## Summary

Keep #39013 open. Current main still has the path-only virtual bootstrap display bug: hook-mutated bootstrap files are only sanitized for a valid `path`, `buildBootstrapInjectionStats` still copies `file.name` directly into report stats, and `/context list` renders that name directly. Current tests cover named bootstrap files only, so a hook file shaped like `{ path, content, virtual: true }` can still surface as `undefined`. The session-memory/accumulation half may need fresh focused reproduction because current main now clears bootstrap snapshots on session rollover, but the issue is not cleanup-ready while the display half remains unfixed.

## Best Possible Solution

Keep the issue open. First fix the stable reporting bug by deriving a safe display label in `buildBootstrapInjectionStats`, for example `file.name`, then basename or normalized `file.path`, then an explicit placeholder, and add regression coverage for path-only virtual bootstrap files injected by `agent:bootstrap` hooks. After that, separately reproduce the session-memory/accumulation path on current main; if hook/bootstrap content is still indexed or duplicated across sessions, exclude injected system/bootstrap content from session-memory sources or split that into a focused follow-up.
