## Summary

Keep PR #39888 open. Current main still does not emit a `model.usage` diagnostic event from the `openclaw agent` CLI command path, while reply-runner does emit that event and diagnostics plugins consume it for token/cost telemetry. The PR is stale because it targets the old `src/commands/agent.ts` implementation, now a re-export, but the underlying behavior gap remains valid and the PR discussion/second commit addressed useful review feedback around `channel: "cli"` and context-token fallback.

## Best Possible Solution

Refresh this PR or replace it with a small current-main change that emits a trusted `model.usage` diagnostic after `updateSessionStoreAfterAgentRun` in the CLI/local agent command path. Include a stable `channel: "cli"`, session/agent/provider/model metadata, usage and last-call usage, context limit/used, cost, duration, and a focused regression test while preserving reply-runner behavior.
