## Summary

Keep PR #39888 open. Current main still lacks `model.usage` diagnostic emission in the `openclaw agent` local/embedded command path, while reply-runner emits that event and diagnostics plugins consume it for OTEL/Prometheus token accounting. The PR itself is stale because `src/commands/agent.ts` is now just a re-export, but the underlying fix remains valid and not superseded.

## Best Possible Solution

Refresh this PR or replace it with a small main-targeted change in `src/agents/agent-command.ts` or a shared agent-run accounting helper. Emit a trusted `model.usage` diagnostic after session-store usage/cost state is updated, include a stable CLI channel label and agent/session/provider/model fields, reuse the current resolved context-token and cost helpers, preserve reply-runner behavior, and add a focused CLI command regression test for diagnostic emission.
