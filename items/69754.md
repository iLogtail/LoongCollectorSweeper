## Summary

Keep #69754 open. Current main has generic `/approve` fallback and Feishu same-chat approval authorization, but it does not yet provide Feishu-specific approval-button delivery or clear Feishu fallback/degradation copy for the built-in approval UX described in the issue.

## Best Possible Solution

Keep the issue open. Implement Feishu-aware built-in approval UX: either add a Feishu approval delivery/rendering path for native cards/buttons, or explicitly degrade to text with copy that says Feishu interactive approval buttons are unavailable for this surface/context, gives the exact `/approve` command, and states where approval can be completed. Update Feishu/exec-approval docs and add focused tests for the Feishu approval fallback path.

## Risks / Open Questions

- Feishu config still accepts a generic `capabilities` array, but current code does not interpret `inlineButtons` as approval-button availability; maintainers should clarify whether that key is legacy, user-facing, or ignored.
- Approval payloads are shared across channels, so the fix should be Feishu-scoped or use the existing channel approval/outbound adapter seams to avoid regressing Slack, Telegram, Discord, and Matrix native approval behavior.
