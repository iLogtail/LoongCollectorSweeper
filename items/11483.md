## Summary

Current `main` does not support per-agent TTS config. `agents.list[]` is still a strict schema without a `tts` field, docs only describe global `messages.tts`, and the speech runtime resolves TTS exclusively from `cfg.messages?.tts`. The issue also has a recent reproduction comment and an open scoped PR (#62573), so conservative maintainer cleanup should keep this issue open.

## Evidence
