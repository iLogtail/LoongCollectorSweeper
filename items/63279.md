## Summary

Keep open. Current main has meaningful fixes around compaction bounds, reset cleanup, config hot reload, restart lifecycle messaging, and subagent announce queuing, but this issue's combined failure chain is not fully closed. In particular, GatewayDrainingError is still handled as a restart lifecycle failure with a "try again" response rather than auto-retry, and subagent docs still describe announce-back work as best-effort and lost across gateway restarts. The remaining behavior is also partly tracked by narrower open items such as #55412 and #62095, but those do not fully subsume this large-session cascade report.

## Best Possible Solution

Keep this issue open for maintainer triage as an umbrella failure-chain report. The best path is to verify current main against a large-session reproduction, then either close it after confirming the cascade no longer occurs or split remaining work into narrower items: GatewayDrainingError retry policy (#55412), durable subagent announce across gateway restarts, and any still-reproducible compaction overflow/reset behavior. The stale fallback-chain portion should be rechecked against the current hot-reload path before adding new work.
