## Summary

Keep PR #71738 open. Current main still has the Matrix gaps the PR targets: room history and watermarks are flat per room, readMessages has no thread-scoped read path, and threaded sends/edits still carry fallback reply metadata that can surface in the main timeline. The PR is not merged or obsolete, but the existing review comments identify real correctness risks that should be addressed before landing.

## Best Possible Solution

Keep the PR open and revise it or land an equivalent Matrix-extension fix: scope room history, watermarks, and message reads by Matrix thread root; avoid fallback reply metadata for thread-only sends/edits; add focused regression tests; and address the review comments around pagination token alignment and stale thread watermark invalidation before merge.
