## Summary

Keep PR #71738 open. Current main still has the Matrix thread isolation and reply-placement gaps described by the PR: thread root IDs are detected for routing, but room history, watermarks, readMessages, and edit/send fallback behavior remain room-flat or main-timeline-leaking. The PR is not obsolete or implemented on main, though it should be revised before merge to address the review comments about pagination and stale thread watermark handling.

## Best Possible Solution

Keep the PR open and ask for a revision that preserves its core direction: add threadRootId-scoped Matrix room-history queues and watermarks, add thread-aware readMessages behavior, and remove unintended Matrix fallback reply metadata. Before merge, address the existing review findings with targeted tests for thread sub-queue eviction/watermark races and pagination correctness.
