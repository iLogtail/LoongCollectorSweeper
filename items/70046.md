## Summary

Keep PR #70046 open. Current main still only applies `--tz` to offset-less ISO datetimes, not bare `HH:MM` / `HH:MM:SS` inputs, and bare `09:00` still falls through to the generic absolute-time parser and is invalid on Node. The PR is the active clean respin for open bug #59441 and supersedes closed noisy PR #59444, so it is not obsolete or superseded.

## Best Possible Solution

Keep #70046 open as the active implementation candidate for #59441. The maintainer path is to review the focused parser change in `src/cli/cron-cli/shared.ts`, keep/add regression coverage for `HH:MM`, `HH:MM:SS`, invalid time ranges, invalid IANA zones, and timezone date resolution, and update help/docs if maintainers want time-only `--at` to become a documented CLI format.
