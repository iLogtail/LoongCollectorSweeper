## Summary

Keep PR #70046 open. Current main still does not support bare `HH:MM` / `HH:MM:SS` in `cron add --at`, and `--tz` is only applied to offset-less ISO datetimes. The supplied discussion also shows this PR is the active clean respin of the older noisy PR #59444 and is linked to the still-open bug #59441, so it is not obsolete or superseded.

## Best Possible Solution

Keep #70046 open for maintainer review or land an equivalent current-main fix. The likely product path is to add time-only parsing in `src/cli/cron-cli/shared.ts`, resolve it against today's date in the supplied IANA timezone or UTC when omitted, reject out-of-range time parts, and add focused parser/CLI regression tests. Help text/docs can be aligned in the already-related help-text track if not included in this PR.
