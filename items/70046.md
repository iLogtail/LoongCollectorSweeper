## Summary

Keep #70046 open. Current main still lacks support for bare HH:MM/HH:MM:SS values in `cron add --at` and does not resolve time-only input against `--tz`; the PR is a focused active respin for the linked bug #59441 and supersedes the noisy closed PR #59444 rather than being superseded itself.

## Best Possible Solution

Keep #70046 open for maintainer review as the active scoped fix for #59441. The likely path is to validate and land the parser/test change, with any remaining help-text/docs wording handled either in this PR or the existing help-text follow-up chain (#59456/#69975).
