## Summary

Keep open. Current main has assistant-message actions for expanding a message and copying markdown, plus code-block copy buttons, but no per-message Print button, isolated print view, window.print path, or print-specific Control UI chat styling. The request is recent, concrete, unprotected, and not covered by the provided related items; #43416 is about native Swift chat copy, not Control UI print.

## Best Possible Solution

Keep the issue open and implement this in the built-in Control UI chat surface: add a print action alongside copy/expand in `ui/src/ui/chat/grouped-render.ts`, factor a small print helper that renders only the target assistant markdown into an isolated print document or print-only DOM, add print styles in the chat stylesheet, and cover it with a focused UI rendering/interaction test. This should stay in core because it changes first-party Control UI message controls, not an optional ClawHub integration.
