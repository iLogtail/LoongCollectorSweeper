## Summary

Keep #69727 open. Current main only exposes the existing Control UI assistant-message Copy as markdown action and optional Expand action; there is no individual-message Print button, print handler, window.print flow, or print-specific styling. The related copy-button item is a different surface and does not supersede this Control UI print request.

## Best Possible Solution

Keep the issue open and implement a focused Control UI feature: add an assistant-only print action beside the existing copy/expand controls in ui/src/ui/chat/grouped-render.ts, likely with a helper next to copy-as-markdown.ts that renders only the target message into an isolated printable document or window, triggers window.print(), and applies white-background, black-text, table/code-friendly print CSS. Add focused UI tests for action visibility and print invocation without changing existing copy behavior.
