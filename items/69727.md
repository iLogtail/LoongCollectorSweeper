## Summary

Keep #69727 open. Current main has the Control UI assistant-message copy affordance and whole-chat markdown export, but it does not implement an individual assistant-message Print button, print-only view, print-friendly styles, or a `window.print()` flow for one message.

## Best Possible Solution

Implement this in the Control UI chat surface by adding an assistant-message print action next to the existing copy action in `ui/src/ui/chat/grouped-render.ts`, backed by a small helper that renders the same sanitized message markdown into a print-only document or isolated window, applies print-friendly CSS for text/code/tables, and triggers `window.print()`. Add focused jsdom tests for action rendering and the print helper, plus update `docs/web/control-ui.md` if the user-facing chat actions list is expanded.
