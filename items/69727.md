## Summary

Keep #69727 open. Current main has an assistant-message copy affordance and optional expand action in the Control UI webchat, but no per-message Print button, print handler, isolated print view, or print-specific styling for assistant messages.

## Best Possible Solution

Implement this in the Control UI webchat message surface: add an assistant-message print action next to the existing expand/copy controls, render only the selected message content into a print-safe view or transient window, call `window.print()`, add clean print styles for markdown/tables/code, and cover the action with focused UI tests. This belongs in the core Control UI surface rather than ClawHub because the request targets built-in webchat chrome and message rendering.
