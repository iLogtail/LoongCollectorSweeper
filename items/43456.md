## Summary

Keep open. Current main still treats Mattermost `oncall` as explicit-mention only: `root_id` is used for threading/session routing, but the mention gate has no reply-to-bot/root-author check and will drop unmentioned channel/thread traffic. The related #45284 context is adjacent outbound threading work, not a duplicate or superseding implementation of inbound reply-wake behavior.

## Evidence

- **Mention gate still requires explicit mention in oncall:** `evaluateMattermostMentionGate` derives `effectiveWasMentioned` only from `wasMentioned`, authorized command bypass, or `oncharTriggered`; when `shouldRequireMention` is true and no mention is detected it returns `dropReason: "missing-mention"`. No implicit `reply_to_bot` or root-author condition is considered.
