## Summary

Keep #43456 open. Current main has Mattermost thread/session routing and the related #45284-style threading/preview work, but it still does not treat a reply to the bot's own Mattermost post as an implicit wake in default oncall mode. The inbound path computes mentions only from raw text/mention patterns plus command/onchar bypasses, and the existing tests still assert that unmentioned oncall channel traffic is dropped.

## Best Possible Solution

Keep the issue open and implement this inside `extensions/mattermost`: before `evaluateMattermostMentionGate`, resolve and cache the reply target/root post (`root_id`, and `parent_id` if Mattermost exposes it in inbound payloads), prove whether it was authored by `botUserId`, and pass that as an implicit `reply_to_bot` mention so default `oncall` wakes while existing sender/group allowlists still run first. Add Mattermost tests for threaded and inline replies to bot-authored posts, plus a negative case for replies to non-bot posts.
