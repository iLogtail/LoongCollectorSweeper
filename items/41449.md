## Summary

Keep open. Current `main` no longer appears to block the configured Ollama host just because it is `localhost` or RFC1918, but the PR's host-pinned embedding SSRF policy is still not implemented on `main`, so there is not strong evidence to close this as already implemented or obsolete.

## Evidence

- **Embedding provider still uses generic base-URL SSRF helper:** The Ollama memory embedding client still sets `ssrfPolicy` with `ssrfPolicyFromHttpBaseUrlAllowedHostname(baseUrl)` and does not use a dedicated Ollama host-scoped policy.
  - file: [extensions/ollama/src/embedding-provider.ts:137](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/extensions/ollama/src/embedding-provider.ts#L137)
  - command: `nl -ba extensions/ollama/src/embedding-provider.ts | sed -n '118,139p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Generic helper does not pin redirects:** `ssrfPolicyFromHttpBaseUrlAllowedHostname` returns only `{ allowedHostnames: [parsed.hostname] }`; it does not add `hostnameAllowlist`.
  - file: [src/infra/net/ssrf.ts:47](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/infra/net/ssrf.ts#L47)
  - command: `nl -ba src/infra/net/ssrf.ts | sed -n '47,58p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Why localhost/private-host access likely no longer reproduces:** SSRF resolution skips private-network checks when the requested hostname is explicitly listed in `allowedHostnames`, so an exact configured Ollama host such as `localhost` or `192.168.x.x` is now allowed.
  - file: [src/infra/net/ssrf.ts:93](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/infra/net/ssrf.ts#L93)
  - command: `nl -ba src/infra/net/ssrf.ts | sed -n '89,97p'; nl -ba src/infra/net/ssrf.ts | sed -n '222,231p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Host-pinning pattern exists elsewhere in Ollama plugin but not embeddings:** Other Ollama surfaces already have `buildOllamaBaseUrlSsrFPolicy`, which returns `hostnameAllowlist: [parsed.hostname]` plus `allowPrivateNetwork: true`, and dedicated tests for localhost/private-network hosts.
