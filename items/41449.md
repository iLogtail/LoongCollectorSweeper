## Summary

Keep PR #41449 open. Current main appears to solve the basic localhost/private-network access failure for configured Ollama embedding hosts, but the PR also requested host-scoped redirect pinning for the Ollama memory embedding path, and that part is still missing on current main. The PR’s old `src/memory/*` files are stale after the Ollama embedding code moved into the bundled plugin, so the right maintainer path is a small port rather than closing it as fully implemented.

## Best Possible Solution

Port the useful part of #41449 to the current bundled Ollama plugin: have `extensions/ollama/src/embedding-provider.ts` use `buildOllamaBaseUrlSsrFPolicy` or an equivalent exact-host policy, then add focused embedding tests for localhost, private-network hosts, and cross-host redirect blocking. Keep the PR open or ask the contributor to rebase/retarget that small change.
