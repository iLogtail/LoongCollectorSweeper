## Summary

Keep open. Current main still has no clear fix for the reported duplicated full JSON object in OpenAI-compatible streamed tool-call arguments: the transport still appends every `toolCall.function.arguments` chunk into `partialArgs`, the malformed-argument repair path only accepts very short non-JSON trailing junk, and isolated cron `agentTurn` jobs still run through the embedded-agent path that uses this streaming/tool-call handling.

## Best Possible Solution

Keep the issue open and fix it in the OpenAI-compatible streamed tool-call argument path. Add a regression test that feeds an `openai-completions` stream with two identical full argument JSON payloads and proves the final tool call receives one object, then apply the same protection through the embedded cron `agentTurn` path used by isolated OpenRouter jobs. The fix should distinguish true delta fragments from repeated snapshot/full-object chunks without weakening malformed-argument safety.
