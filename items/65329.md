## Summary

Keep open. Current main still contains the exact DM fallback gap described in #65329: after direct attachment download returns no media, the Bot Framework fallback is gated on HTML `<attachment>` IDs, and Graph fallback is explicitly skipped for Bot Framework personal-chat IDs. The current docs still promise DM images/file attachments via Teams bot file APIs, so this remains a product bug rather than a cleanup close.

## Best Possible Solution

Keep #65329 open as the focused tracker for the current-main DM media gap. The likely fix belongs in `extensions/msteams/src/monitor-handler/inbound-media.ts`: after direct download returns no media, attempt the Bot Framework attachment download for personal DMs when the activity carries a real non-HTML/downloadable attachment or `msteams://media` placeholder, while preserving the #58617 guard that avoids mention-only HTML 404 noise. Add a regression test for a personal DM inline image/file with no `<attachment id>` HTML wrapper.
