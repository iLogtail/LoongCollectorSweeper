## Summary

Keep PR #39630 open. Current main still does not expose the requested `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only exposes the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`. The linked feature issue #39604 remains open, the PR head is still separate from current main, and the PR discussion contains unresolved product/security policy questions around metadata endpoints and configurable exceptions.

## Best Possible Solution

Keep PR #39630 open for maintainer direction or revision. The best path is to implement a documented `web_fetch` private-network opt-in aligned with the current `ssrfPolicy` surface, thread it through strict schema, public types, generated config metadata, runtime, cache partitioning, tests, docs, and changelog, and explicitly settle the metadata-endpoint policy with #39604 as the canonical feature request.
