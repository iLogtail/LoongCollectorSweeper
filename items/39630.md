## Summary

Keep open. Current main still does not expose the requested `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only exposes the narrower RFC2544 exception under `tools.web.fetch.ssrfPolicy`. The related feature issue #39604 remains open, and the PR discussion still has unresolved security/product policy questions around cloud metadata endpoint handling, so this PR is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open for maintainer direction or revision. A strong path would add a documented `web_fetch` private-network opt-in aligned with the current `ssrfPolicy` config surface, thread it through schema, generated config metadata, types, runtime, cache partitioning, tests, docs, and changelog, and explicitly resolve metadata endpoint policy before merge. Continue treating #39604 as the canonical feature request until the product decision is made.
