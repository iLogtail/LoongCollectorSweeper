## Summary

Keep open. Current main has a partial macOS app and release path, including DMG packaging and Sparkle zip updates, but the requested single click-and-run desktop installer story is not fully implemented: Windows has no companion app/installer, and the macOS app currently expects an external OpenClaw CLI/Node runtime rather than bundling the Gateway into a no-terminal installer.

## Best Possible Solution

Keep the issue open as a product/release tracking item. The best path is to split the work into clear maintainer-owned milestones: verify and document the existing macOS DMG/zip release flow, decide whether the macOS app should intentionally keep using an external CLI runtime or move toward a bundled runtime, and track a Windows companion app plus .exe/auto-update release pipeline separately. If maintainers decide bundled runtime is out of scope, update this issue title/body or create narrower follow-ups so the remaining Windows/no-terminal installer gap is explicit.
