## Summary

Keep open. Current main has ordinary model fallback and same-model overflow compaction, but it does not implement the requested configurable per-candidate retry policy or context-aware safe/strict fallback mode. The related PR #60984 was closed unmerged and explicitly redirected the broader resilient fallback work to #65502, so this issue remains the canonical tracker rather than a duplicate close candidate.

## Best Possible Solution

Keep #65502 open as the canonical feature issue. A good implementation would add a bounded fallback policy under the existing agent model config surface, classify retryable failures consistently with current failover reasons, thread retry/backoff through `runWithModelFallback`, and define safe vs strict behavior for smaller-context fallback candidates using the existing overflow compaction/truncation infrastructure. Add focused tests for timeout/rate-limit retry counts, non-retryable errors, and primary-to-smaller-fallback context overflow recovery.

## Risks / Open Questions

- Retry behavior must be designed around existing auth-profile rotation and cooldown budgets so transient retries do not multiply long stalls.
- Context-aware fallback needs coverage for both thrown overflow errors and embedded-runner error payloads, especially when a fallback model has a smaller context window than the primary.
