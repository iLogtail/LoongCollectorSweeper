## Summary

Keep #69222 open. Current main has only the narrow `/api/web_search` update and `https://ollama.com` base-URL honoring, but it does not implement the PR's main remaining behavior: local endpoint fallback, env-key-only Ollama Cloud fallback after local 404s, or separated local/cloud auth scoping. The linked bug #69132 also remains open in the provided context, and the duplicate PR #69178 was closed unmerged rather than landing the fallback work.

## Best Possible Solution

Keep #69222 open and ask for a rebase/narrowed update against current main. The best path is either to land a revised Ollama web-search compatibility fix that preserves current `/api/web_search` behavior while adding the missing local fallback, env-scoped cloud fallback, docs, and regression tests, or to make an explicit maintainer product decision on #69132 if cloud fallback should not be supported. #69178 should be treated only as a closed duplicate attempt, not as the canonical shipped fix.
