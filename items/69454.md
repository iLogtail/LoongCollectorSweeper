## Summary

Keep #69454 open. Current main does not already implement the PR’s central Control UI churn reductions: it still eagerly loads node/device data on gateway hello, starts node polling globally, reloads sessions immediately on every `sessions.changed`, keeps unbounded chat/session option refreshes, and serves Control UI index HTML with `Cache-Control: no-cache` rather than `no-store`. The PR also has a concrete review finding about guarding deferred session reloads across tab changes, so the best path is to update/fix the PR rather than close it as obsolete.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The useful direction is to land the same narrow Control UI performance hardening, but with the bot review fixed: bound `loadSessions` refreshes, scope node polling and node/device preloads to the Nodes flow, debounce/suppress `sessions.changed` reloads safely across tab switches, serve Control UI `index.html` with `no-store`, and keep the focused UI/gateway regression tests.
