## Summary

Keep open. Current main still uses a single JSON session store path guarded by a storePath-level lock, and there is no directory-backed/per-session store implementation in the checked-out tree. The linked open PR #51946 is relevant tracking work, but the requested capability remains missing on main and the issue has concrete recent reproductions, so it should stay open until an implementation lands.
