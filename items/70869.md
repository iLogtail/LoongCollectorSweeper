## Summary

Keep open. Current main has an important replay-time mitigation: OpenClaw drops `provider:"openclaw", model:"delivery-mirror"` assistant rows from in-memory model replay, so core provider calls should not replay those rows to Anthropic. But the requested core behavior is not implemented: the message tool still constructs a mirror for an outbound route, successful delivery still appends that mirror as a `role:"assistant"` `delivery-mirror` transcript row, there is no delivery-mirror config toggle, and current tests explicitly allow appending a new delivery mirror after a different latest assistant message.

## Best Possible Solution

Keep the issue open and fix the remaining core transcript behavior deliberately. The likely maintainer path is to add a narrow outbound mirror policy around `appendAssistantMessageToSessionTranscript` or the mirror construction sites: either expose a documented `session.deliveryMirror` opt-out, or skip/degrade mirror writes that would create adjacent assistant-only transcript turns, especially for cron/isolated/subagent-origin sends. Keep the replay filter, add regression coverage for message-tool outbound delivery into a session whose latest row is assistant, and document the transcript-only `delivery-mirror` contract for plugins such as LCM.
