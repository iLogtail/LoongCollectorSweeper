## Summary

Keep open. Current main still has only partial human sender-label support; it does not carry or render structured main-agent versus sub-agent identity for Chat UI messages. The related PRs in the provided context are closed and unmerged, and the latest discussion reinforces that durable sender identity/auditability is still the unsolved user problem.

## Best Possible Solution

Keep #22774 open. The best path is a core-backed attribution contract that carries durable sender identity through sessions.send, sessions.steer, and sub-agent announce/agent delivery into transcript/history events, then renders distinct human/main-agent/sub-agent labels and avatars in the Chat UI. The design should include persistent sender id/kind/source session data, not only colors, and regression tests for main-to-subagent steer and subagent-to-main completion messages.
