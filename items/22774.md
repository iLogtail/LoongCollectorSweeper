## Summary

Keep #22774 open. Current main has partial `senderLabel` display and inter-session provenance, but it still does not provide a durable sender attribution contract or Chat UI rendering that distinguishes human, main-agent, and sub-agent identities across `sessions.send`, `sessions.steer`, `chat.send`, and sub-agent announce flows. The issue is also actively discussed, and the related PRs #23336 and #36006 are closed unmerged rather than shipped fixes.

## Best Possible Solution

Keep #22774 open. The best path is a core-backed sender attribution contract that carries durable sender id/kind/name/source session through `sessions.send`, `sessions.steer`, `chat.send`, agent input provenance, and sub-agent announce delivery into persisted transcript/history events, then updates Chat UI grouping/avatar rendering to distinguish human, main-agent, and sub-agent messages. Add regression coverage for main-to-sub-agent steer and sub-agent-to-main completion/announce flows.
