## Summary

Keep open. Current `main` still gates steering on `isStreaming`, rejects embedded queued messages when the run is not streaming, and keeps `collect` as the default queue mode. The docs also still describe steer delivery only after the current assistant turn finishes its tool calls. That matches the reported interrupt/steer failure during tool-execution gaps rather than showing a fix on `main`.
