## Summary

Keep PR #71354 open. Current main still skips `getTailnetHostname` whenever `gateway.tailscale.mode` resolves to `off`, so the linked status bug is not implemented on main. The display and docs already support distinguishing OpenClaw Tailscale exposure from the local Tailscale daemon, and the PR addresses that gap while also following up on the cold-start review feedback. It still needs maintainer review, regression coverage, and a privacy/security decision before merge.

## Best Possible Solution

Keep the PR open. The best path is a focused status-scan fix that preserves `skipColdStartNetworkChecks`, adds a regression test for `gateway.tailscale.mode: "off"` on non-cold-start scans plus a cold-start skip test, and resolves the privacy/PATH concerns either by explicitly accepting the behavior or by gating/redacting the hostname and using a trusted binary-resolution path.
