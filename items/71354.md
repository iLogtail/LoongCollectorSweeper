## Summary

Keep PR #71354 open. Current main still has the reported bootstrap behavior: when `gateway.tailscale.mode` resolves to `off`, `createStatusScanCoreBootstrap` returns `null` without calling `getTailnetHostname`, so the existing display path that can render `off Â· <dns>` never receives runtime DNS data. The PR is a focused core status fix, and the author already addressed the cold-start regression feedback in the follow-up commit described in the PR context. It still needs maintainer review for the remaining security/product tradeoff and ideally a regression test, but it is not obsolete on current main.

## Best Possible Solution

Keep the PR open and ask for maintainer review or a small test update: the likely best path is to land the focused bootstrap change, preserve `skipColdStartNetworkChecks`, and add coverage proving `getTailnetHostname` runs when mode is `off` only on non-cold-start status scans. If maintainers agree with the security concern, refine the implementation to use a trusted resolved Tailscale binary path or gate hostname disclosure explicitly rather than closing the PR as obsolete.
