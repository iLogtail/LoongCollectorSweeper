## Summary

Keep PR #71354 open. Current main still short-circuits Tailscale hostname lookup when `gateway.tailscale.mode` is `off`, so `openclaw status` cannot populate the already-supported `off Â· <tailnet host>` display path. The PR is a focused one-file fix and the author addressed the cold-start regression review feedback, but it still needs maintainer judgment on the privacy/PATH tradeoff and targeted bootstrap regression coverage.

## Best Possible Solution

Keep the PR open. The best path is to merge or adapt this focused bootstrap change after adding targeted regression tests for mode-off runtime detection and cold-start skipping, and after a maintainer explicitly accepts or revises the privacy/PATH behavior raised in review.
