## Summary

Keep open. Current main still makes `--agent` optional for `openclaw cron add`, emits no missing-agent warning, and isolated agent-turn cron runs without a job agent fall back to the configured default agent. The related PR #42245 is an exact attempted fix, but it is still open, unmerged, and dirty against current main.

## Best Possible Solution

Keep the issue open until main lands a small cron CLI UX fix. Prefer a warning in `src/cli/cron-cli/register.cron-add.ts` after payload and agent normalization, only for `agentTurn` jobs where no valid `--agent` was supplied, emitted via `defaultRuntime.error` so JSON stdout remains parseable. Add focused tests in `src/cli/cron-cli.test.ts` for warning without `--agent`, no warning for `--system-event`, warning with `--session-key` but no agent, and no warning with explicit `--agent`. Rebase or supersede PR #42245 to current main rather than closing this issue now.
