## Summary

Current main still matches the reported Control UI live-render race: own-run `final` and `aborted` events append to `chatMessages` before clearing `chatStream`/run state. The open PR #72004 appears to target the exact reorder, but it is not merged into the reviewed main SHA. The issue also has a current comment describing a possible server-side transcript duplicate variant, which overlaps the maintainer-owned #69208 umbrella and should not be auto-closed from this review.

## Best Possible Solution

Keep #71992 open until #72004 or an equivalent main-branch patch lands. The best fix is to reorder both own-run `final` and `aborted` handlers so `chatStream`, `chatRunId`, and `chatStreamStartedAt` are cleared before any `chatMessages` mutation, add a regression test that observes that intermediate ordering, and separately coordinate the server-side duplicate transcript evidence with the maintainer-owned #69208 umbrella.
