## Summary

Keep open. Current main has moved GitHub Copilot token exchange into the bundled provider plugin/runtime-auth path, but it still unconditionally applies the token-derived baseUrl to the runtime and effective models. That means the PR’s central request, preserving a user-configured Copilot baseUrl such as a business endpoint or corporate proxy, is not implemented on main. The PR concept remains useful, but the patch needs to be ported to the current pluginized architecture and address the existing review concerns.

## Best Possible Solution

Keep the PR open as a valid bug report/contribution, but ask for a current-main rewrite in the GitHub Copilot provider/runtime-auth path. The best implementation should preserve explicit `models.providers.github-copilot.baseUrl` values, still apply token-derived Business/Enterprise endpoints when the model is unset/default/runtime-derived, update both runtime and effective models, and add regression coverage for initial auth, refresh, context-window effectiveModel copies, and profile rotation.

## Risks / Open Questions
