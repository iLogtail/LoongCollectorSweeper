## Summary

Keep PR #39412 open. Current main already covers the daemon gateway-token persistence portion with the newer SecretRef-aware install flow, but the provider `*_FILE` env-var behavior proposed by the PR is still absent from main. The branch also has unresolved review findings, so the useful maintainer path is a focused rebase/rewrite rather than closure or direct merge.

## Best Possible Solution

Keep the PR open as useful stale work. Rebase or rewrite it against current main, keep only the still-missing provider `*_FILE` behavior if maintainers want that env convention, and adapt it to current provider auth and SecretRef architecture. Do not reintroduce the old gateway-token persistence changes wholesale because main already has the safer SecretRef-aware install flow and coverage.
