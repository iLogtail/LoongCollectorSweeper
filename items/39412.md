## Summary

Keep open. Current main already covers the daemon gateway-token persistence goal through the newer SecretRef-aware install flow, but it still does not implement the PR's file-backed provider auth env vars. In particular, provider auth resolution reads direct env values only, and live key collection would still treat `OPENAI_API_KEY_FILE` as a prefixed env value rather than reading the referenced file. The PR branch also has unresolved review feedback, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep the PR open or replace it with a focused current-main patch for provider file-backed env auth. The durable fix should add a shared resolver for provider auth env vars, wire it through `resolveEnvApiKey()`, `collectProviderApiKeys()`, and usage auth paths, preserve list separators for `*_API_KEYS_FILE`, ignore `_FILE` entries during prefixed scans, surface unreadable file misconfiguration clearly, and leave the already-shipped gateway token persistence flow intact.
