## Close Comment

Codex review: this is already fixed on `main`, and it shipped in `v2026.3.28`. The changelog explicitly calls out preserving leading indentation when stripping inline reply tags at `CHANGELOG.md:2013`. Current outbound reply parsing still routes through `parseInlineDirectives(...)` in `src/auto-reply/reply/reply-directives.ts:23`, and the implementation in `src/utils/directive-tags.ts:41` preserves fenced and indented blocks during whitespace normalization. There is also regression coverage for both plain indented text and fenced code blocks in `src/utils/directive-tags.test.ts:56` and `src/utils/directive-tags.test.ts:63`. Since the report was against `2026.3.8`, this looks resolved in later releases, so I’m closing it as fixed. If you still see indentation flattening on `2026.3.28+` or current `main`, please open a new issue with the exact current version and a fresh repro path.
