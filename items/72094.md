## Summary

Keep #72094 open. Current main has Codex app-server and ACP/Codex routing, and PeekabooBridge covers macOS UI automation through Peekaboo, but I found no implemented `codex-desktop` desktop-control provider, `computerUse`/`desktopProvider` spawn option, feature probe, or channel-safe routing path for Codex Desktop native Computer Use. The provided discussion also frames this as the narrower follow-up to #67776 rather than a duplicate of the PeekabooBridge implementation.

## Best Possible Solution

Keep the issue open as a focused product/security design item for the bundled Codex/app-server path. The best implementation path is to first verify the supported Codex Desktop/app-server API for native Computer Use, then add an explicit `codex-desktop` desktop-control route or provider with availability probing, permission policy, channel allowlists, result/artifact return handling, and tests/docs. If current plugin APIs are insufficient, add a narrow plugin SDK/design follow-up rather than closing this as ClawHub-only.
