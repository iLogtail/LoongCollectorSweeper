## Summary

Keep #14133 open. Current main still has the Cron Jobs Control UI surface only, with no Reminders tab/route and no cron job classification/display metadata contract that reminder-created one-shot jobs could use for filtering, human-readable reminder text, or a dedicated card view. The existing maintainer-side Codex comment on the issue also explicitly kept it open, and current source inspection confirms that remains true at 9a529ca78bfa0e257f662f0e70ab8c681caa6086.

## Best Possible Solution

Keep this open as a core Control UI plus cron protocol enhancement. The best implementation is an additive cron classification/display metadata contract, reminder-producing paths that set it, typed list filtering, Cron tab filtering/hiding for reminder jobs, and a Reminders tab with human-readable reminder text, fire time, channel/recipient, and delete actions, backed by UI/protocol/service tests and docs updates.
