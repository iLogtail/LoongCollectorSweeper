## Summary

Keep #61145 open. Current main still defaults both TUI and browser WebChat/Control UI chat to the configured main session, and `gateway.webchat.defaultPeerId` is not present in current config types, Zod schema, docs, or UI/TUI default-session wiring. The PR is a plausible focused feature with concrete discussion and an author follow-up to the bot review, but it is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase plus explicit coverage of the claimed surfaces. The best implementation path is to add a documented optional `gateway.webchat.defaultPeerId` config, resolve empty/default TUI and browser WebChat session selection through `buildAgentPeerSessionKey` with `session.dmScope` and `session.identityLinks`, preserve legacy main-session behavior when unset, and add focused tests for TUI default selection, Control UI/WebChat default selection, schema acceptance, and legacy fallback.
