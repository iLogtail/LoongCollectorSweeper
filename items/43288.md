## Summary

Keep open. Current main still accepts arbitrary non-empty CLI session_id values, including the reported "rate-limited" sentinel, and the Claude CLI backend still has resume wiring that can reuse such a value. The related fix PRs in the provided GitHub context were closed without merging and do not supersede the issue.
