## Summary

Current main does not implement the PR's secret-redaction path for Control UI tool events. `sanitizeToolResult()` still only truncates text and strips image data, tool-start events still emit raw `args`, and derived exec/patch events still read from the unsanitized result. The linked security issue remains a real core bug path rather than maintainer cleanup.

## Best Possible Solution

Keep the PR open and review it as a plausible core security fix. The best path is to land a maintainer-approved variant that redacts sensitive text at the agent-event producer boundary for tool start args, partial/final tool result content, result details used by exec/patch event derivation, and matching regression tests for the Control UI event path.
