## Summary

Keep PR #72319 open. Current main still sends full verbose tool result text through `sanitizeToolResult()` without secret redaction, and the Control UI can render that payload when tool verbosity is `full`. The PR is a focused fix for the linked open security bug #72283, though it should address the review gap about user-configured `redactPatterns` before landing.

## Best Possible Solution

Keep the PR open and ask for a small revision: redact text blocks in `sanitizeToolResult()` before truncation, preserve image stripping behavior, add regression tests for JSON/env/bearer/common-prefix tokens and benign passthrough, and resolve the configured-patterns gap from the review. This is a core agent/Control UI security fix, not ClawHub or stale cleanup.
