## Summary

Keep open. Current `main` has a partial solution: an opt-in bundled `session-memory` hook saves recent `/new` or `/reset` context into `memory/*.md`, and startup-context code reloads recent memory files on fresh sessions. But it does not implement the issue’s requested full transcript export plus synthesized, linked cross-session memory package, and related follow-up work is still open.

## Evidence

- **Bundled hook saves reset context, but only as a bounded memory note:** `saveSessionToMemory` runs on `command:new` and `command:reset`, reads a configurable recent-message window (default 15), generates a slug, and writes a single Markdown note into `<workspace>/memory/YYYY-MM-DD-slug.md`. This is preservation, but not full-transcript export or linked synthesis.

## Risks / Open Questions

- Current main partially satisfies the request, so future triage may need to retitle or split this issue to isolate the remaining synthesis/linking gap.
- Closing as implemented would hide that the shipped behavior is opt-in and materially narrower than the requested full-transcript-plus-linked-summary workflow.
- There is active follow-up work and an open bug in the same feature area, so the evidence is not strong enough for conservative cleanup.
