## Summary

Keep this issue open. Current main still lacks the central requested capability: a native browser-based, no-third-party E2EE chat transport with per-device/per-agent chat keys and isolation. Existing WebChat and Control UI chat are gateway-backed browser chat surfaces, and the current WebCrypto/device identity work secures device identity for the Control UI rather than encrypting chat end to end. The related prior ClawSweeper review reached the same conclusion, and the newest maintainer-associated automated comment explicitly kept the item open.

## Best Possible Solution

Keep tracking this as a product/security architecture request until a maintainer decides the direction or an implementation lands. The likely path is a concrete design for browser chat transport, key lifecycle, pairing, per-agent isolation, fallback/relay behavior, gateway/control UI changes, and tests/docs showing that existing third-party channel account sharing is not part of this native browser path.
