## Summary

Current `main` still does not implement the PR's requested streaming text-repetition guard. The shipped loop detection remains scoped to tool calls, assistant text streaming/final-flush paths do not abort or suppress repeated text output, and config/schema/docs surfaces do not expose a `textRepetitionGuard` setting. The PR may need rebase and maintainer review, but it is not obsolete cleanup material.

## Best Possible Solution

Keep #39961 open for maintainer engineering review. The best path is to either land a robust streaming text-repetition guard in core with config/schema/docs/tests covering abort, queued-event, chunked, non-chunked, and fallback flush behavior, or have a maintainer explicitly decide OpenClaw should not add this core guard.
