## Summary

Keep #39961 open. Current main still does not implement the PR's streaming text-repetition guard, and the existing guardrail remains scoped to tool-call loops. The PR contains unique unmerged work and useful contributor follow-up, but the streaming abort/final-flush lifecycle is subtle enough that it needs maintainer engineering review rather than conservative cleanup closure.

## Best Possible Solution

Keep #39961 open for maintainer engineering review. The best path is to land a robust core streaming text-repetition guard with config/schema/docs/tests covering abort, queued-event, chunked, non-chunked, final-flush, and lifecycle fallback behavior, or have a maintainer explicitly decide OpenClaw should not add this core guard.
