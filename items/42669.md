## Summary

Keep #42669 open. Current main still lacks `skills.priority`, still validates the strict `skills` config without that field, and still sorts prompt-facing skills alphabetically before applying count/character truncation. The PR is not obsolete, but it should be narrowed before merge because the latest branch includes unrelated Slack/plugin-SDK seam work beyond the stated skills prompt-ordering feature.

## Best Possible Solution

Keep the item open, but ask for a focused skills-only revision or replacement PR: add `skills.priority` to config types/schema/generated docs, order eligible prompt entries before truncation while preserving current behavior when unset, honor `skillKey` aliases without overriding exact names, use deterministic sorting for non-priority skills, invalidate/rebuild cached snapshots when prompt-affecting skills config changes, and cover count/char truncation plus alias behavior in tests.
