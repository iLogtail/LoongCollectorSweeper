## Summary

Keep open. Current main supports operator-configured Docker sandbox resource limits globally and per agent, but it still does not provide default memory, CPU, or PID limits when those fields are absent. The related PID-limit item/PR covers only part of this request and has not made the broader default resource policy obsolete.

## Best Possible Solution

Keep the issue open for a focused core sandbox change: add documented default Docker resource limits in the sandbox config resolver, preserve explicit global and per-agent overrides, add regression coverage for unset/global/agent/shared-scope behavior and docker create args, and update the public sandbox/config docs to state the defaults and override path.
