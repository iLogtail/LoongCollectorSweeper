## Summary

Keep #12756 open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still has the shared `read` action, but the bundled Telegram plugin does not advertise, gate, dispatch, document, or implement Telegram message reads. The related implementation PR #14367 was closed unmerged, so it is useful prior art rather than a shipped fix.

## Best Possible Solution

Keep #12756 open and implement the feature in the bundled Telegram plugin. The best path is to add Telegram `read` discovery/action mapping, config gating, runtime handling backed by a bounded inbound message store, CLI `--target` compatibility, docs, and focused tests. PR #14367 is useful prior art, but a new or revised change needs to wire the shared message action surface end-to-end.
