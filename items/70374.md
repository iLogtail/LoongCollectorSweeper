## Summary

Keep open. Current main still does not implement the PR's requested Telegram `message.send` gateway routing: the send/poll outbound service dispatches locally before falling back to core send, while gateway `message.action` execution is only wired for non-send/poll plugin actions. The PR is a plausible focused fix, but its reviewed fallback and gateway-routing semantics need maintainer/security handling before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best code direction is to route Telegram `send` through the existing gateway action seam only with clear failure semantics, reuse the existing gateway URL/policy safeguards from the core send path, avoid duplicate sends on ambiguous gateway failures, and keep focused regression coverage for gateway and local fallback behavior.
