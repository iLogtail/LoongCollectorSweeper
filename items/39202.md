## Summary

PR #39202 is superseded by the shipped tokenjuice path on current main: OpenClaw now has an opt-in bundled plugin and documented tool-result middleware seam for compacting noisy exec/bash output without rewriting the approved command. The RTK-specific core rewrite remains absent and still carries the unresolved approval-plan trust concern raised in review, so this PR should close in favor of the shipped tokenjuice/plugin direction rather than land as-is.

## Best Possible Solution

Close this PR as superseded by #69946/current tokenjuice support. Keep exec-output compaction on the documented tool-result middleware/plugin path, where output can be reduced without letting an external binary replace the command that passed approval. Any future RTK-specific integration should come as a fresh security-reviewed design rather than landing this core rewrite PR.
