## Summary

Keep open. The issue is recent, non-maintainer-authored, and has concrete cross-surface reproduction details. Current main still unconditionally sends `stream_options.include_usage` on every OpenAI-compatible streaming request, while the existing compat model says generic custom providers and native Google/Gemini endpoints are not automatically streaming-usage compatible. I did not find a current-main fix, release note, or canonical open duplicate that resolves the reported Gemini/OpenAI-compatible streaming regression.

## Best Possible Solution

Keep this issue open and make the OpenAI-compatible completions payload respect the existing streaming-usage compatibility contract, or add a narrowly documented opt-out/override for providers that complete early when `stream_options.include_usage` is present. Add regression coverage for a custom or Google/OpenAI-compatible Gemini-style backend that still streams incremental `delta.content` while preserving usage chunks for providers that explicitly support them.
