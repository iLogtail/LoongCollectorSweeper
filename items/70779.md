## Summary

Keep open. Current `main` still has a plausible gap for generic OpenAI-compatible Gemini streaming: incremental text is emitted only from `delta.content` or explicitly-whitelisted `reasoning_details` types, and that whitelist is only enabled for OpenRouter-like routes. The Apr 25 follow-up on `main` only recovers the final answer after blank streamed text; it does not restore incremental streaming updates.

## Evidence

- **Generic OpenAI-compatible routes do not opt into visible `reasoning_details` text:** `src/agents/openai-completions-compat.ts` only sets `visibleReasoningDetailTypes` to `['response.output_text', 'response.text']` for OpenRouter-like routes; other OpenAI-compatible routes get an empty list, so visible text carried only in `reasoning_details` is not surfaced incrementally.
  - file: [src/agents/openai-completions-compat.ts:107](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/src/agents/openai-completions-compat.ts#L107)
  - command: `nl -ba src/agents/openai-completions-compat.ts | sed -n '100,125p'`
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Streaming parser only emits text from `delta.content` or whitelisted `reasoning_details`:** `src/agents/openai-transport-stream.ts` first streams `choice.delta.content`; otherwise it parses `reasoning_details` and only promotes entries whose `type` is in `compat.visibleReasoningDetailTypes`. Non-whitelisted visible text there is ignored during streaming.
  - file: [src/agents/openai-transport-stream.ts:1335](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/src/agents/openai-transport-stream.ts#L1335)

- **Tests explicitly limit this behavior to compat-opted providers:** `src/agents/openai-transport-stream.test.ts` has one test that surfaces visible `reasoning_details` text for OpenRouter (`line 2859`) and a separate test asserting ambiguous `reasoning_details` text is not surfaced without explicit compat opt-in (`line 2952`). That matches the reported symptom for a generic custom backend more than a resolved fix.
