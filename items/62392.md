## Summary

Keep PR #62392 open. Current main still lacks the MIME-extension fallback the PR proposes: dropped or selected files with an empty browser MIME type are still rejected before attachment creation. The linked bug #62371 remains plausibly unfixed on main, and the PR discussion contains useful follow-up review that was addressed in the PR head but is not present in the checked-out main code.
