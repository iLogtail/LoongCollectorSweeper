## Summary

Current `main` still loads local skills from `~/.openclaw/skills` for runtime visibility, but the searchable skill surface remains ClawHub-only. The issue's requested behavior is therefore not implemented on `main` and should stay open.

## Evidence

- **Local skills are already auto-loaded from managed storage:** `loadWorkspaceSkillEntries` resolves `managedSkillsDir` to `CONFIG_DIR/skills` and loads it as `openclaw-managed`, which is the `~/.openclaw/skills` shared skill root documented elsewhere.
