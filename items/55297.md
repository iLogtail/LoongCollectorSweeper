## Summary

Keep open. Current main has explicit manual update paths for ClawHub skills and plugins, and the core package auto-updater can run plugin update sync after an OpenClaw update, but there is no native opt-in background updater that independently checks and updates installed ClawHub skills/plugins on gateway start or interval. The remaining request is a real product/security design item rather than a duplicate, shipped fix, or ClawHub-only external plugin idea.

## Best Possible Solution

Keep the issue open and design a native opt-in updater for tracked ClawHub skills and plugin install records. A conservative path is to extend the existing `update.auto` scheduler with explicit `plugins` and `skills` policy, reuse `updateNpmInstalledPlugins` and `updateSkillsFromClawHub`, document pinning/interval/default-off behavior, and add tests for gateway scheduling, no-core-update checks, pinned/exact versions, integrity failures, and skill lockfile updates.

## Risks / Open Questions

- Background updates for executable plugins and agent skills have supply-chain risk; the design should define pinning, integrity/signing, rollback, audit logging, and break-glass behavior before enabling unattended writes.
- There is already partial plugin update behavior inside `openclaw update`; adding independent skill/plugin auto-update should avoid confusing overlap, duplicate writes, or silent behavior changes for pinned installs.
