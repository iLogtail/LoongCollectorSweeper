## Summary

Keep open. Current main still only suppresses exact duplicate final text after Feishu streaming-card closeout; distinct late final payloads can still start another streaming card. PR #72294 is a focused, recent fix attempt for the concrete #71977 reproduction, and the remaining question is maintainer/product judgment about whether dropping distinct late final text is acceptable.

## Best Possible Solution

Keep PR #72294 open for maintainer review. The best path is to decide the Feishu product behavior explicitly: either accept the success-only closed-for-reply suppression with the PR’s media/error regression tests, or change the fix so distinct late finals update the existing card or fall back to a non-streaming follow-up instead of being dropped. After that, validate with the Feishu dispatcher test and the changed gate before landing.
