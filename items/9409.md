## Summary

Keep #9409 open. Current main still surfaces generic user-visible context-overflow text; diagnostic data is logged or carried in metadata, but the requested channel/session label, context size, model limit, and overage are not included in the user error message. The related diagnostic PR #15749 was closed unmerged, and the smaller-context model-switch items do not supersede this request.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, and computed overage, then wires it through runEmbeddedPiAgent, formatAssistantErrorText, sanitize/fallback delivery paths, and focused regression tests.
