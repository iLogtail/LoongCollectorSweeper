## Summary

Keep #9409 open. Current main still does not surface the requested context overflow specifics in user-visible messages. The runner and precheck paths compute/log useful diagnostics, but the delivered payloads, shared formatter, sanitizer, and auto-reply fallbacks remain generic. The related PR #15749 attempted this feature but was closed unmerged, and the later related context hardening/model-switch work does not supersede this diagnostic-message request.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, computed overage, and concise recovery actions, then wires it through the embedded runner, formatAssistantErrorText, sanitizer/normalization paths, and auto-reply fallback paths with focused regression tests.
