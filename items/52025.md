## Summary

Keep open. Current `main` still does not inline-render tool-result image blocks shaped as `{ type: "image", data, mimeType }`, still hides tool-message images behind the collapsed tool-output disclosure, and still lacks the nested `toolResult` text handling called out in review. The PR may be superseded by broader follow-up work, but the underlying core bug is not implemented on `main` or in the latest shipped release snapshot I inspected.
