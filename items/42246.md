## Summary

Keep #42246 open. Current main has Telegram text chunking, preview/edit controls, and retry-after-aware retry for Telegram API errors, but it does not implement the requested configurable outbound batching/digest layer with interval, max-message, grouping, or shutdown flush behavior. The linked #69165 remains complementary rate-limit queueing work, not a superseding tracker for digest aggregation.

## Evidence

- **Telegram outbound sends deliver immediately per payload/chunk:** sendMessageTelegram resolves the target, builds thread params, then sends each text chunk in a loop through api.sendMessage and records the sent message. There is no batching queue, digest formatter, interval flush, or grouping strategy in this path.
