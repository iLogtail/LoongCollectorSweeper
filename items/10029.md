## Summary

Keep #10029 open. Current main still renders WebChat fenced and indented code as escaped monospace HTML with language labels, copy-button chrome, and JSON collapse behavior, but no syntax highlighting is wired into the WebChat markdown path. The related copy-button work is present, but it does not satisfy this syntax-highlighting request.

## Best Possible Solution

Keep #10029 open until WebChat implements real syntax highlighting in `ui/src/ui/markdown.ts` using an approved highlighter or markdown-it highlight integration, adds Control UI theme CSS, and includes focused tests that prove highlighted fenced code while preserving the current language label, copy button, and JSON collapse behavior.
