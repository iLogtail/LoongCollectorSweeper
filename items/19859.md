## Summary

Current main does not implement the requested WebChat option. The browser Control UI chat composer still hardcodes Enter to send and Shift+Enter for newline, and the native macOS WebChat composer has the same fixed behavior. The public/config schema and WebChat docs expose only chat history sizing under gateway.webchat, not send-on-enter behavior. This remains a valid core WebChat/UI enhancement rather than a cleanup close.
