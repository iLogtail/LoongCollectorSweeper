## Summary

Current main still lacks the requested WebChat/Control UI Enter-key behavior option. The browser composer still sends on unmodified Enter, persisted UI settings and gateway webchat config/schema do not expose a send-on-enter or send-shortcut preference, the macOS Swift WebChat composer still sends on Return with Shift-Return for newline, and the public WebChat docs only list chatHistoryMaxChars. The overlapping #8147 tracker remains open, and the latest #19859 discussion explicitly kept this broader issue open pending maintainer consolidation.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should choose the product surface, add a typed chat send-shortcut/send-on-enter preference, wire it through the browser Control UI/WebChat composer and native Swift WebChat composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types/help, settings UI, public docs, and focused composer tests.
