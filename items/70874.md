## Summary

Keep open. On current `main` (`bf0221c5b3de3f290a934a314814426c05b59b84`), `/tools` still reads from `resolveEffectiveToolInventory()`, which only inventories `createOpenClawCodingTools()`. The actual agent runtime appends bundled MCP tools later via `materializeBundleMcpToolsForRun()` / `createBundleMcpToolRuntime()`, so the code still matches the reported mismatch: runtime can see MCP tools, but `/tools` can omit them.
