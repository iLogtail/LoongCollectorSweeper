## Summary

Keep open. Current `main` still passes `job.payload.model` straight into `truncate()` in `printCronList()`, so any object-valued model will still throw `TypeError: value.slice is not a function`. The same code is present in the latest release `v2026.4.22`, and the existing tests only cover string-valued models.

## Evidence

- **Current main still uses payload.model as a string in cron list:** `printCronList()` builds `modelLabel` with `truncate((job.payload.kind === "agentTurn" ? job.payload.model : undefined) ?? "-", CRON_MODEL_PAD)`, with no string coercion or object handling first.
