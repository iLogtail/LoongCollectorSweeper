## Summary

Keep #70508 open. Current main still matches the reported Mattermost bug: streamed partials update one draft post, assistant/reasoning boundaries only clear `lastPartialText`, and tool starts write status text into the same draft stream without rotating the post. The PR is focused on a real channel bug and is not implemented or superseded on main, but its current head has unresolved review concerns around async callback ordering and missing regression tests, so the best maintainer action is request fixes rather than auto-close.

## Best Possible Solution

Keep the PR open and ask for a tightened Mattermost fix: preserve already-streamed assistant text by flushing then rotating the draft post on assistant/reasoning/tool boundaries, decide explicitly whether tool-status posts are transient or persistent, guard the known item-event/onToolStart ordering races, and add Mattermost regression tests for assistant-message start, reasoning end, tool start, and item-event ordering.
