## Summary

Keep #70508 open. Current main still contains the Mattermost draft-stream overwrite path described by the PR: assistant/reasoning boundaries only clear local text state, and tool start writes status text into the current draft post without rotating to a new post. The PR is a plausible focused fix, but the discussion also surfaced real async-ordering and test-coverage concerns that should be resolved before merge rather than closing it as obsolete.

## Best Possible Solution

Keep the PR open. The best path is a focused Mattermost fix that rotates or flushes the draft stream before assistant-message, reasoning, and tool-status boundaries, accounts for the fire-and-forget callback ordering in the embedded runtime, and adds Mattermost monitor/draft-stream regression coverage for preserving prior streamed text across tool starts. Tool-status UX changes should stay minimal or be split unless maintainers explicitly want richer status posts.
