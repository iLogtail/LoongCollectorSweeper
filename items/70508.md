## Summary

Keep #70508 open. Current main still has the Mattermost draft-stream overwrite behavior the PR targets: assistant/reasoning boundaries only clear `lastPartialText`, and tool starts update the active draft post directly. The PR is a plausible focused bug fix, but the review discussion also raised concrete sequencing/race and missing-test concerns, so the best maintainer outcome is to keep it open for revision or replacement rather than close it as obsolete.

## Best Possible Solution

Keep the PR open. The maintainer path is to revise this Mattermost-specific fix so draft boundaries are serialized with partial, tool, and item events; flush pending draft text before rotating; add focused Mattermost monitor/draft-stream regression tests for assistant boundaries and tool starts; and keep or intentionally update the tool-status text contract. If the current branch is too complex, replace it with a narrower PR that preserves already-streamed assistant text without duplicating or fragmenting tool-status posts.
