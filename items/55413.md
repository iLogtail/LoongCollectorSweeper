## Summary

Keep PR #55413 open. Current main still has the Discord PluralKit webhook lookup gap and does not canonicalize inbound Discord MessageSid from PluralKit `original`, so the PR is not obsolete or implemented. However, the latest PR review comment identifies a plausible regression in the PR's revised lookup ordering, so it is useful but not merge-ready as-is.

## Best Possible Solution

Keep the PR open. The best path is to preserve the PR's canonical `MessageSid`/`MessageSidFull` behavior and regression coverage, while adjusting preflight ordering so known bound-thread webhook echoes are suppressed without making degraded PluralKit webhook payloads miss PK identity lookup before the bot-drop gate.
