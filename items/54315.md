## Summary

Current main does not yet implement the PR's central runtime change. Main has bootstrap truncation warnings/reporting, but bootstrap files are still loaded and injected in existing order, `HARD_EXECUTION_RULES.md` is not a recognized bootstrap file, and there is no priority sorting that would keep SOUL/HARD_EXECUTION_RULES ahead of AGENTS under a tight total budget.

## Best Possible Solution

Keep the PR open for maintainer review or author rebase. The best path is a narrow runtime change that preserves current bootstrap-budget diagnostics, adds deterministic priority ordering for critical bootstrap files, makes a deliberate decision on whether HARD_EXECUTION_RULES.md is a supported bootstrap basename, and adds focused tests for tight-budget ordering and skipped-file diagnostics.
