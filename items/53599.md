## Summary

Keep open. Current main intentionally removed the Chrome extension relay and rejects `driver: "extension"`; the available replacements are node-host `browser.proxy`, remote CDP, and Chrome MCP `existing-session`. Those cover cross-machine browser control only through a node host or remote CDP, and do not restore the Chrome-extension-only managed-hosting workflow described in #53599. I did not find a public plugin/browser-driver seam that would let an external extension relay wire back into the existing browser profile/tool path.

## Best Possible Solution

Keep #53599 open for maintainer judgment. The concrete path is either to document an intentional removal/wontfix, or to design a browser plugin/API seam that lets a ClawHosters-style Chrome extension relay register a profile driver or bridge into the existing browser tool safely. Current docs should continue directing supported remote browser control to node-host `browser.proxy` or remote CDP, but that does not close the reported extension-only regression.
