## Summary

Keep open. Current `main` does not implement `skills.entries.<skill>.model`, and the current skills architecture treats skills as on-demand instructions/tool wrappers under the active agent/session model, so there is not strong evidence for `implemented_on_main`, `cannot_reproduce`, `incoherent`, or a clear enough `clawhub` close.

## Evidence

- **Skills are prompt/tool instructions, not separate model runners:** The skills docs define skills as `SKILL.md` folders that teach the agent how to use tools.
  - file: [docs/tools/skills.md:9](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/tools/skills.md#L9)
