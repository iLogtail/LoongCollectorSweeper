## Summary

Keep open. Current main does not implement a framework-level plugin circuit breaker, and this should not be closed as ClawHub because the requested behavior requires core/plugin-runtime participation: hook wrapping, plugin-scoped state, lock attribution, gateway status visibility, and an operator reset path. The only circuit-breaker implementation found is Nostr relay-specific extension code, not a general plugin runtime facility.

## Evidence

- **Hook runner lacks breaker behavior:** The central plugin hook runner executes registered handlers and catches/logs or rethrows errors by policy, but it does not record rolling plugin failure budgets, open/half-open/forced-open state, cooldowns, persisted breaker state, or skip affected plugin hooks.
  - file: [src/plugins/hooks.ts:323](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/plugins/hooks.ts#L323)

  - file: [src/plugins/hook-runner-global.ts:36](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/plugins/hook-runner-global.ts#L36)
