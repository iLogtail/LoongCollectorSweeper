## Summary

Keep open. Current main has partial graceful handling for plugin initialization failures and fail-open hook error logging, plus subsystem-specific circuit breakers, but it does not implement the requested framework-level plugin circuit breaker with timing/error budgets, plugin-hook lock attribution, persisted open/half-open/forced-open state, status visibility, or a reset command.

## Best Possible Solution

Keep #41899 open, or have a maintainer explicitly consolidate it with #55253 if they prefer one canonical tracker. The implementation belongs in core plugin runtime/hook execution: add timing/error accounting around plugin hooks, plugin and hook attribution for lock/session diagnostics, persisted breaker state, open/half-open/forced-open policy, gateway status visibility, reset controls, and focused tests/docs for fail-open and fail-closed behavior.
