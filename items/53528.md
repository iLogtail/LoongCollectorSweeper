## Summary

Keep open. Current main still does not implement the requested jobs.json watcher/reload behavior, and #52772 was closed specifically as duplicate tracking to this PR. The PR is not merge-ready as submitted, but it remains the canonical open item for a real remaining cron scheduler gap.

## Best Possible Solution

Keep PR #53528 open as the canonical tracker, but request a safer implementation: a file or directory watcher plus/or periodic maintenance re-read that works from an empty startup state, wakes the scheduler after external writes, avoids invalidating state during the service's own persist path, survives rename-based store writes, and adds focused tests for external jobs.json creation/update after startup.
