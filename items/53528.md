## Summary

Keep open. Current `main` still does not implement a `jobs.json` watch/invalidation path, and the existing scheduler code only force-reloads cron state on timer ticks. When cron starts with no enabled jobs, it arms no timer at all, so direct external writes remain unobserved until some other cron operation occurs. The PR also has unresolved review concerns in the discussion about watcher correctness and idle wake-up behavior, so there is not strong evidence that it is ready to auto-close as already-fixed or obsolete.

## Evidence
