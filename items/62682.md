## Summary

Keep open. Current main still does not implement this PR's core behavior: the model fallback layer cannot inspect an AbortSignal reason, HTTP disconnect aborts are untagged, and cron/embedded timeout aborts can still normalize into retryable timeout failover. Related PRs #52365 and #42482 are still open and cover adjacent cron-budget/shared-controller behavior; merged #54388 only propagates HTTP abort signals, not terminal abort classification through model fallback.

## Best Possible Solution

Keep the PR open or fold its focused terminal-abort contract into a maintainer-owned fix: add a narrow `abortSignal`/terminal-reason path to `runWithModelFallback`, tag HTTP client disconnect aborts, preserve retryable provider timeouts, and add regression tests for `TimeoutError`, `ClientDisconnectError`, cron string timeout reasons, and failover-normalization ordering. Coordinate with #52365/#42482 so cron budget handling does not duplicate or conflict with the terminal-abort rule.
