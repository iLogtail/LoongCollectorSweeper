## Summary

Keep PR #69261 open. Current main still resolves one Ollama embedding API key independently of the destination URL and attaches it to whichever embedding base URL is selected, so the PR's central auth-scoping bug remains present. The PR is focused and the discussion/review comments identify refinements to make before merge, not a reason to close it as obsolete.

## Best Possible Solution

Keep the PR open and ask for a rebase/follow-up patch: implement host-boundary-aware auth selection in `extensions/ollama/src/embedding-provider.ts`, filter non-secret markers such as `ollama-local` and env marker strings before bearer attachment, add focused regression tests for env-to-local, provider-to-remote, remote opt-out, provider opt-out on cloud, loopback/cloud host normalization, and update the changelog/docs if behavior changes ship.
