## Summary

Keep open. Current main still has status/session display paths that can show preserved or oversized `totalTokens` directly against `contextTokens`, including stale totals, and the issue comment’s Claude CLI stdout-limit failover also still maps to live code that converts output-limit failures into `FailoverError` with reason `format`. Related older context-display reports were about other surfaces and do not fully cover this current status-card/Claude CLI report.

## Best Possible Solution

Keep #72321 open. Fix the status/session context display to use only fresh current prompt/context snapshots for percentages, suppress or mark stale totals instead of showing impossible utilization, and add regression coverage for an inflated cacheRead/cacheWrite session. Separately triage or split the Claude CLI output-limit path so long responses do not get misclassified as a format failover to weaker fallback models.
