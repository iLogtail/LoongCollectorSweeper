## Summary

Current main still does not implement #38881's central behavior: inbound image messages do not auto-switch the main agent run from a text-only selected model to `agents.defaults.imageModel`. The configured `imageModel` is used for media-understanding/tool routing, while the final reply run continues with the selected provider/model. The PR remains relevant, but its review comments identify real design issues that need maintainer/author follow-up before merge.

## Best Possible Solution

Keep #38881 open for maintainer review. A production fix should live around `src/auto-reply/reply/get-reply.ts`, switch the main run only when normalized inbound media contains actual images and the selected model is positively known to lack vision, coordinate with media understanding to avoid duplicate analysis, thread the real alias index, respect heartbeat/session/channel overrides, support `imageModel` fallbacks and URL-only media, and add focused regression tests proving `runPreparedReply` receives the configured image model.
