## Summary

Keep PR #56720 open. Current main at 7e376e5aba32eeea8191e70d5e43e293f8b8398a still does not implement the requested Gmail watcher `--exclude-labels` plumbing: the runtime config lacks `excludeLabels`, the actual `gog gmail watch serve` arg builder omits the flag, the config schema/types omit the field, and the existing test expectation matches the omission. The PR discussion is useful evidence rather than noise: it links #56635, Greptile agreed the core fix was correct, and the author addressed the two minor review comments in commit 5f56d2ee03cf439076f7bf5ab010d1ea990095e2. This should remain for maintainer selection/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep the item open and land one canonical fix for #56635 that adds `hooks.gmail.excludeLabels`, defaults OpenClaw to `SPAM,TRASH,DRAFT,SENT`, threads config/overrides through `GmailHookRuntimeConfig`, passes `--exclude-labels` from `buildGogWatchServeArgs()`, updates generated config metadata/docs as needed, and adds regression tests for default, custom, and empty label lists. After that lands, close duplicate Gmail exclude-label PRs against the landed commit.
