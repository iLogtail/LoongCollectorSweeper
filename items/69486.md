## Summary

Keep PR #69486 open. Current main still rewrites the active branch by branching at the first rewritten entry's parent and re-appending the suffix, while the file-level wrapper only emits a transcript update after a successful rewrite. The flat `readSessionMessages` reader still scans every message line in the chosen JSONL file, so the PR's central duplication concern is not implemented or superseded on main. The PR also has concrete reproduction/test work and recent author follow-up, plus unresolved review feedback about preserving non-parent references such as label `targetId`, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open and review it as a focused runner/session persistence bug fix. The likely best implementation is an ancestry-aware abandoned-entry cleanup after successful transcript rewrites, but it should also preserve or explicitly handle non-parent references such as label `targetId` and include the production-derived/repeated-rewrite/sibling-branch regression tests before merge.
