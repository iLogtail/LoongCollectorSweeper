## Summary

Keep PR #69486 open. Current main still has the rewrite path that branches from the first rewritten entry and re-appends the suffix without any on-disk cleanup, while `readSessionMessages` still returns every message line from the transcript file. The PR is a plausible focused bug fix with production-derived evidence, but the current review record raises a real `label.targetId` integrity concern that should be addressed before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep #69486 open. The best maintainer path is to require a narrow revision that extends the cleanup integrity model beyond parentId ancestry to all load-bearing transcript references, especially `label.targetId`, with regression tests for sibling branches plus labels. After that, validate the rewrite-file cleanup against the current flat `readSessionMessages` behavior and the existing session-truncation invariants before merging.
