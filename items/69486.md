## Summary

Keep PR #69486 open. Current main still has the rewrite path that branches from the first rewritten entry and re-appends the suffix without cleaning the abandoned on-disk entries, while `readSessionMessages` still scans the raw transcript file linearly. The PR also has recent production-derived repro evidence and unresolved review feedback that should be addressed rather than auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow rewrite cleanup that removes only entries abandoned by the current rewrite, preserves active and sibling branch integrity, accounts for non-parent metadata references such as labels, and includes the new raw-file regression tests and production-derived repro coverage.
