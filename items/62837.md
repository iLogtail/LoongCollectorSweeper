## Summary

Keep #62837 open. Current main still depends on `mentionedJids` being extracted from a narrow set of Baileys `contextInfo.mentionedJid` fields, while the report is specifically about a WhatsApp MD group mention where that extraction returns null despite native mention metadata being present. The docs promise native WhatsApp mentions should satisfy group activation, but current tests do not cover the reported alternate/native payload shape, and related ClawSweeper context shows nearby WhatsApp native/LID mention handling is still not safely resolved.

## Best Possible Solution

Keep the issue open. The best fix is in the WhatsApp plugin: add a regression fixture for the actual WhatsApp MD group @-mention payload that produces the `+38m` signal, broaden `extractMentionedJids()` to read the real current-message mention source without leaking quoted-message mentions, and verify group gating sets `wasMentioned=true` while preserving the existing self-chat/LID safeguards tracked by related WhatsApp mention work.
