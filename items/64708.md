## Summary

Keep PR #64708 open. Current main still uses a raw `resolved.to === resolved.lastTo` check before preserving a session-derived `threadId`, so the bare Telegram chat target versus topic-scoped session route case described by the PR is not implemented on main. The PR is a plausible focused bug fix with a concrete repro, but the submitted patch still needs maintainer/contributor changes before merge because its import path and regression test were both flagged as broken in review.

## Best Possible Solution

Keep the PR open and ask for a small revision: import `normalizeTargetForProvider` from `src/infra/outbound/target-normalization.ts`, compare normalized route shapes when deciding whether a session-derived `threadId` is safe to preserve, and update the regression test to use `normalizeTarget` for Telegram so the bare chat ID versus topic-scoped session route case is actually covered. The related ClawSweeper reports provided are adjacent Telegram thread-routing work, but none supersedes this exact cron delivery bug.
