## Summary

Keep #64616 open. Current main still treats any readable ACP session-store entry as a live backing session, without considering stale `running` ACP metadata or a missing transcript file, so the PR is not obsolete or implemented on main. The PR is recent, focused, contributor-authored, and has no protected close label; it should receive maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open. The best path is to review the current head and either merge a corrected implementation in `src/tasks/task-registry.maintenance.ts` with ACP zombie/live regression tests in `src/tasks/task-registry.maintenance.issue-60299.test.ts`, or request changes so stale `running` ACP records are marked lost only when their transcript backing is genuinely missing/unreadable and live ACP sessions remain protected.
