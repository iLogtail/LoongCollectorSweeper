## Summary

Keep #71669 open. Current main documents and enforces the Control UI secure-context/device-identity rule, and the UI now has client-side guidance, but there is still no startup/runtime server warning for the specific configuration the issue requests.

## Best Possible Solution

Add a focused startup warning in the gateway startup-log path when Control UI is enabled, gateway TLS is not enabled, the gateway is exposed beyond loopback or advertises remote HTTP access, and `gateway.controlUi.dangerouslyDisableDeviceAuth` is not true. Reuse existing docs links, avoid noisy repeats, and add coverage in `src/gateway/server-startup-log.test.ts` for warned and non-warned cases.
