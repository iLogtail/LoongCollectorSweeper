## Summary

Current `main` already supersedes this PR: `waitForActiveTasks()` in `src/process/command-queue.ts` no longer uses the recursive 50 ms poll timer described in the PR and instead waits via an `activeTaskWaiters` set plus a single timeout.

## Evidence

- **Current implementation no longer polls recursively:** `waitForActiveTasks()` snapshots active task IDs, installs one timeout, adds a waiter to `activeTaskWaiters`, and immediately checks completion; there is no recursive `setTimeout(check, POLL_INTERVAL_MS)` path anymore.
  - file: [src/process/command-queue.ts:381](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/process/command-queue.ts#L381)

- **Completion is event-driven on task finish:** Task completion calls `notifyActiveTaskWaiters()`, which resolves waiters when their starting task IDs are gone. This removes the need for a drain poll timer entirely.
