## Summary

Keep open. Current main still awaits LINE event processing before sending HTTP 200 in both webhook handlers, and the tests currently lock in that blocking/failure behavior. The related PR #65397 was closed unmerged, and its referenced commit is not present in this checkout. Official LINE docs define `request_timeout` as the bot server not returning a response within 2 seconds, matching the reported symptom. citeturn0search7

## Best Possible Solution

Keep #65375 open and fix the LINE plugin webhook handlers. After raw body read, signature verification, and payload parsing succeed, send HTTP 200 immediately, then process `handleWebhook` / `onEvents` asynchronously with errors routed to `runtime.error`. Add regression tests proving the 200 is sent while event processing is still pending for both Node and Express middleware paths.
