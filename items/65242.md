## Summary

Keep open. Current main has later subagent completion-delivery fallback work, but it does not implement this PR's shared CompletionDeliveryGate/silent-wake coordination. ACP child-session tasks with a deliverable requester origin are still expected to send a visible `Background task done` banner, and the PR has unresolved P1 review concerns that would need fixes before merge.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a focused revision before merge: implement a production-safe single-delivery coordination model or equivalent sequencing, claim only after successful delivery or support release/rollback, keep shadow mode observational, preserve announce cleanup and farewell semantics, and retain Discord parent-thread lookup. Add regression tests covering ACP child-session task-registry banner versus announce/farewell duplicate paths.
