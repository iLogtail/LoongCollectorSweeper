## Summary

Keep open. Current main has Discord-side timeout mitigation for stuck attachment downloads, and #54112/#68427 were closed on that broader behavior, but this PR's narrower shared-reader fix is still not implemented: the bounded response overflow path still awaits `reader.cancel()`, and the regression test for a never-resolving cancel is absent. The PR needs a rebase because Matrix now delegates to the shared helper, but the underlying shared fix remains useful.

## Best Possible Solution

Keep the PR open and rebase it into a narrower current-main patch: change `src/media/read-response-with-limit.ts` so overflow cancels the reader fire-and-forget, add the non-returning `cancel()` regression test in `src/media/read-response-with-limit.test.ts`, rely on the existing Matrix wrapper through `src/plugin-sdk/response-limit-runtime.ts`, and drop obsolete unrelated import cleanup.
