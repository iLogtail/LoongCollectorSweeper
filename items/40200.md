## Summary

Current `main` does not implement PR #40200's shell-startup suppression. `getShellConfig()` still returns plain `["-c"]` for POSIX shells, and the exec/supervisor paths still pass those args through directly, so the reported daemon-env override remains plausible rather than already fixed on `main`.

## Evidence

- **Current shell selection still uses bare `-c`:** On `main`, `getShellConfig()` returns `args: ["-c"]` for fish->bash fallback, fish->sh fallback, direct `envShell`, and the final `sh` fallback. There is no `zsh -f`, `bash --noprofile --norc`, or `fish --no-config` handling in the shipped code.
  - file: [src/agents/shell-utils.ts:71](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/shell-utils.ts#L71)

  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **Exec paths consume those args unchanged:** The main exec runtime builds `childArgv` as `[shell, ...shellArgs, execCommand]`, and the supervisor PTY adapter likewise passes `args: [...shellArgs, ptyCommand]`.
