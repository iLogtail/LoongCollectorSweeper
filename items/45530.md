## Summary

Keep open. Current `main` does not implement persistent TTS voice switching via a `persist=true` directive: the directive parser has no persistence field, the speech-core runtime does not write per-provider voice prefs after synthesis, Discord voice playback also treats directive voice overrides as one-shot, and the docs still describe only `enabled`/`provider`/`maxLength`/`summarize` as stored TTS prefs. There is a separate `/voice set` Talk-voice command, but it edits gateway `talk.providers.<provider>.voiceId` config rather than message-TTS per-user prefs, so it is not a shipped replacement for this PR.
