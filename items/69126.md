## Summary

Keep #69126 open. Current main has global token-budget compaction tuning under `agents.defaults.compaction`, but it does not implement the requested `agents.defaults.sessionCompactionLimit` key or an equivalent global per-session/message-count compaction trigger. The issue is recent, non-maintainer-authored, has no protected labels in the provided context, and the related reports are analogous defaults requests rather than a canonical duplicate.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. If OpenClaw wants this behavior, add a typed and documented `agents.defaults` config field for a global per-session or message-count compaction trigger, wire it into the session compaction decision path without conflicting with existing token-budget `agents.defaults.compaction.*` settings, and add schema/help/docs plus targeted tests. If maintainers decide existing token-based `agents.defaults.compaction` is the intended answer, answer the issue with that clarification rather than auto-closing it as already implemented.
