## Summary

Current main has limited bundle-backed LSP support, but it does not implement the issue's zero-configuration LSP auto-detection. OpenClaw can read enabled Claude bundle `.lsp.json` / manifest `lspServers`, spawn configured stdio servers, and expose hover/definition/references tools. It does not scan project files or package manifests, discover installed LSPs from PATH, suggest or install missing servers, auto-configure per language, or provide the requested diagnostic dashboard. The related #65252 review does not supersede this because that item covers real-time diagnostics, while #65251 is specifically about auto-detection and setup.

## Best Possible Solution

Keep the issue open for product/API design. The best path is to extend embedded Pi LSP setup beyond static bundle defaults: add a safe workspace/project detection layer, discover installed servers without surprising installs, expose explicit opt-in install/fix actions, support user overrides, and surface status through CLI/UI diagnostics. If maintainers want this outside core, the missing piece should be a documented plugin or bundle API that lets a plugin contribute dynamic per-workspace LSP configuration safely.
