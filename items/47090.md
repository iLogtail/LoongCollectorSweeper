## Summary

Keep open. The PR's central session-archiving workflow is not implemented on current main: there is no `archivedAt` session field, no `includeArchived` list parameter, no archive/unarchive patch handling, no archive summary writer, and no Sessions UI archive filter/action. The PR also still needs maintainer review because its diff includes unrelated Telegram `/set_topic_name` and docs changes alongside the archive feature, and the provided review comments show several behavior risks that would need current-main validation before merge.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to rebase it onto current main, split or drop unrelated `/set_topic_name` and docs-heading changes, then land the archive feature with explicit `archivedAt` schema/store support, `includeArchived` list semantics, archive/unarchive UI controls, inbound auto-unarchive persistence, archive summary failure isolation, and regression tests for gateway list/patch, UI filtering, session resolution, spawned visibility, announce-target lookup, and ACP snapshot paths.
