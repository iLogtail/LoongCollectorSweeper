## Close Comment

Closing after a Codex review. I couldn't reproduce the reported "user message during heartbeat gets hidden" path on current `main`. In the normal chat-send flow, Gateway keeps `clientRunId === runId` and registers that same id as the run context (`src/gateway/server-methods/agent.ts:730`, `src/gateway/server-methods/agent.ts:738`), so `resolveHeartbeatContext()` in `src/gateway/server-chat.ts:35` never falls back to a different heartbeat `sourceRunId` for an interactive chat turn. The only in-tree remapped chat path is voice ingress (`src/gateway/server-node-events.ts:351`), and that path creates a synthetic `voice-*` client id while the actual source run is registered as a normal non-heartbeat run in `src/auto-reply/reply/agent-runner-execution.ts:671`. That means the specific primary-non-heartbeat/source-heartbeat state described here does not occur in current main, so there isn't a reproducible bug for this PR to land. If there's a concrete failing path outside those flows, please open a fresh report with the exact entrypoint and run-id mapping.
