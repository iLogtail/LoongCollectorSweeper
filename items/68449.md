## Summary

Keep open. Current main has partial fixes for the original report: expanded concept stop-word filtering, session-store based cron transcript exclusion, NO_REPLY/heartbeat filtering, and tests. But the issue discussion also identifies legacy/read-path and light narrative theme contamination, and current main still passes raw persisted conceptTags into the light dreaming narrative before the short-term repair runs. No narrower canonical item in the provided context covers that remaining scope.

## Best Possible Solution

Keep #68449 open as the canonical memory-core dreaming noise bug until the remaining read-path/narrative scope is handled. The best fix is to preserve the current stopword and cron/NO_REPLY ingestion guards, then add a shared noise-token filter or repair-before-phase path for persisted conceptTags used by REM and light narratives, including regression tests for legacy assistant/the tags, reflection-template words, numeric/date tokens, and cron-store classified sessions. #68876 should be folded or superseded once its corpus-filter coverage is confirmed against current main.
