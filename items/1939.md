## Summary

本 PR 修复了匿名扩展插件 ID 生成冲突导致的资源泄露问题，以及非法 input 类型错误被吞没的缺陷。维护者已就插件 ID 格式变更（引入下划线如 1_1）的向后兼容性提出疑问，目前等待作者回应或调整实现。属于有效的核心缺陷修复，需进一步讨论兼容性方案后合并。

## Best Possible Solution

建议作者回应维护者关于插件 ID 格式变更是否破坏下游解析或现有约定的疑问。若需保持纯数字 ID，可调整实现策略（例如使用独立的全局递增计数器，或基于主插件 ID 的大步长偏移生成唯一数字 ID）。确认兼容性或调整实现后，请维护者重新审查并合并。同时建议补充相关集成测试以验证热更新与退出时的资源清理行为。
