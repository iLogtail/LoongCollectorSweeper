## Summary

Keep PR #68961 open. Current main has adjacent agent-runtime work and a supported `claude-cli` backend, but it does not contain this PR's `claude-sdk` runtime driver, `runtime-dispatch` entry point, `runAgent` SDK surface, TypeBox-to-Zod native-tool bridge, session mirror, or generated `.claude/agents` assets. The prior #68135 is closed unmerged and this PR is the active resubmission with substantive, still-unmerged work plus unresolved P1/P2 review findings that need maintainer/product judgment rather than automated closure.

## Best Possible Solution

Keep the PR open for explicit maintainer judgment. The best path is to decide product direction against the current `agentRuntime.id`/`claude-cli`/agent-harness architecture, then either split and repair the Claude Agent SDK adapter with tests and live smoke coverage or close it manually as a superseded product direction with a canonical replacement issue/PR. Automated cleanup should not close it now.
