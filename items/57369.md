## Summary

Keep open. Current `main` still exposes only `daily` and `idle` session reset modes in the public config types, schema, resolver, and docs, so the requested manual-only `mode: "never"` behavior is not implemented.

## Evidence

- **Public session reset type still excludes `never`:** `SessionResetMode` is defined as `"daily" | "idle"`, and `SessionResetConfig` only documents those two modes.
  - file: [src/config/types.base.ts:124](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/types.base.ts#L124)
  - command: `nl -ba src/config/types.base.ts | sed -n '110,160p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Config schema rejects any mode besides `daily` or `idle`:** `SessionResetConfigSchema` validates `mode` with only `z.literal("daily")` and `z.literal("idle")`; `idleMinutes` is also positive-only, so there is no explicit zero/disable shape here either.
  - file: [src/config/zod-schema.session.ts:17](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/zod-schema.session.ts#L17)
  - command: `nl -ba src/config/zod-schema.session.ts | sed -n '1,70p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Runtime resolver still defaults to daily and has no manual-only branch:** `resolveSessionResetPolicy` falls back to `DEFAULT_RESET_MODE` (`daily`) whenever no explicit reset mode is present, and `SessionResetPolicy.mode` is typed only as `daily | idle`.
  - file: [src/config/sessions/reset-policy.ts:4](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/sessions/reset-policy.ts#L4)
  - command: `nl -ba src/config/sessions/reset-policy.ts | sed -n '1,95p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **User-facing config docs still document only `daily | idle`:** The session config example and field reference list only `daily` and `idle` for `session.reset.mode`, with no `never`/manual-only option described for base, type, or channel overrides.
  - file: [docs/gateway/config-agents.md:1117](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/gateway/config-agents.md#L1117)
  - command: `nl -ba docs/gateway/config-agents.md | sed -n '1110,1175p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Concept docs still describe daily default plus optional idle reset:** The session lifecycle docs say sessions expire via daily reset by default, optional idle reset, or manual `/new`/`/reset`, which matches the current implementation and not the requested manual-only config mode.
  - file: [docs/concepts/session.md:57](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/concepts/session.md#L57)
