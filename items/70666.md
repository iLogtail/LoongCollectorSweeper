## Summary

Current main still has the reported false-positive path. The auto-reply guard appends the unscheduled-reminder note whenever a reminder-looking reply has `successfulCronAdds === 0` and no same-session cron exists, while tool telemetry increments `successfulCronAdds` only for the native `cron` tool `add` action. I found no `reminderGuard` config field or plugin-tool scheduling signal in the current config/plugin SDK surface. The linked PR #71170 directly targets this issue, but it is open/unmerged in the provided context, so #70666 should remain open as the canonical bug until a fix lands.

## Best Possible Solution

Keep #70666 open until the external-scheduler path is actually fixed on main. The pragmatic near-term fix is to review and land #71170 or an equivalent config-backed opt-out with schema/docs/tests. A fuller follow-up could add an explicit plugin SDK scheduling signal so plugin tools can mark a scheduled event as created without disabling the guard wholesale.
