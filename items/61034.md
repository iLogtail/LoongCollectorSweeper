## Summary

Current main does not implement the requested reasoning-language option. OpenClaw has controls for thinking depth and reasoning visibility, but the config/schema/runtime path still models reasoning as visibility only (`off|on|stream`) and formats displayed reasoning with the existing English `Reasoning:` label while preserving the model-emitted reasoning text.

## Best Possible Solution

Keep this open for maintainer product/design judgment. If accepted, add a core config surface such as `agents.defaults.reasoningLanguage` with optional per-agent/session overrides, validate language tags, feed the preference into the system prompt or provider-specific request layer where supported, keep final-response language behavior unchanged, and update schema/help/docs plus focused tests for `/reasoning on` and reasoning payload formatting.
