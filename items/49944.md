## Summary

Keep #49944 open. Current main has a lower-level `autoExpandToolCalls` mechanism, but the primary Control UI chat render path still passes it as `false`, and the effective `verboseLevel` from `agents.defaults.verboseDefault` is not wired into tool-card expansion. The linked PR #50034 was closed unmerged per the provided GitHub context, and current main does not contain the described `activeSession.verboseLevel === "full"` wiring.

## Best Possible Solution

Keep the issue open. Implement a narrow Control UI fix that carries the effective verbose level from `chat.history` or an equivalent session/default source into app state, passes `autoExpandToolCalls: effectiveVerboseLevel === "full"` into `renderChat`, and adds regression coverage for global `agents.defaults.verboseDefault = "full"`, explicit session `verboseLevel = "full"`, and the default collapsed case.
