## Summary

Current main still has the node systemd unit-name mismatch this PR is trying to fix. Node install/uninstall injects OPENCLAW_SYSTEMD_UNIT=openclaw-node, and the unit file path honors that override, but the install activation and uninstall disable paths still derive openclaw-gateway.service from the gateway profile resolver.

## Best Possible Solution

Keep the PR or recreate its behavior on current main: make activateSystemdService and uninstallSystemdService derive unitName through resolveSystemdServiceName(env), preserving the gateway fallback, and add a regression test that node install/uninstall issues systemctl commands for openclaw-node.service.
