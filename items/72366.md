## Summary

Current main fixes the v2026.4.24 Bonjour/CIAO crash-loop reported in #72366. The bundled Bonjour plugin now registers scoped process handlers, classifies CIAO PROBING/ANNOUNCEMENT cancellations as handled, keeps the gateway fatal path from exiting for handled plugin errors, bounds stuck advertiser restarts, and auto-disables Bonjour in detected container environments. The issue discussion's Ubuntu, WSL2, Linux, and AWS ECS confirmations all match this same Bonjour/mDNS failure family and workaround, and related ClawSweeper reviews for #72103, #72152, and #72307 reached the same implemented-on-main conclusion.

## Best Possible Solution

Close #72366 as implemented on main. Keep the current Bonjour-owned fix: scoped CIAO process handling, shared handler registration, bounded advertiser recovery, automatic container disable, and the documented disable-Bonjour workaround. Any backport or publication decision should go through the normal release process rather than keeping this duplicate crash report open.
