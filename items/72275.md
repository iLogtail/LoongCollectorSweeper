## Summary

Keep #72275 open. The report is fresh, concrete, and targets a plausible cross-boundary path: Telegram topic sessions are agent-scoped, normal model selection has per-agent support, but current main does not have source/test/history evidence proving that post-compaction retry/session state preserves the bound agent's configured primary instead of falling back to `agents.defaults.model.primary`.

## Best Possible Solution

Keep the issue open and add a focused regression around the Telegram/topic or generic channel session path: configure `agents.defaults.model.primary` to one model, `agents.list[].model.primary` to another, force overflow or explicit compaction for an `agent:<id>:...` session, then assert both the retry run and persisted session metadata continue selecting the agent-specific provider/model. If that fails, fix the compaction retry/runtime-context or session-store model resolution so it re-resolves from the bound `agentId` instead of global defaults.
