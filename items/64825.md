## Summary

Keep open. Current main mitigates some stuck-streaming cases with the TUI streaming watchdog shipped in 2026.4.15, but it still does not implement the issue's specific `wasActiveRun === false` final-event fallback. The local `/btw`/unbound path remains plausible from source: chat deltas set `streaming`, local BTW runs do not bind `activeChatRunId`, and the empty-final path returns without resetting activity status.

## Best Possible Solution

Keep #64825 open as the canonical bug report until a maintainer lands or supersedes the narrow inactive-final recovery work in #64862/#52745. The best fix is a scoped status-recovery path in `finalizeRun`/`terminateRun` or the local-BTW empty-final branch when no other run is active, with regression coverage for unbound/local-BTW final events and existing concurrent-run/deferred-history tests preserved.

## Evidence

When `wasActiveRun` is false, current main only clears a matching streaming watchdog and does not fall back to idle based on `sessionRuns.size`/`activeChatRunId`, which is the specific fix requested by #64825.
