## Summary

Keep open. Current main still returns cached cron evaluators without promoting the key, then evicts the first Map key when the cache is full, so the FIFO-vs-LRU bug remains present. The PR is focused, addresses the code path directly, and its follow-up test change addresses the earlier review concern by asserting specific cache keys survive or are evicted.

## Best Possible Solution

Keep this PR or a narrow equivalent open until main implements true LRU promotion in src/cron/schedule.ts and adds a regression test that verifies specific cache keys across eviction. If maintainers prefer a different cache abstraction, it should still preserve the observable behavior: cache hits must refresh recency before capacity eviction.
