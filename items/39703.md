## Summary

Keep #39703 open. Current main still returns cached cron evaluators without promoting the Map entry, then evicts the first Map key at capacity, so the FIFO-vs-LRU bug remains. The PR remains the focused useful fix, and the provided context shows its second commit addressed the earlier weak-test review feedback with key-level cache assertions.

## Best Possible Solution

Keep #39703 open as the canonical remaining cron cache LRU fix, or land an equivalent maintainer-owned patch that promotes cache hits with delete/re-set and includes key-level regression coverage proving the promoted key survives eviction while the next-oldest key is evicted.

## Evidence
