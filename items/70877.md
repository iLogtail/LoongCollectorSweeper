## Summary

Keep PR #70877 open. Current main still defaults tool-loop detection to disabled, the production resolver does not synthesize `enabled: true` when config is absent or partially merged, tests still assert the no-config path is disabled, and user-facing docs/schema/type metadata still advertise `default: false`. The PR addresses a real core agent safety problem, but the provided review context also shows unresolved concerns in the PR head around direct detector defaults, schema/UI default drift, generated config artifacts, and inheritance semantics, so it needs maintainer review/fixes rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer handling or consolidation with the overlapping loop-detection PRs. The best code path is a minimal core fix that enables loop detection by default in the actual runtime/default resolver, preserves explicit and inherited disable semantics, updates schema source plus generated config artifacts/docs/types, and adds focused tests for no-config default-on, explicit false, global false with partial agent overrides, and generated schema default/help consistency.
