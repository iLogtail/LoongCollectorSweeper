## Summary

Keep PR #70877 open. Current main still leaves generic tool-loop detection disabled by default: the runtime default is `enabled: false`, the config resolver passes through omitted/partial loopDetection config without default-enabling it, tests still assert the no-config path is disabled, and docs/schema/type metadata still advertise default false. The PR overlaps with earlier open PR #56403, but there is no merged or maintainer-designated canonical replacement, and #70877 contains a concrete live `web_fetch` retry report. Its current implementation still needs fixes before merge.

## Best Possible Solution

Keep the PR open or consolidate it with one canonical maintainer-reviewed loop-detection-default PR. The best implementation should default-enable the actual runtime resolver used by `detectToolCallLoop`, preserve explicit global `enabled: false` when agent overrides omit `enabled`, update docs/help/generated schema and baseline artifacts from source, and add regression coverage for no-config default-enabled behavior plus explicit opt-out behavior.
