## Summary

Current main already solves the task-audit false-positive path that #69880 was trying to address, using a stronger timestamp normalization path instead of the PR's broad invalid-createdAt early return. The implementation normalizes task lifecycle timestamps on create/update/restore, tests cover earlier startedAt inputs, and the existing audit still preserves createdAt-independent inconsistency checks that #69880 would suppress.

## Best Possible Solution

Close #69880 and keep the current main timestamp-normalization implementation. It fixes the observable task-audit false positive without weakening unrelated audit checks; any remaining legacy persisted-record cleanup or audit-tolerance discussion should stay on the canonical task-audit follow-up rather than merging this invalid-createdAt bypass.
