## Summary

Keep open. Current `main` still mutates heartbeat prompts that mention `HEARTBEAT.md` by appending a workspace-path hint, and the current test suite explicitly locks that behavior in. That conflicts with the docs and prompt resolver, which say custom heartbeat prompt bodies are sent verbatim.

## Evidence

- **Current runtime still appends the hint:** `appendHeartbeatWorkspacePathHint()` adds `When reading HEARTBEAT.md, use workspace file ... Do not read docs/heartbeat.md.` to any prompt matching `/heartbeat\.md/i`, and `resolveHeartbeatRunPrompt()` applies it to the resolved heartbeat prompt on the main path.
