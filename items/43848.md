## Summary

Keep open. Current main has a partial fix for announce `threadId` preservation, but the issue's two main regressions are still present in code and tests: `sessions_send` still suppresses outward delivery when the announce step returns `ANNOUNCE_SKIP`, and the ping-pong path still feeds `inter_session` turns back through the requester except for one ACP-specific skip case.

## Evidence

- **Initial `sessions_send` run is internal-only:** The target session is started with `deliver: false` and `inputProvenance.kind = "inter_session"`, so visible delivery depends on the later A2A announce flow rather than the primary run.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/tools/sessions-send-tool.ts#L275)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '268,330p'`
- **`ANNOUNCE_SKIP` still drops the visible send:** The announce step only calls gateway `send` when the announce reply is non-empty and not `ANNOUNCE_SKIP`; otherwise nothing is posted to the target channel/topic.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:121](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/tools/sessions-send-tool.a2a.ts#L121)
  - command: `nl -ba src/agents/tools/sessions-send-tool.a2a.ts | sed -n '112,145p'`
- **Current tests still encode the skip behavior:** The existing `sessions_send supports fire-and-forget and wait` test makes the announce step return `ANNOUNCE_SKIP` and asserts `expect(sendCallCount).toBe(0)`, which matches the reported missing-visible-post behavior rather than a fix.
  - file: [src/agents/openclaw-tools.sessions.test.ts:800](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/openclaw-tools.sessions.test.ts#L800)
  - command: `nl -ba src/agents/openclaw-tools.sessions.test.ts | sed -n '779,925p'`
- **Ping-pong remains generic and requester-facing:** Ping-pong runs whenever requester and target session keys differ; there is no Telegram topic-to-topic or same-agent room-move suppression in this path. The same test suite also asserts each nested agent call carries `inputProvenance: { kind: "inter_session" }`, which is the mechanism behind requester transcript contamination.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:70](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/tools/sessions-send-tool.a2a.ts#L70)
  - command: `nl -ba src/agents/tools/sessions-send-tool.a2a.ts | sed -n '56,111p'`
- **Only a partial fix is present on main:** Current main does preserve `threadId` when resolving/sending announce targets, and there is a regression test for that, but this only addresses the topic-delivery part of the report, not the `ANNOUNCE_SKIP` suppression or transcript-pollution parts.
  - file: [src/agents/tools/sessions-announce-target.ts:21](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/tools/sessions-announce-target.ts#L21)
