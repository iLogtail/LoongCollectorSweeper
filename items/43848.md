## Summary

Keep open. Current main partially fixes the Telegram forum-topic routing portion by preserving threadId through announce target resolution and final gateway send, but the issue's other reported behaviors remain: ANNOUNCE_SKIP still suppresses visible delivery instead of falling back to the target reply, and normal peer sessions_send still runs A2A ping-pong turns that can re-enter the requester transcript.

## Best Possible Solution

Keep #43848 open or split it into focused follow-ups before closing. The likely fix belongs in src/agents/tools/sessions-send-tool.a2a.ts and src/agents/tools/sessions-send-tool.ts: preserve the shipped threadId routing, define explicit behavior for ANNOUNCE_SKIP after a target reply exists, and suppress or make configurable the topic-to-topic/same-agent A2A ping-pong path so requester transcripts are not polluted. Add regression coverage for Telegram topic-thread delivery, ANNOUNCE_SKIP fallback, and requester transcript isolation.
