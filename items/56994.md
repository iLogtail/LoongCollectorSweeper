## Summary

Keep #56994 open. Current main has implemented important parts of the request: `saveJsonFile` now writes through a sibling temp file with file fsync, rename/copy fallback, chmod, and best-effort directory sync, and `writeRestartSentinel` now uses `writeJsonAtomic`. But the shell completion installer still rewrites the user's shell profile with direct `fs.writeFile`, so a concrete affected path from the issue remains unfixed. Exec approvals are also only partially aligned with the requested durability contract: they now use temp+rename and secure mode, but do not fsync the temp file or parent directory like the shared atomic JSON writer does.

## Best Possible Solution

Keep the issue open but narrow it to the remaining actionable work: replace the completion profile create/update writes with `writeTextAtomic` using profile-appropriate mode 0o644, and decide whether `writeExecApprovalsRaw` should be upgraded to the same sync fsync plus directory-sync durability contract now used by `saveJsonFile`. The restart sentinel and `saveJsonFile` sub-items can be treated as already implemented on current main.
