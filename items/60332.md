## Summary

Keep #60332 open. Current main partially counts OpenClaw-triggered context-engine compactions, but the central request remains missing: there is no ContextEngine stats/reporting contract for plugin-owned background or afterTurn/maintain compactions, and /status/session_status still render only the session-store compactionCount.

## Best Possible Solution

Keep the issue open and add a small core/plugin API contract for compaction accounting. A good path is either an optional ContextEngine stats method or a runtimeContext/reportCompaction helper that increments persisted session compactionCount, then wire /status and session_status to that persisted value and add focused tests for ownsCompaction engines that compact in afterTurn or maintain without a built-in auto_compaction_end event.

## Risks / Open Questions

- A core fix needs a backward-compatible source of truth for plugin-owned compaction counts, otherwise OpenClaw-triggered compact() calls and plugin background compactions could be double-counted.
- If status rendering queries live engine stats directly, it must handle inactive engines, background maintenance, and cross-process/session-store persistence without making /status slow or flaky.
