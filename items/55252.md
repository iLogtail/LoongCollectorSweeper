## Summary

Keep open. Current `main` does not implement a Slack `suppressAssistantText` config or equivalent enforcement. The Slack config surface has no such field, repo-wide search finds no references, and the Slack reply/draft delivery path still sends normal assistant text unless the payload is the exact silent token `NO_REPLY`.

## Evidence

- **Slack config surface lacks the requested option:** `SlackChannelConfig` and `SlackAccountConfig` enumerate Slack channel/account settings, but neither type includes `suppressAssistantText` or an equivalent assistant-text suppression flag.
  - file: [src/config/types.slack.ts:31](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/types.slack.ts#L31)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Normal Slack replies are still delivered unconditionally:** `deliverNormally()` calls `deliverReplies(...)`, and the main dispatch path routes final replies through `deliverNormally()` or preview-finalization without any config check that would suppress ordinary assistant text.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:470](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/slack/src/monitor/message-handler/dispatch.ts#L470)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Draft preview streaming still posts assistant text:** `createSlackDraftStream()` sends and edits preview text through `sendMessageSlack()` / `editSlackMessage()` with no suppression setting, so draft assistant previews remain a visible Slack delivery path.
  - file: [extensions/slack/src/draft-stream.ts:22](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/slack/src/draft-stream.ts#L22)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Only exact silent-token replies are suppressed:** `sendMessageSlack()` suppresses only `NO_REPLY`-style silent replies before the Slack API call. There is no broader rule to suppress ordinary assistant text output.
  - file: [extensions/slack/src/send.ts:310](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/slack/src/send.ts#L310)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Repo-wide search finds no implementation or docs for the option:** A repo search for `suppressAssistantText` returned no matches in Slack extension code, config types, UI, docs, or changelog, which strongly indicates the requested config was not landed on `main`.
  - command: `rg -n "suppressAssistantText" CHANGELOG.md docs/channels/slack.md src/config/types.slack.ts extensions/slack ui/src`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Latest shipped Slack change is narrower than the issue request:** The current changelog mentions suppressing verbose tool/plan progress on Slack non-DM surfaces, but not a configurable suppression of all regular assistant text or draft/final replies.
  - file: [CHANGELOG.md:54](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/CHANGELOG.md#L54)
