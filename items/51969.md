## Summary

Keep open. Current main only partially supersedes this PR: the cron schema now says `tz` is an IANA timezone and the CLI/docs already mention `--tz <iana>`, but the exact inline cron-tool description change from this PR is still not on `main`. More importantly, the open review comment is substantive: the proposed prose would incorrectly steer users away from valid fixed-offset IANA zones like `UTC` and `Etc/GMT+5`, which changes scheduling semantics across DST.
