## Summary

Keep open. The PR is still useful docs work, but current main does not fully cover the exact `browser.request` install-drift signature, and the maintainer discussion says the PR should be rebased and expanded to distinguish install drift from plugin filtering, including the `plugins.entries` case. Closing it now would drop a plausible docs improvement rather than recognizing it as obsolete.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and revise the docs to cover both proven branches: an explicit `browser.request` plus newer-config/version-mismatch install-drift signature with Mac app/doctor/install repair guidance, and a sibling plugin-filter signature for `plugins.allow` or restrictive `plugins.entries` omitting `browser`, where the fix is restoring `plugins.entries.browser` or removing the restrictive plugin list plus restarting the Gateway.
