## Summary

Keep open. Current `main` already documents the ID-first/default Slack allowlist behavior, but this PR's proposed warning overstates it by saying name-based keys never work under `allowlist`. Runtime still supports two name-based paths: startup resolution to IDs when token access is available, and explicit `dangerouslyAllowNameMatching`. That makes the PR salvageable but not a strong auto-close candidate.
