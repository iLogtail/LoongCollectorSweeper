## Summary

Keep #53112 open. Current main has Slack docs that mention stable channel IDs and ID-first routing, but it does not include the explicit warning/troubleshooting callout this PR proposes. The user confusion is real, per #28037 comments, but the PR needs revision before merge because its warning overstates runtime behavior and its examples use the legacy `allow` key instead of current `enabled`.

## Best Possible Solution

Keep the PR open but ask for a small rework: add the explicit Slack allowlist ID warning and troubleshooting callout, mention the two supported name-matching exceptions, avoid absolute "never" wording, and update examples to use `enabled` instead of legacy `allow`.
