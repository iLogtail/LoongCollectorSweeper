## Summary

Keep #53112 open. Current main still has the Slack docs gap the PR targets: it only says channel allowlists should use stable IDs and the troubleshooting section does not call out the `groupPolicy: "allowlist"` name-key failure mode. The PR is not obsolete, but it likely needs rework because its proposed warning reportedly says name-based keys will never route, while current runtime supports startup name-to-ID resolution and the explicit `dangerouslyAllowNameMatching` escape hatch.

## Best Possible Solution

Keep the PR open and ask for a small docs revision: make the default guidance stronger that `channels.slack.channels` allowlist keys should be Slack channel IDs for reliable routing, add the troubleshooting callout, but explicitly mention the two supported exceptions: startup name-to-ID resolution when Slack API lookup succeeds and `channels.slack.dangerouslyAllowNameMatching: true` as a break-glass compatibility path.
