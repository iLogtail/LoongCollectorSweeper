## Summary

Keep #7817 open. Current main has useful building blocks, including session labels/groups, parent/spawn metadata, display-only reply pills, and checkpoint branching, but it does not implement the requested project tree metadata/navigation, composer-level thread routing or fanout, arbitrary message split/move triage, or seed-to-branch workflow. The related #55349 is a narrower quote-reply request and does not supersede this broader Control UI feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete solution should add explicit project/session-tree metadata, additive Gateway protocol methods for split/move/seed and route/fanout targeting, a project tree navigator, composer target controls, and focused protocol/UI tests. Keep #55349 separate as the narrower quote-reply/threaded-reply request.
