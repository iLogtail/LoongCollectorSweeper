## Summary

Keep open. Current main still has the reported synchronous help-banner path: `configureProgramHelp()` calls `formatCliBannerLine()` from Commander's `addHelpText("beforeAll")`, `formatCliBannerLine()` calls `readCliBannerTaglineMode()`, and `readCliBannerTaglineMode()` still statically imports the full config barrel and synchronously calls `createConfigIO({ env }).loadConfig()`. The related fix PR #62485 is still open and unmerged, so the issue should remain available for that fix or an equivalent maintainer-approved patch.

## Best Possible Solution

Keep #62484 open. Review and land #62485 or an equivalent fix that removes synchronous full-config loading from the Commander help-banner path, preferably by moving config access behind a lazy runtime boundary or by skipping configured tagline reads during help-only cold paths. Add targeted regression coverage for `openclaw tui -h`/subcommand help so the command prints help and exits without importing or synchronously loading the heavy config/plugin path.
