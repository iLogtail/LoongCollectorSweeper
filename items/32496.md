## Summary

Keep #32496 open. Current main still does not provide first-class `frequency_penalty`, `presence_penalty`, or `repetition_penalty` passthrough from resolved agent/model params. The available `params.extra_body` escape hatch only covers `api: "openai-completions"` proxy payloads, while Responses and WebSocket payload builders still only forward existing fields such as temperature/max tokens/top_p. Related PRs #32540, #32582, and #33620 were closed unmerged; #71350 is adjacent per-agent params/thinking work, not a canonical replacement for this sampling-penalty request.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best implementation is targeted agent/provider transport work: collect supported numeric snake_case and camelCase sampling params from resolved global, per-model, and per-agent params; inject them only into compatible OpenAI/OpenAI-compatible request paths; preserve existing `onPayload`, `extra_body`, and proxy-store behavior; and add focused tests plus docs for supported fields and transports.

## Risks / Open Questions

- Some providers reject unsupported sampling fields, so a fix should be scoped by provider/API capability instead of blindly injecting every configured key into every transport.
- `params.extra_body` already mitigates a subset of OpenAI-compatible Completions proxy cases, so implementation should define merge order and collision behavior with that escape hatch.
