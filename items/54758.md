## Summary

Keep PR #54758 open. Current main still flushes pending `GatewayBrowserClient` requests with a generic `gateway closed (...)` error on WebSocket close, even when `pendingConnectError` contains structured connect-handshake details. The PR remains a focused bug fix with a targeted regression test for preserving `GatewayRequestError.details.code` on pending request rejection.

## Best Possible Solution

Keep the PR open and have maintainers rebase or recreate the same narrow fix on current main: in `ui/src/ui/gateway.ts`, flush pending requests with a `GatewayRequestError` when `pendingConnectError` is present, while preserving the generic close error for ordinary disconnects. Add the PR's targeted `ui/src/ui/gateway.node.test.ts` regression test for pending request rejection with `details.code`.
