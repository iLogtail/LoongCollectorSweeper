## Summary

Keep PR #64684 open. Current main has adjacent safeguards (doctor steps during some update flows, config validation, last-known-good recovery, and restart sentinels with doctor hints), but it does not implement the PR's specific behavior of running a bounded doctor summary after `update.run` failures or surfacing a doctor-backed diagnostic in restart sentinel messages. The PR is a focused code change that may still be useful, and the broader related self-healing issue #55347 remains open rather than a shipped replacement.

## Best Possible Solution

Keep the PR open for maintainer review or author refresh. The best path is to rebase it onto current main, keep the bounded `update.run` doctor-summary behavior if maintainers want that user-facing diagnostic, and either implement config-reload restart-check diagnostics in a non-stale immediate reporting path or fold that portion into the broader #55347 self-healing design.
