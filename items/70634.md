## Summary

Keep #70634 open. Current main still serializes all turns for a session through a FIFO per-session lane, and `sessions_send`/A2A follow-up turns are queued into that same session lane. The code has provenance metadata that distinguishes `external_user` from `inter_session`, but the command queue does not carry or use it for priority. The KinthAI comment is useful confirmation of the product shape: a two-tier per-agent/session queue plus back-pressure is a plausible fix, not something already shipped.

## Best Possible Solution

Keep the issue open and implement a core scheduler fix: extend the per-session command queue with explicit priority/origin metadata, propagate `external_user` for channel/webchat ingress and `inter_session` for `sessions_send`/A2A work, drain human/external tier entries before agent-originated background entries while preserving FIFO within each tier, and add regression tests that enqueue an inter-session loop plus a later human turn for the same session. The KinthAI comment's two-tier queue and back-pressure notes are useful design input; back-pressure can be a follow-up if the ordering fix lands first.
