## Summary

Keep #70634 open. Current main still uses FIFO session command lanes with no origin/priority field, while `sessions_send` inter-session work and human/webchat/channel turns can contend for the same per-session lane. The issue is recent, not maintainer-authored, has no protected label, and the KinthAI comment adds corroborating production experience rather than making it stale or incoherent.

## Best Possible Solution

Keep the issue open and implement origin-aware priority in the core run/queue path: classify channel/webchat/voice/control-user ingress as human/external at the boundary, classify `sessions_send` and other automated follow-ups as inter-session/background, add a priority/tier field to command queue entries, preserve FIFO within each tier, and add regression tests proving a human turn jumps ahead of queued inter-session work for the same session lane. Consider separate follow-up work for budget/back-pressure on agent-to-agent loops.
