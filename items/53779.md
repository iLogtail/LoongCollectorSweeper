## Summary

Keep this PR open. Current main still validates `models.providers` as one strict Zod object and returns a full config validation failure when any provider entry is invalid; I did not find the PR's per-provider sanitization or its regression test on main. The linked #21584 discussion and the later 2026.4.10 user confirmation describe a real remaining failure mode, although the PR still needs maintainer-quality cleanup for sanitized warning output before merge.

## Best Possible Solution

Keep the PR or its linked bug path open. The best maintainer path is to implement provider-level validation isolation in the models/config loading path, preserve valid providers when siblings fail validation, emit sanitized provider-specific warnings, and add focused regression coverage for mixed valid/invalid providers plus all-invalid providers. The PR's current idea is still useful, but it needs the logged-warning safety review addressed before landing.
