## Summary

Keep open. Current main has explicit Telegram ACP paths, but not the requested per-DM/per-chat default ACP binding that auto-spawns on the first message in a new topic. The existing support is manual `/acp spawn --thread ...`, static top-level `bindings[]` for known forum topic IDs, or per-topic `agentId` routing to normal agents.

## Best Possible Solution

Keep #60737 open. If maintainers want this behavior, implement it in the bundled Telegram plugin/config path: add typed and zod-validated `defaultAcp` config for direct/group/account scope, preserve precedence for explicit `topics.<id>.agentId` and top-level `bindings[].type="acp"`, detect unbound new DM/group topics after access checks, initialize the ACP session through the existing ACP binding/session-binding services, and add Telegram config/routing tests plus docs.
