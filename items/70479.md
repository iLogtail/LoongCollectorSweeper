## Summary

Keep PR #70479 open. Current main still contains the compaction-failure session rotation path that this PR removes, and the linked bug #70472 remains open. The PR is a focused, plausible fix with regression coverage in its diff, but it also has an unresolved maintainer review question about retry-loop behavior, so it should remain available for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep the PR open and have maintainers review the proposed session-preserving behavior against the retry harness. The likely product direction is to stop automatic compaction-failure session rotation for WebChat/main sessions, preserve the active session mapping, surface explicit retry `/compact` `/new` guidance, and keep role-ordering reset behavior separate with focused regression tests.
