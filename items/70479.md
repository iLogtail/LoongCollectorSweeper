## Summary

Keep PR #70479 open. Current main still contains the compaction-failure session reset path that the PR is trying to remove, and the linked issue #70472 describes a concrete WebChat session-orphaning failure. This is a plausible focused core auto-reply bug fix, not implemented on main, not a ClawHub/plugin boundary item, and not superseded by the related overflow-budget PR #70473.

## Best Possible Solution

Keep the PR open and continue it as the scoped core fix for #70472: preserve the active session mapping on compaction/overflow recovery failure, remove the compaction-specific reset hook and dead test stubs, return explicit retry `/compact` `/new` guidance, and add regression tests for both embedded overflow payloads and thrown compaction-failure errors. Before merging, the author or maintainer should address the retry-loop/latch and role-ordering-state questions from the latest review and add the requested release-note/changelog coverage if this user-facing behavior changes.
