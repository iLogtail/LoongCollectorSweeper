## Summary

Keep #42273 open. Current main has improved atomic temp-file publication and cleanup, but it still archives the whole state directory, still uses tar file mode, and exposes no exclude, smart-exclude, max-size, or progress controls for multi-GB `.openclaw` installs. Related #42282/#42290 and #44288 track useful follow-up work, and Ryce's keepmyclaw plugin is a helpful workaround, but none of that has landed in core as the user-visible fix for this issue.

## Best Possible Solution

Keep #42273 open until core ships a user-visible fix in `src/infra/backup-create.ts` and `src/cli/program/register.backup.ts`: preserve the existing atomic temp publish path, add either robust streaming/progress/error handling or safe exclude/smart-exclude controls for large state subtrees, and update `docs/cli/backup.md` plus focused backup tests. If maintainers choose a narrower tracker, explicitly mark #42282 or a landed successor PR as canonical before closing this issue as duplicate/superseded.
