## Summary

Keep #42273 open. Current main does not implement the requested large-backup mitigation: the CLI still has no exclude/smart-exclude flags, the backup plan still includes the full state directory by default, and the archive path still writes all included roots through tar without a size/progress/exclusion policy. The related keepmyclaw plugin is useful prior discussion, and PR #44288 plus issue #42282 appear to track proposed follow-up work, but the core bug is not fixed on main.

## Evidence

- **CLI surface lacks large-backup exclude flags:** `backup create` registers `--output`, `--json`, `--dry-run`, `--verify`, `--only-config`, and `--no-include-workspace`; there is no `--exclude-logs`, `--exclude-media`, `--exclude`, `--exclude-file`, `--smart-exclude`, progress, or max-size option on current main.
