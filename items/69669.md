## Summary

Keep open. Current main still treats interactive thread/current-conversation ACP bindings as direct routing to the ACP harness, and the docs explicitly say normal bound follow-ups are sent as prompt text to the ACP session. That is the opposite of the issue's requested parent-orchestrated rewrite-by-default policy.

## Best Possible Solution

Keep the issue open for maintainer design. The likely path is to add explicit metadata/policy for parent-orchestrated ACP thread sessions, route shorthand follow-ups back through the parent session for synthesis, then send the expanded task to the bound ACP child. Preserve current direct prompt routing for explicitly interactive ACP binds or explicit verbatim forwarding, and update ACP docs plus Discord/Telegram binding tests to cover both modes.
