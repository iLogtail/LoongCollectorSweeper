## Summary

Keep open. Current main still treats bound ACP follow-ups as direct ACP prompt routing, and the regression test coverage explicitly asserts that a bound message body such as "continue" is sent unchanged to the ACP runtime without invoking the normal parent/model reply path.

## Best Possible Solution

Keep this issue open. The best fix is to define the intended policy for thread-bound ACP follow-ups, then update the dispatch/orchestration path and tests so shorthand user follow-ups are interpreted by the parent and sent as explicit child tasks by default, while preserving an explicit verbatim-forwarding escape hatch and updating ACP docs accordingly.
