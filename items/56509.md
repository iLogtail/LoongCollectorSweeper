## Summary

Keep PR #56509 open. The central cron bug is not implemented on current main: `onTimer` and `runDueJobs` still call `collectRunnableJobs` without `skipAtIfAlreadyRan`, while the guard only works when that option is passed. The PR is useful but should not be merged wholesale because it bundles unrelated WhatsApp, secrets, transcript repair, and env-security changes, including an unresolved P1 review comment on one unrelated change.

## Best Possible Solution

Keep this open or replace it with a narrow cron-only patch: pass `skipAtIfAlreadyRan: true` from `onTimer` and `runDueJobs`, add a regression test for a completed `schedule.kind === "at"` job with past `nextRunAtMs` and `lastStatus: "ok"`, and split/drop the unrelated WhatsApp, secrets, transcript repair, and env-security changes from this PR.
