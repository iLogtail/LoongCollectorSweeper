## Summary

Keep #69102 open. Current main has a native bundled Google/Gemini provider, but its direct Gemini request path only accepts temperature, maxTokens, thinkingConfig, cachedContent, systemInstruction, tools, and toolConfig; there is no `serviceTier`/`service_tier` plumbing or test/doc coverage for Gemini Flex. The related provided items do not supersede this request.

## Best Possible Solution

Keep the issue open and implement this in the bundled Google provider, not core. Add provider-local parsing for `serviceTier` and `service_tier` under `agents.defaults.models["google/<model>"].params`, validate supported Gemini service tiers, forward the value in the direct Gemini GenerateContent payload, and add focused tests plus a short `docs/providers/google.md` advanced-configuration section.
