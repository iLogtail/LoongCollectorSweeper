## Summary

Keep open. Current main still has the verbose hard-coded TUI footer and does not include the compact token/percent/progress-bar helpers proposed by the PR. The PR is plausibly useful but not obsolete: it also has unresolved review feedback around stale elapsed-time display and unknown-token coloring that should be addressed before maintainer acceptance.

## Best Possible Solution

Keep the PR open and ask for a small revision: add the compact footer helpers/rendering in the TUI, make unknown token pressure neutral, avoid duplicated percent math, refresh elapsed footer text from the same status-timer path or another bounded timer, and add focused formatter/TUI tests plus docs if the visible footer contract changes.
