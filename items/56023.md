## Summary

Keep PR #56023 open for maintainer judgment. Current main does not implement the proposed weighted-recall feedback loop or the three SkillFoundry skills, and the related ClawSweeper items do not supersede it. The skill additions look like ClawHub material, but the memory-core part is a real product/API change, not something I can safely auto-close as ClawHub-only because current memory search results do not expose a stable chunk id for the proposed feedback commands and the PR discussion has unresolved correctness concerns.

## Best Possible Solution

Split the work. Publish or iterate the $anvil, $gate-keeper, and $layer-check skills through ClawHub or a plugin skill bundle. If weighted recall should become part of memory-core, keep it as a focused memory-core PR that first settles the public result-id/feedback UX contract, applies weights consistently across hybrid/vector/FTS paths, re-sorts after weighting, handles long-lived process reload/concurrency, and adds docs plus targeted tests.
