## Summary

Keep PR #69982 open. Current main still has the reported listener-retention path in the delete confirmation popover: Cancel and Delete remove the popover element without removing the document capture listener, while only the outside-click handler unregisters it. The PR is still addressing a real current UI issue, although it should preferably add regression coverage and consider the same-button toggle path as well.

## Best Possible Solution

Keep this PR or a maintainer-owned replacement open and revise it into a complete UI fix: centralize popover dismissal so Cancel, Delete, outside click, and same-button toggle all unregister the document listener, then add a focused grouped-render test that spies on document addEventListener/removeEventListener for each dismissal path.
