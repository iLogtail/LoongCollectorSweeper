## Summary

Keep PR #69982 open. Current main still has the document click listener cleanup gap the PR describes: the delete confirmation popover removes the DOM node on Cancel/Delete, but only the outside-click path removes the capture listener from document.

## Best Possible Solution

Keep the PR open and either merge a cleaned-up version of its dismissPopover approach or apply the same fix directly on main with a focused regression test that proves Cancel, Delete, and outside-click all unregister the document click listener.
