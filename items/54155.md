## Summary

Keep open. Current main has partial mitigations for contributors named in #54155, including `seqByRun` cleanup, stale run-context sweeping, session maintenance/disk-budget controls, and opt-in transcript truncation after compaction. That is not enough for a conservative close: the session store cache still uses `structuredClone`, active load paths still retain cloned store objects, transcript reads can still materialize whole JSONL files, and the truncation behavior that would bound compacted transcripts is disabled by default. #54155 remains the better canonical tracker for the broader long-running Gateway RSS/session-accumulation problem.

## Best Possible Solution

Keep #54155 open as the canonical Gateway memory-growth tracker. The best fix direction is to remove or redesign the `structuredClone` session-store cache path, avoid retaining both serialized and object forms when not needed, stream or cap transcript reads on Gateway paths, make compaction/truncation and session maintenance behavior align with long-running defaults, and add focused regression/soak coverage for completed subagent/cron session cleanup and Gateway RSS plateau behavior. Narrower duplicates such as later gateway RSS reports should point back to #54155 until that broader fix ships.
