## Summary

Keep #71018 open. Current main has a Feishu/Lark streaming card path, but the shipped code only surfaces a transient tool-name status line and tests explicitly assert that the status is omitted from the final card close. That does not satisfy the issue’s requested tool execution area with tool name, parameters, result, and duration, and the report is recent with concrete version/environment details.

## Best Possible Solution

Keep the issue open for Feishu plugin owner review. The most concrete path is to implement a Feishu-owned tool execution card state in `extensions/feishu/src/reply-dispatcher.ts`, fed from existing `onToolStart`, `onToolResult`, and/or `onItemEvent` callbacks, with a regression test that asserts tool name, params/result summary, duration/status, and final card behavior. If maintainers decide the old `openclaw-lark` package is unsupported, document the 2026.4.25+ `@openclaw/feishu` upgrade path and ask the reporter to retest there rather than auto-closing now.
