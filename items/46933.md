## Summary

Keep open. Current main still contains the exact `status --deep` last-heartbeat rendering bug: it feeds `formatTimeAgo(...)` into another ` ago` suffix and still renders missing channel metadata as `unknown`. PR #46933 is not directly mergeable against current main because the status rendering path was refactored, but the underlying bug is still real and the PR remains useful as a focused fix/test direction.

## Best Possible Solution

Keep the PR open or have a maintainer take over the patch: update current `buildStatusLastHeartbeatValue` in `src/commands/status.command-sections.ts` to use `formatTimeAgo(...)` output directly and omit missing channel/account fields, then add focused coverage in `src/commands/status.command-sections.test.ts` or an extracted helper test. Close only after that equivalent fix lands on main.
