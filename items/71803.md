## Summary

Keep #71803 open. Current main still has CLI no-output watchdogs driven only by stdout/stderr activity, including the Claude live-session path used by `claude-cli`; there is no pending Monitor/MCP/tool-call expected-idle state that would suspend or heartbeat the watchdog during a long bounded wait.

## Best Possible Solution

Keep the issue open. The best maintainer path is to teach the CLI backend watchdog about pending Monitor/MCP tool-call liveness: suspend or heartbeat the no-output timer while a bounded Monitor task is pending, keep the overall run timeout enforced, and add focused regression coverage where a live claude-cli turn stays silent past `noOutputTimeoutMs` during a pending Monitor wait and then completes when the monitor emits output.
