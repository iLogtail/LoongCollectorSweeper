## Summary

Current main partially addresses the multi-agent slash-command problem through deterministic Discord routing, bound ACP sessions, and `CommandTargetSessionKey`, but it does not implement the issue's requested `/new @AgentName` or `/reset @AgentName` targeting, nor the proposed smart default based on context usage or active participants. This should remain open as a real remaining product gap rather than being closed as implemented.

## Best Possible Solution

Keep the issue open. The best fix is to add an explicit, authorized target-resolution path for Discord native `/new` and `/reset`: accept a user/agent/session target option or mention-compatible input, resolve it through existing agent/binding/session routing helpers, apply consistent channel and sender authorization, and add Discord native-command tests plus docs. If maintainers prefer not to implement the heuristic smart default, document deterministic binding-based default behavior and track the heuristic separately.
