## Summary

Keep open. Current main has useful inbound debounce infrastructure and some channel-specific batching, but it does not implement the reported multi-part multimodal aggregation generally, and the QQ path still queues and processes each inbound message separately. The public docs also state that `messages.inbound` debouncing is text-only and that media/attachments flush immediately, which leaves the image-followed-by-text use case requested in #60190 unresolved.
