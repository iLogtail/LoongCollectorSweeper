## Summary

Keep #60190 open. Current main has a useful inbound debounce framework, and BlueBubbles has an opt-in coalescing path, but the broad requested behavior for ordinary image/media plus follow-up text is still not implemented across affected bundled channels. The shared/default policy explicitly excludes media, Discord and Telegram ordinary messages preserve that exclusion, and QQBot serializes messages per peer without batching them into one agent turn.

## Best Possible Solution

Keep the issue open and implement this as a shared, media-aware channel ingress capability rather than a one-off field name. The likely path is to extend/reuse `createInboundDebouncer` so channel plugins can opt into same-sender aggregation that carries text plus media, preserves reply/thread metadata, records all source message IDs for dedupe, and remains configurable through `messages.inbound.byChannel` or a clearly documented per-channel opt-in. Start with QQBot plus at least one helper-backed channel test, then update `docs/concepts/messages.md` to distinguish text-only debounce from media-aware aggregation.
