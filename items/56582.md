## Summary

Keep #56582 open. Current main still accepts trailing-at ClawHub specs because `parseClawHubPluginSpec` returns `{ name: spec }` when `atIndex >= spec.length - 1`, so `clawhub:demo@` still proceeds into the ClawHub install lookup path instead of returning `invalid_spec`. The PR's idea remains valid, though it now needs to be rebased onto the current split-out parser file.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer patch that updates `src/infra/clawhub-spec.ts` to reject explicit empty-version specs while preserving `clawhub:@scope/pkg` and `clawhub:@scope/pkg@1.2.3`, with regression coverage in `src/infra/clawhub.test.ts`.

## Evidence
