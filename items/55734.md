## Summary

Keep #55734 open. Current main still has the risky default-selection behavior: raw CDP tab listing admits every target with an id, and default tab selection picks the first `type: "page"` candidate without excluding `chrome://`, `chrome-untrusted://`, or top-chrome UI URLs. The related PR #55740 proposed the right kind of fix but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and land a focused browser-plugin fix: add a shared user-navigable tab predicate that rejects Chrome internal UI targets such as `chrome://...`, `chrome-untrusted://...`, and top-chrome surfaces; apply it to raw CDP `/json/list`, Playwright-backed listing/default selection, and managed tab cleanup where appropriate; fall back to opening `about:blank` when only internal UI targets are present; add regression tests using the target list from #55734.
