## Summary

Keep open. Current `main` still terminates the PowerShell session on installer failure paths, and the user-facing Windows docs still recommend the `iwr ... | iex` invocation from the report.

## Evidence

- **Failure path still uses `exit 1` after `Ensure-Node` fails:** `Main` still does `if (!(Ensure-Node)) { exit 1 }`, so the no-Node/no-package-manager path reported in the issue still ends in a process exit.
  - file: [scripts/install.ps1:400](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/scripts/install.ps1#L400)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **The installer no longer exits inside `Install-Node`, but only returns failure:** `Install-Node` now falls back through winget/chocolatey/scoop and then returns `$false` after printing manual-install guidance. That means the sharp edge was moved to the caller, not removed end-to-end.
  - file: [scripts/install.ps1:146](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/scripts/install.ps1#L146)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **`exit 1` runs at top-level installer evaluation:** The script ends with a direct `Main` call, so the `exit 1` sites in `Main` execute in the same evaluated script session used by `iex` / `scriptblock` invocation.
  - file: [scripts/install.ps1:451](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/scripts/install.ps1#L451)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Current docs still recommend the affected invocation:** The Windows install docs still present `iwr -useb https://openclaw.ai/install.ps1 | iex` as the primary command, which is the exact host-session pattern described in the bug report.
  - file: [docs/install/index.md:27](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/docs/install/index.md#L27)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **PowerShell `exit` still aborts the surrounding evaluated session:** A subprocess check with `pwsh -NoLogo -NoProfile -Command "& ([scriptblock]::Create('function Main { Write-Host before; exit 1; Write-Host after }; Main')); Write-Host outer"` printed only `before` and exited nonzero, never reaching `outer`. That matches the reported host-termination behavior of `exit` inside an evaluated scriptblock.
  - command: `pwsh -NoLogo -NoProfile -Command "& ([scriptblock]::Create('function Main { Write-Host before; exit 1; Write-Host after }; Main')); Write-Host outer"`
- **Latest release still carries the same installer blob:** `HEAD:scripts/install.ps1` and `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) resolve to the same blob `7078c95dc61a624cdcdce14a201b6258bd0e757a`, so this is not already fixed in the latest shipped release either.
  - file: [scripts/install.ps1](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/scripts/install.ps1)
