## Summary

Keep #39857 open. Current main still lets configured or persisted context-token overrides replace a smaller known model window in several user-visible/session paths, and the linked fix PR #39859 remains open/unmerged in the provided context.

## Best Possible Solution

Keep #39857 open until the resolver and all reporting/persistence callers treat agents.defaults.contextTokens or stored contextTokens as a ceiling when a smaller selected-model window is known. PR #39859 appears to be the focused implementation path; maintainers should review or replace it with an equivalent fix plus regression coverage for resolver, status, and session-store behavior.
