## Summary

Keep #39857 open. Current main still lets positive context-token overrides win before known model-window resolution, and the reply, status, and session persistence paths still feed `agents.defaults.contextTokens` or persisted session values through that behavior. The related fix PR #39859 is still separate from `main` in the provided context and remote refs, so the reported correctness bug is not implemented on main.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent maintainer-approved fix lands. The fix should make `resolveContextTokensForModel()` treat overrides and stored values as ceilings when a smaller known selected model window is available, preserve existing behavior when the model window is unknown, and add focused regression coverage for the shared resolver, `/status`, status summary/session tools, and persisted session metadata.
