## Summary

Keep #61428 open. Current main still does not support `hooks.mappings[].sessionTarget`: the config type/schema omit it, mapping resolution/action dispatch do not thread it, and hook agent dispatch still hardcodes cron jobs to `sessionTarget: "isolated"`. The related #58023 is closed unmerged and was explicitly superseded by this smaller PR, so it is not a canonical open replacement.

## Best Possible Solution

Keep the PR open or ask the author to revise it. A mergeable solution should add `sessionTarget` to the hook config type, strict Zod schema, generated config metadata/help/labels, docs, mapping normalization/action/transform/merge, replay-cache scope, effective session-prefix validation, and tests for default isolated behavior, `sessionTarget: "main"`, custom `agentId`, idempotency, and allowed-prefix policy. Maintainers should also decide whether hook agent turns are allowed to target main sessions despite cron's current main-session constraints.
