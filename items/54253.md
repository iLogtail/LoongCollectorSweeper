## Summary

Keep open. Current main does not provide enough evidence to close this RISC-V64 LLM network failure as fixed, duplicated, or non-actionable. The model request path still goes through guarded Undici fetch/dispatchers, and the related #58150 remains open with a more specific report that direct HTTP succeeds while OpenClaw fails on RISC-V64. The proxy comments are useful: current service generation intentionally forwards proxy env vars into systemd/launchd units, which explains several follow-up reports, but it does not resolve the original RISC-V64 failure.

## Best Possible Solution

Keep #54253 open and consolidate investigation with #58150. Maintainers should separate two paths: verify the RISC-V64 Undici/fetch failure on current OpenClaw with Node 24/22.14+ and direct fetch probes, and separately decide whether daemon install should warn/doctor inherited proxy env in systemd units. A fix likely belongs in the provider/network transport layer or a documented/doctor-supported runtime workaround once the RISC-V root cause is reproduced.
