## Summary

Keep #71216 open. Current main partially covers the sandbox ask via `agents.defaults.sandbox.mode`, but it does not implement the requested `routing.rules`, `instances`, or `gateway.nodes.denyPaths` config/runtime enforcement. The related reports are adjacent config/schema work, not canonical duplicates.

## Best Possible Solution

Keep the issue open for maintainer design. A practical path is to acknowledge existing `agents.defaults.sandbox.mode`, then split or refine the remaining asks: decide where tag-based provider routing belongs, decide whether `instances.*` should complement profile-based multi-gateway setup, and implement any `denyPaths` policy with TypeScript types, strict Zod/generated schema, docs, doctor/audit coverage, and runtime enforcement tests.
