## Summary

Keep #71216 open. Current main has adjacent primitives, especially `agents.defaults.sandbox.mode` / per-agent sandboxing and channel/account `bindings`, but it does not implement the requested top-level `sandbox`, tag-based `routing.rules`, `instances.*`, or `gateway.nodes.denyPaths` schema/enforcement. The remaining asks are core config/security/runtime design work, not a ClawHub-only plugin cleanup item.

## Best Possible Solution

Keep the issue open and have maintainers split or scope the work: document the existing `agents.defaults.sandbox.mode` answer for the sandbox portion, then design separate core changes for provider/tag routing policy, multi-instance config validation, and gateway-enforced path deny rules. `gateway.nodes.denyPaths` especially belongs in core/security enforcement rather than a plugin because it must apply at the gateway chokepoint regardless of caller.
