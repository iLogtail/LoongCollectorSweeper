## Summary

Keep #38622 open. Current main still routes workspace bootstrap files through `openBoundaryFile` with the workspace as the boundary, returns boundary failures directly, and converts those failures into missing-file markers. A symlinked `SOUL.md` or `AGENTS.md` whose target resolves outside the workspace still matches the reported `[MISSING]` behavior, and the related fixes are not merged on main.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/workspace.ts`: when a standard workspace bootstrap file path exists as a symlink, resolve and read the target only for bootstrap injection while preserving `MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES`, dangling-symlink missing behavior, and hardlink rejection. Add regression tests for relative and absolute symlinks to shared workspace files, dangling symlinks, and hardlink aliases.
