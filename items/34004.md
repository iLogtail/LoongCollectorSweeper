## Summary

Keep open. Current main still only implements the narrower Discord `set-presence` action behind `channels.discord.actions.presence`; it does not implement the requested `self-profile`/`selfProfile` action, nickname/avatar mutation, self-only target guards, multi-account `accountId` requirement for profile mutation, or `bestEffort` partial profile results. The related PR #34010 was closed unmerged per the provided GitHub context, so it does not supersede this issue.

## Best Possible Solution

Keep #34004 open. If maintainers want this in core, implement it in the bundled Discord plugin as a scoped `self-profile` message action behind `channels.discord.actions.selfProfile`, with self-only target checks, explicit account routing for multi-account configs, strict avatar validation, gateway preflight for presence updates, tests, and docs. If maintainers prefer an external plugin, first define or document the plugin/API path that can safely operate on the relevant Discord bot account/runtime.
