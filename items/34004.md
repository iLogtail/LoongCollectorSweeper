## Summary

Current main still does not implement the requested Discord self-profile action. It has Discord presence and autoPresence support, but no `channels.discord.actions.selfProfile` gate, no `self-profile` message action discovery/dispatch, and no nickname/avatar self-mutation path. The related PR #34010 is closed unmerged, and related issue #33270 is closed rather than an open canonical tracker for this narrower remaining work.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped `self-profile` message action behind `channels.discord.actions.selfProfile` with nickname/avatar/status/activity support, strict bot-self target validation, explicit account handling for multi-account configs, avatar MIME/base64/10MB validation, gateway preflight for presence fields, focused tests, and Discord docs. If maintainers prefer ClawHub, record that product decision and link or define the plugin/API path needed to safely own Discord bot profile mutation outside core.
