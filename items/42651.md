## Summary

Keep open. Current main has substantial adjacent memory work: `openclaw memory search/index/status/promote/promote-explain/rem-harness/rem-backfill`, recall tracking, promotion preview/apply, memory-wiki ingestion/apply, and an active-memory toggle. It still does not implement the issue's explicit operator-facing MVP command set (`remember`, `recall`, `review`, `edit`, `forget`) or first-class review/edit/forget memory flows in the active memory pipeline.

## Best Possible Solution

Keep #42651 open, but narrow it before coding. A maintainer should define the v1 boundary for `extensions/memory-core`: likely map `recall` to `memory search`, `review` to promotion/dreaming candidate review, add a small `remember` write path only if it can preserve the Markdown memory contract, and either implement or explicitly defer first-class `edit`/`forget` to the already-related #42650 work. Ingestion helpers should reuse existing `rem-backfill`, daily-note, session, and memory-wiki source paths instead of creating a parallel memory store.
