## Summary

Keep PR #69954 open. Current main still has the cleanup behavior described by the PR and linked issue #50248: `--fix-missing` trusts a structurally valid but stale `sessionFile` and prunes when that path is absent, without checking the canonical `<sessionId>.jsonl` transcript. The PR is not redundant, though its current diff should address the review finding that repair-only runs can still report `wouldMutate: false`.

## Best Possible Solution

Keep the PR open, or land an equivalent targeted fix in `src/commands/sessions-cleanup.ts`: when the persisted `sessionFile` is missing, check the canonical transcript path derived from `sessionId`; if it exists, keep the entry and repair `entry.sessionFile`. The implementation should also return/report repair counts so `missing`, `wouldMutate`, and applied summaries remain accurate, with regression coverage in `src/commands/sessions-cleanup.test.ts` for dry-run, applied repair, true-missing pruning, and repair-only JSON output.
