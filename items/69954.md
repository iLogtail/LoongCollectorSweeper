## Summary

Keep PR #69954 open. Current main still appears vulnerable to the reported cleanup bug: `sessions cleanup --fix-missing` resolves and checks only the stored `sessionFile` path, so a structurally valid but stale missing path can still cause pruning even when the canonical `<sessionId>.jsonl` transcript exists. The PR is a focused bug fix for open issue #50248, but it should also address the bot review finding that repair-only runs can mutate metadata while reporting `wouldMutate: false`.

## Best Possible Solution

Keep the PR open and revise it, or land an equivalent fix, so cleanup checks the canonical transcript path before pruning a stale stored `sessionFile`, repairs the stored path when the canonical transcript exists, and reports repair-only dry-run/apply mutations accurately in JSON summaries. Add focused tests for dry-run, applied repair, true missing transcripts, and `wouldMutate`/repair count behavior.
