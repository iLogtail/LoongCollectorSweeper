## Summary

Keep #70344 open. Current main does not implement the central requested behavior: tool-use assistant messages without commentary metadata are still not suppressed, and `sessions_send` still starts the A2A ping-pong/announce flow with `delivery.mode: "announce"`. The PR is also not safe to close as obsolete or merge as-is because the review discussion flags real follow-up work, including preserving stored external routes while suppressing only the current turn and removing a branch-only hardcoded-name sanitizer artifact.

## Best Possible Solution

Keep the PR open but require a revised implementation: add a narrow source-backed tool-use suppression guard and regression test for phase-missing tool-use messages; handle `sessions_send` as session-local for the current turn without corrupting persisted external route fields; remove the broad/hardcoded sanitizer heuristic or replace it with a stronger, well-tested extraction boundary; then update the existing sessions tests that currently assert announce delivery.
