## Summary

Keep #70344 open. Current main has only the commentary-phase suppression piece; it still runs sessions_send A2A announce/ping-pong delivery, preserves established external routes for all inter-session turns, and lacks the PR's stopReason/tool-block suppression and sanitizer changes. The PR is not auto-closeable or fully superseded, but it needs revision before merge because review comments identified concrete regressions/artifacts in the proposed patch.

## Best Possible Solution

Keep the PR open for maintainer/product review and ask for a narrower revision: suppress tool-use/prelude delivery with targeted tests, isolate sessions_send current-turn external delivery without corrupting persisted lastChannel/lastTo, remove the sanitizer artifact and tighten heuristics, and update tests/docs if sessions_send's documented A2A announce behavior is intentionally changing.
