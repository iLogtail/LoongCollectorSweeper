## Summary

Keep open: current `main` still does not implement the requested GitHub-style inline image rendering for Control UI agent replies. In `ui/src/ui/markdown.ts`, the renderer explicitly restricts Markdown images to `data:image/...;base64,...` sources and falls back to escaped alt text for any other image URL, so normal `![alt](https://...)` Markdown remains blocked. ([github.com](https://github.com/openclaw/openclaw/blob/main/ui/src/ui/markdown.ts))

## Evidence

- **Renderer explicitly blocks non-data-URI Markdown images:** `ui/src/ui/markdown.ts` contains an explicit comment, `Override image to only allow base64 data URIs`, then checks `if (!INLINE_DATA_IMAGE_RE.test(src)) { return escapeHtml(alt); }`, which means standard remote or relative Markdown image URLs are not rendered inline. ([github.com](https://github.com/openclaw/openclaw/blob/main/ui/src/ui/markdown.ts))

  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **Current allowlist is narrower than GitHub-flavored Markdown image syntax:** The same renderer defines `INLINE_DATA_IMAGE_RE = /^data:image\/[a-z0-9.+-]+;base64,/i` and only returns an `<img class="markdown-inline-image" ...>` when that regex matches.
