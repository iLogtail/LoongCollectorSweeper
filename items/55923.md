## Summary

Keep open. Current `main` does not implement the PR’s core behavior changes: the embedded runner still lacks explicit downgraded-tool/placeholder-reply detection, web chat still hides `state: "error"` events in `lastError` instead of appending a visible message, and the gateway websocket handler still does not fast-reject legacy query-auth URLs before the normal connect-challenge timeout path.
