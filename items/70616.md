## Summary

Keep #70616 open. Current main has partial internal dreaming execution types, but the public memory-core plugin schema still rejects a top-level `dreaming.execution`, the requested fallback model list is not represented in the dreaming config, and the managed dreaming runtime does not pass model/fallback choices into the generated cron job or narrative subagent calls. The request is actionable in the bundled memory-core plugin rather than a ClawHub close: core cron already has per-job `model` and `fallbacks` payload support that memory-core could wire through.

## Best Possible Solution

Keep the issue open and implement it in memory-core: define a public `dreaming.execution` schema shape, decide whether it uses `model`/`fallbacks` or aligns exactly with cron agent-turn payloads, normalize it in `src/memory-host-sdk/dreaming.ts`, carry it through `resolveShortTermPromotionDreamingConfig`, include `model` and `fallbacks` in the managed cron payload and relevant subagent narrative calls, then update docs, generated config metadata, and tests covering schema acceptance plus runtime forwarding.
