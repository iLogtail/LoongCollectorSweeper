## Summary

Keep #70616 open. Current main has partial internal dreaming execution normalization, but the public `memory-core` config schema still omits `dreaming.execution`, docs still describe Dream Diary generation as using the default runtime model, and the active dreaming runtime path does not pass model or fallback settings into the narrative subagent run. The requested per-dreaming primary/fallback model support remains actionable core/plugin work.

## Best Possible Solution

Keep the issue open and implement a first-class, documented memory-core dreaming execution contract. The likely path is to add `plugins.entries.memory-core.config.dreaming.execution` to the plugin manifest/schema/docs/tests, choose a canonical primary/fallback shape aligned with existing `{ primary, fallbacks }` model config, plumb it through `resolveMemoryDreamingConfig` into the active dreaming and narrative subagent paths, and verify model override/failover behavior with focused memory-core tests.
