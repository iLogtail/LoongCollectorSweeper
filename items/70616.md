## Summary

Keep #70616 open. Current main still has the reported memory-core dreaming config gap: the strict plugin schema rejects `dreaming.execution` and per-phase `execution`, while runtime helpers partially resolve execution settings and the managed dreaming paths still do not apply model/fallback overrides.

## Best Possible Solution

Keep the issue open and fix it in memory-core/core config: decide the public `dreaming.execution` shape, add manifest schema and tests for global and per-phase execution, update user-facing memory/dreaming docs, and wire resolved model/fallback settings into the managed dreaming cron payload and dream narrative subagent path. If fallback chains are required for narratives too, add or reuse a typed plugin subagent fallback override instead of only changing validation.

## Evidence
