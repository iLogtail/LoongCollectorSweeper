## Summary

Keep open. Current main appears to have fixed the installed-plugin module-resolution half with jiti aliases for user-installed plugins, and it has a deprecated root-alias/compat layer. But that compat layer still does not expose two issue-named symbols, `resolvePreferredOpenClawTmpDir` and `normalizeAccountId`, through `openclaw/plugin-sdk`; only `buildChannelConfigSchema` is covered via the compat channel-config-schema export. The linked broader compatibility discussion does not make this exact openclaw-weixin failure obsolete because the current source still lacks the requested root-compat exports.

## Best Possible Solution

Keep the issue open and either add a narrow legacy compat export/test for the exact symbols used by affected external plugins, or confirm and document that `@tencent-weixin/openclaw-weixin` has released a fixed version importing `resolvePreferredOpenClawTmpDir` from `openclaw/plugin-sdk/diffs` or `temp-path`, `buildChannelConfigSchema` from `openclaw/plugin-sdk/channel-config-schema`, and `normalizeAccountId` from `openclaw/plugin-sdk/account-id` or `core`. The existing loader alias tests already cover the module-resolution half, so the remaining maintainer work is the export compatibility decision and regression coverage.
