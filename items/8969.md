## Summary

Keep #8969 open. Current main ships the first requested quick win as `openclaw skills check`, but the remaining status-surface and skill prompt-rewrite diagnostics are still not implemented on current main.

## Best Possible Solution

Keep #8969 open, or split it into two focused follow-ups. Preserve `openclaw skills check` as the shipped implementation for the first quick win. Add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`, then emit an operator-visible diagnostic when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus public docs updates.
