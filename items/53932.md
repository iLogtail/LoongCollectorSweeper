## Summary

Current main still does not provide a Slack opt-out for outbound image optimization. Slack uploads call the shared outbound media loader, the shared loader defaults `optimizeImages` to true and optimizes non-GIF images before returning the upload buffer, and Slack config/docs expose only `mediaMaxMb` for this surface. The related PR #53988 directly addressed the request but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #53932 open. The best product path is a Slack-owned delivery option, likely a documented `channels.slack.optimizeUploads` or `channels.slack.preserveOriginalUploads` boolean with account-level inheritance, wired through `sendMessageSlack` into `loadOutboundMediaFromUrl`/`loadWebMedia` as `optimizeImages: false`, plus regression coverage showing PNG bytes/content type are preserved when disabled and current optimized behavior remains the default.
