## Summary

Keep open. Current main still does not provide the requested per-channel tool-error visibility control, and the existing `messages.suppressToolErrors` path still leaves mutating tool failures such as `message` visible on chat surfaces.

## Best Possible Solution

Keep this issue open for a core runtime/config change: define an explicit tool-output visibility policy with a global default and per-channel overrides, wire it through chat dispatch and embedded-run payload shaping so mutating `message`/`exec` errors can be hidden from users while still returned to the agent, then update schema/docs and add focused tests for `buildEmbeddedRunPayloads` plus a streaming channel path such as Telegram.
