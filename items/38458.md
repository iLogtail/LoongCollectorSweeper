## Summary

Keep open. Current main still lacks the requested per-channel tool-error visibility control, and the existing global `messages.suppressToolErrors` path is narrower than the issue: mutating tool failures such as `message` remain user-visible, with tests asserting that behavior.

## Best Possible Solution

Keep this issue open for a core runtime/config change: define an explicit tool-output visibility policy with a global default and per-channel overrides, wire it through embedded-run payload shaping and streaming chat dispatch so mutating `message`/`exec` failures can be hidden from users while still returned to the agent, then update schema/docs and add focused tests for `buildEmbeddedRunPayloads` plus a streaming channel path such as Telegram.
