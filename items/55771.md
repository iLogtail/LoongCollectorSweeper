## Summary

Keep #55771 open. Current main still exposes `before_model_resolve` as an override-only plugin hook, and the runtime does not support terminal `block` / `blockReason` semantics for that hook. The linked budget-guard plugin context is a legitimate external plugin use case that needs a core SDK/runtime contract, not a ClawHub-only implementation detail.

## Best Possible Solution

Keep the issue open and implement this as an additive core plugin SDK/runtime change: add `block?: boolean` and `blockReason?: string` to `PluginHookBeforeModelResolveResult`, give `runBeforeModelResolve` terminal block merging like `before_tool_call`, propagate the blocked result through `resolveHookModelSelection` before model resolution, surface a user-visible reason, and add docs plus focused tests for priority ordering, thrown-hook fail-open behavior, and interaction with legacy `before_agent_start` and model fallback.
