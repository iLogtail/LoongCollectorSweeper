## Summary

Keep #72081 open for maintainer judgment. Current main still prints/logs `Dashboard URL:` without `#token=...`, which matches the reported visible behavior, while separately sending the tokenized URL to browser launch and clipboard for non-SecretRef tokens. That looks intentional and security-backed rather than a clearly implemented/cannot-reproduce close.

## Best Possible Solution

Keep the security behavior that avoids printing gateway bearer fragments. Maintainer follow-up should either close this manually as intended behavior, or improve the dashboard/headless UX so users who cannot use browser launch or clipboard get clear instructions to paste `gateway.auth.token`/`OPENCLAW_GATEWAY_TOKEN` or use a future non-sensitive bootstrap-link flow instead of logging the bearer token.
