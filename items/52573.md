## Summary

Current main documents Kimi as an AI-synthesized web_search provider, but the runtime still accepts the model's final natural-language message as a successful tool result when Kimi stops tool-calling, including with no collected citations. That matches the core failure mode reported in #52573, so this should not be closed as implemented or superseded by the docs-only clarification #52575.

## Best Possible Solution

Keep #52573 open. The maintainer-quality fix belongs in the Moonshot/Kimi web-search provider: preserve the documented synthesized-answer mode, but make successful Kimi web_search require grounding evidence such as search_results/citations or a valid native tool round, return an explicit provider failure for ungrounded/generic fallback answers, and add focused tests simulating no tool calls/no citations/refusal-style content. The existing docs from #52575 can stay as provider-behavior clarification.
