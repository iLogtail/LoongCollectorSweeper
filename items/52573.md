## Summary

Keep open. Current main documents Kimi as an AI-synthesized `web_search` provider, but the reported runtime contract problem remains: when Kimi stops tool-calling, OpenClaw still returns the model's final natural-language text as a successful search payload even if no citations or structured search results were collected. The linked docs follow-up #52575 appears addressed by docs, but that does not implement the requested explicit failure/guardrail for non-grounded fallback answers.

## Evidence

- **Runtime still accepts final free-form model text as success:** `runKimiSearch` returns `{ done: true, content: text ?? "No response", citations: [...] }` whenever Kimi's choice is not a tool-call turn, without requiring `search_results`, citations, or detecting generic fallback text such as inability-to-browse responses.
