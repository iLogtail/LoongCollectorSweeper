## Summary

Keep open. Current main still has the backup planning implementation in core, and plugins can call the broad `openclaw/plugin-sdk/runtime` `createBackupArchive` helper, but there is no `openclaw/plugin-sdk/backup` subpath and no public SDK export for `BackupPlan`, `resolveBackupPlanFromDisk`, or the archive-name helpers requested by this PR. The old review concern about `tsdown.config.ts` is no longer the same blocker on current main because build entries now come from `scripts/lib/plugin-sdk-entrypoints.json`, but the PR still needs rebase, docs/API-contract alignment, and maintainer review before it can land.

## Best Possible Solution

Keep the PR open, but require a current-main revision that adds a narrow `src/plugin-sdk/backup.ts` subpath only if maintainers want this as a supported plugin contract. The revised change should update `scripts/lib/plugin-sdk-entrypoints.json`, `package.json` exports, docs under `docs/plugins/*`, and plugin-SDK contract/API checks, and should re-evaluate whether the public surface should expose the raw `BackupPlan` or a narrower stable backup-plan adapter.
