## Summary

Keep open. Current `main` still strips documented sandbox `docker.env` API keys, so the PR is addressing a real bug, but the latest PR patch still introduces a config-regression by bypassing `skills.entries.<skill>.enabled === false` for skills named in `skillFilter`.

## Evidence

- **Docs direct users to put sandboxed skill secrets in `docker.env`:** The docs explicitly say sandboxed skills do not inherit host `process.env` and tell users to use `agents.defaults.sandbox.docker.env` (or per-agent equivalent) for skill API keys.

## Risks / Open Questions

- Closing this PR would drop a real docs-vs-runtime bug: current `main` still strips documented sandbox `docker.env` API keys for sandboxed skills.
- Merging the PR as it stands would re-enable skills that operators explicitly disabled via `skills.entries.<skill>.enabled = false` whenever those skills remain listed in `skillFilter`.
