## Summary

Keep #54677 open. Current main still lacks the Hetzner warning the PR adds, the repo default Compose file still publishes gateway/bridge ports without a host-IP prefix, and Docker's Compose merge rules make the described override footgun real. ([docs.docker.com](https://docs.docker.com/reference/compose-file/merge/))

## Best Possible Solution

Keep the PR open and either merge this focused warning or fold the same warning into the Hetzner Docker Compose section, with similar wording considered for other VPS guides that show loopback-only Compose snippets. There is no conservative cleanup close reason because the requested docs protection is still absent on main.
