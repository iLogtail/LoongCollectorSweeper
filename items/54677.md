## Summary

Keep open. Current main still lacks the Hetzner warning this PR adds, and the underlying docs footgun is real: the Hetzner guide shows a loopback-only mapping while the repository default Compose file publishes gateway and bridge ports without a host address. The PR is a focused docs change with a positive review note, and the follow-up commit addressed the only style suggestion by making it a warning callout.

## Best Possible Solution

Keep the PR open and, if it still applies cleanly, land the warning in docs/install/hetzner.md near the Hetzner Compose snippet. The best product outcome is to make the loopback-only VPS guidance explicit about replacing Compose ports rather than layering an override that preserves public bindings.
