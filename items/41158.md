## Summary

Keep open. Current `main` does not implement attempt-boundary retry for transient rate limits before failover. The embedded runner still calls `activeSession.prompt(...)` directly, then routes prompt-side `rate_limit` errors into auth-profile rotation/model fallback. Related fixes on `main` improve failover classification and overload pacing, but they do not add the behavior this PR proposes.
