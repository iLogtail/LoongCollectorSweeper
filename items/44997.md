## Summary

Keep #44997 open. Current main has optimistic base-hash conflict detection for config replacement, but it does not serialize `openclaw.json` read/transform/write or provide the PR's fresh-state `updateConfigFile` path. `agents add` still computes the agent entry and bindings from a snapshot taken before the commit path, so the PR remains a plausible focused fix for the concurrent `agents add` lost-update part of #43367.

## Best Possible Solution

Keep the PR open and either rebase/port its core idea onto current main or implement an equivalent current-main fix: serialize the global config read-transform-write path for `agents add`, apply the agent mutation to freshly read state inside the critical section, preserve hardened directory permissions and runtime snapshot invalidation, and add a regression test showing concurrent `openclaw agents add` invocations persist every agent instead of overwriting or failing one writer.
