## Summary

Keep PR #68924 open. Current main has not implemented the central changes: the overview card still uses only `usageResult` for the cost value and token/message hint, `usageCostSummary` is loaded in state but not threaded into the overview card props, and the English UI copy still says `Cost`, `Daily Cost`, `Cost by Type`, and `Avg Cost / Msg`. The PR is not obsolete or duplicated, but it still needs maintainer/code review because the discussion raises substantive concerns about 7d/30d hint scope, calendar-window behavior, incomplete cost data, and preserving i18n/model-auth behavior.

## Best Possible Solution

Keep the PR open for maintainer review or revision. A mergeable version should keep the useful wording change, thread `usageCostSummary` into overview cards, and add focused coverage for the cost-card data scope, calendar-window calculations, incomplete-cost fallback, i18n labels, and preservation of the existing model-auth card.
