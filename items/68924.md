## Summary

Keep PR #68924 open. Current main still has the ambiguous dashboard wording and does not thread usageCostSummary into the overview cards, so the PR is not implemented or obsolete. The PR also has substantive review concerns around 7d/30d cost hint correctness and missing-cost handling, so the best maintainer path is revision/review rather than cleanup closure.

## Best Possible Solution

Keep the PR open and revise it as a focused UI change: keep the Usage Cost wording through i18n, pass usageCostSummary into overview cards, compute or label cost windows against real calendar/range semantics, align the card value and hint to the same data scope, gate cost hints when missingCostEntries indicates incomplete data, and add narrow overview-card tests.

## Risks / Open Questions

- Merging the PR as-is could mislabel cost windows when the loaded usage range is not a full 7 or 30 days.
- Sparse daily cost data can make last-N-entry totals cover older calendar periods than the label implies.
- Missing cost entries can display false $0.00 summaries unless the hint is gated or falls back to token/message context.
- The overview-card behavior currently lacks focused tests for usageCostSummary window hints and missing-cost cases.
