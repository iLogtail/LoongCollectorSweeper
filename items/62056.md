## Summary

Keep open. Current main still only logs Feishu `im.chat.member.bot.added_v1` and `im.chat.member.bot.deleted_v1`; it does not emit internal hook events or typed plugin hook events for bot added/removed. The requested behavior is also not a clean ClawHub close, because the source events are swallowed inside the bundled Feishu monitor and current hook contracts do not expose arbitrary `channel:feishu:*` events for an external hook/plugin to observe.

## Best Possible Solution

Implement this as a Feishu-owned, additive hook contract: emit bot added/removed events from `extensions/feishu/src/monitor.account.ts`, include `accountId`, `chatId`, timestamp, and best-effort `chatName`, keep the existing logs, add focused Feishu monitor tests that invoke both Lark event types, and document the new hook events in the public hooks/Feishu docs. If maintainers want the literal `channel:feishu:*` event names, first extend the internal/plugin hook event model in a backwards-compatible way rather than treating this as an external ClawHub-only plugin.
