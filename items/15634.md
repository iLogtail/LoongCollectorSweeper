## Summary

Keep #15634 open. Current main has shipped substantial Mattermost parity improvements, including markdown-aware delivery, reactions, threading, media upload, typing indicators, and interactive buttons, and the issue discussion correctly links #19957 as covering rich interactions. The remaining read receipts / presence scope is still not implemented or documented for Mattermost, so the umbrella should not be closed yet.

## Best Possible Solution

Keep #15634 open as the Mattermost parity umbrella for now, or create/link a narrower Mattermost read receipts / presence issue and close this umbrella only after that scope is implemented, documented, or explicitly descoped by a maintainer. Keep the already-shipped rich interaction work from #19957 as the canonical evidence for the completed button/rich-interaction portion.
