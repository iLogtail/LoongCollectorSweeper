## Summary

Current main now directly addresses the reported heartbeat prompt-shaping failure. GPT-5/Codex runs get explicit model-facing guidance that heartbeats are proactive wake-ups, concrete HEARTBEAT.md work should be executed, quiet monitoring is not enough for ongoing work, and status-report style summaries should not end the wake when actionable work remains. Provider/runtime tests verify the overlay is attached for OpenAI/Codex and native Codex app-server runs, and the implementation is present in v2026.4.24.

## Evidence

- **Heartbeat work-loop guidance:** The GPT-5 prompt overlay now says to treat heartbeats as proactive wake-ups, execute concrete HEARTBEAT.md work, reject quiet monitoring as sufficient for active workstreams, avoid ending after mere checking/orientation, prefer meaningful action over commentary, and treat useful heartbeat progress as often silent rather than visible status output.
