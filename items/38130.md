## Summary

Keep open. Current `main` already disables Slack SDK auto-reconnect, which addresses the retry-storm part of the PR, but it still does not treat `not_allowed_token_type` as a non-recoverable auth error. The PR is therefore only partially implemented on `main` and still contains useful behavior change.
