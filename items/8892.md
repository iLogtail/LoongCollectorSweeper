## Summary

Keep #8892 open. Current main still does not provide a native `openclaw tui --agent <id>` startup selector. The available paths are full `agent:<id>:<session>` keys via `--session`, CWD workspace inference, and post-launch `/agent`; the discussion’s PR #40176 workaround is useful but explicitly does not satisfy the requested flag for generic directories.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native startup selector for the TUI. The likely path is to add `--agent <id>` in `src/cli/tui-cli.ts`, add a typed `agentId` field to `TuiOptions`, validate it against configured agents, define precedence versus full `--session agent:<id>:...` and CWD inference, and update focused TUI routing tests plus `docs/cli/tui.md` and `docs/web/tui.md`.
