## Summary

Keep #52412 open. Current main has the prerequisite runId hook context and has gateway-level chat.abort wiring for agent RPC runs, but it still does not expose the requested plugin-facing api.runtime.agent.abort({ runId }) method in the PluginRuntime type, implementation, tests, docs, or gateway method surface. The related #67586 context explicitly scoped plugin-facing api.runtime.agent.abort out of that gateway fix, and the provided context does not include a readable canonical duplicate that would justify closing this one.

## Best Possible Solution

Keep this open until OpenClaw either adds a narrow PluginRuntime agent.abort surface or intentionally consolidates it under a canonical issue. The likely implementation belongs in src/plugins/runtime/types.ts and src/plugins/runtime/runtime-agent.ts or a gateway-bound runtime helper, should dispatch through the existing chat abort path/abortChatRunById with clear session/run ownership semantics, and should add runtime shape tests, gateway/plugin-runtime tests, SDK baseline/docs updates, and a note about how it relates to chat.abort and subagent.run runIds.
