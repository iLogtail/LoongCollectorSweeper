## Summary

Keep #52412 open. Current main exposes runId in plugin hook context and has gateway-level `chat.abort` support for agent RPC runs, but it still does not expose the requested plugin runtime method `api.runtime.agent.abort({ runId })` or equivalent `AgentAbort*` runtime contract. The related PR discussion reinforces that the later gateway abort fix explicitly left plugin-facing abort API work out of scope.

## Evidence

- **Plugin runtime agent type lacks abort:** `PluginRuntimeCore.agent` exposes defaults, agent path/identity helpers, embedded-agent runners, timeout/workspace helpers, and session store helpers; there is no `abort` method or `AgentAbortParams`/`AgentAbortResult` type in the injected runtime contract.
  - file: [src/plugins/runtime/types-core.ts:49](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/plugins/runtime/types-core.ts#L49)
