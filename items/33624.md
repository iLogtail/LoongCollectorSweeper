## Summary

Keep #33624 open. Current main documents and implements several Gateway HTTP endpoints, and it generates a Draft-07 JSON Schema for the WebSocket protocol, but there is still no committed OpenAPI/Swagger document, no `/openapi.json` or Swagger route, and no OpenAPI generation/check script for custom client generation.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. Define the public HTTP API boundary, then add either a committed/generated OpenAPI document or a served `/openapi.json` endpoint for Gateway HTTP APIs, with a check/test that keeps the spec synchronized. Treat the existing WebSocket Draft-07 protocol schema as a separate contract unless maintainers explicitly decide to model that surface too.
