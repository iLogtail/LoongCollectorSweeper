## Summary

Keep open. Current main only partially covers this PR's intent: the async read-side stat check now rethrows non-ENOENT errors in the shared allow-from helper, but the sync stat path still turns non-ENOENT into an empty allowlist and the write-side stat still has a bare catch. The PR discussion is useful: Greptile identified the missed sync path and the author followed up, but the patch needs to be refreshed for current main and should add focused regression coverage.

## Best Possible Solution

Keep this PR open, but ask for a refresh against current main: move the sync stat fix to `readAllowFromFileSyncWithExists`, keep the write-side `fs.promises.stat` non-ENOENT rethrow in `writeAllowFromState`, and add targeted pairing/allow-from tests that simulate `EACCES` or `EIO` for the affected stat calls.
