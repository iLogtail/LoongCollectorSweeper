## Summary

Keep #69297 open. Current main still does not implement WhatsApp per-group admin privacy controls: there is no `groups.*.admin` config type/schema support, no admin-aware inbound policy or sender-aware mention bypass, and group allowlist ingress still blocks before any admin command exception could apply. The PR is the active implementation thread for the already-consolidated feature request #69233, but it should be narrowed before landing because the current PR also carries unrelated QA/agent/chat metadata changes.

## Best Possible Solution

Keep the PR open as the canonical implementation thread for WhatsApp group admin privacy controls. Ask the contributor to narrow or split it: keep the WhatsApp admin behavior, schema, docs, generated metadata, and targeted tests together, and move unrelated QA/agent/chat-meta stabilization into separate PRs unless a maintainer explicitly wants them bundled. Then rebase on current main and run the changed WhatsApp/config lanes before maintainer review.
