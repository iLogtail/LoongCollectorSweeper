## Summary

Keep open. Current `main` still does not expand `~/` in `memorySearch.extraPaths`, and the same unfixed logic is present in the latest release tag `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence

- **Current main still resolves `~/...` relative to the workspace:** `src/memory-host-sdk/host/internal.ts:67`-`76` trims each entry and then does `path.isAbsolute(value) ? path.resolve(value) : path.resolve(workspaceDir, value)`. There is no `expandHomePrefix()` or equivalent `~` handling in this path.
