## Summary

Current main still does not implement the PR's requested Anthropic Opus 4.6 xhigh/max advertisement. Anthropic now owns thinking levels through `resolveThinkingProfile`, and that profile advertises `xhigh`/`max` only for Opus 4.7 while explicitly leaving Opus 4.6 at `adaptive`. The PR's current one-line `supportsXHighThinking` approach is stale against that API, but the underlying request remains plausible because the transport can map a forced Opus 4.6 `xhigh` request to Anthropic `output_config.effort: "max"`. This needs maintainer product/API judgment, not automated closure.

## Best Possible Solution

Keep the PR open, but require a rework before merge. If OpenClaw should expose Opus 4.6 max effort, implement it in `extensions/anthropic/register.runtime.ts` through `resolveThinkingProfile`, update the Anthropic profile tests and thinking docs, and keep Sonnet 4.6 excluded unless transport/runtime behavior supports a distinct xhigh/max level. If Opus 4.7-only xhigh/max is intentional product policy, a maintainer should close the PR with that rationale or link a canonical policy issue.
