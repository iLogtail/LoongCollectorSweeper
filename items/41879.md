## Summary

Current main still matches the reported bug: `isHeartbeatEnabledForAgent` enables the default agent when there are no per-agent heartbeat entries, and defaults-only heartbeat config still scopes scheduling to the default agent. Existing regression tests also encode the reported incorrect behavior, so this should stay open.

## Best Possible Solution

Keep the issue open and fix the heartbeat scope contract in the shared helper and scheduler: no heartbeat config should disable all agents; `agents.defaults.heartbeat` with no per-agent heartbeat entries should enable every configured agent; explicit per-agent heartbeat entries should keep opt-in scoping. Update the existing regression test that currently encodes the bug and add scheduler coverage for defaults-only multi-agent configs.
