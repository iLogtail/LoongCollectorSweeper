## Summary

Keep this PR open. Current main still has the reported Feishu DM topic behavior: DM peer routing remains flat per sender, DM replies do not opt into thread replies, and dispatch/debounce partitioning is still chat-based rather than topic-based. The PR addresses a real bundled Feishu channel gap, but it needs maintainer review and likely rebasing onto the current message-handler/sequential-key architecture before it can land.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or port the change onto current main by adding DM topic-scoped peerId/parentPeer/replyInThread handling in extensions/feishu/src/bot.ts, adding root_id/thread_id-aware lanes in extensions/feishu/src/sequential-key.ts and debounce keys in extensions/feishu/src/monitor.message-handler.ts, preserving main-surface DM behavior, and covering root_id, thread_id-only, dynamic-agent, and debounce cases in Feishu tests.
