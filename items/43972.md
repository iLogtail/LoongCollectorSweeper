## Summary

- Raising the cap is a real behavior change, not maintainer cleanup: runtime limits, schema, generated metadata, docs, and likely tests all need to move together.
- A higher cap changes loop/chatter bounds for agent-to-agent flows, so maintainers may want explicit product guidance on safe defaults versus an increased hard maximum.
