## Summary

Keep open. Current main has a real Control UI i18n framework and generated locale bundles, but the reported Web UI problem is still valid: multiple user-visible Control UI strings remain literal English outside the translation map.

## Best Possible Solution

Keep the issue open as Control UI localization debt. Migrate remaining user-visible Web UI literals into `ui/src/i18n/locales/en.ts`, replace call sites with `t(...)`, regenerate non-English bundles with the existing Control UI i18n workflow, and add a focused guard or test so new raw English UI copy is harder to introduce.
