## Summary

Keep open. Current main still has the exact Discord explicit-target bug this PR describes: `parseDiscordExplicitTarget` returns the bare `target.id`, and the downstream delivery path trusts that parsed `to`, after which Discord outbound normalization treats bare numeric IDs as `channel:<id>`. The PR's focused regression tests are not present on main, and the Greptile comment correctly identifies the proposed fix as narrow and directly relevant.

## Best Possible Solution

Keep PR #65123 open and either merge it or reapply the same scoped fix on main: return `target.normalized` from `parseDiscordExplicitTarget` and add the focused regression tests for `user:<id>`, `channel:<id>`, and chat-type inference. This matches the documented Discord target contract and fixes the current delivery path without a broader behavior change.
