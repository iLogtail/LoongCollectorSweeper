## Summary

Keep this PR open. Current main still only supports Slack `thread.requireExplicitMention` at the account level: per-channel Slack config has no `thread` schema/type/resolver field, and inbound mention gating still reads only the account-level monitor flag. The PR is a plausible implementation path, but its current diff should be revised to validate only the implemented per-channel field instead of reusing the full account-level thread schema.

## Best Possible Solution

Land a revised Slack-owned implementation: add a narrow channel-level `thread?: { requireExplicitMention?: boolean }` config schema/type, resolve it through channel entry -> wildcard -> account default in `extensions/slack/src/monitor/channel-config.ts`, use the resolved value in `prepareSlackMessage`, and keep the PR's two override-direction regression tests. Update docs/generated config metadata if that schema surface changes.
