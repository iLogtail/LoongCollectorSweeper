## Summary

Keep #51654 open. Current main has ACP spawn support for model/thinking/cwd/timeouts, and the low-level ACP runtime type already has an optional env field, but the user-facing sessions_spawn tool does not accept env and the ACP spawn/manager path never forwards a session env map into runtime.ensureSession. The issue discussion is concrete and still actionable; related reports cover different ACP/session or environment-variable topics and do not supersede this request.

## Best Possible Solution

Keep the issue open and implement an additive env?: Record<string, string> path for runtime="acp": expose it in sessions_spawn schema/docs, validate it as string-to-string input, thread it through SpawnAcpParams and AcpInitializeSessionInput, pass it to runtime.ensureSession, and add focused tests that prove acpx receives merged/overriding session env without persisting secret values in transcripts or session metadata.

## Risks / Open Questions

- Per-session env support would handle credentials, so the implementation needs a clear redaction/non-persistence policy for tool calls, session metadata, logs, and errors.
- Because ACP sessions run on the host rather than inside the OpenClaw sandbox, env overrides should be documented as host-runtime credentials available to the selected external harness.
