## Summary

Keep #51654 open. Current main still does not expose or wire session-level environment variables for ACP sessions: `sessions_spawn` has no `env` parameter, `spawnAcpDirect`/`SpawnAcpParams` have no `env` field, and ACP session initialization does not pass env into the runtime. There is a core `AcpRuntimeEnsureInput.env` type field, but it is unused on the current spawn path and the pinned `acpx@0.6.1` runtime contract does not include env support.
