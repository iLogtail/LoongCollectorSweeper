## Summary

Close as duplicate/superseded. Current main still suppresses log writes after `logging.maxFileBytes`, so the feature is not implemented; however the same remaining log-rotation work is already tracked by open issue #66394 and an older open implementation PR #58621. This PR is also marked dirty and bundles unrelated ACPX runtime changes with unresolved review findings, so it should not be the canonical vehicle.

## Best Possible Solution

Keep size-based log rotation tracked through #66394 and the narrower open PR #58621. The eventual fix should be logging-only, update `src/logging/logger.ts`, the logging tests, and public logging docs, and should address custom-path retention, rotation races, failure-warning reset, and any new rotation options deliberately. The ACPX queue-owner/client-pooling work in #66457 should be split into a separate PR if it is still desired.
