## Summary

Current `main` still contains the reported bug. Shell-completion install/status logic hardcodes zsh/fish profile paths, and bash still has the `.bashrc`/`.bash_profile` check-vs-install mismatch described in the issue. No landed fix was found in the inspected code; the issue timeline also points to open follow-up PR #63082 rather than a merged change.

## Evidence

- **Hardcoded profile paths in completion status checks:** `getShellProfilePath()` still resolves zsh to `HOME/.zshrc`, bash to `HOME/.bashrc`, and fish to `HOME/.config/fish/config.fish`; `isCompletionInstalled()` and `usesSlowDynamicCompletion()` both rely on that helper, so `$ZDOTDIR` and `$XDG_CONFIG_HOME` are still ignored on current `main`.
  - file: [src/cli/completion-runtime.ts:128](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/cli/completion-runtime.ts#L128)
