## Summary

Current main does not ship a PLaMo/Preferred Networks provider, and PR #65275 contains useful provider work plus addressed review feedback. The conservative cleanup path is still ClawHub/plugin scope: PLaMo is an optional third-party model provider, and current main already exposes provider plugin docs and SDK hooks for API-key auth, catalogs, dynamic model resolution, replay/tool schema hooks, custom stream transports, and guarded transport helpers. If this work reveals a generic SDK gap, that should be split into a focused core/plugin-API PR rather than landing a new bundled provider by default.

## Best Possible Solution

Close the bundled-provider PR and encourage publishing/maintaining PLaMo as an external OpenClaw provider plugin through ClawHub/npm. Keep any genuinely generic transport, auth, or provider-SDK improvements as small standalone core PRs with focused tests.
