## Summary

Keep #18548 open. Current main still truncates oversized bootstrap files by fixed positional head/tail slices, and the active run path still feeds workspace bootstrap files through that logic. The compaction and observation-extraction parts have plausible plugin/context-engine paths, including the documented Lossless Claw plugin, but that does not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep #18548 open as the core tracker for bootstrap relevance, or split it so bootstrap relevance remains in OpenClaw core while incremental compaction follows #58398/context-engine work and observation extraction stays plugin/hook-driven unless a missing SDK API is identified. The likely code change belongs around src/agents/pi-embedded-helpers/bootstrap.ts and buildBootstrapContextFiles: replace fixed head/tail slicing with a relevance-aware or structure-aware selector, then update positional tests and user-facing docs/config wording.
