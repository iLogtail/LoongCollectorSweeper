## Summary

Keep #18548 open. Current main still injects oversized bootstrap files via fixed positional head/tail slicing, and the active run path, tests, and docs continue to describe/protect truncation rather than relevance-aware or structure-aware bootstrap selection. The compaction and observation-extraction portions have plausible plugin/context-engine paths, but they do not resolve the core bootstrap relevance request, and the existing maintainer/Codex review comment already explicitly kept this item open for that reason.

## Best Possible Solution

Keep #18548 open as the core tracker for bootstrap relevance, or split it so bootstrap relevance remains in OpenClaw core while incremental compaction follows the context-engine/Lossless Claw/#58398 path and observation extraction remains hook/plugin-driven unless a missing SDK API is identified. The likely core change belongs around src/agents/pi-embedded-helpers/bootstrap.ts and buildBootstrapContextFiles: replace fixed head/tail slicing with a relevance-aware or structure-aware selector, then update the positional tests and docs/config wording.
