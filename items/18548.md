## Summary

Keep #18548 open. Current main still truncates oversized bootstrap files by fixed head/tail character slices, and active run/compaction paths still feed workspace bootstrap files through that trimmer. Context-engine, compaction-provider, memory-flush, and observation hooks cover adjacent plugin/compaction work, but they do not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep #18548 open until the core bootstrap truncation behavior is fixed or explicitly descoped. The likely implementation path is a relevance-aware or structure-aware selector in src/agents/pi-embedded-helpers/bootstrap.ts, with tests replacing the current fixed head/tail expectation and docs updated for the new bootstrap selection behavior. Track the compaction portion against the context-engine/compaction-provider path and #58398, and treat observation extraction as plugin/hook/context-engine work unless a missing SDK API is identified.
