## Summary

Keep open. Current `main` does not implement one-shot `/think <level> <body>` behavior: slash-command detection still treats `/think` with args as a control command, mixed directive+body parsing clears the think directive instead of preserving a one-message override, and Telegram still builds `CommandBody` through single-line normalization.

## Evidence

- **Control-command gating still intercepts `/think <level> ...`:** `hasControlCommand()` returns true for any registered command alias with whitespace-separated args. There is no one-shot `/think` exception or helper in current `main`, so `/think high explain this` is still classified as a control command.

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Mixed body text clears the think directive:** When inline directives leave remaining body text, `resolveReplyDirectives()` clears directives via `clearInlineDirectives(...)`; later `resolvedThinkLevel` only reads `directives.thinkLevel` or the stored session value. That means `/think high <body>` does not preserve a one-shot think override on current `main`.
