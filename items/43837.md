## Summary

Current main does not implement the PR's `/think <level> <body>` one-shot behavior, and the PR is not superseded by an equivalent merged change. The existing code still treats `/think` as a normal argument-taking control command, has no `oneShotThinkLevel` directive state, resolves thinking from persistent directive/session/default state only, and Telegram command normalization still truncates multiline command bodies before directive parsing.

## Best Possible Solution

Keep the PR open. The best path is for a maintainer to review the remaining one-shot command semantics, require a clean rebase against current `main`, and merge or request changes once `/think <level> <body>` is consistently routed through reply handling, applies only to that turn, preserves Telegram multiline bodies, and has focused regression coverage for the reviewed edge cases.
