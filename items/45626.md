## Summary

Keep open. Current `main` still advertises and requires `fileId` for Slack `download-file` and Telegram sticker sends, but the shared message tool schema on `main` still does not define `fileId`, and the Slack/Telegram channel schema contributions do not add it either.

## Evidence

- **Shared message tool schema still omits `fileId`:** `buildStickerSchema()` only exposes `emojiName`, `stickerId`, `stickerName`, `stickerDesc`, and `stickerTags`; there is no `fileId` field in the shared schema on current `main`.
  - file: [src/agents/tools/message-tool.ts:282](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/tools/message-tool.ts#L282)

- **Current tool schema only gains extra params from channel schema contributions:** `buildMessageToolSchema()` merges `extraProperties` from `resolveChannelMessageToolSchemaProperties(...)`, so if neither core nor the current channel contributes `fileId`, it is not part of the tool schema.
  - file: [src/agents/tools/message-tool.ts:533](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/tools/message-tool.ts#L533)
