## Summary

Keep #67983 open. Current main still exposes `llm_input` as prompt/history-only telemetry, while the final effective built-in and client tool lists are only available inside the runner after policy, bundled tool, and client-tool assembly. The PR is a focused public plugin-contract enhancement, not a ClawHub-only item or duplicate, but it should be amended before merge to address the existing review concerns around safe cloning, normalization fidelity, and coverage.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation is an additive `llm_input` payload extension from the embedded runner after effective tool resolution, with cloned and normalized built-in/client tool descriptors, regression coverage for both fields, API baseline updates, and plugin docs describing the optional event fields.
