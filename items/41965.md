## Summary

Keep open. Current main still lacks the requested WhatsApp/MEDIA-token force-document behavior: there is no #doc parsing in shared media loading, WhatsApp still routes images through image sends, and the only shipped force-document control is Telegram-specific. Related PR #42044 was closed unmerged after review feedback and author follow-up, so it does not supersede the issue.

## Best Possible Solution

Implement a precise, tested force-document hint for outbound media. The likely path is exact terminal `#doc` parsing or a structured `forceDocument/asDocument` media option in the shared media/outbound contract, stripping the marker before local/remote resolution, preserving filename/content type, computing caps from the final document kind, and plumbing the hint through WhatsApp direct send and auto-reply so Baileys receives a document payload for image files. Add tests for `#doc`, nonmatches such as `#document`/`#docx`, local path resolution, image optimization bypass, and WhatsApp document payload selection; update user-facing docs if this becomes a supported syntax.
