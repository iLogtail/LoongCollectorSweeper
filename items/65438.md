## Summary

Current main still does not implement configurable bootstrap dynamic-file classification or configurable bootstrap file injection order. The renderer hardcodes both the order and the dynamic set, the config schema/docs expose only existing injection and size controls, and related context treats #65438 as an unresolved prompt-cache follow-up rather than something already shipped or superseded.

## Best Possible Solution

Keep #65438 open. A conservative implementation would add documented `agents.defaults` and likely per-agent config for `bootstrapDynamicFiles` and possibly `bootstrapFileOrder`, thread it into prompt assembly before `sortContextFilesForPrompt()`, preserve deterministic ordering, update generated config schema/help/docs, and add tests proving selected files move below `OPENCLAW_CACHE_BOUNDARY` without destabilizing the stable prefix.
