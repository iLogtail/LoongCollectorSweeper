## Summary

Keep open. The report is recent, detailed, and points at a current gateway-host exec surface that still exists on main: YOLO gateway exec skips the script-preflight layer and then runs each command through a shell child process. I found no current source, docs, tests, or related tracker evidence that resolves or supersedes the reported macOS/APFS cross-exec stale-read behavior.

## Best Possible Solution

Keep #71326 open for maintainer investigation. The best next step is a minimal macOS/APFS repro that runs the same atomic-rename writer and repeated gateway-host exec reads against current main and, if possible, v2026.4.19/v2026.4.20. If reproduced, add a focused regression test or macOS smoke harness around the exec topology and then decide whether the fix belongs in gateway exec process reuse, documented retry/validation guidance for hot files, or an explicit external macOS/APFS limitation.
