## Summary

Keep open. The report is recent, detailed, and targets OpenClaw core gateway-host exec behavior. Current main still skips script preflight in gateway YOLO mode and runs host exec through a fresh shell child process; I found no source, docs, tests, changelog, or related-item evidence that adds a stale-read/cache-coherence mitigation or supersedes this issue. The supplied related items are only title-search overlaps around stale state/log/memory issues, not a canonical duplicate for this cross-exec APFS stale-read report.

## Best Possible Solution

Keep the issue open for maintainer investigation. The next useful path is a focused macOS/APFS repro against v2026.4.19, v2026.4.20, v2026.4.24, and current main using gateway-host exec with security=full and ask=off. If confirmed, the fix likely belongs in the core host-exec/runtime path or in a documented gateway-host filesystem consistency diagnostic, with regression coverage around atomic rename followed by reads from separate exec calls.
