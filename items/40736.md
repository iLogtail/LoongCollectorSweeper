## Summary

Keep open. Current `main` still has TUI delivery paths that pass assistant string/text content through without the canonical reasoning-tag sanitizer, so the reported leak remains plausible rather than clearly fixed.

## Evidence

- **TUI streaming path leaves assistant string content unsanitized:** `extractContentFromMessage()` returns assistant `content` strings after `sanitizeRenderableText()` only; it does not call `sanitizeAssistantVisibleText()` or any reasoning-tag stripper, so raw `<think>...</think>` text would still be displayable during streaming.
  - file: [src/tui/tui-formatters.ts:268](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/tui/tui-formatters.ts#L268)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '268,360p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Shared assistant visible-text extractor does not sanitize reasoning tags by default:** `extractAssistantVisibleText()` delegates to `extractAssistantTextForPhase()` without a `sanitizeText` callback. For string content or plain text blocks, the helper trims but does not strip reasoning tags.
  - file: [src/shared/chat-message-content.ts:93](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/shared/chat-message-content.ts#L93)
  - command: `nl -ba src/shared/chat-message-content.ts | sed -n '90,190p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Canonical sanitizer exists elsewhere but is not wired into the shared TUI extraction path:** `sanitizeAssistantVisibleText()` explicitly strips reasoning tags, Minimax tool XML, and downgraded tool markers, which shows the intended fix surface exists; the TUI path above is not using this canonical sanitizer for shared visible-text extraction.
  - file: [src/shared/text/assistant-visible-text.ts:560](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/shared/text/assistant-visible-text.ts#L560)
  - command: `nl -ba src/shared/text/assistant-visible-text.ts | sed -n '560,625p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Current TUI tests do not cover the reported regression:** `src/tui/tui-formatters.test.ts` covers final-answer preference, string preservation, and ANSI/control cleanup, but there is no regression test asserting that `<think>` or similar reasoning tags are removed from assistant string/text content when thinking is off.
  - file: [src/tui/tui-formatters.test.ts:10](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/tui/tui-formatters.test.ts#L10)
  - command: `nl -ba src/tui/tui-formatters.test.ts | sed -n '1,120p'`

- **Issue discussion and timeline still point to an unresolved family of leaks:** The provided issue context shows multiple related cross-references after the initial report, including open PR `#40873` and open issues `#41494` and `#59826`, while the only linked closed PR in the timeline was `#40779`. That does not support a conservative 'implemented on main' close.
