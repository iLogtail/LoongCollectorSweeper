## Summary

Keep open. Current `main` still drops `authProfileId` when the active execution provider is not the run's primary provider, and the CLI session reuse path still invalidates stored sessions on any `authProfileId` mismatch. That matches the stale-binding failure described in the issue discussion, and I did not find a main-branch fix or regression test covering CLI fallback-provider session continuity. I could not run an end-to-end repro here because `pnpm` is unavailable in this checkout and no Gemini CLI OAuth environment is configured.
