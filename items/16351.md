## Summary

Current main implements `openclaw config validate` and config write-back protections for env/SecretRef values, but the remaining root-command behavior is still missing: bare `openclaw config` is wired and documented to open guided setup rather than showing help on an already configured system.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely implementation point is the root config action in `src/cli/config-cli.ts`: when no subcommand is provided and the system is already configured, print config command help instead of entering guided setup, while preserving the shipped `config validate` and env/SecretRef-preserving write behavior and updating docs/tests accordingly.
