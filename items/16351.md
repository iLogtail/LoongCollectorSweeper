## Summary

Keep #16351 open. Current `main` already implements `openclaw config validate` and protects config writes from invalid or secret-resolved payloads, but the remaining root-command request is still unresolved: bare `openclaw config` still dispatches into the guided configure wizard, and the docs still document that behavior. The existing maintainer-posted Codex review reached the same conclusion, and the relevant files have not changed since that reviewed commit.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining bare `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely path is to update `src/cli/config-cli.ts` and `src/commands/configure.commands.ts` so no-subcommand `openclaw config` on an already configured system prints config help, while preserving `config validate`, SecretRef/env-preserving writes, and updating docs/tests to match.
