## Summary

Keep open. Current `main` still intentionally uses least-recently-used round-robin ordering for implicit auth-profile selection after reset, and there is an explicit regression test asserting `lastGood` must not be preferred. The silent-402 half looks largely addressed on current `main`, so the issue is only partially resolved.
