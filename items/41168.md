## Summary

Keep open. Current main has fixed adjacent reconnect and heartbeat-label problems, but it still does not satisfy this issue's central restart behavior: TUI without an explicit `--session` initializes to the configured main/global session, not the most recently active non-heartbeat user session. The storage-bloat half also remains plausible because first-turn sessions still persist a `skillsSnapshot` containing the prompt and resolved skills in each session entry, although session maintenance now limits stale/count/disk growth.

## Best Possible Solution

Keep #41168 open. The best maintainer path is to implement a TUI startup resolver that, only when `--session` is omitted, asks `sessions.list` for the current agent, selects the latest real user session by `updatedAt` while excluding heartbeat/cron/synthetic sessions, and falls back to the configured main session. Add focused TUI tests for omitted `--session`, explicit `--session`, reconnect, and heartbeat exclusion. For storage, either replace per-entry `skillsSnapshot.prompt/resolvedSkills` with a shared snapshot keyed by workspace/version/filter or document/extend maintenance if per-session snapshots are intentional.
