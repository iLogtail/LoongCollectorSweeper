## Summary

Keep #46356 open. Current main has partially superseded the Discord proxy pieces, but the central webchat/control-UI bidirectional sync behavior is still not implemented, and the WhatsApp channel-specific proxy config requested by the PR is still absent. The PR should not be merged as-is because it is broad and the provided review context includes unresolved correctness concerns, but no conservative close reason applies.

## Best Possible Solution

Keep the item open but ask for a narrow replacement path: split webchat bidirectional sync into a focused gateway/session PR with mirror-target tests for DM, group, thread, directive, and deliver:true cases; handle WhatsApp explicit proxy config separately if still wanted; reuse the current Discord proxy-request-client path instead of re-patching Carbon; drop unrelated Matrix/Tlon/SDK churn and resolve the existing review findings before maintainer review.
