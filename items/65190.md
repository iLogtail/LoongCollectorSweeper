## Summary

Current `origin/main` still does not implement Vietnamese docs localization. The source docs i18n policy keeps glossary/navigation source assets in `docs/.i18n`, but `main` has no `glossary.vi-VN.json` or `vi-VN-navigation.json`; the docs publish sync locale registry omits `vi-VN`; the release dispatch workflow omits `translate-vi-vn-release`; and the README still lists active docs locales without Vietnamese. The PR discussion also shows reviewer-identified activation gaps were addressed in the PR head, so this remains plausible useful work rather than a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase it onto current `main`, preserve the Vietnamese glossary/navigation source assets, add `vi-VN` to `GENERATED_LOCALES` with the right TM/nav metadata, wire `translate-vi-vn-release`, update `docs/.i18n/README.md`, and coordinate the matching `openclaw/docs` translation workflow before merge.
