## Summary

Keep #71178 open. Current main still lacks the `update.run` restart-continuation producer that would resume an agent turn after a successful in-place update, and maintainer discussion explicitly says this specific report should stay open until that path is fixed and scoped.

## Best Possible Solution

Keep #71178 open until PR #71573 or an equivalent main-branch change lands. The narrow code fix should add `continuationMessage` to the `update.run` protocol params, parse it through restart request handling, build a restart success continuation in `src/gateway/server-methods/update.ts` on successful session-scoped updates, forward it from `src/agents/tools/gateway-tool.ts`, and update the tests that currently assert no continuation. Maintainers should then decide whether graceful-drain and post-reset bridge behavior remain in this issue or move to narrower follow-ups.
