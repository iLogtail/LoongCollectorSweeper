## Summary

Current `main` has a Control UI fix that defers transcript reloads while a session run is active and replays the reload after the run ends. Because heartbeat runs are main-session turns, that directly matches the reported symptom of chat content disappearing mid-heartbeat in the UI.

## Evidence

- **Heartbeat runs in the main session:** Docs state heartbeat is a scheduled main-session turn, not a detached task, so heartbeat transcript updates hit the same session/UI path as normal chat.
