## Summary

Keep #70512 open. Current main still routes ordinary Feishu/Lark DMs by sender open_id only; root_id/thread_id are captured from inbound events but are only used for group topic behavior, not direct-message session isolation. The related #10242 context covers DM new-thread sending/thread replies, not inbound DM topic-based session keys.

## Best Possible Solution

Implement this in the bundled Feishu plugin, not generic core routing: add an explicit Feishu DM topic/session-scope config, derive direct-message peer/session ids from sender plus root_id/thread_id when enabled, update docs/config metadata, and add focused Feishu p2p/private tests for topic and non-topic DMs.
