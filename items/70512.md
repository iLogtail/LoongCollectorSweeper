## Summary

Keep #70512 open. Current main supports Feishu thread replies and group topic session isolation, and the related #10242 new-thread request was implemented, but this issue asks for inbound Feishu DM topics to route to separate gateway sessions. The current Feishu inbound route still uses only the sender open_id for DMs, and the Feishu config schema has no DM topic/sessionScope option.

## Best Possible Solution

Keep the issue open for a Feishu plugin change: add a documented opt-in DM session scope such as channels.feishu.dm.sessionScope or an equivalent Feishu config key, derive a stable direct peer/conversation id from sender plus root_id/thread_id when enabled, preserve current per-user behavior by default, and cover routing, ACP bindings, reply delivery, schema metadata, and docs with Feishu tests.
