## Summary

Keep open. Current main has several abort primitives, but it does not implement the unified run-generation invalidation, stale-output fencing, pre-tool gate, or full new-message takeover requested by #70319. The linked draft PR #70363 is active and explicitly partial, so it is better treated as the current implementation path rather than a reason to close the umbrella issue.

## Best Possible Solution

Keep #70319 open as the umbrella until the coordinated interruptibility contract lands. The best path is to continue through #70363 or smaller follow-up PRs that add a per-session run generation, capture it in ReplyOperation, bump it on /stop and superseding user messages, fence every visible emission path, add a pre-tool gate in the embedded runner, and wire process supervisor cancellation to the run/session scope with regression tests.
