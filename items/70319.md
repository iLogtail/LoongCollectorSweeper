## Summary

Keep open. Current main has several cancellation primitives, but it does not yet provide the issue's requested unified session run-generation invalidation, stale-output fencing across reply/tool/typing/progress surfaces, pre-tool skip gate, or immediate new-message takeover. The related #70363 PR is an open draft and is explicitly partial, with pre-tool gating, new-message takeover, and additional emission fences deferred.

## Best Possible Solution

Keep the issue open as the canonical tracker for unified run invalidation. The best path is to land a small, reviewed run-generation primitive first, then wire it through reply operations, fast abort, queue interrupt/new-message takeover, block and non-block delivery, typing/progress/tool-event emission, and embedded tool dispatch with regression tests. The partial #70363 work can be reviewed as an implementation slice, but it should not close the umbrella issue until the deferred pieces are either shipped or split into narrower tracked follow-ups.
