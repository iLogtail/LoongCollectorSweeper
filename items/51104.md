## Summary

Keep open. Current main still does not implement the requested Mattermost WebSocket protocol ping/pong keepalive. The Mattermost WebSocket wrapper only models open/message/close/error plus send/close/terminate, and the connection open path sends auth then starts the existing bot update_at health check. The secondary stale-event threshold is now configurable, but that remains a fallback idle/staleness monitor rather than transport-level detection of half-dead WebSocket connections. The related PR #57621 tracks the exact fix but is still open/unmerged, and the issue discussion supports this as a real Mattermost session connectivity risk.
