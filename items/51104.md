## Summary

Keep open. Current main still does not implement the requested Mattermost WebSocket protocol ping/pong keepalive. The Mattermost socket wrapper exposes only open/message/close/error plus send/close/terminate, and the connection path sends the auth frame then starts the existing REST bot update_at health check. The configurable stale threshold exists now, but it remains a fallback health-monitor setting rather than transport-level half-dead WebSocket detection. The linked PR #57621 tracks the exact fix but is still open/unmerged in the provided GitHub context, and the issue discussion supports this as a real Mattermost connectivity risk.

## Best Possible Solution

Keep #51104 open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add ping/pong or equivalent transport liveness in extensions/mattermost/src/mattermost/monitor-websocket.ts, terminate missed-pong sockets so runWithReconnect restarts them, clean up timers on close/error/abort, and add focused monitor-websocket tests for healthy pong handling and missed-pong termination.
