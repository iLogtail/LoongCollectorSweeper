## Summary

Do not close #55705. Current main still has the stale-cache-prone Control UI path: `loadSessionsOnce()` only assigns `state.sessionsResult` when `sessions.list` returns a truthy response, while the overview/dashboard count is rendered directly from the cached `sessionsResult.count`. The linked PR #55831 is still open/unmerged and appears to be the right candidate fix, not shipped behavior.

## Best Possible Solution

Keep #55705 open and use #55831 as the canonical candidate fix: update `ui/src/ui/controllers/sessions.ts` so `loadSessions()` clears or replaces `sessionsResult` when the API returns no data, then add a regression test with a previously populated `sessionsResult` and an undefined/empty `sessions.list` response to prove the dashboard count cannot stay stale.
