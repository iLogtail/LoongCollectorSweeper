## Summary

Keep open. Current main covers some adjacent behavior, but this PR's bundle is not fully implemented or clearly superseded: the macOS wizard timeout is still 0.75s, the dashboard nav width is still clamped to 400px, the skill-creator docs still do not expose quick_validate.py, and browser restart-deferral would need a current browser-plugin/service design rather than the obsolete src/browser/server.ts patch. The reset-hook surface now exists and Brave llm-context filter limitations are documented/enforced, but those partial overlaps are not enough to close the PR.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. The best path is to split it into focused current-main PRs: a small macOS timeout change with onboarding proof, a UI nav-width clamp/rendering change with browser tests, a skill-creator docs update for quick_validate.py, and a separate gateway/browser restart-deferral design that exposes active browser work through a current plugin/runtime API. Any pre-reset agentic memory flush should be redesigned on top of the existing reset/before_reset hook surface with idempotent registration, correct config/session-file handling, visible failure logging, and tests.
