## Summary

Current main still matches the reported behavior. For Telegram, an unset `reactionLevel` resolves to `minimal`, the Telegram plugin turns that into prompt reaction guidance, and the system prompt emits the exact `Reactions are enabled for Telegram in MINIMAL mode.` text. The path is channel-driven rather than GPT-specific, but GPT-backed Telegram sessions can still observe it. Related Telegram reaction guidance work such as #9835 is relevant, but it is broader and does not make this specific defaulting/prompt-stability report obsolete.

## Best Possible Solution

Keep the issue open. Maintainers should decide the product contract for unset Telegram reaction levels in existing sessions, then align implementation, schema comments, docs, and tests. Likely fixes are to inject prompt reaction guidance only when `channels.telegram.reactionLevel` is explicit, add a migration/doctor notice for existing configs, or coordinate this with the broader Telegram reaction-guidance work in #9835 while preserving a clear no-surprise defaulting rule.
