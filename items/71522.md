## Summary

Keep open. The issue is recent, concrete, and linked to PR #71678, but current main does not appear to contain that fix. Main still has the reported control-path gaps: non-batch memory embedding indexing falls back to a hardcoded concurrency of 4, config/docs only expose `remote.batch.concurrency`, CLI batch status still renders disabled state as failure counts, and the remote embedding transport lacks the PR-described URL normalization and pre-response diagnostic wrapping.

## Best Possible Solution

Keep #71522 open until #71678 or an equivalent fix lands. The likely maintainer path is to add a dedicated non-batch remote embedding concurrency control, clarify batch disabled status with a reason field, normalize remote embedding endpoint URL joining, wrap guarded-fetch transport failures with sanitized endpoint/cause diagnostics, and cover those seams with focused config, manager, CLI, remote HTTP, and docs/changelog updates.
