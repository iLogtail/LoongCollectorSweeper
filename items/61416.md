## Summary

Keep open. Current main still strips SearXNG image-result `img_src`: the result type and normalizer only preserve `url`, `title`, and `content`, and the final `web_search` payload maps only `title`, `url`, `snippet`, and `siteName`. The PR is a focused additive change to an existing bundled SearXNG plugin, and the current web-search provider contract returns `Record<string, unknown>`, so no core API gap or ClawHub-only boundary justifies cleanup-closing it.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is an additive provider-owned change in `extensions/searxng/src/searxng-client.ts` plus the colocated regression test, preserving `img_src` for SearXNG image-category results without changing core web-search APIs.
