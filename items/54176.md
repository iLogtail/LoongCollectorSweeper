## Summary

Keep open. Current main still lacks the legacy Office binary guard for `application/x-cfb` and `application/msword`; the existing regression coverage only covers `application/vnd.*` Office MIME types. The related discussion is useful: #54190 identified the root cause and #54380 is an open superseding PR for the same fix, but the issue should remain open until that or an equivalent patch lands on main.

## Best Possible Solution

Keep #54176 open and land #54380 or an equivalent narrow patch: add `application/x-cfb` and `application/msword` to the media-understanding binary MIME guard in `src/media-understanding/apply.ts`, then add regression tests in `src/media-understanding/apply.test.ts` proving legacy OLE/CFB `.doc` attachments are skipped instead of embedded as text.
