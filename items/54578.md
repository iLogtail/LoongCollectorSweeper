## Summary

Keep #54578 open. Current main has useful primitives for sessions, subagent status, child-session metadata, tasks, and Control UI session tables, but it does not provide the requested `openclaw tree` or equivalent hierarchical active-agent view. The provided related items are adjacent or unrelated and do not supersede this request.

## Best Possible Solution

Keep the issue open and implement a first-class hierarchy view, likely as `openclaw tree` or a narrowly scoped `openclaw sessions --tree`, backed by `sessions.list` child/parent metadata and task/subagent status. A conservative first version should support text and `--json`; `--watch` and CPU/memory can be follow-ups if they need additional runtime metrics.
