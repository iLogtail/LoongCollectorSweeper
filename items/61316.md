## Summary

Keep PR #61316 open. Current main still has the case-sensitive custom-list behavior and lacks the non-array guard this PR proposes. The earlier related PR #59535 is closed and unmerged, so it is not a canonical shipped replacement.

## Best Possible Solution

Keep the PR open for maintainer review or port the same behavior directly: normalize custom truthy/falsy arrays before comparison, ignore non-array runtime overrides safely, and add the mixed-case and malformed-option regression tests. Consider addressing the bot review suggestion by filtering custom arrays to strings before lowercasing.
