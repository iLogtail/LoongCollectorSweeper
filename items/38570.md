## Summary

Keep open. Current main still does not include the requester agent's configured `identity.name` in `sessions_send` agent-to-agent message, reply, or announce prompt contexts. The linked fix PR #38579 addressed the right path but was closed unmerged, so it does not make current main fixed.

## Best Possible Solution

Implement the narrow core fix in `sessions_send`: derive the requester agent id from the effective requester session key, resolve `agents.list[].identity.name` from the current config, and include `Agent 1 (requester) name: <name>.` in the initial message, reply, and announce A2A prompt contexts when the configured name is non-empty. Add focused regression tests for all three prompt contexts and preserve existing output when no requester identity name is configured.
