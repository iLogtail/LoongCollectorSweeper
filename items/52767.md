## Summary

Current main still matches the reported behavior: the boot-md handler is wired to every `gateway:startup` event, builds per-agent startup tasks, and calls `runBootOnce` for each agent. `runBootOnce` only skips missing or empty `BOOT.md`; when the file exists it invokes `agentCommand` and returns `ran`. I did not find a persisted run-state/sentinel guard, and the docs/tests still describe and lock the “runs when the gateway starts” behavior. The issue comment’s proposed idempotency guard is not implemented on main, so this should stay open.
