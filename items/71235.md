## Summary

Keep #71235 open. Current main does not include the proposed python -> python3 shim in any of the four targeted Dockerfiles, and the PR is a focused Docker image behavior change rather than a duplicate, external-only request, or ClawHub/plugin item. The v2 PR context also addressed the two concrete bot review concerns by using `$(command -v python3)` and checking `command -v python` before creating the shim.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, land a narrowly scoped Dockerfile change that creates the shim only when `python3` exists and no `python` command is already on PATH, ideally with a lightweight Dockerfile text test or CI smoke assertion for the four affected images.
