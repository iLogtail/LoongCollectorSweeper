## Summary

Keep #71235 open. Current main still uses bookworm/bookworm-slim Docker images and includes or allows `python3`, but none of the four target Dockerfiles creates an unversioned `python` shim. The PR is recent, scoped to Dockerfile behavior, and the review comments in the provided context show the author already addressed the two concrete robustness nits in head commit 6b44ae4.

## Best Possible Solution

Keep the PR open for maintainer review. If the product decision is to make agent-invoked `python` work whenever `python3` is present, merge the v2 conditional shim pattern across the four Dockerfiles and add a small Docker smoke assertion for `python --version` on Python-bearing images plus a no-op case where `python3` is absent.
