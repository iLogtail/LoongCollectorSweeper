## Summary

Keep open. Current main has later core handling for empty/error assistant turns, but it does not implement this PR's OpenClawKit history decoding/rendering path: Swift chat history messages still do not preserve `errorMessage`, and the Swift renderer only displays text from `content`. The PR also needs the existing review comments addressed so any fallback is restricted to real assistant error turns.

## Best Possible Solution

Keep the PR open for revision. The best path is to add/pass through `errorMessage` in OpenClawKit and render it only for assistant messages with `stopReason == "error"`, preferably as display-only fallback when visible text is empty or only the core sentinel. Add regression coverage for both the error history refresh and a successful turn with stale `errorMessage`.

## Risks / Open Questions

- Merging the PR as-is could display stale/background provider errors on successful assistant turns.
- A fix should keep provider error text display-only; current core deliberately keeps raw provider errors out of replay-visible `content`.
- Because current core now uses sentinel content for failed empty turns, the OpenClawKit UI fix may need to handle both empty content and the sentinel text when showing detailed provider errors.
