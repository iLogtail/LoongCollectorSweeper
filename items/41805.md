## Summary

Keep open. Current main has partial Discord interaction plumbing: sender id/username are present, and plugin interactive handlers can receive a source messageId. But the ordinary agent callback path still sends only synthesized text/callbackData as BodyForAgent, and the context does not expose structured button metadata such as source message id, component/custom id, or button label. The related sender-id issues address only one part of this request.

## Evidence

- **Agent callback path lacks structured interaction metadata:** Discord component fallback builds the inbound context with BodyForAgent/RawBody/CommandBody set to eventText and SenderId/SenderUsername set from the clicker; MessageSid is the Discord interaction id. There is no UntrustedStructuredContext or interaction object carrying source message id, component id/custom id, or button label.
