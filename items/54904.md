## Summary

Keep open. Current main still has the Feishu webhookPath bug the PR targets: webhook mode records/logs the configured path, but the request handler does not compare the inbound request path before generic guards, body reading, signature validation, and dispatch. The linked PR remains a focused Feishu-owned bug fix with regression tests and addressed review feedback rather than obsolete cleanup.

## Best Possible Solution

Keep #54904 open and finish maintainer review. The best fix path is to land this PR or an equivalent Feishu-owned patch that normalizes the configured webhookPath, rejects malformed/non-origin or wrong request targets before body parsing/signature verification/dispatch, and keeps focused transport regression coverage for wrong path, query-string acceptance, path normalization, malformed absolute-form, and network-path cases. That should also close #54841.
