## Summary

Current `main` does not enforce Feishu `webhookPath` on inbound webhook requests. The open PR still addresses a real gap and is not already implemented on `main`.

## Evidence

- **No request-path check in webhook handler:** `monitorWebhook` records the configured `path` but the request handler never reads `req.url` or compares the incoming path before running guards, reading the body, validating the signature, and dispatching the event.
