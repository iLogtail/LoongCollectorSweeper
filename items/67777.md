## Summary

Keep open. Current main includes a partial direct-fallback fix for the observed yielded-parent/no-visible-output loss window, but the broader durable completion spool/inbox requested in #67777 is not implemented. The current design still uses direct-first delivery, conditional queue fallback, bounded retry/give-up cleanup, and orphan pruning that deletes missing-session runs rather than preserving pending spooled completions.

## Best Possible Solution

Keep #67777 open as the planning thread for the remaining core durability work. The best implementation path is to add a self-contained persisted completion spool to SubagentRunRecord/state, create it before cleanup retry/give-up branches, make direct delivery consume and mark spool state, retry pending spools on resume/restart, prevent orphan pruning while a pending completion spool exists, and add lifecycle/dispatch/restore tests for those transitions.
