## Summary

Keep open. Current `main` does not implement the PR’s user-facing context enrichment for overloaded, rate-limit, or transport errors. The relevant formatters still return generic strings, `formatAssistantErrorText()` still accepts only `cfg/sessionKey/provider/model`, and the current tests still assert the old generic output. The idea remains useful, but the patch would need rebasing onto the current helper split.

## Evidence

- **Transient error copy is still generic:** `formatRateLimitOrOverloadedErrorCopy()` still returns the bare overloaded/rate-limit strings (or provider-specific reset text) and does not append provider/model/profile/trigger/session/request context.
  - file: [src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:36](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts#L36)
- **Transport error copy is still generic:** `formatTransportErrorCopy()` still maps transport failures to fixed strings such as `connection refused by the provider endpoint`, `network connection was interrupted`, and `DNS lookup for the provider endpoint failed`, with no context suffix or actionable hint.
  - file: [src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:108](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts#L108)
- **Formatter API does not accept profile/trigger:** `formatAssistantErrorText()` on `main` only accepts `cfg`, `sessionKey`, `provider`, and `model`; there is no `profileId` or `trigger` input to surface in user-facing error text.
  - file: [src/agents/pi-embedded-helpers/errors.ts:926](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-helpers/errors.ts#L926)
- **Runner still passes only provider/model context:** The embedded runner calls `formatAssistantErrorText()` with `sessionKey`, `provider`, and `model` only, so the requested profile/trigger context is not threaded through current runtime code.
  - file: [src/agents/pi-embedded-runner/run.ts:949](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-runner/run.ts#L949)
- **Payload builder still lacks profile/trigger fields:** `buildEmbeddedRunPayloads()` still only carries `sessionKey`, `provider`, and `model` into `formatAssistantErrorText()`, matching the current limited formatter API.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:121](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-runner/run/payloads.ts#L121)
- **Tests still lock the old generic strings:** Current tests still assert the exact old overloaded and transport messages, which confirms `main` has not adopted the richer context-bearing copy from this PR.
  - file: [src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts:59](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts#L59)
- **Lifecycle logging test still expects bare overloaded output:** The lifecycle test still expects the console message to contain only the generic overloaded text plus raw error, not the proposed profile/trigger/request-context suffix.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.test.ts:95](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-subscribe.handlers.lifecycle.test.ts#L95)

## Risks / Open Questions

- The PR description targets the older `errors.ts` implementation, but transient-copy helpers now live in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`, so the branch will need a rebase onto current `main` structure.
- Current `main` has newer failover-classification logic in `src/agents/pi-embedded-helpers/errors.ts`; any revival should preserve that behavior and add direct tests for the new context suffixes rather than only loosening existing exact-string assertions.
