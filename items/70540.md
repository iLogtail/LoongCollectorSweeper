## Summary

Keep #70540 open. Current main does not implement `memory_add`: the bundled `memory-core` plugin still registers only `memory_search` and `memory_get`, the tool catalog and docs expose only those memory tools, and indexing still chunks daily memory markdown with the generic markdown chunker rather than PR #70540's proposed block parser. The linked proposal #70049 is closed, but it describes the same requested feature and does not supersede this PR with an implemented mainline change.

## Best Possible Solution

Keep the PR open for maintainer product review. If accepted, the work belongs in the bundled `memory-core` plugin plus the catalog, gateway policy, display metadata, docs, and focused tests shown by the PR. If maintainers decide memory writes should be externalized instead, open a clear follow-up defining the needed plugin/API boundary; current main does not provide an implemented or superseding `memory_add` path.
