## Summary

Keep open. Current main still has the unresolved Slack and Discord parts of this PR: both slash/native command routing paths still route with the provider startup config object, while `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`. Telegram has since moved to a fresh runtime config pattern, so the original PR is partially superseded but not obsolete.

## Best Possible Solution

Keep this PR or the linked #39605 bug active. The best maintainer path is to rebase the remaining fix onto `extensions/discord/src/monitor/native-command.ts` and `extensions/slack/src/monitor/slash.ts`, load the current runtime config only for slash/native route resolution and configured binding readiness, preserve the already-updated Telegram path, and add Slack/Discord regression tests for `session.dmScope` or route-binding config changes without provider restart.
