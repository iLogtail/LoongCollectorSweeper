## Summary

Keep open. Current main still passes startup-captured config into Slack slash routing and Discord native command routing, while route resolution still reads cfg.session.dmScope. Telegram has since implemented the fresh-config pattern and has regression coverage, so this PR is partly stale but the Slack and Discord parts remain useful.

## Best Possible Solution

Keep this open or keep #39605 as the canonical bug until Slack and Discord are fixed. The best code path is to rebase/split the PR: preserve Telegram's already-shipped fresh-config implementation, add fresh loadConfig-based routing snapshots for extensions/slack/src/monitor/slash.ts and extensions/discord/src/monitor/native-command.ts or native-command-route.ts, and add Slack/Discord regression tests mirroring the Telegram native command routing reload test.
