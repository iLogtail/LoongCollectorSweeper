## Summary

Keep open. Current `main` still hardcodes `30_000` ms in `runImagePrompt()`, so `tools.media.image.timeoutSeconds` is still ignored by the image tool even though the config contract and related media-understanding pipeline support that setting.

## Evidence

- **Current main still uses hardcoded 30s timeouts in the image tool:** `runImagePrompt()` builds `effectiveCfg`, but the three image-description call sites still pass `timeoutMs: 30_000` directly instead of reading `tools.media.image.timeoutSeconds`. That means the PR's reported bug is still present on `main`.
