## Summary

Keep open. Current main has partial coverage, but it does not implement the central reply-first/default chat behavior bundle. Reply metadata is available, Active Memory has latest-message retrieval guardrails, and sub-agent completion is result-oriented, but the prompt still allows routine progress updates and there is no explicit rule that a direct reply target outranks broader retrieval/history.

## Best Possible Solution

Keep the issue open for maintainer product judgment or split it into narrower actionable issues: one for explicit reply-first prompt/context priority, one for default retrieval/local-context guardrails outside Active Memory, one for progress-update policy, and one for any desired sub-agent orchestration default. If maintainers prefer this as optional behavior, document the plugin/context-engine or prompt-overlay path rather than closing it as already implemented.
