## Summary

Keep open. Current main partially covers pieces of the proposal, but the bundled request is not fully implemented: Telegram reply metadata is surfaced as context, active-memory has latest-message retrieval guardrails, and sub-agent announces are result-oriented, yet there is no explicit reply-first priority contract for the replied-to message and the default prompt still allows brief progress updates for longer work.

## Evidence

- **Issue scope:** #59970 asks for a bundle of defaults: reply-first priority, local context before retrieval, retrieval guardrails, no fragmented progress updates, result-oriented sub-agent reporting, and sub-agent-first complex-work routing. The follow-up comment adds anti-stall defaults around safe internal action, lightweight classification, and fallback routing.
- **Reply metadata exists, but only as context:** Current inbound prompt assembly records reply context and renders it as `Replied message (untrusted, for context)` with sender/body. This supports reply-aware behavior but does not encode the requested higher-priority rule to answer the immediate reply target first.
