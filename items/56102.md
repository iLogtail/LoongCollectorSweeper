## Summary

Keep open. Current main still appears to reject the reported VS Code/Cursor `initialize.params.protocolVersion: "2025-11-25"` on the ACP bridge path: `openclaw acp` passes the raw SDK NDJSON stream into `AgentSideConnection`, while the pinned ACP SDK validates `protocolVersion` as an integer before OpenClaw's `initialize()` handler runs. PR #56176 is a directly related open, unmerged fix attempt, but it is not evidence that main is fixed yet.

## Best Possible Solution

Keep #56102 open. Land a reviewed version of #56176, or an equivalent ACP bridge fix, that normalizes or negotiates date-string MCP `protocolVersion` values before `@agentclientprotocol/sdk` validates `initialize`; add a focused regression test for `"2025-11-25"` and keep the existing `openclaw mcp serve` path separate.
