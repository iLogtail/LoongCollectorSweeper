## Summary

Keep #54014 open. Current main has real OPENCLAW_HOME support for the core state/config/workspace paths, but the issue also explicitly asks that the default OPENCLAW_GIT_DIR derive from OPENCLAW_HOME, and current main still derives the dev checkout default from os.homedir(). Related PR #54015 was closed unmerged, while #65536 is a narrower open sandbox tilde-expansion fix and does not supersede the broader OPENCLAW_HOME request.

## Best Possible Solution

Keep the issue open and narrow the remaining work: decide whether OPENCLAW_HOME should also govern the dev checkout default, then update src/cli/update-cli/shared.ts to resolve the default git install dir through the shared home resolver, add an OPENCLAW_HOME regression case in src/cli/update-cli.test.ts, and align docs if the product decision is yes. Treat #65536 as a separate sandbox tilde-expansion fix rather than the canonical tracker for this broader issue.
