## Summary

Current main has adjacent documentation for local-model fallback, text-only CLI backends, sub-agent model/timeouts/sandbox controls, and local tool-calling caveats, but it does not document the requested first-class cloud-orchestrator plus local text-worker architecture pattern. The issue is recent, non-protected, and still has a concrete docs gap rather than a closeable duplicate or shipped implementation.

## Best Possible Solution

Keep #51333 open and add a concise docs/cookbook page or section, likely under concepts or gateway docs, that combines existing primitives into the recommended pattern: strong hosted/cloud model as the tool-using orchestrator, local models or CLI backends as bounded text-only sub-agents, prompt examples for drafting/refactoring/summarization, config showing sub-agent model selection, `runTimeoutSeconds`, `sandbox: "require"` guidance, and explicit caveats about local tool-calling reliability. Link to existing Local models, Sub-agents, CLI backends, and Ollama pages rather than duplicating their full setup details.
