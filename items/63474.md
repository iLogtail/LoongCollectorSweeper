## Summary

Keep this PR open. Current main still lacks a migration that strips legacy `installs`/`plugins` from `channels.*` configs before strict channel validation, and the linked Feishu upgrade bug remains a plausible current-main failure. The PR is not ready to auto-close as implemented or superseded; it should be rebased/adapted to the current migration files and address the existing review concerns around non-Feishu/account-only detection.

## Best Possible Solution

Keep the PR open as a valid implementation candidate, but update it for current main by adding a channel legacy migration in `src/commands/doctor/shared/legacy-config-migrations.channels.ts` that strips `installs` and `plugins` from every `channels.<id>` block and nested `accounts.<id>` block, with legacy rules broad enough for doctor/read paths to surface and persist the repair. Keep or add Feishu schema rejection tests plus end-to-end migration tests through `migrateLegacyConfig`/current doctor compatibility flow, then resolve the existing review comments before merge.
