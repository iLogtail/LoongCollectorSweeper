## Summary

Keep open. Current main still does not include the answering provider/model in the per-response `/usage` footer. The related #70772 work adds `resolvedRef` observability for operator/tooling traces, but it does not change the user-visible response usage footer. The PR remains potentially useful, especially because its final diff uses the existing `formatProviderModelRef` helper, but it needs a current-main rebase and targeted formatter/runner tests.

## Best Possible Solution

Keep #56081 open unless maintainers intentionally consolidate it into the broader #40215/#48635 usage-footer work. The best implementation is a current-main rebase that passes `providerUsed` and `modelUsed` into `formatResponseUsageLine`, formats the label with `formatProviderModelRef`, preserves existing token/cache/cost/session ordering, and adds targeted tests for no-model, model-only, provider-only, provider/model, cost-enabled, and already-qualified model cases.
