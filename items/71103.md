## Summary

Keep open. Current `main` still intentionally treats any forwarded-header request as non-`direct_local`, and the WebSocket pairing bypass logic only covers `trusted-proxy` Control UI operator auth, not Tailscale-authenticated Control UI sessions. The maintainer comment on #71103 says the reported symptom is credible and points to a narrower fix path via PR #43613, so this is not implemented on `main` and should stay open.
