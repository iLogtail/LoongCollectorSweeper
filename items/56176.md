## Summary

Keep open. Current main still passes the raw `ndJsonStream` directly into `AgentSideConnection`, while the pinned ACP SDK validates `initialize.protocolVersion` as a numeric uint16. The linked bug report’s VS Code/Cursor logs therefore still describe a plausible current failure path, and this PR is a focused attempt to fix that path rather than obsolete cleanup.

## Best Possible Solution

Keep #56176 open and refresh it rather than closing. The best path is a small current-main patch in `src/acp/server.ts` that normalizes MCP date-string initialize `protocolVersion` values before `AgentSideConnection` validates them, plus a focused regression test around the stream transform or server handshake boundary. While refreshing, address the review nits with integer validation and a narrower stream type assertion.
