## Summary

Keep this issue open. Current main still keys classic voice-call auto-response sessions by normalized caller phone number, has no `sessionScope` config in the voice-call schema/manifest, and realtime consult still falls back to phone-number scope unless a call already has an explicit `sessionKey`. The related discussion is accurate: #59245 is broader outbound-task work and #71262 covered realtime tool bridging, but neither supersedes this per-call isolation request.

## Best Possible Solution

Implement this in the voice-call plugin, not core: add a strict `sessionScope` enum such as `per-phone` default and `per-call`, expose it in `VoiceCallConfigSchema`, `openclaw.plugin.json`, UI hints, and docs, resolve a call session key once when the call record is created, persist it as `call.sessionKey`, and use that same key for both classic `generateVoiceResponse` and realtime consult. Add focused tests for inbound classic calls and realtime consult proving repeated calls from the same phone isolate when `sessionScope: "per-call"` and preserve current behavior when unset.
