## Summary

Current main still lacks a user-configurable per-call session scope for the voice-call plugin. Classic response generation hardcodes the embedded agent session to the normalized caller phone number, realtime consult falls back to caller/callee phone when no explicit call sessionKey exists, inbound call records do not resolve/persist a per-call session key, and neither the Zod config schema nor manifest schema exposes `sessionScope`. Existing tests and docs also reflect the current phone-scoped behavior. The maintainer discussion already reached the same conclusion on April 25, 2026, so this should stay open.

## Best Possible Solution

Implement this in the official `voice-call` plugin rather than closing it: add a `sessionScope` enum such as `per-phone` default and `per-call`, expose it in `VoiceCallConfigSchema`, `openclaw.plugin.json`, UI hints, docs, and tests, resolve the session key once when a call record is created, persist it on `CallRecord.sessionKey`, and have both classic response generation and realtime consult consume that stored key.
