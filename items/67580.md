## Summary

Current `main` filters staged dreaming artifacts out of short-term recall and promotion, covers the regression with tests, and documents that dreaming diary/report output is not a promotion source. The fix appears present on `main` but not yet in the latest tagged release `v2026.4.23`.

## Evidence

- **Contaminated dreaming snippets are explicitly detected:** `isContaminatedDreamingSnippet` now treats `Candidate:`/`Reflection:` snippets with dreaming metadata (`confidence`, `evidence`, `status: staged`, `recalls`) as contaminated, including prefixed diff/bracket/bullet forms.
  - file: [extensions/memory-core/src/short-term-promotion.ts:275](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/short-term-promotion.ts#L275)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Short-term recall skips contaminated snippets:** When recording short-term recalls, contaminated snippets are discarded before entering the recall store, which breaks the feedback loop before ranking.
  - file: [extensions/memory-core/src/short-term-promotion.ts:914](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/short-term-promotion.ts#L914)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Promotion path also refuses contaminated snippets:** `applyShortTermPromotions` filters contaminated candidates before selection and again after live-note rehydration, preventing raw dreaming artifacts from being appended into `MEMORY.md`.
  - file: [extensions/memory-core/src/short-term-promotion.ts:1543](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/short-term-promotion.ts#L1543)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Regression tests cover the reported raw-candidate path:** Tests assert contaminated dreaming snippets are ignored during recall recording and are not appended during direct promotion apply.
  - file: [extensions/memory-core/src/short-term-promotion.test.ts:206](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/short-term-promotion.test.ts#L206)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Managed dreaming blocks are no longer re-ingested:** A dreaming-phase regression test verifies repeated light-dreaming runs do not re-ingest managed light blocks from daily notes, matching the feedback-loop part of the report.
  - file: [extensions/memory-core/src/dreaming-phases.test.ts:325](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/dreaming-phases.test.ts#L325)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Docs now state dreaming artifacts are excluded from promotion:** The Dreaming docs now explicitly say the Dream Diary/report artifacts are not a promotion source and only grounded memory snippets can promote into `MEMORY.md`.
  - file: [docs/concepts/dreaming.md:80](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/concepts/dreaming.md#L80)
