## Summary

Current `main` and the latest locally-available release tag already avoid the reported `spawn(..., args, { shell: true })` pattern in the shared command runner, so the gateway-startup DEP0190 report is obsolete against current OpenClaw.

## Evidence

- **Current main disables shell for argv-based execution:** `shouldSpawnWithShell()` explicitly documents that argv-based execution must not use `shell`, and currently returns `false` unconditionally.
  - file: [src/process/exec.ts:121](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/process/exec.ts#L121)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Gateway/runtime spawn path no longer enables `shell: true`:** `runCommandWithTimeout()` still has a conditional spread for `shell: true`, but it is gated by `shouldSpawnWithShell(...)`; with that helper returning `false`, this shared spawn path does not pass `shell: true` with an args array.
