## Summary

Keep open. Current main still renders the Control UI Config Raw mode as a native textarea, styles it as a textarea, and tests textarea-specific behavior. There is no CodeMirror/Monaco/Ace/jsoneditor dependency or integration, so the requested folding, syntax highlighting, formatting, and inline validation are still missing.

## Best Possible Solution

Keep the issue open and implement the Raw config editor as a first-class Control UI component, likely CodeMirror v6-backed for JSON/JSON5. Preserve current raw redaction, rawAvailable/Form-mode fallback, onRawChange, Reset to saved, save/apply, and safe round-trip semantics while adding folding, syntax highlighting, formatting, and parse/validation feedback. Update CSS and browser tests to assert the editor contract rather than native textarea behavior.
