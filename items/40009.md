## Summary

Keep open. Current main still renders the Config Raw mode editor as a native textarea, with textarea-specific styling and browser tests, and no CodeMirror/Monaco/Ace-style dependency or integration. The requested folding, syntax highlighting, formatting, and inline JSON validation are still missing.

## Best Possible Solution

Keep the issue open and implement the Raw config editor as a first-class Control UI component, likely CodeMirror v6-backed for JSON/JSON5 editing. Preserve the existing redaction, raw round-trip, reset, onRawChange, save/apply, and unavailable-snapshot semantics while adding folding, syntax highlighting, formatting, and parse/validation feedback. Update the current textarea-specific tests and CSS to assert the editor contract instead of native textarea behavior.
