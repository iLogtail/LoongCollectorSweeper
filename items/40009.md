## Summary

Keep open. Current `main` still renders the Config Raw mode editor as a native `<textarea>`, keeps textarea-specific styling and tests, and has no CodeMirror/Monaco/Ace integration in the relevant UI/package surfaces. The issue’s requested folding, syntax highlighting, formatting, and inline JSON validation remain unimplemented.

## Best Possible Solution

Keep the issue open and implement the Raw config editor as a first-class UI component, likely a CodeMirror v6-backed JSON/JSON5 editor, preserving redaction and `onRawChange`/save semantics while adding folding, syntax highlighting, formatting, and inline validation. Update the existing textarea-specific tests and styles to assert the editor contract instead of native textarea behavior.
