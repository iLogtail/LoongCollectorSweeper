## Summary

Keep open. The report’s persistent-file-descriptor explanation does not match current main, but current main still appears to miss effective daily rotation for long-lived subsystem loggers: the base logger re-resolves the date-based file path, while subsystem loggers cache a tslog child logger that keeps the original file transport.

## Evidence

- **Date-based log path is still the default:** `resolveSettings()` defaults `file` to `defaultRollingPathForToday()`, and `defaultRollingPathForToday()` builds `openclaw-YYYY-MM-DD.log` from the current local date.
  - file: [src/logging/logger.ts:376](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/logging/logger.ts#L376)
  - command: `nl -ba src/logging/logger.ts | sed -n '338,378p;564,574p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Base logger only rotates by rebuilding on settings change:** `getLogger()` compares cached settings to freshly resolved settings and rebuilds the logger when the file path changes, so date rollover depends on callers obtaining a fresh base logger.
  - file: [src/logging/logger.ts:486](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/logging/logger.ts#L486)
  - command: `nl -ba src/logging/logger.ts | sed -n '482,490p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Subsystem loggers cache a child file logger:** `createSubsystemLogger()` keeps `fileLogger` in closure state, initializes it once with `getChildLogger(...)`, and reuses that same child logger for later writes.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/logging/subsystem.ts#L327)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '325,354p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **tslog child loggers clone attached transports:** In tslog v4.10.2, the constructor copies `attachedTransports`, and `getSubLogger()` creates a new logger from the parent’s current settings. That means a cached child logger keeps the transport list it was created with instead of following later base-logger replacements.
  - command: `curl -L --silent https://raw.githubusercontent.com/fullstack-build/tslog/v4.10.2/src/BaseLogger.ts | nl -ba | sed -n '732,788p;888,910p'`
- **Docs still describe daily rolling logs:** User-facing docs still say the gateway writes a rolling daily log file under `/tmp/openclaw/openclaw-YYYY-MM-DD.log`, so the reported expectation matches current documentation.
  - file: [docs/logging.md:22](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/logging.md#L22)
  - command: `nl -ba docs/logging.md | sed -n '20,26p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)

## Risks / Open Questions

- Did not run a live midnight rollover reproduction in this read-only review; the conclusion is based on current source plus the tslog dependency contract.
- Some direct `getLogger()` call sites should re-resolve the daily path correctly, so the impact likely depends on whether a given logging path uses cached subsystem loggers or uncached top-level logger calls.
