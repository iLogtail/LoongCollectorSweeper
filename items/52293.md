## Summary

Keep open. Current `main` still lacks the `"transport endpoint is not connected"` matcher in the shared systemd-unavailable classifier, and the corresponding availability/fallback/hints coverage is still missing, so there is not strong evidence to auto-close this PR as already implemented, obsolete, or incoherent.

That means the central classifier used by daemon availability logic still misses the reported case on `main`.
