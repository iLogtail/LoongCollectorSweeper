## Summary

Keep #69311 open. Current main still only protects fenced code spans before extracting MEDIA directives, and a 4-space or tab-indented CommonMark code line still reaches the `trimStart()`-based MEDIA check. The linked PR #69312 is still open in the provided context, so the remaining bug is not shipped on main.

## Best Possible Solution

Keep the issue open and fix it via #69312 or an equivalent main-targeted patch. The best implementation should add CommonMark-aware indented-code handling in `splitMediaFromOutput()`, add focused regressions in `src/media/parse.test.ts` for 4-space and tab-indented `MEDIA:` lines, preserve code indentation when real media directives are also present, and re-check streaming directive accumulator assumptions because existing streaming tests currently assume indented `MEDIA:` can be treated as a directive. The secondary fence-closing behavior can be fixed in `src/markdown/fences.ts` with its own tests if it remains in scope.
