## Summary

Keep #14593 open. Current main still does not provide an official Linuxbrew/Homebrew path for the Docker setup flow, the reported `openai-whisper` bundled skill remains brew-only, and the skill installer still returns `brew not installed` when no brew executable is found. The issue also has a same-day maintainer/Codex keep-open comment and active unresolved fix-PR references in the provided discussion context.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely maintainer path is an official opt-in Linuxbrew setup for the non-root Docker image and setup script, OS-scoped or non-brew installer metadata for bundled brew-only skills such as `openai-whisper`, or onboarding/status behavior that avoids offering impossible brew installs in the shipped Docker flow. Use the active fix PRs and discussion (#57553, #42303, #55940) as input and validate across `Dockerfile`, `scripts/docker/setup.sh`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts`.
