## Summary

Keep #14593 open. Current main still does not provide an official Homebrew/Linuxbrew path for Docker onboarding, `openai-whisper` remains brew-only, and the installer still returns a `brew not installed` failure when no brew executable is present. The issue also has concrete same-day discussion saying it should remain open while an open fix PR is under review.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The strongest paths are an official opt-in Linuxbrew setup for the Docker image and setup script, OS-scoped non-brew installer metadata for bundled brew-only skills such as `openai-whisper`, or onboarding/status behavior that avoids offering impossible brew installs in the shipped Docker flow. Validate the chosen path across `Dockerfile`, `scripts/docker/setup.sh`, `docker-compose.yml`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts`.
