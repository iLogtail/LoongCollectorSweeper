## Summary

Keep open. Current `main` still deterministically fails for brew-only skills in the official Docker flow: the bundled `openai-whisper` skill exposes only a `brew` installer, onboarding still recommends/shows Homebrew when `brew` is missing, `installSkill()` still returns `brew not installed`, and the official Docker image/setup/docs still do not provide a Linuxbrew/Homebrew path. The issue discussion also points to active fix PRs that are not on `main` yet.

## Evidence

- **Bundled skill is still brew-only:** `openai-whisper` requires `whisper` and advertises a single installer entry with `kind: "brew"` / `formula: "openai-whisper"`, so Linux/Docker users without Homebrew have no alternate installer path for this skill.
  - file: [skills/openai-whisper/SKILL.md:10](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/skills/openai-whisper/SKILL.md#L10)

  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Installer selection still falls back to unavailable brew:** `selectPreferredInstallSpec()` avoids preferring brew when `brew` is missing, but it still returns `brewSpec` as a last resort specifically to surface a brew-missing error. For a brew-only skill, that means current `main` still selects brew on Linux/Docker.
