## Summary

Keep #14593 open. Current main still does not provide an official Docker/Linuxbrew setup path, the Docker setup script does not pass or persist brew-related build options, the reported openai-whisper skill remains brew-only, and the installer still returns a brew-not-installed failure when no brew executable is present. The issue also has a very recent maintainer/Codex keep-open signal while fix PRs remain unresolved.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely maintainer path is an official opt-in Linuxbrew setup that works for the non-root Docker image and is documented/tested, non-brew or OS-scoped installers for bundled brew-only skills like openai-whisper, or onboarding/status behavior that avoids offering impossible brew installs in the shipped Docker flow. Validate across Dockerfile, scripts/docker/setup.sh, docs/install/docker.md, skill metadata, installer behavior, and onboarding tests.
