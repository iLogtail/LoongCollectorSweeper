## Summary

PR #71115 proposes an optional subagent model-router implementation in core. Current main already has the generic plugin hooks needed to build this as a hook-only plugin: `before_model_resolve` can classify the subagent prompt and override provider/model, model-call hooks expose actual provider/model telemetry, and plugin config can hold router mode/command/policy/telemetry settings. OpenClaw's VISION keeps optional capability in plugins/ClawHub unless a missing generic plugin API is identified, and this review did not find such a missing API.

## Best Possible Solution

Do not add a core `agents.defaults.subagents.modelRouter` branch. Package this as a hook-only ClawHub plugin that uses `before_model_resolve` for shadow/live recommendations, plugin-owned config for mode/command/policy/telemetry path, and `model_call_started`/`model_call_ended` for actual-provider/model telemetry. If implementation exposes a concrete missing generic hook field, open a focused plugin-API issue/PR for that field instead of a one-off core router.
