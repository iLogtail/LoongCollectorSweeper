## Summary

Keep PR #39715 open. Current main still resolves child/supervisor waits and snapshots background exec output without the stdout/stderr drain barrier proposed by the PR, and the focused regression test file from the PR is not present. The related sandbox stdin/write fix does not supersede this stdout/background-output data-loss path.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregation are snapshotted, retain the settled/cancel/runId safeguards from the PR review discussion, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
