## Summary

Keep open. Current main still does not implement PR #39715's central stdout/stderr drain before exec/session output is snapshotted, and the PR's regression coverage is not present on main.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before RunExit/session aggregation is snapshotted, preserve the settled/cancel/runId-reuse guards already discussed on the PR, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
