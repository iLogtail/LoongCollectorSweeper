## Summary

Keep #70739 open. Current main still has no SSE heartbeat helper and the `/v1/chat/completions` and `/v1/responses` streaming paths only write bytes when real events arrive or when finalizing with `[DONE]`. The PR is a focused, recent gateway bug fix with concrete proxy-idle reproduction, existing repo precedent in the session-history SSE keepalive, and relevant hardening follow-up in the PR discussion. It still needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open. The best path is to finish review on #70739, address the small test/validation comments, and then land a narrow SSE heartbeat helper for `/v1/chat/completions` and `/v1/responses`; any broader streaming connection limits should be tracked separately in gateway auth/rate-limit handling.
