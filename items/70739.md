## Summary

Keep PR #70739 open. Current main still lacks an SSE heartbeat on the OpenAI-compatible streaming gateway endpoints, so the PR is not obsolete and addresses a real downstream proxy idle-timeout gap. It should not land exactly as-is until the remaining timer-normalization and test fake-timer cleanup review comments are handled.

## Best Possible Solution

Keep PR #70739 open and request a narrow revision: normalize `intervalMs` with a finite-number fallback before scheduling the interval, add fake-timer cleanup in the new tests, then land the SSE comment heartbeat for `/v1/responses` and `/v1/chat/completions` with the existing backpressure and write-error protections. Track WebSocket and upstream proxy idle-timeout work separately.
