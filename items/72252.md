## Summary

Keep #72252 open. Current main still has the reported heartbeat async-exec leak path: structured exec completions are treated as exec events, user-relay prompts are built whenever heartbeat has a delivery target, and exec completions are explicitly exempted from normal HEARTBEAT_OK/ack skipping. The linked #72253 PR is the right narrow fix, but it is still open and not present on main.

## Best Possible Solution

Keep #72252 open and land a narrow fix like #72253 on main: classify successful structured exec completions and SIGTERM cleanup completions as internal-only, allow normal heartbeat ack skipping for those cases, and continue relaying real failures such as non-SIGTERM code 1 completions. Leave the broader event-audience redesign to #69492.
