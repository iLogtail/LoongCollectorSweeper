## Summary

本 Issue 请求为 Loki Flusher 增加对 Grafana Loki 结构化元数据（Structured Metadata）的支持。经核查当前 main 分支代码，该插件序列化逻辑仅包含时间戳与日志正文，尚未实现 values 数组第三项的元数据组装。需求边界清晰、具备明确 API 规范与落地路径，属于合理且可执行的插件功能增强，无替代工单或关联 PR，符合保持开启条件。

## Best Possible Solution

建议在 plugins/flusher/loki 中新增配置项（如 StructuredMetadataKeys），允许用户指定需提取为结构化元数据的日志字段。在序列化推送载荷时，将匹配字段组装为 JSON 对象并附加至 values 数组的第三项。实现需同步更新插件配置文档、增加单元测试覆盖高基数元数据场景。建议维护者评估后分配里程碑或接受社区 PR。
