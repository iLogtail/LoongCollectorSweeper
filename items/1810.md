## Summary

本 Issue 明确请求为 Loki Flusher 增加对结构化元数据（Structured Metadata）的支持，提供了标准的 Loki Push API 载荷示例与官方文档链接。经保守评估，该需求属于对现有外部 Sink 插件的合理功能增强，描述清晰且具备可实施性。当前无证据表明 main 分支已内置该特性，亦无重复工单或插件替代方案。作为有效的特性请求，建议保持开启以待社区或维护者排期实现。

## Best Possible Solution

建议贡献者在现有 Loki Flusher 插件中扩展配置项，允许用户映射特定字段至 Loki Push API values 数组的第三项（结构化元数据对象）。实现时需同步更新插件文档与单元测试，验证高基数标签在不索引场景下的查询兼容性，并在 Release Notes 中注明新增能力。
