## Summary

本 Issue 请求为内置 Loki Flusher 增加对 Grafana Loki 结构化元数据（Structured Metadata）的支持。经核对当前 main 分支代码，该 Flusher 尚未实现 values 数组第三项的元数据映射功能。需求边界清晰，提供了标准 API 载荷示例与官方文档链接，属于合理且可实施的功能增强，无重复或替代工单，建议保持开启以待排期实现。

## Best Possible Solution

在现有 Loki Flusher 插件中新增配置项（如 StructuredMetadataKeys），允许用户指定需提取为结构化元数据的日志字段。在序列化推送载荷时，将匹配字段组装为 JSON 对象并附加至 values 数组的第三项。实现需同步更新插件配置文档、增加单元测试覆盖高基数元数据场景，并在 Release Notes 中说明兼容性。

## Evidence

- **feature_request_clarity:** Issue 正文提供了清晰的 Loki Push API JSON 示例与 Grafana 官方文档链接，明确指出了需在 values 数组中增加第三项结构化元数据对象。
- **current_implementation_status:** 检索当前 main 分支 Loki Flusher 相关源码，现有实现仅支持标准的 stream labels 与 log line 推送，尚未包含结构化元数据字段的解析与序列化逻辑。
