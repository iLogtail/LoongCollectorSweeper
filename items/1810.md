## Summary

本 Issue 请求为内置 Loki Flusher 增加对 Grafana Loki 结构化元数据（Structured Metadata）的支持。经核查，当前 main 分支的 flusher_loki 插件尚未实现该特性（values 数组仅包含时间戳与日志内容，未组装第三项元数据对象）。需求边界清晰，提供了明确的 API 载荷示例与官方文档参考，属于合理且可落地的插件功能增强，无替代工单或关联 PR，符合保持开启条件。

## Best Possible Solution

建议在 plugins/flusher/loki 中新增配置项（如 StructuredMetadataKeys），允许用户指定需提取为结构化元数据的日志字段。在序列化推送载荷时，将匹配字段组装为 JSON 对象并附加至 values 数组的第三项。实现需同步更新插件配置文档、增加单元测试覆盖高基数元数据场景。建议维护者评估后分配里程碑或接受社区 PR。
