## Summary

Keep open. Current main has a generic Discord startup cleanup that disconnects the gateway before lifecycle starts, which partially overlaps the PR title, but the bot-identity failure behavior discussed by this PR is still not implemented: fetchUser('@me') failures still degrade to an undefined botUserId, startup continues, and guild mention gating still short-circuits on botId. The PR also has substantive review concerns, so it needs maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open or supersede it with a focused maintainer-owned fix for the canonical Discord bot-identity startup bug: resolve botUserId reliably or fail closed, preserve the existing pre-lifecycle gateway cleanup, address reconnect suppression explicitly if needed, remove unsafe botId short-circuits where appropriate, and add deterministic regression coverage.
