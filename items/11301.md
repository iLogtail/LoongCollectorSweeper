## Summary

Keep #11301 open. Current main still does not add a runtime-container `/workspace -> /home/node/.openclaw/workspace` alias or mount; `/workspace` support remains scoped to agent sandbox containers, and the directly related PR #12764 is closed unmerged.

## Best Possible Solution

Keep #11301 open until a maintainer either implements a safe runtime Docker/Podman compatibility alias or bind mount for `/workspace -> /home/node/.openclaw/workspace` and aligns docs/smokes, or explicitly decides that `/workspace` is sandbox-only and updates Docker/runtime documentation plus issue expectations accordingly. PR #12764 is useful prior work but is closed unmerged, so it should not be treated as superseding the issue.
