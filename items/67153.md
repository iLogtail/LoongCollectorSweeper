## Summary

Keep #67153 open. Current main still contains the docs inconsistency the PR targets, the change is actionable in this repository, and there is no provided canonical duplicate or superseding main-side fix. The PR may need slightly stronger wording before merge because OpenClaw’s current parser is more permissive than the upstream Agent Skills spec.

## Best Possible Solution

Keep the PR open and ask for a small docs refinement: change the example to `hello-world`, replace the table’s `snake_case` text with spec-compatible language such as “must match the skill directory name, using lowercase letters, numbers, and hyphens,” and avoid implying runtime validation unless OpenClaw intentionally adds enforcement in a separate code change.
