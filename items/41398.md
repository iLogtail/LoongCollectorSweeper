## Summary

Keep open. Current main has canonical provider/model parsing for normal config paths, but the PR's Google-specific defensive fix is not implemented: Google model-id normalization still only maps preview aliases, the embedded resolver does not split a raw `google/gemini-*` modelId at entry, the Google transport still builds `models/${model.id}`, and the native PDF path still encodes `params.modelId` directly. The PR remains plausible useful work, though it needs rebasing and removal of unrelated baseline churn.

## Best Possible Solution

Keep the PR open or supersede it with a fresh, current-main patch that adds provider-owned Google prefix normalization in the shared/plugin helper, verifies resolver and direct transport/PDF paths with regression tests, and drops unrelated `.secrets.baseline` changes.
