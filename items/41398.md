## Summary

Keep open. Current `main` still lacks the Google model-prefix stripping described by PR #41398, and the remaining code paths would still send or look up `google/gemini-*` IDs without normalizing them to bare Gemini IDs.

## Evidence

- **`resolveModel()` still keeps prefixed Google IDs:** `resolveModel()` normalizes with `normalizeStaticProviderModelId(normalizeProviderId(provider), modelId)` and never splits a `provider/model` ref first. With `provider="google"` and `modelId="google/gemini-2.5-pro"`, the model ID stays prefixed for registry lookup and error paths.
  - file: [src/agents/pi-embedded-runner/model.ts:718](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-embedded-runner/model.ts#L718)
  - command: `nl -ba src/agents/pi-embedded-runner/model.ts | sed -n '714,748p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Native Gemini PDF path still uses raw `params.modelId` in the REST URL:** The Google PDF helper builds `/v1beta/models/${encodeURIComponent(params.modelId)}` directly, so a prefixed ID would still become `models/google%2Fgemini-...` instead of a bare Gemini model name.
  - file: [src/agents/tools/pdf-native-providers.ts:153](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/tools/pdf-native-providers.ts#L153)
  - command: `nl -ba src/agents/tools/pdf-native-providers.ts | sed -n '141,156p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Current Google model normalizer does not strip `google/` or `models/` prefixes:** The bundled Google plugin's `normalizeGoogleModelId()` only remaps preview aliases like `gemini-3.1-flash` and otherwise returns the input unchanged. It does not remove `google/`, `gemini/`, or `models/` prefixes.
  - file: [extensions/google/model-id.ts:3](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/extensions/google/model-id.ts#L3)
  - command: `nl -ba extensions/google/model-id.ts | sed -n '1,40p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Static shared Google normalization also lacks prefix stripping:** The shared `normalizeGooglePreviewModelId()` helper used by core static model normalization likewise only remaps preview aliases and returns other IDs unchanged.
  - file: [src/plugin-sdk/provider-model-id-normalize.ts:3](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/plugin-sdk/provider-model-id-normalize.ts#L3)

  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)

  - file: [docs/providers/google.md:303](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/providers/google.md#L303)
