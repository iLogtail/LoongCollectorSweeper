## Summary

Current `main` still does not support Workers AI model selection for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. The bundled Cloudflare AI Gateway plugin remains Anthropic-only, and the CLI exposes only account/gateway/key flags for that auth choice. The generic custom-provider flow is a partial workaround, but it does not satisfy the requested Cloudflare-specific onboarding path or automatic Workers AI provider/model setup.

## Best Possible Solution

Keep #10480 open. A proper fix likely belongs in the bundled Cloudflare AI Gateway provider plugin: add an explicit Workers AI mode or model selector for non-interactive onboarding, generate an OpenAI-compatible `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` provider config with `@cf/...` model IDs, update the Cloudflare provider and onboarding automation docs, and add focused tests for the generated provider/model config. If maintainers decide the generic custom-provider flow is the intended path instead, document that product decision and the exact Workers AI command rather than closing this as already implemented.
