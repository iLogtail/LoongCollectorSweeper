## Summary

Current main still does not implement #10480. The Cloudflare AI Gateway onboarding path remains Anthropic-only and lacks a Workers AI mode, provider-type selector, or Cloudflare-specific model flag for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. A generic custom-provider onboarding flow can configure an OpenAI-compatible endpoint, but that remains a workaround and does not satisfy the requested bundled Cloudflare onboarding behavior. The provided issue discussion also includes a maintainer comment from 2026-04-26 explicitly keeping this open for the same gap.

## Best Possible Solution

Keep #10480 open. The fix should live in the bundled `cloudflare-ai-gateway` provider plugin: add an explicit Workers AI mode or non-interactive model selector for Cloudflare AI Gateway onboarding, generate an OpenAI-compatible Workers AI provider config using `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` and `@cf/...` model IDs, update the Cloudflare onboarding docs, and add focused tests for the generated provider/model config. If maintainers decide the generic `custom-api-key` flow is the intended answer, document that product decision with a concrete Workers AI command rather than closing this as already implemented.
