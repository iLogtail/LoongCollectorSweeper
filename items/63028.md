## Summary

Current `main` treats WeChat as an external channel plugin owned outside the OpenClaw core repo. The reported missing `allowFrom` file is therefore plugin-scope, and core already has generic pairing storage that creates the credentials directory when it writes pairing state.

## Evidence

- **WeChat is documented as an external plugin, not core code:** Current WeChat docs state that the WeChat code does not live in the OpenClaw core repo, and that login, Tencent iLink API calls, media handling, context tokens, and account monitoring are owned by the external plugin runtime.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/docs/channels/wechat.md#L27)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **WeChat DM access is documented through pairing/allowlist flow:** The same docs say direct messages use the normal OpenClaw pairing and allowlist model for channel plugins, with approval via `openclaw pairing list openclaw-weixin` and `openclaw pairing approve openclaw-weixin <CODE>`. That means successful login is not, by itself, the documented core approval step for inbound DMs.
  - file: [docs/channels/wechat.md:84](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/docs/channels/wechat.md#L84)

  - file: [src/pairing/pairing-store.ts:79](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/pairing/pairing-store.ts#L79)
