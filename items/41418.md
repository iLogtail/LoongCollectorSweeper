## Summary

Keep #41418 open. Current main does not implement a global `openclaw agent --dry-run` mode, the universal Pi tool adapter still executes tools directly, and the existing alternatives are scoped dry-runs or tool omission/blocking rather than structured preview results for all agent tool calls.

## Best Possible Solution

Keep #41418 open for maintainer/product judgment. The best implementation path is a core agent-runtime feature, not cleanup closure: add a CLI-visible dry-run flag and thread it through the agent command/run context to the shared tool execution point, intercept after policy/before-tool hooks but before `tool.execute`, return a structured preview result to the model, log redacted would-execute payloads to stderr, and cover normal tools plus client tools with focused adapter and CLI tests. Gateway support should be designed separately because process-global state is unsafe for concurrent gateway runs.
