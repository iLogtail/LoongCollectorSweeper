## Summary

Current `main` does not implement a global `openclaw agent --dry-run` mode or a universal dry-run interception in the agent tool adapter. The issue still describes missing functionality rather than something already fixed on `main`.

## Evidence

- **Agent CLI has no `--dry-run` flag:** `registerAgentCommands()` defines `openclaw agent` options such as `--message`, `--local`, `--deliver`, `--json`, and `--timeout`, but no `--dry-run` option or startup handling for one.
  - file: [src/cli/program/register.agent.ts:25](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/program/register.agent.ts#L25)
  - command: `nl -ba src/cli/program/register.agent.ts | sed -n '1,120p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Agent docs also omit `--dry-run`:** The CLI reference for `openclaw agent` lists all supported options and examples; `--dry-run` is not documented there.
  - file: [docs/cli/agent.md:23](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/cli/agent.md#L23)
  - command: `nl -ba docs/cli/agent.md | sed -n '1,220p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Universal tool funnel still executes tools directly:** `toToolDefinitions()` runs before-tool-call hooks and then immediately calls `tool.execute(...)`; there is no global dry-run branch returning preview payloads. `toClientToolDefinitions()` likewise has no dry-run handling and only returns the existing `pending` client-delegation result.
  - file: [src/agents/pi-tool-definition-adapter.ts:216](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-tool-definition-adapter.ts#L216)

- **Current dry-run support is scoped, not global:** The agent message tool schema includes its own optional `dryRun` parameter, showing per-tool dry-run support already exists. Separately, the Gateway `/tools/invoke` docs state `dryRun` is 'reserved for future use; currently ignored,' which confirms there is still no universal execution-level dry-run.
