## Summary

Current `main` still has a real store-vs-runtime split for Claude CLI auth. `openclaw models status` builds OAuth health from the auth-profile store without external CLI overlays, while the Anthropic runtime can use live Claude CLI credentials directly. Docs and the Claude doctor command both reflect that distinction, so this issue is not implemented-on-main and should stay open.

## Evidence

- **`models status` uses the store-only auth path:** The command imports `ensureAuthProfileStoreWithoutExternalProfiles`, loads that store, then feeds it into `buildAuthHealthSummary`, so the reported OAuth health comes from persisted auth profiles rather than live external CLI credentials.
