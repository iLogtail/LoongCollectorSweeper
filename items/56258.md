## Summary

Keep open. Current main has some overlapping gateway shutdown hardening for MCP, WebSocket, HTTP close, and channel task abort waits, but it does not implement this PR’s central per-subsystem timeout behavior across the close handler. The PR remains a plausible focused gateway reliability fix, with unresolved review concerns that need maintainer/author handling rather than auto-closure.

## Best Possible Solution

Keep the PR open for maintainer review or author rebase. The best path is to integrate per-subsystem shutdown budgets for the close-handler sequence while preserving HTTP listener close semantics for in-process restart, guarding timed-out promises against late rejection, and adding regression tests for hung subsystems and late rejection after timeout.
