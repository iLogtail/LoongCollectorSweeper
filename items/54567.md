## Summary

Keep open. Current main still runs the non-interactive daemon service install before `ensureSystemdUserLingerNonInteractive`, while the Linux service install path immediately reaches `systemctl --user enable`. The PR remains a focused, plausible fix for the linked headless Ubuntu report, and the existing tests do not cover the requested ordering.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land the linger-before-install behavior with a focused regression test for `installGatewayDaemonNonInteractive`, then decide whether direct `openclaw gateway install` needs the same pre-install linger handling.

## Risks / Open Questions

- The PR may still need a small ordering regression test before merge, because current tests would not catch this bug returning.
- `ensureSystemdUserLingerNonInteractive()` returns when `isSystemdUserServiceAvailable()` is false, so maintainers should validate the exact headless Ubuntu failure mode before treating this as the complete fix.
- The direct `openclaw gateway install` path also calls `service.install()` without a pre-install linger step, so maintainers should decide whether the fix should cover that surface too.
