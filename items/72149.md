## Summary

Keep open. Current main still forwards `options.temperature` into Anthropic Messages payloads whenever thinking is disabled, with no Claude Opus 4.7 guard. The PR is a focused attempt to fix a credible request-shaping bug backed by #72123 live v2026.4.24 logs and by Anthropic's Opus 4.7 migration guidance, so it is not implemented, non-actionable, stale, or safely superseded on main.

## Best Possible Solution

Keep #72149 open, or replace it with an equivalent maintainer patch, that filters unsupported sampling params for Claude Opus 4.7 in `src/agents/anthropic-transport-stream.ts`. At minimum, omit `temperature` for `claude-opus-4-7`/`claude-opus-4.7` before payload construction and add focused tests proving Opus 4.7 omits temperature while non-Opus models keep existing behavior; also confirm whether a broader `top_p`/`top_k` guard is needed on any OpenClaw-owned Anthropic transport path.
