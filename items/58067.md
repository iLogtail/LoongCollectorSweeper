## Summary

Current `main` still appears to match the report: `sessions_yield` intentionally ends the parent run as `end_turn`, the gateway emits a terminal chat event for that parent run, the Control UI clears its active streaming state on that terminal event, and subagent parent streaming remains ACP-only.

## Evidence

- **sessions_yield leaves parent idle:** Current test coverage codifies that `sessions_yield` completes the parent run with `stopReason = "end_turn"`, leaves no pending tool calls, and makes the parent session idle immediately afterward.
  - file: [src/agents/pi-embedded-runner/sessions-yield.orchestration.test.ts:28](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/pi-embedded-runner/sessions-yield.orchestration.test.ts#L28)

- **gateway sends terminal chat event on lifecycle end:** The gateway maps lifecycle `end`/`error` events into `emitChatFinal(...)` for Control UI-visible runs without any special handling for `stopReason: end_turn`, so a yielded parent run is treated as terminal from the chat surface's perspective.
  - file: [src/gateway/server-chat.ts:598](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-chat.ts#L598)
