## Summary

Keep PR #69906 open. Current main still lacks the WhatsApp multi-account `groups` inheritance guard, the docs explicitly describe the old WhatsApp behavior as still present, and the linked maintainer issue #69874 remains the canonical product decision for this breaking change.

## Best Possible Solution

Keep #69906 open for maintainer review or landing, or have a maintainer apply an equivalent fix: mirror Telegram’s multi-account `groups` guard in WhatsApp account resolution, keep `direct` unchanged unless #69874 is explicitly expanded, add the proposed WhatsApp regression tests, and ship the docs/changelog migration note including the `openclaw doctor` allowlist warning.

## Risks / Open Questions

- The change is intentionally breaking for multi-account WhatsApp configs that relied on root `groups` inheritance; maintainer review should preserve clear migration docs and the doctor-warning mitigation discussed on the PR.
- The Aisle/Greptile discussion identifies a narrow fail-open allowlist side effect when root `groups` was used as an implicit allowlist under `groupPolicy="open"`; deciding whether to fail closed across WhatsApp and Telegram is a separate product/security call.
