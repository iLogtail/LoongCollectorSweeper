## Summary

Issue #41100 is already fixed on current `main`: safeguard cancellations now preserve a human-readable reason, the compaction runner replaces generic `Compaction cancelled` with that reason, and manual `/compact` replies classify benign no-op cases as skipped instead of failed. The changelog records this exact fix in release `2026.3.28` under PR/reference `#51072`.
