## Summary

Keep PR #43951 open. Current main still has the reported `/hooks/agent` accountId gap: hook payload normalization omits `accountId`, the gateway hook dispatcher hand-builds a `CronJob` delivery object without it, and the public webhook docs list no `accountId` field. The PR is a focused bug-fix with a detailed repro, tests, and author responses to bot review comments, so it is not obsolete or low-signal. It does need a current-main rebase because the cron message-tool policy code has moved on since the PR branch.

## Best Possible Solution

Keep #43951 open and request a current-main rebase. The best fix remains in core: parse and trim `accountId` in `normalizeAgentPayload`, include it in direct and mapped hook replay scopes, preserve it into `CronJob.delivery` through the cron normalization path or an equivalent current-main construction, keep hook one-shot jobs from deleting reused hook sessions if normalization is used, and add focused tests proving `/hooks/agent` propagates `accountId` into `job.delivery`. Update the public webhook docs and any changelog entry with the required attribution before merge.
