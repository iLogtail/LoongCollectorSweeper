## Summary

Keep open. Current main intentionally clears the gateway token when the WebSocket URL meaningfully changes, and tests lock that behavior in place. This PR removes that guard and would reuse the old credential for a new gateway target.

## Evidence

- **Overview handler clears token on URL change:** The current Overview input handler resets `token` unless the edited URL is trim-equal to the existing URL, so only whitespace-only edits preserve the token.
  - file: [ui/src/ui/views/overview.ts:271](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/views/overview.ts#L271)
  - command: `nl -ba ui/src/ui/views/overview.ts | sed -n '258,276p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Current browser test asserts token must clear after gateway change:** `navigation.browser.test.ts` explicitly verifies that after editing the gateway URL to `wss://other-gateway.example/openclaw`, `settings.token` becomes an empty string.
  - file: [ui/src/ui/navigation.browser.test.ts:311](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/navigation.browser.test.ts#L311)
  - command: `nl -ba ui/src/ui/navigation.browser.test.ts | sed -n '311,340p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Connect path sends the current settings token to the configured gateway:** `connectGateway` constructs `GatewayBrowserClient` with `url: host.settings.gatewayUrl` and `token: host.settings.token`, so preserving the token across a real URL change would send the previous gateway credential to the new endpoint.
  - file: [ui/src/ui/app-gateway.ts:279](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/app-gateway.ts#L279)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '274,282p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Main already treats tokens as gateway-scoped state:** Storage keys session tokens by normalized gateway URL, and the navigation tests keep incoming tokens pending until a gateway change is confirmed. That matches the current security model: tokens are scoped to a gateway target, not preserved across target changes.
  - file: [ui/src/ui/storage.ts:122](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/storage.ts#L122)

## Risks / Open Questions

- Merging the PR as written would silently reuse a gateway credential for a different WebSocket URL and could send it to the wrong server.
- The PR's new test expectation conflicts with current main's explicit behavior and existing regression coverage, so it is not conservative maintainer cleanup.
