## Summary

Keep open. Current main has the Gateway primitives and a separate Sessions control table, but the requested ChatGPT-style Control UI session sidebar is still not implemented: the Chat plus button still sends `/new` into the active session instead of creating and switching to a new persisted session, and session list/search/rename/delete live in the separate Sessions tab rather than beside the chat thread.

## Best Possible Solution

Keep the issue open and implement it in the Control UI chat surface: add a left session sidebar backed by `sessions.list`/`sessions.preview`, make the plus button call `sessions.create` and switch to the returned key, load history with `chat.history` on selection, support label editing through `sessions.patch`, and wire delete/archive only once the corresponding Gateway contract is present. #51802 looks like the closest current PR to use or supersede, but it should be updated to the current `sessions.create` API and covered with focused UI/controller tests.
