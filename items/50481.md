## Summary

Keep #50481 open. Current main has Slack assistant thread status support, but no `assistant.threads.setSuggestedPrompts` call, no `assistantPrompts` config surface, no dynamic message action, and no tests/docs for suggested prompts. The related ClawSweeper items are adjacent Slack/channel reports, not a canonical duplicate or shipped implementation for this request.

## Best Possible Solution

Keep the issue open and implement it in the bundled Slack channel plugin, not generic core. A conservative first pass would add a typed `channels.slack.assistantPrompts` config with schema/docs/tests and call `client.assistant.threads.setSuggestedPrompts` after a successful reply when a delivered Slack thread timestamp is known. The stretch goal can be a Slack-owned message action after validating the action name, payload limits, account scoping, and tests.
