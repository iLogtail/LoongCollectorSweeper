## Summary

Keep #17217 open. Current main documents session lifecycle and provides `/reset` plus `sessions.reset`/`sessions.compact`, but cache-TTL pruning is still tool-result scoped, event-driven, and does not expire normal user/assistant history by wall-clock age. There is also no explicit `sessions.prune` or prune-context API surface.

## Best Possible Solution

Keep the issue open for maintainer product work. A complete resolution should decide whether `contextPruning.ttl` is intended to expire full conversation history or only gate tool-result cache pruning. If full TTL pruning is desired, add an explicit session-control prune surface, implement age-based removal or summarization in the Pi/context-engine path, and cover it with source tests and docs. If that behavior is intentionally out of scope, update user-facing docs to state that TTL does not expire conversation messages and that `/reset` and `/compact` are the supported controls.
