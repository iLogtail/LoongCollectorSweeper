## Summary

Keep #17217 open. Current main documents session lifecycle and provides manual reset/compact controls, but the requested wall-clock TTL pruning of normal conversation history and a manual prune-context API are still not implemented. The existing cache-ttl path is in-memory, runs only during context handling, and only trims/clears old toolResult messages.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether contextPruning.ttl should expire/summarize normal conversation history or remain a tool-result prompt-cache optimization. If full TTL pruning is desired, add an explicit gateway/protocol prune surface and implement age-based removal or summarization in the Pi/context-engine path with tests and docs. If current behavior is intentional, update user-facing docs to make that limitation unambiguous and point users to /compact, /new, and /reset as the supported controls.
