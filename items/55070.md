## Summary

Keep open. Current `main` still lacks the abort-time native-command guard described in the report, and the cross-linked PR `#55084` contains exactly that missing fix rather than something already merged.

## Evidence

- **No native-command lifecycle guard on main:** `createDiscordNativeCommand()` has no `guard` parameter, and `Command.run()` immediately defers and dispatches the interaction without any `isActive()`/generation check.
  - file: [extensions/discord/src/monitor/native-command.ts:634](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/native-command.ts#L634)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Misleading allowlist reply path still present:** The current slash-command path still emits `"This channel is not allowed."` from the native command handler, matching the user-visible symptom in the issue.
  - file: [extensions/discord/src/monitor/native-command.ts:892](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/native-command.ts#L892)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Provider shutdown only deactivates message handler:** `monitorDiscordProvider()` builds native commands with no shutdown guard, then only calls `deactivateMessageHandler?.()` in `finally`; there is no abort listener or native-command deactivation path before teardown completes.
  - file: [extensions/discord/src/monitor/provider.ts:823](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/provider.ts#L823)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Cross-linked PR contains the missing fix, not main:** Remote PR `#55084` patch adds a `guard?: { isActive: () => boolean }` to `createDiscordNativeCommand()`, checks it at the top of `Command.run()`, and installs an abort listener in `monitorDiscordProvider()` to flip the guard off during shutdown. Those hunks are absent from current `main`.
  - command: `curl -L --silent https://patch-diff.githubusercontent.com/raw/openclaw/openclaw/pull/55084.patch | sed -n '1,260p'`
  - sha: [5d9df839dd6c](https://github.com/openclaw/openclaw/commit/5d9df839dd6c878ac9732a79423cf6828a15d484)
- **PR still separate from current checkout:** The issue timeline references commit `5d9df839dd6c878ac9732a79423cf6828a15d484`, and `git ls-remote` shows it as `refs/pull/55084/head`; current `main` is `34896839ba22f3a71a3d5a8fb4ca5f4c6989a534`, so the proposed fix is not part of this checkout.
  - command: `git ls-remote origin 'refs/pull/55084/head' 'refs/pull/55084/merge'`
