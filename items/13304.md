## Summary

Keep #13304 open. Current main still routes normal bindings by static channel/account/peer/guild/team/role metadata only, and neither the strict binding schema nor the plugin hook contracts provide the requested content matcher, prefix stripping, target agent override, or transformed BodyForAgent for normal dispatch. The linked #10969 and #11220 are model-routing requests, not canonical trackers for content-based agent routing.

## Best Possible Solution

Keep the issue open. Implement either bindings[].match.content with deterministic prefix/regex matching and stripMatch behavior, or add a documented pre-routing plugin hook that can select the target agent, session behavior, and transformed BodyForAgent before normal dispatch. Update config types, strict zod schema, generated config help/docs, resolver precedence, representative channel call sites, and route/channel regression tests.
