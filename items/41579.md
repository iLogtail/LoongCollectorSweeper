## Summary

Keep open. Current `main` does not implement a Mission Control left-nav `File Gallery`, and the existing agent file UI is a narrow editor for a fixed set of core markdown workspace files rather than a docs/images gallery with `Refresh Gallery` and `Open in Browser` actions.

## Evidence

- **No top-level File Gallery nav:** Control UI top-level tabs are limited to chat/control/agent/settings groups. There is no `fileGallery` tab or route in the main left-nav definitions.
  - file: [ui/src/ui/navigation.ts:5](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/ui/src/ui/navigation.ts#L5)
  - command: `nl -ba ui/src/ui/navigation.ts | sed -n '1,80p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Only nested agent Files tab exists:** The only related UI entry is the nested `Files` panel inside the `Agents` screen, alongside `Overview`, `Tools`, `Skills`, `Channels`, and `Cron Jobs`. That is not the permanent Mission Control left-nav item requested in the issue.
  - file: [ui/src/ui/views/agents.ts:345](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/ui/src/ui/views/agents.ts#L345)

- **Current Files panel is not a gallery:** `renderAgentFiles` is labeled `Core Files` with subtitle `Bootstrap persona, identity, and tool guidance.` It offers refresh, markdown preview, reset, and save for selected files. There is no gallery UI, image/doc browser, or `Open in Browser` action.
