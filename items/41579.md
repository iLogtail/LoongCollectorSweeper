## Summary

Keep #41579 open. Current main does not implement a permanent Mission Control/Control UI File Gallery left-nav entry, and the existing file UI only edits a small allowlisted set of agent bootstrap files. A ClawHub close is not well-supported because the current plugin API exposes gateway/plugin capabilities and HTTP routes, but not a Control UI left-nav or panel extension point for this requested in-app surface.

## Best Possible Solution

Keep the issue open for maintainer product/API judgment. A complete path would add either a first-party Control UI File Gallery tab backed by a safe workspace file-index RPC, or a documented plugin UI contribution API that can add a left-nav/panel surface while preserving workspace path confinement, refresh/open-in-browser actions, and friendly empty/error states.
