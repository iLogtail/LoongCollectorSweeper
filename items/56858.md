## Summary

Keep PR #56858 open. Current main does not implement the requested opt-in media cache: context pruning still replaces image blocks with text placeholders or hard-clear placeholders, there is no `cacheMedia` config key, no `src/media/media-cache.ts`, and image-reference detection has no `[media cached: ...]` marker support. The related #53869 was closed in favor of this PR, not merged or shipped, so it is not a canonical implemented/superseding item.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase it onto current `main`, adapt the implementation to `src/agents/pi-hooks/context-pruning/*`, add the `cacheMedia` config/schema/docs surface, implement safe disk caching and `[media cached: ...]` rehydration, and keep focused tests for soft-trim, hard-clear, soft-trim plus hard-clear, multi-image ordering, MIME normalization, and cache-write races. If maintainers decide the feature is too broad for core, that should be an explicit product decision rather than an automated cleanup close.
