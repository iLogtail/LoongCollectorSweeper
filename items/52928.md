## Summary

Keep open. Current main still does not expose an `agents.setDefault` Gateway RPC: the agent handler, public method list, scope table, protocol schema, validators, docs, and tests only cover `agents.list/create/update/delete` plus file helpers. The request also remains product-relevant because the Control UI has a `Set Default` button, but it still writes `agents.defaultId` into the config form while the strict config schema models the default as `agents.list[].default`. The linked #59070 is a concrete symptom of that mismatch, not a shipped replacement for this feature.

## Best Possible Solution

Add `agents.setDefault` as an additive Gateway protocol method: define params/result schemas and validators, register the handler and ADMIN scope, update docs and generated protocol surfaces, add focused handler/protocol tests for success, unknown agent rejection, and clearing other `agents.list[].default` flags, then switch the Control UI Set Default action away from `agents.defaultId`. That would also give #59070 a clean fix path for the current UI failure.
