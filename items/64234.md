## Summary

Keep PR #64234 open. Current main does not ship the proposed privacy config, PII redaction hooks, media blocking, system-prompt masking, or session encryption at rest. The PR overlaps with the open LLM-boundary privacy PR #45783, but it is not fully duplicated because this PR also proposes core session-file encryption/media controls. The existing review discussion identifies serious correctness and performance issues, so this is maintainer design/splitting work, not conservative auto-close cleanup.

## Best Possible Solution

Keep open for maintainer judgment, but ask for a focused redesign/split: either route the LLM-boundary PII work through the canonical privacy-filter discussion (#45783) and keep only a smaller core at-rest/media/system-prompt PR here, or replace both with one maintainer-approved privacy design that adds documented config/schema, async crypto, transcript read/write compatibility, and targeted tests across embedded runner, CLI runner, messaging channels, TTS, tool output, and session transcript consumers.
