## Summary

Keep open. Current main has partial preservation plumbing, but the reported `openclaw agents add` behavior is not fully implemented: provider setup still returns `agentModelOverride` when `setDefaultModel` is false, and the add command still writes that override into the agent entry. The PR may need a rebase and test cleanup, but it is not obsolete on current main.

## Best Possible Solution

Keep the item open and ask for a rebased fix against current main. The implementation should update the three `setDefaultModel: false` paths to return `agentModelOverride` only when no inherited `agents.defaults.model` primary exists, preserve provider config/allowlist additions, remove misleading agent-scoped notes, and add current regression coverage for `openclaw agents add` with an existing defaults primary.
