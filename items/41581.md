## Summary

Keep open. Current main still routes active Telegram DM preview streaming through message transport (`sendMessage` + `editMessageText`) instead of letting DM auto transport use `sendMessageDraft`, and the regression tests explicitly lock in that behavior. The linked fix PRs #62207/#62210 attempted the same change but are closed and unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #41581 open. The likely fix belongs in `extensions/telegram/src/bot-message-dispatch.ts`: stop forcing `previewTransport: "message"` for Telegram DMs when partial preview streaming is active, while preserving the duplicate-finalization guard that motivated the current fallback. Update `extensions/telegram/src/bot-message-dispatch.test.ts` to expect DM `previewTransport: "auto"` or draft behavior, and keep coverage for final delivery without duplicate messages.
