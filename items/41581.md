## Summary

Keep #41581 open. Current main still forces Telegram DM partial preview streaming through the message/editMessageText transport for answer streaming, and the current tests and Telegram docs explicitly lock/describe direct-chat streaming as preview message edits. The related #62207/#62210 PRs identified the same root cause but were closed unmerged, and v2026.4.24 still contains the forced message transport.

## Best Possible Solution

Keep the issue open and fix the dispatcher transport selection deliberately: restore native sendMessageDraft/auto transport for DM answer partial streaming when it is safe, keep or explicitly justify message transport for reasoning lanes, and add regression coverage that proves smooth DM answer previews do not reintroduce duplicate final sends or broken reasoning/answer lane finalization.
