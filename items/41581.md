## Summary

Keep open. Current `main` still forces Telegram DM preview streaming onto `sendMessage`/`editMessageText` instead of the DM `sendMessageDraft` path, and the latest shipped release `v2026.4.22` still carries that behavior. The issue looks real and not fixed on `main`.
