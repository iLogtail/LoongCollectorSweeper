## Summary

Keep open. Current main has only a partial zero-default mitigation; it still exposes poll fields on the shared message tool and still rejects `action: "send"` before dispatch whenever non-empty/true/non-zero poll creation fields are present. That matches the central #42820 failure mode rather than an implemented fix. The provided related context shows #46455 is an open unmerged PR for the fuller strip/sanitize approach, while #52141 only addressed zero-valued defaults.

## Best Possible Solution

Keep #42820 open until the message tool either sanitizes/strips poll creation params on `action: "send"`, splits send and poll into narrower LLM-facing schemas, or scopes poll fields to channels/actions that support them. #46455 is the closest known open implementation attempt, but it should be reviewed/updated and merged or superseded before closing this issue; #52141 should be treated as a partial zero-default fix only.
