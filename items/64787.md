## Summary

Keep open. Current main still hard-rejects `sessions_spawn` with `runtime: "subagent"` plus `streamTo: "parent"`, while the linked issue describes strict/schema-materializing providers auto-filling that single-value enum. The PR is a focused fix for a real current behavior, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR, or an equivalent canonical PR, open and land a scoped fix that ignores `streamTo` for non-ACP `sessions_spawn` calls while preserving ACP `streamTo="parent"` behavior, plus a regression test for the strict-provider auto-fill shape and an ACP-only schema description.
