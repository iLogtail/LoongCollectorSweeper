## Summary

Keep #70777 open. The report is recent and includes concrete transcript/config evidence. Current main resolves configured workspaces correctly for normal embedded runs, but ACP session rehydration still treats persisted runtimeOptions.cwd/meta.cwd as authoritative, and there is no clear repair path or regression test proving a stale wrong cwd is corrected for the main ACP session case.

## Best Possible Solution

Keep the issue open and fix it in the ACP/session lifecycle. Derive the desired cwd for default/main ACP sessions from agents.list[].runtime.acp.cwd when present, otherwise from resolveAgentWorkspaceDir for the session agent, then reconcile persisted ACP meta/runtimeOptions before reusing a runtime handle. Add a regression test where a stored main ACP session has cwd=/home/user while agents.defaults.workspace points at /home/user/.openclaw/workspace, and verify the next turn reinitializes or repairs to the workspace cwd. If automatic mutation is risky, add an explicit doctor/fix repair path plus targeted reinitialization for mismatched configured workspaces.
