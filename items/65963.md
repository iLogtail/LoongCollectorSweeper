## Summary

Keep open. Current `main` has reduced part of the original pain: dreaming now defaults to one daily sweep and `memory-core` now best-effort deletes/scrubs stale dreaming narrative sessions. But the core asks in #65963 are still not implemented as user-facing behavior: the public plugin schema/docs still do not expose a `dreaming.model` knob, `openclaw sessions` still has cleanup-only maintenance with no `purge` subcommand or `--kind dreaming` filter, and the session picker/list surfaces show no dreaming-specific hiding/collapse behavior. The issue is also recent and linked to still-open follow-up work (#65589, #69320), so conservative cleanup should leave it open.
