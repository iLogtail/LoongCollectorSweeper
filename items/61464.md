## Summary

Keep #61464 open. Current main does not already ship the Mac Docker keep-awake helper, the Docker-specific migration helper scripts, or a restore workflow equivalent to this PR. The existing backup CLI and generic migration docs partially overlap with the backup problem, but they do not make the PR obsolete, and the PR discussion shows the author addressed several concrete review comments rather than leaving a low-signal drive-by.

## Best Possible Solution

Keep the PR open for maintainer review. The strongest path is to rebase it on current main, decide whether the keep-awake helper belongs standalone or under ClawDock, align the migration docs with the existing backup CLI, and retain or improve the restore helper only if it covers a real gap not served by `openclaw backup` plus the generic migration guide.
