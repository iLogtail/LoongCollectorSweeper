## Summary

Keep open. Current main still reconstructs Feishu card action events with only `action.value` and `action.tag`, and the text fallback only serializes `action.value`; `form_value`, `input_value`, `name`, `option`, and `options` are not passed through. The linked PR #43953 tracks this exact remaining work but is still open/unmerged, while #46196 was closed unmerged.

## Best Possible Solution

Keep #43202 open as the canonical user-facing issue until the Feishu plugin actually preserves these callback fields on main. The best fix belongs in `extensions/feishu`: extend the parsed event/type to retain `form_value`, `input_value`, `name`, `option`, and `options`; build a structured JSON synthetic message only when those interactive fields are present; preserve existing `text`/`command` button behavior; and add handler plus monitor/parser coverage for real callback payloads.
