## Summary

Keep #43202 open. Current main at e640c0a95fcbd06a1311f18fc635518d011d45da still models Feishu card actions around `action.value` only and dispatches only the fallback text derived from that value. The requested `action.form_value`, `action.input_value`, `action.name`, `action.option`, and `action.options` pass-through is not present. The related PR #43953 is useful and directly tracks the same fix, but it is still open/unmerged in the provided GitHub context, and PR #46196 was closed unmerged, so the issue is not implemented or superseded enough to close.
