## Summary

Keep open. Current `main` still has the two core conditions described in the report: session-title extraction does not strip gateway-injected inbound metadata, and the gateway `/sessions` path derives titles from the raw session entry instead of the computed row `displayName`. The TUI also prefers `derivedTitle` over `displayName`, so a bad derived title still wins in `/sessions`.

## Evidence

- **Gateway title extraction still returns raw user text without stripping inbound metadata:** `extractTextFromContent` only applies `stripInlineDirectiveTagsForDisplay(...)` and returns the result directly. `extractFirstUserMessageFromTranscriptChunk` then uses that text as the first-user-message candidate for titles, with no call to the inbound-metadata stripper.
  - file: [src/gateway/session-utils.fs.ts:254](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/gateway/session-utils.fs.ts#L254)
  - command: `nl -ba src/gateway/session-utils.fs.ts | sed -n '254,305p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Gateway session rows compute a displayName, but derived titles still ignore it:** `buildGatewaySessionRow` computes `displayName` from `entry.displayName`, group display-name helpers, `entry.label`, or `origin.label`. Later in the same function, `deriveSessionTitle(entry, fields.firstUserMessage)` is called with the raw `entry`, not the computed `displayName`, so the derived-title path can still fall through to transcript text.
  - file: [src/gateway/session-utils.ts:1238](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/gateway/session-utils.ts#L1238)

- **TUI `/sessions` still prefers derivedTitle over displayName:** The selector label uses `session.derivedTitle ?? session.displayName`, so if the gateway returns a bad derived title, the correct `displayName` is not shown.
  - file: [src/tui/tui-command-handlers.ts:174](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/tui/tui-command-handlers.ts#L174)
