## Summary

Current main still matches the reported failure path for #39722. The transcript title extractor can return the gateway-injected `Sender (untrusted metadata)` block, `buildGatewaySessionRow` computes an outer `displayName` but does not pass it into `deriveSessionTitle`, and the TUI `/sessions` list prefers `derivedTitle` over `displayName`. The related fix PR #39790 is still open/unmerged in the provided GitHub context, so there is no conservative close reason yet.

## Best Possible Solution

Keep #39722 open until PR #39790 or an equivalent maintainer-approved fix lands. The fix should strip inbound metadata before session title and preview extraction, pass or otherwise prefer the computed gateway row displayName during title derivation, preserve the intended TUI fallback behavior, and add regression coverage for both metadata-envelope stripping and external displayName precedence.
