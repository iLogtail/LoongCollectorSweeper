## Summary

Keep #39722 open. Current main still has the reported `/sessions` title precedence and sanitization gap: the TUI requests derived titles and renders `derivedTitle` before `displayName`; gateway session rows compute an outer `displayName` but derive the title from only the inner `entry` plus transcript text; and transcript title extraction still strips directive tags only, not inbound metadata envelopes.

## Best Possible Solution

Keep #39722 open until #39790 or an equivalent maintainer-reviewed fix lands. The implementation should pass the computed outer session `displayName` into derived-title construction, strip inbound metadata envelopes before first-user-message and last-message-preview extraction, preserve subject/message/session-id fallbacks, and add regression coverage for both external displayName precedence and metadata stripping.
