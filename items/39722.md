## Summary

Keep open. Current main at 6a688e33f66f3965968343922e874cc0e9440998 still has the reported TUI /sessions title path: title extraction returns raw transcript text without stripping gateway inbound metadata, buildGatewaySessionRow derives the title from the raw session entry rather than the computed row displayName, and the TUI prefers derivedTitle over displayName.

## Best Possible Solution

Keep #39722 open until OpenClaw either lands PR #39790 or an equivalent fix: strip inbound metadata before using first user transcript text as a title candidate, pass the computed external displayName into title derivation or otherwise prefer it, and add regression coverage for both behaviors in the gateway session title tests. The TUI label precedence should also be checked so displayName can still win when derivedTitle is polluted.
