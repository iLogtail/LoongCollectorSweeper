## Summary

Keep open. Current `main` does not detect multiple Node.js environments or duplicate global OpenClaw installs in the installer or `openclaw doctor`; it only reports the active Node/npm path and some adjacent gateway/service drift. The linked PR #40850 implements the requested installer warning, but it is still open and its latest review history shows an unresolved coverage gap.

## Evidence

- **Installer only reports active Node/npm and an `nvm` hint:** On current `main`, `scripts/install.sh` prints the active Node/npm paths and, when the active version is too old, special-cases `nvm`; there is no function that scans for multiple Node environments or warns about duplicate global OpenClaw installs.
  - file: [scripts/install.sh:1290](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/scripts/install.sh#L1290)
  - command: `nl -ba scripts/install.sh | sed -n '1288,1390p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Doctor install checks are source-checkout-only:** `noteSourceInstallIssues()` only warns about pnpm workspace/source-install problems (`node_modules/.pnpm`, `package-lock.json`, missing `tsx`). It does not inspect global npm roots, multiple Node managers, or duplicate OpenClaw installs.
  - file: [src/commands/doctor-install.ts:5](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/commands/doctor-install.ts#L5)

  - file: [docs/gateway/doctor.md:566](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/gateway/doctor.md#L566)
