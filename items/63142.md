## Summary

The report is about a `mac-calendar` skill behavior on headless macOS, but current `main` does not ship that skill and project scope routes new or skill-specific integrations to ClawHub rather than OpenClaw core.

## Evidence

- **Project scope pushes new skills out of core:** The vision doc says optional capability should usually ship as plugins, and new skills should be published through ClawHub first rather than added to core by default.
