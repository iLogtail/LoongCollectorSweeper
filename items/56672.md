## Summary

Keep open. Current main partially supersedes the subagent portion because successful and failed subagent cleanup goes through `sessions.delete`, and `sessions.delete` now emits `session_end` from the persisted session entry. The cron isolated-agent portion is still not implemented on current main: cron runs return on success, abort, and catch paths without a `session_end` emission unless a later delete-after-run path deletes the session. The PR also has unresolved review concerns, so it should not be closed as implemented or treated as ready to merge unchanged.

## Best Possible Solution

Keep the item open for maintainer or author revision. The best path is to rely on the existing gateway `sessions.delete` emission for subagent session cleanup, and add a centralized cron isolated-session finalization hook that emits exactly once with the real persisted `sessionId` across success, error, and abort paths, with focused tests for those exits. The broader product requirement can continue to be tracked by #10142.
