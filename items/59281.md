## Summary

Keep #59281 open. Current main still routes Windows plugin loads away from native Jiti/import behavior, still treats TypeScript source plugin entries as transpiled Jiti loads, and the production plugin loader call sites remain synchronous Jiti calls. The related closed reports in the provided context cover narrower status/diagnostic or Linux/typebox loader issues, not this Windows source-loading latency lane.

## Best Possible Solution

Keep the issue open and address it in the plugin loader. A maintainer-quality fix should validate a Windows-safe native path for built plugin artifacts and/or add an async TS-aware native import path for source plugin entries, then replace the tests that currently assert Windows disables native Jiti with Windows regression/performance coverage for loadOpenClawPlugins, CLI/status, gateway bootstrap, and capability runtime call sites.
