## Summary

Keep #59281 open. Current main has a partial Windows improvement for built JavaScript plugin artifacts, matching the issue comment's direction, but the requested TypeScript source-loading problem is still intentionally routed through the synchronous Jiti transform path. The issue is recent, author is not protected, and labels are not protected, but the core requested capability remains unresolved rather than obsolete or duplicated.

## Evidence

- **TypeScript source entries still avoid native Jiti:** `shouldPreferNativeJiti` returns true only for `.js`, `.mjs`, `.cjs`, and `.json`; all other extensions, including `.ts`, fall through to false. That preserves the Jiti transform path for the TypeScript source entries named in the issue.
