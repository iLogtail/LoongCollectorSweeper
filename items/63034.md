## Summary

Keep open. Current `main` at `cf858258c71701ef62dddf88aa777873a4962046` still routes OpenRouter Anthropic requests through `applyAnthropicEphemeralCacheControlMarkers`, which only marks `system`/`developer` messages and strips assistant thinking markers. The direct Anthropic path still applies cache control to both the system prompt and the trailing user message, so the reported OpenRouter gap remains present. The issue timeline also points to an open follow-up PR (#63062), which is consistent with this not being resolved on `main` yet.
