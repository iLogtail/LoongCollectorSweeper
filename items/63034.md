## Summary

Keep open. Current main still routes OpenRouter Anthropic payloads through the helper that only marks system/developer blocks and strips assistant thinking markers; it does not reuse the direct Anthropic path’s trailing-user-message cache marker logic. The provided GitHub context also shows an open, unmerged same-author PR #63062 with closing syntax for this issue, so the right maintainer path is to review/land or reject that PR rather than close the issue now.

## Best Possible Solution

Review PR #63062 against current main. If maintainers accept the behavior, land a focused change that reuses the existing trailing-user-message cache-control helper on the OpenRouter Anthropic path and updates regression coverage so user string/array content is marked. If maintainers reject it, document why OpenRouter should remain system/developer-only and close or retitle accordingly.
