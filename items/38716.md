## Summary

Keep #38716 open. Current main still lacks Slack RTS support via assistant action tokens: Slack docs mention `assistant:write` and optional search scopes, and runtime has assistant thread status updates, but there is no `action_token` capture, no `assistant_thread_*` event handling, no `assistant.search.context` call, and no Slack `search` message action. The related Slack search PR context does not supersede this issue because it targeted `search.messages` with a user token, not the requested assistant action-token RTS flow.

## Best Possible Solution

Keep the issue open and implement the feature in the Slack plugin if maintainers want bundled RTS support: subscribe to the relevant Slack assistant thread events, capture and scope `action_token` to the assistant thread context, expose a Slack search message action or dedicated tool, call `assistant.search.context`, and add focused tests/docs. If maintainers decide RTS should live outside the bundled Slack plugin, move it to an explicit ClawHub/plugin API tracker instead of closing this as already handled.
