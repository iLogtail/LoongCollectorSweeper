## Summary

本 Issue 报告 flusher_kafka_v2 依赖的 IBM/sarama v1.42.2 存在上游已知的 broker throttle timer 竞态缺陷，可能导致生产者阻塞或 panic。上游已在 v1.43.1 合并修复。当前 main 分支 go.mod 仍停留在 v1.42.2，属于有效、明确且可执行的依赖升级请求，需维护者评估 API 兼容性后推进升级。

## Best Possible Solution

在 go.mod 与 test/go.mod 中将 github.com/IBM/sarama 升级至 v1.43.1 或更高兼容版本，执行 go mod tidy 后运行 flusher_kafka_v2 相关单元与集成测试验证 API 兼容性与竞态修复效果，随后提交依赖升级 PR 并关联本 Issue 供维护者审查。

## Evidence

  - file: [go.mod](https://github.com/alibaba/loongcollector/blob/d121269844e56531cc9159f7c4874d1a792c6655/go.mod)
