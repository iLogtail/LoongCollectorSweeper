## Summary

本 Issue 报告了 flusher_kafka_v2 依赖的 IBM/sarama v1.42.2 存在上游已知的 broker throttle timer 竞态缺陷，可能导致生产者阻塞或 panic。经核对当前 main 分支（448cb79b7eed）的 go.mod，依赖版本仍停留在 v1.42.2，尚未升级至包含官方修复的 v1.43.1。该问题目标明确、复现路径清晰且属于核心依赖升级范畴，需保持开启以跟踪后续升级与验证工作。

## Best Possible Solution

建议维护者或贡献者提交 PR，将根目录及 test/ 目录下的 go.mod 中 github.com/IBM/sarama 升级至 v1.43.1 或更高兼容版本。升级后执行 go mod tidy，并重点运行 flusher_kafka_v2 相关的单元测试与集成测试，验证 API 兼容性及竞态修复效果。测试通过后合并即可关闭本 Issue。

## Evidence

  - file: [go.mod](https://github.com/alibaba/loongcollector/blob/448cb79b7eedca73a4f5d7742403073d2fe5b1da/go.mod)
