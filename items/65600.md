## Summary

Keep #65600 open. Current main still contains the reported preflight compaction early return: when a session has a positive persisted `totalTokens` and `totalTokensFresh` is not false, `runPreflightCompactionIfNeeded()` returns before estimating or checking the compaction threshold. The latest release tag `v2026.4.24` has the same code, and the related fix PRs in the provided context are not landed on main.

## Best Possible Solution

Keep the issue open until a fix lands on main that restructures `runPreflightCompactionIfNeeded()` so fresh persisted totals are used for threshold evaluation instead of causing an early return. Add focused regression coverage for `totalTokensFresh: true` with cached totals above and below the preflight threshold, then close this issue from the landed canonical PR.
