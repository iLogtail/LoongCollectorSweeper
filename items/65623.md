## Summary

Current main still matches the reported bug: `/config show` clones `snapshot.parsed` and serializes either the selected path value or the full parsed config directly into chat reply text without applying `redactConfigObject` or `redactConfigSnapshot`. The linked fix PR #65637 remains open in the provided GitHub context, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Land PR #65637 or an equivalent maintainer-reviewed patch that loads config schema UI hints and applies `redactConfigObject` or `redactConfigSnapshot` before both full-config and path-specific `/config show` chat replies, with regression tests for sensitive fields such as `gateway.auth.token` and provider/channel tokens.

No redaction is applied on either reply path.
