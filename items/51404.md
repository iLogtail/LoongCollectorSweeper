## Summary

Keep #51404 open. Current main has a related reset-state fix that clears auto-sourced fallback pins on /new and /reset, but I did not find a source-backed fix for the reported startup greeting wording. The bare reset prompt still explicitly tells the model to mention default_model when the runtime model differs, and there is no regression test asserting the greeting names the actual runtime model or switches to the default before greeting. Ryce's comment usefully frames this as session-state drift and mentions KeepMyClaw snapshotting as an external mitigation, but it is not evidence that the core greeting bug is fixed.
