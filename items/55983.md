## Summary

Keep open. Current `main` at `7130e56f872aea96577ac9fcbda0c7555a9ecc69` still rejects cross-agent spawned sessions before checking spawn lineage, so the bug described by this PR is not implemented on `main`. The repo docs still describe `tree` visibility as including spawned sessions, and `session_status` has a separate cross-agent gate, so this PR also does not fully cover every behavior it claims.
