## Summary

Keep open. Current main has some Google Chat thread-send plumbing, but it does not fully implement the issue’s requested preservation of the inbound Google Chat thread in session/delivery context. The linked PR #68967 is relevant and appears to track the remaining work, but it is still open/unmerged, so the issue should not be closed as implemented or superseded yet.

## Evidence

- **Inbound Google Chat context stores thread only as ReplyToId:** The monitor copies message.thread?.name into ReplyToId/ReplyToIdFull, but does not populate MessageThreadId in the finalized inbound context. That leaves the issue’s session/delivery thread preservation concern unresolved on current main.
