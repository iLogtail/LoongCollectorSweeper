## Summary

Keep open. Current `main` has partial Google Chat thread support, but the code path still does not show a complete automatic fix for DM thread stickiness. The inbound monitor records the Google Chat thread in `ReplyToId`, not `MessageThreadId`; session persistence derives `deliveryContext.threadId` from `MessageThreadId`; and outbound Google Chat replies only reuse a thread when `payload.replyToId` is already present. The Google Chat account-scoped threading adapter also falls back to `replyToMode: "off"` when unset, so there is not strong evidence that ordinary DM replies will stay in the existing Chat thread by default.
