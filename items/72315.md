## Summary

Keep #72315 open. Current main still has the Windows npm child-process popup risk identified in the prior review: Telegram and Anthropic are staged-runtime-dependency plugins, and the runtime/postinstall npm staging spawns still do not pass `windowsHide: true`. The issue is new, non-maintainer-authored, has no protected labels, and the reporter followed up asking for a fix.

## Best Possible Solution

Keep #72315 open and fix it in core/plugin runtime staging. Add `windowsHide: true` to the Windows npm child-process options used by `installBundledRuntimeDeps`, `scripts/postinstall-bundled-plugins.mjs`, and `scripts/stage-bundled-plugin-runtime-deps.mjs` or propagate a typed runner spawn option consistently. Add regression tests asserting the option on Windows for the runtime installer and postinstall/stage paths, then verify on Windows 11 with OpenClaw 2026.4.24-or-newer style packaged/npm install, Telegram enabled, and missing staged runtime deps so npm actually runs.
