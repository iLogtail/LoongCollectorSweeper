## Summary

Keep open. This is a fresh, detailed issue against the current release, and current `main` does not provide strong evidence that the reported 100–400s Discord ingest lag is already fixed. The repo shows related Discord reconnect hardening, but not a clear buffered-message replay or catch-up fix, and I did not have a reproducible live Discord environment here to disprove the report.

## Evidence

- **#68159 was a narrower startup-race fix:** The changelog entry for PR #68159 says it fixed `awaiting gateway readiness` during startup when Carbon gateway registration races with lifecycle reconnect, which matches the issue body's claim that this older startup hang was resolved but does not establish a fix for post-login ingest lag.

- Some current-main reconnect hardening may reduce severity, but the repository evidence is not strong enough to claim the reported lag is obsolete.
- One hypothesis in the issue body appears unproven by code inspection, so future triage should separate the symptom (ingest lag) from that specific listener-lifecycle theory.
