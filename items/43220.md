## Summary

Keep open. Current main has backup and plugin-bundle primitives, but it does not implement the requested safe/reproducible agent package or migrate export/import flow; the related migration PR #53520 is still open and unmerged.

## Best Possible Solution

Keep #43220 open as a product/design tracking issue. The best path is to either land a core migrate export/import design with selective components, secret stripping, dry-run/import validation, and docs, or split the shareable agent-content part into plugin bundle/ClawHub guidance while tracking any missing core API explicitly. #53520 is relevant implementation work but should not close this issue until it lands or a narrower canonical follow-up covers the remaining packaging scope.

## Risks / Open Questions

- Closing as implemented would conflate backup archives with a safer shareable agent package; backups can still include credentials, sessions, and other runtime state.
- Closing as duplicate of #53520 would be premature because that PR is open/unmerged and appears focused on cross-device migration rather than the full agent packaging and registry model requested here.
- Closing as clawhub-only would skip the core product/security question around first-class migration, secret stripping, import validation, and path remapping.
