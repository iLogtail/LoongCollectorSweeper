## Summary

Keep open. Current main still documents full state-directory copying for migration, has no `openclaw migrate export/import` or agent packaging command on the core CLI, and the related PR #53520 is open/unmerged rather than proof that the request is implemented on main.

## Evidence

- **Current migration docs still copy full state:** The migration guide tells users to archive and copy `~/.openclaw/`, preserving config, auth, sessions, channel state, and workspace files. It later warns that backups contain auth profiles, channel credentials, and provider state, which is the unsafe all-state copy model the issue asks to improve.
  - file: [docs/install/migrating.md:15](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/docs/install/migrating.md#L15)

  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Workspace docs separate shareable workspace from private state:** The workspace guide says config, auth profiles, credentials, sessions, and managed skills live under `~/.openclaw/` and should not be committed; sessions/config must be copied separately and kept out of version control. That helps users reason about state, but is not a standardized safe package/export format.
