## Summary

Keep #71205 open. Current main still hardens the active config/state directory to 0o700 with no OPENCLAW_SKIP_STATE_DIR_HARDEN or equivalent opt-out, while the PR is a focused core config fix for a concrete multi-UID sidecar deployment problem. The provided PR discussion also shows the bot observability concern was addressed in d500b555e.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow core config/io opt-out guarded by isTruthyEnvValue, with logAcceptedEnvOption observability and the proposed regression test preserving a 0o2770 setgid shared state directory when OPENCLAW_SKIP_STATE_DIR_HARDEN is truthy. This is core runtime/config behavior, not ClawHub plugin work.
