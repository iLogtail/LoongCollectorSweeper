## Summary

Keep #71205 open. Current main still has the unconditional state-dir hardening behavior the PR targets: config writes create the config directory with mode 0o700 and then call `tightenStateDirPermissionsIfNeeded`, which chmods the active state/config directory to 0o700 whenever any group/other permission bits are present. There is no `OPENCLAW_SKIP_STATE_DIR_HARDEN` implementation, import, test, changelog entry, or docs entry on main. The PR is recent, focused, includes a concrete multi-uid sidecar reproduction, and its only bot review concern was addressed in the PR discussion.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product/code path is to add a narrow opt-in `OPENCLAW_SKIP_STATE_DIR_HARDEN` guard in `src/config/io.ts` using `isTruthyEnvValue` and `logAcceptedEnvOption`, preserve the default 0o700 hardening path, add the setgid/group-writable regression test in `src/config/io.write-config.test.ts`, and include an operator-facing changelog/docs note if maintainers accept the env-var surface.
