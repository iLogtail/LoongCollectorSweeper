## Summary

Keep open. Current main still aborts auto provider selection when an adapter omits shouldContinueAutoSelection, so the PR addresses a real remaining behavior. The PR discussion is useful: Greptile agreed the change is aimed at third-party adapters because built-in adapters already provide explicit guidance, but noted the guard should use adapter.transport rather than adapter.id.
