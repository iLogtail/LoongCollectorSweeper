## Summary

Keep #12508 open. Current main still executes mutable in-process plugin hook chains without host-enforced checksum/signature handoff, per-handler tamper detection, detailed modification audit, or manifest/config per-hook allow-lists. The existing April 26, 2026 maintainer-side Codex review comment already kept this open, and current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still supports that conclusion.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely implementation belongs in the hook runner, registry/config policy, plugin manifest/docs contract, and focused tests covering sequential hook mutation, per-hook permissions, deterministic modification auditing, and host-enforced integrity checks. It should not be moved to ClawHub unless core first exposes an API that lets an external plugin enforce these guarantees across the full hook chain.
