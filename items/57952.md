## Summary

Keep open. Current `main` still does not expose or apply configurable concurrency for `CommandLane.Nested`, and `/hooks/agent` still has no `lane` input and dispatches hook runs through the cron-to-nested path. A recent `main` change does appear to reduce the `sessions_send` portion of the report by scoping nested lanes per target session, but that is only a partial match; the cron/hook bottleneck described in this issue is still supported by the current code.
