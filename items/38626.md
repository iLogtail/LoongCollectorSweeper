## Summary

Keep open. Current main has partial subagent/task observability and controls, but it still lacks the central requested durable per-subagent lifecycle timeline, persisted tool/warning/error transitions, artifact-pointer summaries, first-failure/recommended-action diagnostics, and idempotent audit-logged cancel/steer/resume supervision controls. Related issue #62478 is closed background-spawn work and is complementary rather than a canonical replacement.

## Best Possible Solution

Keep this open unless a maintainer deliberately narrows the scope. The product/code path is to add a durable per-run subagent/task event ledger for spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it in the task/subagent stores; expose it through /subagents info, openclaw tasks show, and runtime/gateway APIs; attach artifact pointers and terminal diagnostics; and make cancel, steer, and resume idempotent with auditable action records.
