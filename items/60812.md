## Summary

Keep #60812 open. Current main still has Telegram-specific user-visible fallback handling, but the requested Discord, Slack, Signal, and WhatsApp dispatch paths do not send a typed fallback when dispatch fails or no visible reply is queued. Related #60830 remains a different/open Telegram/model-fallback PR in the provided context, and #60813 only covers LINE.

## Best Possible Solution

Keep the issue open and implement a shared, tested channel fallback path for bundled channel dispatchers. The likely fix is a small shared helper that maps dispatch/lastError and empty/no-visible-output states to safe user-facing copy using the existing failover classification, then wire it into Discord, Slack, Signal, WhatsApp, and LINE with regression tests for dispatcher throw, delivery failure/skip, and no visible final reply.
