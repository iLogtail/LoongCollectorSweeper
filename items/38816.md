## Summary

Keep #38816 open. Current main supports explicit provider-prefixed image models such as `ollama/qwen2.5vl:7b`, and the docs now recommend that form, but the reported bare `agents.defaults.imageModel.primary` path still resolves through the image fallback default provider and can become `anthropic/<model>` instead of inferring a configured Ollama/LiteLLM provider.

## Best Possible Solution

Keep this issue open until maintainers either add imageModel-specific bare-provider inference from `agents.defaults.models` / `models.providers`, or explicitly decide that imageModel must be provider-prefixed and enforce that contract with validation or a clear runtime error. Explicit `provider/model` imageModel values should remain the documented workaround in the meantime.
