## Summary

Keep PR #69139 open. Current main still has the exact Gateway `agent` params schema gap the PR targets: `AgentParamsSchema` is closed with `additionalProperties: false` and still has no root-level `paperclip` field, while the `agent` handler validates that schema before dispatch and returns `invalid agent params` on validation failure. The related discussion is concrete and corroborated: #69137 and the live triage comment describe the Paperclip adapter adding `agentParams.paperclip` unconditionally, and the only shipped Paperclip-related relaxation on main is for `wake` params, not `agent` requests.

## Best Possible Solution

Keep the PR open for maintainer review. The remaining product decision is whether to accept the narrow `paperclip` field as proposed, replace it with a documented generic opaque metadata path for gateway `agent` requests, or coordinate an adapter-side change. In any case, current main still needs a fix plus regression coverage for Paperclip-originated `req agent` payloads if OpenClaw intends to support the documented Paperclip gateway flow.
