## Summary

Keep #71482 open. Current main has not implemented the proposed protection, and the reported root cause remains plausible: `loadSessionStore()` still swallows read/parse failures into `{}`, while `scrubDreamingNarrativeArtifacts()` treats an empty store as an empty reference set and then archives old dreaming-marker transcripts as orphans. The PR should not be auto-closed, but its current patch also has a real review concern: skipping all cleanup for any legitimate empty store would disable orphan transcript cleanup for agents with no active sessions.

## Best Possible Solution

Keep the PR open and ask for a narrower fix: make the scrubber distinguish a failed session-store load from a genuinely empty store, or change `loadSessionStore()`/a new read helper to expose load status for this cleanup path. Add regression coverage for both cases: failed/malformed store must not archive active dreaming transcripts, while a genuinely empty valid store should still allow aged orphan dreaming transcripts to be cleaned up.
