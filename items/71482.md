## Summary

Keep open. Current `main` still has the reported orphan-archival path in `extensions/memory-core/src/dreaming-narrative.ts`, and the PR's proposed guard is not on `main` and has a credible unresolved review concern about suppressing legitimate orphan cleanup for truly empty stores.

## Evidence

- **Current scrubber still archives any unreferenced dreaming transcript:** On `main`, `scrubDreamingNarrativeArtifacts()` loads the store, immediately builds `referencedSessionFiles`, then archives any `.jsonl` not in that set once older than the 5 minute threshold. There is no empty-store guard in the checked-out code.
