## Summary

Keep open. Current main has moved Feishu runtime conversation routing into the shared binding route helper, and it still honors bound sessions even when the bound session agent differs from the resolved route agent. The PR’s agent-mismatch cleanup is not safe as-is because current docs/tests support intentional cross-agent ACP and focused conversation bindings, and the PR already has an unresolved P1 review calling out that regression.

## Best Possible Solution

Do not merge or auto-close this PR as-is. If stale Feishu binding cleanup is still desired, rework it around an explicit stale/invalid binding condition rather than bound-agent-versus-route-agent mismatch, and add paired regressions proving both cleanup and preservation of intentional cross-agent `/focus` and ACP `--thread here` bindings.
