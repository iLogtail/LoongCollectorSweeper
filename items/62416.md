## Summary

Keep open. The report is broad but actionable and current main only has partial accessibility treatment in the Control UI settings surfaces. The Settings tab is a core Control UI surface, not a ClawHub/plugin item, and the related closed aria-live report covers chat transcript behavior rather than the settings panel.

## Best Possible Solution

Keep this open as a core Control UI accessibility issue. The best path is to audit Settings/Quick Settings/Advanced Config with keyboard and at least one screen reader, then add semantic grouping, associated labels/ids, aria-pressed or native radio semantics for segmented choices, clearer status/live regions where state changes, focus management for settings navigation/import flows, and focused DOM tests across ui/src/ui/views/config-quick.ts, ui/src/ui/views/config.ts, and ui/src/ui/views/config-form.node.ts. Treat #46504 as related prior accessibility work, not as a duplicate.
