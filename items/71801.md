## Summary

Keep #71801 open. Current main still forwards Codex app-server request_user_input display strings into chat without sanitizing or bounding header/question/option text, and the existing tests only cover routing and answer mapping. The PR is a focused, still-open hardening patch for a real Codex extension gap, not a duplicate or already implemented change. It should stay under review because the latest PR discussion still raises response-shape/security questions around unsafe question ids and hidden/omitted questions.

## Best Possible Solution

Keep the PR open. The best path is to finish the focused Codex extension hardening in this PR or an equivalent maintainer patch: sanitize and bound only chat display text, keep request ids and answer semantics compatible, add regression tests for control/OSC/bidi/oversized text, and resolve the response-building concerns by protecting untrusted question ids and ensuring only shown questions can be answered when prompt text is omitted.
