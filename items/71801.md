## Summary

Keep #71801 open. Current main still formats Codex request_user_input chat prompts from raw app-server header/question/option strings with no display sanitization, no per-field/full-prompt bounds, and no regression coverage for control/OSC/bidi/oversized prompt text. The PR targets a real current gap, but the posted review comments raise concrete correctness/security issues that should be addressed before merge.

## Best Possible Solution

Keep the PR open and finish it as the canonical fix: add display-only sanitization and bounded formatting in extensions/codex/src/app-server/user-input-bridge.ts, ensure only displayed questions can be answered, avoid prototype-sensitive answer maps for untrusted ids, and add focused tests for terminated and unterminated OSC/control/bidi/large prompt fields plus unchanged answer mapping.
