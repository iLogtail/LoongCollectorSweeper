## Summary

Keep open. Current main still couples `mode="session"` to `thread: true` for both native subagent and ACP `sessions_spawn`, and docs/tests intentionally describe that coupling. The available `mode="run"` path is not an equivalent ACP persistent-session replacement because ACP run mode maps to `oneshot` and the runtime closes it after the turn. Related closed issues do not provide a canonical open replacement or shipped decoupling fix.

## Best Possible Solution

Keep this as the canonical product/runtime request unless maintainers choose a narrower tracking issue. A real fix likely belongs in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`: allow `mode: "session"` without `thread: true`, keep ACP runtime mode `persistent`, return the `childSessionKey`/label for `sessions_send`, define parent/current-conversation delivery semantics for unbound sessions, and update the rejection tests plus docs under `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
