## Summary

Keep #50677 open. Current main partially mitigates the original failure by compacting the skills catalog before dropping entries, documenting the prompt budget, and exposing per-agent skill allowlists/budget overrides. But the remaining user problem is still real: once compact output still exceeds budget, OpenClaw keeps only the sorted prefix, reports only `included X of Y`, does not list omitted skills, does not prioritize user-created or pinned skills, and has no per-topic/context-required skill loading. The related PR #42637 is useful but only covers omitted-name visibility and is not a shipped or complete superseding fix for the broader issue.

## Best Possible Solution

Keep the issue open as the broader tracking item. A complete fix should extend the skills prompt selection contract to record omitted skill names, surface truncation to the operator through `/context` or a one-time session warning, and add a deterministic priority/pinning or required-skill mechanism before the compact-prefix truncation step. PR #42637 can be treated as a candidate implementation for the omitted-name visibility slice, but it should not close this issue unless the remaining pinning/priority or explicit-survivability path is also handled or split into a narrower canonical follow-up.
