## Summary

Keep open. Current main does not implement a configurable umask, fileMode, or dirMode for agent-created user-facing files. This is not a ClawHub close: the request targets built-in core write/edit/exec behavior and no existing plugin surface can change the file modes used by those built-in tools without core support.

## Evidence

- **Strict tools schema has no permission fields:** The runtime config schema defines `tools.exec` via `ToolExecBaseShape` and `tools.fs` via `ToolFsSchema`; both schemas are strict and include exec policy/path settings or `workspaceOnly`, but no `umask`, `filePermissions`, `fileMode`, or `dirMode` keys.

## Risks / Open Questions

- Implementing this will need careful separation between user-facing workspace outputs and sensitive internal files, since related issues intentionally tightened config/credential/state permissions.
- Host, sandbox, and exec-created files currently take different code paths, so a future fix should cover all requested surfaces with focused tests.
