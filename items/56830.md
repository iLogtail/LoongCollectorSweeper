## Summary

Keep open. Current main has docs and policy tests that clarify `allowInsecureAuth` is localhost-only, but the runtime Gateway/UI copy that PR #56830 targets is still the older ambiguous wording. The PR remains a plausible focused fix for #32473, with one actionable review gap around non-English locale copy rather than close-worthy redundancy.

## Best Possible Solution

Keep #56830 open as an actionable Gateway/UI copy fix for #32473. The best path is to rebase it onto current main, preserve the existing localhost-only policy, update the clarified runtime/UI strings and tests, and either update the non-English locale `stayHttp` strings or make a maintainer-approved localization call. The broader docs already point users to the right secure-context model, so the remaining product work belongs in the Gateway protocol/UI surfaces this PR touches.
