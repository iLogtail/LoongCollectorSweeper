## Summary

Keep open. Current main only partially handles env-reference syntax: `${VAR}` can be recognized as a config SecretRef, but bare `$VAR` is still not recognized, `.env` audit values are still reported as `PLAINTEXT_FOUND` whenever non-empty, and auth-profile inline refs still fall through to plaintext findings. The linked #61207 was closed unmerged and did not land the claimed guard; #54141 is open and covers part of the same problem but not the `.env` collector path.

## Best Possible Solution

Keep the issue open or explicitly consolidate it under a canonical maintainer-tracked fix. The best code path is a shared env-ref classifier used consistently by `parseEnvTemplateSecretRef()`, `collectConfigSecrets()`, `collectAuthStoreSecrets()`, and `collectEnvPlaintext()`, with regression tests for `$VAR` and `${VAR}` in `openclaw.json`, `auth-profiles.json`, and `.env` audit inputs. If #54141 is used as the base, it still needs coverage for the `.env` collector path before #61058 is fully resolved.
