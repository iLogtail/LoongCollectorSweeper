## Summary

The reported docs outage does not reproduce against current live infrastructure on 2026-04-24, and the checkout shows docs hosting is external to OpenClaw main. Current evidence supports closing this as an obsolete external incident rather than an actionable `main` bug.

## Evidence

- **Live HTTPS now succeeds:** `curl -Ik --max-time 20 https://docs.openclaw.ai` returned `HTTP/2 200` on 2026-04-24.
  - command: `curl -Ik --max-time 20 https://docs.openclaw.ai`
- **Live TLS certificate is valid:** `openssl s_client` showed a valid `docs.openclaw.ai` certificate issued by Let's Encrypt E8, valid from 2026-04-16 14:27:28 GMT through 2026-07-15 14:27:27 GMT.
  - command: `echo | openssl s_client -connect docs.openclaw.ai:443 -servername docs.openclaw.ai`
- **Reported DNS failure no longer reproduces via Google DNS:** `dig @8.8.8.8 docs.openclaw.ai ANY +dnssec` returned `NOERROR` with `docs.openclaw.ai. CNAME docs.mintlify.builders.`; `dig @8.8.8.8 openclaw.ai DS +dnssec` returned `NOERROR` with signed NSEC3 denial of DS, not the broken DNSSEC chain described in the issue.
  - command: `dig @8.8.8.8 docs.openclaw.ai ANY +dnssec && dig @8.8.8.8 openclaw.ai DS +dnssec`
- **Docs hosting is external to the main app repo:** The docs guide states that docs are hosted on Mintlify at `https://docs.openclaw.ai`.
  - file: [docs/AGENTS.md:7](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/AGENTS.md#L7)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Docs publishing is handled through a separate publish repo:** The docs workflow clones `openclaw/docs.git` and syncs docs there, indicating the live docs site is managed outside the main application codebase.
  - file: [.github/workflows/docs-sync-publish.yml:30](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/.github/workflows/docs-sync-publish.yml#L30)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Repo already treats some docs reachability problems as external/network issues:** The FAQ documents `docs.openclaw.ai` SSL-access problems as external blocking and points users to the GitHub docs mirror if the site is unreachable.
  - file: [docs/help/faq-first-run.md:245](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/help/faq-first-run.md#L245)

## Close Comment

Closing after a Codex review because this does not reproduce on current infrastructure. On 2026-04-24, `https://docs.openclaw.ai` returned HTTP 200, TLS presented a valid `docs.openclaw.ai` certificate, and `dig @8.8.8.8 docs.openclaw.ai ANY +dnssec` returned `NOERROR` with `docs.mintlify.builders` rather than the DNS/TLS failure reported here. Current `main` also shows the docs site is hosted externally on Mintlify (`docs/AGENTS.md:7`) and published through the separate `openclaw/docs` repo (`.github/workflows/docs-sync-publish.yml:30`). This looks like an external docs-hosting/DNS incident that has since cleared, not a current actionable bug in `openclaw/openclaw`. If it happens again, please open a new issue with fresh `dig` output from the affected resolver, validating-resolver logs, and the exact browser/TLS error.
