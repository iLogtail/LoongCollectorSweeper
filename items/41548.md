## Summary

Keep open. Current main implements shared gateway sessions for some direct-message context, but it still does not provide the requested unified inbox or automatic TUI/Telegram/iMessage cross-surface sync. Docs explicitly say history is not fully synced back to every client, channel delivery remains origin-channel based, and the TUI still filters events/history to the active session.

## Best Possible Solution

Keep this open as a product/core UX issue. The best path is an explicit unified inbox or opt-in mirror mode that preserves current origin-channel delivery safety, labels source channel/session clearly, and uses gateway session/event APIs to observe activity without silently faning replies across Telegram/iMessage unless maintainers intentionally design that behavior.
