## Summary

Keep open. Current main has partial shared-session/transcript behavior for TUI and Control UI, but it explicitly does not provide the requested full unified inbox or automatic cross-channel sync back to every Telegram/iMessage/client surface.

## Evidence

- **Docs explicitly say full client sync is not implemented:** The Messages docs state that multiple devices/channels can map to the same session, but history is not fully synced back to every client, and recommend one primary device for long conversations.
  - file: [docs/concepts/messages.md:73](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/docs/concepts/messages.md#L73)

  - file: [docs/channels/channel-routing.md:10](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/docs/channels/channel-routing.md#L10)
