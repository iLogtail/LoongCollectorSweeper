## Summary

Keep #38580 open. Current main still supports Telegram outbound delivery pinning, but it does not expose or implement a Telegram pinned-message read path backed by Bot API getChat.pinned_message. The related PR attempts (#38613 and #38718) were closed unmerged in the provided context, and their proposed action/command/config/doc surfaces are absent from current main.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements a pinned-message read path, likely a message action such as list-pins/read-pinned backed by Bot API getChat.pinned_message, with Telegram action gating, discovery/runtime plumbing, CLI/tool formatting, tests, and user-facing docs. If maintainers decide this belongs outside the bundled Telegram plugin, record that product decision explicitly rather than auto-closing from the current state.
