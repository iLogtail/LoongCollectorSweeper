## Summary

Keep open. Current main does not implement this PR's central backend contract: `sessions.list` still loads the combined session store, computes rows, and returns the full `SessionsListResult` without `lastHash`, response `hash`, or `{ unchanged: true }`. The PR discussion is meaningful rather than low-signal: it was split from #55959 after scope feedback, Greptile found no blockers, and the author addressed a Codex P1 review on the branch, but those commits are not present on current main.

## Best Possible Solution

Keep the PR open for maintainer review or a smaller rebased/superseding backend PR. The remaining product path is to land an additive `sessions.list` cache/`hash`/`lastHash` contract with clear invalidation inputs, regression coverage for config/store/transcript/subagent changes, and then handle Control UI and Apple adoption separately.
