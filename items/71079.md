## Summary

Keep open. Current `main` still hardcodes Feishu card header/footer content in both streaming and non-streaming card paths, and the Feishu config schema/docs do not expose any `channels.feishu.card.header` or `.footer` settings. The issue is recent, coherent, and the existing comment matches the current code rather than contradicting it.

## Evidence

- **Header/footer builders are hardcoded in the Feishu reply dispatcher:** `resolveCardHeader()` always derives a title from agent identity and defaults the template to `blue`; `resolveCardNote()` always starts with `Agent: <name>` and appends model/provider when present.
  - file: [extensions/feishu/src/reply-dispatcher.ts:70](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/feishu/src/reply-dispatcher.ts#L70)

  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Streaming cards always receive that header/note:** On streaming start, the dispatcher calls those helpers and passes both values into `streaming.start(...)`; on close it recomputes and passes the note again.
  - file: [extensions/feishu/src/reply-dispatcher.ts:314](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/feishu/src/reply-dispatcher.ts#L314)

  - file: [extensions/feishu/src/outbound.ts:200](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/feishu/src/outbound.ts#L200)
