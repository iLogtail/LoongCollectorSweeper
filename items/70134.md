## Summary

Keep open. Current main still uses `auth-profiles.json` mtime in the models-config ready-cache fingerprint, and I did not find an equivalent semantic auth-profile fingerprint implementation or regression test on main. The PR is a plausible focused fix for a real cache invalidation surface, but it still needs normal maintainer review because the PR discussion leaves an OAuth token-content fingerprint tradeoff unresolved.

## Best Possible Solution

Keep PR #70134 open for normal maintainer review. The likely best path is a dedicated semantic auth-profile fingerprint used by `buildModelsJsonFingerprint`, with regression coverage for no-op rewrites, meaningful key/ref/OAuth/profile-order changes, `models.json` mtime invalidation, SecretRef/env/file/exec behavior without leaking secrets, and parse/read error behavior. Maintainers should explicitly resolve whether OAuth access/refresh token content must be hashed for cache correctness.
