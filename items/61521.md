## Best Possible Solution

Keep the PR open for maintainer/product judgment on the channel setup API. The likely path is a revised, additive SDK/core setup contract that validates a fully assembled candidate config before writes, handles resolved account IDs consistently, avoids sparse-input false failures, and includes focused wizard plus channels-add tests. The VK channel itself can remain a ClawHub/community plugin, but this shared setup-validation seam is still a core API question.
