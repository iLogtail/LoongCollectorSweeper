## Summary

Keep #19680 open. Current main still has useful VISION.md scope guidance, lean-core/plugin direction, memory notes, security guidance, and PR review mechanics, but it has not been reworked into the requested human/AI decision framework for evaluating PRs against a product thesis. The issue discussion also includes a same-day Codex review comment from steipete explicitly keeping it open, and there is no superseding PR or canonical duplicate in the provided related context.

## Best Possible Solution

Keep the issue open until a maintainer either accepts and implements a docs-only VISION.md restructure or explicitly decides the current guidance is sufficient. A good implementation would add a concise product thesis, three or four PR gut-check questions, and rationale-grounded examples for security, memory, lean core, plugins, and won't-merge categories so humans and AI review tools can reason from principles.
