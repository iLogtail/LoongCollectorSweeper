## Summary

Keep open. Current `main` still does not expose `sessionModelOverride` or `sessionProviderOverride` on `PluginHookAgentContext`, the public hooks docs still say `before_model_resolve` only gets prompt plus attachment metadata, and the runtime still applies `before_model_resolve` overrides after building hook context from the already-selected provider/model. The reported SDK gap is therefore still present and the issue is coherent and actionable.

## Evidence

- **Plugin hook context still lacks session override fields:** `PluginHookAgentContext` currently exposes `runId`, tracing/session ids, workspace, `modelProviderId`, `modelId`, `messageProvider`, `trigger`, and `channelId`, but no `sessionModelOverride`, `sessionProviderOverride`, or override-source field.
  - file: [src/plugins/hook-types.ts:155](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/plugins/hook-types.ts#L155)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '150,175p'`

- **Hook context builder does not populate any session override metadata:** `buildAgentHookContext()` only forwards run/session/workspace/message metadata into the hook context object, so plugins still cannot read pinned session override state from the typed context.
  - file: [src/agents/harness/hook-context.ts:14](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/harness/hook-context.ts#L14)
