## Summary

Keep open. Current main already makes the linked Gemini and Sonnet claims mostly obsolete, and standard inbound metadata now strips NUL bytes, but the PR still contains a useful hardening change that main does not cover: arbitrary `UntrustedStructuredContext.payload` strings are JSON-serialized without NUL stripping. This should be reworked/narrowed rather than closed as fully implemented or obsolete.

## Best Possible Solution

Keep the PR open only for a maintainer or contributor rework: narrow it to stripping NUL bytes inside `sanitizeUntrustedJsonValue` and add a regression test for NUL bytes in `UntrustedStructuredContext.payload`. Drop the Gemini template and Sonnet 4.6 default-context changes from this PR unless maintainers explicitly want separate product follow-ups.
