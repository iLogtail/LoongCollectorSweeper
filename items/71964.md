## Summary

Keep PR #71964 open. Current main still builds gateway/node service PATHs with hard-coded Volta, asdf, Bun, fnm, nvm, and pnpm fallback directories regardless of whether those directories exist. The PR targets a real remaining daemon behavior, but it should still get maintainer review and the small test/changelog cleanup already noted in review comments.

## Best Possible Solution

Keep #71964 open or land a maintainer-equivalent fix: gate only hard-coded version-manager/package-manager fallback directories behind an injectable existence predicate, keep explicit env-configured roots and stable user-bin conventions unconditional, add deterministic Linux/macOS regression tests for missing and existing fallbacks, and clean up the changelog attribution.
