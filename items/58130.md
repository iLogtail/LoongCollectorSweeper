## Summary

Keep open. Current main already has a substantial browser Control UI, but this issue is asking for a CoPaw-style layout redesign and easier centralized multi-account orchestration. That redesign is not implemented on main, and web frontend ergonomics remain in core scope rather than ClawHub-only scope.

## Evidence

- **Core scope:** `VISION.md` keeps "Ergonomics across CLI and web frontend" in the project's next priorities, so Control UI UX/layout work is still core product scope. The ClawHub guidance in the same document is limited to new skills, not the core admin UI.
  - file: [VISION.md:25](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/VISION.md#L25)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Existing Control UI already exists:** Current docs describe a browser Control UI with chat, channel status/config, sessions, cron add/edit/run, skills management, nodes, exec approvals, and config editing. That means this issue is not obsolete as "add a web UI"; it is a request for a different layout/workflow on top of an existing UI surface.
  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/web/control-ui.md#L105)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Multi-account support is partial, not a CoPaw-style redesign:** Current UI has some multi-account affordances, such as optional account ID fields for cron delivery/failure alerts, but those are narrow form fields rather than the centralized orchestration layout the issue asks for.
  - file: [ui/src/ui/views/cron.ts:1102](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/views/cron.ts#L1102)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current layout is still OpenClaw-specific, not a copied CoPaw UI:** The agents view is a tabbed OpenClaw-specific interface, and a repo search found no `CoPaw`, `openclaw-control-center`, or `ClawManager` implementation references on current main.
  - file: [ui/src/ui/views/agents.ts:147](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/views/agents.ts#L147)
