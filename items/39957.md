## Summary

Keep PR #39957 open. Current main still suppresses only explicitly commentary-phased assistant output; it does not suppress the unphased `message_end` + `stopReason: "toolUse"` + tool-call-block case described by the PR and linked bug #24376.

## Best Possible Solution

Keep the PR open and either merge/rebase its narrow `message_end && stopReason === "toolUse" && has tool-call block` suppression with regression coverage, or implement equivalent suppression in the current display-phase model so unphased intermediate tool-use turns cannot leak into Telegram-visible block replies.
