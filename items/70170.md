## Summary

PR #70170 is a duplicate/superseded onboarding fix. Current main still filters setup web-search providers by `onboardingScopes: ["text-inference"]`, and the bundled DuckDuckGo provider base still does not expose that metadata. The same remaining onboarding work is already tracked by the earlier, narrower PR #65940, which is linked in the PRtags group and targets only the two metadata additions for the DuckDuckGo runtime and contract surfaces. PR #70170 also includes unrelated DuckDuckGo client transport/cooldown changes with review comments, so the best cleanup path is to keep the canonical onboarding fix in #65940 and split any DDG transport hardening into a separate scoped PR if maintainers want it.

## Best Possible Solution

Close #70170 as a duplicate/superseded PR, continue the onboarding metadata fix through #65940 or an equivalent narrow patch adapted to the current shared DuckDuckGo provider base, and handle the DuckDuckGo POST/header/cooldown work as a separate focused PR if maintainers want that behavior change.
