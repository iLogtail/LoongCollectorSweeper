## Summary

Keep open. Current `main` still does not extract tool-card text from array-shaped tool result content, so this PR is not obsolete, and the provided PR patch is not strong enough evidence for closure.

## Evidence

- **Current tool-card extraction still drops array content:** `extractToolText` only returns `item.text` or a string `item.content`; any array-shaped `item.content` falls through to `undefined`. That means tool cards still miss payloads like `content: [{ text: ... }]` on current `main`.

- **Current tests do not cover the reported regression:** `tool-cards` tests exercise string `text` and string `content` tool results, but there is no test for array-backed tool result content, so there is no proof the reported case was fixed on `main`.
