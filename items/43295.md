## Summary

Keep #43295 open. Current main still does not provide the requested forced recall-before-response runtime guard with enforce/block behavior. Existing Active Memory and memory-lancedb paths can inject recall context through plugin hooks, but they are scoped/advisory and do not cover the issue's multi-step orchestration and subagent reliability concern.

## Best Possible Solution

Keep the issue open for maintainer design. The likely product path is a narrowly scoped, opt-in runtime policy that can perform recall before selected model responses, inject recalled rule context, and optionally block when recall is unavailable, with clear coverage for embedded, CLI, subagent, automation, and active-memory/plugin interactions. If maintainers prefer plugin ownership, the missing enforce/block hook contract should be tracked explicitly before closing the core issue.
