## Summary

Keep #38430 open. Current main still does not implement the requested default of full absolute file paths in agent replies, and the related PR #38495 was closed unmerged. The code still intentionally compacts some prompt-visible paths to `~/...`, preserves tilde paths in at least one tool success message, and has no `OPENCLAW_EXPAND_PATHS` or equivalent output-format control.

## Best Possible Solution

Keep the issue open for maintainer product direction. A robust fix should decide the exact surface to change, then add either prompt guidance for final replies, a delivery-time formatter for user-facing path text, or a documented config/env switch, with regression coverage for final replies and tool-result summaries. It should explicitly preserve or intentionally revise existing skill prompt compaction and sandbox relative-path guidance.
