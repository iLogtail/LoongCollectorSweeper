## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration order, while the adjacent prompt supplement path is already sorted for stable prompt-cache behavior. The PR is a focused fix with matching regression-test intent, and the current code paths confirm ordering can affect memory_get fallback behavior and loader cache snapshots.

## Best Possible Solution

Keep this PR open and review or land the focused fix after any needed rebase: sort `listMemoryCorpusSupplements()` by `pluginId` in `src/plugins/memory-state.ts` and keep the two regression tests that lock reverse-registration and re-registration behavior.
