## Summary

Keep open. Current main has a related Feishu fix for synthetic card-action/bot-menu reply targets, but the reported generic `message` tool `replyTo` path still has a concrete gap: Feishu's plugin-owned `send` action intercepts message sends and does not pass `replyTo`/`reply_to` through, while only `thread-reply` maps a message id to `replyToMessageId`.

## Best Possible Solution

Keep the issue open and fix the Feishu message-tool `send` path so explicit `replyTo`/`reply_to` is honored for text, card/presentation, and media sends, likely by mapping `resolveFeishuMessageId(ctx.params)` for `send` as well as `thread-reply` while preserving `replyInThread` only for explicit thread replies. Add regression tests in `extensions/feishu/src/channel.test.ts` for `action: "send"` with `replyTo` on text and presentation/card payloads, plus a check that `thread-reply` still uses `reply_in_thread`.
