## Summary

Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still does not expose or apply hook-owned user prompt replacement. The public hook result types, prompt-mutation allowlist, hook merger, embedded runner, CLI runner, agent harness helper, docs, and tests all cover system/context mutation only. The direct implementation PR #8178 was closed unmerged, and related systemPrompt items do not satisfy the requested promptOverride capability.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for before_prompt_build and legacy before_agent_start, or a maintainer explicitly rejects that API. A complete fix should update the public result types, prompt-mutation allowlist, hook merge precedence, resolvePromptBuildHookResult, embedded runner, CLI runner, shared agent harness path, allowPromptInjection filtering, public docs, and focused regression tests for merge order plus submitted prompt text.
