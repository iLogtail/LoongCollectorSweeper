## Summary

Keep #62303 open. Current main still has the original `autoTopicLabel` implementation, but I did not find a later fix for the reported regression: Telegram DM topic auto-labeling is still gated on `!entry?.systemSent`, and the label generation/`editForumTopic` path still only runs when that first-turn gate is true.

## Best Possible Solution

Keep #62303 open as an actionable Telegram regression. A good fix should decouple DM topic auto-labeling from the generic `systemSent` first-turn bit, or add explicit per-topic label-attempt state/provisional-title detection, then add regression coverage for a DM topic whose session entry already has `systemSent: true` before the first meaningful user message while still honoring `autoTopicLabel: false` overrides.
