## Summary

Keep open. Current main has substantial Google provider infrastructure, but it does not contain this PR's `google-genai` provider/auth mode, and the linked Vertex ADC problem is still plausible on the current code path. The PR should not be cleanup-closed, though it likely needs rework into the bundled Google plugin boundary and must address remaining transport/auth review gaps before merge.

## Best Possible Solution

Keep the PR open, but ask for maintainer-guided rework: either fix Google Vertex ADC in the existing bundled Google provider/runtime-auth path, or add a plugin-owned `google-genai` style transport that preserves current Google transport behavior for system prompts, tools, generation controls, aborts, usage, finish reasons, docs, config schema, and tests. The existing open Vertex ADC/proxy reports should remain the canonical bug context unless a narrower maintainer issue is opened.
