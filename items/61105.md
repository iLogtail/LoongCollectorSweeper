## Summary

Keep #61105 open. Current main still does not apply `tools.exec.pathPrepend` to `process.env.PATH` during config load. Skill eligibility/status checks still read `process.env.PATH` through `hasBinary()`, while `pathPrepend` is only merged into per-exec command environments. The PR addresses a real remaining gap, though it likely needs maintainer review for reload semantics and tests.

## Best Possible Solution

Keep the PR open. The best path is a small production-quality fix at the config/runtime boundary: apply normalized `tools.exec.pathPrepend` to the process env using the existing path-prepend helper or equivalent, make reload behavior idempotent and able to remove/update previously managed entries, add focused `src/config/io` coverage plus a skill-status/`hasBinary` regression if feasible, and update docs if `pathPrepend` is intentionally broadened beyond exec runs.
