## Summary

Keep PR #67509 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still has no equivalent CLI root guard, while the linked #67478 report describes a concrete, still-plausible root-run state/systemd corruption path. The PR is a focused active fix that addresses both modern and legacy CLI entrypoints and has already incorporated review feedback for static imports, container forwarding, legacy startup side effects, and effective UID handling.

## Best Possible Solution

Keep #67509 open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks real and effective UID, preserves the intended container-forwarded child CLI path, provides a clear explicit override, and updates DigitalOcean/install documentation if the root-run setup flow changes.
