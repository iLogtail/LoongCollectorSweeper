## Summary

Keep open. Current main still has no CLI root guard, so PR #67509 is not implemented or obsolete. The linked bug #67478 remains a concrete product risk: root execution can still enter CLI startup and state/config paths on both modern and legacy entrypoints.

## Best Possible Solution

Keep #67509 open. The best path is to review and land a root guard that runs early in both `src/entry.ts` and the legacy `src/index.ts` path, checks real and effective UID, keeps an explicit `OPENCLAW_ALLOW_ROOT=1` override, preserves the container-forwarded child flow using the existing hint+bypass contract, and includes regression tests for the modern entrypoint, legacy entrypoint, bypass cases, and setuid-root behavior. That implementation should then close #67478.
