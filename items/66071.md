## Summary

Keep PR #66071 open. Current main has OpenResponses phase metadata support, but the `/v1/responses` HTTP SSE path still emits every assistant delta as `response.output_text.delta` without checking whether the event is commentary. The PR addresses a real remaining gateway leak, though the supplied review comments show it should not be merged as-is.

## Best Possible Solution

Keep the PR open, but require a narrow revision or replacement patch in `src/gateway/openresponses-http.ts`: buffer phase-tagged assistant text, emit only `final_answer` visible deltas, keep unphased assistant deltas streaming for existing producers, exclude commentary from final visible text, preserve completed item phase metadata, and retain current abort-on-disconnect plus untrusted file-content wrapping behavior. Add regression tests in `src/gateway/openresponses-http.test.ts` for commentary suppression, explicit final-answer streaming, mixed commentary/final output, and normal unphased streaming.
