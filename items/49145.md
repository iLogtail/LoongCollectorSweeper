## Summary

Keep #49145 open. Current main has only partial Signal quote-context support: it can expose quoted body/sender in inbound context, but the PR's central remaining behaviors are still missing on main, including Signal ReplyToId propagation, reaction-triggered heartbeat wake, outbound signal-cli quote parameters, and first-chunk-only quoted sends.

## Best Possible Solution

Keep #49145 open for maintainer review or continue the implementation on a maintainer branch. The best product path is to land a focused Signal-channel change that wires quote id/author metadata through inbound context, wakes the targeted session after reaction system events, adds signal-cli quote-timestamp/quote-author support to outbound sends, preserves first-chunk-only quoting across text/media/payload paths, and keeps the existing lazy send-runtime boundary intact with regression tests for direct, group, chunked, media, reaction, and visibility-policy cases.
