## Summary

Keep #38966 open. Current main still emits `sessions.changed` on `sessions.reset`, but TUI clients neither subscribe to session-change events nor dispatch/handle `sessions.changed`, so an external Telegram/other-client `/new` or `/reset` still will not refresh the TUI. The directly linked fix PR #40472 remains open in the provided GitHub context, not shipped on main.

## Best Possible Solution

Keep the issue open until PR #40472 or an equivalent fix lands. The implementation should make the TUI subscribe to session change events, handle `sessions.changed` reset/new events for the current session, reload history and session metadata, clear stale token/run/stream state, ignore unrelated sessions, and add focused TUI handler/client coverage.
