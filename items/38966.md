## Summary

Keep #38966 open. Current main has the gateway-side `sessions.changed` reset signal, but `openclaw tui` still does not subscribe to session change events and its event dispatcher/handler factory still ignore `sessions.changed`, so an external `/new` or `/reset` will not refresh TUI history/token/run state. The provided related context shows PR #40472 is the direct open, unmerged fix attempt.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should have TUI subscribe to session change events, dispatch and handle `sessions.changed` `new`/`reset` events for the current session only, reload history/session metadata, clear stale active run/local run/stream state, ignore unrelated sessions, and add focused gateway/TUI regression coverage.

## Evidence
