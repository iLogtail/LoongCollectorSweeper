## Summary

Keep #55942 open. Current main has generic OpenAI-compatible malformed-tool-call recovery that may mitigate part of the NVIDIA Kimi symptom, and the closely related #44549 was already closed as implemented. But the NVIDIA Kimi route itself still ships as a plain openai-completions catalog entry with no NVIDIA/Kimi-specific compat flags, no string-content opt-in, no schema-compaction implementation, and no NVIDIA/Kimi tool-execution regression test proving this report is fixed.

## Best Possible Solution

Keep the issue open until maintainers either add a focused NVIDIA/Kimi compatibility fix and regression coverage, or produce live/current proof that the existing generic OpenAI-compatible repairs fully restore NVIDIA NIM Kimi tool execution. The likely implementation path is in the NVIDIA provider/model catalog or shared OpenAI-compatible provider compat layer: opt into required string-content normalization if NVIDIA still rejects content arrays, add any needed Kimi schema/tool-call compatibility, and cover nvidia/moonshotai/kimi-k2.5 with a targeted payload/tool execution test.
