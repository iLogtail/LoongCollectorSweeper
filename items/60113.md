## Summary

Keep open. Current main still has the core unsafe transport sanitizer path identified by the issue discussion, and the requested embedded-agent stack logging is not implemented: the lifecycle log records sanitized message/raw-error metadata, while the shared transport failure path stores only `error.message`. The related PR #60157 is closed unmerged in the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #60113 open. The narrow maintainer path is to harden transport serialization by coercing nullish/non-string payload text before `.replace()`, guard malformed replay blocks before `.trim()`/provider payload conversion, preserve a redacted `errorStack` or equivalent diagnostic field for embedded run-end failures, and add regression tests for undefined text/thinking blocks plus stack visibility.
