## Summary

Keep open. Current `main` still creates a Google Chat typing message up front, but only clears or replaces it from the delivery path. Exact silent `NO_REPLY` final payloads can be skipped before any delivery callback runs, so the typing message can persist indefinitely in the reported reaction-only flow. The issue timeline also already points to two follow-up PRs that add this missing cleanup, which is further evidence the fix is not on `main` yet.

## Evidence

- **Google Chat monitor starts a typing message before reply dispatch:** The Google Chat monitor defaults `typingIndicator` to `"message"`, sends `_... is typing..._`, and stores the returned `typingMessageName` before dispatching the agent reply pipeline.
  - file: [extensions/googlechat/src/monitor.ts:251](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/googlechat/src/monitor.ts#L251)
  - command: `nl -ba extensions/googlechat/src/monitor.ts | sed -n '245,315p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Current main only uses `typingMessageName` inside the delivery path:** `typingMessageName` is passed into `deliverGoogleChatReply`, then nulled after a successful delivery callback. The surrounding `replyOptions` only pass `onModelSelected`; there is no post-dispatch cleanup if no payload is delivered.
  - file: [extensions/googlechat/src/monitor.ts:287](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/googlechat/src/monitor.ts#L287)

- **Silent exact `NO_REPLY` finals can be skipped before delivery:** The reply dispatcher resolves silent final payload handling before enqueueing delivery, and when normalization returns null it logs that the exact `NO_REPLY` final payload was skipped before delivery. That means channel `deliver(...)` hooks do not run for this case.
  - file: [src/auto-reply/reply/reply-dispatcher.ts:127](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/auto-reply/reply/reply-dispatcher.ts#L127)

- **Dispatch completion does not help Google Chat because no cleanup callback is wired:** The buffered dispatcher always calls `markRunComplete()` and `markDispatchIdle()` in `finally`, but Google Chat does not provide an `onTypingCleanup`/typing callback hook in its current monitor path, so those generic lifecycle calls never delete the typing message.
  - file: [src/auto-reply/dispatch.ts:159](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/auto-reply/dispatch.ts#L159)
