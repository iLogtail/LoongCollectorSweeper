## Summary

Keep #65405 open. Current main still does not implement the requested dashboard `/model` filtering to only `models.providers`; `models.list` has no filter parameter, the Control UI calls it with `{}`, and the Gateway only narrows the result when `agents.defaults.models` is present as an allowlist.

## Best Possible Solution

Keep the issue open. A conservative fix would add an additive `models.list` parameter such as `configuredOnly` or `source`, wire the Control UI `/model` surface to default or toggle to `models.providers.*.models[]`, preserve the existing `agents.defaults.models` allowlist semantics for authorization, and add regression coverage for a MiniMax/Z.AI-style `models.providers`-only config.
