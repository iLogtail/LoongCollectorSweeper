## Summary

Keep #39780 open. Current main still has the reported Control UI gap: the insecure HTTP Overview recommendation remains manual text plus docs links, while before/after diff and base-hash guarded config write machinery exists only in Config/Quick Settings flows and is not wired into Overview recommendation callouts.

## Best Possible Solution

Keep #39780 open until Control UI recommendation callouts can stage narrow base-hash guarded config changes, show a before/after diff, offer accept/dismiss controls, apply through the existing config write path, and confirm the saved value. The work likely belongs in a shared recommendation/apply component that reuses Config form diff/apply primitives, with focused tests for the insecure HTTP Overview path and public Control UI docs updated.
