## Summary

Current main still has the exact missing behavior from #39780: the Overview insecure-HTTP recommendation renders text-only guidance telling users to set `gateway.controlUi.allowInsecureAuth: true`, with no apply button, before/after diff, accept/dismiss flow, or confirmation. The Config UI has diff/save/apply primitives, and Quick Config can stage/apply preset changes, but those mechanisms are not wired into this Overview hardening callout.

## Best Possible Solution

Keep #39780 open until Control UI hardening/recommendation callouts can stage a small config patch, show a before/after diff, let the user accept or dismiss, call the existing config write/apply API with base-hash protection, and confirm the resulting value. The likely implementation belongs in the Overview recommendation path, reusing the existing Config controller/diff concepts or a small shared config-remediation component rather than asking users to edit JSON manually.
