## Summary

Keep PR #38658 open. Current main still uses pending-only Slack room history: mention-gated skipped messages are recorded, but processed channel messages are cleared after dispatch, including when `requireMention=false`. The PR is not merged, not implemented on main, and not superseded by the related title-search items in the provided context.

## Best Possible Solution

Keep the PR open for maintainer review. If always-on Slack channel context is desired, implement it in `extensions/slack/src/monitor/message-handler` by carrying the resolved `requireMention` policy into dispatch, recording processed room messages when `requireMention=false` in both reply and no-reply paths, updating Slack history tests, and updating public docs to describe the bounded retained history behavior.
