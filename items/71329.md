## Summary

Keep #71329 open. Current `main` still requires Slack status reactions to pass the ack-reaction gate, so `messages.statusReactions.enabled: true` does not independently activate the lifecycle reaction controller in cases like DMs or disabled ack scopes. The PR targets the right Slack activation seam and is not superseded by the related items provided.

## Best Possible Solution

Keep the PR open and ask for the small test assertion requested in review, then consider merging the activation change. The best product/code path is to honor `messages.statusReactions.enabled === true` as an explicit opt-in independent of `ackReaction`, while preserving the existing unset/default behavior where Slack status reactions follow active ack reactions.
