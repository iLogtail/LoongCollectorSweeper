## Summary

Keep #71329 open. Current main still gates Slack lifecycle status reactions on `shouldAckReaction()`, so `messages.statusReactions.enabled: true` is not enough when the ack-reaction gate is false, such as the default `group-mentions` scope in a DM or an explicitly disabled ack reaction. The PR is a focused fix for behavior that remains missing on main, with only a small test assertion gap noted in review.

## Best Possible Solution

Keep the PR open and ask for the small test fix from the review comment, then merge or adapt the prepare.ts change so `messages.statusReactions.enabled: true` independently activates Slack lifecycle reactions while preserving the existing ack-reaction fallback and explicit-disable behavior.
