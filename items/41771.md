## Summary

Keep open. Current `main` does not expose configurable image-compression settings for the `image` tool. `agents.defaults.imageModel` still only supports model routing, the tool surface only exposes `maxBytesMb`/`maxImages`, and image preprocessing still runs through a hardcoded resize/quality grid.

## Evidence

- **Agent defaults lack image compression fields:** `AgentDefaultsConfig` defines `imageModel` only as `AgentModelConfig` and adds PDF-specific knobs like `pdfMaxBytesMb`/`pdfMaxPages`, but there are no image-tool compression/resolution settings alongside it.
  - file: [src/config/types.agent-defaults.ts:176](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/config/types.agent-defaults.ts#L176)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '175,205p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Docs describe `imageModel` as routing only:** The agent config docs say `imageModel` accepts only a string or `{ primary, fallbacks }` and describe it as the vision-model config/fallback path. No compression, quality, or max-dimension options are documented there.
  - file: [docs/gateway/config-agents.md:339](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/docs/gateway/config-agents.md#L339)
  - command: `nl -ba docs/gateway/config-agents.md | sed -n '338,366p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Image tool API has no compression parameters:** The `image` tool schema exposes `prompt`, `image`, `images`, `model`, `maxBytesMb`, and `maxImages`. During execution it forwards images through `loadWebMedia(...)` using only the byte cap/local-root options, with no caller or config hook for resize/quality control.
  - file: [src/agents/tools/image-tool.ts:320](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/tools/image-tool.ts#L320)
  - command: `nl -ba src/agents/tools/image-tool.ts | sed -n '320,500p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Image optimization remains hardcoded:** `loadWebMedia()` always enables image optimization, and `optimizeImageToJpeg()` still uses fixed resize sides `[2048, 1536, 1280, 1024, 800]` and qualities `[80, 70, 60, 50, 40]`. That is the exact area the issue asks to make configurable, and it is not configurable on current `main`.
  - file: [src/media/web-media.ts:596](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/media/web-media.ts#L596)
