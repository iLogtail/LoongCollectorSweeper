## Summary

Current `main` still appears to match the reported bug. The control UI identity RPC still prefers global `ui.assistant.name` over per-agent identity, and the Agent Context panel still renders emoji-only avatars as `—` instead of the configured emoji.

## Evidence

- **Per-agent identity is still overridden by global UI assistant name:** `resolveAssistantIdentity()` resolves `name` from `cfg.ui.assistant.name` before `agents.list[].identity.name` or `IDENTITY.md`, so any global UI assistant name will win for subagents too.
  - file: [src/gateway/assistant-identity.ts:92](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/assistant-identity.ts#L92)
  - command: `nl -ba src/gateway/assistant-identity.ts | sed -n '81,117p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Control UI agent identity endpoint uses that resolver directly:** `agent.identity.get` returns the result of `resolveAssistantIdentity({ cfg, agentId })`, so the Control UI receives the globally overridden name for subagents.
  - file: [src/gateway/server-methods/agent.ts:1024](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/agent.ts#L1024)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '1023,1031p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Agent Context panel still ignores emoji avatars:** `buildAgentContext()` sets `identityAvatar` to `"custom"` only when `resolveAgentAvatarUrl(...)` returns a URL; otherwise it shows `"—"`. It never uses the existing `resolveAgentEmoji()` helper, so configured emoji avatars still render as `—`.
  - file: [ui/src/ui/views/agents-utils.ts:371](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/views/agents-utils.ts#L371)
  - command: `nl -ba ui/src/ui/views/agents-utils.ts | sed -n '365,371p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Emoji-aware helper exists but is not used for Agent Context:** `resolveAgentEmoji()` already prefers runtime identity emoji and avatar-as-emoji values, which reinforces that the Agent Context card is simply not using the right display path.
  - file: [ui/src/ui/views/agents-utils.ts:272](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/views/agents-utils.ts#L272)
  - command: `nl -ba ui/src/ui/views/agents-utils.ts | sed -n '272,293p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)

## Risks / Open Questions

- The issue likely still reproduces whenever `ui.assistant.name` is set globally, because subagent identity fetches inherit that name.
- Even if the name path is corrected elsewhere, the Control UI's Agent Context card still cannot display emoji-only avatars from agent identity.
