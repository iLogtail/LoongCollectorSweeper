## Summary

Keep open. Current main still contains the source pattern described in #59545: the General tab mounts the Tailscale settings section in local mode, that section hydrates SwiftUI state from config, and its save-on-change handlers still call the same persistence path that rewrites gateway and auth fields. I did not find a source fix, regression test, or canonical duplicate that makes this safe to close.

## Best Possible Solution

Keep #59545 open. The likely fix belongs in apps/macos/Sources/OpenClaw/TailscaleIntegrationSection.swift: separate config hydration from user-driven edits with an explicit loading/user-change guard, or compare the normalized output config and no-op when loading produces no semantic change. Add a regression test around the Tailscale settings load path using ConfigStore test overrides or a small test seam to prove opening General does not call save or mutate gateway.bind/auth/meta without an explicit user action.
