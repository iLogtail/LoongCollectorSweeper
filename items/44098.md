## Summary

Keep open. Current main still does not set a default Docker PID limit for sandbox containers: `resolveSandboxDockerConfig` only inherits agent/global config, and `docker create` only emits `--pids-limit` when a positive value is already present. The PR is still a relevant focused fix for the linked open security report #38604, but it should be completed with regression coverage and config/docs alignment before merge.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer patch that adds a default sandbox Docker PID limit in `resolveSandboxDockerConfig`, preserves explicit operator overrides, adds regression tests for unset/global/agent/shared-scope behavior, and updates the user-facing sandbox config docs/types/schema expectations.
