## Summary

Keep open. Current main does not contain the #64563 fix and still leaves the reported WhatsApp profile-isolation surface unguarded: the bundled WhatsApp light export resolves `WA_WEB_AUTH_DIR` eagerly, account auth-dir helpers still call `resolveOAuthDir()` from ambient `process.env`, and the latest release tag has the same implementation. The related PR #64563 directly targeted this issue but is closed unmerged, so it is not a canonical shipped fix or superseding open tracker.

## Best Possible Solution

Keep #64555 open and implement the #64563-style fix on main: make the bundled WhatsApp `WA_WEB_AUTH_DIR` export lazy or remove reliance on it, thread an explicit env/state-dir through WhatsApp account auth-dir resolution where feasible, and add regression coverage for `--profile`/`OPENCLAW_STATE_DIR` import-order behavior on the `web.login.start` path. A doctor warning for creds outside the active profile state dir would be a useful follow-up, but the core bug should be fixed first.
