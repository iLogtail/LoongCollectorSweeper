## Summary

Current main still does not clean up the default Docker sandbox workspace directory during prune. The prune path removes the runtime and registry entry only, the Docker backend runtime removal is just `docker rm -f`, and the sandbox workspace is still a persistent directory created under `agents.defaults.sandbox.workspaceRoot`. The related open PR #43835 is a plausible fix attempt, but it has not landed, so #43797 should remain open.

## Best Possible Solution

Keep #43797 open and land a focused sandbox lifecycle fix. The likely maintainer path is to compute the local Docker sandbox workspace from `cfg.workspaceRoot` plus the registry `sessionKey`, delete it only for non-shared sandbox workspaces after successful runtime removal, and add regression coverage for both prune and `sandbox recreate` so config/bootstrap-file changes take effect without manual directory deletion. PR #43835 can be reviewed as the existing candidate, or replaced with a maintainer-owned patch if it is stale.
