## Summary

Keep open. Current main already has stable/beta/dev release channels, beta-first stable promotion, release branches, and release-check workflows, but this issue asks for a fixed monthly stable cadence, optional weekly stable builds, and an explicit support/compatibility commitment. I did not find that calendar cadence or support promise implemented or clearly superseded by the related items.

## Best Possible Solution

Keep the issue open for release-manager triage. If maintainers want this promise, update `docs/reference/RELEASING.md`, public channel/update docs, and any release automation to define the calendar cadence, support window, and stable/beta/dev comparison. If maintainers do not want a fixed cadence commitment, they should close it manually as a product/release-policy decision rather than via automated cleanup.
