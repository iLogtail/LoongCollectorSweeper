## Summary

Keep open. Current main still rejects `auth-profiles.json` entries whose credential `type` is `aws-sdk`; the linked fix PR #69753 was closed unmerged, and the current parser still classifies every type outside `api_key`, `oauth`, and `token` as `invalid_type`. Provider-level `auth: "aws-sdk"` exists and is documented, but the specific persisted-marker rejection reported in #69708 remains in source.

## Best Possible Solution

Keep #69708 open and land a narrow auth-profile store fix, likely the #69753 approach: treat `type: "aws-sdk"` as an SDK-managed non-secret marker during store load, avoid counting it as an invalid entry, preserve warnings for genuinely invalid profiles, and add regression coverage for both legacy and versioned `auth-profiles.json` shapes.
