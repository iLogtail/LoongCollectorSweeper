## Summary

Keep open. Current main has fixed the broader Bedrock IMDS runtime path by allowing `auth: "aws-sdk"` providers to run without an explicit API key, but it still rejects persisted `auth-profiles.json` entries with `type: "aws-sdk"` as `invalid_type` and drops them during store load. The linked PR #69753 proposed the narrow marker-skip behavior for this exact issue, but it was closed unmerged, so the reported auth-profile parser behavior remains unresolved.

## Best Possible Solution

Keep #69708 open unless maintainers decide that `aws-sdk` must never be a persisted auth-profile credential. The concrete code path to resolve it is a narrow auth-profile store-load change: treat `type: "aws-sdk"` as a non-secret SDK-managed marker and skip it without counting it as an invalid entry, or add an explicit migration/doctor repair that removes stale markers and documents provider-level `auth: "aws-sdk"` as the supported path. Add regression coverage in `src/agents/auth-profiles.ensureauthprofilestore.test.ts` for an `amazon-bedrock:default` marker mixed with a real credential, and keep the existing Bedrock IMDS runtime tests.
