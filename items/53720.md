## Summary

Keep PR #53720 open. Current main does not implement the central feature: there is no `agents.list[].enabled` config field, no `openclaw agents disable/enable` command surface, no list output status, and routing/cron/heartbeat still treat configured agents as active without an agent-level disabled filter. The PR is still relevant, but it should address the later review feedback before merge, especially making cron and heartbeat honor disabled agents and keeping `--json` output clean.

## Best Possible Solution

Keep the PR open and ask for a small follow-up revision: add the typed/schema/doc/CLI `enabled` surface, preserve disabled entries in config mutation paths, make routing/cron/heartbeat/channel polling consistently skip disabled agents, keep `--json` output parseable, and add focused regression tests for those paths. Once those review comments are addressed, this remains the right core location for the feature because current main has no non-destructive per-agent suspension mechanism.
