## Summary

Keep open. Current main still has the exact display-name path this PR targets: `buildGatewaySessionRow` falls back to `origin.label`, and the heartbeat route can still synthesize the sender value `"heartbeat"`. The PR is not cleanup-closeable as implemented, though, because the current patch globally suppresses any origin label equal to `"heartbeat"`; current metadata code also allows `origin.label` to be a real direct-chat sender/conversation label. Related PR #66656 is separate preview filtering work and is still open/unmerged in the provided context.

## Best Possible Solution

Keep the PR open for maintainer review, but revise it before merge: suppress only synthetic heartbeat-origin metadata, ideally at the origin/sender provenance boundary or with a shared predicate, and add focused regression tests proving the heartbeat fallback is hidden while a legitimate `origin.label: "heartbeat"` remains visible. Coordinate separately with #66656 for last-message preview filtering if that symptom still remains.
