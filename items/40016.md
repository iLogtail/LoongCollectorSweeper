## Summary

Current `main` does not implement a CLI command to set a session subject/label, and the existing `openclaw sessions` output still relies on truncated keys rather than session naming metadata.

## Evidence

- **CLI surface has no rename/set-subject subcommand:** `registerStatusHealthSessionsCommands` defines `openclaw sessions` plus only the `cleanup` subcommand. There is no `set-subject`, `rename`, or equivalent mutation command under `sessions`.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/cli/program/register.status-health-sessions.ts#L124)
  - command: `nl -ba src/cli/program/register.status-health-sessions.ts | sed -n '100,210p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Sessions command only lists sessions:** `sessionsCommand` accepts listing/filtering options (`json`, `store`, `active`, `agent`, `allAgents`) and emits rows from the store; it does not accept or apply any subject/display-name update input.
  - file: [src/commands/sessions.ts:110](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/commands/sessions.ts#L110)
  - command: `nl -ba src/commands/sessions.ts | sed -n '110,210p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Displayed rows omit subject/displayName/label:** `SessionDisplayRow` and `toSessionDisplayRows()` do not carry `label`, `displayName`, or `subject`, and `formatSessionKeyCell()` truncates the raw key. That matches the issue's complaint that the CLI output is hard to identify.
  - file: [src/commands/sessions-table.ts:5](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/commands/sessions-table.ts#L5)
  - command: `nl -ba src/commands/sessions-table.ts | sed -n '1,160p'`

- **Docs only cover list and cleanup:** The CLI docs for `openclaw sessions` document listing flags and `sessions cleanup` only; there is no documented rename/set-subject workflow.
  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/docs/cli/sessions.md#L8)

However, the gateway `sessions.patch` schema accepts `label` but not `displayName` or `subject`, so even the mutation API does not implement the full requested feature.
