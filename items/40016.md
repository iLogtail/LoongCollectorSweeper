## Summary

Keep #40016 open. Current main still does not provide `openclaw sessions set-subject` or `openclaw sessions rename`, and the text/JSON sessions CLI output is still built around the raw session key rather than a user-editable label. Gateway-level `sessions.patch` label support exists, so the remaining gap is the requested user-facing CLI surface and docs/tests for it.

## Best Possible Solution

Keep the issue open and implement a small CLI surface under `openclaw sessions`, likely `rename` with `set-subject` as an alias if desired, backed by the existing `sessions.patch` label support. The implementation should also update `docs/cli/sessions.md`, add CLI registration/command tests, and consider displaying the label/displayName in `openclaw sessions` output so the original readability problem is actually solved.
