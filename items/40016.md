## Summary

Keep open. Current main has backend `sessions.patch` label support, and the issue discussion noted an intended implementation, but the requested first-class `openclaw sessions rename` / `set-subject` CLI surface and readable sessions-list label display are still absent.

## Best Possible Solution

Keep the issue open. Implement a focused `openclaw sessions rename <session-key> <label>` command, with `set-subject` or `set-label` as an alias if maintainers want that wording. Reuse the existing `sessions.patch` label validation/path, keep channel-owned `subject` and `displayName` semantics intact, update `openclaw sessions` text/JSON output to show the effective human-readable label before or beside the key, and add CLI tests plus `docs/cli/sessions.md` coverage.
