## Summary

Keep #29100 open. Current main still lacks a supported pricing/cost metadata overlay for auto-discovered Amazon Bedrock models: the plugin discovery schema and legacy type/schema expose discovery controls but no `costs` map, discovered foundation models still receive `DEFAULT_COST`, inference profiles only inherit the base/default cost, and docs still tell users that cost estimates come from explicit provider model `cost` entries. The related PR #31160 attempted this feature but closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models, preferably in the Amazon Bedrock plugin-owned discovery config or through a deliberate general model-cost overlay. The fix should update config types/schema/manifest UI metadata, apply costs to foundation models and inference profiles during discovery, include pricing in discovery cache-key semantics, add focused Bedrock discovery tests, and update the public Bedrock and token-use docs.
