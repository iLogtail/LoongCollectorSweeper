## Summary

Keep #29100 open. Current main still has no supported cost/pricing overlay for auto-discovered Amazon Bedrock models: the plugin discovery schema is strict and has no `costs` field, the legacy Bedrock discovery type/schema also lacks cost metadata, discovered foundation models are assigned zero/default cost, inference profiles only inherit that default/base cost, and docs still describe cost estimation as coming from explicit `models.providers.<provider>.models[].cost` entries. The related PR #31160 attempted this feature but is closed unmerged and is not present in the current checkout history.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models. The likely implementation should live in the Amazon Bedrock plugin-owned discovery config, or in a deliberate general `models.costs` overlay, and should update the manifest/schema/types/help docs, apply costs to foundation models and inference profiles, include pricing in discovery cache semantics, add focused Bedrock discovery tests, and update the public Bedrock and token-use docs.
