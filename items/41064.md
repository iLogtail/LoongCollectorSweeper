## Summary

Keep #41064 open. Current main still clears Dashboard chat busy state on Gateway hello/reconnect, and the Gateway hello snapshot/schema does not expose active chat runs that the UI could use to restore `chatRunId`. The linked PR #41067 directly targets this bug, but it is open and unmerged, so the issue is still a valid tracker.

## Best Possible Solution

Keep the issue open and use #41067 as the current implementation candidate. The best fix is to add a typed Gateway hello snapshot/recovery contract for active chat runs, restore `chatRunId` for the selected session instead of blindly clearing it on hello, keep composer controls in `Stop`/`Queue` while the recovered run is abortable, and add focused UI/protocol regression tests for reconnect/tab-return busy-state recovery.
