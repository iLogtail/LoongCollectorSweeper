## Summary

Keep open. Current `main` still derives the chat toolbar abort/busy state from in-memory `chatRunId`, clears that state on reconnect/hello, and does not recover it from session/history data. That matches the report rather than proving it fixed.

## Evidence

- **Toolbar busy state still depends on `chatRunId`:** The chat view passes `canAbort: Boolean(state.chatRunId)` into the run controls, and the controls render `New session`/`Send` whenever `canAbort` is false.
  - file: [ui/src/ui/app-render.ts:2264](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-render.ts#L2264)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '2190,2310p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Reconnect path explicitly drops active-run state:** On gateway hello, the dashboard unconditionally resets `host.chatRunId = null` and clears stream state instead of restoring the active run for the current session.
  - file: [ui/src/ui/app-gateway.ts:315](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-gateway.ts#L315)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '253,337p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **History reload does not recover busy state:** `loadChatHistory` updates messages/thinking level and clears stream artifacts, but it never restores `chatRunId` or any other active-run marker for the session.
  - file: [ui/src/ui/controllers/chat.ts:179](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/controllers/chat.ts#L179)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '139,202p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Current tests lock in the reconnect-clears-run behavior:** The dashboard gateway test expects `host.chatRunId` to become `null` after reconnect hello, which is the opposite of the requested busy-state recovery.
  - file: [ui/src/ui/app-gateway.node.test.ts:539](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-gateway.node.test.ts#L539)
