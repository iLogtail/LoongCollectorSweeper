## Summary

Keep #59737 open. Current main still has a single comprehensive `.github/pull_request_template.md`, no `.github/PULL_REQUEST_TEMPLATE/` directory, and no CONTRIBUTING guidance for choosing quick vs full PR templates. The request is recent, actionable in this repository, not protected by author association or labels, and the existing community comment provides a plausible implementation sketch rather than evidence that the work landed.

## Best Possible Solution

Keep the issue open for a small repo-DX change: add `.github/PULL_REQUEST_TEMPLATE/quick_fix.md` and `full_change.md`, keep or intentionally adjust `.github/pull_request_template.md` as the fallback, update `CONTRIBUTING.md` with accurate template-selection guidance, and sanity-check any PR triage automation that assumes the current single comprehensive template.

## Risks / Open Questions

- Do not assume GitHub will display an issue-style PR template chooser; implement guidance around supported PR template URLs/query parameters unless maintainers confirm a newer UI.
