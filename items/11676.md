## Summary

Keep #11676 open. Current main still ships OpenClaw as a Node-hosted TypeScript gateway/CLI with Node and desktop/server OS install requirements. The existing plugin, Gateway WebSocket, node-device, and embedded-agent helper surfaces are useful extension points, but they do not provide the requested runtime-agnostic agent core/protocol for C/Rust/WASM, bare-metal, or microcontroller ports. The linked MimiClaw project remains useful external prior art rather than an implementation or compatibility contract in this repo.

## Best Possible Solution

Keep the issue open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference client proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful prior art.
