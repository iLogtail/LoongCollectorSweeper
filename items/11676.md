## Summary

Keep #11676 open. Current main still ships OpenClaw as a Node-hosted TypeScript application with documented Node and desktop/server OS requirements. The existing agent runtime, plugin runtime, and agent harness seams allow trusted in-process or native-hosted harnesses, but they do not provide a stable runtime-agnostic agent core/protocol for bare-metal, C/Rust/WASM, or no-Node embedded ports. MimiClaw remains useful prior art, and the existing issue discussion already includes a prior Codex review by steipete choosing to keep this product-direction item open.

## Best Possible Solution

Keep the issue open until maintainers make an explicit runtime-portability decision. A strong implementation path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, and platform adapters, then add compatibility tests or reference clients proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain cited as useful prior art.
