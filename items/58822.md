## Summary

Keep #58822 open. Current main still implements the reported precedence bug: both the normal subagent model helper and the isolated cron path check the target agent's own model before `agents.defaults.subagents.model`. Existing tests also assert that old order, while the public-facing docs describe the global subagent model setting as the way to override caller inheritance. The related fixing PR #58823 is still open and unmerged, so this is not implemented on main or superseded by shipped work.

## Best Possible Solution

Keep #58822 open and review/merge #58823 or an equivalent maintainer patch. The likely code path is to use `agentConfig.subagents.model -> agents.defaults.subagents.model -> agentConfig.model` in both `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts`, then update tests for main-agent, named-agent, per-agent subagent override, and explicit spawn/job model override behavior.
