## Summary

Keep open. Current checked-out main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still does not emit zsh positional argument specs for leaf commands, so PR #69305 is not implemented or superseded on main. The PR is a relevant fix for #69293, though its remaining heuristic concern around non-filesystem `<path>` arguments should be handled before merge.

## Best Possible Solution

Keep #69305 open as the focused zsh positional-completion fix. Before merge, preserve the new regression tests for `wiki ingest` and optional/variadic positionals, and tighten the completion-action mapping so filesystem completion is used for actual file/directory arguments without misclassifying config-key positionals.
