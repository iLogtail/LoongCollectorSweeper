## Summary

Keep open. Current main does not contain the focused fix described in linked PR #52140, and the skill install download paths still have plausible mid-stream hang gaps. Telegram queueing is documented/expected, but the remaining user-visible bug is that a long or stalled install can leave the agent without a useful completion/failure response.

## Best Possible Solution

Keep #52073 open. The best fix is a narrow skill-install timeout hardening pass: keep one AbortSignal alive through both request and body consumption, pass it to stream pipelines where used, add a body/chunk timeout for ClawHub `arrayBuffer()` downloads or convert them to a bounded streaming helper, and add regression tests that simulate headers followed by a stalled body. Preserve documented per-session queue semantics while ensuring the active turn eventually emits a failure or completion response.
