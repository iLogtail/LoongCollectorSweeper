## Summary

Current main still lacks a generic `/resume` or equivalent restore flow for pre-reset conversations. `/clear` resets the active session, archives the old transcript, and the built-in session listing only exposes live store entries, so the pre-clear context is not recoverable through the normal UI/command surfaces.

## Evidence

- **No core `/resume` command:** Core chat commands include `/reset`, `/new`, and `/compact` for session lifecycle, but there is no generic `/resume` command in the core registry.
  - file: [src/auto-reply/commands-registry.shared.ts:663](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/auto-reply/commands-registry.shared.ts#L663)
- **Only plugin-scoped resume is documented:** The slash-command docs list `/codex ... resume` under bundled plugin commands; there is no top-level `/resume` command documented for normal OpenClaw sessions.
  - file: [docs/tools/slash-commands.md:157](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/tools/slash-commands.md#L157)
- **`/clear` performs a reset:** WebChat's UI-only `/clear` handler calls `sessions.reset` for the current session key, so clearing starts a fresh session instead of restoring prior context.
  - file: [ui/src/ui/app-chat.ts:481](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/app-chat.ts#L481)
- **Reset replaces the live store entry:** The reset service overwrites the primary session-store entry with `nextEntry`, which moves the active key to a new `sessionId` and drops the prior session from the live store.
  - file: [src/gateway/session-reset-service.ts:617](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/session-reset-service.ts#L617)
- **Old transcript is archived, not reattached:** During reset, the previous transcript is archived with reason `reset`, creating `*.reset.<timestamp>` artifacts rather than a built-in restored/live session entry.
  - file: [src/gateway/session-reset-service.ts:629](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/session-reset-service.ts#L629)
- **Session listing only uses current store entries:** `sessions.list` is built from `Object.entries(store)` and maps current store rows only, so archived reset transcripts are not surfaced as selectable sessions.
  - file: [src/gateway/session-utils.ts:1483](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/session-utils.ts#L1483)
