## Summary

Keep #55982 open. Current main does not implement `skipWhenIdle`, and the PR is a focused feature rather than obsolete cleanup. The branch also has unresolved maintainer-review concerns around persistence and production activity wiring, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep the PR open and ask for a focused update: add `skipWhenIdle` across the cron type/protocol/CLI surfaces, persist it in job create/update paths, wire a reliable last-inbound activity source in production gateway cron construction, preserve the `at`-job guard or define explicit deferred behavior, and add regression tests plus user docs for the new option.

## Risks / Open Questions

- Merging the PR without the review fixes could make CLI add/edit appear to accept the option while not storing it.
- Treating missing activity data as idle requires a production source of last inbound main-session activity; otherwise enabled jobs may skip forever.
- The feature changes the cron protocol surface, so schema, CLI, docs, service tests, and gateway wiring need to land together.
