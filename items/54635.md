## Summary

Keep open. Current main still has the exact warning-policy behavior reported: mutating tool errors return `showWarning: true` unconditionally, while recoverable-error suppression is only applied to the non-mutating fallback branch. Existing edit recovery is relevant partial mitigation, but it does not prove this warning-policy issue is fixed, and related open items remain unsuperseded.

## Best Possible Solution

Keep #54635 open. The likely maintainer path is a focused change in `src/agents/pi-embedded-runner/run/payloads.ts` or the upstream `lastToolError` clearing path, with tests proving that recoverable/post-write-applied mutating failures do not emit false `failed` notifications while genuine mutating failures remain visible. Related open work in #59868/#60664 should be considered, but there is no shipped canonical fix to close this against.
