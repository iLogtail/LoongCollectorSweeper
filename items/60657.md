## Summary

Keep #60657 open. Current main still formats collected busy-session messages with `Queued #N` markers only, without the requested total count such as `[Queued 1/3]`. The related #64410 context supports that collect-mode batching is already implemented, but this narrower sequence-total prompt improvement remains missing.

## Best Possible Solution

Implement total-aware collect prompt rendering in the queue helper or its render callbacks, then update the auto-reply collect tests to assert markers like `[Queued 1/3]` and the header total. Consider applying the same pattern to subagent announce collect prompts for consistency.
