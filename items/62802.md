## Summary

Keep open. Current main still contains the undefined-to-0 timeout coercions this PR targets, and `resolveAgentTimeoutMs` still treats `undefined` and `0` differently. The observable `sessions_spawn` default-timeout path is partly addressed by spawn-time default resolution, but the lower-level registry, steer, and reactivation cleanup is not implemented on main and is not superseded by another open canonical item in the provided context.

## Best Possible Solution

Keep #62802 open as the canonical remaining cleanup. Rebase/adapt it to current main by removing the undefined-to-0 coercions in `resolveSubagentWaitTimeoutMs`, `src/agents/subagent-registry-run-manager.ts`, `src/agents/subagent-control.ts`, and `src/gateway/session-subagent-reactivation.ts`; add focused regression coverage proving omitted `runTimeoutSeconds` propagates as `undefined` while an explicit `0` still means no timeout.
