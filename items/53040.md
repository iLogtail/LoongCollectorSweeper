## Summary

Keep open. Current main still uses the same fetch-backed Signal SSE subscription and fetch-backed Signal RPC send path reported in #53040, with no Node 25/macOS/npm-specific fallback or replacement visible in source, docs, or tests. Related #51716 remains open for the same Signal SSE fetch-failure family, but #53040 adds Node 25/macOS/npm and outbound send evidence, so this is not safe to auto-close as implemented or as a shallow duplicate.

## Best Possible Solution

Keep the issue open, or have a maintainer explicitly consolidate it with #51716. The fix likely belongs in `extensions/signal`: replace or harden the Signal SSE and RPC transport paths for Node 24/25 npm installs, then add focused regression coverage for `/api/v1/events` streaming and `/api/v1/rpc` send behavior under the chosen transport.
