## Summary

Current `main` still applies `before_model_resolve` inside `runEmbeddedPiAgent`, after `runWithModelFallback(...)` has already chosen fallback candidates and after `agent-runner-execution` has begun persisting candidate selection. That means plugin `providerOverride` / `modelOverride` is not the source of truth for fallback candidate construction, fallback bookkeeping, or persisted fallback state, so the report is not strongly closeable as implemented-on-main or cannot-reproduce.
