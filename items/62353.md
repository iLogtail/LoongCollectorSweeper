## Summary

Keep open. Current main still has no generic HTTP `/rpc` JSON-RPC gateway endpoint: Gateway RPC remains documented as the WebSocket control-plane transport, and the existing `/mcp` JSON-RPC loopback is MCP-tool-specific rather than a general gateway-method adapter. The PR is a plausible core gateway/protocol feature, but the provided review thread identifies substantial security and contract gaps, so it needs maintainer design review or a rewritten patch rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is either a core-owned HTTP JSON-RPC transport that reuses the existing gateway HTTP auth/rate-limit/body helpers, real GatewayRequestContext, runtime extraHandlers, caller identity/scopes, and protocol tests/docs, or an explicit plugin/API design that makes such an adapter safe outside core.
