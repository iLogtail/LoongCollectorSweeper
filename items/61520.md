## Summary

Keep #61520 open. Current main still silently swallows the exact classes of config IO failures described by the issue: health-state persistence, audit-log append failures, backup maintenance, and runtime listener failures remain best-effort catches without operator-visible logging. The related PRs are useful context, but #61522 is still unmerged and #61609/#61874 were closed unmerged, so there is no shipped fix on main.

## Best Possible Solution

Keep #61520 open and use #61522 or a smaller maintainer-owned patch as the implementation path. The best fix is to preserve best-effort semantics while adding scoped warning/debug/error reporting for health-state persistence, audit append failures, backup maintenance, and other config IO catches where permission, ENOSPC, or locking failures would otherwise be invisible; add focused tests for at least one async and one sync failure path.
