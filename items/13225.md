## Summary

Keep #13225 open. Current main still does not implement the requested per-tool-call delegation contract: `before_tool_call` can rewrite params, block, or require approval, while model routing remains a run-level `before_model_resolve` hook that runs before session messages are available. The related open PRs #58079 and #62701 are useful partial context work, but they do not add model/session redirect semantics, and the latest issue discussion explicitly supports keeping this item open.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. Preserve the existing `before_tool_call` params/block/approval behavior and the run-level `before_model_resolve` override hook, but only close this after maintainers either ship or explicitly reject an additive typed contract that lets plugins inspect the relevant preceding assistant output and redirect a pending tool call to another model or session. Treat #58079 and #62701 as partial context work, not canonical closure for this issue unless they are merged and extended to cover routing.
