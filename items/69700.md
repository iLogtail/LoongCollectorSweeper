## Summary

Keep #69700 open. Current main still treats every `sources/*.md` file as a memory-wiki source page and applies required wiki frontmatter checks, while the plugin docs still describe `sources/` as imported raw material. The ingest and bridge paths wrap raw content in generated source pages with provenance frontmatter, but that does not resolve the reported user-facing ambiguity for raw markdown dropped directly under `sources/`.

## Best Possible Solution

Keep the issue open for a small memory-wiki product/docs fix. The conservative path is to update `docs/plugins/memory-wiki.md`, `docs/cli/wiki.md`, and lint/CLI messaging to state that `sources/` contains OpenClaw source pages with provenance frontmatter, and that raw files should be imported with `openclaw wiki ingest` or bridge import. If raw unmodified files are intended to be supported directly, add an explicit raw-source mode or ignored raw-source location with focused lint/compile tests.
