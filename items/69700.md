## Summary

Current main still treats Markdown files under `sources/` as wiki source pages and applies the same required `id`, `pageType`, and `updatedAt` lint path. The docs still contain the cited “imported raw material” wording, while other CLI docs and import code show the intended source-page wrapper/provenance model. Related #64696 is relevant background for malformed bridge source pages, but it does not supersede this narrower docs/lint contract question.

## Best Possible Solution

Keep this open for maintainer product/docs judgment. The likely fix is to make the memory-wiki contract explicit: either document that `sources/` contains managed source pages that wrap raw imports and require frontmatter, or add an explicit raw/preserve mode that lint can detect without weakening checks for bridge/imported source pages. A narrow docs update plus regression coverage for raw/manual `sources/*.md` behavior would be the safest path.
