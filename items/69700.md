## Summary

Current main still matches the report: memory-wiki compile/lint includes every Markdown file directly under `sources/`, classifies it as a source page, and then requires `id`, `pageType`, and freshness metadata from frontmatter. The docs still contain the `sources/` raw-material wording while the importer/bridge implementations wrap raw content into provenance-bearing wiki source pages, so this remains an actionable product/docs/behavior contract issue rather than a closable duplicate or implemented fix.

## Best Possible Solution

Keep #69700 open for a maintainer decision on the `sources/` contract. The likely fix is either to clarify public docs that `sources/` contains provenance-bearing wiki source pages with raw content preserved inside them, or to add an explicit raw-source mode/subdirectory/lint exemption with regression tests; coordinate any malformed bridge-source handling with open issue #64696.
