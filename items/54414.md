## Summary

Keep open. Current main already ships the central Slack thread mention opt-out under `channels.slack.thread.requireExplicitMention`, but PR #54414 also fixed a reviewed multi-account config-merging edge case that current main does not appear to cover. The PR should not be merged as-is because it adds an overlapping config key, but it is not fully obsolete.

## Best Possible Solution

Do not merge the PR's new `autoReplyOnParticipation` config key. Port the useful part as a narrow Slack follow-up: make `mergeSlackAccountConfig` deep-merge the `thread` object and add tests proving a global `channels.slack.thread.requireExplicitMention: true` survives account-level `thread` overrides such as `historyScope`. After that lands, this PR can be closed as superseded by the shipped `requireExplicitMention` implementation plus the targeted merge fix.
