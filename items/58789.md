## Summary

Keep open. Current main has Discord bot self-filtering and an optional `allowBots: "mentions"` gate, but it does not implement the requested bot-to-bot pair rate limit/cooldown. The linked #65403 RFC is related discussion, not a canonical replacement for this specific Discord loop-protection request.

## Evidence

- **Current allowBots behavior:** Discord preflight maps `allowBots` only to `off`, `all`, or `mentions`, and only filters the current bot's own messages before continuing. There is no per sender/receiver/channel history, threshold, or cooldown in this path.
