## Summary

Keep #58789 open. Current main still admits Discord bot-authored messages when `allowBots: true`, only filters the receiving bot's own messages and the optional `allowBots: "mentions"` mode, and has no pairwise bot-to-bot loop/rate limiter or `botLoopProtection` config. The related #65403 RFC is broader and explicitly treats bot-loop detection as a later safety dependency rather than a shipped fix or canonical duplicate.

## Best Possible Solution

Leave #58789 open for a focused Discord/plugin-owned safety implementation: add a strict config schema/docs entry for bot-loop protection, track bot-author to receiving-bot pairs per guild/channel/account, suppress after a configurable threshold/cooldown, and cover `allowBots=true` plus `allowBots="mentions"` with preflight or inbound-worker tests. If maintainers want this generalized beyond Discord, introduce an additive SDK/channel helper rather than hard-coding channel behavior in core.
