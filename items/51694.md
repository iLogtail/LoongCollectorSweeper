## Summary

Keep open. Current main still exposes Google Chat `appPrincipal` as optional in the Zod schema, TypeScript config type, and generated config metadata, while the runtime still rejects Google Chat app-url add-on tokens when no principal binding is available. The attempted fix PR #51817 was closed unmerged, so there is no shipped implementation that resolves this issue.

## Best Possible Solution

Keep #51694 open and implement the Google Chat config contract deliberately: either make `appPrincipal` required for the app-url/add-on-token path in the schema, TS type, generated metadata, docs, and regression tests, or document and validate the narrower conditional requirement so users fail at config validation instead of getting webhook 401s.
