## Summary

PR #64865 is superseded by merged PR #64779, which closed #64748 and shipped the intended user-facing fix: `openclaw wiki` resolves through the owning `memory-wiki` plugin when `plugins.allow` is restrictive. Current main has manifest-backed command alias metadata, CLI allow-guard coverage, validation guidance, and changelog/release evidence.

## Best Possible Solution

Close #64865 as superseded by merged #64779. Keep the shipped manifest-backed alias ownership and parent-plugin allowlist behavior; any future desire to preserve command aliases as authored allowlist entries should be opened as a separate product decision with tests against the current structured `commandAliases[].name` contract.
