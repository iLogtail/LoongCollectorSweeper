## Summary

Current main still appears vulnerable to the reported sandbox skill-path mismatch. The agent command builds a skills snapshot from the canonical workspace before sandbox execution, non-rw sandbox runs switch to the sandbox workspace and sync skills there, but the embedded runner still passes the original snapshot through. Because snapshots include `resolvedSkills`, runtime skill loading is skipped and `resolveSkillsPromptForRun` returns the snapshot prompt unchanged, including home-compacted host paths. Related PRs #43526 and #51471 proposed the missing refresh/remap behavior, but they are closed and their changes are not present on current main.

## Best Possible Solution

Keep the issue open and implement the sandbox-specific prompt refresh/remap that the closed PRs attempted: when a run is executing from a non-rw sandbox workspace, prefer skill entries loaded from the effective sandbox workspace over the persisted snapshot prompt, then render paths that the sandbox read tool can resolve (`skills/<skill>/SKILL.md` or `/workspace/skills/<skill>/SKILL.md`). Add regression coverage for a populated skills snapshot plus non-rw sandbox execution so host-compacted paths cannot reappear.
