## Summary

Keep open. Current main still returns an existing skills snapshot prompt unchanged in sandboxed runs, while the prompt builder intentionally compacts home-dir skill paths to `~/...`. For `workspaceAccess: "none"`, the sandbox FS bridge only mounts the sandbox workspace and omits the agent workspace, so a host-style managed skill path remains outside the sandbox. Related PR #43526 directly tracks the proposed fix but is still open/unmerged, so this is not implemented on main.
