## Summary

Keep open. Current main preserves iMessage reply context down to the iMessage send helper, and it now strips inline reply tags, but the RPC payload still sends `reply_to` rather than the requested `reply_to_guid`/`replyToGUID` field. The related PR #51966 was closed unmerged, so it does not supersede the issue. This remains a concrete bundled iMessage behavior gap rather than a conservative close candidate.
