## Summary

Current `main` already routes bundled channel secret discovery through lightweight packaged sidecars and explicit channel fast paths, which removes the eager bootstrap/runtime import path that matched this `target-registry -> signal/api -> pi-embedded` regression report. The changelog records that fix as shipped in `v2026.4.8`.

## Evidence

- **Target registry no longer enumerates bootstrap channel runtimes:** `listChannelSecretTargetRegistryEntries()` now walks bundled manifest records and loads `loadBundledChannelSecretContractApi(record.id)` for each channel. It does not call `iterateBootstrapChannelPlugins()` or import full channel runtime barrels from this path.
  - file: [src/secrets/target-registry-data.ts:70](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/secrets/target-registry-data.ts#L70)
  - command: `nl -ba src/secrets/target-registry-data.ts | sed -n '1,140p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Bundled channel secret contracts load through packaged sidecars:** `loadBundledChannelSecretContractApi()` resolves `secret-contract-api.js` / `contract-api.js` via `loadBundledPluginPublicArtifactModuleSync`, which is the lightweight packaged public-surface path instead of the full plugin runtime.
  - file: [src/secrets/channel-contract-api.ts:57](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/secrets/channel-contract-api.ts#L57)
  - command: `nl -ba src/secrets/channel-contract-api.ts | sed -n '1,220p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Signal exposes a lightweight secret-contract sidecar:** `extensions/signal/secret-contract-api.ts` is just an empty `secretTargetRegistryEntries` export plus a no-op collector, so secret-target discovery no longer needs to import Signal's runtime API barrel.
  - file: [extensions/signal/secret-contract-api.ts:1](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/extensions/signal/secret-contract-api.ts#L1)
  - command: `nl -ba extensions/signal/secret-contract-api.ts | sed -n '1,120p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Explicit channel target lookup now has a fast path:** `resolveConfigSecretTargetByPath()` checks `channels/<id>/...` targets through `getCompiledBundledChannelOpenClawTargets()` before the full registry fallback, and the paired fast-path test asserts this path does not scan the manifest registry.
  - file: [src/secrets/target-registry-query.ts:320](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/secrets/target-registry-query.ts#L320)
