## Summary

Keep #64953 open. Current main still has the reported behavior gap: both CLI picker paths and the gateway catalog path call the full catalog loader without a `models.mode === "replace"` branch, and `loadModelCatalog` itself still returns the merged registry/plugin catalog without replace-mode filtering. The PR remains a plausible focused fix for #64950, though maintainer design judgment is still needed on whether replace-mode picker/catalog results should include implicit or plugin-discovered models.

## Best Possible Solution

Keep #64953, #64950, or a canonical successor open. Maintainers should define the replace-mode catalog contract, then implement a shared mode-aware catalog path for `promptDefaultModel`, `promptModelAllowlist`, `loadGatewayModelCatalog`, and `models.list`, with regression tests covering strict configured-provider replacement plus any intended implicit/plugin-discovery behavior.
