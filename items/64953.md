## Summary

Keep open. Current main still has the reported behavior gap: the gateway catalog and both CLI model picker paths call the full catalog loader without checking `models.mode`, while docs and runtime selection already treat `models.mode: "replace"` as a replacement/configured-provider mode. The PR is a plausible focused bug fix, but review discussion also raised a real design question about implicit/plugin-discovered models, so this should stay open for maintainer resolution rather than be auto-closed.

## Best Possible Solution

Keep the PR or a canonical successor open and implement a mode-aware catalog path shared by `loadGatewayModelCatalog`, `promptDefaultModel`, and `promptModelAllowlist`. The maintainer decision should define whether `models.mode: "replace"` means strictly `cfg.models.providers` only or a resolved replacement catalog that still includes implicit/plugin-discovered provider entries; then add regression tests for both CLI picker functions and gateway `models.list`.
