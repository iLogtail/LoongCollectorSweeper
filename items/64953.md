## Summary

Keep #64953 open. Current main still does not implement the requested replace-mode behavior for the CLI model picker or gateway catalog: those paths still load the full model catalog, while the config/schema contract says `models.mode: "replace"` should use only configured providers. Related PR attempts in the provided context are closed unmerged, and #64950 remains the open linked bug, so there is no shipped or canonical superseding fix to close this PR against.

## Best Possible Solution

Keep #64953, #64950, or a canonical successor open. The best fix is a shared mode-aware catalog path for the CLI picker and gateway `models.list` that honors `models.mode: "replace"`, explicitly resolves the intended implicit/plugin-discovery behavior, preserves configured metadata, and adds regression coverage for default picker, allowlist picker, and gateway catalog results.
