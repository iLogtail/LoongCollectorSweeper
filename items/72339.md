## Summary

Current main already handles the reported v2026.4.24 Bonjour/CIAO crash path. The exact `CIAO PROBING CANCELLED` reason is classified as non-fatal, the Bonjour plugin wires scoped process handlers into the advertiser, the global unhandled-rejection path returns before fatal exit when a registered handler handles the reason, and the advertiser now keeps those handlers registered across watchdog restarts so late probe cancellations do not crash the gateway.

## Best Possible Solution

Keep the current main implementation: scoped Bonjour/CIAO process handlers should remain wired through the plugin SDK, shared across plugin module instances, and active across advertiser restarts. For users still pinned to v2026.4.24, point them to upgrade once a fixed build is available or temporarily disable Bonjour/mDNS when LAN multicast is unreliable.
