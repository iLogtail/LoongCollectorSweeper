## Summary

Keep #68987 open. Current main still uses the hard-coded adjacent-token argv sniff for `secrets audit`, while docs describe `secrets audit` as read-only and current consumers still rely on `OPENCLAW_AUTH_STORE_READONLY` to suppress auth-store mutations. The linked #68803 is planning-only and closed unmerged, not a shipped implementation or canonical replacement. The Greptile comments are P2 documentation/style notes and do not make this PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and either land an equivalent command-owned read-only path guard with regression coverage, or ask the author to adjust the manifest approach to the preferred CLI command-metadata pattern. #68803 should not be treated as a superseding item because it was planning/meta work and closed unmerged.
