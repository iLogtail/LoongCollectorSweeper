## Summary

Keep open. Current main still has the reported mismatch: `openclaw memory status` scans the sessions directory by counting every `.jsonl` file, while the session indexer uses the shared usage-counted transcript filter that excludes compaction checkpoint transcripts. The issue discussion correctly identifies checkpoint snapshots as recovery artifacts rather than memory-index input, but the status denominator has not been updated to match that contract.

## Best Possible Solution

Keep the issue open. Update `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles` uses the same usage-counted transcript contract as the indexer, preferably through a shared helper such as `isUsageCountedSessionTranscriptFileName` or `listSessionFilesForAgent`, then add a focused memory CLI status regression test with a main session JSONL plus a `*.checkpoint.<uuid>.jsonl` snapshot proving the denominator stays at the indexable file count.
