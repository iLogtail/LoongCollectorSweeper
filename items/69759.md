## Summary

Keep open. Current main still has the reported mismatch: `memory status` counts every `.jsonl` in the sessions directory for display totals, while the session indexing path uses the shared usage-counted transcript listing that excludes compaction checkpoint snapshots. The issue is recent, concrete, unprotected, and actionable in the bundled `memory-core` plugin rather than stale or external.

## Best Possible Solution

Keep #69759 open and fix `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles` uses the same usage-counted session transcript contract as indexing, either via a plugin-SDK/host export or `listSessionFilesForAgent` plus preserved missing-directory diagnostics. Add a focused `extensions/memory-core/src/cli.test.ts` regression with one normal session and one `.checkpoint.<uuid>.jsonl` file, asserting `Indexed` and `By source` report the checkpoint-excluded total.
