## Summary

Keep open. Current main still counts every `.jsonl` file in the sessions directory for `openclaw memory status` totals, while the session indexing path uses the shared session-file helper that excludes compaction checkpoint transcripts. The rendered `Indexed: files/total` and per-source totals therefore can still show the reported mismatch when a `*.checkpoint.<uuid>.jsonl` file exists.

## Best Possible Solution

Keep the issue open and fix `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles` uses the same session artifact classification as the indexing path, likely `isUsageCountedSessionTranscriptFileName` unless maintainers want a narrower primary-only count. Add focused CLI/status coverage with a normal session transcript plus a `*.checkpoint.<uuid>.jsonl` snapshot so both `Indexed` and `By source` totals stay consistent.
