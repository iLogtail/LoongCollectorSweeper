## Summary

Keep #69759 open. Current main still lets `openclaw memory status` count every `.jsonl` session file for the displayed denominator, while the indexing path uses the shared session-file classifier that excludes compaction checkpoint transcripts. That leaves the reported `Indexed: 3/4 files` style mismatch possible on current main.

## Best Possible Solution

Keep the issue open. The narrow fix should make `extensions/memory-core/src/cli.runtime.ts` use the same checkpoint-aware session transcript classifier as the indexing path, then add focused CLI regression coverage showing checkpoint `.jsonl` files do not inflate `memory status` totals.

## Evidence
