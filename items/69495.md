## Summary

Keep PR #69495 open. Current main still does not implement heartbeat-specific model fallback configuration: `heartbeat.model` is string-only in schema/types/docs and heartbeat runs only forward a single `heartbeatModelOverride`. The PR targets a real gap tied to #69434/#66232, but it still needs maintainer/author follow-up because review comments found that explicit `fallbacks: []` would be dropped instead of disabling the inherited fallback chain.

## Best Possible Solution

Keep the PR open and finish it rather than close it. The right path is to extend `heartbeat.model` to `AgentModelConfig`, thread primary plus fallbacks through heartbeat reply options into `FollowupRun.run.modelFallbacksOverride`, preserve an explicit empty fallback array, update generated schema/help/docs, and add regression tests for object form, no-fallback form, and `fallbacks: []` semantics.
