## Summary

Keep PR #69495 open. Current main still has string-only `agents.*.heartbeat.model` schema/types/docs and heartbeat runtime plumbing only forwards a single `heartbeatModelOverride`, so the PR addresses a real remaining gap. The PR should not be closed as implemented or duplicate; it still needs maintainer-quality follow-up for the review finding that explicit `fallbacks: []` must be preserved instead of inheriting agent-level fallbacks.

## Best Possible Solution

Keep #69495 open and ask for a focused revision: change heartbeat schema/types/generated config surfaces to reuse `AgentModelConfig`, resolve primary plus fallback values in `runHeartbeatOnce`, carry `heartbeatModelFallbacks` through `GetReplyOptions` and `FollowupRun` as an explicit per-run override including empty arrays, update status/pricing/docs/help surfaces, and add regression tests for object form, per-agent merge behavior, and `fallbacks: []` disabling inherited fallbacks.
