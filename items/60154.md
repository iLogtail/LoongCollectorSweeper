## Summary

Keep #60154 open. Current main still downloads Feishu inbound media through the account-aware Feishu API path, but persists every saved attachment through the shared managed media store subdir `inbound`; there is no accepted `channels.feishu.accounts.<id>.inboundDir` config field, and existing tests still assert the shared `inbound` subdir. The related #19330 context is adjacent broader media isolation work, not a shipped or exact canonical replacement for this Feishu multi-account request.

## Best Possible Solution

Keep the issue open and implement an account-aware Feishu inbound media isolation path. The likely fix should add a documented config contract or account-scoped managed media store behavior, update Feishu download/save calls to use it, preserve compatibility for existing `media://inbound/*` and absolute managed-media reads, and add tests for multi-account isolation plus docs/schema metadata updates.
