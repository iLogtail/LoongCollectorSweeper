## Summary

Keep open. Current main still routes Feishu inbound downloads into the shared media store `inbound` subdirectory, and the Feishu account schema/docs do not expose a per-account `inboundDir` or equivalent isolation knob.

## Evidence

- **Feishu account config has no inboundDir:** `FeishuAccountConfigSchema` is strict and lists account fields plus shared channel fields, but no `inboundDir` or per-account media directory setting.
  - file: [extensions/feishu/src/config-schema.ts:192](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/extensions/feishu/src/config-schema.ts#L192)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '1,260p'`

- **Inbound Feishu media still saves to shared inbound subdir:** `resolveFeishuMediaList` saves downloaded message resources via `core.channel.media.saveMediaBuffer(..., "inbound", ...)`; the account id is used for download routing but not for the storage subdirectory.
  - file: [extensions/feishu/src/bot-content.ts:443](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/extensions/feishu/src/bot-content.ts#L443)
