## Summary

Keep open. Current `main` does not implement the timestamp normalization, the reported cron/audit mismatch is plausible from the code path, and the submitted PR head is not safe to merge because its diff removes still-used task-flow exports and weakens existing error handling.

## Evidence

- **Current main still lacks the proposed clamp:** `createRunningTaskRun` on `main` creates the task record and immediately returns it through `ensureSingleTaskFlow`; there is no normalization to enforce `startedAt >= createdAt`.
  - file: [src/tasks/task-executor.ts:108](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/tasks/task-executor.ts#L108)
  - command: `nl -ba src/tasks/task-executor.ts | sed -n '100,116p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Registry can persist `startedAt` earlier than `createdAt`:** `createTaskRecord` stamps `createdAt` from `now` while preserving the caller-supplied `startedAt`, so a caller that captured `startedAt` earlier can produce an `inconsistent_timestamps` record.
  - file: [src/tasks/task-registry.ts:1455](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/tasks/task-registry.ts#L1455)
  - command: `nl -ba src/tasks/task-registry.ts | sed -n '1455,1476p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Cron callers pass pre-captured timestamps into task creation:** The cron service computes `startedAt` before calling `createRunningTaskRun` and forwards it as both `startedAt` and `lastEventAt`, which matches the race described in the PR body.
  - file: [src/cron/service/timer.ts:135](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/cron/service/timer.ts#L135)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '135,156p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Audit still flags this exact condition on main:** Task audit emits `inconsistent_timestamps` when `startedAt` is earlier than `createdAt`, and the user docs still describe `inconsistent_timestamps` as an active `tasks audit` finding.
  - file: [src/tasks/task-registry.audit.ts:49](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/tasks/task-registry.audit.ts#L49)

  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)

  - file: [src/tasks/task-executor.ts:463](https://github.com/openclaw/openclaw/blob/4663ff76ff6d87970ef84514618d897792ea5d59/src/tasks/task-executor.ts#L463)
