## Summary

Keep #62413 open. The provided discussion correctly points at PR #62433 as the intended fix, but that PR is closed/unmerged and the current `main` Dream Diary renderer still renders diary body lines as Lit text nodes rather than using the existing sanitized markdown pipeline. Current tests cover parsed/plain diary prose and structured backfill flattening, but not markdown rendering in the Dream Diary tab.

## Best Possible Solution

Keep the issue open and implement the #62433 direction on current `main`: render Dream Diary body content through the existing sanitized markdown pipeline, preserve the structured backfill cleanup behavior intentionally, add CSS for diary markdown/prose elements as needed, and add a focused `ui/src/ui/views/dreaming.test.ts` regression that proves `**bold**`, `*italic*`, and headings/lists render as markup rather than literal source.
