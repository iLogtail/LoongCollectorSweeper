## Summary

Keep open. Current main has restored duplicate heartbeat suppression, but the plugin-channel heartbeat fallback is still not implemented end-to-end: `resolveHeartbeatDeliveryTarget()` still returns `channel: "none", reason: "no-target"` when `resolveOutboundTarget()` cannot resolve the recovered plugin route. The related PR #46203 was closed unmerged, and the current code still matches the remaining failure mode.

## Best Possible Solution

Keep #43216 open, scoped to the remaining plugin-channel heartbeat fallback. The duplicate-suppression half can be treated as done, but `resolveHeartbeatDeliveryTarget()` should either bootstrap/resolve stored plugin ids before returning `no-target` or preserve the recovered stored route when primary normalization fails in the `target: "last"` path. Add focused regression coverage in `src/infra/outbound/targets.test.ts` for an empty active plugin registry and a stored plugin route, while keeping the existing duplicate suppression test.
