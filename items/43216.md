## Summary

Keep #43216 open. Current main implements the duplicate heartbeat suppression half, but the plugin-channel fallback gap still remains in the heartbeat path. The related PR #46203 is useful context, but the provided GitHub data says it was closed unmerged, so it does not supersede the issue.

## Evidence

- **Heartbeat still degrades to no-target after outbound resolution failure:** resolveHeartbeatDeliveryTarget() resolves the session route, then calls resolveOutboundTarget(); if that returns !ok, heartbeat delivery returns channel "none" with reason "no-target" instead of falling back to the already recovered session lastChannel/lastTo.

This is the same dependency that blocks heartbeat fallback when the plugin registry is empty or unbootstrapped.
