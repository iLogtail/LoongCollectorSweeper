## Summary

Keep #64732 open. Current main at a3c51f91 still routes unresolved primary CLI tokens through the plugin allowlist fallback, and the provided maintainer discussion explicitly made this issue the canonical tracker for help/tool/run/subagents-style command names being confused with plugin IDs. The closed PR #64859 appears unmerged in the provided context, and the current source does not contain the explicit command-classification layer requested in the maintainer comment.

## Best Possible Solution

Keep #64732 open and implement the maintainer-suggested command-classification layer before the plugin allowlist fallback: framework/core help and built-in command surfaces should bypass plugin allowlist guidance, manifest command aliases should point to the owning plugin ID, runtime slash aliases should say they are chat-only or suggest the owning CLI command, and unknown commands should fall through to normal Commander unknown-command handling. Add regression tests for restrictive plugins.allow covering `openclaw help`, non-plugin `tool`, core/built-in command names, a real excluded plugin command, and manifest/runtime alias cases.
