## Summary

Keep PR #58735 open. Current main still hardcodes the control-plane write limit at 3 requests per 60 seconds, exposes only the failed-auth gateway rate limiter in config/schema/docs, and does not implement `gateway.controlPlane.rateLimit`. The PR is a real in-repo gateway feature, not obsolete or duplicated, but it needs maintainer/product review and a clean rebase because the PR context shows generated/config-doc churn around this surface.

## Best Possible Solution

Keep the PR open, ask the contributor to rebase onto current main, preserve the existing bucket pruning and hard cap, wire `gateway.controlPlane.rateLimit` through types, zod schema, generated base schema/help/labels/docs, and keep tests for defaults, custom max/window, invalid values, startup reset, and dynamic error details. If maintainers decide configurable control-plane write limits conflict with the gateway trust model, close only with that explicit product decision.
