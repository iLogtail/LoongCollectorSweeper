## Summary

Keep open. Current main still has no default sandbox Docker `pidsLimit`: the resolver only uses agent/global config, and Docker creation only emits `--pids-limit` when that resolved value is a positive number. The open PR #44098 tracks the same fix direction, but it is not merged, and the broader WASM-runtime comment is an optional future backend idea rather than a fix for the current Docker sandbox default.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The best path is to add a documented `DEFAULT_SANDBOX_PIDS_LIMIT`, apply it in `resolveSandboxDockerConfig` while preserving explicit agent/global overrides, add regression coverage for the default and override behavior, and update the public sandbox/config security docs. #44098 is the existing open PR tracking that direction; the WASM backend idea can remain separate optional execution-backend work.
