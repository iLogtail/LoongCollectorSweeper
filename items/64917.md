## Summary

Keep open. Current main still implements `sessions_send` as an internal `deliver: false` target run followed by A2A reply/announce machinery, and current tests still codify nested webchat calls plus announce delivery. The related implemented reports cover preserving already-established external routes, but they do not solve this issue's fresh visible-webchat/internal-session boundary or provide a strict internal-only handoff mode.

## Best Possible Solution

Keep #64917 open and decide the product contract for `sessions_send`: either add an explicit strict internal-only mode that skips A2A ping-pong/announce and avoids seeding fresh targets with webchat delivery context, or document/rename the current announce-capable handoff behavior clearly. The likely implementation surface is `src/agents/tools/sessions-send-tool.ts`, `src/agents/tools/sessions-send-tool.a2a.ts`, `src/gateway/server-methods/agent.ts`, and the session delivery-context tests/docs.
