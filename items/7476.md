## Summary

Keep #7476 open. Current main still lacks outbound WhatsApp sticker sending: inbound stickers are recognized, but outbound WebP media follows the generic image/media path, no WhatsApp send option exposes `sticker` or `asSticker`, and `action=sticker` is still unsupported for WhatsApp. The related implementation PR #7502 was closed unmerged, and the latest maintainer discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract such as `action=sticker` or `asSticker`, preserve valid WebP sticker media before image optimization, route outbound sends through Baileys `{ sticker: buffer }`, and update WhatsApp action discovery, gateway/message option surfaces, docs, and focused send-api/media-loader tests.
