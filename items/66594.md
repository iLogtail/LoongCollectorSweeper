## Summary

Current main already solves the two user-visible problems from #66594: `chat.send` registers a chat-run link using the canonical session key before dispatch, and startup warmup falls back to async/provider-runtime model resolution before warning for `codex/gpt-5.4`. The PR’s discussion was useful, but the open branch is now stale against the shipped implementation and would duplicate part of the chat-run registration path.

## Best Possible Solution

Close #66594 as already implemented. Keep current main’s canonical pre-dispatch chat-run registration plus the startup async/provider-runtime fallback and regression coverage; do not merge the stale PR patch because its `onAgentRunStart` registration would now be redundant with the shipped path.
