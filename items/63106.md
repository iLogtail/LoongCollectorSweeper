## Summary

Current main still has the reported fast-return path for main-session `systemEvent` cron jobs, and I did not find the requested ghost-run warning, configurable threshold, or run-log annotation. The linked implementation PR #63111 was closed unmerged, so it does not supersede or implement this issue.

## Best Possible Solution

Keep this issue open. The most direct maintainer path is to implement a scoped ghost-run signal in the gateway cron finished-event path, or a stronger cron-service outcome that distinguishes queued heartbeat handoff from confirmed agent processing, then add regression coverage for `status: ok` fast main-session `systemEvent` runs and document the new threshold/config surface.
