## Summary

Keep open. Current `main` still marks `sessionTarget: "main"` + `wakeMode: "next-heartbeat"` cron runs as `ok` immediately after enqueueing the system event and requesting a heartbeat, without waiting for the heartbeat to run. The gateway run-log path persists that `ok` result and there is no ghost-run warning/flag on current `main`. The linked PR #63111 in the issue timeline also indicates active follow-up rather than an obsolete report.
