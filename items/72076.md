## Summary

Keep #72076 open. Current main does not implement the proposed cross-harness cua-driver computer plugin, #71842 is a separate open Codex app-server path rather than a shipped replacement, and the maintainer discussion explicitly says a simple plugin is not sufficient because OpenClaw is exploring a computer-use SDK with security and permissions wiring. The PR has useful prior art from #1946 and cua-driver, but review comments identify concrete blockers before it could merge.

## Best Possible Solution

Keep the PR open for maintainer product/API handling. The best path is to define the computer-use extension SDK and security/permissions contract first, then either rework this cua-driver work against that contract or move it to an external ClawHub/plugin package once the required OpenClaw API exists.
