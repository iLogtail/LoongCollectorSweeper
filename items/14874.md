## Summary

Keep #14874 open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still has no optional loader-time SIE warn/strict verification for skills. The existing ClawHub/plugin archive integrity, skill install scanning, and `before_install` hook are adjacent install-time protections, not a loader boundary that warns or rejects before arbitrary skills become prompt/runtime-visible. The related PR #14893 was closed unmerged, and the existing maintainer comment explicitly says to keep this issue open.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default loader-time verification policy, or a documented core loader verification hook/API, that can warn or reject before skills are exposed to prompts, slash-command discovery, snapshots, sandbox sync, or runtime use. Existing ClawHub install flows and `before_install` hooks should remain adjacent hardening, not treated as this feature.
