## Summary

Keep PR #62845 open. Current main still lacks the webchat-controller changes proposed by the PR: the client silent-token regex is case-sensitive, delta handling suppresses only complete `NO_REPLY`, and final/aborted fallback paths still persist any non-empty `chatStream` unless it is the exact uppercase token. Main does have gateway-level suppression for many live WebChat events, but that does not make this PR obsolete. The PR also includes the `/activate` alias work, and that alias remains unimplemented on main and separately tracked by #62827/#62841.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or split. The WebChat part should add case-insensitive silent-token suppression and prefix-fragment guards in `ui/src/ui/controllers/chat.ts` with regression tests for lowercase finals, prefix deltas, final fallback, abort fallback, and natural `No...` text. The `/activate` alias should either be separated into the existing #62827/#62841 track or kept only if maintainers accept the bundled scope.
