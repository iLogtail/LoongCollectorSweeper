## Summary

Current main still has the exact misleading prompt shape reported here: the system prompt heading is `Current Date & Time`, but the implementation emits only `Time zone: ...`. Existing tests intentionally lock in timezone-only prompt behavior for cache stability, and docs still describe the section with the same heading. Related closed items address startup/reset timestamp grounding and prompt-cache churn, but they do not resolve this narrower label/section-shape mismatch.

## Best Possible Solution

Keep this issue open and fix the narrow prompt-contract mismatch directly. The likely best path is to preserve cache-stable timezone-only behavior, rename the system prompt section to `Time Zone` or add an explicit `Use session_status for current date/time` line, then update `src/agents/system-prompt.test.ts`, `docs/concepts/system-prompt.md`, `docs/date-time.md`, and the stale wording in `docs/concepts/timezone.md` so implementation and docs agree.
