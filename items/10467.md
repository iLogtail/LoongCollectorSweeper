## Summary

Keep #10467 open. Current main still does not expose a `lane` parameter on `sessions_spawn`; native and ACP spawned child runs still dispatch through the hardcoded `subagent` lane, and subagent concurrency config remains a single scalar rather than per-lane limits. The issue is a concrete core/gateway feature request, and the latest issue discussion already includes a same-day review comment reaching the same conclusion.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through native and ACP spawn paths to the Gateway `agent` call, adds backward-compatible per-lane subagent concurrency configuration with fallback to `subagent`, and updates docs plus regression tests for default lane behavior, custom lanes, invalid lane names, and config reload.
