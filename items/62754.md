## Summary

Keep open. Current main has a shipped fix for the exact `Agent listener invoked outside active run` signature in the narrower exec stdout/stderr late-update path, plus timeout/fallback hardening, but I did not find source/test evidence that the broader model abort/timeout fallback race described by #62754 is fully fixed. Related #63220 was closed as a duplicate of #62754, so this remains the canonical tracker.

## Best Possible Solution

Keep #62754 open as the canonical tracker for the gateway/pi-agent-core abort-timeout fallback race. The best next step is a focused repro or regression test around embedded model timeout/abort with configured fallbacks and compaction pressure, then either add a targeted late-listener/processEvents guard in the embedded run lifecycle or document that the remaining behavior is fully covered by the exec-update fix if maintainers can prove that root cause.
