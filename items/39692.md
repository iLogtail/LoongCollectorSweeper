## Summary

Keep PR #39692 open. Current main has the base Control UI i18n framework and zh-CN locale loading, but the PR's core request is still not implemented across the targeted Control UI views: Agents, agent status/files, Logs, channel config/form helpers, channel cards, Debug, and exec approvals still contain user-visible hardcoded English. The PR needs a focused rebase against the generated-locale workflow rather than closure.

## Best Possible Solution

Keep the PR open and ask for a focused rebase. The best implementation is to move the remaining Control UI view strings into the English source locale map, regenerate zh-CN through the Control UI i18n pipeline, preserve the UX details already called out in review, and add targeted coverage or locale-key parity checks where feasible for the affected views.
