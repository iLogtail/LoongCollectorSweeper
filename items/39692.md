## Summary

Keep PR #39692 open. Current main has broad Control UI i18n and zh-CN locale support, but the PR’s central request is still not fully implemented: multiple targeted Control UI views still render hardcoded English strings. The branch is stale against the current generated-locale workflow and needs rebase/regeneration, but it is not obsolete or superseded.

## Best Possible Solution

Keep the PR open, but require a current-main rebase that moves source strings into ui/src/i18n/locales/en.ts, regenerates zh-CN through the Control UI i18n pipeline, wires the remaining hardcoded strings in the targeted views, preserves the Greptile-noted UX details, and adds focused regression coverage where practical. If the branch remains too stale, a maintainer could ask for a smaller replacement PR covering the same missing hardcoded-string surfaces.
