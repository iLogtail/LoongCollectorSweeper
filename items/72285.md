## Summary

Keep PR #72285 open. Current main still renders `/status` context from persisted `entry.totalTokens` without honoring `totalTokensFresh === false` or `compactionCheckpoints[].tokensAfter`, so the reported post-compaction stale-total bug is not implemented on main. The PR is a plausible targeted fix, but it needs rebase/adaptation because the status implementation now lives in `src/status/status-message.ts`, while `src/auto-reply/status.ts` is only a re-export wrapper.

## Best Possible Solution

Keep the PR open and ask for a small rebase onto current main: implement the stale-total fallback in `src/status/status-message.ts`, using a helper that prefers `resolveFreshSessionTotalTokens(entry)` and, when stale or missing, falls back to the latest finite nonnegative `compactionCheckpoints[].tokensAfter` by `createdAt`. Add the proposed regression in `src/auto-reply/status.test.ts` so `/status` shows the post-compaction context size while preserving the existing compaction count display.
