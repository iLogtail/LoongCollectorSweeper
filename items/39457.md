## Summary

Keep #39457 open. Current main still has no updater-owned post-update hook or durable service customization point that runs after the generated service refresh and before the gateway restart. The prior ClawSweeper comment remains accurate against current main SHA 64af2feda0334aecd3bb5101cb628b7040caa437.

## Best Possible Solution

Keep the issue open and implement a core updater-owned lifecycle hook or equivalent durable service customization mechanism. The likely fix belongs in `src/cli/update-cli/update-command.ts`: after successful core/plugin update and after any `gateway install --force` service refresh that can rewrite the plist, but before `runRestartScript` or `runDaemonRestart`. Add focused tests for a loaded macOS LaunchAgent update path proving a custom wrapper or hook-applied plist change is in place before relaunch, and document the supported hook/configuration surface.
