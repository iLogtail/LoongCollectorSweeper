## Summary

Keep #39457 open. Current main still refreshes an installed gateway service with `gateway install --force` and then immediately restarts it; there is no updater-owned post-update hook or durable LaunchAgent `ProgramArguments` customization point in the required slot. The existing ClawSweeper keep-open comment remains accurate against current main, and the only relevant update-command change since that review is timeout propagation for plugin installs, not lifecycle hook support.

## Best Possible Solution

Keep the issue open. The best product/code direction is a core updater-owned lifecycle hook or equivalent durable service customization mechanism in `src/cli/update-cli/update-command.ts`, after `refreshGatewayServiceEnv` and before `runRestartScript`/`runDaemonRestart`. Add focused tests for the loaded macOS LaunchAgent update path proving custom wrapper or hook-applied plist changes are in place before relaunch, and document the supported hook/configuration surface.
