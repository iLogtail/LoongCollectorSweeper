## Summary

Keep open. Current main still has structured failover logging and generic model-call hooks, but it does not expose the requested first-class `model_failover` or `model_failure_terminal` plugin hook surface. The related PR #70990 appears to be the right implementation track, but it is open and unmerged, so the issue remains a valid tracker.

## Best Possible Solution

Keep #70976 open as the canonical tracker and review/land PR #70990 or an equivalent core plugin API change. The right product direction is a small, typed, fire-and-forget hook surface with tests and docs, emitted from the existing failover-decision and terminal failure seams; this should not be moved to ClawHub until the core hook API exists.
