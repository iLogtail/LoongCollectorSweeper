## Summary

Keep open. Current `main` does not implement a `maxCostPerRun` heartbeat cap, and the PR discussion surfaces unresolved correctness issues around transcript-aware cost estimation and effective model resolution that need maintainer review rather than auto-close.

## Evidence

- **No heartbeat cap in current config surface:** `AgentDefaultsConfig.heartbeat` on `main` defines heartbeat knobs like `model`, `timeoutSeconds`, `lightContext`, `isolatedSession`, and `includeReasoning`, but there is no `maxCostPerRun` field.
  - file: [src/config/types.agent-defaults.ts:332](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/config/types.agent-defaults.ts#L332)
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **No pre-dispatch cost guard in heartbeat runner:** `runHeartbeatOnce()` builds `replyOpts` and immediately calls `getReplyFromConfig()`; there is no cost estimation branch or `cost-cap-exceeded` skip path on current `main`. A repo search for `maxCostPerRun|cost-cap-exceeded|estimateRunCost` under `src`/`docs` also returned no matches.
  - file: [src/infra/heartbeat-runner.ts:1061](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/infra/heartbeat-runner.ts#L1061)
  - command: `rg -n "maxCostPerRun|estimateRunCost|cost-cap-exceeded" src docs`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Main reply path resolves per-agent effective model:** Heartbeat replies eventually resolve the model through `resolveDefaultModel({ cfg, agentId })`, not a single global default. That matches the PR review concern that pricing a run before dispatch with only a config-wide default would be incorrect for agent-specific model overrides.
  - file: [src/auto-reply/reply/get-reply.ts:193](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/auto-reply/reply/get-reply.ts#L193)
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Per-agent override logic is real on main:** `resolveDefaultModelForAgent()` applies agent-specific model overrides before returning the configured default model ref. That is the dependency contract the proposed cap logic would need to respect.
  - file: [src/agents/model-selection.ts:199](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/model-selection.ts#L199)
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Transcript size materially affects heartbeat cost today:** Current heartbeat runner comments explicitly state that `isolatedSession` avoids sending the full conversation history and can save about `~100K tokens`, which supports the review feedback that a body-only estimate would undercount real run cost in non-isolated sessions.
  - file: [src/infra/heartbeat-runner.ts:797](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/infra/heartbeat-runner.ts#L797)
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **PR discussion already identified unresolved correctness gaps:** The provided PR review context includes a Codex P1 on excluding transcript history from the estimate, a Codex P2 on using the wrong default model source, and Greptile comments on the contradictory free-model test and missing `claude-3-*` pricing coverage. That makes this unsuitable for conservative auto-close.
