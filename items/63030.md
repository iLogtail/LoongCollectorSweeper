## Summary

Current `main` centralizes system-prompt cache-boundary assembly, removes volatile inbound identifiers from the system prompt, and routes announce wakeups as user-turn agent messages, which addresses the reported chat/heartbeat/announce cache-busting divergence.

## Evidence

- **Volatile suffix order is centralized behind a cache boundary:** `buildAgentSystemPrompt` now inserts `OPENCLAW_CACHE_BOUNDARY` and appends volatile sections in one fixed order: dynamic project context, extra/group context, provider suffix, heartbeat section, then runtime.
  - file: [src/agents/system-prompt.ts:942](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/system-prompt.ts#L942)

- **Later prompt additions are injected after the boundary instead of reshuffling the prefix:** `prependSystemPromptAdditionAfterCacheBoundary` normalizes and inserts per-turn system additions after the boundary, preserving the stable prefix shape across code paths.
  - file: [src/agents/system-prompt-cache-boundary.ts:22](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/system-prompt-cache-boundary.ts#L22)
