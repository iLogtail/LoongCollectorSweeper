## Summary

Current main still has the reported macOS companion-app node contract mismatch. The gateway `exec host=node` path invokes `system.run.prepare` before `system.run`, and gateway command policy rejects commands not advertised by the node. The macOS app node still advertises and dispatches `system.run` without `system.run.prepare`, while the headless node host does implement the prepare command. PR #43330 was a direct attempted fix but is closed unmerged, and the newer fallback PR #69725 is useful active follow-up work rather than a shipped fix.

## Best Possible Solution

Keep #37591 open as the primary tracker until maintainers either implement secure `system.run.prepare` support in the macOS companion-app node or land a reviewed gateway fallback such as #69725. The macOS-side fix should add the shared command enum case, advertise it from `MacNodeModeCoordinator`, implement a `MacNodeRuntime` prepare handler returning the canonical `systemRunPlan` shape, and add focused Swift tests for advertised commands, valid prepare payloads, invalid prepare payloads, and the prepare-to-run contract.
