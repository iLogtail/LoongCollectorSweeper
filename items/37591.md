## Summary

Current main still has the reported mismatch: the gateway `exec host=node` path requires `system.run.prepare`, while the macOS companion app node still advertises and dispatches `system.run` without a matching prepare command. The issue discussion is consistent with the code: `nodes.allowCommands` cannot add a command the node does not implement, PR #43330 attempted the missing macOS prepare support but was closed unmerged, and the later 2026-04-21 comment reports the same failure on recent stable/beta builds. Related issue #49031 tracks the same symptom, but it is a newer duplicate-style report and does not clearly supersede this original issue enough for conservative closure.
