## Summary

Keep open. Current main still enforces a 2 MiB local avatar limit but the user-facing CLI and agent configuration docs do not yet document that limit, so this PR is not obsolete. The PR likely needs a rebase/update because the relevant gateway identity docs now live in docs/gateway/config-agents.md rather than the older configuration-reference location.

## Best Possible Solution

Keep the PR or the underlying #65312 docs issue active. The best path is a small docs update on current main that documents the local avatar size limit in docs/cli/agents.md and docs/gateway/config-agents.md, with wording that applies only to local workspace avatar files and avoids claiming the limit applies to remote URLs or data URIs.
