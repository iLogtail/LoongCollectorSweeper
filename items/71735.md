## Summary

Keep open. Current main has plugin commands and forwards gatewayClientScopes to handlers, and it has scoped gateway RPC methods, but it does not implement declarative plugin command requiredScopes, trusted reserved command-root ownership, or continueAgent/resume semantics. The linked RFC PR #71731 is still an open docs/RFC artifact, and #71676 closed unmerged, so no merged implementation supersedes this issue.

## Best Possible Solution

Keep the issue open as a real SDK/API design and implementation gap. A focused follow-up should add an additive command contract to the plugin SDK, enforce requiredScopes in host command dispatch before handlers run, define trusted reserved-root ownership for bundled/trusted plugins, map continueAgent results into agent continuation on text and native channel paths, and add fixture tests for insufficient scope denial, read-only commands, default stop behavior, continuation, reserved-root rejection, and native/text parity.
