## Summary

Keep PR #70497 open. Current main still does not include the proposed Gemini `if`/`then`/`else` stripping: the Gemini unsupported-keyword set omits those conditionals, the cleaner only skips keys in that set and otherwise copies unknown keys through, and the Gemini provider hooks normalize/inspect via that same cleaner/set. The existing tests cover `not` stripping but not the #69768 conditional-keyword regression. The provided PR context also supports keeping it: #70497 is a reopening of #69964 after the fork became private, and the bot review describes the change as a minimal, targeted regression fix.

## Best Possible Solution

Keep the PR open and integrate or rebase the narrow two-file fix: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` and keep the targeted regression test. If maintainers want a broader provider-tool cleanup instead, land an equivalent current-main patch that preserves the same Gemini behavior proof.
