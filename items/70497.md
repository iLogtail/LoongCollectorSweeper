## Summary

Keep open. Current main has Gemini tool-schema cleanup, but it does not strip JSON Schema conditional keywords `if`, `then`, or `else`; those keys are absent from the Gemini unsupported keyword set and therefore survive the existing stripping loop. The related #69964 is closed unmerged and points to the same head SHA as #70497, so this reopened PR is not superseded by an already-landed fix.

## Best Possible Solution

Keep PR #70497 open and have a maintainer either land it after normal validation or apply the same narrow fix on main: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` and keep the targeted regression test for conditional schemas.
