## Summary

Keep PR #70497 open. Current main does not strip Gemini JSON Schema conditional keywords: `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` omits `if`, `then`, and `else`, and the cleaning loop only drops keys present in that set. The existing tests cover stripping `not` and other Gemini cleanup, but there is no regression coverage for conditional keywords. The PR remains a narrow, relevant fix for the linked #69768/#69964 problem and should be rebased or merged rather than closed.

## Best Possible Solution

Keep the PR open and land the narrow behavior after a rebase: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, retain the targeted regression test in `src/agents/schema/clean-for-gemini.test.ts`, and run the relevant agent/schema changed-lane checks before merge.
