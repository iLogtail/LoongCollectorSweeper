## Summary

Keep #69271 open. Current main still only exposes session deletion through checkbox selection plus the bulk delete bar, and there is no per-row action/dropdown or first-class archive session API/view. The existing delete path archives transcript files as a side effect after removing the active session entry, which does not satisfy the requested soft-archive/filter/recovery behavior.

## Best Possible Solution

Add a row-level Actions column in `ui/src/ui/views/sessions.ts` and wire a single-key delete action through the existing confirmed `deleteSessionsAndRefresh(state, [key])` path. Treat soft archive as a separate product/API change unless maintainers want it in the same issue: define `sessions.archive`, storage/list/filter/recovery semantics, Control UI filters, docs, and gateway/UI tests.
