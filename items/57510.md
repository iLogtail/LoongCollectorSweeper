## Summary

Keep #57510 open. Current main still lacks the requested blocking `pre_route` plugin hook, and the existing `message_received`, `inbound_claim`, `before_dispatch`, and `reply_dispatch` seams do not provide same-turn session/account/agent route override before dispatch routing. The PR has substantive review discussion and author follow-up, so it remains useful work rather than conservative cleanup.

## Best Possible Solution

Keep the PR open. The best path is to rebase it onto current main, implement `pre_route` in the current plugin hook contract/runner files, run it in `dispatch-from-config` before route/session/agent decisions that need to be overridden, thread `sessionKey`, `accountId`, and `agentId` consistently through ACP and normal reply execution, and keep focused docs plus dispatch/hook-runner regression tests.
