## Summary

Current main already preserves durable run history for one-shot `schedule.kind = "at"` jobs through per-job JSONL run logs and exposes that history through `cron.runs` even after delete-after-run cleanup. The gateway path appends a finished record for every completed job, the one-shot delete path emits `finished` before removal, and the checked-in gateway test covers an auto-run `at` job whose history remains queryable by job id.
