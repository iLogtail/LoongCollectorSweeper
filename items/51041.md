## Summary

Current main does not expose Discord slash interaction responder controls to `api.registerCommand` handlers. The plugin command context remains channel-generic and returns a single `ReplyPayload`; the Discord native command adapter invokes the handler first and only then replies or follows up on Discord's interaction. Existing Discord responder helpers are for interactive component handlers, not native plugin commands, so this is a real missing plugin API/design request rather than a ClawHub-only item.
