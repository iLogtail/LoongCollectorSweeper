## Summary

Keep #61213 open. Current main has useful manual session labels and chat-picker display-name handling, but it does not implement the requested automatic friendly label priority of `binding.comment -> agent.name -> technical identifier` for Feishu `ou_xxx`/long session keys. Unlabelled sessions still fall back to parsed technical session-key fragments, and the Sessions table still renders the raw key as the primary cell.

## Best Possible Solution

Add a shared session display-label resolver for Control UI/Gateway session rows that can match configured route/ACP bindings and prefer `binding.comment`, then agent identity/name, then the existing technical key fallback. Wire it into the chat session selector and Sessions table while keeping the raw key available in titles/details for diagnostics, and add focused tests for Feishu `ou_xxx` direct sessions and long ACP/session-key cases.
