## Summary

Current main does not yet provide the requested restart-safe handling for approval-pending exec tool results. Pending approval state is still process-local, approval-pending results still carry the approval id into persisted tool output, and replay/restart recovery does not invalidate or re-register those stale ids. The related #65437 is another approval-routing gap, not a superseding fix.

## Best Possible Solution

Keep this open and fix it in the exec approval/session replay boundary. The conservative implementation is to detect persisted exec tool results with status approval-pending, or their approval-pending text, during restart recovery or replay sanitization and convert them to a terminal error/denial that tells the agent to re-issue the exec call with a fresh approval id. Re-registering from persisted details should only be used if the full original request binding can be reconstructed safely. Add a regression test covering approval-pending exec result, gateway restart, and resumed session behavior.
