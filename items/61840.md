## Summary

Keep #61840 open. Current main still routes Feishu native audio/TTS voice-note replies through `sendMediaFeishu`, and that caller still invokes `uploadFileFeishu` without passing a `duration` value for `audio`/`media` uploads.

## Best Possible Solution

Patch the Feishu plugin media path: after native Ogg/Opus preparation or audioAsVoice transcoding, compute a reliable media duration, pass it as `duration` to `uploadFileFeishu` for `fileType: "opus"` and likely `"mp4"`, and add focused regression tests asserting the upload payload includes duration for native `.ogg`/`.opus`, transcoded voice-note TTS, and video where applicable. Keep the issue open until that code and tests land on main.
