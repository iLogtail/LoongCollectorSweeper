## Summary

Keep open. Current main partially documents Media Understanding provider auth in the broader gateway config docs, but the issue’s linked Media Understanding page still does not clearly state in the Config overview or Model entries section that provider credentials come from auth profiles, env vars, or `models.providers.<provider>.apiKey`, and the directly related PR #51089 was closed unmerged.
