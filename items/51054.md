## Summary

Current main does not fully resolve #51054. The runtime behavior is clear: `tools.media.*.models[]` provider entries do not have an inline `apiKey` field, and media provider execution resolves credentials through the shared model auth path. There is a useful partial doc note in `docs/gateway/config-tools.md`, but the reported user-facing page, `docs/nodes/media-understanding.md`, still shows provider examples without explaining where `apiKey` comes from or that users should configure auth profiles, env vars, or `models.providers.<provider>.apiKey`. The related PR #51089 explicitly targeted this doc gap but is closed and unmerged per the provided GitHub context, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep #51054 open and land a narrow docs fix in `docs/nodes/media-understanding.md`: in `Config overview` or `Model entries`, state that provider media entries do not accept inline `apiKey`; credentials resolve through `profile`/`preferredProfile` auth profiles, provider env vars, then `models.providers.<provider>.apiKey`; include a short `models.providers` example or link to the public tools/custom-provider config page. PR #51089 can be used as prior art, but it is closed unmerged and should not be treated as shipped.
