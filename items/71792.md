## Summary

Keep #71792 open. Current main still creates new sessions for daily/idle stale rollover and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` hook that bundled `session-memory` listens to. The PR remains a focused fix for a live gap, and the related context shows the earlier same-area PR #43533 was closed unmerged while #11960 remains open.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for auto daily/idle rollover memory persistence. The best product path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` values like `auto:daily` and `auto:idle`, pass the resolved agent workspace, and keep/land regression tests for daily, idle, and fresh-session no-op behavior unless maintainers choose a broader lifecycle-hook redesign.
