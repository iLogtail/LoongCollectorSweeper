## Summary

Keep #71792 open. Current main at 631552c still rolls stale daily/idle sessions into a new session and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` event that bundled `session-memory` listens for. The PR is active, focused, and the supplied discussion shows the earlier missing `auto:idle` coverage was addressed; related #11960 remains open and earlier PR #43533 was closed unmerged, so there is no strong close reason.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for auto daily/idle rollover memory persistence. The likely best path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` distinguishing `auto:daily` and `auto:idle`, while preserving the existing `session-memory` command-event contract unless maintainers choose a broader lifecycle-hook redesign.
