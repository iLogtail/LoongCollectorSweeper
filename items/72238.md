## Summary

Keep open. Current main at e672b614 still has browser runtime value imports from `playwright-core`, and there is no browser-local Playwright runtime shim on main. The PR is a focused active fix candidate for linked bug #72168, whose report shows packaged installs on v2026.4.24 still fail with `ERR_MODULE_NOT_FOUND` after `doctor --fix`.

## Best Possible Solution

Keep #72238 open as the active fix candidate for #72168. The best path is to validate the shim in a packaged install or targeted import test, keep the production change limited to runtime value imports (`chromium` and `devices`) while leaving type-only imports untouched, then land this PR or a maintainer-authored equivalent.
