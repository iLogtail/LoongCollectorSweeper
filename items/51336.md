## Summary

Keep open. Current main still drops provider/model context at the user-facing transient error formatter: overloaded and generic rate-limit errors can still become provider-agnostic copy, even though the runner now has active failed-attempt context available. The issue is also the canonical tracker for newer duplicate reports; open implementation PRs are not merged, so closing #51336 now would be premature.

## Best Possible Solution

Keep #51336 open as the canonical tracker. The best fix is to extend the transient error formatting path to accept sanitized provider/model context from the active failed attempt, use it for overloaded and generic rate-limit messages, update the auto-reply mid-turn path as well, and replace the generic exact-string tests with provider-aware assertions. Then close #51336 from the merged PR.
