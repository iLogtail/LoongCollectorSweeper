## Summary

Keep open. Current `main` already routes Telegram traffic through standard `HTTP_PROXY`/`HTTPS_PROXY` env vars, but it does not implement the Telegram-specific `OPENCLAW_TELEGRAM_PROXY` override proposed by PR #41795.

## Evidence

- **No Telegram-specific env var support on main:** `extensions/telegram/src/proxy.ts` only re-exports `getProxyUrlFromFetch` and `makeProxyFetch`; there is no `OPENCLAW_TELEGRAM_PROXY` resolver or equivalent helper on current `main`. A repo-wide search for `OPENCLAW_TELEGRAM_PROXY` returned no matches.
