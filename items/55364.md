## Summary

Keep open. Current `main` still forces WhatsApp outbound `audio/*` media through the PTT voice-note path, and the tests/docs explicitly encode that behavior, so the reported MP3/PTT bug class is not implemented on `main` and is still plausible for `openclaw message send --channel whatsapp --media ...`.
