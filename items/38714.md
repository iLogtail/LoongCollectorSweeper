## Summary

Keep #38714 open. Current main still receives Discord reaction add/remove events, but exposes them only as routed session system events, not as internal Hooks events or typed plugin hook events. The related #52658 and #60805 reports are closed, but the provided discussion/context does not identify a shipped implementation that supersedes this issue.

## Best Possible Solution

Keep this issue open until maintainers choose and document the contract. The likely implementation should add either dedicated typed/internal reaction hook events or structured synthetic message_received delivery for Discord reaction add/remove events, with payload metadata, Discord listener wiring, plugin/internal hook tests, and docs updates. Because current plugin surfaces cannot observe these reactions, this is a core/plugin-API gap rather than ClawHub-only work.
