## Summary

Keep open. Current main has adjacent configured-model inference, but it does not implement this PR's central behavior for persisted gateway/session model overrides. `resolveSessionModelRef()` still returns the persisted selected model directly, and the persisted override resolver has no config-aware unique-provider inference for a bare `modelOverride` like `gpt-5.4`. The submitted PR also has unresolved review findings around wrapper providers, stale runtime providers, and non-default-provider setups, so it should not be closed as implemented or landed unchanged.

## Best Possible Solution

Keep the item open or supersede it only with a clearly canonical follow-up. The best fix is a focused maintainer-reviewed change that makes persisted/session model resolution infer a provider for bare `modelOverride` only when configured models make it unique, while preserving explicit `providerOverride`, preserving wrapper-provider slash model IDs, covering existing sessions with `modelProvider`, and adding regression tests for the gateway/session-status surfaces. Coordinate with #49166 if that PR becomes the canonical narrower fix for the remaining session-status behavior.
