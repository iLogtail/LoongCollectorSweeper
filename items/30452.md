## Summary

Keep #30452 open. Current main implements the compaction summarization model override, but the issue's remaining `memoryFlush.model` and `memoryFlush.resetAfterFlush` requests are still not present in config types, strict schema, docs, memory-flush planning, or runtime routing. The related discussion also explicitly treated the earlier compaction thinking/fallback work as partial and left flush-then-reset as a separate open question.

## Best Possible Solution

Keep the issue open, or split it into focused maintainer-approved follow-ups. The concrete path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, zod schema, generated help/docs/UI surfaces, and tests; route the silent memory-flush turn through the configured model override; then define and test flush-then-reset behavior across session store updates, transcript/session-file handling, queue refresh, active run cleanup, subagent/ACP state, and channel-visible behavior.
