## Summary

Keep #41501 open. The Langfuse-in-core portion is largely superseded by OpenClaw's current plugin/diagnostics direction, but the PR also contains a concrete core delivery-confirmation fix that is not fully covered on current main for the subagent direct-announce/gateway-agent path. Current main improved isolated cron direct delivery, but gateway agent delivery still does not return explicit delivery metadata, and subagent announce can still mark a successful gateway agent call as delivered without confirming the external send.

## Best Possible Solution

Keep the item open, but maintainer action should be to request a split/replacement: move Langfuse to a ClawHub/plugin implementation built on diagnostics/plugin hooks, and extract the delivery-confirmation work into a focused core PR that propagates outbound delivery results from src/agents/command/delivery.ts through gateway agent responses and verifies src/agents/subagent-announce-delivery.ts does not mark external completion delivery successful after best-effort outbound failure with a visible model payload.
