## Summary

Keep open. Current `main` and the latest release still overwrite `identity/device.json` in place with `fs.writeFileSync`, so this is not implemented on `main`. The PR addresses a real durability gap for device key material, but the discussion also shows unresolved review scope: the unsafe temp-file mode was fixed in a follow-up commit, while the `ensureDir` `EEXIST` catch remains questionable, and `main` already has a stronger shared atomic JSON-write pattern that may be a better final shape.
