## Summary

Keep open. Current `main` does not implement the proposed `models.providers.*.preScript` surface, and the PR discussion still shows unresolved regression risk against plugin-owned synthetic auth that current `main` actively uses. OpenClaw already has narrower dynamic-auth mechanisms (`SecretRef` + `secrets.providers.*.source: "exec"`, plus provider `resolveSyntheticAuth` hooks), but those are not the same feature and do not fully replace the PR’s per-provider pre-script/baseUrl flow.
