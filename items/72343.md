## Summary

Keep open. Current main still routes bare `openclaw` directly to Crestodian and does not inspect config state or call classic onboarding for missing/empty configs. The PR is a focused change for a real remaining behavior gap, with one review concern about weakened e2e coverage that should be addressed before merge.

## Best Possible Solution

Keep the PR open and review it as a plausible product fix. The best path is to preserve the proposed config-state split in `src/cli/run-main.ts`: missing or valid empty config should start classic `setupWizardCommand`, authored config and invalid config should keep routing to Crestodian. Before merge, require focused tests for missing, valid empty, authored, and invalid config snapshots, plus a meaningful first-run smoke assertion that exercises the fresh bare-root onboarding branch rather than a pure argv-pattern check.
