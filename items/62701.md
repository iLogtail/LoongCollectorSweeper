## Summary

Keep #62701 open. Current main still does not expose `precedingText` or `messageId` on the public `before_tool_call` event, and the tool-definition path still ignores the fifth execution argument that would carry the session/message context needed to derive those fields. The PR is a plausible narrow additive plugin contract change related to closed #12617 and broader open #13225/#58079, but it is not implemented on main and is not clearly superseded by an exact canonical item.

## Best Possible Solution

Keep the PR open and have a maintainer decide whether to land this narrow additive hook payload or fold it into the broader context/routing work. A current-main-ready patch should update `src/plugins/hook-types.ts`, derive `precedingText`/`messageId` from the assistant transcript entry when available, preserve `extensionContext` through `toToolDefinitions`, `wrapToolWithBeforeToolCallHook`, and `wrapToolWithAbortSignal`, and add focused tests plus public docs updates.
