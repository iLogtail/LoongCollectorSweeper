## Summary

Keep #42539 open. Current main still sends Telegram voice/TTS media with the reply text as a caption when the text fits within Telegram's 1024-character caption limit, and only separates text on overflow or sendVoice caption errors. I found no `messages.tts.delivery` or equivalent configurable delivery mode in the TTS config type, generated schema/docs, or Telegram config hints. The issue's proposed UX remains a real, focused Telegram plugin feature request rather than a duplicate or already-shipped behavior.

## Best Possible Solution

Keep the issue open and implement this in the bundled Telegram plugin/TTS delivery path. A conservative fix would preserve the current bundled-caption default, add a typed `bundled | separate` delivery option in the appropriate TTS/channel override config surface, update generated schema/docs/UI hints, and cover both `sendMessageTelegram` and `deliverMediaReply` with tests proving text-first plus voice-second delivery for Telegram TTS while retaining the existing overflow/error fallbacks.
