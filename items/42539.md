## Summary

Keep open. Current main still lacks a configurable Telegram TTS delivery mode that always sends readable text separately from the voice note; it only separates text on specific fallback paths such as caption-too-long errors or block-streaming TTS-only synthesis.

## Evidence

- **TTS config has no delivery option:** The public TTS config type exposes auto, enabled, mode, provider, summaryModel, modelOverrides, providers, prefsPath, maxTextLength, and timeoutMs only; there is no delivery/bundled/separate field for the requested setting.

  - command: `nl -ba src/config/types.tts.ts | sed -n '1,120p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Runtime schema would reject proposed config:** TtsConfigSchema is strict and likewise does not define messages.tts.delivery, so the proposed config shape is not currently accepted by main.
  - file: [src/config/zod-schema.core.ts:500](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/config/zod-schema.core.ts#L500)
