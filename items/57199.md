## Summary

Keep open. Current main still does not implement the PR’s requested CLI and wake-routing behavior: `openclaw system event` accepts only text/mode/json, the wake protocol schema names only mode/text, and the gateway/cron wake path drops agent/session targeting before enqueueing or requesting a heartbeat. The PR is still useful, but it needs rework for the review feedback: session-key-only targeting should use the same resolved wake target for both the system-event enqueue and heartbeat wake so they cannot split across default and derived agents. Related PR #55775 is narrower agent-id-only work and does not supersede this session-key request.

## Best Possible Solution

Keep the PR open and rework it against current main: add `--agent-id` and `--session-key` to `src/cli/system-cli.ts`, add typed optional wake params to the gateway protocol without breaking forward-compatible metadata, thread both through `src/gateway/server-methods/cron.ts`, `CronService.wake()`, `ops.wakeNow()`, and `timer.wake()`, and update `buildGatewayCronService` so system-event enqueue and heartbeat wake share one resolved target. Add regression tests for agent-id-only, session-key-only, both fields, blank normalization, and the session-key-only mismatch called out in review.
