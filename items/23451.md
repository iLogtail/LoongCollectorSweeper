## Summary

Keep #23451 open. Current main has related exec/ACP/plugin approval primitives, including plugin `before_tool_call.requireApproval`, but it still does not implement the requested built-in `tools.confirmation` risk-tier policy or same-response assistant-text-aware pause before the first tool executes.

## Best Possible Solution

Keep #23451 open for maintainer/product work at the agent tool-execution boundary. The likely path is a documented `tools.confirmation` schema/help/docs surface, deterministic tool risk classification, access to same-response assistant text before first tool execution, user-visible flush before holding tools, explicit approval or timeout handling, and clear execute/skip transcript semantics while preserving plugin `before_tool_call.requireApproval` as the lower-level custom policy primitive.
