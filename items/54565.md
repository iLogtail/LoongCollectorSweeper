## Summary

Keep open. Current main still only resolves WhatsApp inbound debounce from global/channel/account config, not from `channels.whatsapp.groups.<jid>`. The group schema is strict and does not accept `debounceMs`; the runtime passes one listener-wide debounce value and currently opts media messages out of debouncing, so the reported image/image/text group pattern remains unhandled.

## Best Possible Solution

Keep the issue open. The best implementation path is to add documented WhatsApp `groups.<jid>.debounceMs` support in the typed config and Zod schema, resolve the effective debounce per inbound group/conversation using the existing debouncer's per-item timing hook, and add regression tests for both text-only batching and the reported image/image/text burst. Broader Telegram/Discord/Slack support should follow as channel-specific schema/runtime work or a documented common plugin contract.

- Cross-channel parity should be designed carefully because Telegram, Discord, and Slack use different group/channel nesting shapes.
