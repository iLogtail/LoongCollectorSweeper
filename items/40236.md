## Summary

Keep open. Current `main` overlaps with the request only partially: per-job timeout is implemented, and there are some cleanup/event surfaces, but OpenClaw still does not expose the proposed per-job lifecycle-hook config or per-job cleanup policies by run-count/time/status. That is not strong enough for an auto-close, and the existing plugin/hook seams do not clearly prove this belongs outside core either.

## Evidence

- **Per-job timeout exists on current main:** Cron agent-turn payloads already accept `timeoutSeconds`, and the scheduler resolves that into a per-job timeout policy.
  - file: [src/cron/types.ts:123](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/cron/types.ts#L123)

  - file: [src/cron/service/timer.ts:79](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/cron/service/timer.ts#L79)
