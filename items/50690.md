## Summary

Keep open. Current main has useful partial fixes: a generic `message_sending` pre-delivery hook, phase-aware commentary suppression, and an isolated-cron stale-ack retry/suppression path. It does not yet implement the requested cross-surface runtime policy layer with ack classification, execution-state gating, repeated-ack state, progress-debt enforcement, or durable execution-state validation.

## Best Possible Solution

Keep #50690 open and narrow the remaining work to a core runtime/API design: add or extend a pre-send policy surface with durable run/session execution state, typed message classification, per-session ack/progress bookkeeping, progress-debt handling, observability counters, and cross-surface regression tests. Treat current commentary suppression and isolated-cron interim retry as shipped partial mitigations, not completion of the issue.
