## Summary

Keep #64597 open. Current main still has the overflow fallback behavior this PR targets: after overflow compaction fails, the run loop only checks whether any oversized or aggregate tool-result output exists, then attempts truncation, without comparing the reducible tool-output budget to the observed/current overflow. The PR also includes a still-missing auto-reply behavior change for embedded context-overflow payloads. The PR is contributor-authored, has no protected labels, but is recent and focused with author follow-up, so it is not conservative cleanup material.

## Best Possible Solution

Keep the PR open and review it as a focused agents overflow-recovery fix. The best path is to land a version that gates fallback truncation with attempt-local overflow/reduction estimates, preserves recoverable oversized-tool-result retries, avoids stale `lastRunPromptUsage` decisions, narrows auto-reply session resets to real compaction failures, and keeps the added regression coverage for the overflow loop and auto-reply behavior.
