## Summary

Keep PR #39489 open. Current main still lacks Feishu runtime connection/activity status reporting, so the PR is not obsolete or implemented. It also is not a clean duplicate/superseded close: related Feishu health PRs are still open in the provided context, and no maintainer-selected canonical replacement is evident. The PR does need revision before merge because the current gateway health contract uses `lastTransportActivityAt` for stale-socket checks, while this PR reports only `connected`, `mode`, `lastEventAt`, and `lastError`.

## Best Possible Solution

Keep #39489 open and request a rebase/revision against current main. The Feishu plugin should pass `ctx.setStatus` from `extensions/feishu/src/channel.ts` through `monitorFeishuProvider`, `monitorSingleAccount`, event handlers, and websocket/webhook transports; report `connected`, `mode`, `lastConnectedAt`, `lastEventAt`, `lastTransportActivityAt`, and `lastError` at validated lifecycle/activity points; preserve the already-discussed webhook guard-ordering fix; and add focused Feishu tests for websocket and webhook status behavior. Maintainers should also pick the canonical branch among the overlapping Feishu health PRs.
