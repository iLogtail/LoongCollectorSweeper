## Summary

Keep PR #55688 open. The reported bug is still present on current main: the shared Volc coding catalog still marks the five requested coding models as text-only, and both volcengine-plan and byteplus-plan consume that shared catalog. The PR is useful but stale after the catalog moved into the plugin SDK and should be rebased/reworked rather than closed.

## Best Possible Solution

Rework the PR against current main by updating src/plugin-sdk/volc-model-catalog-shared.ts so only the five shared coding model entries advertise text+image, then add focused regression coverage for both bundled provider surfaces, preferably in the Volcengine and BytePlus plugin/provider catalog tests or an implicit-provider test that exercises volcengine-plan and byteplus-plan. Update the public provider docs/changelog if the capability is intentionally user-facing, and leave doubao-seed-code-preview-251028 unchanged unless maintainers confirm it should also be vision-capable.
