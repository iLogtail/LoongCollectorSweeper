## Summary

Keep PR #39772 open. Current main still builds one Telegram draft-stream reply parameter object and reuses it for every message-preview send, so the proposed reply_to gating is not implemented. The PR is a focused Telegram bug fix with relevant tests, but its current approach still needs maintainer review because the discussion identifies substantive failure-recovery and replyToMode="all" contract concerns.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. A robust fix should prevent deleted-message preview artifacts while preserving replyToMode="all" where configured, handle failed first preview sends/materialize recovery without losing the first successful reply context, and add regression coverage for forceNewMessage, thread params, first-send failure recovery, and all-mode behavior.
