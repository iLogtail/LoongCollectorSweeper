## Summary

Keep open. Current `main` still resolves `--agent` to the agent main session before honoring `--to` or `--session-id`, so the reported session-collision behavior remains plausible on `a53fea39050beba21ac5468d876c37323cfdd2e6` rather than being already fixed.

## Evidence

  - file: [src/commands/agent-via-gateway.ts:113](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/commands/agent-via-gateway.ts#L113)
