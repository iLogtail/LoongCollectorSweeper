## Summary

Keep #41483 open. Current main still lets `--agent` preselect `agent:<id>:main` before `--to`/`--channel` can create a recipient-specific session: the CLI gateway path does not pass `channel` into session-key resolution, and both the CLI resolver and gateway handler derive an explicit agent main session when no `--session-id` is present. Existing coverage addresses `--agent` + `--session-id`, not the reported `--agent` + `--channel` + `--to` per-recipient routing.

## Best Possible Solution

Keep the issue open and fix the `openclaw agent` request path so an explicit `--channel` + `--to` target can resolve through the channel-aware routing/session-key machinery, honoring `session.dmScope` and the selected `--agent`. Add regression coverage for both gateway and embedded fallback paths, including two different `--to` values under the same custom channel and agent.
