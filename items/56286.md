## Summary

Keep #56286 open. Current main still resolves Telegram exec approval callbacks by clearing inline buttons / actions, not by deleting the approval message, so the reporter's observable issue remains. The provided timeline also shows an open related PR (#56319) that explicitly proposes deleting the Telegram approval message after resolution, which indicates the fix has not been shipped on main yet.

## Best Possible Solution

Keep #56286 open until Telegram approval finalization actually removes resolved approval messages or maintainers explicitly decide that clear-buttons-only is the intended UX. The likely fix belongs in `extensions/telegram/src/approval-handler.runtime.ts` plus the direct callback path in `extensions/telegram/src/bot-handlers.runtime.ts`, with regression coverage in the Telegram approval/bot tests. PR #56319 is the existing related implementation attempt and should be reviewed, updated, or superseded rather than closing this issue now.
