## Summary

Keep #38939 open. Current main still lacks Feishu group mention arbitration across sibling bot accounts, so the duplicate-reply/drop-risk problem is not obsolete. The PR is a focused attempt at the right bundled Feishu plugin surface, but the existing review comments identify fixable correctness issues that should be revised before merge.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should extract all structured mentioned bot open_ids before per-account dispatch, derive the coordinator from Feishu default-account selection, arbitrate specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add regression tests for those cases.
