## Summary

Keep #70266 open. Current main implements assistant/avatar identity for gateway and Control UI surfaces, but the native macOS Talk Mode overlay still renders `TalkOrbView` directly and only receives phase, level, accent color, and paused state. The Talk config path currently forwards `ui.seamColor` to the macOS Talk runtime, not `ui.assistant.avatar`, and the existing macOS overlay smoke test only covers present/dismiss behavior. The reporter's follow-up correctly distinguishes this from the already-closed web/control UI identity work in #22199.

## Best Possible Solution

Keep the issue open and implement the feature in the native macOS Talk Mode path. The likely product/code direction is to have Talk Mode resolve the effective assistant identity, either by extending `talk.config` with a safe avatar payload or by calling `agent.identity.get`, store the resolved avatar state on the Talk overlay/controller model, render a circular cropped image inside `TalkOrbView` while preserving the existing rings/arcs/paused fallback, and add focused parser/controller/view smoke coverage for avatar-present and avatar-missing fallback cases.
