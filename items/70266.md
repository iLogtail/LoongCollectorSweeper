## Summary

Keep #70266 open. Current main still renders the macOS Talk Mode overlay as the generic SwiftUI orb and only carries the gateway UI accent color into that native surface. The related assistant avatar work is present for Control UI/chat identity, but the issue author explicitly clarified that this request is for the native macOS Talk Mode voice overlay, and I did not find current macOS code that fetches or renders the resolved assistant avatar there.

## Best Possible Solution

Keep the issue open and implement it in the macOS app: fetch the resolved assistant identity/avatar for the active agent when Talk Mode starts, either by adding a native `agent.identity.get` call or by extending the `talk.config` payload with resolved avatar metadata. Store the resolved avatar in Talk/App state, render it as a circular image inside `TalkOrbView`, keep the existing rings/pulse/arcs around it, fall back to the current orb when unresolved, and add focused Swift tests for config parsing/state plus a Talk overlay smoke or view-level regression.
