## Summary

Keep #70266 open. Current main still renders the macOS Talk Mode overlay as the generic SwiftUI orb and only passes phase, level, accent color, and paused state into `TalkOrbView`. Existing avatar resolution is present for Control UI `/avatar` and web chat identity surfaces, but the native macOS Talk Mode path has no avatar field in the overlay model, no `talk.config` avatar transport, and no parser/rendering hook for `ui.assistant.avatar`. The reporter's follow-up correctly distinguishes this from the closed web/control UI identity work in #22199.

## Best Possible Solution

Keep the issue open and implement the feature in the macOS Talk Mode path: expose the resolved assistant avatar for the active agent through a suitable gateway/native contract, parse and cache it in the macOS app, render it as a circular image inside `TalkOverlayView` while preserving the existing rings, pulse, and thinking arcs, fall back to the current orb when unresolved, and add macOS parser/overlay coverage plus protocol tests for the new field.
