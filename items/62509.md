## Summary

Keep #62509 open. Current main still has the reported installer finalization probes as unbounded shell command substitutions: `npm prefix -g` through `npm_global_bin_dir`, and `"$claw" daemon status --json` through `is_gateway_daemon_loaded`. The daemon-status CLI has an RPC timeout, but the installer does not apply a shell-level timeout to the child process itself, and `npm prefix -g` remains unbounded.

## Best Possible Solution

Keep the issue open and patch `scripts/install.sh` with a small portable deadline wrapper for best-effort finalization probes. It should cover `npm_global_bin_dir`/`npm prefix -g`, `resolve_openclaw_bin` paths that call npm helpers, and `is_gateway_daemon_loaded` invoking `openclaw daemon status --json`; passing `--timeout` to daemon status is useful but should not replace a process-level installer timeout. Add focused `test/scripts/install-sh.test.ts` coverage with stubbed hanging commands proving finalization warns and continues.
