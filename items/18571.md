## Summary

Keep #18571 open. Current main has `message_sending` and `message_sent` hooks, including cancel/rewrite behavior, but it still does not satisfy the requested deterministic verification contract: shared outbound delivery and direct reply paths do not propagate `sessionKey`/`runId` into `message_sending`, and the event type has no bounded tool-call/result history or transcript handle.

## Best Possible Solution

Keep the issue open and implement an additive core plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` across shared outbound delivery plus direct reply paths, expose a bounded documented transcript or tool-call/result history view suitable for deterministic pre-send verification, and add focused regression tests for correlation plus cancel/rewrite behavior.
