## Summary

Current main has adjacent Control UI debug and log surfaces, but not the requested ephemeral connection-diagnostics mode. The browser client still reduces pre-auth failures to `lastError`, drops `connect.challenge` from the event log, ignores websocket `error`, and the Logs tab only works after an authenticated `logs.tail` RPC. The closed related PR #49274 was not merged, so this issue remains the active tracker.

## Best Possible Solution

Keep the issue open and implement it in the Control UI: add session-memory-only connection diagnostics state, instrument `GatewayBrowserClient` for lifecycle events, connect challenge/request/response metadata, close codes/reasons, and structured connect errors, render a copyable overlay/drawer reachable from the connection/settings area, and keep gateway-side `logs.tail` as an optional post-connect supplement. Add targeted tests proving pre-auth events are captured, no diagnostics are persisted to localStorage/IndexedDB, and sensitive auth fields are redacted.
