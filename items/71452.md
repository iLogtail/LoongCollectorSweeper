## Summary

Keep #71452 open. Current main still has the generic human-readable message-list formatter capped at 25 rows, while the CLI/provider surfaces expose limits and some providers can return more than 25 messages or pins. Teams-side pagination is partially implemented, but it does not remove the remaining formatter cap reported by the issue.

## Best Possible Solution

Keep this open and implement a narrow message formatting/action-contract fix: pass the requested limit or pagination metadata into `renderMessageList`, or remove the fixed display cap where the payload is already bounded; add regression tests for more than 25 `read`, `list-pins`, and search rows; preserve existing provider-side pagination behavior such as Teams `@odata.nextLink`.
