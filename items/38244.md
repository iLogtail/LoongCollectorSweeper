## Summary

Current `main` already covers this report: isolated cron runs now pass the resolved delivery route into the embedded message-tool context, the message tool infers a missing target from that ambient current-chat target instead of throwing `Action send requires a target.`, and isolated cron fallback delivery goes through the outbound adapter. The changelog records this cron delivery work in `v2026.4.22`.

## Close Comment

Closing as implemented after a Codex review of current `main`. Isolated cron runs now pass the resolved chat target into the embedded message-tool context (`src/cron/isolated-agent/run-executor.ts:155`), the shared message-action normalization infers a missing send target from that ambient current-chat target before raising `Action send requires a target.` (`src/infra/outbound/message-action-normalization.ts:36`), and the Feishu action path also accepts that ambient target (`extensions/feishu/src/channel.ts:497`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the new job JSON, `openclaw cron runs --id <job-id>` output, and current gateway logs so maintainers can check for a different regression.
