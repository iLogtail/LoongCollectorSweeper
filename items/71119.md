## Summary

Keep #71119 open. Current main still has the LM Studio provider normalize hook wired without a provider-id guard, and the core fallback path can invoke non-matched provider hooks. The PR is a narrow, relevant bug fix with a focused regression test for the missing foreign-provider no-op behavior.

## Best Possible Solution

Keep the PR open and review it as the likely correct fix. The best path is to land the provider-id guard in `extensions/lmstudio/index.ts` with the foreign-provider regression test in `extensions/lmstudio/index.test.ts`, then run the normal extension/changed validation before merge.
