## Summary

Keep open. Current main has real mitigations for #38844: dispatcher-path timeout wording is no longer rewritten as a browser-control-service outage, `/hooks/file-chooser` supports atomic upload-plus-click when `ref` is supplied, the browser tool forwards those atomic fields, and selector/batch support reduces stale-ref workflows. The original product gap is still not fully closed because public browser-control docs still present upload/dialog as split arming calls to run before the triggering click, and the bundled browser automation skill still lacks upload/file-chooser atomic guidance.

## Best Possible Solution

Keep #38844 open until maintainers either accept the current mitigations as sufficient or land the missing guidance and coverage. The concrete path is to update `docs/tools/browser-control.md` and `extensions/browser/skills/browser-automation/SKILL.md` to prefer single-call upload with `ref` or direct `inputRef` on dynamic pages, and add a focused browser tool/route contract test that locks the upload action to the atomic `/hooks/file-chooser` behavior.
