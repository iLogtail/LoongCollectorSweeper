## Summary

Keep #60652 open. Current main still does not advertise `upload-file` or `sendAttachment` from the Discord message-tool discovery path, even though the Discord send handler accepts `media`/`path`/`filePath` and the shared message tool builds its `action` enum from discovered actions. The related fix PRs are useful context, especially #61087, but they are not merged into current main per the provided context and the checked-out source remains unfixed.

## Best Possible Solution

Keep the issue open until a canonical Discord plugin fix lands on main. The likely fix belongs in the Discord plugin: add `upload-file` to `describeDiscordMessageTool` under the appropriate messages/files gate, add a dedicated `upload-file` dispatch path or clearly preserve the existing `send` media delegation, and add regression coverage for Discord discovery with messages enabled plus handler behavior. After one fix PR lands, close or supersede the overlapping PRs #60808, #61087, and #61100 as appropriate.
