## Summary

Keep open. Current `main` does not implement the PR’s byte-aware `chat.history` preservation behavior: it still truncates by character count first and then later replaces oversized messages with `[chat.history omitted: message too large]`. `main` added configurability (`gateway.webchat.chatHistoryMaxChars` and RPC `maxChars`), but that is not the same as this PR’s fix, and the PR timeline’s later cross-reference to issue #65238 matches the placeholder path still being a live symptom.

## Evidence

That means the specific behavior proposed by this PR is not present.
