## Summary

Keep open. Current main still does not implement JSON-stringified content-block extraction, and the PR discussion shows the latest patch needs a more conservative detector before it is safe to merge or cleanup-close.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is a shared, conservative parser that only treats a stringified JSON array as chat content when it has a clear chat-content-block shape, extracts text blocks consistently with the array path, preserves arbitrary JSON strings, and is applied to any string-bypass call sites that need the behavior. Add focused regression tests for text-only vLLM blocks, mixed text/image blocks, invalid JSON, and typed non-chat JSON arrays.
