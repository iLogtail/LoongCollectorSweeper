## Summary

Keep open. Current main still does not parse JSON-stringified chat content blocks, the only production helper caller preserves strings verbatim, and the PR’s latest detection heuristic still has unresolved review feedback about dropping non-chat JSON arrays with a `type` key.

## Best Possible Solution

Keep this PR open or replace it with a tighter maintainer patch. The fix should parse only JSON strings that clearly match OpenAI-style chat content blocks, preserve the original string when parsing yields no text or an unrelated typed JSON array, update the actual production boundary that currently returns strings verbatim, and add regression tests for text-only blocks, mixed text/image blocks, invalid JSON, non-block JSON arrays, and typed non-chat arrays.
