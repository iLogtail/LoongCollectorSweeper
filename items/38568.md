## Summary

Current main still does not inject context-window usage into the system-prompt Runtime line. The issue remains a concrete core prompt/runtime enhancement: context percentage is available on status/session_status surfaces, but the prompt builder has no runtime field, no parameter path, and no test coverage for the requested Runtime injection.

## Best Possible Solution

Keep the issue open and implement this in core prompt assembly. Add an explicit context-usage field to the system-prompt runtime params, populate it from the current session/model context snapshot before prompt construction, render a compact `context=49%` or `context=98k/200k (49%)` field in buildRuntimeLine below the cache boundary, and add focused tests for buildRuntimeLine, prompt param wiring, embedded/CLI prompt construction where applicable, and unchanged status/session_status formatting.
