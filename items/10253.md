## Summary

Keep #10253 open. Current main still does not provide a hook/webhook-specific session lifecycle policy. Hooks expose session-key routing and Gmail presets, but no cleanup/archive/keep-transcript settings; hook dispatch still runs isolated cron-style sessions that are persisted; and cron retention only targets cron run aliases, not hook:gmail session keys. The existing maintainer-posted Codex review also explicitly kept this open.

## Best Possible Solution

Keep #10253 open. The best maintainer path is a core hook-session lifecycle policy under hooks, with schema/types/help/docs updates, dispatch/finalization wiring for /hooks/agent plus mapped/Gmail hooks, clear archive/delete/transcript-retention semantics, and focused tests for Gmail per-message keys, delayed cleanup, session-list visibility, and interaction with existing session.maintenance and cron retention.
