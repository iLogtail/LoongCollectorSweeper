## Summary

Keep open. Current main does not include PR #54600's defensive extraction path: detached cron execution still forwards `job.payload.message` directly, and cron isolated-agent prompt construction would still template-coerce a non-string message if malformed persisted data reaches this path. The PR may still be useful, but it needs maintainer validation because the normal CLI/Gateway create path already defines `message` as a string and the linked issue says `jobs.json` contained the correct string, which does not match the PR's stated root cause.

## Best Possible Solution

Keep PR #54600 open for maintainer review. The best path is to add a focused regression test around detached cron execution with a malformed persisted `agentTurn.payload.message` object, then either land a small normalization helper in `src/cron/service/timer.ts` or normalize/repair the persisted payload shape centrally if maintainers want the store contract to stay strict. The PR should also reconcile its root-cause claim with #54579's note that `jobs.json` had a correct string.
