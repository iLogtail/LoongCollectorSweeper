## Summary

Keep PR #53209 open. Current main still lacks the sandbox allowlist and skill-env sync behavior this PR is trying to add, and the PR discussion identifies a remaining P1 review concern rather than a clean supersession or shipped fix.

## Best Possible Solution

Keep the item open. The best path is a core fix that threads an exact skill-declared env allowlist into Docker sandbox env sanitization, mirrors only actually active/eligible injected skill env keys into sandbox exec env, preserves always-blocked runtime-dangerous keys and null-byte validation, and adds regression tests for sandbox.docker.env plus configured skill apiKey/env paths before landing or closing related reports.
