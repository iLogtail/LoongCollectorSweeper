## Summary

Keep open. Current `main` still lacks the sandbox-side allowlist/sync needed for skill-declared secret env vars, so this is not implemented or obsolete. The PR discussion also ends with an unresolved P1 review concern about syncing env for ineligible skills, so conservative cleanup should not auto-close it.

## Evidence

- **Sandbox is created before skill env overrides run:** `runEmbeddedAttempt` resolves the sandbox first, then later loads skill entries and applies `applySkillEnvOverrides*()`. That means current `main` has no chance to thread skill-declared sensitive env names into sandbox creation.
