## Summary

Keep open. Current `main` still preserves the active run state when a `final` chat event arrives for the same session with a different `runId`, which matches the reported stuck-typing behavior instead of proving it fixed.

## Evidence

- **Cross-run final branch still leaves active run intact:** `handleChatEvent` appends the final assistant message for `payload.runId !== state.chatRunId` and returns without clearing `chatRunId`, `chatStream`, or `chatStreamStartedAt`.
