## Summary

Current main still does not implement the requested two-stage sub-agent stall recovery. The native sub-agent surface exposes wall-clock run timeout only; there is no sub-agent tool-call inactivity timestamp, nudge state, nudge delivery, automatic stall kill, or per-spawn stall override. The related closed PR #43017 is useful prior work but was closed unmerged, and the broader activity/watchdog issues #39127/#39141 do not fully supersede this specific request.

## Best Possible Solution

Keep #39305 open until native sub-agent stall recovery ships or a maintainer explicitly redirects it. A complete fix likely belongs in the sub-agent runtime/config surface: add documented defaults and `sessions_spawn` overrides, track last sub-agent tool activity and nudge state in the run record or associated activity model, wire nudge delivery and automatic kill into the lifecycle/sweeper path, and cover recovery/kill/config/docs behavior with focused tests. If #39127/#39141 land first, this issue can be re-evaluated as a narrower implementation task or closed only if a canonical item fully tracks the remaining sub-agent nudge/kill behavior.
