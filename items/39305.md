## Summary

Keep open. Current main still lacks activity-based sub-agent stall recovery: the shipped sub-agent surfaces expose wall-clock `runTimeoutSeconds` and stale/orphan cleanup, but no last-tool-call tracking, nudge threshold, post-nudge kill threshold, or per-spawn stall overrides. The linked PR #43017 is useful prior art but was closed unmerged, and #39127/#39141 are broader session activity/watchdog items rather than a shipped substitute for this concrete nudge-to-kill behavior.

## Best Possible Solution

Keep #39305 open until OpenClaw either implements this concrete activity-based sub-agent recovery path or maintainers explicitly choose a different product direction. The likely implementation belongs in the sub-agent runtime/registry: add strict config and `sessions_spawn` overrides for nudge and kill thresholds, record deterministic child activity such as tool calls or run progress, inject an auditable system/session nudge, terminate through the existing sub-agent kill/completion flow, and cover it with focused tests plus sub-agent docs. PR #43017 can be treated as prior art; #39127/#39141 may provide broader activity/watchdog infrastructure but do not currently replace this behavior.
