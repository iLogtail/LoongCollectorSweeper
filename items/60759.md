## Summary

Current main still appears to embed plugin config schemas as-is, while the bundled qqbot and webhooks manifests still contain local `$defs` plus root-relative `$ref` values. The reported editor-schema bug is therefore not safely closeable as implemented.

## Best Possible Solution

Keep the issue open and land a focused schema-export fix. The maintainable path is to normalize plugin/channel schemas when building the full config schema: either hoist `$defs` to the document root with collision-safe names and rewrite internal `$ref`s, or inline the affected manifest definitions. Add regression coverage in `src/config/schema.test.ts` or CLI schema tests for qqbot/webhooks-style `$defs` so `openclaw config schema` emits resolvable refs.
