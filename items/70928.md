## Summary

Keep #70928 open. Current main partially contradicts the report because `before_message_write` and `agent_end` are wired through the embedded Pi runner, but the central outbound lifecycle gap remains for the default chat surfaces: WebChat and remote openclaw-tui both use `chat.send`, and that path does not route assistant replies through the outbound delivery hook emitter or the dispatcher wrapper that installs `message_sending`.

## Best Possible Solution

Keep the issue open and narrow the fix to the `chat.send` / internal WebChat delivery path. The best implementation is to reuse a shared outbound observer/delivery seam where possible, or at minimum install `message_sending` before WebChat/TUI dispatcher delivery and emit `message_sent` after final assistant delivery/persistence, with regression tests covering WebChat and remote openclaw-tui. The issue should also note that `before_message_write` and `agent_end` are already wired for the embedded Pi runner on current main.
