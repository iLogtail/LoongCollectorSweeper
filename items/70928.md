## Summary

Keep open. The report mixes internal `HOOK.md` events with plugin-hook names, so the stated reproduction is not coherent as written, but current `main` still appears to have a real outbound hook-coverage gap for webchat/TUI reply delivery: the shared outbound path wires `message_sending`/`message_sent`, while `chat.send` webchat/TUI flows use custom reply handling that bypasses that seam. The report likely needs narrowing because `agent_end` and `before_message_write` look independently wired in the embedded runner.

## Evidence

- **Internal hook docs do not match the reported hook names:** The public internal-hook event table documents `message:sent` and other `type:action` names for `HOOK.md` handlers; it does not list plugin-style `message_sent`, `agent_end`, or `before_message_write` as internal hook events. That means the issue’s repro steps are mixing two hook systems.
