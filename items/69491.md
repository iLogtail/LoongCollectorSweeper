## Summary

Keep PR #69491 open. Current main still does not forward default Anthropic beta flags through the bundled `amazon-bedrock` Converse stream path, while the direct Anthropic plugin does maintain and merge those defaults. The PR addresses a real remaining parity gap, but it should not be treated as ready or obsolete because review comments identify concrete branch risks around overwriting existing `anthropic_beta` values and applying Anthropic-only request fields via a broad Bedrock model heuristic.

## Best Possible Solution

Keep the PR open and have the Bedrock provider implementation inject Bedrock-correct `additionalModelRequestFields.anthropic_beta` for definitive Claude Bedrock models, merge/dedupe any existing beta values, compose cleanly with guardrail and cache-point wrappers, and add focused tests for no-guardrail, guardrail, existing beta preservation, non-Anthropic models, and application-profile false positives or resolution behavior.
