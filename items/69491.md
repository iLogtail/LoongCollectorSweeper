## Summary

Keep PR #69491 open. Current main still does not forward the default Anthropic beta flags through the Amazon Bedrock Converse stream payload; the existing Bedrock wrapper leaves Anthropic Bedrock streams unchanged except for guardrail/region/cache-point handling, and the Bedrock tests still assert only guardrail/cache behavior. The PR is also not ready to auto-close or land as-is because its review discussion identified concrete P2 issues around preserving existing `anthropic_beta` values and avoiding broad model-ID heuristic false positives.

## Best Possible Solution

Keep the PR open, but require a revised implementation before merge. The best path is a Bedrock-local stream payload wrapper in `extensions/amazon-bedrock/register.sync.runtime.ts` that adds the default Anthropic beta flags via `additionalModelRequestFields.anthropic_beta`, merges with any existing array without duplicates, composes with the guardrail/cache wrappers, and uses stricter Anthropic Claude detection than the profile-name cache heuristic where invalid Bedrock payloads are possible. Add focused tests in `extensions/amazon-bedrock/index.test.ts` for Claude Bedrock with and without guardrails, preservation of existing `anthropic_beta`, and no injection for non-Anthropic/profile false positives.
