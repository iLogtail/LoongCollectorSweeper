## Summary

Keep open. Current `main` has partial mitigation: auto-compaction can estimate prompt size from raw transcript messages when cached totals are stale, and `/status`/session listing can recover totals from transcript usage snapshots. But the requested feature is not fully implemented for providers that return no usable usage data at all: status/session display still depends on transcript assistant usage snapshots and does not fall back to a local estimated context count with an explicit estimated marker.

## Evidence
