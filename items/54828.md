## Summary

Current main already fixes #54816 by preventing `config.schema`/`config.get` from re-activating the plugin runtime registry on each MCP reconnect. The fix landed in `src/config/runtime-schema.ts` and is covered by an explicit regression test in `src/config/runtime-schema.test.ts`, so PR #54828's `src/plugins/registry.ts` dedupe is no longer the right path.

## Evidence

See `src/config/runtime-schema.ts:11` and `src/config/runtime-schema.ts:21`.

## Close Comment

Closing as implemented on current `main` after Codex review. The underlying #54816 bug is already fixed by changing gateway schema loads to use manifest metadata instead of re-activating the runtime plugin registry on each MCP reconnect: see `src/config/runtime-schema.ts:21` in commit `a52b50ef3c8bb05d4a18302aad6862c37278cc57`. Current `main` also has an explicit regression test for this exact path at `src/config/runtime-schema.test.ts:227`, which calls out repeated `config.schema` / `config.get` requests, Feishu `registerFull`, and asserts the active registry is not replaced across repeated loads. Since the root cause is already addressed on `main`, this PR's extra dedupe in `src/plugins/registry.ts` is no longer needed.
