## Summary

Keep open. The report is recent, has concrete Feishu WebSocket reconnect-loop logs, is not maintainer-authored, has no protected labels, and current main still lacks the requested watchdog/session reset behavior. Current code delegates reconnect behavior to the Lark SDK and `status --probe` remains an HTTP credential/bot-info probe, not a WebSocket reachability check.
