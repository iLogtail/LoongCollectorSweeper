## Summary

Keep #72044 open. Current main still has the DeepSeek V4 replay wrapper filling missing assistant tool-call `reasoning_content` with an empty string, and the tests explicitly assert that blank fallback. The issue reports that this exact fallback is insufficient for long multi-turn DeepSeek thinking/tool-call sessions, so the existing 4.24-era fix is not enough evidence to close as implemented.

## Best Possible Solution

Keep the issue open and fix the DeepSeek V4 replay path in the shared OpenAI-compatible wrapper or adjacent transcript conversion so missing assistant tool-call `reasoning_content` can be populated from the last real DeepSeek thinking payload when the provider requires it. Add focused coverage for multiple DeepSeek V4 thinking tool-call rounds, ideally including a `sessions_spawn` or embedded-runner replay shape, and keep the existing disabled-thinking stripping behavior intact.
