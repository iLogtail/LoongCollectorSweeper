## Summary

Keep #72044 open. Current main still uses the incomplete DeepSeek V4 replay behavior the issue describes: missing `reasoning_content` on assistant tool-call messages is filled with an empty string, and the focused tests explicitly assert that blank placeholder path. The existing live coverage only proves a short single prior-tool-call replay, not the reported long multi-turn `sessions_spawn`/tool-call chain that needs actual prior reasoning content replayed.

## Best Possible Solution

Keep the issue open and fix it in the DeepSeek/provider replay path or upstream `pi-ai`: replay actual non-empty DeepSeek `reasoning_content` for DeepSeek V4 assistant tool-call history instead of inserting `""`, then add a focused regression that exercises a long multi-turn tool-call or `sessions_spawn` chain with DeepSeek V4 Pro thinking enabled. If the durable fix belongs upstream, OpenClaw should either update to the fixed `pi-ai` version or carry a narrow wrapper until the upstream release is available.
