## Summary

Keep #12512 open. Current main still exposes SKILL.md as model-facing skill instructions: the system prompt tells the model to read and follow selected SKILL.md files, docs describe skills as injected into the system prompt, and the available metadata only supports eligibility/visibility controls, not the requested hard instruction-isolation, role-locking, memory, tool, or behavioral-monitoring boundary. The existing Apr 26 issue discussion also explicitly kept this open for the same core security-hardening gap.

## Best Possible Solution

Keep #12512 open as core security hardening. The best path is to define and document an instruction-isolation model for SKILL.md consumption, add enforceable skill capability metadata for role, memory, and tool influence, and cover malicious SKILL.md cases with regression tests and security docs. This should not move to ClawHub unless core first provides a real isolation/API boundary external skills can rely on.
