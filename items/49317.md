## Summary

Keep #49317 open. Current main normalizes WhatsApp LID mentions, but the specific self-chat/allowFrom branch reported here is still present: group mention detection still ignores explicit mention JIDs when the linked self number is in allowFrom, so normalizedMentionedJids can match the bot while wasMentioned remains false.

## Best Possible Solution

Keep the issue open and fix the WhatsApp plugin mention-gating path: make group mention detection compare normalized native mention identities against the bot/self identity even when allowFrom contains the linked self number, or pass an explicit group-safe selfChat flag into debugMention. Add a regression test for a group message with mentionedJids like 216372600647751@lid resolving to selfE164 while allowFrom includes that self number, plus a separate test preserving the owner/self-chat safeguard.
