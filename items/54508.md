## Summary

Keep open. Current main has useful boundary marking, web_fetch metadata, tool-result middleware, and exec approvals, but it does not implement the requested persistent taint/provenance model or default enforcement against content-derived exec, secret access, or outbound actions. The issue is a live core security architecture umbrella with narrower related issues still open, including security-labeled reports.

## Best Possible Solution

Keep #54508 open as a security architecture umbrella, or have a maintainer replace it with an explicit canonical RFC/design issue. The implementation path should preserve existing wrappers and approvals while adding durable provenance on tool outputs and persistence writes/reads, enriching before_tool_call with stable action classification and input provenance, and enforcing provenance/risk/direct-human-intent gates for exec, messaging/outbound actions, and secret-bearing paths.
