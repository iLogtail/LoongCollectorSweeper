## Summary

Keep open. Current `main` has two adjacent capabilities, but not the requested one: TUI-only `/agent <id>` switching, and channel `/focus`/`/agents` thread-binding commands for subagent/session targets. OpenClaw still documents multi-agent routing as config/bindings-driven, so there is no channel command today that lists configured agents and switches the current conversation to another agent by agent id.

## Risks / Open Questions

- `/focus` is a partial workaround on channels that support conversation bindings, so maintainers may view the request as lower priority even though it does not match the requested `/agent <name>` UX.
- The original restart complaint is partly outdated because current docs show hot reload for `agents` and `bindings`; the remaining gap is the missing conversational command surface, not restart behavior.
