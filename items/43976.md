## Summary

Keep open. Current main has TUI-only agent switching and channel slash commands for subagents/thread bindings, but it does not implement the requested in-channel `/agents` list of configured agents plus `/agent <name>` temporary per-conversation agent switch without Gateway restart.

## Best Possible Solution

Keep the issue open for maintainer product/design work. The likely core path is a Gateway/channel chat command that lists configured `agents.list` entries and stores a temporary per-conversation or per-session agent override that routes future turns to that agent's own session/workspace, with clear precedence against configured bindings, `/focus`, ACP/subagent bindings, and deletion/reset behavior. If maintainers want this outside core, first expose a deliberate plugin API for non-plugin-owned conversation-to-agent routing rather than relying on ad hoc binding internals.
