## Summary

Current main already covers PR #41694's central request through the newer `messaging.inferTargetChatType` channel-plugin seam. Plugins can now steer bare targets to direct, group, or channel resolution before directory lookup, outbound resolution uses that inferred kind, and tests cover direct/user inference plus direct session routing from resolved user targets. The exact `defaultTargetKind` field from the PR is not present, but the shipped API solves the same plugin-routing problem more generally.

## Best Possible Solution

Close PR #41694 as implemented on main. Keep the shipped `inferTargetChatType`/`looksLikeId`/`resolveTarget` split as the supported plugin API, and direct any remaining edge-case work to focused follow-up issues against that current contract rather than reviving the older `defaultTargetKind` patch.
