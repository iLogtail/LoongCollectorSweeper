## Summary

Keep #13597 open. Current main still lacks a dedicated AWS deployment guide for EC2/Lightsail, ECS/Fargate, or Lambda applicability. Existing docs cover reusable pieces such as generic Docker Compose, Kubernetes, VPS, and Bedrock provider setup, but they do not satisfy the requested AWS production deployment runbook. The issue discussion is still actionable and includes concrete production concerns plus an AWS sample repository link.

## Best Possible Solution

Keep the issue open until OpenClaw ships an AWS install page, likely under docs/install/aws.md and linked from docs/install/index.md, docs/vps.md, and docs/docs.json. The guide should make Docker Compose the primary EC2/Lightsail path, cover durable state/EFS or equivalent storage, gateway bind/reverse proxy/ALB/TLS/auth/origin settings, security groups, IAM roles, Secrets Manager, logging/monitoring, backups, troubleshooting, performance and hardening, ECS/Fargate task/service/networking, Lambda applicability or non-support, multi-agent topology, and evaluate or link the AWS sample repo from the issue discussion.
