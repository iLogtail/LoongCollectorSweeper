## Summary

Keep PR #71582 open. Current main at 21c51bc140295e4286f1adc871a96b9b33c6a8b9 still has the old exec-event heartbeat wake routing: exec completions request only `reason: "exec-event"` and scoped session keys, with no `target: "last"` heartbeat override and no explicit clears for pinned `to`, `accountId`, or `isolatedSession`. The provided PR context shows the latest PR head addresses the Greptile duplicate-helper review by moving the shared exec-event wake helper into `src/routing/session-key.ts`, so the remaining useful path is normal maintainer review rather than cleanup closure.
