## Summary

Keep PR #71582 open. Current main still has the exec/system-event heartbeat routing gap: exec wakes are scoped by session key, but they do not request `target: "last"` or clear configured `heartbeat.to`, `heartbeat.accountId`, or `isolatedSession`. The latest PR discussion and diff show a focused bug-fix path, including a follow-up that deduplicated the helper flagged by Greptile.

## Best Possible Solution

Keep PR #71582 open. The best path is to continue review of the shared `scopedExecEventWakeOptions` route-clearing approach, preserve legacy synthetic node fallback behavior, and run the changed gates for the touched heartbeat, gateway, cron, routing, and agent surfaces before merging.
