## Summary

Keep #71975 open. Current main has the shared video-generation plugin surface and the fal provider can be extended, but first-class LTX-Video support is not present in docs, provider model metadata, defaults, or a Comfy workflow template. The dedicated Lightricks-provider part likely belongs in a third-party plugin/ClawHub path, but the issue's lowest-risk requested path is an actionable docs/model-catalog improvement for existing bundled video-generation surfaces, so this should not be auto-closed.

## Best Possible Solution

Keep the issue open for a focused maintainer/product decision. The best path is likely a small first-party docs/model-discovery patch for LTX through the existing fal video-generation provider if upstream schema validation checks out, while treating a dedicated Lightricks provider or reusable Comfy LTX workflow bundle as third-party/ClawHub work unless a maintainer explicitly commits to bundled ownership.
