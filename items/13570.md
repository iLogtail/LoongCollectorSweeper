## Summary

Current main still lacks `contextPruning.forcePruneRatio` or any equivalent high-usage TTL bypass. The config type, strict Zod schema, generated schema, docs, runtime settings, and context-pruning extension all preserve the existing TTL gate before pruning can run. The related PR #13572 directly attempted this feature but is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #13570 open until a maintainer implements or explicitly declines the feature. A likely fix should add `agents.defaults.contextPruning.forcePruneRatio` to the TypeScript config type, strict Zod schema, generated schema, docs, effective settings, and runtime; estimate usage before the fresh-TTL early return; bypass TTL only when the configured ratio is met; and add regression tests for both above-threshold bypass and below-threshold TTL preservation.
