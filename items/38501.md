## Summary

Current main still lacks the requested Ctrl+C active-run clarification. The TUI Ctrl+C path only resolves clear/warn/exit behavior and sets generic exit status text, while active run cancellation remains on Esc and `/abort`. The related fix PRs in the provided GitHub context are open or unmerged, so the issue remains actionable rather than implemented, duplicate-closed, or stale.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best fix is to update `src/tui/tui.ts` so Ctrl+C status text appends an active-run hint when `activeChatRunId` is set, while preserving current Ctrl+C clear/exit behavior and leaving Esc and `/abort` as the actual abort paths. A small regression test around the Ctrl+C status construction or handler would make this user-facing contract explicit.
