## Summary

Keep #38501 open. Current `main` still has Ctrl+C wired to clear/warn/exit status only, with no active-run hint, while Esc and `/abort` remain the actual abort paths. The exact fix work is still represented by open/unmerged related PRs rather than shipped main behavior.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best implementation is to update `src/tui/tui.ts` so the Ctrl+C clear/warn status text appends an active-run hint when `activeChatRunId` is set, while preserving existing Ctrl+C clear/exit behavior and leaving Esc plus `/abort` as the actual abort paths. Add a small regression test for active-run Ctrl+C status construction or handler behavior.
