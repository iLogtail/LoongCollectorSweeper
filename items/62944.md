## Summary

Keep open. Current `main` still hardcodes a 30s timeout in the `image` tool, while the config/docs expose configurable media-understanding timeouts. The underlying bug remains on `cf858258c71701ef62dddf88aa777873a4962046`, and the latest release `v2026.4.23` also shipped the same hardcoded value.

## Risks / Open Questions

- Slow self-hosted or local vision models will still abort around 30 seconds even when operators configure higher image/media timeouts.
- The issue title names `tools.media.models.timeoutSeconds`, but the current documented config surface is `tools.media.image.timeoutSeconds` plus per-model `models[].timeoutSeconds`; the exact knob naming may need clarification when fixing, but the core timeout bug is real.
- `src/agents/tools/image-tool.test.ts` does not appear to cover timeout propagation, so this regression can persist without test failure.
