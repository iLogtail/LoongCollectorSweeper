## Summary

Keep PR #57309 open. Current main still does not implement per-channel Slack `replyToMode`: the Slack per-channel config type/schema/resolver only cover mention/users/bots/skills/systemPrompt/tools, and inbound/runtime reply mode is still resolved from account/chat-type config. The PR is a focused implementation path, but its current head needs the already-reported `channelConfig` plumbing fix before it can merge.

## Best Possible Solution

Keep the PR open and request a small fix: add `replyToMode` to Slack per-channel config schema/types/resolution, pass the resolved channel config into Slack routing, propagate the pre-resolved mode into `MsgContext`/`ChannelThreadingContext` and downstream reply filtering, and add focused tests for exact channel, wildcard fallback, and tool-context propagation. The existing PR discussion and bot comments already identify the main branch blocker.
