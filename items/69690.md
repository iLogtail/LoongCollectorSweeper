## Summary

PR #69690 still carries a distinct runtime-registry compatibility fix that is not present on current `main`. Current `main` includes broader gateway-bindable active-registry reuse for default-mode runtime requests, but it does not record the active gateway method names or synthesize them when an explicitly gateway-bindable helper/runtime load omits `coreGatewayHandlers`, so the cache key can still miss the gateway-startup registry described in the PR.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent narrow fix on `main`: record the gateway method names associated with the active gateway-bindable registry, use them only to reconstruct the same cache key for compatible gateway-bindable helper/runtime loads that omit `coreGatewayHandlers`, and keep regression tests proving both the reuse path and mismatched-method rejection. Also preserve the current guard that default-mode callers do not accidentally inherit gateway bindings unless that behavior is intentionally covered by existing policy.
