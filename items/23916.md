## Summary

Keep #23916 open. Current main stores and returns useful pairing metadata, but the human `openclaw devices list` output still only shows compact device/access/token/IP columns and has no `--full`, describe, or editable note/name workflow. Related #13870 and PR #55973 cover narrower naming/timestamp pieces, not the full requested device-purpose and client-detail surface.

## Best Possible Solution

Keep #23916 open and implement a user-facing device detail workflow: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while preserving existing `--json` behavior.
