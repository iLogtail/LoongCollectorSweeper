## Summary

Keep #23916 open. Current main still does not expose the requested client purpose/mode, platform/device family, host/user-agent-style details, editable notes/names, or a full/detail view in the human `openclaw devices list` output. The underlying pairing records and RPC path already carry some of that metadata, so the remaining work is a user-facing CLI/docs workflow, not storage plumbing.

## Best Possible Solution

Keep #23916 open. Add a user-facing device detail workflow in the core devices CLI: show compact client purpose metadata in `openclaw devices list`, add a `--full` or describe-style view for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable names or notes with #13870 rather than treating persisted metadata alone as sufficient.
