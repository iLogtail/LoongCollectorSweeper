## Summary

Keep #19482 open. Current `main` still writes `HEARTBEAT.md` whenever `ensureAgentWorkspace({ ensureBootstrapFiles: true })` sees it missing, even for existing workspaces. That conflicts with the documented and runtime distinction where a missing `HEARTBEAT.md` keeps heartbeat active, while an effectively empty file skips as `empty-heartbeat-file`. The PR remains a narrow, plausible fix; its negative test should be tightened to model a fully initialized workspace with only `HEARTBEAT.md` absent.

## Best Possible Solution

Keep the PR open and either land it after tightening the negative regression test, or implement the same behavior directly on main: only seed `HEARTBEAT.md` for brand-new workspaces while preserving missing-file heartbeat semantics for existing workspaces.
