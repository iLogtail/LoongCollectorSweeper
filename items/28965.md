## Summary

Keep #28965 open. Current main still does not provide the proposed first-class `openclaw models reconcile` workflow or scoped bulk `openclaw sessions model-reset` command. Existing code has targeted model setters/status/listing, per-session model reset paths, `sessions.reset` behavior, and session cleanup, but those do not satisfy the central operator workflow requested by the issue.

## Best Possible Solution

Keep #28965 open until OpenClaw either implements a documented model configuration reconciliation flow plus a scoped bulk session model override reset, or maintainers explicitly consolidate it into a canonical model/config single-source design issue. The likely implementation belongs in the existing `models` and `sessions` CLI surfaces with `--dry-run`, `--json`, `--agent`, `--all-agents`, `--store`, and match/filter support, plus regression coverage for clearing `providerOverride`, `modelOverride`, `modelOverrideSource`, and optional auth-profile override fields without disturbing unrelated session data.
