## Summary

Keep #28965 open. Current main still has only partial normalization through targeted model commands and one-session override clearing through `sessions.patch`; it does not provide the requested `openclaw models reconcile` or bulk `openclaw sessions model-reset` operational surface.

## Best Possible Solution

Keep this issue open until OpenClaw implements first-class core CLI support for model config reconciliation and bulk session override reset, or a maintainer explicitly consolidates it into a canonical model/config hierarchy issue. The likely implementation belongs in the core CLI plus config/session-store code, reusing the existing model-ref normalization, allowlist upsert, fallback dedupe, session patch/reset semantics, scope selection, dry-run, and JSON reporting.
