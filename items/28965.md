## Summary

Keep #28965 open. Current main still has targeted model status/list/set/scan/auth commands and per-session model override patching/reset behavior, but it does not implement the requested first-class `openclaw models reconcile` command or scoped bulk `openclaw sessions model-reset` workflow. The related open model/config issues do not supersede this proposal.

## Best Possible Solution

Keep #28965 open until OpenClaw either implements explicit model configuration reconciliation plus scoped bulk session model override reset, or maintainers consolidate it into a canonical model/config single-source design issue. The likely implementation belongs in the existing `models` and `sessions` CLI surfaces with dry-run/JSON reporting, scoped `--agent`/`--all-agents`/`--store` behavior, regression tests around session override clearing, and docs for model migration/troubleshooting.
