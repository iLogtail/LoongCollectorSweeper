## Summary

PR #71645 should stay open. Current main still updates the active Mattermost draft preview with partial assistant text, then writes tool-start status into the same pending draft without flushing or rotating the message, which matches the reported overwrite bug. The PR is a focused channel bug fix using existing Mattermost draft-stream primitives, but it still needs maintainer review and ideally a regression test/docs decision.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to preserve Mattermost streamed assistant text across tool-start events using the existing draft stream lifecycle, add a focused regression test for partial text followed by `onToolStart`, and either update the Mattermost preview-streaming docs if a separate tool-status post is accepted or adjust the design to preserve text while staying within the documented single-preview-post behavior.
