## Summary

PR #71645 should stay open. Current main still updates the active Mattermost draft preview with tool-start status without first flushing streamed assistant text or rotating to a new preview post, so the reported overwrite remains plausible and the PR is not implemented or superseded.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to add a narrow Mattermost regression test for partial streamed text followed by `onToolStart`, then either merge the existing flush-and-rotate approach with a docs update or choose a same-post preservation design that still prevents overwriting streamed text.
