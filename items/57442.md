## Summary

Current `main` does not show elapsed/response time in the `session_status`/session status card path, and the persisted/transcript-backed status data it reads does not carry that field for normal sessions. This looks like a still-open feature/regression request rather than something already implemented or obsolete.

## Evidence

- **`session_status` uses the shared status-text builder with no timing-specific field:** The tool builds its response by calling `buildStatusText(...)`; the arguments include model/runtime/task context but no elapsed or response-time value.
