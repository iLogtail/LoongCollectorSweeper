## Summary

Current `main` has moved internal gateway restarts onto an authorized `SIGUSR1` path, added restart-reason logging/coalescing plus restart deferral/draining, and now surfaces restart-lifecycle delivery failures explicitly instead of silently dropping new work.

## Evidence

- **Internal restart path is SIGUSR1-only and carries reason/coalescing metadata:** `emitGatewayRestart()` authorizes and emits `SIGUSR1`, while `scheduleGatewaySigusr1Restart()` records `reason`, cooldown, and coalescing state for internal restarts.
  - file: [src/infra/restart.ts:120](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/restart.ts#L120)
  - command: `nl -ba src/infra/restart.ts | sed -n '1,240p'; nl -ba src/infra/restart.ts | sed -n '240,620p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Gateway run loop treats SIGTERM as stop, not self-restart, and drains before restart:** The gateway `SIGTERM` handler requests `stop`, while restart handling waits for active tasks and embedded runs, logs drain/timeout state, and only then closes the server for restart.
  - file: [src/cli/gateway-cli/run-loop.ts:139](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/cli/gateway-cli/run-loop.ts#L139)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Config-triggered restarts now log why they are happening and defer until work drains:** `requestGatewayRestart()` logs restart reasons, counts pending operations/replies/runs/tasks, defers restart until idle, and logs timeout/check-failure cases before continuing.
  - file: [src/gateway/server-reload-handlers.ts:201](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/gateway/server-reload-handlers.ts#L201)

  - file: [src/process/command-queue.ts:24](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/process/command-queue.ts#L24)
