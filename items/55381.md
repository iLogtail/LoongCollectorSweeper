## Summary

Keep #55381 open. Current main still promotes assistant text from mixed assistant+tool-call messages into subagent output selection, then freezes that selected text into `frozenResultText` for completion announce-back. The related PR #55510 was closed unmerged, and I found no shipped replacement fix or regression coverage for the requested mixed-turn filter.

## Best Possible Solution

Keep the issue open and fix it in `src/agents/subagent-announce-output.ts`: when summarizing assistant messages, do not promote text from messages that also contain tool calls, except for explicitly extracted `<final>...</final>` content if maintainers want that escape hatch. Keep timeout/error partial-progress summaries separate from final-result selection, and add focused regression tests for mixed assistant+toolCall text, mixed `<final>` content, assistant-only final text, and timeout partial progress.
