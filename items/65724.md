## Summary

Keep open. Current main has fixed the shared Windows exec helper that likely covered many DEP0190 reports, and the changelog shows that shipped in 2026.2.14, but current main still has a shipped TUI local-auth path that intentionally uses `spawn(command, args, { shell: true })` for Windows `.cmd`/`.bat` shims. Since DEP0190 is specifically about args plus `shell: true`, the broad no-warning request is not fully resolved.

## Best Possible Solution

Keep the issue open and narrow the remaining work to shipped Windows child-process call sites that can still pass a non-empty args array with `shell: true`. The likely code fix belongs in the TUI local auth/Codex `.cmd` shim path: resolve Windows shims through the existing `windows-spawn`/explicit wrapper approach or another non-shell argv-safe path, then update `src/tui/tui.test.ts` to assert no args-based shell spawn is used.
