## Summary

本 Issue 请求在单进程多采集配置场景下支持任务级优先级与资源分级保障（QoS）。经核查当前 main 分支核心调度与配置解析逻辑，尚未原生实现基于任务重要性的资源抢占或降级策略。该能力涉及 Pipeline 调度模型与底层资源管理的核心架构调整，不属于插件可独立承载范围，且需求描述清晰有效，故保持开启。

## Best Possible Solution

保持开启以待维护者评估核心调度器与资源管理架构路线。若纳入规划，需在 Pipeline/Config 层设计 priority/qos 字段及对应的资源隔离与降级策略；若暂不纳入核心路线，建议将社区提出的多实例配合 cgroups/systemd 的分级隔离方案沉淀至官方最佳实践文档，并在此 Issue 中说明后续规划。
