## Summary

本 Issue 请求在单进程多采集配置场景下支持任务级优先级与分级资源保障（QoS）。经核对当前 main 分支核心调度器与 Pipeline 架构，尚未原生实现基于重要性的任务级资源抢占或降级策略。社区讨论提供的多进程+cgroups 方案属外部变通，未满足单进程内原生调度的需求。该特性涉及核心流水线调度改造，属于合理的产品增强建议，无关联 PR 或重复工单，故保持开启。

## Best Possible Solution

保持开启以待维护者进行产品路线与核心调度器架构评估。若纳入规划，需设计任务级优先级配置规范（如在 Pipeline/Config 层级增加 priority/qos 字段）、资源隔离与降级策略，并评估对现有 Flush/Process 队列的影响。若暂不纳入核心，建议将多实例配合 cgroups/systemd 的分级隔离方案沉淀至官方最佳实践文档。可邀请作者或社区贡献者补充详细设计草案（如优先级维度、资源让步机制、压测数据）以便推进技术评审。
