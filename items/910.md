## Summary

本 Issue 提出了在单进程多采集配置场景下实现任务级优先级与分级资源保障（QoS）的架构增强需求。经核对当前 main 分支核心调度与资源管理模型，该能力尚未原生实现，且属于合理的核心产品演进方向，不符合自动关闭条件。

## Best Possible Solution

保持开启以待维护者进行核心调度器架构评估。若纳入规划，需在 Pipeline/Config 层设计 priority/qos 字段及对应的资源隔离与降级策略；若暂不纳入核心路线，建议将社区提出的多实例配合 cgroups/systemd 的分级隔离方案沉淀至官方最佳实践文档，并在此 Issue 中说明后续规划。
