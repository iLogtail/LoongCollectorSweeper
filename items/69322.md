## Summary

Keep PR #69322 open. Current main still exposes `doctor.memory.status` only for the active default agent and has no `perAgent` payload, while the CLI memory status path already iterates configured agents and returns per-agent status objects. The PR addresses a real RPC/CLI parity gap from #69295, but should be revised before merge to handle per-agent manager acquisition failures locally and fix the documented indentation nit.

## Best Possible Solution

Keep the PR open and ask for a small revision: add the additive `perAgent` field to `doctor.memory.status`, enumerate agents via the existing agent-scope helper, reuse the default manager where possible, wrap each non-default manager acquisition/status read/close in local error handling, add a regression test for acquisition failure on a secondary agent, and fix the protocol-doc indentation before merge.
