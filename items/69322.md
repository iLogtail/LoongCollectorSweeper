## Summary

Keep open. Current main still exposes `doctor.memory.status` only for the default agent and has no `perAgent` payload, while the existing CLI memory status path already iterates configured agents and emits JSON results. The PR is a plausible focused fix for the RPC parity gap, but it still needs the unresolved review concern around per-agent manager acquisition errors addressed before merge/close cleanup.

## Best Possible Solution

Keep the PR open and ask for a small update: rebase onto current main, add `perAgent` to `doctor.memory.status`, preserve additive protocol compatibility, catch manager acquisition/status failures inside each per-agent collection so one broken secondary agent returns `{ agentId, dirty: false, files: 0, chunks: 0, error }`, and keep/update the protocol docs plus tests for default-only, multi-agent, missing manager, and acquisition-throw cases.
