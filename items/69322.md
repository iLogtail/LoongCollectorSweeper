## Summary

Current `main` does not implement per-agent `doctor.memory.status`, and this PR still has a substantive unresolved failure path: a non-default agent whose memory manager acquisition throws can break the whole aggregated response. That matches the existing review comment and is not covered by the added tests, so conservative cleanup should leave the PR open.

## Evidence
