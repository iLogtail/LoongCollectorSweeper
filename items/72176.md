## Summary

Keep #72176 open. This is a recent, concrete regression report, and current main still lacks the two protections named by the linked fix PR: inbound duplicate suppression when `MessageSid` is missing or unstable, and user transcript idempotency on the `chat.send` transcript update path. PR #72290 is related and still open, but it has not merged or shipped, so the issue should remain the canonical tracker.

## Best Possible Solution

Keep #72176 open as the canonical regression. Land PR #72290 or an equivalent narrower fix only after removing any unrelated/accidental files, adding regression coverage for unstable or missing MessageSid retry paths and user transcript idempotency, and verifying the duplicate user-message plus duplicate assistant-reply symptom against current main.
