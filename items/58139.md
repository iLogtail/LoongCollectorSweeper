## Summary

Keep open. Current main does not show a clear fix for the reported Windows Docker bind-mount failure: the `memory-lancedb` initialization path is still single-shot with no retry/backoff or Windows-specific handling, and the docs/config surface still lacks a memory-lancedb-specific Docker volume warning or workaround.

## Evidence

- **Single-shot LanceDB initialization remains:** `MemoryDB.doInitialize()` still does `connect()`, `tableNames()`, then `createTable()` followed by an immediate `delete()` with no retry, delay, or Windows/bind-mount handling. That matches the reported failure mode rather than proving a fix.

A repo search did not find a `memory-lancedb` note telling Windows Docker users to prefer a Docker volume over a bind mount.
