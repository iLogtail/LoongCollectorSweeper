## Summary

Keep open. Current main still retries memory embedding batches only for rate-limit/quota/5xx-style messages, so transport errors such as `TypeError: fetch failed`, `ECONNRESET`, `socket hang up`, and `other side closed` would still leave `embedBatchWithRetry()` through the non-retryable path. The PR’s exact patch is stale because memory code now lives under `extensions/memory-core/`, but the linked bug #44166 remains coherent and the PR discussion shows the author addressed the bot review concerns by tightening the network matcher and strengthening split coverage.

## Best Possible Solution

Keep the PR or linked issue open and port the fix into the current memory-core plugin implementation: add a narrowly scoped transient transport-error classifier in `extensions/memory-core/src/memory/manager-embedding-policy.ts`, apply it from `embedBatchWithRetry()` in `extensions/memory-core/src/memory/manager-embedding-ops.ts`, preserve retry/backoff behavior, add recursive split fallback for exhausted multi-item text batches, and add current-path regression tests for `fetch failed` retry plus full split coverage.
