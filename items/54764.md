## Summary

Keep open. Current main still has the cleanup gaps this PR targets: killed runs are coalesced by childSessionKey before cleanup bookkeeping, delete cleanup removes only the passed runId, and orphan pruning still bypasses the lifecycle cleanup path that retries deferred announces.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to port the fix onto the current split registry structure: finalize cleanup per runId in markSubagentRunTerminated, give orphan pruning the same final cleanup/deferred-retry semantics or an equivalent explicit hook, and add focused regressions for repeated delete-mode termination, multiple runIds sharing a childSessionKey, and orphan-prune deferred cleanup retry preservation.
