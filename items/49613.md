## Summary

Current main still has the reported heartbeat inheritance bug. `agents.defaults.heartbeat` is merged into the resolved heartbeat config, but enablement and scheduling still use the old scope rule: explicit per-agent heartbeat entries win, otherwise only the default agent runs. The existing regression test asserts the exact reported `main`/`ops` defaults-only case with `ops` disabled, and the related fix PRs in the provided context are closed/unmerged.

## Best Possible Solution

Keep this issue open and fix the core heartbeat scope contract in `src/infra/heartbeat-summary.ts` and `src/infra/heartbeat-runner.ts`: defaults-only heartbeat config should enable every configured agent, explicit per-agent heartbeat entries should continue to provide opt-in scoping/overrides, and the current regression test should be updated to assert the non-default inherited case.
