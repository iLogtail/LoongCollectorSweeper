## Summary

Keep open. Current main has restart deferral and restart-continuation improvements, but the issue's core durable replay request is not implemented: followup queues are still in-process state, drain-window enqueues still fail with GatewayDrainingError, and that failure path surfaces a retry/restart message rather than persisting inbound work for replay. The related PRs #51623 and #51803 are unmerged and explicitly cover only Phase 1, leaving the broader issue unsolved.
