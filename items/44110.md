## Summary

Keep open. Current main still lacks a non-empty-tools guard before serializing `tool_choice` in the OpenAI transport paths, so the underlying empty-tools-plus-`tool_choice` bug remains plausible. The issue body's exact `tools.allow: []` reproduction is weaker on current main because empty allowlists are permissive here, but that makes the report need clarification, not closure.

## Evidence
