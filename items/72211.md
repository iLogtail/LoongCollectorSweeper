## Summary

Keep PR #72211 open. Current main still lacks the specific local-dashboard-after-update recovery runbook with local curl/lsof checks, LaunchAgent reload context, root dashboard URL recovery, and stale browser-state guidance. The PR is a focused docs improvement against real CLI behavior, but it should address the existing review comment by making restart/start conditional when the gateway is already serving HTML.

## Best Possible Solution

Keep the PR open and request a small docs revision: keep the local-dashboard recovery runbook, but structure it as check `lsof`/`curl` first, use the root dashboard URL if HTML is served, and run `openclaw gateway restart` or `openclaw gateway start` only when the gateway is not listening or not serving the dashboard. After that wording fix, this can proceed as a focused docs improvement.
