## Summary

Current main still misses the requested audit behavior. The Discord security audit only counts legacy `dm.allowFrom` plus the credential-store allowlist before emitting `channels.discord.commands.native.no_allowlists`; it still ignores top-level `channels.discord.allowFrom`, even though runtime slash-command authorization and docs treat that top-level field as valid.

## Best Possible Solution

Keep the issue open and fix the Discord audit in `extensions/discord/src/security-audit.ts` by including normalized top-level `discordCfg.allowFrom` in `ownerAllowFromConfigured`, then add a regression test showing `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists` while preserving the existing legacy `dm.allowFrom` behavior.
