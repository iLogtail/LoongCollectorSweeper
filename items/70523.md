## Summary

Keep open. Current main mitigates the linked daily-memory pollution issue by defaulting Dreaming storage to separate files, and #70509 was already closed on that basis. But #70523 targets the narrower supported inline/both storage path: current main still supports explicit inline mode, writes raw light/REM bodies into memory/YYYY-MM-DD.md, and has no narrative fallback result or suppression path. Related PRs overlap but are still open and not a clear canonical replacement.

## Best Possible Solution

Keep #70523 open for maintainer reconciliation. The best path is either a narrowly scoped memory-core change that prevents raw inline fallback dumps without transiently persisting sensitive raw blocks, or an explicit product decision to retire/deprecate inline/both dreaming output. If maintainers continue with this PR, strip unrelated docs changes and resolve the fallback/write-order review concerns.
