## Summary

Current main still returns the raw ClawHub 429 error without the requested login hint. The linked PR #56394 appears to track the exact fix, but it is still open/unmerged, so the issue should remain open until that implementation or an equivalent lands.

## Best Possible Solution

Keep the issue open and resolve it through PR #56394 or an equivalent main-branch change. The implementation should return whether `clawhubRequest` used auth, append a clear `clawhub login`/`npx clawhub login` tip only for unauthenticated 429s, cover fetch/search plus package and skill archive download throw paths, and add regression tests for hinted unauthenticated 429, suppressed authenticated 429, and non-429 errors.
