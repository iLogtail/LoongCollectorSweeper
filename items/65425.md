## Summary

Keep open. Current main still rejects `messages.groupChat.historyLimit: 0` in the runtime Zod schema and generated public schema, while current docs say `0` disables group history. The PR addresses a real remaining inconsistency, but its branch also contains unrelated memory-core changes and an unresolved review concern, so the best maintainer path is to keep the config/schema fix alive and ask for cleanup rather than auto-close it.

## Best Possible Solution

Keep the PR open, but request a focused branch containing only the `GroupChatSchema.historyLimit` `.min(0)` change, regenerated `schema.base.generated.ts`, and a regression test proving `0` is accepted and negatives are rejected. The unrelated memory-core search changes should be split into a separate PR or dropped from this one.
