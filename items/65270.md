## Summary

Keep open. Current main still lacks the central LINE sticker send path and `STICKER:` reply directive support; it only turns inbound LINE stickers into textual context. The PR is too broad for a clean merge as-is and the bundled `line-sticker` skill/catalog likely belongs on ClawHub unless maintainers explicitly promote it, but the underlying core/plugin and LINE outbound gaps are real enough that this should not be closed as implemented, duplicate, or clawhub-only.

## Best Possible Solution

Keep the item open, but ask for a split before merge: land a narrow core/plugin-SDK structured-payload seam with focused tests if maintainers want sticker payloads in the shared reply pipeline; land LINE send/receive support in the LINE plugin separately; move the `line-sticker` catalog/editor to a ClawHub skill or plugin unless maintainers explicitly approve bundling; drop unrelated Matrix/a2ui/Telegram changes from this PR.
