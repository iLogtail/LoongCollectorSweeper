## Summary

Keep open. Current main has generic headless-node/gateway plumbing, but it does not ship the requested Raspberry Pi/Brabble distributed voice-wake node. The repo still documents native Linux companion apps as planned, and current voice-wake docs only cover macOS/iOS wake words with Android on manual mic flow. This is also not a ClawHub-skill cleanup target: VISION scopes ClawHub to skills, while companion apps on Linux remain an OpenClaw roadmap item.

## Evidence

- **Linux companion app is still future work:** The Linux platform doc says the Gateway is supported on Linux, but native Linux companion apps are still planned and contributions are welcome. That is not an implemented Raspberry Pi/Linux voice node.
  - file: `docs/platforms/linux.md:1`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **Voice wake is not shipped for generic Linux/headless nodes:** The voice-wake doc says macOS and iOS use the global wake-word list, while Android voice wake is disabled and Android uses manual mic capture. There is no shipped Linux/headless wake-word node path documented here.
  - file: `docs/nodes/voicewake.md:1`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **Generic node substrate exists, but it is only groundwork:** The architecture doc says macOS/iOS/Android/headless nodes connect over WebSocket with `role: node`. That supports external implementations, but it is not the same as OpenClaw shipping Brabble-as-node support.
  - file: `docs/concepts/architecture.md:1`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **ClawHub scope does not fit this issue:** VISION puts companion apps on macOS/iOS/Android/Windows/Linux in OpenClaw's roadmap, while ClawHub is the default path for new skills. This request is a companion-node/platform feature, not a ClawHub skill.
  - file: `VISION.md:2`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
