## Summary

Keep PR #72085 open. The underlying docs gap still exists on current main: `commands.bashForegroundMs` is clamped/validated at 0..30000 ms in source and schema, while the user-facing slash-command and channel config docs still show only the default and `0` behavior. The submitted PR is not merge-ready because its one-file docs diff is far larger than the stated one-bullet change, but the requested documentation fix is real and belongs in this repo.

## Best Possible Solution

Keep the PR open and ask for a narrow docs cleanup. The best fix is to update `docs/tools/slash-commands.md` and the schema help/generated config docs so `commands.bashForegroundMs` is documented as an integer from 0 to 30000 ms, default 2000, with `0` backgrounding immediately. Optionally add the same concise range note to `docs/gateway/config-channels.md`. Avoid duplicating channel/config reference content inside `docs/gateway/configuration-reference.md`.
