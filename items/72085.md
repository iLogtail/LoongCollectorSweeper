## Summary

Keep PR #72085 open. The underlying documentation gap still exists on current main: runtime/schema enforce a 0..30000 ms `commands.bashForegroundMs` bound, while the user-facing slash-command and channel config docs still only mention the default and `0` behavior. The submitted branch is not merge-ready because it adds a large unrelated block to `configuration-reference.md`, but the requested docs fix is real and belongs in this repo.

## Best Possible Solution

Keep the PR open and ask for a narrow docs cleanup. The best fix is to update `docs/tools/slash-commands.md` plus the config schema help/generated docs so `commands.bashForegroundMs` is documented as an integer from 0 to 30000 ms, default 2000, with `0` backgrounding immediately. Optionally add the same concise bound note to `docs/gateway/config-channels.md`; avoid duplicating channel/config reference content inside `docs/gateway/configuration-reference.md`.
