## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata to the `message_sending` plugin hook, and the PR remains a real core plugin API gap rather than an obsolete or ClawHub-only item. The linked open issue #69110 is adjacent around provider/model identity but does not supersede the broader tool/token/session context bridge requested here.

## Best Possible Solution

Keep PR #39761 open for maintainer review or adaptation. The best path is a backward-compatible `agentContext` or equivalent field on `PluginHookMessageSendingEvent`, threaded through `DeliverOutboundPayloadsParams` and `applyMessageSendingHook()`, built from current embedded-runner metadata in agent delivery paths, documented in `docs/plugins/hooks.md`, and covered by focused hook contract tests. Coordinate provider/model identity with #69110 rather than closing this PR as obsolete.
