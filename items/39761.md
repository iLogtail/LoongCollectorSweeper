## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata on the `message_sending` plugin hook, and external plugins cannot add that missing core event field or thread embedded-runner metadata through outbound delivery themselves. The PR is a real core plugin API proposal, not obsolete and not ClawHub-only.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is to add a documented, backward-compatible `agentContext` field to the current `PluginHookMessageSendingEvent`, wire it through the shared outbound delivery params and `applyMessageSendingHook()`, derive it from embedded-runner metadata in agent delivery paths, and add focused hook contract tests. Coordinate provider/model identity and retry semantics with #69110 or split those into explicit follow-ups rather than closing this PR as obsolete.
