## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata on the `message_sending` plugin hook, and plugins cannot add that missing event field or delivery-threading themselves. The PR is a real core plugin API proposal rather than cleanup, ClawHub-only work, or a duplicate of #69110.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is a documented, backward-compatible `agentContext` (or equivalent) on `PluginHookMessageSendingEvent`, threaded through `DeliverOutboundPayloadsCoreParams` and `applyMessageSendingHook()`, derived from current embedded-runner metadata in agent delivery paths, and covered by focused hook contract tests. Coordinate model/provider identity and retry semantics with #69110 rather than closing this as obsolete.
