## Summary

Keep #69374 open. Current main still does not expose WhatsApp `contextInfo.forwardingScore` or `contextInfo.isForwarded` to `inbound_claim` or `message_received` plugin hooks. The Baileys source fields exist, but OpenClaw has no current extraction, normalized carrier, canonical hook field, or plugin hook metadata entry for them. Related PR #46291 is open and overlaps, but it targets inbound agent prompt context and does not currently update the hook mapper/types requested by this issue.

## Best Possible Solution

Keep the issue open and implement this as an additive bundled WhatsApp plus plugin-hook contract change. Extract `contextInfo.isForwarded` and `contextInfo.forwardingScore` during WhatsApp inbound normalization, carry them through `WebInboundMessage` and the canonical inbound hook context, expose them in `inbound_claim` and `message_received` metadata, and add focused WhatsApp extraction plus hook-mapper regression tests. PR #46291 may be a useful base if rebased and extended to cover plugin hooks.
