## Summary

Keep #9835 open. Current main still has Telegram inbound reaction events and outbound reaction guidance, but it does not implement the requested configurable two-way prompt guidance: there is no Telegram `reactionGuidance` config key, the strict schema only accepts `reactionNotifications` and `reactionLevel`, and the `## Reactions` prompt remains outbound-only rather than interpreting incoming user reactions as soft CTAs.

## Best Possible Solution

Keep the issue open. If maintainers want this feature, implement it in the Telegram plugin prompt/config path: add a documented `reactionGuidance` mode, define its relationship to `reactionLevel`, gate prompt injection on enabled inbound `reactionNotifications` plus guidance mode, add minimal/full guidance for sparse sending and incoming reaction interpretation, update schema/generated metadata/docs, and add focused tests for off/minimal/full behavior and notification gating.
