## Summary

Keep #9835 open. Current main still has Telegram inbound reaction events and outbound reaction prompt guidance, but not the requested configurable two-way Telegram prompt guidance: there is no Telegram `reactionGuidance` config option, prompt guidance is driven by `reactionLevel`, and the injected `## Reactions` text does not interpret incoming reactions as soft CTAs.

## Best Possible Solution

Keep the issue open. If maintainers want this feature, implement it in the bundled Telegram plugin/config prompt path: add and document `reactionGuidance`, define how it composes with `reactionNotifications` and `reactionLevel`, inject minimal/full incoming reaction guidance only when enabled, update generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
