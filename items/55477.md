## Summary

Do not auto-close PR #55477. Current main has related model-call diagnostic events and a context.assembled diagnostic event, but it does not stamp session_key, message_channel, or context_limit into outbound LiteLLM request options.metadata. The PR still addresses a real remaining integration gap, though it should be revised with tests and the contextTokenBudget nullish guard from review.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best path is a small, tested request-metadata wrapper near the final outbound streamFn path, or a deliberate extension of the provider transport metadata API so the LiteLLM provider can add session_key, message_channel, and context_limit without hardcoding LiteLLM-specific behavior in the runner. Add focused tests for metadata merge behavior and nullish contextTokenBudget handling.
