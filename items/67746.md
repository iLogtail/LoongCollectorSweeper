## Summary

Keep open. Current `main` still has the logout/remove gap the report describes: CLI logout is explicitly auth-only, WhatsApp logout only deletes credential files, and channel removal only edits config without reconciling the live gateway runtime. The same relevant files are unchanged from release `v2026.4.23` to `main`, so there is no shipped fix on the inspected path.

## Evidence

- **CLI logout is still auth-only:** `runChannelLogout` resolves the account and calls `plugin.gateway.logoutAccount(...)` directly. The in-file comment says `Auth-only flow: resolve account + clear session state only.` There is no `callGateway(..., method: "channels.logout")` reconcile path comparable to login.
  - file: [src/cli/channel-auth.ts:246](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/cli/channel-auth.ts#L246)

- **Live gateway logout exists, but the CLI path does not use it:** Gateway-side `logoutChannelAccount` stops the running channel first via `context.stopChannel(...)`, then calls the plugin logout hook and marks the runtime snapshot logged out. This is the behavior the report expects for active-memory cleanup, but it is only present in the gateway RPC path.
  - file: [src/gateway/server-methods/channels.ts:83](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/server-methods/channels.ts#L83)
