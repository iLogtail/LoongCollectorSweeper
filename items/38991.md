## Summary

Keep #38991 open. Current main mitigates the original macOS Cron tab failure by skipping malformed jobs in the app, but it still does not implement this PR's gateway/store-side repair for legacy cron jobs missing `createdAtMs` and/or `updatedAtMs`. Closing linked issue #38971 does not supersede this PR because malformed jobs can still remain hidden/skipped instead of being repaired and persisted.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch: repair missing or non-finite `createdAtMs` and `updatedAtMs` during cron store/service load, persist only when the store was mutated, and extend the regression to prove both `cron.list` and persisted storage expose numeric timestamps. Drop the unrelated secret-baseline/Fastfile changes before merge.
