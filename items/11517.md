## Summary

Keep #11517 open. Current main still has the import-heavy Pi embedded attempt runner and Discord monitor surfaces named in the issue, and the merged harness work is adjacent rather than a RunContext/AgentContext-style dependency aggregation for these files.

## Best Possible Solution

Keep #11517 open as the canonical tracker. The best path is an incremental maintainer-approved refactor that introduces a prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord monitor, preserves plugin SDK and extension boundaries, and adds focused tests around each extracted seam before any large mechanical file split.
