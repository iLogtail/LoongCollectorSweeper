## Summary

Keep #11517 open. Current main still has the import-heavy Pi attempt runner and Discord monitor files named in the issue, and the current harness/runtimePlan work is an adjacent prepared-attempt policy bundle rather than the requested RunContext/AgentContext dependency aggregation. The existing April 26 Codex/maintainer review comment also explicitly kept this item open.

## Best Possible Solution

Keep #11517 open as the canonical tracker. The best path is an incremental maintainer-approved refactor that introduces a prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord monitor, preserves plugin SDK and extension boundaries, separates pure moves from behavior changes, and adds focused tests around each extracted seam.
