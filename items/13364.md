## Summary

Keep #13364 open. Current main still exposes `before_tool_call` and `after_tool_call` through the typed plugin hook runner, but managed/internal `HOOK.md` hooks do not have a typed tool-call event, cannot return block/param decisions, and are not dispatched from tool execution. The latest maintainer discussion also explicitly identifies #13364 as the canonical product/API request, with #17008 closed as duplicate, #67085 related, and #29590 only an open partial implementation attempt.

## Best Possible Solution

Keep #13364 open for maintainer product/API judgment. A complete fix should add typed internal/managed `tool_call` before/after events, define blocking and param-rewrite return semantics for `before`, bridge tool execution without double-firing existing plugin hooks, handle unsupported or unreachable `HOOK.md` event names clearly, and update docs plus focused internal-hook tests. If maintainers decide this must remain plugin-only, close later with an explicit product decision and point users to plugin hooks and ClawHub/plugin packaging.
