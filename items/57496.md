## Summary

Keep open. Current main still does not reuse `models.providers.google.apiKey` or `models.providers.google.baseUrl` for Gemini web search, and the PR addresses a real remaining product gap also reflected by the related Gemini baseUrl issue context. The PR should not be closed as implemented or obsolete; it needs normal review/rebase, including the latest credential precedence concern from the PR review thread.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and finish review by preserving the documented precedence order: scoped Gemini web-search config first, `GEMINI_API_KEY` before `models.providers.google.apiKey`, and provider/webSearch base URL fallback without breaking SecretRef handling or auto-detection. The related #50213 user request can remain the product anchor for the baseUrl part until this or a successor patch lands.
