## Summary

Keep #35447 open. Current main still records cron job ownership on jobs and stamps ownership on cron.add, but list/update/remove/run/runs remain global because neither the cron service contract nor gateway handlers thread caller agent/session context or enforce ownership checks. The closed related PRs were not merged, and the open related items are narrower or broader rather than canonical replacements.

## Best Possible Solution

Keep #35447 open and implement a core cron caller-context contract across src/agents/tools/cron-tool.ts, src/agents/tools/gateway.ts, src/gateway/protocol/schema/cron.ts, src/gateway/server-methods/cron.ts, src/cron/service-contract.ts, and src/cron/service/ops.ts. Add focused tests proving list/listPage filtering plus update/remove/run/enqueueRun denial for jobs owned by another agent/session, with an explicit admin/owner override and documented legacy-job behavior. Track the sandbox execution comment separately if still reproducible.
