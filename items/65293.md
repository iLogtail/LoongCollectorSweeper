## Summary

Keep open. Current main still has the reported repair gap: doctor can invoke channel-specific stale cleanup only when it can resolve a read-only, loaded, setup, or bundled channel doctor adapter. For a deleted external channel plugin like `openclaw-weixin`, no adapter exists, so `channels.openclaw-weixin` is skipped. The separate stale-plugin repair still only scans and removes `plugins.allow` and `plugins.entries`, while config validation continues to reject unknown channel ids. The related open PRs mentioned in the issue context cover stale plugin refs or plugin-clean ergonomics, but they do not cover this dangling `channels.*` case on main.

## Best Possible Solution

Implement a generic doctor repair path for dangling configured channel ids whose owning plugin is no longer discoverable, with a regression test using an external channel id such as `openclaw-weixin`. The repair should remove or clearly disable `channels.<id>` when the corresponding plugin reference is pruned from `plugins.allow`/`plugins.entries`, while preserving valid installed plugin channels and explicitly disabled stale channel config behavior.
