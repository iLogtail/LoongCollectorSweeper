## Summary

本 PR 提议将 k8s.io/cri-api 依赖从 v1alpha2 升级至 v1，方向符合上游演进路线。但当前仅修改了单文件 import 路径，未同步更新 go.mod 与 go.sum，且基线过旧存在合并冲突。作为仍有价值的依赖升级工作，建议保持开启等待贡献者 Rebase 补全依赖，或由维护者在后续统一升级 K8s 依赖时接管。

## Best Possible Solution

建议贡献者将分支 Rebase 至最新 main，执行 go get k8s.io/cri-api@v1 及 go mod tidy 补全依赖锁文件，验证本地编译与 CI 通过后推送。若长期无响应，维护者可在后续统一升级 K8s 相关依赖或重构 CRI 适配层时手动完成此迁移，并引用本 PR 关闭。
