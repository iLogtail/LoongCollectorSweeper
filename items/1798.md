## Summary

本 PR 提议将 k8s.io/cri-api 依赖从 v1alpha2 迁移至 v1，方向符合上游 Kubernetes 演进路线。但当前实现仅修改了单行 import 路径，未同步更新 go.mod 与 go.sum，且基于较旧的 main 分支，直接合并会导致依赖不一致或编译失败。

## Best Possible Solution

建议贡献者将分支 Rebase 至最新 main，执行 go get k8s.io/cri-api@v1 并运行 go mod tidy 更新依赖锁文件，验证本地编译与 CI 通过后推送更新。若贡献者长期无响应，维护者可在后续统一升级 Kubernetes 生态依赖或重构 CRI 适配层时手动完成此迁移，并引用本 PR 关闭。
