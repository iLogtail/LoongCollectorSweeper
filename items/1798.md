## Summary

本 PR 提议将 k8s.io/cri-api 依赖从 v1alpha2 迁移至 v1，符合上游废弃 v1alpha 的演进方向。但当前改动仅替换了单文件 import 路径，未同步更新 go.mod 与 go.sum，且基线过旧存在合并冲突。主分支目前仍使用 v1alpha2，该升级工作仍有价值，需补全依赖变更并 Rebase 后方可合并，故保持开启。

## Best Possible Solution

建议贡献者将分支 Rebase 至最新 main，执行 go get k8s.io/cri-api@v1 及 go mod tidy 补全依赖锁文件，验证编译与 CI 后推送。若长期无响应，维护者可在后续统一升级 K8s 生态依赖时接管此迁移，并引用本 PR 关闭。
