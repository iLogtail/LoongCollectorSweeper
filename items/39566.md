## Summary

Keep open. Current main still lacks Telegram `topics."*"` fallback behavior for group or DM topic config; the PR remains a concrete, useful additive change, though it needs rebasing from old paths onto the current Telegram plugin layout.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch. The best implementation is to add exact-topic-then-`topics["*"]` resolution in the current `extensions/telegram/src/bot-core.ts` resolver, cover both fallback and exact-topic precedence in Telegram tests, document the public config behavior, and drop unrelated `.secrets.baseline` churn during rebase.
