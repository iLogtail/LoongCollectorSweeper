## Summary

Keep PR #39566 open. Current main still resolves Telegram group and DM topic config by exact message_thread_id only, so topics["*"] wildcard defaults are not implemented. The PR is stale against the current Telegram plugin layout and includes unrelated baseline churn, but the core behavior remains concrete and useful.

## Best Possible Solution

Keep the PR open and ask for a refreshed, focused patch against the current Telegram plugin. Implement exact-topic-then-topics["*"] fallback for both group and DM topic config resolution in extensions/telegram/src/bot-core.ts, add focused extension tests for wildcard defaults and exact-topic priority, update src/config/types.telegram.ts and the public Telegram docs, and drop unrelated baseline changes.
