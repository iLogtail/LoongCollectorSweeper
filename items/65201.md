## Summary

Keep #65201 open. Current main still has a plausible doctor false-positive path for a resolvable gateway.auth.token SecretRef: the doctor gateway-auth health check treats any still-SecretRef token as unavailable after resolveGatewayAuth returns no token, but that resolver does not actually resolve arbitrary SecretRefs from the configured secret provider. The linked PR #65229 is directly relevant but remains open/unmerged in the provided context, so it is not shipped evidence for closing the issue.

## Best Possible Solution

Keep the issue open until main changes runGatewayAuthHealth to resolve gateway.auth.token through the configured SecretRef/snapshot path before warning, with a focused regression test showing doctor stays quiet when the SecretRef is resolvable and still warns only when it is genuinely unavailable.
