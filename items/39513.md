## Summary

Keep open. Current main still has the behavior this docs PR documents: cron-triggered main-session heartbeat runs use a specialized cron-event prompt, bypass HEARTBEAT.md file-gate parsing, and current docs/templates still do not explain the HEARTBEAT.md caveat or workaround. The PR discussion also supports revision rather than closure: Greptile called the docs accurate/helpful and only raised style issues.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default HEARTBEAT.md instruction, add a clear workaround in docs/automation/cron-jobs.md, and make docs/reference/templates/AGENTS.md contain operational agent-facing cron handling. Any runtime behavior change should be tracked separately with prompt tests.
