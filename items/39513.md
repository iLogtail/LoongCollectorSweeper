## Summary

Keep open. Current main still has the behavior this docs PR documents: main-session cron system events bypass HEARTBEAT.md file-gate parsing and use the specialized cron-event prompt, while current scheduled-task docs and the AGENTS.md template still do not explain the HEARTBEAT.md caveat or workaround.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default HEARTBEAT.md instruction, add a clear workaround in docs/automation/cron-jobs.md, and make docs/reference/templates/AGENTS.md contain direct operational cron guidance for agents. Runtime behavior changes should be tracked separately with prompt tests.
