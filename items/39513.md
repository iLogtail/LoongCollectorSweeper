## Summary

Current main still needs this PR or an equivalent docs update. Main-session cron events continue to use the specialized cron-event prompt instead of the default heartbeat prompt, that prompt still omits the `Read HEARTBEAT.md` instruction, tests lock in the omission, and current docs/template do not contain the proposed clarification or workaround.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs patch. The best path is to document that main-session cron system events use a self-contained cron-event prompt that does not include the default `HEARTBEAT.md` instruction, add clear workaround guidance in `docs/automation/cron-jobs.md`, and make any `docs/reference/templates/AGENTS.md` guidance directly operational for agents. If maintainers prefer a runtime behavior change, that should be tracked separately with updated tests.
