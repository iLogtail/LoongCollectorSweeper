## Summary

Keep open. Current main still has the behavior this PR documents: main-session cron events use the specialized cron-event prompt, that prompt does not include the default `Read HEARTBEAT.md` instruction, and tests assert the omission. The user-facing cron docs and AGENTS template still do not include the proposed HEARTBEAT.md workaround, so the PR remains relevant documentation work rather than something already implemented or obsolete.

## Best Possible Solution

Keep tracking this PR or replace it with an equivalent maintainer-authored docs update. The best current path is to document that main-session cron system-event prompts are self-contained and do not automatically tell the agent to read `HEARTBEAT.md`, then add clear AGENTS.md/template guidance for users who want cron events to trigger HEARTBEAT.md-defined tasks. If maintainers prefer changing runtime behavior, that should be handled as a separate tested product change.
