## Summary

Keep #40982 open. Current main still has the reported 3-minute resume watchdog cap and does not implement the requested no-cap mode; the related fixing PR #41022 remains open/unmerged and PR #46050 was closed unmerged.

## Best Possible Solution

Keep the issue open until the watchdog defaults and optional disable semantics are fixed on main. The likely maintainer path is to revive or replace #41022 with a small patch that raises the defaults, decides whether `0`/`null` should disable the cap, updates `src/agents/cli-runner/reliability.ts`, `src/config/zod-schema.core.ts`, generated schema/docs if required, and adds focused regression coverage for long global timeouts and any no-cap configuration.
