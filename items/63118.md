## Summary

Keep open. Current main has partial session control surfaces (`/stop`, Control UI Stop, Gateway `sessions.abort`, session cleanup), but it does not implement the requested `openclaw sessions stop/pause/resume` CLI workflow, session-list control buttons, or pause/resume lifecycle management described in the issue.

## Evidence

  - file: [docs/cli/sessions.md:2](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/docs/cli/sessions.md#L2)

## Risks / Open Questions

- The issue bundles several different asks (CLI commands, web controls, pause/resume, quotas). Some parts are partially implemented through other surfaces, but the requested feature set is not fully present on `main`.
- `sessions.abort` and chat `/stop` may reduce urgency for the stop portion, but they are not equivalent to the requested `openclaw sessions stop/pause/resume` CLI and dashboard session-management flow.
