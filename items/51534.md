## Summary

Keep #51534 open. Current main still has Discord native reply support, cached @username mention rewriting, and inbound mention gating, but it does not implement the requested `channels.discord.autoMentionReply` behavior or any equivalent that prepends `<@originalAuthorID>` to outbound guild replies. The related ClawSweeper items are adjacent Discord reply/mention issues, not a canonical duplicate for this outbound notification feature.

## Best Possible Solution

Keep this open as a focused Discord plugin feature. The likely best implementation is a Discord-owned outbound formatting policy, gated by a typed config field, that prepends `<@originalAuthorID>` only on the first outbound guild reply chunk when the text does not already mention that user. It should carry the inbound author id when available, fall back to fetching the referenced message only for explicit/session sends that have a reply id but no author context, exclude DMs, preserve existing `replyToMode` behavior, and add schema/docs plus regression tests for text, media/chunking, component/webhook paths as applicable.
