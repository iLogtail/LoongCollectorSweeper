## Summary

Keep open. Current main does not implement `channels.discord.autoMentionReply` or equivalent Discord reply mention injection. The Discord send path rewrites only mentions already present in outbound text, then sends `message_reference` when `replyTo` is present; it does not resolve the referenced message author or prepend `<@userId>`. The public Discord docs/config surface also only documents `replyToMode`, not an auto-mention reply option. Ryce's issue comment is consistent with the remaining work: this would need to live in the Discord plugin's post-processing/send layer, and it is not already covered by the related ClawSweeper reports.
