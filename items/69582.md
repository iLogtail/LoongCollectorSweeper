## Summary

Current main has partial, adjacent protections, but it does not conclusively fix the reported GLM-5/local tool-argument contamination. The malformed-argument repair is narrow, edit/write validation still accepts strings ending in `{}`, the process tool does not runtime-type-check `action`, and known-tool loop blocking is opt-in.

## Best Possible Solution

Keep this open. Maintainers should ask for or add a focused regression covering the exact local GLM-5 tool-call payload shape, then fix the narrow failing path: schema-aware runtime validation for known tools such as `process.action`, a terminal/no-progress guard for repeated known-tool validation failures, and only provider-specific argument repair if captured GLM output proves a safe transformation. Avoid a global trailing-`{}` string stripper without stronger evidence.
