## Summary

Keep open. Current `main` has a shared `presentation` path for the message tool, but Feishu still degrades buttons/selects into markdown text inside a card instead of sending native interactive controls, and the send schema still does not expose a raw `card` field. The exact proposal to add provider-native `card` to the generic message tool conflicts with current architecture docs, but the underlying Feishu interaction gap remains real and is not implemented on `main` or in the latest release snapshot provided.
