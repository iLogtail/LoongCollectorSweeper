## Summary

Keep #65403 open. Current main has adjacent agent-to-agent building blocks (`sessions_send`, ping-pong/announce flow, per-agent routing, plugin subagent runtime), but it does not implement the requested teleconference lifecycle: dial/pickup/send/wrap/list/show/transcript verbs, meeting IDs, pending/active/closed state, or durable meeting transcript artifacts. The maintainer comment also explicitly points toward a narrow implementation PR rather than cleanup closure.

## Best Possible Solution

Keep the RFC open and ask for the narrow implementation PR described by the maintainer comment: a small lifecycle layer over the existing/canonical A2A handoff contract, with explicit meeting IDs, dial/pickup/send/wrap/list/show/transcript commands, durable pending/active/closed state, visible channel frames, and tests for state transitions and transcript rendering. Defer Google Meet/browser voice, dashboards, broad orchestration, and external A2A interop to separate work.
