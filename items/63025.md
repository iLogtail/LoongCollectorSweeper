## Summary

Keep open. Current `main` still has the zero-output completion path this PR targets: when model dispatch yields no sendable reply and no block/TTS fallback applies, `dispatchReplyFromConfig` can return with `queuedFinal=false` and zero final replies. The PR is a narrow, discussed fix that limits the fallback to direct/private chats and adds regression coverage, but that logic is not present on `main` or in the latest release.

## Evidence

If none of that fires, the function returns `queuedFinal` plus `counts` without queueing any final reply.
