## Summary

Keep open. Current main has overflow mitigations for Telegram and Discord, but it has not implemented the requested default change: `commands.nativeSkills` still defaults to `"auto"`, and Discord/Telegram provider metadata still turns native skill commands on under that auto default. The related default-flip PR #49849 was closed unmerged, while the warning-only follow-up #50191 is still open in the provided context.

## Best Possible Solution

Keep the issue open for maintainer product judgment. The concrete paths are either to flip the config/schema/docs/tests/baselines/changelog default to disabled (`false`, the current config vocabulary for “none”), or to explicitly supersede the request with a merged #50191-style fix that makes Telegram/Discord overflow warnings point users to `commands.nativeSkills: false` and verifies startup does not fail when skill counts exceed platform command limits.
