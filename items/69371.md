## Summary

Keep open. Current main contains related ARM/startup hardening, a fixed gateway handshake timeout path, and global cron retry/backoff, but the issue's central ARM edge-device stability requests are still only partially covered. The missing pieces include a lightweight/adaptive sessions/status path, the requested compaction high-water controls, message-send background retry/queueing, per-job cron retry overrides, and a sessions_send delivery receipt.

## Best Possible Solution

Keep the issue open, preferably split or track the remaining work as narrower maintainer-owned tasks: add a bounded lightweight sessions/status path for large session stores, decide whether compaction needs explicit high-water/interval config beyond existing overflow/near-limit behavior, add per-job cron retry overrides if global retry is insufficient, design durable queued message-send retry semantics, define what sessions_send delivery acknowledgment should mean, and either add doctor --edge-device or fold those checks into the existing ARM/low-memory doctor platform notes.
