## Summary

Keep issue #69371 open. Current main contains several adjacent reliability improvements, including a larger/overridable gateway handshake timeout, pre-prompt compaction checks, cron retry/backoff, outbound delivery queueing, and sessions_send run status. However the issue is a broad ARM64 edge-device stability request and the central requested CLI/session/status ergonomics are not fully implemented or superseded by a narrower canonical item.

## Best Possible Solution

Keep the issue open or split it into narrower maintainer-owned follow-ups: lightweight/compact sessions and status paths for large session stores, explicit compaction threshold policy if reserve-based prechecks are insufficient, per-job cron retry controls if needed, CLI background/retry semantics for message send, and a clearer sessions_send receipt contract. Treat the existing handshake, compaction, cron retry, and delivery queue work as partial progress rather than a complete resolution.
