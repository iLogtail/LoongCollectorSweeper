## Summary

Keep #71656 open. Current main has partial dreaming-corpus noise filtering and session ingestion cursor/budget improvements, and deep promotion ranking is weighted and recency-aware, but the issue's concrete flat source-score complaint is still present: daily ingestion still records every snippet at 0.62 and session-corpus ingestion still records every snippet at 0.58 before downstream ranking. The related scoring issue #57307 and PR #68876 cover adjacent parts, but neither fully supersedes this issue's combined source scoring, pre-filtering, and scan-rotation scope.

## Best Possible Solution

Keep the issue open and narrow it into concrete memory-core work: add source-level, content-type-aware scoring before daily/session results are passed to recordShortTermRecalls; extend corpus filtering/deduplication for repeated blocks and low-signal test/status messages; and add regression coverage proving session-corpus scans continue across files and days under budget pressure. Link #57307 for the broader importance/time-decay policy if maintainers want to share scoring design, and link PR #68876 as partial corpus-noise work rather than treating it as a complete replacement.
