## Summary

Keep #71656 open. Current main partially addresses session-corpus noise and scan progress, and deep promotion ranking is weighted/recency-aware, but daily and session dreaming ingestion still use fixed source scores (`0.62` and `0.58`) before promotion ranking. The issue is recent, unprotected, and still describes concrete remaining memory-core work.

## Best Possible Solution

Keep the issue open and narrow it into concrete memory-core work: add content-type-aware scoring before `recordShortTermRecalls` for daily and session dreaming ingestion, extend session-corpus filtering/deduplication for repeated blocks and low-signal diagnostic/status messages, and add budget-pressure regression coverage proving session corpus scans continue across files/days. Cross-link #57307 only for the broader importance/time-decay policy, not as a full duplicate.
