## Summary

Keep #71656 open. Current main has partial fixes for session-corpus noise, checkpointing, and downstream weighted/recency-aware promotion, but the central remaining complaint is still present: daily and session ingestion still seed fixed scores before promotion ranking, and session scan fairness is not proven against active-file budget starvation.

## Best Possible Solution

Keep this issue open and narrow it into implementation work: add content-type-aware scoring before daily/session `recordShortTermRecalls`, expand corpus filtering/deduplication for repeated blocks and low-signal status/test messages, and add a regression proving session-corpus ingestion rotates across active files under budget pressure. Link #57307 only for the broader importance/time-decay policy, and keep the corpus filtering plus scan-rotation specifics tracked here unless maintainers split them into focused follow-ups.
