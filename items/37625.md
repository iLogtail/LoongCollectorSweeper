## Summary

Keep #37625 open. Current main still lacks the requested Markdown code-fence lint/auto-fix utility, warn/auto-fix configuration, per-write skip option, and integration into workspace Markdown write paths. The related implementation PRs #37628 and #41207 were closed unmerged in the provided GitHub context, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, change records, warn/auto-fix configuration, per-write skip support, and focused coverage across current workspace write/edit/RPC paths, or explicitly decide OpenClaw should not mutate Markdown at write time.
