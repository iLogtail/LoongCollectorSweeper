## Summary

Keep #37625 open. Current main still lacks the requested write-time Markdown code-fence lint/fix utility, config option, per-write skip, and integration into workspace Markdown write paths; the related implementation PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep the issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, wire it into the canonical workspace write/edit/apply_patch paths with config and per-write skip controls, and add targeted tests/docs, or explicitly decide that OpenClaw should not mutate Markdown at write time.
