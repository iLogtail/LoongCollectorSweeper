## Summary

Keep open. Current `main` still contradicts the documented "ClawHub first, then npm" behavior for request failures: docs promise npm fallback for bare package specs, but the install path only falls back when ClawHub returns `package_not_found` or `version_not_found`. A generic ClawHub fetch failure still returns the error and exits instead of trying npm.

## Evidence
