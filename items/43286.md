## Summary

Keep open. Current main still does not expose `session.threadIsolation` or `session.channelIsolation`, Slack thread routing still calls `resolveThreadSessionKeys()` without a config-controlled `useSuffix: false`, and the related implementation PRs #43360 and #43459 are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #43286 open as the focused Slack shared-thread-session request. A likely fix is an opt-in session config, threaded through config types/zod/help/labels/docs, Slack inbound and outbound session resolution, per-thread history keys, and first-thread tracking such as `seenThreadIds`, with regression tests for default isolated behavior and `threadIsolation: false`. The closed PRs #43360 and #43459 remain useful design input but do not close the issue.
