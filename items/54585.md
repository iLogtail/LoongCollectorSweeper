## Summary

Keep open. Current main still only supports compaction modes `default` and `safeguard`; it has pre-prompt overflow recovery, but not the PR's user-controlled `warn`, `error`, or `none` modes or their documented semantics. The PR discussion is also still materially relevant: reviewer comments identify unresolved correctness issues in the PR patch, including the latest P1 that warn mode emits a warning but still submits the prompt.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to revise it against current main: extend the config type, Zod/generated schema, help/docs, and tests; implement `warn`, `error`, and `none` through the existing pre-prompt/overflow recovery flow; preserve llm_input/llm_output/agent_end and context-engine epilogue semantics; ensure `none` disables all automatic compaction paths; and address the unresolved P1/P2 review comments before merge consideration.
