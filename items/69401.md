## Summary

Keep #69401 open. Current main has useful adjacent protections, but the central request is not implemented generically: inbound channel message bodies still reach the agent prompt as ordinary user text for key bundled channels, while only metadata and some supplemental Discord context are wrapped as untrusted. The memory/session path also strips inbound metadata for Dreaming ingestion and skips only inter-session provenance, so the requested external-channel provenance/filtering remains real product/security work.

## Best Possible Solution

Keep the issue open and implement a shared inbound-channel trust boundary: wrap `BodyForAgent` or the final prompt body for all external channel turns with a nonce/random boundary using the existing external-content sanitizer, escape or neutralize boundary breakouts, attach `InputProvenance` such as `external_user` with source channel/session/sender where available, and carry that provenance through session memory/Dreaming so retrieval and promotion can distinguish untrusted inbound content. Add focused tests for Slack, Telegram, Discord, prompt assembly, and memory ingestion.
