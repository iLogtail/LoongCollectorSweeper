## Summary

Keep open. Current main still does not propagate any TLS fingerprint through the shared operator approvals GatewayClient path, while the normal gateway call path already has the local/remote TLS fingerprint resolver that this PR is trying to mirror. The PR discussion and related #41741/#41740 context describe a real approvals-over-local-self-signed-TLS gap; the remaining work is to adjust the PR for the remote-mode local-fallback review comment rather than close it as obsolete.

## Best Possible Solution

Keep the PR or its linked bug open and land a corrected version: add TLS fingerprint resolution to src/gateway/operator-approvals-client.ts using the same semantics as src/gateway/call.ts, pass tlsFingerprint into GatewayClient, and add focused operator-approvals TLS tests for local WSS, env override remote pinning, CLI override skip, and remote-mode missing-url local fallback. The related #41741/#41740 discussion is useful reproduction/context, but current main still lacks the implementation.
