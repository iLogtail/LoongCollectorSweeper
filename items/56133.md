## Summary

Keep open. Current main still has the transcript-header regression that #56133 targets: three session transcript creation paths write `cwd: process.cwd()` even when an agent id is known, while the current agent workspace resolver already exists and returns the configured/default workspace. The PR is a focused, still-relevant gateway fix, not implemented on main and not covered by the provided related items.

## Best Possible Solution

Keep #56133 open and either land it after review or revise it so all transcript-header creation paths receive the relevant `cfg` and `agentId` and write `resolveAgentWorkspaceDir(cfg, agentId)` into the session header, with regression coverage for sessions.create, DM/appendAssistantMessageToSessionTranscript, and chat-created transcript creation.
