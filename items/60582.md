## Summary

Keep #60582 open. Current main still does not support Discord category-level guild channel config for normal guild messages: channel metadata can include a Discord parent/category ID, but the inbound preflight path only passes thread-parent data into channel config fallback. The related #8972 is open for glob/pattern allowlists and mentions category-based listening as an alternative, but it does not clearly supersede this more specific category-inheritance request.

## Best Possible Solution

Implement this in the Discord plugin/config surface: resolve normal guild channel category parent IDs, add a documented category key form or `categories` block, define precedence as direct channel override before category before wildcard, and update Discord types, Zod schema, docs, status/audit handling, and regression tests. Coordinate with #8972 only if maintainers choose to merge category inheritance with broader pattern-based allowlists.

## Evidence

"thread" : "channel"`. For ordinary guild channels under a Discord category, `threadChannel` is false and `threadParentId` is unset, so the category parent ID is not considered.
