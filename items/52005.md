## Summary

Keep #52005 open. Current main has useful Discord PluralKit support and exact bound-thread webhook echo suppression, but it does not implement the reported original/proxy collapse for PluralKit messages with different Discord message IDs. The linked fix PR #52007 is closed unmerged in the provided GitHub context, and current tests explicitly preserve the narrower behavior where differing webhook IDs are not suppressed.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin’s inbound/preflight boundary. Prefer an explicit PluralKit original/proxy canonicalization signal before session dispatch; if Discord/PluralKit metadata cannot provide that reliably, add the narrow bound-thread agent-session content-clone fallback described in the issue, plus regression tests proving one run, one reply, and no lingering typing indicator for original/proxy pairs with different message IDs.
