## Summary

Keep open. Current `main` now drops plain non-self-chat `is_from_me=true` rows, but the broader iMessage self-echo path described in this issue is not clearly fixed: outbound echoes are cached under the outbound `target`, while inbound dedupe still looks up by inbound `sender`. That sender/target scope mismatch means the non-self-chat echo variant discussed in the issue comments remains plausible on current `main`, and the tests only cover self-chat / same-scope cases.

## Evidence
