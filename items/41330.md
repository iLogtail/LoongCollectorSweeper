## Summary

Do not close #41330 yet. Current main has partial iMessage echo-loop protections for `is_from_me`, self-chat, reflected assistant markers, and sent-message cache matching, but the broader non-self echo path discussed in the issue is not fully proven fixed. The code still scopes outbound text echo cache entries by delivery target and inbound lookups by inbound sender, and text-key normalization remains CR/trim-only with a 4s text TTL, matching the open follow-up concerns around delayed or garbage-prefixed reflected iMessage text.

## Best Possible Solution

Keep #41330 open as the broad iMessage echo-loop tracker until maintainers land and verify a fix that covers the remaining non-self/account-scope and garbage-prefixed/delayed reflected-message paths. If #59973/#62191 become the accepted canonical remaining work, add a maintainer comment linking them and close #41330 only after confirming the account-scope mismatch is also covered or intentionally out of scope.
