## Summary

Keep open. Current `main` still routes the default-agent model editor through `agents.list[index].model`, so the reported bug is still present rather than implemented on `main` or obsolete.

## Evidence

- **Default-agent field is still presented as a defaults editor:** The Overview panel labels the selector as `Primary model (default)` when the selected agent is the default agent, and changes go through the shared `onModelChange(agent.id, ...)` callback.
  - file: [ui/src/ui/views/agents-panels-overview.ts:142](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/ui/src/ui/views/agents-panels-overview.ts#L142)
  - command: `nl -ba ui/src/ui/views/agents-panels-overview.ts | sed -n '138,157p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Model edit path is hardcoded to per-agent override:** `resolveAgentModelFormEntry()` always returns `basePath: ["agents", "list", index, "model"]` and reads existing data from `agents.list[index].model`; there is no branch for the default agent or `agents.defaults.model`.
  - file: [ui/src/ui/app-render.ts:774](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/ui/src/ui/app-render.ts#L774)
