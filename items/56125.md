## Summary

Current `main` has manual/session-scoped and context-specific model overrides, but not the requested first-class per-task or skill/complexity-based routing rules. The issue remains a valid feature request.

## Evidence

- **Core model config only supports primary plus fallbacks:** `AgentModelConfig` is just a string or `{ primary, fallbacks }`, and `AgentDefaultsConfig.model` uses that shape. There is no native `routing.rules` or match-by-skill/task schema in the current config types.
  - file: [src/config/types.agents-shared.ts:8](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/types.agents-shared.ts#L8)
  - command: `nl -ba src/config/types.agents-shared.ts | sed -n '1,20p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Docs describe model selection as primary -> fallbacks -> provider auth failover:** The user-facing model-selection docs only document primary model selection, ordered fallbacks, and provider auth failover. They do not describe any native task classifier, skill match rules, or per-turn automatic routing policy.
  - file: [docs/concepts/models.md:14](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/concepts/models.md#L14)
  - command: `nl -ba docs/concepts/models.md | sed -n '8,70p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current chat UX is manual per-session switching:** `/model` changes the model for the current session and persists that session selection. That is a manual session override, not automatic routing based on task complexity or selected skills.
  - file: [docs/concepts/models.md:119](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/concepts/models.md#L119)
  - command: `nl -ba docs/concepts/models.md | sed -n '118,150p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Existing automatic override is channel-scoped, not task-scoped:** OpenClaw documents `channels.modelByChannel` to pin a channel or topic to a model when there is no session override. This is per-channel routing only.
  - file: [docs/gateway/config-channels.md:44](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/gateway/config-channels.md#L44)

  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Runtime applies stored session override first, then channel override:** The reply path resolves a stored session model override and only falls back to `channels.modelByChannel` when there is no session override. There is no corresponding runtime branch for skill-based or complexity-based model routing.
  - file: [src/auto-reply/reply/get-reply.ts:315](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/auto-reply/reply/get-reply.ts#L315)

- **Other scoped model overrides exist, but they are narrow feature-specific overrides:** Hook mappings and Gmail hooks each support a single explicit `model` override field. That shows current `main` supports some targeted override points, but still not a general first-class routing rules engine.
  - file: [src/config/types.hooks.ts:11](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/types.hooks.ts#L11)

## Risks / Open Questions

- There are partial workarounds on `main` (session `/model`, channel model pins, hook/Gmail overrides, upstream proxy routing via docs like LiteLLM/OpenRouter), so the eventual solution may be documentation or composition guidance rather than a new core schema.
- GitHub follow-up context was limited because `gh` could not authenticate in this environment, so duplicate/roadmap linkage beyond the provided cross-reference could not be inspected further.
