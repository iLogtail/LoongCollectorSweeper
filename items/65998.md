## Summary

Keep open: current `main` still applies runtime `toolsAllow` only to the core tool list in `src/agents/pi-embedded-runner/run/attempt.ts`, then materializes and appends bundled MCP/LSP tools without that runtime allowlist. That matches the reported Active Memory failure mode rather than proving it fixed.

## Evidence

- **Active Memory still relies on a narrow runtime allowlist:** The plugin invokes `runEmbeddedPiAgent` with `toolsAllow: ["memory_search", "memory_get"]`, so the reported behavior depends on the runner enforcing that allowlist across every tool source.
