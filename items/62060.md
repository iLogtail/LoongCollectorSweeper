## Summary

Keep open. Current main still uses and documents a 10,000 ms pre-auth/connect-challenge timeout, while PR #62060 specifically proposes raising the default to 15,000 ms for slower local arm64 hosts. Main has the earlier 10s/runtime-override fix, but the requested 15s default remains missing.
