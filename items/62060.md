## Summary

Keep open. Current main still uses and documents a 10,000 ms pre-auth/connect-challenge timeout, while PR #62060 specifically proposes raising that default to 15,000 ms for slower local arm64 hosts. The earlier shipped fix addressed the 3s timeout by moving to 10s plus a runtime override; it does not implement or supersede this PR's 15s default request.

## Best Possible Solution

Keep the PR open for maintainer product review. If maintainers agree 10s plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS` is still too tight for slower local hosts, the best implementation is a focused change to set `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS` to `15_000`, keep the runtime override, update `docs/gateway/protocol.md` to the new default/clamp, and add the invalid-override regression coverage proposed by the PR. If maintainers decide the default should remain 10s, close with that explicit product decision rather than as already implemented.
