## Summary

Keep open. The PR is not obsolete against the checked-out main SHA: the announce delivery path still routes normalized assistant output through `runtime.log(output)`, which reaches `console.log` and then file logging at info level when console capture is enabled. The linked bug report was closed, but I did not find an equivalent implementation or regression test on current main.

## Best Possible Solution

Keep the PR open or recreate the same narrow fix on current main: add announce-run detection in `src/agents/command/delivery.ts`, route info-level announce logging to stable metadata through `createSubsystemLogger("agents/announce")`, keep full body/debug-only diagnostics out of default file logs, move announce warning/error logging off `defaultRuntime`, and retain the strengthened regression test that `runtime.log` is not called for announce payloads.
