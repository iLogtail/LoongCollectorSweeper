## Summary

Keep open. Current main has retry support for one-shot cron jobs and recurring-job error backoff, but it still does not implement the requested configurable execution retry for recurring cron jobs that re-runs a failed daily/weekly job after a delay before the next scheduled slot.

## Evidence

- **Recurring cron errors wait for the later of natural next run and backoff:** The recurring error branch computes normalNext, computes backoffNext, then sets nextRunAtMs to the later value. For a daily cron failure, the natural next run is normally the next day, so a short backoff does not create a same-day retry.
  - file: [src/cron/service/timer.ts:510](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/cron/service/timer.ts#L510)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '463,548p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Actual retry scheduling is one-shot only:** The branch that schedules nextRunAtMs to endedAt + backoff is gated by schedule.kind === "at" and transient error classification.
  - file: [src/cron/service/timer.ts:467](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/cron/service/timer.ts#L467)

- **Public configuration docs scope cron.retry to one-shot jobs:** The configuration reference states maxAttempts is for one-shot jobs and explicitly says cron.retry applies only to one-shot cron jobs; recurring jobs use separate failure handling.
  - file: [docs/gateway/configuration-reference.md:1110](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/docs/gateway/configuration-reference.md#L1110)
