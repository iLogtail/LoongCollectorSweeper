## Summary

Keep open. Current `main` only applies error backoff floors to recurring cron jobs; it does not reschedule a failed recurring execution for a separate retry attempt after a configurable delay, and the proposed per-job retry CLI/config surface is not present.

## Evidence

- **Recurring-job error path uses natural next run or backoff floor, not a dedicated retry run:** For non-`at` jobs, `applyJobResult` computes the normal next scheduled time and then sets `nextRunAtMs` to `max(normalNext, backoffNext)`. For daily/weekly jobs, that means the next scheduled slot still wins when it is later than the short backoff window, so the failed run is not re-executed shortly after failure.
  - file: [src/cron/service/timer.ts:510](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/cron/service/timer.ts#L510)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '460,550p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Docs describe recurring behavior as backoff, not rerun of failed execution:** The user-facing CLI docs say recurring jobs use exponential retry backoff after consecutive errors and then return to the normal schedule after the next successful run. The automation docs likewise distinguish one-shot retry from recurring backoff.
  - file: [docs/cli/cron.md:39](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/docs/cli/cron.md#L39)
  - command: `nl -ba docs/cli/cron.md | sed -n '20,45p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Config retry type is scoped to one-shot jobs:** `CronConfig.retry` is documented in code as an override for one-shot jobs on transient errors. There is no separate recurring execution-retry config with per-job delay arrays or retry reasons.
  - file: [src/config/types.cron.ts:34](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/config/types.cron.ts#L34)

  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Existing tests assert backoff gating rather than immediate recurring reruns:** Restart catch-up tests explicitly preserve a future `nextRunAtMs` during error backoff and only replay a missed cron slot once that backoff window has elapsed. That matches the current backoff-floor design, not the requested `pending_retry` execution flow.
  - file: [src/cron/service.restart-catchup.test.ts:258](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/cron/service.restart-catchup.test.ts#L258)
