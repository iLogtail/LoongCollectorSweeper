## Summary

Keep open. Current main still does not implement the Anthropic billing wording from this PR, and the duplicate-reply half is still unresolved for multi-handle same-person targets. The PR patch itself also needs rework because its unconditional text dedupe conflicts with current cross-target delivery behavior and tests.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. Split the fix into two narrow, tested changes: add a precise Anthropic billing matcher/classification path for the "specified API usage limits" + "regain access" wording while preserving periodic usage-window rate_limit behavior, and solve multi-handle duplicate suppression through identity-aware target matching or a channel/plugin alias seam that preserves current cross-target same-text delivery tests.
