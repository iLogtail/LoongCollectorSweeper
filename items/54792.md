## Summary

Keep open. Current `main` partially mitigates the report by documenting the requirement and throwing a clear runtime error when `hooks.enabled=true` without `hooks.token`, but the requested schema/write-time dependency validation is still not implemented: `config.patch`/`config.apply` do not enforce this dependency before persisting config, and the generated schema/lookup surface still does not encode it.

## Evidence

- **Config writes do not enforce hooks dependency before save:** `config.patch` merges the patch, runs `validateConfigObjectWithPlugins(...)`, then SecretRef preflight, and writes the config. There is no check here for `hooks.enabled === true` requiring `hooks.token` before persistence.
  - file: [src/gateway/server-methods/config.ts:516](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/config.ts#L516)
  - command: `nl -ba src/gateway/server-methods/config.ts | sed -n '470,620p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Runtime hook config does throw a clear error:** `resolveHooksConfig` now throws `hooks.enabled requires hooks.token` when hooks are enabled without a token. This is a clear runtime/startup error, but it is not schema-time validation.
  - file: [src/gateway/hooks.ts:49](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/hooks.ts#L49)
  - command: `nl -ba src/gateway/hooks.ts | sed -n '1,140p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Gateway startup still consumes that runtime validation:** Gateway runtime config construction calls `resolveHooksConfig(params.cfg)` during startup, so a persisted config with `hooks.enabled=true` and no token can still fail boot rather than being rejected at patch time.
  - file: [src/gateway/server-runtime-config.ts:121](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-runtime-config.ts#L121)
  - command: `nl -ba src/gateway/server-runtime-config.ts | sed -n '90,145p'`

- **Schema/help surface still lacks dependency metadata:** The schema help/generated entries for `hooks.enabled` and `hooks.token` provide labels/help text, but there is no machine-readable dependency metadata indicating that enabling hooks requires `hooks.token`.
  - file: [src/config/schema.help.ts:1405](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/config/schema.help.ts#L1405)

## Risks / Open Questions

- Users can still persist a config that enables hooks without `hooks.token`, leaving a latent startup failure for the next full restart.
- `config.schema.lookup` remains insufficient for tooling/UI to warn about this dependency before a write.
- The clearer runtime error and docs may reduce confusion, but they do not fully resolve the issue as filed.
