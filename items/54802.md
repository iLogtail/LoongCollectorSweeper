## Summary

Keep PR #54802 open. Current main has not implemented the requested behavior: the active conversation-label and TTS summarization helpers still embed instruction text in the user message and always pass temperature, while the Codex Responses transport only derives top-level instructions from context.systemPrompt. The PR is stale against refactored file names, but it addresses a real remaining behavior gap rather than cleanup that can be closed.

## Best Possible Solution

Keep the PR open or ask the author/maintainer to rebase it onto current main by porting the fix to `src/auto-reply/reply/conversation-label-generator.ts` and `src/tts/tts-core.ts`: pass instructions through `systemPrompt`, keep user content as user data, omit `temperature` for `openai-codex-responses`, handle `stopReason === "error"`, and add current-path regression coverage in the conversation-label and TTS/speech-core tests. Also consider adopting the review comment's clearer fallback error for TTS unknown completion errors.
