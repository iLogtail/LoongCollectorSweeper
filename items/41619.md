## Summary

Keep open. The issue is eligible for review, but current main does not confidently implement the macOS Gemini CLI OAuth callback workaround that was previously linked from the issue. The current google plugin still sends local macOS through the localhost callback path, while the referenced fix explicitly changed the old plugin to use manual paste flow on macOS.

## Evidence

- **Current main still uses automatic localhost flow for local macOS:** `shouldUseManualOAuthFlow` only returns true for remote environments or WSL2. On a local macOS install, `isRemote` is false and `isWSL2Sync()` is false, so the flow still uses localhost callback instead of the manual paste workaround.
  - file: [extensions/google/oauth.flow.ts:13](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/google/oauth.flow.ts#L13)
  - command: `nl -ba extensions/google/oauth.flow.ts | sed -n '1,260p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Current main opens browser then waits on localhost:8085:** For non-manual flow, `loginGeminiCliOAuth` tells the user the callback will be captured on localhost:8085, calls `ctx.openUrl(authUrl)`, then waits for `waitForLocalCallback`. The fallback to manual mode only covers callback server listen/port failures, not the macOS callback/drop case described by the linked fix.
  - file: [extensions/google/oauth.ts:19](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/google/oauth.ts#L19)
  - command: `nl -ba extensions/google/oauth.ts | sed -n '1,180p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Callback server binds localhost:8085 only in automatic flow:** `waitForLocalOAuthCallback` creates the HTTP server and listens on the requested port/host. There is no current-main special case here that makes local macOS use manual URL paste instead of the local listener.
  - file: [src/plugin-sdk/provider-auth-runtime.ts:58](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/plugin-sdk/provider-auth-runtime.ts#L58)

Its patch changed `shouldUseManualOAuthFlow` to include `process.platform === "darwin"` in the old `extensions/google-gemini-cli-auth/oauth.ts` path.
