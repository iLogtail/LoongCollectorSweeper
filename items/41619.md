## Summary

Keep #41619 open. The issue is a real Gemini CLI OAuth regression report with multiple confirmations, and current main still uses automatic localhost:8085 capture on local macOS instead of the manual-paste fallback that the referenced fix commit claimed was needed for this exact issue. Related reports/PRs cover narrower symptoms, but they do not make this item safely closeable.

## Best Possible Solution

Keep the issue open and fix the current bundled Google plugin path. The likely maintainer path is to restore an explicit macOS manual-paste fallback or otherwise make the OAuth URL/manual path reliable before waiting on localhost:8085, add focused tests in extensions/google/oauth.test.ts for the platform/manual-flow decision, and coordinate separately with the proxy-specific work in #46184/#47518.
