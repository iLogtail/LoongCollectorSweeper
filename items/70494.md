## Summary

PR #70494 is superseded by merged PR #71871. The reopened PR targeted a real task-audit false positive from #70203/#70216, but current main now fixes the root cause at the task registry boundary by normalizing lifecycle timestamps on create, update, and restore, with regression coverage showing `inconsistent_timestamps` remains zero for earlier `startedAt` cases. The later implementation also preserves strict audit checks for real timestamp-ordering violations instead of adding a blanket audit-side jitter tolerance.

## Best Possible Solution

Close #70494 as superseded by merged #71871. Keep the current registry-boundary normalization as the canonical fix, since it resolves the false-positive audit noise without adding an audit-side jitter exemption that could hide larger ordering problems.
