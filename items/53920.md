## Summary

Keep #53920 open. Current main still has the reported bug: `scripts/setup-auth-system.sh` mutates the tracked auth-monitor systemd template before copying it, and the tracked service template still hardcodes `/home/admin/openclaw/scripts/auth-monitor.sh`. The PR is a focused scripts fix for a real behavior gap, but the latest review context still leaves a material idempotency concern for templates already mutated by the old setup flow, so it should be revised rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: copy the service template first, render only the installed user unit, set `ExecStart` to the current checkout's `scripts/auth-monitor.sh`, and make the notification placeholder handling idempotent for both pristine commented placeholders and templates already mutated by prior setup runs. If this PR stalls, implement the same fix directly in `scripts/setup-auth-system.sh` with verification that running setup does not dirty the repository template.
