## Summary

Keep #53920 open. Current main still has the reported behavior: `scripts/setup-auth-system.sh` runs `sed -i` against the tracked `scripts/systemd/openclaw-auth-monitor.service` template before copying it into the user systemd directory, and the template still contains a hard-coded `/home/admin/openclaw/scripts/auth-monitor.sh` `ExecStart`. The PR is a focused fix for a real scripts bug, but the latest review context still includes an unresolved idempotence concern for previously mutated templates, so it should be revised rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: copy the service template first, render only the installed user unit, set `ExecStart` to the current checkout's `scripts/auth-monitor.sh`, and make the renderer idempotent for both pristine commented placeholders and templates already mutated by prior setup runs. If the PR is not continued, the same fix should be implemented directly in `scripts/setup-auth-system.sh` with focused verification that the repository template remains unchanged after setup.
