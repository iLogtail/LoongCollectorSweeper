## Summary

Keep this PR open. Current main still has the core gaps this PR is trying to address: self-hosted provider model discovery uses an unguarded fetch, typed plugin hook priorities are stored without an upper bound even though hooks run highest-priority first, and the built-in skill scanner still has only the narrower baseline rule set. The provided review context also shows the author responded to concrete review feedback by removing public PoCs, fixing the guarded-fetch release path in the PR, correcting bundled-origin detection, and dropping the no-op device-token change, so this is not a conservative cleanup close candidate.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a narrower maintainer-owned security PR. The useful path is to split or revise the remaining three changes: harden `discoverOpenAICompatibleLocalModels` with the existing guarded-fetch seam and precise self-hosted policy, add contract-tested upper bounds for external plugin hook priorities while preserving bundled and negative-priority semantics, and add scanner rules with regression tests that cover both bypasses and benign code. Close only after those fixes land on main or a canonical superseding security PR is opened.
