## Summary

Keep #60506 open. Current main has no `routeTo` support and `agents.defaults.models` entries still only accept `alias`, `params`, and `streaming`, so the derived/virtual model-ref behavior proposed by the PR is not implemented or superseded. The related #31278 only overlaps the context-token part, and current shipped `contextTokens` support is for provider model metadata/global/per-agent paths, not per `agents.defaults.models["provider/model"]` variants. The PR also has review-noted correctness issues that should be addressed before merge rather than closed as obsolete.

## Best Possible Solution

Keep the PR open or replace it with a corrected implementation. The best path is to add `routeTo` and per-entry `contextTokens` to the `agents.defaults.models` config contract, generated schema/help/docs, model selection, runtime resolution, and context-budget resolution, with tests covering selectable derived refs, cross-provider dynamic preparation, cycle handling, params inheritance semantics, and model picker/list behavior.
