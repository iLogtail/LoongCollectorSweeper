## Summary

Keep #14850 open. Current main has the narrower binding-level `session.dmScope` override for direct messages, but it still does not provide the requested binding-level `sessionKey`, `sessionAlias`, or `sessionScope` target for non-direct Discord guild/channel bindings. Channel peers still resolve to isolated channel session keys, and the current docs/tests preserve that behavior.

## Best Possible Solution

Keep the issue open until maintainers either add and document an explicit binding-level session target for non-direct route bindings, or make a product decision to reject channel-to-main session unification. The likely implementation path is `AgentRouteBinding`, `RouteBindingSchema`, `resolveAgentRoute`, and `buildAgentPeerSessionKey`, with tests proving Discord guild/channel bindings can intentionally share an agent main or aliased session while preserving isolated defaults.
