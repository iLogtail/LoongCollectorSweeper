## Summary

Keep open. Current `main` does not implement the proposed `commands.shellProfile` feature: the command runtime still hardcodes default shell invocation without profile sourcing, and the config schema/types/docs in `main` do not expose a `commands.shellProfile` setting. This PR is therefore not implemented on `main`, and none of the allowed conservative auto-close reasons fit.
