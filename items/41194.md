## Summary

Keep open. Current `main` at `e94c0bf51542fe518e49ad4e2ee29882891b5603` still has the reported control flow: `updateCommand` keeps `shouldRestart` true when `resolveGatewayService().isLoaded()` returns false or throws, then `maybeRestartService` falls through to `runDaemonRestart()`, whose `onNotLoaded` path explicitly tries to restart/start the gateway without a loaded service manager. The latest tagged release `v2026.4.22` still contains the same code.

## Evidence

- **Pre-update service check does not disable restart on unloaded service:** `updateCommand` checks `resolveGatewayService().isLoaded()`, but if `loaded` is false it only leaves `restartScriptPath` null and `refreshGatewayServiceEnv` false; it does not change `shouldRestart`, and the empty catch also preserves the fallback path.

- **Post-update path still falls back to daemon restart:** `maybeRestartService` restarts whenever `params.shouldRestart` is true. If no `restartScriptPath` was prepared, it directly executes `runDaemonRestart()`.
