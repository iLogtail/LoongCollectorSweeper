## Summary

Keep #14344 open. Current main still does not implement WhatsApp message delete/revoke: the bundled WhatsApp plugin advertises only `react`/`poll`, accepts only `react` for plugin-dispatched actions, the active WhatsApp listener API has no delete/revoke helper, and user-facing docs exclude WhatsApp from the `delete` action. The related #38723 is closed and does not supersede this older, still-open canonical request.

## Best Possible Solution

Keep #14344 open as the canonical WhatsApp delete/revoke feature request. The implementation should live in the bundled WhatsApp plugin: advertise `delete`, accept it in `supportsAction`, route it through the gateway-owned WhatsApp action runtime, add an active-listener delete/revoke helper using the Baileys message key, validate bot-owned/fromMe deletion and group participant keys, then add focused action/send tests and update `docs/cli/message.md` plus `docs/channels/whatsapp.md`.
