## Summary

Keep PR #72138 open. Current main still lacks Feishu normal-reply `message_sent` and internal `message:sent` emission, while the shared outbound path already emits those hooks for generic delivery. The PR remains a focused Feishu-owned fix for the linked bug, with review comments to resolve before merge rather than a cleanup-close reason.

## Best Possible Solution

Keep PR #72138 open, or land an equivalent Feishu-owned fix. The best implementation path is to emit canonical plugin `message_sent` and internal `message:sent` from `extensions/feishu/src/reply-dispatcher.ts`, pass routed session key and group mirror metadata from `extensions/feishu/src/bot.ts`, include streaming-card message identity where available, avoid false failure hooks for partially delivered streaming text, and cover plain text, card, media, streaming finalization, and failure cases in Feishu tests.
