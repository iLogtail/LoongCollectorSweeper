## Summary

Keep open. Current main has Slack live directory helper functions and user-token read-token selection, but the reported CLI commands still call the config-backed Slack directory methods rather than the live Slack methods. `openclaw directory self --channel slack` also still resolves through the default null self adapter because Slack does not provide a self resolver. The user-token work in current main is useful related progress, but it does not fully solve this issue's observable CLI behavior.

## Best Possible Solution

Keep the issue open and fix the Slack directory CLI path: either have `openclaw directory peers/groups list` prefer live directory methods when a plugin exposes them, or add an explicit live mode and document the distinction. Slack should also add a `directory.self` resolver using the available Slack identity APIs, with regression tests covering `directory self`, `peers list`, and `groups list` for Slack user-token-backed reads.

## Evidence

- **CLI list commands use config-backed methods only:** `runDirectoryList` selects `plugin.directory?.listPeers` or `plugin.directory?.listGroups`; it does not prefer or fall back to `listPeersLive`/`listGroupsLive` for the reported CLI list commands.
