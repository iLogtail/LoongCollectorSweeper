## Summary

Keep open. Current `main` does not implement this PR’s per-job stuck-lock logic: maintenance still uses a fixed 2-hour `STUCK_RUN_MS`, while execution timeouts are already per-job. The startup path clears stale locks on restart, but regular timer maintenance still leaves shorter-timeout jobs blocked behind `runningAtMs` for up to 2 hours, so the proposed change remains relevant.
