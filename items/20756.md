## Summary

Current main does not implement the requested auto-selection of the only enabled account. The message action path still only uses explicit accountId, defaultAccountId, or an agent binding; WhatsApp fallback still resolves configured default/default/sorted account IDs rather than filtering for a single enabled account. The related PR #20801 was closed unmerged in the supplied GitHub context, so it does not supersede the issue.

## Evidence

- **Message tool account resolution lacks enabled-account fallback:** runMessageAction resolves accountId from params.accountId, input.defaultAccountId, then the first matching agent binding, and writes it back only if found. There is no branch that inspects channel account enabled states or selects a sole enabled account.
