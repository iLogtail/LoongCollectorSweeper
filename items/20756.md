## Summary

Keep #20756 open. Current main still does not auto-select the single enabled account for `message` tool sends when `accountId` is omitted and the configured `default` account is disabled. The existing message tool and action runner only honor explicit `accountId`, `defaultAccountId`, or agent binding fallback, while WhatsApp still resolves omitted outbound account IDs through the default-account helper that ignores enabled state. Related PR #20801 attempted the requested behavior but was closed unmerged, and the current docs still describe default/first configured account fallback rather than sole-enabled-account selection.

## Best Possible Solution

Keep #20756 open. Implement shared outbound account resolution in the message action path: preserve explicit account IDs, prefer a usable configured default, auto-select exactly one enabled account when no usable default exists, and return clear errors for zero or multiple enabled accounts. Add focused regression tests for `runMessageAction` and the WhatsApp disabled-default/single-enabled-account case, then update public channel docs.
