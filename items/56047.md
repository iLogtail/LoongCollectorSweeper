## Summary

Keep open. Current `main` does not implement the requested feature: Anthropic response headers are not captured into session/API state, and existing usage surfaces only expose normalized provider usage windows, not raw `anthropic-ratelimit-unified-*` values.

## Evidence

- **Anthropic transport does not retain response headers:** The Anthropic Messages client sends the request, checks `response.ok`, then streams `response.body` directly via `parseAnthropicSseBody(response.body)`. There is no code here that reads, stores, or forwards Anthropic rate-limit headers.
  - file: [src/agents/anthropic-transport-stream.ts:501](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/anthropic-transport-stream.ts#L501)
  - command: `nl -ba src/agents/anthropic-transport-stream.ts | sed -n '486,520p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Anthropic usage path is endpoint-based, not header-based:** Anthropic usage is loaded from `https://api.anthropic.com/api/oauth/usage` or the claude.ai web usage endpoint, then converted into `UsageWindow[]`. This path does not inspect response headers and only builds normalized windows like `5h`, `Week`, and `Sonnet`/`Opus`.
  - file: [src/infra/provider-usage.fetch.claude.ts:19](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/infra/provider-usage.fetch.claude.ts#L19)
  - command: `nl -ba src/infra/provider-usage.fetch.claude.ts | sed -n '1,170p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Provider usage types have no field for raw rate-limit headers:** `ProviderUsageSnapshot` only carries `provider`, `displayName`, `windows`, optional `plan`, and optional `error`. There is no structured field for raw header-derived data such as unified token limits/remaining/reset values.
  - file: [src/infra/provider-usage.types.ts:7](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/infra/provider-usage.types.ts#L7)
  - command: `nl -ba src/infra/provider-usage.types.ts | sed -n '1,80p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **`session_status` only returns formatted text plus basic metadata:** The tool response includes `content` with `statusText`, and `details` only contains `ok`, `sessionKey`, `changedModel`, and `statusText`. There is no `rateLimits` or similar structured field in the tool output.
  - file: [src/agents/tools/session-status-tool.ts:603](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/tools/session-status-tool.ts#L603)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '540,620p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Gateway auth-status enrichment only carries usage windows:** Gateway model auth status enriches providers by calling `loadProviderUsageSummary()` and storing only `{ windows, plan }` per provider. That API shape also has nowhere to expose raw Anthropic unified headers.
  - file: [src/gateway/server-methods/models-auth-status.ts:313](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/gateway/server-methods/models-auth-status.ts#L313)
  - command: `nl -ba src/gateway/server-methods/models-auth-status.ts | sed -n '300,340p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Docs explicitly say current tracing omits raw provider response headers:** The prompt-caching docs state that current output is focused on normalized token usage rather than raw provider response headers, and recommend logging request IDs/rate-limit headers separately.
  - file: [docs/reference/prompt-caching.md:327](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/reference/prompt-caching.md#L327)
