## Summary

Keep PR #39954 open. Current main still documents `tools.profile` mechanically, but it does not include the PR's practical warning that `tools.profile: "messaging"` is intentionally narrow or the direct guidance/example to use `tools.profile: "full"` for the broadest command/control surface. The runtime behavior and tests confirm the distinction is real, and the related PRs are adjacent open work rather than landed replacements.

## Best Possible Solution

Keep the PR open and port the useful wording into the current `docs/tools/index.md` profile section. The best maintainer path is a compact note explaining that `tools.profile: "messaging"` is intentionally narrow, `tools.profile: "full"` is the unrestricted baseline for broader command/control, and the wording should be reconciled with the current profile table plus the adjacent open help/status/troubleshooting PRs.
