## Summary

Keep #39954 open. Current main still documents the raw `tools.profile` profile contents but does not include this PR's practical clarification that `messaging` is intentionally narrow or that users wanting broader command/control should choose `full`. The runtime and tests confirm the underlying restriction is real, and the related PRs are still open rather than merged/superseding this docs change.

## Best Possible Solution

Keep this PR open and have a maintainer either rebase/port the focused `docs/tools/index.md` clarification or fold it into a coordinated docs/config/status troubleshooting update with the related open PRs. The missing user-facing guidance belongs in the tools docs because the current runtime behavior is intentional but still under-explained.
