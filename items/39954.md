## Summary

Keep open. Current `main` still does not include this PR's practical clarification of `tools.profile` behavior: the docs list raw profile contents, but they do not explain that `messaging` is intentionally narrow or direct users toward `full` for broader command/control access. The underlying behavior is real in code/tests, so the docs change remains useful rather than already-implemented.
