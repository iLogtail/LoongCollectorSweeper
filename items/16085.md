## Summary

Keep PR #16085 open. Current main still wires the bundled Signal plugin to native signal-cli JSON-RPC + SSE only; it has no REST/WebSocket container client, no adapter layer, no channels.signal.apiMode config, and no bbernhard/signal-cli-rest-api docs. The PR is the active implementation candidate for the closed tracker #10240, and the latest discussion explicitly narrowed it back to Signal container transport support.

## Best Possible Solution

Keep #16085 open as the active implementation candidate for bundled Signal container REST/WebSocket support. Maintainers should review the adapter/container design in extensions/signal, require focused config-schema/docs/tests coverage, and verify native signal-cli JSON-RPC/SSE behavior remains unchanged before considering merge.
