## Summary

Current main already fixes the reported 2026.4.24 regression. Legacy `agents.defaults.embeddedHarness.runtime: "claude-cli"` is migrated to `agentRuntime`, `claude-cli` is treated as a CLI backend alias rather than an embedded AgentHarness id, gateway fallback paths no longer pass it as `agentHarnessId`, and startup planning includes the Anthropic plugin that owns the `claude-cli` backend.

## Best Possible Solution

Close this as implemented on current main. Keep the shipped `agentRuntime`/CLI-backend split, legacy config migration, and startup owner loading; if users still reproduce after the next stable release, handle it as a fresh runtime-specific regression with new logs and config.
