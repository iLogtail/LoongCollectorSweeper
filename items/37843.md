## Summary

Current main does not implement the requested cron behavior. `payload.model: "default"` is still treated as a literal non-empty model override, which resolves as `<defaultProvider>/default` rather than skipping override resolution and using the configured default model. The linked PRs proposed the expected guard, but both are closed and unmerged, and the guard is absent from current main.

## Evidence

  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/cron/isolated-agent/model-selection.ts#L104)
