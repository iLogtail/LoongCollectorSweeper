## Summary

Keep open. Current main still does not implement `payload.model: "default"` as a cron runtime-default sentinel. The cron model-selection path treats any non-empty trimmed payload model as an explicit override, downstream model parsing resolves bare `default` as a literal model under the effective default provider, and fallback policy also classifies `default` as a payload model override.

## Best Possible Solution

Keep the issue open and implement the sentinel in the current cron model-selection path, likely by treating trimmed `payload.model === "default"` like an omitted model before calling `resolveAllowedModelRef`. Update `src/cron/isolated-agent/run-fallback-policy.ts` so the sentinel preserves omitted-model fallback behavior, and add focused tests for omitted, empty, whitespace-trimmed `default`, explicit provider/model, disallowed, invalid, and fallback cases.
