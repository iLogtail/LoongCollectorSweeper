## Summary

Keep #29552 open. Current main has newer-config guards, update hardening, docs, and a trusted device-token retry path, but it still does not implement the local shared-token fallback proposed for this issue: ordinary local CLI/gateway calls can still prefer stale `OPENCLAW_GATEWAY_TOKEN` over `gateway.auth.token` and make only one request attempt. The issue also has recent reporter confirmation and the directly related fix PR #29676 was closed unmerged.

## Best Possible Solution

Keep this open. The best maintainer path is to either add a guarded local-only retry in `src/gateway/call.ts` that falls back to resolved `gateway.auth` token/password after `AUTH_TOKEN_MISMATCH` when no remote URL or explicit auth override is involved, with regression coverage for stale `OPENCLAW_GATEWAY_TOKEN`, or make an explicit product decision that env-first local call precedence is intended and expand doctor/troubleshooting guidance. If the package/update version-drift symptom is now considered solved by the newer update/service guards, split that into a narrower canonical issue instead of closing this one.
