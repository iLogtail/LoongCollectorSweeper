## Summary

Keep open. Current `main` and the latest shipped release still do not implement native Slack table-block delivery for `markdown.tables: "block"`. The code explicitly coerces `block` to `code`, the shared table converter still falls back to code rendering, Slack send still goes through text mrkdwn chunking, and tests/docs encode that behavior.
