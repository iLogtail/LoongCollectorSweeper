## Summary

Keep open. Current main still does not honor `cfg.agents.list[].thinkingDefault` in the `chat.history` fallback path: the handler resolves the session agent id, but when `entry.thinkingLevel` is absent it goes directly to `resolveThinkingDefault(...)`, whose implementation only checks per-model config and `agents.defaults.thinkingDefault` before model heuristics. The documented resolution order says per-agent thinking defaults should win before global/model fallback, and the related #70302 is an adjacent SessionRow fix rather than a replacement for this `chat.history` RPC fix.

## Best Possible Solution

Keep the PR open and either merge or port the focused `chat.history` change: after `entry.thinkingLevel`, consult `resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault`, then fall through to the existing catalog-aware `resolveThinkingDefault(...)`. Add/retain the focused regression tests for session value precedence, per-agent fallback, resolver fallback, and normalized `agent:main:main` session keys. Treat #70302 as a separate companion fix for SessionRow/TUI footer state, not as a reason to close this PR.
