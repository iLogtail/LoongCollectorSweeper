## Summary

Keep open. Current main still does not show a source-backed fix for the reported main-process text-transform gap: `resolveRuntimeTextTransforms()` reads through `getActiveRuntimePluginRegistry()`, and that helper still probes only `./runtime.js` / `./runtime.ts` via `createRequire`. The linked PR #64924 corroborates the diagnosis but was closed unmerged, so it is not shipped evidence.

## Best Possible Solution

Leave #64921 open. The likely fix belongs in the plugin runtime registry/text-transform path: use a direct active-registry accessor that works inside the bundled main process, keep any external-plugin fallback only if it is actually reachable, and add a regression test proving plugin-registered text transforms are applied through the main runtime prompt/provider path.
