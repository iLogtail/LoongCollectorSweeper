## Summary

Keep open. Current `main` already has partial WhatsApp TTS plumbing: docs say TTS works anywhere OpenClaw can send audio, `/tts audio <text>` returns an audio reply, auto-TTS can emit a TTS-only final payload to avoid duplicate text, and the WhatsApp channel sends outbound audio as `ptt` voice notes. But the specific feature requested here is not implemented: there is no native WhatsApp message action to read the assistant's last/current reply aloud, no channel-specific "Read aloud" action surface, and WhatsApp message actions currently expose only `react` and `poll`.
