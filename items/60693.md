## Summary

Keep open. Current main appears to cover the Discord REST proxy and documents model precedence, but this bundled issue still has actionable remaining work: the ACPX Codex path still shells through npm/npx rather than a bundled/validated codex-acp binary path, the default-model command does not migrate per-agent or cron overrides, and ACP allowedAgents is still treated as a raw allowlist/probe source without installed/disabled-agent filtering.

## Best Possible Solution

Keep the issue open or split it into focused maintainable items. The best product path is to let #72037 carry the ACPX codex-acp packaging/npx fallback work, decide whether a safe bulk model migration command should rewrite defaults, per-agent overrides, and cron payloads, and add ACPX/ACP validation that warns or skips stale allowedAgents entries without breaking legitimate external ACP harness ids.
