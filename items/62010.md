## Summary

Current main does not implement WhatsApp-scoped configured outbound media roots. The WhatsApp send path already accepts and forwards `mediaLocalRoots`, but the current config types/schema/docs do not expose `channels.whatsapp.mediaLocalRoots`, and the shared outbound delivery context only uses agent-scoped `mediaAccess.localRoots`. PR #62010 remains actionable, though it needs revision for the account/defaultAccount path, validation/docs generation, and preferably a plugin-owned or generic adapter hook rather than hard-coding WhatsApp-specific config into shared core delivery.

## Best Possible Solution

Keep this PR or a follow-up open and revise it: add `mediaLocalRoots` to WhatsApp config types, zod schema, docs, and generated config metadata; resolve and merge channel-level plus effective-account roots at the WhatsApp plugin/outbound boundary or through a generic channel adapter hook; validate/normalize entries consistently with local-media policy; and add tests for channel-level, account-level, omitted-account/defaultAccount, and deduplication behavior.
