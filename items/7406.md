## Summary

Keep #7406 open. Current main has partial Telegram topic-name support: it caches topic names and exposes `TopicName` to inbound prompt metadata, but it still does not map that value into persisted session `displayName`/`label` or the Control UI session dropdown/list display path. Related implementation PRs #7407 and #43105 are closed and unmerged in the provided GitHub context, and the current issue discussion includes a same-day keep-open review plus a follow-up “yes please” comment.

## Best Possible Solution

Keep #7406 as the canonical tracker. The best fix is to map cached Telegram forum `TopicName` into a persisted session display field used by `sessions.list` and the Control UI session selector, likely via `ThreadLabel` or `displayName` during inbound session metadata recording. Add regression coverage for Telegram context extraction, session-store metadata, gateway row output, and the Control UI label resolver, with a topic-id fallback only when no human-readable name is known.
