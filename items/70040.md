## Summary

Keep #70040 open. Current main does not already implement Chinese/CJK memory-lancedb auto-capture triggers, and the related PRtags items are all closed/unmerged rather than a live canonical replacement. The PR is incomplete as submitted because the existing `shouldCapture` length gate can still reject short Chinese memory statements before trigger matching, and there is no current Chinese trigger/category regression coverage.

## Best Possible Solution

Keep the PR open but require a scoped revision before merge: add CJK trigger coverage in `extensions/memory-lancedb/index.ts`, handle short high-signal Chinese memory statements by adjusting the length gate or evaluating strong triggers first, and add regression tests in `extensions/memory-lancedb/index.test.ts` for examples such as `记住我叫李雷` and Chinese preference/category cases. If maintainers prefer the broader approach from #20241, reopen or recreate that consolidated CJK trigger/category work as the canonical PR rather than closing this without a replacement.
