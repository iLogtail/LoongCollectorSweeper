## Summary

Keep #70040 open. Current main still does not implement Chinese/CJK auto-capture triggers for the bundled memory-lancedb plugin, and the related PRs cited by PRtags are closed unmerged rather than shipped. The submitted one-line PR is incomplete because short Chinese memory statements still hit the existing `text.length < 10` gate and there is no Chinese regression coverage, but that means it needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open or ask for a revised replacement that implements Chinese/CJK memory-lancedb auto-capture end to end: adjust the short-input gate for high-signal CJK triggers, add Chinese trigger patterns, add `shouldCapture` tests for both short and longer Chinese memory statements, and consider `detectCategory` coverage for Chinese preference/decision/entity/fact cases using the prior closed PRs as design references.
