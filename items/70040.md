## Summary

Keep #70040 open. Current main still lacks Chinese/CJK `memory-lancedb` auto-capture triggers, and the related PRtags items are closed unmerged rather than a shipped canonical fix. The PR addresses a real missing behavior, but the current one-line change needs maintainer review because `shouldCapture` rejects inputs shorter than 10 characters before trigger matching and category detection remains English/Czech-only.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The durable fix belongs in `extensions/memory-lancedb`: add focused Chinese/CJK capture triggers, handle short high-signal Chinese memory statements, extend category detection where appropriate, and add regression tests for short and longer Chinese examples plus non-capture cases.
