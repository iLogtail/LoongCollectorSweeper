## Summary

Keep PR #70040 open. Current main still lacks Chinese/CJK memory-lancedb auto-capture triggers, so the central requested capability is not implemented. The existing PR is also not cleanly obsolete: related PRs in the provided context are closed unmerged, and the current PR has a substantive review concern that common short Chinese inputs still fail the existing `text.length < 10` capture gate.

## Best Possible Solution

Keep the PR open and ask for a small but complete memory-lancedb change: add Chinese/CJK capture triggers, account for short high-signal Chinese inputs in `shouldCapture`, add regression tests for representative Chinese capture strings including short inputs, and consider matching category detection for preference/decision/entity/fact if the captured memories should not all fall through to `other`.
