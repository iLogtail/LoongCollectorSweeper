## Summary

Keep #44111 open. Current main still only ships the local archive `openclaw backup create` / `verify` flow; the PR's encrypted folder-backed snapshots, `setup/run/status/list/restore` commands, `backup.target` / `backup.encryption.key` config, and workspace mirroring are not present on main. The prior snapshot PR #41345 is closed and unmerged, while this PR is the active v2 rather than obsolete. The keepmyclaw link in discussion is useful as an off-machine complement, not a replacement for the native core backup flow proposed here.

## Best Possible Solution

Keep #44111 open as the active native core backup snapshot implementation candidate. The best path is to rebase it onto current main, preserve the security hardening and regression coverage already added in review, resolve any remaining restore-target edge cases, and run the changed backup/CLI/config/docs lanes before maintainer review. Off-machine tools such as keepmyclaw can stay documented as complementary backup options, but they do not replace this core local encrypted snapshot flow.
