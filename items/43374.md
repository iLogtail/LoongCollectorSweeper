## Summary

Keep open. Current main has several relevant hardening changes around model fallback, per-attempt run timers, guarded HTTP/1.1 provider fetches, and Telegram polling transport, but the issue’s core multi-agent simultaneous-timeout report is not conclusively fixed on main. Meaningful requested items remain unimplemented or only partially addressed, especially Telegram polling concurrency still being tied to agents.defaults.maxConcurrent, no launchd/default UV_THREADPOOL_SIZE setting, and no global force of autoSelectFamily=false/dnsResultOrder=ipv4first for LLM traffic.
