## Summary

Keep #72329 open. Current main accepts and preserves `compat.thinkingFormat: "qwen-chat-template"`, and it now has manual `params.chat_template_kwargs` / `extra_body` pass-through plus a Nemotron-specific vLLM auto path. It still does not map the Qwen chat-template format to `chat_template_kwargs.enable_thinking`, so the reported `/think` toggle behavior remains missing. The related #54983 context matches the same remaining gap, but that tracker is closed, so this issue is still useful as the active tracker.

## Best Possible Solution

Implement this as provider-owned vLLM request shaping or a shared OpenAI-compatible chat-template helper: when a vLLM model opts into `compat.thinkingFormat: "qwen-chat-template"`, map OpenClaw's current `thinkingLevel` to top-level `chat_template_kwargs.enable_thinking` while preserving documented precedence for explicit `params.chat_template_kwargs` and `params.extra_body.chat_template_kwargs`. Add focused tests for Qwen-on-vLLM off/on behavior and update `docs/providers/vllm.md`.
