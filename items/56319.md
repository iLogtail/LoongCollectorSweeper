## Summary

Current main does not implement the PR's requested Telegram approval-message deletion. The active Telegram native approval runtime resolves approvals with `clear-actions`, which only removes inline buttons via `editMessageReplyMarkupTelegram`; it does not wire `deleteMessageTelegram` or a `deleteEntry` transport. The linked bug #56286 remains a concrete open product issue, so the PR should not be auto-closed, but it needs a rebase/rework against the current `approval-handler.runtime.ts` adapter and should split unrelated docs cleanup.

## Best Possible Solution

Keep the bug/PR path open, but require a narrow rework against the current native approval adapter: add a Telegram `deleteEntry` path using `deleteMessageTelegram` and either return `{ kind: "delete" }` on resolved approvals or implement a tracked/unref delayed cleanup if maintainers want the 2-second visible transition. Add a regression test for resolved Telegram approval deletion and split unrelated docs cleanup into separate work.
