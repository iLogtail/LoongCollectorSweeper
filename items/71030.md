## Summary

Keep open. Current `main` still preserves QMD diff-format snippets through memory search and short-term recall tracking, while deep promotion rehydration still tries to match that stored snippet against normalized live note text. That means a QMD-backed candidate can rank successfully but still fail promotion exactly as described in #71030.

## Evidence

- **QMD search still returns diff-format snippets unchanged:** `QmdMemoryManager.search()` copies `entry.snippet` straight into each `MemorySearchResult` after line extraction, without stripping the `@@ -x,y` diff header. Current tests on `main` still assert this raw diff-form output.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:1211](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-core/src/memory/qmd-manager.ts#L1211)
  - command: `nl -ba extensions/memory-core/src/memory/qmd-manager.ts | sed -n '1201,1224p' && nl -ba extensions/memory-core/src/memory/qmd-manager.test.ts | sed -n '1666,1676p'`
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Recall tracking stores the snippet as-is:** `recordShortTermRecalls()` only runs whitespace normalization on `result.snippet`, then writes that value into `short-term-recall.json`. There is no QMD diff-header cleanup before persistence.
  - file: [extensions/memory-core/src/short-term-promotion.ts:913](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-core/src/short-term-promotion.ts#L913)
  - command: `nl -ba extensions/memory-core/src/short-term-promotion.ts | sed -n '911,970p'`
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Promotion rehydration still requires matching plain live-note text:** `relocateCandidateRange()` normalizes the candidate snippet and compares it against normalized windows from the live source file using equality/substring checks. A stored snippet like `@@ -3,4 ... - Bob asked...` will not match the plain note text, so `rehydratePromotionCandidate()` returns `null` and the candidate is skipped.
  - file: [extensions/memory-core/src/short-term-promotion.ts:1376](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-core/src/short-term-promotion.ts#L1376)

- **No fix landed after the latest release:** Diffing `v2026.4.23` against current `main` shows only unrelated `qmd-manager` source-filtering changes and test additions in this area; there is still no change that strips diff headers or makes promotion rehydration tolerant of them.
