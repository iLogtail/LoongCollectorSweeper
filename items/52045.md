## Summary

Keep open. This is a recent, concrete regression report with an author-provided reproduction detail: browser status works after install, then reports the gateway closed after Feishu pairing. Current main still routes `openclaw browser status` through `browser.request` and returns browser status from the current resolved browser state; I did not find a landed source change or test proving the reported mismatch is fixed. The linked PR #52057 explicitly tracks this exact issue but remains open and unmerged, so the issue should stay open until a fix lands or maintainers choose a different canonical tracker.
