## Summary

Keep open. Current main still matches the reported failure mode: cross-agent `resolveRequesterOriginForChild` resolves `accountId` from the target child agent binding and then stores that origin for subagent completion routing. The related fix PR #70607 is still open/unmerged, so the bug is not implemented on main.

## Best Possible Solution

Keep #70574 open. Land a focused fix with regression coverage for cross-agent Telegram-style completion routing: preserve the parent/requester `accountId` for completion announce back to the requester session, while retaining any target-agent bound account only where child thread creation or child-session binding genuinely needs it. PR #70607 is the active candidate, but it should be checked against the existing cross-agent binding tests before merging.

That is the account overwrite described in #70574.
