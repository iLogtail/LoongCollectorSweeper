## Summary

Current main still contains the reported routing behavior: cross-agent `sessions_spawn` calls resolve a bound account using the child `targetAgentId`, which can replace the parent requester account. An open PR (#70607) proposes the same fix, but it is unmerged, so the issue should stay open.

## Best Possible Solution

Keep the issue open and use PR #70607, or an equivalent maintainer-authored patch, to make `requesterOrigin` preserve the parent requester account for cross-agent subagent spawns while retaining explicit coverage for same-agent spawns and nested/thread-bound completion delivery.

## Risks / Open Questions

- The current tests assert child-account selection for cross-agent bindings, so the fix needs careful regression coverage for cross-agent Telegram/account binding, same-agent spawns, and nested `streamTo="parent"` behavior.
- Changing this route may affect integrations that relied on target-agent account lookup for cross-agent delivery, so the maintainer review should confirm the intended contract before merging PR #70607 or an equivalent fix.
