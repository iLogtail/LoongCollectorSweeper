## Summary

Current main still contains the reported cross-agent requester-origin behavior. `resolveRequesterOriginForChild` resolves a bound account for the child/target agent when `targetAgentId !== requesterAgentId`, then prefers that value over the parent's `requesterAccountId`. The later completion path still feeds that origin into child-session bound delivery routing, so the Telegram misrouting report should remain open until PR #70607 or an equivalent fix lands.

## Best Possible Solution

Keep #70574 open. The maintainer path is to land PR #70607 or an equivalent change that preserves the parent's `requesterAccountId` for cross-agent `sessions_spawn` completion routing, then add focused regression coverage for Agent A spawning Agent B and nested `streamTo="parent"` behavior before the issue closes.
