## Summary

Current main still looks like a valid feature gap. `sessions_send` records machine provenance for inter-session prompts, and the UI can display a `senderLabel` when one exists, but the session-send path does not populate or resolve a human-readable sender/recipient label for either the receiving transcript or the sender's session.

## Best Possible Solution

Keep the issue open. The likely fix should either extend `InputProvenance` with display-safe source/target identity fields and populate them in `sessions_send`, A2A reply steps, and announce delivery, or resolve `sourceSessionKey`/target keys to labels at `sessions.get` or UI render time. Add regression coverage for both receiving-session sender display and sender-session target visibility/log troubleshooting.
