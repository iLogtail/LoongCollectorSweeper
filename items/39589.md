## Summary

Keep #39589 open. Current main still requires `models` in `models.providers.*` at the Zod schema, generated schema, and TypeScript config contract, so the reporter's provider-only `models.providers.openai` config remains invalid. The OpenAI memory embedding runtime would honor `models.providers.openai.baseUrl` if validation accepted the config, but validation is still the blocker. Related PRs #46977 and #46980 proposed this same schema fix and are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands: make `ModelProviderSchema.models` optional with a default empty array, align `ModelProviderConfig` if needed, regenerate generated schema artifacts, add a regression test proving `models.providers.openai` with only `baseUrl`/`apiKey`/`api` validates, and keep/add a memory embedding test proving provider `baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
