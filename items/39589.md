## Summary

Keep open. Current main still requires `models.providers.*.models`, so the reporter's provider-only `openai` config remains invalid even though the memory embedding runtime would honor `models.providers.openai.baseUrl` after validation. The useful related work is still not merged: #46977 was closed unmerged and #46980 is open per the provided GitHub context.

## Best Possible Solution

Keep #39589 open until OpenClaw either makes `ModelProviderSchema.models` optional with a default empty array and regenerates aligned config artifacts/tests, or maintainers decide that embedding-only provider credentials must instead use `agents.defaults.memorySearch.remote.*` and document/reject the `models.providers.openai` workflow explicitly.
