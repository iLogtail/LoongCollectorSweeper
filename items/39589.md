## Summary

Keep #39589 open. Current main still rejects a provider-only `models.providers.openai` entry because the core config schema, generated schema, and TypeScript contract all still require `models`. The OpenAI memory embedding runtime would honor `models.providers.openai.baseUrl` once the config is accepted, so the remaining issue is still the config contract. Related PRs #46977 and #46980 attempted this optional-models fix but are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands: make `ModelProviderSchema.models` optional with a default empty array, align `ModelProviderConfig`, regenerate generated schema/help artifacts, add a regression test proving `models.providers.openai` with only `baseUrl`/`apiKey`/`api` validates, and keep or add an embedding test proving provider `baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
