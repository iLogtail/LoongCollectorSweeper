## Summary

Keep #39589 open. Current main still rejects the reporter's provider-only `models.providers.openai` entry because `models` remains required in the config schema. The OpenAI memory embedding path would honor `models.providers.openai.baseUrl` after validation, but the reported config cannot pass validation without adding `models: []` or using the separate `memorySearch.remote.baseUrl` workaround. The related #46977 and #46980 PRs are closed unmerged in the provided context, so no shipped or canonical superseding fix closes this issue.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands. The likely fix is to make `ModelProviderSchema.models` optional with a default empty array, regenerate the generated config schema artifacts, add a schema regression test for a provider entry containing only `baseUrl`/`apiKey`/`api`, and keep or add a memory embedding test proving `models.providers.openai.baseUrl` is used when `memorySearch.remote.baseUrl` is not set.
