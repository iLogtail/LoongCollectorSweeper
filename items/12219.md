## Summary

Keep #12219 open. Current main still has no skill.yaml/skill.yml permission manifest standard, no install-time permission consent in openclaw skills install, no openclaw skills audit command, and no shipped signing/attestation/trust-grade path for skills. The related @ellistevo package, ClawHub shell-security-ultimate skill, and globalcaos/tinkerclaw fork are useful implementation references, not substitutes for shipped OpenClaw behavior.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. Define a versioned skill permission manifest for skills, validate it during ClawHub/direct skill install and updates, show declared permissions before openclaw skills install, add an audit/trust report or equivalent, and connect declarations to runtime enforcement hooks where appropriate. Treat @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as useful references, not shipped replacements.
