## Summary

Keep #9912 open. Current main still does not expose a global or per-agent hard `maxTurns`, `maxToolCalls`, or `toolCallTimeout` budget that forces a final response after model-tool iterations. Existing safeguards cover overall agent timeout, LLM idle timeout, optional pattern-based loop detection, and hook-level tool blocking, but those are not equivalent to the requested total turn/tool-call budget or last-tool-result finalization behavior.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard tool-loop budget request. A complete fix should add typed config and schema/docs for max model-tool turns and/or max total tool calls, enforce the budget before executing more tools or continuing the model loop, define behavior for the timeout-after-first-tool-call case, and specify final-response behavior using the last useful tool result or an explicit iteration-limit notice. Keep #13271 separate for per-channel budget and auto-delegation behavior.
