## Summary

Current main does not implement a public `transform_tool_result` plugin hook. OpenClaw now has pre-model tool-result middleware, but it is a different bundled-plugin-only seam, so this issue still represents a real open feature/API question for external plugins.

## Evidence

- **Requested hook name is absent from the public hook contract:** `PluginHookName` and `PLUGIN_HOOK_NAMES` include `before_tool_call`, `after_tool_call`, and `tool_result_persist`, but no `transform_tool_result`.
  - file: [src/plugins/hook-types.ts:72](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/plugins/hook-types.ts#L72)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '64,110p'`
  - sha: [8bead989da0c](https://github.com/openclaw/openclaw/commit/8bead989da0c62d8df91094ef9b881b725b282c8)
- **Public hook docs still describe only the existing tool hooks:** The plugin hook catalog documents `before_tool_call`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; there is no public pre-model transform hook.
  - file: [docs/plugins/hooks.md:74](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/docs/plugins/hooks.md#L74)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '70,82p'`
  - sha: [8bead989da0c](https://github.com/openclaw/openclaw/commit/8bead989da0c62d8df91094ef9b881b725b282c8)
- **Main now has a different pre-model seam:** SDK docs explicitly say bundled plugins can use `api.registerAgentToolResultMiddleware(...)` to rewrite a tool result after execution and before the runtime feeds that result back into the model.
  - file: [docs/plugins/sdk-overview.md:119](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/docs/plugins/sdk-overview.md#L119)
  - command: `nl -ba docs/plugins/sdk-overview.md | sed -n '118,128p'`
  - sha: [8bead989da0c](https://github.com/openclaw/openclaw/commit/8bead989da0c62d8df91094ef9b881b725b282c8)
- **That seam is intentionally not available to external plugins:** Docs and runtime validation both state that external plugins cannot register tool-result middleware; the registry emits `only bundled plugins can register agent tool result middleware`.
  - file: [src/plugins/registry.ts:297](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/plugins/registry.ts#L297)

  - sha: [8bead989da0c](https://github.com/openclaw/openclaw/commit/8bead989da0c62d8df91094ef9b881b725b282c8)
- **Pre-model rewriting is wired for both runtimes and already used by a bundled plugin:** PI and Codex both apply tool-result middleware before returning tool output, and bundled `tokenjuice` registers this middleware for `pi` and `codex` as a concrete result-compaction implementation.
  - file: [src/agents/pi-embedded-runner/extensions.ts:37](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/agents/pi-embedded-runner/extensions.ts#L37)

## Risks / Open Questions

- Maintainers may decide the right follow-up is to expose the existing middleware seam more broadly rather than add a new `transform_tool_result` hook name.
- The issue body is still directionally valid because the current bundled-only middleware does not satisfy third-party/external plugin use cases.
