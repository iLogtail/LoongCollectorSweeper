## Summary

Keep open. Current `main` does not implement a generic `transform_tool_result` plugin hook. The public hook list still only includes `before_tool_call`, `after_tool_call`, and `tool_result_persist`; `after_tool_call` runs after emission, and `tool_result_persist` only rewrites OpenClaw-owned transcript persistence, so the requested immediate model-visible transform gap still exists.

## Risks / Open Questions

- There is partial overlap in the Codex harness via Codex app-server `tool_result` middleware, so future work should decide whether the generic plugin hook should reuse or align with that existing Codex-only path.
- Any implementation touches public plugin SDK/hook contracts, so it needs explicit compatibility design for chaining semantics and live-result vs persisted-transcript behavior.
