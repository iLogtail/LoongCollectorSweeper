## Summary

Keep #70898 open. Current main still rotates silent auto-reply sessions onto a fresh session id/file without replaying recent user/assistant transcript context, and Telegram DMs do not have the group-history fallback that would mask the reported continuity loss. The PR is a plausible focused fix for open issue #70853, but it is not implemented on main and still needs maintainer review of the replay-tail safety concerns before merge.

## Best Possible Solution

Keep #70898 open. The right maintainer path is a core auto-reply/session fix that preserves a small, validated recent user/assistant tail across silent compaction/role-ordering rotations while keeping explicit `/new` and `/reset` fresh. Before merge, tighten the replay helper so the retained tail cannot recreate strict role-ordering failures and does not require a full transcript read on the recovery path.
