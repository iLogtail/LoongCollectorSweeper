## Summary

Keep #70898 open. Current main still has the linked #70853 gap: silent auto-reply resets for compaction failures and role-ordering conflicts mint a fresh session id/file without replaying prior DM context. The PR is a focused attempt to fix a real current behavior, but its latest review context still raises correctness/performance concerns that need maintainer handling before merge.

## Best Possible Solution

Keep #70898 open and have the author address the remaining review blockers: preserve recent user-visible context across silent reset/rebind, but normalize or collapse same-role replay tails so strict alternation providers do not immediately fail again, and avoid full transcript reads in the recovery path. The fix belongs in the auto-reply silent reset path with regression coverage for compaction failure, role-ordering recovery, consecutive same-role records, assistant-only tails, and explicit /new or /reset staying fresh.
