## Summary

Keep open. Current main has a macOS bootstrap path in `scripts/install.sh` that installs Homebrew for the main OpenClaw installer, but the reported skills/onboarding path still only shows the Homebrew install command when `brew` is missing and then continues into installer metadata that returns `brew not installed` for brew-only skill installs. Related ClawSweeper reports #14593 and #57555 also remain open for unresolved brew-only skill installability gaps, so this is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep the issue open and decide the product policy for gateway-backed skill dependency installs. A concrete fix would likely add a safe, explicit `Install Homebrew now?` path in `src/commands/onboard-skills.ts` and/or a shared helper used by `src/agents/skills-install.ts`, with platform-specific safeguards for macOS versus Linuxbrew/Docker and tests covering missing-brew brew-only skill installs. If maintainers intentionally want manual Homebrew setup for skills, the prompt and docs should be changed to say that clearly instead of implying the installer may handle it.
