## Summary

Keep open. Current main does not implement a vi/vim modal input editor: the TUI still constructs a plain CustomEditor with no vi-mode option, the config schema has no `tui`/`tui.viMode`/`tui.inputMode` setting, docs list only the existing shortcuts, and the same-scope PR #55975 was closed unmerged. The related #22770 external-editor compose issue is adjacent but does not cover in-place modal bindings.

## Best Possible Solution

Keep #55836 open as the canonical request for in-place vi/vim modal editing in the TUI composer. A good implementation would add an explicit config surface, pass it into `src/tui/components/custom-editor.ts`, implement modal state and vi motions/deletes with focused unit tests, update the generated config schema/help/docs, and keep #22770 separate for external `$EDITOR` compose mode.
