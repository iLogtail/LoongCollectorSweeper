## Summary

Current `main` does not implement vi/vim input mode for the TUI editor. The TUI still hard-wires `Esc` to abort/dismiss behavior, exposes only the existing Ctrl-based shortcuts in docs, and has no `tui.inputMode` or `OPENCLAW_TUI_INPUT_MODE` config surface. This feature request remains unmet and should stay open.

## Evidence

- **TUI editor consumes Esc for OpenClaw actions:** `CustomEditor.handleInput()` intercepts `Key.escape` and calls `onEscape()` instead of passing Esc through to editor text-handling, which conflicts with vi-style normal/insert mode switching.
  - file: [src/tui/components/custom-editor.ts:49](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/tui/components/custom-editor.ts#L49)
