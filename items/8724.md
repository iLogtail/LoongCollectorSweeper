## Summary

Keep #8724 open. Current main still lacks a typed global/per-model generation timeout under `agents.defaults.models`; the model catalog accepts only `alias`, `params`, and `streaming`, while runtime timeout resolution remains a global run timeout plus an idle/no-chunk watchdog. Related first-token/provider fallback reports are adjacent, but they do not implement the requested total generation budget for a model that keeps streaming non-useful output.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete fix should validate config shape, resolve the effective timeout per fallback candidate, keep it distinct from idle/no-token behavior, abort and log timeout events, integrate with fallback handling, and add regression coverage for a stream that keeps emitting chunks past the configured budget.
