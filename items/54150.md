## Summary

Keep open. Current main still leaves the native PDF provider module with bare global fetch calls for Anthropic and Gemini and no local env-proxy bootstrap or regression test for this path. Broader CLI/embedded-runner bootstrap may mitigate normal runtime flows, but it does not implement the specific direct-provider fix requested here.

## Best Possible Solution

Keep the issue open and make a narrow code fix in src/agents/tools/pdf-native-providers.ts: import ensureGlobalUndiciEnvProxyDispatcher, call it before both Anthropic and Gemini fetches (or via a shared local helper), and add focused regression coverage proving the bootstrap runs before the native PDF provider fetch path.
