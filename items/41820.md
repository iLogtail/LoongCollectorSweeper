## Summary

Keep open. Current main still routes Feishu/Lark HTTP API calls through the Lark SDK Axios default HTTP instance with only timeout injection; it does not inject an HTTP CONNECT-capable proxy agent or set `proxy: false` for Axios. The linked workaround and open PRs are useful context, but the observable bug is not implemented on main yet.
