## Summary

Keep open. Current main still has the reported broad sandbox env-var value heuristic: `validateEnvVarValue` warns on any 80+ character value matching the base64 character class, `sanitizeEnvVars` calls it without key-name context, and tests still assert the warning for a non-credential key. The related fix PRs in the provided context were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep this issue open until `src/agents/sandbox/sanitize-env-vars.ts` tightens the base64 warning heuristic, either by requiring stronger base64 structure or by combining the value heuristic with credential-like key names. Add regression coverage in `src/agents/sandbox/sanitize-env-vars.test.ts` for legitimate long alphanumeric/hex/JWT-segment-like values while preserving existing blocked credential-key behavior.
