## Summary

Current main still has the reported value-only base64 heuristic and still warns for any 80+ character value in the base64 character class, including long alphanumeric, hex-like, and JWT-segment-like values. The prior fix PRs in the provided context are useful attempts but were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep this issue open. Tighten src/agents/sandbox/sanitize-env-vars.ts by making the base64 warning structurally aware and/or key-name aware, then add focused regression tests for legitimate long alphanumeric, hex-like, and JWT-segment-like values while preserving existing explicit secret blocking and null-byte blocking behavior.
