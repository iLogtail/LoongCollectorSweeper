## Summary

Keep open. The report describes a concrete Control UI iOS Safari bug, the reporter confirmed it on a real iPhone, and current origin/main still contains sub-16px font sizes on the main chat composer, agent chat textarea, mobile form controls, and config fields. Linked PRs #64673 and #65188 appear to be proposed fixes, but they are open/unmerged in the provided GitHub context and the relevant CSS is still not fixed on main.

## Best Possible Solution

Keep #64651 open and land a targeted Control UI CSS fix. The best path is to set computed font-size to at least 16px for focusable text controls on touch/iOS surfaces, including `.chat-compose .chat-compose__field textarea`, `.agent-chat__input > textarea`, `.field input/textarea/select`, and config inputs/textareas/selects, while preserving desktop density where appropriate. The linked PRs #64673 and #65188 provide useful implementation direction; add a focused CSS regression test or style assertion if feasible, then manually verify on iPhone Safari/Home Screen Web App.
