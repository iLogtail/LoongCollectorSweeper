## Summary

Keep open. Current main still unconditionally falls back from `openclaw agent` gateway execution to embedded execution for every gateway error, including ambiguous timeout/closed cases. The PR remains a focused, relevant bug fix with tests for the missing fail-closed behavior, and the provided review thread shows the author addressed the additional `gateway request timeout for agent` variant.

## Best Possible Solution

Keep the PR open and land this behavior, preferably with the review-thread hardening: fail closed for gateway timeout, gateway request timeout for `agent`, and gateway closed errors while preserving embedded fallback for clearly local/recoverable errors such as `gateway not connected`. Include the focused `agent-via-gateway` regression tests from the PR or an equivalent structured-error implementation.
