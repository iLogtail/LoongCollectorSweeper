## Summary

Keep #60490 open. Current main has several Windows gateway service hardening pieces, but the reported user problem is native Windows automatic gateway startup after reboot/logon, and that is still being actively tracked by an open, unmerged Windows auto-start PR rather than clearly shipped on main.

## Best Possible Solution

Keep #60490 open as the user-visible native Windows gateway auto-start bug. Triage it with PR #68149 or a successor fix, then verify on a real Windows host: install/reinstall or onboard with daemon install, ensure `gateway.mode=local`, reboot/log in, confirm `OpenClaw Gateway` state plus a real listener on 127.0.0.1:18789, and confirm `openclaw dashboard` reaches the Control UI without a manual `openclaw gateway start`.

## Evidence

- **Issue has concrete recent Windows auto-start reproduction:** The reporter showed the service could be installed and manually started after setting gateway.mode=local, but still needed `openclaw gateway start` after each Windows restart.
