## Summary

Keep open. Current main supports inbound Telegram per-topic agent routing and explicit/current-thread outbound routing, but it does not provide the requested config-level outbound default topic binding per agent. The existing `topics.*` config is strict and only includes inbound/routing fields such as `agentId`; outbound sends resolve a thread from explicit `threadId`, a topic-qualified target, or current tool context, not from the topic-agent config.

## Best Possible Solution

Add a Telegram-owned config and runtime path for outbound defaults, likely in `extensions/telegram` plus shared config/schema/docs/tests. The implementation should define a clear precedence order for explicit `threadId`, topic-qualified `target`/`to`, current conversation context, and any new per-agent topic default, then cover message-tool and proactive-send behavior with targeted Telegram tests.
