## Summary

Keep #9132 open. Current main still has lifecycle data available in the gateway row model and Control UI session state, but the Sessions table does not render active/completed/timed-out status or provide status-based sorting/filtering. The related #46792 is closed and the existing maintainer/Codex comment already treats #9132 as the remaining tracker.

## Best Possible Solution

Keep #9132 open until the Control UI Sessions view renders the existing lifecycle fields. The likely implementation belongs in ui/src/ui/views/sessions.ts as a Status column or badge mapping running/done/failed/killed/timeout to clear labels, with last-activity/runtime detail from updatedAt, startedAt, endedAt, or runtimeMs, plus focused coverage in ui/src/ui/views/sessions.test.ts. If maintainers want the same visibility in the chat session switcher, ui/src/ui/chat/session-controls.ts can be a separate follow-up.
