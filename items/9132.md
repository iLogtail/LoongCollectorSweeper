## Summary

Keep #9132 open. Current main carries session lifecycle status/timing through the gateway data model and Control UI state updates, but the Sessions table still renders only Key, Label, Kind, Updated, Tokens, Compaction, Thinking, Fast, Verbose, and Reasoning, with no active/completed/timed-out status indicator.

## Best Possible Solution

Keep #9132 open until the Control UI Sessions view renders the existing lifecycle fields. The likely fix belongs in ui/src/ui/views/sessions.ts as a Status column or badge mapping running/done/failed/killed/timeout to clear labels, with last activity/runtime detail from updatedAt, startedAt, endedAt, or runtimeMs and focused coverage in ui/src/ui/views/sessions.test.ts. If maintainers also want status visible while switching chat sessions, ui/src/ui/chat/session-controls.ts can be updated as a second scoped UI improvement.
