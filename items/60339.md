## Summary

Keep open. Current main partially addresses the same offloaded attachment metadata problem for `chat.send`, but the `agent.request` node-events path used by iOS share still parses `offloadedRefs` and then drops them except for cleanup on an over-limit parsed message. The issue remains actionable and is not superseded by the related iOS resizing issue.

## Best Possible Solution

Keep the issue open and implement node-events/iOS share transcript media persistence analogous to `chat.send`: retain `parsed.offloadedRefs`, combine them with inline image persistence in original `imageOrder`, write or rewrite the user transcript entry with `MediaPath`, `MediaPaths`, `MediaType`, and `MediaTypes`, and add focused `server-node-events.test.ts` coverage where mocked `parseMessageWithAttachments` returns offloaded refs.
