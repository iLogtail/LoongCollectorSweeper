## Summary

Current main still does not implement WhatsApp outbound native mentions. The WhatsApp outbound path normalizes text/media and quote options, then sends Baileys text/media payloads without deriving `@number` targets or passing any mention metadata. Existing `mentionedJids` handling remains inbound-only, and related context shows an open PR/item still tracking native outbound mentions rather than a merged implementation.

## Best Possible Solution

Keep #39879 open until the bundled WhatsApp channel plugin implements outbound native mentions. A conservative fix would add a typed mention-JID path or carefully scoped extraction in the WhatsApp outbound send layer, attach the Baileys-supported mention metadata for text and caption/follow-up text sends, and cover text, media captions, and voice-note visible-text follow-up cases with focused tests. Related PR #56863 may be the right implementation vehicle if maintainers choose to revive or merge it, but current main has not shipped the feature.
