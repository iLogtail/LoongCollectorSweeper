## Summary

Keep #39879 open. Current main still does not add Baileys outbound mention metadata for WhatsApp text or caption sends, so `@number` text remains just message text from OpenClaw's outbound path. The related open PR #56863 is a plausible implementation vehicle, but it is not merged or shipped; closed related issue #50944 covered a broader quote-reply plus mention report, while only quote-reply support appears implemented on main.

## Best Possible Solution

Keep #39879 open until the bundled WhatsApp plugin implements outbound native mentions. The best product path is to add a typed mention-JID flow or carefully scoped `@+<digits>`/`@<digits>` extraction in the WhatsApp outbound send layer, pass Baileys `mentions` for text and supported media caption sends, handle the separate visible-text send used for voice notes, and add focused tests. PR #56863 may be the right implementation vehicle if maintainers choose to revive or merge it.
