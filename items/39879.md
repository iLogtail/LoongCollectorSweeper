## Summary

Current main still does not implement WhatsApp native outbound @mentions. The WhatsApp send path sends plain text/captions without Baileys `mentions` metadata, the typed send options have no outbound mention field, tests still assert plain text payloads, and the related implementation PR #56863 is open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39879 open until the bundled WhatsApp plugin ships native outbound mention support. A solid fix should add a typed outbound mention flow, safely extract or accept `@+<digits>` / `@<digits>`, resolve group/LID participant mappings where possible, pass Baileys `mentions` for text and supported media captions, handle the separate visible-text send used for voice notes, and add focused unit coverage. PR #56863 can remain the implementation vehicle if maintainers choose to revive or merge it.
