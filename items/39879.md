## Summary

Keep open. Current `main` still does not implement WhatsApp outbound mention metadata for sends: the WhatsApp web send path builds plain `{ text }` payloads for text messages, the send options type has no `mentionedJids` field, and the generic outbound payload path only normalizes/forwards text and media. I found inbound mention parsing and gating support, but no outbound mention injection or regression coverage for it.

## Evidence

- **Outbound text payload omits mention metadata:** The WhatsApp web sender falls back to `payload = { text }` for text-only messages before calling Baileys. There is no `mentionedJids` population in this code path.
