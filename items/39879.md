## Summary

Keep #39879 open. Current main still sends WhatsApp outbound text/captions without Baileys mention metadata, so `@number` remains plain text from OpenClaw's bundled WhatsApp plugin. The related PR #56863 is still open and unmerged; it is a plausible implementation vehicle, but the feature is not shipped on main.

## Best Possible Solution

Keep #39879 open until the bundled WhatsApp plugin implements outbound native mentions. The likely path is to add a typed outbound mention flow or carefully scoped `@+<digits>` / `@<digits>` extraction in `extensions/whatsapp`, pass Baileys `mentions` for text and supported media caption sends, handle the separate visible-text send used for voice notes, account for LID/group participant resolution, and add focused tests. PR #56863 can remain the implementation vehicle if maintainers choose to revive or merge it.
