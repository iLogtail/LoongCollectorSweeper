## Summary

Keep #58889 open. Current main still downloads Telegram voice media and runs audio preflight transcription while building the Telegram message context, before the reply pipeline exists and before `typingMode: "instant"` can trigger Telegram `sendChatAction("typing")`. The related #39075 performance PR is still open and does not supersede the requested receipt-time typing cue; the duplicate-title ClawSweeper report #58887 was also kept open for the same missing capability.

## Best Possible Solution

Keep the issue open and implement a narrow Telegram inbound voice-receipt typing cue before media download/STT, likely in the Telegram handler path using the existing `sendChatActionHandler` and thread-param helpers. Add a focused test proving `sendChatAction("typing")` fires before `resolveMedia`/`transcribeFirstAudio` for voice messages, while preserving auth/group gating and existing `typingMode: "never"` semantics if maintainers want that policy applied before preflight.
