## Summary

Keep #58889 open. Current main still builds the Telegram message context before dispatch, and audio preflight transcription happens inside that context. The Telegram typing callback is only wired when the reply pipeline is created later, so the requested “typing immediately on voice receipt, before STT transcription” capability remains missing. The related #39052/#39075 pipeline performance items do not supersede this specific UX request, and the one issue comment discusses a WeChat plugin setup rather than a Telegram voice typing implementation.
