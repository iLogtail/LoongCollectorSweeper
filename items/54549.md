## Summary

Keep open. Current main already has background exec completion wakes via notifyOnExit, but it does not implement the proposed per-call onComplete="notify" contract, and the PR discussion identifies unresolved edge cases in the proposed patch. This is not strong enough evidence for conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to rebase and narrow it. If OpenClaw wants this API, implement onComplete as a schema-limited per-call option that explicitly backgrounds local/gateway exec, forces the desired notify behavior including quiet-success notifications, handles or rejects node-host exec consistently, avoids promising wakes when no session routing exists, and adds focused tests plus docs updates.
