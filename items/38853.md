## Summary

Keep #38853 open. Current main has generic channel auto-restart backoff and JSON Discord 429 retry handling, but the Discord startup REST paths still do not classify Cloudflare/Error 1015 HTML as a cooldown signal and can still preserve non-JSON HTML in logged errors for `/users/@me/guilds`.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin startup REST layer: classify Cloudflare/Error 1015 and non-JSON 429 HTML as a rate-limit cooldown signal, sanitize or summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when it is absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses.
