## Summary

Keep PR #72051 open. Current main still lacks both requested fixes: mirrored TaskFlow terminal records can still produce `endedAt < updatedAt`, which the TaskFlow audit reports as `inconsistent_timestamps`, and the packaged build entries still do not emit a stable root `task-registry-control.runtime` artifact even though task cancellation tries to require it. The PR is focused and recent, but the existing review comment about preserving historical `endedAt` semantics should be handled before merge.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve historical terminal `endedAt` when there is no newer event timestamp, clamp only the mirrored TaskFlow case where a later `lastEventAt` would violate `endedAt >= updatedAt`, add the stable `task-registry-control.runtime` tsdown root entry, and keep focused regression tests for both timestamp normalization and the build entry.
