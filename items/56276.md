## Summary

Current main does not implement the requested hostname segment in the TUI status/footer. The TUI header includes the connection URL, which partially helps for explicit remote URLs, but the footer renderer still only includes agent, session, model, modes, and token counts, and there is no hostname field in the TUI session state.

## Best Possible Solution

Keep this issue open and implement it in core TUI code. Add a hostname/host label to the footer with focused tests and docs updates, deriving the best user-facing value from the gateway endpoint for explicit remote URLs and from a gateway/local machine-name source for loopback or embedded local mode.

## Risks / Open Questions

- A clean implementation needs a product choice between displaying the connection URL host, the gateway machine hostname, or the local embedded runtime hostname; these differ for loopback, SSH, and remote Tailscale setups.
- Footer space is already dense, so the display should include truncation or responsive formatting tests for narrow terminals.
