## Summary

Keep open. I did not find strong evidence that current `main` fixes or obsoletes this report. The Nostr bus on `main` still creates a single `SimplePool.subscribeMany(...)` subscription, treats `onclose` as an error, and does not add any readiness wait, reconnect, or resubscribe logic that would clearly address timeout/connection-refused subscription failures.

## Evidence

- **Current main still uses the same direct subscription path and surfaces close as an error:** `startNostrBus()` still calls `pool.subscribeMany(relays, [{ kinds: [4], "#p": [pk], since }], ...)` and its `onclose` handler only emits disconnect/error reporting (`Subscription closed: ...`). I did not find new retry or resubscribe handling around that path.
