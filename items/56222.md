## Summary

Keep issue #56222 open. Current main still does not expose the running gateway version on the actual `openclaw gateway status --json` daemon-status surface. The lower-level gateway probe reads `hello-ok.server.version`, but `probeGatewayStatus` discards it before building the `rpc` object, and `DaemonStatus.gateway` has no version/runtimeVersion field. `openclaw status` has `runtimeVersion`, but that is a different command surface than the requested gateway status/MCP status response.

## Best Possible Solution

Keep the issue open and implement an additive status-field change in the daemon status path: carry `probeGateway(...).server.version` through `probeGatewayStatus` into `DaemonStatus` JSON, preferably as the running gateway runtime version under `gateway.version` or a clearly named `runtimeVersion`, with optional separate CLI/service version metadata if maintainers want skew diagnostics. Update human output, JSON tests, docs, and any MCP `gateway_status` wrapper to mirror the same field.
