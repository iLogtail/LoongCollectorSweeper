## Summary

Keep open. Current `main` still treats `![image](...)` as a bang-prefixed command in the TUI submit path, the `/bash` alias handler, and ACP command bypass, so this PR is not implemented on `main` and the underlying bug remains plausible. The PR discussion also shows the author already addressed the main review concern by narrowing the fix to real Markdown image syntax and reverting the unnecessary inline-command regex change, though the patch now needs a rebase onto renamed files on `main`.
