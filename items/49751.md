## Summary

Keep open. Current `main` has already adopted the PR's timeout increase in a different form, but it has not implemented the PR's main semantic change: the WebSocket handshake timer is still cleared only after the async auth flow completes and `hello-ok` is prepared. The prior PR review also surfaced a real safety concern with the proposed move: merging it as written would remove the only watchdog for a stalled auth path without adding a replacement auth deadline.

## Evidence

- **Current main still clears the handshake timer after auth, not before it:** `resolveConnectAuthState(...)` runs first, and `clearHandshakeTimer()` is only called later when building the connected client and sending `hello-ok`. That means the race this PR is trying to fix is not implemented on `main` today.
