## Summary

Keep #9544 open. Current main still builds the bundled WhatsApp socket on Baileys `useMultiFileAuthState`; the shipped hardening only wraps `creds.json` persistence and legacy-file handling, not the requested production replacement for the Baileys multi-file credential plus signal-key store.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces Baileys multi-file credential and signal-key storage with a production auth store, preserves existing auth through a tested migration path, keeps or deliberately supersedes the current atomic `creds.json` protection, updates auth presence/logout/self-id/docs, and adds focused fresh-install plus migration tests.
