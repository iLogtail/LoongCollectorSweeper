## Summary

Keep #69931 open. The issue form has contradictory expected/actual text, but the summary and steps identify a real missing capability: reading WhatsApp emoji reactions on a specific historical message. Current main supports sending WhatsApp reactions and ack reactions, but the WhatsApp inbound path and public plugin hooks do not expose reaction-observation events or target-message reaction data. This is not a ClawHub-only close yet because the requested skill/plugin cannot be built cleanly without an additive OpenClaw channel/plugin API.

## Best Possible Solution

Keep the issue open as a narrow WhatsApp reaction-observation/plugin-API gap. The best path is an additive contract: have the WhatsApp channel observe Baileys reaction/update events, normalize account/chat id, target message id/key, reactor sender, emoji, add/remove state, timestamp, and fromMe/participant metadata, then expose it through a documented SDK hook or channel event with focused WhatsApp tests and docs. Once that core surface exists, a ClawHub skill can build the higher-level automation.
