## Summary

Keep #69931 open. Current main still supports WhatsApp outbound reactions, but I found no inbound WhatsApp reaction notification/event path, no `reactionNotifications` config for WhatsApp, and no plugin-visible hook payload that exposes emoji, reacting sender, target message id/key, or add/remove semantics. The issue is recent and describes a real WhatsApp channel/plugin API gap, so it should not be closed as ClawHub-only.

## Best Possible Solution

Keep this open as a WhatsApp channel/plugin API gap. The best path is to add plugin-owned WhatsApp reaction ingestion that captures Baileys reaction updates, normalizes target message key/id, emoji, reacting sender, chat/account, and add/remove semantics, applies existing WhatsApp access-control/privacy rules, exposes a typed plugin-visible hook or system event, and documents/tests the new reaction notification config. A ClawHub skill can then consume that supported surface without reaching into private WhatsApp internals.
