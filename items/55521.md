## Summary

Keep open. Current main has Feishu typing reactions, but they are tied to the reply lifecycle rather than an immediate receipt acknowledgment, and this PR's current patch is unsafe as reviewed because it reacts before eligibility gates and bypasses existing typingIndicator semantics.

## Best Possible Solution

Keep the PR open only if the maintainer wants this earlier Feishu UX. The safe implementation would move any acknowledgment until after group policy, allowlist, DM pairing, and requireMention gates; honor account-level typingIndicator and global typingMode semantics; avoid permanent reactions on silent drops; reuse the managed add/remove typing lifecycle; and add focused Feishu tests for disabled typing, stale/replayed messages, mention-required groups, and allowlist drops.
