## Summary

Keep open. Current `main` does not implement a macOS LaunchDaemon mode for the gateway: the CLI exposes only the existing service install/start lifecycle, Darwin resolves to a per-user LaunchAgent, and the launchd code hardcodes `~/Library/LaunchAgents` plus the `gui/$UID` domain. The code even documents a custom LaunchDaemon as 'not shipped,' so this feature request is still outstanding rather than already fixed or obsolete.
