## Summary

Keep #62339 open. Current main still has a plausible WhatsApp-plugin-owned bug path for the reported symptom: automated WhatsApp heartbeat/cron delivery can include pairing-store recipients in the same effective recipient set as configured `allowFrom`, even though allowlist-mode inbound access intentionally does not merge persisted pairing approvals. A maintainer comment on the issue explicitly reached the same conclusion and asked to keep it open.

## Best Possible Solution

Keep the issue open and fix it in the WhatsApp plugin. The likely best change is to separate explicit configured `allowFrom` from pairing-store approvals for automated heartbeat/cron delivery in strict allowlist mode: allow explicit `heartbeat.to`, an active session recipient that is also in configured `allowFrom`, or configured `allowFrom` itself, but do not send automation to pairing-store-only recipients unless the user explicitly opts in. Update `extensions/whatsapp/src/heartbeat-recipients.test.ts` and status/log wording to label configured recipients versus paired-store recipients.
