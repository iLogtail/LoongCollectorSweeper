## Summary

Current `main` does not implement this PR's Linux-container onboarding behavior. `setupSkills` still treats brew-backed skill installs the same in containers as on bare Linux, `src/daemon/container-context.ts` still only recognizes `OPENCLAW_CONTAINER*` hints, and the Docker onboarding path in `scripts/docker/setup.sh` runs `openclaw onboard` inside the container without setting those hints. The PR also includes follow-up discussion addressing exactly that gap, so this remains useful work rather than conservative cleanup candidate.
