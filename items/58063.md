## Summary

Keep open. Current `main` still routes Grok `web_search` through the xAI Responses path and still applies the shared 30-second default timeout to that request path. The provided discussion also includes a current-`main` reproduction showing `AbortError: This operation was aborted` at about 30s and success when the timeout is raised to 60s. I found no code or release-history evidence that this was fixed on `main` or in `v2026.4.23`.
