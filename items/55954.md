## Summary

Keep open. Current `main` has partial route-first/fast-JSON CLI startup work for some read-only commands, but it does not fully implement the requested agent-optimized mode: there is no repo-wide `--agent`/`--machine`/`OPENCLAW_AGENT_MODE` switch, and at least one cited command surface (`agents list`) still loads plugins on startup.

## Evidence

- **Route-first work landed for several read-only commands:** The CLI command catalog now routes `status`, `config get`, `config unset`, `models list`, `models status`, `channels list`, and `channels status`. `status` explicitly sets `loadPlugins: "never"`; `config get` and `models status` are routed entries intended to avoid the old full startup path.
  - file: [src/cli/command-catalog.ts:42](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/cli/command-catalog.ts#L42)
  - command: `nl -ba src/cli/command-catalog.ts | sed -n '1,120p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **`status --json` now skips plugin/config bootstrap:** Route tests on current `main` assert that `tryRouteCli([..., "status", "--json"])` does not call either the config guard or plugin registry loader, which directly addresses part of the cold-start/handshake complaint for that command.
  - file: [src/cli/route.test.ts:79](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/cli/route.test.ts#L79)
  - command: `nl -ba src/cli/route.test.ts | sed -n '70,140p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Lean fast-JSON status path exists:** `scanStatusJsonFast()` uses a reduced status scan policy: no channel rows/details, no plugin compatibility work, and memory inspection only when `--all` is requested. This is strong evidence of partial implementation for machine-readable status flows.
  - file: [src/commands/status.scan.fast-json.ts:50](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/commands/status.scan.fast-json.ts#L50)
  - command: `nl -ba src/commands/status.scan.fast-json.ts | sed -n '1,140p'`

- **Requested global agent/machine mode is still absent:** A repo search found no `OPENCLAW_AGENT_MODE`, no agent-optimized CLI mode, and no CLI `--machine` flag for this purpose. The only `--machine` hits are unrelated systemd code/tests, so the specific flag/env API proposed in the issue is not implemented on `main`.
  - command: `rg -n -e "OPENCLAW_AGENT_MODE" -e "agent-optimized" -e "machine mode" -e "--machine" src docs CHANGELOG.md VISION.md`

That means one of the issue's concrete examples is still not on the lean startup path.
