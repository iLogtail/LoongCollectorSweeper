## Summary

Keep #51918 open. Current main has overlapping pieces: configurable `steer`/`steer-backlog` can inject into a streaming active run, and fast abort handles some stop-style messages. But the requested owner/priority preemption is not implemented: there is no owner-priority or priority flag, default queueing is still `collect`, and the steering path only fires when that mode is selected and the active run is streaming. The supplementary-context use case from the issue comments also remains a product behavior request rather than an already-shipped default.
