## Summary

Keep #70439 open. Current main has the partial Signal media fix: Signal now preserves fetched attachment paths and MIME types, including filename-based MIME fallback for voice notes, so the generic media-understanding pipeline can classify audio. But the central request remains unimplemented: Signal still does not run a channel audio preflight equivalent to Telegram, WhatsApp, or Discord, and there is no `channels.signal.audio`/preflight hook config. The existing maintainer comment explicitly called this partial and recommended a shared channel audio-preflight helper plus Signal hook.

## Best Possible Solution

Keep the issue open and implement a shared channel audio-preflight helper, then wire Signal to run it after access checks and before mention/dispatch decisions. Reuse `tools.media.audio`/`transcribeFirstAudio` and mark `MediaTranscribedIndexes` to avoid double STT; add a channel SDK seam only if bundled and third-party channels need the same preflight contract. Avoid a Signal-only shell hook unless maintainers explicitly want a per-channel escape hatch.
