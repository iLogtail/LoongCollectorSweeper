## Summary

Keep open. Current main partially addresses the older Signal audio metadata bug: Signal now forwards fetched attachment paths and MIME types, and the shared media-understanding pass can classify/transcribe audio later. The requested Signal audio preflight/hook is still missing: Signal mention gating runs before attachment fetch/transcription, Signal never sets Transcript or MediaTranscribedIndexes, and there is no Signal config surface for an audio preflight command/hook. Telegram, Discord, and WhatsApp have explicit preflight paths, so the parity request remains valid. The existing maintainer comment on #70439 also explicitly kept this open for the same reason.

## Best Possible Solution

Keep #70439 open. The best fix is a shared channel audio-preflight helper/seam, wired into Signal after sender access checks and before mention/command gating, using the existing tools.media.audio/transcribeFirstAudio pipeline where possible. Signal should replace placeholder-only agent/command text with the transcript, set Transcript and MediaTranscribedIndexes to avoid duplicate STT, and document any supported template variables only if a command-hook surface is intentionally added.
