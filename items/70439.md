## Summary

Keep open. Current main partially fixed Signal audio metadata so generic media understanding can classify voice attachments, but the requested Signal/shared audio preflight hook is still missing. The issue also has an explicit maintainer comment saying the item is partially addressed, not closed, and should remain open for a shared channel audio-preflight helper plus Signal wiring.

## Best Possible Solution

Keep #70439 open. Implement a shared channel audio-preflight helper that Signal can call after resolving/fetching attachments but before mention gating and agent dispatch, reusing transcribeFirstAudio and marking MediaTranscribedIndexes. Then document Signal behavior, available template variables if an external command hook is added, and format conversion expectations such as ffmpeg for OGG/Opus or MP4 inputs.

## Risks / Open Questions

- Generic media understanding may already transcribe some Signal DMs after context finalization when tools.media.audio is configured; any fix should preserve that path and avoid double STT via MediaTranscribedIndexes.
- A shell-command preflight hook would need careful path, sandbox, timeout, and secret handling. The best path is the maintainer-suggested shared channel audio-preflight helper plus Signal wiring before considering a Signal-only command hook.
