## Summary

Keep #71887 open. Current main still has the reported `channels status`/`--deep` mismatch: the subcommand does not register `--deep`, but the CLI description and both status footer paths still use bare `status --deep` wording. The PR is the active canonical follow-up for #69341, though it should be rebased/updated for the current formatter split.

## Best Possible Solution

Keep the PR open and update or land the same narrow fix on current main: clarify the `channels status` help and both channel-status footer formatters to point at top-level `openclaw status --deep`, then add regression coverage for the config-only footer so the stale guidance does not return.
