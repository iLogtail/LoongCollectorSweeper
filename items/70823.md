## Summary

Keep open. Current `main` still models `plugins.slots.*` as plain plugin-id strings, normalizes only string values, and install flows overwrite the selected slot with a new string plus warnings. The RFC is coherent, not implemented on `main`, and aligns with the repo vision of extending plugin API/contracts in core when plugins need a new seam. The issue body’s `@plur-ai/claw` example and cross-reference to plur-ai/plur#51 strengthen that this is a real ecosystem need rather than a ClawHub-only plugin promotion request.
