## Summary

Keep PR #52951 open. Current main still does not implement `tools.fs.roots`: the config type/schema, effective FS policy, host read/write/edit/apply_patch wiring, media root propagation, docs, and generated config metadata remain `workspaceOnly`-only. The PR is active, directly targets a real missing filesystem access mode, and the provided review discussion shows recent author follow-up on security-sensitive P1/P2 edge cases rather than an obsolete or low-signal drive-by.

## Best Possible Solution

Keep the PR open. Maintainers should decide whether `tools.fs.roots` is the desired core config contract; if yes, review and land a narrowed implementation covering schema/docs, host FS tool enforcement, apply_patch, media/outbound propagation, sandbox bypass behavior, and focused regression tests. If maintainers prefer a different filesystem sandboxing model, close this only with an explicit product decision and link the canonical follow-up, not as automated cleanup.
