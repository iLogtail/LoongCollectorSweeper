## Summary

Keep open. Current `main` still lacks the runtime-plugin activation and model-id canonicalization this PR adds, and the linked bug reports (#70776, possibly #69440) still match the shipped/current code paths.

## Evidence

- **`get-reply` still skips runtime plugin activation before session initialization:** Current `getReplyFromConfig` goes from workspace setup into `initSessionState` with no `ensureRuntimePluginsLoaded`/best-effort equivalent on this path, so thinking validation still depends on whatever runtime registry state already exists.
