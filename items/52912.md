## Summary

Keep open. Current main does not contain the PR's claimed write-time Anthropic setup-token sibling auth sync, and the linked bug report remains a real, broader auth/onboarding issue. The PR should not be merged as-is because the existing review flagged an OAuth cross-agent overwrite risk and the branch includes unrelated stale registry work, but none of the allowed close reasons is strong enough for conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer or contributor follow-up, but require a rebase and a narrower implementation: factor a shared auth-profile writer only if it preserves per-agent isolation, gate sibling sync to the Anthropic setup-token/manual-token case or an explicit setup context, add regression coverage for multi-agent Anthropic token setup, and leave the broader UX/persistence items on #51911 unless addressed in the same reviewed patch.
