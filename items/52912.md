## Summary

Keep open. Current `main` does not implement the Anthropic token sibling-sync fix, and the PR as proposed appears to introduce a real per-agent auth regression by broadening sibling sync to all OAuth credentials in agent-scoped setup flows.

## Evidence

- **Current main still writes plugin-provider auth profiles only to the selected agent dir:** `runProviderPluginAuthMethod` on `main` still calls `upsertAuthProfile` directly for each returned profile and passes only the resolved `agentDir`; there is no Anthropic token sibling-sync path here yet.
  - file: [src/plugins/provider-auth-choice.ts:169](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/plugins/provider-auth-choice.ts#L169)
  - command: `nl -ba src/plugins/provider-auth-choice.ts | sed -n '150,270p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Sibling syncing exists today only in the OAuth helper, not generic auth-profile writes:** `writeOAuthCredentials` supports `syncSiblingAgents`, but that logic is scoped to the OAuth helper. Current `main` has no shared `writeAuthProfile` helper or Anthropic-token-specific sibling-sync policy.
  - file: [src/plugins/provider-auth-helpers.ts:269](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/plugins/provider-auth-helpers.ts#L269)
