## Summary

Keep open. Current main still initializes the Control UI Usage tab start and end dates to today's date, so the PR's central behavior change is not implemented or superseded. The controller and render path pass those state dates through directly, while the existing 7d preset is only a manual action after render.

## Best Possible Solution

Keep the PR open and either merge it or reimplement the same inclusive last-7-calendar-days default in `ui/src/ui/app.ts`. The cleanest durable path is to share/extract the existing preset date formatting logic or add a focused initializer test so the default cannot silently regress back to today-only.
