## Summary

Keep open. Current main still does not implement `channels.signal.configPath`: the Signal config type/schema/docs/generated metadata omit it, the daemon option type and arg builder have no `--config` handling, and `monitorSignalProvider` does not forward a config path into daemon startup. The PR remains useful but needs rebasing from the old `src/signal/*` layout into the current bundled Signal plugin under `extensions/signal/src/*`.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase/adapt it. The best implementation is to add optional `channels.signal.configPath` to the current Signal bundled plugin config contract, trim and omit blank values, pass it to daemon startup as `signal-cli --config <path>`, update docs/generated metadata/help, and add focused schema/autostart tests for set, unset, and blank values.
