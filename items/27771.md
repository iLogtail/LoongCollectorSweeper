## Summary

Keep PR #27771 open. The requested ability to set `channels.signal.configPath` for `signal-cli --config` is still not implemented on current main, and the PR discussion already points toward maintainer takeover rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer takeover or a replacement PR. Add optional `channels.signal.configPath` to the current Signal config type/schema, generated metadata and UI hints, and docs; trim and omit blank values; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; place `--config <path>` before daemon startup arguments; and add focused tests for set, blank, unset, and schema acceptance cases in the current `extensions/signal` layout.
