## Summary

Keep #6615 open. Checked-out main still has only the existing exec-approval modes (`deny | allowlist | full`) and ask modes (`off | on-miss | always`), with no denylist file model, `security: "denylist"`, `ask: "on-match"`, or argument-aware deny matching. The SafeClaw/CRE links in the discussion are useful prior art, but the requested core exec-approvals capability remains unimplemented, and the prior implementation PRs were closed unmerged.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements denylist/on-match/argument-aware deny matching across the exec approvals model, config/schema, runtime policy, gateway protocol, CLI/session surfaces, Control UI, docs, and tests, or maintainers explicitly redirect the capability to ClawHub/plugin policy with any missing authorization API tracked separately.
