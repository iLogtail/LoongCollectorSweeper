## Summary

Keep open. Current main has not fixed the reported OpenAI SDK SSE failure: OpenClaw still consumes OpenAI SDK streaming iterables directly, the fetch wrapper passes SSE bytes through unchanged, and the current direct/transitive SDK versions still unconditionally parse empty `sse.data` after flushing event-only SSE frames. The related closed ClawSweeper item #52679 covered an Anthropic SSE path, not this OpenAI-node path.

## Best Possible Solution

Keep the issue open and add a narrow OpenAI streaming hardening: either consume an upstream OpenAI SDK release that skips empty-data SSE frames across all direct/transitive paths, or add an OpenClaw-controlled sanitizer/custom stream parser before SDK `JSON.parse` for OpenAI Responses, Azure Responses, and OpenAI-compatible Completions. Add regression tests using an in-memory `Response` with `event: message\n\n`, a following valid `data:` frame, and CRLF variants.
