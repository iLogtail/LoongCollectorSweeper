## Summary

Keep open. Current main still has `cacheRead: 0` and `cacheWrite: 0` in the Doubao/Volcengine and BytePlus default model costs, and those defaults are used for both standard and coding catalog entries. Cost estimation multiplies cached-token usage by these configured rates, so the reported undercount remains real. Related PR #55913 directly addresses the same issue, but it is still open and its diff targets the old `src/agents/*` files rather than the current `extensions/*` provider files, so it is useful work to adapt rather than evidence that the issue is already fixed.

## Best Possible Solution

Keep the issue open and adapt the fix from #55913 or submit an equivalent patch against `extensions/volcengine/models.ts` and `extensions/byteplus/models.ts`, with regression coverage that asserts nonzero cache pricing on the current provider catalog path and published-rate evidence for the chosen values.
