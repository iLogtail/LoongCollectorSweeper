## Summary

Keep open. Current `main` has already fixed the specific `seqByRun` leak, but the broader report is not obsolete: the session-store cache still retains duplicated serialized/object forms, and `readSessionMessages()` still materializes whole transcript files on active Gateway paths. That is not strong enough evidence for an auto-close.

## Evidence

- **`seqByRun` cleanup is fixed on `main`:** `clearAgentRunContext()` now deletes both `runContextById` and `seqByRun`, and there is a regression test asserting the sequence resets after cleanup. This addresses one concrete sub-problem from the issue, but only that one.
