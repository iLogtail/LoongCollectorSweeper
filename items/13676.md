## Summary

Keep #13676 open. Current main still lacks the requested first-class Project object, Projects dashboard workflow, project-to-home-session binding, project workspace list, and project-scoped tool/skill/runtime constraints. Existing agents, sessions, workspaces, per-agent skills, and sandbox settings are adjacent primitives, not the Project model requested here. The provided discussion also shows #12333 and #13535 were closed as narrower duplicates/superseded items, making #13676 the canonical tracker rather than something to close.

## Best Possible Solution

Continue tracking #13676 as the canonical product/core dashboard request. A complete solution should add an explicit Project storage/protocol model, expose Projects in Control UI, bind each project to a home session plus optional workstreams, associate one or more workspace paths, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, skill-policy, and sandbox primitives where they fit.
