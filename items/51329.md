## Summary

Keep PR #51329 open. Current main still builds PDF extraction fallback contexts with only a user message, then calls `complete(model, context)` for both extraction branches without passing model/API metadata or setting `systemPrompt`. The Codex Responses transport maps `context.systemPrompt` to top-level `instructions`, and the existing image path already has matching Codex coverage, so this PR remains a focused, still-useful fix rather than obsolete cleanup.

## Best Possible Solution

Keep #51329 open and finish it as a narrow bug fix: rebase onto current main, preserve the Codex-only `systemPrompt` gate for both PDF extraction fallback branches, keep the non-Codex regression guard plus text-only branch coverage, and run the appropriate changed checks before landing.
