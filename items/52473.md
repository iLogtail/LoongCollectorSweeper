## Summary

Keep open. Issue #52473 asks for deterministic SKILL.md injection when a skill is explicitly invoked via slash command, but current main still rewrites non-tool skill commands into a normal model request asking the model to use the skill. The prompt/catalog path still exposes only name, description, and location plus soft instructions to read SKILL.md, and the docs explicitly describe default skill commands as forwarded to the model as a normal request.

## Evidence

- **Slash skill invocation remains a prompt rewrite:** For a resolved skill invocation without tool dispatch or a bundle prompt template, current main builds the agent body as `Use the "<skill>" skill for this request.` plus optional `User input`, then assigns that string to `ctx.Body`/`BodyForAgent`. It does not read or inject the target SKILL.md content.
