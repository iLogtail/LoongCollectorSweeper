## Summary

Keep open. Current main still treats explicit skill slash invocation as a prompt rewrite plus the normal available-skills catalog, not as deterministic SKILL.md content injection. The related Claude CLI plugin path is a separate backend-specific accommodation and does not solve the Gateway slash-command path described in #52473.

## Best Possible Solution

Keep #52473 open. The best path is a core deterministic activation change: when an explicit `/skill <name>` or per-skill slash command resolves to a normal skill, carry or re-resolve the selected skill file, read the bounded SKILL.md content, inject it into the model context as hidden/system context before the agent run, and pass only the user args as the user request. Preserve `command-dispatch: tool` and bundle prompt-template behavior, respect allowlists/gating, add focused tests around `handleInlineActions`/`runPreparedReply`, and update the skills docs.
