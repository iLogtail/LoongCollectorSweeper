## Summary

Current `main` already addresses the reported startup window: scheduled services stay inert until gateway sidecars finish, cron/heartbeat activation happens only after that point, heartbeat delivery checks channel readiness before sending, and regression tests now cover both sidecar gating and startup catch-up responsiveness.
