## Summary

Keep open. Current `main` does not contain the Slack A2A thread-routing fix described in PR #49750, and the PR discussion already surfaced unresolved correctness risks in the proposed patch.

## Evidence

- **Current group-session parsing still uses the first `From` segment:** On `main`, `resolveGroupSessionKey()` still derives `head` from `parts[0]` and `second` from `parts[1]`, with no special handling for `agent:<id>:<provider>:...` keys. That means the PR's first claimed fix is not implemented on `main`.
