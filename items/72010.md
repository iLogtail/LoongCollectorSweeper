## Summary

Keep PR #72010 open. Current main still has the old stuck-session behavior: diagnostics only refresh session activity on queue/state changes, the heartbeat warns solely by processing age, there is no progress-marking API/backoff state, ACP/runtime events are not wired as progress, and the docs still describe an age threshold rather than no-progress semantics. The PR appears to be the active successor to closed PR #71969 and includes follow-up commits addressing the Codex P2 diagnostics-enabled review concern, so it is useful work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The best path is to validate the root stuck-warning false-positive behavior, review the progress/reset/backoff implementation and tests on the PR head, and decide whether the incidental rebased fixes should stay in this PR or be split before merge.
