## Summary

Keep open. Current main still has the original age-based stuck-session warning behavior and does not implement the PR's central progress-reset/backoff semantics for reply/tool/status/block or ACP runtime events. The prior PR #71969 was auto-closed by the too-many-PRs guard, and #72010 is the active replacement rather than a superseded duplicate.

## Best Possible Solution

Keep #72010 open. Rebase it onto current main, drop already-on-main incidental cleanup if it is no longer part of the needed diff, and preserve the core fix: config-gated diagnostic progress marking for real reply/tool/status/block and ACP runtime progress, no typing-keepalive progress, repeated session.stuck backoff, and matching tests/docs.
