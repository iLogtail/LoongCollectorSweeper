## Summary

Keep PR #69691 open. Current main still hardcodes MiniMax VLM detection to `MiniMax-VL-01`, so the PR's direct `MiniMax-M2.7` allowlist behavior is not implemented. The linked user-visible bug #69648 was closed via the current VL-01 media-provider route, but this PR remains a concrete product/API decision about whether direct `minimax/MiniMax-M2.7` image requests should also route through the VLM helper.

## Best Possible Solution

Keep the PR open for maintainer review. If direct `minimax/MiniMax-M2.7` image requests should be supported, land a corrected allowlist change with guard/runtime tests and docs reconciled with the current `MiniMax-VL-01` media-provider design. If maintainers want VL-01 to remain the only image-understanding model, close the PR with that explicit product/API rationale and point to the shipped #69648 route.
