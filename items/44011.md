## Summary

Keep PR #44011 open. Current main only partially covers the requested hook context expansion: `PluginHookMessageContext` exposes `sessionKey` but still lacks `agentId`, inbound `message_received` does not derive/pass agent identity into the typed plugin context, and outbound `message_sending` / `message_sent` still do not receive the available session/mirror identity. The PR is stale against current file organization, but the product/API gap remains real.

## Best Possible Solution

Keep the PR open and adapt it to current main: add optional `agentId` to the current typed message context, thread `sessionKey` and `agentId` through inbound `message_received` and outbound `message_sending`/`message_sent` using the existing session/mirror identity sources, and add focused tests for unset, mirror-wins, and session-only fallback cases. Preserve backwards compatibility by keeping the new fields optional.
