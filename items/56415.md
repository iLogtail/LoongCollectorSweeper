## Summary

Keep open. Current main blocks non-webchat gateway callers when `gatewayClientScopes` is present without `operator.admin`, but it does not implement this PR's stricter "all channels" behavior: scope-less external channel callers are still explicitly allowed by both implementation and tests.

## Best Possible Solution

Keep the PR open for maintainer product/security judgment. If the intended policy is truly `operator.admin` for every `/phone arm|disarm` invocation, update the current `requiresAdminToMutatePhoneControl` helper plus the external-channel tests and docs. If the intended policy is the current split between internal gateway scopes and owner-authorized external channels, close or supersede this PR with an explicit maintainer note documenting that boundary.
