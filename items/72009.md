## Summary

Keep #72009 open. Current main still hardcodes the reply typing TTL to 2 minutes and exposes only typing mode/interval configuration; `typingTtlSeconds` is not present in current config types, schemas, generated schema/help, or docs. Although older PR #42105 also tracks configurable typing TTL, its live diff is narrower and has review gaps for docs/help metadata and session override precedence; #72009 is the newer focused PR that addresses those gaps and has active author follow-up.

## Best Possible Solution

Keep #72009 open for maintainer review as the likely best current implementation path: it fixes the still-missing main behavior, adds the config contract and docs/help metadata, includes test coverage, and addresses the session-over-agent precedence review. If maintainers accept this direction, merge or refine #72009, then close older duplicate PR #42105 and related closed issue references as superseded by the merged implementation.
