## Summary

Keep PR #38685 open. Current main documents Telegram multi-account routing and the individual group access gotchas, but it still lacks the requested same-group multi-agent Telegram walkthrough. The PR also has a remaining correctness issue: its sample should scope the intended Telegram group explicitly instead of using broad `groupPolicy: "open"` without a group allowlist.

## Best Possible Solution

Keep the PR open and request a small docs revision. Preserve the dedicated same-group Telegram multi-agent walkthrough in `docs/concepts/multi-agent.md`, but update the sample to scope the intended Telegram group explicitly, preferably under each participating `channels.telegram.accounts.<id>.groups` entry for a multi-account setup, while keeping the corrected `groupAllowFrom` sender-ID wording and account-level groupPolicy precedence notes.
