## Summary

PR #39654 is obsolete. Current main still keeps `commands.allowFrom` as the documented provider-keyed record shape, but the linked gateway-startup failure is now covered by last-known-good config recovery and the public docs no longer show the flat-array form.

## Best Possible Solution

Close this PR and keep the shipped direction: record-shaped `commands.allowFrom`, public docs that show the record form, and last-known-good config recovery for invalid edits. Future flat-array support should be a deliberate config-contract change with schema, types, docs, and tests aligned.

## Risks / Open Questions

- This close does not mean `commands.allowFrom: ["id"]` is accepted; fresh hand-written configs without a last-known-good backup still need to use the documented record form.
- If maintainers later want flat-array shorthand as a public config contract, it should be reopened as an explicit schema/types/docs change rather than merged as a stale compatibility patch.
