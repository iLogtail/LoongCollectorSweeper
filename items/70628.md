## Summary

Keep open. Current main still has the reported Telegram DM no-visible-response path: it synthesizes a `NO_REPLY` payload, projects it through outbound delivery, and direct silent-reply defaults rewrite that into visible filler text. The current regression test still encodes the buggy behavior, and related PR #70630 remains the appropriate fix path rather than evidence that main is already fixed.

## Best Possible Solution

Keep #70628 open and land a focused #70630-style fix in `extensions/telegram/src/bot-message-dispatch.ts`: for successful Telegram direct-message turns with no visible final reply and no dispatch/non-silent delivery failure, do not synthesize or deliver `NO_REPLY`; finalize status as a handled silent turn. Preserve the existing real-error/non-silent-failure fallback path and update `extensions/telegram/src/bot-message-dispatch.test.ts` so the DM no-visible-response case asserts no chat message is sent.
