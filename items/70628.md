## Summary

Current main still matches the reported Telegram DM bug. The no-visible-response, no-error path still synthesizes a NO_REPLY payload and sends it through Telegram delivery, where direct-session silent-reply defaults rewrite it into visible filler text. The linked fix PR #70630 is still a separate open PR head, not merged into main, and the maintainer comment explicitly kept #70628 open for this reason.

## Best Possible Solution

Keep #70628 open until #70630 or an equivalent fix lands. The best fix is to treat successful Telegram direct turns with no visible final response as a handled silent no-op, avoid synthesizing a transport-visible NO_REPLY fallback for that case, finalize status reactions as done, preserve the real error/non-silent-failure fallback path, and flip the current Telegram dispatch regression test to assert no chat message is sent.
