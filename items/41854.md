## Summary

Keep open. Current main still lacks the PR’s central Groq text-inference onboarding/auth UX: the bundled Groq plugin is audio/media-understanding only, and there is no `groq-api-key`, `--groq-api-key`, `buildGroqProvider`, or Groq text-provider registration on main. The PR should not merge as-is because it targets older hard-coded core wiring, but the remaining user problem is real and #21442 was closed as superseded by this PR rather than by an implemented mainline fix.

## Best Possible Solution

Keep the item open, but require a rework onto the current provider-plugin architecture. The concrete path is to extend `extensions/groq` if maintainers want Groq bundled, or publish a ClawHub/npm provider plugin if it should stay external: add manifest `providers` and `providerAuthChoices`, implement `registerProvider(...)`/catalog/default model through the provider SDK, wire `GROQ_API_KEY` and `--groq-api-key` via plugin auth metadata, add focused picker/non-interactive/provider-discovery tests, and remove unrelated secret-baseline churn.
