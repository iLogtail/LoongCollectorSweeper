## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native card buttons for shared presentation payloads: Feishu presentation buttons render as markdown fallback text, and the Feishu outbound adapter does not expose renderPresentation or sendPayload. Existing card-action plumbing is useful groundwork for quick-action and approval cards, but it is not wired to arbitrary inline button rendering from the message/presentation contract. Related PR #47891 is narrower channelData/sendPayload card work and does not supersede this issue.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared MessagePresentation button blocks in the bundled Feishu plugin: add presentationCapabilities/renderPresentation or equivalent runtime outbound support, generate Feishu Schema 2.0 action/button elements using structured interaction envelopes, route clicked values through handleFeishuCardAction, support URL-button fallback semantics, and add tests for button rendering, URL buttons, fallback behavior, and callback dispatch. Keep #47891 as separate narrower sendPayload/channelData card work unless maintainers deliberately make it the canonical implementation for this broader issue.
