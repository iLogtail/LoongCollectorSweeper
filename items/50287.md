## Summary

Keep open. The split guardrail request is not implemented on current main: the config schema and runtime type still accept `audio` and `video` in `models.providers.*.models[].input`, while catalog normalization only recognizes text/image/document and silently drops other values. The related umbrella #50285 is closed, but this narrower model-input validation work remains unresolved.

## Best Possible Solution

Keep this issue open and implement a focused config-contract pass: define the supported modality matrix for `models.providers.*.models[].input`, update `ModelDefinitionSchema` and the model catalog normalizer to either reject unsupported values or support them consistently, add targeted valid/invalid config tests, and document the validation expectations in the model provider docs.

- Closing now would lose a still-actionable config-safety issue: current main accepts the example modality values while downstream catalog and media capability paths do not consistently support them.
