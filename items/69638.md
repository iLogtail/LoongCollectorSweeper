## Summary

Keep PR #69638 open. Current main still checks only `connectAuth.password` in password-mode gateway auth, while the WebSocket handshake path preserves token-only payloads separately and passes them into the same authorizer. The PR remains a focused, actionable fix for the linked Rabbit R1/Funnel regression rather than obsolete or implemented.

## Best Possible Solution

Keep the PR open and either land or recreate the minimal password-mode fallback in `src/gateway/auth.ts`, preserving `connectAuth.password` priority while accepting `connectAuth.token` when password is absent. Keep the PR's regression coverage for token-only success, token mismatch, and dual-field priority.
