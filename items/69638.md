## Summary

Keep PR #69638 open. Current main still does not accept `connectAuth.token` as a password-mode credential, while the linked issue #51953 has already been closed in favor of this exact active PR. The PR is still useful and canonical: it touches the implicated gateway auth branch and adds the happy-path plus edge-case regression tests requested by review.

## Best Possible Solution

Keep #69638 open and review/merge this PR or an equivalent maintainer-authored patch. The right fix belongs in `src/gateway/auth.ts`: in password mode, prefer `connectAuth.password` and fall back to `connectAuth.token`, with regression tests for token-only success, token mismatch rejection, and password-over-token precedence. Longer-term Rabbit onboarding can move toward device-specific bootstrap tokens, but that follow-up should not block this compatibility fix.
