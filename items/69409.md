## Summary

Keep #69409 open. Current main still automatically restarts for restart-required external config changes in the default hybrid reload mode, and the existing controls only offer hot/restart/off behavior, not a queued human approval gate. The later Slack outage comment is useful evidence of operational impact, but it does not make the restart-approval feature obsolete.

## Best Possible Solution

Keep the issue open and design a first-class restart approval policy under `gateway.reload`, likely with auto/notify/manual semantics. The implementation should queue restart-required external config changes with changed paths and reason, expose CLI/API approval such as `openclaw gateway approve-restart`, integrate with existing operator approval/notification routing where appropriate, and preserve current hybrid/hot/off behavior for deployments that do not opt in.
