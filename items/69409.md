## Summary

Keep #69409 open. Current main has reload modes that can avoid automatic restarts (`hot`/`off`) and restart deferral while active work drains, but it does not implement the requested approval/notification queue for restart-required direct config edits. In default `hybrid`, restart-required paths still proceed to restart scheduling; in `hot`, they are only logged/ignored, not queued for later approval.

## Best Possible Solution

Keep this open for maintainer/product judgment on a core gateway reload policy: add a first-class `gateway.reload` policy for restart-required direct-file edits that records a pending restart, emits an operator-visible notification through existing admin/approval surfaces, exposes a CLI/RPC approval action, and preserves current `hybrid`, `hot`, and `off` behavior as compatibility modes. If the Slack 408 crash remains reproducible, split it into a separate Slack socket-mode recovery issue with logs and version details.
