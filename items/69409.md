## Summary

Keep #69409 open. Current main still lacks a human approval/notification queue for restart-required direct config edits. `plugins.allow` remains restart-required, and default `gateway.reload.mode="hybrid"` queues a gateway restart automatically after validation and active-work drain. Existing `hot`/`off` reload modes are workarounds, not the requested pending restart approval flow.

## Best Possible Solution

Keep the issue open for a core gateway reload policy design. A good implementation would add a documented `gateway.reload` policy for restart-required changes, persist or expose a pending restart record, notify configured admin/approval targets, provide CLI/RPC approve/cancel actions, preserve current `hybrid`/`hot`/`off` compatibility, and integrate approval with the existing active-work drain and restart audit paths.
