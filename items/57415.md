## Summary

Keep #57415 open. Current main still lacks a public `openclaw agent --cwd` option, does not forward a CLI cwd override through gateway or embedded fallback paths, and the gateway agent schema still rejects public `workspaceDir` params. The PR is not obsolete; it is a focused implementation of a real gap, but it should resolve the remaining review concern around trusted workspace override normalization before merge.

## Best Possible Solution

Keep the PR open and ask for an update rather than closing it. The best path is to rebase onto current main, preserve the `openclaw agent --cwd <dir>` CLI surface, normalize CLI paths before gateway/local dispatch, add `workspaceDir` to the agent gateway protocol and generated Swift models, enforce trusted-owner authorization for gateway overrides, resolve the remaining trusted-override normalization review, and keep targeted CLI/gateway regression coverage plus docs aligned.
