## Summary

Current `main` already supports OpenRouter backend pinning/routing through model params via `params.provider`; the support landed in commit `a43c1f880757e830b5fb4d3c7b5532a625f0e3e4` on April 24, 2026, after the latest tagged release `v2026.4.23`, and is covered by docs and tests.

## Evidence

- **OpenRouter plugin reads routing from model params:** The OpenRouter provider wrapper reads `ctx.extraParams.provider` when it is an object and injects it into `model.compat.openRouterRouting` before shared stream wrappers run. That is the wiring needed for OpenRouter request-body routing metadata from model config.

## Risks / Open Questions

- The implementation uses the raw OpenRouter request key `params.provider` rather than a separate `providerPreference` alias, so the requested capability exists but under a slightly different config name.
- The change landed after `v2026.4.23`, so users on the latest release will not have it until a newer release ships.
