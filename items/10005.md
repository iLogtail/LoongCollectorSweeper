## Summary

Keep open. Current origin/main still does not provide a config option or tool parameter to hide the rendered chat/session status card. The related closed CLI `openclaw status --json` work is a separate command-line surface and does not satisfy chat `/status` or the `session_status` agent tool requested here.

## Best Possible Solution

Keep #10005 open until OpenClaw either adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer makes a product decision not to support it. A complete implementation should add config schema/types/help/docs, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, preserve current rendered-card behavior by default, and cover both chat and agent-tool paths in targeted tests.
