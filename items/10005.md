## Summary

Keep #10005 open. Current main still renders chat `/status` and the `session_status` tool as a status card, and there is no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression control for those chat/tool surfaces. The related closed #13450 work is CLI `openclaw status --json`, which does not satisfy this request.

## Best Possible Solution

Keep #10005 open until OpenClaw either adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer makes a product decision not to support that behavior. A complete implementation should add config schema/types/help/docs, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, keep current card rendering as the default, and add targeted tests for both chat and agent-tool paths.
