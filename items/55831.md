## Summary

Keep open. Current main still has the stale-cache behavior this PR targets: `sessions.list` is typed as possibly `undefined`, but `loadSessionsOnce` only assigns `state.sessionsResult` inside `if (res)`, so an empty/undefined successful response leaves the previous count visible. The PR is useful, but its current diff also clears `sessionsResult` on every exception, which the existing review correctly flags as a separate UX behavior change that should be revised or explicitly accepted before merge.

## Best Possible Solution

Keep the PR open and revise it to the narrow fix: assign `state.sessionsResult = res ?? null` for successful `sessions.list` calls, keep checkpoint refresh logic guarded to real results, preserve current generic-error cache behavior unless explicitly changed, and add a focused `ui/src/ui/controllers/sessions.test.ts` regression test where a previous cached result is cleared when `sessions.list` resolves to `undefined`.
