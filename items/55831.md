## Summary

Current `main` does not implement the requested stale-session fix, so this PR should not be closed as already handled. The PR also appears to include an extra error-path change that can regress the UI by clearing last-known session state on transient failures, so it is useful work but needs revision rather than cleanup closure.

## Evidence

  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
