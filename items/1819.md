## Summary

本 PR 修复了 external 目录下第三方 syslogparser 副本中 RFC3164 TAG 解析未限制长度导致的潜在数组越界缺陷。修改引入 32 字节长度限制并重构终止符处理逻辑，符合 RFC3164 Section 4.1.3 规范，属于有效的生产问题修复。当前 main 分支未见已移除该 external 副本或已通过依赖管理升级替代此修复的证据，且无其他关联 PR 覆盖此改动。

## Best Possible Solution

建议维护者审查本 PR 对 external 副本的修改。若项目已全面迁移至 Go Modules 且不再维护 external/github.com/jeromer/syslogparser 副本，请引导作者通过更新 go.mod 依赖版本或向上游提交 Patch 解决；若项目仍依赖此 external 代码，请评估合并该边界修复，并建议作者补充针对超长 TAG 及各类终止符场景的单元测试，验证现网 syslog 采集链路兼容性后合入。
