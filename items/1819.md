## Summary

本 PR 修复了 RFC3164 syslog 解析器中 TAG 字段未限制长度导致的潜在越界问题，修改位于 external/ 目录下的第三方依赖代码。因涉及静态拷贝补丁与上游依赖同步的策略选择，且为有效缺陷修复，需维护者审查依赖管理策略并补充边界测试后合入。

## Best Possible Solution

建议维护者审查本 PR 的依赖管理策略。若项目采用 external/ 静态拷贝模式，可合入本 PR 并补充边界测试（如32字符、超长TAG、特殊分隔符）；若已转向 Go Modules，应评估向上游 jeromer/syslogparser 提交修复或通过 go.mod 更新依赖版本。合并前建议补充对应单元测试以验证越界修复与 RFC 3164 兼容性。
