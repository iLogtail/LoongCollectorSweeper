## Summary

本 PR 修复了 RFC3164 syslog 解析器中 TAG 字段未限制长度导致的潜在数组越界问题，修改位于 external/  vendored 依赖目录。该修复符合 RFC3164 规范且作者报告了生产环境实际触发案例，属于有价值的真实缺陷修复，当前 main 分支尚未合并此变更。

## Best Possible Solution

建议维护者进行代码审查。需重点确认 external/ 目录的依赖管理策略：若为静态拷贝可直接合并本 PR 或引导作者将修复提交至上游 jeromer/syslogparser 后同步；若已使用 Go Modules，应评估是否通过 replace 机制或上游发版引入。合并前请作者补充单元测试，覆盖 TAG 长度恰好为 32、超过 32、缺失或含特殊字符等边界场景，确保不破坏现有 syslog 采集兼容性。
