## Summary

本 PR 修复了 RFC3164 syslog 解析中 TAG 字段未限制长度导致的数组越界问题，符合 RFC 3164 Section 4.1.3 规范。但修改直接作用于 external/ 目录下的第三方依赖拷贝，且未补充边界测试用例。属于有价值的真实缺陷修复，需维护者确认依赖管理策略并补充测试后合入，故保持开启。

## Best Possible Solution

维护者需评估 external/ 目录的依赖维护策略：若项目仍手动维护该拷贝，可要求作者补充 32 字符临界与超长 TAG 的单元测试后合入；若已转向 Go Modules，应引导作者向上游 jeromer/syslogparser 提交修复或通过 replace 升级。合并前需人工确认架构归属并验证越界修复与 RFC 兼容性。
