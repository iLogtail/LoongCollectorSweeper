## Summary

本 PR 修复了 RFC3164 syslog 解析器中 TAG 字段未限制长度导致的潜在数组越界问题。当前 main 分支尚未包含此修复，且涉及 external/ 第三方代码修改策略，需维护者人工审查依赖管理方式与测试覆盖后决定合入路径，故保持开启。

## Best Possible Solution

建议维护者进行代码审查。重点评估 external/ 目录下第三方依赖的管理策略：若为静态拷贝可直接合入本 PR 或引导作者向上游 jeromer/syslogparser 提交修复后同步；若已使用 Go Modules 应评估通过 replace 或上游发版引入。合并前建议补充 TAG 长度边界（如正好32字符、超过32字符、含特殊分隔符）的单元测试。
