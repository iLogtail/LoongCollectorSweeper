## Summary

本 PR 修复了 RFC3164 syslog 解析中 TAG 长度未限制导致的潜在数组越界问题，符合 RFC 3164 Section 4.1.3 规范。但修改直接作用于 external/ 目录下的第三方依赖拷贝，且未补充边界单元测试。保守评估建议保持开启，等待维护者明确依赖管理策略（上游修复或本地 Vendor 补丁）并要求补充测试覆盖。

## Best Possible Solution

建议维护者评估依赖管理策略：若项目允许直接修改 external/ 拷贝，请要求作者补充 32 字符边界与超长 TAG 的单元测试后合入；若项目已转向 Go Modules 或要求上游同步，应引导作者向 jeromer/syslogparser 提交修复，并通过升级依赖版本解决。需人工确认架构归属与测试覆盖后再合并。
