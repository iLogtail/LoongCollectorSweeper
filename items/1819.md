## Summary

本 PR 旨在修复 RFC3164 syslog 解析中 TAG 长度未限制导致的潜在数组越界问题，并使其符合 RFC 32 字符限制。但修改直接作用于 external 目录下的第三方库副本，未同步补充单元测试，且未明确项目的依赖管理策略。属于有价值但需工程规范对齐的可挽救 PR，建议保持开启。

## Best Possible Solution

建议维护者确认对 jeromer/syslogparser 的依赖管理策略。若项目已全面转向 Go Modules，应引导作者通过 go.mod replace 或向上游提交 PR 引入修复；若确需维护 external 副本，请作者补充针对超长 TAG 及终止符边界的单元测试，并验证修改后的解析逻辑是否完全兼容现有生产日志格式。待依赖路径明确且测试覆盖完善后推进合并。
