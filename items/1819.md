## Summary

本 PR 修复了 RFC3164 syslog 解析器中 TAG 字段未限制长度导致的潜在数组越界问题。修改位于 external/ 目录下的第三方依赖代码，当前 main 尚未合入此修复。因涉及外部依赖管理策略（静态拷贝 vs Go Modules）及上游同步问题，需维护者人工审查后决定合入路径，故保持开启。

## Best Possible Solution

建议维护者审查本 PR。重点评估 external/ 目录下第三方依赖的管理策略：若项目采用静态拷贝（vendor）模式，可直接合入本 PR 或要求作者补充边界测试（如32字符、超长TAG、特殊分隔符）；若已全面转向 Go Modules，应评估通过 go.mod replace 指令或向上游 jeromer/syslogparser 提交修复后同步更新依赖版本。合并前建议补充对应单元测试以验证越界修复与 RFC 3164 兼容性。
