## Summary

本 PR 修复了 external 目录下第三方 syslogparser 副本中 RFC3164 TAG 解析未限制长度导致的潜在数组越界缺陷。修改引入了 32 字节长度限制并重构了终止符处理逻辑，属于有效的生产问题修复。当前上下文未见 main 已移除该副本或上游已合并此修复的证据，需维护者评估依赖管理策略与测试覆盖后决定合并路径。

## Best Possible Solution

建议维护者审查本 PR 对 external 副本的修改。若项目已迁移至 go.mod 依赖管理或上游 jeromer/syslogparser 已发布包含此修复的版本，请引导作者通过更新依赖版本解决；若项目仍维护此 external 代码副本，请评估合并该边界修复，并建议作者补充对应的单元测试用例以覆盖超长 TAG 及各类终止符场景。合并前需确认该修改不会破坏现有 syslog 采集链路的兼容性。
