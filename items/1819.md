## Summary

本 PR 针对 RFC3164 syslog 解析中 TAG 长度未限制导致的数组越界问题提供了修复，但直接修改了 external 目录下的第三方库副本且未补充单元测试。该修复具有明确的生产价值，但涉及依赖管理策略确认（维护 external 补丁 vs 迁移 go.mod 升级上游）及工程规范完善，属于可能仍有价值的陈旧 PR，故保持开启以待维护者评审与协作。

## Best Possible Solution

建议维护者明确对 jeromer/syslogparser 的依赖演进路线：若已采用 Go Modules，应关闭本 PR 并引导通过升级上游版本解决；若继续维护 external 副本，请作者或社区补充针对超长 TAG 及游标边界条件的单元测试，验证无回归后合入。在依赖策略明确且测试覆盖完善前，保持本 PR 开放。
