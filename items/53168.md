## Summary

Keep open. Current main has generic gateway handshake hardening and Telegram polling-stall recovery, but I could not verify that the exact reported scenario, Telegram long polling active while CLI RPC commands connect, is fixed end-to-end. A newer related open report (#68944) also tracks fresh CLI gateway handshake hangs, so this should not be auto-closed as implemented.

## Best Possible Solution

Keep this open, or have a maintainer explicitly mark #68944 as the canonical remaining tracker and then close #53168 as duplicate. The best code path is a focused regression that starts a gateway with Telegram polling in a controlled stalled/long-poll state, runs CLI RPCs such as `cron.list`, `cron.runs`, and `gateway status`, and asserts the CLI either connects or fails gracefully while the gateway process stays alive. Any fix likely belongs around `src/gateway/client.ts`, `src/gateway/call.ts`, and the Telegram polling session if the interaction is real.

## Evidence

The commenter repeated the same Telegram-polling + CLI-handshake crash concern.
