## Summary

Keep open. Current `main` does not show a full fix for this report: the Control UI still treats reconnects as losing any in-flight chat run, and the only shipped reconnect-related webchat fix I found is a narrower Stop-button abort queueing change, not preservation of active runs or elimination of `terminated` lifecycle endings.

## Evidence

- **Reconnect still drops active local run state:** `connectGateway` explicitly says an in-flight run's final event is lost during the disconnect window and clears the active run/stream state on hello, which is not evidence of preserving an ongoing session across reconnects.
