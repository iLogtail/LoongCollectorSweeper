## Summary

Keep #62126 open. Current main has the native Codex web-search configuration/runtime support and a configure-wizard prompt, but the onboarding path still imports `setupSearch` from `onboard-search.ts`, and that module currently aliases the managed-provider flow directly. The PR remains actionable because its central change is to make onboarding offer the native Codex option before falling through to managed providers.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase that preserves the existing configure behavior, moves the Codex-native prompt into a shared helper or equivalent onboarding path, and keeps focused tests for OpenAI Codex OAuth accept, decline-to-managed-provider, and non-Codex skip cases.
