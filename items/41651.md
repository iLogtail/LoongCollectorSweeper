## Summary

Keep open. Current main still warns when a skill candidate's realpath escapes the configured root before cross-source deduplication, so the issue's requested behavior for symlinked personal skills pointing at already-loaded bundled/plugin skills is not implemented. Related runtime staging work appears narrower: bundled plugin skill trees are copied into dist-runtime, but that does not suppress duplicate warnings for ~/.agents/skills symlinks.
