## Summary

Keep open. Current main fixes one adjacent packaged-bundled case by copying bundled plugin skill trees into dist-owned locations, but the reported personal `~/.agents/skills` symlink-to-bundled workflow still goes through the generic realpath containment path, warns on symlink escapes, and is filtered before source precedence/duplicate merging can suppress the noise.

## Best Possible Solution

Keep the realpath containment guard, but add a narrow loader-level suppression for non-actionable escaped symlink warnings when the same skill is already available from a trusted source such as bundled or managed skills. A good fix should cover `agents-skills-personal`/project `.agents` symlinks, preserve security rejection for unique escaped skills, and add a regression test that proves duplicate symlink warnings are suppressed without loading the escaped copy.
