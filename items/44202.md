## Summary

Keep #44202 open. Current main still does not provide official Apple Silicon Metal/GPU guidance, a CPU-only local embedding config, a `memorySearch.local.gpu`-style control, or process isolation for native `node-llama-cpp` failures. The related #48927 context is not enough to close this because that PR was closed unmerged, and current source still runs the local provider through `node-llama-cpp` directly with concurrent per-batch embedding calls.

## Best Possible Solution

Keep the issue open and resolve it with an explicit maintainer-supported local embedding strategy: document the current safest Apple Silicon path, add a supported CPU-only or GPU backend config if OpenClaw wants to own that surface, serialize or otherwise constrain local `node-llama-cpp` embedding calls, and consider isolating native local embedding work from the gateway process so Metal/ggml assertions cannot terminate the gateway.
