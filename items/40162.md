## Summary

Keep open. Current `main` has a recent fix for the CLI `plugins update` unsafe-override path, but the Feishu/channel setup flow still goes through onboarding install code that does not thread `dangerouslyForceUnsafeInstall`. Feishu setup also still defaults to npm install, so the reported dangerous-code warning can still block setup rather than being clearly resolved on `main`.
