## Summary

Current `main` still has a concrete unresolved path for this report: `applyExtraParamsToAgent()` does not forward `agentDir`/`workspaceDir` into the provider `wrapProviderStreamFn` context, even though the Codex native search wrapper depends on `agentDir` to detect persisted `openai-codex` auth. That means native Codex `web_search` can still be skipped as `codex_auth_missing` for direct `openai-codex` runs, so this issue is not safe to close.

## Evidence

- **Provider wrapper call still drops agentDir/workspaceDir:** `applyExtraParamsToAgent()` passes only `config`, `provider`, `modelId`, `extraParams`, `thinkingLevel`, `model`, and `streamFn` into `providerRuntimeDeps.wrapProviderStreamFn({ context })`. It does not include `agentDir` or `workspaceDir` in that wrapper context on current `main`. This is the exact seam the issue comment identified.

So the omission is not a contract limitation; current callsite is simply not forwarding fields that the hook type supports.
