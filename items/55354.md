## Summary

Current main still has the reported display-name bug path. `buildGatewaySessionRow()` still calls `buildGroupDisplayName()` whenever `entry.channel` is present, without checking whether the session is actually `group`/`channel`, and the TUI footer displays that returned `displayName`. A direct main session with `channel: "telegram"`, no explicit display name, and origin label `openclaw-tui` can therefore still render as a group-style Telegram label.

## Best Possible Solution

Keep the issue open and fix `buildGatewaySessionRow()` so group-style display-name synthesis only runs for actual group/channel sessions, likely using `entry.chatType === "group" || entry.chatType === "channel"` or the parsed group key. Add a regression test for `agent:main:main` with `channel: "telegram"`, `chatType: "direct"`, no explicit `displayName`, and `origin.label: "openclaw-tui"`, asserting the row/TUI-visible display falls back to the direct-session label rather than `telegram:g-agent-main-main`.
