## Summary

Keep PR #43832 open. Current main does not implement the requested graceful cron drain: hot reload still calls the synchronous `stop()`, `stop()` only clears the timer, and current cron timer/manual-run paths can still perform work and persist outside the stop call. The PR is recent, focused on a real remaining code path, and has active follow-up addressing prior review feedback, although its latest form still needs maintainer review around bounded drain behavior.

## Best Possible Solution

Keep the PR open. The best path is to finish a maintainer-reviewed cron graceful-stop implementation that prevents post-reload stale persists for locked operations, timer ticks, and queued manual runs, while bounding or otherwise safely handling hung/long-lived cron executions so config reload cannot block forever. Update the hot reload test from `stop()` to the new graceful path and keep the focused regression tests for locked, timer, and manual-run races.
