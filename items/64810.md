## Summary

Keep open. Current main still lacks the active-reply-run guard described in #64823/#64963; `runHeartbeatOnce` only skips on main/session command-lane depth, while #64810 is specifically about the window where those lanes can look idle but the reply operation is still active before final delivery.

## Best Possible Solution

Keep #64810 open until #64963 or an equivalent patch lands. The likely fix belongs in `src/infra/heartbeat-runner.ts`: after preflight resolves `sessionKey` and before the existing session-lane queue check, call `resolveActiveReplyRunSessionId(sessionKey)` and return `{ status: "skipped", reason: "requests-in-flight" }` when a reply run is active, letting `heartbeat-wake.ts` retry. Add the missing regression with idle main/session lanes, a queued system event, and a live reply operation for the same Telegram-topic session key.
