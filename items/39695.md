## Summary

Keep open. Current main has fresh-onboarding and session-persistence improvements, plus newer partial tool-policy mitigations, but it still does not fully close the reported legacy Discord exec/no-session-log failure: explicit `messaging` profiles still exclude `exec` by default, the targeted legacy migration PR #40202 was closed unmerged, and the silent fake-tool-call surfacing PR #55923 is still open/not present on main.

## Best Possible Solution

Keep #39695 open until maintainers ship a targeted fix for upgraded/explicit messaging-profile installs or a clear diagnostic path. The likely product path is either a narrow legacy local-onboard migration like #40202, a doctor/startup warning when SOUL/profile expectations require `exec` but the effective tool policy excludes it, and regression coverage for a Discord-bound session that both attempts exec and persists a JSONL transcript. The silent fake-tool-call/error-surfacing work from #55923 can be linked as related, but it should not be treated as closing this issue until merged or split into a shipped equivalent.
