## Summary

Keep open. Current main improves fresh local setup and has session transcript plumbing, but it does not fully close the reported Discord/legacy-profile failure: the `messaging` profile still excludes `exec`, existing explicit profiles are preserved, there is no current legacy migration from `tools.profile: "messaging"` to `"coding"`, and the related silent-failure surfacing work remains partial/open in the provided context.

## Best Possible Solution

Keep #39695 open until maintainers either land a narrow legacy local-onboard profile migration/doctor repair or document and diagnose the `messaging`-profile exec exclusion clearly. Add a Discord-bound regression that proves a URL-triggered agent run has `exec` available under the affected upgraded config and creates a transcript, and link or land the visible silent-failure diagnostics work from #55923 as the canonical surfacing fix if that becomes the chosen path.
