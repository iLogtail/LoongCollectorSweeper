## Summary

Keep #39695 open. Current main improves Discord session routing and transcript plumbing, but it still has an unresolved configuration-dependent exec gap: `tools.profile: "messaging"` excludes `exec`, explicit profiles are preserved, and I found no merged legacy migration or Discord-bound regression proving an inbound Discord turn can execute `exec` and persist the JSONL transcript for this report.

## Best Possible Solution

Keep #39695 open until maintainers either ship a narrow migration or doctor warning for legacy/explicit messaging-profile configs that expect runtime tools, or add a targeted Discord-bound regression proving an inbound Discord turn can use `exec` and persist a JSONL session transcript. If the intended product boundary is that messaging profiles must not run shell commands, add a clear user-facing diagnostic telling users to configure `tools.profile: "coding"`, `tools.alsoAllow`, or `tools.exec`.
