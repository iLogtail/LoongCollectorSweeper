## Summary

Keep open. Current main does not provide enough source-backed evidence to close this regression: the macOS app still enables Sparkle auto-checks/downloads by default for signed app bundles, and the config-loss path is only audited/preserved as a suspicious clobber snapshot rather than clearly prevented or restored. The only user-supplied workaround in the discussion (`npx clawaid`) was explicitly reported as not helpful, and the provided related items are title-overlap macOS companion issues, not a canonical duplicate for beta auto-update plus config loss.

## Best Possible Solution

Keep this open and handle it as a macOS app/update-state regression. The best path is a focused macOS owner decision plus tests: make the Sparkle feed/channel policy explicit so stable users cannot be moved onto beta builds without opt-in, consider prompt-only or default-off background downloads, and add a protected config recovery path for reinstall/update clobbers of `~/.openclaw/openclaw.json` (backup/restore or doctor-driven repair with UI/log guidance).
