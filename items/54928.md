## Summary

Keep open. Current `main` still does not implement `plugins.entries.exa.config.webSearch.baseUrl`, and the fallback requested in the issue is not present either: the Exa plugin schema only allows `apiKey`, the runtime hardcodes `https://api.exa.ai/search`, and the docs still document only the API-key path.

## Evidence

- **Exa plugin config schema rejects extra fields:** `webSearch` is strict (`additionalProperties: false`) and its only declared property is `apiKey`, so `baseUrl` is not currently supported in plugin config.
  - file: [extensions/exa/openclaw.plugin.json:26](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/exa/openclaw.plugin.json#L26)
  - command: `nl -ba extensions/exa/openclaw.plugin.json | sed -n '1,220p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Exa runtime uses a fixed endpoint:** The provider defines `EXA_SEARCH_ENDPOINT = "https://api.exa.ai/search"` and later always sends requests to that constant, with no config-derived `baseUrl` path.
  - file: [extensions/exa/src/exa-web-search-provider.runtime.ts:25](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/exa/src/exa-web-search-provider.runtime.ts#L25)
  - command: `nl -ba extensions/exa/src/exa-web-search-provider.runtime.ts | sed -n '1,520p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Fixed endpoint is the one actually called:** `runExaSearch()` passes `url: EXA_SEARCH_ENDPOINT` into `withTrustedWebSearchEndpoint(...)`, confirming there is no alternate configured base URL on current `main`.
  - file: [extensions/exa/src/exa-web-search-provider.runtime.ts:345](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/exa/src/exa-web-search-provider.runtime.ts#L345)
