## Summary

Keep #54475 open. Current main still rejects the first gateway close in the one-shot CLI call wrapper and still reports connect failures before reconnect can complete; the PR's narrow pre-hello code-1000/empty-reason tolerance is not implemented on main. The Greptile concern from the PR discussion was also addressed in the PR context by switching to a structured close error, so the remaining branch is a plausible focused gateway fix rather than cleanup fodder.

## Best Possible Solution

Keep #54475 open and review it as the focused fix for the remaining pre-hello close race. The best implementation is the narrow current PR direction: track hello-ok in src/gateway/call.ts, ignore only pre-hello code 1000 closes with an empty reason, use a structured GatewayClient close error in src/gateway/client.ts instead of message matching, and keep the focused src/gateway/call.test.ts and src/gateway/client.test.ts coverage. Broader transient gateway retry or cron quieting work can remain separate.
