## Summary

Current `main` does not implement this PR's Exa `webSearch.baseUrl` support. The Exa manifest still rejects any `baseUrl` field, the runtime still posts only to the hardcoded `https://api.exa.ai/search` endpoint, and the cache key still does not vary by endpoint. This PR should stay open if the feature is still wanted, though it will need a rebase because the touched surfaces have shifted on `main`.
