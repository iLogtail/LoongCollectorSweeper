## Summary

Keep #54939 open. Current main still does not support `plugins.entries.exa.config.webSearch.baseUrl`: the Exa manifest schema only allows `apiKey`, runtime always posts to the fixed Exa search endpoint, docs still only show `apiKey`, and related issue #54928 remains open. The PR is directionally useful and its latest head addresses earlier review issues, but it needs rebase/adaptation to current main because the Exa implementation has moved into the runtime sidecar file.

## Best Possible Solution

Keep the PR open, ask for or apply a rebase onto current main, and adapt the patch so `webSearch.baseUrl` is added to `extensions/exa/openclaw.plugin.json`, resolved and passed inside `extensions/exa/src/exa-web-search-provider.runtime.ts`, included in the Exa cache key, covered by the endpoint-resolution tests, and reflected in current docs/config metadata paths. The linked issue #54928 should remain tied to this PR until that behavior ships.
