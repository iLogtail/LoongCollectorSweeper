## Summary

Current `main` does not fully implement this request. `runEmbeddedPiAgent` still allows uncaught `runEmbeddedAttemptWithBackend` failures to escape without setting terminal lifecycle metadata, and the embedded-run liveness model still has no distinct `cancelled` state.

## Evidence

- **No outer error boundary around embedded attempt call:** `runEmbeddedPiAgent` awaits `runEmbeddedAttemptWithBackend(...)` directly, and the surrounding structure shown on `main` is a `try`/`finally` for cleanup rather than a catch path that sets fallback terminal lifecycle metadata before rethrowing.
  - file: [src/agents/pi-embedded-runner/run.ts:830](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/pi-embedded-runner/run.ts#L830)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '820,870p'; nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '2280,2315p'`
  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Test still expects thrown attempt errors to bubble out:** Coverage on `main` explicitly asserts that a non-overflow prompt error (`transport disconnected`) causes `runEmbeddedPiAgent(...)` to reject, which matches the missing boundary concern rather than a handled terminal-meta fallback.
  - file: [src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts:456](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts#L456)

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Liveness union still has only four states:** `EmbeddedRunLivenessState` on `main` is still exactly `"working" | "paused" | "blocked" | "abandoned"`; there is no `"cancelled"` variant in the public type used by the embedded runner and subscriber plumbing.
  - file: [src/agents/pi-embedded-runner/types.ts:98](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/pi-embedded-runner/types.ts#L98)

## Risks / Open Questions

- Closing as implemented would hide a still-open lifecycle gap: uncaught runner exceptions can still bypass `setTerminalLifecycleMeta` and leave downstream observers without explicit terminal state.
- Closing as implemented would also overstate current UX semantics, because user cancellation is still conflated with `blocked` in the embedded runner contract and downstream consumers.
