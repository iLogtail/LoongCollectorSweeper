## Summary

Keep open. Current main has partial memory-capability preservation for cached registries, snapshot loads, and failed registrations, but it does not implement this PR's central OC-40 behavior: the active memory capability is still stored inside `memoryPluginState` and is cleared on every active plugin reload. The CLI scope helper also still returns only manifest-planned command plugin ids and does not append the configured memory slot plugin. The related #68041/#67063 items are adjacent open attempts, but they are not a shipped fix and do not make this PR safely obsolete.

## Best Possible Solution

Keep #69458 open for maintainer judgment. The best path is to consolidate the memory-wiki/OC-40 fixes into one clean PR: preserve the active memory capability across active plugin scopes that exclude the memory plugin, keep full-reset semantics explicit, add the scoped-load regression coverage from this PR, and decide whether CLI memory-slot inclusion belongs in the same PR or the narrower #67063/#68041 track. Close only after a canonical PR lands on main or maintainers explicitly choose a different product boundary.
