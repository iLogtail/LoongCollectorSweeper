## Summary

Keep open. Current main has a small generic warning for the all-media-dropped path, but it still does not implement this PR's dropped-media pipeline and still has silent or lossy paths for partial drops, catastrophic normalization failures, JSON/transport projection, and followup routing. Related PR #70109 is closed unmerged, so it does not supersede this open PR.

## Best Possible Solution

Keep the PR open, or replace it with a smaller maintainer-reviewed patch that preserves sanitized dropped-media diagnostics through reply normalization, routing, outbound JSON/transport projection, delivery, transcript mirroring, and followup routing. The implementation should cover partial drops, all-dropped replies, catastrophic normalization failures, media-unsupported channels, and primary-message-id preservation with focused tests.
