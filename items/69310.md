## Summary

Keep open. Current main only adds a generic warning when all reply media are dropped; it still silently loses partial media drops and still has fallback paths that strip media without carrying a structured user-visible dropped-media diagnostic. The related #70109 is open and related, but the provided PRtags context does not establish it as the canonical replacement for this active PR.

## Best Possible Solution

Keep #69310 open for maintainer review unless maintainers explicitly choose #70109 as the canonical implementation. The best path is a smaller, current-main-rebased dropped-media solution that preserves per-item diagnostics for partial drops, catastrophic normalization failures, routed/follow-up replies, JSON/API projections, and transcript mirrors, with focused tests for each delivery path.
