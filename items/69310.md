## Summary

Keep open. Current main has a narrower maintainer implementation that appends a generic warning only when every reply media item is dropped, but it does not cover this PR's remaining useful scope: partial media drops, catastrophic normalization failures, structured propagation, JSON/transcript visibility, or the more precise sanitized dropped-media notice pipeline. Related PR #70109 is closed and unmerged, so there is no canonical open/landed replacement for the remaining work.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase or refactor it against current main's simpler `⚠️ Media failed.` implementation, then decide whether to add a small targeted fix for partial drops and the catastrophic normalization catch, or keep the broader `droppedMedia` contract only if maintainers want structured JSON/transcript propagation and per-item sanitized notices.
