## Summary

Keep open. Current main still does not route macOS voice wake through the session selected in the app’s Session picker. The picker changes the embedded chat view model’s session only, while app-level voice wake forwarding still defaults to main/webchat, and the Talk Mode fallback reads `WebChatManager.activeSessionKey`, which is not updated by the picker switch.

## Best Possible Solution

Keep the issue open. The best fix is to add a macOS app-level selected session target shared by the chat Session picker, voice wake overlay forwarding, and Talk Mode, then route voice wake sends through that selected session using `chat.send` or an equivalent channel-aware gateway path. Add focused Swift tests proving that switching to a non-main session updates the voice target and that voice wake/Talk Mode sends use that session key.
