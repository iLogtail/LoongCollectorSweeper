## Summary

Keep open. Current main still has a macOS chat session picker, but the voice wake send path does not consume that selected session target. Voice wake forwards through default options that route to `sessionKey = "main"` and `channel = .webchat`, so the reported wrong-session behavior remains plausible on main.
