## Summary

Keep open. Current main still collapses Mattermost inbound system-event previews with `bodyText.replace(/\s+/g, " ").slice(0, 160)`, while the agent-visible inbound body separately keeps `bodyText`. I found no Mattermost config, type, schema, docs, or tests for an opt-in newline-preserving/faithful preview mode. The existing newline-preservation work covers mention normalization/body text, not the preview string requested here.

## Best Possible Solution

Keep the issue open. Add a Mattermost plugin setting, likely top-level plus per-account, that defaults to the current compact one-line preview but can opt into a faithful preview helper preserving `\n`, normalizing spaces/tabs within lines, and capping by max lines/chars. Document it on the public Mattermost channel page and add focused tests for default flattening plus opt-in newline-preserving system-event previews.
