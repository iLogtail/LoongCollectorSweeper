## Summary

Keep #28025 open. Current main still does not implement `channels.slack.thread.replyOnToolUse` or any equivalent tool-use-gated Slack reply routing. Slack reply placement is still determined by configured `replyToMode`, chat type, and existing thread state; tool callbacks only update progress/status after the reply delivery plan has already been built. The prior ClawSweeper comment matches current behavior on e23d17da79dc2345ca55902e632d3c7622accacc.

## Best Possible Solution

Keep the issue open until a maintainer accepts or rejects explicit tool-use-aware Slack reply routing. A likely implementation belongs in the Slack plugin/config surface: add typed schema/docs for a default-off option, carry a per-turn tool-use signal into reply planning before final or streaming delivery, and add focused Slack tests covering DMs, channels, existing thread replies, `inheritParent`, and streaming/progress behavior.
