## Summary

Keep #70629 open. Current main still hardcodes `minimal: 128` for Gemini 2.5 Pro and any model id containing `2.5-flash`; `gemini-2.5-flash-lite` therefore still reaches the integer `thinkingBudget` path and would emit `thinkingBudget: 128` for `reasoning: "minimal"`. Google's current Gemini API docs say Gemini 2.5 models use `thinkingBudget`, and list Gemini 2.5 Flash-Lite's supported range as `512` to `24576`, so at least the Flash-Lite part of the PR remains a real unfixed compatibility issue. citeturn1view0

## Best Possible Solution

Keep the PR open and land a focused Google provider fix after maintainer review. At minimum, prevent `reasoning: "minimal"` from sending `thinkingBudget: 128` for `gemini-2.5-flash-lite`; keep or adapt the PR's regression tests for `buildGoogleGenerativeAiParams` so Gemini 2.5 fixed reasoning budgets are covered. If maintainers keep the broader 512 floor for Pro/Flash too, the merge rationale should cite live API behavior or a vendor contract update beyond the currently visible docs.
