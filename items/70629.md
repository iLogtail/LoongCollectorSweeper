## Summary

Keep PR #70629 open. Current main still sends `thinkingBudget: 128` for `reasoning: "minimal"` on Gemini 2.5 Pro and all Gemini 2.5 Flash-family IDs, including `gemini-2.5-flash-lite`; current tests do not cover the PR's `minimal` budget regression cases. The PR is a focused fix vehicle, though the best maintainer path is to reconcile it with current Google docs, which document a 512 minimum specifically for Gemini 2.5 Flash-Lite while listing lower supported floors for Gemini 2.5 Pro and Flash.

## Best Possible Solution

Keep the PR open as a focused bug-fix candidate, but ask for a small maintainer/code review pass against current Google API ranges: at minimum, prevent `gemini-2.5-flash-lite` minimal requests from sending `128`, add the regression tests from the PR, and preserve lower documented floors for Gemini 2.5 Pro/Flash unless live API evidence shows those endpoints also reject them.
