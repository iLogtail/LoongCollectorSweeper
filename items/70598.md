## Summary

Keep PR #70598 open. Current main already implements the central exec-heartbeat prompt fix from #70552: exec prompts now receive exec event text, embed it in the user-relay prompt, suppress empty exec completions with HEARTBEAT_OK, and have targeted prompt/runner tests. However the PR’s latest head also addresses a still-real mixed-queue follow-up from the review discussion: preserving deferred non-exec events after an exec heartbeat. Current main still filters the prompt to exec events but consumes the full inspected queue afterward, so a cron/reminder event queued beside an exec completion can be dropped without being shown or preserved.

## Best Possible Solution

Do not auto-close this PR yet. Maintainers should either merge/adapt only the deferred-event preservation part, with the duplicate-reminder regression tests from the PR, or open a narrower follow-up that fixes current main’s full-queue consumption after exec-only heartbeat prompts. The already-shipped prompt wording/filtering portions can be treated as superseded during review, but the event preservation fix remains relevant.
