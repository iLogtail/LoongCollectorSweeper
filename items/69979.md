## Summary

Keep #69979 open. Current main still advertises Feishu `message react` when reactions are enabled, but the `react` path passes the user-supplied `emoji` string straight through to Feishu's `emoji_type` field. For the reported `emoji="👍"` case, current code would send raw Unicode rather than a Feishu enum such as `THUMBSUP`. The related PR #71970 is the right shape of fix, but it is still open and its converter/test file is not present on this checkout.

## Best Possible Solution

Keep the issue open as the canonical bug until PR #71970 or an equivalent Feishu plugin fix lands. The best fix is to normalize user-facing Unicode emoji to Feishu `emoji_type` enum strings before add/list/remove reaction API calls, add regression coverage for `message action=react emoji="👍"`, and align the Feishu reaction docs/CLI surface if maintainers intend `message react` to be supported for Feishu.
