## Summary

Keep #57247 open. Current main still only renders `runtimeInfo.model` and `defaultModel` in the compact `## Runtime` metadata line; it does not contain the dedicated high-salience model identity line proposed by the PR. The related prompt-identity reports do not provide a merged or open canonical replacement for this exact generic core behavior, and the PR also has a substantive unresolved review concern about sanitizing the model id before inserting it into system-prompt text.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either a sanitized core implementation with regression tests for ordering, absence, autoroute/default-model distinction, and malicious model ids, or an explicit maintainer decision to redirect model-family-specific identity prompting to provider prompt contributions or a ClawHub plugin. Current main does not yet implement or supersede the requested behavior.
