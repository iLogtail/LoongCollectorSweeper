## Summary

Keep open. Current main still reparses session JSONL during session-memory sync and does not reuse a cached parsed entry, so the central performance request is not implemented or superseded. The PR is outdated and unsafe as submitted, but it is concrete core performance work rather than a closeable duplicate or external/plugin request.

## Best Possible Solution

Keep the item open until it is updated or intentionally declined. The best code path is a fresh implementation against `extensions/memory-core/src/memory/manager-sync-ops.ts` and `src/memory-host-sdk/host/session-files.ts` with byte-safe append boundaries, prefix validation before reuse, bounded/cleared cache state, and regression tests for ASCII append, non-ASCII append, rewritten-prefix fallback, and cache growth.
