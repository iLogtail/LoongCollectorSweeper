## Summary

Keep open. Current main still materializes full session transcript `.jsonl` files in key gateway paths before applying response limits, and the linked chunked-read PR #64858 is closed unmerged. Existing maintenance covers `sessions.json`, archive cleanup, and optional directory budgets, but it does not yet solve the central full-transcript read / event-loop stall reported here.

## Best Possible Solution

Keep #64767 open until OpenClaw lands a bounded or streaming transcript line reader for `readSessionMessages`, `readLatestSessionUsageFromTranscript`, and adjacent call sites, with regression coverage for transcripts larger than the internal chunk/window size. Coordinate the per-transcript rotation/size-cap portion with the narrower open #66360 follow-up instead of treating current maintenance as a complete fix.
