## Summary

Keep open. Current main still resolves OpenRouter auto as `openrouter/openrouter/auto`/runtime model id `openrouter/auto` rather than a bare OpenRouter API `auto` id, and the `infer model run` JSON path still hardcodes `attempts: []`, so the reported runtime fallback/visibility problem is not fixed. The linked PR #69580 was closed unmerged, and related #62655 overlaps the catalog mismatch but does not fully supersede this runtime regression report.

## Best Possible Solution

Keep #69527 open and fix it in the OpenRouter provider/model-runtime boundary: preserve `openrouter/auto` as the user-facing config ref, send bare `auto` to the OpenRouter API where required, align the bundled catalog/normalization behavior with #62655, and add regression coverage for `infer model run` plus populated fallback attempt details.
