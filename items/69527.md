## Summary

Keep #69527 open. Current main has not shipped the related fix PR, and `openclaw infer model run --json` still always reports `attempts: []` even though the agent fallback layer records fallback attempts internally. That leaves the reporter's observable problem unresolved when the run lands on `openai/gpt-4.1-mini`: the JSON output still cannot distinguish a successful OpenRouter primary run from a silent fallback. OpenRouter `openrouter/auto` is still documented as a supported default, so this remains an actionable runtime/CLI bug rather than cleanup noise.

## Best Possible Solution

Keep the issue open and fix the runtime/CLI contract directly: add a regression test for `infer model run` with `agents.defaults.model.primary = "openrouter/auto"` plus fallbacks, ensure the configured OpenRouter primary is the first attempted candidate, and include fallback attempt metadata in the JSON result when fallback occurs. If `openrouter/@preset/...` has a distinct model-resolution bug, split or link a narrower follow-up rather than closing this report.
