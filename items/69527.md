## Summary

Keep #69527 open. Current main appears to have improved the OpenRouter dynamic-model path enough that `openrouter/auto` and preset-like OpenRouter ids should no longer be rejected as unknown before runtime, but the reported `infer model run` observability problem is still present: both local and gateway JSON responses hard-code `attempts: []`, so a fallback from OpenRouter to OpenAI would still look like a silent primary-model ignore. The linked PR #69580 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and fix it with a focused regression test around `openclaw infer model run` using `agents.defaults.model.primary = "openrouter/auto"` plus OpenAI fallback. The best implementation should prove the first attempted provider/model is OpenRouter, surface any fallback attempt details in the JSON envelope for both local and gateway transports, and clean up the `models status` display so `openrouter/auto` is not shown as `openrouter/openrouter/auto`.
