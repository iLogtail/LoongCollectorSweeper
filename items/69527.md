## Summary

Keep #69527 open. Current main still appears to preserve `openrouter/auto` as the runtime model id instead of normalizing the OpenRouter API payload to bare `auto`, and `infer model run` still returns `attempts: []` even though the embedded runner records execution trace internally. Related PR #69580 attempted this exact fix but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open. The best fix is provider-owned: preserve `openrouter/auto` as the public OpenClaw model ref, normalize the OpenRouter runtime/API model id to bare `auto` before the OpenAI-compatible payload is built, add regression coverage for `resolveModel` or the provider runtime hook plus `infer model run`, and map embedded runner executionTrace into the `infer model run --json` `attempts` field so fallback use is visible.
