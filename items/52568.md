## Summary

Keep open. Current main still lacks the requested Discord middle-ground behavior: bot-owned `autoThread` threads bypass mention gating entirely, and Discord config has no `requireMentionInThreadReplies` or equivalent option. The related #40895 context is relevant, but it is broader and still open, so it is not a strong canonical duplicate for closing this focused request.

## Best Possible Solution

Keep #52568 open and implement the Discord plugin behavior explicitly. A conservative path is to add a config-backed Discord thread-reply mention policy, likely under the guild/channel entry or `channels.discord.thread`, wire it through `src/config/types.discord.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/help, docs, and `extensions/discord/src/monitor/message-handler.preflight.ts`, then add regression tests for bot-owned `autoThread` follow-up replies with and without explicit mentions. Coordinate with broader #40895, but use this issue as the focused product shape unless maintainers choose a different canonical tracker.
