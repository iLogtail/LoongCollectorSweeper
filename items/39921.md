## Summary

Keep open. Current `main` does not contain this PR's gateway-health/runtime-state fix: `getHealthSnapshot` still probes by default and has no runtime snapshot input, the gateway `health` RPC still serves/refreshed cached snapshots without overlaying live runtime state, and Discord still only publishes `connected:true` when `gateway.isConnected` is already true.

## Evidence
