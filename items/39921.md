## Summary

Keep open. Current `main` still lacks the core behavior this PR proposes: `getHealthSnapshot()` still probes by default and does not accept/runtime-overlay gateway state, `health` RPC still serves cached snapshots plus a background refresh instead of rebuilding from `context.getRuntimeSnapshot()`, and Discord startup still only marks accounts connected when `gateway.isConnected` is already true. The PR discussion also shows the only review blocker was addressed in a follow-up commit, so this is still plausible useful work rather than stale noise.
