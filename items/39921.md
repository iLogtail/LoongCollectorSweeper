## Summary

Keep open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not implement this PR's central behavior: health snapshots have no runtime snapshot input, gateway health still serves through the cached refresh path, and Discord still gates the startup connected-status patch on `gateway.isConnected`. The PR discussion includes a concrete live repro and an addressed review blocker, so this remains plausible maintainer backlog rather than cleanup-close material.

## Best Possible Solution

Keep this PR open or consolidate it into a maintainer-chosen canonical health-runtime PR. The product fix should thread runtime channel/account state into health snapshots, make probe intent explicit, update gateway health cache/broadcast behavior without overwriting runtime fields, remove the Discord `gateway.isConnected` startup gate, and add regression coverage for both the health snapshot and Discord race cases.
