## Summary

Keep open. Current main still lacks the PR's core behavior: health snapshots have no runtime-snapshot input, gateway health still serves cached/refresh snapshots without overlaying `context.getRuntimeSnapshot()`, and Discord still gates the connected status patch on `gateway.isConnected`. The PR discussion also shows the known review blocker was addressed, so this remains plausible useful bug-fix work rather than cleanup noise.

## Best Possible Solution

Keep this PR open or manually consolidate it with the related health-runtime PRs. The product fix should make `getHealthSnapshot()` accept runtime state and opt into probing explicitly, have `gateway.health` build/cache/broadcast a fresh `context.getRuntimeSnapshot()`-backed snapshot without a stale overwrite, and make Discord publish `connected:true` after successful login regardless of the gateway plugin's instantaneous `isConnected` flag, with focused regression tests for both health snapshots and Discord startup.
