## Summary

Current `main` already supports configuring multiple providers/models and automatically failing over when the active provider hits rate limits, billing/auth failures, or similar retry-worthy errors.

## Close Comment

Closing from a Codex review because current `main` already implements the requested behavior well enough. OpenClaw supports multiple configured providers/models and will fail over when the active one is exhausted: `docs/concepts/models.md:16` documents primary -> `agents.defaults.model.fallbacks` -> provider auth failover, `docs/concepts/model-failover.md:10` and `docs/concepts/model-failover.md:215` document cross-model/provider failover for rate limits and related failures, and `src/agents/model-fallback.ts:397` plus `src/agents/model-fallback.ts:652` implement that candidate chain and retry loop. `docs/cli/models.md:28` also documents multi-provider auth/quota visibility in `openclaw models status`. If you configure Claude, Gemini, OpenAI, etc. as primary + fallbacks, OpenClaw already has the "when one burns out, try another" path on current main. If you meant proactive budget-based routing before any failure happens, that narrower behavior is different, but the feature described in this issue is already present.
