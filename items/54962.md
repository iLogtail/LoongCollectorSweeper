## Summary

Keep #54962 open. Current main still forwards generic Telegram callback_query fallback data to the agent as raw callback data, while the Telegram docs still document `callback_data: <value>`. The PR addresses a real remaining mismatch linked to #54909, but its one-line patch needs to be updated for current main because native command callbacks now use a `tgcmd:` path and must keep `commandSource: native`.

## Best Possible Solution

Keep the PR or its linked bug open, then update the fix for current main as `text: nativeCallbackCommand ?? `callback_data: ${data}`` in the generic fallback path, preserving exec approvals, plugin callbacks, pagination, model callbacks, and `tgcmd:` native commands. Add a regression assertion that generic callback payloads reach the agent as `callback_data: cmd:option_a` and keep the existing native-command test proving `CommandSource: native`.
