## Summary

Keep #54507 open. Current main still builds Feishu card note footers from agent/model/provider context and passes them into streaming and static card replies without a config gate. Feishu does have render-mode controls, and auto mode keeps plain text on the non-card path, but there is no option to hide/customize the runtime footer while retaining card/streaming rendering.

## Best Possible Solution

Keep the issue open and implement this in the Feishu channel/plugin presentation config, likely as a top-level and per-account option such as `channels.feishu.showRuntimeFooter` or a small `cardFooter` mode. The change should gate `resolveCardNote` for streaming and static structured cards, update the Feishu config schema/manifest/generated metadata, document it on the public Feishu docs page, and add tests that preserve card rendering while omitting the note.
