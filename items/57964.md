## Summary

Keep open. Current `main` still contains a live chat/TUI text-merge bug that can drop repeated trailing characters, matching the report that repeated symbols collapse in the dashboard or TUI until the view reloads from saved history.

## Evidence

- **Gateway chat merge drops repeated trailing deltas:** In `src/gateway/server-chat.ts:90`, `appendUniqueSuffix()` returns `base` unchanged whenever `base.endsWith(suffix)`. Then `resolveMergedAssistantText()` at `src/gateway/server-chat.ts:109` prefers `nextText` when it starts with `previousText`, so a stale full-text snapshot plus a repeated trailing delta is silently collapsed instead of appended.
  - file: [src/gateway/server-chat.ts:90](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-chat.ts#L90)
  - command: `nl -ba src/gateway/server-chat.ts | sed -n '90,130p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current logic reproduces the reported collapse:** Read-only `node -e` reproduction on the checked-out logic produced `{"prev":"!!!","next":"!!!","delta":"!","result":"!!!"}` and `{"prev":"111","next":"111","delta":"1","result":"111"}`. That is the exact failure mode from the issue: repeated end-of-token characters are lost.
  - command: `node -e 'function appendUniqueSuffix(...) ...; function resolveMergedAssistantText(...) ...; console.log(...)'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **The same faulty merge exists in the embedded TUI path:** `src/tui/embedded-backend.ts:86` contains the same `appendUniqueSuffix()` / `resolveMergedAssistantText()` logic, so local embedded TUI has the same repeated-suffix collapse behavior.
  - file: [src/tui/embedded-backend.ts:86](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/tui/embedded-backend.ts#L86)
  - command: `nl -ba src/tui/embedded-backend.ts | sed -n '86,123p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Live dashboard display uses streamed payloads, while reload uses persisted history:** `ui/src/ui/controllers/chat.ts:451` applies live `delta`/`final` payload text directly to `chatStream` / `chatMessages`, while `ui/src/ui/controllers/chat.ts:180` reloads full saved history and clears streaming state. That matches the reporter's symptom: collapsed characters during live display, then correct text after browser refresh or reconnect.
  - file: [ui/src/ui/controllers/chat.ts:451](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/controllers/chat.ts#L451)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '176,186p' && nl -ba ui/src/ui/controllers/chat.ts | sed -n '451,470p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
