## Summary

Keep PR #53738 open. Current main has implemented the image-side config-aware model resolution, but the PDF tool still uses the registry-only helper that ignores the live `cfg` object, so the PR's PDF/baseUrl problem is not fully solved on main. The related #65746 is useful context, but it is open, narrower, and not a shipped replacement for the full media-tools behavior requested here.

## Best Possible Solution

Keep the item open and either rebase #53738 or replace it with a maintainer-approved PDF-focused fix that preserves the already-shipped image behavior. The remaining code change should make PDF model resolution config-aware for provider `baseUrl`, `api`, `headers`, `compat`, and config-defined custom models, then add targeted PDF regression tests for both registry-backed provider overrides and configured custom providers. Coordinate with #65746 rather than closing this broader PR as already implemented.
