## Summary

PR #41758 should close as superseded by the still-open canonical issue #41750. Current main still does not implement automatic recovery for an unresponsive Chrome process holding the managed CDP port, but this PR is stale, targets the old browser source path, and has unresolved blocking review findings that make it unsafe to merge as the fix.

## Best Possible Solution

Close this stale, unsafe PR and keep #41750 as the canonical tracker. A future fix should target `extensions/browser/src/browser/chrome.ts`, perform safe process enumeration without shell interpolation, verify OpenClaw ownership before killing anything on a port, handle Windows/macOS/Linux explicitly, and add regression tests for an unresponsive process holding the managed CDP port.
