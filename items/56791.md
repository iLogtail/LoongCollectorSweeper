## Summary

Keep open. Current main only forwards Signal quote text/sender/isQuote and still does not forward `ReplyToId` or model signal-cli's `quote.id`/`authorNumber`; #56791 remains a focused channel bug-fix PR rather than obsolete cleanup.

## Best Possible Solution

Keep #56791 open as the canonical active fix for Signal quote metadata. The best path is to land a reviewed version that adds `ReplyToId`, models `quote.id`/`authorNumber`, routes UUID vs phone authors correctly, clears reply metadata on merged debounce flushes, and adds focused Signal inbound-context tests for those cases.
