## Summary

Keep open. Current `main` still does not implement `{context}` or `{contextPercent}` for response-prefix templates, and the PR's follow-up wiring shown in the review context still has an unresolved correctness risk: it prefers aggregate `lastCallUsage.total`, while OpenClaw's existing context-snapshot contract explicitly uses prompt-side tokens only.

## Evidence

There is no `{context}` or `{contextPercent}` handling on `main`.
