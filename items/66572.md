## Summary

Current main already solves PR #66572's root problem. The shipped-on-main resolver no longer lets a partial active speech-provider registry hide cfg-requested providers: it detects requested speech provider ids/aliases from cfg, removes providers already present in the active registry, resolves the cfg-compatible plugin registry with `activate: false`, and merges only the missing requested loaded providers back into the active list. The PR discussion and Greptile comment described the same partial-active-registry Microsoft/Edge TTS failure mode, and current tests now cover the active OpenAI plus cfg-requested Microsoft/Edge case and the unrelated-provider filtering behavior.

## Best Possible Solution

Close this PR as implemented on current main and keep the mainline resolver/tests. The current implementation is broader than the PR patch because it merges active providers with only the cfg-requested missing speech providers rather than replacing the active list wholesale.
