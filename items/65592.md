## Summary

Keep open. Current main documents the SearXNG JSON requirement, but it does not implement the PR's onboarding behavior: the SearXNG provider exposes no setup note, the web-search provider contract has no credential-note field, and the setup flow proceeds from provider selection to credential handling without any SearXNG-specific note before the base URL prompt. The provided PR context is still useful because it contains a focused contract/flow/provider change and reflects prior bot-review feedback about quickstart and secret-ref timing.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to finish it as a small additive plugin-contract change: add a provider-owned pre-credential note surface, show it before SearXNG credential collection including secret-ref setup, skip it in quickstart fast paths, and add focused flow/provider tests. The existing docs tip should remain as supporting documentation, not as a replacement for the onboarding hint.
