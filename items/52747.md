## Summary

Keep PR #52747 open. Current main still lacks the ACP session-lane task timeout, config surface, and hung-initialize regression coverage proposed by the PR. Existing ACP turn timeout coverage is adjacent but does not cover a stuck actor task such as `session/load`/`ensureSession` holding the lane forever. The prior #52628 is the earlier closed copy of this work, not a canonical replacement, and #33884 is unrelated sub-agent notification work.

## Best Possible Solution

Keep the PR open and revise it into a focused ACP fix. The safest path is to add a session-lane timeout only with a cancellation/cleanup contract that preserves per-session serialization: propagate an `AbortSignal` into ACP runtime operations, abort on timeout, and release the lane only after the operation has stopped or after stale side effects are fenced off. Then wire the `acp.sessionLane.taskTimeoutMs` config/schema/help/generated surfaces and add regression coverage for a hung initialize/session-load task followed by queued work.
