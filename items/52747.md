## Summary

Keep this PR open. Current main still lacks the ACP session-lane task timeout, the `acp.sessionLane.taskTimeoutMs` config surface, and the hung-initialize regression coverage proposed here. The prior #52628 is the earlier closed copy of the same work, not a shipped replacement, and #33884 is unrelated sub-agent notification work.

## Best Possible Solution

Keep the PR open and review it as a focused ACP control-plane bug fix. The safest implementation is a session-lane timeout with a cancellation or fencing contract: propagate an `AbortSignal` into ACP runtime operations where possible, prevent stale post-timeout side effects from corrupting later same-session work, then wire `acp.sessionLane.taskTimeoutMs` through types, Zod, generated schema/help, and regression coverage for a hung initialize/session-load followed by queued work.
