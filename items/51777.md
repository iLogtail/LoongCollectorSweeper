## Summary

Keep open. Current `main` still has the underlying Feishu persistence gap: onboarding skips `setPolicy` when the selected policy matches `getCurrent`, and Feishu's `getCurrent` falls back to `"pairing"` when the config key is absent. The PR discussion also surfaced a real Twitch side effect that makes the broader unconditional-write alternative non-trivial, so this is not a safe auto-close candidate.
