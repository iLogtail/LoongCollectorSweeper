## Summary

Keep open. Issue #51395 asks for a browser upload fallback for custom non-standard upload widgets such as ByteUploader. Current main still routes the browser upload tool through Playwright file chooser/input-file mechanisms or Chrome MCP `upload_file`, and the docs still describe upload as file-chooser/ref/input-ref based with existing-session limits. I found no current DataTransfer, clipboard/paste, or native drag-event upload fallback in the browser upload implementation. The related #38844 context is nearby but narrower: it covers upload/file-chooser flakiness, not custom uploader fallback behavior.

## Evidence
