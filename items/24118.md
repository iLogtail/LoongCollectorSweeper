## Summary

Keep #24118 open. Current main still does not implement `channels.signal.linkPreview` or an equivalent Signal JSON-RPC link-preview pass-through. The Signal config schema/types are strict and omit the key, the outbound send path only sends message/style/account/attachment/target fields, Signal docs do not document the setting, and the related #29466 is a closed webchat UI preview request rather than a Signal channel fix. The latest issue discussion also already records a same-day Codex review keeping this open.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` config surface, updates Signal schema/types/generated config metadata/docs, and threads the resolved option plus any supported preview metadata through the Signal JSON-RPC send/updateConfiguration path with focused tests. If maintainers decide Signal preview generation is out of scope, document that product decision explicitly instead of auto-closing this concrete channel enhancement.
