## Summary

Keep #24118 open. Current main still does not expose `channels.signal.linkPreview`, the Signal config schema/types/generated metadata remain strict without that key, and the Signal outbound JSON-RPC send path still sends only message/style/account/attachment/target params. The existing maintainer/Codex issue comment also explicitly kept this open, and the related #29466 is a closed Webchat preview-card request, not a Signal channel implementation.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` surface, updates Signal schema/types/generated metadata/docs, resolves account inheritance/defaults, threads supported preview configuration or metadata through the Signal JSON-RPC send path, and adds focused tests. If maintainers decide Signal preview generation is out of scope, record that product decision explicitly rather than auto-closing this concrete channel enhancement.
