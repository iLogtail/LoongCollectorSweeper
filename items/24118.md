## Summary

Keep #24118 open. Current main still does not implement `channels.signal.linkPreview` or an equivalent Signal JSON-RPC link-preview pass-through; the same-day maintainer comment already reached the same conclusion, and the only linked related issue is a closed Webchat UI preview request rather than a Signal channel fix.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` config surface, updates strict schema/types/generated config metadata/docs, and threads the resolved option plus any supported preview metadata through Signal JSON-RPC send/configuration paths with focused tests. If maintainers decide Signal link-preview generation is out of scope, document that product decision explicitly rather than auto-closing this concrete channel enhancement.
