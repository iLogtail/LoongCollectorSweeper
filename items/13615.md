## Summary

Keep #13615 open. Current main still has only adjacent, reactive protections: provider 429/failover classification, auth-profile cooldown/fallback, channel retry helpers, inbound gateway/control-plane limiters, and provider usage snapshots. It does not implement the requested configurable outbound API rate limiting/throttling layer, token budgets, queue/backpressure controls, warning/metrics surface, or `openclaw limits` CLI. The related implementation PR #13686 was closed unmerged, so it is useful prior art but not shipped.

## Best Possible Solution

Keep #13615 open as the canonical core product/design item for outbound API rate limiting. The best path is an opt-in policy layer at generic provider, channel, and tool request boundaries with config/schema/docs for per-provider/per-model RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat closed PR #13686 as prior art, not as shipped implementation.
