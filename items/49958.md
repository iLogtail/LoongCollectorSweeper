## Summary

Current `main` does not implement the requested behavior, and the proposed helper-level change is too broad to conservatively auto-close this PR. The shared gate still denies `useAccessGroups=true` when no authorizer is configured, current tests lock that behavior in, and adjacent DM/SDK contracts show why a blanket `!anyConfigured => allow` change would need a narrower design before merge.

## Evidence
