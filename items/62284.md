## Summary

Keep #62284 open. Current main does not implement time-bounded exec approvals: the approval decision model and UI/chat/protocol surfaces only support allow-once, allow-always, and deny; the existing expiresAtMs is a pending-request timeout, not reusable command trust; and durable trust remains permanent allow-always allowlist state. The related #39126 PR context says trust windows are still open/unshipped, so this issue is not obsolete.

## Best Possible Solution

Keep the issue open and treat it as core exec approval work. The best path is to design one bounded trust-window model, likely coordinated with #39126, then update the decision/protocol schema, in-memory or session-scoped approval store, enforcement checks before prompting, /approve parsing, Control UI buttons, approvals CLI visibility, docs, and regression tests across gateway/node/chat paths.
