## Summary

Keep #42472 open. Current main still does not suppress the whole reply when substantive text is followed by NO_REPLY; central helpers only classify exact/JSON silent payloads, normalizeReplyPayload strips a trailing token and keeps the preceding text, and cron/subagent/directive paths still have exact-token checks. The linked PRs #42873 and #46626 are useful implementation attempts, but they are open and not present on main.

## Best Possible Solution

Keep the issue open and use #42873/#46626 as implementation references. The best fix is a shared line-aware silent-token helper applied before trailing-token stripping in normalizeReplyPayload, parseReplyDirectives/outbound delivery, cron/subagent follow-up, and gateway chat/history projection, with regression tests that suppress `text\nNO_REPLY` while preserving inline/substantive NO_REPLY cases.
