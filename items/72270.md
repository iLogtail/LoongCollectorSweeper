## Summary

Current main already fixes the reported ENOTEMPTY cascade by making bundled plugin runtime-dependency repair use an OpenClaw-owned staged install root with project-scoped npm config, rather than letting inherited npm global/prefix settings redirect installs into ~/node_modules. Tests and Docker smoke coverage now assert this behavior for Telegram and other bundled plugin runtime deps. The fix is on main, not in the provided latest stable v2026.4.24 release.

## Best Possible Solution

Close this issue as implemented on main. Keep the staged bundled-runtime-deps implementation and its unit/Docker coverage, and ship it in the next stable release rather than tracking a separate core bug for the same ENOTEMPTY startup failure.
