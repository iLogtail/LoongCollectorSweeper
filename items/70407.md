## Summary

Keep #70407 open. Current main still treats `plugins.entries.memory-core.config.dreaming.storage` as an object-only plugin config field, validates plugin config before normal runtime resolution, and does not show a memory-core doctor/plugin compatibility migration for the legacy string values (`"inline"`, `"separate"`, `"both"`). The related ClawSweeper item #68468 covers the default storage mode flip, not this upgrade-time schema migration break.

## Best Possible Solution

Add a narrow compatibility migration before plugin schema validation, or a memory-core doctor contract normalizer, that rewrites `plugins.entries.<active-memory-plugin>.config.dreaming.storage: "inline" | "separate" | "both"` to `{ mode: <value>, separateReports: false }`. Add regression coverage for `readConfigFileSnapshot`/`loadConfig` and `openclaw doctor --fix` with the exact legacy string shape from the report, plus a short upgrade hint in the invalid-config path if migration is intentionally not automatic.
