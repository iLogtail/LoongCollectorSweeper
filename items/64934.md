## Summary

Keep #64934 open. Current main still has only coarse task statuses (`queued`, `running`, and terminal states), and task registry event projection does not handle approval-pending or waiting-for-input runtime events as first-class active task states. The PR is not obsolete, but it needs maintainer-quality rework because review comments correctly identify additional active-state consumers that would also need updating before this can safely land.

## Best Possible Solution

Keep the PR open, but require a refreshed core task-state implementation: add the intermediate statuses to the type and normalizer, centralize active-status logic, update every active/terminal consumer, wire approval and waiting-for-output entry and exit transitions, clear stale progress summaries when states resolve, and cover registry, flow, maintenance, audit, status-snapshot, and executor cancellation behavior with focused tests.
