## Summary

Keep open. Current main still does not verify or repair the global npm bin link after `openclaw update`; it only cleans package-level `.openclaw-*` rename directories and verifies the installed package root. The linked bug #63719 remains a concrete, open report, and this PR is a plausible implementation candidate, but it needs maintainer review and cleanup before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #63719. The best path is to rebase onto current main, remove or split the unrelated context-overflow harness edit, tighten the bin repair to match npm semantics safely, add focused tests through the shared package update path, and then review/land it if the implementation is sound.
