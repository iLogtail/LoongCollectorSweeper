## Summary

Keep #72376 open. Current main still opens the memory-core SQLite database with only `busy_timeout`; it does not set `journal_mode=WAL` or `synchronous=NORMAL`. The PR is a fresh, clean, narrow change against the current `extensions/memory-core` path, while older related WAL PRs are closed or stale/dirty and are not better canonical vehicles.

## Best Possible Solution

Keep #72376 open as the current canonical narrow PR for memory-core WAL defaults. The best path is to amend it with a WAL pragma-result check or an explicit rebuildable-index tradeoff comment, add focused memory-core WAL tests covering fresh open and safe reindex, then merge if maintainers accept the default. Older stale WAL PRs such as #51986/#36535 can be closed separately rather than using them to close this cleaner current-path PR.
