## Summary

Keep open. Current `main` still passes Discord `ThreadStarterBody` unconditionally on thread messages, while the analogous Slack path explicitly suppresses it for existing thread sessions. Core reply assembly still consumes `ctx.ThreadStarterBody` on follow-up turns when it is present, so there is no strong evidence that the reported Discord behavior is already fixed or obsolete on `main`.
