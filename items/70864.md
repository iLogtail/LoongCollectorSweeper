## Summary

Keep open. Current main at 8741a86f does not implement scoped mention-pattern policy: config types/schema only expose the existing mentionPatterns array, the SDK/runtime mention surface lacks policy helpers, bundled channel handlers still build mention regexes directly, and docs only describe existing regex fallback behavior. PR #70864 remains the active clean successor to closed dirty predecessor #70846, not an obsolete duplicate.

## Best Possible Solution

Keep #70864 open as the active implementation path. The maintainer path is to review the latest PR head, ensure the config schema/types/help/generated metadata, SDK/runtime helpers, and every bundled channel agree on routed-agent and conversation-scoped policy semantics, then merge if CI and changed-surface validation are green.
