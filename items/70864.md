## Summary

Keep open. Current main does not implement scoped mention-pattern policy: it only has the existing regex mention-pattern fallback, with no global/agent/provider allow/deny mode, no SDK/runtime policy helper, and bundled channel handlers still build mention regexes directly. The PR remains a coherent core plus bundled-plugin proposal, and the earlier #70846 is the dirty predecessor that this PR supersedes, not a canonical replacement for closing this one.

## Best Possible Solution

Keep #70864 open as the active implementation path. The best maintainer path is to review the latest head, ensure the scoped config schema, SDK/runtime helper, docs, and every bundled channel use the same routed-agent/conversation policy semantics, then merge if CI and changed-surface tests are green. Do not close as implemented, duplicate, ClawHub, or stale: current main lacks the capability and #70864 is the clean successor to #70846.
