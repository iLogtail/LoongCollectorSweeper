## Summary

Keep open. Current main has adjacent chat-composer affordances, including starter prompts, slash-command completion, input history, browser dictation, and Talk mode, but it does not implement configurable custom-term autocomplete/suggestions for the WebChat/Control UI chat input. The remaining work appears to be a real core UI/config feature request, not an already-shipped capability, duplicate, stale report, or clean ClawHub/plugin close.

## Best Possible Solution

Keep the issue open as a focused Control UI/WebChat feature request. A good implementation would add a small, explicit custom-term source, wire a suggestion menu into `ui/src/ui/views/chat.ts` beside the existing slash menu and STT draft updates, validate/persist the source through config or a documented workspace/browser-local mechanism, and update WebChat/Control UI docs. If maintainers want this outside core, the next step should be a deliberate plugin API/design issue for composer UI augmentation rather than closing this request now.
