## Summary

Keep #7909 open. Current main still exposes only the assistant-message Markdown copy path in the Control UI: the helper label is "Copy as markdown", it writes the supplied markdown string directly to the clipboard, and assistant bubble actions render only that button. I found no plain-text copy action, dropdown, copy-format preference, or focused test coverage. The provided issue discussion also includes a maintainer-side Codex review explicitly keeping the issue open, and the related Android copy/selection issue and PR do not supersede this Control UI request.

## Best Possible Solution

Keep #7909 open until the Control UI adds a real assistant-message copy format choice. The likely implementation belongs near ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: expose Markdown vs plain-text copy through a compact action menu or preference, reuse or add a tested Markdown-to-plain-text helper, and add focused UI/clipboard tests proving both formats write the expected text.
