## Summary

Keep #72219 open. Current main still unconditionally serializes `delivery.resolved`, and the `delivery.mode: "none"` path still manufactures a failed resolved target with `error: "delivery is disabled"`. The PR is a focused, recent bug fix for linked issue #72210 and includes a targeted regression test; bot/QC comments in the provided discussion also describe the same small guard-and-test change positively.

## Best Possible Solution

Keep the PR open for maintainer review and land #72219 or an equivalent targeted patch that omits the misleading `delivery.resolved.error` for intentionally disabled cron fallback delivery while preserving current message-tool target behavior and adding the regression test.
