## Summary

Keep PR #72219 open. Current main at edf40ab6 still has the reported behavior: bare `delivery.mode: "none"` cron runs produce a failure-shaped `resolvedDelivery`, and `buildCronDeliveryTrace` unconditionally writes that into `delivery.resolved`. The PR is a focused fix for linked open issue #72210 and adds the missing regression assertion.

## Best Possible Solution

Keep #72219 open for maintainer review and land this PR or an equivalent targeted patch. The fix should omit `delivery.resolved` only when fallback delivery is intentionally disabled by `delivery.mode: "none"`, while preserving explicit target context for message-tool delivery and keeping the new regression test.
