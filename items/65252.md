## Summary

Keep open. Current main has bundle-provided LSP server configuration and agent-facing hover/definition/references tools, but it does not implement the requested real-time LSP diagnostic feed, server health dashboard, in-editor visualization, smart troubleshooting, or code-quality trend UI. The plugin surface can host tools/services/routes, but I did not find an existing core/editor API that would let a ClawHub plugin satisfy the direct in-editor diagnostics request without new product/API work.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete path would add LSP diagnostic collection in the embedded LSP runtime, define server health/log/performance state, expose it through Gateway/ACP/UI contracts, render an editor or diagnostics dashboard surface, and cover the behavior with focused runtime and UI tests. If maintainers want this externalized, the first core task should be a documented plugin/editor diagnostics API rather than a one-off bundled implementation.
