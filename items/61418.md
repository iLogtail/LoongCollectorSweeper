## Summary

Keep open. Current main still filters the TUI model picker by configured model allowlist only, not by provider auth status. If no `agents.defaults.models` allowlist exists, both the gateway-backed and embedded TUI paths can fall back to the full model catalog, so the reported auth-only setup can still show unauthenticated providers/models. Related #40412 covered search/configured-model filtering, but it does not supersede this auth-status filtering request.

## Best Possible Solution

Keep #61418 open and implement a shared auth-aware model-listing path for the TUI `/model` picker and `/models` provider/model browser. Reuse the existing CLI/provider auth availability semantics where possible, preserve explicit allowlist behavior, and add regression coverage for an auth-only setup with no `agents.defaults.models` allowlist so unauthenticated providers are hidden instead of falling back to the full catalog.
