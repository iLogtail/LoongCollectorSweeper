## Summary

Keep open. Current main does not implement PR #56785's actionable missing-python guidance in the sandbox mutation error path, and the PR is not fully superseded by current main. The Docker/source-image side has partial mitigation because Dockerfile.sandbox now installs python3, and the broader default-image root cause is tracked by #51185 / PR #57789, but main still has no friendly error translation in fs-bridge.ts or remote-fs-bridge.ts. The PR is stale and overlapping, not obsolete.

## Best Possible Solution

Keep the PR open but require a refresh against current main. The best path is to coordinate with #51185/#57789 for the Docker default-image root cause, then implement one current, tested error-mapping path for sandbox mutation failures that uses the current pinned-helper marker, preserves the original error as `cause`, and adds coverage for both Docker/local bridge behavior and any intended remote-shell guidance.
