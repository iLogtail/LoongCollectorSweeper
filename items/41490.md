## Risks / Open Questions

- Main's implementation is broader than the PR: it flattens any text-only completion message arrays for `requiresStringContent` models, not only assistant messages with `tool_calls`.
- The shared helper joins multiple text segments with newlines, whereas the PR description implied preserving assistant tool history as plain strings without focusing on separator semantics.
