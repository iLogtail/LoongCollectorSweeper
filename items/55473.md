## Summary

Keep open. Current main still treats startup SecretRef activation as all-or-nothing for active channel refs, so the central bug is not implemented on main. The PR is a focused real bug fix attempt, but the discussion and later review comments show it still needs rework before merge.

## Best Possible Solution

Keep the item open and ask for a reworked, current-main PR. The fix likely belongs in the startup secrets path now centered on `src/gateway/server-startup-config.ts`: isolate only true channel-local missing-secret availability failures, keep non-channel and config/policy errors fatal, preserve runtime/source snapshot invariants, ensure `secrets.reload` can recover and restart affected channels, and add regression coverage around startup degradation plus recovery writes/restarts.
