## Summary

Keep open. Current main still puts the same native `sessions_spawn` task string into both the subagent system prompt and the initial child user message. The issue is fresh, has concrete provider-log evidence, and has active linked fix PRs (#72026, #72028, #72053), but none is merged into the inspected main SHA.

## Best Possible Solution

Keep #72019 open until a focused core fix lands. The best path is to merge or adapt the #72053-style approach: make the full subagent task appear exactly once, preferably in the system `## Your Role` section with multiline content preserved, make the initial user turn a short bootstrap pointer, and add regression tests that inspect both `extraSystemPrompt` and the gateway `message` for a native `sessions_spawn` call.
