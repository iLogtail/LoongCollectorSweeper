## Summary

Keep open. Current main still duplicates the native `sessions_spawn` task text: the same `task` is interpolated into the subagent system prompt and into the first child user message passed to the gateway. The linked ClawHub skill is a useful workaround and the related PRs are plausible fixes, but no fix is present on current main.

## Best Possible Solution

Keep #72019 open until a single canonical PR lands. The best fix is to single-source the subagent task text for native `runtime: "subagent"`, preserve multiline task formatting, and add a focused regression test that inspects both `extraSystemPrompt` and the first gateway `message` so long task prompts are not duplicated or lost.
