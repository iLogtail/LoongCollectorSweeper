## Summary

Keep PR #44423 open. Current main has Telegram polling-stall recovery and documents the retry/network knobs, but it does not include the PR's proposed restart/update recovery troubleshooting row or host-level watchdog/systemd guidance. The PR is focused docs work, and the original systemd `~` review concern appears addressed in the PR diff, but the remaining `openclaw health --json` leakage concern is substantive enough that this should be revised rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and ask for a small revision: preserve the Telegram restart/update troubleshooting row and systemd path guidance, but add an explicit sanitized watchdog pattern that extracts only needed booleans from `openclaw health --json` and warns not to post or persist the full JSON payload. After that, maintainers can review it as a focused Telegram docs improvement.
