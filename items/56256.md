## Summary

Keep PR #56256 open. Current main does not implement the requested ClawHub skill lockfile hashes or `skills.integrity.*` audit findings. The repo has related plugin archive integrity checks and generic plugin security-audit hooks, but those do not cover post-install tamper detection for ClawHub-installed workspace skills. Because this is security/core hardening, it is not a conservative ClawHub cleanup close.

## Best Possible Solution

Keep the PR open for maintainer/security review. The strongest path is to either rework it into a core ClawHub skill integrity feature with tests/docs for lockfile hashes, audit findings, unreadable/tampered directories, hidden-file policy, and update compatibility, or explicitly split the scanner/policy portions into a ClawHub/Clawguard plugin while tracking any missing core extension API separately.
