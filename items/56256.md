## Summary

Keep PR #56256 open. Current main still has only ClawHub skill lockfile version/install metadata plus a reusable SHA-256 fingerprint helper; it does not record install-time skill hashes, expose verifySkillIntegrity/verifyAllSkillIntegrity, or emit skills.integrity.* security audit findings. The PR is security hardening for the core skill/ClawHub audit surface, links the ClawHavoc supply-chain report and Clawguard discussion, and still has review-raised correctness issues to resolve rather than an obvious auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to either rework #56256 into a correct core ClawHub skill integrity design with tests/docs/audit catalog coverage, or explicitly split the work into a smaller core extension point plus an external ClawHub/Clawguard plugin if maintainers decide integrity policy should live outside core.
