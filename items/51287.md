## Summary

Keep open. Current main mitigates the original two-minute pain by using a fixed 30-minute exec approval window, and that behavior shipped by the 2026.4.5 changelog entry. However, the requested user-configurable approval timeout is still not implemented in the public config surface or runtime plumbing, and the related configurable-timeout work remains open/unmerged rather than shipped.

## Best Possible Solution

Keep #51287 open as the canonical feature request until a maintainer-approved configurable approval window ships. A complete fix should choose the public config key, update exported types, strict Zod schema, labels/help/generated schema and docs, then thread the value through exec config resolution, `ExecToolDefaults`, gateway/node approval registration, agent request/wait timeouts, and CLI transport timeout floors with focused regression tests.
