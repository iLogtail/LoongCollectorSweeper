## Summary

Keep open. Current `main` still drops allow-always persistence for bare basename commands when the segment lacks `resolvedPath`, so the PR is not implemented on `main` and the report remains valid.

## Evidence

- **Allow-always persistence still uses candidate-path resolution without PATH lookup:** `collectAllowAlwaysPatterns` persists from `resolveExecutionTargetCandidatePath(segment.resolution, params.cwd)`. It does not pass `env`, and it depends on the resolution helper returning a path for unresolved bare basenames.
  - file: [src/infra/exec-approvals-allowlist.ts:961](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/infra/exec-approvals-allowlist.ts#L961)
  - command: `nl -ba src/infra/exec-approvals-allowlist.ts | sed -n '935,1015p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Current helper still fails closed for unresolved bare basenames:** `resolveExecutableCandidatePathFromResolution` falls back to `resolveExecutablePathCandidate(raw, { cwd, requirePathSeparator: true })`. With `requirePathSeparator: true`, bare basenames like `id` return `undefined` instead of resolving through PATH.
  - file: [src/infra/exec-command-resolution.ts:167](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/infra/exec-command-resolution.ts#L167)

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **PATH helper exists separately but is not used here:** `resolveExecutablePath` would resolve a bare basename via PATH, but `resolveExecutablePathCandidate` explicitly returns `undefined` for bare basenames when `requirePathSeparator` is set.
  - file: [src/infra/executable-path.ts:10](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/infra/executable-path.ts#L10)

That matches the reported bug: PATH can resolve the command, but allow-always persistence drops it when `resolvedPath` is absent.
