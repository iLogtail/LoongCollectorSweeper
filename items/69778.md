## Summary

Keep open. Current main still has the core behavior reported here: restart-aborted subagent sessions with `abortedLastRun: true` remain recoverable regardless of age, and the recovery path sends the synthetic gateway-reload resume prompt. The existing stale-run window mitigates non-recoverable stale runs, but explicitly exempts restart-aborted runs; a current regression test even asserts a 3-hour-old aborted restored run is kept/recovered. The related #66909 is only a closed docs clarification and does not supersede this concrete age-bound bug.

## Best Possible Solution

Keep the issue open and implement a core age bound for restart recovery. The likely fix should add a documented config/default for task or subagent recovery age, make subagent orphan recovery and task reconciliation mark stale restart-aborted work `lost` or otherwise finalize it instead of sending the synthetic resume prompt, log/notice skipped stale recoveries, and update schema, docs, and regression tests around old `abortedLastRun` sessions.
