## Summary

Keep open. Current main still has the reported stale restart-recovery behavior: there is no taskRecovery/maxAgeMinutes config, subagent orphan recovery resumes any active run whose child session has abortedLastRun=true, and tests/docs explicitly preserve 3-hour stale aborted restored runs for restart recovery. Related #66909 is a closed docs clarification and does not supersede this concrete age-bound bug.

## Best Possible Solution

Keep #69778 open and implement a core recovery bound. The likely path is to add a documented agents.defaults task/subagent recovery max-age setting, apply it before synthetic subagent resume and task re-dispatch, mark stale recovered candidates lost/skipped with a clear startup log, and replace the current 3-hour aborted-run recovery test with coverage for both fresh recoverable and stale skipped cases. Preserve the existing `openclaw tasks list --status lost` behavior, which is already documented.
