## Summary

Current `main` contains a post-issue fix for provider-prefixed/nested model IDs such as NVIDIA model refs, and the exact regression path is now covered by targeted tests.

## Evidence

- **Provider inference for configured nested model IDs:** `inferUniqueProviderFromConfiguredModels` now scans configured allowlist/catalog entries and `resolveAllowedModelRefFromAliasIndex` uses that inference before validating a bare or nested model ID, which addresses `model not allowed` failures for refs like `moonshotai/kimi-k2.5` under `nvidia`.
  - file: [src/agents/model-selection-shared.ts:52](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/model-selection-shared.ts#L52)

- **Session/runtime model identity now preserves inferred provider:** `resolveSessionModelIdentityRef` now infers the provider from configured models for runtime and fallback refs before reparsing, so persisted session rows with nested IDs no longer lose the `nvidia` provider when switching models.
  - file: [src/gateway/session-utils.ts:1186](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/gateway/session-utils.ts#L1186)

  - file: [src/gateway/server.sessions.gateway-server-sessions-a.test.ts:1652](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/gateway/server.sessions.gateway-server-sessions-a.test.ts#L1652)
