## Summary

Keep #39690 open. Current main still lets implicit/catalog model metadata overwrite an explicit `models.providers.*.models[].input` value, so a user-configured `input: ["text", "image"]` can still be lost before Gateway decides whether to preserve inline image blocks. The newer text-only offload path helps a related crash/drop case, but it does not implement the requested explicit-input precedence.

## Best Possible Solution

Keep the issue open and fix `mergeProviderModels` so an explicitly present `models.providers.*.models[].input` wins over implicit/catalog metadata, matching the existing `reasoning` override pattern. Add regression coverage where an explicit image-capable model survives a text-only implicit match, then verify Gateway inline image handling still sends native image blocks for that configured model while preserving the text-only offload fallback.
