## Summary

Current main still does not implement the central requested behavior: when an explicit provider model matches an implicit/catalog model, `mergeProviderModels` continues to take `input` from the implicit model instead of preserving explicit `input: ["text", "image"]`. The related text-only image offload path helps a different failure mode, but it does not make custom or overridden vision-capable model metadata survive provider merge and Gateway image-capability checks.

## Best Possible Solution

Keep #39690 open until `mergeProviderModels` preserves explicit `models.providers.*.models[].input` when present, matching the existing explicit `reasoning` override behavior. #39785 is the direct candidate fix; maintainers should update the merge test to expect explicit image input preservation and verify Gateway/native inline image handling for a custom Ollama or OpenAI-compatible vision model.
