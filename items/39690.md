## Summary

Current main still does not implement the requested behavior. Matching explicit and implicit provider models still take `input` from the implicit/catalog model, so an explicit user-configured `input: ["text", "image"]` can be overwritten before Gateway image-capability checks run. The related #67634 text-only offload path helps a different failure mode, but it does not make custom vision-capable models preserve native image input.

## Best Possible Solution

Keep #39690 open until current main changes `mergeProviderModels` so explicit `models.providers.*.models[].input` wins when provided, matching the existing `reasoning` override pattern. #39785 is the direct candidate fix; maintainers should update the merge test to expect explicit image input preservation and verify Gateway/native inline image handling for a custom Ollama or OpenAI-compatible vision model.
