## Summary

Keep open. Current `main` does not implement this PR’s Bitable requester-permission behavior: `createApp()` still only creates the app and returns metadata, `CreateAppSchema` has no `grant_to_requester`, and `feishu_bitable_create_app` is still registered through the generic helper without access to trusted requester context. The analogous trusted-requester permission flow exists only for `feishu_doc` today.
