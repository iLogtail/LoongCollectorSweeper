## Summary

Keep open. Current main still does not implement Feishu status/liveness reporting into gateway runtime state, and the submitted PR has unresolved review comments around premature `connected: true` and early-abort status gaps. The related #41940 context shows a parallel open attempt, not a merged or clearly canonical replacement.

## Best Possible Solution

Keep the PR open until a maintainer either accepts a corrected Feishu implementation or explicitly chooses a canonical replacement. The best fix should wire a status sink from `extensions/feishu/src/channel.ts` through `monitor.ts`, `monitor.account.ts`, and `monitor.transport.ts`; publish connection state only after WebSocket start or webhook listen succeeds; publish disconnected/error state on abort/error including early-abort paths; update inbound activity if still desired; and add regression coverage for both inbound activity and transport lifecycle behavior. If #41940 becomes the maintained canonical PR, then close this one as duplicate with that link after maintainer confirmation.
