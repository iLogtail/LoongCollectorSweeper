## Summary

Keep PR #27061 open. Current main still has the validation gap the PR targets: `validate_skill()` strips `name` and `description`, then only validates them inside truthy guards, so empty or whitespace-only values can still pass. The related superseding PR #35102 was closed unmerged, and the latest maintainer/Codex discussion already called out the same keep-open outcome. The PR should revise its packaging rationale because archives are named from the skill directory basename, not the frontmatter name.

## Best Possible Solution

Keep PR #27061 open, or land an equivalent narrow fix, until `skills/skill-creator/scripts/quick_validate.py` explicitly rejects empty and whitespace-only `name` and `description` after stripping and `skills/skill-creator/scripts/test_quick_validate.py` includes focused regression tests. Before merge, update the PR rationale to focus on invalid skill metadata and discovery/triggering behavior rather than the outdated archive filename claim.
