## Summary

Keep open. Current `main` does not implement this fix, and the proposed blanket `--deliver` bypass is still unsafe because other `deliver: true` callers rely on an LLM rewrite before delivery.

## Evidence

- **Current main still routes deliver requests through agent execution:** `agentCommandInternal` only checks send policy when `opts.deliver === true`, then continues into ACP or embedded-agent execution and finally calls `deliverAgentCommandResult` with the model result. There is no direct `opts.message` delivery path on current `main`.
  - file: [src/agents/agent-command.ts:433](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/agent-command.ts#L433)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '398,565p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Subagent announcements intentionally combine deliver + internalEvents:** Subagent announce delivery calls the gateway `agent` method with `deliver: !requesterIsSubagent` and `internalEvents: item.internalEvents`, so those deliveries depend on the agent turn to synthesize internal context into user-facing text.
  - file: [src/agents/subagent-announce-delivery.ts:366](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/subagent-announce-delivery.ts#L366)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '352,386p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Another current caller uses deliver without internalEvents but still needs model rewriting:** Exec-approval followups build an instruction-heavy prompt (`"Reply to the user in a helpful way."`, `"Continue the task if needed, then reply to the user in a helpful way."`) and send it through the gateway with `deliver: deliveryTarget.deliver`. A generic direct-delivery bypass would send that prompt text verbatim instead of a synthesized follow-up.
  - file: [src/agents/bash-tools.exec-approval-followup.ts:57](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/bash-tools.exec-approval-followup.ts#L57)
  - command: `nl -ba src/agents/bash-tools.exec-approval-followup.ts | sed -n '57,74p;127,160p;214,233p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **PR discussion already identified the same regressions:** Review comments on the PR head called out both the `internalEvents` leak path and the `sendExecApprovalFollowup` regression, matching the current `main` call sites above. That makes this PR useful but not safe to close as implemented or obsolete.
