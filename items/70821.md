## Summary

Keep #70821 open. Current main still does not expose `xhigh` from the `amazon-bedrock` provider profile for Claude Opus 4.7, so `/thinking xhigh` and `thinkingDefault: "xhigh"` remain blocked before the downstream reasoning path can use the higher effort level. The PR is a narrow, product-code-actionable fix with a targeted regression test.

## Best Possible Solution

Keep the PR open and review/land or reimplement the narrow fix: add a Claude Opus 4.7-specific Bedrock matcher in `extensions/amazon-bedrock/register.sync.runtime.ts`, append `xhigh` only for matching Opus 4.7 model IDs, preserve the existing Claude 4.6 `adaptive` behavior, and keep the targeted regression test in `extensions/amazon-bedrock/index.test.ts` covering regional Opus 4.7 positives and 4.6/Sonnet negatives. Run the Amazon Bedrock extension test lane and, if credentials are available, one live Bedrock smoke for `/thinking xhigh`.
