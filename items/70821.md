## Summary

Keep #70821 open. Current main still lacks the Bedrock-specific `xhigh` thinking profile entry for Claude Opus 4.7. The downstream Anthropic transport already knows how to map Opus 4.7 `xhigh`, but `amazon-bedrock` currently advertises only base levels plus Claude 4.6 `adaptive`, so the PR remains a focused actionable fix.

## Best Possible Solution

Keep the PR open and land or recreate the minimal provider-profile fix: add a Claude Opus 4.7 Bedrock matcher in `extensions/amazon-bedrock/register.sync.runtime.ts`, advertise `xhigh` only for matching Opus 4.7 model IDs, preserve the existing Claude 4.6 `adaptive` behavior, and add the focused `extensions/amazon-bedrock/index.test.ts` regression for regional Opus 4.7 positive and 4.6/Sonnet negative cases.
