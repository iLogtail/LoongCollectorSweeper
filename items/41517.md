## Summary

Current `main` does not already implement this fix, and the PR as written is not strong enough to close because it would broaden `resolveModel(...)` normalization beyond the stated Google/OpenRouter case and could strip legitimate provider-prefixed IDs.

## Evidence

- **`resolveModel` still uses exact model-id matching on current main:** `resolveModel(...)` normalizes with `normalizeStaticProviderModelId(...)` and then passes that value into resolver lookups. The lookup path still relies on exact `entry.id === modelId`, `modelRegistry.find(provider, modelId)`, and fallback `candidate.id === modelId`, so `provider=google` with `modelId=google/gemini-2.5-pro` would not match a configured `gemini-2.5-pro` entry.
  - file: [src/agents/pi-embedded-runner/model.ts:263](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/pi-embedded-runner/model.ts#L263)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Current normalization has no generic same-provider prefix stripping:** `normalizeStaticProviderModelId(...)` only applies provider-specific rewrites for Anthropic, Hugging Face, Google, OpenRouter, xAI, and Vercel AI Gateway. There is no generic rule that turns `google/gemini-2.5-pro` into `gemini-2.5-pro` before `resolveModel(...)` lookup.
  - file: [src/agents/model-ref-shared.ts:59](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/model-ref-shared.ts#L59)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Same-provider-prefixed IDs are valid in current main:** `resolveModelFromRegistry(...)` explicitly retries a provider-prefixed variant when the bare lookup misses, and its regression test proves `kimchi/claude-opus-4-6` is a valid model ID for provider `kimchi`. That means stripping any same-provider first segment in `resolveModel(...)` would be too broad.
  - file: [src/agents/tools/media-tool-shared.ts:423](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/tools/media-tool-shared.ts#L423)
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Parser tests already distinguish meaningful prefixed IDs from bare aliases:** Current tests preserve `openrouter/aurora-alpha` as a native OpenRouter model ID and pass through `openrouter/anthropic/claude-sonnet-4-6` as an upstream-provider model ID. The repo already treats prefixed model IDs as semantically meaningful, so a blanket same-provider strip in `resolveModel(...)` needs tighter scoping than this PR currently has.
