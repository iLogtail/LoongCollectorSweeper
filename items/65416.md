## Summary

PR #65416 is obsolete against current main. The requested `video_generate` buffer save cap is implemented more broadly on main: generated videos now pass a generated-media limit resolved for `video`, falling back to the 16MB video cap instead of `saveMediaBuffer`'s generic 5MB default, with regression coverage and docs.

## Best Possible Solution

Close PR #65416 as implemented on current main and keep the broader shipped implementation from `3507efa4ec1e8b7c22889ab5e2c6c9283adeb768`, which preserves the video cap, honors configured media limits, and adds hosted-URL fallback for oversized generated videos.
