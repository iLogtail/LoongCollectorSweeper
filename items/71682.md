## Summary

Keep #71682 open. Current main partially addresses the timeout-setting part with `agents.defaults.llm.idleTimeoutSeconds`, schema support, CLI write coverage, and docs, but the issue also reports that the LLM cannot change the timeout and current owner-only agent gateway config writes still do not allow `agents.defaults.llm.idleTimeoutSeconds`. The timeout/fallback cooldown behavior remains separately tracked by open #51057, so this issue still has unresolved maintainer work rather than being safe to close.

## Best Possible Solution

Keep open for maintainer triage. The practical fix path is to decide whether the owner-only agent gateway config tool should allow a bounded edit to `agents.defaults.llm.idleTimeoutSeconds`, document the exact user CLI workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>`), and leave timeout auth-profile cooldown/faster fallback behavior to the open #51057 thread unless #71682 adds a distinct repro beyond slow local Ollama idle timeouts.
