## Summary

Keep #71682 open. Current main partially addresses long model idle timeouts with `agents.defaults.llm.idleTimeoutSeconds`, schema support, CLI write coverage, docs, and a runtime error hint. The issue is not safely closeable because it also reports that the LLM/agent cannot change the timeout, and current agent-facing `gateway config.patch/apply` still excludes `agents.defaults.llm.idleTimeoutSeconds`; related timeout cooldown/faster fallback behavior also remains open in #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to document the operator command `openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` for slow local models, then decide whether the owner-only agent-facing `gateway config.patch/apply` allowlist should permit bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Keep timeout auth-profile cooldown and faster fallback policy aligned with the existing open #51057 thread unless #71682 adds a distinct repro.
