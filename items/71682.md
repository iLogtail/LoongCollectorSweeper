## Summary

Keep #71682 open. Current main already has an operator-side LLM idle-timeout knob, schema support, CLI write coverage, docs, and timeout error guidance, but the report also asks for the LLM/agent to be able to change that timeout. Current main still fail-closes agent-driven `gateway config.patch/apply` outside the narrow prompt/model/mention allowlist, so `agents.defaults.llm.idleTimeoutSeconds` remains blocked. The related timeout cooldown/fallback behavior is still a separate open product decision tracked by #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical fix path is to document the operator workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>`) and decide whether the owner-only gateway tool should allow bounded edits to `agents.defaults.llm.idleTimeoutSeconds` through `config.patch/apply`. Keep the auth-profile timeout cooldown/faster fallback policy in #51057 unless this issue adds a distinct repro, and use the existing `/model` and `openclaw models set <provider/model>` paths for model-selection guidance.
