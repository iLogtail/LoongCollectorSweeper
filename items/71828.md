## Summary

Keep PR #71828 open. Current main still has the controller-level typing TTL gap the PR targets: keepalive ticks call the typing callback, but they do not refresh `createTypingController`'s own TTL, so cleanup can stop Discord typing during a still-active long run.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land this targeted controller TTL refresh plus its regression test, or intentionally fold the same fix into the broader #65919 active-run presence lease. The implementation should preserve cleanup after run completion plus dispatch idle and avoid refreshing after the controller is sealed.

## Evidence

- **Current main lacks TTL refresh on keepalive tick:** `refreshTypingTtl()` arms the controller safety timer, but `triggerTyping()` only calls `onReplyStart`; the keepalive loop uses `triggerTyping` for interval ticks, so those ticks do not extend the controller TTL. `startTypingLoop()` refreshes TTL only when invoked, and returns early once the loop is already running.
