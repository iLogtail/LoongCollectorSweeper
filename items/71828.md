## Summary

Keep PR #71828 open. Current main still refreshes the run-side typing TTL only when the typing loop is started or text starts typing, while keepalive ticks run through `triggerTyping()` without extending that controller TTL. Discord routes typing through this shared reply pipeline, so the PR's one-line refresh plus regression test still address a real active-run liveness gap rather than obsolete work.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land the targeted `triggerTyping()` TTL refresh with its regression test, or fold the same behavior into the broader #65919 active-run presence lease while preserving cleanup after run completion plus dispatch idle and the sealed-controller guard.
