## Summary

Keep #59949 open. Current main still does not implement `agents.list[].includedWorkDirs`: the agent config contract/schema omit it, resolved agent config does not carry it, workspace path inference only checks the primary workspace, and file-tool wiring still guards against a single workspace root. The PR is also recent and coherent, with useful branch work plus review comments identifying fixable boundary and cleanup issues rather than evidence that the feature is obsolete.

## Best Possible Solution

Keep the PR open as a real missing core capability. The best path is a focused rebase that limits the diff to the `includedWorkDirs` contract and filesystem-boundary plumbing, resolves the posted boundary/security review comments, adds docs for `agents.list[].includedWorkDirs`, and keeps tests across config normalization/schema, agent resolution, read/write/edit, apply_patch, sandbox media, image, and PDF paths.
