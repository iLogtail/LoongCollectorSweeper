## Summary

Keep #70529 open. The issue has a recent maintainer comment explicitly saying to keep it open as the Desktop-specific variant, and current main still defaults the built-in `user` profile to the existing-session Chrome MCP path that spawns `npx chrome-devtools-mcp@latest`. Related #62288 covers broader brittle attach diagnostics, but it does not supersede the packaged Desktop failure where the app bundle may provide `node` without `npm`/`npx`.

## Best Possible Solution

Keep #70529 open for a Desktop/browser-runtime fix. The best path is to remove the default `profile="user"` dependency on ambient `npx` in packaged Desktop builds by bundling or explicitly resolving the Chrome MCP driver, or to add a structured Desktop diagnostic/fallback that points users to the managed `openclaw` profile and the separate login handoff work. Keep #62288 as the broader attach/diagnostics tracker unless it is explicitly expanded to cover this packaged-app `npx` failure.
