## Summary

Keep #65656 open. Current main still routes the reported text-plus-table path as a text Reply API call followed by a separate rich/table Push API call, and the related PR #65673 was closed after reviewers and the author agreed it was a no-op rather than a fix.

## Best Possible Solution

Keep the issue open and implement the real LINE delivery fix in the bundled LINE plugin: allow `sendLineReplyChunks` or the surrounding auto-reply delivery path to coalesce text chunks plus rich/media messages into one Reply API batch up to LINE’s five-message limit, then push only overflow. Add a focused regression test for text plus Markdown table/Flex message with an unused reply token, and preserve the current quick-reply ordering rules deliberately.
