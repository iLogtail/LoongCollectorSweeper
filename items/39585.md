## Summary

Keep #39585 open. Current main has Telegram-native exec approval routing and generic elevated full mode that can skip exec approvals, but it still lacks the requested Telegram-scoped trusted-owner/group/tool-class approval bypass with audit labeling and destructive-operation fallback behavior.

## Best Possible Solution

Keep the issue open. A complete fix should live in the bundled Telegram plugin/config surface and add an explicit opt-in policy requiring resolved numeric owner sender ID, allowlisted group/topic, configured tool classes, an audit reason such as trusted-owner-bypass, destructive-operation fallback behavior, schema/docs updates, and focused Telegram approval-path tests.
