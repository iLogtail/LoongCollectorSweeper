## Summary

Keep #39585 open. Current main still has Telegram-native exec approval delivery and generic host/elevated ways to avoid prompts, but it does not implement the requested Telegram-scoped trusted owner x allowlisted group/topic x tool-class approval bypass with audit labeling. The prior ClawSweeper comment correctly identified the gap, and the relevant files are unchanged in substance on current main.

## Best Possible Solution

Keep the issue open. A complete solution should live in the Telegram plugin/config surface and add an explicit opt-in policy requiring a resolved owner sender plus allowlisted group/topic, scoping bypass to configured tool classes, recording a trusted-owner-bypass audit marker, preserving destructive-operation fallbacks, and adding schema, docs, and focused tests around the Telegram exec approval path.
