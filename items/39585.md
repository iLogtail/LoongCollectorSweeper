## Summary

Keep open. Current main implements Telegram-native exec approval delivery and broad session-level approval bypasses, but it still does not implement the requested Telegram group trusted-owner approval bypass or owner x group x tool-class policy matrix.

## Best Possible Solution

Keep #39585 open until OpenClaw either implements an explicit Telegram trusted-owner approval policy or a maintainer decides the broad session-level elevated/full workflow is the intended answer. A complete implementation would likely touch Telegram config types/schema/docs, Telegram sender/group policy resolution, exec approval context/audit metadata, and tests proving only explicitly allowlisted owners in allowlisted groups can bypass configured tool classes.
