## Summary

Keep #39585 open. Current main has Telegram group sender allowlisting, Telegram-native approval delivery, approval callback handling, and generic `/elevated full` approval skipping, but it does not implement the requested Telegram-scoped trusted owner/group/tool-class approval bypass with audit labeling or destructive-operation fallback behavior.

## Best Possible Solution

Keep the issue open. A complete fix should live in the bundled Telegram plugin/config surface and add an explicit opt-in policy requiring resolved numeric owner sender IDs, allowed group/topic scope, configured tool classes, audit metadata such as trusted-owner-bypass, destructive-operation fallback behavior, schema/docs updates, and focused Telegram approval-path tests.
