## Summary

Keep #39558 open. Current main still has the Feishu reaction-ID bug surface: reaction events synthesize `om_xxx:reaction:EMOJI:uuid` IDs, inbound context preserves that suffixed ID, and several Feishu API boundaries still pass raw message IDs to Feishu SDK calls. The PR is a plausible fix for linked issue #34528, but it needs refresh against current main because newer/changed surfaces such as streaming cards are not covered by the stale patch context.

## Best Possible Solution

Refresh #39558 or replace it with a current-main Feishu plugin patch that centralizes reaction-suffix stripping at API-facing message-ID boundaries while preserving suffixed IDs for internal dedup/session identity. Cover send/reply/get/edit/update, media resource and media replies, reactions, typing indicators, thread/action paths, and streaming-card replies, then add regression tests for the helper and at least one API boundary using a suffixed `om_xxx:reaction:EMOJI:uuid` ID.
