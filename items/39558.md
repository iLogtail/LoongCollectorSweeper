## Summary

Keep open. Current main still generates Feishu synthetic reaction message IDs with the `:reaction:<emoji>:<uuid>` suffix and still routes those IDs into Feishu API path parameters without a normalization helper. PR #39558 is not implemented or obsolete; it remains a useful candidate fix, though it likely needs refresh for newer paths such as streaming-card replies.

## Best Possible Solution

Keep this PR, #34528, or a canonical successor open until Feishu plugin code normalizes OpenClaw's synthetic `:reaction:<emoji>:<uuid>` suffix before every Feishu API `message_id` boundary, with focused regression tests for reply, get/edit/update, reaction, typing, media, and streaming-card paths.
