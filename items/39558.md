## Summary

Keep #39558 open. Current main still preserves Feishu synthetic reaction IDs internally and still passes raw `messageId`/`replyToMessageId` values into Feishu API path parameters. The PR is a plausible fix for the still-open canonical bug #34528 and has useful prior Greptile review support, but it needs a current-main refresh because newer Feishu API boundaries are not covered and one touched file no longer exists.

## Best Possible Solution

Refresh #39558 or replace it with an equivalent Feishu plugin patch that strips OpenClaw's synthetic `:reaction:<emoji>:<uuid>` suffix at every Feishu API-facing message ID boundary while preserving suffixed IDs for internal dedupe/session identity. Cover send/reply/get/edit/update, reactions, typing indicators, media resource and media replies, streaming-card replies, pin/action paths, and add regression tests for the helper plus at least one API boundary receiving a suffixed reaction message ID.
