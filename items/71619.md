## Summary

Keep #71619 open. Current main still includes Linux version-manager directories in the daemon minimal service PATH and tests still assert that behavior, so this PR is not obsolete or implemented on main. It remains a focused fix for #71578's reported doctor/minimal-PATH contradiction.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: make Linux minimal service PATH include only stable user bins, pnpm/NPM prefix, Nix profiles, and system dirs while preserving macOS version-manager behavior; update `service-env` and audit regression tests for #71578's hardened-PATH expectation.
