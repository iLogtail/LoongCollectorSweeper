## Summary

Keep #71619 open. Current main still treats Linux version-manager directories as part of the daemon minimal service PATH, and the current tests still assert that behavior. The PR remains a focused fix for the #71578 doctor/minimal-PATH contradiction rather than something already implemented or superseded.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: make Linux minimal service PATH include stable user bins, PNPM/NPM prefix, Nix profiles, and system dirs while excluding Linux version-manager directories; preserve macOS version-manager behavior; update service-env and service-audit regression coverage for #71578's hardened-PATH expectation.
