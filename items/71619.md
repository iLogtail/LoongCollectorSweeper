## Summary

Keep #71619 open. Current main still treats Linux version-manager directories and env-configured roots as part of the daemon minimal service PATH, and the tests still assert that behavior. The PR remains a focused fix for #71578's minimal-PATH contradiction rather than something already implemented or superseded.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: split stable Linux user-bin/Pnpm/NPM/Nix PATH entries from version-manager entries, keep macOS behavior unchanged, and update service-env plus service-audit regression coverage and any user-facing docs that describe Linux service PATH contents.
