## Summary

Keep #71619 open. Current `main` still treats Linux version-manager directories as part of the daemon minimal service PATH, and current tests still assert that behavior. The PR is not obsolete or implemented on main; it remains a focused fix for #71578's reported doctor/minimal-PATH contradiction.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: make Linux minimal service PATH include only stable user bins, pnpm/NPM prefix, Nix profiles, and system dirs while preserving macOS version-manager behavior; update service-env/audit regression coverage and Linux PATH docs to match the hardened policy.
