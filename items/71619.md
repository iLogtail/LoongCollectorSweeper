## Summary

Keep #71619 open. Current main still treats Linux version-manager directories as part of the daemon minimal service PATH, and existing tests/docs still lock in that older behavior. The PR remains a focused fix for #71578 rather than obsolete or implemented on main.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch. The likely product direction is to make Linux minimal service PATH include stable user bins, pnpm/NPM prefix, Nix profiles, and system dirs while preserving macOS version-manager behavior, then update service-env and service-audit regression tests plus any user-facing docs that still describe the broader Linux PATH.
