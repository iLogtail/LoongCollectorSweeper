## Summary

Keep open: current `main` still has the broken multi-line export HTML placeholders in `template.html`, while `commands-export-session.ts` still performs literal `{{...}}` replacements, so the PR addresses a real unfixed regression and the current tests do not cover the broken template path.

## Evidence

- **Current main still has broken placeholder syntax:** `template.html` on current `main` contains `MARKED_JS;`, `HIGHLIGHT_JS;`, and `JS;` inside nested block syntax instead of literal `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` tokens, matching the regression described in the PR.
  - file: [src/auto-reply/reply/export-html/template.html:61](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/auto-reply/reply/export-html/template.html#L61)
  - command: `nl -ba src/auto-reply/reply/export-html/template.html | sed -n '50,85p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Export code still expects literal `{{...}}` placeholders:** `generateHtml()` still calls `.replace("{{JS}}", ...)`, `.replace("{{SESSION_DATA}}", ...)`, `.replace("{{MARKED_JS}}", ...)`, and `.replace("{{HIGHLIGHT_JS}}", ...)`. With the current template contents, the three JavaScript injections will not match.
  - file: [src/auto-reply/reply/commands-export-session.ts:91](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/auto-reply/reply/commands-export-session.ts#L91)
