## Summary

Keep PR #18860 open. Current main at 560ddd2f9b13fe2cf2073ea2922df481b7c06bb4 still does not expose `after_tools_resolved` or an equivalent plugin-facing API for discovering the resolved per-attempt tool list and schemas. The related issue #21330 is closed as superseded by this PR, and the provided discussion includes a same-day maintainer comment explicitly keeping this PR open.

## Best Possible Solution

Keep PR #18860 open as the active implementation/design context for plugin-facing resolved tool/schema discovery. A maintainer-quality path is a narrow additive contract across `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, and `src/agents/pi-embedded-runner/run/attempt.ts`, with focused contract tests and public docs in `docs/plugins/hooks.md`. If maintainers prefer the pull-based `getResolvedTools` design from #21330, designate that replacement explicitly before superseding this PR.
