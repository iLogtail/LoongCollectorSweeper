## Summary

Keep PR #18860 open. Current main still lacks the requested plugin-facing `after_tools_resolved` hook and lacks an equivalent `listTools`/`getTools`/`getResolvedTools` API on `OpenClawPluginApi`. The newer `tools.effective` surface is operator/UI inventory only and omits JSON Schema parameters. Linked issue #21330 was closed as superseded by this PR, and the latest maintainer/Codex review comment also explicitly kept this PR open.

## Best Possible Solution

Keep PR #18860 open as the active design/implementation context for a plugin-facing resolved-tools/schema discovery seam. The best path is a narrow maintainer-approved contract change in `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, and `src/agents/pi-embedded-runner/run/attempt.ts`, with focused contract tests and docs in `docs/plugins/hooks.md`; if maintainers prefer the pull-based `getResolvedTools()` design from #21330, designate a new canonical follow-up before superseding this PR.
