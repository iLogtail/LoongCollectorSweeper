## Summary

Keep PR #62338 open. Current main still does not implement the PR’s central doctor-path behavior: `doctor.memory.status` only returns embedding/dreaming data, `probeGatewayMemoryStatus` only forwards embedding readiness, and `noteMemorySearchHealth` has no FTS5-unavailable warning. The PR remains a focused, plausible fix for the doctor visibility part of #62328, while the broader node:sqlite/FTS5 fallback problem remains separate.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent focused change on main: add optional `fts` to `DoctorMemoryStatusPayload`, copy `manager.status().fts` into `doctor.memory.status`, extend `GatewayMemoryProbe`, emit a clear FTS5-unavailable doctor note in `noteMemorySearchHealth`, and keep the regression tests from the PR. Continue tracking the larger node:sqlite/FTS5 fallback decision in #62328.
