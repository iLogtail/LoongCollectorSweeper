## Summary

Keep open. Current `main` still implements cron delivery as a single target, with CLI/docs/tests all exposing one `delivery` object and one `--channel`/`--to` pair. I found no evidence that multi-channel cron fanout is already implemented or that it cleanly belongs in ClawHub instead of core/plugin-API work.

## Evidence
