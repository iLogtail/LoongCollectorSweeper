## Summary

Keep #37943 open. Current main still routes the macOS node service through `openclaw node run`, and that startup path still calls strict `loadConfig()` on the main `openclaw.json`. Gateway startup has separate best-effort and last-known-good recovery paths, so the reporter's observation that the gateway can run does not prove the node service config failure is fixed.

## Best Possible Solution

Keep the issue open. A maintainer should either make `openclaw node run` use a deliberate best-effort/recovery config path compatible with gateway startup, or keep strict validation but fail with precise schema diagnostics that explain why the node service rejects a JSON-valid config. The likely implementation surface is `src/node-host/runner.ts` plus the node CLI/service path, with a focused regression test for JSON-valid but schema-invalid main config and docs/logging updates if strict node-host validation is intentional.
