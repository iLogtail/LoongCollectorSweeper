## Summary

Keep open. I did not find strong evidence that current `main` has already fixed or obsoleted this report. The node host still loads the shared OpenClaw config on startup, the shared config path still marks parse failures as invalid JSON5, and the node service still depends on its installed service environment for `OPENCLAW_CONFIG_PATH`/`OPENCLAW_STATE_DIR`, so the reported macOS service-only mismatch remains plausible.
