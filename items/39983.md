## Summary

Keep open. Current `main` still routes Ollama memory embeddings through the superseded `/api/embeddings` endpoint with a `prompt` payload and `embedding[]` response shape, and the bundled tests explicitly lock that behavior in. The report matches current code rather than stale behavior.

## Evidence

- **Current Ollama embedding implementation still uses deprecated endpoint:** `createOllamaEmbeddingProvider` builds `.../api/embeddings`, POSTs `{ model, prompt }`, parses `{ embedding?: number[] }`, and throws if `embedding[]` is missing.
  - file: [extensions/ollama/src/embedding-provider.ts:146](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/ollama/src/embedding-provider.ts#L146)
  - command: `nl -ba extensions/ollama/src/embedding-provider.ts | sed -n '140,170p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Affected surface is the registered Ollama memory embedding provider:** The Ollama plugin wires `createEmbeddingProvider` to `createOllamaEmbeddingProvider`, so this path is the live memory-search embedding path, not dead code.
  - file: [extensions/ollama/index.ts:173](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/ollama/index.ts#L173)
  - command: `nl -ba extensions/ollama/index.ts | sed -n '160,210p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Tests currently assert old endpoint behavior:** `embedding-provider.test.ts` names the behavior as `calls /api/embeddings` and expects requests to `http://127.0.0.1:11434/api/embeddings`, confirming current main still enshrines the deprecated contract.
  - file: [extensions/ollama/src/embedding-provider.test.ts:48](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/ollama/src/embedding-provider.test.ts#L48)
  - command: `nl -ba extensions/ollama/src/embedding-provider.test.ts | sed -n '40,150p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Upstream Ollama API contract has moved to /api/embed:** The official Ollama API docs currently document `POST /api/embed` with `input`, and separately mark `POST /api/embeddings` as superseded.
  - command: `curl -L --silent https://raw.githubusercontent.com/ollama/ollama/main/docs/api.md | sed -n '1690,1848p'`
- **Issue discussion already identified the right fix path:** The issue comments and timeline point to the same migration: `/api/embeddings` -> `/api/embed`, `prompt` -> `input`, and `embedding` -> `embeddings`. Multiple cross-referenced PR attempts exist, but current `main` still contains the old implementation.
