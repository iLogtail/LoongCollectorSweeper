## Summary

Issue #39983 should remain open. Current main still uses Ollama's deprecated `/api/embeddings` endpoint with a `prompt` request body and parses an `embedding` response for the bundled Ollama memory embedding provider, while the issue and related PRs target migration to `/api/embed` with `input` and `embeddings[][]`. The current tests also assert the deprecated contract, so no fix has landed on main.

## Best Possible Solution

Keep the issue open and land a focused fix in the bundled Ollama plugin: update `extensions/ollama/src/embedding-provider.ts` to call `/api/embed`, send `input`, parse `embeddings[0]` or batch `embeddings`, preserve existing base URL/auth/header/SSRF behavior, and update `extensions/ollama/src/embedding-provider.test.ts` to assert the current Ollama API contract. After that lands, close this issue and supersede duplicate open PRs #40038 and #40205 against the merged fix.
