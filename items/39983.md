## Summary

Current main still implements Ollama memory embeddings with the superseded `/api/embeddings` endpoint, `prompt` request field, and `embedding[]` response shape. The linked issue remains a real current bug rather than stale or implemented behavior.

## Best Possible Solution

Keep #39983 open until OpenClaw migrates the bundled Ollama memory embedding provider to `POST /api/embed` with `input` and `embeddings[][]` parsing, updates the colocated Ollama embedding tests, and verifies `memorySearch.provider: "ollama"` still preserves base URL, auth header, SSRF, timeout, and model-normalization behavior.
