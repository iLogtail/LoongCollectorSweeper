## Summary

Keep #39983 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still implements the bundled Ollama memory embedding provider with the deprecated `/api/embeddings` endpoint, sends `prompt`, parses a single `embedding` array, and has tests that assert that old contract. Official Ollama docs now document `/api/embed` with `input` and `embeddings`, and note that `/api/embeddings` is superseded. Related PR #40205 is an open fix attempt, but the fix has not landed on main.

## Best Possible Solution

Keep the issue open and land a focused bundled Ollama plugin fix. Update `extensions/ollama/src/embedding-provider.ts` to call `/api/embed`, send `input` (ideally supporting string and batch input), parse `embeddings[0]` or batch `embeddings`, preserve existing base URL, auth, custom header, timeout, and SSRF behavior, and update `extensions/ollama/src/embedding-provider.test.ts` to assert the current Ollama API contract. PR #40205 can be used as candidate implementation, but the issue should close only after the fix lands on main.
