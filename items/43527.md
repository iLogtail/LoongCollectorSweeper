## Summary

Keep open. Current main still formats the normal status Agents overview as `no bootstrap files` when `bootstrapPendingCount` is zero, and that count is still derived only from `BOOTSTRAP.md` existence. Docs confirm `BOOTSTRAP.md` is a one-time file deleted after completion, so the reported wording bug remains on the `openclaw status` path even though the separate `status --all` helper already uses `N bootstrapping`.

## Best Possible Solution

Keep the issue open until the normal `openclaw status` Agents overview is changed to describe the actual signal, for example `0 bootstrapping` / `N bootstrapping`, and add or update regression coverage so mature workspaces without `BOOTSTRAP.md` are not described as missing bootstrap files. PR #43614 appears to be the closest open implementation path; alternatively apply the same scoped fix directly to `src/commands/status.command-sections.ts` and the status tests.
