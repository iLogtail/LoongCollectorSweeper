## Summary

Close as implemented on current `main`. The reported failure mode assumes a memory/distillation subagent is incrementally editing a shared session file, but current code no longer matches that architecture: active-memory runs are isolated to per-run transcript files, only `memory_search`/`memory_get` are allowed, and core transcript mutations are guarded by a process-aware session write lock with contention tests.

## Evidence

- **Active-memory prompt forbids edit-style tools:** The recall subagent prompt explicitly says `Use only memory_search and memory_get.`, which does not match the issue’s claimed `edit`-tool-based update path.
  - file: [extensions/active-memory/index.ts:789](https://github.com/openclaw/openclaw/blob/272313877dc867f83008810302b3b6eba4bb1900/extensions/active-memory/index.ts#L789)

That removes the shared-session-file collision described in the report.
  - file: [extensions/active-memory/index.ts:1616](https://github.com/openclaw/openclaw/blob/272313877dc867f83008810302b3b6eba4bb1900/extensions/active-memory/index.ts#L1616)

- **Core agent runs acquire a session write lock before transcript work:** `runEmbeddedPiAgent` acquires `acquireSessionWriteLock({ sessionFile, ... })` before opening the run/session machinery, so transcript writers are serialized at the file level.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:520](https://github.com/openclaw/openclaw/blob/272313877dc867f83008810302b3b6eba4bb1900/src/agents/pi-embedded-runner/run/attempt.ts#L520)

- **Session write lock is process-aware and file-based:** `acquireSessionWriteLock` creates a `.lock` file with `fs.open(lockPath, "wx")`, waits/retries on contention, reclaims stale locks, and throws `session file locked` instead of allowing concurrent mutation of the same transcript.
  - file: [src/agents/session-write-lock.ts:475](https://github.com/openclaw/openclaw/blob/272313877dc867f83008810302b3b6eba4bb1900/src/agents/session-write-lock.ts#L475)

  - file: [src/agents/session-write-lock.test.ts:143](https://github.com/openclaw/openclaw/blob/272313877dc867f83008810302b3b6eba4bb1900/src/agents/session-write-lock.test.ts#L143)
