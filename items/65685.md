## Summary

Keep open. Current main still has the reported enqueue-followup path: it enqueues the followup, only schedules an immediate drain when the active snapshot is stale, and then unconditionally calls `typing.cleanup()` before returning. Because cleanup seals the typing controller, the queued followup cannot keep or restart a typing signal while it waits behind a real active run.

## Best Possible Solution

Keep the issue open and fix `src/auto-reply/reply/agent-runner.ts` so the enqueue-followup branch distinguishes stale snapshots from real queued-behind-active runs. For real active runs, keep the typing controller alive and start or refresh a liveness signal until the active run clears; for stale snapshots, continue scheduling the followup drain immediately. Add focused regression coverage for both branches in the runReplyAgent queue tests, with an assertion that active queued followups do not call `typing.cleanup()` prematurely.
