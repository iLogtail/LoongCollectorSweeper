## Summary

Current main still does not preserve Slack CodedError structured fields in delivery queue recovery. Recovery now uses the shared formatter rather than the older direct err.message pattern, but that formatter still only records Error.message plus cause-chain text, so data.needed and data.response_metadata.scopes/acceptedScopes remain lost before lastError is persisted or logged.

## Best Possible Solution

Keep #62391 open. The best fix is a narrow recovery-safe diagnostic formatter, or a shared formatter that recovery can use, which preserves Slack Web API structured fields after redaction and adds focused delivery-queue tests for a Slack missing_scope CodedError. Maintainers should also decide whether missing_scope belongs in the permanent delivery error policy for queued outbound sends.
