## Summary

Current main still renders cron schedules in the Control UI as raw `Cron <expr>` strings, so PR #65382 is not implemented or superseded. The PR is a focused UI fix and its current head, per the provided PR context, adds coverage for common hourly and minute intervals, including the bot-reviewed `0 * * * *` and `*/5 * * * *` cases. It should stay open for maintainer review, with the remaining concern being whether the parser should range-check step values before humanizing malformed cron strings.

## Best Possible Solution

Keep PR #65382 open. The best path is to update or review the PR so `formatCronSchedule` humanizes common valid cron intervals in `ui/src/ui/presenter.ts`, keeps unsupported and malformed expressions on the raw `Cron ...` fallback, preserves timezone suffixes, and lands with focused presenter tests for hourly, minute, fallback, timezone, and invalid step-range cases.
