## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation that PR #41653 targets, and the keepalive loop still exercises that path. The PR discussion and crash signature describe a plausible focused bug, so this should not be closed as stale or implemented, though the patch likely needs maintainer cleanup because it bundles a redundant Swift workflow and lacks a duplicate-pong regression test.

## Best Possible Solution

Keep the item open, then land a cleaned-up one-file `WebSocketTaskBox.sendPing()` fix that guarantees at-most-once continuation resume and adds a focused duplicate-pong regression test. Drop or split the added Swift workflow, and once a canonical fix is selected, close the duplicate ping-continuation PRs against that canonical item.
