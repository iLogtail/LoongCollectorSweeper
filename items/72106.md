## Summary

Keep open. Current main accepts `agents.defaults.compaction.reserveTokensFloor` and reapplies it after Pi resource-loader reloads, but the user-facing overflow recovery paths still emit the exact hard-coded advice to set the floor to `20000 or higher`. The issue is recent, has a concrete configuration and environment, and overlaps with still-open guidance cleanup in #65839/#67314, but it is not safe to auto-close because the reporter's claim may be either a real reload/config-path bug or a misleading recovery message that needs maintainer cleanup.

Keep this open until maintainers either reproduce the Debian service restart path or consolidate it under a canonical context-overflow guidance issue. The likely fix is to replace the hard-coded overflow message in `src/auto-reply/reply/agent-runner-execution.ts` with context/model/config-aware guidance, add tests for both reset paths, and document a troubleshooting flow using `openclaw config file`, `openclaw config validate`, the active model/provider context window, and the service restart path.
