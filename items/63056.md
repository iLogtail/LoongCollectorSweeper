## Summary

Keep open. Current `main` already uses `GET`, so the issue body's stated root cause is outdated, but the linked follow-up PR and the code both show the MiniMax usage fetch still points at `api.minimaxi.com` instead of the documented global `api.minimax.io`. That means the underlying MiniMax usage bug tied to this issue is not implemented on `main`, and the latest release still carries the same endpoint.
