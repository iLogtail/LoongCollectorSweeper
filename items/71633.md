## Summary

Keep open. Current main still ships the default-enabled Bonjour plugin with unpatched @homebridge/ciao@1.3.6, and upstream ciao 1.3.6 still calls the Windows arp polling command without an options object. #71633 is a narrow current-main fix for a real remaining bundled-plugin runtime path, not conservative cleanup to close.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump if one becomes available, with lockfile consistency and Windows-focused verification that ciao's arp polling exec receives windowsHide: true.
