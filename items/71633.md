## Summary

Keep open. Current main still resolves the default-enabled bundled Bonjour plugin to unpatched `@homebridge/ciao@1.3.6`, and the upstream stable package still runs the Windows `arp -a | findstr` exec without `windowsHide`. The PR is a narrow live fix for the linked Windows console-flicker report, not obsolete cleanup; the prior issue closure checked the repo’s own discovery file but missed the bundled dependency’s runtime code path.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow `@homebridge/ciao@1.3.6` pnpm patch, or replace it with a stable upstream ciao release once one ships the same `windowsHide` fix, with lockfile consistency and either Windows manual verification or a focused regression check proving the ciao arp-poll `exec` receives `{ windowsHide: true }`.
