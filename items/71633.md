## Summary

Keep PR #71633 open. Current main at 21c51bc still ships the default-enabled Bonjour plugin with unpatched @homebridge/ciao@1.3.6, and upstream ciao 1.3.6 still runs the Windows arp polling exec without an options object while Node's windowsHide option defaults to false. The linked Windows flicker report (#71492) maps to this runtime path. Later PR #72332 also includes a broader version of the fix, but it is open and carries additional Bonjour/TUI/dependency hardening, so it is not a clean canonical superseder for this narrow PR yet.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump if one becomes available, with lockfile consistency and focused Windows/manual verification that ciao's arp polling exec receives windowsHide: true. If maintainers choose #72332 as the canonical broader fix, close #71633 only after explicitly marking it superseded by that PR.
