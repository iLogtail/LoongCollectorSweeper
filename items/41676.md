## Summary

Keep open. Current main adds a useful structured file-log size cap (`logging.maxFileBytes`), but the reported user-visible problem is specifically the launchd/service stderr file `gateway.err.log`. Current main still writes launchd `StandardErrorPath` directly to that file and I found no max-size, rotation, truncation, or dedupe control for that service log path. The issue's requested capability therefore remains meaningfully incomplete.
