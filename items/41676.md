## Summary

Keep open. Current `main` still sends Gateway service stderr to a fixed `gateway.err.log` path with no built-in rotation or max-size control. The existing `logging.maxFileBytes` cap applies to the separate internal OpenClaw logger, not to launchd/service-managed stdout/stderr files.

## Evidence

- **Gateway service still uses fixed stderr/stdout files:** `resolveGatewayLogPaths` returns `${stateDir}/logs/gateway.log` and `${stateDir}/logs/gateway.err.log`, so the service log target is still a stable append-only file pair.
  - file: [src/daemon/restart-logs.ts:18](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/daemon/restart-logs.ts#L18)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **launchd plist writes those paths directly:** Gateway LaunchAgent installation passes `stdoutPath` and `stderrPath` straight into `StandardOutPath` and `StandardErrorPath`; there is no rotation/truncation layer here.
  - file: [src/daemon/launchd-plist.ts:116](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/daemon/launchd-plist.ts#L116)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Docs still point operators to gateway.err.log:** The troubleshooting FAQ still documents macOS service logs as `$OPENCLAW_STATE_DIR/logs/gateway.log` and `gateway.err.log`, which matches the current daemon codepath.
  - file: [docs/help/faq.md:1565](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/help/faq.md#L1565)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Only the internal file logger has a size cap:** `buildLogger` suppresses writes after `settings.maxFileBytes`, but that logic is for the OpenClaw logger sink, not launchd-captured stderr.
  - file: [src/logging/logger.ts:183](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/logging/logger.ts#L183)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Config/docs confirm external rotation is still expected:** The config schema exposes `logging.maxFileBytes`, and the docs explicitly say to use external log rotation for production deployments; there is no matching gateway service log rotation setting.
  - file: [docs/gateway/configuration-reference.md:770](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/gateway/configuration-reference.md#L770)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Gateway diagnostics still read whole stderr/stdout files:** `readLastGatewayErrorLine` loads both log files fully with `fs.readFile`, which reinforces that the gateway service log files are treated as persistent flat files rather than managed rotating logs.
  - file: [src/daemon/diagnostics.ts:27](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/daemon/diagnostics.ts#L27)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
