## Summary

Keep open. Current main has a global subagent concurrency cap and status/task visibility, but it does not expose remaining subagent capacity in `session_status` and there is no `sessions_capacity` tool. `sessions_spawn` documents immediate acceptance/queueing and only returns structured limit errors for per-session child caps/depth, not for the global concurrency lane budget.

## Evidence

  - file: [src/status/status-text.ts:247](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/status/status-text.ts#L247)
