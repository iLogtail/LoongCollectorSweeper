## Summary

Keep open. Current main has not implemented the PR’s central behavior: the install manifest scan still rejects every `node_modules` symlink whose real target resolves outside the plugin install root, with no trusted exception for `node_modules/openclaw` or `node_modules/.bin/openclaw`. The PR is a focused, plausible fix path with review-thread concerns already addressed in the provided PR context; the related symlink-discovery PRs track different plugin discovery/install-root behavior, not this install security-scan peer-link case.

## Best Possible Solution

Keep PR #70819 open or land a cleaned version of its scanner change: resolve the trusted host OpenClaw package root once per manifest traversal, allow only exact `node_modules/openclaw` and `node_modules/.bin/openclaw` peer-link shapes whose real target is inside that host root, and keep all arbitrary, nested, spoofed, or non-exact escaping `node_modules` symlinks blocked with regression tests in `src/plugins/install.test.ts`. If maintainers do not want the bundled CI-stability tweaks in the same PR, split those into a separate infra PR rather than closing the plugin scanner fix.
