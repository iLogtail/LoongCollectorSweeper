## Summary

Current main has an additive `alsoAllow` path for tools outside a selected profile, but the reported accepted configuration still appears to remain profile-filtered: `tools.profile: "coding"` is applied before later agent `tools.allow`, and `coding` does not include browser/UI or arbitrary plugin tools such as Gmail. This needs maintainer judgment rather than automatic closure.

## Best Possible Solution

Keep this open for maintainer judgment. The likely product path is either to document/diagnose this exact misconfiguration more clearly and direct users to `agents.list[].tools.alsoAllow` for browser/Gmail, or to intentionally change policy semantics with tests if maintainers want agent `tools.allow` to widen a global profile.

## Evidence

- **Issue reproduction:** The issue reports `tools.profile: "coding"` plus `agents.list[0].tools.allow` for browser/Gmail yielded `systemPromptReport.tools.entries: []`, while removing the top-level profile restored `gmail_search`, `gmail_read`, `gmail_send`, and `browser`.
