## Summary

Keep open. Current `main` does not implement a post-update plugin lifecycle hook like `plugin_updated`; the typed hook surface only includes `before_install`, and the plugin update path records install metadata/outcomes without dispatching any post-update hook.

## Evidence

- **Typed hook names do not include `plugin_updated`:** `PluginHookName`/`PLUGIN_HOOK_NAMES` end with `before_install`; there is no `plugin_updated` or equivalent post-update hook in the public typed hook list.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/hook-types.ts#L57)

- **Hook handler map only defines `before_install` for install/update policy:** The typed handler map exposes `before_install(event, ctx)` and no post-update/update-complete hook.
  - file: [src/plugins/hook-types.ts:766](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/hook-types.ts#L766)

  - file: [docs/plugins/sdk-overview.md:232](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/plugins/sdk-overview.md#L232)
