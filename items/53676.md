## Summary

Keep open. Current `main` has already addressed the PR review comment about per-account auth rejection logging, but the two substantive behavior changes in the PR are still not present: app-url/add-on auth still depends only on `email`/`email_verified` and ignores `iss`, and sender personalization still does not fall back to `event.user.email` when `message.sender.email` is absent.

## Evidence

- **App-url auth on main still rejects no-email JWT variants:** `verifyGoogleChatRequest` on `main` lowercases only `payload.email`, immediately fails when `email_verified` is falsey, and never reads `payload.iss`. That means the PR's stated fix for Chat/add-on JWTs that omit `email` is not implemented on current `main`.
