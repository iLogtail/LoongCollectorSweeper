## Summary

Keep PR #53676 open. Current main has implemented and improved the diagnostics portion of the PR, including per-target auth rejection logging and appPrincipal warnings, but it has not implemented the PR’s issuer-based app-url/add-on token matching or the sender email fallback when `message.sender` exists without an email. Because meaningful requested behavior remains unmerged and the PR is a focused Google Chat bug-fix candidate, it should not be auto-closed as implemented or superseded.

## Best Possible Solution

Keep the PR open for maintainer review or author update. The best path is to rebase/adapt only the still-missing pieces: validate the real Google Chat/add-on token claim variants, add focused tests for `iss`-matched app-url/add-on tokens if the contract is confirmed, and separately apply the low-risk `event.user.email` fallback without regressing current per-target diagnostics.
