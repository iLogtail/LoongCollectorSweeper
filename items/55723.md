## Summary

Keep open. Current main still has the exact ACP gap this PR targets: the requester override is accepted by the tool layer and used by the subagent path, but it is not part of SpawnAcpContext, is not passed into spawnAcpDirect, and ACP requester heartbeat/relay eligibility still falls back to the parsed parent session key.

## Best Possible Solution

Rebase and finish the PR on current main: add requesterAgentIdOverride to SpawnAcpContext, pass it from createSessionsSpawnTool into spawnAcpDirect, use override-first requester resolution for ACP heartbeat and parent-stream requester state, and keep the proposed regression coverage for tool passthrough and implicit parent-stream eligibility.
