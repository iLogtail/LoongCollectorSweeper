## Summary

Keep #53447 open. Current main has a generic parent-session fallback for channel model overrides, but Discord only supplies the native thread parent as `ParentSessionKey` when `channels.discord.thread.inheritParent` is enabled. With the default `inheritParent: false`, a Discord thread without its own `modelByChannel` entry still resolves from the thread id/name path rather than the parent channel id requested by this issue.

## Best Possible Solution

Keep the issue open and implement a narrow Discord routing fix: pass the resolved native thread parent channel id into `resolveChannelModelOverride` independently of `channels.discord.thread.inheritParent`, preserving explicit thread overrides first and transcript inheritance behavior unchanged. Add regression coverage for a Discord thread whose own id has no `channels.modelByChannel.discord` entry but whose parent channel id does, plus a case where the thread id override wins.

## Evidence

That means the parent channel id is not generally passed downstream for model routing.
