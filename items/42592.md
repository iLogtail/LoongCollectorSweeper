## Summary

Keep open. Current main has partial installer hardening, but it does not fully implement the requested structured npm error classification or retryable network-error policy, and the related best-effort config probes still use `|| true`.

## Best Possible Solution

Keep the issue open and narrow the implementation to the installer surfaces: add a reusable npm error classifier with remediation hints and retryability decisions in `scripts/install.sh`, add Windows parity in `scripts/install.ps1`, preserve intentional best-effort probes with explicit warnings where appropriate, and cover the behavior with focused installer tests under `test/scripts/install-sh.test.ts` plus PowerShell-facing coverage if available.
