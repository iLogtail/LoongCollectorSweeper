## Summary

Keep open. Current `main` already includes the `tools.web.fetch.maxResponseBytes` schema/docs/test coverage and the benign dual-stack loopback port classification, but the Synology webhook behavior this PR proposes is still absent on `main`: `extensions/synology-chat/src/webhook-handler.ts` only accepts `POST`, returns `405` for `HEAD`, and still ACKs successful requests with `204`/empty body. That means the PR is only partially superseded, not implemented well enough to close.
