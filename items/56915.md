## Summary

Issue #56915 still describes a real current-main bug. `openclaw skills list` flows through the shared skill status loader, and that loader still discovers only root-level skills or immediate child directories under each configured skill root. The discussion workaround from Syu0, moving the skill up or pointing `skills.load.extraDirs` at the parent directory, matches current behavior but does not implement the requested `~/.openclaw/skills/coze/koze-retrieval/SKILL.md` layout. PR #69663 appears to track the exact fix, but it is open and unmerged, while earlier linked PRs were closed unmerged.

## Best Possible Solution

Keep #56915 open. The best fix is a conservative change in `src/agents/skills/workspace.ts` that discovers a bounded grouped layout under configured skill roots, with a regression test for `~/.openclaw/skills/coze/koze-retrieval/SKILL.md`. The implementation should preserve existing realpath containment checks, hidden-directory/node_modules skips, and candidate/loaded-skill limits; PR #69663 is the likely canonical place to review that work.
