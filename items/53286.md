## Summary

Keep #53286 open. Current main still suppresses estimated cost on `/status` and response usage footers when the active provider auth is `aws-sdk`, even when the model has explicit nonzero `cost` metadata. PR #53303 directly targets this issue, but it is still open/unmerged in the provided GitHub context, so the fix has not shipped on main.

## Best Possible Solution

Keep the issue open until main decouples cost display from auth mode. The likely fix belongs in `src/status/status-message.ts` and `src/auto-reply/reply/agent-runner.ts`: resolve configured model cost for all auth modes and show estimated cost when explicit nonzero cost metadata exists, with focused regression tests for `aws-sdk` Bedrock in `/status` and response usage footer behavior. PR #53303 is the related implementation to review or replace with a smaller current-main patch.
