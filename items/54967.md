## Summary

Keep open. Current main still resolves Feishu media only from the triggering message, while mention-gated pending group history is replayed as text context and is not converted into MediaPaths. The PR remains a focused fix for a real remaining Feishu channel gap; related PR #66432 is adjacent thread-history work, not a full duplicate of pending pre-mention history media.

## Best Possible Solution

Keep #54967 open. The best path is to update the PR against current main, preserve raw Feishu content for fetched pending-history messages, resolve a bounded number of pending media items before building the inbound media payload, add a regression test for screenshot-before-mention group flow, and coordinate but not collapse it into #66432 because that PR targets fetched topic thread history.
