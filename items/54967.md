## Summary

Keep open. Current `main` does not implement pending-history media hydration for Feishu group mentions: pending history is stored as text-only entries, inbound media is resolved only from the triggering message, and fetched Feishu messages still discard raw payloads needed to re-resolve prior media messages.

## Evidence

- **Pending group history stores only text plus messageId:** When a mention is required and missing, Feishu group messages are buffered as `body: `${ctx.senderName ?? ctx.senderOpenId}: ${ctx.content}`` with `messageId`, but no media payload or raw Feishu content is preserved in history.
  - file: [extensions/feishu/src/bot.ts:480](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/feishu/src/bot.ts#L480)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '470,490p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Inbound media comes only from the triggering message:** Current inbound handling resolves media only from `ctx.messageId` / `event.message.content` for the current event, builds one `mediaPayload`, and passes only that payload into `finalizeInboundContext`. There is no pass over pending history entries to fetch older media.
  - file: [extensions/feishu/src/bot.ts:712](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/feishu/src/bot.ts#L712)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '712,723p'; nl -ba extensions/feishu/src/bot.ts | sed -n '983,1016p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Non-text Feishu messages are normalized to raw JSON strings in history/body flow:** `parseMessageContent` returns the original `content` string for message types like `image`, `file`, `audio`, `video`, and `sticker`, so pending-history entries for those messages are raw JSON bodies rather than resolved media attachments.
  - file: [extensions/feishu/src/bot-content.ts:125](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/feishu/src/bot-content.ts#L125)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '125,156p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Fetched Feishu messages still drop raw API content:** `parseFeishuMessageItem` reads `rawContent = item.body?.content ?? ""` but only returns normalized `content`; the raw payload is not exposed on `FeishuMessageInfo`, which blocks reusing fetched history messages to resolve prior media on current `main`.
