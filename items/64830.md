## Summary

Keep #64830 open. Current main still does not fully satisfy the requested Web UI default behavior: session list defaults are model-derived instead of config-effective, chat history only carries effective thinking state, reasoning visibility still keys off persisted session rows, and the related PR #66098 is open and explicitly scoped as a partial fix that does not address config.patch or new config keys.

## Best Possible Solution

Keep the issue open and either land a complete fix or use it as the canonical tracker. The best implementation path is to expose effective chat defaults across the Gateway boundary, update Web UI state/rendering to use effective thinking and reasoning for fresh sessions, add regression tests in the gateway and UI chat/session-control paths, and resolve the product/schema mismatch for agents.defaults.reasoningDefault by either adding the supported global key with docs or removing it from tool/config surfaces that advertise it.
