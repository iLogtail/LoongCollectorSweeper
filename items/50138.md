## Summary

Keep open. Current main still lacks Mattermost WebSocket transport liveness detection for silent half-open connections. The generic health policy only restarts stale sockets when a connected account reports lastTransportActivityAt, while Mattermost's WebSocket path only handles open/message/close/error and bot update_at polling; it does not publish transport activity or use ping/pong keepalive. The related fix PR #57621 is still open/unmerged in the provided context, so the original bug remains actionable.

## Best Possible Solution

Keep #50138 open as the canonical Mattermost silent WebSocket disconnect bug until an extension-owned fix lands. The best path is to finish/review #57621 or a successor in extensions/mattermost/src/mattermost/monitor-websocket.ts: add WebSocket ping/pong or equivalent transport liveness, publish lastTransportActivityAt where appropriate, terminate half-open sockets so runWithReconnect restarts, and cover it with monitor-websocket tests plus any needed health-policy/status assertions.
