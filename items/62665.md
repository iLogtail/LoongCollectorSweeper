## Summary

Keep open. Current main still contains the exact logging gaps this PR targets: config health writes and clobbered snapshot persistence still swallow failures, and backup restore still logs success even if copyFile fails. The #71540 cross-reference is a broader config auto-restore bug, not a shipped replacement for this focused diagnostics PR.

## Best Possible Solution

Keep the PR open and ask for maintainer review or small follow-up tests. The best implementation is to keep the proposed diagnostic logging behavior, make the backup restore warning accurately reflect restoredFromBackup, and add targeted tests for restore failure and best-effort persistence failure logging without changing recovery control flow.
