## Summary

Keep #56145 open. Current main includes the adjacent directive-tag indentation fix, but it does not implement this PR's gateway non-streaming recombination fix. The remaining `/btw` path still uses per-payload `.trim()` plus `join("\n\n")`, and the non-streaming final reply path still sanitizes each payload independently, which can remove leading indentation when formatted text is split across payload boundaries.

## Best Possible Solution

Keep the PR open and have a maintainer review or rebase it into current main. The best fix is still a shared gateway helper for non-streaming reply-part recombination that skips whitespace-only chunks, preserves authored indentation and newline boundaries, handles CRLF/CR/LF outer boundaries, and is used by both final replies and `/btw` side results with focused regression coverage.
