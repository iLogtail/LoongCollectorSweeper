## Summary

Keep PR #62528 open. Current main still does not implement the proposed generic `POST /hooks/message` ingress, and the PR remains the focused implementation for open issue #62526 rather than something already shipped or superseded. The supplied review discussion also contains post-head compatibility/error-handling concerns that need author or maintainer handling before merge, not automated closure.

## Best Possible Solution

Keep the PR open and ask the author/maintainer to resolve the remaining review concerns, especially preserving existing mapped-hook behavior for `match.path: "message"`, preserving structured handler error statuses, and proving the new `/hooks/message` path with focused gateway/cron tests. If maintainers still want this capability, it belongs in the gateway hook pipeline with docs for `https://docs.openclaw.ai/gateway/configuration-reference#hooks` and `https://docs.openclaw.ai/automation/cron-jobs`.
