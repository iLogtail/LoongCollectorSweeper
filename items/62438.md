## Summary

Keep #62438 open. Current main still runs the secrets-in-config audit check against the resolved runtime config, so authored `${VAR}` sentinels are gone before `looksLikeEnvRef()` can suppress `config.secrets.gateway_password_in_config` or `config.secrets.hooks_token_in_config`. The related #67171 write-path discussion does not supersede this audit bug.

## Best Possible Solution

Keep the issue open and fix the audit path directly: teach `collectSecretsInConfigFindings` to compare the resolved runtime value with the authored/raw config value for `gateway.auth.password` and `hooks.token`, or otherwise pass an audit-specific snapshot that preserves `${VAR}` sentinels. Add regression tests covering both fields with source `${VAR}` plus resolved secret values, and keep the broader secrets-audit issues (#53998/#61058) separate unless a single implementation intentionally covers them.
