## Summary

Keep open. Current main still matches the reported bug: `openclaw configure channels` removes only the channel config and explicitly leaves credentials/sessions on disk, while startup auto-enable still treats persisted WhatsApp auth as configured state and can write `channels.whatsapp.enabled = true` back into config. The related PR #52672 was closed unmerged, so it does not supersede the issue.
