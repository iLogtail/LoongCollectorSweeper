## Summary

Current main now resolves the reported exec PATH problem by probing the user's trusted login shell for PATH and merging that PATH into gateway-host exec runs. The behavior is covered by tests and documented for users.

## Evidence

## Risks / Open Questions

- This imports and merges login-shell PATH; it does not source arbitrary interactive shell rc code for every command. Users whose version-manager setup only exists in an interactive-only rc file may still need to expose those PATH entries through the login shell or tools.exec.pathPrepend.
