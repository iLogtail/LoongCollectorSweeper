## Summary

Keep #66953 open. Current main still writes delivery-mirror transcript records with the current/effective request agent rather than deriving the transcript-owning agent from the outbound route/session key, and existing gateway coverage even preserves a mismatched explicit-agent/session-key precedence case.

## Best Possible Solution

Keep #66953 open. Add a focused regression for a multi-account writer route with session.dmScope='per-account-channel-peer' where the current/default request agent differs from the outboundRoute.sessionKey agent, then update the mirror contract so transcript ownership follows the outbound route/session key or ensure all callers propagate the writer agent before route construction. Adjust the gateway precedence test to encode the chosen contract.
