## Summary

Keep #30518 open. Current main still supports generic pin/unpin action names and Telegram delivery-time pinning, and it has low-level Telegram Bot API pin/unpin helpers, but the bundled Telegram message action adapter does not advertise or dispatch manual `pin` / `unpin` actions.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding Telegram `pin` and `unpin` to action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases/branches that call the existing `pinMessageTelegram` and `unpinMessageTelegram` helpers, adding focused adapter/runtime contract tests, and updating the CLI plus Telegram docs to list manual Telegram pin/unpin separately from send-time `--pin`.
