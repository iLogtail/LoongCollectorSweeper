## Summary

Keep PR #39520 open. Current main still lacks the requested Feishu `channels.feishu.localRoots` and account-scoped `localRoots` configuration, and `sendMediaFeishu` still only forwards caller-provided `mediaLocalRoots` arrays to `loadWebMedia`. The linked #39506 user problem is not implemented on main and is not superseded by the separate context-root forwarding work.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent Feishu bundled plugin change. The best implementation is to add `localRoots: "any" | string[]` to the shared Feishu top-level/account config schema, resolve it from the merged runtime account so account overrides work, define empty-array behavior explicitly, inject raw `readFile` only for explicit `"any"` or adapt to the current outbound media helper contract, and update Feishu docs plus tests for top-level, account override, `"any"`, empty array, and no `readFile` on array roots.
