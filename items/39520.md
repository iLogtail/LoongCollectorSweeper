## Summary

Keep PR #39520 open. Current main still only forwards core-provided `mediaLocalRoots` into Feishu media upload; it does not define or read `channels.feishu.localRoots`, does not support the requested `"any"` Feishu config override, and has no Feishu tests/docs for account-scoped local root overrides. The linked bug #39506 and closed #27884 do not supersede this PR because #27884 shipped the narrower core-root propagation behavior, not this channel/account config surface.

## Best Possible Solution

Keep the PR open for maintainer review or implementation. The best path is to land an extension-scoped Feishu `localRoots: "any" | string[]` config that resolves through the merged account config, honors empty arrays intentionally, injects raw `readFile` only for the explicit `"any"` bypass, and adds docs/tests; alternatively, a maintainer should explicitly decide that Feishu local media must only use core `mediaLocalRoots` and close with that product rationale.
