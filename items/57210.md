## Summary

Keep open. Current `main` has session-management primitives and a Sessions index, but the Control UI still wires its visible `New session` action to `/new`, which rotates the current `sessionKey` instead of creating a separate independent chat/history. That means the issue's requested multi-session/new-chat workflow is only partially implemented, not closed out end-to-end.

## Evidence

- **Chat UI New session still sends `/new`:** The Control UI chat toolbar labels the action as `New session`, but the handler calls `state.handleSendChat("/new", { restoreDraft: true })` rather than `sessions.create` or any session-creation flow.
  - file: [ui/src/ui/app-render.ts:2338](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/ui/src/ui/app-render.ts#L2338)
