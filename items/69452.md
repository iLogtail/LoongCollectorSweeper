## Summary

PR #69452 should stay open. Current main does not already contain an Anthropic-compatible proxy recipe for the bundled Anthropic provider, and the underlying `baseUrl` behavior is present in source. The PR is still useful docs work, but it needs revision before merge because the provided review context identifies unresolved issues: promotional aiusage wording from an affiliated author, a redundant `api` field in the bundled-provider example, and an auth-mode caveat that conflicts with current OAuth-token transport behavior.

## Best Possible Solution

Keep the PR open and ask for a neutral docs revision: document the generic Anthropic-compatible proxy pattern for `models.providers.anthropic.baseUrl`, remove promotional aiusage cost language, remove or justify the redundant bundled-provider `api` field, and correct the OAuth/API-key caveat against `src/agents/anthropic-transport-stream.ts`. Leave the broader provider API/baseUrl matrix tracked by #61984/#61993.
