## Summary

Keep #39232 open. Current main still has no typed `on_raw_inbound` / `before_channel_filter` plugin hook, and the existing inbound/message hooks are post-filter or post-routing surfaces. WhatsApp now has an explicit `message_received` opt-in, but it still fires from `processMessage` after access control and group mention gating, so it does not solve passive observation of all raw inbound messages.

## Best Possible Solution

Keep the issue open for maintainer/product design. A good fix would add an observe-only, privacy-aware pre-filter inbound channel/plugin contract, expose it through typed SDK docs, wire bundled channels such as WhatsApp and Telegram before access and mention gating, and cover it with contract plus channel tests. If maintainers decide raw ambient observation should not be supported, document that decision and any narrower supported alternative.

## Risks / Open Questions

- A raw pre-filter hook would expose content that access control and mention gating currently suppress from plugins, so any implementation needs explicit opt-in, privacy documentation, and security review.
- The product shape is still a maintainer decision: a global typed hook, per-channel observe config, or a documented decision not to support ambient raw observation.
