## Summary

Current main still does not implement the requested native subagent session reuse for Teams. Native `sessions_spawn` creates a fresh `agent:<agentId>:subagent:<uuid>` on each spawn, the tool schema has no `sessionTarget`/`reuse` option, and `resumeSessionId` is explicitly ACP-only. Persistent native session mode still depends on `thread=true` plus a `subagent_spawning` hook; Teams declares thread capability and has inbound thread session isolation, but its bundled entry does not register the subagent hooks that Discord and Feishu register. The KeepMyClaw comment is a workaround/backstop discussion and even frames registry reuse as the long-term fix, not evidence that OpenClaw main has shipped it.

## Best Possible Solution

Keep the issue open. Maintainers should either add a Teams-owned `subagent_spawning`/delivery-target implementation that binds Teams thread conversations to persistent subagent sessions, or design the broader core native subagent reuse path (`sessionTarget: "reuse"` or equivalent) with TTL and tests/docs. The KeepMyClaw backup comment can remain as operational context, but it is not a shipped OpenClaw fix.

## Risks / Open Questions

- A core registry-reuse implementation would need clear TTL, requester/session isolation, sandbox, model/auth override, and allowlist semantics so one team's expert context is not silently reused from the wrong requester or channel.
- Adding only a Teams `subagent_spawning` hook may solve thread-bound Teams follow-ups but would not satisfy the issue's preferred hook-independent registry reuse for `mode="run"`.
