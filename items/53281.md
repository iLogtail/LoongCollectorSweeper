## Summary

Current `main` does not implement this PR’s caller-facing clarification. `attachAs.mountPath` is still forwarded into subagent attachment handling as a child prompt hint, but accepted spawn results still use only the generic accepted-note helper, and the current docs/tests on `main` do not cover the proposed `mountPath` note contract.

## Evidence

- **`sessions_spawn` still forwards `attachAs.mountPath`:** The tool reads `params.attachAs.mountPath` and passes it through as `attachMountPath` to `spawnSubagentDirect`, so the surface this PR targets still exists on `main`.
