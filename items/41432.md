## Summary

Current `main` already handles real conversation messages in both `messagesToSummarize` and `turnPrefixMessages`, and it includes a regression test for the split-turn case described in the issue. The same logic is present in release `v2026.4.22`, so the issue is already shipped.

## Close Comment

Closing as implemented on `main` after Codex review. The current compaction safeguard already checks both `preparation.messagesToSummarize` and `preparation.turnPrefixMessages` before deciding there is no real conversation content to summarize (`src/agents/pi-hooks/compaction-safeguard.ts:727`). There is also a regression test for the split-turn case where only `turnPrefixMessages` has real content (`src/agents/pi-hooks/compaction-safeguard.test.ts:1875`). I also verified the same logic is present in the shipped `v2026.4.22` release tag, so this is already fixed in released OpenClaw builds.
