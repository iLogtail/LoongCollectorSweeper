## Summary

Keep open. Current `main` still appears to ignore `cacheRetention` for the bundled LiteLLM provider’s real `openai-completions` transport. The new prompt-cache retention code only treats non-Bedrock custom providers as Anthropic-cache-eligible when `modelApi === "anthropic-messages"`, while bundled LiteLLM is still declared as `api: "openai-completions"`. The only payload-level Anthropic `cache_control` wrapper on this path is OpenRouter-specific, so there is no corresponding LiteLLM wrapper on current `main`.
