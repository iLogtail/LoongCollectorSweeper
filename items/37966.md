## Summary

Keep issue #37966 open. Current main still does not implement the reported LiteLLM OpenAI-compatible payload behavior: LiteLLM is configured as `openai-completions`, `resolveCacheRetention` only honors explicit non-Bedrock cache config for `anthropic-messages`, and the only Anthropic `cache_control` payload wrapper is scoped to verified OpenRouter routes. The related PR #38221 was closed unmerged, and the tests now present cover LiteLLM with `anthropic-messages`, not the documented/default LiteLLM OpenAI-compatible path in the bug report.
