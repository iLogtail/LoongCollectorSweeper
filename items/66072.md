## Summary

Keep open. Current `main` still does not implement the `models.mode: "replace"` UI/catalog filtering described in the PR, and the proposed PR logic appears incomplete: it filters only against raw `cfg.models.providers` keys, while current runtime/provider behavior on `main` includes implicit and aliased paired providers (for example `byteplus-plan` and `volcengine-plan`). That makes the PR unsafe to auto-close as already-implemented and too risky to treat as conservatively resolved.

## Evidence

There is no `cfg.models.mode === "replace"` branch on current `main`.
