## Summary

Current main still has the reported bundled 1Password skill bug: the skill requires tmux for all `op` commands and explicitly forbids direct `op` calls, even though its own 1Password reference describes desktop app integration and direct `op vault list` sign-in. This is not superseded by the related closed launchd/config issues, and it belongs in this repo because OpenClaw ships and loads bundled skills from `skills/`.

## Best Possible Solution

Keep #52540 open and update `skills/1password/SKILL.md` to be environment-aware: prefer direct exec for macOS 1Password desktop app integration/Touch ID, keep tmux only for auth modes where it is actually needed, and align any release/operator docs that still repeat blanket tmux-only `op` guidance. A small text regression around the bundled skill wording would be useful if the repo has an existing skill-guidance test pattern.
