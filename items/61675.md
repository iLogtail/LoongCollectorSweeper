## Summary

Keep PR #61675 open. Current main has session_start/session_end plugin lifecycle coverage for stale daily/idle rollovers, but it still does not implement this PR’s central behavior: firing internal command hooks and before_reset for lazy daily/idle resets so bundled session-memory can flush. The bundled hook and docs still list only command:new/command:reset, and the PR’s added stale-hook regression test is not present on main.

## Best Possible Solution

Keep the PR open and have it rebased or reimplemented against current main. The desired fix belongs in core session rollover handling: emit internal command hooks and before_reset for lazy daily/idle resets, widen bundled session-memory metadata/handler/docs to accept those actions, and keep focused tests proving daily/idle dispatch plus no duplicate manual-reset dispatch.
