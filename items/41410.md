## Summary

Keep #41410 open. Current main does not implement Telegram forum topic deletion through the shared message action surface: `delete` still maps only to `deleteMessage`, `topic-delete` is absent from discovery/public action names/docs, and the Telegram transport has create/edit forum topic helpers but no delete helper. The related #11414 is an open umbrella for broader forum topic management, not a merged replacement for this concrete PR. The PR still needs maintainer review and likely cleanup of the late destructive-ID parsing feedback, but it is useful, targeted work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased, narrow Telegram plugin change that adds `deleteForumTopicTelegram`, exposes `deleteForumTopic`/`topic-delete` through the existing message action contract, gates it under `channels.telegram.actions.deleteMessage`, requires explicit decimal-only `topicId`/`threadId` for destructive deletes, and updates docs plus focused action/transport tests. Use #11414 as the umbrella for the remaining close/reopen topic-management actions.
