## Summary

Keep PR #65770 open. Current main does not already implement the PR’s core behavior: subagent announce delivery still tracks only session/source-tool provenance, `SubagentRunRecord` has no persisted `pendingAnnounceId` or `lastAnnounceDeliveredId` fields, restore does not clear stale pending announce state, and the matching regression tests are absent. The PR discussion is also still relevant because Greptile identified a concrete redundant disk-write issue to address before merge.

## Best Possible Solution

Keep the PR open, address the existing Greptile P2 by making restore cleanup persistence single-owner, then rebase/update the patch against current main and run the scoped agents tests plus the repo’s changed/typecheck lanes. Close only if an equivalent source-run announce state implementation lands elsewhere or a canonical superseding PR/issue is linked.
