## Summary

Keep open. Current main still matches the reported root cause: `runEmbeddedAttempt` stores only one `clientToolCallDetected`, `runEmbeddedPiAgent` wraps that single value into a one-item `pendingToolCalls` array, and `/v1/responses` reads only `pendingToolCalls[0]` in both non-streaming and streaming paths. The linked PR #52287 was closed unmerged in the provided GitHub context, and the fix is not present in the checked-out main tree.
