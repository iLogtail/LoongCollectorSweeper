## Summary

Keep open. Current main covers or mitigates several sub-points, but the issue's high-priority token-efficiency asks are still not fully implemented: workspace bootstrap loading is controlled by global modes and character caps, not per-file load conditions, and compaction summaries are still persisted context that future turns see rather than lazy/collapsible title-only entries.

## Best Possible Solution

Keep the issue open for maintainer triage or split it into narrower follow-ups: one for per-file conditional/lazy workspace bootstrap loading and one for collapsible or demand-expanded compaction/LCM summaries. Already-shipped sub-points such as heartbeat task intervals, cron retry/alerts, config schema output, and subagent tool policy can be checked off in a maintainer comment or moved to a checklist.
