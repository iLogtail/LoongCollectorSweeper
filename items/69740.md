## Summary

Keep open. Current main does not implement the proposed centralized production guard for debug env vars: the helper file is absent and the same OPENCLAW_DEBUG_* flags are still read directly across agents, auto-reply, Telegram, health, and memory host code. The PR is also not merge-ready as-is because review discussion identified an incomplete migration and unrelated sessions/install-smoke changes, but the central hardening request is not already implemented or superseded.

## Best Possible Solution

Keep the PR open, but require a focused rewrite: either split out the unrelated sessions/install-smoke changes or drop them, place any shared debug-env helper in an appropriate shared module rather than an agents-only module, migrate every intended OPENCLAW_DEBUG_* call site with tests for production-disabled and non-production-enabled behavior, and document the supported debug flags if maintainers agree that NODE_ENV=production should suppress them.
