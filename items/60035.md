## Summary

Keep #60035 open. Current main has useful partial proxy support for model-provider requests, selected channels, and env-based HTTP proxy dispatchers, but it does not implement the requested broad first-class system/custom HTTP and SOCKS proxy configuration across OpenClaw. There is still no top-level proxy config, no per-agent proxy override, web_fetch intentionally avoids env proxying for untrusted URLs, browser automation explicitly does not inherit gateway/provider proxy env vars, and provider proxy config is documented as HTTP-only.

## Best Possible Solution

Keep the issue open for a maintainer-scoped core networking design. The best path is to define a coherent proxy contract covering top-level env/custom proxy policy, per-provider/tool/channel/browser behavior, SOCKS support or documented non-support, NO_PROXY/ALL_PROXY semantics, and tests/docs that preserve SSRF protections while enabling restricted-network deployments.
