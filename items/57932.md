## Summary

Keep open. Current `main` still has the sandbox-off early return in `stageSandboxMedia`, so inbound media is not copied into the workspace when `tools.fs.workspaceOnly` is enabled and no sandbox/remote host is active. The current vision loader still restricts local file access to `workspaceDir` in that mode, so the PR addresses a real gap rather than an already-landed fix.

## Evidence

- **Current main still returns early with sandbox off:** `stageSandboxMedia()` only uses `sandbox?.workspaceDir ?? remoteMediaCacheDir`; when both are absent it returns without rewriting `ctx.MediaPath`, so local inbound media stays outside the workspace.
