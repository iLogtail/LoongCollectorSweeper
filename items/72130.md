## Summary

Keep #72130 open. The issue is fresh and describes unbounded native runtime checkpoint storage, while current main only proves OpenClaw session-store/transcript maintenance. The bundled Codex app-server harness still opts into extended native history and exposes no checkpoint retention, size limit, or server/VPS default policy in its strict config surface.

## Best Possible Solution

Keep this open and handle it as a Codex/Hermes runtime storage bug: reproduce the VPS growth on current main, identify the native checkpoint directory and trigger, then either pass supported app-server controls to disable/bound checkpoints or add/upstream a protocol/config knob that OpenClaw can expose under `plugins.entries.codex.config.appServer`. Document the recommended VPS/server retention setting once implemented.
