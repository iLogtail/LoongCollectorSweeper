## Summary

Keep open. Current `main` does not already implement this PR, and the review-thread defects are real against current code: the added `skippedPayload` metadata would be dropped by `logVerbose(message: string)`, and widening the visibility bypass to all `tool` stream events would conflict with the existing hidden-run contract and likely surface intermediate tool traffic.

## Evidence

- **`logVerbose` only accepts one string argument:** `src/globals.ts` defines `logVerbose(message: string)`. That matches the PR review concern that adding a second `{ skippedPayload: ... }` argument in `src/auto-reply/reply/block-reply-pipeline.ts` would be ignored at runtime, so that part of the PR is not a real fix.
  - file: [src/globals.ts:10](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/globals.ts#L10)
  - command: `nl -ba src/globals.ts | sed -n '1,25p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Hidden runs are explicitly supposed to omit `sessionKey`:** `src/infra/agent-events.test.ts` has a direct test that runs with `isControlUiVisible: false` must emit no `sessionKey`. The PR's `event.stream === "tool"` critical-event shortcut would bypass that contract for all tool events, not just terminal ones.
  - file: [src/infra/agent-events.test.ts:78](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/infra/agent-events.test.ts#L78)
  - command: `nl -ba src/infra/agent-events.test.ts | sed -n '78,98p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Tool events are broadly fanned out whenever a `sessionKey` exists:** `src/gateway/server-chat.ts` broadcasts tool events to run-scoped recipients and mirrors them onto session subscribers whenever `sessionKey` is present. If hidden runs started carrying `sessionKey` for every `tool` stream event, start/update traffic would become visible to those subscribers, which matches the regression called out in review.
  - file: [src/gateway/server-chat.ts:930](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/server-chat.ts#L930)
