## Summary

本 Issue 详细报告了 flusher_otlp 在成功接收 trace 后无法向 OTLP/gRPC 后端转发且无错误日志的问题，包含完整的上下游链路交叉验证、Docker Compose 配置、插件参数与复现命令。该问题直指核心输出插件的端点解析逻辑、gRPC 拨号错误处理及日志反馈机制，属于有效的功能疑问与潜在缺陷，当前证据不足以支持关闭。

## Best Possible Solution

建议维护者核查 plugins/flusher/otlp 中 Traces.Endpoint 的协议前缀处理逻辑（特别是 http:// 前缀在 gRPC 模式下的兼容性）及底层 gRPC 客户端的初始化/发送错误捕获路径。若当前实现存在静默丢弃或日志缺失，需补充 Warning/Error 日志并增强配置强校验；同时更新插件文档明确 OTLP/gRPC 与 OTLP/HTTP 的 Endpoint 标准写法。可邀请用户尝试移除 http:// 前缀、启用 gRPC 底层调试日志或升级至 v3.3.5 验证。待配置规范明确、日志增强或潜在缺陷修复后再行关闭。
