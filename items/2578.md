## Best Possible Solution

建议维护者或社区核查 flusher_otlp 的 Endpoint 解析与 Protocol 默认回退逻辑。OTLP 输出通常需根据 URL Scheme 或显式 Protocol 字段区分 HTTP 与 gRPC。若默认使用 HTTP 客户端连接 4317(gRPC) 端口会导致协议不匹配。请提供针对 gRPC 后端的正确配置示例（如移除 http:// 前缀或添加 Protocol: grpc），并建议用户开启插件 Debug 日志或使用 grpcurl 验证连通性。待配置澄清或确认为代码缺陷后跟进处理。
