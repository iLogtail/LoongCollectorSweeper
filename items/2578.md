## Summary

本 Issue 详细报告了 flusher_otlp 无法向 OTLP/gRPC 后端转发 trace 的问题。作者已提供完整的四组对照测试、Docker Compose 配置与下游服务验证，成功将问题隔离至 LoongCollector flusher_otlp 组件的 gRPC 转发或 Endpoint 解析环节。属于信息充分的有效配置疑问与潜在缺陷排查请求，需维护者介入核实协议自适应逻辑与连接初始化行为。

## Best Possible Solution

建议维护者核查 flusher_otlp 源码中 Traces.Endpoint 的解析逻辑，确认 gRPC 模式下是否兼容 http:// 前缀或需显式指定协议/传输字段。可邀请用户尝试开启底层 gRPC 调试环境变量（如 GRPC_GO_LOG_VERBOSITY_LEVEL=9 与 GRPC_GO_LOG_SEVERITY_LEVEL=info）以获取连接静默失败的具体原因。若确认为协议推断缺陷、静默失败或初始化日志缺失，应在主仓库修复并补充 Warning 日志与配置示例；若为配置方式问题，请提供正确范例并更新文档。
