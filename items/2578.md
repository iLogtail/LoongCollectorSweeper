## Summary

本 Issue 为有效的配置咨询与插件行为澄清请求，详细提供了复现链路与交叉验证。当前现象高度疑似 flusher_otlp 的 gRPC 客户端默认启用 TLS，而下游 OTel Collector 4317 端口为明文通信，导致握手失败且日志未明确报错。用户配置中未显式声明 Insecure 或 TLS 跳过参数，需维护者或社区确认该插件正确的明文 gRPC 配置字段及默认安全策略。

## Best Possible Solution

建议维护者协助确认 flusher_otlp 插件对 gRPC 明文传输的配置方式（如是否支持 Insecure: true 或 TLS.InsecureSkipVerify），并指导用户更新配置验证。若插件当前对连接失败缺乏有效 Debug 日志，可标记为可观测性改进项。待配置明确或用户验证通过后关闭。
