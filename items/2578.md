## Summary

本 Issue 详细报告了 flusher_otlp 插件在转发 Trace 至 OTLP gRPC 后端时失败的问题。用户提供了完整的复现环境、配置对比与网络验证，明确指向 Endpoint 格式解析或 TLS/Insecure 默认行为可能导致连接静默失败。信息充分且创建不足 60 天，需维护者核查插件 gRPC 客户端逻辑或补充配置文档，不符合任何自动关闭条件。

## Best Possible Solution

建议维护者或熟悉 OTLP 插件的贡献者核查 flusher_otlp 的 gRPC 客户端初始化逻辑：1. 确认 Endpoint 字段对 http:// 前缀的兼容性及 host:port 格式的解析行为；2. 检查是否需显式暴露 Insecure 或 TLS 配置项以支持 gRPC 明文传输；3. 若连接失败时日志静默，建议在 gRPC dial/send 阶段补充 error 级别日志。可邀请用户尝试调整 Endpoint 格式（如移除 http:// 或增加 Insecure: true 配置）或补充插件内部指标/抓包数据。待配置方案明确、文档更新或代码修复后关闭。
