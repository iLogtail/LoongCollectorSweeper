## Summary

本 Issue 详细报告了 flusher_otlp 无法向 OTLP gRPC 后端转发 trace 的问题，提供了完整的交叉验证链路与配置细节。经评估，该问题涉及 gRPC 客户端默认 TLS 行为与明文端点的兼容性配置，以及连接失败时的日志可见性，属于有效的配置咨询与潜在缺陷报告，不符合自动关闭条件。

## Best Possible Solution

建议维护者或贡献者核查 flusher_otlp 插件的 gRPC 客户端初始化逻辑：1. 确认是否需显式配置 Insecure 或 TLS 相关字段以支持明文传输，并在文档中补充 OTLP/gRPC 明文与 TLS 的标准配置示例；2. 检查 Endpoint 解析逻辑是否兼容 http:// 前缀，或明确提示用户去除 scheme；3. 若确认为连接失败时日志静默，请增加 gRPC dial 或 send 阶段的 error 日志输出。待配置方案明确或用户验证通过后关闭。
