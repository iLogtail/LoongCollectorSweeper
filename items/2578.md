## Summary

本 Issue 报告了 flusher_otlp 向 OTLP/gRPC 后端转发 Trace 时静默失败的问题，涉及核心插件的配置解析与错误日志缺失。当前 main 分支代码中 gRPC 拨号逻辑对 http:// 前缀的处理及连接失败时的日志反馈仍需维护者确认或增强，属于有效的功能疑问与潜在缺陷，需保持开启以待官方配置规范明确或代码修复。

## Best Possible Solution

建议维护者核查 plugins/flusher/otlp 中 Traces.Endpoint 的协议前缀处理与 gRPC 拨号逻辑。若 gRPC 模式不支持 http:// 前缀，应在代码中增加强校验与连接失败时的 Warning/Error 日志，避免静默丢弃；同时更新插件文档明确 OTLP/gRPC 与 OTLP/HTTP 的 Endpoint 标准写法。可邀请用户尝试移除 http:// 前缀、开启底层 gRPC 调试日志或升级至最新版本验证。待配置规范明确、日志增强或潜在缺陷修复后关闭。
