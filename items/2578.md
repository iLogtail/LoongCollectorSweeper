## Summary

本 Issue 报告了 flusher_otlp 无法向 OTLP/gRPC 后端转发 trace 的问题，提供了详尽的交叉验证与配置对比。经核查 main 分支 flusher_otlp 源码，协议选择（gRPC vs HTTP）与 Endpoint 解析逻辑可能存在默认行为不明确或静默失败的情况，当前无证据表明已在 main 修复或纯属文档已覆盖的配置失误。属于有效的核心插件功能疑问/潜在缺陷，需维护者介入核查。

## Best Possible Solution

建议维护者核查 flusher_otlp 插件中 Traces 配置的协议默认值与 Endpoint 解析逻辑。若 gRPC 转发需显式配置 Protocol: grpc 或特定 Endpoint 格式（如 grpc:// 或无前缀），请补充文档/示例并直接回复用户；若确认为 gRPC 客户端初始化静默失败或日志缺失缺陷，请安排修复或标记为 bug。可邀请用户升级至 v3.3.5 并提供开启 Debug 级别后的完整插件日志以便进一步定性。
