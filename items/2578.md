## Summary

本 Issue 详细报告了 flusher_otlp 在接收 OTLP/HTTP trace 后无法通过 gRPC 转发至下游后端的问题，提供了完整的交叉验证链路与配置对比。当前无证据表明该行为已在 main 分支修复或纯属文档配置错误，属于核心插件潜在缺陷或配置协议解析歧义，需维护者进一步核查。

## Best Possible Solution

建议维护者核查 flusher_otlp 插件中 Traces 配置的协议默认值与 Endpoint 解析逻辑，确认 gRPC 转发是否需要显式指定协议字段或特定 URL 格式。若确认为连接初始化静默失败或 Debug 日志缺失，请安排修复或补充详细日志；同时可邀请用户升级至 v3.3.5 并提供开启 Debug 级别后的完整插件日志以便进一步定性。
