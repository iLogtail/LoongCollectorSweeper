## Summary

本 Issue 详细报告了 flusher_otlp 在接收 OTLP/HTTP trace 后无法向 OTLP/gRPC 后端转发的问题。作者提供了完整的四组链路对比测试、Docker Compose 配置及下游后端验证，已排除网络与下游服务故障，明确指向 flusher_otlp 的 gRPC 连接初始化或 Endpoint 协议解析逻辑可能存在静默失败或配置规范不清晰的情况。

## Best Possible Solution

建议维护者核查 flusher_otlp 插件对 Traces.Endpoint 的解析逻辑，确认 gRPC 模式下是否兼容 http:// 前缀或需强制去除 scheme；若属配置规范问题，需补充文档说明及初始化阶段的协议不匹配 Warning 日志；可邀请用户开启 GRPC_GO_LOG_VERBOSITY_LEVEL 环境变量获取底层连接详情，或提供最小复现配置以便进一步排查。
