## Summary

本 Issue 详细报告了 flusher_otlp 接收 trace 后无法通过 gRPC 转发至下游的问题，并提供了完整的链路透传验证与配置对比。经分析，该现象高度符合 gRPC 客户端默认启用 TLS 而下游服务端为明文通信导致的连接握手失败，且当前配置未显式声明跳过 TLS 验证或指定明文协议。属于有效的配置咨询与插件行为澄清请求，需维护者确认默认安全策略或补充文档说明，故保持开启。

## Best Possible Solution

建议维护者或社区协助确认 flusher_otlp 的 gRPC TLS 默认行为。若默认启用 TLS，请指导用户在配置中补充 Insecure: true 或相应 TLS 设置；若插件当前不支持明文 gRPC 或存在连接失败日志缺失，请标记为改进项并补充 Debug 日志与配置示例。待用户验证配置调整或维护者明确插件边界后，可据此关闭或转为文档/功能增强工单。
