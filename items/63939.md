## Summary

Keep #63939 open for maintainer judgment. Current main still has the 30-minute WhatsApp inbound-message watchdog hard-coded through runtime tuning, does not expose `web.messageTimeoutMs` in config/schema/docs, and the maintainer-designated canonical fix #71466 is still open. A maintainer comment explicitly said to leave this PR open until #71466 lands, then decide whether the config surface remains useful or the PR is superseded.

## Best Possible Solution

Let the maintainer-owned #71466 path resolve the core quiet-session bug first. After it lands, reassess #63939: close it as superseded if disabling the app-message silence watchdog by default fully addresses the operator problem, or adapt the config/schema/docs pieces if maintainers still want an explicit `web.messageTimeoutMs` opt-in surface.
