## Summary

Keep open. Current main still writes gateway file logs as raw tslog JSON with numbered argument keys and only adds `time`; it does not add top-level `hostname`, `agent_id`, `session_id`, `channel`, or a flattened `message` field to the file log lines. Diagnostic/OTel plumbing is adjacent but intentionally separate from logs and does not satisfy the requested multi-agent log filtering surface.

## Evidence
