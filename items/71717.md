## Summary

Keep #71717 open. It is a fresh Windows 11 report against 2026.4.23, and current main does not show a shipped or source-level fix for the reported `spawn EPERM` path. The host exec path still spawns a Windows PowerShell process, and the shared spawn fallback does not retry `EPERM`; related reports cover different Windows exec issues or fixes that shipped before the reporter's version.

## Best Possible Solution

Keep the issue open and triage it as a native Windows exec spawn failure. The likely maintainer path is to reproduce on Windows 11 with 2026.4.23/current main through Feishu or webchat, capture the exact spawned executable and errno from the supervisor, then add a focused regression test or diagnostic around PowerShell spawn `EPERM` handling if the failure is in OpenClaw rather than host policy.
