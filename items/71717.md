## Summary

Keep #71717 open. The report is recent, non-maintainer-authored, and describes a plausible native Windows exec failure in 2026.4.23. Current main still routes host exec through a spawned PowerShell process, and the relevant spawn wrapper only retries EBADF, not EPERM. I did not find source, test, docs, changelog, or related-report evidence that the specific `spawn EPERM` all-commands-fail path has been fixed or superseded.

## Best Possible Solution

Keep the issue open and ask for a minimal repro against the latest release/current main: exact command sent through Feishu/Web, full exec error stack/logs, `openclaw --version`, gateway run mode, whether `pwsh.exe` or `powershell.exe` is selected, and whether that executable can be spawned from the same gateway process/user. If reproduced, the likely fix belongs in the Windows host exec shell/spawn path with diagnostics or a safe fallback, plus a regression test covering EPERM propagation/recovery.
