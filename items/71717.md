## Summary

Keep #71717 open. It is a recent non-maintainer bug report with a concrete Windows 11 symptom (`spawn EPERM` for all agent exec commands), and current main still has a plausible active failure path: host exec on Windows launches PowerShell through the process supervisor, while the shared spawn fallback only retries `EBADF`, not `EPERM`. The provided related items cover narrower Windows exec issues such as quoting, gateway restart, output encoding, or env leakage; they do not supersede this all-commands spawn failure.

## Best Possible Solution

Keep the issue open and investigate the Windows agent exec host path. The likely maintainer path is to reproduce with a native Windows gateway on 2026.4.23+ using Feishu/Web exec, capture the exact spawned executable and Node error fields, then fix `src/agents`/`src/process` as appropriate: either improve PowerShell resolution/diagnostics, add a safe direct-spawn path for simple commands, or add a narrowly justified Windows fallback for `EPERM` without weakening exec approval or shell-injection hardening. Add focused Windows-mode unit coverage for the chosen behavior and cross-link related Windows exec issues only if a canonical duplicate emerges.
