## Summary

Keep #71717 open. It is a fresh Windows bug report with no protected labels or maintainer author, and current main does not contain a clear fix for the reported `spawn EPERM` failure: Windows exec still resolves to PowerShell/pwsh, spawns that shell through the supervisor child adapter, and propagates spawn errors. Related ClawSweeper reports cover Windows quoting, gateway restart, and CJK decoding issues, but not this all-commands `spawn EPERM` symptom.

## Best Possible Solution

Keep the issue open and ask for a focused Windows repro with gateway logs, resolved PowerShell/pwsh path, `openclaw doctor` output, exec config, and a minimal command such as `echo ok`. The likely code path to harden is Windows exec shell resolution/spawn in `src/agents/shell-utils.ts`, `src/agents/bash-tools.exec-runtime.ts`, and `src/process/supervisor/adapters/child.ts`: add clearer diagnostics for `spawn EPERM`, consider retrying alternate PowerShell candidates when the selected shell fails before command execution, and cover the behavior with mocked win32 spawn-failure tests.
