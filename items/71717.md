## Summary

Keep #71717 open. It is a fresh Windows exec failure report from a non-protected author/label set, and current main does not provide enough evidence that the reported `spawn EPERM` path is fixed. The gateway exec path still spawns a Windows shell process, and the supervisor retry path does not retry `EPERM`; without a Windows live repro or linked fix, conservative cleanup should not close it.

## Evidence

- **Issue context:** Issue #71717 reports OpenClaw 2026.4.23 on Windows 11 via npm global install; all agent exec-tool commands fail immediately with `spawn EPERM`. The issue has no comments, no logs, and only the `bug` label, so there is no protected-label or maintainer-author override, but also no discussion proving a duplicate or fix.
