## Summary

Keep #71569 open. Current main has Mattermost draft-preview code and tests for in-place finalization, but the documented `channels.mattermost.streaming` config surface is still absent from the strict Mattermost schema/types and the main inbound path creates preview streaming unconditionally while forcing block streaming off. The reporter's superseding of #71565 is useful context, and the remaining item is a real docs/runtime/config mismatch plus unresolved notification UX decision.

## Best Possible Solution

Keep the issue open and fix the mismatch in the Mattermost plugin: add the documented `streaming` config schema/types/generated metadata, resolve `off | partial | block | progress` in the Mattermost runtime, add doctor/config compatibility if scalar values are intended, and make an explicit product decision for final notification behavior such as progress mode, silent drafts, or an opt-in final ping. Add regression tests for config acceptance/preservation and for each runtime mode, including `streaming: "off"` disabling draft previews.
