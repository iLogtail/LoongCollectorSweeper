## Summary

Keep open. Current main still has the Mattermost docs/config/runtime mismatch reported in #71569: docs advertise `channels.mattermost.streaming` modes, but the Mattermost schema, public config type, generated config metadata, and normal runtime path still do not implement mode-aware Mattermost preview streaming. Related #70253 overlaps on the ignored streaming setting, but it is broader and does not supersede this focused Mattermost streaming/notification UX report.

## Best Possible Solution

Implement the documented Mattermost preview streaming contract in the bundled Mattermost plugin: add `streaming` support to the Mattermost schema/types/generated metadata and compatibility/doctor paths; resolve `off | partial | block | progress` in the Mattermost runtime; skip draft preview creation for `off`; implement `progress` so the final answer is delivered as a normal notifying post; and add focused config/runtime tests for top-level and per-account Mattermost streaming settings. Keep #71569 as the focused tracker and reference #70253 only for the overlapping ignored-config symptom.
