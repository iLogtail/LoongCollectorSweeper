## Summary

Keep open. Current main still recognizes silent replies only as an exact `NO_REPLY` token or the one-field JSON action envelope, does not strip the reported `NO_REPLY.NO_REPLY` tail, and only strips XML-style reasoning tags. The related PR #69472 is a focused open fix attempt, but this behavior is not implemented on main.

## Best Possible Solution

Keep #69470 open and resolve it through PR #69472 or an equivalent maintainer-reviewed patch. The fix should stay in the shared auto-reply token layer, add a narrowly scoped reasoning-prefaced silent-reply predicate, tolerate repeated trailing silent tokens separated by punctuation/whitespace, and preserve #19537 by continuing to deliver substantive replies that merely end with `NO_REPLY`. Add regression coverage in `src/auto-reply/tokens.test.ts` and a delivery/normalization test for the leaking payload shape.
