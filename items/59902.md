## Summary

Keep open. Current main still implements the Cron job History action as an in-place run-history reload on the existing /cron page. The run-history section is present lower on the page, but the History button does not scroll to it, focus it, open a panel/modal, or otherwise provide the visible feedback requested in #59902.

## Best Possible Solution

Keep #59902 open and implement a small Control UI UX fix in `ui/src/ui/views/cron.ts`/`ui/src/ui/app-render.ts`: when History is clicked, select the job, load its runs, then scroll and focus the Run history card or open a dedicated history panel. Add a focused view test that stubs `scrollIntoView`/focus or asserts the new panel behavior, and keep docs unchanged unless the final interaction changes the public Control UI workflow.
