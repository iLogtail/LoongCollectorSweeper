## Summary

Keep open. Current main still embeds managed `.env`/config/auth-profile values in the gateway install environment before the LaunchAgent plist is written, so PR #70968 is not obsolete or implemented on main. The PR is a focused fix for the closed #70612 report, but its current patch still needs maintainer review because the existing review thread identified a plausible `PATH`/service-owned-variable purge risk.

## Best Possible Solution

Keep PR #70968 open and ask for a small revision: remove literal values for managed secret/durable keys before plist serialization, but do not purge generated service-owned environment entries. Update the existing install-plan tests to cover both the secret-removal behavior and the `PATH` preservation case, then run the targeted command test and changed gate before merge.

## Risks / Open Questions

- The current PR patch should preserve or explicitly exclude service-owned/generated variables such as `PATH`, `HOME`, `TMPDIR`, and `OPENCLAW_PORT` when removing managed literal values, otherwise it may trade the secret-staleness fix for a service runtime regression.
