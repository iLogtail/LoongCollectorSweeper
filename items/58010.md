## Summary

Keep open. Current `main` still enforces `guilds.<id>.users` / `roles` before the `allowBots` mention check, and Discord allowlists still only parse `discord:`, `user:`, and `pk:` prefixes, so the reported `bot:` behavior is not implemented or documented away.

## Evidence

- **`bot:` is still not a recognized Discord allowlist prefix:** `DISCORD_OWNER_ALLOWLIST_PREFIXES` is still `['discord:', 'user:', 'pk:']`. In `normalizeDiscordAllowList`, unknown prefixed strings fall through to slug/name matching instead of ID parsing, so `bot:123` would not be treated as a bot ID.
