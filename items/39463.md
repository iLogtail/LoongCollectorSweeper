## Summary

Keep open. Current main still lacks the low-memory/OOM install handling proposed by PR #39463: `install_openclaw_npm` only has build-tool retry, stale directory cleanup, and `EEXIST` recovery, while the linked DigitalOcean install bug remains represented by user-facing docs that require manual swap. The PR is core installer work, not ClawHub scope, and its automatic swap behavior still carries maintainer-level product/security tradeoffs rather than being safely obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to address the low-memory npm install failure in core installer/docs, likely with conservative OOM detection and explicit opt-in or safer guidance for swap, plus installer-focused coverage where feasible. If maintainers reject automatic swap, the linked bug should still get a documented/manual path that works before the npm install step on 1 GB hosts.
