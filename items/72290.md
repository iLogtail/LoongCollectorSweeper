## Summary

Keep PR #72290 open. Current main still lacks the PR's two central protections for #72176: inbound dedupe has no body/timestamp/peer fallback when `MessageSid` is missing or unstable, and `chat.send` user transcript updates still omit a user-turn `idempotencyKey`. The PR is useful active implementation work, but it is not merge-ready as submitted because it adds `chat.ts.backup`, has a review concern on 64-bit hash truncation, and lacks visible regression tests for the reported duplicate shape.

## Best Possible Solution

Keep #72290 open and request a clean, maintainer-reviewable revision: remove the accidental backup file, add focused regression coverage for #72176, decide whether the fallback hash should use at least 128 bits, and verify the content-based dedupe scope is narrow enough not to collapse legitimate repeated messages. If maintainers prefer a different design, land an equivalent fix against #72176 rather than closing this PR as obsolete.
