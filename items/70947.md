## Summary

Keep open. Current main still treats Control UI/webchat attachments as image-only at the picker, paste/drop filters, client serializer, local optimistic render path, and Gateway parser. PR #70947 addresses a real remaining bug linked to #69447 and appears to be the newer refreshed attempt over #57707, but it should not be closed as implemented or duplicate. It also has unresolved review concerns that should be handled before merge.

## Best Possible Solution

Keep #70947 open, request fixes for the size-validation and image-sniff fallback review comments, and use it or a maintainer-owned successor as the canonical implementation for #69447. The final fix should update the Control UI picker/preview/serializer, Gateway non-image persistence into transcript/media fields, generic file rendering, and regression tests across UI serializer, renderer, Gateway chat.send, and media-store filename/MIME handling.
