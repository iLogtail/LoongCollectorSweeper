## Summary

Keep open. Current main still has the WebChat/Control UI attachment limitation this PR is trying to fix: the UI only accepts image MIME types, the client serializes every attachment as an image, and the gateway parser still drops sniffed non-image payloads. The PR is a plausible focused bug fix with relevant tests, but it has unresolved reviewer concerns and needs maintainer/change-request handling rather than conservative cleanup closure.

## Best Possible Solution

Keep #70947 open and request changes or maintainer review. The best path is to rebase onto current main, put the generic file handling through the current shared gateway attachment path in src/gateway/chat-attachments.ts plus chat.send transcript media fields, keep sniff-based recovery for images with missing or wrong declared MIME, validate base64 and decoded size before Buffer allocation, and retain focused UI/gateway/media regression coverage.
