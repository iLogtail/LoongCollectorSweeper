## Summary

Keep open. Current `main` and the latest shipped release still open the memory index database with `PRAGMA busy_timeout = 5000` only; the proposed WAL change is not implemented, and the existing memory-index test still asserts the old `busy_timeout` behavior. The prior PR comment about a stale test remains material, so there is not strong evidence this PR is obsolete or safely closable.
