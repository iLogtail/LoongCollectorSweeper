## Summary

Keep open. Current main ships Android assistant entrypoints, but it does not contain this PR's hardening: App Action prompts are trimmed but not bounded, assistant launches are handled without restored-launch fingerprinting, and transient launch routing state is still plain in-memory ViewModel state rather than restored through SavedStateHandle. The related #47574 cleanup only covers assistant entrypoint support, not these lifecycle/sanitization fixes.

## Best Possible Solution

Keep #61298 open or refresh it against current main. The best path is a narrow Android fix that preserves the intended composer-vs-auto-send product behavior, adds bounded prompt sanitization, persists only the needed restored-launch identity/state through lifecycle recreation, and keeps the regression tests for prompt bounds, fresh-vs-restored launch handling, and SavedStateHandle restoration. #47574 should not be used as the canonical close target because it implemented the assistant entrypoints, not this hardening.
