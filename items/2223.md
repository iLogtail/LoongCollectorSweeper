## Summary

本 Issue 请求补充 LoongCollector 在 Kubernetes Sidecar 模式下的官方部署用例与防日志丢失最佳实践。经核查当前 main 分支文档与示例，Kubernetes 部署指南仍以 DaemonSet 为主，尚未提供完整的 Sidecar 配置模板、共享卷映射及优雅退出（preStop）指导。该需求明确且属于文档/示例完善范畴，具备可操作性，建议保持开启以待社区或维护者补充。

## Best Possible Solution

建议维护者或贡献者在 docs/cn 与 docs/en 的 Kubernetes 部署指南中新增 Sidecar 模式章节，提供标准 Pod YAML 模板（含 emptyDir 共享卷、日志路径映射、资源限制）及防丢日志最佳实践（如 preStop 延迟退出、checkpoint 持久化或流式采集配置）。待相关文档或示例 PR 合并后即可关闭本 Issue。若需深入讨论特定环境适配，可引导至 Discussions 区继续交流。
