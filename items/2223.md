## Summary

本 Issue 请求提供 LoongCollector 在 Kubernetes Sidecar 模式下的官方部署用例与最佳实践文档，明确指出社区现有讨论仍基于旧版 ilogtail 镜像且存在容器终止顺序导致的日志丢失风险。经核查当前 main 分支文档与部署指南，核心内容仍聚焦于 DaemonSet 部署，缺乏针对 Sidecar 模式的生命周期管理、共享卷配置及防丢日志的官方 YAML 示例。该诉求属于清晰有效的文档与示例补充需求，非无效、重复或已实现项，符合保持开启条件。

## Best Possible Solution

建议维护者或社区贡献者在 docs/cn 与 docs/en 的 Kubernetes 部署章节中补充 Sidecar 模式官方示例。内容需涵盖：1) LoongCollector 容器镜像替换指引；2) 标准 Pod YAML 模板（含 emptyDir/hostPath 共享卷挂载、日志路径配置与资源限制）；3) 针对 Sidecar 终止顺序导致日志丢失的最佳实践（如 preStop hook 延迟退出、checkpoint 持久化或流式采集配置）。待相关文档或示例 PR 合并后即可关闭本 Issue。若用户需针对特定环境进行故障排查，可引导至 Discussions 继续交流。
