## Summary

本 Issue 请求提供 LoongCollector 在 Kubernetes Sidecar 模式下的官方用例与部署文档，并指出社区现有讨论仍基于旧版 ilogtail 镜像且存在日志丢失风险，属于明确的文档与示例补充诉求。

## Best Possible Solution

建议维护者或贡献者在 docs/cn/ 与 docs/en/ 目录下补充 Sidecar 模式的官方部署示例。内容应包含：1) LoongCollector 容器镜像替换指引；2) 标准 Kubernetes Pod YAML 模板（重点展示 emptyDir/hostPath 共享卷挂载、日志路径配置与资源限制）；3) 针对 Sidecar 生命周期导致的日志丢失问题的最佳实践（如 preStop hook 延迟退出、采集位点持久化或流式采集配置）。待相关文档或示例 PR 合并后，可关闭本 Issue。若用户需针对特定环境进行故障排查，可引导至 Discussions 继续交流。
