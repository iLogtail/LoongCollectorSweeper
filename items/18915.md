## Summary

Keep open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still sends regular Telegram videos without probing or forwarding width/height in both direct-send and reply-delivery paths. The PR addresses a plausible, specific Telegram channel bug, but its current patch needs rework because it imports a core helper from plugin production code and includes an unrelated global video-size limit change.

## Best Possible Solution

Keep the PR open and request a focused revision. The fix should live inside the Telegram plugin or use an approved `openclaw/plugin-sdk/*` media seam, probe dimensions for regular video attachments, forward `width` and `height` in both `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, continue excluding video notes and GIF/animation sends, add regression tests for both paths, and remove or split the unrelated media-size-limit change.
