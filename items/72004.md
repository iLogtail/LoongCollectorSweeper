## Summary

Keep PR #72004 open. Current main still has the ordering race the PR targets: final and aborted chat events append to `chatMessages` before clearing `chatStream`, while the renderer can display committed messages and the live stream at the same time. The linked bug #71992 is still open in the provided context, and the earlier equivalent PR #34164 was closed unmerged, so this PR is not obsolete or implemented on main.

## Best Possible Solution

Keep #72004 open as the focused candidate fix for #71992. The best maintainer path is to review and land this stream-clearing reorder, or an equivalent patch, with the regression test and normal UI validation rather than closing it as implemented, duplicate, stale, ClawHub work, or not actionable.
