## Summary

Current main already contains Discord inbound replay-dedupe logic plus regression coverage that matches the reported single-inbound => double-reply failure, and the fix was shipped in v2026.4.12 after the reporter's v2026.4.8 build.

## Evidence

- **Shipped fix note:** The v2026.4.12 changelog explicitly says replay-dedupe behavior was standardized across Discord so duplicate deliveries stay reply-once after success and only retry after pre-delivery failures.
  - file: [CHANGELOG.md:872](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/CHANGELOG.md#L872)

- **Replay key claimed before enqueue:** Current main builds a replay key from accountId, channelId, and messageId, claims it before enqueueing the Discord inbound, and returns early when the claim is not newly acquired.
  - file: [extensions/discord/src/monitor/message-handler.ts:241](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/src/monitor/message-handler.ts#L241)

- **Discord replay guard implementation:** The Discord replay guard exposes explicit claim/commit/release helpers around the inbound replay key, which is the dedupe primitive used by the handler and worker.
  - file: [extensions/discord/src/monitor/inbound-dedupe.ts:22](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/src/monitor/inbound-dedupe.ts#L22)
  - command: `nl -ba extensions/discord/src/monitor/inbound-dedupe.ts | sed -n '1,140p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Successful runs commit dedupe state:** After processing a Discord inbound job, the worker commits the replay key; only explicit DiscordRetryableInboundError paths release it, which prevents a successfully handled inbound from being re-driven into a second visible reply.
  - file: [extensions/discord/src/monitor/inbound-worker.ts:68](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/src/monitor/inbound-worker.ts#L68)
  - command: `nl -ba extensions/discord/src/monitor/inbound-worker.ts | sed -n '68,147p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Regression coverage for duplicate deliveries:** The Discord queue tests cover dropping duplicate inbound deliveries before preflight, retrying only after explicit retryable failure, and keeping replay committed after non-retryable failure so visible side effects do not happen twice.
  - file: [extensions/discord/src/monitor/message-handler.queue.test.ts:241](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/src/monitor/message-handler.queue.test.ts#L241)
