## Summary

Keep open. Current `main` has a newer `requiresStringContent` path for string-only OpenAI-compatible backends, but it does not normalize assistant `content: null` to `""` for tool-call messages. The reported DeepSeek/OpenAI-compatible failure path is still plausible on `main`, so there is not strong evidence to close this PR.

## Evidence
