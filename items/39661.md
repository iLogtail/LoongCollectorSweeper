## Summary

Keep open. Current main still has only a single Discord guild/channel `requireMention` boolean, while `users` is modeled and documented as an allowlist. Preflight resolves one mention requirement before sender allowlist handling, so the requested per-user trigger policy remains missing and is not superseded by the stale label or related reports.

## Best Possible Solution

Keep #39661 open. The likely fix belongs in the bundled Discord plugin/config surface: add an additive per-user mention policy shape to the Discord schema and public types, define guild/channel/user precedence, resolve the effective `requireMention` using the sender before `resolveInboundMentionDecision`, and cover mixed-user same-channel behavior with tests and docs.
