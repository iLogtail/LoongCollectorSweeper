## Summary

Keep #39661 open. The prior ClawSweeper keep-open analysis still matches current main: Discord `requireMention` is one guild/channel boolean, while `users` and `roles` remain sender allowlists. There is no current schema, runtime resolver, test, or doc path for per-user mention-trigger overrides in the same Discord channel.

## Best Possible Solution

Keep the issue open until the bundled Discord plugin implements a typed per-user mention policy, or until a maintainer explicitly rejects the product direction. A likely implementation path is to add guild/channel schema and exported types for sender-specific `requireMention` overrides, define precedence between guild default, guild user override, channel default, and channel user override, resolve the effective value in Discord preflight with sender identity available, and update generated config docs plus regression tests for two users in the same channel with different trigger behavior.
