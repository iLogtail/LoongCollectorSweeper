## Summary

Keep open. Current main has a broad agent-visible cron tool and gateway cron RPC, but it does not implement the RFC's narrower schedule_task-style API with gateway-injected, non-forgeable creator provenance, playbook-only job creation, active-until requirements for recurring jobs, per-agent quotas, or create-call audit ledgering. The issue is also very recent, has no protected close labels, and has a supporting comment rather than stale/missing-info signals.

## Best Possible Solution

Keep #71712 open. The best path is a focused design/implementation plan for a narrow gateway-owned agent scheduling surface: do not accept agentId/sessionKey/createdBy from the payload, inject caller session and agent identity in the gateway, require registered playbook identifiers plus validated params instead of free-form prompts, require activeUntil for recurring jobs in coordination with #71711, enforce per-agent active-job quotas, and add durable audit entries for create/update/disable calls. Existing cron.add can remain the unrestricted operator surface.
