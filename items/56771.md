## Summary

Keep #56771 open. Current main has targeted required-parameter guards for some filesystem tools and improved retry guidance, but it does not implement the requested generic pre-call parameter validation/configuration surface (`tools.validation.enabled` / `autoPrompt`) across tools. The `exec` example still uses an exec-tool-local command check rather than schema-backed pre-dispatch validation, and the config schema has no validation/autoprompt knob.

## Best Possible Solution

Keep the issue open and implement a generic, schema-backed validation step in the tool-definition adapter or shared tool execution pipeline before invoking each tool's `execute`, with focused tests for `read`, `exec`, MCP/plugin tools, malformed/non-object args, and retry guidance. Decide separately whether this should be always-on or exposed as a documented `tools.validation` config surface; if a config key is added, update `types.tools.ts`, `zod-schema.agent-runtime.ts`, generated config docs, and user docs.
