## Summary

Keep open. Current main still has only manual selected-session deletion in the Dashboard plus generic CLI/store cleanup by age, entry count, disk budget, or missing transcript. It does not implement a Dashboard Prune workflow with a 48-hour default, eligibility preview, or Telegram/Discord channel/topic existence checks. Related #50310 remains a narrower Discord cleanup request and does not supersede this broader Dashboard feature.

## Best Possible Solution

Keep #49259 open and implement a core/gateway-backed Dashboard prune workflow: add a preview endpoint that returns age-qualified orphan candidates with reasons, channel-specific existence probes owned by the Telegram/Discord plugins, and an explicit enforce/delete path that revalidates before mutation. Wire that into the Sessions screen with a 48-hour default age input, clear candidate preview, and focused tests for deleted/inaccessible Discord channels/threads and Telegram groups/topics.
