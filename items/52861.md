## Summary

Keep #52861 open. Current main has several partial mitigations, including compact/budgeted skill metadata and built-in compaction, and #14785 remains a related narrower tracker for tool-schema overhead. But the headline request here is broader and still not implemented: default runs still build the full effective tool set and send full tool schemas unless an explicit allowlist/policy has already narrowed them, and workspace bootstrap files still default to per-turn injection.

## Best Possible Solution

Keep the issue open as the broader performance tracker. The best implementation path is a core prompt/tool budget design that preserves explicit policy semantics, adds safe automatic relevance or activation-based tool schema routing before provider payload construction, and separately considers a better default or summary/index mode for recurring workspace bootstrap files. Continue using #14785 as the narrower canonical tracker for tool-schema footprint work.
