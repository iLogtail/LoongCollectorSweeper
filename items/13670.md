## Summary

Keep #13670 open. Current main still has adjacent Control UI model and usage surfaces, but it does not implement the requested always-visible topbar status display for active model plus token/context usage. The topbar status area only renders chat mobile/theme controls, debug status/models remain scoped to the Debug tab, and the existing chat/usage surfaces do not satisfy the specific header/topbar placement request.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive token/context usage from fresh session usage events or sessions data where possible, avoid making Debug-tab polling normal UI state, and add focused UI tests plus docs if the surface ships.
