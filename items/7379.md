## Summary

Keep #7379 open. Current main still has the pieces needed for `/status` to show GitHub Copilot Premium/Chat quota, but the per-response `/usage` footer path still formats only token/cache/cost/session-key data and never loads provider quota windows. Related PR #43098 directly targeted this request but was closed unmerged, so it remains useful prior work rather than a shipped fix.

## Best Possible Solution

Keep #7379 open until the reply footer path intentionally reuses or loads cached provider usage for the active GitHub Copilot provider, appends Premium/Chat quota without changing /usage tokens versus /usage full semantics, and adds a focused regression test. PR #43098 is the best prior implementation to review as a starting point, but it was closed unmerged and is not shipped on current main.
