## Summary

Keep open. Current main does not implement the PR's central reply-routing idempotency plumbing or Discord raw-target normalization, and the provided related items are about different routing bugs. The PR is also not ready to merge as-is because the review discussion identifies a real Telegram chunk/media dedupe regression in the branch.

## Best Possible Solution

Keep the PR open, but require a revised patch before merge. The best path is to thread a stable idempotency key through dispatch/routeReply/deliver/ChannelOutboundContext/Telegram, scope or salt provider-send dedupe per chunk/media send, add long Telegram chunk/media regression tests, and land the Discord channel-target normalization with focused tests.
