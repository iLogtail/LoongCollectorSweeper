## Summary

Keep PR #52782 open. Current main has not landed the cleanup fallback change in the refactored reset helper, and the current caller still drops numeric Telegram topic ids before reset. The PR likely needs retargeting and a broader fix rather than closure.

## Best Possible Solution

Keep the PR open but ask for an updated patch against current main. The robust fix should let resetReplyRunSession accept string|number thread ids, derive the reset topic id from ctx.MessageThreadId, the thread-scoped session key, or the persisted entry thread metadata as appropriate, and pass that id to both next-session path creation and cleanup fallback candidates. Add regression coverage in src/auto-reply/reply/agent-runner-session-reset.test.ts for numeric Telegram topics and restored topic sessions where MessageThreadId is absent.
