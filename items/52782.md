## Summary

Keep PR #52782 open. Current main still has a topic-aware reset-cleanup gap after the reset logic was refactored into `agent-runner-session-reset.ts`, and the PR has an unresolved review comment identifying an additional restored-thread/session-key case that the one-line patch does not fully cover.

## Best Possible Solution

Keep the PR open and ask for an update against current main: make `resetReplyRunSession` accept `string | number`, derive the effective topic id from `MessageThreadId` and the thread-scoped `sessionKey` when needed, add the topic-aware fallback candidate for old transcript cleanup, and add regression tests for numeric Telegram topic ids plus the `SessionKey`-only restored-thread case.
