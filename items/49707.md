## Summary

Keep open. Current `main` still drops all-zero usage snapshots before they can reach `/status`, and the PR's single-line guard change would not fix the end-to-end behavior because later code paths still coerce zero totals back to `undefined` and render `?`.

## Evidence

- **Current embedded subscribe path still rejects all-zero usage:** `resolveAssistantUsage()` only returns a candidate when `hasNonzeroUsage(usage)` is true, so a provider snapshot like `{ input: 0, output: 0, total: 0 }` is discarded before `recordAssistantUsage()` or `getUsageTotals()` can surface it.
  - file: [src/agents/pi-embedded-subscribe.ts:384](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/pi-embedded-subscribe.ts#L384)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '372,406p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Current totals helper still turns zero totals into missing data:** `getUsageTotals()` returns `undefined` unless at least one accumulated field is `> 0`, and then maps zero fields with `|| undefined`, so an all-zero total still disappears instead of being preserved as `0`.
  - file: [src/agents/pi-embedded-subscribe.ts:415](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/pi-embedded-subscribe.ts#L415)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '415,433p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Status transcript fallback also drops zero-valued usage:** `readUsageFromSessionLog()` returns `undefined` when both `promptTokens` and `total` are zero, so later `/status` fallback cannot recover a legitimate zero-usage snapshot from transcript logs either.
  - file: [src/status/status-message.ts:341](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/status/status-message.ts#L341)
  - command: `nl -ba src/status/status-message.ts | sed -n '337,345p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Missing totals render as question mark in status output:** When `totalTokens` is absent, `formatTokens()` renders `?/<context>`, which matches the reported symptom.
  - file: [src/status/status-message.ts:245](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/status/status-message.ts#L245)

- **Review discussion already identified the PR gap:** The PR review comment noted that changing only `recordAssistantUsage()` was incomplete because `getUsageTotals()` still rejected all-zero totals, which matches the current `main` code inspection.
