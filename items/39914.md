## Summary

Keep open. Current `main` already offers a push-based iMessage path through the bundled BlueBubbles plugin, but the legacy `imessage` channel still has a fixed polling-related bottleneck and does not expose a configurable `channels.imessage.pollIntervalMs`. That means the issue's request is only partially satisfied, so there is not strong enough evidence to close it conservatively.

## Evidence

- **Legacy iMessage is still the current `imsg` path:** The docs explicitly describe `imessage` as a legacy `imsg` integration and tell new deployments to use BlueBubbles instead, which means the reported lag is about a still-supported legacy path rather than a removed feature.
