## Summary

Keep open. Current `main` still implements session storage as a filesystem-backed `sessions.json` store, not a PostgreSQL-backed session store, and the issue timeline already points to related ongoing work in open PR #58550.

## Evidence

- **Session config is still a file path:** `SessionConfig` exposes `session.store?: string` and the surrounding session maintenance options are all defined around `sessions.json` file management; there is no session database driver/config in this type.
  - file: [src/config/types.base.ts:158](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/types.base.ts#L158)

  - file: [src/config/sessions/store-load.ts:76](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/sessions/store-load.ts#L76)
