## Summary

Keep open. Current main has useful partial memory-maintenance surfaces: promotion preview/apply, Dreaming UI listings, and memory-wiki claims/lint/apply workflows. It still does not provide the first-class operator MVP requested here: review queue/listing with per-item actions, edit, non-destructive forget/tombstone/supersede states in memory-core, or active-memory conflict-resolution flows usable without direct file/DB access.

## Best Possible Solution

Keep this open as the tracker for first-class bundled memory operator flows. The likely path is a memory-core/Gateway/Control UI contract with explicit item states such as pending-review, active, tombstoned/forgotten, superseded, contested, and resolved; CLI/API/UI actions for list/review/edit/forget/resolve; tests for promotion/search/index behavior around those states; and docs tying the flow to memory-wiki where durable claims and contradiction dashboards are the better surface.
