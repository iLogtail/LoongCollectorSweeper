## Summary

Keep #23353 open. Current main still does not implement Anthropic native server-side web_search/web_fetch/code_execution, Programmatic Tool Calling allowed_callers, or the requested Anthropic server content-block handling. The issue also has a same-day maintainer/Codex comment explicitly keeping it open, and related implementation PRs #23469 and #23573 are closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #23353 open as the broad Anthropic provider/runtime feature tracker. The best implementation path is an explicit Anthropic design covering config/schema/docs, request payload injection for web_search/web_fetch/code_execution server tools, managed tool suppression or coexistence rules, response block preservation/rendering, Programmatic Tool Calling allowed_callers behavior, and focused unit/integration tests. If maintainers want smaller work units, split web_search/web_fetch and code_execution/PTC into canonical follow-up issues rather than closing this tracker.
