## Summary

Keep #23353 open. Current main at bc24b547d0888fb716505b07e3f2dceeb8abd203 still does not implement Anthropic native server-side tools, web_fetch/web_search server block handling, code_execution, or Programmatic Tool Calling allowed_callers. The related implementation PRs #23469 and #23573 are closed and unmerged in the provided context, and the latest issue discussion already recorded a keep-open review that current main still supports.

## Best Possible Solution

Keep #23353 open as the broad Anthropic provider/runtime feature tracker. The best path is an explicit Anthropic implementation covering config/schema/docs, payload injection for server tool definitions, suppression or coexistence rules for managed web_search/web_fetch, response block preservation/rendering, Programmatic Tool Calling allowed_callers behavior, and focused unit/integration tests. If maintainers want narrower tracking, split web_search/web_fetch and code_execution/PTC into canonical follow-up issues rather than closing this tracker.
