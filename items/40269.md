## Summary

Keep open. Current `main` has Anthropic-thinking replay guardrails and unit tests, but I did not find strong evidence that the gateway leak in #40269 is fixed. The exact recovery helper for the reported Anthropic `thinking or redacted_thinking blocks ... cannot be modified` rejection exists only in `thinking.ts`/its tests, with no production call site on current `main`, while invalid-request assistant errors are still formatted into user-visible `LLM request rejected: ...` replies.
