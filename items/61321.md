## Summary

Current main still preserves the stale absolute same-agent `sessionFile` path instead of remapping it into the current state dir. The linked fix PR #61322 is open/unmerged, and the relevant source path would still hand the old absolute path to the session write lock.

## Best Possible Solution

Keep #61321 open until PR #61322 or an equivalent narrow fix lands. The fix should remap canonical same-agent stale-root `sessionFile` paths into the current `agents/<agent>/sessions/<file>` directory before write-lock acquisition, with a focused regression test in the session path tests and no change to intentional cross-agent absolute transcript handling.
