## Summary

Keep open. Current main only applies Slack agent identity on normal reply delivery; the draft-stream preview path still drops the resolved identity, and the monitor path still forwards bare emoji aliases without normalizing them to Slack's documented `:emoji_name:` form. The two linked PRs in the issue discussion accurately identify useful fixes, but they were closed unmerged and current main lacks their advertised draft-stream identity passthrough.

## Evidence

Its first preview message calls `send(..., { cfg, token, accountId, threadTs })`, so the preview `chat.postMessage` call cannot receive `username` or `icon_emoji`.
