## Summary

Keep PR #53108 open. Current main at 06d409dc2738034f6005ede426726e830ea48679 still does not implement sandbox Docker `capAdd`: the public config type, strict Zod schema, resolved Docker config, Docker create args, docs example, generated schema, and tests only cover `capDrop`. ## Best Possible Solution

Keep the PR open and either land a maintainer-reviewed version of `capAdd` or fold it into the broader Docker sandbox-parameters work tracked by #29384. The implementation should update the public config type, strict schema, generated config schema/baseline, resolver defaults and precedence, config hash behavior, Docker create args, docs, and targeted tests, while preserving `capDrop: ["ALL"]` by default and making the security tradeoff explicit.
