## Summary

Keep open. Current main still has the reported Discord voice readiness-abort shape: `/vc join` waits for `VoiceConnectionStatus.Ready` with a hard-coded 15s timeout, destroys the connection on failure, and only records the session after that wait succeeds. The requested configurable/longer connect timeout and retry behavior are not present in config types, schema, docs, or tests. Related #39825 tracks the same symptom family, but it is also open and the provided context does not show a maintainer decision making it canonical or superseding this issue.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path in `extensions/discord/src/voice/manager.ts`: add focused regression tests for initial `entersState` abort/timeout before session activation, decide on a longer default and/or bounded retry/backoff, expose a documented `channels.discord.voice.connectTimeoutMs`-style config if maintainers want user tuning, and update config types, zod schema, UI hints, generated metadata/docs, and related #39825/#63125 triage once the chosen behavior lands.
