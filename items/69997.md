## Summary

Keep PR #69997 open. The linked crash path is still plausible on current main because session-store JSON is only shallowly accepted and `normalizeStoredOverrideModel` still calls `.trim()`/`.toLowerCase()` without runtime type guards. However, this PR is not merge-ready as submitted because it also bundles unrelated heartbeat behavior changes that review comments correctly flag as a regression risk.

## Best Possible Solution

Keep the item open, but ask for a narrow update: drop the heartbeat commit from this PR, add runtime type guards for persisted `providerOverride`/`modelOverride` at the shared model-selection/session-store boundary, and add regression tests using malformed session-store values. If heartbeat empty-object semantics need to change, handle that separately with explicit maintainer product judgment and dedicated tests.
