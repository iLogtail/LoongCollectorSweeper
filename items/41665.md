## Summary

Keep #41665 open. Current main still has bundled skills with Homebrew-only install specs that lack per-install OS filters, and onboarding still builds the installable skill prompt from whatever install options survive `normalizeInstallOptions`. The PR is a focused metadata fix using an existing supported `os` field on install specs; it is not implemented on main and is not superseded by the unrelated local title-search matches.

## Best Possible Solution

Keep the PR open for maintainer review or a small follow-up branch: apply per-install `os` filters to the affected bundled brew install specs, keep xurl's fallback as the supported `node` installer kind, and consider a focused bundled-metadata regression check so Windows onboarding no longer offers brew-only skills. Native Windows installer additions such as winget/choco/scoop should be treated as optional follow-up work, not a reason to close this fix.
