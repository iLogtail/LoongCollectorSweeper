## Summary

Keep open. Current `main` still implements only TTL-gated `contextPruning.mode: "cache-ttl"`; there is no shipped turn-based or rolling pruning option in code, config schema, docs, or tests.

## Evidence

- **Runtime is still TTL-gated:** The pruning hook returns early unless `Date.now() - lastCacheTouchAt >= ttlMs`, so active sessions that keep touching the cache do not trigger pruning on each turn.
