## Summary

Keep open. Current main does not contain the graph-aware chain tracing described in #37842/#37945: `InputProvenance` has no `chainAgentIds`, the gateway schema would reject it, and `sessions_send` still only stamps source provenance before delivery. The linked PR #37945 is useful prior work, but the provided GitHub context shows it was closed unmerged, and the referenced commit is not present in this checkout. Related #51832 is a different gateway/channel error-cascade mechanism, not a canonical duplicate for this remaining sessions_send loop-detection request.
