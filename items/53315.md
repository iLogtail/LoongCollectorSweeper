## Summary

Keep open. Current `main` still treats unknown `channels.<id>` and unknown heartbeat targets as fatal config errors, while the repo’s current mitigation for stale bundled-channel configs is a narrow `plugins install`/doctor recovery path rather than relaxed gateway startup validation. The PR discussion also surfaced unresolved regression risk: the proposed broad downgrade would stop hard-failing typoed channel ids, and the original crash-loop scenario is still not fully covered when heartbeat targets point at the missing channel.

## Risks / Open Questions

- Merging the PR as written would weaken config validation by letting misspelled `channels.<id>` keys start successfully in the common no-`plugins` case.
- The original upgrade/crash-loop report is not fully addressed when a missing channel is also referenced by heartbeat target config.
- Current main already provides a narrower install/doctor recovery path, so changing gateway startup semantics needs explicit maintainer judgment rather than conservative auto-close cleanup.
