## Summary

Keep #33724 open. Current main still hard-codes automatic config-write backups to an adjacent five-file `.bak` ring, with no typed config/schema/UI surface for backup directory, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged, and `openclaw backup` is a separate archive command rather than configuration for automatic `.json.bak` rotation.

## Best Possible Solution

Keep #33724 open until maintainers either implement configurable built-in config backup behavior or explicitly decline that product direction. A complete implementation should add typed config/schema/help/docs/UI support for backup directory, retention count, and trigger policy; thread that policy through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and `src/config/io.observe-recovery.ts`; preserve current defaults; and update rotation/recovery/audit tests.
