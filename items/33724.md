## Summary

Keep open. Current main still hard-codes OpenClaw config backups to `${configPath}.bak` plus four numbered backups, triggers that maintenance from config write paths, and exposes no typed config/schema/UI option for backup path, retention, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged, and the cron/archive backup alternatives do not reconfigure OpenClaw's own config-write `.bak` files.

## Best Possible Solution

Keep #33724 open until a maintainer either implements configurable built-in config backup behavior or explicitly rejects the product direction. A complete implementation likely belongs in typed config/schema/help/docs/UI wiring plus plumbing through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and recovery/audit code that currently assumes `${configPath}.bak`, with tests for default compatibility, custom directory, custom retention, and any supported trigger policy.
