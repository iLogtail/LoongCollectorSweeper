## Summary

Keep open. Current `main` does not restore general recursive skill discovery; it still loads only a skill root itself, its immediate child directories, and one narrow `dir/skills/*` heuristic for extra dirs.

## Evidence

- **Scanner only enumerates one directory level:** `loadSkillsFromDirSafe()` loads either the root skill directory or immediate child directories from `listCandidateSkillDirs()`. It does not recurse into deeper descendants such as `<root>/bundle-a/skill-x/`.
  - file: [src/agents/skills/local-loader.ts:96](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/agents/skills/local-loader.ts#L96)
  - command: `nl -ba src/agents/skills/local-loader.ts | sed -n '1,220p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Workspace loader only accepts immediate subfolders as skill candidates:** After resolving a base dir, `loadSkillEntries()` lists `childDirs` once and then checks only `path.join(skillDir, "SKILL.md")` for each immediate child. The comment explicitly says: "Only consider immediate subfolders that look like skills".
  - file: [src/agents/skills/workspace.ts:439](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/agents/skills/workspace.ts#L439)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '150,520p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Recent main adds only a narrow nested-root heuristic:** `resolveNestedSkillsRoot()` recognizes `dir/skills/*/SKILL.md` by probing `dir/skills` and treating that folder as the real root when any immediate child contains `SKILL.md`. This is not general recursive scanning of configured skill directories.
  - file: [src/agents/skills/workspace.ts:317](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/agents/skills/workspace.ts#L317)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '150,520p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Tests cover only `dir/skills/*` support, not arbitrary recursive discovery:** Current tests validate "nested repo-style skills roots (dir/skills/*)" and later-entry detection under that shape. There is no coverage for deeper layouts like `<root>/agent-browser-skills/agent-browser/SKILL.md` discovered via recursive descent.
  - file: [src/agents/skills.buildworkspaceskillsnapshot.test.ts:227](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/agents/skills.buildworkspaceskillsnapshot.test.ts#L227)
