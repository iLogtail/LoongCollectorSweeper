## Summary

Keep open. Current main has explicit agent add/update and hot reload for `agents.list`, but it still has no `agents.autoDiscover` config surface and no runtime path that scans external workspace globs for `agent.json` or `openclaw-agent.json`. The related zero-restart work reduces the operational pain but does not implement the requested workspace-discovery behavior.

## Best Possible Solution

Keep the issue open and narrow it around external workspace agent discovery now that explicit zero-restart agent management exists. A good implementation should define the `agents.autoDiscover` or plugin-backed config shape, trusted scan roots, accepted file names, schema validation, deterministic merge order, precedence versus explicit `agents.list`, hot reload/watch behavior, docs, and tests covering unsafe workspace config injection.
