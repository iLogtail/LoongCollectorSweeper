## Summary

Keep PR #64436 open. Current main still does not expose `runtime.usage` or an equivalent supported plugin SDK path for model cost resolution, while the internal pricing helpers already exist and are tested. This is a focused plugin API proposal, not a ClawHub-only plugin request, and the provided PR discussion shows the author addressed the initial bot-review concerns in the PR branch.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow additive plugin runtime API, likely based on this PR, after confirming the exposed types are stable, adding or updating plugin runtime docs, keeping the SDK API baseline aligned, and adding a small runtime-shape test for `api.runtime.usage`.

## Risks / Open Questions

- Adding `runtime.usage` is a public plugin SDK expansion, so it needs maintainer API judgment plus aligned docs, API baseline, and preferably a small runtime contract test before merge.
- Exposing `resolveModelCostConfig` directly may freeze an internal pricing helper contract; maintainers may prefer a narrower wrapper type if they do not want plugin authors coupled to `src/utils/usage-format.ts` signatures.
