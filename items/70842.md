## Summary

Keep open. Current `main` still routes `openclaw wiki bridge import` through the local wiki CLI path, while primary-command CLI loading narrows runtime loading to the `wiki` owner plugin. Public artifact discovery still depends on the process-local memory capability registered by `memory-core`, so the reported 0-artifact bridge behavior remains plausible on current `main`. The issue timeline also links an open PR (`#71479`) that matches this exact gap.
