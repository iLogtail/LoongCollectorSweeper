## Summary

Keep #38633 open. Current main still shows `pidsLimit: 256` in the user-facing sandbox Docker config example while the runtime leaves `pidsLimit` unset unless explicitly configured. The related code-default PR #38621 and docs-clarification PR #38914 are closed unmerged, and the broader security/default question remains open in #38604.

## Best Possible Solution

Keep #38633 open until maintainers either land a narrow docs clarification saying `pidsLimit` is an explicit opt-in example and unset unless configured, or add a real default in `resolveSandboxDockerConfig` with matching tests and docs, coordinated with #38604.
