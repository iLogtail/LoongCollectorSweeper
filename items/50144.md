## Summary

Keep open. Current `main` does not implement this Feishu config gate, and the PR as proposed still has the review-flagged regression where `@_all` short-circuits explicit bot mentions in the same message.

## Evidence

- **Current main still treats `@_all` as an unconditional bot mention:** `checkBotMentioned()` returns `true` as soon as message content contains `@_all`, with no config gate.
  - file: [extensions/feishu/src/bot-content.ts:236](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/feishu/src/bot-content.ts#L236)

- **Current main has no `respondToAtAll` schema surface:** Feishu group/shared config currently expose `requireMention` but not `respondToAtAll`, so the proposed opt-in behavior is not on `main`.
  - file: [extensions/feishu/src/config-schema.ts:142](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/feishu/src/config-schema.ts#L142)

- Landing the PR as-is would likely regress mixed messages that include both `@_all` and a direct bot mention, especially in groups using `requireMention`.
