## Summary

Keep PR #71024 open. Current main still lacks a Mattermost read-only account inspector, so Mattermost status inspection falls back to the strict resolver that throws on unresolved SecretRef bot tokens. The linked bug #70942 is a real current-main gap relative to the documented read-only SecretRef behavior, and this PR is still the focused implementation path rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and finish review/merge work if the final branch cleanly adds a Mattermost inspectAccount path, preserves botTokenStatus/configured_unavailable in status snapshots, keeps runtime resolveMattermostAccount strict, and includes the proposed Mattermost regression tests for unresolved SecretRef status inspection.
