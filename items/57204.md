## Summary

Keep open. Current main still has the reported behavior: a dirty build-relevant watched tree returns `dirty_watched_tree` and rebuilds before the source mtime fallback can decide whether `dist` is already current. The linked fix PR #57236 is still open in the provided GitHub context, so the issue remains the canonical bug until that work is reviewed and landed or superseded.

## Best Possible Solution

Keep #57204 open while reviewing or superseding #57236. The likely product/code path is to change `scripts/run-node.mjs` so dirty build-relevant trees fall through to the mtime check, then update `src/infra/run-node.test.ts` with regression coverage for dirty-but-not-newer skipping rebuild and dirty-and-newer still rebuilding.
