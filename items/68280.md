## Summary

Keep open. Current main still contains the anonymous local loopback probe path when local gateway auth is required but unresolved, and the related #61179 cleanup comment explicitly made #68280 the canonical remaining fix. The PR is not ready to land as-is because the latest patch still needs security/maintainer review around user-facing SecretRef failure details, but it is useful focused gateway/CLI work rather than obsolete cleanup.

## Best Possible Solution

Keep #68280 open as the canonical implementation path for the local probe-auth fail-fast behavior. Revise it so local `gateway status`, summary `status`, and daemon status skip anonymous loopback probes only when local auth is explicitly required and unresolved, preserve explicit URL/RPC overrides and unset/none/trusted-proxy auth modes, and surface only redacted/generic user-facing auth failure messages with focused regression tests.
