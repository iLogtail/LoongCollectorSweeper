## Summary

Current main still does not implement `[[tts:...]]` directive handling in the Voice Call telephony TTS path. The shared message-channel TTS path parses directives and passes overrides into synthesis, and ElevenLabs already supports `voiceId` directive tokens, but the Voice Call Twilio streaming path still sends raw text to a telephony provider whose API accepts only text and uses a startup-time merged config. This matches the maintainer's April 25 keep-open review comment, so there is no conservative cleanup close reason.

## Best Possible Solution

Keep the issue open. The best fix is in the bundled Voice Call plugin and TTS runtime boundary: parse `[[tts:...]]` with the existing `parseTtsDirectives` policy before telephony synthesis, strip directives from spoken text, and pass per-utterance overrides into telephony synthesis without duplicating parser logic. Add focused tests for Twilio stream playback text cleanup, allowed `voiceId`/model overrides, disabled override policy, and fallback to the existing static config.
