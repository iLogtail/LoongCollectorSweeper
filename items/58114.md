## Summary

Keep open. Current `main` still does not wire `[[tts:...]]` directive parsing into the Voice Call telephony TTS path. Telephony synthesis still takes raw text plus a startup-merged static config, while the message-channel TTS path still parses directives before synthesis. The 2026-04-25 maintainer comment is consistent with the current code.

## Evidence

- **Telephony TTS provider uses a fixed merged config and only accepts raw text:** `createTelephonyTtsProvider` computes `mergedConfig` once, exposes `synthesizeForTelephony(text)`, and always calls `runtime.textToSpeechTelephony({ text, cfg: mergedConfig })` with no per-utterance override input.
  - file: [extensions/voice-call/src/telephony-tts.ts:38](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/extensions/voice-call/src/telephony-tts.ts#L38)
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Twilio telephony playback forwards raw text directly into telephony synthesis:** `playTtsViaStream` calls `ttsProvider.synthesizeForTelephony(text)` directly. There is no directive parsing or text cleanup in the telephony playback path before synthesis.
  - file: [extensions/voice-call/src/providers/twilio.ts:619](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/extensions/voice-call/src/providers/twilio.ts#L619)
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Message-channel TTS still parses directives before synthesis:** `maybeApplyTtsToPayload` calls `parseTtsDirectives(text, config.modelOverrides, ...)` and derives cleaned/visible TTS text from the result before continuing.
  - file: [extensions/speech-core/src/tts.ts:1083](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/extensions/speech-core/src/tts.ts#L1083)

- **Voice Call config already exposes `tts.modelOverrides`:** The bundled Voice Call plugin schema still includes `tts.modelOverrides` flags such as `allowVoice`, `allowModelId`, and related directive controls, so the requested behavior remains plausible but unwired in telephony.
