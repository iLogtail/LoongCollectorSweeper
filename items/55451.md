## Summary

Current main implements the operator-visible diagnostics requested by #55451: assistant terminal errors keep friendly copy for chat/UI surfaces while structured logs include bounded raw provider error previews, provider/model, failure classification, provider error type/message preview, hashes, and request-id hashes. Rate-limit formatting also preserves provider-specific details such as retry timing instead of always collapsing to the generic rate-limit text. The same implementation is present in the v2026.4.24 release tag.

## Best Possible Solution

Close this as implemented and keep the current bounded/redacted diagnostics design. If maintainers want a richer Control UI diagnostics panel later, that should be a narrower UI enhancement rather than keeping this incident-level observability issue open.
