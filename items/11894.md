## Summary

Keep #11894 open. Current main still has internal auth-profile routing and model-fallback lifecycle notices, plus local cost summaries by provider/model, but it does not persist or expose the actual resolved auth profile per model call or aggregate costs by auth profile. The supplied maintainer comment also explicitly kept this issue open, and the burn0/#55379 workaround covers network-level cost visibility only, not OpenClaw auth-profile attribution.

## Best Possible Solution

Keep this issue open until OpenClaw stores the resolved authProfileId with each model-call usage record, preserves it in transcript/session-safe metadata, aggregates /usage cost or session_status totals by auth profile, and emits or stores a clear user-visible profile rotation/fallback event. Existing runner auth-profile plumbing and model-fallback lifecycle events can be reused, but burn0 remains only an interim network-level workaround.
