## Summary

Keep #28913 open. Current main e23d17da79dc2345ca55902e632d3c7622accacc still lacks source-based filtering for memory-lancedb autoCapture: there is no captureFilter/excludeWrapped config surface, shouldCapture remains trigger-based and wrapper-unaware, and the agent_end auto-capture path stores matching user-message text without checking EXTERNAL_UNTRUSTED_CONTENT boundaries. The prior ClawSweeper comment is still accurate against current main.

## Best Possible Solution

Keep the issue open and implement a narrow, backwards-compatible memory-lancedb option such as captureFilter.excludeWrapped. Add typed config parsing, manifest schema/UI hints, and an agent_end pre-filter that skips configured external-content boundary blocks before shouldCapture. Cover wrapped EXTERNAL_UNTRUSTED_CONTENT containing trigger words such as 'always' while preserving direct user preference auto-capture, and update user-facing plugin docs if the config becomes public.
