## Summary

Keep #39631 open. Current main still intentionally surfaces failed mutating tool warnings as user-facing error payloads even when the assistant already produced normal output, and Telegram still delivers those error payloads rather than suppressing them. The two related fix PRs in the provided context, #39636 and #39717, were closed unmerged, so they do not supersede or implement the issue.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the agent reply payload policy. The likely fix belongs around `resolveToolErrorWarningPolicy` / `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, preserve a fallback warning when no assistant reply exists, and keep full details in logs/debug surfaces. Update the payload tests and Telegram dispatch coverage to lock in the desired channel behavior.
