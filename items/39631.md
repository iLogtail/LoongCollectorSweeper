## Summary

Keep #39631 open. Current main still emits user-visible warning payloads for mutating tool failures such as `edit`, and Telegram delivery still sends those `isError` payloads to chat. Existing tests assert that mutating tool errors remain visible even when assistant output exists, while the related PRs #39636 and #39717 were closed unmerged.

## Best Possible Solution

Keep the issue open until core/channel output behavior changes. A maintainer-quality fix should either suppress mutating tool warning payloads from user chat when a clean assistant reply already exists, or normalize them into a user-facing assistant message, with focused payload tests plus a Telegram dispatch regression for the failed `edit()` scenario. If maintainers intentionally want these safety warnings visible, the issue should be resolved by an explicit product decision and matching docs/config wording.
