## Summary

Keep #39126 open. Current main still does not implement session-based exec trust windows, and the related later PRs are overlapping but not a clear canonical replacement for this broader gateway/CLI/Discord/protocol/security feature. The PR is stale and security-sensitive, but it remains substantive work rather than a conservative cleanup candidate.

## Best Possible Solution

Keep this open until a maintainer either rebases and reviews a trust-window implementation against current gateway/CLI/protocol/exec-host and Discord plugin surfaces, or explicitly declares a canonical successor such as #46956 or #47696 and closes the others as duplicates. The implementation should include focused tests for agent self-escalation, operator/admin authorization, expiry, audit behavior, and deny-policy override ordering, plus user-facing docs for the time-bounded workflow.
