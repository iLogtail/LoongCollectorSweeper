## Summary

Keep #71590 open. Current main still has only the existing exponential temporal decay path, and neither the runtime config schema, generated schema, docs, nor tests contain the proposed Weibull model or `weibullShape` option. The PR is a recent draft with a concrete implementation for the open memory-ranking request in #57307, but it has unresolved review feedback about inverted public JSDoc and should not be closed as obsolete.

## Best Possible Solution

Keep #71590 open as a focused draft implementation candidate. The contributor should fix the public JSDoc direction, complete the PR evidence/template, update docs/help if the config is user-facing, run the memory-core/config changed gates, and let maintainers decide whether the Weibull decay model belongs in bundled `memory-core` or should be folded into the broader #57307 memory-ranking design.
