## Summary

Keep PR #60824 open. Current main still has no ACP stream legacy migration: the schema only accepts the new `acp.stream.maxOutputChars` and `maxSessionUpdateChars` fields, the runtime migration registry does not include an ACP stream migration, and generic doctor unknown-key stripping would remove old values rather than preserve the two renamed limits. The PR needs a rebase into the current modular doctor migration files, but the author already addressed the bot-reviewed edge case in `e6a344ef7`, so the work is still plausibly useful.

## Best Possible Solution

Keep the PR open and ask for a rebase into the current modular legacy-config migration system, likely as an ACP/runtime migration plus tests covering rename, non-overwrite, no-replacement removal, and wrong-typed removed keys. Also clarify whether the PR is really meant to fix the broad legacy-config upgrade problem from #35957 or should be retargeted to a narrower ACP stream legacy-key issue.
