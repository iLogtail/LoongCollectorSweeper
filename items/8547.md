## Summary

Keep #8547 open. Current main still lacks a user-configurable Web UI font size or font scale in browser-local settings, the Appearance UI, gateway `ui` config schema/types/docs, or chat CSS. The related #3027 is closed and does not replace this open request, and the issue discussion already includes a recent Codex review keeping it open.

## Best Possible Solution

Keep #8547 open for core Control UI accessibility/customization. A concrete path is to add a browser-local text-size/font-scale control under Appearance, persist it in `UiSettings`, apply it through CSS custom properties across chat messages, tool cards, sidebars, and compose inputs, and extend gateway schema/types/docs only if maintainers want server-side `ui` font settings as a public contract.
