## Summary

Keep #8295 open. Current main still does not implement Telegram `allowBots` parity for groups/topics: the public Telegram config types, strict Zod schema, generated metadata, docs, and inbound runtime path all lack an account/group/topic `allowBots` policy. The Plaiground ClawHub note in the discussion is useful for Discord-based bot-to-bot setups, but it does not satisfy Telegram group/topic support in the bundled Telegram plugin.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across public config types, strict schema, generated metadata/UI, docs, runtime filtering, and regression coverage. The likely path is to review and land PR #68325 or a successor that handles account/group/topic precedence, keeps self echoes blocked, preserves `channel_post`, and documents safe mention/allowlist defaults.
