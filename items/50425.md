## Summary

Keep open. Current `main` still serializes assistant reasoning blocks through `sessions_history`, and the existing tests explicitly lock that behavior in. The PR targets a real core bug, but its review thread also surfaced unresolved questions around replay semantics and `sessions_list` preview handling, so this needs maintainer judgment rather than conservative auto-close.
