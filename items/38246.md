## Summary

Keep open. Current main still sends normal WhatsApp auto-reply text back through the inbound `msg.reply(...)` path, with no owner-vs-third-party guard and no `forceExplicitTargetOnThirdParty`-style config. Existing owner pinning only protects `lastRoute` updates, not visible auto-reply delivery.

## Best Possible Solution

Keep #38246 open. Add a WhatsApp/channel-level routing guard in the auto-reply delivery path before `deliverWebReply`, backed by config schema/types/docs and regression tests for owner DMs, allowlisted non-owner DMs, and explicit message-tool sends.
