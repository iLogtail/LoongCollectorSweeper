## Summary

Keep open. Current main still sends WhatsApp plain-text auto-replies directly back to the inbound sender, so the requested safeguard is not implemented. Main has only a narrower mitigation: when `session.dmScope` is `main` and `allowFrom` identifies exactly one configured owner, OpenClaw can skip updating the shared session `lastRoute` for non-owner DMs. That does not block or reroute the immediate reply path described in #38246.

## Evidence

- **Immediate WhatsApp reply path still targets the inbound sender:** `deliverWebReply` sends text replies with `msg.reply(...)` and logs `to: msg.from`, so plain-text auto-replies in a DM still go back to the current sender.
