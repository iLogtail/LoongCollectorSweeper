## Summary

Keep open. Current main still directly interpolates the memory schema table identifiers in both the source helper and mirrored package file, and that helper is exported through the plugin SDK storage subpath. The bundled memory-core plugin currently passes fixed constants, which narrows immediate bundled-core exposure, but the public helper still accepts string table names and the PR is not obsolete. The PR also has an unresolved review concern about `quoteTable` regressing pre-quoted identifiers, so the right maintainer path is revision plus focused tests rather than cleanup closure.

## Best Possible Solution

Keep the PR open, but require a revised identifier-handling patch before merge: quote or validate table and column identifiers in the shared memory schema helper and package mirror, preserve schema-qualified and existing valid quoted identifier behavior, and add focused tests covering malicious identifiers plus normal `embedding_cache`/`chunks_fts` behavior.
