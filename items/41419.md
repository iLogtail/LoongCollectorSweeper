## Summary

Keep open. Current main still constructs Discord upload Blobs without forwarding the detected media content type, and shared media loading still optimizes non-GIF images through the JPEG/PNG optimizer, so the WebP preservation behavior requested by this PR is not implemented. The PR is a focused bug-fix attempt for open issue #41256, but it needs maintainer review/rebase against the moved Discord plugin paths and should address the existing review concern about trusting a .webp filename when MIME sniffing says otherwise.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to rebase and rework it against extensions/discord and src/media/web-media.ts. The best fix is to preserve Discord Blob MIME types for both normal and component uploads, add focused regression tests, and make WebP optimization bypass depend on trustworthy detected MIME/bytes so animated WebP is preserved without changing behavior for mislabeled JPEG/PNG inputs.
