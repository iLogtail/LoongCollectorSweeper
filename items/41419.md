## Summary

Keep PR #41419 open. Current main still has the Discord upload MIME gap and still sends WebP images through the shared optimizer, so this PR is not implemented or obsolete. The PR contains useful work for #41256, but the WebP bypass and shared media behavior need maintainer review rather than auto-close.

## Best Possible Solution

Keep this PR open for maintainer review or intentionally supersede it with a focused canonical fix for #41256. The best implementation should preserve Discord upload MIME types for both standard and component uploads, verify MIME from bytes before declaring Blob.type, preserve static/animated WebP only after safe dimension and metadata handling, and update regression coverage for Discord plus the shared WhatsApp media behavior.
