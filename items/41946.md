## Summary

Keep open. Current main still lacks the Feishu URL-underscore escaping/sanitization behavior, and the linked bug #41860 remains represented by an open issue. This PR is not merge-ready as proposed because the diff imports a non-exported helper and uses a markdown-link regex that would break parenthesized URLs, but the underlying product bug is still real enough that conservative cleanup should not close it as implemented or obsolete.

## Best Possible Solution

Keep the PR open or ask for a revised patch. The fix should handle both bare URLs and markdown-link URLs in Feishu send/edit paths, preserve valid URLs with balanced parentheses, add Feishu regression tests for underscore-containing URLs, and avoid exporting a helper unless it is intentionally part of the local module surface.
