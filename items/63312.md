## Summary

Keep this PR open. Current main still rejects symlinked skill directories whose realpaths escape the configured root, including the personal and project `.agents/skills` sources this PR targets. The linked bundled `dist-runtime` issue appears to have been handled separately by copying bundled plugin skill trees into the runtime overlay, but that does not implement the PR’s user-managed `.agents/skills` behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a scoped implementation for `agents-skills-personal` and `agents-skills-project` with the proposed positive and managed-negative regression tests plus any needed docs/security wording, or ask the author to retarget the PR description away from the already-closed bundled `dist-runtime` issue and clarify the intended `.agents/skills` symlink policy.
