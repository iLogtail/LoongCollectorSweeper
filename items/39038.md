## Summary

Keep #39038 open. Current main has adjacent Windows PATH/service hardening and the old `node host PATH` debug print is documented as removed, but I still did not find source, tests, docs, release notes, duplicate tracking, or reporter confirmation proving the native Windows 11 24H2 foreground `openclaw node run` hang now reaches the Gateway connection path.

## Best Possible Solution

Keep tracking this as a native Windows node-host startup bug. The best maintainer path is a focused Windows 11 24H2 regression smoke proving `openclaw node run --host 127.0.0.1 --port 18789` reaches Gateway connection with Node 24 and a non-ASCII Windows profile path, plus clearer docs or doctor validation if `--ignore-scripts` installs are unsupported.
