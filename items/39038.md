## Summary

Keep open. Current main removed the separate `node host PATH` debug log, and there are later Windows PATH/status/plugin-startup fixes, but I found no direct source, test, changelog, release, or reporter-confirmation proof that the reported native Windows 11 24H2 `openclaw node run` hang/no Gateway connection is fixed.

## Best Possible Solution

Keep tracking this as a native Windows node-host startup bug until maintainers either verify a focused Windows 11 24H2 + Node 24 smoke where `openclaw node run` reaches Gateway connection, ideally including a non-ASCII profile path, or explicitly document that this install/runtime combination is unsupported and should use a clean install without `--ignore-scripts` or WSL2.

The report also includes npm global install with `--ignore-scripts`, Node v24.14.0, and a non-ASCII Windows username.
