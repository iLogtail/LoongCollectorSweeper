## Summary

Keep open. Current `main` still has the broken multi-line JS placeholder blocks in the session export HTML template, while the export code still performs literal `"{{...}}"` replacements. This PR’s fix does not appear to be implemented on `main`, and the existing tests do not convincingly cover the real template file regression.

## Evidence

- **Current template still uses broken multi-line placeholders:** `template.html` on current `main` contains `MARKED_JS`, `HIGHLIGHT_JS`, and `JS` inside multi-line brace blocks rather than compact `{{PLACEHOLDER}}` tokens, so the literal replacement strings described in the PR are not present in the shipped template.
