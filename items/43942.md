## Summary

Current `main` still appears vulnerable to the reported `hunter-alpha` shape: OpenRouter reasoning fallback fields are mapped into `thinking` blocks, reply directives are parsed from visible text only, and the newer reasoning-only retry guard is limited to OpenAI/OpenAI-Codex lanes rather than OpenRouter.

## Evidence

- **Hunter Alpha is still configured as a reasoning model:** `openrouter/hunter-alpha` remains registered with `reasoning: true`, so OpenClaw still requests/accepts reasoning-formatted output for this route.
  - file: [extensions/openrouter/provider-catalog.ts:51](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/extensions/openrouter/provider-catalog.ts#L51)
  - command: `nl -ba extensions/openrouter/provider-catalog.ts | sed -n '44,58p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **OpenRouter only treats specific reasoning_details item types as visible text:** OpenAI-completions compat for OpenRouter exposes visible text only for `response.output_text` and `response.text`; other reasoning payloads are not promoted to visible assistant text.
  - file: [src/agents/openai-completions-compat.ts:109](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/openai-completions-compat.ts#L109)
  - command: `nl -ba src/agents/openai-completions-compat.ts | sed -n '78,112p'`

- **Fallback OpenRouter reasoning fields are still mapped to thinking blocks:** When a completions delta carries `reasoning_content`, `reasoning`, or `reasoning_text`, the stream parser pushes that text as `kind: "thinking"`, not as visible text.
  - file: [src/agents/openai-transport-stream.ts:1436](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/openai-transport-stream.ts#L1436)
  - command: `nl -ba src/agents/openai-transport-stream.ts | sed -n '1428,1456p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Visible reply extraction ignores thinking blocks:** `extractAssistantVisibleText`/`extractAssistantText` read text content via `extractTextFromChatContent`; thinking-only assistant messages therefore yield no visible text for reply parsing.
  - file: [src/agents/pi-embedded-utils.ts:116](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/pi-embedded-utils.ts#L116)
  - command: `nl -ba src/agents/pi-embedded-utils.ts | sed -n '102,136p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)

A `[[reply_to_current]]` tag that exists only inside reasoning/thinking would still be missed here.
