## Summary

Keep #56869 open. Current main still lacks the requested Docker Compose proxy examples and Docker install docs section, and this is a focused repo-owned deployment documentation/config improvement rather than ClawHub work. The PR should be revised before merge because its NO_PROXY sample needs to match OpenClaw/Undici env-proxy semantics.

## Best Possible Solution

Keep the PR open and revise it into a conservative Docker docs/config patch: add commented HTTP_PROXY/HTTPS_PROXY examples and Linux host.docker.internal extra_hosts guidance, but make the NO_PROXY sample match current EnvHttpProxyAgent-supported exact/suffix/wildcard/port semantics or explicitly avoid unsupported CIDR claims. Leave broader global proxy coverage questions to the already-open #43821 discussion.
