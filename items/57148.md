## Summary

Keep open. Current main does not implement the requested configurable subagent-result injection cap. It has a coarse 100KB frozen-result storage cap and existing tool-result truncation, but subagent completion findings are still passed into the internal completion event without a dedicated `maxSubagentResultChars`-style limit.

## Best Possible Solution

Keep the issue open and implement a dedicated, documented subagent completion-result cap, preferably aligned with the existing `agents.defaults.contextLimits` / `agents.list[].contextLimits` config family. Apply it before `AgentInternalEvent.result` is formatted for parent injection, reuse the existing truncation helper where appropriate, and add focused tests for direct replies, nested child completion findings, and the existing 100KB frozen-result path.
