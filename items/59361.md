## Summary

Keep open. Current main appears to fix or supersede the `openclaw agent` silent/JSON/gateway-path symptoms, but the issue also asks for `openclaw lobster` and `openclaw llm-task` root CLI commands. Those commands are still not registered on current main: Lobster and LLM Task are implemented as optional agent tools, and the docs now describe Lobster's standalone CLI as a separate `lobster` binary, not an `openclaw lobster` subcommand. That remaining product/API question needs maintainer judgment or a plugin-owned CLI registration change before this issue is safe to close.

## Best Possible Solution

Split or resolve the remaining CLI-command question explicitly. If OpenClaw should support the old workflow, add plugin-owned `api.registerCli` registrations in `extensions/lobster` and `extensions/llm-task` with descriptor-backed lazy root commands and focused CLI tests. If the product decision is that these are tool-only surfaces, update the issue with that rationale and docs links, and close manually rather than auto-closing as implemented.
