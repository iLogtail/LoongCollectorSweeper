## Summary

Keep open. Current main appears to have better error surfacing and documented Lobster/llm-task as plugin tools, but the issue bundles several regressions and at least one requested observable behavior remains unresolved: `lobster` and `llm-task` are still not registered as root OpenClaw CLI commands. I also did not find source/test evidence that the headless gateway discovery delay or agent hang/no-connection path from the reporter's follow-up is fixed end to end.

## Evidence

- **Issue discussion:** Reporter follow-up at https://github.com/openclaw/openclaw/issues/59361#issuecomment-4174333290 narrowed the agent symptom to a swallowed missing selector error, reported zero TCP connections even with `--agent`, and confirmed `openclaw lobster` was still unknown on v2026.3.31.
- **Agent CLI selector handling:** Current gateway path rejects `openclaw agent` without `--to`, `--session-id`, or `--agent`, then `agentCliCommand` catches gateway-path errors and falls back to embedded agent. Embedded agent has the same selector requirement, so this no longer looks like a silent success path from source alone, but there is no targeted #59361 regression proof in the inspected tests.
