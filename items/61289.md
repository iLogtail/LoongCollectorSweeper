## Summary

Keep open. Current main still lacks the PR's specific Telegram action-runtime aliases: `sendMessage` maps only `mediaUrl` and `media` into the outbound media path, while the PR adds `filePath` and `path` plus focused tests. Core `message send` already supports path-style inputs, but that does not close this narrower direct Telegram action gap.

## Best Possible Solution

Keep the PR open. The best path is either to land the narrow alias mapping in `extensions/telegram/src/action-runtime.ts` with the proposed `action-runtime.test.ts` coverage, preserving existing `mediaUrl` then `media` precedence and avoiding PR-only docs, or for maintainers to explicitly decide that direct `sendMessage` should only accept `mediaUrl`/`media` and close it as a product-scope decision.
