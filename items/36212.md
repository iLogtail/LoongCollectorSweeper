## Summary

Keep open. Current main still does not preserve queued platform message IDs through the collect-mode followup path well enough for Feishu/Lark native replies. The original Feishu message ID is captured at ingress and copied into the queued run, but the collect drain does not include per-item message_id metadata in the queued prompt or forward a queued messageId, and the followup runner does not pass currentMessageId into the embedded runtime. Related PR #36235 directly targeted this issue but was closed unmerged, so it does not supersede the open issue.
