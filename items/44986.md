## Summary

Keep open. Current main still always runs the npm global install path for install.sh after resolving the target version, and there is no shipped --force / OPENCLAW_FORCE_INSTALL / already-up-to-date short-circuit. The PR is also not safe to close as implemented because the proposed helper choice has an unresolved maintainer-reproduced false-skip concern: trusting a PATH openclaw binary can skip the npm install even when the npm-managed package is absent.

## Best Possible Solution

Keep the PR open for maintainer judgment or request changes. A safe implementation should compare the target version against the npm-managed package itself, for example via npm root -g/openclaw/package.json or verified npm-global-bin ownership, preserve finalization diagnostics such as PATH and gateway refresh, add the --force / OPENCLAW_FORCE_INSTALL surface, and include focused install.sh tests for same-version skip, forced reinstall, fresh install, and PATH-wrapper false positives.
