## Summary

Keep #44986 open. Current main still does not implement the requested installer no-op: the npm installer resolves the target version for display, then still calls the global npm install path. The PR addresses a real linked request (#44700), but its latest discussion includes an unresolved maintainer review concern that the skip check can trust an unrelated `openclaw` earlier on PATH rather than the npm-managed install.

## Best Possible Solution

Keep the PR open for maintainer/author revision. The safe implementation should base the skip on the npm-managed package itself, such as `$(npm root -g)/openclaw/package.json` or a verified npm global-bin target, add the documented `--force` / `OPENCLAW_FORCE_INSTALL` override, preserve finalization like PATH warnings and gateway refresh, and add focused installer tests/docs for the skip and force paths.
