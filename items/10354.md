## Summary

Keep #10354 open. Current main still defines the shared message tool `channel` parameter as a bare optional string with no description or enum, and the exposed tool schema still reaches agents that way. Runtime fallback reduces some Slack-context failures but does not implement the requested schema guidance, and the related fix PRs were closed unmerged.

## Best Possible Solution

Keep the issue open and land a narrow core change in `src/agents/tools/message-tool.ts` adding a `channel` schema description that says it expects a messaging provider/channel plugin id such as `slack` or `discord`, not the conversation target. Add a focused schema test proving the description is exposed. Defer an enum unless maintainers choose a dynamic-safe source for valid channel ids.
