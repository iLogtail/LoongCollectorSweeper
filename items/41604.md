## Summary

Current main no longer performs unconditional Anthropic/Qwen external CLI auth-profile sync on auth-store load. The runtime external CLI overlay only registers `minimax-portal`, and the reported `synced anthropic oauth credentials from claude cli` log path is absent. The same implementation is present in release `v2026.4.22`.

## Close Comment

Closing from a Codex review because this is already implemented on `main`. Current auth-profile loading goes through `src/agents/auth-profiles/external-auth.ts:57` and `src/agents/auth-profiles/store.ts:264`, and the only runtime external-CLI auth-profile bootstrap still registered is `minimax-portal` in `src/agents/auth-profiles/external-cli-sync.ts:74`. There is no Anthropic/Qwen auth-profile sync path there anymore, and the reported `synced anthropic oauth credentials from claude cli` log string is absent from the repo. I also verified the narrowed implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit this exact log on a current build, please open a fresh issue with the exact version and reproduction path.
