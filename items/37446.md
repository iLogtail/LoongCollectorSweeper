## Summary

Keep #37446 open. Current main still lacks a stable caller-supplied or deterministic recovery identity for the built-in `sessions_spawn` tool. The gateway `agent` method can dedupe when the same idempotency key is reused, and plugin-owned subagent runs can pass an optional idempotency key, but the native and ACP `sessions_spawn` paths still create fresh child sessions and fresh gateway idempotency keys on each spawn attempt. The linked ClawHub skill is useful operational guidance, not a substitute for a core replay/recovery contract.

## Best Possible Solution

Keep the issue open until OpenClaw core adds an additive idempotent recovery contract for `sessions_spawn`: accept or derive a stable recovery key, persist the mapping from recovery key to child session/run, make both native and ACP spawn paths return or reuse the existing child run after timeout/lost acknowledgement, and add regression coverage for both paths. Continue to treat the linked ClawHub skill as operational guidance, not as the core fix.
