## Summary

Keep #13744 open. Current main has typed session write-lock timeout errors and failover suppression, but the main requested behavior is still missing: no supported openclaw.json/env timeout/stale config, and the embedded run still holds the session file lock across the attempt lifecycle rather than only transcript mutation sections.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported openclaw.json and/or OPENCLAW_* surface for session write lock timeoutMs and staleMs, wires it into embedded run and transcript mutation paths, and either narrows the file-lock critical section to actual transcript read/repair/prewarm/append/rewrite work or explicitly documents the broad per-session lifecycle lock as intentional. Preserve the shipped SessionWriteLockTimeoutError failover suppression and add focused contention tests for concurrent same-session callers.
