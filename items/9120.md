## Summary

Keep #9120 open. Current main still has timing-based inbound debouncing and active-run queue collection, but it does not implement the requested user-controlled continuation marker, marker stripping, continuation timeout, or continuation-specific typing behavior. The existing Apr 26 issue comment reached the same conclusion, and the relevant config/debouncer/docs paths have not changed since that review.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation would likely add typed and documented messages.continuationMarker and messages.continuationTimeout settings, validate them in the strict schema and generated help, buffer marked text-only inbound messages before invoking the agent, strip the marker from model-visible text, release on the first non-marker message or timeout, define media/control-command behavior, update shared channel/plugin inbound helpers and tests, and decide whether existing typingMode: "message" is sufficient or a continuation-aware typing mode is needed.
