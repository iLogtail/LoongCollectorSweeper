## Summary

Keep #9120 open. Current main still has timing-based inbound debouncing and active-run followup collection, but not the requested user-controlled continuation marker, marker stripping, continuation timeout, or continuation-specific typing behavior. The existing maintainer/Codex discussion also explicitly kept this open for the same reason.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation would likely add typed and documented messages.continuationMarker and continuationTimeout settings, validate them in the strict config schema and generated help, buffer marked text-only messages before invoking the agent, strip the marker from prompt text, release on the first non-marker message or timeout, define media/control-command behavior, update shared channel/plugin inbound helpers and tests, and decide whether typing needs a dedicated continuation-aware suppression mode.
