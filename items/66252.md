## Summary

Keep open. Current main has implemented the per-agent TTS half, and the maintainer discussion explicitly says to keep #66252 open for the remaining STT/language/provider override work. I confirmed `agents.list[].tts` exists in schema/types/docs/runtime/tests, while there is no corresponding `agents.list[].stt` contract and audio transcription still resolves from global `tools.media.audio`.

## Best Possible Solution

Keep #66252 open for the remaining STT half. The likely product path is to add a typed `agents.list[].stt` or broader per-agent speech config contract, merge it over `tools.media.audio` for the active agent, thread agent identity through the relevant transcription paths, and update docs/tests while leaving the current per-agent TTS implementation intact.
