## Summary

Keep #72262 open. Current main has only shallow WhatsApp disconnect troubleshooting and does not document a `status=408 Request Time-out` runbook, even though the WhatsApp runtime/status path can surface disconnect status, reconnect attempts, and error text. The request is recent, docs-scoped, and complements rather than duplicates the narrower open WhatsApp reliability items.

## Best Possible Solution

Keep the issue open and add a focused docs section in `docs/channels/whatsapp.md`, with a short cross-link from `docs/channels/troubleshooting.md`. It should explain WhatsApp/Baileys 408 disconnect semantics, first diagnostic commands, how to choose wait vs relink vs host/proxy/DNS checks vs bundled plugin runtime-dependency repair, what a healthy credentials directory means, and the safe `channels logout` then `channels login` recovery path with backup/auth-dir cautions.
