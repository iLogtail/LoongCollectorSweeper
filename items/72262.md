## Summary

Keep #72262 open. Current main still has only generic WhatsApp disconnect guidance, while the requested `status=408 Request Time-out` runbook and troubleshooting-table row are absent. The latest issue discussion also explicitly corrects the earlier claim and says the runbook was drafted but not integrated.

## Best Possible Solution

Add the requested docs to current main: a dedicated `status=408 Request Time-out` subsection in `docs/channels/whatsapp.md` and a concise row in `docs/channels/troubleshooting.md`. The runbook should map the observable status/probe/log output to operator actions: wait for transient reconnect, inspect logs and `openclaw doctor`, check gateway/runtime/DNS/proxy conditions, define healthy WhatsApp credential state, and document a backup-aware `openclaw channels logout --channel whatsapp --account <id>` then `openclaw channels login --channel whatsapp --account <id>` recovery path. Verify the public docs anchors for `https://docs.openclaw.ai/channels/whatsapp` and `https://docs.openclaw.ai/channels/troubleshooting`.
