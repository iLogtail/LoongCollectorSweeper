## Summary

Keep open. Current main contains the earlier heartbeat re-arm and requests-in-flight fixes, but this issue is a recent concrete report that those fixes did not solve the observed 30-minute heartbeat cadence on Linux/Node 24. The related #66992 follow-up is macOS/App Nap-specific and remains open, so it does not supersede this Linux report. I found no later changelog/source evidence that the exact ~20h silent-stop behavior is fixed on main.

## Best Possible Solution

Keep #59710 open. The best maintainer path is to add or request diagnostics around heartbeat runner state, wake-layer pending/running/timer state, last scheduled due time, and skip reasons, then reproduce on Linux Node 24 with a 30m interval. If the stall is confirmed, fix the runner/wake lifecycle or add a self-healing watchdog; keep the macOS App Nap work tracked separately in #66992.
