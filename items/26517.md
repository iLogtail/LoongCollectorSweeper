## Summary

Keep #26517 open. Current main has useful session-label display, session switching, and backend `sessions.create` groundwork, but the requested WebChat create/manage flow and agent-facing session label/create API are still not implemented. A maintainer comment on the issue already reached the same conclusion on 2026-04-26, and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #26517 open until maintainers either implement or explicitly narrow the WebChat product scope. The likely implementation should reuse the existing `sessions.create` and `sessions.patch` backend support, add chat UI affordances to create, label/edit, switch, and manage isolated per-agent sessions, and add reviewed agent-facing session create/relabel semantics only if maintainers still want agents to control those labels.
