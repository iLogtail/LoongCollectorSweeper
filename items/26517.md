## Summary

Keep #26517 open. Current main has partial groundwork: the gateway can create and label session rows, and WebChat can display/switch existing labeled sessions. The requested WebChat create/manage flow and agent-facing session label/create tool are still missing, and the latest maintainer comment in the provided issue context explicitly kept this open for the same reason.

## Best Possible Solution

Keep this issue open until maintainers either implement a WebChat flow that can create, label, switch, and manage multiple per-agent sessions using the existing sessions.create/sessions.patch backend, or explicitly narrow the product scope. If the agent-controlled labeling part remains desired, add a reviewed agent tool/API surface rather than treating sessions_spawn labels as a substitute.
