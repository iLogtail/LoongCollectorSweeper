## Summary

Keep #17424 open. Current main still does not implement the requested public `agents.defaults.memorySearch.dimensions` config key, and the OpenAI-compatible memory embedding paths still do not forward a configured dimension value into inline or batch embedding requests. The recent maintainer/Codex issue comment reached the same conclusion, and the relevant files have not changed since that reviewed commit.

## Best Possible Solution

Keep #17424 open. Maintainers should choose the public config contract (`dimensions`, `outputDimensionality`, or an alias pair), then update schema/types/help/docs, default and per-agent merge logic, OpenAI-compatible inline and batch request builders, provider cache key data, and reindex identity. Add focused schema, resolver, request-body, batch, cache-key, and reindex tests.

## Risks / Open Questions

- Some OpenAI-compatible providers may reject a `dimensions` field, so implementation should be provider/model-capability aware instead of blindly adding it everywhere.
- Changing embedding dimensions must affect provider cache keys and reindex identity to avoid mixing vectors of incompatible sizes.
