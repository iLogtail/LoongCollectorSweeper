## Summary

Keep #17424 open. Current main still has partial dimension plumbing under `outputDimensionality`, but it does not implement the requested `memorySearch.dimensions` config key and still does not pass a `dimensions` field through OpenAI/OpenAI-compatible inline or OpenAI batch embedding requests. The latest issue discussion already identified this gap, and the current checkout still matches it.

## Best Possible Solution

Keep #17424 open. Decide whether `dimensions` should be accepted as an alias or whether docs/UI should standardize on `outputDimensionality`, then carry the resolved value into OpenAI/OpenAI-compatible clients, inline `/embeddings` request bodies, OpenAI batch JSONL request bodies, and provider cache/reindex identity. Add focused tests for defaults, per-agent overrides, schema acceptance/rejection, OpenAI inline requests, OpenAI batch requests, and dimension-changing reindex/cache behavior.
