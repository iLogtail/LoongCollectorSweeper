## Summary

Keep open. Current main has improved timeout failover compared with the issue's described cycling: profile rotation walks forward through candidates, model fallback is documented after timeout profile exhaustion, and the default LLM idle watchdog is now 120 seconds. However, the specific requested faster timeout handling is still not implemented: timeout failures are still explicitly excluded from auth-profile failure/cooldown state, tests assert timeout rotation does not cool down the profile, and configurable profile-rotation caps exist for overloaded/rate-limit failures but not timeout failures. The broader related #45589 remains open, but it is not a clean canonical replacement for this narrower timeout-profile fallback request.
