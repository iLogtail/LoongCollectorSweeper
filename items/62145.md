## Summary

#62145 should remain open. Current main still replaces the entire Control UI with the login gate whenever `connected` is false, so prior chat/messages are not accessible during a gateway disconnect. The chat view already has disabled-control behavior for disconnected state, but the top-level render gate prevents that view from being shown.

## Best Possible Solution

Implement this in the core Control UI: keep the already-loaded dashboard/chat visible when the gateway disconnects, disable mutating controls such as send/attach/config saves, show a compact offline/reconnecting banner with a reconnect/login-gate action, and add render/lifecycle tests proving chat history remains readable after a gateway close.
