## Summary

Keep #62495 open. Current main still emits warn-level escaped skill-path messages from the workspace skill loader, and the existing tests assert that warning behavior. The related PR #62518 directly targeted this issue by downgrading the warning to debug, but it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the root realpath containment check, but make benign sibling skill references non-noisy: either stop turning documentation links into candidate skill paths if that parser path exists, or downgrade the escaped-path startup warning to debug for expected sibling/symlink cases while retaining warn-level security/audit coverage where appropriate. Add a focused regression test for a workspace skill doc or sibling skill reference that verifies no warn-level startup noise while escaped skills are still skipped.
