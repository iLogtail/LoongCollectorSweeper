## Summary

Keep open. Current `origin/main` still sends Signal inbound text to the agent through raw `BodyForAgent`, so this is not implemented on main, but the PR as written would only wrap the legacy envelope/body path and does not address the prompt path the agent actually consumes. The existing review comments also identify two concrete regressions in the PR's history wrapping.

## Evidence

- **Signal handler on current main still uses raw inbound text for the agent path:** On `origin/main` (`fe930b987e33f4e1acd93b5ef1ea76c8b7260047`), the Signal event handler builds an envelope `Body` from `entry.bodyText` and separately sets `BodyForAgent: entry.bodyText` without `wrapExternalContent(...)`. That means the change is not already present on main.
  - file: [extensions/signal/src/monitor/event-handler.ts:154](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/extensions/signal/src/monitor/event-handler.ts#L154)
  - command: `git show origin/main:extensions/signal/src/monitor/event-handler.ts | nl -ba | sed -n '150,205p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Shared inbound-context code prefers `BodyForAgent` over `Body`:** `finalizeInboundContext(...)` resolves `BodyForAgent` from `normalized.BodyForAgent` first, only falling back to `CommandBody`, `RawBody`, or `Body` if `BodyForAgent` is absent. So a PR that wraps only `Body` does not protect the actual prompt path when `BodyForAgent` remains raw.
  - file: [src/auto-reply/reply/inbound-context.ts:64](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/auto-reply/reply/inbound-context.ts#L64)

- **Repo contract treats `BodyForAgent` as the inbound prompt surface:** The SDK migration docs deprecate plaintext inbound envelopes in favor of `BodyForAgent` plus structured user-context blocks, which matches the runtime contract comment that `formatInboundEnvelope(...)` is deprecated for prompt construction.
  - file: [docs/plugins/sdk-migration.md:494](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/docs/plugins/sdk-migration.md#L494)

This makes the PR's premise inconsistent with the current channel contract rather than an already-fixed omission unique to Signal.
