## Summary

Keep #30142 open. Current main partially supersedes this PR: Discord and Slack now have shipped sendPayload implementations, but Google Chat, Mattermost, MS Teams, and Synology Chat still lack sendPayload. The PR is stale and should be reworked before merge, but the remaining bundled channel work is still real and not fully tracked by a narrower canonical item in the provided context.

## Best Possible Solution

Keep #30142 open, or replace it with a narrower PR that targets only the still-missing bundled adapters: Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current shared sendPayload/sendTextMediaPayload helper patterns, leave the shipped Discord and Slack implementations from #30144 alone, and add adapter-level payload contract tests plus explicit handling for structured-only payloads and mid-loop cancellation expectations.
