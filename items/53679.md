## Summary

Keep open. Current main still writes an install-time TMPDIR into the macOS LaunchAgent plist, so the Migration Assistant failure described in #53679 remains plausible and is not implemented away on main.

## Best Possible Solution

Keep this issue open and fix the macOS gateway LaunchAgent install path to avoid persisting install-time TMPDIR, or to derive a valid temp directory at gateway startup. Add a regression test that generated macOS LaunchAgent plists omit TMPDIR or reject stale migrated TMPDIR values, and consider a doctor/repair hint for existing affected plists.
