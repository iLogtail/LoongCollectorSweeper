## Summary

Keep #72181 open. Current main still does not implement BlueBubbles `trainerMode`/`trainerNotifyNumber`, owner-gated draft approval, or supervised delivery before outbound DM sends. The request is fresh, concrete, scoped to the bundled BlueBubbles plugin, and asks for BlueBubbles ownership/product review rather than conservative cleanup closure.

## Best Possible Solution

Keep #72181 open for BlueBubbles owner/product review. A good implementation would stay under `extensions/bluebubbles`, add typed/schema-backed `trainerMode` and `trainerNotifyNumber` config, intercept allowed inbound DMs before normal reply dispatch, route drafts and owner commands through existing BlueBubbles send paths, enforce pending-message TTLs, and add monitor/config/docs coverage for both `training` and `supervised` modes.
