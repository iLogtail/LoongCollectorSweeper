## Summary

Keep #9876 open. Current main still decodes `sessionKey` into the macOS exec approval prompt request, but the native `ExecApprovalsPromptPresenter.buildAccessoryView` does not display it. Gateway exec approval payloads already carry turn-source metadata, and tests prove that metadata is preserved through the gateway/forwarding path, but the Swift native prompt model used by the popup still has no requester/channel/tool-context fields to decode or render. The prior Codex review comment on 2026-04-26 also kept this open, and the current checked-out main SHA still shows the same missing native UI behavior.

## Best Possible Solution

Keep #9876 open. The narrow fix should add a trimmed non-empty `Session` detail row in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift` inside `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the Swift prompt request model used by native macOS prompts to decode safe gateway turn-source/requester metadata, map it to user-readable labels, and add focused macOS coverage for socket, gateway, and node-host prompt paths.
