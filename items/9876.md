## Summary

Keep #9876 open. Current main still carries `sessionKey` through the macOS approval request model but the macOS popup presenter does not render it, while the web approval UI does. The richer requester/channel metadata already exists in gateway exec approval payloads and tests, but the Swift prompt model used by the macOS popup does not decode or display those fields.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `ExecApprovalsPromptPresenter.buildAccessoryView`: add a trimmed, non-empty `Session` detail row for `request.sessionKey` and cover socket, node-host, and gateway prompt paths. The fuller fix should extend the Swift prompt request model to decode safe requester metadata already carried by gateway exec approval payloads, derive user-readable channel/requester labels, and add focused macOS prompt/model tests.
