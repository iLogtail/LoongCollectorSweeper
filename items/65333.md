## Summary

Keep open. Current main has not implemented this PR: generated gateway install-token persistence still defaults a missing `gateway.auth.mode` to `"token"`, and the colocated test asserts that behavior. The docs also describe auth as required by default and `auth.mode="none"` as an explicit local-loopback-only choice, so the PR is a product/security-policy change rather than an obsolete or duplicate cleanup candidate.

## Best Possible Solution

Keep this PR open for maintainer product/security judgment. If local friction should be reduced, implement a narrower design that only permits `auth.mode="none"` when the effective exposure is proven local-only, preserves token auth for daemon/non-loopback/private-ingress risk paths, updates gateway docs/security guidance, and adds regression tests for loopback, lan/tailnet/custom, Tailscale Serve, and generated-token persistence.
