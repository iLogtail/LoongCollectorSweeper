## Summary

Keep #71867 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still runs `openclaw doctor` auth health against an implicit/default auth store and emits generic auth note titles without agent attribution. The related PR #71923 is an open candidate fix, but it is not shipped on main.

## Best Possible Solution

Keep the issue open and land a focused doctor-auth fix, likely via #71923 or an equivalent patch. The fix should label `Model auth`, cooldown, and OAuth refresh diagnostics with the agent id and preferably the auth store path; use read-only per-agent auth loads for diagnostics; avoid duplicating inherited-only stores; and add regression tests for divergent multi-agent auth state.
