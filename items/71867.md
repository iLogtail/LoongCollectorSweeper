## Summary

Keep #71867 open. Current main still reports doctor model-auth health from one implicit/default auth store and emits generic `Auth profile cooldowns`, `OAuth refresh errors`, and `Model auth` note titles with no agent id. The same work is being actively addressed by open PR #71923, but it is not merged, so the bug remains on main.

## Best Possible Solution

Keep #71867 open until the doctor auth path is fixed on main. The likely maintainer path is to review/land PR #71923 or an equivalent change that enumerates configured agent auth stores, labels doctor auth diagnostics with the agent id when multiple stores are involved, avoids creating inherited auth files during read-only reporting, and preserves the existing single-agent behavior with regression coverage.
