## Summary

Keep open. Current main still has the exact sandbox tilde-expansion behavior this PR targets: `expandPath` and `shortPath` in `src/agents/sandbox-paths.ts` call `os.homedir()` directly. The sandboxed write/edit path still routes through this resolver, while existing shared home-dir code already prefers `HOME` then `USERPROFILE` before `os.homedir()`. The PR discussion also shows the only bot review concern, `shortPath`, was addressed in the PR head commit.

## Best Possible Solution

Keep the PR open and land this narrow fix, preferably by reusing `resolveRequiredOsHomeDir` or an equivalent local helper for `expandPath` and `shortPath`, plus a focused regression test that simulates `HOME` differing from `os.homedir()` for sandboxed write/mkdir path resolution. Track broader `OPENCLAW_HOME` semantics separately in #54014.
