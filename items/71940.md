## Summary

Keep #71940 open. Current main still lacks the Hyprland/grim screenshot fallback, opt-in config key, strict runtime schema support, docs coverage, and route tests. The linked bug #64317 has concrete reproduction evidence for headed Chromium viewport screenshot timeouts on Omarchy/Hyprland/Wayland, so this is a real browser-plugin fix attempt rather than a cleanup-close candidate.

## Best Possible Solution

Keep #71940 and/or canonical bug #64317 open. The best path is a focused browser-plugin fix that adds an opt-in Hyprland/Wayland headed viewport screenshot fallback only for ordinary viewport screenshots, preserves existing CDP/Playwright paths for non-Hyprland, full-page, element, ref, and label cases, wires the flag through BrowserConfig plus strict schema/help/generated docs, documents grim/hyprctl setup, tests route selection and virtual-output cleanup, uses per-profile headless state, and fixes signal/failure cleanup before merge.
