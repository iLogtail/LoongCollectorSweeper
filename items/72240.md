## Summary

Keep #72240 open. Current main has partial exec diagnostics, but foreground SIGKILL failures still collapse to generic signal/timeout handling and the tool result shape lacks the requested kill cause, supervisor-vs-OS distinction, run/log pointer, and richer stdout/stderr tail metadata. The related issues are real and open, but none clearly supersedes the macOS diagnostic gap described here.

## Best Possible Solution

Keep this open and fix it in the exec/process supervisor path: add structured kill metadata through RunExit, ExecProcessOutcome, ExecToolDetails, node/gateway exec events, and system-event text. Track whether OpenClaw initiated termination and why, include timeout limits and run/context IDs, preserve bounded stdout/stderr tails, add a log-span pointer where available, and consider warning/guarding recursive scans over `$OPENCLAW_STATE_DIR` and large log directories.
