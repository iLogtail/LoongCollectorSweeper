## Summary

Keep open. Current main still does not include the proposed limited-tools troubleshooting section, and this PR is a concrete docs contribution rather than a safe cleanup close. As written, it needs revision because current profile contracts include `minimal`, while the PR text only covers `messaging`, `coding`, and `full`, and it tells users to inspect a `Global tools profile` status row that is not shipped on main.

## Best Possible Solution

Keep #40039 open. Revise the troubleshooting section to cover all current profiles, especially `minimal`; avoid depending on the unshipped `Global tools profile` status row unless #39996 or equivalent status output lands first; and align wording with #39954/#39963 so the troubleshooting page points users to the current public tools/config docs without duplicating stale profile details.
