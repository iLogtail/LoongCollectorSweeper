## Summary

Keep #40039 open. Current main still lacks the proposed symptom-first troubleshooting section, so the PR is not obsolete, but the PR should be revised before merge because current docs/source treat `minimal` as a valid tool profile and current main does not expose the `Global tools profile` status row that the proposed text tells users to inspect.

## Best Possible Solution

Keep #40039 open as a salvageable docs PR. Revise the troubleshooting section to include `minimal`, align the diagnosis steps with status behavior currently shipped on main or explicitly depend on #39996, and keep detailed profile definitions linked to the canonical tools/config docs so the troubleshooting page stays compact and accurate.
