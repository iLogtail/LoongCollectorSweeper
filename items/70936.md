## Summary

Keep open. Current main moved the active PDF fallback path out of `src/media/pdf-extract.ts` and into the bundled `document-extract` plugin, and that active `pdfjs-dist` call still does not pass `standardFontDataUrl`. PR #70936 is stale as submitted, but it contains a useful package-root resolution/test shape for the still-open #51455 bug, so it should be retargeted rather than auto-closed.

## Best Possible Solution

Keep #70936 open and ask the author to rebase/retarget it: move the package-root `standardFontDataUrl` resolver into `extensions/document-extract/document-extractor.ts`, widen that local PDF.js parameter type, and add the focused assertion to `extensions/document-extract/document-extractor.test.ts` so #51455 is fixed on the active plugin-owned fallback path.
