## Summary

Keep open. Current main still lacks the requested PDF.js standard font wiring in the active PDF extraction path. PR #70936 has the right root-cause direction, and Greptile reviewed that narrow patch positively, but its diff targets the old core `src/media/pdf-extract.ts` implementation. Current main delegates PDF extraction to the bundled `document-extract` plugin, where the live `getDocument` call still passes only `data` and `disableWorker`.

## Best Possible Solution

Keep this PR open for maintainer follow-up or consolidation, but require the actual implementation to move into `extensions/document-extract/document-extractor.ts`: resolve `pdfjs-dist/package.json` from the installed package root, pass a trailing-separator filesystem `standardFontDataUrl` into PDF.js `getDocument`, update the local PDF.js type surface, and add focused coverage in `extensions/document-extract/document-extractor.test.ts`.
