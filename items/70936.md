## Summary

Keep open. The underlying PDF.js standard-font issue is still present on current main, but this PR is stale against the current ownership boundary: PDF fallback extraction now dispatches through the bundled document-extract plugin, while the active PDF.js getDocument call in that plugin still omits standardFontDataUrl. The PR contains useful fix/test direction, but it should be retargeted before merge rather than closed as implemented.

## Best Possible Solution

Keep the PR open, but ask for a retargeted patch: resolve pdfjs-dist/package.json from the document-extract plugin runtime, pass the resulting package-root standard_fonts path as standardFontDataUrl in extensions/document-extract/document-extractor.ts, widen the local PdfJsModule type there, and add a focused regression assertion in extensions/document-extract/document-extractor.test.ts. The adjusted PR can remain the implementation for open issue #51455.
