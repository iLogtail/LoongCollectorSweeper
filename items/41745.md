## Summary

Current `main` still matches the bug this PR is trying to fix. `ensureSkillSnapshot()` persists `systemSent: true` on the first turn before the LLM call happens, `get-reply-run.ts` still treats `!systemSent` as the first-turn gate, and `agent-runner.ts` has no compensating post-success `systemSent` persistence path. The same premature write is present in `v2026.4.22`, so this is not implemented on main and is not safe to close.
