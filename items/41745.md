## Summary

Keep open. Current main still has the pre-response persistence path this PR is trying to fix: `ensureSkillSnapshot()` sets and persists `systemSent: true` during first-turn setup, before the LLM response succeeds, and current `agent-runner.ts` does not have an equivalent post-success `systemSent` write. The PR is not implemented on main and is not obsolete, though the PR patch still needs maintainer/author work for the latest queued-followup review concern.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase plus a revised implementation that removes `systemSent` disk persistence from `ensureSkillSnapshot()` and records it only after a successful first-turn model completion across normal payloads, NO_REPLY/payloadless successes, messaging-tool sends, and queued followup/final branches. Add focused regression tests for failed first-call retry and payloadless/followup success so the session does not stay corrupted or permanently first-turn.
