## Summary

Keep open. Current main still does not implement PR #55742's central plugin execute-context feature: plugin factories receive `OpenClawPluginToolContext`, but resolved tools are returned directly and later executed with only `toolCallId`, params, `signal`, and `onUpdate`. The PR discussion shows reviewers pushed the approach toward a safer fifth-argument context that preserves `signal`/`onUpdate`; that remains useful work. The unrelated compaction auth sidecar appears to have already landed separately on main, so the PR likely needs rebasing/splitting rather than closure.

## Best Possible Solution

Keep the item open and ask for a focused rebase that implements only the plugin execute-context seam: add a public `OpenClawPluginToolExecuteContext` type, wrap plugin tools in `resolvePluginTools`, append a frozen identity/session snapshot as the fifth execute argument after `signal` and `onUpdate`, preserve tool own properties and metadata, and add regression tests/docs for context injection and non-leakage. Drop the already-landed compaction auth sidecar from this PR.
