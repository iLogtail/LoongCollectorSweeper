## Summary

Keep PR #72011 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks the proposed streaming-preview guard: the partial reply path sends `parsedFull.text` to `onPartialReply` without a bare `tool_calls` JSON sanitizer, and the existing shared sanitizer only handles XML/scaffold marker forms. The linked bug #70797 remains open, #71954 was closed unmerged, and the review comments on #72011 are merge-review concerns rather than evidence that this PR is obsolete.

## Best Possible Solution

Keep #72011 open for maintainer review. The best path is to land a corrected streaming-preview sanitizer for `src/agents/pi-embedded-subscribe.handlers.messages.ts` plus shared visible-text sanitizer coverage, or fold equivalent behavior into a maintainer-owned fix for #70797 while addressing the open review concerns.
