## Summary

Keep open. Current main partially supersedes this PR via the newer reset-preservation helper and shipped #69419 reset behavior, but it does not fully implement the PR's central resolver guard: `resolveSessionAuthProfileOverride` still rotates from an existing override on new sessions instead of discarding auto-selected overrides and picking the first available profile fresh.

## Best Possible Solution

Keep the PR open for maintainer review, but port only the still-relevant pieces: update `resolveSessionAuthProfileOverride` so new sessions discard auto-selected auth profile overrides and call `pickFirstAvailable()`, add focused tests for auto vs user source, cooldown skipping, and legacy source behavior, and integrate with the current `resolveResetPreservedSelection` helper instead of duplicating older reset-path logic.
