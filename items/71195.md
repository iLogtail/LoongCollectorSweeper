## Summary

Keep open. Current `main` already added the shared/browser realtime Talk pieces, but native macOS Talk still runs the older STT -> `chat.send` -> `chat.history` -> `talk.speak`/local playback loop. Maintainer comments on the issue explicitly narrow the remaining work to macOS client integration and say not to close it.
