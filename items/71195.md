## Summary

Keep #71195 open. Current main already has the gateway/browser realtime Talk path and realtime voice provider SDK surface, but the specific requested macOS native Talk integration is still missing. The maintainer discussion also explicitly says to keep this open and narrow the remaining work to the macOS client consuming the existing realtime session flow, with the ElevenLabs PCM fallback topic split out.

## Best Possible Solution

Keep this issue open as the canonical native macOS Talk realtime follow-up. The likely implementation path is to add a GatewayConnection wrapper for talk.realtime.session, add a native realtime audio/WebRTC session object modeled on ui/src/ui/chat/realtime-talk.ts, wire TalkModeRuntime and overlay state/cancellation into that session, and preserve the current SFSpeechRecognizer -> chat.send -> talk.speak path as fallback. Split any remaining ElevenLabs PCM rejection caching/fallback behavior into a separate focused issue or PR.
