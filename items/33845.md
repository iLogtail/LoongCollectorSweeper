## Summary

Keep PR #33845 open. Current main still uses an in-memory Slack thread participation cache, so the restart-survival behavior proposed by the PR is not implemented. Main now has a disk-backed `plugin-sdk/persistent-dedupe` helper, but Slack does not use it yet, and the related open PRs do not provide a merged Slack consumer migration.

## Best Possible Solution

Keep the PR open. The best maintainer path is to implement Slack thread participation restart persistence on main, preferably by migrating the Slack cache to the existing or accepted shared persistent store helper, updating the prepare/dispatch/action-runtime call sites as needed, and adding cold-restart persistence tests. Close only after a canonical Slack persistence implementation lands or a maintainer explicitly supersedes this PR with another reviewed PR.
