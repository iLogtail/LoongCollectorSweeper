## Summary

Current main still does not implement the requested trusted-proxy shared-secret fallback. The gateway dispatcher still returns trusted-proxy failures before token/password/Tailscale checks, startup validation still rejects trusted-proxy plus shared-token config, tests still assert local direct token attempts are rejected, and docs describe the current fail-closed trusted-proxy contract. The issue discussion also remains active with production-impact reports; the directly linked fix PR #17746 was closed unmerged, while related fallback work #64122 is still only an open PR in the provided context.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide the current fail-closed trusted-proxy contract is final. A complete fallback fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression/e2e tests, and `docs/gateway/trusted-proxy-auth.md`, likely by continuing or superseding #64122.
