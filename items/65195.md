## Summary

Keep open. The issue is recent, concrete, and reports a core macOS LaunchAgent/gateway exec networking failure. Current main still routes default exec to the launchd-managed gateway when no sandbox is active, only reconciles login-shell PATH for gateway-host exec, and keeps macOS app-context execution on the node/system.run path rather than the default gateway exec path. Related #61713 remains open as adjacent SSH/LAN signal, but that does not prove this HTTPS/CalDAV LAN reproduction is fixed or safely duplicate-closeable.

## Best Possible Solution

Keep #65195 open as a core macOS LaunchAgent/gateway exec networking bug. The best maintainer path is a current-main or v2026.4.24 macOS repro with tools.exec.host=gateway, the same sanitized LAN curl from Terminal and OpenClaw exec, gateway status/doctor/config output, and relevant service environment diagnostics. If reproduced, fix the LaunchAgent/gateway exec network-context mismatch or document/reroute through an explicit node/app execution path only if that is the intended product boundary.
