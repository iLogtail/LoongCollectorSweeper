## Summary

Keep this issue open. Current main still passes array-typed `pending.json` / `paired.json` through `loadState` in both device and node pairing, so the reported data-loss path remains plausible. The provided GitHub context also shows an open same-author PR, #63081, with explicit closing syntax for this issue, so the maintainer path is to review or land that PR rather than close the issue early.

## Best Possible Solution

Review #63081, or an equivalent maintainer patch, and land a guarded record-normalization fix in both `src/infra/device-pairing.ts` and `src/infra/node-pairing.ts`. Add a focused regression test that seeds `devices/pending.json` and `devices/paired.json` with `[]` and verifies request-to-approval persistence; mirror or unit-cover the same normalization for node pairing. After the PR merges, let GitHub close this issue through the PR closing syntax.
