## Summary

Keep open. Current `main` and the latest release still deserialize pairing state files without rejecting arrays, so the reported `[]`-vs-`{}` corruption path remains plausible in both device and node pairing. The issue is recent, concrete, and supported by linked PR discussion rather than lacking information.

## Evidence

- **Device pairing still trusts array-typed JSON:** `loadState` still assigns `pending ?? {}` and `paired ?? {}` directly after `readJsonFile(...)`, with no `Array.isArray` or plain-object guard. If `pending.json` or `paired.json` contains `[]`, the array will be kept as state.
  - file: [src/infra/device-pairing.ts:150](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/device-pairing.ts#L150)
  - command: `nl -ba src/infra/device-pairing.ts | sed -n '138,160p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Node pairing has the same unguarded load path:** `node-pairing.ts` uses the same `pending ?? {}` / `paired ?? {}` pattern, so the same array-typed-state bug remains for node pairing too.
  - file: [src/infra/node-pairing.ts:141](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/node-pairing.ts#L141)
  - command: `nl -ba src/infra/node-pairing.ts | sed -n '132,148p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **JSON reader does not validate object shape:** `readJsonFile` only does `JSON.parse(raw)` and returns the parsed value. Arrays therefore pass through unchanged, which makes the pairing loaders responsible for rejecting them.
  - file: [src/infra/json-files.ts:29](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/json-files.ts#L29)

- **Latest release still contains the same code:** The latest release tag `v2026.4.23` still shows the same unguarded `loadState` implementation in both pairing files, so this was not already shipped as fixed.
  - file: [src/infra/device-pairing.ts:150](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/device-pairing.ts#L150)
