## Summary

Keep PR #70918 open. Current main still has the zh-Hans navigation flattened to only `zh-CN/concepts/memory`, while the English source navigation contains a nested Memory group with all seven Memory-related pages. The docs sync pipeline treats zh-Hans as a curated overlay read from `docs/.i18n/zh-Hans-navigation.json`, so the PR addresses a real source-repo docs navigation gap rather than an obsolete or external change.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge after normal docs validation. The best path is to preserve the existing zh-Hans overlay model and restore the localized nested Memory group there, or alternatively make a broader docs-sync change only if maintainers want zh-Hans to stop using a curated overlay.
