## Summary

Keep PR #70918 open. Current main still has the zh-Hans navigation gap the PR targets: the Chinese Sessions/Memory nav only exposes the top-level memory page, while the English source nav has a nested Memory group with the additional memory-builtin, memory-qmd, memory-honcho, memory-search, active-memory, and dreaming pages. The zh-Hans navigation file is a curated source asset used by docs publish sync, so an in-repo docs change is actionable.

## Best Possible Solution

Keep the PR open and either merge it after normal docs validation or apply an equivalent focused patch to docs/.i18n/zh-Hans-navigation.json. The best version should preserve the localized group label 记忆, keep zh-CN paths aligned with the English Memory subtree, and run the usual docs/i18n checks in maintainer CI.
