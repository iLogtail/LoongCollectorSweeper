## Summary

Keep PR #70918 open. Current main still lacks the nested zh-Hans Memory navigation group and only exposes `zh-CN/concepts/memory`, while the English source navigation includes all seven Memory pages. The zh-Hans navigation file is a curated overlay consumed by the docs publish sync, so this is a real docs-source gap rather than an already-shipped or out-of-repo change.

## Best Possible Solution

Keep the PR open and land a narrow docs fix that updates `docs/.i18n/zh-Hans-navigation.json` to match the English Memory group, using the localized `记忆` label. A follow-up parity check for curated zh-Hans navigation would reduce future drift, but the immediate fix belongs in this PR or an equivalent maintainer patch.
