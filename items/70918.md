## Summary

Keep PR #70918 open. The current main branch still has the zh-Hans navigation gap: English navigation has the nested Memory group with all seven memory pages, while `docs/.i18n/zh-Hans-navigation.json` still lists only `zh-CN/concepts/memory` and omits the six additional Memory sub-pages. The PR is narrow, current, and directly addresses a real docs discoverability gap in the curated zh-Hans navigation that the publish sync uses.

## Best Possible Solution

Do not close this PR. The best path is to merge or adapt the scoped zh-Hans navigation update after maintainer docs review, then consider a separate follow-up to reduce future drift between English navigation and the curated zh-Hans overlay.
