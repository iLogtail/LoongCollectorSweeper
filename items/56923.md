## Summary

Keep PR #56923 open. Current main still has the reported audit false positive path: `listKnownNodeCommands()` builds known commands from platform defaults plus configured `allowCommands`, but it does not include `DEFAULT_DANGEROUS_NODE_COMMANDS`. The non-deep security audit then treats any `denyCommands` entry absent from that set as an unknown ineffective deny entry, while new gateway setup intentionally writes the dangerous command list into `gateway.nodes.denyCommands`. The PR is a focused source fix for a real current-main issue, though it should add a regression test before landing.

## Best Possible Solution

Keep the PR open and land a tested version of the same fix: add `DEFAULT_DANGEROUS_NODE_COMMANDS` to `listKnownNodeCommands()`, then add a focused regression case in `src/security/audit-node-command-findings.test.ts` for dangerous commands present only in `gateway.nodes.denyCommands` and run the targeted security audit test lane.
