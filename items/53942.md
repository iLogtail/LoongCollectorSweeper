## Summary

Keep #53942 open. Current main still does not provide user-level per-section prompt overrides or a trust-level setting. The relevant prompt text remains hardcoded for Safety, Tool Call Style, and self-update permissions; the only user-facing override is full prompt replacement. Provider/plugin prompt section overrides exist, but they are limited to provider-owned `interaction_style`, `tool_call_style`, and `execution_bias` sections and do not include Safety or a user config surface.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely core path is a documented, schema-backed user configuration surface under agent defaults and per-agent config for selected prompt sections or a trust policy, with tests proving Safety, Tool Call Style, and self-update permission text can be selectively replaced or relaxed without requiring full prompt replacement. It should be coordinated with the broader #25493 template-editing discussion rather than closed as already implemented.

## Risks / Open Questions

- Treating `agents.defaults.systemPromptOverride` as sufficient would force users to copy and maintain the entire OpenClaw prompt just to relax one section, which is materially broader than the request.
- Closing only as a duplicate of #25493 could lose the narrower product question here: whether OpenClaw should expose safe user-level trust or section override knobs for Safety, Tool Call Style, and self-update permission text.
