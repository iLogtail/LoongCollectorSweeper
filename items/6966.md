## Summary

Keep open. Current main has ordered model failover, auth-profile rotation, and model-scoped rate-limit cooldowns, but it still does not implement the requested dynamic model-tier switching policy with health-aware routing, downgrade thresholds, or gradual upgrade-back behavior. The aashari fork mentioned in the discussion is useful prior art, not upstream implementation.

## Best Possible Solution

Keep the issue open for maintainer product/design work in the core model routing and auth-profile fallback layer. A good upstream implementation would define a documented routing policy such as failover versus health-aware/round-robin, add per-provider/model/profile health state with success/failure recovery semantics, expose observable downgrade and upgrade-back events, update config schema/docs, and add regression tests. Treat the aashari fork as useful prior art rather than shipped upstream behavior.
