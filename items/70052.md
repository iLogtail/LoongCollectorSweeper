## Summary

Keep #70052 open. Current main still uses the upstream `@mariozechner/pi-coding-agent` token estimator in the compaction and preemptive-compaction paths, while the existing CJK-aware `estimateStringChars` helper is only wired into other paths such as context pruning. The provided GitHub context also shows this was triaged as valid/new, with #70112 still open and #70081 closed unmerged, so the reported core bug is not shipped on main.

## Best Possible Solution

Keep the issue open until the compaction path is actually changed on main. The likely fix belongs in core agent compaction: introduce a CJK-aware message token estimator using `src/utils/cjk-chars.ts`, replace all compaction and preemptive-compaction `estimateTokens` call sites that estimate user-visible prompt/history text, and add regression tests for Japanese/Chinese/Korean messages in both normal and preemptive compaction paths. PR #70112 is the relevant follow-up to review or revise.
