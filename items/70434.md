## Summary

Keep PR #70434 open. Current main does not implement the proposed subagent gateway lifecycle resilience: spawn still performs a single gateway `agent` call, announce finalization still uses direct best-effort `sessions.patch` / `sessions.delete` calls with 10s timeouts, and archive sweep cleanup still retries only on the next sweep after any delete failure. The PR is relevant, but the provided review context shows it needs rework before it is a conservative maintainer cleanup candidate.

## Best Possible Solution

Keep the PR open and ask for a rework against current main: use a non-mutating gateway readiness probe or a narrower retry at the actual failing calls, keep all local validation before gateway readiness checks, centralize the transient lifecycle classifier if used in multiple agent files, and add targeted tests for spawn, announce finalize, and sweep cleanup across the specific lifecycle error strings claimed in the PR body.
