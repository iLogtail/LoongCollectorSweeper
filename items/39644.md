## Summary

Keep open. Current main still has both Windows bugs the PR targets: PowerShell completion install is recognized as a valid shell but still falls through to the unsupported branch, and Windows time-format detection still invokes literal `powershell` instead of the existing `resolvePowerShellPath()` helper that prefers `pwsh`. The PR remains a focused, salvageable bug fix, though it needs a port to the current `completion-runtime.ts` layout.

## Best Possible Solution

Keep this PR open and port the small fix to current main: add the PowerShell install/profile branch and dot-source formatting in `src/cli/completion-runtime.ts`, use `resolvePowerShellPath()` plus non-interactive flags in `src/agents/date-time.ts`, and add regression tests covering PowerShell profile installation, reload/source syntax, and PS7 path selection.
