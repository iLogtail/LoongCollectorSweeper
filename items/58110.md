## Summary

Keep #58110 open. Current main has real building blocks for chat UX, multi-agent routing, per-agent session history, and sub-agent/background collaboration, but it does not implement the central requested product shape: an agent-as-contact chat UI with visible agent status plus first-class group chats where users invite multiple agents into one shared conversation. Related ClawSweeper context also shows narrower multi-agent gaps still open (#22774, #43235, #43750), so closing this broad UX request would be premature.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. The best path is to define concrete acceptance criteria for a Control UI agent contact/sidebar experience, per-agent session switching/history, visible agent status, and a first-class multi-agent conversation or invitation model. Reuse existing multi-agent routing, sessions.list, and sub-agent/thread primitives where they fit, while linking narrower follow-ups for sender attribution (#22774), per-agent lanes (#43235), and command targeting (#43750).
