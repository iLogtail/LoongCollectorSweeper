## Summary

Keep open. Current `main` already has core multi-agent primitives, but it does not fully implement the issue's requested chat-style multi-agent UX or shared multi-agent conversation model.

## Evidence

- **Core multi-agent support exists already:** Current docs describe isolated agents with separate workspaces, auth, and session stores, plus routing bindings between channel accounts and agents.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/concepts/multi-agent.md#L8)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current collaboration model is isolated subagents, not a shared group chat:** Sub-agents run in their own sessions and announce results back to the requester chat. The documented `sessions_spawn` flow is separate-session orchestration, not multiple agents co-reading and co-writing one shared conversation.
  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/tools/subagents.md#L10)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **UI navigation is session-oriented, only partially agent-oriented:** The Control UI groups session options by agent in the session selector, which covers some multi-agent navigation, but this is not the requested contacts-style sidebar or per-agent chat-window model.
  - file: [ui/src/ui/chat/session-controls.ts:475](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/chat/session-controls.ts#L475)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Requested chat-style agent switcher is not implemented in the chat view:** `app-render.ts` passes `agentsList`, `currentAgentId`, and `onAgentChange` into `renderChat`, but in `ui/src/ui/views/chat.ts` those identifiers only appear in the prop type block and are not rendered or used by the chat view.
  - file: [ui/src/ui/app-render.ts:2287](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/app-render.ts#L2287)
  - command: `rg -n "agentsList|currentAgentId|onAgentChange|onNavigateToAgent" ui/src/ui/views/chat.ts`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
