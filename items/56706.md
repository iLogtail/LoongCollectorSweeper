## Summary

Keep open. Current main still uses regex-based TTS directive parsing and deliberately accepts inline and bare TTS tags, with tests and docs describing that behavior. The PR is not obsolete, but it changes a public directive contract and bundles unrelated Slack/config-validation changes, matching the maintainer comment that it should not merge as-is.

## Best Possible Solution

Keep this PR open but require a narrower revision: split or drop the unrelated Slack/config-validation changes, make the TTS PR only about directive semantics, and have maintainers decide whether inline/bare TTS tags remain supported or intentionally migrate to standalone-line-only behavior. If standalone semantics are accepted, update docs/tools/tts.md, src/tts/directives.test.ts, test/helpers/plugins/tts-contract-suites.ts, and changelog coverage alongside the parser change.
