## Summary

Keep open. Current `main` still has the reported leak path: the user-visible text stripper only recognizes downgraded `[Tool Call ...]` / `[Tool Result ...]` / `[Historical context ...]` markers, so a bare streamed `{"tool_calls":[...]}` payload is not removed before delivery.

## Evidence

- **Bare `tool_calls` JSON is not matched by the current stripper:** `stripDowngradedToolCallText()` returns early unless the text contains `[Tool Call|Result ...]` or `[Historical context ...]`. A bare `{"tool_calls":[...]}` payload does not satisfy that guard, so it is returned unchanged.
  - file: [src/shared/text/assistant-visible-text.ts:328](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/shared/text/assistant-visible-text.ts#L328)
  - command: `nl -ba src/shared/text/assistant-visible-text.ts | sed -n '324,334p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Direct streamed block chunks rely on that stripper before user delivery:** The direct embedded subscribe path explicitly says it strips downgraded tool-call text, but it only calls `stripDowngradedToolCallText(...)` on outgoing chunks. Because the bare-JSON case is missed, that text can still be emitted to channels.
  - file: [src/agents/pi-embedded-subscribe.ts:620](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/pi-embedded-subscribe.ts#L620)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '616,623p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **ACP visible-text accumulation also forwards raw text deltas:** `createAcpVisibleTextAccumulator().consume()` appends incoming text without any assistant-visible sanitization pass, and the ACP projector forwards `text_delta` content directly into live buffers. That matches the issue discussion's suspicion that a rendering/stream-assembly path can surface raw tool payload text.

- **Existing tests cover marker-style downgraded tool text, not bare `{"tool_calls":...}` payloads:** Current tests exercise `[Tool Call ...]`, `[Tool Result ...]`, and `[Historical context ...]` cases. A repo search for a literal `{"tool_calls"` test case returned no matches.
