## Summary

Keep open. Current main still matches the reported failure mode: the final visible-text sanitizer does not strip bare OpenAI-style `{"tool_calls":[...]}` JSON, and the Telegram/direct partial reply path can pass cleaned stream text to `onPartialReply` without a streaming-aware tool-call JSON sanitizer. The linked PR #72011 appears to be the right active fix path, but it is open/unmerged in the provided context, so the bug is not implemented on main yet.

## Best Possible Solution

Keep #70797 open as the canonical bug until PR #72011 or an equivalent fix lands. The fix should add bare `tool_calls` JSON stripping for final visible text and a streaming-aware sanitizer before Telegram/direct `onPartialReply`, with regressions in `src/shared/text/assistant-visible-text.test.ts` and `src/agents/pi-embedded-subscribe.handlers.messages.test.ts`.

## Evidence

- **Issue discussion confirms a current product bug:** The reporter provided two recurring Telegram direct-chat leaks after background `exec` followed by `process poll`, and the triage comment identifies the same sanitizer/streaming-preview paths as the likely root cause.
