## Summary

Keep open. Current `main` still does not contain the streaming-preview guard needed for #70797: the shared visible-text sanitizer still ignores bare `{"tool_calls":[...]}` JSON, and the Telegram/direct partial reply path can forward accumulated text to `onPartialReply` before final cleanup. The linked #72011 PR appears to be the right active fix attempt, but it is open/unmerged in the provided context, so this issue should remain open until that fix or equivalent lands.

## Best Possible Solution

Keep #70797 open and land or review an implementation equivalent to #72011: add a streaming-aware sanitizer/accumulator before `onPartialReply` that withholds incomplete standalone JSON, drops complete OpenAI-style `tool_calls` payloads, preserves ordinary visible JSON/code examples, and adds regression coverage in `src/agents/pi-embedded-subscribe.handlers.messages.test.ts` plus shared visible-text sanitizer tests. After the fix is merged, close #70797 as implemented with the merge commit and release evidence.
