## Summary

Keep open. Current main still initializes QMD collection adds as `--glob` first, and the fallback only changes to `--mask` when QMD explicitly rejects the option. The PR's silent-ignore regression remains plausible and is not implemented on main.

## Best Possible Solution

Keep this PR open or supersede it with one canonical QMD compatibility PR. The best fix is to prefer `--mask` for fresh `qmd collection add` calls, preserve fallback to `--glob` for older binaries, and add a regression test that covers silent `--glob` ignore for custom collection patterns.
