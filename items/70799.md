## Summary

Keep open. Current main still lacks the PR's proposed outer guards on `markBlueBubblesChatRead` and `sendBlueBubblesTyping`; the only current short-circuit is inside `sendBlueBubblesChatEndpointRequest` after client/account resolution. The latest release tag inspected also has the same shape. This is a focused bundled BlueBubbles channel bug-fix PR, not ClawHub/external work, and the provided related item is an unrelated BlueBubbles SQLite contact lookup PR rather than a canonical duplicate or superseding fix.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge the narrow outer guards with a regression test that covers the reported cache/account-id mismatch, or replace it with a slightly stronger BlueBubbles optional-private-API handling fix that skips typing/read when Private API is known disabled and keeps those best-effort failures from affecting replies.
