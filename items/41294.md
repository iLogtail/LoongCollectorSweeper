## Summary

Keep #41294 open. Current main supports configured automatic model fallback, but it does not expose `fallbackMode`, a confirmation policy, or a chat prompt before switching to the next fallback candidate.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete fix would add a typed and documented fallback policy, validate it in the config schema/UI, and thread it into the core fallback runner and chat surfaces so users can approve, deny, or choose a fallback before the runtime switches models.
