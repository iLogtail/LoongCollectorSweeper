## Summary

Keep open. Current main still has the central `channels.telegram.retry.jitter` failure mode: Telegram uses the shared retry schema, and that schema still accepts only numeric `jitter` values with no boolean coercion/migration. Some adjacent hardening exists for systemd restart limits and SecretRef command-path behavior, but the reported config-triggered gateway startup failure is not implemented on main, and the related PR #52137 was closed unmerged in the provided GitHub context.
