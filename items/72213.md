## Summary

Keep PR #72213 open. Current main still has the Discord block-streaming duplicate progress path: tool progress is deduplicated only against the immediately previous exact string, while active callbacks can emit the same logical tool as both `tool: session_status` and `session_status`. The PR is a narrow Discord display-layer fix with a targeted regression test, and no canonical main-branch implementation makes it obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and CI, or land an equivalent Discord plugin change that deduplicates preview tool-progress lines by logical tool name rather than exact adjacent text, with the regression test in `extensions/discord/src/monitor/message-handler.process.test.ts`.
