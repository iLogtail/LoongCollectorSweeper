## Summary

Keep PR #72213 open. Current main still has the reported Discord block-preview duplicate tool-progress path: `pushPreviewToolProgress` deduplicates only the immediately previous exact string, while active callbacks can emit the same tool once as `tool: session_status` and again as `session_status`. The PR is a focused display-layer fix with a targeted regression test for linked open issue #72205, and I found no main-branch implementation or canonical superseding item that makes it obsolete.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The best code path is to land this narrow Discord plugin fix, or an equivalent change, with the regression test in `extensions/discord/src/monitor/message-handler.process.test.ts` so preview tool-progress lines deduplicate by logical tool name rather than exact adjacent text.
