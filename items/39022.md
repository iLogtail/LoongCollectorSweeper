## Summary

Keep #39022 open. Current main still implements `/model` as a persistent session-scoped model override, but does not implement `/model <name> --turns N`, per-model turn-limit config, warning thresholds, remaining-turn state, or automatic reversion to the default model. The prior ClawSweeper review in the issue reached the same keep-open conclusion, and the current main SHA still matches that behavior.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete fix likely belongs in core `/model` session behavior: extend directive/native command parsing for `--turns`, add durable session budget and revert-target state, add per-model limit schema/docs if desired, decrement only on intended completed turns, warn near expiry, clear the user model override back to the configured default, and cover parser, session persistence, model-selection, warning, and auto-revert behavior with focused tests.
