## Summary

Keep #54354 open. Current main has a narrower `openclaw.install.minHostVersion` floor that blocks/loads plugins after the selected package has already been fetched, but it does not implement the PR's central behavior: resolving npm installs/updates to the latest version compatible with the running OpenClaw version via `engines.openclaw`, nor an `--ignore-engine` escape hatch. The PR itself also has unresolved review concerns, so it should not be merged as-is.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If OpenClaw wants this behavior in core, implement it around the current install/update architecture and documented `openclaw.install.minHostVersion` contract or a clearly documented new `engines.openclaw` contract; honor exact versions and dist-tags, validate untrusted registry metadata, avoid custom semver edge cases where possible, preserve ClawHub/npm fallback and install-safety checks, and add focused install/update/docs tests.
