## Summary

Current main still leaves `/bot-clear-storage` on the QQBot pre-dispatch slash-command path because its registration lacks `requireAuth: true`. The PR addresses a live authorization gap with the same framework-auth pattern already shipped for `/bot-approve` in #70706, and the current test coverage only asserts `bot-approve` is in the auth-gated framework registry.

## Best Possible Solution

Keep the PR open and land the focused fix, or cherry-pick an equivalent change: add `requireAuth: true` to `bot-clear-storage`, add the framework-registry regression assertion, and keep the changelog entry with the required contributor attribution. This should close the same class of QQBot pre-dispatch auth gap as #70706 without broadening core behavior.
