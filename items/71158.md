## Summary

Current main still has the QQBot `/bot-clear-storage` gap that PR #71158 targets. The command is registered without `requireAuth: true`, so it remains in the pre-dispatch command map, while the handler can delete all files under the QQBot downloads directory after only a private-chat guard. The related merged `/bot-approve` fix (#70706) confirms the intended pattern but does not cover this command.

## Best Possible Solution

Keep #71158 open and either merge or recreate the focused fix: add `requireAuth: true` to `bot-clear-storage`, add the `getFrameworkCommands()` regression assertion, and fix the changelog attribution before landing. The existing `/bot-approve` auth fix is useful precedent, not a replacement for this command-specific patch.
