## Summary

Keep #71158 open. Current main still registers QQBot `/bot-clear-storage` without `requireAuth: true`, so the command remains outside the auth-gated framework registry while its `--force` path deletes files from the QQBot downloads directory. The related merged #70706 fixed the same class for `/bot-approve`, but it does not cover this command.

## Best Possible Solution

Keep the PR open and land this patch or an equivalent maintainer-authored fix: add `requireAuth: true` to `bot-clear-storage`, keep the `getFrameworkCommands()` regression assertion, and retain a user-facing changelog/security note. The related #70706 pattern is the right precedent, but it does not eliminate the remaining `/bot-clear-storage` gap.
