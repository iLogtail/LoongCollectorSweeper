## Summary

Keep open. Current main still has a host-side pre-action path before `openclaw channels login --channel openclaw-weixin` reaches the plugin `auth.login` hook, including broad plugin registry loading inherited from the generic `channels` command. I did not find a #62120-specific fix, test, changelog entry, or canonical closed duplicate proving the Weixin pre-QR hang is resolved.

## Best Possible Solution

Keep #62120 open for a focused core/plugin-boundary fix. The likely maintainer path is to make `channels login` avoid broad pre-action plugin loading, then load only the requested/configured login-capable channel through the existing scoped plugin resolution path and add a regression test that `channels login --channel openclaw-weixin` reaches `plugin.auth.login` without unrelated plugin startup or doctor/preflight work blocking first.
