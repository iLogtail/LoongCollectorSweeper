## Summary

Keep #59151 open. Current main partially addresses the report by treating human/PluralKit @everyone/@here payloads as mentions, but role mentions are still only counted as generic mention metadata and are not matched against bot or agent role membership for the mention gate.

## Best Possible Solution

Keep the issue open and implement the remaining Discord role-mention behavior in the Discord plugin preflight path: resolve or cache the bot member role IDs for the guild, compare message.mentionedRoles against those IDs before mention gating, thread the result into resolveDiscordMentionState/wasMentioned, and add focused tests for role-only mentions, non-member role mentions, @everyone/@here behavior, bot-authored broadcasts, and ignoreOtherMentions interactions. Update Discord channel docs and config metadata only if maintainers want an opt-in/opt-out guard.
