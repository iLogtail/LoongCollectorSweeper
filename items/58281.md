## Summary

Keep open. Current `main` still does not expand `$VAR` or `${VAR}` in shared user-path resolution, so the reported behavior is not implemented on `main`. The PR also still needs maintainer judgment because a review comment identified a plausible regression around missing `HOME` fallback semantics.

## Evidence

- **Current path resolution still only handles tilde:** `resolveHomeRelativePath` trims input, expands only `~` prefixes, and otherwise returns `path.resolve(trimmed)`. `resolveOsHomeRelativePath` follows the same pattern. There is no env-var expansion helper on `main`.
  - file: [src/infra/home-dir.ts:102](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/infra/home-dir.ts#L102)
  - command: `nl -ba src/infra/home-dir.ts | sed -n '1,220p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Shared config/state paths depend on that resolver:** `resolveStateDir`, `resolveCanonicalConfigPath`, and `resolveConfigPath` route overrides through `resolveUserPath`, which delegates to `resolveHomeRelativePath`. That means env-var expansion would affect real config/state surfaces, not dead code.
  - file: [src/config/paths.ts:91](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/config/paths.ts#L91)
  - command: `nl -ba src/config/paths.ts | sed -n '1,220p'`

- **Mainline tests do not cover env-var expansion:** `home-dir` tests on `main` cover blank input, relative/absolute paths, and tilde expansion only. There are no landed tests for `$VAR` or `${VAR}` path expansion, matching the missing implementation.
  - file: [src/infra/home-dir.test.ts:162](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/infra/home-dir.test.ts#L162)

  - file: [docs/install/gcp.md:263](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/docs/install/gcp.md#L263)

That means the PR is not an obvious safe auto-close candidate even though the feature request is plausible.
