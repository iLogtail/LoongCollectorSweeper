## Summary

Current `main` no longer gives the health monitor a path to restart or emit events for channels/accounts removed from config. The runtime snapshot is rebuilt from the live config on each check, the monitor iterates only that snapshot, and health-monitor config keys now exist for global and per-channel/account opt-out.

## Evidence

- **Health monitor only scans current runtime snapshot:** `startChannelHealthMonitor()` calls `channelManager.getRuntimeSnapshot()` and iterates `snapshot.channelAccounts`; it does not walk any persistent stale registry.
  - file: [src/gateway/channel-health-monitor.ts:111](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/gateway/channel-health-monitor.ts#L111)

## Risks / Open Questions

- I did not run a live reproduction because this review was constrained to read-only inspection.
- There is no explicit regression test named for the exact removed-from-config scenario; the conclusion rests on the current control flow in `getRuntimeSnapshot()` and `isHealthMonitorEnabled()`.
- `pnpm docs:list` could not be run in this environment because `pnpm` was not installed (`/bin/bash: pnpm: command not found`).

## Close Comment

Closing as implemented on `main` after Codex review. The current gateway health monitor only iterates `channelManager.getRuntimeSnapshot()` (`src/gateway/channel-health-monitor.ts:111`), and that snapshot is rebuilt from the live config via each plugin’s `listAccountIds(cfg)` (`src/gateway/server-channels.ts:645`), so channels/accounts removed from `openclaw.json` are no longer present for the monitor to act on. Current `main` also fails closed if account resolution is missing (`src/gateway/server-channels.ts:222`) and exposes the health-monitor controls that were missing in the report: global disable with `gateway.channelHealthCheckMinutes: 0` plus per-channel/account overrides documented in `docs/gateway/health.md:34`. This appears fixed on `main`; if you still hit it on a newer build, please open a fresh issue with current version details and a repro against that build.
