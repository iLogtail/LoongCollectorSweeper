## Summary

Keep open. Current `main` still has the reported bug: the shared chat-target parser accepts non-decimal `chat_id:` values via `Number.parseInt(..., 10)` and only checks `Number.isFinite`, so a hex-like identifier such as `2ecba0e20f3a4299b0efb228a5990731` still truncates to `2`. The PR's original edit location is stale after a refactor into shared channel-target code, but the underlying iMessage misrouting/allowlist problem is not implemented on `main`.
