## Summary

Keep open. Current main still lacks inbound Feishu interactive-card parsing and raw-card context propagation, while PR #56795 contains useful implementation work for that gap. It is not implemented on main or clearly superseded, though the PR still needs maintainer review because the final review thread flags a RawBody compatibility regression.

## Best Possible Solution

Keep #56795 open or explicitly consolidate it into the canonical Feishu interactive-card tracker/work. The best implementation is a shared Feishu plugin parser used by inbound events, fetched/quoted messages, and thread history, with verified raw-card API request handling, backward-compatible RawBody semantics, and tests for header text, markdown/div content, buttons/actions, i18n variants, fallback card formats, and raw JSON context propagation.
