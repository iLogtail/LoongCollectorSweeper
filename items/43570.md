## Summary

Keep open. Current main still has the Discord `thread-reply` alias gap: `send` honors `media`/`path`/`filePath`, but `thread-reply` still reads only `media`, and the generic runner does not normalize `path`/`filePath` into `media` for that action. The linked PR #43574 remains the relevant unmerged fix path, not shipped behavior.

## Best Possible Solution

Keep #43570 open and land #43574 or an equivalent current-main patch: update `extensions/discord/src/actions/handle-action.guild-admin.ts` so `thread-reply` reads `media ?? path ?? filePath`, then add regression coverage for `filePath`, `path`, and `media` precedence through the Discord message action path while preserving media root/read-file forwarding.
