## Summary

Keep PR #70971 open. Current main has a newer guard that likely fixes the narrow exec-only bundled MCP leak by skipping MCP materialization when `toolsAllow` contains only core tool names, but the requested final allowlist behavior is not fully implemented: LSP bundled tools are still appended without a runtime `toolsAllow` pass, and materialized MCP catalogs are not narrowed to individual allowed MCP tool names.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer patch that applies runtime `toolsAllow` after bundled MCP/LSP materialization, expands the `bundle-mcp` plugin key intentionally, supports exact individual MCP and LSP tool names, and adds focused regression coverage for `toolsAllow: ["exec"]`, one allowed MCP tool, `toolsAllow: ["bundle-mcp"]`, and configured bundle LSP.
