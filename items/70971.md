## Summary

Keep #70971 open. Current main has a partial MCP gating mitigation for non-bundled `toolsAllow` values, but it still does not make runtime `toolsAllow` authoritative over the final combined tool list; bundled LSP tools and namespaced bundled MCP allowlists can still bypass the final allowlist shape this PR is trying to enforce.

## Best Possible Solution

Keep the PR open and land a rebased version or equivalent fix that applies runtime `toolsAllow` after bundled MCP/LSP tools are materialized, or filters `filteredBundledTools` by the same allowlist before appending. Add a focused regression proving scheduled runs with `toolsAllow` do not serialize unallowed bundled MCP/LSP schemas, while preserving current no-op behavior when `toolsAllow` is empty or unset.
