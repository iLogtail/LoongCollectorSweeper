## Summary

Keep open. Current main does not implement or supersede the PR's central Discord/ACP surface overhaul: the MessageClass/surface-policy/receipt/E2E-harness machinery is absent, the current ACP parent relay still emits snippet-style system events, and Discord thread binding lifecycle still removes stale bindings rather than preserving ended bindings for respawn. The PR is draft, very broad, and not landable as-is, but it contains real product work and explicitly calls out remaining strict live verification, so it should be reworked or superseded by narrower maintainer-reviewed PRs rather than auto-closed under the allowed cleanup reasons.

Keep the item open, but require maintainer triage before any merge path. The best product/code path is to split this draft into narrow PRs: first the core message classification and delivery-policy contract with focused tests, then Discord thread-bound routing/identity/respawn changes, then delivery receipts/tools, then live E2E harness/docs. Remove fork-only artifacts such as custom register/merge notes/GitNexus guidance from the upstream PR, rebase onto current main, and complete the strict live Discord verification checklist before considering merge.
