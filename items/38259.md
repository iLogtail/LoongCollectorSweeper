## Summary

Keep open. Current main still loads dotenv values literally: `src/infra/dotenv.ts` parses with `dotenv.parse(...)` and writes each parsed `value` directly into `process.env` without any `${VAR}` or `$VAR` expansion step. The related fix PR #38254 was closed unmerged, and I found no replacement implementation or regression coverage on main.
