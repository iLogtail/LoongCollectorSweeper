## Summary

Keep open. Current `main` still has the same failure path for Control UI cron jobs on local-only/webchat-only setups: new cron jobs default to announce delivery, missing delivery mode still resolves to announce/`last`, and unresolved announce delivery still fails with `Channel is required (no configured channels detected).` There is evidence for opt-out/no-delivery support, but not that the reported default behavior was fixed.

## Evidence

- **Control UI still defaults new cron jobs to announce delivery:** `DEFAULT_CRON_FORM` keeps isolated agent-turn cron jobs on `deliveryMode: "announce"` with `deliveryChannel: "last"` and `deliveryBestEffort: false`. That matches the issue's default UI path, not a fixed `none` default.
