## Summary

Keep #38420 open. Current main has addressed several audit bullets, but the full Control UI dashboard audit is still not implemented: Instances remains presence-list-only, Sessions still has repeated per-row advanced controls and no copy affordance in this table, Config still exposes adjacent Save/Apply/Update semantics, and dashboard-wide R/S/? shortcut/help and feedback/activity UX are not complete. The linked #39780 is a narrower open Config hardening issue, not a superseding tracker for this full-dashboard audit.

## Best Possible Solution

Keep #38420 open unless a maintainer intentionally splits it. The best path is to create or implement focused follow-ups for Instances per-instance actions and clearer presence affordances, Sessions key copy/truncation and less repetitive advanced controls, Config action semantics, dashboard shortcut/help UX, and save/delete/apply feedback or activity history. Treat #39780 as only the narrower Config hardening follow-up, not a superseding full-dashboard tracker.
