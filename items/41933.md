## Summary

Keep #41933 open. Current main still builds the chat session dropdown from `sessions.list` rows plus the active session; it uses `agents.list` only for labeling existing agent-session groups, not to add configured agents with no session. The linked PR #41955 appears to track the right fix, but it remains open in the provided GitHub context, and #42041 was closed unmerged.
