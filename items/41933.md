## Summary

Keep #41933 open. Current main still builds the Control UI chat session dropdown from persisted `sessions.list` rows plus the active session; `agents.list` is loaded and used for labeling existing agent session groups, but it is not merged into the option set to show configured agents with no stored session. The provided related context shows #41955 and #48671 as open proposed fixes and #42041 closed unmerged, so the user-visible request is not shipped on main.

## Best Possible Solution

Keep the issue open and land a corrected Control UI fix in the session-option construction path, likely `resolveSessionOptionGroups()`: merge configured `agents.list` entries with existing `sessions.list` rows, synthesize valid main-session keys for agents without rows, avoid malformed keys when main session defaults are already canonical, dedupe against real sessions, and add regression tests for configured agents with no sessions plus custom main-key/default-agent cases. #41955 or #48671 can be the vehicle if updated to satisfy those constraints.
