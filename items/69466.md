## Summary

Keep open. #69466 is recent and concrete, and current main still has silent pre-output work in `openclaw security audit`, including a 30s gateway SecretRef resolution path before any JSON or human output is written. Current main has useful cold-audit/plugin-index improvements, but I did not find a direct fix that proves the reported no-output hang is obsolete.

## Best Possible Solution

Keep #69466 open. The best fix is to make `openclaw security audit` bounded and observable: avoid or shorten the 30s read-only gateway SecretRef wait, prefer fast local/read-only fallback when safe, emit progress or phase timing before long waits for human output, keep JSON machine-clean, and add regression coverage for a slow `secrets.resolve`/collector path so the command returns an actionable diagnostic within the expected few seconds.
