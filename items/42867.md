## Summary

Keep #42867 open. Current main has only daily/idle session reset policy fields and no session.reset.maxTurns, turn-count freshness check, or heartbeatRefreshesSession/lastConversationAt-style config. There is partial related work that keeps route/inbound metadata from bumping updatedAt, plus optional heartbeat isolated sessions, but the requested absolute turn-count TTL remains missing and #43738 tracks a distinct adaptive daily+idle mode rather than this request.
