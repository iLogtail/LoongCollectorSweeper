## Summary

Keep #42867 open. Current main has implemented the heartbeat-aware staleness half by separating session start/interaction freshness from `updatedAt`, but it still has no `session.reset.maxTurns`, turn-count TTL, or equivalent reset policy. The linked #43738 is a distinct adaptive daily+idle request, and the broader #45390 context is not a shipped implementation of this narrower turn-count TTL request.

## Best Possible Solution

Keep this open as a core session-lifecycle feature unless maintainers intentionally consolidate it into a canonical TTL issue. A complete fix should add a documented/validated turn-count reset policy, wire it into session freshness or rollover, preserve the existing heartbeat-aware freshness behavior, and cover config schema/help/docs plus reset regression tests.
