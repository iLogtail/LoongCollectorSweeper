## Summary

Keep open. Current main still treats inbound audio as media-understanding transcription text, not native audio prompt input. The model/config surface only advertises text/image input, the embedded runner only forwards images to the model prompt, and OpenRouter's bundled media-understanding contract is image-only. This request also needs a core/model-runtime seam, so it is not a clean ClawHub-only plugin close.
