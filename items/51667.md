## Summary

Keep open. Current main has only partial prerequisite support for declaring audio/video model input, but inbound WhatsApp/Telegram voice notes still flow through transcription, media-understanding audio is still modeled as `audio.transcription`, and the embedded runner still submits only text plus optional images to the active model. The related PR #48900 is relevant and should inform the fix, but it is unmerged/not merge-ready, so this issue remains a valid tracker for the missing core runtime behavior.

## Best Possible Solution

Keep #51667 open as the user-facing native-audio tracker. The best path is a core/model-runtime change, likely coordinated with #48900: define a typed native audio content contract in the prompt path, add opt-in config such as `tools.media.audio.native` with STT fallback/audit metadata, wire WhatsApp/Telegram voice-note handling through it, and cover the behavior with focused source, config, and channel tests plus docs.
