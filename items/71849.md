## Summary

Keep #71849 open. Current main still routes Voice Call realtime memory/context questions through the shared full `openclaw_agent_consult` embedded-agent path, has no `realtime.fastContext` config or manifest/docs surface, and the realtime call handler still drops non-final transcript chunks. The open related PR #72237 is related consult tuning, but it does not supersede the requested fast memory/session-context hot path.

## Best Possible Solution

Keep the issue open and implement it in the Voice Call plugin: add a default-off `realtime.fastContext` config/manifest/docs surface, accumulate and normalize partial user transcript chunks for context-trigger detection, call the active memory search manager with a short hard deadline and `sources`/`maxResults`, inject either top snippets or an explicit miss into the realtime session, and retain `openclaw_agent_consult` as the slower fallback for deeper work.
