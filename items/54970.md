## Summary

Keep open. Current `main` still has the active `exec host=node` path unconditionally calling `system.run.prepare`, while the macOS companion node surface still does not implement that command. The PR's old `nodes run` / `nodes invoke` call sites are stale on current `main`, but the underlying bug remains in the live exec path and is not implemented on `main`.
