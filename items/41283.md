## Summary

Keep #41283 open. Current main still intentionally returns `INVALID_REQUEST: unknown agent id` when `tools.catalog` is called with an unconfigured `agentId`, and the issue does not include the config, session key, requested agent id, logs, or install path needed to determine whether this was a stale Control UI selection, a misconfigured agent reference, a web-search provider setup issue, or a fixed runtime bug. The helpful comment about a stale/misconfigured agent reference is plausible, but not enough to close this regression report yet.

## Best Possible Solution

Keep the issue open and ask for a fresh reproduction on current OpenClaw with the `tools.catalog` request parameters, `openclaw agents list`, redacted `agents.list` and `tools` config, the active session key, selected channel/plugin, and gateway logs around the failed request. If the failure is just a stale or deleted agent id, the product path is better UX/diagnostics in Control UI or doctor; if the session truly has `tools.profile: full` but only messaging tools, investigate `resolveEffectiveToolPolicy` inputs for per-agent, provider, group, or session overrides.
