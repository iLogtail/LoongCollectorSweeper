## Summary

Keep open. Current `main` still intentionally returns `INVALID_REQUEST` with `unknown agent id` for `tools.catalog` when the caller supplies an unknown `agentId`, and the current tests lock that behavior in. I found UI fallback handling for a failed runtime catalog, but that only downgrades the Control UI tools panel to a built-in list; it does not prove the reported tool-availability regression is fixed. With no linked fix, no changelog entry clearly matching this report, and no safe runtime reproduction path in this read-only checkout, there is not strong enough evidence to close conservatively.
