## Summary

PR #65051 is a focused, valid UI fallback for the old workspace-skill grouping bug, but current main already fixes the linked user-visible issue at the status payload layer and includes regression coverage. Workspace skills now report `source: "openclaw-workspace"` with `bundled: false`, so the existing Control Panel grouping renders them under Workspace Skills.

## Best Possible Solution

Close PR #65051 as implemented on main. Keep the shipped status-layer fix and backend regression test from v2026.4.22; only reopen or file a narrower follow-up if maintainers want extra defensive UI contract hardening beyond the fixed user-visible bug.
