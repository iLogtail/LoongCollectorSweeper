## Summary

Keep open. Current main partially implements the request via per-agent TTS config, but the central remaining Talk/Voice tab agent/session switching feature is still missing. The Android Voice tab exposes mic, speaker, and Talk controls without a session/agent picker; TalkMode still routes chat.send through a single mainSessionKey; and chat session switching only updates ChatController, not TalkMode. This matches the maintainer comment on the issue saying the TTS half landed but the Talk/Voice UI path remains open.

## Best Possible Solution

Keep the issue open and scope the remaining work narrowly: add an agent/session selector to the Talk/Voice surface, wire that selection through MainViewModel/NodeRuntime into TalkModeManager and manual voice chat.send routing, and cover that the selected agent session key is used. Do not rework the already-shipped agents.list[].tts TTS config path except as needed to consume the active Talk/Voice agent.
