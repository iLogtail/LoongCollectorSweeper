## Summary

Current main has adjacent usage/cost/quota reporting, but it still does not implement the requested configurable TUI footer fields or a plugin hook/template for cost, balance, or rate-limit status. This remains a concrete core TUI/plugin-API feature request, not a cleanup close.

## Best Possible Solution

Keep the issue open or manually consolidate it with a chosen canonical TUI-footer item. The likely product path is a core TUI footer configuration surface plus, if plugins should supply live cost/balance/cooldown data, a documented plugin API such as a footer/status contribution provider wired into `src/tui/tui.ts` and the public SDK docs.
