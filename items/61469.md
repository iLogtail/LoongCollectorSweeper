## Summary

Keep open. Current main still has the exact gating this PR is trying to change: `emitReasoningStream` returns before broadcasting unless `streamReasoning` is true and `onReasoningStream` exists, and the message handlers still only call it from thinking/tagged/final paths when reasoning streaming is enabled. The PR discussion also raised real visibility/suppression concerns that would need maintainer review before landing rather than auto-closing.
