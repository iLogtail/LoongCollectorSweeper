## Summary

Keep #61469 open. Current main still gates `stream:"thinking"` emission behind `reasoningMode:"stream"` plus an `onReasoningStream` callback, so the PR targets a real remaining gap for WS/webchat clients. It should not be auto-closed as implemented; however, the current PR version still needs maintainer/code follow-up for suppression semantics and tests before it is safely mergeable.

## Best Possible Solution

Keep the PR open but require a revised patch that decouples WS `thinking` event emission from the channel `onReasoningStream` callback while preserving `silentExpected` and deterministic-approval suppression. Add focused tests proving `onAgentEvent` receives `stream:"thinking"` without `onReasoningStream`, and proving hidden/silent/approval-suppressed runs do not leak thinking events. Coordinate with #47613 as the narrower related PR rather than closing this one automatically.
