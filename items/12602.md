## Summary

Keep #12602 open. Current main has shipped large parts of Slack Block Kit support, including authored `channelData.slack.blocks`, Slack message-tool `blocks`, shared `presentation` rendering, and interactive reply directives. The remaining table/rich-rendering gap is still real: `markdown.tables: "block"` is schema-valid and has IR support, but runtime delivery still coerces it to code/text instead of native Slack Block Kit table blocks.

## Best Possible Solution

Keep #12602 open unless a maintainer explicitly supersedes it with a narrower canonical Slack rich markdown/table issue. The best implementation path is in the bundled Slack plugin: carry markdown table metadata through runtime delivery and render supported cases as native Slack Block Kit table or richer blocks, with focused outbound tests and public docs updates. Avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive reply directive support.
