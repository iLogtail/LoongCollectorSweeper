## Summary

Keep open. Current main still does not implement the PR's LLM gateway request-context headers or `llm_output.isFallback` hook contract. The related #69463 overlaps on session tracing but does not fully supersede this PR's attempt/fallback/turn metadata, and the supplied review discussion identifies concrete fixups rather than a conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to either land a revised, tested core implementation that injects request-context headers with correct fallback and monotonic turn semantics, or intentionally fold this into a canonical tracing-header design such as #69463 while preserving any still-needed fallback and hook metadata.
