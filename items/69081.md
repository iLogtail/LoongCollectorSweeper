## Summary

Keep #69081 open. Current main still matches the reported TUI desync: the streaming watchdog is armed from chat delta events, tool events do not refresh it, and reconnect/load-history paths do not clear the active run or watchdog state. The related fix PR #69125 was closed unmerged, while the follow-up #69978 is a separate UX/configuration request rather than a canonical replacement for this bug.

## Best Possible Solution

Keep the issue open and land a focused TUI fix: in `src/tui/tui-event-handlers.ts`, treat active-run tool events as liveness and rearm the streaming watchdog before verbosity filtering; add a reconnect/reset seam so `src/tui/tui.ts` clears stale `activeChatRunId` and the pending watchdog before reloading history after reconnect; add regression tests for long tool activity and missed-final reconnect recovery. Keep #69978 separate as a watchdog UX/configuration follow-up.

This leaves the watchdog tied to chat deltas rather than all run liveness.
