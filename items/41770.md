## Summary

Keep open. Current `main` still does not honor env proxy settings for remote memory embedding fetches: both active memory host helpers call `fetchWithSsrFGuard(...)` in default strict mode, and that guard only uses an env proxy when `mode` is explicitly `trusted_env_proxy`. The branch is also stale against the current tree, but the underlying bug has not been implemented on `main`.

## Evidence

- **Active memory remote helper still uses strict guarded fetch:** The packaged memory host helper calls `fetchWithSsrFGuard` without `withTrustedEnvProxyGuardedFetchMode(...)` or any `mode` override.
  - file: [packages/memory-host-sdk/src/host/remote-http.ts:22](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/packages/memory-host-sdk/src/host/remote-http.ts#L22)
