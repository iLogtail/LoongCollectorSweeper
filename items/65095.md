## Summary

Keep #65095 open. Current main still has a strict macOS app-to-gateway/CLI version gate with no grace-period or skip-version-check mode, and the current docs tell users to update the global CLI when versions are incompatible. The related #65081 stable-release request is open and overlaps only the release-channel part; it does not supersede the gateway compatibility/offline-mode request.

## Best Possible Solution

Keep the issue open but narrow it to a maintainer product decision: define a supported app/gateway compatibility policy for remote/offline users, then either add a documented grace-period or explicit compatibility flag with macOS `GatewayEnvironment` and gateway protocol tests, or link the stable-release-channel portion to #65081 if that becomes the canonical release-cadence tracker.
