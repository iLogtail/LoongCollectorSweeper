## Summary

本 PR 为 LogFileReader 与 Go 侧协议转换器新增了纳秒级时间戳支持，通过全局配置项 EnableTimestampNanosecond 控制精度输出。改动涉及 C++ 核心日志读取逻辑与 Go 侧协议扁平化/自定义转换逻辑，属于明确且目标集中的功能增强。当前 main 分支尚未包含此特性，PR 状态为开放且未合并，无重复实现或冲突迹象，符合核心功能扩展路径，应保留供维护者进行代码审查、配置校验与集成测试。

## Best Possible Solution

维护者需审查新增配置项 mEnableTimestampNanosecond / EnableTimestampNanosecond 是否已在全局配置结构体、配置解析器与用户文档中完整定义；验证 GetCurrentTimeInNanoSeconds 在目标平台（Linux/Windows）下的时钟源兼容性与性能开销；确认 Go 侧转换器将秒与纳秒合并为 uint64 后的序列化格式是否符合下游消费端（如 SLS/Kafka/ES）的解析预期。审查通过且 CI 验证无误后可合并，并同步更新配置说明文档。
