## Summary

Keep open. Current `main` does not implement this PR's fix: Control UI favicon links are still rooted at `/`, and the gateway still serves `index.html` unchanged even when `basePath` is configured, so the reported 404 behavior remains plausible on `main`.

## Evidence

- **Control UI source still uses root-relative favicon links:** `ui/index.html` still hardcodes `/favicon.svg`, `/favicon-32.png`, and `/apple-touch-icon.png`, which bypass a configured `basePath`.
  - file: [ui/index.html:8](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/ui/index.html#L8)
  - command: `nl -ba ui/index.html | sed -n '1,20p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Gateway serves index.html without rewriting hrefs:** `serveResolvedIndexHtml()` computes CSP headers and then returns the HTML body unchanged; there is no favicon/basePath rewrite on current `main`.
  - file: [src/gateway/control-ui.ts:612](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/gateway/control-ui.ts#L612)

- **Current basePath flow does not reach index.html rewriting:** `handleControlUiHttpRequest()` normalizes `basePath`, but both `index.html` serve paths call `serveResolvedIndexHtml(res, body)` without passing `basePath`, so mounted Control UI pages still emit the raw favicon hrefs.
  - file: [src/gateway/control-ui.ts:694](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/gateway/control-ui.ts#L694)

  - file: [src/gateway/control-ui-links.ts:30](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/gateway/control-ui-links.ts#L30)
