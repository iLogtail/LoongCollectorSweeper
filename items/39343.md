## Summary

Keep #39343 open. Current main partially solves the Telegram album case with `media_group_id` buffering, but the requested LINE same-sender/time-proximity media batching and configurable media batch delay are still missing. The issue is concrete, author association is `NONE`, and the only label in the provided GitHub context is `stale`, so it remains actionable rather than closeable.

## Best Possible Solution

Keep this issue open and implement a focused channel-owned media batching feature: preserve Telegram `media_group_id` buffering, add LINE sender/chat/time-proximity buffering, expose a documented config knob such as `mediaBatchDelayMs` or a generic media-specific inbound debounce setting, flush on timer or text arrival, and add Telegram plus LINE regression tests and docs.
