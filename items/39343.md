## Summary

Keep #39343 open. Current main already buffers Telegram albums using `media_group_id`, but the requested LINE same-sender/time-window image batching and a documented/configurable media batch delay are still missing. The issue is concrete, author association is `NONE`, and the provided context has no protected labels.

## Best Possible Solution

Keep the issue open and implement a focused channel-owned media batching feature: preserve Telegram album buffering, add LINE sender/chat/time-proximity media buffering, expose documented configuration for the media batch delay or an equivalent media-specific inbound debounce, flush on timer or text arrival, and add LINE plus Telegram regression coverage and docs.
