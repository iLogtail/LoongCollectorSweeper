## Summary

Keep open. Current `main` does not expose a shell-profile selection surface for exec commands. The exec config/docs list host/security/node/PATH-related options but no `shellProfile` field, and runtime code still builds exec argv from the detected shell with fixed flags (`-c` on Unix, `-NoProfile -NonInteractive -Command` on Windows). The only adjacent support on `main` is login-shell env/PATH import, which is not the requested per-command or configurable profile-file feature.

## Evidence
