## Close Comment

Closing this as a core PR after a Codex review. The current repo direction is to keep optional capability out of `main` unless there is a strong reason to bundle it: `VISION.md:54-61` says optional capability should usually ship as plugins, be hosted in its own repo, and has a high bar for landing in core. The plugin docs make the same call for providers: `docs/plugins/building-plugins.md:16-19` says you do not need to add a plugin to this repository, `docs/plugins/sdk-provider-plugins.md:679-689` documents publishing provider plugins to ClawHub, and `docs/plugins/sdk-setup.md:159-167` says onboarding can already surface provider auth choices from plugin metadata and install on demand. This PR’s final diff is also plugin-only (`extensions/zenmux/*` plus `.github/labeler.yml`), so it fits the external plugin path better than `main`. If you still want ZenMux support, please publish it as an external OpenClaw provider plugin on ClawHub or npm and reopen only if you hit a concrete core/plugin-SDK gap.
