## Summary

Keep open. The reported WebChat overlap remains plausible and actionable on current main: action buttons are rendered for assistant messages, but the only CSS padding hook (`.chat-bubble.has-copy`) is still not applied by the bubble class construction.

## Evidence

- **Current renderer omits has-copy:** `bubbleClasses` is built from `chat-bubble`, optional `chat-bubble--tool-shell`, optional `streaming`, and `fade-in`; it does not include `has-copy`. The actions are rendered separately when `hasActions` is true.
  - file: [ui/src/ui/chat/grouped-render.ts:1320](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/ui/src/ui/chat/grouped-render.ts#L1320)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '1180,1395p'`
  - sha: [b64bfc5d9abf](https://github.com/openclaw/openclaw/commit/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab)
- **Actions still overlay the bubble:** The CSS reserves right padding only for `.chat-bubble.has-copy`, while `.chat-bubble-actions` is absolutely positioned at the top right and made visible on hover or on non-hover devices.
  - file: [ui/src/styles/chat/grouped.css:222](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/ui/src/styles/chat/grouped.css#L222)

  - sha: [b64bfc5d9abf](https://github.com/openclaw/openclaw/commit/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab)
- **has-copy appears only in CSS:** Repository search found `has-copy` only in the CSS selector, not in current UI renderer code, which supports the issue comment's CSS/JS mismatch diagnosis.
  - file: [ui/src/styles/chat/grouped.css:222](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/ui/src/styles/chat/grouped.css#L222)
