## Summary

Keep #50195 open. Current main still does not support WhatsApp pre-built linkPreview payloads, a configurable Baileys generateHighQualityLinkPreview setting, or a batch send RPC/action. The related #6722 was closed as superseded by this broader tracker, and the prior implementation PR #43397 was closed unmerged, so this remains the canonical actionable issue.

## Best Possible Solution

Keep #50195 open as the canonical WhatsApp tracker. The best implementation path is in the bundled WhatsApp plugin plus gateway contracts: add a typed pre-built `linkPreview` pass-through to WhatsApp text sends, add an explicit opt-in `channels.whatsapp.generateHighQualityLinkPreview` config if automatic fetching is desired, update tests/docs/config metadata, and handle batch sending either as a carefully specified gateway `send.batch` method or a narrower follow-up if batching needs separate product/API review.
