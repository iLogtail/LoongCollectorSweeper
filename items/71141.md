## Summary

Keep open. Current main at c74fb781943f3bad7d6fec5256729c861d285cab still lacks the PR's core behavior: reaction directives are not parsed into outbound channel data, Telegram send payloads do not execute channelData.telegram.reaction, and reaction-only Telegram replies are still rejected as missing text/media. The linked issue #71140 remains the same open bug, and this draft PR is a focused proposed fix rather than obsolete cleanup.

## Best Possible Solution

Keep PR #71141 open and review it as the active focused fix for #71140. The likely product path is to add structured reaction directive parsing, preserve it as channelData.telegram.reaction during outbound normalization, allow reaction-only Telegram replies when a reply target exists, execute reactMessageTelegram before optional text/media sends, and land targeted normalization plus Telegram adapter coverage.
