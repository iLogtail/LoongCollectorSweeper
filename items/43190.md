## Summary

Keep #43190 open. Current main does not contain the new msteams-channel-archive plugin, does not expose a channel_deleted plugin hook, and does not dispatch Teams conversationUpdate/channelDeleted events. The archive plugin itself likely belongs outside core/ClawHub unless maintainers want it bundled, but this PR also points at a real missing lifecycle/metadata seam for event-driven deleted-channel cleanup, so it is not safe to auto-close as ClawHub-only or implemented.

## Best Possible Solution

Keep the PR open for maintainer judgment, but steer it toward a split: land only a narrow, documented plugin/Teams lifecycle seam if maintainers agree it is needed, with tests and docs; move the durable Teams archive implementation to a ClawHub/npm plugin unless there is an explicit product/security reason to bundle it; then address the remaining cleanup correctness and package-boundary review comments in a smaller PR.
