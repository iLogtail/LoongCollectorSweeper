## Summary

Keep #8913 open. Current main still has a single italic-only reasoning formatter, runner/subscriber/Telegram paths do not accept or pass a display-format option, config/docs expose reasoning visibility and provider transport thinking format only, and the related implementation PR #25336 is closed unmerged. The owner comment already reached the same keep-open conclusion on April 26, 2026, and I found no newer main change that implements the requested channel/agent configurable formatting.

## Best Possible Solution

Keep #8913 open. A good implementation should define a documented reasoning/thinking display-format contract, validate it in the selected config scope, thread it through embedded runner payloads, subscribe handlers, reasoning streaming, and Telegram split/dispatch, preserve backward-compatible italic defaults, and add regression coverage for default italic plus configured code/spoiler/blockquote behavior.
