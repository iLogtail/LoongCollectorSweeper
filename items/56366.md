## Summary

Current main still lacks the requested root-level opt-out for Claude/Agents skill directories. The skills loader unconditionally scans both `~/.agents/skills` and `<workspace>/.agents/skills`, the watcher also includes those roots, and the strict config schema only exposes `extraDirs`, `watch`, and `watchDebounceMs` under `skills.load`. Docs and tests continue to describe `.agents/skills` as always-loaded default roots, so this remains a real core skills config gap rather than a closeable item.

## Best Possible Solution

Keep the issue open. The best maintainer path is to add a documented `skills.load` control, preferably a general `ignoreDirs` list or a narrow `scanAgentsSkills` boolean defaulting to current behavior, then make `loadSkillEntries` and `resolveWatchPaths` honor it. Update `SkillsLoadConfig`, the strict zod schema, generated/help labels as needed, docs under `docs/tools/skills-config.md` and `docs/tools/skills.md`, and add focused tests proving personal and project `.agents/skills` roots can be skipped while managed, bundled, workspace, and `extraDirs` behavior stays intact.

## Risks / Open Questions

- A root-level opt-out must update loader behavior and watcher behavior together; otherwise disabled roots could still trigger refresh churn.
- The default should likely remain current behavior for backward compatibility, while allowing users who coexist with Claude Code to exclude `~/.agents/skills` without disabling every skill by name.
