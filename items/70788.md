## Summary

Keep open. Current main still generates the Windows Startup-folder fallback as a single `.cmd` file that runs `start "" /min cmd.exe /d /c ...`, and the existing startup fallback test explicitly asserts that behavior. The linked #57682 is related Windows console-flash work for the update restart path, but it does not supersede this separate login Startup-folder launcher issue.

## Best Possible Solution

Keep #70788 open for a focused Windows daemon fix in `src/daemon/schtasks.ts`: generate a silent companion launcher or another no-window startup mechanism for the Startup-folder fallback, keep the existing `.cmd` detection path stable, update install/uninstall cleanup, and adjust `src/daemon/schtasks.startup-fallback.test.ts` plus Windows service docs to cover the new behavior.
