## Summary

Keep open. Current main still implements the Windows Startup-folder fallback as a single `.cmd` launcher that renders `start "" /min cmd.exe /d /c ...`, and the regression test explicitly expects that command. The proposed wscript/VBScript silent launcher is not present in the daemon startup fallback path. The related #57682 is a closed update-restart surface, not a canonical tracker for this login Startup-folder flash.

## Best Possible Solution

Implement the Startup-folder fallback fix in `src/daemon/schtasks.ts`: keep the existing `.cmd` path for service detection, add a hidden Windows launcher such as an adjacent `.vbs`/`wscript.exe //nologo` shim or an equivalent no-window mechanism, remove the companion file on uninstall, and update `src/daemon/schtasks.startup-fallback.test.ts` to assert the silent launcher behavior. Maintainers should also verify it on a real Windows 11 login because the bug is compositor-visible behavior.
