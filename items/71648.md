## Summary

Keep PR #71648 open. Current main still has the unbounded `pendingClaudePermissions` and `pendingApprovals` maps, does not clear them on `close()`, and only deletes entries on matched permission replies or approval-resolved events. The PR is a recent, focused bug fix for the linked open issue #71646, with API-visible proof that it remains open and unmerged, plus a targeted diff adding TTL sweeping, close cleanup, post-close guards, and regression tests.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to evaluate and land this focused MCP memory-bounding fix, or request a narrowly scoped update if maintainers want the deferred cap/FIFO guard before merge. Current main still lacks the behavior, and the linked issue #71646 should remain tied to this PR until the fix lands or a canonical replacement PR supersedes it.
