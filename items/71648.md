## Summary

Keep #71648 open. Current main still has the reported MCP bridge leak shape: `pendingClaudePermissions` and `pendingApprovals` are plain instance Maps, are not cleared in `close()`, and are only pruned by matching reply/resolved events. The PR is a focused, recent bug-fix PR with regression coverage in the provided diff, and its follow-up commit `9f16dd4823547354c6d7f85b0dab7fe672008010` addresses the bot review's self-terminating sweeper concern. The linked issue #71646 is the bug report this PR fixes, not a reason to close the PR.

## Best Possible Solution

Keep #71648 open for maintainer review and possible merge. The best path is to land this or an equivalent scoped fix that clears both pending Maps on bridge close, expires stale Claude permission and approval entries without changing the MCP protocol/tool surface, and keeps the added regression coverage. If maintainers want burst caps, track that separately rather than closing this useful PR.
