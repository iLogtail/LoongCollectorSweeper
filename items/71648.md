## Summary

Keep PR #71648 open. Current main at 21c51bc still has the MCP channel bridge leak pattern the PR targets: raw pendingClaudePermissions and pendingApprovals Maps are populated without TTL/sweeper or close-clear, and cleanup depends only on matched Claude replies or approval resolved events. The PR is a focused, recent bug-fix PR linked to still-open issue #71646, includes regression tests, and its one review thread was addressed by commit 9f16dd4823547354c6d7f85b0dab7fe672008010. It is not implemented on main, not superseded by #56420, and has no protected author/label reason to auto-close.

## Best Possible Solution

Keep #71648 open for maintainer review and likely merge after normal validation. The best code path is to land the focused MCP bridge TTL sweeper, close-clear, post-close guard, and regression tests, while leaving the separate cap/FIFO concern to a narrower follow-up if maintainers want stronger burst bounds.
