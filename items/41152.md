## Summary

Keep open. Current main still implements the reported async approval completion path with the same 400-character tail and whitespace-collapsing formatter described in #41152. The related PR #41170 correctly identified this path and proposed preserving multiline output, but it was closed unmerged, and no equivalent fix is present on main.

## Best Possible Solution

Fix the async exec approval completion formatter on main, likely by reviving the narrow #41170 approach: for gateway approval followups, deliver the captured exec output with newlines preserved, include an explicit note when the existing capture cap is hit, and keep ordinary background `notifyOnExit` summaries compact. Add regression coverage for >400-character multiline gateway approval output, and consider auditing the analogous node approval followup path, which also tails and normalizes output.
