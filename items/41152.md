## Summary

Keep open. Current `main` still hardcodes async exec-approval followups to the last 400 characters and collapses all whitespace/newlines before sending the completion back into the resumed agent flow, so the reported bug is still present. The same code is also present in the latest release `v2026.4.22`, and I found no evidence of a merged fix on `main`.
