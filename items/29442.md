## Summary

Keep #29442 open. Current main has real prompt-injection mitigations for external content, cron hook payloads, web fetch/search output, tool policy, sandboxing, and plugin hooks, and the ClawMoat link in the issue discussion is useful defense-in-depth. But current main still does not implement the requested cron-integrated dual-LLM privilege-separation/sanitization path, and the plugin hook catalog still lacks the #30046-style post-external-content-wrap sanitizer seam that would let a plugin replace or block wrapped external content before the privileged cron agent sees it.

## Best Possible Solution

Keep the issue open as the canonical product/security design item. A good path is either to implement a first-class cron/external-content reader-worker flow, or to add a narrow documented plugin API seam that can replace/block wrapped external content before privileged agent turns, then let ClawHub plugins such as ClawMoat provide optional detection/sanitization on top of that seam.
