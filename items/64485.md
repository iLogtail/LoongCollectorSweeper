Keep PR #64485 open. Current main still leaves the root Compose volume sources without defaults, and `docker compose config` fails when the mount env vars are unset. The setup script covers its own path by exporting defaults, but docs still show a direct manual Compose flow, so the PR is a focused Docker usability fix rather than an obsolete cleanup candidate.

## Best Possible Solution
