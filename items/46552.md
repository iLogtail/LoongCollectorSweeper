## Summary

Keep open. Current main still lacks the PR's targeted docs: `docs/concepts/queue.md` describes queue modes and basic troubleshooting, but does not explain why `steer` looks sequential on `partial`/`block` streaming surfaces, that non-streaming `steer` falls back to followup, or that per-session `/queue` overrides can trump config. The PR discussion also shows the two factual review concerns were addressed in later PR commits, so this remains plausible focused docs work rather than obsolete cleanup.

## Best Possible Solution

Keep PR #46552 open and either land it after maintainer docs review or recreate the same focused section in `docs/concepts/queue.md`. The best wording should distinguish Discord `partial` preview finalization from `block` chunks, document non-streaming `steer` followup fallback, and mention `/queue` status/per-session override precedence.
