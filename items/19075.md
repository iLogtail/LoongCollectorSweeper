## Summary

Keep #19075 open. Current main still does not implement a configurable Brave Search baseUrl: Brave runtime uses fixed api.search.brave.com endpoints for both web and llm-context modes, the Brave plugin schema/UI hints only expose apiKey and mode, and docs do not document any Brave baseUrl/proxy setting. Related implementation PRs in the provided context are closed unmerged, while duplicate issues were closed back to this canonical tracker.

## Best Possible Solution

Keep #19075 open as the canonical tracker. Implement this in the bundled Brave plugin: add plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints and Brave config type, normalize and validate it, build both /res/v1/web/search and /res/v1/llm/context URLs from the configured base while preserving the default api.search.brave.com behavior, include the base URL in cache identity, update https://docs.openclaw.ai/tools/brave-search, and add focused runtime/schema tests. The existing provider-owned plugin config seam is sufficient; this is missing bundled Brave behavior, not a ClawHub-only request.
