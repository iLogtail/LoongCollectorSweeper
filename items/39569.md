## Summary

Keep #39569 open. Current main still lacks the Feishu reply orphan-placeholder mention fallback this PR targets, and the linked bug #39568 remains a real open behavior gap. The PR is not cleanup-close material because its current rootId-clearing approach intersects with existing Feishu topic-session routing contracts that main still preserves and tests cover.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix path. The best next step is to rebase it onto the current `bot-content.ts` split, add focused dropped-bot-mention reply tests, make the fallback as bot-specific as Feishu allows, and avoid blanket `rootId` clearing that breaks `group_topic` / `group_topic_sender` routing contracts. Coordinate the final behavior with the linked open bug #39568.
