## Summary

Keep PR #39569 open. Current main still lacks a Feishu reply-message orphan-placeholder mention fallback, so the linked #39568 bug remains plausible. The PR contains useful production verification and regression-test direction, but its current broad orphan-key heuristic and root/thread routing changes still need revision against current Feishu topic-session behavior.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix. Rebase it onto the current `bot-content.ts` split, add focused regression tests for reply events where Feishu drops the bot mention, make the fallback as bot-specific as available payload data allows, and preserve intentional `group_topic`/`group_topic_sender` routing while fixing normal reply+mention behavior tracked by #39568.

## Risks / Open Questions

- The PR's original implementation target moved from `extensions/feishu/src/bot.ts` to `extensions/feishu/src/bot-content.ts`, so it likely needs a rebase rather than a direct merge.
- A broad orphan-placeholder heuristic can bypass `requireMention` if Feishu drops a non-bot mention or user text contains placeholder-like tokens.
- Blanket `rootId` clearing can merge or reroute Feishu topic-scoped sessions that intentionally depend on root/thread metadata.
