## Summary

Keep open. Current main still lacks the Feishu reply-mention fallback this PR is trying to add, so the bug is not obsolete. The PR also remains unsafe to close as implemented because its proposed `rootId` clearing and broad orphan-key heuristic still conflict with current topic-routing contracts and the review discussion already identified those correctness risks.

## Best Possible Solution

Keep the PR open or ask for a revised patch. The safest path is to implement reply-message mention recovery in the current `extensions/feishu/src/bot-content.ts` helper with a stronger bot-specific signal or documented Feishu constraint, preserve `rootId` for topic-scoped session routing, and add regression coverage for both orphaned reply mentions and topic/root reply behavior.
