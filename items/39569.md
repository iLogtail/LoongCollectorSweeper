## Summary

Keep #39569 open. Current main still does not implement the Feishu reply orphan-placeholder mention fallback, so the linked bug in #39568 remains plausible. The PR contains useful reproduction notes and regression-test direction, but it should be revised before landing because the broad orphan-key heuristic and blanket rootId clearing conflict with current topic/thread routing behavior.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix. Rebase it onto the current `bot-content.ts` split, add focused regression tests for reply events where Feishu drops the bot mention, make the fallback as bot-specific as available payload data allows, and preserve topic-session `rootId` behavior while coordinating final semantics with #39568.

## Risks / Open Questions

- A broad orphan-placeholder heuristic can mark the bot as mentioned when Feishu dropped a non-bot mention or a user typed a placeholder-like token.
- Clearing `rootId` for all mentioned replies can merge or reroute topic-scoped Feishu sessions that intentionally use root/thread metadata.
- A final fix may need real Feishu payload samples to make the fallback as bot-specific as the platform allows.
