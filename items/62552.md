## Summary

Keep #62552 open. Current main still uses `acp:<uuid>` for new ACP bridge fallback sessions and pending ACP prompts still require an exact raw session-key match, so the PR's central ACP bridge fix is not implemented or superseded. The PR is not ready to close as obsolete, but its task-registry hunk needs maintainer attention because an existing review comment flags a plausible cron false-lost regression, and current main has since moved cron task recovery in a different direction via durable run-log/job-state recovery.

## Best Possible Solution

Keep #62552 open for maintainer review. The best path is to salvage the ACP-specific fix by switching bridge fallback sessions to a non-runtime-reserved namespace such as `acp-bridge:<uuid>`, matching pending prompts across raw and Gateway-canonical session keys, updating the ACP docs, and adding the two ACP regression tests from the PR. The cron/task-maintenance hunk should be dropped, split, or reworked against current `src/tasks/task-registry.maintenance.ts` so active cron jobs remain authoritative and durable cron recovery remains intact.
