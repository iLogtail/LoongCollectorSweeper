## Summary

Keep open. Current main still has the timeout behavior the PR is trying to change, and the submitted PR has unresolved review feedback about a task-lifecycle race rather than being obsolete or safely implemented elsewhere.

## Best Possible Solution

Keep the PR open or request changes. The best fix should preserve the user-visible goal from #70024, but add task identity or lifecycle guarding around the old `trackedPromise` cleanup/restart path and include a regression test where a timed-out task eventually settles after a restart attempt.
