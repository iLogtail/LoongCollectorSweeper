## Summary

Keep open. Current `main` still sends `openclaw models set` through the normal config rewrite path, and that path rejects writes that shrink the config by more than 50% with `CONFIG_WRITE_REJECTED`. The write layer also reserializes to canonical JSON, so a comment-heavy or otherwise bulky JSON5 config can plausibly hit the reported `size-drop` guard. I found no `models set`-specific bypass or preservation path that would make this clearly fixed on `main`.
