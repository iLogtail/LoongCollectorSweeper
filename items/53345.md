## Summary

Keep open. Current main already has Korean Control UI locale support, registration, fallback behavior, docs, and tests, and that UI portion is present in the latest release tag. The issue also explicitly asks for AI-agent Korean language persistence and localized search behavior; current main does not add a platform-level primary-language directive to the agent system prompt, and the linked PR #53346 is closed unmerged rather than a canonical open follow-up.

## Best Possible Solution

Keep the issue open but narrow it: treat Control UI Korean localization as already shipped, then either implement a generic agent primary-language persistence policy in `src/agents/system-prompt.ts` with focused prompt tests and any provider/search parameter support that is actually available, or split that remaining agent/search behavior into a smaller maintainer-owned follow-up before closing this broad issue.
