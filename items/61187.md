## Summary

Keep #61187 open. Current main still does not add Kimi or Moonshot provider-owned supplemental rows to the picker catalog via `augmentModelCatalog`, and the picker still falls back to the unfiltered global catalog when a preferred provider has no matching catalog rows. The PR remains a focused fix path, with review comments already addressed in the PR branch per the provided discussion.

## Best Possible Solution

Keep the PR or a maintainer-owned successor open. The best fix is to expose Kimi and Moonshot picker rows through the current picker catalog path, either by landing provider-local `augmentModelCatalog` hooks with tests or by teaching `loadModelCatalog`/the picker to consume trusted static manifest catalog rows for provider-scoped setup without regressing third-party plugin boundaries.
