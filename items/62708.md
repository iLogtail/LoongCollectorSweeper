## Summary

Keep open. Current main has not implemented the requested refactor: `claudeCliSessionId` is still part of `SessionEntry`, is still dual-written/cleared by CLI session helpers, and current gateway history import code still uses it as a legacy fallback with a regression test covering that behavior.

## Best Possible Solution

Keep the issue open for maintainer judgment. A good fix would either keep the compatibility fallback intentionally and close this as unwanted, or perform a compatibility-aware cleanup: decide how to handle legacy-only sessions, remove the type/fallback/dual writes/pass-throughs together, and update the tests that currently assert legacy fallback behavior.
