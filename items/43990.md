## Summary

Keep open. Current main contains mitigations for this exact `ENOTEMPTY` stale-rename failure in the installer script and in `openclaw update`, but the repo still documents plain `npm install -g openclaw@latest` and the published package lifecycle does not show an equivalent cleanup/retry path for raw npm global installs. The report therefore does not look safely closable as implemented/cannot-reproduce.

## Evidence

- **Manual npm global install is still a documented path:** `docs/install/index.md:64-72` still tells users to run `npm install -g openclaw@latest` for manual installs.
  - file: [docs/install/index.md:64](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/install/index.md#L64)
  - command: `nl -ba docs/install/index.md | sed -n '52,90p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Package lifecycle does not include a stale-rename cleanup step:** `package.json:1410-1413` only wires `preinstall` and `postinstall`. `scripts/postinstall-bundled-plugins.mjs:1-6` is about bundled plugin runtime handling after install, not recovering npm global-prefix rename collisions before install completes.
  - file: [package.json:1410](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/package.json#L1410)
  - command: `nl -ba package.json | sed -n '1405,1414p'; nl -ba scripts/postinstall-bundled-plugins.mjs | sed -n '1,24p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Installer script has an exact mitigation for this failure signature:** `scripts/install.sh:479-484` deletes `"$npm_root"/.openclaw-*` and `"$npm_root"/openclaw`. `scripts/install.sh:811-818` matches `ENOTEMPTY: directory not empty, rename .*openclaw`, then cleans and retries. That mitigation exists for the installer script, not for raw `npm install -g` invoked directly by the user.
  - file: [scripts/install.sh:811](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/scripts/install.sh#L811)

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Updater also cleans stale rename directories before global package updates:** `src/infra/update-runner.ts:1293-1297` calls `cleanupGlobalRenameDirs(...)` before the global update. The helper removes `.openclaw-*` directories in `src/infra/update-global.ts:588-611`, and that behavior is covered by `src/infra/update-runner.test.ts:1353-1368`. This is another mitigation, but it is scoped to `openclaw update` rather than first-time/manual npm installs.
  - file: [src/infra/update-runner.ts:1293](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/infra/update-runner.ts#L1293)
