## Summary

Keep #11040 open. Current main still has partial lineage support only: persisted sessions and gateway rows carry `spawnedBy`, `parentSessionKey`, and `spawnDepth` in some paths, but the requested Phase 1 contract is not implemented because `rootSessionKey` and a stable session-chain `traceId` are absent from the session model, gateway protocol, spawn write paths, `sessions_list` tool rows, and UI row type. The related implementation PR #11043 is closed unmerged, and the latest issue discussion already identified these same remaining root/trace gaps.

## Best Possible Solution

Keep the issue open until core either implements the additive Phase 1 chain contract across `SessionEntry`, gateway schema/patch/list rows, native and ACP spawn write paths, `sessions_list` output, UI/shared row types, and multi-hop regression tests, or a maintainer explicitly narrows the product direction to the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model. A ClawHub observability plugin can consume current partial lineage, but it cannot create canonical persisted `rootSessionKey` and `traceId` for all spawned sessions without a core contract.
