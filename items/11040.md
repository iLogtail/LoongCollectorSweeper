## Summary

Keep #11040 open. Current main still has only partial spawn lineage (`spawnedBy`, `parentSessionKey`, `spawnDepth`) and does not implement the requested first-class `rootSessionKey` plus stable session-chain `traceId` contract in the persisted session model, gateway protocol/list rows, agent-facing `sessions_list`, or native/ACP spawn write paths. The related implementation PR #11043 was closed unmerged, and the existing Codex review comment also correctly kept this open.

## Best Possible Solution

Keep the issue open until core implements the additive Phase 1 chain contract across `SessionEntry`, gateway schemas/list rows/events, native and ACP spawn write paths, agent-facing `sessions_list`, UI/shared row types where needed, and multi-hop regression tests. If maintainers decide the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model is the intended product boundary, they should narrow or close the issue explicitly with that rationale.
