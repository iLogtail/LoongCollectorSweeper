## Summary

Keep PR #69340 open. Current main still has the underscore-only copy-sensitive guard in the TUI formatter, so long hyphen-separated identifiers without digits are still chunked with inserted spaces. The linked bug #48432 remains open, and the PR is a focused one-line fix against the implicated code path, though it still needs regression coverage before landing.

## Best Possible Solution

Keep the PR open and land this behavior on main after adding a focused regression test in `src/tui/tui-formatters.test.ts` for a long hyphen-separated identifier. The minimal code change proposed by the PR matches the current formatter contract; broader alignment with `src/terminal/note.ts` can be handled as a separate follow-up if desired.

## Evidence
