## Summary

Keep open. Current main still rejects WhatsApp `@hosted.lid` targets in the shared target normalizer, while adjacent runtime identity/JID conversion paths already recognize `@hosted.lid`. PR #70803 remains a narrow, plausible bug fix with matching regression coverage and is not superseded by the provided related items.

## Best Possible Solution

Keep PR #70803 open for maintainer review and normal CI. The best fix is the PR's small regex expansion in `extensions/whatsapp/src/normalize-target.ts` plus the added lower/upper-case `@hosted.lid` regression assertions in `extensions/whatsapp/src/resolve-target.test.ts`; that aligns target normalization with the already-shipped runtime hosted-LID handling.
