## Summary

Current `main` still has the reported gap. When provider resolution is empty, models planning returns `skip`, `ensureOpenClawModelsJson()` exits without creating or bootstrapping `models.json`, and the non-main agent runtime path still depends on that per-agent registry before resolving models.

## Evidence

- **Empty provider plan still skips models.json generation:** `planOpenClawModelsJsonWithDeps()` returns `{ action: "skip" }` immediately when `Object.keys(providers).length === 0`, with no non-main fallback logic.
  - file: [src/agents/models-config.plan.ts:101](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/models-config.plan.ts#L101)
  - command: `nl -ba src/agents/models-config.plan.ts | sed -n '70,170p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Skip path does not bootstrap from main agent:** `ensureOpenClawModelsJson()` returns `{ wrote: false }` on `plan.action === "skip"` and does not read or copy `agents/main/agent/models.json`.
  - file: [src/agents/models-config.ts:174](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/models-config.ts#L174)
  - command: `nl -ba src/agents/models-config.ts | sed -n '115,220p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Non-main agent creation does not create models.json:** `agents.create` computes and persists the new agent config and directories, but there is no `ensureOpenClawModelsJson()` call or other `models.json` bootstrap in the create flow.
  - file: [src/gateway/server-methods/agents.ts:491](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-methods/agents.ts#L491)
