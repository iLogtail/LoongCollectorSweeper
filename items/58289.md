## Summary

Keep open. Current `main` still skips non-main `models.json` generation when provider resolution is empty, and the downstream runtime still surfaces `Unknown model`; the bootstrap-from-main fallback described in the issue discussion is not present in the checked-out code.

## Evidence

- **Empty provider plan still skips outright:** `planOpenClawModelsJsonWithDeps()` returns `{ action: "skip" }` whenever `Object.keys(providers).length === 0`, with no non-main bootstrap branch.
  - file: [src/agents/models-config.plan.ts:101](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/models-config.plan.ts#L101)
  - command: `nl -ba src/agents/models-config.plan.ts | sed -n '95,110p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Skip path still does not create or copy models.json:** `ensureOpenClawModelsJson()` reads the existing file, calls the planner, and immediately returns `{ wrote: false }` on `plan.action === "skip"`; there is no copy-from-main fallback before returning.
  - file: [src/agents/models-config.ts:164](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/models-config.ts#L164)
