## Summary

Keep open. Current main does not auto-discover GitHub Copilot chat models from `/models`: the bundled provider catalog still returns an empty model list and only supports forward-compatible model IDs when users already configure or request them manually. The PR’s core Copilot discovery work is therefore still potentially useful, though the branch should be narrowed before merge because it also carries unrelated agent/test cleanup.

## Best Possible Solution

Keep the PR open, but ask for a focused Copilot-only branch: implement provider-owned chat model discovery under `extensions/github-copilot`, keep transport selection aligned with `resolveCopilotTransportApi`, honor explicit models/disable flags/configured baseUrl and headers, add targeted contract tests, and split unrelated agent replay or ModelStudio fixture fixes into separate PRs.
