## Summary

Keep #8441 open. Current main still does not implement top-level `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the public type and strict schema omit those keys, tests still assert unknown top-level skill entry fields are rejected, skill command specs do not carry per-skill model/thinking defaults, and skill invocation only dispatches a tool or rewrites the prompt. The related implementation PR #8455 is closed unmerged in the provided GitHub context, and the latest issue discussion already explicitly kept this item open.

## Best Possible Solution

Keep #8441 open. A good implementation should extend `SkillConfig` and `SkillEntrySchema` with validated optional `model` and `thinking` defaults, carry those defaults through skill command construction/invocation, apply them only as per-skill defaults that yield to explicit user/session directives, add focused config and invocation tests, and update the public skills config plus thinking precedence docs.
