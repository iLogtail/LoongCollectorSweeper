## Summary

Keep open. Current main still rejects the exact `mode="session"` without `thread=true` behavior for both native subagents and ACP, while tests and docs continue to codify that requirement. Main has added better alternatives for headless orchestration (`mode="run"`, retained sessions, `sessions_send`, `sessions_yield`, and nested announce routing), but that is not the same as this PR’s proposed API change, and the PR discussion also identified an ACP inline-delivery risk that still needs maintainer-quality resolution before merge or close.

## Best Possible Solution

Keep this PR open until a maintainer decides whether headless persistent sessions should become first-class `mode="session"` behavior. If yes, update both native and ACP paths, make ACP inline delivery require an actual binding, adjust notes/docs/tests, and preserve the current run-mode `sessions_yield` orchestration behavior. If no, close with maintainer product rationale and point contributors to the shipped `mode="run"` plus retained-session and `sessions_send`/`sessions_yield` workflow.
