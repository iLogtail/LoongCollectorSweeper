## Summary

Keep open. Current main still rejects the headless `mode="session"` behavior this PR proposes for both native subagents and ACP, and current docs/tests still codify the thread requirement. Related issue #65396 was closed as a duplicate of this PR, so #39372 remains the canonical tracker rather than obsolete cleanup.

## Best Possible Solution

Keep #39372 open as the canonical maintainer decision point. If the feature should land, update both native and ACP spawn paths, make ACP inline delivery require an actual binding, add distinct headless-session notes, preserve cleanup on failed starts, and update tests/docs. If maintainers choose the current `mode="run"` plus `sessions_send`/`sessions_yield` workflow instead, close this PR with that explicit product rationale.
