## Summary

Keep open. Current main still adds `message_reference` to Discord webhook execute payloads, and the bound-thread outbound path can still pass `replyToId` into that webhook sender. The PR is stale against the moved file path, but the underlying fix is still relevant.

## Best Possible Solution

Keep this PR or an equivalent focused fix open. Refresh the patch onto `extensions/discord/src/send.outbound.ts`, remove `message_reference` only from `sendWebhookMessageDiscord`, preserve the bot-token `sendMessageDiscord` reply behavior, and add a webhook payload regression test for `replyTo`.
