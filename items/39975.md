## Summary

Keep open. Current main still builds `message_reference` into Discord Execute Webhook JSON bodies when `replyTo` is present, so PR #39975’s central fix remains relevant. The PR itself is stale against the moved Discord plugin path and should be refreshed with a focused webhook-body regression test.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve `sendMessageDiscord` bot-token reply behavior, and add a webhook request-body regression test for `replyTo`.
