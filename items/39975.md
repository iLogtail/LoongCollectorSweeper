## Summary

Keep open: current `main` still adds `message_reference` to Discord webhook execute payloads in `extensions/discord/src/send.outbound.ts`, the Discord outbound adapter still routes `replyToId` into that webhook path for bound-thread persona sends, and Discord’s current API docs still describe `message_reference` under Create Message rather than Execute Webhook. The latest release tag `v2026.4.23` still contains the same webhook payload code, so this is not implemented on `main` and not obsolete. citeturn0search0turn1view1turn0search2turn1view3
