## Summary

Keep open. Current main still builds and sends `message_reference` in Discord webhook execute payloads, so the PR’s central fix is still relevant. The patch is stale against the moved Discord plugin path and should be refreshed with a focused webhook-body regression test.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve `sendMessageDiscord` bot-token reply behavior, and add a webhook-specific regression test proving `replyTo` does not serialize into Execute Webhook payloads.
