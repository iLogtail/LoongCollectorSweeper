## Summary

Keep #71301 open. Current main has partial coverage: the npm package includes `docs/`, agents can resolve a local docs path, and prompts tell agents to consult local docs plus `config.schema.lookup`. The central request is still missing because there is no native local `docs.search`/`docs.get` style retrieval surface for agents, and `openclaw docs` still searches the live docs index.

## Best Possible Solution

Keep the issue open and narrow it to the remaining core work: retain the existing bundled docs and prompt guidance, then add a first-class local docs retriever over bundled `docs/`, exposed to agents through a gateway/MCP/tool surface such as `docs.search` and `docs.get`, with local-first behavior and optional live-docs fallback. Cover package-install docs discovery, generated-doc exclusions, and agent prompt/tool behavior in focused tests.
