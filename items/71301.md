## Summary

Keep open. Current main partially addresses the request by bundling local docs and pointing agents at the installed docs path, but it does not implement the requested native docs retrieval layer. The existing `openclaw docs` surface still searches the live docs index, and production source searches did not find a local `docs.search`/`docs.get`/`docs_lookup` style agent tool or versioned local docs search index.

## Best Possible Solution

Keep #71301 open, but narrow it around the missing product surface: add a first-class local docs retrieval/index layer for agents, backed by the installed `docs/` bundle and generated schema/help data, with explicit fallback to live docs only when requested or missing locally. The likely implementation belongs near the agent tool/gateway tool surface plus docs packaging/release checks, reusing the existing docs path resolver and `config.schema.lookup` contract rather than treating ad hoc `Read`/`rg` guidance as the final API.
