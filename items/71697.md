## Summary

Keep #71697 open. The prior ClawSweeper comment’s distinction from #66505 still holds on current main: OpenClaw documents and supports direct Ollama Cloud native config, but the native `api: "ollama"` agent/Gateway path still forwards replay history to `/api/chat` without an Ollama-specific alternation policy or converter-level normalization.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer solution is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that reshapes non-alternating replay history into an Ollama Cloud-compatible sequence while preserving native Ollama tool-call behavior, with regression coverage for an agent/Gateway history that currently reaches `/api/chat` as non-alternating roles.
