## Summary

Keep #71697 open. Current main still supports direct Ollama Cloud setup, but the native `api: "ollama"` agent/Gateway path still forwards replay history to `/api/chat` without an Ollama-owned strict user/assistant alternation policy. The prior Codex review comment is still accurate against current HEAD; no later commit changed the relevant Ollama stream, replay-policy, transcript-policy, or Ollama docs files.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. Add an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that produces Ollama Cloud-compatible user/assistant alternation while preserving native Ollama tool-call behavior, then add regression coverage for the agent/Gateway replay history that currently reaches `/api/chat` in a rejected order.
