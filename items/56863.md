## Summary

Keep PR #56863 open. Current main still lacks the PR's core WhatsApp outbound mention implementation and quoted reply-target media injection. The existing code handles inbound `mentionedJids`, group metadata, and ordinary outbound sends, but it does not derive outbound `@+<digits>` mention JIDs, pass Baileys `mentions` metadata on text/media replies, build a participant phone-to-LID map, or expose quoted media paths to agent context. The PR discussion also contains useful follow-up validation and unresolved implementation concerns, so this is still actionable review/implementation work rather than cleanup-close material.

## Best Possible Solution

Keep #56863 open or replace it with a refreshed PR that implements the same WhatsApp-owned behavior on current main: outbound mention extraction/resolution for auto-reply and proactive send paths, participant phone-to-JID/LID mapping via existing group metadata resolution, tests for LID/hosted.lid/code-span/duplicate cases, and quoted reply-target media injection with correct fallback MIME handling.
