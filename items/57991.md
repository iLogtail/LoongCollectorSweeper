Keep the PR open for maintainer judgment, but require it to be narrowed or revised before merge: preserve the useful structured logging/status direction, fix the control socket safety and permission issues, make reload semantics honest for running speech settings, add focused tests for CLI JSON/status/health behavior, and avoid unrelated Package.resolved churn.
