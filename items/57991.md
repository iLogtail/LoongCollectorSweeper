## Summary

Keep PR #57991 open. Current main does not already implement the PR's central Swabble features: structured JSON logging with PII redaction, richer status/health output, runtime status snapshots, a local control socket, or in-process config reload. The PR is also not clearly duplicated or superseded by the provided related items, and Swabble docs still list these capabilities as roadmap/open-decision work. It needs maintainer review and likely splitting/hardening, not automated closure.

## Best Possible Solution

Keep the PR open for explicit maintainer judgment. The best path is to split or review it as focused Swabble work: first land structured logging/redaction with backward-compatible config decoding and tests, then status/health output, then a separately hardened local control socket and reload path with owner-only permissions, timeouts, descriptor hygiene, correct exit semantics, docs, and macOS validation.
