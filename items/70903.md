## Summary

Keep open. Current main has partial mitigations, but the exact reported shape can still remain blocked: billing failures persist `disabledUntil` in `auth-state.json`, default billing disables are still hours long, and a primary plus same-provider fallback sharing the same auth profile is treated as a fallback chain, so it only probes billing recovery near expiry rather than soon after the user tops up credit.

## Best Possible Solution

Keep #70903 open. The best fix likely belongs in `src/agents/model-fallback.ts`: treat fallback chains with no actually usable alternate provider/profile as recoverable on the same throttled billing probe path as the no-fallback case, then rely on the existing successful-use path to clear persisted `disabledUntil`. Add focused coverage for same-provider/same-profile billing recovery after a top-up.

## Risks / Open Questions
