## Summary

Keep #72341 open. The current main code still matches the reported live-rendering failure: gateway chat deltas are cumulative for a run, Control UI snapshots the current stream into a segment at tool start, then assigns the next cumulative delta directly to the fresh active stream card. The issue also has an independent confirmation comment from Szqub, and I did not find UI regression coverage for cumulative post-tool deltas.

## Best Possible Solution

Add a focused Control UI regression test for a live `text -> tool_start -> text -> tool_start -> text` sequence where chat deltas are cumulative snapshots. Then fix the WebChat live state to render only fresh text after each committed stream segment, either by tracking a consumed text cursor per run/card or by having the gateway emit segment-local live text at tool boundaries. Preserve the existing final history reload path because persisted JSONL already appears correct.
