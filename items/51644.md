## Summary

Keep open. Current main partially covers rapid same-sender text bursts with timing-based inbound debounce, but this issue asks for a broader semantic preprocessing layer, including multi-topic long-message detection/splitting and adaptive behavior. I found no implementation or docs for those semantic pieces, and the linked #51361 discussion covers coalescing only rather than superseding the broader request.
