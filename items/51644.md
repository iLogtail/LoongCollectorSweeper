## Summary

Keep open. Current main already covers the simpler coalescing portion with timed same-sender inbound debouncing, and related #51361/#46445 were closed on that basis, but this issue asks for a broader semantic preprocessing layer: long-message multi-topic detection/splitting and adaptive semantic behavior remain absent from code, config, docs, and tests.

## Best Possible Solution

Keep the issue open but narrow the remaining product path: retain the shipped `messages.inbound` timed debounce for rapid text bursts, then design either an opt-in core semantic inbound preprocessing layer or a first-class inbound transform/split plugin API that can safely merge, rewrite, or split pre-agent turns across bundled channels. Multi-topic detection/splitting and adaptive windows need explicit behavior, config, tests, and docs before this can close.
