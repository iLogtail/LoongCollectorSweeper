## Summary

Keep #62102 open. Current main still rejects root-level `paperclip` on Gateway `agent` params: the `AgentParamsSchema` is closed and has no `paperclip` or generic metadata field, and the `agent` handler validates that schema before dispatch. The already-shipped Paperclip-related change only relaxes `wake` params, not `agent` requests, and the related open PR #69139 is the dedicated unmerged fix path.

## Best Possible Solution

Keep #62102 open as the canonical tracker and review or replace PR #69139 with a narrow additive protocol fix. The likely maintainer path is to accept an optional opaque `paperclip` metadata record on `AgentParamsSchema` with regression coverage, or coordinate a Paperclip adapter change to send that metadata through an already-supported Gateway agent parameter path.
