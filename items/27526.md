## Summary

Keep #27526 open. Current main has useful partial session management, including session listing, label/search filters, runtime overrides, chat session switching, checkpoint branching, and subagent metadata, but it still does not implement the requested WebChat conversation directory with folders/tags/archive/session pinning/recommendations or the requested multi-AI collaboration/committee UX. The issue also has explicit maintainer signal to remain the canonical tracker after duplicate #27603 was consolidated here.

## Best Possible Solution

Continue using #27526 as the canonical maintainer/product tracker. The best path is incremental: define session metadata and protocol support for folders, tags, archive state, session pinning, richer search/recommendations, and directory navigation in WebChat/Control UI; then design an explicit multi-model/role collaboration UX and protocol for comparison, consensus, agenda/minutes, and project-task extraction. Narrow PRs such as REST session listing can remain separate but should not close this umbrella until the central user-facing workflow exists.
