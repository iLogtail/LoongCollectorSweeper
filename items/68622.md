## Summary

Keep #68622 open. Current main does not implement a `sessions_send` plugin hook or equivalent delegation seam, and this remains plausible core plugin-SDK work rather than a ClawHub-only plugin because the missing interception point is inside OpenClaw's core `sessions_send` tool flow.

## Best Possible Solution

Keep #68622 open for maintainer review. The best path is a focused core/plugin-SDK implementation: add a typed `sessions_send` claiming hook, expose all needed public types, call the hook before baseline/core delivery work, use trusted hook output for delegated metadata, and cover declined, direct, and delegated wait paths with regression tests.
