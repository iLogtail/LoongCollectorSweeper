## Summary

Current main still does not implement #36630's central behavior. Signal outbound sends do not accept or emit native Signal quote metadata, the Signal plugin outbound paths do not forward reply metadata into `sendMessageSignal`, and inbound Signal quote handling remains partial: it exposes quote body/sender/is-quote but not `ReplyToId` or richer signal-cli quote payload metadata. The PR remains an active implementation candidate, with recent review/security discussion that needs maintainer handling rather than conservative cleanup closure.

## Best Possible Solution

Keep #36630 open. The best path is to land a reviewed, hardened version of the Signal quote-reply implementation or reimplement the same behavior narrowly: strict decimal/safe Signal quote timestamp and ID validation, quote-author resolution for group replies, quote-aware Signal outbound delivery, inbound `ReplyToId` plus richer quote metadata, and regression coverage for delivery consumption, debounce, and plugin boundary cases.
