## Summary

Current main only partially addresses the request. Feishu already accepts legacy `groupPolicy: "allowall"` and legacy `topicSessionMode` routing, but there is still no `botId` to `appId` compatibility path or Feishu doctor/setup config migration, so the central backward-compatibility request should stay open.

## Best Possible Solution

Keep this open and implement a Feishu-owned compatibility repair path: add a doctor/setup config migration that rewrites `channels.feishu.botId` and account-level `botId` to `appId`, preserves or documents `topicSessionMode` behavior, normalizes `allowall`, adds focused tests, and documents the migration in the Feishu channel or doctor docs. Treat #52885 as separate shipped plugin-SDK compatibility work, not the canonical tracker for this Feishu config migration.
