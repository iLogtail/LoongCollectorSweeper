## Summary

Current main has related Telegram hardening, especially the v2026.4.24 polling-offset fix for restart/long-turn replay, and actual Telegram API failures are logged when a send call is attempted. That is not enough to conservatively close #71299: the maintainer comment explicitly left it open for a possible outbound-dispatch/no-attempt path, and PR #71377's proposed all-attempt outbound logging/route change is closed unmerged and not present on main.

## Best Possible Solution

Keep #71299 open. Ask for or perform a current-main/v2026.4.24+ retest, then either implement a narrow Telegram outbound observability fix in `extensions/telegram/src/bot/delivery.send.ts` with regression coverage for no-attempt/failure visibility, or close only after logs prove the v2026.4.24 polling-offset fix fully resolves the user-visible missing-message behavior.
