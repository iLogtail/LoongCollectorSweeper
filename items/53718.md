## Summary

Keep open. Current `main` and latest release still leave `$VAR`/`${VAR}` path segments literal in `resolveHomeRelativePath`, the install path flow still routes through that helper, and the linked issue includes a concrete reproduction showing a real `${XDG_CONFIG_HOME}` directory was created. This PR remains relevant; at most it now overlaps with a newer open PR, which is not a valid cleanup-close reason here.
