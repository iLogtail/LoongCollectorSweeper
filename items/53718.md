## Summary

Keep open. Current main still does not expand `$VAR` or `${VAR}` in the shared `resolveUserPath` path resolver, and the PR contains a substantive fix with regression coverage for the linked ClawHub skill-install bug. Current config-level `${VAR}` substitution is narrower and does not implement the requested shared path behavior or bare `$VAR` handling.

## Best Possible Solution

Keep the PR open for maintainer consolidation. The best path is to land one conservative shared path-expansion fix in `src/infra/home-dir.ts`, preserve existing tilde and unknown-placeholder behavior, include regression tests for both `resolveHomeRelativePath` and `resolveOsHomeRelativePath`, then close or supersede overlapping PRs such as #58281 only after a canonical patch is chosen.
