## Summary

Keep open. Current main has substantial partial fixes for the LaunchAgent recovery and cron run observability pieces, but I did not find high-confidence evidence that the full scheduled auto-upgrade/Discord notification problem is solved. The package update path still relies on the global package manager and treats failed global update or doctor steps as update errors; EACCES gets a recovery hint in human output, not a safer automatic update path. Cron now records delivery status, but that is still OpenClaw's send/delivery result, not proof the user visibly received a Discord notification.

## Best Possible Solution

Keep this open and narrow it. Confirm the current LaunchAgent recovery path on macOS, then either close that subproblem separately or note it as fixed. For the remaining work, define a scheduled-update failure notification contract: machine-readable update failures should carry actionable reasons suitable for cron run history and channel delivery, Discord delivery status should mean a verified outbound send rather than assumed user visibility, and npm/global permission failures should be classified as either an external setup problem with docs/support guidance or a product task for a safer package-update path.
