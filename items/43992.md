## Summary

Keep open. Current main still does not implement the PR's central behavior: `agent.wait` terminal snapshots and responses drop the `result` payload, and subagent completion handling still consumes only status/timing/error from `agent.wait` rather than using result text as a fallback when transcript history has not caught up.

## Best Possible Solution

Keep the PR open or replace it with an updated patch against current main. The best fix is to thread structured `result` through `AgentWaitTerminalSnapshot`, `agent.wait` cached/live responses, and `AgentWaitResult`; parse normal agent result shapes such as `{ payloads, meta }`; use the parsed text as a fallback for subagent `frozenResultText` only when no chat-history capture exists; and add regressions for gateway result propagation, structured payload arrays, subagent fallback delivery, and the lifecycle-before-dedupe race.
