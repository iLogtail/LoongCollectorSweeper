## Summary

Keep open. Current main still derives Docker sandbox container names from the sandbox scope key only, not an instance/workspace identity, so multiple co-hosted instances using the same default agent can still target the same container name. The existing workspace-aware config hash detects a mismatch, but it does not make the container name unique and can still leave a hot running container in place.

## Evidence

- **Container name still omits workspace identity:** `ensureSandboxContainer` computes `scopeKey`, then derives `slug` only from that scope key (`shared` or `slugifySessionKey(scopeKey)`) and builds `containerName` from the prefix plus that slug. `workspaceDir` is included in `expectedHash`, but not in the name.
  - file: [src/agents/sandbox/docker.ts:530](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/agents/sandbox/docker.ts#L530)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '524,603p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Slug hash is only based on the passed key:** `slugifySessionKey` hashes the trimmed input string only. For the default agent scope, that input remains `agent:main`, so different HOME/workspace instances still get the same slug.
  - file: [src/agents/sandbox/shared.ts:8](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/agents/sandbox/shared.ts#L8)
  - command: `nl -ba src/agents/sandbox/shared.ts | sed -n '1,45p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Hash mismatch path does not make containers unique:** When the existing container's config hash differs, a hot running container only logs a recreate hint and stays under the same name; the cold path removes and recreates the same `containerName`. That does not resolve cross-instance name collision.
  - file: [src/agents/sandbox/docker.ts:553](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/agents/sandbox/docker.ts#L553)

  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Browser sandbox uses the same name pattern:** `ensureSandboxBrowser` also derives its container name from `scopeKey` alone before computing a workspace-aware config hash, so the related browser-container side described in PR #51380 is not implemented on main either.
  - file: [src/agents/sandbox/browser.ts:172](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/agents/sandbox/browser.ts#L172)
