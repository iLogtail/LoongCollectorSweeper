## Summary

Current main does not show a reachable path where `params.model` is undefined when `normalizeResolvedModel` is called. The linked bug report’s broader unknown-model failure path is handled before runner setup, and the PR’s private guard would be defensive dead code against current typed call sites rather than a demonstrated fix.

## Best Possible Solution

Close this PR as obsolete for current main. If maintainers want additional hardening, the better path is a new focused issue or PR with a failing regression test for a reachable malformed plugin/registry path, then validate model shape at the plugin or registry boundary instead of adding a private guard that current call sites should never hit.
