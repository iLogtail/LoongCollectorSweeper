## Summary

Keep open. Current main still runs the legacy iMessage integration by spawning `imsg rpc` from the Node/OpenClaw process, and the user-facing docs still state that Full Disk Access is required for the process context running OpenClaw/`imsg`. The requested behavior is therefore not implemented on main, the issue is only about 45 days old, and there is no duplicate, maintainer disposition, or ClawHub plugin/workaround in the provided discussion.
