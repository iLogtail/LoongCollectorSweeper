## Summary

Keep open. Current main still has the separate-report heading problem: REM preview body lines are correctly `###` for inline daily notes, but the same body is written under a standalone `# REM Sleep` report without adjustment. The PR's latest diff is a narrow, still-useful fix that preserves inline nesting and lowers headings only for separate REM reports, with targeted regression coverage.

## Best Possible Solution

Keep the PR open and either land this patch or an equivalent change: preserve `previewRemDreaming()` body headings as `###` for inline daily-note blocks, adjust only the standalone REM report body to `##` under `# REM Sleep`, and keep the added regression tests for both output modes.
