## Summary

Keep open. Current `main` still lacks Feishu lifecycle/status reporting into the gateway runtime snapshot, so this PR is not obsolete. However, the current PR should be revised before merge: current `main` health policy keys stale-socket detection off `lastTransportActivityAt`, not app-level `lastEventAt`, and the provided PR discussion includes an unresolved P1 that the callback was not actually threaded through `monitorSingleAccount`.

## Best Possible Solution

Keep this PR open, but ask for an updated patch against current main: pass `ctx.setStatus` through `monitorFeishuProvider` and `monitorSingleAccount` to the transport/event layer, publish `connected: true/false` on lifecycle transitions, and use `lastTransportActivityAt` for actual Lark WebSocket/ping/transport activity if stale-socket detection is the target. Add focused Feishu lifecycle tests that fail on current main and verify the gateway health policy sees the new snapshot fields.
