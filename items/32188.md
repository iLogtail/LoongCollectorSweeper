## Summary

Issue #32188 remains a valid enhancement. Current main has several related surfaces for model selection, slash-command usage/status, per-message metadata, context warnings, usage pages, overview auth cards, CLI usage, and macOS menu usage, but it still does not provide the requested persistent in-interface status area showing the active agent/model plus live token/quota/subscription context during operations.

## Best Possible Solution

Keep the enhancement open and implement a first-class persistent status surface in the chat/application chrome. It should show active agent/session/run status and selected provider/model, then add privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback information using existing sessions.list, usage.status, model/auth, and session event data where possible.
