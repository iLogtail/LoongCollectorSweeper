## Summary

Keep open. Current main has overlapping cron, task-ledger, failure-alert, and reminder-guard behavior, but it still does not provide the requested native commitment primitive: no `openclaw commit` command/API, no durable commitment ledger with `done`, no overdue commitment surface, and no product semantics tying conversational commitments to completion/escalation. Related PENDING.md reports are open workaround/docs follow-ups, not a shipped or canonical replacement.

## Best Possible Solution

Keep #41833 open for maintainer/product design. The best path is to define a commitment layer on top of existing automation primitives: durable commitment records, `add/list/done` CLI and gateway methods if core-owned, due scheduling through cron, execution state through tasks/run logs, explicit overdue/escalation notification semantics, and docs that explain how it differs from cron, tasks, standing orders, heartbeat, and the current reminder guard. If maintainers want this outside core, first identify the exact plugin/API hook needed so a ClawHub plugin can create, track, surface, and escalate commitments safely.
