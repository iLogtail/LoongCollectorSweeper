## Summary

Keep open. Current main still routes `check:docs` through the Unix `git ls-files ... | xargs oxfmt` docs-format pipeline, and no replacement docs-format helper has landed. The PR addresses a real unresolved Windows/PowerShell tooling bug; its discussion also shows the author addressed the early bot-review issues, so it is still potentially useful implementation work rather than an obsolete cleanup target.

## Best Possible Solution

Keep #48887 open for maintainer review or ask the author to amend/rebase it into the canonical fix. The best implementation is one docs-format wrapper that removes `xargs`, preserves the tracked docs file set, handles Windows command shims using the repo's existing process-launch pattern, reports child-process launch errors clearly, and preferably covers both check and write modes. After one corrected implementation lands, close the remaining duplicate PRs against that shipped fix and the canonical issue #44293.
