## Summary

Keep open. Current main still matches the reported bug: `openclaw security audit` builds denylist `knownCommands` from `resolveNodeCommandAllowlist()` across platform defaults, while the platform defaults intentionally exclude `DEFAULT_DANGEROUS_NODE_COMMANDS`. A config that only denies `camera.snap`, `screen.record`, `sms.search`, etc. can therefore still be classified as unknown/ineffective instead of accepted as valid dangerous command names.

## Best Possible Solution

Keep this issue open and fix the audit vocabulary in `src/security/audit-extra.sync.ts`: build `knownCommands` from the full node command catalog or union `DEFAULT_DANGEROUS_NODE_COMMANDS` into the audit vocabulary, then add a regression test showing `gateway.nodes.denyCommands: DEFAULT_DANGEROUS_NODE_COMMANDS` emits no `gateway.nodes.deny_commands_ineffective` finding while typo and pattern cases still warn. Coordinate with the still-open invalid-denyCommands validation work (#50011/#50076) so write-time validation and security-audit suggestions use the same command vocabulary.
