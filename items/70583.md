## Summary

Current main already fixes the user-facing stale Telegram model picker problem via gateway hot-reload cache invalidation. The PR's exact `useCache: false` caller change is not present, but model-related config edits now clear the shared catalog cache and regression coverage verifies newly added models become visible in-process without a gateway restart.

## Best Possible Solution

Close this PR as implemented on main and keep the broader hot-reload cache invalidation as the shipped fix for #69750. If maintainers still want every interactive model menu to bypass the cache on each invocation, that should be a separate performance/consistency decision rather than this stale-config bug fix.
