## Summary

Keep open. Current main still has the exact Feishu/Lark false-positive path: `checkBotMentioned()` returns true whenever raw message content contains `@_all`, and `parseFeishuMessageEvent()` feeds that directly into `mentionedBot`. The related discussion is useful: several PRs attempted this cleanup, and #44256 is still open for the same remaining fix, but the observable bug is not implemented on main or in the latest release tag inspected.
