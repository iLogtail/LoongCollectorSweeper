## Summary

Keep #37706 open. Current main still treats Feishu/Lark raw content containing `@_all` as a bot mention, exposes that through `mentionedBot`, and uses that flag for group `requireMention` gating. The related PRs and comments support the diagnosis, but no fix is present on current main or the latest release tag.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. Land a focused Feishu plugin fix, likely through #44256 or a refreshed equivalent, that removes the raw `@_all` bot-mention shortcut, relies on actual mention metadata and post `at` tags for real bot mentions, and adds regression coverage for broadcast-only and broadcast-plus-bot-mention cases.
