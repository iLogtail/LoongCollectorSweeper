## Summary

Keep open. Current `main` already fixed the CLI side of shared-workspace deletion, but the gateway `agents.delete` path still unconditionally trashes `workspaceDir`, so this PR still targets a live bug. The patch likely needs refresh because `main` now uses broader overlap-based protection in CLI than this PR's exact-match helper.

## Evidence

- **Original report was a real data-loss bug:** Issue #53705 reports that `openclaw agents delete --force` moved a shared workspace to Trash on OpenClaw 2026.3.23-2, deleting shared agent files and memory.
  - command: `web open https://github.com/openclaw/openclaw/issues/53705`
- **CLI protection is already on current main:** The CLI delete command now computes overlapping workspace ownership and skips workspace trashing when another agent could depend on that path; it also reports `workspaceRetained` in JSON output.
  - file: [src/commands/agents.commands.delete.ts:145](https://github.com/openclaw/openclaw/blob/443b837bd521b0485c87bb7359f370a4ccec10fa/src/commands/agents.commands.delete.ts#L145)
  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)
- **CLI regression coverage and changelog exist on main:** Current tests cover same-path, child-path, parent-path, and symlink workspace sharing for `agents delete`, and the changelog records this as `#70897` fixing `#70889` and `#70890`. That means the CLI half of this PR is already superseded on `main`.
  - file: [src/commands/agents.delete.test.ts:188](https://github.com/openclaw/openclaw/blob/443b837bd521b0485c87bb7359f370a4ccec10fa/src/commands/agents.delete.test.ts#L188)
  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)
- **Gateway path is still unsafe on main:** `agents.delete` in the gateway still defaults `deleteFiles` to `true` and unconditionally calls `moveToTrashBestEffort(workspaceDir)` after pruning config, with no shared-workspace guard.
  - file: [src/gateway/server-methods/agents.ts:645](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-methods/agents.ts#L645)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Gateway tests do not cover shared-workspace preservation:** The current gateway delete tests only cover default deletion, `deleteFiles: false`, invalid main-agent deletion, nonexistent agents, and invalid params. There is no regression test for preserving a shared workspace.
  - file: [src/gateway/server-methods/agents-mutate.test.ts:972](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-methods/agents-mutate.test.ts#L972)

## Risks / Open Questions

- Closing this as `implemented_on_main` would miss a real remaining bug in the gateway/UI delete surface, where shared workspaces can still be trashed.
- The PR's `resolveAgentIdsByExactWorkspacePath` approach is narrower than current CLI behavior on `main`, which now protects overlapping parent/child and symlinked workspaces too; the patch likely needs rebasing or widening before merge.
