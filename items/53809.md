## Summary

Keep open. Current main has shipped a broader CLI fix for shared-workspace deletion, but the gateway `agents.delete` path still defaults `deleteFiles` to true and unconditionally trashes `workspaceDir`. This PR's gateway portion remains relevant, although it should be refreshed to match main's overlap-based CLI behavior rather than the PR's narrower exact-path helper.

## Best Possible Solution

Keep the PR open or replace it with a refreshed gateway-focused fix: factor or reuse main's current overlap-aware workspace-retention logic for `src/gateway/server-methods/agents.ts`, add a gateway regression test for shared workspace preservation, and avoid narrowing behavior to exact-path-only matching.
