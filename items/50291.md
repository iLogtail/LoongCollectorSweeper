## Summary

Keep open. Current main has partial progress: runId is now injected into legacy before_agent_start and agent_end, and a DiagnosticTraceContext carrier exists. But the requested plugin-hook observability surface is still incomplete: PluginHookAgentContext does not carry messageId/senderId or direct span/callDepth fields, message_sending lacks event-level correlation and one generic outbound path passes only channel/account/conversation context, and subagent hooks still do not expose requesterRunId/childRunId pairs for runId-based parent-child tracing. Related PR #70924 is a foundation slice, not a complete fix for this issue.

## Best Possible Solution

Keep the issue open and implement a backward-compatible plugin-hook correlation slice: add messageId/senderId and trace/span/callDepth access consistently where core has that data, make runId consistently available on agent lifecycle hooks, propagate session/run/message context through all message_sending/message_sent paths, and add explicit childRunId/requesterRunId fields to subagent lifecycle hooks. Update docs and SDK baselines/tests alongside the public hook type changes.

- Message hook correlation likely needs both inbound-dispatch and generic outbound-delivery paths updated, not just type additions.
