## Summary

Keep open. Current main appears to cover the related #15091-style skill prompt/snapshot loading path for isolated cron, but #43120’s Windows/global npm command PATH failure is still plausible and not fully solved in core. The provided comment identifies the same remaining surface as a Windows Scheduled Task service-environment PATH/runtime mismatch, and current code still skips Windows login-shell PATH merging and avoids writing PATH into the Windows task script/service environment.

## Evidence

- **Issue context:** #43120 reports Windows isolated cron jobs cannot find/use a workspace-installed ClawHub skill even though `openclaw skills list` says ready; the only issue comment points to Windows Scheduled Task PATH/runtime mismatch and suggests reinstalling the gateway or using main-session cron as a workaround.
