## Summary

Keep open. Current `main` still shows the pre-fix behavior described by PR #55351: bare session keys are canonicalized to `agent:<id>:...`, gateway lookup does not probe legacy UUID aliases, Control UI reads/writes raw `session` query params without UUID normalization, and `sessions.get` still reads transcripts without threading agent context into `readSessionMessages`. That is not strong evidence for `implemented_on_main` or `cannot_reproduce`.
