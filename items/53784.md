## Summary

Keep open. Current `main` still routes ordinary MP3/WAV/M4A-style Feishu sends as generic files unless the caller explicitly opts into `audioAsVoice` transcoding, so this PR's requested/default behavior is not already implemented or obsolete.

## Evidence

- **Current Feishu routing still recognizes only native Opus/Ogg as direct audio:** `resolveFeishuOutboundMediaKind()` returns `msgType: "audio"` only for `.opus`/`.ogg` or `audio/ogg`/`audio/opus`; other audio formats fall through to generic file-type handling.

  - file: [extensions/feishu/src/media.ts:518](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/media.ts#L518)
