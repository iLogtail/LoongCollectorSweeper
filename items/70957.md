## Summary

Keep open. Current main still hard-blocks `qwen3.6-plus` on Qwen Coding Plan endpoints in the catalog, config normalization, runtime suppression, tests, and docs. PR #70957 is still relevant, though it is narrower than the full remaining behavior because it only changes the catalog and catalog test.

## Best Possible Solution

Keep #70957 open unless maintainers explicitly choose #66367 or another broader patch as canonical. The final fix should remove or revise all hard-coded `qwen3.6-plus` Coding Plan guards in `extensions/qwen/models.ts` and `extensions/qwen/index.ts`, update `extensions/qwen/provider-catalog.test.ts`, and align the public Qwen docs with the verified Alibaba/Qwen Coding Plan contract.
