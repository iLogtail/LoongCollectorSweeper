## Summary

Keep #12398 open. Current main still lacks the requested deterministic cron postcondition flow: no `postconditions` job field, no persisted verifier state/output/retry count, no verifier execution after isolated agent completion and before cron state application, and no checker-driven retry/escalation path for a clean `ok` agent run.

## Best Possible Solution

Keep #12398 open for maintainer/product implementation. The likely path is an additive cron job contract and gateway schema for postconditions, deterministic verifier execution after isolated agent completion and before `applyJobResult`, persisted verifier state plus run-log/event fields, retry prompt injection with checker output, and integration with existing cron retry and failure-alert delivery. If maintainers prefer plugin-owned implementation, the prerequisite is a first-class cron post-run hook/API that can fail a cron run and schedule cron-owned retry/escalation.
