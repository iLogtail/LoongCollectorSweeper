## Summary

Current `main` already implements the Feishu `/stop` out-of-band queue behavior this PR targets, with the monitor wired through a dedicated sequential-key helper, regression coverage on `main`, and a shipped changelog entry in `2026.4.10`. The PR is superseded by the current implementation.

## Evidence

- **Feishu aborts route to a dedicated control lane on main:** `getFeishuSequentialKey(...)` parses the Feishu message and sends abort text detected by `isAbortRequestText(...)` to `feishu:<account>:<chat>:control`; normal messages stay on the base per-chat lane.
