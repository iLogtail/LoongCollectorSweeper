## Summary

Keep open. Current main has important server-side `chat.history` payload caps and markdown/rendering hardening, but it does not implement this PR's remaining UI-side total render-character budget. The current Control UI still selects the last 200 history entries by count only, then renders that window; there is no `CHAT_HISTORY_RENDER_CHAR_BUDGET`, `estimateMessageRenderChars`, or equivalent test coverage on main. The PR discussion also shows the author addressed the bot/maintainer review concerns in later PR commits, so the branch appears to contain still-useful focused work, though it needs rebasing onto the current `buildChatItems` split.

## Best Possible Solution

Keep the PR open, rebase the char-budget logic into `ui/src/ui/chat/build-chat-items.ts`, and add focused tests in `ui/src/ui/chat/build-chat-items.test.ts` for count cap, aggregate char budget, tool-result `content`, hidden tool messages, raw-walk cap, and accurate visible history notice. Preserve the existing Gateway `chat.history` truncation as a complementary server-side safeguard rather than treating it as a replacement for the UI render-window budget.
