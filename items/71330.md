## Summary

Keep #71330 open. Current main still hard-codes short-term promotion writes to `MEMORY.md`, and the memory-core config schema, CLI options, docs, and tests do not expose a promotion target path or date-token templating. Existing search indexing can cover configured `memorySearch.extraPaths` or files under `memory/`, but that only supports a workaround; it does not make the promotion destination configurable.

## Best Possible Solution

Add a memory-core-owned `promotionTarget` config, defaulting to `MEMORY.md`, and thread it through `resolveMemoryDeepDreamingConfig`, `ShortTermPromotionDreamingConfig`, `ApplyShortTermPromotionsOptions`, CLI/runtime callers, event metadata, docs, and tests. The target should validate as a safe workspace-relative Markdown path, support documented date tokens if accepted by maintainers, create missing parent directories/files with suitable frontmatter/header, and either automatically include the target in memory indexing or document/configure the required `memorySearch.extraPaths` behavior.
