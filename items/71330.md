## Summary

Keep open. Current `main` still hardcodes short-term promotion writes to `MEMORY.md`, exposes no `promotionTarget`-style config in the `memory-core` plugin schema, and documents deep promotion as `MEMORY.md`-only behavior.

## Evidence

- **Promotion apply path is hardcoded to MEMORY.md:** `applyShortTermPromotions()` sets `const memoryPath = path.join(workspaceDir, "MEMORY.md")` and writes promoted entries there. No alternate target is accepted in the function options.
  - file: [extensions/memory-core/src/short-term-promotion.ts:1537](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/memory-core/src/short-term-promotion.ts#L1537)
  - command: `nl -ba extensions/memory-core/src/short-term-promotion.ts | sed -n '1490,1665p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **memory-core config schema has no promotion target setting:** The bundled `memory-core` plugin schema exposes dreaming toggles, frequency, storage mode, and phase thresholds, but no field for a configurable promotion destination file/path.
  - file: [extensions/memory-core/openclaw.plugin.json:21](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/memory-core/openclaw.plugin.json#L21)
  - command: `nl -ba extensions/memory-core/openclaw.plugin.json | sed -n '1,220p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **User-facing docs still describe deep promotion as MEMORY.md-only:** Docs say long-term promotion writes only to `MEMORY.md`, and the config reference lists dreaming settings without any user-facing target-file option.
  - file: [docs/concepts/dreaming.md:23](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/concepts/dreaming.md#L23)
  - command: `nl -ba docs/concepts/dreaming.md | sed -n '1,120p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Bootstrap truncation concern matches current docs:** The token-use docs say `MEMORY.md` is injected as a bootstrap file and is truncated by `agents.defaults.bootstrapMaxChars` with a default of 12000 characters.
  - file: [docs/reference/token-use.md:24](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/reference/token-use.md#L24)
  - command: `nl -ba docs/reference/token-use.md | sed -n '1,120p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Default indexing does not automatically cover arbitrary root promotion files:** The memory config reference says default QMD indexing is `MEMORY.md` plus `memory/**/*.md`, so a root file like `MEMORY_PROMOTIONS.md` would need extra indexing/search work beyond just changing the writer path.
  - file: [docs/reference/memory-config.md:399](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/reference/memory-config.md#L399)

## Risks / Open Questions

- Closing now would discard a still-unimplemented feature request with code and docs on `main` still anchored to `MEMORY.md`.
- If this is implemented later, it likely needs coordinated changes across writer logic, plugin config/schema, CLI/help text, docs, and default indexing/search behavior for nonstandard target paths.
