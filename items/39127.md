## Summary

Keep #39127 open. Current main still lacks the requested gateway-owned `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract; existing session metadata, `sessions.changed`, and channel busy fields are adjacent but do not satisfy the per-session runtimeActivity/attention/version/stale-run/children API. Related PRs #39438 and #41399 were closed unmerged, so they are prior art rather than shipped fixes.

## Best Possible Solution

Keep #39127 open as the canonical core Gateway API feature. The best path is an additive implementation of `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange`, with exported schemas/types, read-scope authorization, run-id based activity tracking, stale-run cleanup, explicit attention transitions, bounded child rollups across session stores/agents, reconnect resync semantics, public protocol docs, and focused gateway tests. Use closed PRs #39438 and #41399 as prior art, not as shipped or superseding work.
