## Summary

Keep #39127 open. Current main still lacks the requested gateway-owned per-session activity RPCs and WebSocket statechange event; existing nearby activity fields are channel-account scoped or session metadata notifications, not the per-session runtime/attention/stale-run/child-rollup contract described in the issue. The related implementation attempts #39438 and #41399 are useful prior art, but the provided GitHub context shows both were closed unmerged.

## Best Possible Solution

Keep #39127 open as the canonical core/gateway API tracking item. The best implementation path is an additive gateway protocol contract in `src/gateway/protocol/schema/sessions.ts`, validators/exports in `src/gateway/protocol/index.ts` and `src/gateway/protocol/schema/protocol-schemas.ts`, handlers in `src/gateway/server-methods/sessions.ts`, method/scope/event registration in `src/gateway/server-methods-list.ts`, `src/gateway/method-scopes.ts`, and `src/gateway/server-broadcast.ts`, plus focused tests for run transitions, stale-run cleanup, explicit attention states, reconnect/list resync, and cross-agent child rollups. Closed PRs #39438 and #41399 are useful prior art but should not close the issue because neither shipped on main.
