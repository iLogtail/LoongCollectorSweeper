## Summary

Keep #39127 open. Current main still lacks the requested gateway-owned per-session activity RPCs and WebSocket statechange event; existing activity data is either channel-account scoped or lifecycle snapshot metadata, not the run-id-set, attention-state, stale-run, and child-rollup contract requested here.

## Best Possible Solution

Keep #39127 open as the canonical core/gateway API tracking item. The best implementation path is an additive gateway protocol contract in `src/gateway/protocol/schema/sessions.ts`, handlers in `src/gateway/server-methods/sessions.ts`, method/scope/event registration in `src/gateway/server-methods-list.ts`, `src/gateway/method-scopes.ts`, and `src/gateway/server-broadcast.ts`, plus focused tests for run transitions, stale-run cleanup, explicit attention states, reconnect/list resync, and cross-agent child rollups. The closed PRs #39438 and #41399 are useful prior art, but neither shipped on main.
