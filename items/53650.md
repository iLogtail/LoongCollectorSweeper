## Summary

Keep #53650 open. Current main still does not implement source-code memory files or declaration-boundary code chunking: memory indexing accepts Markdown plus enabled multimodal files, and both FTS-only and embedding paths still chunk text through `chunkMarkdown()`. The PR is stale against the current memory-core package/plugin layout and has review feedback to resolve, but the requested capability remains unshipped and potentially useful.

## Best Possible Solution

Keep the PR open or ask the author/maintainer to rebase it onto the current memory-core layout. The best implementation path is to add code-file detection and declaration-boundary chunking in `packages/memory-host-sdk/src/host/internal.ts`, route indexing in `extensions/memory-core/src/memory/manager-embedding-ops.ts`, update watcher filtering in `manager-sync-ops.ts`, and add focused package/extension tests plus memory docs for supported code-file behavior.
