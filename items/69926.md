## Summary

Keep #69926 open. Current main still lacks WhatsApp per-group `allowFrom`: the WhatsApp group config type and strict Zod schema only accept `requireMention`, tool policy, and `systemPrompt`, while inbound sender authorization still resolves one account/channel-level `groupAllowFrom` with `allowFrom` fallback and never looks up `groups[<jid>].allowFrom`. The requested parity is real: Telegram already has per-group sender allowlist behavior, and related #69874/#63302 track different issues rather than superseding this feature.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin and config contract: add optional `allowFrom` to `WhatsAppGroupConfig` and `WhatsAppGroupEntrySchema`, regenerate config metadata, update WhatsApp/group docs, resolve the effective root/account group entry for the inbound group JID, and use `groups[<jid>].allowFrom` as an override over account/channel `groupAllowFrom`. Add regression tests for allow, block, wildcard, empty override, fallback, and account-scoped group-map behavior. Keep #69926 open until that lands or a narrower PR tracks it.
