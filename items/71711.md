## Summary

Keep open. Current main does not implement the requested `activeUntil` / `expiresAt` expiration behavior for recurring cron jobs. The core cron model, gateway protocol schema, CLI add/list surfaces, docs, and scheduler result handling expose normal schedule kinds plus `deleteAfterRun` for one-shot jobs, but no expiration timestamp, `onExpire`, or `cron-expired` ledger behavior. The cross-reference to #71712 is relevant, but it is an open sister RFC rather than a canonical duplicate or superseder. This also should not be closed as ClawHub: the request changes the built-in scheduler lifecycle and persisted cron contract, not an optional plugin capability.
