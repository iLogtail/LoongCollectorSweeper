## Summary

Keep #71711 open. Current main does not implement per-recurring-job `activeUntil`/`expiresAt` or `onExpire`: the cron job contract, Gateway protocol schema, CLI, docs, and scheduler lifecycle still only model `enabled`, `deleteAfterRun`, schedule state, and one-shot auto-delete behavior. The only related item in the provided context is open RFC #71712, which depends on this idea rather than superseding it.

## Best Possible Solution

Implement this in core cron rather than closing it. The likely path is an additive `activeUntil`/`expiresAt` plus `onExpire: "remove" | "disable"` contract in `src/cron` types, Gateway protocol schemas/validators, CLI add/edit/list/show, UI/macOS models if applicable, docs, and focused scheduler tests. The scheduler should check expiration before recurring execution, reject it for `at` jobs, persist remove/disable behavior, and align event/audit naming with the existing cron/task ledger. Coordinate naming and agent-facing policy with open RFC #71712, but keep #71711 as the concrete core scheduler feature.
