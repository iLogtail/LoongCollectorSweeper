## Summary

Keep #71711 open. Current main does not implement a recurring-job expiration field: cron job types, protocol schema, CLI add params, docs, and scheduler execution paths have no `activeUntil`/`expiresAt`/`onExpire` equivalent. The only built-in cleanup path is still `deleteAfterRun` for successful one-shot `at` jobs. The cross-reference to #71712 is related but broader and open, not a canonical duplicate that replaces this concrete operator-facing cron lifecycle request.

## Best Possible Solution

Keep the issue open for maintainer/product decision and a focused implementation in the core cron scheduler: add an optional recurring-job expiration field to cron types/protocol/normalization, expose CLI and UI/Mac edit support, reject it for `at` jobs, check it before scheduled recurring runs, apply remove/disable behavior, document it, and add service/protocol/CLI tests. Coordinate terminology with the broader open #71712 scheduling RFC, but do not treat that RFC as a replacement for this operator-facing feature.
