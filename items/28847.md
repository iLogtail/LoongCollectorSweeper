## Summary

Keep open. Current main has meaningful per-profile cooldown/backoff, rate-limit classification, auth-profile ordering, and generic API-key rotation, but it still does not implement the requested Gemini/project quota-bucket grouping. The issue discussion also points to useful prior art: pyro0ownz’s sidecar and aashari’s fork, while aashari explicitly notes bucket/project grouping is not native there either.

## Best Possible Solution

Keep this open until OpenClaw either adds first-class quota-bucket metadata to auth profiles and applies bucket-level cooldown fanout in the provider/auth-profile failover path, with Gemini multi-project tests/docs, or maintainers explicitly choose a sidecar/plugin design and add any missing extension API needed to implement it cleanly outside core.
