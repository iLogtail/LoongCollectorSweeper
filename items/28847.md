## Summary

Keep open. Current main has useful adjacent behavior: multiple auth profiles, per-profile cooldown/backoff, 429/RESOURCE_EXHAUSTED rate-limit classification, profile ordering, model-scoped rate-limit cooldowns, and limited same-provider probe throttling. It still does not implement the central requested Gemini quota bucket/project grouping: there is no first-class quotaBucket/quotaGroup/project-bucket metadata in auth config, no bucket-level state, and no failure fanout that cools every key in the same project before selecting another bucket.

## Best Possible Solution

Keep the issue open. The best product path is to add explicit quotaBucket/quotaGroup metadata for auth profiles or provider-owned external profiles, then teach markAuthProfileFailure, resolveAuthProfileOrder, and the runtime auth failover path to cool and skip all profiles in the same Gemini project bucket before trying a different bucket. Add Gemini multi-project tests and docs, or make an explicit maintainer decision to expose a sidecar/plugin API that can update shared auth-health state safely outside core.
