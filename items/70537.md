## Summary

Keep #70537 open. The issue is recent, has concrete reproduction details for direct Anthropic via Telegram/gateway, and the provided triage comment marked it as a valid bug. Current main does not show a targeted fix or release note for this direct Anthropic array-argument failure; existing malformed tool-call argument repair is scoped to Kimi anthropic-messages and openai-completions, not `anthropic:default`.

## Best Possible Solution

Keep the issue open and ask for or add a focused regression around direct Anthropic `input_json_delta` tool calls whose arguments include arrays of objects. If reproduced, fix the provider/runner tool-argument path so malformed streamed JSON is either repaired deterministically before dispatch or fails with a clearer retryable provider-stream error; then cover the exact direct Anthropic + array-field case in tests and note the fix in the changelog.
