## Summary

Keep #70807 open. Current main does not implement the PR's central timeout propagation or media timeout change, and the Ollama tooling portion is not present. The PR is recent, concrete, and not superseded by the provided related items, but it should be revised before merge because review comments identify real issues in the proposed implementation.

## Best Possible Solution

Keep the PR open for revision. The best path is to split or tighten the gateway fix: pass a runtime-only agent timeout into the compat chat handler without widening the user config schema, cover both endpoint-config and CLI/runtime-enable paths with regression tests, and validate timeout semantics consistently with `agent-command.ts`. Handle the media timeout as a deliberate provider/media policy change. For the Ollama tooling, either move it to docs/ClawHub-style optional tooling or fix the socket cleanup, configurable host/port defaults, 11434 default alignment, and comments before considering it for core scripts.
