## Summary

Keep #70807 open. Current main still does not include the PR's compat timeout propagation, media HTTP timeout widening, or Ollama helper scripts, and the PR has substantive unresolved review feedback that should be addressed rather than auto-closed.

## Best Possible Solution

Keep the PR open for author or maintainer revision. The best path is to split or tighten the core timeout fix, pass runtime timeout through a schema-aligned runtime-only option that also works when the endpoint is enabled by startup override, add/adjust focused regression coverage, and either fix or move the Ollama helper scripts after addressing fd cleanup and documented 11434 defaults.
