## Summary

Keep open. Current `main` has the newer WhatsApp login controller and restart-on-515 handling, but it still does not implement the PR's QR-rotation refresh or Control UI polling/QR-refresh loop. The PR discussion also contains unresolved review findings on the proposed tests and timeout handling, so there is not strong evidence for a conservative close.

## Evidence

- **Control UI still does a single long wait call:** `waitWhatsAppLogin` still makes one `web.login.wait` request with `timeoutMs: 120000` and only clears the QR on success. There is no polling loop and no `web.login.start` refresh path, so the relink flow described in the PR is not implemented on `main`.
