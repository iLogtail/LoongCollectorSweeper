## Summary

Keep #57053 open. Current `main` still exposes `session_status` primarily as a `/status`-style text card with only minimal metadata in `details`; it does not provide structured `remaining_seconds`, `elapsed_seconds`, `session_timeout_seconds`, or usable remaining context/token budget fields. The related #57705 is adjacent concurrency-capacity work, not a duplicate of this session time/token budget request.

## Best Possible Solution

Keep the issue open and implement this in core session tooling, either by adding structured `budget` fields to `session_status.details` while preserving the current text card, or by adding a small `session_budget`/capacity-style tool. The implementation should compute wall-clock remaining time from the active run timeout/start state where available, expose context-token remaining from the same context accounting used by `/status`, document unavailable/unknown values explicitly, and add focused tests in `src/agents/openclaw-tools.session-status.test.ts` plus docs under the public session tools page. Coordinate the concurrency portion with #57705 rather than treating it as already solved here.
