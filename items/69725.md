## Summary

Keep PR #69725 open. Current main still unconditionally sends `system.run.prepare` before `system.run` for `exec host=node`, while the macOS companion app still advertises and handles `system.run` without `system.run.prepare`. The PR is not implemented on main, and the related open macOS PR/issue do not conservatively supersede this agent-side fallback.

## Best Possible Solution

Keep this open for maintainer review. The best product fix is either a complete macOS companion `system.run.prepare` implementation with advertisement, runtime handling, and tests, or this PR's backwards-compatible agent-side local-plan fallback for legacy nodes that advertise `system.run` without prepare. In either path, preserve the `systemRunPlan` approval contract and add regression coverage for the macOS command list that lacks `system.run.prepare`.
