## Summary

Keep open. Current main still contains the reported mismatch: `exec host=node` only checks for `system.run` and then unconditionally invokes `system.run.prepare`, while the current macOS node command list still does not advertise or implement `system.run.prepare`. The PR remains a plausible focused fix, and the related open issue/PR are adjacent alternatives rather than a merged or shipped superseding change.

## Best Possible Solution

Keep #69725 open for maintainer review. The strongest path is to decide the compatibility contract explicitly: either merge a local-plan fallback for legacy macOS nodes, implement and ship `system.run.prepare` in the macOS node, or combine both with regression coverage so `exec host=node` works for existing paired macOS apps and remains plan-bound for newer node hosts.
