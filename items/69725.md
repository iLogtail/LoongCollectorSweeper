## Summary

Keep PR #69725 open. Current main still unconditionally calls `system.run.prepare` in the agent `exec host=node` path, while the macOS companion node advertises and dispatches `system.run` without `system.run.prepare`. The PR is a plausible focused fix for a real current compatibility bug, not something already implemented on main or safely superseded by the related open macOS-side PR #38781.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a compatibility fallback in `src/agents/bash-tools.exec-host-node.ts` for nodes that declare `system.run` but not `system.run.prepare`, or land the macOS prepare implementation plus an explicit legacy-node compatibility decision. Any accepted fix should include tests for a macOS-style command list of `system.run`, `system.which`, and `system.notify`, and should keep the existing remote `system.run.prepare` path covered for nodes that support it.
