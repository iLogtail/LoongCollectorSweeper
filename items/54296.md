## Summary

Keep open. Current main already fixes the exec-approvals atomic write concern, but the transcript side of the report is still present in the gateway idempotency and manual compaction paths, plus related async session transcript helpers. No provided related item supersedes this specific remaining work.

## Best Possible Solution

Keep the issue open and scope the remaining work to transcript readers: replace full-file read/split scans in gateway chat idempotency, session transcript idempotency/equivalence helpers, and `sessions.compact` maxLines with streaming or bounded tail/head scans, then add regression coverage with oversized transcript files. Leave the current atomic exec-approvals implementation in place.
