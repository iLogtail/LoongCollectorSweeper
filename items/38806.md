## Summary

Keep #38806 open. Current main still gates Feishu thread-aware reply delivery on group chats, so P2P thread messages can still lose the reply target and fall back to a standalone send. The prior ClawSweeper keep-open comment remains accurate against current main b67d9bf7f06f4c655b1e0869395056c7d631abc2, and PR #38808 is still the relevant open fix candidate from the provided GitHub context.

## Best Possible Solution

Keep #38806 open and review/land a targeted Feishu fix like PR #38808: detect direct thread messages with isDirect && Boolean(ctx.threadId), preserve P2P quote-only behavior, pass the thread root/reply_in_thread semantics through the dispatcher for P2P threads, and add regression coverage for both P2P thread messages and P2P plain quote replies.
