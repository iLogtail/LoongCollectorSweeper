## Summary

Keep #38806 open. Current main still preserves Feishu `root_id`/`thread_id` on inbound events, but the reply-routing path remains group-gated, so P2P/direct thread messages cannot set `replyInThread`, do not select the thread root as the reply target, and still strip reply metadata before dispatch. The related context identifies #38808 as the open targeted fix candidate; #38814 and #38996 were closed unmerged.

## Best Possible Solution

Keep #38806 open and resolve it through #38808 or an equivalent Feishu patch: add a direct-thread condition such as `isDirect && Boolean(ctx.threadId)`, use the P2P thread root as the reply target, set `replyInThread`/`threadReply` for that case, preserve reply metadata instead of using `skipReplyToInMessages` for direct threads, keep root_id-only P2P quote replies on the triggering message, and add regression coverage for both cases.
