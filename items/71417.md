## Summary

Current `main` still silently resumes an agent's existing main session when `openclaw agent` is invoked with `--agent <id>` and no `--session-id`. The exact long-gap reproduction depends on session freshness/reset policy, but the core behavior in #71417 remains present on both the gateway and embedded paths, and the current CLI/docs still do not make that resume behavior explicit.

## Evidence

- **CLI help still frames agent-only runs as using the existing main session/channel:** The `agent` command help says omitting `--channel` uses the main session channel, and the examples include `openclaw agent --agent ops --message "Summarize logs"` without any fresh-session flag or warning.

  - file: [src/gateway/server-methods/agent.ts:521](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/gateway/server-methods/agent.ts#L521)
