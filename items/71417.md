## Summary

Keep #71417 open. Current main still defaults an omitted `openclaw agent --agent <id>` run onto the agent's main session key and reuses the existing fresh session id; omitted delivery channel is still normalized to `last`. There is no `--new-session`/`--fresh` flag, no default-to-new behavior, and no visible resume banner documented or implemented for this CLI path.

## Best Possible Solution

Implement an explicit, operator-safe fresh-session path for `openclaw agent`: either make `--agent` without `--session-id` mint a fresh session by default, or add a documented `--new-session`/`--fresh` flag plus a clear stderr banner when an existing session is resumed. The change should cover both Gateway and `--local` paths, update `docs/cli/agent.md` and `docs/tools/agent-send.md`, and add focused tests around `--agent` with and without the new control.
