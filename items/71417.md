## Summary

Keep #71417 open. Current main still routes `openclaw agent --agent <id>` with no `--session-id` to the agent's canonical main session and reuses that session when it is fresh. The related `--session-id` remap path has tests, but it does not cover the no-session-id fresh-run problem described here, and there is still no `--fresh`/`--new-session` flag or resume banner.

## Best Possible Solution

Keep this open for a focused CLI/Gateway UX fix. The best path is to add an explicit fresh-session control such as `--fresh` or `--new-session`, or print a clear resume banner when `--agent` resolves to an existing fresh main session; update `docs/cli/agent.md` and add targeted tests around `agent-via-gateway`, embedded `agentCommand`, and Gateway `agent` session resolution. Do not overload `--channel` unless the product decision is to change its documented delivery-only meaning.
