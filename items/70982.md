## Summary

Keep open. Current main has partially narrowed the report because `models: []` now follows the Ollama discovery path and uses the configured `baseUrl`, but the reported schema/docs contradiction remains: a provider block with only `baseUrl` and no `models` field is still rejected by the source schema, generated schema, and exported config type. Related PRs #71515/#71636 were closed unmerged, and open PR #39766 tracks the broader optional-provider-models contract but has not landed.

## Best Possible Solution

Keep #70982 open until the config contract accepts provider entries that omit `models` and materializes them as an empty array, with `src/config/zod-schema.core.ts`, `src/config/schema.base.generated.ts`, `src/config/types.models.ts`, validation behavior, and docs aligned. Add focused Ollama coverage proving `models.providers.ollama` with `baseUrl` and omitted `models` discovers `/api/tags` from that configured host, or explicitly document that users must set `models: []` if maintainers choose that contract instead.
