## Summary

Keep open. Current `main` partially fixes the runtime side of the report by allowing remote Ollama discovery when `models.providers.ollama` includes a custom `baseUrl` and `models: []`, but the reported docs/schema contradiction is still present: the config schema still requires `models`, while the Ollama docs still show a custom-base-URL example without `models` and still say explicit provider config disables autodiscovery.

## Evidence

- **Schema still requires `models` in every provider block:** `ModelProviderSchema` still defines `models` as a required array, so a config like `models.providers.ollama = { baseUrl, apiKey, api }` remains invalid at schema-validation time.
  - file: [src/config/zod-schema.core.ts:359](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/zod-schema.core.ts#L359)

  - file: [docs/providers/ollama.md:153](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/providers/ollama.md#L153)
