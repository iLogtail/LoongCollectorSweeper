## Summary

Keep open. Current `main` still does not expose per-run usage on the `agent` WebSocket lifecycle stream: lifecycle emitters only send `start`/`end`/`error`, and the documented contract matches that. Main has adjacent passive WS usage metadata on `session.message`/`sessions.changed`, but that is a different surface and does not include the full requested per-run fields such as cache counters or last-call usage. The Bridge and Paperclip discussion in the issue comments shows a real operator-facing consumer, so this is not strong enough for conservative auto-close.
