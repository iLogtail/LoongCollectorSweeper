## Summary

Keep open. Current main still recognizes only `/activation`, not `/activate`, and the WhatsApp group path can still skip `/activate mention` before the command handler in mention-gated groups. The linked fix PR #62838 is closed unmerged and #62841 remains open, so there is no shipped fix on main.

## Best Possible Solution

Keep #62827 open until main accepts `/activate` everywhere `/activation` is accepted: add `/activate` as an activation command text alias, update `parseActivationCommand`, and add regression coverage for `/activate mention`, `/activate always`, colon form behavior, no-slash rejection, and the WhatsApp mention-gating bypass path.
