## Summary

Keep #6625 open. Current main at d1f40731 still exposes only hard sub-agent run timeouts plus post-timeout completion/partial-progress handling; I found no configurable pre-timeout grace window or injected checkpoint warning for native or ACP sub-agents. The existing maintainer/Codex issue comment also explicitly keeps this open and separates it from related partial-result persistence work.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented agents.defaults.subagents warning/grace setting, schedule a warning before runTimeoutSeconds, deliver it through the existing child-session control/message path for native and ACP sub-agents where feasible, and add focused tests/docs while preserving current post-timeout partial-progress and fallback-delivery behavior.
