## Summary

Keep PR #69948 open. Current main has the underlying gateway `sessions.send` method, but it still does not expose a plugin-facing `api.runtime.interSession.send` runtime namespace, SDK type, error contract, or docs. The PR is a plausible core plugin API extension rather than an obsolete or ClawHub-only item, and the remaining review feedback should be resolved before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, resolve the remaining runtime cache-mode review, and let maintainers decide whether the minimal send-only `api.runtime.interSession.send` contract belongs in core. If accepted, land it with updated SDK types, API baseline hash, docs, and focused runtime/gateway tests.
