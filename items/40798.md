## Summary

Current `main` still carries `toolCallHistory` forward with no heartbeat-gap reset, so the underlying false-positive report remains plausible. This PR is not implemented on `main`, and its proposed fix still has unresolved review concerns around legitimate `>60s` tool runs/poll intervals, so it should stay open for maintainer judgment rather than cleanup-closing.
