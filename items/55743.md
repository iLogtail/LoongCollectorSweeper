## Summary

Keep open. Current main has adjacent pieces: code blocks have Copy, and real exec/Codex execution requests can raise approval prompts in the Control UI. But the requested Control UI behavior for assistant-suggested terminal commands in ordinary chat text, especially an Approve and Run action on a command card with command/cwd/host/risk metadata, is not implemented on main.

## Best Possible Solution

Keep the issue open and implement it as a Control UI chat command-card surface: detect or receive structured assistant-suggested shell commands, show exact command plus cwd/host/risk metadata, keep the existing Copy affordance, and route Approve and Run through the existing exec approval/gateway execution path so output returns to the conversation with the current approval safeguards.

## Risks / Open Questions

- Adding this feature must not execute sanitized display text or arbitrary markdown; it should create a structured execution request from an exact preserved command, cwd, host, and approval metadata.
- There is overlap with existing exec/plugin approval flows, so the implementation should reuse those approval records and decision semantics instead of creating a parallel unsafe path.
