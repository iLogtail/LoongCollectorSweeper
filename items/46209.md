## Summary

Current main already solves PR #46209's user-visible bug by preserving existing session display name, label/origin, and delivery metadata for heartbeat/cron/exec system-event turns. The PR's `deriveSessionTitle` refactor and label fallback are now redundant for the Control UI selector because gateway session rows already expose preserved `entry.label`/`origin.label`, and the UI selector prefers label/displayName.

## Best Possible Solution

Close PR #46209 as implemented on main. Keep the shipped session-state fix, which addresses the underlying heartbeat metadata overwrite at the mutation boundary; do not merge the now-redundant title-helper refactor unless maintainers separately want that cleanup.
