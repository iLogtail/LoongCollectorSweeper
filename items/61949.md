## Summary

Keep open. Current main still injects the contested Model Aliases section into full system prompts, still tells the model to “Prefer aliases when specifying model overrides,” and builds that list from every configured `agents.defaults.models` alias without distinguishing primary versus fallback or overlapping provider/model entries. The provided issue/comment describes a plausible current-main failure path: the parent model chooses the wrong `sessions_spawn.model` value after seeing alias guidance, and the tool then forwards that string as requested.

## Best Possible Solution

Keep the issue open and fix the core prompt/model-alias behavior. A conservative maintainer patch should stop teaching the parent model to prefer aliases when an exact `provider/model` is supplied, avoid presenting fallback or overlapping provider aliases as interchangeable routing choices, and add focused tests around `buildModelAliasLines`/system prompt rendering plus `sessions_spawn` model override preservation for a GitHub Copilot versus Bedrock overlap case.
