## Summary

Keep open. Current `main` has meaningful partial work for graceful restart handling, but it does not fully satisfy this issue's broader session/cron recovery scope. Active task draining now exists, main-session and subagent restart recovery paths exist, but interrupted cron runs are still marked failed rather than retried/recovered, and the process supervisor explicitly says active runs are not recovered after process restart. A maintainer comment on 2026-04-25 also says PR #71465 only fixed the active-run-kill portion and intentionally left this issue open for the remaining scope.

## Evidence

- **Restart drain is implemented, but only as partial mitigation:** Gateway restart now marks the gateway as draining, waits for active tasks and embedded runs to finish within the drain budget, and preserves queued work across in-process restart. This addresses silent loss during some restart paths, but it is a drain/deferral fix rather than full post-restart session recovery.
