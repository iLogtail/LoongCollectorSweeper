## Summary

Keep open. Current `main` still retains `toolCallHistory` across heartbeat-sized idle gaps, and the loop detector counts the entire retained history without any staleness reset, so the reported false-positive path remains plausible on `bfa6708c03f24c7caa5d0a96a298c292be0d85c1`.

## Evidence

- **Loop detection uses full retained history:** `detectToolCallLoop()` reads `state.toolCallHistory` directly and counts repeated/no-progress calls from that full array. There is no timestamp cutoff or heartbeat-cycle boundary check before warning/block decisions.
  - file: [src/agents/tool-loop-detection.ts:438](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/tool-loop-detection.ts#L438)

  - file: [src/agents/tool-loop-detection.ts:569](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/tool-loop-detection.ts#L569)
