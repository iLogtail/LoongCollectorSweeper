## Summary

Keep open. Current `main` still accumulates `toolCallHistory` across heartbeat turns: loop detection reads the full stored history without any age-based reset, diagnostic session state is retained for 30 minutes, and heartbeats still run in the shared conversation session unless `heartbeat.isolatedSession` is explicitly enabled.

## Evidence

- **Loop detection still ignores staleness:** `detectToolCallLoop()` reads `state.toolCallHistory` directly, and `recordToolCall()` / `recordToolCallOutcome()` append timestamped entries without clearing old ones. There is no time-gap check before counting repeats.
  - file: [src/agents/tool-loop-detection.ts:428](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/tool-loop-detection.ts#L428)
  - command: `nl -ba src/agents/tool-loop-detection.ts | sed -n '428,460p' && nl -ba src/agents/tool-loop-detection.ts | sed -n '569,654p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **State lifetime exceeds reported heartbeat interval:** Diagnostic session state is kept for 30 minutes while idle and reused by `sessionKey`/`sessionId`. A 5 minute heartbeat interval would therefore keep prior tool-call history alive across cycles.
  - file: [src/logging/diagnostic-session-state.ts:30](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/logging/diagnostic-session-state.ts#L30)

- **Heartbeats still share the conversation session by default:** The session code says automated heartbeat turns run inside the conversation session, and `heartbeat-runner` only creates a fresh session when `heartbeat.isolatedSession === true`. That means normal heartbeat cycles keep reusing the same session-backed loop-detection state.

- **Test coverage does not cover stale-gap reset behavior:** `tool-loop-detection` tests verify append behavior, sliding-window size, timestamps, and configured history length, but there is no test asserting that history resets after a large time gap between tool calls.
