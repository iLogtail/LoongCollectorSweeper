## Summary

Keep #70586 open. Current main already solves the stale-socket health failure via transport-activity health policy, and normal Slack messages already update `lastEventAt`/`lastInboundAt` once through `createSlackMessageHandler`. However, the PR still identifies real untracked early-return Slack message branches, while its current patch would double-update normal message/app_mention events. This should be revised, not auto-closed.

## Best Possible Solution

Request changes on #70586 rather than close it: preserve current once-per-normal-message tracking, add `trackEvent` only for accepted message-event branches that return before `handleSlackMessage` if maintainers want those branches reflected in `lastEventAt`/`lastInboundAt`, and add regression tests for early-return subtype/DM branches plus no double-tracking for normal message and app_mention flows.
