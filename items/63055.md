## Summary

Keep open. The report is recent, plausible, and current `main` does not provide strong evidence that the macOS/Homebrew `qmd` spawn failure is fixed or obsolete. Current code still probes QMD by directly spawning the configured command and still falls back to builtin on spawn failure; docs still describe PATH/service-environment troubleshooting rather than a confirmed core fix.

- The reported failure is macOS/Homebrew-specific and could not be safely reproduced in this Linux checkout, so closing as cannot-reproduce would be weak.
