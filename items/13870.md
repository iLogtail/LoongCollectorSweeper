## Summary

Keep #13870 open. Current main can display a paired device `displayName` when client metadata supplies one, but it still does not provide the requested user-facing paired-device rename workflow in the CLI, gateway protocol, server handlers, Control UI, or devices docs. The related #23916 metadata-display request overlaps but does not replace this issue's specific user-settable label request.

## Best Possible Solution

Keep #13870 open until paired-device label management is added end to end: an additive gateway `device.rename` method and schema, an authorized storage update path with a clear precedence rule for client-reported metadata versus operator labels, `openclaw devices rename --device <id> --name <displayName>`, a Control UI rename action, focused protocol/CLI/UI/storage tests, and docs on `https://docs.openclaw.ai/cli/devices`. Keep `node.rename` as a separate paired-node label surface.
