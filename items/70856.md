## Summary

Keep open. Current main has WhatsApp reconnect and delivery-recovery mitigations, but it does not prove the reported Windows/libsignal Bad MAC closed-session failure is fixed, and the active-listener gap can still surface as a runtime send failure while the listener is reconnecting.

## Best Possible Solution

Keep the issue open for a maintainer-owned WhatsApp reliability fix. The likely path is a focused regression test around the reported libsignal `Bad MAC`/closed-session error, safe broadening of the crypto-error reconnect classifier if that signature is recoverable, and a delivery-path review with #37428 so cron/tool sends do not burn retries or fail permanently while the WhatsApp listener is reconnecting.
