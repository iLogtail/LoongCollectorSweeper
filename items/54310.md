## Summary

Keep open. Current `origin/main` still ships `skills/canvas/SKILL.md` without YAML frontmatter, and the current skill loader drops any skill that lacks a frontmatter-derived description. The linked PR #54361 directly tracks the fix but remains open/unmerged in the provided GitHub context, so the underlying shipped defect is still actionable in this repository.

## Best Possible Solution

Keep #54310 open until the bundled canvas skill gains valid YAML frontmatter with `name: canvas` and a trigger-rich `description`, and add focused regression coverage so bundled skills like canvas cannot silently ship without required frontmatter. PR #54361 is the direct follow-up to review or supersede with an equivalent main-branch fix.
