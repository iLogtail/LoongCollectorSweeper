## Summary

Keep #39102 open. Current main still has only the global `tools.agentToAgent.allow` gate for cross-agent session tools; the requested `agents.list[].tools.agentToAgent.allow` requester-specific outbound override is not implemented in policy, schema/types, docs, prune behavior, or tests.

## Best Possible Solution

Keep the PR open or replace it with an updated implementation on current main. The right product path is a strict `agents.list[].tools.agentToAgent.allow` schema/type/docs surface, a policy evaluator that applies global participation first and requester outbound overrides second, consistent denial formatting across `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, immutable pruning of deleted agent ids from per-agent allowlists, and focused tests for fallback, deny-all, outbound-only behavior, global participation, and prune immutability.
