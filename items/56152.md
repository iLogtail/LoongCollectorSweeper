## Summary

Keep open. Current main has not taken this PR's local-media path-variant fix, and the PR as submitted has unresolved security review feedback. The right maintainer path is to revise the fix, not close it as implemented or obsolete.

## Best Possible Solution

Keep the PR open for maintainer/author revision. A safe direction is to keep the canonical media path authoritative when `fs.realpath(mediaPath)` succeeds, use a lexical media candidate only when the media path cannot be realpathed, consider both safe root forms as needed, and add regression tests for `/tmp`/`/private/tmp`, symlink escapes outside allowed roots, and the workspace-sibling restriction.
