## Summary

Keep open: current main still drops trailing text after an unclosed reasoning tag in strict mode, so this is not implemented yet, and the PR still needs maintainer review because its fallback changes preserve-mode behavior used in user-facing sanitization paths.

## Evidence

- **Current main still drops strict-mode trailing text after an unclosed tag:** `stripReasoningTagsFromText()` only appends the trailing slice when `!inThinking || mode === "preserve"`. If parsing ends inside an unclosed `<think>` block in strict mode, the remaining text is discarded.

- **Mainline tests still encode the reported bug:** The current test suite expects strict mode to turn `Before <think>unclosed content after` into `Before`, while preserve mode keeps the trailing text. That confirms the requested behavior change has not shipped on `main`.
