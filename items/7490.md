## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` field: the typed config, strict runtime schema, generated schema, `agents_list` response type/mapper, tests, and docs all still reflect id/name/model/runtime metadata without descriptions. The linked implementation PR #7530 is closed unmerged in the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #7490 open until OpenClaw either implements the additive core config/tool behavior or a maintainer explicitly rejects this product surface. A complete fix should add `description?: string` to `AgentConfig`, accept it in `AgentEntrySchema`, regenerate and label/help/document the config schema, preserve strict validation for other unknown keys, include a trimmed optional `description` in `agents_list` for configured agents, and add focused inclusion/omission tests.
