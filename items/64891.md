## Summary

Keep open. The reported regression still matches current main: Bedrock memory auto-detection calls the AWS SDK default credential provider, and there is no current guard for `AWS_EC2_METADATA_DISABLED` or equivalent IMDS suppression. The linked #64944 PR described the right root cause and fix direction, but it is closed/unmerged in the provided GitHub context and its proposed guardrails are not present in the checked-out code.

## Best Possible Solution

Keep the issue open and apply a narrow Bedrock memory credential-detection fix: make `hasAwsCredentials()` respect `AWS_EC2_METADATA_DISABLED=true`, suppress IMDS probing during the auto-detection `defaultProvider()` call without breaking explicit AWS env/profile/shared-config credentials, and add regression tests in `extensions/amazon-bedrock/embedding-provider.test.ts` for the disabled-IMDS behavior and env restoration.
