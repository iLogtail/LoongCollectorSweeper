## Summary

Keep open. Current `main` still threads native Telegram command replies through the incoming command message, and there is no equivalent `/new` or `/reset` direct-chat exception on `main`. The PR’s approach matches the existing delivery contract, but the stale branch still needs a rebase and the missing `/reset` regression coverage noted in review.

That means direct-chat `/new` and `/reset` still use the same threaded reply policy as every other native command.
