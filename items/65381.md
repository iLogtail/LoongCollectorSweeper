## Summary

Keep open. Current main still undercounts `/status` cost when usage is only or partly in `cacheRead`/`cacheWrite`; the estimator supports cache pricing, but the status renderer only forwards input/output. PR #65381 is outdated because the implementation moved to `src/status/status-message.ts`, but the change is still useful and should be ported rather than closed by automation.

## Best Possible Solution

Keep the bug-fix work open, port the PR to `src/status/status-message.ts` by including `cacheRead` and `cacheWrite` in `hasUsage` and the `estimateUsageCost` usage object, then add the PR's regression case to `src/auto-reply/status.test.ts`. Coordinate with #52329/#65109 so only one refreshed fix lands.
