## Summary

Keep open. Current `main` still has the reported gap: `toToolDefinitions()` calls `runBeforeToolCallHook()` without `ctx`, while the parallel client-tool path already passes hook context. The embedded-runner production path still routes tools through `splitSdkTools()` without any `hookContext` threading, so this PR is not implemented on `main` and remains relevant.
