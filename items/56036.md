## Summary

Keep open. Current main still has the exact fallback gap this PR targets: `toToolDefinitions()` calls `runBeforeToolCallHook()` without `ctx`, `splitSdkTools()` cannot accept or forward hook context, and embedded runner call sites have session/agent metadata available but only pass it to the client-tool path. The fallback is not hypothetical because bundled MCP/LSP tools are materialized as plain AgentTools and then appended to `effectiveTools`, so they rely on `toToolDefinitions()` for hook dispatch.

## Best Possible Solution

Keep the PR open and update/rebase it against current main. The best fix is to add an optional `HookContext` to `toToolDefinitions()`, thread it through `splitSdkTools()`, pass populated context from both attempt and compaction runners, and keep the focused regression test for the `splitSdkTools -> toToolDefinitions -> runBeforeToolCallHook` path. Maintainers should also decide whether to include `loopDetection` in that fallback context before merge.
