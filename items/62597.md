## Summary

Current main has not implemented the PR's optimization. `search()` still calls `hasIndexedContent()` on each search path, and `hasIndexedContent()` still executes direct SQLite probes without an in-memory cache. The PR remains a plausible focused fix for open issue #62513, with the main remaining maintainer concern being dedicated cache/invalidation tests before landing.

## Best Possible Solution

Keep #62597 open. The best path is to rebase it on current `main`, keep the cache/invalidation approach if it still applies cleanly, and add focused memory-core tests proving repeated searches do not re-query SQLite until writes/deletes/resets invalidate the cached result.
