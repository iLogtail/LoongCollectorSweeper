## Summary

Keep open. Current `main` does not contain the PR's UTF-8 bootstrap behavior: Windows `gateway.cmd` generation still starts with only `@echo off`, and scheduled-task parsing still does not skip a `chcp 65001>nul` bootstrap line. The native Windows gateway path still relies on `gateway.cmd`, so the reported fix is neither implemented on `main` nor obsolete.

## Evidence

  - file: [src/daemon/schtasks.ts:248](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/daemon/schtasks.ts#L248)
