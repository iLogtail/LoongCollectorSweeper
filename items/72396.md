## Summary

Keep #72396 open. Current main supports a single writable bundled plugin runtime dependency stage directory, but it does not support layered read-only baseline roots plus a final writable install root. The issue is a fresh core runtime/container deployment request, not a stale report, duplicate, or ClawHub/plugin-owned item.

## Best Possible Solution

Keep the issue open. The best implementation is a core bundled plugin runtime-deps change that separates dependency search roots from the writable install root: parse either a PATH-style `OPENCLAW_PLUGIN_STAGE_DIR` or a new explicit read-only stage env var, check sentinels left-to-right across all roots, install and write retention metadata only in the final writable root, register aliases/node paths for the actual root containing each dependency, and document/test Docker and systemd examples.
