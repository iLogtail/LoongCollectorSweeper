## Summary

Keep #9987 open. Current main still has backend support for listing pending exec approvals, but there is no user-facing `openclaw approvals pending` CLI command, no CLI tests for that command, and the CLI docs still document only `get`, `set`, and `allowlist` flows. The related implementation PRs #10013 and #10341 were closed unmerged, so this issue remains the canonical tracker.

## Best Possible Solution

Keep #9987 open until OpenClaw adds a real `openclaw approvals pending` command in `src/cli/exec-approvals-cli.ts` that calls the existing `exec.approval.list` gateway method, renders useful command details, supports `--json`, and includes focused CLI tests and docs. Treat `--watch` as part of that change or split it into an explicit follow-up if maintainers want an incremental rollout.
