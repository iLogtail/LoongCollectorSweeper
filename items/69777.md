## Summary

Keep open. Current main does not show compaction creating a `ttsAuto: "always"` session override, but it still applies TTS conversion to internal webchat/dashboard replies whenever effective TTS auto state is enabled by session state, local prefs, or config. The reporter added that this happened twice and that a full reset fixes it, so this recent bug is not stale and cannot be safely closed as invalid or cannot-reproduce from source inspection alone.

## Best Possible Solution

Keep the issue open and triage the effective TTS state around the failure: inspect `sessions.json` for the affected session's `ttsAuto`, local `settings/tts.json`, and `messages.tts` config before and after compaction. If dashboard text chats should never auto-synthesize unless explicitly requested, add a regression around `chat.send`/`sessions.compact` and change the webchat/internal dispatch path or default TTS policy so text replies remain visible text unless the user selected `/tts chat on`, `messages.tts.auto: "always"`, or an explicit TTS directive/tool call.
