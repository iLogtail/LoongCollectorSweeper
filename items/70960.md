## Summary

Keep open. Current main only partially covers the Control UI agent-name path, does not show token/cost usage in the chat header/status area, and the QClaw Electron installer described in the report is not present in this repository, so there is not enough implemented-on-main evidence to close the issue.

## Best Possible Solution

Keep this issue open or split it into narrower follow-ups. For OpenClaw core, align every Control UI agent/session selector to prefer `agents.list[].name` before identity-derived names, and either restore a chat header token/cost usage pill from session usage data or explicitly document the intended replacement surface. For the QClaw Windows installer deletion report, route to the QClaw/Electron installer owner with the supplied install log and registry details.
