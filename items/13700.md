## Summary

Keep #13700 open. Current main has useful adjacent primitives, including compaction checkpoint list/branch/restore and `sessions_spawn` `context:"fork"`, but it still lacks the requested user-facing native session snapshot surface: no `/session save`, `/session load`, named snapshot API, reusable snapshot template, or restore path for `.reset`/`.deleted` archives. The latest maintainer comment in the issue also explicitly kept this open, and the relevant files have not changed since that review.

## Best Possible Solution

Keep #13700 open until OpenClaw implements a deliberate native session snapshot surface or splits it into canonical follow-ups. A strong implementation path is typed Gateway methods for named snapshot save/list/load/delete, slash/TUI controls, deterministic preservation of transcript/model/session metadata where feasible, reusable spawn-from-snapshot support, and a safe restore path for `.reset`/`.deleted` transcript archives.
