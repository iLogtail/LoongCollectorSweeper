## Summary

Keep open. Current local `main` at `c74fb781943f3bad7d6fec5256729c861d285cab` and read-only remote `main` at `8344fae38720f415bffa09aa6ba18b4d942188d5` still emit `gateway.trusted_proxies_missing` as `warn` for `gateway.bind="loopback"`. The latest release tag `v2026.4.24` also still has that behavior. PR #70368 tracks the requested fix, but it is open and unmerged, so the issue should remain open until that or an equivalent patch lands.

## Best Possible Solution

Keep #70357 open and review/land #70368 or an equivalent patch. The right code path is `collectGatewayConfigFindings`: emit `gateway.trusted_proxies_missing` as `info` when `gateway.bind` is `loopback`, keep or add `warn` behavior for `lan`/`tailnet`/`custom` with empty `trustedProxies`, and update the loopback audit regression test plus any user-facing audit severity docs as needed.
