## Summary

Keep open. Current main improves the default path because Tailscale cleanup is skipped unless `resetOnExit` is enabled, but the issue's core invariant is still not fully implemented: when `gateway.tailscale.resetOnExit` or `--tailscale-reset-on-exit` is used, gateway restart still runs a cleanup hook that calls global `tailscale serve reset`, which can remove non-OpenClaw Serve bindings.

## Best Possible Solution

Keep the issue open. The best fix is to remove the global `tailscale serve reset` from gateway-managed cleanup, or replace it with an owner-scoped Tailscale operation such as exact `off` arguments or get-config/set-config read-modify-write that removes only OpenClaw's binding. Add regression coverage for restart with another Serve binding preserved, and update the resetOnExit docs/help to warn or reflect the safer behavior.
