## Summary

Keep #55459 open. Current main has generic exec-provider knobs that can carry ordered `args` and explicit `env`, so the reporter's workaround is feasible, but main does not implement `OP_CACHE=false` auto-injection, a `globalArgs`/`prependArgs` field, or 1Password docs that disable the cache. Official 1Password docs confirm the cache defaults on and can be disabled with `OP_CACHE=false` or `--cache=false`. 

## Best Possible Solution

Keep the issue open and address it with a narrow maintainer-approved change. The lowest-risk path is to update the 1Password exec-provider docs to show `OP_CACHE: "false"` or `--cache=false` before `read`, plus required LaunchAgent/service-account env such as `OP_SERVICE_ACCOUNT_TOKEN`; a core auto-detect default for `/op` should be a separate product/security decision because it is service-specific behavior in the generic exec provider.
