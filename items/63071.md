## Summary

Close as implemented on main: current OpenClaw no longer injects a changing clock into the system prompt, places volatile prompt content below an explicit cache boundary, and Anthropic payload shaping only applies `cache_control` to the stable prefix. The same implementation is present in the latest release `v2026.4.23`.

## Evidence

- **System prompt no longer includes a per-turn timestamp:** `buildTimeSection()` now emits only `## Current Date & Time` plus `Time zone: ...`; it does not include `userTime` or formatted clock text, so the changing timestamp described in the issue is absent from the system prompt.
