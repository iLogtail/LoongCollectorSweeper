## Summary

Keep #64695 open. Current main still routes task-registry and task-flow persistence through `node:sqlite` without a runtime availability fallback, and the linked PR #64722 that proposed JSON fallback storage is closed unmerged. The issue is still actionable core behavior, not stale or superseded.

## Best Possible Solution

Keep the issue open and implement the fallback in OpenClaw core task persistence. A good fix would add a runtime `node:sqlite` availability check, route both task and task-flow registries to a durable fallback store or packaged sqlite dependency when unavailable, add regression tests that mock missing `node:sqlite` through the maintenance mark-lost path, and update task storage docs/changelog. PR #64722 is useful prior art but is closed unmerged, so it should be revisited or replaced rather than treated as canonical completion.
