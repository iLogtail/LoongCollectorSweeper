## Summary

Keep open. Current main still classifies adjacent `*.trajectory.jsonl` files as primary/usage-counted transcript files, and `openclaw doctor` still feeds that matcher directly into orphan transcript detection. The PR is recent, focused, and its follow-up commit addresses the valid review concern about real session IDs ending in `.trajectory`; there is no evidence that this fix is already shipped or superseded.

## Best Possible Solution

Keep PR #70812 open. The best path is to land a corrected implementation that excludes real trajectory sidecars from orphan and usage-counted transcript scans only when a matching primary transcript exists, preserves valid `.trajectory` session IDs, adds the PR's helper and doctor regression tests, and explicitly settles disk-budget sidecar retention semantics either in this PR or a narrow follow-up.
