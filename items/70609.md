## Summary

Keep open. Current main does not implement the PR's central behavior: outbound opaque/Hermes metadata is not accepted by gateway send, not passed through sendMessage/direct delivery, not exposed to channel outbound contexts, and not persisted through queue recovery. The PR is also not safe to merge as-is because the provided review context identifies real head-branch issues and unrelated work, but the remaining generic core/plugin API gap is meaningful enough that this should not be closed as implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep the PR open, but request a narrower revision: add a generic, backwards-compatible outbound metadata seam through gateway SendParams, generated client models, sendMessage, deliverOutboundPayloads, ChannelOutboundContext, queue storage/recovery, and focused tests. Split out the unrelated auth commit, fix the PR-head review defects, and move Hermes-specific topic/bot defaults into configuration or a plugin/ClawHub integration unless maintainers explicitly want those constants in core.
