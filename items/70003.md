## Summary

Keep open. Current main has useful mitigations for parts of the workflow, including `/btw`, `/stop`/`chat.abort`, `sessions.steer`, queue modes, non-blocking `sessions_spawn`, and `sessions_yield`, but it still does not implement ordinary concurrent same-session user turns during an active long-running tool execution. Same-session runs remain serialized, and steering only applies at streaming/model boundaries or falls back to queued follow-up behavior.

## Best Possible Solution

Keep the issue open as a core gateway/agent-runtime design item. The best path is to specify the desired behavior for normal inbound messages while a same-session run is inside a long-running tool, then implement either cooperative preemptive interrupt, a safe side-turn channel, or async background-result delivery with transcript ordering guarantees. Add focused coverage for foreground `exec`, `sessions_spawn`/subagent waits, queue modes, `chat.abort`/`sessions.steer`, and docs that distinguish `/btw`/`/stop` mitigations from true concurrent turns.
