## Summary

Keep PR #61374 open. Current main still lacks the two core behaviors the PR targets: Slack block-action system events are enqueued passively without an immediate wake, and threadTs is not passed into Slack system-event session resolution. The related broader tracker #61502 and overlapping PR #67751 mean maintainer coordination is needed, but they do not make this PR obsolete enough for conservative auto-close.

## Best Possible Solution

Keep #61374 open for maintainer review or rework. The best path is to resolve #61502 by landing a thread-aware Slack interaction wake/dispatch fix, with focused regression coverage for Slack reply buttons and generic block actions, while reconciling this PR with the overlapping #67751 approach rather than auto-closing useful code.
