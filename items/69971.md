## Summary

Keep open. Current main still does not implement either central behavior from PR #69971: plugin install still reaches installer paths without a profile-aware `extensionsDir`, and discovery still skips top-level symlinked plugin directories via `Dirent.isDirectory()`. The linked issue #69960 was closed as tracked by this PR, and closed PR #45744 is symlink-only/unmerged, so #69971 remains the canonical fix vehicle rather than cleanup-closeable.

## Best Possible Solution

Keep #69971 open for maintainer review. The best path is to land a focused fix that computes the managed plugin install root after profile env setup while matching `resolvePluginSourceRoots(...).global`/`resolveConfigDir` discovery semantics, then keep/add regression coverage for profile-aware installs, default installs, config/state-dir divergence, symlinked plugin directories, and broken symlink diagnostics. The unresolved bot review should be addressed rather than closing the PR as stale, implemented, or superseded.
