## Summary

Keep #69971 open. Current main still lacks both requested behaviors: `plugins install` does not pass a profile-aware `extensionsDir` into direct npm/path install calls, and plugin discovery still skips symlinked directory entries. The PR remains the canonical useful fix candidate, but it needs revision for the unresolved install/discovery root-alignment review concern before merge.

## Best Possible Solution

Keep #69971 open and ask for a revision that makes plugin install and discovery share one managed extensions-root contract after profile/env resolution. The fix should pass that root through npm/path/link install paths, consider the onboarding/channel setup install path called out in the PR body, and keep regression coverage for `--profile`, `OPENCLAW_CONFIG_PATH` without `OPENCLAW_STATE_DIR`, legacy-state-dir fallback, and symlink plus broken-symlink discovery.
