## Summary

Keep open. Current `main` does not show this fallback-winner persistence implemented, and the open PR appears stale/risky rather than obsolete: `src/agents/agent-command.ts` only mutates `sessionEntry.authProfileOverride` in memory during failover, `src/agents/command/session-store.ts` persists runtime model/token data but not auth-profile winner data, and `src/agents/pi-embedded-runner/types.ts` still has no `finalAuthProfileId` return field. The provided PR discussion also raises unresolved correctness concerns about dropping the `allowModelOverride` ingress guard and omitting a required test option.
