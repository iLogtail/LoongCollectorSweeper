## Summary

Keep #71899 open. Current main still has the reported workspace-default behavior: setup stores the main workspace in `agents.defaults.workspace`, and non-default agents with no explicit workspace are resolved as `<agents.defaults.workspace>/<agentId>`. The `agents add` wizard uses that resolver for the visible initial value, while current docs describe peer-level `workspace-<agentId>` defaults. The PR is a focused fix for #71889 and the provided discussion shows review feedback was actively addressed, so it is not obsolete or superseded.

## Best Possible Solution

Keep the PR open and proceed with maintainer review rather than conservative cleanup. The best path is to land a resolver fix that treats setup-managed main workspace defaults as the main workspace, falls back to peer-level `workspace-<agentId>` for added agents, preserves explicit per-agent workspaces and intentional shared bases, and includes regression coverage for the resolver plus the `agents add` prompt default.
