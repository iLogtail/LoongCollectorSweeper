## Summary

Keep #71899 open. Current main still resolves a non-default agent without an explicit workspace by appending the agent id under any `agents.defaults.workspace`, while setup stores the main workspace in that default and the `agents add` wizard uses that resolver for the prompt initial value. The PR is a live, focused fix for #71889 rather than obsolete or closeable cleanup.

## Best Possible Solution

Keep the PR open and continue review. The best path is to land a focused resolver fix that treats setup-managed main workspaces as main-agent workspaces, preserves intentional shared workspace bases and explicit per-agent workspaces, and includes regression coverage for the wizard default, profile-transition case, symlink/canonicalization behavior, and custom shared bases. Once merged, #71889 can close as fixed.
