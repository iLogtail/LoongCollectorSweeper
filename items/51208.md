## Summary

Keep open. Current main still follows the reported failure path for hook agent runs: the gateway creates hook jobs as isolated runs, the cron runner turns isolated into forceNew sessions, and the CLI executor suppresses stored CLI session reuse whenever the resolved cron session is new. The linked PR #51337 is an exact proposed fix, but it is not merged in the provided context; later related work only cleaned up stale CLI identity for isolated rollovers and does not solve stable hook session continuation.
