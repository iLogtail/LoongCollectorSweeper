## Summary

Keep open. Current `main` does not implement shared Feishu webhook servers across accounts on the same `host:port`, so this PR's core fix is not already present and the request is still live.

## Evidence

- **Current webhook transport still creates one HTTP server per account:** `monitorWebhook()` allocates `const server = http.createServer()` for each account, stores it under that account id, and calls `server.listen(port, host, ...)` directly. There is no shared pool keyed by `host:port` in current `main`.
