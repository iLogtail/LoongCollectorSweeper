## Summary

Keep PR #41637 open. Current main still starts one Feishu webhook HTTP server per account, records it by account id, and calls `server.listen(port, host)` for every webhook account. Multi-account startup remains concurrent in the gateway path, and the Feishu docs still expose a shared default webhook port of 3000. The PR is a focused fix attempt for a real remaining Feishu multi-account webhook collision, though it needs rebasing and maintainer review for the unresolved pooled-lifecycle and security-regression concerns noted in review comments.

## Best Possible Solution

Keep the PR open as a real Feishu plugin bug-fix attempt, but require a refreshed implementation that shares one webhook listener per host:port, preserves current signed-request/body/rate-limit guards, handles creator/joiner abort and listen/error propagation correctly, disambiguates routing by token/app/path, and adds regression tests for two webhook accounts on the same host:port plus cleanup/error paths.
