## Summary

Keep #39031 open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent: the imsg payload, parser, inbound decision, monitor subscription, config schema, docs, and tests all cover normal messages/attachments but not reaction events. BlueBubbles does have reaction system-event support and is the recommended iMessage path, and the related #60446 duplicate was closed back to this issue, but that does not implement the requested legacy imsg behavior. Open PR #39322 remains the relevant implementation candidate.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. Maintainers should either review/supersede PR #39322 with a typed reaction/system-event implementation, config and docs, parser/subscription changes, access-control behavior, debounce semantics, and tests, or make an explicit product decision that reaction event support is BlueBubbles-only and document that legacy imsg parity is intentionally out of scope.
