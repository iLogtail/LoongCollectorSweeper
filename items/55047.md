## Summary

Keep open. Current `main` still has a Control UI render path that unconditionally dereferences `job.payload.kind`, while cron-store normalization can preserve malformed persisted jobs without a `payload`. That matches the reported "blank page with only the title" regression more than it proves a fix.

## Evidence

- **Control UI render still assumes every cron job has a payload:** `ui/src/ui/app-render.ts` builds `cronModelSuggestions` on every render and does `if (job.payload.kind !== "agentTurn" || typeof job.payload.model !== "string")`, which will throw if a persisted cron row reaches the UI without `payload`.
  - file: [ui/src/ui/app-render.ts:795](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/ui/src/ui/app-render.ts#L795)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '760,820p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Cron normalization does not guarantee payload exists:** `normalizeCronJobInput` only infers `payload` when legacy top-level `message`/`text`/`command` fields are present; otherwise it returns the job without synthesizing a payload.
  - file: [src/cron/normalize.ts:480](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/cron/normalize.ts#L480)
  - command: `nl -ba src/cron/normalize.ts | sed -n '405,620p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Loaded cron jobs are kept even when only partially normalized:** `ensureLoaded` replaces each stored job with `normalizeCronJobInput(raw)` when possible, but does not reject/drop jobs that remain malformed apart from `enabled`; those jobs are kept in memory and can still reach `cron.list`.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/cron/service/store.ts#L37)
  - command: `nl -ba src/cron/service/store.ts | sed -n '36,70p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Doctor repair path is advisory, not a proof the UI bug is fixed:** `maybeRepairLegacyCronStore` tells users to run `openclaw doctor --fix` for legacy cron storage, but that is a manual repair flow and not evidence that current Control UI rendering is hardened against malformed rows by itself.
  - file: [src/commands/doctor-cron.ts:163](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/commands/doctor-cron.ts#L163)
