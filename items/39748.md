## Summary

Keep open. Current main still leaves the Nostr relay config schema as an unconstrained string array, setup entry points only reject non-ws/wss schemes, and runtime forwards configured relays directly into the Nostr bus. The PR addresses a real remaining gap and its latest head fixed the earlier review concern by restricting plaintext ws:// to localhost/loopback, but it needs a refresh against current main and should share that validator across setup surfaces too.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best fix is one shared Nostr relay URL validator that allows wss:// for remote relays and ws:// only for localhost/loopback, then uses it from NostrConfigSchema, CLI setup, Control UI setup, and focused regression tests in the current Nostr test files.
