## Summary

Keep open. Current main still leaves the Nostr config schema as an unconstrained string array for relay URLs, setup paths only check that a URL is `ws:` or `wss:` without restricting plaintext `ws://` to local hosts, and runtime forwards configured relay strings directly into the Nostr bus. The PR addresses a live gap, and the discussion already improved the branch by adding the localhost/loopback restriction after review feedback.

## Best Possible Solution

Keep this PR or an equivalent maintainer patch open until Nostr relay validation is implemented across the config schema, CLI setup adapter, and control UI setup surface. The best code direction is a shared plugin-local relay URL validator that allows remote `wss://`, allows `ws://` only for localhost/loopback development hosts, rejects other protocols, and has colocated regression tests for accepted secure/local URLs and rejected `http://`, `https://`, `javascript:`, malformed, and remote `ws://` URLs.
