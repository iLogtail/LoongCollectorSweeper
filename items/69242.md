## Summary

Keep #69242 open. Current main has an adjacent Linux OOM-victim-selection change, shipped by the merged #70419 path, but that PR explicitly scoped #69242 as remaining SIGKILL observability work. The exec path still classifies an unexplained signal-killed child as a generic signal failure, and the foreground exec tool result does not expose exitSignal/failureKind or enough cause metadata to distinguish OS kill, timeout, no-output timeout, manual cancellation, or supervisor-initiated kill. No current docs/tests found broad find/grep guardrails or a user-facing diagnostic fix for this Linux report.

## Best Possible Solution

Keep the issue open and implement shared exec/process diagnostics for signal-killed commands. The best fix likely belongs in the process supervisor plus exec tool result surface: preserve stdout/stderr tail, expose exitSignal/failureKind/termination reason, distinguish OpenClaw-initiated timeout/no-output/manual-cancel kills from raw OS signal exits when knowable, include duration and a run/log correlation id, and add focused tests in the exec runtime/supervisor suites. Optional broad-scan guardrails for known-large OpenClaw state/log directories can be a follow-up, but the core remaining bug is insufficient SIGKILL cause reporting.
