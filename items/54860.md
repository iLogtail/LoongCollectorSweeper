## Summary

Current main already namespaces configured bundle MCP tools as `serverName__toolName`, so cross-server MCP tool name collisions are no longer flattened into one global name.

## Evidence

- **Namespaced MCP tool materialization:** `materializeBundleMcpToolsForRun` builds each exposed MCP tool name from the sanitized server name plus the original tool name (`serverName__toolName`) and registers that name instead of the flat MCP name.
  - file: [src/agents/pi-bundle-mcp-materialize.ts:90](https://github.com/openclaw/openclaw/blob/a52b50ef3c8bb05d4a18302aad6862c37278cc57/src/agents/pi-bundle-mcp-materialize.ts#L90)
  - sha: [a52b50ef3c8b](https://github.com/openclaw/openclaw/commit/a52b50ef3c8bb05d4a18302aad6862c37278cc57)
- **Sanitized server prefixes are captured per server:** The session MCP runtime records a `safeServerName` for each configured MCP server before materialization, so multiple servers can contribute same-named tools without one being dropped.
  - file: [src/agents/pi-bundle-mcp-runtime.ts:169](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/pi-bundle-mcp-runtime.ts#L169)
