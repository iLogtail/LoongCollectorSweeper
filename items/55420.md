## Summary

Keep open. Current main still implements `tools.sessions.visibility` as the coarse `self | tree | agent | all` enum, with no `allowlist` value and no `tools.sessions.allowlist` config. The existing `session.sendPolicy` prefix rules are delivery controls, not visibility controls for `sessions_list` or `sessions_history`. Related #48785 is adjacent broader authorization work, but it does not supersede this specific key-pattern allowlist request.

## Best Possible Solution

Keep the issue open and implement this as core session-tool authorization, not ClawHub. The likely fix is to extend the config contract with `tools.sessions.visibility: "allowlist"` plus `tools.sessions.allowlist`, add a deterministic glob/prefix matcher over canonical session keys in `src/plugin-sdk/session-visibility.ts`, thread that through `sessions_list`, `sessions_history`, `sessions_send`, and likely `session_status`, and update config schema/help/docs/tests while keeping `session.sendPolicy` as a separate outbound-delivery policy.
