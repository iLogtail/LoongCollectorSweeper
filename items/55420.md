## Summary

Keep open. Current `main` does not implement `tools.sessions.visibility=allowlist` or a companion `tools.sessions.allowlist`; the session-tools config, docs, and enforcement code still only support `self`, `tree`, `agent`, and `all`. The existing docs also explicitly note that `agent` visibility can include other users when per-sender sessions share one agent id, which is the gap this feature request is trying to address.

## Evidence

- **Runtime config schema only accepts four visibility values:** `ToolsSchema.sessions.visibility` is `z.enum(["self", "tree", "agent", "all"])`; there is no `allowlist` enum value or sibling allowlist field.
  - file: [src/config/zod-schema.agent-runtime.ts:875](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/zod-schema.agent-runtime.ts#L875)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Core session visibility type/resolver has no allowlist mode:** `SessionToolsVisibility` is defined as `"self" | "tree" | "agent" | "all"`, and `resolveSessionToolsVisibility()` only accepts those four strings before defaulting unknown values back to `tree`.
  - file: [src/agents/tools/sessions-access.ts:13](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/tools/sessions-access.ts#L13)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Enforcement logic has no pattern/allowlist branch:** `createSessionVisibilityGuard()` only special-cases cross-agent access (`visibility` must be `all`) plus same-agent `self` and `tree` restrictions. There is no code path that matches target session keys against configured patterns.
  - file: [src/agents/tools/sessions-access.ts:206](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/tools/sessions-access.ts#L206)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **User-facing docs still describe only self/tree/agent/all:** The config reference lists exactly four `tools.sessions.visibility` values and says `agent` can include other users if per-sender sessions run under the same agent id, matching the issue's stated gap.
  - file: [docs/gateway/config-tools.md:263](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/docs/gateway/config-tools.md#L263)
