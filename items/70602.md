## Summary

Keep PR #70602 open. Current main still lacks the proposed heartbeat keep-full-tools behavior, and the PR remains useful but appears incomplete because core tools are filtered before the PR’s bundled-tool policy hook would run.

## Best Possible Solution

Keep the PR open for maintainer review, but request revisions before merge: apply heartbeat keep-full-tools semantics at the core `createOpenClawCodingTools` boundary as well as bundled tools, use one owner-only predicate for filtering and guard wrapping, and add focused tests proving heartbeat tool-array parity plus denied execution for `cron`, `gateway`, and name-only `nodes`.
