## Summary

Keep open. Current main still does not implement the PR's central behavior: no `/emotions` directive, no persisted `emotionMode`, no emotion-tag sanitizer, no `ttsSourceText`/`ttsPlainText` metadata contract, and no provider capability routing for expressive TTS tags. The PR is an active stacked implementation tied to #69051, #72239, and #72266 with fresh review discussion, so it is not a conservative cleanup-close candidate.

## Best Possible Solution

Keep #72229 open as the active PR-B implementation path. Review it in stack order after #69051/PR-A is settled, require the latest TTS fallback and empty-plain-text edge cases to be addressed, and keep targeted tests for directive parsing, session persistence, metadata writing, provider routing, and sanitizer boundaries before merge.
