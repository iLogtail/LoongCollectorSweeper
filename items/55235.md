## Summary

Keep open. Current main exposes `openclaw config schema`, but it still only prints the schema to stdout and documents manual piping; it does not maintain a colocated `openclaw_schema.json`, set `$schema` in `openclaw.json`, or expose the requested `update.maintain_config_json_schema` setting. The linked PR #55390 appears to track the intended implementation, but it is still open/unmerged, so the issue remains the canonical feature request.

## Best Possible Solution

Keep #55235 open until #55390 or a replacement lands. The best implementation path is a minimal core CLI/config update that writes `openclaw_schema.json` beside the active config, adds `$schema` only when absent or empty, preserves custom `$schema` values, adds the documented opt-out setting under `update`, and covers the startup/update behavior with focused CLI/config tests and config docs.
