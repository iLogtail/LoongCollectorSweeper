## Summary

Current `main` does not implement this feature request. OpenClaw can print the config schema on demand, but it does not auto-generate/update a sibling schema file during bootstrap/update, it does not define `update.maintain_config_json_schema`, and the docs still describe a manual redirection flow.

## Evidence

- **Manual-only schema command:** `config schema` is wired as an explicit CLI subcommand, and its implementation just serializes the generated schema to stdout via `writeRuntimeJson(...)`; there is no file-write path here.
  - file: [src/cli/config-cli.ts:1431](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/cli/config-cli.ts#L1431)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Command registration is explicit:** The only exposed path is `openclaw config schema`; the command table registers `schema` under `config` and invokes `runConfigSchema({})` directly.
  - file: [src/cli/config-cli.ts:1636](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/cli/config-cli.ts#L1636)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)

  - file: [docs/cli/config.md:74](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/docs/cli/config.md#L74)
