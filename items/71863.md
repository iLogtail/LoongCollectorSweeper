## Summary

Keep #71863 open. Current main still has the Signal daemon shutdown race described by #22676: `stop()` is synchronous, returns `void`, only sends SIGTERM, and the monitor does not await daemon shutdown during abort/finally cleanup. The PR is a focused implementation for the linked open bug and its supplied context shows the bot review follow-ups were addressed in the PR head.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to validate the PR's async Signal daemon stop semantics and regression tests, then merge or adapt that implementation to close #22676 rather than auto-closing the only focused fix.
