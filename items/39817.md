## Summary

Keep open. Current `main` does not already implement this PR’s fallback, and the current gateway/UI flow still has a plausible silent-failure path for `chat.send` runs that start an agent run and terminate with lifecycle error / error-only output.

## Evidence

- **Agent-run branch still has no terminal fallback:** In current `main`, `chat.send` sets `agentRunStarted = true` in `onAgentRunStart`, but the success path only does `emitUserTranscriptUpdate()` in the agent-run branch; it does not call `broadcastChatError` or `broadcastChatFinal` there. The explicit final/error broadcast logic remains only in the non-agent-run branch and the rejected-promise `.catch()` path.
  - file: [src/gateway/server-methods/chat.ts:2498](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/gateway/server-methods/chat.ts#L2498)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '2498,2698p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **Lifecycle chat errors are intentionally suppressed for active chat.send runs:** The gateway agent-event handler computes `skipChatErrorFinal = isChatSendRunActive(evt.runId) && !chatLink` for lifecycle errors. That means an active `chat.send` run without a chat-link remap will not emit the normal terminal `chat state:"error"` event from lifecycle processing.
  - file: [src/gateway/server-chat.ts:988](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/gateway/server-chat.ts#L988)

- **Web UI still depends on a terminal chat event to clear the active run:** The web UI defers `session.message` history reloads while `host.chatRunId` is active, and only clears that active run on `chat` terminal states (`final`, `aborted`, or `error`). If no terminal chat event arrives, transcript updates alone do not resolve the active-run state.
  - file: [ui/src/ui/app-gateway.ts:495](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/ui/src/ui/app-gateway.ts#L495)
