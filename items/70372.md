## Summary

Keep #70372 open. Current main suppresses commentary only when a message or textSignature already carries phase metadata, but it does not track Anthropic/Bedrock thinking events or derive a phase for later phaseless text. The related #70344 is broader and still open, not a clear canonical replacement for this narrower provider/thinking path, and #70372 also has an unresolved review concern about provider scoping.

## Best Possible Solution

Keep the PR open or explicitly fold it into a canonical broader fix. The best path is a provider-safe implementation that handles Anthropic/Bedrock phaseless thinking turns, preserves the OpenAI phase-pending guard, and adds a `subscribeEmbeddedPiSession` regression test for thinking events followed by unphased text so only user-visible final content reaches channel delivery.
