## Summary

Keep #70372 open. Current main has not implemented the proposed Anthropic/Bedrock thinking-state fallback: unphased text still bypasses phase-aware delivery, while Anthropic streaming emits thinking and text events without phase metadata. The PR is still plausible work, but it needs maintainer/code review rather than cleanup closure because the overbroad fallback could interfere with current OpenAI Responses buffering, where text phase is not known until text_end.

## Best Possible Solution

Keep the PR open, but request changes: add a targeted regression for unphased Anthropic/Bedrock thinking plus text delivery, then implement a scoped fix either by having those transports mark final visible text with proper phase/textSignature metadata or by deriving phase only for identified unphased Anthropic/Bedrock streams. Preserve the existing OpenAI Responses pending-phase guard.
