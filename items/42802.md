## Summary

Current main supports route/ACP bindings in config and CLI/docs, but the specific requested Gateway UI/API work is still missing: there is no `routing.list` Gateway method, no routing schema/handler, and the Agents Channels panel still renders only a gateway-wide channel status snapshot rather than per-agent binding rows. The external fork commit mentioned in the issue is not present in this checkout.

## Best Possible Solution

Keep this open and implement the feature in core: add a typed read-only `routing.list` Gateway method under `READ_SCOPE`, backed by existing configured binding helpers; add protocol schema/docs/tests; add Control UI state and `loadRouteBindings()`; then filter the Agents tab Channels panel by the selected `agentId` and render route/ACP binding rows plus the requested empty state.
