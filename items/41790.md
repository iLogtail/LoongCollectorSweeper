## Summary

Current `main` does not implement this feature. Runtime config loading still forces `session.mainKey` back to `"main"`, and route building still hardcodes `DEFAULT_MAIN_KEY`, so the requested behavior remains actionable.

## Evidence

- **Runtime defaults still discard custom mainKey:** `applySessionDefaults()` overwrites any configured `session.mainKey` with `"main"` and emits a warning that the field is ignored.
  - file: [src/config/defaults.ts:107](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/config/defaults.ts#L107)
  - command: `nl -ba src/config/defaults.ts | sed -n '107,132p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Loaded runtime config always applies that override:** `materializeRuntimeConfig()` unconditionally calls `applySessionDefaults()`, and `loadConfig()` returns the materialized runtime snapshot used across Gateway/heartbeat/cron codepaths.
  - file: [src/config/materialize.ts:53](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/config/materialize.ts#L53)
  - command: `nl -ba src/config/materialize.ts | sed -n '50,70p'; nl -ba src/config/io.ts | sed -n '1964,1976p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Routing still hardcodes the legacy main key:** `buildAgentSessionKey()` passes `DEFAULT_MAIN_KEY`, and `resolve-route` also builds `mainSessionKey` with `DEFAULT_MAIN_KEY`, so channel routing does not honor configured `session.mainKey`.
  - file: [src/routing/resolve-route.ts:93](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/routing/resolve-route.ts#L93)

  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Main docs still describe session.mainKey as ignored:** The current config docs explicitly call `session.mainKey` a legacy field and say runtime always uses `"main"` for the main direct-chat bucket.
  - file: [docs/gateway/config-agents.md:1136](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/gateway/config-agents.md#L1136)

## Risks / Open Questions

- There is implementation drift in this area: some helpers and docs still describe `session.mainKey` as meaningful (for example `src/config/sessions/main-session.ts` and `docs/gateway/tools-invoke-http-api.md`), but the loaded runtime config still disables it.
