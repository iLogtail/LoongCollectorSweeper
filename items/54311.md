## Summary

Keep #54311 open. Current main still reproduces the reported `gh-issues` validation failure against the repo's skill-creator validator, and the bundled `skills/gh-issues/SKILL.md` is still far above the reported size target. One part has shifted: OpenClaw runtime docs and code now treat `user-invocable` as a supported skill frontmatter key, so the best fix may be to align the validator/spec guidance rather than blindly remove the field.

## Best Possible Solution

Keep the issue open and resolve it with a focused skills-quality cleanup: align `skills/skill-creator/scripts/quick_validate.py`, `skills/skill-creator/SKILL.md`, and `docs/tools/skills.md` on the supported frontmatter contract; remove redundant `user-invocable: true` from `skills/gh-issues/SKILL.md` only if maintainers want bundled skills to avoid default-valued keys; then split the oversized gh-issues orchestration details into referenced files while keeping the main `SKILL.md` concise. The description concern appears mostly addressed on current main.
