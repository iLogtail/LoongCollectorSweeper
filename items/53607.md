## Summary

Keep open. Current `main` still does not mirror Discord component labels into the session transcript, so the requested behavior is not implemented. The PR is directionally useful and the thread already resolved earlier duplicate-write/design concerns, but the latest review still identifies an unresolved regression in generic captioned-media transcript mirroring.
