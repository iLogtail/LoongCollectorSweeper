## Summary

Do not close #53607. The PR targets a real remaining gap: current main still sends Discord Components v2 and registers component entries, but the delivered component labels/options are not mirrored into the target session transcript. The linked bug #21649 is closed, but I did not find an equivalent implementation on main, and this PR still contains useful work that needs maintainer review/rework rather than cleanup closure.

## Best Possible Solution

Keep the PR open and rework it against current main. The best fix is a narrow Discord-owned component transcript summary that flows through an explicit enriched-mirror signal, is bounded/sanitized before persistence, preserves existing media filename behavior for non-enriched sends, and includes tests for button/select/modal labels, media attachments, mixed results, and no double transcript writes.
