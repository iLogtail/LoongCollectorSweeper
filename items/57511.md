## Summary

Keep PR #57511 open. Current main does not implement Teams live voice, and the PR is a recent draft with concrete but unresolved scope, API, compliance, and verification questions. The maintainer review explicitly describes the direction as valuable and asks for a narrower first PR rather than treating it as obsolete or out of scope.

## Best Possible Solution

Keep the draft PR open, but treat it as design/implementation work that must be split. The best path is a narrow first PR for documented Teams voice capability tiers, plugin config schema, and no-op capability negotiation without the .NET worker or live audio path. Follow-up PRs should add the worker bridge and then live audio only after dotnet build, mock gRPC integration, real Teams/Azure VM E2E, RSC permission validation, and compliance-gate proof are available. Any public SDK seam should be justified as a generic, documented plugin contract or kept plugin-local.
