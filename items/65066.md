## Summary

Keep #65066 open. Current target main still does not register a memory runtime/capability from `memory-lancedb`, while doctor and status still resolve memory health through the active `MemoryPluginRuntime`. The PR is a focused fix for linked open bug #60177 and its review comments were addressed in the PR discussion, so it remains useful rather than obsolete or closeable.

## Best Possible Solution

Keep the PR open, refresh it against current main if needed, and land a scoped implementation that registers a `memory-lancedb` memory capability/runtime plus regression coverage. After that lands and changed-lane validation passes, use it to close linked issue #60177; leave broader doctor/status false-positive reports to their existing narrower trackers.
