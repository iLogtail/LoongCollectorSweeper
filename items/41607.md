## Summary

Keep open. Current main has Feishu card-action callback plumbing and some read-path extraction for card markdown/div text, but it does not satisfy the issue's requested full Interactive Card parsing for inbound message events, button labels/values, dropdown options, or form inputs.

## Evidence

- **Inbound message parser lacks interactive branch:** Feishu inbound `im.message.receive_v1` events flow through `parseMessageContent`; it handles `post`, `text`, `share_chat`, and `merge_forward`, then returns the raw content for other message types. There is no `interactive` card parser on the direct inbound event path.
  - file: [extensions/feishu/src/bot-content.ts:132](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/feishu/src/bot-content.ts#L132)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '132,164p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Card action callback exists but is narrow:** Current main registers `card.action.trigger` and routes valid payloads to `handleFeishuCardAction`, so part of the issue's suggested callback path exists.
  - file: [extensions/feishu/src/monitor.account.ts:372](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/feishu/src/monitor.account.ts#L372)
  - command: `nl -ba extensions/feishu/src/monitor.account.ts | sed -n '360,420p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Card action fallback does not extract labels/options/forms:** The legacy card-action fallback only returns `action.value.text`, `action.value.command`, JSON-stringifies the action value, or stringifies the primitive. It does not parse button display labels, select options, or form input fields requested by the issue.
  - file: [extensions/feishu/src/card-interaction.ts:81](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/feishu/src/card-interaction.ts#L81)
  - command: `nl -ba extensions/feishu/src/card-interaction.ts | sed -n '81,92p'`

- **Existing card content extraction is partial and used on message fetch/list path:** `parseInteractiveCardContent` extracts only markdown and div text from card `elements` or `body.elements`. It omits button labels/values, dropdown options, and form inputs, and it lives in the message fetch/list helper rather than the direct inbound parser.
  - file: [extensions/feishu/src/send.ts:182](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/feishu/src/send.ts#L182)
