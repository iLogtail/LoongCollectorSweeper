## Summary

Keep open. Current `main` still lacks a live inbound Feishu interactive-card parsing path, so the reported gap is not clearly implemented or obsolete.

## Evidence

- **Inbound live message parser does not handle `interactive`:** `parseFeishuMessageEvent()` feeds live event payloads through `parseMessageContent()`. That parser explicitly handles `post`, `text`, `share_chat`, and `merge_forward`, then returns the raw content for everything else; there is no `interactive` branch here.
