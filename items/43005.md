## Summary

Keep open. Current main still hard-codes the cross-origin redirect safe-header set, exposes no `additionalSafeHeaders` or equivalent option on `GuardedFetchOptions`, and the existing tests explicitly assert that `x-api-key` is stripped on cross-origin redirects. The related #57588 item is open but covers a different fetch-auth token fallback concern, so it does not supersede this request.
