## Summary

Keep PR #41670 open. Current main still omits `encoding_format` from the active remote memory embedding request path, and OpenAI-compatible memory providers route through that helper. The PR likely needs a rebase and possibly coverage for batch embedding payloads, but the underlying compatibility issue is not implemented on main.

## Best Possible Solution

Keep this open and have maintainers either rebase the minimal `encoding_format: "float"` fix onto the current memory-host-sdk remote provider path with regression tests, or consolidate it into the broader configurable encoding-format work in #43910 while ensuring OpenAI-compatible inline and batch embedding requests are covered.
