## Summary

Keep PR #60456 open. Current main does not contain the four proposed skill-scanner rule IDs or equivalent tests, and the built-in scanner is a real core security path used by plugin/skill install scans and deep security audit. This is not a conservative ClawHub close: the request hardens an existing OpenClaw code-safety gate rather than adding an optional external skill/plugin. The PR still needs maintainer/code-review follow-up because the supplied review comments identify concrete false-positive and false-negative issues in the proposed regex rules.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise the PR in core scanner code: tighten network-listener and credential-access matching to real imports/call arguments, make prototype-pollution detection cover mutation forms including dot notation without flagging benign reads as critical, preserve symlink coverage with fs-context tests, and keep focused regression tests in src/security/skill-scanner.test.ts. If maintainers choose a larger scanner-engine direction, explicitly supersede this PR with that canonical open PR instead of auto-closing it now.
