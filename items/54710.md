## Summary

Kept open. Current `main` shows the exec path is channel-agnostic and forwards `params.command` unchanged into the shell/runtime, while Discord-specific fields are only threaded into exec-approval routing/UI. That weakens the report’s claimed Discord-only root cause, but this read-only review did not include a live Discord-bound reproduction, so there is not strong enough evidence to close as fixed or non-reproducible.

## Evidence

- **Exec tool forwards the command unchanged:** `createExecTool()` passes `params.command` directly to node execution, gateway allowlist handling, and then `runExecProcess()`. The only Discord/channel fields carried here are `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, and `turnSourceThreadId` for approval routing.
