## Summary

Keep PR #63789 open. Current main has adjacent `sessions_send` agent-to-agent behavior, but it does not implement the proposed Phase 1 handoff contract (`handoff_id`, structured delivered/accepted/queued/rejected acknowledgement, control-only visibility contract, or append-only handoff ledger). The recent maintainer comment explicitly says the proposal is still useful as the lower-level primitive and should stay separate from #65403.

## Best Possible Solution

Keep #63789 open as the narrow transport/control primitive. The next useful maintainer path is a focused implementation PR that adds a stable handoff id, structured acknowledgement states, control-only outcome visibility, an append-only audit trail, and regression tests for `NO_REPLY`/`REPLY_SKIP`/`ANNOUNCE_SKIP`, while leaving meeting lifecycle work to #65403.
