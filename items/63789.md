## Summary

Keep #63789 open. Current main has partial `sessions_send` agent-to-agent behavior, but it does not implement the proposed Phase 1 contract (`handoff_id`, structured delivered/accepted/queued/rejected acknowledgement, control-only outcome visibility, append-only handoff ledger). A maintainer comment also explicitly says this proposal is still useful as the lower-level primitive and should stay separate from the lifecycle work in #65403.

## Best Possible Solution

Keep the draft open as the focused transport/control-plane proposal. The eventual core change should add a first-class `sessions_send`/handoff contract with a stable handoff id, structured ack states, visibility for `NO_REPLY`/`REPLY_SKIP`/`ANNOUNCE_SKIP` outcomes, and an append-only audit event stream, with regression coverage in the existing sessions_send/A2A tests. Leave meeting lifecycle, pickup queues, transcripts, and teleconference UX to #65403.
