## Summary

Keep open. Current main still does not provide an end-to-end DM-level toolsBySender policy for the requested Signal/Telegram-style allowlist use case: Signal only exposes toolsBySender under group overrides, and the shared tool-policy resolver still derives sender-scoped policy from group/channel session contexts rather than direct-message sessions.

## Best Possible Solution

Keep the issue open and implement a shared direct-message tool-policy contract: add typed DM tools/toolsBySender config where missing, pass direct-session sender identity into the effective tool-policy resolver, wire Signal and Telegram through the same enforcement path, and add focused tests/docs proving allowlisted non-owner DM senders can be denied tool groups while the owner keeps normal access.
