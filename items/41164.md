## Summary

Keep open. Current `main` still defaults new cron delivery to the implicit/explicit `"last"` route, and the config surface only exposes `cron.failureDestination` for failure alerts, not a global default delivery channel/target for new jobs.

## Evidence

- **CLI still hardcodes `last` as the default cron delivery channel:** `openclaw cron add` defines `--channel <channel>` with default value `"last"`, so omitted `--channel` still falls back to the last active conversation.
  - file: [src/cli/cron-cli/register.cron-add.ts:103](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/cron-cli/register.cron-add.ts#L103)
