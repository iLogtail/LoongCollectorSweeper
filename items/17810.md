## Summary

Keep #17810 open. Current main still injects SOUL.md, IDENTITY.md, USER.md, and MEMORY.md as ordered Project Context files with only a fixed SOUL hint; it does not implement configurable context-file elevation, custom section labels, or exclusion from the flat Project Context block. The matching PR #18110 is closed unmerged in the provided GitHub context, and the latest maintainer/Codex discussion explicitly kept this issue open.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted, implement this in the agent prompt/bootstrap path with documented config/schema support, deterministic matching and ordering, elevated sections before Project Context, exclusion from the flat context unless explicitly retained, and focused tests for prompt rendering plus config validation/docs. If maintainers prefer externalization, first add or document a public plugin/API surface that can safely restructure bootstrap prompt sections without replacing the whole system prompt.
