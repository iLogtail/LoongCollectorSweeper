## Summary

Keep #70061 open. Current main still classifies chat type by looking for `:group:`, `:channel:`, `:direct:`, `:dm:`, or an exactly three-part main session key. Telegram DM topic sessions are still generated as `agent:main:main:thread:<chatId>:<topicId>`, so they miss every Active Memory chat-type branch and fail the allowed-chat-type gate. Active Memory has coverage for a `telegram:direct:...:thread:...` shape, but not for the current default Telegram DM topic key shape reported in the issue.

## Best Possible Solution

Fix this in the bundled Active Memory plugin by classifying thread-suffixed session keys through their parent/base session before applying `allowedChatTypes`. Add focused tests for `agent:main:main:thread:<chatId>:<topicId>` with `messageProvider: "telegram"` and `channelId: "telegram"`, plus a guard that non-direct thread parents still classify as their parent chat type. Consider adding a skip/debug line for unrecognized chat types so future session-key formats do not fail silently.
