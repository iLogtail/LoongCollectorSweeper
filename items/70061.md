## Summary

Keep #70061 open. Current main still matches the reported failure: Active Memory does not classify default Telegram DM topic session keys like `agent:main:main:thread:<chatId>:<topicId>` as `direct`, so the chat-type gate fails before memory recall runs. The related Telegram DM thread work is real and intentional, but it does not update Active Memory for this default key shape.

## Best Possible Solution

Keep the issue open and fix `extensions/active-memory` by classifying thread-suffixed DM session keys whose base is the configured main key as `direct`. Add a regression test using `agent:main:main:thread:<chatId>:<topicId>` alongside the existing `:direct:` thread coverage, and consider adding a trace/status line for chat-type filter skips so future format gaps are observable.

## Evidence
