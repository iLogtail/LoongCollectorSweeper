## Summary

Keep open. Current main still does not implement a durable macOS desktop launch-at-login preference or post-update reconciliation for the `ai.openclaw.mac` LaunchAgent. The report is specific, recent, and not covered by the related gateway LaunchAgent fixes or the narrower #40213 KeepAlive/manual-quit fix.

## Best Possible Solution

Keep #39588 open and fix the desktop macOS launch-at-login contract directly: persist the user's intended launch-at-login preference separately from live launchd loaded status, distinguish disabled from installed-but-unloaded/stale/replaced LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` after app startup and Sparkle updates, and add focused tests for stale/unloaded plist state so the UI cannot silently convert update-time launchd drift into a disabled toggle.

## Evidence
