## Summary

Keep #39588 open. Current main still models the macOS app Launch at login toggle as live LaunchAgent state, not persisted user intent, and I found no app-side Sparkle/post-update reconciliation path for `ai.openclaw.mac`. The related merged #40213 work is narrower: it removed `KeepAlive` so the app does not relaunch after manual quit, but it does not address update-time persistence or UI/launchd drift.

## Best Possible Solution

Keep #39588 open and fix the macOS app Launch at login contract directly: persist the user's intended launch-at-login preference separately from live launchd state, distinguish disabled from stale/unloaded/repair-needed LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale plist, unloaded job, UI reconciliation, and post-update behavior. Treat #40213 as the separate relaunch-after-quit fix, not the canonical resolution for this issue.
