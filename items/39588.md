## Summary

Keep open. Current main still has the macOS desktop app launch-at-login toggle driven by live launchd status and then written back through the same state property, with no separate persisted user intent or Sparkle/update reconciliation. The related gateway LaunchAgent fixes and #40213 KeepAlive fix are useful adjacent work, but they do not solve the reported toggle persistence/update drift.

## Best Possible Solution

Keep #39588 open and implement a desktop-app-specific reconciliation path: persist the user's desired launch-at-login intent separately from live launchd loaded state, make startup/post-update repair `ai.openclaw.mac.plist` to the current bundle path when intent is enabled, expose loaded/stale/error state separately from the toggle, and add Swift regression coverage for status drift and stale plist/update repair.
