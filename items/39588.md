## Summary

Keep #39588 open. Current main still models the macOS desktop app Launch at login setting as live LaunchAgent state, not as persisted user intent, and I found no Sparkle/post-update reconciliation path for `ai.openclaw.mac`. The related shipped fix #40213 is narrower: it removed `KeepAlive` to stop relaunch-after-quit behavior, but it does not address update-time persistence or UI/launchd drift.

## Best Possible Solution

Keep #39588 open and fix the desktop macOS Launch at login contract directly: persist the user's intended launch-at-login preference separately from live launchd state, represent disabled versus stale/unloaded/repair-needed LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale plist/unloaded job/UI reconciliation behavior. #40213 should remain treated as the separate relaunch-after-quit fix, not as the canonical resolution for this issue.
