## Summary

Keep open. Current main already has named browser profiles and manual create/delete support, but it does not implement the issue's requested per-agent dynamic/ephemeral profile flow. The agent-facing browser tool only accepts selecting an existing `profile`, and the gateway browser request layer explicitly blocks persistent profile mutations such as `/profiles/create` and `/profiles/:name`.

## Evidence

  - file: [docs/cli/browser.md:88](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/docs/cli/browser.md#L88)
