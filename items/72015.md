## Summary

Keep open. Current main partially addresses the hard-timeout symptom from related #65517, but the core reliability request remains: Active Memory is still a blocking before_prompt_build sub-agent, still inherits the current/primary model by default, and QMD gateway startup still arms all configured QMD-backed agents without an active-only or max-concurrent boot control.

## Best Possible Solution

Keep the issue open and narrow it into concrete reliability work: preserve the shipped hard-timeout guard, then add an explicit nonblocking/deadline mode for Active Memory, per-agent/session recall concurrency or drop/reuse behavior, safer default model selection or warning, QMD boot startup controls such as max concurrent boot updates or active-only/lazy initialization, and status/doctor warnings for expensive configurations.
