## Summary

Keep open. Current main still stores workspace setup state under `<workspace>/.openclaw/workspace-state.json`, and the tests still assert that legacy path. The PR targets a real remaining workspace compatibility/path cleanup, but its latest patch has unresolved review concerns around best-effort legacy probing and root filename collision, so it should be revised rather than closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open for a maintainer or contributor revision: move state persistence out of the dot directory, add backward-compatible legacy reads that treat dot-path errors as best-effort, avoid clobbering foreign root files with a namespaced/non-dot filename or ownership marker, and update the workspace tests plus any e2e fixtures that still seed `.openclaw/workspace-state.json`.
