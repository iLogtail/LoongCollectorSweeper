## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` setting, and the Feishu inbound path still downloads file/media resources, forwards them as `MediaPaths`, and lets the shared media-understanding pipeline extract text/PDF/text-like attachment content into the agent prompt.

## Best Possible Solution

Add a real, documented Feishu attachment-ingestion control, likely in `extensions/feishu/src/config-schema.ts`, docs/channels Feishu config, and Feishu inbound tests. The fix should short-circuit file/document media before `downloadMessageResourceFeishu` and omit `MediaPaths` for ignored file content while preserving a placeholder such as `<media:document> (filename)` in `BodyForAgent`. Consider whether the option should be Feishu-specific (`ignoreFileContent`) or a generic channel media policy if other channels need the same control.
