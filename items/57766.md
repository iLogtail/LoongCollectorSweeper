## Summary

Keep open. Current `main` still exposes `deliverOutboundPayloads()` as a bare `Promise<OutboundDeliveryResult[]>`, still drops hook-cancelled payloads by returning no results, and still rethrows non-`bestEffort` delivery errors without surfacing any partial-send metadata. Current callers also continue to interpret empty results as delivery failure, so the ambiguity described in #57766 remains on `main` at `c65aa1d2a6e573d49619c4dd0698a76010476e7d`.

## Evidence

- **Public return type still has no outcome metadata:** `deliverOutboundPayloads` still returns `Promise<OutboundDeliveryResult[]>`, and `OutboundDeliveryResult` itself only carries channel/message identifiers plus optional metadata. There is no `cancelledByHook`, `sentBeforeError`, or equivalent discriminator in the current API.
