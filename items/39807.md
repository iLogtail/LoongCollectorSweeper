## Summary

Keep open. Current `main` still does not attach a trackable `profileId` to `models.providers.<id>.apiKey` auth, so billing/auth cooldown state cannot be recorded for inline-key providers. The linked PR #39836 proposed a synthetic inline profile fix, but that change is not present on `main`.

## Evidence

- **Inline provider auth still returns no profileId:** Current `resolveApiKeyForProvider()` returns `{ apiKey, source, mode }` for `models.providers` API keys with no synthetic or default profile id, so inline-key providers still do not enter auth-profile cooldown tracking.
  - file: [src/agents/model-auth.ts:553](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/model-auth.ts#L553)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '540,560p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Failure tracking is skipped when profileId is missing:** Embedded runner only calls `markAuthProfileFailure()` when `profileId` is present. For inline-key providers returned from `models.json`, that path is skipped entirely.
  - file: [src/agents/pi-embedded-runner/run.ts:658](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/pi-embedded-runner/run.ts#L658)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '651,666p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Fallback cooldown gating only applies when profile ids exist:** Model fallback checks cooldown/disable state only when `resolveAuthProfileOrder()` returns one or more profile ids. Inline-only providers with no auth-profile entries bypass this gate.
  - file: [src/agents/model-fallback.ts:755](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/model-fallback.ts#L755)

  - file: [src/agents/model-auth.profiles.test.ts:609](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/model-auth.profiles.test.ts#L609)
