## Summary

Keep open. Current main still implements the old sandbox workspaceAccess semantics: rw uses the agent workspace as /workspace, none/ro use the sandbox workspace but keep it read-only for mutations, and split ro/rw workspaces can still expose the agent workspace at /agent. The issue is recent, concrete, unprotected, and not fully superseded by narrower writable-none trackers.

## Best Possible Solution

Keep #36525 open for maintainer product direction and implementation. A complete fix would make /workspace consistently represent the per-scope sandbox workspace, make mount and file-tool writeability follow resolved mount writeability, remove implicit /agent exposure unless explicitly configured through documented binds, update Docker/browser/SSH/OpenShell paths together, and revise tests plus public sandboxing docs. Keep #37634/#46300 as narrower writable-none implementation context, not as full supersession of this broader refactor.
