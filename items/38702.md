## Summary

Current `main` still has only partial WebChat input-history support. It can store locally sent drafts and recall the most recent item only while the composer is empty, but it does not implement the requested ArrowUp/ArrowDown navigation semantics, draft preservation/restoration, caret/selection/modifier/IME guards, or session-message-derived history. The related PR #38709 explicitly targets this gap, but it is open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38702 open until WebChat implements the full in-session input-history semantics, likely by merging or reworking PR #38709: derive recallable entries from current session user messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around loading/reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
