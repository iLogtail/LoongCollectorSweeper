## Summary

Keep #38702 open. Current main still only has partial WebChat input history: it stores locally sent drafts and recalls them only when the composer is empty. It does not implement the requested draft preservation/restoration, session-message-derived history, caret boundary rules, modifier/IME/selection guards, or loading/reload hardening. The related PR #38709 directly targets the missing behavior, but it remains open and unmerged in the provided context.

## Best Possible Solution

Keep the issue open until WebChat implements the requested in-session history semantics, likely by merging or reworking PR #38709: derive recallable entries from rendered/current session user messages, capture and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around session reload/loading boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
