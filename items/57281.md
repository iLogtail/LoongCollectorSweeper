## Summary

Current `main` still performs auth-profile failure marking before profile rotation for both assistant-side and prompt-side failover. Because the rate-limit path does not bypass that marking and the marking path takes the auth-profile store lock, the reported lock-contention failure mode remains plausible on `main`; this item should stay open. The issue timeline also already links an open follow-up PR (#57283) with the same proposed fix direction.
