## Summary

Keep open. Current main still lacks the requested Control UI session-switcher/modal creation path: the only visible New session control sends `/new`, which resets the current `sessionKey` rather than creating and selecting a new dashboard session via `sessions.create`. The backend API exists and is tested, so this PR or a cleaned-up replacement remains useful, but the PR discussion shows real implementation concerns that still need maintainer review before merge.

## Best Possible Solution

Keep the PR open for maintainer review, or replace it with a focused Control UI implementation that adds a session-switcher-adjacent New Session flow using `sessions.create` with a user-visible label, unique dashboard key generation, robust current-agent resolution, offline/error handling, and targeted UI/unit coverage.
