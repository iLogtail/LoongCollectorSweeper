## Summary

Keep open. The reported Control UI blank-page crash was confirmed by two other users, and I do not have strong enough current-main evidence to conservatively auto-close it. Current UI entry analysis suggests the specific `src/utils.ts -> home-dir.ts -> process.cwd()` leak path described in the issue is not reachable from `ui/src/main.ts`, and the only reachable local `process` reference is guarded for Vitest. But `src/plugin-sdk/browser-config-runtime.ts` and `src/plugin-sdk/browser-config-support.ts` still re-export `CONFIG_DIR` from `src/utils.ts`, so without a live browser/build reproduction or a clear fixing commit in history, the report is not strong-closeable as obsolete.
