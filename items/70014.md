## Summary

Keep #70014 open. Current main still appears to skip the primary/trigger workspace whenever `resolveMemoryDreamingWorkspaces(cfg)` returns any configured agent workspaces. That matches the report: a config with AGI sub-agent entries can process those workspaces while never adding `/root/.openclaw/workspace` from the cron/heartbeat context.

## Best Possible Solution

Keep the issue open and fix memory-core workspace enumeration so scheduled dreaming processes the trigger/primary workspace plus configured agent workspaces, deduped by normalized path. The likely implementation should adjust `runShortTermDreamingPromotionIfTriggered` and the doctor/status aggregation to include `ctx.workspaceDir` or manager status workspace even when configured workspaces exist, then add regression tests for a non-empty `agents.list` containing subagents while the primary workspace is only available from the runtime context.

## Evidence

- **Trigger path only falls back to primary workspace when configured list is empty:** `runShortTermDreamingPromotionIfTriggered` derives `workspaces` from `resolveMemoryDreamingWorkspaces(params.cfg)` and only pushes `params.workspaceDir` when `workspaces.length === 0`. If four AGI workspaces resolve from config, the primary context workspace is not added.
