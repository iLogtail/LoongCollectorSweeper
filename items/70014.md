## Summary

Keep open. Current main still has the reported product gap: the memory-core dreaming workspace resolver enumerates `agents.list` and only falls back to the default agent when that list is empty, so a config with AGI/sub-agent entries but no explicit primary `main` entry can skip the primary workspace instead of processing it alongside the sub-agent workspaces.

## Best Possible Solution

Keep #70014 open and fix the resolver: include the primary/default agent workspace in `resolveMemoryDreamingWorkspaces` even when `agents.list` is non-empty, dedupe by normalized workspace path, preserve listed-agent attribution, and add regression tests for `agents.defaults.workspace` plus AGI/sub-agent-only `agents.list`. The dreaming trigger and phase/session-ingestion paths should continue to share that single resolver.
