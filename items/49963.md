## Summary

Keep open. Current `main` and latest release still hardcode the direct embedding batch timeout to 2 minutes remote / 10 minutes local, so the reported behavior is not implemented. The open PR also has unresolved review issues: `timeoutMinutes` is defaulted to 60 when unset, which makes the proposed fallback branch unreachable, and the later 30-minute clamp would truncate explicit higher user configs.
