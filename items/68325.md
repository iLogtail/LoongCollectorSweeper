## Summary

Keep PR #68325 open. Current main still lacks Telegram `allowBots` config/schema/runtime support, and the PR is a concrete attempt at a real missing Telegram parity feature. The PR likely needs maintainer review and a rebase because its broad bot drop interacts with the existing `channel_post` bot-to-bot path, but that is not a conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, add Telegram `allowBots` across public config types, Zod schema, generated metadata/help/UI hints, docs, and runtime filtering, and add regression tests that separately cover bot-authored group messages and existing `channel_post` behavior. The broader related feature request #8295 should remain the canonical product need until this or equivalent work lands.
