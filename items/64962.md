## Summary

Keep #64962 open. Current main still implements the reported timeout-compaction fall-through: high-context timeouts trigger `contextEngine.compact()`, but only `compacted: true` retries the prompt; `compacted: false` logs the same "compaction did not reduce context" warning and falls through to normal timeout/error handling. The regression tests also explicitly assert this current behavior, so the requested escalation is not implemented on main.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/agents/pi-embedded-runner/run.ts`: after timeout-triggered compaction returns `compacted:false` or otherwise fails to reduce context, route to an explicit recovery ladder instead of normal timeout handling. A conservative implementation would reuse the existing overflow recovery pieces where possible, such as model/profile fallback only when dependency contracts allow it, `compact_then_truncate`/tool-result truncation when the precheck data supports it, and finally a clear blocked-session/user-facing recovery path. Add regression coverage replacing the current fall-through expectations in `run.timeout-triggered-compaction.test.ts`.
