## Summary

Keep #71416 open. Current main still catches any Gateway CLI failure, emits only a stderr diagnostic, and invokes the embedded agent. The relevant tests and CLI docs describe/codify that fallback behavior, and the registered `openclaw agent` options still do not include an opt-in fallback flag or a machine-readable fallback transport marker.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change in `src/commands/agent-via-gateway.ts` and option registration. The best fix is to make Gateway-to-embedded fallback explicit and script-detectable: either fail unless an `--allow-embedded-fallback` flag is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus clear non-JSON warning behavior, with tests and `docs/cli/agent.md` updated together.
