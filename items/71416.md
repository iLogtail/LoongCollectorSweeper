## Summary

Keep #71416 open. Current main still catches any Gateway agent failure, emits only the existing stderr diagnostic, then runs the embedded agent path. The current tests and docs codify that fallback rather than adding an opt-in flag, nonzero exit, stdout banner, or stable JSON transport/fallback marker.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best fix is to make Gateway fallback explicit and script-detectable: either fail closed unless an `--allow-embedded-fallback` flag is passed, or add stable JSON metadata such as `meta.transport`/`meta.fallback` plus a clear non-JSON warning path, then update `src/commands/agent-via-gateway.ts`, CLI option registration, tests, and `docs/cli/agent.md` together.
