## Summary

Keep #17683 open. Current main has useful partial controls for cron and hook-triggered isolated agent runs, including timeouts, light context, tool allow-lists, and post-run usage/cost telemetry, but it still lacks the requested first-class scoped/script-limited mode with native per-job token caps, daily spend ceilings, budget aborts, dry-run cost preview, and admin-defined script-only execution.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best path is a native scoped automation mode across cron, hooks, gateway protocol, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts using runtime usage accounting, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual run logs.
