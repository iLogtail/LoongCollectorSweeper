## Summary

Keep open. Current main still has the reported legacy SQLite schema hazard: webhook `create_flow` correctly relies on the route-bound session, but the flow store inserts only `owner_key` while the migration only adds/copies `owner_key` from legacy `owner_session_key` and does not rebuild the table to remove or satisfy a surviving `owner_session_key TEXT NOT NULL` column. The linked PR #65767 appears to target the same root cause, but it is still open/unmerged in the provided context, so this is not implemented on main.

## Best Possible Solution

Keep the issue open until a migration lands on main that handles existing `flow_runs` tables with `owner_session_key TEXT NOT NULL`, preserving legacy data and allowing new `owner_key` inserts. The fix should include a regression test that constructs the legacy schema, runs restore/migration, and verifies webhook or TaskFlow `createManaged` inserts succeed. PR #65767 is the natural candidate to review or supersede.
