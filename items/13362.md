## Summary

Keep #13362 open. Current main still does not implement `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode, and doctor still has repair paths that can write config and mutate runtime/plugin state. The related global agent dry-run issue #41418 is complementary, not a replacement for doctor/config validation.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements or explicitly rejects a doctor-specific dry-run mode. The likely implementation is to add `openclaw doctor --dry-run`, thread `dryRun` through `DoctorOptions`, the prompter/repair mode, config flow, and every doctor contribution, make all repair/write paths emit planned changes without mutating config/state/services/runtime deps, define exit-code behavior for findings, update `docs/cli/doctor.md` and public docs, and add focused tests proving dry-run does not mutate config, cron/state stores, service state, bundled plugin deps, or plugin registry state.

- The requested non-zero exit behavior for findings still needs product semantics so automation can distinguish validation findings from command failures.
