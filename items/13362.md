## Summary

Keep #13362 open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still has no `openclaw doctor --dry-run` flag or equivalent doctor-wide no-write mode. The prior issue discussion already kept this open, and the related global agent dry-run issue #41418 is complementary rather than a superseding doctor/config validation fix.

## Best Possible Solution

Keep #13362 open until OpenClaw implements a doctor-specific `--dry-run` mode or maintainers explicitly reject the product direction. A good fix should add the CLI flag, thread `dryRun` through `DoctorOptions` and doctor repair/write contributions, block all config/state/service/runtime-dependency writes while still producing recommendations, define exit-code behavior for detected issues, update `docs.openclaw.ai/cli/doctor` and `docs.openclaw.ai/gateway/doctor`, and add focused tests proving doctor recommendations run without mutating config or service state.
