## Summary

Current main still preserves Slack DM auto-thread session routing when `replyToMode="all"`: top-level DMs get `canonicalThreadId` from their own message timestamp and `resolveThreadSessionKeys` appends `:thread:<ts>`. The channel regression from #32320 is covered, but the DM case reported in #58832 remains present and is even asserted by an existing test.

## Evidence

- **DM auto-thread routing still active:** For non-roomish Slack messages, `canonicalThreadId` becomes `autoThreadId`; `autoThreadId` is the message's own `messageTs` when `!isThreadReply && replyToMode === "all"`. The inline comment says this preserves existing DM auto-thread behavior.
