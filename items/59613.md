## Summary

Keep open. Current main still matches the reported failure path: Docker sandbox `workspaceAccess: "none"` mounts `/workspace` read-only, the tool layer still exposes sandbox write/edit for `none`, and the filesystem bridge rejects writes unless access is `rw`. The related #59718 is adjacent design work for least-privilege writable binds, but it is still open and does not supersede this bug report by itself.

## Evidence
