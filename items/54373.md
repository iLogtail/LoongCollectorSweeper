## Summary

Keep open. Current main does not implement inline source/injected_at/volatile metadata for injected bootstrap context, and the discussion explicitly leaves Phase 2-4 work beyond open PR #54830. This is core prompt/memory/runtime behavior, not a ClawHub-only extension item and not safely duplicate-closable yet.

## Best Possible Solution

Keep the RFC open for maintainer product/design review. The concrete path is to review or supersede PR #54830 for Phase 1, then either split Phase 2-4 into narrower canonical issues or implement them in core prompt assembly, memory host contracts, and embedded runner refresh logic with tests around cache-boundary behavior and bootstrap budget preservation.
