## Summary

Do not close. Current `main` has already absorbed the skills-install hardening and some orphan-run cleanup, but it does not implement the PR's session lock queue cap and it still does not do orphan detection inside the sweeper path described in the PR. That means the PR is only partially superseded, not implemented-on-main.
