## Summary

Current main still matches the report: `ui.seamColor` is validated and returned by `talk.config`, but the Control UI bootstrap contract/endpoint does not include it, the frontend bootstrap state does not consume it, and the CSS theme variables remain static unless a separate local custom theme is used. The linked PR #56103 is open and unmerged, so the issue should remain open.

## Best Possible Solution

Keep #56068 open. The best maintainer path is either to wire `ui.seamColor` into the Control UI bootstrap or another authenticated frontend config path, derive the relevant CSS variables (`--accent`, `--accent-hover`, `--accent-subtle`, `--ring`, `--primary`) with focused tests, or explicitly document that web Control UI theming is intentionally controlled elsewhere. PR #56103 can be reviewed as a candidate implementation, but the issue should not close until main ships a fix or maintainers make the scope call.
