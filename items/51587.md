## Summary

Keep open. Current main still has the strict `minuteField === "0"` checks, so cron expressions such as `*/15 * * * *`, `0,30 * * * *`, and `0-30 * * * *` still do not receive the default top-of-hour stagger. The related PR #51594 tracks the same fix but is open and unmerged, so the underlying issue remains unresolved on main.
