## Summary

Keep issue #51587 open. Current main still uses literal `minuteField === "0"` checks for default top-of-hour stagger detection, so step/list/range cron expressions such as `*/15 * * * *` and `0,30 * * * *` still do not receive the default stagger. The linked PR #51594 tracks a plausible fix but is still open/unmerged, so the issue remains the canonical bug tracker rather than cleanup-closeable.

## Best Possible Solution

Keep #51587 open and use the linked PR work as input. The best fix belongs in `src/cron/stagger.ts`: replace literal minute equality with a small cron-minute parser or a Croner-aligned helper that detects whether minute 0 is included for 5-field and 6-field recurring expressions, then add focused tests in `src/cron/stagger.test.ts` plus at least one create/normalize integration assertion for `*/15 * * * *` and `0,30 * * * *`. Maintainers should explicitly settle edge semantics for every-minute and unusual step/range forms before merging.
