## Summary

Current main already fixes the Telegram/DM sandbox bypass by deriving a runtime-only per-account direct-chat policy key for external DMs while preserving `dmScope="main"` session continuity. The PR's dedicated `agent:<id>:dm` bucket is no longer the best implementation, and review comments correctly noted peer-mixing and history-continuity concerns with that approach.

## Best Possible Solution

Keep the current main runtime-policy/sandboxSessionKey implementation, which preserves shared DM session history while sandboxing external direct-chat runtime behavior. Do not merge #70442's broad branch; handle unrelated bundled fixes in focused PRs or their canonical issues.
