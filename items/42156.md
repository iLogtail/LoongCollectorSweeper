## Summary

Current main has not implemented the requested broad fix. Telegram callback queries are still answered inside the callback_query handler, which is registered after the sequentialize middleware. Main only appears to have a narrower approval-callback lane workaround, while non-approval callback data still uses the normal chat/topic sequential key and can queue behind an active turn.
