## Summary

Keep #13601 open. Current main has partial persistent cron run history through per-job JSONL files plus Gateway/UI status, delivery, query, sort, and paging filters, but the issue's central CLI log/history command surface, time-window filtering, success-rate/list aggregates, and manual retention/prune workflow are still missing. The existing steipete/Codex issue comment also kept it open for the same reason.

## Best Possible Solution

Keep #13601 open until maintainers either complete the requested surface or deliberately narrow the issue. The likely implementation path is to add timestamp filters to `readCronRunLogEntriesPage`, `cron.runs`, protocol schemas, UI state, and CLI options; expose a clearer `cron logs` or `cron history` command with status/time/follow/export behavior; add success-rate and failure aggregate calculations to `cron list` or a dedicated history summary; and decide whether manual prune/vacuum/history-retention-days belongs in core or should be explicitly documented out of scope.
