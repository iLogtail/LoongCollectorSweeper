## Summary

Keep #69445 open. Current main still builds `/status` and `session_status` with provider usage when available, and there is no per-agent `status.hideUsage`, section allowlist, or equivalent config/schema/API surface. Related #10005 overlaps, but it is broader full status-card hiding; this issue captures a narrower per-agent usage/billing privacy requirement that remains actionable.

## Best Possible Solution

Keep #69445 open as a concrete feature gap. A good implementation would add a typed, schema-validated per-agent status visibility setting such as `agents.list[].status.hideUsage` or a section allowlist, thread it through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, preserve current behavior by default, and add focused tests/docs for both chat `/status` and the tool path. Coordinate with #10005 if maintainers want one canonical status-card visibility design.
