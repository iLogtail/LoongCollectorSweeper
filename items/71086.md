## Summary

Keep PR #71086 open. Current main still always surfaces mutating-tool warnings, so the PR targets a real unfixed behavior, but the posted review concern is valid: the patch suppresses warnings based only on assistant reply presence and could hide unresolved mutating-tool failures after partial/error turns. The Greptile comment about the stale `payloads.ts` comment is also accurate.

## Best Possible Solution

Keep the PR open and ask for a narrower implementation: suppress stale mutating-tool warnings only when the turn has a successful terminal state or the failed mutating action was actually recovered, while preserving warnings for unresolved mutating failures after error/interrupted turns. Add regression coverage for both the recovered-success case and the partial/error case, and update the stale `payloads.ts` comment noted in review.
