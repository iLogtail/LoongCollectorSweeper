## Summary

Keep open. Current main has a newer ACP task-completion notification path, but the specific #70264 gap remains: `streamTo:"parent"` ACP relay still wakes heartbeat with `acp:spawn:stream`, that reason is not action-priority, and heartbeat prompt selection still falls through to the generic heartbeat prompt for ACP terminal system events. The direct task terminal mirror path also sends a channel message plus transcript mirror without invoking the parent agent turn path used by subagent announcements.

## Best Possible Solution

Keep #70264 open and use #56442 as implementation background rather than a close target. The best fix is to route ACP terminal completions through the subagent-style announce/parent-turn path with structured `task_completion` internal events, or otherwise make ACP terminal events action-priority heartbeat wakes with an ACP-completion prompt. Add regression coverage for `runtime:"acp", streamTo:"parent", sessions_yield()` with heartbeat enabled, and for heartbeat-disabled behavior if the announce path is used. Clarify `streamTo:"parent"` as progress relay only unless completion wake/notify is explicitly enabled.
