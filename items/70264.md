## Summary

Keep #70264 open. Current main still matches the reported heartbeat-enabled ACP gap: ACP stream completion is mirrored as system events and requests a generic `acp:spawn:stream` heartbeat wake, but that reason is not action-priority and the heartbeat runner only builds special event prompts for exec completions and cron events. The related #56442 direction is still only an open/unmerged implementation path, not shipped behavior on main.

## Best Possible Solution

Keep the issue open and use it to drive a core fix. The best path is to adapt the open #56442-style `parentUpdates: "notify"`/subagent-announce routing or equivalent opt-in `wakeOwnerOnDone` behavior so ACP `mode:"run"` terminal completions can wake the parent with an event-carrying prompt. Add regression coverage for heartbeat-enabled `streamTo:"parent"` completion, heartbeat-disabled completion if the fix decouples from heartbeat, and unchanged lightweight `streamTo:"parent"` progress relay semantics. Update `docs/tools/acp-agents.md`/public docs to clarify that `streamTo` is progress relay unless paired with the new completion-routing option.
