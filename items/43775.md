## Summary

Keep #43775 open. Current main has partial newer-config hardening for service/process mutation paths, and merged GPT-5.4 delivery hardening reduces silent follow-up drops, but the exact reported config-stamp mismatch is not fully implemented: ordinary config loads still only warn, the proposed in-memory downstamp PR was closed unmerged, and manual gateway startup can still continue with a future `meta.lastTouchedVersion` outside service-marker/force paths.

## Best Possible Solution

Keep the issue open for a targeted maintainer fix or explicit product decision. The strongest path is to either fail closed for all gateway runtime starts when `openclaw.json` was written by a newer OpenClaw, or safely downstamp the metadata in memory for the current session, then add regression coverage around future-version config plus Discord/OpenAI-Codex delivery or startup behavior. If maintainers decide service-mode protection is sufficient, close only with an explicit explanation that manual `gateway run` remains warn-only by design.
