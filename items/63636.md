## Summary

Keep open. Current main does not contain the guarded `estimateMessageTokens()` implementation from this PR: compaction and pre-prompt token estimation still call the pinned upstream `estimateTokens()` directly, and the pinned upstream implementation still has unchecked `.length` reads on assistant/toolResult content. The linked bug report #63612 is still open, so this PR remains a plausible implementation candidate, though it needs rebase/conflict resolution and maintainer review of the remaining fallback-shape scope.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, resolve the remaining fallback-shape review decision, keep the shared guarded estimator in `src/agents/compaction.ts`, route compaction/pre-prompt/metrics token accounting through it, and run the changed agent checks before landing. After it lands, #63612 can close as implemented.
