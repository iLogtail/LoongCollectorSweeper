## Summary

Keep open. Current main at 6a688e33f66f3965968343922e874cc0e9440998 still lacks the Docker/DooD listen-vs-advertise split for sandbox browser bridge/CDP URLs, and the PR remains a useful but non-mergeable draft because it targets superseded browser paths and needs a current plugin-sdk/browser-runtime rewrite.

## Best Possible Solution

Keep the PR open as a real problem area, but require a current-path rewrite: add an advertised/container-reachable browser bridge URL contract through `src/plugin-sdk/browser-bridge.ts` and the browser plugin runtime API, derive Dockerized gateway CDP/bridge reachability deliberately, propagate the advertised URL through `SandboxBrowserContext` and `buildEmbeddedSandboxInfo()`, add Docker/DooD and non-Docker regression coverage, and avoid unrelated baseline or broad compose changes.
