## Summary

Keep open. Current main still has the PR's central Docker/DooD sandbox browser networking gap: the sandbox browser bridge and sandbox CDP readiness path are still loopback-only, and the active plugin SDK/browser runtime bridge contract has no advertised/container-reachable URL. The PR is not mergeable as-is because it is a draft with obsolete-path and unrelated-churn concerns, but the product bug it is trying to fix remains real.

## Best Possible Solution

Keep this open or replace it with a current-path PR. The best fix is to add an explicit advertised/container-reachable bridge URL through src/plugin-sdk/browser-bridge.ts and the bundled browser runtime, keep the host-local bridgeUrl for gateway/browser-tool calls, pass an advertised URL into embedded sandbox info, resolve sandbox browser CDP host/port separately for Dockerized gateways, make attach-only non-loopback tab creation bypass persistent Playwright when needed, and cover Dockerized gateway plus non-Docker host cases with focused tests/docs.
