## Summary

Keep open. Current `main` still hardcodes loopback-only sandbox browser bridge/CDP paths, and the draft PR is aimed partly at superseded core browser files rather than the active plugin-sdk/browser extension seam, so this is still a real problem area but not mergeable as-is.

## Evidence

- **Sandbox browser flow still assumes host loopback for CDP reachability:** `waitForSandboxCdp()` probes `http://127.0.0.1:${cdpPort}/json/version`, `buildSandboxBrowserResolvedConfig()` hardcodes `cdpHost = "127.0.0.1"` with `cdpIsLoopback: true`, Docker publish args bind CDP/noVNC to `127.0.0.1`, and the timeout message still reports `127.0.0.1:${mappedCdp}`. That matches the maintainer review that Dockerized gateway/container-reachable addressing is still unresolved on `main`.
