## Summary

Keep open. Current `main` still lacks the Mattermost WebSocket ping/pong keepalive proposed here, so this PR is neither implemented nor obsolete. The supplied PR discussion only surfaces a minor timer-cleanup nit, not a reason to conservatively close it.

## Evidence

- **Current websocket monitor has no ping/pong keepalive path:** On `main`, `MattermostWebSocketLike` only exposes `open`/`message`/`close`/`error` plus `send`/`close`/`terminate`, and `createMattermostConnectOnce` only accepts `getBotUpdateAt`/`healthCheckIntervalMs`. There is no `pong` listener, `ping()` call, `pingIntervalMs`, or missed-pong termination logic.
