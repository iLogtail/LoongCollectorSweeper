## Summary

Keep #44068 open. Current main still lacks an `application/msword` binary guard and has no legacy `.doc` regression test, while the existing media-understanding path still lets non-vendor `application/msword` reach UTF-16/text heuristics. The PR is a focused bug fix with a directly relevant regression test and positive review discussion, so it is still useful rather than obsolete.

## Best Possible Solution

Keep the PR open and land or reimplement the minimal current patch: add `application/msword` to `isBinaryMediaMime` and keep the focused regression test for the UTF-16/tabular legacy `.doc` false-positive path. Avoid expanding scope into `.doc` extraction, session compaction, or unrelated provider bugs.

## Evidence

The `application/vnd.*` catch-all does not cover plain `application/msword`.
