## Summary

Keep open. Current `main` does not include the proposed `application/msword` binary guard or the regression test, and the existing media-understanding path still appears able to coerce non-vendor binary payloads into text when they look UTF-16/text-like.

## Evidence

- **`application/msword` is still not treated as binary:** On current `main`, `isBinaryMediaMime` covers image/audio/video, `application/octet-stream`, archive MIME types, `+zip`, and `application/vnd.*`, but there is no special case for legacy Word `application/msword`. That means legacy `.doc` MIME still falls through this binary gate.
  - file: [src/media-understanding/apply.ts:309](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/media-understanding/apply.ts#L309)
  - command: `nl -ba src/media-understanding/apply.ts | sed -n '309,340p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Text coercion still applies after the missing binary guard:** `extractFileBlocks` only skips attachments when `isBinaryMediaMime(normalizedRawMime)` is true. If that check misses `application/msword`, the code continues into UTF-16 / UTF-8 text heuristics and can relabel content as `text/plain` or TSV/CSV; only PDFs are excluded from that heuristic.
  - file: [src/media-understanding/apply.ts:387](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/media-understanding/apply.ts#L387)
  - command: `nl -ba src/media-understanding/apply.ts | sed -n '387,405p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Current tests cover nearby cases but not legacy `.doc`:** The current suite has regressions for PDF non-reclassification and `application/vnd.*` Office binaries, but no `application/msword` coverage. A repo search for `application/msword` or `legacy .doc` under `src/media-understanding/apply.test.ts` returned no matches.
  - file: [src/media-understanding/apply.test.ts:1217](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/media-understanding/apply.test.ts#L1217)
