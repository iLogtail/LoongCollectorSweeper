## Summary

Keep PR #72158 open. Current main has a bundled, enabled-by-default Microsoft Foundry provider, but `docs/providers/index.md` still does not list Microsoft Foundry. The PR's one-line index addition targets a real docs gap, though it should be coordinated with the related provider-page PRs because current main also lacks `docs/providers/microsoft-foundry.md`, so the link is not independently merge-ready until that page exists or lands with it.

## Best Possible Solution

Keep #72158 open for now, but fold or rebase its provider-index entry into the canonical Microsoft Foundry docs PR that adds `docs/providers/microsoft-foundry.md` and `docs/docs.json` navigation. The final docs change should include the page, navigation, and index entry together, aligned with the current `extensions/microsoft-foundry` auth and API-selection behavior.
