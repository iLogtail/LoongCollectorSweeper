## Summary

Keep open. Current `main` still writes ACP spawn labels back to the requester's session store instead of resolving the spawned target session store from the child `sessionKey`, so the bug described by PR #57597 is not implemented on `main`. The PR discussion also shows the author already addressed the review-found test defect in a follow-up commit, so this remains useful unresolved work rather than an auto-close candidate.

## Evidence

- **Current main still writes label to requester store:** `persistSpawnedSessionLabel` mutates `params.commandParams.sessionStore` and then calls `updateSessionStore(params.commandParams.storePath, ...)`; it never resolves the spawned session's store from `sessionKey`, so cross-agent spawns still write to the requester's store on `main`.
