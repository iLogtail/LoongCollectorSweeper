## Summary

Keep #32515 open. Current main still does not implement the requested one-click Anthropic browser OAuth onboarding flow. It has useful narrower auth support: Claude CLI reuse, setup-token paste, API-key auth, env-backed ANTHROPIC_OAUTH_TOKEN, and runtime handling for sk-ant-oat credentials. Those do not satisfy the issue's central request for openclaw onboard to open a browser, let the user sign in with Anthropic, detect account type, and configure auth automatically.

## Best Possible Solution

Keep the issue open for maintainer/product handling. If accepted, implement this inside the bundled Anthropic provider as a provider-owned OAuth auth method and manifest auth choice, using the existing provider auth `openUrl`/OAuth handler seams, persisting credentials through auth profiles, updating onboarding/configure/docs, and adding regression coverage for browser OAuth selection plus API-key-versus-OAuth token behavior. If Anthropic does not support a public browser OAuth flow, maintainers should explicitly retitle or close around the narrower Claude CLI/setup-token/API-key UX direction.
