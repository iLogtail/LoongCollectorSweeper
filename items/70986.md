## Summary

Current main still matches the reported memory-wiki lint defects. The extractor runs raw regexes over the full page text after removing only the managed related block, and `toWikiPageSummary` feeds it the raw file, so fenced/inline code, comments, and frontmatter are still candidates. The broken-link resolver still accepts only the page path without `.md` and its basename, not frontmatter `title`. The issue is recent, has independent vault confirmation in the provided comment, and related #64696 is broader/open rather than a canonical shipped fix.

## Best Possible Solution

Keep #70986 open and fix it in `extensions/memory-wiki`: make wikilink extraction markdown-aware or parser-backed, avoid scanning frontmatter/fenced code/inline code/comments, and update `collectBrokenLinkIssues` to use a shared normalized target lookup that includes relative paths, basenames, frontmatter titles, ids, and likely slugified title forms. Add focused regression tests for fenced/inline literal `[[...]]`, title-based wikilinks, and relative links before considering this issue resolved or deduplicated.
