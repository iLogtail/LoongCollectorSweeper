## Summary

Keep #70986 open. Current main still has both reported memory-wiki lint gaps: wikilinks are extracted by a raw regex over the full markdown string, and broken-link validation only recognizes relative path and basename targets, not frontmatter titles. The related #64696 is narrower and still open, so it does not supersede this broader root-cause report.

## Best Possible Solution

Keep #70986 open and fix it in `extensions/memory-wiki`: make wikilink extraction markdown-aware by excluding frontmatter, fenced code, inline code, and comments, then unify lint target resolution with a normalized lookup that includes relative path, basename, title, slugified title, id, and future aliases. Add focused regression tests for fenced/inline literal `[[...]]`, bridge/unsafe-local fenced source pages, frontmatter-title links with Unicode/special characters, and relative markdown links if folded into this fix. Treat #64696 as related narrower evidence, not the canonical replacement for this broader issue.

## Evidence
