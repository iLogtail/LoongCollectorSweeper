## Summary

Current `main` still matches the report. `memory-wiki` link extraction is still a raw regex over the full Markdown minus only the managed related block, and lint still resolves wikilinks only against path/basename, not `frontmatter.title` or aliases. I found no relevant change between the latest release `v2026.4.23` (`a979721`) and current `main` for the affected files, so this is not conservatively closable.

## Evidence

- **Extractor is still markdown-blind on current main:** `extractWikiLinks()` still scans `markdown.replace(RELATED_BLOCK_PATTERN, "")` with raw `[[...]]` and Markdown-link regexes. It does not strip frontmatter, fenced code, inline code, or HTML comments before matching.
