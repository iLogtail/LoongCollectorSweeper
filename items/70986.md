## Summary

#70986 is still actionable on current main. The memory-wiki extractor still scans raw Markdown with the wikilink regex after removing only the managed related block, and lint's broken-link resolver still accepts only extensionless relative paths plus basenames, not page titles or normalized title slugs. The linked #64696 is related but broader/narrower in different places and does not cover the frontmatter-title resolver bug, so this should not be closed as implemented or duplicate.

## Best Possible Solution

Keep #70986 open and fix it in `extensions/memory-wiki`: make `extractWikiLinks` markdown-aware by excluding frontmatter, fenced code, inline code, and comments before Obsidian wikilink extraction; update lint target resolution to use normalized path, basename, page title, title slug, id/aliases if supported, and page-relative Markdown links; add focused regression tests in `markdown.test.ts` and `lint.test.ts` for fenced literals, inline literals, natural-title wikilinks, Unicode/special-character titles, and relative links.
