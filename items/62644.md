## Summary

Keep open. Current main still defaults `tools.sessions.visibility` to `tree`, `sessions_send` still returns the same sparse forbidden error for cross-agent sends, and the guard still blocks delivery before the message is sent. The changelog/config docs now document the security default, but I did not find a current migration warning, doctor/startup validation, or OC-07-specific runtime guidance. The related #18967 parent-scoped send request remains adjacent, not a full superseder for this migration/UX report.

## Best Possible Solution

Keep #62644 open. A conservative maintainer path is to improve the `sessions_send` cross-agent forbidden message with security-default context and the full required knobs (`tools.sessions.visibility=all`, `tools.agentToAgent.enabled`, and sandbox clamp notes), add a doctor/security-audit or startup warning for likely affected multi-agent isolated cron setups where feasible, and link the config docs/migration guidance. Keep #18967 as the separate safe parent-scoped send feature request.
