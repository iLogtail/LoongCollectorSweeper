## Summary

Current main still allows the reported contradictory cron state. The manual/task execution path passes status, error, and delivered directly into applyJobResult; applyJobResult still writes job.state.lastError = result.error, while resolveDeliveryState independently maps delivered: true to lastDeliveryStatus: "delivered". Related PR #41869 tracks the proposed fix but is still open/unmerged, and PR #42086 was closed unmerged, so this should not be closed as implemented.
