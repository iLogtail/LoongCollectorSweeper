## Summary

Keep open. In checked-out `main` at `350d3221800a18283915cd2f3b612ae04ddd490c`, cron still stores `lastError` unconditionally from `result.error` inside `applyJobResult`, so a successful/delivered run can still retain an error string. The closed PR #42086 contains the intended fix for #41764, but that change is not present on this checkout or in the shipped `v2026.4.22` code available locally.
