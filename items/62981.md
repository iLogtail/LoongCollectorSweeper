## Summary

Keep open. Current `main` still appears to have the reported failure mode: the CLI fallback path immediately reruns the embedded agent after any gateway error without changing session selection, `resolveSession()` will reuse a fresh stored `sessionId`, gateway client timeouts only close the client side, and gateway connection close handling does not abort the in-flight server run. The provided timeline cross-references newer lock-related reports (#70004, #70026), which is consistent with this still being an active bug family rather than an obsolete report.

## Evidence

  - file: [src/commands/agent-via-gateway.ts:191](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/commands/agent-via-gateway.ts#L191)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)

  - file: [src/agents/command/session.ts:241](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/command/session.ts#L241)
