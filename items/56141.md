## Summary

Current `main` already addresses the stale `sessionFile` rollover problem and includes regression coverage. The cron/session rollover path now clears inherited `sessionFile` on fresh sessions, the session-file resolver rotates transcript paths when `sessionId` changes, and mainline thread-fork code preserves intentionally supplied forked transcript paths so the earlier PR review concern is avoided.
