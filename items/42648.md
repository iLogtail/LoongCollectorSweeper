## Summary

Keep #42648 open. Current main has useful adjacent memory work, but the requested write pipeline remains meaningfully unimplemented: there is no candidate-memory write resolver that classifies, attaches source, searches near-duplicates/conflicts, chooses create/merge/update/review, preserves revisions, and blocks low-confidence or source-less candidates. The related #49108 / smart-memory ClawHub work is useful evidence for possible implementation direction, but it is open, covers only part of this issue, and does not supersede the broader request.
