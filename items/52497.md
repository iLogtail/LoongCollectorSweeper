## Summary

Keep open. Current main only propagates a timeout-side error when `agent.wait` already returns one, but the gateway wait path can still collapse to a bare `{ status: "timeout" }` when no lifecycle/dedupe terminal snapshot resolves before the timer. That leaves the core model-error-timeout ambiguity from the issue still present, although the broader taxonomy expansion is narrower than stated because label/session resolution failures already return explicit errors.
