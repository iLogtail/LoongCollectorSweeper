## Summary

Keep #52497 open. Current main partially preserves timeout-side errors when `agent.wait` supplies one, but the remaining gateway wait path can still collapse a target run into `{ status: "timeout" }` without carrying a recent model/lifecycle error. The March comment by GMTekAI is still accurate: the broad taxonomy expansion is partly already handled for missing/forbidden targets, but the core timeout reason passthrough gap remains.

## Best Possible Solution

Keep the issue open and fix the gateway wait path so a recent target run model/lifecycle error is preserved when `agent.wait` resolves as `timeout`. A focused implementation should carry the last-known error from pending lifecycle errors into timeout snapshots or the bare timeout fallback, add regression coverage for `agent.wait` and `sessions_send`, and treat the broader failure-taxonomy request as follow-up only where distinct failures still collapse to timeout.
