## Summary

Keep #49665 open. Current main still has the reported lazy-loader failure mode: `createLazy()` caches the dynamic import promise after `.then()` only, has no rejection handler, and never resets `s.promise` when the import fails. The Channels tab still renders through that shared lazy path, so a rejected `import("./views/channels.ts")` can leave the page body empty while `channels.status` has successfully populated state. The referenced fix PR #49697 was closed unmerged in the provided GitHub context, and the current main implementation does not contain its proposed recovery behavior.

## Best Possible Solution

Keep the issue open and fix the shared Control UI lazy-loader in `ui/src/ui/app-render.ts`: catch rejected dynamic imports, surface a diagnostic state instead of rendering `nothing`, reset the cached promise so a later render can retry, and add focused regression coverage for a rejected lazy view import. The prior PR #49697 is useful context, but because it was closed unmerged and current main still lacks the recovery path, it is not a sufficient canonical fix.
