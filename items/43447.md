## Summary

Keep #43447 open. Current main has fixes for related fallback lockout paths, but the specific Moonshot/Kimi failure mode remains plausible: a billing-looking payload carried on HTTP 429 is still classified and surfaced through the rate-limit lane, so the original “insufficient funds masked as rate limit” bug is not resolved well enough to close. The TeamoRouter comment is a useful external workaround for multi-provider failover, but it does not replace a core/provider classification fix.
