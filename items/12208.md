## Summary

Keep #12208 open. Current main still lacks a bounded inner vertical scroll body for every large native chat bubble; SwiftUI and Android chat bodies are rendered as plain stacks inside only the outer transcript scroll/list.

## Best Possible Solution

Keep #12208 open for a focused native chat UI fix. Define the current equivalent of a large bubble, wrap only the bounded body area in a vertical scroll container in apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift, add Android parity in apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt, preserve outer transcript scrolling, and add focused render/smoke coverage or documented manual mobile verification for tall-but-not-truncated content.
