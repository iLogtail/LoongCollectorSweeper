## Summary

Keep #13219 open. Current main still provides transcript-backed cost summaries, session/model aggregations, provider quota snapshots, and diagnostics telemetry, but it does not implement the requested native append-only per-call usage ledger with durable usage.jsonl-style storage, event ids, source/job/run attribution, backfill/idempotency semantics, or ledger-first readers. Related #49064 and #70205 are adjacent transcript-based work and explicitly do not cover the ledger request.

## Best Possible Solution

Keep #13219 open until OpenClaw adds a typed append-only `UsageLedgerEntry` written from the normalized model-call usage/cost seam. The ledger should store provider/model, agentId, sessionKey, sessionId, runId/callId, source/job attribution, token/cache/cost fields, status, cost status/breakdown, latency, and idempotency metadata without prompt/completion content. Then `usage.cost`, session usage readers, CLI/UI summaries, and a backfill command should prefer the ledger while preserving transcript scanning as a compatibility fallback; provider quota snapshots and optional real-time external tools should remain separate adjacent surfaces.

## Risks / Open Questions

- Adjacent transcript summaries, provider quota snapshots, Prometheus/OTel diagnostics, and third-party real-time tools are useful but easy to confuse with the requested native durable ledger.
- A future implementation needs explicit idempotency/backfill design so live ledger entries and transcript-derived historical entries do not double-count usage.
