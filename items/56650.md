## Summary

Keep PR #56650 open. Current main still lacks the non-interactive `paste-token --token` path and still routes GitHub Copilot credentials through the `/copilot_internal/v2/token` exchange, so the central user problem is not implemented on main. The PR is stale and needs revision because review comments flagged missing PAT header propagation, but current main now has a `prepareRuntimeAuth.request` transport override seam that looks suitable for a corrected implementation.

## Best Possible Solution

Keep the PR open or ask for a refreshed version against current main. The best fix is to add non-interactive `paste-token` inputs with tests, implement GitHub Copilot PAT detection/direct enterprise auth in the provider-owned token path, attach Copilot IDE headers through `ProviderPreparedRuntimeAuth.request.headers`, update simple-completion/runtime paths if needed, and document the supported headless auth workflow.
