## Summary

Keep open. Current main still implements the reported behavior: startup failures call the stability bundle failure writer, but the bundle error shape and serializer only preserve `name` and `code`, not `message`, `stack`, or `cause`. The related discussion correctly identifies the same root cause, and PR #71155 appears to track a plausible fix, but it is open/unmerged while #71092 and #71352 were closed unmerged, so the issue is still actionable in this repo.

## Best Possible Solution

Keep #71071 open until #71155 or an equivalent maintainer-reviewed fix lands. The fix should extend the writer-side safe error metadata to include sanitized, bounded `message` and `stack` and, if accepted, a bounded/sanitized `cause` chain; preserve conservative imported-bundle sanitization; and update `src/logging/diagnostic-stability-bundle.test.ts` with a regression covering `gateway.startup_failed` bundles when the recorder snapshot is empty.
