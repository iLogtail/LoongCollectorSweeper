## Summary

Keep open. Current `main` still drops `error.message` and `error.stack` from startup-failure stability bundles, and the current tests explicitly lock in that behavior, so this is not implemented on `main` yet.

## Evidence

- **Bundle error schema only allows `name` and `code`:** The persisted `DiagnosticStabilityBundle` type defines `error` with only `name?: string` and `code?: string`; there is no field for `message`, `stack`, or `cause`.

  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Serializer still strips everything except `name` and `code`:** `readSafeErrorMetadata()` reads only `name` and `code`, returns `undefined` if both are absent, and never inspects `message` or `stack`.
