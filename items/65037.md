## Summary

Keep open. Current main does not implement the PR's central durable pending-final-delivery payload state: subagent retry cleanup still rebuilds announce attempts from the live run record, and the run record type has no pendingFinalDelivery fields. The PR is a plausible focused hardening change tied to the still-open durable subagent completion delivery work, so it is not conservative cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation into the active durable subagent completion delivery work. The best implementation path is to add typed durable pending-final-delivery payload state to the subagent run record, snapshot it at the first failed final delivery boundary, reuse it across restart/retry while allowing safe live outcome/frozen-text refreshes, and clear it on success, give-up, and steer replacement. Any unrelated auto-reply abort queue changes should be split or dropped if current main already covers them.
