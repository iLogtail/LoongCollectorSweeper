## Summary

Keep open. Current main documents and implements ACPX as a host-local runtime path, and remote nodes are documented for `system.run`/`system.which` exec forwarding, not ACP harness placement. The issue is recent, concrete, unprotected, and the requested official Gateway-on-Pi plus harness-on-Mac Discord ACP topology still appears to lack an explicit supported configuration or documented feature boundary.
