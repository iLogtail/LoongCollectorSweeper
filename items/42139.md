## Summary

Keep open. Current main has improved and documented the Discord ACP binding model, and it now clearly says ACPX harness prerequisites live on the Gateway host, but it still does not provide the requested first-class topology where Discord `/acp spawn` on a Raspberry Pi gateway selects and runs Claude/Codex ACP harnesses on a remote Mac node. No canonical follow-up in the provided related context tracks that remaining product/API gap.

## Best Possible Solution

Keep the issue open as a focused product/docs/API item. The maintainers should either explicitly document that bundled ACPX harnesses must run on the Gateway host and show a supported workaround such as a custom `agents.<id>.command` SSH wrapper, or add a first-class ACP remote-node binding API/config path that can select a paired/SSH Mac node for ACP harness execution from Discord `/acp spawn` and typed ACP bindings.
