## Summary

Keep #44027 open. Current main still suppresses text-only tool-result delivery for ordinary group chats regardless of a session's verbose level: `shouldSendToolSummaries` is tied to the group/forum surface gate, and the payload resolver drops non-media, non-approval tool payloads when that gate is false. The existing tests cover default group suppression and forum delivery, but not group + `/verbose on|full`; docs still describe verbose mode as showing tool summaries and explicitly warn that verbose output can expose tool output in group settings. The PR discussion also contains a concrete DingTalk AI Card validation path and addressed the bot review's native-source side-effect concern, so this remains a plausible focused bug fix rather than a stale cleanup close.

## Best Possible Solution

Keep the PR open but rework it against current main: add focused regression coverage for `ChatType: "group"` with `verboseLevel: "on"` and `"full"` delivering text tool summaries, preserve default group suppression, media-only and exec-approval passthrough, forum-topic delivery, and Slack non-DM suppression, then adjust `dispatch-from-config.ts` to let the current verbose state override only the ordinary group-chat suppression.
