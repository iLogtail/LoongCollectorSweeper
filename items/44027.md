## Summary

Keep open. Current `main` still suppresses text tool summaries for group chats regardless of verbose state, so this PR's requested behavior is not implemented on `50e36983bb2d827adc617282c090c0a0359c6eee`.

## Evidence

- **Current dispatch gate ignores verbose level for group tool summaries:** `shouldSendToolSummaries` is derived only from `shouldSendVerboseProgressMessages`, which is `false` for non-forum group chats. The fallback/session verbose level is computed earlier, but it is not used in this gate.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:280](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/auto-reply/reply/dispatch-from-config.ts#L280)
  - command: `nl -ba src/auto-reply/reply/dispatch-from-config.ts | sed -n '260,310p'; nl -ba src/auto-reply/reply/dispatch-from-config.ts | sed -n '646,650p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Tool summary delivery still hard-stops when group suppression is active:** `resolveToolDeliveryPayload` returns `null` for text-only tool payloads when `shouldSendToolSummaries` is false, so verbose group sessions still cannot deliver normal tool summaries.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:863](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/auto-reply/reply/dispatch-from-config.ts#L863)
