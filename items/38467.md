## Summary

Keep #38467 open. Current main still does not ingest or persist Baileys `contacts.upsert` data for WhatsApp, and `openclaw directory peers list --channel whatsapp` remains backed by configured `allowFrom` entries rather than the phone contact list. The generic outbound target resolver can already resolve names from directory entries, but the WhatsApp plugin does not supply contact-name entries for it to use.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add an account-scoped local contact store populated from Baileys `contacts.upsert`, add explicit privacy/config handling such as `storeContacts` if maintainers want opt-in persistence, merge stored contacts into `directory.listPeers` or `listPeersLive` with `name`/`handle`/phone-number fields, and cover contact-event ingestion, directory filtering, outbound unique-name resolution, schema, and docs.
