## Summary

Keep #70403 open. Current main still appends Dream Diary narratives from light and REM phase paths, uses one sweep timestamp across phases, and documents/tests per-phase diary generation. Related open PRs #70332 and #70523 cover adjacent diary hardening and fallback pollution, but they do not clearly supersede this one-entry-per-sweep invariant.

## Best Possible Solution

Keep the PR open for maintainer review. The concrete path is to rebase it onto current main, address the type-only import review comment, and decide the product policy explicitly: either preserve current per-phase diary entries, or land the one-entry-per-sweep behavior with matching updates in extensions/memory-core/src/dreaming-phases.ts, extensions/memory-core/src/dreaming-phases.test.ts, and docs/concepts/dreaming.md. If maintainers choose a broader canonical diary hardening PR, link that PR only after it explicitly covers this invariant.
