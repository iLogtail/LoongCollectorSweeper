## Summary

Keep open. Current main still treats configured `logging.file` paths as literal strings, so the requested `YYYY-MM-DD` expansion is not implemented. The PR is a focused core logging change, but the proposed patch likely needs to be extended to the shared log-file resolver and made deterministic in tests before it is merge-ready.

## Best Possible Solution

Keep the PR open, but ask for a more complete implementation: centralize `YYYY-MM-DD` expansion in the shared logging path resolver, use it from both logger settings and status/reporting paths, add a deterministic fake-timer regression test, and keep broader midnight rollover behavior separate from the rotation work tracked by related logging PRs such as #47370.
