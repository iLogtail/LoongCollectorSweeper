## Summary

Keep open. Current `main` still lacks the WhatsApp Control UI QR-refresh and wait-loop behavior described in the report, and the linked fix PR #63022 remains open rather than merged.

## Evidence

- **Control UI wait path is still a single one-shot request:** `waitWhatsAppLogin` on current `main` sends one `web.login.wait` call with a 120s timeout, updates state once, and returns. There is no polling loop and no QR refresh/handoff logic after scan.

  - file: [ui/src/ui/views/channels.whatsapp.ts:78](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/ui/src/ui/views/channels.whatsapp.ts#L78)
