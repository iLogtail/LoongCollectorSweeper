## Summary

Keep #13487 open. Current main still handles Discord mention/reply activation inside each running account, not as a cross-account target router with the requested mention > reply-target > default-owner precedence. Mention and reply checks compare only against the current bot user ID, route/default resolution happens independently, and `ignoreOtherMentions` remains optional rather than the requested mandatory suppression invariant.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to the owning OpenClaw account/agent, select explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases.
