## Summary

Keep #39037 open. Current main still does not extract Click-to-WhatsApp attribution data from WhatsApp/Baileys contextInfo, does not carry it on WebInboundMessage, and does not persist it into prompt or session metadata. The previous ClawSweeper comment correctly identified the same gap, and the code at current main has not added a CTWA/referral carrier since then.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Extract CTWA ad attribution from Baileys contextInfo.externalAdReply and any Cloud-style referral shape if present, normalize it as untrusted structured context and/or a typed session metadata attribute, expose it to the agent prompt and plugin hook metadata, and add tests for extraction plus prompt/session propagation.
