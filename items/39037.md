## Summary

Keep #39037 open. Current main still does not extract or propagate Click-to-WhatsApp `ctwaClid`/referral metadata from WhatsApp inbound messages. The existing code can locate Baileys `contextInfo` and has a generic untrusted structured context carrier, but the WhatsApp plugin only uses those paths for mentions, replies, contact cards, location/media, and generic hook metadata today.

## Best Possible Solution

Keep #39037 open and implement this in the bundled WhatsApp plugin. Add a focused extractor for Baileys and Cloud-API CTWA/referral data, normalize fields such as `ctwaClid`/`ctwa_clid`, `sourceUrl`, `sourceType`, `sourceId`, `headline`/title, and related ad metadata, then route it into an explicit untrusted structured context block and, if maintainers want durable attribution, hook/session metadata. Add targeted tests for extraction, queueing into `WebInboundMessage.untrustedStructuredContext`, prompt rendering, and any chosen hook/session propagation behavior.
