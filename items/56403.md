## Summary

Keep PR #56403 open. Current main still ships tool-loop detection disabled by default, the runtime resolver uses that disabled default when no config is provided, docs/schema/type metadata still advertise `default: false`, and existing tests still assert the default-disabled behavior. The related #56787 was only a closed companion docs PR, not a merged replacement for this runtime change.

## Best Possible Solution

Keep this open or intentionally supersede it with one canonical maintainer-reviewed PR that flips `tools.loopDetection.enabled` to default true, preserves explicit opt-out, stabilizes repeated exec/no-progress hashing if needed, defines safe generic-repeat blocking semantics, and updates docs, config help, generated schema metadata, TypeScript docs, and regression tests together.
