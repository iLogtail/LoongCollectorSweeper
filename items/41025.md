## Summary

Keep open. Current main still intentionally flattens remote markdown images to alt text, and the Control UI CSP/docs/tests explicitly prohibit remote http(s) image fetches. The PR is a focused implementation for #40945, but it conflicts with the current privacy/networking model and needs maintainer product/security judgment or a broader rework rather than conservative auto-close.

## Best Possible Solution

Keep #41025 open for maintainer review rather than auto-closing. If maintainers want remote inline images, rework the design with an explicit safe path such as a same-origin managed image proxy, allowlist/config, or other privacy-preserving flow, then update the renderer, CSP, tests, and Control UI docs together. If maintainers want the current hardening to stand, close #40945/#41025 with an explicit product/security rationale.
