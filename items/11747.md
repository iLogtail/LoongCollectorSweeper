## Summary

Keep #11747 open. Current main still treats Discord, Telegram, and Slack reactions as session-scoped system events that are queued into the next prompt, not as inputs that wake an immediate agent turn. The closed related plugin-forwarding reports identify valid plugin use cases, but current plugin hooks still do not expose a reaction event or a reaction-driven wake path, so this is not safely closeable as implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake contract, such as `reactionAutoRespond` or `reactionTriggersTurn`, that preserves existing passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn for Discord, Telegram, and Slack. If maintainers prefer plugin-first automation, core still needs reaction event forwarding plus a wake/turn API before this can move cleanly to ClawHub/plugin implementations.
