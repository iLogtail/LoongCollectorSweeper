## Summary

Keep #51832 open. Current main has same-account Slack self-message filtering, but I did not find a core or Slack fix for the reported sibling-bot cascade: with `allowBots: true`, bot-authored messages from another Slack bot account can still be prepared as inbound turns, and gateway failure text is still emitted as normal outbound reply payload. The issue discussion is specific and still actionable, and the related #37842 tracks a different sessions_send loop mechanism rather than this gateway-error channel feedback loop.

## Evidence

Comment 4104261094 identifies self-message/gateway-error suppression as the base fix; comment 4104051112 adds diagnostic value around preserving cascade state. Neither comment makes this obsolete or ClawHub-only.
- **Slack inbound bot filtering is same-account only:** The Slack authorization path drops bot messages only when the message user matches this monitor context's `botUserId`; otherwise bot messages are allowed when `allowBots` is true.
