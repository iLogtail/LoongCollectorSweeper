## Summary

Keep open. Current `main` at `4e4aeacae4a87f2fe09125d462d88cd71a18cb66` still computes `status` session defaults from one global configured model path and does not represent the per-agent-varying case this PR addresses.

## Evidence

- **Status summary still resolves one global default:** `getStatusSummary()` calls `resolveConfiguredStatusModelRef({ cfg, defaultProvider, defaultModel })` once, with no per-agent iteration, then stores that single `configModel`/`configContextTokens` in `sessions.defaults`.
  - file: [src/commands/status.summary.ts:157](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/commands/status.summary.ts#L157)

  - file: [src/commands/status.types.ts:56](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/commands/status.types.ts#L56)
