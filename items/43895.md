## Summary

Keep PR #43895 open. Current main still builds `status` session defaults from a single global configured model and has no `variesByAgent`/`varies by agent` state in the CLI, JSON payload type, or TUI formatter, while per-agent `agents.list[].model` overrides remain a supported config surface.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase it onto current main. The best fix is to compute `sessions.defaults` from each agent's effective default model using the existing status-safe resolver path, keep the concrete model only when all agent defaults match, expose an optional `variesByAgent` state through `StatusSummary`/`GatewayStatusSummary`, render `varies by agent` in CLI and TUI, and add focused regression tests for divergent and uniform per-agent defaults.
