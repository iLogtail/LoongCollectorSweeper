## Summary

Current `main` already contains the Control UI fix for the stuck streaming state, and the same guard logic is present in the latest release tag I could inspect.

## Evidence

- **Tracked side-result terminal events no longer tear down the active run:** `handleChatGatewayEvent` detects terminal `chat` events that belong to tracked `chat.side_result` runs and returns early after removing the tracked run id, so detached BTW/side-result completions do not clear the main run or its streaming state.
  - file: [ui/src/ui/app-gateway.ts:448](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/ui/src/ui/app-gateway.ts#L448)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '440,456p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Session reloads are deferred until the active run finishes:** `session.message` reloads are deferred while `chatRunId` is active and replayed only after terminal chat state when the active run is gone, preventing the mid-stream history reload/reset race.
  - file: [ui/src/ui/app-gateway.ts:461](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/ui/src/ui/app-gateway.ts#L461)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '458,500p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Terminal chat events clear streaming state:** `handleChatEvent` clears `chatStream`, `chatRunId`, and `chatStreamStartedAt` on `final`, `aborted`, and `error`, which is the UI state transition needed to stop the stuck timer/spinner.
  - file: [ui/src/ui/controllers/chat.ts:470](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/ui/src/ui/controllers/chat.ts#L470)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '456,496p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Regression tests cover the stuck-streaming paths:** The Control UI tests explicitly assert that tracked BTW terminal finals do not tear down the active run and that deferred `session.message` reloads replay after terminal chat state clears the run.
  - file: [ui/src/ui/app-gateway.node.test.ts:653](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/ui/src/ui/app-gateway.node.test.ts#L653)
