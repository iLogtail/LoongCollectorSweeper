## Summary

Current `main` does not implement the requested out-of-order `message_start` handling, and PR #57853 still has an unresolved correctness concern around synthetic finalization of buffered assistant text with no dedicated regression test for the reported sequence. The linked bug report also has multiple user confirmations and workaround discussion, so this is not a conservative auto-close candidate.
