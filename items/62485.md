## Summary

Keep open. Current main still has the synchronous CLI banner config path that #62485 is trying to replace: help rendering calls the banner formatter, the banner formatter reads `cli.banner.taglineMode`, and that helper still statically imports the config barrel and calls `createConfigIO({ env }).loadConfig()` synchronously. The PR is therefore not implemented or superseded; it needs rebase/review/validation rather than cleanup closure.

## Best Possible Solution

Keep #62485 open. The best path is to rebase or rework the PR into current `main`, keep the heavy config read out of synchronous help rendering through a runtime boundary or equivalent lazy path, preserve configured `cli.banner.taglineMode` behavior for normal startup, and add focused coverage/manual proof for the help-only command path.
