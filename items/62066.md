## Summary

Current main only partially addresses the request. Slack has a shipped `channels.slack.thread.requireExplicitMention` control, and Discord has coverage for the later channel-bound-session no-mention repro, but the broader requested strict/explicit-only mention mode is still missing for Discord reply-to-bot and other implicit-reply channels such as Telegram, WhatsApp, Microsoft Teams, and ZaloUser.

## Best Possible Solution

Keep the issue open and implement a shared explicit-only mention policy for implicit-reply capable channels. A conservative path is to add a config-backed `requireExplicitMention`/`disableImplicitMention` equivalent at the relevant Discord guild/channel, Telegram group/topic, WhatsApp group, Microsoft Teams team/channel, and ZaloUser group scopes, then wire those settings into the existing `allowedImplicitMentionKinds: []` resolver path. Preserve Slack's shipped `channels.slack.thread.requireExplicitMention` as a backward-compatible alias or channel-specific surface.
