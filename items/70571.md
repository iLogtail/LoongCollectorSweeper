## Summary

Keep open. Current main does not implement compound `safeBins`; it still matches only the executable name and requires a safe-bin profile. The PR is trying to address a real open report (#59600), but it is not mergeable as-is because the proposed compound-prefix path bypasses existing safe-bin argv validation and also includes unrelated README churn. This needs maintainer/product judgment or a safer rework, not conservative cleanup closure.

## Best Possible Solution

Keep #70571 open for maintainer review or author rework, but do not merge the current branch. A safe path should either preserve the documented `safeBins` model as executable-name plus explicit argv profile and steer `gh`, `curl`, and `python3` use to host allowlist entries, or introduce a new explicit command/profile contract that validates every argument beyond the matched command prefix. The PR also needs the unrelated README rewrite dropped.
