## Summary

Keep open. Current main still does not implement compound `safeBins` matching, and the PR is a plausible attempt to fix linked issue #59600, but it is not ready to merge as submitted because the proposed branch would bypass the existing safe-bin argv/profile safety model and also contains unrelated README churn.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but require a narrower branch and a security-preserving design before merge. The likely path is either to close or reframe #59600 as documentation/product scope if compound commands should use explicit exec allowlist entries, or implement compound support only with exact argv semantics that preserve trusted-path checks, case-sensitive flag handling, and `validateSafeBinArgv`/profile enforcement for all remaining arguments.

## Evidence
