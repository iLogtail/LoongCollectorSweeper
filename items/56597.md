## Summary

Keep #56597 open. Current main still persists `/exec` session defaults and still resolves persisted session exec fields before agent/global exec config for later turns. The related closed #57896 may cover stale runtime config reloads, but it does not remove or invert this per-session exec override behavior.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. A fix should choose one explicit contract: make channel `/exec` overrides ephemeral, invalidate stored exec fields when global `tools.exec` changes, prefer global config over stored session exec fields, or add an operator-facing bulk/session reset for exec overrides. The chosen path should update `get-reply-exec-overrides`, session reset/patch behavior as needed, regression tests for Discord/external channel sessions, and the `/exec` docs.
