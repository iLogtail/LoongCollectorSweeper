## Summary

#9865 should remain open. Current main still does not provide first-class provider Batch API routing for background LLM turns in cron, heartbeat, sessions_send, or sessions_spawn/subagent execution. The only batch-provider support I found is scoped to memory embeddings/media-style mechanics, and the provided discussion already has a same-day maintainer/Codex review comment explicitly keeping this as the canonical tracker.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The likely maintainer path is an additive provider/runtime batch-turn contract with submit, poll, cancel, timeout, failure, usage, and reconciliation semantics, followed by explicit opt-in surfaces for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, plus public plugin SDK docs and focused regression coverage. Treat #49326, #70606, and #56126 as closed duplicate/follow-up context rather than replacements.
