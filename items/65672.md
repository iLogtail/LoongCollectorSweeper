## Summary

Current main already preserves OpenClaw/Pi compaction overrides after `DefaultResourceLoader.reload()` in both production embedded Pi reload paths. The same core bug was fixed by merged PR #67146 and is present in release `v2026.4.24`, so PR #65672 is now obsolete even though its root-cause analysis was valid.

## Best Possible Solution

Close #65672 as implemented on main by the merged #67146 fix. Keep the shipped per-call post-reload reapplication unless a future change introduces more reload call sites, at which point a small shared helper could be reconsidered.
