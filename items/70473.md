## Summary

Keep open. Current main still only extracts observed overflow counts from three provider message shapes and only forwards `currentTokenCount` to overflow compaction when that extraction succeeds. The PR’s broader MiniMax/OpenAI-compatible extraction and unparseable-overflow fallback are not implemented on main, and related PR #70479 is explicitly scoped to the later session-remap failure rather than this overflow-budget path.

## Best Possible Solution

Keep #70473 open, rebase it onto current main, retain the broader provider error parsing plus the minimally over-budget fallback, and run the targeted overflow helper/runner tests plus the changed gate before merge. Treat #70479 as the separate session-preservation follow-up rather than a replacement for this PR.
