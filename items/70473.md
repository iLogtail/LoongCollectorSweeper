## Summary

Keep #70473 open. Current main still has the exact overflow-budget gap the PR targets: confirmed overflow errors can be recognized while their attempted token counts remain unparsed, and overflow compaction only receives `currentTokenCount` when parsing succeeds. The related #70479/#70472 work tracks later session-preservation behavior, not this provider-error budget extraction fix.

## Best Possible Solution

Keep #70473 open as the focused overflow-budget extraction fix. Maintainers should review the final PR diff against current main, rerun the targeted agent tests/build, and handle #70479/#70472 separately for session-preservation behavior after compaction failure.
