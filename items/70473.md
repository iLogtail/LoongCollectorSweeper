## Summary

Keep open. Current main does not implement this PR's overflow-budget behavior: token extraction still recognizes only three older formats, and overflow compaction only receives `currentTokenCount` when that older extractor returns a value. The PR remains a focused, non-obsolete bug fix with tests/docs, and related PR #70479 is explicitly a separate follow-up for session preservation after compaction failures.

## Best Possible Solution

Keep #70473 open and either merge it after normal CI/review or apply the same focused changes on main: broaden `extractObservedOverflowTokenCount`, pass a synthetic minimally over-budget `currentTokenCount` for confirmed but unparseable overflow, and keep the regression tests/docs. Track the later session-preservation behavior separately in #70479/#70472.
