## Summary

Current main still does not implement automatic pruning or collapsing of consecutive assistant NO_REPLY transcript turns before model replay. The related PR #13388 was closed unmerged, and current code only covers adjacent mitigations: heartbeat-pair filtering, delivery/display suppression, tool-result pruning, image replay cleanup, and empty/error-turn handling.

## Best Possible Solution

Keep #13387 open. Implement a focused in-memory replay-context pruning pass near sanitizeSessionHistory, validateReplayTurns, or immediately after limitHistoryTurns that collapses consecutive assistant silent replies while preserving isolated legitimate NO_REPLY turns. Use the shared silent-token semantics where possible, and add regression coverage for text-block NO_REPLY, legacy string content, JSON/silent-token variants if supported, and empty-content silent stop turns. Only add config/docs if maintainers decide this should be configurable.
