## Summary

Keep PR #65364 open. Current main does not implement `api.registerProviderRuntimeAuthOverride`, and the existing provider-owned runtime-auth path only exchanges an already-resolved credential for the matched provider plugin. That does not solve the PR/linked issue’s central gap: an external plugin supplying runtime auth and request metadata for built-in providers without replacing their provider implementation.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to decide the generic plugin-SDK seam for runtime auth overrides, resolve the remaining review concerns, and add focused contract tests/docs if the API is accepted. If maintainers prefer a different API shape, keep #61910 as the canonical design tracker and supersede this PR with that narrower implementation rather than closing it as already implemented or ClawHub-only.
