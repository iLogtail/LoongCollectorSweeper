## Summary

Keep #53628 open. Current main still routes configured workspace paths through resolveUserPath/resolveHomeRelativePath, and that helper only trims, tilde-expands, and path.resolve()s the input; it does not expand ${XDG_CONFIG_HOME} or $VAR placeholders. The issue has related fix PRs, but the relevant ones are still open or closed unmerged, so the bug is not shipped as fixed on main.

## Best Possible Solution

Keep #53628 open as the canonical bug. The best fix is to add conservative ${VAR} and $VAR expansion to the shared path-resolution layer used by resolveUserPath/resolveHomeRelativePath, preserve existing tilde and unknown-placeholder behavior, add regression tests for ${XDG_CONFIG_HOME}/workspace in workspace/skill install resolution, then close or supersede the open duplicate PRs once the chosen patch lands.
