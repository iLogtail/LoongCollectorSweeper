## Summary

Keep #42122 open. Current main still runs the full `pnpm ui:build` during the root Dockerfile build, and the install-smoke workflow does not pass an opt-in UI-skip build arg. The PR is stale against the current workflow shape because main now builds one matrix-enabled smoke image and tags it as both root and extension smoke, but the underlying optimization is not implemented or superseded.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation to current main: add an opt-in Docker build arg scoped inside the `build` stage, pass it only from the current install-smoke Dockerfile build, preserve full UI assets for normal/release Docker builds, and validate with a real install-smoke Docker build or equivalent CI proof.
