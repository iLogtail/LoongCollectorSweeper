## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK downloads in GitHub releases: public Android docs say the app is not publicly released, the Android README remains alpha/local-build oriented, CI only validates a Play debug APK, the release helper builds signed AAB bundles rather than APK release assets, and the latest release assets are macOS-only. Related #35515 is closed back to #9443, while community APK/client links in the discussion are useful workarounds but do not satisfy the request for official OpenClaw APK release assets.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK/flavor policy, add tag or release automation that builds and uploads official APK assets to GitHub releases, update public Android install docs, and align release checks with the Android app readiness path. Continue pointing duplicate APK requests back to #9443 until that ships or maintainers explicitly decide against official APK distribution.
