## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK downloads in GitHub releases: the public Android docs say the app is not publicly released, the Android README is still alpha/local-build oriented, CI only validates a Play debug APK build, the Android release helper builds signed AAB bundles rather than APK release assets, and the latest release assets contain only macOS artifacts. Related #35515 was already closed as a duplicate pointing back to #9443, so this remains the canonical tracker.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK/flavor policy, add release automation that builds on tag or release and uploads APK assets to GitHub releases, update Android install docs, and add release checks that match the Android app readiness path. New duplicate APK requests should point back to #9443.
