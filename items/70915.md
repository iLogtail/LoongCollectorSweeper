## Summary

Keep PR #70915 open. Current main still registers the bundled ZAI image provider as a direct generic media-understanding wrapper, while ZAI config can point at Coding Plan base URLs and docs promise ZAI image understanding with no extra config. The linked bug #70886 and this PR’s diff identify a plausible remaining product bug; it is not implemented on main, not superseded, and not a ClawHub/core-scope close.

## Best Possible Solution

Keep the PR or the linked issue active. The preferred fix is to route ZAI media-understanding requests for vision models to the standard ZAI endpoint while preserving Coding Plan routing for text/code models, with a regression test covering `zai/glm-4.6v` under `zai-coding-global`/`zai-coding-cn`. If the PR is revised, address the request-scoped cfg/models.json race noted in review, likely by moving the routing decision into a provider-owned transport or model-resolution hook rather than cloning config per request.
