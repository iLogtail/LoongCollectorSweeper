## Summary

Current main still does not implement SKILL.md signature/hash verification for native skills. ClawHub skill install records only registry/slug/version/time metadata, native skill downloads are extracted and installed without comparing to trusted skill-version integrity metadata, runtime skill loading reads local SKILL.md files without verification state, and plugin archive verification explicitly excludes ClawHub skill-family packages.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity issue. A complete fix should add trusted skill-version hash/signature/publisher metadata to the ClawHub skill API, verify native skill downloads before extraction/install, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state on SkillEntry/status surfaces, and add maintainer-approved policy/docs/tests for unsigned or unverified skills and capability restrictions.
