## Summary

Keep open. Current `main` added broader transcript/runtime-context sanitization, but it does not implement the PR's actual change: non-assistant WebChat message extraction still does not strip `<relevant-memories>` blocks. That means the requested behavior is not proven implemented on `main`, especially for legacy or transient user-message transcripts.
