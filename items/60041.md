## Summary

Keep open. Current main has the narrower Twitch schema/setup/status improvements, but the reported runtime failure still appears present: Twitch startup can log a connection, return from `startAccount`, get marked stopped by the channel manager, and enter the auto-restart loop described in the issue.

## Best Possible Solution

Fix Twitch as a bundled channel bug, not as issue cleanup. The likely path is to make `extensions/twitch/src/plugin.ts` wrap `monitorTwitchProvider` with `runStoppablePassiveMonitor` or equivalent `waitUntilAbort` lifecycle handling, pass a `createAccountStatusSink` into the monitor, and add regression coverage proving Twitch `startAccount` stays pending until abort and does not trigger the channel manager auto-restart path after setup. While touching this, reuse the existing `probeTwitch` connection-event pattern or Twurple events to set truthful connected/liveness status and keep the existing config/docs improvements from #60033 as supporting work, not the final fix.
