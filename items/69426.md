## Summary

Keep #69426 open. Current main still gates `context1m` on provider IDs `anthropic` or `claude-cli`, and current tests explicitly preserve the non-Anthropic-provider fallback for `github-copilot` Claude model IDs. The newer shipped work appears to cover Claude CLI only, not the proxied-provider problem reported in #69353 and addressed by this PR.

## Best Possible Solution

Keep the PR open and either merge a focused version of it or replace it with a maintainer-owned patch that aligns runtime context budgeting with provider capability. The concrete path is to allow explicit `context1m: true` for Claude Opus/Sonnet 4 model IDs through verified proxy providers, add regression tests for `github-copilot` success and nonmatching-model fallback, and confirm the request transport behavior for each affected provider family.
