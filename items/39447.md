## Summary

Keep open. Current main still uses the same quickstart/global npm installer path implicated by the report, still defaults `SHARP_IGNORE_GLOBAL_LIBVIPS` to `1`, and has no visible OOM/swap recovery for a process killed during npm install. The discussion’s `install-cli.sh` workaround is documented and useful, but it is an alternative path rather than a fix for `install.sh`; the directly linked fix PR #39463 remains open/unmerged in the supplied context.

## Best Possible Solution

Keep #39447 open until #39463 or an equivalent main-branch change lands. The best fix likely belongs in `scripts/install.sh` and the matching installer docs: either avoid the low-memory sharp/npm failure by default, add explicit OOM detection/retry guidance or swap handling, or adjust the DigitalOcean quickstart order so 1 GB hosts have swap before the npm install path runs.
