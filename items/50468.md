## Summary

Keep open. Current main still passes `SKILL.md` glob patterns directly to chokidar while the repo is pinned to chokidar 5.0.0, whose supported upgrade path no longer treats watch inputs as globs. Existing-session refresh still depends on watcher events bumping the skills snapshot version, and the current unit test asserts the incompatible glob target shape instead of protecting against this bug. The related fix PR #50473 was closed unmerged in the provided GitHub context, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep the issue open and fix the core skills watcher. Replace direct glob targets with chokidar-compatible directory targets, or pre-expand file paths with a real glob API before calling chokidar; then filter events to `SKILL.md`, preserve FD-exhaustion safeguards such as bounded depth/ignored paths, and update `src/agents/skills/refresh.test.ts` with regression coverage for directory targets and non-SKILL.md event filtering.
