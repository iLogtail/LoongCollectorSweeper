## Summary

Keep open. Current main fixed the linked daily-reset path through `resolveAndPersistSessionFile`, but it did not implement this PR's gateway `sessions.reset` change. `performGatewaySessionReset` still creates a new `sessionId` while passing the previous generated `sessionFile` straight through, leaving a generated filename that no longer matches the new session id. The PR remains a focused fix, and its review-thread custom-filename concern was addressed by the author's second commit.

## Best Possible Solution

Keep #57126 open or land an equivalent narrow patch: reuse/export the existing generated-transcript rewrite helper in `performGatewaySessionReset`, preserve custom `sessionFile` names when no rewrite pattern applies, and add regression coverage for generated UUID, topic-suffix, fork-pattern, and custom-name reset cases.
