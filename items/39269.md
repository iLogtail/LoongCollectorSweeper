## Summary

Keep #39269 open. Current main still only partially overlaps this batch PR: Codex-prefixed error sanitization and subagent completion fallback handling exist, but the generic outbound delivery queue still lacks the Discord Unknown Channel/10003 permanent-error rule, chat.history still lacks a general delivery-mirror display filter, resolveAnnounceOrigin still drops webchat before merging persisted route state, and plugin duplicate-warning policy intentionally differs from the PR. The PR also has concrete unresolved review comments, so it is not a clean merge candidate as-is.

## Best Possible Solution

Keep this PR open only as a source of focused follow-up work, not as a batch merge. Split or reimplement the still-valid pieces: add a generic outbound queue Unknown Channel/10003 permanent-error rule with policy coverage if maintainers want it; add any delivery-mirror filtering in the shared chat display projection before max-message limiting; add a targeted webchat plus stale external route regression before changing resolveAnnounceOrigin; and keep the current duplicate-plugin warning policy unless a maintainer explicitly chooses quieter cross-origin diagnostics.
