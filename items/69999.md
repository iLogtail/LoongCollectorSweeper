## Summary

Keep open. Current main does not implement the macOS wallboard shell, `--wallboard` launch flag, Wallboard resources, or Wallboard smoke tests. The PR is recent, focused, and still potentially useful, but it should not be merged unchanged because review discussion identified a packaged-app resource lookup risk that current macOS code already avoids elsewhere.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If OpenClaw still wants a wall dashboard shell, rework this PR or a successor against current main, use the existing main-bundle-first resource lookup pattern, resolve the review comments, preserve focused macOS smoke tests, and add user-facing docs only once the wallboard entry point is accepted.
