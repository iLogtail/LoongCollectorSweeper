## Summary

Keep #62346 open. Current main still has the `incomplete-text -> prefill: true` helper behavior and matching test expectation, while the actual embedded-run recovery path now appears to use reasoning-only retry instructions instead of this helper. The PR is plausible but needs maintainer refinement rather than cleanup closure, especially because existing review comments flagged that the proposed whole-message drop can discard meaningful assistant content in mixed content-block cases.

## Best Possible Solution

Keep the PR open for maintainer-directed refinement. The best path is to first decide whether `sanitizeThinkingForRecovery` is dead legacy code and remove it/tests, or make it part of the active recovery contract. If the behavior is still wanted, update the patch to avoid `prefill: true` for reasoning-only/incomplete text while preserving meaningful assistant content in mixed-block messages, and add regression tests for signed-thinking-only, signed-thinking-plus-empty-text, and signed-thinking-plus-meaningful-text cases tied to #62322.
