## Summary

Keep open. Current main still does not implement the PR's central `worker-state.json` snapshot: the task registry has an observer seam and CLI JSON inspection, but no default observable-state observer, no `worker-state.json` writer, and docs still describe SQLite as the persistent task store. The PR also has unresolved review comments on its unrelated doctor `--json` changes, so it is not obsolete and not ready to auto-close as-is.

## Best Possible Solution

Keep the PR open for maintainer review. The clean path is to rebase and narrow it to the observable worker-state feature, wire a default task-registry observer that writes `worker-state.json` through existing atomic JSON helpers, keep restore/upsert/delete/error handling covered by tests, and split or drop the unrelated doctor `--json` changes unless they are fixed in a separate PR.
