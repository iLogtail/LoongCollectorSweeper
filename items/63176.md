## Summary

Keep open. Current main still does not expose `api.runtime.acp.spawn()` or `api.runtime.acp.prompt()` to plugins, and the request is a real plugin API gap rather than a ClawHub-only plugin idea. The PR discussion also includes a concrete downstream `task-dispatch` use case and follow-up validation. It does need maintainer API/security judgment, and the Discord snowflake normalization part is already present on main, so the PR likely needs a rebase/split rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, land a narrow plugin runtime API change that adds a lazy `api.runtime.acp` surface, an explicit default-off config gate, runtime mocks/types/tests, public docs/config reference updates, and channel-agnostic delivery target handling. Rebase or drop the already-landed Discord snowflake normalization before merge.
