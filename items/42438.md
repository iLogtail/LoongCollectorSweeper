## Summary

Keep open. Current main already supports the silent-drop half via `dmPolicy: "allowlist"`, but the requested owner notification for blocked/unknown WhatsApp DMs is still missing. The existing WhatsApp message hook is emitted only after access control accepts a message, so a ClawHub/plugin-only workaround would need a new pre-blocked-message channel/API seam first.
