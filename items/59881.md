## Summary

Current main still does not expose the provider-specific conversation/channel identifier in agent hook ctx.channelId. The issue's requested provider split is only partially represented: ctx.messageProvider is wired in several paths, but ctx.channelId is still sourced from the provider-level message channel/originating channel fallback rather than the Discord snowflake or equivalent conversation id. A meaningful requested capability remains missing, so this should stay open.
