## Summary

Keep open. Current main still skips bundled pre-install resolution for scoped specs like `@openclaw/feishu`, and the existing test suite explicitly asserts that old behavior. The PR is a focused direct-CLI fix that is not superseded by the related catalog-entry PR context.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase or recreate the narrow `resolveBundledInstallPlanBeforeNpm` change, preserve exact-match `npmSpec` behavior for scoped bundled specs, keep unmatched/versioned specs falling through, add the PR's match/no-match regression tests, and update `docs/cli/plugins.md` if maintainers accept scoped bundled specs as the default direct-CLI behavior.
