## Summary

Keep open. Current main has cron catch-up and health-monitor hardening, and the linked cron-run and Discord loop reports are now closed, but #51828 still contains a concrete current core gap: the cron retention sweep only considers cron-run session keys, while the linked #51767 subagent/ACP retention issue remains open and matches current code. The KeepMyClaw snapshot comment is useful external context/workaround for gateway-state recovery, but it does not make the OpenClaw core session-cleanup issue implemented or purely out-of-repo.

## Best Possible Solution

Keep #51828 open or manually narrow it around the remaining core session-retention/state-survival work. The concrete implementation path is to add a restart-safe completed subagent/ACP retention policy, document its relationship to `cron.sessionRetention` and generic `session.maintenance`, preserve active/main sessions, archive/delete transcripts safely, and add focused tests for subagent and ACP session-store rows. The external KeepMyClaw snapshot layer can be acknowledged as a useful ecosystem workaround, but it should not replace core cleanup semantics.
