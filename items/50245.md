## Summary

Keep open. Current main still exposes `durationMs` in the health JSON payload but does not print the gateway health snapshot duration in the default `openclaw health` text path. The linked PR #50250 is an exact open implementation attempt, but it is not merged, so the issue remains a valid tracker.

## Best Possible Solution

Keep #50245 open and land PR #50250 or an equivalent small patch that prints `Gateway probe duration: <n>ms` from `summary.durationMs` in the non-JSON `openclaw health` output, with a focused regression assertion in the health command tests. Leave `--json` unchanged.
