## Summary

Keep open. Current main has partial session-title plumbing in gateway/TUI code and supports manual labels through `sessions.patch`, but the user-facing `openclaw sessions` CLI still lists technical keys only, the docs still show key-only output, and there is no `/title` slash command. Related #55249 overlaps on labels/nicknames, but it does not supersede the full auto-title plus manual `/title` request here.

## Best Possible Solution

Keep this issue open. The best path is to wire existing session label/display-name/derived-title data into `openclaw sessions` text and JSON output, add a user-facing title or label command such as `/title` backed by `sessions.patch`, and decide whether automatic titles should be persisted or remain a bounded derived-title read from transcripts. Coordinate with #55249 for the narrower CLI label workflow instead of duplicating that implementation.
