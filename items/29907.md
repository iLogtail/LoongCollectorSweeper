## Summary

Keep #29907 open. Current main still supports only `systemEvent` and `agentTurn` cron payloads, so the requested no-LLM `exec`/`lobster` runner, condition DSL, and conditional `agentTurn` handoff are not implemented. The existing Lobster plugin is an agent tool, not a cron payload executor, and related #18160 only covers part of the direct-exec request while remaining open.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into a broader cron-execution issue. The best product path is an additive core cron/protocol feature with deterministic exec/script/Lobster execution, timeout and security policy, run logs/state, condition evaluation, silent no-op behavior, and conditional enqueue/delivery of an `agentTurn`, with CLI, Control UI, public docs, and protocol/runtime tests updated together.
