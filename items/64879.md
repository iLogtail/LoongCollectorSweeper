## Summary

Keep open. Current main has the mock parity workflow from the companion #64878 and adds providerVariant tagging, but the mock dispatcher still does not branch scenario plans by provider, and the live QA workflows do not run an OpenAI-vs-Anthropic parity-report lane. The specific #64879 gap remains actionable.

## Best Possible Solution

Keep #64879 open. The best path is to implement Option A in extensions/qa-lab/src/providers/mock-openai/server.ts by deriving a provider variant inside buildResponsesPayload and adding provider-specific planned-tool/prose branches for at least three agentic parity scenarios, with tests asserting divergent OpenAI vs Anthropic plans. If maintainers prefer Option B, add a separate secret-gated CI lane that runs the same parity pack against real openai/gpt-5.x and anthropic/claude-opus-4-6 summaries and feeds them into openclaw qa parity-report.
