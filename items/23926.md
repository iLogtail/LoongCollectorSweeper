## Summary

Keep open. Current main still has only partial plugin-level `before_install` coverage for gateway-backed skill dependency installs. The requested skill install lifecycle remains incomplete: no `skill:pre-install` / `skill:post-install` events, no post-install observational hook, and native ClawHub skill install/update still copies downloaded skill folders without the skill scan/hook path. The recent issue discussion already called this out, and current `main` still matches that assessment.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The likely product path is to extend the current `before_install` contract so native ClawHub skill install/update/sync also route through a blocking policy point, then add a documented post-install observational hook or record an explicit maintainer decision that those remaining events belong outside OpenClaw core.
