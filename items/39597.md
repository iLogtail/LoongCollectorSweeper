## Summary

Keep open. Current main still distinguishes Mac app and browser/Control UI at Gateway connect time, and a prior ClawSweeper comment already documented that useful partial implementation. The missing requested behavior remains: the agent runtime/trusted inbound context does not expose a trusted client type such as `client=macapp`; chat dispatch only maps the Gateway caller into sender metadata and scopes.

## Best Possible Solution

Keep #39597 open until core adds a trusted, normalized client-context path. The most direct implementation is to carry selected Gateway `connect.client` fields from `chat.send` into agent runtime/trusted inbound metadata, render an explicit stable field such as `client=openclaw-macos` or `client_type=macapp/browser`, and add focused tests for Mac app vs browser WebChat. Preserve the current trust boundary by avoiding raw/sensitive user-agent exposure unless it is deliberately sanitized or treated as untrusted.
