## Summary

Keep open. Current main already has Gateway client identity for browser/Control UI and native macOS connections, and chat.send carries the connected client id into sender/untrusted context. The requested trusted agent runtime distinction is still missing: RuntimeInfoInput and buildRuntimeLine have no client/client_mode/client_platform fields, so the system prompt runtime line still cannot distinguish Mac app vs browser WebChat as requested.

## Best Possible Solution

Keep #39597 open and implement a narrow trusted Gateway client-context path: sanitize client metadata from client.connect.client in chat.send, propagate it through auto-reply/followup run params into RuntimeInfoInput, render deterministic fields such as client=openclaw-macos, client_mode=ui, and client_platform=macos in buildRuntimeLine, and add Gateway chat plus system-prompt tests and WebChat/Control UI docs for the privacy-safe fields.

- Adding runtime prompt fields changes prompt bytes and cache behavior, so field ordering and tests need to be deterministic.
