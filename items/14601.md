## Summary

Current main still does not add `openrouter/moonshotai/kimi-k2.5` or `moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-picker catalog. The OpenRouter plugin supports dynamic/manual refs and now includes Kimi K2.6, but the requested K2.5 selector row remains absent, matching the prior issue comment’s keep-open conclusion.

## Best Possible Solution

Keep #14601 open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with matching docs and focused catalog/onboarding tests, or have maintainers explicitly document/comment that K2.6 supersedes K2.5 for OpenRouter onboarding/model selection.
