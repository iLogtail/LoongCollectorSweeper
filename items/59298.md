## Summary

Keep #59298 open. Current main does not have the proposed `openclaw update hint` subcommand or automatic update preambles in `.agents/skills`, and the existing gateway/status update hints do not fully solve the PR’s stated “agent sees it during skill use” workflow. The linked #59304 proposal is still open discussion, not a shipped replacement or clear superseding implementation.

## Best Possible Solution

Keep the PR open for maintainer judgment. If the feature is wanted, rebase it onto current main, decide whether the update hint belongs as a core updater CLI command or a ClawHub/plugin skill workflow, update all intended skill surfaces or centralize the preamble injection, and verify Claude Code skill-preamble behavior plus tests for cached JSON handling.
