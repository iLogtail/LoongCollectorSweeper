## Summary

Keep open. Current `main` only partially covers the request: per-channel queue mode already exists and per-channel debounce is now implemented, but queue `cap` and `drop` are still global-only settings.

## Evidence

- **Queue config schema only exposes per-channel mode and debounce:** `QueueConfig` defines `byChannel` for mode and `debounceMsByChannel` for per-channel debounce, but `cap` and `drop` remain single top-level fields with no per-channel override shape.
  - file: [src/config/types.messages.ts:13](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/types.messages.ts#L13)
  - command: `nl -ba src/config/types.messages.ts | sed -n '13,20p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Runtime resolution applies per-channel debounce only:** `resolveQueueSettings()` reads `queueCfg.byChannel` for mode and `queueCfg.debounceMsByChannel` for debounce, but resolves `cap` only from session/global values and `drop` only from session/global values.
  - file: [src/auto-reply/reply/queue/settings.ts:23](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/auto-reply/reply/queue/settings.ts#L23)
  - command: `nl -ba src/auto-reply/reply/queue/settings.ts | sed -n '23,58p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Validation schema has no per-channel cap/drop fields:** The Zod `QueueSchema` allows `byChannel`, `debounceMsByChannel`, `cap`, and `drop`, but there is no `capByChannel`, `dropByChannel`, or object-valued per-channel queue override schema.
  - file: [src/config/zod-schema.core.ts:674](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/zod-schema.core.ts#L674)
  - command: `nl -ba src/config/zod-schema.core.ts | sed -n '674,683p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Docs still describe cap/drop as global queue options:** Current queue docs show `byChannel` only for queue mode. They document `debounceMs`, `cap`, and `drop` as queue options, with no per-channel `cap` or `drop` override documented.
  - file: [docs/concepts/queue.md:45](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/concepts/queue.md#L45)
  - command: `nl -ba docs/concepts/queue.md | sed -n '45,70p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)

## Risks / Open Questions

- The report is partially obsolete because `messages.queue.debounceMsByChannel` now exists, so any future fix should narrow the issue to per-channel `cap` and `drop` (and possibly align the requested config shape with current schema conventions).
