## Summary

Keep open. Current main still has the #72019 duplicate task path: `buildSubagentSystemPrompt` embeds the task in the subagent system prompt, and `spawnSubagentDirect` also sends `[Subagent Task]: ${task}` as the first child user message. PR #72028 is a focused active fix attempt with useful regression coverage, but the discussion includes a concrete Aisle concern about moving task binding out of the system prompt for persistent sessions, and overlapping PR #72053 remains open rather than a landed canonical replacement.

## Best Possible Solution

Keep #72028 open for maintainer review while consolidating the #72019 fixes. The best outcome is one canonical implementation that keeps exactly one long-form task copy in the child model request, preserves multiline/structured task text, and resolves the task-binding tradeoff for persistent subagent sessions with regression tests covering both `message` and `extraSystemPrompt`.
