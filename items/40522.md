## Summary

Current `main` still routes `GET /` to a 404 when `gateway.controlUi.basePath` is set, so this PR remains a valid, narrowly scoped fix rather than stale cleanup.

## Evidence

- **Routing still rejects `/` outside the configured basePath:** With a configured Control UI basePath, `classifyControlUiRequest` returns `not-control-ui` for any pathname that is neither exactly `basePath` nor under `basePath/`. There is no special case for `pathname === "/"` on current `main`.
