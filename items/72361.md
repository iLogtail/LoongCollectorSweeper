## Summary

Keep PR #72361 open. Current main still has the reported `/allowlist --store` gap: the `configWrites` check only runs inside the config-update branch, while the store-only branch can call `updatePairingStoreAllowlist` directly. The linked issue #72360 describes a real access-control boundary, the docs say `/allowlist add|remove` honors `configWrites`, and pairing-store entries feed into effective DM command authorization. The PR is a focused fix with matching tests and positive review comments, not obsolete maintainer-cleanup material.

## Best Possible Solution

Keep #72361 open and review/merge or manually port its narrow fix: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and keep the channel-level false, account-level false, and allowed regression tests.
