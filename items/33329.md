## Summary

Keep #33329 open. Current main has per-provider runtime toggles for GitHub Copilot, Hugging Face, Ollama, and Bedrock, but public docs still only document the Bedrock discovery config path. The later request for a global disable-all model discovery switch also remains unimplemented: model planning still resolves implicit providers before applying merge/replace behavior, and the strict models config schema has no global discovery control.

## Best Possible Solution

Keep the issue open. Document the shipped per-provider toggle paths in the public provider/config docs: `plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, and `plugins.entries.ollama.config.discovery.enabled`, alongside the existing Bedrock documentation at https://docs.openclaw.ai/providers/bedrock. Then either add a typed global model-discovery disable control checked before `resolveImplicitProviders`, or split that global kill switch into a focused follow-up with clear interaction rules for `models.mode`.
