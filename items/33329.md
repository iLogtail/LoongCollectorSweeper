## Summary

Keep #33329 open. Current main implements provider-owned discovery toggles for the originally named Copilot, Hugging Face, Ollama, and Bedrock paths, with tests and manifest metadata, but the user-facing docs still document only Bedrock's config path. The later request for a global disable-all model discovery switch also remains unimplemented, and PR #33365 is closed unmerged rather than a canonical follow-up.

## Best Possible Solution

Keep the issue open until maintainers either document the shipped plugin-owned disable paths in the public docs, or explicitly decide that manifest UI hints are sufficient. The likely docs work belongs in `docs/gateway/config-tools.md` plus the affected provider pages for `plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, `plugins.entries.ollama.config.discovery.enabled`, and any newer model-discovery plugin toggles such as Codex. Separately, make a product call on the requested global disable-all model discovery switch; if accepted, add a typed config field in the model/provider discovery path and document how it composes with provider-specific controls.
