The PR is still coherent, but it needs to be refreshed for the current shared status overview pipeline and the unresolved global-vs-effective profile wording needs maintainer judgment.

## Best Possible Solution

The best path is to retarget the change through `StatusOverviewSurface` and the shared overview row builders, add focused tests for both status commands, and have a maintainer decide whether the row should show only the global baseline or a clearer effective-profile summary that accounts for agent/provider overrides.
