## Summary

Current main still does not implement this PR's requested `Global tools profile` overview row for `openclaw status` or `openclaw status --all`. The request remains coherent, and related docs/config-help PRs do not supersede the missing status output. The main open design issue is whether status should show only the global baseline or a more effective profile view that accounts for agent/provider overrides.

## Best Possible Solution

Keep this PR open until a maintainer decides the desired status surface. If accepted, implement the row in the shared status overview surface/row builder used by both `status` and `status --all`, add focused overview-row tests, and choose wording that clearly distinguishes global baseline from effective agent/provider policy. Coordinate with related docs/help text PRs #39954, #39963, and #40039 rather than treating them as replacements for this status change.
