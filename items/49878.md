## Summary

Keep open. Current main has partial heartbeat fixes, including preflight reads for empty-file gating and structured `tasks:` prompts, but the core stale-context problem remains actionable for ordinary `HEARTBEAT.md` checklist/bootstrap content: bootstrap files are still cached by session key and heartbeat runs still resolve bootstrap context through that cache. The related closed #60080 context describes a different heartbeat pollution issue and does not supersede this original cache-refresh report.

## Best Possible Solution

Keep #49878 open and fix the heartbeat/bootstrap refresh path. The narrow product/code direction is to make heartbeat turns either bypass the per-session bootstrap snapshot for `HEARTBEAT.md`, invalidate that snapshot when the file identity changes, or inject the freshly read `HEARTBEAT.md` content into each heartbeat prompt. Add a regression test that runs heartbeat once with old `HEARTBEAT.md`, edits the file, runs the same persistent session again, and proves only the new content is visible.
