## Summary

Keep open. Current main does not implement either acceptable resolution from #71710: `openclaw agent` has no top-level SIGTERM-to-abort-and-exit path for gateway or embedded runs, and the CLI docs do not document the `timeout -k` cron workaround. The related #66399 remains a process-supervisor/exec-tool issue, not a complete replacement for this cron-spawned `openclaw agent` entry-point bug.

## Best Possible Solution

Implement a bounded shutdown path for `openclaw agent`: install a top-level SIGTERM/SIGINT handler for the command, convert it into an AbortController passed through gateway calls, ACP, embedded PI runs, and CLI backend runs, stop progress/gateway clients, release session locks, and force exit after a short grace window. Also fix shared cleanup handlers so multiple OpenClaw signal listeners do not collectively suppress process termination. Until that ships, add user-facing CLI/cron docs recommending `timeout -k <delay> <duration>` or an equivalent supervisor escalation policy for cron-spawned agent runs.
