## Summary

Keep open. Current main does not implement the requested `openclaw agent` SIGTERM path or the proposed cron documentation fallback. The CLI command still sends gateway requests without a cancellation signal, the embedded path only consumes an `abortSignal` if a caller already supplied one, and the docs do not mention `timeout -k` or equivalent cron supervision guidance.

## Best Possible Solution

Keep this issue open. The best fix is to add a top-level `openclaw agent` shutdown controller for SIGTERM/SIGINT, pass its signal through both gateway and embedded execution, stop the Gateway client promptly, allow bounded drain/persistence, and exit with a signal-appropriate code. Until that ships, add `docs.openclaw.ai/cli/agent` guidance recommending `timeout -k <grace>` or an equivalent supervisor escalation policy for cron-spawned one-shot agents.
