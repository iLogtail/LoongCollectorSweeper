## Summary

Keep #38671 open. Current main still only models cron payloads as systemEvent or agentTurn, validates the gateway protocol against those two shapes, exposes only --system-event/--message in the cron CLI, and executes detached cron through runIsolatedAgentJob. The issue discussion describes a real zero-token shell mode and hybrid command->agent path that remains unshipped; related PR #62789 is an open implementation attempt, not merged or superseding evidence.

## Best Possible Solution

Keep #38671 open until core cron supports a direct command payload end to end, or maintainers explicitly reject that product direction. The likely path is to rework or land the backend/protocol pieces from #62789, add CLI/UI/docs coverage for pure command and command+message hybrid jobs, and align execution with existing exec security, timeout, output truncation, run-log, and delivery contracts.
