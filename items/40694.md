## Summary

Keep open. Current `main` does not implement the requested default of closing temporary browser tabs/windows when a primary agent finishes a task. Instead, primary-agent tabs are intentionally kept reusable and only cleaned up later by idle/excess sweeps; end-of-run browser cleanup is limited to subagents, cron, and explicit session teardown paths.

## Evidence

- **Browser docs describe different primary-session policy:** The user-facing browser docs explicitly say primary-agent sessions keep active tabs reusable and only close idle or excess tracked tabs in the background, while subagent/cron/ACP lifecycle cleanup closes tracked tabs at session end.
  - file: [docs/tools/browser.md:176](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/tools/browser.md#L176)
  - command: `nl -ba docs/tools/browser.md | sed -n '170,182p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Primary cleanup code is sweep-based, not task-completion-based:** `runTrackedBrowserTabCleanupOnce()` only sweeps tracked tabs for primary sessions using `idleMinutes` and `maxTabsPerSession`; it excludes subagent/cron/ACP sessions from that path and has no notion of 'task completed, now close temporary tab'.
  - file: [extensions/browser/src/browser/session-tab-cleanup.ts:29](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/extensions/browser/src/browser/session-tab-cleanup.ts#L29)

- **Default settings leave tabs around for long reuse:** The resolved cleanup config defaults to enabled, but the defaults are `idleMinutes = 120`, `maxTabsPerSession = 8`, and `sweepMinutes = 5`, which is materially different from closing browser-opened temporary tabs/windows when the task ends.
  - file: [extensions/browser/src/browser/constants.ts:7](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/extensions/browser/src/browser/constants.ts#L7)

- **Browser tool tracks opened tabs instead of auto-closing them:** The browser tool records opened tabs with `trackSessionBrowserTab(...)` after `open`, and only untracks on explicit `close`; there is no automatic close after a successful inspection/fetch/verification action in the primary-agent path.
  - file: [extensions/browser/src/browser-tool.ts:632](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/extensions/browser/src/browser-tool.ts#L632)
