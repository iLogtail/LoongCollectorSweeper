## Summary

Keep open. Current main still takes the single-store fallback for any literal `session.store` path, so the reported `sessions.list` omission remains plausible for the literal per-agent path case. The linked PR #54716 is directly relevant and includes reproduction/fix evidence, but it is open and unmerged in the provided context, so this issue should continue tracking the bug until that or an equivalent fix lands.

## Best Possible Solution

Keep #54435 open and land or adapt #54716. The fix should make `loadCombinedSessionStoreForGateway` treat literal paths matching the per-agent layout (`<root>/agents/<agentId>/sessions/sessions.json`) as discovery roots, while preserving the old single shared-store behavior for arbitrary literal paths. Add/keep a regression that exercises the real `sessions.list` path with main plus another agent store.
