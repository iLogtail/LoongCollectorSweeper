## Summary

Keep open. Current main has improved native-channel setup guidance and the relevant CLI/docs exist, but the shared exec-approval unavailable/timeout copy still does not include the central requested recovery hints: a discoverable Control UI URL or `openclaw dashboard --no-open`, and it does not point users to `openclaw approvals get` from the error itself. The linked fix PR #53347 was closed unmerged, so it is not a shipped implementation.

## Best Possible Solution

Keep the issue open and implement a scoped update to `src/infra/exec-approval-reply.ts` that adds concrete recovery steps to unavailable/timeout approval copy: use `openclaw dashboard --no-open` for the Control UI URL/token, point to `openclaw approvals get` or `openclaw exec-approvals get`, and preserve channel-specific native `execApprovals` setup guidance. PR #53347 is a useful prior attempt but was not merged.
