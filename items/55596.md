## Summary

Keep open. Current main still renders code-mode markdown tables using UTF-16 string length, so CJK cells are still padded as width 1 per character. The PR is a focused draft for a real shared formatter gap affecting Telegram and Discord; related #59562 is broader and points back to this PR rather than superseding it.

## Best Possible Solution

Keep the PR open and finish it or an equivalent fix in the shared markdown table renderer. The likely path is to add a small display-width helper for code-mode tables in `src/markdown/ir.ts`, cover CJK and emoji/grapheme cases in `src/markdown/ir.table-code.test.ts`, and let Telegram/Discord benefit through the existing shared `convertMarkdownTables` and `renderTelegramHtmlText` paths. Treat #59562's paragraph-spacing report as separate follow-up work.
