## Summary

Keep open. The issue reports a real runtime-snapshot failure mode, and current main still does not implement the discussed fix: `setRuntimeConfigSnapshot` stores the runtime snapshot verbatim, while Slack startup still resolves `botToken`/`appToken` from that active config path. I found no regression test for restoring Slack token redaction sentinels in `io.runtime-snapshot-write.test.ts`, and local history/search did not show a merged fix for this specific issue.

## Evidence
