## Summary

Keep open. Current main still requires `agents.defaults.cliBackends.<id>.command` in both the source Zod schema and generated JSON schema, so the reported partial-override bug is not implemented on main. The PR targets a real mismatch because the runtime merge already inherits base backend fields and `resolveCliBackendConfig` already rejects unresolved commandless results, but the submitted patch still needs generated schema sync and command-invariant typing across prepared runner paths before it is safely mergeable.

## Best Possible Solution

Keep the PR open and have it finish the focused bug fix: make `CliBackendSchema.command` and `CliBackendConfig.command` optional for user overrides, introduce or propagate a resolved backend type with non-empty `command`, update generated schema artifacts, and add a regression test for a `modelAliases`-only override inheriting the plugin-owned base command.
