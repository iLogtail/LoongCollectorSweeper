## Summary

Current `main` still preserves text that precedes a trailing `NO_REPLY`, so the reported behavior is not implemented there, and this PR also has unresolved review regressions around legitimate text-only replies and directive parsing.

## Evidence

- **Current normalization still strips trailing token and keeps remaining text:** `normalizeReplyPayload` on `main` strips `NO_REPLY` from non-exact matches and only suppresses when the stripped payload is empty; if text remains, it is returned to callers.
  - file: [src/auto-reply/reply/normalize-reply.ts:64](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/auto-reply/reply/normalize-reply.ts#L64)

  - file: [src/auto-reply/reply/reply-utils.test.ts:122](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/auto-reply/reply/reply-utils.test.ts#L122)
