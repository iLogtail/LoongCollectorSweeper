## Summary

Keep this PR open. Current main still rejects `channels.telegram.accounts.<id>.agentId` because the strict Telegram account schema does not include that field, and Telegram route resolution still only consumes topic-level `agentId` plus normal `bindings[]`. The PR remains a focused implementation candidate for the linked regression in #62985, and the author addressed the review follow-ups captured in the provided discussion.

## Best Possible Solution

Keep this PR open and review it as the active fix candidate for #62985. The desired path is to rebase/validate the branch on current main, keep explicit bindings and topic-specific `agentId` ahead of the account default, keep top-level `channels.telegram.agentId` rejected, and land the schema, type, routing, and regression-test coverage together.
