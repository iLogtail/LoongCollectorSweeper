## Summary

Keep open. Current main still does not document the Active Memory `config.cacheTtlMs` range or default in `docs/concepts/active-memory.md`, while the runtime and manifest still enforce the `1000-120000` ms bounds and `15000` ms default that this PR documents. The linked issue #65708 is closed, but this PR's specific docs improvement has not landed on main and remains useful.

## Best Possible Solution

Keep the PR open or recreate its tiny docs change on current main: document `config.cacheTtlMs` as `1000-120000 ms` with default `15000 ms` in `docs/concepts/active-memory.md`, matching the active-memory runtime clamp and plugin manifest. No core behavior change is needed unless maintainers choose to relax the schema instead.
