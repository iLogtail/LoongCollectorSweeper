## Summary

Keep open. Current main has not implemented this PR's central guard: `runReplyAgent` still drains `pendingToolTasks` with a bare `Promise.allSettled`, so a never-settling pending tool callback can still block the reply pipeline. The PR is tied to a plausible real bug and #53889 remains relevant, but the proposed hard 30s `Promise.race` also has an unresolved ordering concern for slow but healthy serialized tool-result delivery.

## Best Possible Solution

Keep the PR open but ask for a safer implementation: add a runner-level guard for orphaned pending tool tasks that does not advance while serialized `onToolResult` delivery is still making progress, and add targeted tests for both a never-settling task and slow healthy tool-result delivery ordering. If maintainers prefer a broader fix, keep #53889 as the canonical bug and supersede this PR with a tested main-side implementation.
