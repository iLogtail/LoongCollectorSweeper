## Summary

Keep #69548 open. Current main already has generic cron `staggerMs` support and a default 5-minute top-of-hour stagger, but the memory-core dreaming config surface still does not expose `dreaming.staggerMs`, the resolved dreaming config does not carry it, and the managed dreaming cron job builder never passes an explicit stagger into the cron schedule. The PR remains a focused implementation of a real missing memory-core config pass-through, though it should resolve or document the acknowledged unset/clear semantics before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the memory-core pass-through on top of the existing cron `staggerMs` contract, add focused resolver/reconciliation tests for positive values, `0`, blank strings, unset/default behavior, and manual/previous stagger drift behavior, and update the public dreaming/memory config docs if the setting is accepted.
