## Summary

Keep #69512 open. Current main does not forward `exec-approvals.json` allowlist policy into `claude-cli` sessions. The bundled Claude backend only maps OpenClaw YOLO policy (`security: "full"`, `ask: "off"`) to Claude Code `--permission-mode bypassPermissions`; for `allowlist`/`on-miss`, tests explicitly expect no Claude permission mode to be added. Native allowlist enforcement remains attached to OpenClaw-owned gateway/node exec paths, not Claude Code's internal exec path. The provided related PR #67666 is a merged Codex CLI resume sandbox fix and does not supersede this Claude CLI allowlist-forwarding request.

## Best Possible Solution

Keep the issue open and implement the policy bridge in the bundled Anthropic CLI backend or shared CLI runner: derive the effective per-agent exec approval policy, then enforce it through a Claude Code-supported permission hook/tool bridge or by disabling Claude's internal shell execution and routing command execution through OpenClaw MCP/native exec. Add focused tests for allowlist hit, allowlist miss, per-agent overrides, resume/live-session behavior, and docs for the supported Claude permission mode mapping.
