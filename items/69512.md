## Summary

Keep #69512 open. Current main has partial Claude CLI permission-mode hardening: OpenClaw only adds Claude `bypassPermissions` for YOLO exec policy, and safe/allowlist policy leaves Claude permission mode unset. That does not implement the requested forwarding of `exec-approvals.json` allowlist decisions into Claude Code internal exec. Native gateway/node exec still runs OpenClaw allowlist checks, but the Claude CLI runner still launches Claude with backend args and no OpenClaw exec-approval resolver in the CLI internal exec path.

## Best Possible Solution

Keep the issue open and implement a first-class Claude CLI permission bridge in the bundled Anthropic CLI backend/CLI runner. The bridge should map effective per-agent OpenClaw exec policy and `exec-approvals.json` allowlist state into Claude Code in a way that allows matches and denies misses, or route Claude permission requests through OpenClaw's existing allowlist/approval resolver. Add regression coverage for safe policy, explicit raw permission overrides, live stdio sessions, resume sessions, and docs in `docs/gateway/cli-backends.md` plus exec-approval docs.
