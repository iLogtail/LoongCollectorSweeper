## Summary

Keep open. Current main partially mitigates Claude CLI safety by deriving `bypassPermissions` only from OpenClaw's YOLO exec policy, but it does not forward or enforce `exec-approvals.json` allowlist entries for Claude Code internal exec. Explicit Claude `--permission-mode` overrides are still preserved, the Claude backend only allows OpenClaw MCP tools statically, and the native permission relay is currently Codex-only.

## Best Possible Solution

Keep this open and implement a Claude-specific permission bridge. The best path is either to translate the effective per-agent OpenClaw exec policy into Claude `--allowedTools`/`--disallowedTools` entries when safe, or wire Claude permission requests through an OpenClaw handler that evaluates `exec-approvals.json` with the same allowlist/deny behavior as gateway and node exec. The existing #70723 behavior should stay as a baseline mitigation, but it does not satisfy this issue by itself.
