## Summary

Keep #69512 open. Current main has Claude CLI policy hardening, but it still does not forward OpenClaw `exec-approvals.json` allowlist decisions into Claude Code exec handling. The code maps only YOLO exec policy to Claude `--permission-mode bypassPermissions`; for allowlist policy it deliberately leaves Claude permission mode unset and does not pass allowlist entries or an OpenClaw approval resolver into the CLI backend path.

## Best Possible Solution

Keep the issue open and implement a bundled Anthropic/Claude CLI permission bridge that translates OpenClaw effective exec policy plus `exec-approvals.json` allowlist matches into Claude Code permission decisions. The likely home is the `extensions/anthropic` CLI backend normalizer/adapter plus `src/agents/cli-runner` live-session handling, with regression coverage proving allowed commands pass, non-allowlisted commands deny, and existing YOLO `bypassPermissions` behavior is preserved.
