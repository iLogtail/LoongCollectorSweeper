## Summary

Keep #56545 open. Current main still has no `failureAlert.mode: "agent-turn"` implementation, and the PR is a focused attempt to solve open issue #56521. It should not be closed as implemented, duplicate, ClawHub, or non-actionable; the right path is maintainer review/revision because the provided PR discussion also identifies real completion gaps around CLI parsing, gateway protocol validation, scoped heartbeat wake, and `failureDestination` semantics.

## Best Possible Solution

Keep the PR open and request a complete core implementation or maintainer product decision. A complete path should add `agent-turn` consistently across cron types, config/zod validation, gateway protocol schema, CLI parser/help, agent tool schema, and any UI surface that edits failure alerts; implement the failure-alert runtime by enqueuing a system event and making a targeted heartbeat wake for the failing job's agent/session; explicitly decide whether `delivery.failureDestination.mode` supports `agent-turn` or rejects it; and add focused regression tests for CLI parsing, protocol validation, `emitFailureAlert`, gateway failure-destination behavior, and scoped wake targeting.
