## Summary

Keep #51350 open. Current main contains the Slack media pipeline pieces, but this issue is an audit/documentation task under the still-open Slack attachment vision epic, and the requested final behavior map is not present as a source doc, issue-resolution artifact, or full acceptance coverage on the finalized Slack `Media*` handoff.

## Best Possible Solution

Keep #51350 open and complete it as a focused audit under #51349: write the current behavior map for direct image, thread-root inherited image, multi-image, and forwarded/shared attachment cases; add or link acceptance tests that assert the finalized Slack `ctxPayload.Media*` fields; then let the remaining stabilization/docs/live-validation items close separately.
