## Summary

Keep open. Current `main` does not implement this PR's Linux `apt-get` fallback for `kind: "brew"` skill installs, and the proposed change still raises a real trust-boundary question for non-bundled skills, so there is not strong evidence for a conservative close.

## Evidence

- **Current main still fails immediately when brew is missing:** `installSkill()` returns `resolveBrewMissingFailure(spec)` as soon as a brew installer is requested and no Homebrew executable is found; there is no Linux `apt-get` fallback path on `main`.
  - file: [src/agents/skills-install.ts:507](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/skills-install.ts#L507)
  - command: `nl -ba src/agents/skills-install.ts | sed -n '460,560p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current main only documents manual Linux fallback:** `resolveBrewMissingFailure()` tells Linux users to install the formula manually with their system package manager. That matches the absence of any implemented automatic brew-to-apt behavior on `main`.
  - file: [src/agents/skills-install.ts:278](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/skills-install.ts#L278)
  - command: `nl -ba src/agents/skills-install.ts | sed -n '120,340p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Main explicitly treats non-bundled skill install metadata as attacker-controlled:** The current code warns that workspace/project/personal skill install metadata can be attacker-controlled and only emits a warning for non-bundled sources. Extending `kind: "brew"` to run `apt-get`/`sudo apt-get` would enlarge that privilege boundary, so the PR still needs maintainer judgment rather than cleanup closure.
  - file: [src/agents/skills-install.ts:466](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/skills-install.ts#L466)

## Risks / Open Questions

- Closing as implemented would be incorrect: `main` still returns the Homebrew-missing error path for brew skill installs on Linux.
- Closing as obsolete would discard a still-open product/security decision: whether OpenClaw should automatically elevate from skill metadata into `apt-get`/`sudo apt-get` for non-bundled skills.
