## Summary

Keep #38829 open. Current main still lacks a TUI path for cross-surface transcript updates: the Gateway exposes `sessions.messages.subscribe` and broadcasts `session.message`, and the Control UI handles that event, but `src/tui` still only routes `chat`, `chat.side_result`, and `agent` events and has no transcript subscription method or handler.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI change: add GatewayChatClient/TuiBackend methods for `sessions.messages.subscribe` and unsubscribe, subscribe to the active session on connect and session changes, handle `session.message` for the current session by appending or reloading history while preserving active local stream state, ignore other-session events, and add TUI tests for cross-surface delivery and session filtering. Coordinate with #41964 so any chat-event broadcast changes complement, rather than replace, the transcript subscription path.
