## Summary

Keep open. Current main still does not implement Control UI quote-reply/threaded-reply composition. The UI can display assistant-authored reply metadata, but users cannot choose an earlier message, show a quoted draft above the composer, or send reply-target metadata through `chat.send`.

## Best Possible Solution

Keep #55349 open. A good implementation would add a message-level Reply affordance in `ui/src/ui/chat/grouped-render.ts`, composer quoted-message state in `ui/src/ui/views/chat.ts`/chat controller state, additive `chat.send` reply-target metadata in the Gateway protocol, transcript rendering/history support for quoted previews, and focused UI/protocol tests. Coordinate with #7817 for broader project/thread routing, but do not collapse this request into that umbrella yet.
