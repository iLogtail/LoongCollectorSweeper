## Summary

Keep #69220 open. Current main still combines tagged Gemini reasoning-output selection with native Gemini thinking payloads/parts, so the reported conflict is not implemented away. The current fix needs to address both the direct `google-generative-ai` built-in fallback and the bundled Google `google-gemini` provider hook, not only the old dist constant.

## Best Possible Solution

Keep the issue open and fix the Google/Gemini reasoning-output contract in source: remove the direct `google-generative-ai` tagged fallback, change the `google-gemini` provider hook or Google provider registration so native Gemini thinking uses `native` output mode, update docs/tests that currently describe/assert tagged mode, and add regression coverage for native Gemini thought parts plus a tool call before final visible text so the run does not enforce `<final>` and collapse into `payloads=0`.
