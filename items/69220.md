## Summary

Keep #69220 open. Current main still configures direct Gemini as tagged reasoning while also sending native Google thinking payloads, and tests/docs explicitly preserve those two behaviors. The reporter’s and commenters’ failure description is therefore still plausible against current source rather than implemented, duplicate, ClawHub-only, or stale.

## Best Possible Solution

Keep the issue open for a provider-owner fix. The likely path is to remove or narrow the built-in/bundled tagged reasoning mode for direct Gemini when native Google `thinkingConfig` is active, preserve Gemini replay sanitation/tool handling, update `provider-utils` and `provider-model-shared` expectations, and add a regression covering native Gemini thinking plus a tool-call turn that must not require `<final>` text tags.
