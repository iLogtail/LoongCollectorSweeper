## Summary

Keep open. Current main still rejects `messages.groupChat.historyLimit: 0` through `GroupChatSchema`, the generated schema reflects that rejection, and the user-facing docs say `0` disables group history. The related fix PR #65359 is still open/unmerged, so the bug is not implemented on main yet.

## Best Possible Solution

Keep #65305 open until #65359 or an equivalent maintainer patch changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)`, updates/regenerates the generated schema if required, and adds a focused regression test proving `0` is accepted and negative values are rejected for both global and per-agent group chat config paths.
