## Summary

Current `main` still drops `identity.name` for gateway agent summaries when `agents.list[].name` is unset, and the PR’s added coverage is not present on `main`. The PR remains a valid, scoped fix rather than cleanup-closable noise.

## Evidence

- **Basic gateway list still uses only configured name:** `listGatewayAgentsBasic` on current `main` stores `name: normalizeOptionalString(entry.name)` with no fallback to `entry.identity?.name`, so an agent with only `identity.name` still produces an undefined summary name.
