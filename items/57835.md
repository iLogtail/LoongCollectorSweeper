## Summary

Keep open. Current main still does not copy `agents.list[].identity.name` into gateway agent summary `name` fields when `agents.list[].name` is unset. The linked Web UI issue is mitigated by UI-side label fallback, but this PR’s gateway/basic-summary behavior and regression tests remain absent on main.

## Best Possible Solution

Keep the PR open and have a maintainer review the API semantics. If the intended contract is that gateway summary `name` means the best display name, the best implementation is the PR’s narrow `configuredName ?? identity.name` fallback in both `listGatewayAgentsBasic` and `listAgentsForGateway`, with explicit configured `name` winning and regression tests for fallback, override, and missing-name cases.
