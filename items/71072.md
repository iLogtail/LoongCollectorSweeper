## Summary

Keep open. Current main preserves explicit empty plugin scopes semantically, but `loadOpenClawPlugins` still proceeds through cache lookup, plugin discovery, manifest registry loading, registry construction, and candidate iteration before returning an empty registry. The PR is a focused performance optimization that is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open. The best maintainer path is to add or require a `src/plugins/loader.test.ts` regression proving an explicit empty scope returns an empty registry without discovery, manifest parsing, or cache lookup, while preserving reentry protection and activation-state clearing for activating loads.
