## Summary

Keep open. Current main preserves explicit empty plugin-scope semantics, but `loadOpenClawPlugins` still performs the normal cache/reentry/Jiti/discovery/manifest path before filtering, so the proposed O(1) empty-scope fast path is not implemented on main. The PR's activation-planner motivation should be tightened because the current CLI planner path omits `onlyPluginIds` when no command owner is found, but explicit empty scopes still exist in runtime loader paths.

## Best Possible Solution

Keep the PR open. The best path is to add the empty-scope fast path in `loadOpenClawPlugins` with side-effect parity and the reentry guard preserved, plus a focused `src/plugins/loader.test.ts` regression that spies/mocks discovery and manifest loading to prove explicit `onlyPluginIds: []` returns an empty registry without doing filesystem/plugin discovery. The PR should also clarify that current CLI command planning falls back to full loading on no match, while explicit empty scopes from runtime/load callers are the applicable surface.
