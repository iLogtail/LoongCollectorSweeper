## Summary

Keep #53762 open. Current main still has the silent-drop control-flow hole this PR targets: a final answer can be buffered after skipped reasoning delivery, then the segment early-return exits without flushing it. The PR is useful, but its proposed flush path also needs to address the outstanding review concern about emitting the internal `message:sent` hook for buffered preview finalization.

## Best Possible Solution

Keep the PR open, but revise it before merge: add a final-segment flush before the `segments.length > 0` early return, make the buffered flush return or emit the `LaneDeliveryResult` so `message:sent` hooks fire for preview-finalized answers, and add regression coverage for both the skipped-reasoning final-answer delivery and the internal hook path.
