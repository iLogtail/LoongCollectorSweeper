## Summary

Keep #39141 open. Current main still lacks the prerequisite `sessions.activity` API and the requested opt-in session activity watchdog/alert engine. The existing channel health checks, diagnostics `session.stuck` signals, cron alerts, and heartbeat alerts are adjacent but do not provide session-scoped activity state, attention states, dedupe/cooldown policy, operator delivery/webhooks, or optional tokenized alert summaries.

## Best Possible Solution

Keep #39141 open until #39127 or an equivalent session-scoped activity API lands, then implement an opt-in watchdog that consumes that state and provides no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped delivery/webhooks, optional tokenized summaries, and focused tests/docs. Revisit plugin placement after the activity API is stable and plugin-accessible.
