## Summary

Keep #39141 open. Current main still lacks the prerequisite `sessions.activity` gateway API and the requested opt-in session activity watchdog/alert engine. The closest shipped pieces are channel-level health restarts plus diagnostics `session.stuck` events/OTel metrics, which do not provide session activity state, attention states, alert dedupe/cooldowns, operator delivery, webhooks, or optional tokenized summaries.

## Best Possible Solution

Keep #39141 open until the per-session activity API in #39127 lands or maintainers make an explicit scope decision. The best path is to implement `sessions.activity.get/list` plus WS activity events first, then add an opt-in watchdog that consumes that state, supports no-token structured alerts, dedupe/cooldown thresholds, scoped delivery/webhooks, optional tokenized summaries, and focused tests/docs. Do not close this as ClawHub yet because the missing core session activity API prevents a complete external implementation.
