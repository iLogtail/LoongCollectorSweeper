## Summary

Keep #39141 open. Current main still does not implement the prerequisite per-session activity/attention API from #39127 or the requested opt-in gateway-side watchdog/alert engine. Adjacent task audits, channel health monitors, cron failure alerts, and TUI/client watchdogs cover narrower surfaces and are not a substitute for session-scoped activity state and operator alerts.

## Best Possible Solution

Keep #39141 open until #39127, or an equivalent session-scoped activity/attention API, lands. Then implement an opt-in gateway watchdog or a plugin-accessible watchdog surface that consumes reliable session activity state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped channel/webhook/operator delivery, optional tokenized summaries, docs, and focused tests. Reuse existing task audit, cron failure alert, channel health, and subagent recovery code only where their narrower contracts fit.
