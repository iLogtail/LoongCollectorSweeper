## Summary

Keep #42391 open. The issue describes an external Feishu progress-card plugin that already works for send/update, but the requested core/plugin SDK seam is still missing on current main: `before_tool_call`, `after_tool_call`, and `agent_end` expose correlation/timing fields, not structured monotonic progress metrics. This is not a ClawHub-only close because the blocker is a missing core hook payload/API surface, and VISION.md explicitly keeps plugin API extension discussions in scope when a useful plugin cannot be built with current seams.
