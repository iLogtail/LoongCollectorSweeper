## Summary

Keep open. Current main improves the related `gateway.mode` diagnostic and has newer clobber/last-known-good recovery, but it still does not handle the specific `~/.openclaw/moltbot.json` upgrade gap reported in #54200. Active config resolution and Doctor legacy migration cover `openclaw.json` plus `clawdbot.json`, not `moltbot.json`, and the migration docs do not document the adjacent-file case from this issue.

## Best Possible Solution

Keep #54200 open for a narrow maintainer fix. Per repo guidance, avoid reviving load-time legacy auto-detection; instead add an explicit Doctor/startup diagnostic for `~/.openclaw/moltbot.json` when the canonical `openclaw.json` is missing or skeletal, offer a safe `doctor --fix` copy/merge path where feasible, and update the public migration/changelog wording so users with `moltbot.json` already under `~/.openclaw/` know what to do.
