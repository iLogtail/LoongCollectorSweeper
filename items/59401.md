## Summary

Current main does not implement `ignoreAtAll` or `ignoreBroadcast`, and Feishu still treats `@_all` as a bot mention before normal mention checks. The linked PR #59422 clearly tracks the request, but it is open, unmerged, and review comments identify a defect in its proposed implementation, so the issue should remain open until the behavior lands correctly.

## Evidence

- **Current main treats @all as a bot mention:** `checkBotMentioned` returns `true` as soon as message content contains `@_all`, before checking `event.message.mentions` for the actual bot open_id.
  - file: [extensions/feishu/src/bot-content.ts:236](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/extensions/feishu/src/bot-content.ts#L236)
