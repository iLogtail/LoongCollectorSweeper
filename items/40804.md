## Summary

Close PR #40804 as superseded. Current main still does not ship generic backup exclude patterns, but this PR targets the older inline backup implementation and its final head still has unresolved matcher/negation review defects. Open PR #44288 is the explicit successor for the same `--exclude` / `--exclude-file` user problem, touches the current `src/infra/backup-create.ts` architecture, and keeps the remaining work centralized with #40786.

## Best Possible Solution

Close #40804 and keep the remaining product work consolidated in open issue #40786 and successor PR #44288. The implementation that eventually lands should target the current `src/infra/backup-create.ts` path, use a dependency-backed gitignore matcher, protect restorable paths, and include archive-level tests rather than reviving this older inline `src/commands/backup.ts` patch.
