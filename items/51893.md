## Summary

Current `main` does not implement this fix: non-Azure custom onboarding still hardcodes `input: ["text"]`, and the tests/docs still describe that behavior. The underlying bug remains real, but this PR’s heuristic auto-enable approach is not conservative enough to close the item one way or the other, because prompt-image loading is gated directly by `model.input` and the review thread identified credible overmatch and config-mutation risks.

## Evidence

- **Current main still defaults non-Azure custom models to text-only:** In `applyCustomApiConfig()`, the non-Azure branch builds `nextModel` with `input: ["text"]` and no vision inference. That means `openclaw onboard --custom` on current `main` still disables image input for non-Azure custom models unless the user edits config manually.

## Risks / Open Questions

- The bug described by the linked issue is still present on current `main`: non-Azure custom model onboarding remains text-only by default, so image attachments stay disabled unless users edit config manually.
- Merging the PR as-is would rely on model-id heuristics for a security-relevant capability flag; that can misclassify text-only custom deployments and can also mutate existing configs on re-onboard.
