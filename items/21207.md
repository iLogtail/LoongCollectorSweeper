## Summary

Keep #21207 open. Current main has related partial work, including GPT-5 `parallel_tool_calls`, context-engine lifecycle hooks, pruning/compaction, subagents, and the experimental `update_plan` tracker, but it still does not implement the requested native batched read/write/search inputs, AI-callable named context block tools, or stateful plan-submit/step-done/status enforcement surface.

## Best Possible Solution

Keep #21207 open and split it into concrete tracks: native or gateway-level batching for core file/search tools; an explicit AI-callable context block API if maintainers want named save/load/unload semantics; and runner-owned plan state with opt-in strict gating if enforceable sequential execution is desired. Treat current `parallel_tool_calls`, context engine, pruning/compaction, and `update_plan` as useful partial groundwork, not closure.
