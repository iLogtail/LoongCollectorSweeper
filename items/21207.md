## Summary

Keep open. Current main has relevant partial work, including GPT-5 parallel tool-call defaults, context-engine lifecycle hooks, and `update_plan`/strict-agentic planning-only retry behavior. It still does not implement the issue’s requested native batched read/write/search interfaces, AI-callable context block save/load/unload/list tools, or stateful plan_submit/plan_step_done/plan_status enforcement.

## Best Possible Solution

Keep #21207 open and split it into concrete maintainer-owned tracks: native/gateway batching for core read/write/search surfaces, an explicit context block tool/API or context-engine extension if AI-controlled load/unload is desired, and runner-owned plan state with opt-in strict gating if maintainers want enforceable sequential plans. Treat the reporter’s batch-tools skill workaround plus current strict-agentic/context-engine work as useful partial evidence, not closure.
