## Summary

Keep open. Current `main` still has the mismatch the PR describes: update checks compare versions through `compareSemverStrings`, but that path still treats every `-suffix` as prerelease instead of treating numeric `-N` suffixes as OpenClaw revision publishes. The bug is still user-visible because the update banner and startup update state both depend on that comparator.

## Evidence

- **Update checks still use the generic semver comparator:** `compareSemverStrings()` in `src/infra/update-check.ts` still delegates directly to `parseComparableSemver(..., { normalizeLegacyDotBeta: true })` and `compareComparableSemver(...)`, so all update-status callers use the shared comparator path.
  - file: [src/infra/update-check.ts:371](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/infra/update-check.ts#L371)
