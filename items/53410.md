## Summary

Keep PR #53410 open. Current main has not implemented the PR's requested update-check behavior: the update path still delegates to the generic semver comparator, which treats a numeric `-N` suffix as a prerelease identifier rather than an OpenClaw correction revision. Separate config-version code already models numeric suffixes as revisions, but that logic is not used by `compareSemverStrings()` or the status/startup/update command paths affected by the PR.

## Best Possible Solution

Keep the PR open and ask for a small refresh rather than closing it. The best implementation is to make update-version comparison reuse or mirror `src/config/version.ts` revision-aware ordering for OpenClaw package versions, add the missing update-check regression cases from #53410, and consider hardening numeric suffix comparison against unsafe integer precision before landing.
