## Summary

Keep open. Current main sends Feishu audio as native `audio` messages and has a lower-level `duration` upload field, but the message-tool media path never computes or passes duration into `uploadFileFeishu`. Existing tests assert `file_type: "opus"` and `msg_type: "audio"`, not the duration field, so the reported Feishu UI duration problem is not implemented or superseded.

## Best Possible Solution

Implement this in the Feishu plugin media path: after loading or transcoding audio/video, derive duration with a dependency-backed method such as ffprobe/ffmpeg metadata, pass the duration to `uploadFileFeishu` for Feishu `opus`/`mp4` uploads in the units expected by the Lark API, and add regression tests that `message` tool sends for native `.opus` and transcoded voice audio include `duration` in `client.im.file.create`.
