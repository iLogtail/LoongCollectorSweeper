## Summary

Current main already fixes the user-visible stuck `streaming` TUI status via the shipped #67401 streaming watchdog. The watchdog resets the active run to `idle` after delta silence, existing tests cover reset/rearm/cancel behavior, and release `v2026.4.15` documents the fix. This PR's broader fallback is no longer needed and carries the non-active BTW/error status risk already noted in review.

## Best Possible Solution

Close this PR and keep the shipped #67401 watchdog implementation. It addresses the stuck `streaming` symptom without broadening non-active final/abort/error status updates; any remaining current-main stuck-status report should be opened as a fresh issue with event ordering logs and version details.
