## Summary

Keep #71396 open. Current main still has the Feishu mention-cascade behavior the PR is trying to remove: inbound `ctx.mentionTargets` are passed into the reply dispatcher, and the dispatcher still attaches them to the first outbound plain-text/card reply and streaming-card close. The linked issue #70065 being closed does not make the PR obsolete because the fix is not present on current main, and the PR also has unresolved review/security discussion that needs maintainer handling before merge.

## Best Possible Solution

Keep the PR open and have the Feishu plugin owner/maintainer finish the fix in the bundled Feishu plugin: stop unconditional automatic mention propagation for fallback/error/ack/ordinary replies, keep mention context safe for the agent, and address explicit-mention behavior with allowlisted structured conversion plus regression tests for plain text, card, streaming-card, and fallback paths.
