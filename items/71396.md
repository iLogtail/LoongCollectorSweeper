## Summary

Keep #71396 open. Current main at 8741a86f still forwards inbound Feishu mention targets into the reply dispatcher and still auto-applies them to first outbound text/card replies and streaming-card closes. The PR is not implemented on main, and the provided PR discussion includes unresolved security/review concerns that need maintainer handling before merge.

## Best Possible Solution

Keep #71396 open and have the Feishu plugin maintainer finish the bundled-plugin fix: stop unconditional automatic mention propagation for ordinary/fallback/error/ack replies, keep mention context safe for the agent, preserve or redesign explicit mention conversion with allowlisted structured handling, and add regression coverage for plain text, card, streaming-card, and fallback paths.
