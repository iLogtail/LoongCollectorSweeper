## Summary

Keep #71396 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still contains the Feishu mention-cascade behavior: inbound mentionTargets are extracted, passed into the reply dispatcher, and automatically converted into outbound Feishu mentions on first plain/card replies and streaming-card close. The PR is a focused, recent Feishu plugin fix for linked issue #70065, and the latest provided PR context shows the author pushed follow-up hardening after prior review feedback rather than abandoning the work.

## Best Possible Solution

Keep the PR open for Feishu plugin maintainer review. The best path is to finish and land a bundled Feishu fix that stops unconditional inbound mention propagation for fallback/error/ack/ordinary replies, keeps mention context safe and bounded for the agent, preserves any intended explicit-mention behavior with tests, and ships the regression coverage already proposed in #71396.
