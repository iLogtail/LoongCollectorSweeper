## Summary

Keep #71396 open. Current main still contains the Feishu automatic mention cascade that the PR is trying to remove: `ctx.mentionTargets` still flows into the reply dispatcher, and the dispatcher still turns those targets into outbound Feishu mention markup on first replies and streaming-card closes. The linked issue #70065 being closed does not make this PR obsolete, and the PR discussion also has unresolved card-mode explicit mention and prompt/mention safety concerns that need maintainer handling before merge.

## Best Possible Solution

Keep the PR open and have the Feishu plugin maintainer finish the bundled-plugin fix: stop unconditional propagation of inbound `mentionTargets` into fallback/error/ack/ordinary replies, keep inbound mentions as safe agent context, preserve explicit mention delivery through an allowlisted structured conversion path for text/card/streaming-card outputs, and keep regression coverage for those reply paths.
