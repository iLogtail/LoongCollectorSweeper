## Summary

Keep #33975 open. Current main covers the model-attribution half through `messages.responsePrefix` templates such as `{modelFull}` and `{provider}`, but it still does not implement the requested opt-in approval gate before moving from a failed primary model to a fallback model. The latest issue discussion explicitly confirms that partial coverage and reframes the remaining ask as a per-agent fallback-approval policy.

## Best Possible Solution

Keep #33975 open, or split it so this becomes the canonical fallback-approval tracker. Treat `messages.responsePrefix` with `{modelFull}` and `{provider}` as the shipped attribution solution, and implement the remaining feature as an opt-in per-agent fallback approval policy in the core model fallback path with schema/types, docs, and tests for approval, timeout, fail-fast/proceed, cancellation, and channel delivery behavior.
