## Summary

Keep #33975 open. Current main satisfies much of the model-attribution problem through `messages.responsePrefix` template variables, but it still has no opt-in approval gate before moving from the primary model to configured fallback models.

## Best Possible Solution

Keep this issue open until OpenClaw either implements a real opt-in fallback approval mode in the core model fallback path, with config schema/types/docs/tests and channel-appropriate approval UX, or a maintainer explicitly splits that remaining behavior into a narrower canonical follow-up. The already-shipped response-prefix attribution path should remain the model-visibility answer unless maintainers choose a dedicated `modelAttribution` config.
