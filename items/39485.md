## Summary

Keep PR #39485 open. Current main still lacks the requested Discord `accountId` to matching `agents.list[].id` fallback, and the linked issue #39428 remains the canonical open bug. The PR is useful but not merge-ready because review comments identify a real empty-agent-list routing regression and an observability issue with reusing `matchedBy: "binding.account"` for an implicit match.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer-owned fix that adds a narrow Discord `accountId` to existing-agent fallback after explicit binding tiers, skips the `default` account id, verifies the resolved candidate exists in `agents.list`, uses a distinct match reason such as `implicit.account`, and adds regression coverage for configured-agent matching, default-account behavior, and the empty-agent-list edge case. Keep #39428 as the canonical bug until the guarded behavior lands.
