## Summary

Keep PR #39485 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still lacks the requested Discord accountId-to-agentId fallback: the router evaluates explicit binding tiers and then falls to the default agent. The PR remains a focused fix attempt for open issue #39428, but the review comments identify real blockers that should be addressed before merge.

## Best Possible Solution

Keep this PR open, or supersede it with a maintainer-owned fix that adds a narrow Discord-only implicit accountId-to-existing-agent fallback after explicit binding tiers, skips accountId "default", requires the candidate to exist in agents.list, uses a distinct match reason such as "implicit.account", and adds regression coverage for positive matching, default-account behavior, and the empty-agent-list guard. Keep #39428 as the canonical bug until guarded behavior lands.
