## Summary

Current main still uses the reported absolute, above-trigger delete confirmation popover and has no viewport-aware vertical positioning. Related PR attempts exist, but the provided context says they were closed unmerged, so the user-facing bug is not implemented on main.

## Best Possible Solution

Implement the #50316-style fix on current main: measure the delete trigger/popover, render or position the confirmation relative to the viewport, flip below the trigger when there is insufficient room above, clamp horizontal/vertical bounds, and add focused regression coverage for top-edge and narrow-viewport cases. The issue can close after that behavior is present on main.
