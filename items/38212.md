## Summary

Keep open. Current `main` still appears to treat `codex-cli` as a separate provider in embedded/helper paths, while the Codex helper/model/auth/thinking hooks remain `openai-codex`-only, so the reported mismatch is not clearly fixed on `53aac30f5104052db56da2b26bcee0f4e5140325`.

## Evidence

- **Helper path forwards configured provider unchanged:** The bundled slug-generator helper parses the configured primary model and passes its provider/model straight into `runEmbeddedPiAgent()`. A primary model of `codex-cli/gpt-5.4` therefore reaches the embedded runner as `provider: "codex-cli"` rather than being rewritten first.
  - file: [src/hooks/llm-slug-generator.ts:59](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/hooks/llm-slug-generator.ts#L59)
  - command: `nl -ba src/hooks/llm-slug-generator.ts | sed -n '50,80p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Embedded model resolution does not alias codex-cli to openai-codex:** `resolveModel()` keeps the original provider and only normalizes the model id with `normalizeProviderId(provider)`. `normalizeProviderId()` has aliases for providers like `z.ai`, `kimi-code`, and `aws-bedrock`, but no `codex-cli -> openai-codex` mapping.
  - file: [src/agents/pi-embedded-runner/model.ts:701](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/agents/pi-embedded-runner/model.ts#L701)

- **Codex runtime hooks are still openai-codex-only:** The bundled OpenAI Codex provider registers dynamic-model and thinking hooks under `PROVIDER_ID = "openai-codex"`. The generic provider-hook matcher only recognizes `provider.id`, `aliases`, or `hookAliases`, and a search shows no `aliases`/`hookAliases` on `extensions/openai/openai-codex-provider.ts` while `extensions/openai/openai-provider.ts` does define Azure aliases.
  - file: [extensions/openai/openai-codex-provider.ts:466](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/openai/openai-codex-provider.ts#L466)
