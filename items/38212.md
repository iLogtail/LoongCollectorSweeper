## Summary

Keep #38212 open. Current main has adjacent progress for OpenAI Codex auth aliases and CLI-runtime routing in normal replies, but the reported session-memory slug helper still resolves the configured default model and calls the embedded runner directly with `codex-cli/gpt-5.4`. I do not see source or regression-test evidence that this helper now takes the normal `runCliAgent` path or normalizes the provider to `openai-codex` before embedded model resolution.

## Best Possible Solution

Keep #38212 open until the session-memory slug helper has a supported path for CLI-backed Codex defaults. A maintainer-quality fix should either route CLI-runtime primary models through the same `runCliAgent` execution path used by normal replies, or explicitly normalize this helper to a supported `openai-codex` embedded model/auth route when that is the intended compatibility behavior. Add focused regression coverage for `generateSlugViaLLM` with `codex-cli/gpt-5.4`, plus auth/profile alias coverage that proves OpenAI Codex OAuth credentials are selected without relying on the normal reply path.
