## Summary

Keep open. Current main contains some adjacent hardening for OpenAI Codex auth/profile aliases, but the exact #38212 helper path still appears unresolved: the session-memory slug generator passes the configured `codex-cli/*` ref into `runEmbeddedPiAgent`, and the embedded model resolver/harness selection path does not currently normalize `codex-cli` to `openai-codex` before model resolution. The related #38215 and #47074 fixes were closed unmerged, and #70743 addresses broader GPT-5.4 runtime/auth issues without source evidence that this exact `codex-cli/gpt-5.4` embedded resolution gap is fixed.
