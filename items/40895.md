## Summary

Current `main` no longer matches the issue's broad claim that all Discord threads stay 'active' after one mention: thread mention gating is re-evaluated per inbound message, and user-owned threads still require mentions when `requireMention` is enabled. But the code also intentionally bypasses mention gating for bot-owned `autoThread` threads and for bound thread sessions, which matches the follow-up comment describing `autoThread: true`. Because the issue discussion mixes those two behaviors and current code/docs do not clearly settle whether the bot-owned-thread behavior is a bug or intended session semantics, this is not strong enough for a conservative auto-close.
