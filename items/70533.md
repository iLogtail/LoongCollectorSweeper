## Summary

Keep #70533 open. Current `main` has moved several startup planning and read-only metadata paths onto the cold persisted plugin registry, but the runtime plugin load path still discovers `dist/extensions` broadly and builds a manifest registry for every discovered candidate before `plugins.allow` / `onlyPluginIds` prevents module import or registration. The issue’s central ask, discovery-time allowlist filtering before manifest reads/validation, is not implemented on the active `loadOpenClawPlugins` path.

## Best Possible Solution

Keep the issue open and fix the runtime plugin load path by applying a safe plugin-id scope before manifest reads where possible. In practice that likely means passing the gateway `onlyPluginIds` / normalized `plugins.allow` scope into discovery or manifest-registry candidate selection, using cheap candidate id hints or the persisted installed-plugin index to preserve duplicate precedence, config validation, explicit channel exceptions, and unknown-id diagnostics. Add regression coverage that spies on manifest reads for off-allowlist bundled plugins, not just module imports or runtime dependency installation.
