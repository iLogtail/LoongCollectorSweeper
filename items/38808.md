## Summary

Keep #38808 open. Current main still lacks the Feishu P2P thread reply behavior from this PR: direct messages with `thread_id` still get `replyInThread: false`, direct-message reply metadata is still stripped before sending, and tests only cover the group-thread path. The linked bug #38806 remains open while duplicate/replacement attempts #38814 and #38996 are closed unmerged, so #38808 is still the canonical useful PR rather than conservative cleanup material.

## Best Possible Solution

Keep #38808 open and either merge or adapt its narrow Feishu plugin fix: add a direct-thread guard for `isDirect && Boolean(ctx.threadId)`, target the thread root for P2P thread messages, preserve plain P2P quote-reply behavior, keep reply metadata through the dispatcher/send layer, and land the targeted P2P thread/plain-reply regression tests. Continue treating #38808 as the canonical fix for #38806 unless a maintainer opens or merges a newer replacement.
