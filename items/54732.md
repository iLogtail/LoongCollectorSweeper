## Summary

Keep open. Current `main` still scopes the `action` enum and channel-provided `schemaContributions`, but it does not trim the built-in message-tool sub-schemas by enabled/resolved actions. The reported gap remains in `src/agents/tools/message-tool.ts`.

## Evidence

- **Core schema props are still unconditional:** `buildMessageToolSchemaProps()` always spreads sticker, thread, event, moderation, gateway, channel-management, and presence schemas into the tool properties with no action-based gating.
  - file: [src/agents/tools/message-tool.ts:380](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/tools/message-tool.ts#L380)
  - command: `nl -ba src/agents/tools/message-tool.ts | sed -n '380,400p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Resolved actions only affect the enum:** `buildMessageToolSchemaFromActions()` uses the resolved `actions` only for `action: stringEnum(actions)`; it still calls `buildMessageToolSchemaProps(options)` without passing the action set, so the extra core properties remain present.
  - file: [src/agents/tools/message-tool.ts:403](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/tools/message-tool.ts#L403)
