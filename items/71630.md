## Summary

Keep #71630 open. Current main has local chat lifecycle signals, abort/queue controls, session lifecycle metadata, and a Talk-only composer status row, but it does not implement the requested compact Web UI composer runtime chip with In progress / Interrupted / Done semantics. The related #39127 and #52640 items are adjacent and still open; neither supersedes this narrower Web UI composer request.

## Best Possible Solution

Keep the issue open. The best product path is a core Control UI change: add a session-scoped runtime status model, render a compact accessible chip in the composer toolbar near the attach/mic/Talk controls, derive initial states from existing chat lifecycle events and reconnect/gap/shutdown handling, fail closed to Interrupted when continuity is uncertain, and later wire the chip to the gateway-owned activity API tracked in #39127 when that lands.
