## Summary

Keep #71630 open. Current main does not implement the requested compact Web UI composer runtime status chip. The existing composer has attach, voice, Talk, token count, and send/stop/queue controls, but no visible In progress/Interrupted/Done indicator. The related #39127 session activity API is still open and would be the right backing source; #52640 is a separate channel-status surface and does not supersede this Web UI-specific request.

## Best Possible Solution

Keep this issue open as a focused Control UI feature. Implement a compact, accessible composer status chip in ui/src/ui/views/chat.ts near the attachment/voice/Talk controls, add styling/tests for desktop and mobile layout stability, and back it with #39127’s session-scoped activity state once available. An interim implementation may use chatRunId/chat events, but it should explicitly fail closed on disconnect/reconnect uncertainty and avoid transcript messages.
