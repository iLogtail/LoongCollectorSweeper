## Summary

Keep #71630 open. Current main does not implement a compact Web UI composer runtime status indicator, and the broader gateway-owned per-session activity API it references (#39127) is still absent. Existing Web UI behavior only has local send/queue/stop controls, streaming state, compaction/fallback toasts, and a Talk-specific status row, which do not satisfy the requested session-scoped In progress / Interrupted / Done chip.

## Best Possible Solution

Keep the issue open and implement it in the Control UI after or alongside #39127: add a typed session-scoped activity source from the gateway, map active/completed/uncertain terminal states into a compact composer chip in ui/src/ui/views/chat.ts, style it without toolbar layout shifts, and cover send, final, abort, disconnect/reconnect, and mobile/accessibility behavior in focused UI and gateway tests.
