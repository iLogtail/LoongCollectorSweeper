## Summary

Keep PR #31172 open. Current main still hardcodes the Feishu streaming card update throttle at 160ms and does not expose or plumb a `streamingThrottleMs` config field through schema, manifest, docs, or dispatcher construction. The PR is not implemented on main or superseded; it needs maintainer review for the default value and a rebased, tested implementation.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased follow-up. A complete fix should add `streamingThrottleMs` to the Feishu config schema and `openclaw.plugin.json`, pass the resolved account value into `FeishuStreamingSession`, document it in the Feishu channel docs, and add focused tests for default behavior plus configured throttle plumbing. Maintainers should explicitly decide whether the default should preserve current main's 160ms or use the PR's proposed 100ms.
