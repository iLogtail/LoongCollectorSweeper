## Summary

Keep open. Current main still does not clearly preserve an in-flight Control UI chat run across a WebSocket reconnect: the browser client auto-reconnects, the gateway unsubscribes session events on socket close, and the UI explicitly clears orphaned chat run/stream state after reconnect because the final event was lost. Current health-monitor safeguards may reduce channel stale-socket restarts during active runs, but that is not the same as fixing the reported Control UI reconnect/session-termination behavior. The related #38888 context is useful but does not provide a canonical remaining tracker that supersedes this specific issue.
