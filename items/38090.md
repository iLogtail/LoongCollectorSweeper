## Summary

Current `main` does not provide strong evidence that #38090 is fixed. The reconnect path still auto-reconnects on ordinary closes and then clears the active chat run state locally, while the only reconnect-related change I found since `v2026.4.22` is a narrower fix for Stop-button aborts during a disconnect, not for unsolicited reconnects terminating in-flight runs.
