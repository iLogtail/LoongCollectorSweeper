## Summary

Keep open. Current main still does not implement the requested user-facing `plugins.entries.memory-core.config.dreaming.model` override: the memory-core manifest rejects that top-level key, the host resolver does not read it, and Dream Diary narrative subagent calls do not pass a model override. PR #69209 still addresses real remaining work, though it should be consolidated with the related duplicate #66583 before landing.

## Best Possible Solution

Keep the item open and land one canonical implementation of `plugins.entries.memory-core.config.dreaming.model`: add the manifest/schema/UI hint, resolve the top-level model into dreaming execution defaults while preserving phase overrides, pass it to light/REM/deep Dream Diary subagent runs, retry only appropriate configured-model failures with the session default, and cover the behavior in host and memory-core tests plus user-facing docs/changelog. Then close or supersede the duplicate PR #66583 explicitly.
