## Summary

Keep open. Current main still prepends inbound metadata to channel-triggered prompts in `src/auto-reply/reply/get-reply-run.ts`, and `src/auto-reply/reply/inbound-meta.ts` still unconditionally emits `Conversation info` / `Sender` untrusted metadata blocks when values are present. I found config knobs for context/history scope, but no shipped toggle to disable or minimize this metadata injection for trusted DMs or senders.
