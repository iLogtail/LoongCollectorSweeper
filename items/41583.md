## Summary

Keep #41583 open. Current main still injects inbound channel metadata into both the system prompt and user prompt path, including direct external-channel chats. The available channel config only covers supplemental context visibility, not a toggle to disable or minimize the metadata envelopes requested here.

## Best Possible Solution

Add a conservative channel configuration surface, likely under `channels.defaults` with per-channel and possibly per-DM overrides, that can minimize or suppress user-visible inbound metadata only for explicitly trusted contexts while preserving the current default and the trusted system metadata needed for safety. Cover Telegram/Discord DMs, direct external-channel prompts, transcript sanitization, and docs.

## Risks / Open Questions

- Disabling metadata can weaken sender verification and prompt-injection defenses, so any implementation should default to current safe behavior and be scoped carefully to trusted DMs or explicit allowlisted senders.
- The prompt/transcript sanitizers rely on existing metadata sentinels, so a minimization mode should be tested across reply generation, transcript display, and copied-metadata stripping.
