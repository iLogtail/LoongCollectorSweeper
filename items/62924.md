## Summary

Keep open. Current `main` still does not expose the chosen media-understanding provider/model inside the inbound `[Image]/[Audio]/[Video]` body that the assistant reads. The runtime records provider/model in media decision metadata and `/status`, but the agent-visible body remains `User text` plus `Description`/`Transcript` only, so the reported gap is still present upstream.

## Evidence

- **Inbound media body formatter omits provider/model fields:** `formatMediaUnderstandingBody()` builds sections from `User text`, `Description`, and `Transcript` only. It never reads `output.provider` or `output.model`, so no `Analysis model:` line is emitted into the agent-visible body.

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)

The chosen model is not injected into `ctx.Body` before agent processing.
