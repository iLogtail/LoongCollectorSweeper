## Summary

Keep #54386 open. Current main still requires `--path` for `skill-creator` initialization and still documents relative output paths, while OpenClaw loads user skills from the active workspace `skills/` directory. The PR addresses a real product/data-loss boundary, but its proposed default needs maintainer-quality alignment with the existing workspace resolver, especially `OPENCLAW_HOME` and non-default `OPENCLAW_PROFILE` behavior noted in review comments.

## Best Possible Solution

Keep the PR open and update it rather than closing it. The best fix is to make `--path` optional, derive the default from the same effective workspace rules OpenClaw uses in `src/agents/workspace-default.ts`, expand user paths consistently, update the bundled `skill-creator` instructions to avoid relative-path examples for persistent user skills, and add focused tests for default, `OPENCLAW_HOME`, `OPENCLAW_PROFILE`, and explicit custom path behavior.
