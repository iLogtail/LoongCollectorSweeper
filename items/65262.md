## Summary

Keep open. Current main still returns the old misleading `plugins.allow` diagnostic for `openclaw commands` when `plugins.allow` is non-empty, because `/commands` is a built-in chat command but is not represented in the plugin command-alias metadata used by `resolveMissingPluginCommandMessage`. The PR is not obsolete, though it should be cleaned up before merge because the latest PR discussion identified a brittle string-matching control-flow issue and the branch includes unrelated ACP test changes.

## Best Possible Solution

Keep the PR open or replace it with a narrower maintainer patch that introduces a structured missing-command diagnostic for built-in runtime slash commands, preserves plugin allow/disabled diagnostics for real plugin-owned aliases, keeps Commander unknown-command UX, and adds focused regression tests for `openclaw commands` with and without `plugins.allow`. Drop the unrelated ACP test changes from this PR before landing.
