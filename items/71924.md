## Summary

Keep PR #71924 open. Current main still has the reported GitHub Copilot behavior: provider discovery returns an empty model catalog, so GPT-5.5 falls through to synthetic dynamic metadata rather than `/models` capability metadata. The related issue #71936 was already closed as superseded by this PR, making this PR the active implementation tracker rather than an obsolete item.

## Best Possible Solution

Keep #71924 open as the canonical implementation path for #71936, or replace it with an equivalent provider-owned fix in `extensions/github-copilot/index.ts`. Before landing, address the mocked-fetch test issue, verify the current Copilot `/models` response shape, and update the catalog/discovery tests that currently lock in `models: []`.
