## Summary

Current main still has the gap PR #71924 addresses. The GitHub Copilot provider catalog returns an empty model list, unknown `github-copilot/gpt-5.5` would fall through to the synthetic 128k/8192 defaults, and `gpt-5.5` is not in the Copilot xhigh thinking list. This is a focused bundled-provider fix, not a cleanup close candidate.

## Best Possible Solution

Keep PR #71924 open. The best path is to land a provider-owned GitHub Copilot catalog fix that maps Copilot `/models` capability fields into `ModelDefinitionConfig`, adds `gpt-5.5` xhigh thinking support, and includes fully mocked tests that do not rely on live network fallback behavior.
