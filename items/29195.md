## Summary

Keep #29195 open. Current main documents ACPX permission configuration and has partial ACP improvements, but the central failure mode remains: ACPX still defaults to approve-reads plus fail for non-interactive permission prompts, and docs still describe permission failures and indefinite stalls as active troubleshooting cases rather than fully fixed behavior.

## Best Possible Solution

Keep #29195 open or split it into focused follow-ups for the remaining ACPX/Core work: decide the non-interactive permission default or preflight policy, surface ACP permission/config failures back to the spawning session, preserve an agent-callable status/progress path for owned ACP children, and replace the documented manual stale-process cleanup path with automatic timeout/reporting semantics. Keep the shipped docs, /acp status, background task, and runTimeoutSeconds improvements.
