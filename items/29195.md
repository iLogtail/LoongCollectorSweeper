## Summary

Keep #29195 open. Current main still preserves the ACPX non-interactive permission footgun, still blocks owned cross-agent ACP child visibility under default tree visibility, and still documents manual stale-process cleanup. Docs and task/status/timeout plumbing are real partial improvements, but they do not close the umbrella issue.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability issue. The best path is to decide a safe default or preflight policy for ACPX permissions, surface permission/config failures with actionable requester-facing messages, allow spawning agents to inspect their own ACP children without global visibility, and replace manual stale-process cleanup with automatic timeout/reporting semantics while preserving the shipped docs, /acp status, background tasks, and runTimeoutSeconds improvements.
