## Summary

Keep open. Current main does not implement the PR's opt-in `parentUpdates:"notify"` API or its ACP-through-subagent-announce completion path. The existing ACP parent relay still uses system events plus heartbeat wake, and the related ACP parent-wake gap remains represented by open follow-up context (#70264/#52249).

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best path is to decide the ACP parent-completion product contract alongside #70264/#52249, then either rebase this opt-in `parentUpdates` approach onto current main with schema/docs/tests and ACP internal-event-source support, or close it only after an equivalent merged fix provides the same parent wake/announce behavior.
