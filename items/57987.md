## Summary

Current `main` does not implement PR #57987’s OpenRouter conversation-history cache behavior. Verified OpenRouter Anthropic requests still use the legacy marker helper that only tags system/developer blocks, current OpenRouter tests still assert unchanged user turns, and the docs still describe OpenRouter cache markers as system/developer-only. The requested trailing-user-turn cache marker exists on the separate direct Anthropic payload-policy path, which confirms the gap is still specific to OpenRouter.
