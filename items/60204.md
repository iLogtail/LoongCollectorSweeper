## Summary

Keep #60204 open. Current main has related gateway service and legacy cron-store cleanup, but not the reported legacy user-crontab `ensure-whatsapp.sh` health-check problem. The existing scanner is limited to launchd/systemd/schtasks gateway services, and the cron doctor only normalizes OpenClaw's persisted cron job store. The related #58129 DBus service issue and #60212 cron-output PR are distinct and do not retire or detect this legacy health script path.

## Best Possible Solution

Keep the issue open and implement a narrow diagnostic/repair path: have `openclaw doctor` or gateway health diagnostics inspect the user crontab for legacy invocations of `ensure-whatsapp.sh`, warn that the check is unsupported/stale, and optionally provide a safe removal or migration path. If the script remains intentionally supported, its systemctl failure handling should distinguish user-bus unavailability from a genuinely inactive gateway.
