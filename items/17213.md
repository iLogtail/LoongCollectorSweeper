## Summary

Keep #17213 open. Current main has narrower reliability work, including strict-agentic GPT-5 plan-only retries, commentary-phase suppression, and lifecycle end/error fallback, but it still does not implement the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers across shared agent/gateway surfaces. The related #40710 execution-intent gate PR is closed unmerged, and #55721 is a narrower shipped tool-dispatch/incomplete-turn fix rather than a canonical replacement for this core lifecycle contract.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved, backward-compatible lifecycle contract that adds explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across shared AgentEvent types, Gateway protocol/schema, agent emitters, harness/plugin SDK surfaces, chat/session finalization, clients, docs, and regression tests. Treat #40710 and #55721 as useful prior art/narrow fixes, not as superseding this broader issue.
