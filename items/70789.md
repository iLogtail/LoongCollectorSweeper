## Summary

Current main still has the reverse `findLatestCard` pairing logic that this PR targets, and the existing regression coverage only exercises interleaved call/result ordering, not the sequential same-name call-call-result-result shape described in the PR and linked issue. The PR remains a focused candidate fix, but it should stay open for maintainer review and a stronger regression test.

## Best Possible Solution

Keep the PR open. The best path is to land a FIFO/two-pass pairing fix in `ui/src/ui/chat/tool-cards.ts` that prioritizes exact tool-call IDs over name fallback, and add a focused `extractToolCards` regression test for sequential same-name calls with results arriving afterward, including the exact-ID preemption case.
