## Summary

Keep PR #70789 open. Current main still contains the reverse newest-first `findLatestCard` result-pairing logic that the PR targets, and the Control UI expanded-card path only renders a tool output when the extracted card has `outputText`. The existing regression coverage on main covers exact-id and interleaved same-name results, but not the sequential call-call-result-result ordering described by the PR and linked issue #70746.

## Best Possible Solution

Keep the PR open and review it as a plausible focused Control UI bug fix. The best path is to ensure the PR includes a regression test for sequential same-name calls where calls are emitted before results, then merge or request adjustments based on normal maintainer review and CI.
