## Summary

Keep PR #65415 open. Current main partially mitigates the original 10s subagent-spawn timeout path with hardcoded 60s/derived timeouts, but it does not implement the requested `agents.defaults.subagents.gatewayTimeoutMs` config surface, and several related subagent gateway lifecycle/control paths still use fixed 10s timeouts. The PR discussion is useful, especially the Greptile review noting the configured floor can lower current defaults, so the branch should be reworked rather than auto-closed as already implemented.

## Best Possible Solution

Rework the PR against current main: decide whether OpenClaw should expose `agents.defaults.subagents.gatewayTimeoutMs`; if yes, implement it as a documented config/schema/generated-schema/help/docs surface, preserve current built-in minimums so configuration can raise rather than lower them, route the relevant subagent spawn/control/announce/orphan/sweep gateway calls through a shared timeout helper, and add focused tests for default floors, configured higher floors, and non-regression of the existing 60s behavior. If maintainers prefer the current hardcoded 60s/derived policy, close this manually with that product decision rather than via automated cleanup.
