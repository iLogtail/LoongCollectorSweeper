## Summary

Keep PR #65415 open. Current main has partly superseded the original 10s native sessions_spawn failure mode by using 60s/60s-to-300s timeouts on the main subagent spawn path, but it does not implement the requested configurable `agents.defaults.subagents.gatewayTimeoutMs` surface and still has subagent-adjacent gateway calls with 10s budgets. The PR likely needs a rebase/redesign rather than auto-close.

## Best Possible Solution

Keep the PR open, but require a rebase onto current main and a narrower maintainer decision: either add a real `agents.defaults.subagents.gatewayTimeoutMs` contract that only raises current defaults and covers all intended subagent-owned gateway lifecycle paths, or explicitly decline the config knob because current main's 60s/300s native spawn behavior is sufficient. Any accepted config surface should update config types, Zod schema, generated schema/docs, and focused tests without per-test heavy module resets.
