## Summary

Keep open. Current main still has the subagent gateway calls routed through `callSubagentGateway` with hardcoded 10s RPC budgets, and the proposed `agents.defaults.subagents.gatewayTimeoutMs` config key is not present in the config types, Zod schema, generated schema, or docs. This PR addresses a real remaining reliability gap rather than something already shipped or clearly superseded.

## Best Possible Solution

Keep the PR open and ask for a small update against current main: implement the timeout floor in the centralized `callSubagentGateway` path, preserve the per-method defaults as minimums, add `agents.defaults.subagents.gatewayTimeoutMs` to types/Zod/generated schema and user docs, and keep targeted tests for default floors plus configured overrides. The related timeout reports in the supplied context track handshake/status/announce/startup concerns; they do not supersede this specific subagent spawn/lifecycle RPC timeout gap.
