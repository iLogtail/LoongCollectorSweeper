## Summary

Keep open. Current `main` has partial mitigation for the reported stop-path regression, but there is not strong evidence that the underlying long-task Control UI disconnect is fixed, and the current UI still drops local run tracking on reconnect, which can remove the Stop control after a reconnect/reload.

## Evidence

- **Docs still define Stop and `/stop` as the expected out-of-band abort path:** The Control UI docs say Stop should call `chat.abort`, `/stop` should abort out-of-band, and `chat.abort` supports `{ sessionKey }` without a `runId` to abort all active runs for the session.
  - file: [docs/web/control-ui.md:180](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/docs/web/control-ui.md#L180)
  - command: `nl -ba docs/web/control-ui.md | sed -n '180,186p'`
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Reconnect currently clears the local active-run state:** On `hello` after reconnect, the Control UI processes any queued abort, then unconditionally clears `host.chatRunId` and `chatStream`. That means the browser no longer considers the run active after reconnect.
  - file: [ui/src/ui/app-gateway.ts:305](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/ui/src/ui/app-gateway.ts#L305)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '250,380p'`
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Stop button rendering still depends only on local `chatRunId`:** The chat view passes `canAbort: Boolean(state.chatRunId)`, and the run controls only render the Stop button when `canAbort` is true. Once reconnect clears `chatRunId`, the Stop button disappears.
  - file: [ui/src/ui/app-render.ts:2288](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/ui/src/ui/app-render.ts#L2288)

  - file: [ui/src/ui/controllers/chat.ts:640](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/ui/src/ui/controllers/chat.ts#L640)
