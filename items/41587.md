## Summary

Keep open. Current `main` still watches only the single active config file path, so edits to `$include`-referenced files do not have evidence of triggering config hot reload. The codebase has include-path scanning utilities, but they are used in security flows, not the gateway reload watcher.

## Evidence

- **Gateway reloader watches one path:** `startGatewayConfigReloader` creates a chokidar watcher on `opts.watchPath` only, and the write-notification fast path also filters on `event.configPath === opts.watchPath`. There is no watcher expansion to included files.
  - file: [src/gateway/config-reload.ts:382](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/gateway/config-reload.ts#L382)

  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Server passes only root config path into reloader:** Gateway startup wires the reloader with `watchPath: configSnapshot.path`, i.e. the single active config file path rather than an include set or watched directory.
  - file: [src/gateway/server.impl.ts:864](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/gateway/server.impl.ts#L864)
