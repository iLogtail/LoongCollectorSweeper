## Summary

Current `main` no longer uses Slack’s broad `contract-api.js` surface for config-doctor rule loading. The config-read path now resolves `doctor-contract-api*` first, Slack ships that dedicated doctor artifact, and the shipped `v2026.4.23` release contains the same behavior plus tests that lock the precedence in place.
