## Summary

Keep open. Current `main` still treats `connected === false` as immediately unhealthy once startup grace expires, the health monitor does not pass any disconnect timestamp into policy evaluation, and the tests still assert restart of a long-running disconnected channel. The PR discussion points to a real unsolved race; its review feedback was mostly polish, and one cited type gap (`ChannelAccountSnapshot.lastDisconnect`) is already typed on `main`, so the core reconnect-grace fix is not obsolete.
