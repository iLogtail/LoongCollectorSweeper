## Summary

Keep open. Current main still does not cap Pi compaction summarization output against the selected model's `maxTokens`; the reported `reserveTokensFloor: 300000` path can still feed `reserveTokens=300000` into Pi, whose dependency code computes `maxTokens = floor(0.8 * reserveTokens)` and would request 240000 output tokens. The linked fix PR #54392 is still open/unmerged in the provided GitHub context, so this issue is not implemented or superseded on main.

## Best Possible Solution

Land an OpenClaw-side clamp before Pi receives compaction reserve tokens, or otherwise wrap the summary call, so the effective reserve for summarization cannot make Pi request more than the selected model's output limit. Cover both default and safeguard compaction paths with a regression for an Anthropic 1M model, `maxTokens: 128000`, and `reserveTokensFloor: 300000`; coordinate with or supersede PR #54392.
