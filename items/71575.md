## Summary

Keep #71575 open. Current main still deliberately renders Telegram tool-progress preview updates by default when preview streaming is active: it builds a visible `Working…` draft preview, appends tool names/progress text/command summaries, and only offers `streaming.preview.toolProgress: false` or `streaming.mode: off` as opt-outs. The related #71663 is closed as the narrower duplicate/context item, but it does not supersede the broader requested behavior here: “only final assistant reply” is still not implemented on main.

## Best Possible Solution

Keep the issue open for maintainer product/behavior work. The likely fix belongs in the Telegram preview streaming path: either make Telegram hide tool-progress previews by default, add a stricter final-only delivery mode for Telegram sessions, or sanitize/summarize progress so raw tool output never appears. Update `extensions/telegram/src/bot-message-dispatch.ts`, Telegram dispatch tests, and Telegram/streaming docs together.
