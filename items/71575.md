## Summary

Keep open. Current main does not satisfy the issue's default expected behavior of showing only the final Telegram reply. Telegram tool-progress is still intentionally visible by default in preview streaming, and the tests cover the same `Working…` / `tool: exec` style output reported here. A config opt-out exists, but that is not the same as the requested default behavior.

## Best Possible Solution

Keep this open for maintainer judgment on Telegram privacy/defaults. If the desired behavior is final-only by default, change the Telegram dispatch/config contract so tool/progress payloads are suppressed unless explicitly enabled, update the Telegram and streaming docs, and adjust the current tests that intentionally assert visible `Working…` tool progress. If maintainers want the current released behavior, close manually with a product-position comment pointing users to `channels.telegram.streaming.preview.toolProgress: false`.
