## Summary

Current `main` at `350d3221800a18283915cd2f3b612ae04ddd490c` still matches the reported failure mode. The Anthropic provider wrapper only injects OAuth-aware beta merging when betas come from `extraParams`, and the transport path still merges headers with plain overwrite semantics, so a model-level `headers["anthropic-beta"]` can replace the default `oauth-2025-04-20` beta for `sk-ant-oat*` tokens.

## Evidence

- **Anthropic beta wrapper is still skipped when only model headers set the beta:** `resolveAnthropicBetas()` only looks at `extraParams.anthropicBeta` and `extraParams.context1m`, and `wrapAnthropicProviderStream()` only applies `createAnthropicBetaHeadersWrapper()` when that result is non-empty. A model configured only with provider/model `headers` never enters the OAuth-aware beta merge path.
  - file: [extensions/anthropic/stream-wrappers.ts:82](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/anthropic/stream-wrappers.ts#L82)
  - command: `nl -ba extensions/anthropic/stream-wrappers.ts | sed -n '75,210p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **OAuth default beta can still be overwritten by model headers:** `createAnthropicTransportClient()` seeds OAuth requests with `anthropic-beta: claude-code-20250219,oauth-2025-04-20,...` and then calls `mergeTransportHeaders(defaults, model.headers, options?.headers)`. `mergeTransportHeaders()` uses `Object.assign` in source order, so `model.headers["anthropic-beta"]` replaces the OAuth beta instead of merging with it.
  - file: [src/agents/anthropic-transport-stream.ts:561](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/anthropic-transport-stream.ts#L561)
  - command: `nl -ba src/agents/anthropic-transport-stream.ts | sed -n '520,600p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Header merge helper is overwrite-only:** The shared helper does a straight `Object.assign` across header sources and has no special-case merge for comma-delimited beta headers, which preserves the regression when model-level Anthropic headers are present.
  - file: [src/agents/transport-stream-shared.ts:47](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/transport-stream-shared.ts#L47)
  - command: `nl -ba src/agents/transport-stream-shared.ts | sed -n '40,60p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Current transport test does not cover the reported regression case:** The OAuth transport test asserts Bearer auth, `x-app`, and `user-agent`, but it does not assert preservation of `oauth-2025-04-20` when `model.headers` supplies `anthropic-beta`, so there is no proof on `main` that this path was fixed.
