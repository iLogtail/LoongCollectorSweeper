## Summary

Keep open. Current main still appears to have the reported gap: Anthropic beta header merging is only installed when `resolveAnthropicBetas()` returns betas from extra params, so a configuration that only supplies `anthropic-beta` via model/provider headers can still skip the OAuth beta wrapper. The provided timeline also shows the exact fix PR (#41461) is still open, while related attempted PRs were closed unmerged.

## Evidence

- **Wrapper still gated on extra params:** `wrapAnthropicProviderStream()` calls `resolveAnthropicBetas(ctx.extraParams, ctx.modelId)` and only applies `createAnthropicBetaHeadersWrapper` when `anthropicBetas?.length` is truthy. This preserves the issue's reported condition: no `context1m` or `anthropicBeta` in extra params means no OAuth beta wrapper.
  - file: [extensions/anthropic/stream-wrappers.ts:194](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/anthropic/stream-wrappers.ts#L194)
  - command: `nl -ba extensions/anthropic/stream-wrappers.ts | sed -n '191,208p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Beta resolver ignores model headers:** `resolveAnthropicBetas()` only inspects `extraParams.anthropicBeta` and `extraParams.context1m`; it does not consider `model.headers` or provider/model-level `anthropic-beta` headers.
  - file: [extensions/anthropic/stream-wrappers.ts:82](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/extensions/anthropic/stream-wrappers.ts#L82)
