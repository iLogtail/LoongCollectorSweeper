## Summary

Current main does not yet prove this regression fixed. The Anthropic plugin still only applies the beta-header wrapper when `extraParams` yields Anthropic betas, so a model-level `headers: { "anthropic-beta": "context-1m-2025-08-07" }` path can bypass the wrapper that preserves `oauth-2025-04-20`. Related PR #41461 tracks the same fix but is still open, and earlier PRs in the context were closed unmerged.

## Best Possible Solution

Keep #41444 open. The best fix is to land or rework the #41461-style change in the Anthropic plugin so OAuth-required betas are preserved even when no `context1m`/`anthropicBeta` extra param is present, or otherwise explicitly merge/reject model-level `anthropic-beta` headers for OAuth. Add a focused regression test for `sk-ant-oat*` plus model-level `headers["anthropic-beta"] = "context-1m-2025-08-07"`; update docs if model-level beta headers are intentionally unsupported.
