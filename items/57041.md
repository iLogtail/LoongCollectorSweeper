## Summary

Keep #57041 open. Current main has shipped adjacent descendant-pending fixes, but it has not implemented this PR's narrower cleanup-pending state: ended runs with no pending descendants and no cleanupCompletedAt still format as done/recent, and the status/list active predicates do not gate on cleanupCompletedAt plus suppressAnnounceReason. The PR likely needs rebasing onto the newer subagent-list/status helper structure rather than closure.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation onto current main: add a shared cleanup-pending predicate that treats ended runs without cleanupCompletedAt as active unless suppressAnnounceReason is set, wire it through subagent list, target resolution, status summary, and active-run counting, and add focused tests for completing display plus suppressed killed/steer-restart behavior.
