## Summary

Keep open. Current main still lacks the PR's cleanup-pending active-run behavior: ended runs are considered inactive unless they have pending descendants, status formatting still maps ended ok runs to done, and spawn quota counting does not include an ended run whose cleanupCompletedAt is still unset. The PR discussion and Greptile review describe a plausible race, and current tests cover pending descendants but not the cleanup-pending window this PR targets.

## Best Possible Solution

Keep the PR open for maintainer review or an equivalent rebased fix. The best implementation should treat ended runs with missing cleanupCompletedAt and no suppressAnnounceReason as active/completing in subagent list/status, target ordering, and countActiveRunsForSession spawn gating, with regression tests for cleanup pending, cleanupHandled-but-not-completed, and suppressed steer/killed runs.
