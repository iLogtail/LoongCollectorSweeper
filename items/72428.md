## Summary

Keep #72428 open. Current main has related liveness and empty-result guardrails, but it does not contain the PR's generic reply-run progress watchdog or deterministic user-visible empty-final fallback in the outer auto-reply runner. The PR is recent, contributor-authored, has no protected labels, and still appears to contain potentially useful work, with review comments that should be addressed before merge rather than used as close evidence.

## Best Possible Solution

Keep the PR open and continue review. The best path is a corrected implementation in `src/auto-reply/reply/` that adds a generic quiet-run watchdog only for appropriate user-visible routes, resets on the existing visible activity callbacks, logs transport failures, and handles both empty-final branches without duplicating replies after block streaming or messaging-tool side effects. Require focused watchdog/empty-final tests plus the normal changed gate before merge.
