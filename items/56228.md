## Summary

Keep open. Current main still resolves Discord reply-context text from content, media placeholders, embeds, and fallback text, but does not inspect `flags` or the `components` tree. The related PR #56248 proposed the same fix but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Implement a small, schema-tolerant Components v2 text extractor in the Discord plugin, wire it into `resolveDiscordMessageText()` and `resolveDiscordSnapshotMessageText()`, and add focused regression tests for referenced replies and forwarded snapshots where `content` and `embeds` are empty but TextDisplay components contain visible text.

## Risks / Open Questions

- A fix should traverse only display-text fields from Discord Components v2 and avoid treating button labels, custom IDs, or callback data as quoted message body unless product intent says otherwise.
- Forwarded snapshot handling likely needs the same extractor as normal referenced messages, because current snapshot text resolution also stops at content, media placeholders, and embed text.
