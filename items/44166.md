## Summary

Keep #44166 open. Current main still routes inline memory embedding batches through `embedBatchWithRetry()`, but the retry classifier only matches rate-limit/429/5xx/cloudflare/token-limit style text and does not classify `fetch failed`, `ECONNRESET`, `socket hang up`, `terminated`, or `other side closed` as retryable. There is also no recursive split fallback after retry exhaustion. The linked PR #44167 is directly relevant but remains open/unmerged, and #59137 is related rollback/cache recovery work rather than a direct transport retry fix.

## Best Possible Solution

Keep #44166 open until memory-core implements a narrow fix in `extensions/memory-core/src/memory/manager-embedding-policy.ts` and `extensions/memory-core/src/memory/manager-embedding-ops.ts`: classify common transient transport errors as retryable, retry with the existing backoff loop, recursively split multi-item batches after retry exhaustion, and add focused tests for one-shot `fetch failed` recovery plus repeated multi-item transport failure splitting. #44167 is the most directly relevant candidate patch, but it needs to be landed or adapted to current `extensions/memory-core` main before the issue can close.
