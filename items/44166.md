## Summary

Keep open: current `main` still only retries memory embedding batch failures that match the narrow rate-limit/5xx regex, and `embedBatchWithRetry()` still has no retry-on-transport-error or split-the-batch fallback after retries are exhausted.

## Evidence

- **Retry classifier still excludes transport failures:** `isRetryableMemoryEmbeddingError()` in `extensions/memory-core/src/memory/manager-embedding-policy.ts:83` only matches rate-limit/429/5xx/Cloudflare/tokens-per-day text. It does not include the issue’s transient transport cases like `fetch failed`, `ECONNRESET`, `socket hang up`, `terminated`, or `other side closed`.
  - file: [extensions/memory-core/src/memory/manager-embedding-policy.ts:83](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/memory-core/src/memory/manager-embedding-policy.ts#L83)
  - command: `nl -ba extensions/memory-core/src/memory/manager-embedding-policy.ts | sed -n '1,200p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Batch embedding path still just retries or throws:** `embedBatchWithRetry()` in `extensions/memory-core/src/memory/manager-embedding-ops.ts:234` passes `provider.embedBatch(texts)` into `runMemoryEmbeddingRetryLoop()` and returns its result directly. There is no recursive split-and-continue fallback when retries are exhausted on a multi-item batch.
  - file: [extensions/memory-core/src/memory/manager-embedding-ops.ts:234](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/memory-core/src/memory/manager-embedding-ops.ts#L234)
  - command: `nl -ba extensions/memory-core/src/memory/manager-embedding-ops.ts | sed -n '230,330p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Current tests cover only rate-limit/5xx retry behavior:** `extensions/memory-core/src/memory/manager-embedding-policy.test.ts:46` and `:74` test retries for `429`, `502 Bad Gateway (cloudflare)`, and `Too many tokens per day`. There is no test for transient transport errors or for splitting a failed multi-item batch into smaller retries.
  - file: [extensions/memory-core/src/memory/manager-embedding-policy.test.ts:46](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/memory-core/src/memory/manager-embedding-policy.test.ts#L46)

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Changelog wording still stops at 5xx retries:** `CHANGELOG.md:6075` says memory indexing retries transient `5xx errors (Cloudflare)`, while `:6079` separately mentions splitting batches for token limits. That matches the current code and does not indicate a shipped fix for transient embedding transport retries plus split fallback.
  - file: [CHANGELOG.md:6075](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/CHANGELOG.md#L6075)
