## Summary

Keep #53557 open. Current main moved memory search into the memory-core plugin, so the PR's original files are stale, but the requested behavior is still only partially present: CLI status does not print Mode or Provider issue, memory_search still omits providerUnavailableReason, and the Ollama/auto unavailable hint has not been ported.

## Best Possible Solution

Keep the PR open and ask for a rebase/port to the current memory-core plugin surface: update extensions/memory-core/src/cli.runtime.ts to show Mode and Provider issue from status.custom, update extensions/memory-core/src/tools.ts to include providerUnavailableReason in memory_search responses, update extensions/memory-core/src/tools.shared.ts for the Ollama auto-mode hint, and add regression coverage in extensions/memory-core/src/cli.test.ts and extensions/memory-core/src/tools.test.ts while addressing the duplicate fallback review comment.
