## Summary

Keep open. Current `main` still prunes sandbox runtime containers and registry entries but does not remove sandbox workspace directories, so the cleanup request is not implemented on `main`. The PR also appears stale against current tree because its docs file target no longer exists locally.

## Evidence

- **Current prune path only removes runtime + registry entry:** `maybePruneSandboxes()` calls `pruneSandboxContainers()` and `pruneSandboxBrowsers()`. In `pruneSandboxRegistryEntries()`, the `finally` block only calls `params.remove(entry.containerName)` and `params.onRemoved?.(entry)`; there is no filesystem deletion of sandbox workspaces.
  - file: [src/agents/sandbox/prune.ts:38](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/sandbox/prune.ts#L38)
  - command: `nl -ba src/agents/sandbox/prune.ts | sed -n '38,123p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Container backend removal does not clean workspace directories:** Docker backend `removeRuntime()` only runs `docker rm -f` for the container name and ignores workspace paths entirely.
  - file: [src/agents/sandbox/docker-backend.ts:143](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/sandbox/docker-backend.ts#L143)
  - command: `sed -n '1,240p' src/agents/sandbox/docker-backend.ts`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Sandbox workspaces are real on-disk directories derived from scope:** `ensureSandboxWorkspaceLayout()` resolves `sandboxWorkspaceDir` from `cfg.workspaceRoot` and `resolveSandboxWorkspaceDir(...)` for non-shared scopes, or uses the root directly for shared scope. Those directories are created/ensured, so pruning only the container leaves disk state behind.
  - file: [src/agents/sandbox/context.ts:38](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/sandbox/context.ts#L38)
  - command: `nl -ba src/agents/sandbox/context.ts | sed -n '38,82p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Registry stores the scope/session key needed for workspace cleanup:** Sandbox registry entries include `sessionKey`, so current `main` has the metadata needed to map a pruned runtime back to its workspace, but prune code does not use it.
  - file: [src/agents/sandbox/registry.ts:8](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/agents/sandbox/registry.ts#L8)
