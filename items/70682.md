## Summary

Keep PR #70682 open. Current main has not implemented this PR's `createHostWorkspaceReadTool` change, and the linked privacy-isolation report (#70573) remains a real security-sensitive product decision rather than cleanup. Current main still treats `tools.fs.workspaceOnly` as opt-in/default false, and existing tests document that `read` outside the workspace is allowed when the flag is false. The PR is not safe to close as implemented or obsolete, though it likely needs maintainer review, narrowing, and alignment with the intended default/security policy.

## Best Possible Solution

Keep #70682 open for maintainer/security review, but require a narrowed fix. The best path is to resolve #70573 explicitly: decide whether `read` should become workspace-contained by default, whether agent-specific filesystem isolation/path permissions are needed, or whether operation-level read enforcement is only defense-in-depth for `tools.fs.workspaceOnly: true`. Rebase/split out unrelated changes and address the existing review comment before considering merge.
