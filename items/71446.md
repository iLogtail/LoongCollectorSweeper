## Summary

Keep open. Current main still lacks the null/inaccessible-proxy guards that #71446 is trying to add, so this PR is not implemented on main or superseded. The PR should still get maintainer review because its current diff drops existing public-surface TOCTOU regression coverage and does not add channel-entry regression coverage for the new guard.

## Best Possible Solution

Keep #71446 open for maintainer review. The best path is a narrowed patch that adds null/undefined and inaccessible-proxy validation to both bundled channel-entry and public-surface loaders, deletes bad cache entries on failure, preserves the existing validated canonical path/hardlink/TOCTOU protections and tests, and adds focused regression tests for the channel-entry and public-surface bad-loader cases.
