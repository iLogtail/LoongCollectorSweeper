## Summary

Keep #15073 open. Current main still implements model-only fallback, not fallback-agent/workspace fallback: candidates carry provider/model only, retries reuse the active session agent, workspace, agentDir, session file, and skills snapshot, and docs/config schema expose only `{ primary, fallbacks }` model refs. The issue discussion also includes a current maintainer/Codex review comment explicitly keeping this open, and the related closed issues cover narrower context-budget/profile/file-injection ideas rather than superseding the broader fallback-agent/workspace request.

## Best Possible Solution

Keep #15073 open until OpenClaw adds explicit fallback-agent/workspace/context-profile semantics or maintainers choose a narrower model-specific context-budget/profile design as the canonical product direction. The likely core path is config schema/types/docs changes, fallback candidate metadata beyond provider/model, retry plumbing that can resolve a fallback agent/workspace safely, bootstrap/context assembly updates, and regression tests proving fallback retries use the intended agent context.

## Risks / Open Questions

- Plugin hooks can partially work around prompt shaping, but the current hook/config surfaces do not provide a first-class way for model fallback to switch the active agent workspace, agentDir, or session context.
- A real implementation needs careful session/transcript ownership handling so fallback-agent retries do not corrupt the original agent session or accidentally persist fallback-only state.
