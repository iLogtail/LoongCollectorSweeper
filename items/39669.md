## Summary

Keep open. Current main still lacks a generic queued-message feedback hook for messages enqueued behind an active session run. Telegram and Discord have partial immediate status-reaction behavior on message receipt, but the core enqueue-followup path still returns silently for channels like Feishu/Lark that only wire feedback through reply-start typing callbacks.

## Best Possible Solution

Keep this issue open. The best fix is a generic non-blocking queued-message feedback seam on the active-run enqueue-followup path, likely an onQueued callback on GetReplyOptions/replyOptions plus channel wiring for Feishu/Lark typing or queue reactions. Add focused tests proving the callback fires when a message is queued behind an active run, callback failures do not block enqueueing, and existing Telegram/Discord status reactions and normal run-start typing remain unchanged.
