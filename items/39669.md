## Summary

Keep open. Current main still lacks the generic queued-message feedback hook requested by #39669. The active-run enqueue-followup path queues the message, touches session state, cleans up typing, and returns without any user-visible queued callback. Feishu still starts its visible typing reaction from reply-start callbacks, which are only reached when the run actually starts. Telegram and Discord have partial initial status-reaction behavior, but that does not solve the cross-channel/Feishu queue feedback gap. The related PR #39676 proposed the missing onQueued seam but was closed unmerged in the provided GitHub context, and current main contains no onQueued implementation.

## Best Possible Solution

Keep #39669 open until OpenClaw adds an additive queued-feedback seam in the reply runtime, such as an onQueued callback or equivalent typed channel-feedback hook invoked in the enqueue-followup branch before cleanup. Channel plugins should then map it to their native reaction/typing/ack behavior, with focused tests for the core enqueue-followup path and Feishu plus existing Telegram/Discord status reaction behavior.
