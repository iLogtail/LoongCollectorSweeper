## Summary

Keep #29736 open. Current main still resolves exec approvals defaults through `~/.openclaw` via `OPENCLAW_HOME`/home expansion, not the active `OPENCLAW_STATE_DIR` or profile-derived state root. The issue discussion contains concrete reproductions across systemd, profile isolation, and container deployments, and the current source/tests/docs still match the reported split-state behavior rather than a fixed state-dir contract.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. The default approvals file and socket should derive from the active state root/profile semantics, likely via `resolveStateDir()`, while preserving a compatibility or migration path for existing `~/.openclaw/exec-approvals.json`. Update local CLI, gateway RPC, node-host RPC, docs, and regression tests for `OPENCLAW_STATE_DIR`, `--profile`, `OPENCLAW_HOME`, container writable-state deployments, and symlink safety.
