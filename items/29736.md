## Summary

Keep #29736 open. Current main still resolves exec approvals through the home-based `~/.openclaw` path, using `OPENCLAW_HOME`/OS home expansion rather than the active `OPENCLAW_STATE_DIR` or profile state root. The issue discussion includes multiple concrete reproductions across systemd, profile isolation, and containers; the later comment that it seemed fixed does not match the current default source path unless a workaround such as `OPENCLAW_HOME` is being used.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. Default approvals file and socket resolution should derive from the active state directory/profile semantics, with a deliberate compatibility/migration story for existing `~/.openclaw/exec-approvals.json`, consistent local/gateway/node behavior, and regression coverage for `OPENCLAW_STATE_DIR`, profiles, and container-style writable state roots. Update the CLI and exec approvals docs once the storage contract is settled.
