## Summary

Keep #39528 open. Current main has meaningful hardening around owner-only gateway tooling, admin-scoped RPCs, config path allowlists, dangerous-flag blocking, and rate limits, but it still does not enforce a fresh runtime user-consent gate before model-callable restart, allowlisted config writes, or update.run actions.

## Best Possible Solution

Keep this open as a product/security-behavior bug. The best fix is an enforceable, test-backed recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run, while preserving the existing owner-only/admin-scope, allowlist, dangerous-flag, protected-path, and rate-limit guardrails. Add regression coverage for long sessions and compaction where SOUL.md, USER.md, MEMORY.md, and user red-line instructions remain available or are re-read before sensitive control-plane actions.
