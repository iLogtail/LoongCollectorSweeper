## Summary

Keep open. Current main has meaningful hardening since the 2026.3.2 report, but it still does not enforce a hard explicit-consent gate for agent-initiated gateway restarts, allowed config writes, or updates. The context-retention side has bootstrap and post-compaction mitigations, including Red Lines refresh, but I did not find source/test evidence that the full reported long multi-turn loss of SOUL/USER/MEMORY plus red-line instructions is fixed.

## Best Possible Solution

Keep this open as a product/security-behavior bug. The best path is a hard, test-backed explicit-consent mechanism for agent-initiated gateway restart, config.apply, config.patch, and update.run actions, while preserving the existing protected-path allowlist. Add regression coverage for long sessions and compaction where SOUL.md, USER.md, MEMORY.md, and user red-line instructions remain available or are safely re-read before sensitive control-plane actions.
