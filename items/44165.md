## Summary

Keep PR #44165 open. Current main still has the exact Channels UI behavior the PR targets: `resolveChannelOrder()` returns only `channelMeta` or `channelOrder` when either is present, so a custom-only gateway snapshot can still hide the built-in channel cards. The linked issue #31379 remains open, and the PR is the concrete focused fix rather than a duplicate or obsolete cleanup item.

## Best Possible Solution

Keep #44165 open and resolve #31379 through this PR or an equivalent focused UI change in `ui/src/ui/views/channels.ts`: merge snapshot-provided channel ids with a stable built-in channel id list, preserve snapshot/custom order, dedupe ids, and add regression coverage for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback.
