## Summary

Keep open. Current `main` still does not implement the PR's reported behavior for non-Gemini, non-Anthropic OpenAI-compatible providers, and the linked bug #57443 remains a plausible open product issue. The PR needs a rebase because the schema-cleaning logic moved, but the underlying fix is not obsolete.

## Best Possible Solution

Keep the PR open, ask for a rebase onto current `main`, and move the implementation into `normalizeToolParameterSchema()` or an explicit provider/model-compat path. Add a regression test that proves `patternProperties` is stripped for Volcengine/BytePlus or generic non-Anthropic OpenAI-compatible providers while preserving Anthropic behavior.
