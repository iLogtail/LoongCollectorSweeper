## Summary

Keep open. Current main still implements the Signal SSE subscription with `fetch()` against `/api/v1/events`, while the issue specifically reports Node 24 native-fetch failure and asks for a `node:http` SSE path. The Signal monitor still routes inbound events through that fetch-backed SSE loop, and Docker/current docs make Node 24 a supported/recommended runtime, so this is not safe to auto-close.
