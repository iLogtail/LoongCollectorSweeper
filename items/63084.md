## Summary

Keep open. Current main does not emit the requested internal `gateway:shutdown` or `gateway:pre-restart` hook events from the gateway close path, and a maintainer closure comment on #6711 explicitly points to #63084 as the canonical active tracker for the remaining exact internal hook work. The PR is therefore neither implemented on main nor obsolete, though its current diff still needs maintainer/reviewer resolution for bounded shutdown-hook execution.

## Best Possible Solution

Keep #63084 open as the canonical PR for the internal gateway shutdown/pre-restart hook events. The best path is to revise the PR so shutdown hook dispatch is explicitly bounded/non-blocking, keep focused `server-close` tests, and update internal hook docs if maintainers accept the new events.
