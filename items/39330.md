## Summary

Keep open. Current main still appends the unscheduled reminder note into the final reply payload and current tests assert that visible-note behavior. PR #39330 is a still-relevant focused bug-fix candidate, though the existing review concern about only stripping text that OpenClaw internally appended still needs maintainer resolution.

## Best Possible Solution

Keep #39330 open for maintainer review. The best path is a narrow fix in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents internally injected reminder-guard notes from reaching final channel-visible payloads, while preserving legitimate model output that contains the same text. Update the reminder-guard tests to assert clean delivery behavior and reconcile the approach with broader reminder-guard follow-ups for isolated crons and plugin-managed schedulers.
