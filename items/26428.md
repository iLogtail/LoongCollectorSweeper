## Summary

Current main still installs fish completion by sourcing the cached `openclaw.fish` from `~/.config/fish/config.fish`, not by installing `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The related attempted fix PR #27135 is closed unmerged, and current docs, doctor/setup/update hints, and tests still reflect the profile-based path.

## Best Possible Solution

Keep this issue open and implement the fish-specific path in the CLI completion runtime: write/copy the cached fish completion to `~/.config/fish/completions/<binName>.fish`, detect that installed state there, clean up legacy `config.fish` source blocks safely, update doctor/setup/update reload messaging and docs, and add focused regression tests. PR #27135 is useful prior art but needs rebasing/review because it did not land.
