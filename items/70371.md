## Summary

Keep open. Current main still treats plugin-local config schema failures as invalid global config and strict plugin-load errors in key startup/registry paths, so the requested graceful per-plugin quarantine is not implemented. The triage comment correctly identifies a real bug, and linked PR #70394 is a relevant open fix attempt rather than a shipped resolution.

## Best Possible Solution

Keep #70371 open and finish a core fix. The best path is to quarantine plugin-entry schema failures per plugin: keep clear diagnostics, mark the bad plugin failed or disabled in runtime/metadata registries, skip its registration and tools, continue booting unaffected plugins/channels/jobs, and extend doctor/status repair output for `plugins.entries.<id>.config` schema failures. PR #70394 is a useful starting point but needs maintainer review against the config-load, runtime-loader, metadata-loader, and doctor acceptance criteria.
