## Summary

Keep #70371 open. Current main still treats plugin-local config schema failures as invalid startup config and still enables fatal plugin-load behavior in the runtime/metadata loaders. The linked PR #70394 is relevant and acknowledges the same bug, but it is open/unmerged in the provided context and its proposed behavior is not present in the current checkout.

## Best Possible Solution

Keep the issue open and implement a focused plugin-local graceful-degradation path. Invalid `plugins.entries.<id>.config` should mark only that plugin entry failed or disabled in the in-memory registry, skip its registration and tools, keep unrelated gateway/channels/plugins running, and make `openclaw doctor` report and repair or quarantine the exact config path/schema error. Preserve fail-closed behavior for global config corruption and required runtime-critical surfaces, with regression tests for startup, CLI/bootstrap, status/doctor output, and strict-loader boundaries.
