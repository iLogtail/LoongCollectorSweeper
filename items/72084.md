## Summary

Keep open. Current main still lacks the postinstall stale-version reinstall behavior that PR #72084 targets; it only treats missing sentinels or missing optional native children as install triggers. The adjacent runtime/doctor staging path has version-aware checks, but that does not make this postinstall PR obsolete, and the supplied PR discussion still has actionable review feedback.

## Best Possible Solution

Keep PR #72084, or a focused successor, open. The best path is to add version-aware stale dependency detection to `scripts/postinstall-bundled-plugins.mjs`, reuse or match the existing semver contract used by runtime staging, add regression coverage for stale exact/range versions, keep the optional native child reinstall test exercising that path, and remove or split unrelated lockfile churn before merge. If maintainers decide postinstall should remain sentinel-only, they should explicitly close this in favor of a canonical doctor/runtime repair tracker rather than treating current main as implemented.
