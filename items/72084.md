## Summary

Keep open. Current main has version-aware runtime-dependency checks in the doctor/startup staging path, but the postinstall script changed by this PR still treats any existing root sentinel package.json as installed unless a platform-targeted optional native child is missing. The current postinstall tests explicitly assert that an existing `{}` sentinel skips reinstall, so the stale-version postinstall behavior proposed by #72084 is not implemented on main. The linked bug #72058 remains a live regression report, and the PR has concrete review comments to address rather than being obsolete.

## Best Possible Solution

Keep the PR open and ask for a focused cleanup: add postinstall stale-version detection for bundled runtime deps, prefer shared semver semantics or a clearly documented supported-spec subset, preserve optional-native-child coverage by giving sentinel fixtures real versions, address the existing review comments, and rebase away unrelated lockfile changes already present on main.
