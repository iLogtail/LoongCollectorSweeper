## Summary

Current main still lacks the requested configurable suppression policy for transient mutating tool-error warnings. It has partial retry-success cleanup for the same mutating action, but payload policy still intentionally surfaces mutating tool failures even when an assistant reply exists, and the related suppression PR #39636 is closed unmerged.

## Best Possible Solution

Keep this issue open until maintainers choose and implement the intended policy in the shared embedded-run payload path, likely as an explicit message/tool-error delivery setting with tests for mutating failures, recovered final replies, heartbeat per-run suppression, and docs/schema alignment.
