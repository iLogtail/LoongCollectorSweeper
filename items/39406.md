## Summary

Current main still does not implement the requested normal-chat policy for suppressing transient mutating tool error warnings after an eventual successful assistant reply. The existing `messages.suppressToolErrors` setting only suppresses non-mutating failures, `suppressToolErrorWarnings` remains a per-run/heartbeat option, and tests still assert that a mutating `write` failure is emitted even when the assistant says `Done.` The related PR #39636 in the provided GitHub context was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep #39406 open for maintainer product judgment or implementation. A good fix would add a documented normal-chat tool-error delivery policy, preserve the current safety default, cover mutating retry-success/final-success cases in payload tests, and update config schema, help text, generated schema, and docs.
