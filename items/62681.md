## Summary

Keep #62681 open. Current main has general Control UI localization, but the central Telegram request is not implemented: built-in command names/descriptions remain English in the shared command registry, Telegram registers a single command menu via setMyCommands without language-scoped options, and Telegram config exposes native/custom command knobs but no locale or per-language description map. The issue's mention of an i18n skill is useful prior art, but it does not satisfy bundled Telegram native command registration or Control UI slash-command labels. The provided related Telegram reports cover other command/API surfaces and do not supersede this request.

## Best Possible Solution

Keep the issue open as a valid bundled Telegram and Control UI i18n request. The best path is to preserve canonical English command names for compatibility, add locale-aware display descriptions/help text through the shared command registry and existing UI i18n pipeline, and extend Telegram menu sync/config to register localized descriptions per language. If localized command aliases are desired, model them as explicit backward-compatible aliases with Telegram/runtime/UI tests rather than replacing existing commands.
