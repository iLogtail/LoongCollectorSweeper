## Summary

Keep open. Current main still has generic health-monitor restart/cooldown/hourly-cap behavior, but not the requested repeated-restart escalation, Discord clean-restart/fresh-IDENTIFY hook, or catch-up behavior for delayed Discord ingest. The issue also has fresh current-version evidence and an open PR (#40413) that explicitly proposes the missing implementation.

## Best Possible Solution

Keep #38596 open until a maintainer lands or rejects a focused Discord recovery design. The likely path is to evaluate PR #40413 or a narrower replacement: add monitor-level repeated-restart escalation, expose a clean Discord restart/fresh IDENTIFY path across the plugin/channel boundary, reset escalation after a stable period, and cover it with health-monitor plus Discord lifecycle regression tests. #71546 can remain related evidence for user-visible ingest lag, but it does not replace this issue’s monitor-escalation request.
