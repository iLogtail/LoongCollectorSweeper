## Summary

Keep open. Current main has related prompt-cache mitigation and telemetry, including cache-ttl pruning and documented heartbeat-based cache warming, but it does not implement the requested invisible, non-turn, cost-capped prompt-cache keep-warm ping mechanism. Existing heartbeats are real agent runs with a prompt, scheduling, visibility/delivery behavior, optional isolated sessions, and no adaptive TTL or hourly cost cap. The related PR context also does not provide a merged canonical replacement: #62179 only exposes prompt-cache telemetry, #56575 remains open and covers compact/reset maintenance modes, and #62501 was closed unmerged.

## Best Possible Solution

Keep #62475 open for maintainer product judgment. The best implementation path is either a scoped, opt-in Anthropic prompt-cache keep-warm service that reuses the exact production payload shaping path, records cache-read/cache-write telemetry, cancels on dirty prompt/tool changes, and enforces a hard cost cap; or an explicit maintainer decision that heartbeat is the supported workaround, with the issue closed manually after documenting that tradeoff.
