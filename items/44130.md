## Summary

Keep open. Current `main` does not show a shipped fix for the reported TUI scroll-jump behavior: the TUI chat log is still a plain append-only `Container`, history reloads still clear and rebuild the log, and event handling still forces frequent full rerenders without any explicit viewport-preservation or auto-follow gating. There are also no functional `src/tui` changes between the latest release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` and current `main` `50e36983bb2d827adc617282c090c0a0359c6eee`.

## Evidence

- **Chat log has no explicit viewport state or scroll-anchor handling:** `ChatLog` only appends/removes child components and tracks tool/run references; there is no scroll position, follow-bottom, or viewport anchor state in the TUI chat surface.
  - file: [src/tui/components/chat-log.ts:11](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/tui/components/chat-log.ts#L11)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **History reload still clears and rebuilds the entire chat log:** `loadHistory()` calls `chatLog.clearAll()`, re-adds the session banner, then replays every message before requesting a render. That rebuild path matches the kind of disruptive viewport reset described in the issue.
  - file: [src/tui/tui-session-actions.ts:288](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/tui/tui-session-actions.ts#L288)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Streaming/event paths still request rerender repeatedly without viewport guards:** TUI chat/agent event handlers update assistant/tool content and call `tui.requestRender()` on delta/final/tool/lifecycle paths, but there is no adjacent logic preserving reader position or only auto-following when already at bottom.
  - file: [src/tui/tui-event-handlers.ts:313](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/tui/tui-event-handlers.ts#L313)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **User docs show no TUI scroll control or auto-follow toggle:** The TUI docs list pickers, settings, and shortcuts, but nothing for scroll lock, jump-to-latest, or disabling auto-follow in the terminal UI.
  - file: [docs/web/tui.md:79](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/web/tui.md#L79)
