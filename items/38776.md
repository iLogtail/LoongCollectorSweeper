## Summary

Keep open. Current main still does not implement the Feishu quoted interactive-card variants from #38776: the read/quote parser only handles flat `elements` or `body.elements` with `markdown`/`div` text. The PR is not implemented away, and the related superseder #39339 was closed unmerged. There are newer overlapping Feishu parser PRs/issues, but they are unmerged and not a clean shipped replacement for this narrow fix yet.

## Best Possible Solution

Keep #38776 open until a Feishu quote/read parser fix lands, or until maintainers explicitly pick a canonical successor. The best implementation path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and the Feishu raw/post fallback shapes covered by newer reports.
