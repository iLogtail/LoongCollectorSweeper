## Summary

Keep open. Current main has related OpenAI-compatible completions hardening, but the maintainer discussion explicitly leaves #66125 open because the broad fallback failure is not yet proven fixed. The remaining blocker is the exact real-agent request/response from the proxy, since current code still sends a normal agent completions payload rather than a guaranteed strict-minimal chat/completions request.

## Best Possible Solution

Keep #66125 open and ask for a captured real-agent OpenAI-compatible proxy request body plus response status/body. Once that evidence exists, implement the smallest targeted fix: another payload sanitizer, a documented strict-minimal `openai-completions` compatibility mode, or clearer fallback diagnostics if the request is already correct.
