## Summary

Keep open. Current main does not contain the PR's compatibility shim: `normalizeTargetForProvider` still only accepts string normalizer output, so a runtime legacy `{ ok, to }` value is discarded rather than extracting `to`. The exact PR needs rebase to the current split outbound test layout, but the underlying compatibility work is still plausible and not superseded by main.

## Best Possible Solution

Keep the PR open, rebase it onto the current outbound test-file layout, and either merge the narrow runtime adapter that extracts a trimmed string `to` from legacy `{ ok, to }` normalizer results or reimplement the same compatibility behavior directly on main with focused target-normalization and send-runner regression coverage. Resolve the bot comments by documenting or avoiding the broad OAuth mock and trimming unrelated generated baseline churn if it is no longer needed.
