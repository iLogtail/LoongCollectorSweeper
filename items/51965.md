## Summary

Keep open. Current `main` and the latest shipped release still hardcode non-Azure custom onboarded models to `input: ["text"]`, so #51869 is not implemented yet, and PR #51965 is only a partial fix with unresolved correctness and coverage gaps.

## Evidence

- **Current main still stores non-Azure custom models as text-only:** `applyCustomApiConfig` on current `main` uses an Azure-only `isLikelyReasoningModel` branch for image input, but the non-Azure branch still writes `input: ["text"]` and `reasoning: false`. The same code is present in release `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`), so the reported behavior is still shipped.
