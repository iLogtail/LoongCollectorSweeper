## Summary

Keep open. Current main has zh-CN lazy locale support and generated zh-CN translations, but the reported Control UI tabs still contain many user-visible English literals in source, so the observable problem remains.

## Best Possible Solution

Keep the issue open and treat it as Control UI i18n debt. Add English source keys for all user-visible strings in the affected tab views and channel subcards, switch those render paths to t(...) calls, then regenerate non-English locale bundles with the existing Control UI i18n pipeline and add focused render/i18n coverage for zh-CN on the affected tabs.
