## Summary

Keep open. Current main still appears to have the reported Discord DM classification failure path: when channel-info lookup fails for a guild-less Discord message, preflight does not infer a DM from the missing guild_id/sender context, so routing can still build and persist a `discord:channel:<dmChannelId>` session key. Existing direct-session normalization only helps after the context is already classified as direct.

## Best Possible Solution

Fix Discord preflight to prefer direct routing for guild-less inbound messages when channel-info lookup is unavailable and the sender/user context is sufficient, then thread that inferred direct classification through route peer construction, binding conversation id, persisted `ChatType`, and last-route target. Add a focused regression test with `session.dmScope: "per-channel-peer"` and a DM event whose `fetchChannel` returns null or rejects, asserting the session key stays `agent:main:discord:direct:<userId>` and no `discord:channel:<dmChannelId>` session is created. Consider a conservative session-key repair only as a backstop, not the primary fix.

## Evidence
