## Summary

Current main does not implement #69748. Discord still uses `requireMention?: boolean` plus `users`/`roles` sender allowlists, with role allowlist support and per-channel overrides only for the existing fields. It does not provide the proposed `deny`/`bypass`/`mention` tiers, permission aliases, sender-specific no-mention bypass, or additive channel inheritance with `inherit: false`.

## Best Possible Solution

Keep #69748 open for bundled Discord plugin work. A good path is to add typed guild/channel `deny`, `bypass`, and `mention` lists with audited role and permission matching, define merge versus `inherit: false` precedence, preserve legacy `requireMention`/`users` via migration or doctor warnings, and cover schema, docs, and preflight tests for deny-first behavior and per-channel overrides.
