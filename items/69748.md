## Summary

Keep #69748 open. Current main still uses Discord `requireMention` as one guild/channel boolean and `users`/`roles` as sender allowlists. It has role allowlist support and per-channel overrides, but it does not implement separate `bypass`/`mention`/`deny` access lists, `all`/`perm:*` entries, channel-level union inheritance, or sender-specific mention bypass semantics.

## Best Possible Solution

Keep the issue open for a Discord plugin design/implementation pass. The likely path is to add typed guild/channel access policy fields for bypass/mention/deny, define precedence and inheritance/merge semantics, support role and selected permission predicates, preserve legacy `requireMention`/`users` through doctor or migration compatibility, update docs/generated config metadata, and add preflight regression tests for owner-bypasses-while-everyone-else-mentions, deny precedence, role/permission matches, and per-channel inheritance.
