## Summary

Keep #69748 open. Current main has Discord role allowlists and per-channel `requireMention`, but it does not implement the requested access model: there are no `bypass`/`mention`/`deny`/`inherit` config fields, no `perm:*` matcher support, and runtime still applies `users`/`roles` as one sender allowlist before a single guild/channel mention boolean.

## Best Possible Solution

Keep this open as a Discord plugin-owned config/runtime enhancement. The maintainable path is an additive, backward-compatible Discord access-policy schema with explicit `bypass`, `mention`, and `deny` precedence, role and selected permission matchers, documented channel inheritance/merge rules, migration or doctor guidance for existing `requireMention`/`users` configs, and regression tests for guild-level, channel-level, role, permission, and deny-first cases.
