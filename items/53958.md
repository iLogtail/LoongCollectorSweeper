## Summary

Keep open. Current main has the backend pieces to expose the active config path to admin clients, and the related #54025 correctly pointed in that direction, but the Control UI still does not visibly render the config file path in Overview or Settings. The only Settings/Config use is an `Open` button tooltip/title, which does not satisfy the requested prominent troubleshooting surface.

## Best Possible Solution

Keep the issue open and implement a small Control UI change that visibly shows the active config path in the Overview snapshot card and/or the Settings > Config header using the existing admin-only `hello.snapshot.configPath` or `configSnapshot.path`. Preserve the current low-privilege omission behavior and add a focused render test asserting the path is visible when present and absent when not authorized.
