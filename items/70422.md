## Summary

Keep #70422 open. Current main still only treats `MiniMax-VL-01` as the MiniMax VLM route, while this PR proposes a real behavior/docs/test change for explicit `MiniMax-M2.7` and `MiniMax-M2.7-highspeed` refs. The related #69691 work is still open and narrower, and review discussion leaves a material endpoint-contract question unresolved.

## Best Possible Solution

Keep #70422 open for maintainer review. Verify the MiniMax VLM endpoint contract, then either land a corrected allowlist/docs/test change for explicit `MiniMax-M2.7` and `MiniMax-M2.7-highspeed` refs, or reject/narrow the change with documented rationale if `MiniMax-VL-01` must remain the only supported MiniMax image-understanding model.
