## Summary

本 PR 针对 metric_system_v2 插件中 fd_max 指标因 float64 转换导致精度丢失的问题，提出了改为字符串上报的修复方案。PR 处于开放状态且未合并，改动范围清晰，属于有效的缺陷修复，应保持开启等待维护者审查与合并决策。

## Best Possible Solution

建议维护者审查本 PR 引入的 addMetricStringVal 及下游 helper.NewMetricLogStringVal 的兼容性。若 CI 通过且下游消费端支持字符串类型指标，可直接合并；也可根据 PR 讨论评估是否直接移除恒定的 fd_max 指标以简化实现。
