## Summary

本 PR 提议将 metric_system_v2 插件的 fd_allocated 与 fd_max 指标类型由 float64 改为 string 以解决大整数精度丢失问题。该变更属于下游破坏性更新，且作者在正文中提出了改用 int64 或直接移除恒定 fd_max 指标等替代方案。当前 PR 基线较旧需 Rebase，且指标类型变更涉及产品兼容性与下游消费端适配，需维护者介入评估技术选型与破坏性影响后再行合并或调整。

## Best Possible Solution

建议维护者评估指标类型变更的下游兼容性影响。若下游强依赖数值类型，可指导作者改用 int64/uint64 上报以保留精度，或采纳移除恒定 fd_max 指标的建议；若确认改为 string 可接受，请作者 Rebase 至最新 main 分支，补充或更新相关单元测试与文档后合并。当前保持开启等待维护者技术选型裁决与作者 Rebase。
