## Summary

本 PR 修复了 metric_system_v2 插件中 fd_max 与 fd_allocated 指标因 float64 转换导致的大数精度丢失问题，方案为将指标值改为字符串上报。该缺陷真实存在，但涉及指标数据类型变更（float64 转 string），可能影响下游消费端兼容性，且 PR 基线较旧需 Rebase。符合保守维护策略中“真实缺陷/可能仍有价值的陈旧 PR 保持开启”的原则，需维护者人工评估与指导。

## Best Possible Solution

维护者需审查指标类型变更的下游兼容性。若确认字符串上报可接受，请指导作者 Rebase 至最新 main 分支并解决潜在冲突后合并；若下游强依赖数值类型，可评估作者提出的替代方案（直接移除恒定的 fd_max 指标）或采用其他精度保留方案（如 int64 上报）。
