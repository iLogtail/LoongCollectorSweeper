## Summary

本 PR 针对 metric_system_v2 插件中 fd_max 与 fd_allocated 指标因 float64 转换导致的大数精度丢失问题提出了修复方案，缺陷真实存在且修改意图明确。但由于该方案涉及指标数据类型变更（float64 转 string），可能影响下游监控消费端的兼容性与解析逻辑，且 PR 基线较旧需 Rebase 至当前 main。符合保守维护策略中“真实缺陷/可能仍有价值的陈旧 PR 保持开启”的原则，需维护者人工评估兼容性边界并指导后续操作。

## Best Possible Solution

维护者需审查指标类型变更的下游兼容性。若确认字符串上报可接受，请指导作者 Rebase 至最新 main 分支并解决潜在冲突后合并；若下游强依赖数值类型，可评估作者正文中提出的替代方案（直接移除恒定的 fd_max 指标）或采用 int64 上报方案以保留精度。
