## Summary

本 PR 修复了 metric_system_v2 插件中 fd_max 和 fd_allocated 指标因转换为 float64 导致的精度丢失问题，改为以字符串形式上报。

## Best Possible Solution

建议维护者审查本 PR。需确认 pkg/helper 中是否已导出 NewMetricLogStringVal 函数以支持字符串指标值；若 CI 通过且下游消费端兼容字符串类型，可直接合并。也可在 PR 讨论中评估是否直接移除恒定的 fd_max 指标以简化实现。

## Evidence

- **pr_diff:** PR 将 fd_allocated 和 fd_max 的上报方式从 strconv.ParseFloat 转换为直接传递字符串，避免 float64 精度丢失。
  - file: [plugins/input/systemv2/input_system_linux.go:162](https://github.com/alibaba/loongcollector/blob/2f363a2cffd36a43e3bb2da731d33edadce41454/plugins/input/systemv2/input_system_linux.go#L162)
