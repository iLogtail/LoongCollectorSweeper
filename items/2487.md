## Summary

本 PR 提议将 metric_system_v2 插件的 fd_allocated 与 fd_max 指标类型从 float64 改为 string 以解决大整数精度丢失问题。由于涉及指标类型变更（破坏性更新）、基线显著落后于当前 main，且作者提出了移除恒定指标或改用其他类型的替代方案，需维护者评估下游兼容性并裁决最终实现路径，故保持开启。

## Best Possible Solution

建议作者先将分支 Rebase 至最新 main 分支并解决冲突。维护者需介入评估指标类型变更的下游兼容性：若下游强依赖数值类型，可考虑改用 int64/uint64 上报以保留精度，或采纳作者建议移除恒定的 fd_max 指标；若确认改为 string 可接受，则指导作者完善对应测试与文档后合并。
