## Summary

本 PR 提议将 metric_system_v2 插件中的 fd_allocated 与 fd_max 指标类型由 float64 改为 string，以修复 max int64 超出 float64 精度范围导致的数据失真问题。该变更涉及指标数据类型的 Breaking Change，可能影响下游消费端兼容性，且 PR 基线较旧需 Rebase。当前需维护者对技术方向（改为 string、改用 uint64 或直接移除恒定 fd_max）进行裁决，并等待作者同步代码与补充测试。

## Best Possible Solution

建议维护者评估将 fd 指标类型改为 string 的下游兼容性影响。若方向确认，请作者 Rebase 至最新 main 分支解决潜在冲突，并补充或更新相关单元测试后合并。若维护者倾向于直接移除恒定的 fd_max 指标或改用 uint64 处理，请指导作者相应调整实现。当前保持开启等待技术裁决与代码同步。
