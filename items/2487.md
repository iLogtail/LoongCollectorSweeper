## Summary

本 PR 修复了 metric_system_v2 插件中 fd_max/fd_allocated 指标因 float64 转换导致的大数精度丢失问题，改为字符串上报。属于有价值的真实缺陷修复，尚未合入 main，建议保持开启以待维护者审查。

## Best Possible Solution

维护者应审查本 PR 的 CI 状态与代码变更，确认下游消费端对字符串类型指标值的兼容性，以及 helper.NewMetricLogStringVal 的导出与行为是否符合预期。若兼容性无虞且 CI 通过，可直接合并；也可在讨论中评估作者提出的替代方案（移除恒定的 fd_max 指标）以简化实现。

## Evidence

- **pr_diff:** PR 将 fd_allocated 和 fd_max 的解析与上报方式从 strconv.ParseFloat 改为直接传递字符串，避免大数精度丢失。
  - file: [plugins/input/systemv2/input_system_linux.go:162](https://github.com/alibaba/loongcollector/blob/2f363a2cffd36a43e3bb2da731d33edadce41454/plugins/input/systemv2/input_system_linux.go#L162)
  - sha: [2f363a2cffd3](https://github.com/alibaba/loongcollector/commit/2f363a2cffd36a43e3bb2da731d33edadce41454)
- **issue_context:** 作者指出 fd_max 固定为 max int64 (9223372036854775807)，超出 float64 精度范围，导致指标值失真。改为字符串上报是合理的修复路径。
  - file: [plugins/input/systemv2/input_system_linux.go:166](https://github.com/alibaba/loongcollector/blob/68ba314f5176e6101fbe03007efa871b84cb3e6a/plugins/input/systemv2/input_system_linux.go#L166)
