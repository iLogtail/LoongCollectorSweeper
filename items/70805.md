## Summary

Keep #70805 open. Current main still explicitly skips clean watch-mode runtime-postbuild sync, and the existing run-node test asserts that behavior. Main has adjacent loader/runtime-mirror improvements that cover some plugin-sdk import cases, including Telegram-style bundled plugin imports from an external stage dir, but those do not implement the PR's broader watch-mode missing-output and metadata freshness checks.

## Best Possible Solution

Keep the PR open for maintainer review/rebase. The best path is to preserve the current loader/runtime-mirror improvements, then land a focused watch-mode fix that makes `scripts/run-node.mjs` verify required runtime-postbuild outputs before launch instead of unconditionally skipping clean watch sync, with targeted tests for missing alias wrappers, dist-runtime overlays, package metadata freeze/recovery, skills/static assets, staged runtime deps, and the existing build-lock concurrency behavior.
