## Summary

Keep PR #70805 open. Current main still skips runtime-postbuild sync entirely for clean `OPENCLAW_WATCH_MODE=1` launches, and the current tests explicitly assert that behavior. The PR targets a real watch-mode/build-staging gap rather than obsolete Telegram behavior, and the related context does not identify a canonical superseding fix.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused fix in `scripts/run-node.mjs`, `scripts/runtime-postbuild.mjs`, `scripts/copy-bundled-plugin-metadata.mjs`, and `scripts/stage-bundled-plugin-runtime-deps.mjs` that validates required runtime-postbuild outputs in watch mode before launching, preserves/rebuilds package metadata safely, and keeps targeted regression coverage for missing outputs, staged dependency freshness, and runtime-postbuild lock serialization.
