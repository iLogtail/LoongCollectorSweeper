## Summary

Keep open. Current main has substantial macOS launchd hardening, but I did not find source or test evidence that the specific raw-SIGTERM LaunchAgent repro from #53475 is fixed. The gateway still treats an unmarked SIGTERM as a clean stop and relies on launchd KeepAlive to respawn it, which is exactly the path the reporter says failed with `domain in on-demand-only mode`.

## Best Possible Solution

Keep #53475 open and add a macOS launchd integration reproduction: install a test Gateway LaunchAgent, read the runtime PID, send SIGTERM to that PID, then assert `launchctl print` reaches `state = running` with a new PID and the health endpoint returns. If it still fails, patch the launchd service lifecycle or document the exact GUI-domain/session limitation; if it passes on current main, close later with that live/integration evidence.
