## Summary

Current `main` does not appear to fix this end-to-end. The gateway has support for preserving channel-scoped external routing, including Feishu, when `chat.send` is invoked with `deliver: true`, but the shipped Control UI chat path currently hardcodes `deliver: false`, and current tests explicitly expect that to keep replies on the internal surface instead of routing back to the original channel.

## Evidence
