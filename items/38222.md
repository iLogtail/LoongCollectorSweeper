## Summary

Keep open. Current main still does not enforce length limits on hook-injected prompt context. The hook merger, legacy-plus-phase join, and embedded runner injection paths remain unbounded, and current tests still cover concatenation/precedence rather than truncation. The PR remains a focused core hardening change rather than a conservative close candidate.

## Best Possible Solution

Keep the PR open and land or adapt a core implementation that clamps hook-injected prompt strings after per-hook merge and after the final legacy-plus-phase join, preserves model/provider override fields, logs truncation, and adds regression coverage for both `before_prompt_build` and legacy `before_agent_start`.
