## Summary

Current main still appears to have the reported sandbox write/edit permission regression. The shared pinned mutation helper creates replacement temp files with mode 0600 and `write_atomic` replaces the original without applying the original file mode. Sandboxed write and edit tools still delegate through this helper, and the related fix PR #44114 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open. The fix should land in `src/agents/sandbox/fs-bridge-mutation-helper.ts`: before overwriting an existing regular file, capture its mode, apply that mode to the temp fd before fsync/replace, and preserve the existing hardlink/symlink safety checks. Add focused regression coverage in `src/agents/sandbox/fs-bridge-mutation-helper.test.ts` for overwriting a 0644 file without collapsing it to 0600, and consider the same shared helper impact on remote sandbox writes.
