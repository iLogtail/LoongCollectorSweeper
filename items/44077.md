## Summary

Keep open. Current `main` still routes sandbox file-tool writes through an atomic replace helper that creates the temp file with mode `0o600` and replaces the destination without restoring the original file mode. The only permission-preservation logic on `main` is in the cross-device rename fallback, not the normal write/edit path described in the issue.
