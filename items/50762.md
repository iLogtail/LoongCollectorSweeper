## Summary

Keep #50762 open. Current main has useful adjacent pieces, including queued chat sends, live tool-output cards in chat, and a generic markdown/canvas side panel, but it does not implement the requested dedicated split chat plus live exec/log panel with live stdout/stderr, command status/duration/exit-code metadata, or live-bound sidebar state.

## Best Possible Solution

Keep this open as a Control UI feature request. The best implementation path is to build on the existing agent tool-event stream and chat sidebar, but make the sidebar stateful by exec toolCallId/runId, preserve exec details from the tool result/update payload, render stdout and stderr distinctly when available, show command/status/duration/exit code/session id, keep Queue/Stop behavior, and add focused UI tests for live updates, final history reload, and side-panel refresh behavior. If the current tool-event payload cannot carry those fields without losing details, add an additive gateway/agent event contract before wiring the UI.

## Evidence

That is broader than suppressing noisy exec messages or rendering ordinary tool cards.
