## Summary

Current main still appears to have the reported strict-agentic planning-only retry gaps. The issue is recent, unprotected, and externally authored, but it describes a real core runner behavior that is not implemented or superseded.

## Best Possible Solution

Keep the issue open. A conservative fix should update `src/agents/pi-embedded-runner/run/incomplete-turn.ts` and add regression tests in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts` for the reported non-allowlisted imperative prompt and for an accepted ack such as `go ahead` followed by present-continuous no-tool narration like `I'm running it now.` Maintainers should also decide whether bare `GO` belongs in the ack set while keeping casual chatter and completed-result summaries non-retryable.
