## Summary

Keep #71658 open. The provided issue discussion already identified the same gaps, and current main at 631552c554095b0c520376db9131dac0f5427f94 still has the narrow actionable-prompt gate and present-continuous planning narration miss.

## Best Possible Solution

Keep the issue open. Fix `resolvePlanningOnlyRetryInstruction` in `src/agents/pi-embedded-runner/run/incomplete-turn.ts` with focused regression tests for the reported non-allowlisted imperative prompt and for an accepted ack prompt followed by present-continuous no-tool narration such as `I'm running it now.`. Broaden the actionable-prompt and planning-only narration detection conservatively so casual chatter and completed-result summaries still do not retry, and make an explicit maintainer call on whether bare `GO` should be an ack.
