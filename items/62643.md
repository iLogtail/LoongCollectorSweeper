## Summary

Keep PR #62643 open. Current main still exposes and uses the `purpose: "status"` memory-runtime contract, but it does not document the default/status semantics that this PR adds for plugin authors. The PR remains a focused core SDK/status contract improvement, not a ClawHub-only plugin item and not something already implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or revise the `MemoryPluginRuntime` JSDoc and focused status tests so plugin authors have a clear core contract, while tracking any stronger runtime enforcement or live-gateway status redesign separately under the related memory-status issues.
