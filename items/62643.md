## Summary

Keep open. Current main already passes `purpose: "status"` through the status memory path, but it does not document the `MemoryPluginRuntime` status/default contract that this PR adds. The status path still loads the active memory runtime locally and probes the manager, so the PR remains a focused in-repo SDK/docs/test improvement rather than a ClawHub-only plugin item or an already-implemented change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or revise the core `MemoryPluginRuntime` contract documentation and focused status tests, then leave any runtime enforcement or live-gateway status redesign to a separate follow-up tied to #62599/#56968 if maintainers want stronger behavior than a documented plugin contract.
