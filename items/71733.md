## Summary

Keep #71733 open. Current main still has only adjacent prompt-hook and in-memory system-event surfaces; it does not implement the requested durable next-turn injection queue, public `enqueueNextTurnInjection` API, typed `agent_turn_prepare` hook, exactly-once dequeue, expiry, dedupe, or disabled-plugin suppression. The linked #72082 is an open draft implementation proposal, not merged/shipped current-main behavior.

## Best Possible Solution

Keep the issue open as a core/plugin-SDK RFC until the generic host-hook work lands. The best path is to review and finish #72082 or an equivalent focused implementation that adds durable plugin-owned next-turn injection storage, public enqueue API, typed `agent_turn_prepare`, retry-safe exactly-once consumption, expiry/dedupe/disablement diagnostics, and fixture tests proving next-run delivery without duplicating retries.
