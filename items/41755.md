## Summary

Keep open. Current `main` still clears Control UI active-run state on reconnect/page refresh, so the chat toolbar falls back to idle controls until another chat event arrives. I did not find an on-main fix for #41755.

## Evidence

- **Reconnect handler clears active run state:** On gateway hello, the Control UI explicitly nulls `chatRunId`, `chatStream`, and `chatStreamStartedAt` with a comment that any in-flight run is treated as orphaned after disconnect/reconnect.
  - file: [ui/src/ui/app-gateway.ts:315](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/app-gateway.ts#L315)

  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Chat refresh path reloads history on page load:** The chat tab refresh path calls `refreshChat(...)`, and `refreshChat(...)` always calls `loadChatHistory(...)` on reconnect/startup.
  - file: [ui/src/ui/app-settings.ts:384](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/app-settings.ts#L384)
