## Summary

Current `main` does not implement immediate standalone delivery of early assistant commentary on Discord. The core stream handler still suppresses `commentary`-phase block replies, Discord interim output is routed through the draft preview stream instead of a committed normal reply, and restart deferral only tracks registered reply dispatchers rather than draft-preview work. This issue should stay open.
