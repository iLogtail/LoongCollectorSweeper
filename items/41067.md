## Summary

Keep PR #41067 open. Current main still lacks the PR's core dashboard chat recovery behavior: hello snapshots do not expose active chat runs, the Control UI still clears chat run state on reconnect, the composer busy calculation still ignores abortability, and shutdown does not abort active chat runs with a durable transcript notice. The PR is still relevant, but it needs rebase/integration work and should address the remaining review edge cases before merge.

## Best Possible Solution

Keep the PR open. Rebase it onto current main, preserve the active-chat-run hello snapshot and UI reconciliation behavior, include `canAbort` in chat busy state, keep last-active chat navigation, and integrate shutdown/restart notices with the newer restart recovery paths. Before merge, address the remaining review issues by selecting the newest or currently tracked active run deterministically and catching transcript-injection failures so a system-event fallback still runs, then regenerate protocol artifacts and run the targeted gateway/UI/protocol gates.
