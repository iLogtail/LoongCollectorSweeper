## Summary

Keep open. Current `main` does not implement the PRâ€™s reconnect recovery and restart-notice behavior: the gateway hello snapshot still omits active chat runs, the Control UI still clears chat run state on reconnect, sidebar chat navigation still prefers the main session path, the chat composer busy state still ignores abortability, and gateway shutdown/restart paths still lack the proposed active-run abort and durable transcript notice handling.
