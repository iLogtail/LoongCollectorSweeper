## Summary

Keep open: current `main` still does not provide a separate Agents overview control/write path for the global `agents.defaults.model`, so this PR is addressing a real unshipped UI gap rather than something already implemented or obsolete.

## Evidence

- **Overview select still uses the per-agent model handler:** The Overview panel labels the default agent field as `Primary model (default)`, but the `<select>` still calls `onModelChange(agent.id, ...)`, the same callback used for agent overrides. There is no separate default-model callback in this view.

- **Current props expose no dedicated default-model write path:** `AgentsProps` includes `onModelChange` and `onModelFallbacksChange`, but no `onDefaultModelChange` or equivalent global-default setter on current `main`.
  - file: [ui/src/ui/views/agents.ts:107](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/ui/src/ui/views/agents.ts#L107)
