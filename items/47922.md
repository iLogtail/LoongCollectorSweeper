## Summary

Keep PR #47922 open. Current main does not implement the proposed `messages.outbound.text.emDash` config or shared outbound em-dash transform, and the PR is a focused, plausible feature rather than an obsolete duplicate. It should not be auto-closed as ClawHub yet because existing plugin hooks can rewrite outbound content, but they do not clearly cover the PR’s exact final-pass/mirror-normalization goals without maintainer product judgment. The PR also has concrete unresolved review findings that need author or maintainer handling before merge.

## Best Possible Solution

Keep the PR open for maintainer/product decision. If core-owned outbound style normalization is desired, revise the PR to fix the leading/trailing comma defects, add explicit tests for leading em dashes, trailing commas without em dashes, post-hook output, direct/gateway sends, and transcript mirror text, and document hook ordering. If maintainers prefer the plugin path, close only with an explicit ClawHub/plugin direction and, if needed, open a narrower plugin API issue for a true final outbound text-transform stage.
