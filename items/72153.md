## Summary

Keep #72153 open. Current main still builds the Windows dashboard config opener as `Start-Process -LiteralPath ...`, while PowerShell's installed `Start-Process` command exposes `FilePath` and no `LiteralPath` parameter. The linked #69933 Windows log, the PR patch, and the Greptile/QC comments all align with a focused real bug fix that has not landed on main.

## Best Possible Solution

Keep the PR open for maintainer review and merge, or land the same minimal fix on main: change the Windows branch of `resolveConfigOpenCommand()` to use `Start-Process -FilePath ...`, preserve the existing single-quote escaping, and update `src/gateway/server-methods/config.test.ts` to assert `-FilePath`.
