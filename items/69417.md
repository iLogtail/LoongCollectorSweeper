## Summary

Keep PR #69417 open. Current main still does not support SecretRef/SecretInput values on the core `mcp.servers.*.env.*` or `mcp.servers.*.headers.*` surfaces. Existing shipped SecretRef MCP support is for the `acpx` plugin config path, and related closed items cover different behavior such as preserving MCP env vars or forwarding static headers. The PR discussion also shows active review issues were addressed on the PR branch, so this remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the likely implementation path for core `mcp.servers` SecretRef support. The best maintainer path is to land a narrowed, green version that updates the core schema/types, SecretRef target registry, command/runtime secret resolution for active MCP transports, `secrets configure`/`secrets audit` behavior, generated docs/baselines, and regression tests. If maintainers choose not to land this branch wholesale, the same behavior still needs an equivalent core implementation; existing ACPX plugin SecretRef support and static MCP env/header fixes do not cover it.
