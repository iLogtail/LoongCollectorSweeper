## Summary

Keep open. Current main has safeguards around `delivery-mirror` replay into model context, but I do not see a source- or test-backed fix that rules out the reported user-visible outbound delivery. The `openclaw.cache-ttl` entry appears to be a passive transcript/context-pruning marker, while the remaining plausible path is stale pending outbound delivery recovery plus transcript mirroring, which current main still intentionally performs.

## Best Possible Solution

Keep the issue open and investigate the outbound delivery durability boundary: add regression coverage that a cache-ttl custom entry never initiates delivery, then decide whether recovered outbound queue entries need a max-age, stronger idempotency/source metadata, or operator-visible audit gating before sending user-visible payloads after long idle gaps.
