## Summary

Keep #71185 open. Current main at c74fb781 still has the consumer-side crash surface this PR addresses: trajectory metadata maps every `resolvedSkills` entry directly, redacts `filePath`/`baseDir` through a string-only redactor, and sorts with `left.name.localeCompare(...)` without validating partial entries. The PR is a recent focused contributor bug fix with targeted regression tests in the provided diff and positive bot review, so no conservative close reason applies.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this defensive consumer fix with regression tests, or replace it with an equivalent producer-plus-consumer hardening patch that preserves support bundle export stability and the existing symlink-escape security behavior.
