## Summary

Keep PR #71185 open. Current main still has the pre-PR support-capture behavior: `buildSkillsCapture` trusts every `resolvedSkills` entry and sorts via `left.name.localeCompare(...)`, while `redactPathForSupport` still accepts only `string` and immediately calls `startsWith`. The PR remains a focused defensive fix with regression tests, and the supplied related items do not supersede this trajectory support-capture issue.

## Best Possible Solution

Keep PR #71185 open for maintainer review. The best path is to land or refine the focused consumer-side hardening in `src/trajectory/metadata.ts` and `src/logging/diagnostic-support-redaction.ts` with the proposed regression coverage in `src/trajectory/metadata.test.ts`; any producer-side cleanup in `src/agents/skills/workspace.ts` can remain a separate follow-up if maintainers want to eliminate partial entries at the source.
