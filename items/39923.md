## Summary

Current `main` still implements numbered config-backup rotation (`.bak`, `.bak.1` ... `.bak.4`) rather than datetime-suffixed backups, and the regression tests still assert that numbered ring behavior. The issue remains a valid, unimplemented feature request. Timeline evidence also shows two linked PRs are still open and the one closed PR was not merged, so there is no strong basis for conservative auto-close cleanup.
