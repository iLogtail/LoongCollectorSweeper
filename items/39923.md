## Summary

Keep #39923 open. Current main still uses the numbered `.bak` / `.bak.1` through `.bak.4` config backup ring, cleanup still deletes non-numbered `.bak.*` suffixes including timestamp-shaped files, and tests assert that behavior. The related PRs #39982 and #40392 are open implementation candidates, while #46353 is closed unmerged, so they are useful prior art but not shipped behavior.

## Best Possible Solution

Keep #39923 open until a datetime-suffix config backup implementation lands or maintainers explicitly decide to retain the numbered backup ring. A complete fix belongs in `src/config/backup-rotation.ts` with matching `src/config/config.backup-rotation.test.ts` coverage for bounded pruning, chmod discovery, collision handling, and legacy `.bak.N` migration; open PRs #39982 and #40392 can remain implementation candidates.
