## Summary

Keep #39923 open. Current main still uses the numbered config backup ring (`.bak`, `.bak.1` through `.bak.4`) rather than datetime-suffixed config backups, and current tests still assert that numbered behavior. The related implementation PRs in the provided GitHub context are open or closed unmerged, so the requested behavior has not shipped.

## Best Possible Solution

Keep the issue open until a datetime-suffix implementation lands or maintainers explicitly decide to preserve the numbered backup ring. A correct fix should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts`, with deliberate coverage for bounded pruning, chmod discovery, collision behavior, and legacy `.bak.N` migration.
