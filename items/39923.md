## Summary

Current `main` still uses the numeric config-backup rotation ring (`.bak`, `.bak.1` ... `.bak.4`) and prunes non-numeric `.bak.*` files, so the requested datetime-suffixed backup scheme is not implemented.

## Evidence

- **Rotation logic is still numeric:** `rotateConfigBackups` renames `openclaw.json.bak` to `.bak.1` and shifts numbered slots up to `.bak.4`; it does not create timestamped backup names.
  - file: [src/config/backup-rotation.ts:16](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/config/backup-rotation.ts#L16)
