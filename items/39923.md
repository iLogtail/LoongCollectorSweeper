## Summary

Keep #39923 open. Current main still implements the numbered config backup ring (`.bak`, `.bak.1` through `.bak.4`) and the current tests still assert that behavior, so the datetime-suffix feature remains unimplemented. The related PR context does not provide a landed replacement: #39982 and #40392 are open, while #46353 was closed unmerged.

## Best Possible Solution

Keep the issue open as the canonical feature request unless a maintainer decides to preserve numeric rotation. A suitable fix belongs in `src/config/backup-rotation.ts` with updated colocated tests for timestamp formatting, bounded retention, permission hardening, orphan cleanup, collision handling, and legacy numeric backup migration/preservation behavior.
