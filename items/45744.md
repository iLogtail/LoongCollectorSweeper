## Summary

Keep open. Current `main` still skips symlinked plugin directories during auto-discovery, the report is corroborated by later discussion, and main still lacks the top-level symlink coverage that reviewers asked for. This is not implemented on `main`, and the PR still contains useful work despite branch drift.

## Evidence

- **Current discovery loop still drops non-directory dirents before package scanning:** In `discoverInDirectory`, file entries are handled first, then any entry where `entry.isDirectory()` is false is immediately skipped. A symlinked plugin directory therefore never reaches the package-manifest discovery path on current `main`.
  - file: [src/plugins/discovery.ts:873](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/discovery.ts#L873)
  - command: `nl -ba src/plugins/discovery.ts | sed -n '821,900p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **The affected path is the documented global plugin auto-discovery path:** `discoverOpenClawPlugins` still calls `discoverInDirectory` for `roots.global`, and the docs describe global plugin scanning under `~/.openclaw/<plugin-root>/*/index.ts`. That matches the PR's reported `~/.openclaw/extensions/...` use case.
  - file: [src/plugins/discovery.ts:1199](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/discovery.ts#L1199)
  - command: `nl -ba src/plugins/discovery.ts | sed -n '1180,1238p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Main's existing symlink test covers a different case:** The only current symlink-focused discovery test on `main` is `rejects package extension entries that escape via symlink`, which exercises a symlink inside a package directory. It does not cover a top-level symlinked plugin directory in `extensions/`, which is the behavior this PR is trying to fix.
  - file: [src/plugins/discovery.test.ts:813](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/discovery.test.ts#L813)
  - command: `nl -ba src/plugins/discovery.test.ts | sed -n '790,860p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **PR discussion provides concrete corroboration and a test follow-up:** Greptile's review called out the missing tests for top-level symlink discovery and broken symlink handling. On 2026-03-29, Dithilli reported hitting the exact bug with a symlinked `openclaw-affect` plugin and supplied a two-test diff plus a branch/commit (`6ed753c9d3`) to cover both scenarios.
  - sha: [6ed753c9d3](https://github.com/openclaw/openclaw/commit/6ed753c9d3)
