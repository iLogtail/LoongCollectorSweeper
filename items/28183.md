## Summary

Avian is a reasonable optional OpenAI-compatible LLM provider, but current OpenClaw intentionally routes service-specific provider additions through provider plugins and ClawHub/npm distribution unless maintainers explicitly promote them into the bundled set. The PR discussion also shows the contributor already adapted the work into a provider-plugin shape, and current main exposes the auth, onboarding, catalog, install, and publish surfaces needed outside core.

## Best Possible Solution

Close the OpenClaw core PR as a ClawHub/plugin-distribution fit. The Avian work should be published and maintained as an external provider plugin using the existing provider SDK and ClawHub/npm install path; official bundled status can be revisited only through an explicit maintainer-owned promotion decision.
