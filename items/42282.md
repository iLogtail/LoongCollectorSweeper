## Summary

Current main does not implement the requested streaming backup pipeline for #42282. The core backup path still writes through `tar.c({ file: tempArchivePath, gzip: true })`, there is no progress or ENOMEM-specific user guidance, and the related implementation PRs remain open or closed-unmerged. The external keepmyclaw plugin linked in the discussion is a useful workaround, but it does not replace the core `openclaw backup create` request.
