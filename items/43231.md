## Summary

Keep #43231 open. Current main has partial Telegram forum-topic support: it caches topic_id -> topic_name and exposes topic_name in prompt/plugin metadata, but it still does not provide deterministic topic-name -> topic_id resolution, a list-topics/tool action, or a topic catalog in Telegram directory data. The linked PR #65973 should be treated as useful partial progress, not a full superseding fix.

## Best Possible Solution

Keep the issue open and implement a Telegram-owned best-effort topic catalog over the existing persisted topic-name cache: expose a list/lookup action or metadata endpoint scoped to the current chat, support normalized name matching with ambiguity handling, keep numeric topic IDs as the authoritative send/edit target, and add docs/tests that explain the Telegram API limitation and cache freshness behavior.
