## Summary

Current main still does not implement configurable `session-memory` output directories or filename templates. The bundled handler writes under a hardcoded `<workspace>/memory` root and creates `${dateStr}-${slug}.md`; docs still describe only `messages` as configurable and document the fixed `<workspace>/memory/YYYY-MM-DD-slug.md` output. This remains a valid bundled-hook enhancement, not an implemented or duplicate cleanup close.

## Best Possible Solution

Keep the issue open. The best implementation path is a small bundled-hook enhancement: add typed/validated `session-memory` config for relative output path and filename templates, keep the current flat `memory/YYYY-MM-DD-slug.md` behavior as default, update `HOOK.md` and public hook docs, and add focused tests for default behavior, custom nested directories, placeholder rendering, and path traversal rejection. If maintainers prefer this as optional policy rather than bundled behavior, document an explicit hook-pack/plugin replacement path instead of closing this as already handled.
