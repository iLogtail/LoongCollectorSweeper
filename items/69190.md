## Summary

Keep open. Current main has documented hard tool policy and many browser-timeout fixes since the reporter's 2026.4.2 install, but this issue cannot be closed from current evidence: the target URL/logs are missing, and the tool-restriction side overlaps an unresolved current-main bug (#47487) where `tools.exec` can still re-expose `exec`/`process` through profile `alsoAllow`. Disabling `web_fetch` alone would not prevent `curl` if `exec` remains available.

## Best Possible Solution

Keep the issue open and ask for a retest on the latest release with the real URL, sanitized config, browser profile/driver, `openclaw browser doctor --deep` output, and logs around the timeout. For the curl bypass, either route the reporter to a hard policy example (`tools.allow: ["browser"]` or deny `exec`/`process`/`web_fetch`) or fix the unresolved #47487 behavior so global `tools.exec` configuration cannot silently re-enable runtime tools when a profile/allowlist was meant to exclude them. If the browser timeout reproduces on current main, track it separately with the affected profile/action path.
