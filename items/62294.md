## Summary

Keep open. Current main still contains the reported heartbeat scheduler behavior and the macOS launchd plist still lacks the proposed App Nap mitigation. The linked follow-ups are useful, but they are open and unmerged, and no single canonical follow-up fully supersedes this original report with its reproduction and two-part diagnosis.

## Best Possible Solution

Keep #62294 open as the original bug report until maintainers either land an equivalent scheduler fix plus a macOS launchd/App Nap decision, or explicitly mark #62310 and #62308/#66992 as the canonical tracking items. The likely code work is in `src/infra/heartbeat-runner.ts` for non-interval minimum spacing/schedule advancement and `src/daemon/launchd-plist.ts` plus launchd tests for any accepted plist policy change.
