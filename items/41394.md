## Summary

Keep open. Current `main` has strict config-write validation plus recovery/doctor repair paths, but it does not implement the requested configurable strict-vs-tolerant validation modes or tolerant unknown-key stripping during normal startup/read flows.

## Evidence

- **Config writes stay strict:** `config.set` validates the submitted full config before write, and `config.patch` refuses to run against an invalid snapshot and rejects merged configs that fail validation.
  - file: [src/gateway/server-methods/config.ts:452](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/gateway/server-methods/config.ts#L452)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Startup still requires a valid snapshot:** Gateway startup only tries last-known-good recovery or non-JSON-prefix repair, then calls `assertValidGatewayStartupConfigSnapshot`; it does not strip unknown keys and continue in a tolerant mode.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/gateway/server-startup-config.ts#L71)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Docs describe fail-closed startup behavior:** The docs explicitly say validation failures prevent boot, and recovery is via last-known-good restore plus `openclaw doctor --fix`, not a tolerant read mode.
  - file: [docs/gateway/configuration.md:80](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/gateway/configuration.md#L80)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Unknown-key stripping is a doctor repair path:** `doctor --fix` is documented to drop unknown config keys, and the doctor flow removes unknown keys as a repair step. That is separate from normal gateway read/startup behavior.
  - file: [docs/cli/doctor.md:43](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/cli/doctor.md#L43)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **No validation-mode config surface found:** Search for the proposed config knobs (`writeMode`, `readMode`, `unknownKeyLogLevel`, or a top-level `validation` block) returned no matches in `src/config`, `src/gateway`, or `docs`.
  - command: `rg -n "writeMode|readMode|unknownKeyLogLevel|validation\s*:\s*\{" src/config src/gateway docs || true`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
