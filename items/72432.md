## Summary

Keep #72432 open. Current main does not implement the proposed Matrix `com.openclaw.approval` payload, and the PR is a focused Matrix-plugin enhancement rather than obsolete or outside scope. However, the supplied PR diff and review comments show the current patch is not safe to close or merge as-is because it routes default approval delivery through the single-event sender, which current main documents and tests as throwing for oversized text instead of preserving the existing chunked Matrix approval prompt behavior.

## Best Possible Solution

Keep the PR open and ask for a revision that preserves existing chunked approval delivery while adding structured metadata to the primary Matrix `m.room.message` event. A good fix likely extends the Matrix send transport or approval delivery helper so the first chunk can receive `extraContent`, keeps follow-up chunks for long command text, updates the injected test seam to exercise metadata, and adds Matrix approval-handler tests for both structured metadata and long-command fallback behavior.
