## Summary

Keep open. Current `main` does not contain the abort-time native-command guard from PR #55084 or an equivalent teardown change, so this is not implemented on main and the duplicate-interaction race remains plausible.

## Evidence

  - file: [extensions/discord/src/monitor/provider.ts:823](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/provider.ts#L823)

- **Provider cleanup still happens only in `finally`:** Current cleanup sets disconnected status, disposes the supervisor, and stops thread bindings in `finally`, but there is no `nativeCommandsActive` flag, abort listener, or early deactivation path before `waitForDiscordGatewayStop()` completes.
  - file: [extensions/discord/src/monitor/provider.ts:1136](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/provider.ts#L1136)
