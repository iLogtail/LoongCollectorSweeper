## Summary

Keep PR #70790 open. Current main still does not have the general embedded-runner payload sanitizer proposed by the PR. The related toolsAllow forwarding has landed, but the remaining empty `tools: []` and orphaned `tool_choice` provider-payload cleanup is still missing on the embedded stream paths, and the PR discussion includes fresh active-memory impact.

## Best Possible Solution

Keep the PR open for maintainer review or a successor patch. The best path is a narrow embedded-runner/provider-payload sanitizer that strips empty `tools` arrays and orphaned `tool_choice` at the egress boundary, composes with existing payload wrappers, preserves non-empty tools, and avoids duplicating the already-landed toolsAllow forwarding work.
