## Summary

Keep PR #70790 open. Current main has related toolsAllow forwarding and some narrower safeguards, but it does not implement the PR's general embedded-runner payload guard. Embedded stream resolution still returns session/boundary/custom stream functions without stripping `tools: []` or orphaned `tool_choice`, while OpenAI-compatible HTTP payload construction can still emit `tools: []` for tool-history replay. The latest PR discussion also shows active user impact for active-memory, so this is not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or a successor patch. The best path is a narrow, tested embedded-runner/OpenAI-compatible payload sanitizer that strips `tools: []` and orphaned `tool_choice` at egress, preserves non-empty tools, composes with existing payload hooks, and avoids duplicating the already-landed toolsAllow forwarding work.
