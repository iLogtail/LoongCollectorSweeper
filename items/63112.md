## Summary

Current `main` does not contain this warning logic. The add/edit CLI paths still accept main-session `--system-event` payloads without any shell-command heuristic or stderr warning, while the runtime still treats main-session cron jobs as queued system events. The PR remains a live UX fix, not redundant cleanup.
