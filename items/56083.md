## Summary

Keep open. Current main still has the cron direct-delivery gap this PR targets: `dispatchCronDelivery` builds `payloadsForDelivery` from cron payloads or `synthesizedText`, only normalizes silent-reply tokens, and sends those payloads to `deliverOutboundPayloads` without resolving or applying `messages.responsePrefix`. Normal reply routing does apply the response-prefix path through `resolveEffectiveMessagesConfig` and `normalizeReplyPayload`, and the PR discussion shows the author addressed review feedback around boundary-aware idempotency and trailing-whitespace prefixes. The PR likely needs rebasing onto the newer delivery-dispatch shape, but the work is not obsolete.

## Best Possible Solution

Keep the PR open and rebase/refine it against current main. The best code path is to resolve the effective response prefix for the cron delivery target in `deliverViaDirect`, apply it to text payloads after silent-token normalization and before `deliverOutboundPayloads`, preserve media-only payloads, avoid double-prefixing and double spaces, and add focused regression coverage under `src/cron/isolated-agent`.
