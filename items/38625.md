## Summary

Keep open. Current main still only partially overlaps the request: the gateway exposes runtime version in WebSocket `hello-ok` and has an admin-scoped `update.run` WebSocket RPC, but `GET /health`/`/healthz` still returns only liveness status and there is no core `POST /admin/update` HTTP endpoint or `target_version` parameter.

## Best Possible Solution

Keep the issue open until maintainers either implement the requested HTTP contract or explicitly choose a WS-only product direction. A conservative fix would add a source-backed `version` field to the shallow `/health` and `/healthz` responses with tests/docs, then decide whether `POST /admin/update` should be a thin authenticated HTTP wrapper around `update.run`. If `target_version` is required, extend `UpdateRunParamsSchema` and the update runner contract instead of adding an ad hoc host-package path.
