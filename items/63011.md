## Summary

Keep open. Current `main` still has the split the issue describes: direct `message send` can be satisfied by channel-owned action handlers that bypass the shared outbound hook path, while gateway `send` still turns an empty delivery result into a generic `No delivery result` error instead of an explicit guarded denial. The shared `message_sending` hook path also does not supply the documented session/run correlation fields on outbound sends.

## Evidence

- **Direct CLI send still short-circuits into plugin-owned action handlers before core outbound delivery:** `executeSendAction()` first calls `tryHandleWithPluginAction()` and returns immediately when a channel plugin handles `send`, so the shared `sendMessage()` / `deliverOutboundPayloads()` path is skipped for those channels.
  - file: [src/infra/outbound/outbound-send-service.ts:143](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/infra/outbound/outbound-send-service.ts#L143)

  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Telegram send remains a concrete bypass of the shared hook-bearing path:** Telegram exposes `handleAction()` for `send`, and its runtime executes `sendMessageTelegram()` directly. That block returns a delivery payload but does not call `message_sending`, which matches the issue's direct-send concern for Telegram.
  - file: [extensions/telegram/src/action-runtime.ts:308](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/telegram/src/action-runtime.ts#L308)

- **Shared outbound delivery still treats hook cancellation as 'no result':** In the shared outbound path, `applyMessageSendingHook()` can mark a payload cancelled and the loop just `continue`s without appending a delivery result. A fully cancelled send therefore produces an empty `results` array.
  - file: [src/infra/outbound/deliver.ts:851](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/infra/outbound/deliver.ts#L851)

  - file: [src/gateway/server-methods/send.ts:531](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/gateway/server-methods/send.ts#L531)

Current outbound hook calls do not consistently meet that contract.
