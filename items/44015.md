## Summary

Keep open. Current main does not implement the requested `chat.send` 512 KB message cap: the protocol schema still accepts an unbounded string, the sanitizer only normalizes/rejects null bytes/strips control characters, and the post-auth transport cap is 25 MB. The PR remains a focused gateway hardening change, but should address the posted review concerns before merge.

## Best Possible Solution

Keep the PR open and revise it: enforce a byte cap in the `chat.send` server path before expensive downstream work, clarify or adjust the schema-level guard so maintainers do not treat character `maxLength` as byte enforcement, add ASCII boundary/multibyte/normalization-shrink tests, and document the final limit in the gateway protocol docs.
