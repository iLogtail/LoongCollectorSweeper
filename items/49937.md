## Summary

Keep open. Current `main` still has the hardcoded 120s remote non-local embedding batch timeout, so this is not implemented on `main`, and the PR discussion already identified unresolved regressions in the proposed fix path.

## Evidence

- **Current main still hardcodes 120s for remote batch embedding calls:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:307` returns `EMBEDDING_BATCH_TIMEOUT_REMOTE_MS` for remote `"batch"` requests, and the call sites at `extensions/memory-core/src/memory/manager-embedding-ops.ts:244` and `:276` still wrap `provider.embedBatch(...)` / `embedBatchInputs(...)` with that timeout. That matches the bug report rather than proving it fixed on `main`.
