## Summary

Current `main` only partially supersedes this PR: `## Group Chat Context` now lands after project context, but `## Messaging` and `## Reactions` still sit above `# Project Context` and above the cache boundary. The underlying cache-stability improvement is therefore not implemented on `main`, so this PR should stay open even though it now needs a rebase against the newer system-prompt/cache-boundary work.

## Evidence

- **Current prompt order still keeps dynamic channel sections before project context:** `buildAgentSystemPrompt()` appends `## Workspace Files (injected)`, `## Messaging`, and later `## Reactions` before it appends `# Project Context`. `extraSystemPrompt`/`## Group Chat Context` is appended later, so `main` only partially reflects the PR's intended reorder.
  - file: [src/agents/system-prompt.ts:862](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/system-prompt.ts#L862)
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Messaging guidance is runtime-variant:** `buildMessagingSection()` depends on available tools, `runtimeChannel`, inline-button capability, and optional message-tool hints. That means it is not byte-stable across channels/agents, so keeping it ahead of project context still perturbs the cacheable prefix.
  - file: [src/agents/system-prompt.ts:335](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/system-prompt.ts#L335)
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Reaction guidance is runtime-variant too:** `## Reactions` is built from `reactionGuidance.level` and `.channel`, so it also changes across channel/config combinations while remaining above the cache-boundary marker on `main`.
  - file: [src/agents/system-prompt.ts:882](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/system-prompt.ts#L882)
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Cache shaping only protects the prefix above the boundary:** Anthropic payload shaping splits the system prompt at `SYSTEM_PROMPT_CACHE_BOUNDARY` and marks only the `stablePrefix` portion as cacheable. With the boundary inserted later in `buildAgentSystemPrompt()`, everything before that marker remains part of the cache key.
  - file: [src/agents/anthropic-payload-policy.ts:89](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/anthropic-payload-policy.ts#L89)
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Docs still describe moving volatile content below the boundary:** The prompt-caching docs say content above the boundary should stay byte-identical across turns and explicitly recommend moving volatile content below the boundary when cache churn appears. That matches the remaining gap this PR is trying to close.
  - file: [docs/reference/prompt-caching.md:167](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/docs/reference/prompt-caching.md#L167)
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Main lacks the ordering regression guard proposed by the PR:** Current tests cover project-context presence/absence plus separate group-chat/reaction presence, but there is no assertion that `# Project Context` precedes `## Messaging` or `## Reactions` on `main`.
  - file: [src/agents/system-prompt.test.ts:578](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/system-prompt.test.ts#L578)
