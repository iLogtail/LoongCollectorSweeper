## Summary

Keep open. Current `main` at `a6aa151653ab478e9aa23e95c6446c41adc4fce1` does not implement this PR's literal per-agent store-path fix: `loadCombinedSessionStoreForGateway` still treats every non-template `session.store` as a single-store fallback for the default agent, so a literal path like `agents/main/sessions/sessions.json` will not discover sibling agent stores.

## Evidence
