## Summary

Keep PR #54716 open. Current `main` still treats any literal non-template `session.store` as a single default-agent store, so a literal `.../agents/main/sessions/sessions.json` path does not fall through to multi-agent discovery. The PR’s central bug fix is therefore not implemented or obsolete, and the linked bug #54435 remains a plausible current behavior gap.

## Best Possible Solution

Keep the PR or its linked bug open and adapt the fix into `src/config/sessions/combined-store-gateway.ts`: detect literal paths that match `agents/<id>/sessions/sessions.json`, route those through scoped multi-agent discovery for the same agents root, preserve the resolved literal path for downstream lookups, canonicalize each store using the agent id derived from its on-disk path, and add focused regressions for #54435 plus the leakage and config-order cases called out in review.
