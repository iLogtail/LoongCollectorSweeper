## Summary

Keep PR #67227 open. Current main has provider request proxy schema/reference coverage, but it still does not contain the user-facing model-provider clarification this PR adds: provider requests do not automatically honor HTTP_PROXY/HTTPS_PROXY, env proxying requires explicit request.proxy.mode, provider entries must satisfy the current schema, and support is limited to transport-aware provider APIs. The PR also already responded to the bot review by moving the section and narrowing the provider-scope claim.

## Best Possible Solution

Keep the PR open and update it against current main. The best path is a concise docs addition on the model-provider/custom-provider page that says provider requests do not automatically inherit HTTP_PROXY/HTTPS_PROXY, shows request.proxy.mode: "env-proxy" in a complete schema-valid provider entry, and clearly scopes support to transport-aware APIs listed in src/agents/provider-transport-stream.ts.
