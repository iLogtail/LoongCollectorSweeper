## Summary

Keep open. Current main still does not implement this PR's core hardening: both uncaughtException handlers remain fatal, the gateway upgrade handler lacks the proposed raw-socket error listener, and there is no strict uncaught-exception transient classifier. The PR also has a substantive unresolved review concern around suppressing EPIPE, so the right maintainer action is revision or replacement, not cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with an updated patch. The best fix should add shared uncaught-exception classification for both entrypoints, preserve fatal/config error precedence, use a stricter network/TLS allowlist that excludes EPIPE and broad fetch/network strings, add the gateway upgrade socket error listener with logging, and include regression coverage for both fatal precedence and transient socket/TLS behavior.
