## Summary

Keep #68502 open. Current main has Microsoft Foundry/Azure OpenAI implementation support, but it still lacks a dedicated user-facing Azure AI Foundry or Microsoft Foundry provider guide in the provider docs. The PR is not safe to close as implemented or superseded; it likely needs revision to align with the current `microsoft-foundry` plugin and current API/auth paths before merge.

## Best Possible Solution

Keep the PR open, but ask for a revision against current main: add or retarget a provider guide for the bundled `microsoft-foundry` plugin, link it from `docs/providers/index.md`, document Entra ID and API-key onboarding, and clearly separate any manually configured Claude-on-Foundry `/anthropic/` path if it is still supported and verified.
