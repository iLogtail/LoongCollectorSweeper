## Summary

Keep #14079 open. Current main still does not implement Linux desktop delivery for the existing `system.notify` / `openclaw nodes notify` surface: the Linux/headless node host does not advertise `system.notify`, the node-host dispatcher has no notify handler, docs and CLI still frame notify as macOS-node behavior, and plugin node-host commands cannot replace the reserved core `system.notify` command. The related #24616 only addressed the unsupported-command error experience, not notification delivery.

## Best Possible Solution

Keep the issue open. The likely product fix belongs in the Linux/headless node-host path: advertise `system.notify` only when the host can support it, implement `notify-send` with a D-Bus/libnotify fallback and clean headless degradation, add focused node-host/gateway/CLI tests, and update the nodes/CLI docs. If maintainers prefer this outside core, first add an explicit plugin API that can safely extend or own the reserved `system.notify` surface.
