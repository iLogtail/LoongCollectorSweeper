## Summary

Keep open. Current `main` does not implement this: the Telegram channel is a bot-only grammY integration, the inbound context builder exposes sender/chat/message metadata but no device/client field, and the Telegram config schema has no `exposeDeviceHint` toggle. The issue's suggested data source (`api_id` / `application`) does not line up with the current bot-based adapter, but the request is still understandable and not strong enough to close as `incoherent` or `clawhub`.
