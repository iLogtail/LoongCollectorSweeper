## Summary

Keep open. Current main still ignores OPENCLAW_STATE_DIR for the default agent workspace path, so this PR is not obsolete. The submitted patch is directionally useful but needs revision because the proposed raw env handling would not expand `~`; the best fix is to derive the workspace base from the existing state-dir resolver and add focused tests for state-dir, tilde expansion, and OPENCLAW_PROFILE behavior.

## Best Possible Solution

Keep the PR open but request changes: implement the default workspace base by reusing `resolveStateDir(env, homedir)` or equivalent `resolveUserPath` normalization, preserve existing OPENCLAW_HOME fallback semantics when no state dir is set, and add tests for unset state dir, absolute state dir, `~` state dir, and OPENCLAW_PROFILE suffixing under a custom state dir.
