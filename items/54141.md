## Summary

Keep open. Current main still has the auth-profile audit false positive this PR targets: bare `$VAR` strings are not parsed as env SecretRefs, and auth profile credential values are still reported as plaintext even when `resolveSecretInputRef()` finds an inline ref. The PR is a focused bug fix, has recent author follow-up, and the review discussion already identified and addressed the main false-negative risk for explicit `keyRef` plus leftover plaintext.

## Best Possible Solution

Keep the PR open and have maintainers review or land a rebased version that adds `$VAR` parsing, skips `PLAINTEXT_FOUND` only for inline SecretRefs, preserves warnings/findings when explicit `keyRef` or `tokenRef` coexists with leftover plaintext, and adds the auth-profile audit regression tests proposed in the PR.
