## Summary

Current `main` still keeps every dynamically created command lane in the global `lanes` Map with no production eviction path. The reported leak remains plausible for `session:*` and `auth-probe:*` lanes, so this PR should stay open for maintainer review rather than be auto-closed.

## Risks / Open Questions

- Long-running gateways can accumulate idle `LaneState` objects for unique `session:*` and `auth-probe:*` names with no current reclamation path.
- The PR may still need design review around preserving explicit per-lane concurrency settings after pruning; that review concern should be resolved by maintainers, not auto-closed.
