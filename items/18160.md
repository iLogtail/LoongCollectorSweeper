## Summary

Keep #18160 open. Current main still has no direct `exec`/`shell` cron payload: cron types, Gateway protocol schema, validation, runtime dispatch, CLI, UI types, and docs remain centered on `systemEvent` and `agentTurn`. The issue has concrete production-use evidence, a closed/unmerged implementation PR (#51276), and the latest maintainer/Codex discussion already treats #18160 as the substantive core tracker rather than a duplicate.

## Best Possible Solution

Keep #18160 open as the core cron direct-exec tracker. The best implementation path is an additive, versioned `exec`/`shell` payload contract across `src/cron/types.ts`, Gateway schema/validation, cron job normalization and runtime dispatch, CLI add/edit, Control UI, docs, and focused tests. It should use a controlled process runner with timeout/abort, stdout/stderr capture, run-log/state updates, retry/failure handling, channel/webhook delivery semantics, and security behavior aligned with existing exec approvals and sandbox policy.
