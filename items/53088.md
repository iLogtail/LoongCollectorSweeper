## Summary

Current main still renders the Control UI Sessions table Updated column as a relative timestamp, not a full date/time. The linked PR #53445 is relevant and appears to implement the requested `YYYY-MM-DD HH:mm` display, but it is open/unmerged in the provided GitHub context, so the issue should remain open until that PR or an equivalent change lands.

## Best Possible Solution

Keep #53088 open and resolve it by reviewing/landing PR #53445 or an equivalent focused UI change: add a deterministic full local date-time formatter, render it in `ui/src/ui/views/sessions.ts` for the Updated column, preserve useful relative-time context in a tooltip if desired, and add targeted tests in `ui/src/ui/format.test.ts` and/or `ui/src/ui/views/sessions.test.ts`.
