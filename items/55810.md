## Summary

Keep open. Current main does not implement `sandbox.docker.workspaceMountPropagation`: there is no config type/schema field, no resolver wiring, and `appendWorkspaceMountArgs` always emits Docker bind specs without propagation mode. The PR is still a coherent, scoped feature request, but it should address the remaining browser-container propagation gap before merge.

## Best Possible Solution

Keep the PR open and ask for a small follow-up on the branch: add the config field/type/schema/generated schema, thread it through `resolveSandboxDockerConfig`, apply it to both main and browser workspace mounts, update config-hash/recreate and workspace-mount tests, and document the Docker propagation behavior and host shared-mount prerequisite.
