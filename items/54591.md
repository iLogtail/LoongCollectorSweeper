## Summary

Keep #54591 open. Current main still enforces the sandboxed-requester to unsandboxed-child block, exposes no `dangerouslyAllowUnsandboxedSubagentSpawn` config/runtime/schema surface, and documents that `sessions_spawn` rejects unsandboxed targets from sandboxed requester sessions. The PR is a concrete but security-sensitive core behavior change, not cleanup-ready or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the best implementation is a core change that threads a requester-owned dangerous flag through typed config, schema/help/generated metadata, sandbox runtime status, and `spawnSubagentDirect`, while preserving `sandbox="require"` as a hard gate and adding focused regression tests for default block, per-agent/default opt-in, allowlist interaction, and `sandbox="require"` behavior.
