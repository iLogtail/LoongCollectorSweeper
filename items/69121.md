## Summary

Keep #69121 open. Current main still has only pre-restart deferral for queued work and pending replies; it does not include the PR's second-stage bounded drain inside `server.close()`, remaining restart budget handoff, or close-time active chat-run abort behavior. The PR discussion describes an active, narrow gateway restart correctness fix, not an obsolete or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before sockets/channels are torn down, aborts still-active chat runs when the bounded budget is exhausted, and keeps unrelated qa-lab changes out unless they are required for the regression scenario.
