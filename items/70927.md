## Summary

Keep open. Current `main` still intentionally filters `qwen3.6-plus` from Qwen Coding Plan endpoints, strips it from configured provider models, documents that limitation, and tests for it; the issue is therefore still live rather than obsolete or already fixed.

## Evidence

- **Current model catalog still removes `qwen3.6-plus` on Coding Plan hosts:** `isQwen36PlusSupportedBaseUrl()` returns false for Coding Plan base URLs, and `buildQwenModelCatalogForBaseUrl()` filters out `QWEN_36_PLUS_MODEL_ID` when those hosts are used.
  - file: [extensions/qwen/models.ts:126](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/qwen/models.ts#L126)
  - command: `nl -ba extensions/qwen/models.ts | sed -n '111,136p'`

- **Configured models are normalized to drop `qwen3.6-plus` and built-in resolution is explicitly suppressed:** The Qwen plugin removes `qwen3.6-plus` from `providerConfig.models` for Coding Plan URLs and returns an explicit error saying the model is not supported on the Qwen Coding Plan endpoint.
  - file: [extensions/qwen/index.ts:168](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/qwen/index.ts#L168)
