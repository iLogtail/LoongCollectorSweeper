## Summary

Keep open. Current `main` does not implement agent-level channel account binding for outbound `sessions_send` delivery. Agent config has no per-agent `accountId` field, the `sessions_send` announce path still sends with whatever `accountId` is stored on the target session, and session initialization falls back to channel `defaultAccount` rather than resolving an agent-bound account.

## Evidence
