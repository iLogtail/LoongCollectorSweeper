## Summary

Keep open. Current main still persists plugin auto-enable materialization into config during gateway startup and doctor repair paths. The older restart-churn symptom appears partially mitigated by the internal-write hash guard, but the requested runtime-only/canonical-config behavior is not implemented.

## Best Possible Solution

Keep the issue open and narrow the fix to separating effective runtime activation from persisted config repair: gateway startup should pass an auto-enabled effective snapshot into plugin/channel bootstrap without writeConfigFile, and doctor should report auto-enable reasoning without treating it as a repair mutation that writes channels.<id>.enabled or plugins.entries.<id>.enabled.
