## Summary

Current `main` has partial redaction and extensibility seams, but it does not implement automatic sensitive-data sanitization for outbound agent responses as requested in #43830. The existing built-in redaction is limited to logging/tool-summary style paths and selected tools like `sessions_history`, while outbound reply delivery only exposes plugin hooks for custom interception.

## Evidence

- **Logging redaction is explicitly tools-only:** The public config surface documents `logging.redactSensitive` as only `off | tools`, and the type comment says it redacts sensitive tokens in tool summaries.
  - file: [docs/gateway/configuration-reference.md:766](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/gateway/configuration-reference.md#L766)

  - file: [src/logging/redact.ts:8](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/logging/redact.ts#L8)
