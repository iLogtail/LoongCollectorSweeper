## Summary

Keep open. Current main has log/tool-summary redaction and plugin hooks that could support some external sanitization work, but it does not implement the issue’s requested automatic final-output sanitization policy or `outputSanitization`/`privacy.enabled`-style configuration. The related open PR #45783 overlaps on privacy filtering, but its stated scope is LLM traffic egress/restoration, not redacting final agent replies before channel delivery, so it does not fully supersede this issue.

## Best Possible Solution

Keep #43830 open until maintainers either land a scoped core final-output sanitizer for user-visible replies/tool-result delivery, or split the problem explicitly: let #45783 track LLM egress privacy and create/point to a canonical follow-up for channel-delivery response redaction. A safe implementation likely belongs near reply dispatch/block streaming/message-sending, with config/schema/docs/tests proving what text and media paths are filtered and how operators opt out.
