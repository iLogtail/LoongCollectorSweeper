## Summary

Keep open. Current `main` still hardcodes the agent file browser to a small allowlist of core markdown files, and the gateway protocol/UI have no scope-level support for browsing arbitrary workspace files or the wider filesystem.

## Evidence

- **Gateway still enforces a hardcoded filename allowlist:** `src/gateway/server-methods/agents.ts` defines `ALLOWED_FILE_NAMES` from fixed bootstrap + memory filenames and rejects anything else with `unsupported file` in `resolveAgentWorkspaceFileOrRespondError()`.
