## Summary

Keep #69333 open. Current main has generic browser status/doctor paths and agent guidance that partially cover browser readiness checks, but there is still no Hubitat installed-app discovery guidance, Maker API fallback documentation, or Hubitat-specific inspector/plugin surface in the repo. The issue is recent and the production comment adds concrete workaround evidence, so this is not stale or safely closeable.

## Best Possible Solution

Keep the issue open or split it into two narrower tasks: add/update Hubitat workflow guidance that says static curl of the JS-rendered installed-app UI is insufficient, prefer Maker API where it covers the use case, and use rendered browser inspection only when needed; separately ensure browser-dependent workflows perform a preflight with `browser` `action="status"`/`doctor` or `openclaw browser doctor --deep` and surface clear fallback guidance. If Hubitat remains service-specific, the durable implementation likely belongs in a ClawHub plugin/skill using the existing browser and plugin surfaces rather than new core behavior.
