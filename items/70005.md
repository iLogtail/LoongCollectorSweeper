## Summary

Keep #70005 open. Current main has useful mitigations for transcript boilerplate, cron/heartbeat chatter, and the literal `assistant` tag, but the requested semantic-importance material-selection layer is still not implemented. REM still selects reflections from recurring concept tags and candidate truths from score/recall/consolidation-style signals, while the narrative writer only consumes the fragments already selected upstream.

## Best Possible Solution

Keep this issue open for memory-core product/implementation work. The likely fix belongs in the pre-REM material-selection path: add a deterministic or model-backed semantic density/importance pass that down-ranks routine role labels and boilerplate, promotes decisions/preferences/conclusions/unusual events, and adds focused regression coverage using a fixture shaped like the reported Persona/DQ/SRW session versus high-frequency `assistant` noise. Keep the existing noise filters and `assistant` tag blacklist as supporting safeguards, not as the complete fix.
