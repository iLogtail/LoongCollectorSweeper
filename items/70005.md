## Summary

Keep #70005 open. Current main has useful mitigations for the concrete noisy “assistant/HEARTBEAT_OK” failure mode, but the scheduled REM/dream diary path still ranks reflection material from concept-tag counts across recent short-term traces rather than a semantic importance or decision/preference filter. The author’s follow-up says a prompt-only change was helpful but insufficient and that the remaining bottleneck is pre-narrative material selection, which still matches the current code shape.

## Best Possible Solution

Keep the issue open and fix it in `memory-core` by adding a semantic material-selection layer before scheduled REM/narrative generation. The likely implementation should score session/daily chunks for durable user decisions, preferences, conclusions, unusual events, and explicit outcomes; demote role/scaffolding/heartbeat/current-state chatter; feed ranked grounded snippets and themes into `previewRemDreaming` and the dream diary; and add regression tests with a high-frequency role word plus low-frequency meaningful decisions to prove the meaningful material wins. If maintainers want a user knob, document and validate it alongside `plugins.entries.memory-core.config.dreaming`.
