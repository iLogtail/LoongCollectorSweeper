## Summary

Keep #43235 open. Current main still routes normal inbound agent runs through a per-session lane and then the shared global `main` lane, with only `agents.defaults.maxConcurrent` configuring main-lane concurrency. I found manual per-run custom lane support in the gateway/plugin path, but not the requested per-agent or per-agent-group lane policy for channel ingress such as Feishu. The related issue/comment context points to broader multi-agent deployment work and corroborates the same `lane=main` bottleneck; it does not supersede this request.
