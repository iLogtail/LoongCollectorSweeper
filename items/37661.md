## Summary

Keep #37661 open. Current main still has partial duplicate suppression for stream bookkeeping, block streaming, heartbeat payloads, and messaging-tool echoes, but I did not find an assistant-text repetition detector, n-gram/sentence guard, generation abort path, or outbound per-turn duplicate/message cap that would stop the reported pure text generation flood.

## Best Possible Solution

Keep this issue open and implement an output-level guard in the assistant streaming/final reply pipeline, with an outbound per-turn duplicate/message cap in the channel delivery planning path. Add focused tests for repeated deltas, repeated sentence/ngram fragments inside one assistant message, repeated block-stream payloads, and repeated channel sends. Keep #34574 separate because it tracks tool-call loops, while #37661 is about pure text generation and delivery flooding.
