## Summary

Keep #60745 open. Current main still has only retention/cap based cleanup for cron/webhook-created sessions, plus separate one-shot job deletion and spawned-session cleanup. It does not expose an `ephemeral` flag for cron jobs or hook mappings, and it does not delete the hook/cron run session entry immediately after completion on success or error. The related closed ClawSweeper reports cover isolated cron sessions, retention pruning, UI hiding, and `sessions_spawn cleanup: "delete"`, but they do not supersede this completion-based disposal request.

## Best Possible Solution

Keep the issue open and implement this as a core cron/webhook session lifecycle feature: add an `ephemeral` or explicit cleanup policy to cron job create/patch/protocol/CLI and hook mapping/request dispatch types, carry it through `runCronIsolatedAgentTurn`, and in a completion/finally path remove the relevant run-session entry and archive/delete its transcript via the existing session-store transcript cleanup helpers while preserving cron run logs. Tests should cover recurring cron success, cron error, mapped hook success/error, and non-ephemeral sessions remaining unchanged.
