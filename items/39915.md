## Summary

Current `main` already has Telegram preview streaming, so the PR body’s core premise is obsolete, but the specific throttle change in this PR is not implemented on `main` and the proposed 100ms floor still looks risky because preview failures stop the stream outright. That leaves this as a stale optimization PR rather than a clear close candidate under the allowed reasons.
