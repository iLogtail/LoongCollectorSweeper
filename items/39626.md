## Summary

Keep open. Current main still has user-visible sanitizer coverage for XML-style tool-call leaks, MiniMax XML, model control tokens, memory/reasoning tags, and downgraded `[Tool Call: ...]` scaffolding, but I did not find implementation, docs, tests, or changelog evidence that the exact reported bare `exec{"command": ...}` pseudo-tool-call text is stripped or executed correctly before Telegram/shell delivery.

## Best Possible Solution

Keep the issue open and add a focused fix in the shared assistant-visible delivery path, with regression coverage for the exact `exec{"command": "openclaw status", "description": "Check OpenClaw version and status"}` text on both final assistant extraction and streaming/block delivery. The fix should either convert a recognized pseudo-call into a real tool call before delivery or safely suppress that bare pseudo-call from user-visible channel output without stripping ordinary prose/code examples.
