## Summary

Keep open. Current main still does not handle the reported bare `exec{...}` visible-text leak. It has a narrow proxy-like `tool_choice: "auto"` mitigation and sanitizers for XML/downgraded tool-call scaffolding, but no stage that strips tool-name-plus-JSON pseudo calls before shell/Telegram delivery.

## Best Possible Solution

Keep this open as a behavior bug. Preserve the proxy-like `tool_choice: "auto"` mitigation, then add either provider-specific root-cause handling for the affected model/config or a conservative visible-text sanitizer for actual tool-name-plus-JSON pseudo calls such as `exec{...}`. Add focused regression coverage for `sanitizeAssistantVisibleText`, `pi-embedded-utils.extractAssistantVisibleText`, text_end/message_end block replies, and the shell/Telegram delivery path.
