## Summary

The report's cited failure path no longer matches current OpenClaw main. OpenRouter Anthropic refs are treated as proxy-style OpenAI-compatible traffic, and embedded-runner fallback traffic is routed through OpenClaw's own boundary-aware transports rather than the upstream `@mariozechner/pi-ai` Anthropic provider path described in the issue.

## Evidence
