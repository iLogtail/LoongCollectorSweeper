## Summary

Keep open. Current main still makes plugin `api.runtime.subagent.run()` available from gateway hook paths, but it forwards the call as an ordinary gateway `agent` request and does not register the run with subagent completion metadata. The `subagent_delivery_target` hook and final completion delivery still hang off the tracked subagent announce flow, so the specific `before_dispatch` deferred-delivery gap in #55517 remains unresolved.

## Best Possible Solution

Keep #55517 open and resolve it deliberately: either add an explicit plugin-runtime completion-delivery mode that registers runs with requester origin and `expectsCompletionMessage` metadata, with a regression test for `before_dispatch` returning a handled progress reply while the child later delivers via `subagent_delivery_target`, or document that `api.runtime.subagent.run()` is manual orchestration and does not participate in core completion announce delivery.
