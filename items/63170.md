## Summary

Keep open. Current `main` has Feishu card interaction plumbing and same-chat approver authorization, but it does not expose a native exec-approval capability for Feishu or document any `channels.feishu.execApprovals` flow. The requested feature is still unimplemented on `main`.

## Evidence

- **Feishu channel wires only the auth adapter into approvals:** The Feishu plugin registers `approvalCapability: feishuApprovalAuth` and nothing else for approval delivery/runtime/native surfaces.
  - file: [extensions/feishu/src/channel.ts:666](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/extensions/feishu/src/channel.ts#L666)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '650,675p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Feishu approval adapter is auth-only:** `feishuApprovalAuth` is built with `createResolvedApproverActionAuthAdapter`, resolving approvers from `allowFrom` and normalizing sender IDs; the adapter definition only returns `authorizeActorAction` and does not define native delivery/runtime hooks.
  - file: [extensions/feishu/src/approval-auth.ts:15](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/extensions/feishu/src/approval-auth.ts#L15)
  - command: `nl -ba extensions/feishu/src/approval-auth.ts | sed -n '1,80p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Auth helper does not provide native approval surfaces:** `createResolvedApproverActionAuthAdapter` returns an object with only `authorizeActorAction(...)`; there is no `native`, `nativeRuntime`, `getExecInitiatingSurfaceState`, or delivery implementation in this helper path.
  - file: [src/plugin-sdk/approval-auth-helpers.ts:40](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/plugin-sdk/approval-auth-helpers.ts#L40)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Native exec approval clients require capability.native plus initiating-surface state:** `hasNativeExecApprovalCapability()` only returns true when a channel approval capability exposes `native` and an initiating-surface availability hook. Feishu does not provide those via its registered approval capability, so it is not a native exec-approval client on current `main`.
  - file: [src/infra/exec-approval-surface.ts:32](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/exec-approval-surface.ts#L32)

  - file: [extensions/feishu/src/card-ux-launcher.ts:16](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/extensions/feishu/src/card-ux-launcher.ts#L16)
