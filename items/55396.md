## Summary

Close as `clawhub`: the Kudosity SMS work is a useful plugin, but current OpenClaw scope and docs point vendor-specific channel integrations toward external plugin distribution rather than bundling them into `openclaw/openclaw`.

## Evidence

- **Vision: optional capability should stay out of core when possible:** Current project vision says optional capability should usually ship as plugins, the preferred path is npm/local plugin distribution, plugins should be hosted in their own repository, and the bar for adding optional plugins to core is intentionally high.

- **Vision: commercial integrations are out of scope for now:** The roadmap guardrails explicitly list 'Commercial service integrations that do not clearly fit the model-provider category' under 'What We Will Not Merge (For Now)'. Kudosity SMS is a vendor-specific commercial messaging integration.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/VISION.md#L99)

## Close Comment

Closing from a Codex review because this looks like the right feature in the wrong home. Current project scope says optional capability should usually ship as plugins hosted in their own repo rather than being added to core (`VISION.md:54`, `VISION.md:57`), and the roadmap explicitly excludes commercial service integrations that do not clearly fit the model-provider category (`VISION.md:99-104`). The plugin docs also say channel plugins do not need to be added to this repository and should be published to ClawHub or npm instead (`docs/plugins/building-plugins.md:16-19`, `docs/plugins/community.md:9-16`, `docs/tools/plugin.md:14-16`). On current `main` (`34896839ba22f3a71a3d5a8fb4ca5f4c6989a534`) there is no existing `kudosity` surface in `extensions/`, `docs/`, or `src/`, so this review is not closing it as already shipped. If you want to keep the implementation moving, the maintainable path is an external OpenClaw plugin package published via ClawHub/npm instead of bundling it into `openclaw/openclaw`.
