## Summary

Keep open. Current `main` still contains the conservative `\<newline>` rejection in allowlist analysis, current tests still expect that rejection, and both gateway/node-host surfaces still collapse the resulting parse failure into an `allowlist miss` error. The issue comment about a fix branch is useful context, but that fix is not present on `main`.
