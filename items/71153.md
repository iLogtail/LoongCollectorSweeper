## Summary

Keep open. Current main still rejects `\<newline>` commands before allowlist analysis, still reports that analysis failure as `allowlist miss`, and the current test suite explicitly asserts the failing behavior. The linked PR #71154 is an open, unmerged fix proposal; #71225 was closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #71153 open until a focused core fix lands. The likely path is to normalize POSIX `\<newline>` continuations before allowlist analysis in `src/infra/exec-approvals-allowlist.ts` / `src/infra/exec-approvals-analysis.ts`, add regression coverage for allowlisted multi-line commands, and split unparseable syntax from true allowlist misses in the gateway and node-host denial paths. PR #71154 is the canonical existing fix proposal to review or supersede.
