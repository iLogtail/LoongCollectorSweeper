## Summary

Keep #71153 open. Current main still contains the exact early `\<newline>` allowlist rejection and still maps analysis failures to `allowlist miss` in both gateway and node-host policy paths. The related fix PR #71154 is open and directly tracks the requested behavior; PR #71225 was closed unmerged, and its branch comment does not change current main behavior.

## Best Possible Solution

Keep #71153 open until a focused fix lands. The best implementation path is the one already described by the reporter and PR #71154: normalize safe POSIX `\<newline>` continuations before allowlist analysis, preserve shell/parser safety around quotes and heredocs, add gateway and node-host `unsupported-shell-syntax` denial plumbing for true parse failures, and replace the current regression test that asserts line continuations fail with coverage for the allowed multiline curl case and distinct parse-error messaging.
