## Summary

Keep open. Current `main` still contains both reported QMD behaviors, so this is not implemented on `main`, and the PR's checkpoint-file exclusion is too broad because it would also skip valid primary transcripts whose session IDs contain `.checkpoint.`.

## Evidence

- **QMD status probe still ignores higher configured timeout on main:** `probeVectorAvailability()` still calls `runQmd(["status"])` with `timeoutMs: Math.min(this.qmd.limits.timeoutMs, 5_000)`, so configs above 5s are still capped on current `main`.

## Risks / Open Questions

- The underlying timeout bug and checkpoint-export noise still appear to be real on current `main`.
- Merging this PR as-is would likely introduce a regression by silently excluding searchable memory for legitimate session IDs containing `.checkpoint.`.
