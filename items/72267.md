Keep open. Current main still emits the zsh completion script with a leading blank line before `#compdef`, so the PR's central bug is not implemented on main. The related same-title prior PR is closed and unmerged, and the current tests cover sourced/deferred registration behavior rather than the `fpath` autoload first-line case.

Keep the PR open and either land the targeted newline removal with a small regression assertion in `src/cli/completion-cli.test.ts`, or ask the contributor to add that assertion before merge. The fix belongs in the existing CLI completion generator, not in docs or a plugin.
