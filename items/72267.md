## Summary

Keep open. Current main still emits the zsh completion script from a template literal that begins with a blank line before `#compdef`, while the PR removes exactly that leading newline. The existing zsh test covers sourcing/deferred `compdef` registration, not the `fpath` autoload first-line contract described in the PR and zsh docs.

## Best Possible Solution

Keep PR #72267 open and either merge/adapt its one-line generator fix or add an equivalent change on main. A small regression test should assert `getCompletionScript("zsh", program).startsWith("#compdef openclaw\n")`; an optional zsh integration test can cover `fpath`/`compinit` autoload when zsh is available.
