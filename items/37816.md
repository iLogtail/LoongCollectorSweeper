## Summary

Keep #37816 open. Current main still has Telegram preview streaming, edited in-place tool/progress lines, /verbose tool bubbles, and /reasoning visibility, but it does not provide the requested opt-in Telegram mode that emits every relevant agent activity step as persistent separate messages. The related Telegram verbose/streaming/NO_REPLY issues are narrower closed reports and do not supersede this feature request.

## Best Possible Solution

Keep tracking this as a Telegram plugin feature. Implement an opt-in mode in extensions/telegram, for example channels.telegram.streaming.mode:"full" or channels.telegram.activityMessages:true, with config schema, docs, and tests. It should emit durable separate messages for relevant agent activity events, keep the current preview-edit behavior as the default, and suppress internal NO_REPLY housekeeping turns.

## Risks / Open Questions

- A full persistent Telegram activity mode could create high message volume and run into Telegram rate limits or noisy group-chat UX.
- Reasoning, memory, tool output, plans, and command summaries have privacy and safety implications, so any implementation should be opt-in and filtered.
- The implementation needs explicit NO_REPLY/internal-turn suppression so housekeeping does not leak into user-facing chats.
