## Summary

Keep #69110 open. Current main still does not expose resolved runtime provider/model identity to `message_sending`, and there is no landed final-reply hook or gateway enforcement layer that can verify or rewrite model self-tags before delivery. The related open PRs are useful context, especially #65149, but they are not merged and do not yet make the reported behavior obsolete.

## Best Possible Solution

Keep the issue open and coordinate it with the message hook architecture work. A good fix would thread resolved provider/model provenance from the agent run result into the final reply or delivery hook boundary, add typed hook fields/tests, and decide whether core should only expose provenance or also enforce/overwrite user-visible model tags when fallback changes the runtime model.
