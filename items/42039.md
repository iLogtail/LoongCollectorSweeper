## Summary

Keep #42039 open. Current main still keeps `maxMissedJobsPerRestart` and `missedJobStaggerMs` as internal `CronServiceDeps` only; they are absent from the public `CronConfig`, strict config schema, docs, and gateway wiring, so users still cannot set them via `openclaw.json`. Related PR #42069 was closed unmerged, and the current code does not contain its proposed config exposure.

## Best Possible Solution

Keep the issue open and implement the narrow public config path: add `maxMissedJobsPerRestart` and `missedJobStaggerMs` to `CronConfig`, the strict zod schema, labels/generated schema/help, and public cron docs; wire them from `buildGatewayCronService` into `CronService` deps; add focused tests proving config passthrough, `0` disables immediate startup catch-up, and staggered deferred jobs keep the existing default behavior when unset.

## Risks / Open Questions

- Changing startup catch-up defaults could break users who rely on missed cron jobs running after sleep or downtime; the implementation should preserve current defaults unless explicitly configured.
- `maxMissedJobsPerRestart: 0` needs explicit tests because it is both the requested disable-catch-up behavior and an edge case for deferred job scheduling.
