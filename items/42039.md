## Summary

Current main still does not expose `maxMissedJobsPerRestart` or `missedJobStaggerMs` through public cron config. The internal CronService dependency knobs exist and tests can inject them, but `CronConfig`, strict config validation, docs, and gateway CronService construction do not wire those fields from `openclaw.json`. The linked PR #42069 attempted this but was closed unmerged; #48345 is related for skip-catch-up behavior but does not fully supersede the requested max/stagger config exposure.
