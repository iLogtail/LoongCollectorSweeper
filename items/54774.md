## Summary

Keep open. Current main still treats a pre-hello gateway close as terminal for one-shot callers, and the PR's attempt-scoped, capped transient-close handling plus regression tests are not present on main. The PR discussion also shows useful follow-up addressing review concerns rather than a stale or obsolete change.

## Best Possible Solution

Keep open for maintainer review or manual consolidation with #54475. The likely fix belongs in `src/gateway/call.ts` and `src/gateway/client.ts`: track pre-`hello-ok` state, suppress only bounded `(1000, "")` transient closes tied to the correct connect attempt, preserve fatal handling for other close shapes and post-hello closes, and add focused `call.test.ts` and `client.test.ts` regression coverage.
