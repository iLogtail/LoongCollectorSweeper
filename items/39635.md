## Summary

Keep #39635 open. Current main still lacks the requested `agents.defaults.extraBootstrapFiles` or per-agent equivalent, and the existing `bootstrap-extra-files` hook remains limited to recognized bootstrap basenames, so custom files such as `CONTEXT_BRIEF.md` are still not supported as requested.

## Best Possible Solution

Keep this issue open as the canonical tracker for first-class config support. A complete fix likely needs a strict schema/type addition for `agents.defaults.extraBootstrapFiles` plus per-agent override semantics, safe workspace-relative path loading for arbitrary basenames, ordering after standard bootstrap files, budget integration with existing bootstrap limits, docs/schema-help updates, and focused tests covering missing files, custom basenames, and default-vs-agent override behavior.
