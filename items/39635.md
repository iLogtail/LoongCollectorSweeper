## Summary

Keep #39635 open. Current main still lacks `agents.defaults.extraBootstrapFiles` or a per-agent equivalent, and the existing `bootstrap-extra-files` hook is not a substitute because it only accepts recognized bootstrap basenames, so arbitrary files like `CONTEXT_BRIEF.md` remain unsupported through the requested config surface.

## Best Possible Solution

Keep #39635 open. A complete fix should add a strict typed/schema-backed `agents.defaults.extraBootstrapFiles` field plus a per-agent override or replacement policy, resolve configured paths relative to the agent workspace with the existing boundary/realpath guards, inject custom files after standard bootstrap files in configured order, apply existing per-file and total bootstrap budgets, document the public config, regenerate schema/help artifacts, and add focused tests for missing files, invalid or escaping paths, ordering, arbitrary basenames, truncation budgets, and defaults-vs-agent override semantics.
