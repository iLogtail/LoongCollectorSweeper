## Summary

Keep #39635 open. Current main still lacks `agents.defaults.extraBootstrapFiles` or a per-agent equivalent, and the existing `bootstrap-extra-files` hook is still restricted to recognized bootstrap basenames, so files like `CONTEXT_BRIEF.md` are not supported as requested. Related context points the other way too: #51789 was closed as a duplicate of this issue, while #39120 remains a broader arbitrary-file tracker rather than a replacement for this first-class config request.

## Best Possible Solution

Keep #39635 open as the canonical tracker for first-class config support. A complete fix should add a strict schema/type field for defaults plus per-agent override semantics, safely resolve workspace-relative custom files, inject them after standard bootstrap files, apply existing per-file and total bootstrap budgets, document the config, update generated schema/help, and add focused tests for missing files, custom basenames, ordering, and default-vs-agent override behavior.
