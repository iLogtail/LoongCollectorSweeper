## Summary

Keep #23906 open. Current main still does not expose a per-spawn `sessions_spawn` tool profile override, and the request remains active because the discussion covers local-model tool-less spawns plus plugin/MCP tool inheritance that is not solved by the existing global subagent tool policy or by the overlapping #15032 thread.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete implementation path is to add `tools: "none" | string[]` or explicit `disableTools`/`toolsAllow` to `sessions_spawn`, validate it in `sessions-spawn-tool`, carry it through `SpawnSubagentParams`, extend the gateway `agent` schema/server/command ingress path, map tool-less spawns to runner `disableTools: true`, map subsets to runner `toolsAllow`, and add docs/tests for core tools, plugin/bundle MCP tool names, global subagent policy precedence, sandbox/security behavior, and small local-model tool-less spawns.
