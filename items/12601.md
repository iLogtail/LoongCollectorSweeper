## Summary

Keep #12601 open. Current main still does not expose Slack Canvas content reading: the Slack message action surface has no Canvas/read-canvas action, the dispatcher/runtime only handle existing message/file/pin/member/emoji actions, setup/docs omit `canvases:read`, and related PR #18752 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #12601 open for the bundled Slack plugin. A complete fix should live primarily under `extensions/slack`: add a Canvas read action or dedicated tool, expose and dispatch it through the Slack message action surface, add `canvases:read` to setup/docs/config as needed, call Slack's Canvas read API through the existing account/token model, return bounded markdown or section data, and cover discovery, dispatch, runtime behavior, docs, and live Slack validation.
