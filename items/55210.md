## Summary

Keep open. Current main already has the normal-stream stop chunk machinery, but it still emits the streaming error message and `finish_reason: "stop"` in the same choice, and the current regression test asserts that old behavior. The PR is stale relative to newer usage-aware finalization and should be narrowed rather than auto-closed.

## Best Possible Solution

Keep the PR open but ask for a current-main refresh that preserves `writeAssistantStopChunk`, `maybeFinalize`, `stream_options.include_usage` behavior, `finish_reason: null` on non-final chunks, and disconnect cleanup. The narrow code change should make the catch path emit the error text as a normal content chunk, leave `wroteStopChunk` false, then let `maybeFinalize` send the empty stop chunk; tests should assert the error content and terminal stop choice are separate.
