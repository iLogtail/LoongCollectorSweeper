## Summary

Keep open. Current main still lacks the central plugin-tool fixes from #56574: `registerTool` appends duplicate registrations unconditionally, and plugin tool factories do not receive `plugins.entries.<id>.config`. Some gateway-bindable/test-env work is already present on main, so the useful path is to rebase and narrow/split the PR rather than close it as implemented or superseded.

## Best Possible Solution

Keep #56574 open for maintainer review, but rebase and narrow it. The best code path is a focused plugin PR that adds exact per-plugin duplicate registration suppression while preserving partial-overlap registrations, threads `pluginConfig` through `PluginToolRegistration` into `OpenClawPluginToolContext`, and locks both behaviors with registry/tool tests and public SDK/docs updates if the factory context becomes a supported contract. Drop or separate the already-landed gateway-bindable and live-test-home/docs changes.
