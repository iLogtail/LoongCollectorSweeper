## Summary

Keep open. Current main still sends full accumulated chat delta snapshots after tool boundaries, while the Control UI commits pre-tool stream text into a separate segment above the tool card. That combination can still produce the duplicated pre-tool text this PR is about. The PR discussion is useful, especially the bot review notes that ACP/TUI depend on full-snapshot chat deltas, so this needs a compatible fix rather than cleanup-close.

## Best Possible Solution

Keep the PR open or replace it with a narrower compatible fix: preserve full-snapshot chat deltas for generic gateway, ACP, TUI, and node subscribers, but make the Control UI render only the post-tool portion after it commits a pre-tool stream segment. A schema-compatible optional hint inside `message`, or UI-side subtraction based on committed `chatStreamSegments`, should be covered by tests in the gateway chat event path and Control UI chat/tool-stream rendering path.
