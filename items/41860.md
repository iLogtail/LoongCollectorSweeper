## Summary

Keep open. Current main still routes Feishu text sends through a raw `tag: "md"` post payload with the original message text, and there is no merged regression coverage or shipped fix for underscore-containing URL rendering. The linked PRs are relevant, but they remain open/unmerged rather than proof that the issue is solved on main.

## Best Possible Solution

Keep #41860 open and use one canonical Feishu fix, preferably the safer shape from #41880: detect bare URL-only text payloads and send them as explicit Feishu link elements or otherwise escape URL markdown spans in `extensions/feishu/src/send.ts`, with regression tests for underscore-containing URLs in the send/edit paths. After that lands on main, close the issue as implemented and close duplicate PRs against the landed fix.
