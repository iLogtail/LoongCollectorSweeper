## Summary

Keep PR #55944 open. Current main still has no `visionos` platform id, prefix rule, device-family fallback, or spatial command defaults in the gateway node command policy, so a real Vision Pro node reporting `platform: "visionos"` still falls through to the conservative unknown-platform policy. The PR is a narrow core gateway-policy change, not ClawHub scope, and the related visionOS app PR #56130 remains open while explicitly depending on this platform-policy work.

## Best Possible Solution

Keep the PR open and move it toward merge as a narrow gateway core policy change: rebase if needed, add focused tests for `platform: "visionos"`, `deviceFamily: "Vision Pro"`, spatial command defaults, dangerous camera command gating, and non-matching generic headset metadata, then coordinate with the related visionOS node app PR #56130.
