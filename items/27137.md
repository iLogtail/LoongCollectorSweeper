## Summary

Keep #27137 open. Current main still has shared DM/group-policy primitives, but it does not contain the requested unified `authorizeIngress()`/`IngressContext` gate, and representative ingress surfaces continue to authorize locally per channel or event type. The existing issue comment already reached the keep-open conclusion, and current main at `bc24b547d0888fb716505b07e3f2dceeb8abd203` still supports that outcome.

## Best Possible Solution

Keep #27137 open for maintainer design and implementation. The best path is a narrow core/security or SDK-backed ingress authorization helper that centralizes DM/group policy, allowlist merging, command gating, event-kind handling, and reason codes while letting channel adapters provide identity normalization and chat-context classification. Adoption should migrate representative message, reaction, pin, interaction, callback, slash, and voice paths and add contract tests proving new ingress surfaces cannot bypass the shared gate.
