## Summary

Keep #27137 open. Current main still has shared DM/group policy primitives, but it does not implement the requested unified `authorizeIngress()` / `IngressContext` gate, and representative channels continue to authorize ingress through channel- or event-local helpers. The existing steipete Codex review comment also explicitly kept this open, and the newer main SHA does not change that conclusion.

## Best Possible Solution

Keep #27137 open for maintainer design. The likely path is a narrow core/security or SDK-backed ingress authorization helper that centralizes DM/group policy evaluation, allowlist merging, command gating, event-kind handling, and reason codes while letting channel adapters supply identity normalization and chat-context classification. Adoption should migrate representative message, reaction, pin, interaction, callback, slash, and voice paths, with contract tests proving new ingress surfaces cannot bypass the shared gate.
