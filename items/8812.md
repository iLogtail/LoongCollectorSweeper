## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as escaped/plain text in Control UI tool-card paths and native WebChat tool-result cards. The related implementation attempt, PR #8880, is closed unmerged, and the latest maintainer/Codex review comment also explicitly kept this issue open.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware renderer for Control UI tool output cards across expanded output, raw details, collapsed previews, inline short cards, and sidebar/side-panel content, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace. If WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString/Link handling rather than plain Text.
