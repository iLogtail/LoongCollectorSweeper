## Summary

Keep open. Current main still contains the exact heartbeat dedupe write pattern this PR targets: `runHeartbeatOnce` loads a session-store snapshot, mutates `lastHeartbeatText` / `lastHeartbeatSentAt`, then saves that snapshot. The session-store helper contract confirms `updateSessionStore` is the safer read-modify-write API because it re-reads inside the lock, so this PR is not implemented on main or clearly superseded.

## Best Possible Solution

Keep the PR open or land an equivalent change on main: replace the heartbeat dedupe `loadSessionStore` / `saveSessionStore` block with `updateSessionStore`, remove the now-unused import if applicable, and add a focused regression test that simulates a concurrent session-store update while the heartbeat dedupe write is pending.
