## Summary

Keep PR #70051 open. Current main still handles every node-level `exec.started` / `exec.finished` / `exec.denied` event by canonicalizing the session key, enqueuing a system event, and requesting an `exec-event` heartbeat wake; it does not inspect the existing `heartbeatIsolatedBaseSessionKey` marker. The related #59606 fix stabilized repeated `:heartbeat` suffix growth but did not suppress this relay path, and #67059 was closed unmerged and explicitly scoped away from `server-node-events.ts`.

## Best Possible Solution

Keep the PR open and have maintainers decide whether the heartbeat-isolated marker check belongs directly in `src/gateway/server-node-events.ts` as proposed, or in a shared exec-event/wake helper that also covers gateway-hosted exec notifications. The minimum product path is a regression test proving node `exec.*` events for a stored synthetic heartbeat session neither enqueue a system event nor request another heartbeat wake, while ordinary agent/session exec notifications remain unchanged.
