## Summary

Keep open. Current main still keeps WebChat chatQueue in memory only, and the linked canonical issue #51549 remains open. PR #71336 is not superseded by #56542, which was already closed as stale, and it is now focused on the queue-persistence path after dropping the unrelated session-store changes discussed in review. The PR is not merge-ready as written, but that is a reason for contributor follow-up, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open but request changes: fix the incorrect import/export wiring, remove the unused restoreQueue import, put queue restore/drain on a lifecycle point that runs after gateway connection is established, and add focused UI tests for enqueue persistence, remove/drain cleanup, TTL expiry, dedupe on restore, and reconnect/session-switch restore. Treat #71336 as a focused queue-persistence PR for #51549, with draft/history either included explicitly or left to a narrower follow-up.
