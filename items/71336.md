## Summary

Keep open. Current main does not yet have WebChat chatQueue persistence: the queue is still local UI state and is cleared on session switches/refresh-style reinitialization. The PR targets a real open bug (#51549) and the author responded to review by narrowing scope, so it is not obsolete. However, the latest PR head still needs maintainer review/changes because it appears to import `flushChatQueue` from a module that does not export it and CI is failing type/build checks.

## Best Possible Solution

Keep #71336 open for normal review. The best path is to fix the import/export mismatch, add focused UI tests for queue persist/restore, stale-entry removal, session-switch restore, and reconnect drain, and then either scope the PR explicitly to queue persistence while #51549 tracks draft/history follow-up or extend it to cover draft persistence too.
