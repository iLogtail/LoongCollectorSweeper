## Summary

Keep #65445 open. Current main partially addresses the session-cruft problem: pruned transcripts are archived as `.deleted`, old `.deleted` and `.reset` archives can be removed, and optional disk-budget cleanup can remove archive artifacts under pressure. The specific requested default garbage collection is still missing: age cleanup explicitly leaves `.jsonl.bak` transcript archives, there is no recycle/soft-delete config, and `sessions_saved.json` is not recognized anywhere in current source/docs/tests.

## Best Possible Solution

Keep the issue open and implement a focused session-maintenance enhancement: add tests and code for explicit artifact garbage collection covering orphaned `.jsonl.deleted`, `.jsonl.reset`, `.jsonl.bak`, and any legacy `sessions_saved.json` behavior maintainers still want to support. Align config/schema/docs around either a recycle/retention model or clearly document why the existing `resetArchiveRetention` plus optional `maxDiskBytes` replaces part of the proposal.
