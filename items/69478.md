## Summary

Keep open. Current main still lacks the requested dedupe/cancellation behavior: system events only suppress consecutive duplicate text, exec approvals are generated under fresh UUIDs, and the gateway only treats duplicate approval IDs as idempotent. The related fix PRs referenced in the issue timeline were closed unmerged, and there is no changelog/release evidence that #69478 shipped.

## Best Possible Solution

Keep #69478 open and fix the core path. Add stable idempotency or cancellation for exec approval/system events keyed by session or agent plus command intent/runId/contextKey, so failover retries with fresh approval IDs coalesce instead of sending repeated prompts. Include regression coverage for a retry of the same approval intent producing only one queued/native approval prompt, and update protocol/docs if a new public idempotency key is introduced.
