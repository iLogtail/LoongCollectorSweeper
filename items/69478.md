## Summary

Keep #69478 open. Current main still does not deduplicate system events or exec approval requests by runId/contextKey; it only suppresses consecutive identical system-event text and approval records remain keyed by fresh approval IDs. The related PRs called out in the issue timeline were closed without merge, and the latest release tag in this checkout has the same system-events behavior.

## Best Possible Solution

Keep the issue open and fix it in the approvals/system-event boundary. The likely maintainer path is to add bounded idempotency for pending exec approval requests using stable intent fields such as agentId/sessionKey plus command/systemRunPlan and context/run identity, then add regression tests for non-consecutive duplicate system events and repeated exec approval requests under fresh approval IDs. The existing exec.finished dedupe can remain as a separate downstream defense.
