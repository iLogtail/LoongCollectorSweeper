## Summary

Keep open. Current main still does not make macOS Talk use Gateway `talk.speak` as the primary TTS path for provider-backed Talk playback. macOS parses `talk.config`, but playback routing still only has direct ElevenLabs, local MLX, and local system voice plans; unknown/custom providers such as `openai` fall through to system voice.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `apps/macos/Sources/OpenClaw/TalkModeRuntime.swift`: make Gateway `talk.speak` the primary path for every non-local Talk provider, keep explicit local MLX and `system` paths local, decide whether direct ElevenLabs remains an optimized special case, and add macOS tests for custom/OpenAI provider routing plus any provider-selection UI/docs updates.
