## Summary

Keep open. Current main has strong tool_use/tool_result pairing repair, including after compaction and history truncation, so the compaction-root-cause part overlaps with closed #51423. But this issue also asks that Anthropic `unexpected tool_use_id` validation failures not be surfaced raw to Discord; current main still has a generic `invalid_request_error` formatter that can return `LLM request rejected: <provider message>` and the channel reply payload path emits that formatted assistant error.
