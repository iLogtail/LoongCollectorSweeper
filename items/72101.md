## Summary

Keep #72101 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still contains both behaviors the PR is meant to change: `sessions_spawn` rejects `runtime="subagent"` when `streamTo:"parent"` is present, and it forwards `model:"default"` as a literal model override. The related #68397 is closed unmerged and does not cover the model-default normalization, so it is not a live canonical replacement.

## Best Possible Solution

Keep #72101 open, or land an equivalent agents-tool patch that ignores ACP-only `streamTo` for `runtime="subagent"`, normalizes empty or `default` model input to no override through shared helper logic, preserves ACP behavior, and updates the `sessions_spawn` and `session_status` regression tests. Maintainers should explicitly decide whether to include the adjacent `resumeSessionId` silent-drop behavior from #68397 before merging.
