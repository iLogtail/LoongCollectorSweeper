## Summary

Keep #39507 open. Current main still forces Feishu thread replies from inbound thread_id/root_id even when replyInThread is "disabled", and the tests explicitly lock in that behavior. Related PR #33789 fixed reply target selection, not the stricter disabled-thread semantics requested here; the #39507 discussion also has a follow-up commit addressing the topic-session review concern.

## Best Possible Solution

Keep #39507 open and resolve the Feishu semantics explicitly. If disabled should be a hard off switch, gate non-topic threadReply in extensions/feishu/src/bot-content.ts on the configured replyInThread value while preserving group_topic/group_topic_sender anchoring, then update the existing regression test that currently expects forced thread replies under disabled config. If maintainers prefer the current follow-existing-thread behavior, update the schema/docs/test names so "disabled" no longer promises normal inline replies.
