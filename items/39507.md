## Summary

Keep PR #39507 open. Current main still lets inbound Feishu `thread_id` or `root_id` force `threadReply` and therefore `replyInThread` even when `replyInThread: "disabled"`; the current regression test explicitly expects that behavior. The PR discussion is useful: the author reports live Feishu verification, Greptile agreed the minimal fix, and the review concern about topic-session anchoring was addressed in the PR commits, but the remaining work needs to be ported to the current `bot-content.ts` split and resolved as a maintainer semantics decision.

## Best Possible Solution

Keep #39507 open and resolve the Feishu semantics explicitly. If `disabled` is meant to be a hard off switch, port the PR to `extensions/feishu/src/bot-content.ts`, gate non-topic `threadReply` on configured `replyInThread`, preserve `group_topic` and `group_topic_sender` anchoring, and update the regression test that currently expects forced thread replies under disabled config. If maintainers prefer follow-existing-thread behavior, update the schema/docs/test names so `"disabled"` no longer promises normal inline replies.
