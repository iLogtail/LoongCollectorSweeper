## Summary

Keep open. Current `main` still lacks a webchat/control-UI inline button surface and still documents inline buttons as unavailable for the runtime channel. The issue is not implemented on `main` as of `a6aa151653ab478e9aa23e95c6446c41adc4fce1`.

## Evidence

- **Control UI chat content model has no button/select block type:** `MessageContentItem` only allows `text`, `tool_call`, `tool_result`, `attachment`, and `canvas`. There is no normalized UI content type for inline buttons or selects.
  - file: [ui/src/ui/types/chat-types.ts:24](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/types/chat-types.ts#L24)
  - command: `nl -ba ui/src/ui/types/chat-types.ts | sed -n '20,80p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Message normalization does not preserve actionable button blocks:** Assistant message normalization has special handling for attachments, canvas previews, and text. Other block types fall through to the generic text/tool shape, so a `buttons` block would not survive as a clickable UI element.
  - file: [ui/src/ui/chat/message-normalizer.ts:315](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/chat/message-normalizer.ts#L315)
  - command: `nl -ba ui/src/ui/chat/message-normalizer.ts | sed -n '300,380p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Webchat reply assembly only builds text plus media:** `buildAssistantDisplayContentFromReplyPayloads` iterates normalized reply payloads and appends sanitized text, audio blocks, and managed image blocks. It does not read `payload.presentation` or `payload.interactive`, so Control UI chat replies do not gain inline buttons from the shared presentation contract.
  - file: [src/gateway/server-methods/chat.ts:302](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/chat.ts#L302)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '300,360p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Runtime prompt still says inline buttons are not enabled:** Current system-prompt generation still emits: `Inline buttons not enabled for <channel>. If you need them, ask to set <channel>.capabilities.inlineButtons ...` when the runtime lacks that capability.
  - file: [src/agents/system-prompt.ts:370](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/system-prompt.ts#L370)
  - command: `nl -ba src/agents/system-prompt.ts | sed -n '365,377p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **No Control UI capability/config knob exists in schema help:** The documented `gateway.controlUi` config only exposes enablement, basePath, root, embed sandbox, origin/auth hardening, and related security toggles. There is no `gateway.controlUi.capabilities.inlineButtons` or similar config surface.
  - file: [src/config/schema.help.ts:74](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/config/schema.help.ts#L74)
