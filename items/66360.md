## Summary

Current `main` does not implement transcript-specific size or line caps for session `.jsonl` files. `session.maintenance.rotateBytes` still applies only to `sessions.json`, and the only transcript-related enforcement on `main` is archive cleanup plus oldest-session eviction under a directory-wide disk budget. That does not bound a single active transcript file, so the reported gap remains actionable and should stay open.
