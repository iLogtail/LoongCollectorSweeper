## Summary

Keep open. Current main already has iMessage text chunking, but it does not implement the PR's audioAsVoice split-delivery behavior. The iMessage channel still exposes only generic attached `sendText`/`sendMedia` outbound hooks, `deliverOutboundPayloads` still sends media payloads through the generic caption+media path unless `channelData`/interactive content is present, and `sendMessageIMessage` still has no `audioAsVoice` handling. That means the core bug described by PR #39970 is not implemented on `main`, even though the chunking regression portion is already covered.
