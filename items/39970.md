## Summary

Keep open. Current `main` has the iMessage chunking behavior again, but it still does not implement this PR's `audioAsVoice`-specific delivery path: the iMessage plugin drops the flag before the runtime send call and the send layer still treats audio attachments as generic media/placeholder text.

## Evidence
