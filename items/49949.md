## Summary

Keep #49949 open. Current main still does not implement Anthropic native web search, and the linked open PR #50692 is an unmerged implementation attempt with review blockers, not a shipped or clearly accepted canonical replacement. A recent duplicate closure on #15647 also explicitly pointed to #49949 as the active tracker for this remaining feature request.

## Best Possible Solution

Keep #49949 open as the canonical feature tracker until Anthropic native web search lands or maintainers explicitly choose a different canonical item. The likely implementation belongs in the bundled Anthropic provider: add an Anthropic web-search provider/config surface, inject `web_search_20260209`/`web_search_20250305` through the Anthropic provider stream wrapper only when web search is enabled and `provider: "anthropic"` is selected, restrict it to direct Anthropic API traffic, pass domain/location/max-use config through, and cover manifest/schema/docs/runtime tests.
