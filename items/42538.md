## Summary

Keep #42538 open. Current main still has the reported health/status divergence path: `health` builds channel snapshots without gateway runtime state, while `channels.status` explicitly reads and merges runtime state. The linked PR #42586 appears to be the right fix track, but it is still open/unmerged in the provided GitHub context, so closing the original bug now would be premature.

## Evidence

- **Health snapshot lacks runtime input:** `getHealthSnapshot()` only accepts `timeoutMs` and `probe`, then constructs each channel snapshot from `accountId`, `enabled`, `configured`, optional probe, and `lastProbeAt`; it does not accept or merge `ChannelRuntimeSnapshot`.
  - file: [src/commands/health.ts:215](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/commands/health.ts#L215)

  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Gateway health refresh does not inject runtime:** `refreshGatewayHealthSnapshot()` calls `getHealthSnapshot({ probe: opts?.probe })` directly, with no channel manager runtime snapshot involved.
  - file: [src/gateway/server/health-state.ts:72](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/server/health-state.ts#L72)

  - file: [src/gateway/server-methods/channels.ts:161](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/server-methods/channels.ts#L161)
