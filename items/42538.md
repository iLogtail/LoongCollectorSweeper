## Summary

Current main still has the reported health/status divergence. The gateway `health` RPC refreshes `getHealthSnapshot()` without passing channel-manager runtime state, and `getHealthSnapshot()` builds each channel snapshot from config/probe fields only. WhatsApp then defaults missing `snapshot.running` and `snapshot.connected` to `false`, while `channels.status` explicitly reads `context.getRuntimeSnapshot()` and passes per-account runtime into the shared channel snapshot builder. The linked PR #42586 appears to track the right fix but is open/unmerged in the provided context, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #42538 open until a narrow gateway fix lands. The likely path is to thread `channelManager.getRuntimeSnapshot()` into `refreshGatewayHealthSnapshot()` / `getHealthSnapshot()`, merge per-account runtime fields before `plugin.status.buildChannelSummary()`, and add a regression test proving health and `channels.status` agree for runtime-backed `running` and `connected` values. PR #42586 can be reviewed, updated, or superseded by an equivalent focused patch.
