## Summary

Keep open. Current main does not yet implement this PR's core behavior: `sessions_spawn` still displays as `Sub-agent`, `spawnSubagentDirect` does not return `resolvedModel`, `/subagents spawn` still replies with the old generic message, and tool meta is still derived only from current tool args through the existing display formatter. The PR is focused and still addresses a real gap, but the provided review comments identify safety/regression issues that should be fixed before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep the PR open and request a narrower, safer revision: return `resolvedModel` from the accepted subagent spawn result, update both tool-display resources and tests, format `/subagents spawn` success from the raw resolved model string without truncating slash-containing ids, and keep start-notice metadata on the existing redaction/truncation display path. Add regression coverage for redacted/long prompts, rejected spawns, ACP runtime behavior, and resolved-model reporting.
