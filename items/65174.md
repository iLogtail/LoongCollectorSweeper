## Summary

Keep open. Current main still has the old `sessions_spawn` title, the old `/subagents spawn` success reply, and no accepted spawn-result `resolvedModel` field for the slash command to report. Related model-selection work is already implemented on main, but it does not cover this PR's user-facing start notice/reporting change. The PR is still useful, though the supplied review comments identify safety/regression issues that need fixes before merge.

## Best Possible Solution

Keep the PR open and request a narrower, safer revision: return `resolvedModel` from accepted native subagent spawns, update both tool-display resources and tests, format `/subagents spawn` from the raw resolved model string without truncating slash-containing IDs, and keep start-notice metadata on the existing redaction/truncation path. Add regression coverage for redacted or long prompts, rejected spawns, ACP runtime behavior, and resolved-model reporting.
