## Summary

Keep open. Current main still uses the old sessions_spawn title and /subagents spawn success reply, and SpawnSubagentResult does not expose resolvedModel to the command path. The PR is not implemented on main or clearly superseded, but the proposed diff needs revision because the provided review context flags redaction, truncation, false-positive start notice, and model-ref formatting risks.

## Best Possible Solution

Keep the PR open for maintainer/contributor revision. The best implementation is a small, focused change that returns the actual resolved subagent model from the native spawn path, formats /subagents spawn with that value without lossy model splitting, and updates the start notice/title while preserving existing redaction, summary length normalization, and failure-reporting behavior. ACP runtime notices should avoid claiming a native subagent model unless that runtime actually uses it.
