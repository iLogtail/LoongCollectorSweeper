## Summary

Keep PR #61444 open. Current main still does not provide `openclaw agents import` or an equivalent agent-archive restore path, and the provided related items cover different agent/model commands rather than this import workflow. The PR is plausible core CLI work because it mutates agent config and workspaces, but it still needs maintainer review and fixes for the remaining validation and partial-write concerns raised in review.

## Best Possible Solution

Keep this PR or a maintainer-owned successor open as the canonical path for `openclaw agents import`. A landable version should validate the full `agent.json` shape before use, avoid trusting archive-controlled workspace paths by default, perform reserved-id and overwrite checks before filesystem writes, minimize partial imports if config writes or workspace copies fail, use the existing archive safety helpers, and keep CLI docs plus focused tests aligned.
