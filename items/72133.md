## Summary

Keep #72133 open. Current main partially covers channel replies with the existing `/usage off|tokens|full` per-response footer, but the requested mobile and richer channel metadata is not fully shipped: the footer lacks context-window percentage/model, Swift/iOS chat decodes per-message usage but does not render it, and Android chat neither models nor renders usage metadata.

## Best Possible Solution

Keep the issue open. The best path is to reuse the existing gateway usage/session metadata, add a compact usage footer component to the shared Swift/iOS chat UI, add usage/cost/context fields plus parsing/rendering to Android chat, and extend the existing channel `/usage` footer to optionally include model and `% ctx` when `contextTokens` and fresh session totals are available.

## Risks / Open Questions

- Channel footers need an opt-in/default policy because appending cost/context data to third-party chat surfaces can add noise or leak operational details into shared conversations.
- Context percentage should be defined consistently, likely from cumulative session total over `contextTokens`, not just last-call input/output tokens.
