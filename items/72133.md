## Summary

Keep #72133 open. Current main already covers the Control UI metadata and an optional chat/channel reply footer via `/usage`, which overlaps positively with related #8485, but the requested native iOS/Android chat view support is still missing: iOS decodes usage but does not render it, and Android neither carries usage in its chat model nor renders a footer.

## Best Possible Solution

Keep the issue open, scoped to native mobile parity: render a compact assistant-message usage footer in OpenClawKit/iOS and Android, reuse the existing token/cache/cost/context semantics from the web renderer and `/usage` formatter, and compute context percentage from the current model context window where available. Treat channel footer support as already implemented through `/usage`, with any follow-up limited to documentation or a clearer per-channel toggle if maintainers want that product surface.
