## Summary

本 Issue 请求为 flusher_http 增加便捷配置以提取指定 JSON 字段作为 HTTP Body 输出。经核查当前 main 分支，flusher_http 仍依赖 Convert/Serializer 机制进行完整日志序列化，未内置单字段提取选项。该需求涉及数据转换职责归属（Processor 链路 vs Flusher 便捷配置），属于需维护者产品裁决的可选特性，描述清晰且具备实际场景价值，应保持开启以待评估。

## Best Possible Solution

保持开启以待维护者产品裁决。建议维护者评估是否将此便捷提取逻辑纳入 flusher_http 核心配置，或明确推荐通过标准 Processor 链路（如前置 processor_json/processor_drop 组合或自定义 Processor）完成字段裁剪与重组后再交由 flusher_http 发送。若决定不纳入核心，可补充标准 Processor 实践文档或示例配置后关闭本 Issue。
