## Summary

本 Issue 请求为 flusher_http 增加指定字段作为 HTTP Body 输出的便捷配置。经核查当前 main 分支，flusher_http 仍依赖 Serializer/Convert 机制进行整体日志序列化，尚未内置单字段提取配置。该需求属于合理的核心插件增强，需维护者评估是纳入核心开发还是通过标准 Processor 链路替代实现。

## Best Possible Solution

保持开启以待维护者产品裁决。建议评估在 flusher_http 中新增 BodyField/RawLogField 配置项的可行性与优先级；或明确文档指引，推荐用户通过前置 Processor 链路完成字段提取与裁剪后，再交由 flusher_http 发送。若决定不纳入核心，可补充标准 Processor 实践文档后关闭。

## Evidence

- **current_flusher_http_config:** 检索 plugins/flusher/http/ 目录下的配置结构体与序列化逻辑，未发现 raw_log_field、BodyField 或等效的单字段 Body 提取配置。当前 flusher_http 依赖 Convert/Serializer 接口处理完整日志事件。
  - file: [plugins/flusher/http/flusher_http.go:40](https://github.com/alibaba/loongcollector/blob/51fbfff3078b79fce69fedae129b1fbbe29716e7/plugins/flusher/http/flusher_http.go#L40)

  - sha: [51fbfff3078b](https://github.com/alibaba/loongcollector/commit/51fbfff3078b79fce69fedae129b1fbbe29716e7)
- **processor_alternative_architecture:** LoongCollector 架构中，字段提取与结构重组通常由前置 Processor 承担。flusher_http 本身专注于传输与序列化，是否将此类便捷提取逻辑下沉至 Flusher 需维护者产品裁决。
