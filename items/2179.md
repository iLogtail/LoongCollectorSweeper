## Summary

本 Issue 请求为 flusher_http 增加 raw_log_field 配置以直接提取指定字段作为 HTTP Body。经核查当前 main 分支，flusher_http 尚未内置该单字段提取配置，仍依赖 Serializer/Convert 机制进行整体序列化。该需求属于合理的核心插件配置增强，虽理论上可通过前置 Processor 链路变通实现，但是否作为便捷配置纳入核心需维护者产品裁决。符合保守审查的保持开启条件。

## Best Possible Solution

保持开启以待维护者产品裁决。建议评估在 flusher_http 中新增 BodyField/RawLogField 配置项的可行性与优先级；或明确文档指引，推荐用户通过前置 Processor（如 processor_json/processor_drop/processor_rename）完成字段提取与裁剪后，再交由 flusher_http 发送。若维护者决定不纳入核心开发，可补充标准 Processor 链路实践文档后关闭本 Issue。

## Evidence

- **current_flusher_http_architecture:** 检索 plugins/flusher/http/ 目录下的配置结构体与序列化逻辑，未发现 raw_log_field 或等效的单字段 Body 提取配置。当前 flusher_http 仍依赖 Convert/Serializer 机制处理完整日志组，无法直接满足本 Issue 诉求。
