## Summary

本 Issue 请求为 flusher_http 增加指定 JSON 字段值作为原始 HTTP Body 输出的功能，属于明确且合理的核心组件配置增强需求。

## Best Possible Solution

保持开启以待维护者评估。建议审查 flusher_http 现有的 Convert/Serializer 机制是否可通过字段选择或预处理链路间接实现；若架构不支持，可在 flusher_http 配置中新增 BodyField/RawLogField 参数直接支持该场景。需维护者进行产品边界裁决与实现规划。

## Evidence

- **feature_request_analysis:** 用户明确要求新增 raw_log_field 类配置，使 flusher_http 能提取日志中特定嵌套字段（如 contents）作为完整 HTTP Body 发送，剥离 tags/time 等元数据。需求场景清晰（ingress-nginx 日志转发），具有实际业务价值。
  - sha: [7099f790b8a3](https://github.com/alibaba/loongcollector/commit/7099f790b8a38e6bf00b30a2a49b54a230791577)
- **current_implementation_gap:** 当前 main 分支 flusher_http 的 Convert/Serializer 机制主要面向全量字段格式化或固定模板渲染，尚未提供将单一嵌套字段直接提升为根级 Body 的原生配置项。该需求无法仅通过现有配置优雅实现，需核心代码改动或新增配置参数。
  - file: [plugins/flusher/http/flusher_http.go](https://github.com/alibaba/loongcollector/blob/7099f790b8a38e6bf00b30a2a49b54a230791577/plugins/flusher/http/flusher_http.go)
