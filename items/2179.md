## Summary

本 Issue 请求为 flusher_http 增加指定 JSON 字段作为原始 HTTP Body 输出的配置项（如 raw_log_field）。经核查，当前 main 分支的 flusher_http 依赖 Convert/Serializer 机制进行整体序列化，尚未提供单字段提取为 Raw Body 的原生配置。该需求属于合理的核心插件配置增强，且无证据表明已通过其他 PR 合并或可由现有配置直接替代。建议保持开启，待维护者评估是纳入核心实现，还是推荐通过前置 Processor 链路完成。

## Best Possible Solution

保持开启以待维护者产品裁决。建议评估在 flusher_http 中新增 BodyField/RawLogField 配置项的可行性；或明确文档指引，推荐用户通过前置 Processor（如 processor_json/processor_drop/processor_rename）完成字段提取与裁剪后，再交由 flusher_http 发送。若纳入核心开发，可补充设计草案并标记 help wanted。

## Evidence
