## Summary

本 Issue 请求为 flusher_http 增加指定字段作为 HTTP Body 输出的便捷配置。经核查当前 main 分支，flusher_http 仍依赖 Serializer/Convert 机制进行整体日志序列化，尚未内置单字段提取配置。该需求涉及 Flusher 与 Processor 的职责边界，需维护者进行产品裁决。

## Best Possible Solution

保持开启以待维护者评估。建议维护者确认是否将此类字段提取逻辑作为 flusher_http 的便捷配置纳入核心，或明确推荐通过标准 Processor 链路（如前置 processor_json/processor_drop 组合或自定义 Processor）完成字段裁剪后再交由 flusher_http 发送。若决定不纳入核心，可补充标准 Processor 实践文档后关闭。

## Evidence

- **current_flusher_http_config:** 检索 plugins/flusher/http/ 目录下的配置结构体与序列化逻辑，未发现 raw_log_field、BodyField 或等效的单字段 Body 提取配置。当前 flusher_http 依赖 Convert/Serializer 接口处理完整日志事件。
  - file: [plugins/flusher/http/flusher_http.go:40](https://github.com/alibaba/loongcollector/blob/d3305220b0b8b9bd4339df9441ace92983c2253f/plugins/flusher/http/flusher_http.go#L40)
  - sha: [d3305220b0b8](https://github.com/alibaba/loongcollector/commit/d3305220b0b8b9bd4339df9441ace92983c2253f)
- **processor_alternative_architecture:** LoongCollector 架构中，字段提取与结构重组通常由前置 Processor 承担。flusher_http 本身专注于传输与序列化，是否将此类便捷提取逻辑下沉至 Flusher 需维护者产品裁决。
  - file: [plugins/processor/README.md:1](https://github.com/alibaba/loongcollector/blob/d3305220b0b8b9bd4339df9441ace92983c2253f/plugins/processor/README.md#L1)
