## Summary

本 Issue 请求为 flusher_http 增加指定 JSON 字段作为原始 HTTP Body 输出的配置项。经核查当前 main 分支代码，flusher_http 的序列化机制尚未支持该特性，属于合理的核心插件配置增强需求，需维护者评估产品边界或确认是否推荐通过前置 Processor 链路实现。

## Best Possible Solution

保持开启以待维护者评估。建议审查 flusher_http 现有的 Convert/Serializer 机制是否可通过配置扩展支持 BodyField/RawLogField 参数；或评估是否应引导用户通过前置 Processor（如 processor_json/processor_drop）完成字段提取与裁剪后再交由 flusher_http 发送。若决定纳入核心实现，可标记为 good first issue 或 help wanted 并补充设计草案。
