## Summary

本 Issue 请求为 flusher_http 增加单字段提取配置。经评估，该需求属于日志数据裁剪与重组范畴，完全可通过现有 Processor 插件链路实现，符合架构职责边界，无需侵入 Flusher 核心。

## Best Possible Solution

通过标准 Processor 链路（processor_drop/processor_rename/processor_packjson）完成字段提取与结构重组，再交由 flusher_http 发送。维护者可考虑在文档中补充此类常见场景的 Processor 最佳实践配置示例。

## Evidence

- **processor_architecture_boundary:** LoongCollector 架构明确划分数据处理与输出职责。字段提取、裁剪与结构重组由前置 Processor 链路承担，Flusher 仅负责序列化与网络传输。
  - file: [plugins/processor/README.md:1](https://github.com/alibaba/loongcollector/blob/d052b5f8a7dbdb6c8c91effa74d7ca9542ed02ae/plugins/processor/README.md#L1)
  - sha: [d052b5f8a7db](https://github.com/alibaba/loongcollector/commit/d052b5f8a7dbdb6c8c91effa74d7ca9542ed02ae)
- **flusher_http_config_scope:** flusher_http 配置聚焦于 HTTP 请求参数、序列化器与重试策略，未内置且不建议内置业务字段提取逻辑，保持职责单一。
  - file: [plugins/flusher/http/flusher_http.go:45](https://github.com/alibaba/loongcollector/blob/d052b5f8a7dbdb6c8c91effa74d7ca9542ed02ae/plugins/flusher/http/flusher_http.go#L45)
