## Summary

本 Issue 请求为 flusher_http 增加指定 JSON 字段作为原始 HTTP Body 输出的配置能力。经核查当前 main 分支代码，flusher_http 的序列化与转换机制尚未原生支持提取单一嵌套字段作为完整 Body 发送。该需求场景清晰（如 ingress-nginx 日志剥离元数据转发），属于合理的核心插件配置增强，需维护者评估产品边界与实现方案。证据不足且非陈旧/重复，故保持开启。

## Best Possible Solution

保持开启以待维护者评估。建议审查 flusher_http 现有的 Convert/Serializer 机制是否可通过配置扩展支持 BodyField/RawLogField 参数，或确认是否应通过前置 Processor 链路（如 processor_rename/processor_drop）实现该字段提取逻辑。若决定接纳，可标记为 enhancement 并分配里程碑。

## Evidence

- **feature_request_clear:** 用户明确要求新增 raw_log_field 类配置，使 flusher_http 能提取日志中特定嵌套字段（如 contents）作为完整 HTTP Body 发送，剥离 tags/time 等元数据。需求场景清晰，具有实际业务价值。
- **implementation_gap:** 当前 main 分支 flusher_http 的 Convert/Serializer 机制主要面向全量字段格式化或固定模板渲染，尚未提供将单一嵌套字段直接提升为根级 Body 的原生配置项。该需求无法仅通过现有配置优雅实现，需核心代码改动或新增配置参数。
