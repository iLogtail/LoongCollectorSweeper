## Summary

Keep open. Current main at 6a688e33f66f3965968343922e874cc0e9440998 still does not implement a `tools.exec.shell` config surface, and the PR discussion identifies real follow-up work for PTY and gateway approval paths rather than a cleanup-safe close.

## Best Possible Solution

Keep PR #39413 open until OpenClaw either implements a supported `tools.exec.shell` contract across config types, zod schema, docs, direct exec, PTY, and gateway approval paths, or a maintainer explicitly decides custom shell selection is out of scope. Related PR #65271 overlaps on Windows shell customization but is narrower and also open, so it is not strong enough evidence for a duplicate/superseded close of this broader config-surface PR.
