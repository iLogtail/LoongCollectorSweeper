## Summary

Keep open. Current main still does not expose or thread a `tools.exec.shell` configuration option, and the PR review discussion identified real direct exec, PTY, and gateway approval paths that would need a coordinated implementation before this feature is complete or obsolete.

## Best Possible Solution

Keep this PR or create a canonical follow-up until maintainers decide the shell override product contract. A complete fix should add the config schema/type/docs surface and thread the selected shell consistently through direct exec, PTY supervisor execution, and gateway approval follow-up runs, with focused tests for precedence and fallback behavior.
