## Summary

Keep open. Current main still does not implement `tools.exec.shell` or an equivalent global/per-agent shell override, and the PR remains a plausible implementation attempt for a real exec-tool gap. The related #49931 issue tracks the broader request, but it is open and does not supersede this PR with a shipped implementation or maintainer decision.

## Best Possible Solution

Keep the PR open for maintainer review or explicit consolidation with #49931. A complete solution should define the supported `tools.exec.shell` contract, add typed global and per-agent config plus schema/help/docs, route the selected shell through direct exec, PTY supervisor, gateway approval follow-up runs, and node/sandbox policy decisions, then add focused regression tests for precedence, fallback, Windows arguments, validation, approval paths, and docs/schema drift.
