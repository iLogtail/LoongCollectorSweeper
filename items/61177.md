## Summary

Keep PR #61177 open. Current main still lacks any `SO_NOSIGPIPE` or process-wide `SIGPIPE` suppression on the macOS exec approval socket path, while the PR is a focused fix with a concrete repro, a regression test, and a review comment saying the core change is correct apart from a harmless listener-fd nit. This is not implemented, duplicate, stale, or out of scope.

## Best Possible Solution

Keep the PR open and either merge or adapt its narrow fix: add a small descriptor guard that applies `SO_NOSIGPIPE` to the client socket before `FileHandle.write` and to each accepted server-side client fd before response writes, with a regression test that verifies the option via `getsockopt`. Address the review nit by omitting the listener-fd guard if maintainers want the implementation to show that accepted sockets are handled explicitly.
