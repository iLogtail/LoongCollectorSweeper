## Summary

Keep open. Current `main` does not implement this PR: the Overview WebSocket URL input still clears `settings.token` on URL edits, and browser tests explicitly assert that behavior. The PR’s one-line change would bypass the current per-gateway token scoping model and send an existing bearer token to a newly entered gateway URL without confirmation, so there is not strong evidence for a safe cleanup close.
