## Summary

Keep open. Current main still clears the in-memory gateway token when the Overview WebSocket URL changes, so the underlying bug is not implemented on main. However, PR #41552's exact patch is not clearly safe because it would carry the existing token to a newly selected gateway URL, conflicting with the current per-gateway token-scoping contract and the existing review warning.

## Best Possible Solution

Keep the item open for maintainer judgment or a revised patch. The best fix should improve the Overview editing UX without reusing a token across different normalized gateway scopes, likely by separating a draft WebSocket URL from the committed selected gateway URL and only clearing or switching token scope when the new gateway is confirmed.
