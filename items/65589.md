## Summary

Keep #65589 open. Current main has important mitigations for the linked runaway incident, including queued-cron gating, default separate dreaming storage, candidate dedupe, and promotion thresholds, but it does not implement the PR’s central cost/budget circuit breaker. The PR also is not merge-ready as submitted because its enforcer was only added as an integration sketch and was never wired into the live dreaming pipeline.

## Best Possible Solution

Do not merge #65589 as submitted. Either ask for a narrow rework that wires a real dreaming budget/session guard into `runDreamingSweepPhases`/`generateAndAppendDreamNarrative` with config schema and regression tests, or have a maintainer explicitly decide that the shipped cron/storage/threshold fixes are sufficient and close it manually as superseded while tracking remaining session/model/burst cleanup in #65963.
