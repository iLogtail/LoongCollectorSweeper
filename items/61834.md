## Summary

Keep open. Current main still does not expose QMD no-rerank at either config level or memory_search request level. The QMD config/resolved types, config docs/help, direct CLI arg builder, and mcporter MCP call builder all lack a noRerank/rerank control and never emit --no-rerank or rerank:false.

## Best Possible Solution

Keep the issue open and implement it in the bundled memory-core/QMD backend rather than closing as cleanup. A complete fix should add a typed QMD rerank/noRerank control, thread it through config schema/help/docs and resolved backend config, append --no-rerank for direct qmd query only when applicable, pass rerank:false for supported MCP qmd.query calls, and add focused tests for direct CLI, mcporter, config resolution, and memory_search request behavior if a request-level option is chosen.
