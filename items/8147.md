## Summary

Keep #8147 open. Current main still sends Control UI/WebChat messages on plain Enter, with only Shift+Enter and IME composition exemptions. There is no Cmd/Ctrl+Enter send branch, no chat send-shortcut prop, no persisted UI setting, and no gateway.webchat config/docs option for this behavior. The related context does not make this closeable: #39821 was closed as a duplicate, PR #38363 was closed unmerged, and #19859 is an overlapping open follow-up rather than a shipped fix.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed Control UI/WebChat send-shortcut preference or make an explicit product decision to change the default, persist it in the appropriate UI/settings surface, update composer placeholder and send-button shortcut hints, document any new config/user setting, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
