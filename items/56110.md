## Summary

Current `main` does not implement `STATE.md` as an auto-loaded workspace bootstrap file. The documented bootstrap set omits it, the runtime bootstrap enumeration omits it, and the bundled `bootstrap-extra-files` hook explicitly rejects unrecognized basenames like `STATE.md`, so this request is still actionable and should stay open.

## Evidence

- **System prompt docs omit STATE.md:** The workspace bootstrap injection docs list `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and optional `MEMORY.md`; `STATE.md` is not listed.
  - file: [docs/concepts/system-prompt.md:99](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/concepts/system-prompt.md#L99)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Runtime bootstrap file set omits STATE.md:** `VALID_BOOTSTRAP_NAMES` and `loadWorkspaceBootstrapFiles()` only recognize/load the existing eight bootstrap filenames plus `MEMORY.md`; there is no `STATE.md` constant or entry.
  - file: [src/agents/workspace.ts:172](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/workspace.ts#L172)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Extra bootstrap hook cannot load STATE.md:** The documented and implemented `bootstrap-extra-files` hook only accepts recognized bootstrap basenames, and `loadExtraBootstrapFilesWithDiagnostics()` emits `invalid-bootstrap-filename` for anything else. That means users cannot currently opt into `STATE.md` through the existing hook path.
  - file: [docs/automation/hooks.md:192](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/automation/hooks.md#L192)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Hook implementation delegates to basename validation:** The bundled `bootstrap-extra-files` hook loads extras through `loadExtraBootstrapFilesWithDiagnostics()`, so the runtime basename restriction applies to hook-loaded files too.
  - file: [src/hooks/bundled/bootstrap-extra-files/handler.ts:41](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/hooks/bundled/bootstrap-extra-files/handler.ts#L41)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **No STATE.md references on main:** Repository search found no `STATE.md` references under `docs`, `src`, `packages`, `extensions`, `ui`, or `test`, which is consistent with the feature not being present.
  - command: `rg -n "STATE\.md" docs src packages extensions ui test --glob '!**/*.generated.ts'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)

## Risks / Open Questions

- `docs/concepts/system-prompt.md` says subagents only inject `AGENTS.md` and `TOOLS.md`, but `src/agents/workspace.ts` currently also allows `SOUL.md`, `IDENTITY.md`, and `USER.md` for subagent/cron sessions. That inconsistency does not change the conclusion that `STATE.md` is unimplemented.
