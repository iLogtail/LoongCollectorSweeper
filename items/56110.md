## Summary

Keep #56110 open. Current main still does not support `STATE.md` as an auto-loaded workspace bootstrap file, and the related open issues do not fully supersede this request: #59050 is about ordering existing bootstrap files, while #39120 is about arbitrary extra-file injection rather than making `STATE.md` a first-class default bootstrap file.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A conservative implementation would add a `DEFAULT_STATE_FILENAME`, include it in the bootstrap filename type/validation and `loadWorkspaceBootstrapFiles()` ordering, add prompt sort order near `MEMORY.md`, decide whether missing `STATE.md` should inject a marker or be optional like `MEMORY.md`, exclude it from subagent/lightweight contexts as intended, and update docs plus focused bootstrap tests.

## Risks / Open Questions

- A custom internal hook can mutate `context.bootstrapFiles`, but the public typed/documented bootstrap filename path is still fixed and does not make `STATE.md` a standard workspace bootstrap file.
- Subagent filtering needs explicit maintainer design if `STATE.md` is added, because current code and docs are not perfectly aligned on which non-AGENTS/TOOLS files subagent sessions receive.
