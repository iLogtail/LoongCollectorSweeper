## Summary

Keep open. Current main still has the three generic transport-stream throws this PR targets, and the existing shared-helper test file does not cover propagating a preexisting provider `output.errorMessage`. The related discussion is useful but does not supersede this narrow fix.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent small fix on main: change the three transport-stream throw sites to prefer `output.errorMessage || "An unknown error occurred"`, then add focused shared-helper regression coverage for both provider-message and fallback branches.
