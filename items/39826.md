## Summary

Keep PR #39826 open. Current main still emits the Discord no-mention guild-drop log with only `channelId` and `reason`; `accountId` and `guildId` are available in the same preflight scope but are not included in that structured log payload. The PR is stale against the old `src/discord/...` path, but the underlying two-field diagnostic improvement remains valid for `extensions/discord/...`.

## Best Possible Solution

Keep PR #39826 open or reapply the same log-only change on current main in `extensions/discord/src/monitor/message-handler.preflight.ts`: add `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` object, ideally with a focused test assertion for that structured payload.
