## Summary

Current main still does not implement PR #39826. The Discord no-mention guild drop log remains limited to `channelId` and `reason`, so the PR's accountId/guildId diagnostic improvement is still useful and should not be closed.

## Best Possible Solution

Keep PR #39826 open, or apply the same log-only change to `extensions/discord/src/monitor/message-handler.preflight.ts` on current main by adding `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` object. Prefer adding a focused regression assertion for that structured log payload if feasible.
