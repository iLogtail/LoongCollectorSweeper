## Summary

Keep open. Current main still omits `accountId` and `guildId` from the Discord `no-mention` auto-reply drop log, so the PR's focused logging improvement has not been implemented or superseded.

## Best Possible Solution

Keep the PR open and rebase or recreate the same additive change in `extensions/discord/src/monitor/message-handler.preflight.ts`, ideally with a narrow test assertion that the `discord-auto-reply` no-mention log includes `accountId`, `guildId`, `channelId`, and `reason`.
