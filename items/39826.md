## Summary

Keep PR #39826 open. Current main still logs the Discord no-mention guild drop with only `channelId` and `reason`, while the requested `accountId` and `guildId` fields are already available in the same preflight scope. The PR is stale against the old `src/discord/...` path, but the underlying log-only diagnostic improvement remains valid for the current `extensions/discord/...` implementation.

## Best Possible Solution

Keep PR #39826 open or apply the same log-only fix on current main in `extensions/discord/src/monitor/message-handler.preflight.ts` by adding `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` object. Prefer adding a small regression assertion for that structured log payload if feasible.
