## Summary

Keep #70472 open. Current main still contains the reported compaction-failure reset/rotation behavior: compaction failures call a reset helper that writes a new sessionId for the same session key, and missing compaction checkpoint store entries still only warn and return null. The linked PR #70479 is explicitly the pending fix and is open/unmerged in the provided GitHub context, so the canonical issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #70472 open and land #70479 or an equivalent mainline fix: preserve the active WebChat session mapping on overflow/compaction failure, avoid hidden sessionId rotation unless explicitly requested, make recovery/audit events visible, and add regression coverage for agent:main:main continuity after compaction/checkpoint failure.
