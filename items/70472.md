## Summary

Keep open. Current main still has the reported compaction/overflow failure path wired to session reset/rotation, and the focused fix PR #70479 is still open/unmerged in the provided GitHub context. The issue remains a real product bug rather than cleanup-closeable.

## Best Possible Solution

Keep #70472 open until the product path preserves the active session mapping on compaction/overflow failure or intentionally records an auditable recovery event. The focused direction is to land or revise #70479 so failed compaction produces explicit retry guidance without rotating agent:main:main, then add regression coverage around overflow/compaction failure followed by a subsequent WebChat message using the same sessionId.
