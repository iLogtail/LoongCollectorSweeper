## Summary

Keep #70472 open. Current main still contains the reported checkpoint sentinel and still has an auto-recovery path that rotates the active session after context-overflow or compaction-failure handling. The related fix PR #70479 is explicitly tied to this issue but is open/unmerged in the provided context and its head commit is not present in this checkout.

## Best Possible Solution

Keep #70472 open until PR #70479 or an equivalent mainline fix lands. The fix should remove the automatic compaction-failure session rotation for WebChat main sessions, preserve the active `agent:main:main` mapping unless the user explicitly resets, add an auditable recovery/rotation event if a new session is unavoidable, and add regression coverage around overflow/compaction failure followed by a subsequent WebChat message continuing the same active session id.
