## Summary

Current main still appears vulnerable to the reported Control UI round-trip failure: internal exec approval allowlist metadata can contain `source`, gateway/node snapshots do not strip it, the Control UI preserves the loaded file when saving, and `exec.approvals.set` still rejects unknown allowlist properties. The related PR #69732 is directly relevant but is open/unmerged in the provided context, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #69698 open and land PR #69732 or an equivalent current-main fix that redacts internal exec approval allowlist metadata (`source`, and any other transport-only/internal fields) from gateway and node-host approval snapshots before the Control UI can round-trip them. Add regression coverage for both `exec.approvals.get`/`set` and `exec.approvals.node.get`/`set` snapshot transport behavior.
