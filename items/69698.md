## Summary

Current main still has the reported round-trip contract mismatch. Exec approval allowlist entries can carry persisted internal `source: "allow-always"` metadata, the gateway snapshot redaction currently only trims socket data, and the Control UI sends the cloned snapshot back through `exec.approvals.set`. The protocol schema for set rejects allowlist entry properties outside `id`, `pattern`, `argPattern`, and usage metadata, so `source` is still an invalid property on save. The linked PR #69732 is directly relevant but remains open/unmerged in the provided GitHub context, so this issue should stay open until that or an equivalent fix lands on main.

## Best Possible Solution

Keep #69698 open until #69732 or an equivalent main-branch fix redacts internal exec approval allowlist metadata from transport snapshots, covers both gateway and node exec approvals paths, and adds a regression test proving `exec.approvals.get` payloads can be saved back through `exec.approvals.set` without rejected `source` properties while preserving internal metadata in the store.
