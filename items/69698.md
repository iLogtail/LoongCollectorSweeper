## Summary

Current main still has the reported Control UI round-trip bug: allow-always allowlist entries can contain internal `source` metadata, `exec.approvals.get` returns the file without stripping that metadata, the UI clones and sends the snapshot file back on save, and `exec.approvals.set` validates against a schema that rejects extra allowlist properties. The linked fix PR #69732 is open and unmerged, so the issue should remain open.

## Best Possible Solution

Keep #69698 open until PR #69732 or an equivalent main-branch fix lands. The fix should redact internal allowlist metadata such as `source` from `exec.approvals.get` and node snapshots, or sanitize the UI payload before `exec.approvals.set`, with regression coverage for saving a source-bearing allowlist snapshot from Control UI.
