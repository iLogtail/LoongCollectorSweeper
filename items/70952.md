## Summary

Keep open. Current main has useful exact-SecretRef deduping/batching for a single `resolveSecretRefValues` call, but I did not find the requested startup-resolution scoped provider lookup memoization. The gateway startup path can still prepare the full secrets runtime snapshot twice when active gateway auth SecretRefs are present, and batch resolver calls do not populate/read the full-ref cache across runtime phases for exec refs.

## Best Possible Solution

Keep the issue open and implement a scoped core fix: make `resolveSecretRefValues` honor/populate `SecretRefResolveCache` for exact refs and batch results, avoid or share cache across the startup preflight/final activation path where safe, and add a regression test with a counting exec/provider fixture. If vault/item-level 1Password Connect caching is provider-specific, document and implement it in a batch-aware resolver while preserving the generic SecretRef contract.
