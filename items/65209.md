## Summary

Keep open. Current main still matches the reported CLI UX problem: `openclaw mcp list` reads only OpenClaw `mcp.servers`, but its command description and human-readable output still say generic "configured MCP servers" / "MCP servers" and do not mention mcporter. Docs now explain the OpenClaw-owned registry boundary, and the closed PR #69726 proposed the exact low-risk CLI/docs/test clarification, but that PR is closed unmerged and its changes are not present in the checkout.

## Best Possible Solution

Keep #65209 open and apply a narrow version of PR #69726: relabel `openclaw mcp list/show/set/unset` help/output as OpenClaw-managed `mcp.servers`, add a non-JSON note that mcporter `config/mcporter.json` servers are not included, keep `--json` output unchanged, and add focused CLI regression tests. The broader native mcporter/MCP discovery work should remain separate from this UX clarification.
