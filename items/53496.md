## Summary

Keep #53496 open. Current main still does not implement Feishu group `toolsBySender`: the Feishu schema accepts only a local `tools` policy with `allow`/`deny`, the runtime resolver returns only `groupConfig?.tools`, and Feishu generated metadata/docs do not expose `channels.feishu.groups.*.toolsBySender`. The shared `resolveChannelGroupToolsPolicy` support exists in core, so this PR is a plausible narrow Feishu parity slice rather than obsolete work. The related broad PR #44602 was closed unmerged, and #53530 explicitly treats #53496 as the separate group-policy slice, not as a replacement.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to refresh it. The best implementation path is the narrow one described by the PR: make Feishu group config use the shared ToolPolicySchema, add `groups.*.toolsBySender`, delegate Feishu group policy resolution to `resolveChannelGroupToolsPolicy` with case-insensitive group matching, and keep focused Feishu schema/policy regression tests plus generated config metadata in sync.
