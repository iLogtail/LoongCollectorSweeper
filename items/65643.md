## Summary

Keep #65643 open. Current main still leaves `OPENCLAW_LAUNCHD_LABEL` intact when `--profile` is applied, while launchd plist resolution still gives that inherited label priority over `OPENCLAW_PROFILE`. The linked PR #66000 is still open and tracks a possible fix, but the issue remains valid until an equivalent fix lands.

## Best Possible Solution

Keep the issue open and land a targeted fix, likely alongside or instead of PR #66000: when `--profile` is explicitly provided, reconcile inherited stock gateway labels so the requested profile derives the correct plist, while preserving intentional custom `OPENCLAW_LAUNCHD_LABEL` overrides. Add regression tests in `src/cli/profile.test.ts` or the launchd/status path for inherited-label conflict, matching-label preservation, and custom-label behavior.
