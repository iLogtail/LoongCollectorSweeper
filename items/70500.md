## Summary

PR #70500 should stay open. Current main documents the safer signed-app/TCC model, but it still does not contain the PR's explicit warning that granting macOS Accessibility to a shared `node` runtime can extend GUI automation privileges to any package run by that Node executable. The predecessor #69654 is not a superseding merged PR; it has the same head SHA and was closed because the fork became private. The Greptile comment also supports that this docs-only change is technically accurate and safe to merge.

## Best Possible Solution

Keep PR #70500 open for maintainer review or merge. The best path is to land this focused docs warning, or an equivalent maintainer-authored wording, in `docs/platforms/mac/permissions.md` and link it from `docs/platforms/mac/peekaboo.md`, then run the repository docs check.
