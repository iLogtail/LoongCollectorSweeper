## Summary

Keep PR #39323 open. Current main still lacks owner notifications for new DM pairing requests, lacks a `pairing:request` hook or plugin hook for this event, and only supports requester-side pairing replies plus requester approval confirmation. The closed related issue #38998 does not supersede the PR; this PR remains a concrete implementation candidate for the remaining capability.

## Best Possible Solution

Keep the PR open as the active implementation candidate, or replace it with a maintainer-owned narrower design. The core/API work should add a documented, versioned pairing-request event or equivalent plugin hook emitted only for newly created DM pairing requests, then implement owner notification in core config or as a plugin once that event exists. Before merge, address Discord target normalization and add/update docs and focused tests for the final notification path.
