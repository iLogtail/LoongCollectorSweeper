## Summary

Keep #39323 open. Current main still does not implement owner notifications for new DM pairing requests: the channel pairing store creates pending requests without firing a pairing hook or outbound owner alert, the public CLI docs only document `--notify` as a post-approval confirmation to the requester, and the current hook/event surfaces do not expose a `pairing:request` event. The newer `device-pair` plugin does implement Telegram notifications for node device-pairing requests, but that is a separate pairing flow and does not supersede this PR’s DM pairing owner-notification feature.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-owned follow-up that adds a supported owner-notification path for DM pairing requests. The likely core work is a small, documented pairing-request event or plugin API plus a sanitized, non-agent outbound notification implementation with channel-specific target validation, tests, and docs. The existing Telegram `device-pair` notifier should remain as node-device-pairing behavior, not be treated as covering DM pairing owner alerts.
