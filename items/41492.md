## Close Comment

Closing from a Codex review because this regression is already implemented on current `main` and documented as shipped in `v2026.3.11` (`CHANGELOG.md:2673`, `CHANGELOG.md:2792`, #41711). The fallback runner now limits transient cooldown probes to one attempt per provider per fallback run (`src/agents/model-fallback.ts:746`), which avoids repeatedly retrying multiple cooldowned models from the same provider and allows cross-provider fallbacks to continue. The current regression tests cover both the reported rate-limit case and the one-probe-per-provider behavior (`src/agents/model-fallback.test.ts:1584`, `src/agents/model-fallback.test.ts:1633`), and the docs describe the same behavior (`docs/concepts/model-failover.md:286`).
