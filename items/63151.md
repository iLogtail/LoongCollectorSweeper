## Summary

Current `main` already contains and has shipped a fix for the reported late async callback race: exec/subagent update callbacks are disabled once the run ends or aborts, and the changelog for `v2026.4.23` records the exact crash string and scenario.

## Evidence

- **Current exec runtime suppresses late updates after run end:** `runExecProcess` now guards `emitUpdate()` with `updatesDisabled` and explicitly sets that flag before marking the session exited, preventing late stdout/stderr callbacks from reaching a disposed pi-agent run.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/bash-tools.exec-runtime.ts#L561)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
