## Summary

Current main already addresses the central gateway RSS/startup-loading problem by narrowing Gateway startup to a computed plugin id scope backed by the cold plugin registry, rather than by changing bundled `scanDir` precedence. The PR's exact `scanDir` reorder is not needed and would conflict with current explicit override semantics, which main preserves with source and tests.

## Best Possible Solution

Close #70730 and keep the current mainline startup-scoping/cold-registry approach. If #70717 still reproduces after the next release, continue that investigation against the current startup plugin-id selection and runtime-load paths rather than merging a `scanDir` precedence reversal.
