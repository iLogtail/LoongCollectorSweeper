## Summary

Do not auto-close this PR. Current main already has the non-mutating `toSorted()` change in `.pi/extensions/files.ts`, but the second paged-select change is not safely implemented on main or in the PR: the local helper type says `render()` returns `string`, while the pinned upstream Pi TUI `Component`/`Container` contract returns `string[]`, so joining to a string would fight the actual dependency contract rather than fix it cleanly.

## Best Possible Solution

Keep the PR open for maintainer/revision handling, but do not merge it as-is. The best fix is a narrow update to `.pi/extensions/ui/paged-select.ts` that types `CustomUiContext.ui.custom` against the actual Pi component contract (`render(): string[]`, correct `done(result)` shape, and preferably imported upstream types where feasible), leaves `container.render(width)` returning `string[]`, and drops the whitespace churn. The sort change can be omitted because it is already on current main.
