## Summary

Keep PR #70151 open. Current main still has the stale-snapshot empty-turn path: `runEmbeddedPiAgent` destructures the attempt assistant once, builds terminal payloads from `attempt.lastAssistant`, and has no recovery poll/readback for a later persisted assistant message. The PR is a focused runner bug fix with a regression test and no superseding related item in the provided context.

## Best Possible Solution

Keep the PR open and have maintainers review or port the core recovery idea onto current main: before surfacing the zero-payload incomplete-turn error, re-read the persisted session for a late final assistant message, use that refreshed assistant for payload building, and keep focused regression coverage in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts`.
