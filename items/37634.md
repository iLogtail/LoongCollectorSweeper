## Summary

Keep #37634 open. Current main still does not implement the requested isolated-but-writable `workspaceAccess: "none"` behavior: `none` selects the isolated sandbox workspace, but the mount layer and filesystem bridges still treat that workspace as read-only, and tests/docs encode that current behavior. Related fix PRs #37276 and #38154 were useful but are closed unmerged; the open related work does not supersede this focused bug yet.

## Best Possible Solution

Keep #37634 open as the focused tracker. The best fix is to preserve `workspaceAccess: "none"` isolation from the agent workspace while making the isolated sandbox workspace writable consistently across Docker/browser mounts, local and remote filesystem bridges, write gating, memory-flush behavior if applicable, regression tests, and docs; alternatively, make an explicit maintainer-approved product decision and then close or supersede related items accordingly.
