## Summary

Current main does not implement the requested behavior. `workspaceAccess: "none"` still uses an isolated sandbox workspace, but the Docker mount, local fs bridge, remote fs bridge, and tests all continue to treat that workspace as read-only. The related fix PRs in the provided GitHub context are closed and unmerged, so the issue is not superseded by merged work.
