## Summary

Keep #41304 open. Current main has a useful narrow fix for explicit allowlists that resolve to no callable tools, but the issue discussion reports broader behavior: registered plugin/MCP tools sometimes appear available while the agent gives simulated success or text instructions instead of executing. That broader tool-invocation/debugging problem is not fully proven fixed, the issue is not old enough for stale-insufficient-info, and no canonical duplicate in the provided context covers the remaining behavior.

## Best Possible Solution

Keep the issue open and narrow the maintainer path: ask affected users for a trajectory export or equivalent proof showing the final tool list, submitted tool schema, model/provider, and `tool_execution_start`/`tool_execution_end` events. If the remaining failures are only missing allowlisted tools, point them to the shipped no-callable-tools guard; if registered tools are present but not invoked, add a focused regression/live smoke around one plugin or MCP action tool and improve diagnostics for model tool support, profile filtering, and dispatch.
