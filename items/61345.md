## Summary

Keep PR #61345 open. Current main still sends every disconnected Control UI state through the login gate, and it does not have the PR's `shouldRenderLoginGate` helper or topbar `Connecting...` indicator. The PR addresses a real remaining UI-state bug rather than obsolete work.

## Best Possible Solution

Keep the PR open and review it as the likely fix: move the login-gate decision into a small tested helper that uses structured gateway error detail codes plus narrow string fallbacks, render a dashboard-shell `Connecting...` status for transient reconnects, and preserve login-gate behavior for missing gateway configuration and high-confidence auth/config/device-auth failures.
