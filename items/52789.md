## Summary

Keep #52789 open. Current main still models cron delivery as only `none`, `announce`, or `webhook`; the gateway schema, agent tool schema, UI type, docs, and delivery runtime do not implement a framework-owned `delivery.mode: "session"` path that sends finished cron output through `sessions_send`/`sessions.send` to another internal session. The provided related items cover adjacent cron delivery bugs and cron tooling, but none is a canonical tracker or implementation for this feature.
