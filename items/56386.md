## Summary

Keep open. Current main has not absorbed the PR's central behavior: stop/reset still do not await subagent stop propagation, and the stop-subagents cascade still does not cancel ACP-backed child sessions or persist child aborted state. The PR also has review feedback that should be addressed before merge rather than auto-closing it as obsolete.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to refresh the patch against current main by making `stopSubagentsForRequester` awaitable, adding bounded ACP cancellation for active child sessions, persisting child aborted state where appropriate, awaiting the cascade from stop/reset callers, and adding a passing regression test with the latest-run mock setup fixed.
