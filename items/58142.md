## Summary

Keep open. Current `main` does not implement per-skill model routing: skill metadata/types and frontmatter parsing do not support `model` or `modelProfile`, the public skills docs do not document such fields, and the embedded runner still drops `skillEntries` whenever a cached `resolvedSkills` snapshot is present. This is also not a strong `clawhub` close because the proposal changes core skill/runtime behavior rather than adding a new bundled skill.

## Evidence

- **Skill metadata has no per-skill model fields:** `OpenClawSkillMetadata` on current `main` only includes gating/UI/install fields (`always`, `skillKey`, `primaryEnv`, `emoji`, `homepage`, `os`, `requires`, `install`). There is no `model` or `modelProfile` field in the exported type.
