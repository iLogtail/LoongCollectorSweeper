## Summary

Keep #50701 open. Current main has partial transcript-backed usage support: reset/deleted archive transcripts are counted and can be discovered as separate usage sessions. The central request is still missing: no durable logical-session family/lineage model, no lineage-aware usage API contract, no response metadata for included historical session instances, and no WebUI current-instance vs historical-lineage toggle or 90d/1y/all presets. The KeepMyClaw comment is a useful external durability workaround, but it does not close the OpenClaw WebUI/backend lineage gap.

## Best Possible Solution

Keep #50701 open. The best product path is a core usage-history design that records or reconstructs a stable logical session family for a session key, exposes a lineage-aware sessions.usage API with explicit current-instance versus historical-lineage grouping, returns included sessionIds/instance count/first and last activity metadata, and updates the Control UI with a lineage toggle plus 90d/1y/all/custom range controls. Coordinate with #45003/#60409 archive-recovery work, but do not close this as their duplicate because this issue is specifically about usage rollups and WebUI presentation continuity.
