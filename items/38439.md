## Summary

Keep open. Current main has the `/avatar/<agentId>` route, workspace `IDENTITY.md` fallback, metadata, and 2 MB size-limit reporting, but the exact reported Windows/non-default-agent webchat path is still not proved fixed on main. The linked open PR #38923 still tracks the remaining IDENTITY.md-only, path-normalization, backtick parsing, and regression-test work.

## Best Possible Solution

Keep #38439 open until #38923 or an equivalent maintainer-reviewed fix lands. The narrow best fix is to normalize Windows namespaced and drive-letter paths before avatar containment checks, strip single-backtick wrappers from `IDENTITY.md` avatar values, add a non-default-agent Control UI/webchat regression that resolves a workspace-relative `IDENTITY.md` avatar through the real route, and preserve the existing `too_large` metadata/UI explanation.
