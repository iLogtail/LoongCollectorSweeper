## Summary

Keep #38439 open. Current main still has the Windows namespaced-path containment gap described in the issue discussion, so a valid workspace avatar can still resolve as `outside_workspace`, causing `/avatar/<agentId>?meta=1` to return `avatarUrl: null` and `/avatar/<agentId>` to return 404. The linked fix PR #38923 is still separate from current main.

## Best Possible Solution

Keep #38439 open until #38923 or an equivalent maintainer-reviewed fix lands on main. The fix should normalize Windows namespaced and drive-letter paths before avatar containment checks, preserve workspace-boundary and symlink protections, keep the existing `too_large` behavior, and add regression coverage for a non-default Windows-style agent resolving a workspace-relative `IDENTITY.md` avatar through both `/avatar/<agentId>?meta=1` and `/avatar/<agentId>`.
