## Summary

Keep #71035 open. Current main still does not expand `${ENV_VAR}` in HTTP MCP launch `url` or `headers`; it trims the URL and stringifies header values only. The PR targets a real gap that is also contradicted by current docs, but it needs regression tests and maintainer resolution of the empty-env review comment and URL-schema scope.

## Best Possible Solution

Keep the PR open and land a revised, tested fix: implement expansion in `src/agents/mcp-http.ts`, add focused coverage in `src/agents/mcp-transport-config.test.ts` for header interpolation and unresolved/empty env behavior, and either align `mcp.servers.*.url` schema/docs for URL interpolation or narrow the PR to headers only.
