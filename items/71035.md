## Summary

Keep #71035 open. Current main still does not expand `${ENV_VAR}` placeholders in HTTP MCP `url` or `headers`; it trims/parses the URL as-is, stringifies headers as-is, and passes them directly into the MCP SDK transports. The PR is a focused, relevant fix for a real docs/runtime mismatch, though it should add regression coverage and resolve the existing empty-env review concern before merge.

## Best Possible Solution

Keep the PR open and land this behavior, or an equivalent maintainer patch, at the HTTP MCP launch-config boundary in `src/agents/mcp-http.ts`. Add targeted tests in the MCP transport/config area and decide explicitly whether empty env vars should remain visible as unresolved placeholders or fail validation.

## Risks / Open Questions

- The submitted PR currently treats an empty environment variable as a successful substitution, producing values like `Bearer `; the existing review comment correctly flags that this may hide misconfiguration.
