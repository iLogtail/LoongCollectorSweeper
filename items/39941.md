## Summary

Keep PR #39941 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still recurses into the first object-valued anyOf/oneOf variant during config form coercion, so the PR's discriminator-aware object variant selection remains useful unmerged work.

## Best Possible Solution

Keep PR #39941 open. The best path is to land or rework the small discriminator-aware fix in ui/src/ui/controllers/config/form-coerce.ts, preserving primitive union behavior and array fallback behavior, and add anyOf/oneOf regression tests in ui/src/ui/controllers/config/form-utils.node.test.ts, including the unconstrained-variant edge case raised in review.
