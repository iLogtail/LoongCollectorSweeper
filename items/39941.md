## Summary

Keep open. Current `main` still has the reported bug: `coerceFormValues()` recurses into the first object variant in `anyOf`/`oneOf` unions, so discriminated object unions can still coerce nested fields with the wrong schema. The PR is not implemented on `main`, and the bug is directly reproducible from the current logic.
