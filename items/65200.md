## Summary

Current main only clears auto-sourced fallback model/provider/auth overrides on `/new` and `/reset`; it intentionally preserves explicit user selections such as `/model codex`. Because #65200 asks bare `/new` and `/reset` to clear those explicit session overrides too, the requested behavior is still not implemented and should remain open.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If OpenClaw should adopt #65200's behavior, change the shared reset-selection policy so bare `/new`, `/reset`, and `sessions.reset` clear explicit user model/provider/auth overrides, keep `/new <model>` as the opt-in path for a fresh override, and update the auto-reply, Telegram, gateway tests, and session docs accordingly. If maintainers prefer the current sticky explicit-selection policy, close with an explicit product decision rather than automated cleanup.
