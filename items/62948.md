## Summary

Keep open. Current `main` still does not ship the dedicated `openclaw browser doctor` surface from PR #62948. OpenClaw has partial browser diagnostics through the global `openclaw doctor` flow, but the browser CLI, browser tool schema, client, and HTTP routes do not expose a dedicated browser-doctor command/report on `cf858258c71701ef62dddf88aa777873a4962046`. The prior PR discussion also identified attach-only/remote CDP correctness issues that would need resolution before merge.
