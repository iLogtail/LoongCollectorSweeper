## Summary

Keep open. Current main still lacks Mattermost WebSocket protocol ping/pong keepalive; the shipped code listens for open/message/close/error and has an update_at REST health check, but that does not detect a half-dead WebSocket when REST still works and update_at is unchanged. PR #57621 is an open, unmerged fix for this exact issue, while PR #46345 was closed unmerged, so the original bug should remain open until a keepalive implementation lands.

## Best Possible Solution

Keep #44160 open and land PR #57621 or an equivalent Mattermost-owned change that extends the WebSocket abstraction with ping/pong, adds a default pong-timeout heartbeat with cleanup on close/error/abort, preserves reconnect behavior through terminate(), and adds focused monitor-websocket tests for missed-pong termination and healthy pong handling.
