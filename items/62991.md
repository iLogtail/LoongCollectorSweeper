## Summary

Keep open. Current `main` still builds the gateway model catalog from the Pi registry plus provider-plugin augmentation only, with no merge from user-configured `models.providers`, so the PR’s reported gap is not implemented on `main`. The submitted PR also has an unresolved correctness concern around provider-alias deduping and does not add the targeted regression test this path needs.

## Evidence

- **Current catalog path still skips config-declared providers:** `loadModelCatalog()` on `main` loads registry entries, then calls `augmentModelCatalogWithProviderPlugins()`. There is no merge step for `cfg.models?.providers` before sorting/returning the catalog.
