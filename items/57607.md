## Summary

Keep open. Current `main` still drops Mattermost group messages from non-allowlisted senders before pending history is recorded, so the report matches shipped code. The linked follow-up PR `#57609` is still open in the provided timeline, so this is not implemented on `main` yet.

## Evidence

- **Early return before history recording:** In `extensions/mattermost/src/mattermost/monitor.ts`, the `GROUP_POLICY_NOT_ALLOWLISTED` branch logs and immediately returns. That happens before `recordPendingHistory` is defined or called, so non-allowlisted group senders never reach pending-history capture.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:1329](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/mattermost/monitor.ts#L1329)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '1300,1465p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Pending history is only recorded on mention-gate drops:** The same handler records pending history for `onchar-not-triggered` and `missing-mention` drops via `recordPendingHistory()`, but there is no equivalent call on the non-allowlisted sender path.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:1436](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/mattermost/monitor.ts#L1436)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '1300,1465p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Recorded pending history is the source for later group-context injection:** When a group message does trigger a run, `buildPendingHistoryContextFromMap(...)` and `inboundHistory` both read from `channelHistories`. Missing entries on the non-allowlisted path therefore directly remove context from later replies.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:1498](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/mattermost/monitor.ts#L1498)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '1488,1530p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Current docs describe group allowlists as trigger authorization, not a separate history filter:** `docs/channels/mattermost.md` says `groupPolicy = "allowlist"` is mention-gated and `groupAllowFrom` allowlists senders, while `docs/gateway/security/index.md` says group allowlists run before mention activation to restrict who can trigger inside a group session. Nothing in these docs documents a separate 'drop from pending history' behavior for non-allowlisted senders.
  - file: [docs/channels/mattermost.md:177](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/channels/mattermost.md#L177)
