## Summary

Keep #11487 open. Current main still does not accept optional human-readable `name`, `label`, or `description` metadata inside group config entries. Telegram and WhatsApp group-entry schemas remain strict and behavior-only, generated config metadata still emits `additionalProperties: false` for Telegram groups without any metadata field, and representative bundled plugin group schemas have the same omission. The existing @steipete Codex review comment that kept this open still matches current main.

## Best Possible Solution

Keep the issue open. The best implementation is to choose one canonical optional metadata field, add it to group-entry config types and Zod schemas for core and bundled channel plugins, regenerate bundled config metadata, update public group/channel docs, and add focused tests proving the field is accepted and ignored functionally.
