## Summary

Current `main` already routes plugin trust/provenance checks through the activation source config and has a regression test that suppresses the exact false allowlist/provenance warnings described in #63182.

## Evidence

- **Loader resolves trust from source snapshot when given the runtime snapshot:** `resolvePluginActivationSourceConfig` returns the source config snapshot when the loader is called with the current runtime snapshot, which addresses the reported mismatch between runtime config and source allowlist/install records after gateway restart.
  - file: [src/plugins/activation-source-config.ts:7](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/plugins/activation-source-config.ts#L7)
  - command: `nl -ba src/plugins/activation-source-config.ts | sed -n '1,40p'`
  - sha: [a43c1f880757](https://github.com/openclaw/openclaw/commit/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4)
- **Gateway loader now passes activationSourceConfig through to plugin loading:** `loadGatewayPlugins` re-derives auto-enable reasons from `activationSourceConfig`, uses that same source config for startup plugin ID resolution, and passes `activationSourceConfig` into `loadOpenClawPlugins` so trust checks do not rely only on the derived runtime config.
  - file: [src/gateway/server-plugins.ts:451](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/gateway/server-plugins.ts#L451)
  - command: `nl -ba src/gateway/server-plugins.ts | sed -n '436,500p'`
  - sha: [a43c1f880757](https://github.com/openclaw/openclaw/commit/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4)
- **Loader merges trust config from activation source before warning logic:** `resolvePluginLoadCacheContext` computes `activationSourceConfig`, builds an activation source from it, and merges trust config from that source before caching/loading. The warning logic itself only emits `plugins.allow is empty` when the resolved allowlist is actually empty, and skips provenance warnings when the plugin is allowlisted or tracked by provenance.
  - file: [src/plugins/loader.ts:924](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/plugins/loader.ts#L924)
  - command: `nl -ba src/plugins/loader.ts | sed -n '920,965p'; nl -ba src/plugins/loader.ts | sed -n '1626,1825p'`
  - sha: [a43c1f880757](https://github.com/openclaw/openclaw/commit/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4)
- **Install-path provenance still covers global plugin entrypoints under the install directory:** `matchesInstalledPluginRecord` resolves the candidate entrypoint path and considers it tracked when it is equal to or inside `plugins.installs.<id>.installPath` or `sourcePath`, matching the issue's `~/.openclaw/extensions/<id>/index.ts` layout.
  - file: [src/plugins/manifest-registry.ts:475](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/plugins/manifest-registry.ts#L475)
  - command: `nl -ba src/plugins/manifest-registry.ts | sed -n '475,505p'`
  - sha: [a43c1f880757](https://github.com/openclaw/openclaw/commit/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4)
- **Regression test covers the reported warning pattern:** `uses the source runtime snapshot allowlist for plugin trust checks` sets different runtime and source plugin allowlists, then asserts the trusted global plugin loads, the untrusted one is disabled, and neither `plugins.allow is empty` nor `loaded without install/load-path provenance` is warned for the trusted plugin.
  - file: [src/plugins/loader.test.ts:6261](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/plugins/loader.test.ts#L6261)
