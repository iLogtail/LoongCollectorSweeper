## Summary

Current main still has the no-cfg early return that the PR is trying to fix, and the announce-delivery classifier still depends on that no-cfg depth lookup. The PR should not be closed as cleanup; it is a plausible bug fix, though it should be narrowed before landing.

## Best Possible Solution

Keep the PR open, but ask for a narrow rebase or split: retain the `src/agents/subagent-depth.ts` no-cfg default-store fix plus focused `src/agents/subagent-depth.test.ts` and `src/agents/subagent-announce-delivery.test.ts` regressions, drop unrelated drift changes, and address the temp-dir cleanup review.
