## Summary

Keep open. Current main still skips the default on-disk session-store lookup when getSubagentDepthFromSessionStore() is called without cfg, so the PR addresses a real remaining agents/announce bug rather than obsolete work.

## Best Possible Solution

Keep the PR open, but narrow it before merge: remove the no-cfg early return in src/agents/subagent-depth.ts, call resolveStorePath(params.cfg?.session?.store, ...), add focused no-cfg default-store and announce-classification regression tests, clean temporary directories in those tests, and drop unrelated drift from the branch.
