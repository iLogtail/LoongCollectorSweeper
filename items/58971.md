## Summary

Keep #58971 open. Current main still has no `cron.webhookAllowPrivateNetwork` config key, no schema/help/docs entry, and cron webhook POSTs call `fetchWithSsrFGuard` without passing an SSRF policy, so the requested opt-in is not implemented. The related #68055 is the same idea but is closed and unmerged, so it does not track or ship the remaining work.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by a canonical open PR. The best implementation path is a small, explicit `cron.webhookAllowPrivateNetwork` boolean accepted by the config type and strict Zod schema, documented in cron and configuration-reference docs, wired through all cron webhook POST paths as `policy: { allowPrivateNetwork: true }` only when the flag is exactly true, with tests proving both default blocking and opt-in forwarding.
