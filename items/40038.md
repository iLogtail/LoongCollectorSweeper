## Summary

Close PR #40038 as superseded by the newer open PR #63313. The Ollama memory embedding bug is still real on current main, but #40038 is stale against the current plugin file layout and #63313 is the better current implementation tracker because it covers the same `/api/embed` migration plus fallback behavior for the compatibility concern raised in this PR discussion.

## Best Possible Solution

Close this stale old-layout PR as superseded and keep implementation review on open PR #63313. The final fix should update the active Ollama plugin embedding provider under `extensions/ollama/src/embedding-provider.ts`, preserve auth/base URL/SSRF behavior, cover `/api/embed` response validation, and include fallback or explicit compatibility handling for Ollama versions that still respond on `/api/embeddings`.
