## Summary

Keep PR #47687 open. Current main still has only generic Telegram reaction notifications: it handles added standard emoji reactions, enqueues a generic system event, and does not expose `channels.telegram.reactionSemantics`, custom-emoji reaction keys, semantic actions, or wake-trigger behavior. The PR directly addresses the linked open feature request #47677 and the provided review thread shows the author responded to several concrete correctness issues. It needs maintainer/code-review cleanup rather than conservative closure.

## Best Possible Solution

Keep the PR open and ask for a focused rebase/revision: preserve the Telegram-owned reaction semantics feature, custom emoji normalization, schema validation, docs, and tests; remove unrelated CI churn; fix changelog attribution; and route heartbeat wake behavior through a public plugin SDK boundary or add a small additive SDK export if needed. The linked feature request #47677 remains real on current main.
