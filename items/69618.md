## Summary

Keep PR #69618 open. Current main still has only the older per-session usage footer path: no `messages.responseFooter`, no `agents.defaults.responseUsage` schema/type/help surface, no late-bound footer template composition, and `/usage full` only adds the session key rather than model/context-window details. The PR also has unresolved review concerns and broad unrelated changes, so it should not be auto-closed as implemented or superseded, but it also needs maintainer review/cleanup before merge.

## Best Possible Solution

Keep the item open for maintainer review or supersede it with a narrower PR. The best code path is a focused implementation of `messages.responseFooter`, `agents.defaults.responseUsage` inheritance with explicit-off and clear-to-inherit semantics, enriched `/usage full` model/context/cost output, and block-streaming trailing footer delivery, with schema/help/docs/generated artifacts and targeted reply/config/TUI tests. Unrelated dependency, `.gitignore`, and memory-lancedb changes should be split out or dropped.
