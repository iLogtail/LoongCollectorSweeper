## Summary

Keep #40078 open. Current main still lacks a Discord guild/channel opt-in that silently transcribes unmentioned audio in `requireMention` channels and stores the transcript for later history context. Discord can preflight audio for DMs and for spoken mention detection, and it can pass that transcript into the dispatch context when the message proceeds, but the unmentioned guild drop path still records the prebuilt history entry based on `resolveDiscordMessageText`, which remains `<media:audio>` for audio attachments.

## Best Possible Solution

Keep #40078 open. The best implementation is an explicit Discord guild/channel opt-in such as `preflightTranscribe`, added to config types, strict schema, generated metadata, docs, and focused tests. In the Discord preflight path, after authorization but before the no-mention return, transcribe eligible unmentioned audio, replace or supplement the pending history entry with transcript text, suppress agent dispatch, and add a regression proving a later @mention sees transcript text instead of `<media:audio>` placeholders.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications; any implementation should remain explicit opt-in and run only after guild/channel/member access checks.
- The implementation should avoid duplicate transcription when a later mentioned turn processes the same original audio attachment.
