## Summary

Keep #40078 open. Current main still lacks the requested Discord guild/channel `preflightTranscribe` option. Discord can preflight-transcribe DMs and audio-only guild messages for spoken mention detection, but an unmentioned `requireMention` guild audio message still records the original `<media:audio>` history entry and returns without storing the transcript for a later @mention.

## Best Possible Solution

Keep #40078 open. Add an explicit Discord guild/channel option such as `preflightTranscribe` across config types, strict schema, generated metadata, docs, and focused tests. In Discord preflight, after authorization and before the no-mention return, transcribe eligible unmentioned audio when enabled, store transcript text in pending guild history instead of `<media:audio>`, suppress agent dispatch, and prove a later @mention receives the transcript in history context.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications, so any implementation should remain explicit opt-in and run only after guild/channel/member access checks.
- The implementation should avoid a second STT call if the same audio is later processed during a mentioned turn.
