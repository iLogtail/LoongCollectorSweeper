## Summary

Keep #40078 open. Current main still lacks the requested opt-in silent Discord preflight transcription/history ingestion for unmentioned audio in requireMention guild channels. The shipped/current Discord path only uses preflight transcription for DMs and spoken mention detection; when a requireMention guild audio message is not considered mentioned, OpenClaw records the prebuilt <media:audio> history entry and returns without dispatching or storing the transcript.

## Best Possible Solution

Keep #40078 open. The best implementation path is an explicit Discord guild/channel opt-in such as preflightTranscribe, added to config types, strict schema, generated metadata, docs, and tests. In the Discord preflight path, after authorization but before mention-gate drop, transcribe matching unmentioned audio, record transcript-bearing pending history without dispatching an agent reply, and add a regression proving a later @mention sees transcript text rather than <media:audio> placeholders.
