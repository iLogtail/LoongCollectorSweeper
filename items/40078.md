## Summary

Keep #40078 open. Current main has Discord audio preflight for direct-message audio and spoken mention detection, but it still does not implement the requested opt-in silent transcription/history ingestion for unmentioned audio in requireMention guild channels.

## Best Possible Solution

Keep #40078 open. The best product/code path is an explicit Discord guild/channel opt-in, such as preflightTranscribe, wired through config schema/types/docs and the Discord inbound path after authorization but before mention drop. For matching unmentioned audio, run the existing tools.media.audio transcription pipeline, store transcript-bearing pending history without dispatching an agent reply, and add regression tests proving a later @mention receives transcript text rather than <media:audio> placeholders.
