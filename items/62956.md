## Summary

Keep open. Current `main` still only pushes a bare `error` status into the TUI status bar and clears elapsed timing when the status leaves the busy-state path, so the UX gap described by PR #62956 is not implemented on `main`. The PR also accumulated broader stale-run/gap-recovery changes with unresolved review concerns, so it needs maintainer judgment rather than auto-close cleanup.
