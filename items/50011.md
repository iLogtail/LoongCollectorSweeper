## Summary

Keep #50011 open. Current main still accepts arbitrary string entries for `gateway.nodes.denyCommands` at config-write/schema level, while the unknown-command and suggestion logic remains in `security audit`. The linked fix PRs are still open and unmerged, so this is not shipped or safely superseded yet.

## Best Possible Solution

Keep the issue open and land a focused config-write warning for direct `gateway.nodes.denyCommands` writes and ancestor writes such as `gateway.nodes={...}`. Reuse or share the audit known-command/suggestion logic, warn rather than hard-fail so custom/plugin commands remain possible, and add regression coverage in `src/cli/config-cli.test.ts` or a dedicated config validator test. PR #50076 is the closest existing implementation to evaluate, but it needs current-main refresh and maintainer review before this issue can close.
