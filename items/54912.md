## Summary

Keep open. Current main adds generic `memory_search` unavailable warnings/actions, but it still does not surface the active `auth-profiles.json` path/profile or give leaked-key-specific recovery guidance in `memory status` or the memory failure path, which is the core maintainer concern in #54912.

## Evidence

- **`memory_search` only has generic provider-error guidance:** `buildMemorySearchUnavailableResult` special-cases quota/429 only. A Gemini leaked-key 403 would fall through to the generic warning/action (`embedding/provider error` / `Check embedding provider configuration and retry memory_search`) rather than a stored-key rotation hint.

It does not print the auth-store path/profile or a fix command for stale stored credentials.
