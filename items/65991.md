## Summary

Current `main` does not implement `silent` for cron `failureAlert`. The cron config/types, gateway/tool schemas, runtime alert resolution, and docs all omit that field, while Telegram outbound already supports `silent`. This is a coherent, recent feature request, so conservative cleanup should leave it open.
