## Summary

Keep open. Current main still uses an exact, case-sensitive gateway catalog lookup for image support and has no generic fallback to the configured `models.providers.<id>.models[].input` entry described by the issue. The linked PR #65178 is relevant and directly attempts this fix, but it is open/unmerged in the provided GitHub context, so the bug is not implemented on main.

## Best Possible Solution

Keep #65165 open until a fix lands. The likely maintainer path is to land a corrected version of #65178 or a broader fix shared with the custom-provider image-support reports: make `resolveGatewayModelSupportsImages` use normalized provider/model matching and consult configured provider model metadata when the catalog misses, then add regression tests for case-insensitive IDs, provider aliases, slash-containing model IDs, and explicit `input: ["text", "image"]` custom providers.
