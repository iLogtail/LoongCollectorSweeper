## Summary

Keep #52732 open. Current main still exposes `compaction` and `contextPruning` only under `agents.defaults`; `agents.list[]` has neither type nor strict-schema fields for these keys, resolved per-agent config does not carry them, and Pi compaction/pruning runtime paths still read global defaults only. The related #52751 implementation PR is useful prior art but remains unmerged, so this issue is not cleanup-closeable.

## Best Possible Solution

Keep the issue open until a core implementation lands. The best fix should add optional `agents.list[].compaction` and `agents.list[].contextPruning` to the config type and strict Zod schema, reuse or extract the defaults schemas, define deep-merge precedence over `agents.defaults`, thread effective per-agent values through compaction reserve settings, safeguard runtime, context-pruning setup, manual `/compact`, memory-flush, timeout/model/provider paths, then update generated schema/help/docs and focused schema/resolver/runtime tests. Treat #52751 and #13736 as related prior art unless maintainers explicitly choose a canonical tracker.
