## Summary

Keep PR #43911 open. Current main still does not ship a Russian Control UI locale: `ru-RU` is absent from the locale type, lazy registry, browser-locale resolver, generated locale files, `.i18n` metadata, and registry tests. The PR addresses a real missing capability, and the provided review discussion shows the author handled the earlier missing-key feedback, but the branch should be rebased and cleaned up against the expanded generated-locale pipeline before merge.

## Best Possible Solution

Keep the PR open, but ask for a fresh rebase onto current main and a clean generated-locale implementation: add `ru-RU` to the current `Locale` union, lazy registry, navigator resolution, generator `LOCALE_ENTRIES`, English language labels, generated Russian locale bundle and `.i18n` metadata, plus registry/translation tests. Drop unrelated branch drift before review.
