## Summary

Current `main` does not implement a `ru-RU` Control UI locale, so this PR should stay open. The locale type, lazy-locale registry, navigator fallback logic, shipped locale files, and locale tests all exclude Russian on `main`. The PR is also stale relative to current UI i18n workflow, because non-English locale bundles are now generated output and `ru-RU` would need matching generator and `.i18n` metadata updates before landing cleanly.
