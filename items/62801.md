## Summary

Keep #62801 open. Current main does not expose the PR's requested `runtime.subagent.spawnDetached(...)` plugin API. Main has an internal native subagent spawner used by `sessions_spawn`, and plugins can run an already-known subagent session through `api.runtime.subagent.run(...)`, but there is still no public plugin runtime method that creates a fresh detached child session and returns `runId` plus `childSessionKey`. The PR discussion also shows review feedback was addressed on the branch, so this is not a low-signal cleanup close; it needs maintainer product/API review and rebase against current main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased, narrow implementation that exposes a small optional plugin runtime detached-spawn API, routes it through the existing native `spawnSubagentDirect` lifecycle, preserves plugin id/request scope and model override policy, documents it on the public SDK runtime page, and adds gateway/runtime/registry regression tests for late binding and authorization.
