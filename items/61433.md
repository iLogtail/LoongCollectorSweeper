## Summary

Keep open. Current main has partial mitigations for explicit internal-runtime-context wrappers, including display sanitization and a newer ACP-specific plain-prompt path, but it still does not provide the hard fail-closed boundary requested here. Subagent completion fallback can still send the raw task result verbatim when the announce-agent call fails or returns no visible payload, and isolated cron direct delivery still forwards selected reply payload text without a general rewrite/sanitization layer for operational text such as file paths or routing labels. Related #62306 and #53145 cover narrower remaining surfaces, but neither fully supersedes this broader cron/subagent/announce boundary issue.

## Best Possible Solution

Keep #61433 open as the broad hardening tracker. The best fix is a shared fail-closed completion/announce delivery boundary: avoid raw `event.result` external fallback, require a clean synthesized user-facing payload or a narrowly sanitized fallback, apply the same rule to isolated cron and subagent completion paths, and add regression tests proving internal wrappers, session ids, stats, file paths, routing labels such as `Message for Telegram`, and operational summaries do not reach user-visible channel delivery or persisted visible chat.
