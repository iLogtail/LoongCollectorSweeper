## Summary

Current `main` already has explicit Anthropic thinking/replay protections and silent-by-default compaction notices, with regression coverage for preserving latest assistant thinking blocks and avoiding mutation of provider-owned signed thinking content.

## Evidence

- **Latest assistant thinking is preserved verbatim:** `dropThinkingBlocks()` now strips `thinking`/`redacted_thinking` from non-latest assistant messages only, and explicitly preserves the latest assistant turn unchanged.
  - file: [src/agents/pi-embedded-runner/thinking.ts:59](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/pi-embedded-runner/thinking.ts#L59)
- **Provider-owned signed thinking is not mutated during repair:** When signed Anthropic thinking shares a turn with tool calls, transcript repair preserves the turn only if it is already replay-safe; otherwise it drops the whole assistant turn instead of normalizing sibling blocks and corrupting provider-owned content.
  - file: [src/agents/session-transcript-repair.ts:285](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/session-transcript-repair.ts#L285)
- **Anthropic replay policy distinguishes models that must preserve thinking:** Replay policy keeps thinking blocks for Claude 4.5+/4.6-era models and only enables `dropThinkingBlocks` for older Claude families, matching Anthropic's newer replay rules.
  - file: [src/plugins/provider-replay-helpers.ts:73](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/plugins/provider-replay-helpers.ts#L73)
- **Regression coverage for latest assistant thinking + immutable replay:** Tests assert that Anthropic replay preserves the latest assistant thinking block and that immutable signed-thinking turns are handled without mutating provider-owned content.
  - file: [src/agents/pi-embedded-runner.sanitize-session-history.test.ts:965](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/pi-embedded-runner.sanitize-session-history.test.ts#L965)
- **Compaction notices are silent by default:** Compaction notices are only sent through the optional notice path, and the execution test verifies that a compaction start event does not call `onBlockReply` by default.
  - file: [src/auto-reply/reply/agent-runner-execution.test.ts:1000](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/auto-reply/reply/agent-runner-execution.test.ts#L1000)
- **Shipped changelog entries for the Anthropic compaction/replay fixes:** The changelog records shipped fixes for dropping replayed assistant thinking blocks and for preserving latest assistant thinking/redacted-thinking ordering during sanitization.
  - file: [CHANGELOG.md:2172](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/CHANGELOG.md#L2172)

## Close Comment

Closing from a Codex review because current `main` already implements this behavior. The Anthropic replay path now preserves the latest assistant `thinking`/`redacted_thinking` blocks instead of mutating them (`src/agents/pi-embedded-runner/thinking.ts:59`), transcript repair avoids rewriting signed provider-owned thinking content (`src/agents/session-transcript-repair.ts:285`), and regression tests cover preserved latest-thinking replay for Anthropic sessions (`src/agents/pi-embedded-runner.sanitize-session-history.test.ts:965`). The secondary chat-noise part is also addressed in current behavior: compaction notices are silent by default unless `agents.defaults.compaction.notifyUser` is enabled (`src/auto-reply/reply/agent-runner-execution.test.ts:1000`). The shipped changelog also records the relevant Anthropic replay/ordering fixes (`CHANGELOG.md:2172`, `CHANGELOG.md:2660`).
