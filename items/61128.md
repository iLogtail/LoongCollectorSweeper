## Summary

Keep #61128 open. The issue’s linked implementation PR #59547 was closed unmerged, and current main still performs direct skill discovery by scanning directories and reading/parsing SKILL.md on each load path. Existing session snapshot reuse and watcher invalidation reduce repeated prompt/session work, but they are not the requested safe, metadata-validated, bounded in-memory loader cache.

## Best Possible Solution

Keep the issue open. The best path is to revive or rework the #59547 idea into a narrow cache in the skill loader/workspace loading path: key by source plus resolved skill directory, validate hits with SKILL.md metadata, recheck root containment and anti-symlink constraints before reuse, prune entries under scanned roots, bound process-wide growth with TTL/LRU, and add regression tests for cache hit, edit/delete invalidation, symlink escape, oversized files, root pruning, and watcher/session snapshot interactions.
