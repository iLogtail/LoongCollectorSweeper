## Summary

Current `main` still matches the bug report: the gateway exec path always sends `system.run.prepare` for node shell execution, and the headless node host supports/advertises that command, but the macOS companion app code still neither defines, advertises, nor dispatches `system.run.prepare`. The same omission is present in the latest release tag `v2026.4.23`, so there is not strong evidence to close this issue.

## Evidence
