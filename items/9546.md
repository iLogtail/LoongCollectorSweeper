## Summary

Current main still lacks the requested WebChat new-messages banner preference. The UI still derives the `New messages` control directly from scroll state, the render path displays it whenever `showNewMessages` is true, persisted UI settings do not include a banner/notification/autoscroll preference, and the strict `gateway.webchat` config contract still only exposes `chatHistoryMaxChars`. The existing issue comment that kept #9546 open remains accurate against `ee2ab9a6449219e1c425308a4832ddbb7eeb0ca2`.

## Best Possible Solution

Keep #9546 open until maintainers choose the product surface. A narrow implementation should add either a persisted browser-local Control UI/WebChat preference or a documented `gateway.webchat` config option, gate `chatNewMessagesBelow`/`showNewMessages` behavior through it, and update focused UI tests plus the public WebChat/config documentation.
