## Summary

Current main still lacks the requested WebChat option. The existing April 26 issue comment keeping #9546 open remains accurate: the New messages control is still driven directly by scroll state, rendered without a settings/config guard, and neither persisted UI settings nor `gateway.webchat` config expose a banner/autoscroll/disabled mode.

## Best Possible Solution

Keep #9546 open. A narrow implementation should add a persisted WebChat/Control UI preference or documented `gateway.webchat` option for the new-message indicator mode, gate both scroll-state behavior and chat rendering through that setting, and update focused UI tests plus the public WebChat/config docs.
