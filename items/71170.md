## Summary

Keep open. Current main still lacks `agents.defaults.reminderGuard` or any per-agent opt-out, and the runtime still appends the unscheduled-reminder note based only on reminder-like reply text, `successfulCronAdds === 0`, and absence of an existing session cron job. The PR remains a focused core fix for #70666 rather than an obsolete or ClawHub-only item.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, fix the changelog attribution, and either land the minimal `agents.defaults.reminderGuard` plus per-agent override or explicitly supersede it with a maintainer-approved SDK scheduling-signal design.
