## Summary

Keep PR #71170 open. Current main still has the false-positive reminder guard behavior and does not implement the proposed `agents.defaults.reminderGuard` or per-agent `reminderGuard` opt-out. The linked issue #70666 remains a real product bug in core auto-reply behavior, not a cleanup-close candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/update #71170 against current main, preserve the default-on behavior, keep the global and per-agent config schema/docs/tests aligned, and decide separately whether #70666 also needs a future plugin SDK scheduling signal.
