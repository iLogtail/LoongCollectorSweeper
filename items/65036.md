## Summary

Keep PR #65036 open for maintainer review. Current main has not implemented the proposed line change, and the PR is a focused bug-fix attempt with active discussion rather than an obsolete or duplicate cleanup item. However, the provided review thread and current code show a real design question: when `config` is omitted, `loadConfig()` can return the resolved runtime snapshot, while current docs/tests emphasize source-authoritative marker persistence for generated `models.json`. That needs maintainer judgment or a safer revision, not automated closure.

## Best Possible Solution

Keep the PR open and require maintainer review or revision. The best path is to preserve source-authoritative `models.json` secret marker behavior, add a regression test that distinguishes source and resolved runtime snapshots for the omitted-config branch, and decide whether the remaining bloat belongs in this PR or in the broader plugin-discovery behavior described in #65020.
