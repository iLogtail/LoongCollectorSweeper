## Summary

Keep open. Current main still recognizes `max` as a possible thinking level, but the default thinking profile excludes it, explicit `/think max` directives are rejected when the provider does not advertise `max`, and the bundled Ollama plugin does not currently provide an Ollama-specific thinking profile. The existing Ollama stream path only maps non-`off` thinking to top-level `think: true`, so the reported request remains actionable rather than implemented or duplicate/superseded.

## Best Possible Solution

Keep #71584 open. The likely fix belongs in the Ollama provider/plugin layer, using the existing provider thinking profile API to advertise `max` for the relevant Ollama models or model capabilities, then adding regression coverage for `/think max` validation and the Ollama payload mapping. Avoid adding a broad core default that exposes `max` for providers that have not declared it.
