## Summary

Keep #71584 open. Current main still parses `max` as a canonical thinking level, but the default provider profile excludes it, explicit `/think max` is rejected unless the selected provider advertises `max`, and the bundled Ollama provider does not currently advertise an Ollama-specific `max` profile. The Ollama runtime forwards native thinking as top-level boolean `think: true/false`, not an Ollama `reasoning_effort: "max"` control, so the reported bug remains actionable.

## Best Possible Solution

Keep #71584 open. The likely fix belongs in the bundled Ollama provider/plugin layer: use the existing provider thinking profile API to advertise `max` only for Ollama models/capabilities that support it, then add regression coverage for `/think max` validation and the Ollama payload mapping. Avoid adding `max` to the global base profile, because current docs/tests deliberately reserve it for provider-advertised support.
