## Summary

Keep open. Current `main` does not implement the requested path-scoped RWX permission model for exec and file tools. Exec approvals are still binary-path allowlist based, and filesystem tool policy is still a single `workspaceOnly` boolean rather than a path-keyed rule map.

## Evidence

- **Exec approvals remain binary-path allowlists:** The user-facing docs still define exec allowlist enforcement in terms of resolved binary paths only, not target-path permissions.
  - file: [docs/tools/exec.md:178](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/docs/tools/exec.md#L178)
  - command: `nl -ba docs/tools/exec.md | sed -n '170,190p'`

- **Exec approvals schema has allowlist entries, no path-RWX map:** `ExecAllowlistEntry` only stores allowlist pattern metadata, and `ExecApprovalsAgent` only exposes `allowlist`; there is no `filesystem.rules` / `deny` / `default` structure like the issue proposes.
  - file: [src/infra/exec-approvals.types.ts:1](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/infra/exec-approvals.types.ts#L1)
