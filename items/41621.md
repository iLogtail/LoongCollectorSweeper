## Summary

Keep PR #41621 open. Current main still uses one `skillsFilter` for both the Agents -> Skills panel and the top-level Skills tab, so the focused separate-state fix is not implemented or clearly superseded. The related autofill/click mitigation does not remove the shared-state behavior this PR targets.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and land the same narrow product fix: add a dedicated `agentSkillsFilter` state field, include it in `AppViewState`, route only the Agents -> Skills panel through that field, and add a focused regression test proving the Agents panel filter no longer changes the top-level Skills tab filter.
