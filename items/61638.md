## Summary

Current main fixes the same observable heartbeat overflow failure through merged PR #71478: oversized heartbeat scheduler delays are clamped through the shared safe timeout helper, logged once, and covered by regression tests. This supersedes PR #61638's config-schema rejection approach, which is no longer needed to stop the 1 ms heartbeat loop.

## Evidence

## Risks / Open Questions

- Current main clamps and warns instead of rejecting oversized heartbeat.every at config load time. That solves the resource/log-flood/user-message failure, but maintainers could still choose stricter config UX later as a separate product decision.
- Repo tests were not executed during this read-only review; pnpm was unavailable in the checkout. Confidence comes from inspected current code, committed regression tests, release notes, PR #71478 merge metadata, and a local Node timer smoke check.
