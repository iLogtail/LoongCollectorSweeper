## Summary

Keep open. Current `main` added documentation and isolated-agent auth fallback that reduce the original confusion, but the core CLI/setup gaps from the report are still present: `models auth paste-token` is still interactive-only, and the missing-auth error still points isolated users at `openclaw agents add <id>`/copying `auth-profiles.json` rather than a concrete custom-provider auth path. The cross-referenced PR #63050 also suggests the issue is still active work, not stale cleanup.
