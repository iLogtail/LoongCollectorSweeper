## Summary

Keep open. Current `main` does not include this dedup behavior, and the PR as written still does not address the reported two-pass gateway startup path because that path rebuilds a fresh plugin registry under a different cache key. It also misses factory-registered Feishu tools that do not pass explicit `name`/`names` options to `registerTool()`.

## Evidence

- **Current main still registers tools without same-plugin dedup:** `registerTool()` on `main` still just normalizes names, appends them to `record.toolNames`, and pushes a new `registry.tools` entry. There is no guard for duplicate registrations by the same plugin.
  - file: [src/plugins/registry.ts:371](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/plugins/registry.ts#L371)
  - command: `nl -ba src/plugins/registry.ts | sed -n '371,398p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Registry cache keys diverge across startup modes:** The plugin load cache key includes `preferSetupRuntimeForChannelPlugins`, so a startup load with that flag and a later reload without it produce different cache keys and therefore different registry instances.
  - file: [src/plugins/loader.ts:842](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/plugins/loader.ts#L842)
  - command: `nl -ba src/plugins/loader.ts | sed -n '840,855p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Gateway startup enables the setup-runtime pass when any deferred channel plugin exists:** Initial gateway bootstrap passes `preferSetupRuntimeForChannelPlugins: deferredConfiguredChannelPluginIds.length > 0`, which is exactly the mode split the PR description relies on.
  - file: [src/gateway/server-startup-plugins.ts:92](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-startup-plugins.ts#L92)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '91,102p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Deferred reload reuses the startup plugin set in a later pass:** After listen, the server reloads deferred gateway plugins with `pluginIds: startupPluginIds`, but without the earlier `preferSetupRuntimeForChannelPlugins` flag, so the same startup plugin set can execute registration again in a fresh registry.
  - file: [src/gateway/server.impl.ts:774](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server.impl.ts#L774)
