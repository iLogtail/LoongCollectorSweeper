## Summary

Keep open. I did not find strong evidence that current `main` already fixes this Control UI/webchat formatting bug, and the report is recent enough that `stale_insufficient_info` does not apply. The current code still streams assistant text directly into the live chat bubble and then swaps in the final message, but there is no targeted regression coverage for the reported case where code indentation is wrong until a manual refresh.
