## Summary

Current main has retry support for retryable embedding 5xx failures, but the core reported problem is still not fully fixed: normal `openclaw memory status` can still report `Dirty: no` from existing index metadata without reconciling current source-file coverage against indexed rows. Related #26772 remains adjacent checkpoint/resume work, not a canonical duplicate for this transient embedding/status gap.

## Best Possible Solution

Keep #70567 open. Fix memory-core so failed or incomplete indexing is persisted or recomputed from source coverage: compare scanned memory/session source files against indexed `files` rows, surface missing-file counts clearly in `memory status`, mark status dirty when coverage is incomplete, and add regression coverage for an embedding failure after some files have already been indexed. Keep #26772 separate for durable checkpoint/resume of long full-reindex operations.
