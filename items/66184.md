## Summary

Keep open. Current `main` does not contain this conversational reply-shaping change, and the PR discussion surfaced two still-valid correctness gaps: computing the conversational flag before inline body rewrites, and applying the hint only to the immediate prompt while queued follow-up prompts use a different body.

## Evidence

- **Current main does not include the proposed feature:** A repo search on `fe930b987e33f4e1acd93b5ef1ea76c8b7260047` found no `conversationalFreeform` plumbing, no `[Reply shaping hint]`, and none of the added anti-flattening system-prompt strings. The current prompt sections and reply prelude still lack this behavior.
  - file: [src/agents/system-prompt.ts:299](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/agents/system-prompt.ts#L299)
  - command: `rg -n "conversationalFreeform|Reply shaping hint|anti-flattening|confirmation-heavy|Preserve persona signals|prefer acting over asking for reassurance|explanation-first framing" src/agents src/auto-reply`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Inline actions can rewrite the body after directive resolution:** `resolveReplyDirectives()` finalizes `cleanedBody` early, but `getReplyFromConfig()` calls `handleInlineActions()` afterward. Inline actions can rewrite `ctx.Body`, `sessionCtx.Body`, `sessionCtx.BodyForAgent`, and `sessionCtx.BodyStripped` from skill `promptTemplate` expansion or inline command extraction, so a conversational/freeform classification computed before that point can become stale.
  - file: [src/auto-reply/reply/get-reply-inline-actions.ts:310](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/auto-reply/reply/get-reply-inline-actions.ts#L310)

  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Queued follow-up prompts use a different body than the immediate prompt:** `buildReplyPromptBodies()` returns both `prefixedCommandBody` and `queuedBody`, and active-run follow-up/steer paths later set `followupRun.prompt` from `queuedBody`. The PR patch shown in context only injected the new hint into `prefixedBody`, so queued conversational turns would miss the shaping hint entirely.
