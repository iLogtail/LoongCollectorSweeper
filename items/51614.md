## Summary

Keep open. Current main exposes a global `gateway.channelStaleEventThresholdMinutes`, but it still does not expose the requested `channels.whatsapp.healthMonitor.staleEventThresholdMs` or any per-channel/per-account stale threshold. The shared channel health-monitor schema/type remain `enabled` only, WhatsApp reuses that strict schema, and the WhatsApp Web watchdog still falls back to an internal 30-minute `tuning.messageTimeoutMs` default with no config wiring.
