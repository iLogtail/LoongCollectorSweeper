## Summary

Current `main` now preserves provider-qualified custom-provider model refs in the Control UI model-selection path, including nested slash-containing model ids, and adds focused tests for the exact failure class reported here. The fix landed in commit `bc0f54bd04469d79095283c3b7ab55a3118fc3ae` and is contained in the current `main` SHA.

## Evidence

- **Control UI now normalizes and preserves qualified model refs:** `resolvePreferredServerChatModelValue(...)` keeps provider-qualified refs, corrects stale providers from the catalog, and avoids bogus double-prefixing for slash-containing model ids.
  - file: [ui/src/ui/chat-model-ref.ts:106](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/ui/src/ui/chat-model-ref.ts#L106)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Slash-command model switch mirrors the resolved qualified ref back into UI session state:** The `/model` Control UI path resolves the patched model/provider and stores a qualified `modelOverride`, instead of leaving a stripped model id in local state.
  - file: [ui/src/ui/chat/slash-command-executor.ts:212](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/ui/src/ui/chat/slash-command-executor.ts#L212)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Regression coverage for nested custom-provider refs:** Tests now cover nested ids such as `nvidia/moonshotai/kimi-k2.5`, asserting that picker state keeps the provider-qualified value and label instead of degrading to an allowlist-breaking raw id.
  - file: [ui/src/ui/chat-model-select-state.test.ts:118](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/ui/src/ui/chat-model-select-state.test.ts#L118)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Post-patch Control UI test covers provider-qualified nested ids after model changes:** The new node test asserts that after `sessions.patch` returns `moonshotai/kimi-k2.5` with provider `nvidia`, the Control UI keeps `nvidia/moonshotai/kimi-k2.5` as the session override.
  - file: [ui/src/ui/chat/slash-command-executor.node.test.ts:418](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/ui/src/ui/chat/slash-command-executor.node.test.ts#L418)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
