## Summary

Keep #15022 open. Current main still has no opt-in final/tool-interleaved text coalescing config, final payload assembly preserves separate assistant text entries, and block streaming still flushes before tool execution instead of holding text across tool calls. The linked PR #16275 was a plausible fix but is closed unmerged in the provided GitHub context and its hold/resume API is absent from current main.

## Best Possible Solution

Keep #15022 open as the focused core reply-pipeline issue. The likely fix should add an opt-in coalescing/final-only mode that covers final embedded-run assistantTexts and block-streamed text around tool execution, with a hold/defer mechanism for sequential and parallel tool calls, config schema/docs, and regression tests for text -> tool -> text -> tool -> text turns while preserving media, reasoning, duplicate-suppression, and assistant-boundary behavior where required.
