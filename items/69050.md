## Summary

Keep #69050 open. Current main still does not include `ui:build` in `pnpm build`/`BUILD_ALL_STEPS` or the `ciArtifacts` profile, and the current build-all test locks in that omission. Main has important adjacent mitigations: gateway startup can auto-build missing Control UI assets, and prepack/release paths run or require `pnpm ui:build` separately. That makes this a build-contract decision for maintainers, not a conservative ClawSweeper close.

## Best Possible Solution

Keep the PR open for maintainer build-contract judgment. If `pnpm build` should produce a complete local distribution, add `ui:build` to `BUILD_ALL_STEPS` and `ciArtifacts`, update `test/scripts/build-all.test.ts`, and review docs/prepack/cross-OS release comments so the contract is consistent. If the intended contract is to keep UI bundling separate, close this PR manually with that rationale and point to the existing startup auto-build plus `pnpm build && pnpm ui:build` release/prepack flow.
