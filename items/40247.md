## Summary

Keep open. Current `main` does not implement the PR's dual-bind behavior: the gateway still binds only the resolved specific host for tailnet/custom modes, while internal/local gateway clients and tooling still default to loopback. The PR is addressing a live code/docs gap, so conservative cleanup should not auto-close it.
