## Summary

Keep open. Current main still parses `--resources` case-sensitively: `parse_resources` strips entries but does not lowercase them before comparing against `ALLOWED_RESOURCES`, so mixed-case values like `Scripts` would still be treated as invalid. The related PRs in the supplied GitHub context are closed and unmerged, so they do not supersede the issue.
