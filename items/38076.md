## Summary

Keep #38076 open. Current main still parses `init_skill --resources` case-sensitively: tokens are stripped but not lowercased before validation against a lowercase-only allowlist, and the CLI passes `--resources` directly through that parser. The related fix PRs in the provided GitHub context are closed and unmerged, and current tests do not include `test_init_skill.py` coverage for mixed-case resources.

## Best Possible Solution

Keep the issue open and implement the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: lowercase each nonempty `--resources` token before validation and order-preserving deduplication, while preserving the existing unknown-resource error path. Add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case accepted values, lowercase output/deduplication, and unknown-resource rejection.
