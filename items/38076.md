## Summary

Keep open. Current `main` still parses `--resources` case-sensitively in `skills/skill-creator/scripts/init_skill.py`, and a direct read-only reproduction on the checked-out code rejects mixed-case values like `Scripts,References,ASSETS` with `SystemExit 1`.

## Evidence

- **Current parser is still case-sensitive:** `parse_resources()` splits and trims values, then validates each token against `ALLOWED_RESOURCES` without lowercasing first, so mixed-case inputs remain invalid.
