## Summary

Keep #70438 open. Current main still has image-only WebChat attachment handling and no `gateway.webchat.attachments` config for PDF offload, text inlining, or inline text caps. Related #30759 and #60226 track broader document-upload/context gaps, and open PR #70947 may address generic non-image persistence, but it explicitly has no config/env changes and does not implement this configurable PDF/text policy request.

## Best Possible Solution

Keep #70438 open unless maintainers deliberately fold it into a canonical WebChat attachment-policy issue. The best product path is a typed `gateway.webchat.attachments` config with documented defaults, Control UI accept/filter behavior driven by that policy, Gateway PDF handling that produces `media://inbound/<id>` refs, capped UTF-8 inline handling for text/JSON/XML/YAML/CSV/code MIME types, unsupported-file feedback, and focused UI/Gateway/config/docs tests coordinated with #30759, #60226, and PR #70947.
