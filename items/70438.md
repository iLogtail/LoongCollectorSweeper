## Summary

Keep #70438 open. Current main still treats WebChat attachments as image-only in the Control UI and Gateway attachment parser, and there is no `gateway.webchat.attachments` or equivalent config/docs surface for PDF offload, inline UTF-8 text handling, or per-tenant policy. The related open issues #30759 and #60226 cover broader document upload/injection problems, but they do not fully supersede this narrower configurable handling request.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally folds it into a canonical WebChat document-upload tracker. The likely product path is a typed `gateway.webchat.attachments` policy in config schema/labels/docs, Control UI accept/filter/preview changes for allowed document/text files, Gateway parser support for PDF media refs and capped UTF-8 text inlining, and focused tests covering current image behavior plus PDF/text/drop cases.
