## Summary

Keep open. Current `main` still matches the report for non-isolated heartbeat runs: session-usage persistence writes the just-used `model`/`modelProvider` into the session entry, `resolveSessionModelRef` still prefers those persisted runtime fields for UI/session summaries, and heartbeat still runs on the main session by default unless `isolatedSession` is explicitly enabled.
