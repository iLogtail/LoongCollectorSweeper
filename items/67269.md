## Summary

Keep #67269 open. Current main still does not implement arbitrary Feishu event handlers, and this cannot be safely closed as a ClawHub-only item because existing plugin hooks observe OpenClaw message flow, not arbitrary Feishu/Lark events on the bundled Feishu channel's existing WebSocket. The PR is recent and addresses a concrete integration gap, but it still needs maintainer product/API review and likely revisions before merge.

## Best Possible Solution

Do not auto-close. The best path is maintainer/API review for a typed, documented Feishu raw-event extension seam that lets external plugins or trusted operator config share the bundled Feishu connection safely. AgentLegion-specific Bitable behavior should live outside core, likely as a ClawHub plugin, once the missing shared-connection event seam exists. If maintainers reject config-loaded handlers, supersede this PR with a narrower plugin-SDK/channel-runtime design issue rather than closing it as stale cleanup.
