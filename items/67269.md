## Summary

Keep PR #67269 open. Current main does not implement `channels.feishu.customEventHandlers`: the Feishu monitor registers a fixed built-in event map, and the strict Feishu config schema has no `customEventHandlers` field. Existing plugin hooks cover OpenClaw message/tool/agent lifecycle, not arbitrary raw Feishu/Lark event subscription on the existing Feishu dispatcher/WebSocket, so this is not a high-confidence ClawHub-only close. The PR also has unresolved design/quality questions from review, including schema/docs coverage, path normalization, fire-and-forget behavior, and regression concerns.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is either a documented, typed Feishu/channel raw-event subscription seam that third-party plugins can use, or a Feishu-plugin implementation that includes schema metadata, docs, tests, path normalization, built-in event collision handling, fire-and-forget semantics, and regression coverage for existing Feishu handlers.
