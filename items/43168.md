## Summary

Keep open. Current main still routes non-isolated heartbeat runs through the resolved session key, and the reply runner persists the submitted heartbeat prompt as a user transcript entry. Optional `isolatedSession: true` mitigates this, but it is documented as default false. The direct fix PR #43189 remains open, so the requested behavior is not implemented on main.
