## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` directly to the OpenAI embeddings request, and the existing regression test explicitly asserts that old upstream-forwarding behavior. The PR is a focused memory-lancedb compatibility fix with matching regression coverage in the provided diff and a follow-up commit addressing the bot review about non-positive dimensions. The nearby memory-lancedb reports are related provider-compatibility work, but they do not supersede this exact vLLM/non-Matryoshka `dimensions` forwarding issue.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land a scoped fix in `extensions/memory-lancedb/index.ts` that omits `dimensions` from the upstream OpenAI-compatible request, truncates locally to the configured vector dimension, fails clearly when the returned vector is too short or the configured dimension is invalid, and replaces the existing forwarding assertion with the PR's focused regression tests. If maintainers need provider-specific behavior instead, that should be captured as an explicit memory-lancedb contract rather than closing this PR as obsolete.
