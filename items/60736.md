## Summary

Current main has partial `google-vertex` recognition, but it does not implement the issue's requested Vertex AI provider behavior. `google-vertex` is still a hook-only alias of the Google AI Studio provider, onboarding has no `google-vertex` choice or location prompt, and the Google native transport still builds Gemini API-style `/models/{model}:streamGenerateContent` URLs instead of Vertex project/location/publisher paths. The linked PRs #60860 and #62103 are active implementation attempts and should remain the path for this work, not a reason to close the issue yet.

## Best Possible Solution

Keep #60736 open and resolve it by landing or consolidating one of the active Google plugin PRs. The implementation should register or otherwise activate a real `google-vertex` path, resolve project and location from config/env/ADC with the requested fallbacks, construct Vertex AI project/location/publisher model URLs, use OAuth/ADC bearer auth safely, add onboarding/docs for location setup, and add focused transport/auth tests proving Vertex URLs and headers.
