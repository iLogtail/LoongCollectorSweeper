## Summary

Keep open. Current main does not implement `tool_result_before_model` as a typed plugin hook: the hook name, hook runner, docs, and tests do not expose it. Main does have pre-model tool-result rewriting through agent tool-result middleware, but that seam is explicitly bundled-plugin-only, so it does not solve the public Plugin SDK/API gap this PR is trying to address. The PR discussion and bot review context show a focused implementation and test plan rather than a low-signal drive-by; related #34144 is closed, and #62123 is a broader async request rather than a clear superseding PR.

## Best Possible Solution

Keep the PR open for maintainer product/API review. The likely path is either to finish this narrowly scoped synchronous text-only hook with explicit trust and prompt-injection policy semantics, or to intentionally supersede it with a documented public policy for external pre-model tool-result transforms. Closing as implemented, duplicate, or ClawHub would hide a real core SDK gap because external plugins cannot add this timing themselves today.
