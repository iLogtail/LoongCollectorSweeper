## Summary

Keep PR #64907 open. Current main still does not expose `tool_result_before_model` as a public typed plugin hook, and the existing same-turn tool-result middleware is intentionally limited to bundled plugins. The PR is a real core/plugin API proposal rather than obsolete cleanup; it also contains useful discussion and review history around sync-only behavior, prompt-injection policy, and persistence semantics. It should not be auto-closed as implemented or superseded by the related async-hook issue because #62123 asks for a different broader async validation seam.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a narrow, well-tested `tool_result_before_model` hook with the PR's text-only/sync/prompt-injection-policy constraints, or request revisions that address the remaining review comments and clarify how this hook coexists with bundled-only tool-result middleware and the broader async transform request in #62123.
