## Summary

Keep #66912 open. Current main at 560ddd2f9b13fe2cf2073ea2922df481b7c06bb4 still downloads media from `msg.reply_to_message` without checking whether the replied-to message was authored by the bot, while the existing self-authored guard only skips top-level bot-authored Telegram message updates. The PR appears to target the right owner module and the latest provided diff now wires the guard, but it should be reviewed with a regression test and the unrelated generated baseline change removed before landing.

## Best Possible Solution

Keep the PR open and ask for a clean, focused Telegram plugin patch: add the self-authored reply-target guard in `resolveReplyMediaForMessage`, preserve normal user-authored reply-media ingestion, add regression coverage in `extensions/telegram/src/bot.test.ts` proving no file/media fetch occurs when `reply_to_message.from.id` matches `ctx.me.id`, and drop unrelated generated baseline changes before maintainer review.
