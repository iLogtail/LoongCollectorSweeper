## Summary

Keep open. Current main still leaves the reported mcporter-backed QMD search path vulnerable to stdout noise: the daemon-start path runs before the first call, but the subsequent mcporter call response is still parsed with raw JSON.parse(result.stdout). The newer noisy-output parser only covers direct qmd command output, not mcporter call JSON.
