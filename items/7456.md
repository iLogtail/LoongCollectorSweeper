## Summary

Keep #7456 open. Current main still lacks wizard-wide Back navigation for `openclaw onboard`: the CLI path is a linear call into `runSetupWizard`, the shared prompt contract and Clack adapter expose cancellation but no back/navigation result, and the Gateway wizard RPC only supports start/next/cancel/status. The related #7762 PR was closed unmerged, #5880 is interrupted-session resume rather than in-session back navigation, and #18286 was only a closed/unmerged foundational dialog proposal, not shipped onboarding behavior.

## Best Possible Solution

Keep #7456 open. The best product/code path is a core wizard navigation contract: add an explicit Back/previous-step result to `WizardPrompter` and Gateway wizard schemas/session handling, refactor `runSetupWizard` into a typed state machine that preserves answers across backward and forward movement, align local submenu Back behavior with that model, and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped because it was closed unmerged.
