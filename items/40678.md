## Summary

Current `main` still lacks an opt-in TUI mirror/observe mode. The TUI intentionally isolates `/new` into a fresh `tui-<uuid>` session, chat events are filtered to the active session key, and the published TUI docs expose only `--session`/`--deliver` while still pointing users to `openclaw logs --follow` as the monitoring path. That means the requested cross-channel visibility is not implemented well enough to close.

## Evidence
