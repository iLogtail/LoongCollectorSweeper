## Summary

Keep PR #39389 open. Current main still has the Feishu-specific docs gap this PR targets, and the PR is not superseded by the broader accountId docs work because Feishu docs still omit the routing field and wildcard example. The submitted wording also needs revision before merge because it can conflate Feishu `defaultAccount` outbound selection with the literal `default` account used by binding matching.

## Best Possible Solution

Keep the PR open and revise the Feishu docs wording. The best patch should document the actual binding contract precisely: omitted `match.accountId` matches the literal `default` account id in the generic matcher, use an explicit Feishu account id such as `main` for a named account, and use `accountId: "*"` to match any configured Feishu account. Coordinate with #49769 or a successor for the broader CLI/type/docs cleanup, but keep a Feishu-specific note and example on `docs/channels/feishu.md`.
