## Summary

Keep PR #40961 open. Current main has partially moved since the PR: subagent announce delivery now defaults to 120s and is configurable, but the PR's central remaining change is still not implemented because `sessions_send` A2A still caps its initial `agent.wait` phase at 60s. The PR likely needs a rebase/rework rather than cleanup closure.

## Best Possible Solution

Keep #40961 open for maintainer review, but rework it against current main: preserve or explicitly revisit the shipped 120s subagent announce default, replace the sessions_send A2A 60s cap with a bounded/configurable timeout policy, decide whether the `agent-step.ts` cap is in scope, and add focused regression coverage for A2A waits above 60s.
