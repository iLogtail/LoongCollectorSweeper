## Summary

Keep PR #39905 open. Current main still does not implement the PR's core feature: Telegram reply threading and preview streaming are resolved once from account-level config, and the direct/group/topic config types and Zod schemas do not accept conversation-scoped `replyToMode`, `streaming`, or `streamMode`. The related quoted-DM streaming subproblem has been fixed independently on main, but that does not supersede the conversation-scoped override work.

## Best Possible Solution

Keep this PR open until a maintainer either implements the conversation-scoped Telegram override surface on current `extensions/telegram` code or explicitly rejects the product direction. A current implementation should add typed/schema-supported `replyToMode` and streaming overrides for direct, group, and topic config; resolve precedence as topic over group/direct over account before normal dispatch; apply the same reply mode resolution to native command delivery; and update Telegram docs/tests on the extension paths.
