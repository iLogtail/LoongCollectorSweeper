## Summary

Current `main` does not implement the PR's core feature: Telegram `replyToMode` and preview `streaming` are still account-level only, with no per-direct/group/topic config fields or resolution path. One sub-problem from the PR was implemented independently on `main` later: DM preview streaming now stays on a single edited message. Because the main request remains unshipped and the PR is still salvageable, this should stay open rather than be auto-closed.
