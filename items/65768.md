## Summary

Keep #65768 open. Current main still does not let MiniMax web_search resolve MINIMAX_OAUTH_TOKEN: the runtime lookup, provider metadata, contract artifact, and tests only cover the configured web-search key, MINIMAX_CODE_PLAN_KEY, MINIMAX_CODING_API_KEY, and MINIMAX_API_KEY. The linked PR #65785 appears to target the right fix, while #65779 was closed unmerged, so the bug is not implemented on main yet.

## Best Possible Solution

Keep the issue open until a focused MiniMax plugin fix lands. The likely patch is to add MINIMAX_OAUTH_TOKEN to the MiniMax web-search runtime lookup, public web-search provider metadata/contract artifact, manifest or UI help where applicable, and add regression tests proving OAuth-token-only web_search succeeds while preserving existing coding-plan key precedence.
