## Summary

Keep open. Current `main` still decodes gateway exec child-process stdout/stderr with bare `Buffer.toString()` before the text is sanitized and wrapped into `Exec finished` system messages, which matches the failure mode described in the issue comments. There is a newer Windows code-page-aware decoder in `src/node-host/invoke.ts`, but that fix has not been applied to the gateway exec supervisor path, and the docs still describe Windows `system.run`/`exec` Chinese mojibake as an active workaround case.

## Evidence

- **Gateway exec child adapter still uses implicit UTF-8 decoding:** `createChildAdapter()` forwards both stdout and stderr with `chunk.toString()` and no Windows code-page handling. That is the same buffer-to-string step the reporter called out as the likely corruption point.
  - file: [src/process/supervisor/adapters/child.ts:111](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/process/supervisor/adapters/child.ts#L111)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '90,125p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Exec system-message path only consumes already-decoded strings:** `runExecProcess()` receives `data: string`, sanitizes it, and appends it to the aggregated exec session output. Later gateway follow-up messages reuse that aggregated text for `Exec finished ...` summaries, so any mojibake introduced earlier will propagate into the system message path.
  - file: [src/agents/bash-tools.exec-runtime.ts:599](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/agents/bash-tools.exec-runtime.ts#L599)

  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Windows-aware decoding exists, but only on the node-host path:** `src/node-host/invoke.ts` probes `chcp`, maps Windows code pages like 936/54936, and decodes captured buffers with `TextDecoder`. That is a partial mitigation elsewhere in the codebase, not a fix for the gateway exec adapter above.
  - file: [src/node-host/invoke.ts:123](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/node-host/invoke.ts#L123)
