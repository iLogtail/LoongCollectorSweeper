## Summary

Keep open. Current `main` still rebuilds the next attempt from the original `params.prompt` and explicitly `continue`s after successful auto-compaction, while the retry handler only resets subscriber state and replay/liveness bookkeeping. That means the codebase acknowledges fresh-prompt replay as unsafe in one compaction-timeout path, but does not appear to implement a stable post-compaction resume for the successful auto-compaction path described in #66126.

## Evidence

- **Successful auto-compaction still retries the same prompt:** `runEmbeddedPiAgent()` rebuilds each loop iteration from `params.prompt`, and on successful auto-compaction it logs `retrying prompt` and `continue`s the run loop instead of switching to a distinct resume payload or retry marker.
