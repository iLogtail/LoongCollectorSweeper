## Summary

Keep open. Current main does not implement the PR’s wrapper-owned durable ACP artifacts, and the AcpX integration has since moved to a thin wrapper around the published acpx package. The PR is stale in shape and has unresolved review risks, but the requested durability behavior is concrete and potentially useful, so it needs maintainer/upstream acpx direction rather than automated closure.

## Best Possible Solution

Keep the item open for maintainer judgment. The best path is to move the durable terminal-truth artifact design into openclaw/acpx, include redaction and abort-cleanup fixes there, publish a new acpx release, then update extensions/acpx/package.json and add focused OpenClaw integration coverage for the consumed behavior.

## Risks / Open Questions

- The current PR is unlikely to apply cleanly because current main delegates to acpx/runtime instead of owning the spawn/process code this patch modifies.

- The artifact creation and abort-listener lifecycle needs redesign around the current acpx startTurn/runTurn contract, with tests in the upstream runtime package and an OpenClaw package bump.
