## Summary

Keep open. Current `main` no longer has the extension-local ACPX runtime shape this PR patches; it now wraps upstream `acpx/runtime`, and I could not verify that the PR’s terminal-artifact behavior is already shipped there. The PR discussion also contains useful follow-up fixes plus at least one unresolved review risk, so there is not strong evidence for any safe auto-close reason.

## Evidence

- **Current ACPX runtime is a thin wrapper over upstream acpx/runtime:** `extensions/acpx/src/runtime.ts` imports `AcpxRuntime as BaseAcpxRuntime` from `acpx/runtime` and delegates `ensureSession`, `runTurn`, `getStatus`, `cancel`, and `close` to that upstream runtime instead of implementing the old local wrapper logic this PR modifies.
  - file: [extensions/acpx/src/runtime.ts:1](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/acpx/src/runtime.ts#L1)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Delegation happens in the live code path:** The constructor instantiates `BaseAcpxRuntime`, and `runTurn` simply yields from the resolved delegate. That means the PR's proposed `createRunArtifacts` / `finalizeRun` integration points do not exist on current `main` in this extension.
  - file: [extensions/acpx/src/runtime.ts:228](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/acpx/src/runtime.ts#L228)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current extension tree has no run-artifacts module:** Listing `extensions/acpx/src` on current `main` shows `runtime-internals/error-format.mjs`, `mcp-command-line.mjs`, and `mcp-proxy.mjs`, but no `runtime-internals/run-artifacts.ts` or matching artifact tests from the PR.
  - command: `find extensions/acpx/src -maxdepth 2 -type f | sort`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Main ships an upstream acpx bump, but not proof of this PR’s behavior:** The bundled ACPX extension now depends on `acpx` `0.6.0`, and the changelog notes an ACP/OpenCode runtime update. That shows the implementation moved upstream, but the repo does not prove that this specific terminal-artifact contract was shipped from OpenClaw `main`.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/acpx/package.json#L7)

I therefore do not have high-confidence evidence that the PR is already implemented upstream.
