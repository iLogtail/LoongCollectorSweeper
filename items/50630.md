## Summary

Keep open. Current main still allows the reported `gateway.auth.mode="none"` plus `gateway.tailscale.mode="serve"` startup path: runtime config only rejects Funnel without password and non-loopback unauthenticated binds, while Serve forces/defaults loopback. The auth layer still accepts `mode="none"` unconditionally, and an existing test explicitly asserts that Tailscale Serve with `mode="none"` resolves successfully. Related PR #50631 remains open/unmerged and PR #51339 is closed/unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep #50630 open until main rejects `gateway.auth.mode="none"` whenever `gateway.tailscale.mode` is `serve` or otherwise requires a real shared-secret/identity-bearing auth mode for Tailscale Serve HTTP exposure. The likely fix belongs in `src/gateway/server-runtime-config.ts`, with a regression test replacing the current successful `mode=none` + `serve` case, and docs/audit wording updated to match the enforced startup contract. PR #50631 can be used as prior art, but the issue should remain open until a narrow fix lands and ships.
