## Summary

Keep #71932 open. Current main does have shared Google Generative AI transport code that prefixes bare model ids with `models/`, and Google model normalization covers `gemini-3.1-flash-lite-preview`, but I did not find enough source/test/history evidence to prove the reported sub-agent/reset 404 is fixed or obsolete. The issue is new, the maintainer comment already asked for the missing provider config, sub-agent runtime, spawn payload, and outbound request URL, and those details are still needed to distinguish a real sub-agent/reset divergence from a Google `baseUrl` or inline-provider configuration problem.

## Best Possible Solution

Keep the issue open and ask the reporter to retest on v2026.4.24/current main with redacted `models.providers.google` config, runtime (`subagent` vs `acp`), full `sessions_spawn` payload, and the debug outbound URL. If it reproduces, add a focused regression test around the sub-agent/reset path for `google/gemini-3.1-flash-lite-preview` and fix the layer that mutates model id or baseUrl before `extensions/google/transport-stream.ts`.
