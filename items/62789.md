## Summary

Keep open. Current main still lacks the requested cron command payload contract and runtime execution path: cron payload types, Gateway schema, validation, timer dispatch, CLI, agent tool docs, and scheduled-task docs are still limited to systemEvent and agentTurn. The PR discussion narrowed this branch to backend/runtime/server work and split CLI follow-up separately, so earlier CLI-only review comments do not make the backend PR obsolete.

## Best Possible Solution

Keep #62789 open or manually consolidate it with #18160. If maintainers want this in core, review the narrowed backend/runtime diff and complete the command/exec payload contract across cron types, Gateway schema, normalization, runtime dispatch, delivery/run-log behavior, CLI/UI/docs, and tests with exec approval/sandbox semantics aligned. If maintainers instead decide this belongs only in plugins/ClawHub, close it with explicit product guidance after reconciling the still-open core tracker.
