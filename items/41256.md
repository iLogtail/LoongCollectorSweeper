## Summary

Keep open. Current main still has the reported Discord upload/content-type bug and the WebP/animation preservation issue: Discord upload paths construct untyped Blobs, and `loadWebMedia` still only exempts GIFs from image optimization before converting optimized images to JPEG/PNG. The linked fix PRs in the provided GitHub context (#41265, #41419, #41714) are open/unmerged rather than shipped.

## Best Possible Solution

Keep #41256 open and land one focused fix, likely by reworking one of #41265/#41419/#41714 against current main: make Discord file Blob creation accept/preserve `contentType`, thread `media.contentType` through shared sends, component sends, and native command file responses, and update `src/media/web-media.ts` so WebP and animated images bypass lossy JPEG optimization with regression tests for static WebP, animated WebP/GIF/APNG where feasible, and typed Discord Blobs.
