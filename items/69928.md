## Summary

Keep open. Current main still has the reported provider-agnostic Anthropic Messages adaptive-effort emission: Opus 4.7 is treated as adaptive, high/xhigh map to explicit `output_config.effort` values, and `buildAnthropicParams` emits that field without a `github-copilot` provider/model compatibility guard. The related PRs mentioned in the issue context were closed unmerged, and the open low/minimal follow-up does not fully supersede this high/xhigh report.

## Best Possible Solution

Keep #69928 open until the Anthropic Messages transport has a provider/model compatibility rule for `github-copilot/claude-opus-4.7`. The conservative fix is to omit or coerce `output_config.effort` to `medium` for that provider/model pair for all non-medium levels, with focused regression tests for high, xhigh, low/minimal, direct Anthropic Opus 4.7, and Copilot Opus 4.6 scope guards. A small per-provider/model effort capability helper would be the best long-term shape if maintainers want to cover future proxy drift.
