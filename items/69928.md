## Summary

Current main still has the reported provider-agnostic Anthropic Messages payload behavior. Claude Opus 4.7 is treated as adaptive-thinking capable, `high` and `xhigh` resolve to explicit `output_config.effort` values, and the transport emits that field without a `github-copilot` provider/model guard. The existing regression coverage only checks direct Anthropic-style Opus 4.7 xhigh emission, not the Copilot proxy path from #69928.

## Best Possible Solution

Keep the issue open and land a focused Anthropic Messages transport fix for `github-copilot/claude-opus-4.7`: either omit `output_config.effort` for that provider/model pair or normalize all unsupported efforts to `medium` through a provider/model capability table. Add regression tests for Copilot Opus 4.7 with `high`, `xhigh`, and low/minimal, plus scope guards proving direct Anthropic Opus 4.7 and other Claude adaptive models keep their intended behavior.
