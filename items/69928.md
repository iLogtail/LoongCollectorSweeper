## Summary

Keep open. Current main still contains the provider-agnostic Anthropic Messages payload path that maps Claude Opus 4.7 `high`/`xhigh` to `output_config.effort` and emits it without checking `model.provider`. The default GitHub Copilot catalog now marks bundled Copilot models as `reasoning: false`, which reduces exposure for the common default setup, but it is not the requested provider/model capability guard and configurable or discovered reasoning-capable Copilot Claude models can still reach the bad branch. Related fix PRs #69937, #69944, and #70376 were closed unmerged.

## Best Possible Solution

Keep the issue open and implement a provider/model capability guard in `src/agents/anthropic-transport-stream.ts`: for `github-copilot/claude-opus-4.7`, either omit `output_config.effort` or clamp all non-medium adaptive efforts to `medium`. Add transport tests covering Copilot Opus 4.7 high/xhigh and low/minimal, plus scope guards for direct Anthropic Opus 4.7 and other Copilot Claude models. If maintainers prefer consolidation, link this issue to the broader open Copilot Opus 4.7 effort-capability work instead of closing it as fixed.
