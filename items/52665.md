## Summary

Current main still does not implement the requested sandbox-common apt retry/fix-missing hardening. The documented setup script builds Dockerfile.sandbox-common, whose apt update/upgrade/install commands remain plain apt-get calls, and the cold-start base image path also lacks retry handling. The related #52724 context is the same attempted fix, but main has not picked up the behavior, so this issue remains actionable.

## Best Possible Solution

Keep the issue open and implement apt retry hardening in the sandbox image build path, likely matching the existing install-sh-smoke pattern: retry apt-get update, add Acquire::Retries to upgrade/install, decide deliberately whether --fix-missing is appropriate, and cover both Dockerfile.sandbox-common and the cold-start Dockerfile.sandbox path with a focused dockerfile test.
