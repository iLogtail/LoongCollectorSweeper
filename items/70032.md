## Summary

Current main does not implement the requested Nextcloud Talk typing indicator. The bundled plugin sends messages and reactions, but its inbound reply path has no typing lifecycle and the strict Nextcloud Talk config/docs expose no `typingIndicator` option. This is a recent, scoped plugin-owned enhancement, not a duplicate or obsolete report.

## Best Possible Solution

Keep #70032 open as a scoped Nextcloud Talk plugin enhancement. The likely fix belongs under `extensions/nextcloud-talk`: add a typed `sendTypingNextcloudTalk` helper for the Talk typing endpoint, verify auth against upstream Nextcloud Talk behavior, add config/schema/docs/tests for the chosen default, and wire start/stop cleanup into the inbound reply lifecycle. If `dispatchInboundReplyWithBase` cannot pass plugin typing callbacks cleanly, extend that SDK helper additively or use the lower-level reply pipeline in the plugin.
