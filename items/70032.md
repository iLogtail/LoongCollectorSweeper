## Summary

Keep #70032 open. Current main owns a bundled Nextcloud Talk plugin, but the requested typing indicator is not implemented: the plugin sends messages/reactions only, inbound replies use the shared dispatch path without typing callbacks, and the Nextcloud Talk config/docs expose no typing-indicator option. The provided related items are other-channel typing/feedback reports and do not supersede this Nextcloud Talk request.

## Best Possible Solution

Keep the issue open. Best path is a focused bundled-plugin change in extensions/nextcloud-talk: add a typed start/stop typing helper for the Nextcloud Talk typing endpoint, wire it into the inbound reply lifecycle with cleanup on success/error/timeout, add schema/docs if configurable, and cover it with Nextcloud Talk unit tests around request shape and reply lifecycle behavior.
