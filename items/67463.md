## Summary

Keep PR #67463 open. Current main already covers the task timestamp normalization half, but the doctor SecretRef false-positive remains present on main and is also tracked by open bug #65201. The PR is not safe to auto-close as implemented or duplicate because its remaining doctor fix is still useful, although it needs maintainer/author cleanup to preserve doctor lazy-loading boundaries and avoid resolving SecretRefs on non-token auth paths.

## Best Possible Solution

Keep the PR open or ask the author to revise it into a narrow doctor-only fix: rebase on current main, drop the already-implemented task timestamp test additions, dynamically import resolveGatewayAuthTokenForService inside runGatewayAuthHealth, gate SecretRef resolution behind the existing needsToken decision, then pass the resolved token as an authOverride and keep the regression test proving a resolvable SecretRef suppresses the false-positive doctor note.
