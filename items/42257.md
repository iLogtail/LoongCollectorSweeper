## Summary

Keep #42257 open. Current main still contains the Feishu external-key behavior that PR #42301 identifies as the likely root cause, and the related PR is still open/unmerged in the supplied GitHub context. The Feishu inbound media pipeline exists, but a legitimate Feishu image key containing `..` would still be rejected before download, so this is not implemented on main.
