## Summary

Keep open. Current main still hard-rejects any `sessions_send` payload that contains both `sessionKey` and `label`, which is the exact failure reported. The reporter’s follow-up says they resolved it by locally patching source and the ClawX bundled runtime, but that fix is not present on `main`; the related #64846 fix PR is still open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a maintainer-reviewed fix in the `sessions_send` implementation and relevant client argument assembly. The narrow code path is `src/agents/tools/sessions-send-tool.ts`: decide the intended contract for duplicate values, prefer `sessionKey` when safe, keep true ambiguous/different targets guarded if that is the product decision, and add regression coverage for duplicate `sessionKey`/`label` payloads. Treat the separate `tools.agentToAgent` policy denial as a distinct configuration/policy issue.
