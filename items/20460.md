## Summary

Keep open. Current main still writes config backups beside the config as `${configPath}.bak` plus numbered `.bak.1` through `.bak.4`; it does not create a `config-backup/` directory or use datetime-suffixed backup names. The linked PR #20877 is useful prior art, but the provided GitHub context shows it was closed unmerged, and current tests/docs still codify adjacent numbered backups.

## Best Possible Solution

Keep this open as a core config enhancement. Implement the backup directory and datetime naming in `src/config/backup-rotation.ts`, update `writeConfigFile`, config mutation/include writes, recovery/audit/log paths, tests, and user-facing docs, and use PR #20877 as prior art while fixing its retention-count issue and preserving compatibility with existing adjacent backups.
