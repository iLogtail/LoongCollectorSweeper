## Summary

Keep open. Current `main` still implements `/acp close` as close-plus-unbind only; it does not archive ACP-created Discord threads, does not carry ACP-created-thread ownership metadata in Discord thread bindings, and does not have a Discord outbound archive-after-reply path. The PR discussion also matters here: the two concrete review concerns called out in comments were reportedly fixed in follow-up commits, so there is not strong evidence this should be auto-closed as stale or superseded. The work looks unshipped and still relevant, but it now needs a rebase/port into the current Discord extension layout.
