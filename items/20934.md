## Summary

Keep #20934 open. Current main still exposes session reset, status/list/usage, compaction, and model/reasoning updates through Gateway RPC/control-plane handlers and related tools, but it does not provide the requested REST-native `/v1/sessions/*` endpoints for HTTP `/v1/chat/completions` integrations. The related REST PR #23941 was closed unmerged, and the existing issue discussion already identifies this as the canonical remaining HTTP API gap.

## Best Possible Solution

Keep #20934 open as the canonical REST session-management tracker. A complete solution should add authenticated Gateway HTTP endpoints for reset, status/list or usage, compact, and model/reasoning patch operations by reusing the existing `sessions.reset`, `sessions.list`/`sessions.usage`, `sessions.compact`, and `sessions.patch` semantics, with route/auth/scope tests and docs under the OpenAI-compatible HTTP API.
