## Summary

Keep open. Current main still does not provide the requested REST session-management endpoints for `/v1/chat/completions` integrations. The related discussion is useful: #23941 attempted the REST endpoints but is closed/unmerged, and the #29985 comment explicitly says the REST endpoints remain a follow-up. Current main also does not expose the claimed `api.resetSession()` Plugin SDK surface, so this is not a clean ClawHub/plugin close either.

## Evidence

- **HTTP route table omits requested endpoints:** The built-in HTTP dispatcher routes `/v1/models`, `/v1/embeddings`, `/tools/invoke`, `/sessions/{key}/kill`, `/sessions/{key}/history`, `/v1/responses`, and `/v1/chat/completions`; there is no `/v1/sessions/reset`, `/v1/sessions/status`, `/v1/sessions/compact`, `/v1/sessions/model`, or `/v1/sessions/reasoning` route.
  - file: [src/gateway/server-http.ts:953](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/server-http.ts#L953)
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Requested endpoints not present by search:** Targeted search for `/v1/sessions`, `sessions-http`, requested session endpoint paths, and `api.resetSession` found no matching implementation in gateway, plugin, plugin-sdk, or gateway/plugin docs.
  - command: `rg -n '"/v1/sessions|/v1/sessions|sessions-http|pathname: "/v1/sessions|isSessionsHttp|resetSession\b|api\.resetSession' src/gateway src/plugins src/plugin-sdk docs/gateway docs/plugins`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
