## Summary

Keep #69197 open. Current main still has the active `process poll` vs `maybeNotifyOnExit()` race the PR targets, and the PR is a recent focused bug fix with concrete linked reports and author live-validation. It is not obsolete or superseded, but it should be revised or reviewed for the overlapping-polls concern before merge.

## Best Possible Solution

Keep the PR open and ask for a small revision or maintainer decision on the active-poll tracking model. The best code path is to land this fix or an equivalent one that suppresses notify-on-exit while any poll is actively waiting, marks completion as consumed when poll reports the exit, preserves normal notify-on-exit when no poll is active, and includes focused regression tests for poll, no-poll, timeout-still-running, kill, and overlapping-poll cases.
