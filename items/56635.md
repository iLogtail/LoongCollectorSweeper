## Summary

Keep open. Current main still omits `--exclude-labels` from `buildGogWatchServeArgs()`, the Gmail config schema/types still have no `excludeLabels` field, and the linked PRs that would fix #56635 are still open rather than merged. The reported regression remains actionable in this repo.

## Best Possible Solution

Land one canonical fix that restores the default `SPAM,TRASH,DRAFT,SENT` exclusion in the Gmail watcher path, ideally with a supported `hooks.gmail.excludeLabels` config field if maintainers want user override behavior. The fix should update the schema/types/help/docs as needed and add focused regression tests for default labels, custom labels, and empty-label behavior.
