## Summary

Keep open. Current `main` still contains the mismatch described in #57775: the headless node host advertises and implements `system.execApprovals.get/set`, but Gateway node-command reconciliation filters declared commands through platform defaults that do not include those exec-approvals commands for Windows. That means `node.describe`/known-node surfaces can still omit them, and the Control UI still depends on advertised commands to show exec-approvals-capable nodes.
