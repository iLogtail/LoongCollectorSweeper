## Summary

Keep open. Current main still has the Windows-sensitive direct-run guard in `scripts/generate-base-config-schema.ts`, and the package scripts still route schema check/generation through that file. PR #52989 remains a focused, plausible fix and is not implemented or superseded on main.

## Best Possible Solution

Keep PR #52989 open. The best path is to land the `pathToFileURL` fix for `scripts/generate-base-config-schema.ts`, preferably with a small regression test or shared entrypoint helper; separately consider whether the same direct-run guard should be fixed in sibling generator scripts.
