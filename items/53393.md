## Summary

Keep PR #53393 open. Current main already has a narrower shipped fix for the user-visible #49568 model-catalog hot-reload problem, but it does not implement this PR’s broader config-derived cache registry, and the context-window cache invalidation concern is still distinct. The PR also has an unresolved P1 review note about stale async context loading repopulating the cache after invalidation, so it should not be auto-closed as implemented or superseded.

## Best Possible Solution

Do not auto-close this PR. Maintainers should either ask for a revised version that keeps the shipped targeted model-catalog behavior, adds a generation or cancellation guard for context-window invalidation, and proves the remaining cache behavior with tests, or explicitly close it manually if the product direction is to avoid a registry and keep per-cache invalidation local.
