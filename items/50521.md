## Summary

Keep open. Current `main` still does not reject null bytes in `validateArchiveEntryPath()`, so the PR is not implemented on `main`. The existing archive extraction flow relies on Node's lower-level filesystem rejection instead of returning the clearer archive-path validation error proposed here. The PR stays small, tested, and aligned with similar explicit null-byte guards elsewhere in the repo, so there is not strong evidence for an auto-close.
