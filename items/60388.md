## Summary

Keep #60388 open. Current main still routes run-budget timeouts through the timeout failover path that can rotate profiles and escalate to model fallback. The supplied related PR #62682 directly tracks a plausible fix, but it is open and not present on main.

## Best Possible Solution

Keep #60388 open and land or adapt PR #62682: pass enough abort context into the fallback layer to distinguish a run-budget TimeoutError from retryable provider/model failures, preserve fallback for genuine provider/rate-limit/auth failures, and add regression tests around run-budget timeout short-circuiting.
