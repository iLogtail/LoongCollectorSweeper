## Summary

Keep #36614 open. Current main still has the write-side contamination path: `recordInboundSession` writes delivery route fields to the caller-provided last-route target, and several direct-message channel/plugin paths still pass the main session key under scoped DM routing. #38418 mitigated read-side WebChat stale-route inheritance, and WhatsApp has a better scoped guard, but the reported iMessage/main-session write pollution remains in current source.

## Best Possible Solution

Keep #36614 open and fix the write-side target selection. The best maintainer path is to centralize inbound last-route target resolution, or update each direct-message caller to use `resolveInboundLastRouteSessionKey`, so `lastRoutePolicy: session` writes delivery metadata only to the scoped inbound session while `dmScope: main` continues to update the main session. Add regression coverage for iMessage plus representative Slack, Signal, LINE, Matrix, Mattermost, and Discord component paths, using the existing WhatsApp guard/tests as the model.
