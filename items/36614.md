## Summary

Keep #36614 open. Current main still has write-side last-route contamination paths: per-channel-peer routing is session-scoped, but several direct-message channel handlers still target the main session for inbound last-route writes, and the shared update path persists deliveryContext/lastChannel/lastTo even when ctx is suppressed. #38418 only addressed the read-side stale-route symptom, and the related write-side PRs in the provided context (#36625 and #40233) were closed unmerged.

## Evidence
