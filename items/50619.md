## Summary

Keep open. Current main has post-switch mitigations for stale context windows and oversized prompts, but it still does not implement the requested `/model` pre-switch check: the command applies the model immediately, the gateway patch path stores the override without comparing current session size to the target model budget, and the runner-side overflow precheck only runs later when building a prompt. The issue’s TPM/rate-limit case remains especially relevant because OpenAI Codex `gpt-5.4` is capped to a 272k runtime context, so a ~200k carried context can still be submitted and hit provider TPM limits rather than triggering context-overflow compaction.

## Best Possible Solution

Keep this open or explicitly merge it into a maintainer-chosen canonical issue. The likely fix belongs in the `/model`/`sessions.patch` path: resolve the target model budget, compare it with the current session’s fresh prompt/context estimate, warn or block before switching when above a configurable safe threshold, and optionally offer `/compact` or `/new`. Keep the existing runner-side overflow compaction as a fallback, not as the only user-facing protection.
