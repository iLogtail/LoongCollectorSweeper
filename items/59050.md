## Summary

Current `main` already exposes bootstrap ordering as an extension seam: `agent:bootstrap` hooks/plugins receive a mutable `bootstrapFiles` array before prompt assembly, and the prompt builder preserves that array order. Given `VISION.md`'s plugin-first scope rule for optional capabilities, configurable bootstrap ordering fits a hook pack/plugin better than new core config.
