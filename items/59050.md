## Summary

Current `main` still hardcodes workspace bootstrap prompt ordering and exposes no config for per-agent/default reordering. Hooks can mutate the bootstrap file set, but the final system prompt renderer re-sorts files with a fixed precedence, so the requested `bootstrapFileOrder` behavior is not implemented on `main` and this is not a safe cleanup close.

## Evidence

- **Prompt renderer hardcodes bootstrap file precedence:** `CONTEXT_FILE_ORDER` fixes the rendered order to `AGENTS.md`, `SOUL.md`, `IDENTITY.md`, `USER.md`, `TOOLS.md`, `BOOTSTRAP.md`, `MEMORY.md`.
