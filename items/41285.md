## Summary

Keep open. Current main intentionally returns `AUTH_TOKEN_MISSING` when the Control UI connects without a shared token, and `openclaw dashboard --no-open` intentionally omits `#token=` for SecretRef-managed gateway tokens. That explains parts of the report, but there is no strong proof that the March 2026 failure is fixed or obsolete, and current Control UI copy still tells users that `openclaw dashboard --no-open` yields a tokenized URL, which conflicts with the documented/implemented SecretRef behavior.
