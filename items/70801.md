## Summary

Keep open. Current `main` still restarts the macOS LaunchAgent with direct `launchctl kickstart -k` on the normal path, with no two-phase stop/wait/start flow and no test coverage for the ordering the issue asks for.

## Evidence

- **Current restart path is still single-phase:** `restartLaunchAgent` does stale-port cleanup, clears disabled state, then immediately runs `launchctl kickstart -k`. On success it returns without any `bootout`/wait-for-old-pid phase; on failure it only bootstraps and retries.
  - file: [src/daemon/launchd.ts:685](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/daemon/launchd.ts#L685)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '685,718p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Detached handoff path also uses kickstart-first:** The launchd restart handoff script waits for the caller PID to exit, then runs `launchctl enable` followed by `launchctl kickstart -k`, with bootstrap fallback only after failure. It does not implement the issue's proposed stop + ensure-old-pid-gone + bootstrap/start ordering.
  - file: [src/daemon/launchd-restart-handoff.ts:96](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/daemon/launchd-restart-handoff.ts#L96)
  - command: `nl -ba src/daemon/launchd-restart-handoff.ts | sed -n '96,118p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Unit tests lock in the current kickstart-only behavior:** The restart test explicitly expects `restartLaunchAgent` to call `kickstart -k` and asserts there is no `bootout` or `bootstrap` on the normal success path, which is the opposite of the requested two-phase restart behavior.
  - file: [src/daemon/launchd.test.ts:703](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/daemon/launchd.test.ts#L703)
  - command: `nl -ba src/daemon/launchd.test.ts | sed -n '703,771p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Integration coverage does not prove the requested ordering:** The integration test only checks that restart eventually yields a different PID. It does not assert that the old PID is fully gone before the new process is started, so the acceptance criteria in the issue are not already covered on `main`.
  - file: [src/daemon/launchd.integration.e2e.test.ts:177](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/daemon/launchd.integration.e2e.test.ts#L177)
