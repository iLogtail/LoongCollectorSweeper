## Summary

Keep open. Current main has a stale-port cleanup before macOS `kickstart -k`, but it has not implemented the requested two-phase LaunchAgent stop/wait/start ordering, and the unit tests still assert the direct kickstart path with no bootout/bootstrap on the normal restart path. The referenced PR #43961 is closed unmerged in the provided context, so it does not supersede this issue.

## Best Possible Solution

Keep the issue open for maintainer judgment or implementation. A complete fix would update `src/daemon/launchd.ts` and likely `src/daemon/launchd-restart-handoff.ts` to use a launchd-aware two-phase restart that stops/unloads or otherwise confirms the old service process is gone before starting a replacement, while preserving self-restart handoff behavior. Tests should replace the current no-bootout assertion with explicit ordering coverage and retain unloaded-service recovery coverage.

## Evidence
