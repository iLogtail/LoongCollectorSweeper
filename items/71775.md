## Summary

Keep open. Current main strongly suggests the reported 11 PM to 5 AM context loss may be the default 4:00 AM daily session reset, but the issue is fresh and lacks the config, session store timestamps, WeChat plugin version, session key, and logs needed to distinguish expected reset behavior from a core session-key/plugin regression.

## Best Possible Solution

Keep the issue open for maintainer triage. Ask the reporter for session.reset, session.resetByChannel, session.dmScope, WeChat plugin version, /status before and after the gap, the affected session key, redacted sessions.json timestamps (sessionStartedAt, lastInteractionAt, updatedAt), and gateway logs around 4:00-5:00 AM. If the default daily reset explains it, the product fix should be clearer reset notice/docs/config guidance; if not, add a focused regression test around WeChat direct-session reuse or session-key stability.
