## Summary

Keep open. Current `main` still renders Slack outbound markdown as `mrkdwn` text and Block Kit section/context/action blocks; there is no markdown-to-`rich_text` auto-conversion path on the outbound Slack renderer, so this RFC remains unimplemented and still in scope for the bundled Slack plugin.

## Evidence

- **Slack markdown renderer still targets `mrkdwn`:** `markdownToSlackMrkdwn()` parses markdown with `markdownToIR()` and renders it with Slack marker syntax (`*`, `_`, `~`, code fences) via `renderMarkdownWithMarkers()`. There is no `rich_text` block builder on this path.
  - file: [extensions/slack/src/format.ts:121](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/slack/src/format.ts#L121)
  - command: `nl -ba extensions/slack/src/format.ts | sed -n '1,180p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Tests assert `mrkdwn` output, not `rich_text` blocks:** Current tests expect markdown normalization into Slack `mrkdwn` strings like `*bold*`, bullet text, and `<url|label>` links. No outbound tests cover `rich_text` block generation from markdown.
  - file: [extensions/slack/src/format.test.ts:9](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/slack/src/format.test.ts#L9)

- **Slack Block Kit renderer emits section/context/actions/header/divider only:** `buildSlackPresentationBlocks()` converts presentation text/context into `mrkdwn` section/context blocks and interactive controls into `actions`; it does not emit Slack `rich_text` blocks.
  - file: [extensions/slack/src/blocks-render.ts:124](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/slack/src/blocks-render.ts#L124)
