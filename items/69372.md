## Summary

Keep PR #69372 open. Current main still has the raw protected Control UI canvas fallback the PR is trying to remove, so it is not obsolete or implemented on main. The PR should not be auto-closed, but it also should not be merged as-is because its timer-driven WebSocket reconnect matches the P1 review concern: current close/re-hello paths reject pending RPCs and clear active chat run state.

## Best Possible Solution

Keep the PR open and ask for a revision: preserve the `resolveCanvasIframeUrl` fail-closed behavior for protected `/__openclaw__/canvas` and `/__openclaw__/a2ui` paths, but replace the timer-driven transport reconnect with a non-disruptive Control UI capability refresh path or an idle-gated refresh that does not reject pending RPCs or clear active chat state. Add regression coverage for no raw iframe fallback, positive a2ui rewriting, and active chat/pending RPC survival across capability refresh.
