## Summary

Do not close PR #69372. Current main still has the Control UI canvas behavior the PR is trying to change, and the PR has a source-backed blocking review concern around forced websocket reconnects interrupting active work.

## Best Possible Solution

Keep the PR open, but rework it before merge: retain the fail-closed `resolveCanvasIframeUrl` behavior and tests, then replace the forced websocket reconnect with a non-disruptive Control UI capability refresh path, likely by adding an operator-safe refresh RPC or explicit hello expiry metadata and updating protocol/schema/tests accordingly.
