## Summary

Keep #38302 open. Current main still lacks an account-scoped Slack native slash command prefix or nativeNames mapping. The existing implementation supports only provider-level native command overrides such as Slack's /status -> /agentstatus rename, while Slack account config, schema validation, native registration, lookup, docs, and tests do not expose or exercise per-account native command names. The related attempts #38326 and #38391 were closed unmerged, and another user confirmed the same multi-app Slack collision in the issue discussion.

## Best Possible Solution

Keep #38302 open and implement an account-scoped Slack native command naming contract in the bundled Slack plugin. The likely path is to add an account-level slashCommand.nativeNames mapping or validated nativePrefix-derived mapping, thread it through Slack native spec generation and findCommandByNativeName dispatch for the resolved account, update Slack manifest/setup docs, and add focused multi-account tests that prove /oz-reset and /dross-reset style commands route to the intended internal command without breaking the existing /agentstatus provider override.
