## Summary

Keep open. Current main still only has the Telegram sendChatAction per-account 401 backoff/circuit breaker; it does not implement the PR's transient network/429/5xx cooldown path for the bot message-context handler. The linked issue #55811 was closed, but this PR's central behavior is not present in the inspected current main source or tests.

## Best Possible Solution

Keep this PR open or supersede it only with a canonical Telegram sendChatAction transient-cooldown PR. The implementation should live in `extensions/telegram/src/sendchataction-401-backoff.ts`, reuse the existing Telegram network/rate-limit/server-error helpers, preserve 401 suspension semantics, keep rejected transient starts visible to the typing guard, and add focused tests for cooldown suppression, expiry/recovery, 429/5xx/network cases, and mixed 401/transient sequences.
