## Summary

Keep open. Current `main` does not implement this PR’s transient `sendChatAction` cooldown behavior; the global Telegram handler still only backs off/suspends on 401s, and the existing typing guard is per-reply rather than a cross-message cooldown.

## Evidence

- **Global Telegram handler still only handles 401 failures:** `createTelegramSendChatActionHandler` only detects `401` via `is401Error`, tracks `consecutive401Failures`, and rethrows all other errors. There is no transient cooldown state, no 429/5xx/network classification, and no suppression window for later calls.

- **Core wiring still documents and injects only the 401 circuit breaker:** `bot-core.ts` still describes the shared handler as a `401 backoff / circuit breaker` for issue `#27092`; there is no mention of transient cooldown behavior in the injected account-global handler.
