## Summary

Keep #45710 open. Current main does not contain the proposed VS Code launch/task files, the `OUTPUT_SOURCE_MAPS` tsdown toggle, `clean:dist`, or the VS Code debugging docs, and I found no landed/superseding implementation. The PR is actionable DX work in this repo, not ClawHub scope, but its review thread also documents remaining launch-path concerns that need maintainer/product review before merge rather than auto-close.

## Best Possible Solution

Keep this PR open for maintainer review or a refreshed revision. A mergeable direction would preserve the useful pieces, especially env-gated source maps and clear debugging docs, while replacing the brittle `clean:dist` + `run-node` launch path with a repo-supported script or build profile that regenerates or preserves required runtime artifacts such as `dist/export-html` and copied UI assets. If maintainers do not want checked-in VS Code workspace defaults, convert the work to docs plus a local command/script instead of closing it as already shipped.
