## Summary

Keep #69186 open. Current main has backend lifecycle hooks (`agent_end`, `subagent_ended`) and Control UI web-push subscription/test plumbing, but it does not implement the requested Control UI completion/success sound, on/off setting, custom sound URL, or automatic turn-completion notification. The related closed #59199 covers the plugin hook signal only; it does not satisfy this user-facing browser audio feature.

## Best Possible Solution

Keep the issue open for maintainer product judgment. The concrete implementation path is a small Control UI preference stored with `UiSettings`, rendered under the existing Notifications/Communications settings, and triggered from the active-run terminal `final` path in `ui/src/ui/controllers/chat.ts` or the gateway event wrapper. It should distinguish success/final from aborted/error as desired, respect browser autoplay constraints, and include focused tests. If maintainers want this in ClawHub instead, first define a browser-side Control UI extension/event API that a plugin can use safely.
