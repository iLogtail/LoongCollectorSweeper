## Summary

Keep open. Current main has partial Telegram-side degradation for unavailable tokens, but the reported startup path is not fully fixed: the dotenv loader still does not read ~/.openclaw/secrets/.env, active Telegram botToken SecretRefs are still included in startup secret resolution, and the env SecretRef resolver still throws when an env var is missing or empty. The issue discussion's symlink/merge workaround remains consistent with current loader behavior.
