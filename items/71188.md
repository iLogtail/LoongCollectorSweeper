## Summary

Keep open. Current `main` still enables the bundled Bonjour advertiser by default, still ships the same `@homebridge/ciao` dependency path, and shows no Windows-specific suppression or alternate implementation for Gateway LAN discovery. There is a current workaround on `main` (`discovery.mdns.mode: "off"` or `OPENCLAW_DISABLE_BONJOUR=1`), but that does not resolve the reported flashing-console behavior while discovery remains enabled.

## Risks / Open Questions

- No Windows reproduction was run in this Linux checkout, so the exact `cmd.exe` spawn path was not directly verified here.
- The flashing behavior may live inside the shipped `@homebridge/ciao` dependency rather than OpenClaw-owned source, but OpenClaw still ships and enables that path by default.
- Because the workaround only disables discovery, closing as implemented would be too aggressive for a report asking to preserve discovery without visible console flashes.
