## Summary

Current main does not implement the requested MCP stdio proxy/NODE_OPTIONS propagation. The stdio config path still passes only configured `env`, the stdio transport still relies on the MCP SDK default environment plus that configured env, and newer docs/tests intentionally block `NODE_OPTIONS` in per-server stdio env while allowing ordinary proxy vars only when explicitly configured. The linked open PR #65574/#65526 remains the active product/fix discussion, especially because the later stdio env safety policy conflicts with the issue’s original `NODE_OPTIONS` workaround.

## Best Possible Solution

Keep #65525 open. Maintainers should resolve it together with #65526/#65574 by choosing a supported MCP stdio proxy propagation design: likely auto-propagate safe proxy variables such as `HTTP_PROXY`, `HTTPS_PROXY`, lower-case variants, and `NO_PROXY` as low-priority defaults, while explicitly deciding whether `NODE_OPTIONS` remains blocked, gets a narrowly-scoped trusted propagation path, or is replaced by a documented OpenClaw-supported proxy bootstrap for MCP child processes.
