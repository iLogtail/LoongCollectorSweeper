## Summary

Keep open. Current main still has the reported Control UI race: the model picker fires an async `sessions.patch`, updates only local override cache during the round trip, and neither the send button nor `handleSendChat()` tracks or awaits that pending model-switch operation before `chat.send`. Related provider-prefix/dropdown issues appear fixed, but those fixes do not cover this persistence-before-send race.

## Best Possible Solution

Keep this issue open and fix it in the Control UI. Add explicit pending model-switch state, likely on `AppViewState`/chat host, set it around `switchChatModel()`'s `sessions.patch`, disable the model picker and send controls while it is pending, and have `handleSendChat()` await the pending switch before `sendChatMessageNow()`. If the patch fails, roll back the picker state and do not send. Add a regression test with a deferred `sessions.patch` proving `chat.send` is not called until after the patch resolves, plus a failure case proving no send occurs after patch rejection.
