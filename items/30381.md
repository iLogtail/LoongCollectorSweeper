## Summary

Keep #30381 open. Current main still selects the agent from `x-openclaw-agent-id`, but `resolveOpenAiCompatModelOverride` independently rejects any non-OpenClaw request body `model` before dispatch. That means a generic OpenAI SDK model such as `gpt-4` with the agent header would still return HTTP 400 instead of using the selected agent's configured model.

## Best Possible Solution

Keep #30381 open. The likely fix belongs around `src/gateway/http-utils.ts` and `src/gateway/openai-http.ts`: when `x-openclaw-agent-id` resolves to a valid agent for chat completions, ignore or bypass OpenAI body `model` validation while still using the selected agent's configured model and still validating any explicit `x-openclaw-model` override. Add a regression test with `model: "gpt-4"` plus `x-openclaw-agent-id` proving dispatch succeeds and routes to the header-selected agent, and update the OpenAI HTTP docs if this becomes the intended compatibility contract.
