## Summary

Keep PR #55713 open. Current main still omits Mattermost from the reaction-capable platform guidance and from the reactions channel-behavior docs, while the Mattermost plugin already exposes and tests reaction support. The PR is therefore still useful, but it should be revised before merge to satisfy docs ordering and to document Mattermost's non-empty emoji requirement.

## Best Possible Solution

Keep the PR open and revise the docs-only change: add Mattermost to `docs/reference/templates/AGENTS.md` as `Discord, Mattermost, Slack`; add a Mattermost section to `docs/tools/reactions.md` in alphabetical service order; explicitly state that Mattermost uses emoji names, requires a non-empty emoji, and `remove: true` removes that specific reaction. No runtime code change is needed for this PR, and zh-CN docs should not be added under this repo.
