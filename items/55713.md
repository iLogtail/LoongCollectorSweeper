## Summary

Keep PR #55713 open. Current main still lacks the requested Mattermost entry in the workspace template and cross-channel reactions docs, while the Mattermost runtime and tests already support `react`; the PR is useful but should be amended before merge for docs ordering and the Mattermost non-empty-emoji exception.

## Best Possible Solution

Keep the PR open and ask for a small docs amendment: change the template list to `Discord, Mattermost, Slack`; add Mattermost to `docs/tools/reactions.md` in the appropriate ordered position; document that Mattermost uses emoji names and does not support empty `emoji`; leave localized docs alone in this repo. No runtime code change is needed.
