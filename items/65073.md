## Summary

Google Vertex service-account embeddings are useful, and the PR body plus review comments contain implementation details worth preserving. The current main architecture now treats memory embedding providers as plugin-owned adapters, with manifest ownership via `contracts.memoryEmbeddingProviders`, so a `google-vertex` memory embedding provider can be shipped as an external/plugin/ClawHub package without adding another core memory switch path.

## Best Possible Solution

Do not merge this core PR. Preserve the useful Vertex implementation notes and move the work to a ClawHub/plugin package that declares `contracts.memoryEmbeddingProviders: ["google-vertex"]`, registers a `MemoryEmbeddingProviderAdapter`, and fixes the review issues before publication.

## Risks / Open Questions

- A future external plugin still needs tests for service-account parsing, token refresh, Vertex request/response parsing, auto-selection skip behavior, and document/query task types.
- A maintainer could later decide to promote a well-used `google-vertex` memory embedding plugin into the bundled Google plugin, but this PR should not remain open for that product decision.
