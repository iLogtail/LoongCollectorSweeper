## Summary

Keep open. Current main still leaves the Usage header and daily chart on different data scopes: the UI loads `sessions.usage` and `usage.cost` separately, the header uses `sessions.usage.totals`, and the daily chart uses `usage.cost.daily`. On main, `sessions.usage` discovers configured agents, while `usage.cost` calls the cost summary once with no `agentId`, which resolves to the default/main agent session directory. The related PR #42063 directly tracks this missing aggregation but is still open/unmerged, so the issue is not implemented on main.
