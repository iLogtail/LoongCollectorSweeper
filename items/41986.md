## Summary

Current main has not fixed the reported Usage dashboard mismatch. The UI still renders the header from `sessions.usage` totals and the daily chart from `usage.cost`, while `sessions.usage` is multi-agent and `usage.cost` still calls the cost loader once without an agent scope. Open PR #70205 appears to be the active follow-up for the remaining aggregation fix, but it is not merged.

## Best Possible Solution

Keep #41986 open until the `usage.cost` aggregation fix lands. The likely product/code path is to land a narrow version of #70205 or a maintainer-approved successor that resolves `usage.cost` over the same configured-agent scope as `sessions.usage`, includes agent-scoped cache keys and explicit-agent behavior, and adds regression coverage for the Usage header versus daily chart scope.
