## Summary

Keep open. Current main still lacks create/update-time Feishu/Lark announce target validation, and canonical issue #39599 remains open. PR #39602 is unsafe as submitted because its head wipes the cron CLI registration module, but the underlying bug-fix work is focused and salvageable rather than a conservative auto-close candidate.

## Best Possible Solution

Keep this PR open for maintainer or contributor repair, or supersede it only with a clean canonical fix. The best implementation is to restore `src/cli/cron-cli/register.cron-add.ts`, add gateway/service validation that rejects Feishu/Lark announce jobs only when no explicit or deterministic target can be resolved, add clear CLI feedback, and keep #39599 as the canonical bug until regression coverage lands.
