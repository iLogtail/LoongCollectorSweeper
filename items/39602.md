## Summary

Keep #39602 open. Current main still does not reject Feishu/Lark announce cron jobs without a delivery target at create/update time, #39599 remains the canonical open bug, and this PR is still unsafe as submitted because its head wipes the cron CLI registration module.

## Best Possible Solution

Keep #39602 open for contributor repair, or supersede it with a clean PR linked to #39599. The fix should restore the cron CLI file, add service/gateway validation for Feishu/Lark announce jobs when no explicit or deterministic target exists, optionally add CLI fail-fast feedback, and include create/update regression coverage.
