## Summary

Keep open. Current main has partial runtime fixes for Telegram native commands, but the reported `sandbox explain --session telegram:slash:<id>` diagnostic path still cannot resolve the bare Telegram slash key to `telegram`, and docs still describe that bare Telegram native-command session form.

## Best Possible Solution

Keep the issue open and land a narrow diagnostics fix in `src/commands/sandbox-explain.ts`: either normalize bare native-command keys like `telegram:slash:<id>` into the agent-scoped store format or add a safe channel inference fallback for `<channel>:<subtype>:...` keys using bundled/static channel ids. Add regression tests proving `telegram:slash:<id>` and `agent:main:telegram:slash:<id>` report `channel: telegram` and honor `tools.elevated.allowFrom.telegram`. Reconcile docs if Telegram native session keys are now always agent-scoped.
