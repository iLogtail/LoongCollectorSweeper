## Summary

Keep open. Current main still has the incorrect Plugin SDK migration table entry for `plugin-sdk/allowlist-resolution`, while the actual SDK export contract exposes `mapAllowlistResolutionInputs` from `openclaw/plugin-sdk/allow-from`. The PR is a small, accurate docs fix and is not implemented on main.

## Best Possible Solution

Keep the PR open or land the same docs correction on main: update `docs/plugins/sdk-migration.md` so `mapAllowlistResolutionInputs` is documented under `plugin-sdk/allow-from` and remove the phantom `plugin-sdk/allowlist-resolution` row. A maintainer may also consider whether `docs/plugins/sdk-subpaths.md` should mention the broader allow-from helper set, but that is separate from this PR's narrow migration-table fix.
