## Summary

Keep open. Current `main` has partial reasoning-related mitigations, but I did not find strong evidence that issue #41555 is fully fixed. `/new` still preserves session thinking/reasoning overrides across reset, the Anthropic change is narrow to Opus 4.7 defaults, and the existing Anthropic recovery path only handles invalid replayed thinking blocks, not the reported stuck reasoning-loop behavior.

## Evidence

- **`/new` keeps reasoning toggles across reset:** `performGatewaySessionReset()` clears model/auth overrides and allocates a fresh `sessionId`, but it copies `thinkingLevel` and `reasoningLevel` from the prior session into the new entry. That means reset does not fully clear reasoning-related session state.
