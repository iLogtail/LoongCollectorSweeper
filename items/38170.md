## Summary

Keep PR #38170 open. Current main still lacks Telegram `download-file` support and still does not preserve Telegram `file_id` as a parser-safe media placeholder/history contract. The submitted branch is stale because it targets removed pre-plugin Telegram paths, but the requested behavior remains useful and should be ported into `extensions/telegram`.

## Best Possible Solution

Keep this PR or a refreshed follow-up open, but port the idea to `extensions/telegram`: expose `download-file` from `channel-actions.ts`, implement it in `action-runtime.ts` using the existing Telegram media transport/download helpers, preserve inbound `file_id` in a parser-safe body/history contract, and add focused Telegram tests plus docs updates.
