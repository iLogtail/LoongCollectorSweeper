## Summary

Keep open. Current `main` does not implement the PR’s requested Telegram behavior end to end. Telegram placeholders are still bare `<media:...>` tags on the main path, Telegram does not expose a `download-file` message action, and the current tree has moved Telegram code into `extensions/telegram/*`, so this PR would need a port plus follow-on fixes rather than a conservative close.

## Evidence

There is no `includeFileId` option or embedded `file_id="..."` attribute on current `main`.
