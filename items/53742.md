## Summary

Keep #53742 open. Current main partially fixes the restart/doctor token-drift loop and the special case for `gateway.auth.token` SecretRefs, but the broader install-path concern remains: gateway install still builds a persisted service environment from `~/.openclaw/.env`, config env vars, and auth-profile env refs, then renders those values directly into the LaunchAgent plist `EnvironmentVariables` block.

## Best Possible Solution

Keep the issue open for the remaining install-path work. The maintainer path is to decide whether LaunchAgent installs should load durable secret sources at runtime instead of embedding secret values in `EnvironmentVariables`; if yes, implement that in the gateway install plan/plist path and add macOS-focused tests for env, FILE, and structured SecretRefs. If the intended product contract is that `.env` values are service metadata and may be persisted into launchd/systemd definitions, then update the public docs and close only after explicitly narrowing #53742 to the already-fixed `gateway.auth.token` drift-loop portion and linking narrower follow-ups such as #70752.
