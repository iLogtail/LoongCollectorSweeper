## Summary

Keep open. Current `main` still applies an LLM idle watchdog before the first streamed chunk, so the reported heavy-prefill/local-model failure mode is still plausible by inspection. The exact `~60s` claim looks stale on current code because the default is now 120s, and there is now a config escape hatch (`agents.defaults.llm.idleTimeoutSeconds`), but that is not strong evidence that the underlying first-token timeout problem is fixed.
