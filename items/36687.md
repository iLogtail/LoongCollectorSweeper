## Summary

Keep #36687 open. Current main has guardrails for the WebChat/TUI stale external-route leakage symptom, and the related maintainer PRs/comments support that part as fixed. The stronger claim that the 2026.3.2 update reset, dropped, or ignored an existing isolated `session.dmScope` remains unproven and not fixed by current default behavior: omitted `session.dmScope` still means `main`.

## Best Possible Solution

Keep the issue open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore an existing isolated `session.dmScope`, add a targeted migration or doctor repair plus regression coverage around config preservation. If omitted `dmScope` intentionally remains `main` and no reset-path reproduction appears, close only after explicit maintainer product judgment while retaining the shipped WebChat/UI stale-route guardrails.
