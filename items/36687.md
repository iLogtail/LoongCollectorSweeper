## Summary

Keep #36687 open. Current main contains guardrails and regression coverage for the WebChat/UI cross-channel reply leakage symptom, but the stronger 2026.3.2 upgrade/reset claim remains unresolved: unset session.dmScope still means main in runtime routing, outbound session-key construction, doctor security checks, docs, and schema. Maintainer comments also explicitly left the issue open for evidence on whether the update path changed or lost session.dmScope.

## Best Possible Solution

Keep the issue open until maintainers either verify the 2026.3.2 update/reset path and add a targeted migration/repair/regression test if dmScope can be lost, or explicitly decide that an omitted session.dmScope should continue to mean main and close with that product decision. The existing WebChat/UI stale-route guardrails should remain in place.
