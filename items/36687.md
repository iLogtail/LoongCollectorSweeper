## Summary

Keep open. Current main has routing guardrails/tests for the WebChat/UI cross-channel leakage symptom, but the issue's stronger claim that the 2026.3.2 update/reset path changed `session.dmScope` is still not proven fixed or disproven. Runtime code and docs still treat an unset `session.dmScope` as `main`, while onboarding only writes `per-channel-peer` when the value is unset. The issue is clear and is not older than 60 days, so stale/incoherent/cannot-reproduce closure is not appropriate.
