## Summary

Keep #70578 open. Current main and v2026.4.24 have strong guardrails for the central silent-clobber symptom: source/runtime projection now preserves source-only keys, large destructive writes are rejected before replacement, and last-known-good recovery handles sharp size drops. But the issue also asks for root-schema unknown top-level preservation and bundled CLI allowlist exemptions, and current main still does the opposite in code and tests. Related PR #67557 fixed the gateway restart-loop trigger only, not the broader config round-trip/allowlist behavior.

## Best Possible Solution

Keep the issue open but narrow it. Maintainers should decide whether root-level unknown keys are supposed to be inert source-preserved data or invalid config with doctor migration guidance. If preservation is desired, add a focused regression around `plugins install` and gateway/startup writes with source-only root keys. Separately, either exempt core-owned bundled maintenance command surfaces from `plugins.allow` or split that CLI allowlist behavior into its own issue if the current restriction is intentional.
