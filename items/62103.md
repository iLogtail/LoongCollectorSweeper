## Summary

Keep PR #62103 open. Current main has partial `google-vertex` awareness, but the PR's central behavior is still not implemented on main: `google-vertex` remains a hook alias of the Google AI Studio provider, plugin metadata does not declare it as an owned provider/auth choice, and there is no Google plugin Vertex URL builder or onboarding path that collects location/project and constructs `{region}-aiplatform.googleapis.com` endpoints. The PR also has an unresolved review concern about ADC/OAuth-aware catalog setup, so the best cleanup path is maintainer review or revision, not auto-close.

## Best Possible Solution

Keep the PR open and ask for maintainer review or author revision. The likely product path is a Google-owned `google-vertex` provider surface that is declared in `extensions/google/openclaw.plugin.json`, builds validated Vertex base URLs from project/location, supports ADC/OAuth-aware auth and discovery, and adds focused tests for provider ownership, URL construction, region validation, and catalog/auth discovery.
