## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp/Baileys plugin to Baileys `call` events, and the recent maintainer discussion explicitly says the request is still valid and should remain WhatsApp-plugin-owned.

## Best Possible Solution

Keep the issue open and implement the feature inside the bundled WhatsApp plugin: add an opt-in channel/account setting if desired, subscribe to Baileys `call`, normalize and sanitize the call payload, apply the existing account, DM/group, pairing, and allowlist boundaries, then expose contact activity through a documented hook/session-event/tool surface with focused config, type, docs, and tests. Do not treat this as solved by the PSTN `voice-call` plugin; it is a separate WhatsApp Web event stream.
