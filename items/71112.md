## Summary

Keep open. Current main does not implement the proposed `memory-lancedb` non-vector `ltm query` command: the bundled plugin CLI only registers `list`, `search`, and `stats`, and `MemoryDB` does not expose `getTable()`. The PR is a focused, recent change to an existing bundled memory plugin, with review discussion showing earlier correctness concerns were addressed and only a remaining performance/style concern around full scans.

## Best Possible Solution

Keep the PR open for maintainer review. If the feature is accepted, land it as a memory-lancedb plugin CLI enhancement with targeted tests for filtering, column selection, ordering before limiting, omitted sort columns, invalid filters/limits, plus user-facing docs for `openclaw ltm query` or the memory plugin CLI surface. The remaining full-scan behavior should either be capped, delegated to LanceDB ordering if supported, or explicitly documented before merge.
