## Summary

Keep open. Current `main` does not implement Telegram native command-menu localization or any per-language command-description configuration, and the current config/docs do not provide a supported workaround for localizing built-in `/help`, `/tts`, and similar Telegram menu entries.

## Evidence

- **Built-in Telegram-native command descriptions are hardcoded in English:** The built-in native command catalog defines English descriptions such as `/help -> "Show available commands."` and `/tts -> "Control text-to-speech (TTS)."` with no locale-specific variants.
  - file: [src/auto-reply/commands-registry.shared.ts:131](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/commands-registry.shared.ts#L131)
  - command: `nl -ba src/auto-reply/commands-registry.shared.ts | sed -n '128,330p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Telegram menu registration forwards those descriptions directly to `setMyCommands`:** Telegram command registration builds `{ command, description }` entries from the native command specs, then calls `bot.api.setMyCommands(retryCommands)` without any locale/language-specific options or alternate description source.
  - file: [extensions/telegram/src/bot-native-commands.ts:505](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/telegram/src/bot-native-commands.ts#L505)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '480,610p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Config schema exposes only plain `customCommands`, not localized native-command descriptions:** Telegram config currently supports `commands` and `customCommands?: TelegramCustomCommand[]`, where each custom command has only `command` and `description`. There is no locale/language field for Telegram command-menu descriptions.
  - file: [src/config/types.telegram.ts:103](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/types.telegram.ts#L103)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '90,130p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **No documented config workaround for overriding built-in native command descriptions:** The Telegram docs document startup `setMyCommands` registration and `customCommands`, but not localized native descriptions. The test suite also asserts that when native commands are disabled, registered custom commands still cannot reuse reserved built-in native names, so users cannot replace `/help` or `/tts` menu entries via config alone.
  - file: [extensions/telegram/src/bot.command-menu.test.ts:177](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/telegram/src/bot.command-menu.test.ts#L177)
  - command: `nl -ba extensions/telegram/src/bot.command-menu.test.ts | sed -n '176,214p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
