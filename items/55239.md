## Summary

Keep #55239 open. Current main still registers Telegram native command-menu descriptions from the built-in English command catalog, and the existing `customCommands` config only adds non-conflicting extra menu entries; it cannot localize or override built-in commands like `/help` and `/tts`.

## Best Possible Solution

Keep the issue open and implement this in the bundled Telegram plugin/config surface: add a config-backed way to override or localize native command descriptions, apply it before `syncTelegramMenuCommands`, document the behavior on the public Telegram channel/config pages, and add regression tests showing Chinese descriptions for built-ins such as `/help` and `/tts` while preserving the existing custom-command conflict rules.
