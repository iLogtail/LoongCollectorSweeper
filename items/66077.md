## Summary

Current `main` already fixes the reported CLI/runtime mismatch for Ollama memory embeddings: the deep-status path resolves plugin capability adapters, Ollama still registers its memory embedding adapter, regression coverage explicitly checks `ollama` fallback resolution, and the changelog records the fix as shipped in `2026.4.14`.

## Evidence

- **Current resolver falls back to plugin capability lookup for requested memory embedding providers:** `getMemoryEmbeddingProvider(id, cfg)` now checks the registered adapter first and otherwise calls `resolvePluginCapabilityProvider({ key: "memoryEmbeddingProviders", providerId: id, cfg })`, which matches the failure mode described in the issue comments and fixes the early-miss path for plugin-backed providers like `ollama`.
