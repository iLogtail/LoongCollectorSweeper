## Summary

Keep #61320 open. The related PR #61943 was closed unmerged, and current main still has Moonshot registered without a provider-owned tool-schema normalizer for boolean schemas. The runtime only applies a provider plugin hook when present; existing OpenAI tool compatibility is scoped to OpenAI/OpenAI Codex and does not cover `moonshot`, while core tools still expose native boolean parameters that can reach the Moonshot request path unchanged.

## Best Possible Solution

Keep the issue open and implement a Moonshot-owned tool-schema compatibility hook, or a shared named provider-tool helper if another provider needs the same behavior. The fix should normalize boolean JSON-schema nodes before tool registration and add Moonshot tests for nested properties, arrays/tuples, combinators, passthrough cases, and provider hook wiring. PR #61943 is the right direction but needs to be revived or superseded on current main.
