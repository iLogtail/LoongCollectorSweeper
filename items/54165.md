## Summary

Keep open. Current main still does not implement the requested heartbeat sanitization for uppercase `[TOOL_CALL]` / `[TOOL_RESULT]` bracket blocks, though PR #54165 should be narrowed before merge because it bundles unrelated Telegram `message:sent` streaming hook changes that review comments flagged as risky.

## Best Possible Solution

Keep the item open, but do not merge PR #54165 as-is. The best path is a narrow core/shared sanitizer for uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` blocks on heartbeat user-visible delivery, with focused `buildReplyPayloads` tests for `isHeartbeat=true`; split or drop the unrelated Telegram streaming `message:sent` hook work into a separate PR that resolves the P1 review concerns.
