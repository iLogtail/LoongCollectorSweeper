## Summary

Keep open. Current main still does not add an internal operator.admin gate to /stop, so the underlying bug is not implemented on main. The PR is also not safe as a cleanup close because its approach replaces the existing sender authorization gate instead of layering the internal scope check on top.

## Best Possible Solution

Keep the PR open or replace it with a maintainer patch that preserves rejectUnauthorizedCommand for /stop, adds requireGatewayClientScopeForInternalChannel with allowedScopes ["operator.admin"] as an additional internal-channel gate, and adds focused regression tests for internal operator.write rejection, internal operator.admin success, and external unauthorized sender rejection.
