## Summary

Keep this PR open. Current main still has bare `sessions_send` schema fields without the proposed field descriptions, and the public session-tool doc still lacks the proposed guidance to use either `sessionKey` or `label`, not both. The runtime mutual-exclusion guard already exists, so the PR is a plausible docs/schema clarification rather than obsolete work.

## Best Possible Solution

Keep this PR open or consolidate it with the overlapping `sessions_send` guidance work. The useful path is to add schema field descriptions for `sessionKey`, `label`, and `agentId`, update the session-tool docs with the same target-selection rule, and optionally add a focused schema metadata assertion so the model-facing guidance does not regress.
