## Summary

Keep #7433 open. Current main still lacks the requested Baileys reliability hooks for the bundled WhatsApp plugin: no socket-level `getMessage` or `cachedGroupMetadata`, no group metadata event maintenance, and no `lid-mapping.update` persistence path. The prior maintainer-side Codex comment remains consistent with current `b67d9bf7` source.

## Best Possible Solution

Implement this in the bundled `@openclaw/whatsapp` plugin: add bounded full-message retry storage and pass `getMessage`; share a group metadata cache through `cachedGroupMetadata`; seed and maintain it from connect hydration plus `groups.upsert`, `groups.update`, and `group-participants.update`; then decide whether Baileys `signalRepository.lidMapping` already persists enough LID state or add an explicit `lid-mapping.update` persistence path. Add focused WhatsApp plugin tests for socket options, group cache event updates, retry-message lookup, and LID persistence behavior.
