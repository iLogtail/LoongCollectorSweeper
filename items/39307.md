## Summary

Keep open. Current main has partial compaction surfacing through a transient Control UI indicator, persisted Control UI divider markers, and opt-in chat notices, but it still does not satisfy the issue's requested default user-visible reset signal plus assistant-side disclosure, re-grounding, and reload offer. The related #50790 context is not a duplicate because it covered session-scoped reset/forget controls, while this issue is about mid-thread compaction visibility and recovery behavior.

## Best Possible Solution

Keep #39307 open as the product/runtime tracking item. A complete fix should make compaction/reset events visible by default on supported surfaces with sensible noise controls, preserve or expose a durable context-refresh marker where appropriate, and add assistant-side recovery instructions that disclose the refresh, re-ground before substantive reasoning, and offer to reload active documents/context when accuracy may depend on prior state. Update docs and tests around the chosen default behavior.
