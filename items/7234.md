## Summary

Keep #7234 open. Current main still exposes Discord message gating as one coarse `messages` boolean in config/schema/docs and runtime checks that same gate for read, send, edit, and delete. The prior Codex review comment by steipete remains accurate against current main b67d9bf7f06f4c655b1e0869395056c7d631abc2.

## Best Possible Solution

Keep #7234 open. The best path is a backward-compatible Discord gate design, for example `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level Discord gates, then wire it through config types, zod/generated schema, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. This belongs in the bundled Discord plugin/config surface because the official Discord integration currently exposes and enforces the coarse gate.
