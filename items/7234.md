## Summary

Keep open: current main still implements Discord message gating as one boolean `actions.messages` flag, and the code/docs/tests still treat read, edit, delete, and runtime send behavior as grouped rather than granular sub-actions.

## Evidence

- **Config shape is still a single boolean:** `DiscordActionConfig` exposes `messages?: boolean` and does not define nested `read`, `edit`, `delete`, or `send` sub-gates.
  - file: [src/config/types.discord.ts:96](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/src/config/types.discord.ts#L96)
  - command: `nl -ba src/config/types.discord.ts | sed -n '91,112p'`
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Action discovery still groups read/edit/delete under `messages`:** Discord action discovery always includes `send`, then adds `read`, `edit`, and `delete` together only when `discovery.isEnabled("messages")` is true.
  - file: [extensions/discord/src/channel-actions.ts:79](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/discord/src/channel-actions.ts#L79)
  - command: `nl -ba extensions/discord/src/channel-actions.ts | sed -n '79,104p'`
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Runtime still gates send/read/edit/delete with the same `messages` check:** `fetchMessage`, `readMessages`, `sendMessage`, `editMessage`, and `deleteMessage` each check `isActionEnabled("messages")`, so there is still no separate runtime gate for destructive vs non-destructive message actions.
  - file: [extensions/discord/src/actions/runtime.messaging.ts:274](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/discord/src/actions/runtime.messaging.ts#L274)
  - command: `nl -ba extensions/discord/src/actions/runtime.messaging.ts | sed -n '273,455p'`

- **Current tests still encode the grouped-gate behavior:** The Discord actions contract test sets `messages: false` and expects only `send`, `poll`, `react`, `reactions`, and `emoji-list`, which matches the current all-or-nothing gate rather than granular message permissions.
  - file: [extensions/discord/src/channel-actions.contract.test.ts:20](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/discord/src/channel-actions.contract.test.ts#L20)

  - file: [docs/channels/discord.md:926](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/docs/channels/discord.md#L926)
