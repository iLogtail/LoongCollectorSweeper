## Summary

Keep open. Current main partially addresses the anti-spike problem for heartbeat by using a stable per-agent phase, but the issue asks for broader scheduling controls that are still missing: no prime cadence mode, no exposed heartbeat phase-jitter options, and no cron `every` jitter/stagger support. The existing cron staggering remains limited to `cron` expressions, especially top-of-hour schedules.
