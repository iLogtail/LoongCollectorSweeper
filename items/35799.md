## Summary

Keep #35799 open. Current main partially mitigates the heartbeat herd problem with an internal stable per-agent phase, but the requested product/API surface is still missing: no prime cadence mode, no exposed heartbeat phase-jitter controls, and cron `every` schedules remain fixed intervals without stagger/jitter support. The user comment asking for a stable :55-style heartbeat phase remains a valid use case that is not directly configurable.

## Best Possible Solution

Keep the issue open and split or implement the remaining product surface deliberately: add backward-compatible heartbeat scheduling controls for deterministic/user-selectable phase and optional prime cadence if maintainers want it, extend cron `every` schema/protocol/CLI/UI/status/docs/tests with explicit jitter/stagger semantics, and preserve the existing internal stable heartbeat phase.
