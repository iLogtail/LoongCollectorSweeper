## Summary

Keep open. Current `main` does not contain the PR's chat-channel session-switching feature: there is no `/sessions` chat command, `/session` still only handles `idle` and `max-age`, and the proposed `session.historyLimit` config/schema additions are absent on `main`. The existing `/sessions` mention in TUI docs is a different Gateway/TUI session-picker surface, not this chat-channel feature.
