## Summary

Keep open. Current main does not implement `system.read` / `system.write`, and the standard `read`, `write`, and `edit` tools still route through gateway-local host operations or sandbox filesystem bridges, not a node host when `tools.exec.host=node`. This is an actionable core protocol/tool-routing feature request, not a stale or ClawHub-only cleanup item.

## Best Possible Solution

Keep #61500 open for maintainer design and implementation. The likely path is an additive node protocol extension for structured file operations, advertised by node hosts, covered by gateway/node command policy and docs, plus filesystem-tool routing that either adds an explicit `tools.fs.host`/split mode or reuses the existing workspace boundary so gateway-owned workspace/bootstrap files remain accessible while remote project files can route to the selected node.
