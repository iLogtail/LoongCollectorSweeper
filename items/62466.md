## Summary

Keep open. Current main still delegates out-of-range read offsets directly to @mariozechner/pi-coding-agent, whose pinned 0.70.2 read tool throws `Offset ... is beyond end of file`. The related PR #62491 proposed the right class of fix, but it was closed unmerged, and neither current main nor v2026.4.24 contains equivalent EOF-offset handling or regression coverage.

## Best Possible Solution

Keep #62466 open and implement the fix in `src/agents/pi-tools.read.ts`: catch the pinned upstream EOF-offset error in both explicit-limit and adaptive paging paths, retry with a bounded offset or return accumulated content where appropriate, and add focused regression tests for offset one past EOF, far past EOF, explicit `limit`, and empty-file behavior. The closed unmerged #62491 is useful prior art but not a shipped superseding fix.
