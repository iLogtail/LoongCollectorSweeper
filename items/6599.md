## Summary

Keep open. Current main still has fallback runtime support and `openclaw models status --probe` auth probing, but it does not expose `/models test-fallback` or an equivalent CLI command that intentionally skips the primary model and verifies the configured fallback chain. The related implementation PR #43342 was closed unmerged, and the latest issue discussion already contains a maintainer-side Codex review choosing to keep #6599 open.

## Best Possible Solution

Continue tracking #6599 until maintainers either implement or explicitly decline a first-class fallback-chain probe. A strong implementation would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, add focused slash/CLI tests, and document how it differs from `models status --probe`.
