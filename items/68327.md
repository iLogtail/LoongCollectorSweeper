## Summary

Keep open. Current main has a cleaner secrets-runtime fast path that overlaps with the PR, but the PR is not fully obsolete: when the resolver path is needed, main still calls `loadAuthProfileStoreForSecretsRuntime`, which still overlays external auth profiles through provider plugin resolution. The PR also has unresolved review blockers in the provided discussion, including a Windows shell-argument sanitizer bypass and unconditional production profiling logs, so it should not be merged as-is.

## Best Possible Solution

Keep the item open for maintainer judgment, but do not merge the PR as-is. The best path is to narrow it against current main: either prove and patch the remaining resolver-path external-auth overlay latency with regression coverage, or split the Windows mDNS, async-lock, and UI spawn fixes into separate PRs after addressing the existing security and production-logging review comments.
