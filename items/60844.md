## Summary

Keep open. Current main has not implemented the PR’s central behavior: unset heartbeat `isolatedSession` still reuses the main session, and docs/tests still describe that behavior. The PR is focused and potentially useful, but it needs maintainer review/rebase and likely additional handling around wake/system-event preflight before it can be considered obsolete or safe to close.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a refreshed patch against current main that changes heartbeat isolation semantics consistently across `runHeartbeatOnce`, preflight/wake event inspection, system-event consumption/routing, tests, config type comments, and the current heartbeat/config-agent docs. Add or adjust regression coverage for unset `isolatedSession` with wake/cron/exec queued events before merging.
