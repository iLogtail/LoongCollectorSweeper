## Summary

Current main has substantial env-proxy support, but it is still piecemeal and mode-specific rather than the comprehensive global HTTP proxy layer requested in #43821. The codebase bootstraps an ambient Undici env-proxy dispatcher for CLI startup, yet SSRF-guarded fetches and multiple channel/provider integrations still rely on explicit trusted-proxy modes or subsystem-specific proxy agents. That is not strong enough evidence to close this broad feature request as fully implemented.
