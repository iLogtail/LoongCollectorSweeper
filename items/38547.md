## Summary

Keep #38547 open. Current main still does not implement native remote gateway delegation for `sessions_spawn` or `sessions_send`: the session tool schemas do not accept `gatewayUrl`/`gatewayToken`, the subagent and ACP dispatch paths still call the current gateway through the local `agent` RPC path, and the existing remote gateway override support is limited to separate gateway-style tools. The prior ClawSweeper keep-open comment remains accurate on current main.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation would add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential rules, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tool execution, transcripts, and credentials are handled by the remote gateway without leaking local credentials.
