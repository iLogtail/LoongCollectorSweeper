## Summary

Keep open. Current main has useful adjacent primitives, but it still does not provide a documented, channel-generic coordinator/reporter model where one primary agent owns inbound routing while contributor agents can post/report into the same channel with clean session-context semantics.

## Best Possible Solution

Keep the issue open for explicit maintainer product/API judgment. The best path is either a core/channel-routing extension to model primary responder plus non-intercepting contributors, or a documented plugin API seam that lets a ClawHub plugin implement that safely across channels without bypassing session routing, delivery context, and permissions.

## Evidence

- **Bindings schema has only route/acp ownership, not contributors:** Agent bindings are typed as either normal route bindings or ACP bindings, each with a single agentId. There is no contributor/primary role field in the public config type.
