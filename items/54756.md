## Summary

Current main already fixes the gateway service reinstall/doctor mismatch through a shared index-first gateway entrypoint resolver, shipped in v2026.4.24. However, one explicit part of this PR is still not present: the root-based auto-update fallback in `src/infra/update-startup.ts` still checks `dist/entry.js` before `dist/index.js`, and there is no focused regression test for that path. Conservative cleanup should not close this PR as fully implemented yet.

## Best Possible Solution

Keep the PR open for maintainer judgment, but narrow it: current main already shipped the service reinstall/doctor mismatch fix, so the only useful remaining product/code direction is to make `runAutoUpdateCommand` use the same shared index-first gateway entrypoint candidate helper for its root fallback and add the focused `src/infra/update-startup.test.ts` regression. If maintainers decide spawning `entry.js` for this one-shot auto-update fallback is intentional or harmless, they can close the PR with that explicit rationale.
