## Summary

Keep open. Current main includes the related v2026.3.28 fix for status cards that should use a persisted runtime model when no explicit override exists, but #42690 asks for a different behavior: keep the selected/default model visible and add a distinct active model line for runtime-routed turns. Current code does not render a generic `Active` line, does not read `model-snapshot` custom entries for status, and only persists the final runtime model after the run completes.

## Best Possible Solution

Keep #42690 open. The product fix should pass the current resolved run model into `session_status` or teach status transcript fallback to read the latest `model-snapshot`, then render a separate `↪️ Active: provider/model` line whenever it differs from the selected/default model. Add a regression test for a `before_model_resolve` or equivalent runtime route that calls `session_status` before the final assistant message.

The tool therefore cannot directly report the in-flight routed model for the current run.
