## Summary

Keep #55549 open. Current main still has the race-prone Control UI approval path: rendered buttons do not pass the rendered approval id, and the handler still reads `execApprovalQueue[0]` at click time after a cleanup timer can remove that entry. The PR is still a plausible fix for the linked exec bug, but the current patch needs follow-up because it passes only the id and can misroute plugin approvals if the queue entry was already pruned.

## Best Possible Solution

Keep the PR or a successor open and update the fix to pass render-time approval identity plus kind or resolver method from `renderExecApprovalPrompt` into `handleExecApprovalDecision`. Add a focused UI regression test that renders an exec approval and a plugin approval, clears/prunes `execApprovalQueue` before invoking the button click, and verifies the original id is submitted through the correct resolver method.
