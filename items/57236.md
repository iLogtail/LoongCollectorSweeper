## Summary

Keep open. Current `main` and the latest release still contain the exact unconditional dirty-tree rebuild branch this PR removes, the existing tests still assert that behavior, and the repo's own `gateway:watch` regression checker treats `dirty_watched_tree` rebuilds as an invalid local run. This looks like a still-relevant bugfix PR, not a conservative auto-close candidate.
