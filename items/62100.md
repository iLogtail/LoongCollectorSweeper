## Summary

Keep open. Current main still treats slash-containing refs like `minimax/...` as provider-qualified model refs rather than OpenRouter native model IDs, and the PR has unresolved P1 review findings around provider-qualified ref preservation and allowlist widening.

  - file: [src/agents/model-selection-normalize.ts:87](https://github.com/openclaw/openclaw/blob/1d49b8cdaa4a9c57d44a42505984aa7104fa6953/src/agents/model-selection-normalize.ts#L87)

- **Slash inputs do not infer an OpenRouter provider in allowed-model resolution:** `resolveAllowedModelRefFromAliasIndex` only infers a default provider when the trimmed input does not include `/`; slash inputs keep the caller's default provider before parsing.
