## Summary

Keep PR #62100 open. Current main does not implement the PR's requested behavior for OpenRouter native slash IDs such as `minimax/...` or `qwen/...`, but the submitted patch also has unresolved review findings that could break provider-qualified refs and widen model allowlists. This is still a plausible model-resolution improvement, not a conservative auto-close candidate.

## Best Possible Solution

Keep the PR open for maintainer or contributor follow-up. A safe fix would preserve explicit provider-qualified refs, avoid adding bare model IDs to `allowedKeys`, and add regression tests for both the desired OpenRouter native slash compatibility path and the negative cases where real `qwen` or `minimax` providers remain distinct. If OpenClaw wants this shorthand, docs should also explain when `minimax/...` or `qwen/...` is accepted versus the canonical `openrouter/<provider>/<model>` form.
