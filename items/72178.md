## Summary

Keep open. Current main still lacks the PR's cursor-based `chat.history` contract and UI load-older flow, while the PR discussion contains concrete review feedback that should be resolved rather than auto-closed. The partial `NO_REPLY` leak is partly covered server-side on main, but the pagination work is not implemented or clearly superseded by a merged fix.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up rather than auto-closing it. The best path is to choose a canonical pagination PR or issue, fix cursor semantics around the raw index of the first retained message after sanitization and byte trimming, add a regression test for mid-page sanitized rows, and either split or explicitly justify the separate partial-NO_REPLY UI fallback change.
