## Summary

Keep open. Current `main` still applies `MIN_REFIRE_GAP_MS` only to `cron` schedules, while the `every` scheduling path can still compute a next run at the current completion timestamp. That means the requested behavior is not implemented on `main`, and the report remains plausible rather than obsolete.
