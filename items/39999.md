## Summary

Keep open. Current `main` has post-install verification for global package updates, but it does not implement the PR's requested rollback to `beforeVersion` when post-update doctor/verification fails. The request is still materially unmerged, and the remaining behavior difference is visible in code, docs, and tests.

## Evidence

- **Current package update flow has no rollback branch:** `runPackageInstallUpdate` records `beforeVersion`, runs the global update, re-resolves the package root, runs install verification and `openclaw doctor`, then returns the first failed step. There is no reinstall of the prior version and no conditional rollback path after doctor failure.
