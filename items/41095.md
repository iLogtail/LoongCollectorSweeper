## Summary

Current `main` already implements the underlying request with a different, safer design: isolated single-top-level `$include` writes go directly to the included file, and flattening writes fail closed. This PR is superseded.

## Evidence

- **Single-file include write-through exists on main:** `tryWriteSingleTopLevelIncludeMutation` detects a single changed top-level section backed by a lone `$include` string and writes that section directly to the included file; `replaceConfigFile` and `mutateConfigFile` call it before falling back to root config writes.
  - file: [src/config/mutate.ts:111](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/mutate.ts#L111)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Regression test covers the reported plugins case:** The test creates `openclaw.json` with `plugins: { "$include": "./config/plugins.json5" }`, applies a plugin mutation, asserts the root file still contains the `$include`, and verifies the included `plugins.json5` received the new plugin entry/install data.
  - file: [src/config/mutate.test.ts:140](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/mutate.test.ts#L140)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Flattening protection is enforced:** `preserveUntouchedIncludes` throws when a write would flatten `$include`-owned config, and the write-config test confirms root-include partial writes are rejected instead of rewriting the root file.
  - file: [src/config/io.write-prepare.ts:124](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/io.write-prepare.ts#L124)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Docs and shipped release match the implementation:** Current docs say `plugins install/update/enable/disable/uninstall` write through a single-file `plugins` include and that root/include-array/sibling-override cases fail closed; the same fix is present in release tag `v2026.4.22`.
  - file: [docs/cli/plugins.md:71](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/cli/plugins.md#L71)
