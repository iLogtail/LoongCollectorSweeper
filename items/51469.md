## Summary

Keep open. Current main still only has the earlier 10s pre-auth handshake timeout plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS`; the issue’s remaining request is specifically that 10s is not enough for the reported 12s+ cold-start CLI path and that a persistent `gateway.handshakeTimeoutMs` config should exist. The related PR #51503 proposed that exact 15s/config change, but the provided GitHub context says it was closed unmerged, and current main does not contain the proposed config field or 15s default.
