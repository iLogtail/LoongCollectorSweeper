## Summary

Keep #51469 open. Current main still has a 10s pre-auth gateway handshake default and an env-only server override, while the issue’s remaining report is that 10s is insufficient for observed 12s+ cold-start ESM compilation and asks for a 15s default plus persistent `gateway.handshakeTimeoutMs` config. The matching PR #51503 is closed unmerged in the provided GitHub context, so it does not supersede or ship the requested behavior.

## Best Possible Solution

Keep the issue open and use it as the canonical tracker for the remaining gateway handshake timeout work: decide whether to raise the server pre-auth default to at least 15s, add a validated `gateway.handshakeTimeoutMs` config field, and pass that config into the WebSocket handshake timer while preserving the existing env override and pre-auth abuse budget. #51503 is a useful prior implementation reference, but because it closed unmerged it should be revived or replaced rather than treated as shipped.
