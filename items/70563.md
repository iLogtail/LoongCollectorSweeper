## Summary

Keep #70563 open. Current main still lacks a Kimi Coding/subagent fix for provider requests that require `reasoning_content` on assistant tool-call replay when thinking is enabled. The broader #35023 thinking-block preservation work does not cover Kimi’s required-field behavior, and the existing placeholder fix is provider-scoped to DeepSeek/Venice-style wrappers.

## Best Possible Solution

Keep the issue open and fix it in the Kimi provider or a documented generic provider-compat seam. The likely fix is a Kimi/Moonshot-style replay or payload wrapper that preserves or fills `reasoning_content` on assistant tool-call replay when thinking is enabled, with focused coverage for `sessions_spawn` and tool-call continuation under Kimi thinking. If maintainers choose a shared solution, align it with the open #50178 class of provider-required `reasoning_content` replay gaps.
