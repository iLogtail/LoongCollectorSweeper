## Summary

Keep #70563 open. Current main has generic thinking-block replay work, but I do not see a Kimi Coding fix for the reported provider rejection: the Kimi plugin can enable thinking for `kimi/kimi-code`/upstream `kimi-for-coding`, yet its stream wrapper and replay policy do not add or preserve `reasoning_content` on replayed assistant tool-call messages. Existing `reasoning_content` repair coverage is provider-specific for DeepSeek/Venice, not Kimi/subagent continuation.

## Best Possible Solution

Keep the issue open and implement a Kimi Coding replay/payload fix: when Kimi thinking is enabled and an assistant tool-call turn is replayed, either preserve/map the provider reasoning into `reasoning_content` or add the blank `reasoning_content` placeholder Kimi requires, backed by a regression test for `sessions_spawn`/tool-continuation replay. If upstream Kimi's contract differs from DeepSeek's, document that provider-specific rule in the Kimi/Moonshot docs.
