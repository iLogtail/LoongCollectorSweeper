## Summary

Keep #70563 open. The report is recent and concrete, and current main still does not show a Kimi Coding/Moonshot repair for thinking-enabled assistant tool-call replay when `reasoning_content` is absent. Related #35023 covered latest-turn thinking-block preservation, but this Kimi-specific missing-field path remains unproven and appears unfixed.

## Best Possible Solution

Keep the issue open and add a provider-scoped Kimi Coding/Moonshot replay repair, likely reusing or generalizing the existing DeepSeek placeholder behavior so thinking-enabled assistant tool-call replay includes required `reasoning_content` when absent. Add focused regression coverage for Kimi `sessions_spawn` or follow-up tool-call replay with thinking enabled, plus docs/changelog if the fix changes supported Kimi thinking behavior.
