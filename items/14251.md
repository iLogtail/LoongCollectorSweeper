## Summary

Keep #14251 open. Current main still has macOS gateway lifecycle management wired to per-user LaunchAgent paths/domains only. System-level LaunchDaemon handling is limited to best-effort status/doctor discovery for extra services, and the canonical `/Library/LaunchDaemons/ai.openclaw.gateway.plist` case is not a stop/uninstall path or `--system` lifecycle option.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management becomes launchd-scope-aware. A good fix should model user vs system scope, detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist`, avoid reporting stop/uninstall success while a KeepAlive system daemon remains active, provide exact sudo guidance or explicit `--system` handling for stop/uninstall, and add focused tests/docs for both user LaunchAgents and system LaunchDaemons.
