## Summary

Current `main` now covers the main symptom clusters from #40082: GPT-5/OpenAI Codex planning-only or filler turns are explicitly retried/blocked instead of treated as progress, embedded-run provider failures are surfaced to webchat instead of disappearing behind `surface_error`, Control UI renders those chat errors visibly, and gateway handshake/disconnect diagnostics are much clearer. This looks implemented on `main`, but I could not tie the full composite fix to a released tag with confidence.

GPT-5/OpenAI Codex runs auto-enable the strict-agentic path and explicitly retry/block planning-only or filler turns instead of treating them as progress (`src/agents/execution-contract.ts:46`, `src/agents/pi-embedded-runner/run/incomplete-turn.ts:389`, `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:158`). I could not confirm a released tag for the full composite fix set, so this is being closed as implemented on `main` rather than as a specific shipped release.
