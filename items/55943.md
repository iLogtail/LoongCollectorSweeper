## Summary

Keep open: current `main` still limits the bundled `session-memory` hook to `/new` and `/reset`, and persistent subagent sessions intentionally skip completion-end hooks, so the reported missing memory file behavior remains a real, actionable gap.

## Evidence

- **Bundled session-memory is documented as reset/new-only:** The public hooks docs and bundled hook metadata both scope `session-memory` to `command:new` and `command:reset`, with no subagent-completion event coverage.
  - file: [docs/automation/hooks.md:154](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/automation/hooks.md#L154)
  - command: `nl -ba docs/automation/hooks.md | sed -n '150,176p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Handler hard-gates on reset/new commands:** `saveSessionToMemory` returns unless the event is a `command` with action `new` or `reset`, so a subagent finishing normally cannot reach this hook path.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/hooks/bundled/session-memory/handler.ts#L53)
  - command: `nl -ba src/hooks/bundled/session-memory/handler.ts | sed -n '40,90p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Session-mode subagents intentionally suppress ended-hook emission:** `shouldKeepThreadBindingAfterRun()` returns true for `spawnMode === "session"`, and `shouldEmitEndedHookForRun()` negates that, so persistent subagent sessions do not emit the completion ended-hook path on normal finish.
  - file: [src/agents/subagent-registry.ts:494](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/subagent-registry.ts#L494)
  - command: `nl -ba src/agents/subagent-registry.ts | sed -n '494,535p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Test suite locks in current behavior:** There is an explicit regression test asserting that persistent session-mode runs do not emit `subagent_ended` on completion and leave `endedHookEmittedAt` undefined.
  - file: [src/agents/subagent-registry.steer-restart.test.ts:333](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/subagent-registry.steer-restart.test.ts#L333)
  - command: `nl -ba src/agents/subagent-registry.steer-restart.test.ts | sed -n '328,356p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current docs describe session-mode subagents as persistent sessions:** Subagent docs tell users to use `sessions_spawn` with `thread: true` and `mode: "session"` for persistent thread-bound sessions, which matches the reporter's scenario and makes the missing memory write a meaningful product gap rather than a misunderstanding.
  - file: [docs/tools/subagents.md:62](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/tools/subagents.md#L62)
