## Summary

Current `main` does not implement this. The bundled `session-memory` hook is still scoped only to internal `command:new`/`command:reset` events, while subagent completion goes through separate plugin lifecycle hooks. For persistent `mode: "session"` subagents, current tests explicitly assert that completion does not emit `subagent_ended`, so there is no existing completion path that would create `memory/YYYY-MM-DD.md` when a subagent reaches `done`.
