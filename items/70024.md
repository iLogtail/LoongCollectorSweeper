## Summary

Keep open. Current `main` still has the stop-timeout behavior reported in #70024: the timeout branch leaves `store.tasks`/`store.aborts` intact, records `running: true`, and an existing regression test currently asserts that a restart after timeout is skipped. The linked PR #70056 appears to be the right active fix path, but it is still open/unmerged in the provided GitHub context, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #70024 open and use PR #70056, or an equivalent maintainer-authored patch, to change the timeout path to delete the stale task/abort entries, mark the account `running: false`, set `lastStopAt`, and update the regression test so restart after a stop timeout succeeds.
