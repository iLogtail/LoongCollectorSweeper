## Summary

Keep open. Current main has several duplicate-tool-call mitigations, including failed-assistant replay filtering and generic occurrence-aware ID rewriting, but the bundled Moonshot provider still intentionally opts out of strict replay tool-call-ID sanitization. The reported #51593 symptom is WhatsApp group-chat-specific, uses Moonshot/Kimi, and is not fully covered by the narrower retry-corruption fix in #65924.

## Best Possible Solution

Keep #51593 open until maintainers either reproduce the WhatsApp group-chat Moonshot path on current main or add a targeted regression/fix. The likely maintainer path is a focused replay/request-history test for same-model Moonshot group-chat transcripts with repeated `exec:*` IDs, preserving native Kimi `functions.<name>:<index>` IDs while still preventing duplicate `tool_call_id` values from reaching Moonshot.
