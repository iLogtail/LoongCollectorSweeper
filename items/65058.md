## Summary

Current main still appears to reject the Google Workspace Add-on lifecycle payload variants that PR #65058 targets. The parser only normalizes add-on envelopes with `chat.messagePayload`; upstream Google docs also list `chat.addedToSpacePayload` and `chat.removedFromSpacePayload` as valid Chat add-on payload union members. The PR is still a plausible focused fix, though it should address the review comment and add regression tests before merge.

## Best Possible Solution

Keep the PR open. Ask for a small update that keeps payload selection and event labelling in the same branch, adds focused webhook parser tests for message/added/removed add-on envelopes with body-carried `systemIdToken`, and leaves non-`MESSAGE` lifecycle events acknowledged but ignored downstream. Track the remaining #65007 issues separately.
