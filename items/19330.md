## Summary

Keep this issue open. Current main still saves inbound channel media into the shared `media/inbound` directory, the read-side media reference path still rejects nested inbound paths, and sandbox staging still copies into `media/inbound/*`. The related implementation PR #46162 is closed and unmerged in the provided GitHub context, so the requested per-agent/account isolation has not shipped.

## Best Possible Solution

Keep tracking this as core media-pipeline work. A maintainer-quality fix should add a backwards-compatible account/agent scoped media subdirectory contract, update bundled channel/plugin callers to pass account identity, update `media://inbound` and absolute-path resolution, adjust sandbox staging and cleanup behavior, and add regression coverage plus docs. PR #46162 can be reviewed as prior art, but because it is closed and unmerged, the remaining work should be implemented or replaced on a new branch.
