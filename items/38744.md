## Summary

Keep PR #38744 open. Current main still does not implement ElevenLabs realtime WebSocket STT for the native iOS Talk Mode microphone path or the proposed STT settings. Main does have ElevenLabs Scribe v2 Realtime support, but it is registered as a gateway/plugin realtime transcription provider and wired to Voice Call streaming, not to the iOS app's TalkModeManager. The PR also has unresolved review comments about WebSocket readiness, PTT committed-transcript waiting, and fallback recovery, so it is not conservative to close as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rework it around a robust native iOS Talk Mode realtime STT design or add a gateway/native seam that lets companion apps consume the existing realtime transcription or realtime voice provider surface. Address confirmed WebSocket readiness, committed-transcript waiting for PTT, Apple Speech permission/fallback recovery, and Settings/docs alignment before merge.
