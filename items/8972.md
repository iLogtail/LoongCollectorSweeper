## Summary

Keep #8972 open. Current main still treats Discord guild channel allowlist keys as exact channel id/slug/name matches plus thread-parent fallback and the literal `*` wildcard; there is no glob, prefix, or `ws-*` pattern matching. The category-based alternative is also not implemented in the Discord config type/schema and is already tracked separately by open issue #60582 in the provided context.

## Best Possible Solution

Keep #8972 open until Discord guild channel allowlists support a documented pattern form such as `ws-*`, with matcher precedence, config schema/types/help, docs, status/audit behavior, and regression tests showing dynamically created matching channels are allowed without per-channel config edits. Treat category inheritance as the separate #60582 path unless maintainers choose to combine the designs.
