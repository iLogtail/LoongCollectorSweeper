## Summary

Keep #70363 open. Current main does not contain the run-generation registry, ReplyOperation generation stamp, or block-reply stale-output fence proposed by the draft PR. The linked umbrella issue #70319 is still open, and the PR is a focused partial implementation rather than obsolete or superseded cleanup.

## Best Possible Solution

Keep the draft PR open for maintainer review or supersede it with a maintained implementation that closes the same gap. The best product path is to finish the run-scoped invalidation contract in core auto-reply: add a per-session generation/staleness primitive, wire it through abort and new-message takeover, gate tool dispatch and visible emission sites, and cover it with focused regression tests tied to #70319.
