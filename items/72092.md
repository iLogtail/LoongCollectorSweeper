## Summary

Keep PR #72092 open. Current main still has the bug this PR targets: media image, MiniMax VLM fallback, and audio/video provider execution all require a static API key, while Bedrock `aws-sdk` auth may legitimately resolve credentials at call time with an empty API key. The PR is a focused bug fix for open issue #72031, and the latest PR commit addresses the reviewed empty-key rotation gap.

## Best Possible Solution

Keep PR #72092 open for maintainer review and CI. The best path is to validate or iterate on this focused implementation that carries the `aws-sdk` missing-key allowance through image, MiniMax fallback, and audio/video execution, including the empty-key rotation contract, then merge it to close #72031 if checks pass.
