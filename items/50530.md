## Summary

Current main does not implement the requested Telegram soft-newline collapse. The Telegram send paths preserve single newlines through Markdown-to-IR rendering, chunking, and final send, and the documented `chunkMode="newline"` behavior is paragraph-boundary chunking rather than in-paragraph line wrapping cleanup.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled Telegram plugin formatting/send layer: add an opt-in `channels.telegram.collapseNewlines`-style setting or equivalent formatting option, apply it before Telegram HTML chunking/sending, preserve double-newline paragraph breaks, and avoid changing fenced code blocks, intentional list structure, and blockquotes without regression tests.
