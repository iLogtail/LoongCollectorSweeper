## Summary

Keep #39317 open. Current main still treats an exit-0 direct `codex-cli` JSONL run as successful after parsing stdout, without a semantic failure signal for sandbox-denied/tool/action errors. The prior focused fix PR #39460 was closed unmerged, and the related #66081 work is scoped to final-message artifacts rather than failure classification.

## Best Possible Solution

Keep the issue open and revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied and failed command/tool/action/error/status signals after exit 0, fail before `buildCliRunResult`, add a local `sandbox_denied` failover/status/auth-profile policy if maintainers want that public contract, avoid assistant-message keyword matching, and add focused parser plus runner regression tests for true denials and false-positive discussion cases. Treat #66081 as related final-answer reliability work, not a replacement for semantic failure classification.
