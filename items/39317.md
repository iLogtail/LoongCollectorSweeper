## Summary

Keep #39317 open. Current main still treats a zero-exit codex-cli run as successful once stdout parses, and there is no sandbox_denied detection or failover reason. The proposed fix in #39460 was closed unmerged, while #65074/#66081 are related final-output artifact work and do not supersede failure classification.

## Best Possible Solution

Implement the #39460-style fix on current main: parse structured Codex JSONL for sandbox-denied tool/action/error signals, return a typed parser flag, throw a FailoverError with a new sandbox_denied reason mapped to 403, exclude that reason from auth-profile health, and add regression tests that prove exit-0 sandbox denials fail while normal sandbox discussion text does not false-positive. Keep #65074/#66081 as separate final-output artifact work.
