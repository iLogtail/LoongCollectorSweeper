## Summary

Keep open. Current main partially changed the old behavior by giving subagent/cron sessions a hard-coded reduced allowlist that includes persona files, and lightweight bootstrap context exists for heartbeat/cron paths. But the requested configurable `agents.defaults.bootstrapTier` surface (`minimal` / `standard` / `full`) is still not implemented in source, config schema, types, tests, or docs. The linked implementation PR #22439 is still open/unmerged in the provided GitHub context, and the latest maintainer/Codex issue comment also explicitly kept this item open.

## Best Possible Solution

Keep #22438 open until maintainers either land a schema-backed `agents.defaults.bootstrapTier` design or explicitly decide that the current fixed subagent/cron allowlist plus `lightContext` is the intended product direction. If implemented, the likely scope remains `src/agents/workspace.ts`, `src/agents/bootstrap-files.ts`, `src/config/*agent-defaults*`, generated schema/docs, and focused bootstrap/workspace tests; docs should also be reconciled with current subagent allowlist behavior.
