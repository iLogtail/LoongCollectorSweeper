## Summary

Current `main` has a newer Feishu plugin surface and explicit `feishu`/`lark` domain handling, but repo inspection does not show a direct fix for this reported redirect loop. The default path still targets `https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal`, so there is not strong enough evidence to close this as fixed or obsolete.
