## Summary

Current `main` does not contain this fix. User-visible history and Control UI rendering still sanitize user text with `stripInboundMetadata(...)` only, which strips injected metadata blocks and timestamps but not leading `System (untrusted)` exec-completion lines; session transcript rewrite logic also still only backfills media paths. This PR remains a plausible bug fix rather than something already implemented on `main`.

## Evidence
