## Summary

Keep open. Current main still has the reported failure mode for raw model pricing entries: `estimateUsageCost` defaults missing usage counters to zero, but multiplies by `cost.cacheRead` and `cost.cacheWrite` without defaulting those cost rates. `models.json` pricing is read directly into the cost index without applying config defaults, so a model cost with only `input` and `output` can still make the total `NaN`, return `undefined`, and prevent diagnostics from emitting `openclaw.cost.usd`. The provided GitHub context shows an external fork PR attempt, but there is no corresponding fix in the checked-out main code.

## Best Possible Solution

Keep this issue open and fix the shared pricing path: normalize missing `cost.cacheRead` and `cost.cacheWrite` to `0` when building/resolving model cost configs or inside `estimateUsageCost`, include tiered pricing normalization if applicable, and add regression tests proving input/output cost still estimates and `openclaw.cost.usd` can emit when cache rates are omitted.

If `cost.cacheRead` or `cost.cacheWrite` is absent, the total becomes non-finite and the function returns `undefined`.
