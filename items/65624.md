## Summary

Current main still has the reported Mattermost slash-command behavior: omitted callbackUrl derives an http:// URL, registration proceeds with that URL, and callback authorization checks only token membership before trusting the caller-supplied command trigger. The linked fix PR #65655 remains open, while #65647 and #65648 were closed unmerged, so the bug should stay open until an equivalent fix lands.

## Best Possible Solution

Keep #65624 open and land or adapt the linked #65655 approach: reject non-HTTPS resolved Mattermost native slash callback URLs before registration, preserve token-to-trigger bindings from Mattermost registration through HTTP callback validation, and add regression coverage plus docs for HTTPS fail-closed and cross-trigger replay rejection.
