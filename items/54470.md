## Summary

Keep open. Current main still appears to route `openclaw webhooks gmail setup` through a bare `gcloud` spawn on Windows, while the Windows command shim path only handles selected package-manager shims. The reporter's native PowerShell reproduction remains actionable, and the related generic Windows `.cmd` fixes do not cover this Gmail/gcloud path.

## Best Possible Solution

Keep the issue open. Implement a targeted Windows-safe resolution for the Gmail setup `gcloud` invocations, such as resolving `gcloud.cmd` through the existing shim/wrapper machinery or a dedicated executable resolver, then add regression coverage for `runCommandWithTimeout(["gcloud", ...])` and the Gmail setup auth path under `win32`. After that, verify the patched build on native PowerShell using the reporter's scenario.
