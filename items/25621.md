## Summary

Keep #25621 open. Current main still hard-limits managed Gateway service adapters to darwin, linux, and win32, and `resolveGatewayService()` still throws on unsupported platforms. Both `openclaw status` and `openclaw gateway status` still reach that resolver before their later fallback handling can degrade gracefully, so the OpenBSD report remains actionable. PR #25690 is useful prior art for the requested stub behavior, but it was closed unmerged and current main does not contain that fix.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a graceful unsupported-platform GatewayService adapter/stub or guards read-only service resolver calls so `openclaw status` and `openclaw gateway status` work on OpenBSD and other unsupported platforms. Lifecycle commands can still reject managed service operations, but should include manual foreground/service-wrapper guidance. PR #25690 remains useful prior art, though it was closed unmerged.
