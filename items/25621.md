## Summary

Keep #25621 open. Current main still limits managed Gateway services to macOS, Linux, and Windows, and unsupported platforms still throw from `resolveGatewayService()`. The read-only status paths for both `openclaw status` and `openclaw gateway status` still reach that resolver before they can degrade, so the OpenBSD failure remains actionable. The related PR #25690 is useful prior art, but it was closed unmerged and its stub behavior is not present on main.

## Best Possible Solution

Keep the issue open until OpenClaw adds a graceful unsupported-platform GatewayService adapter or guards read-only status paths so `openclaw status` and `openclaw gateway status` work on OpenBSD and similar platforms. Reuse #25690 as prior art for the stub shape, but add current-main regression coverage for unsupported `process.platform` status behavior and preserve clear failures for managed lifecycle operations.
