## Summary

Keep #70600 open. Current main has the intended preview-only CLI behavior and the main CLI docs are corrected, but the bundled node-connect skill and Android README still tell operators to use `openclaw devices approve --latest` as if it completes approval. PR #70667 directly addresses those stale docs, but it is still open and unmerged.

## Best Possible Solution

Keep #70600 open until PR #70667, or an equivalent docs-only fix, lands on main. The fix should update `skills/node-connect/SKILL.md` and `apps/android/README.md` to make `approve --latest` explicitly preview-only and require `openclaw devices approve <requestId>` for the actual approval; then re-grep markdown for remaining one-step `approve --latest` pairing guidance.
