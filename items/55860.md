## Summary

Current `main` does not match the reported failure mode. Auth-profile resolution for a custom provider happens before any plugin-owner lookup, token profiles still resolve to an API key, and runtime callers keep the original resolved key when `prepareProviderRuntimeAuth()` returns `undefined`. The cited `resolveOwningPluginIdsForProvider()` path is only reached after auth lookup has already failed and is used for missing-auth messaging, not for passing the token into the runtime request.
