## Summary

Keep #2597 open. Current main still has explicit context-visibility surfaces such as `/status`, `session_status`, `/context`, UI metadata, persisted token snapshots, and the community `openclaw-workspace-sync` recovery workaround, but it does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. The latest provided maintainer-facing comment also explicitly kept the issue open for that reason.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest fresh session token snapshot into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests and docs, or a maintainer explicitly decides that `/status`, `session_status`, `/context`, UI warnings, memory flush/context-engine plugins, and community backup/sync workflows are the intended product boundary.
