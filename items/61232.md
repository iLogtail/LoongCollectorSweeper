## Summary

Keep #61232 open. Current main does not implement the requested Android third-party Always/background location path: the app still exposes only Off/While Using, migrates stored `always` to `whileUsing`, lacks the background-location manifest/service declarations, and rejects every background `location.get`. This is also not a clean ClawHub close: current plugin docs expose gateway/plugin node invocation, but not a way for an external plugin to change the native Android manifest, settings UI, foreground-service type, or node-local location handler.

## Best Possible Solution

Keep the PR open for maintainer review as a native Android app feature. If accepted, the implementation should live in the Android app with third-party-flavor gating, Play manifest removal, an explicit Always settings flow, persisted reconciliation for canceled/system-settings grants, foreground-service type gating based on real background capability, and targeted Android tests/docs/changelog. If maintainers decide background location should remain out of core, close it manually as a product/security decision rather than an automated ClawHub cleanup, because current plugin surfaces cannot add the required native Android permission/runtime behavior.
