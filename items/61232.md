## Summary

Keep #61232 open. Current main is still Android foreground-location only, and the PR is a concrete native Android app change rather than something current ClawHub/plugin APIs can provide externally.

## Best Possible Solution

Leave the PR open for maintainer review. If OpenClaw wants Android background location, it belongs in the native Android app surface with manifest, Settings UI, runtime permission, foreground-service, docs, and tests aligned. If maintainers instead want this outside core, the actionable path is an explicit Android/native extension design or a documented decision to reject background location, not an automatic ClawHub close under the current plugin API.
