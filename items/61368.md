## Summary

Keep #61368 open. Current main still only exposes the constrained `agents.files.*` bootstrap-file API, not the requested full `agents.workspace.*` directory/file-management API, and the linked implementation PR #62417 is open/unmerged rather than shipped. This is a real core gateway/control-UI security surface that needs maintainer product and security review, not conservative cleanup.

## Best Possible Solution

Keep #61368 open as the product/API tracking issue until maintainers either merge a reviewed implementation such as #62417 or intentionally reject/narrow the scope. A proper implementation should add typed `agents.workspace.*` protocol schemas, gateway handlers using `fs-safe` boundary checks, method scopes, tests for traversal/symlink/hardlink/size limits, Control UI workspace browsing/editing, and user-facing docs.
