## Summary

Keep #69975 open. Current main still exposes the old cron-only `--tz` help text in both `cron add` and `cron edit`, while current source/tests/docs show `--tz` also applies to offset-less ISO `--at` values. The PR is still relevant, but it needs amendment because its proposed `default: UTC` wording conflicts with current cron behavior: cron schedules without `--tz` use the gateway host timezone.

## Best Possible Solution

Keep the PR open and ask for a small wording amendment in both `src/cli/cron-cli/register.cron-add.ts` and `src/cli/cron-cli/register.cron-edit.ts`: clarify that `--tz` applies to `--cron` expressions and offset-less ISO `--at` datetimes, while avoiding a generic UTC default claim. The help should mirror the existing docs distinction: offset-less `--at` defaults to UTC unless `--tz` is provided; cron without `--tz` uses the gateway host timezone.
