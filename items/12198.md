## Summary

Keep #12198 open. Current main has adjacent safety work, especially the optional Skill Workshop scanner, but it does not implement the requested shared Markdown validation/linting contract for prompt-bearing Markdown. Workspace bootstrap files, extra bootstrap files, discovered SKILL.md files, and native ClawHub skill installs still reach the prompt/read path without a common ruleset such as HTML-comment rejection. The existing maintainer/Codex issue comment also explicitly kept this item open.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-Markdown validation contract with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap hooks, SKILL.md discovery/on-demand loading, native ClawHub skill install/publish policy, and any plugin API needed for third-party enforcement. The existing Skill Workshop scanner can inform the initial rules, but it should be generalized and explicitly cover high-confidence blockers such as HTML comments plus warn/block/customization behavior.
