## Summary

Keep open. Current main and the latest release still render the Control UI refresh button as enabled during active chat-run states; the linked PR #66395 is the right direction but is not on main.

## Best Possible Solution

Keep #65522 open until #66395 or an equivalent main-branch change disables the refresh action whenever `chatSending`, `chatRunId`, or `chatStream` indicate an active turn. Add a focused browser render test for the refresh button disabled state so this guard does not regress.

## Evidence
