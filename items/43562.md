## Summary

Keep open. Current main still hardcodes the first system-prompt identity line as “You are a personal assistant running inside OpenClaw.” for both full and `promptMode=none` prompts. The only adjacent shipped knob is `systemPromptOverride`, which replaces the entire assembled prompt rather than making the preamble configurable or neutral when `SOUL.md` is present. The related PR #44497 directly attempted the requested fix but was closed unmerged, so no canonical shipped fix supersedes this issue.

## Best Possible Solution

Keep the issue open and implement a narrow configurable identity preamble: add typed and schema-backed `agents.defaults.preamble` plus `agents.list[].preamble` or an equivalent neutral SOUL-aware behavior, plumb it through embedded prompt construction and compaction paths, and add focused tests/docs showing default fallback, per-agent precedence, and SOUL.md persona behavior.
