## Summary

Current `main` already contains a shipped fix for the reported `gateway closed (1008): pairing required` loopback approval path. The built-in exec approval flow still uses the backend `gateway-client`, and current code plus regression coverage show loopback backend/shared-auth connections are treated as local and no longer require device pairing.

## Evidence
