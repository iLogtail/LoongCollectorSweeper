## Summary

Keep #72382 open. It is a fresh, concrete bug report against the documented macOS/Linux installer, and current main still has the same observable class of behavior: the npm install path can warn that the npm global bin directory is missing from the user's original PATH while still printing a successful install message, with post-install command verification disabled unless `--verify` is passed.

## Best Possible Solution

Keep the issue open and fix the default installer behavior. The best maintainer direction is to make a successful npm install leave `openclaw` immediately discoverable for the user, or avoid printing unconditional success when the command will not be available in the user's shell. Add focused coverage for npm global prefix/PATH behavior and update the public install troubleshooting docs if the intended workflow requires opening a new shell.
