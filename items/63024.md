## Summary

Keep open. I did not find strong evidence that current `main` already fixes or obsoletes this pnpm global-install failure. The current source still imports `tslog` directly in the logging path, the latest published tarball still contains a root logger chunk importing `tslog`, and the new postpublish checks on `main` only prove manifest/runtime-dependency coverage, not that this specific Node 24 + pnpm resolution error is gone.
