## Summary

Current main has not implemented this PR's localized abort-trigger additions. The core abort path still depends on exact matches in `ABORT_TRIGGERS`, and the current tests/docs only cover the existing abort phrases. The PR remains a plausible focused safety/accessibility improvement, but it needs maintainer review and at least one fix before merge because the current PR diff adds `abandonner` in implementation while the test still expects `avorter`.

## Best Possible Solution

Keep the PR open. The best path is to update the branch so implementation and tests agree on the French trigger (`abandonner` vs `avorter`), review the added words for exact-match false positives across the target languages, and add or update user-facing docs if these new natural-language abort triggers are accepted as supported behavior.
