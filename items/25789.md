## Summary

Keep open. Current main still lacks a user-configurable exec approval timeout: `tools.exec.approvalTimeoutMs` is absent from exported config types, strict schemas, labels/help/docs, runtime config resolution, and the approval request path. Main did raise the fixed default to 30 minutes, but that does not implement the PR’s configurable knob, and related configurable-timeout items remain open/unmerged.

## Best Possible Solution

Keep the PR open for maintainer judgment or supersede it with a canonical configurable-timeout implementation. A complete fix should choose the config shape, update exported types, strict schemas, labels/help/generated metadata/docs, runtime config resolution, exec host registration, gateway/node approval paths, CLI transport timeout handling, and focused tests against the current 30-minute default.
