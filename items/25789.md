## Summary

Keep open. Current main still does not implement a user-configurable exec approval timeout; it only has a fixed 30-minute default. The PR remains potentially useful or should be superseded by a canonical implementation, but there is no merged replacement on main.

## Best Possible Solution

Keep this PR open for maintainer judgment, or supersede it with a canonical configurable-timeout implementation. A complete fix should choose the public config shape, update exported types, strict schemas, labels/help/generated metadata and docs, then thread the value through runtime config resolution, exec host registration, gateway/node approval paths, CLI transport timeout handling, and focused tests against the current 30-minute default.
