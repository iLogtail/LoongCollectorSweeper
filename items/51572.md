## Summary

Keep open. Current main partially improved session lifecycle visibility with plugin `session_end`/`before_reset` hooks for some reset/delete paths, but the requested bundled `session-memory` behavior is still missing for idle/daily rollovers and maintenance pruning. The bundled hook remains registered and guarded for explicit `command:new`/`command:reset` events only, while stale rollover emits plugin lifecycle hooks and pruning deletes/archive entries without firing the internal session-memory hook first. The issue discussion's KeepMyClaw snapshot workaround is relevant external context, but it does not make the OpenClaw core request implemented.

## Evidence
