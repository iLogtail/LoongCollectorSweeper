## Summary

Keep PR #43910 open. Current main still does not implement `memorySearch.remote.encodingFormat` in schema/types/config merge or the active remote embedding request/parse paths, and the PR discussion shows the author addressed real review gaps around base64 decoding and batch propagation in later commits. This is stale/rebase-needed work, not conservative cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review, but require a rebase onto current main and a complete implementation across schema/types/help/docs, `resolveMemorySearchConfig`, shared remote embedding clients, direct response decoding, and provider batch request/output handling. Coordinate it with #41670 so OpenClaw lands one coherent encoding-format design rather than two partial fixes.
