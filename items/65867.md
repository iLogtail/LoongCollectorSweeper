## Summary

Keep #65867 open. Current main has broader shared/UI sanitization in some paths, but the specific regression is not safe to close: the issue discussion identifies #66225 as the active self-closing `<final/>` fix, and the two callsites named there still use narrower `<final>` regexes on main. The latest issue comments also show the prompt-driven Google/Gemini tag behavior is still relevant, so this needs a maintainer fix or an explicit canonical successor before cleanup closure.

## Best Possible Solution

Keep the issue open and have maintainers land or supersede #66225 with a shared final/reasoning-tag parser used by the streaming subscriber and user-facing sanitizer, plus regression tests for `<final/>`, streamed Control UI display, and final history/replay rendering. Close only after the chosen fix is on main and the shipped release or canonical successor is clear.
