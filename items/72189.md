## Summary

Keep #72189 open. Current main still lacks the PR's Google Live tool-response fixes: Google realtime responses omit the function name, the consult tool is not declared NON_BLOCKING, the realtime voice bridge has no continuation option, and Voice Call submits only one final tool result after the handler finishes. Related #60093 confirms broader Gemini Live voice support is already shipped, but this PR addresses a narrower remaining tool-response bug. The PR is recent and the author reports live validation, but the Greptile P1/P2 review remains actionable before merge.

## Best Possible Solution

Keep the PR open and request a narrow revision: include Google FunctionResponse.name, preserve call-id-to-name mapping, mark only openclaw_agent_consult as NON_BLOCKING, add and forward realtime voice continuation options through the SDK/session bridge, send the provisional working response only for that consult tool, schedule the final consult result appropriately, preserve single-result behavior for OpenAI and other providers/tools, and keep focused regression tests plus a maintainer-acceptable validation note.
