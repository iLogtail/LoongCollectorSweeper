## Summary

Keep PR #68528 open. Current main still defaults `resolveChannelGroupRequireMention` to `true` when no explicit group or wildcard `requireMention` is configured, and Telegram falls back to that core helper. The linked issue was closed because an explicit `requireMention: false` workaround exists, but this PR is a distinct default-behavior change for `groupPolicy: "open"`, and that behavior is not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to either land or adapt the core resolver change with the proposed regression tests, or explicitly decide that open groups should still require an explicit `groups` / `groups.*` `requireMention: false` override and close the PR with that rationale.
