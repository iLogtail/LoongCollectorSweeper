## Summary

Keep open. Current main still does not auto-propagate ambient proxy or NODE_OPTIONS state into MCP stdio children, and the PR has specific production confirmation from the filer of #65525/#65526. It should be reviewed or rebased against the newer MCP env safety filter, not closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open and have maintainers review an updated implementation that adds a dedicated MCP stdio child-env resolver: propagate trusted ambient proxy/NO_PROXY and any explicitly approved host TLS/runtime preload state from the gateway, preserve user-configured proxy env precedence, keep #69540's per-server dangerous-env blocking intact, and add focused tests for case precedence, empty-string proxy disables, no-op behavior, and NODE_OPTIONS handling.
