## Summary

Keep open. Current `main` still models gateway token auth as a single shared secret, not a per-token identity map, and the documented multi-user path remains `trusted-proxy` rather than multiple gateway tokens.

## Evidence

- **Gateway auth config is single-token only:** `GatewayAuthConfig` exposes `token?: SecretInput` and has no `tokens` map or per-token identity/scopes structure.
  - file: [src/config/types.gateway.ts:144](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/config/types.gateway.ts#L144)

  - file: [src/config/zod-schema.ts:700](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/config/zod-schema.ts#L700)
