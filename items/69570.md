## Summary

Keep #69570 open. Current main still has per-approval async follow-up behavior for exec approvals, and tests assert that approved or denied approvals start agent follow-up turns. The recent issue comment provides a current deterministic repro and related context; #57568 was closed unmerged and does not supersede this issue.

## Best Possible Solution

Keep this issue as the canonical core bug. A maintainer fix likely belongs in the exec approval follow-up and session/queue control paths: coalesce multiple approval completions for the same logical task/session, add a cancellation token or generation check so /stop/chat.abort can discard pending approval continuations, and ensure new user input can preempt or steer queued exec-event follow-ups. Related #69386 can remain a narrower denial/stop symptom and #72062 can cover wrapper-policy hardening, but neither replaces this core async completion cascade work.
