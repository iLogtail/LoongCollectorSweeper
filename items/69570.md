## Summary

Keep #69570 open. Current main has useful stop/queue cleanup, but it still does not solve the reported approved-exec completion cascade: each approval owns an independent async waiter and completion follow-up, and the follow-up path starts an agent turn keyed per approval rather than batching, cancelling stale completions, or prioritizing later user input. The recent commenter’s deterministic 2026.4.21 repro is also directly relevant and points to this issue as the canonical tracker, while #57568 was closed unmerged and covered a narrower /approve-as-shell loop.

## Best Possible Solution

Keep #69570 as the canonical core bug. The best fix likely belongs in the exec approval/follow-up and session queue boundary: add a session/task-level cancellation or generation marker that /stop and task completion can advance, suppress stale exec completion follow-ups after that marker, optionally batch multiple completions for the same logical turn, and add regression tests for many approved execs plus a mid-cascade /stop or new user message.
