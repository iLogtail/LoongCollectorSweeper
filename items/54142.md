## Summary

Keep #54142 open. Current main partially addresses the report: MiniMax VLM now has an explicit 60s fetch timeout, but the native PDF Anthropic and Google provider fetches still pass only method/headers/body with no `signal` or timeout, and the Ollama pull path uses `fetchWithSsrFGuard` without a per-call `timeoutMs` or `signal`. The related closed issues #54137, #54139, and #54140 appear to be component reports feeding this umbrella, not a canonical open replacement.

## Best Possible Solution

Keep the umbrella issue open and narrow it to the remaining provider calls. Add explicit, tested timeouts to `anthropicAnalyzePdf` and `geminiAnalyzePdf`; then either pass an explicit `timeoutMs`/`signal` through the Ollama pull `fetchWithSsrFGuard` call or document why the guarded/global timeout path is the intended product behavior for streaming model pulls.
