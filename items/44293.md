## Summary

Keep open. Current `main` still routes `pnpm check:docs` through `format:docs:check`, and that script still uses the Unix-only `git ls-files ... | xargs oxfmt` pipeline reported in #44293. The related PRs #44824 and #49913 are open, not merged, so they are candidate fixes rather than evidence that the issue is implemented or superseded.

## Best Possible Solution

Keep #44293 open until a cross-platform implementation lands on `main`. The likely fix belongs in the repo scripts/package command surface: replace `format:docs` and `format:docs:check` with a Node helper that enumerates tracked docs files, invokes `oxfmt` without `xargs`, handles Windows argument length limits, and keeps Linux/macOS behavior unchanged. The open PRs #44824 and #49913 are the relevant candidate implementations to review or consolidate.
