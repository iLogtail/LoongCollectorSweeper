## Summary

Keep open. Current main still wires Discord inbound agent processing only through Carbon message-create events, with reaction/thread/presence listeners alongside it but no messageUpdate or messageDelete listener. The related open #38505 covers only the edit/reprocess half, while #53654 also asks for delete-to-cancel, so this should not be closed as implemented or duplicate.

## Best Possible Solution

Keep #53654 open and implement it in the bundled Discord plugin: add Carbon MessageUpdate and MessageDelete listeners, reuse the existing Discord preflight/mention policy for edited messages, maintain a message-id to session/run mapping for active or recent inbound Discord messages, call sessions.abort/chat.abort for delete-to-cancel, and add focused tests for edit re-dispatch, delete abort, completed-session no-op behavior, mention gating, and dedupe interactions. Coordinate with #38505 so the edit half is not solved twice.
