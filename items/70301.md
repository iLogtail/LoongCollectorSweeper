## Summary

Keep open. Current main has the central `/bot-approve` auth-gating line, but this PR still carries useful unmerged follow-up fixes from review: preserving `/bot-approve ?` help after framework routing and restricting the framework-registered command to QQBot channel context.

## Best Possible Solution

Keep the PR open or port its remaining pieces onto main: retain `requireAuth: true`, add a QQBot-only guard in `registerQQBotFrameworkCommands`, preserve `/bot-approve ?` usage help in the framework path, and keep a focused regression test that proves unauthorized pre-dispatch `/bot-approve off` does not mutate `tools.exec.security` or `tools.exec.ask`.
