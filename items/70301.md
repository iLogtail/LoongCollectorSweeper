## Summary

Keep open. Current main already has the core `/bot-approve` `requireAuth: true` change and a minimal registry test, but PR #70301 still contains useful unmerged hardening from review: the QQBot framework command bridge needs a QQ-only channel guard, and the PR also addresses the `/bot-approve ?` help regression introduced by moving the command to the framework path.

## Best Possible Solution

Keep #70301 open. Rebase it on current main, preserve the unmerged QQ-only guard and `/bot-approve ?` help handling, and add focused regression coverage for non-QQ framework contexts plus unauthorized pre-dispatch non-execution/config non-mutation. If maintainers prefer smaller changes, split the remaining guard/help/test work into a new PR rather than closing this one as fully implemented.
