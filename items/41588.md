## Summary

Current `main` mitigates parts of the report, but it does not implement an activity-aware run timeout. The embedded runner still uses a single wall-clock abort timer for the whole run, so active multi-tool runs are not exempted; only LLM-response idle timeout was added as a separate watchdog. The old silent-failure symptom is improved, and the default timeout is now much higher, but the feature request itself remains unresolved.
