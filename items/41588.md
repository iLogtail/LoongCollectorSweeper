## Summary

Keep #41588 open. Current main mitigates the original 600s default-timeout symptom by raising the default agent runtime cap to 48 hours, adds an LLM-stream idle watchdog, and surfaces explicit timeout errors instead of silence. But the requested activity-aware run timeout remains meaningfully unimplemented: the embedded run still uses a single wall-clock abort timer that is not reset on normal tool/assistant activity, and config-level `agents.defaults.timeoutSeconds: 0` is still rejected.
