## Summary

Keep PR #71860 open. Current main still treats successful exec completions as relayable user-facing events when a delivery target exists, and tests still encode that behavior. The PR is a focused, recently updated fix for a real remaining heartbeat/exec delivery contract gap, with the mixed-batch review concern addressed in the provided PR discussion, so it is not implemented on main and should not be auto-closed as cleanup.

## Best Possible Solution

Keep #71860 open for maintainer review. If the intended contract is that successful structured exec completions are internal-only while failed, signaled, and legacy ambiguous completions remain relayable, refine or land this focused heartbeat implementation with its regression tests, then use it as a concrete slice of the broader #69492 cleanup. If maintainers want successful exec completions to remain user-visible, close with that explicit product rationale instead of treating the PR as stale cleanup.
