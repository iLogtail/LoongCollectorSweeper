## Summary

Keep open. The issue reports a concrete Feishu bundled-plugin UX gap, and current main still handles `card.action.trigger` as a void/deduped dispatch path rather than returning an immediate Feishu callback response with a disabled/transitional card.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin: add an additive `ocf1` disable-state/processing metadata contract, have `card.action.trigger` return an immediate Feishu callback response with a disabled/transitional card and optional toast while dispatching the synthetic command asynchronously, preserve the current duplicate-token protection, and add focused tests for the returned WS/webhook callback payload plus existing quick-action and approval-card regressions.
