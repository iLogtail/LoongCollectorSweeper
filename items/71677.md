## Summary

Keep open. Current main still handles Feishu `card.action.trigger` as a void async callback: it awaits or fire-and-forgets `handleFeishuCardAction`, but never returns a Feishu callback response for an instant card update. The existing implementation only deduplicates repeated tokens and lacks the proposed `ocf1` disable-state/transitional-card contract.

## Best Possible Solution

Keep the issue open and implement the behavior inside the bundled Feishu plugin: extend the typed `ocf1` interaction envelope with an opt-in disable-state or equivalent metadata, have `card.action.trigger` immediately return a Feishu callback response containing a transitional disabled card/toast while dispatch continues asynchronously, preserve duplicate-token protection and retry behavior, and add focused tests for webhook/WebSocket returned callback payloads plus existing quick-action and approval-card regressions.
