## Summary

Keep open. Current `main` still drops outbound session identity from the `message_sending` hook context and still turns a hook-cancelled gateway send into a generic `No delivery result`, so the PR's stated behavior is not implemented on `main` yet.

## Evidence

- **Gateway send still collapses empty outbound results to a generic error:** `sendHandlers.send` calls `deliverOutboundPayloads(...)`, then immediately throws `Error("No delivery result")` when the result list is empty. There is no guarded-cancellation branch that would surface a `message_sending` denial.
  - file: [src/gateway/server-methods/send.ts:531](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/gateway/server-methods/send.ts#L531)
  - command: `nl -ba src/gateway/server-methods/send.ts | sed -n '531,556p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Outbound message_sending hook context still omits agent/session identity:** `applyMessageSendingHook` invokes `runMessageSending(...)` with a context containing only `channelId`, `accountId`, and `conversationId`. No `agentId` or outbound `sessionKey` is passed here.
  - file: [src/infra/outbound/deliver.ts:567](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/infra/outbound/deliver.ts#L567)
