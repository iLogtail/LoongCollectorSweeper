## Summary

Keep #49828 open. Current main still renders the Control UI WebChat session/agent switcher as a header dropdown, and the existing chat sidebar is for tool/markdown side content, not a persistent agent conversation list. Related ClawSweeper reports and the current changelog cover dropdown label/list fixes, not the requested WhatsApp/Telegram-style sidebar UX.

## Best Possible Solution

Keep the issue open as a valid Control UI feature request. The best path is a focused WebChat UI design/implementation in `ui/src/ui/views/chat.ts` and related state helpers that renders configured agents from `agents.list` or existing session data as a persistent conversation list, switches via the existing session-switch path, and adds regression tests for multi-agent rendering, active selection, missing session rows, and optional preview/history behavior. Treat #48666 as related dropdown completeness work, not as the canonical replacement for this sidebar request.
