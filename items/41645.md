## Summary

Keep open. Current `main` has session-scoped usage tracking and a usage dashboard, but the data model, server aggregation, docs, and UI filters do not implement per-user token accounting or user-based daily/weekly/monthly summaries.

## Evidence

- **Docs describe session-local usage surfaces:** `/status` shows the current session model/context usage, `/usage off|tokens|full` is per-response and persists per session, and `/usage cost` is a local cost summary from session logs. No per-user reporting surface is documented.
  - file: [docs/reference/token-use.md:66](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/docs/reference/token-use.md#L66)
  - command: `nl -ba docs/reference/token-use.md | sed -n '60,110p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Shared usage result schema has no user aggregate:** The usage API returns sessions plus aggregates by model, provider, agent, and channel. There is no `byUser` or equivalent user-dimension aggregate in the current contract.
  - file: [src/shared/usage-types.ts:13](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/shared/usage-types.ts#L13)
  - command: `nl -ba src/shared/usage-types.ts | sed -n '1,120p'`
  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Gateway usage aggregation only rolls up agent/channel/model/provider:** `sessions.usage` merges totals into `byAgentMap`, `byChannelMap`, `byModelMap`, and `byProviderMap`, then returns those aggregates. It does not build a per-user rollup even though session origin metadata exists.
  - file: [src/gateway/server-methods/usage.ts:662](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/gateway/server-methods/usage.ts#L662)

  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
- **Transcript summarization counts roles, not distinct users:** Session cost usage summarization increments generic `user` and `assistant` message counters and daily totals, but does not extract or aggregate a sender/user identifier for billing-style per-user stats.
  - file: [src/infra/session-cost-usage.ts:576](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/infra/session-cost-usage.ts#L576)

  - sha: [bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b17d180e4bfc28973a7e4fd1646a93)
