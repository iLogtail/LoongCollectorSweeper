## Summary

Keep open. Current main has substantial usage tracking, including session-level token/cost summaries, daily breakdowns, provider quota surfaces, and Control UI usage views, but it does not implement the issue's central request: aggregating input/output/total token usage by user for multi-user billing/quota management.

## Best Possible Solution

Keep the issue open and implement it as an extension of the existing usage pipeline: derive a stable user/sender identity from session origin/account/sender data and identityLinks, add byUser/bySender aggregates with input/output/total/cost and daily/weekly/monthly buckets to sessions.usage, add Control UI filters/columns/charts and /usage cost output as appropriate, and cover it with backend aggregation tests plus Usage UI query/filter tests.
