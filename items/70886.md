## Summary

Keep #70886 open. Current main still routes ZAI image understanding through the configured provider/model runtime without a ZAI coding-endpoint-to-general-endpoint override for vision models. The linked PR #70915 appears to target the right remaining bug, while #70908 was closed unmerged, so the issue should remain the canonical tracker until a fix lands on main.

## Best Possible Solution

Keep #70886 open as the canonical bug. Land a focused ZAI fix on main that routes vision models such as `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo` to the matching non-coding ZAI endpoint when the provider is configured with a Coding Plan endpoint, preferably with regression tests for both `zai-coding-global` and `zai-coding-cn` plus image-runtime coverage. Once that ships, close the issue and mark redundant PRs accordingly.
