## Summary

Keep #70886 open. Current main still routes ZAI image understanding through the generic media-understanding helper using the resolved model/baseUrl, but the bundled ZAI catalog/onboarding code does not add a per-model non-coding endpoint override for vision models when the provider is configured with `zai-coding-global` or `zai-coding-cn`. The related fix attempts are useful evidence, but #70908 is closed unmerged and #70915 remains open/unmerged, so the reported bug is not implemented on main.

## Best Possible Solution

Keep the issue open and fix it in the ZAI plugin boundary, preferably by making ZAI onboarding/catalog generation assign same-origin non-coding `baseUrl` overrides to vision models (`glm-4.6v`, `glm-4.5v`, `glm-5v-turbo`) when the selected provider endpoint is `zai-coding-global` or `zai-coding-cn`. Add regression coverage in `extensions/zai/onboard.test.ts` or `extensions/zai/model-definitions.test.ts` proving coding endpoint setup keeps text/coding models on the coding URL while routing vision catalog entries to `https://api.z.ai/api/paas/v4` or `https://open.bigmodel.cn/api/paas/v4`. Use #70915/#70908 as prior art, but do not close #70886 until a fix lands on main.
