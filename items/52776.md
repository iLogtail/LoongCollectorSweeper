## Summary

Keep this PR open. Current main partially implements the login-gate visibility localization, but the PR’s central shell work is still missing: the command-palette trigger, shortcut hint, docs tooltip, update banner, theme mode labels, mobile chat settings label, and gateway status prefix still contain hard-coded English on `origin/main`. The PR likely needs rebase/rework for the current generated Control UI locale pipeline rather than closure.

## Best Possible Solution

Keep the PR open and ask for a focused rebase against current main: retain the missing shell/topbar/update/status/theme shortcut localization, drop or reconcile the already-present login-gate hunks, add the English source keys in `ui/src/i18n/locales/en.ts`, regenerate non-English locale output via the current Control UI i18n pipeline, and keep/add tests for zh-CN shell labels plus platform-specific command-palette shortcut behavior.
