## Summary

Keep #56876 open. Current main has MIME helpers, an explicit PDF analysis tool, and a bundled PDF document-extract plugin, but the core `read` tool still only special-cases images and otherwise reads arbitrary non-image files as UTF-8 text. The issue asks for binary document detection/preprocessing in the read path for `.docx`, `.xlsx`, `.pdf`, and `.pptx`; that central behavior is not implemented on main, and the referenced attachment PR was closed/unmerged and does not solve read-tool preprocessing.

## Best Possible Solution

Keep the issue open and implement a narrow core read-tool guard: detect MIME from magic bytes and extension before converting non-image files to UTF-8, return a concise binary/document notice for unsupported types, and route supported document types through a documented extractor seam where available. Start with PDF via the existing document-extractor path, then add or explicitly defer DOCX/XLSX/PPTX support with tests for binary suppression, extraction success, unsupported binary fallback, size limits, and sandbox reads.
