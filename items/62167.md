## Summary

Keep #62167 open. Current main has adjacent usage visibility in the Usage tab, `/status`/`/usage`, a conditional context-pressure notice, and collapsed per-message metadata, but it does not implement the requested persistent chat-header quota widget or the requested OpenRouter, LLMAPI, and Ollama quota/unlimited provider handling. Related #13670 is adjacent but still open and narrower, so it does not fully supersede this issue.

## Best Possible Solution

Keep #62167 open as the broader Control UI chat quota-widget request. A good implementation should add a focused chat-header/collapsible usage panel backed by existing session usage data plus provider usage snapshots, extend or document provider hooks for OpenRouter credits and local/unlimited providers such as Ollama/LLMAPI, preserve the existing per-message metadata and Usage tab instead of duplicating them, and add focused UI tests plus docs if the feature ships.
