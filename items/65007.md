## Summary

Keep open. Current main still contains the reported Google Chat failure modes: add-on parsing only handles messagePayload, the wildcard routed-space case preserves an empty sender allowlist, and reply delivery can still pass a message resource as the Google Chat thread. The related PR #65058 is open and explicitly only partially addresses #65007.

## Best Possible Solution

Keep #65007 open. The best path is a Google Chat plugin fix that adds add-on lifecycle payload parsing with tests, resolves or deliberately redocuments the wildcard routed-space sender-policy behavior, and normalizes reply threading so Google Chat API sends receive a spaces/.../threads/... resource. PR #65058 can be updated or followed for the first part, but the sender-policy and reply-thread failures still need maintainer-visible fixes.
