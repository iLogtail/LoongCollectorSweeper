## Summary

Keep #53198 open. Current main still does not implement the documented Discord elevated fallback end to end: runtime elevated auth only consults a channel plugin `elevated.allowFromFallback` hook, the Discord plugin does not provide that hook, and `sandbox explain` still reads only `tools.elevated.allowFrom.<channel>`. The related #53259 PR appears to track the right fix but is open/unmerged in the provided context, so the user-visible bug is not shipped as fixed.

## Best Possible Solution

Keep #53198 open and use #53259, or an equivalent maintainer patch, to add a Discord-owned elevated `allowFromFallback` that resolves `channels.discord.allowFrom` through the same account-aware normalization as Discord command access. `sandbox explain` should share the runtime resolution logic or the same plugin fallback so diagnostics match runtime behavior. Add regression tests for the omitted `tools.elevated.allowFrom.discord` fallback, explicit empty override behavior, and the `allowedByConfig` diagnostic result.
