## Summary

Keep open. Current `main` does not already contain the PR’s vLLM-specific lifecycle management, multi-endpoint handling, or stale-default cleanup. The shipped vLLM plugin still routes setup through the generic single-endpoint self-hosted flow and docs still describe only one explicit `models.providers.vllm` entry.

No vLLM-specific management flow is wired on `main`.
