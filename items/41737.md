## Summary

Keep #41737 open. Current main still has only single-endpoint vLLM setup through the generic self-hosted provider flow; the requested endpoint lifecycle management, multi-endpoint keys, multi-model selection, and stale vLLM default cleanup are not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review or author follow-up. The useful path is to rebase and narrow it to vLLM/provider setup behavior, add focused tests for endpoint add/update/delete, vllm/vllm-2 key handling, model discovery/multi-select, and stale default/fallback cleanup, and split unrelated CI/build/test-infra edits into separate work.
