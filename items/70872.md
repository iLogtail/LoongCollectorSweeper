## Summary

Keep #70872 open. Current main still forwards bundled MCP tool input unchanged from the materialized agent tool into the session MCP runtime, and the runtime only checks that the top-level input is an object before sending it as MCP `arguments`. That means `{ params: "{}" }` remains string-valued at the property level. The exact proposed fix exists in open, unmerged PR #70882, but it has not landed on main.

## Best Possible Solution

Keep the issue open until a hardened version of PR #70882 or equivalent lands on main. The fix likely belongs in `src/agents/pi-bundle-mcp-materialize.ts`, using the MCP tool `inputSchema` to coerce stringified top-level object/array properties before `runtime.callTool`, with regression tests in `src/agents/pi-bundle-mcp-tools.materialize.test.ts` and attention to the PR review notes around unsafe keys, input size limits, and JSON Schema array-form `type`.
