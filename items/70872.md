## Summary

Keep open. Current `main` still forwards bundle-MCP tool arguments without schema-based coercion, so the reported `params: must be object` failure remains plausible, and linked PR #70882 appears to add the missing fix rather than proving it is already shipped.

## Evidence

- **Materialized MCP tools pass model input through unchanged:** `materializeBundleMcpToolsForRun()` exposes each MCP tool with `execute(..., input)` and calls `params.runtime.callTool(tool.serverName, tool.toolName, input)` directly. There is no pre-call coercion for stringified nested object/array properties such as `params: "{}"`.
  - file: [src/agents/pi-bundle-mcp-materialize.ts:109](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/agents/pi-bundle-mcp-materialize.ts#L109)
  - command: `nl -ba src/agents/pi-bundle-mcp-materialize.ts | sed -n '70,130p'`
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Runtime only enforces top-level record shape:** `createSessionMcpRuntime().callTool()` passes `arguments: isMcpConfigRecord(input) ? input : {}` into the MCP client. That accepts any top-level object but does not repair nested schema-mismatched strings, so `{ params: "{}" }` would still be forwarded as a string value.
  - file: [src/agents/pi-bundle-mcp-runtime.ts:351](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/agents/pi-bundle-mcp-runtime.ts#L351)
  - command: `nl -ba src/agents/pi-bundle-mcp-runtime.ts | sed -n '340,375p'`
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Current main test coverage does not cover the reported regression:** Existing `pi-bundle-mcp-tools.materialize` tests exercise normal execution and sorted tool registration, but the current file has no assertion covering coercion of stringified nested object/array params before MCP calls.
  - file: [src/agents/pi-bundle-mcp-tools.materialize.test.ts:55](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/agents/pi-bundle-mcp-tools.materialize.test.ts#L55)

- **Linked PR targets this exact gap and is not on main yet:** The linked PR patch is titled `fix(bundle-mcp): coerce stringified object/array params before MCP tool calls` and adds `coerceStringifiedObjectProperties()` plus regression tests for issue #70872, which is consistent with the missing logic in current `main`.
