## Summary

Keep open. The report is recent, concrete, and current main still forwards bundled MCP tool input without schema-aware coercion of nested object/array properties such as `params: "{}"`. Open PR #70882 proposes the exact missing coercion path, but it is still open and unmerged in the provided context, so #70872 remains the issue-level tracker until a fix lands.

## Best Possible Solution

Keep #70872 open and land or revise PR #70882 with a focused regression test proving `{ params: "{}" }` and stringified arrays are converted before `SessionMcpRuntime.callTool`, while already-correct objects/arrays and ordinary strings remain unchanged.
