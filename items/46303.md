## Summary

Keep PR #46303 open. Current main still does not implement the PR's central behavior: globally flushing inbound debounce buffers and waiting for all followup queues during SIGUSR1 restart. Related shipped restart work covers active task/subagent deferral and orphan recovery, but not the closure-local debounce timers or all-queue drain requested here.

## Best Possible Solution

Keep the PR open. The best path is a smaller additive core fix that registers active inbound debouncers without breaking existing SDK signatures, performs a bounded restart-time debounce flush, waits for followup queues using the same pending predicate as the drain loop after active work can complete, wires debouncer lifecycle cleanup across channel teardown paths, and covers the SIGUSR1 ordering with focused tests.
