## Summary

Keep open. Current main still lacks the PR's central behavior: no process-wide inbound debouncer flush, no all-followup-queues drain waiter, and the SIGUSR1 restart path only drains active tasks/runs before closing the server. The PR is an active, focused bug-fix attempt with real review discussion; it is not implemented, duplicated, or obsolete enough for conservative cleanup closure.

## Best Possible Solution

Keep #46303 open, but require a narrower maintainer-reviewed implementation: preserve the public SDK contract or version it deliberately, add a restart-time inbound debounce flush capability without a mutable public global, wait for followup queues after active turns can actually drain, account for `droppedCount` summary work, and cover SIGUSR1 ordering plus channel teardown races with focused tests.
