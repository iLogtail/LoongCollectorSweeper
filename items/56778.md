## Summary

Keep #56778 open. Current main still has no dedicated `openclaw cron list --agent <id>` filter and the gateway/tool path does not auto-scope `cron.list` to the requesting agent. The closest existing behavior is a generic text `query` filter that includes `job.agentId`, but that is not equivalent to exact agent isolation and does not hide unset/default-agent jobs from non-default agents.

## Best Possible Solution

Implement this in core, not ClawHub: add an exact `agentId` or `agent` filter to `CronListParamsSchema`/types, wire `openclaw cron list --agent <id>` through the CLI, filter in `src/cron/service/ops.ts`, and update the agent tool/gateway path so calls made from an agent session default to that session's agent scope unless an explicit admin/global mode is requested. Define and test the default-agent rule for jobs with unset `agentId`, then document the new list behavior on the cron CLI page.
