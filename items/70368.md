## Summary

Keep open. Current main still emits `gateway.trusted_proxies_missing` only for literal `gateway.bind="loopback"` and hard-codes it as `warn`; the existing test and audit-check docs also still encode that behavior. The related duplicate PR #70377 is closed unmerged and the linked issue #70357 was closed as superseded by this active PR, not as implemented. The PR also needs maintainer/code follow-up before merge because the latest review discussion raises real effective-bind classification concerns, especially around `auto`.

## Best Possible Solution

Keep #70368 open and request a corrected implementation. The safest path is to downgrade only definitely local-only exposure, keep/fail-closed warnings for ambiguous `auto` or exposed effective binds, preserve coherent sibling checks (`bind_no_auth`, `loopback_no_auth`, Control UI origin checks), and add regression coverage for plain loopback, `lan`, custom loopback, exposed custom, and `auto`/container or fallback behavior. Update the audit-check docs if the severity taxonomy changes.
