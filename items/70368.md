## Summary

Keep #70368 open. Current main still has the reported behavior: loopback Control UI with empty `gateway.trustedProxies` emits `gateway.trusted_proxies_missing` at `warn`, and the current test/docs still encode that severity. The PR is also not safe to close as implemented or superseded, because the linked issue #70357 remains open and the related duplicate PR #70377 was closed unmerged. The PR head still needs maintainer/reviewer handling because later review comments identified real false-negative risks around classifying `auto`/`custom` binds as loopback without matching the runtime bind fallback behavior.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-reviewed patch that solves #70357 without audit false negatives: derive severity from the effective bind outcome where feasible, fail closed when bind resolution is unknown or environment-dependent, keep `warn` for non-loopback exposure, and add focused tests for loopback, lan/tailnet, custom loopback with bind failure, and auto/container behavior. The docs reference should be updated only after the final severity taxonomy is settled.
