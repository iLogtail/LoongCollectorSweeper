## Summary

Keep #72218 open. Current main still has the heartbeat final-payload bypass in `buildReplyPayloads`, so an `isHeartbeat: true` final payload containing only `HEARTBEAT_OK` is not run through the shared token-stripping path. The PR remains a focused fix for a live source/test gap; adjacent heartbeat runner and followup-delivery filtering narrow some delivery paths but do not make this payload-builder change obsolete.

## Best Possible Solution

Keep #72218 open for maintainer review or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts`: apply `HEARTBEAT_OK` stripping to heartbeat final payloads, preserve media-bearing payloads after stripping, keep Bun socket error formatting scoped to non-heartbeat runs, and add focused `isHeartbeat: true` regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #72253 and #69492 as separate broader exec/system-event leakage work.
