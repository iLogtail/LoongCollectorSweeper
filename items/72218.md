## Summary

Keep #72218 open. Current main at 637bd33e69b2117331dafe92dfaed831fb684654 still has the heartbeat-specific payload sanitation bypass that this PR removes, while docs say heartbeat `HEARTBEAT_OK` acknowledgments are stripped and dropped by default. The related open items cover broader exec/system-event leakage, not this narrow final-payload helper gap.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent narrow fix in `src/auto-reply/reply/agent-runner-payloads.ts` that applies heartbeat token stripping to heartbeat final payloads while preserving the `isHeartbeat` guard around Bun socket error formatting. Keep the focused regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`; treat broader async exec/system-event leakage as separate work under #72253/#69492.
