## Summary

Keep #72218 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still has the heartbeat-specific early return in `buildReplyPayloads`, so heartbeat final payloads do not use the same `HEARTBEAT_OK` stripping branch as non-heartbeat final payloads. The docs say heartbeat OK acknowledgments are suppressed by default, and current tests do not include the PR's focused `isHeartbeat: true` final-payload sanitation cases. The linked/adjacent issues and PRs (#71860, #72253, #69492) cover broader exec/system-event leakage, not this exact payload-builder gap.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent narrow fix in `src/auto-reply/reply/agent-runner-payloads.ts` that applies heartbeat token sanitation to heartbeat final payloads while preserving the `isHeartbeat` guard around Bun socket error formatting. Keep the focused regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`, and treat broader exec/system-event leakage as separate work tracked by #69492/#72253.
