## Summary

Current main does not fully eliminate the reported chain. The spawned host exec command now uses a non-login shell and receives `OPENCLAW_SHELL=exec`, but `host=gateway` still runs a login-shell PATH probe before execution, and `openclaw gateway start` still routes an already installed/loaded service through restart. The linked #62086 appears related to a separate SIGTERM path, not a canonical replacement for this profile/systemd restart report.

## Best Possible Solution

Keep #60087 open. The best fix is to make `openclaw gateway start` idempotent when the managed service is already active/running, and also remove or harden the exec PATH login-shell probe so it cannot trigger user startup autostart commands. Add regression coverage for `host=gateway` exec with shell-startup side effects and for Linux systemd `gateway start` not restarting an already running gateway.
