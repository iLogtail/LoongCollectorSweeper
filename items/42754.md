## Summary

Keep #42754 open. Current main still drops Feishu card callback fields outside `action.value`: the typed event only includes `value` and `tag`, the production parser reconstructs `action` with only those two fields, and the fallback path serializes only `action.value`. The linked PR #42908 is useful and directly related, but it is open/unmerged and needs current-main parser-to-handler coverage before this issue can be treated as implemented or obsolete.

## Best Possible Solution

Keep the issue open and land one coordinated Feishu plugin fix, likely through #42908/#43953/#43202: preserve extra action fields in `parseFeishuCardActionEventPayload`, extend the typed event/fallback serialization to include `option`, `options`, and `form_value` without breaking existing button command/text flows, and add an ingress-path regression test starting from a raw `card.action.trigger` payload.

- A handler-only fix would still miss real Feishu callbacks unless `parseFeishuCardActionEventPayload` preserves the sibling action fields from the raw payload.
