## Summary

Keep PR #69199 open. Current main still routes memory_search unavailable payloads through the generic embedding/provider warning/action when the underlying error is a missing node:sqlite runtime, and current tests do not lock in the sqlite-missing case. The PR is a narrow, active respin of the UX fix for closed issue #59457 and is not superseded by the separate open fallback-search feature #59451.

## Best Possible Solution

Keep PR #69199 open for maintainer review or merge as the narrow UX fix. The best path is to land a targeted sqlite-missing branch in buildMemorySearchUnavailableResult with the accompanying unit test, while leaving the broader file-based fallback behavior tracked separately in #59451.
