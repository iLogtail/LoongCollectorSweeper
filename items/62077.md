## Summary

Keep #62077 open. Current main still emits full user-role inbound metadata for direct external-channel messages, including Telegram DMs, and there is no `envelopeMode` or equivalent minimal-DM config surface. The earlier cache-stability work keeps volatile metadata out of the system prompt, but it does not solve the repeated per-turn user prefix described in this issue.

## Best Possible Solution

Keep the issue open for a core prompt/envelope policy change. A good implementation would add a documented config contract for inbound metadata verbosity, likely under `agents.defaults` with any intended per-agent override, thread it through prompt assembly, and update `buildInboundUserContextPrefix` plus tests so Telegram and other direct external channels can use compact per-turn metadata while preserving safety-critical untrusted blocks for groups, replies, forwards, locations, and history.
