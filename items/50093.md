## Summary

Keep #50093 open. Current main has adjacent WhatsApp reconnect hardening and a narrow Baileys `append` upsert path, but it does not implement the requested per-chat/cursor backfill after reconnect. In fact, current tests assert that older history/offline append messages are skipped, so the reported multi-minute or multi-hour outage window remains unsolved.

## Best Possible Solution

Keep the issue open for a WhatsApp-owned design decision. The best path is to either add a bounded, deduped WhatsApp reconnect catchup mechanism if Baileys exposes a reliable history/sync primitive, or document and implement gap-detected warnings when provider history replay cannot be guaranteed. The BlueBubbles catchup implementation is a good shape to mirror where WhatsApp's dependency contract permits it.

## Evidence

- The remaining implementation depends on WhatsApp/Baileys capabilities; if no reliable history query is available, a useful partial solution may be bounded gap detection plus operator warnings rather than full replay.
