## Summary

Keep #70253 open. Current main still contains the Mattermost inbound streaming config mismatch, and the channel-level A2A behavior described in the discussion is not implemented for Mattermost routing.

## Best Possible Solution

Keep the issue open, fix the Mattermost inbound reply path to honor the resolved account block-streaming setting, and separately make a product/API decision for channel-level bot-authored/A2A ingress before adding Mattermost `allowBots` or broader channel fan-out semantics.
