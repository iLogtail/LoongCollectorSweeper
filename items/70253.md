## Summary

Keep #70253 open. Current main still matches both reported problem areas: the main Mattermost inbound reply path wires draft preview streaming and hard-disables block streaming instead of honoring the account setting, and Mattermost/channel routing still has no channel-level bot-to-bot/A2A delivery policy comparable to other channels' allowBots support.

## Best Possible Solution

Keep the issue open. The best maintainer path is to first fix the Mattermost inbound reply path so preview streaming and block streaming respect channels.mattermost.streaming and account blockStreaming settings, with a focused regression test. Separately define the product/API contract for Mattermost bot-message intake or channel-level agent-to-agent routing before changing mention/policy gates.
