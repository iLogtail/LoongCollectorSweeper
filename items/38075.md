## Summary

Current `main` does not contain this fix. Delivery-mirror transcript entries are still written to session transcripts, and the `chat.history` sanitization path still only drops commentary/suppressed-control assistant messages, not `model: "delivery-mirror"` entries. The web UI also only client-filters silent replies and synthetic transcript-repair tool results, so this PR still addresses a real gap rather than something already implemented on `main`.
