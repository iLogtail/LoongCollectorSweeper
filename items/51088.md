## Summary

Keep #51088 open. Current main still lacks native acknowledgement behavior for the legacy `imessage` channel: its schema and monitor path do not expose `ackReactionScope`, read-receipt, or tapback acknowledgement handling. BlueBubbles already covers read receipts and tapbacks for the recommended iMessage path, and the existing `message_received` / `message:received` hooks are useful workaround surfaces, but they do not implement the requested native legacy iMessage acknowledgement parity.

## Best Possible Solution

Keep this open as a channel-owned feature request. The best implementation path is either to add plugin-local legacy iMessage acknowledgement support in `extensions/imessage` using a reliable `imsg`/macOS API for read receipts or tapbacks, wired through `messages.ackReactionScope` semantics and tests, or make an explicit maintainer product decision to document BlueBubbles plus `message:received` hook packs as the supported acknowledgement path for iMessage and not extend the legacy `imsg` channel.
