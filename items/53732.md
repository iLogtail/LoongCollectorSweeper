## Summary

Keep open. Current main does not implement either requested mitigation: the boot-md path still embeds raw BOOT.md contents in the agent prompt, and the model fallback retry path preserves the same prompt body rather than adding boot-specific echo detection or suppression. The related fallback/model-status items in the provided context do not supersede this specific BOOT.md leakage report.

## Best Possible Solution

Keep #53732 open and fix it in the boot-md/agent fallback boundary. A conservative implementation would mark boot-md runs as internal/silent, add a deterministic guard that detects substantial literal echoes of the generated BOOT.md prompt before final delivery or message-tool sends, and add regression tests where the primary fails and the fallback returns the boot prompt verbatim.
