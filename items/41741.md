## Summary

Current main no longer has the old Discord-only direct GatewayClient construction targeted by this PR. Discord approval button decisions now go through resolveApprovalOverGateway and the shared operator approvals client; that shared client still lacks tlsFingerprint propagation, so the underlying bug is not fixed, but the remaining work is already tracked by #41740 and the newer current-path PR #62532.

## Best Possible Solution

Close #41741 as superseded by the current shared operator-approvals tracking path. Continue the actual fix in #62532 or #41740 by adding TLS fingerprint resolution to src/gateway/operator-approvals-client.ts or its bootstrap helper, with tests covering local loopback wss, remote mode, and URL overrides.
