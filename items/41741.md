## Summary

Keep open. Current `main` does not implement this fix in the active Discord approvals path. Discord approvals now flow through `resolveApprovalOverGateway()` into `withOperatorApprovalsGatewayClient()`, and that client still creates `GatewayClient` without any `tlsFingerprint`, so a local self-signed `wss://` gateway would still fail TLS verification on this path.
