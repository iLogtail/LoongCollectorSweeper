## Summary

Current `main` does not implement this cleanup. The Google Chat monitor still creates a real typing message, only clears it from the `deliver` path, and returns immediately after `dispatchReplyWithBufferedBlockDispatcher(...)` with no fallback cleanup. Because exact `NO_REPLY` finals are skipped before delivery, the reported orphaned typing message remains plausible on current `main`.
