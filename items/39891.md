## Summary

Keep PR #39891 open. Current main still creates a Google Chat typing message before dispatch and only clears it from the delivery path; exact allowed `NO_REPLY` finals in Google Chat spaces can skip delivery entirely, leaving the manually-created typing message orphaned. The PR targets the right behavior, but should be refreshed against current main and get a focused regression test before merge.

## Best Possible Solution

Keep PR #39891 and linked issue #39843 active. The best maintainer path is to refresh the Google Chat fix against current main, delete the leftover `typingMessageName` after dispatch when delivery never ran, catch and log cleanup failures, avoid double-deleting after normal delivery, and add a focused regression test for Google Chat space reactions followed by exact `NO_REPLY`.
