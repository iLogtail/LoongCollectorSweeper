## Summary

Current main still appears to have the Google Chat NO_REPLY typing-message leak described by the PR and linked issue. Google Chat creates a real typing message, clears it only inside the delivery path, and exact silent final payloads can be skipped before delivery, so this PR remains relevant even if the final patch should be refreshed to the current typing-cleanup seam.

## Best Possible Solution

Keep this PR or the linked #39843 bug open until Google Chat deletes the real typing message on exact silent NO_REPLY runs. The likely maintainer-grade fix is to wire Google Chat's `typingMessageName` cleanup into `onTypingCleanup` or add an equivalent post-dispatch guard, then add a focused regression test for the NO_REPLY path so future dispatcher changes cannot reintroduce the leak.
