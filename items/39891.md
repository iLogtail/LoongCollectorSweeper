## Summary

Keep PR #39891 open. Current main still has the Google Chat typing-message leak described by #39843: the channel creates a real typing message before dispatch, clears it only when `deliver` runs, and exact `NO_REPLY` finals can skip delivery entirely.

## Best Possible Solution

Keep PR #39891 and linked issue #39843 active. The best maintainer path is to refresh the Google Chat fix against current main, delete `typingMessageName` after dispatch when it remains set, catch and log cleanup failures, and add a focused Google Chat monitor-level regression test for the reaction plus exact `NO_REPLY` path.
