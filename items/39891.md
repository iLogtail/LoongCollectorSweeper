## Summary

Keep open. Current `main` still matches the bug described in #39891: Google Chat posts a real typing message, only clears `typingMessageName` inside `deliver`, and provides no channel-specific cleanup when an exact `NO_REPLY` final payload is skipped before delivery.

## Evidence

- **Current Google Chat flow still leaves cleanup inside `deliver` only:** `extensions/googlechat/src/monitor.ts` on `main` sends `_BotName is typing..._`, stores `typingMessageName`, calls `dispatchReplyWithBufferedBlockDispatcher`, and only resets the typing message state inside the `deliver` callback. There is no post-dispatch fallback cleanup after the dispatcher returns.
  - file: [extensions/googlechat/src/monitor.ts:258](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/extensions/googlechat/src/monitor.ts#L258)
  - command: `nl -ba extensions/googlechat/src/monitor.ts | sed -n '258,315p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Exact silent finals are skipped before delivery:** The reply dispatcher returns without calling channel delivery when the normalized final payload is an exact silent reply (`NO_REPLY`). That means a channel that only cleans up typing inside `deliver` will miss the silent path.
  - file: [src/auto-reply/reply/reply-dispatcher.ts:228](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/auto-reply/reply/reply-dispatcher.ts#L228)
  - command: `nl -ba src/auto-reply/reply/reply-dispatcher.ts | sed -n '226,236p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Core has a typing-cleanup seam, but Google Chat is not using it here:** `get-reply.ts` wires `opts.onTypingCleanup` into the typing controller, but this Google Chat call site passes only `onModelSelected` in `replyOptions` (`extensions/googlechat/src/monitor.ts:312`). The silent-reply cleanup seam exists, yet this path does not hook into it.
  - file: [src/auto-reply/reply/get-reply.ts:235](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/auto-reply/reply/get-reply.ts#L235)

  - file: [extensions/googlechat/src/monitor.ts:286](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/googlechat/src/monitor.ts#L286)
