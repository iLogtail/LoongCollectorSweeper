## Summary

Keep open. Current `main` does not implement the PR’s core change: the Linux gateway install path still has no sudo-entry guard, and systemd unit paths still resolve from `HOME` rather than the invoking non-root user. The docs still describe this as a systemd user service with `loginctl enable-linger` as a separate sudo step, which matches the PR motivation. The PR discussion also shows unresolved review concerns on the branch itself, so there is not strong evidence to close it conservatively.
