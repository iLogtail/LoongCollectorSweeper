## Summary

Keep open. Current main still allows the sudo gateway install mismatch this PR is trying to address: the CLI install path has no sudo guard, systemd unit writes still resolve through HOME, and systemctl under SUDO_USER targets the sudo caller's user scope. The PR is not safely obsolete or superseded; the related #42051 is closed and unmerged, while this PR also has review concerns that should be resolved rather than auto-closed.

## Best Possible Solution

Keep the PR open and steer it toward a narrow fix: reject sudo gateway installs before every first-party service.install entrypoint can reach the systemd writer, keep loginctl enable-linger as the only sudo-required step, add focused CLI/unit coverage, and remove unrelated build-config churn. If maintainers prefer allowing sudo installs instead, that should be a separate, security-reviewed systemd path/ownership design rather than this cleanup PR.
