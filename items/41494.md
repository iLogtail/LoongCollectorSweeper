## Summary

Keep open. Current `main` only strips tagged reasoning blocks (`<think>`, `<thinking>`, `<thought>`, `<final>`) or explicit `Reasoning:` payloads. The plain-text Gemini leak shapes described in #41494 are not covered by the current sanitizer or channel-delivery logic, so there is not strong evidence that the issue is implemented or obsolete.

## Evidence

- **Core reasoning sanitizer is tag-only:** `src/shared/text/reasoning-tags.ts` only matches XML-style reasoning/final tags. There is no handling for plain-text preambles like `Thinking:`, `Analysis:`, or freeform Gemini self-narration.
  - file: [src/shared/text/reasoning-tags.ts:5](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/shared/text/reasoning-tags.ts#L5)
  - command: `nl -ba src/shared/text/reasoning-tags.ts | sed -n '1,92p'`
- **Telegram split path only recognizes tagged reasoning or `Reasoning:` blocks:** `splitTelegramReasoningText()` returns a reasoning lane only for `Reasoning:\n...` messages or extracted tagged `<think>/<thinking>/<thought>` content; otherwise it leaves the text as the answer unchanged.
  - file: [extensions/telegram/src/reasoning-lane-coordinator.ts:65](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/telegram/src/reasoning-lane-coordinator.ts#L65)
  - command: `nl -ba extensions/telegram/src/reasoning-lane-coordinator.ts | sed -n '1,91p'`
- **Generic outbound reasoning detection is prefix-based:** `isReasoningReplyPayload()` treats a payload as reasoning only when `isReasoning === true` or the text starts with `Reasoning:` (optionally quoted). It does not classify plain Gemini reasoning prose as hidden reasoning.
  - file: [src/plugin-sdk/reply-payload.ts:39](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/plugin-sdk/reply-payload.ts#L39)
  - command: `nl -ba src/plugin-sdk/reply-payload.ts | sed -n '39,64p'`
- **Tests confirm the narrow scope:** Current tests cover tagged reasoning stripping and explicit `Reasoning:` prefixes. They also explicitly preserve mid-message `Reasoning:` mentions instead of stripping them, which is consistent with the current narrow implementation.
  - file: [src/plugin-sdk/reply-payload.test.ts:20](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/plugin-sdk/reply-payload.test.ts#L20)
  - command: `nl -ba src/plugin-sdk/reply-payload.test.ts | sed -n '20,33p'; nl -ba extensions/telegram/src/reasoning-lane-coordinator.test.ts | sed -n '1,28p'`
- **Previous shipped fix was specifically for `<think>` leakage:** The changelog entry for #26551 says OpenClaw treated Google as a reasoning-tag provider to prevent `<think>` leakage. That is narrower than this issue's report about plain-text reasoning leaking through Gemini responses.
  - file: [CHANGELOG.md:3840](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/CHANGELOG.md#L3840)
  - command: `nl -ba CHANGELOG.md | sed -n '3838,3841p'`
- **No evidence of current plain-text Gemini leak handling:** Repo-wide search did not find sanitizers or tests for the reported plain-text Gemini patterns (`Thinking Process`, `The SOUL.md says`, `Actually, the rule says`, `Tankegang`, `Analyse`, `Begrunnelse`).
