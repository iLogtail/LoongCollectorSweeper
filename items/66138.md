## Summary

Keep open. Current `main` already has significant dashboard/config UX, including schema-driven config forms, direct Control UI config apply, per-channel/plugin config, and ClawHub-backed skill install. But this issue asks for a broader product-level UX direction ('feel like WordPress') rather than a narrow, already-complete change, and OpenClaw's vision still explicitly includes setup UX and web frontend ergonomics as active core priorities.
