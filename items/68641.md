## Summary

Current main already fixes the Control UI image-history problem that PR #68641 targeted. The browser Control UI now reconstructs image previews from structured image blocks and transcript MediaPath/MediaPaths, the gateway persists uploaded chat images back into MediaPath/MediaPaths on the user transcript, and both UI and gateway tests cover the behavior. The PR's Swift shared-UI implementation is therefore no longer needed for the linked #68605 webchat bug.

## Best Possible Solution

Close PR #68641 as implemented on main for the linked Control UI webchat bug. Keep the shipped gateway/web UI implementation and its tests; handle any remaining native OpenClawKit image-preview work as a separate, scoped issue if maintainers want that surface changed.
