## Summary

Keep #54821 open. Current main still does not implement the PR's central capability-subscription behavior: there is no `thinking-events` client cap, gateway reasoning streaming is still gated on `onReasoningStream`, thinking deltas are still computed from formatted text, and non-tool agent events still use the generic broadcast/node path. The related #48995 is closed, but the provided timeline shows it was cross-referenced to this still-open PR rather than superseded by a merged implementation.

## Best Possible Solution

Keep the PR open, rebase it onto current main, address the remaining recipient-cleanup review concern, and land a focused implementation that adds the `thinking-events` cap, raw reasoning delta tracking, cap-gated gateway routing, and regression coverage. The docs for gateway client capabilities should be updated as part of the same protocol-facing change.
