## Summary

Keep PR #70554 open. Current main at b67d9bf still has the memory-core dreaming fallback behavior the PR targets: request-scoped subagent runs write a local fallback instead of retrying through a gateway background runtime, cleanup still uses the same request-scoped subagent, and plugin subagent dispatch does not accept or forward bootstrap run kind. The PR is recent and tied to real open bug #67152; it needs maintainer review and likely narrowing, not conservative closure.

## Best Possible Solution

Keep #70554 open and have a maintainer either finish this PR or replace it with a narrower patch for #67152. The best implementation should retry request-scoped dreaming narrative runs through a narrow host-owned gateway subagent accessor, forward `bootstrapContextMode` and `bootstrapContextRunKind` through plugin subagent dispatch, preserve cron/lightweight behavior across embedded and CLI execution, keep gateway runtime mutators out of public SDK barrels, and retain focused regression coverage for non-transient rethrow plus transient cleanup fallback.
