## Summary

Keep #70554 open. Current main still has the memory-core dreaming narrative request-scope fallback behavior, and the PR is a live, focused attempt to fix the remaining bug tracked by #67152 rather than obsolete or duplicate cleanup.

## Best Possible Solution

Keep #70554 open as useful work for #67152. The best path is a narrowed maintainer-reviewed patch that keeps background gateway-subagent access on a purpose-specific non-general SDK surface, forwards `bootstrapContextMode` and `bootstrapContextRunKind` through plugin subagent dispatch and both CLI/embedded reply execution paths, preserves best-effort cleanup for transient retry failures, and keeps focused regression coverage for retry, cleanup, and non-transient error behavior.
