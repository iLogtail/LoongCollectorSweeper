## Summary

Keep PR #72225 open. Current main still contains the three Control UI chat-session refresh callsites that hardcode `activeMinutes: 0`, and `loadSessions` only forwards positive `activeMinutes`, so those paths still bypass the shared active-session window this PR targets.

## Best Possible Solution

Keep PR #72225 open for maintainer review or merge an equivalent scoped fix: replace the three remaining Control UI `activeMinutes: 0` session-list refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES` and keep regression coverage for initial chat refresh and session-switch refresh.
