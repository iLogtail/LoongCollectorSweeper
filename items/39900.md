## Summary

Current main still does not implement OpenAI TTS extraBody passthrough. The validation part of the report is partly stale because provider-specific TTS config now accepts arbitrary JSON-ish keys, but the bundled OpenAI speech provider normalizes only known fields and sends a fixed /audio/speech payload.

## Best Possible Solution

Keep the issue open and implement a provider-owned extraBody/extra_body object for messages.tts.providers.openai. The OpenAI speech provider should normalize it as a plain JSON object, merge it into the /audio/speech payload with a documented collision policy, and add focused provider tests plus TTS docs updates.
