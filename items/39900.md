## Summary

Keep open. Current main does not implement the requested `extraBody` passthrough for OpenAI-compatible TTS endpoints. The issue body's validation complaint appears outdated on current main, but the core feature request remains unshipped.

## Evidence

- **TTS schema now accepts arbitrary provider keys:** `TtsProviderConfigSchema` uses `.catchall(...)`, so `messages.tts.providers.openai.extraBody` would not be rejected by current config validation.
  - file: [src/config/zod-schema.core.ts:485](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/config/zod-schema.core.ts#L485)
  - command: `nl -ba src/config/zod-schema.core.ts | sed -n '482,522p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **OpenAI TTS config reader has no extraBody field:** The OpenAI speech provider only normalizes and reads `apiKey`, `baseUrl`, `model`, `voice`, `speed`, `instructions`, and `responseFormat`; there is no `extraBody` plumbing.
  - file: [extensions/openai/speech-provider.ts:32](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/openai/speech-provider.ts#L32)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '1,170p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **OpenAI TTS request body is fixed and does not merge provider-specific fields:** `openaiTTS()` serializes only `model`, `input`, `voice`, `response_format`, optional `speed`, and optional `instructions` before POSTing to `/audio/speech`. No custom body merge exists.
  - file: [extensions/openai/tts.ts:111](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/openai/tts.ts#L111)
  - command: `nl -ba extensions/openai/tts.ts | sed -n '90,155p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **User-facing docs list no extraBody support:** The OpenAI provider docs enumerate the supported TTS config keys and stop at `baseUrl`; there is no documented passthrough field for custom OpenAI-compatible TTS payload parameters.
  - file: [docs/providers/openai.md:331](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/docs/providers/openai.md#L331)
  - command: `nl -ba docs/providers/openai.md | sed -n '329,357p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
