## Summary

Keep open. Current main suppresses only assistant text that is explicitly phase-tagged as `commentary`, but the Anthropic transport still maps streamed text before/around `tool_use` into ordinary assistant `text` blocks with no phase metadata. I did not find strong evidence on `main` that this Telegram/Claude Opus 4.6 work-note leak is fully fixed.

## Evidence

- **Anthropic transport still emits plain assistant text around tool use:** `src/agents/anthropic-transport-stream.ts:811` creates streamed Anthropic `content_block_start` text as plain `{ type: "text" }` blocks, and `src/agents/anthropic-transport-stream.ts:882` forwards `text_delta` chunks without any `phase` or `textSignature` tagging. That means Claude pre-tool narration still enters the pipeline as ordinary assistant text.
