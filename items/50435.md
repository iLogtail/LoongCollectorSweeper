## Summary

Keep open. Current `main` still prefers a header-derived `Content-Disposition` filename over the Feishu message payload filename, so the reported mojibake bug is still plausible, but this PR’s unconditional Latin-1-to-UTF-8 recovery heuristic would also rewrite legitimate Latin-1 filenames (`"Â£ rates.txt"` -> `"£ rates.txt"`, `"Ã©.txt"` -> `"é.txt"`). That means the item is neither already implemented on `main` nor safe to close.
