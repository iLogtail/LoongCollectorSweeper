## Summary

Current `main` does not implement this PR’s behavior. The OpenAI TTS logic now lives in `extensions/openai/tts.ts`, and it still drops `instructions` unless the model name includes `gpt-4o-mini-tts`, even for custom OpenAI-compatible endpoints. Tests and docs on `main` still encode that restriction, so there is not strong evidence to close this as already implemented or obsolete.
