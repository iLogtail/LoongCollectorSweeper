## Summary

Keep open. Current `main` only preserves transcript-backed message fields for `before_reset`; `agent_end` and `before_compaction` still pass in-memory snapshots or omit `messages`, so the requested provenance-preservation behavior in #57796 is not implemented end-to-end.

## Evidence

- **`agent_end` still uses in-memory snapshot:** `runAgentEnd` is called with `messages: messagesSnapshot` from the live embedded session state, not transcript-backed messages. That matches the issue's reported provenance loss path rather than the proposed JSONL-backed fix.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2856](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/pi-embedded-runner/run/attempt.ts#L2856)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '2850,2888p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Engine-owned `before_compaction` omits messages entirely:** When the context engine owns compaction, `before_compaction` is fired with `{ messageCount: -1, sessionFile }` and no `messages` payload, so there is no runtime surface where plugin hooks could receive preserved `provenance` fields.
  - file: [src/agents/pi-embedded-runner/run.ts:712](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/pi-embedded-runner/run.ts#L712)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '704,724p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Other `before_compaction` path still uses live session messages:** The embedded subscribe compaction handler passes `ctx.params.session.messages` directly into `runBeforeCompaction`, which is the same in-memory message surface the issue says lacks `provenance`.
  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:26](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L26)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.compaction.ts | sed -n '20,38p'`

- **`before_reset` is already transcript-backed:** Gateway reset reads messages via `readSessionMessages(...)` before firing `before_reset`, and `readSessionMessages` returns parsed transcript messages without stripping custom fields like `provenance`. This means the issue is now only partially satisfied on `main`.
  - file: [src/gateway/session-reset-service.ts:441](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/session-reset-service.ts#L441)

  - file: [src/gateway/session-utils.fs.ts:113](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/session-utils.fs.ts#L113)

Current `main` does not reflect that full approach because only `before_reset` is transcript-backed in the inspected code.
