## Summary

Keep open. Current main and the latest release still contain the reported Gateway URL input behavior, so the UI bug is not implemented or obsolete. However, this PR’s current UI fix preserves a token across arbitrary endpoint edits, which matches the provided security review concern and conflicts with the existing per-gateway token-scoping model. The separate LanceDB retry change appears already implemented on main, so the best path is maintainer judgment or a split/rework, not cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer judgment, or ask the contributor to split/drop the already-implemented LanceDB retry change and rework the UI fix around the existing per-gateway token-scope contract. The eventual UI fix should address the annoying per-keystroke clearing without silently carrying a token from gateway A to gateway B.
