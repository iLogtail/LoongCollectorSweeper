## Summary

Keep open. Current `main` does not implement the PR title change: `ui/src/ui/views/overview.ts` still clears `token` when `gatewayUrl` changes, and that behavior is reinforced by current tests and storage scoping. The PR’s separate `memory-lancedb` retry fix appears to already be implemented on `main` in a broader form, but the UI change would regress current per-gateway token isolation, so none of the allowed close reasons fit cleanly.
