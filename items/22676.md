## Summary

Current main still matches the reported Signal restart race. The Signal daemon stop contract is synchronous, sends SIGTERM without awaiting process exit or escalating to SIGKILL, and the gateway reload path can start the channel again immediately after stopChannel returns. A maintainer Codex review also explicitly kept the issue open and points to open PR #71863 as the active candidate fix.

## Best Possible Solution

Keep #22676 open until Signal daemon shutdown is made asynchronous and awaited through monitor/channel reload teardown: SIGTERM, wait for exit, escalate to SIGKILL on timeout, and only then allow restart. PR #71863 appears to be the active candidate; maintainers should review it or land an equivalent scoped fix with regression coverage for daemon stop semantics and aborted/reload shutdown.
