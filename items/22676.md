## Summary

Current main still matches the reported Signal restart race: Signal daemon shutdown is synchronous, only sends SIGTERM, and monitor/reload teardown has no awaited child-process exit barrier before a channel restart. The issue should remain open while PR #71863 or an equivalent fix is reviewed and merged.

## Best Possible Solution

Keep #22676 open until Signal daemon shutdown is changed to an awaited `Promise<void>` flow that sends SIGTERM, waits for the process to exit, escalates to SIGKILL on timeout, and is awaited by monitor/channel reload teardown before restart. PR #71863 appears to be the active candidate for that fix; maintainers should review it or land an equivalent scoped patch with focused regression coverage for daemon stop semantics and aborted/reload shutdown.
