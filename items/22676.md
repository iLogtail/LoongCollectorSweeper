## Summary

Keep open. Current `main` still implements Signal daemon shutdown as a synchronous `SIGTERM` fire-and-forget path, and the Signal monitor teardown/restart path still does not await daemon exit. The latest release `v2026.4.23` ships the same behavior. A related PR (#46357) exists but was closed unmerged, and the reporter posted another confirmation on April 24, 2026.

## Evidence

  - file: [extensions/signal/src/daemon.ts:16](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/signal/src/daemon.ts#L16)
