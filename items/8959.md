## Summary

Keep #8959 open. Current main still exposes a portable message presentation contract with buttons and single-select blocks only; it does not provide the requested cross-platform multi-select/message-tool API with min/max selection bounds and one structured selected-values callback. Discord has some channel-native select/minValues/maxValues types, but the shared message contract and Slack renderer remain single-select, and related PR #18286 is closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract, likely in src/interactive/payload.ts and src/agents/tools/message-tool.ts, exported through openclaw/plugin-sdk/interactive-runtime, documented at docs.openclaw.ai/plugins/message-presentation, and rendered/handled by Slack, Discord, Telegram, and fallback text paths with focused contract tests. A plugin-only answer is not enough because the requested feature is a shared message-tool API and callback contract.
