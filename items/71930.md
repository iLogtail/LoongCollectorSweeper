## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before post parsing, mention detection, debounce, and wake routing. This is an actionable bundled Mattermost plugin bug, not a ClawHub/off-repo request, and the provided related items do not supersede this specific edit-to-mention behavior.

## Best Possible Solution

Keep #71930 open. Fix it in the bundled Mattermost plugin by accepting `post_edited` as a post-bearing WebSocket event, passing the updated post through the existing mention/access/session path, and making replay dedupe edit-aware so an original unmentioned post does not suppress a later edit-to-mention. Add focused tests for posted-without-mention followed by `post_edited` with a mention, plus a non-mention edit that stays quiet.
