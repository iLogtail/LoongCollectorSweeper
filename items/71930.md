## Summary

Keep open. Current main still routes only Mattermost WebSocket `posted` events into the post/mention pipeline; `post_edited` is a real Mattermost WebSocket event but is rejected before `onPosted` and mention gating can run. Mattermost is a bundled OpenClaw plugin, so this is actionable in this repo rather than a ClawHub close.

## Best Possible Solution

Keep #71930 open and fix it in the bundled Mattermost plugin. Add a typed `post_edited` path in `extensions/mattermost/src/mattermost/monitor-websocket.ts` that parses the updated post body and routes it through the existing debouncer/`handlePost`/mention-gate flow, with tests for an edit that adds an @mention, a non-triggering edit, and duplicate edit behavior.
