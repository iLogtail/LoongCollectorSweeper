## Summary

Current main already covers the reported memory-flush failure mode: flush gating uses fresh projected token counts derived from transcript usage when cached totals are stale/unknown, and it also supports a transcript-size forced-flush path. The docs now also clarify that `softThresholdTokens` is headroom below compaction, not an absolute total-token trigger.
