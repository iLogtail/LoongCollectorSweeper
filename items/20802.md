## Summary

Current `main` does not implement this PR’s mutable-hook behavior. `llm_input`, `llm_output`, and `after_tool_call` are still typed and executed as observe-only `void` hooks, embedded attempts still fire `llm_input`/`llm_output` fire-and-forget without consuming a return value, and current tests/changelog explicitly preserve the opposite `after_tool_call` adapter behavior this PR proposes.
