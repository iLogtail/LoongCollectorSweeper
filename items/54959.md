## Summary

Keep PR #54959 open. Current main has OpenShell backend/configuration docs, but it does not contain the proposed OpenShell deployment guide or a Platforms navigation entry. The PR is still a plausible docs contribution for an existing bundled sandbox backend, but it needs maintainer/docs/security review because the latest PR context includes an unresolved P1 concern about documenting a blanket SSRF/fetch-guard bypass.

## Best Possible Solution

Keep the PR open for maintainer/docs/security review. The best path is to rework the guide so it complements `docs/gateway/openshell.md`, remove or replace the unsafe fetch-guard/SSRF workaround with source-backed shipped behavior and any canonical follow-up such as the trusted-proxy work, and consider splitting the platform deployment runbook from troubleshooting/workaround sections if that makes review tractable.
