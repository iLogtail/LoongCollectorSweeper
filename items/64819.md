## Summary

Keep #64819 open. Current main does not expose `openclaw agent --spawn`; it only fixes the older `--agent` plus `--session-id` resolver problem. The PR still represents a real CLI capability, but it needs maintainer product review and a much narrower, rebased branch before merge.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a narrow rebase that only touches the agent CLI, focused tests, and docs. If accepted, the implementation should reuse the existing `sessions.patch`/subagent metadata contract, pin scopes deliberately, handle `--run-id` retries without orphaning sessions, and document when callers should use `--session-id` versus `--spawn`.
