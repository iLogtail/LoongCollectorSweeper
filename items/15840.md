## Summary

Keep #15840 open. Current main still synthesizes visible `<media:audio>` / `<media:${kind}>` fallback text for blank media-only Signal and iMessage sends, and the strict `messages` config surface still has no suppression option. The provided related PRs for this feature were closed unmerged, and the latest issue discussion already identifies this as the canonical cross-channel request.

## Best Possible Solution

Keep #15840 open as the canonical product/config request. A conservative implementation should add a documented default-off `messages` option, update `MessagesConfig`, strict zod schema, generated config metadata/help/docs, and route the setting through bundled Signal and iMessage media-only send paths while preserving current accessibility/fallback behavior by default and adding focused tests for audio/media-only sends.
