## Summary

Keep #15840 open. Current main still synthesizes visible media placeholder text for blank media-only Signal and iMessage sends, the shared MIME helper maps audio attachments to the `audio` kind used in `<media:audio>`, and the strict `messages` config surface still has no `suppressMediaPlaceholders` or granular `mediaPlaceholders` option. The issue also has a recent maintainer-side Codex review comment explicitly keeping it open as the canonical cross-channel request, while related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #15840 open as the canonical product/config request. A conservative implementation should add a documented, default-off `messages` option, update `MessagesConfig`, the strict zod schema, generated config metadata/help/docs, and route that setting through the bundled Signal and iMessage media-only send paths while preserving the current accessibility fallback by default and adding focused tests for audio/media-only sends.
