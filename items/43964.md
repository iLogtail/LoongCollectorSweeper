## Summary

Keep open. Current main statically routes Feishu group replies and media sends through `chat_id`, so the exact `receive_id_type` mismatch described in the issue is not evident in code. But there is no end-to-end group-TTS regression test or live reproduction proof here, and there is no substantive Feishu send-path change between `v2026.4.22` and `main`, so the evidence is not strong enough for a conservative close.
