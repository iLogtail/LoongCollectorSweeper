## Summary

Keep open. Current main improves the reported UX by classifying gateway-draining as a restart lifecycle condition instead of falling through to the raw generic error, but it still returns a visible restart notice and marks the run failed rather than retrying the turn. The related PR #55470 proposed the one-shot retry behavior but was closed unmerged, and #63279 is a broader failure-chain investigation rather than a canonical replacement for this narrow request.

## Best Possible Solution

Keep #55412 open unless maintainers explicitly decide that a visible restart notice is the desired product behavior. A complete fix would add a bounded retry or restart-continuation path for `GatewayDrainingError` before finalizing the reply, with tests for direct and fallback-summary-wrapped errors; coordinate only the broader cascade/subagent/fallback aspects with #63279.
