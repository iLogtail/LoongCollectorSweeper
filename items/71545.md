## Summary

Keep open. Current `main` does not implement streamable-HTTP session auto-reinitialization, so the reported bug is still plausible, and PR #71545 has unresolved correctness regressions called out in review.

## Evidence

- **Main still has no reconnect path:** On current `main`, `callTool` does a single `session.client.callTool(...)` and throws `bundle-mcp server ... is not connected` when the session is missing; there is no retry or streamable-HTTP reinit logic in this path.
