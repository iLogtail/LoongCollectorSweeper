## Summary

Keep #71545 open. Current main still supports `streamable-http` MCP servers but `createSessionMcpRuntime().callTool()` only uses the cached session/client and forwards one `client.callTool()` call; it does not detect the SDK's stateful-session loss errors, rebuild the `StreamableHTTPClientTransport`, refresh tool metadata, or retry. The open PR is a plausible focused fix, and the maintainer force-pushed hardening commit addresses the bot-review concerns with a shared client factory, in-flight reconnect guard, narrower lost-session detection, retryable failed reconnects, and targeted streamable-http regression tests.

## Best Possible Solution

Keep #71545 open. The best path is to continue reviewing and, if validation passes, land the streamable-http reconnect implementation in `src/agents/pi-bundle-mcp-runtime.ts` with the added regression coverage in `src/agents/pi-bundle-mcp-runtime.test.ts`, preserving the custom schema validator, single in-flight reconnect, narrow session-loss detection, metadata refresh, and retryability after failed reconnect attempts.
