## Summary

Keep open. Current main still does not implement this PR's central behavior: there is no `markdown.strip` config, no `shouldStripMarkdown()` helper, and shared outbound delivery only runs a channel `sanitizeText` hook. The PR remains a useful but flawed fix candidate; related #58948 is narrower, and #24943 tracks part of the user problem but does not supersede this broader outbound PR.

## Best Possible Solution

Keep the PR open for maintainer-directed revision or replacement. The best path is an account-aware outbound Markdown stripping design that respects registered plugin/channel markdown capability, preserves WhatsApp/lightweight-markup channels, fixes fenced-code and underscore edge cases in `stripMarkdown()`, adds focused delivery and helper tests, and links the narrower iMessage tracking issue #24943 plus fenced-code PR #58948 as related work.
