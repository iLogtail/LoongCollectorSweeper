## Summary

Keep PR #72177 open. Current main still lacks the PR's central LiteLLM image-generation hardening: LiteLLM edits are JSON/data-URL based, there is no LiteLLM 180s provider timeout default, the agent tool still exposes model-supplied timeoutMs, and there is no chat-completions image route or Gemini duplicate-image parsing/dedupe path.

## Best Possible Solution

Keep #72177 open for maintainer review. The best path is to land this PR or an equivalent focused patch that switches LiteLLM edits to multipart, gives LiteLLM image requests a provider default timeout, removes or ignores model-authored image timeout overrides if that is the intended contract, adds a robust chat-completions route for chat-native LiteLLM image models, and covers parsing/dedupe behavior with regression tests before running the relevant changed lanes.
