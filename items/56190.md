## Summary

Keep open. Current main still only supports plain `${VAR}` config env substitution and write-back preservation; `${VAR:-default}` is not implemented or documented. The PR is a focused core config feature, not a ClawHub/plugin item, and the provided review discussion shows useful follow-up commits for parser edge cases rather than grounds for cleanup closure.

## Best Possible Solution

Keep the PR open and land a carefully reviewed core implementation that updates `src/config/env-substitution.ts`, `src/config/env-preserve.ts`, and the `containsEnvVarReference`/`applyConfigEnvVars` interaction together. Preserve existing `${VAR}` and `$${VAR}` behavior, add regression tests for defaults with URLs, colons, braces, quotes, empty defaults, `onMissing` bypass, write-back restoration, and `config.env` chaining, then document the new `${VAR:-default}` syntax in the public config/plugin docs.
