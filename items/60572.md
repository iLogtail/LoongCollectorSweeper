## Summary

Keep open. Current main has one exclusive `plugins.slots.memory` selector plus additive memory-adjacent supplement/hook APIs, and it includes a narrower fix for `memory-core` dreaming coexistence. It does not implement the requested purpose-specific `memory.recall`, `memory.compaction`, `memory.capture`, or `memory.userModel` slots, nor per-agent memory slot scoping; memory-only runtime plugins still remain mutually exclusive.

## Best Possible Solution

Keep this open as a core memory/plugin architecture RFC. Maintainers should either define a minimal additive memory-layer contract that maps recall, capture, user modeling, compaction, and dreaming onto existing supplement/hook surfaces, or explicitly add a backward-compatible slot/subslot schema with loader/runtime semantics and docs. Any implementation should update config types, Zod schema, plugin SDK docs, loader tests, and clarify how per-agent scoping interacts with the existing `plugins.slots.memory` fallback.
