## Summary

Keep #33102 open. Current main still has no config-backed default for TUI delivery: `openclaw tui` only forwards the explicit `--deliver` flag, `runTui` falls back to `false`, sends use that option-derived value, and the config schema/types/docs still do not define `tui.deliver`. The linked config PRs #33198 and #33208 were closed unmerged, and the later stuck-spinner report #66991 is related but separate from this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved implementation adds a documented config key, likely `tui.deliver`, wires it through config types, zod schema, generated schema/help/docs, and CLI precedence, and adds focused tests for config default, explicit flag override, and fallback behavior. Track the stuck-spinner behavior in #66991 separately unless a single PR intentionally fixes both.
