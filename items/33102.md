## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The CLI exposes only an explicit `--deliver` flag defaulting to false, the TUI runtime derives delivery from startup options, and the config type/schema/docs have no `tui.deliver` surface. The related PRs #33198 and #33208 were closed unmerged, and the external workaround plus #66991 discussion are useful prior art but do not satisfy this persisted-default request on main.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented config key such as `tui.deliver`, wires it through `OpenClawConfig`, zod schema/generated schema help, CLI precedence, and TUI send behavior, and adds focused tests for config default, explicit CLI override, and false fallback. Treat #33198/#33208 and the arcabot workaround as prior art, and track #66991 only for the related stuck-spinner failure mode unless one PR intentionally fixes both.
