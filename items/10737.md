## Summary

Keep #10737 open. Current main still does not implement legacy iMessage channel auto-acknowledgment or native typing feedback before dispatch, and the latest maintainer discussion explicitly kept this open despite BlueBubbles having typing support.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly decline legacy iMessage feedback. A concrete implementation should live in the bundled imessage plugin/config surface, likely adding typed channels.imessage autoAck or native-typing-with-text-fallback config, wiring feedback before dispatchInboundMessage, honoring DM-only behavior, and adding regression tests to prevent duplicate final replies. BlueBubbles should remain the recommended iMessage path, but it is not a drop-in resolution for legacy channels.imessage.
