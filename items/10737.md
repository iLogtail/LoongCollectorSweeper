## Summary

Keep #10737 open. Current main still lacks legacy iMessage channel-layer auto-acknowledgment or native typing wiring for `channels.imessage`; BlueBubbles has typing support, but it is a separate recommended iMessage channel and does not satisfy the requested legacy `channels.imessage.autoAck` behavior. The latest provided issue discussion also already contains a maintainer-side Codex review keeping this open.

## Best Possible Solution

Keep the issue open until maintainers either implement channel-level feedback for the bundled legacy iMessage plugin or explicitly decline it. A concrete implementation would likely live in `extensions/imessage`: add typed config/schema/docs for `channels.imessage.autoAck` or native typing with a text fallback, send the ack before `dispatchInboundMessage`, preserve DM-only behavior, and add regression tests for no duplicate final replies. BlueBubbles should remain documented as the recommended path, but it should not close this legacy iMessage request.
