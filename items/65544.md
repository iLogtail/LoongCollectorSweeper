## Summary

Keep open. Current main still runs `agent_end` through the generic void hook runner without a deadline, timeout option, or AbortSignal contract. The runner/harness warning paths only handle rejected promises, so a never-settling embedding call can still hang post-turn memory capture exactly as reported.

## Best Possible Solution

Keep the issue open and fix it in core plugin hook execution, not ClawHub. A good fix would add a configurable default timeout for `agent_end` or void lifecycle hooks, expose a per-hook or config-level timeout option, log a clear timeout warning, and ideally provide a cooperative AbortSignal in hook context so plugin HTTP work can be cancelled rather than only ignored. Add focused hook-runner tests for a never-settling handler plus embedded/CLI harness warning coverage.
