## Summary

Keep #62328 open. Current main still uses node:sqlite for the builtin memory database, creates keyword search only through SQLite FTS5, and skips keyword/BM25 results when FTS is unavailable. Related #44352 and #50453 were closed as duplicates pointing to this issue, while #64776 fixed a separate sqlite-vec vector-loading problem and #62338 remains an open, narrower doctor-visibility PR.

## Best Possible Solution

Keep #62328 as the canonical tracker. The best fix should land in memory-core: either add a non-FTS lexical fallback over chunks when FTS5 is unavailable, or introduce a supported SQLite driver/runtime path that guarantees FTS5. The narrower doctor warning from #62338 can be merged or respun as visibility work, but it should not close the core fallback issue by itself.
