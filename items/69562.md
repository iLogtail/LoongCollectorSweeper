## Summary

Keep #69562 open. Current main still validates cron announce channels but not explicit `delivery.accountId` or `delivery.failureDestination.accountId`, while the linked open bug #69513 reports that invalid cron account IDs are accepted and persisted. The PR is a focused gateway fix with regression tests, and the earlier related PR #69518 is closed unmerged rather than a shipped superseding fix.

## Best Possible Solution

Keep the PR open for maintainer review and rebase. The best path is a focused gateway-layer fix for #69513 that validates explicit cron announce `delivery.accountId` and `delivery.failureDestination.accountId` against the resolved configured channel accounts, while preserving valid default/unlisted/default-normalized account behavior and adding the focused rejection/acceptance tests shown in #69562.
