## Summary

Keep open. Current main still has the behaviors this PR is trying to change: request-scoped fallback diary text is written from raw snippets/promotions, Dream Diary dedupe still fingerprints timestamp plus body, append does not skip duplicate bodies, and managed daily dreaming stripping still treats any heading as a boundary. The related PRs #70403 and #70523 cover adjacent cleanup, but they do not fully supersede this PR's heading-strip and Dream Diary body-dedupe work. The PR also has an unresolved review finding around italic diary-line fingerprinting, so it needs maintainer/author follow-up rather than conservative closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either a corrected focused version of this PR, or an explicit maintainer split where #70403/#70523 handle their narrower cleanup and a separate follow-up keeps the heading-level strip and Dream Diary body-dedupe fixes with regression coverage. Before merge, fix the diary fingerprint review issue and confirm the desired body-only dedupe semantics.
