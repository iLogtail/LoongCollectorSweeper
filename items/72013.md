## Summary

Keep open. Current main still runs startup ACP identity reconciliation for any persisted ACP session whose identity is pending and has a stable ACP/session id; it does not skip terminal one-shot records before ensuring/reconciling the runtime handle. The related PR #72023 appears to be the right active fix path, but it is still open and not present in the checked-out main.

## Best Possible Solution

Keep #72013 open until PR #72023 or an equivalent maintainer-approved patch lands. The best code path is to skip terminal one-shot and any valid closed ACP metadata before pending-identity runtime re-ensure in `AcpSessionManager.reconcilePendingSessionIdentities`, with focused regression tests proving the runtime is not invoked and the gateway warning stays quiet for those records.
