## Summary

Keep open. Current main still enforces the exact HTTPS-only and `api.firecrawl.dev` host-only checks reported by the user, the Firecrawl test suite currently asserts that localhost/custom hosts are rejected, and the docs still say base URL overrides must stay on the official Firecrawl API host. The supplied GitHub context also shows an open implementation candidate, #63941, linked to this issue, so this should not be closed before that PR is reviewed, landed, or explicitly closed as obsolete.

## Best Possible Solution

Keep this issue open and handle it through the Firecrawl plugin implementation: review #63941 or another scoped PR that relaxes `resolveEndpoint` for explicitly configured local/private self-hosted Firecrawl endpoints while preserving HTTPS requirements for public hosts, then update `extensions/firecrawl/src/firecrawl-tools.test.ts` and `docs/tools/firecrawl.md` to match the shipped contract.
