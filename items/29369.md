## Summary

Keep open. Current `main` still does not expose or honor a spawn-time `tools` allowlist for `sessions_spawn`; the requested feature remains a core tool/API and tool-policy pipeline change, not a ClawHub/plugin cleanup or an already-shipped behavior.

## Best Possible Solution

Keep tracking #29369 until maintainers either design and implement a spawn-time tool policy through the `sessions_spawn` schema, native/ACP spawn parameter types, child-session metadata or run payload, and tool-policy resolution pipeline with tests, or explicitly document that dynamic per-spawn tool allowlists are unsupported in favor of configured agent profiles and `tools.subagents.tools` policy.
