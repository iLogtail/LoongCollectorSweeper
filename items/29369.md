## Summary

Keep open. Current main still does not expose or persist a `tools` parameter on `sessions_spawn`; native and ACP spawn paths only accept the existing task/runtime/model/thread/sandbox/context/attachment fields, while subagent tool filtering is still derived from config plus stored session capability metadata. The prior ClawSweeper comment remains accurate against current main.

## Best Possible Solution

Keep tracking this issue until maintainers either add a designed spawn-time tool policy through the `sessions_spawn` schema, native/ACP spawn param types, child session metadata, and tool-policy pipeline with tests, or explicitly document that per-spawn dynamic tool policies are unsupported in favor of config/agent profiles.
