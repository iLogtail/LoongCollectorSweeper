## Summary

Current `main` already provides exec isolation through the sandbox runtime architecture (`host=sandbox` / `host=auto` with Docker, SSH, or OpenShell backends), so this draft macOS-only `sandbox-exec` wrapper is superseded.

## Evidence

- **Exec docs already describe sandbox routing:** The current exec docs say `host` supports `auto|sandbox|gateway|node`, with `auto` resolving to `sandbox` when a sandbox runtime is active and `gateway` otherwise.
  - file: [docs/tools/exec.md:43](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/docs/tools/exec.md#L43)

  - sha: [f896c3935ef8](https://github.com/openclaw/openclaw/commit/f896c3935ef8d706fe4193443c5c5eede5e64178)
- **Exec fails closed without a sandbox runtime:** The exec tool currently requires a real sandbox runtime for `host=sandbox` and throws if none is available, showing sandboxing is an existing first-class path rather than a missing feature.
  - file: [src/agents/bash-tools.exec.ts:1497](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/agents/bash-tools.exec.ts#L1497)
