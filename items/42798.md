## Summary

Keep open. Current main still runs the Tailscale Serve command and then immediately probes `tailscale status --json` without retry/backoff, and the shared executor still emits the exact verbose `Command failed: tailscale status --json` log before the caller can swallow the hostname lookup failure. The alternate suggested `gateway.tailscale.binaryPath` option is also not present in the current config type.

## Evidence
