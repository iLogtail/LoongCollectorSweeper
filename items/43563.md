## Summary

Keep #43563 open. Current main has a related Feishu broadcast path, but it is an inbound-message fan-out to configured agent sessions, not the requested send-time cross-bot broadcast that replays bot-authored Feishu group messages to other registered bot accounts. The Feishu config/docs also do not expose the proposed `channels.feishu.crossBotBroadcast` loop-guarded feature.

## Best Possible Solution

Keep this open for maintainer/product judgment. The best implementation would extend the existing Feishu/broadcast routing deliberately: add an opt-in Feishu cross-bot broadcast config, register active bot accounts/groups, inject synthetic bot-originated inbound events on successful sends, enforce per-group loop limits/reset on human messages, and add focused tests/docs. If maintainers want this to become cross-channel, define a narrow channel/plugin API before generalizing beyond Feishu.
