## Summary

Keep open. Current `main` does not implement native Feishu `at` elements for post messages, so the reported behavior is not already fixed, and the PR discussion still shows unresolved correctness risks in the proposed approach.

## Evidence

- **Current main still wraps Feishu post replies in a single `md` element:** `buildFeishuPostMessagePayload()` always serializes `messageText` as `[[{ tag: "md", text: messageText }]]`; there is no path that emits native post `at` elements on `main`.

  - file: [extensions/feishu/src/mention.ts:79](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/feishu/src/mention.ts#L79)
