## Summary

Keep open. Current main still has an ACP-specific gap: the ACP reply-dispatch path calls `applyMediaUnderstanding` without `agentDir`, while image media-understanding provider runs still require `agentDir`. That means the documented fallback media-understanding path for non-vision models is not clearly implemented for ACP/agent-channel turns on main.

## Evidence

- **ACP dispatch omits `agentDir` when invoking media understanding:** `tryDispatchAcpReply` applies media understanding with only `{ ctx, cfg }`. No `agentDir` or `activeModel` is forwarded at this call site.
  - file: [src/auto-reply/reply/dispatch-acp.ts:394](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/reply/dispatch-acp.ts#L394)
  - command: `nl -ba src/auto-reply/reply/dispatch-acp.ts | sed -n '394,400p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Image media-understanding provider runs require `agentDir`:** Provider-backed image understanding throws `Error("Image understanding requires agentDir")` before calling the model when `agentDir` is absent.
  - file: [src/media-understanding/runner.entries.ts:528](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/media-understanding/runner.entries.ts#L528)
