## Summary

Keep #71140 open. Current main already has MEDIA/GIF parsing and Telegram native send/reaction primitives, but the reported reaction directive path is still missing: `[[react...]]` / `[[react_to_current:...]]` is not parsed into structured outbound data, Telegram delivery still rejects reaction-only replies with no text/media, and `telegramOutbound.sendPayload` only consumes Telegram buttons/quote metadata. The linked fix PR #71141 is open/draft and unmerged, so it is not implemented on main.

## Best Possible Solution

Keep #71140 open and use draft PR #71141, or an equivalent maintainer-owned patch, as the active implementation path: extend the shared directive parser to emit structured reaction metadata, preserve reaction-only payloads as sendable content, apply Telegram reactions via `reactMessageTelegram` in both reply delivery and outbound adapter paths, and add parser, normalization, adapter, and delivery tests for reaction-only and text/media-plus-reaction replies.
