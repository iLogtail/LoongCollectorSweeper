## Summary

Keep open. Current main still lacks the Telegram reaction-directive path the issue describes, even though Telegram already supports native GIF/media sends and has a direct reaction primitive. The provided maintainer comment and the current code both point to a real, still-open gap, and the issue is also linked to an open PR (#71141).

## Evidence

- **Reply directive parser has no reaction output:** `parseReplyDirectives` only returns text, media, reply threading, audio-as-voice, and silent flags. There is no reaction field or `channelData.telegram.reaction` projection from inline reply text.
  - file: [src/auto-reply/reply/reply-directives.ts:5](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/auto-reply/reply/reply-directives.ts#L5)

  - file: [src/infra/outbound/payloads.ts:138](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/infra/outbound/payloads.ts#L138)
