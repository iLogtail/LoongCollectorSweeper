## Summary

Keep #71140 open. Current main still lacks the reaction-directive path: reply normalization handles MEDIA/reply/audio directives, but not [[react...]], Telegram delivery still rejects payloads with no text/media, and sendPayload only preserves Telegram buttons/quote metadata. The linked #71141 is an open draft PR, not a shipped fix.

## Best Possible Solution

Land #71141 or an equivalent focused fix: extend shared reply directive parsing to emit structured reaction metadata, preserve it through outbound payload normalization, allow reaction-only Telegram replies when a reply target exists, and teach Telegram delivery/sendPayload to call reactMessageTelegram before optional text/media. Keep the media/GIF scope narrow because current main already has MEDIA parsing and Telegram sendAnimation/sendPhoto paths.
