## Summary

Current `main` still appears to drop WhatsApp reaction-only inbound events. The WhatsApp inbox path only enqueues inbound messages when it can derive text, contact, location, or media-placeholder content, and the extractor has no reaction-specific branch. Docs and action discovery also describe WhatsApp reactions only as agent/ack/outbound behavior, not inbound reaction notifications.

## Evidence

- **Inbound WhatsApp processing skips unsupported message shapes:** `handleMessagesUpsert` normalizes each `messages.upsert` entry, then calls `enrichInboundMessage`; if enrichment returns `null`, the message is skipped. `enrichInboundMessage` only succeeds when it can derive text, location text, contact context, or a media placeholder.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:469](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/whatsapp/src/inbound/monitor.ts#L469)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Extractor has no reaction payload handling:** `extractText` only handles conversation text, extended text, captions, and contact placeholders. `extractMediaPlaceholder` only handles image/video/audio/document/sticker payloads. There is no reaction-specific extraction path here.
  - file: [extensions/whatsapp/src/inbound/extract.ts:234](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/whatsapp/src/inbound/extract.ts#L234)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **WhatsApp docs describe reactions as reaction-level/ack/outbound tooling only:** The WhatsApp docs define `reactionLevel` as ack plus agent-initiated reactions, document `ackReaction`, and list `channels.whatsapp.actions.reactions` under tools/actions. They do not document any inbound reaction notification setting for WhatsApp.
  - file: [docs/channels/whatsapp.md:404](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/channels/whatsapp.md#L404)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **WhatsApp action surface is outbound-only:** `describeWhatsAppMessageActions` only advertises `react` and `poll` actions based on config. There is no corresponding inbound reaction event/action surface in the channel action description.
  - file: [extensions/whatsapp/src/channel-actions.ts:61](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/whatsapp/src/channel-actions.ts#L61)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
