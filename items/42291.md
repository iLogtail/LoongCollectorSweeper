## Summary

Keep open. Current main still routes `openclaw models auth login` through the local agent auth store and does not detect or block remote node-host usage before consuming OAuth credentials. The directly related fix PR #42381 is still open and unmerged, and the alternative sync PR #42407 was closed unmerged, so the bug is not implemented on main and should remain tracked.

## Best Possible Solution

Keep #42291 open until a mainline fix lands. The likely minimal product fix is the #42381 behavior: detect an active remote node-host configuration before OAuth login, fail with a clear instruction to run auth on the gateway host, and keep regression coverage in `src/commands/models/auth.test.ts`. A richer sync solution would need a real gateway auth-profile write API and security review first.
