## Summary

Keep open. Current main no longer appears to bypass Feishu plugin action dispatch for `message(action=send)`, but the specific reported behavior remains: a Feishu card JSON string in the `message` parameter is treated as plain text/post content. Main supports shared `presentation` payloads as Feishu cards, not auto-detection of native Feishu card JSON in `message`. The related closed Feishu items do not cover this remaining raw-card-JSON send path.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin send path. Add a Feishu-owned detector for native card JSON strings in `extensions/feishu/src/channel.ts` before the text fallback, call `sendCardFeishu` for supported v1/v2/wrapped interactive card shapes, and add focused regression coverage in `extensions/feishu/src/channel.test.ts` or the outbound/message-action tests for `message(action=send)` with raw card JSON. If maintainers intentionally want to drop raw Feishu JSON compatibility, document the migration to `presentation` instead of closing this as already fixed.
