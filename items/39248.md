## Summary

Keep #39248 open. Current main still classifies native `sessions_spawn` child sessions as non-main sandbox candidates, resolves the sandbox backend before the embedded child run proceeds, and has no regression test proving Docker-backed `agents.defaults.sandbox.mode: "non-main"` child runs initialize and write session/transcript state.

## Best Possible Solution

Keep the issue open and add a targeted regression test or live verification for native `sessions_spawn` with `agents.defaults.sandbox.mode: "non-main"`. The proof should assert the child run starts, writes transcript/messages, updates session state such as `systemSent` or token usage when applicable, and surfaces sandbox initialization or dispatch failures instead of silently timing out. If the repro still fails, patch the Docker-backed `resolveSandboxContext` or async child dispatch path with explicit error propagation/logging.
