## Summary

Current main still appears to contain the reported subsystem log rollover bug. `createSubsystemLogger` caches one `fileLogger` per subsystem logger and reuses it for normal and raw file logs, while the root logger can rebuild when the date-dependent default file path changes. The related fix PRs in the provided context are open or closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep the issue open and land a focused logging fix, likely by refreshing the subsystem child logger when the parent/root logger or date-resolved file path changes, with a regression test that proves a subsystem logger writes to the new dated file after local midnight. Existing PRs #54527 and #54589 are relevant candidates, but the fix still needs to be merged or reimplemented on current main.
