## Summary

Keep #38260 open. Current main still has Linux/Node WebChat image flows that can reach sharp/libvips via media-understanding-core, and I did not find source, test, docs, changelog, or related-item evidence proving the reported libvips-cpp SIGILL crash is fixed or superseded.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce or collect a core/backtrace on Linux npm-global Node 22, identify whether bundled sharp/libvips, system libvips, CPU feature selection, or image optimization is the root cause, then stage a known-safe native dependency, add a Linux-safe fallback/kill switch, or isolate native image work out of the gateway process with a WebChat image-flow regression smoke.
