## Summary

Current main still has the reported Feishu interactive-card parsing gap. The `msg_type: interactive` read path still uses `parseInteractiveCardContent`, which only extracts flat `markdown` and `div.text.content` elements; it does not extract top-level fallback `title`, `header.title`, nested post-format element arrays, `tag:"text"`, `tag:"a"`, or image placeholders. The linked PR #60383 appears to be the right focused follow-up, but it is still open/unmerged in the provided GitHub context, so the issue should not be closed as implemented.

## Best Possible Solution

Keep #60380 open until a Feishu parser fix lands. The likely fix belongs in `extensions/feishu/src/send.ts`, either by extending `parseInteractiveCardContent` or sharing the existing post parser so interactive reads handle top-level fallback `title`, `header.title`, `body.elements`/`elements`, nested row arrays, `text`, `a`, `img`, and `plain_text` variants. Add focused regression tests in `extensions/feishu/src/send.test.ts` for `getMessageFeishu` and thread-history reads against the reported post-format fallback payload, then close via PR #60383 or a successor.
