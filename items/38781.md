## Summary

Keep #38781 open. Current main still has the reported macOS node contract gap: the agent exec path requires `system.run.prepare`, while the Swift macOS node does not define, advertise, or dispatch that command. The related #69725 is still only an open alternate fallback path in the provided context, and #71939 was a closed duplicate of this same authorâ€™s rebased work, so there is no merged superseding fix to justify closing this PR.

## Best Possible Solution

Keep #38781 open until maintainers merge a direct macOS `system.run.prepare` implementation or intentionally supersede it with a merged fallback such as #69725. The best direct fix is to align the Swift macOS node with the TypeScript node-host contract: add shared prepare payload/plan types, advertise `system.run.prepare`, dispatch it in `MacNodeRuntime`, return the approval plan shape expected by `executeNodeHostCommand`, and cover command declaration, shell-wrapper/rawCommand consistency, env behavior, and path canonicalization with focused verification.
