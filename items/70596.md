## Summary

Keep PR #70596 open. Current main still only initializes QMD memory during gateway startup, so the requested builtin explicit-local embedding prewarm is not implemented. The PR is a plausible canonical place for the remaining #70460 work, but it is not closeable as obsolete or implemented, and the provided diff still needs maintainer review for startup gating and failed-prewarm cache behavior.

## Best Possible Solution

Keep #70596 open and ask for a revised patch: update the post-attach startup gate and tests so builtin memory with explicit `memorySearch.provider = "local"` actually schedules the startup work, and make the prewarm failure path retry-safe by avoiding or clearing poisoned cached default-manager provider initialization. Then verify with focused gateway startup tests plus the usual changed gate/build for this lazy startup surface.
