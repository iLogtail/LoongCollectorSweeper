## Summary

Keep open. Current main at 7e376e5aba32eeea8191e70d5e43e293f8b8398a has an implicit-stream guard for thread-context requesters, but it does not implement PR #55896's explicit Discord thread parent-relay suppression. When streamTo="parent" is active, the ACP parent relay still surfaces progress, stall, and resumed system events, and the PR discussion plus linked #55807 describe a plausible core Discord/ACP bug rather than obsolete or ClawHub work.

## Best Possible Solution

Keep #55896 or the linked #55807 open. The best maintainer path is to rebase or port the narrow suppression into the current ACP parent relay/spawn code, preserving relay logs and start/done/error/timeout lifecycle notices, and add regression coverage for explicit agentThreadId and thread-bound parent-session-key cases.
