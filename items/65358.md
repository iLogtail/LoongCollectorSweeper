## Summary

Current main has useful mitigations, but #65358 is not fully implemented. The upgrade path still does not take an automatic pre-update state/workspace/session backup, there is no first-class backup restore or rollback command, the general CLI sessions surface does not provide a targeted reset/delete command for one poisoned session, and legacy session model overrides without `modelOverrideSource` are intentionally preserved across `/new` and `/reset` rather than invalidated.

## Best Possible Solution

Keep the issue open and narrow it into concrete maintainer tasks: add an opt-out pre-update backup using the existing backup archive planner, design a supported restore/rollback workflow, expose a safe CLI session reset/delete or override-clear command for one session key, and add doctor/migration validation for stale model/provider/auth overrides that distinguishes real user selections from auto/fallback poison.
