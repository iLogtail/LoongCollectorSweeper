## Summary

Keep open. Current main still drops session label/displayName from the status summary contract and renders raw session keys in both CLI `/status` and TUI `/status`; the TUI statusline is still key-first. The Control UI session dropdown portion was addressed separately, and PR #51257 tracks part of the remaining work, but it is open/unmerged and does not clearly replace the full statusline/session-picker request.

## Best Possible Solution

Keep #51184 open as the canonical product request for friendly cron/session identity in `/status` and the TUI statusline. The likely implementation path is to rework/land #51257 or an equivalent patch that carries `label` and `displayName` through `SessionStatus` and `GatewayStatusSummary`, renders a sanitized friendly name while preserving a copyable resolvable selector, adds CLI/TUI status regression tests, and splits any remaining session-picker/statusline work into explicit follow-ups if it is intentionally out of scope.
