## Summary

Keep open. Current main still documents the Discord elevated allowFrom fallback but does not implement it for runtime auth or sandbox diagnostics, so this PR remains relevant to #53198 rather than obsolete. The PR likely needs maintainer review/rework because the best current shape is a Discord plugin `elevated.allowFromFallback` implementation plus shared diagnostics behavior, not a durable core special-case.

## Best Possible Solution

Keep the PR or its linked bug open, but steer the implementation toward adding `elevated.allowFromFallback` to the Discord plugin and making `sandbox explain` reuse the same effective fallback resolution with session/account metadata. Preserve current human output fields such as `mainSessionKey`, restrict provider inference to real channel ids, and keep focused regression tests for runtime auth, diagnostics, account-level fallback, and explicit-empty override behavior.
