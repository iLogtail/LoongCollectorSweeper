## Summary

Keep open. Current `main` still contradicts the documented Discord elevated `allowFrom` fallback and `sandbox explain` still ignores that fallback, so this PR is addressing a live bug rather than something already implemented, obsolete, or incoherent.

## Evidence

- **Docs still promise Discord elevated fallback:** The user-facing docs explicitly say `tools.elevated.allowFrom.discord` falls back to `channels.discord.allowFrom` when omitted, and Discord account docs say named accounts inherit the top-level allowlist when their own one is unset.
