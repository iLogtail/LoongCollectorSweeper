## Summary

Keep #39659 open. Current main still supports manual Docker/env/profile/container primitives and ClawDock convenience helpers, but it does not implement the requested first-class named Docker instance lifecycle (`openclaw instance create/list/delete/update`) or an equivalent managed create/list/update/delete workflow. The prior ClawSweeper comment reached the same keep-open conclusion, and the current `dc05c93` checkout has not changed that product surface.

## Best Possible Solution

Keep #39659 open until maintainers either implement a minimal Docker instance lifecycle or explicitly choose another product direction. A strong implementation would add a documented command or supported helper that stores per-instance metadata, validates/allocates ports and paths, sets compose project/container names, and provides safe create/list/update/delete cleanup with tests and Docker install docs.
