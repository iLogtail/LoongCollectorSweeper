## Summary

Keep #39659 open. Current main still has only manual Docker/profile/container primitives and ClawDock convenience helpers; it does not provide the requested first-class named Docker instance lifecycle (`openclaw instance create/list/delete/update`) or an equivalent managed create/list/update/delete workflow. The issue is recent, author association is NONE, labels are only `enhancement`, and the existing issue discussion already reached the same keep-open conclusion with no maintainer override in the provided context.

## Best Possible Solution

Keep #39659 open until maintainers either implement a minimal Docker instance lifecycle or explicitly choose another product direction. The strongest implementation would add a documented command surface or supported helper that maintains per-instance metadata, allocates/validates ports and paths, sets compose project/container names, and provides safe create/list/update/delete cleanup with tests and Docker install docs.
