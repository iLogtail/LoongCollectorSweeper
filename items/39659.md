## Summary

Keep #39659 open. Current main has generic `--profile` state isolation and `--container` execution targeting, plus Docker/ClawDock convenience helpers, but it still lacks the requested first-class Docker multi-instance lifecycle with named create/list/delete/update, per-instance metadata, managed port/path allocation, and compose/container isolation. Related PR #54126 was a closed, unmerged draft for profile management, not a canonical shipped Docker lifecycle replacement.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly decline a supported Docker multi-instance lifecycle. A good implementation would add a documented `openclaw instance ...` surface or Docker/ClawDock equivalent that records per-instance metadata, validates or allocates unique host ports and paths, isolates compose project/container names, and provides safe create/list/update/delete cleanup with focused tests and Docker install docs.

## Risks / Open Questions
