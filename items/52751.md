## Summary

Keep #52751 open. Current main still does not implement per-agent `compaction` or `contextPruning` overrides: the per-agent type and strict Zod schema omit both keys, resolved agent config does not carry them, and Pi runtime compaction/pruning paths still read `agents.defaults` only. The PR discussion also contains still-relevant runtime no-op review feedback, so this is not a cleanup-close candidate.

## Best Possible Solution

Keep #52751 open for maintainer review or replace it with an explicitly canonical PR that covers the full feature. A complete fix should add `agents.list[].compaction` and `agents.list[].contextPruning` to the config type, strict Zod schema, generated schema/help/docs, safely share or extract the defaults schemas, resolve effective per-agent values over `agents.defaults`, thread the resolved agent id/config through Pi settings, safeguard, context-pruning, memory-flush, manual `/compact`, and normal reply paths, and add focused schema/resolver/runtime tests.
