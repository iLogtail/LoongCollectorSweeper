## Summary

Current `main` does not implement per-agent `compaction` / `contextPruning` overrides. The feature is still defaults-only in types, schema validation, runtime resolution, and user-facing docs, so this PR is not superseded and should stay open for maintainer review.

## Evidence

- **Agent list type is still defaults-only for these settings:** `AgentConfig` in `src/config/types.agents.ts` defines per-agent fields like `thinkingDefault`, `fastModeDefault`, `skills`, `memorySearch`, etc., but it does not define `compaction` or `contextPruning`. By contrast, `AgentDefaultsConfig` still owns those keys in `src/config/types.agent-defaults.ts`.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '1,180p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Runtime schema for `agents.list[]` does not accept the requested keys:** `AgentEntrySchema` enumerates the allowed per-agent config keys for `agents.list[]` and includes `skills`, `memorySearch`, `contextLimits`, `heartbeat`, `subagents`, `embeddedPi`, `sandbox`, `tools`, and `runtime`, but no `compaction` or `contextPruning`. Current main therefore has no schema-level support for the PR’s feature.
