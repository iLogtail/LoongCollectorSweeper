## Summary

Keep open. Current main still exposes the exact marker as model replay text and has no source/test evidence that literal user input containing `[image data removed - already processed by model]` is escaped, filtered, or hidden from Kimi Coding/WebChat. Related #66702 is open and explicitly tracks a broader same-family stale-image contamination problem, but it does not supersede this exact Kimi Coding literal-marker recursion report.

## Best Possible Solution

Keep the issue open and fix it in the agent replay/user-visible text boundary. A conservative path is to stop using a human-natural bracket marker that models can mirror, or wrap old image-prune placeholders in a hidden/non-user replay representation; add a focused regression test for a user message whose literal text is `[image data removed - already processed by model]` through the Kimi Coding/WebChat path, and coordinate with the still-open #66702 work for stale image/history contamination.
