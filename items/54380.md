## Summary

Keep PR #54380 open. Current main still does not classify `application/x-cfb` or `application/msword` as binary in the media-understanding MIME classifier, and the linked bug #54176 remains a real core media pipeline issue. The PR is still useful as the freshest narrow branch, though its regression test should be strengthened per the Greptile review before merge.

## Best Possible Solution

Keep #54380 open as the likely canonical fix path for #54176. Before merge, update the regression test to use `createMediaDisabledConfigWithAllowedMimes([mime])` for the two legacy Office MIME types, keep the production MIME additions scoped to `isBinaryMediaMime`, then run the changed media-understanding gate. Older duplicate PRs can be cleaned up after maintainers choose this canonical branch.
