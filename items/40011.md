## Summary

Current `main` does not implement per-signal OTLP endpoints. The diagnostics OTEL service still reads only the global `diagnostics.otel.endpoint` / `OTEL_EXPORTER_OTLP_ENDPOINT` and derives `v1/traces`, `v1/metrics`, and `v1/logs` from that single base URL, while config schema and docs expose only the global endpoint.

## Evidence
