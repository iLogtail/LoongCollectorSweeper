## Summary

Current `main` does not contain this fix. The cron announce path still ignores `isReasoning`, still falls back to `payloads[0]?.text`, and the embedded runner still drops `isReasoning` when mapping reply items into returned payloads, so the PR remains materially useful and should stay open for maintainer judgment.
