## Summary

Keep #55372 open. Current main has improved restart-sentinel routing and wake behavior, including webchat delivery-context capture, but it still lacks the requested first-class option to suppress only the outbound restart notification while preserving the wake/continuation path.

## Best Possible Solution

Add a first-class config option, preferably under a gateway-owned restart-sentinel/restart-recovery config surface, that gates only deliverRestartSentinelNotice. Preserve sentinel consumption, enqueueRestartSentinelWake, requestHeartbeatNow, durable continuation queueing, and BOOT.md/internal-hook startup behavior. Update exported config types, zod/runtime schema, schema help/docs, and add tests proving notify=false does not enqueue or deliver outbound restart notices but still wakes and drains continuations.
