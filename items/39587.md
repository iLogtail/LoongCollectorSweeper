## Summary

Keep open. Current `main` still routes `openclaw docs` through the live docs search tool, and the public docs search endpoint still returns HTTP 500 for the reported search queries. The linked PR #39632 was closed unmerged and only addressed a missing-`mcporter` error-message path, not the wrong/no-result search behavior.

## Best Possible Solution

Keep #39587 open until maintainers restore the live docs search endpoint/index and verify `openclaw docs` returns relevant results for queries such as `browser extension` and `Skills`, or intentionally replace the CLI with a repo-backed/search-provider-backed implementation and update the docs accordingly.
