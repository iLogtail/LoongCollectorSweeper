## Summary

Keep #71728 open. Current main still has adjacent truthfulness mechanisms, but not the RFC's requested structured claim-level confidence/source metadata on assistant output or autonomous completion reporting for unverified claims. The item is fresh, coherent, non-maintainer-authored, and has no protected labels, so there is no conservative close reason.

## Best Possible Solution

Keep the RFC open for maintainer design. The strongest path is an additive core or SDK-visible trust-signal contract for agent output: claim/evidence metadata where available, explicit known-vs-inferred wording guidance, and autonomous completion/report fields for unverified claims. Existing memory citations, memory-wiki claims, prompt guidance, and untrusted internal-event framing should be reused as inputs, not treated as the complete solution.
