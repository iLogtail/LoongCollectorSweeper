## Summary

Keep open. Issue #71728 is a fresh, coherent RFC asking for claim-level truth/confidence signaling across agent output and autonomous completion reporting. Current main still only has adjacent mechanisms: prompt guidance to verify mutable facts, memory-search citations, memory-wiki claim metadata, input provenance, and internal task-completion context marked as untrusted. None of those implement confidence tiers or source attribution for every factual claim, nor a completion-event field for unverified claims.

## Best Possible Solution

Keep the RFC open for maintainer product/design judgment. A credible path would define a core or SDK-visible trust signal for agent output, including claim-level source/evidence metadata where available, known-vs-inferred confidence semantics, and autonomous completion-report fields for unverified claims. Existing memory citations and memory-wiki claims can be reused as inputs, but they are not sufficient as the whole solution.
