## Summary

本 Issue 提议新增基于 C++ 的原生 Kafka Flusher 插件。当前 main 分支尚未包含该实现，且 C++ 插件因深度依赖主仓库 CMake 构建系统与底层 Pipeline 接口，通常需在主仓库内开发集成。讨论区已提出与现有 kafka_v2 在 Zstd 压缩下的性能对比需求，表明该特性具备明确的性能优化场景价值。因尚处意向征集与技术设计阶段，无关联 PR 或替代工单，保守保持开启以待维护者架构评估与社区贡献。

## Best Possible Solution

保持开启以待维护者技术路线评估。建议作者或贡献者补充与现有 kafka_v2 插件的性能对比基准（重点关注 CPU、内存与吞吐），并提供简要技术设计草案（如 C++ Kafka 客户端选型、配置结构、V2 Pipeline 集成方式）。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
