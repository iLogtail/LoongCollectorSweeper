## Summary

本 Issue 提议新增基于 C++ 的原生 Kafka Flusher 插件以优化日志投递性能。当前 main 分支已存在 kafka_v2 插件，但尚无原生 C++ 实现。C++ 插件深度依赖主仓库 CMake 构建链与核心 Pipeline 接口，无法通过外部插件系统独立承载，必须在主仓库内开发集成。讨论区已明确与现有插件在 Zstd 压缩下的性能对比需求，目前处于技术设计与基准测试准备阶段，无关联 PR 或替代工单。属于有效且需主仓库集成的性能增强请求，符合保守保留原则。

## Best Possible Solution

保持开启以待维护者技术路线评估。建议贡献者补充与现有 kafka_v2 插件在 Zstd 压缩等场景下的 CPU/内存/吞吐基准测试数据，并提供 C++ Kafka 客户端选型（如 librdkafka）与 V2 Pipeline 集成设计草案。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
