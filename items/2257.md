## Summary

本 Issue 提议新增基于 C++ 的原生 Kafka Flusher 插件。当前 main 分支主要提供 kafka_v2 实现，尚未合并请求的原生 C++ 版本。讨论区已提出针对 Zstd 压缩场景的性能对比诉求，属于有效的核心插件扩展提议。无关联 PR 或替代工单，符合保守保留原则，建议保持开启以待维护者技术评估与社区贡献。

## Best Possible Solution

保持开启。建议贡献者补充与现有 kafka_v2 插件在 Zstd 压缩等场景下的 CPU/内存/吞吐基准测试数据，并提供 C++ Kafka 客户端选型（如 librdkafka）与 V2 Pipeline 集成设计草案。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
