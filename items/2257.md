## Summary

本 Issue 请求新增基于 C++ 的原生 Kafka Flusher 插件。当前 main 分支尚未包含该特定实现，且 C++ 原生插件深度依赖主仓库 CMake 构建系统与底层 Pipeline 接口，不适合转为外部生态插件。讨论区已有关于 Zstd 压缩场景下 CPU 性能对比的合理询问，表明该需求具备明确的性能优化价值。因缺乏具体技术设计与基准数据，需维护者进行架构评估后方可推进，保守保持开启。

## Best Possible Solution

保持开启以待维护者技术路线评估。建议作者或贡献者补充与现有 kafka_v2 插件的性能对比基准（重点关注 CPU、内存与吞吐），并提供简要技术设计草案（如 C++ Kafka 客户端选型、配置结构、V2 Pipeline 集成方式）。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
