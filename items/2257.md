## Summary

本 Issue 请求新增基于 C++ 的原生 Kafka Flusher 插件。当前 main 分支仅存在 kafka_v2 实现，尚无原生 C++ 版本。该需求明确，属于有效的核心插件扩展，需与主仓库 CMake 构建链及 Pipeline 接口深度集成，无法通过外部插件系统独立承载。讨论区已提出与现有插件的性能对比需求，无关联 PR 或替代工单，符合保守保留原则，建议保持开启以待维护者技术评估与社区贡献。

## Best Possible Solution

保持开启以待维护者技术路线评估。建议贡献者补充与现有 kafka_v2 插件在 Zstd 压缩等场景下的 CPU/内存/吞吐基准测试数据，并提供 C++ Kafka 客户端选型（如 librdkafka）与 V2 Pipeline 集成设计草案。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
