## Summary

本 Issue 提议新增基于 C++ 的原生 Kafka Flusher 插件以优化特定压缩算法（如 Zstd）下的性能。当前 main 分支已提供 kafka_v2 插件，但社区对原生实现的性能收益仍有明确探讨诉求。该需求属于有效的核心插件扩展提议，尚无关联 PR 或替代工单，且讨论区已提出基准测试建议。符合保守保留原则，建议保持开启以待贡献者补充性能对比数据、C++ 客户端选型与 V2 Pipeline 集成设计后提交 PR。

## Best Possible Solution

保持开启。建议贡献者补充与现有 kafka_v2 插件在 Zstd 等压缩场景下的 CPU/内存/吞吐基准测试数据，并提供 C++ Kafka 客户端选型（如 librdkafka）与 V2 Pipeline 集成设计草案。待维护者确认核心 API 兼容性与主仓库集成方案后，可提交 PR 进入代码审查流程。
