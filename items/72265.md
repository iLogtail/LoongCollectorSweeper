## Summary

Keep issue #72265 open. Current main still has the README quick start pattern the issue reports: daemon install followed by a foreground gateway command, without splitting daemon mode from foreground/debug mode. The docs have partial related coverage elsewhere, but the Node setup/troubleshooting pages still do not give a clear service-vs-shell Node runtime check ladder. The provided related items are title-overlap only and do not supersede this README/Node docs request.

## Best Possible Solution

Keep the issue open and make a docs-only patch: split the README TL;DR into daemon mode (`openclaw onboard --install-daemon` then `openclaw gateway status`) and foreground/debug mode (`openclaw gateway stop` only when intentionally disabling the managed service, then `openclaw gateway --port 18789 --verbose` or `openclaw gateway run`). Add a Node troubleshooting section that compares shell and service runtime paths using the existing status output, `which node`, `which openclaw`, `node -v`, `npm -v`, and Linux `systemctl --user cat openclaw-gateway.service`, and mention restarting/reinstalling the gateway service after Node upgrades or version-manager changes. Also update the README logging link to the canonical gateway logging page if maintainers agree.
