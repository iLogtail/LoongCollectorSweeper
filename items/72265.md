## Summary

Keep issue #72265 open. The current README still shows `openclaw onboard --install-daemon` followed by a foreground `openclaw gateway --port 18789 --verbose` in the same TL;DR block, and the requested Node service-runtime verification checklist is not present in the Node or gateway troubleshooting docs. The QEEK-AI comment describes the desired docs changes, but those changes are not in current `main`.

## Best Possible Solution

Update the README TL;DR to split daemon mode from foreground/debug mode, keep the beginner docs link, and add a focused service-runtime verification section to `docs/install/node.md` plus the gateway troubleshooting ladder. The docs should explain that shell Node and service Node can differ with nvm/fnm/mise/n or after upgrades, and should tell users to restart or reinstall the gateway service after changing Node.
