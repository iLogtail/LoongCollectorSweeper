## Summary

Keep #25883 open. Current main still does not implement the proposed Redis/BullMQ-style distributed job orchestration or horizontal worker coordination; the relevant cron, task, and command-queue surfaces remain Gateway-local, file/SQLite-backed, and in-process. The linked ScalyClaw prototype and the existing 2026-04-26 maintainer/Codex comment are useful design evidence, not an implementation on main.

## Best Possible Solution

Keep #25883 open for maintainer design review. The best path is to decide whether OpenClaw should support a core distributed scheduling/job abstraction, then either implement a versioned/plugin-capable queue backend with worker claiming, retry/backoff, dead-letter/failure-state, cron, visibility, and shared state contracts, or document horizontal orchestration as an external deployment pattern. The ScalyClaw prototype should remain linked as prior art.
