## Summary

Keep open. Current main has improved Telegram topic threading when the message tool targets the current chat or omits a target, but the reported failure mode is an explicit user-id target in a Telegram forum topic. That path is still preserved as a DM target and the Telegram auto-thread resolver deliberately declines to inject the topic thread when the target chat differs from the current topic chat.

## Best Possible Solution

Keep the issue open and fix the current-reply attachment workflow rather than silently rewriting explicit DM targets. The safest product path is to expose a first-class “reply with attachment/media” route, or update the message-tool prompts/schema so same-turn file delivery to the current conversation can omit target or use the topic-qualified current chat target. Add a Telegram forum-topic regression test for a file/media send from a topic, including the model-facing target behavior, to prove files stay in chat_id + thread_id without breaking intentional DMs.
