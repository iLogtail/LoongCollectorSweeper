## Summary

Keep open. Current main has partial support for topic-safe attachment replies via normal MEDIA reply delivery and message-tool auto-threading when the target is the current chat, but it does not fix the reported explicit-target failure: if the model sends `message(action=send, target=<user_id>, media=...)`, the explicit target is preserved and Telegram auto-threading returns no topic thread for a different chat id, so the send still goes to the DM target.

## Evidence

- **Explicit message target is preserved:** The message action normalizer only infers `toolContext.currentChannelId` when no explicit `target`/legacy destination is present. An explicit user target is kept and then mapped to `to`.
  - file: [src/infra/outbound/message-action-normalization.ts:24](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/infra/outbound/message-action-normalization.ts#L24)
  - command: `nl -ba src/infra/outbound/message-action-normalization.ts | sed -n '20,46p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Telegram auto-threading skips different chats:** `resolveTelegramAutoThreadId` returns undefined when the message-tool destination chat id differs from the current channel chat id, which is exactly the reported `target=user_id` from a forum topic case.
  - file: [extensions/telegram/src/action-threading.ts:16](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/extensions/telegram/src/action-threading.ts#L16)
  - command: `nl -ba extensions/telegram/src/action-threading.ts | sed -n '4,24p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Thread id is only applied when resolver returns one:** The outbound message action path writes `actionParams.threadId` only from an explicit threadId or the channel `resolveAutoThreadId` result; when Telegram returns undefined for a user DM target, no topic thread is applied.
  - file: [src/infra/outbound/message-action-threading.ts:26](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/infra/outbound/message-action-threading.ts#L26)

This does not prevent the explicit message-tool DM target described in the issue.
