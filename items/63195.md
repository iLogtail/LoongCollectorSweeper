## Summary

Keep open. This is a recent cross-surface continuity report, and current main shows only partial safeguards for Control UI and TUI history/session event races. I did not find a direct shipped fix or canonical superseding item for the broader intermittent loss across Control UI/WebChat and TUI; the linked #63229 remains related but narrower and open.

## Best Possible Solution

Keep this issue open until maintainers can reproduce or disprove the current-main failure with focused gateway logs and cross-surface traces. The likely path is to correlate it with #63229, then add targeted regression coverage around gateway session store updates, `sessions.changed`/`session.message` delivery, Control UI reconnect/history reload behavior, and TUI concurrent-run history refreshes. If #63229 proves to be the whole root cause, close this issue only after that narrower fix lands and validates the Control UI plus TUI continuity symptoms.

## Evidence

- **Issue discussion and related context:** The provided issue body reports repeated, non-deterministic session/chat continuity loss across Control UI/WebChat and TUI after normal messages, tool execution, reconnect-like moments, and session switching. It also says pinned token/session URLs and disabling a local LaunchAgent did not fully resolve the issue. The only timeline item links #63229, which is open and frames gateway timeout/fallback cascades as related rather than a complete duplicate.
