## Summary

Keep open. Current main routes Discord `delete` through the normal messaging action path and the low-level delete call only needs `channelId` plus `messageId`, so I found no checked-in DM-specific guard to justify the reported failure. But there is no DM-delete regression test, the exact reported errors do not appear in the current source, and I could not perform an authenticated live Discord reproduction from this read-only review. That leaves the report plausible but not strongly closeable.
