## Summary

Keep open. Current main still lacks the requested public async plugin hook that can mutate or suppress a tool result after execution and before transcript/model visibility. Main has async tool-result middleware, but it is a trusted bundled-plugin seam only; public hooks remain async/void `after_tool_call` plus synchronous `tool_result_persist`. The related PR #64907 is open and narrower: synchronous, text-only, and explicitly not the async validation path requested here.

## Best Possible Solution

Keep #62123 open until OpenClaw either adds a documented, trust-gated public async hook for post-execution/pre-transcript tool-result mutation, or maintainers explicitly decide the capability must remain bundled-only. A complete design should define ordering relative to `after_tool_call`, bundled `registerAgentToolResultMiddleware`, `tool_result_persist`, and transcript caps; specify failure/block semantics; and add runtime tests proving the model/transcript only sees the transformed or suppressed result.
