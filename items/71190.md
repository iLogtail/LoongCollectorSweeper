## Summary

Keep #71190 open. Current main still prepends the entire orphan leaf when it is not already present in the next prompt, and I found no marker/abort-note stripping or regression coverage for an already-accumulated queued block. The reporter's follow-up is also relevant: a pure no-op merge would prevent snowballing but would drop legitimate cross-channel orphan text because the caller removes the leaf after repair.

## Best Possible Solution

Patch the transcript repair helper to preserve legitimate orphan content while normalizing away previously injected queued markers and abort-note prefixes before merging. Add focused tests for repeated orphan repair with an already-queued block, plus a cross-channel preservation case so the no-op regression stays covered.
