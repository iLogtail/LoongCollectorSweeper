## Summary

Current main still has the reported mismatch: the message tool `send` schema advertises `buffer`, `filename`, `contentType`, and `mimeType`, but the shared send path only normalizes media/path-style hints, attachment hydration excludes `send`, and the Telegram runtime forwards only `mediaUrl` into `sendMessageTelegram`. The related PRs in the provided timeline proposed fixes for #41779 but were closed unmerged, and their behavior is not present in the checked-out main.
