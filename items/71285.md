## Summary

Keep #71285 open. Current main still has no `memorySearch.ignorePaths`/exclude config, the memory indexer still recursively walks `memory/` without a dreaming-directory exclusion, and the watcher only has a hardcoded dependency-dir ignore list that does not include `dreaming`. Related dreaming promotion fixes reduce durable-memory contamination, but they do not solve memory_search indexing or watcher scope.

## Best Possible Solution

Keep the issue open and implement a real memory-search ignore contract if maintainers want this behavior: add a schema/type/docs field, resolve default plus per-agent patterns, apply it consistently to `listMemoryFiles`, watcher setup, and any QMD/native indexing scope, include it in the reindex scope hash, and add focused tests for `memory/dreaming/**`, dependency dirs, extra paths, and watcher/index parity. Maintainers should separately decide whether `memory/dreaming/` should be opt-in ignored as requested or a default exclusion.
