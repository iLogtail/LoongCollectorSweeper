## Summary

Keep open. Current `main` does not implement a `memorySearch.ignorePaths` config, and the builtin memory indexer/watcher still traverses `memory/` broadly enough to include dream-report files under `memory/dreaming/`.

## Evidence

- **No ignorePaths in memorySearch config contract:** `MemorySearchConfig` exposes `extraPaths` but no exclusion field, and the runtime schema matches that shape.
  - file: [src/config/types.tools.ts:336](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/types.tools.ts#L336)
  - command: `nl -ba src/config/types.tools.ts | sed -n '330,430p'; nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '624,680p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Resolved config only merges extraPaths:** `resolveMemorySearchConfig`/`mergeConfig` normalizes `sources` and `extraPaths`, but there is no ignore/exclude path handling to pass downstream.
  - file: [src/agents/memory-search.ts:201](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/memory-search.ts#L201)

  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:81](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/memory-core/src/memory/manager-sync-ops.ts#L81)
