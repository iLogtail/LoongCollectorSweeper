## Summary

Keep #71285 open. Current main still has no `memorySearch.ignorePaths`/exclude-pattern config, the builtin memory indexer recursively scans `memory/` for Markdown while only skipping root-memory auxiliary paths, and the watcher ignore list is a fixed set that does not include `dreaming`. The related merged dreaming work blocks promotion/self-ingestion, but it does not solve search-index or watcher exclusion for `memory/dreaming/**`; the earlier `ignorePaths` PR #6362 was closed unmerged.

## Best Possible Solution

Keep the issue open and implement a single memory-search exclusion contract if maintainers want the configurable path: add a typed/schema-documented `agents.*.memorySearch.ignorePaths` or similarly named field, resolve it alongside `extraPaths`, apply the same matcher to builtin `listMemoryFiles()`, the builtin chokidar watcher, and the QMD managed collection/update path where feasible, then add focused tests for `memory/dreaming/**`, default dependency-directory skips, and backend/watch/index consistency. If maintainers decide dreaming reports should never be searchable, a narrower core fix would hard-exclude generated `memory/dreaming/**` from default memory roots and document that boundary while preserving explicit `extraPaths` behavior intentionally.
