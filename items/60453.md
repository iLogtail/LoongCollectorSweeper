## Summary

Keep open. Current main appears to cover the likely config-form rendering failure for untyped/dynamic channel schemas, but I did not find strong evidence that the Gmail account status problem is fixed. The generic channel status path still only propagates runtime `running` state when the plugin supplies a status adapter, and the reported `openclaw-gmail` plugin is not present in this repository to verify that it does so.

## Best Possible Solution

Keep this open as a narrow core/plugin-contract triage item. Treat the schema rendering side as likely addressed, then resolve the remaining status mismatch either by making `src/channels/plugins/status.ts` merge runtime lifecycle fields into the generic fallback or by updating/linking the Gmail plugin to provide `status.buildAccountSnapshot`/`buildChannelSummary` using the SDK status helpers and a setup entry for read-only status surfaces.
