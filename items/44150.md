## Summary

Keep open. Current main still does not expose transcript messages in the direct `before_compaction` plugin hook payload. The hook event type allows `messages`, but the current direct compaction path builds metrics from `currentMessages` and then drops the array before calling `runBeforeCompaction`. The PR’s branch also needs updating for the current `compaction-hooks.ts` refactor and should address the review concern about passing a live mutable `session.messages` reference.

## Best Possible Solution

Keep the PR open, but update it for current main by threading a defensive pre-compaction message snapshot through `buildBeforeCompactionHookMetrics`/`runBeforeCompactionHooks`, widening the local `CompactionHookRunner` payload type to include `messages`, and adding focused tests that assert the exact pre-compaction contents remain stable after compaction. Docs can remain minimal unless maintainers want to document payload fields publicly.
