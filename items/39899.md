## Summary

Keep PR #39899 open. Current main prevents the common replay path by stripping unpaired surrogate code units from bundled text transports and dropping failed assistant turns before replay, but it still lacks the PR's surrogate-specific detector, friendly error formatting, and auto-reset branch. Source inspection also shows tool-call argument replay is not covered by the text sanitizer, so the PR is not safely obsolete from current-main evidence alone.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main and add a focused regression test that injects an unpaired surrogate into the remaining plausible replay surfaces, especially provider-error and tool-call-argument paths. If the test still reaches the runner error path, implement the smallest surrogate detector, friendly message, and session reset branch consistent with the existing role-ordering and Gemini corruption handling; if the test proves every supported path is already sanitized, close with that proof or replace the PR with a narrower sanitizer hardening patch.
