## Summary

Keep open. Current main has a useful preventative sanitizer for several HTTP transport builders and now drops failed assistant turns before transport replay, but it still does not implement this PR's surrogate-specific detector, friendly formatting, or session reset path. There is also a current core OpenAI WebSocket path that carries raw transcript/system text into request payloads without the transport sanitizer, so the PR is not safely obsolete.

## Best Possible Solution

Keep the PR open until maintainers choose the final product shape: either add surrogate sanitization to every core payload path, especially OpenAI WebSocket, or add the PR's surrogate-specific detector, friendly error text, and tested session-reset recovery. A narrow regression test should cover a transcript containing a lone surrogate and prove the next turn does not loop.
