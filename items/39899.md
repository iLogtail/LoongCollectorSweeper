## Summary

Keep PR #39899 open. Current main has useful partial mitigation for unpaired surrogates in normal text transport payloads and failed assistant-turn replay, but it still lacks the PR's surrogate-specific detector, friendly error formatting, and runner-level session reset. Source inspection also shows plausible non-text payload paths, especially replayed tool-call arguments, still bypass the current text sanitizer.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main and add focused regression coverage for surrogate-bearing provider error formatting, runner reset behavior, OpenAI Responses tool-call arguments, and Anthropic tool_use input replay. If those tests show every supported path is already sanitized, replace the PR with a narrower sanitizer-hardening patch or close with that proof; otherwise implement the smallest surrogate detector, friendly message, and session-reset branch consistent with the existing role-ordering and Gemini-corruption recovery.
