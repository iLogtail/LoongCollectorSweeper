## Summary

Keep open. Current main partially covers startup-failure diagnostics through diagnostic stability bundles, but it does not ship this PR's `gateway watchdog` command, external watchdog script, or persistent outage notification/restart loop. The PR also still has concrete implementation risks, while related rescue-watchdog PR #46502 is open/unmerged and not a clean shipped supersession.

## Best Possible Solution

Keep #53716 open for maintainer judgment or contributor follow-up. The best path is to either rework this PR into a smaller, tested watchdog/notification slice that fits current diagnostics and packaging, or explicitly supersede it from the product tracker once #46502 or another canonical rescue-watchdog implementation lands. Do not close it as implemented because current main still lacks the persistent watchdog behavior requested by #53684.
