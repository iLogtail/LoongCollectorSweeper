## Summary

Keep open. Current `main` has internal `sessionChannelId` session metadata for Discord voice sessions, but it does not expose `voice.autoJoin[].sessionChannelId` in the config contract or docs, `autoJoin()` still only joins each configured voice `channelId`, and `processSegment()` still calls ingress with `deliver: false`, so the requested paired text-channel transcript routing is not implemented.

## Evidence

- **Config type only allows guildId + voice channelId:** `DiscordVoiceAutoJoinConfig` defines only `guildId` and `channelId`; there is no optional `sessionChannelId` field in the public config surface.
  - file: [src/config/types.discord.ts:121](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/types.discord.ts#L121)

- **Auto-join code ignores any companion text-channel concept:** `autoJoin()` reads `discordConfig.voice?.autoJoin` and calls `join({ guildId, channelId: entry.channelId })` for each entry, so startup joins are still keyed only by the configured voice channel.
  - file: [extensions/discord/src/voice/manager.ts:356](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/discord/src/voice/manager.ts#L356)
