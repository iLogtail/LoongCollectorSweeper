## Summary

Keep #59149 open. The request is concrete, recent, and supported by a +1 comment plus open PR #59170, and current main still only exposes `tools.sessions.visibility` and `tools.agentToAgent` as global runtime policy. The per-agent `agents.list[].tools` schema/type surface does not include `sessions` or `agentToAgent`, and the session tool access helpers/callers resolve visibility and A2A policy from `cfg.tools` without threading requester-agent tool overrides.

## Best Possible Solution

Keep the issue open as the canonical product request unless maintainers prefer to close it in favor of PR #59170. The likely core path is to add a backwards-compatible per-agent schema/type surface for session visibility and A2A policy, resolve the requester agent from the session key, pass requester-agent tools into list/history/send/status access checks, document the precedence rules, and cover the manager/lead/specialist access matrix with focused tests.
