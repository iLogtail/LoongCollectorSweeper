## Summary

Keep open. Current main still has the redundant inbound context finalization that this PR targets: wrapper setup finalizes for silent-reply policy, hook setup can finalize again, and the wrapper then calls dispatchInboundMessage(), which finalizes again before dispatch. The PR is small, focused, unmerged, and adds the missing call-count guardrail rather than being obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review or merge an equivalent scoped change: finalize the inbound context once in the wrapper paths, route the shared dispatch body through a private FinalizedMsgContext helper, preserve the public dispatchInboundMessage() input contract, and keep the added unit tests that lock wrapper finalization to one call.
