## Summary

Keep open. Current main still ships the bundled xurl skill, but its SKILL.md does not contain mandatory confirm-before-write or audit-log instructions, and its rate-limit guidance is only a generic 429 note. Generic skill prompting and exec-approval docs do not implement the requested xurl-specific semantic safety flow. The related wacli ClawSweeper report supports the same boundary: generic exec guardrails are not a substitute for service-specific send/post policy in the skill instructions.

## Best Possible Solution

Keep #56897 open and make a focused bundled-skill update in skills/xurl/SKILL.md: add a clear Safety section that classifies read-only versus write actions, requires explicit user confirmation with the exact target/content before any post/reply/quote/delete/like/repost/bookmark/follow/block/mute/DM/media-post workflow, requires redacted audit entries for write attempts/results, and gives conservative rate-limit/backoff guidance. If maintainers want hard enforcement beyond instructions, track that separately as a broader skill policy/runtime capability rather than closing this xurl documentation request.
