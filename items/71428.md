## Summary

Keep open. Current main has the config-level `plugins.entries.<id>.hooks.allowConversationAccess` path and validation, but it still does not implement the requested manifest-owned `permissions.conversationAccess` capability. Hook enforcement receives only config hook policy, the manifest parser/registry do not carry a permissions field, and docs/tests still describe only the host config opt-in.

## Best Possible Solution

Keep the issue open and implement an additive manifest-level permission contract, likely `permissions.conversationAccess`, through `src/plugins/manifest.ts` normalization, `src/plugins/manifest-registry.ts` records, and the loader/registry hook-policy merge path. Preserve `plugins.entries.<id>.hooks.allowConversationAccess` for backward compatibility and explicit operator override, then update plugin docs and add tests for manifest-only allow, config-only allow, denial without either, and precedence when both are present.
