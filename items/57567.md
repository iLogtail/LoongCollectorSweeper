## Summary

Keep open. Current `main` has partial invalid-config recovery, but the specific Feishu upgrade problem is not clearly fixed: Feishu account config is still strict on `name`, the generic account-promotion migration does not include legacy `botName`, and doctor/preflight only auto-repair a narrow non-JSON-prefix corruption case rather than restoring arbitrary malformed JSON from backups.

## Evidence

- **Feishu account schema still rejects legacy `botName` shape:** `FeishuAccountConfigSchema` is strict and defines `name` as the display-name field; there is no `botName` property in the accepted per-account schema.
  - file: [extensions/feishu/src/config-schema.ts:192](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/feishu/src/config-schema.ts#L192)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '188,240p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Generic account-promotion keys do not include `botName`:** The shared single-account promotion key list includes `name`, `dmPolicy`, `groupPolicy`, and related fields, but not `botName`, so the generic migration path would not rename that legacy Feishu field.
  - file: [src/channels/plugins/setup-promotion-helpers.ts:11](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/channels/plugins/setup-promotion-helpers.ts#L11)
  - command: `nl -ba src/channels/plugins/setup-promotion-helpers.ts | sed -n '1,55p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Doctor migration only seeds `accounts.default` from the shared key list:** `seedMissingDefaultAccountsFromSingleAccountBase` moves top-level single-account values into `channels.<id>.accounts.default`, but it just copies keys chosen by the helper; it does not do any Feishu-specific field rewrite such as `botName -> name`.
  - file: [src/commands/doctor/shared/legacy-config-core-normalizers.ts:121](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/commands/doctor/shared/legacy-config-core-normalizers.ts#L121)

- **Doctor preflight only auto-repairs prefixed-config corruption:** The doctor preflight repair path only calls `recoverConfigFromJsonRootSuffix`, which is the narrow non-JSON-prefix recovery flow; it does not attempt backup restore for arbitrary malformed JSON.
  - file: [src/commands/doctor-config-preflight.ts:84](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/commands/doctor-config-preflight.ts#L84)
