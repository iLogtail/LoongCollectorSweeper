## Summary

Keep open. Current `main` still invokes `before_model_resolve` inside each `runWithModelFallback` candidate attempt, so an unconditional routing override can still pin every retry back to one model and defeat failover. `main` has added `ctx.runId` plus current candidate model fields, which helps plugin-side memoization, but the runtime behavior and the misleading `"this run"` docstring are not fixed.

## Evidence

- **Fallback loop still runs the callback once per candidate:** `runWithModelFallback` resolves candidates, loops over them, and calls `runFallbackAttempt({ run: params.run, ...candidate })` for each iteration.
  - file: [src/agents/model-fallback.ts:715](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/model-fallback.ts#L715)

  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Agent command rebuilds an attempt for each fallback iteration:** `agent-command` increments `fallbackAttemptIndex` and calls `runAgentAttempt(...)` from the `runWithModelFallback` callback, so each candidate becomes a fresh embedded attempt.
  - file: [src/agents/agent-command.ts:877](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/agent-command.ts#L877)

  - file: [src/agents/command/attempt-execution.ts:413](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/command/attempt-execution.ts#L413)

  - file: [src/agents/pi-embedded-runner/run.ts:326](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/agents/pi-embedded-runner/run.ts#L326)

## Risks / Open Questions

- Routing plugins that always return a provider/model override can still collapse the fallback chain onto one failing model on current `main`.
- Because the contract is still per-iteration but undocumented as such, plugin authors may reasonably implement one-shot routing logic and get incorrect failover behavior without noticing.
