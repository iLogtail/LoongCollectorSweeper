## Summary

Keep open. Current main at bc24b547d0888fb716505b07e3f2dceeb8abd203 still has the reported gap: both the CLI picker and gateway/UI models.list path load the full catalog and only apply agents.defaults.models allowlist filtering, not models.mode="replace" filtering from models.providers. The linked PRs #64953 and #64967 are relevant attempts but remain unmerged per the provided GitHub context.

## Best Possible Solution

Keep #64950 open as the canonical bug. Implement a mode-aware shared catalog path for picker and gateway surfaces: when cfg.models?.mode === "replace", use buildConfiguredModelCatalog({ cfg }) while preserving alias/metadata handling and configured synthetic entries; keep merge behavior unchanged. Add focused regression tests for promptDefaultModel(), promptModelAllowlist(), and models.list, then consolidate or close the unmerged duplicate PRs against that implementation.
