## Summary

Keep open. Current main has channel awareness in the main embedded chat flow and trusted inbound metadata, but the linked #21271 discussion identified a slash-command system-prompt gap and current main still has that gap: `resolveCommandsSystemPromptBundle()` builds runtime params with `host/os/arch: "unknown"` and does not pass channel, capabilities, channel actions, or shell into the prompt. The related PR was closed unmerged, so the issue is only partially addressed.
