## Summary

Keep #20837 open. Current main already gives agents channel metadata for normal chat turns and trusted inbound metadata, but the slash-command prompt rebuild path still drops the available command channel and builds runtime info with hardcoded unknown host/os/arch and no channel, capabilities, channelActions, or shell. The linked fix PR #21271 was closed unmerged, so the remaining command-path scope is still actionable in OpenClaw core.

## Best Possible Solution

Keep the issue open and implement the missing slash-command runtime metadata in src/auto-reply/reply/commands-system-prompt.ts. Forward the normalized command channel/channelId, account-aware channel capabilities/actions, shell, host, os, and arch using the same helper pattern as the embedded runner, then add focused regression tests proving /context or /export-session generated from Telegram includes channel=telegram and expected capabilities. Treat broader selective cross-channel reply policy as a separate follow-up if maintainers want that behavior.
