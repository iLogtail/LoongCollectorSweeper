## Summary

Keep #20837 open. Current main exposes channel metadata for normal chat turns and inbound metadata, but the remaining slash-command path is still missing it: `resolveCommandsSystemPromptBundle()` rebuilds `/context` and `/export-session` prompts with hardcoded runtime fields and no channel/capabilities/shell/action metadata. The linked #21271 was closed unmerged, and the latest issue discussion already kept this open for that narrower scope.

## Best Possible Solution

Keep the issue open and implement the missing command-path runtime metadata in `src/auto-reply/reply/commands-system-prompt.ts`: normalize and forward `params.command.channel` or `params.command.channelId`, include account-aware channel capabilities/actions and shell/host/os/arch using the same helpers as the embedded runner, and add focused regression tests proving `/context` or `/export-session` generated from Telegram includes `channel=telegram` and expected capabilities. Track any broader selective cross-channel reply policy separately.
