## Summary

Current main still leaves #37705 actionable. The bundled Bonjour plugin can pass a machine-derived service instance name longer than the 63-byte DNS label limit into @homebridge/ciao, and the related fix PRs #37712 and #37714 were closed unmerged. The supported workaround is `OPENCLAW_DISABLE_BONJOUR=1`, but that does not replace a code fix for the default crash path.

## Best Possible Solution

Keep #37705 open. Add a byte-aware DNS-label sanitizer in the bundled `bonjour` plugin before calling `@homebridge/ciao`, covering both the service instance `name` and the configured mDNS `hostname`, preserving UTF-8 boundaries and retaining a non-empty fallback. Add regression tests for the reported Kubernetes-style 64-byte service label, longer hostnames, exact-boundary labels, and multibyte names. Also clarify in any maintainer response that the current workaround is `OPENCLAW_DISABLE_BONJOUR=1`, not `CLAWDBOT_DISABLE_BONJOUR=1`.
