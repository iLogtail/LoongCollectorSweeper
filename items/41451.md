## Summary

Keep open. Current `main` still has the exact gap the issue describes: `memory-lancedb` defaults every agent to the same LanceDB path, stores everything in one `memories` table, does not persist an `agentId` on entries, and does not scope recall/forget/auto-capture by agent.

## Evidence

- **Shared default database path:** `memory-lancedb` still resolves the default database path to `~/.openclaw/memory/lancedb`, so agents share one backing DB unless users manually override config. The same shared path is surfaced as the schema default/placeholder.
  - file: [extensions/memory-lancedb/config.ts:28](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/memory-lancedb/config.ts#L28)
  - command: `nl -ba extensions/memory-lancedb/config.ts | sed -n '1,280p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Schema and storage lack agent namespace:** `MemoryEntry` still has only `id`, `text`, `vector`, `importance`, `category`, and `createdAt`. `MemoryDB` still opens/creates a single hard-coded `memories` table and inserts/searches/deletes rows without any agent field or filter.
  - file: [extensions/memory-lancedb/index.ts:30](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/memory-lancedb/index.ts#L30)
  - command: `nl -ba extensions/memory-lancedb/index.ts | sed -n '1,240p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **User-facing tool paths are unscoped:** `memory_recall` calls `db.search(...)` directly, `memory_store` dedupes and stores directly, and `memory_forget` searches/deletes directly. None of those tool handlers pass or derive an agent namespace.
  - file: [extensions/memory-lancedb/index.ts:349](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/memory-lancedb/index.ts#L349)

  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Hook paths are also unscoped despite agent-aware hook context existing:** Auto-recall and auto-capture hooks call `db.search(...)` / `db.store(...)` without any agent scoping, and the handlers only accept `(event)` even though plugin hook context includes optional `agentId`/`sessionKey` fields.
  - file: [extensions/memory-lancedb/index.ts:582](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/memory-lancedb/index.ts#L582)
