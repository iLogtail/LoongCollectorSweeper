## Summary

Keep #65212 open. Current main still resolves QMD custom paths as directory roots with a default markdown glob and has no direct-file normalization or regression test. The PR is a focused, plausible bug fix, though it should address the remaining literal filename/glob edge case before merge.

## Best Possible Solution

Keep the PR open and finish the direct-file path support in `resolveCustomPaths`: normalize existing file targets to parent directory plus an exact literal filename pattern, preserve existing directory behavior, add regression coverage for ordinary filenames and glob-metacharacter filenames, and update user-facing memory/QMD docs if direct-file paths are now supported configuration.
