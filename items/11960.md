## Summary

Keep #11960 open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still does not persist bundled session-memory on automatic daily/idle rollovers: the bundled hook is registered for command:new/command:reset and the stale rollover path archives the old transcript and fires plugin session_end/session_start, but does not emit the internal reset hook or before_reset path. The secondary hook-config schema mismatch is fixed on main. The provided issue context shows active implementation PRs #61675 and #71792, but both are open/unmerged, so this issue remains the active user-facing tracker.

## Best Possible Solution

Keep #11960 open until OpenClaw ships the daily/idle auto-rollover memory persistence path on main, likely by landing #71792 or an equivalent fix that emits the existing reset hook contract with previousSessionEntry for stale rollovers and covers it with regression tests. Treat the hook config schema portion as already fixed, but do not close the issue while bundled session-memory still misses automatic reset persistence.
