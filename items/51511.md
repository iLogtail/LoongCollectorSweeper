## Summary

Keep open. Current main still contains Cursor ACP routing guidance and the bundled ACP router skill maps plain-language Cursor harness intent to `sessions_spawn(runtime="acp")`; Feishu also supports bound ACP conversations. I found no current source, test, or history evidence proving that ordinary pasted Cursor-related links are ignored and cannot spawn or bind a new ACP session from a custom Feishu agent.

## Best Possible Solution

Keep the issue open and add a focused fix around explicit intent: Cursor-related URLs or support-issue links should remain ordinary user content unless the user clearly asks to run work in Cursor/Cursor CLI/ACP or uses `/acp`. The likely implementation spans the system prompt/ACP router guidance and ACP spawn intent tests, plus a Feishu custom-agent regression proving a pasted Cursor URL does not call `sessions_spawn`, does not create/bind an ACP thread/session, and preserves the routed agent workspace context such as SOUL.md, USER.md, and MEMORY.md.
