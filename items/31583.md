## Summary

Keep #31583 open. Current main has generic run-scoped skill env injection, but the reported `GOG_KEYRING_PASSWORD` case is still blocked because sensitive-looking keys from `skills.entries.<skill>.env` are only allowed when declared in skill metadata, and the bundled `gog` skill does not declare that key. The issue discussion also points to open PR #50960 as candidate work, not shipped behavior.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent maintainer-reviewed fix lands. The narrow core fix belongs in `src/agents/skills/env-overrides.ts`: explicitly allow user-configured `skills.entries.<skill>.env` keys such as `GOG_KEYRING_PASSWORD` through the sensitive-name sanitizer while preserving the dangerous host-env blocklist, then add regression coverage for undeclared configured `_PASSWORD`/`_TOKEN` keys and document any separate sandbox env contract decision.

- Sandbox env propagation is documented as host-only today; if maintainers want sandboxed skill env support, that should be tracked as an explicit contract change rather than used to close this host-sensitive-key bug.
