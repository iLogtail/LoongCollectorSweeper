## Summary

Current main still does not fully satisfy #31583. Host exec inherits `process.env`, but the reported `GOG_KEYRING_PASSWORD` value is never injected because the gog skill does not declare that sensitive-pattern env key and the sanitizer only re-allows sensitive keys from skill metadata. Sandbox exec also still builds a clean environment with no skill-env merge path, while docs currently describe `skills.entries.*` injection as host-only.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent narrow fix lands. The fix should allow explicitly user-configured `skills.entries.<skill>.env` keys such as `GOG_KEYRING_PASSWORD` to bypass only the sensitive-name sanitizer while preserving the dangerous host-env blocklist, add regression coverage for an undeclared password/token-style key, and either implement applied skill-env forwarding for sandbox exec or keep the host-only sandbox contract clearly documented.
