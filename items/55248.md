## Summary

Current `main` does not show a maintainer-cleanup basis to close this. The Anthropic replay policy still preserves thinking blocks for Claude 4.x models such as `claude-opus-4-6`, compaction feeds the sanitized transcript directly back into the live session before `session.compact()`, and subagent announce still re-enters the target session through a nested `runAgentStep()`. I found no code path wired on `main` that specifically protects subagent announce from post-compaction Anthropic thinking-block corruption, and no targeted regression test covering this flow.

## Evidence

- **Claude 4.x still preserves thinking blocks:** `shouldPreserveThinkingBlocks()` returns true for `opus-4`, `sonnet-4`, and `haiku-4`, and `buildAnthropicReplayPolicyForModel()` only sets `dropThinkingBlocks` when preservation is false.

  - file: [src/agents/transcript-policy.ts:32](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/transcript-policy.ts#L32)
