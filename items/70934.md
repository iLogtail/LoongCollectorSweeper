## Summary

Keep #70934 open. Current main has made onboarding easier with QuickStart and a modern Crestodian preview, but the default first-run path is still setup/configuration-first rather than the requested value-first flow with templates or a quick useful task before optional personalization.

## Best Possible Solution

Keep the issue open as a product/UX enhancement. The best path is to make first-run onboarding genuinely value-first: default new users into a chat-first bootstrap with a small set of example task templates and one successful useful run, then defer optional identity, theme, persona, channel, and deeper settings customization until after that first success. The implementation likely spans classic CLI onboarding, Crestodian/modern onboarding, Control UI/macOS/mobile entrypoints, and the onboarding docs/tests that define the first-run sequence.
