## Summary

Keep open. Current main has a QuickStart/defaults path and a dashboard/TUI handoff, but it does not implement the requested value-first first-run flow with example tasks/templates before optional persona/customization. The existing flow still completes setup/auth/gateway/channel/search/skill work before the first chat handoff, and the handoff is a bootstrap/hatching prompt rather than a quick useful task.

## Best Possible Solution

Keep this open for maintainer UX/product triage. The likely path is to extend the first-run onboarding surfaces around `src/wizard/setup.ts`, `src/wizard/setup.finalize.ts`, and the macOS/app onboarding docs so users can see or run a small useful task/template before optional identity, persona, channel, search, and skill customization, while preserving the existing QuickStart defaults and skip flags.
