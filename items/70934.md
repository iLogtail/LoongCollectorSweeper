## Summary

Keep #70934 open. Current main has useful adjacent pieces, including QuickStart defaults, Control UI chat suggestions, and the Crestodian preview, but the default first-run path still configures auth/workspace/gateway/channels/search/skills before the user reaches chat, then runs the bootstrap identity/preference ritual before ordinary use. That does not satisfy the requested value-first onboarding flow.

## Best Possible Solution

Keep the issue open and design a narrow value-first first-run path: offer a small set of safe starter tasks or a guided dashboard/TUI first message as soon as model and gateway basics are viable, then defer bootstrap identity/style questions and optional channel/skill/search setup until after a successful first result. The implementation likely belongs in `src/wizard/setup.ts`, `src/wizard/setup.finalize.ts`, the bootstrap templates under `docs/reference/templates/`, and the Control UI/TUI starter surfaces, with tests for the default QuickStart sequence.
