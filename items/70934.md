## Summary

Keep open. Current `main` has some friction-reduction paths (`quickstart`, `Configure later`), but first-run still routes users through the bootstrap workflow that collects identity/preferences before normal replies, so the requested value-first onboarding flow is not implemented.

## Evidence

- **CLI onboarding already has a lighter setup mode, but only as a partial mitigation:** `quickstart` is documented as using minimal prompts, which shows maintainers have already reduced some setup friction without implementing the issue's broader value-first onboarding request.
  - file: [docs/cli/onboard.md:160](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/cli/onboard.md#L160)

- **macOS onboarding already allows deferring setup entirely:** The current first-run app flow includes a `Configure later` branch, so the repo already supports skipping setup in one path, but that is not the same as showing product value before customization within the normal onboarding flow.
  - file: [docs/start/onboarding.md:39](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/start/onboarding.md#L39)
