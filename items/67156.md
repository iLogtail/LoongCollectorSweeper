## Summary

Keep open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, and the active local skill loader requires a frontmatter-derived `description`; without it, the bundled canvas skill is dropped from discovery/status/prompt entries. The PR is a narrow one-file fix for a real remaining defect, not an implemented, external, incoherent, or safely closeable item.

## Best Possible Solution

Keep the PR or another canonical fix open until current main gains valid `name` and `description` frontmatter for `skills/canvas/SKILL.md`. The best implementation is a minimal metadata-only patch with no `metadata.openclaw.os` hard gate, plus a small bundled-skill frontmatter regression check so canvas cannot silently fall out of skill discovery again.
