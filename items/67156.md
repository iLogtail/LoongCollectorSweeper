## Summary

Keep PR #67156 open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, and the active skill loader drops skill directories that do not provide a frontmatter-derived `description`. The PR's final patch directly addresses that still-present defect and incorporated review feedback by removing the hard Darwin OS gate.

## Best Possible Solution

Keep the PR open and either land it or land a maintainer-equivalent patch that adds minimal `name` and trigger-rich `description` YAML frontmatter to `skills/canvas/SKILL.md`, with optional non-gating metadata such as the emoji. The best follow-up is to expand the bundled skill frontmatter regression test to cover canvas or all bundled skills.
