## Summary

Keep #12047 open. Current main now covers part of the request by downloading Telegram document attachments and feeding text/PDF-capable files into the shared media/file extraction path, but the requested Office/.docx-style document extraction is still missing. Office MIME types are skipped as binary before extraction, the default file MIME set omits Office formats, the file extraction dispatcher only uses document extractors for PDFs, the bundled extractor only declares PDF support, and tests assert Office vendor attachments are skipped.

## Best Possible Solution

Keep this as an OpenClaw issue. Extend the shared media/input-file document extraction path so safe Office document MIME types can reach a document-extractor contract, add a bundled or plugin-owned Office extractor implementation with size limits and untrusted-content wrapping, and cover Telegram attachments plus generic shared extraction with tests and docs. Telegram should continue forwarding MediaPath/MediaType into the shared path rather than parsing documents itself.
