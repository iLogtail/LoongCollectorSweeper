## Summary

Keep #69319 open. Current main still lacks a `skills.uninstall` Gateway RPC in the method registry, handler surface, protocol schema, admin scope map, generated client model surface, and Gateway docs. The PR remains the canonical implementation path for the closed #69299 request, but it needs revision before merge because review comments flagged destructive non-ClawHub deletion and malformed-slug error classification.

## Best Possible Solution

Keep #69319 open, or replace it with a maintainer-owned patch that adds `skills.uninstall` through the Gateway protocol schema, validator exports, method registry, `operator.admin` scope, handler, generated client models, public docs, and focused backend/handler tests. The mergeable version should delete only ClawHub-tracked skill folders or lock entries, preserve idempotent not-installed behavior, treat malformed slugs as client errors, and cover the reviewed edge cases.
