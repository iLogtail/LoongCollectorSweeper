## Summary

Keep PR #71553 open. Current main still drops the WhatsApp group ack reaction participant when the inbound sender is LID-only: `maybeSendAckReaction` only forwards `sender.jid`, while identity normalization deliberately moves `@lid` identities into `sender.lid` and clears `sender.jid`. The PR is a focused contributor bug fix with a matching regression test and is not superseded by current main.

## Best Possible Solution

Keep PR #71553 open for maintainer review or merge. The best fix is the PR's narrow change in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts`: use `sender.jid ?? sender.lid` for WhatsApp group ack reaction participants, with the proposed regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`.
