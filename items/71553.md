## Summary

Keep #71553 open. Current main still builds WhatsApp ack-reaction options from `sender.jid` only, while the inbound identity path normalizes `@lid` participants into `sender.lid` with `sender.jid` unset. The existing send layer already preserves LID participants when one is provided, so the PR is aimed at the right local gap; it just needs to be rebased/adapted to the current ack-reaction handle shape on main.

## Best Possible Solution

Keep the PR open and rebase/adapt it to current main: change `maybeSendAckReaction` to set the reaction participant from `sender.jid ?? sender.lid` while preserving existing JID precedence and direct-message behavior, add the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`, then run the targeted WhatsApp extension test/type/format lanes before merge.
