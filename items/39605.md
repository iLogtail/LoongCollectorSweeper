## Summary

Keep #39605 open. Current main still has the reported bug surface for Discord and Slack: their slash/native command routing paths resolve conversation routes from the provider startup config snapshot, while `resolveAgentRoute` still derives DM session keys from `cfg.session.dmScope`. Telegram has a fresh-runtime-config path and a regression test, but that partial coverage does not fix Discord or Slack. The related PR #39617 remains open and unmerged, so the issue is not implemented or safely superseded on main.

## Best Possible Solution

Keep the issue open and land a focused fix, likely via #39617 or an equivalent maintainer-reviewed patch, that reloads runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests matching the Telegram pattern, proving `dmScope` or route binding changes are honored between slash/native command invocations without recreating the provider.
