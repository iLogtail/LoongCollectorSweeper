## Summary

Keep #39605 open. Current main still routes Discord native commands and Slack slash commands with the monitor startup config object, while route/session-key resolution still derives DM session keys from the supplied `cfg.session.dmScope`. Telegram has a fresh-runtime-config implementation and regression coverage, but the Discord and Slack surfaces named in the issue remain unresolved. PR #39617 remains the canonical open fix attempt in the provided GitHub context.

## Best Possible Solution

Keep #39605 open and land PR #39617 or an equivalent maintainer-reviewed fix that supplies freshly loaded runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests mirroring Telegram’s fresh-config coverage, proving slash/native command routing honors updated `session.dmScope` or route bindings without recreating the provider.
