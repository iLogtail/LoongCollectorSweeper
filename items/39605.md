## Summary

Keep #39605 open. Current main still routes Discord and Slack slash/native command route resolution through startup-captured config, while `resolveAgentRoute` derives direct-message session keys from `cfg.session.dmScope`. Telegram has a fresh-config path and a regression test, but the cross-channel issue is only partially fixed and related PR #39617 is still open in the provided GitHub context.

## Best Possible Solution

Keep #39605 open until the remaining Discord and Slack paths are fixed or PR #39617 is landed/superseded. The narrow product/code direction is to load current runtime config immediately before Discord native route-state resolution and Slack slash-command `resolveAgentRoute` calls, preserve startup config only where command registration needs it, and add Discord plus Slack regression tests for direct-message slash commands with non-main `session.dmScope`.
