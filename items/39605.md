## Summary

Keep #39605 open. Current main at edf40ab6 still routes Discord native commands and Slack slash commands through config objects captured when the monitor starts, while resolveAgentRoute derives direct-message session keys from the supplied cfg.session.dmScope. Telegram has a fresh-runtime-config pattern for native commands, but the Discord and Slack surfaces named in the issue remain unresolved; the provided GitHub context also identifies PR #39617 as the open canonical fix attempt.

## Best Possible Solution

Keep #39605 open and land PR #39617 or an equivalent maintainer-reviewed fix that loads fresh runtime config before Discord native command route resolution and Slack slash command route resolution. Add focused Discord and Slack regression tests proving native/slash commands in DMs honor updated session.dmScope without recreating the provider.
