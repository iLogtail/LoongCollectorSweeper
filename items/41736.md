## Summary

Keep open. Current `main` still uses separate config/auth/runtime layers and still resolves agent models through a per-agent-plus-global-default hierarchy, so the requested single-source-of-truth simplification is not implemented. Part of the report is outdated (`workspace-*/agent.json` is not a current documented config surface), and missing-auth failures are surfaced more clearly now, but that is not enough to close this DX issue as implemented or obsolete.

## Evidence

- **Workspace is explicitly not the config/auth source of truth:** Current docs say the workspace is separate from config and auth, and list `~/.openclaw/openclaw.json` plus per-agent `auth-profiles.json` under `~/.openclaw/` as the live locations. That means the issue's `workspace-*/agent.json` premise does not match current main, but it also confirms the hierarchy is still split rather than unified.
