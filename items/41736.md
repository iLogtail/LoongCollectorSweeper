## Summary

Keep open. Current main has useful mitigations for per-agent model overrides, fallback visibility, and auth status, but it does not implement the issue’s requested single source of truth or remove the multi-location config/auth hierarchy. The fallback behavior the issue complains about is still documented as intentional when fallbacks are configured.

## Evidence

- **Current model selection still centers global defaults:** Docs describe primary selection from `agents.defaults.model.primary`/`agents.defaults.model`, then `agents.defaults.model.fallbacks`, with per-agent defaults only as overrides through `agents.list[].model`. This is not a workspace-only SSOT.
