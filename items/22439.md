## Summary

Keep PR #22439 open. Current main still does not implement the requested schema-backed `agents.defaults.bootstrapTier` or `minimal`/`standard`/`full` bootstrap filtering, and the same-day maintainer-side review in the discussion explicitly kept this PR open alongside canonical issue #22438. Existing `lightContext` support is a different per-run full/lightweight switch, not a configurable tiered bootstrap replacement.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If maintainers accept the direction, land a current-main version that adds schema-backed `agents.defaults.bootstrapTier`, wires tier resolution through bootstrap file resolution and the `bootstrap-extra-files` hook, updates generated config/docs, and keeps focused tests for default behavior plus `minimal`, `standard`, and `full`. If maintainers prefer the existing `lightContext` or model-aware bootstrap direction instead, close it manually after that explicit product decision.
