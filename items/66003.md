## Summary

Keep open. Current `main` still wires `memory-wiki` around one plugin-level vault config resolved at registration time, with ctx-aware factory registration only for `wiki_search`/`wiki_get`. `vault.path` still supports only `~` expansion, not per-agent templating or overrides, and the issue discussion points to an open follow-up PR (#67584) rather than a landed fix.

## Evidence

- **Registration still uses one shared config:** `register(api)` resolves `const config = resolveMemoryWikiConfig(api.pluginConfig)` once, then registers `wiki_status`, `wiki_lint`, and `wiki_apply` directly against that shared config. Only `wiki_search` and `wiki_get` are registered through a `(ctx) => ...` factory that receives `ctx.agentId` / `ctx.sessionKey`.
