## Summary

Keep #59718 open. Current main recognizes custom bind writability in mount resolution, but the normal sandbox file write path still has a global `workspaceAccess === "rw"` gate, so a `:rw` custom bind remains unwritable when `workspaceAccess: "none"`. The request is a concrete core sandbox/file-tool capability gap, not a ClawHub item or a duplicate of the broader workspace-writability issues.

## Best Possible Solution

Keep the issue open and implement a core sandbox file-tool change: authorize writes based on the resolved mount’s writability for custom bind mounts, while keeping the sandbox workspace and agent workspace governed by `workspaceAccess`. Add tests for `workspaceAccess: "none"` with `:rw` and `:ro` binds, path/symlink boundary checks, and apply_patch behavior; update the sandbox docs once the contract is explicit.
