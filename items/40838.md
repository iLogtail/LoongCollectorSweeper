## Summary

Keep #40838 open. Current main already exposes Telegram sender id, username, name, chat type, provider/surface, and mention state to the inbound agent context, but the requested bot/source classification is still not available: `msg.from.is_bot` is read in Telegram update handling paths yet is not preserved into the finalized context or prompt metadata.

## Best Possible Solution

Keep the issue open and implement a narrow Telegram-owned inbound context expansion: preserve `msg.from.is_bot` through the Telegram message pipeline, expose it as a safe typed context field such as `SenderIsBot` or provider-scoped structured metadata, render it in inbound prompt metadata where agents can use it, and add regression tests plus docs for the new context field.
