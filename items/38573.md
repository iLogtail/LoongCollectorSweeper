## Summary

Keep #38573 open. Current main still forces isolated cron deliveries with a resolved thread/topic ID through direct outbound delivery, and the cron type/schema/tool/config/docs surfaces still do not expose an opt-in route for text-only topic/forum completions to use the main-session announce/agent-turn flow. Related #34160 covered a separate subagent group/topic announce path, not this cron route-selection feature.

## Best Possible Solution

Keep this issue open for maintainer product/API work. A complete fix should add a backward-compatible explicit cron delivery route option across CronDelivery, gateway protocol schema, cron tool/CLI/docs, config docs if a global default is chosen, and dispatchCronDelivery; preserve current direct-send behavior by default, keep structured payloads on direct delivery, limit the opt-in to text completions, define fail-closed/direct-fallback semantics, and add regression coverage proving a Telegram topic opt-in reaches the main-session announce/agent-turn path before posting.
