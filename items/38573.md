## Summary

Keep #38573 open. Current `main` still forces isolated cron deliveries with a resolved thread/topic ID through direct delivery, and the cron public type/schema/tool/docs surfaces still do not expose an opt-in route that would let text-only topic/forum completions go through the main-session announce/agent-turn path before posting. Related #34160 fixed a different group/topic subagent announce problem; it does not implement this cron-specific route-selection option.

## Best Possible Solution

Keep this issue open for maintainer product/API work. A complete fix would add a backward-compatible explicit cron delivery route option across `CronDelivery`, gateway protocol schema, cron tool schema/docs, config/docs if chosen, and `dispatchCronDelivery`; preserve current direct-send behavior by default, keep structured payloads on direct delivery, restrict the opt-in to text completions, define fail-closed/direct-fallback semantics, and add regression coverage proving a Telegram topic opt-in reaches the main-session announce/agent-turn path before any channel post.
