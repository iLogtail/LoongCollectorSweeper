## Summary

Keep open: current `main` still hard-codes the bundled Discord skill requirement to `channels.discord.token`, while Discord runtime/config surfaces accept `channels.discord.accounts.<id>.token` for multi-account setups. The issue's reported false negative remains valid, and the referenced fix does not appear to have landed on this checkout.

## Evidence

- **Bundled Discord skill still requires only the single-account path:** The bundled skill frontmatter on current `main` lists `requires.config = ["channels.discord.token"]` and does not mention account-scoped Discord tokens.
  - file: [skills/discord/SKILL.md:4](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/skills/discord/SKILL.md#L4)
