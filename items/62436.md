## Summary

Keep #62436 open. Current main still lacks a true `/v1/chat/completions` passthrough or no-persistence mode: the endpoint resolves a real session key, runs the normal agent command path, and that path still persists session metadata, skills snapshots, and transcript files. The related open issues cover overlapping persistence/session-management problems, but none is an exact canonical replacement for this HTTP gateway passthrough request.

## Best Possible Solution

Keep the issue open and treat it as a core Gateway/API feature request. The best path is either an explicit `gateway.http.endpoints.chatCompletions.mode: "passthrough"` or a documented no-store/ephemeral execution mode for OpenAI-compatible HTTP endpoints that preserves Gateway auth, model selection, and fallback while bypassing session-store, transcript, skills snapshot, system prompt, and tool setup. It should be coordinated with #48159, #55334, and #20934 but not closed as a duplicate today.
