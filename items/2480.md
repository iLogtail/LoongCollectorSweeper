## Summary

本 Issue 询问如何在使用 flusher_kafka_v2 与 custom_single_flatten 转换器时输出纳秒级时间戳。经核查，全局配置 EnableTimestampNanosecond 主要作用于采集端内部 LogEvent 精度，当前转换器序列化逻辑未自动继承该设置，且缺少显式的时间精度控制参数。属于有效的配置咨询与转换器功能/文档增强需求，复现路径清晰，建议保持开启以待维护者评估实现或补充文档。

## Best Possible Solution

建议维护者核查 custom_single_flatten 转换器的时间戳格式化逻辑。若可通过 Convert 配置项（如 TimestampPrecision 或 TimeFormat）指定纳秒输出，请在本 Issue 补充标准配置示例并关闭；若当前实现确实未读取全局纳秒开关或硬编码为秒/毫秒级，建议将其标记为文档补充或转换器功能增强，在后续版本中增加精度控制参数或实现全局配置自动对齐。
