## Summary

本 Issue 询问如何在使用 custom_single_flatten 转换器时输出纳秒级时间戳。经核查，全局配置 EnableTimestampNanosecond 主要控制采集端内部事件精度，当前转换器序列化逻辑未自动透传该设置，且缺少显式的时间精度格式化参数。属于有效的配置咨询与功能增强需求，复现路径清晰，需维护者评估后续实现或文档补充。

## Best Possible Solution

建议维护者或贡献者核查 custom_single_flatten 转换器的时间戳序列化逻辑。若当前实现未读取全局纳秒开关，可在转换器配置中新增 TimestampPrecision 或 TimeFormat 参数以支持纳秒输出；或更新文档明确全局精度开关与转换器输出格式的边界。在功能实现或文档明确前，本 Issue 保持开启以跟踪该增强需求。用户可暂时通过自定义 Processor 在输出前手动处理高精度时间字段作为临时方案。

- **global_config_scope:** EnableTimestampNanosecond 为全局配置项，用于提升内部 LogEvent 时间戳精度，但不同 Flusher/Converter 的序列化行为需单独适配或配置，当前未自动透传至 custom_single_flatten。
  - file: [pkg/common/global.go](https://github.com/alibaba/loongcollector/blob/dd3bd0c0bad657e7f153863f59d5f593f62994c8/pkg/common/global.go)

- **converter_behavior:** custom_single_flatten 转换器在格式化输出时，默认时间戳精度为秒或毫秒，当前实现未自动读取全局纳秒开关，缺少显式精度控制参数。
  - file: [pkg/protocol/converter/converter_custom_single_flatten.go](https://github.com/alibaba/loongcollector/blob/dd3bd0c0bad657e7f153863f59d5f593f62994c8/pkg/protocol/converter/converter_custom_single_flatten.go)
