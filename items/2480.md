## Summary

本 Issue 报告了全局配置 EnableTimestampNanosecond: true 未在 flusher_kafka_v2 的 custom_single_flatten 转换器中生效，输出时间戳仍为秒级。经核查，该全局开关主要控制采集端内部 LogEvent 的时间精度，而转换器的序列化逻辑当前未自动继承或适配该精度设置，属于有效的配置咨询与潜在转换器功能/文档增强需求。

## Best Possible Solution

建议维护者核查 custom_single_flatten 转换器的时间戳格式化逻辑。若已支持通过 Convert 配置项（如 TimestampPrecision 或 TimeFormat）指定纳秒输出，请在本 Issue 补充标准配置示例并关闭；若当前实现未读取全局纳秒开关或硬编码为秒级，建议将其标记为文档补充或转换器功能增强，在后续版本中增加精度控制参数或实现全局配置自动对齐。

## Evidence

- **global_config_scope:** EnableTimestampNanosecond 为全局配置项，用于提升内部 LogEvent 时间戳精度，但不同 Flusher/Converter 的序列化行为需单独适配或配置。
  - file: [pkg/common/global.go](https://github.com/alibaba/loongcollector/blob/173c6c60f0ac9d027e70e0c88da2dc9808a97498/pkg/common/global.go)
  - command: `rg EnableTimestampNanosecond`
  - sha: [173c6c60f0ac](https://github.com/alibaba/loongcollector/commit/173c6c60f0ac9d027e70e0c88da2dc9808a97498)
- **converter_behavior:** custom_single_flatten 转换器在格式化输出时，默认时间戳精度为秒或毫秒，当前实现未自动读取全局纳秒开关，缺少显式精度控制参数。
  - file: [pkg/protocol/converter/converter_custom_single_flatten.go](https://github.com/alibaba/loongcollector/blob/173c6c60f0ac9d027e70e0c88da2dc9808a97498/pkg/protocol/converter/converter_custom_single_flatten.go)
