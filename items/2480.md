## Summary

本 Issue 报告了全局配置 EnableTimestampNanosecond 未在 flusher_kafka_v2 的 custom_single_flatten 转换器中生效的问题。经核查，该全局开关主要控制采集端内部 LogEvent 的时间精度，而转换器的序列化输出格式通常需独立配置或显式继承。当前主仓库代码未见自动对齐该精度的实现或明确文档说明，属于有效的配置咨询与潜在转换器功能/文档增强需求。

## Best Possible Solution

建议维护者核查 custom_single_flatten 转换器的时间戳格式化逻辑。若已支持通过 Convert 配置项（如 TimestampPrecision 或 TimeFormat）指定纳秒输出，请在本 Issue 补充标准配置示例并关闭；若当前实现未读取全局纳秒开关或硬编码为秒级，建议将其标记为文档补充或转换器功能增强，在后续版本中增加精度控制参数或实现全局配置自动对齐。

## Evidence

- **global_config_scope:** EnableTimestampNanosecond 为全局配置项，用于提升内部 LogEvent 时间戳精度，但不同 Flusher/Converter 的序列化行为需单独适配或配置。
  - file: [pkg/common/global.go](https://github.com/alibaba/loongcollector/blob/d3305220b0b8b9bd4339df9441ace92983c2253f/pkg/common/global.go)
  - sha: [d3305220b0b8](https://github.com/alibaba/loongcollector/commit/d3305220b0b8b9bd4339df9441ace92983c2253f)
- **converter_behavior:** custom_single_flatten 转换器在格式化输出时，默认时间戳精度可能为秒或毫秒，且当前实现未自动继承全局纳秒设置，缺少显式精度控制参数。
  - file: [pkg/protocol/converter/converter_custom_single_flatten.go](https://github.com/alibaba/loongcollector/blob/d3305220b0b8b9bd4339df9441ace92983c2253f/pkg/protocol/converter/converter_custom_single_flatten.go)
  - sha: [d3305220b0b8](https://github.com/alibaba/loongcollector/commit/d3305220b0b8b9bd4339df9441ace92983c2253f)
- **issue_reproduction:** 用户提供了完整的配置文件与 Kafka 消费侧实际输出（time 字段为秒级 1764663807），复现路径清晰，非信息不足或陈旧无响应工单。
