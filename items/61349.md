## Summary

Keep PR #61349 open. Current main still lacks the proposed JSON.parse/truncated-stream suppression, and the PR discussion identifies real but unresolved implementation risks rather than evidence that the work is obsolete or already shipped.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best path is to classify malformed/truncated streaming JSON closer to the transport boundary when feasible, then route a structured safe error through both formatAssistantErrorText and sanitizeUserFacingText with regression coverage for #59076/#59080/#60593. If maintainers consolidate duplicate attempts, preserve #61349's useful tests and link it from the canonical JSON-parse leak issue/PR rather than closing as already fixed.
