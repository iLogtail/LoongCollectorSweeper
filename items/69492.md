## Summary

Keep #69492 open. Current main has partial transcript/display hygiene for heartbeat acknowledgements, internal envelopes, and delivery-mirror assistant replay, but it does not implement the issue's central system-event-shape audience primitive or an equivalent backend consumer-path boundary for exec/cron/heartbeat system events.

## Best Possible Solution

Keep #69492 open as the canonical system-event-shape Track B item. The likely fix belongs in core: add an event-level audience or equivalent typed intent to SystemEvent/enqueueSystemEvent, derive exec completion audience from explicit defaults/trigger context, honor it in heartbeat relay and transcript/history consumers, and cover exec, cron, heartbeat, chat.history, sessions.get/SSE, and at least one non-webchat delivery path with tests.
