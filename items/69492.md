## Summary

Current main does not implement #69492's central event-audience primitive or a shared consumer-path filter for system-event-shape artifacts. There are narrower fixes for delivery-mirror replay, heartbeat prompt/ack display, and silent cron `delivery.mode="none"`, but the broad system-event classification gap remains actionable in core.

## Best Possible Solution

Keep #69492 open as the canonical core design/implementation issue. The likely fix belongs in core system-event and heartbeat/session-history paths: add a backward-compatible system-event audience or equivalent classification, carry completion intent through exec sessions, mark internal producers appropriately, and apply a shared consumer-path predicate to `chat.history`, `sessions.get`, session history HTTP/SSE, and channel relay tests.
