## Summary

Current main does not implement the requested per-event audience classification for system-event-shaped payloads. The system event type and enqueue options still only carry text, timestamps, context, delivery context, and trusted state; heartbeat relay still decides user delivery from channel/target/showAlerts rather than an event audience; and tests still assert user relay for exec completion events when a delivery target exists. The issue is also active, with PR #72201 described by the author as addressing only one awareness-event slice, not the broader canonical problem.

## Best Possible Solution

Keep this issue open as the canonical maintainer design/tracking item for system-event-shaped leakage. The best path is to coordinate PR #72201 and the narrower heartbeat/exec fixes, then either add a typed event audience or hidden-runtime-context lane to SystemEvent/enqueue options with consumer-boundary tests, or explicitly document a narrower tactical-patch policy if maintainers reject the broader primitive.
