## Summary

Current main does not yet implement the issue's proposed system-event audience boundary. System events still carry only text/timestamp/context/delivery/trust metadata, exec completion still enqueues unclassified events, heartbeat relay still decides user delivery from route/visibility only, and transcript/history filtering is still pattern/assistant-message oriented rather than per-event audience based. The linked open PR #72201 appears to be the active implementation path, so the issue should stay open until that or an equivalent maintainer-approved fix lands.

## Best Possible Solution

Keep #69492 open as the canonical backend system-event audience issue. Land or revise PR #72201, or an equivalent maintainer-approved patch, to add a SystemEvent audience primitive, classify exec/cron/heartbeat producers, honor the classification in heartbeat relay and transcript/history/channel consumer paths, and add regression coverage for internal versus user-facing events.
