## Summary

Keep open. Current main still does not emit `command:new`/`command:reset` internal hooks for idle/daily session rollovers; it only preserves/archive state for stale sessions and emits command hooks for explicit reset command paths. The linked fix PR #38027 is closed unmerged in the provided GitHub context, and its advertised auto-rollover tests are not present in this checkout.

Auto stale rollovers have `resetTriggered=false`, so this does not satisfy the issue request.
