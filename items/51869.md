## Summary

Keep open. Current main still hardcodes newly onboarded non-Azure custom provider models to text-only input, and the existing test suite explicitly asserts that behavior for non-Azure URLs. The issue discussion also points to open PRs #51893 and #51965 as proposed fixes, but the requested behavior is not implemented on current main.
