## Summary

Keep open. Current main still creates new non-Azure custom-provider models with `input: ["text"]`, and there is no wizard prompt or heuristic for image-capable model IDs. The existing tests also assert text-only metadata for non-Azure onboarding. Gateway image handling still depends on the resolved model catalog `input` metadata, so a vision-capable custom model saved as text-only will be treated as not supporting inline images. The related PRs #51893 and #51965 directly target this issue, but the provided GitHub context shows both remain open and unmerged.

## Best Possible Solution

Keep #51869 open until a maintainer lands a fix. The likely product path is to update `src/commands/onboard-custom-config.ts` and the wizard/non-interactive tests so non-Azure custom onboarding can mark image-capable models as `input: ["text", "image"]`, either through an explicit prompt/flag or a conservative model-name heuristic, while preserving existing user-customized model metadata on re-onboard. The open PRs #51893 and #51965 are useful starting points but need maintainer review against current main.
