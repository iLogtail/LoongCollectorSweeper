## Summary

PR #41332 should stay open for maintainer judgment. Current main has not implemented the gateway shutdown runtime change or the configurable cron startup stagger, and the cooldown portion is only partially overlapped by later shipped work; current docs/tests intentionally preserve transient cooldown probes in cases this PR would tighten. The related #41317 was an earlier closed unmerged attempt, not a canonical shipped replacement.

The gateway shutdown fix can be evaluated independently around a Node-only createGatewayRuntime/process.reallyExit path; the cooldown change needs reconciliation with the current documented transient-probe policy; the cron startup stagger should either become a supported config/docs addition or be split into its own focused PR with tests.
