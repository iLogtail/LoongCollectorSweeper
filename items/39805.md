## Summary

Keep open. Current `main` still does not implement Feishu Sheet content reading. The bundled Feishu plugin exposes doc, wiki, drive, permission, chat, and bitable tools, but no sheet-reading tool; wiki nodes can resolve to `sheet`, yet the documented workflow still routes content access through `feishu_doc`, whose schema and actions are docx/document-only.

## Evidence

- **No sheet tool in Feishu plugin exports:** The Feishu plugin exports `registerFeishuDocTools`, `registerFeishuChatTools`, `registerFeishuWikiTools`, `registerFeishuDriveTools`, `registerFeishuPermTools`, and `registerFeishuBitableTools`, with no `registerFeishuSheetTools` or equivalent sheet surface.
  - file: [extensions/feishu/api.ts:2](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/api.ts#L2)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Doc tool remains docx/document-focused:** `feishu_doc` is still described only as document operations, and its action list contains doc/block/table operations only; there is no sheet/range/value read action.
  - file: [extensions/feishu/src/docx.ts:1429](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/src/docx.ts#L1429)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Doc schema expects `/docx/` tokens:** The read schema still requires `doc_token` with the description `extract from URL /docx/XXX`, which does not cover Feishu Sheet URLs or range-based reads.
  - file: [extensions/feishu/src/doc-schema.ts:17](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/src/doc-schema.ts#L17)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Drive tool handles sheet metadata/comments, not sheet content:** `feishu_drive` supports `sheet` as a file type, but its registered actions are limited to list/info/folder moves/deletes and comment operations. There is no action for reading spreadsheet cells, rows, columns, or ranges.
  - file: [extensions/feishu/src/drive.ts:758](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/src/drive.ts#L758)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Wiki flow still depends on feishu_doc for content:** The bundled Feishu wiki skill says `get` may return `obj_type: "sheet"`, but its documented workflow still says to use `feishu_doc` to read/write the resolved content, which matches the reported gap for wiki-backed sheets.
  - file: [extensions/feishu/skills/feishu-wiki/SKILL.md:43](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/skills/feishu-wiki/SKILL.md#L43)
