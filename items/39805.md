## Summary

Keep #39805 open. Current main still has no ordinary Feishu Sheet content, row/column, or A1-range read tool; the existing Feishu surfaces cover docs, wiki metadata/navigation, drive metadata, permissions, chat, and Bitable records only. The request remains actionable in the existing `@openclaw/feishu` plugin unless a maintainer explicitly decides to move this capability out to a separate ClawHub plugin.

## Best Possible Solution

Implement this in the existing `@openclaw/feishu` plugin: add a Sheet tool module and schema for wiki/sheet token resolution plus row/column/range reads, wire it through `extensions/feishu/index.ts`, `api.ts`, tool config, skills/docs, permissions guidance, and focused tests. If maintainers do not want ordinary Sheet support in the bundled Feishu plugin, they should make that product decision explicitly and point contributors to a ClawHub plugin path.
