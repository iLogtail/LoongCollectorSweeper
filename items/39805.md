## Summary

Keep #39805 open. Current main still does not provide an ordinary Feishu Sheet content reader, row/column reader, or A1-range reader. The bundled Feishu plugin handles doc, chat, wiki, drive, permissions, scopes, and Bitable records; sheet appears only as metadata/object type support, not as a cell/range data surface. This remains an actionable enhancement for the existing `@openclaw/feishu` plugin unless maintainers explicitly decide to move ordinary Sheet support to a separate ClawHub plugin.

## Best Possible Solution

Implement ordinary Sheet support in the existing `@openclaw/feishu` plugin: add a Sheet tool module and schema for wiki/sheet token resolution plus row, column, and A1-range reads; wire it through `extensions/feishu/index.ts`, `extensions/feishu/api.ts`, Feishu tools config/types, plugin skills/docs, permission guidance, and focused unit tests. If maintainers do not want this in the bundled Feishu plugin, make that explicit as a product decision and point contributors to a separate ClawHub plugin path.
