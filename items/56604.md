## Summary

Keep open. Current main still only has global voice-call settings for greeting, agent, model, system prompt, and TTS; inbound events preserve the dialed `to` number, but no schema, docs, tests, or runtime path uses it to select a per-number persona or voice. The maintainer discussion also explicitly reviewed current main and kept the request open.

## Best Possible Solution

Keep the issue open and implement this inside the bundled `voice-call` plugin: add a typed, documented per-number route table to the config schema/manifest, normalize and match inbound `event.to` at call creation, store a stable per-call effective config, and make response generation, realtime consults, greetings, and TTS playback use that effective config. Add focused tests for Twilio/Telnyx inbound `to` routing, default fallback behavior, and per-number overrides for greeting, agent/model/system prompt, and TTS voice.
