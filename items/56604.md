## Summary

Keep #56604 open. Current `main` still has only global Voice Call settings for TTS, response model, system prompt, greeting, and agent selection. Inbound events preserve the dialed `to` number on the call record, but there is no `numbers` route table or merge path that resolves a per-dialed-number effective config. This matches the 2026-04-25 maintainer review comment that explicitly kept the issue open.

## Best Possible Solution

Implement this inside the bundled Voice Call plugin: add a typed per-number route table to the config schema, manifest schema, docs, and tests; resolve an immutable effective config from the inbound `event.to` when the call record is created; store enough per-call routing metadata so later response generation, greeting, agent/model selection, and TTS voice selection use the same effective config for the life of the call. Preserve global settings as defaults and cover fallback, exact E.164 matching, and stable per-call behavior.
