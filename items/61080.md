## Summary

Keep PR #61080 open. Current main still has the Logs screen user-facing copy hardcoded in `ui/src/ui/views/logs.ts`, and the current locale source only has navigation/subtitle `logs` entries, not the `logs.*` screen keys this PR adds. The bot-flagged export filename regression was already addressed in the PR by keeping the file suffix locale-independent, so the remaining work is still useful but needs rebase/update for the current generated-locale workflow.

## Best Possible Solution

Keep the PR open or replace it with an equivalent current-main patch: add Logs screen keys to `ui/src/i18n/locales/en.ts`, regenerate supported locale bundles through the Control UI i18n pipeline, wire `ui/src/ui/views/logs.ts` through `t(...)`, and preserve the stable ASCII export filename suffix separate from the localized button label.
