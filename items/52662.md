## Summary

Keep #52662 open. Current main still treats the sandbox browser as a Docker-managed Chromium/noVNC container, has no strict config/schema fields for external CDP or external noVNC endpoints, and built-in non-Docker sandbox backends still do not support sandbox browsers. The Lightpanda/external-CDP comment reinforces a real remaining use case rather than making the issue obsolete.

## Best Possible Solution

Keep the issue open and implement an additive core path: either the proposed `agents.defaults.sandbox.browser` external CDP/noVNC endpoint config or a broader browser-backend contract that lets sandbox backends provide browser context. The change should update config types/schema/help/docs, avoid Docker lifecycle calls when external endpoints are configured, preserve the bridge/noVNC auth model, and add focused tests for Docker default behavior plus external endpoint behavior.
