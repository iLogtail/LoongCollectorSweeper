## Summary

Keep open. Current main has the related `agents.defaults.compaction.model` implementation for compaction summarization, including local-model refs such as Ollama, but it does not implement the broader requested housekeeping routing. The pre-compaction memory flush still runs through the active follow-up run's provider/model fallback path, and `memoryFlush` config has no model override.

## Best Possible Solution

Keep the issue open or split it into a narrower follow-up for a dedicated housekeeping/local model policy. A concrete path would be to add an explicit model/provider override for pre-compaction memory flush, such as `agents.defaults.compaction.memoryFlush.model` or a broader `agents.defaults.housekeeping.model`, wire it through `runMemoryFlushIfNeeded`, document how it differs from `agents.defaults.compaction.model`, and clarify that startup bootstrap files are currently injected into the active conversation context rather than read by a separate model.
