## Summary

Keep open. Current `main` intentionally leaves `sessions_spawn.agentId` optional by default, but it already ships a configurable `subagents.requireAgentId` guard for both native subagent and ACP spawns while preserving the documented `acp.defaultAgent` fallback. That means this PR is not implemented as-written, and the review thread shows a real product/design tradeoff rather than cleanup-only noise, so it needs explicit maintainer judgment instead of auto-close.
