## Summary

Keep open. Current main does not implement this PR as written: `sessions_spawn.agentId` remains optional in the tool schema and implementation. Main does ship an opt-in `subagents.requireAgentId` runtime guard and preserves the documented ACP `acp.defaultAgent` fallback, so the PR is now a product/design tradeoff rather than a conservative cleanup close.

## Best Possible Solution

Keep the PR open for explicit maintainer judgment. A mergeable direction would either revise the PR to preserve ACP default-agent behavior while improving native subagent schema guidance, or document that the shipped `subagents.requireAgentId` runtime guard is the accepted solution and close the PR manually with that product decision.
