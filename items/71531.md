## Summary

Keep #71531 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not accept `channels.slack.logLevel`, does not type it, and does not forward any log level or custom logger into Slack Bolt. The referenced PR #71550 is a relevant active implementation path, but it is open/unmerged in the provided GitHub context, so the requested behavior is not shipped on main.

## Best Possible Solution

Keep #71531 open and use PR #71550, or an equivalent maintainer-reviewed patch, as the implementation path. The fix should add a validated and typed `channels.slack.logLevel` surface aligned with Slack Bolt logger levels, forward it to Bolt App construction only when configured, preserve omitted/default behavior, and update docs/generated config surfaces plus focused tests before merge.
