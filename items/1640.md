## Summary

本 Issue 请求为现有 Elasticsearch Flusher 增加 Data Streams 模式支持。该功能需适配 ES Bulk API 的 op_type: create 语义及索引路由规则，属于对现有输出插件的合理增强。当前 main 分支尚未实现该配置项，且该需求明确、具备实现路径，符合有效功能需求特征，故保持开启以跟踪后续社区贡献或维护者排期。

## Best Possible Solution

保持开启作为功能需求跟踪。建议后续贡献者在 plugins/flusher/elasticsearch 中增加 data_stream 或 op_type 配置开关，适配 ES Data Streams 要求的索引行为。维护者可添加 help wanted 标签，或等待原作者及社区提交设计草案与 PR。

## Evidence

- **current_implementation_gap:** 当前 main 分支的 flusher_elasticsearch 插件未提供 data_stream 或 op_type 配置项，Bulk 请求默认使用 index/update 行为，尚未适配 ES Data Streams 必需的 create 语义。
