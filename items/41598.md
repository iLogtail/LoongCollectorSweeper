## Summary

Do not close PR #41598. Current main still has the chat-bash-specific gap the PR is trying to fix: `handleBashChatCommand` builds its own exec tool and forwards timeout/notification defaults, but not `tools.exec.host`, `tools.exec.security`, or `tools.exec.ask`. The PR needs changes before merge because its added markdown-image routing change has an unresolved review concern about breaking punctuation-prefixed bang aliases, but the core security inheritance work is still useful and not superseded by the broader exec-policy fixes.

## Best Possible Solution

Keep the PR open or replace it with a narrow maintainer patch that passes `cfg.tools.exec.host`, `cfg.tools.exec.security`, and `cfg.tools.exec.ask` into the chat-bash `createExecTool` call, adds focused regression coverage for those defaults, and handles markdown image text without breaking existing punctuation-prefixed `!` shell aliases.
