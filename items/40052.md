## Summary

Keep PR #40052 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still saves default-agent model and fallback edits through the per-agent `agents.list[index].model` path, while the documented defaults surface is `agents.defaults.model`. The PR targets a real bug, but review discussion identified a stale default-agent override cleanup gap that should be addressed before landing unchanged.

## Best Possible Solution

Keep this open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, keeps non-default edits on `agents.list[].model`, clears or migrates stale default-agent `agents.list[].model` overrides when saving defaults, and adds regression coverage for default-agent, non-default-agent, and `defaultId === null` path resolution.
