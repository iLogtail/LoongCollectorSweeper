## Summary

Keep open. Current main still routes the Agents UI model editor through `agents.list[index].model`, so the PR's reported bug remains present. The PR also has unresolved review discussion about stale default-agent overrides that should be handled before landing or in a coordinated follow-up.

## Best Possible Solution

Keep the PR or a maintainer-owned successor open until the Agents UI writes default-agent primary and fallback edits to `agents.defaults.model`, preserves non-default agents on `agents.list[index].model`, and removes or migrates stale default-agent `agents.list[*].model` overrides so the saved default actually takes effect. Add focused tests for default, non-default, `defaultId === null`, and stale-override behavior. Issue #40065 is useful bug context, but it does not supersede this implementation PR.
