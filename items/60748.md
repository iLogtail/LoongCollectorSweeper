## Summary

Keep open. Current main only partially addresses the Feishu card UX by omitting the generic `main` header; the broader requested capability remains missing: no configurable card header/footer surface, no elapsed/status footer rendering, no footer hiding/toggles, and no reply-to context header mode. Related issue #71079 was closed because #60748 is the canonical tracker, and PR #54979 only covers a header-hide subset.

## Best Possible Solution

Keep #60748 open as the canonical Feishu card customization tracker. The best implementation belongs in the Feishu plugin: add top-level and per-account config for card header mode and footer fields, wire it through `resolveCardHeader`/`resolveCardNote` for streaming and static card paths, preserve backward-compatible defaults, update plugin manifest/generated config metadata and `docs/channels/feishu.md`, and add dispatcher/schema tests for hidden/custom/reply-to headers plus elapsed/status/footer toggles. PR #54979 can be reviewed as a narrower header subset or superseded by a broader PR against this issue.
