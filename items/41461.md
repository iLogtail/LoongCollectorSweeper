## Summary

Keep open. Current main still has the same class of gap the PR is trying to fix: Anthropic OAuth beta injection lives inside the beta wrapper, but the provider wrapper is only installed when extra params resolve at least one Anthropic beta. The PR will need a rebase because the active implementation moved into the Anthropic plugin and native transport paths, but the underlying fix is not obsolete or shipped.

## Best Possible Solution

Keep the PR or its linked issue active, then rebase the fix onto the current Anthropic plugin architecture: always preserve OAuth-required Anthropic betas for `sk-ant-oat-*` tokens even when no extra-param beta is configured, and ensure model-level/options-level `anthropic-beta` values merge with rather than replace `oauth-2025-04-20`. Add regression coverage for OAuth token + model-level `anthropic-beta` headers without `params.context1m`.
