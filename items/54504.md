## Summary

Keep open. Current main has mature `sessions_spawn` support, model/thinking defaults, context modes, attachments, and plugin hooks, but it does not implement lightweight per-label subagent instruction files or frontmatter-driven per-role tool policy. Related #50678 and #51598 cover adjacent prompt customization, not this exact label-scoped file convention.

## Best Possible Solution

Keep open for maintainer product/API design. The likely best path is a narrow, versioned subagent role/profile seam: resolve `sessions_spawn.label` to workspace/user role metadata, inject the markdown body as child system context, apply safe model/thinking defaults, and define whether per-role tools are supported as model-visible tool filtering, enforcement-only policy, or explicitly out of scope. If maintainers prefer this outside core, first add the missing plugin hook/API needed for a ClawHub plugin to implement it cleanly.
