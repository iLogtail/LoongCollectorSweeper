## Summary

Current `main` only partially addresses #62525. Managed `HOOK.md` hooks register through the internal `registerInternalHook()` / `triggerInternalHook()` bus, and `message_received` plus `message_sent` now bridge into internal `message:received` / `message:sent` events. The central request remains open because `before_dispatch` and `agent_end` still run only through the plugin hook runner, and there is no current internal `before:dispatch` or `agent:end` emission.

## Best Possible Solution

Keep #62525 open for maintainer design and implementation. The concrete path is either to add tested internal emissions for selected lifecycle events such as `before_dispatch` and `agent_end` with a documented event naming contract, or to document that managed `HOOK.md` scripts are not intended to mirror plugin lifecycle hooks and point users to a plugin-based implementation for those events.
