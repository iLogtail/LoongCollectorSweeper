## Summary

Keep #10944 open. Current main still lacks a config-driven Telegram parse mode: normal sends, media captions, edits, and streaming previews continue to use Telegram HTML mode, while the strict config schema/types/docs expose related Telegram knobs but no parseMode, MarkdownV2, or plain-text setting. The issue discussion already corrected the outdated Markdown claim to current HTML behavior, and the request remains actionable.

## Best Possible Solution

Keep #10944 open and implement this in the bundled Telegram plugin/config surface. A good maintainer path is a typed channels.telegram.parseMode setting, with per-account override only if maintainers want that granularity, wired through normal sends, media captions, edits, action delivery, and stream/draft preview materialization. Update the strict schema, exported config types, generated config metadata, Telegram docs, and focused tests for HTML, plain/no-parse-mode, and MarkdownV2 or any supported Markdown mode.
