## Summary

Current main still does not ship `openclaw sessions rm` or `openclaw sessions clear`. The PR addresses a real CLI gap, and the related issue #35682 remains open, but this implementation still needs maintainer review and updates for the session-store maintenance semantics raised in review comments.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is a rebased implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that defines safe mutation semantics, dry-run parity, transcript archival/removal behavior, stable JSON output, and explicit interaction with `session.maintenance`; alternatively, a maintainer can manually close this PR only after deciding to track all remaining work solely in #35682.
