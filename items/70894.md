## Summary

#70894 should stay open. Current main still routes `/hooks/agent` requests into an isolated cron job, and `runCronIsolatedAgentTurn` still passes `forceNew: true` for every `sessionTarget === "isolated"` run. The session resolver only reuses stored history when `forceNew` is false, so the reported webhook session-continuity regression remains present. The linked PR #70895 is relevant and appears to target the right surface, but it is open and unmerged, so it does not make the issue closeable yet.

## Best Possible Solution

Keep #70894 open and review/land a focused fix like #70895: compute a webhook-aware `forceNew` value in `src/cron/isolated-agent/run.ts`, add regression coverage proving `/hooks/agent` or hook-sourced isolated runs pass `forceNew: false`, and retain coverage that ordinary isolated cron runs still force a fresh session.
