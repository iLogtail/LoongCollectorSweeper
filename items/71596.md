## Summary

Keep open. Current `main` still has the configure/model-picker behavior this draft PR is targeting, and the proposed cleanup changes a documented fallback-vs-allowlist contract that needs maintainer review rather than conservative auto-close.

## Evidence

- **Current main still rewrites fallbacks from picker selection:** `promptAuthConfig` applies the scoped model allowlist and then unconditionally rewrites `agents.defaults.model.fallbacks` from the picker selection without passing scope metadata, so provider-scoped configure flows still mutate fallback config on current `main`.
  - file: [src/commands/configure.gateway-auth.ts:171](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/commands/configure.gateway-auth.ts#L171)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Allowlist prompt does not seed configured fallbacks on main:** `promptModelAllowlist` seeds initial selections from configured allowlist keys, the resolved primary model, and explicit initial selections, but not from `agents.defaults.model.fallbacks`, so existing fallback refs are not surfaced for intentional keep/remove decisions.
  - file: [src/flows/model-picker.ts:618](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/flows/model-picker.ts#L618)
