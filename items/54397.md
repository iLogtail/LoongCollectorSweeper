## Summary

Keep #54397 open. Current main has useful partial session-management plumbing, but the requested Topic/Session Management UI is not fully implemented: Web UI can list/filter/label/delete sessions and switch from chat, Gateway has list/create/patch/delete APIs, but TUI/CLI creation-renaming-deletion, a full topic list/sidebar workflow, topic metadata such as tags/descriptions/message counts, and import/export remain missing.

## Best Possible Solution

Keep the issue open as a broad product/UI enhancement. The best path is to build on existing `sessions.list/create/patch/delete/resolve/preview` Gateway methods and the current Web UI session table/select, then add the missing TUI switcher, CLI create/rename/delete aliases, topic metadata/search/display fields, and explicit archive/restore/import/export behavior with docs and tests.
