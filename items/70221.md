## Summary

Keep #70221 open. Current main still has a height-only hosted embed/canvas preview contract: `[embed]` shortcode parsing recognizes `height`, the normalized preview types only carry `preferredHeight`, the chat and sidebar iframe renderers only apply inline height, and the shared preview CSS still forces iframe width to `100%`. This is a focused Control UI feature request, not a duplicate, ClawHub candidate, or stale report.

## Best Possible Solution

Keep the issue open for a focused Control UI implementation. A good fix would add a bounded `width`/`preferredWidth` contract through `src/chat/canvas-render.ts`, UI preview/sidebar types, chat and sidebar iframe rendering, CSS layout behavior, tests for `[embed width="..."]`, and user-facing guidance for the supported syntax and limits.
