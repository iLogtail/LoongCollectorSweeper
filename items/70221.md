## Summary

Keep #70221 open. Current main still has a real Control UI feature gap: `[embed]` accepts and applies `height`, but there is no parsed, typed, normalized, documented, or rendered width parameter for hosted/canvas embeds. The issue has no protected labels and no maintainer author signal, but the requested behavior is a core Control UI directive/rendering contract, not a ClawHub/plugin cleanup item.

## Best Possible Solution

Keep the issue open and implement a first-class Control UI embed width contract: add a normalized/clamped `preferredWidth` or equivalent field to the canvas preview and UI types, parse it from `[embed width="..."]` and JSON preview payloads if desired, apply it in inline and sidebar iframe rendering with responsive overflow/max-width behavior, then update Hosted embeds docs and tests alongside the existing height coverage.
