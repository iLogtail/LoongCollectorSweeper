## Summary

Keep open. Current `main` still drops parsed `/think` and `/verbose` levels when mixed-content messages go through inline-directive clearing, so this PR appears to fix a real bug that is not already implemented on `main`.

## Evidence

- **Mixed-content cleanup still clears think/verbose on main:** `resolveReplyDirectives` still sends mixed-content messages through `clearInlineDirectives(parsedDirectives.cleaned)` once it decides the message has real content after stripping directives. There is no preservation step for `thinkLevel` or `verboseLevel` before that reset.
