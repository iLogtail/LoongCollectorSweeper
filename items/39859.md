## Summary

Current `main` does not implement the PR's core fix. `resolveContextTokensForModel()` still returns a positive `contextTokensOverride` immediately, before checking any known model window, so OpenClaw can still over-report the effective session context window. The alias-normalization part has effectively been addressed on `main` through the current configured-provider lookup, but the override-capping behavior and matching regression coverage are still missing.

## Evidence

## Risks / Open Questions

- Closing this PR would leave a live correctness bug where `/status` and persisted session metadata can overstate effective context for a model when a larger override is set.
- The current tree has no regression test in `src/agents/context.test.ts` covering the PR's core `min(override, knownModelWindow)` behavior.
- Only the alias-normalization subproblem appears handled on `main`; treating the whole PR as already implemented would overstate what shipped.
