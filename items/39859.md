## Summary

Keep open. Current main still does not implement PR #39859's central behavior: a positive contextTokensOverride wins before known model/config/discovery limits are resolved, so status/session reporting can still overstate the effective model context window.

## Best Possible Solution

Keep #39859 and the linked #39857 open until main lands the narrow product fix: resolve the known model context window before honoring a contextTokens override, return Math.min(contextTokensOverride, modelTokens) when both values are known, preserve existing override behavior when no model window is known, normalize provider aliases for configured model-window lookup, and add focused resolver/status/session regression tests. Avoid carrying over the unrelated CI and baseline churn from the stale PR branch.
