## Summary

Keep open. Current `main` still does not implement PR #39859's central fix: a positive `contextTokensOverride` returns before any known model window is resolved, so session metadata and `/status` can still over-report the effective context window. The linked issue #39857 remains open and tracks the same unresolved behavior.

## Best Possible Solution

Keep #39859 or linked issue #39857 open until main implements the narrow resolver behavior: resolve the known model context window before returning an override, return `Math.min(contextTokensOverride, modelTokens)` when both values are known, preserve existing override behavior when the model window is unknown, and add focused regression coverage for larger-override capping, lower-override preservation, and normalized provider-alias lookup.
