## Summary

Current main does not implement PR #39859's core fix. `resolveContextTokensForModel()` still returns a positive override before checking configured or discovered model windows, so a larger `agents.defaults.contextTokens`/agent override can still overstate the effective session context window in status/session metadata paths.

## Best Possible Solution

Keep this PR or the linked issue #39857 open until main implements the focused resolver behavior: when both a positive context override and a known model context window are available, resolve the effective session context as the smaller value, while preserving existing provider-alias lookup, Anthropic 1M, discovery-cache, and OpenRouter slash-model safeguards.
