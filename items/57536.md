## Summary

Keep open. The reporter's follow-up narrows this to daemon/service proxy handling, and current `main` still forwards ambient proxy variables into managed gateway services, which matches the remaining complaint rather than showing an implemented fix or an obsolete report.

## Evidence

- **Reporter follow-up changed the diagnosis from generic Feishu auth failure to proxy-specific daemon behavior:** The only follow-up comment says Feishu works with proxy disabled in foreground `openclaw gateway run`, but the background service started via `openclaw gateway restart` still appears to use the proxy.
