## Summary

Keep #39610 open. Current main at b67d9bf still only supports global `agents.defaults.userTimezone`; it does not implement `agents.list[].userTimezone` in config types/schema/docs or thread an agent-aware timezone through prompts, cron/current-time helpers, heartbeat, envelopes, system events, or gateway timestamp injection. The linked issue #39579 was closed by the PR reference, but this PR is still the canonical unmerged implementation path.

## Best Possible Solution

Keep #39610 open as the canonical implementation path for per-agent user timezone support. The maintainer-ready direction is to add `agents.list[].userTimezone` to config types, strict runtime schema, generated schema, and docs; centralize config-aware agent timezone resolution; thread `agentId` through every `"user"` timezone path; validate IANA timezone strings consistently; and refresh targeted tests for system prompts, `session_status`, heartbeat active hours and body injection, envelopes/system events, gateway/chat timestamp injection, memory flush/current-time helpers, post-compaction context, and plugin inbound-envelope surfaces.
