## Summary

Keep open. Current main does not implement the requested behavior: the Control UI still stores gateway tokens in sessionStorage, not localStorage, so browser-close persistence remains absent. The current source, tests, and docs appear to make that a deliberate security/product choice rather than an already-fixed bug, so this needs maintainer judgment rather than automated closure.

## Best Possible Solution

Do not auto-close. A maintainer should decide whether browser-close token persistence is desired. If yes, implement an explicit opt-in remember-token flow in `ui/src/ui/storage.ts` and update Control UI docs/tests; if no, close manually as intentional behavior and point reporters to the documented sessionStorage/current-tab security model or tokenless identity-bearing auth modes.
