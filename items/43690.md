## Summary

Keep #43690 open. Current main still sends Feishu markdown/table replies through interactive-card paths without a plain-text/table-conversion fallback when card delivery fails. The linked PR #50429 is a relevant open fix attempt for the reply-dispatcher path, and the related #70651 report indicates a separate outbound direct-send path is also still unresolved.

## Best Possible Solution

Keep the issue open and route it to the active Feishu fix work: add a narrowly tested fallback around `sendStructuredCardFeishu()`/`sendMarkdownCardFeishu()` failures caused by Feishu card table limits, reuse the existing `sendMessageFeishu()` table-conversion path for the retry, and make sure both the inbound reply-dispatcher path and outbound direct-send path are covered by regression tests.
