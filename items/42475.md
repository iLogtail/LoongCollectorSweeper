## Summary

Keep #42475 open. Current main has cost observability, provider quota/cooldown handling, and documented external spend-limit workarounds, but it does not provide per-agent daily/monthly budget config or a gateway-level pre-dispatch enforcement gate. The related #66012 item is a closed optimization/dashboard request, not a canonical tracker for this enforcement feature.

## Best Possible Solution

Keep the issue open and implement a narrow core guardrail or first design an explicit blocking pre-model-call plugin API. The core path should add validated `agents.defaults` and `agents.list[]` budget config, enforce it in the shared model-dispatch path before provider calls, account for daily/monthly periods and in-flight calls, and surface status/doctor warnings. External proxies such as LiteLLM can remain documented workarounds, but they do not replace the requested per-agent gateway control.

## Risks / Open Questions

- Budget enforcement based only on transcript scans could race with in-flight calls; a robust design likely needs a synchronized per-agent budget ledger or reservation/refund model.
- Cost estimates are approximate because provider pricing, cache behavior, and subscription-style quota surfaces differ by provider.
- A native gate needs clear policy for which spend surfaces count: core chat, retries/fallbacks, compaction, media understanding/generation, embeddings, web tools, and plugin-owned providers.
