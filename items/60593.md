## Summary

Keep open. Current main still has the core failure mode described in #60593: Anthropic SSE frames are parsed with a bare `JSON.parse`, the resulting SyntaxError is not classified as failoverable when there is no timeout signal, and the short raw parse message can still become the user-facing error. The related closed items and the pi-ai partial-JSON parser work address adjacent tool-argument parsing paths, but not this Anthropic SSE-frame parse/failover path.

## Best Possible Solution

Keep #60593 open and fix the Anthropic streaming boundary directly: wrap Anthropic SSE frame parsing in a typed helper, convert malformed/truncated stream JSON into a safe provider-stream error with a retry/failoverable reason, add regression coverage for malformed Anthropic SSE frames and failover decisions, and ensure any remaining JSON.parse SyntaxError is rewritten to a generic user-facing LLM streaming failure instead of raw parser text.
