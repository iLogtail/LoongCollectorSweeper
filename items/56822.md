## Summary

Current main already solves the PR’s fan-out/fan-in reliability problem through the shipped push-based subagent completion path: parent/orchestrator turns defer while descendants are pending, child completion findings are aggregated, completed descendants can wake the orchestrator to synthesize, and agents are guided to use `sessions_yield` instead of polling. The exact `sessions_await` API is not present, but the observable orchestration problem is covered without adding the PR’s risky suppress-auto-announce surface.

## Best Possible Solution

Close this PR as implemented by current main’s shipped push-based orchestration path. Keep the existing `sessions_yield` plus descendant-settle wake/aggregation behavior as the product direction unless maintainers later want a separate, carefully designed blocking tool proposal with fresh ownership and tests.
