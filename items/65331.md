## Summary

Keep open. Current main still does not clearly implement PR #65331's central fix: plugin text transforms are still resolved through the indirect `createRequire`-based runtime-registry helper, while the PR's direct active-registry lookup is not present. The related bug report #64921 also remains open, and the inspected tests only prove registration, not main-process application.

## Best Possible Solution

Keep PR #65331 open for maintainer review or a revised patch. The likely fix belongs in the plugin runtime registry/text-transform path: use a direct active-registry accessor that works in the bundled main process, remove or justify any unreachable fallback, and add a regression test proving plugin-registered text transforms are applied through the main runtime prompt/provider path. Related issue #64921 should remain the bug-tracking anchor until this behavior is fixed and tested.
