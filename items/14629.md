## Summary

Current main still has the reported sanitizer gap. `sanitizeUserFacingText()` strips `<final>` tags textually and only collapses exact adjacent duplicate blocks separated by blank lines, while outbound replies still pass through this sanitizer before delivery. Existing tests cover blank-line duplicate paragraphs and intentionally preserve ordinary single-newline text, but they do not cover the reported same-line `</final>` duplicate, single-newline duplicate collapse, or partial-prefix cleanup.

## Best Possible Solution

Keep #14629 open. Implement a narrow sanitizer change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` for same-line/single-newline repeats, the reported `</final>` boundary duplicate shape, and conservative partial-prefix duplicates. Add focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that prove the reported cleanup while preserving legitimate repeated wording and normal single-newline formatting.
