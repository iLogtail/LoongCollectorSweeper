## Summary

Current main still has the sanitizer gap described in #14629. `sanitizeUserFacingText()` strips `<final>` tags as plain text, then only collapses exact adjacent duplicate blocks separated by blank lines, so the reported same-line `</final>` boundary duplicate and single-newline/partial-prefix duplicates remain unhandled. Existing tests cover blank-line duplicate paragraphs and explicitly preserve ordinary single-newline text, but do not cover the requested cases. Other current dedupe paths address streaming/final-payload or messaging-tool duplicate delivery, not duplicate text inside one sanitized payload. The April 26 issue comment also already reached a keep-open conclusion for the same code path.

## Best Possible Solution

Keep #14629 open. The best path is a narrow change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` for same-line/single-newline repeats, the reported `</final>` boundary duplicate shape, and conservative partial-prefix duplicates, with focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that also preserve legitimate repeated text and ordinary single-newline formatting.
