## Summary

Keep #33478 open. Current main has partial thread/topic routing preservation for the A2A announce path, but the requested structured sessions_send skip/action contract is still missing, and maintainer discussion explicitly says to keep this as acceptance criteria for the broader handoff work in #63789.

## Best Possible Solution

Keep #33478 open as acceptance criteria for A2A handoff reliability. The best path is to implement a structured sessions_send outcome/action contract, or the equivalent first-party handoff protocol from #63789, that supports skip/no-reply/announce outcomes without exact model text, preserves channel/thread context, keeps legacy tokens as compatibility behavior, and adds regression coverage for chatter around REPLY_SKIP plus threaded announce routing.
