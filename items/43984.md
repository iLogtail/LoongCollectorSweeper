## Summary

Keep open. Current `main` still applies the `activeMinutes` cutoff uniformly in `listSessionsFromStore`, so stale primary/main agent sessions are still hidden, and the PR's regression coverage is not present on `main`.

## Evidence

- **Current main still filters stale main sessions out:** `listSessionsFromStore` computes `cutoff` and then filters with `sessions.filter((s) => (s.updatedAt ?? 0) >= cutoff)` without any exemption for primary/main agent sessions. A stale `agent:main:main` entry would still be removed here.
