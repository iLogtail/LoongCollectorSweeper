## Summary

Keep PR #57260 open. The Discord stale-warning bug described by the PR is not implemented on current main: mutating message fingerprints still use the raw `to`/`target` value, while successful mutating retries only clear `lastToolError` when the stored fingerprint matches. The PR should not land as-is because it also carries an unrelated cron deadlock commit/files; the best path is a focused Discord fingerprint-normalization PR plus regression tests, with the cron change split separately if still needed.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require it to be narrowed before merge: preserve Discord recipient validation, add the Discord-specific fingerprint normalization and the two regression tests described in the PR body, and split the cron lane deadlock change into its own issue/PR if that bug still needs handling.
