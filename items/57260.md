## Summary

Keep open. Current `main` still fingerprints Discord `target`/`to` values generically in `src/agents/tool-mutation.ts:184`, so a failed bare numeric send and a successful `channel:<id>` retry do not converge to the same action fingerprint. `lastToolError` is only cleared on exact fingerprint match in `src/agents/pi-embedded-subscribe.handlers.tools.ts:839`, and Discord still intentionally rejects bare numeric recipients as ambiguous in `extensions/discord/src/target-parsing.ts:38` and `extensions/discord/src/send.sends-basic-channel-messages.test.ts:300`. The PR is therefore not implemented on `main`, and it should not be auto-closed.
