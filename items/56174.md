## Summary

Keep open. Current main does not provide a supported Termux/Android Gateway host path or Termux command list. The docs still scope install/service support to macOS, Linux/WSL2, and Windows, while Android is documented as a companion node that does not host the Gateway. The only Termux-specific repo hits are optional auth/widget helper scripts, not OpenClaw CLI/Gateway availability on Termux.

## Best Possible Solution

Keep the issue open until maintainers choose the Termux product path. A complete path would either add and test a supported Android/Termux CLI/Gateway install/runtime flow with docs, or explicitly document Termux Gateway hosting as unsupported and consolidate this with a canonical Android Gateway/Termux tracking issue such as the service-install and Termux platform blockers.
