## Summary

Keep PR #65541 open. Current main and the latest release still buffer text-only and captioned block replies when block streaming is disabled, while docs describe `blockStreamingDefault: "off"` as final-reply-only behavior. The PR is a real behavior change, not already implemented or safely obsolete.

## Best Possible Solution

Keep the PR open for maintainer judgment. The clean path is to decide whether `blockStreamingDefault: "off"` should remain final-reply-only or gain a progressive, naturally bounded text delivery mode; if adopted, update `reply-delivery.ts`, the disabled-streaming tests, streaming docs, and the direct-send dedupe/fallback contract together.
