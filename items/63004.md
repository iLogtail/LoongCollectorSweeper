## Summary

Current `main` already gets the Discord voice gateway bridge behavior from the released `@buape/carbon@0.16.0`, so this PR's local stopgap is now redundant.

## Evidence

- **OpenClaw now depends on the released Carbon version that superseded the stopgap:** `extensions/discord/package.json` pins `@buape/carbon` to `0.16.0`. The PR body explicitly described its bridge classes as a temporary workaround until Carbon shipped the upstream fix; npm metadata shows `0.16.0` was published on 2026-04-16.
  - file: [extensions/discord/package.json:7](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/package.json#L7)
  - command: `nl -ba extensions/discord/package.json | sed -n '1,20p'; curl -fsSL https://registry.npmjs.org/%40buape%2Fcarbon | jq -r '.time["0.16.0"]'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **The released Carbon package now registers the voice bridge listeners itself:** In the published `@buape/carbon@0.16.0` package, `VoicePlugin.registerClient()` registers `VoiceStateUpdate` and `VoiceServerUpdate` listeners, which is the behavior this PR was adding locally.
  - command: `curl -fsSL https://registry.npmjs.org/@buape/carbon/-/carbon-0.16.0.tgz | tar -xzO package/dist/src/plugins/voice/VoicePlugin.js | nl -ba | sed -n '1,80p'`
- **Those upstream listeners forward gateway events into the adapter callbacks:** `VoiceServerUpdateListener.js` calls `voice.adapters.get(guildId)?.onVoiceServerUpdate(data)`, and `VoiceStateUpdateListener.js` reconstructs the raw payload and calls `voice.adapters.get(guildId)?.onVoiceStateUpdate(raw)`, matching the PR's intended fix path.
  - command: `curl -fsSL https://registry.npmjs.org/@buape/carbon/-/carbon-0.16.0.tgz | tar -xzO package/dist/src/plugins/voice/VoiceServerUpdateListener.js | nl -ba | sed -n '1,80p'; curl -fsSL https://registry.npmjs.org/@buape/carbon/-/carbon-0.16.0.tgz | tar -xzO package/dist/src/plugins/voice/VoiceStateUpdateListener.js | nl -ba | sed -n '1,100p'`
- **Current OpenClaw main relies on the upstream VoicePlugin instead of local bridge classes:** When voice is enabled, OpenClaw now adds `new VoicePlugin()` in startup, while `provider.ts` only wires `DiscordVoiceManager` and `DiscordVoiceReadyListener`. There is no local `adapter-bridge.ts` on current `main`, so merging this PR would duplicate behavior already supplied by Carbon.
  - file: [extensions/discord/src/monitor/provider.startup.ts:139](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/discord/src/monitor/provider.startup.ts#L139)
  - command: `nl -ba extensions/discord/src/monitor/provider.startup.ts | sed -n '133,168p'; nl -ba extensions/discord/src/monitor/provider.ts | sed -n '1033,1047p'; rg -n 'adapter-bridge|DiscordVoiceServerUpdateBridge|DiscordVoiceStateUpdateBridge' extensions/discord/src`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
