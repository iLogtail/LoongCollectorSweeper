## Summary

Current main does not implement the RFC’s distributed control-plane/runtime split, nor the narrower Phase 0/Phase 1 request from the discussion: start, stop, or restart one agent without restarting the gateway. This is core gateway/runtime architecture work, not a ClawHub-only plugin cleanup item.

## Evidence

- **Issue discussion leaves concrete remaining work:** The later issue comment narrows the most valuable minimum to per-agent lifecycle commands (`start`, `stop`, `restart`) that bring an agent online without a full gateway restart. Current main does not expose that surface.
- **Agents CLI has config/routing commands, not lifecycle commands:** The public docs describe `openclaw agents` as list/add/delete/bindings/bind/unbind/set-identity for isolated agents. There is no documented `agents start`, `agents stop`, or `agents restart`.
