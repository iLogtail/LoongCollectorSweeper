## Summary

Current main does not yet fix #65130. The package-manager update path still invokes npm global install without `--force`, and the tests assert that argv. The linked PR #65147 is an open, unmerged fix proposal for the same EEXIST failure, so this issue should remain open until a fix lands or maintainers intentionally make the PR the canonical tracker.

## Best Possible Solution

Keep #65130 open until a fix reaches main. The likely product path is to land or rework PR #65147, or otherwise add explicit handling for npm EEXIST on `/usr/local/bin/openclaw` in the package-manager update path, with a regression test covering container global updates.
