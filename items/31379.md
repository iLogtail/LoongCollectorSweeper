## Summary

Keep #31379 open. Current main at b67d9bf still renders the Channels settings cards from `channelMeta` or `channelOrder` exclusively, while `channels.status` still builds those arrays from loaded channel plugins only. A custom-only snapshot can still hide the built-in channel cards. The provided context shows PR #44165 as the focused open fix and PR #31350 as closed/unmerged, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #31379 open and resolve it through #44165 or an equivalent focused Control UI change in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom channel ids with the built-in channel id list, preserve snapshot/custom order, de-dupe ids, keep configured/active channels promoted by the existing render-time sort, and add regression coverage for custom-only `channelMeta` and `channelOrder`.
