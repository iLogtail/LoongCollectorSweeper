## Summary

Keep PR #70525 open. Current main handles the normal Control UI upload path by persisting uploaded images as transcript MediaPath/MediaPaths and the UI renders those paths, but the PR's specific gateway behavior is not implemented: current chat.history projection still strips top-level inline image data into an omitted marker. The PR is recent, active, and its review discussion addressed the payload-budget, MIME, and caller-scope concerns, so this needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and have maintainers review or rework it as a narrow chat.history compatibility fix: preserve inline image bytes only for authorized Control UI/WebChat history consumers, keep metadata-only output for non-UI/session-history consumers, enforce cumulative byte budgets and MIME normalization, and retain focused gateway regression coverage.
