## Summary

Keep #50165 open. Current main has a shipped partial fix for premature completion when descendant subagents are still pending, but the issue discussion also covers background exec or other external work still running after the subagent/session is shown as done; current status and task code still primarily resolve subagent completion from `endedAt` plus pending descendant subagents, not process/external-work liveness.

## Best Possible Solution

Keep the issue open and narrow it to the remaining lifecycle contract: when a subagent starts background exec or other externally tracked work, OpenClaw should either keep the subagent/task visibly active until that work exits, or explicitly surface a distinct state such as `agent turn done; background process still running` in `/status`, `/subagents`, `/tasks`, and completion announcements. The implementation likely belongs around subagent lifecycle/task status plus exec/process liveness tracking, with regression coverage for a subagent that backgrounds an exec and ends its agent turn before the process completes.
