## Summary

Keep #41275 open. Current main still rejects `timeoutSeconds: 0` in the Control UI and omits `0` from the payload builder, while core cron schemas/runtime already support `0` as no-timeout. The PR is still addressing a real UI/core contract mismatch, but it should add regression coverage and address the reviewer-noted malformed-input path before landing.

## Best Possible Solution

Keep the PR or linked bug active. The best fix is a small Control UI change that aligns `validateCronForm` and `buildCronPayload` with the existing core contract: accept finite `0` and positive values, reject negative/non-finite/non-numeric input, preserve `timeoutSeconds: 0` in create/update payloads, update the English validation/help copy if needed, and add focused controller tests for `0`, positive values, and malformed input.
