## Summary

Keep open. Current main has the stale-unended subagent run registry/liveness work mentioned in the issue discussion, but the specific persisted `sessions.json` cleanup/finalization requested here is still not implemented. The restore path prunes stale subagent run records; it does not remove or terminally update the direct child session-store row that external monitors read.

## Best Possible Solution

Keep #67902 open until the stale-orphan restore path either removes the child `sessions.json` entry or persists a terminal status/`endedAt` through the session-store writer, with a regression test that seeds both `subagents/runs.json` and `agents/<id>/sessions/sessions.json` and verifies direct `sessions.json` readers no longer see a zombie running row. If maintainers want #71756 to own all remaining lifecycle finalization work, close #67902 manually as duplicate with that explicit canonical link.
