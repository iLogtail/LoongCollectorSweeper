## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` entries in the shared MIME-to-extension map, and the affected media save/outbound filename paths still depend on `extensionForMime()`. The PR is a narrow, plausible bug fix and is not superseded by the provided related context.

## Best Possible Solution

Keep the PR open. Rebase or recreate the patch on current main, add `audio/webm` and `video/webm` mapping to `.webm` plus `video/x-matroska` mapping to `.mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`. A saved-media or outbound filename regression case would also protect the reported user-visible behavior.
