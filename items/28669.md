## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` mappings in the shared MIME-to-extension helper, while saved media and outbound filename paths still depend on that helper. The PR remains a narrow plausible bug fix and is not superseded by the related context provided.

## Best Possible Solution

Keep the PR open. Rebase or recreate the patch on current main, add `audio/webm` and `video/webm` to `.webm` plus `video/x-matroska` to `.mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts` plus a save/outbound filename test for MIME-only attachments without an original extension.
