## Summary

Keep open. Current main documents browser downloads as explicit control-server download interception endpoints, and the implementation only handles downloads when `/download` or `/wait/download` is called. I did not find evidence that the isolated browser profile's native `chrome://downloads` behavior from #18598 is repaired on `main`. The issue also has an open linked PR (#64558) specifically targeting the isolated download seam, which argues against conservative auto-close.
