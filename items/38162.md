## Summary

Keep PR #38162 open. Current main still does not implement `sessionSaveRedirectPath` or the supporting `postHookActions` coordination path, and the PR is still tied to real security-sensitive session-memory policy work rather than an obsolete cleanup candidate.

## Best Possible Solution

Keep #38162 open. Resolve or supersede the foundational #38161 hook contract first, then review the redirect implementation with explicit workspace containment, symlink/retraction semantics, absolute and relative escape tests, docs for the new hook context fields, and a clear product/security decision on session-save blocking semantics.
