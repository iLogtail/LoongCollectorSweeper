## Summary

Keep open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still lacks both behaviors targeted by PR #39790: transcript title extraction does not strip inbound metadata, and the computed outer session-row displayName is not passed into deriveSessionTitle. The linked #39722 user-visible TUI /sessions bug remains actionable.

## Best Possible Solution

Keep PR #39790 or linked issue #39722 open until main implements the narrow gateway fix: strip inbound metadata before session title text extraction, pass the computed row displayName into deriveSessionTitle or equivalent title logic, and add regression tests for metadata stripping plus external displayName precedence/fallback.
