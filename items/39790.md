## Summary

Current main still lacks the two behaviors PR #39790 targets: session title extraction does not strip OpenClaw inbound metadata, and `buildGatewaySessionRow` computes an outer `displayName` but does not pass it into `deriveSessionTitle`, so the TUI can still prefer a derived transcript title over the intended display label. The PR remains actionable rather than obsolete or safely closeable.

## Best Possible Solution

Keep PR #39790, or the linked issue #39722, open until main implements the narrow gateway fix: strip inbound metadata before extracting session title text, pass the computed row `displayName` into `deriveSessionTitle` or equivalent title logic, and add regression tests for metadata stripping plus external displayName precedence/fallback.
