## Summary

Keep PR #39790 open. Current main still lacks both behaviors the PR targets: session-title text extraction does not strip inbound metadata, and derived titles ignore the externally computed session-row displayName. The linked issue #39722 remains applicable, and the PR discussion/Greptile review supports the patch direction while noting only test coverage gaps.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, apply the targeted gateway changes, and add focused regression coverage in `src/gateway/session-utils.fs.test.ts` for stripping inbound metadata from title fields plus `src/gateway/session-utils.test.ts` for external displayName precedence and whitespace fallthrough.
