## Summary

Current `main` still shows both code paths the PR is trying to fix: derived session titles do not strip inbound metadata before extracting user text, and the session listing computes an external row `displayName` but never passes it into `deriveSessionTitle`. I did not find evidence that the report is obsolete or already implemented on `main`, so this should stay open for maintainer review rather than stale cleanup.

## Evidence

- **Session title extraction still does not strip inbound metadata:** `extractTextFromContent` normalizes with `stripInlineDirectiveTagsForDisplay(...)` for both string and array text content, but it never calls `stripInboundMetadata(...)`, so OpenClaw-injected inbound metadata can still flow into the first-user-message title path.
