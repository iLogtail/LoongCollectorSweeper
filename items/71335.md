## Summary

Current main still defaults `agents.defaults.memorySearch.sync.watch` to `true` unless the user explicitly overrides it, and the memory index manager still creates chokidar watchers whenever that flag is enabled. I found no gateway-mode-specific override in the memory config path, so #71335 remains actionable and should stay open.

## Evidence

- **Sync default is still unconditional true:** `resolveSyncConfig()` resolves `watch` as `overrides?.sync?.watch ?? defaults?.sync?.watch ?? true`, with no branch on `cfg.gateway.mode` before `resolveMemorySearchConfig()` returns the resolved settings.
  - file: [src/agents/memory-search.ts:350](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/memory-search.ts#L350)
