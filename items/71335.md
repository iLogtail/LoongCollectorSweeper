## Summary

Keep #71335 open. Current main still resolves `memorySearch.sync.watch` to `true` unless the user explicitly sets a default or per-agent override, and the gateway memory startup path passes that resolved config into the builtin/QMD memory managers without a gateway-mode override. Both memory backends still create chokidar watchers when the resolved sync config has `watch: true`, so the requested headless-gateway default is not implemented or superseded by #64549.

## Best Possible Solution

Implement a gateway-runtime-aware default in the memory sync config resolution path or at the gateway memory-manager startup boundary: default `agents.defaults.memorySearch.sync.watch` to `false` for headless gateway startup when no default/per-agent override is set, preserve explicit `true`, keep non-gateway CLI/desktop behavior at `true`, and add focused tests for builtin and QMD manager creation under gateway mode plus docs/schema help noting the conditional default.

## Risks / Open Questions

- The implementation needs a precise definition of gateway/headless mode because current config schema exposes `gateway.mode` as `local | remote`, where `remote` usually means this config targets another gateway rather than that this process is the gateway host.
- Changing the default must preserve explicit `sync.watch: true` overrides and avoid surprising desktop/CLI workflows that rely on near-real-time memory freshness.
