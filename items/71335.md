## Summary

Keep #71335 open. Current main still defaults `agents.defaults.memorySearch.sync.watch` to `true` whenever neither defaults nor per-agent overrides set it, and the memory-core builtin and QMD managers still create chokidar watchers when that resolved setting is true. I found no gateway/runtime-mode conditional in the memory config resolver or watcher setup, and the related #64549 is a separate open bug about watch correctness rather than this default-policy change.

## Best Possible Solution

Keep the issue open and implement a narrow, tested runtime-aware default: extend the memory search config resolution path or manager creation API with an execution context that can distinguish headless gateway server startup from CLI/desktop use, default `sync.watch` to false only for that gateway context, and add resolver plus builtin/QMD manager tests proving explicit `sync.watch: true` and `sync.watch: false` still win.
