## Summary

Current main has adjacent sanitizer and WebChat transcript-repair fixes, but I did not find source or test evidence that the specific Telegram leak format from this report (`[TOOL_CALL]...[/TOOL_CALL]`) is blocked. The report is recent, concrete, non-maintainer-authored, and not protected by an open paired PR in the provided context, so conservative cleanup should keep it open.

## Best Possible Solution

Keep this open. The best fix is a shared user-visible outbound guard that strips complete and truncated bracketed internal transcript blocks such as `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]`, then route Telegram final replies and streaming previews through it with regression tests for both delivery paths. The transcript-repair ordering should also be checked so synthetic repair content cannot become assistant-visible text before sanitization.
