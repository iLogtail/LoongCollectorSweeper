## Summary

Keep open. Current main still lacks the PR's requested behavior: TUI slash autocomplete is built from `getSlashCommands()`, and that function still merges built-in commands plus chat/gateway commands only. Plugin commands are registered and already listed in `/commands`, but they are not surfaced to the TUI autocomplete path. The PR is focused and not superseded by #71592, which is about command execution fallthrough rather than autocomplete.

## Best Possible Solution

Keep the PR open or reimplement the same narrow fix on main: merge `listPluginCommands()` into `getSlashCommands()` with normalization and deduplication, then add a unit test that registers and clears a plugin command and verifies it appears in TUI slash autocomplete without duplicating existing command names. Track local-mode command execution gaps separately under #71592.
