## Summary

Keep PR #56741 open. Current main still has the exact Telegram native-command authorization gap described by #56699: `/commands` resolve `topicConfig`, but command policy evaluation still passes `useTopicAndGroupOverrides: false`, so topic/group `groupPolicy` overrides are skipped. The PRâ€™s one-line change is still relevant and aligns command handling with the regular-message path, though a focused regression test should be added before landing.

## Best Possible Solution

Keep the PR open. The best maintainer path is to apply the one-line `useTopicAndGroupOverrides: true` fix for native Telegram commands and add a regression test in `extensions/telegram/src/bot-native-commands.group-auth.test.ts` or its helper that supplies a forum `topicConfig` with `groupPolicy: "disabled"` and verifies `/status`/`/reset` is blocked in that topic. Once shipped, use it to close #56699 and the related duplicate report #56777.
