## Summary

Current main still matches the reported failure class: vision-capable runs keep exposing the fallback `image` tool, and prompt image reference loading still intentionally ignores remote HTTP(S) image URLs. The linked PR #63710 is relevant and may address the mixed inline+URL path, but it is open and explicitly does not change fallback image-tool exposure, so this issue should not be auto-closed yet.

## Best Possible Solution

Keep #62514 open until maintainers either merge/adapt #63710 for mixed inline+HTTP(S) image refs and decide whether to add current-turn image-tool gating, or split the remaining behavior into a narrower issue covering fallback `image` tool exposure and silent local-ref drops. A complete fix likely belongs in `src/agents/pi-embedded-runner/run/images.ts`, `src/agents/pi-embedded-runner/run/attempt.ts`, and the image-tool/tool-inventory tests.
