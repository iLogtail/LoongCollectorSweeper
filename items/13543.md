## Summary

Keep #13543 open. Current main still does not implement per-tool sandbox execution routing. The existing implementation has session/agent sandbox modes, per-agent sandbox overrides, exec-only host routing, browser sandbox support only when the session already has a sandbox browser bridge, and sandbox tool allow/deny filtering, but no schema or runtime path that can force web_fetch/browser into a sandbox while exec/message/cron stay on host in the same session. The provided GitHub discussion already contains a maintainer/Codex review keeping this open, and current-main inspection did not find a later implementation.

## Best Possible Solution

Keep this as a core sandbox/tool-routing enhancement. A good fix should add a documented, schema-backed per-tool execution policy, route selected tool calls such as web_fetch/browser through a sandbox while keeping other tools such as exec/message/cron on host in the same session, and add regression tests plus docs that clearly distinguish execution placement from tools.sandbox.tools availability filtering.
