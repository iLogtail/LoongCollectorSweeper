## Summary

Keep open. Current main still rejects Synology Chat HEAD probes and still ACKs accepted webhook POSTs with 204 No Content, while PR #53441 targets exactly that linked bug with scoped Synology Chat changes and added test coverage. It is not implemented on main and is not superseded by the related #53439 issue, which is the bug report this PR fixes.

## Best Possible Solution

Keep #53441 open and either merge it after normal review/CI or apply the same focused fix on main: handle Synology Chat HEAD probes with 200, return 200 + {"success":true} for accepted webhook ACKs, rename the misleading no-content helper, and update Synology Chat webhook/integration tests from 204 expectations to the new ACK contract.
