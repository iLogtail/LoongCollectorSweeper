## Summary

Keep open: current `main` and the latest shipped release still implement the old Synology webhook behavior that this PR is trying to fix. The handler only accepts `POST`, returns `405` to `HEAD`, and ACKs successful webhook requests with `204 No Content`; the current tests still assert that behavior, and the available history does not show this landing elsewhere.
