## Summary

Keep #70197 open. Current main still ships the default-enabled Bonjour gateway discovery path, still resolves `@homebridge/ciao` to 1.3.6, and has no local ciao patch. The published ciao 1.3.6 source still runs the reported Windows `arp -a | findstr` command on a 15 second poll without a `windowsHide` option, so the observable Windows console-flash bug is not implemented/fixed on main. Related #71188 was closed as a duplicate of this item, which supports keeping #70197 as the canonical tracker.

## Best Possible Solution

Keep #70197 open and fix the bundled Bonjour/ciao path. Prefer consuming or upstreaming a `@homebridge/ciao` change that passes `{ windowsHide: true }` for the Windows ARP probe; if that cannot land quickly, carry a narrow dependency patch for the locked ciao version. Keep `OPENCLAW_DISABLE_BONJOUR=1`, `openclaw plugins disable bonjour`, and `discovery.mdns.mode="off"` as documented workarounds rather than treating them as the final fix.
