## Summary

Current main does not implement the requested mid-session plugin-tool refresh. Plugin install paths persist/refresh registry metadata but still tell users to restart the gateway, while embedded Pi agent runs construct plugin tools once, convert them to fixed customTools, and set a fixed active-tool allowlist on session creation. I found no source or test evidence of a plugin-install toolset-delta broadcast or per-turn tool re-query for already-active sessions.

## Best Possible Solution

Keep the issue open. The likely core fix is a generic plugin-registry/tool inventory invalidation path: after successful plugin install/update/enable that introduces tool registrations, refresh the active gateway registry as needed and make active sessions rebuild or merge their tool definitions before the next LLM turn. The implementation should cover CLI/message-channel install paths, gateway sessions, embedded Pi runs, and tests proving a newly installed tool is visible without a manual gateway restart.
