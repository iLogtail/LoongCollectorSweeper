## Summary

Do not auto-close. Current `main` has not merged this alias-specific cleanup, and the remaining install-log behavior the PR targets is still present in source. At the same time, part of the proposed discovery change appears redundant because discovery already prefers the manifest id when `openclaw.plugin.json` is present, so the item needs maintainer judgment rather than conservative cleanup closure.
