## Summary

Keep PR #72348 open. Current main still advertises Nextcloud Talk reactions and documents them as supported, but the Nextcloud Talk plugin does not register a channel message action adapter, so the shared `message` tool cannot dispatch `react` to the existing `sendReactionNextcloudTalk` sender. The PR remains a focused Nextcloud Talk-owned fix for #70110 and its review comments were reportedly addressed in the PR head commit.

## Best Possible Solution

Keep #72348 open and review/land this PR or an equivalent Nextcloud Talk-owned fix that registers a `ChannelMessageActionAdapter`, exposes `react` only when a configured account can support it, dispatches to `sendReactionNextcloudTalk`, and keeps focused regression coverage for both adapter dispatch and HTTP request shape. No current-main implementation or canonical superseding PR was found.
