## Summary

Keep open. Current `main` still undercounts `/status` cost for cached prompts: `buildStatusMessage` estimates cost from `input`/`output` only, while cache usage is shown on a separate line and the session usage pipeline already computes cost from full usage including `cacheRead`/`cacheWrite`. The PR likely needs a refresh because the implementation moved from `src/auto-reply/status.ts` to `src/status/status-message.ts`, but the underlying bug is not implemented on `main`.
