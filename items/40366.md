## Summary

Keep open. Current `main` still lacks both behaviors this PR adds: agent delivery still has no normalized-empty fallback, and the Slack preview path still clears draft state and returns without any user-visible fallback when partial/progress output occurs but no final reply is delivered. The PR is also against pre-plugin Slack paths, so it needs rebasing/porting rather than stale auto-close.

## Evidence

- **Agent delivery still gates on raw payload presence, not normalized deliverable output:** `deliverAgentCommandResult` builds an outbound payload plan and JSON projection first, but then only treats `!payloads || payloads.length === 0` as the no-reply case. There is no branch that detects `deliveryPayloads` collapsing to empty after normalization and emits the fallback text described in the PR.
