## Summary

Keep open. Current main has partial TUI status-state handling (`idle`, `sending`, `waiting`, `streaming`, `running`, disconnect handling, and a streaming watchdog), but it does not implement the issue's requested stable operator-facing label model. In particular, yielded/subagent waiting is still not represented as a distinct long-lived TUI state, and reconnecting is not surfaced as a separate label from disconnected/reconnected connection text.

## Best Possible Solution

Keep the issue open and implement a small TUI status-state model instead of rendering raw internal strings. Map active token deltas to `Generating`, tool/lifecycle work to `Working`, idle connected sessions to `Idle`, reconnect backoff to `Reconnecting`, and disconnected/unreachable state to `Disconnected`. For `sessions_yield`, add a durable signal or session/run metadata that lets the TUI show `Waiting` while child subagent completion is pending, with focused tests in `src/tui` and docs updated at `docs/web/tui.md`.
