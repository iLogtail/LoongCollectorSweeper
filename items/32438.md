## Summary

Keep #32438 open. Current main still writes only the configured JSONL file and offers formatted output only as a CLI/UI rendering layer, not as the requested persistent sibling `-formatted` log file for direct `tail`/`grep` when the Gateway is down. The earlier related issue #29378 is closed and PR #32442 was closed unmerged, so this issue remains the active tracker.

## Best Possible Solution

Keep this issue open until a maintainer either implements the core logger behavior or explicitly rejects the product direction. A complete fix should derive a sibling path from `logging.file`, write one human-readable line whenever the JSONL file is written, preserve existing JSONL consumers (`logs.tail`, Control UI, scripts), define formatted-file rotation/size behavior, add focused logger/log-tail tests, and update the public logging docs.
