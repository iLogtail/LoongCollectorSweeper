## Summary

Keep open. Current `main` still has the exact stale-reload path described in the issue: the Agents "Reload Config" action calls `loadConfig(state)`, but `loadConfig()` does not clear `configFormDirty`, and `applyConfigSnapshot()` refuses to replace `configForm` while dirty. The model dropdown is rendered from `configForm`, so an explicit reload can still show stale model state. The report is concrete, recent, and already has useful linked follow-up work in the timeline.

## Evidence
