## Summary

Keep #14526 open. Current main has manual backup creation/verification, update doctor checks, post-restart gateway/version/plugin diagnostics, and timing-only auto-update config. It still does not implement the requested opt-in safe-update workflow with automatic pre-update state backup, retention, channel or real-agent smoke verification, or automatic rollback/restore. The linked openclaw-auto-update-kit is useful operator evidence/workaround, not a merged core implementation or superseding tracker.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core update/auto-update path or explicitly decide this operator workflow should remain external. A complete core path would likely extend `openclaw update` and `update.auto` with pre-update backup creation, retention limits, post-update gateway/status/channel/real-agent smoke checks, and controlled rollback/restore built on the existing backup and health primitives, while coordinating with narrower backup and deep-health follow-ups.
