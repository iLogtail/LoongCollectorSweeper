## Summary

Keep #51028 open. Current main still sorts the Control UI Sessions table by `updatedAt`, and the gateway `sessions.list` result exposes only `updatedAt` for ordering. The related sub-agent label concern was split to #51030, but the central request here, first-class last meaningful activity ordering with heartbeat/background noise suppression, parent rollup, and visible reason labels, is not implemented on main.

## Best Possible Solution

Keep this issue open and implement a real activity model rather than closing as shipped. The likely path is to add persisted/API fields for `lastEventAt`, `lastMeaningfulActivityAt`, `lastVisibleConversationAt`, and a compact activity reason; classify transcript/lifecycle/system events so heartbeat and bookkeeping do not bubble sessions; roll child/sub-agent activity up to parents; then update `sessions.list`, `sessions.changed`, the Sessions table, and focused gateway/UI tests. The sub-agent label follow-up can remain handled by the split #51030 work, but it does not supersede the sorting/rollup request here.
