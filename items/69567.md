## Summary

Keep PR #69567 open. Current main still constructs base64 Basic auth headers in custom checkout blocks and passes them to git via http.extraheader without registering the derived value with GitHub Actions masking. The provided PR discussion and demo evidence support the proposed hardening, and there is no canonical related item in the supplied context that supersedes this workflow-specific fix.

## Best Possible Solution

Keep the PR open and review it as a narrow CI security hardening. The best path is to add `echo "::add-mask::${auth_header}"` immediately after every derived `auth_header` construction and before any git command can use or log it, with maintainers explicitly checking whether `.github/workflows/ci-check-testbox.yml` should be covered too.
