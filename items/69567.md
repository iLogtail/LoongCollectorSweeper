## Summary

Keep PR #69567 open. Current main still has the exact CI hardening gap the PR targets: `.github/workflows/ci.yml` builds base64 Basic-auth headers from `github.token` in 12 custom checkout blocks and passes them to `git fetch` via `http.extraheader` without registering the derived value with `::add-mask::`. The PR is a focused security hardening change, and the existing discussion includes useful positive review evidence from Greptile plus the author’s manual masking demo. It is not implemented on main and is not superseded by another provided item.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to merge or request a small revision that adds `echo "::add-mask::${auth_header}"` immediately after every derived `auth_header` assignment and before any git command that can use it, ideally covering the matching custom checkout in `ci-check-testbox.yml` as well. No core product code change is needed; this belongs in CI workflow hardening.
