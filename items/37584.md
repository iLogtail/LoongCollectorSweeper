## Summary

Keep open. Current main still has no per-agent `agents.list[].tools.web.search.*` override path for web-search provider, credentials, or timeouts. Per-agent `tools` is a strict policy/sandbox/exec object, while web-search config is resolved from global `tools.web.search` plus plugin-scoped provider config. The reporter reaffirmed the request on 2026-04-20, and the related items do not supersede this exact per-agent web-search settings gap.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a documented agent-scoped web-search settings contract, validate it in the agent schema/types, merge global/plugin/default/agent config when constructing the agent runtime, thread the effective search config or agent id through `createOpenClawTools()` -> `createWebSearchTool()` -> `runWebSearch()`/provider context, and add regression tests plus docs. If maintainers want this to stay plugin-owned, first add a generic SDK/core seam that lets web-search providers resolve per-agent credentials and settings safely.
