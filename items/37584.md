## Summary

Keep open. Current main still has global web-search provider configuration only; per-agent `agents.list[].tools.web.search.*` is neither accepted by the strict agent tools schema nor threaded into the web_search runtime/provider context. The reporter also confirmed on 2026-04-20 that the issue is still valid, and there are no protected labels or maintainer author association in the provided issue context.

## Evidence

- **Agent tools schema excludes nested web settings:** `AgentToolsSchema` is a strict object containing policy fields, `elevated`, `exec`, `fs`, `loopDetection`, and `sandbox`; it has no `web` or `search` field, so `agents.list[].tools.web` is not an accepted config surface.

  - file: [src/config/types.tools.ts:304](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/config/types.tools.ts#L304)
