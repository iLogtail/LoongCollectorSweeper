## Summary

Keep open. Current main still does not implement a configurable Gemini web search base URL: the Google plugin manifest accepts only apiKey/model for webSearch, the runtime hardcodes DEFAULT_GOOGLE_API_BASE_URL, the cache key does not include a base URL, and the public Gemini search docs do not document a baseUrl override. The PR discussion also shows the author addressed the main review concern by normalizing trailing slashes, so this remains useful work rather than conservative cleanup material.

## Best Possible Solution

Keep this PR open unless maintainers explicitly choose #57496 as the canonical replacement. The best implementation path is to port the feature to the current split files: add webSearch.baseUrl to extensions/google/openclaw.plugin.json, extend GeminiConfig and a safe string-normalizing resolver using normalizeGoogleApiBaseUrl, thread baseUrl through executeGeminiSearch/runGeminiSearch and the cache key, update docs/tools/gemini-search.md, and keep focused tests for default, override, trailing-slash normalization, and malformed config handling.
