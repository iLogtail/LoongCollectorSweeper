## Summary

Keep #49350 open. Current main still does not resolve both Google Chat UX bugs: reply delivery sends chunked text directly to the Google Chat API without a Google Chat markdown conversion/strip step, and the typing display-name fallback still checks only account config name and `agent.name` before hardcoded `OpenClaw`, not `agent.identity.name`. The related PR context is useful but not sufficient for cleanup closure: #49361 is still open and only covers the typing fallback, while #49406 is closed/unmerged and covered the markdown half.

## Best Possible Solution

Keep the issue open and land a focused Google Chat plugin fix: add a tested Google Chat text-normalization helper that converts common model Markdown to Google Chat text syntax or plain text before both reply delivery and outbound `sendText`/`sendMedia`, and update `resolveBotDisplayName` to fall back through account config name, `agent.name`, `agent.identity.name`, then `OpenClaw`. The existing related PRs can be used as prior analysis, but main needs one complete, tested implementation.
