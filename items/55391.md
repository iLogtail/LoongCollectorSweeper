## Summary

The request is for a domain-specific external memory/graph integration, and current main explicitly scopes optional memory/integration work to plugins rather than OpenClaw core. The repo already exposes memory-plugin seams that an Engramic/graph adapter could use, so this belongs as a ClawHub/community plugin, not a core issue.

## Close Comment

Closing after Codex review. This looks like a useful idea, but it does not fit OpenClaw core scope as the codebase stands today. Current main explicitly keeps optional capability in plugins rather than core (`VISION.md:54-64`), treats memory as a plugin slot (`VISION.md:63-64`), and already exposes memory-plugin seams such as `registerMemoryCapability`, `registerMemoryCorpusSupplement`, and public artifact export for companion integrations (`docs/plugins/sdk-overview.md:102-103`, `docs/plugins/sdk-overview.md:195-215`). The documented path for third-party capability like this is a community plugin published via ClawHub (`docs/plugins/community.md:9-22`). I also found no existing Engramic/Grey Matter implementation in this repo, so this is not cleanup for an in-tree subsystem. If someone wants to pursue this, the right path is a separate ClawHub/OpenClaw plugin repo built on the memory plugin APIs, not a core OpenClaw issue.
