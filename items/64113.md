## Summary

PR #64113 adds Yep as a new optional third-party web-search provider bundled in OpenClaw core. Current main does not already ship Yep, but the current project direction and public plugin API put this kind of provider integration on the external plugin/ClawHub path: optional capabilities should usually ship as plugins, plugin promotion belongs in ClawHub, and web-search providers can already be implemented through `api.registerWebSearchProvider(...)` with exported SDK subpaths. The PR author is not a maintainer and the provided labels are not protected, so conservative cleanup can close this as a ClawHub/plugin-distribution item rather than keep a core-bundling PR open.

## Best Possible Solution

Do not merge Yep as another bundled core plugin by default. The best path is to move the same provider into an externally maintained OpenClaw plugin, declare its manifest/config contracts, and publish it through ClawHub or npm so users can install it with `openclaw plugins install`. If maintainers later want Yep bundled officially, that should be a deliberate product/security ownership decision rather than conservative cleanup keeping this PR open.
