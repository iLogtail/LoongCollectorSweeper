## Summary

Keep #52960 open. Current main has chat streaming, tool cards, queued follow-ups, Stop/abort controls, and narrow compaction/fallback status toasts, but it does not implement the requested compact active-run progress model with phase, latest action, last activity, elapsed time, subagent/tool waiting state, or reconnect-during-run visibility. The related issue comment correctly notes PR #31192 was delivery-queue retry work and not this Control UI feature.

## Best Possible Solution

Keep this issue open and implement a Control UI active-run progress model fed by gateway chat, agent lifecycle, tool, subagent, delivery, and reconnect signals. Render a compact strip near the thread/composer with current phase, latest meaningful action, last activity, elapsed time, and reconnect count; add focused UI/controller tests and update Control UI docs.
