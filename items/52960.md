## Summary

Keep #52960 open. Current main has chat streaming, tool cards, queue/stop controls, and narrow compaction/fallback indicators, but it does not implement the requested compact active-run progress model with phases, latest meaningful action, last activity, elapsed time, subagent/tool waiting state, or reconnect-during-run visibility. The only related comment correctly notes PR #31192 was delivery-queue retry work and not this UI feature.
