## Summary

Keep #13613 open. Current main still lacks multi-gateway cron coordination: cron config/schema/docs expose local cron fields only, scheduler execution uses local JSON/state plus an in-process lock chain, status exposes no leader/coordination health, and the current plugin SDK does not provide a scheduler lock-provider seam. The provided maintainer discussion already kept this open, and related #13582 is a separate closed local scheduler reliability issue rather than a canonical HA coordination tracker.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements explicit HA cron coordination in the cron scheduler, or first adds a documented scheduler/lock-provider plugin API that lets Redis, etcd, DynamoDB, or similar backends coordinate due-job ownership safely. The path should include config/schema/docs, status or CLI health/leader visibility, and multi-process or multi-node regression coverage. Treat #13582 as separate shipped/local scheduler reliability work, not as the canonical tracker for this HA request.
