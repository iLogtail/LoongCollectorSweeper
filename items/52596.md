## Summary

Keep #52596 open. Current main has the pre-tool hook primitive and plugin approval support, but it does not implement the requested framework-owned session PIN gate: there is no `session.auth.pin` config/schema field, no session-authenticated state on `SessionEntry`, and no trusted core PIN verification path before tool execution. This is security/core session enforcement rather than a clean ClawHub-only optional integration.
