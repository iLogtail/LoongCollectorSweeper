## Summary

Keep #41949 open. Current main has meaningful mitigations, including browser snapshot caps, optional efficient snapshots, persisted tool-result caps, and overflow recovery. But the issue asks for browser interaction output to be task/context-budget aware by default; current browser snapshots are still primarily raw/structural dumps with size caps, and efficient mode is opt-in or config-driven rather than an automatic remaining-context policy.

## Best Possible Solution

Keep the issue open and narrow it to a browser-plugin enhancement: make browser snapshot/tool output context-budget aware by default, likely in extensions/browser/src/browser-tool.actions.ts and extensions/browser/src/browser/routes/agent.snapshot.plan.ts, using remaining context/config to prefer efficient or scoped snapshots, preserve actionable refs, and add tests/docs for large-page multi-step flows.
