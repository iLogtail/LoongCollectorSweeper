## Summary

Keep open. Current main still renders the Control UI chat transcript as a polite live log while streaming content is appended inside that same region. The related PRs in the issue discussion describe plausible fixes, but the shipped/main code has not adopted `aria-live="off"` or `aria-busy` behavior yet.

## Best Possible Solution

Keep #65538 open as the canonical accessibility bug. Land a focused Control UI fix in `ui/src/ui/views/chat.ts` that suppresses live-region announcements while sending or streaming, add targeted tests in `ui/src/ui/views/chat.test.ts` for idle, sending, streaming, and final-render states, then close this issue from the landed PR and mark unmerged duplicate PRs as superseded.
