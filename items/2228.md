## Summary

本 PR 针对 Prometheus 抓取器在遇到非 2xx 响应（如 404）时错误解析指标及样本计数统计异常的问题提出了有效的核心修复。修改收紧了指标名称首字符合法性校验以符合 Prometheus 官方数据模型规范，并在抓取失败时清理事件组及重置样本计数。审查讨论中维护者对 ClearEventGroup 后调用 SendMetrics 的行为提出合理疑问，作者尚未完全澄清该下游影响。属于有价值的核心缺陷修复候选，需维护者进一步代码审查、确认空事件组发送安全性或调整逻辑，并补充相关单元测试后合并。

## Best Possible Solution

建议维护者继续审查本 PR。重点确认 ClearEventGroup 后 SendMetrics 发送空事件组是否符合下游 Pipeline 预期，或是否应直接跳过 SendMetrics 调用。验证 IsValidMetric 的首字符校验边界是否完全覆盖 Prometheus 规范。若逻辑确认无误且 CI 通过，可批准合并。建议作者补充针对 4xx/5xx 响应边界及非法指标名称的单元测试。
