## Summary

本 PR 针对 Prometheus 抓取器在遇到非 2xx 响应（如 404）时错误解析指标及样本计数统计异常的问题提出了核心逻辑修复。修改涉及收紧 IsValidMetric 首字符校验以对齐 Prometheus 数据模型规范，并在抓取失败时清理 EventGroup 及修正元数据。目前 PR 处于维护者代码审查与技术讨论阶段，作者已回应规范依据疑问，属于有效的核心缺陷修复候选，应保持开启等待审查结论、CI 验证与合并。

## Best Possible Solution

建议维护者继续审查 ClearEventGroup 调用后 SendMetrics 的下游行为是否符合预期（确认空事件组发送是否安全或需跳过发送），并验证 IsValidMetric 的首字符校验逻辑完全覆盖 Prometheus 规范边界。若核心逻辑无误且 CI 通过，可批准合并。建议作者补充针对 4xx/5xx 响应边界及非法指标名称的单元测试以提升覆盖率。
