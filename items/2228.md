## Summary

本 PR 修复了 Prometheus 抓取失败（如返回 404）时响应体被误判为有效指标的问题，通过严格限制指标名称首字符范围优化了合法性校验逻辑，并修正了 up 状态为 false 时的样本计数与 EventGroup 清理行为。当前处于代码审查阶段，维护者已就校验依据与清理逻辑提出疑问，作者已引用 Prometheus 官方数据模型规范进行回应。该修改针对核心抓取链路中的真实缺陷，具有明确修复价值，需维护者进一步复核后合并。

## Best Possible Solution

维护者需复核 IsValidMetric 的字符白名单逻辑是否完全覆盖 Prometheus 命名规范（[a-zA-Z_:][a-zA-Z0-9_:]*），并确认 ClearEventGroup 后调用 SendMetrics 的遥测上报行为是否符合预期（如空 EventGroup 是否会被下游正确丢弃或仍需上报元数据）。若审查与 CI 通过，可批准合并；若需调整实现细节或补充边界单元测试，请在 PR 评论中给出具体修改建议。
