## Summary

本 PR 修复了 Prometheus 抓取返回非 2xx 状态码（如 404）时，响应体被误判为有效指标及样本计数错误的问题。修改涉及指标名首字符合法性校验及 upState 为 false 时清理缓存的逻辑，符合 Prometheus 官方数据模型规范。当前维护者审查仍在进行中，需确认 ClearEventGroup 后调用 SendMetrics 的下游影响及补充边界测试，具备明确合并价值，应保持开启。

## Best Possible Solution

建议维护者继续审查 ClearEventGroup 与 SendMetrics 的交互逻辑，确认空 EventGroup 发送是否符合下游预期或应直接跳过。若逻辑无误且 CI 通过，可批准合并；建议作者补充针对 4xx/5xx 响应及非法指标名称的单元测试以覆盖边界场景。
