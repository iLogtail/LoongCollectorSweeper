## Summary

本 PR 修复了 Prometheus 抓取返回 404 等非指标响应时被误判为有效指标的问题，通过严格校验指标名称首字符及在 up 状态为 false 时清理 EventGroup 来保证计数准确。当前处于代码审查阶段，维护者已就实现细节提问且作者已引用官方规范回复，属于核心抓取链路的真实缺陷修复，需维护者进一步复核后合并。

## Best Possible Solution

维护者需继续复核 IsValidMetric 的白名单逻辑是否完全符合 Prometheus 数据模型规范，并确认 ClearEventGroup 后调用 SendMetrics 的下游行为是否符合预期。若 CI 通过且审查无异议，可批准合并；若需补充边界单元测试或调整清理时机，请在 PR 评论中给出具体修改建议。
