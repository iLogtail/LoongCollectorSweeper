## Summary

本 PR 修复了 Prometheus 抓取器在遇到非 2xx 响应（如 404）时错误解析指标及统计样本数的问题。修改了 IsValidMetric 的首字符校验逻辑以对齐 Prometheus 官方规范，并引入 ClearEventGroup 在抓取失败时清理事件组及修正计数。目前维护者已就校验依据及清理逻辑对下游的影响提出审查意见，作者已回复并提供官方文档链接。该 PR 属于有效的核心缺陷修复，处于正常审查流程中，应保持开启等待进一步评估与合并。

## Best Possible Solution

建议维护者继续审查 ClearEventGroup 调用后 SendMetrics 的下游行为是否符合预期，并确认 IsValidMetric 的首字符校验逻辑完全对齐 Prometheus 数据模型规范。若核心逻辑无误且 CI 通过，可批准合并。建议作者补充针对 4xx/5xx 响应边界及非法指标名称的单元测试以提升覆盖率。
