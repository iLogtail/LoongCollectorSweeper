## Summary

本 PR 修复了 Prometheus 抓取返回 404 等非成功响应时，响应体内容被误判为有效指标导致计数错误的问题。修改涉及 core/prometheus/Utils.cpp 的指标名首字符合法性校验，以及 StreamScraper 在 upState 为 false 时清理缓存事件并修正样本计数。当前处于代码审查阶段，维护者已就实现细节与规范依据提问，作者已引用 Prometheus 官方数据模型规范回复。该 PR 针对核心抓取链路的真实缺陷，逻辑清晰且具备合并价值，需维护者继续复核下游行为与边界用例后合并。

## Best Possible Solution

建议维护者继续审查 ClearEventGroup 后调用 SendMetrics 的下游影响，确认空 EventGroup 的发送是否符合预期或是否应直接跳过发送。若逻辑无误且 CI 通过，可批准合并；建议补充针对 4xx/5xx 响应及非法指标名称的单元测试以覆盖边界场景。
