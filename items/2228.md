## Summary

本 PR 旨在修复 Prometheus 抓取场景中 404 响应体被误判为有效指标的问题，并优化 up 状态为 false 时的样本计数逻辑。当前处于代码审查阶段，维护者已就指标名称校验规范及 ClearEventGroup 调用时机提出疑问，作者已补充 Prometheus 官方数据模型文档链接。该修改涉及核心抓取与元数据上报逻辑，需维护者进一步评估实现细节与规范的一致性后合并。

## Best Possible Solution

维护者需复核 IsValidMetric 的白名单逻辑是否完全覆盖 Prometheus 规范（如首字符限制、数字与冒号的使用场景），并确认 ClearEventGroup 后调用 SendMetrics 是否符合预期遥测上报行为（避免上报空 EventGroup 引发下游异常）。若审查通过且 CI 检查通过，可批准合并；若需调整实现细节或补充单元测试，请在 PR 评论中给出具体修改建议。
