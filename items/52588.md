## Summary

Keep open: current `main` does not implement the proposed `OPENCLAW_TUI_USER_BG` / `OPENCLAW_TUI_USER_TEXT` feature, and the PR as described still leaves a readability regression for background-only overrides.

## Evidence

- **Current theme code has no user-color env override path:** `palette` is still exported directly from `lightMode ? lightPalette : darkPalette`; there is no `readHexColorEnv` helper and no `OPENCLAW_TUI_USER_BG` / `OPENCLAW_TUI_USER_TEXT` handling on `main`. That rules out `implemented_on_main`.
  - file: [src/tui/theme/theme.ts:128](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/tui/theme/theme.ts#L128)
  - command: `nl -ba src/tui/theme/theme.ts | sed -n '1,220p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **User messages always use both palette colors:** The TUI user-message component always renders with `theme.userBg` and `theme.userText`. Any feature that allows overriding only the background needs matching text-color derivation or validation to stay readable.
  - file: [src/tui/components/user-message.ts:7](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/tui/components/user-message.ts#L7)
