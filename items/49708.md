## Summary

Keep open. Current main appears to have fixed the doctor false-positive comparison path by canonicalizing transcript references, but it still resolves and persists session transcript paths through the real sessions directory. The linked PR #49796 is the right focused fix path, but it is still open/unmerged in the provided context, so the original user problem is not fully implemented on main.

## Best Possible Solution

Keep #49708 open until #49796 or an equivalent patch lands. The best fix is to retain realpath-based containment checks for safety, but return and persist logical paths under the caller-visible sessions directory or a relative path, while keeping the current doctor canonical comparison for legacy realpath entries.
