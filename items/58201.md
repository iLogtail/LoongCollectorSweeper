## Summary

Keep open. Current `main` does not implement agent-level plugin configuration overrides: per-agent config has no `plugins` field, agent entries are strict-schema objects, and plugin config remains documented/generated only under top-level `plugins.entries.*`. The request is coherent, recent, and still represents missing behavior, although plugin authors can partially work around storage isolation by using agent-specific runtime directories.

## Evidence

- **Per-agent config type has no plugin override field:** `AgentConfig` lists per-agent overrides such as `skills`, `memorySearch`, `sandbox`, `params`, `tools`, and `runtime`, but no `plugins` property.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/types.agents.ts#L70)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Agent config schema is strict and excludes plugins:** `AgentEntrySchema` enumerates allowed agent keys and ends with `.strict()`. A proposed `agents.list[].plugins` object would be an unknown key, not a supported config surface.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/zod-schema.agent-runtime.ts#L812)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Plugin config exists only at top level:** The root config schema defines `plugins` as a top-level object with `entries`, `allow`, `deny`, `load`, `slots`, and `installs`; there is no parallel agent-scoped plugin block.
  - file: [src/config/zod-schema.ts:939](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/zod-schema.ts#L939)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Generated config metadata documents only top-level plugin entries:** Generated config help includes `plugins.entries.*.config` and related top-level paths, but there is no generated `agents.list[].plugins` path.
  - file: [src/config/schema.base.generated.ts:27242](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/schema.base.generated.ts#L27242)
  - command: `rg -n 'agents\.list\[\]\.plugins|agents\.list\.\*\.plugins|agents\.list.*plugins' docs src/config/schema.base.generated.ts src/config/doc-baseline.ts src/config/doc-baseline.integration.test.ts`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **User-facing agent docs mention per-agent skills, not per-agent plugins:** The `openclaw agents` docs call out per-agent skill visibility via `agents.defaults.skills` / `agents.list[].skills`; they do not document any per-agent plugin config override.
  - file: [docs/cli/agents.md:37](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/cli/agents.md#L37)

That supports plugin-authored isolation strategies but does not implement the requested override mechanism.
  - file: [docs/plugins/sdk-runtime.md:34](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/plugins/sdk-runtime.md#L34)
