## Summary

Current `main` does not implement the requested generic agent-level plugin override surface. `AgentConfig` has no `plugins` field, and the agent config docs enumerate per-agent overrides without any plugin-entry merge contract. There are plugin-side/per-memory workarounds on `main` for agent-scoped storage, but they do not satisfy this issue’s requested core config behavior.

## Evidence

- **Agent config type has no per-agent plugins field:** `AgentConfig` includes fields like `workspace`, `memorySearch`, `sandbox`, `tools`, and `runtime`, but no `plugins` property or related merge hook.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/config/types.agents.ts#L70)
