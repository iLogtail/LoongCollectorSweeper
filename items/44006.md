## Summary

Keep open. Current `main` still sends low-text PDFs through `@napi-rs/canvas` rasterization on inbound file extraction and PDF-tool fallback, and I found no musl/CPU-feature guard, no PDF-preview disable switch, and no changelog/history evidence that this crash path was fixed after `v2026.3.8` or in the latest release `v2026.4.22`. This is not safely closable as implemented or obsolete.
