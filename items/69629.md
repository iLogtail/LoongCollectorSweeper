## Summary

Current main does not implement the requested Discord per-channel slash-command visibility. OpenClaw starts multiple Discord accounts and enforces guild/channel/user authorization when a native command is invoked, but the native command registration path still deploys the command set at the application level for each bot account; docs explicitly say commands may still appear in Discord UI for unauthorized users. The related routing/authorization reports are useful context, but they do not cover this remaining command-palette visibility request.

## Best Possible Solution

Keep #69629 open as a Discord plugin feature/design item. The likely path is to add a Discord-native command visibility layer tied to channels.discord.accounts.<id>.guilds.<guildId>.channels where Discord supports it, or document the platform limitation and provide a first-class text-only/native-disabled workaround for multi-bot same-guild deployments. The existing execution authorization should stay as the safety backstop.
