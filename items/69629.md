## Summary

Keep #69629 open. Current main protects Discord slash-command execution with account/channel allowlists, but it does not implement the requested Discord UI visibility scoping for multi-bot same-guild setups. The docs explicitly acknowledge that commands may remain visible to unauthorized users, and the implementation still registers/clears native commands at the application level rather than per configured guild channel.

## Best Possible Solution

Keep #69629 open for the bundled Discord plugin. A maintainer-grade path is to investigate Discord's current application-command permissions/contexts support, then either scope registered commands or command permissions to the configured `channels.discord.accounts.<id>.guilds.<guild>.channels` set for each account, or document the Discord platform limitation and add an explicit supported text-only/native-disable workflow for multi-bot same-guild deployments. Preserve the existing execution-time authorization and add focused tests around multi-account channel allowlists plus any new registration payload/permission behavior.
