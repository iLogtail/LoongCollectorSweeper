## Summary

Keep #69629 open. Current main already enforces Discord slash-command authorization and account-scoped routing, but it still documents that commands may remain visible in Discord's UI for unauthorized users. I did not find a current OpenClaw implementation or config contract that syncs per-account/per-channel Discord application-command visibility for multi-bot same-guild deployments.

## Best Possible Solution

Keep this open as a Discord plugin feature. The best path is a maintainer design that either adds safe Discord application-command permission sync from `channels.discord.accounts.*.guilds.*.channels`/bindings when OpenClaw has the required authorization, or documents and assists the Discord Server Settings > Integrations workflow as the supported workaround. Also add regression coverage for a same-guild multi-account setup so command deployment, execution auth, and any permission-sync payload stay aligned.
