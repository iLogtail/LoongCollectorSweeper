## Summary

Keep open. Current main still contains the exact semantic misclassification called out in the report: a stale/non-redirectable LiveSessionModelSwitchError is converted into a FailoverError with reason "overloaded". Some related hardening exists around bounded retries and longer subagent gateway waits, but the core reported behavior is not fully implemented or obsolete.

## Best Possible Solution

Keep this issue open. The best fix is to handle LiveSessionModelSwitchError as session/model-switch state rather than provider overload, add regression coverage that prevents it from being recorded as "overloaded", and add a focused current-main reproduction for local vLLM streaming/idle timeout behavior so healthy sub-second endpoints are not moved through a minutes-long fallback chain.
