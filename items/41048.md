## Summary

Current `main` does not implement this PR’s proposed behavior. The Telegram path still intentionally preserves unresolved-media placeholders in the agent body, and the only user-visible download warning on `main` is for thrown download errors, not the silent `getFile()` -> `null` path this PR targets.
