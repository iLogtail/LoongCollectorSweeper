## Summary

Keep open. Current main still lacks the buffered text-only block-reply collapse that this PR is trying to add: text-only blocks are intentionally not delivered live when block streaming is disabled, but the subscriber only collapses interim assistant text when reasoning is enabled and no onBlockReply is present. The existing regression tests cover the reasoning/no-block-reply path, not the buffered onBlockReply path from this PR.

## Best Possible Solution

Keep the PR open or ask the contributor for changes. The likely correct path is to finish the explicit buffered-block-reply flag end to end: set it when blockStreamingEnabled is false and onBlockReply is wired, carry it through RunEmbeddedPiAgentParams, run.ts, run attempts, buildEmbeddedSubscriptionParams, and SubscribeEmbeddedPiSessionParams, then collapse interim assistantTexts to the final assistant text under that flag with focused subscriber and runner propagation tests.
