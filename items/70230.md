## Summary

Keep open. Current main still exposes `health` as a service/channel/agent/session snapshot, not a connection-level reachability/latency surface. There is a gateway tick keepalive and client-side stall watchdog, but no `rttMs`, `lastHeartbeatAt`, or `connected` fields on the health response and no `gateway.ping` RPC. The provided related reports are channel-health issues, not canonical tracking for this gateway connection-level UI signal.

## Best Possible Solution

Keep the issue open and implement a core gateway transport-health surface: either add a small `connection` object to `health` with `connected`, `rttMs`, and `lastHeartbeatAt`, or add a lightweight `gateway.ping` RPC and document how UI clients should compute/display live reachability. The change should update gateway protocol docs, TypeScript response types, browser/client behavior, and targeted gateway/UI tests.
