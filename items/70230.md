## Summary

Keep #70230 open. Current main has gateway reachability via the existing WebSocket client and server tick keepalive, but the health RPC still returns only the service/channel/agent/session health summary and does not expose the requested connection-level `connected`, `rttMs`, or `lastHeartbeatAt` fields. I also found no `gateway.ping` method in the current gateway method inventory.

## Best Possible Solution

Keep the issue open and implement a small gateway connection-health contract. The narrow path is to add a lightweight `gateway.ping` or a `connection` object on `health` with `connected`, `rttMs`, and `lastHeartbeatAt`, backed by client/server timing state rather than channel probe duration, then cover it with gateway protocol tests, CLI JSON expectations, and docs.
