## Summary

Keep open. Current main still returns early for Feishu group messages with `requireMention=true` before command probing, so bare slash commands cannot bypass the mention gate. The PR remains a plausible focused fix, and its discussion contains useful follow-up: the broadcast scoping issue flagged by review was addressed in the PR by hoisting `hasCommandInGroup` and including it in broadcast active-agent selection. It should get maintainer review and regression tests rather than be closed as obsolete.

## Best Possible Solution

Keep the PR open and revise it into a Feishu-owned mention-gate fix: compute normalized command presence and effective command authorization before the group mention return, bypass the gate only for authorized command traffic, carry that bypass state into broadcast active-agent selection, and add targeted tests for single-agent and broadcast requireMention groups.
