## Summary

Keep open. Current main still lacks the core provider-hook API requested by this PR: `ProviderPrepareExtraParamsContext` does not expose `sessionKey` or `sessionId`, and the embedded runner/runtime-plan extra-param path does not thread those fields into provider preparation. The request is not a ClawHub-only plugin task because plugin authors need a core SDK/context field before they can set per-session LiteLLM `user` attribution dynamically.

## Best Possible Solution

Keep the PR open or ask for a refreshed patch that adds optional `sessionKey` and `sessionId` to the provider extra-param/wrap contexts, threads them through `AgentRuntimePlan.resolveExtraParams`, `runEmbeddedAttempt`, compaction/direct apply call sites, and covers the behavior with a focused extra-params provider-hook test. That would unblock LiteLLM or custom provider plugins from setting per-session request attribution without hardcoding channel-specific behavior in core.
