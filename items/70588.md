## Summary

Keep #70588 open. Current main has internal Discord webhook creation for thread-bound sessions, but the shared message-tool action list, Discord action discovery, and Discord action dispatch do not expose an agent-callable `webhook-create` action or return a generated webhook URL.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. If accepted, implement it in the Discord plugin-owned message-action surface: add a shared action name, Discord discovery/config gate, schema params for channel/name/avatar, runtime REST creation via Discord channel webhooks, safe result handling for the webhook URL, docs, and focused tests. A ClawHub tool plugin could also satisfy the optional workflow, but current main does not provide the requested built-in message-tool action.
