## Summary

Keep open. Current `main` does not implement xAI multi-agent model support; it intentionally supports the standard Grok 4.20 beta reasoning/non-reasoning models while excluding `multi-agent` ids from the bundled xAI provider.

## Evidence

- **Docs explicitly mark the model unsupported:** The current xAI provider docs say `grok-4.20-multi-agent-experimental-beta-0304` is not supported on the normal xAI provider path because it requires a different upstream API surface.
  - file: [docs/providers/xai.md:407](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/docs/providers/xai.md#L407)
  - command: `nl -ba docs/providers/xai.md | sed -n '400,430p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Built-in xAI catalog only includes non-multi-agent Grok 4.20 models:** The bundled catalog defines `grok-4.20-beta-latest-reasoning` and `grok-4.20-beta-latest-non-reasoning`, but no Grok 4.20 multi-agent model entry.
  - file: [extensions/xai/model-definitions.ts:148](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/xai/model-definitions.ts#L148)
  - command: `nl -ba extensions/xai/model-definitions.ts | sed -n '120,190p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Runtime resolution rejects multi-agent ids:** `resolveXaiCatalogEntry()` returns `undefined` for model ids containing `multi-agent`, which prevents the bundled xAI provider from resolving this family at runtime.
  - file: [extensions/xai/model-definitions.ts:215](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/xai/model-definitions.ts#L215)
  - command: `nl -ba extensions/xai/model-definitions.ts | sed -n '190,250p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Modern-model detection also excludes multi-agent refs:** `isModernXaiModel()` returns `false` when the normalized model id contains `multi-agent`, so the provider deliberately does not treat these refs as supported forward-compatible xAI models.
  - file: [extensions/xai/provider-models.ts:14](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/xai/provider-models.ts#L14)
  - command: `nl -ba extensions/xai/provider-models.ts | sed -n '1,260p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Tests lock in the current non-support behavior:** The xAI test suite asserts that the multi-agent id is excluded from modern model detection and that forward-compatible resolution for `grok-4.20-multi-agent-experimental-beta-0304` returns `undefined`.
  - file: [extensions/xai/web-search.test.ts:507](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/xai/web-search.test.ts#L507)
  - command: `nl -ba extensions/xai/web-search.test.ts | sed -n '430,530p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
