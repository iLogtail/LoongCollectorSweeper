## Summary

Keep #38204 open. Current main supports ordinary Grok 4.20 reasoning/non-reasoning refs, but it still deliberately excludes Grok 4.20 multi-agent model IDs and documents them as unsupported. Upstream xAI docs now describe `grok-4.20-multi-agent` on `/v1/responses`, so the remaining work is not just stale catalog cleanup; it needs xAI-provider product/implementation judgment and regression coverage.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. A good implementation would verify the current xAI multi-agent Responses contract, decide whether bundled `extensions/xai` should support it or whether it belongs in a separate plugin, then update model resolution, any payload handling such as the current xAI reasoning-field stripping, provider docs, and tests for both supported standard Grok 4.20 refs and multi-agent refs.
