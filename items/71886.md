## Summary

Keep open. Current main still serves the Control UI CSP with `connect-src 'self' ws: wss:` and no `https://tweakcn.com`, while the current browser UI has a tweakcn importer that normalizes theme links to `https://tweakcn.com/r/themes/{id}` and calls `fetch` from the browser. The PR’s proposed one-host CSP addition and regression assertion are still relevant and not implemented on main. Related configurable-CSP issue #47565 is broader and does not supersede this focused bug fix.

## Best Possible Solution

Keep PR #71886 open for maintainer review and likely merge the narrow CSP allowlist change plus regression assertion, preserving the broader configurable-header discussion as separate follow-up work.
