## Summary

Keep open. Current main has spawnedBy on session snapshots and session/tool-oriented events, but it does not implement the specific chat and non-tool agent broadcast payload enrichment requested by this PR. The PR remains a plausible focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review and either land a cleaned-up version of this PR or make an equivalent gateway/protocol change on main: add optional spawnedBy to chat and relevant non-tool agent broadcast payloads, keep the protocol schemas in sync, and include targeted tests for chat delta, flush, final-done, final-error, seq-gap, non-tool agent events, non-subagent omission, and any chosen lookup/cache behavior.
