## Summary

Keep #55507 open. Current main still accepts and resolves `memorySearch.outputDimensionality`, but the OpenAI memory embedding path does not pass it to the `/embeddings` request body as `dimensions`; the OpenAI batch path also omits it. The PR is stale against the current pluginized file layout, but it addresses a real remaining bug and the provided PR discussion shows the author fixed the earlier review concerns about default omission, range validation, and English test names.

## Best Possible Solution

Keep the PR open and ask for an update against current main: carry `outputDimensionality` through the shared remote embedding client/request body as OpenAI `dimensions`, validate only for supported OpenAI embedding-3 models without changing the default request shape when unset, mirror any SDK/package source as required by this repo, update the OpenAI batch request body, and add focused regression tests for inline and batch OpenAI memory embeddings.
