Keep open. ## Best Possible Solution

Keep #49660 open. The best path is a maintainer-reviewed revision in src/agents/session-write-lock.ts that adds startup-cleanup orphan-self detection without allowing contention acquisition to delete a just-created same-process lock, keeps session-dir realpath handling consistent, adds regression coverage for no starttime, matching starttime, PID recycle, symlinked sessions dirs, and active in-process lock preservation, and removes the accidental log file from the PR.
