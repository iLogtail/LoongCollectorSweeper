## Summary

Keep open. Current main does not implement the PR's compaction event observability contract: compaction start events still drop the upstream `reason`, compaction end events still omit `hasResult` and sanitized `errorMessage`, and tests assert the older payload shape. The PR discussion also contains concrete review feedback to address before merge, so this is actionable work rather than cleanup-close material.

## Best Possible Solution

Keep the PR open for maintainer review or restack. The best implementation is a focused update to `src/agents/pi-embedded-subscribe.handlers.compaction.ts` and its call site/tests that forwards upstream compaction `reason`, emits `hasResult` alongside `completed` and `willRetry`, sanitizes any `errorMessage`, and keeps normal successful compaction lifecycle logging at a non-warning level while preserving warnings for retry/failure paths.
