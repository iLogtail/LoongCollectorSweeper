## Summary

Keep PR #56394 open. Current main still formats ClawHub non-OK responses directly from the HTTP body and has no unauthenticated 429 login hint, while the open PR addresses the linked issue #56368 on the right OpenClaw-owned ClawHub client paths. The bot review identified a small test-quality fix, but not a reason to close the PR as obsolete.

## Best Possible Solution

Keep the PR open and have the author or maintainer refresh it against current main, preserve the production approach of adding the unauthenticated 429 hint in the shared ClawHub client, and fix the two silent negative tests before merge. This belongs in OpenClaw core because the affected behavior is the built-in `openclaw skills` and gateway ClawHub client error surface, not an optional external plugin.
