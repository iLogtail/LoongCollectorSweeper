## Summary

Keep open. Current main still has no refusal or warning when `sessions_send` targets a Slack/Discord-style `:thread:` session key. The tool resolves the supplied key, applies normal visibility/A2A policy, injects the inter-session message into the resolved session, and the announce path explicitly preserves thread IDs for delivery rather than rejecting them.
