## Summary

Keep open. Current `main` and the latest release still do not pre-install `iproute2` in the official Docker image. The repo only provides an opt-in custom-build path via `OPENCLAW_DOCKER_APT_PACKAGES`, which does not satisfy the issue's request for the published `ghcr.io/openclaw/openclaw:*` image.

## Evidence

- **Default runtime packages exclude iproute2:** The runtime Docker apt step installs `procps hostname curl git lsof openssl` and does not include `iproute2`.
  - file: [Dockerfile:162](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/Dockerfile#L162)
  - command: `nl -ba Dockerfile | sed -n '158,170p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Extra system packages are only opt-in at build time:** `OPENCLAW_DOCKER_APT_PACKAGES` is an optional build arg. The image can install extra packages only when a custom build sets that arg.
  - file: [Dockerfile:199](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/Dockerfile#L199)
  - command: `nl -ba Dockerfile | sed -n '199,207p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Docs describe a custom-image workaround, not an official-image default:** Docker docs document `OPENCLAW_DOCKER_APT_PACKAGES` and explicitly tell power users to bake extra system deps such as `git curl jq` into their own build.
  - file: [docs/install/docker.md:121](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/install/docker.md#L121)

- **Setup script only forwards extra apt packages during local builds:** `scripts/docker/setup.sh` passes `OPENCLAW_DOCKER_APT_PACKAGES` only when building `openclaw:local`; if `OPENCLAW_IMAGE` points to a remote tag, it pulls the image unchanged.
  - file: [scripts/docker/setup.sh:474](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/scripts/docker/setup.sh#L474)

## Risks / Open Questions

- The default compose setup drops `NET_ADMIN` on `openclaw-cli` (`docker-compose.yml:55`), so even if `iproute2` were bundled, the reporter's routing workflow may still need extra container-capability changes. That is a product/design question, not evidence that the issue is already resolved.
