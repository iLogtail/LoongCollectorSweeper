## Summary

Keep open. Current main has related probe/status cleanup, but the dedicated `openclaw gateway probe` path still can mark a target unreachable when WebSocket connect succeeds and the follow-up read/detail RPCs time out, which is the user-visible problem reported in #42019.

## Evidence

- **Issue context:** The report describes `Connect: ok` followed by an RPC timeout during cold startup, with later normal subagent/session recovery. The reporterâ€™s follow-up says they verified this was not just their bare-metal/Nix host issue.
- **Gateway probe still runs full read-detail RPCs:** `runGatewayStatusProbePass` calls `probeGateway` for each target without `includeDetails: false`, so `probeGateway` defaults to the full detail path for `gateway probe`.
