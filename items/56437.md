## Summary

Keep open. Current `main` already preserves system-prompt/bootstrap context during model fallback and exposes the active fallback model in status output, so the issue's reported 'raw cron payload only' behavior does not match current code. But the requested native `fallbackRunbook` config field is not implemented, and invalid fallback model refs are still detected at runtime rather than validated at config-load/doctor time.
