## Summary

Keep open. Current main still serializes the dashboard form state directly for `config.set` / `config.apply`; I do not see the focused #60971 behavior on main that would drop unresolved `__OPENCLAW_REDACTED__` placeholders while preserving placeholders backed by the original raw config.

## Best Possible Solution

Keep the issue open and implement the focused UI-side fix proposed in #60971: before form-mode `config.set` / `config.apply`, strip unresolved `__OPENCLAW_REDACTED__` paths that were not present in the original raw config, preserve redacted paths that do correspond to original config secrets, and add regression coverage in `ui/src/ui/controllers/config.test.ts` plus a helper test in `ui/src/ui/controllers/config/form-utils.node.test.ts`.
