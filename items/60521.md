## Summary

Keep #60521 open. Current main still matches the reporter's narrowed root cause: direct `POST /hooks/agent` normalization does not preserve `allowUnsafeExternalContent`, so the direct hook path dispatches a webhook-provenance agent turn without the opt-out flag. The later cron agent runner only skips wrapping when the agent-turn payload contains that flag, or when Gmail config opts out, so the reported direct request body cannot affect the wrapping decision today.

## Best Possible Solution

Keep the issue open for an explicit maintainer decision. If direct hook callers are allowed to opt out, add `allowUnsafeExternalContent?: boolean` to `HookAgentPayload`, preserve `payload.allowUnsafeExternalContent === true` in `normalizeAgentPayload()`, include it in direct dispatch replay scope if idempotency should distinguish trust mode, update `/hooks/agent` docs/security guidance, and add regression coverage proving direct `POST /hooks/agent` with the flag reaches `runCronIsolatedAgentTurn()` unwrapped. If direct callers should not control trust, close only after changing docs/API behavior to reject or clearly ignore the field.
