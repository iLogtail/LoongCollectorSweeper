## Summary

Keep #17925 open. Current main implements the Google/Gemini side as a bundled `gemini` web-search provider using Google Search grounding, but the ZAI/GLM side requested by the issue is still missing: the ZAI plugin has no `webSearchProviders` manifest contract, no `registerWebSearchProvider` registration, no web-search config/docs surface, and no native `tools: [{ type: "web_search" }]` passthrough path. The issue-linked Zhipu/ZAI PRs #10454 and #13814 are closed and unmerged in the provided context, and the latest issue discussion already keeps this tracker open for the remaining ZAI work.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider with manifest ownership, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Maintainers can separately decide whether Gemini's existing managed provider fully resolves the Google portion or whether automatic fallback semantics need a narrower follow-up.
