## Summary

Keep #71734 open. Current main still only has the normal priority-ordered `before_tool_call` hook plus basic plugin tool catalog projection; it does not expose the requested trusted pre-plugin tool policy tier, first-class tool metadata/display/safety registry, or core-tool decoration rule. The new linked #72082 is a draft implementation PR that may become the vehicle for this work, but it is not merged or shipped on the checked-out main.

## Best Possible Solution

Keep #71734 open and use #72082 as the implementation vehicle if maintainers accept the RFC direction. The best path is a focused host-hook PR that adds a bundled/trusted pre-plugin tool policy registration contract, rejects privileged registration by external plugins, projects plugin-owned display/category/safety metadata through catalog/effective inventory/UI lookup, explicitly decides core-tool decoration, and adds fixture tests for ordering, override resistance, metadata projection, and disablement cleanup. Close this issue only after #72082 or an equivalent implementation lands, or if maintainers deliberately choose #72082 as the sole canonical tracker.
