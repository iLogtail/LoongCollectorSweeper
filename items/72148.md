## Summary

Keep #72148 open. Current main still sends the user-facing `Automatic session resume failed, so sending the status directly.` prefix when exec approval follow-up session resume fails, and tests still assert that behavior. The PR is recent, focused, and not superseded by the related #72268 parser PR, which addresses a different nested-metadata parsing bug.

## Best Possible Solution

Keep #72148 open for maintainer review. The best path is to land a focused fix that removes the internal resume-failure prefix from user-facing exec approval fallback messages, logs resume failures internally with safe redaction/sanitization, and tracks retry/backoff separately unless maintainers want it in the same issue scope. Coordinate with #72268 only as a related parser fix; it does not replace this PR.
