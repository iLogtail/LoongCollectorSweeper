## Summary

Keep open. Current main partially addresses the safer-default half by rendering new systemd units with `KillMode=control-group`, but the central observability request remains missing: doctor/status do not inspect or surface `KillMode`, `TasksCurrent`, `MemoryCurrent`, or cgroup hygiene warnings for existing Linux gateway services.

## Best Possible Solution

Keep the issue open and implement the missing observability path: extend the Linux systemd service/runtime inspection to read `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID`; expose them in `openclaw gateway status --deep` human/JSON output; add doctor warnings and an explicit repair path that rewrites old `KillMode=process` units to the current safe default while respecting `OPENCLAW_SERVICE_REPAIR_POLICY=external`. Tests should cover parsing, warning thresholds, status JSON, and doctor repair prompts.

## Risks / Open Questions

- Changing defaults alone does not help existing installations that still have older unit files or local overrides unless doctor/status can detect and guide repair.
- Thresholds for `TasksCurrent` and `MemoryCurrent` need to be conservative and Linux/systemd-specific to avoid noisy warnings on legitimately busy gateways.
