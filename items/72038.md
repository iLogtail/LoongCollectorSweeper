## Summary

Keep #72038 open. Current main still finalizes normal text-only Telegram preview-streamed replies by editing the existing preview message, with no visible-since metadata, no long-lived-preview threshold, and docs/tests that still describe and assert the old in-place final edit behavior. The PR is an active, focused bundled Telegram plugin bug fix, and the provided review discussion shows the main P1 review note was addressed in the latest PR head.

## Best Possible Solution

Keep #72038 open for maintainer review. The best path is to land a focused bundled Telegram plugin fix that records when message previews become visible, carries that timestamp through active, superseded, and archived preview state, sends a fresh final for previews older than the accepted threshold, cleans up stale previews, resets active lane state so later assistant turns cannot rematerialize stale text, preserves short-preview edit-in-place behavior, and keeps the active/archived/send-failure regression tests plus docs/changelog updates.
