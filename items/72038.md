## Summary

#72038 should stay open. Current main still finalizes normal text-only Telegram preview-streamed replies by editing the existing preview message, and it has no visible-since metadata or long-lived-preview fresh-final branch. The PR is an active, focused Telegram plugin bug fix with review discussion showing the earlier stale-preview rematerialization concern was addressed in PR head dbc52c476c0c992d965be99a99ad7d89e0b4824f.

## Best Possible Solution

Keep #72038 open for maintainer review. The best path is to land a focused Telegram plugin fix that records when message previews become visible, carries that timestamp through active, superseded, and archived preview state, sends a fresh final for previews older than the accepted threshold, cleans up stale previews, resets active lane state so later assistant turns cannot rematerialize stale text, preserves short-preview edit-in-place behavior, and keeps the active/archived/send-failure regression tests plus docs/changelog updates.
