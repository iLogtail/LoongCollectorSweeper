## Summary

The request targets the wrong layer for OpenClaw core. Current main already gives agent-wide identity/context through injected workspace bootstrap files, while skills themselves are prompt instructions (`SKILL.md`), not a separate programmable runtime. For structured dynamic agent-policy context, the existing extension point is a plugin/context-engine, which fits the repo vision of keeping optional capability out of core.

## Close Comment

Codex review: this looks like the right idea for the plugin/ClawHub layer, not a new OpenClaw core surface. On current `main`, agent-wide identity/context already comes from injected workspace bootstrap files such as `IDENTITY.md`, `USER.md`, and `AGENTS.md` (`docs/concepts/agent.md:25`, `docs/concepts/system-prompt.md:99`). Skills themselves are `SKILL.md` prompt instructions that the agent reads on demand, not a separate programmable execution environment (`docs/tools/index.md:28`, `src/agents/system-prompt.ts:163`). For structured dynamic agent policy, OpenClaw already exposes a plugin context-engine hook with `systemPromptAddition` (`docs/concepts/context-engine.md:67`). Given the current architecture and `VISION.md` scope bar, this belongs as a ClawHub/plugin implementation rather than a core issue, so I’m closing it on that basis.
