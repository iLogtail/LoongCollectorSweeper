## Summary

Keep open. Current main still does not relay incoming ACP `system_event` stream events to the parent session for `streamTo: "parent"`; the parent relay handles assistant deltas and lifecycle events, then returns for every other stream. The PR is not obsolete, but it needs a current-main rework with regression coverage and the review-noted stall-timer handling before it should merge.

## Best Possible Solution

Keep #54471 open and rework it against current main: add a tested parent-relay branch for ACP system/progress events, validate where the event text and context key come from, count those events as progress by updating `lastProgressAt`/`stallNotified`, and preserve the existing assistant/lifecycle behavior. If the modern Codex path now uses `codex_app_server.*` instead of bare `system_event`, either broaden the relay deliberately or split that into a separate canonical ACP parent-stream visibility issue.
