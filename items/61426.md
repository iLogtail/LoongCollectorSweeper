## Summary

Keep open. Current main still mirrors successful isolated cron delivery into the main-session system-event queue, and the existing regression coverage explicitly expects that awareness event for isolated cron jobs. The linked fix PR #61658 described the same root cause and intended suppression for implicit/default delivery, but it was closed unmerged, so it does not supersede or ship the issue.

## Best Possible Solution

Keep the issue open and either revive the intent of PR #61658 or implement the same narrow fix on main: make cron awareness conditional on an explicit delivery target or configurable policy, so implicit/default `channel: last` isolated cron runs do not enqueue main-session awareness while deliberate explicit-target awareness remains covered by tests.
