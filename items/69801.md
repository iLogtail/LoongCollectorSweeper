## Summary

Keep open. Current main has an icon-only New session control in the chat composer toolbar, but it is wired to send `/new` against the current session rather than call `sessions.create`, switch to a newly created direct session key, and update `/chat?session=...`. The Sessions view exposes refresh/filter/table/edit/delete/navigation controls, not a create-new-session action. A gateway `sessions.create` API and tests already exist, so this is a real Control UI integration gap rather than a missing backend primitive. The related PR #61423 appears to cover similar remaining work but was previously kept open with unresolved implementation concerns, so it is not strong enough to close this issue as superseded.

## Best Possible Solution

Implement a focused Control UI create flow: add a clearly labeled New Chat/New Session action in the chat header and/or Sessions header, call `sessions.create` with the current agent and optional label/model/parent context, then switch to the returned key with the existing chat session switch/URL sync path. Use #61423 as useful prior work, but resolve its modal/review concerns before landing.
