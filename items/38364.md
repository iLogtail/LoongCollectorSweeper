## Summary

Keep #38364 open. Current main partially fixes the confusing no-output path by filtering silent `NO_REPLY` assistant output in Control UI events/history, but the central reply-to metadata request is still not implemented for `chat.send`: the protocol schema has no reply/parent field, the Control UI send path cannot collect or forward one, and the gateway-created `MsgContext` does not populate existing reply metadata fields for Control UI sends.

## Best Possible Solution

Keep #38364 open. The best path is an additive Gateway protocol field for `chat.send` reply metadata, Control UI composer/thread state that captures the selected parent message, gateway mapping into existing `MsgContext` reply fields plus an optional parent-content summary, and focused protocol/UI/server tests. Keep the current silent-output filtering as a partial fix.
