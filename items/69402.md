## Summary

Current main already fixes the reported media-preview auth gap. Instead of syncing `host.settings.token` into UI state after `hello`, the shipped implementation resolves Control UI media auth from `hello.auth.deviceToken`, then falls back to the explicit settings token/password, and appends that token to assistant-media preview URLs. Existing tests cover authenticated assistant media rendering, auth-token refresh, and server acceptance of paired operator device tokens. The behavior is also listed in the v2026.4.24 changelog.

## Best Possible Solution

Close this PR as already implemented. Keep the shipped shared Control UI auth resolver plus hello-issued device-token media path; if a new failure remains, it should be filed as a fresh, narrower bug against the current v2026.4.24-or-newer behavior with a reproduction.
