## Summary

Keep #6975 open. Current main has landed the related Telegram native-quote delivery fix from #71952, but it still does not implement the requested `channels.telegram.replyContextLength` configuration knob. The workaround discussed in the issue, `replyToMode: "all"`, and the new `reply_parameters.quote` plumbing improve native replies but do not let users configure how much quoted context OpenClaw includes.

## Best Possible Solution

Keep the issue open. Build on the landed #71952 native quote path by adding a typed and validated `channels.telegram.replyContextLength` setting, generated config metadata/UI hints, Telegram docs, and tests. Apply the configured limit when deriving Telegram reply quote text before passing it to `reply_parameters.quote`, while preserving existing `replyToMode` behavior and fallback to legacy replies.
