## Summary

Keep open. Current `main` does not implement a Telegram `replyContextLength` setting or any equivalent configurable quote-length control. The Telegram config/docs expose `replyToMode` and `textChunkLimit`, and the send path accepts a quote string but no length parameter. The March comments suggest `replyToMode` addressed a no-reply-threading setup issue, but that is different from the feature requested in #6975, so there is not strong evidence to close this item.
