## Summary

Keep #43564 open. Current main has mature OpenClaw skill discovery and per-agent skill allowlists for native prompt construction, plus ACP spawning, but it does not implement the requested ACP-session skill context injection, `sessions_spawn.skills`, automatic relevance selection, or default ACP skill set. The ACP spawn path still sends only the task text and runtime options into the ACP child session, and the current docs/tests describe ACP parameters without a skills parameter.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A likely implementation path is to add a typed ACP-safe skill selection surface to `sessions_spawn` and/or per-agent ACP defaults, reuse existing skill eligibility and per-agent allowlist logic, render bounded skill excerpts or skill references into the ACP initial prompt, and document/test behavior for Codex/Pi/OpenCode/Gemini ACP harnesses. If maintainers decide full skill contents should not cross the ACP boundary, document the supported alternative explicitly.
