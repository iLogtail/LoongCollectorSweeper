## Summary

Keep #59730 open. Current main has not implemented the requested Vitest consolidation: the repo still carries many separate Vitest config files, the root config delegates to a project graph, package/test scripts still pass explicit `--config` files, and docs describe shard configs rather than `VITEST_MODE`. This remains actionable repo-internal test infrastructure work, with no protected-label or maintainer-authored close override needed and no duplicate/superseding item in the provided context.

## Best Possible Solution

Keep the issue open and treat it as a test-infrastructure refactor. The best path is to design a consolidation that preserves the current shard/changed-lane semantics in `test/vitest/`, `scripts/test-projects*.mjs`, package scripts, CI, and testing docs, with focused regression coverage for routing and config selection before removing old configs.
