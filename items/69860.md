## Summary

Keep #69860 open. Current main still lacks Feishu-native exec approval action cards: Feishu handles general card.action.trigger callbacks and same-chat approval authorization, but it does not expose channels.feishu.execApprovals, native approval runtime/delivery wiring, or docs claiming Feishu as a native exec approval client. Open PR #60328 is the focused implementation path, but it remains unmerged with unresolved review concerns, so it should not close the user-facing feature tracker yet.

## Best Possible Solution

Keep #69860 open as the Feishu native exec approval tracker. The best path is to finish PR #60328 or a maintained successor that adds Feishu execApprovals schema/docs, native approval runtime/delivery targets, card rendering for pending/resolved/expired states, callback handling through the shared gateway approval resolution path, and targeted auth/routing/expiry tests while addressing the approver-ID validation concern.
