## Summary

Keep #69860 open. Current main still lacks Feishu-native exec approval action cards: Feishu has general interactive card callback support and approval authorization, but no `channels.feishu.execApprovals` schema, native approval delivery/runtime wiring, or docs claiming Feishu as a native exec approval client. Open PR #60328 tracks a plausible implementation path, but it is unmerged and still has an unresolved review concern, so it is not enough to close the feature request.

## Best Possible Solution

Keep the issue open as the Feishu native exec approval tracker. The best path is to update and land PR #60328 or an equivalent successor that adds Feishu `execApprovals` config/schema/docs, native delivery targets, card rendering for allowed decisions, callback handling that resolves approvals through the shared gateway path, resolved/expired card updates, and focused Feishu/native approval tests while addressing the existing approver-ID review blocker.
