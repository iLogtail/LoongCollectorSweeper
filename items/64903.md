## Summary

Keep open. Issue #64903 has a concrete Android 16 crash trace, and current `origin/main` still uses a long-running `dataSync` foreground service on `targetSdk 36` without an `onTimeout` path or fallback for exhausted `dataSync` foreground-service budget. This is still an actionable Android app bug, not an implemented, duplicate, ClawHub, or stale-info close.

## Best Possible Solution

Keep #64903 open and fix the Android foreground-service lifecycle. The likely maintainer path is to stop treating the persistent gateway connection as unbounded `dataSync`, add Android 15/16 timeout handling such as `Service.onTimeout(int, int)` with `stopSelf()` or a safer reconnect strategy, and add regression coverage or a documented device test for the SDK 36 timeout/exhausted-budget path.
