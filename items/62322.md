## Summary

Keep open. Current main still contains the reported recovery ingredients: `incomplete-text` recovery returns `prefill: true`, the abort hint is injected and then cleared as a one-turn marker, and abort/session persistence still toggles `abortedLastRun`. The linked PR #62346 is still open and narrow, and the issue comments explicitly say broader recovery stability remains beyond that patch.

## Best Possible Solution

Keep #62322 open until main ships an equivalent of the #62346 no-prefill behavior and maintainers decide whether the broader aborted-run state should become a stable structured recovery fact. The likely implementation path is a focused regression test combining an aborted embedded run with an `incomplete-text`/reasoning-only assistant turn, then changes in `src/agents/pi-embedded-runner/thinking.ts` or the newer `run/incomplete-turn.ts` path plus `src/auto-reply/reply/body.ts`/session persistence so partial assistant text and abort state cannot produce misleading next-turn context.
