## Summary

Keep open. Current main still wires the Control UI New session toolbar button directly to the `/new` reset path with no confirmation dialog, relocation, danger state, or undo mitigation. The focused related PR #27110 proposed this exact confirmation flow but was closed unmerged, and the latest release tag visible in this checkout still has the same direct handler.

## Best Possible Solution

Implement a focused Control UI confirmation or equivalent maintainer-approved mitigation before `onNewSession` dispatches `/new`. The likely code path is `ui/src/ui/chat/run-controls.ts` plus parent wiring as needed, with regression coverage in `ui/src/ui/chat/run-controls.test.ts`; broader `/new` and `/reset` confirmation policy can remain tracked separately by the wider reset-safety issues.
