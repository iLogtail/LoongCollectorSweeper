## Summary

Keep open. Current main still renders the Control UI New session button as a direct click handler into `onNewSession`, and the parent still wires that to `handleSendChat("/new", { restoreDraft: true })`. I found no confirmation dialog, relocation, danger styling, or undo/safety step for this toolbar action. Existing tests assert the direct callback behavior, and the closed related PR #27110 was not merged according to the provided GitHub context.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI safety step before the toolbar New session action dispatches `/new`. The likely change belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for confirm and cancel behavior. Broader `/new` and `/reset` policy can remain tracked separately, but this toolbar misclick issue remains real on current main.
