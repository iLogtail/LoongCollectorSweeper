## Summary

Keep open. Current main still omits `--exclude-labels` from the Gmail watcher args, and the gateway/CLI serve paths use that arg builder directly. The PR is a plausible focused fix for #56635 rather than something already implemented or obsolete.

## Best Possible Solution

Keep the PR open or consolidate it with the canonical Gmail regression work in #56635/#56720/#56736. The best code path is to add the exclusion to `buildGogWatchServeArgs()` and add regression coverage; maintainers can decide whether to hardcode `SPAM,TRASH,DRAFT,SENT` for compatibility or expose `hooks.gmail.excludeLabels` through schema/types/docs.
