## Summary

Keep open. Current main handles ordinary Telegram `video` messages in some paths, but it still does not fully solve this report: the linked animation/GIF-video fix PR #56336 was closed unmerged, the current inbound media file-ref and placeholder code still omit `msg.animation`, and the reporter's 60MB example also remains vulnerable to the current `getFile` too-large path returning no saved attachment.

## Best Possible Solution

Keep this open until the Telegram plugin either fixes the missing `animation` inbound-media pipeline with regression coverage or maintainers explicitly split that into a separate canonical issue. Also clarify/user-surface the difference between OpenClaw's `mediaMaxMb` cap and Telegram Bot API `getFile` download limits for files such as the reported 60MB video. ([core.telegram.org](https://core.telegram.org/bots/api%20))
