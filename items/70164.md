## Summary

Keep #70164 open. Current main has the normal chat streaming path for assistant events, but the reporter’s newer OpenClaw gateway log shows a tool result followed by lifecycle end and chat final with no assistant event at all. The code still emits `chat` text only from buffered assistant events, so this is not safely implemented or obsolete on main.

## Best Possible Solution

Keep the issue open and investigate the MiniMax tool-turn path through `subscribeEmbeddedPiSession` and gateway chat finalization. A good fix should prove whether the provider/embedded runner loses the final assistant `message_end` after tool use or whether `server-chat.ts` needs a documented fallback from final assistant message state, then add a regression test for tool result -> visible assistant text -> chat delta/final before lifecycle completion.
