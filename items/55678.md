## Summary

Current main does not implement the requested DashScope/Qwen usage-window support. Qwen is a bundled provider and has streaming token-usage compatibility, but provider usage/quota reporting is still gated by a hard-coded UsageProviderId list that excludes qwen/modelstudio/dashscope, and the qwen plugin does not currently provide usage auth or snapshot hooks.

## Best Possible Solution

Keep this open and implement it in the bundled qwen provider/provider-usage surface: add qwen/modelstudio/dashscope alias coverage, implement resolveUsageAuth and fetchUsageSnapshot against the appropriate DashScope/Qwen usage or billing endpoint, update tests for resolveUsageProviderId, loadProviderUsageSummary, and models.authStatus, and update the Qwen plus API usage/cost docs. If DashScope only exposes quota/billing snapshots, label that separately from per-message token/cost estimates; if reliable per-model pricing is added, replace the current zero Qwen cost metadata with source-backed values.
