## Summary

Keep #71350 open. Current main already covers part of the request through `agents.list[].thinkingDefault` and per-agent `agents.list[].params` overrides, including `temperature` and `maxTokens`, but the broader feature is not fully implemented: there is no `agents.list[].models` per-agent per-model override map and no `thinkingStrategy` rules/escalation/AGENTS.md-driven policy. The related context also points to this issue as the canonical remaining adaptive-thinking tracker, with PR #44126 still open as implementation discussion rather than shipped behavior.

## Best Possible Solution

Keep #71350 open as the canonical product/API discussion. Preserve the shipped `agents.list[].thinkingDefault` and `agents.list[].params` behavior, then decide whether OpenClaw should add a true per-agent per-model override map, broaden provider param pass-through, and implement a core `thinkingStrategy` policy for rule matching and failure-based escalation, likely coordinated with PR #44126 and the open #32496 parameter work.
