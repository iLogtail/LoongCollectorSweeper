## Summary

Keep open. Current main already implements the issue's per-agent `thinkingDefault` and flat per-agent model params via `agents.list[].params`, but it does not implement the requested nested `agents.list[].models` per-agent per-model layer or any rules/escalation/AGENTS.md-driven `thinkingStrategy` behavior. Related #64792 was closed into this issue, while PR #44126 remains only a related open implementation discussion, not a shipped replacement for the broader request.

## Best Possible Solution

Keep this issue open as the canonical product/design tracker. A maintainer-facing resolution should acknowledge that `agents.list[].thinkingDefault` and flat `agents.list[].params` already ship, decide whether OpenClaw wants a new `agents.list[].models` merge layer or clearer docs around `agents.list[].params`, and design any adaptive behavior deliberately as a core agent-loop strategy or approved tool/API surface with schema, docs, generated config help, and tests for rule matching, provider-supported thinking levels, and failure escalation.
