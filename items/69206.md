## Summary

Keep #69206 open. Current main still generates and documents `/dock_discord`, and Telegram native commands are routed into the shared command pipeline, but there is no dock command handler that performs Discord handoff. With no matching handler, the command pipeline continues to normal agent/model reply, matching the reported fall-through behavior. Related #45596 addressed Telegram native skill/custom command dispatch generally, but it does not implement dock-channel handoff.

## Best Possible Solution

Implement a first-class dock command handler in the shared command pipeline, backed by tests. The handler should resolve `dock:<channel>` commands from the generated registry, validate the target channel/account, use `session.identityLinks` or another explicit route mapping to find the corresponding conversation, update the target delivery/session metadata deterministically, and return a clear error when no linked target exists. Add focused coverage for Telegram native `/dock_discord` so it proves native dispatch does not fall through to the model.
