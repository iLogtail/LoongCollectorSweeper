## Summary

Keep #70541 open. Current main still contains the reported `process.cwd()` call in the auto-compaction completion path, even though the queued followup run already carries `workspaceDir` and an adjacent preflight compaction path uses it correctly. Related items in the issue body are adjacent post-compaction feature/fallback work, not duplicates of this remaining call-site bug.

## Best Possible Solution

Patch `src/auto-reply/reply/agent-runner.ts:1555` to pass `followupRun.run.workspaceDir` to `readPostCompactionContext` (optionally with a defensive fallback), and add a regression test for the `autoCompactionCount > 0` branch so the post-compaction system event reads the configured agent workspace rather than the runner process cwd.
