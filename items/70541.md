## Summary

Keep #70541 open. Current main still contains the reported live post-run auto-compaction path that reads post-compaction AGENTS.md context from `process.cwd()` instead of the queued run workspace, while adjacent preflight and execution paths already carry `followupRun.run.workspaceDir`. The issue is recent, has no protected/maintainer labels, and is a concrete fixable source bug rather than stale or already implemented.

## Best Possible Solution

Keep the issue open and fix `src/auto-reply/reply/agent-runner.ts` to call `readPostCompactionContext(followupRun.run.workspaceDir, { cfg, agentId: ... })` in the post-run auto-compaction branch. Add a regression test around `runReplyAgent` or the nearest injected seam that makes `process.cwd()` differ from `followupRun.run.workspaceDir` and proves the refresh is read from the queued workspace AGENTS.md.
