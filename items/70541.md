## Summary

Keep #70541 open. Current main still has the reported post-compaction follow-up hook reading AGENTS.md from process.cwd(), while the queued run already carries a required workspaceDir and adjacent compaction code uses that workspaceDir correctly. The issue is recent, source-backed, and actionable in OpenClaw core.

## Best Possible Solution

Patch `src/auto-reply/reply/agent-runner.ts` so the auto-compaction completion hook passes `followupRun.run.workspaceDir` to `readPostCompactionContext` while preserving the current cfg and resolved agentId. Add a focused regression test where process.cwd() differs from `followupRun.run.workspaceDir`, the workspace contains AGENTS.md, and an auto-compaction metadata result enqueues or appends the post-compaction refresh from the workspace file. Keep the existing `agent-runner-memory.ts` behavior as the reference path.
