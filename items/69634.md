## Summary

Keep #69634 open. Current main still has the exact UI contract gap the PR addresses: the `.chat-bubble.has-copy` padding rule exists, and the floating message action bar still renders, but `renderGroupedMessage` does not add `has-copy` when `canCopyMarkdown || canExpand` is true. The linked bug #69605 remains open in the provided context, and the related duplicate PRs are closed/unmerged rather than shipped replacements.

## Best Possible Solution

Keep the PR or replace it with an equivalent maintainer patch that adds `has-copy` to assistant bubbles whenever `canCopyMarkdown || canExpand` is true, preserves current tool-shell behavior, and adds a regression test proving action-bearing assistant bubbles reserve space while user/no-action bubbles do not.
