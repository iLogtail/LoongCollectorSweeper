## Summary

Keep PR #69634 open. Current `main` still renders assistant bubble action controls without applying the existing `.chat-bubble.has-copy` padding class, so the overlap/mis-click bug described by #69605 is not implemented on main or in the latest release. The related duplicate PRs in the provided context are closed and unmerged, while the canonical bug #69605 remains open.

## Best Possible Solution

Keep the PR or canonical bug #69605 active, rebase the one-line layout fix onto current `ui/src/ui/chat/grouped-render.ts`, and add a focused `grouped-render.test.ts` assertion for the `has-copy` class when assistant copy/expand actions are rendered. The existing CSS contract should be used rather than inventing a new layout mechanism.
