## Summary

Keep PR #69634 open. The reported Control UI overlap is not implemented on current main: the existing `.chat-bubble.has-copy` padding rule still exists, but current `renderGroupedMessage` renders message-level actions without adding `has-copy` to the bubble. The PRâ€™s fix remains directionally useful, though it should be rebased and ideally paired with the regression test described in related PR #69689.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main that adds `has-copy` under the same `hasActions` condition while preserving the current `chat-bubble--tool-shell` class. Add or reuse the focused grouped-render regression test described in #69689: assistant bubbles with markdown actions should include `has-copy`, while bubbles without those actions should not.
