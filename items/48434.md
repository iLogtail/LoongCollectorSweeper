## Summary

Keep open. Current `main` at `c65aa1d2a6e573d49619c4dd0698a76010476e7d` still lacks a broadcast `runInboundClaim(...)` call in the inbound dispatch path, so globally registered `inbound_claim` handlers are not invoked unless the conversation is already plugin-bound. The issue discussion also points to an open fix PR (#71644), which matches the code on `main` rather than an already-shipped resolution.

## Evidence

There is no generic `runInboundClaim(...)` call after the plugin-bound block.
