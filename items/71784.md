## Summary

Keep #71784 open. Current main still does not add bounded retry for the live builtin memory-search query embedding path, and the related open PR #44167 is batch/reindex-focused rather than a complete fix for per-query `openclaw memory search` failures.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded retry/backoff around the live query embedding path, preferably shared with the single remote embedding HTTP helper or `embedQueryWithTimeout()`, while keeping auth/config errors fail-fast. Regression coverage should include `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, and TLS socket disconnect cases for live memory search, plus a check that `memory status --deep` readiness does not imply live query retry coverage.
