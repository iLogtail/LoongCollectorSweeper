## Summary

Keep open. Current main still runs the pre-compaction memory flush as an embedded agent prompt on the active session, with `silentExpected` suppressing assistant delivery but no evidence that the memory-flush user prompt is kept out of the persisted transcript or compaction input. The issue is recent, not protected/maintainer-authored, and related open items address adjacent reliability/blocking problems rather than fully superseding this persisted-prompt contamination report.

## Best Possible Solution

Keep this open until memory flush is made truly internal: either run it on a non-user-visible/non-transcript lane or pass a safe `transcriptPrompt`/custom hidden runtime context so the flush directive is not stored as a normal user message. Add regression coverage that `chat.history` and session JSONL do not expose the pre-compaction prompt, and that compaction ignores any historical leaked flush prompts to break poisoned-session loops.
