## Summary

Keep open. Current `main` still rejects `gateway.bind="lan"` when `gateway.tailscale.mode` is `serve`/`funnel`, and the runtime/doctor legacy migration set does not contain a targeted repair for that legacy combination. There is a generic last-known-good startup recovery path now, and a comment points to ClawAid as a workaround, but that is not the in-core migration requested here and does not prove the issue is implemented on `main`.
