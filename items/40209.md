## Summary

Current `main` still has the case-sensitivity mismatch described in the PR: `src/infra/system-events.ts` stores and looks up queues with the raw trimmed `sessionKey`, while heartbeat routing canonicalizes agent session keys to lowercase before peeking the queue. That makes the reported mixed-case channel-session failure still plausible on `main`, so this PR should stay open for maintainer review rather than be auto-closed.

## Evidence

- **System-event queue is still keyed by raw sessionKey:** `requireSessionKey()` only trims and returns the original string, and every queue access path (`getSessionQueue`, `getOrCreateSessionQueue`, `enqueueSystemEvent`, `drainSystemEventEntries`) goes through it. There is no lowercase normalization in current `main`, so queue keys remain case-sensitive.
  - file: [src/infra/system-events.ts:43](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/infra/system-events.ts#L43)

If `session.sessionKey` is canonicalized differently from the enqueue key, the queue lookup will miss.
