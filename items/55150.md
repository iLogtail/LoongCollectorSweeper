## Summary

Keep open. Current `main` has related deleted-agent cleanup and guards, but I did not find strong evidence that the specific bug in #55150 is fixed. The code still intentionally supports/discovers orphaned on-disk agent state, and the repo still documents manual stale-dir cleanup rather than a startup prune path.
