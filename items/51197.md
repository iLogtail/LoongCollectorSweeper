## Summary

Keep #51197 open. Current main still has the reported post-open write failure leak in both TypeScript lock helpers, and the PR addresses a plausible live resource-cleanup bug. The PR should not merge as-is without addressing the existing review comment on the Go pipe cleanup and adding focused regression coverage, but it is not obsolete or outside the repo.

## Best Possible Solution

Keep the PR open, or replace it with an equivalent maintainer patch. The best implementation is to wrap the post-open write sections in src/infra/gateway-lock.ts and src/plugin-sdk/file-lock.ts so every non-success path closes the FileHandle and removes the partial lock file, add focused regression tests that simulate writeFile failure after open succeeds, and either fix or drop the scripts/docs-i18n/pi_rpc_client.go pipe-cleanup part by closing stdout on the StderrPipe failure path.
