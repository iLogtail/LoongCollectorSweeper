## Summary

Keep #65564 open. Current main fixes the original stale `sessionFile` carryover for fresh isolated cron/heartbeat sessions, but the active follow-up in the issue is still valid: the normal `heartbeat.isolatedSession` rotation path overwrites the current isolated session entry without archiving the previous transcript at that same key. The closed, unmerged PR #64808 describes the missing reset-archive behavior, and no open canonical item fully supersedes this issue. #70680 only covers trajectory false positives in doctor counts, not the primary heartbeat transcript leak.

## Best Possible Solution

Keep #65564 open and fix the heartbeat-owned rotation lifecycle: before overwriting `cronSession.store[isolatedSessionKey]` for `heartbeat.isolatedSession === true`, capture the prior entry at that same key and archive its transcript with `reason: "reset"` when the new forced session rotates to a different `sessionId`. Add a heartbeat regression test equivalent to the closed #64808 proposal. Track #70680 separately for the trajectory-file doctor false positive, because it is related noise but not the primary heartbeat transcript leak.
