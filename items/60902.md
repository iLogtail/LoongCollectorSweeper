## Summary

Keep open. Current main still exposes only `/vc join`, `/vc leave`, and `/vc status`; the requested `/vc switch <agent>` handoff is absent from the Discord voice command, manager, tests, and docs. The PR is a focused change to the existing bundled Discord plugin, and the provided discussion says the author addressed the bot-reviewed route consistency and queued-audio race concerns in head commit `601e12b36619094dd6887e17cc241244ddf850d4`. No canonical duplicate or superseding item in the provided related context tracks this exact agent handoff feature.

## Best Possible Solution

Keep #60902 open for maintainer review. The best path is to land or revise a focused Discord plugin change that adds `/vc switch <agent>`, validates configured agents, rebuilds the full route atomically, snapshots per-segment routes for queued audio, and keeps docs/tests/changelog aligned.
