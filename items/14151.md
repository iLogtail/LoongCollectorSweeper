## Summary

Keep #14151 open. Current main still strips/parses existing thinking tags and supports native reasoning events, but it does not provide a configurable global or per-model option to prepend a missing opening `<think>` tag before streaming/final parsing. Related PR #14323 attempted this feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, validate and document it, resolve it through agent/model config, thread it through `RunEmbeddedPiAgentParams` and `SubscribeEmbeddedPiSessionParams`, inject exactly once before streaming and final-message reasoning parsing, guard existing tags/native thinking events, and add focused tests for streaming, non-streaming, existing `<think>` tags, native thinking, and `enforceFinalTag` interactions.
