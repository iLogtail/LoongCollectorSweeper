## Summary

Keep open. Current main does not implement the requested suppression: `gateway` calls are still considered mutating unless their full `action` string is in `READ_ONLY_ACTIONS`, and `config.schema.lookup` is not in that set. The gateway correctly returns `config schema path not found`, but the agent warning path still treats a failed `gateway` lookup as a mutating tool error, which matches the user-facing notification reported in #42856. The linked PRs #42898 and #42962 are relevant fixes, but they are open and not present on current main.

## Best Possible Solution

Keep #42856 open and review or consolidate the linked fixes from #42898 and #42962. The most direct product fix is to make the agent mutation classifier treat read-only compound gateway actions such as `config.schema.lookup` as non-mutating, ideally aligned with gateway method scopes or with leaf-verb handling, then add regression coverage for `isMutatingToolCall("gateway", { action: "config.schema.lookup" })` and the `config schema path not found` warning-payload path.
