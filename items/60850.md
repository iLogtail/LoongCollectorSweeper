## Summary

Current main still has the reported low-contrast Webchat/Control UI selection styling. The global `::selection` rule uses the low-opacity `--accent-subtle` token, and the linked fix PR #60854 is still only a PR, not behavior present in the checked-out `main`.

## Best Possible Solution

Keep #60850 open until the Webchat/Control UI selection styling is changed on main. The likely fix belongs in `ui/src/styles/base.css`, using a higher-contrast selection background such as `--accent-muted` with `--accent-foreground`, either by landing #60854 after review or applying an equivalent maintainer-approved patch.
