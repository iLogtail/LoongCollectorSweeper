## Summary

Current main partially supports Bedrock-discovered vision metadata, but the reported stale/explicit catalog failure mode is still present. Bedrock discovery maps AWS IMAGE modalities to `input: ["text", "image"]`, yet the merge path for an existing non-empty `models.providers.amazon-bedrock.models` list still preserves the stale explicit list instead of refreshing it from discovery, and missing/unknown input still defaults to text-only. This should stay open for a focused fix and regression coverage.

## Best Possible Solution

Keep the issue open. The best fix is provider-owned: make Bedrock discovery metadata refresh stale explicit/static `amazon-bedrock` model rows instead of discarding the discovered catalog when explicit models exist. Add regression coverage for an existing `models.providers.amazon-bedrock.models` row missing `input` or stuck at `["text"]` while discovery returns the same Claude foundation/profile model with image support; include ID-normalization coverage for regional/global profile suffix variants. A secondary hardening path is to repair known Bedrock Claude 3.5+/4.x text-only or missing-input rows during Bedrock provider normalization, but that should not replace the merge/discovery fix.
