## Summary

Current main still leaves the reported stale Bedrock catalog failure mode open. Bedrock discovery can populate `input: ["text", "image"]` when AWS discovery returns IMAGE, and tests cover inference-profile inheritance, but existing non-empty Bedrock model lists are still preserved and unmatched explicit model rows are returned unchanged. The image path still treats missing `input` as text-only. The linked #72404 PR is open and unmerged, so it is useful follow-up work rather than a shipped or superseding fix.

## Best Possible Solution

Keep #71921 open until #72404 or an equivalent maintainer-approved fix lands. The best fix should refresh or repair stale explicit/static `amazon-bedrock` model rows when discovery has fresher capability metadata, preserve intentional explicit opt-outs, and add regression coverage for missing or text-only `input` on Bedrock Claude Opus/Sonnet 4.x regional/global IDs.
