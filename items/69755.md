## Summary

Keep open. Current main still emits the same generic post-restart `Run: openclaw doctor --non-interactive` hint for config patch/apply, gateway restarts, and updates, then delivers that message through the restart-sentinel wake/notice path without knowing whether the receiving surface can complete approvals. The issue is recent, concrete, unprotected by maintainer labels, and not resolved by the related ClawSweeper matches provided.

## Best Possible Solution

Keep this issue open and implement context-aware restart follow-up guidance. A conservative fix would add a structured restart-sentinel follow-up/hint helper that can say `Recommended follow-up: run openclaw doctor --non-interactive in a terminal or approvals-capable UI` for chat/heartbeat/unknown delivery contexts, while preserving the direct `Run:` command in terminal/control-plane contexts known to be actionable. Add focused tests around `formatRestartSentinelMessage`, config patch/apply payloads, gateway restarts, and routed versus session-only restart wake delivery.
