## Summary

Current main still emits a generic `Run: openclaw doctor --non-interactive` post-restart hint without distinguishing whether the receiving heartbeat/chat/session surface can actually complete approval-gated execution. The issue is recent, non-protected, and describes a real core UX gap that is not covered by the unrelated related-item matches.

## Best Possible Solution

Keep the issue open. The best fix is a small core UX change around restart sentinel doctor hints: make `doctorHint` or its formatter aware of the restart/request delivery surface and emit explicit fallback text when approvals cannot be completed there, then add focused tests for config patch/apply, update/restart, and restart-sentinel formatting/delivery.
