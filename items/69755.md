## Summary

Current main still presents the post-restart doctor hint unconditionally. The restart/config-patch/update sentinel paths continue to store `Run: openclaw doctor --non-interactive`, and tests assert that exact wording, so the issue's requested context-aware/actionable-here distinction is not implemented.

## Best Possible Solution

Keep this issue open. The best fix is to make restart sentinel doctor hints context-aware: pass enough surface/route/approval capability information from config patch, update, `/restart`, and gateway restart call sites, then render either an actionable local command or explicit fallback guidance such as running doctor in a terminal or approvals-capable UI. Add focused tests for routed chat/heartbeat/background contexts so the generic `Run: openclaw doctor --non-interactive` hint is only used where it is actually actionable.
