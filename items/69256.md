## Summary

Keep open. Current main still deletes isolated cron direct-delivery sessions from `cleanupDirectCronSessionIfNeeded` without checking active descendants, and structured/threaded direct delivery reaches that cleanup path outside the text-delivery subagent wait/guard. The PR is addressing a real gap, but the current patch should be revised because review comments correctly flag that a simple early return can skip later `deleteAfterRun` cleanup permanently.

## Best Possible Solution

Keep #69256 open and ask for an updated patch against current main that makes direct-delivery cleanup subagent-aware without leaking one-shot isolated sessions. The best fix should either defer/retry cleanup after descendants finish or route direct delivery through a shared subagent-finalization/cleanup state machine, and should add focused tests for active descendants, later cleanup after drain, and the existing zero-descendant cleanup cases.

- A regression test is still needed for `deleteAfterRun: true` plus structured or threaded direct delivery with active descendants, including proof that cleanup runs later after descendants drain.
