## Summary

Keep #69256 open. Current main still has the reported direct-delivery cleanup gap: structured payloads and threaded direct deliveries call the direct cleanup wrapper before the text-only subagent follow-up guard, and that cleanup deletes the cron session without checking active descendants. The PR is relevant, but its current patch needs maintainer/author follow-up because review comments correctly flag missing regression coverage and a likely need for second-stage cleanup after descendants drain.

## Best Possible Solution

Keep the PR open for a revised fix: preserve the parent session while `countActiveDescendantRuns(params.agentSessionKey) > 0`, add a second-stage cleanup path after descendants settle so `deleteAfterRun` remains honored, and add focused tests for structured/threaded direct delivery with active descendants plus the eventual zero-descendant cleanup case.
