## Summary

Keep open. Current `main` still does not thread `timeoutMs` into the remote batch HTTP helpers, so the PR's stated fix is not implemented on `main`, and the docs still promise a full-operation batch timeout. The PR discussion also surfaced a remaining behavioral risk around retrying the entire batch pipeline after an HTTP timeout, which argues for maintainer review rather than cleanup closure.

## Risks / Open Questions
