## Summary

Keep #55788 open. Current main still does not implement openai-codex audio media understanding: the runtime provider and manifest metadata both advertise openai-codex as image-only, and the audio runner skips providers without transcribeAudio. The PR discussion and later commits are useful, but the patch needs rework for the current manifest-first media metadata path and the unresolved Codex audio base-URL/auth behavior.

## Best Possible Solution

Keep the PR open, but require a maintainer-quality update before merge: add openai-codex audio support in the bundled OpenAI plugin provider and manifest metadata, ensure auto audio selection/default model resolution works through the current manifest-first path, add tests for capability plus handler wiring, and verify the Codex OAuth transcription request uses the intended audio endpoint without discarding legitimate custom base URL overrides. If maintainers prefer the newer #68539 product framing, link this PR there rather than auto-closing it until a replacement implementation exists.
