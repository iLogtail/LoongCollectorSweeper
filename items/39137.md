## Summary

Keep this PR open. Current main still has the split YAML inline sequence for the `skills-python-tests` hook, which parses into two dependency entries and leaves pip/pre-commit with the invalid standalone `<9` requirement. The PR remains a narrow, valid fix and is not superseded by the related closed voice-call PRs.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or apply the same one-line `.pre-commit-config.yaml` change on main so `skills-python-tests` passes one valid `pytest>=8,<9` requirement string to pip. Do not close it before the fix merges.
