## Summary

Keep #70238 open. Current main still has an OpenClaw-owned TUI relaunch spawn without `windowsHide`, the bundled Bonjour discovery path remains enabled by default with `@homebridge/ciao@1.3.6`, and the active fix PR #72332 is open but not merged. This is a recent regression with multiple confirmations, so it should not be cleanup-closed until the fix lands or maintainers intentionally consolidate it under a canonical tracker such as #70197.

## Best Possible Solution

Keep the issue open while PR #72332 is reviewed. The likely product fix is to add `windowsHide: true` to OpenClaw-owned Windows background spawn surfaces such as `src/tui/tui-launch.ts`, patch or upstream-fix `@homebridge/ciao` ARP/network-interface exec calls used by the bundled Bonjour plugin, and add focused regression coverage for the spawn options where feasible. Once that lands, close #70238 as fixed; if maintainers prefer one canonical issue, explicitly duplicate it to the open root-cause tracker #70197 only after confirming the TUI startup surface is also tracked.
