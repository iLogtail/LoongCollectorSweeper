## Summary

Current `main` still relays failed or undelivered `/hooks/agent` isolated-run results into the main session as untrusted system events, and the docs/config surface does not show a suppression knob for that relay. This matches the reported behavior closely enough that the issue should stay open.

## Evidence

- **Hook agent fallback still enqueues relay events:** `dispatchAgentHook` enqueues `Hook <name>: <summary>` when `runCronIsolatedAgentTurn()` returns `delivered=false`, and enqueues `Hook <name> (error): ...` on exceptions. Both go to `mainSessionKey` as untrusted system events.
  - file: [src/gateway/server/hooks.ts:43](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/server/hooks.ts#L43)
