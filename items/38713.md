## Summary

Keep PR #38713 open. Current main still resolves `openclaw skills info <name>` with exact-only `name`/`skillKey` matching, while the PR targets a real CLI mismatch class where visible skill names/keys can differ by case or separators. The predecessor PR #38701 was closed unmerged, and the linked issue #38546 is closed but does not supersede this focused fix.

## Best Possible Solution

Keep PR #38713 open. Refresh or land the focused `skills info` resolver in `src/cli/skills-cli.format.ts`: trim the requested name, preserve exact matches first, allow only unique case-insensitive and separator-normalized matches, sanitize not-found output, and keep regression tests in `src/cli/skills-cli.test.ts`. Track any separate agent-facing skill activation issue independently rather than closing this PR as obsolete.
