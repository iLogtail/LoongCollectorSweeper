## Summary

Keep PR #38713 open. Current main still resolves `openclaw skills info <name>` with exact `name`/`skillKey` equality only, the CLI passes the raw argument into that formatter, and the current tests do not include the PR's case-insensitive or separator-normalized regression coverage. The predecessor PR #38701 was closed unmerged, and the linked bug #38546 was closed without this code landing.

## Best Possible Solution

Keep this PR open. The best path is to refresh it against current main, land a focused `skills info` resolver in `src/cli/skills-cli.format.ts` that trims the requested name, preserves exact matches first, allows only unique case-insensitive and separator-normalized matches, sanitizes not-found output, and adds regression tests in `src/cli/skills-cli.test.ts`; split any broader agent skill activation problem into a separate tracked issue if it is not covered by the CLI lookup fix.
