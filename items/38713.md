## Summary

Keep PR #38713 open. Current main still uses exact-only `skills info` lookup, lacks the PR's case-insensitive/separator-normalized resolver and regression tests, and the related predecessor PR #38701 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep this PR open. The best path is to refresh it against current main, land a focused resolver in `src/cli/skills-cli.format.ts` that trims the requested name, preserves exact matches first, permits only unique case-insensitive and separator-normalized matches, sanitizes not-found output, and adds regression tests in `src/cli/skills-cli.test.ts`; split any remaining agent-facing skill activation problem into a separate issue if it is not covered by the CLI lookup fix.
