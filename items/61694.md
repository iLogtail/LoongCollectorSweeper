## Summary

Current main still lacks the requested global opt-out. External CLI auth overlays are resolved unconditionally from the auth-profile runtime path, the synced providers include Codex CLI, Claude CLI, and MiniMax CLI, and neither config types/docs nor environment handling expose `auth.externalCliSync.enabled` or `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC`.

## Best Possible Solution

Keep the issue open. The likely fix belongs in core auth-profile loading: add a supported config/env gate for external CLI sync, thread it through `overlayExternalAuthProfiles` and `readManagedExternalCliCredential`/runtime auth resolution, update config schema/help/docs, and add tests covering env precedence, config disablement, default-enabled behavior, and the existing readonly distinction.
