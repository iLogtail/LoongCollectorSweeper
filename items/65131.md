## Summary

Keep open. Current main has partial cleanup: the newer session-history snapshot path hides heartbeat prompt/ack rows, and queued System: events are kept out of the transcript prompt. But the Control UI chat tab still loads `chat.history`, whose server/client filters do not hide user-role system prompts broadly, and the pre-compaction memory-flush path still submits its prompt as the visible run prompt without a separate transcript prompt. The issue's broader request is not fully implemented.

## Best Possible Solution

Keep the issue open and implement a unified visible-chat filter for system/internal turns on the `chat.history` path used by Control UI/WebChat. Reuse the existing heartbeat classifier and runtime-context stripping, add explicit handling for memory-flush/system housekeeping prompts, and keep raw transcripts intact for audit/replay. Add focused tests in `src/gateway/server-methods/chat*.test.ts` and `ui/src/ui/controllers/chat.test.ts` that cover heartbeat prompts, `HEARTBEAT_OK`, pre-compaction memory flush prompts, and restart/system-event notices.
