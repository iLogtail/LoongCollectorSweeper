## Summary

Keep #55231 open. Current main still replays all pending outbound delivery queue entries on gateway startup when they are first-replay crash leftovers, without a stale-age cutoff. The transcript echo path also still sends through the normal outbound delivery wrapper without `skipQueue`, so those ephemeral echo messages can be persisted in the recovery queue.

## Best Possible Solution

Keep the issue open and fix it in core outbound delivery. A conservative path is to add an age/TTL policy for startup delivery recovery, with tests proving stale zero-retry entries are moved aside or logged without delivery, and to route ephemeral transcript echo/confirmation sends with `skipQueue: true` or an explicit non-recoverable delivery class. Update config/docs only if the stale threshold is user-configurable.
