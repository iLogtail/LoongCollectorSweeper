## Summary

Keep #39142 open. Current main still does not implement the requested configurable user-facing exhaustion message. It has static friendlier handling for some pure billing/rate-limit fallback summaries, but `messages.modelExhausted` or equivalent config is absent, and mixed billing/rate-limit exhaustion is still covered by a regression test that expects the raw `Agent failed before reply` / `All models failed` text.

## Best Possible Solution

Keep the issue open until OpenClaw either implements an explicit core config surface for model-exhaustion copy or a maintainer rejects that product direction. A likely implementation would add a typed and schema-backed `messages.modelExhausted` or more precise exhaustion-message config, wire it into `runAgentTurnWithFallback` for billing/rate-limit fallback summaries, document it in the messages/model-failover docs, and add regression coverage for pure billing, pure rate-limit, and mixed billing/rate-limit exhaustion.
