## Summary

Keep PR #53171 open for maintainer judgment rather than auto-closing. Current main has stronger Control UI packaging guardrails, but the exact `pnpm ui:build`-from-published-package request is not implemented, and the proposed `scripts/`-only fix is not sufficient because the UI runner also requires the `ui/` source tree.

## Best Possible Solution

Do not merge this PR as-is. Maintainers should decide the package boundary explicitly: either keep published installs read-only and remove or guard source-only `ui:*` scripts from the published package metadata, or intentionally support post-install UI rebuilds by shipping the complete `scripts/ui.js` plus `ui/` source/dependency surface with release checks. The existing `dist/control-ui` packaging guardrails should remain.
