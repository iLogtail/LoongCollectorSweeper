## Summary

Keep #69298 open. Current main improved one adjacent Codex accounting path by using `tokenUsage.last` instead of cumulative `tokenUsage.total`, and current main also has the separate checkpoint-snapshot usage fix related to #70749. But the core bug here remains: Codex app-server `inputTokens`/`cachedInputTokens` are still projected as additive OpenClaw `input`/`cacheRead`, while generic session/context accounting still derives prompt usage as `input + cacheRead + cacheWrite`.

## Best Possible Solution

Keep the issue open and fix the Codex harness adapter at the source: normalize Codex app-server `inputTokens` to uncached input before storing OpenClaw usage, or teach shared prompt/session accounting to prefer provider `totalTokens - output` when the provider reports prompt totals that already include cached input. Add a focused Codex event-projector regression test using the issue's invariant where `totalTokens - outputTokens == inputTokens` and `cachedInputTokens` is a subset of input.
