## Summary

Keep open. Current main contains the earlier Moonshot tool-call ID fix, but I did not find an OpenClaw-side fix for this follow-up: Moonshot/Kimi replay still lacks a provider-specific guard that restores or synthesizes `reasoning_content` for assistant tool-call messages after compaction or cross-model fallback when the original thinking block is gone. The issue discussion also links the LCM-side bug, but that does not supersede the OpenClaw-side compatibility gap described here.

## Best Possible Solution

Keep this issue open and implement a focused Moonshot/Kimi replay guard: preserve `reasoning_content` when the thinking block survives, and when Kimi thinking is enabled synthesize an acceptable `reasoning_content` value for replayed assistant `tool_calls` that arrive from compacted or cross-model history without one. Add regression coverage for a compacted/cross-model transcript and coordinate with Martian-Engineering/lossless-claw#488 so LCM preserves OpenAI-style top-level reasoning fields during compaction.
