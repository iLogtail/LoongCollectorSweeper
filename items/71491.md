## Summary

Keep open. The report is recent, concrete, and not covered by the current main implementation. Main has the earlier Moonshot tool-call ID fix and an optional `thinking.keep = "all"` pass-through, but I did not find a Moonshot replay sanitizer or payload patch that restores missing top-level `reasoning_content` for assistant tool-call turns after LCM compaction or cross-model fallback. The linked LCM discussion also points to an external compaction-side strip, so this still needs coordinated handling rather than cleanup closure.

## Best Possible Solution

Keep the issue open and implement a focused Moonshot replay fix in the bundled provider/plugin path. The maintainer path should add provider-owned replay sanitization or a payload patch that preserves top-level OpenAI-style `reasoning_content` when present and safely handles missing reasoning on replayed assistant tool-call turns during Kimi thinking sessions, then cover same-model long replay, LCM/context-engine compaction, and cross-model fallback cases. Coordinate with Martian-Engineering/lossless-claw#488 so LCM preserves top-level `reasoning_content` rather than forcing OpenClaw to recover data after it is stripped.
