## Summary

Keep open. Current main still lacks Feishu reply-dispatcher wiring for media local roots: the dispatcher accepts no `mediaLocalRoots` parameter, its `sendMediaReplies` call to `sendMediaFeishu` omits roots, and both Feishu bot call sites create that dispatcher without any media-root context. The related ClawSweeper reports for PRs #49082 and #51375 also identify the same remaining gap on current main.

## Best Possible Solution

Keep the issue open and fix the Feishu reply path directly: add a typed media-root/media-access field to `createFeishuReplyDispatcher`, derive the appropriate agent-scoped roots at the Feishu bot call sites, pass them through to `sendMediaFeishu`, and add a reply-dispatcher regression test that asserts local media replies include `mediaLocalRoots`.
