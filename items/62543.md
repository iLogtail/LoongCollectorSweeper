## Summary

Keep open. The PR identifies a real stale-docs problem: the current blanket OpenClaw note still says Bedrock requires model access enabled in each AWS account/region, while AWS now says commercial-region access is enabled by default with the right Marketplace prerequisites. But the PR’s one-line deletion is not safe because AWS still documents third-party subscription prerequisites, Anthropic first-time-use details, and GovCloud manual model-access steps. citeturn1view0turn1view2

## Best Possible Solution

Keep the item open for a maintainer/docs revision: replace the blanket line in `docs/providers/bedrock.md` with a scoped caveat explaining that commercial-region model access is generally automatic with the required AWS Marketplace permissions, while third-party models can require first-use subscription prerequisites, Anthropic requires first-time use-case submission, and GovCloud still requires manual Model access enablement. The current PR should not be merged as a pure deletion unless it is updated to preserve those caveats. citeturn1view0turn1view2

## Evidence

- **Current OpenClaw docs still have the broad caveat:** `docs/providers/bedrock.md` currently says, under Notes and caveats, “Bedrock requires model access enabled in your AWS account/region.” That is the exact line the PR deletes, but no replacement caveat exists on current main.
  - file: [docs/providers/bedrock.md:329](https://github.com/openclaw/openclaw/blob/6cd047e7c27021d8955e09fa3598029eede5a1a7/docs/providers/bedrock.md#L329)

The same current AWS page also says first third-party-model invocation can fail with `AccessDeniedException` when prerequisites are missing. citeturn1view0turn1view2
