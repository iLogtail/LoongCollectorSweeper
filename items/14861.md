## Summary

Keep #14861 open. Current main still has only partial startup diagnostics: compact gateway ready logging, startup trace timings, plugin load failure summaries, and targeted Gmail watcher logs. It still does not provide the requested aggregate gateway boot summary, and the `hooks.enabled=true` plus missing `hooks.gmail.account` startup case remains silent.

## Best Possible Solution

Keep #14861 open. The best maintainer path is a structured gateway startup outcome surface in `startGatewaySidecars` or `startGatewayPostAttachRuntime` that records started/skipped/failed results for Gmail watcher, internal hooks, channels, plugin services, memory, cron, heartbeat, scheduled services, and related sidecars, then logs a concise post-sidecar boot summary. Add regression coverage for the aggregate summary and for `hooks.enabled=true` with missing `hooks.gmail.account`.
