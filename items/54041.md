## Summary

Keep open. Current main still only wires Anthropic request betas for `anthropicBeta` and `context1m`; the requested Anthropic context-management betas, `context_management` request body support, and `memory_20250818` tool path are not implemented or documented. The related production comment reinforces a real use case, and there is no canonical merged issue/PR in the provided context that supersedes it.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. The likely implementation belongs in the bundled Anthropic provider or a clearly documented provider plugin path: add provider-owned beta header handling, a safe Anthropic `context_management` params contract, tests for payload/header shaping, and decide whether Memory Tool support can be a plugin tool plus provider payload wrapper or needs a new provider-defined-tool SDK seam. Full native server-side compaction should also account for Anthropic compaction stop reasons and post-compaction reinjection before claiming Option A is done.
