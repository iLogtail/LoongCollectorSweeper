## Summary

Keep PR #65143 open. Current main still lacks `models.providers.*.timeoutSeconds` in the model provider type, strict parser, generated schema, labels/help metadata, and embedded runner provider path. The PR is not obsolete or implemented on main, but its current approach still needs maintainer-quality revision because the proposed timeout is applied through a process-global Undici dispatcher rather than a provider/request-scoped path.

## Best Possible Solution

Keep the PR open and request a scoped revision: add `models.providers.*.timeoutSeconds` consistently across `ModelProviderConfig`, `ModelProviderSchema`, labels/help, generated schema artifacts, and docs, then plumb it into model-provider request execution through a provider/request-scoped timeout or dispatcher path rather than mutating global Undici state per attempt. Add focused tests proving schema acceptance/generated metadata and that overlapping providers cannot affect each other's timeout behavior.
