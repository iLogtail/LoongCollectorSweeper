## Summary

Keep #16121 open. Current main still lacks first-class Telegram inline custom emoji support for normal bot replies, captions, action sends, and preview streaming. The bundled Telegram formatter can only render the existing Markdown IR styles and links, raw HTML is deliberately escaped in normal markdown mode, and the only custom-emoji hits are forum-topic icon or reaction metadata rather than `<tg-emoji>` message entities.

## Best Possible Solution

Keep #16121 open and implement this in the bundled Telegram plugin formatting path: add a structured custom emoji representation or explicit markdown-level syntax carrying `emoji-id` plus fallback emoji, validate/escape the id, render Telegram HTML as `<tg-emoji emoji-id="...">fallback</tg-emoji>`, preserve plain-text fallback, and cover normal messages, captions, replies/quotes, chunking, HTML parse fallback, preview streaming, docs, and Telegram formatter/delivery tests.

## Risks / Open Questions

- A naive raw-HTML allowlist would weaken the formatter's deliberate HTML escaping and could regress Telegram parse-fallback behavior.
