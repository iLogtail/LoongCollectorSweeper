## Summary

Current main still has the bug this PR is trying to fix: direct `/hooks/wake` cannot carry `sessionKey`, mapped `action: "wake"` drops `agentId`/`sessionKey`, and wake dispatch still enqueues only to the resolved main session. #35231 is related but closed unmerged, so it is not a canonical superseding fix. The PR should stay open for maintainer review/landing, with the remaining global-scope review comments checked before merge.

## Best Possible Solution

Keep #70268 open. The best path is to finish maintainer review of the PR as the focused fix for #64556: preserve explicit wake `sessionKey`/`agentId` routing, canonicalize the queued event and heartbeat target consistently, enforce `allowedSessionKeyPrefixes` for explicit and computed wake targets, and add regression tests for direct wake, mapped wake, agentId-only mapped wake, and global-scope behavior. #35231 should remain historical related work rather than the canonical replacement because it is closed and unmerged.
