## Summary

Keep open. Current `main` still has multiple lazy imports of `../../config/sessions/store.runtime.js`, but `buildCoreDistEntries()` still does not emit a stable `config/sessions/store.runtime` entry and the current tsdown regression test suite does not cover it. The latest release tag `v2026.4.23` shows the same omission. The PR timeline’s cross-reference to the already-closed `subagent-registry.runtime` fix is useful supporting evidence: `main` adopted that stable-entry pattern for a similar bug, but not for `store.runtime`, so this PR still appears relevant rather than obsolete.
