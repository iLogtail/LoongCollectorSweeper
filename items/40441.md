## Summary

Keep open. Current `main` still has the stale-config bug the PR targets, but this PR's unconditional `configFormDirty = false` inside shared `loadConfig()` would also discard unsaved edits during automatic refresh paths like tab switches and reconnects, not just explicit Reload clicks.

## Evidence

- **Current main still leaves dirty form state intact during config loads:** `loadConfig()` fetches `config.get` and immediately calls `applyConfigSnapshot()` without clearing `configFormDirty`. `applyConfigSnapshot()` intentionally refuses to replace `configForm`/`configFormOriginal`/`configRawOriginal` when the form is dirty, so a reload path can leave stale form state visible.
  - file: [ui/src/ui/controllers/config.ts:39](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/ui/src/ui/controllers/config.ts#L39)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '35,120p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **The proposed fix changes shared loader semantics, not just the explicit Reload action:** The UI wires the Reload button directly to `loadConfig(state)`, but the same function is also reused from non-user-initiated refresh flows. Putting `state.configFormDirty = false` into `loadConfig()` would therefore affect every caller, not only explicit reload.
  - file: [ui/src/ui/app-render.ts:859](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/ui/src/ui/app-render.ts#L859)

- **Automatic tab refreshes call the same loader:** `refreshActiveTab()` calls `loadConfig(app)` for multiple tabs (`config`, `communications`, `appearance`, `automation`, `infrastructure`, `aiAgents`, `nodes`, `dreams`). Those are ordinary navigation/data refresh paths, so the PR would clear pending edits on tab changes.
  - file: [ui/src/ui/app-settings.ts:335](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/ui/src/ui/app-settings.ts#L335)

  - file: [ui/src/ui/app-gateway.ts:341](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/ui/src/ui/app-gateway.ts#L341)

That matches the current shared-loader behavior and reinforces that any override should be narrowly scoped to an explicit reload action, not all config fetches.

## Risks / Open Questions

- Closing this PR would lose a still-valid user-facing bug report; current `main` still lacks an explicit-reload-only fix for stale config form state.
- Merging the PR unchanged would introduce a behavioral regression by silently discarding unsaved config edits during automatic refreshes such as tab switches or reconnects.
