## Summary

Keep open. Current main still has a real unresolved Telegram inbound-document gap for files that exceed the public Bot API download limit: the resolver returns no media attachment, the message body can remain `<media:document>`, and the docs/config surface do not clearly distinguish `mediaMaxMb` from Telegram's 20 MB public `getFile` limit. The related PR #55970 is open and only scopes a clearer fallback reason, not guaranteed attachment download support.

## Best Possible Solution

Keep #55917 open and handle it as a Telegram inbound media UX/docs bug: clarify `channels.telegram.mediaMaxMb` versus Telegram's public 20 MB `getFile` limit, surface an explicit agent-visible fallback reason for failed document downloads, and document or improve the self-hosted Bot API `apiRoot`/`trustedLocalFileRoots` path for users who need >20 MB inbound documents. PR #55970 can be used as a partial starting point if maintainers want that fallback-reason behavior, but it does not close the full reported attachment expectation by itself.
