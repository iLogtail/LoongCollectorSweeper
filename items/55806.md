## Summary

Keep open. Current main has partial tool execution diagnostics and OTel spans for sanitized parameter shape, duration, and error category, but the broader request is not fully implemented: persisted/tool-result JSON still lacks structured error fields, runtime logs still use the old debug-only start/end lines in the embedded handler, and there is no generic context writeback tracking for write/edit/memory-style mutations. The related PR #55846 is still open and appears to cover only part of the issue, so it is not enough to close this issue as superseded.

## Best Possible Solution

Keep the issue open and narrow the implementation plan around existing diagnostics: decide the stable schema for sanitized tool input summaries, add structured tool error categorization where persisted/tool-result consumers can read it, and add generic writeback summaries for write/edit/apply_patch/memory mutations without logging raw sensitive content. #55846 should either be updated to cover that full contract or treated as a partial precursor.
