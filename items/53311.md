## Summary

Keep open. The report is concrete, recent, and still matches current main: plugin-aware config validation still records an unknown configured channel id as a fatal issue, and gateway startup rejects invalid config snapshots. The linked fix PR #53315 is open/unmerged, so the bug is not implemented on main yet.

## Best Possible Solution

Keep #53311 open until main changes the unknown configured channel-plugin path in src/config/validation.ts from a fatal issue to a warning/skipped channel, updates the Matrix/plugin install recovery tests that currently expect invalid config, and adds a regression test proving gateway startup continues with remaining channels when one configured channel plugin is missing. PR #53315 is the obvious starting point but still needs maintainer review, tests, and merge/supersession.
