## Summary

Keep #39762 open. Current main still does not implement a Feishu read-only group mode: the schema/docs expose group admission and mention controls, but no outbound group-reply guard; allowed group messages still reach real reply dispatchers; explicit group/chat targets still resolve normally. The PR is focused, unmerged Feishu bundled-channel work with substantive review context, not a duplicate or safe cleanup close.

## Best Possible Solution

Keep #39762 open. If maintainers want this behavior, finish it in the existing Feishu bundled channel plugin: add typed disableGroupReplies/read-only group config, enforce it in both broadcast and single-agent reply dispatch, block explicit and bare chat_id outbound targets while keeping DMs working, document it, and cover config/send-target/bot regressions. If maintainers do not want this as first-party Feishu behavior, that should be an explicit product decision rather than automated cleanup.
