## Summary

Keep #39762 open. Current main still does not implement Feishu read-only group mode: the Feishu config schema/docs have no `disableGroupReplies` key, group dispatch paths still create real Feishu reply dispatchers, and outbound Feishu target resolution has no read-only group guard. The PR remains a substantive bundled Feishu plugin change, though it still needs maintainer review for the bare `oc_*` chat-id bypass flagged in PR review.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to top-level and per-account Feishu config/schema/docs, enforce no-op group dispatch for both broadcast active-agent and single-agent paths, block explicit and bare Feishu group/chat outbound targets when enabled, and cover those paths with targeted regression tests while avoiding unrelated generated/baseline churn.
