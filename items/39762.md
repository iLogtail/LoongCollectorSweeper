## Summary

Keep #39762 open. Current main still does not implement Feishu read-only group mode, and the PR is a substantive bundled Feishu plugin change rather than obsolete cleanup. The supplied PR discussion also identifies a remaining raw `oc_*` chat-id outbound bypass that should be resolved before merge.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to top-level and per-account config schemas and docs, enforce no-op group dispatch for both broadcast active-agent and single-agent paths, block explicit and bare Feishu `chat_id` outbound targets when enabled, and cover those paths with targeted regression tests.
