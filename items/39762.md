## Summary

Keep #39762 open. Current main still does not implement Feishu read-only group mode: there is no `disableGroupReplies` config/schema/docs surface, accepted Feishu group messages still create real reply dispatchers in both broadcast active-agent and single-agent paths, and outbound Feishu target resolution still allows `chat_id` targets. The PR discussion remains actionable because the author addressed earlier broadcast and whitespace review feedback, while a later review still flags the raw `oc_*` chat-id bypass and the PR includes unrelated `.secrets.baseline` churn.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to the Feishu config/schema/docs surfaces, enforce a no-op dispatcher for accepted group messages in both broadcast active-agent and single-agent paths, block explicit and raw Feishu `chat_id` outbound targets when enabled, add targeted regression tests, and remove or separately justify the `.secrets.baseline` churn before merge.
