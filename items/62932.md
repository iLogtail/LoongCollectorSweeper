## Summary

Current `main` already handles the reported silent-after-tools failure mode: it retries empty tool-followup turns for non-frontier models and, if that still fails, emits a user-visible incomplete-turn error instead of leaving the session with no text reply.

## Evidence

- **Model-agnostic silent-after-tools retry:** `runEmbeddedPiAgent` now has an explicit retry path for turns that finish after a tool-call sequence with `stopReason="error"`, zero output tokens, and empty content; the inline comment says this previously caused a "silent gap until the user nudges."
