## Summary

Current main and the latest checked release no longer match the reported v2026.3.23 behavior. The bundled ACPX Claude alias now resolves to the Claude Agent SDK ACP adapter, which offers Claude subscription login through the Claude Code CLI instead of requiring only a standalone Anthropic API key.

## Best Possible Solution

Close this issue as implemented, keep the shipped acpx + Claude Agent SDK ACP adapter path, and optionally clean the stale internal ACP router skill reference. If users still hit auth failures on v2026.4.24+ or current main, they should open a fresh issue with current OpenClaw, acpx, `@agentclientprotocol/claude-agent-acp`, and Claude Code versions plus logs.
