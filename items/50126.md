## Summary

Keep open. Current main has fixed some Telegram/shared-delivery hook paths, but the broader inconsistency reported in #50126 still exists: channel reply dispatchers and plugin-handled message actions can send outbound replies without a universal `message:sent`/`message_sent` observer. Related context also shows #70928 remains open for webchat/TUI hook gaps, so this issue is not safely obsolete or fully superseded.

## Best Possible Solution

Keep #50126 open as the umbrella until maintainers either centralize outbound delivery observation around a single hook-emitting seam or split the remaining work into canonical narrower issues that cover webchat/TUI, plugin-handled message actions, Feishu and other direct channel reply dispatchers. The eventual fix should make `message_sending` and `message_sent`/`message:sent` fire consistently for every successful and failed outbound dispatch path, with channel-level regression tests.
