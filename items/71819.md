## Summary

Keep PR #71819 open. Current main still releases the inbound dedupe claim on every dispatch error and has no poisoned terminal state for errors after externally visible output. The PR is a focused fix for open bug #69303 with regression coverage and recent review follow-up, not stale or superseded.

## Best Possible Solution

Keep #71819 open for maintainer review and likely landing, or land an equivalent fix. The right product/code direction is a third inbound dedupe terminal state: release for failures before any visible output, but stamp/poison the dedupe key after externally visible dispatch so replayed provider message_id values cannot duplicate visible replies or rerun side effects. Track the tool-result-only exclusion separately if maintainers want broader idempotency.
