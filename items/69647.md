## Summary

Current main still preserves a stale legacy/default persisted gatewayUrl when the Control UI is opened under a different reverse-proxy path, so this PR is not obsolete or safely closeable. The PR discussion also shows active review iterations that addressed real same-host override risks, so the right maintainer action is to keep it open for review and validation rather than auto-close it.

## Best Possible Solution

Keep the PR open. The best path is to land a reviewed variant that distinguishes page-derived gateway defaults from explicit user overrides, updates or seeds the current page-scoped settings key safely, preserves intentional custom gateway URLs including same-host different-path deployments, and adds focused storage regression tests for /a -> /b same-host reverse-proxy fallback from legacy localStorage.
