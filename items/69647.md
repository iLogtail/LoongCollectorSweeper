## Summary

Current main still has the reported stale-gateway failure mode: the Control UI derives a path-prefixed default gateway URL, but `loadSettings()` can fall back to the shared legacy localStorage key and preserve a sibling path's cached `gatewayUrl`. PR #69647 is not merged or superseded and targets the implicated storage code, so it should remain open for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep the PR open. The best path is to refine or replace this patch with a storage migration that distinguishes page-derived defaults from intentional user gateway overrides, add regression tests for same-host `/a` to `/b` path-prefix fallback plus custom same-host WebSocket paths, then merge once the behavior is proven against current main.
