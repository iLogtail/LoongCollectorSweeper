## Summary

Keep open. Current main has partial timeout hardening, but it still does not implement phase-aware timeout attribution for embedded runs or `agent.wait`, and the auth-profile policy suppresses all timeout cooldowns rather than distinguishing provider-started timeouts from queue/preflight/post-turn timeouts. The related PR #65511 claimed to close this issue but was closed unmerged and its head SHA is not present in this checkout.

## Best Possible Solution

Keep the issue open and implement it in the embedded runner plus gateway wait surface: add structured timing/phase metadata, mark the exact provider request start and first-token/provider elapsed points, carry timeoutPhase through lifecycle/dedupe/agent.wait snapshots, prefer terminal snapshots during post-turn cleanup, and update auth-profile failure handling so only provider-started timeout failures can affect profile cooldown/rotation. Add focused tests around queued/preflight/provider/post-turn timeout attribution and Anthropic auth-profile cooldown behavior.
