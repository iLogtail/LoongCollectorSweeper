## Summary

Keep #9820 open. Current main still does not implement native JSONL support for builtin memorySearch.extraPaths: discovery admits Markdown plus enabled multimodal media, memory_get still rejects non-.md targets, the watcher is Markdown/media-oriented, and docs/types continue to describe extraPaths as Markdown files/directories. Existing session-transcript JSONL indexing is a separate sessions source and does not satisfy arbitrary raw JSONL logs configured through extraPaths.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to builtin memorySearch.extraPaths or explicitly document a narrower product direction. A complete core fix should update extra-path admission, watcher globs, text/chunk handling with JSONL line mappings, memory_get/read behavior, focused tests, config help/types, and public memory docs. If arbitrary JSONL should stay outside builtin memory, document the intended alternative through session transcript memory, QMD/custom collections, or a plugin-owned converter path.
