## Summary

Keep open. Current main still has the reported Control UI bug: `validateCronForm` rejects `timeoutSeconds: "0"`, the focused UI test still expects that rejection, and `buildCronPayload` only forwards positive timeout values. Backend and gateway code already accept/preserve `timeoutSeconds: 0` and treat it as no timeout, so the remaining work is the UI/controller fix, not cleanup closure. The supplied GitHub context also shows linked PR #41275 is still open/unmerged and PR #46003 was closed unmerged.

## Best Possible Solution

Keep #41272 open until a focused Control UI fix lands. The implementation should allow `timeoutSeconds === 0` in `ui/src/ui/controllers/cron.ts`, preserve explicit zero in `buildCronPayload`, update the UI test expectation/add a regression test for negative vs zero values, and adjust the English help/error copy so the form documents zero as no-timeout mode. The linked open PR #41275 or an equivalent patch is the right path to close this bug.
