## Summary

Keep open. Current `main` still rejects `timeoutSeconds: 0` in the cron UI and drops `0` on submit, while core cron paths explicitly support `0` as no-timeout.

## Evidence

- **UI validation still rejects 0:** `validateCronForm` treats any non-empty timeout `<= 0` as invalid, so a form value of `"0"` still produces `cron.errors.timeoutInvalid`.
  - file: [ui/src/ui/controllers/cron.ts:150](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/controllers/cron.ts#L150)
  - command: `nl -ba ui/src/ui/controllers/cron.ts | sed -n '135,170p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **UI submit path drops explicit 0:** `buildCronPayloadFromForm` only writes `payload.timeoutSeconds` when the parsed value is `> 0`, so even if the field were accepted, `0` would not be preserved in the saved payload.
  - file: [ui/src/ui/controllers/cron.ts:591](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/controllers/cron.ts#L591)
  - command: `nl -ba ui/src/ui/controllers/cron.ts | sed -n '585,600p'`

- **Current UI test codifies the bug:** The existing controller test sets `timeoutSeconds: "0"` and expects `errors.timeoutSeconds` to equal `cron.errors.timeoutInvalid`, confirming the current intended UI behavior still matches the report.
  - file: [ui/src/ui/controllers/cron.test.ts:1081](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/controllers/cron.test.ts#L1081)
  - command: `nl -ba ui/src/ui/controllers/cron.test.ts | sed -n '1068,1086p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Backend cron paths support 0 as no-timeout:** Gateway/server and cron normalization tests accept and preserve `payload.timeoutSeconds: 0`, so the mismatch is specifically in the UI layer rather than the underlying cron contract.
  - file: [src/gateway/server.cron.test.ts:431](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/gateway/server.cron.test.ts#L431)
