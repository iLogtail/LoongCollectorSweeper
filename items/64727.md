## Summary

Keep #64727 open. Current main does return the newest `chat.history` window, which mitigates one suspected window-order failure, but the subagent announce extraction path still hardcodes `limit: 100`, still returns sanitized `toolResult`/`tool` text before consulting the latest-assistant fallback, and the completion capture retry stops as soon as any non-empty raw tool output is seen. I did not find source, docs, tests, release notes, or related-item evidence that the reported tool-heavy announce selection bug is fixed.

## Best Possible Solution

Keep the issue open and fix it in the subagent announce output seam. A conservative implementation would prefer an explicit latest assistant lookup before accepting `latestRawText`, or only use raw tool/toolResult fallback after a bounded retry/pagination path proves there is no visible assistant reply. Add a regression test around `src/agents/subagent-announce-output.ts` or the existing announce format suite with more than 100 tool-heavy messages and a final assistant summary, plus a timing case where the first history snapshot has tool output and a later snapshot has the final assistant summary.
