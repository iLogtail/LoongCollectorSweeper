## Summary

Do not close #70287. Current main still constructs `msteams:search` requests with Graph `$search`, and the mainline tests still assert that URL shape, so the linked app-only Graph bug is not implemented or superseded on main. The PR is a focused msteams plugin bug fix with relevant review follow-up; it needs normal maintainer review/merge judgment, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open and review or land a corrected msteams plugin fix: remove `$search` from app-only chat/channel message search, page recent messages with `$top` capped at 50, filter locally with HTML-stripped case-insensitive matching, retain supported sender `$filter`, and keep regression tests covering the URL shape and pagination tradeoff.
