## Summary

Current main and the latest release still build `msteams:search` with Graph `$search`, and the existing tests explicitly assert that old URL shape. PR #70287 is a focused, still-relevant fix for the linked app-only Microsoft Graph failure, not obsolete or implemented elsewhere.

## Best Possible Solution

Keep #70287 open as the canonical remaining-work PR for the MS Teams app-permission search bug. The best path is maintainer review, rebase or port onto current main, then land the list-and-local-filter implementation with `$top=50` pagination, HTML-aware local matching, and regression tests that prevent `$search` from returning on chat/channel message endpoints.
