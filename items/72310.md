## Summary

Keep open. Current main still defaults both plugin discovery and manifest registry caches to 1 second, activation still performs filesystem discovery before narrowing to requested plugin ids, and the checked-out tree contains 121 bundled plugin directories. Existing mitigations for the older node_modules recursion issue are present, but they do not satisfy this issue's broader request for production-safe default scanning behavior on constrained hardware.

## Best Possible Solution

Keep this open and turn it into a measurable plugin-discovery performance task: profile gateway startup, reload, and plugin read-only commands on constrained hardware or a synthetic low-core benchmark; then lengthen or adapt discovery/manifest cache windows, use persisted plugin index data or activation planning earlier, add reload hysteresis where plugin state changes cause repeated scans, or add an explicit constrained-host performance preset with regression coverage.
