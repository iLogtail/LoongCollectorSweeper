## Summary

Keep open. Current main still depends on structured `message`/interactive payloads for Telegram inline keyboards and does not show evidence of parsing raw `[[{"text":...,"callback_data":...}]]` text output into buttons, so the reported Gemini behavior remains plausible.

## Evidence

- **System prompt only adds guidance:** The runtime prompt tells models with Telegram inline-button capability to use the `message` tool with `buttons=[[{text,callback_data,style?}]]`, which is guidance rather than a text-output parser.
  - file: [src/agents/system-prompt.ts:371](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/system-prompt.ts#L371)

  - file: [extensions/telegram/src/outbound-adapter.ts:81](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/telegram/src/outbound-adapter.ts#L81)
