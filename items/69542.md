## Summary

Keep open. Current main still lacks `acp.fallbacks` and has no ACP backend failover loop; `runTurn` only retries fresh handles on the same resolved backend. The PR targets a real core behavior gap, but its current diff should be reworked before merge because the provided review context identifies concrete correctness issues and the branch includes unrelated smoke/runtime-deps changes.

## Best Possible Solution

Keep the item open and ask for a focused rework. The best path is a narrow ACP core PR that splits out the unrelated smoke/runtime-deps commits, corrects the linked issue, adds `acp.fallbacks` to the config/schema/docs only if maintainers accept the contract, and implements backend switching inside `AcpSessionManager.runTurn` using resolved session metadata and existing binding/agent/global backend precedence. It should only fail over before user-visible streamed output or tool calls, include session-init and turn unavailable cases, close or evict the old runtime handle before switching, record each failed backend attempt in observability, and add targeted ACP manager/config tests.
