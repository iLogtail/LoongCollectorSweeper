## Summary

Keep open. Current main already handles Feishu reply context when `message.parent_id` is present, but the issue discussion identifies the remaining Feishu UI quote path as `quote.message_id`, and current main still has no event type, context field, parser, fetch path, or regression test for that quote ID.

## Best Possible Solution

Keep #29487 open. Implement the remaining Feishu plugin behavior by verifying the UI quote event payload, extending the event/context types to carry the quoted message ID, parsing `quote.message_id`, preferring it for explicit quote context while preserving `parent_id` fallback, and adding regression tests for both quote-only UI replies and existing parent_id replies under current group context visibility rules.
