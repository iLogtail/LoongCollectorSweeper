## Summary

Keep open. Current main still implements Feishu quoted/replied context only through `message.parent_id`; it does not model, parse, fetch, or test the Feishu UI quote identifier discussed in the issue comments (`quote.message_id`/quote ID). The linked PR #29501 was closed unmerged and only described parent-message parsing robustness, while #60891 is a related parse-failure issue rather than the missing quote-ID path.

## Best Possible Solution

Keep #29487 open. Implement the Feishu plugin behavior by verifying the UI quote payload, extending the event/context types to carry the quoted message ID, parsing `quote.message_id` or the verified equivalent, preferring that explicit quote ID when fetching quoted content while preserving `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent_id replies under group context visibility rules.
