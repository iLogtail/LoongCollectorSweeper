## Summary

Keep #64721 open. Current main partially addresses the report by exposing `payload.model` and `payload.timeoutSeconds` in the agent-facing cron tool schema, but the requested `payload.contextTokens` and `maxSpendUsd` fields are still not part of the cron payload/tool/protocol contract, and gateway validation remains closed to unknown cron payload fields.

## Best Possible Solution

Keep the issue open and narrow the implementation to the remaining gap: add supported per-job context cap and spend cap fields intentionally across the cron tool schema, gateway protocol schemas/validators, `CronAgentTurnPayload` types, normalization/patching, runtime enforcement, CLI/docs, and regression tests. Preserve the already-shipped `model` and `timeoutSeconds` behavior.
