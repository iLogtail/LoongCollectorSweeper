## Summary

Keep PR #70518 open. Current main does not implement a heartbeat-specific skill allowlist: the heartbeat config schema/type/docs omit `allowSkills`, and `runHeartbeatOnce` never passes a heartbeat `skillFilter` into `getReplyFromConfig`. The PR is a focused, plausible additive fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation is an additive `agents.defaults.heartbeat.allowSkills` / `agents.list[].heartbeat.allowSkills` config field in the heartbeat type, zod schema, generated schema/help/labels, docs, and config-honor inventory, with `runHeartbeatOnce` passing the resolved value as `skillFilter` to `getReplyFromConfig` while preserving the distinction between unset and `[]`. Regression tests should cover defaults, per-agent overrides, and empty-list disabling.
