## Summary

Keep PR #70518 open. Current main still does not implement a heartbeat-specific skill allowlist: the heartbeat schema/type/docs/inventory omit `allowSkills`, and `runHeartbeatOnce` calls `getReplyFromConfig` without passing a heartbeat `skillFilter`. The PR is a focused, plausible fix with useful discussion evidence from the Greptile review, but it needs normal maintainer review/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and have the contributor or maintainer rebase it onto current main, then implement the same narrow product path: add `agents.defaults.heartbeat.allowSkills` and `agents.list[].heartbeat.allowSkills` to the config schema/types/help/generated surfaces, pass the merged heartbeat allowlist as `skillFilter` from `runHeartbeatOnce`, and keep regression coverage for non-empty and empty allowlists.
