## Summary

Keep PR #55004 open. Current main still has the user-visible gap: `dashboardCommand` resolves and logs the dashboard URL, then copies and auto-opens it without a local gateway reachability or daemon-state preflight. The PR is a plausible focused fix, though the final implementation should preserve valid custom/tailnet bind behavior noted in review discussion.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that adds a local-mode dashboard preflight before clipboard/browser side effects, skips the gate for `--no-open` and remote flows as intended, treats auth/device-identity close responses as reachable when appropriate, and probes in a way that does not regress valid custom or tailnet local bind configurations. Add focused regression coverage in `src/commands/dashboard.links.test.ts` or nearby dashboard command tests for unreachable local gateway, reachable/auth-close gateway, `--no-open`, and non-loopback bind behavior.
