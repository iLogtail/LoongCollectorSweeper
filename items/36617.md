## Summary

Keep #36617 open. Current main has meaningful Japanese localization for the browser Control UI and a docs i18n publish pipeline, but the requested core UX scope remains broader: CLI onboarding, wizard prompts, help text, and workspace templates still use English source strings without a shared key-based core i18n path.

## Best Possible Solution

Keep #36617 open until maintainers either define a narrower localization scope or add a core/key-based i18n path beyond the existing Control UI and docs pipeline. The likely path is to preserve the current UI ja-JP support, then extend locale selection and fallback behavior into CLI onboarding/wizard/help text and workspace templates, with tests for locale resolution, fallback, and generated bundle/schema drift.
