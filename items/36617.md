## Summary

Keep open. Current main has meaningful partial implementation for Control UI i18n, including a generated ja-JP locale, lazy loading, browser-locale detection, persistence, and English fallback. The issue asks for first-class Japanese localization across core UX/onboarding/help/templates, and current main still has English-only core UI/help surfaces, so the requested capability is not fully implemented enough for conservative auto-close.
