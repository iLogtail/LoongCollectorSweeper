## Summary

Keep open. Current main still has the exact unguarded `hashText(event.text)` and downstream `emitSystemStatus(event.text, ...)` paths, so the PR is not implemented or obsolete. The PR discussion also shows the initial bot review found a real adjacent crash path and the author followed up with the second guard in `9c524dcee2`, making the PR still relevant directionally.

## Best Possible Solution

Keep the PR open and either land this small guard with a focused regression test or recreate the same fix on main. The best code path is to make ACP status projection resilient to missing text at both dedupe and delivery, then add a test that exercises a visible `usage_update` without `text` so the Telegram group follow-up crash cannot regress.
