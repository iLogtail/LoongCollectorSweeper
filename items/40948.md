## Summary

Keep #40948 open. Current main still has no `session-recap`/`session_recap` handling in the shared inbound metadata stripper, and both the TUI and Control UI user-message paths depend on that stripper. The issue is concrete, recent, non-maintainer-authored, and only has the non-protected `stale` label. Related PRs include one open unmerged fix candidate (#41013) and several closed unmerged attempts, so the remaining work is not shipped or superseded.

## Best Possible Solution

Keep the issue open until main gains a narrow shared fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a `session-recap`/`session_recap` quick check and leading-block stripper, preserve mid-message literal tags, and add regression coverage in the shared stripper plus TUI and Control UI user-message consumers. The existing open PR #41013 can be reviewed, updated, or superseded by a fresh patch that matches current main.
