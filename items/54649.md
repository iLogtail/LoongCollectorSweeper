## Summary

Keep open. Current main appears to have already fixed the Stop/Cancel visibility gap, but it still clears the only draft state on every session switch, so the per-session draft persistence request is not implemented.

## Evidence

- **Session switch still discards the active draft:** `resetChatStateForSessionSwitch()` unconditionally assigns `state.chatMessage = ""`, and `switchChatSession()` always calls that reset path before loading the next session.
  - file: [ui/src/ui/app-render.helpers.ts:68](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/app-render.helpers.ts#L68)
  - command: `nl -ba ui/src/ui/app-render.helpers.ts | sed -n '56,90p;509,522p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Draft is still a single global field, not a per-session cache:** The chat view reads and writes the draft through `state.chatMessage` (`getDraft` / `onDraftChange`), so there is no separate per-session draft store in the current render path.
  - file: [ui/src/ui/app-render.ts:2257](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/app-render.ts#L2257)
