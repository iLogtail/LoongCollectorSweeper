## Summary

Keep #47994 open. Current main still lacks either of the two concrete designs that would close the sticky fallback problem: the PR's `modelIsFromFallback` provenance flag or #68991's persisted-selection/runtime-accounting split. Main has a useful `modelOverrideSource: "auto"` self-heal path for auto fallback overrides, but session runtime persistence still writes the actual fallback provider/model into `modelProvider`/`model`, and model resolution still prefers those fields on later turns.

## Best Possible Solution

Keep #47994 open for maintainer follow-up. The best product direction is to explicitly separate per-turn fallback runtime accounting from next-turn selected model persistence, or add a robust fallback provenance flag, while preserving the current `modelOverrideSource: "auto"` self-heal behavior. Close this PR only after equivalent behavior lands on main with regression coverage across session persistence, model resolution, and status/sessions/API display surfaces.
