## Summary

Keep #49879 open. Current main implements `/usage off|tokens|full` as a per-session setting and footer renderer, but there is no `agents.defaults.responseUsage` config key in the typed config, zod schema, generated schema/help, docs, or response runner fallback path. The issue is concrete, non-maintainer-authored, has no protected labels, and the provided related reports cover analogous `agents.defaults.*` requests rather than a canonical response-usage tracker.

## Best Possible Solution

Keep the issue open as a core config/runtime feature request. The likely implementation should add `agents.defaults.responseUsage` to the config type, zod schema, generated schema/help, and docs; then seed new sessions from that default when no per-session `responseUsage` exists, while preserving explicit per-session `/usage` overrides and adding focused tests for new-session inheritance and reset/session-patch behavior.
