## Summary

Keep open. Current `main` has a related partial mitigation for session-scoped orphaned tabs, but it does not strongly establish that managed browser processes are now cleaned up after automation tasks. The run-finish cleanup path closes tracked tabs only; explicit process stop still lives in the browser shutdown/stop path, and I found no startup orphan-browser sweep or `browser.idleTimeoutMs` implementation on `main`.

## Evidence

## Risks / Open Questions

- The newer session-scoped tab cleanup may reduce the reported symptom for cron/subagent workflows, but it does not prove that long-lived managed Chrome processes are cleaned up after ordinary automation tasks.
- Docs now say tracked browser tabs/processes are cleaned up on some run completions, but the implementation I inspected is tab-focused; that mismatch makes an auto-close decision unsafe.
