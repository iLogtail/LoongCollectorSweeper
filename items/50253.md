## Summary

Keep #50253 open. Current main still does not implement IPv6 LAN gateway binding: `gateway.bind=lan` resolves only to the IPv4 wildcard `0.0.0.0`, `gateway.bind=custom` accepts only canonical IPv4 addresses, and the active listener setup simply listens on the resolved host list. The issue comment’s suggestion to allow `::` or a specific IPv6 address remains a real core gateway feature gap, not a ClawHub/plugin item or a duplicate of the nearby Docker/LAN auth reports.

## Best Possible Solution

Keep the issue open and implement first-class IPv6 or dual-stack gateway binding in core. The likely path is to extend the gateway bind contract to support IPv6 custom hosts or a clear dual-stack LAN mode, update `resolveGatewayBindHost`/`resolveGatewayListenHosts` and runtime validation, add regression tests for `::` and specific IPv6 addresses, and update gateway config/security docs.
