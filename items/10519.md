## Summary

Keep #10519 open. Current main still renders Telegram fenced code blocks as `<pre><code>` without preserving the markdown fence language, and the Telegram tests still assert that missing-language output. The separate WebChat copy-button/language-class work does not affect Telegram Bot API HTML rendering.

## Best Possible Solution

Keep the issue open. Implement fenced-code language preservation in the shared Markdown IR/render path or a Telegram-specific fence-aware path, then update Telegram format, wrapping, chunking, and outbound tests to expect `class="language-js"` while preserving HTML escaping, file-reference wrapping rules, chunk limits, and plain-text fallback behavior.
