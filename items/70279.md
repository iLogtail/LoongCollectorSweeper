## Summary

Keep open. Current main partially addresses related claude-cli pitfalls, but the specific #70279 maintainer requests are not fully implemented: root/systemd Claude CLI guidance and the `IS_SANDBOX=1` workaround are not documented on the CLI backend/auth pages, skip diagnostics are generic `model-fallback/decision` logs rather than a CLI-backend-scoped skipped event, and the session store still records the successful runtime fallback model after a run even though normal next-turn selection now uses explicit `modelOverride` fields rather than those runtime fields.

## Best Possible Solution

Keep this as a narrow CLI backend diagnostics/docs bug. Best maintainer path: add or explicitly document a CLI-backend-scoped pre-dispatch skip diagnostic with reason/provider/model, add public CLI backend docs for Claude CLI under root/systemd with the supported non-root or `IS_SANDBOX=1` guidance if maintainers accept that workaround, and clarify/test the distinction between runtime `model` metadata and explicit `modelOverride` selection so fallback success does not look like a silent primary override.
