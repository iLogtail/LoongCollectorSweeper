## Summary

Keep open. Current main has meaningful partial fixes around fallback visibility, Claude CLI defaults, token-env clearing, and auto-failover session self-healing, but the root/systemd Claude CLI constraint and an operator-facing Claude-CLI-specific skip diagnostic are still not fully covered. The reporter’s follow-up already documents a working workaround (`IS_SANDBOX=1`, omit the stale `cliBackends` override, add the `anthropic:claude-cli` auth profile, avoid conflicting `CLAUDE_CODE_OAUTH_TOKEN`) and explicitly leaves the maintainer requests open.

## Best Possible Solution

Keep the item open, but narrow it. The best maintainer path is to document the supported root/systemd stance on `docs.openclaw.ai/gateway/cli-backends`, add a Claude CLI doctor warning for root-managed gateway processes if `claude-cli` is configured, and optionally mirror model-fallback `skip_candidate` decisions into an `agent/cli-backend` diagnostic when the skipped provider is a CLI backend. The already-shipped fixes for `jsonl`/stdin defaults, env-token clearing, auth-profile diagnostics, and auto-failover self-healing should stay as-is.
