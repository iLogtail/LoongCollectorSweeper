## Summary

Keep #70055 open. Current main has addressed the worst reported symptom by moving the old external CLI credential behavior to runtime-only overlays with debug/warn logging rather than the reported repeated info-level "synced ... credentials from external cli" output, but it still does not provide the requested config/env/CLI opt-out. External CLI auth resolution is still invoked unconditionally from the auth-profile overlay path, and the public auth config type/schema surface has no `auth.externalCliSync` or equivalent disable control.

## Best Possible Solution

Keep the issue open and implement a small, documented gate for built-in external CLI auth reuse: either `auth.externalCliSync` in `openclaw.json`, an env override such as `OPENCLAW_DISABLE_EXTERNAL_CLI_SYNC=1`, or both. Wire it into the `external-auth`/`external-cli-sync` overlay path, update generated schema/help/docs, and add regression tests for disabled default, provider include/exclude behavior if supported, and preservation of plugin-owned external auth profiles.
