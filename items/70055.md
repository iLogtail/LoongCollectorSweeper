## Summary

Keep #70055 open. Current main has removed the exact noisy `synced ... credentials from external cli` info-log path and now treats external CLI credentials as runtime-only overlays, but it still has no user-facing config/env/CLI opt-out for external CLI auth profile discovery. The remaining requested capability is concrete and belongs in core auth/config if maintainers want it.

## Best Possible Solution

Keep the issue open and implement an additive opt-out only if maintainers want this policy surface: add a documented `auth.externalCliSync` config/env gate, thread it into `overlayExternalAuthProfiles` or `resolveExternalCliAuthProfiles`, cover provider include/exclude/default-on behavior in auth-profile tests, and update the generated config schema/help plus the public auth/OAuth docs.
