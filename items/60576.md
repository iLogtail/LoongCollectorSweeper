## Summary

Keep PR #60576 open. Current main still treats `undefined` and `null` unhandled rejection reasons as unclassified failures and falls through to the generic fatal `process.exit(1)` path. The requested core behavior is not implemented on main or in the latest release tag inspected, and the regression coverage proposed by the PR is still absent.

## Best Possible Solution

Keep the PR open and land or rebase the narrow core fix: handle `undefined`/`null` after registered plugin handlers and before abort/fatal/config/transient classification in `src/infra/unhandled-rejections.ts`, with regression tests in `src/infra/unhandled-rejections.fatal-detection.test.ts`. This should remain core infra work because the global unhandled-rejection fallback owns `process.exit(1)`; plugin handlers can add scoped suppressions but do not make the default gateway behavior safe for missing rejection reasons.
