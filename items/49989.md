## Summary

Current main still appears to pass prefixed Synology Chat peer IDs through to the outbound client path without stripping the `synology-chat:` prefix. The linked fix PR #50109 was closed unmerged in the provided GitHub context, and the current regression test coverage does not include `sendMessage(..., "synology-chat:4")`.

## Best Possible Solution

Keep the issue open and land a focused Synology Chat fix: strip the `synology-chat:` prefix before numeric user ID parsing or add an `outbound.resolveTarget` path that guarantees outbound targets reach `sendMessage` as bare numeric IDs, then add a regression test for `sendMessage(..., "synology-chat:4")` producing `user_ids: [4]`. Consider also adding the requested warning when `sendMessage` returns `false` after retries.

## Evidence
