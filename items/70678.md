## Summary

Current main still contains the reported WhatsApp extension-local no-message watchdog. It defaults to 30 minutes, updates its baseline only on inbound app messages, and force-closes the socket with status 499 when the baseline ages past the timeout. The newer gateway stale-socket policy is transport-activity based, but that does not remove the WhatsApp-local watchdog, matching the maintainer comment that the issue is still valid. Keep #70678 open as the canonical parent while the open fix PR #71466 or an equivalent patch is reviewed.

## Best Possible Solution

Keep #70678 open as the parent bug. The best fix is to make WhatsApp liveness transport-backed, or otherwise disable/configure the extension-local no-message watchdog for quiet linked-device sessions, while preserving handled-message counters. Add regression coverage showing a quiet connected WhatsApp session does not receive a watchdog 499 reconnect, then close this issue when the fix ships.
