## Summary

Keep #70623 open. The Telegram phantom fallback half is implemented on current main and documented in the 2026.4.24 changelog, but maintainer comments explicitly left the issue open for the separate Discord inbound-blackout report. Current main still has Discord inbound support and diagnostics, but I found no source, test, changelog, or related-item evidence proving the Discord 'bot online/doctor ok/no session created' behavior is fixed or superseded.

## Best Possible Solution

Keep the issue open but treat the Telegram half as fixed. Narrow the remaining work to Discord inbound-blackout investigation: collect sanitized startup and message-receipt logs, effective Discord account/guild/channel config, status/probe output, and proof of whether the gateway receives Discord WebSocket/messageCreate events. Once a root cause is found, fix it in the Discord plugin monitor/gateway/preflight path and add a focused regression test around the failing stage.
