## Summary

Keep open. Current main has not absorbed this PR's Feishu comment hardening: there is no strict `to_user_id.open_id` recipient gate before comment processing, no delivery guard to suppress duplicate automatic final replies after tool-visible comment delivery, no angle-bracket escaping for Drive comment writes, typing reaction state is still not account-scoped, and bot identity probing still lacks `forceFresh` plus credential-aware cache keys. The related #66427 is closed and unmerged in the provided context, so it is not a shipped replacement for #70471.

## Best Possible Solution

Keep #70471 open for maintainer review. The best path is to land a focused version of the PR's core fixes on the Feishu plugin: strict comment-recipient gating, tool-delivery duplicate-final suppression, `<`/`>` escaping for Drive comment writes, account-scoped typing cleanup, and force-fresh credential-aware bot identity probing with regression tests for each path. If maintainers prefer the broader policy surface from #66427, split that into a separate follow-up rather than closing this unshipped hardening PR.
