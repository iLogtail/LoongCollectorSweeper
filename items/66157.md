## Summary

Keep open. Current `main` has related Active Memory improvements, including forwarding `toolsAllow` and shrinking embedded-run prompt surface, but there is no high-confidence proof that the reported Discord channel timeout on 15s budgets is resolved. The plugin still explicitly times out and records `status=timeout`, and the repo does not contain a Discord channel regression proving this case is fixed.

## Evidence

- **Active Memory now requests only memory tools:** The blocking recall subagent invokes `runEmbeddedPiAgent` with `toolsAllow: ["memory_search", "memory_get"]`, which is a relevant post-report reduction in tool surface.
  - file: [extensions/active-memory/index.ts:1680](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/extensions/active-memory/index.ts#L1680)

This matches the later cross-referenced tool-filtering discussion, but by itself does not prove the Discord timeout is gone.
