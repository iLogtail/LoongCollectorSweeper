## Summary

Current `main` still has the array-shaped device-pairing state bug the PR describes, and the PR remains useful but incomplete because it adds no regression test. This should stay open for maintainer review rather than be auto-closed.

## Evidence

- **Current main still trusts parsed state shape:** `loadState` on current `main` still assigns `pending ?? {}` and `paired ?? {}` directly into the in-memory records. Because arrays are truthy, `[]` would still flow through unchanged instead of being coerced to an object.
  - file: [src/infra/device-pairing.ts:143](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/device-pairing.ts#L143)
  - command: `nl -ba src/infra/device-pairing.ts | sed -n '143,155p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **JSON loader does not validate object shape:** `readJsonFile` returns raw `JSON.parse(raw)` with no schema or plain-object validation, so array-typed or primitive JSON values can reach `loadState` as-is.
  - file: [src/infra/json-files.ts:29](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/json-files.ts#L29)
  - command: `nl -ba src/infra/json-files.ts | sed -n '25,35p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **No current regression coverage for corrupted array state:** The existing `device-pairing` test file has many `requestDevicePairing` coverage points, but no test referencing `pending.json`, `paired.json`, or `loadState` corruption handling. That matches the review discussion already attached to the PR.
  - file: [src/infra/device-pairing.test.ts:27](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/device-pairing.test.ts#L27)
  - command: `rg -n "pending.json|paired.json|loadState|requestDevicePairing\(" src/infra/device-pairing.test.ts`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **PR discussion already identifies remaining gap:** The provided PR discussion includes review feedback that the bug is real and the patch is tightly scoped, but it also flags the missing regression test and a possible broader non-object guard. The timeline shows no follow-up commits after that feedback.

## Risks / Open Questions

- Closing this PR would drop a still-live device-pairing state-corruption fix on current `main`.
- Merging as-is would still leave the bug without automated regression coverage, making future reintroduction easy.
- There is a plausible follow-up hardening gap for other non-object JSON values beyond arrays, so maintainers may want a slightly broader guard before merge.
