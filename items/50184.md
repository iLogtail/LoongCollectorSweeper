## Summary

Keep #50184 open. Current main still matches the reported regression: Telegram DM answer/reasoning lanes force `previewTransport: "message"`, which bypasses the lower draft stream's `auto` behavior that would prefer `sendMessageDraft` for DM threads. The issue comments correctly identify the transport-precedence problem, and the current tests codify the forced message behavior rather than a fix.

## Best Possible Solution

Keep the issue open for a Telegram maintainer fix. The likely product/code path is to remove or narrow the unconditional DM `message` override in `extensions/telegram/src/bot-message-dispatch.ts`, let the answer lane pass `auto` so `extensions/telegram/src/draft-stream.ts` can choose draft for DMs, or add an explicit Telegram config preference for DM preview transport. Update the dispatch tests that currently assert `previewTransport: "message"` and add a regression test for `channels.telegram.streaming = partial` in a DM thread.
