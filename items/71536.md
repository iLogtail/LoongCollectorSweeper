## Summary

Keep open. The issue is new, non-maintainer-authored, and describes a concrete Telegram `/restart` authorization failure on 2026.4.23. Current main still has a two-stage Telegram native-command path where pre-dispatch allowlist authorization and later owner-only `/restart` authorization can both reject with the same “You are not authorized to use this command.” message, and I did not find source/test/release evidence that this specific regression is already fixed or superseded.

## Best Possible Solution

Keep the issue open and ask for or reproduce the exact Telegram config path. The likely maintainer path is a focused regression test in `extensions/telegram/src` that drives native `/restart` through Telegram authorization into `handleRestartCommand`, covering numeric `channels.telegram.allowFrom`, `channels.telegram.groupAllowFrom`, `commands.allowFrom.telegram`, and `commands.ownerAllowFrom`. If the repro confirms the report, fix the Telegram native preauth/owner handoff or fresh config resolution so a properly owner-allowlisted sender can execute `/restart` without weakening non-owner command gating.
