## Summary

Keep open. Current `main` still has the reported Docker sandbox bug: when `openclaw-sandbox:bookworm-slim` is missing, `ensureDockerImage()` silently pulls and retags plain `debian:bookworm-slim`, even though the intended default image contract includes `python3`. The pinned mutation helper still fails when Python is missing, and `fs-bridge` still has no Docker-specific rewrite/recovery path for that failure. The docs on `main` also do not yet include the incompatible-image troubleshooting this PR adds.
