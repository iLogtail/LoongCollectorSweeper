## Summary

Keep #69443 open. Current main has related fixes for subagent delivery context and account-scoped channel routing, but the reported RPC shape still appears unsupported: `agent` routing is selected from `sessionKey` or `agentId`, while `to` is treated as an outbound delivery target. I did not find evidence that a WeChat session key placed in `to` is canonicalized into the target session, so this should not be auto-closed.

## Best Possible Solution

Keep the issue open for a targeted maintainer fix or explicit product decision. The likely fix is either: detect a valid `agent:*` session key in `agent` RPC `to` when `sessionKey` is omitted and canonicalize it to `sessionKey`, then use the stored delivery route; or reject that shape with `INVALID_REQUEST` and tell callers to pass `sessionKey` separately. Add a regression test using an `openclaw-weixin`-style direct session key so callbacks cannot silently fall back to `agent:main:main`.
