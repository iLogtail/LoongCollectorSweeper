## Summary

Keep open. Current main still lacks the PR's core resource-cleanup changes in `downloadToFile`: redirect/error responses are not drained with `res.resume()`, the size-limit path destroys only the request, there is no response-level error handler or shared settle guard, and the tests do not cover the proposed cleanup cases. The PR is stale against newer main behavior, but it remains a plausible focused bug fix with targeted tests rather than an obsolete refactor-only change.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best path is to rebase the resource-cleanup changes onto current `src/media/store.ts`, preserve current `maxBytes` and redirect-header behavior, keep the existing pipeline temp-file removal, and add focused tests for response draining/error handling plus leftover `.tmp` cleanup.
