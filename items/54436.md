## Summary

Keep open. Current main has not implemented the PR's core resource-cleanup changes in `downloadToFile`: redirect/error responses are still rejected without `res.resume()`, the size-limit path still destroys only the request, there is no response-level error handler, and promise settlement is still unguarded. The PR is stale against newer main changes, but it remains a plausible focused bug fix with tests rather than an obsolete cleanup-only PR.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best path is to rebase the resource-cleanup changes onto current `src/media/store.ts`, preserve current `maxBytes` and redirect-header behavior, keep the existing pipeline temp-file removal, and add focused tests for response draining/error handling and leftover `.tmp` cleanup.
