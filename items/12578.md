## Summary

Keep #12578 open. Current main still does not implement a persistent `agents.defaults` usage-footer default: the config schema/types have no `usageDefault`, `responseUsageDefault`, or `agents.defaults.responseUsage` key; the reply runner resolves usage footers from session `responseUsage` only; `/usage` still mutates that per-session field; and docs still describe the footer setting as per-session. The related PRs #48635 and #69618 are relevant implementation attempts, but the provided GitHub context shows they are still open and unmerged, so they are not current-main behavior.

## Best Possible Solution

Keep #12578 open until a focused implementation lands. The likely maintainer path is to choose one public key name, such as `agents.defaults.responseUsage` or `agents.defaults.responseUsageDefault`, add it to config schema/types/generated help/docs, apply it only when a session has no explicit `responseUsage` override, preserve `/usage off|tokens|full` as the per-session override, provide a clear-to-inherit path, and add regression tests for inherited default, explicit `off`, and session override behavior.
