## Summary

Keep #38505 open. Current main still registers only the Discord message-create inbound listener for conversation triggers; there is no messageUpdate listener, content-change filter, edited-message re-dispatch path, or edit metadata path. The prior ClawSweeper comment and related open #53654 support the same conclusion, but #53654 is broader and does not make this focused edit-reprocess issue obsolete.

## Best Possible Solution

Implement the missing behavior in the bundled Discord plugin: add a Carbon MessageUpdateListener, ignore non-content edits, reuse the existing Discord preflight/routing/inbound worker path with the edited message as authoritative, and add focused tests for mention-added edits, DMs or active threads, duplicate/update replay behavior, and non-content update suppression. Coordinate with open #53654 so the edit half is not implemented twice.
