## Summary

Keep open. Current main still does not implement the PR's central behavior: chat history sanitization preserves only `usage.cost.total`, and the chat UI metadata extractor reads cost only from top-level `m.cost`, not nested `m.usage.cost`; there is also no savings badge rendering. The PR is a plausible core UI/gateway compatibility fix for provider plugins emitting OpenAI-style `usage.cost`, not a cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to adapt the change on current main, add focused gateway sanitizer and grouped-render tests for nested `usage.cost`, and address the multi-message savings aggregation edge case before merge. Related cost exposure issues such as #7006 and #9016 remain open, but they do not supersede this narrow UI/gateway implementation.
