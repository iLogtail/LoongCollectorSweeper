## Summary

Keep #12163 open. Current main still has only partial OpenAI TTS speed support: configured speed and Talk overrides reach the OpenAI request body, but the reported `[[tts:speed=1.5]]` directive path still is not parsed by the OpenAI speech provider.

## Best Possible Solution

Keep the issue open and implement the missing OpenAI directive handling in `extensions/openai/speech-provider.ts`: parse `speed` only when `policy.allowVoiceSettings` is enabled, validate OpenAI's documented `0.25..4.0` range, return `{ speed }`, and add focused tests proving both `[[tts:speed=1.5]]` with OpenAI active and `[[tts:provider=openai speed=1.5]]` reach the OpenAI request body. Preserve the existing config, Talk, and request-body plumbing.

## Evidence
