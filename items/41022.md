## Summary

Keep open. Current main still has the old CLI watchdog defaults: fresh min 180s, fresh max 600s, and resume max 180s. The resolver still consumes those defaults and clamps computed no-output timeouts through maxMs before passing them to the CLI process supervisor, so the PR is not obsolete. The linked issue #40982 remains the same live product request.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer-owned patch that raises the shipped defaults and updates the existing watchdog/default assertions. A broader follow-up can decide whether maxMs should be nullable or disableable, but the current PR's core default-cap change is still live and useful.
