## Summary

Keep open. Current main has not implemented this PR’s central behavior: `shouldSkipRespawnForArgv` still skips only help/version invocations, and the existing test suite still asserts `openclaw status` keeps the respawn path. Related startup-performance work appears adjacent rather than superseding this respawn-policy change.

## Best Possible Solution

Keep #54085 open for maintainer review. The likely implementation path is still a scoped update to `src/cli/respawn-policy.ts` with dedicated command/root-option tests, plus an explicit treatment of the existing `NODE_EXTRA_CA_CERTS` behavior in `buildCliRespawnPlan` so the performance win does not accidentally drop startup TLS configuration for commands that need it.
