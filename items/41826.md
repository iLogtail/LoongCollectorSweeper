## Summary

Current main does not implement the requested custom TUI client display name. The TUI CLI exposes no --client-name/--username option, TuiOptions has no field for it, and GatewayChatClient still passes a hardcoded clientDisplayName of "openclaw-tui" when connecting to the Gateway. This is a concrete core TUI/Gateway UX request, not a ClawHub/plugin cleanup close.
