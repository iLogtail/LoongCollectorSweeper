## Summary

Keep open. Current `main` still does not export `stringEnum` or `optionalStringEnum` from the root `openclaw/plugin-sdk` barrel, and `main` now contains explicit docs and contract tests that keep that root runtime surface intentionally tiny while steering plugin authors to narrow subpaths such as `openclaw/plugin-sdk/core` and `openclaw/plugin-sdk/channel-actions`. The PR discussion also includes a concrete external reproduction (`memory-lancedb-pro@1.0.32` on OpenClaw 2026.4.14/2026.4.15), so this is not a safe `cannot_reproduce` or `implemented_on_main` close. It needs maintainer judgment on compatibility policy, not conservative auto-close.
