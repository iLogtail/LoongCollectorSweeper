## Summary

Keep PR #41807 open. Current main has inline `agents.defaults.systemPromptOverride` and CLI-backend prompt-file handoff plumbing, but it still does not implement the requested operator config key `agents.defaults.systemPromptFile` that reads an external file and prepends it for all sessions. The PR remains a plausible core-agent config feature, not an implemented, duplicate, or ClawHub-only item.

## Best Possible Solution

Keep the PR open and treat it as the active implementation candidate for file-based global system prompt injection. If maintainers accept the feature, the best path is to rebase it onto current main, add `agents.defaults.systemPromptFile` to the exported type, zod schema, generated schema/help/docs, inject it through both CLI and embedded prompt assembly paths, and keep focused tests for missing files, whitespace, precedence with `systemPromptOverride`, and no duplicate injection.
