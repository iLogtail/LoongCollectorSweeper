## Summary

Keep open. Current main still does not implement the requested Volta-specific normalization: gateway install resolves the selected Node path through `resolveStableNodePath`, but that helper only rewrites Homebrew Cellar paths and returns non-Cellar paths unchanged, so a Volta `tools/image/node/<version>/bin/node` executable would remain version-pinned instead of becoming the stable Volta shim. The related PR #52252 remains the narrower implementation track, but it is not merged and the issue should stay open until that work lands or is superseded.

## Best Possible Solution

Keep #52184 open and either revise #52252 or replace it with a maintainer-approved fix. The implementation should extend the runtime node-path resolver to rewrite Volta image-node paths to a stable shim when safe, include default and custom Volta home coverage, and add focused regression tests in `src/infra/stable-node-path.test.ts` and `src/daemon/runtime-paths.test.ts`.

## Evidence
