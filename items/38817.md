## Summary

Keep #38817 open. Current main still strips unsupported tool-schema keywords only when resolved model compat declares them, and the bundled Volcengine/Kimi provider still does not add that compat or a provider-owned schema normalization hook. The related fix attempts #38883 and #46887 were closed unmerged, so there is no canonical merged fix to close against.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Add unsupportedToolSchemaKeywords to the affected resolved model compat, or have the Volcengine plugin contribute it through normalizeResolvedModel, contributeResolvedModelCompat, or normalizeToolSchemas. Add focused regression coverage for volcengine, volcengine-plan/Ark, and Kimi model IDs, plus a negative case proving unaffected providers keep validation keywords.
