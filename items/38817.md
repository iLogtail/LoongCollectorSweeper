## Summary

Keep open. Current main still does not implement automatic unsupported tool-schema keyword stripping for bundled Volcengine/Kimi models. The generic schema normalizer strips `minLength`/`maxLength`/`minItems` only when resolved model compat declares unsupported keywords, and the Volcengine/Kimi catalog/provider does not attach that compat or a provider hook. The linked fix PRs #38883 and #46887 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Implement this as provider-owned compatibility, not a hard-coded core branch: add `unsupportedToolSchemaKeywords` to the affected bundled Volcengine/Kimi model definitions or add a Volcengine provider hook (`normalizeResolvedModel`, `contributeResolvedModelCompat`, or `normalizeToolSchemas`) that applies the existing shared unsupported-keyword stripping. Add focused regression coverage for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs while proving unaffected providers keep validation keywords.
