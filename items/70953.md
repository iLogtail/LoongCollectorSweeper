## Summary

Keep #70953 open. Current main still treats foreground host exec approval as a two-phase asynchronous flow: the exec tool returns `approval-pending` after registration, while approval resolution and command execution continue in a background follow-up. Docs and tests explicitly codify that immediate-return behavior, so the requested turn-pause semantics are not implemented on main.

## Best Possible Solution

Implement foreground host exec approvals as an awaited agent/tool execution path: after registration, await `exec.approval.waitDecision` with run cancellation and bounded timeout, then return the approved command result or deterministic denial/timeout result from the original exec tool call. Keep async follow-up behavior only for explicit background mode or deliberately asynchronous delivery paths. Add regression coverage proving no later tool calls or assistant reply are emitted while a foreground exec approval is pending.
