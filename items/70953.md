## Summary

Keep #70953 open. Current main still implements and documents exec approvals as an immediate `approval-pending` tool result with detached follow-up execution after approval, so the requested hard pause of the agent turn is not implemented. The provided related items are adjacent exec-approval cleanup reports, but none is a canonical duplicate for this agent-turn pause behavior.

## Best Possible Solution

Keep the issue open and implement the requested behavior in the foreground exec approval path by suspending the active agent turn, or otherwise gating subsequent tool calls, until `exec.approval.waitDecision` resolves or times out. The change likely belongs around the exec tool/embedded runner contract rather than only gateway approval delivery, with regression tests proving that a second tool call cannot run while a foreground exec approval is pending and docs updated if the immediate-return contract changes.
