## Summary

Keep open. Current main partially addresses the broader private-network story through the SearXNG web_search plugin and model-provider request allowPrivateNetwork support, but the central request is not fully implemented: generic web_fetch still has no private-network opt-in, and Gateway URL input fetching still hard-codes private-network blocking for input_image/input_file. The related PR #53532 was closed unmerged, while the later #62147 provider-request work covers a different surface.

## Best Possible Solution

Keep the issue open and scope the remaining product/security decision around a narrow, explicit private-network opt-in for generic web_fetch and Gateway URL inputs. The implementation should extend the existing SSRF policy/config schema, preserve default blocking, add regression tests for localhost/RFC1918 denied-by-default and allowed-with-explicit-opt-in paths, and update the public web_fetch/Gateway docs. SearXNG and model-provider allowPrivateNetwork can be referenced as related shipped precedent, not as complete resolution.
