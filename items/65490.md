## Summary

Keep #65490 open. Current main has shipped meaningful pieces of the production-discipline story, including fallback notices/status state, live-session fallback fixes, model status/probe docs, and cron CLI/docs improvements. The umbrella issue still contains concrete unimplemented or only partially implemented requests: startup does not fail or block turns on an unreachable primary model, cron add still lacks a dry-run path and allows announce delivery without an explicit target, and the agent-facing diagnostic surface still does not cover the full requested last-error/fallback/runtime state story.

## Best Possible Solution

Keep open, or have a maintainer split it into focused follow-ups: a startup provider/model reachability gate with clear operator-visible failure semantics; an agent-safe runtime diagnostics surface that includes active model, selected model, fallback reason, and last actionable error; and cron add validation/dry-run that previews the resolved delivery route before persistence. Keep the shipped fallback notice/live-switch fixes as evidence of progress, and link narrower canonical issues where maintainers choose to track the remaining work.
