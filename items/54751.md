## Summary

Keep open. Current `main` still performs the package-manager self-update while the gateway process is running, then schedules restart only after a successful update. I found no shipped stop-update-restart flow for the Control UI `Update now` path, so the Windows `EBUSY` report remains plausible and not implemented on `main`.

## Evidence
