## Summary

Keep #20950 open. Current main still has a single file-backed cron definitions store (`cron.store` -> `jobs.json`) plus a derived `jobs-state.json` sidecar, but no `cron.storeDir`, `jobs.d`, or per-job JSON directory loader. Open PR #41206 is the relevant unmerged implementation track, and closed #53581 only shipped the complementary definitions/runtime-state split.

## Best Possible Solution

Keep #20950 open until directory-backed cron definitions are implemented or maintainers explicitly decide against the storage shape. The likely path is core cron storage/config work: an additive `cron.storeDir` or store-backend contract, deterministic per-file loading, compatibility with existing `cron.store`, state sidecar behavior, doctor/config schema/help/docs updates, and focused tests. PR #41206 is the current implementation candidate to evaluate or refresh.
