## Summary

Keep #20950 open. Current main still has only the aggregate `cron.store` / `jobs.json` cron definition store plus `jobs-state.json`; it does not support `cron.storeDir`, `jobs.d`, or one JSON file per cron job. The related duplicate chain currently treats #20950 as the canonical open issue, while #41206 is only an open, unmerged implementation candidate.

## Best Possible Solution

Keep #20950 open as the canonical issue until maintainers land #41206 or a replacement implementation. The implementation should add an additive, documented directory-backed cron definitions store, deterministic per-file loading, compatibility with existing `cron.store`, preservation of the current `jobs-state.json` runtime sidecar behavior, config schema/help/docs/doctor updates, and focused load/save/service tests.
