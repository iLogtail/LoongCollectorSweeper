## Summary

Keep open. Current main still does not implement per-agent compaction overrides: `agents.list[]` has no `compaction` type or strict schema field, resolved agent config does not merge or expose per-agent compaction, and runtime compaction/memory-flush paths still read `agents.defaults.compaction` only. The related implementation PRs #14598 and #19329 are useful prior art but were closed unmerged, and the related ClawSweeper reports still treat this as unfinished core config/runtime work.

## Best Possible Solution

Keep #13736 open as the root feature request unless maintainers choose a newer canonical item such as #69988. The implementation should add optional `agents.list[].compaction` typing and strict schema validation, reuse or extract the defaults compaction schema, resolve effective per-agent compaction by deep-merging over `agents.defaults.compaction` including nested `memoryFlush`, thread the resolved agent id/config through reserve-token, safeguard, timeout/model, memory-flush, and compaction runtime paths, then add focused schema/resolver/runtime tests and config docs. Closed PRs #14598 and #19329 should be treated as prior implementation work rather than closure evidence.
