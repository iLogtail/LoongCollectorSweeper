## Summary

Current main still does not implement tool policy audit logging with rule attribution. The PR is a plausible implementation for open issue #55801, but it should stay open for maintainer review because the sandbox audit log placement proposed in the PR would run from an error-formatting helper that current main calls from multiple render paths, so duplicate audit lines are a real risk.

## Best Possible Solution

Keep the PR open and ask for a focused revision: retain the pipeline removal audit with deterministic tool names, but move sandbox audit emission to a single enforcement point or dedupe it per run/tool/policy decision. Add tests that prove one audit entry per blocked sandbox decision and preserve the existing pipeline tests, then close #55801 when the shipped behavior covers the requested operator diagnostics.
