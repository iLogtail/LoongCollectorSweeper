## Summary

Current main already fixes the reported packaged `session_status` crash by keeping the tool on a dist-stable lazy local runtime shim, shipped in v2026.4.24 via the #65807 changelog entry. PR #66671's static-import approach is no longer needed and would remove the lazy boundary that current main preserves.

## Best Possible Solution

Close #66671 as implemented on main and keep the shipped #65807 design: `session_status` should continue using the local lazy runtime shim, preserving deferred loading while avoiding source-tree-relative runtime imports. Any future regression should be handled with a focused test or build-output assertion around the local shim boundary.
