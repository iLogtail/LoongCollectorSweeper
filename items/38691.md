## Summary

Keep #38691 open. Current main and latest release still hard-stop the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, with no OEM `voice_recognition_service` lookup or targeted recognizer creation. The PR is a plausible focused fix with real device verification, but it is not implemented on main and still needs maintainer review/revision for the unresolved retry-loop concern and current file path drift.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The likely path is to rebase onto the current `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, add a generic OEM recognizer fallback using the configured secure `voice_recognition_service` component and `SpeechRecognizer.createSpeechRecognizer(context, component)`, and preserve graceful disable behavior when no backend exists, especially for `ERROR_CLIENT`/unusable recognizers. Add focused Android coverage if feasible, or document manual verification on a Chinese OEM device plus a no-recognizer safety check.
