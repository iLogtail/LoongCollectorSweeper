## Summary

Keep #71586 open. It is a fresh, concrete macOS companion app hang report against v2026.4.22 with a specific scrollbar-thumb drag reproduction. Current main still uses the same native SwiftUI WebChat scroll path as the latest v2026.4.24 release, I found no source/changelog/test evidence of a targeted fix, and the older related #42527 closure does not cover this newer native-scrollbar regression.

## Best Possible Solution

Keep the issue open for a maintainer or macOS-capable tester to reproduce on macOS 26.4.1 or nearby versions using `dist/OpenClaw.app/Contents/MacOS/OpenClaw --webchat`, capture `WebChatSwiftUI` logs or a hang sample, then fix the native `OpenClawChatView` scroll behavior. The likely implementation path is to remove or debounce any `.scrollPosition` feedback during manual scrollbar dragging, or replace the message list with an AppKit-backed scroll view that can distinguish user thumb-dragging from auto-scroll updates. Add focused regression coverage where feasible, plus a manual verification note if native scrollbar dragging cannot be automated.
