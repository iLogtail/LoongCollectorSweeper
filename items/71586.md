## Summary

Keep open. This is a recent, specific macOS companion bug report for scrollbar-thumb dragging, and current main still routes the chat window through the same SwiftUI OpenClawChatView ScrollView/LazyVStack + scrollPosition stack. I found smoke coverage for opening/closing the macOS WebChat window, but no implementation, test, changelog, or related-item evidence proving scrollbar-thumb dragging has been fixed or made obsolete.

## Best Possible Solution

Keep the issue open and investigate the macOS native WebChat scroll path. Reproduce on macOS with a large chat history, especially while dragging the scrollbar thumb, then determine whether the SwiftUI scrollPosition bottom-pinning/animated auto-follow logic fights manual scrollbar dragging. A conservative fix likely gates auto-scroll while the user is actively scrolling/dragging, or introduces a macOS-specific NSScrollView-backed message list if SwiftUI remains unstable. Add focused regression coverage where feasible plus manual verification notes for scrollbar-thumb dragging.
