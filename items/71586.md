## Summary

Keep open. This is a fresh, concrete macOS companion-app bug report against v2026.4.22, and current main still has the relevant native SwiftUI chat scroll surface with scroll-position-driven auto-scroll state. I did not find a current-main scrollbar-thumb fix, release-note evidence, or regression coverage that would justify closing it. The related #42527 report was a broader older scroll/crash cleanup after the native SwiftUI rewrite; #71586 targets that current surface more specifically, so it should not be auto-closed as implemented or duplicate.

## Best Possible Solution

Keep the issue open and investigate the native shared `OpenClawChatView` scroll mechanics on macOS. The likely fix belongs in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift`, either by removing/debouncing scrollbar-driven `.scrollPosition` feedback on macOS or by using an AppKit-backed scroll view/bottom-sentinel approach that does not fight manual scrollbar dragging. Add focused macOS UI coverage where feasible, or at least a documented manual verification with a long chat history and scrollbar-thumb drag.
