## Summary

Keep open. Current main still treats this as a real cross-agent/session-visibility surface, not an already-implemented fix or a duplicate. Telegram group messages produce per-agent group session keys, while `sessions_list` and `sessions_send` remain gated by generic session visibility plus `tools.agentToAgent`; the related #59933 is open and explicitly says it is distinct from this cross-agent visibility report.

## Best Possible Solution

Keep #53783 open and reproduce with two configured agents in one Telegram group under explicit `tools.sessions.visibility: "all"` and enabled/allowlisted `tools.agentToAgent`. If the asymmetry reproduces, add a regression test for `agent:<a|b>:telegram:group:<id>` list/resolve/send symmetry and fix the gateway/session visibility path. If maintainers decide same-group cross-agent access should remain opt-in, improve the error/docs so it returns clear config guidance instead of one-way `No session found`. Track #59933 separately for the same-agent group-session listing gap.
