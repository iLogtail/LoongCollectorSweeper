## Summary

Current main still has the requested installer maintainability work outstanding: `scripts/install.sh` remains a 2,770-line single script with downloader, UI/gum, platform detection, build-tool setup, npm/git install logic, diagnostics, and the main flow in one file. No `scripts/install/` module tree or canonical duplicate/superseding item was available in the provided issue context or local checkout, so this should not be auto-closed.

## Best Possible Solution

Keep the issue open for maintainer judgment. The most conservative implementation path is to introduce tested installer modules under `scripts/install/` while preserving `scripts/install.sh` as the published self-contained entry point, likely generated or bundled from those modules. The refactor should move downloader, UI/gum, platform/package-manager, npm, git, and build-tool logic behind shell-testable seams and keep `docs/install/installer.md`, package scripts, and installer smoke workflows aligned.
