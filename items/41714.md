## Summary

Keep open. Current main still drops Discord upload MIME types and still optimizes WebP through the shared image path, so the user-visible bug is not implemented on main. The PR is also not safe to close as obsolete because it contains useful Discord MIME work, but its broad shared WebP bypass has unresolved review concerns about non-Discord regressions and validation/security tradeoffs.

## Best Possible Solution

Keep the PR open, but ask for a narrower revision: preserve media.contentType in the Discord Blob construction paths with focused tests, and handle animated/WebP preservation through a Discord-specific/raw media path or a validated/clamped WebP path rather than an unconditional shared loadWebMedia bypass.
