## Summary

Keep open. Current `main` does not implement the Discord final-reply sanitizer requested by PR #60982. Discord partial preview updates strip reasoning tags, and shared assistant-visible sanitizers exist, but the Discord final preview-edit path still derives final edit content from `payload.text` without reasoning/scaffolding stripping, and `deliverDiscordReply` forwards `params.replies` to shared outbound unchanged. The PR is also still useful but not clearly merge-ready because review discussion on the PR identified a remaining media-final loss risk at the latest head.

## Best Possible Solution

Keep the item open and land a scoped Discord final-boundary fix. The best direction is to reuse the existing shared assistant-visible/scaffolding sanitizer for both Discord preview-final edits and normal `deliverDiscordReply` payload text before table conversion/chunking/send, with regression tests for `<thinking>`, `<relevant-memories>`, code-fenced literals, sanitized-empty text-only finals, and media finals whose caption sanitizes to empty.
