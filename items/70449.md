## Summary

Current main already solves the Gemini tool-call replay problem: the direct Google transport captures tool-call thought signatures, replays them on same-provider/model history, and the OpenAI-compatible Google path captures/re-emits `extra_content.google.thought_signature`. The shipped implementation uses the part-level `thoughtSignature` shape for direct Gemini content, which matches the current `@google/genai` Part/FunctionCall types rather than adding `functionCall.thought_signature`.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped part-level direct Google replay implementation and the OpenAI-compatible `extra_content.google.thought_signature` path; reopen only if a fresh live repro against current main shows Google rejecting the current direct Gemini part-level `thoughtSignature` replay shape.
