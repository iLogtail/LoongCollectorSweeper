## Summary

Keep PR #62653 open. Current main still rejects document-kind media in both targeted image-load paths, and the related reports cover adjacent media URI/session-send issues rather than this exact document-kind compatibility gap. The PR is not merge-ready as-is, but it is a plausible focused agents fix rather than an auto-close candidate.

## Best Possible Solution

Keep the PR open and request a focused rework: remove `patch-image-tool.diff`, rebase onto current main, align MIME fallback behavior, add regression coverage in the current image-tool and native prompt-image tests, and accept document-classified media only when the content is actually image-compatible rather than accepting arbitrary documents.
