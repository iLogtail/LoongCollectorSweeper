## Summary

Keep PR #62653 open. Current main has not implemented the requested document-kind media acceptance in the two image-load paths: both the explicit image tool and native prompt image loader still reject any `loadWebMedia` result whose `kind` is not exactly `image`. The PR is not ready to merge as-is because the diff includes a patch artifact and the proposed broad `document` acceptance can mislabel real documents as images, but the underlying surface is not obsolete or covered by a canonical shipped fix.

## Best Possible Solution

Keep the PR open but request a focused rework: drop `patch-image-tool.diff`, rebase onto current main, add regression coverage in `src/agents/tools/image-tool.test.ts` and `src/agents/pi-embedded-runner/run/images.test.ts`, and implement a narrower helper that only accepts document-classified media when the content is actually image-compatible instead of blindly coercing all documents to image MIME types.
