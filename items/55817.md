## Summary

Keep PR #55817 open. Current main still persists a fresh sessionId/sessionFile during rollover before creating the corresponding transcript file, and the only current header-creation path runs later during transcript append/mirroring. The PR addresses a real remaining gap, though the unresolved atomic-create review note should be handled before merge.

## Best Possible Solution

Keep the PR open and either update it or replace it with an equivalent main-branch fix: create the fresh transcript header before publishing the rotated sessionFile in sessions.json, do it atomically, and add regression tests for explicit /reset, stale/daily rollover, and ideally the concurrent EEXIST path.
