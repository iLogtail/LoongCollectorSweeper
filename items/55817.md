## Summary

Current `main` does not implement this rollover fix. `initSessionState()` still persists the new `sessionId`/`sessionFile` through `resolveAndPersistSessionFile()` before any transcript file is created, and transcript header creation still happens later in the append path. The PR remains useful, but it should stay open because the bug is still present on `main` and the proposed patch also has an unresolved atomic-create concern from review.

## Evidence

## Risks / Open Questions

- The open PR's reviewed patch uses a check-then-write header bootstrap; as noted in review, that should be made atomic (`wx`/create-only semantics) to avoid concurrent rollover/file-creation clobbering.
