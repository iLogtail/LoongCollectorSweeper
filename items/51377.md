## Summary

Keep open. Current main still does not implement first-class Excel or generic binary uploads for Gateway/OpenResponses `input_file`: defaults exclude `.xlsx`/`.xls`, unsupported MIME types are rejected, accepted files are decoded into prompt context rather than persisted with a stable binary receipt, and the WebChat attachment path remains image-only. The related WebChat work overlaps only part of the issue; it does not cover the broader `input_file`/durable binary ingest request.

## Best Possible Solution

Keep the issue open and implement a core Gateway/OpenResponses upload path for allowlisted Excel and generic binary files: explicit MIME/size limits, fail-closed errors, durable workspace/media persistence with name/sha256/mime metadata, and a stable receipt/path that tools and skills can consume. The WebChat image-only work can be tracked through #69447/#70947, but this issue should remain for the `input_file` and binary artifact contract.
