## Summary

Keep open. Current main still does not provide first-class Excel or generic binary `input_file` uploads: the default OpenResponses file allowlist excludes `.xlsx`/`.xls`, unsupported file MIME types are rejected, decoded `input_file` content is injected into prompt context rather than persisted with a stable path/receipt, and the chat attachment path remains image-oriented. The issue discussion specifically asks for workspace-persisted Excel/binary artifacts with metadata, and that central Gateway/API capability remains missing rather than superseded by the related items.

## Evidence
