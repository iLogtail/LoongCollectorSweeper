## Summary

Keep open. Current `main` still has the strict `bundledById.npmSpec !== npmSpec` rejection in `resolveBundledInstallPlanForCatalogEntry`, so the proposed change is not already implemented. The PR body's specific `openclaw plugins install acpx` repro does not match the current direct CLI path, but the patch still appears relevant for onboarding/channel-setup flows that route catalog `install.npmSpec` values through this helper.
