## Summary

Keep open. Current main partially improves ACP spawned-run tracking, but the issue's main hook/headless persistent-session blocker remains: `sessions_spawn(runtime:"acp", mode:"session")` still hard-requires `thread:true`, the accepted result still exposes only OpenClaw `childSessionKey`/`runId` rather than the underlying ACP/harness session id, and `/hooks/agent` still runs through the cron isolated embedded-agent path rather than routing an ACP session key to `acpManager.runTurn`.

## Best Possible Solution

Keep the issue open, likely narrow it to the remaining core ACP/hook work: add an explicit headless persistent ACP mode or otherwise allow `mode:"session"` without thread binding under a safe contract, return the underlying ACP/harness session id needed for resume, and teach `/hooks/agent` or a session tool to route known ACP session keys through `acpManager.runTurn`. Preserve the current background task tracking and add focused tests for hook-triggered ACP session spawning, resume identity exposure, and no-duplicate delivery behavior.
