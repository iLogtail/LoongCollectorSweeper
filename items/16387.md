## Summary

Keep #16387 open. Current main still treats configured provider headers as static strings, the Fireworks plugin does not implement `x-session-affinity`, and the provider-owned turn-state hook is not a generic `models.providers.*.headers` templating solution for custom providers. The supplied discussion and related closed PR attempts support the same conclusion: this remains real product/API work, not a duplicate or shipped fix.

## Best Possible Solution

Keep the issue open. The best maintainer path is a narrow, documented dynamic-value resolver for `models.providers.*.headers` and likely `models.providers.*.request.headers` at request dispatch, or an explicit product decision to solve only provider-owned cases through `resolveTransportTurnState`. If config templating is chosen, cover at least session identity, agent identity, and turn/run identity, with tests for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, and OpenAI-compatible transports.
