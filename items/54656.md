## Summary

Keep PR #54656 open. Current main still lacks the Hetzner-specific access clarifications the PR is trying to add: the Hetzner guide sets `OPENCLAW_GATEWAY_BIND=lan`, shows only an `openclaw-gateway` Compose service, then tells users to paste a shared secret without documenting the Docker CLI service, `dashboard --no-open`, or device pairing approval flow. The original allowed-origins crash-loop concern is partly mitigated by current startup seeding, so the patch should be refreshed rather than auto-closed.

## Best Possible Solution

Keep the PR open and refresh it against current main: preserve the useful Hetzner Docker access clarifications (`openclaw-cli` service, `dashboard --no-open`, and `devices list/approve`), then adjust the allowed-origins wording to match current startup seeding and explicit-origin security guidance. If the GCP guide has the same CLI-service gap, handle that as a small follow-up or include it with maintainer approval.
