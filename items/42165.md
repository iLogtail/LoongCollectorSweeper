## Summary

Keep open. Current main still unconditionally promotes `tools.fs` config into `read`, `write`, and `edit` profile-stage allowances, so `tools.profile: "minimal"` can still expand beyond the documented `session_status` surface when an `fs` block is present. Related PR #42308 was closed unmerged and #43061 is still open, so the fix has not shipped on main.

## Best Possible Solution

Keep #42165 open until main either lands a focused fix that skips implicit `tools.fs`/`tools.exec` expansion for `profile: "minimal"` while preserving explicit `alsoAllow`, or maintainers intentionally update docs and tests to declare `tools.fs` blocks as capability opt-ins for all restrictive profiles. The likely code path is `src/agents/pi-tools.policy.ts`, with regression tests in `src/agents/pi-tools.policy.test.ts` and an end-to-end agent config test in `src/agents/pi-tools-agent-config.test.ts`.
