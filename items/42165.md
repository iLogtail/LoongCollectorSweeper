## Summary

Keep open. Current main still has the reported widening path: `tools.profile="minimal"` is documented/cataloged as `session_status` only, but `resolveImplicitProfileAlsoAllow()` still adds `read`, `write`, and `edit` whenever a `tools.fs` block is present, and that list is merged into the profile allowlist before filtering. The related PR #42308 positively identified the same root cause and added regression coverage, but the provided GitHub context says it closed unmerged; #43061 is still open, so the issue is not implemented on main or superseded by a landed fix.
