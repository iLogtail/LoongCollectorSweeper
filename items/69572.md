## Summary

Keep #69572 open. Current main still implements Feishu reply typing as a `Typing` emoji message reaction, and the Feishu docs/config describe the feature as typing reaction calls rather than a native Feishu/Lark typing status. The related #8722 cleanup confirms reaction-based typing was implemented, but this issue reports that implementation as the bug.

## Best Possible Solution

Keep the issue open. The Feishu plugin should either switch the reply typing lifecycle to a verified native Feishu/Lark typing API while preserving the existing `typingIndicator` config, keepalive/backoff behavior, stale-message suppression, and tests, or document/product-accept that Feishu only supports the current reaction or streaming-card fallback and update the issue accordingly.
