## Summary

Keep open. Current main still does not expose `channels.slack.logLevel`, and #71550 is a recent active PR tied directly to open RFE #71531 rather than obsolete cleanup. However, the fetched PR patch likely needs one more correctness pass: OpenClaw currently creates a custom `SocketModeReceiver`, so forwarding `logLevel` only to Bolt `App` may not reach the socket-mode `SlackWebSocket` logger that emits the pong-timeout WARN.

## Best Possible Solution

Keep #71550 open as the canonical implementation path for #71531. Finish it by adding a lowercase `channels.slack.logLevel` config contract across schema, manual config types, docs/generated config reference, and tests; then forward the configured level to the actual socket-mode receiver/logger as well as Bolt `App` if needed. Add regression coverage that asserts configured `error` reaches `SocketModeReceiver`/socket-mode and that omitted config preserves the default behavior, then merge after the normal changed gate/CI is green.
