## Summary

Current main does not implement the requested fallback chain for compaction or a Lossless Claw summary-model fallback. Normal agent runs have model fallback support, but compaction config still exposes only a single `agents.defaults.compaction.model`, and the compaction runtime resolves one provider/model target without using `runWithModelFallback`. Lossless Claw is documented as an external context-engine plugin with plugin-specific config, and no `summaryModelFallbacks`/summary-model fallback surface exists in this repo.

## Best Possible Solution

Keep the issue open. Implement a compaction-specific fallback chain by extending the config type/schema/help/docs for `agents.defaults.compaction`, then route built-in compaction summarization through the same failover classification/candidate machinery used by normal agent runs. For Lossless Claw, either add a plugin-owned `summaryModelFallbacks` implementation or expose a small shared SDK/runtime helper so context-engine plugins can use the same ordered fallback semantics without copying core internals.

## Risks / Open Questions

- The Lossless Claw half likely needs coordination with the external `@martian-engineering/lossless-claw` plugin or a shared context-engine/runtime helper, not just a core config field.
- Adding compaction fallback should preserve existing `compaction.model` behavior and avoid accidentally applying chat-session fallback semantics to plugin-owned context engines without explicit API design.
