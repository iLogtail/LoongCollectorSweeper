## Summary

Keep open. Current `main` already fixed the linked directive-tag indentation bug (#41699) through the directive parser, but this PR’s narrower duplicate-block sanitizer behavior is still present in the moved helper: duplicate collapsing still pushes `block.trim()`, which can strip indentation when duplicates are actually collapsed. The PR should be rebased and tightened rather than closed as obsolete.

## Best Possible Solution

Rebase the functional change onto current `main` in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`, preserve the original block on the output path while keeping normalized comparison for duplicate detection, add a focused `sanitizeUserFacingText()` regression test for duplicated indented/fenced code blocks, and drop the unrelated `.gitignore` change.
