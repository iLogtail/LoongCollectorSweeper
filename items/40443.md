## Summary

Keep open. Current `main` still does not implement the PR's explicit-reload semantics: config reload paths call `loadConfig(...)` without any discard option, and `applyConfigSnapshot(...)` still preserves dirty local state instead of providing a way for explicit reloads to replace it. The related gateway hash-conflict contract is still active, and the regression coverage described in the PR is not present on `main`. The review discussion also shows the author addressed earlier bot feedback, so this remains substantive pending work rather than something already superseded.
