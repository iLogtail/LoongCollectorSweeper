## Summary

Keep open. Current main still contains the reported Telegram inbound mismatch: the body builder can send `<media:image>` to the agent without deriving that tag from the resolved media content type, and the related PR cited in the issue timeline was closed unmerged rather than shipped.

## Best Possible Solution

Keep #69793 open and fix it in the Telegram plugin. The best path is to add a small, tested media-placeholder helper that derives the body tag from resolved `allMedia[].contentType`/media kind, preserves audio transcript framing, keeps a Telegram message-shape fallback only when no attachment was resolved, and adds regression coverage for `application/octet-stream`/document, mixed media, multiple images, and normal photo cases. If the same hardcoded pattern is confirmed in other bundled channel plugins, migrate them through the same helper or equivalent contract tests in a follow-up rather than closing this Telegram-specific report.
