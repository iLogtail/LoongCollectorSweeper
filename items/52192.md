## Summary

Keep open. Current `main` still has the duplicate-send branch the PR changes in `extensions/feishu/src/outbound.ts`, but the shipped core call paths I inspected only invoke channel `sendMedia` when a media URL is present. That leaves reachability ambiguous: the PR is still plausible cleanup, and the evidence is not strong enough to auto-close it as obsolete or already implemented.

## Evidence

- **Feishu adapter still has the duplicate branch:** `extensions/feishu/src/outbound.ts` sends text before checking `mediaUrl`, then falls back to another text send when `mediaUrl` is absent. A direct `sendMedia({ text, mediaUrl: undefined })` call would send twice.
  - file: [extensions/feishu/src/outbound.ts:254](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/extensions/feishu/src/outbound.ts#L254)

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **CLI/runtime text-only sends do not call sendMedia:** `src/cli/send-runtime/channel-outbound-send.ts` computes `hasMedia = Boolean(opts.mediaUrl)` and routes text-only sends to `sendText`, not `sendMedia`.
  - file: [src/cli/send-runtime/channel-outbound-send.ts:60](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/cli/send-runtime/channel-outbound-send.ts#L60)
