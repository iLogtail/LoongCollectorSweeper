## Summary

本 PR 旨在为火山引擎环境扩展 ConfigServer 客户端与元数据上报，但自 2023 年 6 月起已停滞且存在合并冲突。当前 ConfigServer 管控链路已全面演进至 V2 协议架构（#1793 已合并），且 V2 Provider 扩展支持正由 #1710 专项推进。本 PR 基于旧版架构的实现已被新方案替代，审查反馈中的架构调整与 C++ 规范问题亦未解决，故予以关闭。

## Best Possible Solution

建议关注或参与 #1710（ConfigServer V2 Provider 支持）的后续开发。若火山引擎集成仍需合入，请基于 V2 架构与最新 main 分支重新实现，并将环境特有逻辑封装于独立 Provider/Client 实现中，避免侵入核心 AppConfigBase。
