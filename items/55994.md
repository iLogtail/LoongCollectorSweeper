## Summary

Keep open. Current `main` still contains the channel-agnostic `send` poll guard in `src/infra/outbound/message-action-runner.ts`, and the latest shipped release `v2026.4.22` contains the same code. The repo also still has a regression test that intentionally rejects `send` when poll-shaped params are present on a non-Telegram test channel. I could not do an end-to-end reproduction because `openclaw-weixin` is not present in this checkout, so there is not strong evidence the reported regression is fixed or obsolete.
