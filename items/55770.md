## Summary

Keep PR #55770 open. Current main still hard-codes exec command display truncation at 120 characters and does not expose a persisted per-session `verboseLimit` through the session schema, directive parser, TUI, or web slash-command path. The PR is not obsolete or implemented on main; the submitted patch also has unresolved review findings around stale TUI/web state that should be fixed before merge.

## Best Possible Solution

Keep the PR open or replace it with a cleaned-up implementation. The best path is to add a typed `verboseLimit` to session storage, gateway rows, and `sessions.patch`; thread an `execDetailMaxLength` option through tool display resolution and agent event metadata; support `/verbose limit <n>` consistently in directive, TUI, web, and messaging surfaces; and add focused tests for default behavior, invalid input, set/query, clearing, session switch/reset, and web refresh after patch.
