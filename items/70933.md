## Summary

Current `main` still makes Control UI `New session` a one-click `/new` action with no confirmation prompt or remember-my-choice option. The implementation does preserve the unsent draft and attachments after the reset request succeeds, but the documented `/new` behavior still rotates the current `sessionId`, so the prior transcript/context is intentionally left behind. This is a live core-UI enhancement request, not already implemented or closeable on conservative cleanup grounds.
