## Summary

Keep #37131 open. Current main has partial direct invocation via `POST /tools/invoke`, `/bash`, and some skill-command dispatch paths, but the umbrella request is not implemented: the Gateway direct-invoke resolver still builds only `createOpenClawTools`, not `createOpenClawCodingTools`, so `exec`/`read`/`write` coding tools are not available through the generic HTTP surface. Cron also still accepts only `systemEvent` and `agentTurn` payloads, with no direct `toolCall` payload contract. The related open PR #63919 and open issue #18160 cover useful slices, but neither is merged/shipped nor broad enough to supersede this umbrella item.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either land a security-reviewed direct coding-tool composition for `/tools/invoke`, explicitly decide the slash/CLI direct invocation surface, and add an additive cron direct-tool payload contract, or split those remaining surfaces into canonical narrower issues and close this as superseded by those maintainers-approved trackers.
