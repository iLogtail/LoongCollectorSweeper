## Summary

Keep open. Current main has a partial direct invocation surface via `POST /tools/invoke`, but it does not satisfy the main requested capability for direct invocation of coding tools like `exec`, `read`, and `write`, nor the proposed cron `toolCall` payload. The linked PR #63919 is relevant and positively tracks the missing gateway coding-tool wiring, but it is still open/unmerged in the provided GitHub context, so the issue should remain open until that work lands or maintainers decide the narrower HTTP surface is sufficient.

## Evidence

- **HTTP direct invoke exists:** `handleToolsInvokeHttpRequest` handles `POST /tools/invoke`, parses `body.tool`, resolves tools, runs the before-tool-call hook, executes the selected tool directly, and returns `{ ok: true, result }` without an LLM turn.
  - file: [src/gateway/tools-invoke-http.ts:129](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/tools-invoke-http.ts#L129)

  - file: [src/gateway/tool-resolution.ts:2](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/tool-resolution.ts#L2)
