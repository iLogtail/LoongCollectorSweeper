## Summary

Current `main` already routes embedded runner stream selection through `resolveEmbeddedAgentStreamFn(...)` and explicitly passes `authStorage`/`resolvedApiKey`, while the helper now reinjects `options.apiKey` for provider-owned stream functions. A regression test on `main` covers the custom-provider auth path, so the reported `No API key for provider` failure is implemented on current `main`.
