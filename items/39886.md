## Summary

Current main no longer matches the bug model this PR was written against. Webchat route inheritance for channel-agnostic sessions is now blocked in gateway routing, while session-delivery on main intentionally preserves established external routes for externally originated sessions. This PR would re-flip current tested behavior rather than fix a current-main bug.

## Evidence

That is the current entry-point behavior for the reported webchat leak.

## Close Comment

Closing this based on a Codex review of current `main`. The original leak no longer matches the current routing model: `src/gateway/server-methods/chat.ts:554` now blocks webchat clients from inheriting external delivery routes on channel-agnostic sessions, and `CHANGELOG.md:3137` documents that shipped behavior. Current `main` also intentionally preserves established external routes for externally originated sessions/subagent completion delivery (`src/auto-reply/reply/session-delivery.ts:103`, `src/auto-reply/reply/session.test.ts:3372`), so this PR would now flip `agent:main:main` and similar keys in the opposite direction and conflict with current regression coverage rather than fix a current-main bug. If there is still a reproducible leak on current `main`, please open a fresh report with the exact session-key shape, caller surface, and expected vs. actual routing.
