## Summary

Keep open. Current main b67d9bf7f06f4c655b1e0869395056c7d631abc2 still does not implement agent-scoped internal hook policy overrides: internal hook dispatch is global-only, config/types/schema lack `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`, and the docs still describe only global internal-hook configuration plus external webhook agent routing. The merged `before_agent_reply` plugin hook is related context but does not supersede this PR’s core config/runtime policy proposal.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants agent-scoped internal hook policy, the best path is a core config/runtime change that adds explicit policy types and strict schema support for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; resolves policy only for real agent-scoped events; documents exact event matching; and adds focused tests for global/default/per-agent precedence and non-agent hook behavior. If maintainers reject the product direction, close it explicitly with that rationale rather than automated cleanup.
