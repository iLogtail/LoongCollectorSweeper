## Summary

Keep #20237 open. Current main covers some pieces, including cron list/edit/run/history, delete-after-run in the WebUI, browser Web Push, an inline context notice with Compact, and automatic pre-compaction memory flush. The central request still remains: a native in-app notification/toast system, cron-result popups with actions, and snooze/reschedule controls are not implemented in the Control UI or first-class cron RPC surface.

## Best Possible Solution

Keep this open until the missing Control UI notification work is implemented or explicitly descoped. The likely product path is a native in-app toast/popup layer, cron run completion events wired into actionable result notifications, snooze/reschedule controls backed by cron.update or a dedicated validated RPC, and a decision on whether context warnings should stay inline or also produce toast-style Flush/Save prompts alongside the existing Compact and automatic memory-flush behavior.
