## Summary

Keep #62212 open. Current main still does not implement a working Google Vertex ADC OAuth path for Gemini chat/text requests: it detects ADC only as the non-secret `gcp-vertex-credentials` marker, the bundled Google provider still hooks `google-vertex`, and the Google transport sends non-JSON credentials as `x-goog-api-key` rather than exchanging ADC for a bearer token or constructing Vertex AI project/location endpoints. The linked #65023 is relevant and appears to be the right direction, but the provided GitHub context says it remains open/unmerged, so this issue should remain a live tracker.

## Best Possible Solution

Keep the issue open until OpenClaw ships a real Google Vertex path: either merge a focused `google-genai`/Vertex implementation like #65023 or add native `google-vertex` handling that exchanges ADC through the Google auth/default credential chain, builds project/location Vertex AI endpoints, avoids sending `gcp-vertex-credentials` as `x-goog-api-key`, and adds regression coverage for ADC auth and endpoint routing.
