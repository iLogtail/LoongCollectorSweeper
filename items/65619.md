## Summary

Keep #65619 open. Current main still has the reported loopback fallback path: `resolveGatewayBindHost("loopback")` returns `0.0.0.0` when the loopback preflight fails, and `resolveGatewayRuntimeConfig` then throws the exact `gateway bind=loopback resolved to non-loopback host ...` error. The linked targeted PR #65635 was closed unmerged, and the latest release tag inspected still contains the same fallback behavior.

## Best Possible Solution

Keep the issue open and land a focused gateway fix, likely equivalent to #65635: explicit `gateway.bind=loopback` should not downgrade to `0.0.0.0` after a failed preflight. Add a regression test for the failed-preflight path in `resolveGatewayBindHost` or runtime config, and include a user-facing changelog entry once fixed.
