## Summary

Keep #72020 open. Current main still does not wire Signal inbound messages into the shared status reaction lifecycle: the Signal handler dispatches only typing/model callbacks, while Signal reaction primitives and shared status-reaction callbacks already exist. The linked PR #72025 is the active proposed fix and should remain the canonical implementation path until merged.

## Best Possible Solution

Keep the issue open and use #72025 or an equivalent Signal-owned patch to add a Signal status reaction controller in `extensions/signal/src/monitor/event-handler.ts`, route reply lifecycle callbacks into `dispatchInboundMessage`, and add focused regression tests for direct and group inbound messages with `messages.statusReactions.enabled: true`.
