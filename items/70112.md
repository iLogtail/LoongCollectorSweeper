## Summary

Keep PR #70112 open. Current main still has the reported compaction bug: the compaction and pre-prompt paths delegate to upstream `estimateTokens()`, while the existing CJK-aware helper is only present elsewhere. The PR remains a plausible focused fix, though the latest review context still needs maintainer handling before merge.

## Best Possible Solution

Keep PR #70112 open. The best path is to rebase or otherwise land a CJK-aware estimator in `src/agents/compaction.ts`, use it consistently from embedded compaction and pre-prompt estimation, keep ASCII behavior unchanged, cover CJK text and tool-call arguments with focused regression tests, and resolve the remaining token-vs-character threshold concern in the preemptive truncation route.
