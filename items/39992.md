## Summary

Keep open. Current main still does not implement the requested `openclaw doctor` warning for synthetic/provider-supplemented runtime catalog models that are available but missing from an explicit `agents.defaults.models` allowlist. The related PR #46235 directly attempted this feature, but the provided GitHub context says it was closed unmerged, and current main still only has a narrow hooks.gmail.model allowlist warning.

## Best Possible Solution

Keep #39992 open until a maintainer accepts and lands a doctor health contribution that loads the read-only runtime catalog, identifies provider-owned synthetic/supplemental catalog rows with a reliable marker or contract, compares them to an explicit `agents.defaults.models` allowlist, and emits a warn-only diagnostic with model IDs and `openclaw models allowlist add <model-id>` guidance. The check should skip empty/absent allowlists and include focused tests for no allowlist, all allowlisted, missing synthetic rows, and provider/cost-profile wording if maintainers want that broader warning.
