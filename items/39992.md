## Summary

Current main still does not implement the requested `openclaw doctor` warning for provider-owned/synthetic runtime catalog models that are available but missing from an active `agents.defaults.models` allowlist. The catalog/listing surfaces now include provider-owned supplemental rows, but doctor only has a narrow hooks model check and no catalog-vs-allowlist gap diagnostic; the direct PR #46235 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39992 open and implement a read-only core doctor health contribution. It should load the same runtime catalog used by model list/gateway surfaces, identify provider-owned forward-compatible or synthetic catalog rows through a stable marker/contract, compare them with a non-empty `agents.defaults.models` allowlist, and warn with concrete model ids plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, no gaps, gap detection, provider marker behavior, and warning wording. Treat the later API-vs-subscription cost-profile warning as a separate doctor/product decision.
