## Summary

Keep open. Current main still does not implement the requested `openclaw doctor` diagnostic for runtime/catalog synthetic or provider-owned model rows that are available but missing from an active `agents.defaults.models` allowlist. The source has adjacent allowlist filtering, hooks-specific model validation, provider catalog augmentation, and runtime-allowed model listing, but no general doctor warning that computes and reports available-but-not-allowlisted model gaps. The directly related PR #46235 attempted this exact feature and is closed unmerged in the provided GitHub context, so this issue remains the canonical actionable item.

## Best Possible Solution

Implement a read-only core doctor health contribution that loads the same runtime model catalog used by gateway/model-list surfaces, identifies provider-owned synthetic or forward-compatible rows through a stable marker or documented provider-owned contract, compares them with a non-empty `agents.defaults.models` allowlist, and warns with concrete model ids plus current allowlist-edit guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, all rows already allowed, gap detection, non-synthetic/provider filtering, and warning wording. The later cost-profile warning from the comments should be handled as a separate doctor warning or an explicit extension only if maintainers want that product behavior.
