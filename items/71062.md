## Summary

Keep PR #71062 open. Current main does not contain the core fixes described by the PR: the Pi client-tool adapter still returns a normal pending jsonResult without terminate:true, the /v1/responses SSE path still emits function_call args only on output_item.added/output_item.done, and the Responses stream parser only handles function_call_arguments.delta, not function_call_arguments.done. The PR is a focused, recent bug fix with concrete regression-test intent and active author follow-up, so it is not conservative cleanup-close material.

## Best Possible Solution

Keep this PR open and either land it or an equivalent maintainer-owned patch after review. The best fix path is to add terminate:true to the Pi client-tool handoff stub, emit response.function_call_arguments.delta and .done for streamed /v1/responses function_call items while preserving populated args in response.completed, teach the upstream Responses stream parser to consume function_call_arguments.done, update the schema/docs if this is a public SSE event, and keep the targeted regression tests for the adapter, gateway SSE sequence, and .done-only parser path.
