## Summary

Keep #50989 open. Current main has useful phase-aware suppression for assistant output marked as `commentary`, and that partially addresses related reports, but it does not implement the requested generic/group-channel suppression for unphased pre-tool narration. The reported Claude/Gemini Slack path can still emit ordinary text before tool calls, and current tests intentionally preserve that behavior.

## Best Possible Solution

Keep the issue open, or explicitly dedupe it into the broader open pre-tool text tracker if maintainers want one canonical item. The product fix should add a scoped agent/channel option that buffers block replies until tool-use vs final/text-only status is known, discards configured pre-tool narration for tool-use turns, preserves normal text-only replies and final answers, and adds regression coverage for Slack/group channels plus Anthropic and Google unphased streams.
