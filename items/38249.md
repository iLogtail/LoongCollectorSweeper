## Summary

Keep #38249 open. Current main has active-schema validation, legacy config repair, unknown-key pruning, and a Control UI pending-change diff, but it still does not implement the requested `openclaw doctor --config-diff` CLI workflow, version-to-version schema comparison from `meta.lastTouchedVersion`, categorized upgrade reporting, migration suggestions, or `--config-diff --fix`.

## Best Possible Solution

Keep #38249 open until maintainers implement or explicitly reject a CLI upgrade-diff workflow. The likely path is to add `openclaw doctor --config-diff` with docs and tests, compare the active config with the current schema and a trustworthy previous-version baseline from `meta.lastTouchedVersion`, report unknown keys, invalid enum values, removed/renamed/new keys, and migration suggestions, and reserve `--fix` for explicit safe migrations with backups. Keep #39780 separate for Control UI apply UX.
