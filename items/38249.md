## Summary

Keep open. Current `main` has already fixed much of the issue's reported premise: `openclaw config validate` now fails on unrecognized keys and invalid enum values, and `openclaw doctor --fix` already detects and migrates many legacy config shapes. But there is no `openclaw doctor --config-diff` flag or equivalent version-to-version schema diff/migration report, so the requested feature itself is not implemented on `main`.
