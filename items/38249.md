## Summary

Keep open. Current main has useful config hardening: schema validation rejects unknown/invalid config shapes, `doctor --fix` can remove unknown keys, and `meta.lastTouchedVersion` is tracked. But the requested `openclaw doctor --config-diff` surface is not implemented: there is no CLI flag, no per-version schema diff output, no “new keys since last touched version” report, and no version-to-version migration guidance. The related open Control UI issue #39780 overlaps only on before/after UI fixes, and closed #40015 is about safe agent-driven config writes, so neither fully supersedes this CLI doctor request.
