## Summary

Keep #38249 open. Current main has stricter config validation, `meta.lastTouchedVersion` tracking, and generic `doctor --fix` unknown-key pruning, but it still lacks the requested `openclaw doctor --config-diff` CLI surface, version-to-version schema comparison, new/removed/renamed key reporting, and actionable upgrade migration guidance. Related #39780 remains a separate Control UI apply/diff request, and closed #40015 covered safe config writes rather than this CLI doctor upgrade-diff workflow.

## Best Possible Solution

Keep #38249 open until maintainers either implement a real CLI upgrade-diff surface or explicitly reject the product direction. A complete fix would add `openclaw doctor --config-diff` plumbing, tests, and docs; compare the active config against current schema plus a trustworthy previous-version baseline from `meta.lastTouchedVersion`; report unrecognized keys, invalid enum values, new/removed/renamed keys, and migration suggestions; and reserve `--fix` for safe migrations with backups. #39780 can remain the separate Control UI before/after apply-flow item.
