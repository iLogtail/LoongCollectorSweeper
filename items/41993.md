## Summary

Keep open. Current main still matches the reported root cause: DNS results are checked with an any-blocked-address fail path before IPv4 preference/deduping, and the pinned address list is built from the unfiltered results. The related fix PRs in the provided GitHub context were closed unmerged, so they do not prove this is implemented on main.

## Evidence

- **Current main still throws on the first blocked resolved address:** `assertAllowedResolvedAddressesOrThrow` iterates every DNS result and throws `SsrFBlockedError` as soon as `isBlockedHostnameOrIp(entry.address, policy)` is true. That is the exact behavior described in #41993.
