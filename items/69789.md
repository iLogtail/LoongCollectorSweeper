## Summary

Keep open. Current main still routes Feishu markdown table output into interactive card sends without specific 230099/11310 table-limit handling or a plain-text fallback. The linked larksuite/openclaw-lark#65 fix and open PR #70651 are useful implementation references, but the bundled OpenClaw Feishu plugin has not shipped that behavior yet.

## Best Possible Solution

Keep #69789 open as the canonical bundled Feishu bug until a fix lands. The best path is to use #70651 and larksuite/openclaw-lark#65 as references, add bundled-plugin handling that detects Feishu Card Kit table-limit errors such as 230099/11310/table-number-over-limit on card send paths, and fall back to plain post/text or split card output with regression tests for direct outbound and reply-dispatcher delivery.
