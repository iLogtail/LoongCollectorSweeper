## Summary

Keep #69789 open. Current main still routes Feishu markdown tables through interactive card send paths and has no Feishu 230099 / ErrCode 11310 table-limit detection or fallback. The reporter’s linked standalone Lark fix is useful context, and open PR #70651 appears to be the relevant proposed OpenClaw fix, but the bug is not implemented on main yet.

## Best Possible Solution

Keep #69789 open and review/land PR #70651 or an equivalent Feishu plugin change. The fix should detect Feishu Card Kit table-limit failures from sendMarkdownCardFeishu/sendStructuredCardFeishu, preserve other error behavior, fall back to plain post text or another graceful delivery path, and add regression tests for direct outbound and reply-dispatcher card paths.
