## Summary

Keep #72381 open. Current main still lacks the requested Google Meet `realtime.agentId` config/schema surface and does not pass an agent id into the shared realtime consult path. The shared consult runtime already supports optional agent routing, Voice Call already uses that capability, and the PR is a narrow plugin fix with positive review context rather than a duplicate of the related chrome-node cleanup PR.

## Best Possible Solution

Keep the PR open and review it as a plausible narrow plugin fix: add `realtime.agentId` to the Google Meet config type, resolver, manifest labels/schema, and pass it into `consultRealtimeVoiceAgent`, with resolver and embedded-agent invocation tests. Consider documenting the advanced Google Meet config key if manifest help is not considered enough.
