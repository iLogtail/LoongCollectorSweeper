## Summary

Keep open. Current `main` partially improved one `sessions_spawn(mode="session")` thread-binding error path in `835f7680`, but the PR's remaining UX cleanup is still not present: the `threadBindingReady:false` path still uses the old copy, `session_status` tree-visibility errors remain generic, related tests still assert the old wording, and the docs do not yet add the persistent-target vs live-session vs visibility distinction this PR proposes.
