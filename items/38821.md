## Summary

Current main now fixes the stale-image/media-rehydration failure described by #38821. Old image blocks and textual media references are scrubbed from the provider replay view after the recent-turn window, while current Discord attachments still flow through MediaPaths and remain available to vision-capable models.

## Best Possible Solution

Close #38821 as implemented on main and keep the current replay-pruning implementation. If users still see unrelated text-context contamination, track that separately against the context-engine/compaction path rather than reopening this stale-image media rehydration report.
