## Summary

Keep open. Current `main` still ships the Kimi stream wrapper path the report points at, that wrapper is active for the bundled Kimi provider, the same logic is present in the latest release tag, and the only documented generic mitigation for runaway repeated tool calls is disabled by default. I did not find local evidence that the referenced PR or another change fixed this on `main`.
