## Summary

Keep open. Current main does not contain the proposed Kimi Coding loop fix, the directly linked PR #71274 was closed unmerged, and the existing Kimi/Moonshot fixes cover adjacent tool-call parsing or ID replay problems rather than this newly reported Kimi Coding final-answer/tool-loop behavior.

## Best Possible Solution

Keep #71273 open. The best path is a focused Kimi Coding regression test for the reported after-tool-results repeated-call transcript, then a narrow fix in `extensions/kimi-coding/stream.ts` so `toolUse` is emitted only when the final normalized assistant message actually contains structured tool calls, following the same invariant already used by OpenAI-compatible stream handling. If a broader Kimi loop fix lands first, close this against that canonical merged PR.
