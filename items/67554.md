## Summary

Keep #67554 open. Current main still hardcodes file-based memory behavior in workspace templates while memory-core's prompt section only provides recall guidance, so the PR's decoupling is not implemented on main. The existing memory plugin prompt APIs make the PR direction plausible, and the provided related items do not supersede this specific template-vs-plugin conflict.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is a narrowed version that removes memory-backend-specific instructions from default workspace templates, moves memory-core's file-backed persistence guidance into its active memory prompt builder, preserves the platform-level privacy rule, and adds a small regression check that fresh templates no longer force file-based memory when another memory plugin owns the slot.
