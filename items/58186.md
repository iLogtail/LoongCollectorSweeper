## Summary

Current `main` does not implement the PR’s requested behavior. Gmail hook mappings still have no hook-level `systemPrompt` field, no dispatch/runtime plumbing carries one, and the docs/config schema on `main` still describe only `messageTemplate`-driven hook content.

## Evidence

- **Hook config contract still omits mapping-level systemPrompt:** `HookMappingConfig` defines `sessionKey`, `messageTemplate`, and `textTemplate`, but no `systemPrompt` field.
  - file: [src/config/types.hooks.ts:19](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/types.hooks.ts#L19)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Gateway mapping/runtime still only builds agent messages:** The Gmail preset on `main` sets `sessionKey` plus `messageTemplate`, and `buildActionFromMapping` constructs agent actions from `message` without any `systemPrompt` field.
  - file: [src/gateway/hooks-mapping.ts:72](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/hooks-mapping.ts#L72)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Dispatch payloads still cannot carry hook system prompts:** `HookAgentPayload` and cron `agentTurn` payload fields do not include `systemPrompt`, so there is no current path from hook mapping to isolated agent run for this data.
  - file: [src/gateway/hooks.ts:213](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/hooks.ts#L213)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Schema/docs on main still expose only messageTemplate-based hook behavior:** Generated config metadata has entries for `hooks.mappings[].sessionKey` and `hooks.mappings[].messageTemplate` but no `hooks.mappings[].systemPrompt`, and the Gateway hook docs still describe payload templates generically without a literal-only `systemPrompt` option.
  - file: [src/config/schema.base.generated.ts:26662](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/schema.base.generated.ts#L26662)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current Gmail hook docs/example still lack systemPrompt:** The Gmail hook example on `main` shows `sessionKey` and `messageTemplate` only; there is no built-in preset/system-prompt guidance documented for Gmail hooks.
  - file: [docs/gateway/configuration-reference.md:438](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/gateway/configuration-reference.md#L438)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
