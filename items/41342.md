## Summary

Keep open. Current `main` still routes OpenAI-compatible voice-note TTS to `opus` by default, only offers a manual `responseFormat` override for custom endpoints, and has no Kokoro-specific mitigation or new mainline fix to point to. The issue is recent and has a second confirmation, so there is not strong evidence for any close reason.

## Evidence
