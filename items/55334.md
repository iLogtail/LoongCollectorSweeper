## Summary

Keep open. Current `main` still stores `skillsSnapshot` inside session entries, duplicates that session entry for isolated cron run records, eagerly deserializes the full `sessions.json` into memory, and only has targeted automatic pruning for cron run sessions. The issue’s main performance complaint is not implemented away on `main`.

## Risks / Open Questions

- One-shot subagent runs do have auto-archive support elsewhere, so not every subagent session is unbounded; that does not change the unresolved `skillsSnapshot` duplication and eager whole-store load on `main`.
- `session.maintenance` can cap total entries, but the default cap does not remove per-entry snapshot duplication or the cost of loading/parsing the entire store.
