## Summary

Keep open. Current main has important mitigations, including enforced session maintenance and cron-run retention, but the core issue is not fully implemented: `skillsSnapshot` and `systemPromptReport` are still persisted per session entry, the session store is still read as a whole JSON file before any load-time pruning can help, `sessions.list` and `chat.history` still traverse the JSON-backed store path, and default one-shot subagent spawns still use `cleanup: "keep"` with regression coverage showing keep-mode runs do not get an archive time.

## Best Possible Solution

Keep #55334 open as the canonical session-store OOM/perf tracker. The best fix should reduce persisted per-session metadata by deduplicating or referencing `skillsSnapshot`/prompt reports, add explicit retention or cleanup for ephemeral one-shot subagent runs without deleting thread-bound or intentionally persistent sessions, and move `sessions.list`/`chat.history` toward lazy or paged metadata access so they do not parse and materialize the whole store on each hot path. Keep the existing cron reaper and generic maintenance as mitigations, not as the final closure condition.
