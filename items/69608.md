## Summary

Keep open. Current main only has partial startup-context support for canonical plus date-prefixed slugged daily files; it does not contain the PR's shared daily-memory/session-summary model, sentinel/provenance handling, SDK exports, or aligned doctor/dreaming/short-term behavior. The PR is broad and needs maintainer review/rebase, but it still contains potentially useful bug-fix work that is not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to rebase onto current main, split or narrow the change around the missing shared daily-memory/session-summary helpers and their runtime callers, preserve the useful regression coverage, and avoid bundling unrelated docs/bootstrap/log cleanup unless it is required for the same bug path.
