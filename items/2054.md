## Summary

本 PR 提议修改核心 Encoder 接口（EncoderV1/EncoderV2）签名以支持变量键解析与 Buffer 回收，属于破坏性 API 变更。当前 PR 存在合并冲突，且直接修改核心接口签名会影响所有现有 Encoder 插件实现。该特性具有明确业务价值（如支持 flusher_http 动态 Query 参数），但需维护者评估向后兼容策略（如接口版本隔离、默认适配层或扩展包下沉）并解决冲突后方可进入实质审查。

## Best Possible Solution

建议贡献者先 Rebase 解决与 main 分支的冲突。维护者需审查核心接口签名变更的破坏性：若接受该设计，需同步更新仓库内所有 Encoder 实现，或提供默认适配层/接口版本隔离以平滑过渡；同时确认 varKeys 解析逻辑与 BufferRecycler 是否应下沉至公共扩展包。待兼容方案确定、全量 Encoder 适配完成且 CI 通过后合并。
