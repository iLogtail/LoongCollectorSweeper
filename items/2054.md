## Summary

本 PR 提议增强 Encoder 核心接口以支持变量键解析（如 %{metadata.xxx}），涉及 EncoderV1/EncoderV2 签名变更及 BufferRecycler 新增。该变更属于核心 API 扩展，具有明确的产品价值（统一 Converter 与 Encoder 的变量处理能力），但存在向后兼容性影响。当前 PR 代码结构清晰且已同步调整内置 Prometheus Encoder 与 HTTP Flusher 的调用链及测试，建议保持开启等待维护者审查接口兼容性与合并路径。

## Best Possible Solution

保持开启，建议维护者审查接口变更的向后兼容性，确认是否需提供适配层、弃用过渡期或迁移指南。作者可补充所有内置 Encoder 实现的适配情况（当前仅见 Prometheus 示例），并完善兼容性说明。待审查通过且解决主线冲突后合入。
