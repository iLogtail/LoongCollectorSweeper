## Summary

本 PR 提议增强 Encoder 接口以支持变量键解析与 Buffer 回收，但直接修改了核心 EncoderV1/EncoderV2 的方法签名，属于破坏性 API 变更。目前仅适配了 Prometheus Encoder 与 HTTP Flusher，仓库内其他实现未同步更新，直接合并将导致编译失败，需维护者进行架构评审并制定向后兼容或全量迁移方案。

## Best Possible Solution

保持开启。建议贡献者先 Rebase 解决与 main 的冲突。维护者需审查核心接口签名变更的破坏性：若接受该设计，需同步更新仓库内所有 EncoderV1/V2 实现，或提供默认适配层/接口版本隔离（如新增独立扩展接口或可选能力接口）以平滑过渡。待兼容方案确定、全量 Encoder 适配完成且 CI 通过后合并。
