## Summary

Keep open. Current `main` still sends Feishu streaming replies as CardKit `card_id` references, while quoted-message reads still parse interactive content only from inline `elements` / `body.elements` and otherwise fall back to `[Interactive Card]`. The quoted-message fetch path does not request any resolved raw card payload, and the latest release `v2026.4.22` has the same Feishu code on these paths, so there is no strong evidence that #54972 is already fixed on main or in the latest release.
