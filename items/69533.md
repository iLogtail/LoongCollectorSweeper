## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and request prompt in `memory-core`, the accepted dreaming config/schema does not expose any `prompt` field, docs only document `enabled`/`frequency` as user-facing dreaming settings, and the Dreams UI can toggle dreaming but does not expose prompt editing.

## Best Possible Solution

Keep the issue open as a real memory-core product enhancement. A good implementation would add a typed dreaming narrative prompt/template field under `plugins.entries.memory-core.config.dreaming`, update `openclaw.plugin.json`, `src/memory-host-sdk/dreaming.ts`, `extensions/memory-core/src/dreaming-narrative.ts`, the Dreams settings UI, docs, and focused tests for placeholder rendering and safe fallback to the default prompt.
