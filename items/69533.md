## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and request text, the memory-core dreaming schema/config resolver expose no prompt/template field, and the Dreams UI only patches the enabled flag. The request is coherent, recent, unprotected, and targets bundled memory-core behavior rather than an already-satisfied or clearly external ClawHub item.

## Best Possible Solution

Keep the issue open. The best path is a memory-core design/implementation that adds a documented, schema-validated Dream Diary customization contract, likely starting with language/locale and a narrowly scoped diary prompt/template with explicit allowed placeholders, then threads it through buildNarrativePrompt/NARRATIVE_SYSTEM_PROMPT, updates the plugin manifest, Dreams UI, docs, and focused config/narrative tests.
