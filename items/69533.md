## Summary

Keep #69533 open. Current main still hardcodes the memory-core Dream Diary narrative system prompt and fixed English request labels, the dreaming config schema/resolver expose no prompt/template field, and the Dreams UI only patches the enabled flag. The related prompt items in the provided context do not implement or supersede this specific Dreaming customization request.

## Best Possible Solution

Keep the issue open and implement this inside the bundled memory-core plugin surface: add a schema-validated Dream Diary prompt/template or locale contract under plugins.entries.memory-core.config.dreaming, thread it through buildNarrativePrompt/NARRATIVE_SYSTEM_PROMPT/generateAndAppendDreamNarrative, expose the setting in the Dreams/Settings UI, document supported placeholders, and add focused config/narrative/UI tests.
