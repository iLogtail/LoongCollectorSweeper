## Summary

Keep PR #64758 open. Current main still has the active-but-not-streaming steer gap the PR is trying to address, but the PR is not safe to auto-close or treat as already shipped: its proposed abort fallback needs to preserve heartbeat/drop policy and address the existing review comments.

## Best Possible Solution

Keep the PR open and ask for a revision: preserve the successful streaming steer fast path, add a non-streaming steer fallback that only aborts when queue policy would enqueue a real user followup, add explicit heartbeat no-abort coverage, fix the misleading `steer-backlog` test name, and remove or justify the now-unused `isStreaming` parameter.
