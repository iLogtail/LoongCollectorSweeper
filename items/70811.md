## Summary

Keep PR #70811 open. Current main still unconditionally validates `entry?.config` in both the plugin snapshot loader and CLI metadata path, while `validatePluginConfig` converts `undefined` to `{}`. That means strict required schemas can still produce the false-positive invalid-config errors this PR targets. The PR is a focused, recent bug-fix attempt with regression coverage and follow-up work, not obsolete cleanup.

## Best Possible Solution

Keep #70811 open for maintainer review. The best path is a context-aware loader fix that skips schema validation only when `activate:false` and config is actually `undefined`, preserves normalized/defaulted `pluginConfig` for non-activating registration, and keeps activation plus explicit-config validation strict. Retain focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`, then merge or request final changes based on current-head CI and the security/defaults review thread.
