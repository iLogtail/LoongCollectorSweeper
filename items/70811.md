## Summary

Keep PR #70811 open. Current main still unconditionally validates `entry?.config` in both the snapshot/non-activating loader path and the CLI metadata loader path, while `validatePluginConfig` converts `undefined` to `{}` before AJV validation. That means the linked false-positive required-config failure remains actionable in core and is not implemented, duplicate, or ClawHub-only cleanup.

## Best Possible Solution

Keep the PR open for maintainer review and likely land a focused loader fix: defer validation only when config is `undefined` in non-activating snapshot/CLI metadata paths, preserve normalized/defaulted `api.pluginConfig`, continue validating real activation and explicit config, and keep the new regression coverage for both affected paths.
