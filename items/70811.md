## Summary

Keep PR #70811 open. Current main still unconditionally validates `entry?.config` in the non-activating snapshot and CLI metadata loader paths, and `validatePluginConfig` still normalizes `undefined` to `{}`, so the linked false-positive strict-schema failure remains actionable in core. The PR has a concrete linked bug (#70658), targeted loader/test changes, and active follow-up addressing review feedback; it is not implemented on main, superseded, or a ClawHub/out-of-repo item.

## Best Possible Solution

Keep PR #70811 open for maintainer review. The best path is a scoped core loader fix that defers validation only for non-activating omitted config, continues validating real activation and explicit config, preserves normalized/defaulted `pluginConfig`, and lands with the focused snapshot and CLI metadata regression tests already proposed by the PR.
