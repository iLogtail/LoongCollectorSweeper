## Summary

Keep open. Current `main` still derives `sessions.list` row models from the generic session model resolver and does not read ACP runtime metadata such as `entry.acp.runtimeOptions.model`, so the reported misleading default-model behavior for ACP sessions still appears valid on `a6aa151653ab478e9aa23e95c6446c41adc4fce1`.

## Evidence

- **sessions.list still uses the shared store serializer:** The gateway `sessions.list` handler loads the combined session store and passes it straight to `listSessionsFromStore(...)`; there is no ACP-specific branch in the handler.
  - file: [src/gateway/server-methods/sessions.ts:579](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/sessions.ts#L579)

  - file: [src/gateway/session-utils.ts:1461](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/session-utils.ts#L1461)
