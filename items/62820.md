## Summary

Keep open. Current main has partial protection in the assistant-visible text pipeline, but the specific `sanitizeUserFacingText` / reply-normalization path described in #62820 still does not strip tool-call XML. The related PR #62883 directly targeted this gap but was closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #62820 open and implement the targeted sanitizer fix on main: add regression tests for `sanitizeUserFacingText` and the reply-normalization path, then route that sanitizer through the existing assistant-visible XML stripping helpers or explicitly call both `stripMinimaxToolCallXml` and `stripToolCallXmlTags` before outbound delivery.
