## Summary

Current main contains Discord reconnect/shutdown hardening, but I did not find strong evidence that issue #55340 is already fixed. The `message:sent` internal hook emission still lives in the generic outbound delivery path, while the Discord auto-reply path sends through Discord-specific helpers that do not obviously invoke that hook pipeline. The existing reconnect tests cover gateway lifecycle/status recovery, not `message:sent` hook persistence across reconnects, so the close evidence is not strong enough.
