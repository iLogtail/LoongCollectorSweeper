## Summary

Keep #10118 open. Current main preserves multiline text once the editor produces it, and OpenClaw delegates unhandled editor input to the pinned pi-tui editor, but the remaining requested user-facing work is still missing: TUI docs and /help do not mention Shift+Enter, there is no configurable Enter/Ctrl-Enter send behavior, and related PR #43327 is still open/unmerged in the provided context. A maintainer comment on Apr 26, 2026 also explicitly kept this remaining scope open.

## Best Possible Solution

Keep the issue open and narrow it to the remaining user-visible work: document Shift+Enter in https://docs.openclaw.ai/web/tui and /help, add focused assertions for visible shortcut/help text, and decide whether OpenClaw should expose configurable Enter vs Ctrl/Cmd+Enter send behavior through config or CLI options. Treat PR #43327 as related prior work while it remains open and unmerged, not as a superseding canonical fix.
