## Summary

Keep #10118 open. Current `main` already has the basic Enter-to-submit and Shift+Enter-to-newline behavior through the pinned `@mariozechner/pi-tui` editor, and OpenClaw preserves submitted multiline text. The issue should stay open because the requested docs/help visibility and configurable send/newline behavior are still missing, and a maintainer-posted Codex review already preserved that remaining scope.

## Best Possible Solution

Keep the issue open and narrow the remaining work: document Shift+Enter in `docs/web/tui.md` and `/help`, add focused assertions for visible shortcut text, and make an explicit maintainer product decision on whether OpenClaw should expose pi-tui keybinding configuration for Enter vs Ctrl/Cmd+Enter send behavior. Treat PR #43327 as related but not canonical while it remains open and unmerged.
