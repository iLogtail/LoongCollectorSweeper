## Summary

Current `main` does not implement quoted-message media download for Feishu replies. The inbound path still resolves media only from the triggering message, then fetches the parent message as text-only context. For image replies, `getMessageFeishu()` parses the parent payload into a placeholder like `[image message]`, and `FeishuMessageInfo` still has no raw-content field needed to re-run media resolution on the quoted message.

## Evidence

- **Inbound media resolution only covers the current message:** `dispatchMessage()` builds `mediaPayload` from `resolveFeishuMediaList()` using `ctx.messageId` and `event.message.content`, before quoted-message fetch. The quoted-message branch only assigns `quotedContent = quotedMessageInfo.content`; there is no second `resolveFeishuMediaList()` call for `ctx.parentId`.
