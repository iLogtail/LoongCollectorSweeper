## Summary

Keep open. Current main still collapses channel config reload actions into full-channel restarts, while the existing channel manager already supports account-scoped start/stop. PR #43938 remains a focused, plausible fix for open issue #43935, but it needs rebasing onto current main reload-plan fields.

## Best Possible Solution

Keep this PR or its underlying issue open. The best path is to rebase the change onto current main, add an account-scoped restart plan field, parse strictly account-scoped channel account paths, clear account restarts when a full channel restart is also required, and route those account-only restarts through stopChannel(channel, accountId) and startChannel(channel, accountId) with updated regression tests.
