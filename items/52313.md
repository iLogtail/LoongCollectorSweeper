## Summary

Keep #52313 open. Current main still respawns unsupervised Linux gateways with `process.execArgv + process.argv.slice(1)`, which is the brittle package-manager realpath behavior reported in the issue. The related PR #52319 documented a plausible fix but is closed/unmerged, and the later #65668 cross-reference covers a different supervisor/orphaning failure mode rather than replacing this issue as the canonical tracker.
