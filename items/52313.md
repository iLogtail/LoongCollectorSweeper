## Summary

Keep #52313 open. Current main still respawns unsupervised gateway restarts with `process.execArgv + process.argv.slice(1)`, so the reported package-manager realpath failure remains possible. The related fix PR #52319 was closed unmerged, and I found no current source, test, docs, or changelog evidence that the stable-wrapper respawn behavior shipped.

## Best Possible Solution

Keep the issue open and implement a narrowly scoped respawn target resolver in `src/infra/process-respawn.ts`: preserve source/dev entrypoints, but for installed OpenClaw package roots and pnpm versioned realpaths respawn through a stable package entrypoint such as `openclaw.mjs` or the established source wrapper. Add focused tests for pnpm realpath rewriting, package-root fallback, source checkout passthrough, and update/restart SIGUSR1 flow behavior. Coordinate any supervisor-default changes with #65668 instead of merging that broader policy into this package-manager path fix.
