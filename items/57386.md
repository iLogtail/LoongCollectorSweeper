## Summary

Keep open. Current `main` still implements the same Google Chat auth behavior the report is about, the docs still present the disputed `project-number` path as supported, and the only post-`v2026.4.23` changes in this area are diagnostics/warnings rather than an auth fix.

## Evidence

- **Current auth code still uses the disputed verification paths:** `verifyGoogleChatRequest()` still requires `app-url` requests to pass `verifyIdToken(...)` and match `payload.sub` against `appPrincipal`, and still verifies `project-number` requests with `verifySignedJwtWithCertsAsync(...)` against Chat certs. I did not find any fallback or alternate handling that would resolve the reported 401s.
