## Summary

Keep PR #39322 open. Current main has BlueBubbles reaction-event support, but it still does not implement the PR's legacy `channels.imessage`/`imsg rpc` tapback reaction forwarding, and the related feature issue #39031 remains open rather than shipped or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, verify the upstream `imsg` reaction payload and subscribe contract, add the `includeReactions` config/schema/docs surface, add parser and monitor tests, and explicitly define how legacy iMessage group reactions interact with mention gating. If maintainers decide legacy `imsg` should not gain this feature, close or supersede it with a clear product decision on #39031 rather than treating it as already implemented.
