## Summary

Keep #39322 open. Current main still lacks legacy imsg tapback inbound-agent support: no includeReactions config/schema, no reaction parser/type, and the monitor subscribes only for attachments while handling only message/error notifications. BlueBubbles has tapback support, but docs describe it as the separate recommended iMessage path, not an implementation of the legacy imsg channel. The PR is still potentially useful, though it needs maintainer review and likely contract updates against current imsg reaction-stream semantics.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to update legacy iMessage support against the current imsg reaction contract, add includeReactions config/schema/docs, add parser and monitor tests, and define group mention-gating behavior for synthetic reaction events. Close only if maintainers decide the legacy imsg channel will not gain this feature and choose to route users to BlueBubbles or a canonical follow-up issue instead.
