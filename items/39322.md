## Summary

Keep #39322 open. Current main still does not implement legacy `imsg` iMessage tapback forwarding: there is no `includeReactions` config/schema, no reaction payload/parser, and the monitor subscribes only for attachments while handling only `message` and `error` notifications. The PR remains a relevant implementation candidate for the open canonical feature request #39031, though it still needs maintainer review against the current `imsg` reaction contract and group mention-gating behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the legacy iMessage implementation against the current `imsg` reaction stream, add `includeReactions` config/schema/docs, add parser and monitor tests, and explicitly define how synthetic reaction events interact with group mention gating. If maintainers decide legacy `imsg` will not support tapback forwarding, close or supersede this only with an explicit BlueBubbles-only/canonical #39031 product decision.
