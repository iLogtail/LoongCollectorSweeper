## Summary

Current `main` still has the schema/runtime mismatch this PR is trying to fix. The `message` tool advertises `buffer` on `send`, but outbound hydration still skips `send`, and the send delivery path still only forwards `media`/`mediaUrl`, not hydrated base64 buffers. The PR therefore should remain open for maintainer review rather than be auto-closed.
