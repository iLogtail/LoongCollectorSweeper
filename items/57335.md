## Summary

Keep PR #57335 open. Current main still has the reported schema/runtime mismatch: the `send` schema advertises `buffer`, but `hydrateAttachmentParamsForAction` still skips `send`, and `handleSendAction` only turns media/path-style fields into outbound media. The PR discussion also shows the author addressed the two substantive bot-review concerns by wiring hydrated buffers into delivery and applying channel media limits, so the PR still contains potentially useful bug-fix work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open or refresh it against current main, then review the final implementation as a focused bug fix: hydrate non-empty `send.buffer`, materialize it into the existing media path when needed, honor per-channel media limits, and keep the added unit/integration coverage for plain base64, data URLs, empty buffers, and buffer-only sends.
