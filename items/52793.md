## Summary

Keep open. Current main still models `tools.fs.workspaceOnly` as a single boolean guard for the filesystem tool family. There is a separate `tools.exec.applyPatch.workspaceOnly` setting for apply_patch, but no per-operation or per-tool policy for read/search/write/edit that would allow external reads while keeping writes and edits workspace-confined.
