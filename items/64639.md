## Summary

Keep open. Current main has useful mitigations for prompt bloat, especially agent-level skill allowlists and prompt limits, but it still hardcodes and scans the personal `~/.agents/skills` source and the schema/docs do not expose a per-source disable option. The provided issue discussion also points at the same missing short-term config toggle, and the related ClawSweeper items do not supersede this request.

## Best Possible Solution

Keep the issue open for a core skills-loader change. Add a typed, documented config such as `skills.load.disableSources` with stable source ids or explicit booleans, apply it before each built-in `loadSkills` call and in watcher path resolution, preserve current behavior by default, and add focused tests for disabling `agents-skills-personal` without breaking agent allowlists, prompt snapshots, status reporting, or precedence.

## Risks / Open Questions

- Existing `agents.defaults.skills` and `agents.list[].skills` reduce prompt bloat, so the eventual implementation should clearly distinguish visibility filtering from source scanning.
- Source gating should cover both initial loading and watcher targets to avoid a config that appears to disable a source but still scans or refreshes from it.
