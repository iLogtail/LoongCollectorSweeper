## Summary

Keep PR #69732 open. The linked bug #69698 still maps to current main: gateway/node exec approval snapshots can carry allowlist `source` metadata back to clients, while `exec.approvals.set` and `exec.approvals.node.set` validate against a schema that rejects unknown allowlist fields. The PR is a focused, recent fix attempt, but its latest direction preserves `source` to avoid durable-approval data loss, so it needs a coherent schema/transport/save-path design before it can be closed or merged.

## Best Possible Solution

Keep the PR open and request changes around one explicit contract: either redact `source` from transport snapshots and rehydrate/preserve durable metadata on SET, or make `source` an accepted exec approvals metadata field in the gateway protocol and update all clients/tests accordingly. Add gateway and node GET -> edit -> SET coverage that proves socket tokens stay redacted and durable allow-always entries survive the round trip.
