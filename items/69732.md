## Summary

Keep #69732 open. Current main still has the exec-approval transport/schema mismatch behind #69698: GET payloads can preserve `allowlist[].source`, while gateway SET validators reject unknown allowlist fields before normalization. The PR is recent, active, and not superseded; however, the provided final file list does not update the protocol schema or Control UI save contract, so maintainer review or revision is still needed rather than cleanup-closing it.

## Best Possible Solution

Keep #69732 open and require a narrow contract fix. The safest direction is to make exec-approval snapshots round-trip without data loss and without schema rejection, either by explicitly accepting `source?: "allow-always"` in the transport schema/client types or by redacting it from transport while rehydrating it from the current snapshot during SET. Add regression coverage for gateway and node GET-to-SET round trips with durable allow-always entries and socket token redaction.
