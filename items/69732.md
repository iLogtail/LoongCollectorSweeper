## Summary

Keep PR #69732 open. The linked Control UI round-trip problem remains real on current main: exec approvals snapshots can expose internal allowlist `source` metadata, the Control UI sends the loaded file back on save, and the gateway/node `exec.approvals.set` schemas still reject `source` as an unexpected allowlist property. The PR is a focused, recent bug-fix attempt, but its latest described direction preserves `source` in transport snapshots without changing the strict set schema, so it needs maintainer review/changes rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a narrow fix that makes `exec.approvals.get` snapshots safely writable through `exec.approvals.set` while preserving durable `allow-always` metadata. The likely maintainer path is to define a transport/edit DTO or set-side merge that accepts/rehydrates internal `source` by stable allowlist identity, add gateway and node-host regression coverage for GET -> SET round-trips, and keep socket token redaction intact.
