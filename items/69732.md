## Summary

Keep #69732 open. Current main still has the exec approvals round-trip mismatch: transport snapshots redact only the socket token, so allowlist `source` metadata can still reach clients, while `exec.approvals.set` still validates allowlist entries with `additionalProperties: false` and no `source` field. The PR is a recent, focused attempt tied to open issue #69698, but its final provided diff preserves `source` without changing the set schema or adding a rehydration path, so it is not clearly obsolete, implemented, or safely closeable.

## Best Possible Solution

Keep the PR open and request a tighter fix. The maintainer path should make GET -> edit -> SET valid for both gateway and node-host approvals while preserving durable `allow-always` metadata: either explicitly support `source` in the exec approvals transport schema and UI types, or redact it from client-editable payloads while rehydrating/preserving it on SET by stable id or pattern/argPattern. Add regression coverage for the Control UI-style round-trip and durable approval preservation.
