## Summary

Keep #43737 open. Current main has mitigations: the chat dropdown refresh asks for an unbounded session list, and the gateway hello snapshot exposes the canonical main session key. But the dropdown builder still only renders rows returned by `sessions.list` plus the current session, so the exact heartbeat-only/current-heartbeat state would still omit main. The linked gateway fix PR #43984 remains open and unmerged in the provided context.

## Best Possible Solution

Keep the issue open until maintainers either land a narrow UI fallback that always includes the resolved main session key in `resolveSessionOptionGroups`, or land/supersede #43984 with tests proving stale main plus active heartbeat stays selectable. Add regression coverage for a `sessions.list` result containing only heartbeat while the canonical main session exists in `hello.snapshot.sessionDefaults`.
