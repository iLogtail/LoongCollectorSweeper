## Summary

Keep open. Current main still hardcodes non-Azure custom-provider onboarding to `input: ["text"]`, and runtime transports drop image parts when the resolved model metadata does not include `image`. #72171 is a focused PR for the directly linked open bug #51869, with tests in its diff for new and re-onboarded custom models. It may still need review on heuristic coverage/Azure scope, but it is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR or a canonical follow-up open. The best fix belongs in `applyCustomApiConfig()`: detect or ask for image-capable custom models, set `input: ["text", "image"]` for new matching models, upgrade existing text-only entries on re-onboard without clobbering user-customized richer metadata, and cover both new and existing-config paths in `src/commands/onboard-custom-config.test.ts`. Maintainers should decide whether Azure custom deployment vision detection is part of this PR or a separate follow-up.
