## Summary

Keep open. Current main still hardcodes non-Azure custom-provider onboarding to `input: ["text"]`, and image payloads are filtered out at runtime unless model metadata includes `image`. #72171 remains a focused PR for the linked open bug #51869; the review comments identify refinements to the PR, not evidence that it is obsolete or already implemented.

## Best Possible Solution

Keep #72171 or a canonical follow-up open. The best fix belongs in `applyCustomApiConfig()`: detect or ask for image-capable custom models, set `input: ["text", "image"]` for new matching non-Azure models, upgrade existing text-only entries on re-onboard without clobbering richer user metadata, and cover both new and existing-config paths in `src/commands/onboard-custom-config.test.ts`. Maintainers should decide whether Azure custom deployment vision detection is part of this PR or a separate follow-up.
