## Summary

Keep open. Current `main` still preserves active run state for foreign `final` events, and this PR’s proposed change would likely regress busy-state and queue semantics by treating unrelated terminal events as completion of the active run.

## Evidence

- **Current main intentionally preserves active state on foreign finals:** `handleChatEvent` appends an assistant message from another run and returns `null` instead of clearing `chatRunId`/stream state or reporting a terminal state.
  - file: [ui/src/ui/controllers/chat.ts:667](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/ui/src/ui/controllers/chat.ts#L667)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '657,675p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Behavior is covered by an explicit unit test:** The current test suite asserts that a `final` payload from another run should append the message while keeping `chatRunId`, `chatStream`, and `chatStreamStartedAt` unchanged.
  - file: [ui/src/ui/controllers/chat.test.ts:112](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/ui/src/ui/controllers/chat.test.ts#L112)
  - command: `nl -ba ui/src/ui/controllers/chat.test.ts | sed -n '112,133p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Marking a foreign final as terminal would trigger terminal-event side effects:** `app-gateway` treats `state === "final"` as terminal, clears pending queue items for that run, flushes queued chat work, and may reload chat history.
  - file: [ui/src/ui/app-gateway.ts:406](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/ui/src/ui/app-gateway.ts#L406)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '406,447p'; nl -ba ui/src/ui/app-gateway.ts | sed -n '466,484p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Busy-state and queue gating depend on `chatRunId` remaining set:** The UI considers chat busy when `chatRunId` is present, and `flushChatQueue` exits early while busy. Clearing `chatRunId` on a foreign final would unlock queued sends before the active run actually finishes.
  - file: [ui/src/ui/app-chat.ts:63](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/ui/src/ui/app-chat.ts#L63)

- **Foreign `final` events are an expected gateway behavior:** `chat.inject` broadcasts `state: "final"` with a synthetic `runId` prefixed by `inject-`, so a terminal event from another run can legitimately arrive while the user's tracked run is still active.
  - file: [src/gateway/server-methods/chat.ts:2823](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/server-methods/chat.ts#L2823)
