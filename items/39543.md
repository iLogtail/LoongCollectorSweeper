## Summary

Keep #39543 open. Current main has a PDF analysis tool for paths, URLs, and managed media refs, but the requested local PDF upload workflow is still missing from the Control UI and Gateway chat attachment path.

## Best Possible Solution

Keep the issue open until OpenClaw either implements local PDF chat upload end to end or a maintainer explicitly narrows the product scope to path/ref-based PDF analysis. The likely implementation spans Control UI attachment acceptance/preview for `application/pdf`, Gateway document offload with preserved MIME metadata, and agent prompting/tool handoff so uploaded PDFs become `media://inbound/*` refs usable by the PDF tool or media-understanding pipeline, with tests and docs updated together.
