## Summary

Keep #39543 open. Current main implements a PDF tool for local paths, URLs, and managed inbound refs, but the requested local PDF upload-and-summarize workflow is still missing from the Control UI/Gateway chat attachment path.

## Best Possible Solution

Keep the issue open. The likely implementation belongs in the Control UI and Gateway chat attachment pipeline: accept `application/pdf` in file selection/drop, preserve filename/MIME/document metadata, render a document attachment state, serialize/offload PDFs as managed inbound media refs, and route those refs to the existing `pdf` tool with focused UI, Gateway, and PDF-tool integration coverage.
