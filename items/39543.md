## Summary

Keep #39543 open. Current main has a PDF analysis tool and managed-media PDF support at the agent/tool layer, but the requested local PDF upload-and-summarize workflow is still missing from the Control UI/WebChat attachment path and Gateway chat attachment parser.

## Best Possible Solution

Keep the issue open and scope it to a Control UI/WebChat local PDF upload workflow: allow PDF selection/drop where intended, preserve file name/MIME/document metadata in `ChatAttachment`, serialize document attachments distinctly from images, teach Gateway chat attachments to validate/offload PDFs as managed media refs, and ensure the agent can route those refs to the existing `pdf` tool for summarization with focused UI, Gateway, and PDF-tool integration tests.
