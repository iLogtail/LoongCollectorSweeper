## Summary

Keep #70354 open. Current main mitigates fresh canonical Linux installs by rendering `Restart=always`, but the reported `update.run`/SIGUSR1 path still exits `0` for supervised systemd restarts, and service audit does not clearly migrate existing/manual `Restart=on-failure` units. PR #70466 is still the direct unmerged fix for this issue.

## Best Possible Solution

Keep #70354 open until #70466 or an equivalent fix lands. The likely best fix is to split supervised restart exit behavior by supervisor: preserve launchd clean-exit behavior and spawned restart exit `0`, but make Linux/systemd supervised SIGUSR1/update restarts exit non-zero or otherwise explicitly support `Restart=on-failure`. Add a Linux/systemd supervised regression test and consider doctor/audit coverage for existing units still using `Restart=on-failure`.
