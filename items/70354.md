## Summary

Keep #70354 open. Current main improves the default systemd unit by using `Restart=always`, but the reported `update.run`/SIGUSR1 path still exits `0` for supervised restarts, and existing/manual `Restart=on-failure` units remain a real gap. The provided context’s PR #70466 is the right-looking fix, but it is still open, not shipped on main.

## Best Possible Solution

Keep #70354 open until main either lands the #70466-style fix or an equivalent one: systemd-supervised SIGUSR1/update restarts should exit non-zero when relying on `Restart=on-failure`, while preserving launchd’s clean-exit behavior. Add regression coverage for the Linux/systemd supervised exit code and consider auditing or repairing existing `Restart=on-failure` gateway units.
