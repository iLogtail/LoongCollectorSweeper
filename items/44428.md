## Summary

Keep #44428 open. Current main still contains the reported behavior: isolated cron direct delivery waits for active descendants with the full cron timeout and then suppresses delivery while descendants remain active, without checking `deliveryBestEffort`. The related PR #46160 was closed unmerged, and I did not find an equivalent regression test or later source change on main.

## Best Possible Solution

Keep the issue open and implement the narrow cron fix on current main: when `delivery.bestEffort` is true, isolated cron delivery should skip descendant-summary waiting and the active-descendant suppression return, then deliver the parent job output immediately. Add a regression test where `deliveryBestEffort: true`, `countActiveDescendantRuns() > 0`, and `waitForDescendantSubagentSummary` would otherwise block, asserting direct delivery is attempted without waiting.
