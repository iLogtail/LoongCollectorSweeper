## Summary

Keep open. Current main still does not wire Signal inbound messages into the shared status reaction lifecycle, while the PR is a focused, recent bug-fix attempt with a concrete linked reproduction in #72020. The right maintainer path is to keep the PR active and require the outstanding review fixes before merge, not close it as obsolete.

## Best Possible Solution

Keep #72025 open. The best path is to revise the PR so Signal mirrors the established Slack/Telegram/Discord lifecycle pattern: create the status reaction controller only when the ack/status gate passes, pass reply lifecycle callbacks into dispatchInboundMessage, route reactions through sendReactionSignal for direct and group messages, and finalize with restoreInitial/setDone/setError based on actual final delivery. Add tests for direct, group, mention-gated group, error, and queuedFinal:false paths; update Signal/config docs if Signal becomes a documented status-reaction surface.
