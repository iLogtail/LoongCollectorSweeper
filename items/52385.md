## Summary

Keep open. Current main partially addresses the report by making the initial `sessions_send` injection internal-only, and the issue discussion's Slack `replyToModeByChatType.channel="first"` workaround is a useful mitigation for top-level channel noise. But the central request is broader: agent-to-agent `sessions_send` traffic should not render on the human Slack surface at all. Current main still documents and tests an A2A announce path that asks the target agent for a visible announce and sends any non-`ANNOUNCE_SKIP` reply to the resolved channel/thread.

## Best Possible Solution

Keep #52385 open for a core sessions/A2A delivery policy fix. The likely path is to add a first-class no-surface mode for agent-to-agent `sessions_send` (either automatic for agent-to-agent sends or an explicit `silent`/`surface` parameter), preserve delivery into the target session transcript/context, and add Slack regression coverage for both thread-bound and channel-bound sessions proving no raw coordination or announce text reaches Slack unless the caller explicitly requests visible delivery.
