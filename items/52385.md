## Summary

Keep open. Current main does suppress the initial `sessions_send` injection from direct channel delivery, which addresses part of the report and aligns with the commenters' workaround direction for reducing Slack noise. But the requested behavior is broader: inter-agent coordination should not render on the Slack surface at all. Current main still has a documented A2A announce path that asks the target agent for a visible announce and sends any non-`ANNOUNCE_SKIP` reply to the resolved channel/thread, so the central no-surface-rendering request is not fully implemented.
