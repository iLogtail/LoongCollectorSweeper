## Summary

Keep open. Current `main` only exposes coarse session lifecycle state (`running|done|failed|killed|timeout`) and start/end timing in session rows; it does not implement per-session heartbeat polling, stalled/unresponsive detection, or Control UI live status indicators for model wait/tool execution gaps.

## Evidence

- **Session row schema is coarse only:** `GatewaySessionRow` exposes `status`, `startedAt`, `endedAt`, and `runtimeMs`, with `SessionRunStatus` limited to `running|done|failed|killed|timeout`. There is no field for heartbeat state, last token time, active tool, or stalled/unresponsive markers.
  - file: [src/gateway/session-utils.types.ts:24](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/gateway/session-utils.types.ts#L24)

  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Lifecycle updates only track start/end/error:** Session lifecycle derivation maps events to `start`, `end`, or `error`, setting `status: "running"` on start and a terminal status on end/error. No intermediate waiting/tool/streaming heartbeat state is persisted.
