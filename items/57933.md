## Summary

Current `main` has chat-level streaming/tool activity and coarse session lifecycle fields, but it does not implement per-session heartbeat polling, stall/unresponsive detection, or live session status indicators in the Control UI session list.

## Evidence

- **Sessions view has no status/heartbeat column:** The Sessions table renders `Key`, `Label`, `Kind`, `Updated`, `Tokens`, `Compaction`, `Thinking`, `Fast`, `Verbose`, and `Reasoning`, with no live status, heartbeat, stall, or unresponsive indicator.
  - file: [ui/src/ui/views/sessions.ts:380](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/views/sessions.ts#L380)
  - command: `nl -ba ui/src/ui/views/sessions.ts | sed -n '330,560p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Gateway session rows only expose coarse lifecycle status:** `GatewaySessionRow` includes `status`, `startedAt`, `endedAt`, and `runtimeMs`, but no per-turn heartbeat state like awaiting model, streaming, tool exec, tool wait, stalled, or unresponsive.
  - file: [src/gateway/session-utils.types.ts:16](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/session-utils.types.ts#L16)
  - command: `nl -ba src/gateway/session-utils.types.ts | sed -n '1,90p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Control UI reloads sessions on change; no heartbeat/stall logic:** On `sessions.changed`, the UI just calls `loadSessions(...)`. There is no session-specific heartbeat timer, stall threshold handling, or dead-session detection in this path.
  - file: [ui/src/ui/app-gateway.ts:570](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/app-gateway.ts#L570)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '560,585p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Documented Control UI surface does not include per-session health polling:** The Control UI docs describe Sessions as `list + per-session model/thinking/fast/verbose/trace/reasoning overrides`, while the only heartbeat RPC surfaced in UI code is the global `last-heartbeat` debug snapshot. A repo search found no `sessions.heartbeat` method or equivalent endpoint.

## Risks / Open Questions

- There is partial overlap already: chat has live streaming/tool-output cues, so future work should avoid duplicating chat-only progress signals while still adding cross-session status.
- Because coarse `status`/timing fields already exist in gateway session rows, the eventual fix may be smaller than the issue's full heartbeat proposal, but the requested Control UI behavior is not present today.
