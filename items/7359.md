## Summary

Keep #7359 open. Current main still separates Slack DMs/main sessions from Slack channel sessions, mirrors outbound `message` tool sends only into the destination Slack session, and default session-tool visibility blocks a DM/main session from reading an unrelated same-agent channel transcript. The linked thread-context issues address narrower Slack threading/session bugs, not the requested cross-context self-activity recall. The existing 2026-04-26 maintainer/Codex issue comment reached the same conclusion, and I found no later current-main implementation of a scoped Slack self-activity ledger or bridge.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path rather than broad cross-session visibility. The maintainer-safe shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack account, destination channel/thread, and sent Slack message id, exposed from DM context only for the agent's own outbound activity. Add regression coverage for sending from a Slack DM/main session to a Slack channel and then asking from the DM whether the agent sent that message.
