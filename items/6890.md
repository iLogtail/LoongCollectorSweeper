## Summary

Keep open. Current main still does not implement Ralph Loop or an agent-scoped max Ralph iteration/default loop setting. The agent config type and strict schema have no `loop` or max-iteration field, resolved agent config cannot propagate one, and the only adjacent behavior is strict-agentic planning-only retry plus tool-loop detection, not a STOP-driven repeated same-prompt loop.

## Best Possible Solution

Keep the issue open. If accepted, implement an OpenClaw-native loop/default surface across `AgentConfig`, strict zod schemas, schema help/labels/generated reference, docs, and `resolveAgentConfig()`, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add tests for default/global behavior, per-agent override, higher-precedence override, disabled, finite, and unlimited semantics, and keep it distinct from strict-agentic planning-only retry and tool-loop detection unless maintainers intentionally merge those concepts.
