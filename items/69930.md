## Summary

Current main already fixes the user-visible lazy subcommand help problem by registering the real primary command before Commander parses `--help`. PR #69930's exact `placeholder.helpOption(false)` hardening is not present, but the linked #59449 behavior is already solved and shipped in `v2026.4.24`, so this PR is obsolete as a bug fix.

## Best Possible Solution

Close PR #69930 as implemented by the current shipped primary-command preregistration path. Keep the shipped `runCli` behavior and its tests; open a narrower maintainer-owned cleanup only if the team wants extra lower-level `registerLazyCommand` hardening beyond the fixed CLI behavior.

## Evidence

The inline comment states this keeps help and command parsing correct with lazy command registration.
