## Summary

Current main already handles this failure mode: incomplete trailing `[[...` directive fragments are stripped before final assistant parsing, and complete reply tags are converted into reply metadata before outbound delivery, preventing Telegram-visible leakage of internal reply markers.

## Evidence

- **Incomplete reply/audio directive tails are explicitly buffered and dropped on final parse:** `splitTrailingDirective` treats an unclosed `[[...` tail as buffered directive content and, in final mode, returns only the safe text prefix.
  - file: [src/auto-reply/reply/streaming-directives.ts:43](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/reply/streaming-directives.ts#L43)

## Risks / Open Questions

- I did not run a live Telegram/Codex reproduction in this read-only review; the conclusion is based on current code paths, shipped tag contents, and existing regression tests.
- The original report mixed provider `server_error` with reply corruption. This close decision covers the leaked/truncated directive-delivery path, not upstream provider outages themselves.

## Close Comment

Closing as implemented on main. Codex review verified that the current/released code already suppresses this class of leaked Telegram reply-marker corruption: unclosed trailing `[[...` directive fragments are buffered and dropped in `src/auto-reply/reply/streaming-directives.ts:43`, final assistant text is sanitized through `splitTrailingDirective(..., { final: true })` before reply parsing in `src/agents/pi-embedded-subscribe.handlers.messages.ts:651`, and outbound delivery converts complete `[[reply_to_*]]` tags into metadata instead of sending them as plain text in `src/infra/outbound/payloads.ts:136`. The shipped `v2026.4.22` release already contains this path (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), which is newer than the reported `2026.3.13` build. If you still hit a fresh corruption case on a current build, please open a new issue with a current transcript/log slice and the exact leaked text.
