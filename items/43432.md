## Summary

Keep open. Current main has adjacent memory-flush and cache machinery, but it does not implement the three requested config semantics: no `hardThresholdTokens`, no `compaction.strategy: "preserve-priority"`, and no `memorySearch.cache.ttlSeconds`. Related reports address narrower flush-trigger failures, but they do not supersede this broader durability feature request.

## Best Possible Solution

Keep the issue open for maintainer product/design review. A complete solution likely needs a core or bundled-memory design that either adds the requested config keys with tests/docs, or deliberately maps them to existing supported mechanisms: a true hard token-triggered flush path, a documented priority-preservation compaction contract or provider API, and an age-based embedding-cache invalidation option if content-hash invalidation is not sufficient.
