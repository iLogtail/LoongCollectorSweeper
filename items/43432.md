## Summary

Keep open. Current main has related durability work, including transcript-size forced flush and content-hash based embedding invalidation, but the specific requested capabilities are not implemented: no token hard flush threshold, no preserve-priority compaction strategy, and no embedding cache TTL config.

## Evidence

- **Compaction schema lacks requested hard threshold and strategy:** The strict agents.defaults.compaction schema only accepts mode default|safeguard plus existing fields; memoryFlush accepts enabled, softThresholdTokens, forceFlushTranscriptBytes, prompt, and systemPrompt. There is no memoryFlush.hardThresholdTokens or compaction.strategy field.
