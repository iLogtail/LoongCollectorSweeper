## Summary

Keep open. Current main still skips CLI providers in both preflight compaction and memory-flush paths, the regression test still asserts CLI memory flush is skipped, and the user-facing compaction reference still documents CLI backends as excluded. The related PR #71081 was closed unmerged in favor of this discussion, while the existing issue comment argues the gate removal is safe and worth accepting, so this remains an active core runtime/product decision rather than conservative cleanup.

## Best Possible Solution

Keep #71084 open for maintainer direction. If accepted, implement this in the core auto-reply memory lifecycle by allowing CLI-backed agents through the same threshold checks, preserving heartbeat and read-only workspace guards, proving the flush/compaction execution path is correct for CLI providers, and updating tests plus compaction docs. If rejected, close after documenting the CLI skip as an intentional design choice and pointing users to the existing CLI transcript compaction behavior.
