## Summary

Current main still explicitly skips CLI-backed providers for both preflight compaction and memory flush, and the docs/test suite codify that behavior. The related PR #71081 was closed unmerged in favor of this discussion, so the reporter's actual question remains unresolved and needs maintainer product judgment rather than conservative cleanup.

## Best Possible Solution

Keep the issue open for maintainer direction. If CLI-backed agents should participate, land a focused PR that removes the `isCli` gates in `agent-runner-memory.ts`, updates the CLI-provider test from skip to allow, and adds/keeps coverage for both memory flush and preflight compaction with configured CLI backends. If the skip is deliberate, document the rationale and close after answering the reporter.
