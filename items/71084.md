## Summary

Keep open. Current `main` still explicitly skips both preflight compaction and silent memory flush for CLI-backed providers, and the new unit tests codify that skip. At the same time, current docs and session accounting treat CLI backends as persistent sessions with context snapshots, so the report matches current behavior and raises a real design/product question rather than a stale or incoherent one.

## Evidence

- **Preflight compaction still short-circuits for CLI providers:** `runPreflightCompactionIfNeeded()` computes `isCli` and returns early when `params.isHeartbeat || isCli`, so CLI-backed runs never enter the preflight compaction path on current `main`.
