## Summary

Keep open. Current main still has the reported /tools mismatch: the command inventory path only renders the result of resolveEffectiveToolInventory, while the real run path separately materializes configured MCP tools and appends them to effectiveTools. The PR targets a real remaining bug, but it is not merge-ready as-is because it also carries unrelated install-package-dir peer-dependency changes with a test/implementation mismatch noted in review.

## Best Possible Solution

Keep the PR open for maintainer or contributor follow-up, but require a focused revision: drop the install-package-dir changes, add MCP tools to /tools and /tools verbose through the existing session MCP runtime path, mark them as an MCP inventory source/group, preserve runtime reserved-name collision behavior, filter through applyFinalEffectiveToolPolicy, and add regression tests for compact and verbose MCP inventory output. If maintainers prefer a cleaner history, supersede this with a fresh narrow PR linked to #70874.
