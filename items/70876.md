## Summary

Current main still does not include configured MCP server tools in `/tools`; the command builds its inventory from the core/plugin/channel inventory only, while the actual run path separately materializes and policy-filters bundle MCP tools. The PR remains relevant to #70874, but should be cleaned up before merge because it also carries unrelated plugin-install peer dependency changes noted in review discussion.

## Best Possible Solution

Keep the PR open. The best path is to make the change MCP-focused or split the install-package peer dependency work into its own PR, then implement `/tools` inventory against the existing session MCP runtime without read-only side effects, pass the real session id and reserved tool names, apply the same final tool policy as `attempt.ts`, and add regression tests for MCP inventory, policy-denied MCP tools, and collision-adjusted tool names.
