## Summary

Keep #71382 open. Current main still lacks a WhatsApp channel/account `readOnly` config surface and does not enforce a transport-level outbound gate. The PR is the canonical implementation path for #71166, but the discussion identifies concrete follow-ups that should be addressed before merge.

## Best Possible Solution

Keep #71382 open. The best maintainer path is to finish the WhatsApp transport-level `readOnly` implementation on this PR or a replacement patch, preserving channel/account config support and hard outbound gating for sends, polls, reactions, composing, read receipts, presence, and pairing replies. The follow-up should also keep pairing request creation possible in read-only mode while suppressing the outbound pairing message.
