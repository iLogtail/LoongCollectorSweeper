## Summary

Keep #8321 open. Current main still lacks per-job `disableCache` or `contextPruning` on cron jobs, and isolated cron execution still derives pruning/cache behavior from the resolved agent defaults rather than job metadata. The related PR #8341 is closed unmerged in the provided GitHub context, and #24597 covers direct script/no-LLM cron execution rather than this per-job cache/pruning override.

## Best Possible Solution

Implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override, using the existing `mode: "off" | "cache-ttl"` contract. Thread it through cron create/update/list schemas, normalization/store, CLI/UI/macOS and agent-tool surfaces, then merge it into `cfgWithAgentDefaults` before isolated cron calls `runEmbeddedPiAgent`. Add regression coverage proving job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Keep direct script/no-LLM cron execution tracked separately.
