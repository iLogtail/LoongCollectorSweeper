## Summary

Keep open. Current main still has per-chunk compaction summarization retries with no shared circuit breaker or consecutive-failure budget, and the linked circuit-breaker PRs #58846 and #58857 were closed unmerged. There are partial mitigations for timeout-classed failures, but the requested persistent-unavailable/rate-limit/misconfiguration guard is not implemented.

## Best Possible Solution

Keep the issue open and implement a targeted compaction retry budget in the current compaction owner path: thread shared consecutive-failure state through summarizeInStages()/summarizeWithFallback()/summarizeChunks(), skip further LLM summarization once the budget opens, preserve previous summary or structured fallback, and add regression tests for multi-split persistent failure, AbortError exclusion, and recovery after cooldown. If maintainers prefer consolidation, explicitly supersede it under the broader open compaction circuit-breaker issue rather than closing it as already fixed.
