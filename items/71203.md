## Summary

Keep open. Current main still refreshes `models.json` only through the default-agent primary-model warmup path, and that path returns before any cache regeneration when no explicit default primary is configured. It does not enumerate configured non-default agent dirs at startup, while the PR targets that remaining gap from #71192 and has recent author follow-up rather than being obsolete cleanup.

## Best Possible Solution

Keep the PR open for review. The best product/code path is a startup helper that refreshes the unique configured agent dirs independently of primary-model prewarm, with regression coverage for non-default agents and the no-default-primary case, while preserving the selected default/override agent-dir semantics noted in review.
