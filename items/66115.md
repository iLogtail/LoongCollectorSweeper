## Summary

Keep open. Current `main` intentionally defines `GET /v1/models` as an agent-target list (`openclaw`, `openclaw/default`, `openclaw/<agentId>`), not a provider/model catalog, so the requested behavior is not implemented. There is a separate gateway `models.list` RPC that returns the runtime-allowed model catalog, which makes the request coherent and still actionable rather than stale or incoherent.

## Risks / Open Questions

- Closing as implemented or cannot-reproduce would contradict current docs, code, and tests: `/v1/models` still returns agent ids on current `main`.
- Changing `/v1/models` to raw provider/model ids would be a compatibility change because the documented OpenAI HTTP contract currently treats the `model` field as an OpenClaw agent target.
- There is an existing workaround surface (`models.list` RPC), but that does not satisfy the standard OpenAI HTTP discovery use case described in the issue for remote third-party clients.
