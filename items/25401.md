## Summary

Keep #25401 open. Current main still lacks a provider-level `models.providers.<id>.enabled` public contract and does not use such a flag to filter models.json planning, fallback candidates, configured model catalog entries, or `openclaw models status` output. The related implementation PR #25415 was closed unmerged, so it remains useful reference work rather than shipped behavior.

## Best Possible Solution

Keep the issue open until OpenClaw core adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults omitted values to enabled, excludes disabled providers from generated models.json and model/fallback candidate selection, preserves the config block for re-enable, and displays disabled state in `openclaw models status` JSON and rich output. PR #25415 is useful prior art but was closed unmerged.
