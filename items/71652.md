## Summary

Keep PR #71652 open. Current main still does not implement the PR's host-pressure refusal, force override, or heavy-check temp-directory routing for local tsgo. The PR is a plausible scripts fix, but the review thread identifies a real correctness concern around checking `/tmp` after redirecting tsgo temp files, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep PR #71652 open. The best maintainer path is to revise the PR so the pressure guard checks the filesystem backing the effective heavy-check temp directory, preserves the sparse guard and existing heavy-check lock behavior, keeps the explicit force override, and retains focused regression tests for pressure refusal, force bypass, and temp-dir routing.
