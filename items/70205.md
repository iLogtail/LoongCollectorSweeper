## Summary

Keep open. Current main still does not implement the PR's central bug fix: `sessions.usage` scans configured gateway agents, but `usage.cost` still loads one unscoped transcript directory and caches only by date range. The related prior PR #42063 is closed unmerged, and #13219 tracks a broader dedicated usage-log feature rather than this narrower aggregate bug. This PR is not obsolete, though it needs maintainer review/cleanup because the branch includes unrelated touched surfaces.

## Best Possible Solution

Keep the PR open, but ask for maintainer review focused on narrowing or splitting the unrelated branch changes. The best product/code path is to implement the gateway fix in `src/gateway/server-methods/usage.ts`: resolve the cost-usage agent scope from `listAgentsForGateway`, validate explicit `agentId` requests if supported, merge per-agent `CostUsageSummary` daily/totals, include the resolved scope in the cache key, and keep regression coverage in gateway tests plus the UI date-label test. Do not treat #13219 as a substitute; it is a broader future usage-log proposal.
