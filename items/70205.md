## Summary

Keep open. Current main still has the reported gateway bug: `usage.cost` uses a single unscoped transcript-cost summary and cache key, while `sessions.usage` already discovers configured agents. PR #70205 addresses a real remaining issue, but its branch also contains unrelated changes outside the stated gateway/UI usage fix, so the best maintainer path is to narrow or cherry-pick it rather than close it as obsolete.

## Best Possible Solution

Keep the item open, but ask for a rebased/narrowed PR or maintainer cherry-pick that only changes the gateway usage-cost aggregation, cache scoping, agent validation/error handling, and UI date-only label behavior. The implementation should resolve configured gateway agent ids for unscoped `usage.cost`, preserve explicit `agentId` scoping, merge successful per-agent summaries, include agent scope in the cache key, add gateway/UI regression tests, and leave the broader dedicated `usage.jsonl` proposal in #13219 separate.
