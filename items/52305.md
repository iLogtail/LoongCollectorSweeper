## Summary

Keep open. Current main has session-scoped system-event storage and internal exec/node completion paths now pass sessionKey into heartbeat wake, but the user-facing external-runner path described in #52305 is still not explicitly session-targeted: `openclaw system event` exposes no `--session-key`, the gateway `wake` handler forwards only `mode` and `text`, and the separate `system-event` RPC still targets the configured main session. The provided discussion also shows this was still reported after those internal improvements, with #50818 open as related but not merged/sufficient for the CLI boundary.
