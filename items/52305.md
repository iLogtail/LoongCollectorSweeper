## Summary

Keep open. Current main has session-scoped system-event storage and internal exec/node completion wakes can carry a sessionKey, but the external-runner path reported here is still not reliably session-targeted: `openclaw system event` has no `--session-key`, the gateway `wake` handler forwards only `mode` and `text`, cron wake requests enqueue/wake without a target, and the documented CLI behavior is still main-session only. Related PR #50818 remains useful/related but does not close the external CLI boundary on current main.

## Best Possible Solution

Keep the issue open and fix the external completion callback boundary directly: add a documented, validated session-targeting path for `openclaw system event`/gateway `wake` (or a first-class async completion notification API), propagate that target through cron wake and heartbeat wake, preserve existing internal exec session-scoped behavior, and add regression tests for CLI payload construction, wake RPC handling, and delivery to the originating channel/session.
