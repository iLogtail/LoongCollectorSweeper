## Summary

Keep open. Current `main` intentionally removed the legacy Chrome Extension Relay in `v2026.3.22`, still rejects `driver: "extension"`, and only offers migration to Chrome MCP `existing-session` / `user`. The documented `existing-session` limits do not establish parity with the requested relay behavior, so this is not implemented on `main`. The issue is also too recent for `stale_insufficient_info`, and it concerns built-in browser transport behavior rather than a ClawHub skill/plugin.
