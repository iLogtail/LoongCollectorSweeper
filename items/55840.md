## Summary

Keep #55840 open. Current main provides a documented Chrome MCP `existing-session` migration path for real signed-in Chrome tabs, but it does not restore the old Chrome Extension Relay optional driver requested here. The extension relay is intentionally removed, `driver: "extension"` is rejected or migrated away, and current docs acknowledge existing-session limitations, so closing this as already implemented would overstate the replacement.

## Best Possible Solution

Leave the issue open for maintainer product judgment. If OpenClaw wants this capability again, the best path is not an implicit core rollback: define whether a separate browser plugin/ClawHub package can own the extension relay using the current plugin surfaces, or explicitly approve a core browser-plugin/security design for a new optional extension driver with docs, migration behavior, package assets, and tests. If maintainers consider Chrome MCP `existing-session` the final answer, close manually as a documented product decision rather than as already implemented.

## Risks / Open Questions

- Chrome MCP existing-session solves part of the user problem, but it is not the same architecture as the old extension relay and current docs list feature limits.
- Reintroducing a Chrome extension relay with `chrome.debugger` access would be a product and security decision, not a conservative cleanup close.
- The old relay had many related bug/security reports in the provided context, so restoring it would need explicit owner review and regression coverage.
