## Summary

Keep #52510 open. Current main still has the reported dangerous cleanup-hint shape: macOS service scanning can still treat unrelated LaunchAgents that mention `openclaw` as extra gateway-like services, and doctor/status still render hard-coded cleanup commands for the canonical active gateway rather than the detected plist. The linked PR #52515 is open/unmerged in the provided context, so it is not shipped evidence for closure.

## Best Possible Solution

Keep the issue open and fix the gateway-service audit path on main: make Darwin LaunchAgent detection parse the plist label/program arguments and only flag real legacy or parallel gateway services; make cleanup hints target the detected stale service or suppress them for the active canonical gateway; add regression tests for an active `ai.openclaw.gateway.plist` and an unrelated sibling LaunchAgent such as `openclaw backup`. Then separately verify the `exec:` SecretRef doctor behavior and either include a focused warning-only fix or split it to a dedicated issue/PR.
