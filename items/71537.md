## Summary

Keep PR #71537 open. Current main only has a narrow session-memory fallback from an empty live transcript to same-base `.jsonl.reset.*` siblings, and the `session-logs` skill still documents active `*.jsonl` searches. The broader PR behavior, especially resolving reset-only archives from `findPreviousSessionFile` when the live transcript is absent, is not implemented on main. The linked #60409 is complementary gateway RPC work, not a replacement for this hook/docs change.

## Best Possible Solution

Keep #71537 open. The best path is to land a scoped session-memory fix in `src/hooks/bundled/session-memory/transcript.ts` with regression coverage for reset-only canonical/topic/archive ordering, plus the `skills/session-logs/SKILL.md` archived transcript search guidance. Treat #60409 as a complementary gateway RPC/UI history fallback, not the canonical replacement for this PR.
