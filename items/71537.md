## Summary

Keep #71537 open. Current main has adjacent reset-archive handling, but it does not fully cover this PR: `session-memory` still cannot discover a reset archive when only archived copies remain and the prior live transcript path is missing, and the bundled `session-logs` skill still teaches active-only `*.jsonl` searches. The related #60409 is complementary gateway history work, not a duplicate of this hook/skill surface.

## Best Possible Solution

Keep #71537 open for normal maintainer review. The best path is to land the non-duplicative hook recovery and `session-logs` documentation updates after review, while letting #60409 proceed separately for gateway history/RPC archive fallback.
