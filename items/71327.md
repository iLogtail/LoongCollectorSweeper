## Summary

Keep PR #71327 open. Current main does not enforce LINE `dmPolicy: "open"` requiring an explicit `allowFrom: ["*"]`, while LINE runtime still treats `dmPolicy: "open"` as allowing any DM sender. The PR is a focused, plausible fix with tests; it is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused LINE schema guard plus colocated schema tests, using the shared `requireOpenAllowFrom` behavior and preserving the existing account inheritance contract where intended; add docs/changelog migration text if maintainers treat the stricter validation as user-visible.

## Risks / Open Questions

- The fix is intentionally not fully backward compatible for hand-written LINE configs that currently use `dmPolicy: "open"` without `allowFrom: ["*"]`; docs/changelog or migration wording may be needed before landing.
- LINE account configs inherit top-level fields at runtime, so review should confirm the final schema handles account-level effective `allowFrom` consistently with the resolver and other multi-account channels.
