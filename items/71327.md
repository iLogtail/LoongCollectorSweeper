## Summary

Keep PR #71327 open. Current main still has the LINE schema gap the PR describes: `dmPolicy: "open"` is accepted by the LINE config schema without requiring `allowFrom: ["*"]`, while the LINE runtime treats `open` as accepting all DM senders. The provided PR discussion is useful and favorable: Greptile summarized the fix as reusing the shared guard and only noted minor P2 coverage/message issues, and the provided updated PR file list shows added tests for the missing top-level and account-level cases. This is not implemented on main, not a duplicate of the provided related items, and is a plausible focused LINE-owned hardening PR that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix belongs in the LINE plugin schema, reusing the shared `requireOpenAllowFrom` primitive and adding focused LINE schema tests for top-level and account-level open DM configs, while preserving the existing setup wizard behavior that writes `allowFrom: ["*"]`. Before merge, align the account-level rule with LINE's inherited config semantics and decide whether docs/changelog migration notes are needed.
