## Summary

Keep PR #59522 open. Current main still contains the stale Himalaya command and configuration examples that the PR is trying to fix, and upstream Himalaya v1.1 source supports the proposed command-shape corrections. The PR is not ready to merge unchanged because the existing review identified environment-specific wording in the diff, but that is a fixup request rather than a conservative close reason.

## Best Possible Solution

Keep the PR open and ask for a small docs fixup: preserve the v1.1 CLI syntax corrections and provider guidance, but rewrite the worktree-local phrases generically. After that, maintainers can decide whether maintaining this existing bundled skill remains appropriate or whether a later follow-up should move Himalaya skill maintenance to ClawHub.
