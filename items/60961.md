## Summary

Keep #60961 open. Current main still routes sandboxed write-tool calls through the OpenShell mirror filesystem bridge, and the mirror backend still uploads the full local file path directly to `openshell sandbox upload` with the remote parent directory. The flat staging logic the issue identifies exists only for full workspace uploads, not for `syncLocalPathToRemote`, so the reported subdirectory write failure is still a plausible current bug.

## Best Possible Solution

Keep the issue open and fix `extensions/openshell/src/backend.ts` so mirror-mode single-file sync stages the file under a flat temporary directory using its basename before invoking `openshell sandbox upload`, then add focused OpenShell backend coverage for writing into an already-existing remote subdirectory. The existing full-workspace `uploadPathToRemote` staging path is the right pattern to reuse or factor.
