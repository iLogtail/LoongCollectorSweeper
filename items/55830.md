## Summary

Keep open. Current `main` does not implement the PR's per-plugin load-status lines or consolidated startup summary; gateway startup still only emits plugin diagnostics when errors/info diagnostics exist.

## Evidence

- **Current bootstrap only logs diagnostics:** `prepareGatewayPluginLoad()` calls `logGatewayPluginDiagnostics()` only when `loaded.pluginRegistry.diagnostics.length > 0`. There is no `logPluginLoadSummary()` or equivalent success/disabled summary logging on `main`.
