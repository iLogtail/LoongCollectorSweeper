## Summary

Keep open. Current main has partial plugin startup visibility through the compact `ready (...)` line and failure-only warnings, but it does not implement this PR's per-plugin loaded/disabled/error log lines or consolidated loaded/disabled/errored startup/config-reload summary. The PR discussion also shows the bot review defects were addressed by the branch's follow-up commit, so this remains salvageable work rather than conservative cleanup material.

## Best Possible Solution

Keep the PR open for maintainer judgment or contributor refresh. The best path is to rebase it onto current main, preserve the `logDiagnostics` suppression behavior for deferred reloads, keep error values parser-friendly, and add targeted gateway bootstrap tests that cover loaded, disabled, errored, summary, and config/deferred reload behavior. If maintainers decide the compact `ready (...)` line is sufficient product behavior, that should be an explicit maintainer close rather than automated cleanup.
