## Summary

Keep open. Current main has an LLM idle-timeout watchdog that may mitigate some silent streaming cases, but the requested per-fallback-candidate attempt budget is not implemented in `runWithModelFallback`: each candidate still awaits the supplied `run()` sequentially with no `attemptTimeoutMs`/`Promise.race` cap, and there is no `fallbackAttemptTimeoutSeconds` config. The related PR #46193 proposed the exact fix but was closed unmerged, so it does not supersede the issue.
