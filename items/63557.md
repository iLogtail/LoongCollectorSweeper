## Summary

Keep PR #63557 open. Current main still does not implement the PR's central user-facing behavior: there is no `gateway-tool-audit.jsonl` helper/file, `/tools/invoke` executes tools without appending an audit record, OpenResponses ingress does not thread any gateway audit context into agent runs, and existing diagnostic tool events only carry summarized params rather than a durable append-only audit record with redacted effective args. The linked issue #13131 is closed, but the requested gateway-level audit trail is not present on main, so this PR is not obsolete or implemented-on-main. The PR does need maintainer review/rebase because it carries unrelated test-fix drift and has unresolved hardening/performance review comments.

## Best Possible Solution

Keep the PR open but require a maintainer-guided rebase/scope cleanup: retain the core gateway audit implementation, remove unrelated shared CI/test-fix drift, verify the audit sink against current main, and address the remaining file-mode, async/queued write, and correlation-id review comments. If maintainers prefer a different design, open a narrower canonical follow-up for #13131 before closing this PR.
