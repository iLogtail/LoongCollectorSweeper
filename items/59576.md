## Summary

Keep #59576 open. Current main still resolves Mattermost inbound media only from the WebSocket post's existing `file_ids`; when that array is empty, the resolver returns no media and attachment-only messages can be dropped. The related fix PR #59591 was closed unmerged, so it is not a shipped canonical fix, and the later commenter reported the same large-file symptom.

## Best Possible Solution

Implement and test the Mattermost plugin fix on main: add a typed GET `/api/v4/posts/{post_id}` helper, have the monitor retry or poll briefly when a non-merged WebSocket post arrives with empty/null `file_ids`, then pass the refreshed IDs into `resolveMattermostMedia`. Add focused tests for delayed `file_ids`, attachment-only messages, and the no-attachment/text-only path so the workaround does not regress.
