## Summary

Keep #61716 open. Current main supports model metadata in config and has some provider-specific/default handling, but the generic custom/OpenAI-compatible onboarding flow still does not prompt for context window, max output tokens, or modalities. The related #64213 is a runtime image-capability bug, not a replacement tracker for onboarding UX.

## Best Possible Solution

Keep the issue open and implement this in the core/custom-provider onboarding surface: add validated optional prompts and matching non-interactive flags for `models.providers.<id>.models[]` metadata (`contextWindow`, `maxTokens`, and `input`/modalities), preserve current defaults and existing user-customized model entries, update `docs/cli/onboard.md` plus custom-provider docs, and add focused tests for interactive and non-interactive `custom-api-key` plus shared self-hosted OpenAI-compatible helpers. Treat #64213 as the separate runtime fallback/capability follow-up.
