## Summary

Current main still appears to pass OpenClaw's synthetic Feishu reaction message IDs into API-facing reply/typing/media paths. The issue is a real bundled Feishu plugin bug with active unmerged fix PR context, so it should remain open.

## Best Possible Solution

Keep #34528 open until a Feishu plugin fix lands on main. The likely path is a narrow message-ID normalizer for OpenClaw's `:reaction:<emoji>:<uuid>` suffix, applied at the reply target/API boundary before Feishu API calls, with regression coverage for text/card replies, typing indicators, streaming cards, media replies, and reaction API paths as applicable. PR #39558 appears to be the active candidate to review or supersede.
