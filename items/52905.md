## Summary

Keep open. Current main mitigates the normal `openclaw status` path by making status scans use a presence-only gateway probe, but the broader reported `probeGateway` / `openclaw gateway probe` timeout path is still present: the dedicated gateway probe still defaults to full detail probing, which includes the expensive `status` RPC under a 3000 ms default budget, and that RPC still scans all configured agents/session stores.
