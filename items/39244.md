## Summary

Keep PR #39244 open. Current main still does not implement the requested `tools.exec.nsSandbox` / bubblewrap namespace layer for safeBins-approved gateway exec commands. Main has safeBins argv/path-policy hardening and full sandbox backends, but there is no safeBins-only Linux filesystem namespace wrapper, no config/schema surface for `nsSandbox`, and no runtime bwrap spawn path.

## Best Possible Solution

Keep PR #39244 open for maintainer/security judgment. The best path is a narrowed, rebased optional Linux bwrap layer for safeBins-approved gateway exec, wired through config types, strict schema/help/generated docs, per-agent defaults, gateway approval/follow-up execution, runtime spawning, and focused regression tests. If maintainers reject core bwrap support, close only with an explicit product/security rationale and a documented alternative using current sandbox backends.
