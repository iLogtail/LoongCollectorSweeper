## Summary

Keep open. Current main still does not implement the PR's `tools.exec.nsSandbox` config or a bubblewrap wrapper path for safeBins-approved exec commands. Main has meaningful safeBins path hardening now, which partially reduces the original absolute-path/path-shadowing motivation, but it does not provide the PR's kernel-level filesystem isolation. This is plausible core exec/security hardening and needs maintainer judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/security judgment. The best path is either a narrowed, rebased core implementation of optional bubblewrap isolation for safeBins-approved gateway exec, with config types/schema/docs/tests and resolved review issues, or an explicit maintainer decision that current safeBins hardening plus the existing sandbox backends are sufficient and namespace isolation should not be added.
