## Summary

Keep open. Current main still does not implement the proposed bubblewrap namespace sandbox, `tools.exec.nsSandbox` config surface, or bwrap propagation through the exec runtime. The existing safeBins path does have resolved-path/trusted-dir hardening, but that only partially addresses the PR motivation and does not replace the requested filesystem namespace isolation.

## Best Possible Solution

Continue tracking PR #39244 until a maintainer either accepts a focused implementation of Linux bwrap namespace isolation for safeBins, decides the feature does not belong in OpenClaw core, or asks for a narrower follow-up. The likely core path is to add a validated exec config surface, thread sandbox parameters through gateway approval and runtime execution consistently, document the behavior, and resolve the existing review concerns before merge.
