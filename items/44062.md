## Summary

Keep open. Current main does not implement the PR's configured channel/account `mediaLocalRoots` merge; it only builds default + agent-scoped roots and can widen from concrete media source parent directories when policy allows. Related #44021 shipped Mattermost agent-scoped reply-root propagation, but it does not supersede this generic configured-root behavior. The PR also needs rework against current main because review feedback identified real config-schema/account-resolution blockers.

## Best Possible Solution

Keep the PR open, but rework it against current main. The best fix is a shared config-aware media-root resolver integrated into `resolveAgentScopedOutboundMediaAccess`/outbound delivery, merging default roots, agent roots, and configured channel/account roots with normalized/default/case-insensitive account resolution. Update the real Telegram/Feishu schemas/types/metadata if those channels are supported, and add regression tests for parsed config plus `deliverOutboundPayloads`, plugin message actions, and outbound send service paths.

## Risks / Open Questions

- Applying the old PR shape directly may conflict with current main's newer `resolveAgentScopedOutboundMediaAccess` and source-widening flow.
- A fix that only threads raw `accountId` can still miss default-account and case-insensitive account resolution paths.
- A fix that only updates in-memory tests can remain unusable if strict channel schemas reject `mediaLocalRoots` in real config.
