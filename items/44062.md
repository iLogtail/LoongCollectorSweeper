## Summary

Keep open. Current `main` does not implement the PR's advertised fix: outbound local-media roots are still derived from default roots plus optional agent workspace only, without merging channel/account `mediaLocalRoots`. The same gap remains in message-action handling. There is also a follow-up issue: the strict Telegram and Feishu config schemas on `main` still do not define `mediaLocalRoots`, so real config-driven use of this setting is not yet wired through normal parsing.

## Evidence

- **Root resolver is still config-blind:** `getAgentScopedMediaLocalRoots` only builds default media roots and optionally appends the agent workspace. It does not accept channel/account context and cannot merge `channels.<channel>.mediaLocalRoots` or account-scoped overrides.
  - file: [src/media/local-roots.ts:54](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/media/local-roots.ts#L54)

On current `main`, configured channel/account roots are therefore not merged into outbound delivery.
