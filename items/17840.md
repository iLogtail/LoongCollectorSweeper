## Summary

Keep #17840 open. Current main still exposes reaction notification/queueing behavior, not an opt-in reaction-triggered agent turn, and the issue has concrete Slack, Telegram, Discord, and Matrix/Signal-adjacent evidence plus a recent maintainer keep-open comment.

## Best Possible Solution

Keep #17840 open as the canonical product request unless maintainers explicitly choose #56708 or another tracker. Implement an opt-in per-channel/plugin setting such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist"; apply it after existing reaction authorization/notification filtering, enqueue the reaction system event, then request a scoped heartbeat/agent turn with the existing wake primitive. Update channel schemas, docs, and focused reaction tests while preserving the passive default.
