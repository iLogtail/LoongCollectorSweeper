## Summary

Keep #49376 open. Current main has adjacent auth-profile/model-fallback controls for overloaded errors, but it still does not provide the requested configurable LLM request retry policy around the main conversation prompt path. The related PR #49800 is still open/unmerged in the provided context and explicitly says it does not add the requested config surface.

## Best Possible Solution

Keep the issue open as the canonical feature request unless maintainers prefer to supersede it with a narrower open PR. The best implementation path is an agent/default-level LLM retry policy that wraps the main prompt request path, reuses the existing transient error classification and retry helpers, respects abort/timeout caps, avoids auth/billing/format retries, and documents how it interacts with auth-profile rotation and model fallbacks.
