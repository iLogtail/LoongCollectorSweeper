## Summary

Keep #42408 open. Current main has some diagnostics for missing `memorySearch.extraPaths`, but the requested core/docs work is not fully implemented: builtin memory indexing still has no `excludePaths`/glob exclusion config, and the public memory docs only document `extraPaths` without the requested path-hygiene or benchmark-contamination guidance.

## Best Possible Solution

Keep the issue open and implement a scoped memory-search hygiene improvement: add documented path-hygiene guidance to the public memory docs, add a typed/config-schema-backed builtin exclusion mechanism such as `agents.defaults.memorySearch.excludePaths` or per-root glob filters, apply it in the builtin scanner/watcher/reindex scope hash, and add tests covering benchmark/queryset files being excluded. The existing missing-extra-path status diagnostic should be preserved and can be expanded if maintainers want stronger outside-workspace intent warnings.
