## Summary

Keep #39120 open. Current main still restricts `bootstrap-extra-files` to recognized bootstrap basenames, so arbitrary workspace files such as `hooks/REGISTRY.yaml` are still skipped rather than injected. The related implementation PR #39174 was closed unmerged, and current docs/tests still describe and assert the restricted behavior.

## Best Possible Solution

Keep the issue open for maintainer/product decision. The direct implementation path is to widen the bootstrap-file contract beyond `WorkspaceBootstrapFileName`, relax or replace the `VALID_BOOTSTRAP_NAMES` check in `loadExtraBootstrapFilesWithDiagnostics()`, preserve `openBoundaryFile` workspace/size guards and bootstrap budget enforcement, intentionally handle subagent/light-context filtering for arbitrary paths, and update the bundled hook docs/tests. If maintainers prefer this outside core, document a supported hook/plugin or ClawHub pattern that gives deterministic bootstrap-time injection with equivalent guards before closing.
