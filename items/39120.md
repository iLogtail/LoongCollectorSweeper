## Summary

Keep #39120 open. Current main still rejects arbitrary `bootstrap-extra-files` basenames before reading them, the bundled hook has no alternate arbitrary-file path, docs/tests still describe the restriction, and the related implementation PR #39174 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct implementation would widen `WorkspaceBootstrapFile.name`, replace or relax `VALID_BOOTSTRAP_NAMES` inside `loadExtraBootstrapFilesWithDiagnostics()`, preserve workspace boundary and size guards, define subagent/light-context handling for arbitrary files, and update the bundled hook docs/tests. If maintainers prefer this outside core, document a supported hook/plugin or ClawHub pattern that provides deterministic bootstrap-time file injection with equivalent safeguards before closing.
