## Summary

Keep open. Current main still archives the old transcript on `sessions.reset`, then replaces the `sessions.json` row with a new `sessionId` for the same key. `sessions.list` is built from the current session store only, and the UI session selector reloads from `sessions.list`, so `.jsonl.reset.*` archives are still not represented as selectable prior sessions. The related #60409/#45003 context supports this: #60409 is open and only addresses `chat.history` fallback, with UI archive exposure left as follow-up.
