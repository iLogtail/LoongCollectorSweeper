## Summary

Keep #38309 open. Current main at b67d9bf7f06f4c655b1e0869395056c7d631abc2 still does not implement SecretRef support for `sandbox.docker.env`: the config schema and public type remain string-only, sandbox resolution merges env values without secret materialization, Docker consumes the resolved env as plain strings, and the secrets collector/target registry do not cover Docker sandbox env paths. PR #38310 is relevant prior work but was closed unmerged; #33073 covered other SecretRef credential fields, not this Docker sandbox env surface.

## Best Possible Solution

Keep the issue open until OpenClaw implements SecretRef support end-to-end for Docker sandbox env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, collect and resolve default and per-agent `sandbox.docker.env` refs through the existing secrets runtime, add target registry coverage for audit/plan/configure, keep the resolved launch config as `Record<string, string>`, and add focused schema/runtime tests. PR #38310 remains useful design context but is not shipped or canonical.
