## Summary

Keep #55815 open. Current main still has no warning-only exemption in `handleToolExecutionEnd()`, while Telegram reaction soft-fail results still flow through `jsonResult(...)` with the warning payload under `result.details`. The provided PR discussion shows the branch addressed the important `details.warning` review feedback, but that logic is not present on current main.

## Best Possible Solution

Keep the PR open for maintainer review or refresh. The best fix remains a narrow subscriber-side classifier that treats root-level or `details.warning` payloads with no hard error fields as non-error, gates `handleToolExecutionEnd()` before setting `lastToolError` or emitting failed tool/item events, and includes regression coverage for the real `jsonResult(...)` `details.warning` shape plus the end-to-end handler `isError: false` event.
