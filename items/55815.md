## Summary

Keep open. Current `main` at `7130e56f872aea96577ac9fcbda0c7555a9ecc69` does not contain this PR’s warning-only classification fix, and the Telegram reaction path still produces warning-only payloads that this handler would treat as failed tool executions whenever `evt.isError` is true.

## Evidence

- **Current main SHA inspected:** Reviewed the checked-out `main` at commit `7130e56f872aea96577ac9fcbda0c7555a9ecc69`.
  - command: `git rev-parse HEAD`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Tool-end handler still flags warning payloads as errors:** `handleToolExecutionEnd()` still computes `isToolError` as `isError || isToolResultError(result)` with no warning-only exception, so any warning-only result will still be surfaced as a failed tool when `evt.isError` is true.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:797](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-subscribe.handlers.tools.ts#L797)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Main lacks the proposed warning-only helper:** `src/agents/pi-embedded-subscribe.tools.ts` defines `isToolResultError()` and `extractToolErrorMessage()`, but there is no `isWarningOnlyToolResult()` or equivalent warning-only classifier on current `main`.
  - file: [src/agents/pi-embedded-subscribe.tools.ts:349](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-embedded-subscribe.tools.ts#L349)

  - file: [extensions/telegram/src/action-runtime.ts:295](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/extensions/telegram/src/action-runtime.ts#L295)

  - file: [src/agents/tools/common.ts:285](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/tools/common.ts#L285)
