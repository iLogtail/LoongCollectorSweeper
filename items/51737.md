## Summary

Keep open. Current `main` still has the agent-overview picker gap this PR targets: provider-qualified refs from `agents.defaults.model.primary` / `fallbacks` are collected elsewhere but are not included in the overview picker options, so the UI can still fall back to a synthetic `Current (...)` entry. The chat-header part of the PR looks partly superseded by newer provider-qualified model handling already on `main`, but the overview-picker fix is not implemented there.
