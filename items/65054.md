## Summary

Keep #65054 open. Current main still hardcodes MiniMax usage polling to the CN host, and the MiniMax plugin calls that helper without passing or deriving the configured provider base URL. The related open PR #65059 appears to track the same fix, but it has not landed in current main, so the original bug report should remain open until a fix ships.

## Best Possible Solution

Keep #65054 open and land a focused MiniMax fix. Derive the usage endpoint host from `models.providers.minimax.baseUrl`, `models.providers.minimax-portal.baseUrl`, or the same region logic used by MiniMax Search, while retaining the CN endpoint fallback for existing callers. Add tests that assert global config calls `https://api.minimax.io/v1/api/openplatform/coding_plan/remains` and CN config calls `https://api.minimaxi.com/v1/api/openplatform/coding_plan/remains`, then update the MiniMax provider docs if the usage endpoint behavior changes.
