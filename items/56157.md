## Summary

Keep open. Current `main` does not already implement this documentation work, and `docs/tools/subagents.md` still has live gaps/inconsistencies around sub-agent params, always-denied tools, and nesting depth.

## Evidence

- **Sub-agent docs still omit documented runtime/config knobs:** `docs/tools/subagents.md` lists `sessions_spawn` params but does not mention `announceTimeoutMs` or `streamTo` in the current `main` doc.
  - file: [docs/tools/subagents.md:88](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/tools/subagents.md#L88)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Runtime supports `announceTimeoutMs`, and `streamTo` exists with an ACP-only guard:** The spawn tool schema includes `streamTo`, and the executor rejects it unless `runtime="acp"`; announce delivery reads `announceTimeoutMs` from config. That means current docs are still missing/underspecified behavior rather than already fixed on `main`.
  - file: [src/agents/tools/sessions-spawn-tool.ts:125](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/tools/sessions-spawn-tool.ts#L125)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Config schema still exposes deeper nesting than the top-level subagent doc explains:** `maxSpawnDepth` is accepted in the range `1..5`, but the current sub-agent doc still frames setup as only enabling one extra level (`maxSpawnDepth: 2`) and marks depth 2 as terminal.
  - file: [src/config/zod-schema.agent-defaults.ts:257](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/zod-schema.agent-defaults.ts#L257)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current sub-agent doc is internally inconsistent on nesting depth:** The same page says depth 2 is a leaf and only describes the main→orchestrator→worker pattern, but later also says maximum nesting depth is 5. That inconsistency shows the documentation area this PR targets is not already resolved on `main`.
  - file: [docs/tools/subagents.md:162](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/tools/subagents.md#L162)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current docs still understate always-denied tools for sub-agents:** Runtime policy always denies `gateway`, `agents_list`, `session_status`, `cron`, and `sessions_send` for sub-agents, while the current doc only calls out session-tool exclusions and does not fully document the always-denied set.
  - file: [src/agents/pi-tools.policy.ts:34](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-tools.policy.ts#L34)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
