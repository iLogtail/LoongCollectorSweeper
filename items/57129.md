## Summary

Keep #57129 open. The issue asks for fresh isolated Slack runs for unrelated new top-level questions, but current main still deliberately routes top-level Slack channel messages to the same per-channel session. Only real Slack thread replies get a thread-scoped session key, and tests/docs currently guard that behavior.

## Best Possible Solution

Keep the issue open for maintainer/product decision. A concrete fix would likely live in the Slack plugin routing path, adding an explicit top-level channel isolation mode that starts a fresh session for each new top-level Slack question while routing replies inside the resulting Slack thread back to that question's session. That change should update Slack config schema/docs and add routing/session tests covering top-level isolation plus same-thread follow-up reuse.
