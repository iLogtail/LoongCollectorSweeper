## Summary

Keep PR #57239 open. Current main still queues active-session followups and does not implement the PR's fast-lane override, so it is not implemented or superseded. The PR is a focused implementation attempt for open issue #56880, but the supplied review context shows unresolved concurrency/state-ordering concerns that need rework rather than automated closure.

## Best Possible Solution

Keep the PR open as a concrete but unfinished implementation attempt for #56880. The best path is maintainer-guided rework: define the intended active-session concurrency contract, preserve queue ordering and parent session/system-event accounting, decide whether command-lane isolation is acceptable, and add focused integration coverage for long active run plus quick followup before merge or replacement.
