## Summary

Keep #61019 open. Current main has manual ways to move a session back to the configured default, but the reported global-default propagation is not implemented: `openclaw models set` updates config only, and existing session model resolution still prefers persisted runtime or per-session model state before falling back to the new default. The provided related reports (#42690 and #42788) cover adjacent model-status/display issues, but they are not canonical fixes for this active-session inheritance request.

## Best Possible Solution

Keep the issue open and implement a deliberate session-default migration path: after `openclaw models set`, either offer/perform a scoped refresh for sessions that only carry stale runtime model identity, or expose a documented command/UI action that clears existing session model state back to `agents.defaults.model.primary` without losing context. Explicit user-selected per-session overrides should remain preserved unless the user opts to migrate them.
