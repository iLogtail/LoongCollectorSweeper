## Summary

Keep open. Current main still does not implement a token-free static cron channel message payload, and the issue discussion already contains an April 26, 2026 Codex review from steipete saying this exact item should remain open. The cron contract, Gateway schema, normalization/update paths, runtime execution, CLI, UI, and docs all still expose only the AI/session-mediated `systemEvent` and `agentTurn` payloads.

## Best Possible Solution

Keep #11473 open until maintainers either implement a first-class token-free static message cron payload or explicitly supersede it with a broader non-agent cron payload design. The likely fix belongs in the cron type/normalization/update contract, Gateway protocol schema, runtime delivery path, CLI and Control UI creation/editing surfaces, docs, and focused tests for direct channel delivery and failure reporting.
