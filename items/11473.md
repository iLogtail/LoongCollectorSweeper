## Summary

Keep #11473 open. Current main still has no token-free static cron message payload: the cron contract, Gateway schema, normalization, runtime validation/execution, CLI, Control UI, and docs remain built around `systemEvent` and `agentTurn`. The related #18160 direct-exec request overlaps on avoiding model calls but is broader and does not fully supersede this narrower static text-to-channel request. The issue also already has a maintainer-authored Codex review comment explicitly saying to keep it open.

## Best Possible Solution

Keep #11473 open until OpenClaw either implements an additive token-free static message cron payload or maintainers explicitly supersede it with a broader non-agent cron payload design. A complete implementation would update the cron payload type and Gateway schema, normalization and validation, runtime direct channel delivery with audit/failure reporting, CLI and Control UI creation/editing flows, docs, and focused tests.
