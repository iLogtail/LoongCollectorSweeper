## Summary

Keep open. Current `main` still supports sending WhatsApp polls/reactions, but it does not implement inbound poll-vote or emoji-reaction processing for agents/skills.

## Evidence

- **WhatsApp inbound listener only wires `messages.upsert`:** The WhatsApp web inbox monitor handles inbound traffic through `handleMessagesUpsert(...)` and only attaches listeners for `messages.upsert` and `connection.update`. There is no `messages.update` listener on the current main path.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:637](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/whatsapp/src/inbound/monitor.ts#L637)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Inbound extraction only recognizes text/media-style message bodies:** The inbound extractor's supported message content keys are limited to conversation/text/media/location/contact/button/list variants, and `extractText(...)` only pulls text/caption-style content. It does not define handling for reaction or poll-response payload types.
  - file: [extensions/whatsapp/src/inbound/extract.ts:24](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/whatsapp/src/inbound/extract.ts#L24)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Inbound WhatsApp message shape has no poll/reaction event fields:** `WebInboundMessage` exposes text, sender, reply, mention, media, and location fields only. There is no payload for poll selections, reacted message id, emoji, or add/remove state.
  - file: [extensions/whatsapp/src/inbound/types.ts:54](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/whatsapp/src/inbound/types.ts#L54)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **WhatsApp action surface is outbound-only for polls and reactions:** The channel action discovery advertises only `react` and `poll`, and the web send API implements `sendPoll(...)` and `sendReaction(...)`. I found no corresponding inbound poll-response or reaction handler on main.
  - file: [extensions/whatsapp/src/channel-actions.ts:61](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/extensions/whatsapp/src/channel-actions.ts#L61)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Current docs/config describe WhatsApp outbound reactions and ack reactions, not inbound poll/reaction events:** WhatsApp docs document `reactionLevel`, `ackReaction`, and action gates for outbound `react`/`poll`. The reactions doc explicitly calls out inbound reaction notifications for Signal, but not WhatsApp. The WhatsApp config type also has no `reactionNotifications` or poll-response setting.
  - file: [docs/channels/whatsapp.md:356](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/docs/channels/whatsapp.md#L356)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Generic live event surface also lacks poll/reaction-specific event types:** The MCP event model currently lists only `message`, approval, and Claude permission events. There is no dedicated queue event for poll responses or message reactions.
  - file: [src/mcp/channel-shared.ts:76](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/src/mcp/channel-shared.ts#L76)
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
