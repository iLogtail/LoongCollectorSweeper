## Summary

Keep open. Current main added a partial `agents_list` discovery tool, but it does not satisfy the issue's requested sibling-agent roster: it is scoped to `sessions_spawn` targets, returns top-level `name` only, and does not expose `identity.name`, `identity.theme`, or whether agent-to-agent messaging is allowed. The system prompt also only advertises generic tooling, not a concrete sibling roster.

## Evidence
