## Summary

Keep #52146 open. Current main has an `agents_list` tool, but it is scoped to `sessions_spawn` sub-agent targets and does not provide the requested sibling-agent roster with `identity.name`, `identity.theme`, or agent-to-agent send permission. The system prompt advertises the tool name only; it does not inject configured peer agents. Ryce's KeepMyClaw comment is relevant broader state-divergence context, but it is not a core implementation or canonical duplicate for this request.

## Best Possible Solution

Add a core-owned, deterministic sibling-agent context surface: either augment `agents_list` or add a small prompt section that lists configured peer agents from `agents.list` with `id`, `identity.name` fallback behavior, `identity.theme`, and an explicit `canSendAgentToAgent`/reason derived from `tools.sessions.visibility` and `tools.agentToAgent` policy. Keep it compact and sorted for prompt-cache stability, and cover it with focused tests around default allowlists, `*`, identity fields, and A2A allow/deny cases.
