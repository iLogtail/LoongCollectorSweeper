## Summary

Keep open. Current `main` does not expose `voice.autoJoin[].sessionChannelId` in the Discord config contract or docs, so the requested auto-join text-channel routing is not implemented on `main`. The PR discussion also surfaces unresolved correctness risks: the proposed `deliver: true` path needs an explicit Discord target to resolve delivery, and decrypt-failure recovery currently rejoins with only the voice channel, which would drop any separate session-channel routing.
