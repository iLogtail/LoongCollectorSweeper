## Summary

Keep PR #53741 open. Current main still does not expose `sessionChannelId` on Discord voice `autoJoin`, still routes voice sessions from the voice channel id, and still calls ingress with `deliver: false`, so the central requested behavior is not implemented. The linked feature issue #53562 also has a maintainer update from 2026-04-25 explicitly saying this remains open on current main. The PR is still relevant, but it needs maintainer/code-review follow-through before it can land.

## Best Possible Solution

Keep the PR open and ask for a focused update: add `sessionChannelId` to the Discord voice autoJoin config type/schema/docs/generated baseline, forward it through autoJoin and join, validate that the target is a same-guild text-capable channel before routing, preserve it during decrypt-failure rejoin, pass an explicit Discord delivery target or runContext for text delivery, use best-effort/catch delivery failures so voice playback continues, destroy any joined connection on validation failure, and cover these paths with Discord voice tests.
