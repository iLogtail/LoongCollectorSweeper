## Summary

Keep open. In the checked-out `main` snapshot, `scripts/ui.js` still resolves `pnpm` to an absolute path and then launches `.cmd` shims with `shell: true`, which matches the reported Windows `C:\Program Files\...\pnpm.cmd` failure mode rather than proving a fix on `main`.

## Evidence

- **`scripts/ui.js` still resolves an absolute pnpm path:** `resolveRunner()` calls `which("pnpm")` and returns that filesystem path as `runner.cmd`, so a standard Windows install can produce `C:\Program Files\nodejs\pnpm.cmd` exactly as reported.
  - file: [scripts/ui.js:48](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/scripts/ui.js#L48)
  - command: `nl -ba scripts/ui.js | sed -n '48,53p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **The UI runner still uses `shell: true` for `.cmd` launchers:** `shouldUseShellForCommand()` treats `.cmd` as shell-only on Windows, `createSpawnOptions()` sets `shell: true`, and both install/build execution paths call `run(runner.cmd, ...)` or `runSync(runner.cmd, ...)`. That is the same launch pattern the issue reports as breaking when the command path contains spaces.
  - file: [scripts/ui.js:56](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/scripts/ui.js#L56)
  - command: `nl -ba scripts/ui.js | sed -n '56,95p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Current regression test does not cover `pnpm.cmd` under `Program Files`:** `test/scripts/ui.test.ts` asserts that `.cmd` launchers require shell mode, but its only `Program Files` case is `node.exe`, not `pnpm.cmd` in a spaced path. There is no test proving the reported failure was fixed.
  - file: [test/scripts/ui.test.ts:5](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/test/scripts/ui.test.ts#L5)
  - command: `sed -n '1,40p' test/scripts/ui.test.ts`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **OpenClaw already uses a safer Windows pnpm strategy elsewhere, but not here:** `scripts/pnpm-runner.mjs` uses `cmd.exe /d /s /c` with `buildCmdExeCommandLine("pnpm.cmd", pnpmArgs)` and `shell: false`, and `test/scripts/pnpm-runner.test.ts` covers arguments containing spaces. That safer wrapper has not been adopted in `scripts/ui.js` on this snapshot.
  - file: [scripts/pnpm-runner.mjs:63](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/scripts/pnpm-runner.mjs#L63)
  - command: `nl -ba scripts/pnpm-runner.mjs | sed -n '43,69p'; nl -ba test/scripts/pnpm-runner.test.ts | sed -n '109,123p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
