## Summary

Keep #55458 open. Current main still silently swallows `manager.probeVectorAvailability()` errors in the status memory scan, and the memory status snapshot/rendering path has no `probeError` field or display logic. The PR discussion identifies a real diagnostics improvement, but the branch also bundles unrelated Fly/Docker/plugin SDK changes and has unresolved review feedback, so the best maintainer path is a focused rework rather than auto-closing.

## Best Possible Solution

Keep the item open and ask for a focused rework: add a `probeError` diagnostic to the current `resolveSharedMemoryStatusSnapshot()` path, preserve error presence without a truthy-string guard, render it through `buildStatusMemoryValue()` with warning styling, and add a targeted test where `probeVectorAvailability()` rejects. The unrelated Fly/Docker/workflow/plugin SDK changes should be split out or dropped from this PR.
