## Summary

Keep #54559 open. Current main still implements `generic_repeat` as warning-only, `process(action: "list")` is still not treated as a known poll tool, and the before-tool-call hook only blocks when a detector returns `level: "critical"`. The issue is also the older, more specific canonical report for later duplicate/similar ClawSweeper items rather than something to close as duplicate.

## Best Possible Solution

Keep #54559 open as the canonical generic-repeat loop-detection bug. The likely maintainer path is to add a `criticalThreshold` escalation for generic repeated no-progress calls, decide whether `process(action: "list")` should be classified as a known poll action, update the warning/blocking docs to match the intended contract, and add/flip focused unit plus before-tool-call e2e tests so generic repeats block at the intended critical threshold while the global breaker remains a last-resort guard.
