## Summary

Keep #41594 open. Current main has partial mitigations for this problem family, but it does not implement the requested alias/model-scoped `contextProfile`: model allowlist entries only support alias/params/streaming, workspace bootstrap injection is controlled globally, skills are scoped by agent rather than model, and no-tool model compat disables callable tool schemas without providing a first-class way to strip the Tooling/Skills prompt sections per alias.

## Best Possible Solution

Keep this open for a core design change. The best path is to add a model/alias-scoped context profile under the model selection surface, likely `agents.defaults.models[ref].contextProfile`, then resolve it after `/model` selection and apply it through `resolveBootstrapContextForRun`, `resolveSkillsPromptForRun`, tool exposure, and `buildEmbeddedSystemPrompt`. If #59168 changes model keying first, this issue should remain the focused follow-up for the prompt/context behavior.
