## Summary

Keep open. Current `main` has partial mitigations for tool-incompatible models (`compat.supportsTools: false`, `localModelLean`), but it does not implement alias-scoped or per-model context profiles for injected workspace files, skills, or tool-list prompt sections. The runtime still builds the normal injected context and skills prompt for these runs, and the system prompt still renders a fallback Tooling block even when no tools are available.

## Evidence

- **Model alias config has no context-profile field:** The validated per-model config under `agents.defaults.models` only supports `alias`, `params`, and `streaming`. There is no alias-scoped `contextProfile`, workspace-file selector, or skills/tool prompt toggle in the current config surface.
  - file: [src/config/zod-schema.agent-defaults.ts:52](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/config/zod-schema.agent-defaults.ts#L52)
  - command: `nl -ba src/config/zod-schema.agent-defaults.ts | sed -n '40,90p'`
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Bootstrap context injection is not model-aware:** `resolveBootstrapContextForRun(...)` only accepts workspace/config/session/agent/context-mode inputs and builds injected `contextFiles` from bootstrap files. There is no provider/model/alias parameter that could select different workspace-file sets per model.
  - file: [src/agents/bootstrap-files.ts:255](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/agents/bootstrap-files.ts#L255)

- **System prompt still emits tooling guidance even with no tools:** `buildAgentSystemPrompt(...)` always emits a `## Tooling` section. When `toolLines.length === 0`, it falls back to hardcoded `This runtime enables:` tool bullets instead of omitting tooling guidance entirely.
  - file: [src/agents/system-prompt.ts:679](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/agents/system-prompt.ts#L679)
