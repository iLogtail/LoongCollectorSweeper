## Summary

Keep open. Current main has partial mitigations for the token/tool-pressure problem, especially `compat.supportsTools: false`, but it does not implement the requested opt-in per-model or per-alias `contextProfile` for workspace file selection, tool prompt visibility, and skills visibility. The sole issue comment reinforces that the desired design is alias-scoped and backward-compatible, and the related #59168 is complementary model-key work rather than a canonical tracker for these context-injection controls.

## Evidence

- **Model schema has compat flags, not context profiles:** The model definition schema accepts `compat.supportsTools`, `contextWindow`, and `contextTokens`, but because the object is strict and has no `contextProfile`, `workspaceFiles`, `tools`, or `skills` profile fields, the proposed model-level config shape is not accepted on current main.
