## Summary

Current `main` (`cf858258c71701ef62dddf88aa777873a4962046`) initializes the global hook runner during synchronous gateway plugin bootstrap, before the server starts listening and before channel sidecars start, so the cold-start `getGlobalHookRunner() === null` race described in PR #62871 does not match the present startup flow.

## Evidence

- **Hook runner activation is immediate during plugin activation:** `activatePluginRegistry()` calls `initializeGlobalHookRunner(registry)` immediately after `setActivePluginRegistry(...)`. This is inside the synchronous `loadOpenClawPlugins()` path, not an awaited post-step.
  - file: [src/plugins/loader.ts:1831](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/plugins/loader.ts#L1831)
  - command: `nl -ba src/plugins/loader.ts | sed -n '1831,1841p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Gateway startup loads plugins before serving traffic:** `prepareGatewayPluginBootstrap()` awaits startup maintenance first, then calls `loadGatewayStartupPlugins()` before returning bootstrap state.
  - file: [src/gateway/server-startup-plugins.ts:24](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/gateway/server-startup-plugins.ts#L24)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '24,117p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Server bind happens after plugin bootstrap completes:** `server.impl.ts` awaits `prepareGatewayPluginBootstrap(...)` before continuing startup, and only later calls `startListening()`. That ordering prevents inbound traffic from arriving before plugin bootstrap finishes.
  - file: [src/gateway/server.impl.ts:348](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/gateway/server.impl.ts#L348)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '348,355p;809,817p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Channels and sidecars start after post-attach:** `startGatewayPostAttachRuntime()` starts channels and sidecars from a deferred `setImmediate` block after startup, so first channel traffic does not get ahead of hook-runner initialization on current `main`.
  - file: [src/gateway/server-startup-post-attach.ts:471](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/gateway/server-startup-post-attach.ts#L471)
