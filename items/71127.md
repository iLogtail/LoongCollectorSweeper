## Summary

Keep open. Current `main` still treats stuck sessions as diagnostics-only: the heartbeat logs/emits `session.stuck` but does not invoke any abort or queue-release path, the public config/docs only expose `diagnostics.stuckSessionWarnMs`, and the existing tests cover warning emission rather than recovery. The report is also fresh and contains concrete production evidence, so none of the conservative auto-close reasons apply.

## Evidence

- **Current heartbeat only warns about stuck sessions:** `startDiagnosticHeartbeat()` computes `ageMs` for processing sessions and only calls `logSessionStuck(...)` when the threshold is exceeded; there is no recovery callback, abort, or queue-drain in that loop.
