## Summary

Keep #47660 open. The PR is a real Dockerfile refactor, but current main still deliberately supports the Docker contracts the PR removes or weakens, including selective bundled plugin dependency baking, runtime variants, digest-pinned base images, apt cache mounts, and retry hardening. The PR also has unresolved review comments identifying stale comments and a documented OPENCLAW_EXTENSIONS regression, so it is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer judgment or author revision. A mergeable version should preserve or intentionally replace OPENCLAW_EXTENSIONS across Dockerfile, setup scripts, Podman/QA flows, docs, install-smoke, and tests; either keep digest pinning/cache/retry hardening or update the documented/tested policy explicitly; and resolve the stale Dockerfile comments before review.
