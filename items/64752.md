## Summary

Keep #64752 open. Current main receives Telegram `message_reaction` updates and records them as system events, but the reaction handler still does not request an immediate heartbeat/agent wake or dispatch a Telegram message turn. The related #47677 is an open broader feature request, not a shipped fix or clearly canonical replacement for this focused bug report.

## Best Possible Solution

Keep this focused bug open. The likely fix belongs in the Telegram plugin path: after an authorized reaction notification is enqueued, add an explicit wake/agent-turn mechanism for the resolved session when configured, align schema/docs around the supported config key, and add regression tests that prove a reaction-only update schedules the agent turn. Use #47677 for broader follow-up semantics such as richer structured reaction context and custom emoji normalization.

## Evidence

- **Issue discussion confirms live repro:** The issue body plus comments 4229938784 and 4261225850 report live Telegram DM repros on OpenClaw v2026.4.9, v2026.4.10, and v2026.4.11: `Telegram reaction added...` appears in session context, but no agent turn starts until later text arrives. The timeline links #47677, but that related item is an open broader feature request for first-class Telegram reaction triggers, not an implementation proof.
