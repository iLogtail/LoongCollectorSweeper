## Summary

Keep PR #30310 open. Current main still omits the PR's central behavior: shared account listing returns only named accounts when an accounts map exists, so base/default credentials are not started beside independent named accounts. The PR discussion also remains substantive, with active contributor updates plus unresolved security/product review around implicit default startup.

## Best Possible Solution

Keep #30310 open as the canonical focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage, or explicitly decide that named accounts suppress base/default startup and document that contract. Before landing, resolve the implicit-default startup policy and channel-specific edge cases raised in review.
