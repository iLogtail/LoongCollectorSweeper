## Summary

Keep #30310 open. Current main still lacks the PR's central token-aware default-account inclusion: the shared account helper returns only configured account IDs when an accounts map exists, and gateway startup starts exactly those IDs. Slack and Discord still use that helper while allowing named accounts to inherit top-level credentials, so the reported startup behavior remains a plausible core/channel bug rather than cleanup material.

## Best Possible Solution

Keep #30310 open as the focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage in src/channels/plugins/account-helpers.test.ts, or make an explicit product/security decision that named accounts suppress base/default startup and document that contract. Until then, this PR remains useful and not conservative cleanup material.
