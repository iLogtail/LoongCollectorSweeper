## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before registry lookup, while the PR addresses a concrete CLI/gateway mismatch reported in #71767 and is not implemented, duplicate, or out-of-repo cleanup.

## Best Possible Solution

Keep PR #71857 open for maintainer review. The best path is the focused parser change proposed by the PR: accept optional `<owner>/<slug>` for requested ClawHub skill install/update, validate both segments with the existing ASCII slug rules, strip the owner before registry/install-dir/lockfile use, preserve legacy tracked-slug update behavior, and keep targeted regression tests plus the changelog entry.
