## Summary

Keep PR #71857 open. Current main still rejects `owner/slug` ClawHub skill inputs before registry lookup, while live ClawHub resolves `skill-vetter` as the canonical API slug for owner `spclaudehome` and returns 404 for the owner-prefixed API path. The PR addresses a concrete mismatch reported in #71767 and is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep PR #71857 open for maintainer review. The best path is a narrow shared parser in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` for requested ClawHub skill install/update, validates both segments under the existing slug rules, canonicalizes to the bare slug before registry/install-dir/lockfile use, preserves legacy tracked-slug update compatibility, and lands with the focused regression tests and changelog entry proposed by the PR.
