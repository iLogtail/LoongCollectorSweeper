## Summary

Keep #72293 open. Current main has pieces that would help if an aborted stop reason reached the gateway, but the embedded agent lifecycle path still emits a normal `phase: "end"` for `stopReason: "aborted"`, and the subagent registry then records an `ok` completion. The related #38300 announce-back routing fix is different and does not cover this aborted-empty terminal state.

## Best Possible Solution

Keep the issue open and fix the embedded subagent terminal path: propagate `stopReason`, `errorMessage`, and/or `meta.aborted` into lifecycle events; have the subagent registry treat `stopReason: "aborted"` as killed/error rather than ok; persist `abortedLastRun: true`; and add regression coverage that an empty aborted assistant turn produces a parent-visible failed/aborted completion instead of an ok/no-output completion.
