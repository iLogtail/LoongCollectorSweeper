## Summary

Keep open. Current main still allows the reported Feishu gateway startup path to enter multiple per-account bot identity probes concurrently, and the linked implementation candidate #63481 remains open and unmerged.

## Best Possible Solution

Keep this issue open until #63481 or an equivalent Feishu-owned fix lands. The best fix is to serialize bot identity probes on the actual gateway per-account startup path, keep failure/abort behavior isolated per account, and add a regression test that simulates concurrent gateway startup for multiple Feishu accounts.

## Evidence
