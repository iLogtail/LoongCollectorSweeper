## Summary

Keep open. Current `main` still does not honor LiteLLM `--custom-base-url` in the non-interactive onboard path: the LiteLLM plugin uses the generic single-provider API-key helper, that helper's `runNonInteractive` never reads `ctx.opts.customBaseUrl`, and LiteLLM's config applier only preserves an already-configured `baseUrl` or falls back to `http://localhost:4000`. The PR discussion shows an active, concrete bugfix with follow-up review work, not an obsolete or incoherent item.
