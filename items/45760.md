## Summary

Current `main` intentionally excludes `MEMORY.md` from the minimal bootstrap passed to subagent and cron sessions, and both tests and docs describe long-term memory as main/private-session context with on-demand memory-tool access for delegated runs. That is strong evidence against merging this PR as-is, but it is not already implemented on `main` and it does not cleanly fit any allowed conservative auto-close reason, so this item should stay open for explicit maintainer judgment.
