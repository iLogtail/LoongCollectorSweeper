## Summary

Keep open. Current `main` still suppresses commentary-only assistant text in the TUI render path, which matches the reported behavior where the TUI shows `running` but no assistant text until a final payload arrives. The newer watchdog logic helps stale status handling, but it does not address the core visibility bug.

## Evidence

- **Shared assistant extraction still hides commentary-only phased text:** `src/shared/chat-message-content.ts:150-154` refuses unphased fallback once explicit phased blocks exist, and `src/shared/chat-message-content.ts:182-187` only returns `final_answer` text before falling back to legacy unphased extraction. For commentary-only phased messages, that resolves to `undefined`, so nothing is considered visible yet.
