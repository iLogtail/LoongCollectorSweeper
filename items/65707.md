## Summary

Keep #65707 open. Current main still computes minimal/full prompt mode for subagent and cron sessions, but that signal is not threaded into `buildEmbeddedExtensionFactories`, and context-pruning still installs the default `keepLastAssistants: 3` unless the user explicitly configures a different value. The PR remains a focused perf change rather than an obsolete or duplicate cleanup item; the review discussion also shows the initial `skillsSnapshot` signal problem was addressed by the PR's second commit using `promptMode`.

## Best Possible Solution

Keep the PR open for maintainer review or reimplementation on current main. The likely best path is to thread `promptMode`/`effectivePromptMode` into `buildEmbeddedExtensionFactories` from both run and compaction paths, cap `settings.keepLastAssistants` to 1 only for the intended autonomous minimal-mode sessions while preserving explicit 0, and add focused runtime-setting tests for minimal versus full prompt modes.
