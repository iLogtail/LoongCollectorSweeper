## Summary

Keep open. Current `main` still only applies backoff/circuit-breaker behavior to `401` `sendChatAction` failures, while transient non-401 failures still flow through the normal typing loop and are logged at error level on each failure.

## Evidence

- **Handler is still 401-only:** `createTelegramSendChatActionHandler` tracks only `consecutive401Failures`, sleeps only when that counter is non-zero, and only increments/suspends inside the `is401Error(error)` branch. Non-401 network/429/5xx failures are rethrown with no cooldown or duplicate suppression.
  - file: [extensions/telegram/src/sendchataction-401-backoff.ts:83](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/extensions/telegram/src/sendchataction-401-backoff.ts#L83)
