## Summary

Keep #51977 open. Current main has partial cron/session freshness cleanup, but the central reported retention path remains: the persisted session shape still includes the full rendered `skillsSnapshot.prompt`, session-store saves still stringify the whole store, the cache still keeps parsed and serialized store copies, isolated cron persistence still clones the full session entry into both base and `:run:` keys, and reset/follow-up paths still carry the snapshot forward. The issue discussion also contains concrete heap evidence and an independent v2026.3.22 production reproduction, so this is not stale or low-signal.
