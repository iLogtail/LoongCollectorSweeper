## Summary

Keep #50768 open. Current main has sub-agent defaults for model, thinking, and global run timeout, plus explicit `sessions_spawn` overrides, but it does not implement named reusable sub-agent templates, `promptFile` resolution, a `template` tool parameter, or template discovery in `agents_list`. The linked external agent-swarm PR in the issue timeline is useful evidence that the workflow exists, but it is not an OpenClaw implementation or canonical duplicate.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. A good implementation would add a strict, documented `agents.list[].subagents.templates` schema with prompt-file resolution relative to the agent workspace, merge template defaults into `sessions_spawn` with explicit call parameters winning, expose available templates through `agents_list` or a narrow companion discovery surface, and add tests for schema validation, prompt prepending, override precedence, missing prompt files, and timeout/model behavior. A ClawHub plugin could prototype a separate templated-spawn tool, but it should not close this core `sessions_spawn(template=...)` request unless maintainers explicitly choose that product boundary.
