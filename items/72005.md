## Summary

Keep PR #72005 open. Current main still lacks server-rendered outbound status footers, fabricated-footer stripping, and context-threshold warning persistence, and the PR is the canonical implementation thread that superseded #71984. ## Best Possible Solution

If maintainers prefer ClawHub/plugin ownership, first define a documented outbound/session-telemetry plugin API that makes the feature implementable outside core.
