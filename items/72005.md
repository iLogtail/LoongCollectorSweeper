## Summary

Keep #72005 open. Current main at 560ddd2f9b13fe2cf2073ea2922df481b7c06bb4 still lacks the proposed server-rendered outbound status footer, fabricated-footer stripping, context-warning config, and once-per-session threshold state. The PR is still useful as the canonical implementation thread for closed parent issue #71984, but should not merge unchanged because review discussion identifies concrete gateway, dry-run, threshold-recording, and concurrent-send issues.

## Best Possible Solution

Keep #72005 open and request changes. The best path is a core outbound-pipeline implementation that strips fabricated runtime footers before every real delivery path, renders optional server-authored footer text from session telemetry, skips state mutation for dry runs, handles direct, gateway, and mirror sends consistently, records all crossed thresholds atomically, and adds focused tests for direct/gateway/mirror/dry-run/skipped-threshold/concurrent-send behavior.
