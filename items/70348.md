## Summary

Keep open. Current main still allows synthetic compaction markers into the chat.history projection, and the related fixes do not supersede this PR. The PR likely needs rework because its proposed raw text-prefix filter can hide legitimate user content, but the underlying issue is still real and belongs in the gateway display-history path.

## Best Possible Solution

Keep the PR open, but request a safer implementation in the shared chat display projection: hide __openclaw.kind="compaction" markers from chat.history, and handle memory-flush/post-compaction artifacts using provenance or synthetic metadata rather than broad text prefixes. Add gateway-level tests proving internal artifacts are hidden while user-authored messages with similar text remain visible.
