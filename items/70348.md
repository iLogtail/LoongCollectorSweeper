## Summary

Keep open. Current main does not already implement PR #70348's requested filtering, and the related closed PR #32068 is not present as the canonical fix. Main still converts compaction transcript records into `chat.history` messages, intentionally tests that behavior, and the shared display projection does not drop compaction markers, pre-compaction memory-flush prompts, or post-compaction refresh text. The PR also has a substantive review concern around text-prefix false positives, so this needs maintainer/product judgment rather than conservative auto-close.

## Best Possible Solution

Keep this open for maintainer decision. The best implementation path is a provenance/source-backed visible-history filter in the shared chat display projection, with explicit agreement on whether compaction markers should remain as UI dividers or be hidden from `chat.history`, plus gateway regression tests for the JSONL-to-history path. Avoid raw text-prefix filtering unless it is constrained to synthetic/internal entries.
