## Summary

Keep #50090 open. Current main has implemented important pieces of the skill ecosystem, especially local/workspace/extra-dir discovery and documentation, but the issue still bundles live core/security work: the skills watcher still uses glob-style chokidar targets while the repo depends on chokidar 5, native ClawHub skill installs still bypass the installer-metadata scan path, and the permission-manifest/install-lifecycle trust work remains tracked by open related issues rather than shipped behavior.

## Best Possible Solution

Keep #50090 open until a maintainer explicitly consolidates it. The clean path is to note which parts are already shipped (skill discovery/docs, likely from the v2026.4.22-era work), then link or split the remaining work into concrete trackers: watcher glob/chokidar fix (#54927/#50468 context), permission manifest and supply-chain trust (#12219/#10827), skill install lifecycle hooks (#23926), ClawHub CLI/service publishing issues in the ClawHub repo, and any separate trigger-quality/linter proposal if maintainers want that in core.
