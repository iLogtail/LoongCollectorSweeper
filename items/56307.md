## Summary

Current main partially addresses the underlying thread-bound subagent flow through plugin hooks and channel thread bindings, including Feishu current-conversation binding, but it does not implement the requested automatic `thread: true` / `mode: "session"` injection from channel plugin policy. `sessions_spawn` still reads `thread` only from tool input, rejects `mode: "session"` without explicit `thread: true`, and the Feishu hook only runs after `threadRequested` is already true. This is a real core/plugin-SDK ergonomics/API request, not a ClawHub close or duplicate/superseded cleanup item.

## Best Possible Solution

Keep open as a focused core/plugin-SDK design item. The best path is an additive channel-owned policy or manifest/control-plane field that `sessions_spawn` can read before mode/thread validation, with tests proving Feishu or another thread-required channel can auto-select `thread: true` and `mode: "session"` while channels without that policy keep today’s explicit behavior.
