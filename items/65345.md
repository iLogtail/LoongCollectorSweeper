## Summary

Keep #65345 open. Current main still contains several source-backed docs/code alignment gaps from the report, including CLI JSON parsing, OpenResponses session/body behavior, /tools/invoke responses, $include merge semantics, and onboarding legacy-key handling.

## Best Possible Solution

Keep the issue open and split it into focused follow-ups by area. The likely lowest-risk path is to update the public docs for behavior that is already intentional and tested, especially CLI `--strict-json`, OpenResponses session/body/ignored-field behavior, `/tools/invoke` 403 and hook behavior, `$include` merge/security semantics, and onboarding legacy-key handling; any behavior changes should be handled separately with owner review and regression tests.
