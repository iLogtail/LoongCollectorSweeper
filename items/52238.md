## Summary

Keep #52238 open. Current main has added explicit Feishu `topic_group` support and tests for topic routing, but it still treats `chat_type: "p2p"` as direct even when the chat id is an `oc_...` conversation with `root_id`/`thread_id`, which is the core scenario reported in the issue and echoed in the comments.

## Evidence

- **Discussion identifies the remaining shape:** The issue reports Feishu topic-group messages logged as `p2p` in an `oc_...` chat and asks for supplemental detection via `thread_id`/`root_id`; the first maintainer-adjacent comment also proposed recognizing `chat_type === "p2p"` with `chat_id.startsWith("oc_")`.
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
