## Summary

Keep #52238 open. Current main has native Feishu `topic_group` support, but the reported shape is `chat_type: "p2p"` with topic metadata; that still flows through the direct-message route instead of group/topic routing.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin by adding a shared classifier for inbound Feishu conversation kind that treats topic-context `p2p` events as group/topic only when there is reliable topic metadata such as `thread_id` or `root_id`, while keeping ordinary `p2p` chats direct. Use that helper in `bot.ts` and `mention.ts`, preserve native `topic_group` behavior, and add regression coverage for `chat_type: "p2p"` with `oc_...` plus topic ids so routing resolves to the Feishu group/topic session and replies stay in the thread.
