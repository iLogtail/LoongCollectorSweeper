The linked issue is closed, but the provided PR discussion and patch describe a focused gateway stability fix that is not obsolete on current main.

## Best Possible Solution

Keep the PR open for maintainer review or a refreshed narrow patch that adds a per-connection Gateway WebSocket protocol ping, clears the timer on all close paths, and ideally includes a focused fake-timer/unit test around ping and cleanup. If maintainers do not want server-side WS control-frame keepalive, close only with explicit product judgment, not automated cleanup.
