## Summary

本 PR 提议在构建分发脚本中将 `strip` 改为 `strip --strip-debug` 以保留静态符号表，便于生产环境使用 `pstack`/`perf` 进行堆栈分析。维护者认可其调试价值，但明确指出不应在 Release 模式下无条件保留调试信息，建议根据 `CMAKE_BUILD_TYPE` 条件化应用该参数。作者尚未推送条件化逻辑的更新，且当前 PR 与 main 存在冲突，仍需进一步修改方可合并。

## Best Possible Solution

建议作者或维护者先将本 PR rebase 至最新 main 分支解决冲突。随后在 `scripts/dist.sh` 中引入构建类型判断（例如读取 `$CMAKE_BUILD_TYPE` 环境变量或构建参数），仅在 `RelWithDebInfo` 或 `Debug` 模式下使用 `strip --strip-debug`，在 `Release` 模式下保持原有的 `strip` 行为以控制二进制体积。若作者长期未响应，维护者可基于现有讨论结论直接提交条件化 strip 逻辑并合并。
