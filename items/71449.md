## Summary

Keep PR #71449 open. Current main has adjacent startup improvements, but it does not implement this PR's central user-visible changes: the `ready (` log still does not clarify gateway-listening vs channel recovery, there is no `sidecars settled` channel health summary, Telegram webhook cleanup still blocks/retries before polling without a startup timeout signal, and Feishu startup only treats timeout/abort specially. The PR is currently dirty against main and needs maintainer review/rebase, but it is still plausible focused work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the legacy `ready (` token while clarifying gateway-listening semantics, add the sidecar/channel health summary with deterministic output, keep Telegram webhook cleanup bounded with a safe background retry, keep Feishu startup probe behavior explicitly time-bounded and transient-network tolerant, and add focused tests for those behaviors before merge.
