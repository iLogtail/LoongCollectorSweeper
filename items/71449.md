## Summary

Keep PR #71449 open. Current main still lacks the PR’s central behavior: the startup log still says only `ready (...)`, there is no `sidecars settled` per-channel health summary, Telegram polling still waits and retries webhook cleanup before polling, and Feishu startup handling is only partially defensive. The PR is recent, focused, and its discussion shows review feedback was addressed rather than abandoned.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a polished version of #71449 or request targeted changes that preserve the legacy `ready (` marker for tooling while adding clear `gateway listening` wording, a deterministic sidecars/channel health summary, bounded non-blocking Telegram webhook cleanup, and Feishu transient network handling with focused tests.
