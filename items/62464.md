## Summary

Keep open. Current main has partial timeout handling, but not the issue's requested retry after the MiniMax fallback chain is exhausted. Idle timeouts can retry the same model only when no model fallbacks are configured, and the generic fallback loop still attempts each candidate once before throwing an "All models failed" summary when every candidate times out or fails.

## Best Possible Solution

Keep the issue open and implement a bounded, replay-safe timeout retry path for model requests before or after model fallback exhaustion. The likely fix belongs in the embedded runner/model-fallback layer with tests covering `minimax/MiniMax-M2.7` and `minimax/MiniMax-M2.7-highspeed` both timing out, plus clear docs/config for retry budget and backoff behavior.
