## Summary

Keep #59693 open for maintainer judgment. Current main does not implement the PR's requested server-side Control UI selector filter, auth-gated visibility, `_meta` response, or model-selection preflight. Main has partial related behavior for `agents.defaults.models` allowlist filtering, but that does not cover the proposed authenticated/configured selector modes, and the PR discussion includes unresolved security/behavior review findings that need a rework rather than an automatic close.

## Best Possible Solution

Keep the PR open for explicit maintainer review, but do not merge it as-is. The best path is a smaller rework that aligns with current `models.list` allowlist behavior and #48483, adds any new Gateway protocol/config surface deliberately with docs/tests, reuses the existing model-auth status/auth resolution semantics including synthetic local provider auth, and fixes the reviewed fallback and normalization issues before reconsidering merge or closure.
