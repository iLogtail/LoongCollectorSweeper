## Summary

Current main does not implement the requested behavior. Telegram group command authorization still intentionally uses explicit group allowlists (`groupAllowFrom`, per-group `allowFrom`, or `commands.allowFrom`) and excludes the pairing credentials store. The linked PR #57060 attempts the reported fix, but it is open/unmerged in the provided GitHub context and its old `src/auto-reply/command-auth.ts` approach does not match the current Telegram-owned group command path.

## Best Possible Solution

Keep the issue open for maintainer judgment. If maintainers want paired Telegram users to be authorized in groups, the fix should be implemented in the current Telegram-owned group command/access path with updated tests, docs, and the no-pairing-store group-auth guard. If maintainers prefer the current security boundary, they should close it explicitly as intentional design and point users to `groupAllowFrom`, per-group `allowFrom`, or `commands.allowFrom`.
