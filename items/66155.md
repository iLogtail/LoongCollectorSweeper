## Summary

Keep open. Current `main` still has a plausible mobile overlap bug in the Control UI chat bubble actions: the action icons are absolutely positioned and always visible on touch devices, but the renderer does not apply the spacing class that would reserve room for them.

## Evidence

- **Unused right-padding guard:** The chat bubble CSS only reserves right-side space for action icons via `.chat-bubble.has-copy { padding-right: 70px; }`, but that selector is not applied anywhere on current `main`.
  - file: [ui/src/styles/chat/grouped.css:219](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/ui/src/styles/chat/grouped.css#L219)
  - command: `nl -ba ui/src/styles/chat/grouped.css | sed -n '180,245p'`
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Touch devices force actions visible:** On touch devices, `@media (hover: none)` sets `.chat-bubble-actions` to always visible. Those actions are absolutely positioned at the top-right of the bubble, which matches the reported iPad/Safari clipping risk.
  - file: [ui/src/styles/chat/grouped.css:223](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/ui/src/styles/chat/grouped.css#L223)
  - command: `nl -ba ui/src/styles/chat/grouped.css | sed -n '223,243p'`
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Renderer never adds spacing class:** `renderGroupedMessage` builds `bubbleClasses` from `chat-bubble`, optional `chat-bubble--tool-shell`, `streaming`, and `fade-in`, then renders `.chat-bubble-actions` inside the bubble when assistant actions exist. There is no `has-copy` or equivalent spacing class in that path.
  - file: [ui/src/ui/chat/grouped-render.ts:1408](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/ui/src/ui/chat/grouped-render.ts#L1408)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '1408,1456p'`
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
