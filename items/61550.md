## Summary

Keep open. Current main still has a Telegram preview-streaming final-delivery path that can finalize by editing or materializing a preview instead of going through `deliverReplies`, and Telegram currently overrides the generic auto-reply `message_sending` before-delivery hook with an identity callback. The normal Telegram `deliverReplies` path is hooked, but the streaming preview finalization path remains unhooked.

## Best Possible Solution

Keep this issue open and fix the Telegram streaming preview finalization path. The likely product/code direction is to compose Telegram's dispatcher `beforeDeliver` with the generic `message_sending` hook, or add a narrow preview-finalization interception point before `editMessageTelegram`/draft materialization, then add regression tests for `streaming: partial` final text rewrite, cancellation, and the existing non-streaming `deliverReplies` behavior.
