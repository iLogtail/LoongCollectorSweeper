## Summary

Keep open. Current `main` still has the same initial `/vc join` failure path, and the shipped Discord voice recovery logic only helps after a voice session reaches `Ready` and later hits receive/decrypt errors. There have been no Discord voice code changes since `v2026.4.22`, so there is no strong evidence that issue #39825 is fixed on `main` or in the latest release.

## Evidence

- **Initial join still fails with the same surfaced error:** `DiscordVoiceManager.join` still waits up to 15s for `VoiceConnectionStatus.Ready` and then returns `Failed to join voice channel: ${formatErrorMessage(err)}` on failure. This matches the issue's user-visible abort symptom and does not create a session when the initial join never becomes ready.

That means the current mitigation does not address the reported case where `/vc join` aborts before any active voice session exists.
