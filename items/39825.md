## Summary

Keep #39825 open. Current main still has the same Discord voice join failure shape: `/vc join` waits up to 15 seconds for `VoiceConnectionStatus.Ready`, destroys the connection on readiness failure, and records the session only after that wait succeeds, so `/vc status` can still report no active sessions after an abort. Recent issue discussion and the newer #63098 report provide live-user signal that this remains unresolved rather than implemented or superseded.

## Best Possible Solution

Keep #39825 open as the Discord native voice readiness-abort tracker unless maintainers intentionally consolidate it into #63098. A real fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts/timeouts before session activation, improve diagnostics, and add either bounded retry/backoff or a documented configurable connect timeout if that is the intended product behavior.
