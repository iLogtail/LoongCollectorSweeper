## Summary

Keep open. Current main still has the same core failure shape reported here: `/vc join` waits up to 15s for `VoiceConnectionStatus.Ready`, destroys the connection on readiness failure, and only stores a session after that point, so `/vc status` can still report no active sessions after an abort. The newer main changes since v2026.4.24 are unrelated voice-model/docs work, while the issue discussion includes newer-build reproductions and a maintainer comment explicitly saying this should stay open.

## Best Possible Solution

Keep #39825 open until the Discord voice join path is fixed or deliberately superseded. The likely product fix belongs in `extensions/discord/src/voice/manager.ts` with regression coverage around readiness aborts/timeouts before session activation, clearer diagnostics, and possibly retry/backoff or a documented connect-timeout control if maintainer product judgment wants that surface.
