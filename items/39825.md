## Summary

Keep #39825 open. Current main still matches the reported failure shape: Discord voice join waits for `VoiceConnectionStatus.Ready` with a fixed 15s timeout, destroys the connection on readiness failure, and only records the session after that wait succeeds. The issue discussion also has newer reproductions and maintainer signal saying to keep it open.

## Best Possible Solution

Keep #39825 open as the Discord native voice readiness-abort tracker unless maintainers choose a newer canonical issue such as #63098. A real fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts/timeouts before session activation, improve diagnostics, and either add bounded retry/backoff or a documented configurable connect timeout if that is the intended product behavior.
