## Summary

Keep open. Current main still lacks the requested safe-mode recovery instructions for token updates, visible OAuth re-auth, and clearing the safe-mode flag; the newer OAuth mirroring code only partially overlaps and does not document or implement the safe-mode workflow described in the issue.

## Best Possible Solution

Keep the issue open until a maintainer either adds the safe-mode recovery instructions to the generated TOOLS.md/public docs or identifies the external installer/safe-mode workspace repo that owns the generator and moves/links the work there. The eventual docs should cover both token sources, visible OAuth re-auth without detached tmux, and the required safe-mode flag/reset exit procedure.
