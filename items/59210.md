## Summary

Keep open. Current main fixed much of the related headless QR workflow by adding a PNG-backed WhatsApp QR path through gateway/UI/agent tools, and related issue #45652 is already closed as implemented. But the WhatsApp docs still only show the CLI QR login path and do not include this PR's remote/headless caveat or point users at the current image-based QR surfaces, so the docs PR is not obsolete enough for conservative auto-close.

## Best Possible Solution

Keep the PR open, but ask for a small refresh before merge: document the real current remote/headless story in `docs/channels/whatsapp.md`, including that CLI login still prints a terminal QR while gateway/UI/agent login can surface a PNG data URL, and preserve the useful warning that WhatsApp setup is still QR-only rather than phone-number pairing-code based.
