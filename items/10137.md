## Summary

Keep #10137 open. Current main only partially covers the report: generic `api: "openai-responses"` requests already use array `input` and `stream: true`, but the requested stable compatibility options for stricter Responses-compatible proxies are still missing, especially generic top-level fallback `instructions` and an option to omit `max_output_tokens`.

## Best Possible Solution

Keep #10137 open as the canonical tracker. A solid fix would add documented stable provider/model compatibility options around `ModelCompatConfig`/`ModelCompatSchema` and the Responses transport/payload policy, covering generic top-level fallback `instructions`, optional omission of `max_output_tokens`, and focused tests for proxy-like `api: "openai-responses"` providers. Treat #10140 as the closed duplicate copy.
