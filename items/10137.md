## Summary

Keep #10137 open. Current main still only partially covers the report: generic `api: "openai-responses"` requests already send array `input` and `stream: true`, but there is still no stable compatibility layer for stricter Responses-compatible proxies to default/force top-level `instructions` or omit `max_output_tokens`. The provided discussion identifies #10140 as the closed duplicate, leaving #10137 as the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. A good fix belongs in the Responses transport/payload policy and stable provider/model compatibility config: add opt-in controls for generic `api: "openai-responses"` proxy compatibility, especially fallback/default top-level `instructions` and optional omission of `max_output_tokens`, with focused regression tests and docs updates. Treat #10140 as the closed duplicate copy.
