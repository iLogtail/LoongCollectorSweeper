## Summary

Current `main` suppresses heartbeat ack text for both Control UI chat and Telegram when `showOk` is false, matching the issue’s expected behavior. The fix is present on `main` after `v2026.4.23`, so this looks implemented but not yet released in the latest stable tag.

## Evidence

- **Control UI final chat output now strips/suppresses heartbeat ACK text for heartbeat runs:** `normalizeHeartbeatChatFinalText` checks heartbeat run context, applies `resolveHeartbeatVisibility({ channel: "webchat" })`, strips `HEARTBEAT_OK`, and suppresses the final message when the remainder is only ACK-sized content.
