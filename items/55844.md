## Summary

Current `main` already contains an unreleased fix for this exact symptom: pinning the startup channel registry so later plugin-registry churn does not leave gateway health/status with empty channel maps.

## Evidence

- **Unreleased changelog entry matches the report:** `CHANGELOG.md` explicitly says gateway channel registry pinning keeps configured channels visible and stops `channels.status` from falling back to empty `channelOrder` / `channels` payloads after runtime plugin loads.
  - file: [CHANGELOG.md:1405](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/CHANGELOG.md#L1405)
