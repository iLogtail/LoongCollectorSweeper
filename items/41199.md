## Summary

Keep open. Current main still contains both reported failure paths: `sessions_send` rejects calls containing both `sessionKey` and `label`, and `message` with `action: "send"` still throws when non-empty poll creation parameters are present. The related PRs in the provided issue timeline (#59324/#41255 for `sessions_send`, #46904 for poll params) remain unmerged follow-up work, not proof that main is fixed.

## Best Possible Solution

Keep #41199 open until maintainers land or consolidate the focused fixes. The likely product path is to choose one canonical `sessions_send` PR (#59324 or #41255) to prefer `sessionKey` or otherwise enforce mutual exclusion with regression coverage, and separately resolve the `message` action schema/runtime behavior tracked by #46904 or an equivalent maintainer-owned patch.
