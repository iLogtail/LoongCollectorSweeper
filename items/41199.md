## Summary

Keep open. Current `main` still contains both reported parameter-conflict paths: `sessions_send` accepts both `sessionKey` and `label` in its schema and then rejects calls that include both, and the `message` tool still exposes poll-creation fields in the same flat schema used for `action: "send"` while runtime validation rejects those fields for send actions.
