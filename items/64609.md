## Summary

Keep #64609 open. Current main still has the inconsistent Telegram multi-account config behavior: `requireMention` falls back to channel-level groups, but the group/topic prompt path only reads the resolved account snapshot, where channel-level groups are intentionally suppressed in multi-account setups.

## Best Possible Solution

Keep the issue open and implement the targeted fallback from PR #64650 or equivalent in `extensions/telegram/src/bot-core.ts`: for inbound group/topic prompt and skill resolution, read `channels.telegram.accounts.<accountId>.groups ?? channels.telegram.groups`, while leaving `mergeTelegramAccountConfig` unchanged for account routing. Add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` covering two accounts plus channel-level group/topic `systemPrompt` and `skills`.
