## Summary

Keep PR #56167 open. Current main has upstream cron/direct-delivery NO_REPLY suppression that likely addresses much of the linked heartbeat symptom, but the Feishu outbound adapter still has no defense-in-depth guard before Feishu API calls. The PR is therefore not obsolete, though its own review feedback about the unguarded sendMedia fallback still needs maintainer attention before landing.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to add a complete Feishu adapter-level guard using the shared isSilentReplyText helper. The fix should cover sendText, media captions with mediaUrl, comment-thread/media fallback behavior as appropriate, and the no-mediaUrl sendMedia fallback, with focused tests for each path. If maintainers decide upstream cron suppression is sufficient, close only after opening or linking a narrower canonical item for the missing Feishu defense-in-depth behavior.
