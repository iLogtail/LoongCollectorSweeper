## Summary

Current main still exposes the reported auth-profile poisoning path: the session-memory slug generator uses an embedded run without helper-local auth failure scoping, while the runner persists billing-classified failures into shared auth profile state and Anthropic extra-usage text is still classified as billing.

## Best Possible Solution

Keep #71709 open. Fix the slug-generator call site by passing `authProfileFailurePolicy: "local"` or equivalent helper-local failure scope to `runEmbeddedPiAgent`, then add a regression test asserting slug-generator billing/auth failures fall back to timestamp/null without writing shared auth-profile failure state. Preserve user-facing/provider billing classification unless maintainers choose a broader product policy change.
