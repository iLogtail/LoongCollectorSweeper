## Summary

Keep #71709 open. Current main has the helper-local auth-profile failure policy needed for this class of bug, but the session-memory slug generator does not pass it when it calls runEmbeddedPiAgent. The exact reported Anthropic extra-usage text is still classified as billing, and billing failures still disable the shared auth profile for about 5 hours by default.
