## Summary

Keep #71709 open. Current main still has the reported slug-generator-to-shared-auth-state path: `generateSlugViaLLM` runs `temp:slug-generator` through `runEmbeddedPiAgent` without `authProfileFailurePolicy: "local"`, while the embedded runner persists non-timeout classified failures such as billing into shared auth-profile state. The helper-local policy and tests already exist, and another helper uses it, so this is a focused missing call-site fix rather than a ClawHub, duplicate, or stale-info close.

## Best Possible Solution

Keep the issue open and fix the session-memory slug generator call site by passing `authProfileFailurePolicy: "local"` to `runEmbeddedPiAgent`, then add a focused regression test in `src/hooks/llm-slug-generator.test.ts` or a nearby seam test asserting slug-generator billing/auth failures fall back locally without writing shared auth-profile failure state. Treat broader billing classifier changes as a separate maintainer decision because current tests intentionally classify Anthropic extra-usage as billing for user-facing/provider runs.
