## Summary

Keep open. Current main still persists and reuses session skill snapshots without recording the workspace that produced them, so a workspace migration can keep stale absolute `SKILL.md` paths from `sessions.json` until a separate skill version/filter refresh happens.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation: persist a normalized/canonical workspace identity or skill-root source metadata in `SessionSkillSnapshot`, rebuild when it differs from the current resolved agent workspace or when legacy snapshots lack that identity, and add regression coverage for an existing `sessions.json` snapshot with stale iCloud `resolvedSkills`/prompt paths after `openclaw.json` points to a new workspace.
