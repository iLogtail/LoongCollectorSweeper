## Summary

Current main still does not invalidate a persisted session skill snapshot when the configured workspace path changes. The snapshot has no workspace/source provenance, `agent-command` refreshes only on missing snapshot, version bump, or skill filter changes, and the embedded runner continues to trust the cached prompt/resolved skills instead of re-resolving against the current workspace.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source provenance to `SkillSnapshot`/`SessionSkillSnapshot`, rebuild when it differs from the current resolved `workspaceDir`, invalidate snapshots on `agents.defaults.workspace` and per-agent workspace config changes, and add a regression test with an old absolute `SKILL.md` path snapshot plus a current config pointing at the new workspace.
