## Summary

Keep #39719 open. Current main still has no workspace provenance in persisted skill snapshots, and the normal agent/chat refresh decisions only compare snapshot version and skill filter. A workspace move can still reuse a stale `sessions.json` snapshot with old absolute `SKILL.md` paths when the cached version is `0` or absent and the new workspace version is also `0`. The related sandbox skill-path work does not cover ordinary iCloud-to-local workspace migration.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source provenance to `SkillSnapshot` and `SessionSkillSnapshot`, compare it with the current resolved workspace before reusing cached snapshots in agent command, auto-reply, cron, and embedded/CLI runtimes, invalidate snapshots when `agents.defaults.workspace` or per-agent workspace changes, and add a regression test with a persisted old absolute `SKILL.md` path plus current config pointing at the new workspace.
