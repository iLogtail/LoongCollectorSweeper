## Summary

Keep #12855 open. Current main has a useful default-off package auto-updater under `update.auto`, but it does not implement the issue’s first-class workflow: mode selection, HH:MM/cron timezone scheduling, configured channel confirmation/notify-only delivery, or automatic post-update version/changelog/health reporting. The related PR #12896 was closed unmerged, and #65011 tracks UI/discoverability rather than replacing this workflow request.

## Best Possible Solution

Keep #12855 open as the canonical product/implementation tracker for the remaining first-class auto-update workflow. Extend the existing `update.auto` surface through config schema/types/help/docs, HH:MM or cron timezone scheduling, `confirm`/`silent`/`notify-only` modes, configured channel delivery, and post-restart version/changelog/health notifications. Keep #65011 scoped to UI/discoverability, and treat #12896 as a closed unmerged attempt.
