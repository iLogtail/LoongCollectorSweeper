## Summary

Keep PR #66985 open. Current main still has the raw `boundNode !== requestedNode` guard before node canonicalization, while the config, docs, and resolver contracts all allow node names/IPs/prefixes. The PR is a focused fix for the still-present bug tracked from #61229 and the discussion shows review feedback was addressed.

## Best Possible Solution

Keep the PR open and land a focused version that canonicalizes both `boundNode` and `requestedNode` through the existing node resolver before enforcing the bound-node guard, while preserving the stale/unresolvable bound-node failure path and adding the PR's regression coverage.
