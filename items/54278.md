## Summary

Keep open. Current main has important pieces of the requested experience: OpenAI Codex usage/quota snapshots, Control UI model-auth usage hints, automatic refresh of normal expired OAuth tokens, and doctor/log diagnostics for permanent OAuth refresh failures. The central feature is still not implemented as requested: there is no native `ok`/`expiring_soon`/`quota_wait`/`reauth_required` state model, Codex usage errors like 429 remain generic snapshot errors, and the Codex app-server bridge can refresh stored OAuth credentials but does not provide a bounded brokered host-side reauth execution path or Control UI reauth action.

## Best Possible Solution

Keep the issue open and implement it as an extension of the existing OpenAI Codex provider, provider-usage snapshot model, gateway `models.authStatus`, and Control UI model-auth card. Add an explicit quota/auth diagnosis state, map Codex rolling-window exhaustion separately from permanent OAuth refresh failures, preserve automatic refresh for normal expired access tokens, and design a bounded brokered reauth adapter only for `reauth_required` when an approved host-side adapter is configured.
