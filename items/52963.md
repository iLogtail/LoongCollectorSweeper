## Summary

Keep open. Current main still does not implement the requested dedicated Slack user DM routing: the bundled Slack setup/docs only configure bot event subscriptions, monitor replies are still sent with the bot token, and tests/docs explicitly preserve the current behavior that user-token writes are only used when no bot token is available.

## Best Possible Solution

Keep #52963 open and implement it in the bundled Slack plugin if maintainers want this product behavior: add documented user-event setup (`message.im` under Slack user event subscriptions and required user scopes), preserve a marker for user-level inbound DM events through the monitor pipeline, select the xoxp user token for those replies when `userTokenReadOnly: false`, and cover the routing/token choice with Slack monitor and send/action tests.
