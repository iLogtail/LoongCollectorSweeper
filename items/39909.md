## Summary

Keep open. Current `main` no longer rejects unknown keys under `messages.tts.providers.*`, so the issue's old validation symptom appears obsolete on `main`. But the requested feature is still not implemented: OpenAI TTS config normalization only reads a fixed set of fields, and the actual `/audio/speech` payload is hard-coded with no `extraBody` or generic provider-specific body merge.

## Evidence

- **TTS provider config now allows arbitrary provider-owned keys:** `TtsProviderConfigSchema` uses `.catchall(...)`, so unknown keys under `messages.tts.providers.<id>` are accepted by config validation on current `main`. That means the reported "Unrecognized key" symptom is likely outdated.
  - file: [src/config/zod-schema.core.ts:485](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/src/config/zod-schema.core.ts#L485)
  - command: `nl -ba src/config/zod-schema.core.ts | sed -n '480,500p'`
  - sha: [86fa8eeb68c6](https://github.com/openclaw/openclaw/commit/86fa8eeb68c64eb0ae25e1eb8c6196061862a010)
- **OpenAI TTS normalization ignores extra provider-specific fields:** `normalizeOpenAIProviderConfig` only reads `apiKey`, `baseUrl`, `model`, `voice`, `speed`, `instructions`, and `responseFormat`. There is no `extraBody` field and no pass-through of arbitrary provider parameters like `lang`.
  - file: [extensions/openai/speech-provider.ts:99](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/extensions/openai/speech-provider.ts#L99)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '99,132p'`
  - sha: [86fa8eeb68c6](https://github.com/openclaw/openclaw/commit/86fa8eeb68c64eb0ae25e1eb8c6196061862a010)
- **OpenAI TTS request body is fixed-shape:** `openaiTTS()` serializes only `model`, `input`, `voice`, `response_format`, optional `speed`, and optional `instructions` into the POST body for `/audio/speech`. There is no merge point for provider-specific payload fields.
  - file: [extensions/openai/tts.ts:111](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/extensions/openai/tts.ts#L111)
  - command: `nl -ba extensions/openai/tts.ts | sed -n '111,148p'`
  - sha: [86fa8eeb68c6](https://github.com/openclaw/openclaw/commit/86fa8eeb68c64eb0ae25e1eb8c6196061862a010)
- **User-facing docs expose only fixed OpenAI TTS settings:** The OpenAI provider docs list only model, voice, speed, instructions, responseFormat, apiKey, and baseUrl for `messages.tts.providers.openai`; no `extraBody` or equivalent is documented.
  - file: [docs/providers/openai.md:331](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/docs/providers/openai.md#L331)
  - command: `nl -ba docs/providers/openai.md | sed -n '331,339p'`
  - sha: [86fa8eeb68c6](https://github.com/openclaw/openclaw/commit/86fa8eeb68c64eb0ae25e1eb8c6196061862a010)
- **No repo-wide `extraBody` support for TTS/OpenAI speech:** A repo search for `extraBody` in `src`, `extensions`, and `docs` found no OpenAI TTS implementation or docs using that config key.
  - command: `rg -n "extraBody" src extensions docs -g '!node_modules'`
