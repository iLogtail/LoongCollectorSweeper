## Summary

Keep #22770 open. Current main still does not implement a $VISUAL/$EDITOR external editor compose flow for the TUI composer, and the provided issue discussion already includes a maintainer-side Codex review keeping it open. Related #55836 is adjacent vi/vim in-place editing work, not a duplicate or shipped replacement.

## Best Possible Solution

Implement this in the core TUI composer: add a deliberate shortcut or command, suspend the TUI with the existing suspension helper, write the current composer contents to a temporary markdown file, launch $VISUAL then $EDITOR, wait for exit, read the saved contents back into CustomEditor without auto-submitting, surface missing-editor/non-zero-exit/file-I/O errors, update the TUI docs, and add focused tests for the roundtrip and error cases. Treat #55836 as related but separate in-place vi/vim editing work.
