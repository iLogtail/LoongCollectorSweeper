## Summary

Keep #22770 open. Current main still does not provide a TUI external-editor compose flow: the CustomEditor shortcut surface has no Ctrl+E/external editor hook, runTui only wires existing picker/toggle/exit handlers and uses TUI suspension for auth, slash commands have no edit/compose command, and the public TUI docs do not document $VISUAL/$EDITOR handoff. The related #55836 remains only adjacent vi/vim in-place editing, not this temp-file editor roundtrip.

## Best Possible Solution

Implement this in the core TUI composer rather than closing the issue: add a dedicated shortcut or slash command, suspend the TUI, write the current editor text to a temporary markdown file, launch $VISUAL then $EDITOR and wait, read the file back into CustomEditor with setText/getExpandedText semantics, do not auto-submit, surface missing-editor/non-zero-exit/file I/O status messages, update docs/web/tui.md, and add focused CustomEditor/TUI tests. Keep #55836 as the separate in-place vi/vim keybinding request.
