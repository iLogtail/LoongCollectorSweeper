## Summary

Keep #22770 open. Current main still does not implement a $VISUAL/$EDITOR external editor compose flow for the TUI composer. The related #55836 remains a distinct open request for in-place vi/vim keybindings, and the latest issue discussion already contains a Codex review comment from steipete keeping this item open for the same product gap.

## Best Possible Solution

Implement this in the core TUI composer rather than closing or moving it to ClawHub: add a dedicated shortcut or command that suspends the TUI, writes the current composer text to a temporary markdown file, launches $VISUAL then $EDITOR, waits for exit, reads the saved content back into CustomEditor without auto-submitting, reports missing editor/non-zero exit/file I/O errors in the status surface, updates TUI shortcut docs, and adds focused tests. Resolve the Ctrl+E conflict with the existing editor line-end behavior before choosing the final default key.
