## Summary

Keep #58932 open. Current main still uses ASCII-only slug normalization, the active group display-name path still depends on that helper, and the existing tests do not cover the reported CJK/Cyrillic/Arabic behavior. The discussion usefully points to fix PR #58995, with related attempts #58942 and #59068, but the provided PR state shows those fixes are still open/unmerged and #58973 was closed unmerged.

## Best Possible Solution

Keep the issue open until a focused fix lands on main. The best path is to update the shared slug normalizers to preserve Unicode letters/numbers, add regression tests in `src/shared/string-normalization.test.ts`, and verify `buildGroupDisplayName` keeps non-Latin group labels while preserving current ASCII outputs. Maintainers should pick or consolidate one of the open fix PRs (#58942, #58995, or #59068) rather than closing the canonical bug report first.
