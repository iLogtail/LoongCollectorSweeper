## Summary

Keep PR #33914 open. Current main only covers the message-hook sessionKey slice; it still does not expose sourceProvider, senderId, senderName, senderIsOwner, groupId, or spawnedBy on PluginHookAgentContext, and normal agent plus compaction hook emission paths still build contexts without those trust fields. This is a real core plugin API gap, not ClawHub-only work, because external plugins cannot add missing core hook context fields themselves.

## Best Possible Solution

Keep this open for maintainer review or a narrowed rebase. The best product/code path is an additive plugin SDK/context update that exposes sourceProvider, senderId, senderName, senderIsOwner, groupId, and spawnedBy on PluginHookAgentContext; derives sourceProvider from normalized origin metadata; preserves documented null semantics; threads the fields through all agent, queued, memory, compaction, reset, and relevant message hook emission paths; and keeps the already-landed message-hook sessionKey work as baseline rather than reintroducing stale changes.
