## Summary

Keep open. Current main still does not implement the central PR request: `PluginHookAgentContext` does not expose `sourceProvider`, sender identity, `groupId`, or `spawnedBy`, and the main agent/compaction hook emission paths still build minimal contexts. The message-hook `sessionKey` slice has landed independently, but that only covers a small part of PR #33914. The PR remains a real public plugin API gap, supported by plugin-author discussion, rather than a cleanup-close candidate.

## Best Possible Solution

Keep PR #33914 open for maintainer review. The best product path is an additive plugin SDK/context update that exposes `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`, threads them consistently through agent, queued, memory, compaction/reset, and relevant outbound hook emission paths, preserves the already-landed message-hook `sessionKey` work, and lands with contract tests, docs, and API baseline updates.

## Risks / Open Questions

- The PR branch is large and has repeatedly been rebased against moving hook/compaction code, so maintainers likely need a careful rebase or cherry-pick rather than a blind merge.

- This changes the public plugin SDK contract, so a final landing path should include API baseline/docs updates and focused tests for propagation plus null/undefined semantics.
