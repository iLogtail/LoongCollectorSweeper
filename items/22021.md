## Summary

Keep #22021 open. Current main still lacks `X-Actual-Model` or an equivalent OpenAI-compatible transport response contract. The existing maintainer/Codex issue comment remains accurate: HTTP handlers still return/stream the request-side agent target model, while actual provider/model data exists deeper in agent metadata and session persistence but is not surfaced as an HTTP response header or documented response field.

## Best Possible Solution

Keep the issue open. Thread resolved provider/model metadata from `agentMeta`, `onModelSelected`, or a new run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`; define the non-streaming response header contract, decide SSE semantics, document it on the public Gateway HTTP API pages, and add focused tests in both Gateway HTTP test files.
