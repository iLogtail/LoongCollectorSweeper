## Summary

Keep #22021 open. Current main still does not implement `X-Actual-Model` or an equivalent transport-layer response contract for the resolved runtime provider/model. The prior Codex/maintainer comment in the issue remains consistent with `bc24b547`: the OpenAI-compatible Gateway surfaces use the request-side agent target model in response bodies and SSE events, while the actual selected-model callback exists deeper in the reply pipeline and is not wired into these HTTP handlers.

## Best Possible Solution

Keep the issue open. The best implementation path is to thread the resolved provider/model from `GetReplyOptions.onModelSelected` or an equivalent run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`, set a documented response header for non-streaming and streaming responses where HTTP semantics allow it, and add focused coverage in `src/gateway/openai-http.test.ts` and `src/gateway/openresponses-http.test.ts`. Update the public Gateway docs once the exact header value format is decided.
