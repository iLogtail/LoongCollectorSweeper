## Summary

Keep #69422 open. Current main still lacks a Google Chat thread carrier through the outbound reply payload, block-streaming coalescer, and follow-up queue path. The issue discussion includes multiple independent reproductions and a source mapping comment that still matches current main; related closed items do not cover the full remaining bug.

## Best Possible Solution

Keep #69422 open as the canonical broad bug. The fix should make Google Chat thread metadata first-class from inbound webhook context through OutboundReplyPayload, block reply keys/conflict checks, Google Chat reply delivery, and FollowupRun/routeReply; add regression tests for long streamed replies and concurrent queued followups from the same Google Chat thread. Also align the interim config mitigation by adding googlechat to messages.queue.byChannel schema if maintainers want a channel-scoped workaround.
