## Summary

Keep #61009 open. Current main still has a docs/runtime mismatch for the sandbox-active case: docs say per-call `host=node` is allowed from `auto`, but the runtime rejects `host=node` from `auto` whenever a sandbox runtime is available, and tests explicitly lock in that rejection. The issue comments asserting resolution appear to have missed the node-with-sandbox rejection case.

## Best Possible Solution

Keep the issue open for a narrow docs/runtime decision. The conservative fix is probably to update `docs/tools/exec.md` and the published docs to say per-call `host=node` from `auto` is only allowed when no sandbox runtime is active, matching `isRequestedExecTargetAllowed` and existing tests. If maintainers instead want node override to escape an active sandbox, the runtime and tests should be changed intentionally.
