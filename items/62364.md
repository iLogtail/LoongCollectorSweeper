## Summary

Keep #62364 open. Current main still makes the first `models.list` and some `chat.history` calls wait on the cold gateway model catalog, provider catalog discovery still runs serially, ordinary startup discovery has no default timeout, and startup prewarm still runs before channel startup. Existing startup-gating is only a partial mitigation, not the requested lazy, parallel, or optional provider validation behavior.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make the cold model-catalog path non-blocking or bounded: defer or move `prewarmConfiguredPrimaryModel()` after channel startup, avoid making `chat.history` wait on the catalog just to compute a thinking default, parallelize or yield between provider catalog hooks, add a normal-startup timeout or stale cached fallback for `models.list`, and add regression/timing coverage around `ensureOpenClawModelsJson`, provider discovery, `models.list`, and `chat.history`.
