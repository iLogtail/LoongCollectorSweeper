## Summary

Current main partially addresses the related hook-path concern from #70118 by adding an optional `beforeDeliver` seam and wiring `message_sending` for inbound dispatcher replies, but it does not implement the issue’s requested universal fail-closed structured-output validation gate. The dispatcher still calls `options.deliver(...)` after optional hook processing, the inbound hook wrapper is text-only and absent when no hook exists, and presentation handling normalizes/degrades structured payloads rather than validating and blocking invalid structured output pre-send.

## Best Possible Solution

Keep the issue open and scope a real shared validator for structured operator-facing reply payloads. The likely implementation belongs at or just before `src/auto-reply/reply/reply-dispatcher.ts`’s `beforeDeliver`/`deliver` handoff, backed by a strict `ReplyPayload` presentation/interactive/channelData validation helper and regression tests covering text-only preservation, structured payload acceptance, structured payload rejection, hook preservation, and inbound/native/tool/followup delivery paths.
