## Summary

Keep open. Current main has meaningful partial coverage for restart-scoped continuation, but it does not implement the RFC’s durable task checkpoint lifecycle, explicit paused/resuming/resumed/failed_resume states, or plan/artifact checkpoint model for in-progress tasks across gateway restarts. The related merged PR #63406 also explicitly scoped itself away from adding a durable continuation queue.

## Best Possible Solution

Keep #60864 open unless maintainers explicitly consolidate it into #42816. The implementation path should build on the existing restart sentinel, session-delivery queue, main-session/subagent recovery, and Task Flow concepts, but add the missing durable checkpoint schema, explicit task/run status transitions, atomic claim/at-most-once resume semantics, plan cursor/artifact context, user-visible failure states, and focused tests for restart-required and startup-resume flows.
