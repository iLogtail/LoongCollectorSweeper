## Summary

Keep PR #55001 open. Current main still has the reported `/queue` directive loss: `resolveReplyDirectives` treats `hasQueueDirective` as an inline directive, then clears inline directives when a prompt remains, and `clearInlineDirectives` explicitly resets every queue field. The supplied PR context shows the author added a second commit to restore the queue raw/option fields called out by bot review, so the PR remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open or reimplement the same fix on main: exclude queue directives from the generic inline-clear trigger, preserve the full queue parse metadata when other inline directives trigger clearing, and add regression coverage for `/queue steer <prompt>` and `/queue debounce:500ms /think <prompt>` so the downstream per-message queue mode/options and validation paths remain intact.
