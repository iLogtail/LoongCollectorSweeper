## Summary

Keep PR #39894 open. Current main still does not implement the central fix: outbound payload planning parses raw reply text before delivery and JSON projection, so echoed inbound metadata can remain visible text while MEDIA directives are parsed. The PR is draft and mixed with unrelated CI/macOS/test-runner edits, but its core outbound normalization fix and regression remain useful for the still-open canonical bug #39847.

## Best Possible Solution

Keep PR #39894 or canonical issue #39847 open until current main lands a narrow core fix: strip copied inbound metadata before `parseReplyDirectives()` in `src/infra/outbound/payloads.ts`, and add a regression in `src/infra/outbound/payloads.test.ts` covering `Conversation info (untrusted metadata)` followed by visible text plus `MEDIA:`. Split unrelated CI/macOS/test-runner edits into separate work if still needed.
