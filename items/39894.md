## Summary

Keep open. Current main still does not strip echoed inbound metadata before outbound reply directive parsing, so PR #39894 is not obsolete and still contains useful unmerged bug-fix work for linked issue #39847. The PR likely needs splitting or rebasing because it bundles unrelated CI/macOS test-runner changes with the outbound fix.

## Best Possible Solution

Keep this PR open as useful unmerged bug-fix work, but ask for a focused rebase or split: outbound payload normalization should stripInboundMetadata(payload.text ?? "") before parseReplyDirectives(), and the metadata-plus-MEDIA regression test should land with that focused change. Track or separate the unrelated CI/macOS changes outside this bug fix.
