## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading for normal non-heartbeat runs. HEARTBEAT.md is still part of the unconditional workspace bootstrap read/cache path, and current policy keeps it for default-agent normal runs when heartbeat guidance is enabled. The closed #55311 lightContext fix addresses lightweight heartbeat runs, but not this issue's requested non-heartbeat omission before disk reads and snapshot caching.

## Best Possible Solution

Keep #13968 open. Implement a run-kind-aware bootstrap loader/cache contract that excludes HEARTBEAT.md for default non-heartbeat runs before disk reads, session snapshot caching, hooks, and prompt injection, while preserving HEARTBEAT.md for actual heartbeat runs and the closed #55311 lightweight heartbeat behavior. Add focused tests for enabled default non-heartbeat omission, cache behavior, disabled cadence/prompt regressions, and heartbeat retention, then update the public heartbeat docs if the contract changes.
