## Summary

Current main still does not implement the requested strict on-demand HEARTBEAT.md loading. The code now has run-kind-aware filtering for some cases, but HEARTBEAT.md is still part of the unconditional workspace bootstrap read/cache path and normal default-agent runs still keep it when heartbeat guidance is enabled.

## Best Possible Solution

Keep #13968 open. Implement a run-kind-aware bootstrap loading and cache contract that can exclude HEARTBEAT.md for default non-heartbeat runs before disk reads and session snapshot caching, while preserving HEARTBEAT.md for actual heartbeat runs and lightweight heartbeat context. Add focused tests for enabled default non-heartbeat omission, disabled/cadence regressions, cache behavior, and heartbeat retention, then update heartbeat/system-prompt docs to describe the final contract.
