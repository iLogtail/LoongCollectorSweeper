## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading. Non-heartbeat default runs can still read and inject HEARTBEAT.md when heartbeat guidance is enabled, and the loader still includes HEARTBEAT.md before any run-kind filtering.

## Best Possible Solution

Keep #13968 open. Implement a run-kind or isHeartbeat-aware bootstrap loading path that omits HEARTBEAT.md for default non-heartbeat runs before disk read, cache reuse, hook processing, and prompt injection, while preserving HEARTBEAT.md for actual heartbeat runs and preserving heartbeat lightContext behavior. Add focused tests for default enabled omission, heartbeat retention, disabled cadence/prompt-section behavior, cache behavior, and #55311 lightContext forwarding; then update heartbeat/system-prompt/token-use docs.
