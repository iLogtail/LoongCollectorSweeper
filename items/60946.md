## Summary

Keep open. Current main has partial edit-error hardening, but it does not implement the requested rich tool error UX across file edits, apply_patch, exec failures, and generic tool bridges. The related title-search items are about different tool/message bugs and do not supersede this request.

## Best Possible Solution

Keep the issue open and implement a core/tooling error-enrichment path with focused tool-specific enrichers: edit/apply_patch should report closest and multiple matches with line snippets and concrete retry steps; exec command-not-found should provide deterministic local suggestions where safe; generic adapter and MCP bridge errors should carry structured, model-visible troubleshooting fields. Add focused regression tests for edit mismatch, multiple matches, apply_patch expected-lines failures, exec command-not-found, and generic tool bridge errors.
