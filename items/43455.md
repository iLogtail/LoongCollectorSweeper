## Summary

Current main has not implemented the requested Browserbase session lifecycle. The Browserbase docs still recommend the static `cdpUrl`, and the browser code still treats profiles as already-resolved CDP URLs rather than creating Browserbase sessions and using returned connection URLs.

## Best Possible Solution

Keep #43455 open. The best fix is in the bundled browser plugin/docs: either implement Browserbase-aware session lifecycle support around `browser.profiles.*` configuration, including API key/project id handling, `POST /v1/sessions`, `connectUrl` use, cleanup semantics, and regression tests, or revise the Browserbase docs to remove the broken static-URL claim and document a supported manual `connectUrl` workflow.
