## Summary

Current main does not implement the requested Browserbase session lifecycle. The docs still recommend the static Browserbase CDP URL and say no manual session creation is needed, while the browser runtime only connects to the configured CDP URL/fallback endpoint and has no Browserbase Sessions API create/close flow.

## Evidence

- **Docs still advertise the broken static Browserbase URL:** The Browserbase docs section still shows `cdpUrl: "wss://connect.browserbase.com?apiKey=<BROWSERBASE_API_KEY>"` and states Browserbase auto-creates a session on WebSocket connect, matching the issue's reported outdated guidance.
