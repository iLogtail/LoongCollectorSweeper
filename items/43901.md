## Summary

Keep open. Current `main` at `50e36983bb2d827adc617282c090c0a0359c6eee` still contains the L1 bootstrap snapshot cache and still bypasses the inode/mtime-validated workspace file cache for session-keyed runs, so the PR's core bug is not already implemented on `main` and remains plausible.

## Evidence

- **L1 cache still exists on main:** `getOrLoadBootstrapFiles` still stores `WorkspaceBootstrapFile[]` in a `Map<string, ...>` keyed only by `sessionKey`, with no file identity, mtime, or inode validation.
  - file: [src/agents/bootstrap-cache.ts:3](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/bootstrap-cache.ts#L3)
  - command: `nl -ba src/agents/bootstrap-cache.ts`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Session-keyed bootstrap resolution still uses the L1 cache:** `resolveBootstrapFilesForRun` still calls `getOrLoadBootstrapFiles(...)` whenever `params.sessionKey` is present, which short-circuits the lower-level workspace loader for the common session-keyed path.
  - file: [src/agents/bootstrap-files.ts:229](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/bootstrap-files.ts#L229)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '224,246p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **L2 workspace cache already validates file identity:** `readWorkspaceFileWithGuards` computes a cache key from `dev:ino:size:mtimeMs` and only reuses cached content when that identity matches, which is exactly the invalidation behavior the PR says the L1 cache bypasses.
  - file: [src/agents/workspace.ts:46](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/workspace.ts#L46)

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Current rollover invalidation is conditional and incomplete:** `clearBootstrapSnapshotOnSessionRollover` returns early unless both `sessionKey` and `previousSessionId` are present, so stale snapshots are not cleared for paths that reuse an existing session entry without a previous-session rollover.
  - file: [src/agents/bootstrap-cache.ts:23](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/bootstrap-cache.ts#L23)

  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
