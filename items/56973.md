## Summary

Keep open. Current main still has the two exported `normalizeChannelId` wrappers, the plugin barrel still re-exports the ambiguous name, and the cited call site still aliases both imports. The request remains concrete and actionable, though any removal needs public SDK compatibility care.

## Best Possible Solution

Keep the issue open. The likely fix is to migrate internal call sites to `normalizeChatChannelId` and `normalizeAnyChannelId`, export a canonical `normalizeAnyChannelId` from the relevant plugin SDK subpath, and keep `normalizeChannelId` only as a deprecated compatibility alias until a breaking-change window.
