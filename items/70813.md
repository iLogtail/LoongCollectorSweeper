## Summary

Keep #70813 open. Current main does not implement WhatsApp edit or unsend actions: the bundled WhatsApp channel advertises only react/poll, gates runtime action support to react, and its active listener/send API expose send, poll, reaction, and typing methods but no edit/delete wrapper. The PR discussion and diff describe a concrete, recently active implementation with tests and follow-up on bot review feedback, so this is not stale, duplicate, or ClawHub-only cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish reviewing the WhatsApp extension implementation, ensure the bot review feedback is actually reflected in the latest PR head, validate the Baileys edit/delete contract and action-gating/security behavior, then either land the PR or request focused changes. Closing now would drop a concrete implementation for behavior that current main still lacks.
