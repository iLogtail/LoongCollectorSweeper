## Summary

Keep PR #70813 open. Current main does not implement WhatsApp edit/unsend message actions: the WhatsApp plugin advertises only react/poll, supports only react through the channel action adapter, and the runtime/send API has no edit or unsend path. The shared message-action contract already includes edit and unsend, so this is a plausible WhatsApp-plugin implementation rather than ClawHub or out-of-repo cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a WhatsApp-plugin-owned implementation that advertises edit/unsend only when message sends are enabled, routes both actions through the existing gateway action surface, reuses outbound target authorization, constructs Baileys own-message keys with fromMe: true, and adds the focused unit/seam tests plus a WhatsApp docs update for the new actions and gating.
