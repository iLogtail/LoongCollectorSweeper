## Summary

Keep open. Current `main` still surfaces raw media fetch failure text through outbound error paths, so this is not implemented on `main`, and the PR discussion already identified a diagnostic regression in the proposed patch that still needs maintainer judgment.

## Risks / Open Questions

- Merging the PR as last discussed could regress non-media debugging by collapsing non-`Error` throws to `unknown error` unless that review point is addressed.
