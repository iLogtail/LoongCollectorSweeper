## Summary

Keep open. Current main has broader mitigations for user-visible reply/media failures, but it does not implement this PR's MediaFetchError-specific sanitization at the message-action hydration, broadcast aggregation, or delivery hook error boundaries. The PR also has an unresolved review concern about preserving non-Error diagnostics, so the best maintainer path is to rework or supersede it with a targeted fix and regression tests rather than auto-close it.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. The best fix is a narrow MediaFetchError-aware sanitizer at the message action and delivery boundaries, with tests proving no group IDs, phone numbers, names, or local paths reach channel-facing output, while preserving String(err) for unrelated non-Error failures. If maintainers decide the broader reply/tool-error protections are sufficient, close only after documenting that exact behavior against the #20279 reproduction path.
