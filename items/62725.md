## Summary

Keep open. Current main still supports typingMode and typingIntervalSeconds only at agents.defaults and session, not as agents.list[] per-agent overrides. The strict agent-entry schema would still reject typingMode on an agents.list[] item, and normal reply plus heartbeat typing resolution read defaults/session rather than a per-agent merged typing config.

## Best Possible Solution

Implement a narrow per-agent typing contract: add typingMode and typingIntervalSeconds to AgentConfig, AgentEntrySchema, generated schema/help metadata, and docs; expose them through the resolved per-agent config path; then make normal replies and heartbeat typing resolve precedence as session override, agents.list[] override, agents.defaults fallback. Add schema acceptance/rejection tests and runtime precedence tests for both mode and interval.
