## Summary

Keep open. The issue specifically reports that the `[object Object]` hallucination persisted after applying the heartbeat trigger-policy fix from #69278, and current `main` only proves that heartbeat prompt injection is now limited to heartbeat-triggered runs. I found diagnostics that would help reproduce it, but no source, test, changelog, or related-item evidence that the post-fix fresh-session hallucination itself has been fixed or conclusively traced to external translator behavior.

## Best Possible Solution

Keep #70734 open as a focused follow-up to #69079/#69278. Ask for a current-main reproduction using `/export-trajectory` or equivalent prompt/provider-boundary capture, plus the translator before/after payload. If OpenClaw's `prompt.submitted` and provider request are clean, close or redirect to the external translator; if `[object Object]` or stale support context appears before the provider boundary, fix that prompt composition or memory/bootstrap source in core and add a regression test.
