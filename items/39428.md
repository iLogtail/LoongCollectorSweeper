## Summary

Keep #39428 open. Current main still has no Discord accountId-to-agentId fallback: after explicit binding tiers, the generic resolver falls directly to the configured default agent. The related fix PR #39485 is the right pending implementation path, but it is not merged or present on main.

## Best Possible Solution

Keep #39428 open until PR #39485 or an equivalent maintainer-approved change lands. The likely fix belongs in the generic routing path after explicit binding tiers and before default fallback, should guard the literal default account from surprising overrides, and should add regression coverage for Discord accountId matching plus default-account behavior.
