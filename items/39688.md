## Summary

Keep open. Current main still emits `message:received` and `message:sent` internal hooks without retaining the hook event or routing messages pushed into `event.messages`. The related fix work remains open in the provided GitHub context, with PR #41375 looking like the likely maintainer path.

## Best Possible Solution

Keep #39688 open until maintainers merge PR #41375 or an equivalent fix that drains internal hook replies only on reply-capable surfaces, adds regression coverage for hook-produced message delivery, and updates the hook docs/contracts to clarify which events can auto-reply.
