## Summary

Current main still does not deliver `event.messages` returned by `message:received` or `message:sent` internal hooks. The relevant paths fire-and-forget `triggerInternalHook(...)` and never await the event or drain `hookEvent.messages` into `routeReply`/outbound delivery. This is a real, still-actionable behavior bug with related open fix PRs, including maintainer-labeled PR #41375, so it should stay open for maintainer resolution.

## Best Possible Solution

Keep #39688 open and resolve it through the maintainer-reviewed hook reply work. The best path is to land or revise the canonical fix so `message:received` hook replies are awaited and drained only when a safe reply target exists, then update the public hooks docs to state exactly which internal hook surfaces auto-deliver `event.messages`. If `message:sent` should remain lifecycle-only, close that part by documenting the boundary rather than silently dropping messages under a contract that says they are sent.
