## Summary

Keep #72220 open. Current main still creates runtime-deps staging temp directories without owner metadata and deletes every `.openclaw-runtime-deps-*` entry before staging, so the cleanup race described by the PR remains plausible on the active build path. The adjacent bundled runtime-deps install lock is separate runtime repair/install logic and does not guard `runtime-postbuild` staging cleanup. The PR is still useful, but the backup-directory owner gap noted in review should be addressed or explicitly accepted before merge.

## Best Possible Solution

Keep PR #72220 open for maintainer review. The best path is to land this PR or an equivalent fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that stamps active runtime-deps staging temp dirs with owner metadata, skips cleanup for live owners, preserves or deliberately scopes backup-directory safety in `replaceDirAtomically`, and keeps the concurrent-cleanup regression test in `test/scripts/stage-bundled-plugin-runtime-deps.test.ts`.
