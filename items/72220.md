## Summary

Keep #72220 open. Current main still has the unguarded build-time runtime-deps temp cleanup path the PR targets: staging, stamp, and backup temp dirs in `scripts/stage-bundled-plugin-runtime-deps.mjs` are ownerless, and `removeStaleRuntimeDepsTempDirs` deletes every `.openclaw-runtime-deps-*` directory before staging. The existing runtime install lock in `src/plugins/bundled-runtime-deps.ts` is adjacent but does not supersede this build/postbuild staging race. The PR is recent, focused, and includes targeted regression coverage; Greptile’s backup-dir owner gap should be addressed or accepted in normal review rather than closing this cleanup item.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this PR or an equivalent fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active staging temp dirs with an owner pid, skips cleanup for live owners, covers the concurrent cleanup regression, and either fixes or consciously accepts the backup-dir owner gap called out by review.
