## Summary

Keep #72220 open. Current main still has the build-time runtime-deps cleanup race the PR targets: staging, install, stamp, and backup temp dirs are ownerless, and `removeStaleRuntimeDepsTempDirs` deletes every `.openclaw-runtime-deps-*` entry before staging. Adjacent bundled-runtime lock work protects runtime install/mirror paths, not this `runtime-postbuild` staging cleanup. The PR is recent, focused, and has targeted regression coverage; Greptile's backup-dir owner gap should be handled in normal review rather than closing the PR.

## Best Possible Solution

Keep the PR open for maintainer review. Land this PR or an equivalent fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active runtime-deps staging temp dirs with owner metadata, skips cleanup for live owners, includes the concurrent cleanup regression test, and resolves or deliberately accepts the backup-dir owner gap called out in review.
