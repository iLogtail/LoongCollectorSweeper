## Summary

Keep #27574 open. Current main at e23d17da79dc2345ca55902e632d3c7622accacc still has browser tooling for tabs, screenshots, snapshots, console/errors, and actions, but not a live browser preview panel in the Control UI/Dashboard. This should not be closed as ClawHub/plugin work yet because the current plugin API exposes tools, hooks, routes, gateway methods, CLI, services, channels, providers, and similar runtime seams, but no supported Control UI view/panel contribution seam. The related #63926 overlaps, but remains open and is narrower around login handoff/browser management rather than the broader remote-debugging live preview requested here.

## Best Possible Solution

Continue tracking #27574. The best path is either a core Control UI browser preview backed by the existing browser.request/control API, or first adding a supported plugin API for Control UI panel/view contributions and then implementing the browser preview as a plugin. The resulting surface should cover current tab URL/title, refreshed screenshots or streaming preview, console/page errors, active tabs, and carefully scoped manual controls, while coordinating with but not automatically replacing #63926.
