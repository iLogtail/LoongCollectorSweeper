## Summary

Keep PR #70681 open. Current main still contains the Unix `lsof -Fpc` command-name filter that the PR targets, and the linked bug #70664 remains open with a concrete macOS reproduction. The PR is not obsolete, but it needs maintainer review or author follow-up because the bot review identified two timeout fixes in the new `pollPortOnce` verifier calls and GitHub reports the PR branch as dirty against current main.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The product fix belongs in `src/infra/restart-stale-pids.ts`: parse listener PIDs independently from lsof command names, add Unix argv verification using the existing `isGatewayArgv` contract, apply that verifier in `findGatewayPidsOnPortSync` and `pollPortOnce`, pass `POLL_SPAWN_TIMEOUT_MS` on the polling path, and update the focused restart-stale-pids/restart tests to cover macOS `cnode` plus `ps` argv `openclaw-gateway`.
