## Summary

Keep #70681 open. Current main still has the Unix `lsof -Fpc` command-name filter that the PR is trying to replace, and the stale-cleanup path still calls that filtered result directly. The PR is a focused, plausible fix for the linked macOS bug, but its own review thread has actionable follow-up before merge.

## Best Possible Solution

Keep the PR open and either update it or land an equivalent main-branch fix: parse Unix lsof output as candidate listener PIDs, verify gateway identity via argv before cleanup and poll classification, preserve the ancestor/self exclusion, use `POLL_SPAWN_TIMEOUT_MS` on poll-path verifier calls, and update the restart-stale-pids tests that currently assert lsof command-name filtering.
