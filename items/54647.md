## Summary

Current `main` does not implement the ACP session-model reporting fix described by PR #54647, so this is not closeable as already implemented or obsolete. The PR also appears to need follow-up before merge because the proposed identity-path change conflicts with existing wrapper-provider preservation behavior.

## Evidence

- **ACP runtime model metadata exists but current identity resolver ignores it:** `SessionAcpMeta.runtimeOptions.model` is part of the persisted session schema, but `resolveSessionModelIdentityRef()` only checks `entry.model`, `entry.modelProvider`, `fallbackModelRef`, and then falls back to `resolveSessionModelRef()`; it never reads `entry.acp.runtimeOptions.model`.
  - file: [src/gateway/session-utils.ts:1186](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/session-utils.ts#L1186)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1186,1325p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Gateway session rows still default missing model identity to DEFAULT_MODEL:** `buildGatewaySessionRow()` resolves model identity through `resolveSessionModelIdentityRef()` and then substitutes `DEFAULT_MODEL` when the resolver has no model. That means ACP rows with no explicit `entry.model` still report a default model instead of leaving it unknown, which is exactly the behavior the PR is trying to change.
  - file: [src/gateway/session-utils.ts:1284](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/session-utils.ts#L1284)

- **Current tests do not cover the ACP sessions-list regression:** The `session-utils` tests on `main` cover selected-model display and generic identity resolution, but there is no ACP-specific case for `resolveSessionModelIdentityRef()` or `listSessionsFromStore()` reading `acp.runtimeOptions.model` / suppressing default-model fallback for ACP rows.
