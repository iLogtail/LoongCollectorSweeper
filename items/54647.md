## Summary

Keep PR #54647 open. Current main still builds `sessions.list` rows through the generic session model resolver, which ignores `entry.acp.runtimeOptions.model` and falls back to configured defaults. The PR targets a real remaining bug from #54640, but the patch also has a review concern about preserving explicit wrapper `modelProvider` values when model ids contain provider/vendor prefixes, so it should be revised rather than closed.

## Best Possible Solution

Keep the PR open and revise it: make `buildGatewaySessionRow` ACP-aware, read `entry.acp.runtimeOptions.model` for ACP rows, avoid default-model fallback when an ACP runtime model is unknown, preserve explicit `modelProvider` as authoritative for prefixed model ids, and add focused regression tests in `src/gateway/session-utils.test.ts`.
