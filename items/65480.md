## Summary

Keep open. Current main still initializes QMD collection adds with `--glob`, and the fallback only runs when QMD rejects that option. That does not address the reported QMD 2.0.1/2.1.0 behavior where `--glob` is accepted or ignored and custom patterns fall back to `**/*.md`. The related #65481 remains the focused open fix proposal, and the issue discussion includes a second macOS/arm64 confirmation with a working `--mask` workaround.

## Best Possible Solution

Keep #65480 open until a focused memory-core fix lands. The best code path is to prefer `--mask` for QMD collection patterns by default, keep a fallback to `--glob` for compatibility, and update `qmd-compat`, `qmd-manager` regression tests, and QMD docs language so custom `memory.qmd.paths`, transcript collections, and `memorySearch.qmd.extraCollections` are covered against silent `--glob` ignores.
