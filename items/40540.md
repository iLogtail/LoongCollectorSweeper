## Summary

Keep open. Current main still performs package-manager self-update from the running CLI process, only does gateway restart/recovery after the update succeeds, and has no EBUSY-specific recovery path. The Windows same-guest smoke harness still stops gateway processes before invoking `openclaw update`, so there is not strong evidence that issue #40540 is obsolete or fully fixed in core.
