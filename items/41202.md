## Summary

Current main already implements the requested direction: OpenClaw exposes structured filesystem tools (`read`, `write`, `edit`, and `apply_patch`) so common file reads/edits do not need raw `exec`, and this shipped in `v2026.4.22`.

## Risks / Open Questions

- `apply_patch` is provider-gated to OpenAI/OpenAI Codex, so non-OpenAI models still rely on `read`/`write`/`edit` plus `exec` for some workflows.
- If the reporter meant model prompt behavior rather than tool availability, some models may still sometimes choose `exec`; that would need a fresh issue with a concrete repro transcript.

## Close Comment

Closing as implemented on main after a Codex review. OpenClaw already ships structured filesystem tools so common file reads/edits do not need raw `exec`: `createOpenClawCodingTools()` wires in `read` / `write` / `edit`, removes the base `bash` tool, and adds `apply_patch` when supported (`src/agents/pi-tools.ts:453`, `src/agents/apply-patch.ts:97`).
