## Summary

Keep #54128 open. Current main still does not expose `memorySearch.local.maxThreads` or pass a thread-count option to `node-llama-cpp` for local embeddings; the config, schema, docs, and local embedding tests only cover `modelPath`, `modelCacheDir`, and `contextSize`.

## Best Possible Solution

Add a typed `memorySearch.local.maxThreads` positive-integer config, thread it through resolved memory config and the local embedding provider, update the node-llama wrapper/types to pass `getLlama({ logLevel, maxThreads })`, and add focused tests plus docs/schema/help updates. Maintainers should also decide whether a separate per-context `threads` option is needed, since node-llama-cpp distinguishes the Llama instance limit from context evaluation threads.
