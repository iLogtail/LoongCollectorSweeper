## Summary

Keep open. Current `main` still documents the Windows one-liner `iwr -useb https://openclaw.ai/install.ps1 | iex`, and the installer script still calls `exit 1` directly on multiple failure paths with no pause/readkey/no-exit handling. The same behavior is present in the latest release tag `v2026.4.22`, so there is not strong evidence that this issue was fixed on shipped OpenClaw.
