## Summary

Keep open. Current `main` still appears to bypass request-policy attribution on the media-understanding image path, and the submitted patch is not the best fix because it uses raw provider-based attribution instead of the current endpoint-gated request-policy helper used elsewhere.

## Evidence

- **Current image path still calls `complete()` without request-policy headers:** `describeImagesWithModelInternal()` builds `completeImage()` with only `apiKey`, `maxTokens`, `signal`, and optional `onPayload`; it does not pass `headers` or route through `resolveProviderRequestPolicyConfig`/`resolveProviderRequestHeaders` before calling `complete()`.
  - file: [src/media-understanding/image.ts:320](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/media-understanding/image.ts#L320)
  - command: `nl -ba src/media-understanding/image.ts | sed -n '300,430p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Current main has centralized, endpoint-gated attribution policy:** `resolveProviderRequestPolicy()` only enables OpenRouter attribution on known OpenRouter/default routes and only enables hidden OpenAI attribution on verified native OpenAI/Codex routes. `resolveProviderAttributionHeaders()` alone is provider-only and does not apply that base-URL/API gating.
  - file: [src/agents/provider-attribution.ts:524](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/provider-attribution.ts#L524)
  - command: `nl -ba src/agents/provider-attribution.ts | sed -n '380,620p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Current stream path uses request-policy config, not raw attribution headers:** `createOpenAIAttributionHeadersWrapper()` resolves the attribution provider via request-policy capabilities, then merges headers through `resolveProviderRequestPolicyConfig(...)`, preserving endpoint gating and protected header precedence.
  - file: [src/agents/pi-embedded-runner/openai-stream-wrappers.ts:45](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/pi-embedded-runner/openai-stream-wrappers.ts#L45)

  - file: [src/agents/provider-request-config.test.ts:445](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/provider-request-config.test.ts#L445)
