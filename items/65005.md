## Summary

Keep open. Current main does not implement `openclaw models explain`, and the PR is a focused CLI feature with tests rather than obsolete or out-of-scope work. It is not merge-ready as-is because the review discussion identifies unresolved correctness issues around default-agent resolution and session-mode CLI overrides, but those are reasons to request changes, not to close the PR.

## Best Possible Solution

Keep PR #65005 open and request changes. The best path is to rebase on current main, add the command in `src/cli/models-cli.ts` and `src/commands/models.ts`, implement the explain logic by reusing `resolveDefaultAgentId`, `resolveGatewaySessionStoreTarget`, and `resolveSessionModelRef` consistently, add regression tests for default-agent no-`--agent` behavior and `--session` plus CLI override handling, and document `openclaw models explain` in the models CLI docs.

## Risks / Open Questions

- The PR should not be merged until it resolves the default-agent mismatch: gateway/session paths commonly resolve an omitted agent through `resolveDefaultAgentId`, while the proposed command can leave `agentId` undefined.
- Session explain mode needs a clear contract for positional model/`--provider` combined with `--session`: either reject those combinations or apply them before reporting override fields.
- Docs should be added for the new CLI surface if the command lands.
