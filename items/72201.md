## Summary

Keep #72201 open. Current main still lacks the PR's `SystemEvent.audience` primitive, drain-time internal runtime-context wrapping, and the migrated cron-awareness producer behavior. The PR is a recent focused implementation for open canonical issue #69492, and the provided review discussion shows its only concrete bot feedback was addressed in head commit c2ae546b8f16.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update/rebase it against current main, preserve the addressed shared-header fix from c2ae546b8f16, run the normal changed gates, and decide whether this narrow hidden-runtime-context lane is the accepted first implementation step for #69492 while leaving broader delivery-routing and remaining leakage work tracked separately.
