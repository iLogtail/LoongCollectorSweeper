## Summary

Keep open. Current `main` contains partial mitigation for plugin-registry drift in tool/provider runtime paths, but the Gateway WebSocket path still dispatches from a startup-time copied `extraHandlers` table, and runtime state has no gateway-handler pin/live-registry surface analogous to the HTTP-route and channel registry protections. That is not strong enough evidence to close this as fixed.

## Evidence

- **WS dispatch still uses a copied handler table:** Gateway startup attaches WS handlers with `extraHandlers: { ...pluginRegistry.gatewayHandlers, ...extraHandlers }`, and request dispatch resolves methods from `opts.extraHandlers?.[req.method]` before core handlers. That is still a snapshot, not a live registry lookup.
  - file: [src/gateway/server.impl.ts:804](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server.impl.ts#L804)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '768,807p'; nl -ba src/gateway/server-methods.ts | sed -n '130,185p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Runtime pinning exists for HTTP routes and channels, not gateway handlers:** Plugin runtime state tracks separate `httpRoute` and `channel` surfaces, with `pinActivePluginHttpRouteRegistry` / `pinActivePluginChannelRegistry` and corresponding getters. There is no parallel gateway-handler surface in this state.
  - file: [src/plugins/runtime.ts:19](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/runtime.ts#L19)

- **Main added a partial mitigation for secondary runtime loads:** `resolveRuntimePluginRegistry()` now reuses a compatible active registry instead of always loading a fresh one, and the dedicated regression test asserts that behavior. This reduces one suspected trigger from the report, but it does not change the WS snapshot dispatch path above.
  - file: [src/plugins/loader.ts:1015](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/loader.ts#L1015)
