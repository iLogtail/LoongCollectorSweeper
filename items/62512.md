## Summary

Keep #62512 open. Current main does not implement an AI-guided in-app walkthrough, and the request is not a clean ClawHub close because it needs first-class Control UI integration or a new documented UI extension surface before an external plugin could safely provide the proposed DOM snapshot, element catalog, overlay, navigation, and validated action loop.

## Best Possible Solution

Keep the issue open for product/design triage. A conservative path would define the Control UI contract first: stable guide IDs or an equivalent element registry, a typed validated action protocol, safe navigation/highlight/click semantics, tests for catalog drift, and a Gateway/session integration plan. If maintainers want this outside core, the first actionable core work is a documented UI extension API that lets a plugin provide the guide without reaching into private Control UI internals.
