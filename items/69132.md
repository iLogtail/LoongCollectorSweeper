## Summary

Keep #69132 open. Current main no longer calls `/api/experimental/web_search`, but it still makes one request to the configured/default Ollama host at `/api/web_search` and throws on 404. The issue and linked #69222 discussion both report local `/api/web_search` returning 404 on Ollama 0.16+/0.17.x, and the fuller local-404/cloud-fallback/auth-scoping fix is still represented by open PR #69222 rather than shipped on main.

## Best Possible Solution

Keep #69132 open until maintainers land #69222 or an equivalent focused fix. The likely implementation is to preserve the current `/api/web_search` path, add tested 404 compatibility behavior for local Ollama hosts, retry hosted `https://ollama.com/api/web_search` only when an env-sourced `OLLAMA_API_KEY` is available, keep provider-config and cloud bearer-token scoping explicit, and update the Ollama Web Search docs to match the chosen contract.

- PR #69222 likely needs a rebase against the partial #71741-era implementation so it preserves the shipped `/api/web_search` and `https://ollama.com` base URL behavior while adding the missing fallback and auth-boundary tests.
