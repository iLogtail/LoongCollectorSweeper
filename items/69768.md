## Summary

Keep #69768 open. The current main checkout does not contain the requested Gemini conditional-schema cleanup: `if`, `then`, and `else` are still absent from the Gemini unsupported-keyword set, and the current regression tests do not cover conditional schemas. The related PR #69964 explicitly targeted this issue but was closed unmerged, so it is not a shipped fix.

## Best Possible Solution

Keep #69768 open. The best maintainer path is to add `if`, `then`, and `else` to Gemini schema cleanup or an equivalent provider compatibility layer, add regression coverage for nested conditional schemas through `cleanSchemaForGemini` and the Gemini tool-normalization path, then run the changed agent/plugin test lanes. The cron failure-notification docs already exist in `docs/cli/cron.md`, but they do not replace the missing schema fix.
