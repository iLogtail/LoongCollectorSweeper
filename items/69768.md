## Summary

Keep #69768 open. Current main does not implement the concrete schema-cleaning fix identified in the issue comments and linked PR context: Gemini tool schemas are still cleaned by `cleanSchemaForGemini()`, but its unsupported-keyword set does not include JSON Schema `if`, `then`, or `else`, so conditional TypeBox-style schemas can still pass through to Gemini. There is partial fallback-related hardening for no-body HTTP 400/422 handling, but that does not solve the reported direct Gemini Flash tool-schema rejection.

## Best Possible Solution

Keep the issue open and land a focused agents fix: add `if`, `then`, and `else` to the Gemini unsupported schema keyword handling, add a regression test that nested conditional schemas are stripped while ordinary properties remain, and consider a small docs note for production cron users about failure destinations and provider-specific model/tool compatibility once the behavior decision is made.
