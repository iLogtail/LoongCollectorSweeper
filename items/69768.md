## Summary

Keep open. Current main still sends Gemini tool schemas through the shared Gemini cleaner, and that cleaner still does not strip JSON Schema conditional keywords `if`/`then`/`else`. The related PR #69964 proposed the exact root-cause fix but was closed unmerged, so the observable Gemini 400 schema-rejection path remains plausible on main. Failure-destination docs are now visible, but that only addresses incident visibility, not the Gemini schema payload rejection.

## Best Possible Solution

Keep the issue open and land the narrow schema-cleaning fix: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` in `src/agents/schema/clean-for-gemini.ts`, add the #69768 regression test in `src/agents/schema/clean-for-gemini.test.ts`, and run the focused agent schema tests. A short docs note about avoiding Gemini Flash for tool-heavy cron jobs can be considered after the code fix or if live validation shows provider behavior still rejects cleaned schemas.
