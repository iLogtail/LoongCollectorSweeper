## Summary

Keep open. Current main still does not strip JSON Schema conditional keywords `if`, `then`, and `else` from Gemini tool schemas, even though the issue discussion identified that as the likely root cause and PR #69964 attempted the same fix but was closed unmerged. Some related fallback and cron failure-notification behavior has improved on main, but the schema sanitizer gap remains.

## Best Possible Solution

Keep #69768 open and land a narrow agents fix: add `if`, `then`, and `else` to Gemini schema stripping or implement an equivalent recursive conditional-keyword sanitizer, then add regression coverage for top-level and nested conditionals through `cleanSchemaForGemini()` or `normalizeToolParameterSchema()`. PR #69964 is useful prior art but was closed unmerged; cron failureDestination docs/fallback changes can be treated as adjacent improvements, not the core fix.
