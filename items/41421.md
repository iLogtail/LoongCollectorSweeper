## Summary

Keep PR #41421 open. Current main does not provide `openclaw agents view-system-prompt` or an equivalent full system-prompt dump; the existing `/context` path only reports sizes and contributors. The PR is a plausible core CLI feature, but it still needs maintainer review because prior review comments identified fidelity gaps between static preview assembly and real runtime prompt assembly.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-owned implementation that factors prompt-preview assembly into a shared helper used by both runtime and CLI. The final command should prove parity for heartbeat defaults, model override versus configured default model, channel normalization/capabilities/reaction hints, workspace cwd/bootstrap notes, provider prompt contributions, prompt overrides, and documented limitations for live-only sandbox/tool filtering. Add focused tests and update `docs/cli/agents.md` plus user-facing docs if the command ships.
