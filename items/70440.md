## Summary

Keep #70440 open. Current main has related mitigations for duplicate incomplete-turn fallback after committed message-tool delivery and for group silent replies, but it still has no global/per-channel config option to silently swallow genuine empty or incomplete turn error payloads, and the maintainer discussion explicitly kept this feature request separate after the #70396/#70623 fix.

## Best Possible Solution

Keep #70440 open as a focused core feature request. The best path is a documented config-controlled empty/incomplete-turn delivery policy at the PI runner and auto-reply/channel delivery boundary, preserving the current default error behavior while allowing selected global, surface, or channel contexts such as Signal TTS to log WARN and deliver nothing. It should include config schema/docs plus tests for direct Signal/TTS behavior, existing group silent replies, committed messaging-tool suppression, replay/liveness metadata, and model fallback classification.
