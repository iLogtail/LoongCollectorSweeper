## Summary

Keep #70440 open. Current main has the #70396-style suppression for duplicate fallback text after committed message-tool delivery, plus group silent-reply handling, but it still does not expose a global/per-channel empty-turn delivery policy for direct/TTS channels. Genuine empty turns can still return the hard-coded user-visible "Agent couldn't generate a response" payload.

## Best Possible Solution

Add an explicit empty-turn delivery policy, probably resolved from global defaults plus surface/channel overrides, and consult it in the embedded runner before returning the incomplete-turn error payload. Keep the current visible-error default for text/direct channels, allow opt-in silent delivery for TTS/voice-note channels, log the suppressed failure at warn level, and cover direct Signal/TTS plus existing group/silentReply behavior in tests and docs.
