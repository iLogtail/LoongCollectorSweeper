## Summary

Keep #70440 open. Current main has partial silent-reply plumbing, but it does not implement the requested general empty-turn delivery policy. The current fallback path still returns a user-visible `Agent couldn't generate a response` error for genuine empty turns unless a narrow group-chat silent path is enabled, and the maintainer discussion explicitly kept this feature request open after the related false-positive fix.

## Best Possible Solution

Keep the issue open and implement a first-class empty-turn delivery policy, likely in the embedded runner or reply-run configuration path, with global and surface/channel overrides. The policy should default to current visible error behavior, allow `silent` for selected channels such as Signal with TTS, keep WARN/server logging, and add tests for direct and group/channel conversations plus genuine empty turns, reasoning-only turns, side-effect/tool-delivery cases, and existing `NO_REPLY` semantics.
