## Summary

Current main still has the exact unguarded assistant surface_error throw path described by PR #70900, and the PR’s proposed regression test is not present. This is a focused, plausible bug-fix PR rather than obsolete cleanup.

## Best Possible Solution

Keep PR #70900 open and review/land the minimal guard plus regression test, or apply an equivalent main-side fix that requires failoverFailure before throwing from assistant surface_error while preserving the #70848/#70124 real provider-failure path. Before merge, run the focused assistant-failover test and the repo’s changed gate when available.
