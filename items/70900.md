## Summary

Keep #70900 open. Current `main` still contains the unguarded `surface_error` throw path added by #70848, and the PR addresses a real remaining edge case: `failoverReason` can be classified from stale assistant `errorMessage` text even when `failoverFailure` is false because the assistant did not stop with `stopReason: "error"`. The provided PR patch adds the missing guard and a focused regression test; I did not find an implemented or canonical superseding fix on current `main`.

## Best Possible Solution

Keep the PR open and either merge #70900 or land an equivalent minimal fix in `src/agents/pi-embedded-runner/run/assistant-failover.ts`: require `params.failoverFailure` before throwing `FailoverError` from the assistant `surface_error` branch, while preserving #70848/#70124 behavior for real provider failures with `stopReason: "error"`. Add the proposed focused regression test in `src/agents/pi-embedded-runner/run/assistant-failover.test.ts` for `failoverFailure=false` plus classified `failoverReason`, and run the changed runner tests before landing.
