## Summary

Keep open. Current main has prompt-cache retention, cache-ttl pruning, heartbeat keep-warm guidance, and cache observability, but it does not implement a dedicated per-agent Cache TTL Warmer config or subsystem. The related heartbeat work (#70417/#70602) is adjacent, not a canonical replacement for this design proposal.

## Best Possible Solution

Keep #70418 open for maintainer design. The best path is to decide whether OpenClaw should add a core per-agent cache-warmer subsystem, or first expose a narrow plugin/provider API that can build the exact next-turn prefix, issue a minimal non-persisted request, and prove behavior with config schema, docs, unit tests, and live Anthropic cache-read validation. Coordinate with #70417/#70602 heartbeat alignment, but do not close this as implemented or superseded yet.
