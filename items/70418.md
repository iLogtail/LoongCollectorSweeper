## Summary

Keep #70418 open. Current main has prompt-cache support, cache-ttl pruning, and semantic heartbeat keep-warm guidance, but it does not implement the proposed opt-in Cache TTL Warmer: there is no `cacheWarmer` config/schema/docs surface, no background subsystem that sends a minimal non-persisted Anthropic request, and the existing heartbeat path still runs a full heartbeat agent turn with HEARTBEAT semantics, visibility/delivery handling, and session effects. The related discussion is useful: #70417/#70602 concerns heartbeat cache alignment, while the commenter’s CausaMem link is adjacent memory work, not an OpenClaw core/plugin implementation of this warmer.

## Best Possible Solution

Keep the issue open for maintainer design review. A good path is to either add a core `agents.defaults.cacheWarmer` / `agents.list[].cacheWarmer` subsystem near the heartbeat scheduler and embedded run path, with schema/docs/tests proving non-persistence and no delivery, or first define a narrow plugin/provider API that can safely perform the same active-prefix refresh without exposing mutable core internals. Do not treat #70417/#70602 as superseding this unless maintainers explicitly decide heartbeats are the only supported cache keep-warm mechanism.
