## Summary

Keep this issue open. Current main still does not provide the requested user-facing `plugins.entries.memory-core.config.dreaming.model` override: the memory-core plugin manifest rejects extra dreaming keys, docs still say Dream Diary uses the default runtime model, and the Dream Diary subagent path does not pass any model override. The discussion has concrete cost/model-separation use cases, and related PR #69209 appears to be the active implementation path but is not merged.

## Best Possible Solution

Keep this issue open until #69209 or an equivalent patch lands. The best implementation path is a backward-compatible, documented memory-core dreaming model override, most likely `plugins.entries.memory-core.config.dreaming.model`, with manifest/schema/docs updates, resolver tests, and runtime wiring into light, REM, and deep Dream Diary subagent runs while preserving fallback to the agent primary when unset and respecting plugin subagent model-override authorization.
