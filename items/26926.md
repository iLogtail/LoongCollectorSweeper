## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only: uploaded images, CSVs, and PDFs are not parsed, downloaded, saved, or forwarded into the agent media context. The existing issue discussion already noted this gap, and the current source still supports only outbound URL-based file delivery.

## Best Possible Solution

Keep #26926 open and implement this in the bundled Synology Chat plugin. Confirm Synology Chat's uploaded-file webhook/API shape, add typed inbound attachment parsing for images, CSV, and PDFs, download and store files through the existing channel media helpers with size/type/security limits, populate MediaPath/MediaPaths/MediaTypes in the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook/inbound-context tests for the supported attachment types.
