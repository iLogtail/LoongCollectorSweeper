## Summary

Keep #26926 open. Current main still only parses and forwards Synology Chat inbound webhook text; uploaded images, CSVs, and PDFs are not represented in the webhook payload type, parser, delivery message, inbound context, docs, or tests. The existing issue discussion already includes a maintainer/Codex review keeping this open for the same remaining gap.

## Best Possible Solution

Keep #26926 open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's uploaded-file webhook/API shape, add typed attachment parsing for images, CSVs, and PDFs, download/store accepted files through existing inbound media helpers with size/type/security limits, pass media fields into the inbound context, update docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
