## Summary

Keep #7661 open. Current main has the generic `channel-create` action name and Discord-side channel creation, but the Slack-centered request and CLI surface are still missing: Slack does not advertise or dispatch `channel-create`, the Slack setup scopes omit `channels:manage`, and `openclaw message channel` still exposes only `info` and `list`.

## Best Possible Solution

Keep #7661 open. Implement Slack-owned `channel-create` in `extensions/slack` with `channels:manage` setup/docs, a scoped action gate, a runtime wrapper for Slack `conversations.create`, support for the proposed name/topic/private fields where product wants them, discovery and dispatch tests, and an `openclaw message channel create` CLI mapping through the existing message-action runner. Preserve the existing Discord implementation and add docs for the provider-specific flags.
