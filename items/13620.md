## Summary

Keep #13620 open. Current main still lacks outbound Telegram album/media-group sending: Telegram reply delivery, direct sends, shared outbound payload helpers, tests, and docs all treat multiple media URLs as sequential one-by-one sends. The related implementation attempts #21309, #51449, and #51457 were closed unmerged in the provided GitHub context, so they do not supersede this issue.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is a `sendMediaGroup` flow for 2-10 groupable photos/videos across reply delivery, direct Telegram send/action, and outbound payload delivery, with focused Telegram tests and docs for any new `mediaUrls`/`mediaGroup`/album user-facing parameter. Existing per-item behavior should remain the fallback for unsupported media and edge cases.
