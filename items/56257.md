## Summary

Keep open. Current main does not implement this PR's requested declarative plugin capability model: the plugin manifest has no top-level capability declaration, docs do not document that schema, and security audit has no `plugins.capabilities.legacy_unrestricted` finding. The PR is plausible core plugin/security work rather than ClawHub work, but it still needs maintainer review and fixes for the outstanding review concerns before it is mergeable.

## Best Possible Solution

Keep the PR open for maintainer security/API review. The best path is a focused core implementation that adds the manifest capability types and normalization tests, documents the public manifest schema, scopes the audit to enabled plugins, resolves manifests from the installed extensions root, preserves the intended empty-declaration semantics, and links the enforcement follow-up to #12517 rather than treating Phase 1 as a full isolation fix.
