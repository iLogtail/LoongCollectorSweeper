## Summary

Keep #38945 open. Current main still lacks the PR’s broader Unicode, combining-mark, and NFC tokenizer fix after the memory implementation moved from src/memory to extensions/memory-core. Later CJK work improves Chinese/Japanese/Korean search and chunking, but it does not cover accented Latin, Cyrillic, Arabic/Hebrew, Thai/Hindi combining marks, or NFC/NFD normalization for MMR/FTS tokenization.

## Best Possible Solution

Keep the PR open and port the narrow tokenizer fix to current memory-core: update extensions/memory-core/src/memory/mmr.ts, extensions/memory-core/src/memory/hybrid.ts, and the shared token path in extensions/memory-core/src/memory/manager-search.ts as needed. Add focused tests for accented Latin/Finnish, Cyrillic, Arabic/Hebrew, Thai/Hindi combining marks, and NFC/NFD normalization while preserving existing CJK MMR and FTS behavior.
