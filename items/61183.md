## Summary

Keep #61183 open. Current main still commits embedded message-tool bookkeeping from stale tool-start args and pending maps, while adjusted before_tool_call params are only consumed later for after_tool_call payloads. The PR remains useful, although its unrelated session-transcript repair changes should be split or reconciled before merge.

## Best Possible Solution

Keep the PR open and reconcile it into a narrow agents fix: in handleToolExecutionEnd, derive effective post-hook args before message/cron bookkeeping, commit sent text/target/media from those effective args, use the same args for successfulCronAdds and after_tool_call, and add focused regressions for adjusted message sends plus hook-rewritten cron add/list cases. Split or drop the session-transcript-repair changes unless a maintainer wants that separate behavior in this PR.
