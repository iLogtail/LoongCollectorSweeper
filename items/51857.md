## Summary

Keep #51857 open. Current main has shipped/partial fixes for several linked symptoms, including Gemini usage mapping and MS Teams Graph attachment diagnostics, but the umbrella issue is not fully obsolete: one concrete linked bug (#51593, Moonshot/Kimi duplicate tool-call IDs in WhatsApp groups) is still open in the provided GitHub context, and current main intentionally avoids Moonshot tool-call ID sanitization to preserve native Kimi IDs. The broader content-verification/vision-routing question also is not fully reduced to a single implemented behavior.

## Best Possible Solution

Keep the umbrella open or convert it into a maintainer-owned tracking checklist: mark Gemini usage and MS Teams Graph diagnostics as addressed, keep #51593 as the canonical Moonshot/WhatsApp duplicate-ID bug, and decide whether core should add a shared post-fetch/content-verification contract plus clearer imageModel fallback observability across Gateway and channel plugins.
