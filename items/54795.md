## Summary

Keep open. Current `main` does not implement a macOS Keychain / `kc` SecretRef backend, and this is not a clear `clawhub` close because SecretRef provider sources are hard-coded in core and the public plugin SDK does not expose a secret-provider registration seam.

## Evidence

- **SecretRef provider types are fixed in config types:** `SecretProviderConfig` only allows `env`, `file`, and `exec`; there is no `kc`/keychain source today.
  - file: [src/config/types.secrets.ts:222](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/config/types.secrets.ts#L222)
  - command: `nl -ba src/config/types.secrets.ts | sed -n '220,270p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Runtime resolution only handles three provider sources:** `resolveProviderRefs` dispatches only `env`, `file`, and `exec`, then throws for any other source.
  - file: [src/secrets/resolve.ts:791](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/secrets/resolve.ts#L791)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Docs describe the same three-source SecretRef contract:** The user-facing secrets docs define SecretRefs as `{ source: "env" | "file" | "exec", ... }` and document only those provider configs.
  - file: [docs/gateway/secrets.md:81](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/gateway/secrets.md#L81)

- **Plugin SDK has no secret-provider registration hook:** The public plugin registration API lists providers/channels/tools/hooks/etc., but no method to add a SecretRef backend, so this is not plainly an external-plugin-only issue on current `main`.
  - file: [docs/plugins/sdk-overview.md:60](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/plugins/sdk-overview.md#L60)

## Risks / Open Questions

- There may be a workable user-level workaround today via existing `env`/`exec` SecretRef providers plus an external adapter, but that is not the feature requested here.
- Closing as `clawhub` would be premature because current public plugin contracts do not let third-party plugins register a new SecretRef source directly.
