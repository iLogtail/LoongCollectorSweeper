## Summary

Current main does not make Slack `requireMention: true` suppress implicit thread mentions by itself. The repo added a separate opt-in setting, `channels.slack.thread.requireExplicitMention` (default `false`), so the issue’s reported behavior still matches current main unless that extra flag is enabled.

## Evidence

- **Mention gating still keys off separate thread flag:** `prepareSlackMessage` passes `requireMention: shouldRequireMention`, but only disables implicit thread mentions when `ctx.threadRequireExplicitMention` is true via `allowedImplicitMentionKinds: ctx.threadRequireExplicitMention ? [] : undefined`. That means `requireMention` alone does not suppress implicit thread mentions.

## Risks / Open Questions

- Closing as implemented would be inaccurate: current main still reproduces the reported behavior for configs that only set `requireMention: true`.
- The multi-agent Slack collision described in the issue remains plausible unless operators know to add the newer `channels.slack.thread.requireExplicitMention: true` setting.
