## Summary

Current `main` does not implement the requested fix, and the PR as proposed appears to regress an existing supported contract: explicit `OPENCLAW_LAUNCHD_LABEL` overrides must win over profile-derived labels. The underlying bug in #65643 may still be real, but this PR should stay open for maintainer follow-up rather than being closed as already-fixed or obsolete.

## Evidence

- **Current main still leaves launchd label untouched in profile env setup:** `applyCliProfileEnv()` on `main` sets `OPENCLAW_PROFILE`, state dir, config path, and dev port, but does not clear or reconcile `OPENCLAW_LAUNCHD_LABEL`, so the PR's requested behavior is not already implemented on `main`.

- **Core launchd resolution explicitly prefers label override over profile:** `resolveLaunchAgentLabel()` returns `OPENCLAW_LAUNCHD_LABEL` first and only falls back to `resolveGatewayLaunchAgentLabel(OPENCLAW_PROFILE)` when no override is present. That is the current behavior on `main`.
