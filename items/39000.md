## Summary

Keep #39000 open. Current main still exposes plugin gateway method registration and request-path dispatch, but there is no plugin-facing `api.callGatewayMethod()` or equivalent generic inter-plugin request/response seam. The linked metabolism/stability report is concrete and points to an SDK/runtime API gap that needs maintainer design, not cleanup or ClawHub migration alone.

## Best Possible Solution

Keep #39000 open for maintainer product/API design. The likely fix belongs at the plugin SDK/runtime boundary: add a deliberate inter-plugin request/response seam, either `api.callGatewayMethod()` or a more explicit typed service/RPC abstraction, with tests for plugin isolation, caller identity and scope propagation, missing methods, handler failures, recursion/startup timing, and documentation for plugin authors.
