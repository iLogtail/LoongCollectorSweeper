## Summary

Keep #71862 open. Current main at 21c51bc140295e4286f1adc871a96b9b33c6a8b9 still exposes `before_model_resolve` and legacy `before_agent_start` model-selection output as provider/model-only, and the embedded runner still chooses auth profiles from request/session state rather than hook output. The related #68621 is an older overlapping predecessor, not a shipped or clearly canonical superseding item.

## Best Possible Solution

Keep #71862 open as the active candidate implementation. The best path is to add `authProfileOverride` to the typed `before_model_resolve` and legacy `before_agent_start` result contract, merge it with the same first-defined priority rules as provider/model overrides, consume it in runner auth-profile selection only when no explicit user pin is present, preserve legacy source/compaction semantics, and update SDK/docs/tests together.
