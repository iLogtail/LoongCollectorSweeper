## Summary

Keep PR #71862 open. Current main at 637bd33e69b2117331dafe92dfaed831fb684654 still exposes `before_model_resolve` as a provider/model-only hook and the embedded runner still selects auth profiles from request/session state rather than hook output. The PR remains a focused plugin API extension with tests/docs in the PR context, not something already implemented or superseded on main.

## Best Possible Solution

Keep #71862 open as the active candidate implementation. The best path is to add `authProfileOverride` to the typed `before_model_resolve` and legacy `before_agent_start` result contract, merge it with the same priority rules as provider/model overrides, consume it in runner auth-profile selection only when no explicit user pin is present, keep SDK/docs/tests aligned, and then close older PR #68621 as superseded if #71862 lands.
