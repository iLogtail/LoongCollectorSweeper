## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as plain instance maps with no TTL/sweeper, no close-time clear, and no cap. The linked fix PR #71648 remains the appropriate active path rather than evidence that main is fixed.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands on main. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the MCP protocol surface, add focused regression tests, and let maintainers decide whether hard caps/FIFO burst protection land in the same change or a tracked follow-up.
