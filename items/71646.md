## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as unbounded instance maps, does not clear them in `close()`, and has no TTL/sweeper or cap. The linked fix PR #71648 is useful context but is not merged into current main, so the issue should remain open.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands on main. The fix should clear both maps on bridge close, expire stale Claude permission and approval entries, preserve the MCP protocol surface, and add focused regression tests; a hard cap/FIFO burst guard can be included or tracked as a follow-up at maintainer discretion.
