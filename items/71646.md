## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps the two reported pending maps without TTL, close cleanup, or cap, and the linked fix PR #71648 is still the active path rather than evidence that main is fixed.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the MCP protocol surface, add focused regression tests, and let maintainers decide whether cap/FIFO burst protection is part of the same fix or a tracked follow-up.
