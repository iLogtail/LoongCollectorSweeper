## Summary

Current main still matches #71646: `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as unbounded instance maps, does not clear them on bridge close, and has no TTL/sweeper or cap for either collection. The linked fix PR #71648 is still the active proposed path in the provided GitHub context, not evidence that main is fixed.

## Best Possible Solution

Keep #71646 open and use #71648, or an equivalent maintainer-approved patch, as the implementation path. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries using the existing approval expiry metadata where available, avoid post-close writes, preserve the MCP notification/tool protocol, and add focused regression tests. A hard cap/FIFO guard can be included if maintainers want burst resistance in the same fix; otherwise it should be tracked as a narrow follow-up.
