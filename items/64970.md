## Summary

Keep open. Current main still hardcodes the sessions_history per-text cap at 4000 characters and does not read SESSIONS_HISTORY_TEXT_MAX_CHARS. The linked fix PR #65029 was closed unmerged, and I found no changelog/docs/source evidence that the env override shipped elsewhere.

## Best Possible Solution

Implement the narrowly scoped fix from #65029 on current main: read `SESSIONS_HISTORY_TEXT_MAX_CHARS` through the existing `readEnvInt()` helper, preserve the default 4000-character cap when unset/invalid, and add focused sessions_history regression coverage proving the env override affects text truncation while the 80 KiB hard cap remains enforced.
