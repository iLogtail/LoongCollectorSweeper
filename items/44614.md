## Summary

Keep #44614 open. Current main still performs global npm updates synchronously from the running gateway process and only schedules a restart after the install reports success, so the Windows locked-file problem described by the PR is not implemented on main. The PR is a plausible core gateway fix, but the submitted diff also has unresolved review findings that need author/maintainer changes before merge.

## Best Possible Solution

Keep the PR open or supersede it with a focused core gateway update fix. The best implementation should delegate Windows npm global updates before replacing locked files, preserve the prepared install environment and npm fallback retry, quote cmd arguments safely, default and clamp restart timing, require Scheduled Task supervision before forced exit, reconcile the detached result on startup, and cover the Windows branch with targeted tests.
