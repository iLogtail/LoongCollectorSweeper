## Summary

Keep open. Current `main` does not clean sandbox workspace directories during prune or recreate, so the request is not implemented on `main`, and the PR context still shows unresolved correctness issues in the proposed approach.

## Evidence

- **Current prune path only removes runtimes and registry entries:** `src/agents/sandbox/prune.ts:38-61` prunes by calling `removeRuntime`, then `remove(entry.containerName)` and optional `onRemoved`. There is no workspace-directory cleanup in the current `main` implementation. Both container and browser prune paths call this helper at `src/agents/sandbox/prune.ts:65-122`.
  - file: [src/agents/sandbox/prune.ts:38](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/sandbox/prune.ts#L38)

This confirms the cleanup target is a real sandbox workspace path that current prune code never removes.
  - file: [src/agents/sandbox/context.ts:38](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/sandbox/context.ts#L38)
