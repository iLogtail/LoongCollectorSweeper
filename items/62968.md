## Summary

Current main still matches the reported failure mode: completion-mode dispatch records a failed direct-primary phase, returns a successful queue-fallback result when the fallback delivers, and the caller only logs when the top-level delivery result failed with path direct. The remoteclaw cross-references in the timeline are relevant upstream-report context, but they do not supersede this OpenClaw bug.

## Best Possible Solution

Keep this issue open and fix the recovered completion-delivery path. The narrowest product behavior is to emit a warn-level diagnostic when a direct-primary phase has an error and a later queue-fallback phase delivers, without marking the announce as failed. The implementation likely belongs at the runSubagentAnnounceFlow caller boundary or via a dispatch hook/logger, with a focused test covering delivered=true/path queued or steered plus a direct-primary error in phases.
