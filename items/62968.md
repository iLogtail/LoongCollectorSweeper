## Summary

Keep open. Current `main` still preserves the direct-send failure only inside `phases` and still skips logging when the completion-mode queue fallback succeeds, so the reported bug remains present on `cf858258c71701ef62dddf88aa777873a4962046`.

## Evidence

- **Direct-send catch still returns an error result without logging:** `sendSubagentAnnounceDirectly` catches the gateway failure and returns `{ delivered: false, path: "direct", error: summarizeDeliveryError(err) }`; there is no `defaultRuntime.error` or warn call in that catch path.
  - file: [src/agents/subagent-announce-delivery.ts:615](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/subagent-announce-delivery.ts#L615)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '560,640p'`
  - sha: [a43c1f880757](https://github.com/openclaw/openclaw/commit/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4)
- **Dispatch still converts a recovered fallback into a delivered queued/steered top-level result:** In completion mode, `runSubagentAnnounceDispatch` appends the failed `direct-primary` phase, then returns `withPhases(fallbackQueue)` when the queue fallback delivers. That changes the top-level result away from the direct failure.
  - file: [src/agents/subagent-announce-dispatch.ts:88](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/subagent-announce-dispatch.ts#L88)
  - command: `nl -ba src/agents/subagent-announce-dispatch.ts | sed -n '1,240p'`

- **Caller still logs only unrecovered direct failures:** `subagent-announce.ts` only emits `Subagent completion direct announce failed...` when `!delivery.delivered && delivery.path === "direct" && delivery.error`. A recovered queue fallback has `delivery.delivered === true`, so the logged branch is skipped even if `phases` contains the direct failure.
  - file: [src/agents/subagent-announce.ts:566](https://github.com/openclaw/openclaw/blob/a43c1f880757e830b5fb4d3c7b5532a625f0e3e4/src/agents/subagent-announce.ts#L566)
