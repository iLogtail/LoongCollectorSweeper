## Summary

Keep open. Current main has a bundled Volcengine provider, but I did not find strong evidence that the reported Volcengine Lite / UUID-style API key 401 is fixed or obsolete. The provider still passes the stored key unchanged into the generic OpenAI-compatible completions transport, docs still advertise Volcengine DeepSeek setup, and tests do not cover a real Volcengine completion request with this key shape.

## Best Possible Solution

Keep the issue open until maintainers can reproduce against a Volcengine Lite account using the UUID-style key. The likely product path is to add a focused Volcengine completion/auth regression test or live-smoke lane, then either document the required key/endpoint pairing or add a Volcengine-specific auth/request adaptation in `extensions/volcengine` or the provider runtime if the vendor needs something other than the current OpenAI-compatible Bearer-key path.
