## Summary

Keep #9455 open. Current main has the Cron Jobs filter bar and partial filtering, but it still does not satisfy job-id search or an explicit unknown last-run-status filter. The existing owner-side Codex comment already identified these same gaps, and the relevant files have not changed since that review.

## Best Possible Solution

Keep #9455 open but narrow it to the remaining delta on top of merged #9510: include job.id in the cron.list search haystack and UI/search tests, add unknown to the Cron Jobs last-status filter type and select options, map missing lastStatus/lastRunStatus to that filter, and add focused controller/service/view coverage for both behaviors.
