## Summary

Keep #9455 open. Current main has a Cron Jobs filter bar and partial filtering from merged PR #9510, but two requested behaviors still remain missing: searching by cron job id/jobId and an explicit unknown last-run-status filter for jobs without status.

## Best Possible Solution

Keep #9455 open and narrow the remaining work: include `job.id` in the `cron.list` query haystack, add `unknown` to the Cron Jobs last-run-status filter type and UI option, map missing `lastStatus`/`lastRunStatus` to that option, and add focused service/controller/view tests for job-id search and unknown-status filtering. The already-merged #9510 filter bar should remain as the baseline implementation.
