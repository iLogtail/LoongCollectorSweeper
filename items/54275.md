## Summary

Current main still has the stale `pnpm docs:build` references the PR is trying to fix, while `package.json` exposes `check:docs` instead. The PR is therefore not obsolete, but it needs rework before merge because its localized `docs/zh-CN/**` edit conflicts with current docs i18n policy and that file is no longer present on main.

## Best Possible Solution

Keep the PR open but request a narrow rework: update only the current English source docs (`docs.acp.md` and `docs/install/bun.md`) from `docs:build` to `check:docs`, drop the `docs/zh-CN/install/bun.md` change, and let the separate docs i18n publish pipeline handle localized output.
