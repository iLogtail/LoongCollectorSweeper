## Summary

Keep PR #38789 open. Current main still lacks the PR’s sandbox-specific skills prompt refresh: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots skip live entry loading and resolveSkillsPromptForRun still returns the existing snapshot prompt before considering entries. The provided review history also shows unresolved correctness concerns in the PR patch itself, so the right outcome is a focused follow-up fix, not closure.

## Best Possible Solution

Keep this PR open or supersede it with a focused core agents fix that rebuilds or remaps the skills prompt for non-rw sandbox workspaces while preserving the saved session skillFilter, remote eligibility context, and env/runtime contracts. Add regression coverage for snapshot-present sandbox runs, empty sandbox skill sets, filtered skill snapshots, and attempt/compaction parity.
