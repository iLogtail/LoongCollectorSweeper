## Summary

Keep open. Current main has meaningful cron safeguards for the simultaneous catch-up/reaper parts of the report, but it still does not implement the requested daemon heap-limit default/config and still persists full isolated cron session state in both base and per-run entries, so the central OOM pressure report is not resolved enough to close.

## Evidence

- **Current daemon launch args lack heap sizing:** The generated gateway service arguments for the Node runtime are `[nodePath, cliEntrypointPath, ...params.args]`; there is no `--max-old-space-size` or `gateway.nodeMaxOldSpaceMb` equivalent in the service launch path.
