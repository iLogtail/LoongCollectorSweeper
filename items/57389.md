## Summary

Keep open. Current `main` still does not accept `"api_key"` as an alias for auth-profile API key credentials, and the loader path still allows those entries through without an upfront validation error, which matches the reported silent failure.

## Evidence

- **Current loader only normalizes `apiKey`, not `api_key`:** `normalizeRawCredentialEntry` maps `apiKey` to `key`, but there is no equivalent mapping for `api_key`. `parseCredentialEntry` then accepts the object as long as `type` and `provider` are present.
  - file: [src/agents/auth-profiles/persisted.ts:50](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/auth-profiles/persisted.ts#L50)
  - command: `nl -ba src/agents/auth-profiles/persisted.ts | sed -n '50,90p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Store load does not reject missing-key api_key profiles:** `coercePersistedAuthProfileStore` only rejects entries for non-object, invalid type, or missing provider. A profile with `type: "api_key"`, a provider, and an unrecognized `api_key` field is kept in the store rather than flagged during load.
  - file: [src/agents/auth-profiles/persisted.ts:128](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/auth-profiles/persisted.ts#L128)
  - command: `nl -ba src/agents/auth-profiles/persisted.ts | sed -n '128,152p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Runtime still treats missing `key`/`keyRef` as ineligible:** `evaluateStoredCredentialEligibility` checks only `credential.key` and `credential.keyRef` for `api_key` profiles and returns `missing_credential` when both are absent, which is the silent-ignore path described in the issue.
  - file: [src/agents/auth-profiles/credential-state.ts:79](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/auth-profiles/credential-state.ts#L79)
  - command: `nl -ba src/agents/auth-profiles/credential-state.ts | sed -n '72,86p'`

- **Current docs still document `key`, not `api_key`:** Docs describe `type: "api_key"` credentials as `{ provider, key }` and mention `keyRef`, with no documented `api_key` alias for `auth-profiles.json`.
  - file: [docs/concepts/model-failover.md:68](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/concepts/model-failover.md#L68)
