## Summary

Current main still has the hybrid memory-search gap described by this PR. Provider-backed hybrid search performs one full-query FTS keyword lookup, while the FTS query builder AND-joins all tokens; the per-keyword fallback exists only in the no-provider FTS-only branch. The PR is stale against the moved memory-core path and still needs a bounded fan-out design, but the underlying bug is not implemented away or clearly superseded.

## Best Possible Solution

Keep #39555 open as an actionable bug-fix candidate. The best path is to port the fix to extensions/memory-core, add an explicit cap or ranking step for extracted query terms, and add focused regression coverage for multi-word hybrid queries whose terms span chunks. Maintainers may later choose between this per-keyword approach and related #57711, but current main does not justify closing this PR as implemented or superseded.
