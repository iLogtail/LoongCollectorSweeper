## Summary

Keep #39555 open. Current main still has the reported provider-backed hybrid memory-search bug: the hybrid path sends the full cleaned multi-word query to a single FTS lookup, while the FTS builder AND-joins tokens, so terms that do not co-occur in one chunk can still return zero keyword hits. The PR itself is stale against the current `extensions/memory-core` layout, but the underlying fix remains actionable.

## Best Possible Solution

Keep #39555 open as an actionable memory-core bug-fix candidate. Port the idea to `extensions/memory-core`, add bounded/ranked per-keyword FTS recall for provider-backed hybrid search, and add an active regression test where relevant query terms span chunks. If maintainers choose #57711 as the replacement, mark that PR explicitly as canonical and ensure it fixes OR-style cross-chunk recall, not only stop-word query expansion.
