## Summary

Keep open. Current main still appears to have the reported builtin memory gap: provider-backed hybrid search performs one full-query FTS lookup, while the FTS query builder still AND-joins every token. The broader per-keyword fallback exists only in the no-provider FTS-only path, and the PR’s regression test is not present on main. The PR remains useful, but it needs to be ported from the old src/memory layout to extensions/memory-core and should address the unresolved fan-out review concern.

## Best Possible Solution

Keep this open or replace it with a fresh PR that ports the behavior into extensions/memory-core/src/memory/manager.ts, reuses the existing extractKeywords strategy with a bounded term limit, merges/deduplicates keyword hits by chunk id, and adds active regression coverage under extensions/memory-core/src/memory for multi-word provider-backed hybrid searches whose terms land in separate chunks.
