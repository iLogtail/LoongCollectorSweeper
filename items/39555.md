## Summary

Keep #39555 open. Current main still has the provider-backed builtin hybrid memory-search behavior this PR targets: it calls one keyword search with the full cleaned query, and the FTS query builder AND-joins all tokens. The PR patch is stale against the current `extensions/memory-core` layout, and its proposed fan-out needs bounding, but the underlying bug remains actionable. Related #57711 overlaps hybrid query expansion, but it is still open and not a canonical replacement for this cross-chunk OR-recall case.

## Best Possible Solution

Keep the PR open as an actionable memory-core bug-fix candidate unless maintainers explicitly replace it with a canonical PR. Port the OR-style per-keyword keyword recall to `extensions/memory-core/src/memory/manager.ts`, preserve source/model filtering, bound or rank extracted terms, and add an active regression test for provider-backed hybrid search where relevant multi-word query terms appear in separate chunks.
