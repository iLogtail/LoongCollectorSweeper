## Summary

Keep #43549 open. Current main still does not explicitly diagnose oversized, malformed, or unreadable Telegram-related session/state JSON in status output, and there is no first-class Telegram session reset/recovery command. The discussion’s Keep My Claw link is useful as a backup safety net, but the commenter explicitly noted it is not a fix for the requested detection/recovery UX.

## Best Possible Solution

Keep the issue open and implement a conservative diagnostic/recovery path in the Telegram/status surface: report the resolved session/state file path, readability, parse/schema status, size threshold warnings, and actionable backup/reset guidance. If mutation is added, prefer an explicit non-destructive backup-and-reset command scoped to the affected Telegram/account/session artifact before considering automatic quarantine.
