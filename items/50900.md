## Summary

Keep #50900 open. Current main has partial targeted cleanup for cron run sessions and subagent run cleanup, but it still does not implement the requested `session.maintenance.rules` / per-pattern retention policy, and the provided KeepMyClaw comment is a backup workaround rather than an OpenClaw core fix.

## Best Possible Solution

Keep the issue open for maintainer design. The likely core path is to add a first-class `session.maintenance.rules` contract in config types, zod schema, generated config help/docs, and the session-store maintenance resolver, then apply first-match retention by session key while preserving the global `pruneAfter` fallback and existing active-session/transcript/disk-budget safeguards. Existing cron and subagent cleanup should be treated as related special cases or migration guidance, not as proof that the requested generic rule system exists.
