## Summary

Keep open. Current `main` still lacks the replay guard this PR adds, so `subscribeEmbeddedPiSession(...)` can still treat replayed historical assistant lifecycle events as fresh outward delivery when attaching to an existing session. The PR remains materially useful even though the review discussion identified one minor redundant guard.
