## Summary

Keep open. Current main still lacks a typed Gemini safety-settings config surface and does not map safety settings into the direct Google Generative AI request payload. The supplied discussion also includes an explicit maintainer Codex review keeping #12008 open, and the related implementation PR #49529 is described there as open/unmerged rather than shipped.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The likely maintainer-ready path is a schema-validated provider/model config field such as models.providers.*.safetySettings, docs for the public provider config, runtime mapping into direct google-generative-ai Gemini payloads without overriding explicit request-level safety settings, and regression tests for both bundled Google and custom provider keys using the Google adapter.
