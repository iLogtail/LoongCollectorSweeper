## Summary

Current `main` does not implement the PR’s Feishu 502 fallback. The inbound Feishu media path still maps non-image resources to `type: "file"`, and `downloadMessageResourceFeishu` still performs a single `messageResource.get` call with no `502 -> media` retry. The report remains plausible on current `main`, and the PR discussion shows the author already addressed the review note about preserving the original error on retry failure.
