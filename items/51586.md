## Summary

Keep #51586 open. Current main still builds the bare `/new` and `/reset` prompt from hardcoded constants, and the config contract only exposes `agents.defaults.startupContext` for runtime startup data, not a string prompt override or per-agent reset-prompt override. Existing plugin hooks can prepend context, replace system prompt, synthesize/silence replies, or observe resets, but that is not equivalent to a documented `openclaw.json` reset-prompt config surface. The related #54246 describes the same core product gap but is closed, so it should not supersede this still-open tracker.

## Best Possible Solution

Keep the issue open as a core config/API gap. A focused fix would add a documented `agents.defaults.sessionResetPrompt` plus per-agent override, validate it in the config schema/types/help, resolve it in both reset call paths before `appendCronStyleCurrentTimeLine`, preserve existing bootstrap/startupContext behavior when unset, and add regression tests plus docs for default and per-agent behavior.
