## Summary

Keep #51586 open. Current main still does not provide the requested configurable replacement for the bare `/new` or `/reset` prompt. Main has related improvements (`startupContext`, bootstrap-aware wording, hooks), but those prepend context or observe/reset-adjacent events rather than replacing the hardcoded reset prompt per default or per agent.
