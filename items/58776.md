## Summary

Keep open. Current main has a partial timeout-grace path for aborted lifecycle end events, but the requested additive lifecycle phases and typed startup-failure distinction are still missing.

## Evidence

- **Lifecycle phase contract is still only start/end/error:** Gateway session lifecycle code accepts only `start`, `end`, and `error`; unknown phases are ignored, and there is no `first-progress` or `startup-failed` phase in this path.
  - file: [src/gateway/session-lifecycle-state.ts:6](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/src/gateway/session-lifecycle-state.ts#L6)

  - file: [src/gateway/server-methods/agent-job.ts:127](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/src/gateway/server-methods/agent-job.ts#L127)
