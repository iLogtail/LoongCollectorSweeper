## Summary

Keep #71116 open. Current main has strong SecretRef runtime documentation and SDK helpers, but it does not yet implement the issue's requested hotspot guidance for `src/gateway/server-methods/secrets.ts` or a central plugin credential-persistence contract. The linked PR #71156 is a useful follow-up, but it is still open in the provided GitHub context and should not be treated as shipped.

## Best Possible Solution

Keep #71116 open until maintainers either merge #71156 or another focused documentation PR that adds secrets-surface AGENTS guidance and a central plugin credential-persistence contract. The likely best path is to review #71156, then add or cross-link a concise public docs section for plugin authors covering file locks, atomic writes, lifecycle/TTL semantics, and failure behavior for credential persistence.
