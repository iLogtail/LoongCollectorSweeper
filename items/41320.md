## Summary

Keep open. Current main does not implement tirith, `tools.exec.commandSecurity`, or an equivalent enforced content-level scan before exec runs. The PR is focused security/core hardening with useful review history; it may ultimately belong in core, a generic plugin policy seam, or a ClawHub native plugin, but that needs maintainer product/security judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep #41320 open for maintainer security/product review. Decide whether enforced command-content scanning should land in core exec, be exposed as a generic core/plugin policy hook, or be implemented as a ClawHub native plugin; if core, it should update config schema/docs/tests and validate gateway, sandbox, node, approval, and elevated paths.
