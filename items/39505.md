## Summary

Current main still has the reported formatter gap: punctuation-wrapped Home Assistant entity IDs can fall through copy-sensitive classification and then be chunked with literal inserted spaces. The direct fix remains tracked by open PR #39565, while PR #39553 was closed unmerged.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped core token for file-like/dotted IDs, preserve backtick- and quote-wrapped Home Assistant entity IDs exactly, and add regression tests for both bare and markdown-wrapped long entity IDs without weakening the existing narrow-terminal wrapping guard.

## Evidence

- **Current formatter still tests raw punctuation-wrapped token for file-like IDs:** `isCopySensitiveToken` computes a punctuation-stripped `candidate`, but the file-like branch still checks `token.includes("_") && FILE_LIKE_RE.test(token)`. A token like `` `binary_sensor.sense_energy_monitor_power` `` has underscores but fails the raw `FILE_LIKE_RE` because of the backticks, so it is not preserved by this branch.
