## Summary

Current main still has the reported formatter bug for punctuation/backtick-wrapped Home Assistant-style entity IDs. The sanitizer strips edge punctuation into a candidate, but the file-like underscore preservation branch still checks the raw token, so wrapped dotted IDs are treated as generic long tokens and split with literal spaces. The assistant display path reaches this sanitizer, existing tests do not cover wrapped entity IDs, and the focused fix remains tracked by open PR #39565 while earlier PR #39553 was closed unmerged.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped candidate for copy-sensitive file-like checks, preserve bare and backtick/quote/markdown-wrapped Home Assistant-style dotted entity IDs exactly, add regression tests for those shapes, and retain chunking for generic long prose or punctuation-only tokens.
