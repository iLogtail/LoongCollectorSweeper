## Summary

Current main still has the reported copy-safety gap. The sanitizer computes a punctuation-stripped candidate but still uses the raw token for the underscore/file-like check, so a backtick-wrapped Home Assistant entity ID can fall through to long-token chunking and receive inserted spaces. The related PR #39565 remains the useful fix path, but this issue should stay open until that or an equivalent fix lands.

## Best Possible Solution

Keep #39505 open and land PR #39565 or an equivalent fix that classifies punctuation-stripped dotted entity IDs as copy-sensitive before long-token chunking. Add focused regression tests for bare, backtick-wrapped, and punctuation-wrapped Home Assistant entity IDs, and consider whether the similar terminal note wrapping helper needs the same copy-sensitive policy.
