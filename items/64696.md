## Summary

Keep #64696 open. The issue is a concrete memory-wiki bug report, and current main still lacks the source changes proposed by the linked PR #65197: relative Markdown link resolution is not implemented in lint, stale-pages native links are still generated relative to the vault root, bridge rendering still fences raw markdown without reply-tag sanitization or renderer-version invalidation, and cached imported source pages are still skipped based on file existence only. Related #70986 is open and broader, but it does not cover the full bridge import/cache invalidation scope in #64696.

## Best Possible Solution

Keep #64696 open until memory-wiki source receives the bounded bridge/lint fixes or a new canonical maintainer issue/PR covers the entire remaining scope. The likely implementation path is to port the safe parts of #65197 into `extensions/memory-wiki/src`: resolve native Markdown link targets relative to the containing page, generate report-relative native links, sanitize bridge-rendered reply tags, include a bridge renderer-version token in the sync fingerprint, and make imported-source skip logic validate existing generated source pages before skipping. #70986 can remain the broader parser/title-resolution follow-up, but it should not be used to close this bridge/cache issue by itself.
