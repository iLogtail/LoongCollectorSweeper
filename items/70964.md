## Summary

Keep #70964 open. Current main still has the reported Gemini CLI OAuth deadlock path: the local flow only prints the authorization URL when `ctx.openUrl(authUrl)` throws, while the actual `models auth login` wiring discards the browser-open boolean and resolves even when browser opening fails. Related PR #71469 appears to be the active fix, but it is open and not merged in the provided GitHub context, so the issue should remain open until that fix or an equivalent lands.

## Best Possible Solution

Keep the issue open and use PR #71469, or an equivalent maintainer patch, to print the Gemini OAuth authorization URL before best-effort browser launch and add a focused regression test for the local non-remote flow where `openUrl` resolves without providing a usable browser callback.
