## Summary

Keep open. Current `main` still models Mattermost mention gating as a single `requireMention` setting for channel/group traffic. Thread context is passed through the monitor path, but the gating logic does not use it, so the requested separate behavior for top-level channel posts vs thread replies is not implemented on `main`.

## Evidence

- **Config schema only exposes one mention toggle:** The Mattermost config schema defines `requireMention` on the group schema and account schema, but there is no thread-specific field such as `requireMentionInThread`.
  - file: [extensions/mattermost/src/config-schema-core.ts:11](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/config-schema-core.ts#L11)
  - command: `nl -ba extensions/mattermost/src/config-schema-core.ts | sed -n '1,120p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Account resolution produces a single effective requireMention value:** `resolveMattermostRequireMention` collapses `chatmode` and config into one `requireMention` boolean on the resolved account; there is no separate thread policy in the resolved runtime state.
  - file: [extensions/mattermost/src/mattermost/accounts.ts:66](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/mattermost/accounts.ts#L66)

  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Thread id is ignored by mention gating:** `MattermostMentionGateInput` includes `threadRootId`, but `evaluateMattermostMentionGate` computes `shouldRequireMention` only from chat kind, channel id, and the single `requireMentionOverride`. No branch checks whether the inbound message is in a thread.
  - file: [extensions/mattermost/src/mattermost/monitor-gating.ts:25](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/mattermost/src/mattermost/monitor-gating.ts#L25)

- **Runtime still drops non-mentioned group/thread traffic under the same gate:** The Mattermost monitor passes `threadRootId` into the mention gate, then drops the message on `missing-mention` without any thread-specific exception. The user-facing docs also describe only per-channel `requireMention` overrides.
