## Summary

Keep #69966 open. Current main still treats the bootstrap profile knobs as global `agents.defaults` settings: `AgentConfig`/`AgentEntrySchema` do not allow `agents.list[]` overrides for `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`; runtime helpers resolve those values from defaults only; and the Control UI profile presets patch only `agents.defaults`. Related per-agent context-limit work does not cover bootstrap injection budgets.

## Best Possible Solution

Keep #69966 open. The best implementation is a first-class per-agent bootstrap profile contract: add typed optional `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` or a named bootstrap-profile reference to `agents.list[]`; validate it in the strict schema; resolve and deep-merge per-agent values over `agents.defaults`; thread the effective values through bootstrap context injection, diagnostics, `/context`, doctor output, and the Control UI; and update docs/help/generated schema plus focused schema/resolver/runtime/UI tests.
