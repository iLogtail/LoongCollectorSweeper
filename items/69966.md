## Summary

Keep #69966 open. Current main still treats `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` as global `agents.defaults` settings, and there is no current per-agent bootstrap profile/schema/runtime/UI path for `agents.list[]`. The provided related-item context does not identify a canonical duplicate or shipped implementation; the closest analogous per-agent config request (#13736) is also still open.

## Best Possible Solution

Keep the issue open for a core config/runtime/UI implementation. The likely path is to add explicit `agents.list[]` fields or named profile references for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars`, update the strict config schema/types/help/docs and generated config metadata, then thread agent-aware resolution through embedded and CLI runner bootstrap preparation plus the Control UI profile workflow. The implementation should define clear precedence against `agents.defaults` and should include config schema tests plus runtime resolution tests for at least two agents with different bootstrap budgets.
