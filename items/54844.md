## Summary

Current main already carries the Responses reasoning-compat fix for this class of failure. GitHub Copilot Responses routes are excluded from OpenAI-only reasoning compat injection, unsupported disabled reasoning payloads are stripped before send, and the changelog says the unsupported `reasoning.effort: "none"` fix shipped in v2026.4.20.

## Close Comment

Closing as fixed. This was a Codex review against current `main`, and the current code path already avoids sending the unsupported disabled-reasoning payload that was causing these GitHub Copilot GPT-5 mini failures. In particular, Responses payload policy now strips unsupported `reasoning.effort: "none"` before send (`src/agents/openai-responses-payload-policy.ts:125`, `src/agents/openai-responses-payload-policy.ts:174`), native GitHub Copilot Responses is explicitly not treated as an OpenAI reasoning-compat route (`src/agents/provider-attribution.test.ts:941`), and the transport path applies that policy while skipping the fallback disabled-reasoning branch for Copilot (`src/agents/openai-transport-stream.ts:897`). The shipped changelog for `v2026.4.20` also calls out the exact fix class: `/think off` no longer sends unsupported `reasoning.effort: "none"` payloads for GPT Responses models (`CHANGELOG.md:384`, `CHANGELOG.md:395`).
