## Summary

Keep open. Current main has added the shared `channels.modelByChannel` config, which covers much of the channel/group side of this request and matches the related #54476 ClawSweeper finding for Discord per-channel overrides. The broader #53638 request still includes persistent per-DM model overrides for Slack and WhatsApp, and current main does not appear to route direct peer IDs into `resolveChannelModelOverride`, so a meaningful requested capability remains unimplemented.

## Best Possible Solution

Keep #53638 open and narrow the remaining work to direct-conversation support: decide the public config shape for DMs, teach `resolveChannelModelOverride` to consider direct peer identifiers from session/origin context for Slack and WhatsApp, document the key format, and add resolver plus channel integration tests proving group/channel behavior still wins in the intended precedence order.
