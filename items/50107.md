## Summary

Keep open. Current main has a real partial mitigation for string-only OpenAI-compatible endpoints, but it is opt-in via `compat.requiresStringContent`; the NVIDIA bundled catalog and the issue's NVIDIA-style config do not set that flag, so the reported NVIDIA message-format failure can still occur. The linked PR #50152 remains open/unmerged and does not make the issue obsolete.

## Best Possible Solution

Keep the issue open and land a narrow provider-compatible fix: either mark NVIDIA model entries/custom NVIDIA resolution with `compat.requiresStringContent: true` or add a documented provider capability that makes NVIDIA's OpenAI-compatible transport flatten pure text content automatically while preserving mixed media arrays. Then add NVIDIA-specific regression coverage for the issue's `openai-completions` payload shape and update NVIDIA docs if the flag remains user-facing.
