## Summary

本 Issue 报告了特定环境下 Docker 容器 inspect 超时导致采集失败的问题。日志显示客户端请求超时设置为 30s，且触发重试机制，符合预期保护逻辑。该问题目前仅在一个环境中复现，手动 inspect 正常，疑似与特定环境 Docker Daemon 负载、并发限制或个别容器状态有关。维护者已跟进请求日志，用户已提供，但尚未明确代码侧缺陷或提供通用复现路径。证据不足以支持关闭，需进一步排查环境差异或确认是否可通过调整超时/并发参数缓解。

## Best Possible Solution

建议维护者继续跟进：1. 确认该环境 Docker Daemon 资源使用率及是否存在响应慢的异常容器；2. 指导用户尝试调整 client request timeout 或 max fetchOne count 参数观察是否缓解；3. 若确认为高负载环境下的普遍痛点，可评估在代码侧引入自适应退避或更细粒度的超时配置。当前保持开启等待进一步诊断或配置验证。
