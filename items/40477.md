## Summary

Keep open. Current `main` still uses the generic `Reply with: /approve <id> ...` text in the shared exec-approval forwarding fallback, and the tests explicitly lock that behavior in. OpenClaw now has a newer Telegram/native approval path with prefilled `/approve ...` commands and inline buttons, so the original Telegram/mobile pain is partly mitigated, but the exact shared notification change requested in #40477 is not implemented on `main`.
