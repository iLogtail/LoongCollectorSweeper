## Summary

Keep #40067 open. Current main still does not suppress the Playwright `Page.handleJavaScriptDialog` / `No dialog is showing` unhandled rejection, but the PR's current shape hard-codes Playwright-specific matching into core infra instead of using the browser plugin-owned unhandled-rejection seam called out in maintainer discussion.

## Best Possible Solution

Keep the PR open for a rewrite: move the Playwright dialog-race predicate under `extensions/browser/src/**`, register and unregister it from the browser plugin service lifecycle via `openclaw/plugin-sdk/runtime-env`, and add browser-plugin tests for the exact `Protocol error (Page.handleJavaScriptDialog): No dialog is showing` message plus nested/wrapped error shapes. Core unhandled-rejection infra should remain generic.
