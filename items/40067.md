## Summary

Keep open. The underlying Playwright dialog-race crash path still appears unhandled on current `main`, but this PR fixes it in the wrong layer: it adds Playwright-specific matching to core unhandled-rejection infra even though OpenClaw already exposes a plugin-owned unhandled-rejection seam and the browser plugin has its own lifecycle to register browser-specific handlers.

## Evidence

There is no Playwright/browser-specific branch on current `main`.
