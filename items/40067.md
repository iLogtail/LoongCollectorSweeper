## Summary

Keep open. The reported Playwright dialog-race crash path still is not implemented on current main, and the PR's current shape puts Playwright-specific matching into core unhandled-rejection infrastructure instead of the browser plugin-owned lifecycle/seam that already exists.

## Best Possible Solution

Keep the PR open for a rewrite: move the Playwright dialog-race predicate under `extensions/browser/src/**`, register/unregister it from the browser plugin service lifecycle via `openclaw/plugin-sdk/runtime-env`, and add browser-plugin tests for the exact message plus nested/wrapped error shapes. Core unhandled-rejection infra should remain generic.
