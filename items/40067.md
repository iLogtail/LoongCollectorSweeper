## Summary

Keep #40067 open. The underlying Playwright dialog-race crash remains plausible on current main because unknown unhandled rejections still fall through to the fatal gateway exit path, and current source/tests have no `Page.handleJavaScriptDialog` / `No dialog is showing` matcher. The PR should not be closed as obsolete, but the current patch shape puts Playwright-specific matching in core infra; maintainer discussion and current plugin seams point to a browser-plugin-owned rewrite instead.

## Best Possible Solution

Keep the PR open for a rewrite. Move the Playwright dialog-race predicate under `extensions/browser/src/**`, register and unregister it from the browser plugin service lifecycle via `registerUnhandledRejectionHandler`, and add browser-plugin tests for the exact `Protocol error (Page.handleJavaScriptDialog): No dialog is showing` message plus wrapped/nested error shapes. Core unhandled-rejection infra should remain generic.
