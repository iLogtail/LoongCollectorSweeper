## Summary

Keep PR #71237 open. Current main has partial orphan-transcript handling through doctor state-integrity repair and disk-budget cleanup, but it does not include the proposed `pruneOrphanedTranscripts` utility or an on-demand sessions-cleanup path that hard-prunes old orphan JSONL files by caller-supplied preserved paths. The PR is recent, focused, includes field performance data, tests, and active author follow-up addressing bot/Codex review concerns, so it remains useful rather than cleanup-closeable.

## Best Possible Solution

Keep the PR open for maintainer review, but rebase it onto current main and aim it at a dedicated `openclaw sessions cleanup` orphan-prune flow that reuses current session target/path resolution, preserves checkpoint/custom paths, supports dry-run/enforce, and avoids duplicating doctor/disk-budget semantics.
