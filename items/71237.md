## Summary

Keep PR #71237 open. Current main has related cleanup paths, but they are partial: store maintenance archives transcripts only for entries it removes, doctor can archive top-level orphan transcripts, and disk-budget cleanup removes unreferenced transcript files only when a configured budget is exceeded. Current main does not provide the PR's standalone `pruneOrphanedTranscripts` utility or a normal `sessions cleanup` path that directly reclaims arbitrary orphan `.jsonl` files.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge/iterate the standalone primitive with clear preserved-path requirements, then wire it into `openclaw sessions cleanup` using a union of all live transcript and checkpoint paths across stores, or supersede it with an equivalent mainline implementation that directly reclaims orphan transcripts without relying only on doctor archival or disk-budget pressure.
