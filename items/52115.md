## Summary

Keep open. Current main still matches the core freshness bug for the default builtin memory backend: the gateway reuses a long-lived memory manager, file-watch changes only mark the manager dirty and schedule debounced sync, and `search()` starts search-triggered sync asynchronously before querying the existing index. The existing `manager.async-search.test.ts` also explicitly asserts that searching does not await sync. QMD has a synchronous dirty-search path, but the default backend is still `builtin`, so the issue is not implemented away for the reported default `memory_search` surface.
