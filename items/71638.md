## Summary

Keep #71638 open. Current main still exposes only generic routing/sender metadata on Telegram message_received/internal message:received hook events; it does not surface Telegram forward, via-bot, or reply provenance in MessageReceivedHookEvent.context.metadata. The prior ClawSweeper comment also reached the same conclusion, and the current main SHA still shows no via_bot/viaBot implementation.

## Best Possible Solution

Keep the issue open and implement a Telegram-owned additive fix: normalize forward_from/forward_origin/forward_from_chat, via_bot, and reply_to_message from the inbound Telegram msg into bounded hook metadata; thread that through the canonical message hook context and both internal/plugin received-event mappers; cover the normal dispatch path plus mention-skip ingest with focused tests.
