## Summary

Keep #53500 open. Current main has the plugin seams needed for a discovery/trust layer, including before_tool_call and plugin-facing SSRF helpers, but it still does not implement /.well-known/agent.json discovery, trust-registry verification, pricing/trust signal surfacing, or revoked-operator blocking. The related maintainer-guided work is still open/draft and explicitly leaves the #53500 agent.json resolver and blocking semantics as follow-ups, so this is not implemented, not superseded, and not a clean ClawHub close.

## Best Possible Solution

Keep the issue open as the canonical design thread for the agent.json trust-verification branch of structured discovery. Let #66717/#66474 continue plugin-first validation, using before_tool_call plus plugin-sdk/ssrf-runtime, and close #53500 only after the agent.json resolver, trust-registry verification, and any revoked-operator/blocking policy are either shipped or explicitly moved to a canonical external/plugin tracking item.
