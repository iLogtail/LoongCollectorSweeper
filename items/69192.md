## Summary

Keep open. Current main still has the reported diagnosability gap: most hook 4xx responses are returned directly without a `logHooks.warn` journal line, and the ambiguous `hook mapping requires message` rejection string remains. The linked PR #69267 was closed unmerged, so it does not supersede or implement the issue.

## Best Possible Solution

Keep the issue open and implement this in `createHooksRequestHandler` with a small centralized hook-rejection helper that logs one warn line before every non-2xx hook response. The log should include a stable reason code, path/subpath, status, content type/body metadata where safe, and request/client correlation without secrets or raw payloads. Add focused regression tests for invalid JSON, mapped `hook mapping requires message`, auth/method/path rejects, and keep the existing 4xx response contract intact.
