## Summary

Keep open. Current main still has the reported diagnosability gap: most hook 4xx rejection branches return directly without a `logHooks.warn` journal line, including invalid JSON/body errors and mapped-hook validation failures. The related PR #69267 was closed unmerged, and its referenced commit is not present in this checkout.

## Best Possible Solution

Keep the issue open and fix it in the core gateway hook handler. A good maintainer path is to add a small centralized hook-rejection logging helper in `src/gateway/server-http.ts` that logs warn-level branch-specific reason codes and safe request metadata before every hook non-2xx response, then add focused tests that inject a mock `logHooks` and cover invalid JSON/body errors, auth/method/not-found, `/hooks/agent` validation, mapped-hook validation, channel/agent/session policy failures, and the ambiguous mapped-message case. If the user-facing error strings are intentionally stable, add separate internal reason codes rather than changing response bodies casually.
