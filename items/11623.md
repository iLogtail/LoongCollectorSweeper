## Summary

Keep open. Current main still does not implement the requested macOS per-agent floating Clawi bubbles, and the current plugin/ClawHub surface does not expose a native macOS SwiftUI/AppKit window extension point that would make this safely externalizable today.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw/ that connects bubble state to agent/session activity, persisted positions, and chat routing, or a maintainer-led design for a native macOS UI extension API before moving this to ClawHub/plugins.
