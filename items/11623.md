## Summary

Keep #11623 open. Current main still has adjacent macOS UI surfaces only: a menu-bar anchored hover HUD, WebChat window/panel, agent-events debug window, and a single Talk Mode orb. It does not implement the requested multiple configurable per-agent floating Clawi bubbles with drag positioning, anchor/mouse-follow behavior, persisted positions, or bubble-launched per-agent chat routing. A ClawHub close is premature because the repo’s documented plugin surface does not expose native macOS SwiftUI/AppKit window extension points for plugins.

## Best Possible Solution

Keep #11623 open. The best path is a native macOS implementation under apps/macos/Sources/OpenClaw/ that connects bubble state to agent/session activity, persists positions, and routes chat to the selected agent, or a maintainer-led native macOS UI/window plugin API design before this can move to ClawHub/plugins.
