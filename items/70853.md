## Summary

Keep open. Current `main` still rotates a stable Telegram DM session key onto a fresh `sessionId` during reset, archives the prior transcript, and starts a new live transcript file; I did not find a replay/hydration path on `main` that would preserve DM continuity after that rotation. The issue is also fresh and already cross-referenced to an open PR (#70898), so conservative cleanup should not close it.

## Evidence
