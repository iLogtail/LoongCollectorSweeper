## Summary

Keep PR #50483 open. Current main does not implement several central pieces of the PR: Swift agent events still drop `sessionKey`, `ChatViewModel` still filters agent streams by `sessionId`/`runId` only, assistant text segment IDs are regenerated with `UUID()`, and streaming scroll updates still fire immediately on every token change. This is plausible shared iOS/macOS chat UI work, not implemented, duplicate, ClawHub, or external cleanup.

## Best Possible Solution

Keep the PR open. The best maintainer path is to require a clean rebase/split of the unrelated Windows installer change, resolve the latest pending-run/session-key review comments, and land the shared `OpenClawChatUI` fixes with targeted Swift tests for session-key agent events, session switching during active runs, stable assistant segment IDs, bootstrap retry/idempotency, and streaming scroll throttling. If `sessionKey` is now part of the public agent event contract, align the gateway protocol schema too.
