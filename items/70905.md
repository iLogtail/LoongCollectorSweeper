## Summary

Keep #70905 open. Current main still does not expose `agentId` on `ChannelOutboundContext`, even though the outbound pipeline already carries agent scope for media policy, hooks, gateway sends, and plugin action discovery. This is a plausible additive core plugin-SDK seam, not a ClawHub-only item.

## Best Possible Solution

Keep the issue open for a focused additive SDK change: add optional `agentId?: string` to `ChannelOutboundContext`, populate it from `session.agentId` or mirror context where adapter ctx is assembled, and add narrow contract tests plus channel-plugin docs/API-baseline updates for the outbound send paths.

## Risks / Open Questions

- Public SDK surface change: adding the field should remain optional and needs API-baseline/export checks if the generated SDK baseline covers this type.
- Channel adapter coverage should include direct text, media, payload, and formatted send paths so agent attribution does not work only for one delivery mode.
- Maintainers should confirm the intended privacy/security semantics before exposing stable agent ids to all channel plugins.
