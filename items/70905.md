## Summary

Keep #70905 open. Current main already carries agent ids through upstream outbound/session paths for media scoping, hooks, gateway sends, and plugin message actions, but the public `ChannelOutboundContext` delivered to channel outbound adapters still has no `agentId`, and the direct delivery context builder does not copy `params.session.agentId` or `params.mirror.agentId` into sendText/sendMedia/sendPayload contexts.

## Best Possible Solution

Implement the additive SDK/core change: add optional `agentId?: string` to `ChannelOutboundContext`, propagate it from `DeliverOutboundPayloadsCoreParams.session?.agentId` or mirror context through `ChannelHandlerParams` and `createChannelOutboundContextBase`, cover sendText/sendMedia/sendPayload/formatted outbound paths with focused tests, and update plugin SDK docs/API baseline because this is a public plugin contract.
