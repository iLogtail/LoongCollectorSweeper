## Summary

Keep open: current `main` still does not expose `agentId` on the outbound channel-plugin context, even though core already tracks that value in outbound session state and uses it in non-inbound delivery paths. The request is coherent and fits OpenClaw's stated direction of extending the plugin API when a useful capability cannot yet be built as a plugin.

## Evidence

- **Public outbound channel context has no agentId:** `ChannelOutboundContext` currently includes `cfg`, `to`, `text`, media/thread/account/identity fields, `deps`, and `silent`, but no `agentId` field.
