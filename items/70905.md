## Summary

Keep #70905 open. Current main still does not expose agentId on ChannelOutboundContext or propagate the existing outbound session agentId into channel adapter send contexts. The request is a plausible additive plugin SDK/core plumbing change rather than a ClawHub-only plugin idea, because plugin authors cannot supply this metadata themselves without a core context field.

## Best Possible Solution

Add optional agentId?: string to ChannelOutboundContext, thread it through DeliverOutboundPayloadsParams/ChannelHandlerParams/createChannelOutboundContextBase from the existing session or mirror agent context, and cover sendText, sendMedia, sendPayload/renderPresentation/formatted outbound paths with focused tests. Update the channel plugin SDK docs to state when agentId is present and that display identity remains separate from stable agent attribution.
