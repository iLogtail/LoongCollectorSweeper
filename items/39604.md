## Summary

Current `main` still blocks private/internal `web_fetch` targets by default, and the runtime config schema still does not expose a `tools.web.fetch.allowPrivateNetwork` opt-in or equivalent. The issue remains actionable rather than stale: there are multiple corroborating reports in the discussion, and the linked PR #39630 is still open.
