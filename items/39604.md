## Summary

Keep open. Current main still does not expose `tools.web.fetch.allowPrivateNetwork` or an equivalent private-network opt-in for `web_fetch`; it only supports `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`. The issue remains actionable, with multiple corroborating comments and an open implementation PR (#39630).

## Best Possible Solution

Keep this issue open until OpenClaw either lands a guarded `web_fetch` private-network opt-in, preferably under a clearly named SSRF policy with tests/docs/config schema updates, or a maintainer explicitly decides the product should not expose that capability. PR #39630 is the current canonical implementation attempt for the same remaining work.
