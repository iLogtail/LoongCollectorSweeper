## Summary

Keep #39604 open. Current main still does not expose a config-level localhost/LAN/private-network opt-in for `web_fetch`; it only supports the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` exception for 198.18.0.0/15 fake-IP proxy setups. The linked PR #39630 remains the active unmerged implementation attempt in the provided GitHub context, while #43700 was closed unmerged, so the requested behavior is not implemented or superseded.

## Best Possible Solution

Keep #39604 open until maintainers either land an explicit default-off `web_fetch` private-network opt-in or make a deliberate product/security decision not to support it. The likely implementation belongs under `tools.web.fetch.ssrfPolicy`, preferably with a deliberately named option such as `dangerouslyAllowPrivateNetwork`, runtime policy wiring through `runWebFetch()`, cache-key isolation, schema/help/generated metadata, docs at https://docs.openclaw.ai/tools/web-fetch, and SSRF regression tests while preserving the existing RFC2544-only option.
