## Summary

Current `main` mitigates one narrow overwrite path by making memory-flush writes append-only, but it does not implement the requested general agent-callable `append` tool. The issue should stay open.

## Evidence

- **No `append` tool is registered in core tool creation:** `createOpenClawCodingTools()` only preserves `read`/`write` for memory-flush runs and wraps `write`; there is no separate `append` tool added to the returned tool list.
