## Summary

Keep open. Current main has adjacent network/startup changes, but I did not find source, tests, docs, or release evidence proving the Windows 10 + Node 22 gateway-wide outbound HTTP failure is fixed. The requested model-prewarm workaround is still absent, and the startup path still awaits primary-model prewarm before channel startup.

## Best Possible Solution

Keep the issue open. Maintainers should reproduce on native Windows 10 with Node 22 and multiple Discord/Telegram providers, isolate whether global Undici dispatcher mutation or startup concurrency corrupts gateway-process HTTP, and consider a bounded `prewarmConfiguredPrimaryModel` path plus a focused `OPENCLAW_SKIP_MODEL_PREWARM` workaround so channels can start while the root cause is investigated.
