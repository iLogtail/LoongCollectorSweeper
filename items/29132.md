## Summary

Keep open. Current main still lacks a public same-turn pre-LLM tool-result sanitization hook. OpenClaw has trusted bundled tool-result middleware that can rewrite results before model consumption, but current code and docs explicitly keep that seam unavailable to external plugins, while public hooks only cover pre-tool params, after-tool observation, transcript persistence, prompt additions, or inbound dispatch.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a documented, trust-gated public API for same-turn pre-model tool-result rewriting, or maintainers explicitly decide that this security-sensitive capability remains limited to bundled plugins. A complete path should define ordering relative to `before_tool_call`, `after_tool_call`, and `tool_result_persist`, specify failure behavior, and add Pi/Codex coverage proving the model only sees sanitized output.
