## Summary

Keep #65229 open. The underlying doctor false-positive is still present on current main, but the PR is not ready to auto-close or merge: the first fix was a no-op, the later approach uses a throwing SecretRef resolver that can abort doctor in the unresolved case that current doctor intentionally warns about, and the PR branch includes unrelated drift beyond the stated doctor fix.

## Best Possible Solution

Keep #65229 open, but request a narrowed revision. The best fix is in src/flows/doctor-health-contributions.ts: resolve gateway.auth.token through the existing non-throwing SecretRef token-resolution path, pass an authOverride only when a token is actually resolved, preserve the current read-only warning when unresolved, and add a doctor regression test for a custom env-backed SecretRef that is present plus the missing-env warning case. Keep #65201 as the canonical bug until that lands.
