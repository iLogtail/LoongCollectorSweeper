## Summary

Keep open. Current main still has the reported doctor false-positive path: `runGatewayAuthHealth` resolves gateway auth without first resolving `gateway.auth.token` SecretRefs, while `resolveGatewayAuth` deliberately drops SecretRef token values and only falls back to `OPENCLAW_GATEWAY_TOKEN`. Gateway startup has a separate pre-resolution path, so the PR is addressing a real mismatch, but the current PR head also needs revision because its proposed `resolveGatewayTokenSecretRefValue` path can throw for unavailable refs and the branch includes unrelated changes.

## Best Possible Solution

Keep the PR or linked issue open, but revise to a focused doctor-health fix: resolve `gateway.auth.token` through a non-throwing SecretRef-aware path such as `resolveGatewayAuthToken` before computing `needsToken`, preserve the current warning when the ref is genuinely unavailable, add a regression test for a custom env-backed SecretRef, and drop unrelated branch drift before merge.
