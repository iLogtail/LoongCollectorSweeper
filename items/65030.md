## Summary

Keep PR #65030 open. Current main includes the related #65952 known Bedrock context-window lookup for known models, but it does not implement #65030's complementary provenance behavior for unknown/future Bedrock fallbacks. The PR should not be closed as superseded, though it needs fixes before it is mergeable.

## Best Possible Solution

Keep the PR or a narrower successor open as a provenance follow-up on top of the #65952 known-window lookup. The best path is to preserve known Bedrock context-window values, add explicit provenance only for hardcoded fallback values, propagate it through the resolved runtime model type into resolveContextWindowInfo, and add focused tests for unknown foundation models, unknown inference profiles, explicit defaultContextWindow, and cache behavior.
