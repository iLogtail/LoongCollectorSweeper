## Summary

Keep #54982 open. Current main still lacks a TUI flag or option to preserve intermediate assistant streaming output, and the related feature issue #11669 was closed as superseded by this PR, making this PR the canonical implementation thread. It should not be auto-closed, but it needs maintainer/code review because the proposed per-delta preservation has unresolved concerns around component flooding, duplicate final chunks, and no-output suppression behavior.

## Best Possible Solution

Keep #54982 open as the canonical implementation PR. The best path is to revise it so preservation records meaningful intermediate snapshots without unbounded per-token log churn, deduplicates a final response that matches the latest streamed snapshot, removes all preserved snapshots when a run is finalized with no displayable output, documents the history-cap tradeoff, and keeps focused ChatLog/CLI/event-handler regression coverage.
