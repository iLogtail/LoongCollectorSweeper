## Summary

Keep #9658 open. Current main still models the affected tool payloads as inline string parameters and does not provide streamed or file-backed arguments for exec, process write/paste, write, or apply_patch. The provided issue discussion also includes an April 26, 2026 maintainer comment explicitly keeping this item open for the same core-transport/API gap.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete fix likely belongs in core tool transport and schemas: add a backward-compatible streamed or file-backed payload/reference mechanism for exec command/stdin, process write/paste, write content, and possibly apply_patch; document supported limits and workarounds; and add regression coverage for large and special-character payloads. If maintainers decide not to support this, document the supported limits and safe chunking/file workflows instead.
