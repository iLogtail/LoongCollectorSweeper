## Summary

Keep #9658 open. Current main still does not provide streamed or file-backed tool argument transport for the affected built-in tools: exec command payloads, process write/paste payloads, write content, and apply_patch all remain inline tool-call strings. The existing maintainer discussion comment by steipete explicitly kept this issue open, and current source/docs still support that assessment.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete fix likely belongs in core tool transport and schemas: add a backward-compatible streamed or file-backed payload mechanism for exec stdin/command, process write/paste, write content, and possibly apply_patch, then document limits/workarounds and add regression coverage for large and special-character payloads. If maintainers decide not to support this transport, document the supported limits and safe chunking/file workflows instead.
