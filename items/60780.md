## Summary

Keep open. Current main still appears to allow the reported double-delivery path for `sessions_spawn(runtime: "acp", thread: true, mode: "session")`: the ACP child turn inline-delivers to the bound thread, while `sessions_spawn` still registers the accepted ACP run for completion announcement by default. The native subagent path has an explicit suppression guard for this inline-thread case, but the ACP path does not show the same guard.

## Best Possible Solution

Keep the issue open and fix ACP parity with native subagents: when an ACP `mode: "session"` spawn is already inline-delivered to its bound thread, `sessions_spawn` should either skip completion announcement tracking or register it with `expectsCompletionMessage: false`/`deliveryStatus: "not_applicable"`. Add a regression test for Discord thread-bound ACP session mode proving one visible response per turn, while preserving `streamTo: "parent"` and run-mode completion behavior.
