## Summary

Keep #25592 open. Current main has commentary/final_answer hardening, but it still preserves channel-facing delivery for unphased assistant text before tool execution and has no replyMode/tool-only or configurable final-only channel policy. The latest issue discussion also includes a maintainer Codex review explicitly keeping this as the canonical broader product bug.

## Best Possible Solution

Keep #25592 as the canonical product bug until OpenClaw implements or explicitly rejects a channel-safe policy such as final-only delivery, opt-in replyMode/tool-only explicit sends, or configurable enforceFinalTag for channel sessions. The existing commentary-phase hardening should stay, but it does not close the broader unphased intermediate-text leak.
