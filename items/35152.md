## Summary

Keep #35152 open. Current main has an adjacent installer-time warning for multiple global npm OpenClaw installs, but `openclaw doctor` still does not implement this PR's duplicate binary/version/systemd diagnostic. The PR is stale against the current doctor flow and would need a refresh rather than closure as implemented.

## Best Possible Solution

Keep #35152 open until a maintainer refreshes or replaces it with a current doctor health contribution that safely scans duplicate OpenClaw installs, reports versions and relevant service metadata, respects current `--deep` service-scan semantics, and includes focused tests. If maintainers decide the installer warning plus gateway service hardening is sufficient, close it explicitly as a product decision rather than as already implemented.
