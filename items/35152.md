## Summary

Keep #35152 open. Current main still does not implement the requested `openclaw doctor` duplicate-installation diagnostic. It has source-workspace install hygiene, extra gateway-service checks, multiple-state-dir warnings, and an installer-time duplicate global-install warning, but no doctor binary/install-root inventory with versions, realpath deduplication, service metadata, and remediation as proposed by this PR.

## Best Possible Solution

Keep #35152 open until it is refreshed or superseded by a current doctor-health implementation that safely detects duplicate OpenClaw binaries/install roots, deduplicates by realpath, reports versions and relevant service metadata, respects the current `doctor --deep` service-scan boundary, uses safe process APIs, derives npm package/bin locations correctly, and includes focused regression tests.
