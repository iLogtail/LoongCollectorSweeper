## Summary

Keep #71973 open. Current main has the broader timezone-only system prompt and cache-boundary work, but the narrower report still matches source: exact wall-clock `Current time:` text is still generated and injected into heartbeat, cron, and post-compaction prompt content, with tests explicitly asserting that behavior. The related closed prompt-caching issues addressed stable system-prompt boundaries, not this remaining per-turn automation prompt/cache-control placement question.

## Best Possible Solution

Keep the issue open for a targeted prompt-cache optimization. The likely fix belongs in the automation prompt/payload composition layer: preserve stable timezone metadata, split or late-bind exact wall-clock time outside cache-tagged stable blocks, include exact time only when the run needs it, and add payload-level regression tests for heartbeat, cron, and post-compaction turns so provider cache markers do not cover volatile timestamp text unnecessarily.
