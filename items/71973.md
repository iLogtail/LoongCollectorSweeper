## Summary

Keep #71973 open. Current main already fixed the older stable system-prompt problem, but this newer issue is narrower: exact wall-clock `Current time:` text is still intentionally injected into heartbeat, cron, and post-compaction turn bodies, with tests codifying that behavior. That remaining optimization is actionable and is not fully covered by the closed related prompt-cache issues.

## Best Possible Solution

Keep the issue open for a narrow performance/product fix: preserve stable timezone metadata in the system prompt, then audit heartbeat, cron/system-event, and post-compaction body prompts so exact wall-clock time is only injected when that turn actually needs it or is moved as late as the provider payload shape allows. Add regression coverage that proves cache-sensitive stable prefixes remain unchanged while date-dependent workflows still get enough time context.

## Risks / Open Questions

- Provider caching semantics differ across Anthropic, OpenAI, Google, and CLI transports, so the eventual change should include provider-shape tests or cache-trace/live evidence for the affected recurring automation paths.
