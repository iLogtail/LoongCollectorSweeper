## Summary

Keep open. Current main still only preserves the selected Node runtime's own bin directory for daemon service PATH planning, and there is no `.vite-plus` / `js_runtime` companion-bin handling or regression coverage. The PR addresses a concrete QMD service-runtime gap with a narrow shared helper/test change rather than obsolete or duplicate work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow implementation like the proposed helper plus a focused regression test proving vite-plus-managed Node paths add both the runtime bin and the matching `.vite-plus/bin`, while non-vite-plus absolute paths and bare executable names keep current behavior.
