## Summary

Keep open. Current main mitigates the original zombie-session accumulation via cron run-session pruning, but it does not implement this PR's narrower behavior of persisting isolated cron session terminal status (`done`/`failed`/`timeout`) on completion. The PR is also not merge-ready as-is because the discussion reports merge conflicts and prior review comments raised status-persistence edge cases that still need maintainer review.

## Best Possible Solution

Keep the PR open or replace it with a narrower maintainer-owned follow-up that fits current main: persist isolated cron run session status and end metadata best-effort after all terminal outcomes, after delivery result classification, with regression coverage for success, fatal payload, thrown error, timeout, delivery error, and persistence failure paths.
