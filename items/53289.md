## Summary

Keep open. Current `main` does not already carry this docs clarification, and the PR text shown in the review context is not yet correct because `openclaw sessions list` is not a valid CLI form on current `main`.

## Evidence

- **Current docs do not yet implement the clarification:** `docs/tools/index.md` on `main` still jumps straight from the built-in tools heading to the table. It lists tool identifiers like `sessions_*` / `agents_list`, but it does not include the proposed note explaining that these are tool IDs rather than shell commands.
  - file: [docs/tools/index.md:50](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/tools/index.md#L50)
  - command: `nl -ba docs/tools/index.md | sed -n '45,85p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Sessions CLI surface on main is bare `openclaw sessions`, not `openclaw sessions list`:** The sessions docs and command registration both show `openclaw sessions` as the listing command, with `cleanup` as the only subcommand. That means the PR example `openclaw sessions list` from the provided patch/review context would document an invalid command.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/program/register.status-health-sessions.ts#L124)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Sessions docs confirm the valid examples:** The public-facing sessions CLI docs use `openclaw sessions`, `openclaw sessions --agent work`, and `openclaw sessions cleanup ...`; they do not document any `sessions list` subcommand.
  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/cli/sessions.md#L8)

- **Agents CLI surface does support `list`:** The agents command does register a `list` subcommand, and the docs say bare `openclaw agents` is equivalent to `openclaw agents list`. So the PR mixes one correct example (`openclaw agents list`) with one incorrect one (`openclaw sessions list`).
  - file: [src/cli/program/register.agent.ts:90](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/program/register.agent.ts#L90)

- **Existing PR discussion already found unresolved review issues:** The provided PR review context already calls out two problems: a duplicate introductory sentence in `docs/tools/index.md`, and a Codex review noting that `openclaw sessions list` is not a registered CLI form because `sessions` has no `list` child command on current main.
