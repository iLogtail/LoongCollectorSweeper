## Summary

Keep open. Current main has general CLI docs that use the `openclaw` prefix, but it still lacks the explicit tools-vs-shell clarification at the built-in tools surface, and the PR as written needs correction because `openclaw sessions list` is not a valid CLI form.

## Best Possible Solution

Keep the PR open or replace it with an updated docs change that adds the explicit tools-vs-shell callout to `docs/tools/index.md`, removes the duplicate sentence, and uses valid examples such as `openclaw sessions` and `openclaw agents list`; optionally cross-link to the public CLI reference at `https://docs.openclaw.ai/cli`.
