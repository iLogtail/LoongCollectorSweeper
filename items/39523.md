## Summary

Keep open. Current main still does not implement the PR's requested Feishu channel/account `mediaLocalRoots` config field or config fallback. Feishu can send local media when core injects runtime roots, but that does not satisfy the explicit config-surface request, and the submitted patch also needs revision for the current `mediaAccess`/`readFile` safety contract.

## Best Possible Solution

Keep this open, but require a revised Feishu-owned fix or a superseding PR. The fix should add the Feishu top-level/account config field only if that remains the desired product contract, update generated config metadata and docs, define whether configured roots merge with or override injected agent-scoped outbound roots, route any `"any"` behavior through the current `mediaAccess`/`mediaReadFile` safety contract, and add regression tests for configured arrays, any supported `"any"` path, and the normal outbound send flow.
