## Summary

Keep open. Current main still does not implement Feishu `mediaLocalRoots` as a user-facing channel/account config field, and PR #39523 is not obsolete. The PR is incomplete for current main because ordinary outbound sends inject agent-scoped roots that would shadow a simple config fallback, and current `loadWebMedia` also treats `localRoots: "any"` as unsafe without a readFile-backed path.

## Best Possible Solution

Keep the PR open, but require a revised Feishu-owned fix or a superseding PR. The fix should define precedence between configured Feishu roots and agent-scoped outbound roots, align any `"any"` support with the current readFile/sandbox media contract or restrict config to explicit directories, update schema/docs/generated metadata, and add regression tests for configured directory arrays, `"any"` if still supported, and the normal outbound message-send path.

## Evidence

- **Feishu schema omits mediaLocalRoots:** `FeishuSharedConfigShape` includes `mediaMaxMb` and then `httpTimeoutMs`, and that shared shape is spread into both top-level and per-account Feishu config; there is no `mediaLocalRoots` field on current main.
  - file: [extensions/feishu/src/config-schema.ts:173](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/extensions/feishu/src/config-schema.ts#L173)
  - sha: [b67d9bf7f06f](https://github.com/openclaw/openclaw/commit/b67d9bf7f06f4c655b1e0869395056c7d631abc2)
- **Feishu media sender only uses runtime roots:** `sendMediaFeishu` accepts `mediaLocalRoots?: readonly string[]` and passes only that parameter to `loadWebMedia`; it does not read `account.config?.mediaLocalRoots` and does not accept the config value `"any"`.
  - file: [extensions/feishu/src/media.ts:681](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/extensions/feishu/src/media.ts#L681)
  - sha: [b67d9bf7f06f](https://github.com/openclaw/openclaw/commit/b67d9bf7f06f4c655b1e0869395056c7d631abc2)
- **Normal outbound path injects media roots before Feishu:** The shared outbound service resolves agent-scoped media access and dispatches `mediaLocalRoots: mediaAccess.localRoots`, which supports the existing review concern that PR #39523's `mediaLocalRoots ?? account.config?.mediaLocalRoots` fallback would usually be bypassed on regular outbound sends.
  - file: [src/infra/outbound/outbound-send-service.ts:79](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/src/infra/outbound/outbound-send-service.ts#L79)

- **Feishu outbound forwards injected roots:** Feishu's outbound media adapter forwards the received `mediaLocalRoots` directly into `sendMediaFeishu`, so the regular send path reaches Feishu with an explicit roots parameter.
  - file: [extensions/feishu/src/outbound.ts:268](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/extensions/feishu/src/outbound.ts#L268)

- **Current media loader rejects unsafe any bypass:** `loadWebMedia` now rejects `localRoots === "any"` unless a readFile override is present, so PR #39523's direct `localRoots: "any"` forwarding would not satisfy the current local-file access contract by itself.
  - file: [src/media/web-media.ts:501](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/src/media/web-media.ts#L501)

  - file: [extensions/feishu/src/media.test.ts:446](https://github.com/openclaw/openclaw/blob/b67d9bf7f06f4c655b1e0869395056c7d631abc2/extensions/feishu/src/media.test.ts#L446)

## Risks / Open Questions

- Closing this PR would drop a still-real Feishu config/schema/docs gap that current main does not implement.
- Merging the PR exactly as submitted may not fix the main outbound path because injected agent-scoped roots can shadow the account-config fallback.
- The PR's `"any"` behavior conflicts with current `loadWebMedia` unsafe-bypass protection unless the fix is updated to the current readFile/sandbox contract.
