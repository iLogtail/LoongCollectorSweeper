## Summary

Keep open. Current main still does not implement user-facing `channels.feishu.mediaLocalRoots` support, and the PR is a plausible focused fix for a linked Feishu local-media bug. It should not be closed as implemented, duplicate, or stale; however, the existing patch needs revision because normal outbound sends already pass injected agent-scoped roots, which would shadow a simple `mediaLocalRoots ?? account.config?.mediaLocalRoots` fallback.

## Best Possible Solution

Keep the PR open, but require a revised Feishu fix: decide the intended precedence between channel/account `mediaLocalRoots` and injected agent-scoped roots, add schema/docs/generated metadata if the config surface is accepted, and cover array plus `"any"` cases with focused security-sensitive media tests.
