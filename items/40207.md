## Summary

Keep open. Current `main` still generates macOS LaunchAgent `StandardOutPath`/`StandardErrorPath` from `$OPENCLAW_STATE_DIR/logs` (defaulting to `~/.openclaw/logs`), does not resolve that path through `realpath` or move it to `~/Library/Logs/openclaw`, and still treats install success as `launchctl print`/loaded-state success rather than a post-install health check. The issue discussion also has a second independent confirmation plus the same workaround.

## Evidence

- **LaunchAgent logs still come from the state dir:** `resolveGatewayLogPaths` still derives `stdoutPath` and `stderrPath` from `resolveGatewayStateDir(env)/logs`, so the macOS LaunchAgent log targets remain under the OpenClaw state dir rather than a separate `~/Library/Logs/openclaw` location.
