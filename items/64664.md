## Summary

Keep open. Current main still keeps exec/plugin approvals in per-process memory, still returns the raw `unknown or expired approval id` gateway error through `/approve`, and Telegram approval callback tests still expect that raw stale-id failure. The related PR #67982 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep #64664 open. The smallest maintainer cleanup is the narrow UX fix already described in the discussion: map `isApprovalNotFoundError(...)` in `/approve` and Telegram approval callback handling to an actionable message that says the approval is no longer available, may have expired or been lost after restart, and asks the user to re-request the action or double-check a manually typed ID. Add/update focused tests in `src/auto-reply/reply/commands-approve.test.ts` and `extensions/telegram/src/bot.test.ts`. Disk persistence for approvals can remain a larger follow-up/product decision if maintainers want restart-surviving approvals.
