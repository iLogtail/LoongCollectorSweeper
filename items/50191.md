## Summary

Keep open. Current `main` still emits the older Discord/Telegram overflow messages, so this is not implemented on `main`. The PR addresses a real config-guidance gap, but the review discussion also identified one Telegram warning path that would lose fallback guidance, so there is still maintainer work to do rather than an auto-close.

## Evidence

- **Current docs advertise `nativeSkills` as the targeted setting:** The docs say native skill-command registration is controlled by `commands.nativeSkills` and per-provider `channels.<provider>.commands.nativeSkills`, which makes the broader `commands.native` disablement an overbroad recommendation for this overflow case.
