## Summary

Keep PR #68341 open. Current main already covers some MCP loopback owner-spoofing concerns with a stronger owner/non-owner token model, and dedupe overflow still evicts by timestamp, but the PR is not obsolete: current main still lacks the poll inflight dedupe path the PR adds, and the cron trust/default and failover-classification changes still need maintainer review rather than auto-close.

## Best Possible Solution

Keep the PR open, but ask for it to be narrowed or split: preserve current main's owner/non-owner MCP token contract, keep timestamp-based dedupe eviction, and focus any remaining patch on the poll inflight dedupe race with a concurrent regression test. Treat the cron trust default and failover matcher changes as separate maintainer-reviewed changes because they affect trust and retry classification semantics.
