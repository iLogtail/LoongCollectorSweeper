## Summary

Keep open. This is a same-day regression report against v2026.4.24 with a second user confirmation, and current main still starts the automatic Gateway model-pricing refresh path that fetches OpenRouter and LiteLLM pricing with timed upstream requests. I did not find a post-report mainline fix or regression coverage proving Windows npm-global startup, dashboard reachability, and Ctrl+C shutdown are fixed when those pricing requests time out.

## Best Possible Solution

Keep the issue open. Reproduce on Windows 11 with npm-global v2026.4.24 and current main while forcing OpenRouter and LiteLLM pricing requests to timeout, then verify HTTP readiness, Control UI/dashboard reachability, and SIGINT handling. If reproduced, decouple pricing refresh from startup/shutdown so pricing timeouts cannot block dashboard availability or Ctrl+C, and add a regression test around the Gateway startup path rather than only the pricing-cache helper.
