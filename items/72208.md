## Summary

Keep open. Current main does not contain the targeted pricing-refresh shutdown fix, and the linked PR #72306 is still only an open candidate fix in the provided context. The reported regression remains plausible because gateway startup still starts model-pricing refresh automatically, the upstream fetches still only use timeout signals, and the stop path only clears the refresh timer without aborting in-flight fetches or unref'ing the 24-hour timer.

## Best Possible Solution

Keep #72208 open until a fix such as #72306 lands on main or another patch proves the same behavior: gateway startup remains usable when LiteLLM/OpenRouter pricing requests stall or time out, and Ctrl+C shutdown tears down pricing refresh timers and in-flight fetches. The fix should live in `src/gateway/model-pricing-cache.ts` and be covered by focused tests for unref'd refresh timers and abort-on-stop behavior, plus Windows npm-global/manual verification if feasible.
