## Summary

Current main does not implement the requested silent reasoning mode. It has reasoning visibility controls and provider replay sanitization, but assistant thinking blocks can still be persisted in the session transcript, and the existing dropThinkingBlocks path is scoped to replay/context handling rather than transcript write-time persistence.

## Best Possible Solution

Keep this open. The best path is a core transcript-persistence policy that is separate from thinking production and reasoning display, with tests proving assistant thinking/redacted_thinking blocks are stripped before session transcript writes while provider-required replay signatures or safe omitted-reasoning placeholders are handled deliberately. Update /think or config docs only after the setting and replay behavior are implemented.

## Risks / Open Questions

- Provider replay contracts can require signed reasoning/thinking blocks for follow-up turns, so any implementation needs a replay-safe policy rather than blindly deleting every block in every path.
- The built-in memory-host extractor already avoids non-text thinking blocks, so the remaining product decision should target transcript persistence, replay context, compaction, and third-party transcript consumers explicitly.
