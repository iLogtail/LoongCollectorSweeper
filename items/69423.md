## Summary

Keep #69423 open. Current main still has the unsafe Anthropic tool conversion path: it casts `tool.parameters` and immediately reads `.properties`/`.required`, while plugin registration and collection still accept/preserve tools without schema validation. The related PR #69432 was closed unmerged, and the current release tag still contains the same vulnerable converter.

## Best Possible Solution

Patch the Anthropic tool payload builder or a shared pre-transport normalization path so malformed plugin tools cannot throw. Prefer a small helper that resolves `parameters` first, optionally accepts `inputSchema` as a compatibility alias, falls back to an empty object schema or drops the bad tool with a diagnostic, and adds regression tests for both `inputSchema` and missing-schema tools. A follow-up can add plugin-load/status diagnostics, but the crash guard should live on the runtime path that builds Anthropic payloads.
