## Summary

Keep #69423 open. Current main still has the reported Anthropic hard-crash path: the Anthropic transport maps tools by unconditionally reading `tool.parameters.properties`, while plugin tool resolution accepts factory-returned tool objects without validating or normalizing a missing `parameters` schema. The only linked fix PR in the provided GitHub context (#69432) was closed unmerged, and current tests only cover valid Anthropic tool schemas, not malformed plugin tools or `inputSchema` fallback.

## Best Possible Solution

Keep the issue open and fix the Anthropic tool conversion plus plugin boundary defensively: normalize `parameters` from `parameters` or `inputSchema`, degrade or skip malformed tools with a plugin/tool diagnostic, and add focused regression tests in `src/agents/anthropic-transport-stream.test.ts` plus plugin registration/tool-resolution coverage so third-party plugin mistakes cannot crash every Anthropic run.
