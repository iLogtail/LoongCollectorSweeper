## Summary

Keep #69423 open. Current main still has the reported Anthropic tool-schema crash path: the Anthropic converter unconditionally dereferences `tool.parameters`, plugin tool registration/collection does not validate or normalize missing schemas, and the related fix PR #69432 was closed unmerged rather than shipped.

## Best Possible Solution

Keep the issue open and land a focused core fix in the Anthropic transport path: resolve `parameters`, tolerate `inputSchema`/`input_schema` where appropriate, and degrade malformed tools to an empty object schema or skip them with a plugin diagnostic instead of throwing. Add regression coverage in `src/agents/anthropic-transport-stream.test.ts` for inputSchema-only and schema-less plugin tools, then consider a follow-up for plugin registration/status diagnostics.

## Evidence

- **Anthropic converter still dereferences missing parameters:** `convertAnthropicTools()` maps all context tools, casts `tool.parameters`, then reads `parameters.properties` and `parameters.required`. A plugin tool with only `inputSchema` or no `parameters` would still throw before the request is sent.
