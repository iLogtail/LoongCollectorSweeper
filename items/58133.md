## Summary

Keep open. Current `main` still returns an immediate `status: "timeout"` from `sessions_send` instead of leaving A2A delivery pending, so the underlying behavior reported by the PR is not implemented. The PR addresses a real gap, but the review comments identify two plausible correctness regressions that still need maintainer follow-up before merge: `agent.wait` timeout can also mean a terminal aborted run, and the deferred A2A path has no baseline snapshot to prevent stale-history announce content.
