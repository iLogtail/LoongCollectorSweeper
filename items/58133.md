## Summary

Current `main` does not implement this PR's behavior: `sessions_send` still returns an immediate `status: "timeout"` when the initial wait times out. Conservative cleanup should keep this PR open, because the request is still unmet and the proposed patch has unresolved correctness concerns around `agent.wait` timeout semantics and stale-reply delivery.

## Evidence

- **Current main still hard-fails timed-out waits:** `createSessionsSendTool()` still returns `{ status: "timeout" }` immediately when `waitForAgentRunAndReadUpdatedAssistantReply()` times out, with no delayed announce flow started from this branch.
