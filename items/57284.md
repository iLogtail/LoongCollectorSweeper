## Summary

Current `main` still implements `openclaw agent --deliver` as "run one agent turn, then optionally deliver the agent's reply." The requested direct-send/session-inject behavior is not present on `main`, and the linked follow-up PR in the issue timeline indicates this was treated as real follow-up work rather than an already-shipped fix.

## Evidence

- **CLI docs define reply-delivery semantics:** The current CLI reference says `--deliver` will "send the reply back to the selected channel/target," and every example frames `openclaw agent` as an agent turn whose reply may be delivered.
  - file: [docs/cli/agent.md:36](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/cli/agent.md#L36)
