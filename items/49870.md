## Summary

Keep open. Current `main` still does not expose `conversationId`/`accountId` on agent hook contexts or implement an `after_agent_complete` runner, so the PR's requested behavior is not implemented on `main`. At the same time, inbound-claim conversation normalization has since moved to channel-plugin resolvers, which means this PR now needs rework against current architecture rather than a cleanup close.
