## Summary

Current main already implements the queued subagent announce fallback requested by PR #70585 and linked issue #59201. The queued send path now derives `deliver` and route fields from the external best-effort target resolver, and regression tests cover no-route, channel-only, internal-origin, and valid external-route cases. I could not tie this to a shipped release; `v2026.4.24` still shows the older unconditional queued `deliver: true` shape.

## Best Possible Solution

Close PR #70585 as implemented on current main. Keep the resolver-based queued delivery behavior and existing regression coverage; no additional core change is needed unless maintainers want to separately require an explicit `bestEffortDeliver` flag on queued no-target announces.
