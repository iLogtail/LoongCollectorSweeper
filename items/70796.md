## Summary

Keep #70796 open. Current main already has session switching in the chat header and a Sessions page that can jump to chat sessions, but the requested UX is specifically a conversations/session dropdown or expandable list under Chat in the left sidebar. The current sidebar still renders Chat as a single nav item with no per-session child list.

## Best Possible Solution

Keep the issue open for a focused Control UI UX decision. If maintainers want this behavior, add a compact nested session/conversation switcher under the Chat sidebar section, reusing the existing `resolveSessionOptionGroups`, `switchChatSession`, and `loadSessions` plumbing rather than inventing a new session API. If maintainers decide the current header dropdown plus Sessions page is the intended UX, that should be an explicit product call rather than an automated cleanup close.
