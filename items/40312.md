## Summary

Keep open. Current `main` still has the control-UI route-sync fallback that converts unknown paths like `/tasks` into `chat` and then rewrites the browser URL to `/chat`. The only identified fix, PR #50997, was closed for the author's active-PR limit and was not merged. Public docs currently return 404 for `/tasks`, so the issue may need retitling or scope clarification, but the underlying web-route bug remains on `main`.
