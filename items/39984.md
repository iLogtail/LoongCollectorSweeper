## Summary

Current `main` still lacks the requested `web_fetch` / `browser` fallback guidance in runtime missing-key messages. The PR remains useful, but it targets the old core `src/agents/tools/web-search.ts` implementation; current web search execution is provider-owned, so the patch needs a refresh rather than closure.

## Best Possible Solution

Keep the PR open and ask for a refresh against current `main`: update the missing-key payloads in the bundled provider-owned web-search runtimes, preserve real environment variable names and current config paths, avoid baseline/generated churn, and add focused message coverage where practical. The docs fallback is already present, but the runtime guidance requested by this PR is still missing.
