## Summary

Keep open. Current main still lacks the requested `web_fetch` / browser fallback guidance in runtime `web_search` missing-key messages, and the PR targets an obsolete core file rather than the provider-owned runtime messages that now execute web search.

## Best Possible Solution

Keep this open and ask for a refresh against the provider-owned web search runtimes. Update the missing-key messages in the relevant `extensions/*` providers, preserve valid env var names, avoid `.secrets.baseline` churn, and add focused message coverage where practical. Consider applying the same guidance to newer API-key-backed web search providers too, so the UX is consistent.
