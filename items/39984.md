## Summary

Current main still does not implement the PR's requested runtime guidance. The old central `src/agents/tools/web-search.ts` edit target is now obsolete, but provider-owned missing-key messages still tell users only how to configure API keys and do not mention `web_fetch` or the `browser` tool. Docs already contain the fallback guidance, so the remaining work is specifically runtime error-message coverage.

## Best Possible Solution

Keep the PR open or replace it with a refreshed patch that updates the provider-owned missing-key messages across the relevant API-backed `web_search` providers, preserves literal env var names, avoids baseline churn, and adds focused coverage where practical for the missing-key message text.
