## Summary

Keep open. Current `main` still synthesizes `<media:image>`/`<media:...>` for media-only iMessage sends, then passes that value as outbound `text`, and the current iMessage tests still encode that placeholder behavior. Two related fix PRs (#40341 and #46343) were opened from the issue timeline, but both were closed unmerged, so there is no evidence the bug was implemented on `main` or shipped in the latest release.
