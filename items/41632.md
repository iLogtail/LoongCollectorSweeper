## Summary

Current `main` still sends `image` tool file/URL inputs through the auto-optimization path with fixed resize/quality presets, and `agents.defaults.imageModel` still only supports model selection/fallbacks. The requested configurable compression quality/resolution is not implemented on `main`, so this issue should remain open.
