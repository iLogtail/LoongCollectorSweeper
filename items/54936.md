## Summary

Current `main` already honors `agents.defaults.subagents.runTimeoutSeconds` when `sessions_spawn` omits `runTimeoutSeconds`, and the same behavior is present in release `v2026.4.22`.

## Close Comment

Closing from a Codex review: current `main` already honors the configured default subagent timeout when `sessions_spawn` omits `runTimeoutSeconds`. The spawn path now resolves omitted values through `resolveConfiguredSubagentRunTimeoutSeconds()` in `src/agents/subagent-spawn-plan.ts:21` and applies that in `src/agents/subagent-spawn.ts:594`, instead of falling through to an infinite timeout. There is direct regression coverage in `src/agents/openclaw-tools.subagents.sessions-spawn.model.test.ts:134`, and the docs match this behavior in `docs/tools/subagents.md:86` and `docs/gateway/config-tools.md:349`. This behavior is also present in the released `v2026.4.22` tag (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this issue is already implemented on main.
