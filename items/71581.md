## Summary

Keep #71581 open. Current main still routes exec-event and cron-event heartbeat wakes through the explicit/canonicalized session key, while `session.dmScope=main` single-owner pinning is only used by inbound DM routing paths. The linked PR #72160 is closed/unmerged and unrelated to heartbeat/session routing, so it does not supersede this issue.

## Best Possible Solution

Keep the issue open and implement a narrow core routing helper for internal system-event wakes. For `dmScope=main` with the same single-owner allowlist semantics as inbound DM routing, map same-agent direct-message session keys to `resolveAgentMainSessionKey` before both enqueue/drain and heartbeat wake resolution. Add regression tests for bash exec notify-on-exit, explicit exec system events, and cron next-heartbeat system events, including negative cases for wildcard/multi-owner allowlists, non-main dmScope values, subagent keys, and isolated heartbeat sessions.
