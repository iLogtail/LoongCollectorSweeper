## Summary

Keep open. Current main still propagates `os.networkInterfaces()` failures through the gateway LAN and tailnet helpers, and the linked fix PR #71950 remains open/unmerged. The issue describes an active gateway crash path rather than a stale, duplicate, or already-shipped cleanup item.

## Best Possible Solution

Keep #71721 open and land PR #71950 or an equivalent maintainer-reviewed fix that routes gateway LAN and tailnet interface discovery through a safe `os.networkInterfaces()` wrapper, then updates regression tests to assert graceful fallback instead of thrown `uv_interface_addresses` errors.
