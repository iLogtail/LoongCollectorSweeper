## Summary

Keep open. Current main still has the reported failure mode: the gateway LAN and tailnet discovery helpers call the throwing `readNetworkInterfaces()` path directly, while tests still assert that `uv_interface_addresses` failures are propagated. The linked PR #71950 is relevant prior work but is open/unmerged in the provided context, so the issue is not implemented or superseded on main.

## Best Possible Solution

Keep #71721 open and land PR #71950 or an equivalent maintainer-reviewed patch that switches `src/gateway/net.ts` and `src/infra/tailnet.ts` from `readNetworkInterfaces()` to the safe helper, then update the gateway/tailnet regression tests to assert graceful fallback instead of thrown errors.
