## Summary

Keep #62042 open. Current main still routes Slack proactive/message.send delivery through the Slack outbound adapter, and that sender resolves U-prefixed user targets with `conversations.open` before posting. That directly matches the reported failure class where direct `chat.postMessage(channel=<USER_ID>)` works but OpenClaw's proactive DM path can fail with `missing_scope`.

## Best Possible Solution

Keep the issue open and fix the Slack plugin sender. A conservative implementation would avoid `conversations.open` for text/block-only U-user DM sends, or fall back to `chat.postMessage(channel=<USER_ID>)` when `conversations.open` fails with a scope error, while still resolving D-channel IDs for file uploads that require them. Add targeted tests for proactive/message.send text DM delivery to a U ID and for the missing_scope fallback path; update Slack troubleshooting docs if the behavior or required scopes change.
