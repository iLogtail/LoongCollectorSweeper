## Summary

Keep open. Current main still matches the reported bug: disconnected Debug snapshot refresh returns before setting any user-visible state, the Snapshots card only reflects loading in the Refresh button label, and the Debug view does not receive connection state needed to render an inline disconnected reason. Related PR #39806 is an open, unmerged fix attempt, so it does not make the issue obsolete.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into `renderDebug()`, show an inline loading/progress state inside the Snapshots card while `debugLoading` is true, show a disconnected callout or disabled reason when refresh cannot run, and add focused tests for `loadDebug()` and `renderDebug()` behavior. Review or refresh PR #39806 as the existing implementation attempt.
