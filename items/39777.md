## Summary

Current main still lacks the requested Debug Snapshots inline loading/disconnected UI. The controller still returns immediately when disconnected, and the view only reflects loading in the Refresh button label/disabled state.

## Best Possible Solution

Keep the issue open until the Control UI implements the missing Debug Snapshots states. A good fix should pass connection/error state into `renderDebug()`, show an inline loading indicator while `debugLoading` is true, show a clear disconnected callout or disabled refresh affordance when refresh cannot run, and add focused render/controller tests. PR #39806 can be used as prior art, but it still needs maintainer review or replacement before this issue can close.
