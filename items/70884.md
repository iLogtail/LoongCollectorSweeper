## Summary

Keep PR #70884 open. Current main still lacks the trusted-proxy/WebSocket scope-clearing docs this PR adds, and the underlying behavior is real in current source and tests. The provided review context also shows the PR addressed prior bot/Codex wording concerns rather than being superseded.

## Best Possible Solution

Keep the PR open for maintainer review or merge an equivalent docs update that explicitly documents trusted-proxy WebSocket scope clearing, the HTTP-only role of `x-openclaw-scopes`, the `allowInsecureAuth` exception, and the break-glass nature of `dangerouslyDisableDeviceAuth`.
