## Summary

Keep PR #70884 open. Current main still has the docs gap the PR targets: trusted-proxy WebSocket/device-identity docs do not explain scope clearing, while current gateway code and tests confirm device-less trusted-proxy/shared-auth scopes are cleared outside explicit preservation cases. The provided PR context also shows review feedback was addressed rather than leaving an obsolete or low-signal patch.

## Best Possible Solution

Keep the PR open and either merge it after maintainer review or land an equivalent docs update in `docs/gateway/trusted-proxy-auth.md` and `docs/gateway/protocol.md` that explicitly separates HTTP `x-openclaw-scopes` behavior from WebSocket handshake scopes, documents scope clearing without device identity, keeps `dangerouslyDisableDeviceAuth` framed as break-glass, and includes the local `allowInsecureAuth` exception.
