## Summary

Keep #69513 open. Current main still validates cron announce delivery channels but not configured delivery account IDs at `cron.add`/`cron.update`, while the CLI and normalization paths preserve any non-empty `--account` value into `delivery.accountId`. The linked OpenClaw PR #69518 directly addressed this but is closed and unmerged in the provided GitHub context, so it cannot be treated as shipped behavior.

## Best Possible Solution

Land a focused core fix in `src/gateway/server-methods/cron.ts` that validates `delivery.accountId` and announce-mode `delivery.failureDestination.accountId` against the selected channel plugin/configured account IDs during `cron.add` and `cron.update`, while preserving flat single-account setups and omitted accountId behavior. Add regression coverage in `src/gateway/server.cron.test.ts` for reject/accept cases and update user-facing docs or changelog if the behavior change ships.
