## Summary

Keep #69513 open. Current main still lacks create/update validation for cron announce delivery accountIds: the CLI forwards `--account`, normalization only trims it, gateway validation checks the channel but not the account, and delivery resolution gives the explicit job accountId highest precedence. The linked repo PR #69518 was closed unmerged, so there is no shipped canonical fix on main.

## Best Possible Solution

Keep the issue open and implement central gateway validation in `src/gateway/server-methods/cron.ts` for both `cron.add` and `cron.update`, checking `delivery.accountId` and announce `delivery.failureDestination.accountId` against configured accounts for the selected channel. Add focused coverage in `src/gateway/server-methods/cron.validation.test.ts` or `src/gateway/server.cron.test.ts` for rejecting an unknown account and accepting a configured one, while preserving omitted accountId/default-account behavior.
