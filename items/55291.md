## Summary

Keep PR #55291 open. Current main still saves media before hostname/port/server setup and only cleans up in the “server required but not allowed to start” branch, so the later failure paths described by the PR can still leave saved media behind. The PR discussion and bot review match a real code gap rather than obsolete or duplicate work.

## Best Possible Solution

Keep the PR open and have a maintainer either rebase/land the shared post-save cleanup path with the targeted regression tests, or apply the same fix directly on main. The implementation should keep `saveMediaSource()` outside the catch, wrap only the later hosting setup, remove `saved.path` best-effort on any post-save exception, preserve the original error, and verify via the repo’s `pnpm test src/media/host.test.ts` lane when dependencies are available.
