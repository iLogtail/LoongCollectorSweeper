## Summary

Current main does not implement the requested wrapper support. The gateway install path still builds LaunchAgent ProgramArguments from Node/Bun plus the CLI entrypoint, the install CLI exposes no --wrapper option, reinstall preservation skips OPENCLAW_* service env keys, and doctor has no wrapper-aware signal. PR #69425 is a relevant open implementation attempt for the env-var portion, but it is not merged on main and does not cover the full CLI/doctor request.

## Best Possible Solution

Keep #69400 open until a maintainer lands or declines the core installer behavior. The likely implementation belongs in the daemon install plan/program-args path with focused tests, plus an explicit decision on whether to include --wrapper persistence and a doctor/status signal or split those into narrower follow-up issues after PR #69425 is reviewed.
