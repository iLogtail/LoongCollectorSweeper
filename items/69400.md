## Summary

Current main does not honor OPENCLAW_WRAPPER or expose --wrapper for gateway/service install. The linked PR #69425 is the right active implementation path, but it is open, unmerged, non-mergeable, and has a review finding that its env lookup would not preserve the wrapper across reinstalls. Keep the issue open until that PR or a successor lands.

## Best Possible Solution

Keep #69400 open and use #69425 as the active implementation thread. The fix should make resolveCliProgramArguments honor a validated wrapper from the install plan, explicitly persist or preserve OPENCLAW_WRAPPER (or a --wrapper flag) in the service environment across reinstall/update, add regression coverage for the sticky LaunchAgent case, and optionally add the requested status/doctor visibility once the core behavior is correct.
