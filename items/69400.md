## Summary

Keep #69400 open. Current main still does not honor OPENCLAW_WRAPPER or provide a --wrapper install option, and the only directly related implementation path is open PR #69425, which is not merged or shipped. The issue is recent, author is not protected, labels are not protected, and it describes a concrete daemon/LaunchAgent behavior gap rather than a stale or external-only request.

## Best Possible Solution

Keep #69400 open and let maintainers review, land, or supersede #69425. The best implementation path is to add validated OPENCLAW_WRAPPER support in the daemon ProgramArguments resolver, wire a --wrapper option through gateway/service install if maintainers want CLI persistence, preserve or deliberately manage that env key in the service environment, add a doctor/status signal, and cover the behavior with focused daemon install/program-args tests and docs.
