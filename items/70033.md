## Summary

Keep open. Current main has related malformed tool-call argument repair, but it is only enabled for Kimi Anthropic-compatible streams and `openai-completions`; the reported Amazon Bedrock `bedrock-converse-stream` path is not covered, and the Bedrock plugin wrapper does not add an equivalent empty/truncated-arguments guard.

## Best Possible Solution

Keep this issue open. The best path is a targeted Bedrock/converse-stream regression for a large `write`/`exec` tool call whose streamed argument JSON ends up `{}`, then either extend the existing malformed argument repair to `bedrock-converse-stream` when safe or surface a clear one-shot retry/chunking error before repeated validation loops occur.
