## Summary

Keep #70033 open. The report is recent, Bedrock-specific, and current main still lacks a Bedrock live-stream guard, retry, chunking path, or malformed-argument repair for large `write`/`exec` tool inputs. Adjacent empty-argument fixes cover Kimi/OpenAI-compatible streams, not `amazon-bedrock` on `bedrock-converse-stream`.

## Best Possible Solution

Keep the issue open and implement a Bedrock-specific runtime fix: simulate Bedrock `contentBlockStart` plus missing/truncated `delta.toolUse.input` for large `write`/`exec` arguments, then add a guard that detects final empty required-argument tool calls on `bedrock-converse-stream` and either performs a bounded retry with a clearer error or prevents repeated execution attempts. If automatic large-content handling is desired, design it separately as chunked file writes or temp-file guidance rather than treating the existing OpenAI/Kimi repair as sufficient.
