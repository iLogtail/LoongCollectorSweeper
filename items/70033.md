## Summary

Keep #70033 open. The report is recent and concrete for Amazon Bedrock `bedrock-converse-stream` with large `write`/`exec` arguments. Current main has malformed streamed tool-call argument repair, but that repair is gated to Kimi `anthropic-messages` and `openai-completions`, not the Bedrock Converse path reported here. The related empty-argument closes are useful context, but they do not supersede this Bedrock-specific issue.

## Best Possible Solution

Keep the issue open and fix it in the Bedrock/embedded-runner streaming path. A maintainer-quality fix should either safely extend malformed tool-call argument repair to `amazon-bedrock`/`bedrock-converse-stream`, or add Bedrock-specific detection for empty required arguments that stops the retry loop with a clear provider/model error. Add focused regression coverage for a Bedrock Converse stream where large `write.content` or `exec.command` deltas currently leave the tool call with `{}`.
