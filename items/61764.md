## Summary

Keep open. Current main still contains the concrete Channel dashboard perf paths called out in the issue: inline pretty JSON rendering, per-render channel sorting/meta-map work, repeated activity/timestamp calculations, and full per-channel config schema rendering. The related items in the provided context address other channel/status/dashboard problems, but none supersede this dashboard overhaul.

## Best Possible Solution

Keep this issue open and split or implement it in focused Control UI work: first land the low-risk perf fixes in `ui/src/ui/views/channels.ts` and `channels.shared.ts` by deferring raw snapshot stringify, hoisting channel/account indexes, avoiding repeated recent-activity work, and memoizing stable order/meta derivations; then address `channels.config.ts` and `config-form.node.ts` with schema-analysis memoization plus true lazy rendering for collapsed config sections; finally do the UX pass with channel search/filtering, clearer status/actions, developer-mode disclosure for debug surfaces, and focused regression/perf tests around multi-account channel snapshots.
