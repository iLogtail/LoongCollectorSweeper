## Summary

Keep open. Current main has strong partial coverage for OpenClaw-controlled gateway restarts: a durable restart sentinel, post-startup wake, routed restart notice, and optional continuation agent turn. But #42816 asks for broader task-context persistence and proactive verification across system reboots or arbitrary commands that kill the agent process. Current source/docs still show process sessions and supervisor-owned active runs are not recovered after process restart, so a meaningful part of the requested capability remains missing.
