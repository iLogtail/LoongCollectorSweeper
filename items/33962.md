## Summary

Keep open. Current main still routes session-memory slug generation through the default agent's effective primary model and the normal embedded runner lanes. The only mitigation present is the general `agents.defaults.timeoutSeconds` fallback; there is no slug-specific lightweight model config, fast heuristic timeout/fallback, or debounce/concurrency change covering the requested behavior.

## Best Possible Solution

Keep #33962 open until maintainers either add a dedicated slug-generation model surface, such as `agents.defaults.slugGenerator.model` with schema/help/docs/tests, or explicitly choose a different product direction. The likely implementation belongs around `src/hooks/llm-slug-generator.ts`, `src/hooks/bundled/session-memory/handler.ts`, agent-default config schema/help/generated docs, and regression tests for model selection, timeout/fallback behavior, queue behavior, and the existing `llmSlug` disable path.
