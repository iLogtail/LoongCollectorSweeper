## Summary

Keep #33413 open. Current main still exposes the tool-start signal and Slack has a thread-status setter, but Slack dispatch does not map tool-start events to `assistant.threads.setStatus`; it only sets the static `"is typing..."` status at reply start and clears it at stop. The linked PR #33420 was closed unmerged, and the related #67129 customization request is adjacent rather than a canonical replacement.

## Best Possible Solution

Keep #33413 open until Slack dispatch maps tool-start events to `ctx.setSlackThreadStatus` for assistant thread targets, with focused tests that simulate `onToolStart` and assert an additional Slack `assistant.threads.setStatus` call for a tool label. If maintainers decide draft-preview progress replaces assistant status labels, close only with an explicit supersession comment linking the chosen behavior or tracking issue.
