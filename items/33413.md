## Summary

Keep open. Current main still does not update Slack `assistant.threads.setStatus` with per-tool labels. The Slack plugin has the core `onToolStart` signal, but current Slack handling routes tool activity to status reactions and draft-preview progress, while assistant thread status remains the static start/clear `"is typing..."` behavior.

## Best Possible Solution

Keep #33413 open until the Slack plugin maps `onToolStart` events to assistant thread status updates for thread targets, ideally with a small tool-name-to-label helper and a focused Slack monitor test that simulates `onToolStart` and asserts an additional `assistant.threads.setStatus` call. If maintainers decide the newer draft-preview progress UX intentionally replaces assistant thread status updates, close only after an explicit product decision and link that decision to `channels.slack.streaming.preview.toolProgress` and the related #67129 customization discussion.
