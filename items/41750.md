## Summary

Keep open. Current `main` still lacks the recovery the issue asks for: when a loopback Chrome process is holding the CDP port but OpenClaw does not own it in current runtime state, the browser availability path errors and tells the user to run `reset-profile`, but neither `reset-profile` nor idle stop logic actually terminates that external/stale Chrome process. The latest shipped release `v2026.4.22` has the same behavior.

## Evidence

- **Loopback stale-port path still errors instead of reclaiming the process:** When HTTP responds but the CDP websocket is unhealthy and `profileState.running` is absent, `ensureBrowserAvailable()` throws `Port <port> is in use ... Run action=reset-profile ... to kill the process` rather than probing ownership and killing/restarting the stale browser automatically.
  - file: [extensions/browser/src/browser/server-context.availability.ts:285](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/browser/src/browser/server-context.availability.ts#L285)
  - command: `nl -ba extensions/browser/src/browser/server-context.availability.ts | sed -n '250,310p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **`reset-profile` does not kill a non-owned process:** The reset path comment says `Port in use but not by us - kill it`, but the implementation only calls `closePlaywrightBrowserConnectionForProfile(profile.cdpUrl)` and never sends a signal or inspects/kills the OS process holding the CDP port.
  - file: [extensions/browser/src/browser/server-context.reset.ts:39](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/browser/src/browser/server-context.reset.ts#L39)

  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Gateway/browser shutdown only stops tracked children from current runtime:** `stopKnownBrowserProfiles()` stops `runtime.running` children, otherwise it delegates to `stopRunningBrowser()`. For local managed profiles with no tracked `running` child, `stopRunningBrowser()` returns without killing any process, so a stale Chrome that survived a restart is not reclaimed.
  - file: [extensions/browser/src/browser/server-lifecycle.ts:31](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/browser/src/browser/server-lifecycle.ts#L31)
