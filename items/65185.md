## Summary

Keep open. Current main does not implement the PR's mixed-auth precedence fix, and the iOS manual TLS behavior is still the old non-loopback-forces-TLS path. The PR also cannot land unchanged: review comments identify compile-breaking tests, and its blanket LAN/Tailscale plaintext behavior needs to be reconciled with current strict mobile gateway security policy.

## Best Possible Solution

Keep this PR open as useful but require a rework before maintainer action: split or preserve the password-over-bootstrap precedence fix with regression coverage across the relevant Swift/TypeScript client paths, align iOS plaintext handling with the current mobile gateway security policy instead of blanket Tailscale `ws://`, and fix the Swift tests so they compile and assert the intended product boundary.
