## Summary

Keep #55576 open. Current main has not implemented the PR's config-schema change: `google-vertex` is still absent from `MODEL_APIS` and both generated provider/model `api` enum lists, so the reported validation gap remains. The PR is plausible useful work, but the existing review concern is still valid: Google runtime detection still excludes `google-vertex`, while transcript policy depends on that helper for Gemini/Google sanitation and turn-order safeguards.

## Best Possible Solution

Keep the PR open or ask for a follow-up revision. The best implementation would add `google-vertex` to `MODEL_APIS` and regenerated schema only together with the runtime Google-detection update and focused regression coverage proving that `api: "google-vertex"` validates and receives the same Google/Gemini transcript safeguards as the supported Google transports.
