## Summary

Keep open. Current `main` at `dc7c7034254971fa6e9769e99a18da88f432f8c7` still has the reported bug: `normalizeExtraMemoryPaths()` does not expand `~/...` before resolving relative paths, so tilde-prefixed `memorySearch.extraPaths` still resolve under the workspace instead of the user home directory.

## Evidence

- **Current implementation still resolves non-absolute paths against the workspace:** `normalizeExtraMemoryPaths()` trims entries and then does `path.isAbsolute(value) ? path.resolve(value) : path.resolve(workspaceDir, value)`. Because `~/...` is not absolute on Node, this code path still mis-resolves tilde-prefixed entries on current `main`.
  - file: [src/memory-host-sdk/host/internal.ts:70](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/memory-host-sdk/host/internal.ts#L70)
  - command: `nl -ba src/memory-host-sdk/host/internal.ts | sed -n '65,82p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Existing home-expansion helper is available but not used here:** `expandHomePrefix()` already exists in core infra and expands `~` to the effective home directory, which matches the reporter's proposed fix direction.
  - file: [src/infra/home-dir.ts:82](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/infra/home-dir.ts#L82)
