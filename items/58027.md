## Summary

Keep open. Current `main` still resolves `agents.defaults.memorySearch.extraPaths` with `path.isAbsolute()` only, so `~/...` remains workspace-relative instead of expanding to the user home directory.

## Evidence

- **Current normalization logic still lacks home expansion:** `normalizeExtraMemoryPaths` trims and resolves each entry with `path.isAbsolute(value) ? path.resolve(value) : path.resolve(workspaceDir, value)`. There is no `expandHomePrefix` call or `~` handling, so `~/OneDrive/MyVault` is treated as relative to the workspace.
