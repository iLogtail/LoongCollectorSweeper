## Summary

Keep open. Current main still contains the raw default-model status behavior this PR targets, and the PR discussion’s review-noted gaps are still visible in the current code paths. The branch is stale and bundled, but the central `openclaw status` / `models status --json` bug is not implemented well enough on main to close conservatively.

## Best Possible Solution

Keep the item open, but narrow the implementation: validate raw default model refs against configured providers before using them for `models status --json`, rich `models status`, probe candidates, and the separate `openclaw status` summary resolver. Add focused regression tests for a removed provider falling back to the resolved configured provider/model, then rebase or replace this bundled PR with a status-only fix.
