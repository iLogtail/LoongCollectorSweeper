## Summary

Keep PR #39646 open. Current main still exposes the stale raw default-model path that the PR targets in both `models status` and `openclaw status`, while the PR itself remains too broad and incomplete to merge as-is.

## Best Possible Solution

Keep this open or replace it with a narrower maintainer patch. Centralize effective default-model resolution for `openclaw status`, `models status --json`, rich status output, and auth-probe candidates; validate removed configured providers without rejecting CLI-backed or plugin/synthetic providers; add regression coverage for the removed-provider case; split the unrelated PR changes into separate items.
