## Summary

Keep issue #49909 open. Current main still routes all ClawDock commands through `_clawdock_compose()`, which explicitly passes only `docker-compose.yml` and optional `docker-compose.extra.yml`; there is no `docker-compose.override.yml` handling, and no docs/tests/history evidence of a shipped fix or canonical duplicate.

## Best Possible Solution

Add explicit, ordered `docker-compose.override.yml` handling in `scripts/clawdock/clawdock-helpers.sh` after the base file and before/after project-specific overlays according to the intended precedence, then document the supported ClawDock compose overlay order and add a focused shell-helper regression test or static command-construction test so `clawdock-start` and stop/start paths cannot silently drop user overrides again.
