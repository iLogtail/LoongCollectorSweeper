## Summary

Keep open. Current main appears to have fixed the raw Codex JSON leak by sanitizing provider error payloads before chat, but it still does not implement the issue’s requested behavior that failed cron runs with announce delivery should not post to the user-facing channel. Related PR #46236 proposed a cron delivery guard, but it was closed unmerged and that guard is not present on main.
