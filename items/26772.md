## Summary

Keep #26772 open. Current main still lacks durable checkpoint/resume for interrupted full memory indexing: the safe path builds a fresh temp SQLite database, swaps it only after the full build succeeds, removes temp files on errors, and has no checkpoint schema/config/test coverage. The related #70567 is adjacent and open, but it tracks silent partial success after transient embedding failures, not process-crash resume.

## Best Possible Solution

Keep the issue open until memory-core has a durable full-reindex checkpoint/resume design or a maintainer explicitly chooses a different direction. A good fix would persist indexing generation/checkpoint state, resume or safely discard incomplete work on restart, prevent search from serving incomplete generations, update memory status/config/docs if new controls are added, and add regression coverage for interrupted full reindex behavior.
