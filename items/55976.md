## Summary

Current `main` no longer exposes a top-level `buttons` parameter in the `message` tool schema. `buttons` is only an optional nested field inside `presentation.blocks[]`, and the schema builder only requires `action`, so the reported validation failure is already fixed on `main` and in the latest tagged release checked here.
