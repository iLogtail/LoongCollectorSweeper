## Summary

Keep open. Current `main` still uses the same status path that can report plugin-backed memory as unavailable when the live gateway runtime is healthy: the local status scan nulls out memory if `getActiveMemorySearchManager({ purpose: "status" })` cannot materialize a manager, and the gateway memory probe still reduces readiness to `payload.embedding.ok` rather than a separate runtime-health signal.

## Evidence

- **Local memory runtime still falls back to config-only bootstrap:** `ensureMemoryRuntime()` only reuses the in-process runtime from `getMemoryRuntime()` or bootstraps from config via `resolveRuntimePluginRegistry(...)`; it does not pull live gateway runtime state into the status path.
