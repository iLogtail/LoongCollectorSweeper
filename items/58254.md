## Summary

Keep open. Current `main` still contains the macOS launchd restart paths that re-bootstrap without a preceding `bootout`, and the tests explicitly codify that no-`bootout` behavior. I did not find source evidence that issue #58254's launchd throttle-race fix landed on `main`.

## Evidence

- **Auto-update still restarts a loaded gateway through the restart helper:** `openclaw update` prepares the restart helper when the gateway service is loaded, so the reported auto-update outage still depends on the macOS helper path.
  - file: [src/cli/update-cli/update-command.ts:1387](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/cli/update-cli/update-command.ts#L1387)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '1340,1425p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **macOS update restart helper still retries with enable/bootstrap/kickstart and no bootout:** The Darwin restart script does `launchctl kickstart -k`, then `launchctl enable`, `launchctl bootstrap`, and `launchctl kickstart -k` again. There is no `launchctl bootout` before re-bootstrap.
  - file: [src/cli/update-cli/restart-helper.ts:119](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/cli/update-cli/restart-helper.ts#L119)
  - command: `nl -ba src/cli/update-cli/restart-helper.ts | sed -n '90,140p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Detached launchd handoff still re-bootstraps without clearing registration/throttle state first:** Both `kickstart` and `start-after-exit` handoff scripts call `launchctl bootstrap` directly on failure paths. Neither branch performs a `bootout` before the re-bootstrap that the issue says is needed to clear the throttled state.
  - file: [src/daemon/launchd-restart-handoff.ts:96](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/daemon/launchd-restart-handoff.ts#L96)

  - file: [src/daemon/launchd.test.ts:703](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/daemon/launchd.test.ts#L703)
