## Summary

Keep open: current main still contains the macOS launchd refresh/restart behavior that can hit the reported throttle race, and the relevant code paths are unchanged from v2026.4.23.

## Evidence

- **LaunchAgent refresh still bootouts and bootstraps during reinstall:** `activateLaunchAgent` still runs `launchctl bootout`, then `launchctl unload`, then re-registers with `bootstrap` during `openclaw gateway install --force`, which matches the update-time service refresh sequence described in the issue.

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Generated LaunchAgent still starts immediately under launchd:** The plist generator still emits `RunAtLoad`, `KeepAlive`, and `ThrottleInterval`, so a refreshed LaunchAgent is still started immediately and remains subject to launchd throttle timing.
  - file: [src/daemon/launchd-plist.ts:116](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/daemon/launchd-plist.ts#L116)
