## Summary

Keep open. Current `main` still treats ACPX `permissionMode` and `nonInteractivePermissions` as global plugin config, while per-agent and per-binding ACP overrides only cover `agent`, `backend`, `mode`, `cwd`, and binding `label`. The request is recent, specific, and not implemented on `main` at `f00d65a30496335fb068f01fe529dbba484d4312`.

## Evidence

- **Agent runtime ACP schema lacks permission overrides:** `AgentRuntimeAcpConfig` exposes only `agent`, `backend`, `mode`, and `cwd`; there is no `permissionMode` or `nonInteractivePermissions` field for `agents.list[].runtime.acp`.
  - file: [src/config/types.agents.ts:17](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/config/types.agents.ts#L17)

  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **Binding ACP schema lacks permission overrides:** `bindings[].acp` is limited to `mode`, `label`, `cwd`, and `backend`. The Zod schema rejects any additional ACP binding keys because the object is `.strict()`.
  - file: [src/config/zod-schema.agents.ts:47](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/config/zod-schema.agents.ts#L47)

- **Binding resolution only propagates mode/cwd/backend/label:** Configured ACP binding resolution reads agent defaults for `agent/runtime.acp.{agent,mode,cwd,backend}`, merges binding overrides for `mode/cwd/backend/label`, and stores only those values in the binding spec/metadata. No per-binding or per-agent permission fields are carried through session creation.
