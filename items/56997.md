## Summary

Keep #56997 open. Current main can list and manually switch to channel-backed sessions in the Control UI, but it still does not implement the requested channel-based separated/merged session view toggle or unified multi-channel timeline.

## Best Possible Solution

Keep the issue open and implement a Control UI session monitoring mode: use existing `GatewaySessionRow.channel` metadata for separated channel/session grouping, add a deliberate merged timeline path that can fetch or subscribe to multiple session histories, and add UI plus gateway/controller tests proving cross-session ordering and `session.message` updates across active and non-active sessions.

## Risks / Open Questions

- A merged timeline needs careful timestamp ordering, session labels/channel badges, and authorization behavior across multiple sessions.
- The existing per-session selector partially addresses viewing other channels, so implementation should avoid duplicating that surface and instead add an explicit separated/merged mode where it improves monitoring.
