## Summary

Keep open. Current `main` still lacks the WhatsApp symlink-rejection and auth-directory hardening this PR proposes, but the implementation surface has moved from the PR's old `src/web/*` paths into `extensions/whatsapp/src/*`. This is still a plausible security-hardening change, so it should be rebased/reviewed rather than auto-closed.
