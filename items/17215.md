## Summary

Keep #17215 open. Current main still has adjacent update safety pieces, but it does not implement the requested native pre/post-update compatibility scanner, risk-scored JSON/human report, smoke recommendations, rollback hints, or `doctor --upgrade-scan`-style surface. The existing maintainer Codex review comment from 2026-04-26 reached the same conclusion, and the current source/docs/history still support it.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best product path is a native upgrade scan under `openclaw update` and/or `openclaw doctor` that emits human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and tests for the report contract.
