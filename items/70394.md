## Summary

Keep open. Current main still uses strict plugin-load failure behavior on the exact loaders this PR targets, and tests still assert plugin-local invalid config can throw during startup. The bug is real, but this PR should not be cleanup-closed as already implemented; it also should not be merged as-is because the branch bundles unrelated changes and review comments identify regressions in the proposed approach.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a focused replacement patch: handle `plugins.entries.<id>.config` validation failures as per-plugin disabled/error diagnostics where safe, preserve fail-closed behavior for required runtime/security-critical plugin failures, avoid caching transient failed registries, and add focused loader/startup/doctor coverage. Split the unrelated QQBot, Vertex, and media-delivery fixes into separate PRs or rely on current-main equivalents where already present.
