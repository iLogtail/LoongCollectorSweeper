## Summary

Keep open. Current main has not implemented the PR's headline change: the metadata and runtime plugin registry loaders still force strict plugin-load failure behavior. The PR also should not be landed as-is because it bundles unrelated fixes and the supplied review discussion identifies plausible regressions in media delivery and failed-registry caching/activation.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a focused plugin-only fix. The best path is to split unrelated changes out, preserve fail-closed behavior for full production runtime loads, make validate/metadata/CLI diagnostic paths report plugin config errors without process-wide crashes, avoid caching failed full-runtime registries, and add regression coverage for gateway startup plus doctor/plugin status reporting for disabled/error plugins.
