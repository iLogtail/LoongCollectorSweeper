## Summary

Keep open. Current main does not implement `openclaw models status --probe --all`: the CLI has no status-level `--all` flag, the routed args do not parse one, and the probe planner still selects one candidate model per provider. The linked open PR #63259 is the relevant implementation candidate, but it is unmerged and should be reviewed or superseded rather than using this issue for cleanup closure.

## Best Possible Solution

Keep this issue open and review #63259 or a successor PR. The implementation should add a `models status --probe --all` flag to both Commander and routed CLI paths, thread it through `modelsStatusCommand` and `runAuthProbes`, expand probe targets across every configured primary/fallback/image candidate while preserving provider/profile filters, and add focused target-builder plus CLI/JSON docs coverage before closing as implemented.
