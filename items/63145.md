## Summary

Keep open. Current `main` at `cf858258c71701ef62dddf88aa777873a4962046` does not implement `openclaw models status --probe --all`: the CLI parser has no `--all` support for `models status`, the docs only describe provider/profile auth probing, and the probe planner still chooses one representative model per provider. There is also a still-open linked PR (#63259) that implements this exact feature, so conservative cleanup should not close the issue.

## Evidence
