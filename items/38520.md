## Summary

Keep #38520 open. Current main has adjacent compaction safeguards: silent pre-compaction memory flush, optional start/end user notices, post-compaction context reinjection, UI-only context pressure notices, context-engine/plugin observation hooks, and context-engine-owned compaction. It still does not implement the central requested lifecycle: configurable pre-compaction warning events, a structured handoff window/timeout, or an agent-callable bounded compaction deferral mechanism.

## Best Possible Solution

Keep this open for maintainer product/API work. A good implementation would extend the core compaction lifecycle with validated config for warning thresholds and handoff policy, define where warning events are emitted to agents/channels/UI, and add a bounded deferral contract if maintainers want agents or plugins to pause imminent compaction. It should update `src/config/*`, the compaction runner/event path, plugin or tool contracts as needed, public docs, and focused tests covering warnings, handoff timeout behavior, and deferral limits.
