## Summary

Keep open. Current `main` still does not implement any of the requested behavior changes: Tailscale Serve is still loopback-only, `gateway.auth.allowTailscale` still applies only to the Control UI/WebSocket auth path, and there is still no dual-bind support. BlueBubbles webhooks are plugin-owned/password-authenticated and separate from gateway shared-secret auth, so the issue body also appears to mix two different auth models. That makes the report at least partly confused, but not strong enough to close without a live repro of the reported Serve WebSocket drop.

## Evidence

- **Serve is still loopback-only by design:** The Tailscale docs still say Serve keeps the gateway on `127.0.0.1`, and config validation still rejects any non-loopback bind when `gateway.tailscale.mode` is `serve` or `funnel`. That means the suggested `loopback,tailnet` style support is not implemented on current `main`.

- I did not run a live repro of the reported 'WebSocket drops after ~5 seconds' symptom because this review was read-only; that part of the report could still reflect a real Tailscale or third-party client behavior outside what code inspection alone can settle.
