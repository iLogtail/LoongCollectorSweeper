## Summary

Current `main` still appears to expose heartbeat user prompts in Control UI chat history. The gateway `chat.history` sanitization only drops assistant-side silent/commentary messages, and the Control UI history loader only filters assistant `NO_REPLY`/synthetic tool-result entries, not heartbeat-shaped user messages.

## Evidence

- **Gateway history filter is assistant-only:** `shouldDropAssistantHistoryMessage` immediately returns unless `entry.role === "assistant"`, and `sanitizeChatHistoryMessages` only applies that assistant-only drop path before returning chat history.
  - file: [src/gateway/server-methods/chat.ts:1213](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-methods/chat.ts#L1213)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '1180,1265p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **chat.history uses that sanitizer directly:** The `chat.history` handler reads raw session messages, sanitizes them with `sanitizeChatHistoryMessages(...)`, and returns them. There is no heartbeat-specific filtering step in this path.
  - file: [src/gateway/server-methods/chat.ts:1874](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-methods/chat.ts#L1874)
