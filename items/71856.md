## Summary

Keep open. Current main does not implement a TUI startup recent-session summary, and the PR is a focused implementation attempt for the same core TUI behavior requested by #71719. It is not conservative cleanup material, though it still needs maintainer review/changes because the proposed onConnected hook would run on reconnects and needs direct test coverage.

## Best Possible Solution

Keep the PR open and ask for targeted changes: make the startup summary a first-connection-only TUI behavior, keep it best-effort and non-blocking, add focused TUI tests around the formatter and connection/reconnection path, and avoid Gateway/core changes because the existing sessions.list contract already supplies derived titles and last-message previews.
