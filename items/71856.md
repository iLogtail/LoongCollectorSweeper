## Summary

Keep open. Current main still does not implement a TUI startup recent-session summary: the connection flow refreshes agents, updates the header, loads history, and sets connected status, while history loading rebuilds the transcript without a distinct recap block. The linked issue #71719 was closed as superseded by this PR, so #71856 is currently the canonical implementation discussion rather than cleanup material. The PR still needs maintainer review because its proposed startup summary runs on every non-local connection/reconnection and needs focused regression coverage.

## Best Possible Solution

Keep PR #71856 open as the canonical implementation discussion for #71719. Revise the TUI startup-summary implementation in `src/tui/tui.ts`, gate it to initial connection or otherwise avoid duplicate reconnect messages, clarify current-session versus previous-session semantics, and add focused tests around formatting and connection-flow behavior using the existing TUI backend/listSessions seam.
