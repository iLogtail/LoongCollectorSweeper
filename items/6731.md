## Summary

Keep #6731 open. Current main has advisory safety prompt text, opt-in sandboxing, configurable exec approvals, allowlists, strict inline-eval handling, and security audit warnings, but it still intentionally permits no-approval host exec by default for trusted gateway/node operation and explicitly documents that YOLO mode has no separate heuristic dangerous-command gate. The remaining request is a core safety/product decision, reinforced by the Apr 26 maintainer-posted review already keeping this issue open.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy, or explicitly decide that the current opt-in model of exec approvals, sandboxing, tool policy, audit warnings, and docs is the final intended behavior. A concrete path should cover default safe mode, dangerous-command classification, approval UX, sandbox interaction, Windows PowerShell cases such as Remove-Item, and regression tests/docs.
