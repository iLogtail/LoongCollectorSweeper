## Summary

Keep open. Current `main` still has the same capability-lookup mismatch the report describes: provider-prefixed refs are canonical elsewhere, but `resolveGatewayModelSupportsImages` still exact-matches `entry.id === params.model`, so a `qwen/qwen3.6-plus` lookup can still miss the `qwen3.6-plus` catalog row and force image offload/drop behavior. The latest release tag `v2026.4.23` still ships that matcher, and the current tests do not cover this qwen-style prefixed-ref case.
