## Summary

Keep open. Current `main` and the latest release still ship `docker-compose.yml` without a `build:` stanza for `openclaw-gateway`, and the current Docker docs/setup script still require a separate image build before `docker compose up`. This PR addresses a real remaining gap rather than something already implemented or obsolete.

## Evidence

- **Current main compose still lacks a build context:** At `fe930b987e33f4e1acd93b5ef1ea76c8b7260047`, `docker-compose.yml` defines `image: ${OPENCLAW_IMAGE:-openclaw:local}` for `openclaw-gateway` and `openclaw-cli`, but there is no `build:` key on the gateway service.
