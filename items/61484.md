## Summary

Keep #61484 open. Current main has media support in adjacent send/chat surfaces, but the `sessions_send` agent tool itself still only accepts text and the A2A announce path only sends a text message unless another layer/model emits media directives manually.

## Best Possible Solution

Keep the issue open and implement this as an additive core session-tool feature: extend `sessions_send` with media/mediaUrls/attachments params, reuse existing message-action local media normalization and chat attachment handling, pass media through the target `agent`/`chat.send` run, and update the A2A reply/announce path to call gateway `send` with `mediaUrl`/`mediaUrls` so channel `sendMedia` adapters can use native platform delivery. Add focused tests in `src/agents/openclaw-tools.sessions.test.ts`, `src/gateway/server.sessions-send.test.ts`, and the relevant outbound media tests, plus docs updates for `/concepts/session-tool`.
