## Summary

Keep #58557 open. The docs need is plausible, but current main does not already cover cron language-output troubleshooting, and the PR still needs a narrower, source-backed rewrite before it is safe to merge. The current cron API exposes prompt text through `message`; it does not expose a cron `language` config key, prompt-template file option, or `{{language}}` substitution surface.

## Best Possible Solution

Keep the PR open for revision. The best path is a narrow English-only docs change under `docs/automation/cron-jobs.md` that recommends explicit language instructions in `--message` and concrete prompt wording, while removing references to cron language config, template files, or `{{language}}` substitution unless a real cron language/template API is first added to `src/cron/types.ts`, the Gateway protocol schema, CLI add/edit commands, tests, and docs. Do not add localized docs in this repo.
