## Summary

Keep open. Current main still hard-blocks `qwen3.6-plus` on Qwen Coding Plan endpoints in the bundled qwen plugin, and docs/tests still describe that behavior as expected. The linked fix PR #66367 is useful context, but it is represented as open/unmerged in the provided GitHub context, so this issue is not implemented or safely superseded on `main`.

## Best Possible Solution

Keep the issue open and land a validated qwen plugin fix: remove the blanket Coding Plan exclusion from the catalog, config normalization, and runtime suppression; let Alibaba/Qwen return plan-specific unsupported-model errors when needed; update qwen provider tests and docs to match the new provider-truth behavior. PR #66367 is relevant prior work, but maintainers should require current validation before landing.
