## Summary

Keep open. Current `main` still emits WhatsApp pairing/access-not-configured outbound replies for unauthorized inbound DMs, and there is no WhatsApp transport-level read-only enforcement covering those sends. The issue also already has an active linked PR (#71382), so conservative maintainer cleanup should not close it.
