## Summary

Keep open. Current `main` still has the ordering path that can delay a Discord final reply behind earlier tool-summary sends in the default config. The linked fix PRs for this issue were both closed without merging, and the only mitigation now on `main` is preview-tool-progress suppression when Discord preview streaming is enabled; docs still say Discord preview streaming defaults to off.

## Evidence

- **Dispatcher still serializes tool before final:** `createReplyDispatcher` uses one `sendChain` for all reply kinds and explicitly serializes tool/block/final order, so an earlier tool summary still blocks a later final reply.
  - file: [src/auto-reply/reply/reply-dispatcher.ts:151](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/reply/reply-dispatcher.ts#L151)

- **Direct/forum turns still send tool summaries:** `dispatch-from-config` sets `shouldSendToolSummaries` true for non-group turns, and `resolveToolDeliveryPayload` returns the tool payload unchanged in that case; tool results are then enqueued through `dispatcher.sendToolResult(...)`.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:646](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/reply/dispatch-from-config.ts#L646)
