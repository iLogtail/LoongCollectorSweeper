## Summary

Keep open. Current `main` does not implement ACP support in the `subagents` tool, and the repo now documents a deliberate split: `subagents` is for native OpenClaw sub-agents, while ACP sessions are managed through `sessions_spawn`/`sessions_send` and `/acp` controls. That means this PR is neither already implemented on `main` nor safely closable under another allowed reason.

## Evidence

- **Current subagents tool is subagent-only:** `src/agents/tools/subagents-tool.ts:33-176` still scopes the tool to spawned sub-agents only. It lists `listControlledSubagentRuns(...)`, kills via `killAllControlledSubagentRuns(...)` / `killControlledSubagentRun(...)`, and steers via `steerControlledSubagentRun(...)`; there is no ACP path or ACP manager/session-meta usage. The tool description at `src/agents/tools/subagents-tool.ts:37-38` explicitly says "spawned sub-agents for this requester session."
