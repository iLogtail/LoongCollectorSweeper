## Summary

Keep open. Current main still does not implement the requested Telegram mode: observe or ingest all group messages so non-allowlisted users can be identified later, while only allowlisted users can trigger replies. The existing Telegram `ingest` path only handles mention-skipped messages after access gates pass; `groupPolicy: "open"` lets any mentioned sender trigger, while `groupPolicy: "allowlist"` drops unauthorized senders before ingest/context handling.

## Best Possible Solution

Keep this issue open for Telegram plugin product/API work. A good implementation would split observation/ingest authorization from reply-trigger authorization, for example an explicit observe-only or trigger-allowlist policy that lets Telegram receive and optionally ingest non-triggering group messages while only configured senders can start agent turns. Update Telegram/group docs and add tests for unauthorized sender ingest without reply plus allowlisted sender reply.
