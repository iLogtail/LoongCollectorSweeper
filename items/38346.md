## Summary

Current main still does not provide the requested Telegram mode: receive/ingest all allowed-group messages so non-allowlisted users can be identified later, while only an allowlisted subset can trigger replies. Telegram currently ties sender authorization to `groupPolicy: "allowlist"`, which drops unauthorized senders before inbound processing and mention-skip ingest; `groupPolicy: "open"` allows non-allowlisted senders to trigger when mention gating passes.

## Best Possible Solution

Keep #38346 open for bundled Telegram plugin work. The best implementation is to split observation/ingest authorization from reply-trigger authorization: allow configured groups to receive and optionally ingest non-triggering messages, then apply a sender trigger allowlist before starting agent turns or replies. Update Telegram/group docs, config schema/types, and tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
