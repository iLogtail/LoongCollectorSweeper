## Summary

Keep open. Current main has adjacent Telegram group controls, but it does not implement the requested mode of observing all group messages and learning non-allowlisted senders while allowing only configured users to trigger the bot. `groupPolicy: "open"` bypasses sender allowlists, while `groupPolicy: "allowlist"` blocks unauthorized senders before the no-mention context/ingest path can run.

## Evidence

That only controls transport visibility; it does not add an OpenClaw policy that observes all senders but restricts triggers.
