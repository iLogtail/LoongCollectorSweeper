## Summary

Keep open. The underlying Control UI display-url problem is not fully implemented on current main, and this PR still contains useful work, but it is not merge-ready as a cleanup-close candidate because the branch includes unrelated agent error-message changes and a new skill, plus review comments flagged user-facing leakage risks in those unrelated changes.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up rather than closing it. The best path is to split out the unrelated skill and agent error-message commits, rebase onto current main, and land a focused Control UI display-link fix that normalizes lan -> loopback for user-facing URLs across configure wizard, status/status-all, daemon status, setup finalization, and dashboard, preferably through a small shared helper plus regression tests. If a cleaner replacement PR is opened for that exact scope, this PR can then be superseded by the replacement.
