## Summary

Keep open. Current main has partly moved the Codex-specific path away from the reported `codex/*` shape, but the central generator issue is still plausible: explicit providers with custom `models` and no resolved credential can still flow into `models.json`. The session/model-list divergence also still needs a maintainer decision rather than an auto-close.

## Best Possible Solution

Keep the issue open and fix it narrowly in the model generator/runtime contract: skip or neutralize unresolved custom providers with model rows unless their auth mode is explicitly keyless-supported, preserve known exceptions such as `auth: "aws-sdk"`, and add a regression test with one invalid custom provider plus one unrelated valid/discovered provider. Separately, make `sessions_spawn` either accept catalog-visible provider/model refs under a documented rule or return a clear allowlist/configuration error when `agents.defaults.models` excludes the requested model.
