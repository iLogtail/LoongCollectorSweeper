## Summary

Keep PR #71898 open. Current main still hard-codes `memory_search` surfaced memory-manager results as `corpus: "memory"`, even though the tool accepts `corpus=sessions`, filters session hits by `hit.source === "sessions"`, and the shared search result contract carries `source: "memory" | "sessions"`. The PR is a focused bug fix with a regression test, and the only bot readability concern was addressed in the PR context by moving the test into a dedicated corpus-label describe block.

## Best Possible Solution

Keep the PR open for maintainer review and merge once CI/review are satisfactory. The best code direction is the PR’s narrow memory-core fix: type surfaced search results with `MemorySource`, set `corpus` from `result.source`, and keep the focused regression test that proves `corpus=sessions` returns session hits as `corpus: "sessions"`.
