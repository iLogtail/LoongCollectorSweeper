## Summary

Keep PR #71898 open. Current main still hard-codes `memory_search` surfaced memory-manager hits as `corpus: "memory"` even after filtering `corpus=sessions`, while the memory-manager contract carries `source: "memory" | "sessions"`. The PR is a focused bug fix with a regression test, and the bot review concern about test placement was addressed in the follow-up commit.

## Best Possible Solution

Keep the PR open and land the focused fix after normal maintainer/CI validation. The best code direction is exactly the PR's approach: preserve `MemorySearchResult.source` when shaping `memory_search` results and keep the focused regression test for sessions-sourced hits.
