## Summary

Keep open. Current main still ships the Lobster plugin pinned to @clawdbot/lobster 2026.4.6, and that published runtime still compiles user/workflow schemas through long-lived Ajv instances at the hot paths reported in #71148. OpenClaw runs that runtime in-process inside the gateway, so this remains an actionable bundled-plugin dependency/runtime issue rather than a stale or external-only report.

## Best Possible Solution

Keep #71148 open until the bundled Lobster runtime stops compiling equivalent schemas repeatedly on long-lived Ajv instances. The likely maintainer path is to add content-hash validator memoization and/or removeSchema cleanup in @clawdbot/lobster for sharedAjv and llm.invoke output schemas, add a focused regression that repeated identical schemas do not produce repeated Ajv compilations, release a fixed Lobster package, and bump extensions/lobster/package.json plus pnpm-lock.yaml in OpenClaw.
