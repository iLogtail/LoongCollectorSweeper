## Summary

Keep open. Current OpenClaw main still bundles `@clawdbot/lobster` 2026.4.6, and the embedded Lobster runtime on upstream `openclaw/lobster` main still has the reported per-call Ajv `.compile()` paths for `ask`, `llm.invoke`, workflow input validation, pipeline resume validation, and SDK resume input validation. This is a real OpenClaw gateway surface because the bundled Lobster plugin imports and runs `@clawdbot/lobster/core` in-process.

## Best Possible Solution

Fix the root cause in `openclaw/lobster`: add a bounded/content-hash validator cache or equivalent reuse path around `sharedAjv` and the local `llm_invoke` Ajv instance, update the workflow loader/runtime paths to reuse compiled validators where practical, and add a focused regression or stress test proving repeated identical schemas do not trigger unbounded compiles. Then publish a new `@clawdbot/lobster` version and bump `extensions/lobster/package.json` plus `pnpm-lock.yaml` in OpenClaw, with a user-facing changelog entry because this affects gateway memory behavior.
