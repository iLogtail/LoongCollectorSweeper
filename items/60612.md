## Summary

Current main still intentionally warns when the managed Gateway service uses a version-manager Node path such as nvm/asdf, and there is no user-facing durable `--node-path` or warning-suppression setting. The newer external service repair policy only prevents doctor from rewriting service lifecycle files for externally managed setups; it does not suppress the health warning. The ASDF follow-up comment is therefore a valid remaining instance, not a duplicate of the already-fixed launchd path drift issues.

## Best Possible Solution

Keep the issue open for maintainer product/API judgment. A concrete fix would either add a documented suppression/acknowledgement setting for intentionally version-managed Gateway runtimes, add a durable explicit Node binary path option for `openclaw gateway install` and doctor repair, or update docs to clearly state that the warning is expected and how to run an externally managed service when users choose NVM/ASDF anyway.
