## Summary

Keep open. Current main still single-pages the bundled Feishu `feishu_wiki` `nodes` action: it calls `client.wiki.spaceNode.list` once, sends only `parent_node_token`, and returns only `res.data.items`. The schema does not expose pagination controls, latest release `v2026.4.24` has the same path, and related PR #37641 was closed unmerged.

## Best Possible Solution

Keep #37626 open until the bundled Feishu plugin implements pagination for `feishu_wiki` `nodes`. The likely fix belongs in `extensions/feishu/src/wiki.ts`: request `page_size` up to Feishu's max, continue while `has_more` and `page_token` are present, accumulate all node items, and add a focused regression test that mocks multiple `spaceNode.list` pages. Schema additions for manual `page_size`/`page_token`/`fetch_all` should be a maintainer product choice; automatic full pagination would solve the central reported data-loss problem without changing user workflow.
