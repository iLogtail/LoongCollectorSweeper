## Summary

Keep PR #69268 open. Current main has the Anthropic adaptive payload shape for Claude Opus 4.7, but it does not implement the PR's remaining core fixes: routing pi-coding-agent wrapped Anthropic sessions through the boundary-aware OpenClaw transport, or pruning/short-circuiting assistant-tailed transcripts before no-input Anthropic calls. Related issue #69460 only covers the native transport payload shape, not this wrapper path or the assistant-prefill cleanup.

## Best Possible Solution

Keep the PR open and rework/land a scoped version against current main. The best path is to route supported Anthropic default-wrapper sessions through OpenClaw's boundary-aware transport without accidentally breaking intentional custom stream functions, then add the assistant-tail cleanup after effective prompt/images are finalized, write sanitized transcript state back consistently, and keep focused regression coverage in stream-resolution plus validate-turns/attempt helper tests.
