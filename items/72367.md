## Summary

Keep PR #72367 open. Current main still lacks the core behavior the PR is trying to add: bundled LSP servers are spawned without a disposable process group, per-run cleanup only calls `process.kill()` on the immediate child, and Gateway shutdown only sweeps bundled MCP runtimes. The linked bug #72357 is concrete and the PR is not obsolete, though it should be narrowed before landing because its branch includes unrelated session-lock, Docker-test, npm-runner, and changelog changes plus an unresolved shutdown-latency review comment.

## Best Possible Solution

Keep the PR open, but ask for a scoped revision: land the bundled-LSP process-group spawn, process-tree disposal, active-runtime Gateway shutdown sweep, and regression tests for #72357; split unrelated session-lock/Docker/npm-runner changes unless they are tied to a separate issue; and address the parallel-disposal review before maintainer merge consideration.
