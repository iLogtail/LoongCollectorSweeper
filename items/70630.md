Keep #70630 open. Current main still contains the exact Telegram no-visible-response fallback path the PR is trying to fix: when the turn has no queued final, no fallback, no dispatch error, and no delivered reply, it synthesizes a `NO_REPLY` payload and delivers the projected fallback. Current tests still assert the old visible-chatter behavior for Telegram direct messages, while the PR context and review comments show a focused source fix plus regression-test update that remains relevant.
