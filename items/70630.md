## Summary

Keep PR #70630 open. Current main still contains the behavior the PR is trying to change: Telegram no-visible turns synthesize a `NO_REPLY` payload and deliver it when direct silent-reply policy rewrites it to visible fallback text. The current Telegram test still asserts the old DM behavior, while the PR's supplied patch changes that assertion to true silence and marks the status reaction done. The related bug #70628 remains the canonical user problem, and the Greptile review suggestion was addressed by the PR's second commit per the provided discussion.

## Best Possible Solution

Keep the PR open and land this PR or an equivalent Telegram-owned fix after normal validation. The best code path is to suppress synthetic silent-reply fallback for no-visible Telegram DM turns, treat the silent turn as handled for status reactions, and update the Telegram dispatch regression test to assert no visible delivery and `done` status for that path.
