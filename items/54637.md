## Summary

Keep open. Current `main` at `a6aa151653ab478e9aa23e95c6446c41adc4fce1` does not contain the PR's channel-aware explicit-disable fix: auto-enable still checks explicit disable by `pluginId` only, the prefer-over path still passes only `pluginId`, and the BlueBubbles-disabled/iMessage-fallback regression test from the PR is not present on `main`.
