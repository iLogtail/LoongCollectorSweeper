## Summary

Keep open. Current main does not implement automatic rendering of plain remote image URLs in the chat window. Chat text is routed through the sanitized markdown renderer, and that renderer explicitly flattens remote markdown images to alt text while only allowing inline data-image URIs as <img>. The issue is recent, clear, not maintainer-authored, and has no protected labels in the provided GitHub context.

## Evidence

- **Chat messages use sanitized markdown for visible text:** Message bubbles render markdown text with toSanitizedMarkdownHtml inside .chat-text, so plain URL behavior is governed by the UI markdown renderer.
  - file: [ui/src/ui/chat/grouped-render.ts:1479](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/ui/src/ui/chat/grouped-render.ts#L1479)
