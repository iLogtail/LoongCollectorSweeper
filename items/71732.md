## Summary

Keep #71732 open. Current main still lacks the requested generic plugin session-extension and host-routed patch-action contract. The existing code has adjacent surfaces only: closed core `sessions.patch` fields, post-write internal hook notification, plugin gateway RPCs, and raw trusted runtime session-store helpers. Draft PR #72082 is the right implementation path, but it is still open/draft in the provided GitHub context and not present on current main.

## Best Possible Solution

Keep #71732 open as the focused session-extension RFC/tracking issue until a maintainer accepts and lands the generic host-hook implementation, likely via #72082 or a successor. The best implementation is a typed plugin session-extension registration surface on OpenClawPluginApi, registry storage, host-owned sessions.pluginPatch or equivalent protocol support, schema validation before writes, public projection into session list/detail rows, cleanup on reset/delete/compaction/plugin disablement, stable error codes, scope enforcement, and fixture-plugin contract tests.
