## Summary

Keep #69269 open. Current main still does not implement the coordinated invariant fix: compaction hooks pass raw provider values, reset hook contexts still omit messageProvider, and the auto/command compaction bookkeeping still ignores tokensAfter=0 instead of clearing stale token/cost state. The related #69270 PR appears to be the right fix path, but it is open and not present in this checkout.

## Best Possible Solution

Keep #69269 open until #69270 or an equivalent main-branch patch lands. The best fix is a shared hook message-provider resolver used by auto compaction, engine-owned compaction, command reset, and gateway reset paths, plus bookkeeping that refreshes totalTokens, totalTokensFresh, and estimatedCostUsd from compaction results including tokensAfter=0, with regression tests for mixed-case providers, channel-scoped inference, non-channel keys, and zero-token compactions.
