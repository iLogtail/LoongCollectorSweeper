## Summary

Keep #69505 open. Current main does not implement an `assistantPrefix`/assistant-prefill contract for cron `agentTurn`, agent defaults, the agent RPC/CLI, or provider request construction. The existing `responsePrefix` feature is an outbound post-generation reply transform, so it does not solve the reported failure mode where streamed narration reaches WhatsApp before the intended reminder.

## Best Possible Solution

Keep the issue open for maintainer/product design. A proper implementation should add an additive assistant-prefill field to cron payload/schema/storage and agent defaults or agent RPC as appropriate, expose CLI/docs for cron and agent runs, and implement/test provider handling for at least Anthropic plus one OpenAI-compatible path while leaving `messages.responsePrefix` as a separate outbound formatting feature.
