## Summary

Keep PR #70466 open. Current main still exits 0 for both spawned and supervised restart modes, so the PR's requested systemd `Restart=on-failure` hardening is not implemented on main. The linked issue #70354 was closed because OpenClaw-managed systemd units now use `Restart=always`, but this PR still targets a real remaining custom/legacy `Restart=on-failure` behavior and has focused discussion rather than being obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer judgment. If accepted, split the spawned and supervised branches in `src/cli/gateway-cli/run-loop.ts`, preserve spawned exit 0 and launchd's delay, make non-launchd supervised restarts exit non-zero, and add a Linux/systemd-style regression test in `src/cli/gateway-cli/run-loop.test.ts`. If maintainers prefer to rely solely on `Restart=always`, close explicitly as a product decision rather than automated cleanup.
