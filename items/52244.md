## Summary

Keep open. Current `main` still appears to miss the OpenRouter-specific default-resolution hooks this PR is trying to add, so the underlying bug remains plausible. The two March 2026 bot review comments are partly stale on current `main` because catalog-hook routing and thinking-default resolution have both changed since then, but I did not find an equivalent fix landed in `main` for dynamic OpenRouter reasoning defaults.

## Evidence

- **Current main now routes catalog hooks through bundled provider compat plugins:** `resolveCatalogHookProviderPluginIds()` merges enabled provider plugins with `resolveBundledProviderCompatPluginIds(params)`. That means the earlier review comment claiming bundled OpenRouter could never run `augmentModelCatalog` no longer matches current `main`.

  - sha: [16b7dee1](https://github.com/openclaw/openclaw/commit/16b7dee1)
- **Current main now resolves thinking defaults through the plugin-aware profile path:** `resolveThinkingDefaultForModel()` in `src/auto-reply/thinking.ts` calls `resolveThinkingProfile()` and uses provider hooks before falling back. That makes the earlier review comment about `thinking.shared.ts` being the active path stale on current `main`.
  - file: [src/auto-reply/thinking.ts:230](https://github.com/openclaw/openclaw/blob/16b7dee1/src/auto-reply/thinking.ts#L230)

I did not find an equivalent replacement in this file.
