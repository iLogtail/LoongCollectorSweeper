## Summary

Keep open. Current `main` has not implemented this PR's core case-insensitive gateway image-support lookup, and the related #65211 was closed unmerged rather than providing a canonical shipped replacement. The PR is still aimed at a real remaining gap, but it should not land as-is because its unconditional model-only fallback would weaken the current fail-closed provider mismatch behavior noted in review.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require a narrower patch: use `findModelInCatalog` for provider-scoped lookups, only run model-only fallback when no provider was supplied, preserve the existing provider-mismatch fail-closed behavior, and add tests for mixed-case provider/model matches plus explicit-provider mismatch.
