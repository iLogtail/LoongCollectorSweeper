## Summary

Keep open. On current `main` (`c65aa1d2a6e573d49619c4dd0698a76010476e7d`), `openclaw devices list` still renders paired devices with a single `Device` column and hides `deviceId` whenever `displayName` is present, so multiple same-type devices can still be hard to distinguish. The pairing model already carries richer metadata, and a still-open PR (#57449) proposes the exact missing `Device ID`/`Name`/`Client`/`OS`/`Last active` output, which is further evidence this is not yet implemented on `main`.

## Evidence

- **Current CLI still hides device IDs behind display names:** The paired-devices table only has `Device`, `Roles`, `Scopes`, `Tokens`, and `IP` columns, and the `Device` cell is `device.displayName || device.deviceId`. If multiple devices share the same display name/type, the human-readable list does not expose the exact `deviceId` needed by `devices remove`.
  - file: [src/cli/devices-cli.ts:406](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/cli/devices-cli.ts#L406)

The missing part is the CLI presentation, not underlying state availability.
