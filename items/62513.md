## Summary

Keep #62513 open. Current main still performs the reported SQLite-backed `hasIndexedContent()` check from the `search()` hot path, and the linked fix PR #62597 is still open/unmerged rather than shipped on main.

## Best Possible Solution

Keep #62513 open until #62597 or an equivalent memory-core patch lands. The best fix is a small cache around `hasIndexedContent()` in `extensions/memory-core/src/memory/manager.ts`, with updates/invalidation in chunk insert, delete, and reset paths plus a focused regression test proving repeated searches avoid redundant SQLite probes without returning stale indexed-content state.
