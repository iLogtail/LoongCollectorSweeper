Keep #71069 open. Current main supports canonical Gemma 4 refs, but the exact reported `Gemma4-26b-a4-it-gguf` ref is not documented or recognized in-tree, and main still emits the reported generic reset message when a stored override is outside the agent allowlist. The issue is recent and asks either for acceptance or a clearer rejection, so there is no conservative auto-close reason.

Ask the reporter for their model/provider config and session override fields, plus whether the target backend is Google, Ollama, LM Studio, inferrs, or another GGUF/OpenAI-compatible server.
