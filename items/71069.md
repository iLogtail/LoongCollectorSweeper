## Summary

Keep open. Current `main` still contains the exact reset path that emits the reported message when a persisted session model override falls outside the agent allowlist, and there is no evidence that the reported `Gemma4-26b-a4-it-gguf` override is already implemented on `main`. The most likely explanation is a real model-selection inconsistency: persisted overrides without `providerOverride` are still normalized to the session default provider, while user-facing docs for `/model` prefer explicit `provider/model` refs and only infer bare IDs in limited cases.
