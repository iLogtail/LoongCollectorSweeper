## Summary

Keep #39245 open. Current main has related provider-prefixed tool-name normalization for dispatch/replay and canonical Kimi `functions.<name>:<index>` ID preservation, but it still does not implement the PR's core behavior: safely pairing canonical `functions.exec:0` and space-mangled `functions exec:0` IDs. The PR is also recently active and has an unresolved security review about normalization-induced collisions, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is a narrow, regression-tested fix that preserves current name-side normalization, adds collision-safe matching for space-mangled versus canonical Kimi-style tool-call IDs only where needed, binds any normalized pairing to additional context such as tool name or adjacency, and resolves or splits the unrelated logging diagnostic change before landing.
