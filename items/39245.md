## Summary

Keep PR #39245 open. Current main has related Kimi canonical-ID preservation and provider-prefixed tool-name normalization for dispatch/replay, but it still does not implement the PR's central behavior: safely matching or sanitizing mixed canonical `functions.exec:0` and space-mangled `functions exec:0` tool-call IDs. The older related PR #39091 is closed, while #39245 remains the active branch and has an unresolved security review about normalization-induced ID collisions.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is a narrow agent-side fix that keeps current name normalization, adds collision-safe handling for leading `functions ` versus `functions.` IDs only where provider interop requires it, binds any normalized pairing to additional context, preserves replay-safe raw IDs, adds regression tests for the mixed dot/space ID case, and splits unrelated logging cleanup before landing.
