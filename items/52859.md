Keep open. The reported Telegram inbound regression is still present on current main: inbound text passed to the agent still goes through expandTextLinks(), and that helper still ignores bold/italic/code/pre/strikethrough/underline entities except for text_link. The existing tests also explicitly assert that a bold entity is returned unchanged, so current coverage has not been updated for the requested markdown preservation behavior.
