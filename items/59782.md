## Summary

Keep #59782 open. Current main partially hardens generic runner failures by suppressing some boilerplate in group/channel contexts, but it still does not provide WhatsApp DM support for silent/custom/suppressed agent failure replies. WhatsApp config/schema/docs lack `silentErrorReplies`, `suppressAgentErrors`, `errorPolicy`, or a custom error-message option, while Telegram has channel-specific error policy machinery. Related PR #59801 remains the useful implementation path rather than proof that this is shipped.

## Best Possible Solution

Keep the issue open and handle it with the focused WhatsApp error-reply implementation path, likely by reviewing/reworking PR #59801 or a successor. The product/code path should add a documented WhatsApp config contract, account/group/direct resolution behavior, dispatch suppression/customization for `isError` agent failure payloads, and regression tests covering WhatsApp DMs as well as any group/channel behavior maintainers want to preserve.
