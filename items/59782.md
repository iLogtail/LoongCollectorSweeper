## Summary

Keep open. Current main does not implement WhatsApp support for silent/custom/suppressed agent failure replies: the agent runner still constructs user-facing failure text, WhatsApp outbound delivery sends payloads without an error-suppression policy, and WhatsApp config schema/types do not expose silentErrorReplies, suppressAgentErrors, or a custom error-message option. The comparable silentErrorReplies support remains Telegram-specific.
