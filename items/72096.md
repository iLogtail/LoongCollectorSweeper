## Summary

Keep open. Current main still appends each light/REM dream diary narrative with the current timestamp and has no per-phase last-emitted cutoff for narrative inputs. The only DREAMS.md diary dedupe path fingerprints the timestamp together with the body, and its test explicitly preserves identical diary bodies when timestamps differ, which matches the reported hourly duplicate shape rather than fixing it. Related #68882 remains a different promotion-candidacy contamination filter layer, not the diary append path reported here.

## Best Possible Solution

Keep #72096 open. The likely fix belongs in `extensions/memory-core/src/dreaming-phases.ts` and/or `extensions/memory-core/src/dreaming-narrative.ts`: persist a per-workspace/per-phase diary emission cursor or recent narrative fingerprint, suppress repeated light/REM narrative bodies across adjacent cycles, and add a regression test with two light-phase cycles producing the same snippets at different timestamps. A follow-up doctor cleanup could remove existing duplicate-body diary entries, but it should not be the only fix.
