## Summary

Keep #56692 open. Current main still lacks the per-message target metadata the issue requests: group history and prompt context are carried as generic sender/body/timestamp entries, Telegram records and forwards that generic shape, and prompt rendering does not annotate whether a history or reply item was addressed to this bot, another bot, or nobody. The linked PR #58102 proposed the exact missing contract (`mentionedBot`, `repliedToBot`, `OtherBotUsernames`, `ReplyToBotUsername`) but is closed and unmerged; the community triage/plugin resources mentioned in the issue are useful mitigations, not a replacement for the missing core/channel context seam.

## Best Possible Solution

Keep the issue open and implement the narrow core/channel seam: add backward-compatible optional target metadata to `HistoryEntry` and `MsgContext.InboundHistory`, add reply-target metadata to `TemplateContext`, populate those fields from Telegram and other channel plugins where platform data allows, render them in `buildInboundUserContextPrefix()` and group prompt guidance, and add regression tests covering current-agent mention, other-bot mention, reply-to-current-bot, reply-to-other-bot, and unrelated history. Community plugins like `openclaw-triage-gate` can remain useful mitigations, but they should not replace a core prompt/SDK contract for target-aware group context.
