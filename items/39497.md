## Summary

Keep #39497 open. Current main at b67d9bf7f06f4c655b1e0869395056c7d631abc2 still only remaps sandbox file-tool paths through `containerWorkdir` and still has regression coverage expecting `/agent/...` to be rejected under the workspace-root guard, even though the sandbox mount layer and docs define `/agent` as the read-only agent workspace mount for `workspaceAccess: "ro"`.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the sandboxed file-tool guard/wiring: make `wrapToolWorkspaceRootGuardWithOptions` mount-aware via the sandbox mount table, or pass an explicit `/agent` mapping only for `workspaceAccess: "ro"`, then add regression coverage proving `/agent/...` reads resolve while writes remain disabled. Treat `/tmp` reads as a separate policy decision.
