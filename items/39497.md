## Summary

Keep #39497 open. Current main still lacks a `/agent` mount mapping in the workspace-root guard: the guard maps only `containerWorkdir`, sandboxed read-tool wiring passes only that option, and current coverage still expects guarded `/agent/...` reads to fail with `Path escapes sandbox root`. Related fix PRs for this issue were closed unmerged, and the merged adjacent sandbox PR explicitly left `/agent` read-guard remapping out of scope.

## Best Possible Solution

Keep the issue open. The best fix belongs in the sandboxed file-tool workspace-root guard/wiring: validate through the sandbox mount table, or pass an explicit `/agent` mapping only for the `workspaceAccess: "ro"` agent workspace mount. Add regression coverage proving `/agent/...` reads succeed in the read-only workspace path while write/edit/apply_patch remain blocked. Treat `/tmp` reads as a separate policy decision.
