## Summary

Current main has the generic transcript-to-Control-UI update path for Feishu-originated turns, but it does not implement the issue's full bidirectional dashboard/Feishu mirroring. The Control UI still sends normal chat input as session-only (`deliver: false`), and the gateway explicitly prevents webchat clients from inheriting external channel delivery routes, so a dashboard message will not be relayed back to Feishu through the normal UI send path.

## Best Possible Solution

Keep this open as a behavior/product gap. Maintainers should decide whether Control UI sessions bound to Feishu should offer an explicit reply-via-Feishu path; if yes, add a UI affordance/RPC route that sets an explicit Feishu origin or safe deliver mode, preserve current session-only webchat behavior by default, and cover Feishu inbound transcript updates plus dashboard-to-Feishu delivery with focused gateway/UI/plugin tests.
