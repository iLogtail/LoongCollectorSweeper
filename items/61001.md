## Summary

Keep #61001 open. Current main has coarse bootstrap controls (`contextInjection`, size caps), heartbeat-only lightweight mode, and a hardcoded subagent/cron filter, but it does not implement the requested additive per-file `bootstrapFiles` session rules or frontmatter-driven loading. Default direct/group runs still load the full bootstrap set, heartbeat runs only get HEARTBEAT.md when `lightContext` is explicitly enabled, and subagent/cron filtering is hardcoded rather than configurable by session type.

## Best Possible Solution

Keep the issue open for a core config/runtime change. The likely implementation should add an additive, backward-compatible per-file bootstrap policy under `agents.defaults` (for example `bootstrapFiles` with `sessions` rules), define a normalized session-kind contract for direct, group, heartbeat/isolated, subagent, cron/default runs, apply it inside `resolveBootstrapFilesForRun()` before hook overrides and budget truncation, and update strict config schema/types, generated help/docs, and focused tests for default blanket behavior plus direct/group/heartbeat/subagent filtering. Frontmatter support should either be implemented deliberately with parsing/security tests or left out of the first pass.
