## Summary

Keep PR #63662 open. Current main already has proactive recall through plugin-owned paths such as `active-memory` and `memory-lancedb`, but it does not implement the PR's core `agents.defaults.memoryInjection` mode or a `memory-core`/`agents.defaults.memorySearch.autoRecall` config surface. The PR is not safe to merge as-is because the discussion and file list show unresolved behavior concerns plus unrelated changes, but it still contains potentially useful product work that is not fully shipped or tracked by a better canonical item.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but ask for a focused rework: split out unrelated changes, decide whether `memory-core` should own `agents.defaults.memorySearch.autoRecall` given existing `active-memory`, and keep the core change narrowly on a validated `agents.defaults.memoryInjection` mode that can skip only `MEMORY.md` while preserving the rest of bootstrap context. If maintainers prefer the active-memory model, open or link a narrower canonical issue for the remaining selective MEMORY.md injection capability before closing this PR.
