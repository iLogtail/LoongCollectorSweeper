## Summary

Keep #32868 open. Current main still has the reported same-channel block streaming delivery gap: cross-channel block replies await sendPayloadAsync, but same-channel block replies enqueue through ReplyDispatcher without awaiting the dispatcher's actual async deliver chain. The multi-channel issue discussion and closed unmerged fix PRs still describe a real core bug rather than a closeable stale report.

## Best Possible Solution

Keep this issue open and fix the core same-channel delivery contract. The likely path is to make ReplyDispatcher.sendBlockReply return a delivery promise or add an awaited delivery method, await it in the same-channel onBlockReply path with abort/timeout handling, update ACP/plugin SDK types and mocks, and add a regression test proving text_end block delivery completes before tool_execution_start continues.
