## Summary

Keep #32868 open. Current main still has the same-channel block streaming delivery gap: cross-channel block replies are awaited, but same-channel replies only enqueue through the dispatcher, whose API still returns a boolean while actual channel delivery drains later on an async send chain. The related fix PRs in the provided GitHub context were closed unmerged, and newer #71664 is duplicate evidence that the bug remains observable.

## Best Possible Solution

Keep #32868 open as the canonical core bug. The likely fix is to change the same-channel delivery contract so `sendBlockReply` can expose delivery completion, await it from `dispatch-from-config.ts` before tool execution continues, update ACP/plugin SDK types and mocks accordingly, and add a regression test proving `blockStreamingBreak: "text_end"` channel delivery completes before `tool_execution_start` proceeds.
