## Summary

Keep open. Current main still unconditionally copies ConfigMap `openclaw.json` and `AGENTS.md` into the PVC on every pod init, so the reported restart clobbering path is real. The PR is not safely closeable as implemented-on-main, duplicate, or non-actionable; however, its current one-time-copy fix has an unresolved trade-off because the documented Kubernetes redeploy flow says ConfigMap edits are picked up by applying manifests and restarting the pod.

## Best Possible Solution

Keep this PR open for maintainer/product judgment and revise the Kubernetes init/sync contract rather than auto-closing it. The best path is a targeted manifest/deploy change that preserves user-modified PVC files across ordinary pod restarts while still providing an explicit, documented way to apply ConfigMap updates on redeploy, plus docs that clearly distinguish initialized runtime state from declarative ConfigMap defaults.
