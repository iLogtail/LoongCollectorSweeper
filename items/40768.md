## Summary

Keep open. Current `main` still treats a Feishu bot mention as an exact `open_id` match and drops group messages when `requireMention` is on and that exact match fails. I found no landed fallback that reconciles app-scoped cross-bot IDs via `union_id`, `user_id`, display-name identity, or other cross-app mapping, and the latest shipped release `v2026.4.23` carries the same mention-check implementation. The linked issue discussion and PR references therefore still describe a plausible unfixed bug rather than something safe to auto-close.
