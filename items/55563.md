## Summary

Keep open. Current main appears to have addressed the gateway/systemd nvm half through service audit and reinstall/restart repair, but the browser CLI syntax regression is still not fully resolved: `--browser-profile` remains a parent `browser` option, the root Commander program enables positional options, and the WSL2 troubleshooting guide still shows the reported post-subcommand form. The related compatibility PR #55567 was closed unmerged, so no canonical shipped fix covers the remaining browser/docs break.

## Best Possible Solution

Keep #55563 open, but narrow the remaining work to the browser CLI/docs regression. Either restore compatibility for `openclaw browser tabs --browser-profile <name>` across browser subcommands or update every affected doc/changelog/migration note to make the breaking syntax change explicit; add a focused browser CLI regression test. The gateway/systemd nvm repair path can be treated as already substantially addressed on current main.
