## Summary

Keep open. Current `main` supports routing shell execution to a paired Windows node, but only through explicit `exec host=node` / `tools.exec.host=node` configuration. The node host surface is still limited to `system.run` / `system.which`, and the normal filesystem tools (`read`/`write`/`edit`/`apply_patch`) are still implemented against the gateway workspace or sandbox, not a remote node filesystem. That means the issue’s core complaint about a Linux-hosted gateway still operating Linux-side files instead of local Windows files is not clearly resolved on `main`.
