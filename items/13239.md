## Summary

Keep #13239 open. Current main has provider lists, model auth/probe status, model catalog rows, and a new plugin-contract compatibility/deprecation registry, but it still does not implement the requested public provider x model-family x capability lifecycle matrix with stable/experimental/deprecated/broken states, deadlines, tracked issue links, generated docs, and doctor output. The Matrix plugin comment remains useful evidence that compatibility visibility matters, but the linked regression (#52899) does not supersede this broader provider policy request.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, status, deadline, replacement, and canonical issue link. Generate the public docs page from that registry and surface a concise doctor or models-status view while keeping live auth/probe health separate from support lifecycle state.
