## Summary

Keep PR #70391 open. Current main still performs the redundant final-event history reload this PR targets: final assistant payloads are appended locally, but `shouldReloadHistoryForFinalEvent` still returns true for every final event and `handleChatGatewayEvent` still calls `loadChatHistory`. The PR is not merge-ready as-is because review raised real edge cases around deferred `session.message` replay and role handling, but no close reason applies because the requested behavior is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open or replace it with a maintainer patch that skips only the direct redundant final-event `chat.history` reload when the final payload is already displayable, while preserving deferred `session.message` replay for unrelated updates and adding regression tests for final assistant payloads, missing/non-assistant roles, tool-output finals, and deferred reload replay.
