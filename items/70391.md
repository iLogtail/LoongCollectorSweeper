## Summary

Keep PR #70391 open. Current main still reloads chat history after ordinary final chat events even when `handleChatEvent` already appended the final assistant payload locally, so the PR's central behavior is not implemented on main. The PR is relevant but likely needs revision for the review concern about deferred `session.message` reloads before it should land.

## Best Possible Solution

Keep the PR open and ask for a targeted revision: make final events with a confirmed displayable assistant payload skip only the redundant direct final-event history reload, but preserve or replay deferred `session.message` reloads when they may represent other transcript updates. Add regression coverage in `ui/src/ui/chat-event-reload.test.ts` and `ui/src/ui/app-gateway.node.test.ts`, and cross-link the broader remaining reload race tracked by #66875.
