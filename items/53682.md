## Summary

Keep open. Current `main` still appears to have the reported heredoc bug in gateway allowlist approvals, and the PR's proposed fix has a credible unresolved security concern in the timeout-fallback path.

## Evidence

- **Current main still special-cases heredocs for approval:** `processGatewayAllowlist()` marks allowlist-satisfied heredoc commands as requiring explicit approval and warns that heredoc execution needs approval in allowlist mode.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:164](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/bash-tools.exec-host-gateway.ts#L164)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Approved gateway exec still runs the rebuilt enforced command:** In the async approval follow-up path on current `main`, `runExecProcess()` is still called with `execCommand: enforcedCommand`; there is no heredoc-specific override to preserve the original command text.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:371](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/bash-tools.exec-host-gateway.ts#L371)

## Risks / Open Questions

- Current `main` likely still mangles approved heredoc commands in gateway allowlist mode because execution is routed through the rebuilt `enforcedCommand` path.
- The PR as proposed may weaken allowlist hardening for `askFallback === "allowlist"` timeout approvals, because that path also becomes `approvedByAsk = true` without an explicit human allow decision.
- There is no heredoc-specific regression test on current `main` to prove either the bug or a safe fix path is covered.
