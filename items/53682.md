## Summary

Keep PR #53682 open. Current main still routes approved gateway heredoc execution through `enforcedCommand`, while the enforced-command builder reconstructs commands from planned argv and does not preserve heredoc redirection/body syntax. The PR targets a real remaining bug, but the posted review comment identifies a concrete safety issue in the current patch around timeout fallback approvals, so it needs revision rather than closure.

## Best Possible Solution

Revise PR #53682 so gateway heredocs execute the original authored command only after an explicit human `allow-once` or `allow-always` decision, while timeout fallback approvals keep using `enforcedCommand`. Add focused gateway tests for explicit heredoc approval and allowlist timeout fallback behavior.
