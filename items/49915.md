## Summary

Current main has a partial Feishu slash-command fix for mention/content normalization, but the reported group-command failure is still present in the current authorization path. Feishu group commands with no group/channel allowFrom become CommandAuthorized=false under access-groups mode, and downstream whole-message control commands are dropped rather than delivered as normal text. Related PR #49958 is an open, unmerged attempt to fix the same remaining path, so this issue should stay open for maintainer handling.

## Best Possible Solution

Keep #49915 open and resolve it through PR #49958 or an equivalent maintainer-reviewed patch. The fix should define the intended security semantics for group commands when the group is already allowed but no sender allowFrom is configured, update src/channels/command-gating.ts or the Feishu group-command authorizer accordingly, and add Feishu regression tests for group-bound /new, /status, and /model with explicit @mention plus the no-allowFrom configuration.
