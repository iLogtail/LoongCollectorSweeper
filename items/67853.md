## Summary

Keep this PR open. Current main still depends on the memory-wiki corpus supplement being registered for `memory_search`/`memory_get` wiki access, and `resolvePluginTools` still has the gateway-bindable active-registry reuse path that the PR is trying to correct. The PR is not obsolete or implemented on main; it contains plausible, still-relevant fixes, though it needs maintainer review because it is broad and has review-thread concerns.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased, scoped implementation that adds a boundary-safe memory-wiki fallback API, updates package exports/alias/boundary checks, fixes `resolvePluginTools` to avoid incompatible active registry reuse while preserving graceful behavior during in-flight loads, and carries focused regression tests for wiki fallback, plugin disable policy, duplicate fallback avoidance, and gateway/tool-registry compatibility.
