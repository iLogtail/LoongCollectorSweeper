## Summary

Current main already includes the channel-registry pinning fix that prevents configured channels from disappearing from the send path after later plugin-registry churn. That matches this report’s `unsupported channel: whatsapp` symptom for a still-running channel, and the same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

## Risks / Open Questions

- I did not reproduce this live against a real WhatsApp setup in this checkout; the close decision is based on code-path inspection plus targeted regression coverage.
- The report ties the symptom to a health-monitor restart window; the evidence strongly supports a now-fixed registry-desync class, but does not prove the restart itself was the only trigger.

## Close Comment

Closing from a Codex review because current `main` already carries the channel-registry pinning fix that matches this failure mode, and that implementation shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

Why this matches the report:
- `send` returns `INVALID_REQUEST: unsupported channel: <id>` when channel normalization can no longer resolve the registered channel (`src/gateway/server-methods/send.ts:101`, `src/gateway/server-methods/send.ts:423`).
- That normalization is driven by the registered channel plugin registry (`src/channels/registry.ts:72`).
- Current startup code now pins the initial channel registry so later plugin-registry churn cannot evict configured channels from lookup (`src/gateway/server-plugin-bootstrap.ts:98`, `src/plugins/runtime.ts:151`).
- There is direct regression coverage that later registry replacement still preserves channel lookup and outbound adapter resolution (`src/channels/registry.helpers.test.ts:43`, `src/plugins/runtime.channel-pin.test.ts:178`, `src/gateway/server-plugins.test.ts:364`).

Given that, this looks implemented on current `main` rather than an open cleanup item. If you still hit the same behavior on `v2026.4.22` or newer, please open a fresh issue with the exact version and gateway logs around the restart window.
