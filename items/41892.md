## Summary

Keep #41892 open. Current main still has the Cron Control UI as a summary strip, flat jobs list, run-history panel, and edit form; it does not include the proposed 24h cron timeline/calendar view. The PR is unmerged, tied to still-open feature issue #40644, and contains useful focused UI work, even though review comments show it needs scheduler-correctness and UI/i18n rework before merge.

## Best Possible Solution

Keep #41892 open as prior art and an implementation candidate for #40644. The best path is to rebase and rework it so timeline occurrences are derived from the existing scheduler/server contracts where possible, add focused tests for timezone, stagger, every, DST, and Croner-token behavior, regenerate i18n through the UI pipeline, and keep #40644 as the canonical feature request until a reviewed implementation lands.
