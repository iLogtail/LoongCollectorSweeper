## Summary

Keep PR #69344 open. Current main still uses the old TUI long-token sanitizer policy: generic non-copy-sensitive tokens are split at 32 characters, no dot/paren/bracket code-fragment preservation exists, and the tests still assert the 32-character guard. Related #39565/#39505 overlap the copy-safety area but do not fully supersede this PR's broader 128-character threshold and code-fragment proposal.

## Best Possible Solution

Keep #69344 open for maintainer review or replace it with an equivalent current-main formatter fix. The best path is a focused TUI sanitizer policy in src/tui/tui-formatters.ts that preserves genuinely copy-sensitive/code-like fragments without broadly disabling narrow-terminal protection, with regression tests in src/tui/tui-formatters.test.ts for both code snippets and generic long-token wrapping. Coordinate with #39565/#39505 and related TUI copy-safety work rather than closing this PR as implemented or superseded.
