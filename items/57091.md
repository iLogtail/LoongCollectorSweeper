## Summary

Keep #57091 open. Current main still does not provide the requested SKILL.md frontmatter validation or user-visible parse diagnostics: malformed/missing frontmatter can collapse to an empty metadata object, incomplete skills are skipped by returning null, `openclaw skills` has list/info/check but no validate/doctor subcommand, and the Gateway/UI status path only reports successfully loaded skills. The existing docs include a minimal frontmatter example, but not the requested validation command or malformed-frontmatter troubleshooting flow.

## Best Possible Solution

Keep this open and implement a structured skill-load diagnostics path in core: make the frontmatter parser/loader return `SkillLoadIssue` records with path, reason, and parser details; surface those through `openclaw skills list/check` and `skills.status` for the Web UI; add a dedicated validation command or extend `check` with validation semantics; and update the public skills docs with valid frontmatter examples and common malformed-SKILL.md fixes.
