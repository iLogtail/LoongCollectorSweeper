## Summary

Current main still does not broadcast the restart sentinel system event to other recently active sessions. It only wakes the sentinel session, while newer recovery code is narrower and does not replace the requested cross-session GatewayRestart behavior. The PR discussion also shows the author addressed the original bot review defects on the branch, so this remains plausible unmerged work rather than a safe close.

## Best Possible Solution

Keep PR #44401 open for maintainer review. The best path is to adapt the restart sentinel to current main's architecture: enumerate known agent session stores, use SessionEntry.updatedAt for recent activity, skip cron/global/unknown pseudo-sessions, enqueue the restart system event and heartbeat wake for each selected session, and add regression coverage. If maintainers prefer a generic broadcast primitive, the restart sentinel still needs to call that primitive so the restart-specific behavior is actually fixed.
