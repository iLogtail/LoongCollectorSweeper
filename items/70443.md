## Summary

Current main does not implement the requested airgapped Docker setup rerun path. `scripts/docker/setup.sh` still has no `--offline` or equivalent argument parsing, still builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before the rest of setup, and still builds the optional sandbox image when sandbox setup is enabled. The Docker docs and existing e2e coverage also describe/test the current build-or-pull path rather than a preloaded-image rerun. The related `OPENCLAW_DOCKER_PLATFORM` items are separate cross-platform build requests and do not supersede this issue.

## Best Possible Solution

Keep this issue open. The best implementation is a narrow `--offline` flag in `scripts/docker/setup.sh`, backed by `src/docker-setup.e2e.test.ts`, that parses args before setup, skips main image build/pull only when explicitly requested, verifies `docker image inspect "$OPENCLAW_IMAGE"` succeeds with a clear `docker load` message when missing, applies warning-only local-image verification for the optional sandbox image, and leaves the default online path behavior unchanged. The Docker install docs should then document the airgapped rerun workflow on the public Docker install page.
