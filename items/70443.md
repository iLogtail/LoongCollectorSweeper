## Summary

Keep #70443 open. Current main still lacks a `--offline` or equivalent Docker setup path: `scripts/docker/setup.sh` does not parse user flags, builds `openclaw:local` or pulls any non-local image before the rest of setup, and still builds the sandbox image when sandbox setup is enabled. The Docker docs and e2e coverage also do not document or test an offline/preloaded-image rerun path. Related #69143/#69314 were about cross-platform build selection, not this offline rerun behavior, and #69143 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement it in core Docker setup. Add explicit `--offline` parsing near the top of `scripts/docker/setup.sh`; when set, skip `docker build`/`docker pull`, require `docker image inspect "$OPENCLAW_IMAGE"` for the gateway image, warn rather than fail for a missing optional sandbox image, and leave all later env sync, permission repair, onboarding, gateway config, compose startup, and sandbox overlay steps unchanged. Update `docs/install/docker.md` and add focused e2e coverage for the offline and default paths.
