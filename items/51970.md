## Summary

Keep open. Current `main` still ships the old masking behavior that exposes trailing secret characters in user-facing auth output, and this PR is not safely superseded because review already identified a short-secret regression in the proposed patch.

## Evidence

- **Current main still exposes secret suffixes:** `maskApiKey` on `main` still returns `first...last` for values up to 6 chars, `first2...last2` up to 16 chars, and `first8...last8` for longer values. The requested reduction to prefix-only masking is not implemented on current `main`.

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Tests still lock in the old disclosure behavior:** Current tests explicitly expect masked outputs like `ab...op`, `s...t`, and `12345678...ijklmnop`, which confirms the shipped behavior still reveals tail characters rather than only a short prefix.
