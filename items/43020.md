## Summary

Keep #43020 open. Current main still uses a single Mattermost draft preview post with block streaming disabled for the core path, so text-only block replies at assistant message boundaries are not delivered independently. The linked fix PRs were closed unmerged, and I did not find an equivalent implementation on main.

## Best Possible Solution

Keep the issue open. A maintainer-quality fix should either rotate/finalize the Mattermost draft preview at assistant message boundaries while avoiding duplicate final replies, or explicitly decide that the product should keep one preview post and update the issue/docs accordingly. The implementation likely belongs in extensions/mattermost/src/mattermost/monitor.ts with tests that simulate multiple assistant messages separated by tool activity.
