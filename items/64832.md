## Summary

Keep #64832 open. Current main has the #65203-style sessionFile rotation fix, but it does not implement this PR's remaining behavior: proactively archiving the prior isolated-session transcript as a .reset archive during heartbeat or cron rotation. The PR also has an unresolved correctness concern around cron run-history rows blocking archiveRemovedSessionTranscripts, so the useful cleanup work should be revised rather than closed.

## Best Possible Solution

Keep #64832 open or replace it with a narrower maintainer patch that archives the prior isolated-session transcript during both heartbeat and cron fresh-session rotation. The implementation should capture the prior entry before overwriting the active session key, archive after the new entry is persisted with reason: "reset" and restrictToStoreDir: true, and compute referencedSessionIds so inactive cron run-history rows do not block archiving the just-rotated transcript. Keep the existing #65203 sessionFile behavior and add focused heartbeat plus cron run-key regression tests.
