## Summary

Keep #6757 open. Current main has manual `/compact`, Gateway `sessions.compact`, UI/admin compaction, CLI post-turn/preemptive compaction, auto-compaction, and context-engine/plugin compaction hooks, but it still lacks the requested in-session agent-triggered self-compaction surface via `session_status(action: "compact")`, an agent `gateway` compact action, or a dedicated CLI/session self-compact command. The exact PR that attempted this, #43401, was closed unmerged, and the latest issue comment also kept the item open.

## Best Possible Solution

Keep #6757 open until maintainers either implement a first-class in-session self-compaction API, likely `session_status(action: "compact")` or an equivalent agent/session tool wired through direct compaction when already in a run, or explicitly decide agents must not compact their own active session. The implementation should classify the call as mutating, avoid session-lane deadlocks, respect visibility/permission policy, persist compaction counters/token snapshots, and add focused tests around `session_status`, mutation tracking, and active-run behavior.
