## Summary

Keep open. Current main still does not map Discord `MessageType.ChannelNameChange` or related native system-message types to queued system events, while the preflight path depends on that mapper to drop system messages before normal inbound handling. The PR is a focused bug fix for #24108, not obsolete or superseded by current main.

## Best Possible Solution

Keep #49286 open and move it through normal maintainer review. The best path is to land the focused Discord mapper/preflight regression fix after minor polish: preserve user-authored passthrough types, keep the rename preflight regression, improve mapper-test diagnostics if desired, add the required changelog attribution, and run the targeted Discord extension test lane before merge.
