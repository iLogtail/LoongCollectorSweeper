## Summary

Current main still does not implement the requested cost parsing behavior. Numeric-string costs are ignored, booleans are accepted as numeric costs, and NaN/Infinity can enter totals and model selection. The linked PR #37877 is closed unmerged, and current tests do not cover these edge cases.

## Best Possible Solution

Keep the issue open and land a narrow fix in skills/model-usage/scripts/model_usage.py: add a shared parse_cost helper that accepts finite numeric strings and finite numeric int/float values, rejects bool and non-finite values, and route aggregate_costs, pick_current_model, and latest_day_cost through it. Add focused tests in skills/model-usage/scripts/test_model_usage.py for numeric-string aggregation, bool/non-finite rejection, current-model selection, and latest-day cost.
