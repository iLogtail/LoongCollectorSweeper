## Summary

Keep #37878 open. Current main still has the reported model-usage parsing gap: numeric-string costs are skipped, booleans are accepted through Python's int/float check, and NaN/Infinity are not filtered. The linked PR #37877 is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #37878 open and land the narrow fix proposed by the issue: add a shared parse_cost helper in skills/model-usage/scripts/model_usage.py that accepts finite numeric strings and finite numeric values, rejects bool and non-finite values, and use it from aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py for aggregation, current-model selection, latest-day cost, bool rejection, and NaN/Infinity rejection.
