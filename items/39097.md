## Summary

Keep #39097 open. Current main still only has browser SSRF/private-network policy with hostname allowlist exceptions; it does not expose an operator-configurable browser URL/domain/path denylist or warnlist, and the proposed config keys would not be accepted by the strict runtime schema.

## Best Possible Solution

Keep #39097 open. The best implementation is a maintainer-approved browser URL policy surface, likely browser.denylist or browser.blockedUrls plus optional warnlist, wired through typed config, strict zod/generated schema/help, browser config resolution, shared navigation policy options, managed Playwright, existing-session Chrome MCP, raw CDP fallback, clear user-facing errors or warnings, docs, and focused tests for exact host, wildcard domain, redirect, and path-pattern behavior.
