## Summary

Keep open. Current `main` has partial call-level model observability through diagnostic events and OTEL export, but the plugin hook contract still does not expose the requested `before_model_call` / `after_model_call` lifecycle hooks, and Codex docs still describe `llm_input` / `llm_output` as higher-level adapter observations rather than real provider-call boundaries.

## Evidence

- **Requested hook names are not in the public plugin hook contract:** `PluginHookName` includes `llm_input`, `llm_output`, `before_tool_call`, and `after_tool_call`, but not `before_model_call` or `after_model_call`. The typed handler map likewise has no call-level model hook entries.
