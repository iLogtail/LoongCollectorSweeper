## Summary

Keep open. Current `main` still exposes `openclaw sessions` plus `openclaw sessions cleanup`, but not the requested `sessions rm` / `sessions clear` CLI surface. The underlying gateway and UI already support session deletion, so this remains a real, actionable CLI gap rather than a ClawHub/plugin-scope request. The reporter’s linked PR #35735 may still be useful, but there is no evidence in `f00d65a` that it landed.
