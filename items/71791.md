## Summary

Keep PR #71791 open. Current main still records WhatsApp pending group history with only `senderJid` and filters supplemental group-history allowlists using only that cached JID, while inbound messages already carry `senderE164`. The PR remains a narrow WhatsApp plugin bug fix with a targeted regression test for the LID sender plus E.164 allowlist case; no provided related item supersedes this exact work.

## Best Possible Solution

Keep PR #71791 open for maintainer review and normal validation. If CI passes, the best path is to land the focused WhatsApp plugin fix: preserve optional `senderE164` in cached group history, compare it during supplemental-context allowlist filtering, and keep the new LID plus E.164 regression test.
