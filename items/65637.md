## Summary

Keep PR #65637 open. Current main still serializes `/config show` replies from the raw cloned parsed config, so the linked leak is not implemented on main and the PR remains a useful focused fix. The existing redaction pipeline and gateway config handlers support the proposed direction, but the bot-review note about the PR tests not exercising `sensitive: true` schema hints should be addressed before landing.

## Best Possible Solution

Keep #65637 open and have the author or a maintainer land the same core fix: load runtime config schema hints in `handleConfigCommand`, pass the parsed config through `redactConfigObject`, use the redacted object for both full and path-specific `/config show`, and add regression tests that cover both heuristic and `sensitive: true` schema-hint redaction.
