## Summary

Keep PR #64257 open. Current main still does not ignore `docker-compose.sandbox.yml`: `.gitignore` only has the override and extra compose overlays, while the Docker setup path still writes `docker-compose.sandbox.yml` when `OPENCLAW_SANDBOX` is enabled. The related ClawDock reports track compose-loading behavior, not this generated-file ignore rule, so they do not supersede this narrow PR.

## Best Possible Solution

Merge or independently apply the narrow `.gitignore` addition for `docker-compose.sandbox.yml` next to the existing local compose overlays. No product change or ClawHub/plugin path is needed; this belongs in repository hygiene because `scripts/docker/setup.sh` generates the file in the repo root.
