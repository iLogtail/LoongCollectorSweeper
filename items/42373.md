## Summary

Current main does not implement the requested `costCurrency` configuration or an equivalent currency-display path. The model/provider config contract still treats model costs as USD-denominated numeric rates, the strict schema has no currency field, and the UI/CLI formatting paths still hardcode dollar output. This should stay open rather than be closed as implemented or moved to ClawHub because the missing behavior is in core config/schema/display surfaces that a plugin cannot currently override cleanly.
