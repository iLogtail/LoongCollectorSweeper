## Summary

Keep #42373 open. Current main still has no `costCurrency` provider config surface, the model cost type documents prices as USD per million tokens, the strict provider schema only accepts the existing provider fields, and user-facing cost formatters still hardcode `$` in status, reply usage, CLI, and UI usage paths.

## Best Possible Solution

Keep the issue open and implement a typed currency contract rather than only swapping symbols. A good path is to add a validated `costCurrency` or model-cost currency field, define whether stored `cost` values are provider-native or normalized USD, thread the resolved currency through `formatUsd`/`formatCost` call sites, cover session status, `/usage cost`, gateway CLI usage-cost, and UI usage dashboards with tests, then update provider/config docs.
