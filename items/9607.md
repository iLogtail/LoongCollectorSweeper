## Summary

Keep #9607 open. Current main still ships the bundled Himalaya skill with the reported wrong move/copy examples, the message-composition export tip still omits the required message ID, and the broader requested email-formatting/html/plain-text/common-mistakes/save-copy guidance remains absent. The linked PRs do not fully supersede it: #9610 is closed unmerged, #54032 is a narrow three-line syntax/export fix, and #59522 only covers related v1.1/provider guidance.

## Best Possible Solution

Keep #9607 open until a broader Himalaya skill docs fix lands or a maintainer explicitly narrows the issue. A complete fix should update `skills/himalaya/SKILL.md` and `skills/himalaya/references/message-composition.md`, add or intentionally remove the missing HTML-email reference, document plain-text versus HTML expectations, correct the CLI examples against Himalaya v1.1.x, and include the save-copy/BINARY caveat if that upstream limitation remains relevant. PR #54032 can be reused for the narrow syntax/export portion, but it does not close the whole issue.
