## Summary

Keep open. Current main at b67d9bf7f06f4c655b1e0869395056c7d631abc2 still lacks the requested Telegram processing-indicator message lifecycle. Telegram has adjacent feedback mechanisms: typing, ack/status reactions, preview-streaming tool-progress text, and native-command progress placeholders. None provide a configurable standalone ⌛️ message sent immediately for normal inbound Telegram agent turns, deleted after final reply, and edited on failure. The related PR attempts #43021 and #43720 were closed unmerged, and the latest issue discussion already records a maintainer/Codex review keeping this open.

## Best Possible Solution

Keep the issue open. The best path is an opt-in implementation in the bundled Telegram plugin: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its message id through dispatch, delete it after successful final delivery, and edit or replace it on dispatch failure. Reuse the native-command progress placeholder and preview cleanup code as prior art, with focused Telegram dispatch tests for success, error, preview-streaming, group/topic, and cleanup behavior.
