## Summary

Keep open. Current `main` still persists session transcript headers as UTC `...Z` strings and still uses plain `SessionManager.open(...)` append paths without the local-offset wrapper proposed in PR #50300, so the reported behavior is not implemented on `main`. The PR discussion also contains unresolved correctness/robustness concerns, so this is not a conservative auto-close candidate.
