## Summary

Current main still has the old explicit-opt-in behavior. Chat-context background execs with empty successful output are not notified by default, and PR #39726 remains a focused, relevant change that also addressed the review concerns in the provided discussion.

## Best Possible Solution

Keep PR #39726 open or rework it onto current main: default `notifyOnExitEmptySuccess` from `resolveGatewayMessageChannel(messageProvider)` only when the config value is unset, preserve explicit `false`, keep generic provider labels like `openai` and `heartbeat` silent, add the PR's regression cases, and update the changelog/docs if maintainers accept the product behavior.
