## Summary

Keep PR #39726 open. Current main still requires `notifyOnExitEmptySuccess` to be explicitly true, so chat-context empty successful background execs remain silent by default. The PR is a focused, still-relevant behavior change, and the provided PR discussion shows its latest head addressed the explicit-opt-out and generic-provider review concerns.

## Best Possible Solution

Keep PR #39726 open or rebase/rework it onto current main. The best implementation is the narrowed PR behavior: default empty-success notifications only for real reply-capable chat channels when the config value is unset, preserve explicit `false`, keep generic provider labels such as `openai` silent, and land the regression coverage with any accepted changelog/docs update.
