## Summary

Keep #38775 open. Current main still has usage reporting for Anthropic OAuth utilization, but request-time auth-profile selection remains explicit-order/round-robin/lastUsed plus reactive cooldown failover. There is no `auth.balancing` schema, usage-aware selector, proactive utilization circuit breaker, or async plugin API sufficient to implement this externally across Gateway request paths.

## Best Possible Solution

Keep #38775 open for maintainer/product decision. A strong implementation would add first-class usage-aware auth-profile routing across Gateway request paths: provider-owned Anthropic usage parsing, cached per-profile headroom state, proactive utilization thresholds/circuit breakers, shadow accounting for concurrent requests, fallback to current reactive rotation for profiles without usage data, `auth.balancing` schema/help/docs, and regression tests for utilization-based profile choice plus cooldown interaction. If maintainers prefer externalization, first add a documented async plugin API for pre-request profile selection and usage-cache integration.
