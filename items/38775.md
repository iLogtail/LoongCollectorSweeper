## Summary

Keep #38775 open. Current main still has ordered/round-robin auth profile selection, reactive cooldown/failover, and Anthropic usage snapshot reporting, but not the requested usage-aware multi-account routing strategy or `auth.balancing` configuration surface.

## Best Possible Solution

Keep #38775 open for maintainer/product decision. A strong implementation would add first-class usage-aware auth-profile routing for Gateway-wide request paths, likely combining provider-owned Anthropic usage parsing with a core-owned pre-request selector, cached usage state, config/schema/docs, and regression tests for routing by utilization/headroom plus fallback to current reactive behavior for profiles without usage data.
