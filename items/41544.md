## Close Comment

Closing after Codex review. Current `main` already covers the behavior this PR was trying to restore: streamed OpenAI-compatible requests now always send `stream_options: { include_usage: true }` in `src/agents/openai-transport-stream.ts:1686`, and streamed `usage` chunks are parsed into persisted token totals in `src/agents/openai-transport-stream.ts:1266` and `src/agents/openai-transport-stream.ts:1741`. The transport tests also lock this in for custom providers and local backends like llama.cpp in `src/agents/openai-transport-stream.test.ts:1404` and `src/agents/openai-transport-stream.test.ts:1432`. On current `main` (`a53fea39050beba21ac5468d876c37323cfdd2e6`), the zero-usage failure described here is already implemented well enough, so this PR is no longer needed.
