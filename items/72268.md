## Summary

Keep PR #72268 open. Current main still uses the fragile `[^)]*` metadata regex, while current gateway code can emit nested denial metadata such as `approval-timeout (allowlist-miss)`. That means the PR addresses a real remaining bug rather than duplicating or superseding #72148, which is about the resume-failed user-facing prefix.

## Best Possible Solution

Keep this PR open for maintainer review and normal CI. The best code path is to replace the current regex-only parser with a balanced metadata parser and keep the added nested denied/finished plus followup-routing regression tests. Coordinate with #72148 only for overlapping fallback-message expectations; do not close this as a duplicate of that prefix-only fix.
