## Summary

Keep #72268 open. Current main still uses the fragile `[^)]*` exec approval metadata regex, while the gateway can emit `Exec denied` metadata containing nested parentheses via `approval-timeout (allowlist-miss)`. That payload still fails denied parsing and can route through the generic completion/direct-output path. The related #72148 is adjacent but not a replacement; it addresses resume-fallback wording, not the parser misclassification.

## Best Possible Solution

Keep the PR open or recreate its core change as a narrow mainline fix: replace the regex parser in `src/agents/exec-approval-result.ts` with balanced metadata parsing for `Exec denied` and `Exec finished`, add nested-parentheses regression tests in `src/agents/exec-approval-result.test.ts` and `src/agents/bash-tools.exec-approval-followup.test.ts`, and remove the unrelated generated plugin SDK baseline churn unless it is proven necessary.
