## Summary

Keep open. Current `main` still routes iMessage inbound text to raw `BodyForAgent`, and the PR as proposed wraps the deprecated envelope `Body` path instead of the model-bound `UntrustedContext` path. That matches the review discussion: the bug remains plausible on `main`, but this PR is not the right fix to close out conservatively.

## Evidence

- **Current iMessage context still sends raw model body:** `buildIMessageInboundContext` builds an envelope-shaped `Body` but separately sets `BodyForAgent` to raw `decision.bodyText`, with no `UntrustedContext` entry for iMessage inbound text on current `main`.

A wrapper applied only to `Body` does not change the normal model-bound text when `BodyForAgent` is already set.

## Risks / Open Questions

- Closing this PR would discard a still-plausible iMessage prompt-injection hardening gap on current `main` without a landed replacement.
- Merging this PR as-is would give a false sense of security because it targets deprecated `Body` formatting rather than the prompt path that uses `BodyForAgent` and `UntrustedContext`.
- I could not run `pnpm`-based tests in this environment because `pnpm` is unavailable, so this review is based on static code and provided PR-discussion evidence.
