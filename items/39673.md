## Summary

Keep #39673 open. Current main still has a direct gateway install path that can fail with the same user-systemd bus-unavailable class when the fallback cannot recover, and the broad fallback predates the reported 2026.3.7 failure. Related #63561 is useful adjacent context, but it does not fully replace this locale/no-medium install report.

## Best Possible Solution

Keep #39673 open. The maintainer path is to make direct `openclaw gateway install` handle unavailable Linux user-systemd consistently: recover when safe by deriving or repairing the user bus environment or targeting the correct user scope, and when recovery is impossible fail with explicit headless/Linux remediation hints in the install path. Add a regression test where the real install path, not just the preliminary `isLoaded` probe, sees `systemctl --user unavailable: Failed to connect to bus` / no-medium.
