## Summary

Keep #39673 open. Current main has systemd user-bus classification and machine-scope fallback, but the direct `openclaw gateway install` path can still surface the same `Gateway install failed: Error: systemctl --user unavailable: Failed to connect to bus` failure when the user systemd scope remains unreachable. The generic fallback was already documented before the reported 2026.3.7 failure, so it is not enough evidence to close this as implemented.

## Best Possible Solution

Keep #39673 open. The best fix is to make direct `openclaw gateway install` handle unavailable user-systemd consistently: either recover when possible by repairing/deriving the user bus environment or using the correct machine user scope, or fail with the same explicit headless/Linux remediation used by onboarding/status. Add a regression test for direct gateway install with `systemctl --user unavailable: Failed to connect to bus`/No medium so this path is covered.
