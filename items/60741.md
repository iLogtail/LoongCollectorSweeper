## Summary

Keep #60741 open. Current main has partial session-management infrastructure: Control UI can list existing sessions, switch to them, label them, and delete them, and the gateway has a `sessions.create` API. However the requested WebChat topic-session workflow is still not implemented: the chat “new session” control still sends `/new`, which is documented/tested as resetting the same session rather than creating a distinct selectable WebChat conversation, and there is no Control UI call to `sessions.create`.

## Best Possible Solution

Keep the issue open and implement the missing Control UI/WebChat product flow: add a chat-adjacent session list/sidebar or equivalent selector, create new topic sessions via `sessions.create` instead of `/new` reset, keep old transcripts/selectable rows intact, and reuse the existing label/delete/navigate APIs already present in the Sessions tab. This belongs in OpenClaw core Control UI, not ClawHub, because the API exists and the gap is first-party WebChat UX wiring.
