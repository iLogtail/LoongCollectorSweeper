## Summary

Keep open. Current main still has the exact Slack DM preview-streaming gate that PR #56544 changes, and the current tests still assert that top-level DMs without a thread stay off. The PR rationale is plausible because the draft preview sender posts to the target with an optional threadTs, but current Slack docs still describe top-level DMs as not showing thread-style preview, so this needs maintainer review or merge/adaptation rather than automated cleanup.

## Best Possible Solution

Keep PR #56544 open for maintainer handling. If the product intent is to support draft post-and-edit previews in flat Slack DMs, adapt or merge the PR by removing the DM threadTs gate, updating the affected eligibility tests, and clarifying Slack streaming docs to distinguish native thread streaming from draft preview streaming. If the documented no-preview flat-DM behavior is intentional, close with an explicit maintainer decision rather than automated cleanup.
