## Summary

Keep open. Current main still has no per-agent `tools.selfDeny`, `delegateOnly`, or equivalent delegate-only policy. Tool policy remains a narrowing pipeline: per-agent deny removes a tool before subagent policy is applied, and `tools.subagents.tools` cannot add it back. Related #50038 was only an umbrella that points back to this issue for the remaining delegate-only gap, while #59225 tracks a different per-agent schema-parameter filtering request.

## Best Possible Solution

Keep #44253 open for core design. A good implementation would add a documented per-agent delegate-only/self-deny contract to config types, strict zod schema, generated schema/help, and docs; apply it after normal allow/deny resolution for the agent's own model tool list; preserve the resolved eligible pool for subagent filtering; keep `deny` absolute; and add focused tests for an orchestrator that cannot see/use `exec`/`write`/`edit` while its spawned workers can.
