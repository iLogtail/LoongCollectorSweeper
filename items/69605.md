## Summary

Keep #69605 open. Current main still has the layout condition described by the report: assistant bubbles render message-level copy/expand actions, the CSS has a `.chat-bubble.has-copy` padding rule to reserve space, but `renderAssistantBubble` never adds `has-copy` to the bubble class list. The code-block copy handler itself copies `dataset.code`, so the reported full-message copy behavior remains consistent with the related PR discussion that the message-level `Copy as markdown` button overlaps the code-block copy target. Related PR #69634 is still open and directly targets this issue; closed PRs #69635 and #69689 were not merged.

## Best Possible Solution

Keep #69605 open until a focused Control UI fix lands. The likely path is to apply `has-copy` to assistant chat bubbles whenever copy/expand actions render, preserve the existing code-block clipboard handler, and add a grouped-render regression test that locks the spacing contract between `.chat-bubble-actions` and code-block copy controls. PR #69634 is the current canonical open implementation candidate.
