## Summary

Keep #69605 open. Current main still renders assistant bubble-level copy actions without applying the existing `has-copy` padding class, while `.chat-bubble-actions` remains absolutely positioned over the top-right of the bubble. The code-block copy handler itself copies only `data-code`, so the reported “entire message” clipboard result is still consistent with the message-level `Copy as markdown` button being hit by the overlap. Related PR #69634 tracks the same focused fix but is still open; #69689 and #69635 are closed unmerged and do not prove the issue is fixed on main.

## Best Possible Solution

Keep the issue open until a focused Control UI fix lands. The likely fix is to add `has-copy` to assistant `.chat-bubble` classes whenever `canCopyMarkdown` or `canExpand` is true, preserve the existing code-block copy handler, and add a grouped-render regression test proving assistant bubbles with actions get the spacing class while bubbles without actions do not. PR #69634 can be revived or superseded with that patch against current main.
