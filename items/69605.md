## Summary

Keep #69605 open. Current main still renders assistant message-level copy/expand actions without applying the existing `has-copy` spacing class to the `.chat-bubble`, so the CSS intended to keep those floating actions away from code-block copy controls is inactive. The code-block copy handler itself copies `data-code`, but the reported overlap/mis-click path remains unaddressed on main. Linked PR #69634 is the active focused fix candidate; closed PR #69689 proposed the same root cause but was not merged.

## Best Possible Solution

Keep the issue open until a focused UI fix lands. The likely fix is to add `has-copy` to assistant `.chat-bubble` when `hasActions` is true, then add a regression test in `ui/src/ui/chat/grouped-render.test.ts` proving assistant bubbles with copy/expand actions get the spacing class and bubbles without actions do not.
