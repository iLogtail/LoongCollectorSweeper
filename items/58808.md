## Summary

Keep PR #58808 open. Current main still lacks the requested trusted sender context on `ChannelAgentToolFactory`, while adjacent channel message-action discovery and the core `message` tool already receive `requesterSenderId` and `senderIsOwner`. This is a real public plugin SDK seam gap, not a ClawHub-only implementation item or something already implemented on main.

## Best Possible Solution

Keep the PR open or land an equivalent core SDK change: extend `ChannelAgentToolFactory` params with optional trusted `requesterSenderId` and `senderIsOwner`, pass those fields from `createOpenClawCodingTools` into `listChannelAgentTools`, and add a narrow regression test using a fake channel plugin whose `agentTools` factory records the received context.
