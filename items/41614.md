## Summary

Keep open. Current `main` does not implement this fix: `src/cron/service/store.ts` still reuses the in-memory store with no mtime or external-change check on normal paths. The PR body is partly stale because cron runtime state now lives in `jobs-state.json`, but that makes closure less appropriate, not more: the underlying external-edit reload gap still exists and the proposed patch would need refresh for the split store format.

## Evidence
