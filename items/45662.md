## Summary

Keep open. Current `main` still routes proactive IRC sends through a transient client that is created without joined channels, so the reported channel-send failure mode remains plausible. The PR is not implemented on `main`, and the review thread also raised unresolved correctness concerns about the proposed in-memory active-client approach.

## Evidence

- **Current main still uses transient send path:** `sendMessageIrc` uses `opts.client` when present, otherwise opens a fresh IRC connection, sends `PRIVMSG`, and immediately quits. There is no active-client registry or monitor-client reuse on `main`.
