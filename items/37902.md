## Summary

Keep open. Current main does not implement the requested `agents.defaults.subagents.minRunTimeoutSeconds` floor, and the linked fix PR #38048 was closed unmerged. The current timeout resolver still lets an explicit `sessions_spawn.runTimeoutSeconds` override the configured default, so the reported undercut behavior remains possible.
