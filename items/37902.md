## Summary

Keep open. Current main still has no configured minimum or lock for subagent run timeouts: `agents.defaults.subagents.runTimeoutSeconds` remains only a fallback when `sessions_spawn.runTimeoutSeconds` is omitted, and explicit model-provided values still win even when they undercut the configured default. Related PRs #37973 and #38048 attempted the requested `minRunTimeoutSeconds` fix but are closed unmerged; #60211 is a different timeout/fallback issue.

## Best Possible Solution

Keep #37902 open until OpenClaw implements a maintainer-approved subagent timeout constraint. A narrow fix should add an explicit config contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or an authoritative lock option, update the TypeScript config type, strict zod schema, generated schema/docs, and clamp model-supplied `sessions_spawn.runTimeoutSeconds` in the shared timeout resolver before subagent registration. Regression tests should cover omitted timeout fallback, explicit values above the floor, explicit values below the floor, zero/no-timeout compatibility, unset compatibility, and schema acceptance/rejection. Closed PRs #37973 and #38048 are useful prior art but do not close the issue on current main.
