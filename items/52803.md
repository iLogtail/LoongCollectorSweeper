## Summary

Keep open. Current main has useful primitives and a Sessions UI, but it does not implement the requested Control UI orchestration surface: hierarchical agent/session/subagent tree, active-first status filters, bulk kill/steer/retry/cleanup, per-row model/elapsed/status display, or virtualized/scalable rendering. The closest related open issue, #46807, covers a narrower live sub-agent mission board and does not clearly supersede this broader hierarchy/bulk/scalability request.

## Evidence
