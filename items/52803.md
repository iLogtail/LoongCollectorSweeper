## Summary

Keep open. Current main has useful pieces around session metadata, subagent controls, and the `/tasks` chat/CLI surfaces, but the requested Control UI orchestration board is not implemented: the browser UI still exposes a flat Sessions table with pagination and bulk delete only, not a hierarchy, active-first mission board, bulk kill/steer/retry, or virtualized task tree. Related #46807 and #54226 cover narrower surfaces and do not supersede this broader Control UI request.

## Best Possible Solution

Keep the issue open and implement it as a Control UI orchestration surface backed by the task ledger plus session/subagent metadata: a tree or virtualized grouped list for Agent -> Session/Task -> Subagent, active/recent/history filters, status/model/elapsed/last-update columns, and bulk cancel/steer/retry/cleanup actions. Reuse existing `sessions.list`, task registry, `/tasks`, `openclaw tasks`, and subagent control semantics, adding gateway methods/tests where current browser APIs are too narrow.
