## Summary

Keep #23718 open. Current main still does not implement `heartbeat.model: { primary, fallbacks }`: heartbeat config validation, public config types, generated schema, docs, runtime option forwarding, and tests remain heartbeat-model string-only. Existing model fallback machinery and per-agent `agents.list[].model.fallbacks` support are real, but they do not expose a heartbeat-specific fallback chain on `heartbeat.model`. Related PR #40875 is open and docs-only, so it does not supersede the runtime feature request.

## Best Possible Solution

Keep the issue open and implement heartbeat-specific fallback-chain support in core. The likely path is to change heartbeat schema and public config types to use `AgentModelConfig`, regenerate config schema/help/docs, resolve heartbeat primary/fallbacks through the existing model-input helpers, pass a heartbeat-specific fallback override into the reply/run path and `runWithModelFallback` while preserving explicit empty-array semantics, and add focused config plus heartbeat runtime tests. PR #40875 can remain docs context but should not be treated as the implementation.
