## Summary

Current `main` already covers the reported restart path and reconstructs cron announce routing from persisted delivery config, including Telegram topic/thread targets, so this issue is implemented on main.

## Close Comment

Closing from a Codex review because current `main` already implements the reported path. Restarted cron now clears stale running markers and replays interrupted recurring jobs on startup (`src/cron/service/ops.ts:93`), and isolated cron announce delivery rebuilds the target from persisted job config on each run (`src/cron/isolated-agent/run.ts:329`). Telegram explicit targets are parsed back into `to` + `threadId` through the plugin seam (`extensions/telegram/src/channel.ts:404`), and cron delivery resolution preserves explicit thread ids while refusing to leak stale session thread ids onto a different destination (`src/cron/isolated-agent/delivery-target.ts:157`). The shipped `2026.3.28` changelog also calls this out directly under `Telegram/cron topics` (`CHANGELOG.md:1932`).
