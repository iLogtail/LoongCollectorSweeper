## Summary

Keep open. Current main has partial inline media support for Control UI chat, especially managed/generated images and MEDIA-driven image/audio/video attachments, but the full #64607 scope is not implemented. Web/link previews are absent, Control UI CSP still blocks remote http(s) image fetches by design, and generated audio is persisted as audio content blocks that the current UI normalizer does not convert into renderable attachments.

## Best Possible Solution

Keep #64607 open but narrow the maintainer path: preserve the shipped managed-image and MEDIA attachment work, then track the remaining core UI pieces explicitly: server-side link preview/unfurling with a CSP/privacy model, renderer normalization for persisted audio/video content blocks, and the already-related tool-result/remote-image media shapes. Link or consolidate with the narrower open follow-ups where they cover the same remaining behavior.
