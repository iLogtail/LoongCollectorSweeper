## Summary

Keep open. Current `main` still awaits auth-profile cooldown writes before rotating profiles on both the prompt-side and assistant-side failover paths, so this PR has not been superseded. The PR also has an unresolved review note about warning logs losing the `failedProfileId` guard, which means it still needs normal review rather than cleanup-closing.
