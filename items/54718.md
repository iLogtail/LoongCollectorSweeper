## Summary

Keep open. #54718 is not implemented on current main and is not clearly superseded by a shipped or canonical replacement. The PR still proposes a meaningful gateway auth behavior change: local shared-secret fallback in trusted-proxy mode and same-host loopback reverse-proxy acceptance. Current main deliberately rejects those paths in source, tests, and docs, while #69066 is an open RFC that references #54718 as future/context work rather than a merged replacement.

## Best Possible Solution

Keep #54718 open for maintainer review. The maintainer path should either adapt the PR to the current fail-closed trusted-proxy policy and #69066 service-identity direction, or explicitly close it with a human product/security rationale if same-host loopback trusted-proxy/token fallback is no longer desired.
