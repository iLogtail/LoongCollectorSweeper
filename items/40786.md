## Summary

Current `main` does not implement `.gitignore`-style exclude patterns for `openclaw backup create`. The shipped CLI and docs expose only coarse-grained scope controls (`--no-include-workspace`, `--only-config`), and the archive writer still sends every planned source path straight into the tarball with no exclude matcher. The issue discussion adds useful prior art, including PR #44288 and the ClawHub `keepmyclaw` workaround, but that does not change the fact that the requested core behavior is still missing on `main`.

## Evidence

- **CLI surface lacks exclude flags:** `registerBackupCommand` defines `--output`, `--json`, `--dry-run`, `--verify`, `--only-config`, and `--no-include-workspace` for `openclaw backup create`. There is no `--exclude`, `--exclude-file`, `--smart-exclude`, or similar flag on current `main`.
  - file: [src/cli/program/register.backup.ts:23](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/cli/program/register.backup.ts#L23)

  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Public docs still document only whole-workspace omission:** `docs/cli/backup.md` lists only `openclaw backup create`, `--output`, `--dry-run --json`, `--verify`, `--no-include-workspace`, and `--only-config`, and its size guidance says to use `--no-include-workspace` for smaller backups. No ignore-pattern or exclude-file behavior is documented.
  - file: [docs/cli/backup.md:13](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/cli/backup.md#L13)
