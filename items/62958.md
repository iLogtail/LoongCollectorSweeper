## Summary

Close: the `openclaw agent` timeout/hang reported on `2026.4.7` lines up with later timeout fixes that shipped after that release, and current `main` now threads explicit CLI run timeouts into the LLM idle watchdog with regression coverage.

## Evidence

- **Release note: explicit run timeouts:** The `2026.4.11` changelog explicitly says agent timeout handling was fixed so the LLM idle watchdog honors explicit run timeouts and slow models can keep working until the configured limit.
  - file: [CHANGELOG.md:899](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/CHANGELOG.md#L899)

  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Release note: slower-model idle window:** The `2026.4.10` changelog also extended the default LLM idle window to `120s`, specifically to help slow models before surfacing a timeout.
  - file: [CHANGELOG.md:957](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/CHANGELOG.md#L957)
  - command: `nl -ba CHANGELOG.md | sed -n '950,960p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **CLI forwards timeout to gateway agent runs:** `openclaw agent` parses `--timeout`, derives a larger gateway RPC budget from it, and sends `timeout: timeoutSeconds` in the `agent` RPC request.
  - file: [src/commands/agent-via-gateway.ts:107](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/commands/agent-via-gateway.ts#L107)

  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Embedded runner now uses per-run timeout for the idle watchdog:** The embedded runner passes `runTimeoutMs` into `resolveLlmIdleTimeoutMs` when the run timeout differs from the configured default, which is the path needed for CLI `--timeout` to affect slow local-model runs.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1653](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/agents/pi-embedded-runner/run/attempt.ts#L1653)

- **Idle-timeout resolution matches the fix:** Current timeout resolution prefers explicit `agents.defaults.llm.idleTimeoutSeconds`, then an explicit run timeout, then `agents.defaults.timeoutSeconds`, and only then falls back to the default idle window.
  - file: [src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:28](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/agents/pi-embedded-runner/run/llm-idle-timeout.ts#L28)
  - command: `nl -ba src/agents/pi-embedded-runner/run/llm-idle-timeout.ts | sed -n '28,58p'`

- **Regression coverage:** Tests now cover inheriting `agents.defaults.timeoutSeconds`, honoring explicit run-timeout overrides, and disabling the idle watchdog when timeouts are effectively unlimited.
  - file: [src/agents/pi-embedded-runner/run/llm-idle-timeout.test.ts:53](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/agents/pi-embedded-runner/run/llm-idle-timeout.test.ts#L53)
