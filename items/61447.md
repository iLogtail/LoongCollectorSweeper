## Summary

Keep open. Current main still does not include the generated compaction summary text or summary token count in the public `after_compaction` plugin hook payload. The summary exists at compaction call sites and is sometimes persisted elsewhere, but the hook contract and all inspected emitters still pass only counts/session metadata, so this is a real additive plugin API request rather than an auto-close candidate.

## Best Possible Solution

Add an additive public hook contract change: extend `PluginHookAfterCompactionEvent` with optional `summaryText` and, if maintainers want it, `summaryTokens`; thread normalized `result.summary` through all `after_compaction` emitters that have it; leave it `undefined` for runtimes such as Codex-native compaction where docs say no summary payload is available yet; update focused hook tests, plugin API surface checks, and hook docs.
