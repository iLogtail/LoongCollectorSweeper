## Summary

Keep open. Current `main` does not implement the outbound sanitization from this PR, and the PR discussion surfaced a real scope question about whether outbound delivery should use broad `stripInboundMetadata()` or the narrower leading-only helper. This is still actionable maintainer work, not conservative close material.

## Evidence

- **Current outbound delivery still parses raw payload text:** `createOutboundPayloadPlanEntry()` calls `parseReplyDirectives(payload.text ?? "")` directly and then formats that parsed text for delivery. There is no `stripInboundMetadata()` or `stripLeadingInboundMetadata()` call anywhere on the delivery path in this file, so echoed inbound metadata blocks are still not sanitized before channel delivery.
