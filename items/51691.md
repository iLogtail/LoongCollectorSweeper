## Summary

Keep open. Current main has shared-session foundations (`dmScope`, `identityLinks`, gateway-backed transcripts, `sessions.send`, and session transcript websocket events), but it still does not implement the requested end-to-end real-time Telegram <-> TUI sync. The TUI only reacts to `chat`, `chat.side_result`, and `agent` gateway events, while transcript/session update events are emitted separately as `session.message` / `sessions.changed`, so Telegram-originated transcript changes are not wired into the TUI as live conversation updates.
