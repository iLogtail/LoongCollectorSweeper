## Summary

Keep issue #70493 open. The issue is recent, has an explicit maintainer comment saying not to close until isolated-session coverage exists, and current main still has Playwright-required browser routes that can return the reported unavailable-runtime error. I found partial routing/proxy work for host, sandbox, node, and existing-session profiles, but not an isolated sub-agent regression test proving `navigate` or `snapshot` works through the reported path.

## Best Possible Solution

Keep the issue open. The best fix is to make the isolated session browser path use the same Playwright-capable runtime as the main gateway, or explicitly delegate browser actions to the host/node browser owner when that is the intended architecture. Add an isolated sub-agent regression test that exercises `browser({ action: "navigate" })` or `browser({ action: "snapshot" })` through the real isolated-session path and proves it does not return the Playwright-unavailable error when the main gateway/browser node supports Playwright.
