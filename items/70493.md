## Summary

Keep issue #70493 open. Current main still has a distinct sandbox/isolated browser path, the Playwright-unavailable fallback remains present on browser routes, and local test coverage verifies pieces of browser tool wiring/proxying but not the requested isolated sub-agent navigate/snapshot path. The issue also has an explicit maintainer comment saying not to close until isolated-session coverage exists.

## Best Possible Solution

Keep the issue open. The best fix is to make isolated/sandboxed sub-agent browser calls use a Playwright-capable runtime or explicitly delegate to the browser-capable host/node, then add a regression test that exercises browser navigate or snapshot from an isolated sub-agent session and proves the Playwright-unavailable fallback is not returned when the main browser capability is available.
