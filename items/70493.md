## Summary

Keep open. The issue has a recent maintainer comment explicitly saying not to close until isolated-session browser coverage exists, and current main still shows Playwright availability as a runtime gate on the relevant browser routes rather than a proven isolated-session path fix.

## Best Possible Solution

Keep #70493 open. The best fix is to make isolated sub-agent browser control use the same host/node Playwright-capable runtime as the main gateway, or reliably stage the bundled browser plugin runtime dependencies for the isolated browser bridge, then add an isolated-session regression that exercises `browser` `snapshot` or `navigate` through a spawned sub-agent and proves the Playwright-unavailable error no longer appears.
