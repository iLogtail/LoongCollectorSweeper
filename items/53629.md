## Summary

Keep #53629 open. Current main has partial Telegram Business plumbing in allowed update constants, but it still lacks explicit `business_connection` / `business_message` handlers and outbound `business_connection_id` routing. The PR is not implemented on main and is not obsolete; it needs revision for the existing review feedback before it can safely land.

## Best Possible Solution

Keep the PR open for contributor or maintainer revision. The best path is to finish Telegram Business support inside the bundled Telegram plugin: register business connection/message handlers, route business messages through the existing inbound pipeline, track `business_connection_id` per Telegram account with disconnect cleanup, inject it only for supported outbound API methods, and add regression tests for multi-account and disconnect cases. Keep #20786 as the canonical feature issue until that lands.
