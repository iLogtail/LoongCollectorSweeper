## Summary

Current main has useful mitigations, but it does not implement the requested lazy/names-only skills prompt. The prompt still carries a skill catalog with locations every turn, uses full name+description+location until the configured character budget is exceeded, and only then downgrades to name+location compact XML. The issue comment correctly identified existing budget knobs, but the remaining request for a lower-token names-only or auto-matched skill prompt remains actionable.

## Best Possible Solution

Keep this open and treat the existing compact fallback as partial mitigation. The best path is to design a skills prompt mode in `src/agents/skills/workspace.ts` and config schema that can use a names-only or auto-selected catalog while preserving deterministic ordering, safe `SKILL.md` read paths, docs, and regression tests for large skill sets.

## Risks / Open Questions

- A names-only catalog could reduce skill recall because current routing uses descriptions to decide whether a skill applies.
- Changing the skills prompt format must preserve deterministic ordering and prompt-cache stability, and should update docs plus tests around budget fallback and per-agent skill limits.
