## Summary

Keep #53982 open. Current main still does not implement either requested outcome: the Nextcloud Talk setup docs/onboarding text omit `--feature response`, and the outbound 401 path still reports only a bot-secret authentication error. The related PRs in the provided context (#53987 and #56043) were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open and land a narrow Nextcloud Talk fix: update `docs/channels/nextcloud-talk.md` and `extensions/nextcloud-talk/src/setup-surface.ts` to show `--feature webhook --feature response --feature reaction`, add a troubleshooting note for HTTP 401 after successful webhook receipt, and preferably add focused tests around the setup wizard text and/or 401 error mapping.
