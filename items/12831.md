## Summary

Keep #12831 open. Current main has useful tool-event plumbing and live tool cards, but it still does not provide the requested first-class activity observer mode: Control UI renders tool activity inside the chat log, tool output is still tied to `verbose=full`, verbose remains session-scoped, and the TUI still gates tool visibility on `/verbose`. Related #46604 is a narrower open web split-panel request, not a full superseding tracker for the broader observer-mode request.

## Best Possible Solution

Keep #12831 open as the broad product/UX tracker for a first-class activity observer mode. Build on the existing `tool-events` and `session.tool` plumbing, but add a dedicated Control UI activity panel with independent scrolling, connection-scoped observer controls or a documented subscription workflow, output visibility that is not coupled to `/verbose full`, TUI split/observer support, and focused tests/docs. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly make it canonical.
