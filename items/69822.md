## Summary

Keep #69822 open. Current main does not implement the requested socket.drain teardown path for message-only session subscribers, and the PR is a focused replacement for the earlier closed dirty PR rather than already merged or superseded work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to evaluate the focused gateway change on its merits: add socket.drain to the event contract/scope guard, expose the message-subscriber lookup through GatewayRequestContext, route teardown drains to both sessions.subscribe and sessions.messages.subscribe connections, preserve normal dropIfSlow behavior for non-teardown traffic, and keep targeted regression coverage in src/gateway/session-message-events.test.ts.
