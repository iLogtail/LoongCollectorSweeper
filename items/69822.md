## Summary

Keep PR #69822 open. Current main does not implement the PR's central `socket.drain` behavior: the event is not registered, unknown events are rejected by gateway event scope checks, teardown broadcasts still only target `sessions.subscribe` clients, and message-subscriber IDs are not exposed through `GatewayRequestContext`. The related issue #63374 is closed in the provided context, but the observable fix is not present on current main, and prior PR #65539 was an unmerged predecessor rather than a canonical replacement for this focused PR.

## Best Possible Solution

Keep #69822 open for maintainer review. The best path is to either land a refined version of the PR or implement the same core gateway fix on main: register `socket.drain` as a scoped gateway event, expose message-subscriber connection IDs through the gateway request context, emit teardown drains for `sessions.reset`/`sessions.delete` while preserving `sessions.changed`, and keep focused regression coverage in `src/gateway/session-message-events.test.ts`.
