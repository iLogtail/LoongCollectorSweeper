## Summary

Keep open. Current main has selected-vs-runtime model display and OpenClaw fallback notices, but it does not implement this issue's central request: capturing the actual backend model returned by an OpenAI-compatible routing proxy and exposing it as resolvedModel/resolvedProvider/fallbackUsed in session_status, GatewaySessionRow, persisted usage, or agent runtime metadata. The Ryce comment and KeepMyClaw workaround are useful evidence that this is still a core observability/session-state request, not something already solved by the current in-band OpenClaw surfaces.

## Evidence

The stream processing handles response id, usage, status, deltas, and finish reason, but the source search found no `response.model` or `chunk.model` capture path in this transport.
