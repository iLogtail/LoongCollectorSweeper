## Summary

Keep open. Current main already supports per-agent `tools` and `tools.exec` overrides plus per-agent sandbox/workspace configuration, so the validator-rejection part of the report is partly outdated. But the requested command-variant blocking is still not implemented: OpenClaw documents `gateway.nodes.denyCommands` as exact command-name matching only, not shell-text filtering, and multi-agent workspaces are only a default cwd unless sandboxing is enabled. That leaves a real multi-tenant hardening gap.
