## Summary

Keep open. Current main still has the Telegram health-status path this PR targets: health builds a minimal snapshot without running or tokenSource, Telegram forwards that snapshot unchanged, and the shared status helper defaults the missing fields to running:false and tokenSource:"none". The PR also responded to the Greptile review by adding regression tests, so it remains useful work rather than cleanup-close material.

## Best Possible Solution

Keep the PR open and either rebase/merge a maintained version of it or fold its tested fallback into a broader Telegram health fix. The implementation should make health summaries populate tokenSource from the resolved account and avoid defaulting a successful Telegram health probe to running:false, with regression tests covering successful and failed probes.
