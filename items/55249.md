## Summary

Current main has session-label support in gateway/session APIs, but the `openclaw sessions` CLI still does not expose labels in text or JSON output and does not provide a label-setting subcommand. The issue is only partially implemented, so there is not strong evidence to close it.

## Evidence

- **CLI surface still lacks any label command:** The `sessions` CLI registers the base `openclaw sessions` command plus only the `cleanup` subcommand. There is no `label` subcommand or other CLI entrypoint to set a session nickname.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/cli/program/register.status-health-sessions.ts#L124)
  - command: `nl -ba src/cli/program/register.status-health-sessions.ts | sed -n '110,210p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **CLI listing/output does not include labels:** `sessionsCommand` builds rows from `toSessionDisplayRows()`, emits JSON by spreading those rows, and renders a table with columns `Agent Kind Key Age Model Tokens Flags`. No label/display-name column is included.
  - file: [src/commands/sessions.ts:146](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/commands/sessions.ts#L146)
  - command: `nl -ba src/commands/sessions.ts | sed -n '120,240p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Display row model omits label entirely:** `SessionDisplayRow` has no `label` field, `toSessionDisplayRows()` copies no label from the session entry, and `formatSessionKeyCell()` formats only the opaque key.
  - file: [src/commands/sessions-table.ts:5](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/commands/sessions-table.ts#L5)
  - command: `nl -ba src/commands/sessions-table.ts | sed -n '1,120p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Programmatic session labels already exist:** Gateway `sessions.patch` accepts and persists `label`, while gateway session rows include `label`; tests assert that a patched label appears in `sessions.list` and as `displayName`. This shows partial implementation, not a stale request.
  - file: [src/gateway/sessions-patch.ts:219](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/gateway/sessions-patch.ts#L219)
  - command: `nl -ba src/gateway/sessions-patch.ts | sed -n '214,244p'`

  - file: [src/gateway/session-utils.ts:1381](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/gateway/session-utils.ts#L1381)
