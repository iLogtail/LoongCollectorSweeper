## Summary

Keep open. Current main has partial session-label infrastructure through the gateway, agent tools, and Control UI, but the issue's central CLI workflow is still missing: there is no `openclaw sessions label` subcommand, and `openclaw sessions` text/JSON output does not carry or display session labels.

## Best Possible Solution

Keep the issue open but narrow the implementation scope: add a `openclaw sessions label <session-key> <label>` command, include `label` and probably `displayName` in `SessionDisplayRow`, show a compact Label column or label-preferred key display in `openclaw sessions`, include labels in `--json`, and add focused CLI tests plus docs for `docs.openclaw.ai/cli/sessions`. Reuse the existing `SessionEntry.label` and `sessions.patch` validation semantics rather than inventing a second label store.
