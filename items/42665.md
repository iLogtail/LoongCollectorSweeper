## Summary

Keep open. Current main partially addresses the report by preserving full IDs in structured details, but the model/user-facing `memory_forget` candidate output still prints only 8-character IDs while `memoryId` deletion rejects anything that is not a full UUID. The related PR #42765 is useful prior work, but it was closed unmerged, so it does not supersede the issue.

## Evidence
