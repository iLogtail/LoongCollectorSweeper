## Summary

Keep open. Current main still leaves the agent-visible targeted deletion workflow broken: `memory_forget(memoryId)` requires a full UUID, but `memory_forget` query-mode candidate text only shows the 8-character prefix, and tool-result `details` are stripped before provider conversion. `memory_recall` now carries full IDs in `details`, but its visible text has no ID, so that does not solve the user-facing issue for agents.

## Best Possible Solution

Keep the issue open and fix `memory-lancedb` so the full UUID needed by `memory_forget(memoryId)` is present in agent-visible `content` for forget candidates, and preferably in recall output as well. Add focused regression coverage that stores/searches candidate memories and asserts the visible tool text contains full UUIDs, not only `details`.
