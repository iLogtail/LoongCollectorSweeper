## Summary

Keep open. Current `main` does not already implement the requested duplicate reset-hook suppression, and the submitted PR still has unresolved correctness concerns from review, so there is not strong evidence to conservatively close it.

## Evidence

- **Main still emits reset hooks without persistent dedupe:** `emitResetCommandHooks` builds the hook event and immediately awaits `triggerInternalHook`, then only marks `command.resetHookTriggered` afterward. There is no process-level or per-session dedupe guard in this function on current `main`.

  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Fallback guard is only a mutable command flag:** The fallback path skips only when `command.resetHookTriggered` is already set. That prevents some same-command re-emissions, but it is not the bounded duplicate suppression requested in the PR body for repeated re-entry keyed by prior session.
