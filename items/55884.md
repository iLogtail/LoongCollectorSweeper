## Best Possible Solution

Keep the PR open for maintainer judgment, but require a revised patch before merge: scope dedupe to the actual re-entry/request window, mark the key before the first await if a key-based guard is retained, and add focused tests for same-prior-session duplicate suppression plus valid later reset allowance.
