## Summary

Keep #14785 open. Current main still sends full tool parameter schemas for the effective tool set, the `message` tool still exposes broad cross-channel parameters, and `cron` still embeds tutorial-style guidance in the always-sent tool description. Existing policy controls and partial message action scoping are useful mitigations, but they do not implement the issue's requested channel-aware parameter pruning, lazy/stub schemas, tutorial extraction, or mega-tool splitting.

## Best Possible Solution

Keep #14785 open as the parent performance tracker. The best path is incremental measured work: avoid sending schema payloads when no effective tools remain, prune `message` parameters by active channel/capability without breaking cross-channel jobs, move the long `cron` tutorial out of the always-sent description, and design a compatible lazy or slim schema mode for plugin and declarative tool growth.
