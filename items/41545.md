## Summary

Keep open. Current `main` still contains the reported behavior in the Overview Gateway Access form: editing the WebSocket URL explicitly resets `token` to `""` whenever the URL value changes, so the issue is not implemented/fixed on `main` and should not be conservatively closed.

## Evidence

- **Current Overview handler still clears token on URL edit:** In `ui/src/ui/views/overview.ts:266-272`, the WebSocket URL `@input` handler calls `props.onSettingsChange({...props.settings, gatewayUrl: v, token: v.trim() === props.settings.gatewayUrl.trim() ? props.settings.token : ""})`. Any actual URL edit therefore clears the token immediately.
  - file: [ui/src/ui/views/overview.ts:266](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/ui/src/ui/views/overview.ts#L266)
