## Summary

Keep open. Current `main` still treats Feishu `@_all` as a bot mention before checking explicit `mentions`, so the reported group-chat bug remains live and this PR still addresses a real defect rather than obsolete behavior.

## Evidence

- **Current main still short-circuits `@_all` to `true`:** `checkBotMentioned()` returns `true` whenever raw message content contains `@_all`, before consulting the explicit Feishu `mentions` array. That means a broadcast tag is still treated as a targeted bot mention on current `main`.
  - file: [extensions/feishu/src/bot-content.ts:240](https://github.com/openclaw/openclaw/blob/c1133714/extensions/feishu/src/bot-content.ts#L240)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '228,252p'`
  - sha: [c1133714](https://github.com/openclaw/openclaw/commit/c1133714)
- **Group mention gating still depends on that `mentionedBot` flag:** For group chats, Feishu reply policy resolves `requireMention`, then drops the message only when `requireMention && !ctx.mentionedBot`. Because `ctx.mentionedBot` is derived from the buggy `@_all` shortcut, `@所有人` can still bypass mention-gating on current `main`.
  - file: [extensions/feishu/src/bot.ts:479](https://github.com/openclaw/openclaw/blob/c1133714/extensions/feishu/src/bot.ts#L479)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '456,488p'`
  - sha: [c1133714](https://github.com/openclaw/openclaw/commit/c1133714)
- **User-facing docs say Feishu mention gating requires an actual `@mention`:** The Feishu channel docs state `channels.feishu.requireMention: true` means 'require @mention'. The current `@_all` shortcut does not match that documented contract because it is not tied to an explicit bot `open_id` mention.
