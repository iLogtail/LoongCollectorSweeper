## Summary

Keep open. Current `main` does not implement Cmd/Ctrl+Enter send in Control UI: the chat composer still advertises plain Enter-to-send and the key handler sends on `Enter` whenever Shift is not held.

## Evidence

- **Composer placeholder still says Enter sends:** The Control UI chat placeholder is `Message ... (Enter to send)`, which matches the old behavior rather than the requested Cmd/Ctrl+Enter shortcut.
  - file: [ui/src/ui/views/chat.ts:775](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/ui/src/ui/views/chat.ts#L775)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Key handler still sends on plain Enter:** `handleKeyDown` calls `props.onSend()` when `e.key === "Enter" && !e.shiftKey`; there is no `metaKey`/`ctrlKey` requirement in the send path.
  - file: [ui/src/ui/views/chat.ts:1041](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/ui/src/ui/views/chat.ts#L1041)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Cmd/Ctrl is used for search, not send:** The only Ctrl/Cmd branch in the nearby composer handler is `Cmd/Ctrl+F` for search. That reinforces that a Cmd/Ctrl+Enter send shortcut was not wired here.
  - file: [ui/src/ui/views/chat.ts:1030](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/ui/src/ui/views/chat.ts#L1030)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Repo search found no implemented Cmd/Ctrl+Enter send path:** A source search for `Cmd+Enter`, `Ctrl+Enter`, `metaKey`, and `ctrlKey` in the UI found the chat composer still on Enter-to-send and no matching send-shortcut implementation or related tests.
