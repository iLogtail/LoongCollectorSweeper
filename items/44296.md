## Summary

Current main still shows the generic PowerShell onboarding hint instead of a concrete profile path. The referenced fix PR #47333 is closed/unmerged, and the current implementation/test coverage does not include the requested Windows-focused behavior.

## Best Possible Solution

Keep #44296 open. The best fix is to share or export a single profile-path resolver from `src/cli/completion-runtime.ts`, use it in `src/wizard/setup.completion.ts` for PowerShell onboarding notes, provide a concrete dot-source reload command when safe, and add a Windows-focused Vitest case covering `$PROFILE`/known PowerShell profile-path behavior. While there, maintainers should decide whether `installCompletion()` should actually support PowerShell or whether onboarding should avoid claiming install success for that shell.
