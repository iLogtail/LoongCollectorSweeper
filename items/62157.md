## Summary

Keep #62157 open. Current main still suppresses normal final payloads after successful block streaming, so the central reported bug is not implemented on main. The PR targets the right shared code path, but its current patch still needs revision for coalesced streamed chunks before it is safely mergeable.

## Best Possible Solution

Keep the PR open, but revise it before merge: preserve genuinely new post-tool final payloads after successful block streaming while retaining duplicate suppression for exact, coalesced, and directly sent block content. Update `agent-runner-payloads` tests and add a coalescing regression so the fix does not re-send already streamed chunks.

## Risks / Open Questions

- The PR's current `hasSentPayload()`-only direction can duplicate already-streamed content when the block coalescer sends multiple chunks as one outbound block and final payloads arrive as separate items.
- This is shared auto-reply delivery behavior across block-streaming channels, so the fix needs targeted regression coverage for both post-tool final delivery and coalesced duplicate suppression.
