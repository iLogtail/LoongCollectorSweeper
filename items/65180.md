## Summary

Keep #65180 open. Current main still appears to use the default agent main session for local `infer model run` calls and still has the transcript fallback rooted through `resolveSessionTranscriptPath(...)` instead of the active store path's sessions directory. The PR is a focused draft bug fix with tests/canary evidence rather than an obsolete or duplicate cleanup item.

## Best Possible Solution

Keep the PR open for maintainer review or rebasing. The best path is to port the narrow behavior change onto current main: generate an explicit ephemeral `sessionId`/`sessionKey` for default local `infer model run` calls, keep explicit reuse opt-in, make transcript fallback paths use the active store path's sessions directory when available, and add the PR's regression assertions around both behaviors.
