## Summary

Keep open. Current main still has the Nextcloud Talk reaction sender, but it is not wired into the shared `message` tool/action path, so the reported bug remains actionable.

## Best Possible Solution

Implement a Nextcloud Talk message action adapter that advertises `react`, resolves the target room and message id from tool parameters or the current inbound context, validates the emoji/remove semantics, calls `sendReactionNextcloudTalk`, and adds regression coverage proving `message({ action: "react" })` reaches that helper. If reactions are no longer intended for this plugin, remove the advertised capability and docs claim instead, but the current docs point toward wiring the existing helper.
