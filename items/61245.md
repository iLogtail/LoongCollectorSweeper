## Summary

Keep #61245 open. Current main has not implemented the proposed .dockerignore cleanup, but the PR is not merge-ready as-is because its final `test/` exclusion conflicts with current root-context Docker E2E builds that explicitly copy `test/`. This is still an actionable infra cleanup if revised against all Dockerfiles that use the repo root as build context.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: enumerate every Dockerfile built with repo-root context, keep any copied sources available for those builds, drop or scope the `test/` exclusion, and optionally add .dockerignore coverage to `src/docker-build-cache.test.ts` so future context-pruning changes do not break Docker smoke images.
