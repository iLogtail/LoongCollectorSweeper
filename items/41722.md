## Summary

Keep #41722 open. Current main still preserves the macOS Skills settings path that calls gateway `skills.install`, and gateway installer mode still runs declared Homebrew installer metadata on the gateway host. The reported CLT/Homebrew failure is therefore not proven obsolete by the newer ClawHub skill-install flow, because that flow is separate from the dependency-installer path used by the macOS app buttons for bundled skills.

## Best Possible Solution

Keep the issue open and fix the gateway-backed skill dependency installer path. A good maintainer fix would detect the Homebrew/CLT macOS 26 failure, return actionable remediation instead of a raw package-manager error, and/or add a supported fallback or preference escape hatch for bundled skills whose only exposed installer is Homebrew. Regression coverage should target `installSkill` failure classification plus the macOS Skills settings display path.
