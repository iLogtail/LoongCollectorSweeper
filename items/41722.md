## Summary

Keep open. Current `main` still routes the macOS Skills UI through a single preferred installer, defaults that preference to Homebrew when `brew` exists, and does not retry a different installer when the chosen Homebrew command fails. The failure formatter also produces the same `Install failed (exit 1): ...` shape reported in #41722, so there is no strong evidence this regression is fixed on `main`.

## Evidence

- **macOS Skills UI still uses gateway `skills.install`:** The macOS/controller path forwards the selected `installId` directly to `skills.install`; there is no UI-side fallback logic here.
