## Summary

Keep PR #53473 open. Current main still has the older `openclaw doctor` local-embedding guidance and lacks the PR's missing-GGUF regression coverage, so the work is not implemented or obsolete. The PR does need a refresh before merge because its auto-mode `hf:` guidance conflicts with the current runtime contract.

## Best Possible Solution

Refresh the PR against current main: keep the `openclaw doctor` missing-GGUF guidance and focused regression test, change the auto-mode hint to say `provider: "local"` can use the default or `hf:` model while `provider: "auto"` needs an existing local file path, explicitly say Ollama requires `provider: "ollama"`, and remove unrelated formatting churn. FAQ/docs wording should be aligned with the existing public memory configuration reference rather than creating conflicting guidance.
