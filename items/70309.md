## Summary

Keep open. Current main supports saved MCP server definitions and per-run MCP tool discovery, but it still does not provide a generic Control UI MCP connectivity panel, per-server live status, tool-name listing, reconnect/refresh control, or surfaced per-server connection errors. A related open tracker (#64205) covers agent-visible MCP startup diagnostics, but it is narrower and does not supersede this Control UI preflight/debugging request.

## Best Possible Solution

Keep the issue open and implement a core Gateway diagnostics surface, likely `mcp.status` plus a scoped refresh/reconnect action, that reads configured `mcp.servers`, safely probes or reuses MCP runtime state, returns per-server status, tool count/names, and redacted errors, then renders that in the Control UI Infrastructure/MCP area with tests and docs. Coordinate with #64205 so the underlying MCP diagnostic contract is shared instead of duplicating runtime probing logic.
