## Summary

Keep #70309 open. Current main can store MCP server definitions, dispose cached MCP runtimes on config changes, and show live effective tools for an active session, but it does not expose the requested Control UI MCP connectivity panel with per-server connected/error state, tool names per server, manual reconnect/refresh, or structured error details.

## Best Possible Solution

Keep the issue open. The best path is an additive Gateway operator diagnostics API backed by the MCP config/runtime loader that reports each configured server's name, transport summary, connected/error state, listed tools, redacted last error, and a scoped refresh/reconnect action, then render it under Control UI Infrastructure > MCP and document the behavior.
