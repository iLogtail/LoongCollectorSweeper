## Summary

Keep open. Current main still appends `/usage` footers as `Usage: ...` token/cache/cost text, with the session key only in `full` mode; it does not prepend or otherwise include the active provider/model in the usage footer. Existing model attribution support is available through `messages.responsePrefix` templates and `/status`, but that is a different surface from the requested footer behavior. Related #33975 and PR #48635 remain open/unshipped, so this item should not be closed as implemented or superseded yet.

## Best Possible Solution

Keep #65477 open. The likely implementation belongs in the existing response usage footer path: pass the resolved `providerUsed`/`modelUsed` label into `formatResponseUsageLine`, prepend it before `Usage:` for the appropriate `/usage` modes, preserve current `tokens` vs `full` behavior and cost hiding, and add focused tests for both modes. Coordinate with #33975/#48635, but do not close this item until a merged main change actually ships the footer attribution.
