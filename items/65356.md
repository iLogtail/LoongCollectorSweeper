## Summary

Keep open. Current main still uses a module-local sandbox backend Map, while plugin loading and the public SDK make sandbox backend registration a real plugin boundary. The PR is a focused fix for an unimplemented cross-module-graph registry issue, not obsolete or already shipped.

## Best Possible Solution

Keep the PR or an equivalent fix open. The core sandbox backend registry should become Symbol.for/globalThis-backed so plugin-registered backends are visible to runtime lookup across native and jiti-loaded module graphs, with a focused regression test that registers through a plugin SDK/jiti-loaded path and resolves through requireSandboxBackendFactory.
