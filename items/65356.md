## Summary

Keep open. Current main still uses a module-local sandbox backend registry Map, while sandbox backend registration is a public plugin SDK surface and plugin entries are loaded through Jiti. The PR addresses a plausible core/plugin-SDK registry split, and none of the provided related ClawSweeper reports supersede this exact fix.

## Best Possible Solution

Keep the PR open and either merge this narrow core fix or ask the contributor to add a regression test around a Jiti-loaded plugin registering a sandbox backend. This is not a ClawHub-only feature request: the missing behavior is in the core plugin SDK registry contract that third-party sandbox backend plugins depend on.
