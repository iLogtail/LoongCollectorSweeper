## Summary

Keep open. Current main does not contain the proposed singleton fix: the sandbox backend registry is still a module-local Map, while sandbox backend registration is a public plugin SDK seam and plugin runtime loading uses jiti. This is a core extension-point reliability fix, not a ClawHub-only plugin request or an already-implemented change.

## Best Possible Solution

Keep the PR open. The best path is to land this resolveGlobalSingleton change, or an equivalent core fix, with a targeted regression test that proves a sandbox backend registered through the plugin SDK remains visible to the runtime lookup when plugin code is loaded through jiti. If maintainers do not want the exact PR as-is, request that coverage and rerun the normal changed gates before merge.
