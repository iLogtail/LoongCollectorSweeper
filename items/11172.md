## Summary

Keep #11172 open. Current main still does not implement subagent or exec identity environment variables; it only injects the existing OPENCLAW_SHELL runtime marker. The related implementation PR #11497 was closed unmerged, and the open #53262 work is about requiring agentId in the sessions_spawn schema, not exposing that identity to spawned exec processes.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented core exec identity contract. The best implementation belongs in shared exec environment construction: derive identity from resolved session context, inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY consistently, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define whether request env may override identity, and add focused tests/docs for gateway, node, sandbox, PTY, approvals, and background process paths.
