## Summary

Keep open. Current main still does not set OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY in exec/subagent command environments. The exec path has resolved agent/session context, but the environment builders only merge inherited/request/sandbox env and add existing runtime markers such as OPENCLAW_CLI and OPENCLAW_SHELL. The related implementation PR #11497 was closed unmerged, and the recent maintainer/Codex comment on #11172 also concluded this remains missing.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented core exec identity contract for subagent sessions. A likely fix belongs in shared exec environment construction: inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY from resolved agent/session context, decide whether CLAWDBOT_AGENT_ID compatibility is wanted, define whether tool-call env can override these values, and add focused tests/docs for gateway, node, sandbox, and PTY paths.
