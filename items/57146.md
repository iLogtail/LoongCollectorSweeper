## Summary

Keep PR #57146 open. The linked bug is still present on current main: `skills/gh-issues/SKILL.md` still contains hardcoded `/data/.clawdbot` paths for token fallback, claims, cursor files, and embedded sub-agent prompts. The PR is a focused docs/skill fix, but it should not be merged as-is because the review correctly found that it turns the legacy fallback into a duplicate no-op read of the same config path, and current main now uses env-aware `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` patterns that an updated fix should preserve.

## Best Possible Solution

Keep the PR or linked issue active, but update the patch before merge: remove every hardcoded `/data/.clawdbot` reference in `skills/gh-issues/SKILL.md`, keep the current `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` config-path pattern, use the resolved state directory for claims and cursor files, and delete the duplicate fallback block instead of replacing it with another read of the same `~/.openclaw/openclaw.json` path.
