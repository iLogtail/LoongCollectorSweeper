## Summary

PR #40091 is a focused old-path fix, but current main has since moved reset model selection into a shared `modelOverrideSource` policy used by both chat reset and gateway reset. That policy intentionally clears auto-sourced fallback pins while preserving explicit `/model` and legacy user selections, so this PR's one-file unconditional clear is stale. The exact remaining product question, whether bare `/new` after `/model` should return to the agent default, is already tracked by open issue #55063.

## Best Possible Solution

Close this stale PR as superseded and keep the remaining behavior discussion on #55063. If maintainers decide bare `/new` and `/reset` should clear explicit model pins, implement it centrally in `resolveResetPreservedSelection()`, update both `src/auto-reply/reply/session.ts` and `src/gateway/session-reset-service.ts` through that helper, preserve `/new <model>` via `applyResetModelOverride()`, and align tests/docs with the chosen reset contract.
