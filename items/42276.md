## Summary

Keep open. Current main still does not implement the requested OpenAI/Grok-style live reasoning stream in the TUI: `/reasoning stream` exists in the broader session/backend contract, but the TUI command metadata/help still advertises only `on|off`, the TUI agent-event handler has no `thinking` stream branch, and the footer/status area only shows generic activity plus a `reasoning:stream` mode label, not live reasoning text.

## Best Possible Solution

Keep the issue open and implement a narrow TUI-facing fix: expose `stream` in TUI completions/help/usage, render `stream: "thinking"` agent events when `/reasoning stream` is active, clear or finalize transient reasoning text when the run ends, decide whether the footer/status line should show the latest reasoning delta, and add focused TUI event-handler tests. If maintainers choose to route this through a broader gateway reasoning-stream PR, link that item as the canonical follow-up before closing #42276.
