## Summary

Keep open. Current main already supports the proposed `agents.defaults.compaction` tuning path, but it still uses a fixed default reserve floor and does not implement hardware-aware or model-aware higher usable-context defaults for 32GB/64GB local Ollama setups.

## Best Possible Solution

Keep the issue open for maintainer product judgment on local-model compaction defaults. A good resolution would either add a conservative hardware/model-aware preset or auto-tuning path for local Ollama contexts, or explicitly document the high-RAM tuning recipe as the supported answer while keeping low-memory defaults safe.
