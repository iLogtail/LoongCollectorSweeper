## Summary

Current main already exposes and documents the exact tuning knobs described in the report (`agents.defaults.compaction.reserveTokensFloor`, `keepRecentTokens`, and `memoryFlush.softThresholdTokens`), and the threshold math is explicit in code. But OpenClaw still does not auto-tune compaction thresholds by machine RAM or change the conservative defaults for high-memory Macs, so this is not strong enough to close as implemented.

## Evidence

- **Compaction config already supports the proposed knobs:** `applyPiCompactionSettingsFromConfig()` reads `cfg.agents.defaults.compaction.reserveTokensFloor` and `keepRecentTokens`, so the manual tuning path described in the issue is already part of current main.
  - file: [src/agents/pi-settings.ts:43](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/pi-settings.ts#L43)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Preflight threshold uses the reported formula:** Preflight compaction logs and computes `threshold = contextWindowTokens - reserveTokensFloor - softThresholdTokens`, matching the issue's claimed behavior.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:429](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/auto-reply/reply/agent-runner-memory.ts#L429)

  - file: [src/auto-reply/reply/memory-flush.ts:49](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/auto-reply/reply/memory-flush.ts#L49)

## Risks / Open Questions

- The issue can reasonably be read as asking for RAM-aware default behavior for 32GB/64GB machines; current main still keeps conservative generic defaults, so closing as implemented would overstate what shipped.
- This review is source-and-docs based. No hardware reproduction was attempted for the M4 Pro / Ollama scenario, so there is still some uncertainty about the practical performance claim even though the config surface is clear.
