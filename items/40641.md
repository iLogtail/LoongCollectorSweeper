## Summary

Keep open. Current `main` and the latest release still ship the unquoted bracket-notation examples that trigger shell glob errors on shells like zsh, while the CLI parser itself still supports bracket notation once the path reaches OpenClaw. The cross-referenced PR #40642 matches the likely fix direction, but that docs change is not present on `main`.
