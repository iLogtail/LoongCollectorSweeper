## Summary

Keep #71795 open. Current main still enqueues each ordinary Mattermost inbound post as a trusted system event before dispatch, and the prompt builder prepends drained system events ahead of the user prompt. That matches the reported `System: [time] Mattermost message in ...` first-line behavior. Discord’s normal inbound path uses clean `BodyForAgent` text and only enqueues Discord system events from the system-event preflight path.

## Best Possible Solution

Keep the issue open and fix the Mattermost monitor so ordinary inbound posts do not enqueue a trusted `System:` event for the same user message. Preserve clean `BodyForAgent`/`RawBody`/`CommandBody`, keep system events for real channel events such as reactions or button interactions, and add a Mattermost monitor regression test proving an ordinary inbound post dispatches as user text without a prepended `Mattermost message in ...` system line.
