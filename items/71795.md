## Summary

Keep #71795 open. Current main still enqueues a Mattermost system event for every normal inbound post before dispatching the user turn; the shared prompt path drains queued system events as `System:` lines and prepends them ahead of the user body. Discord’s normal user-message path does not enqueue a per-message system event, so the reported Mattermost/Discord parity bug is still present even though Mattermost also sets a clean `BodyForAgent`.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Mattermost normal post handling path: remove or gate the per-message `enqueueSystemEvent` preview, carry sender/channel metadata through structured or untrusted context when needed, and add a regression that a normal Mattermost inbound message reaches the model as user text without a leading `System: Mattermost message ...` line while real Mattermost system/reaction/interaction events still work.
