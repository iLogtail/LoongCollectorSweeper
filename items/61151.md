## Summary

Keep #61151 open. Current main has related replay safeguards for errored/aborted tool turns, but it still does not implement the PR's requested `partialJson` cleanup in `repairToolCallInputs()`, and the OpenAI Responses transport still leaves `partialJson` on completed tool-call blocks.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is to port the PR's conservative complete-block `partialJson` strip into `src/agents/session-transcript-repair.ts`, keep incomplete artifact dropping and `sessions_spawn` attachment redaction intact, and add the regression tests from the PR against current main. If maintainers intentionally want to tolerate persisted `partialJson`, document that decision and add coverage proving replay/display behavior remains safe.
