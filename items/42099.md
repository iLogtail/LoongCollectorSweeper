## Summary

Keep open. Current main still emits `duplicate plugin id detected` diagnostics for distinct-root cross-origin duplicates, including config-over-bundled shadowing. The same physical-root case is suppressed, but the broader behavior requested by #42099 and PR #42192 is not implemented and current tests/docs intentionally preserve logged duplicate drops.

## Best Possible Solution

Keep #42099 open as the user-facing tracker and continue the fix in PR #42192 or a successor. The best implementation is a narrow manifest-registry change that defines which cross-origin duplicate cases should be silent, preserves duplicate state for validation/Doctor/loader consumers, adds regression coverage for the msteams/config-over-bundled reproduction and same-root promotion edge cases, and updates the public duplicate-precedence docs if user-visible logging changes.
