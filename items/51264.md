## Summary

Keep open. Current main still matches the core reported failure mode: `loadCombinedSessionStoreForGateway` resolves all agent session store targets and calls `loadSessionStore` for each one, while `loadSessionStore` still reads/parses whole JSON files and returns a `structuredClone`. Later session maintenance reduces stale/oversized stores, but it does not make the gateway path lazy or target-agent scoped, and `sessions.list` still eagerly builds a combined store.
