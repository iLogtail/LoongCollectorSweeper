## Summary

Keep open. Current main still preserves the launchd-sensitive restart behavior described in the report: `update.run` and config-driven restarts still go through SIGUSR1/in-process restart scheduling, the macOS launchd respawn path still explicitly avoids `launchctl kickstart`/bootstrap handoff, and the LaunchAgent plist still ships `ThrottleInterval = 1` with no `ProcessType`. The latest release `v2026.4.22` still contains those same code paths, so there is no strong evidence this issue is implemented on main or obsolete.

## Evidence

The code comment explicitly says to avoid detached kickstart/start handoffs on launchd.
