## Summary

Keep open. Current main has adjacent Telegram protections, including grammY throttling on live bot instances, retry_after-aware immediate retries for some Telegram helper calls, and a generic write-ahead delivery queue. It still does not implement the requested shared outbound scheduler with per-chat/topic serialization, configurable rate/depth policy, drop-oldest best-effort behavior, or queueing across cron/message-tool/direct outbound, final replies, previews, and media sends.

## Best Possible Solution

Keep #69165 open. The best implementation is a Telegram-owned outbound scheduler shared by direct outbound, message tools, cron/heartbeat delivery, bot final/block replies, previews, and media sends. Key it by Telegram account plus chat/topic, serialize same-key sends, enforce configurable per-chat/group/global pacing, honor Telegram retry_after before retrying, integrate with the existing write-ahead delivery queue for crash recovery, and add config/docs/tests for bestEffort, maxQueueDepth, and drop-oldest behavior.
