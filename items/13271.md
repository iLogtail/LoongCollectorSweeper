## Summary

Keep #13271 open. Current main still has adjacent pieces, including per-channel allow/deny tool policy, opt-in pattern-based loop detection, plugin before_tool_call blocking, and explicit sessions_spawn, but it does not implement the requested per-channel, per-turn tool budget with warn/block/auto-delegate overflow semantics. The existing maintainer-side Codex comment also explicitly kept this issue open, and related #9912 remains a separate global/per-agent budget tracker rather than a replacement for this channel-scoped request.

## Best Possible Solution

Keep #13271 open until OpenClaw adds a typed product/API path for per-channel, per-turn tool-call budgets. A complete fix should add documented channel or agent/channel-scoped config, pass reliable channel/request context into enforcement, define warn, block, and auto-delegate semantics, and cover Discord plus at least one non-Discord channel in tests. If maintainers prefer plugin ownership, add the missing hook/runtime contract first rather than closing this as ClawHub-only.
