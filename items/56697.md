## Summary

Keep PR #56697 open. Current main has improved this surface with canonical required-parameter validation and `(received: ...)` hints, but it still does not include the PR's requested tool-specific examples for empty-object parameter failures. The PR is stale against current main and should not merge as-is because its edit example uses the old top-level `oldText`/`newText` shape while current main requires `edits[]`, but the underlying idea remains a plausible focused improvement rather than something already implemented or clearly superseded.

## Best Possible Solution

Keep the PR open only as a salvageable idea: rebase it onto current main, update examples to the current canonical schemas (`read({"path": ...})`, `write({"path": ..., "content": ...})`, `edit({"path": ..., "edits": [{"oldText": ..., "newText": ...}]})`), remove the unused `acceptedKeys` draft code or deliberately surface it, and add focused tests for empty-object validation errors. Retry-loop prevention should continue under the canonical retryability/loop item rather than being treated as solved by this PR alone.
