## Summary

Keep open. Current main still contains the reported sessions_spawn behavior: when ACP affordances are available, the shared schema advertises streamTo, but the executor still rejects runtime="subagent" calls that include streamTo:"parent" before reaching spawnSubagentDirect. The current unit test suite also asserts that rejection, and the related PRs that propose the silent-drop fix are still described as open in the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a focused fix in src/agents/tools/sessions-spawn-tool.ts plus src/agents/tools/sessions-spawn-tool.test.ts: preserve ACP behavior, but ignore or strip ACP-only streamTo when runtime is subagent so strict-mode provider autofill cannot block native subagent spawning. The existing rejection test should become a regression test proving runtime:"subagent" with streamTo:"parent" reaches spawnSubagentDirect without forwarding ACP-only fields.
