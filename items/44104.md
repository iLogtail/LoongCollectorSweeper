## Summary

Keep open. Current `main` still uses the remote release asset name directly in the local temp download path for Signal CLI, so the PR's security fix is not already implemented. The code also validates archive type only after the download starts, matching the PR's stated gap. The implementation moved from `src/commands/` to `extensions/signal/src/`, but the underlying issue remains on `main` and in the latest release tag `v2026.4.22`.

## Evidence
