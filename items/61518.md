## Summary

Keep PR #61518 open. Current main still does not implement the requested strict web_fetch HTTP(S)_PROXY behavior: web_fetch calls the guarded fetch without a dispatcherPolicy, strict guarded fetch falls back to pinned direct dispatch when the policy is missing, and the current regression test explicitly expects HTTP_PROXY not to use EnvHttpProxyAgent. The PR also has a substantive review comment about redirect/protocol handling, so it remains useful but not superseded or safely closable.
