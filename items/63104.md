## Summary

Keep open. Current `main` does not contain this PR's cross-handler Discord inbound dedupe fix: replay claiming is still scoped to each `createDiscordMessageHandler()` instance, worker-level active-handler dedupe is absent, and the new regression coverage described in the PR discussion is not present.
