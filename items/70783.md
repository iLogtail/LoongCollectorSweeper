## Summary

Keep open. Current main has session infrastructure, a session dropdown, and a Sessions management table, but it does not implement the requested chat tab system for opening, switching, closing, reordering, and persisting multiple chat sessions as simultaneous UI tabs. The issue is recent, author association is NONE, label is only enhancement, and the provided related items are unrelated system-message/session-noise reports rather than a canonical duplicate.

## Best Possible Solution

Keep the issue open as a Control UI feature. The best path is a core UI implementation that uses existing `sessions.create`, `sessions.list`, `sessions.patch`, and `sessions.delete` primitives, adds a persisted open-tab model with per-tab draft/scroll/run state, and covers switching, closing, refresh restore, and long-running-session behavior in UI tests and docs.
