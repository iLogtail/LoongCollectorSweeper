## Summary

Keep #70783 open. Current main has multiple persisted Gateway sessions and a Control UI session picker/table, but it does not implement the requested chat tab system with an ordered set of open chat sessions, create/switch/close tab affordances, per-tab UI state, reorder, or tab persistence. The provided related items are unrelated system-message/session-filtering reports, not a canonical tracker for this UX feature.

## Best Possible Solution

Keep the issue open for Control UI work: add an ordered, browser-persisted chat tab strip backed by Gateway session keys, with create/switch/close controls, clear active-state styling, optional reorder, and careful semantics for close versus `sessions.delete`. Reuse existing `sessions.create`, `sessions.patch`, `sessions.delete`, `chat.history`, and `chat.send` surfaces rather than adding backend API unless a concrete gap appears.
