## Summary

Keep open. Current main has useful generic redaction for logs and persisted transcript text, but the exec foreground path still builds tool result text and `details.aggregated` directly from raw aggregated stdout/stderr. The lower runtime appends stdout/stderr into `session.aggregated` without applying `redactSensitiveText`, and the agent event path only truncates tool-result text/images; exec command-output events can still use raw `execDetails.aggregated`. This remains an actionable OpenClaw security hardening item rather than a stale or duplicate report.

## Best Possible Solution

Keep the issue open and patch the exec-owned surface. Apply `redactSensitiveText` to exec stdout/stderr before `buildExecForegroundResult` emits result text/details, and preferably at the runtime aggregation/update/event boundary so foreground results, background process polling/logs, `tail`, `details.aggregated`, and command-output events share the same redacted view. Add focused regression coverage for a command that prints env-style and bearer-token secrets, asserting redaction in result content, details, live updates, and process/command-output surfaces while preserving the existing persisted transcript redaction behavior.
