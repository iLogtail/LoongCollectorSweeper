## Summary

Keep open. The issue body’s reported foreground exec path still matches current main: stdout/stderr are accumulated into `outcome.aggregated`, then returned as model-visible tool result text and as `details.aggregated` without a secret-redaction pass. Existing redaction code covers logging/tool summaries and some persisted metadata/history cleanup, but I did not find it wired into exec/process stdout delivery, live updates, or command-output events.

## Best Possible Solution

Keep this open for a core security fix. The likely implementation belongs in the exec/process output boundary: redact stdout/stderr before tool-result content, `details.aggregated`, live update text/tails, background process poll/log results, and command-output events, using or extending `src/logging/redact.ts`. Add focused regression tests for env-style `TOKEN=...`, bearer/API-key formats, foreground exec, failed exec, and background process polling/logging. Maintainers should decide whether raw secret-bearing output should ever be revealable and under what explicit approval/opt-in semantics.
