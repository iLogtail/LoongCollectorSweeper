## Summary

Keep open. Current `main` and the latest shipped release still implement the pre-PR behavior: Feishu group replies are forced into thread mode when `thread_id` is present even if `replyInThread` is disabled, and current tests explicitly lock that behavior in. The PR addresses a real config/runtime mismatch, but it is not already implemented and the discussion still leaves a DM `root_id` regression risk that needs maintainer review rather than auto-closure.
