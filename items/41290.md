## Summary

Current main already fixes the heartbeat timer overflow that #41290 targeted. The shipped implementation clamps oversized heartbeat scheduler delays through the shared safe timer helper, logs the clamp once, reschedules safely when a chunk fires before the heartbeat is due, and has regression coverage for oversized intervals. The v2026.4.24 changelog records the canonical shipped fix via #71478 / #71414, so this older unmerged PR is obsolete.

## Best Possible Solution

Close #41290 as implemented on main and keep the shipped #71478-style implementation: shared timeout cap helper, heartbeat scheduler clamp, and regression coverage. If maintainers later want to reject long heartbeat intervals at config parse time, that should be tracked as a separate policy/API decision.
