## Summary

Keep #70605 open. Current main still injects context-file content into the system prompt after only heartbeat-block removal and blank-line normalization; it does not strip Markdown or HTML strikethrough. The linked #47668 discussion includes a concrete user report that struck-through deprecated bootstrap text is still being passed along, and this PR is a recent, focused attempt to address that exact context path with tests. It may still need maintainer review for safe Markdown parsing semantics, but it is not obsolete, duplicated, or outside the repository.

## Best Possible Solution

Keep the PR open for normal maintainer review or replacement by an equivalent maintainer patch. The right product path is a narrowly tested sanitizer on the context-file-to-system-prompt path that removes intentional struck-through deprecated instructions while preserving code fences, unmatched tildes, and ordinary Markdown examples; #47668 should remain the broader tracker for context accumulation and compaction work.
