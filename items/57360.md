## Summary

Keep open. Current `main` still lacks any OpenClaw config/runtime path that forwards Bedrock `requestMetadata` into pi-ai, even though upstream `@mariozechner/pi-ai` already supports that option for Bedrock Converse requests. The request is recent, coherent, and in scope for the bundled `amazon-bedrock` provider rather than ClawHub.

## Evidence

- **Provider config schema has no requestMetadata field:** `ModelProviderConfig` only exposes `baseUrl`, `apiKey`, `auth`, `api`, `injectNumCtxForOpenAICompat`, `headers`, `authHeader`, `request`, and `models`. There is no provider-level `requestMetadata` field in the runtime type surface.
  - file: [src/config/types.models.ts:115](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/types.models.ts#L115)
  - command: `nl -ba src/config/types.models.ts | sed -n '60,140p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Provider config validation is strict and excludes requestMetadata:** The Zod schema for `models.providers.*` is `.strict()` and lists the same fixed keys; unknown provider fields like `requestMetadata` would not be part of the supported schema.
  - file: [src/config/zod-schema.core.ts:348](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/zod-schema.core.ts#L348)
  - command: `nl -ba src/config/zod-schema.core.ts | sed -n '348,382p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Agent/model params are merged but Bedrock requestMetadata is not forwarded:** `resolveExtraParams` merges `agents.defaults.params`, per-model `params`, and per-agent `params`, but `createStreamFnWithExtraParams` only copies a narrow set of keys into stream options: `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, `cachedContent`, and `cacheRetention`. A config value like `params.requestMetadata` would therefore be ignored.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:67](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/pi-embedded-runner/extra-params.ts#L67)

- **Bedrock plugin wrapper adds region/cache/guardrail behavior only:** The bundled `amazon-bedrock` provider's `wrapStreamFn` patches region, guardrails, and prompt-caching behavior, but there is no Bedrock-specific hook that maps config or extra params into `requestMetadata`.
  - file: [extensions/amazon-bedrock/register.sync.runtime.ts:352](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/amazon-bedrock/register.sync.runtime.ts#L352)
