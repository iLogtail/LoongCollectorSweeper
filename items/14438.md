## Summary

Keep #14438 open. Current main still does not provide a supported plugin hot-reload or jiti/cache-clear UX: docs explicitly tell plugin authors to restart the Gateway for native plugin runtime code changes, the `openclaw plugins` CLI has no `reload` or `clear-cache` command, config reload still treats broad `plugins.*` changes as Gateway-restart work, and the loader cache clearing/reload behavior remains internal rather than exposed as a developer command or Gateway RPC.

## Best Possible Solution

Keep this open as core plugin-loader and Gateway lifecycle developer UX work. The strongest implementation path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that invalidates jiti/module state, plugin registry state, plugin-owned registrations, tools/hooks/services/provider hooks, and documents container/wrapper-process behavior. If OpenClaw intentionally does not want native plugin hot reload, close only after documenting that product decision clearly in the plugin CLI/docs and giving a supported cache/restart workflow.
