## Summary

Keep #48621 open. Current main still does not provide the requested raw-config pending changes view: raw mode only tracks string dirtiness, while the existing diff panel is explicitly form-mode-only. The PR is concrete implementation work for the still-missing #39831 capability, though it needs maintainer review/rebase and should address the bot review concerns before merge.

## Best Possible Solution

Keep the PR open as the active implementation path for raw-mode pending changes. The best mergeable version should add a raw JSON/JSON5 diff panel in `ui/src/ui/views/config.ts`, reuse the existing `onRequestUpdate` path instead of abusing `onRawChange`, guard parsing with `hasRawChanges`/panel state, preserve raw sensitive-value redaction, and add focused `config.browser.test.ts` coverage for raw diff rendering and redaction.
