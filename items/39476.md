## Summary

Keep #39476 open. Current main at 64af2feda0334aecd3bb5101cb628b7040caa437 still exposes the requester session key in A2A prompt context, still runs the bounded requester/target reply-back loop and final announce step, and has no per-run guard that removes `sessions_send` from the target agent's tool surface. The issue discussion also includes recent April 24 reports that this still reproduced, while the related fix PRs were closed unmerged and #44309 is only a broader dispatch-mode request.

## Best Possible Solution

Keep the issue open and fix the A2A `sessions_send` path in core. The likely maintainer path is a code-level per-run guard that prevents A2A target/reply/announce turns from calling `sessions_send` back to the requester, plus narrowing or removing requester-session prompt exposure where feasible. Add regression coverage for duplicate visible delivery and keep #44309 separate as the broader dispatch-only/handoff mode request.
