## Summary

Keep #59245 open. Current main has useful voice-call building blocks: outbound notify/conversation modes, DTMF, status/control actions, streaming/realtime audio, and a simple voice-call skill. It does not implement the requested outbound task-call agent: there is no task-call mode/state machine for IVR navigation, hold-state classification, user escalation over a primary chat channel, authorized-info policy, timeout/reschedule behavior, or callback continuity. The April 25 maintainer review in the issue reaches the same conclusion and the item was subsequently labeled `channel: voice-call`, so this should remain a voice-call product/plugin issue rather than be auto-closed as ClawHub-only.

## Best Possible Solution

Keep the issue open and scope the next implementation around the existing `extensions/voice-call` primitives: add a task-call orchestrator/state machine on top of outbound conversation, DTMF, streaming/realtime audio, and call status. The implementation should explicitly model IVR navigation, hold/wait detection, owner escalation through the originating/primary channel, authorized information to share, user-response timeouts, cost/duration caps, and callback/resume behavior. VoIPBin or another telephony media-offload layer can be considered as a provider/backend option, but the product gap is the OpenClaw voice-call task orchestration itself.
