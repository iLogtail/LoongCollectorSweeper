## Summary

Keep open. Current main has adjacent queue work and an empty-reply guard, but it does not implement the requested configurable busy acknowledgment or `queue:wait-exceeded` hook. The current queue config only covers mode/debounce/cap/drop, session lanes still wait silently apart from diagnostics/typing, and the public/plugin hook catalogs do not expose a queue-wait event.

## Best Possible Solution

Keep the issue open and implement the remaining behavior in the auto-reply/session queue path: add `messages.queue.busyAckMs` and `messages.queue.busyAckText` to config types/schema/help/docs, schedule a one-shot acknowledgment when an active session followup waits past the threshold, route that acknowledgment to the queued message's originating channel/thread/account, and add a documented queue-wait hook/event with tests covering Telegram/no-empty-send behavior. Preserve the existing empty-payload guard rather than reworking it.
