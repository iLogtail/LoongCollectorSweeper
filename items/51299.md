## Summary

Keep #51299 open. Current main still has the exact unsupported gap the reporter describes: the legacy iMessage plugin runs `imsg rpc` as a child of the gateway process, while the managed macOS gateway LaunchAgent is still a CLI/Node service rather than an app-bundle process that can own TCC prompts for Messages DB and AppleEvents. The current docs recommend BlueBubbles for new iMessage deployments and mention process-context permissions, but they do not provide a working Sequoia/Tahoe LaunchAgent FDA/Automation approval path or clearly mark this LaunchAgent setup as unsupported. The Keep My Claw comment is backup-related and does not resolve the TCC issue; the provided related ClawSweeper items cover different iMessage/gateway behaviors and are not canonical duplicates.

## Best Possible Solution

Keep the issue open for maintainer handling. The best path is either to document this legacy `imsg` LaunchAgent setup as unsupported on modern macOS and steer users to BlueBubbles, or implement a supported macOS path such as an app-bundle-mediated iMessage helper / explicit MDM-profile guidance / clearer diagnostic status for FDA and Automation failures. If the desired answer is product-level deprecation of LaunchAgent `imsg`, update `docs.openclaw.ai/channels/imessage` and channel setup/status guidance accordingly.
