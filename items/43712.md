## Summary

Keep open. The issue is not stale and current main still lacks a targeted fix for the central source-update edge case: git updates build the live checkout in place after only a normal clean-worktree check, while ignored stale A2UI vendor output can remain under `vendor/a2ui/renderers/lit/dist/` and collide with the A2UI TypeScript emit. The reported Telegram helper mismatch appears fixed on main, but that is secondary to the build/update failure.

## Best Possible Solution

Keep the issue open and fix the source-update path conservatively: before git-update builds, clean or validate ignored A2UI renderer output such as `vendor/a2ui/renderers/lit/dist/`, or make `scripts/bundle-a2ui.mjs` remove its own stale emit directory before `tsgo`. Add a regression test that an otherwise clean git checkout with stale ignored A2UI output does not fail the updater build, and document the recovery/unsupported-layout behavior for source checkouts under node_modules-style paths.
