## Summary

Keep #71049 open. Current main still lacks a server-side Teams DM option to disable the Copilot-style informative stream status: the msteams config schema exposes only `typingIndicator`, `blockStreaming`, and `blockStreamingCoalesce` around this surface; personal conversations always create a `TeamsHttpStream`; and existing tests explicitly assert that `typingIndicator: false` skips only the native typing keepalive while still sending the informative update.

## Best Possible Solution

Keep the issue open and implement a narrow msteams plugin config option that gates DM stream-status creation while preserving the current default. The change should update `MSTeamsConfig`, `MSTeamsConfigSchema`, generated channel metadata, docs at `docs/channels/msteams.md`, and msteams reply-stream tests so DMs can choose native typing plus final-message delivery without disabling existing defaults for users who rely on Teams streaming.
