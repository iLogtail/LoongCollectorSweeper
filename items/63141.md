## Summary

Keep open. Current `main` still silently skips parent-session forking when `session.parentForkMaxTokens` is exceeded, uses a static 100k default unless manually overridden, and does not surface a user-facing warning or auto-compaction path for this case.

## Evidence

- **Silent skip still on current main:** `initSessionState()` checks `parentTokens > parentForkMaxTokens`, logs `skipping parent fork (parent too large)`, and only sets `forkedFromParent = true`; this block does not send a channel notice or trigger compaction before starting fresh.
