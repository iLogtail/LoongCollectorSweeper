## Summary

Keep open. Current main has a Discord-specific Undici FormData fix for proxied attachment uploads, but it does not implement #66243's shared `makeProxyFetch` / `resolveProxyFetchFromEnv` changes. The generic proxy fetch wrappers still pass `Request` objects and global `FormData` bodies through to `undiciFetch` without normalization, and that surface is exported through the public plugin SDK.

## Best Possible Solution

Keep #66243 open or consolidate it with #48556 into one maintained core proxy-fetch fix. The best implementation belongs in `src/infra/net/proxy-fetch.ts`, with regression tests for global FormData to Undici FormData conversion, Request/Request-like flattening, header/signal override semantics, and hop-by-hop/framing header stripping across both explicit proxy and env-proxy paths.
