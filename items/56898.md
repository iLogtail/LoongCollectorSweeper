## Summary

Keep open. Current main still ships the bundled `wacli` skill with only brief soft send guidance, while the issue asks for concrete recipient allowlist, always-confirm, audit-log, and bulk-send policy instructions. Generic exec approvals exist, but they are command/binary-level controls and do not satisfy the requested WhatsApp-recipient send policy.

## Best Possible Solution

Keep the issue open and update `skills/wacli/SKILL.md` with explicit send-safety instructions: always show recipient name/number, full message text, and attachments before sending; require explicit user approval; define how a user/workspace recipient allowlist should be consulted; require per-recipient confirmation for bulk-like sends; and document a redacted audit trail. If maintainers want stronger guarantees than prompt-level instructions, track a follow-up for a dedicated wacli wrapper or plugin-owned send tool that can enforce recipient policy before invoking the external CLI.
