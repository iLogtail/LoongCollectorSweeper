## Summary

Keep #39392 open. Current main still lacks the focused Control UI isolation fix for the reported agent-switch model selector drift: the Agents overview renders directly without a selected-agent remount boundary, agent switching resets adjacent panel state but not the shared config/model form path, and no regression test covers the main -> another agent -> main model selector scenario. The linked PRs #39401 and #39495, plus the broader related PR #54724, remain useful candidate fix paths rather than proof that this issue is obsolete.

## Best Possible Solution

Keep #39392 open and have maintainers choose a canonical fix path among #39401, #39495, #54724, or an equivalent fresh patch. The best implementation is a focused Control UI change that isolates/remounts or explicitly synchronizes the Agents Overview model/fallback controls by selected agent, plus a regression test that switches main -> another agent -> main and verifies the displayed model selector matches the selected agent's resolved config.
