## Summary

Keep #39392 open. Current main still does not implement the agent-switch model-selector isolation described by the issue and linked PRs: the Agents overview panel is rendered directly, agent switching does not reset the shared config form state, and there is no regression coverage for the main -> other agent -> main model dropdown path. The issue is concrete, non-maintainer-authored, and not protected-label blocked, but it remains a plausible unfixed Control UI bug rather than a stale cleanup candidate.

## Best Possible Solution

Keep the issue open and let a maintainer choose or land a focused UI fix, likely by isolating/remounting or explicitly resyncing the Agents overview model controls on selected-agent changes and adding a regression test for switching main -> another agent -> main without saving. The linked PRs #39401 and #39495 remain useful prior art, but current main does not yet contain the behavior.
