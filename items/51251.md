## Summary

Keep open. Current main still preserves explicit or legacy user `modelOverride`/`providerOverride` across gateway reuse, `/new`, `/reset`, and persistent session-store reloads. Main has improved the auto-fallback case and has explicit per-session clear paths, but it does not satisfy this issue's core restart/config-default visibility and stale user-override complaint.

## Best Possible Solution

Keep #51251 open as the canonical restart/config-default visibility issue. If maintainers want config changes or `/new` to clear explicit user model pins, update `resolveResetPreservedSelection`, session model selection, `/status` labeling, and reset tests/docs together. If persistent user pins are the intended contract, document that explicitly and add a first-class `/model clear` or `/model inherit` plus any desired bulk-clear command before closing.
