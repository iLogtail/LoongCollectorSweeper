## Summary

Keep open. Current main only clears auto-sourced fallback model overrides and exposes `session_status(model=default)` as an explicit reset path; it still intentionally preserves user/legacy `modelOverride` and `providerOverride` across `/new`, `/reset`, normal session reuse, and gateway-side `sessions.reset`, so the core complaint remains current.
