## Summary

Keep #72139 open. Current main has partial auth hardening, but the Control UI browser client still has an intentional AUTH_TOKEN_MISMATCH reconnect path when no cached device-token retry is available, and chat.history still performs synchronous transcript read/projection/JSON sizing work on the gateway event loop. The Slack Socket Mode symptom is plausible and actionable rather than obsolete.

## Best Possible Solution

Fix the Control UI browser gateway client so AUTH_TOKEN_MISMATCH stops automatic reconnects when no cached device-token retry is possible, and after the one bounded retry fails, while surfacing clear token re-entry guidance. Separately make chat.history non-starving by reducing synchronous full-transcript work on the gateway event loop, for example by tail-bounded reads, async/chunked processing, or yielding around heavy projection/JSON sizing, with regression coverage for stale Control UI token restarts and large-history handling under active Slack Socket Mode.
