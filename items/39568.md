## Summary

Keep #39568 open. Current main still has the reported Feishu mention-detection failure: if Feishu sends a non-empty `mentions` array that does not include the bot, `checkBotMentioned()` returns false without checking the text placeholder path. The related PR #39569 directly tracks a proposed fix, but the provided context says it is still open and unmerged, so the bug is not implemented on main.

## Best Possible Solution

Keep the issue open until #39569 or an equivalent maintainer-approved fix lands. The fix should make Feishu mention detection handle orphaned text placeholders when the bot is omitted from `mentions`, preserve the existing normal-group/topic-thread routing semantics, and add focused regression tests for reply @mentions and root/thread routing.
