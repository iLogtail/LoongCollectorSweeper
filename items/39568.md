## Summary

Current main still appears to contain the Feishu reply-mention bug. `checkBotMentioned()` still trusts a non-empty Feishu `mentions` array and does not fall back to orphaned `@_user_N` placeholder detection, while reply `root_id` still feeds the Feishu thread-reply path. The linked fix PR #39569 is still open/unmerged in the provided context, so this issue should remain open.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The likely maintainer path is to update `checkBotMentioned()` to detect orphaned reply mention placeholders, clear or suppress root/thread routing for explicit bot @mentions in normal reply messages, and add regression tests that assert both mention detection and non-thread reply behavior.
