## Summary

Keep #39568 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still has both reported Feishu paths: reply mention detection returns false when the structured mentions array omits the bot, and reply root/thread metadata still causes thread-style replies. The related fix PR #39569 is open/unmerged in the provided GitHub context, so there is no implemented or superseding main-branch fix to close against.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to update `checkBotMentioned()` to detect safe orphaned Feishu mention placeholders in reply events, suppress `rootId`/`threadReply` only for normal group reply @mentions that should stay in the main chat, and add regression tests for both mention detection and non-thread reply dispatch while preserving topic groups and explicit thread-reply configuration.
