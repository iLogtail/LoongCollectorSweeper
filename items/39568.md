## Summary

Keep #39568 open. Current main still has the reported Feishu reply-mention drop path: mention detection stops at the structured mentions array and does not detect orphaned `@_user_N` placeholders. The thread-routing half is only partially addressed by a #32980 reply-target change; `root_id` still marks the message as a thread reply and the dispatcher still sends `reply_in_thread: true`. The linked fix PR #39569 remains the right canonical implementation path, but the provided context says it is still open and unmerged.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to make `checkBotMentioned()` detect orphaned Feishu reply mention placeholders, clear or suppress `rootId`/`threadReply` for explicit bot @mentions in normal reply messages that should stay in the main group chat, and add regression tests for both mention detection and non-thread reply dispatch.
