## Summary

Keep open. Current main does not implement generic per-agent plugin configuration overrides: `agents.list[]` remains a strict schema without a `plugins` field, plugin config is still modeled globally under `plugins.entries.<id>.config`, and the SDK config resolver reads only that global entry. The related closed #62700 is about per-agent plugin allow/deny lifecycle scoping, not per-agent deep-merged plugin config values like separate Mem0 API keys or user IDs.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. A complete implementation would add a documented, backwards-compatible `agents.list[].plugins.entries.<id>.config` or equivalent override contract, define merge precedence with global `plugins.entries.<id>.config`, update config schema/types/generated docs, and expose a request-time helper that resolves effective plugin config by `pluginId` and `agentId` for hooks/tools/providers.
