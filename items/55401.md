## Summary

Keep open. Current `main` does not implement native per-agent plugin configuration overrides: agent entries are strict and have no `plugins` field, resolved agent config only carries known agent-scoped settings, and plugin runtime/config loading still reads only top-level `plugins.entries.<id>.config`.

## Evidence

- **Agent schema has no per-agent plugins field:** `AgentEntrySchema` is strict and enumerates agent keys such as `skills`, `memorySearch`, `sandbox`, `params`, `tools`, and `runtime`, but no `plugins` field is accepted.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/zod-schema.agent-runtime.ts#L812)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '808,875p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Agent config type has no plugins property:** `AgentConfig` defines the persisted agent-level surface and includes no `plugins` property.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/types.agents.ts#L70)
