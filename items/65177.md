## Summary

Keep open. Current main still has the reported gap: Feishu account config is strict and uses `name`, while the doctor legacy channel migrations do not include any `botName` to `name` repair. The linked fix PR #65202 was closed unmerged, and the related open PR #65384 appears to track the same missing migration work.

## Best Possible Solution

Keep #65177 open until OpenClaw lands a focused doctor compatibility migration, likely in `src/commands/doctor/shared/legacy-config-migrations.channels.ts`, that rewrites `channels.feishu.botName` and `channels.feishu.accounts.*.botName` to `name` while preserving an existing `name`. Add regression tests for both account-level and top-level legacy shapes, then close the issue from the landing PR, or use the related open PR #65384 as the implementation vehicle if it remains focused and correct.
