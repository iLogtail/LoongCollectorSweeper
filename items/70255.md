## Summary

Keep open. Current main does not contain this PR's one-shot exit behavior for hooks CLI commands, and the reported mechanism is still plausible: hooks reports build plugin diagnostics, plugin diagnostics can execute plugin register handlers, and the successful hooks command path currently returns without forcing process exit. The PR discussion also shows review feedback was addressed rather than leaving an unresolved blocker.

## Best Possible Solution

Keep the PR open, rebase it if needed, and land a scoped version of the proposed fix after changed-lane validation: add a one-shot hooks CLI runner that drains stdout before `defaultRuntime.exit(0)`, guard tests with the shared Vitest environment detection, and preserve the existing deprecated install/update and relay command behavior.
