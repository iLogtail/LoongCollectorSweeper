## Summary

Keep open. Current main still has the original hooks CLI behavior: hooks list/info/check/enable/disable and bare hooks run the generic action wrapper and do not force a clean one-shot exit after output. The PR is a focused, still-relevant fix with review feedback already addressed in follow-up commits.

## Best Possible Solution

Keep the PR open for maintainer review and landing, or reapply the same focused fix on current main: add a one-shot hooks CLI runner for list/info/check/enable/disable/bare hooks, drain stdout before exiting, skip forced exits under the shared Vitest runtime detector, and retain deprecated hooks install/update behavior unchanged with regression coverage.
