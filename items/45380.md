## Summary

Keep PR #45380 open. Current main has partially moved some live-test setup into Node wrappers, but the PR's central Windows failure remains: `gateway:dev`, `gateway:dev:reset`, `tui:dev`, `android:test:integration`, and install e2e scripts still begin with POSIX-style inline environment assignments in `package.json`. The submitted PR also addressed the review feedback about spawn errors and signal exit codes, so it remains a plausible focused fix that needs rebase/current-script adjustment rather than cleanup closure.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and narrow/update it to replace the remaining POSIX inline env prefixes in `package.json` with a cross-platform Node path. Prefer aligning with the existing Windows command helper/runner pattern, add a small unit test for env forwarding and signal/error behavior if feasible, and keep the reporter's Windows `pnpm gateway:dev` verification as the main manual acceptance proof.
