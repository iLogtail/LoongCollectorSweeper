## Summary

Keep open. Current main still has Unix-style inline environment assignments in root `package.json`, including the central reported `pnpm gateway:dev` path. Some Windows runner fixes have landed separately and `test:live` now uses a Node wrapper, but `scripts/with-env.mjs` is not on main and the observable Windows shell failure remains for the gateway/dev scripts this PR targets.

## Best Possible Solution

Keep the PR open, rebase it on current main, and update the remaining env-prefixed package scripts through a small cross-platform Node wrapper or existing script-runner seam. Add focused tests for parsing env assignments, spawn error handling, and Windows command execution so this Windows DX fix can land without broad infra risk.
