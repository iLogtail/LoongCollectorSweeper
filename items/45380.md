## Summary

Keep open. Current `main` still has Windows-incompatible inline env assignments in package scripts, there is no repo-level pnpm shell-emulation workaround configured, and pnpm's own docs say this syntax fails on non-POSIX systems unless `shellEmulator` or `scriptShell` is configured. The PR is addressing a still-real bug rather than something already implemented or obsolete.
