## Summary

Keep #65729 open. Current main still lacks several requested Mattermost threading reliability changes: WebSocket-level ping/pong keepalive is absent, DMs are still explicitly forced to non-threaded `replyToMode: "off"`, per-DM threading config is rejected, thread history is only an in-memory map with no Mattermost `/posts/{root}/thread` backfill, and bare mention-only messages are still normalized to an empty body and dropped. The reporter’s related links are relevant: #51104 covers the keepalive part, while #53186/#59981 show the unresolved product conflict around whether DMs should always stay flat or become configurable.

## Best Possible Solution

Keep this open as a Mattermost plugin behavior issue, or split it into focused follow-ups: add configurable WebSocket ping/pong keepalive, decide and document the DM threading contract alongside #53186/#59981, add thread-history backfill from Mattermost when in-memory history is empty, and handle bare mention-only messages as an explicit wake/invocation path.
