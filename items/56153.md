## Summary

Keep open. Current `main` does not contain this Telegram typing retry change or an equivalent fix: `sendTypingTelegram` still uses the default channel retry path without `strictShouldRetry`, without a bounded attempt cap, and without a non-blocking catch, while the separate `sendChatAction` backoff helper still only special-cases 401 errors.
