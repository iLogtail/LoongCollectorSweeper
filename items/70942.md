## Summary

Keep open. Current `main` still has the reported failure shape: Mattermost status rendering can fall back to strict `resolveMattermostAccount()` on unresolved SecretRefs, unlike Discord/Telegram which provide read-only `inspectAccount` adapters. The touched status code paths are unchanged from the latest release, and the issue is already cross-referenced by open PR #71024.

## Evidence

- **Status summary still falls back to strict account resolution:** `resolveInspectedChannelAccount()` uses `resolvedInspectedAccount ?? params.plugin.config.resolveAccount(...)`. If the plugin has no read-only inspector and `resolveAccount` is strict about SecretRefs, the status path can still throw before it can keep the source-side `configured_unavailable` snapshot.
  - file: [src/channels/account-inspection.ts:55](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/channels/account-inspection.ts#L55)

Default status resolution uses the strict branch and calls `normalizeResolvedSecretInputString(...)` for `channels.mattermost.accounts.${accountId}.botToken`.
