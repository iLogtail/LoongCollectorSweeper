## Summary

Keep #67128 open. Current main has a Telegram-callable `/usage` command, but it is a footer toggle plus a compact current-session/today/30-day cost summary. The richer session usage aggregation exists in Gateway/UI surfaces, including per-agent totals and tool-call counts, but there is no Telegram chat report that breaks usage down by session/sub-agent as requested.

## Best Possible Solution

Keep the issue open and implement a Telegram-native usage report backed by the existing `sessions.usage`/session-cost aggregation path. The likely product shape is `/usage stats` or `/usage sessions`, authorized like other management commands, showing current session/sub-agent plus top sessions/agents, token totals, cost when available, and tool/API-call counts. If this should live outside the bundled Telegram plugin, first expose a narrow supported plugin/runtime helper for session usage aggregation so a ClawHub command plugin does not copy internal parsers.

## Risks / Open Questions

- A ClawHub-only implementation would likely duplicate internal session/transcript usage parsing unless OpenClaw exposes a small supported session-usage SDK/runtime helper.
- Changing `/usage` semantics directly could surprise users who already use it as a response-footer toggle; a subcommand such as `/usage stats` or `/usage sessions` would be safer.
