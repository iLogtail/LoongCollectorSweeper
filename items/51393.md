## Summary

Current main still does not implement statement support for `browser act kind=evaluate`. The managed Playwright page and element paths still parse `fn` with `eval("(" + fnBody + ")")`, so inputs beginning with `var`, `let`, or `const` remain syntax errors. The cross-referenced #55598 closed the broader request for an arbitrary-JavaScript browser capability under the existing `evaluate` surface, but it does not supersede this narrower statement-body request.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled browser plugin. Add a shared evaluator wrapper that preserves current expression/function behavior, but detects or falls back to async-IIFE execution for statement bodies; support both page and element-ref calls, keep timeout and navigation-guard behavior, add tests for `var`/`let`/`const` page and ref cases, and update CLI/docs wording to describe supported function, expression, and statement inputs.
