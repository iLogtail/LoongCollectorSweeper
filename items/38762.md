## Summary

Keep open. Current main still does not accept the reported mixed browser act shape `action=act ref=... request={kind: ...}`. The misleading reachability wrapping appears improved for service 400s, but the core compatibility bug remains, and the related PR #38790 was closed unmerged.

## Best Possible Solution

Keep the issue open and implement a narrow browser tool normalizer for `action: "act"`: when `request` exists, fill missing `request.ref`, `request.targetId`, and valid `request.timeoutMs` from top-level params while preserving nested request precedence. Add a focused regression test for `ref="e146", request={ kind: "click" }` and keep the improved service-error handling intact.
