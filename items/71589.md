## Summary

Keep #71589 open. Current main still suppresses user-visible assistant output only when the assistant phase resolves to `commentary`; it does not suppress unphased tool-use assistant continuations by `stopReason: "toolUse"` or tool-call content. The PR is an active, focused fix for #71588, and the provided discussion shows the author followed up on review gaps for `text_end` coverage and snake_case tool-call block variants.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is to land or refine the narrow suppression for unphased tool-use assistant continuations, preserve explicit `final_answer` delivery, clear already-buffered suppressed text, and keep regression coverage for both `message_end` and `text_end` block-reply modes.
