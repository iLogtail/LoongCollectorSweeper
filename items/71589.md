## Summary

Keep open. Current main still suppresses user-visible assistant output only when the assistant phase resolves to `commentary`; it does not cover phaseless `stopReason: "toolUse"` or tool-call content blocks, which is the central #71589/#71588 behavior. The PR is recent, focused, and its follow-up discussion says it addressed the two bot-review gaps by adding `text_end` coverage, snake_case tool-call variants, and buffer clearing for delayed phase metadata.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is a narrow subscription-layer fix that suppresses or clears user-visible buffers for tool-use continuation assistant messages (`stopReason: "toolUse"` or structured tool-call content), preserves explicit `final_answer` delivery, and keeps regression coverage for both `message_end` and `text_end` delayed-phase paths.
