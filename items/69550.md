## Summary

Keep open. The reported overlap still matches current main: the chat bubble CSS only reserves action-button space when `.has-copy` is present, but the grouped chat renderer never adds that class when copy/expand actions are rendered. The linked PR #69655 proposed the likely fix for #69550, but it was closed unmerged, so it is not a shipped implementation or canonical open replacement.

## Best Possible Solution

Implement the narrow UI fix in `ui/src/ui/chat/grouped-render.ts` by adding `has-copy` to assistant chat bubbles whenever `hasActions` is true, and add a focused `ui/src/ui/chat/grouped-render.test.ts` regression assertion that an assistant message with copy/expand actions receives the class. Keep #69550 open until that fix lands on main or a canonical merged replacement exists.
