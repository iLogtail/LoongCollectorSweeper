## Summary

Current main still appears to have the overlap bug reported in #69550. The CSS reserves room only for `.chat-bubble.has-copy`, but the chat bubble renderer never adds `has-copy` when copy/expand action buttons are rendered. The linked PR #69655 identified the same root cause, but it is closed unmerged in the provided GitHub context, and its fix is not present on current main.

## Best Possible Solution

Keep #69550 open. The best fix is to apply the substance of #69655 on current main: compute `hasActions` before building the chat bubble class string, add `has-copy` when action buttons are present, and add a focused regression test in `ui/src/ui/chat/grouped-render.test.ts` asserting assistant bubbles with copy/expand actions receive `has-copy`.
