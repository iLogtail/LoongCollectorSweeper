## Summary

Keep open. Current `main` still uses the delivery channel for `sessions.usage` attribution, so the reported bug is not implemented, and the PR’s proposed 2-line swap is not a safe conservative close because it would also rewrite session-row semantics and still bucket whole-session totals by a mutable session-level origin snapshot.

## Evidence

## Risks / Open Questions

- The underlying bug report appears still plausible on current `main`: `aggregates.byChannel` is still driven by the delivery-side session channel.
- Merging this PR as-is would likely trade one attribution problem for another by changing session-row `channel`/`chatType` semantics and by attributing whole-session totals to a mutable latest-origin snapshot.
