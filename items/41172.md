## Summary

Current `main` does not implement the Groq `tool call validation failed: attempted to call tool ... which was not in request.tools` parsing described in PR #41172. The formatter and loop-detection code on `main` still only recognize the older `unknown tool` / `tool ... not found|is not available` variants, and there is no corresponding test coverage for the Groq format. This remains a plausible, narrowly scoped fix rather than cleanup-close material.
