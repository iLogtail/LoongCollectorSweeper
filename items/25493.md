## Summary

Keep #25493 open. Current main still does not support editable system prompt templates or section-level workspace overrides such as SAFETY.md; core Safety and CLI quick-reference sections remain hardcoded, provider overrides cover only three unrelated section ids, and the documented config option replaces the entire assembled prompt rather than editing selected OpenClaw-owned sections.

## Best Possible Solution

Keep #25493 open as a core prompt-builder/product decision. A complete fix should add documented and tested section/template overrides for selected OpenClaw-owned prompt sections, or explicitly document that Safety/core sections are intentionally not section-editable and that systemPromptOverride is the supported full-replacement escape hatch.
