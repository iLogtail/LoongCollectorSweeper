## Summary

Keep #25493 open. Current main still does not provide editable system-prompt templates or section-level workspace overrides such as `SAFETY.md`; Safety and the OpenClaw CLI quick reference remain hardcoded, provider prompt overrides are limited to three other section ids, and the only documented user-facing override replaces the entire assembled prompt. The related PR #25938 directly attempted the requested behavior but was closed unmerged, and the existing maintainer/Codex discussion also kept the issue open for the same missing capability.

## Best Possible Solution

Keep #25493 open as a core prompt-builder/product decision. A complete fix should add a documented and tested section/template override path for selected OpenClaw-owned prompt sections, likely through `src/agents/system-prompt.ts` plus workspace/config loading, with clear safety and prompt-cache semantics. If maintainers do not want Safety or other core sections to be editable, document that boundary and keep `systemPromptOverride` as the supported full-replacement escape hatch.

## Risks / Open Questions

- Closing this now would lose a still-missing user-facing capability: section-level customization without copying the entire OpenClaw-owned prompt.
- The existing whole-prompt override is only a workaround; it forces users to duplicate Safety, CLI, and other core prompt text and manually track future OpenClaw prompt changes.
- A real implementation needs explicit maintainer judgment around editable Safety/core sections, prompt cache boundaries, and docs/tests for supported override semantics.
