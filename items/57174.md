## Summary

Keep open. Current main still does not implement per-agent compaction overrides: `agents.list[]` entries do not accept or type a `compaction` field, resolved agent config does not merge one, docs describe `agents.defaults.compaction` only, and runtime compaction paths still read defaults-only config. The linked PR #60807 is open and unmerged, with other overlapping open PRs, so this is active unfinished core config/runtime work rather than implemented or ClawHub-only.

## Best Possible Solution

Keep #57174 open until an equivalent core change lands. A complete fix should add `agents.list[].compaction` to the type and strict schema, share the defaults compaction schema safely, resolve per-agent compaction over `agents.defaults.compaction`, thread the resolved agent id through Pi settings, safeguard extension setup, timeout/model/post-compaction/manual compact paths, update generated schema/help/docs, and add focused schema/resolver/runtime tests. The issue's `session-guardian` ClawHub workaround is useful prior art, but current plugin hooks/providers cannot fully configure the core compaction trigger and runtime behavior per agent.
