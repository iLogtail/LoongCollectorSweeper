## Summary

Keep open. The linked fix PR #61132 was closed unmerged, and current main still routes assistant visible text to Telegram-style block replies without the requested guard that ties exec-looking fenced shell replies to real tool activity. The broader strict-agentic work helps some plan-only stalls, but it does not close this Kimi/Telegram-specific zero-tool-call failure mode.

## Best Possible Solution

Keep the issue open and either revive/supersede #61132 with a narrow guard in the embedded assistant reply pipeline or add an equivalent output-validation seam. The fix should block first-person execution-intent replies with runnable fenced shell blocks when the same turn has no real tool activity, while preserving legitimate tool-backed replies and adding regression coverage for both cases.
