## Summary

Keep #52757 open. Current main only fixed the earlier zero-valued poll-default case from #52118; it still treats non-zero poll defaults like `pollDurationHours: 24` as poll intent on `action: "send"`, and Discord still parses any non-empty `components` object before rejecting an empty `components.modal.fields` array. The workspace media-root portion appears partially addressed by agent-scoped media roots, but the central GPT-5.4 default-parameter compatibility bug is still present.

## Best Possible Solution

Keep the issue open and fix the message tool input boundary or semantic intent guards: poll creation should require a real poll question/options rather than default duration alone, and Discord component parsing should return null for all-empty default component objects before modal validation. Preserve the existing agent-scoped workspace media-root behavior and add targeted tests for GPT-5.4-style filled defaults.
