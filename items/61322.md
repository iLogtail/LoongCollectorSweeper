## Summary

Keep open. The linked bug is still present on current main: same-agent stale cross-root absolute sessionFile paths still fall through to the structural fallback that preserves the old absolute path. The PR is a plausible targeted fix, but it should not be auto-closed or auto-landed because the existing review comment identifies a real custom session.store consistency risk in the proposed implementation.

## Best Possible Solution

Keep #61322 open for maintainer review or author changes. The best patch should remap stale same-agent cross-root sessionFile paths relative to the active resolved sessionsDir, preserve existing cross-agent fallback behavior, and add regression coverage for both the default state dir and a custom session.store/sessionsDir case before closing #61321.
