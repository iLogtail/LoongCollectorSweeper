## Summary

Keep open. Current main supports runtime `toolsAllow` in the embedded runner and cron path, but `sessions_spawn` still has no per-run tool allowlist parameter and the subagent spawn path does not forward one. The related #62264 `lightContext` and #62675 cron `toolsAllow` work are useful adjacent evidence, but they do not implement this requested spawn-time API.

## Best Possible Solution

Implement a native-subagent `sessions_spawn` per-run allowlist, likely as `toolsAllow?: string[]`: add it to the tool schema and parser, `SpawnSubagentParams`, child gateway `agent` protocol/schema, `AgentCommandOpts`, and the `runEmbeddedPiAgent` call so it reaches the existing embedded-runner `toolsAllow` filter. Add seam tests proving `sessions_spawn({ toolsAllow: [...] })` reaches `runEmbeddedAttempt`, rejects or ignores it for unsupported runtimes as intended, and preserves existing static policy precedence/fail-closed behavior. Update the subagents docs after the API is real.
