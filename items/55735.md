## Summary

Keep open. Current main has partial mitigations for stale answers, including system-prompt guidance to consult local docs/source, use `config.schema.lookup`, and run status checks, plus an `openclaw docs` search command. But the issue's central request is not implemented: the agent runtime prompt does not include the OpenClaw product version or version-delta notes, there is no known-behavior registry, and there is no general `openclaw explain <topic>` command.

## Best Possible Solution

Keep this open for maintainer product/design work. A concrete core path would be to add current OpenClaw version metadata to the agent runtime prompt/status surface, teach diagnostics to consult release notes or a generated versioned behavior index before declaring regressions, and add focused tests around `buildAgentSystemPrompt` and the update/diagnostic flow. The existing docs/source/config.schema.lookup guidance should stay as a fallback, not be treated as the complete fix.
