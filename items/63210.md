## Summary

Keep open. Current main has adjacent fixes, but it still does not implement main interactive-session recovery for provider `stopReason:"length"`: no generic length branch, no refusal to execute parsed-but-truncated tool parameters, no truncation recovery prompt, and no proposed config surface. Related #66146 only partially reduces OpenAI-compatible truncation risk and is still open; #63188 remains the separate subagent variant.

## Best Possible Solution

Keep this issue open as the canonical main-session truncation recovery request. A maintainer-quality fix should preserve provider length stop reasons across all main-session transports, add a central runner/tool-dispatch guard that treats `stopReason:"length"` as unsafe for tool execution unless arguments are proven complete, inject an agent-facing continuation/recovery prompt with a bounded retry budget, add config/schema/docs if the behavior is configurable, and cover parsed-but-incomplete tool parameters with regression tests.
