## Summary

Keep open. Current main still evaluates skill OS requirements by exact string match against `process.platform`, so a skill declaring `metadata.openclaw.os: macos` will not satisfy macOS/Darwin eligibility. The code path for `openclaw skills check` uses that evaluator and formats the resulting OS miss as a missing requirement. Tests and docs cover `darwin`, not a `macos` alias, so this is still an actionable alias/validation UX issue rather than an implemented fix.

## Best Possible Solution

Keep #61338 open. The best fix is to add a shared platform canonicalization/validation layer for skill OS metadata, with regression tests for `macos` on Darwin and docs or diagnostics that make accepted platform names unambiguous. The likely code surface is `src/shared/frontmatter.ts` or `src/shared/requirements.ts`, plus skills CLI/status tests and docs updates if aliases are officially supported.
