## Summary

Current main still has the reported behavior. `chunkTextByBreakResolver` trims chunks emitted inside the loop but pushes the final `remaining` string unchanged, and a read-only reproduction of the current logic returns `["! "]` and `["a","b "]` for the reported counterexamples. The related PRs #64054 and #64079 were closed unmerged, so they are not shipped fixes.

## Best Possible Solution

Keep this open and land a narrow maintainer patch in `src/shared/text-chunking.ts`: trim the final `remaining` chunk before pushing it and skip it if trimming leaves it empty. Add regression tests for the two reported counterexamples in `src/shared/text-chunking.test.ts`, and consider a `src/plugin-sdk/text-chunking.test.ts` case because plugin authors consume the helper through `chunkTextForOutbound`. Review the early-return behavior separately if maintainers want the stronger invariant that every returned chunk is right-trimmed.
