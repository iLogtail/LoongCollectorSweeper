## Summary

Keep open. Current main has nearby shipped mechanisms for plugin-initiated agent turns, especially `api.runtime.subagent.run`, but it does not expose the PR's proposed `runtime.followup.enqueueFollowupTurn()` namespace and does not include the proposed bundled `gateway-restart` plugin. The PR is also not safely obsolete: its discussion contains real follow-up review concerns around restart safety, tool authorization, session metadata preservation, and queue semantics.

## Best Possible Solution

Keep this PR open for maintainer review or author re-scope. The best path is to decide whether current `api.runtime.subagent.run({ sessionKey, message, deliver: false })` should be documented as the supported plugin-initiated existing-session turn API, or whether a narrower `runtime.followup` API is still needed. If the core API is not needed, move the `gateway-restart` capability to an external ClawHub/plugin discussion; if it is needed, reduce this PR to the generic API with tests first, then handle any optional restart plugin separately.
