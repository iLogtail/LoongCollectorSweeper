## Summary

Keep open. Current `main` still normalizes outbound reply payloads with `parseReplyDirectives()` only, without calling `stripInboundMetadata()`, and the latest shipped release shows the same behavior. The issue discussion also points to unresolved source-level fix attempts rather than an accepted fix on `main`.

## Evidence

- **Outbound normalization still skips inbound-metadata stripping:** `createOutboundPayloadPlanEntry()` parses `payload.text` with `parseReplyDirectives(payload.text ?? "")` and never imports or calls `stripInboundMetadata()`. `normalizeReplyPayloadsForDelivery()` is just a thin wrapper over that plan, so current outbound delivery normalization still lacks the reported safety net.
