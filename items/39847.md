## Summary

Keep #39847 open. Current main has a newer user-facing assistant-text mitigation for the Discord/vLLM echo case, but the requested shared outbound safety net is still missing: the outbound payload planner still parses raw payload text and the final channel delivery path only runs channel text sanitizers, not `stripInboundMetadata()`.

## Best Possible Solution

Keep #39847 as the canonical bug until a focused source-level fix lands in `src/infra/outbound/payloads.ts` or an equivalent final delivery boundary. The fix should strip inbound metadata before directive/media parsing while preserving reply directives, media directives, channel data, and silent-reply behavior, with regression coverage in `src/infra/outbound/payloads.test.ts`. If maintainers decide the newer `sanitizeUserFacingText()` boundary is intentionally sufficient, document that decision and close the open duplicate PRs explicitly.
