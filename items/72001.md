## Summary

Keep PR #72001 open. Current main still has the reported Gmail wizard/config-validator bug, so this is not obsolete cleanup; however the PR's current unconditional prefix write needs adjustment because it can invalidate existing configurations with a non-hook defaultSessionKey.

## Best Possible Solution

Keep the PR open and revise the implementation so runGmailSetup writes a validator-compatible session policy for the resulting effective mappings without invalidating existing defaultSessionKey configurations. The fix should add regression coverage for the normal wizard output and for an existing non-hook defaultSessionKey/static-shadowing configuration, then keep the changelog entry once the patch is correct.
