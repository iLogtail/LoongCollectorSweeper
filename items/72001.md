## Summary

Keep PR #72001 open. Current main still has the Gmail wizard/config-validator gap the PR targets: `runGmailSetup` writes the Gmail preset without `hooks.allowedSessionKeyPrefixes`, while the Gmail preset has a templated `sessionKey` that `resolveHooksConfig` rejects without prefixes. The PR is useful and focused, but it likely needs maintainer/code updates before landing because current docs/tests also require `hooks.allowRequestSessionKey=true` for templated mapping session keys, and the existing PR review comment identifies a real edge case around non-`hook:` `defaultSessionKey` values.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is to make the Gmail setup wizard emit a complete startup- and runtime-valid hooks configuration for the Gmail preset while preserving existing user policy: merge required prefixes only when compatible, handle the `allowRequestSessionKey` requirement or choose a static-session mapping strategy, and add regression coverage for startup validation, `/hooks/gmail` dispatch, and the `defaultSessionKey`/static-shadow edge case raised in review.

## Risks / Open Questions
