## Summary

Keep #72282 open. This is a fresh, detailed Mattermost channel-only regression report against v2026.4.24, and current main has not changed the relevant Mattermost receive path since that release. The code shows the nominal posted-event and channel-processing paths exist, but I did not find source, test, docs, release, or related-item evidence proving this symptom is fixed or superseded.

## Best Possible Solution

Keep the issue open. Reproduce against v2026.4.24/current main with verbose Mattermost logging and a known-good sender id in groupAllowFrom, then capture whether channel posted WebSocket frames arrive. If frames arrive, add a focused regression test for O/P/G channel posted events under chatmode=onmessage and fix the access/mention/routing drop. If frames do not arrive while REST sees the posts, continue the Mattermost WebSocket subscription/transport investigation and coordinate with the adjacent keepalive/deaf-socket issues rather than closing this report.
