## Summary

Keep #71157 open. Current main has the bundled WhatsApp plugin and Baileys supports an `ephemeralExpiration` send option, but OpenClaw does not expose `channels.whatsapp.disappearingMessagesSeconds`, the strict WhatsApp config schema would reject that field, and the text/media/poll send paths do not forward expiration metadata to Baileys. The related items cover quotes, mentions, and delivery reliability, not disappearing-message expiration.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add `disappearingMessagesSeconds?: number` to shared and account WhatsApp config, zod schema, generated metadata, docs, and account resolution; let account config override channel config; extend `ActiveWebSendOptions` and the Baileys send wrapper to merge `ephemeralExpiration` with quoted options; apply it to text, media, visible text after voice notes, polls, tool/channel sends, and quoted replies; add focused tests for channel default, account override, unset/zero behavior, quoted sends, media sends, and polls.
