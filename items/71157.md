## Summary

Keep #71157 open. Current main does not expose `channels.whatsapp.disappearingMessagesSeconds` or an account-level equivalent, and the WhatsApp outbound paths do not pass Baileys `ephemeralExpiration` on text, quoted, media, audio follow-up text, or poll sends. This is actionable in the bundled `@openclaw/whatsapp` plugin rather than a ClawHub close: WhatsApp is already an OpenClaw channel plugin shipped from this repo. The provided related items cover quote replies, mentions, and delivery bugs, but not disappearing-message expiration.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add root and account-level typed config/schema/docs, resolve account override through the existing account merge path, carry the resolved positive expiration through `sendMessageWhatsApp`, `ActiveWebSendOptions`, `createWebSendApi`, `deliverWebReply`, and poll sends, and add regression tests for unset, channel default, account override, quoted text, media, audio follow-up text, and polls.
