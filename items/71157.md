## Summary

Current main does not implement WhatsApp outbound disappearing-message expiration. This is a recent, actionable bundled WhatsApp plugin enhancement: config/schema, resolved account state, outbound send options, text/media/poll send paths, and docs/tests all lack the requested expiration handling.

## Best Possible Solution

Keep the issue open and implement it inside the bundled WhatsApp plugin. Add channel/account disappearingMessagesSeconds schema and resolved-account support, compute the effective positive expiration with account-level override semantics, pass it through ActiveWebSendOptions into Baileys MiscMessageGenerationOptions.ephemeralExpiration for text, quoted, media, voice-note companion text, poll, auto-reply, and tool/payload paths, then update WhatsApp docs and targeted tests for configured, overridden, unset, quoted, and media cases.

## Risks / Open Questions

- Partial implementation could leave privacy gaps if it misses quoted replies, media sends, voice-note companion text, fallback/trailing auto-reply chunks, polls, or tool/payload sends.
- The optional behavior to honor a Baileys credential default should be verified against the actual Baileys auth-state shape before shipping; configured overrides can be implemented independently.
- Because the WhatsApp zod schema is strict, schema, docs, examples, and tests need to land together or user config containing the new field will still be rejected.
