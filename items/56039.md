## Summary

Current main no longer exposes a top-level `buttons` parameter on the shared `message` tool for Telegram. The schema uses optional semantic `presentation.blocks[].buttons` instead, Telegram advertises `presentation` capability instead of a plugin-specific `buttons` schema, and the same implementation is present in release `v2026.4.22`.
