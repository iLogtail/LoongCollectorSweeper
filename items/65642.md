## Summary

Keep #65642 open. Current main still lacks the PR's core behavior: the OpenResponses streaming handler only emits assistant text/client-tool-call events, `OutputItemSchema` does not allow `function_call_output` output items, and docs only describe client-side tool calls. The PR remains a focused unmerged implementation for the still-open #48489 request, with discussion showing review feedback was addressed on the PR branch.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best product path is to land a tested core Gateway implementation that translates existing internal tool lifecycle events into OpenResponses-compatible `function_call` and `function_call_output` SSE output items, updates the output schema/docs as needed, and preserves existing sanitized event-bus behavior for non-HTTP consumers.
