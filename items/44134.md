## Summary

Keep #44134 open. Current main still normalizes provider tool schemas inside each embedded attempt, and the Google Gemini CLI provider still wires the Gemini normalization hook without any cache or session-persistence/backoff mitigation. The issue has concrete logs and a follow-up allocation-pressure report that map to current source, so this is a real core/provider runtime item rather than stale cleanup.

## Best Possible Solution

Keep the issue open. The best fix is a provider/runtime tool-schema cache keyed by the effective tool catalog plus provider/model normalization context, with invalidation when dynamic tools change, and regression coverage proving repeated embedded attempts with unchanged tools do not re-run per-tool schema normalization. If separate logs show actual external session recreation beyond normalization, handle that with a narrower session-persistence/backoff follow-up.
