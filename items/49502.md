## Summary

Keep open. Current `main` still does not include usage/cost/session metadata in the `agent.wait` terminal response, so this PR is neither implemented nor obsolete. The request is coherent and matches existing gateway metadata patterns used elsewhere.

## Evidence

- **`agent.wait` snapshot type still has no metadata field:** `AgentWaitTerminalSnapshot` on current `main` contains only `status`, `startedAt`, `endedAt`, and `error`; there is no `meta` or usage/cost payload in the dedupe snapshot type.
  - file: [src/gateway/server-methods/agent-wait-dedupe.ts:4](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/gateway/server-methods/agent-wait-dedupe.ts#L4)
  - command: `nl -ba src/gateway/server-methods/agent-wait-dedupe.ts | sed -n '1,240p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **`agent.wait` response on current `main` omits usage/cost/session metadata:** Both the cached-snapshot fast path and the waited response path only return `runId`, `status`, `startedAt`, `endedAt`, and `error`. No `meta`, `usage`, `costUsd`, `provider`, `model`, or `sessionId` is forwarded.
  - file: [src/gateway/server-methods/agent.ts:1118](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/gateway/server-methods/agent.ts#L1118)
