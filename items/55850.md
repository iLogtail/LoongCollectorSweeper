## Summary

Keep #55850 open. Current main has not shipped the collapsed-sidebar clipping fix: the Control UI still defines the collapsed rail as 78px, wires that rail through the main grid/sidebar shell, and lacks the proposed geometry regression test that asserts content starts after the nav rail. The directly linked fix PR #55861 is still open, unmerged, and currently dirty, with review feedback about scoping the rail-width change to desktop layouts. The earlier chat-composer PR #55854 was closed unmerged and does not cover the reporter's later global-layout evidence.

## Best Possible Solution

Keep the issue open until a focused Control UI fix lands. The likely path is to update #55861 or a successor PR so the desktop collapsed-nav content area is provably clear of the rail, avoid widening tablet/mobile drawer behavior unintentionally, and add a settled browser regression test that compares `.content` and `.shell-nav` geometry in the collapsed desktop layout.
