## Summary

Keep open. Current main does not implement the requested `bailian` provider env mapping or `BAILIAN_API_KEY` support. The auth-env system has moved to manifest-backed provider env vars, and current bundled Alibaba/Qwen surfaces use `alibaba`/`qwen` with `MODELSTUDIO_API_KEY`, `DASHSCOPE_API_KEY`, and `QWEN_API_KEY`, but that does not satisfy the reported `bailian` + `BAILIAN_API_KEY` behavior. Related PR #42978 still appears to track the exact requested work, but because the behavior remains absent on main, the issue should not be closed as implemented.
