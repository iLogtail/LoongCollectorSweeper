## Summary

Current main still does not implement the requested WhatsApp high-quality link-preview support. The Baileys socket setup in `extensions/whatsapp/src/session.ts` does not enable `generateHighQualityLinkPreview` even though Baileys documents that option as defaulting to `false`, and OpenClaw’s outbound WhatsApp send surface still only carries basic send options and plain/media payloads. This remains a valid feature request, so it should stay open. ([github.com](https://github.com/openclaw/openclaw/blob/main/extensions/whatsapp/src/session.ts))

## Evidence

- **Socket config still omits the requested flag:** `createWaSocket()` calls `makeWASocket({...})` with `auth`, `version`, `logger`, `printQRInTerminal`, `browser`, `syncFullHistory`, `markOnlineOnConnect`, `agent`, and `fetchAgent`, but no `generateHighQualityLinkPreview` or `linkPreviewImageThumbnailWidth`. Baileys documents `generateHighQualityLinkPreview` as an explicit socket option with default `false`. ([github.com](https://github.com/openclaw/openclaw/blob/main/extensions/whatsapp/src/session.ts))
  - file: `extensions/whatsapp/src/session.ts:1112`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **Outbound listener API has no preview-specific options:** `ActiveWebSendOptions` only contains `gifPlayback`, `accountId`, and `fileName`, and `ActiveWebListener.sendMessage()` only accepts `(to, text, mediaBuffer, mediaType, options)` with that options type. There is no preview toggle or preview payload in the current contract. ([github.com](https://github.com/openclaw/openclaw/blob/main/extensions/whatsapp/src/inbound/types.ts))
  - file: `extensions/whatsapp/src/inbound/types.ts:414`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`
- **Outbound send path forwards only basic send options:** `sendMessageWhatsApp()` builds `sendOptions` only from `gifPlayback`, `accountId`, and `documentFileName`, then calls `active.sendMessage(to, text, mediaBuffer, mediaType, sendOptions)`. No preview-generation hook or preview metadata is added there. ([github.com](https://github.com/openclaw/openclaw/blob/main/extensions/whatsapp/src/send.ts))

There is still no `linkPreview` field in the payload builder. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openclaw/openclaw/main/extensions/whatsapp/src/inbound/send-api.ts))
  - file: `extensions/whatsapp/src/inbound/send-api.ts`
