## Summary

Current main already implements thread-bound ACP delivery routing well enough: ACP spawns derive an explicit bound-thread delivery target, persist requester-origin routing metadata including `threadId`, and terminal ACP task updates deliver back through that stored requester origin. Regression coverage and a shipped changelog entry both match this behavior.

## Close Comment

Closing from a Codex review because current `main` already implements this well enough. Thread-bound ACP spawns now derive an explicit bound-thread delivery target in `src/agents/acp-spawn.ts:951`, dispatch the initial ACP turn with explicit `channel`/`to`/`threadId` in `src/agents/acp-spawn.ts:1264`, and send terminal ACP task updates back through the stored requester origin including `threadId` in `src/tasks/task-registry.ts:1079`. Regression coverage exists in `src/agents/acp-spawn.test.ts:661` and `src/tasks/task-registry.test.ts:539`. This also matches the shipped `v2026.4.10` changelog entry at `CHANGELOG.md:898` for preserving ACP delivery targets back to the originating thread/topic/room. If you still hit a misroute on a newer build, please open a fresh issue with the exact version and current session metadata so it can be checked against the post-fix path.
