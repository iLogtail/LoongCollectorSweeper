## Summary

Keep #71744 open. Current main has structured `model_fallback_decision` subsystem logs and default-on trajectory runtime capture, but the model fallback loop does not emit per-step fallback/root-cause events into the runtime JSONL trajectory stream. The related #62684 context supports the same observability family, but it is a different closed Ollama timeout issue and does not supersede this requested fallback-step telemetry.

## Best Possible Solution

Keep the issue open and add core fallback-step telemetry on the existing trajectory/runtime-event surface. A conservative implementation would pass a recorder/callback into `runWithModelFallback` or emit from its caller for each candidate skip/probe/failure/success/exhaustion, with sanitized from/to provider-model refs, failure reason/detail, chain position, next candidate or final outcome, and regression tests plus `docs/tools/trajectory.md` updates. Preserve the existing `model_fallback_decision` logs as operational diagnostics.
