## Summary

Keep open. Current main has structured model-fallback log records and default-on trajectory runtime capture, but the fallback loop does not write per-step fallback/root-cause events into the runtime trajectory stream. The issue is recent, unprotected, and describes a concrete observability gap that remains actionable in core.

## Best Possible Solution

Keep the issue open and implement core fallback-step telemetry on the existing trajectory/runtime-event surface. The cleanest path is to extend `runWithModelFallback` or its caller with a recorder/callback that emits a sanitized per-candidate event when a candidate fails, is skipped/probed, succeeds after fallback, or exhausts the chain. Include from/to provider/model, failure reason/detail, chain position, next candidate/final outcome, and regression tests plus docs updates for `/export-trajectory`. Preserve the existing `model_fallback_decision` logs as tail-friendly diagnostics.
