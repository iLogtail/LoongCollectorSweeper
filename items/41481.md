## Summary

Keep open. Current `main` still only flattens `openai-completions` message content to plain strings when a model explicitly opts into `compat.requiresStringContent`; the default path still sends the raw `convertMessages(...)` output. That means the reported NVIDIA NIM / DeepSeek-style failure is mitigated by config and docs, but not actually fixed as a general behavior change.
