## Summary

Keep PR #71503 open. Current main still uses first-matching LAN/private IPv4 selection in the gateway, pairing setup, and bundled device-pair plugin paths, so the Docker bridge advertisement bug described in #71493 remains plausible on main. The PR is a focused, recent bug fix with relevant tests, and the author addressed the bot review’s CNI/default-route concerns in follow-up commits.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix: update the gateway LAN picker to prefer default-route/physical LAN candidates before bridge/container/CNI interfaces, wire the same behavior into pairing setup, mirror it in the bundled device-pair plugin without violating the plugin boundary, and retain regression tests for Docker bridge, CNI overlay, default-route preference, and fallback behavior.
