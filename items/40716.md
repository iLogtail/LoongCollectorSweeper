## Summary

Keep open. Current `main` still writes `delivery-mirror` assistant transcript entries and does not filter them in the three consumer paths this PR targets: LLM `transformContext`, RPC `chat.history`, and RPC `sessions.get`. Main has only partial mitigations on adjacent surfaces (HTTP/SSE session history and embedded subscribe block-reply suppression), so this PR remains materially useful rather than stale/implemented.
