## Summary

Keep open. Current main has useful pieces around memory promotion, mention gating, and broadcast groups, and the linked openclaw-listen-only plugin is a relevant partial workaround for separating background context from direct interaction. But the central request is still not implemented: there is no first-class durable rule layer with explicit conflict/supersession semantics, live sessions can still retain stale durable bootstrap context, and existing multi-agent/broadcast behavior does not guarantee that every explicitly mentioned bot replies once while unmentioned bots stay silent.

## Best Possible Solution

Keep the issue open as a product/API design item. The best path is to define a durable rule surface with explicit promotion, provenance, conflict/supersession behavior, and live-session refresh semantics, then add channel-level tests for explicit multi-mention arbitration. Existing memory-core/dreaming, memory-wiki, mention helpers, broadcast groups, #42400, and openclaw-listen-only are useful ingredients or related work, but none is a canonical implementation of this request.
