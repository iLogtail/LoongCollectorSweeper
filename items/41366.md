## Summary

Keep open. Current `main` has adjacent pieces for memory promotion and multi-agent broadcasting, but it does not implement the issue’s requested behavior as a coherent shipped feature. Durable learning today is documented as file-backed memory (`MEMORY.md`, daily notes, dreaming promotion), while stable cross-session instructions still live in `AGENTS.md`/`USER.md`; there is no first-class rule layer or conflict-resolution flow for session-trained rules. On the multi-mention side, Feishu’s current broadcast path still gives only one agent the real reply dispatcher and runs the others as observer/no-op sessions, so it does not provide explicit “every mentioned bot replies once” semantics.
