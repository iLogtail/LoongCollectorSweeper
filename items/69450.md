## Summary

Keep #69450 open. Current main is more targeted than the report implies for ordinary `sessions.send`: it resolves the requested key to one session entry and reads that entry's transcript. However, the requested guarantee to skip all other agent directories is not fully implemented: gateway resolution still performs all-agent discovery for templated `session.store` roots, combined gateway store surfaces intentionally load every discovered agent store, and the docs explicitly describe broader Gateway/ACP discovery. The reporter's concrete beta setup and cleanup workaround therefore still map to a plausible performance issue rather than a safely closeable duplicate or already-shipped fix.

## Best Possible Solution

Keep the issue open and narrow it into a performance fix: avoid full all-agent discovery when the gateway already has a canonical agent id and deterministic target store path, or add a cached per-agent discovery index for templated stores. Preserve intentional aggregate behavior for `sessions.list`, sessionId/label resolution, ACP/disk-only agent discovery, and deleted/retired store compatibility, with regression coverage proving ordinary message sends do not scan unrelated agent stores.
