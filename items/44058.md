## Summary

Current `main` does not implement Feishu bot/account-level TTS configuration. The Feishu channel schema has no `tts` field at either channel or account scope, and the active TTS resolver still reads only `messages.tts`, so the requested feature remains open work.

## Evidence

- **Feishu account schema has no TTS surface:** `FeishuSharedConfigShape` and `FeishuAccountConfigSchema` enumerate supported per-account overrides, but there is no `tts` property for `channels.feishu` or `channels.feishu.accounts.<id>`.
  - file: [extensions/feishu/src/config-schema.ts:155](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/feishu/src/config-schema.ts#L155)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '120,220p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Core TTS resolver is still global-only:** `resolveTtsConfig` builds TTS settings exclusively from `cfg.messages?.tts`, with no channel/account/agent merge path.
  - file: [extensions/speech-core/src/tts.ts:313](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/speech-core/src/tts.ts#L313)
  - command: `nl -ba extensions/speech-core/src/tts.ts | sed -n '313,333p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **User docs still document TTS under messages.tts only:** The TTS docs explicitly state that TTS config lives under `messages.tts` in `openclaw.json`.
  - file: [docs/tools/tts.md:72](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/tools/tts.md#L72)
  - command: `nl -ba docs/tools/tts.md | sed -n '68,76p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Feishu multi-account docs do not expose account TTS overrides:** The Feishu docs show multi-account configuration with `appId`, `appSecret`, `name`, and `enabled`, and the configuration reference lists account keys like `appId`, `appSecret`, and `domain`, but no account-level `tts` setting.
  - file: [docs/channels/feishu.md:204](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/channels/feishu.md#L204)
  - command: `nl -ba docs/channels/feishu.md | sed -n '204,231p'; nl -ba docs/channels/feishu.md | sed -n '375,395p'`

- **Other channels do document dedicated TTS override surfaces:** Discord documents `voice.tts` overriding `messages.tts`, and QQ Bot documents `channels.qqbot.tts` with `messages.tts` fallback. That contrast makes the Feishu omission explicit rather than undocumented-by-accident.
  - file: [docs/channels/discord.md:997](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/channels/discord.md#L997)
