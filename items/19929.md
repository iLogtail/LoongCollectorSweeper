## Summary

Keep #19929 open. Current main does not implement selective shared sessions across channel/group/thread chats: non-direct peers still derive isolated session keys from channel, peer kind, and peer id; the session config schema exposes dmScope and identityLinks but no channelGroups or equivalent grouping map. The linked PR #20078 is useful prior work, but the provided GitHub context shows it was closed unmerged, and the requested capability remains absent on main.
