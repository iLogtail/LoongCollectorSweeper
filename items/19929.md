## Summary

Keep #19929 open. Current main still isolates group/channel/thread sessions by channel and peer id, and the config/schema/docs expose DM sharing controls only, not selective channel/thread session groups. The closed PR #20078 is useful prior work but was not merged, and the current issue discussion already contains a recent Codex review keeping this open.

## Best Possible Solution

Keep #19929 open as the concrete tracker for opt-in shared sessions across selected channel/group/thread chats. The likely core path is a backward-compatible session grouping config, routed through buildAgentSessionKey/resolveAgentRoute and outbound/thread-aware paths, plus schema/help/docs updates and regression tests for routing, queue serialization, reply delivery, and thread inheritance semantics.
