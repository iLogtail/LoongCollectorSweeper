## Summary

Keep open. Current main partially addresses the Linux CI symptom by waiting for the marker file to contain "0", but the issue also reports the Windows failure on the same QA-lab abort-catalog test. The checked-out/current main still asserts the SIGTERM cleanup marker on every platform, while Node's process docs say SIGTERM is not supported on Windows and process.kill() terminates the target process unconditionally there. The related PR #70669 proposed the missing pid/process-exit verification and non-Windows-only SIGTERM cleanup assertion, but that PR is closed unmerged and its remaining diff is not present on main.

## Best Possible Solution

Keep #70659 open and land a focused QA-lab test hardening patch equivalent to the still-missing parts of #70669: write a structured started marker with the spawned pid, verify `lab.stop()` terminates that pid, and keep the graceful `runner-catalog-stopped.txt` SIGTERM cleanup assertion only on non-Windows platforms. Then validate the targeted QA-lab test on Linux and, if feasible, Windows.
