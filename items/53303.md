## Summary

Keep open. Current `main` still hides cost estimates for non-`api-key` providers even when costs are configured, so the underlying bug is not implemented on `main`. The submitted PR also appears to mix the targeted fix with broad unrelated churn, and its proposed `resolveModelCostConfig` gate would also pick up `models.json` and cached gateway pricing rather than only explicit user config, so this needs maintainer-guided follow-up rather than conservative auto-close.
