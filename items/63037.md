## Summary

Keep open. Current `main` still has the indefinite `waitForWaConnection` behavior this PR targets, so the request is not already implemented or obsolete. The PR discussion also surfaced a real follow-up concern on the timeout failure path rather than evidence that the item should be closed.

## Risks / Open Questions

- If left untouched, WhatsApp startup/reconnect can still hang indefinitely when Baileys never emits either terminal `connection.update` event.
- The PR appears directionally useful, but the timeout failure path likely still needs explicit socket cleanup in `monitorWebInbox` before it is ready to land.
