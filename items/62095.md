## Summary

Keep open. Current main has addressed some pieces of the upgrade report, especially doctor migrations and clearer plaintext ws:// guidance, and related #62760 was closed for a narrower sessions_spawn gateway-RPC teardown fix. But #62095 is not fully implemented or safely superseded: the requested subagent announce timeout default reduction is still absent, and the test suite explicitly preserves the 120s default that the reporter called out as harmful.

## Best Possible Solution

Keep #62095 open or split it. The most concrete remaining core work is to decide and implement the subagent announce timeout policy, likely lowering the default or adding a bounded cumulative retry budget, then update src/agents/subagent-announce.timeout.test.ts accordingly. Separately, maintainers should confirm whether the Slack stale-socket cleanup fully resolves the reported gateway RSS growth under repeated health-monitor restarts; if it does, document that in the issue and keep any remaining large-session/drain behavior on canonical follow-ups such as #63279.
