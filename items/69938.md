## Summary

Keep open. Current main still slugifies group session display names and has no legacy display-name migration, so the central user problem remains real. The PR is too dirty to merge as-is, but it contains useful unimplemented core work that is not fully superseded by the narrower Unicode-slug PRs because it also proposes raw/original display formatting and migration behavior.

## Best Possible Solution

Keep the item open for maintainer judgment, but do not merge this branch as-is. The best path is a narrow replacement PR that only changes `src/config/sessions/group.ts`, `src/config/sessions/store-migrations.ts`, focused session tests, and the public group-label docs if raw labels become the intended contract. Link that work to #58932 while explicitly deciding whether OpenClaw wants Unicode-preserving slugs or raw human display labels.
