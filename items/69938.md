## Summary

Keep open. Current main still slugifies group session display names through the ASCII-only slug helper and has no legacy display-name migration, so the central user problem is not implemented. The PR is too dirty to merge as-is, but it contains useful core behavior that is not fully superseded by the narrower Unicode-slug reports because it also proposes raw/original display formatting and migration behavior.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge this branch as-is. The best path is a narrow replacement PR limited to `src/config/sessions/group.ts`, `src/config/sessions/store-migrations.ts`, focused session tests, and the public group-label docs if raw labels become the intended contract. Link it to #58932 while explicitly deciding whether OpenClaw wants Unicode-preserving slugs or raw human display labels.
