## Summary

Keep #60103 open. Current main still has separate Feishu mention helpers for text and card output, but outbound reply text is passed through to post/card/streaming payloads without rewriting manually authored `<at user_id="...">Name</at>` tags into the format-specific Feishu syntax. The related ClawSweeper items cover inbound mention preservation, typing indicators, reply context, rate limiting, and dedup, not this outbound cross-format normalization gap.

## Best Possible Solution

Implement this inside the Feishu plugin, not generic core outbound: add an extension-local mention normalizer that recognizes Feishu `<at>` tokens in agent-authored reply text and applies the correct representation at final send/edit/streaming boundaries. Cover `sendMessageFeishu` post payloads, structured/markdown card sends, streaming card update/close, and outbound adapter render modes with focused tests.
