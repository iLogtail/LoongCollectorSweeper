## Summary

Keep #30529 open. Current main implements Discord typing infrastructure for inbound replies and heartbeat delivery, but it still does not expose an agent-callable `typing` or `typing-indicator` message-tool action. The issue and comments explicitly include that manual/shared action scope, and current action names, Discord discovery, runtime handling, CLI docs, and Discord skill guidance still omit it.

## Best Possible Solution

Keep tracking #30529 until maintainers either implement an explicit `typing` or `typing-indicator` message-tool action, or explicitly decide automatic typing is sufficient and close on that basis. A concrete implementation should add a shared action name and target mode, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with existing account/target resolution, and update CLI docs, Discord skill guidance, and focused tests. If cross-provider parity is desired, design the shared contract first so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
