## Summary

Keep #35203 open. Current main still has isolated hierarchical subagents, plugin lifecycle hooks, per-agent memory/search/promotion, Honcho as an external memory plugin option, and token/cost reporting, but it does not implement the RFC's central capabilities: learned capability profiling, shared blackboard/pub-sub, layered memory visibility/promotion, or multi-agent token budget enforcement. The issue discussion contains useful partial/external paths, especially Octo/teleconference staging and KinthAI's promotion-gated memory notes, but those are not shipped superseding OpenClaw behavior in current main.

## Best Possible Solution

Keep #35203 open as the umbrella RFC until a maintainer scopes it. The practical path is to split it into narrower tracked work: capability metrics/routing, blackboard or artifact search/query, memory visibility and promotion gates, and token budget aggregation/enforcement. Plugin-first prototypes should use runtime.subagent, taskFlow, context/lifecycle hooks, and memory plugin seams where sufficient; any missing runtime data or enforcement hook should become a focused plugin API issue.
