## Summary

Keep PR #62733 open. Current main has shipped the logging config-path part of the problem, but it has not implemented the PR's local memory embedding VRAM retry or per-operation CPU fallback behavior, and the PR still needs rebase/rework before maintainer review.

## Best Possible Solution

Keep the PR open but ask for a focused rebase: drop or adapt the already-shipped logging config-path work, preserve the missing local embedding resilience behavior, validate the node-llama-cpp GPU-fit/CPU-only/dispose APIs through the local adapter types, handle CPU fallback cleanup on partial initialization failure, and add targeted regression tests for single and concurrent VRAM-style failures.
