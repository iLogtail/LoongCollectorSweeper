## Summary

Keep open. Current main still reproduces the reported observability gap: the CLI runner prepares bundle MCP config for codex-cli, but the system prompt report is still built with an empty tool list, so `systemPromptReport.tools.entries` and `tools.schemaChars` remain empty even when MCP tools are injected. The linked PR #65409 directly tracks the fix, but it is still open and not present on current main.

## Best Possible Solution

Keep #65404 open until the CLI runner actually materializes the effective/report MCP tool catalog for bundle MCP runs and passes it to `buildSystemPromptReport`. The best path is to update or supersede #65409 against current main, preserve the final merged backend MCP config/env semantics, dispose any temporary report runtime, and add regression coverage that proves codex-cli/CLI bundle MCP reports non-empty `tools.entries` and positive `tools.schemaChars`.
