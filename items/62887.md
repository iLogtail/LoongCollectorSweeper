## Summary

Current `main` applies global, per-agent, and subagent tool policies when building spawned-session tool inventories, and current tests cover subagent-envelope enforcement. This appears fixed on `main`, but not yet in the latest tagged release.

## Evidence

- **Global and per-agent deny policies are still sourced for spawned sessions:** `resolveEffectiveToolPolicy()` derives `globalPolicy` from `config.tools` and `agentPolicy` from the resolved agent config, so `tools.deny` and per-agent `tools.deny` remain part of tool resolution for spawned sessions.
