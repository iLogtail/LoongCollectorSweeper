## Summary

Keep open. Current main documents the low-level `openclaw system event` command and `/hooks/wake`, but it still does not ship this PR's dedicated automation recipes page or nav entry. The PR remains the canonical place for the remaining wakeup-script documentation work, especially after #48869 was closed as superseded by this PR, but it needs revision before merge because review feedback flags an unsafe webhook shell example and misleading troubleshooting claims.

## Best Possible Solution

Keep #39365 open as the canonical docs PR or replace it with a focused maintainer-authored docs change. The right fix is a safe recipes page that links the existing CLI, heartbeat, and `/hooks/wake` docs; recommends `openclaw system event --mode now` for scripts; explains that writing inbox files alone does not wake an agent; sanitizes untrusted webhook examples; and corrects the `openclaw agent --message` and channel-trigger troubleshooting language.
