## Summary

Keep #71736 open. Current main still has adjacent plugin slash-command discovery and generic plugin approval rendering, but it does not expose the requested Control UI contribution/descriptor registry for plugin-projected chat modes, input guards, event classifiers, status surfaces, or broader UI actions. The follow-up implementation PR #72287 exists and explicitly covers this RFC, but it is still an open PR, not shipped on main.

## Best Possible Solution

Keep #71736 open as the focused Control UI contribution-slot RFC until maintainers land or explicitly supersede it. The best implementation path is an additive, data-only SDK/gateway/UI descriptor contract: plugin registration or manifest projection for chat modes, approval/status descriptors, event classifiers, input guards, and UI actions; host-owned renderer catalogs; enable/disable projection semantics; JSON/sanitization validation; and fixture coverage proving plugin enablement, action dispatch, input guarding, and removal on disable. If PR #72287 lands with that contract, close this as implemented or superseded by the landed PR and cite the merge SHA.
