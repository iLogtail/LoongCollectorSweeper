## Summary

Keep #71736 open. Current main still has adjacent slash-command discovery and generic plugin approval rendering, but not the requested Control UI contribution registry for plugin-projected chat modes, input guards, event classifiers, status surfaces, or descriptor projection. The related implementation PR #72082 is open/draft in the provided context, so this is not implemented on main yet and should remain a live core SDK/UI design item.

## Best Possible Solution

Keep #71736 open as the focused Control UI contribution-slots RFC. The best path is maintainer review of a narrow additive descriptor contract: SDK or manifest registration for chat modes, input guards, status surfaces, event classifiers, generic UI actions, and safe host-owned renderer catalogs; Gateway schema/method/event support; enable/disable projection semantics; sanitization tests; and a fixture plugin proving the contract. If #72082 lands with that contract, this issue can then close as implemented or as resolved by the merged PR.
