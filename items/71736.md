## Summary

Keep #71736 open. Current main has adjacent slash-command discovery and generic plugin approval UI, but not the requested Control UI contribution registry for plugin-projected chat modes, input guards, event classifiers, status surfaces, or broader descriptor projection. The linked RFC PR #71731 is still open and explicitly docs-only/proposed, so this issue remains a live core SDK/UI design thread rather than a ClawHub cleanup candidate.

## Best Possible Solution

Keep #71736 open as the focused RFC thread. The best path is a narrow additive Control UI descriptor contract: SDK or manifest projection for chat modes, input guards, status surfaces, event classifiers, and generic UI actions; safe host-owned renderer catalogs; gateway schema/method/event support; enable/disable projection semantics; sanitization tests; and a fixture plugin proving the contract. The existing generic plugin approval card can be reused as one slice, but it should not be treated as the whole RFC.
