## Summary

Keep #71736 open. Current main still has adjacent plugin slash-command discovery and generic plugin approval UI, but no Control UI contribution or descriptor registry for plugin-projected chat modes, input guards, event classifiers, status surfaces, or broader safe UI descriptors. The related #72287 PR is an open implementation candidate, not merged or shipped, so this remains a live core SDK/UI design item.

## Best Possible Solution

Keep #71736 open and review #72287 as the implementation candidate. The best path is a narrow additive Control UI descriptor contract with safe host-owned renderers, Gateway projection such as plugins.uiDescriptors, enable/disable semantics, sanitization, and fixture coverage for chat modes, approval descriptors, input guards, status surfaces, event classification, and plugin-owned UI actions. If #72287 lands, close this issue as implemented with the landed SHA and release evidence.
