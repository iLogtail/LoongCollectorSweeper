## Summary

Keep #54830 open. Current `main` does not implement the PR's context-provenance behavior: bootstrap context files still carry only `path` and `content`, and system-prompt rendering emits plain project context sections without `ctx:provenance` tags or Context Provenance guidance. The related RFC #54373 is still the live product discussion, and this PR remains a plausible focused Phase 1 implementation rather than obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review against RFC #54373. If the provenance direction is accepted, land a cleaned-up Phase 1 that extends `EmbeddedContextFile`, tags bootstrap context in `buildBootstrapContextFiles()`, renders sanitized provenance comments and conditional guidance in `buildAgentSystemPrompt()`, and includes focused tests for metadata attachment, rendering, escaping, and no-provenance prompts.
