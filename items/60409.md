## Summary

Keep PR #60409 open. Current main still reads only the first existing primary transcript candidate in `readSessionMessages` and returns `[]` when no primary transcript exists, while reset paths still archive transcripts as `.reset.<timestamp>`. The linked PR addresses a real remaining gateway/chat.history gap; related PRs #68765 and #71537 are complementary surfaces, not superseding fixes.

## Best Possible Solution

Keep the PR, or land an equivalent gateway fix, with focused tests around `readSessionMessages`: primary transcript preferred when present, latest valid `.reset.*` archive used when primary is missing, legacy/candidate directories covered, custom transcript basenames handled, and stale candidate basenames excluded. Coordinate but do not replace it with #68765 or #71537 because those cover different recovery paths.
