## Summary

Keep #60237 open. Current main supports `autoThreadName: "generated"`, but the generated-title prompt is still a hardcoded Discord system prompt with no `threadPromptGuideline` config field, schema entry, resolved channel config field, runtime parameter, or test coverage for user-provided prompt guidance. The linked #43366 PR is useful context and shipped the base generated-title strategy, but it does not supersede this follow-up customization request.

## Best Possible Solution

Keep the issue open as a focused Discord bundled-plugin enhancement. A good implementation would add an optional per-channel generated-title guideline field to Discord channel config/types/schema/metadata/docs, validate length, merge it through resolved channel config, pass it into `generateThreadTitle`, compose it safely with the existing message/channel context and fallback prompt, and add unit tests for default behavior, custom guideline behavior, validation rejection, and backward compatibility with `autoThreadName: "generated"`.
