## Summary

Issue #56140 asks for expanding WeCom channel file/document intake, but current `main` treats WeCom as an external community plugin owned outside this repository, so this does not belong in `openclaw/openclaw` core cleanup scope.

## Evidence

## Close Comment

Closing this as a Codex maintainer review because the requested WeCom file/document receive support belongs in the plugin ecosystem, not in `openclaw/openclaw` core. Current `main` documents WeCom as an external community plugin in `docs/plugins/community.md:115`, points the channel catalog at `@wecom/wecom-openclaw-plugin@2026.4.23` in `src/channels/plugins/contracts/channel-catalog.contract.test.ts:40`, and the project vision says optional capability should usually ship as plugins hosted and maintained in their own repositories (`VISION.md:54`). There is no bundled WeCom channel implementation in this checkout to change. If this is still needed, the right follow-up is a feature request against the WeCom plugin repo/package rather than the OpenClaw core repo.
