## Summary

Keep open. Current main has the ordinary WebChat Stop button and `chat.abort` path, but it does not contain this PR's heartbeat-specific stop UI, `sessions.activity` API, heartbeat-disable wiring, or progress/milestone system-prompt changes. The PR is still potentially useful, but needs maintainer review/rework because its own review discussion identified concrete correctness risks.

## Best Possible Solution

Keep the PR open for maintainer review, ideally with a rebase and split scope. The heartbeat/autonomous-task stop work should be separated from the progress-prompt work, use a clear session activity/stop contract, avoid making normal `chat.abort` disable unrelated cron jobs, and include focused tests for queue size, heartbeat state drift, abort cleanup, and progress-update prompt behavior.
