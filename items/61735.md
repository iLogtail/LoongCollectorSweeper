## Summary

Keep #61735 open. Current main still does not apply Gemini schema cleaning based on `modelId` when the provider id is a custom OpenAI-compatible proxy, and the PR's exact regression tests are not present. The requested bug fix remains plausible, but the patch needs to be rebased onto the current split `pi-tools-parameter-schema.ts` implementation and should address the existing review concern about composing Gemini cleaning with `modelCompat.unsupportedToolSchemaKeywords`.

## Best Possible Solution

Keep the PR open for author or maintainer revision. The best fix is a rebased change in `src/agents/pi-tools-parameter-schema.ts` that treats Gemini-like `modelId` values behind custom OpenAI-compatible providers as requiring Gemini schema cleanup, while composing that cleanup with any explicit `modelCompat.unsupportedToolSchemaKeywords`. Add focused tests in `src/agents/pi-tools.schema.test.ts` for custom provider plus Gemini model id, `patternProperties`/`const` cleanup, non-Gemini custom provider non-regression, and compat keyword stripping on the Gemini-modelId path.
