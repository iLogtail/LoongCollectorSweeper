## Summary

Keep open. Current main has already moved WhatsApp delivery into the `extensions/whatsapp` plugin and now covers the reconnect-window problem through listener-level socket replacement/retry handling, but the PR's second requested behavior, surfacing final delivery failures to the agent as system events, is not implemented on current main. The PR is useful but needs to be rebased/retargeted and narrowed to the remaining failure-visibility gap, including the existing bot-review concern that only `kind === "final"` should enqueue the event.

## Best Possible Solution

Keep this PR open for maintainer/author follow-up, but do not merge it as-is. Treat the reconnect retry portion as already superseded by the listener-level socketRef/reconnect retry path on current main, and retarget the remaining work to a small, tested WhatsApp dispatch change that enqueues a system event only for final delivery failures. If maintainers prefer cleanup, open/link a narrower follow-up for final delivery failure visibility, then close this stale pre-plugin-move PR as superseded by that canonical item.
