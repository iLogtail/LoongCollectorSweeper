## Summary

Keep open. Current `main` still exposes `streamTo` in the `sessions_spawn` schema without any ACP-only description, still rejects `streamTo` for `runtime="subagent"`, and still lacks runtime-specific guidance in the tool description, so the reported failure mode remains plausible on current code.

## Evidence

- **`streamTo` still has no ACP-only schema description:** `SessionsSpawnToolSchema` defines `streamTo` as `optionalStringEnum(SESSIONS_SPAWN_ACP_STREAM_TARGETS)` with no description, while neighboring runtime-specific fields like `resumeSessionId` and `lightContext` do carry explicit runtime guidance.
