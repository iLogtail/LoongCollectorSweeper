## Summary

Keep open. Current `main` still gates preflight compaction and memory flush from `reserveTokensFloor` alone, so the reported `reserveTokens > reserveTokensFloor` bug remains present and this PR addresses a real behavior gap.

## Evidence

- **Current preflight threshold still ignores configured reserveTokens:** `runPreflightCompactionIfNeeded` resolves `reserveTokensFloor` and computes `threshold = contextWindowTokens - reserveTokensFloor - softThresholdTokens`, then passes only `reserveTokensFloor` into `shouldRunPreflightCompaction`. There is no `max(reserveTokens, reserveTokensFloor)` path on `main`.
