## Summary

Keep #69379 open. Current main still computes preflight and memoryFlush thresholds from reserveTokensFloor alone, while the config/docs and Pi settings path treat reserveTokensFloor as only a floor for reserveTokens. The PR is the canonical active fix for #66830 after the narrower #67297 was closed as duplicate/superseded, and the review discussion supports it as a real, targeted bug fix.

## Best Possible Solution

Keep #69379 open as the canonical fix path for #66830. Maintainers should review and land this PR or an equivalent patch that resolves an effective reserve as max(agents.defaults.compaction.reserveTokens, reserveTokensFloor), threads it through both preflight compaction and memoryFlush gates, keeps the documented floor semantics intact, and lands the regression tests described in the PR.
