## Summary

Current main does not implement per-agent Bedrock `requestMetadata` injection. The runner has `agentId` available when applying extra params, but the stream option wrapper only forwards selected params and has no `requestMetadata` path. The bundled Amazon Bedrock plugin handles discovery, guardrails, region, cache behavior, and overflow classification, but not per-agent request metadata. Related PR #60698 is an open, unmerged implementation attempt, but it also contains unrelated Feishu changes and unresolved review findings, so it is not a clean superseding tracker. Related issue #57360 covers a broader configurable metadata surface, not this exact automatic per-agent injection request.

## Best Possible Solution

Keep the issue open until OpenClaw ships a scoped Bedrock implementation. The likely path is in the Bedrock provider or runner stream-options path: inject validated per-agent `requestMetadata` when provider is `amazon-bedrock`, preserve existing option precedence, add focused unit coverage for zero-extra-param agents and invalid agent names, and coordinate with #57360 if configurable user metadata should merge with the automatic agent tags.
