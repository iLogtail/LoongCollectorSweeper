## Summary

Keep #44292 open. Current main still does not implement the requested autofix/generator or copy-paste-ready failure output for missing config field labels: the parity test still checks each key with terse `missing ...` messages, the linked PR #46304 that proposed stub output was closed unmerged, and the current config schema/doc generators still consume `FIELD_LABELS` rather than producing missing label stubs.

## Best Possible Solution

Implement the requested developer workflow in `src/config/schema.help.quality.test.ts` or a small config-label helper: collect all missing root/help label keys, generate deterministic copy-paste-ready `FIELD_LABELS` stubs, and cover that output with a focused test if the logic is factored out. A fuller schema-driven label source could follow later, but the low-risk maintainer path is improving the existing parity failure first.
