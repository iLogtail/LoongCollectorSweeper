## Summary

Keep #71783 open. Current main still has the unresolved QMD add-error/name-only conflict path: it can rebind a stale collection only when `collection list` exposes path and pattern metadata, but it does not parse the conflicting name from the `collection add` error shown in the issue. Open PR #72297 is a relevant proposed fix and explicitly targets this issue, but it is not present in current main, so the issue should remain the canonical tracker until that or an equivalent fix lands.

## Best Possible Solution

Keep #71783 open and land #72297 or an equivalent narrow memory-core fix: parse the conflicting collection name from QMD add errors such as `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, verify the add command path and pattern match the configured managed collection, then rebind to the managed name. Add regression coverage for `life` versus `life-main` with incomplete `collection list` metadata and the missing-collection search retry.
