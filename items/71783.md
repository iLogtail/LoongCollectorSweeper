## Summary

Keep #71783 open. Current main can rebind a same-path QMD collection only when `collection list` exposes path and pattern metadata; it still does not parse the stale collection name from the `collection add` error shown in the issue, and the existing regression test intentionally expects the metadata-missing path to warn and skip.

## Best Possible Solution

Keep the issue open and add a narrow memory-core repair. Parse the conflicting collection name from QMD add errors like `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, verify the failed add command path and pattern match the configured collection, then safely remove/re-add or rename/rebind that stale entry to the managed name. Add regression coverage for `life` versus `life-main` and the missing-collection search retry. If automatic mutation is too risky, add a targeted diagnostic and `openclaw doctor` repair instead.
