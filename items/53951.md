## Summary

Keep PR #53951 open. Current main has moved web_fetch HTML extraction behind a plugin-oriented extractor resolver and still ships only the bundled Readability extractor; Defuddle is not implemented on main, and the current external plugin API does not expose a first-class web content extractor registration seam that would make this a clean ClawHub-only close. The PR is stale against the current architecture and should be reworked, but it is a focused feature with review discussion and author follow-up rather than an obsolete/low-signal item.

## Best Possible Solution

Ask for a rebase/rework: implement Defuddle as a web content extractor in the current architecture, likely as a bundled `web-defuddle` extractor with a lower `autoDetectOrder` than `web-readability`, `useAsync: false`, tests asserting Defuddle runs and Readability fallback still works, and updated docs/schema text. If maintainers prefer Defuddle outside core, first add a public external web content extractor registration/ClawHub path, then publish Defuddle as a plugin.
