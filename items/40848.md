## Summary

Keep open. Current `main` does not expose Telegram `entities` on the message tool path, the Telegram message-tool discovery/docs still advertise only `to`, `content`, `mediaUrl`, `replyToMessageId`, and `messageThreadId`, and the current formatting path escapes raw HTML unless a caller can force `textMode: "html"`—which the message tool path does not surface. The request remains plausible and unimplemented on `80b6da72f5f0ac5ade8262142f816b9287fb1cfe`.
