## Summary

Keep open. Current main still contains the reported Unix `lsof -Fpc` command-name filter, so a macOS listener reported as `cnode` would be dropped before stale-gateway cleanup or port polling can recognize it. The linked fix PR #70681 is open and unmerged in the provided GitHub context, so the bug is not implemented on main or superseded by shipped code.

## Best Possible Solution

Keep #70664 open until PR #70681 or an equivalent fix lands. The likely maintainer path is to remove gateway classification from the lsof parser, add Unix argv verification via `ps -ww -p <pid> -o command=` using the existing `isGatewayArgv` contract, apply that verifier in both `findGatewayPidsOnPortSync` and `pollPortOnce`, and replace tests that currently assert `cnode`/non-`openclaw` lsof command filtering with argv-based coverage.
