## Summary

Keep #70664 open. Current main still contains the reported Unix/macOS stale gateway PID bug: non-Windows discovery parses `lsof -Fpc` output and drops any listener whose `c` command token does not include `openclaw`, so a Node-backed gateway reported by lsof as `cnode` is filtered out before any argv-based gateway verification can run. The linked PR #70681 is the right active implementation path, but it is not present in the checked-out main code.

## Best Possible Solution

Keep #70664 open until PR #70681 or an equivalent fix lands. The best fix is to stop classifying Unix gateway listeners by lsof `c`/p_comm, collect candidate listener PIDs, verify them with argv inspection (`ps` on macOS and an appropriate Unix path using the existing `isGatewayArgv` contract), and add regression coverage for lsof output like `p<PID>\ncnode` where the process argv/title identifies `openclaw-gateway`. Close the issue via the fixing PR after it is merged and verified on main.
