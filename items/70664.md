## Summary

Current main still has the Unix stale-PID bug described in #70664: `lsof -Fpc` results are classified by the `c` command token containing `openclaw`, so a real Node-based gateway reported as `cnode` is dropped before cleanup. The linked PR #70681 is the right canonical fix path, but it is still open and the fix is not on main, so the issue should stay open until that PR or an equivalent patch lands.

## Best Possible Solution

Keep #70664 open as the bug record while maintainers review and land #70681 or an equivalent patch. The best fix is to make Unix stale-PID cleanup parse raw lsof listener PIDs, verify each candidate through full argv inspection (`ps`/shared `isGatewayArgv` semantics), and add regression coverage for the macOS `lsof` `cnode` plus `ps` `openclaw-gateway` case before closing the issue.
