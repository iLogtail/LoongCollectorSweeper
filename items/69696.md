## Summary

Keep PR #69696 open. Current main has the broader stale OpenRouter `/v1` canonicalization from #68574, but it does not implement this PR's narrower case-insensitive matching: the OpenRouter, Arcee, and embedded-runner helpers still trim and strip trailing slashes only, then compare against lowercase constants. Current regression tests cover lowercase stale URLs, not uppercase host/path variants, and the provided PR context shows a focused patch plus bot review for exactly that missing edge case.

## Best Possible Solution

Keep #69696 open and review a rebased version that makes OpenRouter base URL canonicalization compare case-insensitively in the OpenRouter plugin, Arcee plugin, and embedded-runner support, with focused regression tests for uppercase legacy URLs, already-canonical no-op behavior, and custom proxy passthrough. A maintainer may prefer URL parsing or lowercasing only the comparison key, but the behavior should remain provider-owned and should preserve non-OpenRouter proxy URLs.
