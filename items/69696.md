## Summary

Keep PR #69696 open. Current main has the lowercase stale OpenRouter /v1 canonicalization from #68574, but the three normalization helpers still compare the trimmed URL string case-sensitively and current tests cover only lowercase `https://openrouter.ai/v1` variants. The PR's added `.toLowerCase()` and uppercase-regression coverage still represent a focused, plausible fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and ask the author to rebase if needed. The best code path is to make only the comparison key case-insensitive in the OpenRouter, Arcee, and embedded-runner normalization helpers, while returning the canonical `OPENROUTER_BASE_URL` constant and preserving custom proxy passthrough. The regression tests proposed in the PR should cover uppercase OpenRouter host input, already-canonical no-op behavior, and non-OpenRouter proxy no-op behavior.
