## Summary

Keep open. Current main still has the case-sensitive behavior this PR targets: OpenRouter, Arcee-via-OpenRouter, and embedded-runner normalization trim trailing slashes but do not lower-case before comparing against lowercase OpenRouter constants. The shipped #68574 fix handles stale lowercase `/v1` URLs, but the uppercase-domain edge case remains missing on main, so this PR is still useful rather than obsolete.

## Best Possible Solution

Keep #69696 open for maintainer review. The best path is to land or adapt the case-folding change across `extensions/openrouter/provider-catalog.ts`, `extensions/arcee/provider-catalog.ts`, and `src/agents/pi-embedded-runner/model.provider-runtime.test-support.ts`, with regression coverage for uppercase OpenRouter host variants and no-op behavior for already-canonical/custom proxy URLs.
