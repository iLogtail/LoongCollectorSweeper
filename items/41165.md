## Summary

Keep open. Current `main` still encodes the reported behavior: default-account Telegram DMs remain on `agent:main:main` under shared `session.dmScope="main"`, and exec/system-event heartbeat wakes still forward the originating session key so they can override a configured heartbeat session.

## Evidence

- **Telegram DM base session still defaults to main session:** `resolveTelegramConversationBaseSessionKey()` returns `route.sessionKey` for default-account DMs unless this is a named-account fallback. When the route session is `agent:main:main`, DMs still stay on the main session.

  - file: [extensions/telegram/src/conversation-route.base-session-key.test.ts:9](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/extensions/telegram/src/conversation-route.base-session-key.test.ts#L9)
