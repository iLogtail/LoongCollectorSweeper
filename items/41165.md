## Summary

Keep open. Current main still has source and tests showing default-account Telegram DMs can resolve to `agent:main:main` when `session.dmScope` is `main`, and the heartbeat runner still prioritizes a forced wake session before `heartbeat.session`. Related PR #41939 was closed unmerged, while #56092 remains an unmerged fix for only the system-event heartbeat reproducer.

## Best Possible Solution

Keep #41165 open until maintainers either land a replacement for #41939 plus #56092, or split this into two canonical issues. The likely code path is to make Telegram private DMs that would otherwise resolve to `route.mainSessionKey` use an isolated direct-session key, and to make `heartbeat.session`/isolated heartbeat routing win over exec forced-session wakes with regression coverage for queued system events.
