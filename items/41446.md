## Summary

Keep open. The PR is not implemented on `main`, and the proposed change is incorrect as written: adding unanchored `config/` to `.gitignore` would match any nested `config` directory, including the tracked `src/config/` tree. Current `main` also documents mutable runtime config outside the repo under `~/.openclaw`, so the PR needs a rewrite or maintainer decision rather than closure.

## Evidence

- **PR patch is unanchored:** The submitted diff adds `config/` to `.gitignore` rather than a repo-root-anchored `/config/` pattern.
  - file: [.gitignore:5](https://github.com/openclaw/openclaw/blob/2e53838dde3c1ad44e2de5eef5cf1ffc3caedd16/.gitignore#L5)

- **Existing local-data ignore rules are root-anchored:** Current `main` ignores mutable local data with root-anchored patterns `/memory/` and `/local/`, which is the established convention for repo-root-only directories.
  - file: [.gitignore:101](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/.gitignore#L101)

## Risks / Open Questions

- Closing this PR would discard a real concern about local secret hygiene, but the proposed fix is not safe to merge as-is.
- The likely correct follow-up, if maintainers want repo-root protection, is a narrower root-anchored rule such as `/config/` after confirming that a repo-root runtime `config/` directory is still part of the intended workflow.
