## Summary

Keep #38066 open. Current main still does not expose Telegram `streamThrottleMs` or `minInitialChars` config, the dispatcher still calls `createTelegramDraftStream` without `throttleMs`, and the draft stream still falls back to `DEFAULT_THROTTLE_MS = 1000` when no caller value is provided. Related PR #38255 tracks the requested work but is open and unmerged, while PR #38226 was closed unmerged.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add Telegram `streamThrottleMs` and, if accepted, `minInitialChars` to the config type/schema/generated metadata/docs, wire the values through `extensions/telegram/src/bot-message-dispatch.ts` into `createTelegramDraftStream`, and cover default, clamp, fallback, and per-account behavior with focused tests. Maintainers should also decide separately whether DM native draft transport should remain forced to message transport or be re-enabled when the throttle surface lands.
