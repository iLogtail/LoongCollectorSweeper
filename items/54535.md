## Summary

Keep open. Current main has useful routing and workaround improvements, including `browser.request` and Chrome MCP existing-session profiles, but the reported default `openclaw browser navigate` problem is not clearly fixed: the CLI still reaches the browser `/navigate` route, and non-`existing-session` profiles still use the Playwright `connectOverCDP` path that the reporter identified as crashing Chrome 146.

## Best Possible Solution

Keep the issue open until maintainers either reproduce and fix the Playwright/CDP crash path, route affected CLI/default-profile operations through a non-crashing transport or documented fallback, or confirm an upstream Chrome/Playwright version bump fixes Chrome 146 with regression/manual verification. The existing-session Chrome MCP profile and loopback/browser.request APIs should be documented as workarounds, not treated as closing evidence for the default CDP `navigate` bug.
