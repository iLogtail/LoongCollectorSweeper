## Summary

Keep open. At the provided `main` SHA `dc7c7034254971fa6e9769e99a18da88f432f8c7`, the shared Slack reaction helper still calls `reactions.add` without handling `already_reacted`, and the generic Slack `react` action still routes through that helper directly. An `already_reacted` catch exists only in the monitor status-reaction path, so the reported bug remains on the user-facing action/tool surface.
