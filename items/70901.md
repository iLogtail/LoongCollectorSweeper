## Summary

Keep #70901 open. Current main still coerces HTTP MCP header values to strings and passes them directly into the SSE/streamable-http transports, so `Bearer ${SUPABASE_PAT}` remains literal rather than expanded. The linked PR #71035 is the right active fix attempt, but it is open/unmerged in the provided GitHub context and the behavior is not implemented on current main or the latest release tag inspected.

## Best Possible Solution

Keep the issue open and land or adapt PR #71035: implement env-template expansion or structured SecretRef resolution for HTTP MCP `url` and `headers` before transport creation, add regression tests for SSE and streamable-http header handling, and align the config schema/docs/SecretRef surface with the supported syntax.
