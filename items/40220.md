## Summary

Keep open. Current `main` still writes two different timestamp representations into each JSONL log record: `_meta.date` remains tslog's ISO UTC value, while `time` is added separately in local-offset form. CLI/UI code prefers `time`, which mitigates user-facing display, but raw log consumers still see the inconsistency reported in #40220.

## Evidence

- **File logger still emits separate local `time` field alongside original log object:** The file-log transport formats `time` with `formatTimestamp(..., { style: "long" })` and then serializes `{ ...logObj, time }`. Because `_meta.date` remains inside `logObj`, the written JSON line still contains both values.

  - file: [src/logging/logger-timestamp.test.ts:35](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/logging/logger-timestamp.test.ts#L35)
