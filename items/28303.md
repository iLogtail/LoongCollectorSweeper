## Summary

Current main still uses `zh-CN` and `zh-TW` as the Control UI runtime locale contract, lazy import keys, persisted setting values, selector option values, generator entries, docs, workflow matrix, and tests. The only `zh-Hans`/`zh-Hant` references found are docs-publish or docs-MDX metadata, not the requested Control UI runtime rename or saved-locale migration.

## Best Possible Solution

Keep #28303 open for a focused Control UI i18n contract change: rename the runtime locale IDs to `zh-Hans`/`zh-Hant`, rename/generated locale metadata and lazy imports, update selector language keys, docs/workflow/test expectations, and add saved-setting migration from `zh-CN`/`zh-TW`. If compatibility should preserve the old IDs, maintainers should record that decision explicitly on the issue.
