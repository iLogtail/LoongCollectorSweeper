## Summary

Keep open. Current main partially covers the available-skills concern through existing agent skill allowlists, but it does not implement the requested per-section full-prompt gates or the requested systemPromptOverride cookbook. The relevant prompt blocks are still hardcoded into the full prompt, and config/docs only expose the existing full override, prompt overlays, skills allowlists, and heartbeat section gate.

## Best Possible Solution

Keep the issue open, but narrow the maintainer path: implement core-owned prompt section controls under an agent-scoped config surface for the full prompt blocks, document which sections are structural versus optional for full overrides, and either point users to `agents.defaults.skills`/`agents.list[].skills` for include-style skill filtering or add a clearly non-overlapping subtractive skill filter if maintainers want that specific workflow.
