## Summary

Keep #51602 open. The report is recent, concrete, and specific to macOS `openclaw tui` with Korean/English IME switching. Current `main` still delegates the relevant input and markdown rendering paths to `@mariozechner/pi-tui`; the dependency is now `0.70.2` and has general East Asian width/IME cursor handling, but I did not find a targeted #51602 fix, regression test, changelog entry, or reproduction proof that the stray glyph/separator artifacts are fixed.
