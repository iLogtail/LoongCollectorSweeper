## Summary

Keep open. Current main has partial recovery and sanitization for some provider failures, including Anthropic thinking-block recovery, but it still forwards sanitized exception detail as user-visible auto-replies on external chat channels. That leaves the central request, a framework-level guarantee to never expose raw LLM/API error details to users, unresolved.

## Best Possible Solution

Keep this issue open until OpenClaw implements a framework-level external-channel failure policy: catch LLM/provider failures before channel delivery, retry only where safe, avoid forwarding raw or sanitized provider exception detail to users by default, and add regression tests for WhatsApp/Telegram-style auto-replies. The existing Anthropic thinking-block recovery and provider-specific sanitizers should remain as partial mitigations, not as the close condition for this broader request.
