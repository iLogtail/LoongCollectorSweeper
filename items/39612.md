## Summary

Keep open. Current main substantially improves the default behavior: generic external runner failures no longer expose raw details by default, and group/channel failures are silenced. But the issue’s literal framework-level request remains unresolved because external direct-chat `/verbose on|full` still intentionally forwards sanitized provider/runner error details to users, and the repo documents that behavior.

## Best Possible Solution

Keep the issue open for an explicit maintainer product/security decision on external-channel failure policy. The likely fix is to keep current generic direct-chat and silent group/channel behavior, but stop forwarding provider/runner exception detail to external chat users even under `/verbose`, or clearly restrict verbose detail to operator/internal surfaces. Add regression coverage for WhatsApp/Telegram direct verbose failures, preserve internal Control UI/log detail, and track any desired multi-agent auto-escalation separately if it remains in scope.
