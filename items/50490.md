## Summary

Keep #50490 open. Current main still appears to have the reported Feishu gap: `/activation mention` can update session activation state and the group intro prompt, but Feishu’s hard message-routing gate still derives `requireMention` from Feishu config/group policy only.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin path. The likely implementation is to resolve the effective group session/route early enough to read the relevant session entry before the mention gate, then let `groupActivation: "mention"` force hard mention gating and `groupActivation: "always"` relax it for that scoped group session, with regression coverage in `extensions/feishu/src/bot.test.ts` or `policy.test.ts`. If maintainers decide Feishu should not support `/activation`, the command should be blocked or documented clearly instead of returning a successful mode switch that does not affect routing.
