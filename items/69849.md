## Summary

Keep #69849 open. Current main still pins @agentclientprotocol/sdk 0.20.0, and the upstream 0.20.0 generated schemas still contain optional array fields without default([]). I did not find an OpenClaw provider-side sanitizer for the reported Claude/API Pro path: current tool-schema normalization is scoped to OpenAI/OpenAI Codex Responses, while the Anthropic and OpenCode provider registrations do not add an equivalent optional-array default transform. The related OpenCode discussion points toward a provider transform-layer fix, but that work is not implemented in OpenClaw main.

## Best Possible Solution

Keep the issue open and ask for or add a focused reproduction that captures the exact OpenClaw provider/ACP registration path and failing schema. The best fix is either to upgrade to an upstream ACP SDK release that emits Claude-compatible defaults, or to add a narrowly scoped provider/tool-schema compatibility transform for affected Claude/API Pro/OpenCode-style paths that fills safe default([]) values for optional array schema fields, with regression coverage for the ACP lifecycle schemas that currently fail.
