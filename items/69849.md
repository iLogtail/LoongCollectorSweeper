## Summary

Keep #69849 open. Current main upgraded OpenClaw to @agentclientprotocol/sdk 0.20.0 and acpx 0.6.1, but the latest ACP SDK package still contains optional array Zod fields without default([]), and OpenClaw does not appear to add a Claude/strict-provider schema normalization layer that would compensate for those fields. The linked OpenCode issue is still open and its follow-up comment points to a provider transform as the preferred fix, which maps to a plausible OpenClaw-side path rather than evidence that the issue is obsolete.

## Best Possible Solution

Keep the issue open and fix it either by consuming an upstream @agentclientprotocol/sdk release that adds safe defaults for optional arrays, or by adding a narrow OpenClaw provider/tool-schema normalization path for Claude/strict validators that turns optional array schema defaults into [] without changing runtime ACP semantics. Add focused regression coverage using representative ACP schemas such as newSession/resumeSession/toolCall so this does not regress across ACP SDK updates.
