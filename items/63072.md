## Summary

Keep open. Current `main` still lacks the array-shape guard this PR proposes, and the underlying load/persist path still matches the reported failure mode.

## Evidence

- **Device pairing loader still passes arrays through:** On current `main`, `loadState()` still assigns `pending ?? {}` and `paired ?? {}` directly. If `pending.json` or `paired.json` parse as `[]`, those arrays remain in state instead of being coerced to plain objects.
  - file: [src/infra/device-pairing.ts:150](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/device-pairing.ts#L150)
  - command: `sed -n '1,220p' src/infra/device-pairing.ts`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Node pairing loader has the same unfixed pattern:** `node-pairing.ts` still uses the same `pending ?? {}` / `paired ?? {}` pattern in `loadState()`, so the bug is present for node pairing as well.
  - file: [src/infra/node-pairing.ts:141](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/node-pairing.ts#L141)
  - command: `sed -n '1,260p' src/infra/node-pairing.ts`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Deserializer does no runtime shape validation:** `readJsonFile()` returns `JSON.parse(raw) as T` with no object-vs-array guard, so array-typed JSON can reach the pairing loaders exactly as described in the PR.
  - file: [src/infra/json-files.ts:29](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/json-files.ts#L29)
  - command: `sed -n '1,220p' src/infra/json-files.ts`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Persist path still relies on plain `JSON.stringify`:** `writeJsonAtomic()` serializes the in-memory value with `JSON.stringify(value, null, 2)`. That aligns with the PR's explanation that keeping an array-shaped state object is unsafe for UUID-keyed pairing maps.
  - file: [src/infra/json-files.ts:38](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/json-files.ts#L38)
  - command: `nl -ba src/infra/json-files.ts | sed -n '29,48p'`
