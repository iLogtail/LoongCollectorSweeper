## Summary

Keep open. Current `main` still hard-codes `web_fetch` to strict guarded fetch, the exposed `tools.web.fetch` config/docs do not provide any env-proxy toggle, and tests explicitly assert that `HTTP_PROXY` is ignored for ordinary `web_fetch` requests. The issue also already has an active cross-referenced PR (#71056), so conservative maintainer cleanup should not auto-close it.

## Evidence

- **`web_fetch` always uses strict mode:** `runWebFetch()` calls `fetchWithWebToolsNetworkGuard()` without any `useEnvProxy` flag, so ordinary `web_fetch` requests stay on the default strict path.
  - file: [src/agents/tools/web-fetch.ts:415](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/src/agents/tools/web-fetch.ts#L415)
  - command: `nl -ba src/agents/tools/web-fetch.ts | sed -n '400,700p'`
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Env proxy is only enabled behind an explicit internal flag:** `fetchWithWebToolsNetworkGuard()` chooses `withTrustedEnvProxyGuardedFetchMode()` only when `useEnvProxy` is true; otherwise it uses `withStrictGuardedFetchMode()`. `withTrustedWebToolsEndpoint()` sets that flag for trusted internal endpoints, but ordinary `web_fetch` does not.
  - file: [src/agents/tools/web-guarded-fetch.ts:37](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/src/agents/tools/web-guarded-fetch.ts#L37)
