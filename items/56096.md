## Summary

Keep open. Current main still does not implement the issue's requested Telegram sendChatAction network/529 retry policy. The affected inbound reply typing path uses the per-account sendChatAction handler, and that handler only backs off/suspends 401 Unauthorized errors; tests explicitly assert non-401 failures do not count toward suspension. The related PR #56153 is open and unmerged in the provided GitHub context, and the proposed retry symbols from that PR are absent from current main.

## Best Possible Solution

Keep #56096 open. The best fix is to land or replace PR #56153 with a maintained implementation in extensions/telegram/src/sendchataction-401-backoff.ts that treats recoverable network/429/529 sendChatAction failures as bounded best-effort typing errors with exponential backoff or suspension, and to add regression tests covering repeated sendChatAction network failures plus repeated onReplyStart/keepalive cycles so the typing loop cannot be revived indefinitely.
