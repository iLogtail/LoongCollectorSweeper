## Summary

Keep open. Current `main` still records ClawHub installs with the resolved `clawhub:<pkg>@<version>` spec in the CLI install path, and the updater reuses `record.spec` for ClawHub updates, so an unversioned install can remain pinned and fail to advance to newer ClawHub releases.

## Evidence

- **CLI install persists resolved ClawHub version:** The `openclaw plugins install` CLI path writes `spec: formatClawHubSpecifier({ name, version })`, which always includes the resolved version returned by ClawHub instead of preserving the user’s unversioned request.
  - file: [src/cli/plugins-install-command.ts:517](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/cli/plugins-install-command.ts#L517)
  - command: `nl -ba src/cli/plugins-install-command.ts | sed -n '490,575p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Updater reuses stored ClawHub spec:** `updateNpmInstalledPlugins` sets `effectiveSpec` to `record.spec` for non-npm sources, then calls `installPluginFromClawHub({ spec: effectiveSpec ?? `clawhub:${record.clawhubPackage!}` })`. If the stored spec is versioned, update re-resolves that same version.
  - file: [src/plugins/update.ts:333](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/plugins/update.ts#L333)
  - command: `nl -ba src/plugins/update.ts | sed -n '300,420p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Update persists the pinned ClawHub spec again:** After a successful ClawHub update, the install record is rewritten with `spec: effectiveSpec ?? record.spec ?? `clawhub:${record.clawhubPackage!}``, preserving the pinned version instead of normalizing back to an unversioned ClawHub locator.
  - file: [src/plugins/update.ts:610](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/plugins/update.ts#L610)
  - command: `nl -ba src/plugins/update.ts | sed -n '420,625p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Current tests still expect the pinned behavior:** The CLI install test for `openclaw plugins install clawhub:demo` expects the persisted record to contain `spec: "clawhub:demo@1.2.3"`, which matches the reported bug rather than a fix.
  - file: [src/cli/plugins-cli.install.test.ts:423](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/cli/plugins-cli.install.test.ts#L423)
  - command: `nl -ba src/cli/plugins-cli.install.test.ts | sed -n '380,535p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
