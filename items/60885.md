## Summary

Keep open. Current main has important launchd restart hardening, but #60885 is not fully implemented or cleanly superseded: the generated LaunchAgent still hardcodes `ThrottleInterval=1`, still does not emit `ExitTimeOut`, and the auto-update service refresh path can still run `gateway install --force` before the detached restart script, reaching a `bootout` activation path from the managed gateway context.

## Best Possible Solution

Keep #60885 open for maintainer-owned macOS LaunchAgent and auto-update hardening. The best path is to make the LaunchAgent shutdown/retry policy explicit, likely by adding or deliberately rejecting an `ExitTimeOut` default with tests, revisiting `ThrottleInterval=1` versus targeted backoff for crash/config/update storms, and making gateway-owned auto-update independent of the managed gateway process before any `gateway install --force`/LaunchAgent `bootout` can occur. Coordinate with open #58890 and #66436, but do not close this item unless a canonical follow-up covers the missing `ExitTimeOut` and auto-update downtime concerns.
