## Summary

Keep PR #71399 open. Current main still does not implement the requested ACP `agent:turn:end` internal hook, and this PR is the active focused implementation attempt after #37833 was closed as superseded by this PR. The branch still needs review follow-up because the proposed payload omits the advertised failure `errorCode` and uses unbounded fire-and-forget dispatch even though a bounded helper exists.

## Best Possible Solution

Keep PR #71399 open for maintainer review or a successor patch. The best path is to emit `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, include `sessionKey`, `success`, `durationMs`, and optional failure `errorCode`, use bounded non-blocking hook dispatch or document why this event intentionally differs, and cover the type guard plus ACP manager success/failure emission in tests. If the event becomes operator-facing, update `docs/automation/hooks.md` and the public hooks docs.
