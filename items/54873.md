## Summary

Issue #54873 is a useful idea, but current main scopes it as an external OpenClaw plugin for ClawHub/npm rather than a bundled/core repo feature.

## Evidence

## Close Comment

Closing as a ClawHub/plugin-scope request after Codex review. Current `main` explicitly keeps optional capability out of core: `VISION.md:52-60` says optional capability should usually ship as plugins hosted in their own repo, with a high bar for adding optional plugins to core, and `VISION.md:99-107` calls out commercial service integrations outside the model-provider category as out of core scope for now. The existing plugin SDK already supports the shape proposed here: `docs/plugins/building-plugins.md:11-19` says tool plugins should be published to ClawHub or npm rather than added to this repo, and `docs/plugins/building-plugins.md:202-244` documents `api.registerTool(...)`, `{ optional: true }`, and `tools.allow` for allowlisted tools. `docs/plugins/community.md:9-16` and `docs/plugins/community.md:129-137` also point community plugins to ClawHub as the canonical distribution path. A Sentry integration still seems useful, but it fits better as an external OpenClaw plugin published on ClawHub/npm than as a bundled/core change in `openclaw/openclaw`.
