## Summary

Keep open. Current main still only supports `agents.defaults.userTimezone`; `agents.list[].userTimezone` is absent from the per-agent type/schema/docs, and runtime timezone readers still resolve from the global default instead of an agent-specific override. The issue discussion’s proposed precedence is coherent and remains unimplemented.
