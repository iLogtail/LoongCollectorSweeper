## Summary

Keep open. Current main still lacks heartbeat-specific model fallback support: `heartbeat.model` is schema/type/docs string-only, heartbeat execution forwards only a single `heartbeatModelOverride`, and fallback resolution still comes from the normal agent/default model chain rather than a heartbeat fallback chain. The linked PR #69495 is open and directly targets this issue, but it has not landed on main.

## Best Possible Solution

Keep #69434 open until the implementation lands. The best product path is to review and finish PR #69495 or an equivalent core change: make `agents.*.heartbeat.model` accept the shared `AgentModelConfig` shape, resolve primary/fallback values with the existing helpers, thread heartbeat fallback overrides into `runWithModelFallback`, update pricing/docs/help/schema labels, preserve string compatibility and `fallbacks: []` behavior, and add focused schema/runtime/docs regression coverage.
