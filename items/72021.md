## Summary

Keep open. Current main still implements the behavior reported in #72021: short-term promotion gates `minRecallCount` on a combined signal count made from `recallCount`, `dailyCount`, and `groundedCount`, and the tests explicitly preserve daily-only promotion through the default gates. This is an active product/behavior question in the bundled memory-core plugin, not implemented-on-main, a duplicate, or ClawHub scope.

## Best Possible Solution

Keep #72021 open for maintainer product judgment. If accepted, split gate semantics so `minRecallCount` checks real `recallCount` only, keep or rename combined daily/grounded/session support as a separate signal metric, recalibrate promotion scoring/default thresholds, and update the current daily-only promotion tests, CLI diagnostics, and docs to match the chosen semantics.
