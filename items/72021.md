## Summary

Keep #72021 open. Current main still uses combined recall/daily/grounded signal counts for the short-term promotion recall-count gate and scoring inputs, while daily and session ingestion record dreaming signals as `daily` signals. There is even current test coverage asserting that `recallCount: 0, dailyCount: 3` can clear the default promotion gates, so this is an actionable product/code decision rather than an obsolete or duplicate report.

## Best Possible Solution

Keep the issue open for maintainer/product judgment in `extensions/memory-core`. The best fix path is to decide whether `minRecallCount` means organic user recalls only, then either gate on `recallCount` while keeping separate weighted scoring for daily/session/grounded signals, or document and retune the existing combined-signal policy with explicit tests for real recall-only candidates near the default thresholds.
