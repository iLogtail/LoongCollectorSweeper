## Summary

Keep open. Current main still has a plausible duplicate path for Discord block-mode preview tool progress: tool lifecycle and item lifecycle events for the same tool are both appended to the same preview message, and the accumulator only dedupes adjacent identical strings.

## Best Possible Solution

Fix the Discord preview progress aggregation rather than closing the issue: canonicalize or replace per `toolCallId`/`itemId`, avoid rendering both `onToolStart` and tool-kind `onItemEvent` as separate visible lines for the same tool, and add a Discord block-mode regression test that simulates `session_status` tool start plus item start/update and asserts only one stable progress entry appears.
