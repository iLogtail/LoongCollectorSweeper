## Summary

Keep open. Current `main` still does not implement declarative `sandbox.docker.volumes`, and the existing docs/tests/runtime remain bind-only. The PR is coherent, recent, and still targets a real missing capability rather than something already shipped on `main`.

## Evidence

- **Sandbox docker config type is still bind-only:** `SandboxDockerSettings` exposes `binds` plus the dangerous overrides, but there is no `volumes` field or `SandboxDockerVolumeSetting` type on current `main`.
  - file: [src/config/types.sandbox.ts:45](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/types.sandbox.ts#L45)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Runtime schema validates binds only:** `SandboxDockerSchema` defines `binds` and its bind-specific validation, with no `volumes` array or strategy validation on current `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:138](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/zod-schema.agent-runtime.ts#L138)
