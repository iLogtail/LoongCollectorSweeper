## Summary

Current main has better Telegram preview/tool-progress observability and ACP binding support, but I did not find high-confidence evidence that the reported failure mode is fixed. The Telegram path still delivers visible assistant text through partial/final reply callbacks independently of any verified tool completion, so a model can still claim success without a tool call unless a separate verifier catches it.
