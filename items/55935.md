## Summary

Keep open. Current `main` still injects a locale-formatted `Current time:` line for heartbeat/cron-style prompts and does not add a separate trigger timestamp field.

## Evidence

- **Current time formatter still non-ISO:** `resolveCronStyleNow` formats the local side with `formatUserTime(...)` and builds `Current time: ${formattedTime} (${userTimezone}) / ${utcTime}`. This is not ISO 8601 and there is no second `Heartbeat triggered at` field.
  - file: [src/agents/current-time.ts:29](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/current-time.ts#L29)
  - command: `nl -ba src/agents/current-time.ts | sed -n '1,220p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Heartbeat prompt uses startedAt as Current time:** `runHeartbeatOnce` injects `appendCronStyleCurrentTimeLine(prompt, cfg, startedAt)` into the heartbeat body, so the `Current time:` line still reflects the heartbeat start/trigger timestamp rather than a separately labeled processing timestamp.
  - file: [src/infra/heartbeat-runner.ts:974](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/infra/heartbeat-runner.ts#L974)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '940,1005p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Tests still encode legacy human-readable format:** Prompt expectations still contain strings like `Current time: Sunday, March 15th, 2026 - 9:30 PM (America/Los_Angeles) / 2026-03-16 04:30 UTC`, which matches the reported behavior rather than an ISO 8601 form.
  - file: [test/helpers/agents/prompt-composition-scenarios.ts:571](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/test/helpers/agents/prompt-composition-scenarios.ts#L571)
  - command: `nl -ba test/helpers/agents/prompt-composition-scenarios.ts | sed -n '540,620p'`
