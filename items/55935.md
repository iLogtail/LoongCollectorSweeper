## Summary

Keep open. Current main still emits the old locale-formatted single-line `Current time: <human local> (<tz>) / <YYYY-MM-DD HH:MM UTC>` heartbeat/cron-style timestamp, with no ISO 8601 local timestamp and no separate `Heartbeat triggered at` field. The linked PR #55957 proposed part of this but was closed unmerged, and adjacent ClawSweeper reports also show this timestamp surface remains unresolved.

## Best Possible Solution

Keep the issue open. The best product path is to update the shared cron-style time helper to emit a machine-readable ISO 8601 local timestamp with offset, decide whether heartbeat needs a separate `Heartbeat triggered at` line versus a more general `Reference UTC`/trigger timestamp, and then update all shared prompt-context tests and any public date/time docs that describe the resulting contract. PR #42654 and issue #63181 are adjacent context but do not supersede this exact heartbeat/cron-style ISO timestamp request.
