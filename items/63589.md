## Summary

Keep this issue open. Current main still omits WhatsApp participant identity and quoted reply metadata from the gateway inbound logger payload, even though the same handler already computes those fields for the queued inbound message. The linked PR #63615 targeted the right fix but is closed unmerged, so the "Fixed, closing." comment is not supported by current main behavior.

## Best Possible Solution

Keep the issue open and implement the narrow WhatsApp plugin change in `extensions/whatsapp/src/inbound/monitor.ts`: copy the already-computed participant identity and `replyTo*` fields into the `inboundLogger.info()` payload, then add a regression assertion in the existing inbound logger test for a group message with quoted reply context.
