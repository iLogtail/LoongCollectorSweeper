## Summary

Keep #70680 open. Current main still lets doctor orphan-transcript scanning treat `*.trajectory.jsonl` as a primary transcript candidate, while current trajectory code/docs still create session-adjacent trajectory sidecars with that suffix. The related PR #70812 is useful prior work, but the provided GitHub context says it is still open/unmerged, so the fix is not shipped on main.

## Best Possible Solution

Keep #70680 open as the canonical bug until a fix lands. The likely path is to land or adapt #70812: narrow `isPrimarySessionTranscriptFileName()` so trajectory sidecars are not primary transcript candidates, add helper coverage for `*.trajectory.jsonl`, and add a doctor state-integrity regression proving a real orphan transcript is still archived while a neighboring trajectory sidecar is ignored.
