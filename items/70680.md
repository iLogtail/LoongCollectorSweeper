## Summary

Keep open. Current main still treats `*.trajectory.jsonl` as a primary session transcript candidate, and the doctor orphan scan still uses that matcher before comparing only `sessions.json` transcript paths. The linked PR #70812 is a relevant candidate fix, but it is still open in the provided GitHub context and the fix is not present in the checked-out main SHA.

## Best Possible Solution

Keep #70680 open until PR #70812 or an equivalent fix lands on main. The likely fix is to model trajectory sidecars as non-primary session artifacts, add helper coverage in `src/config/sessions/artifacts.test.ts`, add a doctor regression test proving a real orphan transcript is still reported while a neighboring `*.trajectory.jsonl` sidecar is ignored, and audit shared callers of `isPrimarySessionTranscriptFileName()` for disk-budget and usage-count behavior.
