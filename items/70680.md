## Summary

Keep open. Current main still treats `*.trajectory.jsonl` as primary session transcripts, so the doctor orphan-transcript scan can still include live trajectory sidecars while the referenced transcript set only contains the primary session file path. The related PR #70812 targets the right fix, but it is still open and not present in the checked-out main behavior.

## Best Possible Solution

Keep #70680 open as the canonical bug until a scoped helper fix lands. The maintainer path is to exclude `*.trajectory.jsonl` from primary and usage-counted transcript classification, add artifact-helper coverage, and add a doctor regression test proving a real orphan transcript is still reported while a neighboring trajectory sidecar is ignored. PR #70812 appears aligned with that direction and should be reviewed or superseded by an equivalent main-branch fix.
