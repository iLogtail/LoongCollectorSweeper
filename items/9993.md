## Summary

Keep #9993 open. Current main has built-in config validation, destructive-clobber protection, SecretRef preflight, and an agent-facing config mutation guard, but it still does not expose the requested synchronous config-write hook/plugin API that receives current and pending config and can veto direct config.patch/config.apply writes before persistence. The existing config-guard skill and linked RFC discussion are useful prior art, not a replacement for the missing enforced interception point.

## Best Possible Solution

Keep #9993 open as a plugin API/product-direction issue. The best path is a documented synchronous config-write interception point invoked by config.patch, config.apply, and likely config.set after the pending config is computed and built-in validation/preflight has passed but before writeConfigFile. It should provide current config, pending config, action, raw patch or full payload, baseHash, actor/client metadata, changed paths, and a typed reject result, with focused tests proving a plugin can reject direct RPC writes without changing the active config.
