## Summary

Current main already provides persisted per-session labels as the operator-facing alias/display-name mechanism in Control UI and gateway session APIs, which addresses the unreadable session-key problem without manual `sessions.json` edits.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw now has persisted per-session labels that cover the display-name/alias use case without manual `sessions.json` edits: `sessions.patch` stores a unique `label` (`src/gateway/sessions-patch.ts:219`), the Control UI Sessions page lets operators edit that label directly (`ui/src/ui/views/sessions.ts:517`), and gateway session resolution/tooling can target sessions by label (`src/gateway/sessions-resolve.ts:189`, `src/gateway/server.sessions-send.test.ts:161`). This also shipped in release `v2026.4.22` (`CHANGELOG.md:6640`). This is not a wildcard `session.aliases` config map, but the underlying alias/display-name capability requested here is already present on main and in the latest release. If a pattern-based config alias layer is still desired on top of labels, please open a new focused feature request for that specific workflow.
