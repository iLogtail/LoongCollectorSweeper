## Summary

Keep PR #54671 open. Current main still has the exact Control UI docs gap the PR targets: the pairing section only shows host-native `openclaw devices list` / `approve` commands and tells users to rerun `openclaw devices list`. Docker docs elsewhere confirm the `docker compose run --rm openclaw-cli ...` pairing commands are valid, but that guidance has not been carried into `docs/web/control-ui.md`, so the PR remains a focused useful docs fix rather than implemented, obsolete, or external.

## Best Possible Solution

Keep the PR open and refresh or merge the docs change into `docs/web/control-ui.md`, preserving the Docker Compose `openclaw-cli` pairing examples and the generalized rerun wording. A maintainer may also choose to include a Docker revocation example, but that is a follow-up polish item rather than a reason to close this PR.
