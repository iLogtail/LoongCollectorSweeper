## Summary

Keep open. Current main still shows only host-native `openclaw devices list` / `approve` commands in the Control UI device-pairing section, and still tells users to rerun `openclaw devices list`. Docker install docs do contain the Docker Compose equivalents, which supports the PR's proposed command pattern, but that does not make the Control UI page itself fixed or superseded.

## Best Possible Solution

Keep PR #54671 open and adapt it to the current `docs/web/control-ui.md` structure: add Docker Compose CLI equivalents in the Device pairing section and make the retry instruction refer to the matching `devices list` command. The existing Docker install troubleshooting commands can be used as the source pattern.
