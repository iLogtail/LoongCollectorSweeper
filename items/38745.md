## Summary

Current main still has the reported Telegram `/models` callback bug. The Telegram provider-list and pagination buttons emit raw provider ids such as `hf.co`, but `parseModelCallbackData()` still rejects `.` in `mdl_list_{provider}_{page}` callbacks, so the list handler is bypassed for `mdl_list_hf.co_1`. The related PRs #40251 and #46035 identified the same narrow fix, but both are closed unmerged.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix: either allow `.` in `mdl_list` provider parsing or parse the callback by the `mdl_list_` prefix plus the final page separator. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and dotted-provider pagination/list handling.
