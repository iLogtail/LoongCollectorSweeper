Keep open. Current main still has empty fire-and-forget catches on `initSessionState()` session lifecycle hook calls, and adjacent lifecycle paths already log equivalent hook-runner rejections. The PR is not ready as-is because the live head removes the `createSubsystemLogger` import while still using `createSubsystemLogger(...)`, and the bug rationale should be narrowed because normal plugin handler failures are already logged inside the hook runner.

## Best Possible Solution

Keep the PR open but request changes: restore the logger import/use the existing `session-init` subsystem logger correctly, remove unrelated formatting churn, and add a focused unit test around `initSessionState()` with mocked `runSessionEnd` and `runSessionStart` rejections. The maintainer decision should also clarify that handler-thrown errors are already logged by the hook runner; the missing improvement is consistency for rejected hook-runner promises on this specific fire-and-forget call site.
