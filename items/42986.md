## Summary

Keep open. Current main has Telegram conversation/topic binding and Telegram ACP thread spawn support, but native subagent `sessions_spawn(mode="session", thread=true)` still depends on a registered `subagent_spawning` hook, and Telegram does not register one. The linked #49073 was already closed as a duplicate of this earlier canonical tracker, so #42986 remains the right item for the missing native Telegram subagent thread-binding work.

## Best Possible Solution

Keep #42986 open. The best fix is to add Telegram subagent lifecycle hook wiring, likely a Telegram `subagent-hooks-api` registered from the bundled entry, that honors `channels.telegram.threadBindings.spawnSubagentSessions`, calls the existing Telegram session binding manager with child/current topic placement as appropriate, returns a delivery origin for the spawned topic, and cleans up on `subagent_ended`. Add targeted tests covering Telegram `sessions_spawn(mode="session", thread=true)` plus docs once shipped.
