## Summary

Keep #42986 open. Current main has Telegram conversation/thread-binding primitives and config keys, but the requested subagent session binding is not wired: `sessions_spawn(mode="session", thread=true)` still depends on a `subagent_spawning` hook, and Telegram does not register one. This is not a ClawHub close because it is a bundled Telegram channel parity gap using existing core/plugin seams, not an optional external capability. The issue is also not stale under the 60-day rule and has no protected-label or maintainer-author blocker.
