## Summary

Keep PR #69413 open. Current main still hydrates Signal native mention placeholders into `@uuid`/`@phone` text and then gates only through text regexes derived from configured mention patterns or agent identity names. It does not compare `dataMessage.mentions[].uuid` or `.number` to the bot account identity before `requireMention` gating, so the PR’s reported Signal group drop bug remains plausible and unimplemented on main.

## Best Possible Solution

Keep PR #69413 open. The best path is to merge or adapt its Signal-owned native mention detector and regression tests, then decide whether to fold the `canDetectMention` native-identity capability concern into this PR or track it as a narrow follow-up. Closing now would leave the current `requireMention` native Signal mention drop behavior unresolved.
