## Summary

Keep #36314 open. Current main has several related robustness mechanisms, but the issue asks for a broader Gateway-level watchdog/error-recovery/health-degradation feature set. Meaningful requested pieces remain missing, especially a configurable process watchdog with auto-stop/notification semantics and a richer health surface with memory/connection/error counters. The request is core Gateway reliability work, not a ClawHub plugin candidate.
