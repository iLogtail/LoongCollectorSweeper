## Summary

Keep #36314 open. Current main has meaningful robustness pieces, but they are partial: channel-level restart caps, platform supervisor restart policies, global unhandled rejection/exception handling, last-known-good config recovery, and health diagnostics. The requested Gateway-level configurable watchdog, configurable global recovery strategy map, richer HTTP health payload with memory/connection/error counters, and deliberate degraded startup mode are still not implemented as a cohesive core feature.

## Best Possible Solution

Keep the issue open as the umbrella for Gateway robustness, or split it into focused tracked work: a first-class gateway.watchdog config/schema/docs surface, restart-loop detection and notification semantics above platform supervisors, a configurable global recovery strategy model if maintainers want user policy there, richer health/RPC metrics with privacy-safe counters, and a deliberate degraded startup design that composes with existing last-known-good config and channel health recovery.
