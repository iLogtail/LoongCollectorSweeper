## Summary

Keep #29384 open. Current `main` still does not expose Docker sandbox GPU, IPC, raw Docker args, `capAdd`, `devices`, or `privileged` support for the requested `--gpus all` / `--ipc=host` workflow. OpenShell GPU support exists, but it is a different sandbox backend and does not satisfy Docker-backend users; the related `capAdd` PR is adjacent rather than a shipped duplicate.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox config extension lands. The best path is a narrow explicit contract such as `agents.defaults.sandbox.docker.gpus` plus a reviewed `ipc` policy or break-glass override, not broad raw-args passthrough by default. Update the public config type, strict Zod schema, generated schema/help/docs, resolver and config-hash behavior, runtime validation/audit messaging, Docker create-arg builder, and focused regression tests.

## Risks / Open Questions

- The requested Docker flags weaken sandbox isolation: GPU/device exposure and host IPC are materially different from the current default `capDrop: ["ALL"]`, `no-new-privileges`, read-only-root, and blocked namespace-join posture.
- A broad raw Docker args passthrough would bypass existing validation, config schema, audit messaging, and config-hash/recreate behavior unless carefully constrained.
- OpenShell GPU support may help some users, but closing this as implemented would leave Docker-backend AI/ML workflows without the requested support.
