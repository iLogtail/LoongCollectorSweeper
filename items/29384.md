## Summary

Keep #29384 open. Current main still does not expose Docker sandbox GPU, IPC, device, privileged, cap-add, or raw extra Docker create-arg support. The related #53108 is only an open capAdd PR and would not satisfy the central `--gpus all` / `--ipc=host` request.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox extension lands. A good implementation would add explicit, security-reviewed fields such as `sandbox.docker.gpus` and `sandbox.docker.ipc`, or a constrained extra-create-args mechanism, then update `SandboxDockerSettings`, the strict Zod schema, config resolution/hash behavior, runtime security validation/audit messaging, `buildSandboxCreateArgs`, public docs, and regression tests. The docs/config surface should clearly describe the isolation tradeoff.
