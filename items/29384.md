## Summary

Keep #29384 open. Current main still does not expose Docker sandbox GPU, IPC, device, privileged, cap-add, or raw extra create-arg support; the type, strict schema, docs, tests, and Docker argument builder only cover the existing hardening/resource/network/env/bind fields. OpenShell has a separate `gpu` option, but that is a different backend and does not satisfy Docker `--gpus all` or `--ipc=host`.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox extension lands. A good implementation would add explicit, security-reviewed fields such as `sandbox.docker.gpus` and `sandbox.docker.ipc`, or a narrowly constrained extra-create-args mechanism, then update `SandboxDockerSettings`, the strict Zod schema, config resolution/hash behavior, security validation/audit messaging, `buildSandboxCreateArgs`, public docs, and regression tests. Because GPU/device access and `--ipc=host` weaken isolation, the UI/docs/config surface should make that risk explicit instead of silently passing arbitrary Docker flags.
