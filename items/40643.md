## Summary

Current main already solves the PR's stated assistant-error replay poisoning problem with a broader implementation: failed assistant transport turns (`stopReason: "error"` or `"aborted"`) are removed before OpenAI/Anthropic replay, and empty zero-output error turns are retried before they can accumulate. The same protection is present in `v2026.4.24`, so this stale PR is obsolete.

## Best Possible Solution

Close this PR and keep the shipped main implementation: failed assistant transport turns are dropped before strict-provider replay, and empty error turns are retried before they accumulate. If routed Mistral strict9 behavior is still a live problem, it should be re-filed as a focused change against the current provider plugin/replay-policy architecture.
