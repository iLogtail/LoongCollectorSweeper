## Summary

Keep open. Current main handles the related OpenRouter `Key limit exceeded` case, but I could not verify handling for the exact reported `403 API key budget limit exceeded (monthly limit)` string. The current 401/403 classifier only bypasses auth when the message is already classified as billing, and the OpenRouter-specific matcher only covers `key limit`, not `key budget limit` or `budget limit exceeded`.
