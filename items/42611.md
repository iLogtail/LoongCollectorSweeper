## Summary

Keep #42611 open. Current main does not already contain a corporate proxy/SSL-inspection runbook for Linux systemd/VPS deployments, and the PR contains focused, reviewed docs work that matches real current daemon/proxy behavior. It needs rework for the current docs layout and docs i18n policy rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a focused refresh: rebase onto current main, drop the `docs/zh-CN/**` changes, add an English-only corporate proxy/SSL-inspection section to `docs/vps.md`, and cross-link the existing public environment/config docs for `NODE_EXTRA_CA_CERTS` and `models.providers.*.request.proxy`. The section should reflect current daemon behavior that installer-generated service environments preserve proxy vars, while still showing explicit `systemctl --user edit openclaw-gateway.service` overrides for operators who need manual systemd configuration.
