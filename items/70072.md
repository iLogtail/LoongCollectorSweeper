## Summary

Keep open. Current main has partial introspection through `openclaw memory promote-explain` and the Dreams UI’s short-term entry list, but it still does not satisfy the issue’s central request: automatic deep-pass artifacts do not list each considered candidate, rejected candidates are filtered out before the deep report is built, verbose scoring is logged to the gateway logger rather than written as structured workspace data, and there is no `openclaw memory dreaming doctor` command.

## Best Possible Solution

Keep the issue open. The best path is to add a first-class deep-candidate evaluation result in `memory-core` that preserves both passing and rejected candidates with gate reasons, use it to enrich `memory/dreaming/deep/YYYY-MM-DD.md`, optionally write structured verbose scoring under `memory/.dreams/`, and expose aggregate gate reachability through a CLI diagnostic such as `openclaw memory dreaming doctor` or an equivalent `openclaw memory status --deep` expansion.
