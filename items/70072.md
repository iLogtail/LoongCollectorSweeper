## Summary

Keep #70072 open. Current main partially helps with one-off candidate inspection through `openclaw memory promote-explain`, and related deep-promotion scoring has moved beyond raw `recallCount` alone, but the requested introspection artifact is not implemented: scheduled deep reports still contain only ranked/promoted counts, verbose candidate detail still goes to logger output rather than a workspace file, and there is no aggregate dreaming doctor/gate-reachability command.

## Best Possible Solution

Keep this issue open and implement an additive memory-core introspection path: extend the scheduled deep report with ranked candidates, thresholds, pass/fail reasons, and promotion outcomes; optionally write verbose structured traces under `memory/.dreams`; and add an aggregate CLI/gateway diagnostic that reuses the existing promote-explain scoring/pass logic for gate reachability.
