## Summary

Keep open. Current main has partial introspection via `memory promote`, `memory promote-explain`, `memory rem-harness`, and Dreams UI status counts, and the related #68882 zero-promotion bug is closed. But #70072 asks for per-run deep report candidate/rejection reasoning, workspace-persisted verbose scoring, and an aggregate dreaming doctor diagnostic; those are not implemented on main.

## Best Possible Solution

Keep #70072 open and implement this in `memory-core`: add an evaluation/diagnostic path that records all considered short-term entries with pass/fail gate reasons, use it to enrich `memory/dreaming/deep/YYYY-MM-DD.md`, optionally persist verbose scoring as structured workspace JSONL when `verboseLogging` is enabled, and add either `openclaw memory dreaming doctor` or an equivalent documented memory diagnostic command for aggregate threshold reachability.
