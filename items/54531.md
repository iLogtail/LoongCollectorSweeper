## Summary

Current main does not implement the requested force/persist-origin behavior. The central guard still suppresses external routing for internal webchat turns unless `ExplicitDeliverRoute` is set, session origin metadata has no immutable `persistedOriginChannel`/`persistedOriginTo`, and tests explicitly preserve the non-routing behavior for UI clients on main sessions. The linked PR #57802 from the issue discussion was closed unmerged per the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #54531 open. The maintainer path is to decide the desired product behavior, then implement a narrow force-origin or persisted-origin route at the session/routing boundary, with tests covering Telegram/Discord-style origin delivery while preserving explicit safeguards against stale-route or wrong-channel delivery. PR #57802 is useful prior art but was closed unmerged, so it should not be treated as the canonical resolution unless revived or replaced.
