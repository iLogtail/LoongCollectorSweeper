## Summary

Keep open. Current main treats Control UI scope upgrades as an intentional approval gate and surfaces clearer `PAIRING_REQUIRED` guidance, but I did not find a landed fix proving the reported "approve the request and the loop still continues" behavior is resolved. The available evidence points to better messaging and one commenter-reported workaround, not a high-confidence closure of this specific bug report.

## Evidence

- **Scope-upgrade reconnects are still intentionally blocked pending approval:** The Control UI gateway auth suite expects an already-paired operator device requesting broader scopes to fail the reconnect with `PAIRING_REQUIRED`, reason `scope-upgrade`, while keeping the prior approved scope set active and creating a pending upgrade request.

  - file: [docs/web/control-ui.md:55](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/web/control-ui.md#L55)
