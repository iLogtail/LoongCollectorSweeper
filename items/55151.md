## Summary

Keep open. Current `main` does not contain the proposed `inboundRootId` change, so this PR is not implemented on `main`, and the patch as described would regress `replyToMode: "off"` by restoring threaded delivery when no explicit reply target was requested. The underlying Mattermost bug still looks plausible, but this PR is not safe to close as fixed or obsolete.
