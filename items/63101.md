## Summary

Keep open. There is an open implementation PR, #63474, whose body explicitly uses closing syntax for this issue, so the issue should not be closed before that PR is reviewed, landed, or rejected. Current main still uses strict Feishu channel/account schemas that do not include the reported legacy `installs`/`plugins` keys or the later-commented legacy keys, and the current doctor legacy migrations do not include a Feishu stale-key cleanup. The related #63485 was closed unmerged, so it is not shipped evidence.

## Best Possible Solution

Keep this issue open and review #63474 as the current implementation candidate. The likely maintainer path is a narrow doctor/legacy-config migration that strips or migrates the legacy channel keys before strict schema validation, with regression tests and clear upgrade documentation; after a suitable PR lands, close this issue as implemented.
