## Summary

Keep open. The current `main` checkout still shows a strict Feishu config schema with no visible Feishu-specific compatibility/doctor migration surface for the legacy keys reported in the issue discussion, so there is not strong evidence that the v4.5/v4.8 upgrade break is fully fixed on `main`. The discussion link to PR #63485 is useful context, but the current tree does not show the corresponding Feishu doctor-contract hook that other channels use for this kind of repair.

## Evidence

- **Feishu config remains strict and excludes legacy keys:** `FeishuConfigSchema` and `FeishuAccountConfigSchema` are both `.strict()`, and the shared/top-level property lists do not include the reported legacy keys such as `installs`, `plugins`, `ackReaction`, or `threadSession`.
  - file: [extensions/feishu/src/config-schema.ts:155](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/extensions/feishu/src/config-schema.ts#L155)
  - sha: [f896c3935ef8](https://github.com/openclaw/openclaw/commit/f896c3935ef8d706fe4193443c5c5eede5e64178)
- **Runtime config reads only auto-heal channel config when a compatibility normalizer exists:** Config loading applies `applyRuntimeLegacyConfigMigrations(...)` before validation, so this issue would only be auto-fixed if Feishu exposed a channel compatibility normalizer.
  - file: [src/config/io.ts:1072](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/config/io.ts#L1072)
  - sha: [f896c3935ef8](https://github.com/openclaw/openclaw/commit/f896c3935ef8d706fe4193443c5c5eede5e64178)
- **Channel compatibility migrations depend on doctor-contract exports:** `applyChannelDoctorCompatibilityMigrations(...)` looks for `doctor-contract-api` / `plugin.doctor.normalizeCompatibilityConfig` hooks per channel. Without a Feishu hook here, strict validation would still see stale Feishu keys unchanged.
  - file: [src/commands/doctor/shared/channel-legacy-config-migrate.ts:24](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/commands/doctor/shared/channel-legacy-config-migrate.ts#L24)
  - sha: [f896c3935ef8](https://github.com/openclaw/openclaw/commit/f896c3935ef8d706fe4193443c5c5eede5e64178)
- **Other channels expose doctor-contract shims; Feishu does not in this checkout:** Discord and Slack both export `normalizeCompatibilityConfig` / `legacyConfigRules` from `doctor-contract-api.ts`, but `rg --files | rg 'doctor-contract-api'` in the current tree lists Discord/Slack/WhatsApp/iMessage/ZaloUser only, not Feishu.
  - file: [extensions/discord/doctor-contract-api.ts:1](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/extensions/discord/doctor-contract-api.ts#L1)
