## Summary

Keep open. Current `main` only replaces the consent card with a `FileInfoCard` on the successful accept path when a stored `consentCardActivityId` exists. It does not clean up the stale `FileConsentCard` on decline, expired/missing upload, conversation mismatch, or upload failure, so this PR still addresses behavior missing from `main`.
