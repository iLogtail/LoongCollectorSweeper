## Summary

Keep open. Current main has some Discord self-message protections and now clears queued followups during session rollover, but the specific fail-open path is still present: Discord bot identity lookup can still return undefined, and the main self-message filters only run when `botUserId` is known. The linked PR #46338 targeted this issue but was closed unmerged, so it is not a shipped canonical fix.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin/core queue boundary: add a reliable bot identity fallback or fail-closed behavior when identity is unavailable, keep the early/preflight self-filters, re-check Discord self-authored queued/replayed turns before followup execution, and add focused regression tests for failed `fetchUser("@me")`, self-authored replay, and `/new` queue cleanup.
