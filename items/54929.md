## Summary

Current `main` does not contain the PR's core behavior. Ordinary web chat turns still do not copy persisted image paths into the live dispatch `MsgContext`, and `getReplyFromConfig` still has no path that promotes inbound image turns from a text-only active model to `agents.defaults.imageModel`.

## Evidence

- **Web chat dispatch still omits persisted media fields for normal turns:** `chat.send` only computes `pluginBoundMediaFields` when `explicitOriginTargetsPlugin` is true, then spreads those fields into `ctx`. Normal web chat turns therefore still skip `MediaPath`/`MediaPaths` on the live dispatch context.
  - file: [src/gateway/server-methods/chat.ts:2263](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/chat.ts#L2263)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Current regression test still expects missing dispatch media:** The web chat test asserts the persisted transcript message has `MediaPath`/`MediaPaths`, but `mockState.lastDispatchCtx?.MediaPath` and `MediaPaths` remain `undefined`, which matches the unfixed behavior the PR is trying to change.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:1743](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/server-methods/chat.directive-tags.test.ts#L1743)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Reply model selection still lacks image-model promotion:** `getReplyFromConfig` resolves default, heartbeat, stored-session, and channel overrides, then immediately runs media/link understanding. The file has no `agents.defaults.imageModel` handling, no model-catalog lookup, and no vision-capability promotion before reply dispatch.
  - file: [src/auto-reply/reply/get-reply.ts:193](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/auto-reply/reply/get-reply.ts#L193)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Image understanding is still separate from reply-model routing:** The media-understanding runner skips only when the active model itself supports vision. That preserves a separate image-understanding path, but it does not switch the actual reply turn onto the configured image model.
  - file: [src/media-understanding/runner.ts:819](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/media-understanding/runner.ts#L819)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
