## Summary

Keep #60381 open. Current main already exposes `act:evaluate` with config gating, docs, and tests, but the requested `force?: boolean` click parameter is still absent from the browser act type, tool schema, normalization path, and Playwright click execution. Related #55598 only covers the arbitrary-JS/evaluate half, so it does not fully supersede this issue.

## Best Possible Solution

Keep the issue open but narrow the remaining work to `click.force`: add `force?: boolean` to `BrowserActRequest`, `BrowserActSchema`, legacy flattened act keys, and `normalizeActRequest`; thread it through `executeSingleAction` into `clickViaPlaywright`; pass it to Playwright `locator.click` and, if supported, `locator.dblclick`; add route/tool contract tests plus docs/tool-description guidance that `force` is a fallback after normal click/keyboard/scroll attempts. Preserve the existing `act:evaluate` implementation and keep #55598 as the closed evaluate-only reference.
