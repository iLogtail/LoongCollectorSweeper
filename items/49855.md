## Summary

Current main still does not implement the requested Feishu/Lark `type=file` to `type=media` fallback for iOS photo-library videos. The inbound path maps `file`, `video`, and `media` messages to Feishu message-resource `type: "file"`, and `downloadMessageResourceFeishu()` forwards that type directly with no 5xx retry or video-file heuristic. The linked fix PRs #49967 and #50164 are useful, exact attempts, but the provided context shows both remain open and unmerged, so the original bug report should stay open until a fix lands or a maintainer chooses a canonical tracker.

## Best Possible Solution

Keep #49855 open as the user-facing bug report until a Feishu fix lands. The likely code path is `extensions/feishu/src/media.ts`: add a narrowly scoped fallback that retries `messageResource.get` with Feishu `type=media` when an initial `type=file` download fails with a retryable Feishu/server 5xx, and add regression tests for 502 fallback plus the iOS file-type-video metadata heuristic. Maintainers should also choose between #49967 and #50164 or ask one contributor to adapt their PR to current main with tests.
