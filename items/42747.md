## Summary

Keep open. Current main still treats Slack credentials as static token inputs and does not implement Slack oauth.v2.access token rotation. The request is a concrete security-oriented gap in the bundled Slack channel, and the provided related items do not supersede it.

## Best Possible Solution

Keep the issue open and implement this as Slack-channel-owned auth/runtime work under extensions/slack: add a backward-compatible config/schema and secret contract, bootstrap/status/doctor or CLI support, an oauth.v2.access refresh path with tests for expiry, refresh failure, multi-account, SecretRef/plaintext handling, and Socket Mode/HTTP behavior. If maintainers decide this should live outside the bundled Slack plugin, capture that as an explicit product/API decision rather than closing this cleanup item.
