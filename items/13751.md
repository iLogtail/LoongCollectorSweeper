## Summary

Keep open. Current main still defaults Feishu inbound sender-name resolution to a live contact/profile API call, and the directory peer path still uses live contact user listing. Recent code suppresses one stale `contact:contact.base:readonly` prompt and documents an opt-out, but it does not satisfy the issue’s requested contact-free default for basic bot operation.

## Best Possible Solution

Keep #13751 open. The best product path is to make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise attribute with `open_id`/`user_id` without calling `contact.user.get`; make profile lookup an explicit opt-in. Live directory peer listing should also become static-first or explicitly opt-in for contact-directory access, with docs and regression tests updated around the new default.
