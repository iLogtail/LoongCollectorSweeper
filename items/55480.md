## Summary

Keep open. Current main still strips non-ASCII Discord channel names from the display slug, so the underlying bug is not implemented. The PR is directionally useful, but it is not merge-ready because it changes Discord `GroupSpace` from ID-first to slug-first and the current session display-name path also normalizes non-ASCII characters away.

## Best Possible Solution

Keep the PR open for rework. The best path is a focused display-only fix that preserves Unicode through the full Discord session display-name construction path, leaves config matching slugs unchanged, keeps `GroupSpace` ID-first, and adds regression tests for pure Unicode channel names, mixed ASCII/non-ASCII names, and ID-keyed guild policy resolution.

## Risks / Open Questions

- The PR's `GroupSpace` slug-first changes can regress ID-keyed `channels.discord.guilds` policy resolution for requireMention and tool overrides.
- The proposed fallback only handles purely non-ASCII names, while the PR body also claims mixed names like `baseline-검증` are preserved.
- On current main, preserving Unicode in `displayChannelSlug` alone may still be insufficient because session display-name normalization later strips non-ASCII text.
