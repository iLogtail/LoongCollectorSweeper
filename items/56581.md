## Summary

Keep PR #56581 open. Current main has some webchat/session-only follow-up handling, but the two central fixes from this PR are not implemented on main: exec child environments still inherit provider auth env vars through the host env sanitizer, and two-phase approvals with no approver route still expire before the two-phase accepted response. The PR is not landable as-is because it also bundles live-test HOME/profile changes with unresolved review comments, but the remaining product/security fixes are real and should not be auto-closed.

## Best Possible Solution

Keep the item open for maintainer handling. The best path is to refresh or split #56581: land the provider-auth env scrub in the exec host sanitizer with regression coverage, land the two-phase no-route approval pending fix with the #56398-style test, and either drop or separately resolve the live-test HOME/profile changes and their review comments. If narrower PRs such as #56398 and #56506 land first, then close this consolidation PR manually as superseded with links to those landed fixes.
