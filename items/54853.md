## Summary

Current `main` does not implement a general per-agent cross-channel account binding. `AgentConfig` still has no `feishuAccountId`/`channelBindings` field, and Feishu tool routing only reuses the ambient `agentAccountId` from the active session when that id already matches a real enabled Feishu account. That is a partial convenience path, not the requested agent-level cross-channel binding mechanism.

## Risks / Open Questions

- Operators can get partial behavior today by aligning account ids across channels, which may make the request look solved in some deployments while still failing for mismatched ids.
- There is a recent Feishu tool-account improvement on main, but it still does not add the requested config surface or general cross-channel mapping semantics.
