## Summary

Keep #62288 open. Current main has partial browser existing-session improvements, including managed `openclaw` as the default profile and clearer DevToolsActivePort messaging, but the canonical issue’s remaining acceptance bar is not met: diagnostics are still per selected profile, generic browser route errors lack machine-readable reason codes, and default target selection does not explicitly avoid browser/devtools/internal tabs.
