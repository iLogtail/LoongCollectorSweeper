## Summary

Keep open. Current main has partial fixes for this area: the managed `openclaw` profile is the default, `user` is an explicit existing-session profile, and DevToolsActivePort failures now produce a more specific fallback message. But the issue was recently marked by a maintainer as the canonical existing-session attach/fallback diagnostics tracker, and current code still does not meet the full acceptance bar: doctor/status remains profile-scoped rather than explicitly showing managed healthy versus existing-session unhealthy together, default tab selection still picks the first page tab without filtering browser-internal/devtools pages, and the related Desktop `spawn npx ENOENT` subcase remains open in #70529.

## Best Possible Solution

Keep #62288 open as the canonical tracker. The best product/code path is to finish the diagnostic model and existing-session attach behavior here: preserve explicit `profile="user"` failure without managed fallback, add actionable reason codes for DevToolsActivePort/no tabs/missing `npx`/wrong profile, make browser doctor/status report managed and existing-session health side by side, and skip devtools/browser-internal tabs during default target selection. Keep #70529 as the Desktop packaging subcase and treat #45812 as folded into this canonical issue.
