## Summary

Keep open. Current `main` does not implement cross-Gateway `sessions_send`, and the request still fits OpenClaw core rather than ClawHub. The code resolves and delivers sessions only through the currently connected gateway and that gateway's own session stores; the multiple-gateways docs describe those instances as intentionally isolated by profile/state/port.

## Evidence

- **`sessions_send` has no gateway-targeting parameter:** The tool schema only accepts `sessionKey`, `label`, `agentId`, `message`, and `timeoutSeconds`, then uses a single `callGateway` client for delivery. There is no `targetGateway`, host/port, or gateway-prefixed session-key support in the tool contract.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/tools/sessions-send-tool.ts#L37)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '28,120p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Session resolution is gateway-local:** `sessions.resolve` resolves by key via `resolveGatewaySessionStoreTarget(...); loadSessionStore(target.storePath)` and by sessionId/label via `loadCombinedSessionStoreForGateway(cfg)`. That means lookup happens against the current gateway's configured/discovered stores, not peer gateway instances.
  - file: [src/gateway/sessions-resolve.ts:101](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/gateway/sessions-resolve.ts#L101)
  - command: `nl -ba src/gateway/sessions-resolve.ts | sed -n '86,210p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Combined session store stays within one gateway config/state root:** `loadCombinedSessionStoreForGateway` merges session stores returned by `resolveAllAgentSessionStoreTargetsSync(cfg)` for the current config/state. It does not enumerate or RPC into other gateway processes.
  - file: [src/gateway/session-utils.ts:1011](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/gateway/session-utils.ts#L1011)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1011,1060p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Docs describe multiple gateways as isolated instances:** The maintainer docs say separate gateways should use isolated profiles, state directories, workspaces, and ports. That matches the current implementation and contradicts the requested cross-instance `sessions_send` behavior being already supported.
  - file: [docs/gateway/multiple-gateways.md:11](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/docs/gateway/multiple-gateways.md#L11)
