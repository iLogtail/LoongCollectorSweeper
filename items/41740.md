## Summary

Keep open. Current `main` at `350d3221800a18283915cd2f3b612ae04ddd490c` still routes Discord exec approvals through the operator-approvals gateway client, and that client still constructs `GatewayClient` without any `tlsFingerprint`. The shared gateway request path does have explicit local-TLS fingerprint hydration, so the reported self-signed `wss://` gap remains credible and appears unfixed on `main`.
