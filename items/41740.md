## Summary

Keep open. Current main still does not pass a local TLS fingerprint into the shared operator-approval GatewayClient path used by Discord native approvals and approval-button resolution. The related PR #46232 targeted this issue but is closed unmerged, and the fix is not present on current main.

## Best Possible Solution

Keep #41740 open and fix this once in the shared operator-approval gateway bootstrap/client path. Resolve the local gateway TLS runtime under the same conservative conditions as `callGateway` (local TLS enabled, target `wss://`, no CLI/env URL override or remote target) and pass `tlsFingerprint` into `GatewayClient`; add focused tests around `operator-approvals-client` or `client-bootstrap` for local TLS, override skip behavior, and current native approval callers.
