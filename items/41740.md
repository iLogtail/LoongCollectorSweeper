## Summary

Keep open. Current main still does not pass a TLS fingerprint through the operator/native exec approval GatewayClient path, while the normal gateway call path does resolve the local self-signed TLS runtime fingerprint. The related PR #46232 was closed unmerged, and its proposed fix is not present on main.

## Evidence

- **Operator approvals client constructs GatewayClient without tlsFingerprint:** createOperatorApprovalsGatewayClient resolves url/auth and passes url, token, password, client identity, scopes, and callbacks to GatewayClient, but no tlsFingerprint option is included.
  - file: [src/gateway/operator-approvals-client.ts:21](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/operator-approvals-client.ts#L21)

  - file: [src/gateway/client-bootstrap.ts:16](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/client-bootstrap.ts#L16)
