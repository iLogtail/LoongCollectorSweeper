## Summary

Keep open. Current `main` has timeout hardening around isolated cron agent turns, but I did not find strong evidence that the underlying ARM64 bug is fixed. The report says isolated `agentTurn` cron jobs never execute; current code still routes those jobs through `runEmbeddedPiAgent`, and the new safeguards mostly bound hangs rather than proving successful execution on Linux/macOS ARM64.
