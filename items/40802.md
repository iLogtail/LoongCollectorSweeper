## Summary

Keep open. Current `main` has Feishu routing hardening and some reply-related fixes, but I did not find strong evidence that this specific 400 `Invalid ids` DM-reply regression is fixed. The direct DM reply path still targets `user:${ctx.senderOpenId}`, existing tests cover target-type/session parsing rather than the reported long-`ou_...` corruption, and the issue is only 47 days old so `stale_insufficient_info` does not apply.
