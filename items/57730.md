## Summary

Keep open. Current `main` still does not surface a machine-readable `deliveryStatus` for `openclaw agent --json --deliver` on either the embedded or gateway path, and the issue already has a related open PR (#57755) referenced from the timeline.

## Evidence

- **Embedded `agent --json` emits only payloads/meta before delivery completes:** `deliverAgentCommandResult()` builds the JSON envelope with only `payloads` and `meta`, writes it immediately when `opts.json` is true, and only then continues into the actual outbound delivery path. There is no `deliveryStatus` field added to the JSON envelope.
  - file: [src/agents/command/delivery.ts:323](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/agents/command/delivery.ts#L323)

- **Outbound JSON envelope type has no `deliveryStatus` field:** `OutboundResultEnvelope` only defines `payloads`, `meta`, and optional `delivery`, and `buildOutboundResultEnvelope()` only returns those keys.
  - file: [src/infra/outbound/envelope.ts:5](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/infra/outbound/envelope.ts#L5)
