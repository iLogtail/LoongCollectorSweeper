## Summary

Keep #13962 open. Current main partially covers the context side by buffering unmentioned WhatsApp group messages and injecting that pending history on a later triggered reply, and existing bindings can route an entire group to an agent/model. It still does not implement mention-state-aware model routing, background-model processing for unmentioned group chatter, or per-group mention context sizing. The existing maintainer/Codex comment on the issue also reached the same keep-open conclusion on April 26, 2026.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing plus configurable group mention context sizing, or exposes a clear pre-gating plugin/API path that lets trusted plugins process unmentioned group messages with a background model before the main mentioned reply. The likely touch points are WhatsApp group gating, route/model selection, WhatsApp config types/schema/docs, and targeted tests for skipped unmentioned messages, triggered mentions, pending-history injection, and per-group sizing.
