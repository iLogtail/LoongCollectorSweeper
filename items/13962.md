## Summary

Keep #13962 open. Current main still only partially addresses the request: skipped unmentioned WhatsApp group messages are buffered for later prompt context, but they are not routed through a background model, there is no mention-state-aware model selection, and WhatsApp per-group config still lacks `mentionModel`, `backgroundModel`, or `mentionContextMessages`.

## Best Possible Solution

Keep the issue open until OpenClaw either implements mention-state-aware routing plus configurable per-group mention context sizing in the WhatsApp/group pipeline, or deliberately adds a trusted pre-gating plugin/API hook that lets unmentioned group messages be processed by a background model. Likely touch points are WhatsApp group gating, route/model selection, WhatsApp config schema/types/docs, and regression tests for skipped unmentioned messages, triggered mention context injection, background processing, and per-group sizing.
