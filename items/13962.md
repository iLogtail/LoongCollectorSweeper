## Summary

Keep #13962 open. Current main already buffers skipped WhatsApp group messages as pending context and supports channel/group model overrides, but it still does not implement per-mention model routing, background-model processing for unmentioned messages, or per-group mention context sizing. The existing maintainer/Codex comment on the issue already keeps it open for these same remaining gaps.

## Best Possible Solution

Keep the issue open until the WhatsApp/group pipeline supports mention-state-aware model routing plus configurable per-group mention context sizing. A conservative implementation should add typed config/schema/docs/tests for the new fields, preserve existing group safety gates, and either process unmentioned group messages through an explicitly safe background-model path or add a trusted pre-gating plugin/API hook that can do so without bypassing group policy.
