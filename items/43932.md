## Summary

Current `main` still does not implement #43932 end-to-end. Rate-limit errors shown to chat are formatted from raw error text only, with optional upstream retry wording, but they still omit provider/model/task/session context even where that context is already available in the runner.

## Evidence

- **Rate-limit formatter is still raw-string-only:** The shared formatter keeps a generic `RATE_LIMIT_ERROR_USER_MESSAGE` and `formatRateLimitOrOverloadedErrorCopy(raw)` only accepts the raw provider error string, so it has no inputs for provider/model/task/session metadata.

  - command: `nl -ba src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts | sed -n '25,110p'`
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Assistant error formatting does not add context for rate limits:** `formatAssistantErrorText(...)` routes rate-limit handling through `formatRateLimitOrOverloadedErrorCopy(raw)` and only uses `opts.provider` / `opts.model` for billing errors, not for rate limits.
  - file: [src/agents/pi-embedded-helpers/errors.ts:1065](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/agents/pi-embedded-helpers/errors.ts#L1065)
