## Summary

Keep open. Current main does not implement the proposed session working-context capsule or an equivalent persisted working-state object, and the linked umbrella #67511 remains open with concrete user confirmation of the compaction/resume pain point. The PR is draft and shape-only, but it is focused work toward a real core agents/session problem rather than stale cleanup, ClawHub scope, or a duplicate that has already shipped.

## Best Possible Solution

Keep #71135 open for maintainer review against #67511. The best path is to settle the core session storage shape, then either land the primitive with its focused tests or ask the author to add the first runtime integration point, likely `/status`, compaction lifecycle, exec-derived updates, or subagent spawn propagation. If maintainers prefer to track only the umbrella, that should be an explicit review decision linking #67511, not a conservative cleanup close.
