## Summary

Keep open. Current main has partial fallback UX, including structured model-fallback warning logs, lifecycle fallback events, UI fallback indicators, and final cooldown copy. A meaningful part of the request remains unresolved: transient provider retry still goes through an error-level log path before the retry delay, and external user-facing fallback notices are gated behind verbose mode rather than being a default transient/retrying state.
