## Summary

Keep open. Current main still lacks MathJax/KaTeX/LaTeX rendering in the Control UI chat markdown path. PR #61510 is a real related implementation attempt, but it is still open and unmerged, and its patch targets an older marked-based renderer while current main uses MarkdownIt, so the canonical feature issue should not be closed yet.

## Best Possible Solution

Keep #42840 open as the canonical feature request until math rendering lands. The best implementation path is to rework or replace #61510 against the current MarkdownIt-based Control UI renderer in ui/src/ui/markdown.ts, add a vetted math renderer such as KaTeX or MathJax with explicit DOMPurify tag/attribute allowlisting, preserve existing markdown/task-list/image/code behavior, avoid parsing ordinary dollar amounts as math, handle invalid LaTeX gracefully, and add focused tests in ui/src/ui/markdown.test.ts plus rendering coverage if needed.
