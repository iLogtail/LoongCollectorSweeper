## Summary

Keep #11460 open. Current main still supports WhatsApp outbound reaction writes only; it does not advertise or handle the generic `reactions` list/query action, does not persist or retrieve historical WhatsApp reactions, and the inbound WhatsApp monitor does not surface reaction add/remove events. The related PR #13641 was closed unmerged and only attempted inbound event surfacing, so it does not satisfy the query/history request.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin implements a complete reaction read path: retrieve or persist WhatsApp reaction history, advertise and handle the generic `reactions` message action for WhatsApp, surface inbound reaction add/remove events with correct routing and access checks, and add focused tests plus docs for CLI/tool behavior.
