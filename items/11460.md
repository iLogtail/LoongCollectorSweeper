## Summary

Current main still does not satisfy the issue’s central request. WhatsApp supports outbound `react`, but it does not expose the generic `reactions` query action, inbound reaction events are not wired in current main, and related PRs are not shipped replacements.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin has a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist WhatsApp reaction history, surface inbound add/remove events through normal routing and access checks, and add focused tests plus docs. PR #48570 can inform implementation, but it is not a shipped superseding fix for the query/history request.
