## Summary

Keep this PR open. Current main still lacks the requested `openclaw models status --probe --all` behavior: the CLI and routed parser do not accept or forward a status-level `--all` flag, and `buildProbeTargets` still selects a single representative model per provider. The PR remains a plausible implementation candidate for open feature request #63145, with branch review discussion showing the routed-parser and no-op-warning concerns were addressed and one small type-surface follow-up still worth reviewing.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #63145. The best path is to rebase onto current main, keep the CLI registration plus routed-parser forwarding plus status-command warning, add `probeAll?: boolean` to `AuthProbeSummary.options` if the branch still serializes it, preserve existing one-model behavior when `--all` is omitted, and land after focused tests cover the routed `--probe --all` path and probe target expansion.
