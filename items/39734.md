## Summary

Keep open. Current main handles Bedrock discovery and the concrete Claude 4.6 1M context-window metadata path, and the issue discussion notes that the original 1M preview flag may now be obsolete for GA models. But the requested broader capability, custom Bedrock `anthropic_beta` / provider-field passthrough into Bedrock Converse `additionalModelRequestFields`, is still not implemented on main.

## Best Possible Solution

Keep this issue open for a maintainer/product decision or a targeted Bedrock plugin implementation. The likely path is provider-owned support in `extensions/amazon-bedrock`: add a validated per-model or per-provider Bedrock request-field/beta passthrough, merge it into Bedrock Converse request payloads/additional model request fields, and cover it with Bedrock runtime tests and provider docs. If maintainers decide arbitrary Bedrock beta flags are out of scope, close explicitly with that product rationale rather than treating current main as implemented.
