## Summary

Keep open. Current main covers the concrete Bedrock Claude 4.6 1M context metadata path and direct Anthropic beta headers, but the requested configurable Bedrock `anthropic_beta` / `additionalModelRequestFields` passthrough is still absent from the bundled `amazon-bedrock` provider config, model schema, runtime wrapper, and docs.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A concrete fix should be Bedrock-owned: add a validated config/params surface for allowed `anthropic_beta` or Bedrock Converse additional request fields, merge it in the `amazon-bedrock` stream wrapper, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`. If maintainers decide arbitrary Bedrock beta passthrough is out of scope now that the 1M use case is GA, that should be an explicit product decision rather than automated cleanup.
