## Summary

Keep open. Current `main` still interpolates unsanitized `reason`, `target`, and `error` in `src/channels/logging.ts`, and those helpers are exported through the plugin SDK and used by multiple bundled channels. The PR targets a real hardening gap, but the existing review discussion also surfaces unresolved implementation concerns on the proposed patch, so there is not strong evidence to auto-close it.

## Evidence

- **Current main still lacks sanitization:** `logInboundDrop`, `logTypingFailure`, and `logAckFailure` on `main` directly interpolate `params.reason`, `params.target`, and `String(params.error)` into log messages with no control-character filtering.
