## Summary

Keep open. Current main still does not implement the PR's central cron delivery recovery for a numeric Telegram chat ID stored in `delivery.channel`, and the PR contains directly relevant regression coverage for #70758. It should not be closed as implemented or obsolete, though the proposed implementation needs maintainer review because the PR discussion raised a concrete allowFrom/explicit-mode security concern and the branch also includes unrelated macOS formatting changes.

## Best Possible Solution

Keep the PR open and request maintainer review/changes. The best path is a narrow cron delivery fix that preserves valid numeric Telegram `delivery.to` targets and handles or repairs legacy/misfiled numeric `delivery.channel` safely, with regression tests in `src/cron/isolated-agent/delivery-target.test.ts`; address the allowFrom/explicit-mode concern before landing and avoid bundling unrelated macOS formatting churn.
