## Summary

Keep open. Current main still has the exact remote `configRemote` timeout cap described in #65355: `gateway probe --timeout 30000` is parsed as an overall timeout, but non-loopback probe targets still receive `Math.min(1500, overallMs)`. The prior shipped fix only changed active local loopback behavior, and the current tests explicitly preserve the non-local 1500ms cap rather than covering the requested remote behavior.

## Best Possible Solution

Keep the issue open and change the gateway status/probe budget policy for remote `configRemote` and explicit non-loopback targets. A likely fix belongs in `src/commands/gateway-status/helpers.ts`, with tests replacing the current `configRemote -> 1500ms` expectation and command/output coverage if per-target effective budgets are surfaced.
