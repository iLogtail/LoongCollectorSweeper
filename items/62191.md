## Summary

Keep open. Current main has related iMessage fixes for destination_caller_id self-chat classification and wrapped imsg RPC text, but it still does not implement the remaining U+FFFD/control-prefix normalization requested by this PR. The PR is not obviously merge-ready because the posted global scrub is lossy, but the underlying OpenClaw bug is still real enough that this item should not be closed as implemented or superseded.

## Best Possible Solution

Keep #62191 open, but ask for a narrower implementation: strip only the known leading U+FFFD/control-byte attributedBody artifact before echo-cache key comparison, preserve normal message content, and add focused echo-cache/self-chat regression tests for prefixed reflected text and collision cases.
