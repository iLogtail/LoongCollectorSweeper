## Summary

Keep open. Current main has related iMessage fixes for DM self-chat false positives and wrapped imsg RPC text, but it does not implement the remaining requested U+FFFD/control-prefix normalization in the echo cache. The true self-chat `is_from_me=true` path still depends on an echo-cache match and dispatches on cache miss, so a garbage-prefixed reflected agent reply can still miss the cache. The PR likely needs refinement because the proposed global scrub is lossy, but the item should not be closed as implemented or superseded.
