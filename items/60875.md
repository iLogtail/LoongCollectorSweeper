## Summary

Keep PR #60875 open. Current main still lacks the typed manifest failure reason and still promotes every failed manifest load to an error, while workspace discovery can still create candidates from loose files or index-based directories under `<workspace>/.openclaw/extensions`. The broader #60686 workspace-recursion problem appears partly mitigated by scoping discovery to `.openclaw/extensions`, but this PR's narrower diagnostic downgrade is not implemented on main.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch: add a typed manifest-load failure reason to OpenClaw and bundle manifest loaders, downgrade only workspace-origin `not-found` manifest diagnostics to `warn`, preserve `error` for unsafe paths, parse errors, invalid manifests, config/global origins, and add focused registry tests for both downgrade and retained-error paths.
