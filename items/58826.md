## Summary

Keep open. Current `main` already has configurable tool-loop detection, which partially addresses the issue's runaway-loop concern, but I found no built-in token/request budget enforcement for per-request, per-session, per-user, or daily caps. Existing usage/quota surfaces are observational (`/status`, `/usage`, provider-usage windows), not active limit controls, so the core request is still not implemented on `main`.
