## Summary

Keep open. Current main has related Control UI theming support, but it does not ship the requested built-in Accessible high-contrast, colour-vision-deficiency-safe preset. The built-in theme registry and pickers only expose Claw, Knot, Dash, plus a browser-local Custom slot; the default semantic tokens still use green for OK and red for danger/destructive. Related #49775 addresses muted text contrast, but it does not supersede this specific preset request.

## Best Possible Solution

Keep the issue open as focused Control UI accessibility/product work. A complete fix would add an `accessible` theme family through `ui/src/ui/theme.ts`, `ui/index.html`, the full and quick settings pickers, CSS variable blocks in `ui/src/styles/base.css`, and focused tests/docs. It should not be closed as implemented by the browser-local Custom/tweakcn slot or by the separate muted-text contrast fix.
