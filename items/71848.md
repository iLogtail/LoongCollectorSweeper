## Summary

Keep #71848 open. The report is fresh, detailed, and not covered by the existing macOS launchd hardening: current main still serializes TMPDIR into the LaunchAgent environment, including session-shaped /var/folders values, while the issue specifically identifies a stale hardcoded TMPDIR as a likely contributor. The discussion also cites launchd/TCC and memory-pressure behavior that may need code, doctor, or docs work rather than conservative cleanup.

## Best Possible Solution

Keep the issue open and investigate the macOS LaunchAgent install/runtime path. The most concrete repo change to evaluate is replacing persisted session TMPDIR with a stable temp directory or omitting it from LaunchAgent plists, plus adding doctor/service-audit detection for stale /var/folders TMPDIR values and macOS gateway docs for launchd TCC/Full Disk Access and supported Node runtime expectations.
