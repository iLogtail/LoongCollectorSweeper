## Summary

Keep #70341 open. The linked bug is real and current main still does not implement the PR's central Windows PowerShell cmdlet allowlist fallback: bare cmdlets without a resolved filesystem path still cannot match a PowerShell-path allowlist entry or generate an allow-always entry. The PR is not ready to close as implemented or obsolete, and the provided review discussion identifies a remaining execution-path bug that should be fixed before merge.

## Best Possible Solution

Keep the PR open as useful work, but require a revision that fixes both halves of the flow: cmdlet-aware allowlist/allow-always matching in infra, plus node-host execution that preserves the PowerShell wrapper for allowlisted bare cmdlets. Add regression coverage for a Windows shell payload such as `Get-ChildItem -LiteralPath ...` after an allow-always/manual allowlist entry, then merge this PR or supersede it with a narrower maintainer-authored fix.
