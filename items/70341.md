## Summary

Keep #70341 open. Current main still lacks the proposed Windows PowerShell cmdlet allowlist fallback, so the PR is not obsolete or implemented elsewhere. The PR also is not ready to close as solved because the submitted approach appears to leave a real execution-path bug: once a bare cmdlet is considered allowlist-satisfied, the Windows node-host path can still rewrite execution to the bare cmdlet argv and drop the PowerShell host.

## Best Possible Solution

Keep the PR open, but require a revision that treats bare PowerShell cmdlets as allowlist-matchable while preserving the PowerShell execution host in the Windows node/gateway rerun path. Add regression coverage for both allowlist evaluation and `resolveSystemRunExecArgv` so an allowlisted `Get-ChildItem ...` does not become a direct `Get-ChildItem` process spawn.

## Evidence

A PowerShell cmdlet like `Get-ChildItem` has no filesystem executable path, matching the bug described by #52952 and #70341.
