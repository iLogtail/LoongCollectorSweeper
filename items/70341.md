## Summary

Keep #70341 open. Current main still lacks the PR's Windows PowerShell cmdlet allowlist fallback: unresolved bare cmdlets cannot match a PowerShell-path allowlist entry or derive an allow-always pattern. The PR is still relevant, but the existing review also identifies a real execution-layer gap that should be fixed before merge.

## Best Possible Solution

Keep the PR open and ask for a revised patch that implements the cmdlet allowlist and allow-always fallback, keeps PowerShell-path resolution in an infra-safe location, preserves the PowerShell execution wrapper for cmdlet matches, and adds regression coverage for allowlist evaluation, allow-always persistence, and the node-host Windows execution path.
