## Summary

Keep open. Current `main` still discovers and includes on-disk session stores for unconfigured/retired agent directories, and Control UI still hydrates/preserves explicit stale agent session keys instead of validating them against configured agents. That means the reported inconsistency is not implemented away on `main`, and the newer comment's broader stale-session restoration variant also appears plausible in current code.

## Evidence
