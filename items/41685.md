## Summary

Keep open. Current main still has a concrete path where disk-only or unregistered agent session stores are discovered and surfaced through `sessions.list`, and the Control UI normalization still preserves explicit non-default agent session keys such as `agent:local:main`. Related issue #41686 only covers directory cleanup, while PR #41687 remains open/unmerged and therefore does not make this implemented on main.
