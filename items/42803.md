## Summary

Keep open. Current main partially addresses Feishu abort handling by routing abort-like text such as `/stop` to a `:control` sequential lane, but the issue asks for Feishu text commands `/stop`, `/new`, and `/status` to bypass the active-run queue. Current code still routes `/status` and `/new` through the normal per-chat queue, and the current test suite explicitly expects `/status` to stay on the normal lane.
