## Summary

Current main does not implement this PR. The registry/control paths still coerce omitted `runTimeoutSeconds` to `0`, and `resolveAgentTimeoutMs` treats `0` as explicit no-timeout instead of falling back to the configured default. The ACP `sessions_spawn` path can still pass `undefined` into `registerSubagentRun`, so the reported behavior remains plausible on main.
