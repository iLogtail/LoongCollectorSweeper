## Summary

Keep #29707 open. Current main still does not provide native Amazon Nova prompt caching on Bedrock: Nova model IDs are treated as non-Anthropic Bedrock models, explicit cacheRetention is dropped or overwritten with "none", and tests/docs still assert that Nova/Mistral are forced to no cache. The related #20815/#20866 work fixed malformed Nova cachePoint crashes by disabling non-Anthropic Bedrock cache metadata, not by adding a Nova-native cache path.

## Best Possible Solution

Keep the issue open for the bundled Amazon Bedrock provider owner path. A proper fix should verify AWS Nova prompt-caching semantics, add Nova model-family eligibility in the shared cache-retention resolver, pass through or translate cacheRetention for Nova without Anthropic cachePoint injection, preserve the #20815 crash fix with regression tests, and update the prompt-caching docs once the supported behavior is proven.
