## Summary

Keep open. Current main supports image attachments in the Control UI through file select, paste, drag/drop, thumbnail preview, and `chat.send` attachment plumbing, but it still has no screenshot/scissors capture button or region-capture flow in the chat composer. The related title-search items are adjacent Control UI button requests, not a canonical duplicate. This is a core Control UI surface gap rather than a ClawHub close because the documented plugin surfaces do not expose a chat-composer UI extension point.

## Best Possible Solution

Implement this in the Control UI chat composer: add a screenshot/scissors button beside the existing paperclip control, capture screen or region pixels via the appropriate browser/native path, convert the result into the existing `ChatAttachment` data URL shape, reuse the current thumbnail/remove UI and `chat.send` image attachment pipeline, and add focused UI/controller tests for capture success, cancel/error handling, replacement/removal, and send behavior. If maintainers prefer this to be plugin-owned, first add and document an explicit Control UI composer extension API; current plugin surfaces are not enough.
