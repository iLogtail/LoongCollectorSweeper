## Summary

Keep open. Current `main` still limits Control UI uploads to `image/*`, renders outgoing webchat attachments as images only, and the gateway `chat.send` path still persists transcript media fields from images only while `parseMessageWithAttachments()` drops non-image payloads. The linked refresh PR #70947 is useful context, but it does not make #57707 implemented on `main` or otherwise eligible for conservative auto-close.
