## Summary

Keep open. Current main still only lets `before_model_resolve`/legacy `before_agent_start` hooks steer provider/model selection, not auth profile preference. The runner's auth profile order is still based only on the incoming session/user auth profile, and the live-switch comparison still uses that earlier preferred profile rather than the active profile. The PR is an active, focused core/plugin API change with tests and docs in the PR context, not a ClawHub-only item or duplicate of the related auth failover reports.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a core/plugin API implementation that adds `authProfileOverride` to the hook result contract, merges it with the same priority rules as provider/model overrides, applies it only as a non-user-pinned preference in runner auth profile ordering, and documents/tests the explicit user profile guardrail.
