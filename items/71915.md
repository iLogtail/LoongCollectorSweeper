## Summary

Keep #71915 open. Current main has post-2026.4.24 hardening for the bundled runtime-deps loop and DeepSeek V4 replay handling, but the issue's direct DeepSeek V4 gateway CPU/memory symptom is not proven fixed or duplicated. The maintainer comment and related PR #72132 both explicitly say more production evidence is needed before closure.

## Best Possible Solution

Keep the issue open and request a repro on current main or the next build with openclaw --version, the last gateway log lines, process CPU/memory output, and ideally a Node CPU profile. If the logs show repeated bundled runtime-deps staging, mark it as duplicate/superseded by the fixed #71818 class; otherwise investigate the direct DeepSeek V4 gateway startup/stream path and land a production fix with a focused regression or performance guard.
