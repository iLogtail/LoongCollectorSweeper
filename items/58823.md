## Summary

Keep PR #58823 open. Current main still implements the opposite precedence from this PR in both the shared subagent model helper and isolated cron path, and checked-in tests assert that current behavior. The PR is not implemented on main and is not clearly superseded; it remains a focused fix candidate for #58822, but it conflicts with the behavior intentionally merged in #58003, so maintainer product judgment is needed.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended model precedence contract first; if the global subagent default is meant to be a true subagent default, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage for explicit spawn override, per-agent `subagents.model`, global `agents.defaults.subagents.model`, and fallback to agent primary model. If #58003's named-agent behavior is intended, update the subagents docs and close #58822/#58823 with that rationale.
