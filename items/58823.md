## Summary

Keep PR #58823 open. Current main still implements and tests the opposite precedence from this PR in both the shared subagent helper and the isolated cron path. The docs and linked bug #58822 support the PR's intended behavior, but #58003 intentionally shipped the current named-agent precedence and the latest PR discussion raises model allowlist/security concerns, so this needs maintainer contract judgment rather than conservative cleanup.

## Best Possible Solution

Keep the PR open. Maintainers should decide the subagent model precedence contract explicitly: if global `agents.defaults.subagents.model` is intended to override the caller/agent primary model, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage and resolve the allowlist validation question; if #58003's named-agent precedence is intended, update `docs/tools/subagents.md` and close #58822/#58823 with that rationale.
