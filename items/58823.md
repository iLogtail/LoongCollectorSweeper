## Summary

Keep PR #58823 open. Current main still implements and tests the opposite subagent model precedence from this PR, while user-facing docs still support the reporter's expectation that `agents.defaults.subagents.model` should affect spawned subagents. The PR is a focused candidate for open issue #58822, but it reverses the behavior intentionally merged by #58003 and the latest PR discussion raises model allowlist concerns, so this needs maintainer product/security judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended subagent model precedence contract first; if global `agents.defaults.subagents.model` should be a true subagent default, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage for explicit spawn override, per-agent `subagents.model`, global subagent default, fallback to agent primary/default model, and allowlist behavior. If #58003's agent-primary precedence is intended, update `docs/tools/subagents.md` and close #58822/#58823 with that rationale.
