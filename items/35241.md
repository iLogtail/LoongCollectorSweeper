## Summary

Keep PR #35241 open. Current main still does not expose `--session-key` on `openclaw agent`, while the embedded agent command, session resolver, Gateway protocol, Gateway server validation, and cron CLI already understand session keys. The PR addresses a real remaining CLI surface gap with downstream demand in the discussion, but it should be refreshed and should include validation parity before merge.

## Best Possible Solution

Keep the PR open. The best path is a small core CLI change that registers `openclaw agent --session-key <key>`, adds `sessionKey` to `AgentCliOpts`, accepts it as a valid selector, passes it through both Gateway and embedded fallback paths, mirrors malformed-key and `--agent` conflict validation consistently, and updates CLI docs/help plus focused tests. Treat `--no-fallback` as a separate follow-up.
