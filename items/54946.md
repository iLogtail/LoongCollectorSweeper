## Summary

Keeping this open. The report is recent and plausible, but it is too underspecified to close, and current `main` only shows related Ollama/TUI hardening rather than strong proof that this exact Windows Docker -> host Ollama TUI hang is fixed or obsolete.

## Evidence

- **Issue body is too thin to verify or dismiss:** The report repeats the same sentence across summary/steps/expected/actual, provides no logs, no screenshots, no install method, and no concrete config details beyond `windows11-docker-openclaw/windows11-ollama` and `qwen3:8b`. That is not enough to conservatively call it fixed or non-reproducible.
- **Current docs still document remote Ollama configuration footguns:** `docs/providers/ollama.md` explicitly warns remote Ollama users not to use `/v1`, and to use the native Ollama URL instead. That means the reported setup still has real configuration-sensitive failure modes today.
