## Summary

Keep #69312 open. Current main still has the reported false-positive path: `splitMediaFromOutput()` skips only fenced code spans, then trims indentation before checking for a `MEDIA:` directive, so 4-space/tab indented code-block lines can still be extracted. The PR is a focused bug-fix attempt with useful regression coverage and active follow-up, not obsolete or superseded.

## Best Possible Solution

Keep the PR open and request a maintainer-quality revision: implement indented-code-block protection in `splitMediaFromOutput()`, preserve code-line whitespace during cleanup, add focused media and fence regression tests, and explicitly reconcile lowercase `media:` semantics with current reply-delivery and streaming-directive behavior.
