## Summary

Keep PR #69312 open. Current main still has the reported MEDIA false-positive path: `splitMediaFromOutput()` only protects fenced code spans, then removes leading indentation before a case-insensitive `MEDIA:` line check, so 4-space/tab indented code-block lines remain eligible for extraction. The PR is recent, focused, linked to #69311, and its discussion shows concrete iteration on review findings rather than stale or low-signal work.

## Best Possible Solution

Keep the PR open and have a maintainer review or land a corrected version that adds indented-code protection in `src/media/parse.ts`, preserves code indentation during cleanup, adds the missing regression tests, and resolves the streaming-buffering case-sensitivity coupling if MEDIA parsing becomes case-sensitive.
