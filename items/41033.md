## Summary

Keep open. Current `main` deliberately treats `/agent` as outside the `tools.fs.workspaceOnly` root for sandboxed file tools, and the docs/tests describe `workspaceAccess: "ro"` as a split-mount mode where `/agent` is a separate read-only agent-workspace mount, not an alias of `/workspace`. This PR would widen that boundary, so there is no strong supported close reason.
