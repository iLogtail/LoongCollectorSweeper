## Summary

Keep #60841 open. Current main still applies `toolsAllow` only as a final post-filter after `createOpenClawCodingTools()` has already applied restrictive tool policy, so it cannot re-add `exec`, `read`, or `write` once `tools.allow: ["group:openclaw"]` filtered them out. The linked fix PR #60842 is open/unmerged in the provided GitHub context, and its described forwarding/force-allow behavior is not present in current main.

## Best Possible Solution

Keep #60841 open and review/land #60842 or an equivalent maintainer-owned fix. The implementation should pass the embedded run's `toolsAllow` into the core tool construction/policy stage, re-expose only explicitly requested known core tools, preserve explicit deny/sandbox/subagent/provider/runtime restrictions, and add regression coverage for `tools.profile: "full"`, `tools.allow: ["group:openclaw"]`, and `toolsAllow: ["exec", "read", "write"]`.
