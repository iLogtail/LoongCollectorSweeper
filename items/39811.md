## Summary

Keep #39811 open. Current main has runtime fallback observability, but it still does not make `openclaw config validate` reject or warn on arbitrary configured model names. Agent defaults and provider model entries remain schema/shape validated only, and the model selection/listing path still synthesizes configured allowlist refs that are absent from the runtime catalog. Related #39818 and #45294 are closed, but they do not supersede the remaining config-validation request.

## Best Possible Solution

Keep the issue open. The best path is to add catalog-aware validation or doctor/startup warnings for configured `agents.defaults.model.primary`, fallback refs, channel/agent model overrides, and provider-owned model refs where the provider catalog can be trusted, with explicit semantics for custom and dynamic providers. Runtime fallback visibility should remain, but it does not replace config validation.
