## Summary

Current `main` only partially addresses the report. OpenClaw now warns when a configured primary model cannot be resolved at startup and logs a warning when fallback succeeds after a `model_not_found` error, but `openclaw config validate` still does not validate configured model refs against provider catalogs, the model schema still accepts arbitrary strings, and docs still direct users to manually verify availability with `openclaw models list`. That is not strong enough evidence to close #39811 as implemented or obsolete.
