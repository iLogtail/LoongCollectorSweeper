## Summary

Keep open. Current `main` still hard-fails workspace bootstrap when a template file cannot be read, so the PR’s runtime fallback behavior is not already implemented. `main` has added packaging/release guards to make missing templates less likely, but those guards are preventive rather than a runtime fix, and the PR as written still has unresolved correctness concerns from review.

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect: the runtime fallback proposed by the PR is not present in `src/agents/workspace.ts` on current `main`.
- Closing as `cannot_reproduce` would be weak: current `main` lowers recurrence via packaging checks, but an incomplete install would still hit the same `Missing workspace template` failure path.
- Merging the PR as-is would still need revision because the reviewed implementation masked non-ENOENT I/O errors and introduced non-neutral fallback template contents that differ from the shipped templates in `docs/reference/templates/IDENTITY.md` and `docs/reference/templates/USER.md`.
