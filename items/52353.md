## Summary

Keep open. Current main has improved timeout handling by surfacing timeout payloads to users and making timeout behavior more configurable, which overlaps with the related #60123/#34644 cleanup, but it still does not emit a plugin-observable `lifecycle:error` agent event with `errorType: "timeout"` when the embedded run timeout timer fires.

## Evidence

- **Timeout handler still only logs and aborts:** The embedded attempt timeout path schedules `abortTimer`, logs `embedded run timeout...`, flags compaction timeout when relevant, and calls `abortRun(true)`. There is no `emitAgentEvent` or `params.onAgentEvent` call in this timer path.
