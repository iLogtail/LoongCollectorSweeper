## Summary

Keep #18967 open. Current main still lacks the requested parent-scoped native send path for sub-agents. The existing implementation has broad `sessions_send`, configurable subagent allow/alsoAllow escape hatches, tree/agent/all session visibility, ACP-only parent streaming on spawn, and announce/yield orchestration, but none of those provide `sessions_send_parent` or a literal `parent` target with direct-parent-only, same-tree, no-chaining enforcement.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool, or a literal `parent` target on `sessions_send`, that resolves from stored direct-parent metadata, enforces same OpenClaw agent tree and direct-parent-only access, never auto-forwards further up the tree, and adds resolver/policy/visibility tests plus user-facing docs. The existing broad `sessions_send` opt-in and ACP parent streaming can stay as separate capabilities but should not close this issue.
