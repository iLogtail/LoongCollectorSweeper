## Summary

Current main does not implement the requested parent-scoped sub-agent send path. It can optionally expose the general `sessions_send` tool to sub-agents, but that is broader than the requested direct-parent-only/no-cross-agent/no-chaining capability, and default visibility is descendant-tree scoped rather than parent scoped.

## Evidence

- **Sub-agent default deny still includes sessions_send:** `SUBAGENT_TOOL_DENY_ALWAYS` still lists `sessions_send` with the comment that sub-agents communicate through the announce chain. Config can remove it from the deny list, but that exposes the existing general tool rather than a parent-only capability.
