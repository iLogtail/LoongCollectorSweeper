## Summary

本 Issue 请求为 flusher_kafka_v2 增加类似 SLS 的本地磁盘缓存（DiskBufferWriter）能力，以应对 Kafka/网络异常及日志快速轮转导致的数据丢失问题。维护者已在讨论中确认该需求价值并明确邀请社区贡献。当前 main 分支尚未实现该特性，属于合理、清晰且可在仓库内通过代码变更落实的增强请求，应保持开启以等待设计讨论或贡献者 PR。

## Best Possible Solution

保持开启以跟踪该增强需求。建议贡献者参考 flusher_sls 的 DiskBufferWriter 实现或 LoongCollector 现有的队列/缓存架构，为 flusher_kafka_v2 设计并实现本地磁盘缓存机制。下一步需明确配置参数（如缓存路径、最大容量、刷盘策略、淘汰规则）、与现有 checkpoint 机制的兼容性，以及异常恢复时的数据重放逻辑。欢迎提交设计草案或直接贡献 PR，维护者可在 PR 阶段进行架构评审与合并。
