## Summary

本 Issue 请求为 flusher_kafka_v2 增加本地磁盘缓存（DiskBuffer）能力以提升网络或 Kafka 异常时的灾备与抗压能力。维护者已确认该需求有效并明确邀请社区贡献。当前 main 分支尚未内置该机制，属于合理且待实现的增强特性，符合保守保持开启条件。

## Best Possible Solution

保持开启以跟踪该增强需求。建议维护者添加 help wanted 标签以吸引社区贡献。贡献者可提交设计方案（如扩展底层 Queue 架构支持磁盘落盘，或为 flusher_kafka_v2 增加本地缓存模块及刷盘策略）或直接提交 PR 实现。在功能落地前，用户可通过调大内存队列参数、优化 Checkpoint 间隔或结合外部可靠传输组件作为临时缓解措施。

## Evidence

- **maintainer_acknowledgment:** 维护者在评论中明确表示该功能需要社区贡献，确认当前未实现且欢迎共建，未标记为拒绝或重复。
- **feature_gap_on_main:** 当前 main 分支 flusher_kafka_v2 实现仍依赖内存队列与 Checkpoint 机制，源码中未见 DiskBuffer 或本地磁盘持久化缓存逻辑，功能缺口确实存在。
