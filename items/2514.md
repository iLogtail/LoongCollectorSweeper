## Summary

本 Issue 请求为 flusher_kafka_v2 增加本地磁盘缓存（DiskBufferWriter）能力，以应对 Kafka/网络异常及日志快速轮转导致的数据丢失问题。维护者已在讨论中确认该需求价值并明确邀请社区贡献。当前 main 分支尚未实现该特性，属于清晰、合理且需核心 Flush/Queue 架构改动的增强请求，应保持开启以等待设计讨论或社区 PR。

## Best Possible Solution

保持开启以跟踪该增强需求。建议贡献者参考 flusher_sls 的磁盘缓存实现或 LoongCollector 现有的队列/缓存架构，为 flusher_kafka_v2 设计并实现本地磁盘缓存机制。下一步需明确配置参数（如缓存路径、最大容量、刷盘策略、淘汰规则）、与现有 checkpoint 机制的兼容性，以及异常恢复时的数据重放逻辑。欢迎提交设计草案或直接贡献 PR，维护者可在 PR 阶段进行架构评审与合并。

## Evidence

- **maintainer_acknowledgment:** 维护者 messixukejia 在评论中明确回复“这个要社区来贡献了。可以来参与共建，也可以社区里号召下。”，确认需求有效且尚未实现，属于待社区接手的增强项。
- **feature_gap_on_main:** 当前 main 分支的 flusher_kafka_v2 实现依赖内存队列与 checkpoint 机制，未内置类似 SLS DiskBufferWriter 的本地磁盘持久化缓存能力，符合 Issue 描述的功能缺口。
