## Summary

Keep open. Current `main` shows partial follow-up work around missing-parameter diagnostics and narrow tool-argument repair, but not a general fix for long-session `exec`/`write`/`sessions_spawn` calls arriving as `{}`. I did not find strong evidence that the reported behavior is implemented or obsolete on `main`.

## Evidence

- **Linked follow-up shipped as diagnostics, not a root-cause fix:** The changelog entry that includes `#55317` says OpenClaw now includes value-shape hints in missing-parameter tool errors. That matches better error reporting, not reliable preservation of dropped tool arguments.
  - file: [CHANGELOG.md:1398](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/CHANGELOG.md#L1398)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Runtime missing-param checks are scoped to file tools:** `REQUIRED_PARAM_GROUPS` only defines `read`, `write`, and `edit`, and the wrappers are only applied by the read/write/edit tool constructors. I found no equivalent wrapper for `exec` or `sessions_spawn`.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/pi-tools.params.ts#L75)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **File-tool wrappers only:** Current tool construction applies `wrapToolParamValidation` to sandboxed/hosted write+edit tools and `assertRequiredParams` to read. This is still limited to file tools.
  - file: [src/agents/pi-tools.read.ts:637](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/pi-tools.read.ts#L637)

I found no generic long-session/channel repair path for other providers.
  - file: [src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts:297](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts#L297)
