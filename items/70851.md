## Summary

Keep PR #70851 open. Current main does not implement the picker filtering this PR proposes: the chat picker still filters only global/unknown and cron rows before adding session options. The gateway already has `spawnDepth` lineage metadata, but the UI row type/controller path does not expose or use it in the picker. The PR is also not cleanly superseded by #69552, which is closed unmerged and narrower, and the review comments identify changes needed before this patch is a final answer.

## Best Possible Solution

Keep the PR open, but ask for changes or maintainer review: implement an explicit internal-session visibility/classification contract from the gateway/session source, propagate that field or existing `spawnDepth` safely through the UI session row path, hide internal rows by default in the chat picker while preserving the active-session exception, and keep admin/debug session surfaces able to show internal rows. If a smaller interim UI filter is accepted, it should use parsed `isSubagentSessionKey`/`isAcpSessionKey`-style helpers and cover the `spawnDepth` fallback in tests.
