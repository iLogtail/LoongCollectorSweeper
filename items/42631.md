## Summary

Keep #42631 open. Current main still only supports cron `payload.model` for `agentTurn` jobs, not `sessionTarget="main"` + `payload.kind="systemEvent"`. The provided related PR #42708 attempted this exact feature but is closed and unmerged, and the current source/schema/docs still reflect the old boundary.

## Best Possible Solution

Implement an explicit supported override contract for main-session system-event cron jobs, likely by adding a validated `payload.model` or top-level `model`/`provider`/`sessionOverride` field to the cron type, protocol schema, normalization, CLI/tool/docs surfaces, and `executeMainSessionCronJob`/gateway session handling so the override is applied for that cron run without requiring manual session-store edits. Add regression coverage for accepted allowed overrides, invalid/blocked fallback behavior, and persistence/non-persistence semantics for the main session.
