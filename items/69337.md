## Summary

Keep #69337 open, but it should not merge as written. Current main still does not implement the requested commentary-phase fallback, and the PR identifies a plausible TUI streaming/no-final-answer visibility gap. However, the proposed shared-helper fallback conflicts with current tests and with deliberate non-TUI commentary suppression in previews/history/live chat, so the best path is a TUI-specific fix or caller option with regression coverage.

## Best Possible Solution

Do not merge the current shared-helper fallback. If maintainers want TUI to show commentary-phase text while a run is streaming or when a model never produces final_answer text, implement that as a TUI-specific rendering option or a narrowly scoped caller flag, then add tests proving commentary-only TUI deltas/finals render while session previews, chat history, live WebChat, and shared helper defaults continue hiding commentary-only assistant text.
