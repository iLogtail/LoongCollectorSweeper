## Summary

Keep #53032 open. Current main has useful partial session management in Control UI: a chat header session dropdown, a separate Sessions table with label editing and links back to chat, a plus button that sends `/new`, and backend session reset/list/patch APIs. It still does not implement the requested native conversation-management sidebar/workspace with in-chat session history navigation and preserved archived conversations. The related PR #61721 is open/unmerged in the provided context and its referenced commit is not present in this checkout, so it does not supersede the issue yet.

## Best Possible Solution

Keep the issue open until a maintainer either lands a native Control UI conversation sidebar or explicitly consolidates it into the broader WebChat conversation-management tracker. The likely implementation path is to build on existing `sessions.list`, `sessions.patch`, chat history loading, and `/new` wiring, then add sidebar UX plus backend preservation/archival semantics if the product wants old sessions to remain switchable after `/new`.
