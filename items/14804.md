## Summary

Keep #14804 open. Current main still has sandbox scopes, workspace access controls, Docker tmpfs mounts for temporary container paths, manual recreate, and idle/age pruning, but it does not expose a first-class ephemeral or tmpfs-backed workspace mode for isolated sessions. The prior review comment remains accurate against current main 9a529ca78bfa0e257f662f0e70ab8c681caa6086.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds a documented disposable workspace mode. A good fix should add an explicit setting to AgentSandboxConfig and AgentSandboxSchema, create a fresh per-run workspace where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, and update sandbox lifecycle tests plus public docs for sandboxing and agent sandbox config.
