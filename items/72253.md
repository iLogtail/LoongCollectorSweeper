## Summary

Keep PR #72253 open. Current main still has the reported heartbeat behavior: benign structured exec completions are recognized as exec events, but there is no internal-only classification for `code 0` or `signal SIGTERM`, and exec completions still bypass normal HEARTBEAT_OK skipping. The PR is recent, focused, linked to concrete issue #72252, and the provided review discussion says the latest hardening pass addressed bot feedback rather than making the PR obsolete.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is to refine or land a narrow heartbeat fix that keeps trusted/local structured `code 0` and restart-cleanup `signal SIGTERM` exec completions internal, preserves user relay for real failures such as `code 1`, and either documents the interim text-shape classifier or aligns it with the broader #69492 event-audience design later.
