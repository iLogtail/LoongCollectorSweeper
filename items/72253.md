## Summary

Keep PR #72253 open. Current main still treats structured `Exec completed (..., code 0)` and `Exec failed (..., signal SIGTERM)` as generic relayable exec completion events, and the PR is a recent focused fix for the linked #72252 restart-cleanup chat-noise bug rather than obsolete, implemented, or fully superseded by the broader #69492 audience-classification discussion.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is a narrow core heartbeat fix that trust-gates local structured exec completions, keeps benign `code 0` and cleanup `signal SIGTERM` completions internal, preserves user relay for actionable failures such as `code 1`, and coordinates with #69492 without waiting for the broader audience-classification redesign to solve #72252.
