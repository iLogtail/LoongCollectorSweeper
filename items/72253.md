## Summary

Keep PR #72253 open. Current main still does not implement the requested benign exec-completion suppression: structured `code 0` and `signal SIGTERM` exec events are recognized only as exec completions, then treated as relayable exec events and kept out of the normal HEARTBEAT_OK skip path. The PR remains a focused fix for open issue #72252, while broader related items do not fully supersede its SIGTERM cleanup behavior.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is a narrow heartbeat fix that keeps trusted/local structured `code 0` and benign restart-cleanup `signal SIGTERM` exec completions internal, preserves user relay for real failures such as `code 1`, adds regression coverage at the heartbeat prompt/runner seams, and leaves the broader event-audience redesign to #69492.
