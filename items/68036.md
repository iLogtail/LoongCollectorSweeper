## Summary

Keep open. Current `main` still clones cached session stores and still routes `readSessionUpdatedAt` through the full mutate-safe load path, so the PR's core optimization is not implemented on `main`. One of the PR's target call sites was independently refactored away from `loadSessionStore`, which makes the patch partially stale and likely in need of a rebase, but that is not a strong close reason.
