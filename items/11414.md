## Summary

Keep #11414 open. Current main implements Telegram forum topic creation and editing, but still lacks the requested close, reopen, and delete lifecycle actions. The related PR #41410 is open and deletion-only in the supplied context, so it does not supersede this dedupe-parent issue.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add documented message actions for close, reopen, and delete topic handling, wire them through the shared action names, Telegram channel action map/discovery, action runtime, Telegram Bot API send/runtime helpers, config/schema metadata, docs, and Telegram contract/runtime tests. PR #41410 may be useful for deletion if it is updated and merged, but close and reopen remain unimplemented on current main.
