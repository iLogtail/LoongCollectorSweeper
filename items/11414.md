## Summary

Current main still implements only Telegram forum topic creation/editing. The requested close, reopen, and delete topic lifecycle actions are absent from the shared action list, Telegram action adapter/runtime, send helper exports, config schema, docs, and contract/runtime tests. The related PR #41410 is open and deletion-only, so it does not supersede this dedupe-parent issue.

## Best Possible Solution

Keep #11414 open as the canonical dedupe-parent tracker until bundled Telegram topic lifecycle support is complete. A complete fix should add generic topic close/reopen/delete action names or well-documented aliases, map and advertise them in `extensions/telegram/src/channel-actions.ts`, implement Telegram runtime/send helpers for `closeForumTopic`, `reopenForumTopic`, and `deleteForumTopic`, extend config/schema/docs coverage, and add Telegram contract/runtime/send tests. PR #41410 can help with deletion, but close and reopen remain missing.
