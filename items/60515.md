## Summary

Keep open. Current main still installs two `uncaughtException` handlers that always call `process.exit(1)`, while the `ENETUNREACH` transient-network suppression is only wired into the `unhandledRejection` path. The linked fixes are not shipped: #60627 is closed unmerged and #63780 remains open as proposed work for this issue.

## Best Possible Solution

Keep #60515 open until the uncaught-exception path is fixed. The best implementation is a shared uncaught-exception handling helper wired into both `src/index.ts` and `src/cli/run-main.ts`, preserving fatal/config error precedence before any transient suppression, suppressing the reported `ENETUNREACH`/AbortError class when appropriate, and adding regression tests for both transient uncaught exceptions and fatal-wrapper precedence. PR #63780 can be revised or replaced to land that behavior.
