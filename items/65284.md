## Summary

Keep #65284 open. Current main still has the root stale-provider merge behavior and several bare-model context-window lookup paths, while some adjacent provider-aware persistence paths have already moved in the right direction and the PR needs a rebase/narrowing rather than auto-closure.

## Best Possible Solution

Keep the PR open, but rebase and narrow it. The best path is an ownership-aware fix for `mergeWithExistingProviderSecrets()` or a documented merge-mode behavior change, plus provider-qualified context resolution in `getSessionDefaults`, `commands/sessions`, and `cron/isolated-agent/run`. Persisted session context should remain a fallback behind configured/provider-specific context, and focused regressions should cover orphaned provider removal plus provider-qualified defaults/status/sessions/cron accounting.
