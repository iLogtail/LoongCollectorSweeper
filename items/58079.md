## Summary

Keep open. Current `main` does not implement the requested `before_tool_call` context expansion: the public hook event still only exposes `toolName`, `params`, `runId`, and `toolCallId`, and the runtime still dispatches only those fields. The docs also still treat conversation access as limited to `llm_input`/`llm_output`/`agent_end`, so this PR is neither implemented-on-main nor obsolete. The PR discussion links the change to active guardrail/plugin use cases, which argues against conservative auto-close.
