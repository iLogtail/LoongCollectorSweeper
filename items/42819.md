## Summary

Keep #42819 open. Current main still installs optional Docker Chromium under `/home/node/.cache/ms-playwright`, docs still tell users to persist that hidden cache path, and there is no stable `chrome`/`chromium` PATH symlink in the Dockerfile. The linked fix PR #42771 is still open/unmerged, while related PR #42844 was closed unmerged, so the reported Docker packaging issue is not implemented on main.

## Best Possible Solution

Keep the issue open and land or rework #42771, or an equivalent maintainer-reviewed Docker packaging change, to move Playwright browsers out of the hidden cache path, expose stable `chrome`/`chromium` PATH symlinks after install, update Docker/browser docs, and add Dockerfile regression coverage for the chosen path and symlinks.
