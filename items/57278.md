## Summary

Keep open. Current `main` still downloads `reply_to_message` media without checking whether the replied-to Telegram message was bot-authored, and that reply media is still appended into the agent-visible inbound attachment context.

## Evidence

- **Top-level self-message guard exists only for the inbound message itself:** The `message` handler skips processing when `normalizedMsg.from.id === ctx.me.id`, but that guard is only applied to the outer inbound message, not to `msg.reply_to_message`.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1956](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/telegram/src/bot-handlers.runtime.ts#L1956)
  - command: `nl -ba extensions/telegram/src/bot-handlers.runtime.ts | sed -n '1938,1970p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Reply-media resolver still fetches replied media with no bot-authored check:** `resolveReplyMediaForMessage` reads `msg.reply_to_message`, checks only `hasInboundMedia`, resolves a file id, and calls `bot.api.getFile(replyFileId)` / `resolveMedia(...)`. There is no self-authored/bot-authored exclusion before download.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:489](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/telegram/src/bot-handlers.runtime.ts#L489)
  - command: `git show c65aa1d2a6e573d49619c4dd0698a76010476e7d:extensions/telegram/src/bot-handlers.runtime.ts | nl -ba | sed -n '489,525p'`
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Resolved reply media is injected into agent-visible media context:** The inbound context builder concatenates `replyMedia` onto the normal inbound `allMedia` list via `const contextMedia = [...currentMediaForContext, ...replyMedia]`, so any downloaded replied-to media is exposed to the agent as attachment context.
  - file: [extensions/telegram/src/bot-message-context.session.ts:321](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/extensions/telegram/src/bot-message-context.session.ts#L321)
