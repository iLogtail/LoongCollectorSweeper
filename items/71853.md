## Summary

Keep open. Current main still has source-backed google-vertex ADC risk: ADC resolves to the non-secret `gcp-vertex-credentials` marker, the runner can store unresolved credentials as runtime API keys when no provider runtime-auth hook returns a replacement, and the Google auth helper serializes any raw string as `x-goog-api-key`. The Node 25/gaxios part also is not visibly fixed in repo: the lockfile still resolves `@google/genai` to a gaxios dependency with `node-fetch`, and there is no patch/override for that path. Related ClawSweeper reports already kept the same ADC issue open, and this issue adds current user confirmation on 2026.4.23.

## Best Possible Solution

Keep #71853 open. The best fix belongs in the bundled Google provider/runtime-auth path: handle google-vertex ADC as provider-owned runtime auth instead of passing `gcp-vertex-credentials` as an API key, set the correct Vertex project/location/endpoint or SDK options for google-vertex, and either update/patch/avoid the @google/genai gaxios Node 25 node-fetch path. Add focused regression tests for ADC marker stripping, emitted auth headers, google-vertex project/location behavior, and the global-fetch/Node 25 compatibility path.
