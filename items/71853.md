## Summary

Keep open. Current main still resolves google-vertex ADC to the non-secret marker `gcp-vertex-credentials`, treats that env result as `api-key` auth, stores/forwards it into runtime/provider stream options, and the bundled Google transport serializes any non-OAuth JSON string as `x-goog-api-key`. The issue also has a same-problem confirmation on 2026.4.23, and related ClawSweeper reviews/PR context already point to the same remaining ADC marker leak.

## Best Possible Solution

Keep the issue open and fix google-vertex as a provider-owned ADC/Vertex auth path: stop representing ADC as an API key that can reach `setRuntimeApiKey` or `options.apiKey`, add a Google provider runtime auth/transport path that omits `x-goog-api-key` and uses Vertex project/location/ADC correctly, and add regression tests proving the marker never becomes a Google API-key header. Track the Node 25/gaxios crash either in the same fix if still reproducible with the locked dependency set, or split it into a narrower dependency compatibility follow-up.
