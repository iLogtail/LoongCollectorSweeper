## Summary

Keep PR #64901 open. Current main has related Bedrock context-window improvements, but it does not implement this PR's requested default-source warning, and the linked follow-up PR #64935 is closed unmerged rather than a canonical merged replacement. The PR may need revision because the review correctly notes Bedrock discovery fallbacks are still represented as model metadata, but that argues for maintainer review or an adapted fix, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer judgment or revision. The best product path is to either adapt the warning so it can distinguish true provider-discovery fallback values from real model metadata, or close it only after a merged canonical fix explicitly covers the default/fallback warning behavior with source and test evidence.
