## Summary

Keep #37813 open. Current main has partial fixes for the issue: session model overrides now feed configured fallback chains, xAI/Grok model IDs from the report are forward-compatible, and fallback status is persisted/rendered. The remaining security/cost-relevant request is still not implemented: fallback tool policy is resolved from the active fallback provider/model, not from the originally requested model whose `tools.byProvider` deny list the reporter expects to preserve.

## Best Possible Solution

Keep #37813 open until maintainers either implement requested-model tool-policy preservation across fallback attempts, or explicitly decide and document that fallback attempts should use only the active fallback modelâ€™s policy. A concrete fix should carry the selected/requested provider/model into the fallback attempt tool-policy path and add regression coverage proving `tools.byProvider[requestedProvider/requestedModel].deny` remains enforced when the run falls back to another provider/model.
