## Summary

Keep open. Current main has TaskFlow groundwork, but it does not implement the PR's central managed child-task flow UX: `sessions_spawn(taskFlow=...)`, managed child completion auto-closing parent flows, retryable managed child-task relaunch, retry/guidance fields, CLI retry, or web/Gateway flow controls. The PR discussion also shows useful author follow-up and review findings rather than an obsolete or duplicate item.

## Best Possible Solution

Keep the PR open, but require maintainer/author follow-up before merge. The best path is to split or tighten the core slice around `sessions_spawn(taskFlow=...)`, managed child completion sync, stored-launch retry eligibility/relaunch, CLI/runtime/docs coverage, and then separately decide the web/Gateway control surface. The unresolved P1/P2 review comments should be addressed with focused tests before landing.
