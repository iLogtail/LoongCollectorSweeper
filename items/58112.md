## Summary

Keep open. Current `main` still reads `cwd` for `sessions_spawn` but does not pass it into the subagent runtime, so the reported bug is not implemented on `main`. The PR also has an unresolved regression risk around attachment materialization for cross-agent spawns, so conservative cleanup should not close it.

## Evidence

- **Current main still drops `cwd` on subagent spawns:** `createSessionsSpawnTool` reads `cwd` from tool params, forwards it for ACP spawns, but the subagent path still passes only `workspaceDir: opts?.workspaceDir` in the spawn context. That means `runtime="subagent"` still ignores the user-supplied `cwd` on current `main`.
