## Summary

Keep PR #66542 open. Current main still starts bundled MCP servers sequentially with one connection attempt, drops failed startups, and `callTool` has no reconnect/retry/dead-server state. The PR targets real missing runtime behavior, but the review discussion also identifies correctness and security issues that should be resolved before merge.

## Best Possible Solution

Keep the PR open, but require a narrowed update before merge: implement bounded startup concurrency, explicit retry/jitter config, call-time reconnect with in-flight joining, safe dead-server resurrection, strict transport-error classification, sanitized catalog/error metadata, and focused tests in `src/agents/pi-bundle-mcp-runtime.test.ts` for startup retry, concurrent reconnect callers, JSON-RPC error handling, and exhausted/dead recovery.
