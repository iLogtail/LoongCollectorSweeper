## Summary

Keep open. Current main has not absorbed this PR or an equivalent Feishu target validation fix: `resolveReceiveIdType` still always returns a truthy `chat_id | open_id | user_id`, `resolveFeishuSendTarget` still forwards that value without a null/invalid-target guard, and the linked regression issue #53376 remains the active report in the provided context. The PR may still need maintainer review and regression coverage before merge, but it is not obsolete or safely closeable.

## Best Possible Solution

Keep #53787 open as the active Feishu regression fix candidate. The best path is a Feishu-owned patch that preserves existing valid bare direct IDs, rejects truly malformed targets before calling the send/streaming APIs, and adds regression coverage around `resolveReceiveIdType`, `resolveFeishuSendTarget`, and the linked #53376 failure mode.
