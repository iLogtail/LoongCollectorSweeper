## Summary

Keep #38235 open. Current main still has the reported Slack streaming identity gap: normal Slack delivery resolves and forwards `slackIdentity`, but the draft preview stream cannot receive it and the native streaming path starts `client.chatStream` without identity fields. The focused fix PR #38237 is still open in the provided context, while #46432 was closed unmerged, so this is not implemented or clearly superseded on main.

## Best Possible Solution

Keep this issue open. The best maintainer path is to land or adapt the focused draft-preview fix from #38237 by adding an optional Slack send identity to `createSlackDraftStream`, passing `slackIdentity` from dispatch, and adding a regression test for the initial draft send. Then verify the current Slack native streaming API against the pinned Node SDK: if `chat.startStream` identity can be passed safely, thread it through `startSlackStream`; otherwise document the native limitation and prefer draft preview when per-agent Slack authorship is required.
