## Summary

Keep open. Current `main` still resolves Slack custom agent identity for normal delivery, but does not thread that identity through the draft preview stream, and native streaming still starts with `chatStream()` parameters that contain no identity override fields.

## Evidence

- **Normal Slack replies still support custom identity:** `dispatchPreparedSlackMessage` resolves `slackIdentity` from the routed agent, and the normal delivery path passes it into `deliverReplies`, so non-streaming replies still use the customize flow.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:223](https://github.com/openclaw/openclaw/blob/eb6e1245ac881566a7f31adc7d94d9e3c721a6ac/extensions/slack/src/monitor/message-handler/dispatch.ts#L223)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/dispatch.ts | sed -n '216,280p' && nl -ba extensions/slack/src/monitor/message-handler/dispatch.ts | sed -n '488,780p'`
  - sha: [eb6e1245ac88](https://github.com/openclaw/openclaw/commit/eb6e1245ac881566a7f31adc7d94d9e3c721a6ac)
- **Draft stream API still has no identity parameter:** `createSlackDraftStream` accepts `target`, `cfg`, `token`, `accountId`, etc., but no `identity` field. Its initial `send()` call forwards only `cfg`, `token`, `accountId`, and `threadTs` to `sendMessageSlack`.
  - file: [extensions/slack/src/draft-stream.ts:22](https://github.com/openclaw/openclaw/blob/eb6e1245ac881566a7f31adc7d94d9e3c721a6ac/extensions/slack/src/draft-stream.ts#L22)

- **Dispatch still constructs draft stream without identity:** The preview-stream initialization calls `createSlackDraftStream(...)` with `target`, `cfg`, `token`, `accountId`, threading callbacks, and logging only; it never passes `slackIdentity`.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:751](https://github.com/openclaw/openclaw/blob/eb6e1245ac881566a7f31adc7d94d9e3c721a6ac/extensions/slack/src/monitor/message-handler/dispatch.ts#L751)
