## Summary

Current main implements the useful part of PR #68682 with a refined policy-based behavior. The bundled Anthropic CLI backend no longer hardcodes `--permission-mode bypassPermissions` in its default args, the normalizer only adds bypass when OpenClaw's effective exec policy is YOLO, explicit Claude permission-mode overrides are preserved, malformed permission flags are dropped, and tests/docs cover the contract. This also addresses the earlier Codex review concern that removing the noninteractive permission mode outright could make default Claude CLI runs block on prompts.

## Best Possible Solution

Close PR #68682 and keep the shipped main implementation: raw Anthropic CLI defaults no longer force Claude bypass, but the normalizer deliberately maps Claude permission mode to OpenClaw's existing exec policy and leaves explicit raw overrides available for users who want Claude `default`, `acceptEdits`, or `bypassPermissions`.
