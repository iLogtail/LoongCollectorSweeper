## Summary

Keep open. Current main still lacks the requested core security/API contract: skills cannot declare sandboxed callable functions in metadata, heartbeat remains a prompt/model turn, and the plugin/runtime surfaces do not provide a general core-enforced path for remote structured heartbeat invocations that bypass LLM instruction interpretation.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete path likely needs a versioned skill metadata schema for declared sandboxed functions, frontmatter/package parsing and docs, a heartbeat remote-invocation fetch/parse layer, runtime validation against the declared schema, execution through existing tool/plugin capability boundaries without sending remote heartbeat content to the model as free-form instructions, and regression tests for allowed calls, unknown calls, schema failures, and prompt-injection non-delivery. ClawHub trust badges can follow once the core contract exists.
