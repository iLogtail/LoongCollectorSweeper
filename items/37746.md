## Summary

Keep open. Current main still lacks the requested core security/API contract: skills cannot declare heartbeat-callable sandboxed functions with parameter schemas, heartbeat config remains a strict prompt-oriented object with no remote structured invocation mode, and heartbeat execution still sends a prompt through the model rather than validating remote calls outside the LLM instruction channel. The earlier ClawSweeper keep-open comment remains accurate after the newer skills/logging commits on main.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete solution should add a versioned skill/plugin metadata contract for declared sandboxed functions, parse and document it, add a structured remote heartbeat fetch/parse path, validate call names and params against declared schemas, execute only through existing tool/plugin capability boundaries, and add regression tests proving unknown calls, schema failures, and prompt-injection text are not delivered to the model as free-form heartbeat instructions. ClawHub trust indicators can follow once the core contract exists.
