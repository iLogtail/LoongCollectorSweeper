## Summary

Keep open. Current `main` still resolves exec-approvals storage to hardcoded `~/.openclaw/*` paths instead of honoring `OPENCLAW_STATE_DIR`, while the rest of state storage and doctor logic use the configured state dir. The reported split-state warning is therefore still plausible on `main`, and no shipped fix is present at `cf858258c71701ef62dddf88aa777873a4962046`.

## Evidence

- **Exec approvals path ignores state-dir override:** `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` return `expandHomePrefix(DEFAULT_FILE|DEFAULT_SOCKET)` directly, with `DEFAULT_*` set to `~/.openclaw/...`; there is no `OPENCLAW_STATE_DIR` branch.
  - file: [src/infra/exec-approvals.ts:173](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/infra/exec-approvals.ts#L173)

  - file: [src/config/paths.ts:60](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/config/paths.ts#L60)
