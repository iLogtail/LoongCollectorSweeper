## Summary

Keep open. Current `main` does not implement Volta shim rewriting at all, so this PR is not obsolete, and the PR as proposed still has an unresolved correctness gap for supported custom `VOLTA_HOME` layouts called out in review.

## Evidence

- **Current main only rewrites Homebrew Cellar node paths:** `resolveStableNodePath` on `main` matches only `.../Cellar/.../bin/node` and returns early for everything else; there is no Volta handling in the shipped implementation.
  - file: [src/infra/stable-node-path.ts:11](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/infra/stable-node-path.ts#L11)
  - command: `nl -ba src/infra/stable-node-path.ts | sed -n '1,120p'`
  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Current tests prove no Volta coverage on main:** `runtime-paths.test.ts` exercises Homebrew resolution and non-Cellar passthrough only; there are no Volta path assertions in the current test suite.
  - file: [src/daemon/runtime-paths.test.ts:153](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/daemon/runtime-paths.test.ts#L153)
  - command: `nl -ba src/daemon/runtime-paths.test.ts | sed -n '150,220p'`
  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **OpenClaw explicitly supports custom VOLTA_HOME bin roots:** Service PATH construction adds `VOLTA_HOME/bin` when present, and the test suite asserts `/opt/volta/bin` is included. That means non-default Volta install roots are part of the current contract.
  - file: [src/daemon/service-env.ts:98](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/daemon/service-env.ts#L98)

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Custom VOLTA_HOME gap already identified in PR review:** The PR discussion includes repeated review comments that the new regex only matches `/.volta/tools/image/...`, so it misses supported custom `VOLTA_HOME` layouts (and a reviewer also called out Windows Volta layout coverage). No follow-up patch addressing that is present in the provided PR context.
  - sha: [36e3ec16e1e5](https://github.com/openclaw/openclaw/commit/36e3ec16e1e55cd5d62806e17c1ac7710c54978c)

## Risks / Open Questions

- Closing this PR would discard a still-relevant fix path while `main` still lacks any Volta image-path stabilization.
- Merging the PR unchanged would leave supported custom `VOLTA_HOME` setups with stale version-pinned daemon paths after Volta upgrades.
