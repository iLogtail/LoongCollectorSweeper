## Summary

Keep open. Current `main` still exposes only concurrency/active-child controls for subagents, not a lifetime-per-session spawn cap, and subagent completions are still handed back into the requester session as completion events/internal context. The requested `maxPerSession` behavior is not implemented on `main` and the report remains plausible/in-scope.

## Evidence

- **Subagent config schema has no lifetime-per-session cap:** `agents.defaults.subagents` currently defines `allowAgents`, `maxConcurrent`, `maxSpawnDepth`, `maxChildrenPerAgent`, `archiveAfterMinutes`, `model`, `thinking`, `runTimeoutSeconds`, `announceTimeoutMs`, and `requireAgentId`. There is no `maxPerSession`/`maxTotal` field in the runtime schema.
  - file: [src/config/zod-schema.agent-defaults.ts:255](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/zod-schema.agent-defaults.ts#L255)
