Keep open. Current main still has the unsafe gateway and tailnet `readNetworkInterfaces()` paths this PR targets, and existing tests on main still assert the throwing behavior. The linked bug #71721 remains open, and the provided PR discussion includes a maintainer comment saying the fix direction is right and landable after the follow-up preserving diagnostics.

## Best Possible Solution

The best product fix is to make gateway LAN/tailnet interface discovery use the existing safe/inspection network-interface path, preserve diagnostic warnings for status/display surfaces, and update the gateway, tailnet, and status tests so sandbox `uv_interface_addresses` failures degrade to loopback/empty network details instead of crashing.
