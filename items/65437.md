## Summary

Keep open. Current main still has a plausible Slack-specific command gate path that can drop `/elevated` in group/channel messages before `resolveElevatedPermissions()` checks `tools.elevated.allowFrom.slack`, and I did not find a Slack fallback implementation or regression test proving the reported configuration works.

## Best Possible Solution

Keep the issue open. Add a focused Slack regression test for a group/channel message like `@bot /elevated on` where only `tools.elevated.allowFrom.slack` contains the sender, then either let elevated-only directives reach `resolveElevatedPermissions()` before the generic Slack command block or explicitly document and test the intended prerequisite if maintainers choose to require `channels.slack.allowFrom` / `channels.slack.channels.<id>.users`. The code path should avoid granting other control commands from the elevated allowlist alone.

## Evidence

- **Slack command gate excludes elevated allowlist:** For Slack channel/group messages, `commandAuthorized` is computed only from `channels.slack.allowFrom` (`allowFromLower`) and per-channel `channels.slack.channels.<id>.users`; `tools.elevated.allowFrom.slack` is not one of the authorizers. If the resulting gate says to block, the handler logs `control command (unauthorized)` and returns `null` before reply directive handling.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:409](https://github.com/openclaw/openclaw/blob/e23d17da79dc2345ca55902e632d3c7622accacc/extensions/slack/src/monitor/message-handler/prepare.ts#L409)

- **Generic control gate blocks recognized commands before directive handling:** `resolveControlCommandGate()` sets `shouldBlock` when text commands are enabled, the message has a control command, and `commandAuthorized` is false. With default access-group enforcement and no configured Slack channel/user authorizer, that matches the issue's reported setup.
  - file: [src/channels/command-gating.ts:31](https://github.com/openclaw/openclaw/blob/e23d17da79dc2345ca55902e632d3c7622accacc/src/channels/command-gating.ts#L31)
