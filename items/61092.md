## Summary

Keep PR #61092 open. Current main still has the requested Skills screen labels hardcoded in English, and the English locale source has no `skillsPage` section to back `t("skillsPage.*")` calls. The PR discussion addressed a real review gap by adding locale stubs, but current UI i18n rules require generated non-English locale bundles, so this needs rebase/rework rather than closure.

## Best Possible Solution

Keep the PR open and ask for a current-main rework through the Control UI i18n pipeline: add the Skills page keys in `ui/src/i18n/locales/en.ts`, wire `ui/src/ui/views/skills.ts` through `t(...)` for the top-level labels, regenerate all locale bundles and `.i18n` metadata with `pnpm ui:i18n:sync`, and keep the structural i18n test passing. Do not close as ClawHub; this is core Control UI localization, not an external plugin capability.
