## Summary

Current `main` does not implement configurable channel pairing message text. The pairing reply is still hardcoded in core, channel callers still use that default path, and the config surface on `main` has no `channels.*.pairingMessage` field or equivalent override.

## Evidence

- **Hardcoded pairing reply text in core:** `buildPairingReply` still emits fixed strings including `OpenClaw: access not configured.`, `Pairing code:`, and the `openclaw pairing approve ...` CLI hint with no config input.
  - file: [src/pairing/pairing-messages.ts:4](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/pairing/pairing-messages.ts#L4)
