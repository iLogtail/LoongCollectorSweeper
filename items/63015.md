## Summary

Keep this PR open. Current main still normalizes loose outbound reply payloads from conversational delivery with only `mediaUrl`, so `filePath`, `path`, or `media` aliases would still be dropped on that path. The existing message action path does accept media aliases, so the PR addresses a real parity gap, though the review comment about fallback precedence should be resolved before merge.

## Best Possible Solution

Keep the PR open and have the author or maintainer finish the review cleanup: decide the intended precedence order, update `normalizeOutboundReplyPayload` accordingly, add/keep focused regression tests for `media`, `mediaUrl`, `filePath`, and `path`, then run the relevant plugin SDK or changed-lane checks before landing.
