## Summary

Keep open. Current `main` still only normalizes `payload.mediaUrl` in `normalizeOutboundReplyPayload`, while the inbound auto-reply path routes loose reply payloads through that helper and other outbound runtimes already accept alias fields like `media`, `path`, and `filePath`. The PR remains coherent and useful; the discussion only surfaced minor review nits about alias precedence/test placement, not a reason to close it.

## Evidence
