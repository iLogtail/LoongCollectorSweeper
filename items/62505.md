## Summary

Keep open. Current main has GPT-5/OpenAI Codex strict-agentic recovery work, but the latest narrowed repro in the issue remains plausible: background exec completions are still delivered through exec-event heartbeat wakes, while `heartbeat.every = "0m"` still disables the heartbeat runner before exec-event preflight can run. The related PR #67913 was closed/cancelled, not merged, and current tests cover exec-event routing only with enabled heartbeat intervals.

## Best Possible Solution

Keep #62505 open and fix the narrowed background-completion path in core: allow exec-event wakes to bypass per-agent heartbeat interval disabling while preserving the global heartbeat enable switch, create a one-shot targeted runner entry for session-scoped exec wakes when no interval-backed heartbeat agent is scheduled, preserve the captured delivery context for relay, and add regression tests for `heartbeat.every = "0m"` plus exec completion delivery.
