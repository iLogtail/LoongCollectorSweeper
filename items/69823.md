## Summary

Keep open. Current main does not contain a native Windows companion app: the repo has macOS, iOS, Android, shared app code, and Windows CLI/installer/service support, but no `apps/windows`, Tauri/WebView2 project, tray app, or Rust watchdog implementation. The project vision explicitly lists Windows companion apps as a next priority, and the Windows docs say the companion app is still planned/not present, so this is a real platform product request rather than a ClawHub-only extension or an implemented feature.

## Best Possible Solution

Keep the issue open as the Windows companion app tracking/RFC item unless maintainers create or identify a canonical Windows desktop-app tracker. The concrete path would be a maintainer-reviewed `apps/windows` implementation with documented Tauri/Rust or alternative architecture, gateway lifecycle/security decisions, installer/update story, and Windows smoke coverage, while retaining the existing CLI installer and Scheduled Task fallback until a native app ships.
