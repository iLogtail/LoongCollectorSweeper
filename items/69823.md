## Summary

Keep open. Current main does not implement the requested native Windows companion app: the repo has Windows CLI, PowerShell installer, and Scheduled Task/startup fallback support, but docs explicitly say there is no Windows companion app yet and no `apps/windows` or Tauri/Rust scaffold is present.

## Best Possible Solution

Keep the issue open as a real Windows platform gap. The best path is an owner-reviewed design for a native Windows companion app, likely under `apps/windows`, reusing current CLI/Scheduled Task installer support while defining tray UX, gateway lifecycle management, WebView2/native bridge scope, installer/update behavior, docs, labels, and release packaging.
