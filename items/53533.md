## Summary

Keep open. Current main still does not implement Feishu websocket/webhook lifecycle health surfacing, and PR #53533 remains a plausible focused implementation attempt rather than cleanup-close material. It is not merge-ready as-is because the normal gateway startup path still is not wired to pass `ctx.setStatus` into the Feishu monitor stack, and current health policy expects `lastTransportActivityAt` for stale-socket checks.

## Best Possible Solution

Keep #53533 open and ask for a focused revision: pass `ctx.setStatus` into the Feishu monitor stack, publish connected/disconnected/reconnect/error patches for both websocket and webhook modes, report provider-proven transport activity through `lastTransportActivityAt`, and add Feishu gateway/transport tests that fail on current main and prove the runtime snapshot receives those fields.
