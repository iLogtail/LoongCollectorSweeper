## Summary

Keep open. Current main has useful browser profile options, including remote CDP and a built-in Chrome DevTools MCP existing-session path, but it does not provide the requested third-party browser backend/plugin-driver registration interface. The browser profile driver contract is still closed over built-in modes, and the plugin API does not expose a browser-driver registration surface. This is not a ClawHub close because the missing piece is a core/plugin SDK seam, which VISION.md explicitly treats as valid design/API work when a useful capability cannot yet be built as a plugin.

## Evidence
