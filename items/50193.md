## Summary

Keep open. Current `main` still renders the chat-model picker as a flat `<select>`, so the requested grouped-by-provider UI is not implemented. The PR is also stale against current `main`: it edits an older `app-render.helpers.ts` path, while the live picker logic now lives in `ui/src/ui/chat/session-controls.ts` and `ui/src/ui/chat-model-select-state.ts`, and current tests explicitly preserve provider/id disambiguation that the PR discussion flagged as a regression risk.
