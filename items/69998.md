## Summary

Keep #69998 open. Current main still returns the mutable `skills.update` config object directly after writing it, so the secret-echo bug is not implemented on main. The PR is a focused fix attempt for open bug #66769, while the related PRs #66928 and #66986 are closed unmerged and do not supersede it. Before merge, maintainers should resolve the redaction-sentinel round-trip concern or deliberately choose a response shape that cannot be written back as a credential.

## Best Possible Solution

Keep #69998 open. The best product fix is to stop returning plaintext secret-bearing config from `skills.update` while preserving existing credentials on any redacted-sentinel round trip, either by restoring sentinel values from the existing entry before writing or by changing the success payload to return only non-sensitive confirmation metadata. Keep the regression test for response redaction and close #66769 only after that behavior lands.
