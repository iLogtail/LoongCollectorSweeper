## Summary

Keep open. Current main still returns the updated `skills.entries.<skillKey>` object directly from `skills.update`, so the plaintext `apiKey` and secret-like `env` leak described by #66769 remains present. PR #69998 is a relevant focused fix, but the review discussion also identifies a real follow-up concern around redaction sentinel round-trips that should be handled before landing.

## Best Possible Solution

Keep the PR open and finish it as the canonical fix for #66769: redact or omit secret-bearing response config, preserve full values on disk, restore or reject redaction sentinels on `skills.update` writes, and add regression tests for response redaction, non-secret env passthrough, persisted full values, and sentinel round-trips. The older duplicate PRs #66928 and #66986 can remain closed because they were not merged.
