## Summary

Keep issue #55878 open. The current macOS/OpenClawKit source still contains the exact `WebSocketTaskBox.sendPing()` checked-continuation bridge named by the crash stack, the keepalive loop still calls it periodically, and I did not find a source, test, release-note, or related-item signal that this crash has been fixed or superseded on current main.

## Best Possible Solution

Keep the issue open and fix the native WebSocket ping bridge in `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift`, likely by making the `sendPing` callback continuation single-resume/cancellation-safe and adding a focused Swift test with a fake `WebSocketTasking` that invokes the ping callback more than once. The menu bar symptom should be rechecked after the crash path is fixed, because the current `MenuBarExtra` source already creates a status item and the missing icon is likely a consequence of the process crash.
