## Summary

Keep PR #69857 open. The requested Ollama synthetic-auth locality gate is still missing from the reviewed main SHA, and the PR is not superseded by a merged change. It is a plausible focused fix, but it needs a current-main rebase because main now has a manifest/provider-discovery synthetic-auth path that the provided PR diff does not update.

## Best Possible Solution

Keep the PR open and ask for a rebase against current main. The best fix is to add a shared, tested `isLocalOllamaBaseUrl`-style helper in `extensions/ollama/src/discovery-shared.ts`, use it from both `extensions/ollama/index.ts` and `extensions/ollama/provider-discovery.ts`, and add regression coverage for cloud URLs, public IPs, LAN/loopback/mDNS/bare-hostname cases, and the review-comment IPv6/loopback edge cases. The broader #43945 privacy/fallback work should remain tracked separately.
