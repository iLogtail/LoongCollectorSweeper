## Summary

Keep #8061 open. Current main still has only a global subagent concurrency setting at `agents.defaults.subagents.maxConcurrent`; `agents.list[]` remains strict without a per-agent `maxConcurrent`, and both native plus ACP `sessions_spawn` dispatch through the shared `subagent` lane. The issue discussion includes another user confirming the same need and a recent maintainer/Codex comment that also kept it open.

## Best Possible Solution

Keep #8061 open. Add a validated per-target-agent subagent concurrency setting under `agents.list[]`, update generated schema/help/docs, and enforce it in the shared native plus ACP `sessions_spawn` path alongside the existing global `CommandLane.Subagent` cap. Add focused tests showing a capped target such as `tts-agent` serializes while uncapped target agents can still run concurrently.
