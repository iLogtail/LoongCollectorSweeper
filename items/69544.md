## Summary

Keep #69544 open. Current main still derives mention patterns from an agent's identity emoji, Matrix room handling uses those derived regexes for mention gating, and the current Matrix/shared group-chat config schemas do not expose the requested emoji-mention switch.

## Best Possible Solution

Keep the issue open. Add a backward-compatible emoji mention control, defaulting to the current enabled behavior, then plumb it into mention-pattern derivation for Matrix and per-agent resolution. Update the TypeScript types, zod schemas/generated config metadata, docs, and Matrix/core mention tests so emoji-only room text no longer triggers when disabled while explicit Matrix mentions and configured mentionPatterns still work.

## Risks / Open Questions

- The current workaround, overriding mentionPatterns, replaces derived defaults rather than giving a focused emoji-only toggle.
- A Matrix-only switch likely needs channel/account context in mention-pattern derivation or Matrix-specific filtering; a shared per-agent switch would affect every channel using the generic mention resolver, so the default must preserve compatibility.
