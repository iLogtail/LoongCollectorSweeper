## Summary

Keep #69544 open. Current main still derives fallback mention patterns from both `identity.name` and `identity.emoji`, and Matrix uses those regexes for room mention gating. I found no Matrix/global/per-agent config switch that disables emoji-derived mentions without replacing `mentionPatterns` entirely.

## Best Possible Solution

Add a backward-compatible typed config path that lets users disable emoji-derived fallback mention patterns, ideally with Matrix/account defaulting and optional per-agent override. Update Matrix and shared group-chat schemas/docs, keep explicit `mentionPatterns` semantics intact, and add tests covering a Matrix room where ordinary text containing an agent emoji no longer triggers that agent when the new switch is disabled.
