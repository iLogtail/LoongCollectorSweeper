## Summary

Keep #69544 open. Current main still derives mention regexes from `identity.emoji` with no channel-level or per-agent emoji-mention switch, and the Matrix room handler uses those generic regexes for required-mention gating. The existing workaround remains explicit `mentionPatterns`, but the requested configurable default is not implemented.

## Best Possible Solution

Keep the issue open and implement a backward-compatible Matrix/group mention control, likely defaulting to current emoji-derived behavior for compatibility. The change should update the config schema/types, docs, and tests around `src/auto-reply/reply/mentions.ts` plus the Matrix room path, so users can disable identity emoji matching without replacing every agent's full `mentionPatterns` list.
