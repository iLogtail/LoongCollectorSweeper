## Summary

Keep open. Current main still builds npm global update commands without `--force`, while linked issue #65130 reports the exact container EEXIST failure this PR targets. The PR may need a small follow-up because review discussion correctly notes the omit-optional fallback should also carry `--force`, but the work is not obsolete or superseded. ([github.com](https://github.com/openclaw/openclaw/issues/65130))

## Best Possible Solution

Keep the PR open and ask for a focused update, or land an equivalent maintainer patch, adding `--force` to npm global install updates on both the primary and omit-optional fallback paths with matching regression tests in `src/infra/update-global.test.ts` and `src/infra/update-runner.test.ts`.
