## Summary

Current main still does not implement WhatsApp Newsletter/Channel JID support for the message tool. The bundled WhatsApp plugin accepts E.164/user JIDs and group JIDs, but `@newsletter` still falls through the generic `candidate.includes("@")` rejection path before outbound send routing. The prior discussion remains accurate: PR #13424 was closed unmerged after review noted incomplete routing/session behavior, and the related #62697 issue is adjacent inbound newsletter support rather than a superseding outbound message-tool tracker.

## Best Possible Solution

Keep #13417 open until the bundled WhatsApp plugin supports Newsletter/Channel targets end to end: add a newsletter JID recognizer/normalizer, define outbound allowlist and default-target semantics, model newsletter/channel routing separately from direct/group sessions, route sends through the active listener, add focused normalization/resolver/session/send tests, and update the message and WhatsApp docs. Treat newsletter metadata/follow/react operations as possible follow-up work after basic message-tool send support is correct.
