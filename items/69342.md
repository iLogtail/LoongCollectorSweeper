## Summary

Current main already covers the linked OpenRouter use case through the shipped `agents.defaults.models["provider/model"].params.extra_body` per-model pass-through path for OpenAI-compatible proxy request bodies. The PR's exact `models.providers.*.models[].extraParams` field is not present, but the observable problem is solved by the documented config surface already released in v2026.4.24.

## Best Possible Solution

Keep the shipped, documented per-model `agents.defaults.models["provider/model"].params` contract and use `params.extra_body` / `extraBody` for OpenAI-compatible proxy request-body fields such as OpenRouter `reasoning.max_tokens`. Do not add a parallel `models.providers.*.models[].extraParams` public config field unless maintainers intentionally decide to expand the model-definition contract and document precedence across both surfaces.
