## Summary

Keep open: current `main` still derives cron run failure from structured `payload.isError` only, while current denial strings like `SYSTEM_RUN_DENIED: approval cannot safely bind this interpreter/runtime command` still exist, so the reported false-`ok` path remains plausible and unaddressed.

## Evidence
