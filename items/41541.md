## Close Comment

Closing from a Codex review because current `main` already implements this behavior. The session transcript reader preserves file order (`src/gateway/session-utils.fs.ts:93`), the TUI history loaders keep that order instead of reversing it (`src/tui/tui-session-actions.ts:288`, `src/tui/embedded-backend.ts:242`), and the chat log appends new entries at the bottom while pruning only the oldest entries (`src/tui/components/chat-log.ts:63`). The history-overwrite path is also covered by the current TUI event-handler logic and tests that defer reloads until the active run finishes (`src/tui/tui-event-handlers.ts:240`, `src/tui/tui-event-handlers.test.ts:507`, `src/tui/tui-event-handlers.test.ts:622`). The changelog also records the pending-message/history reload fix in `CHANGELOG.md:2105` under `2026.3.24-beta.1`. If you still see a separate terminal-rendering problem on a newer build, please open a fresh issue with exact repro steps, terminal app, shell, and a screenshot/video.
