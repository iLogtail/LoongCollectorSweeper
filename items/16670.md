## Summary

Keep #16670 open. Current main still has memory-search config types, provider auto-detection, docs, `doctor` diagnostics, and `openclaw memory status --deep`, but neither `openclaw onboard` nor `openclaw configure` includes the requested mandatory or prominent memory/embedding setup step for provider, model, API key/SecretRef, and endpoint verification. The issue discussion also has an April 26, 2026 Codex/maintainer review explicitly keeping the same gap open.

## Best Possible Solution

Keep #16670 open until OpenClaw either adds a dedicated memory/embedding setup path or a maintainer explicitly decides against making it mandatory. The likely implementation belongs in onboarding/configure: use the existing memory embedding provider registry and config schema, prompt for provider/model/API-key or SecretRef settings under `agents.defaults.memorySearch`, verify readiness with a small embedding probe or `openclaw memory status --deep`, and update public docs to clarify API-key requirements, Codex OAuth limitations for embeddings, and daemon/LaunchAgent environment visibility.
