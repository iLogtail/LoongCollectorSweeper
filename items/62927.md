## Summary

Keep open. Current `main` still has the two gaps the PR targets, and the PR as posted also has an unresolved regression called out in review for id-less Slack files.

## Evidence

- **`app_mention` still loses `files` at the debounce merge point:** `app_mention` events are routed into the shared Slack message handler, but the debounced synthetic message is still built from `last.message` plus merged text only. If the last event in the debounce window is `app_mention`, attachments from the earlier `message` event are not carried forward.
  - file: [extensions/slack/src/monitor/events/messages.ts:61](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/extensions/slack/src/monitor/events/messages.ts#L61)
  - command: `nl -ba extensions/slack/src/monitor/events/messages.ts | sed -n '56,82p'`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Current handler does not union files across debounced entries:** `createSlackMessageHandler` on `main` computes `combinedText` and `combinedMentioned`, then constructs `syntheticMessage` from `last.message` without any `files` merge. That means the core behavior the PR describes is not implemented on `main` yet.
