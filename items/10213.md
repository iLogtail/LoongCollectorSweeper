## Summary

Keep #10213 open. Current main still does not implement Anthropic-native server-side compaction config, request injection, compaction block replay, `stop_reason: "compaction"` handling, focused tests, or docs. The related PR #10214 was closed unmerged, #65287 is broader cache/TTL work rather than a clear canonical replacement, and the existing issue discussion already kept this focused tracker open.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic server-side compaction. A good fix should add explicit opt-in config or per-model/provider params, validate and document them, inject the required Anthropic beta header and `context_management.edits` only for direct Anthropic-compatible support, preserve compaction blocks in streamed output and replay history, handle `stop_reason: "compaction"`, add logging/verbose behavior, and add focused Anthropic provider/transport tests. Treat #65287 as related broader cache/TTL design unless maintainers explicitly mark it canonical.
