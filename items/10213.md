## Summary

Keep #10213 open. Current main at ee2ab9a6449219e1c425308a4832ddbb7eeb0ca2 still lacks Anthropic-native server-side compaction config, Anthropic request payload/header injection, compaction block/stream handling, docs, and focused tests. The linked PR #10214 is closed and not present in this checkout, and #65287 overlaps but is broader cache/TTL work rather than a clear replacement for this focused tracker.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic server-side compaction. A good fix should add an explicit opt-in config or per-model/provider param, validate and document it, inject the required Anthropic beta header and `context_management.edits` without clobbering existing headers/payload hooks, preserve provider compaction blocks in replay history, handle compaction deltas and `stop_reason: "compaction"`, add logging/verbose behavior, and add focused Anthropic tests alongside the existing OpenAI Responses server-side compaction coverage. Treat #65287 as related broader cache/TTL design unless maintainers explicitly choose it as canonical.
