## Summary

Keep open. Current `main` still drops Slack API structured error details at the relevant call sites, and this PR remains a substantive bug fix rather than conservative cleanup. The latest PR head also appears to have addressed the earlier `retryAfter` review by matching the upstream Slack SDK shape and adding focused tests.

## Evidence

- **Current formatter is still message-only:** On current `main`, `formatErrorMessage()` formats `err.message` (plus causes) and does not read Slack-specific fields like `code`, `data`, `statusCode`, or `retryAfter`, so those details are lost before logging.
