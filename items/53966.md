## Summary

Keep #53966 open. Current main still drops Slack Web API structured error fields at the Slack logging/probe surfaces this PR targets, and the PR head remains a focused, useful fix rather than obsolete or superseded. The related #53971 appears to duplicate this PR, while #62391 tracks an additional delivery-queue path that this PR does not cover.

## Best Possible Solution

Keep #53966 open. The best path is a rebased Slack-owned formatter that extracts `@slack/web-api` structured fields, preserves shared redaction, updates the current Slack API catch sites, and keeps focused unit coverage. Treat #53971 as the duplicate attempt and handle the broader delivery-queue recovery gap separately under #62391 or a dedicated core diagnostic formatter.
