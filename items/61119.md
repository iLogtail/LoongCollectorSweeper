## Summary

Keep open. Current main still has the connect command tooltip, aria label, and copy-button label hardcoded as English, and no `connectCommand.*` locale keys exist. The PR is a plausible focused fix and the bot review found it well scoped, but it needs refresh against the current generated locale set before it is merge-ready.

## Best Possible Solution

Keep this PR open, but ask for a refresh against current main: add `connectCommand.copyTitle` and `connectCommand.copyAriaLabel` to `ui/src/i18n/locales/en.ts`, run the Control UI i18n sync path so all generated locale bundles stay aligned, wire `connect-command.ts` through `t(...)`, and keep/adjust the translation parity coverage. Consider a separate follow-up for shared copy-button success/error labels if maintainers want the whole copy affordance localized.
