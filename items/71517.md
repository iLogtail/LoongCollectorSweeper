## Summary

Keep PR #71517 open. Current main still has split, narrow filename parsing in media fetch and Feishu paths, with no shared `src/media/filename.ts`, no mojibake recovery helper, and no multi-charset/RFC 5987 language-tag coverage. The latest PR context also shows the initial bot path-sanitization finding was addressed by the follow-up `feishu: sanitize inbound filenames` commit, so this is not an obsolete or low-signal cleanup candidate.

## Best Possible Solution

Keep PR #71517 open for maintainer review and CI. If accepted, land the shared decoder plus Feishu inbound recovery, then leave #48788 as the canonical follow-up for migrating the remaining channel filename paths and any public SDK documentation/API baseline work.
