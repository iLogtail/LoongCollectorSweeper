## Summary

Keep PR #71517 open. Current main still has only ad hoc Content-Disposition filename parsing in media/Feishu paths, lacks the proposed shared decoder, and lacks Feishu inbound mojibake recovery/sanitization. The PR is a focused, recent bug/refactor tied to open issues #48388 and #48788, and the provided PR context shows the initial P1 filename traversal review was addressed by the second commit.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land or rework this implementation in the shared media layer with focused Feishu integration, while leaving #48788 open for full cross-channel migration and ensuring SDK export/baseline checks cover any new plugin-facing helpers.
