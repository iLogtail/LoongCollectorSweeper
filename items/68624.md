## Summary

Keep #68624 open. Current main does not implement the proposed session-run cancel seam or the dependent sessions_send hook, and the PR still contains plausible core/API work for #68550 rather than an already-shipped or clearly superseded change. It should not be auto-closed, but it needs maintainer review and fixes for the SDK-boundary and cancellation-race review comments before merge.

## Best Possible Solution

Keep the PR open as the active implementation attempt for #68550. If maintainers want this seam in core, the best path is to land or rebase on the sessions_send hook work first, then revise #68624 to keep `emitSessionRunCancel` core-internal, make abort state sticky/replayed for late subscribers, guard requester wiring in minimal gateways, and update the SDK entrypoint/API baseline/docs with focused tests.
