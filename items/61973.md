## Summary

Keep #61973 open. Current main does not implement `openclaw update review`: the CLI only wires `update`, `update wizard`, and `update status`, and the public update docs likewise omit a review/risk-assessment subcommand. The PR still targets a real missing CLI feature, though it needs maintainer review and fixes for the unresolved review comments around channel/git/unknown-state behavior before it could land.

## Best Possible Solution

Keep the PR open as a plausible implementation path for a missing `openclaw update review` command. Maintainers should decide whether this narrower command is the desired product shape versus the broader open compatibility-scanner work in #17215, then require the PR to mirror `updateCommand` channel/git/downgrade semantics, handle unknown network states conservatively, add focused Vitest coverage, and document the final public CLI behavior.
