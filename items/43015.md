## Summary

Current main does not fully implement the requested fix. It appears to have narrowed one part of the problem by hiding provider-native components from the generic message tool schema and by allowing zero-valued poll defaults, but the flat message schema still exposes poll creation fields across actions, the send path still rejects present poll params, and Discord modal skeletons with empty fields still error.
