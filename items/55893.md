## Summary

Keep #55893 open. Current main still lacks any `deactivated_workspace` / `deactivated workspace` matcher or regression coverage, and the failover path still depends on this classifier before it can mark the condition as `auth_permanent`. The related #56693 report is broader active work, not a canonical replacement for this narrow PR.

## Best Possible Solution

Keep the PR open and ask maintainers to either land the narrow matcher/tests or re-home the same behavior into the bundled OpenAI Codex provider via the existing provider failover-classification hook. The minimum acceptable fix should add regression coverage for both `classifyFailoverReason("deactivated_workspace")` and `resolveFailoverReasonFromError({ message: "deactivated_workspace" })`.
