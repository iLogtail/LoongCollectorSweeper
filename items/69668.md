## Summary

Keep issue #69668 open. Current main has adjacent multi-agent support and a chat session selector grouped by agent, but it does not implement the requested team/project parent concept, configurable team metadata, or a hierarchical team view in the Control UI session list. The related #45086 context is adjacent WebChat agent/session switching, not a duplicate of this team/project grouping request.

## Best Possible Solution

Keep this open for maintainer product/design judgment. A conservative path is an additive per-agent grouping field such as `agents.list[].project` or `agents.list[].group`, surfaced through `agents.list` and session-list contracts, rendered as collapsible groups in Control UI, with docs and tests. If maintainers want real team isolation, design that explicitly instead of inferring it from agent ID prefixes.
