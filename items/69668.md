## Summary

Keep #69668 open. Current main supports isolated multi-agent routing and an agent-grouped chat session selector, but it does not implement the requested team/project parent entities or a hierarchical Control UI session list. The remaining work is a core UI/protocol/config product feature, not an already-shipped behavior or obvious ClawHub-only plugin item.

## Best Possible Solution

Keep the issue open for a core product/API/UI design. A maintainable implementation would add an explicit team/project grouping contract, likely as strict agent metadata or a documented workspace-derived policy, expose it through agents.list/sessions.list or derived Control UI state, render collapsible hierarchical groups in the chat/session surfaces, and update multi-agent/config docs plus focused UI and protocol tests. It should not be closed as ClawHub-only unless a future plugin/UI extension point can own Control UI session-list layout safely.
