## Summary

Keep #42715 open. Current main has better fallback observability, but it still intentionally falls back on auth failures and override credential failures, including back to `agents.defaults.model.primary`; Cloudflare AI Gateway onboarding still writes a single provider entry rather than gateway-level/shared credentials for additional gateway-backed providers.

## Best Possible Solution

Keep the issue open and implement a targeted fix: either add gateway-level/shared credential support for AI Gateway-backed provider entries, or require/document per-provider `apiKey` for additional gateway providers; separately, make 401/403 on an explicit per-agent model user-visible and avoid falling back to `agents.defaults.model.primary` unless fallback was explicitly configured for that auth-failure case. Add regression coverage for a second Cloudflare AI Gateway provider with missing `apiKey`, plus docs for the supported configuration.
