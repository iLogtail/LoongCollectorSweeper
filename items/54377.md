## Summary

Keep open. Current main still has no `npm:<package>` override for `openclaw plugins install`: the install flow goes from local path handling to explicit `clawhub:` handling, preferred ClawHub lookup, then npm with the raw spec, while npm spec validation rejects protocol-style strings containing `:`. Docs also explicitly say there is no matching `npm:` override. The PR remains relevant core CLI work, but the provided review context includes a substantive remaining concern about parsing `npm:` after local path existence checks.

## Best Possible Solution

Keep the PR open. The best path is a focused core CLI change that recognizes `npm:<spec>` before local path routing, strips the prefix before npm install and hook-pack fallback, preserves Matrix invalid-config recovery for `npm:@openclaw/matrix`, adds regression tests for success, failure/no-ClawHub, local-path precedence, and recovery behavior, then updates the plugin install docs/help.
