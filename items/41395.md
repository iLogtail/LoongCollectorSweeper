## Close Comment

Closing after a Codex review. This issue reads as an operations audit note for one specific Hostinger deployment, not an actionable OpenClaw bug or feature request. Current main already documents the supported update paths: Hostinger 1-Click says updates are handled automatically, Hostinger VPS updates are done from Docker Manager in hPanel, and self-managed installs use `openclaw update` (`docs/install/hostinger.md:19`, `docs/install/hostinger.md:75`, `docs/install/updating.md:11`). The code also treats being behind main as update-status information and points operators to `openclaw update` rather than exposing it as a product defect (`src/commands/status.update.ts:61`). If you are seeing a real update failure on a current Hostinger deployment, please open a new issue with the exact install type, exact update path attempted, expected vs actual behavior, logs/screenshots, and the current OpenClaw version.
