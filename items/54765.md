## Summary

Keep #54765 open. The PR targets a real remaining gap in current main: durable subagent restore still latches `restoreAttempted` before the restore path and silently ignores failures without rolling back partially restored registry state or allowing a clean retry. The proposed PR is also not ready to close as obsolete because the posted review comment identifies an additional rollback state (`resumedRuns`) that should be handled before landing an equivalent fix.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best fix is an atomic durable-restore retry path in `src/agents/subagent-registry.ts`: snapshot and restore both `subagentRuns` and `resumedRuns`, reset `restoreAttempted` after transient restore/resume failure, and add the focused restore-retry regression test so later init cycles prove they retry from clean state.
