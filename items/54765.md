## Summary

Keep open. Current `main` still has the restore-latch bug in `src/agents/subagent-registry.ts`, so this is not implemented on `main`, and the PR patch shown in the review context appears incomplete because it rolls back `subagentRuns` without also rolling back `resumedRuns`.

## Evidence

- **Current main still suppresses retries after a restore failure:** `restoreSubagentRunsOnce()` sets `restoreAttempted = true` before restore work and swallows all exceptions in the catch block without resetting the latch or restoring prior map state.
  - file: [src/agents/subagent-registry.ts:504](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/subagent-registry.ts#L504)
  - command: `nl -ba src/agents/subagent-registry.ts | sed -n '400,560p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Resume guard depends on `resumedRuns`:** `resumeSubagentRun()` immediately returns when `resumedRuns.has(runId)`. A rollback that restores only `subagentRuns` can still leave resumed entries skipped on the next init cycle.
  - file: [src/agents/subagent-registry.ts:412](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/subagent-registry.ts#L412)
  - command: `nl -ba src/agents/subagent-registry.ts | sed -n '400,560p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Current main has no landed restore-retry regression test:** The repo contains `subagent-registry.persistence.resume.test.ts` and `subagent-registry.announce-loop-guard.test.ts`, but no `subagent-registry.restore-retry.test.ts`; search results only show the existing restore/persistence coverage.
  - file: [src/agents](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents)
  - command: `rg -n "restoreAttempted|initSubagentRegistry\(|restore.*retry|retry.*restore|transient restore|poison" src/agents/*.test.ts src/agents/*.e2e.test.ts`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **PR patch in review context appears incomplete:** The submitted diff restores `subagentRuns` and resets `restoreAttempted` in the catch block, but the attached Codex review correctly points out that it does not clear `resumedRuns`, which can still poison a subsequent retry after a partial restore/resume failure.
  - file: [src/agents/subagent-registry.ts](https://github.com/openclaw/openclaw/blob/23e36522eaba01c4f8e6f6f4f65168bd30f8396c/src/agents/subagent-registry.ts)
  - sha: [23e36522eaba](https://github.com/openclaw/openclaw/commit/23e36522eaba01c4f8e6f6f4f65168bd30f8396c)

## Risks / Open Questions

- There appears to be a real bug on current `main`: a transient restore failure can permanently block later restore attempts because `restoreAttempted` is never reset.
- Closing this PR would discard a useful bug report/fix direction even though the current patch likely needs one more change to roll back `resumedRuns` consistently.
