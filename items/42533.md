## Summary

Keep open. Current main still feeds the macOS input node's native AVAudioFormat directly into SFSpeechAudioBufferRecognitionRequest for Push-to-Talk and Talk Mode, with no mono downmix or AVAudioMixerNode/AVAudioConverter path found. The issue is recent, concrete, and not protected/maintainer-authored, but the reported capability still appears missing rather than implemented or superseded.

## Best Possible Solution

Keep the issue open and fix the macOS Speech capture pipeline. The best path is a shared macOS audio helper used by VoicePushToTalk, TalkModeRuntime, VoiceWakeRuntime, and the voice-wake tester that converts multi-channel input to a Speech-compatible mono format before appending to SFSpeechAudioBufferRecognitionRequest, with focused coverage for >2-channel input-format handling where feasible.

## Evidence

This is the exact PTT path that can finalize as emptyOnRelease len=0 when Speech never produces a transcript.
