## Summary

Keep open. Current checked-out main still lacks the parent-session deliveryContext backfill that #69201 proposes for native and ACP `sessions_spawn` children. The gateway already seeds child session delivery from the `agent` request, but when the spawn path sends only `channel` and no `to`/`threadId`, current tests show the child remains channel-only. The PR discussion is also substantive: multiple bot and human review findings were addressed, and the remaining work is a plausible focused core routing fix, not a duplicate or ClawHub item.

## Best Possible Solution

Keep #69201 open for maintainer review or land an equivalent core fix. The best implementation direction is to backfill missing delivery fields from the parent session entry in both `spawnSubagentDirect` and `resolveAcpSpawnRequesterState`, keep current ctx values authoritative, guard channel/to/root-level stale-thread cases, and retain regression coverage that proves the child session entry is seeded with `deliveryContext.to` and `deliveryContext.threadId` after spawn.
