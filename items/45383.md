## Summary

Keep open. Current `main` still stages inbound attachments under `media/inbound/*`, but the read-tool path on `main` does not detect those inbound paths or wrap returned text with the existing untrusted-data guard. The PR discussion also shows the author addressed follow-up review findings on the branch, so this remains useful work rather than something safely closable.
