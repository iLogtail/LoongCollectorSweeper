## Summary

Keep #70191 open. Current main has adjacent health, readiness, status, and model-status primitives, but it does not provide the proposed `/health/deep` or an equivalent auth-gated HTTP endpoint that returns active provider/model/fallbacks and routing/error telemetry without inference. The issue is recent, concrete, contributor-authored, and asks for maintainer API/design direction before a PR, so it should not be auto-closed.

## Best Possible Solution

Keep the issue open for maintainer product/API direction. The likely implementation path is an additive core Gateway probe, either `/health/deep` or an extension of `/ready` details, reusing the existing readiness-detail auth gate where appropriate, backed by the existing configured model/fallback resolution code plus new bounded routing/error telemetry hooks. It should include source tests for local/remote auth behavior, 401/403 leakage boundaries, no-inference behavior, redaction, and docs under the public Gateway health/model docs.
