## Summary

Keep open. This is a recent design proposal with concrete prior art from the reporter's Nix overlay and tracking links, and current main does not yet provide a native `/health/deep` or equivalent authenticated HTTP model-contract probe. Existing core probes are only `/health`, `/healthz`, `/ready`, and `/readyz`; readiness details expose readiness/failing/uptime only, while health/status snapshots expose channel/session/default-model data through existing CLI/RPC paths rather than the requested cheap external model/provider/fallback/routing/error endpoint.

## Best Possible Solution

Keep #70191 open for maintainer design direction. The likely path is a focused core Gateway proposal or PR that either adds an authenticated `/health/deep` endpoint or extends `/ready` details, reuses the existing readiness auth gate, resolves the active model/provider/fallbacks through shared model-selection helpers, wires routing/error telemetry only where core inference dispatch can do so accurately, and adds HTTP probe tests plus docs at `docs/gateway/health.md`. If maintainers prefer a plugin, first define the missing plugin/runtime API for model-contract and routing telemetry rather than treating the current plugin HTTP route surface as sufficient.
