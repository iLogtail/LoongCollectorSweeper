## Summary

Keep #70202 open. Current main has related pieces, but not the central onboarding feature: `openclaw configure --section model` can multi-select model allowlist/fallback entries, and the Control UI can switch the active chat session model. Classic onboarding still prompts for and applies one default model only, with no call to the multi-select allowlist/fallback flow. The issue is recent, author association is NONE, label is only `enhancement`, and the reporter explicitly asked for human input, so this should stay open for maintainer product judgment or implementation.

## Best Possible Solution

Implement the missing onboarding step by reusing the existing configure model flow: after the default model is selected in `runSetupWizard`, prompt for additional `/model` picker entries with `promptModelAllowlist`, persist them through `applyModelAllowlist`, and map selected non-primary entries to `agents.defaults.model.fallbacks` with `applyModelFallbacksFromSelection`. Add focused wizard tests and update onboarding docs so they consistently describe one primary default plus optional additional models.
