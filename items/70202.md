## Summary

Keep #70202 open. Current main has the underlying model config pieces, model fallbacks, allowlist helpers, and UI/session model switching, but the actual `openclaw onboard` wizard still only prompts for one default model and applies that primary model. The multi-select allowlist/fallback flow exists, but it is wired into configure/auth flows rather than the onboarding setup path, so the central onboarding request remains actionable.

## Best Possible Solution

Keep the issue open and implement it by wiring the existing `promptModelAllowlist` flow into `runSetupWizard` after the default-model prompt for interactive local onboarding. The selected default should remain `agents.defaults.model.primary`, additional checked models should populate `agents.defaults.models` and, when appropriate, `agents.defaults.model.fallbacks`. Add setup-wizard tests for quickstart/manual auth choices and update the public onboarding docs to describe the default-plus-enabled-models flow. Leave automatic complexity routing to a separate provider/plugin policy discussion.
