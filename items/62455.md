## Summary

Keep #62455 open. Current main still treats `channels.discord.guilds.<guild>.channels.<channel>.agentId` as an invalid Discord channel config key, while the supported per-channel routing path remains top-level `bindings[]`. The missing piece is a targeted migration hint or doctor repair that rewrites/removes the unsupported `agentId` without losing the user's routing intent.

## Best Possible Solution

Keep the issue open and fix it in the Discord config/doctor surface: either support `agentId` on Discord guild channel entries by normalizing it into an equivalent top-level route binding, or reject it with a targeted validation/doctor message that says to use `bindings[]` with `match.channel: "discord"` and `peer: { kind: "channel", id: "<channelId>" }`. A focused regression test should cover the exact invalid config path and the chosen repair or hint.
