## Summary

Keep PR #39853 open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, and the PR addresses a real multi-bot routing gap. The submitted patch still needs revision because it preserves only `open_id` mentions, lacks focused regression coverage, and overlaps with the broader open Feishu arbitration work in #38939 rather than being cleanly superseded by it.

## Best Possible Solution

Keep #39853 open. The best maintainer path is to add a shared Feishu mention extraction helper that excludes the receiving bot, preserves a stable ID across `open_id`/`user_id`/`union_id` where available, exposes a typed structured-mentions metadata contract, and adds regression tests for Feishu extraction plus inbound conversation-info serialization. Reconcile that contract with #38939 before merging either PR.
