## Summary

Keep open. Current `main` still does not expose structured non-self Feishu mentions in inbound conversation metadata, so the PR addresses a real gap. The patch also appears incomplete as proposed because Feishu mention payloads in this codebase can use IDs beyond `open_id`, which deserves maintainer review rather than conservative auto-close.
