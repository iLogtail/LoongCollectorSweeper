## Summary

Current main still does not expose structured non-self Feishu mentions in inbound conversation metadata. Feishu mentions are still normalized into message-body tags/system hints, and the finalized inbound context passes `WasMentioned` but no `Mentions` array. The PR remains actionable, though its current head still needs maintainer review for Feishu ID fallback and tests.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-ready Feishu plugin change: add a typed structured mentions field to inbound context metadata, populate it from `event.message.mentions` excluding the receiving bot with `open_id`/`user_id`/`union_id` fallback, serialize it in `buildInboundUserContextPrefix()`, and add targeted tests for no mentions, self-only mentions, non-self mentions, and missing-`open_id` payloads. Coordinate with #38939 if maintainers choose dispatcher-level multi-bot arbitration instead.
