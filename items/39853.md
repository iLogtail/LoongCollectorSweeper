## Summary

Keep open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, and the PR addresses a real multi-bot routing gap. It should not be auto-closed as stale or duplicate, but it needs maintainer revisions before merge because the submitted patch only keeps `open_id` mentions and has no focused regression coverage. The related #38939 overlaps on Feishu sibling-bot arbitration, but it is also open and represents a different product approach that maintainers should reconcile rather than using it to close this focused metadata PR automatically.

## Best Possible Solution

Keep #39853 open. The best maintainer path is to add a robust Feishu mention extraction helper that excludes the receiving bot, preserves a stable ID across `open_id`, `user_id`, and `union_id` where available, exposes a typed structured-mentions metadata contract, and adds regression tests for Feishu extraction plus inbound conversation-info serialization. Reconcile that metadata approach with the broader sibling mention arbitration work in #38939 before merging either PR.
