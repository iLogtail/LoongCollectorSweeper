## Summary

Keep open. Current main has partial mitigation, but it does not fully solve the reported user problem: the UI still defaults new cron delivery fields to `last`, new submissions can omit the channel rather than selecting a concrete target, and the runtime still resolves omitted/`last` announce delivery through `last` semantics. A current regression test explicitly documents that with a single configured channel and no previous session target, delivery still fails with a missing-target error, which matches the reported failure class. The related PR #52769 was closed unmerged, so it is not a canonical merged fix.
