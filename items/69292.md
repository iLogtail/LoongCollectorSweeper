## Summary

Keep open. Current main has not implemented this PR's doctor auto-enable skip: non-built-in channel candidates still materialize `plugins.entries.<id>.enabled = true` and allowlist entries. The PR is a focused core doctor/plugin-loader behavior change, not ClawHub work, but it still needs maintainer review because the proposed workspace-origin skip conflicts with current workspace plugin trust rules and the allowlist review concern remains unresolved.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The best path is to reproduce the duplicate-warning case on current main, then make the fix narrowly target the proven global-installed duplicate scenario while preserving workspace plugin trust/enablement and restrictive `plugins.allow` behavior. Add regression coverage for global, workspace, bundled, and allowlist cases before landing.
