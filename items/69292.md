## Summary

Keep open. Current main does not implement the proposed doctor skip: non-built-in channel candidates are still considered already enabled only when `plugins.entries.<id>.enabled === true`, and doctor still applies that auto-enable mutation. The related PRs/issues in the provided context are closed or unmerged rather than canonical open superseders. The PR may need revision because current activation policy requires explicit trust for global/config channel owners and disables workspace plugins by default, matching the unresolved review concerns.

## Best Possible Solution

Keep the PR open and ask for a revision that aligns doctor auto-enable with manifest-owner trust semantics: avoid duplicate `plugins.entries` only when the installed channel plugin is already explicitly trusted/activated under current policy, preserve workspace disabled-by-default behavior, handle restrictive `plugins.allow`, and add regression tests for global, workspace, bundled, and allowlist cases.
