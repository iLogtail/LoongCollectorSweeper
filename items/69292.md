## Summary

Keep PR #69292 open. Current `main` still does not implement the PR's central doctor auto-enable fix: configured plugin-owned channels still materialize `plugins.entries.<pluginId>.enabled = true` for non-built-in plugins, with no global/workspace-origin skip. The PR is not safely closeable as implemented-on-main, duplicate/superseded, or ClawHub work; it targets core doctor/config/plugin-loader behavior. The submitted patch likely needs maintainer revision because the workspace-origin skip conflicts with current workspace plugin enablement rules and the `plugins.allow` path can still fall through.

## Best Possible Solution

Keep the PR open and ask for a focused revision rather than closing it. The likely maintainable direction is a core doctor/config fix that avoids redundant `plugins.entries` for genuinely installed global channel plugins already loaded from manifest discovery, preserves explicit enablement/allowlist behavior for workspace plugins, handles restrictive `plugins.allow`, and adds regression tests for global, workspace, bundled, and allowlist cases.
