## Summary

Current main has stronger partial support for config-injected and forward-compatible models, especially in `models list`, runtime resolution, allowlist metadata, and provider-owned thinking hooks. It still does not implement the RFC's central invariant: one shared resolved-model descriptor consumed by status, list, runtime, picker/auto-reply consumers, capability gates, and docs.

## Best Possible Solution

Keep #37804 open as the umbrella design tracker until maintainers either land a typed shared resolved-model descriptor or replace it with narrower canonical follow-ups. The likely implementation should combine registry/catalog entries, `models.providers`, provider dynamic and forward-compat hooks, transport, capabilities, provenance, and availability, then migrate `models status`, `models list`, runtime execution, picker/auto-reply consumers, thinking gates, and docs to consume that one result.
