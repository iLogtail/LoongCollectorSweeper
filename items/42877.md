## Summary

Keep #42877 open. Current main has memory_search/memory_get, bootstrap injection truncation, append-only pre-compaction memory flushes, and dreaming promotion, but it does not implement the requested bounded CRUD memory write tool with per-store hard character limits, reject-with-usage behavior, substring replace/remove, duplicate detection, or credential scanning.

## Best Possible Solution

Keep the issue open for a maintainer design/implementation pass. The likely path is to extend memory-core, and possibly the memory Plugin SDK contract, with a bounded write/update tool that supports add, replace, and remove; configurable per-store character budgets; strict reject-with-current-usage behavior; duplicate and credential checks; docs; and regression tests across tool registration, policy, and memory-file writes.
