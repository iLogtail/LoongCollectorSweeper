## Summary

Keep open. Current main has bounded read/injection behavior for memory and a promotion path that appends selected candidates to MEMORY.md, but it does not implement the requested bounded write tool with add/replace/remove actions, hard per-store character limits, rejected overflow writes, substring mutation, or duplicate enforcement for arbitrary memory writes. The related #26949 discussion is relevant to prompt injection/cost, but it does not supersede this write-path request.
