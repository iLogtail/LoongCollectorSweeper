## Summary

Keep open. Current `main` still generates macOS LaunchAgent plists without `ProcessType`, and default non-dev installs still omit `WorkingDirectory`, so the reported launchd behavior is not implemented or documented away on current `main`.

## Evidence

- **LaunchAgent plist renderer still has no `ProcessType` entry:** `buildLaunchAgentPlist` renders `RunAtLoad`, `KeepAlive`, `ThrottleInterval`, `Umask`, `ProgramArguments`, optional `WorkingDirectory`, and log/env keys, but no `ProcessType` key at all.
  - file: [src/daemon/launchd-plist.ts:89](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/daemon/launchd-plist.ts#L89)

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Default production gateway install still leaves `workingDirectory` undefined:** `resolveGatewayProgramArguments` only sets `workingDirectory` for dev/Bun repo-root launches. Normal node/non-dev installs return only `programArguments`, so macOS production installs do not get an explicit working directory by default.
  - file: [src/daemon/program-args.ts:189](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/daemon/program-args.ts#L189)
