## Summary

Keep open. Current `main` still generates macOS LaunchAgent plists without `ProcessType`, and it only writes `WorkingDirectory` when one is explicitly supplied, so the reported launchd slow-start workaround is not implemented on `main`. The report is concrete, recent, and backed by before/after launch timing plus a linked follow-up PR, so there is not enough evidence to close this conservatively.

## Evidence

- **Current plist builder omits `ProcessType` and does not default `WorkingDirectory`:** `buildLaunchAgentPlist` renders `RunAtLoad`, `KeepAlive`, `ThrottleInterval`, `Umask`, `ProgramArguments`, optional `WorkingDirectory`, and log paths. There is no `ProcessType` key, and `WorkingDirectory` is only emitted when the caller passes one.
  - file: [src/daemon/launchd-plist.ts:89](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/daemon/launchd-plist.ts#L89)
  - command: `nl -ba src/daemon/launchd-plist.ts | sed -n '89,117p'`
  - sha: [8bead989da0c](https://github.com/openclaw/openclaw/commit/8bead989da0c62d8df91094ef9b881b725b282c8)
- **Install path writes exactly that plist:** `installLaunchAgent` calls `writeLaunchAgentPlist`, which builds and writes the LaunchAgent plist before bootstrap. There is no extra macOS-specific `ProcessType` or default working-directory handling in the install path.
  - file: [src/daemon/launchd.ts:585](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/daemon/launchd.ts#L585)

  - file: [src/daemon/launchd.test.ts:469](https://github.com/openclaw/openclaw/blob/8bead989da0c62d8df91094ef9b881b725b282c8/src/daemon/launchd.test.ts#L469)
