## Summary

Keep open. Current main still enqueues ACP parent relay progress, stall, and resumed events into the parent session, and the existing tests still assert those intermediate relays are surfaced. The linked PR #55896 is the exact narrow fix, but the provided GitHub context shows it is still open and unmerged, so the bug remains actionable in OpenClaw core.

## Best Possible Solution

Keep #55807 open until #55896 or an equivalent maintainer-approved patch lands. The best implementation path is a narrow event-level visibility guard in the ACP parent relay for visible Discord thread parent sessions, with regression coverage in src/agents/acp-spawn-parent-stream.test.ts and src/agents/acp-spawn.test.ts proving :progress, :stall, and :resumed are suppressed while final lifecycle notices still surface.
