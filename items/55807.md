## Summary

Keep open. Current `main` still routes ACP parent relay `:progress`, `:stall`, and `:resumed` events into parent-session system events with the parent delivery context, and the current tests explicitly assert that behavior. I found no Discord visible-thread suppression on the spawn path.

## Evidence

- **Relay enqueues surfaced events without channel-specific filtering:** `emit()` logs and then calls `enqueueSystemEvent(...)` whenever `surfaceUpdates !== false`; there is no Discord/thread check before enqueueing into the parent session.
  - file: [src/agents/acp-spawn-parent-stream.ts:190](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/acp-spawn-parent-stream.ts#L190)
  - command: `nl -ba src/agents/acp-spawn-parent-stream.ts | sed -n '190,350p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Progress, stall, and resumed are all emitted as parent-session system events:** `flushPending()` emits `:progress`; the no-output watcher emits `:stall`; resumed output emits `:resumed`. These are exactly the intermediate relay events the issue reports as leaking into Discord threads.
  - file: [src/agents/acp-spawn-parent-stream.ts:243](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/acp-spawn-parent-stream.ts#L243)
  - command: `nl -ba src/agents/acp-spawn-parent-stream.ts | sed -n '190,350p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Spawn path passes Discord delivery context into the relay but does not enable any suppression mode:** `spawnAcpDirect()` creates the parent relay with `deliveryContext: parentDeliveryCtx` and `emitStartNotice: false`, but no flag or branch suppresses intermediate relay events for visible Discord threads.
  - file: [src/agents/acp-spawn.ts:1250](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/acp-spawn.ts#L1250)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '1250,1315p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current tests codify the leaking behavior:** The relay tests expect a progress relay message (`"codex: hello from child"`), a stall notice (`"has produced no output for 1s"`), and a resumed notice (`"resumed output."` / `"codex: resumed output"`). That means current `main` still treats these as expected surfaced events, not suppressed Discord-only internals.
