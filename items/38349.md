## Summary

Keep open. Current main still does not implement contextual group activation: the activation contract remains `mention | always`, config/schema/docs/tests still expose only mention-based or always-on group behavior, and the WhatsApp/Telegram runtime paths do not have a peeking/engaged decision-model state machine. The linked implementation PR #38350 is closed unmerged in the provided GitHub context, so the requested behavior has not shipped. This also should not be closed as ClawHub-only today because the request changes pre-dispatch activation for bundled WhatsApp and Telegram groups, and current plugin/message hook surfaces do not clearly let a plugin replace those bundled channel gating decisions before skipped messages are dropped.

## Best Possible Solution

Keep #38349 open as the canonical product/design issue. A good path is either to implement an explicit `contextual` activation contract across config, session state, gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability, CLI/docs, and tests, or first add a typed plugin/channel activation seam so this can be implemented outside core without patching bundled channel internals. PR #38350 and the older #1721 are useful prior design/implementation context, but neither is shipped on current main.
