## Summary

Close as `clawhub`. The PR is a large port of an already-existing external E-Claw plugin into the bundled `extensions/` workspace, but current OpenClaw direction says optional plugins should generally stay in their own repos and be distributed through ClawHub/npm unless there is a strong reason to promote them into core.

## Evidence

- **Vision prefers external plugin ownership:** The repo vision says the preferred plugin path is npm distribution plus local extension loading, plugin authors should host and maintain plugins in their own repositories, the bar for adding optional plugins to core is intentionally high, and plugin promotion belongs in ClawHub.
  - file: [VISION.md:68](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/VISION.md#L68)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Plugin docs point channel/plugin authors to ClawHub publish/install flow:** The plugin authoring guide tells external plugin authors to publish through ClawHub and install with `openclaw plugins install clawhub:@myorg/openclaw-my-plugin`; bundled in-repo plugins are a separate path, not the default recommendation for community integrations.
  - file: [docs/plugins/building-plugins.md:127](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/docs/plugins/building-plugins.md#L127)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **User-facing plugin docs support installable external native plugins:** The plugin docs describe native plugins as including community npm packages, require external plugins to be installed with `openclaw plugins install`, and list official plugins that are installable via npm rather than bundled in core.
  - file: [docs/tools/plugin.md:64](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/docs/tools/plugin.md#L64)
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **Current main does not ship E-Claw:** A repo search on current `main` found no `eclaw` references under bundled extensions, channel docs, labeler config, package metadata, or lockfile. This is not implemented on main; the close is about scope and placement, not duplication.
  - command: `rg -n 'eclaw' docs/channels extensions .github package.json pnpm-lock.yaml`
  - sha: [7bd74758c5f5](https://github.com/openclaw/openclaw/commit/7bd74758c5f54c546b52708359a25c08cae74ad7)
- **PR discussion identifies an existing standalone plugin:** The PR body says this change ports the existing `@eclaw/openclaw-channel` package from the author's separate repository into the bundled workspace, which is exactly the kind of external plugin path the current vision/docs prefer.
  - sha: [fee2299e8a2d](https://github.com/openclaw/openclaw/commit/fee2299e8a2d7d48be5e7a6c0486d6c517d46f29)
