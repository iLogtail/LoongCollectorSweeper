## Summary

Current `main` already refreshes session skill snapshots on subsequent turns when the skills snapshot version changes, and Slack DMs flow through that generic reply path. The behavior is also documented as mid-session hot reload on the next agent turn, so the reported stale `available_skills` behavior from 2026.3.8 is implemented on current `main` and is present in the latest inspected release code as well.

## Evidence

- **Session snapshots refresh on later turns when skills change:** `ensureSkillSnapshot` reads the current snapshot version, enables the skills watcher, and rebuilds/persists `skillsSnapshot` for non-first turns when the cached version/filter is stale.
