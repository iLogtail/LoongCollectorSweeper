## Summary

Keep open. Current main still sends webhook agent actions through an isolated cron agent turn: hook mappings and /hooks/agent accept sessionKey, but no sessionTarget/targetExistingSession field is exposed or threaded, and sessionTarget: "isolated" forces a new session even when a session key is supplied. The related closed PRs appear unmerged, and the open wake-hook PR/comment is explicitly a workaround that does not solve action: "agent".
