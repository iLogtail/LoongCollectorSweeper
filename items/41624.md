## Summary

Keep open. Current `main` still contains the exact unfixed Dockerfile behavior from this PR: the optional browser-install path creates `/home/node/.cache/ms-playwright` as `root` and only `chown`s that subdirectory, while the final image runs as `node`. I did not find evidence in the checked-out `main` that this ownership fix was implemented.
