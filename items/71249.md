## Summary

Keep PR #71249 open. Current main still lacks the generic turn-start cleanup and failed-clear suppression that this PR proposes for stale `pluginDebugEntries`; existing main only refreshes the session entry from disk before appending verbose/trace plugin payloads, so stale persisted Active Memory lines can still be rehydrated. The PR discussion and bot review describe a focused bug fix with targeted regression coverage, and no protected-label or maintainer-authorship rule requires auto-keeping for policy reasons; the technical reason is that the work still appears useful and not implemented on main.

## Best Possible Solution

Keep the PR open and review/land this class of runtime-scoped fix if CI and maintainer review pass: clear `pluginDebugEntries` at reply-turn start with preserve-activity semantics, make cleanup best-effort, suppress same-turn rehydration when persisted clearing fails, and keep the Active Memory writer path extension-owned. The regression tests proposed in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts` are the right guardrail surface.
