## Summary

Keep PR #71249 open. Current main still has the stale plugin debug banner behavior the PR is trying to fix: verbose/trace reply assembly refreshes persisted session state and appends `pluginDebugEntries`, but there is no turn-start clear, no best-effort cleanup failure handling, and no suppression path to prevent stale disk entries from being rehydrated. The PR is a focused, recent bug-fix PR with targeted tests in the supplied context, and its bot review concern was addressed by a follow-up commit rather than left unresolved.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best product/code path is to add a core reply-runtime turn-start cleanup for turn-scoped `pluginDebugEntries`, preserve session activity timestamps for that metadata-only clear, make cleanup best-effort, and prevent verbose/trace output from rehydrating stale debug lines after a clear failure. The targeted regression tests described in the PR belong in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`.
