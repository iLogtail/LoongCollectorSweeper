## Summary

Keep open. Current `main` still has the reported behavior: `openclaw dashboard` builds a localhost URL from config, copies it, and opens the browser without any gateway reachability preflight, so it can still imply success while the local gateway is down or unconfigured.

## Evidence

- **Dashboard command does not preflight gateway reachability:** `dashboardCommand` reads config, resolves the Control UI URL, logs it, copies it, and calls `openUrl(...)` when browser opening is supported. There is no call to `probeGatewayReachable`, `waitForGatewayReachable`, `probeGateway`, or any other health check before logging browser-open success.
