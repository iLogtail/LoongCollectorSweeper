## Summary

Keep PR #69697 open. Current main still has the exact `hashToolCall` crash path for undefined params: the tool-loop serializer returns `JSON.stringify(value)` for non-object values, `JSON.stringify(undefined)` is the JavaScript value `undefined`, and `digestStable` passes that into `createHash().update(...)`. The PR is focused on the implicated file and remains useful, though it should add a regression test before merge.

## Best Possible Solution

Keep the PR open and either ask the contributor to add focused regression coverage or have a maintainer apply the same minimal serializer guard with tests. The likely patch belongs in `src/agents/tool-loop-detection.ts` plus `src/agents/tool-loop-detection.test.ts`, covering undefined params through `hashToolCall` and preferably `recordToolCall`/`recordToolCallOutcome`.
