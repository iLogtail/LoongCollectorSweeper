## Summary

Keep open. Current main still lacks the requested CLI warning for shell-looking text passed via main-session cron system events, and the same author has an open unmerged implementation candidate, #63112, that explicitly links this issue with GitHub closing syntax.

## Best Possible Solution

Keep this issue open and review #63112. The best fix is a small shared helper used by both `cron add` and `cron edit` to emit a non-blocking stderr warning before the RPC call when `sessionTarget` resolves to `main`, payload kind is `systemEvent`, and the text looks like a shell command, with focused CLI tests for warning and non-warning cases.
