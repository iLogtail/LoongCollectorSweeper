## Summary

Keep #39599 open. Current main still has no create-time Feishu/Lark announce target validation: gateway validation only checks configured/ambiguous announce channels, the cron service only requires `delivery.to` for webhook delivery, and the CLI still forwards `--channel feishu` with an undefined `to`. The runtime missing-target path is still present for Feishu when no explicit or resolved target exists. Related PRs #39602 and #39683 are open/unmerged candidate fixes, not shipped behavior.

## Best Possible Solution

Keep #39599 open and land a gateway-side validation fix for cron creation and update. The fix should reject isolated announce delivery for Feishu/Lark when neither an explicit `delivery.to` nor a deterministic resolved target is available, then add focused regression coverage in `src/gateway/server-methods/cron.validation.test.ts`, `src/cron/service.jobs.test.ts`, and CLI tests if CLI fail-fast validation is added.
