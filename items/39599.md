## Summary

Issue #39599 should remain open. Current main still accepts Feishu announce cron delivery without `delivery.to` at the protocol, gateway/server-method, cron service, and CLI creation paths, while runtime target resolution can still fail later when no explicit or deterministic target exists. The related PRs #39602, #39683, and #40031 are open/unmerged candidate fixes, not shipped replacements.

## Best Possible Solution

Keep #39599 open and land one focused canonical fix. Add create/update validation at the gateway/service boundary for Feishu/Lark announce cron delivery when neither explicit `delivery.to` nor a deterministic resolved target is available, with focused regression coverage for cron service/gateway behavior and CLI fail-fast behavior if the CLI layer changes. Use #39602, #39683, and #40031 as candidate implementations or reference material, but do not close this issue until one fix lands on main.
