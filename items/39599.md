## Summary

Keep open. Current main still does not reject `announce + feishu + missing delivery.to` at cron creation time, and the two related fix PRs in the provided GitHub context are still open/unmerged rather than shipped on main.

## Best Possible Solution

Keep #39599 open until a maintainer lands one canonical fix. The fix should add gateway/service-side validation in `src/cron/service/jobs.ts` for Feishu/Lark announce delivery without an explicit or deterministically resolved target, mirror the error in CLI/UI validation for better feedback, and add regression tests for rejected missing-target creation/update plus accepted explicit-target creation.
