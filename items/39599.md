## Summary

Keep #39599 open. Current main still lacks create-time validation for Feishu/Lark announce cron jobs without `delivery.to`; the protocol accepts the shape, gateway/service validation does not reject it, and the CLI still forwards an omitted `--to`. The existing ClawSweeper comment is still consistent with current main: #39602, #39683, and #40031 are candidate fixes in the provided context, not shipped replacements.

## Best Possible Solution

Keep #39599 open and land one focused fix at the gateway/service boundary: reject Feishu/Lark announce cron create and update requests when neither explicit `delivery.to` nor a deterministic resolved target is available. Add regression coverage for service and gateway behavior, and add CLI validation only as an earlier user-feedback layer.
