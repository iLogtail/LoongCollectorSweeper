## Summary

Keep PR #72216 open. Current main still has the exact Nostr setup-status import chain the PR targets: the setup-only Nostr plugin delegates status resolution to the full setup surface, and that surface reaches `nostr-tools` through `nostr-key-utils`. The broader onboarding cold-path work on main defers some QuickStart status loading, but it does not remove the Nostr status delegation or cover non-deferred channel setup status collection.

## Best Possible Solution

Keep the PR open and land it, or apply an equivalent main change, so `extensions/nostr/src/channel.setup.ts` keeps setup status on the lightweight setup-only surface while preserving normal full Nostr configuration/runtime behavior. Address the small relay-count test robustness nit before merge if practical.
