## Summary

Keep PR #72216 open. Current main still has the targeted Nostr setup-status import chain: the setup-only Nostr plugin delegates status to the full setup surface, that surface reaches `nostr-tools`, and setup status collection still calls those resolvers outside deferred-selection paths. The PR remains a focused fix for the linked #72182 failure rather than obsolete cleanup.

## Best Possible Solution

Keep #72216 open and land it, or apply an equivalent main change, so `extensions/nostr/src/channel.setup.ts` keeps setup status resolution on the lightweight setup-only surface while preserving full Nostr setup/runtime behavior in `setup-surface.ts`. Before merge, address the minor relay-count test brittleness if practical.
