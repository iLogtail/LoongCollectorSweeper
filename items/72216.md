## Summary

Keep PR #72216 open. Current main still has the exact Nostr setup-status import chain the PR targets: the setup-only Nostr plugin delegates status resolution to the full setup surface, and that surface reaches `nostr-tools` through `nostr-key-utils`. The provided PR discussion and linked #72182 reproduction support landing this focused fix rather than closing it as obsolete.

## Best Possible Solution

Keep PR #72216 open and land it, or apply an equivalent main change, so `extensions/nostr/src/channel.setup.ts` keeps status resolution on the lightweight setup-only surface while preserving full Nostr configuration/runtime behavior. Add the regression test that proves status checks do not load `setup-surface.js`, and make the expected relay count derive from `DEFAULT_RELAYS.length` for test robustness.
