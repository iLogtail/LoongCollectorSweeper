## Summary

Keep #8495 open. Current main still treats WhatsApp group allowlisting as admission/skip behavior only; it does not auto-leave unauthorized groups, emit a WhatsApp group-join hook/event, or expose a WhatsApp leave-group action. The issue also has a maintainer-side Codex review comment explicitly keeping it open, and related #11489 remains broader/open rather than a shipped replacement.

## Best Possible Solution

Keep #8495 open. The best path is a deliberate bundled WhatsApp plugin change: add opt-in config/schema/docs for auto-leave, implement a Baileys-backed leave path in the WhatsApp runtime with account and target authorization, cover unauthorized group admission/join behavior with tests, and decide whether join notification belongs in a WhatsApp-specific event, a generic group-join plugin hook, or the broader #11489 group-pairing flow. Manual leaveGroup should only be added if the channel action/runtime surface can enforce safe target authorization.
