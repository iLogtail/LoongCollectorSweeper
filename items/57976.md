## Summary

Keep open. Current `main` does not implement Docker GPU passthrough: the sandbox Docker config has no `gpus` field, the runtime schema would reject it as an unknown key, and Docker create args never emit `--gpus`. The request is coherent and actionable, and current GPU support exists only for the OpenShell backend, not Docker.

## Evidence

- **Sandbox Docker config type has no `gpus` field:** `SandboxDockerSettings` lists Docker knobs such as `memory`, `memorySwap`, `cpus`, `ulimits`, `dns`, `extraHosts`, and `binds`, but there is no `gpus` property to represent the requested config.
  - file: [src/config/types.sandbox.ts:3](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/types.sandbox.ts#L3)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Runtime schema would reject `sandbox.docker.gpus`:** `SandboxDockerSchema` is a strict object and enumerates accepted Docker keys; `gpus` is not among them, so the issue's proposed config would currently be invalid on `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:102](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/zod-schema.agent-runtime.ts#L102)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Resolved Docker config has no GPU slot:** `resolveSandboxDockerConfig()` merges supported Docker options into the effective sandbox config, but only carries fields like `network`, `user`, `pidsLimit`, `memory`, `memorySwap`, `cpus`, `ulimits`, `dns`, `extraHosts`, and `binds`. There is no GPU field to propagate.
  - file: [src/agents/sandbox/config.ts:83](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/sandbox/config.ts#L83)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Docker create args never emit `--gpus`:** `buildSandboxCreateArgs()` adds flags for network, user, env, seccomp/AppArmor, DNS, pids, memory, swap, CPUs, ulimits, and binds, but there is no branch that appends Docker's `--gpus` flag.
  - file: [src/agents/sandbox/docker.ts:377](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/sandbox/docker.ts#L377)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Docs list current Docker sandbox knobs and omit GPU passthrough:** The documented `agents.defaults.sandbox.docker` example includes image, network, user, setup, resource limits, security profiles, DNS, extra hosts, and binds, but no GPU option.
  - file: [docs/gateway/config-agents.md:670](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/gateway/config-agents.md#L670)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Main and latest release contain no Docker GPU passthrough references:** Searching both the current main SHA and latest release SHA for `gpus` or `--gpus` under `src` and `docs` returned no matches.
  - command: `git grep -n 'gpus\|--gpus' 90877e0d4274c53fd91bf5787265c4d6889dc9a7 -- src docs; git grep -n 'gpus\|--gpus' a9797214338ba31c52c796adbb75afb16e0684a9 -- src docs`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **GPU support exists only on the OpenShell backend:** OpenShell already exposes a `gpu` boolean in its plugin config and turns that into an `--gpu` CLI flag, which shows the request is still specifically unaddressed for the Docker backend rather than generally solved.
  - file: [extensions/openshell/src/backend.ts:357](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/openshell/src/backend.ts#L357)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)

## Risks / Open Questions

- There is existing GPU support on the OpenShell backend (`docs/gateway/openshell.md:129`, `extensions/openshell/src/config.ts:9`), so maintainers may want to decide explicitly whether Docker should reach parity or whether GPU users should be steered to OpenShell in docs.
- The issue has a linked open PR in the provided timeline (`#58124`), so closing the issue now would be premature even aside from the missing implementation.
