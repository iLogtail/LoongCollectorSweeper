## Summary

Keep PR #72124 open. Current main still routes `openclaw models auth paste-token --provider anthropic` through a validator that only accepts `sk-ant-oat01-`, while OpenClaw already supports Anthropic API-key auth elsewhere. The PR is a focused, non-obsolete bug fix for #72121 rather than something already implemented, duplicated, or out of scope.

## Best Possible Solution

Keep #72124 open for maintainer review and normal CI. The best fix is the PR's direction: widen the shared Anthropic token validator used by `models auth paste-token` and the Anthropic plugin token flow to accept both `sk-ant-oat01-` setup tokens and `sk-ant-api03-` API keys, with focused regression tests for both prefixes and error messages.
