## Summary

本 Issue 报告了 metric_system_v2 中系统启动时间计算存在误差的缺陷，建议改用 /proc/stat 的 btime 字段。经核查当前 main 分支代码，GetSystemBootTime 仍基于 uptime 倒推计算，尚未引入 btime 解析。该问题真实有效且维护者已确认修复方案，标记为 good first issue，目前无关联 PR 或重复工单，应继续保持开放等待贡献。

## Best Possible Solution

保持开启。下一步可由贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的解析逻辑，以消除 time(NULL) - uptime 带来的计算抖动。实现后补充单元测试验证边界条件，并提交 PR 关联本 Issue。
