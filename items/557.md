## Summary

本 Issue 报告了 metric_system_v2 中系统启动时间计算存在微小漂移的问题，建议优先解析 /proc/stat 的 btime 字段。经核查当前 main 分支代码与讨论记录，维护者已确认方案并标记为 good first issue，但尚未有 PR 合并修复。问题依然有效且待贡献，故保持开启。

## Best Possible Solution

保持开启。建议贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的读取与解析逻辑，将其置于现有判断链最前方以消除计算抖动。实现后需补充单元测试验证精度与回退路径，并提交 PR 关联本 Issue。维护者可在 PR 合并后关闭本 Issue。
