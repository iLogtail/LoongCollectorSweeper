## Summary

本 Issue 报告 metric_system_v2 中系统启动时间计算存在误差，指出当前使用 time(NULL) - uptime 倒推会导致重启 Logtail 时指标漂移，建议改用 /proc/stat 的 btime 字段。经核查当前 main 分支源码与讨论记录，该计算逻辑尚未修改，问题仍然存在。讨论区维护者已确认方案可行并标记为 good first issue，属于有效且信息充分的缺陷报告，应继续保持开放。

## Best Possible Solution

保持开启。下一步建议贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的读取与解析逻辑，将其置于现有 if 判断链的最前方，以消除 time(NULL) - uptime 带来的计算抖动。实现后需补充对应单元测试验证边界条件与回退路径，并提交 PR 关联本 Issue。

## Evidence

- **current_code_state:** GetSystemBootTime 函数仍采用 time(NULL) - upTime 计算启动时间，未接入 /proc/stat btime 解析，Issue 报告的误差问题在 main 分支依然存在。
