## Summary

本 Issue 报告了 metric_system_v2 中系统启动时间计算存在微小漂移的问题，指出当前使用 time(NULL) - uptime 倒推存在误差，建议优先解析 /proc/stat 的 btime 字段。经核查，该问题为有效缺陷，维护者已确认方案可行性并标记为 good first issue。当前 main 分支相关逻辑尚未合并修复 PR，核心计算路径仍依赖原有实现，故保持开启以等待贡献。

## Best Possible Solution

保持开启。下一步建议贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的读取与解析逻辑，将其置于现有判断链最前方以消除计算抖动。实现后需补充单元测试验证精度与回退路径，并提交 PR 关联本 Issue。维护者可在 PR 合并后关闭本 Issue。

## Evidence

- **current_code_state:** GetSystemBootTime 函数当前仍依赖 time(NULL) - upTime 计算启动时间，尚未合并 /proc/stat btime 解析逻辑，报告的精度漂移问题在 main 分支依然存在。
