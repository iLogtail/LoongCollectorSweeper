## Summary

本 Issue 报告 metric_system_v2 中 system_boot_time 计算存在漂移，建议改用 /proc/stat 的 btime 字段。经核查当前 main 分支代码，GetSystemBootTime 函数仍依赖 uptime 倒推逻辑，尚未合并读取 btime 的修复。维护者已确认方案并标记为 good first issue，问题依然有效且待贡献。

## Best Possible Solution

保持开启。建议贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的读取与解析逻辑，消除计算抖动。实现后补充单元测试验证精度与回退路径，并提交 PR 关联本 Issue。维护者可在 PR 合并后关闭。
