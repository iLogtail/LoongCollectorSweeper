## Summary

本 Issue 报告了 metric_system_v2 中系统启动时间计算存在微小漂移的缺陷，指出应优先解析 /proc/stat 的 btime 字段而非使用 time(NULL) - uptime 倒推。维护者已确认方案可行性并标记为 good first issue。经核查当前 main 分支代码，GetSystemBootTime 函数仍依赖原有倒推逻辑，尚未合并修复 PR，问题依然存在，故保持开启等待贡献。

## Best Possible Solution

保持开启。建议贡献者在 core/common/TimeUtil.cpp 的 GetSystemBootTime 函数中优先添加对 /proc/stat btime 字段的读取与解析逻辑，将其置于现有判断链最前方以消除计算抖动。实现后需补充单元测试验证精度与回退路径，并提交 PR 关联本 Issue。维护者可在 PR 合并后关闭本 Issue。
