## Summary

Current main already implements the Telegram `/think` native menu behavior this PR was trying to add: dynamic command choices can receive an explicit provider/model context, the Telegram handler resolves that context from the target session model override before rendering menus, and tests/docs cover target-session model menus. The PR’s older `max -> xhigh` behavior is also obsolete because current main treats `max` as its own provider-profile thinking level.

## Best Possible Solution

Close this stale PR and keep the current main implementation, which resolves Telegram native argument menus from the target session model using normalized session-store lookup and provider-profile thinking levels. Do not merge the PR branch because its `max` handling predates current first-class `max` semantics.
