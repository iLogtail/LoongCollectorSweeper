## Summary

Keep #12394 open. Current main still lacks the requested WhatsApp channel-level heartbeat recipient contract: `channels.whatsapp.heartbeat` is visibility-only, and heartbeat OK plus alert delivery still share the same resolved heartbeat delivery target instead of an OK-only `target`/`targets` list.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive WhatsApp channel heartbeat recipient contract. A complete fix should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` or `targets`, validate recipients against account and allowlist policy, apply those recipients only to `HEARTBEAT_OK` delivery without changing alert/content routing, and add regression coverage for single target, multiple admin targets, omitted-target backward compatibility, per-account config, and invalid targets.
