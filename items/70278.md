## Summary

Keep PR #70278 open. Current main still has the broken `docs/plugins/memory-wiki.md` QMD bridge example with `"memory-wiki"` nested under `memory`, while the docs and source contract both put plugin config under `plugins.entries.memory-wiki.config`. The PR is a focused one-file docs correction and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and land or reapply the focused docs correction so the QMD + bridge example nests `memory-wiki` under `plugins.entries`, matching the runtime schema and existing Memory Wiki docs. No core code change appears necessary for this item.
