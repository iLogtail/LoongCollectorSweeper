## Summary

Keep PR #70278 open. Current main still contains the broken QMD + bridge example in `docs/plugins/memory-wiki.md`, nesting `memory-wiki` under `memory`, while the same page and config schema require plugin config under `plugins.entries.memory-wiki.config`. The related memory-wiki bridge reports in the supplied context concern runtime bridge behavior and do not supersede this narrow docs fix.

## Best Possible Solution

Keep the PR open and either merge this focused docs-only correction or apply the same `plugins.entries["memory-wiki"].config` nesting fix directly to `docs/plugins/memory-wiki.md`. No core behavior change is needed; the existing config schema already supports the corrected shape.
