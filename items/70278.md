## Summary

Keep PR #70278 open. Current main still contains the incorrect QMD + bridge example nesting in `docs/plugins/memory-wiki.md`, and the config contract confirms `memory-wiki` plugin configuration belongs under `plugins.entries`, not under `memory`.

## Best Possible Solution

Keep the PR open and either merge it or recreate the same docs-only fix on current main: close the `memory` object after `backend: "qmd"`, then place the `memory-wiki` entry under `plugins.entries["memory-wiki"]`. No core code or plugin API change is needed.
