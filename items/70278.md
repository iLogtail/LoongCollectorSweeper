## Summary

Keep open. Current main still has the broken `docs/plugins/memory-wiki.md` QMD + bridge example: `memory-wiki` is nested inside `memory`, even though the same docs and config types put plugin configuration under `plugins.entries.<id>.config`. The PR is a narrow docs-only correction and is not superseded by the related Memory Wiki bridge runtime bug reports.

## Best Possible Solution

Keep the PR open and land the docs-only fix, or apply an equivalent maintainer patch on main that moves `memory-wiki` under `plugins.entries` in the QMD + bridge example while leaving `memory.backend: "qmd"` under `memory`.
