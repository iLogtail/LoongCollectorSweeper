## Summary

Keep #56419 open. Current main still sends a fixed `timeoutMs: 30_000` from the shared `image` tool into the media-understanding provider path, and that runtime still aborts with `AbortController` when the passed timeout expires. The related fixing PR #56454 is still open/unmerged in the provided GitHub context, and the later overlapping ClawSweeper report #62944 also found this timeout path still unresolved.

## Best Possible Solution

Keep the issue open and land a narrow fix in the image tool path: resolve a configured image-analysis timeout, preserve the current `30_000ms` default when unset, pass the resolved value through every `describeImage(s)` call, document the chosen config surface, and add focused regression tests that prove default and configured timeout propagation.
