## Summary

PR #72174 should remain open. Current main still has the attach-only launchd-management bug: both the `--attach-only` / `--no-launchd` app launch path and the Debug Settings toggle write the disable marker and then call the daemon uninstall path. The PR's marker-only change and focused regression test still address real current behavior and are not superseded by the related launchd lifecycle work.

## Best Possible Solution

Keep PR #72174 open for maintainer review or landing. The best fix is this PR, or an equivalent marker-only attach-only path, so both launch flags and the Debug Settings toggle only persist `disable-launchagent` and never call the launchd uninstall path; keep the focused regression test proving no daemon commands are issued.
