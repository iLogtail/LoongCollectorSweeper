## Summary

Keep open. Current main only partially overlaps this PR: ordinary non-JSON `status` already uses `buildPluginCompatibilitySnapshotNotices`, but `status --all` and setup still call the full diagnostics-backed `buildPluginCompatibilityNotices`. The remaining request is real, but the submitted direct swap is not clearly safe because hook-only and `before_agent_start` compatibility notices are derived from hook registrations that the current metadata-only path can skip.

## Best Possible Solution

Keep the PR open but request rework: either teach the metadata snapshot path to carry the compatibility signals needed for `hook-only` and `before_agent_start`, or add a narrower lightweight compatibility report that preserves those notices without full diagnostics loading. Then switch `status --all` and setup to that safe path, add targeted tests for both surfaces, and split unrelated timeout/TTS/capability changes out of this PR.
