## Summary

Keep open. Current main has only partially addressed the performance lane: regular status/config validation use the snapshot helper, but `status --all` local diagnosis and setup still use the diagnostics-backed compatibility helper. The PR is therefore not implemented on main or cleanly superseded. It also should not be auto-closed/merged as-is because the unresolved review concern is supported by current code: snapshot notices derive `hook-only` and `before_agent_start` warnings from registered hooks, while the metadata snapshot path returns before module hook registration.

## Best Possible Solution

Keep the item open and ask for a narrower fix: either enrich the metadata snapshot/manifest path so it can produce the same compatibility notices without runtime module loading, or gate the snapshot route to cases where notice fidelity is provably unchanged. Then update focused tests for `status --all` and setup compatibility prompts, preserve full diagnostics for `plugins inspect`/doctor, and remove unrelated timeout/TTS/capability-provider churn from this PR.
