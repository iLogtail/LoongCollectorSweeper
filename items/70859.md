## Summary

Keep open. Current main has already moved setup and regular `status` compatibility checks onto the snapshot path, but the specific `status --all` path in this PR is still on the full diagnostics notice builder. The PR also has an unresolved correctness concern: the real `loadModules: false` loader path stops before runtime module registration, so a status-all switch needs stronger implementation/test proof before it is safe to treat as shipped or obsolete.

## Best Possible Solution

Keep the PR open, but require changes before merge: either implement a metadata-backed compatibility snapshot that can faithfully report `hook-only` and `before_agent_start` without loading full plugin runtimes, or keep `status --all` on diagnostics until that contract exists. The setup part is already shipped on main via `2f81c5f5`; the remaining useful work is a focused, well-tested status-all optimization or a narrower follow-up issue/PR that addresses the snapshot fidelity gap.
