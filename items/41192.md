## Summary

Close as ClawHub/plugin-scoped. The PR is a draft with an unfilled template and proposes a llama.cpp-specific tiered KV cache/speculative decoding sidecar in core agent/gateway config. Current OpenClaw direction keeps optional provider/runtime capabilities out of core, and the existing plugin API already exposes provider hooks, long-lived services, gateway methods, config, and ClawHub/npm distribution for this kind of integration.

## Best Possible Solution

Do not merge this broad core PR. Keep OpenClaw core provider-agnostic and move the tiered KV cache/speculative decoding work into a ClawHub/npm provider plugin using existing provider hooks and plugin services; if implementation reveals a precise missing seam, open a narrow plugin-API proposal instead of adding a one-off core sidecar.
