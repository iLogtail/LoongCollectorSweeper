## Summary

Keep #50681 open. Current main supports fail-closed Telegram group allowlisting and logs/skips unknown groups, but it does not implement the requested first-class unknown-group response path: no `unknownGroupAction`, warning message/cooldown config, or automatic `leaveChat` behavior exists in schema, runtime, tests, or docs.

## Best Possible Solution

Add this in the bundled Telegram plugin inbound preflight: extend the Telegram config schema/docs with an explicit unknown-group action (`ignore`, `warn`, `leave` or equivalent), detect `group-chat-not-allowed` before assistant dispatch, optionally send a configured warning with a per-chat cooldown, then call Telegram `leaveChat` for the `leave` action. Cover with focused unit tests around schema parsing, blocked-group handling, cooldown state, warning send failures, and leave failures; live Telegram validation would be useful before landing because the issue is about real Bot API behavior.
