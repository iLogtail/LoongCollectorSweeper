## Summary

Keep open. Current main still skips session delivery-context lookup for Discord A2A announce targets, so the resolved announce target carries `channel`/`to` but no session `accountId`. The related #51626 was closed as a duplicate of this issue, making #42652 the canonical remaining tracker rather than something to close.

## Best Possible Solution

Fix the remaining Discord A2A announce routing path by hydrating the target session route before announce delivery when the parsed target lacks `accountId`, likely by setting Discord `preferSessionLookupForAnnounceTarget: true` or by making `resolveAnnounceTarget()` merge session metadata into parsed targets without regressing channels that rely on parsed session keys. Add a regression test where a Discord target session has `deliveryContext.accountId: "work"` and the announce gateway `send` includes `accountId: "work"`.
