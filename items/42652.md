## Summary

Keep open. Current main still skips session lookup for Discord announce targets because Discord does not set `preferSessionLookupForAnnounceTarget`; the fallback parsed from the session key carries `channel`/`to` but no `accountId`, and the A2A announce send forwards that missing account id. That leaves the reported multi-account Discord routing problem unresolved unless the intended account happens to be the configured default account.

## Evidence

- **Discord announce target still returns fallback before session lookup:** `resolveAnnounceTarget` returns the parsed fallback immediately when the channel plugin lacks `meta.preferSessionLookupForAnnounceTarget`, so `sessions.list` deliveryContext is not consulted for Discord.

  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Session-key fallback does not include accountId:** `resolveAnnounceTargetFromKey` builds an `AnnounceTarget` with `channel`, `to`, and optional `threadId`; there is no `accountId` extraction from the session key path.
