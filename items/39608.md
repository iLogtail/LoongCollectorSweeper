## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`; the bundled Baileys 7.0.0-rc.9 source defaults omitted `shouldSyncHistoryMessage` to `() => !!syncFullHistory`, which makes the user-suggested root cause still plausible. The issue has recent non-maintainer confirmations on 2026.3.28 and 2026.4.20, and the older #14069 report is closed stale rather than a live canonical tracker.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin either passes an explicit Baileys history-sync predicate, exposes a deliberate config/product choice for linked-device history sync, or documents with maintainer approval that WhatsApp group inbound after relink is unsupported. The likely implementation point is `extensions/whatsapp/src/session.ts`, with a focused regression test in `extensions/whatsapp/src/session.test.ts` asserting the options passed to `makeWASocket`.
