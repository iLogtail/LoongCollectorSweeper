## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no `shouldSyncHistoryMessage` override, while the bundled Baileys 7.0.0-rc.9 package defaults that predicate to `() => !!syncFullHistory` when omitted. The issue has recent confirmations on 2026.3.28 and 2026.4.20 after the older #14069 report was closed stale, so this is not obsolete or implemented on main.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin either passes an explicit Baileys history-sync predicate, exposes a deliberate config/product choice for linked-device history sync, or documents with maintainer approval that WhatsApp group inbound after relink is unsupported. The likely code path is `extensions/whatsapp/src/session.ts`, with a focused regression test in `extensions/whatsapp/src/session.test.ts` asserting the options passed to `makeWASocket`.
