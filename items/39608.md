## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while Baileys 7.0.0-rc.9 derives the omitted predicate from `syncFullHistory`. The issue has recent, concrete linked-device group-inbound repro comments and the existing inbound monitor can handle group upserts once Baileys emits them, so this remains a plausible product bug rather than a stale or config-only report.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin deliberately fixes or documents linked-device group inbound behavior. The likely maintainer path is to update `extensions/whatsapp/src/session.ts` to pass a reviewed `shouldSyncHistoryMessage` predicate or expose a product-backed config choice, add a focused regression assertion in `extensions/whatsapp/src/session.test.ts` for the `makeWASocket` options, and run a live linked-device group smoke before closing. If maintainers decide this linked-device group mode is unsupported, document that clearly in the WhatsApp channel docs instead of closing as stale.
