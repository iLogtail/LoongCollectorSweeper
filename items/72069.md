## Summary

Keep PR #72069 open. Current main still lacks the Claude CLI fallback-context seed this PR is trying to add: fallback retries only use OpenClaw's own session-history flag, while Claude CLI JSONL import is limited to chat history display/backfill. The PR is recent, contributor-authored, and the discussion shows active follow-up addressing review findings, so there is no conservative cleanup close reason.

## Best Possible Solution

Keep #72069 open for maintainer review. The likely best code path is the one the PR is pursuing: when a fallback chain originally started on claude-cli and retries on a non-Claude-CLI candidate, read the bound Claude CLI JSONL, build a bounded summary/recent-turn prelude, prepend it to the retry prompt, gate it by original provider to avoid stale binding leakage, and cover the behavior in src/agents/command and src/gateway tests.
