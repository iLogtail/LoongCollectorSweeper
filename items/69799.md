## Summary

Keep open. Current main mitigates parts of the macOS TCC problem with OpenClaw.app owning many permission-facing workflows, but it still does not ship or bundle a dedicated Node runtime for the Gateway/CLI/node service path. The docs explicitly say OpenClaw.app no longer bundles Node/Bun or the Gateway runtime, and the LaunchAgent/install code continues to resolve and run an external user/system Node path.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. The likely path is either to make the signed macOS app own all TCC-relevant macOS tool execution and document the remaining CLI/headless limits, or to add a verified dedicated Node runtime in the installer/update flow with signing/checksum/notarization expectations.
