## Summary

Keep open. Current main has useful macOS app and installer work, but it does not yet provide a bundled/signed OpenClaw-owned Node runtime for the gateway/npm workflow described in #69799. The current LaunchAgent path still resolves and runs an external/current Node path, and the docs explicitly say OpenClaw.app no longer bundles Node/Bun or the gateway runtime.

## Best Possible Solution

Keep this open as macOS security/product work. The best path is to choose and document a supported OpenClaw-owned runtime story for TCC-sensitive flows: either a signed OpenClaw.app-owned gateway/runtime path, or a clearly dedicated installed Node runtime that launchd, onboarding, doctor, iMessage, and automation docs consistently use and verify with signed/checksummed updates.
