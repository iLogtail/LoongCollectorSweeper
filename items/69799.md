## Summary

Keep open. Current main has partial macOS TCC mitigation through the native app and PeekabooBridge, but it has not shipped the requested bundled/dedicated Node runtime for the Gateway/npm workflow. The macOS app docs explicitly say OpenClaw.app no longer bundles Node/Bun or the Gateway runtime, local mode still installs/uses an external CLI, daemon install still resolves the current or system Node path, and the legacy iMessage docs still require Full Disk Access for the process context running OpenClaw/imsg. The related #7227 remains open with a security label, so this should stay in maintainer hands rather than be auto-closed.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is to either implement a signed app-bundled or OpenClaw-managed dedicated Node/Gateway runtime for LaunchAgent and TCC-sensitive macOS workflows, or explicitly route all TCC-gated macOS capabilities through the signed OpenClaw.app/bridge and document the residual npm/global-node risk. If maintainers want one canonical tracker, consolidate this with open security issue #7227 rather than closing it as already fixed.
