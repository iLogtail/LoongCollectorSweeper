## Summary

Keep open. Current main still does not have the requested stable `openclaw/plugin-sdk/embeddings` subpath, package export, entrypoint registration, or docs entry. This is a narrow plugin-SDK surface proposal, not a ClawHub-only optional plugin, and no provided related item supersedes it.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase: add `src/plugin-sdk/embeddings.ts` as a narrow re-export of `src/memory-host-sdk/engine-embeddings.ts`, register it in `package.json` and `scripts/lib/plugin-sdk-entrypoints.json`, update current plugin SDK docs (`docs/plugins/sdk-subpaths.md` and `docs/plugins/sdk-migration.md`), migrate bundled embedding-provider imports as appropriate, then run the plugin-SDK export/API checks and build in a normal dev environment.
