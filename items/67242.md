## Summary

Current main does not yet provide the requested stable `openclaw/plugin-sdk/embeddings` subpath. The PR is still useful because it addresses a real core plugin-SDK boundary gap; this is not a ClawHub-only item since the missing piece is a package export, SDK entrypoint registration, and public docs surface.

## Best Possible Solution

Keep the PR open. The best path is to land a narrow additive core SDK export for embeddings, with `package.json`, `scripts/lib/plugin-sdk-entrypoints.json`, SDK docs, and bundled plugin imports aligned on current main; this is a plugin API extension, not an external plugin implementation to move to ClawHub.
