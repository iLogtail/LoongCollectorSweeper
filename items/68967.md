## Summary

Keep open. Current main already has Google Chat threaded outbound placement support, but it still routes inbound Google Chat sessions by space and has no `sessionThread`/`threadSessionMode` config, schema, docs, or regression test. PR #68967 still contains a concrete remaining implementation for the per-thread session isolation requested by #13373, and the provided review thread shows the author addressed several substantive routing/session-key concerns rather than leaving a low-signal or obsolete PR.

## Best Possible Solution

Keep the PR open for maintainer review or ask for final polish, because the remaining per-thread Google Chat session isolation is not implemented on current main. The best path is to land a narrow Google Chat-owned implementation that preserves space-based route matching, derives a canonical thread-scoped session key without exposing case-sensitive Google thread names through generic `:thread:` parsing, passes the derived key into elapsed-metadata/session-meta paths, and updates config schema, generated metadata, docs, changelog, and targeted tests.
