## Summary

Keep #68967 open. Current main already has partial Google Chat thread reply placement support, but it still uses space-scoped Google Chat sessions and has no `sessionThread`/`threadSessionMode` config, schema, docs, or per-thread session-key helper. The PR remains a focused implementation for the still-missing per-thread session isolation requested by #13373, with substantive review discussion already addressed in the provided PR context.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow Google Chat-owned implementation that preserves space-based route matching, derives a stable thread-scoped session key without exposing case-sensitive Google thread names through generic `:thread:` parsing, passes the derived key through elapsed-metadata/session-meta paths, and updates config schema, generated metadata, docs, changelog, and focused Google Chat tests.
