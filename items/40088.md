## Summary

Keep open. Current `main` still lacks any watcher health/error recovery for builtin memory search, and `memorySearch.sync.intervalMinutes` still defaults to `0`, so the stale-index failure mode described in #40088 is not clearly fixed or obsolete.

## Evidence

- **Builtin watcher still has no health/error handling:** `ensureWatcher()` creates a chokidar watcher and only subscribes to `add`, `change`, and `unlink`. There is no `error`, `ready`, heartbeat, restart, or watchdog path to detect a silent stop and recover.
