## Summary

Current `main` already routes Slack native `/stop` to the active conversation session and aborts that run instead of treating the stop request as a separate prompt.

## Evidence

- **Slack docs describe target-session routing for native slash commands:** The Slack channel docs state that Slack slash sessions use isolated command-session keys while routing command execution to the target conversation via `CommandTargetSessionKey`.
  - file: [docs/channels/slack.md:665](https://github.com/openclaw/openclaw/blob/bc0f54bd/docs/channels/slack.md#L665)
  - command: `nl -ba docs/channels/slack.md | sed -n '650,690p'`
  - sha: [bc0f54bd](https://github.com/openclaw/openclaw/commit/bc0f54bd)
- **Slack slash handler attaches the target session key:** The Slack native slash-command path resolves native command session targets and writes `CommandTargetSessionKey` into the inbound context payload for dispatch.
  - file: [extensions/slack/src/monitor/slash.ts:602](https://github.com/openclaw/openclaw/blob/bc0f54bd/extensions/slack/src/monitor/slash.ts#L602)
  - command: `nl -ba extensions/slack/src/monitor/slash.ts | sed -n '560,625p'`
  - sha: [bc0f54bd](https://github.com/openclaw/openclaw/commit/bc0f54bd)
- **`/stop` aborts the target session instead of the wrapper command session:** `handleStopCommand` resolves the abort target from `ctx.CommandTargetSessionKey`, clears queued followups for that target, aborts the run, and returns the stop reply.
  - file: [src/auto-reply/reply/commands-session-abort.ts:40](https://github.com/openclaw/openclaw/blob/bc0f54bd/src/auto-reply/reply/commands-session-abort.ts#L40)
  - command: `nl -ba src/auto-reply/reply/commands-session-abort.ts | sed -n '34,173p'`
  - sha: [bc0f54bd](https://github.com/openclaw/openclaw/commit/bc0f54bd)
- **End-to-end test covers native stop targeting the active session:** The regression test creates a target session with queued followups, sends a native `/stop` with `CommandTargetSessionKey`, and asserts that the target run is aborted and the queued followups are cleared.
  - file: [src/auto-reply/reply.triggers.trigger-handling.targets-active-session-native-stop.e2e.test.ts:575](https://github.com/openclaw/openclaw/blob/bc0f54bd/src/auto-reply/reply.triggers.trigger-handling.targets-active-session-native-stop.e2e.test.ts#L575)

## Risks / Open Questions

- I verified that current `main` aborts the active target run instead of opening a new prompt, but I did not verify that exec-backed terminal subprocesses receive a literal `SIGINT`/Ctrl+C rather than the runtime's broader cancel path.
- The implementation is present on current `main` at `bc0f54bd`; I could not tie it to a tagged release newer than `v2026.4.23` from the local checkout.
