## Summary

Keep #12931 open. Current main still runs bare `/new` and `/reset` through a model-backed Session Startup greeting, the only reset model override path is still tied to trailing reset text, and the session config/schema/docs do not expose `resetGreeting`, `resetGreetingModel`, silent reset, or gateway-generated acknowledgement behavior. The prior Codex review comment in the issue remains accurate against current main.

## Best Possible Solution

Keep this open for a core session/reset behavior change. A complete fix should add typed and schema-validated config for bare reset greeting behavior, wire both chat.send and gateway agent reset paths, support both a configured cheaper greeting model and a no-model silent/static-ack path, document the options in the public session/slash-command/config docs, and add regression tests for bare `/new`, bare `/reset`, `/new <model>`, startup-context interactions, and existing model override persistence.
