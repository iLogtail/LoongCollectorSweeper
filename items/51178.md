## Summary

Keep #51178 open. Current main still uses an in-process lane queue with per-session serialization and a global concurrency lane; it has some non-blocking/session-tool options, but not the requested durable/event-driven agent message bus or broader decoupling. The related lane-timeout work in #48488/#48690 is also still open, so this should not be closed as implemented or superseded.
