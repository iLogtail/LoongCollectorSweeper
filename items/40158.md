## Summary

Keep open. Current `main` still requires `commands.allowFrom` to be a provider-keyed record, so the legacy array compatibility in PR #40158 is not implemented on `main` and the underlying bug from #39500 remains plausible.

## Evidence

- **Current session schema still uses record-only allowFrom:** `CommandsSchema` still wires `commands.allowFrom` directly to `ElevatedAllowFromSchema.optional()` with no preprocess/normalization step for a legacy array form.
  - file: [src/config/zod-schema.session.ts:223](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/config/zod-schema.session.ts#L223)

  - file: [src/config/zod-schema.agent-runtime.ts:434](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/config/zod-schema.agent-runtime.ts#L434)
