## Summary

Keep open. Current main still ignores top-level `channels.discord.allowFrom` in the Discord owner-auth adapter, while adjacent Discord DM routing, docs, and tests treat the top-level field as valid. The linked fix attempts were closed unmerged, and the observable bug path still exists on current main.

## Best Possible Solution

Keep this issue open. The best fix is a focused Discord plugin change in `extensions/discord/src/shared.ts` so owner auth resolves `account.config.allowFrom ?? account.config.dm?.allowFrom`, with a regression test that top-level `channels.discord.allowFrom` produces a non-empty owner list and `senderIsOwner=true` for a matching Discord DM sender. Maintainers should also decide whether `extensions/discord/src/security.ts` should be aligned to the same top-or-nested allowFrom contract if that resolver remains active.
