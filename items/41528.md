## Summary

Keep open. Current `main` only surfaces `update.run` errors when the response has `ok === false`, but the gateway still returns `ok: true` for `status: "skipped"`, so the Control UI can still appear non-functional for skipped/no-op update paths. This PR's core behavior is not fully implemented on `main`.

## Evidence

- **Control UI only handles `ok === false`:** `runUpdate` sets `lastError` only inside `if (res && res.ok === false)`, so payloads with `status: "skipped"` and `ok: true` produce no feedback.
  - file: [ui/src/ui/controllers/config.ts:191](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/controllers/config.ts#L191)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '178,215p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Gateway marks skipped updates as ok:** `update.run` responds with `ok: result.status !== "error"`. For `status: "skipped"`, that evaluates to `true`, so the UI's current guard does not fire.
  - file: [src/gateway/server-methods/update.ts:132](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/gateway/server-methods/update.ts#L132)
  - command: `nl -ba src/gateway/server-methods/update.ts | sed -n '129,138p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Current UI test coverage misses skipped case:** Current `main` has a test for an error payload (`status: "error"`) but no test asserting skipped-result feedback, matching the missing behavior in `runUpdate`.
  - file: [ui/src/ui/controllers/config.test.ts:436](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/ui/src/ui/controllers/config.test.ts#L436)
