## Summary

Keep PR #68669 open. Current main still has the duplicated browser cleanup wrapper dispatch path described by the PR: both the lifecycle listener and gateway wait path call `completeSubagentRun(..., triggerCleanup: true)`, and `completeSubagentRun` still invokes `cleanupBrowserSessionsForLifecycleEnd` without a per-run dispatch flag. The PR is a focused, recent bug-fix PR with maintainer-positive discussion, not an obsolete, duplicate, ClawHub, or low-signal cleanup item.

## Best Possible Solution

Keep PR #68669 open for normal maintainer review/landing. The best code path is the PR's narrow controller-level guard: add a per-run browser cleanup dispatch flag, set it synchronously before the wrapper call, reset it on steer-restart rearm, and keep the regression test that proves two parallel `completeSubagentRun(... triggerCleanup: true)` callers invoke `cleanupBrowserSessionsForLifecycleEnd` once.
