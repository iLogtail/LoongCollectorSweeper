## Summary

Keep open. Current main still has the reported duplicate-dispatch path: both embedded completion routes can call `completeSubagentRun(..., triggerCleanup: true)` for the same run, and the browser cleanup wrapper is still invoked without a per-run dispatch guard. The PR is a focused, recent bug fix with a maintainer comment saying it is correct and worth landing, so this is not a conservative cleanup close candidate.

## Best Possible Solution

Keep #68669 open and land a narrow implementation like the proposed PR: add a per-run browser cleanup dispatch flag, reset it on steer-restart rearm, and keep the regression test that proves parallel `completeSubagentRun` callers invoke the wrapper once.
