## Summary

Keep open. Current `main` does not show a maintainer-safe fix for container auto-update regressions: docs say container installs should be updated by rebuilding/pulling the image, but the gateway still auto-applies package updates when `update.auto.enabled` is on. The exact `ENOENT: mkdir '/app'` may be specific to the reporter's third-party `coollabsio/openclaw` image, but there is not strong evidence that the underlying report is implemented, obsolete, or disproven on `main`.
