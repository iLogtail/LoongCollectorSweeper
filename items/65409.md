## Summary

Keep #65409 open. Current main still leaves the CLI runner's `systemPromptReport.tools` populated from an explicit empty tool list, so the PR's central fix is not implemented on main. The PR also contains follow-up fixes for review findings around report-runtime lifetime, merged MCP config, backend env placeholders, and plugin activation state; those are still relevant because main has no report MCP materialization path in the CLI runner.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to refresh it against current main. The product direction should be to make the CLI runner materialize the effective bundle MCP tool catalog for reporting, dispose that report runtime before the real CLI run, use the same merged MCP config/env the backend receives, preserve plugin activation config, and keep regression coverage asserting non-empty report tool entries and schema chars for bundle MCP CLI runs.
