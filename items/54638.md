## Summary

Keep open. Current main has partial support for mid-run steering (`messages.queue.mode=steer` and the explicit `sessions.steer` RPC), but ordinary inbound messages still default to `collect`, which queues them until the active turn ends. That means the reported Telegram behavior is not fully implemented away on current `main`; it is only avoidable via opt-in steering paths.
