## Summary

Keep open. Current main still has the exact-dedup-only Feishu streaming fix from #68491, but it does not handle #71977's reported case where distinct final segments arrive after the first streaming card has been closed. The dispatcher clears the active streaming session on idle, skips only exact duplicate final text, and starts a new streaming card for any later distinct final in card mode.

## Best Possible Solution

Keep #71977 open and fix it in the Feishu plugin reply dispatcher. The likely maintainer path is to make one logical assistant turn map to one Feishu streaming card: keep the card/session open until all final segments for the turn are settled, patch the existing card for later distinct finals, or suppress post-close finals when the streaming card already contains the completed response. Add a regression test for a tool-heavy Feishu card-streaming turn with partial streaming followed by multiple distinct final payloads after idle/close.
