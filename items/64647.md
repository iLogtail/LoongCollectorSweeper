## Summary

Keep open. Current main still has only root-level `memory` plus per-agent `memorySearch`; it does not accept or resolve `agents.list[].memory`. The PR is a focused implementation for the still-open linked feature request #64612, and the discussion says the supported boundary matches the reporter's rollout need. It may need rebase/type fixes, but it should not be conservatively closed.

## Best Possible Solution

Keep the PR open and have maintainers review or refresh it against current main. The best product path is to add `agents.list[].memory` as a narrow config/schema/runtime overlay for `backend` and `qmd`, keep `memory.citations` global, update generated schema/docs, and land the regression/runtime coverage already proposed by the PR.
