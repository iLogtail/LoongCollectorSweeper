## Summary

Keep #15032 open. Current main still has only global/config-derived sub-agent tool policy; `sessions_spawn` does not accept, persist, or enforce per-spawn `tools` allow/deny restrictions. The linked Claw Drive use case and PR #23506 are directly relevant, but #23506 is closed/unmerged in the provided context, and #23906 is an open related request rather than a shipped replacement.

## Best Possible Solution

Keep #15032 open as the canonical core API/security gap. Implement an additive `sessions_spawn` tool-policy parameter, likely `tools: { allow?: string[]; deny?: string[] }` and possibly `tools: "none"` to cover #23906. Thread it through the tool schema, `SpawnSubagentParams`, immutable child-session metadata, and `resolveSubagentToolPolicyForSession`; enforce deny-wins and narrow-only semantics; add focused unit/e2e coverage; update https://docs.openclaw.ai/tools/subagents.
