## Summary

Keep open. Current main has useful adjacent recovery and diagnostics work, but it does not implement the requested persistent gateway watchdog, user-facing outage notifications, backup-channel alerting, or first-class multi-instance health monitoring. The directly related PR #53716 is still open/unmerged in the provided context, and its proposed watchdog/startup-error files are absent from this checkout.

## Best Possible Solution

Keep #53684 open as the product tracker unless maintainers explicitly make #53716 or a narrower issue canonical. The best implementation path is to design a persistent, out-of-process watchdog using existing `/healthz`/`/readyz`, gateway service abstractions, and diagnostic stability bundles, then add tested notification paths for local macOS/Linux/Windows alerts and optional backup-channel or multi-gateway monitoring. PR #53716 can be reviewed as a starting point, but it needs rebasing to current main, maintainer product review, tests, and docs before the issue should close.
