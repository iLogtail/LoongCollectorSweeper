## Summary

Keep this issue open. The report is still valid on current main: memory-wiki compile discovers only direct `.md` files under each page-group directory, so nested files like `sources/topic/AGENTS.md` are not included in page summaries, indexes, related blocks, dashboards, or digests. Existing bridge and unsafe-local import paths can ingest nested external artifacts, but they flatten them into generated `sources/*.md` pages and do not satisfy preserving a user-organized wiki vault subdirectory structure.

## Best Possible Solution

Keep this open and implement recursive wiki page discovery inside the bundled `memory-wiki` plugin. A good fix should share one deterministic Markdown discovery helper across compile/query/status as appropriate, preserve nested relative paths in indexes/digests/links, decide and document stable ID behavior for nested pages without frontmatter, and add regression tests for nested `sources/` pages with Unicode/spaced directories plus search/get and digest coverage.

- A compile-only change would likely be incomplete because query/status surfaces have similar direct-child scans.
- Recursive discovery needs deterministic sorting, index-page skip rules for nested `index.md`, safe path normalization, and a clear policy for page IDs when nested Markdown lacks `id` frontmatter.
