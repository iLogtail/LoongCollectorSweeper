## Summary

Keep PR #6457 open. Current main has shipped part of the thread-routing work for Telegram native command auth/diagnostic replies, but the PR's central group-scope command registration/cleanup is still missing, and the plugin-command "Command not found." fallback still sends without topic thread params.

## Best Possible Solution

Keep #6457 open, or replace it with a focused successor, until the Telegram plugin registers native command menus for both default and `all_group_chats` scopes, clears both scopes when native commands are explicitly disabled, threads the plugin-command "Command not found." fallback through the existing Telegram thread helpers, and adds focused tests for dual registration, dual cleanup, retry/cache behavior, and threaded fallback replies.
