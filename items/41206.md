## Summary

Keep open. Current `main` still implements cron persistence as a single `~/.openclaw/cron/jobs.json` file plus `jobs-state.json`; the proposed per-job `cron/jobs/*.json` directory store is not present on `main`, so this PR is neither implemented nor obsolete enough to close conservatively.
