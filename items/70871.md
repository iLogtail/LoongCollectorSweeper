## Summary

Keep open. Current `main` still gives `before_prompt_build` plugins the latest prompt plus the full prepared `messages` array, and the public docs describe that contract directly. The only bounded-query and configurable-timeout behavior I found lives inside the bundled `active-memory` plugin itself, not in a generic core memory-plugin seam. That means the reported failure mode remains plausible for third-party memory plugins like `openclaw-mem0`, so this is not safe to auto-close as fixed or obsolete.
