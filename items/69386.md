## Summary

Keep open. Current main has partial mitigations for stop/queue handling and denial wording, but the central report is not fully fixed: denied async exec approvals still dispatch an agent follow-up for ordinary sessions, and regression coverage asserts that behavior. The linked fix PR #69387 was closed unmerged, and the related async-cascade issue #69570 remains open for overlapping remaining behavior.

## Best Possible Solution

Keep #69386 open, but narrow it to the residual core behavior: denied async exec approval completions for normal sessions should not recursively start new agent turns or generate variant exec attempts. The likely fix belongs around `src/agents/bash-tools.exec-approval-followup.ts` and the gateway-host/node exec denial paths, with regression coverage for repeated denials, stop/abort priority, and pending async completion cancellation or suppression. #69570 can track the broader approved-completion cascade, but this issue still carries the denial-specific backoff and stop-recognition requirements.
