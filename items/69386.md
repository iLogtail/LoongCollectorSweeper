## Summary

Keep open. Current main has useful stop/abort plumbing, but it does not fully cover this report: Control UI only intercepts a narrow set of exact stop words before queueing busy-chat input, denied exec approvals still trigger agent follow-up turns for normal sessions, and `runTaskInFlow` has no consecutive-denial guard. The directly linked PR #69387 and related issue #69570 are both open, so the remaining work is not shipped or cleanly superseded.

## Best Possible Solution

Keep #69386 open and use it as the denial-specific bug. A maintainer-quality fix should combine the shipped abort plumbing with: Control UI reuse of the shared abort phrase matcher, priority handling that bypasses the busy-chat queue for abort text, suppression or non-agent delivery for denied exec completions, a bounded consecutive-denial guard before spawning more flow tasks, and regression tests covering denied exec followups plus queued webchat stop phrases. #69570 can remain the broader approved-completion cascade tracker, and #69387 should not close this issue unless it is updated, validated, and merged.
