## Summary

Keep open. Current `main` does not implement declarative skill trigger metadata, scheduler registration from `SKILL.md`, last-trigger reporting, or a `skills audit --never-triggered` surface.

## Evidence

- **Skills docs enumerate current frontmatter and gating fields only:** The skills docs say the parser supports single-line frontmatter keys, describe `metadata` as single-line JSON, and list current supported fields under `metadata.openclaw` as `always`, `emoji`, `homepage`, `os`, `requires`, `primaryEnv`, and `install`. There is no documented `triggers`, `trigger_description`, scheduler integration, or audit command.
  - file: [docs/tools/skills.md:155](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/tools/skills.md#L155)
  - command: `nl -ba docs/tools/skills.md | sed -n '120,230p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Skill metadata types do not model triggers or last-run state:** `OpenClawSkillMetadata` only contains eligibility/install fields, and `ParsedSkillFrontmatter` is still `Record<string, string>`. There is no type for schedule/event triggers, trigger descriptions, or last-trigger timestamps.
  - file: [src/agents/skills/types.ts:19](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/skills/types.ts#L19)

- **Skill status report has no trigger-related output fields:** `SkillStatusEntry` contains readiness, requirements, source, homepage, and install data only. No trigger metadata, audit state, or last-trigger fields are produced for the CLI/UI layer.
  - file: [src/agents/skills-status.ts:31](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/skills-status.ts#L31)

There is no `audit` command, and the list/info rendering only shows readiness/source/requirements, not trigger definitions or 'last triggered'.

## Risks / Open Questions

- The request is still a plausible core feature request: current `main` has heartbeat and cron systems, but no visible bridge from skill frontmatter into those systems.
- The proposed YAML shape may need design work against the current documented constraint that skill frontmatter keys are single-line and `metadata` is carried as single-line JSON.
