## Summary

Keep open. Current main has implemented the local CLI login -> gateway `channels.start` handoff, so one part of the original dead-end improved, but the WhatsApp monitor logged-out close path still stops and breaks without invoking the existing credential-cleanup/logout lifecycle. The later issue comment’s cleanup/restart-loop concern is still supported by current source and is not clearly covered by the open #67816 timeout/status PR.

## Best Possible Solution

Keep #49305 open until the WhatsApp monitor logged-out path is wired to the existing lifecycle cleanup: on `healthState === "logged-out"`, clear auth via `logoutWeb()` or the gateway logout lifecycle with the credential barrier, mark runtime/status as logged out or not configured, avoid auto-restarting with stale creds, correct the stale `--channel web` relink hint, and add a monitor regression test for 401/logged-out cleanup plus truthful status/outbound behavior. Preserve the already-shipped `channels.start` handoff after successful local login.
