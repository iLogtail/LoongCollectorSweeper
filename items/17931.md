## Summary

Keep #17931 open. Current main still does not implement automatic read-only per-skill bind mounts. The related #22669 fix makes overlapping custom binds work as a manual workaround, but Docker container creation still derives only workspace/agent mounts plus configured custom binds, and `workspaceAccess: "rw"` still exposes workspace-local skills under the writable `/workspace` mount. Non-rw modes are still copy/mirror based rather than direct read-only source mounts.

## Best Possible Solution

Keep #17931 open until OpenClaw either implements automatic read-only handling for eligible skill directories in sandboxed sessions or explicitly rejects that direction. A robust implementation should derive eligible skill source directories, add deterministic read-only Docker overlays after broader workspace mounts for `rw`, avoid writable or stale copied skill state where source mounts are available, update sandbox config hashing/fs-bridge resolution/docs, and add tests covering `rw`, `ro`, and `none`. Treat #22669 as a useful manual workaround only.
