## Summary

Keep #17931 open. Current main still stages sandbox skills as writable workspace contents in non-rw modes and exposes workspace skills through the writable agent workspace in rw mode. The closed #22669 bind-order fix provides a manual overlapping-bind workaround, but there is still no automatic per-skill read-only bind-mount behavior.

## Best Possible Solution

Keep the issue open until OpenClaw either implements automatic read-only handling for eligible skill directories in sandboxed sessions or explicitly rejects that product/security direction. A robust fix should derive deterministic eligible skill source directories, add read-only overlays after broader workspace mounts for rw, avoid writable copied skill state where source mounts are available for ro/none, update sandbox fs-bridge/config-hash/docs, and add tests for all three workspaceAccess modes while treating #22669 as only a manual workaround.
