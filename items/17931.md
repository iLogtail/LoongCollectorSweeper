## Summary

Keep #17931 open. Current main still does not implement automatic read-only per-skill bind mounts: non-rw sandbox modes still copy eligible skills into the sandbox workspace, and rw mode still exposes workspace skills through the writable workspace mount. The closed #22669 bind-ordering fix provides a manual workaround with custom binds, but not the requested default core behavior.

## Best Possible Solution

Keep #17931 open until OpenClaw either implements automatic read-only handling for eligible skill directories in sandboxed sessions or explicitly rejects that sandbox-hardening direction. A robust fix should derive eligible SkillEntry source directories, add deterministic read-only overlays after workspace mounts for rw mode, avoid writable copied skills where a backend can mount source directories directly, update sandbox config hashing/recreate behavior and fs-bridge mount resolution, and cover the behavior in tests/docs. Treat #22669 as a useful manual workaround, not the canonical fix.
