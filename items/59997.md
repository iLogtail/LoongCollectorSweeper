## Summary

Keep PR #59997 open. Current main still only classifies numeric `sessions.json.bak.<digits>` store backups, while disk-budget cleanup depends on that classifier before evicting session entries. The PR's broader `sessions.json.bak.*` classifier and regression cases are not present on main.

## Best Possible Solution

Keep PR #59997 open or land an equivalent narrow fix: classify any non-empty `sessions.json.bak.*` store backup suffix as a session archive artifact, keep the focused regression tests for the observed backup names, and consider adding one disk-budget test proving non-numeric store backups are removed before live session entries are evicted.
