## Summary

Keep open. Current main still only classifies numeric `sessions.json.bak.<digits>` store backups, and the PR's broader `sessions.json.bak.*` handling plus regression cases are not present. Disk-budget cleanup depends on this classifier before it falls back to evicting session entries, so the reported non-numeric backup artifacts remain plausible current behavior rather than obsolete work.

## Best Possible Solution

Keep PR #59997 open or land an equivalent small fix: classify any non-empty `sessions.json.bak.*` store backup suffix as a session archive artifact, keep the focused regression tests for the observed backup names, and optionally add one disk-budget test proving non-numeric store backups are removed before live session entries are evicted.
