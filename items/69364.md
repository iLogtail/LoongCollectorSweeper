## Summary

Keep #69364 open. Current main has useful Control UI primitives for agents, sessions, per-agent file editing, tool access, and skill toggles, but it does not implement the requested cross-agent team relationship graph or debugging/configuration view. The closest related item, #52803, overlaps on multi-agent orchestration UI but does not clearly supersede this issue's delegate/dataflow/sequence graph and side-by-side team config workflow. This is also not a strong ClawHub close because current plugin surfaces do not expose a supported way to add a Control UI tab or panel.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. The best path is a scoped Control UI team relationships design that reuses existing agents, sessions, config, task, and Gateway event data where possible; adds a narrow relationship/graph RPC only if current contracts are insufficient; and explicitly coordinates with #52803 so the broader orchestration board and this delegate/dataflow/sequence graph do not diverge. If maintainers want this outside core, first define a supported Control UI plugin extension point.
