## Summary

Keep #69364 open. Current main has useful Control UI primitives for agents, sessions, per-agent files, tools, and skills, but it does not implement the requested cross-agent relationship view or team graph. This is not a strong ClawHub close either: the plugin SDK exposes capabilities, tools, hooks, routes, RPC methods, and CLI registration, but not a supported way to add a first-party Control UI tab/panel. The closest related open tracker, #52803, covers multi-agent orchestration hierarchy and bulk controls, but it does not clearly cover #69364's delegate/dataflow/sequence graph, symlink/TASK.json inspection, or side-by-side per-agent config workflow.

## Best Possible Solution

Keep open for maintainer product/design judgment. The best path is a scoped Control UI team/relationships design that reuses existing sessions, agents, config, task, and gateway event data where possible, adds a narrow relationship/graph RPC only if the current contracts are insufficient, and either links or deliberately subsumes #52803. If maintainers want this outside core, first define a supported Control UI plugin extension point; current plugin surfaces alone do not make this a clean ClawHub close.
