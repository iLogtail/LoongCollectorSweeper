## Summary

Keep open. Current `main` at `b67d9bf7f06f4c655b1e0869395056c7d631abc2` still uses fixed Ctrl+C status strings with no active-run hint, while abort remains a separate `/abort`/Escape path. The PR is still a narrow TUI UX clarification, and its current patch addresses the earlier review concern by advertising `/abort` only instead of promising unconditional Escape behavior.

## Best Possible Solution

Keep this PR open or land an equivalent narrow TUI change on main: append an active-run hint to the Ctrl+C clear/warn statuses when `activeChatRunId` is set, prefer `/abort` as the universally correct stop instruction, and add focused coverage if the status surface is testable.
