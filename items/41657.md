## Summary

Keep open. Current `main` already ingests Slack attachments as downloaded media plus filename placeholders, so the issue body overstates the bug, but the requested attachment metadata delivery is still not implemented.

## Evidence

- **Slack inbound reads `message.files` and turns them into media plus placeholders:** `resolveSlackMessageContent()` reads `params.message.files`, downloads them via `resolveSlackMedia()`, and only appends placeholder text like `[Slack file: <name>]` into `rawBody`.
  - file: [extensions/slack/src/monitor/message-handler/prepare-content.ts:75](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/slack/src/monitor/message-handler/prepare-content.ts#L75)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare-content.ts | sed -n '70,190p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Downloaded Slack file result has no metadata fields beyond path/content type/placeholder:** `SlackMediaResult` only contains `path`, optional `contentType`, and `placeholder`; file id, original file name, MIME from Slack, and size are not preserved in the returned structure.
  - file: [extensions/slack/src/monitor/media.ts:168](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/slack/src/monitor/media.ts#L168)

  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)

  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:662](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/slack/src/monitor/message-handler/prepare.ts#L662)

## Risks / Open Questions

- The original report text is partly obsolete because Slack attachments are not fully stripped anymore; closing as `cannot_reproduce` would discard the still-valid metadata gap in the title and expected behavior.
- A future cross-channel attachment schema could supersede this exact Slack-specific request, but current `main` does not provide that schema yet.
