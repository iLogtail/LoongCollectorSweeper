## Summary

Keep #41657 open. Current main does ingest Slack files as downloadable media plus filename/fileId placeholders, but it still does not expose the requested structured attachment metadata block with Slack file id/name/MIME/size to agents. The related open PR #62792 appears scoped to media hydration reliability, not the broader structured inbound attachment model discussed on the issue.
