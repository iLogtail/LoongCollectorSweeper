## Summary

Keep #50876 open. Current main still lacks an `agents.defaults.bootstrapFiles` or per-agent equivalent, and the bundled no-code `bootstrap-extra-files` hook only accepts recognized bootstrap basenames, not arbitrary files like `RULES.md` or `CONVENTIONS.md`. The existing `agent:bootstrap` hook surface can support a plugin/hook implementation, but that does not fully satisfy the requested config UX.

## Best Possible Solution

Keep the issue open for maintainer product/API judgment. The narrow implementation path would be either a schema-backed `agents.defaults.bootstrapFiles` and optional per-agent override that feeds the existing bootstrap budget/truncation path, or an explicit maintainer decision to publish a ClawHub/internal hook that reads arbitrary workspace-relative files such as `RULES.md` and appends them through `agent:bootstrap`. The current built-in hook should not be treated as a complete replacement unless it is intentionally broadened beyond recognized bootstrap basenames.
