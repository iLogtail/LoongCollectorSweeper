## Summary

Keep #52997 open. Current main still has intentional Telegram forum-topic and DM-thread session isolation, while preserving topic/thread metadata for replies. I found no `sessionMode: "shared"` or equivalent Telegram config in types/schema/docs, and the related topic-name issues track a different display-label problem rather than this shared-session routing request.

## Best Possible Solution

Keep the issue open and implement this in the bundled Telegram plugin as a typed, schema-backed opt-in, defaulting to current isolated behavior. A maintainer-quality fix should add a group/direct setting such as `sessionMode: "isolated" | "shared"`, route shared-mode forum topics and DM threads to the parent/base session, preserve `MessageThreadId`/`IsForum`/`TopicName` and reply delivery thread metadata, avoid breaking per-topic `agentId` and ACP/thread bindings, and update Telegram docs plus regression tests for forum topics, DM topics, replies, native commands, and session metadata.
