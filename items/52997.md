## Summary

Keep open. Current main still intentionally isolates Telegram forum topics and DM threads into thread/topic-qualified sessions, while preserving topic/thread metadata for replies; there is no Telegram `sessionMode: "shared"` or equivalent config to collapse all topics/threads into one shared main session.

## Evidence

- **Telegram config has per-topic controls but no shared session mode:** `TelegramGroupConfig` and `TelegramDirectConfig` expose `topics` maps and topic settings such as `agentId`, `requireMention`, and `systemPrompt`, but no `sessionMode`/shared-session option for groups or DMs.
  - file: [src/config/types.telegram.ts:223](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/config/types.telegram.ts#L223)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '220,330p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Forum topics are routed as topic-qualified group sessions:** Inbound Telegram group routing builds the peer id with `buildTelegramGroupPeerId(chatId, resolvedThreadId)`, so forum topics become distinct peers before the agent route is resolved.
  - file: [extensions/telegram/src/conversation-route.ts:37](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/extensions/telegram/src/conversation-route.ts#L37)
