## Summary

Keep open. Current `main` still waits on and suppresses delivery based on all active descendant runs under the requester session key, without scoping those waits to the current cron run window. The PR’s claimed fix is therefore not implemented on `main` today.

## Evidence

- **Delivery path still uses unscoped active-descendant counts:** `dispatchCronDelivery` reads `countActiveDescendantRuns(params.agentSessionKey)` before and after follow-up wait, and only passes `runStartedAt` to fallback-read calls, not to the wait/count path.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:685](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/cron/isolated-agent/delivery-dispatch.ts#L685)
  - command: `nl -ba src/cron/isolated-agent/delivery-dispatch.ts | sed -n '680,735p'`
  - sha: [81666e586aa6](https://github.com/openclaw/openclaw/commit/81666e586aa6d4305e6472d7075e62f63f701831)
- **Wait helper has no current-run-window filter:** `waitForDescendantSubagentSummary` does not accept `runStartedAt`; `getActiveRuns()` filters only on `endedAt`, so stale active descendants from prior isolated runs would still be awaited.
  - file: [src/cron/isolated-agent/subagent-followup.ts:75](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/cron/isolated-agent/subagent-followup.ts#L75)

- **Registry helpers are session-wide, not run-window scoped:** `countActiveDescendantRunsFromRuns` and `listDescendantRunsForRequesterFromRuns` traverse all descendant runs for the requester session and apply no timestamp/window filtering.
  - file: [src/agents/subagent-registry-queries.ts:248](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/subagent-registry-queries.ts#L248)
