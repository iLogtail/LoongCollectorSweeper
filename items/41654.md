## Summary

Keep open. Current `main` still contains explicit cron/subagent paths that deliver error payloads outward instead of suppressing them, so there is not strong evidence that #41654 is implemented or obsolete.

## Evidence

- **Cron helper still selects error payloads for delivery:** `resolveCronPayloadOutcome` returns successful payloads when present, but if all deliverable payloads are errors it falls back to the last error payload and also records `embeddedRunError` separately.

## Risks / Open Questions

- There are partial mitigations on current `main` for some adjacent cases, including swallowed cron failure-destination send errors and session-internal completion announces for active requesters, so the remaining leak may be narrower than the original report but is not eliminated.
- The exact raw Anthropic thinking-block rejection string may now be friendlier in some paths, but the code still intentionally delivers fatal error payloads outward for cron/subagent delivery flows.
