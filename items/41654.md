## Summary

Keep open. #41654 overlaps the closed #40579 thinking-block report, and current main has mitigations for Anthropic thinking replay, but the broader requested behavior is still not clearly implemented: external agent-run failures are intentionally formatted into visible channel replies, and the subagent announce direct path does not carry a contract that suppresses provider/API errors as internal-only delivery failures.

## Evidence

- **Linked related issue:** Provided GitHub context cross-references #40579, a closed issue describing the same cron announce / thinking-block rejection being sent to a user channel. That is useful prior discussion, but #41654 also asks for a broader guarantee that API-level announce/delivery errors are logged internally and not forwarded to Discord/Telegram/etc.
