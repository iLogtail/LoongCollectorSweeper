## Summary

Keep #8122 open. Current main still supports session reset policy at global, chat-type, and channel scope only. Telegram group/topic config accepts other per-group controls, but not `session.reset`; the strict schema rejects the proposed shape, and session initialization only resolves channel-level reset overrides before applying type/global policy. Related PR #8134 is closed unmerged, and #20888 does not supersede this per-Telegram-group request.

## Best Possible Solution

Keep the issue open. Implement an additive, account-aware reset override for Telegram group/topic config, or add a generic group/topic reset resolver to the channel plugin API if maintainers want the behavior across plugins. Update Telegram types, strict Zod schema, generated config metadata/help, public docs, and focused precedence tests for topic/group over channel over type/global, including multi-account cases.
