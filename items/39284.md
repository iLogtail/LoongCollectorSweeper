## Summary

Keep open. Current main still does not implement this PR's public `contextFiles` / `inheritContext` `sessions_spawn` API, and the remaining cross-agent workspace behavior is intentionally different from the PR's requester-workspace inheritance model. The submitted PR also has unresolved review blockers in the provided discussion.

## Best Possible Solution

Keep this PR open for maintainer/product judgment. The best path is either to implement a bounded `contextFiles` / `inheritContext` API that reads from the intended requester workspace with tests, or explicitly decide that current automatic target-workspace bootstrap is the supported behavior and document that instead.
