## Summary

Keep open. Current main has automatic bootstrap/context handling for spawned subagents and a `lightContext` option, but it still does not expose this PR's requested `contextFiles` / `inheritContext` `sessions_spawn` API. Current cross-agent workspace behavior also intentionally uses the target agent workspace, which conflicts with the PR's requester-workspace inheritance claim.

## Best Possible Solution

Keep this PR open for maintainer product judgment. If OpenClaw wants explicit file-selection APIs, rework it against current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles`, regression tests for same-agent and cross-agent behavior, docs, and removal of the unrelated logger change. If maintainers decide automatic bootstrap plus `lightContext` is the supported direction, close manually with that rationale.
