## Summary

Keep PR #39284 open. Current main has automatic workspace bootstrap injection and `sessions_spawn(lightContext)`, but it still does not expose or honor the PR's explicit `contextFiles` or `inheritContext` API. The related bootstrap work partially solves the same-agent context-loss problem, but it does not cover spawn-time file selection or the requester-workspace semantics promised by this PR.

## Best Possible Solution

Keep the PR open for maintainer product judgment. If explicit spawn-time file selection is desired, rework it on top of the current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles`, regression tests for same-agent and cross-agent spawns, docs updates, and removal of the unrelated plugin logger type change. If automatic bootstrap plus `lightContext` is the intended product direction, maintainers should close it manually with that rationale.
