## Summary

Keep #11055 open. Current main still has no `memoryFlush.visibleWarning` or `warningPrompt` config and no agent-visible pre-compaction warning path. The implemented nearby behavior is either a silent `memoryFlush` embedded run with `NO_REPLY` guidance or an opt-in `notifyUser` notice sent to the user channel, which does not satisfy the requested visible agent-facing warning before compaction.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or a maintainer rejects that product direction. A concrete implementation would add validated config for `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt`, inject a visible agent-facing warning at the soft threshold before auto-compaction, preserve the existing silent memory-flush run and separate `notifyUser` channel notices, and add config/runtime/docs tests for the distinction.
