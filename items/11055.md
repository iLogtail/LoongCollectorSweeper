## Summary

Keep #11055 open. Current main still has silent pre-compaction memory flush and optional user-channel compaction notices, but it does not implement the requested `memoryFlush.visibleWarning` / `warningPrompt` config or an agent-visible warning turn before compaction. The existing issue discussion also explicitly keeps it open for that same reason.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or a maintainer rejects that product direction. A concrete fix would add validated config for the warning toggle/prompt, inject a visible agent-facing turn before compaction at the soft threshold, preserve the existing silent `memoryFlush` and separate `notifyUser` user-channel notices, and cover the behavior in config/runtime tests plus compaction docs.
