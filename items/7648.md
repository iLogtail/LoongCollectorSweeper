## Summary

Keep #7648 open. Current main still has bottom-follow chat scrolling in the browser Control UI and native WebChat surfaces, no persisted UI setting or gateway config for off/top behavior, and only a downward “New messages” affordance rather than a Scroll to Top control. The prior issue comment also explicitly kept this open, and the only linked related issue is TUI-specific, not a replacement for this Control UI/WebChat request.

## Best Possible Solution

Keep #7648 open and implement an explicit chat-scroll preference for Control UI and native WebChat. A conservative path is to add a persisted UI setting and/or documented gateway.webchat/controlUi config, thread it through the browser scheduleChatScroll logic and shared Swift OpenClawChatView, support bottom-follow plus off/top behavior, optionally add a Scroll to Top control, and cover send, streaming, refresh, initial load, manual-scroll, and native WebChat states with focused tests.
