## Summary

Keep #24107 open. Current main has adjacent cooldown work, including shorter stepped cooldowns, model-scoped rate-limit cooldowns, same-provider transient probing, provider usage-window display, and OpenAI Codex WHAM-aware cooldowns, but it still does not implement the requested generic provider-level `rpm`/`rpd` config contract or pre-request quota-window accounting in model fallback. The existing maintainer discussion already reached the same keep-open conclusion, and current main still matches it.

## Best Possible Solution

Keep this issue open as a core model fallback/auth-profile enhancement unless maintainers choose a different product direction. A complete fix likely needs a validated provider quota config/API contract, persisted per-provider/profile quota-window counters and reset timestamps, pre-request skip/mark-cooldown logic in `runWithModelFallback`, integration with `cooldownUntil` and cooldown probing, generated schema/docs updates, and focused RPM/RPD tests for minute/day reset behavior.
