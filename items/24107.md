## Summary

Keep open. The current main branch improves generic cooldown behavior and has OpenAI Codex WHAM-aware cooldown probing, but it still does not implement the requested generic per-provider RPM/RPD quota contract or pre-request quota accounting in model fallback. The same-day issue discussion also already recorded a Codex/maintainer review keeping #24107 open for this reason, and current source inspection still agrees.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. A complete fix should add an explicit provider quota contract, likely in the models provider schema/types and generated config docs; track RPM/RPD usage before provider calls; set cooldownUntil to the exact minute/day reset window; make probing respect quota-derived windows; and add focused tests for RPM/RPD skip, fallback, expiry, and non-quota backoff behavior.
