## Summary

Keep open. Current main has several cooldown improvements, including OpenAI Codex WHAM-aware cooldown probing and shorter generic transient cooldowns, but it still does not implement the requested generic per-provider RPM/RPD quota contract or pre-request quota-window accounting for model fallback.

## Best Possible Solution

Keep tracking this as a core model fallback/auth-profile enhancement unless maintainers choose a different product direction. A complete fix likely needs an explicit config/API contract for provider quota windows, pre-request quota accounting in the fallback path, cooldownUntil integration, and focused tests for RPM/RPD skip and probe behavior.
