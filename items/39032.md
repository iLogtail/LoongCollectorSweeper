## Summary

Keep open. Current main still extracts `toolResult` / `tool` message content into the subagent completion fallback without checking failed-tool markers such as `isError` or `is_error`. That extracted raw text can then become the task-completion `result` and direct completion fallback content delivered to the requester. The direct fix discussed in #39367 remains closed unmerged, while the merged adjacent hardening in #59649 did not add this narrow guard.

## Best Possible Solution

Keep the issue open and implement the narrow extraction fix in `src/agents/subagent-announce-output.ts`: add a failed-tool-result guard for top-level `isError`, content/block-level `is_error`, and the existing tool-result status shape before populating `latestRawText`. Add focused regression coverage proving failed tool results are skipped while successful tool-only completion fallback still works. The optional secondary hardening is to improve `message` tool target validation or guidance in subagent contexts, but that does not replace fixing the announce extraction leak.
