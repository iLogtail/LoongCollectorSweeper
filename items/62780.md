## Summary

Keep #62780 open. Current main has useful related pieces: `message_sending` can rewrite or cancel outbound delivery, and gateway auto-reply fallback can classify empty/incomplete terminal model results as `format` and advance to configured fallbacks. But the requested product surface is not implemented: there is still no pre-send/content-quality hook result that can reject a completed model response and trigger model fallback with model/finish metadata, so arbitrary near-empty or quality-gate failures remain uncovered.

## Best Possible Solution

Keep the issue open, but narrow it to a model-outcome/content-quality gate in the fallback decision path. The clean direction is likely an outcome classifier or pre-delivery hook that runs before `runWithModelFallback` accepts a candidate, can return a typed `format` rejection, receives bounded model/provider/finish metadata, and is documented separately from delivery-only `message_sending` cancellation.
