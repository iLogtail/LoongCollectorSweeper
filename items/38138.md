## Summary

Keep open. Current `main` still routes Discord thread messages by normal binding priority, with `parentPeer` inheritance ahead of fallback/default routing, and only evaluates `mentionPatterns` after the route has already been chosen. I found no code path, test, or release evidence for a mention-based routing override that would let `@Dex` reroute a thread inherited from another agent.

## Evidence
