## Summary

Keep #38138 open. Current main still resolves the Discord agent from static routing inputs before mention detection, then builds mention regexes for the already selected agent and uses mention state for gating. The provided related context does not identify a canonical duplicate for this Discord thread-parent override, and the prior ClawSweeper keep-open comment remains consistent with current main at d1f40731e30b992d42217e84c14cb4457f241e9d.

## Best Possible Solution

Implement a Discord mention-based routing override in the routing or preflight path, with regression coverage for a thread that inherits a parentPeer binding while explicitly mentioning another configured agent. The fix should map native Discord bot identity and/or per-agent mentionPatterns to the intended agent before gating and dispatch, add a clear route match kind or documented preflight override, and update the public channel-routing docs if this becomes supported behavior.
