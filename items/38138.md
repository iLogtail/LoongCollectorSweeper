## Summary

Current main still routes Discord messages before mention detection, then uses the selected route's agent only for mention gating. A thread inheriting a parent channel binding can therefore still route to the parent-bound agent instead of the explicitly mentioned different agent. The only issue discussion is the stale bot comment; no related plugin, workaround beyond /focus, or superseding item was available in the provided context, and unauthenticated gh search was unavailable.
