## Summary

Current main still appears to have both reported gaps. The reminder guard only matches enabled cron jobs by sessionKey, so isolated same-agent jobs are not covered, and the compaction retry reset still clears successfulCronAdds. Related PR #43404 remains open and tracks the same fix; earlier related PRs were closed unmerged, so this issue should stay open until the fix lands or maintainers choose a canonical replacement.

## Evidence

- **Guard still only matches sessionKey:** hasSessionRelatedCronJobs accepts cronStorePath and sessionKey only, then suppresses the note only when an enabled job has job.sessionKey equal to params.sessionKey. There is no agentId or isolated-session fallback, so an enabled isolated cron with no sessionKey would still miss this check.
