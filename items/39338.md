## Summary

Keep open. Current main still rejects `"audio"` and `"video"` in `models.providers[].models[].input`, and the PR's stated user problem remains real. The submitted two-file schema/type change is plausible but should be refreshed against current main because nearby model input surfaces still narrow or filter to text/image only.

## Best Possible Solution

Keep the PR open or replace it with a refreshed narrow fix that widens the shared model input contract consistently: update the zod schema, exported config/types and SDK-facing model types, any list/normalization filters that currently drop audio/video, and add regression tests proving `models.providers[].models[].input` accepts `text`, `image`, `audio`, and `video` while still rejecting malformed values.
