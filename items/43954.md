## Summary

Keep open. Current `main` still does not expose a `feishu_card` tool, and the PR as described also misses the current Feishu config/schema surface needed to support its advertised `tools.card` toggle.

## Evidence

## Risks / Open Questions

- Closing this PR would leave `main` without any agent-facing `feishu_card` tool despite existing internal card send/update support.
- The PR body promises `tools.card: false`, but current Feishu config validation/types/defaults do not support that toggle; the review comment calling this out is still materially valid.
- The Feishu entrypoint has since moved to the bundled entry loader pattern in `extensions/feishu/index.ts`, so the PR needs a rebase/update rather than conservative closure.
