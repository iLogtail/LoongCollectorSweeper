## Summary

Keep open. Current `main` improved Google Chat webhook diagnostics, but the issue discussion points to `channels.googlechat.appPrincipal` as the real blocker, and that requirement is still not fully surfaced in the current user-facing setup/docs/status paths. The reported config from the issue body would still be rejected on `main` unless `appPrincipal` is added.

## Evidence

- **Issue discussion identified the real missing field:** The April 9, 2026 comment from `calrloco` says the failing path was an undocumented required `channels.googlechat.appPrincipal` value for add-on JWT `sub` validation, and the April 11, 2026 follow-up from `flybyaircraft` confirms that config fix worked without a code bypass.
- **Current auth path still rejects missing add-on principal:** `verifyGoogleChatRequest` accepts `app-url` tokens only when `expectedAddOnPrincipal` is present and matches the JWT `sub`; otherwise it returns `missing add-on principal binding` or `unexpected add-on principal`.
