## Summary

Keep #37656 open. Current main still does not implement the PR's central behavior: loading the resolved selected-agent workspace `.env` into the `exec` tool environment at request time. Main only loads CWD/global dotenv into `process.env` and then builds exec env from `process.env` plus explicit request overrides. The PR remains a focused implementation path for the linked #37401 behavior, with active review history addressing real edge cases.

## Best Possible Solution

Keep #37656 open as the canonical implementation path for the remaining per-agent workspace `.env` exec behavior. The best maintainer path is a reviewed core `exec` change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, filters dangerous host env overrides, skips node/sandbox hosts where workspace env is not executed, adds isolated regression tests, and updates environment/security docs once shipped.
