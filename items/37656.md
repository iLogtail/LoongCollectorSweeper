## Summary

Keep open. Current main still does not implement per-agent, exec-time workspace `.env` loading for the exec tool: gateway/node exec env is built from `process.env` plus request `params.env`, while dotenv loading remains CWD/global startup behavior. The PR remains a plausible core exec change with active review history rather than an obsolete cleanup candidate.

## Best Possible Solution

Keep the PR open as the canonical implementation path for #37401-style per-agent workspace `.env` loading. The best product/code path is to land a carefully reviewed core exec implementation that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics, filters dangerous host env overrides, skips node/sandbox hosts where appropriate, and adds focused regression tests without writing to the repo root.
