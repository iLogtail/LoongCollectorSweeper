## Summary

Keep open. Current `main` still does not expose `promptMode` on `sessions_spawn`, and sub-agent runs are still resolved to `minimal` prompt mode by session type rather than caller input.

## Evidence

- **`sessions_spawn` schema has no `promptMode` field:** The public tool schema lists `task`, `label`, `runtime`, `agentId`, `model`, `thinking`, `thread`, `mode`, `cleanup`, `sandbox`, `context`, `lightContext`, and attachments, but no `promptMode` option.
