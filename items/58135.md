## Summary

Keep open. Current `main` still does not expose a `promptMode` input on `sessions_spawn`, and the subagent runner only resolves child sessions to `full` or `minimal`, so the requested `promptMode: "none"` path is not implemented.

## Evidence

- **`sessions_spawn` public schema has no `promptMode` field:** The tool schema exposes `task`, `runtime`, `agentId`, `model`, `thinking`, `thread`, `mode`, `cleanup`, `sandbox`, `context`, `streamTo`, `lightContext`, and attachments, but no `promptMode` parameter.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/tools/sessions-spawn-tool.ts#L100)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '80,160p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Subagent spawn params still omit `promptMode`:** `SpawnSubagentParams` includes `model`, `thinking`, `mode`, `sandbox`, `context`, and `lightContext`, but there is no `promptMode` field to carry through to the child run.
  - file: [src/agents/subagent-spawn.ts:109](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/subagent-spawn.ts#L109)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '98,135p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Runner only resolves subagent sessions to `minimal` or `full`:** `resolvePromptModeForSession()` returns only `"minimal" | "full"`, and subagent session keys are forced to `minimal`; there is no route here to `promptMode: "none"` for spawned children.
  - file: [src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:92](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts#L92)

- **Internal `none` mode exists but is not surfaced by spawn:** The system prompt builder still supports `promptMode === "none"` and returns only the base identity line in that case, which matches the reporter's observation that the capability exists internally but is not reachable from `sessions_spawn`.
  - file: [src/agents/system-prompt.ts:641](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/system-prompt.ts#L641)
