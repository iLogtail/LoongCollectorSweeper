## Summary

Current `main` does not fix this report. Telegram partial preview streaming still emits tool-progress lines by default, and current tests explicitly assert raw tool identifiers/command text like `tool: exec` and `exec ls ~/Desktop` in the preview. Docs on `main` describe that as the shipped default since `v2026.4.22`, with `streaming.preview.toolProgress: false` as the opt-out, so there is not strong evidence to auto-close this as already implemented or obsolete.
