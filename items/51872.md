## Summary

Keep open. Current main does not implement the requested transport-level proxy authentication for iOS/Android gateway connections. The existing QR/setup-code flow carries the gateway URL plus OpenClaw bootstrap auth, and both mobile clients authenticate inside the OpenClaw WebSocket `connect` RPC rather than adding proxy auth headers to the initial HTTP(S)/WebSocket upgrade. The related trusted-proxy work is a different gateway-side identity mode and does not satisfy this mobile client outbound-header request.
