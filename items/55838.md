## Summary

Keep open. Current `main` and the latest release still only implement `sendChatAction` backoff/circuit-breaking for 401/unauthorized failures; the transient-network backoff proposed by PR #55838 is not on `main` yet. The repo already suppresses typing-start failures at a higher layer, so the PR is narrower than its summary suggests, but it is not obsolete.
