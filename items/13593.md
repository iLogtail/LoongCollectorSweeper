## Summary

Keep #13593 open. Current main has cron run state, run history, failure alerts/backoff, and generic gateway probes, but the central request remains missing: structured per-attempt cron execution logging, skipped-run warnings, cron-specific health reporting, and `openclaw cron list` warning annotations. The existing issue comment reached the same keep-open conclusion, and PR #46501 is closed unmerged.

## Best Possible Solution

Keep #13593 open. The best product path is to add structured per-attempt cron execution logging across scheduled timer runs, startup catch-up, and manual runs; warn on skipped outcomes; add cron health semantics through `/health/cron` or an explicit cron component in the existing readiness/health model; and teach `openclaw cron list` to annotate overdue and repeatedly failing jobs with service, CLI, probe, docs, and regression-test coverage. PR #46501 remains useful prior art, but it was closed unmerged and is not present on current main.
