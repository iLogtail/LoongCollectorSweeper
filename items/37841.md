## Summary

Keep #37841 open. Current main still suppresses final and follow-up session text based on same-target message-tool target metadata even when the tool sent only media and recorded no sent text. PR #37900 targeted the right bug, but the provided GitHub context shows it was closed unmerged, and the relevant files are unchanged from the prior ClawSweeper review SHA to current main.

## Best Possible Solution

Keep #37841 open and treat it as the canonical bug. Patch buildReplyPayloads and resolveFollowupDeliveryPayloads so same-origin target suppression only drops final or follow-up session text when the message tool actually sent text to that same target. Media-only sends should still dedupe already-sent media URLs while preserving session text. Add focused regression tests for same-target media-only sends preserving text, same-target text sends still suppressing duplicate text, and the follow-up delivery path.
