## Summary

Keep open. Current `main` does not implement this fix: gateway-RPC CLI still coerces `opts.timeout` with `Number(...)`, invalid values still fall through to the gateway's `10_000ms` fallback, and the stricter timeout parsing/test coverage from this PR is not present.

## Evidence

## Risks / Open Questions

- The PR's original patch targeted `src/cli/gateway-rpc.ts`, but current main has moved the transport logic into `src/cli/gateway-rpc.runtime.ts`, so it likely needs a small rebase/adaptation rather than a straight merge.
- The current permissive shared parser means malformed numeric strings such as `10ms` can still be accepted in multiple CLI surfaces, so closing this PR would leave the reported behavior unresolved.
