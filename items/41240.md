## Summary

Keep open. Current `main` still schedules heartbeat timers with the raw computed delay and does not clamp values above Node.js's 32-bit `setTimeout` limit, so the reported overflow path is not implemented away on `main`. The specific claim that heartbeat now defaults to `30d` is not supported by current code/docs, but the overflow bug itself still appears valid and an issue-linked fix PR is still open.
