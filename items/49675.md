## Summary

Current `main` still has the lazy-import failure mode described by this PR, and the proposed patch has unresolved review issues. This is not implemented on `main`, not obsolete, and not safe to auto-close.

## Evidence

- **Current main still caches a rejected lazy import forever:** `createLazy` stores `loader().then(...)` in `s.promise` with no rejection handler. If the dynamic import rejects, `s.promise` stays set and later calls skip reloading and return `null` again.

## Risks / Open Questions

- The underlying bug still exists on current `main` and on the latest release: a rejected lazy import can strand a Control UI tab in a blank state.
- Closing this PR would discard a valid bug report even though the current implementation still matches the reported failure mode.
- Merging the PR as-is would likely introduce a retry storm on permanent import failures and leaves the promise typing unsafely widened via `as any`.
- There is no targeted regression coverage in the checked-out tree for rejected `createLazy()` imports, so the failure path remains unguarded.
