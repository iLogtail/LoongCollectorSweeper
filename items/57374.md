## Summary

Keep open. Current `main` does not implement this restart-probe auth fix: gateway restart health checks still authenticate from process env only, the restart/status/update call sites still do not pass resolved local probe auth, and the latest PR discussion points to an additional unresolved flaw in the proposed helper because local credential resolution can still fall back to remote credentials.

## Evidence

- **Current restart health probes are still env-only:** On `main`, `confirmGatewayReachable()` reads only `OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` from `process.env`, and the restart-health APIs do not accept a resolved probe-auth argument. That matches the bug described in the PR rather than an implemented fix.
