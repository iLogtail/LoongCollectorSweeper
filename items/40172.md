## Summary

Current main already provides the core primitives: Telegram inbound media is downloaded to local files and exposed as `MediaPath`/`MediaPaths`, and avatar updates already exist as a generic agent identity feature. The remaining ask is Telegram-specific automation to turn an inbound image into an avatar update, which fits the repo vision for optional plugin/skill behavior rather than OpenClaw core.

## Evidence

## Close Comment

Thanks for the request. I reviewed current `main` in Codex. Telegram already downloads inbound media into local files in `extensions/telegram/src/bot/delivery.resolve-media.ts:188`, and the resolved files are already exposed to agent context as `MediaPath` / `MediaPaths` in `extensions/telegram/src/bot-message-context.session.ts:366`. OpenClaw also already has a generic avatar-setting path via `openclaw agents set-identity --avatar ...` (`docs/cli/agents.md:165`). What remains here is Telegram-specific automation that turns an inbound image into an avatar update. Per `VISION.md:54` and `VISION.md:69`, optional workflow capability like this should usually ship as a plugin/skill, with new skills going to ClawHub first. Closing this as a better fit for a ClawHub skill/plugin rather than OpenClaw core.
