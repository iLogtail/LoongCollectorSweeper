## Summary

Keep open. The report is recent, concrete, and still maps to current bundled MCP client code. Current main still defaults URL-only remote MCP configs to SSE unless the OpenClaw `transport: "streamable-http"` field is set, while `type: "http"` is only a downstream CLI-backend shape. I found no source, changelog, or history evidence that #70753 / AgentMail hosted MCP timeouts have been fixed on main.

## Best Possible Solution

Keep this issue open and fix it in the bundle-MCP client path. A focused maintainer path is to add regression coverage for a hosted-style streamable HTTP MCP endpoint, normalize or reject CLI-native `type: "http"` in OpenClaw-managed `mcp.servers` before embedded runtime startup, and only then investigate undici/SDK timeout tuning if the corrected `streamable-http` path still fails against AgentMail.
