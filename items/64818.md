## Summary

Keep open. Current main still has the same core failure shape: the generated A2UI hash file is tracked, the bundler can rewrite it, and the dev update preflight walkback still uses plain `git checkout --detach` in the disposable worktree without resetting or stashing dirty tracked files. The linked fix PR #64822 was closed unmerged, and the open #67547 is related A2UI reproducibility work rather than a canonical replacement for this updater failure.

## Best Possible Solution

Keep #64818 open. The best fix is to remove `src/canvas-host/a2ui/.bundle.hash` from git tracking again and add updater hardening so each disposable preflight candidate starts from a clean worktree before checkout. A focused regression test in `src/infra/update-runner.test.ts` should simulate a preflight build dirtying a tracked generated file, then verify fallback candidates remain reachable instead of collapsing into `preflight-no-good-commit`.
