## Summary

Keep PR #70676 open. Current main still has the three revocation paths described in the PR using best-effort `socket.close()` with bare `catch {}` blocks and no warning or `terminate()` fallback, including the device-token removal/revoke path. The PR is a focused gateway bug/security hardening change with regression tests and the one bot-review concern was addressed in its final commit, so there is not enough basis to close it as implemented, superseded, ClawHub, or unactionable.

## Best Possible Solution

Keep this PR open or recreate its narrow fix on current main: add a shared revocation close helper that logs `socket.close()` failures, falls back to `socket.terminate()`, threads gateway loggers through shared-auth revocation call sites, and adds focused unit tests for throwing close, throwing terminate, skipped clients, and continued eviction of remaining clients.
