## Summary

Keep #70676 open. Current main still has the three silent gateway revocation catch blocks the PR targets, with no warning log, no terminate fallback, and no regression coverage for throwing socket.close() on those paths. The PR is a focused gateway/security hardening change; the related #70515 is only a similar bare-catch pattern in config recovery, not a canonical duplicate or shipped fix.

## Best Possible Solution

Keep the PR open and have a maintainer either land the same log-then-terminate behavior with the added fault-injection tests, or rework it into an equivalent main-branch patch. Preserve the arrow-wrapper logger fix from ab83788042055df96c421fa6474417202bb88561, and keep #70515 separate because it addresses config restore audit/logging rather than gateway WebSocket revocation.
