## Summary

Keep #70676 open. Current main still has the exact silent WebSocket revocation close-failure paths described by the PR, with no warning log and no terminate fallback, and the current tests do not cover throwing close calls on those paths. The PR is focused and the provided discussion shows the one bot review concern was addressed in the latest PR commit.

## Best Possible Solution

Keep the PR open for maintainer review. The best product/code path is to land this focused log-then-terminate handling with the targeted regression tests, or apply an equivalent fix on main that covers shared-auth rotation, config reload/restart revocation, and device removal/token revocation paths.
