## Summary

PR #41613 should stay open. Current main has not fully made the docs work obsolete: `OPENCLAW_CLI=1` is implemented and released but still missing from `docs/help/environment.md`, and the macOS remote gateway token field exists in app code but is still absent from `docs/start/onboarding.md`. The PR needs rework before merge because current main already has a one-line ACP `--provenance` option, the Gmail PubSub page is now a redirect with no stale `OPENCLAW_HOOK_TOKEN` examples, and the PR's direct `docs/zh-CN/**` edit conflicts with current docs i18n policy.

## Best Possible Solution

Keep #41613 open but request a rebase/rework against current main: add `OPENCLAW_CLI=1` to `docs/help/environment.md`, document the macOS remote gateway token behavior in `docs/start/onboarding.md`, optionally expand the existing ACP `--provenance` option text, drop the direct `docs/zh-CN/**` change, and only touch Gmail docs if the current `docs/automation/cron-jobs.md` surface still has a live stale token example.
