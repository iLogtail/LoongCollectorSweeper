## Summary

Keep open. Current `main` still normalizes request origins through `URL.origin`, so exact `chrome-extension://<id>` allowlist entries cannot match and the Gateway browser-origin path still rejects them. The PR remains a narrow, relevant fix with corresponding unit/integration coverage, and there is no equivalent implementation on `main` at `f00d65a30496335fb068f01fe529dbba484d4312`.

## Evidence
