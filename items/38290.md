## Summary

Keep PR #38290 open. Current main still normalizes the incoming Origin with `URL.origin`, so exact `chrome-extension://<id>` or `app://localhost` allowlist entries collapse to `"null"` and do not match. The PR remains a narrow gateway hardening bug fix with relevant unit and WebSocket regression coverage in the provided diff; scheme wildcards remain separate under #39739.

## Best Possible Solution

Keep PR #38290 open and land it after rebase, or land an equivalent patch in `src/gateway/origin-check.ts` that symmetrically normalizes opaque-origin schemes for incoming `Origin` values and configured `allowedOrigins`. Preserve literal `Origin: null` rejection, exact matching, global `"*"` behavior, host-header fallback semantics, and add/keep gateway WebSocket regression coverage. Track scheme-level wildcard support such as `chrome-extension://*` separately in #39739.
