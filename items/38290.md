## Summary

Keep open. Current `main` still collapses non-standard origins through `URL.origin`, so an allowlisted `chrome-extension://<id>` origin will not match `gateway.controlUi.allowedOrigins`. The PR addresses a real, still-unfixed gateway browser-origin bug.

## Evidence

- **Current parser uses `url.origin` for matching:** `parseOrigin()` returns `origin: normalizeLowercaseStringOrEmpty(url.origin)`. For non-standard schemes this becomes `"null"`, so the match key is lost before allowlist comparison.
  - file: [src/gateway/origin-check.ts:22](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/src/gateway/origin-check.ts#L22)
  - command: `nl -ba src/gateway/origin-check.ts | sed -n '14,27p'`
  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **Allowlist comparison cannot match extension origins:** `checkBrowserOrigin()` lowercases raw allowlist strings, then compares them against `parsedOrigin.origin`. With `origin="chrome-extension://abcdefghijklmnop"`, the allowlist entry stays `chrome-extension://abcdefghijklmnop` but `parsedOrigin.origin` is `null`, so exact matching fails.
  - file: [src/gateway/origin-check.ts:45](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/src/gateway/origin-check.ts#L45)

  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **WHATWG URL behavior reproduces the root cause:** `node -e` on current environment returned `{"origin":"null","protocol":"chrome-extension:","host":"abcdefghijklmnop","hostname":"abcdefghijklmnop"}` for `new URL('chrome-extension://abcdefghijklmnop')`.
  - command: `node -e "const u=new URL('chrome-extension://abcdefghijklmnop'); console.log(JSON.stringify({origin:u.origin, protocol:u.protocol, host:u.host, hostname:u.hostname}))"`
- **Gateway handshake still depends on this check:** Browser/operator-ui WebSocket handshakes still call `checkBrowserOrigin()` and reject with `origin not allowed` when it fails, so the bug remains user-visible in the active auth path.
