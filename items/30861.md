## Summary

Keep #30861 open. Current main still has only a chat session selector grouped by agent plus model/thinking controls; it does not implement a persistent chat-header active-agent/workspace indicator, workspace tooltip, switch confirmation feedback, or single-agent minimal display. The gateway already exposes agent workspace data, so this remains a UI feature gap rather than a backend contract blocker. The related implementation PR #31602 was closed unmerged.

## Best Possible Solution

Keep #30861 open as the canonical Control UI tracker. A good implementation should derive the active agent from the current session key, show the agent id/name and workspace path from `agents.list` in the chat header with a full-path tooltip, keep `switchChatSession`, `/agent`, and persisted `lastActiveSessionKey` synchronized after reload/reconnect, add switch feedback, keep the single-agent case visually minimal, and add focused UI helper/render tests. Closed PR #31602 is useful prior art but does not supersede the issue.
