## Summary

Keep open. Current main still does not implement the Control UI session/chat visibility work from #55927, although the gateway already exposes much of the needed session metadata. The PR also still needs the review-thread fixes folded in before it would be merge-ready.

## Best Possible Solution

Keep #55927 open or consolidate it with a maintainer-selected canonical Control UI session visibility PR. The best path is a focused rebase on current main that syncs the UI session row type with the gateway contract, requests bounded `includeDerivedTitles`/`includeLastMessage`, renders the useful session/channel/delivery/usage context in Sessions and Chat, fixes the token fallback and duration math from the review thread, and adds targeted controller/view tests.
