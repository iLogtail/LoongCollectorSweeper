## Summary

Keep PR #54900 open. Current main still replays first-attempt outbound delivery queue entries regardless of age during gateway startup recovery, so the PR is not implemented or superseded. The branch is a plausible focused fix, but it still needs maintainer review for the noted budget-deferral edge case before merge.

## Best Possible Solution

Keep the PR open, have the author or maintainer address the budget-deferral edge case, then validate with targeted recovery tests covering stale first-attempt entries, legitimate retries, and budget-deferred never-attempted entries. The fix belongs in `src/infra/outbound/delivery-queue-recovery.ts` with focused tests in `src/infra/outbound/delivery-queue.recovery.test.ts`.
