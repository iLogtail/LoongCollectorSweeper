## Summary

Keep #52442 open. Current main still closes the long-lived WhatsApp inbound monitor socket through the raw websocket close path, and the existing test coverage asserts that behavior. The linked fix PR #52443 was closed unmerged, so it does not supersede the issue. Baileys 7.0.0-rc.9 is still the pinned dependency and exposes an `end` teardown path that clears timers/listeners before closing the websocket, so the reported cleanup gap remains plausible and actionable in the WhatsApp plugin.
