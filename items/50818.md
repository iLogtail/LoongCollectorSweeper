## Summary

Keep open. Current main has part of the surrounding reply-route fallback, but it does not implement this PR's remaining session-key targeting work: there is no `resolveEventSessionKey`, cron-run event queues are not remapped with `session.mainKey`, and `heartbeat.target: "none"` still intentionally keeps exec completions internal-only. The linked remaining bug #52305 is still open in the provided context, while #39978 was narrowed/fixed elsewhere by #70258.

## Best Possible Solution

Keep #50818 open. The best path is to rebase it onto current main, preserve the already-shipped #70258 reply-route fallback, and revalidate only the remaining #52305 session-targeting scenarios: cron-run enqueue/wake key alignment, custom `session.mainKey`, ACP/CLI watchdog events, node exec events, and explicit `target: "none"` exec-completion delivery semantics.
