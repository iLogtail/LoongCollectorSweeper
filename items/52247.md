## Summary

Keep open. Current `main` still appears to have the reported `session_status` bug: the status card model selection path does not apply `channels.modelByChannel` before building the display, the fallback note logic still suppresses the divergent-model case, and the only local test still uses the same model for both default and override. The PR also targets an implementation file that has since been refactored into a re-export, so it needs a rebase/update rather than cleanup closure.

## Evidence

- **Status message still ignores channel override during selected-model resolution:** `buildStatusMessage` still derives `selectedProvider`/`selectedModel` only from per-session overrides or the agent default before calling `resolveSelectedAndActiveModel`. There is no `channels.modelByChannel` lookup in this selection step, so a channel override cannot change the displayed `🧠 Model:` line here.
  - file: [src/status/status-message.ts:498](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/status/status-message.ts#L498)
  - command: `nl -ba src/status/status-message.ts | sed -n '492,505p'`
  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Channel-override note still only appears when selection already matches the override:** The `channelModelNote` branch resolves the channel override, then returns `undefined` whenever the resolved override differs from `selectedProvider`/`selectedModel`. That means the note is suppressed in the exact divergent-model scenario described in the PR.
  - file: [src/status/status-message.ts:820](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/status/status-message.ts#L820)

- **session_status tool still seeds the card from the configured default path:** `session-status-tool` calls `resolveSessionModelIdentityRef` with the configured default model as `fallbackModelRef`, then forwards the resulting `providerForCard`/`defaultModelForCard` into `buildStatusText`. That path does not itself resolve `channels.modelByChannel`.
  - file: [src/agents/tools/session-status-tool.ts:510](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/tools/session-status-tool.ts#L510)

In the `session_status` call path above, that means the helper returns the configured default model rather than any derived channel override.
