## Summary

Current `main` does not implement the requested transcript self-identification. Session headers are still written with only `type`, `version`, `id`, `timestamp`, and `cwd`; docs and tests match that shape, and disk-only session discovery still falls back to filename plus first user message rather than recovering a `sessionKey` from transcript metadata.
