## Summary

Keep open. Current `main` still has the cron `agentId` autofill gap this PR targets, and the behavior remains meaningful because a missing cron `agentId` falls back to the default agent rather than the caller's sub-agent.

## Evidence

- **Cron tool still only autofills when the key is absent:** `createCronTool` assigns `job.agentId` only under `!("agentId" in job)`, so a job object carrying `agentId: undefined` skips the autofill path on current `main`.
  - file: [src/agents/tools/cron-tool.ts:540](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/agents/tools/cron-tool.ts#L540)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '520,552p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Normalizer preserves an undefined `agentId` property:** `normalizeCronJobInput` clones `base` with `{ ...base }`; when `base.agentId` is `undefined`, the `agentId` branch neither normalizes nor deletes it, so the property can survive up to the gateway call site.
  - file: [src/cron/normalize.ts:413](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/cron/normalize.ts#L413)
  - command: `nl -ba src/cron/normalize.ts | sed -n '405,434p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Gateway transport drops undefined fields on the wire:** Gateway requests are sent as `JSON.stringify(frame)`. A direct check in this checkout showed `JSON.stringify({type:"req",params:{job:{name:"wake-up",agentId:undefined}}})` becomes `{"type":"req","params":{"job":{"name":"wake-up"}}}`, so the server receives no `agentId` field at all.
  - file: [src/gateway/client.ts:923](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/gateway/client.ts#L923)
  - command: `nl -ba src/gateway/client.ts | sed -n '914,954p'; node -e 'console.log(JSON.stringify({type:"req",params:{job:{name:"wake-up",agentId:undefined}}}))'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Missing cron `agentId` falls back to the default agent:** Cron isolated-agent preparation resolves `requestedAgentId` from explicit input or `job.agentId`, then uses `normalizedRequested ?? defaultAgentId`. If the tool fails to stamp the caller's sub-agent ID, execution falls back to the default agent.
  - file: [src/cron/isolated-agent/run.ts:428](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/cron/isolated-agent/run.ts#L428)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '424,440p'`

- **Current tests do not cover the missing/undefined autofill case:** The existing cron-tool tests on `main` cover `agentId: null` preservation, but the added regression cases from this PR are not present. That leaves the reported path unguarded in current test coverage.
  - file: [src/agents/tools/cron-tool.test.ts:206](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/agents/tools/cron-tool.test.ts#L206)
