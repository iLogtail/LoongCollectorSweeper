## Summary

Keep PR #62070 open. Current main still returns OpenAI-compat gateway images through the embedded runner early-return path without `sanitizeImagesWithLog`, and current streaming coverage only proves rejected-command errors, not lifecycle `phase: "error"` events that arrive before the async catch path. The PR addresses a real linked bug (#59913), and its commits are not present in the current checkout history.

## Best Possible Solution

Keep this PR or a maintainer-refined replacement open as the active fix path for #59913. The best implementation is to sanitize `existingImages` on the no-ref early return, add a regression test for that path, and add a current-main streaming test where a lifecycle `phase: "error"` fires before assistant deltas so clients receive visible error content exactly once.
