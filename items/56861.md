## Summary

Keep open. Current main does not implement the PR's central Signal changes: quoted media is not fetched from quote attachments or routed into model media context, native Signal mentions are still reduced to rendered text plus regex gating, and outbound Signal media still sends local file paths only. The PR is relevant to a bundled Signal channel, but it needs maintainer review and targeted tests before merge because the existing review thread raises concrete edge-case risks.

## Best Possible Solution

Keep the PR open and ask for a focused maintainer-ready revision: route quoted Signal media through the existing MediaPaths/MediaTypes ingestion path, implement structured bot mention and reply-to-bot facts using the shared mention decision helper, add a deliberate remote signal-cli attachment mode instead of a localhost heuristic if needed, and cover these paths with Signal extension tests. If the PR is split, keep this item as the canonical Signal-channel implementation thread until a replacement PR is opened and linked.
