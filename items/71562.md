## Summary

Keep open. Current `main` code that assembles workspace bootstrap/context files reads a fixed allowlist of known workspace markdown files as raw UTF-8 text, and the extra-bootstrap glob path rejects unrelated markdown basenames like `README.md`. That makes the reported `.md`->JSON parsing bug non-obvious in the inspected workspace-context path, but with read-only inspection only and no runtime repro/logs/comments, the evidence is not strong enough to conservatively close this issue.

## Evidence

- **Workspace bootstrap loader reads known workspace files as plain text:** `loadWorkspaceBootstrapFiles()` enumerates only the known workspace bootstrap filenames (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, `MEMORY.md`) and passes each through `readWorkspaceFileWithGuards()`, which uses `readFileSync(..., "utf-8")` and returns the raw string content.
  - file: [src/agents/workspace.ts:65](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/workspace.ts#L65)

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Extra bootstrap file loading filters by recognized basename:** `loadExtraBootstrapFilesWithDiagnostics()` resolves glob matches but explicitly skips any file whose basename is not in the bootstrap allowlist before reading it, so arbitrary root markdown files are not accepted by that path.
  - file: [src/agents/workspace.ts:706](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/agents/workspace.ts#L706)
