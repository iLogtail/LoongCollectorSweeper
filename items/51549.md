## Summary

Keep #51549 open. Current main appears to restore WebChat conversation history from the gateway on connect/refresh, but the pending message queue and unsent draft are still stored only in component state. The open PR #71336 tracks queue persistence, but it is not merged and does not fully cover the draft persistence part of the issue.

## Best Possible Solution

Keep the issue open and either extend PR #71336 or land a follow-up that persists pending queue entries and unsent drafts per session, restores them on reconnect/session switch, and adds focused UI tests. The existing `chat.history` reconnect path should be preserved as the history restoration mechanism rather than replaced.
