## Summary

Keep #7006 open. Current main still does not expose the concrete model OpenRouter selected for `openrouter/auto`, and its response/footer/session cost paths still use normalized token usage plus local pricing estimates rather than OpenRouter's provider-reported `usage.cost` and `cost_details`. The ClawRouter comment is a useful external workaround, but the reporter's clarification is correct: that is a separate router and does not satisfy OpenRouter auto transparency. Related #9016 is cost-only, and PR #43026 was closed unmerged.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's routed response `model` and numeric per-response cost (`usage.cost` plus relevant `usage.cost_details`) in the OpenAI-compatible transport, preserve selected versus resolved model identity in assistant/session metadata, and thread that through response footers, `/usage`, transcripts, status/session surfaces, and focused regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for this resolved-model transparency request.
