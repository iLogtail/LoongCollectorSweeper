## Summary

Current `main` does not implement the requested `openclaw agent --new-session` flag. The CLI still exposes only `--to`, `--session-id`, or `--agent` as session selectors, and agent-targeted runs still derive a stable per-agent main session key by default. A recent mainline fix addressed `--agent` + `--session-id` scoping, but not `--new-session` or a new JSON session-id contract for this CLI flow.
