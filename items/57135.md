## Summary

Keep open. Current main already shows key names for Zod strict-object errors, but it does not fully implement the PR’s observable request for schema-backed config validation: AJV `additionalProperties` errors are still passed through as generic messages and then wrapped as `invalid config: ${error.message}` for channel/plugin config. The PR is also unsafe to merge as-is because the provided review context shows unrelated Feishu localization and config behavior changes bundled with the narrow error-message fix.

## Best Possible Solution

Keep the item open only as a signal for the narrow config-error improvement. The maintainable path is to split or replace this PR with a small change in `src/plugins/schema-validator.ts` that formats AJV `additionalProperties` errors using `params.additionalProperty`, adds focused tests for channel/plugin config validation, and drops the unrelated Feishu localization and `plugins.allow` behavior changes.
