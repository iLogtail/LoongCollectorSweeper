## Summary

Keep #24943 open. Current main still does not strip general Markdown before iMessage delivery: the iMessage send path converts Markdown tables and strips inline directive tags, while reply delivery strips assistant-internal scaffolding and converts tables before sending. The shared stripMarkdown helper exists and BlueBubbles uses it, but iMessage does not. The supplied GitHub context also shows PR #56283 as the current open/unmerged fix attempt and an earlier iMessage-specific PR #25349 closed unmerged.

## Best Possible Solution

Keep #24943 open until PR #56283 or an equivalent fix lands. The best path is to strip or render Markdown to plain text for iMessage/plain-text outbound delivery after existing HTML/table/internal-scaffolding handling and before final send, with focused tests for bold, italic, headings, blockquotes, inline code, syntax-only inputs, media captions, and reply delivery.
