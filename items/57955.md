## Summary

Current `main` does not implement session-scoped `chat` delivery. The PR’s exact visible symptom is partly stale because the Control UI filters `chat` events by `sessionKey` before rendering, but the gateway still broadcasts `chat` payloads to every `operator.read` websocket client, so the underlying session-isolation work is still not on `main` and should remain open.
