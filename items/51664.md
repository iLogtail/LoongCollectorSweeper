## Summary

Keep open. Current main still routes WebChat code-block Copy through `navigator.clipboard.writeText(code)` and silently ignores failures, with no Edge-oriented fallback. The two related fallback PRs in the issue timeline were closed unmerged, and the latest release commit shows the same Clipboard API-only behavior.
