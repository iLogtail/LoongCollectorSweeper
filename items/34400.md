## Summary

Keep #34400 open. Current main recursively discovers default memory files and watches the default memory directory, but the issue discussion’s remaining runtime and user-facing gaps still exist: nested dated memory files are treated as evergreen, dirty search sync is still fire-and-forget, and several tool/prompt/docs strings still describe only `memory/*.md`. The related fixing PRs #34585 and #35691 were closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely path is to support nested dated memory paths in temporal decay, decide whether the first dirty `memory_search` must await sync for freshness, review extra-path watcher glob behavior under current chokidar, and align tool descriptions, prompt guidance, config help, CLI output, and public docs with the shipped recursive behavior. Add regression coverage for nested dated files and the selected freshness behavior.
