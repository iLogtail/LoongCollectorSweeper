## Summary

Current main still has bundled Qwen/DashScope Coding Plan support, but it does not officially enable Bailian Coding Plan thinking/reasoning for the requested Qwen, GLM, and Kimi model set. The relevant catalog entries remain non-reasoning, and the shared OpenAI-compatible transport only emits reasoning parameters when the selected model advertises reasoning support.

## Best Possible Solution

Keep #26037 open and implement this in the Qwen provider surface. The likely path is to verify Bailian Coding Plan reasoning controls per supported Qwen/GLM/Kimi model, mark only confirmed catalog entries as reasoning-capable or add a provider-owned thinking profile/payload adapter, then update Qwen docs and regression tests for catalog discovery, OpenAI-compatible payload construction, and the reported gateway restart/config path.
