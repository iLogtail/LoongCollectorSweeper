## Summary

Keep #26037 open. Current main has a bundled Qwen Cloud / DashScope Coding Plan provider, but the requested Bailian Coding Plan reasoning support is still not implemented for qwen3.5-plus, glm-5, or kimi-k2.5, and the existing transport only emits reasoning payload fields when model/provider metadata makes that path available.

## Best Possible Solution

Keep this open and implement it in the bundled Qwen provider once Bailian's current Coding Plan reasoning contract is verified. The likely maintainer path is to add Qwen-owned reasoning metadata or a Qwen-specific thinking payload adapter for verified models such as qwen3.5-plus, glm-5, and kimi-k2.5, plus docs and regression tests covering catalog discovery, payload construction, and the reported gateway restart/config path.
