## Summary

Keep PR #56705 open. Current main has absorbed or superseded the unrelated Slack `ignoreSelf` and TTS parser pieces, but the PR's named config-validation change is not implemented on main: validation still derives known plugin IDs only from the manifest registry, and the focused missing-bundled-dir/severity tests from the PR are not present.

## Best Possible Solution

Keep the PR open but do not merge it as-is. Ask for a refreshed, focused config-validation patch that drops the already-superseded Slack/TTS changes, then decide whether `src/config/validation.ts` should centralize known plugin ID resolution with bundled alias/auto-enable metadata or rely solely on the current installed-registry design. Any accepted config change should add focused tests for missing bundled discovery and stable warning/issue severity across `plugins.allow`, `plugins.entries.*`, and `plugins.slots.memory`.
