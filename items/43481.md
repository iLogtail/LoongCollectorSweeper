## Summary

Keep open. Current main does not implement provider-level trust policy or the linked #57959 fix. Explicit `opts.images` are still stripped on every fallback retry rather than resolved against provider boundaries, while prompt-detected image references are still loaded unconditionally for vision-capable fallback models. The related open PR #57959 directly tracks this issue and #43492, but it is not merged/shipped in the checked-out main.

## Best Possible Solution

Keep #43481 open until #57959 or an equivalent core change lands. The implementation should live at the `runWithModelFallback` / `runAgentAttempt` boundary, compare primary and fallback providers, route explicit attachments through a tested `resolveRetryImages` policy, suppress prompt-image detection on untrusted cross-provider retries, and add any resulting config/docs/tests for the chosen trust policy.
