## Summary

Keep open. Current `main` does not implement the requested general-purpose private/incognito session mode. The codebase only has limited lightweight bootstrap-context controls for cron/heartbeat/subagent paths, while normal session transcripts are still persisted to disk and ordinary runs still inject workspace context.

## Evidence

- **Only lightweight bootstrap modes exist:** The bootstrap context switch is only `full` vs `lightweight`. In lightweight mode, heartbeat keeps only `HEARTBEAT.md`; default/cron lightweight runs skip bootstrap files entirely. There is no broader private/incognito mode that suppresses all workspace context for normal chat sessions.
  - file: [src/agents/bootstrap-files.ts:165](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/src/agents/bootstrap-files.ts#L165)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '160,220p'`
  - sha: [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c)
- **Gateway API exposes only bootstrap context knobs:** The gateway agent RPC schema and handler expose `bootstrapContextMode: "full" | "lightweight"` plus run kinds `default|heartbeat|cron`; there is no session-level `private`, `ephemeral`, or `noPersist` flag.
  - file: [src/gateway/protocol/schema/agent.ts:157](https://github.com/openclaw/openclaw/blob/a2221d6b47094cd34c12025ce809aba02c5e1c1c/src/gateway/protocol/schema/agent.ts#L157)
