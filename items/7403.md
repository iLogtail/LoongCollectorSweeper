## Summary

Keep open. Current main still does not implement the requested enforceable Private/Secret/Incognito mode. There is a global context-injection control that can suppress bootstrap-file injection, but no per-session or per-channel private flag, no Mission Control toggle, no no-persist session mode, and transcript/session-store writes still occur on the normal Gateway, CLI, ACP, and Codex paths. The provided related context also shows #12242 was closed as a duplicate of this issue and #52478 was closed unmerged, so #7403 remains the canonical tracker.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete implementation should add an explicit private/ephemeral session mode through Gateway protocol schemas, session config/store entries, transcript creation and append paths, workspace bootstrap and memory read/write paths, Mission Control and applicable channel controls, with tests proving private sessions neither read workspace memory/bootstrap files nor write session history or memory files.
