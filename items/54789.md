## Summary

Current main already implements the reported TUI queue/escape fixes: queued prompts stay in `chatQueue` until dispatch, terminal chat events clear stale streaming/tool state and advance the queue, and aborts are replayed safely after reconnect instead of failing from stale state.

## Close Comment

Closing after a Codex review because current `main` already covers the reported TUI queue/escape behavior. Queued prompts are kept in `chatQueue` and only enter the transcript when actually dispatched (`ui/src/ui/app-chat.ts:112`, `ui/src/ui/controllers/chat.ts:294`). Terminal chat handling now clears pending run markers, resets streaming/tool UI state, reloads persisted history after tool runs, and flushes the next queued item (`ui/src/ui/app-gateway.ts:396`, `ui/src/ui/controllers/chat.ts:179`, `ui/src/ui/controllers/chat.ts:456`). The abort path also queues disconnect-time aborts and replays them on reconnect instead of leaving stale state behind (`ui/src/ui/app-chat.ts:98`, `ui/src/ui/app-gateway.ts:300`). Regression coverage is in `ui/src/ui/app-chat.test.ts:502`, `ui/src/ui/app-gateway.node.test.ts:539`, `ui/src/ui/app-gateway.node.test.ts:860`, and `ui/src/ui/controllers/chat.test.ts:184`.
