## Summary

Keep #9797 open. Current main still lacks the requested agent-facing `queue_status` tool and also lacks a stable core `diagnostics.queue`/CLI queue snapshot surface. Related PR #43618 is relevant, but its provided scope explicitly says it does not add the in-agent tool requested here.

## Best Possible Solution

Keep this issue open and implement a shared read-only queue snapshot provider in core. Expose it through the requested agent-facing `queue_status` tool, and preferably through the operator CLI/RPC surfaces discussed in the comments, with tool catalog, permissions, docs, and tests updated together. If #43618 becomes the canonical backend, use #9797 to track the remaining agent-tool consumer until that ships.
