## Summary

Keep open. Current `main` contains evidence that some bundled-plugin startup regressions from `v2026.3.8` were addressed later, but there is not strong evidence that the overall crash-loop report is fully obsolete. In particular, the WeakMap crash path referenced in the issue thread still appears plausible on `main`, and this checkout has no built `dist/` tree to verify the packaged-release SyntaxErrors against current artifacts.

## Evidence

- **WeakMap null/undefined crash path still looks live:** `estimateMessageCharsCached(...)` calls `cache.get(msg)` and `cache.set(msg, estimated)` before any object/null guard. If a malformed `null`/`undefined` message reaches this path, it can still throw `Invalid value used as weak map key`, which matches the linked issue comment's suspected crash class.
