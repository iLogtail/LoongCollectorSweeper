## Summary

Keep open. Current main still has only the generic legacy plugin missing-register diagnostics and coarse bundled-channel contract warning, so the PR's requested diagnostic improvement is not implemented or superseded. The PR is plausible, but the supplied review comments identify a real regression risk: its current head drops the documented OPENCLAW_PLUGIN_LOAD_DEBUG module-shape detail for ordinary plugins.

## Best Possible Solution

Keep the PR open and ask for a narrow amendment: add bundled-channel-entry-specific diagnostics in both loader paths, preserve `formatMissingPluginRegisterError(mod, env)` for ordinary missing-register failures, use the already-normalized `record.kind` where possible, and add targeted tests for both the new bundled-channel diagnostic and the existing debug-shape contract.
