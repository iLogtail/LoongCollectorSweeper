## Summary

Keep open. Current main still emits the generic missing register/activate diagnostic for this path, and the bundled channel loader still collapses malformed bundled-channel-entry contracts to a generic skip warning. The PR is a plausible focused bug-fix, but it should not be auto-closed or merged as-is because the provided diff/review shows it drops the documented debug module-shape detail for ordinary plugin missing-register failures.

## Best Possible Solution

Keep the PR open and request a narrow revision: add the bundled-channel-entry-specific diagnostic without removing formatMissingPluginRegisterError for normal plugin failures, use the already-resolved record kind or an explicit local type guard instead of unnecessary resolved.definition casts, and add focused regression coverage for the new bundled-channel diagnostic plus the existing OPENCLAW_PLUGIN_LOAD_DEBUG module-shape contract.
