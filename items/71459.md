## Summary

Keep open. Current main still emits generic missing register/activate diagnostics in both plugin loader paths and still collapses bundled channel entry validation to a generic contract-missing warning. The PR addresses a real remaining diagnostic gap, but its current branch also drops the existing OPENCLAW_PLUGIN_LOAD_DEBUG module-shape details for ordinary missing-register failures, so it needs revision rather than conservative closure.

## Best Possible Solution

Keep #71459 open and ask for a narrow revision: keep `formatMissingPluginRegisterError(mod, env)` for non-bundled missing-register failures in both loader paths, detect bundled channel entries from the normalized record kind with the existing kind helper, and add focused tests for the new bundled-channel-entry diagnostic plus field-level bundled entry validation logging.
