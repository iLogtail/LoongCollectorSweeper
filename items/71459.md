## Summary

Keep #71459 open. Current main does not implement the PR’s central bundled-channel-entry diagnostic: both plugin loader missing-register branches still use the generic missing-register path, and bundled channel entry validation still emits only a generic missing-contract warning. The PR is a focused, plausible fix, but its current diff also drops the documented/tested debug module-shape diagnostic for ordinary missing-register errors, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: add the bundled-channel-entry diagnostic without regressing `formatMissingPluginRegisterError(...)` for ordinary plugins, prefer the already-normalized `record.kind`/`hasKind(...)` style where applicable, and add focused tests covering both `loadOpenClawPlugins` and `loadOpenClawPluginCliRegistry` plus the bundled entry missing-field warning if that behavior remains part of the PR.
