## Summary

Keep #14248 open. Current main at 9a529ca78bfa0e257f662f0e70ab8c681caa6086 still does not expose Slack Socket Mode ping/pong timeout tuning in config types, zod schema, docs, tests, or runtime Bolt/SocketModeReceiver construction. The related PR #41473 is still only an open implementation path in the provided GitHub context, not shipped behavior on main.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands on main. The best fix is to add a supported `channels.slack.socketMode` config surface for `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled`, wire it through the Slack plugin runtime, cover account inheritance and strict schema behavior, and update Slack docs/help/UI metadata and focused tests.
