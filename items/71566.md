## Summary

Do not auto-close PR #71566. Current main has channel directory adapters plus CLI/docs for self, peers, and groups, but it does not expose the requested `directory.self`, `directory.peers.list`, or `directory.groups.list` gateway methods. The PR is recent, focused on a plausible missing gateway surface, and the related #61744 report is about WhatsApp directory/reconnect behavior rather than a canonical duplicate for these gateway RPC methods.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused core gateway implementation that exposes `directory.self`, `directory.peers.list`, and `directory.groups.list` through the existing `ChannelDirectoryAdapter`, registers method names and read scopes, preserves cache-first list behavior, maps plugin runtime failures to `UNAVAILABLE`, and adds targeted gateway handler/method-scope tests plus public protocol/docs notes if these methods are intended as stable RPC surface.
