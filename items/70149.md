## Summary

Keep open. Current main still routes the WhatsApp Baileys WebSocket `agent` through the generic ambient `proxy-agent` path, which is the exact path the reporter says times out while an explicit `HttpsProxyAgent` succeeds. The related #61825 context is useful, but it only supports that ambient proxy wiring exists; it does not prove this narrower ProxyAgent-vs-HttpsProxyAgent failure is fixed.

## Best Possible Solution

Keep #70149 open. Implement a WhatsApp WebSocket-specific proxy resolver that parses `HTTPS_PROXY`/`HTTP_PROXY` with lowercase precedence and `NO_PROXY` matching for `web.whatsapp.com`, instantiates `HttpsProxyAgent` for Baileys `agent`, keeps the undici dispatcher path for media `fetchAgent`, and adds regression coverage for the reported explicit-agent behavior.
