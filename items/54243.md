## Summary

Keep #54243 open. Current main still makes `--no-include-workspace` only suppress workspace discovery/separate workspace assets; it does not filter nested workspace directories out of the archived `~/.openclaw` state tree. The CLI/docs still describe the flag as excluding workspace files, and the broader exclude-pattern request #40786 remains open rather than shipped.

## Best Possible Solution

Keep the issue open and fix the backup contract directly. The best product/code path is to make `--no-include-workspace` materially exclude resolved workspace directories even when they sit under the state dir, likely by adding a canonical nested-exclude filter to archive creation plus manifest/test coverage. If maintainers decide the state asset must remain indivisible, then the CLI help and docs should be changed to say the flag only skips workspace discovery/separate workspace roots, while #40786 continues to track general exclude patterns.
