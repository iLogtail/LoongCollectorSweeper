## Summary

Current main still matches the reported Twitch lifecycle bug. `twitchPlugin.gateway.startAccount` awaits `monitorTwitchProvider`, but that monitor connects/registers the handler and immediately returns `{ stop }`; the gateway supervisor treats any resolved `startAccount` as an exited channel and schedules auto-restart. Related fixed passive plugins use `runStoppablePassiveMonitor`/`runPassiveAccountLifecycle` to keep `startAccount` pending until abort, and Twitch lacks equivalent regression coverage.

## Best Possible Solution

Keep this issue open and fix Twitch in core. The likely patch is to wrap `monitorTwitchProvider` with `runStoppablePassiveMonitor` or `runPassiveAccountLifecycle` in `extensions/twitch/src/plugin.ts` so `startAccount` stays pending until `ctx.abortSignal` fires and calls `stop()`. Add a Twitch lifecycle regression test using `test/helpers/plugins/start-account-lifecycle.ts` to assert the task is unsettled after monitor startup and settles only after abort. A changelog fix entry is appropriate because this is user-visible bundled plugin behavior.
