## Summary

Keep open. Current main still uses the broad 450px near-bottom threshold with no explicit manual-scroll release lock or wheel-intent handler, so the PR's central chat-scroll behavior is not implemented. The related #70105 is scoped to the debug event log and does not supersede this chat auto-follow PR.

## Best Possible Solution

Keep #69520 open for maintainer review. The best path is to merge or adapt its explicit manual-scroll release lock, wheel-intent handling, bottom-only reacquire rule, nested-scrollable guard, and focused app-scroll tests, or land an equivalent implementation on main.
