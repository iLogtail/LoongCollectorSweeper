## Summary

Keep #61709 open. Current main has per-agent auth login, paste-token, and auth-order controls, and runtime auth scheduling state now lives in per-agent `auth-state.json`, but there is still no supported host-wide atomic auth-profile switch or migration across all agent stores.

## Best Possible Solution

Keep the issue open and implement a first-class all-agent auth switch/migration path. The likely product shape is an extension of `openclaw models auth order set` with `--all-agents` or a new `openclaw auth set-active/migrate` command that discovers configured agent dirs, validates the target provider/profile in each affected store before writing, updates current `auth-state.json` order/lastGood semantics rather than stale `auth-profiles.json` scheduling fields, supports dry-run/reporting, and has focused tests plus docs for multi-agent auth rotation.
