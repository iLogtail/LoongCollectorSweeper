## Summary

Current `main` exposes plugin hook seams around tool calls, but it does not have a native `tools.write.qualityGates` config surface, and the public external-plugin path still cannot rewrite tool results before the model sees them. That means the request is not implemented on `main`, and there is not enough evidence to close it as a ClawHub/plugin-only item with today’s public seams.

## Evidence

- **Core direction: prefer plugins, extend plugin API when needed:** The project vision says optional capability should usually ship as plugins, but if a useful feature cannot be built as a plugin yet, the preferred path is extending the plugin API rather than adding one-off core behavior.
