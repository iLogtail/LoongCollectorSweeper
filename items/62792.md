## Summary

Keep #62792 open. Current main has partial Slack media fixes, but it still does not implement the PR's central remaining behavior: inbound Slack files with only an id are not hydrated through `files.info`, media reads have no alternate token fallback in the inbound path, and sandbox staging still reuses the original basename across turns.

## Best Possible Solution

Keep the PR or its linked bug path open, rebase it onto current main, and narrow it to the still-missing Slack inbound behavior: hydrate id-only/partial Slack file payloads via a runtime-aware `files.info` path, retain safe token fallback semantics for media reads, preserve forwarded/thread-root attachment hydration where intended, and add focused regression coverage for inbound partial metadata and repeated-basename sandbox staging.
