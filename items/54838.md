## Summary

Do not close this PR. Current main has lower-level, stateful `autoExpandToolCalls` plumbing, but Control UI still exposes no persisted expand-by-default setting or toolbar toggle, and `renderApp` hardcodes the prop to `false`. The PR addresses a real remaining UX gap, but it needs a rebase/rework onto the current stateful expansion path and should address the existing review comments before merge.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase it. The best implementation is to add a persisted `chatExpandToolCalls` UI setting and desktop/mobile toolbar control, wire it to `renderChat({ autoExpandToolCalls })`, reuse `syncToolCardExpansionState` so manual expansions survive rerenders, align disabled/tooltip behavior with onboarding and hidden tool calls, and add focused storage/render tests. If maintainers prefer not to take this PR, open or restore a narrow canonical issue for the remaining Control UI toggle work before closing it.
