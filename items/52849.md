## Summary

Keep open. Current main has a bundled-plugin-only tool-result middleware that can rewrite results before the model sees them, but it does not provide the requested public typed hook or external plugin seam for same-turn tool-result mutation/blocking.

## Evidence

- **Requested hook absent from public hook names:** The typed hook union and hook-name list include `before_tool_call`, `after_tool_call`, `tool_result_persist`, and `before_message_write`, but no `before_tool_result_emit` or equivalent typed hook name.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/plugins/hook-types.ts#L57)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '45,130p;720,780p'`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Existing hook contracts do not satisfy live mutation:** `after_tool_call` is a void observation hook, while `tool_result_persist` returns only a synchronous persisted-message replacement, matching the report's concern that these are not a same-turn live result transform hook.
  - file: [src/plugins/hook-types.ts:753](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/plugins/hook-types.ts#L753)

## Risks / Open Questions

- Current main partially solves this for bundled plugins such as tokenjuice, so any future fix needs an explicit trust/security policy for third-party or self-hosted plugins that can rewrite high-trust tool output before model consumption.
