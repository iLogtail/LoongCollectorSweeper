## Summary

Keep open. Current main has a same-turn tool-result middleware path, but it is intentionally limited to bundled plugins. The requested public typed hook or external plugin seam for post-execution, pre-model tool-result mutation/blocking is still absent, and docs/tests preserve that limitation.

## Best Possible Solution

Keep #52849 open for a core/plugin API design. The best path is either a documented, trust-gated public hook or a deliberately expanded `registerAgentToolResultMiddleware` contract that supports async post-execution/pre-model result replacement, structured blocking, metadata/provenance, failure semantics, ordering relative to `after_tool_call` and `tool_result_persist`, and runtime tests proving raw tool output does not reach model-visible content when transformed or blocked.
