## Summary

Keep PR #61046 open. Current main still has the bare-model provider resolution gap in `openclaw models list`, so the PR is not obsolete or implemented on main. The PR also has an unresolved review concern about alias keys, so the best maintainer path is to revise/validate the fix rather than close it.

## Best Possible Solution

Keep the PR open or replace it with a maintainer patch that resolves bare model refs in `resolveConfiguredEntries()` through the existing unique-provider inference path, keeps alias indexing aligned with the inferred provider, and adds focused tests for bare configured keys, bare fallback values, ambiguous duplicate model ids, explicit prefixed refs, and aliases.
