## Summary

Keep open. Current main still has the exact followup-drain gap described by this PR: queued followups retain the triggering message id, and `applyReplyThreading` can use `currentMessageId`, but `resolveFollowupDeliveryPayloads` neither accepts nor forwards it and `createFollowupRunner` does not pass `queued.messageId`. The linked issue #54456 was closed as superseded by this open PR, so this PR remains the canonical implementation candidate rather than an obsolete item.

## Best Possible Solution

Keep this PR open and either land it or apply an equivalent maintainer patch: add `currentMessageId` to `resolveFollowupDeliveryPayloads`, pass `queued.messageId` from `createFollowupRunner`, forward it into `applyReplyThreading`, and add regression coverage in `followup-delivery.test.ts` or `followup-runner.test.ts` for a drained followup producing `replyToId` under `replyToMode: "all"`.
