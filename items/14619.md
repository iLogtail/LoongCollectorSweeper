## Summary

Keep #14619 open. Current main still builds a human-readable `## Tooling` section from active tool names/summaries and separately sends structured tool definitions with descriptions and schemas. I found no scoped `skipToolListText` or equivalent agent/default config; the only nearby user-facing control is the broad `systemPromptOverride`, which replaces the whole generated prompt rather than omitting only the duplicate tool-list text.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that suppresses only the human-readable Tooling section while preserving callable tool schemas. A complete fix should update the prompt builder, embedded runner plumbing if needed, config types/zod/generated schema/help/docs, `/context` reporting, and focused tests proving schemas remain available when the in-band list is omitted.
