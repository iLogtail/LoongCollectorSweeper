## Summary

Keep #14619 open. Current main still builds a human-readable Tooling list from active runtime tools while also serializing the same tools as callable schemas, and I found no scoped `skipToolListText`-style config. The existing `systemPromptOverride` and `promptMode` paths remove or replace much more than the requested tool-list text. The prior maintainer-side Codex comment already kept this open, and there were no relevant path changes from that reviewed SHA to current main.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default setting, such as `agents.defaults.skipToolListText`, that suppresses only the human-readable Tooling list while preserving structured callable tool schemas. The implementation should update prompt building and run plumbing, config types/zod/generated schema/help/docs, `/context` accounting, and focused tests proving the prompt omission and continued tool availability.
