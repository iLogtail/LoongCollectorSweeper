## Summary

Keep #57077 open. Current main has the related #59611 fixes for preserving user-selected sessions across reconnect/tab navigation and stale `chat.history` responses, but it does not contain this PR’s specific session-picker hardening: the desktop and mobile pickers still rely on `.value` plus `?selected`, the mobile picker still shares `chat-controls__session`, and the fallback-rerender regression tests described in #57077 are still absent.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to either rebase the narrow picker hardening onto `ui/src/ui/chat/session-controls.ts` and add focused desktop/mobile fallback-rerender regression tests, or have a maintainer explicitly close it as superseded by #59611 only after deciding current `.value` plus `?selected` behavior is sufficient.
