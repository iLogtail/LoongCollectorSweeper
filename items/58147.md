## Summary

Keep open. Current `main` does not plumb `agents.defaults.timeFormat` into Control UI/WebChat, and existing UI session/chat timestamp rendering still relies on browser or platform-local formatting rather than the agent config.

## Evidence

- **Config contract scopes `timeFormat` to prompts, not UI:** `agents.defaults.timeFormat` is documented in code as a system-prompt setting, and the docs repeat that it controls prompt time format rather than UI/message timestamps.
  - file: [src/config/types.agent-defaults.ts:255](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/types.agent-defaults.ts#L255)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '248,258p'; nl -ba docs/gateway/config-agents.md | sed -n '267,286p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Control UI bootstrap does not expose any time-format preference:** The Control UI bootstrap contract only carries base path, assistant identity, version, media roots, and embed settings. The server response and browser bootstrap loader likewise have no `timeFormat` field, so the browser UI has no config path to honor `agents.defaults.timeFormat`.
  - file: [src/gateway/control-ui-contract.ts:5](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/control-ui-contract.ts#L5)
  - command: `nl -ba src/gateway/control-ui-contract.ts | sed -n '1,40p'; nl -ba src/gateway/control-ui.ts | sed -n '746,765p'; nl -ba ui/src/ui/controllers/control-ui-bootstrap.ts | sed -n '1,90p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Control UI chat timestamps still use browser locale formatting:** Chat message and streaming timestamps are rendered with `new Date(...).toLocaleTimeString(...)`, which follows browser/runtime locale behavior and does not consult any OpenClaw time-format setting.
  - file: [ui/src/ui/chat/grouped-render.ts:241](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/ui/src/ui/chat/grouped-render.ts#L241)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '235,330p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **WebChat session list still uses platform-local shortened time formatting:** The shared SwiftUI WebChat sessions sheet formats session update times with `Date.formatted(date: .abbreviated, time: .shortened)`, again with no linkage to `agents.defaults.timeFormat`.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatSheets.swift:20](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatSheets.swift#L20)
