## Summary

Keep #56498 open. The report is recent, from a supported Hostinger setup path, and describes a combined WhatsApp/Telegram connection failure plus grey Save state that current main does not conclusively prove fixed. Current main has channel setup, QR login, Telegram network/polling improvements, and troubleshooting docs, but those only cover parts of the symptom and do not close the no-error/save-disabled Hostinger workflow described by the reporter and commenter.

## Best Possible Solution

Keep the issue open and ask for a focused Hostinger reproduction: current OpenClaw version, exact Hostinger 1-Click/VPS path, screenshots of the channel setup screen with the grey Save button, browser console errors, gateway logs around `config.set`, `web.login.start`/`web.login.wait`, and `channels.status --probe` for WhatsApp and Telegram. If reproduced, the likely fix belongs in the Control UI channel setup flow: show why Save is disabled, surface config validation and web-login failures inline, and document any Hostinger-specific provisioning workaround in the Hostinger/channel troubleshooting docs.
