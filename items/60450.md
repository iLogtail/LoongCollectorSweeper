## Summary

Keep open. Current main has partial failover safety, including a finite per-run retry loop and finite fallback candidate chain, but it does not implement the issue’s central requests: a global spend/rate guard, an all-provider auth-failure pause/alert, or a 24h default billing lockout. The related #13615 context also remains open for broader outbound API rate limiting rather than superseding this incident.

## Best Possible Solution

Keep #60450 open as a core reliability/safety incident. The best path is a cross-provider failure and spend circuit breaker at model-call boundaries: track repeated auth/billing failures across profiles/providers over a rolling window, pause further model calls for affected agents or providers, surface an immediate user-facing alert with recovery steps, add config/docs/tests for the policy, and coordinate any generic outbound request/token budget work with the still-open #13615.
