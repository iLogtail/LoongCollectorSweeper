## Summary

Keep PR #57308 open. Current main still has the liveness gap the PR is trying to fix: Slack block-action and modal lifecycle handlers enqueue session-scoped system events but do not request a heartbeat wake for the same session. The PR is also the rebased port of #56766, so it should not be closed as a duplicate of the older original without maintainer direction; instead it needs a small update for the current `infra-runtime` import surface.

## Best Possible Solution

Keep this PR or a refreshed successor open, update it against current main by using the current `openclaw/plugin-sdk/infra-runtime` surface for both `enqueueSystemEvent` and `requestHeartbeatNow`, and extend `extensions/slack/src/monitor/events/interactions.test.ts` to assert that successful block actions, modal submissions, and modal closes request an `exec-event` heartbeat for the same routed session only when `enqueueSystemEvent` returns true. After maintainers pick #57308 as canonical, close or supersede the older #56766 to avoid duplicate review.
