## Summary

Keep open. Current `main` still queues Slack interaction system events without requesting a heartbeat wake, so the liveness gap described in PR #57308 is not implemented on `main`. The PR is also stale against current plugin SDK seams, because Slack interaction code now imports `openclaw/plugin-sdk/infra-runtime` rather than the deprecated `channel-runtime` shim.
