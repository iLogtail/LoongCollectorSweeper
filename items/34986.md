## Summary

Keep #34986 open. Current main still lacks a configurable heartbeat pre-flight script/command that can return HEARTBEAT_OK and skip agent session creation; the linked implementation PR #35504 was closed unmerged, and current cost-saving paths are built-in heartbeat skips rather than arbitrary external checks.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete fix would add an optional heartbeat pre-flight hook such as `agents.defaults.heartbeat.preFlightCommand`, merge it through defaults and per-agent heartbeat config, wire exported types, strict Zod schema, labels/help/generated docs, and execute it in `runHeartbeatOnce` before session/model work. Tests should cover HEARTBEAT_OK skip, non-OK output, command failure, timeout, unset command, default/per-agent merging, and unchanged existing heartbeat behavior.
