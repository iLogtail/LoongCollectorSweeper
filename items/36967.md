## Summary

Keep open. Current main still does not solve the central Feishu behavior in PR #36967: `/allowlist channel=feishu` is not resolved through the plugin-aware channel path, the Feishu plugin still lacks the shared config-backed allowlist adapter, Feishu is not marked markdown-capable in metadata, and Feishu envelope headers are not recognized. Some PR chunks are now superseded by generic main behavior, so the right maintainer path is a narrower rework rather than closure.

## Best Possible Solution

Keep PR #36967 open for maintainer/contributor follow-up, but narrow it before merge: add a Feishu allowlist adapter through openclaw/plugin-sdk/allowlist-config-edit with merged-account semantics, mark Feishu markdown-capable in both runtime/package metadata, add Feishu envelope recognition and regression tests, and drop the now-obsolete hook-schema/security-fix chunks already handled on main.
