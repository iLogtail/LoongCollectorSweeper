## Summary

Keep PR #36967 open. Current main supersedes some supporting chunks, especially generic security-fix handling and open-ended hook channel config, but the central Feishu work is still not implemented: Feishu has no config-backed /allowlist adapter and is not declared markdown-capable. The existing PR also has unresolved review concerns around multi-account inherited allowlists/default-account semantics, so the best path is a narrower rework rather than cleanup closure.

## Best Possible Solution

Keep PR #36967 open for a narrower rework: add Feishu-owned allowlist support in extensions/feishu/src/channel.ts using buildDmGroupAccountAllowlistAdapter or a small custom wrapper that reads effective merged Feishu account config and avoids dropping inherited allowFrom/groupAllowFrom entries; declare Feishu markdownCapable in both package/channel metadata as appropriate; add focused Feishu /allowlist channel=feishu and markdown-capability regression tests; remove the security-fix and hook-schema chunks already handled generically on main; address the existing review comments before merge.
