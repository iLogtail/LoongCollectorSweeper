## Summary

Keep #71257 open. Current main still returns raw session transcript text and writes it directly into session-memory files, so the PR is not obsolete or implemented on main. The existing sanitizer-like code is scoped to chat history/display surfaces, not the bundled session-memory persistence path. The PR remains useful, but it is not merge-ready as-is because the branch includes generated `.artifacts/tsgo-cache/*.tsbuildinfo` files and should be reconciled with overlapping open PR #69961 plus the existing review feedback.

## Best Possible Solution

Keep the PR open and finish a focused session-memory fix on the transcript extraction path: strip model artifacts before memory persistence, use existing directive/media parsing helpers where practical, add regression tests for `NO_REPLY`, chat-template tokens, tool-call XML, double-bracket directives, `MEDIA:` lines, and code-block whitespace, remove generated `.artifacts` files, then reconcile or supersede overlapping PR #69961.
