## Summary

Keep #55484 open. Current main does not implement the requested upstream contract: `sessions_spawn(runtime="acp")` only accepts `mode: "run" | "session"`, `mode: "session"` still requires `thread: true`, ACP spawn session keys are UUID-based rather than affinity-keyed, and there is no `affinityKey` surface. The issue comments are useful evidence of a downstream implementation, but they explicitly say the upstream item should remain open until OpenClaw main ships the same capability.

## Best Possible Solution

Keep the issue open and implement the upstream ACP spawn contract directly: add `sessions_spawn({ runtime: "acp", mode: "persistent", affinityKey, agentId })`, define requester eligibility for cron/hook/orchestrator sessions, derive deterministic ACP session keys scoped by requester owner identity plus backend/cwd/affinity key, preserve existing affinity-backed sessions on dispatch failure, and cover the new contract in `src/agents/acp-spawn.test.ts`, `src/agents/tools/sessions-spawn-tool.test.ts`, system-prompt/docs updates, and any cron/orchestrator integration tests.
