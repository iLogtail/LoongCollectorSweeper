## Summary

Keep #65557 open. Current main has useful pieces of the requested surface, including per-session model overrides and a global configured model allowlist, but it does not implement the central request: admin-controlled provider/model allowlists and defaults scoped per account/user with server-side enforcement.

## Best Possible Solution

Keep the issue open and scope it as a core policy plus UI feature: define what “account” means across Control UI trusted-proxy users, channel accounts, and sessions; add config schema/types for global fallback plus per-account provider/model allowlists/defaults; enforce that policy in `models.list`, `sessions.patch`, chat/OpenAI-compatible model overrides, and runtime execution; then add Control UI admin settings and a chat/session selector filtered by the effective policy, with docs and regression tests.
