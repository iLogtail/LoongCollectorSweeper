## Summary

Keep #20173 open. Current main still handles Discord inbound user turns through Carbon's MessageCreateListener path only: OpenClaw does not import/register MessageUpdateListener, the strict Discord config schema and exported config types have no handleEdits/editDebounceMs surface, and the Discord docs only document create-message listener/worker controls. The request is concrete Discord bundled-channel work, not a ClawHub close.

## Best Possible Solution

Implement this in the bundled Discord plugin: add an opt-in config/type/schema/docs surface for edited-message handling, register a Carbon MessageUpdateListener, normalize update payloads through the existing Discord preflight and inbound worker path, preserve bot/self/access filters, debounce/rate-limit per edited message, define in-flight response behavior, and add listener/startup/config/message-handler regression tests.
