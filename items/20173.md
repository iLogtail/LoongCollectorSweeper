## Summary

Current main still does not implement Discord edited-message reprocessing. The Discord plugin only wires inbound agent turns through the Carbon MessageCreate listener, startup registration omits MessageUpdate, config/schema/types/docs lack handleEdits/editDebounceMs, and the existing replay key would suppress same-message reprocessing unless changed.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/UI-doc coverage for edited-message handling, register a Carbon MessageUpdate listener, normalize edited payloads through the existing Discord preflight and inbound worker pipeline, preserve bot/self/access filters, make the replay guard edit-aware, add debounce or rate limiting per message, define in-flight cancellation/reply semantics, and cover listener registration, config validation, and message-handler behavior with regression tests.
