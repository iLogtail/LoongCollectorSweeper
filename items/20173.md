## Summary

Keep #20173 open. Current main still handles Discord inbound user turns through the MessageCreateListener path only; there is no MESSAGE_UPDATE/messageUpdate listener, no handleEdits/editDebounceMs config surface, and no docs or tests for edited-message reprocessing. A maintainer-side Codex comment on 2026-04-26 already kept this open, and the same gaps remain on d9c5040fc5bc35b3b48be0086697915dfb079de0.

## Best Possible Solution

Implement this in the bundled Discord plugin: add an opt-in config/type/schema/docs surface for edited-message handling, register a Carbon/Discord gateway listener for MESSAGE_UPDATE, normalize edited payloads through the existing Discord preflight and inbound worker path, preserve bot/self/access filters, use edit-aware replay/debounce/rate limiting, define in-flight response behavior, and add listener/startup/config/message-handler regression tests.
