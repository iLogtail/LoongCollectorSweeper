## Summary

Keep open. Current main still lacks the central Telegram-sourced ambiguity fix: unbound Telegram approval requests can match every eligible Telegram account. The PR also has material review feedback about regressing `approvals.exec.mode: "both"`, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a revision that preserves `both` mode session delivery while adding a Telegram-sourced no-bound-account ambiguity guard and target-account scoping for approver-DM delivery. The regression suite should cover multi-account Telegram-sourced unbound requests, explicit `turnSourceAccountId`, `targets[].accountId` delivery, and `both` mode with non-Telegram targets.
