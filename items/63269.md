## Summary

Keep this issue open. Current main still supports the nominal Mattermost WebSocket `posted` path and channel mention gating, but I did not find a shipped/current-main runtime fix for the reported 2026.4.x symptom where DMs work and public/group channel posts silently do not arrive. The related fresh report #72282 was already closed as a duplicate of this canonical investigation, not as proof that the bug is fixed.

## Best Possible Solution

Keep this issue as the canonical Mattermost channel-inbound regression tracker. Reproduce on current main/latest release with the reporter's `oncall`, `groupPolicy: "open"`, `blockStreaming`, and `replyToMode: "off"` config; add a focused regression test for a public-channel `posted` WebSocket frame with an @mention reaching dispatch; then fix either the WebSocket receive/diagnostic path or the access/mention/routing drop that reproduction identifies.
