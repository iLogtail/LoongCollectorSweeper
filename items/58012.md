## Summary

Issue should stay open. Current `main` still applies `strict9` only in the native `mistral` provider path; proxy providers like `openrouter` and `together` do not route Mistral-family models to a `strict9` replay policy, so the reported regression remains plausible and is not implemented on `main`.

## Evidence

- **Transcript policy prefers provider-owned replay hooks over generic fallbacks:** `resolveTranscriptPolicy()` uses the provider runtime plugin's `buildReplayPolicy()` when present, and only falls back to transport defaults otherwise. That means OpenRouter's plugin policy fully overrides any model-id fallback path.
