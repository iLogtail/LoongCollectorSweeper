## Summary

Keep open. Current `main` still applies Mistral `strict9` replay policy only through the native `mistral` provider hook; proxy-provider paths on `openrouter` and similar OpenAI-compatible routes do not inherit that policy, and the current tests only cover native Mistral.

## Evidence

- **Transcript policy is chosen by matched provider hook only:** `resolveTranscriptPolicy()` resolves the runtime plugin from `provider`, uses that plugin's `buildReplayPolicy()` when present, and otherwise falls back to a generic transport policy. The generic OpenAI-compatible fallback hard-codes `toolCallIdMode: "strict"`, not `"strict9"`.
  - file: [src/agents/transcript-policy.ts:74](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/transcript-policy.ts#L74)

  - file: [extensions/mistral/index.ts:11](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/extensions/mistral/index.ts#L11)
