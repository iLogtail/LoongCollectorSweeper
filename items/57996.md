## Summary

Issue should stay open. Current `main` still creates an isolated QMD home and SQLite index per agent, documents that per-agent layout, and does not expose OpenClaw-level controls for a shared QMD cache, WAL/checkpoint tuning, vacuum/compaction, or a reduced-write mode.

## Risks / Open Questions

- This was a read-only code review, so I did not run a long-lived multi-agent workload to quantify the reported 34 GB/day write rate or reproduce the macOS crashes directly.
- Some write amplification may come from QMD/SQLite behavior outside OpenClaw's code, so the exact eventual fix may require coordination with QMD as well as OpenClaw changes.
