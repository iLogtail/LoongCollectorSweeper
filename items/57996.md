## Summary

Keep open. Current `main` and the latest shipped release still implement QMD as a separate per-agent home and SQLite index under `agents/<agentId>/qmd/xdg-cache/qmd/index.sqlite`, with boot-time and periodic `qmd update`/`qmd embed` runs. The exposed config only covers command/search/scope/session/update cadence and prompt-size limits; there is no OpenClaw-side shared-cache mode, WAL/vacuum/checkpoint maintenance, or index-size eviction control that would resolve the reported multi-agent disk-I/O pressure.
