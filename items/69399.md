## Summary

Keep PR #69399 open. Current main still only surfaces model fallback and fallback-cleared notices when verbose mode is enabled, and the config/type/schema/docs surface does not contain `agents.defaults.notifyOnModelFallback`. The PR is a focused implementation of the linked open feature request #67197, and its second commit addresses the bot-noted lifecycle coverage gap rather than making the PR obsolete.

## Best Possible Solution

Keep #69399 open for maintainer review. The best product path is to land a narrow opt-in config such as `agents.defaults.notifyOnModelFallback`, wire it into `runReplyAgent` alongside verbose mode for fallback and fallback-cleared notices, and keep the config schema/docs plus once-per-state lifecycle tests aligned. If maintainers prefer a different config location or naming, that should be resolved on this PR rather than closing it as obsolete.
