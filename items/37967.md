## Summary

Keep open. Current `main` added Feishu sender-scoped session isolation and DM-only dynamic agent creation, but it does not implement the issue’s requested automatic per-group-sender agent/subagent provisioning with isolated per-user skills/config in Feishu group chats.

## Evidence

- **DM-only dynamic agent feature:** `DynamicAgentCreationSchema` is documented as creating a new agent for each unique DM user, not group senders.
  - file: [extensions/feishu/src/config-schema.ts:72](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/feishu/src/config-schema.ts#L72)

- **Runtime gate excludes groups:** Feishu auto-agent creation only runs when `!isGroup`; group messages never enter `maybeCreateDynamicAgent`.
  - file: [extensions/feishu/src/bot.ts:601](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/extensions/feishu/src/bot.ts#L601)

## Risks / Open Questions

- Feishu sender-scoped sessions on `main` may already address part of the reporter’s context-isolation goal, so future triage should separate memory/session isolation from full per-user agent provisioning.
- There is Feishu-specific subagent hook code in `extensions/feishu/src/subagent-hooks.ts`, but the documented and wired user-facing behavior still does not show automatic per-group-user agent creation with isolated skills/config.
