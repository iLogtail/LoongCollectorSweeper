## Summary

Keep PR #41696 open. Current main still parses a bare numeric Discord allowlist entry as a channel ID before any guild lookup, and the current regression test explicitly documents that a matching guild ID like `999` remains unresolved unless prefixed as `guild:999`. The PR is a focused bug fix with useful review discussion; it should be ported/rebased onto the current `extensions/discord` path rather than closed as obsolete.

## Best Possible Solution

Keep the PR or its equivalent work open, port the guild-first bare-numeric behavior into `extensions/discord/src/resolve-channels.ts`, preserve explicit `channel:` and `guild:` semantics plus channel fallback, and update the colocated resolver tests to cover guild-first and fallback cases before merging.
