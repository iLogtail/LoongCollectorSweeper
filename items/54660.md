## Summary

Keep PR #54660 open. Current main still contains the broken Docker troubleshooting example exactly as described: it passes `devices list --url ws://127.0.0.1:18789` without explicit `--token` or `--password`. The underlying Gateway client intentionally rejects CLI URL overrides unless credentials are passed explicitly, and tests cover that behavior. The PR is a focused docs fix that remains useful rather than implemented, obsolete, or external to the repo.

## Best Possible Solution

Keep the PR open or apply an equivalent docs update to `docs/install/docker.md`: change the Docker troubleshooting command to pass an explicit gateway token or password and add a short note that `--url` overrides do not reuse config or environment credentials. No core behavior change is needed because the source and tests already enforce the intended auth policy.
