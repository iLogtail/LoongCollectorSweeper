## Summary

Keep open. Current main still contains the Docker troubleshooting command that PR #54660 is trying to fix, and the current gateway auth implementation/tests confirm that an explicit CLI `--url` must not reuse config/env credentials. Existing general CLI docs explain the contract, but they do not fix the Docker-specific snippet users follow in `docs/install/docker.md`.

## Best Possible Solution

Keep PR #54660 open or apply its narrow docs change: update `docs/install/docker.md` so the Docker troubleshooting `devices list --url ...` example passes an explicit gateway token/password, ideally using the `.env` token generated by Docker setup, and add the local note that `--url` overrides require explicit credentials.
