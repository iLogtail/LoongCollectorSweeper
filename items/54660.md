## Summary

Keep open. Current `main` still has the Docker troubleshooting command without explicit auth, while the current gateway call path still rejects CLI `--url` overrides unless `--token` or `--password` is passed. This PR's docs fix is not already implemented on `main`.

## Evidence

- **Docker doc on main still shows the unauthenticated override command:** `docs/install/docker.md` still tells users to run `docker compose run --rm openclaw-cli devices list --url ws://127.0.0.1:18789` with no explicit credential, which is the exact snippet this PR proposes to fix.
  - file: [docs/install/docker.md:393](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/install/docker.md#L393)
  - command: `nl -ba docs/install/docker.md | sed -n '387,396p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Current code still requires explicit credentials for CLI `--url` overrides:** `src/gateway/call.ts` returns early only when a CLI URL override has explicit token/password; otherwise it throws `gateway url override requires explicit credentials`. That matches the PR rationale and means the current Docker doc remains inaccurate.
  - file: [src/gateway/call.ts:252](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/gateway/call.ts#L252)
  - command: `nl -ba src/gateway/call.ts | sed -n '231,258p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **The auth rule is already documented elsewhere, but not on the Docker page:** Current docs already say CLI `--url` does not auto-apply config/environment credentials in `docs/cli/logs.md`, and `docs/gateway/remote.md` says CLI URL overrides never reuse implicit config/env credentials. That reinforces that the Docker troubleshooting page is the stale surface this PR addresses.
  - file: [docs/cli/logs.md:38](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/cli/logs.md#L38)
