## Summary

Keep open. Current `main` still does not implement the legacy queued-entry normalization this PR is about, and the PR itself picked up unrelated gateway changes with unresolved blocking review feedback.

## Evidence

- **Current loader only parses raw JSON and only backfills `lastAttemptAt`:** `readQueueEntry` casts on-disk JSON straight to `QueuedDelivery`, and `normalizeLegacyQueuedDeliveryEntry` only fills `lastAttemptAt` from `enqueuedAt`. There is no current-main remap for legacy fields like `payload` -> `payloads`, `attempt` -> `retryCount`, or `createdAt` -> `enqueuedAt`, so the outbound recovery fix described in the PR is not already present on `main`.
  - file: [src/infra/outbound/delivery-queue-storage.ts:94](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/infra/outbound/delivery-queue-storage.ts#L94)

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Failure bookkeeping still assumes canonical queue entries:** `failDelivery` reads the stored entry and immediately does `entry.retryCount += 1` before any legacy normalization. That means the PR's stated fix for normalizing legacy entries before retry bookkeeping is not implemented on current `main`.
  - file: [src/infra/outbound/delivery-queue-storage.ts:192](https://github.com/openclaw/openclaw/blob/bc73141e8284c219933970dbd4f6aeacb05bbb77/src/infra/outbound/delivery-queue-storage.ts#L192)
