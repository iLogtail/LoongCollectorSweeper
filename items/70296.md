## Summary

Keep #70296 open. Current main still has no `sentinel.suppressChannelNotify`-style config surface, and the restart-sentinel startup path still formats the sentinel message, enqueues a wake, and delivers the restart notice to any resolved channel route. The Control UI model read-back portion appears addressed on current main, but the bundled restart-notification and BOOT.md footer behavior remain actionable.

## Best Possible Solution

Keep the issue open, preferably split into targeted follow-ups: add a documented restart-sentinel channel-notification policy/config key and guard the outbound notice path in `src/gateway/server-restart-sentinel.ts`, then make an explicit product decision for BOOT.md silent-token handling. Preserve the current Control UI read-back behavior/tests unless a fresh reproduction shows a remaining UI regression.
