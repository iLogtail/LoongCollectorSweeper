## Summary

Keep #70296 open. Current main appears to handle the Control UI model read-back path for string and object-shaped model values, but the issue bundles two restart/boot behaviors that still remain: restart sentinel notices are still delivered to routed channels with no suppress config, and BOOT.md prompts still get a hardcoded NO_REPLY footer appended after the workspace file content.

## Best Possible Solution

Keep the issue open or split it. The restart-sentinel path needs an explicit config/API decision that can suppress end-user channel notices while preserving internal system events, durable restart continuations, and diagnostics. The BOOT.md path needs a decision to remove the injected footer or align it with the documented silent-token contract. The Control UI model display portion can be separately verified as implemented if maintainers want a narrower closure.
