## Summary

Keep #69172 open. The central user problem is real on current main: Google/Gemini web search, image generation, and media understanding are still hardwired to native Google Generative AI request paths, and embeddings only honor explicit remote/config baseUrl, not the documented env aliases requested by the PR. However, the submitted PR is not merge-ready as-is because the provided review context shows unrelated/accidental files and P1/P2 correctness regressions in several touched surfaces.

## Best Possible Solution

Keep the item open, but ask for a narrower, mergeable PR: restore Google/Gemini base URL and apiType override support in the Google plugin with a shared resolver, cover web search, embeddings, image generation, and media understanding, add focused regression tests for env/config precedence and native-Google defaults, and remove unrelated changes to scripts, memory-host SDK, channel-entry contract, BytePlus/Volcengine/Anthropic, plus the accidental root files. If maintainers prefer not to support OpenAI-compatible proxying through the Google plugin tools, close only after opening or linking a canonical product/API decision item.
