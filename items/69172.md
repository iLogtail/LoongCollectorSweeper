## Summary

Keep open. Current main does not already implement the PR's central Gemini override behavior: web search is still hardcoded to the native Google endpoint, the Google plugin manifest exposes only webSearch apiKey/model, and image/media helpers explicitly reject non-Google Generative AI base URLs. The PR is not ready as-is because the provided review context shows unresolved P1 regressions and unrelated/dirty changes, but closing would discard a real missing bundled Google plugin capability rather than cleanup an obsolete item.

## Best Possible Solution

Keep the item open, but require a narrowed replacement patch or force-push that only touches the Google proxy override surface and its tests. The best implementation should add shared Google baseUrl/apiType resolution, wire it through web search, embeddings, image generation, and media understanding without breaking native googleapis.com behavior, preserve existing auth parsing and modality-specific payloads, remove unrelated files/changes, and include focused regression tests for config precedence, GEMINI_* env aliases, /openai normalization, and OpenAI-compatible request/response shapes.
