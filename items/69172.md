## Summary

Keep #69172 open. Current main does not implement the PR’s central behavior: Gemini web search is still hardwired to the native Google endpoint, image/media providers still reject non-Google Gemini base URLs through the shared helper, and embeddings do not support the requested env aliases or OpenAI-compatible request/response mode. The PR is not mergeable as-is because the provided review context flags unrelated edits and real regressions, but the underlying bug/feature request remains actionable in this repository.

## Best Possible Solution

Keep the item open as the active track for Gemini custom endpoint/OpenAI-compatible support, but require a narrowed replacement or substantial cleanup: keep changes inside the Google plugin unless a real SDK seam is missing, remove unrelated script/memory/channel changes and root artifacts, preserve native googleapis.com defaults, preserve OAuth and edit/media behavior, add shared baseUrl/apiType resolution, and cover web search, embeddings, image generation, and media understanding with focused tests.
