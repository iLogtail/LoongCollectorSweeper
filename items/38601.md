## Summary

Keep #38601 open. Current main still suppresses an explicit `pty: true` request whenever exec is routed through a sandbox, and I found no user-facing warning, error, docs note, or regression test that covers the requested sandbox PTY feedback. The provided GitHub context also shows the focused fix PR #39349 is still open/unmerged, while #46392 was closed unmerged.

## Best Possible Solution

Keep the issue open and land a narrow fix in the exec tool routing layer: when `params.pty === true && sandbox`, emit a clear warning or intentionally reject the request before `usePty` is suppressed. Add a focused regression test for sandboxed exec with `pty:true`, and update `docs/tools/exec.md` plus the exec tool schema/help text to state the sandbox PTY limitation.
