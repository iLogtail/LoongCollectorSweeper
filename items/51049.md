## Summary

Keep #51049 open. The report is recent, non-maintainer-authored, has no protected labels, and current main does not provide high-confidence evidence that the nested Docker -> k3s WhatsApp inbound failure is fixed or fully superseded. Current WhatsApp code handles standard inbound `notify` and recent `append` upserts and now supports ambient proxy configuration, but there is no current-main code, doc, or merged PR that specifically resolves the reported nested k3s/OpenShell network path.
