## Summary

Current main still lacks a per-instance Bonjour service-name config path. The bundled bonjour plugin always formats the gateway machine display name into `opts.instanceName`, the plugin manifest rejects any bonjour plugin config properties, and `OPENCLAW_MDNS_HOSTNAME` only affects the advertiser fallback/hostname path when `opts.instanceName` is absent. The issue is recent, contributor-authored, and an issue comment confirms a PR is being prepared rather than indicating the request is obsolete.

## Best Possible Solution

Keep the issue open. The likely maintainer-friendly fix is to add `instanceName` to `extensions/bonjour/openclaw.plugin.json` under the bonjour plugin `configSchema`, read and normalize it from `api.pluginConfig` in `extensions/bonjour/index.ts`, pass the configured value through the existing suffix/formatting semantics to `startGatewayBonjourAdvertiser`, and add focused tests plus docs under the public Bonjour/plugin configuration pages. A core `gateway.bonjour.*` key is not necessary unless maintainers intentionally want to move this out of plugin-owned config.
