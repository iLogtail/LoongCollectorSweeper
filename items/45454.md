## Summary

Keep open. Current `main` still appears to have the early-abort `responsePrefix` interpolation gap: reply-prefix context is only populated with model/provider/thinking metadata in `onModelSelected`, but fast-abort replies are emitted before that callback runs, and routed abort replies still normalize `responsePrefix` without any template context.

## Evidence

- **Reply prefix context is not pre-seeded from session metadata on main:** `createReplyPrefixContext` initializes only `identityName`; `model`, `modelFull`, `provider`, and `thinkingLevel` are set later only inside `onModelSelected`. There is no `sessionKey` parameter or session-store lookup in current `main`.

  - command: `nl -ba src/channels/reply-prefix.ts | sed -n '1,120p'`
