## Summary

Keep open. Current main has not implemented the PR's requested runtime behavior: the active `src/memory-host-sdk` default QMD scope still allows channel sessions when `memory.backend` is `qmd` and `memory.qmd.scope` is unset, and the colocated test still asserts that behavior. The PR is still a plausible targeted bug/security-hardening change, though it should reconcile docs and the divergent package mirror before merge.

## Best Possible Solution

Keep the PR open as a real, still-unshipped hardening change. The best path is to update the active `src/memory-host-sdk/host/backend-config.ts` default and source regression test as proposed, then align the QMD docs and reconcile the divergent `packages/memory-host-sdk` copy so the shipped runtime, package surface, and docs agree on the default scope.
