## Summary

Keep open. Current main still exposes the behavior this PR targets in the live `src/memory-host-sdk` path: the default QMD scope includes both direct and channel sessions, and the colocated regression test still asserts channel sessions are allowed by default. The private `packages/memory-host-sdk` copy is already direct-only, so this PR is not obsolete; it is a focused alignment/security-hardening change that still needs maintainer review and likely docs cleanup.

## Best Possible Solution

Keep PR #70297 open for maintainer review. If direct-only is the desired secure default, land a scoped fix in `src/memory-host-sdk/host/backend-config.ts` with the regression test change, then align QMD docs and confirm whether the existing direct-only `packages/memory-host-sdk` copy is intentional or should be covered by a mirror/contract test.
