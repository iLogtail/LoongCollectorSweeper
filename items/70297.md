## Summary

Keep open. Current `main` has not implemented #70297's direct-only QMD default: the active resolver still allows channel sessions when `memory.backend` is `qmd` and `memory.qmd.scope` is unset, and the colocated regression test still asserts that behavior. The PR remains a focused hardening fix; before merge, docs should be aligned because some QMD docs still describe the old direct+channel default.

## Best Possible Solution

Keep #70297 open as a valid focused bug/security-hardening PR. The best maintainer path is to merge the direct-only `DEFAULT_QMD_SCOPE` behavior and regression test, then update the QMD docs to state that channel QMD access requires an explicit `memory.qmd.scope` opt-in.
