## Summary

Current `main` has a coarse workaround for Claude CLI `--bare` via raw backend args, but it does not implement the requested first-class scoped surface (`slimMode`/`--bare`) for `infer`/`capability model run` or cron `agentTurn` payloads. The report is recent, concrete, and still describes a real gap.

## Evidence

- **Documented raw-args workaround exists:** The CLI backend docs explicitly say Claude-specific behavior can be forced with raw backend args under `agents.defaults.cliBackends.claude-cli.args` and matching `resumeArgs`, which is enough for a global `--bare` workaround.
  - file: [docs/gateway/cli-backends.md:175](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/gateway/cli-backends.md#L175)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Config schema supports global backend args/resumeArgs:** `CliBackendConfig` exposes `args` as base args for every invocation and `resumeArgs` for resumed sessions, so current main can carry custom Claude flags globally but only at backend-config scope.
  - file: [src/config/types.agent-defaults.ts:93](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/types.agent-defaults.ts#L93)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Infer/capability model-run surface has no slim/bare flag:** `model.run` advertises only `--prompt`, `--model`, `--local`, `--gateway`, and `--json`, and the registered command defines only those options. There is no first-class `--slim` or `--bare` CLI switch here.
  - file: [src/cli/capability-cli.ts:102](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/cli/capability-cli.ts#L102)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Model run uses the normal agent runtime:** The infer docs state `model run` reuses the agent runtime, which means the existing raw backend-args workaround would apply to one-shot infer runs; the docs also note `openclaw capability ...` is just an alias for `openclaw infer ...`.
  - file: [docs/cli/infer.md:148](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/cli/infer.md#L148)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Cron agentTurn payload has no slim-mode field:** The cron payload type allows `message`, `model`, `fallbacks`, `thinking`, `timeoutSeconds`, `allowUnsafeExternalContent`, `externalContentSource`, `lightContext`, and `toolsAllow`, but no `slimMode`/`bare` override.
  - file: [src/cron/types.ts:116](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/cron/types.ts#L116)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **No dedicated slim/bare symbol found in current source/docs:** A targeted search for `slimMode`, `--slim`, and `--bare` across current `src/cli`, `src`, and `docs` returned no matching first-class feature surface for this request.
  - command: `rg -n --glob '!**/*.test.ts' --glob '!extensions/diffs/assets/**' -- '--slim|--bare|slimMode' src/cli src docs`

## Risks / Open Questions

- The existing workaround is backend-global: adding `--bare` through `agents.defaults.cliBackends.claude-cli.args`/`resumeArgs` would also affect interactive or resumed Claude CLI runs, which does not satisfy the requested opt-in per-job/per-command behavior.
- Closing as implemented would discard a still-missing product surface: there is no first-class cron payload field or `infer`/`capability model-run` flag to scope slim mode to heavy-workspace automation only.
