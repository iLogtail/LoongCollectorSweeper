## Summary

Keep PR #70495 open. Current main still has the user-facing ambiguity this PR addresses: `openclaw mcp list/show` use generic MCP wording, while the implementation only reads OpenClaw config under `mcp.servers`. The related earlier PR #69726 was closed unmerged and #70495 is its stated re-open, so this is not superseded by the prior attempt.

## Best Possible Solution

Keep #70495 open and have a maintainer review or rebase it onto current main. The best fix is still narrow: label `openclaw mcp list/show/set/unset` human output/help as OpenClaw-managed `mcp.servers`, add the non-JSON mcporter exclusion note, keep JSON output unchanged, update the current CLI docs surface, and add the focused CLI regression tests from the PR.
