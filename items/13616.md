## Summary

Keep #13616 open. Current main has first-party local backup archive creation and verification, but not the requested native restore/import, selective cron/session merge, cloud listing/restore, encryption, or complete disaster-recovery workflow. The linked `openclaw-workspace-sync` plugin and `keepmyclaw` ClawHub skill are useful workarounds, but the latest maintainer comment explicitly kept this native umbrella open.

## Best Possible Solution

Keep #13616 open as the native umbrella for first-party backup restore/import and disaster-recovery behavior. The concrete path is to extend the existing `openclaw backup create` and `openclaw backup verify` implementation with safe restore/import, dry-run validation, rollback/overwrite rules, component-aware cron and session semantics, and explicit cloud/encryption product decisions if maintainers want those in core. Community options such as `keepmyclaw` and `openclaw-workspace-sync` should remain documented as workarounds, not as a reason to close this native issue unless maintainers explicitly change the product boundary.
