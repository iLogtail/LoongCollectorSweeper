## Summary

Keep #38876 open. Current main still has the Web UI tool-card bug: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped `content[0].text` still falls through to the visible “No output” fallback. The PR remains a focused, relevant fix path for the linked bug, though it should add regression tests before landing.

## Best Possible Solution

Keep #38876 open as the focused fix path, or land an equivalent maintainer patch in `ui/src/ui/chat/tool-cards.ts` that extracts and joins text from array-shaped `text` and `content` fields for embedded `toolresult` / `tool_result` blocks. Add regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for array `content`, array `text`, typed text blocks, ignored non-string fragments, and multi-fragment joining.
