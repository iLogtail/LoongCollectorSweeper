## Summary

Keep open. Current main still does not extract array-shaped `text` or `content` from `tool_result` / `toolresult` blocks in the Web UI tool-card path, while PR #38876 directly targets that missing behavior and is broader than related PR #38228.

## Best Possible Solution

Keep #38876 open and land it, or an equivalent maintainer patch, with regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for array `content`, array `text`, typed text blocks, and multi-fragment joining. After selecting the canonical fix, close the weaker duplicate PR with a link to the chosen implementation.
