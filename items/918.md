## Summary

本 Issue 请求为 LoongCollector 内置进程异常监控、自动重启及自升级能力。该需求涉及 Agent 核心生命周期管理与部署架构边界，当前 main 分支未实现此类内置模块，且插件系统仅覆盖数据流处理，无法承载进程级守护与升级逻辑。该特性属于需维护者进行产品裁决的新核心特性，依据保守审查原则保持开启，等待维护者明确产品边界或架构规划。

## Best Possible Solution

建议维护者评估产品边界：若遵循云原生 Agent 轻量化原则，可明确回复推荐依赖 systemd、K8s 或外部运维系统负责进程守护与升级，并关闭本 Issue；若计划内置该能力，需补充架构设计（如安全签名验证、失败回滚、权限隔离、跨平台兼容）并拆分为具体 RFC 或开发工单跟进。当前保持开启等待维护者产品决策。
