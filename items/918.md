## Summary

本 Issue 请求为采集端增加进程异常监控、自动重启及自升级能力。该需求涉及 Agent 生命周期管理与部署架构，通常由外部编排系统（如 systemd、Kubernetes DaemonSet）或独立的 Supervisor/Agent Manager 负责，而非内置于日志采集核心进程中。当前 main 分支未见内置 Watchdog 或自升级模块，且该架构决策需维护者评估核心边界、安全策略与跨平台兼容性。

## Best Possible Solution

建议维护者进行产品裁决：若坚持核心轻量化原则，可回复引导用户使用 systemd Restart 策略、K8s restartPolicy 或外部运维脚本/独立 Supervisor 工具，并关闭本 Issue；若计划内置生命周期管理，需明确 Watchdog 与升级模块的架构设计、安全边界（如包签名验证、失败回滚机制、权限隔离），并拆分为具体实现工单跟进。
