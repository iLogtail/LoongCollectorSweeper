## Summary

Keep #10960 open. Current main still does not implement true mid-stream soft steer for inbound queue mode: steer injection is gated on streaming, active non-streaming runs fall back to follow-up queueing, and the docs/tests still describe boundary or fallback behavior rather than pause/abort, inject, and resume.

## Best Possible Solution

Keep #10960 open as the runtime/product tracker. A complete fix should update the inbound auto-reply queue and embedded/reply-run control paths so steer-mode input during active non-streaming generation or tool waits can cleanly interrupt or pause active work, preserve usable partial context, append the steering message, and resume in the same session. Add regression coverage for streaming steer success, active-but-not-streaming/tool-wait behavior, follow-up fallback, explicit gateway `sessions.steer`, and backend failure handling.
