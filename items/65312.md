## Summary

Current main still has the reported docs gap. Source enforces a 2 MB local avatar cap and tests cover oversized avatars, but the relevant CLI and gateway docs only describe the avatar value types and do not mention the size limit. The linked fix PR #65675 is still open/unmerged, and PR #69630 was closed unmerged, so this should not be auto-closed as implemented.

## Best Possible Solution

Keep #65312 open until the docs change lands on main. The best maintainer path is to merge or adapt PR #65675, adding a clear note that local workspace avatar image files must be 2 MB or smaller in `docs/cli/agents.md` and the gateway agent/config reference, while avoiding claims that the limit applies to remote URLs or data URIs.
