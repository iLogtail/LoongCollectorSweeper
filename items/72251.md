## Summary

Keep PR #72251 open. Current main still has the reported live-backup EOF failure surface: backup creation uses a single direct `tar.c()` call with no volatile-file filter, no EOF retry loop, no enriched final EOF diagnostics, and no `skippedVolatileCount` result/JSON/summary field. The PR is a focused fix for open issue #72249, and the provided review context shows the author addressed the two bot review findings with follow-up commits and tests, so this is still useful maintainer-review work rather than obsolete cleanup.

## Best Possible Solution

Keep PR #72251 open for maintainer review. The best path is to rebase and review a narrowly scoped backup fix on current main that adds a volatile-path filter, tar EOF retry with partial-temp cleanup, enriched final errors, stdout/JSON skip-count reporting, and focused regression tests, while separately tracking the deletion-race and broader configurable exclusion/service-aware backup proposals.
