## Summary

Do not close PR #72251. Current main still writes backup archives with one direct `tar.c()` call and has no volatile-path filter, tar EOF retry loop, enriched EOF failure message, or `skippedVolatileCount` result field. The PR targets a concrete live-backup failure described in #72249, and the author already followed up on the bot review findings with additional commits and tests, so this is still useful maintainer-review work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased backup-archiver fix on main that adds a narrowly scoped volatile-file filter, tar EOF retry with temp archive cleanup, error enrichment, stdout/JSON skip-count reporting, and focused regression tests, while leaving #67417 and #67990 as separate follow-up work unless maintainers choose to consolidate backup consistency design.
