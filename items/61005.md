## Summary

Keep open. Current main still gates Android onboarding completion on both operator connectivity and node connectivity, so the reported `Connected (operator offline)` state remains blocked. The linked PR #61269 is a plausible narrow fix and remains open, not merged.

## Best Possible Solution

Keep #61005 open until an Android app fix lands. The likely product/code path is to change the onboarding completion gate in `apps/android/app/src/main/java/ai/openclaw/app/ui/OnboardingFlow.kt` so node session readiness is sufficient when the operator is offline, update `OnboardingFlowLogicTest` to cover node-connected/operator-offline completion, and then close this issue via PR #61269 or an equivalent maintainer-owned patch.
