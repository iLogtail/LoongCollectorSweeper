## Summary

Keep open. Current `main` still does not implement an OpenClaw-owned Feishu WebSocket supervisor: WebSocket mode creates one Lark `WSClient`, calls `start()` once, and then waits only for abort. The linked PR #57978 is the right active fix candidate, but it is open/unmerged, while earlier PRs #52623 and #52666 were closed unmerged.

## Best Possible Solution

Keep #52618 open as the bug tracker until #57978 or an equivalent maintainer-approved patch lands. The fix should live in `extensions/feishu/src/monitor.transport.ts`: wrap `WSClient` lifetime in an abort-safe supervisor loop, detect SDK stall/retry exhaustion using the available observable state or a watchdog, recreate/close clients cleanly, use bounded backoff via existing helpers, fail fast on permanent credential/app errors, and add Feishu transport tests that prove client recreation after a simulated stall.
