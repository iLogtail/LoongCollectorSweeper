## Summary

Keep open. Current main still has the single-shot Feishu WebSocket monitor described in #52618: account monitoring delegates directly to `monitorWebSocket`, that function creates one `WSClient`, calls `start()` once, and then only resolves on abort. The Lark SDK behavior in the pinned package still makes OpenClaw-owned supervision relevant, and the related fix PRs in the provided context are open or closed-unmerged rather than implemented on the checked-out main.
