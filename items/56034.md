## Summary

Keep open. Current `main` still performs enforced session-store pruning/capping by age and recency only. `SessionEntry.status` includes `running`, but the maintenance code does not consult it; it only preserves an explicitly passed `activeSessionKey`, and load-time oversized-store maintenance has no active-session preservation at all.

## Evidence

- **Running state is persisted in session entries:** `SessionEntry` has a persisted `status` field with `"running" | "done" | "failed" | "killed" | "timeout"`.
  - file: [src/config/sessions/types.ts:155](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/types.ts#L155)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Lifecycle code marks sessions as running:** Gateway lifecycle snapshots set `status: "running"` on start events, so the store has an explicit active/running marker available to maintenance code.
  - file: [src/gateway/session-lifecycle-state.ts:102](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/gateway/session-lifecycle-state.ts#L102)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Age pruning ignores running status:** `pruneStaleEntries()` removes entries solely by `updatedAt < cutoff` unless the key is in `preserveKeys`; it does not inspect `entry.status`.
  - file: [src/config/sessions/store-maintenance.ts:156](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/store-maintenance.ts#L156)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Count capping ignores running status:** `capEntryCount()` sorts by `updatedAt` and deletes everything past the retention window. There is no exemption for `status === "running"`.
  - file: [src/config/sessions/store-maintenance.ts:269](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/store-maintenance.ts#L269)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Enforced write-time maintenance preserves only one explicit key:** `saveSessionStoreUnlocked()` builds `preserveSessionKeys` from only `opts.activeSessionKey` before calling `pruneStaleEntries()` and `capEntryCount()`. Other concurrently running sessions are not protected.
  - file: [src/config/sessions/store.ts:283](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/store.ts#L283)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Load-time maintenance has no active-session preservation:** `loadSessionStore()` applies enforce-mode pruning/capping to oversized stores with no `preserveKeys`/active-session handling at all.
  - file: [src/config/sessions/store-load.ts:131](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/store-load.ts#L131)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
