## Summary

Keep open. Current `main` has local workspace-template primitives and ClawHub skill/plugin installs, but no shipped `agent-pack`/`install-pack`/`publish-pack` feature. VISION.md leans against adding optional packaging surfaces to core, yet that scope signal is not strong enough to close this request while the linked PR #38070 still exists upstream.

## Evidence

- **Workspace templates are documented as manual local files:** Current docs tell users to create `~/.openclaw/workspace` and copy `docs/reference/templates/AGENTS.md`, `SOUL.md`, and `TOOLS.md` into it manually. That shows the underlying template concept exists, but not the requested shareable-pack workflow.
