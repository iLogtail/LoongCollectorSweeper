## Summary

Keep open. Current `main` still resolves `contextWindow`/`contextTokens`/`maxTokens` by explicit-override precedence rather than taking a conservative minimum across agent config, provider config, and discovered metadata. The only minimum-style logic found is the guard path for `agents.defaults.contextTokens`, which affects warnings/blocking and compaction budgeting, not the resolved model metadata used elsewhere.

## Evidence

- **Resolved models keep configured token limits verbatim:** `applyConfiguredProviderOverrides()` copies configured `contextWindow`, `contextTokens`, and `maxTokens` over discovered metadata with nullish coalescing. There is no clamp/min against discovered limits here.
