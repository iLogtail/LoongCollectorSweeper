## Summary

Keep #71136 open. Current main has a partial default-context cap in the embedded runner, but it does not implement the requested unified minimum across agent config, provider model config, discovery/defaults, and maxTokens. Override/session-store paths still prefer explicit contextTokens, maxTokens is merged and forwarded without a global model-limit clamp, and provider model merging still prefers explicit model token metadata when present.

## Best Possible Solution

Keep this open and implement a central token-limit resolver for both contextTokens and maxTokens. It should gather all positive candidates from per-agent/default agent config, models.providers model metadata, runtime/discovered model metadata, and hard defaults, then use the minimum for the effective runtime/session value. Thread per-agent contextTokens into the embedded runner’s resolveContextWindowInfo path, clamp or normalize maxTokens before provider transports receive it, and update context/maxTokens tests plus config docs to describe the effective-min behavior.
