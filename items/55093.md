## Summary

Keep open. Current `main` does not contain the PR's two substantive changes: runtime facade entries are still manually hand-listed instead of covering `src/**/*.runtime.ts`, and local dirty-build version marking is still absent from `build-info` generation and `openclaw --version` output.

## Evidence

- **Runtime facades are still hand-listed:** `tsdown.config.ts:204` builds the core dist graph from a fixed object literal. The stable runtime entries are a short explicit list at `tsdown.config.ts:210`-`223`, not an automatic `src/**/*.runtime.ts` walk like the PR proposed.
  - file: [tsdown.config.ts:204](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/tsdown.config.ts#L204)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Current main has far more runtime facades than the dist graph pins:** `find src -type f -name '*.runtime.ts' | wc -l` reports 164 runtime facade files on current `main`. A representative facade exists at `src/agents/cli-runner.runtime.ts:1`, but there is no matching entry for `cli-runner.runtime` in `tsdown.config.ts`'s stable-entry list.
  - file: [src/agents/cli-runner.runtime.ts:1](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/cli-runner.runtime.ts#L1)
