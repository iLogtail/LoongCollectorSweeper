## Summary

Keep PR #53821 open. Current main still has only the config/schema surface for `agents.list[].sandbox.sessionToolsVisibility`; the runtime session tools continue to resolve sandbox visibility from the global default and do not thread `requesterAgentIdOverride` through list/history/send/status access control. The PR remains a plausible core fix, though it needs maintainer review and rescoping because the supplied PR file list includes unrelated changes.

## Best Possible Solution

Keep the PR open or ask the author to rescope it. The best implementation is a focused core change that resolves `agents.list[].sandbox.sessionToolsVisibility` by requester agent, threads `requesterAgentIdOverride` through `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, roots spawned/self/tree guards on the effective requester key including custom `session.mainKey`, and adds targeted tests for override, fallback, direct-key, label, and status paths. If maintainers prefer #59170 as the broader access-control vehicle, explicitly fold this sandbox-clamp behavior into that PR rather than closing this as already handled.
