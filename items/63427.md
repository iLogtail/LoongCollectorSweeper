## Summary

Keep this issue open. Current main still lacks the per-URL/module-level `device-required` short-circuit described by the reporter and by the closed-but-unmerged fix PR. The existing `GatewayClient` has per-instance reconnect backoff and pauses reconnect after `DEVICE_IDENTITY_REQUIRED`, but `probeGateway()` still constructs and starts a fresh probe client for each call, so the reported cross-session/cron burst case is not fixed on main.

## Best Possible Solution

Keep this issue open until a maintainer lands a source-level fix. The most direct path is to revive the PR #63446 approach in `src/gateway/probe.ts`: track recent `DEVICE_IDENTITY_REQUIRED` probe failures per gateway URL, short-circuit after a small threshold/TTL without constructing another WebSocket, clear the cache on successful connection, and add focused regression tests proving the fourth repeated device-required probe does not instantiate `GatewayClient`. If PR #68280 is intended to supersede this, it should explicitly cover this issue's repeated device-required probe storm with equivalent tests before closing.
