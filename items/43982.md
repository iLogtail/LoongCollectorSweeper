## Summary

Keep open. Current main still lacks the PR's core chat.history sanitization for inline image payload shapes: it strips top-level `type: "image"` `data` and base64 audio `source.data`, but not dashboard/Anthropic-style `content[].source.data`, top-level image data URLs, or OpenAI-style `image_url.url`. The PR is stale against the current sanitizer location, but the underlying fix is still useful and should be ported rather than closed.

## Best Possible Solution

Keep the PR open and port the sanitizer change to `src/gateway/chat-display-projection.ts`: strip/mark inline image payloads from `image.source.data`, data-url `image.url`, and `image_url.url`, accumulate omitted bytes across multiple stripped fields, preserve normal remote HTTP image URLs, and add focused regression coverage in the current gateway/server-methods tests.
