## Summary

Keep PR #71137 open. Current main still has the Slack external-arg menu in-memory store and its runtime wiring, but it does not contain the companion `external-arg-menu-store.test.ts` file or equivalent direct unit coverage for TTL pruning, collision retry, bounded stale-entry cleanup, and token parsing/charset rejection. The provided PR is a focused test-only addition, and the existing bot review says its assertions match the implementation.

## Best Possible Solution

Keep the PR open for maintainer review or landing after normal CI. The best path is to merge focused companion coverage for `extensions/slack/src/monitor/external-arg-menu-store.ts`, preserving the production implementation and using the existing Slack monitor test patterns.
