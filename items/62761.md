## Summary

Keep #62761 open. Current main implements the later `is_from_me` echo-loop mitigation, but the original restart/backlog replay problem is still not implemented: the legacy iMessage monitor subscribes without a persisted rowid/start cursor and repo search found no per-account cursor persistence path.

## Best Possible Solution

Keep the issue open and retitle/scope it to legacy iMessage restart replay cursor persistence. The likely fix belongs in `extensions/imessage/src/monitor/monitor-provider.ts`: persist a per-account last processed message rowid or equivalent replay marker, pass it to `watch.subscribe` on startup, update it only after safe handling/drop decisions, and add regression tests for restart replay plus docs/config notes if a new state file or setting is introduced.
