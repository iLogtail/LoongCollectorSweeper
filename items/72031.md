## Summary

Keep #72031 open. Current main still has the reported Bedrock aws-sdk auth regression in the media-understanding image and audio/video auth paths, while the chat path has the missing-key allowance and the Bedrock docs say aws-sdk auth requires no apiKey. PR #72092 is the right canonical fix candidate, but it is still open/unmerged in the provided context.

## Best Possible Solution

Keep #72031 open and land or adapt PR #72092: add a narrow missing-api-key allowance for auth.mode === "aws-sdk" in resolveImageRuntime, resolveMinimaxVlmFallbackRuntime, and resolveProviderExecutionAuth; avoid storing empty runtime keys; and add regression coverage for amazon-bedrock aws-sdk media-understanding calls with no static apiKey.
