## Summary

Keep #72031 open. Current main still has the reported Bedrock `aws-sdk` media-understanding auth regression: image and audio/video paths still require a resolved API key, while Bedrock config/docs and other runtime paths intentionally allow `aws-sdk` auth without a static key. PR #72092 is the right fix candidate from the provided context, but it is still open/unmerged, so the bug is not implemented on main.

## Best Possible Solution

Keep #72031 open and land or adapt PR #72092: add a narrow missing-API-key allowance for `auth.mode === "aws-sdk"` in `resolveImageRuntime`, `resolveMinimaxVlmFallbackRuntime`, and `resolveProviderExecutionAuth`; avoid storing empty runtime keys; and add regression coverage for `amazon-bedrock` media-understanding calls with AWS SDK credentials and no static API key.
