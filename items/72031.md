## Summary

Keep #72031 open. Current main still unconditionally requires a resolved API key in the media-understanding image and provider execution paths, while Bedrock `aws-sdk` auth intentionally has no static API key. The linked fix PR #72092 is still open/unmerged in the provided GitHub context, so the reported regression is not shipped on main yet.

## Best Possible Solution

Keep #72031 open until #72092 or an equivalent maintainer-approved patch lands. The fix should add the same missing-key allowance used by chat/completion to the media-understanding image, MiniMax fallback, and provider execution auth paths, and include regression coverage for `amazon-bedrock` with `auth.mode === "aws-sdk"` and no static API key.
