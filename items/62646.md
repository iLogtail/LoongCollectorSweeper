## Summary

Keep PR #62646 open for maintainer judgment/rework. Current main has shipped partial QMD fixes, including removal of phantom `memory-alt-*`, same-name/path-pattern collection repair, and a one-time missing-collection search repair, but it does not implement the PR’s central invariant of searching only collections confirmed to exist after `ensureCollections()`. The PR also includes an unmerged `--mask`-first behavior change and has unresolved P1 review concerns about the proposed pruning approach, so it should not be auto-closed as already implemented or merged as-is.

## Best Possible Solution

Keep #62646 open for maintainer judgment or rework. A safer fix would derive the per-search collection list from verified sidecar state when that state is reliable, preserve valid configured collections when QMD listing is unavailable, fail closed after repair instead of running unscoped searches, and make the `--glob` versus `--mask` default decision with current QMD compatibility evidence. If maintainers decide the shipped `memory-alt` removal and collection repair are sufficient, they should close this PR manually with that product call rather than through automated cleanup.
