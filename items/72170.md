## Summary

Keep PR #72170 open. Current main has a generic Kubernetes guide and an Azure VM guide, but no AKS deployment instructions, ACR integration, or `az aks` workflow. Provider-specific hosting docs are already an established docs surface, and the PR is a focused docs contribution rather than something already implemented or clearly out of scope.

## Best Possible Solution

Keep the PR open. The best path is maintainer review of a concise AKS deployment section or a dedicated AKS install page, preserving the existing non-loopback bind warning for ingress, using globally unique placeholder registry names, and keeping Azure cost/security claims conservative.
