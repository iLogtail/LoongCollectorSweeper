## Summary

Keep #51458 open. Current main still rejects the reported Slack `send` message-action combination: `interactive` is rendered into Slack blocks, and the dispatch path throws when `media` and blocks are both present. The lower Slack action/runtime send paths also still reject `mediaUrl` with blocks. The separate reply/outbound-payload path does have a two-message split for media plus interactive blocks, but that behavior is not used by the message action path reported here. Related PR #62792 is scoped to inbound Slack file access and explicitly does not change outbound Slack behavior, so it does not supersede this issue.
