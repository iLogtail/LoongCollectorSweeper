## Summary

Keep open. Current main only has partial Feishu interactive-card parsing on the message read/thread-history helper path, and the inbound message-event path still does not parse `interactive` card content into useful text. The issue’s core request remains unresolved, with related open follow-up work in #42218, #60380, and #60383 rather than a shipped fix.

## Best Possible Solution

Keep #41609 open as the broad Feishu inbound interactive-card parsing bug. The best fix is a shared Feishu card parser used by both inbound event parsing and message read/thread-history paths, plus any needed raw-card callback/API request handling, with regression coverage for `parseFeishuMessageEvent`, `getMessageFeishu`, and thread history across header text, markdown/div content, buttons/actions, dropdown/form values, and post-format fallback cards. The open related work in #42218, #60380, and #60383 can inform that implementation, but it does not replace this issue yet.
