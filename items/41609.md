## Summary

Current main only partially handles Feishu interactive cards. It can extract simple markdown/div text from some read/thread-list API responses, but the original issue asks for full interactive-card content in inbound message handling, including buttons/options/form data. That capability is still missing, and linked follow-up PRs/issues remain open.
