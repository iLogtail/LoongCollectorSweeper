## Summary

Keep open. Current `main` only parses Feishu interactive-card content on the message-fetch path (`getMessageFeishu` / thread history), but the live inbound message-event path still does not handle `message_type === "interactive"`. That means the issue's core request, parsing card content from Feishu message events, is not implemented end-to-end on `main` at `350d3221800a18283915cd2f3b612ae04ddd490c`.

## Evidence
