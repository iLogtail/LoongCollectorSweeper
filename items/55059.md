## Summary

Keep open. Current `main` does not implement a distinct `openai-chat` adapter, and the existing `openai-completions` path still defaults to `max_completion_tokens` for Qwen/DashScope-style providers, so the issue’s reported compatibility gap is still plausible.

## Evidence

- **No `openai-chat` API type exists:** The model API enum only includes `openai-completions`, `openai-responses`, `openai-codex-responses`, `anthropic-messages`, `google-generative-ai`, `github-copilot`, `bedrock-converse-stream`, `ollama`, and `azure-openai-responses`. There is no `openai-chat` entry.
