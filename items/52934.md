## Summary

Keep #52934 open. Current main still intentionally tries ClawHub before npm for bare plugin specs, and tests/docs assert that behavior. Main has archive integrity/security scans and plugin allowlist guidance, but I did not find a shipped fix for the reported namespace/name-squatting concern, such as explicit trust, publisher/signature verification, an install warning/confirmation for ClawHub-first resolution, or npm-first behavior when the user typed an npm package spec.

## Best Possible Solution

Keep the issue open for maintainer triage. A conservative fix should make the user's intended source explicit or verifiable: for example require `clawhub:` for ClawHub installs, prefer npm when the user typed an npm package spec, or add strong ClawHub provenance/trust UX such as publisher/signature verification, community-package warnings, and trust/integrity pins. The docs and tests should then be updated to reflect the chosen source-resolution contract.
