## Summary

Current main still builds Feishu card headers from the raw `identity.emoji` string plus the agent name, so descriptive non-emoji text in that field can still be rendered in card headers. This is a live Feishu plugin bug, not a conservative cleanup close.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin. A narrow fix should centralize Feishu card-header title resolution, avoid rendering descriptive text from `identity.emoji` as a header prefix, add regression tests for non-emoji `identity.emoji` values on both outbound card and reply/streaming card paths, and align with any still-open canonical card-header configuration work such as PR #54979 rather than closing this bug prematurely.
