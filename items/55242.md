## Summary

Keep open. Current `main` still concatenates raw `identity.emoji` into Feishu card header titles for both outbound card sends and the reply dispatcher, and the identity config accepts arbitrary strings with no emoji-only sanitization. I did not find evidence on `main` that this request is already fixed.

## Evidence

- **Outbound card path still leaks raw identity.emoji:** Feishu outbound card sends still build the header title as `${identity.emoji} ${identity.name ?? ""}` when `identity.emoji` is set, which would render descriptive text verbatim in the card header.
  - file: [extensions/feishu/src/outbound.ts:206](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/feishu/src/outbound.ts#L206)
  - command: `nl -ba extensions/feishu/src/outbound.ts | sed -n '180,235p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Reply dispatcher still uses raw identity.emoji in card headers:** The current Feishu reply dispatcher trims `identity.emoji` but does not sanitize it; it returns `title: emoji ? `${emoji} ${name}` : name`, so non-emoji text remains in streaming and markdown-card headers.
  - file: [extensions/feishu/src/reply-dispatcher.ts:46](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/feishu/src/reply-dispatcher.ts#L46)
