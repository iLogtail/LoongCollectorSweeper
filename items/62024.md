## Summary

Keep open. Current main still contains the reported Discord explicit-target bug: `parseDiscordExplicitTarget` returns the bare `target.id`, while the Discord outbound normalizer still maps bare numeric recipients to `channel:<id>`. That preserves the failure chain described in #62024 for cron delivery to `user:<id>`. The linked PR #62052 targeted the right one-field fix but is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #62024 open. Reapply the intent of PR #62052 on current main by returning `target.normalized` from `parseDiscordExplicitTarget`, then add focused tests that `user:<id>` stays prefixed, `channel:<id>` stays prefixed, bare numeric IDs keep the existing channel behavior, and isolated cron delivery does not rewrite explicit Discord DM targets to channels.
