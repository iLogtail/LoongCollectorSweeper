## Summary

Current main still ignores deleteAfterRun for recurring schedule kinds because applyJobResult only returns shouldDelete for at schedules. Two open PRs, #63871 and #64375, explicitly target this issue, so the issue should remain open until one is reviewed and merged or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review one implementation candidate, preferably the narrow fix that removes the schedule-kind restriction in src/cron/service/timer.ts and adds regression tests for every and cron schedules deleting on successful runs while staying present on failed runs. Once that lands on main, close this issue as implemented.
