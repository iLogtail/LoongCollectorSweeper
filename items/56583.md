## Summary

Keep PR #56583 open. Current main has a different overload retry mechanism: a fixed, opt-in `auth.cooldowns.overloadedBackoffMs` delay defaulting to 0 ms. It does not implement the PR’s central request for an embedded Pi exponential overload failover policy with a 30s default ceiling and `agents.defaults.embeddedPi.overloadBackoffMaxMs` config. The PR still needs maintainer/author cleanup because its branch also carries unrelated live-test HOME/profile changes with unresolved review concerns.

## Best Possible Solution

Keep the item open but ask for a focused rebase or replacement PR: decide whether overload pacing should extend the existing `auth.cooldowns.overloadedBackoffMs` surface or add the proposed embedded Pi-specific `agents.defaults.embeddedPi.overloadBackoffMaxMs`, then implement the 30s default/exponential policy with aligned types, zod/schema/help/docs, and regression tests. Split the unrelated live-test environment changes into a separate PR.
