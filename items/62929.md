## Summary

Current `main` has heartbeat/session-history protections that prevent heartbeat runs from truncating the active transcript and make the Gateway/UI keep reading session history even when transcript files rotate or archive, which matches the reported `v2026.4.5` symptom of UI history disappearing while files still existed.

## Evidence

- **Heartbeat transcript is now append-only:** Regression test explicitly covers the heartbeat bug class: after `runHeartbeatOnce(...)`, the transcript size must not shrink, and `HEARTBEAT_OK` is kept in the file instead of being removed via truncate.
  - file: [src/infra/heartbeat-runner.transcript-prune.test.ts:17](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/infra/heartbeat-runner.transcript-prune.test.ts#L17)
  - command: `nl -ba src/infra/heartbeat-runner.transcript-prune.test.ts | sed -n '17,82p'`
  - sha: [f896c3935ef8](https://github.com/openclaw/openclaw/commit/f896c3935ef8d706fe4193443c5c5eede5e64178)
- **Gateway resolves rotated/archived transcript candidates:** Session transcript lookup now searches current, stale, archived, agent-scoped, and legacy candidate paths rather than assuming only one live `.jsonl` path. That directly addresses reports where the file still exists on disk but the UI goes blank after rotation/archive.
  - file: [src/gateway/session-transcript-files.fs.ts:70](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/gateway/session-transcript-files.fs.ts#L70)

  - file: [src/gateway/server-methods/sessions.ts:1470](https://github.com/openclaw/openclaw/blob/f896c3935ef8d706fe4193443c5c5eede5e64178/src/gateway/server-methods/sessions.ts#L1470)
