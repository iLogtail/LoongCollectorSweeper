## Summary

Keep PR #38640 open. Current main has adjacent provider SDK retry caps, prompt/assistant failover, auth-profile cooldown/rotation, and model fallback, but it still does not implement the PR's prompt-level exponential-backoff retry wrapper around `activeSession.prompt(...)`. The PR also has material unresolved review concerns, so it needs maintainer design/review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer judgment, or explicitly supersede it with a canonical LLM retry design such as the open configurable provider retry work. A best implementation would place retry at the intended provider/runner boundary, honor abort signals during backoff, parse numeric and string status/statusCode consistently with `failover-error.ts`, respect provider SDK retry behavior, and add focused unit plus runner/failover coverage.
