## Summary

Keep open. Current main still lacks both requested behaviors: gateway startup does not warn when the selected agent provider lacks auth, and skill discovery still only loads a root skill or immediate child skill directories after the narrow dir/skills heuristic. The PR also has unresolved review blockers around auth-store I/O on a startup logging path and recursive scanning without the existing candidate budget.

## Best Possible Solution

Keep this open and rework it against current main. The skill side should add bounded recursive discovery with ignored dirs, symlink containment, maxCandidatesPerRoot enforcement, refresh/watch follow-through, and tests for deeper bundle layouts. The auth side should use the current auth resolution contract in a read-only, nonblocking startup diagnostic path, or otherwise route through an existing auth-status helper, with tests for env, profiles, models.providers apiKey, aws-sdk, Bedrock, and local synthetic providers.
