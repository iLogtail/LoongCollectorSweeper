## Summary

Keep open. Current `main` is built to persist Control UI device identity and device tokens in browser `localStorage`, so a plain browser restart should not reset the device ID in a normal profile. But the issue is still plausible: the report has no repro, logs, browser details, or origin details, and the docs distinguish persistent device identity from per-tab shared-secret storage. That is not strong enough for a conservative close.
