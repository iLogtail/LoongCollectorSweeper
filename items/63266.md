## Summary

Keep open. Current main does not implement the requested Docker-to-host split-layout detection, repair, or guidance. Workspace resolution treats `agents.defaults.workspace` or the default workspace literally, doctor’s legacy-workspace detector currently returns no legacy directories, and the migration/docs surfaces describe state and workspace locations without addressing a `workspace-local` versus `workspace-local/local` split or secrets stranded above the active workspace. The related `/home/node` reports in the provided context cover stale absolute session paths, not this distinct workspace/secrets layout problem.

## Best Possible Solution

Keep this issue open and implement it as a doctor/fix plus docs path: detect parent/child active workspace splits such as `workspace-local` and `workspace-local/local`, report exactly which root is active, identify sensitive/config-like files stranded above or below it, offer a conservative repair or explicit manual guidance, and add regression tests for the detection/reporting behavior. Avoid a startup-time core migration unless maintainers explicitly choose that product behavior.
