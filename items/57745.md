## Summary

Keep open. Current `main` does not implement SKILL.md conditional activation based on tool or toolset availability. The shipped skill spec and eligibility pipeline only support OS, binary, env, config, allowlist, and plugin-enable gating.

## Evidence

- **Skills docs describe current gating surface:** The public skills docs say skills are filtered at load time based on environment, config, and binary presence, then enumerate `metadata.openclaw` fields as `always`, `emoji`, `homepage`, `os`, `requires.bins`, `requires.anyBins`, `requires.env`, `requires.config`, `primaryEnv`, and `install`. They also say skills without `metadata.openclaw` are otherwise always eligible.
  - file: [docs/tools/skills.md:9](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/docs/tools/skills.md#L9)
