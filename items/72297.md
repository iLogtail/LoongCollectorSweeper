## Summary

Keep #72297 open. Current main still cannot rebind a QMD path-pattern conflict when `collection list --json` only returns collection names; it only rebinds when list output exposes path/pattern metadata. The PR is a focused, recent memory-core fix with regression coverage, but the Aisle security comment about using untrusted qmd stderr to select a collection for removal needs maintainer review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to preserve the useful regression coverage for name-only `qmd collection list` output, but amend the implementation so any fallback from add-error text is verified against authoritative collection metadata before removing a collection, or degrades to a warning/manual repair path when qmd cannot provide verifiable metadata.
