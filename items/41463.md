## Summary

Current `main` already implements the reported allowlist behavior for custom subagents. `agents_list` resolves `agents.list[].subagents.allowAgents` / `agents.defaults.subagents.allowAgents`, includes configured agents when the allowlist is `['*']`, and `sessions_spawn` uses the same allowlist source before rejecting a target. The remaining `/agents` behavior is now explicitly the active-run/binding view, while discovery of spawnable agent ids is documented under `agents_list`.

## Evidence

- **`agents_list` now exposes allowlisted/configured targets:** The tool resolves the requester agent, reads per-agent or default `subagents.allowAgents`, treats `*` as allow-any, and returns the ordered allowed agent list including configured agent names/flags.
  - file: [src/agents/tools/agents-list-tool.ts:21](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/tools/agents-list-tool.ts#L21)
- **`sessions_spawn` uses the same allowlist source:** Cross-agent spawns read the same `subagents.allowAgents` config and only return `allowed: none` when the resolved allowlist is actually empty; `['*']` sets `allowAny` and skips that rejection path.
  - file: [src/agents/subagent-spawn.ts:653](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/subagent-spawn.ts#L653)
- **Regression coverage for default allowlist fallback and `['*']`:** Tests assert that `agents_list` falls back to `agents.defaults.subagents.allowAgents` and that `allowAgents: ['*']` returns configured agents like `coder` and `research` instead of `(none)`.
  - file: [src/agents/openclaw-tools.agents.test.ts:76](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/agents/openclaw-tools.agents.test.ts#L76)
- **Current docs separate discovery from `/agents`:** Docs now say `agents_list` is the way to discover agent ids allowed for `sessions_spawn`, while `/agents` is described separately as a binding/active-run view.
  - file: [docs/tools/subagents.md:127](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/docs/tools/subagents.md#L127)
- **`/agents` implementation is an active-run/binding view:** The command renders current visible subagent runs and conversation binding state; when there are no runs it prints `(none)`, which matches the current documented behavior rather than configured-agent discovery.
  - file: [src/auto-reply/reply/commands-subagents/action-agents.ts:31](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/auto-reply/reply/commands-subagents/action-agents.ts#L31)
- **Latest release already contains the implemented allowlist logic:** `git show v2026.4.22:src/agents/tools/agents-list-tool.ts` matches the current allowlist implementation, and `git diff v2026.4.22..HEAD -- src/agents/tools/agents-list-tool.ts src/agents/openclaw-tools.agents.test.ts` shows no relevant changes after the release for this path.
  - command: `git diff --unified=0 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5..HEAD -- src/agents/tools/agents-list-tool.ts src/agents/openclaw-tools.agents.test.ts`

## Risks / Open Questions

- I did not run a live gateway reproduction in this read-only pass; the decision is based on current code, docs, and regression coverage.
- `/agents` still does not list configured idle agents; users must use `agents_list` for discovery. That is consistent with current code/docs, but it is still a behavior change from the issue's expectation.

## Close Comment
