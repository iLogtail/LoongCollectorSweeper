## Summary

Keep #39223 open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the Gateway-location Clack select before model/auth prompts, and the configure command still has no flag to bypass that prompt. The direct `models auth` commands and Anthropic docs are valid workarounds already noted in the issue discussion, but they do not fix the reported configure wizard hang.

## Best Possible Solution

Keep #39223 open until the configure path itself is robust: add TTY/non-interactive detection with an actionable auth bypass, add a configure flag or section mode that can skip Gateway-location selection for model/auth work, or fix the Clack input hang at `Where will the Gateway run?`. Keep the documented `openclaw models auth login/setup-token/paste-token` routes as workarounds, not as the closing fix.
