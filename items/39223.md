## Summary

Keep open. Current main still routes `openclaw configure --section model` through the interactive Gateway-location prompt before reaching model/auth setup, and `openclaw configure` still exposes no bypass flag for that prompt. The `models auth setup-token` / `paste-token` workaround from the discussion is real, but it does not fix the configure wizard input hang or add an Anthropic browser OAuth URL flow from configure.

## Best Possible Solution

Keep tracking this as a real configure wizard/product issue. A good fix would either make the Gateway-location prompt robust in Docker/pseudo-terminal conditions or add an explicit bypass such as a local-gateway/default mode flag for `openclaw configure --section model`, then cover it with a regression test. Maintainers should also decide whether Anthropic should expose a browser OAuth-style flow from configure or whether the intended path is Claude CLI/setup-token/API key with clearer docs.
