## Summary

Keep open. Current main still has the reported macOS gateway PATH audit behavior: the audit compares the LaunchAgent PATH against the full minimal-path superset, while the macOS resolver unconditionally appends common user-bin/version-manager directories from HOME. The warning text still says "missing required dirs", and there is no existence/configuration filter for the nonexistent Darwin user directories called out in #70228. The related ClawSweeper matches are adjacent PATH/runtime issues, not a canonical duplicate or shipped fix for this audit wording/false-positive bug.

## Best Possible Solution

Keep #70228 open. The best fix is in the core gateway service audit: make the Darwin PATH audit compare only truly required system/runtime entries plus user-bin directories that are configured or exist on disk, or split the broad generated PATH into required versus recommended entries and reword the warning away from "required dirs". Add focused regression tests in `src/daemon/service-audit.test.ts` for a healthy macOS LaunchAgent PATH that omits nonexistent common user-bin dirs, while preserving warnings for existing/configured-but-missing directories.
