## Summary

Keep open. Current main still computes macOS gateway service PATH expectations from the full generated minimal PATH, including common user/version-manager directories appended solely from HOME. The audit then reports every absent expected entry as `Gateway service PATH missing required dirs`, and `openclaw gateway status` prints those audit issues as service-config warnings. There is no current test or source path that filters nonexistent Darwin user-bin directories or downgrades the wording requested in #70228.

## Best Possible Solution

Keep #70228 open and fix the Darwin audit path specifically: distinguish required system/service PATH entries from optional HOME-derived user-bin fallbacks, filter or downgrade nonexistent optional dirs, and update the message away from `required dirs` for optional recommendations. Add focused tests in `src/daemon/service-audit.test.ts` for a healthy macOS LaunchAgent PATH that omits nonexistent user-bin defaults while still warning for meaningful existing or env-configured missing dirs.
