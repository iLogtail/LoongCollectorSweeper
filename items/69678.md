## Summary

Keep #69678 open. Current main still truncates isolated cron summaries at 2000 UTF-16 characters before they are emitted into cron run history, and the `cron.runLog` config/docs/schema still only expose file pruning controls (`maxBytes`, `keepLines`), not a per-entry summary length limit or full-output run-log field.

## Best Possible Solution

Keep the issue open and implement a configurable cron run-log summary cap, probably `cron.runLog.maxSummaryChars`, then plumb it through the isolated cron summary extraction path before `appendCronRunLog`. Update config schema/help/generated docs and add regression coverage proving long agent output is not fixed at 2000 chars while run-log file pruning still applies.
