## Summary

Keep #69678 open. Current main still hard-codes isolated cron run summaries to 2000 characters, persists only that `summary` into `cron/runs/<jobId>.jsonl`, and exposes only `cron.runLog.maxBytes` / `keepLines` in config and docs. The existing test suite also asserts that long summaries are truncated, so the reported behavior remains current and actionable rather than implemented or obsolete.

## Best Possible Solution

Keep the issue open and fix it in cron core: add a bounded `cron.runLog.maxSummaryChars` config field or otherwise store/audit full delivered text while preserving file-size pruning, then update `CronConfig`, Zod schema, generated/help docs, cron docs, and regression tests for long summaries.
