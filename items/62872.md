## Summary

Keep open. Current `main` still runs the `sessions_send` peer-session A2A follow-up path that does reply ping-pong and an announce step, and it still marks those injected turns with `sourceTool: "sessions_send"`. The only implemented skip is a narrower case for parent-owned one-shot ACP children; persistent sessions are explicitly treated as interactive, so the reported persistent-session duplicate flow is not fixed on `main`.
