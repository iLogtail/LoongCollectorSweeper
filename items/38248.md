## Summary

Current `main` still has model/auth failover and cost reporting, but no native request-level spend ceiling or `agents.defaults.costGuard`-style enforcement. The issue remains a valid core feature request rather than something already implemented or safe to close.

## Evidence

- **Failover docs describe retries, not spend caps:** The current failover doc defines a two-stage flow of auth-profile rotation and model fallback via `agents.defaults.model.fallbacks`, with no hourly/daily spend guard in the runtime description.
  - file: [docs/concepts/model-failover.md:10](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/docs/concepts/model-failover.md#L10)
  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **Failover behavior still advances on provider failures:** The same doc says OpenClaw moves to the next configured fallback on auth failures, rate limits, and timeouts, and only exposes cooldown tuning knobs such as `auth.cooldowns.overloadedProfileRotations` and `auth.cooldowns.rateLimitedProfileRotations`. There is no cost-budget stop action here.
  - file: [docs/concepts/model-failover.md:215](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/docs/concepts/model-failover.md#L215)
  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **Runtime fallback loop has no spend-budget check:** `runWithModelFallback()` builds candidates from the requested model plus configured fallbacks, loops through them, applies cooldown/probe logic, and records failed attempts. In this path there is no check against accumulated hourly/daily spend or a `pause`/`reject` budget action before retrying the next model.
  - file: [src/agents/model-fallback.ts:652](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/src/agents/model-fallback.ts#L652)
  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **Cost surface is reporting only:** The API usage doc says costs show up in `/status`, `/usage full`, and other summaries as estimated cost. It documents visibility, not enforcement or kill-switch behavior.
  - file: [docs/reference/api-usage-costs.md:15](https://github.com/openclaw/openclaw/blob/2fb9c7e3e586044467f360edf8d8f5170f1c60f5/docs/reference/api-usage-costs.md#L15)
  - sha: [2fb9c7e3e586](https://github.com/openclaw/openclaw/commit/2fb9c7e3e586044467f360edf8d8f5170f1c60f5)
- **`/usage cost` is historical summary output:** The `/usage cost` command loads session and 30-day summaries and replies with a text report; it does not block or pause future requests when a threshold is crossed.
