## Summary

Keep open. Current `main` still does not implement a `clear completed` / per-entry dismiss / completed-subagent separator in the Control UI chat session dropdown. The dropdown renders plain `<optgroup>/<option>` entries from `resolveSessionOptionGroups()` and does not use per-session `status` or `endedAt` to filter or split completed subagent sessions. There is now a partial UI workaround: the separate Control UI Sessions tab supports selecting sessions and deleting them via `sessions.delete`, and subagent docs still describe auto-archive after `archiveAfterMinutes` (default 60). That means the issue body's "no way from the UI" claim is outdated, but the dropdown-specific feature request itself is not implemented on `main`.
