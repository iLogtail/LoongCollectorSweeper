## Summary

Current main still has the hostname fallback this PR removes. `isSecureWebSocketUrl()` still returns `net.isIP(hostForIpCheck) === 0` in `allowPrivateWs` mode, so arbitrary non-IP hostnames are accepted, and multiple current tests still assert that behavior. The PR is not implemented on main and is not obsolete.

## Best Possible Solution

Keep the PR open. The best path is to resolve the product/security decision explicitly: either merge a fix that removes the hostname fallback and updates all gateway/client/onboarding tests to reject plaintext hostname targets, or document and implement a narrower explicit hostname/DNS allowlist if maintainers want to support private DNS names safely.
