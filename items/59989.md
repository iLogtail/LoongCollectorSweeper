## Summary

Keep #59989 open. Current main still tears down the OpenClaw session subscription synchronously: it aborts in-flight compaction and immediately removes the session listener, so the PR's central event-loss problem is not implemented on main. The PR is also not ready to close as obsolete because the underlying concern is plausible under the pi-coding-agent event contract, but the submitted branch has unresolved P1 review issues around async unsubscribe semantics and willRetry handling.

## Best Possible Solution

Keep the PR open and ask for a safer revision: preserve the current synchronous unsubscribe contract or update all callers to await a shared idempotent cleanup promise; register the compaction cleanup waiter without a state-check race; resolve only on the final !willRetry compaction_end; clear any timeout handle; and add fast fake-timer regression coverage that proves abort-triggered compaction_end is delivered before the listener is removed.
