## Summary

Keep open. Current `main` still disables Feishu block streaming in the reply pipeline, drops `kind === "block"` payloads unless they are consumed by the streaming-card path, and the streaming card still uses Feishu CardKit playback animation. The report matches current code, and docs still advertise `channels.feishu.blockStreaming`, so this is not implemented on `main` and not obsolete.

## Evidence

  - file: [extensions/feishu/src/reply-dispatcher.ts:494](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/feishu/src/reply-dispatcher.ts#L494)
