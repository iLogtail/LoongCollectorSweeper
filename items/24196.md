## Summary

Keep #24196 open. Current main still has only scalar Discord outbound media inputs and uploads exactly one file per Discord message; when payloads contain multiple media URLs, the generic/Discord fallback sends them as a sequence of separate messages instead of one Discord message with up to 10 attachments. The existing 2026-04-26 maintainer-side Codex comment in the issue remains accurate against current main.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape, preserve existing single media/path/filePath behavior, validate Discord count and file-size limits, upload up to the supported limit in one files array, split overflow into follow-up messages, and update focused Discord docs and tests.
