## Summary

Keep #24196 open. Current main still does not implement native up-to-10 Discord file attachments for the general Discord message action/outbound send path: the action/runtime/outbound API accept one media reference, the payload builder uploads exactly one file, and the generic multi-media helper fans media out as separate sends. A narrow direct status interaction path can pass multiple media files to a Discord interaction reply, but that is not the requested general Discord message integration behavior.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the Discord plugin: add an ordered multi-attachment input shape for message actions and outbound payloads, validate Discord per-message count and size limits, build one Discord `files` payload for up to 10 files, split overflow into follow-up messages, and update focused Discord docs/tests.
