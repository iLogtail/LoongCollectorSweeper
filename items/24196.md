## Summary

Keep #24196 open. Current main still does not support Discord native multi-file uploads in one message: the Discord action paths normalize a single media/path/filePath value, sendMessageDiscord accepts one mediaUrl, and the Discord payload builder creates a files array with exactly one upload. Existing multi-media payload support fans media out across separate sends, which does not satisfy the requested up-to-10 attachments per Discord message.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the Discord plugin: add an ordered multi-attachment input shape for message actions and outbound payloads, validate Discord per-message file count and size limits, build one Discord payload files array for up to 10 files, split overflow into follow-up messages, and update focused Discord docs and tests.
