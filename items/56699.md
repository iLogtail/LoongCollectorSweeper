## Summary

Keep open. Current main still has the reported Telegram forum-topic command auth gap: native command auth resolves topic config but explicitly disables topic/group groupPolicy overrides, while normal Telegram message handling enables those overrides. The open related PR #56741 targets the same one-line root cause but is not merged into the checked-out main.

## Best Possible Solution

Keep #56699 open until PR #56741 or an equivalent fix lands. The likely minimal product/code path is to make Telegram native command auth use topic/group policy overrides just like regular message auth, then add a regression test for a forum topic with `topics["N"].groupPolicy: "disabled"` rejecting native commands for non-owned bot accounts.
