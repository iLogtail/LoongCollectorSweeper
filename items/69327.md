## Summary

Keep #69327 open. Current main has evidence for related workspace/recreate improvements, but I do not see an implementation that carries the parent sandbox Docker env/context through `sessions_spawn(sandbox="inherit")`. The issue comment’s root-cause analysis still matches current source: spawn context carries workspace/group metadata, `inherit` is treated as a sandbox eligibility mode, and the child sandbox is resolved fresh from child session/agent config.

## Best Possible Solution

Keep the issue open and add a focused regression for `sessions_spawn` from a sandboxed parent with `sandbox.docker.env` configured. If parent inheritance is intended, thread a narrowed parent sandbox context or `SandboxContext.docker.env` through the spawned-run path and merge it before target-agent overrides. If target-agent-only semantics are intended, update the `sandbox="inherit"` docs/tool text and add diagnostics showing which agent config supplies the child sandbox env.
