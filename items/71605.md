## Summary

Keep open. #71605 is recent, concrete, non-protected, and current main still matches the reported Gateway/CLI/embedded-lock failure shape rather than providing close evidence.

## Best Possible Solution

Keep #71605 open as a real Gateway/CLI integration bug. The best maintainer path is to either make `openclaw agent` return the accepted `runId` promptly and use `agent.wait` or another explicit wait path for final results, or stop falling back to same-state embedded mode when a daemon is running. Add a daemon/state-dir guard or separate state-dir support for embedded fallback, improve the error message, document the limitation in `docs/cli/agent.md`, and add regression coverage around the 60s Gateway timeout and 10s session-lock fallback behavior.
