## Summary

Keep open. Current main has related compaction tuning via reserveTokens/keepRecentTokens, but it does not implement the requested subagent operatingPrinciples/maxFailureAttempts config, nor the proposed proactiveThresholdRatio/recentMessagesToKeep fields. PR #61021 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If accepted, implement a first-class config surface in src/config types/schema/help/generated docs and thread it through src/agents/subagent-spawn.ts into src/agents/subagent-system-prompt.ts with focused tests. For compaction, either map the requested threshold behavior onto the existing reserveTokens/keepRecentTokens model or explicitly document why those existing knobs are the supported replacement.
