## Summary

Keep PR #61051 open. Current main does not implement the central approval-routing changes: approval requests are still broadcast before forwarding is evaluated, turn-source routing still lacks approval-kind/source-target context, and Telegram target-only recipients are still treated as authorized exec approvers in current tests. The PR is not safely closeable as implemented, duplicate, or superseded; it likely needs a rebase onto the newer shared approval helper plus maintainer review of the existing bot findings.

## Best Possible Solution

Keep the PR open for maintainer/product review and rework. The best path is to rebase it onto current main, implement target-owned broadcast suppression in the shared approval request helper, extend turn-source route checks with approval kind plus origin fields, preserve explicit target forwarding semantics without broadening approval authority, and keep the exec/plugin/Telegram regression tests aligned with the reviewed bot comments.
