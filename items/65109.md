## Summary

Keep open. Current main still displays cache read/write usage in /status but estimates the status cost from input/output tokens only, so the reported undercount remains actionable in OpenClaw core.

## Best Possible Solution

Keep the issue open and fix the /status path by passing cacheRead and cacheWrite into estimateUsageCost in src/status/status-message.ts, then add a regression test in src/auto-reply/status.test.ts covering the issue's Anthropic-style pricing case where cacheRead materially changes the displayed Cost line.
