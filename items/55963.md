## Summary

Keep open. Current `main` still matches both reported gaps: the bundled `session-memory` hook derives the filename date from `toISOString()` (UTC), and it only handles/registers `command:new` and `command:reset`, not `subagent_ended`.

## Evidence

- **Filename date still comes from UTC ISO string:** `saveSessionToMemory` builds `dateStr` with `now.toISOString().split("T")[0]`, so the file prefix is UTC-based rather than local-date-based.
  - file: [src/hooks/bundled/session-memory/handler.ts:85](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/hooks/bundled/session-memory/handler.ts#L85)
  - command: `nl -ba src/hooks/bundled/session-memory/handler.ts | sed -n '83,86p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Handler still ignores non-command events:** The hook returns unless `event.type === "command"` and the action is `new` or `reset`, so `subagent_ended` cannot reach this code path.
  - file: [src/hooks/bundled/session-memory/handler.ts:55](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/hooks/bundled/session-memory/handler.ts#L55)

- **Bundled hook metadata only subscribes to command events:** The shipped HOOK metadata declares only `command:new` and `command:reset` for `session-memory`; there is no `subagent_ended` registration.
  - file: [src/hooks/bundled/session-memory/HOOK.md:10](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/hooks/bundled/session-memory/HOOK.md#L10)
