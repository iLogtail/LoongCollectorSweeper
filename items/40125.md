## Summary

Keep open. Current `main` has reactive overflow recovery and session-status token reporting, but it does not implement the requested proactive near-limit user warnings with one-time `/reset` or `/new` guidance before failure.

## Evidence

- **Overflow UX is still post-failure:** The embedded runner only emits reset guidance after overflow recovery and compaction attempts have already failed, returning `Context overflow: prompt too large for the model. Try /reset (or /new)...` as a terminal error payload.
