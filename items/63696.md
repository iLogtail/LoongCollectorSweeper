## Summary

Keep open. Current main does not contain the PR's run-scoped duplicate-persistence suppression, and the current fallback path still sends a prompt on each embedded fallback attempt without any callback from the transcript append path to suppress a later duplicate user write. The PR remains a plausible focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The maintainer path is to review or land a current-main equivalent that wires a run-local persisted-current-turn callback from the session append path through `guardSessionManager`, `runEmbeddedPiAgent`, and `runAgentAttempt`, then suppresses only the duplicate persistence on later fallback retries with regression coverage in the agents tests.
