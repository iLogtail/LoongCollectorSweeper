## Summary

Keep open. Current main still rejects a bare `OSTYPE=linux` before reaching the Linux installer path, while this PR is a narrow one-line compatibility fix that broadens the match to `linux*`. The existing discussion and Greptile comment support the reported openSUSE compatibility gap, and I did not find a shipped or canonical superseding fix on main.

## Best Possible Solution

Keep the PR open. The best path is to land or request a tiny revision of the `linux-gnu*` to `linux*` detection change, ideally with a focused regression test for `OSTYPE=linux`, while tracking any broader openSUSE package-manager support separately if needed.
