## Summary

Keep #55216 open. Current main still drops recoverable Telegram media-group download failures from the album payload with only a log warning, while the single-media path sends a user-visible failure message. Existing tests cover continuing with the remaining media, but they do not assert any warning for the skipped album item. The partial-download subclaim appears mitigated by the shared media fetch reader rejecting on idle timeout before saving a buffer, but the central user-visible silent-loss bug remains actionable in the Telegram plugin.

## Best Possible Solution

Implement the warning in `extensions/telegram/src/bot-handlers.runtime.ts` by tracking recoverable media-group failures and `null` media results, then sending a user-visible Telegram reply such as “Received N of M images; K could not be fetched” after or before processing the remaining media. Add focused tests for private/chat media groups and channel posts that assert both the partial album still reaches the agent and the skipped item is reported. Treat configurable media-group flush timing as a separate follow-up, likely under the broader batching/config issue rather than as the blocker for this bug.
