## Summary

Keep #54685 open. Current main still lacks the proposed generic VPS administration hardening note in `docs/vps.md`; adjacent docs cover Gateway access and Tailscale Serve, but they do not clearly separate securing VPS admin access from exposing/accessing the OpenClaw Gateway. The PR is a small, coherent docs-only contribution and is not protected or maintainer-authored.

## Best Possible Solution

Keep the PR open and route it through normal docs review. The best product/docs path is to land a short provider-agnostic `docs/vps.md` note that distinguishes VPS administration hardening from OpenClaw Gateway access, while linking the existing public Gateway remote access and Tailscale docs for implementation details.
