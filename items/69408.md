## Summary

Keep issue #69408 open. Current main still emits generic loop warnings through `log.warn`, and the before-tool-call loop-detection path does not carry isolated cron target, run lifetime, or cron interval metadata needed to implement the requested downgrade. Related #40217 covers stale history false positives, but it does not supersede this narrower isolated-cron log-level policy.

## Best Possible Solution

Keep the issue open and implement a narrow agents/runtime fix: propagate cron trigger, job id, sessionTarget, and enough bounded-run timing or schedule metadata into the before-tool-call loop-warning policy; downgrade only non-critical warnings for fresh isolated cron runs whose lifetime is bounded by the schedule interval; preserve warn/error behavior for interactive, persistent, unknown-tool, global circuit-breaker, and critical loop cases; add focused tests for both the isolated-cron downgrade and inverse cases.
