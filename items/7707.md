## Summary

Keep #7707 open. Current main has adjacent source/provenance concepts, session visibility filtering, temporal decay, Memory Wiki provenance, and active-memory review surfaces, but it still lacks persistent source-based memory trust tiers, trust-policy config, review/expiration enforcement by trust level, and trust-weighted retrieval/downranking.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete implementation should add structured origin trust metadata at memory ingestion, persist it in memory schemas/backends and SDK contracts, document config policy, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction and Memory Wiki flows, expose trust state where useful in UI/docs, and add regression coverage across builtin, QMD, active-memory, session-memory, and compaction paths. HMAKT99's AKF comment is useful prior art but should not be treated as shipped OpenClaw behavior.
