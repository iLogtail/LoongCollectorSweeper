## Summary

Keep open. Current main still lacks abort-aware queued-entry removal in `enqueueCommandInLane`, and `runEmbeddedPiAgent` still does not thread `params.abortSignal` into the session/global lane enqueue calls. The related PR #43243 remains the apparent open implementation track; closed PR #46174 was not merged per the provided context.

## Evidence

- **Queue API still has no abort option:** `enqueueCommandInLane` options only expose `warnAfterMs` and `onWait`; there is no `signal?: AbortSignal` option for queued-entry cancellation.
  - file: [src/process/command-queue.ts:242](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/process/command-queue.ts#L242)

  - file: [src/process/command-queue.ts:258](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/process/command-queue.ts#L258)
