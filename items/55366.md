## Summary

Keep open. Current `main` still handles context-overflow/compaction-failure hard resets by switching to a fresh session and telling the user to try again; there is no automatic injection of the prior task goal or in-progress state into that new session. Main does have post-compaction AGENTS refresh and manual compaction-checkpoint restore APIs, but those are adjacent features, not a fix for this bug.
