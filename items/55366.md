## Summary

Keep #55366 open. Current main still has the exact hard context-overflow reset path and user-facing message, and the reset helper rotates to a fresh session without injecting the last task goal, last progress state, or a reset note into the new transcript. Existing compaction recovery, memory flush, and reset hooks reduce related context-loss risk but do not solve the reported hard-reset recovery path.

## Best Possible Solution

Implement a hard-overflow reset recovery path in the agent runner: before rotating the session, persist or synthesize a small recovery prelude containing the reset reason, last user/task instruction, and best available progress summary, then inject it into the replacement session or queued retry. Keep related retry-loop diagnostics in #63216 separate unless maintainers choose to consolidate them explicitly.
