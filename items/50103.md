## Summary

Keep issue #50103 open. Current main still does not implement additive `agents.defaults.systemPrompt`, `agents.defaults.rules`, or per-agent `systemPrompt`/`rules`; the existing `systemPromptOverride` surface is a full replacement path, not the requested config-injected rules appended after bootstrap files. The related PR #52038 attempted this feature but is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and implement a distinct additive prompt contribution surface rather than reusing `systemPromptOverride`. The likely path is to add typed/schema-backed `agents.defaults.systemPrompt` and/or `agents.defaults.rules` plus per-agent overrides, resolve them into the existing prompt-building path after workspace bootstrap/channel context according to a documented precedence rule, preserve `systemPromptOverride` as a full replacement escape hatch, and add schema/runtime/docs tests for CLI and embedded runs. Existing top-level config `$include` support can help externalize config files, but nested rule include semantics should be designed deliberately before shipping.
