## Summary

Keep #13911 open. Current main improved subagent completion delivery fallback behavior, but it still does not provide a first-class per-channel suppression or reroute policy for subagent completion announcements. The docs and tests still describe/assert requester-channel delivery when an external route exists, and #69492 remains only a broader open design proposal, not shipped behavior.

## Best Possible Solution

Keep #13911 open. The best product path is a core-supported subagent completion delivery/audience policy in config, schema, docs, and tests, applied before external requester-channel delivery is resolved. It should cover Telegram/iMessage-style channels without relying on AGENTS.md behavior, sentinel replies, sanitizer proxies, or the current routing-only hooks.
