## Summary

Keep PR #54308 open for maintainer judgment. Current `main` does not implement the PR’s central Discord teardown behavior: the Discord inbound worker exposes only `deactivate()`, `KeyedAsyncQueue` has no idle-drain API, provider teardown only calls `deactivateMessageHandler?.()`, and there is no managed bot identity account map or `SenderManagedAccountId` context field. The related PR #54270 is closed unmerged, so it is not a canonical shipped replacement. The PR may need refocusing because it bundles unrelated Telegram/plugin type changes and has an unresolved review concern, but the Discord race it tries to address is not clearly solved on `main`.

## Best Possible Solution

Keep this PR available for maintainer review, but require it to be narrowed into a focused Discord lifecycle fix: expose an idle-drain path from `KeyedAsyncQueue` through the Discord inbound worker/message handler, use account-safe managed bot identity attribution only if still needed, and add provider/queue regression tests. Unrelated Telegram/plugin-schema changes should be removed or handled separately unless they are still missing on current `main`.
