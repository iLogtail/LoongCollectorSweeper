## Summary

Keep #70990 open. Current main still does not implement `model_failover` or `model_failure_terminal`; the failover paths expose logs, user replies, and coarse model-call hooks, but not the requested structured plugin events. This is plausible core plugin API work rather than ClawHub-only work because external plugins need a missing in-process hook surface before they can observe these failover seams. The linked #70976 was closed as superseded by this PR, and the Greptile review described the PR as safe with only minor P2 cleanup notes.

## Best Possible Solution

Keep #70990 open for maintainer review. The best path is a narrow, documented core plugin hook API for failover decisions and terminal all-models-failed outcomes, with tests at the hook runner and failover emission seams; broader fallback UX, prompt re-injection, and circuit-breaker behavior should remain separate follow-up work under #65824 or narrower issues.
