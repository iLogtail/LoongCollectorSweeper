## Summary

Keep open. Current main logs model failover decisions and exposes generic model-call telemetry, but it does not provide the requested `model_failover` or `model_failure_terminal` plugin hooks. This is a plausible core plugin/API seam, not ClawHub-only work, because an external plugin cannot observe these failover and terminal-before-reply outcomes without core emitting a first-class hook. The PR discussion and linked #70976/#65824 context support the gap; maintainer review should focus on API shape, payload sanitization, docs, and tests rather than cleanup closure.

## Best Possible Solution

Keep #70990 open for maintainer review. The best path is to land a generic core hook seam if maintainers accept the API, with sanitized payloads, targeted hook-runner tests, failover emission tests, and public plugin-hook documentation. Leave broader #65824 items such as non-verbose notices and workspace-context re-injection tracked separately.
