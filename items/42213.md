## Summary

Keep open. The provided GitHub context shows an open, unmerged same-author PR (#42258) with closing syntax for this issue, so this issue should remain open while that candidate fix is reviewed. Current main also still matches the reported failure shape: Control UI busy state is keyed off `chatRunId`, queued messages do not flush while busy, and the UI clears `chatRunId` through chat terminal events rather than an `agent.wait` watchdog.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. If the PR's watchdog approach is accepted, land it or an equivalent Control UI `agent.wait` recovery with regression coverage; if the PR is rejected, keep this issue as the tracker for a safer UI recovery path.
