## Summary

Keep open. Current main still has the reported failure path: host exec uses the shared env sanitizer, the sanitizer/policy do not strip `OPENCLAW_SERVICE_*` or `OPENCLAW_SYSTEMD_UNIT`, and gateway credential resolution still changes local CLI token precedence when `OPENCLAW_SERVICE_KIND=gateway` is inherited. PR #62204 is an open, directly related fix, but it has not landed.

## Best Possible Solution

Keep #61095 open until a current-main-compatible fix lands. The likely path is to adapt/rebase #62204 or equivalent: strip `OPENCLAW_SERVICE_*` and relevant supervisor runtime markers from host exec inherited env, preserve intentional exec markers like `OPENCLAW_SHELL=exec` and `OPENCLAW_CLI=1`, and add focused regression coverage for both sanitized env contents and clean `openclaw cron list`-style CLI child exit behavior.
