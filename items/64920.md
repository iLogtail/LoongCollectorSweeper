## Summary

Keep open. Current main still keys Feishu message processing by account/chat for ordinary messages, so topic-group turns in the same chat still share one sequential lane. The PR addresses a real remaining bug, but it should be narrowed and rebased because the submitted branch includes unrelated CI-unblock changes and current main now documents/implements Feishu native topic sessions around canonical `thread_id` semantics.

## Best Possible Solution

Keep this PR open for maintainer review, but require a narrow rebase or replacement patch: change only the Feishu sequential-key path, derive topic lanes from the effective group session scope and current `resolveFeishuGroupSession` topic precedence, preserve chat-wide `/stop` and `/btw`, preserve DM and normal-group FIFO behavior, and add focused `sequential-key` regression tests. Split unrelated CI/chore fixes into separate PRs if they are still needed.
