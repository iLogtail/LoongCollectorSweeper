## Summary

Keep open. Current main still constructs and uses IPv4-only Telegram fallback attempts even when `channels.telegram.network` is explicitly configured. The issue's exact requested behavior is acknowledged by open PR #42059, but that PR is still open/unmerged and current main/v2026.4.24 retain the fallback behavior.
