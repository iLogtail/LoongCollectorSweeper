## Summary

Keep #51394 open. Current main has partial browser network inspection through the control API/CLI (`/requests` and `/response/body`), which overlaps the related closed #42369, but the agent-facing `browser` tool still does not expose a `network` act kind, `requests` action, `responsebody` action, or a true request-interception API. The central request remains actionable as a browser plugin/tool surface decision rather than already implemented.

## Best Possible Solution

Keep the issue open and narrow the implementation path: add agent-facing browser tool actions for request listing and response-body capture by reusing the existing control routes, with schema/docs/tests, then separately design any true interception API around Playwright routing/CDP with explicit SSRF, auth, and target-scope safeguards.
