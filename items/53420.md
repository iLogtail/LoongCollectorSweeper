## Summary

Keep open. Current `main` does not already implement this PR's permission normalization, and the proposed one-line redirect-path fix still leaves the broader `saveMediaBuffer`/local-file write path with the same umask-sensitive behavior that the review discussion called out.

## Evidence

- **Media store contract says files should be readable by non-owner UIDs:** `src/media/store.ts` documents that saved media files are intentionally readable by non-owner UIDs for Docker sandbox access and sets `MEDIA_FILE_MODE = 0o644`.
  - file: [src/media/store.ts:19](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/store.ts#L19)
  - command: `nl -ba src/media/store.ts | sed -n '15,25p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Buffer-backed saves still rely only on `fs.writeFile(..., { mode })`:** `writeSavedMediaBuffer` still writes with `fs.writeFile(dest, params.buffer, { mode: MEDIA_FILE_MODE })` and does not normalize permissions afterward.
  - file: [src/media/store.ts:301](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/store.ts#L301)
  - command: `nl -ba src/media/store.ts | sed -n '301,310p'`

- **Redirected URL saves still rename without post-rename chmod:** The URL branch in `saveMediaSource` still downloads to a temp file and then `rename`s it into place with no subsequent `chmod`, so the PR's proposed behavior is not on `main`.
  - file: [src/media/store.ts:376](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/store.ts#L376)

  - file: [src/gateway/chat-attachments.ts:406](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/chat-attachments.ts#L406)

  - file: [src/media/store.redirect.test.ts:86](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/store.redirect.test.ts#L86)

That means this is not a stale/no-longer-needed PR; it still needs maintainer judgment.
