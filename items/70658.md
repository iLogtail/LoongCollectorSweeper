## Summary

Keep #70658 open. Current main still validates `entry?.config` even when it is undefined in non-activating plugin load paths, and the exact fix is only represented by open PR #70811, not by current main.

## Best Possible Solution

Keep #70658 open until PR #70811 or an equivalent change lands on main. The best fix is to make non-activating snapshot/CLI metadata loads defer plugin config validation only when config is not materialized, while preserving validation for real activation and for non-activating loads that do have explicit config. Add focused regression coverage for manifest-only snapshots and CLI metadata loads with a strict `configSchema` and undefined `plugins.entries.<id>.config`.
