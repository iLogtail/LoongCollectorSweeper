## Summary

Keep open. The reported bug still exists on current main: plugin config validation still receives `entry?.config` unconditionally in non-activating snapshot and CLI metadata paths, and `validatePluginConfig` still normalizes `undefined` to `{}` before AJV validation. PR #70811 is the right active fix path, but it remains open and unmerged.

## Best Possible Solution

Keep #70658 open until PR #70811 or an equivalent maintainer-reviewed patch lands. The fix should guard validation in non-activating snapshot and CLI metadata paths when config is truly undefined, preserve schema-default behavior for deferred plugin config where safe, keep real activation validation strict, and retain focused regression tests in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.

## Evidence

For a strict schema with top-level `required`, an omitted plugin config is still validated as `{}`, matching the false-positive missing-required-property failure described in #70658.
