## Summary

Keep open. Current main has added session maintenance, rotation, and orphan transcript cleanup, but the reported doctor failure mode is not fully fixed: `openclaw doctor` still loads and clones the whole `sessions.json`, and neither doctor nor `openclaw sessions cleanup` specifically removes accumulated `sessions.json.*.tmp` files.

## Best Possible Solution

Keep the issue open. A good fix would add a doctor/session-cleanup maintenance path for stale `sessions.json.*.tmp` files, avoid or cap full `sessions.json` parsing/cloning during doctor health checks, and add regression coverage with an oversized store plus stale temp files. The SQLite-backed metadata store can remain a broader follow-up if maintainers want a storage migration.
