## Summary

Keep open. Current `main` at `4e4aeacae4a87f2fe09125d462d88cd71a18cb66` still drops `/hooks/agent` `accountId` in normalization and still hand-builds a cron delivery object without `accountId`, while cron delivery resolution still expects `job.delivery.accountId` to carry the explicit account selection.

## Evidence

- **Hook payload type and normalizer still omit accountId:** `HookAgentPayload` has `message/name/agentId/.../channel/to/model/thinking/timeoutSeconds` but no `accountId`, and `normalizeAgentPayload()` returns an object with the same fields only.
  - file: [src/gateway/hooks.ts:213](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/gateway/hooks.ts#L213)
  - command: `nl -ba src/gateway/hooks.ts | sed -n '200,310p'; nl -ba src/gateway/hooks.ts | sed -n '399,452p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Hook dispatch still bypasses cron normalization and does not include accountId:** `dispatchAgentHook()` still constructs `delivery` inline with only `{ mode, channel, to }` for delivered runs, then passes that raw job to `runCronIsolatedAgentTurn()`; it does not call `normalizeCronJobCreate()` or otherwise populate `delivery.accountId`.
  - file: [src/gateway/server/hooks.ts:43](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/gateway/server/hooks.ts#L43)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '43,77p'`
  - sha: [4e4aeacae4a8](https://github.com/openclaw/openclaw/commit/4e4aeacae4a87f2fe09125d462d88cd71a18cb66)
- **Cron delivery logic still relies on delivery.accountId for explicit account routing:** `resolveCronDeliveryPlan()` reads `delivery.accountId`, and `resolveDeliveryTarget()` prefers explicit `jobPayload.accountId` from the job delivery config when choosing the delivery account.
  - file: [src/cron/delivery-plan.ts:54](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/cron/delivery-plan.ts#L54)

- **Hook docs still do not advertise accountId on /hooks/agent:** Both gateway config docs and cron-job docs list `/hooks/agent` fields without `accountId`, which matches the current implementation rather than a shipped fix.
  - file: [docs/gateway/configuration-reference.md:468](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/docs/gateway/configuration-reference.md#L468)
