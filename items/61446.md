## Summary

Keep #61446 open. Current main still only checks nested `gemini-cli-core` credential paths under `@google/gemini-cli`, while the PR targets a distinct npm v7+ hoisted sibling layout that is not covered by the existing shim test or the recursive fallback.

## Best Possible Solution

Keep the PR open for maintainer review and either land this additive fix or recreate it on main: add hoisted sibling `gemini-cli-core` credential paths in `extensions/google/oauth.credentials.ts`, keep the regression test in `extensions/google/oauth.test.ts`, and validate with the relevant Google extension tests/changed gate.
