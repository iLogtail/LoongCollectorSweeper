## Summary

Keep open. Current `main` already has partial overlap for oversized tool-result truncation and compaction safety, but it does not implement the session-level cacheRead/token-budget guardrails and retained-context attribution requested in #41312.

## Evidence

- **Tool-result caps and transcript truncation already exist:** `agents.defaults.contextLimits.toolResultMaxChars` is documented as a live tool-result cap used for persisted results and overflow recovery, and the runtime truncation path rewrites oversized or aggregate tool results in the session transcript.
  - file: [docs/gateway/config-agents.md:164](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/gateway/config-agents.md#L164)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Overflow handling uses those truncation guardrails:** Prompt precheck and overflow recovery both call tool-result truncation before or after compaction when oversized retained tool results are the likely cause of context overflow.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2203](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-embedded-runner/run/attempt.ts#L2203)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Compaction config typos fail closed, but there is no effective-config surface:** Compaction config is defined with `z.object(...).strict()`, and invalid config paths raise `INVALID_CONFIG` instead of silently falling through. That addresses silent unsupported keys differently than the issue proposes, but it is not an explicit doctor/status 'effective compaction config' view.
  - file: [src/config/zod-schema.agent-defaults.ts:169](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/zod-schema.agent-defaults.ts#L169)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Requested session-budget and attribution features are still absent:** A repo-wide search found no current implementation or docs for the issue's proposed session-level controls/diagnostics such as `cacheRead/totalTokens`, `avgTokensPerTurn`, `block-heavy-tool-result`, `effective compaction config`, or retained-context-category attribution.
