## Summary

Keep open. Current main has adjacent mechanisms: Discord preview edits/tool-progress, status reactions, and a background task ledger with lost reconciliation. Those do not satisfy the requested first-class persistent per-run/channel status surface with explicit terminal/unknown semantics. Normal interactive chat turns still do not create task records, there is no session activity API or WS statechange surface, and the related #59719/#71630 items are open narrower work rather than merged superseding fixes.
