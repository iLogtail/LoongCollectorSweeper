## Summary

Keep open. Current `main` still appears to have the same-provider transient probe slot behavior this PR fixes: the primary candidate is identified, but the cooldown probe dedup check and slot-consumption path do not exempt `isPrimary`, and the regression tests described in the PR are not present on `main`.

## Evidence

- **Primary candidate is tracked but not exempted from probe-slot dedup:** `runWithModelFallback` sets `const isPrimary = i === 0`, but the transient cooldown probe skip path only checks `cooldownProbeUsedProviders.has(candidate.provider)` with no `!isPrimary` guard.
  - file: [src/agents/model-fallback.ts:748](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/model-fallback.ts#L748)

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Transient probe slot is still consumed unconditionally after failure:** When a transient cooldown probe attempt fails, `cooldownProbeUsedProviders.add(transientProbeProviderForAttempt)` still runs without any primary-candidate exception, matching the root cause described in the PR.
