## Summary

Current main already fixes the proxy-only Gemini memory embedding failure through the shared memory-host HTTP helper rather than this PR's Gemini-specific fetchImpl propagation. Gemini direct and batch embedding requests route through that helper, which now enables trusted env-proxy mode when HTTP_PROXY/HTTPS_PROXY applies and keeps strict mode for NO_PROXY/ALL_PROXY-only cases. The unreleased changelog records the shipped-on-main fix as #71506 for #52162.

## Best Possible Solution

Close #64095 as implemented on current main by the shared #71506 memory-host SDK proxy-mode fix. Keep the helper-based implementation and do not merge the stale Gemini-specific fetchImpl propagation PR.
