## Summary

Keep #8285 open. Current main still has no supported configurable pre-processing text acknowledgement, the documented config surface remains reaction/status/typing based, and the existing hook paths either short-circuit dispatch or are documented as terminal dispatch ownership rather than a stable interim-ack send contract. The issue is concrete, has multi-channel user evidence, and the latest provided maintainer-side Codex review also explicitly kept it open.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and per-channel/account overrides or a documented plugin hook/send delegate that is explicitly safe for interim delivery, then update config types, zod schema, generated help/schema, public docs, and channel tests for the affected messaging surfaces.

## Evidence
