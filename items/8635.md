## Summary

Keep #8635 open. Current main still does not expose raw, structured token usage through the agent-facing `session_status` tool. It returns a `/status`-style text card with `details.statusText`; structured session usage aggregation exists separately under Gateway `sessions.usage`, and the related fixes/docs only cover display and transcript fallback behavior.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured usage payload to `session_status` or defines an equivalent supported agent-facing usage API. The implementation should reuse existing transcript/session usage parsers and Gateway usage result types, include stable fields for session totals, last turn, costs, and provider windows/availability, and add focused session tool tests plus docs updates.
