## Summary

Current main still lacks a cron-scoped tool allow/deny config surface or inheritance from `tools.subagents.tools`. Cron has only per-job `payload.toolsAllow` plus a hard-coded delivery-mode policy that keeps the `message` tool enabled for announce/none delivery, and existing docs/tests describe that behavior as current.

## Best Possible Solution

Keep #12736 open for maintainer product direction. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny` or explicitly define cron inheritance from `tools.subagents.tools`, specify precedence against global/agent/provider/group/sandbox policy and per-job `payload.toolsAllow`, integrate with cron delivery-mode message-tool controls, then update schema/types/help/generated config metadata, docs, CLI/config reference, and tests for denied `message` across announce, none, and webhook delivery modes.
