## Summary

本 Issue 请求支持将原生自监控输入插件 input_internal_metrics 的事件直接路由至 Go 扩展插件 flusher_prometheus。维护者已确认该需求有效，指出当前阻塞点在于跨语言桥接时缺乏对应的事件类型转换逻辑，并提供了基于文件中转的临时替代方案。该请求属于核心管线适配增强，需主仓库代码改动，不符合保守关闭条件，故保持开启。

## Best Possible Solution

保持开启以跟踪该核心管线增强需求。建议贡献者或维护者跟进实现 input_internal_metrics 输出事件到 Go 扩展 Flushers 的类型转换适配逻辑。在核心适配完成前，用户可遵循维护者评论中提供的文件中转链路作为临时替代方案。若后续有配套 PR 提交，应关联本 Issue 并进入常规代码审查流程。

## Evidence

- **maintainer_acknowledgment:** 维护者 Takuka0311 在评论中确认需求合理，明确指出核心阻塞点为 input_internal_metrics 事件类型发往 Go 扩展代码时缺少转换逻辑，并非 native/extended 架构缺陷，理论上改动不大。
- **workaround_and_dev_path:** 维护者提供了经文件中转的可行替代链路（input_internal_metrics -> flusher_file -> input_file -> flusher_prometheus），并指出直接支持需一定开发量，属于主仓库核心管线/桥接层增强，无法通过外部插件独立绕过。
