## Summary

本 Issue 请求支持将原生自监控插件 input_internal_metrics 的指标直接通过 flusher_prometheus 推送至 Prometheus RemoteWrite。维护者已确认需求有效性，指出核心阻塞点为跨语言桥接层缺少事件类型转换逻辑，并提供了文件中转的临时替代方案。该需求属于核心管线增强，需主仓库代码改动，当前 main 分支尚未实现，故保持开启以跟踪后续开发。

## Best Possible Solution

保持开启以跟踪核心管线适配增强。建议维护者或贡献者针对 input_internal_metrics 到 Go 扩展插件的事件类型转换逻辑进行开发。在核心适配完成前，用户可遵循维护者评论中提供的文件中转链路（input_internal_metrics -> flusher_file -> input_file -> flusher_prometheus）作为临时替代方案。待配套 PR 提交并合并后，由 GitHub 自动关闭本 Issue。
