## Summary

本 Issue 请求支持将 input_internal_metrics 自监控指标直接通过 flusher_prometheus 推送至 RemoteWrite。维护者已确认该需求有效，指出当前阻塞点在于内部指标事件类型发往 Go 扩展插件时缺乏转换逻辑，需核心管线适配开发。当前无配套 PR，且已提供文件中转的临时替代方案，属于合理的核心功能增强请求，故保持开启。

## Best Possible Solution

保持开启以跟踪该核心管线增强需求。建议贡献者或维护者跟进实现 input_internal_metrics 输出事件到 Go 扩展 Flushers 的类型转换适配逻辑。在核心适配完成前，用户可遵循评论中提供的文件中转链路作为临时替代方案。若后续有配套 PR 提交，应关联本 Issue 并进入常规代码审查流程。

## Evidence

- **maintainer_acknowledgment:** 维护者 Takuka0311 在评论中确认需求有效，明确指出阻塞点为 input_internal_metrics 输出的事件类型发往 Go 代码时缺乏转换逻辑，需核心管线适配开发。
- **workaround_provided:** 维护者提供临时替代方案：input_internal_metrics -> flusher_file -> input_file -> flusher_prometheus。直接打通该链路需额外核心开发工作量。
- **core_pipeline_limitation:** 当前 main 分支尚未实现 native input 到 extended flusher 的自监控指标事件类型自动转换，属于核心功能增强请求，非外部插件或文档可独立解决。
