## Summary

本 Issue 请求支持将 input_internal_metrics 直接路由至 flusher_prometheus (RemoteWrite)。维护者已明确确认需求有效，并指出当前阻塞点在于原生输入插件输出的事件类型在跨语言桥接至 Go 扩展插件时缺乏转换逻辑。该请求属于合理的核心管线/桥接层增强，已有明确的临时替代方案与开发路径，不符合任何保守关闭条件，故保持开启。

## Best Possible Solution

保持开启以跟踪该核心管线增强需求。建议贡献者或维护者跟进实现 input_internal_metrics 输出事件到 Go 扩展 Flushers 的类型转换适配逻辑。在核心适配完成前，用户可遵循维护者评论中提供的文件中转链路（input_internal_metrics -> flusher_file -> input_file -> flusher_prometheus）作为临时替代方案。若后续有配套 PR 提交，应关联本 Issue 并进入常规代码审查流程。

## Evidence

- **maintainer_acknowledgment:** 维护者 Takuka0311 在评论中确认需求合理，明确指出核心阻塞点为 input_internal_metrics 事件类型发往 Go 扩展代码时缺少转换逻辑，并非架构缺陷，并提供了经文件中转的可行替代链路。
- **core_architecture_scope:** 该需求涉及 Native (C++) 到 Extended (Go) 插件管线的跨语言事件类型适配，属于主仓库核心路由与桥接逻辑增强，无法通过外部插件或配置独立绕过，需核心代码改动。
