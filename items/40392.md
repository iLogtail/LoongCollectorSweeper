## Summary

Current `main` and the latest release still use the numbered config backup ring (`.bak.1`-`.bak.4`), so this PR is not implemented or obsolete. The linked issue describes a real unmet behavior change, and PR review raised an unresolved migration/retention concern about deleting legacy backups too early, so conservative maintainer cleanup should leave it open.
