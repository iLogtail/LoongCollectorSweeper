## Summary

Current `main` already contains a targeted fix for subagent completion delivery routing. The spawn path preserves the parent requester route, and the announce path now falls back to the external requester route for completion delivery instead of trusting an internal/non-deliverable origin. The matching regression tests cover the channel-backed fallback case.

## Evidence

- **Spawn path preserves parent requester route:** Subagent spawn resolves `requesterOrigin` from the parent session/channel/thread context before launch, rather than relying on an empty child origin.
