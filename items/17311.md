## Summary

Keep open. Current main supports the SecretRef foundation with env/file/exec providers and documents 1Password through an exec-provider recipe, but it still does not implement native keyring/Keychain or native 1Password SecretRef provider sources requested by #17311. The latest issue discussion also already recorded a Codex review saying this should remain open for that narrower follow-through.

## Best Possible Solution

Keep #17311 open until native keyring/Keychain and 1Password SecretRef/provider sources are implemented across the SecretRef type contract, config schema, resolver runtime, configure/config CLI UX, docs, and tests, or until a maintainer explicitly decides that env/file/exec plus documented exec recipes are the final supported direction and opens or links a focused replacement for any remaining keyring/macOS work.
