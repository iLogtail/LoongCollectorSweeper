## Summary

Keep #7717 open. Current main still lacks a Discord role-mention wake trigger: Discord config/schema/docs expose `requireMention`, `ignoreOtherMentions`, sender `users`, and sender `roles`, but no `triggerOnRoles`-style role-mention trigger. The preflight path preserves `mentionedRoles` as metadata and uses it for generic mention/other-mention handling, but never compares mentioned role IDs to configured trigger role IDs.

## Best Possible Solution

Keep #7717 open. Implement the feature in the bundled Discord plugin as an additive guild/channel config surface such as `triggerOnRoles?: string[]`, wire it through config types, strict zod schema, generated metadata, docs, and preflight. Preflight should compare Discord `mentionedRoles`/`mention_roles` against configured role IDs and treat a match as a wake mention while preserving `requireMention`, `ignoreOtherMentions`, @everyone/@here, explicit bot mentions, reply-to-bot behavior, sender user/role allowlists, and routing. Add focused schema/metadata tests plus preflight tests for matched and unmatched role mentions.
