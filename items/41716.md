## Summary

Keep open. Current main does not implement the requested gateway-to-node file transfer surface: the `nodes` agent tool has no `file_push`/`file_pull` actions, the node command policy/default command sets have no `file.read`/`file.write`/`file.list`, and the headless node host still falls through unsupported non-plugin commands to `UNAVAILABLE`. The issue is also a real core/native-node request, not a clean ClawHub close, because it spans gateway allowlisting, native node runtimes, node-host security/approvals, CLI helpers, and the agent tool surface.

## Best Possible Solution

Keep the issue open and treat it as a core node-command feature request. A conservative implementation should add `file.read`/`file.write`/optional `file.list` as explicit node commands with size caps and path authorization, update platform allowlists and native/headless node runtimes, then expose `file_pull`/`file_push` through the `nodes` agent tool and CLI with focused tests/docs. A ClawHub plugin could cover only a headless-node subset today, but it would not satisfy the requested all-node core behavior without additional core/native surfaces.
