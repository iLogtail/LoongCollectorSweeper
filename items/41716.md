## Summary

Keep open. Current `main` at `bae7b54a85b17d180e4bfc28973a7e4fd1646a93` does not implement the requested gateway↔node file-transfer surface: there are no `nodes` tool actions for `file_push`/`file_pull`, no documented CLI `openclaw nodes file ...` commands, and the built-in node-host command handling still centers on `system.run`/`system.which` plus plugin-registered commands.

## Evidence

- **Nodes tool has no file transfer actions:** `NODES_TOOL_ACTIONS` lists status/describe/pending/approve/reject/notify/camera/photos/screen/location/device/invoke only; there is no `file_push` or `file_pull` action.
  - file: [src/agents/tools/nodes-tool.ts:19](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/tools/nodes-tool.ts#L19)

- **Generic invoke path is JSON RPC only, not workspace file transfer:** The `invoke` action forwards arbitrary `node.invoke` JSON and returns the raw result; it does not read a local file, write a pulled file into the gateway workspace, or provide dedicated file-transfer semantics.
