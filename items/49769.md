## Summary

Keep open. Current `main` still has the default-account-only binding behavior and lacks the docs/type updates proposed here, so this is not implemented on `main`. The PR also has unresolved review concerns: its JSDoc/documentation claim about `defaultAccount` does not match routing code, and its shared `parseBindingSpecs()` change would alter `agents unbind` behavior for existing channel-only bindings.

## Evidence

- **Current bind parsing still omits `accountId` by default:** On current `main`, `resolveBindingAccountId()` returns the explicit account id, a plugin hook result, or the forced default account; otherwise it returns `undefined`. There is no current logic that auto-promotes multi-account `--bind <channel>` to `"*"`.
