## Summary

Keep #8181 open. Current main has made useful progress for explicit local embeddings: `provider: "local"` falls back to the default downloadable GGUF model. But the zero-config request is still not satisfied because `node-llama-cpp` remains an operator-installed optional runtime, setup errors still tell users to install/rebuild it manually, and `provider: "auto"` still skips local unless an existing local file path is configured.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly document that local embeddings remain a manual optional runtime. A complete fix should cover native runtime install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, `auto` versus explicit `local` semantics, and matching docs/tests.
