## Summary

Keep open. Current `main` has strong tool-execution and dispatch hooks, but it still does not provide the requested validator-driven pre-response hard gate or two-phase response protocol. The repo’s own docs explicitly say final-answer gating is not exposed as a v1 plugin contract, and `VISION.md` frames this kind of gap as a valid core/plugin-API extension area rather than something to auto-close as ClawHub-only.

## Evidence

- **Current plugin hook surface stops at pre-tool, pre-LLM, and dispatch hooks:** The documented hook catalog includes `before_agent_reply` (synthetic reply/silence before the LLM call), `before_tool_call` (rewrite/block/require approval), `message_sending` (rewrite/cancel outbound delivery), and `reply_dispatch` (claim final dispatch). It does not describe a validator stage that must pass before a final answer is emitted.
  - file: [docs/plugins/hooks.md:66](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/plugins/hooks.md#L66)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '60,90p'`
  - sha: [80b6da72f5f0](https://github.com/openclaw/openclaw/commit/80b6da72f5f0ac5ade8262142f816b9287fb1cfe)
- **Typed hook contracts do not define a pre-response validator/gating API:** `PluginHookBeforeAgentReplyResult` only exposes `handled`, `reply`, and `reason`; `PluginHookReplyDispatchResult` only exposes dispatch ownership/result counts; `PluginHookBeforeToolCallResult` covers tool param rewrite/block/approval. There is no typed contract for mandatory-tool validation, retry/replan, or a final-answer gate that must pass before completion.
  - file: [src/plugins/hook-types.ts:173](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/plugins/hook-types.ts#L173)

- **Repo docs explicitly say final-answer gating is not supported in the Codex runtime contract:** The Codex harness support contract lists `Stop or final-answer gating` as not supported and says OpenClaw does not expose final-answer gating as a v1 plugin contract, with only a future opt-in primitive noted.
  - file: [docs/plugins/codex-harness.md:622](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/plugins/codex-harness.md#L622)
