## Summary

#13583 should remain open. Current main has adjacent plugin hook surfaces for tool-call blocking, final-answer revision, and outbound message cancellation, but it does not implement the requested pre-response policy-as-code gate: no two-phase tool-only planning protocol, responseGate/require_tools config, mandatory-tool accounting, fail-closed response validators, retry/fallback contract, or audit model.

## Best Possible Solution

Keep #13583 open as the canonical response-gate policy-as-code design item. The best path is a versioned core/plugin API for mandatory tool-call accounting, typed validators, fail-closed final-response gating, bounded retry/fallback semantics, and audit output; once that contract exists, domain-specific policy packs can live in plugins/ClawHub.
