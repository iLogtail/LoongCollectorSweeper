## Summary

Keep open. Current main still appears to have the reported false-positive path: `gateway.nodes.denyCommands` entries are considered known only if they appear in the resolved platform/default allowlist or explicit `allowCommands`, while the documented dangerous commands named in the issue remain outside those defaults. The related PR context is useful and correctly identifies the root cause, but those PRs are closed/unmerged and current main does not contain the needed `DEFAULT_DANGEROUS_NODE_COMMANDS` inclusion in `listKnownNodeCommands`.
