## Summary

Current `main` does not show this fix as already implemented. The schema normalization path now runs through `normalizeToolParameterSchema`, but that code still passes top-level `required: null` through unchanged for non-Gemini/non-compat paths, and the current tests do not cover the malformed-`required` cases this PR is trying to fix.
