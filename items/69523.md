## Summary

Keep open. Current main has internal plumbing that can render tool calls/results expanded when an `autoExpandToolCalls` boolean is supplied, but the shipped Control UI passes `false` unconditionally. Persisted Control UI settings only include show/hide tool calls, and the OpenClaw config schema does not accept the proposed `ui.chat.expandToolsByDefault` key.

## Best Possible Solution

Keep the issue open and implement a real user-facing preference. The smallest aligned path is to add a browser-local `UiSettings` boolean such as `chatExpandToolsByDefault`, expose it in the Control UI chat/settings surface near the existing tool-call visibility toggle, pass it into `renderChat`, and add storage/render tests. If maintainers prefer the proposed global config key, add `ui.chat.expandToolsByDefault` to the config types, zod schema, help/docs, generated baselines, and wire the Control UI config payload into the chat render path.
