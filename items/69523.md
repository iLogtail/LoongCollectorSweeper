## Summary

Current main has the rendering/state plumbing for auto-expanding tool cards, but it is not exposed as a Control UI setting or config option. The chat view always passes `autoExpandToolCalls: false`, existing persisted UI settings only cover focus/thinking/show-hide tool calls, and the root `ui` config schema has no `chat.expandToolsByDefault`-style field. The requested user-facing option remains valid and should stay open.

## Best Possible Solution

Keep the issue open. Best implementation is to add a user-facing Control UI preference, likely a persisted browser UI setting near the existing chat toolbar/settings controls, wire it to `autoExpandToolCalls`, and add storage/render tests. If maintainers prefer a shared `openclaw.json` policy, add a strict schema-backed `ui.chat.expandToolsByDefault` field plus docs/config-schema metadata, then consume that value in Control UI bootstrap/render state.
