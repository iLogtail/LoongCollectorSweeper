## Summary

Current main already ships startup-triggered outbound notification capability via the existing `gateway:startup` hook and bundled `boot-md` startup automation, so this PR's user-facing goal is already covered without adding a new `gateway.startupNotification` config surface.

## Close Comment

Closing from a Codex review because current OpenClaw already covers this use case. The gateway already emits a `gateway:startup` hook after channels start (`src/gateway/server-startup-post-attach.ts:268`), the bundled `boot-md` hook runs `BOOT.md` on that startup event (`docs/automation/hooks.md:159`, `docs/automation/hooks.md:204`), and the boot runner explicitly supports sending outbound messages via the message tool (`src/gateway/boot.ts:50`). This behavior is already present in the released tree for `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so the PR’s requested startup notification capability is already implemented through the existing startup-hook automation path.
