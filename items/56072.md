## Summary

Keep #56072 open. Current main still rotates daily-stale sessions by creating a new session and archiving the old transcript without invoking the compaction memory-flush turn; the bundled session-memory hook remains manual-command scoped, and reset config still lacks an off/disabled mode or reset-time flush option.

## Best Possible Solution

Keep the issue open for a core session-reset design. The likely fix belongs in the session rollover/reset path: run the existing `agents.defaults.compaction.memoryFlush` silent agentic flush before archiving daily/idle reset transcripts when configured, add a clear reset policy such as `onReset: "flush" | "archive-only"` plus a first-class manual-only/off mode if accepted, and add focused tests for daily, idle, explicit `/new`/`/reset`, gateway reset, hook behavior, and docs/config schema alignment.
