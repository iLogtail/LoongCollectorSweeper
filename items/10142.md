## Summary

Keep #10142 open. Current main still does not implement or document the requested HOOK.md/internal `session:end` event. The repo now has stronger typed plugin `session_end` coverage for session rollover/reset cases, but docs and code still distinguish plugin hooks from internal hooks, and there is no `session:end` internal type guard, docs table entry, or completion-path emitter.

## Best Possible Solution

Keep #10142 open until maintainers either implement the internal HOOK.md `session:end` contract or explicitly document that typed plugin `session_end` is the supported replacement for workflow-orchestration completion hooks. A complete internal-hook implementation should add a typed `SessionEndHookEvent`/guard, emit `createInternalHookEvent("session", "end", ...)` from the relevant gateway, cron, and subagent completion paths, add regression tests, and update https://docs.openclaw.ai/automation/hooks.
