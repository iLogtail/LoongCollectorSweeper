## Summary

Keep #13627 open. Current main still serializes concurrent Slack sends per target/thread, but it does not add a configurable minimum delay between Slack outbound posts. The related implementation attempt, PR #26732, is closed and unmerged in the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #13627 open and implement a focused Slack outbound minimum-delay feature in the bundled Slack plugin send path. Add a documented configurable delay to Slack config types/schema/UI metadata, enforce it around every path reaching sendMessageSlack/postSlackMessageBestEffort including chunks, media follow-ups, DMs, channel messages, thread replies, and Slack action sends, and cover it with fake-timer tests. If maintainers prefer a generic channel throttle, track that separately rather than treating closed unmerged PR #26732 as the fix.
