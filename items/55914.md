## Summary

Keep #55914 open. Current main has solid QR/setup-code pairing and a bundled `device-pair` plugin that can send pasteable setup codes through chat, so the issue's "QR-only" premise is partly outdated. But the requested short cloud-resolved `openclaw invite` flow is not implemented: there is no invite CLI command, no `gateway.cloud`/`cloud.apiUrl` config surface, no `/api/v1/invite` integration, and the native mobile code only accepts setup-code/manual/deep-link inputs rather than resolving a 4-character invite code. The related Cloud Relay issue #55915 remains open and adjacent, not a shipped canonical replacement.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. The practical path is either to implement a first-class core/mobile invite flow with config, CLI, backend protocol, and native app resolution, or first define an explicit plugin/API seam that lets a ClawHub plugin provide short-code registration and mobile resolution. Existing `openclaw qr --setup-code-only` and `/pair` setup-code flows should remain documented as the current workaround, not treated as a full replacement for this request.
