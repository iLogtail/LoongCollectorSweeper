## Summary

Keep open. Current main already offers non-QR mobile pairing via setup codes (`openclaw qr --setup-code-only` plus mobile setup-code entry), so the issue's "no alternative to QR" rationale is outdated, but the specific requested feature is still not implemented: there is no `openclaw invite` command, no invite-code relay registration flow, and no `cloud.enabled` / `cloud.apiUrl` gateway config on `main` at `7130e56f872aea96577ac9fcbda0c7555a9ecc69`.
