## Summary

Keep open. Current main partially addresses the user problem through session-list rows and the TUI footer: gateway session rows resolve `modelOverride` ahead of stale runtime/default model data, and TUI session selection can use `label`/`displayName`. But the TUI header itself still renders only the formatted session key and agent label, with no active model and no label/displayName support. The issue discussion also explicitly narrowed the already-fixed Control UI work to a remaining TUI-only scope, so this is not safely closeable as implemented or duplicate/superseded.
