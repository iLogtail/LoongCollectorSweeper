## Summary

Keep open. Current main still does not implement PR #57016's ASCII fast-path or module-level emoji regex: `normalizeTextForComparison` still runs the Unicode property regex inline on every call, and the shared lowercase helper does not move or optimize that work. The PR is a focused, still-actionable performance change; the provided review context is positive and the author already addressed the only style nit in the second PR commit.

## Best Possible Solution

Keep PR #57016 open.
