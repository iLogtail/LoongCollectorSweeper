## Summary

Keep open. Current `main` has a config-gated workaround for string-only Chat Completions backends, but it does not implement this PR's automatic normalization for general OpenAI-compatible `openai-completions` providers. The reported NVIDIA/vLLM/LiteLLM/Ollama failure path remains plausible unless each affected model is manually marked with `compat.requiresStringContent: true`, and the PR discussion still contains unresolved correctness concerns.

## Evidence

- **Normalization on main is still opt-in via model compat:** `shouldFlattenOpenAICompletionMessages()` returns true only when `model.api === "openai-completions"` and `model.compat.requiresStringContent === true`; there is no universal wrapper on current `main`.
