## Summary

Keep open. Current `main` still has the behaviors this PR is trying to fix: Feishu local-image auto-send failures still fall through to plain text, media upload failures still echo `mediaUrl` back as `📎 ...` even when it is a local path, and speech-core still treats `feishu` as a native voice-note/opus channel.

## Evidence

- **Local image fallback still leaks the original path:** In the Feishu text send path, a failed auto-upload logs the error and then intentionally falls through to the normal text path, which will send the original absolute file path as message text.
  - file: [extensions/feishu/src/outbound.ts:185](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/src/outbound.ts#L185)
  - command: `nl -ba extensions/feishu/src/outbound.ts | sed -n '169,188p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Media fallback still sends local paths back to users:** On `sendMediaFeishu` failure, current main always falls back to `text: `📎 ${mediaUrl}`` with no local-path guard, so `/tmp/...` and `file://...` values would still be surfaced to end users.
  - file: [extensions/feishu/src/outbound.ts:275](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/extensions/feishu/src/outbound.ts#L275)
