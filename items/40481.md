## Summary

Keep open. Current `main` still sends the generic `Reply with: /approve <id> ...` text in forwarded exec-approval notifications, so this PR's requested UX change is not implemented on main. Main does already attach interactive approval buttons to forwarded payloads, which reduces urgency, but that is additive rather than a shipped replacement for the proposed text change.
