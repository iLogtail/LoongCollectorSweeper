## Summary

Current `main` already removed the PR's stated root cause: `wakeMode: "now"` main-session cron jobs no longer return `ok` immediately after enqueueing. They await `runHeartbeatOnce(...)` and only report `ok` after a `ran` result, with `skipped`/`error` propagated instead. Existing tests cover the wait-for-heartbeat path and the intentional busy-lane fallback, so the logging-only PR is obsolete against current `main`.

## Evidence
