## Summary

Keep open. Current `main` still returns every combined session-store entry through `sessions.list`, and the row builder explicitly serializes missing activity as `updatedAt: null` with an optional `sessionId`, so a stray `agent:<id>:sessions` store key would still surface as the reported phantom row. The linked PR #57419 is useful evidence that the problem was already identified, but the equivalent guard is not present on current `main` or on the latest shipped release tag I inspected.

## Evidence

  - file: [src/gateway/server-methods/sessions.ts:579](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/server-methods/sessions.ts#L579)

There is no guard requiring a real `sessionId` or non-null `updatedAt`.
  - file: [src/gateway/session-utils.ts:1511](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/session-utils.ts#L1511)
