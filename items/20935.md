## Summary

Keep open. Current main has useful memory event-journal pieces, including append-only JSONL events for recall, dreaming completion, and short-term promotion into MEMORY.md, but it does not satisfy the requested audit log for all agent memory-file changes with what/why/trigger/session/user context or tamper-evidence. The Pinchy comment is useful prior art, but it does not replace the missing OpenClaw memory-change audit coverage.

## Evidence

- **Memory event journal is narrower than requested:** The typed host event journal only models memory.recall.recorded, memory.promotion.applied, and memory.dream.completed events. The promotion event includes timestamp, memoryPath, count, and candidate source metadata, but not session/user/trigger/why fields and not a general changed-content diff for every memory-file mutation.

- **Long-term promotion is audited only for the short-term promotion path:** applyShortTermPromotions writes MEMORY.md and then appends a memory.promotion.applied host event. That is useful coverage for automated short-term promotion, but it is path-specific rather than an audit layer for all MEMORY.md or memory/** changes.
