## Summary

Keep #20935 open. Current main still only has a narrow memory event journal for recall, promotion, and dreaming, plus an unrelated config write audit log. It does not provide an append-only audit log for every MEMORY.md or memory/*.md mutation with what/when/why/session/user or tamper-evidence fields. The Pinchy comment is useful prior art, but it is external to OpenClaw and does not satisfy the requested OpenClaw memory audit surface.

## Best Possible Solution

Keep #20935 open. The best implementation is a first-class memory-change audit layer owned by memory-core plus shared host/tool write hooks, emitting append-only records for every MEMORY.md and canonical memory-file mutation with timestamp, operation, path, changed-content summary or diff, trigger/why, session/run/user context, and optional integrity signing. Add regression tests for promotion, memory flush, dreaming, and ordinary write/edit mutation paths, and document the audit surface in public memory docs.
