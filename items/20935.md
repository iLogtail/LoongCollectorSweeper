## Summary

Current main still does not implement the requested audit log for all agent memory-file changes. OpenClaw has a useful memory event journal for recall, promotion, and dreaming, plus Memory Wiki bridge support for those events, but ordinary memory flush/write/edit paths can still mutate MEMORY.md or memory files without the requested what/when/why/session/user/tamper-evidence audit record.

## Best Possible Solution

Keep #20935 open. The best implementation is a first-class OpenClaw memory-change audit layer owned by memory-core plus shared host/tool write hooks, emitting append-only records for every MEMORY.md and canonical memory-file mutation with timestamp, operation, path, changed-content summary or diff, trigger/why, session/run/user context, and optional integrity/signing. Add regression tests for promotion, memory flush, and ordinary write/edit mutation paths, and document the audit surface in the public memory docs.
