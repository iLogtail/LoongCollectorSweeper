## Summary

Keep #20935 open. Current main has a narrow memory event journal for recall, promotion, and dreaming, plus Memory Wiki bridge support for those events, but it still does not audit every MEMORY.md or memory-file mutation with what/when/why/session/user/tamper-evidence metadata. The issue discussion also supports this: Pinchy is useful external prior art, and the latest maintainer/Codex review comment explicitly kept the issue open for the same gap.

## Best Possible Solution

Keep #20935 open. The best implementation is a first-class memory-change audit layer owned by memory-core and shared host/tool write hooks, emitting append-only records for every MEMORY.md and canonical memory-file mutation with timestamp, operation, path, changed-content summary or diff, trigger/why, session/run/user context, and optional integrity signing. Add regression coverage for promotion, memory flush, ordinary write/edit tool mutations, and document the audit surface in the public memory docs.
