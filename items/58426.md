## Summary

Keep PR #58426 open. Current main still has the exact CLI install metadata bug the PR describes: unversioned ClawHub installs are persisted as resolved versioned specs, and update reuses that recorded spec, so users can remain unintentionally pinned. This is core CLI/plugin-install behavior, not optional ClawHub plugin work.

## Best Possible Solution

Keep the PR open and either merge or adapt its narrow fix on current main: in `src/cli/plugins-install-command.ts`, persist `raw` for explicit `clawhub:` installs and `preferredClawHubSpec` for bare-name ClawHub-preferred installs; keep explicit `clawhub:pkg@version` records pinned; remove the now-dead test mock entry; and retain focused regression coverage in `src/cli/plugins-cli.install.test.ts` plus the existing update tests.
