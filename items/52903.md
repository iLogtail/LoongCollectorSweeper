## Summary

Keep open. Current `main` still has the gap this draft targets: the OpenAI Responses/WebSocket conversion drops empty assistant error turns but does not collapse the adjacent duplicate user turns, `prepareSessionManagerForRun` does not strip retry-generated orphaned users after an assistant error exists, and the pre-prompt repair path only runs when the session leaf is already a user message. The PR discussion also usefully narrowed scope: the author called out a separate HTTP-path fix in `badlogic/pi-mono#2547`, so this OpenClaw PR is not obsolete just because that companion change landed elsewhere.

## Evidence
