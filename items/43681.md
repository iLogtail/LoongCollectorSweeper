## Summary

Keep open. Current main has generic auth-profile rotation/failover, but it does not implement the requested way to register multiple OpenAI Codex OAuth profiles for the same email/team account through `models auth login`. The CLI login path has no profile alias/profile id option, and OpenAI Codex OAuth still derives the profile id from email whenever email is present, so repeated same-email logins can still target the same `openai-codex:<email>` profile. The cross-referenced PR #43694 proposed `--profile-alias`, but it was closed unmerged.

## Best Possible Solution

Keep #43681 open and implement a supported OAuth profile discriminator for `models auth login`: either a generic `--profile-id`/`--profile-alias` passed through provider auth, or OpenAI Codex-specific team/account/org id extraction that wins over plain email when multiple subscriptions share an email. Add coverage that two OpenAI Codex OAuth logins with the same email but different alias/team id persist distinct `openai-codex:*` profiles and that existing same-provider rate-limit rotation can use them.
