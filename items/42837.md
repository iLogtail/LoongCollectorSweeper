## Summary

Keep open. Current main does not appear to contain the linked PR's account-aware Feishu dynamicAgentCreation fix: the dynamic agent helper still writes Feishu direct-message bindings without match.accountId, while Feishu routing re-resolves with the runtime accountId. Core routing intentionally treats bindings with missing accountId as default-account-only, so the reported named-account regression still has a plausible active code path on main.
