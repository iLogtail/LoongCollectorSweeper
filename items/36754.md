## Summary

Keep open. Current main still has the reported behavior: auto-discovery scans global/workspace plugin directories with `fs.readdirSync(..., { withFileTypes: true })`, accepts regular files, then skips anything where `entry.isDirectory()` is false. There is no `entry.isSymbolicLink()` handling in `src/plugins/discovery.ts`, and the only symlink coverage in `src/plugins/discovery.test.ts` is for rejecting escaping entries inside an already-discovered package, not for discovering a symlinked plugin directory. The related PRs in the provided GitHub context were closed unmerged, and the April 15 comment confirmed the issue still reproduced in a shipped build.

## Evidence

- **Current discovery loop skips symlink dirents:** `discoverInDirectory` reads entries with `withFileTypes`, handles `entry.isFile()`, then immediately continues when `!entry.isDirectory()`. Node dirents for symlinks do not satisfy `isDirectory()`, and this function has no fallback `statSync`/`isSymbolicLink` handling.
