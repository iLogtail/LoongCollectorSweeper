## Summary

Current main still matches the reported bug: extension-root discovery reads directory entries as Dirents, handles regular files, then continues unless `entry.isDirectory()` is true, so symlinked plugin directories under global or workspace `extensions` roots are skipped before package discovery. The explicit `plugins.load.paths` workaround remains a separate `fs.statSync()` path, and related fix PRs in the provided context are closed unmerged.

## Best Possible Solution

Keep #36754 open until `discoverInDirectory` resolves symlink Dirents safely for plugin package directories, keeps the existing security checks for package entries and hardlinks, handles broken symlinks deliberately, and adds a focused regression test proving a symlinked plugin directory under `~/.openclaw/extensions` or `<workspace>/.openclaw/extensions` is discovered.
