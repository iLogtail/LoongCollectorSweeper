## Summary

Keep open. Current main has exec/process cleanup hardening for service-managed runtimes, but that evidence is not strong enough to close this specific macOS launchd orphan-`grep` report: the relevant non-detached child/runtime cleanup appears to predate the reporter's 2026.3.24 version, and I could not do a fresh launchd reproduction in this read-only review.

## Evidence

- **Exec runs through the process supervisor with closed stdin for non-PTY shell commands:** `runExecProcess` builds a normal host exec as `[shell, ...shellArgs, execCommand]` with `stdinMode: "pipe-closed"`, so foreground shell pipelines are supervised rather than fire-and-forget.
  - file: [src/agents/bash-tools.exec-runtime.ts:669](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/bash-tools.exec-runtime.ts#L669)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '623,710p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Service-managed child spawns are explicitly non-detached:** The child adapter now sets `detached` only when not running under `OPENCLAW_SERVICE_MARKER`; the code comment says this is so `systemd/launchd can stop the full process tree reliably`. That is directly relevant to the reported orphan-under-launchd behavior.
  - file: [src/process/supervisor/adapters/child.ts:42](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/process/supervisor/adapters/child.ts#L42)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '21,72p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Foreground child stdin is actively closed:** When `stdinMode === "pipe-closed"`, the adapter immediately calls `child.stdin.end()`, which reduces the chance of pipe stages lingering because stdin stays open.
  - file: [src/process/supervisor/adapters/child.ts:75](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/process/supervisor/adapters/child.ts#L75)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '72,88p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Regression tests cover service-managed non-detached mode:** The test suite asserts that `OPENCLAW_SERVICE_MARKER` disables detached mode and removes the no-detach fallback, which is the main code path relevant to launchd-managed gateways.
  - file: [src/process/supervisor/adapters/child.test.ts:205](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/process/supervisor/adapters/child.test.ts#L205)

The reporter says they reproduced on `2026.3.24`, so I cannot confidently treat the current code as a later fix for this exact issue without a fresh repro proving the report is obsolete.
