## Summary

Current main still has the fail-open path this PR targets: an unpaired node reconnect creates a pending node-pair request but still publishes its allowlisted declared commands into the live node session, and `node.invoke` authorizes from that live command list. The PR is not obsolete or superseded by a merged fix; the linked issue remains open and the related PR #65543 was closed unmerged in the provided context.

## Best Possible Solution

Keep the PR open or replace it with a canonical maintainer fix. The concrete implementation should fail closed in `src/gateway/node-connect-reconcile.ts` by returning no effective commands for nodes without an approved node-pair record, update the allowlist and invoke regression tests accordingly, and consider a direct `node.invoke` approved-pair check if maintainers want an additional guard beyond live session command filtering.
