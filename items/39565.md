## Summary

Keep PR #39565 open. Current main still has the pre-PR TUI sanitizer behavior: it classifies copy-sensitive paths/URLs/file-like tokens using the raw token, only preserves file-like tokens with underscores, and does not strip markdown asterisks at token edges. That still lets long punctuation-wrapped dotted IDs and long dotted IDs without underscores be split with inserted spaces in assistant/TUI rendering. The PR remains a focused formatter/test fix for still-open issue #39505, with review feedback already addressed in the provided PR diff.

## Best Possible Solution

Keep #39565 open, or land a rebased/cherry-picked equivalent. The best maintainer path is the narrow TUI formatter fix in src/tui/tui-formatters.ts plus focused regression tests in src/tui/tui-formatters.test.ts: classify the punctuation-stripped candidate, strip markdown '*' at token edges, preserve dotted identifiers and digit-bearing hyphenated IDs, keep punctuation-only and natural-language hyphenated long tokens chunkable, and avoid unrelated lifecycle.test.ts formatting churn.
