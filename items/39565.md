## Summary

Keep #39565 open. Current main still uses the pre-PR TUI copy-sensitive-token classifier for the remaining cases this PR targets: punctuation/markdown-wrapped dotted identifiers and dotted identifiers without underscores can still be split by `sanitizeRenderableText`. The linked bug #39505 is still open in the provided context, and the PR contains a focused formatter/test change rather than obsolete or duplicate work.

## Best Possible Solution

Keep this PR or the linked #39505 work open. The best path is a focused TUI formatter update in `src/tui/tui-formatters.ts` that classifies punctuation-stripped candidates, strips markdown asterisks at token edges, preserves dotted identifiers and digit-bearing hyphenated IDs, and keeps punctuation-only/prose hyphen tokens chunkable, with regression coverage in `src/tui/tui-formatters.test.ts`.
