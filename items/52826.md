## Summary

Keep #52826 open. Current main has adjacent live `agent` streams for lifecycle/tool/item/plan events and session status broadcasts, but it does not provide the requested session-level `process.*` / `step.*` event contract with stable step identity and reload/reconnect source-of-truth semantics. The related ClawSweeper matches are adjacent but do not supersede this request.

## Evidence

- **No requested event names on main:** Targeted search found no `process.started`, `process.replanned`, `process.completed`, `process.failed`, or requested `step.*` event names. The only close string match was unrelated exec diagnostics (`exec.process.completed`).
  - command: `rg -n '"process\.(started|replanned|completed|failed)"|"step\.(created|started|updated|blocked|completed|failed)"|process\.started|process\.replanned|step\.created|step\.started|step\.updated|step\.blocked|step\.completed|step\.failed|process\.completed|process\.failed' src ui packages extensions docs test apps`
  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Existing agent event shape is adjacent but not the requested contract:** `AgentEventPayload` carries `runId`, `seq`, `stream`, `ts`, `data`, and optional `sessionKey`; `AgentItemEventData` has `itemId`, `phase`, `kind`, `title`, `status`, and optional `toolCallId`, while `AgentPlanEventData` only carries free-form string steps. This is not a stable session process event type with `process.*` / `step.*` names or stable per-plan step IDs.
  - file: [src/infra/agent-events.ts:5](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/infra/agent-events.ts#L5)

  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Session event subscribers get tool/status mirrors, not generic process steps:** Gateway mirrors live tool events to session subscribers as `session.tool`, and lifecycle start/end/error only become `sessions.changed` status payloads. There is no `session.process`, `session.step`, or persisted process-event replay path in this handler.
  - file: [src/gateway/server-chat.ts:930](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/gateway/server-chat.ts#L930)

It does not persist step events, plan steps, related message/artifact IDs, or a process event log that would survive reload.
