## Summary

Keep open. Current `main` still resolves exec-approvals paths from hardcoded `~/.openclaw` defaults instead of `resolveStateDir()`, so the reported `OPENCLAW_STATE_DIR` bug is not implemented on `main`. The PR also surfaced two adjacent hardcoded state-dir paths that remain on `main`, which argues against conservative closure.
