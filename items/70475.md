## Summary

Current main does not satisfy the requested dedicated Dreaming LLM model. The Dreaming docs still describe Dream Diary generation as a background subagent turn using the default runtime model, and the memory-core narrative calls do not pass a provider/model override. There is a partial `MemoryDreamingExecutionConfig.model` parser in host helpers, but the bundled memory-core manifest/docs do not expose that as supported config and the parsed value is not wired into the subagent run. The commenter’s point about local embedding models is useful for memory search cost, but it does not cover the Dream Diary subagent chat-model calls that currently run after light, REM, and deep phases.

## Best Possible Solution

Keep the issue open. The likely product/code path is a memory-core-owned implementation: declare a supported dreaming execution model config in `extensions/memory-core/openclaw.plugin.json`, document it in Dreaming and memory config docs, pass the resolved model/provider into `api.runtime.subagent.run` for Dream Diary generation, preserve default inheritance when unset, and add tests for global/per-phase config plus the existing subagent override trust behavior.

## Risks / Open Questions

- The existing parsed-but-unused `MemoryDreamingExecutionConfig.model` field could make the feature look more complete than it is; implementation should add regression coverage that proves the selected model reaches `api.runtime.subagent.run`.
- Subagent model overrides are intentionally gated by plugin trust policy and optional allowlists, so a fix should document the required `plugins.entries.memory-core.subagent.allowModelOverride` path or provide an explicit safe bundled-plugin policy.
