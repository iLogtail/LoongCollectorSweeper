## Summary

Keep open. Current main still does not provide a documented, schema-valid Dreaming-specific LLM model override. The Dream Diary path explicitly uses the default runtime model and the subagent run call does not pass any model/provider override. The existing local/Ollama embedding configuration mentioned in the issue comment can reduce embedding costs, but it does not satisfy the requested dedicated chat model for Dreaming narrative/subagent work.

## Best Possible Solution

Keep #70475 open. A proper fix should make a deliberate memory-core config shape, add manifest schema/docs/tests, preserve unset-as-inherit behavior, and wire the resolved Dreaming model into the existing plugin subagent model override path with the needed cron/fallback authorization coverage.
