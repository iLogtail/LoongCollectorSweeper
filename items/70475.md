## Summary

Keep #70475 open. Current main has partial, undocumented Dreaming execution config parsing, but the resolved model is not wired into the actual Dream Diary subagent run, and user docs still say Dreaming uses the default runtime model. The commenter’s embedding note is valid for the embedding-heavy memory search path, which already supports local/Ollama embeddings, but that does not satisfy the requested dedicated Dreaming LLM/runtime model.

## Best Possible Solution

Keep the issue open and complete the memory-core implementation: define the public config shape, document it under Dreaming/memory config, wire the resolved default or per-phase execution model into `api.runtime.subagent.run` using the existing `model`/`provider` runtime fields, preserve default-model inheritance when unset, and add tests that prove a configured Dreaming model reaches the narrative LLM call.
