## Summary

Keep open. Current main appears to mitigate the runtime credential-resolution part by merging the main auth profile store into non-main agent auth stores, including secrets-runtime reads, and that behavior is shipped in v2026.4.24. However, the issue asked for the setup-token/paste-token onboarding path itself to sync or validate credentials and clarify the Anthropic setup-token flow; current setup-token/paste-token write paths still upsert only the selected agentDir, the Anthropic prompt remains terse, and the related direct-sync PR context is still unresolved rather than merged.

## Best Possible Solution

Keep this open for maintainer judgment or narrow it into follow-ups: either explicitly bless the shipped main-store fallback and close the direct sync request, or implement/test direct setup-token and paste-token sibling propagation plus a clearer Anthropic setup-token prompt. Also align the multi-agent/auth docs with whichever product contract is intended.
