## Summary

Keep open. Current main still writes Anthropic setup-token and paste-token credentials only to the selected/current agent directory in the plugin auth-choice and models auth command paths, while only the OAuth helper has sibling-agent propagation. The Anthropic setup-token prompt remains a bare paste-token prompt, and I did not find post-write validation for the reported silent persistence class. The open related PR #52912 may address part of this, but it is not implemented on current main.
