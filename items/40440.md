## Summary

Current `main` still appears to have the reported Telegram group-history media loss. The Telegram no-mention path records only `sender/body/timestamp/messageId` into pending history, `HistoryEntry` itself has no media fields, and the later inbound context rebuild only forwards text-only history while current-turn media comes only from the newly mentioned message/reply. The issue discussion also points to PR #40490 as an unlanded fix for exactly this gap.

## Evidence

- **HistoryEntry is text-only on main:** `HistoryEntry` currently contains only `sender`, `body`, `timestamp`, and `messageId`; there is no `mediaPaths`/`mediaTypes` field to preserve skipped-message attachments.
  - file: [src/auto-reply/reply/history.ts:30](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/auto-reply/reply/history.ts#L30)

- **Telegram no-mention path stores placeholder/body only:** When a group message is skipped for lack of mention, the plugin records a pending history entry with only `sender`, `body: rawBody`, `timestamp`, and `messageId`. The downloaded `allMedia` paths are not stored in the history entry.
  - file: [extensions/telegram/src/bot-message-context.body.ts:294](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/extensions/telegram/src/bot-message-context.body.ts#L294)
