## Summary

Keep #29387 open. Current main still builds bootstrap context from the configured workspace directory only; there is no `agentDir` merge/override path and no warning for recognized bootstrap files placed in `agentDir`. The related fixes #29450 and #29545 are closed and unmerged in the provided GitHub context, so they do not supersede this issue.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best fix is either a narrow diagnostic like #29450 that threads explicit agentDir/agentId context through bootstrap callsites and warns when recognized bootstrap filenames exist in an explicitly configured `agentDir`, or a deliberate merge/override contract like #29545 with allowlist filtering, hook precedence, caller coverage, regression tests, and docs that distinguish workspace bootstrap files from `agentDir` state.
