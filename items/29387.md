## Summary

Current main still does not implement either requested remedy for #29387. Bootstrap injection still loads recognized files from the resolved workspace directory only, and I found no runtime diagnostic that warns when bootstrap-named files are present under `agentDir`. Docs now clearly tell users to put `AGENTS.md`/`SOUL.md`/`USER.md`/`MEMORY.md` in the workspace and explain that `agentDir` is state/auth/session storage, but documentation clarity is not the requested runtime warning or agentDir merge/override behavior. Related PRs #29450 and #29545 addressed the issue in different ways, but both are closed/unmerged in the provided GitHub context and their head SHAs are absent from this checkout.

## Best Possible Solution

Keep #29387 open as the canonical tracker. The best maintainer path is to choose and implement one explicit contract: either a focused runtime diagnostic like #29450 that warns when recognized bootstrap filenames exist under an explicitly configured `agentDir` but are not injected, or a deliberate agentDir merge/override design like #29545 with safeguards, callsite coverage, regression tests, and docs explaining precedence. The existing per-agent `workspace` option should remain the documented workaround until that runtime behavior is implemented.
