## Summary

Keep open. Current `main` still does not implement same-profile retry for transient `overloaded` errors: the run loop only rotates auth profiles, optionally waits before that rotation/fallback, then surfaces or falls back. Mainline docs and tests also describe and assert rotation/backoff behavior rather than retrying the same profile. The PR discussion shows maintained follow-up work and added coverage, so this is still useful work rather than cleanup fodder.

## Evidence

- **Prompt failover on main still rotates-or-surfaces, never same-profile retries:** In `src/agents/pi-embedded-runner/run.ts`, overloaded prompt failures call `advanceAuthProfile()` when the decision is `rotate_profile`; if rotation fails, the code recomputes the decision with `profileRotated: true`, which then falls through to `surface_error` or `fallback_model`. There is no branch or helper for retrying the same profile.
