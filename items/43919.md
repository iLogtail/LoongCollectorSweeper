## Summary

Keep open. Current `main` has partial env-proxy bootstrap for some fetch paths, but it does not implement this PR’s gateway-startup global dispatcher or its `ALL_PROXY`/SOCKS fallback work.

## Evidence

- **Current gateway startup has no global proxy bootstrap hook:** `startGatewayServer` reads config during startup, but there is no `applyGlobalProxyDispatcher()`-style call in the gateway startup path on current `main`. A repo search for `applyGlobalProxyDispatcher` returns no matches.
  - file: [src/gateway/server.impl.ts:342](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/gateway/server.impl.ts#L342)

- **Existing global dispatcher helper is narrower than the PR:** `ensureGlobalUndiciEnvProxyDispatcher()` only activates when `hasEnvHttpProxyConfigured("https")` is true, and then installs `new EnvHttpProxyAgent()` with no explicit `ALL_PROXY` bridging options.
  - file: [src/infra/net/undici-global-dispatcher.ts:91](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/infra/net/undici-global-dispatcher.ts#L91)
