## Summary

Keep #38721 open. Current main has bounded shutdown fixes for adjacent blockers, including websocket close, HTTP close, channel stop aborts, and bundled MCP runtime disposal, but I did not find evidence that gateway shutdown now cancels or detaches arbitrary active exec/process child handles before the shutdown watchdog. The provided maintainer discussion also explicitly says #61565 fixed a websocket close hang and did not supersede this child-process/pipe-handle report.

## Best Possible Solution

Keep #38721 as the canonical tracker for gateway shutdown blocked by active child-process/pipe handles. The best fix is a shutdown-phase policy for supervisor-managed exec/process sessions and any other gateway-owned child-process surfaces: stop accepting new process work, cancel or detach active runs with a bounded grace window, and add regression coverage proving active child handles cannot trip the shutdown watchdog. Existing websocket and MCP disposal fixes should remain as adjacent mitigations, not closure evidence for this report.
