## Summary

Keep open. #65260 is a recent, concrete OpenAI Codex OAuth latency report with measurements, and current main does not contain a source-backed fix or canonical duplicate that resolves the reported steady-state ~12s server-side latency. The current implementation still routes `openai-codex` through the Codex Responses/ChatGPT backend path, explicitly excludes it from the OpenAI WebSocket transport path, and only documents/configures related cache, service-tier, and transport knobs rather than establishing that the observed latency is expected or fixed.

## Best Possible Solution

Keep #65260 open for maintainer/provider investigation. The best next step is to add a focused Codex OAuth latency diagnostic or live probe that records transport, endpoint, prompt-cache fields, first-token/completion timing, service_tier, and auth-refresh state for `openai-codex-responses`; then either fix the OpenAI plugin/PI transport path if OpenClaw is adding avoidable delay, or document the expected Codex OAuth backend latency and the recommended configuration/workaround if it is upstream behavior.
