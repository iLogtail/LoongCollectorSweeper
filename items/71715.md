## Summary

Keep #71715 open. Current main still lacks WhatsApp error-visibility parity and does not have a WhatsApp-specific outbound transform that would apply the canonical assistant-visible XML sanitizer across the reported delivery paths. The linked fix PR #71830 appears to be the right active path, but it is open/unmerged in the provided context, so closing the issue now would be premature.

## Best Possible Solution

Keep #71715 open and let #71830, or an equivalent maintainer-reviewed patch, be the canonical fix path: apply the canonical assistant-visible sanitizer at WhatsApp outbound projection points after coalescing/fallback merge, add a documented channels.whatsapp exposeErrorText/errorPolicy-style setting with account inheritance and tests, and decide separately whether configurable outboundSanitizers belongs in core or plugin configuration.
