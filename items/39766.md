## Summary

Keep open. Current main still requires `models.providers.<id>.models` during config validation, and the generated config schema still marks it required. The PR's central behavior is not implemented on main, while the linked bug #39589 remains open and the overlapping alternative PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep this PR or canonical issue #39589 open until main accepts baseUrl/credential-only `models.providers.<id>` entries. The likely fix is a small config-contract change that defaults omitted provider `models` to `[]`, keeps the parsed `ModelProviderConfig.models` contract as an array, regenerates the generated config schema, and adds focused regression coverage for schema parsing plus the OpenAI embedding provider `baseUrl` fallback with network preflight stubs.
