## Summary

Keep #39766 open. Current main still requires `models.providers.<id>.models` in both the Zod source schema and generated JSON schema, so the PR's central config-validation bug is not fixed on main. The related Ollama merge only injects Ollama-specific defaults and does not solve the generic OpenAI/custom-provider baseUrl case tied to #39589, which remains open in the provided GitHub context.

## Best Possible Solution

Keep this PR or canonical issue #39589 open until main accepts `models.providers.<id>` entries that omit `models`, preserves the parsed/runtime provider contract as an array such as `[]`, updates the generated config schema, and adds focused regression coverage for schema parsing plus OpenAI embedding `baseUrl` fallback behavior with network preflight stubs. The Ollama-specific #69370 path can remain as shipped, but it does not replace the generic schema fix requested here.
