## Summary

Keep PR #44144 open. Current main still contains the `skills/model-usage/SKILL.md` Linux CLI guidance TODO and does not include the PR's platform-specific `--input` clarification. The underlying script already supports file/stdin input, so the docs change is still relevant, though existing bot review comments identify wording and metadata-scope issues that should be addressed before merge.

## Best Possible Solution

Keep the PR open and ask for a narrow wording revision: document that macOS live reads use the `codexbar` binary installed via the skill metadata/Homebrew path, and that Linux/other platforms can run the Python summarizer directly with `--input` outside OpenClaw skill availability unless the frontmatter and install contract are intentionally broadened.
