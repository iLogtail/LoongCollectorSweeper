## Summary

Keep #51871 open. The issue discussion narrowed the bug from a generic dashboard mismatch to malformed persisted cron rows missing `schedule` or `payload`, and current main still has Control UI code paths that directly dereference `job.schedule.kind`, `job.payload.kind`, and `job.payload.message`. The gateway list path also returns stored jobs without filtering out rows that remain malformed after best-effort normalization, so the reported UI crash is still a plausible current-main bug rather than implemented cleanup.
