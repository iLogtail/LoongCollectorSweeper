## Summary

Keep open. Current main still hard-rejects `sessions_send` calls that include both `sessionKey` and `label`, which is the exact behavior this PR tries to change. The PR remains a focused bug fix with regression-test coverage in its diff, and the related context does not show a merged canonical replacement.

## Best Possible Solution

Keep this open until a maintainer lands this PR or another canonical fix that makes `sessions_send` prefer `sessionKey` when redundant `label`/`agentId` fields are also present, with regression coverage for the mixed-parameter case and preserved label-only and permission-check behavior.
