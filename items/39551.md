## Summary

Keep open. Current main still hard-rejects `sessions_send` calls that include both `sessionKey` and `label`, while PR #39551 is a narrow fix for that live behavior and includes focused regression coverage. The related context shows overlapping open work, but no merged or maintainer-selected canonical replacement.

## Best Possible Solution

Keep #39551 open until maintainers choose and land one canonical fix for the live `sessions_send` mixed-selector bug. The best implementation is the narrow executor-side precedence rule proposed here: normalize a non-empty `sessionKey` as authoritative, ignore redundant `label` and `agentId` for target lookup, preserve label-only behavior plus downstream visibility/A2A checks and the missing-target error, update the tool description, and keep regression coverage proving mixed inputs send to the `sessionKey` target without calling `sessions.resolve`.
