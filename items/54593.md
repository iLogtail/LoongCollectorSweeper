## Summary

Keep PR #54593 open. Current main still has the reported legacy-key depth bug: `getSubagentDepth("subagent:worker")` would return 0, while the surrounding session-key code already recognizes `subagent:<id>` as a legacy subagent key. The PR is a focused bug fix with directly relevant tests, not obsolete or superseded by current main.

## Best Possible Solution

Keep the PR open for maintainer review, or apply the same targeted fix on main: make `getSubagentDepth` count `subagent:` token boundaries after agent-scope parsing and add regression tests for `subagent:worker` and nested legacy `subagent:parent:subagent:child`.
