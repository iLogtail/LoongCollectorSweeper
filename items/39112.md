## Summary

Keep PR #39112 open. Current main still writes native Ollama usage as zero when `/api/chat` omits `prompt_eval_count` and `eval_count`, and the dashboard aggregation path does not add a later character-based fallback.

## Best Possible Solution

Keep this PR open or land an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, include image/tool payloads in prompt estimates, and add focused Ollama stream regression tests for missing counters and explicit zero counters.
