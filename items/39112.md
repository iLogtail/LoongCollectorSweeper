## Summary

Current main still lacks the PR's central native Ollama omitted-counter fallback. The Ollama stream path records absent `prompt_eval_count` / `eval_count` as zero, and downstream usage/reporting paths still rely on positive token totals for visibility, so this PR is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with an equivalent native Ollama implementation in `extensions/ollama/src/stream.ts`. The best fix should estimate tokens only when Ollama omits counters, preserve explicit zero counters, include image/tool payloads in prompt estimation, and add focused Ollama stream regression tests.
