## Summary

Keep open. Current main still does not implement the requested behavior with high confidence: custom `logging.redactPatterns` are not additive, calls that pass partial options bypass config-loaded patterns, and transcript persistence paths still write messages without built-in redaction. The linked PRs in the issue context are useful prior work, but they are open/unmerged and the observable problem remains on main.

## Evidence

- **Custom patterns still replace defaults and partial options bypass config:** `resolvePatterns` uses configured patterns instead of defaults when a non-empty value is supplied, and `resolveRedactOptions` only reads config when `options` is undefined. That matches the linked PRs' stated root cause rather than an implemented fix.
