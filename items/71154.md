## Summary

Keep open. Current main still hard-rejects POSIX backslash-newline shell continuations before allowlist analysis and still reports unparseable allowlist analysis failures as `allowlist miss`. The PR is a plausible focused fix for the linked #71153 behavior, and the related #71225 is closed/unmerged rather than a shipped replacement.

## Best Possible Solution

Keep #71154 open. The best path is to finish maintainer review and land a safe version that normalizes POSIX `\<newline>` continuations before allowlist analysis, preserves fail-closed behavior for genuinely unparseable syntax, emits a distinct `unsupported-shell-syntax` reason through gateway/node-host surfaces, and includes the regression tests/docs updates needed to close #71153 afterward.
