## Summary

Keep PR #71154 open. Current main still hard-rejects POSIX backslash-newline commands before allowlist analysis and still reports analysis failures as `allowlist miss`; the requested `joinShellLineContinuations` normalization and `unsupported-shell-syntax` denial reason are not present on main. The PR remains a focused fix for the user-visible problem from #71153, though it needs maintainer review of the security-sensitive durable-approval behavior flagged in discussion before merge.

## Best Possible Solution

Keep the PR open and ask for maintainer/security review. The best product path is to land an adapted version that performs quote/heredoc/comment-aware POSIX `\<newline>` normalization before allowlist analysis, emits a distinct `unsupported-shell-syntax` denial across gateway and node-host paths, updates tests and user-facing docs/changelog, and makes the durable-approval behavior for unparseable commands explicit and safe.
