## Summary

Keep open. Current main does not implement `channels.whatsapp.denyFrom`, and the PR is a plausible WhatsApp plugin feature, but it is not merge-ready as-is because the branch includes unrelated deployment/config files and has a review-identified command-authorization forwarding gap to resolve.

## Best Possible Solution

Keep the PR open for cleanup rather than auto-closing it. The best path is a focused WhatsApp plugin change only: remove `DEPLOY.md` and `config.baseline.json`, keep `denyFrom` in the WhatsApp config/schema/generated metadata/account resolution, use a dedicated strict `isSenderDenied` matcher with no self-chat or wildcard shortcut, forward it through `resolveDmGroupAccessWithCommandGate`, and add regression tests plus WhatsApp docs/changelog coverage.
