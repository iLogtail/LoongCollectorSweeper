## Summary

Keep open. Current main has substantial session transcript indexing, QMD export, post-compaction sync, and visibility controls, but the specific request is to graduate sessionMemory from experimental and solve multi-channel continuity by default. That is not implemented: builtin memory still defaults to `experimental.sessionMemory=false` and `sources=["memory"]`, docs still label session memory search experimental/opt-in, QMD session indexing is separately disabled by default, and session transcript hits are still filtered through session-tools visibility rather than being a broad default continuity layer. The KeepMyClaw comment is useful external context for crash/recovery continuity, but it does not replace the requested OpenClaw product/config/docs decision. Related PR #51341 was closed unmerged, #60742 is a narrower closed recovery-layer proposal, and the provided ClawSweeper report #53550 kept a related gateway-dispatched session-memory gap open.

## Best Possible Solution

Keep the issue open for maintainer/product decision. A good resolution would either implement a conservative graduation path in core/plugin config and docs, or explicitly decline/default-scope it: decide stable naming, set safe defaults for new installs if desired, align builtin and QMD session indexing behavior, document DM/group privacy and embedding-cost implications, and add tests proving the intended cross-channel continuity surface. If broader same-agent session recall remains intentionally gated by `tools.sessions.visibility`, document that constraint rather than presenting sessionMemory as a universal multi-channel continuity fix.
