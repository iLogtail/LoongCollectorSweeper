## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on vector-only LanceDB retrieval, while the existing BM25+vector hybrid implementation is in the separate built-in `memory-core`/`memory_search` path. The related PR #7636 was closed unmerged, and the existing maintainer/Codex issue comment already correctly kept this focused bundled-plugin parity request open.

## Best Possible Solution

Keep #7629 open as a focused bundled `memory-lancedb` parity enhancement. The best implementation belongs in `extensions/memory-lancedb`: add typed `hybrid` config plus manifest/docs entries, create or maintain an FTS index on `text`, pass the original query text through recall/forget/CLI/hooks, use verified LanceDB hybrid search or a verified fallback/reranker when enabled, preserve vector-only fallback, and add targeted tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query text propagation.
