## Summary

本 PR 提议使用 SIMD (AVX2) 优化多行日志解析性能，具有实际价值，但当前存在合并冲突、CLA 未签署、逆向扫描边界处理缺陷及全局编译标志兼容性问题，属于可挽救的陈旧 PR，需作者或维护者进一步修复与更新。

## Best Possible Solution

建议作者或维护者：1. 解决当前合并冲突并签署 CLA；2. 修复 LogFileReader.cpp 中 SIMD 逆向扫描的头部边界漏检问题，将标量回退逻辑与 SIMD 路径结合而非使用 #else 隔离，确保不足 32 字节的头部数据被正确处理；3. 移除 CMakeLists.txt 中的全局 -march=haswell 编译标志，改为使用函数级 target attribute 配合运行时 CPUID 检测进行动态分发，以保障跨平台与旧 CPU 兼容性。修复完成后可重新提请审查。
