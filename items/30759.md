## Summary

Current main still does not implement WebChat document upload/extraction. The WebChat composer accepts and filters only image MIME types, serializes accepted attachments as images, and the gateway chat.send attachment parser drops non-image payloads such as PDFs. Existing PDF/text file-input support is for the OpenResponses HTTP API, not the WebChat chat.send path, and it does not cover Word/doc/docx extraction requested here.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into a broader canonical WebChat document-upload tracker that explicitly includes PDF, TXT, DOC, and DOCX extraction. A complete fix belongs across the WebChat composer and gateway chat.send attachment path: broaden accept/filter/preview/serialization, add explicit MIME and size validation with user-visible errors, connect safe text/PDF/Word extraction or offload behavior, preserve useful metadata, and add UI/gateway tests for TXT, PDF, DOC, and DOCX behavior. Existing OpenResponses/document-extract infrastructure may be reusable, but it is not currently wired into WebChat uploads.
