## Summary

Keep open. Current main has WebChat session switching, labels, and a sessions page, but it does not implement the requested session-switching sidebar or a Control UI session-create flow backed by `sessions.create`. The related sidebar PR #52746 was closed unmerged, and current main still renders the chat session picker in the content header/mobile dropdown rather than the left sidebar. The visible plus button sends `/new`, which resets the current session key to a fresh session id, not a named additional session entry for sidebar switching.
