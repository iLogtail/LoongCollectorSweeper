## Summary

Current main partially supports WebChat session switching through a chat-header select and a separate Sessions page, but it does not implement the requested chat sidebar/session list, distinct WebChat-created isolated sessions, or WebChat `sessions_spawn({thread:true, mode:"session"})` support. The related sidebar PRs in the supplied GitHub context (#52746 and #61721) were closed unmerged, so the request is still useful and should remain open.

## Best Possible Solution

Keep the issue open. The best product path is a Control UI chat sidebar that reuses `sessions.list`/`sessions.subscribe`, `switchChatSession`, and `sessions.patch` labels, adds a create flow that produces distinct selectable WebChat sessions instead of only `/new` reset semantics, and shows lightweight metadata such as updated time, model, and tokens. Treat `sessions_spawn({thread:true, mode:"session"})` as a related but separate design decision unless WebChat gets a real or virtual thread-binding hook contract.

## Risks / Open Questions

- Closing this now would conflate existing header/select and Sessions-page support with the requested first-class in-chat sidebar workflow.
- A future implementation should avoid duplicating the existing header picker and Sessions page, and should clarify whether WebChat gets a virtual-thread/session API or only ordinary session switching.
