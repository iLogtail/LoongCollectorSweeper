## Summary

Keep open. Current main still does not implement the requested HTTP lightContext/voice mode, and the only stable-session behavior on main is `x-openclaw-session-key` or the OpenAI `user` field. Related PR #69060 is explicitly partial and does not address the full 15-20K-token context assembly problem.

## Best Possible Solution

Keep #68920 open as the canonical HTTP low-latency context issue. Land or replace the narrow stable-session work from #69060 separately, then implement a deliberate `/v1/chat/completions` light-context/voice mode in the gateway path that can opt out of skills, tool exposure, memory prompt/sync, and workspace bootstrap while preserving auth, owner-tool policy, and clear docs/tests.
