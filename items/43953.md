## Summary

Keep open. Current main still drops Feishu card-action `form_value`, `input_value`, `option`, `options`, and `name` data before the synthetic message reaches the agent. PR #43953 is relevant to open issue #43202, but the submitted patch needs to preserve those fields at the webhook parser boundary and add end-to-end regression coverage before it can be treated as implemented or obsolete.

## Best Possible Solution

Keep the PR open and revise it into a complete Feishu plugin fix: preserve the extra Feishu action fields in `parseFeishuCardActionEventPayload`, define the payload contract in `handleFeishuCardAction` or `card-interaction.ts`, add parser-to-handler tests for `form_value`, `input_value`, `option`, and `options`, and coordinate with #43202/#42908 so one canonical PR lands the behavior.
