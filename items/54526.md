## Summary

Keep open. Current main already contains the Control UI channel-card configured-state fallback, but it does not contain the PR’s Slack auth-shaped unhandled-rejection fix. The remaining Slack bug is plausible and belongs in the Slack-owned runtime path or a plugin-registered unhandled-rejection handler, with a focused regression test, rather than closing this PR as implemented.

## Best Possible Solution

Keep the PR open, but rework it before merge: drop the already-shipped Control UI configured-state files, implement the Slack auth floating-rejection handling in the Slack plugin via `registerUnhandledRejectionHandler` or another extension-owned runtime seam, reuse the Slack auth classifier/error graph traversal, and add a focused regression test showing `account_inactive`/`invalid_auth` no longer triggers process exit while unrelated unhandled rejections still do.
