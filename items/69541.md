## Summary

Keep #69541 open. Current main has point fixes for known injected metadata and memory tags, but it does not implement the requested declarative manifest field, aggregated registry/getter, or shared strip/sanitize/ingest contract. New plugin-injected XML or sentinel blocks still require hardcoded updates in each consumer.

## Best Possible Solution

Add a backwards-compatible manifest contract such as `contracts.injectedContextTags`, normalize and expose it through the manifest/registry loader in deterministic order, document it in plugin docs, and refactor display, replay/history, dreaming/session ingestion, and assistant-visible sanitizers to consume the registry while retaining existing hardcoded tags as compatibility defaults and regression tests.
