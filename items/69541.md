## Summary

Keep #69541 open. Current main has several point sanitizers for known injected blocks, but it does not implement the requested declarative `contracts.injectedContextTags` manifest/API registry or a shared downstream strip/sanitize/ingest contract. This is a real core/plugin-SDK contract gap, not a ClawHub-only plugin idea.

## Best Possible Solution

Keep the RFC open and implement an additive plugin manifest/SDK contract for injected context blocks. The likely path is to add typed `contracts.injectedContextTags`, normalize and preserve it in the manifest registry, expose a narrow getter/helper for downstream consumers, migrate bundled producers such as `active-memory` and `memory-lancedb`, and update display, session-history/dreaming, compaction, docs, and contract tests to consume the shared registry.
