## Summary

Current main has partial alignment for WhatsApp DM session routing, but not enough evidence to say issue #65902 is fully fixed. The code/docs now clearly treat WhatsApp DMs as the agent main session when `session.dmScope=main`, `sessions_list` can surface that mailbox as `channel: "whatsapp"`, and gateway delivery from `main` honors WhatsApp `lastChannel`/`lastTo`. However, the only explicit fallback for implicit `...:default:direct:...` keys is in `session_status`; `sessions_send` still resolves strictly from a stored session key or label and has no matching fallback. Since the report is specifically about discovery plus `sessions_send` targeting, the conservative call is to keep it open.
