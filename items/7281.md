## Summary

Current `main` does not implement configurable hook-delivery failover. Hook mappings still accept only one delivery route (`channel` + `to`), hook dispatch forwards only that single route into isolated cron delivery, and a delivery failure still bubbles back as an error that produces the `Hook <name> (error): ...` summary path described in the issue.

## Evidence
