## Summary

Keep open. Current main still strips a trailing NO_REPLY from auto-reply text and delivers the remaining preamble, with tests explicitly asserting that behavior. The linked #66755/#66701 context tracks the same symptom, but #66755 is a narrower alternative rather than a clear superseder of this PR's broader raw-preamble case, and #69394 also has a concrete review concern that should be fixed before merge.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is a maintainer decision on trailing NO_REPLY semantics for text-only auto-replies, then a targeted normalizeReplyPayload fix using a robust trailing-token detector such as the cron path's trimmed-before comparison or a structured strip result. The test update should cover raw reasoning preambles, think-block preambles, substantive text, emoji/media/channelData preservation, and non-trailing NO_REPLY with trailing whitespace, coordinated with #66701 and #66755.
