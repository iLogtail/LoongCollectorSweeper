## Summary

Keep #70895 open. Current main still has the regression the PR targets: webhook agent requests are dispatched as isolated cron jobs, but the isolated runner always passes `forceNew: true`, so `resolveCronSession` cannot reuse a stable hook `sessionKey`. The PR is narrow, directly tied to #70894, and the current tree does not yet contain its regression test or behavior change.

## Best Possible Solution

Keep #70895 open as the focused fix for #70894. The best path is to merge or rebase the narrow `forceNewSession` change so hook/webhook-sourced isolated runs can pass `forceNew: false`, keep ordinary isolated cron runs fresh-per-run, and land the regression test that asserts stable webhook `sessionKey` reuse.
