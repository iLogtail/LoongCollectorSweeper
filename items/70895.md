## Summary

Keep #70895 open. Current main still routes /hooks/agent into an isolated cron run and still passes forceNew: true for every isolated run, so accepted stable hook sessionKey values cannot reuse an existing session. The PR is a narrow, relevant fix for #70894 with a targeted regression test, and it is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and review or adapt the narrow fix: compute whether an isolated run is hook/webhook-sourced before resolveCronSession, pass forceNew: false for those stable hook sessions, keep normal isolated cron runs forceNew: true, and land the regression test linked to #70894 and the broader #11665 discussion.
