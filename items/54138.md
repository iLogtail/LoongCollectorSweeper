## Summary

Keep open. Current main still sends heartbeat replies through code paths that only strip HEARTBEAT_OK/prefix text, not bracket-format [TOOL_CALL]/[TOOL_RESULT] blocks. The exact proposed fix is still only an open PR (#54165) in the provided GitHub context, and related #41329 remains open for broader heartbeat transcript leakage.

## Best Possible Solution

Keep #54138 open until #54165 or an equivalent scoped fix lands. The best implementation is to strip bracket-format [TOOL_CALL]/[TOOL_RESULT] blocks at the heartbeat delivery boundary or in the shared assistant-visible sanitizer, with regression coverage in the reply payload or heartbeat runner tests. Track the broader interactive tool-call stream-transition symptoms separately if they remain reproducible.
