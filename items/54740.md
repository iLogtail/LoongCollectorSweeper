## Summary

Keep open. Current `main` still uses heuristic extra-service detection for doctor and still prints generic cleanup commands for the current gateway label whenever any extra services are found. The reported `gateway.mode` mismatch is also not clearly obsoleted, because doctor seeds from `sourceConfig` while `config get` reads `runtimeConfig`.

## Evidence

- **Launchd extra-service scan is still marker-based, not binary/argv-based:** `scanLaunchdDir()` flags any plist whose contents contain an `openclaw` marker unless it matches a narrow gateway heuristic; it does not inspect the actual executable or command line for a gateway binary.
  - file: [src/daemon/inspect.ts:209](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/daemon/inspect.ts#L209)
  - command: `nl -ba src/daemon/inspect.ts | sed -n '196,242p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Gateway exemption remains a loose heuristic:** `isOpenClawGatewayLaunchdService()` treats a plist as the real gateway if it has embedded service markers, or otherwise if the contents merely contain `gateway` and the label starts with `ai.openclaw.`. That is still broader than verifying an actual gateway service definition.
  - file: [src/daemon/inspect.ts:98](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/daemon/inspect.ts#L98)
  - command: `nl -ba src/daemon/inspect.ts | sed -n '82,117p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Cleanup hints still target the current gateway label unconditionally:** When any extra services are found, `maybeScanExtraGatewayServices()` always calls `renderGatewayServiceCleanupHints()` and prints those commands. The rendered macOS hints are `launchctl bootout gui/$UID/<current-label>` and `rm ~/Library/LaunchAgents/<current-label>.plist`, regardless of which extra service triggered the warning or whether the current gateway is the active one.
  - file: [src/commands/doctor-gateway-services.ts:450](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/commands/doctor-gateway-services.ts#L450)
  - command: `nl -ba src/commands/doctor-gateway-services.ts | sed -n '398,454p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Doctor and `config get` still read different config views:** Doctor preflight initializes from `snapshot.sourceConfig`, while `config get` reads `snapshot.config`; `ConfigFileSnapshot` defines `sourceConfig` separately from `config`/`runtimeConfig`. That means a doctor/config disagreement is still structurally possible on current main.
  - file: [src/commands/doctor-config-preflight.ts:113](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/commands/doctor-config-preflight.ts#L113)
