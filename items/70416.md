## Summary

Keep open, but narrow it. Current main no longer appears to have the missing `claude-opus-4-7` catalog-entry problem: the pinned `@mariozechner/pi-ai@0.70.2` catalog includes Opus 4.7 at the same $5/$25/$0.50/$6.25 rates as Opus 4.6. The remaining actionable bug is that OpenClaw still uses a flat `cacheWrite` rate through `calculateCost`, so `cacheRetention: "long"` cache writes are not priced at the official 1-hour rate.

## Best Possible Solution

Keep the current Opus 4.7 catalog entry, then add retention-aware Anthropic cache-write cost handling in OpenClaw or extend the model cost contract to represent 5-minute and 1-hour cache creation separately. Add a focused regression test for `claude-opus-4-7` with `cacheRetention: "long"` showing `cache_creation_input_tokens` uses $10/MTok, while short/default cache writes continue using $6.25/MTok. A warning for unknown/zero-priced paid models would be a useful follow-up but should not block the retention fix.
