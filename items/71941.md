## Summary

Keep PR #71941 open. Current main still pre-fills the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir`, and that resolver still nests a non-default agent under `agents.defaults.workspace` when that default is set by setup. The PR addresses a real current mismatch with the documented peer-level workspace convention and is not implemented or superseded on main, though the PR needs the existing review comments handled before merge.

## Best Possible Solution

Keep the PR open and ask for the review fixes: stub `OPENCLAW_HOME` or `HOME` in the new `src/commands/agents.add.test.ts` wizard test, correct the changelog attribution, then land this PR or an equivalent narrow implementation that changes only the brand-new-agent wizard prefill while preserving `resolveAgentWorkspaceDir` runtime semantics and the #59789 shared-base behavior.
