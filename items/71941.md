## Summary

Keep PR #71941 open. Current main still seeds the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir`, which nests a non-default agent under `agents.defaults.workspace`; docs still describe peer-level `~/.openclaw/workspace-<agentId>` defaults. The PR addresses a real core CLI/docs mismatch, but its current review context includes fixable test/changelog comments, so it should not be closed as obsolete.

## Best Possible Solution

Keep #71941 open and ask for a small follow-up on the PR: keep the narrow wizard-only peer-level suggestion helper, preserve `resolveAgentWorkspaceDir` runtime behavior for #59789/custom shared roots, stub `OPENCLAW_HOME` or `HOME` in the new wizard test, and fix the changelog attribution before running the normal changed gate.
