## Summary

Keep #69051 open. Current main has the broader TTS persona foundation, but it still lacks this PR-A's source-text metadata, speech-provider capability contract, and per-provider fallback text routing. The PR is active, recently recentered to a narrow 7-file diff, has dependent follow-up PRs, and still has review discussion that needs maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep #69051 open as the canonical narrow PR-A proposal for provider-aware TTS source-text routing. The best path is either to land this after review fixes, or to supersede it with an equivalent maintainer-owned implementation that adds the metadata contract, SDK/public provider capability types, ElevenLabs v3 gating, per-attempt fallback text resolution after provider/persona config resolution, and focused tests for preserve/strip fallback behavior, model allowlists, empty plain variants, and public runtime reason-code typing.
