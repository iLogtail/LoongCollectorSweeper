## Summary

Keep PR #71713 open. Current main still lacks the Telegram gateway-watch inbound summary that the PR adds, while WhatsApp already has the comparable no-content inbound log and current Telegram tests do not cover that observability path.

## Best Possible Solution

Keep PR #71713 open for maintainer review. The best path is a narrow Telegram plugin change that emits exactly one no-content inbound summary after context acceptance and before dispatch, with focused unit coverage for the dispatch and no-context paths.
