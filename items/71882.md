## Summary

Keep #71882 open. Current main still has the underlying third-party memory-plugin status false negative, so the work is not already implemented and does not cleanly belong only in ClawHub. The PR is not mergeable as-is, though: it hard-codes `memory-lancedb-pro` into shared core status logic and its console filter does not cover raw plugin `console.log` output.

## Best Possible Solution

Keep the item open but request a generic fix. The status resolver should consult a plugin-owned memory runtime/status capability, or carry a generic capability bit from plugin/runtime metadata, without naming `memory-lancedb-pro` in core. The JSON path should suppress or reroute raw plugin console output during status probing if that remains part of the scope. Use #56968 as the canonical issue for the broader third-party memory status false-negative behavior.

## Risks / Open Questions

- Merging this PR as-is would put a third-party plugin id into shared core status logic and leave forks or other runtime-backed memory plugins broken.
- The JSON-output fix is incomplete for raw plugin `console.log` or `console.info` calls.
- Closing outright would drop a real unresolved product gap already reproduced for multiple third-party memory plugins and tracked by open issue #56968.
