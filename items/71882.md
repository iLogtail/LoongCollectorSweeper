## Summary

Keep #71882 open. Current main still has the runtime-only memory status false-negative path, so the PR is addressing a real core/plugin-boundary bug. The submitted patch should not be closed as ClawHub-only or obsolete, but it also should not merge as-is because it hard-codes one third-party plugin id and does not fully protect JSON status output from raw plugin console writes.

## Best Possible Solution

Keep the PR open and request changes. The best fix is a generic status path that can ask an active memory runtime for status without requiring built-in sqlite/qmd config or store files, preferably via runtime/capability metadata rather than a hard-coded slot id. The JSON path should also suppress or redirect raw plugin console output during the status probe, with regression tests covering both runtime-only memory status and raw console noise under `openclaw status --json --all`.

## Risks / Open Questions

- Merging the PR as submitted would add a core special-case for `memory-lancedb-pro` instead of a plugin-agnostic runtime/capability contract.
- The JSON-output cleanup in the PR would still allow raw plugin `console.log` or `console.info` writes to reach stdout in relevant command paths.
- Closing the PR now would discard a focused reproduction and regression-test direction for an unresolved status false-negative affecting runtime-backed memory plugins.
