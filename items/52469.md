## Summary

Keep open. Current `main` does not contain this PR’s token-threshold override, and the existing safeguard still intentionally returns a minimal compaction boundary when `messagesToSummarize` and `turnPrefixMessages` are both empty. I also do not have high-confidence evidence that the PR is safely obsolete: current code and tests preserve the `#41981` boundary-writing behavior, while the PR discussion leaves an unresolved regression path where the override can still fall through to `cancel: true` on model/auth/summarization failure.
