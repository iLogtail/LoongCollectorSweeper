## Summary

Keep #55901 open. Current main still flattens IRC newlines and only sends/receives plain PRIVMSG; there is no draft/multiline CAP negotiation, IRCv3 tag parsing, or BATCH reconstruction. The PR targets the existing bundled IRC plugin and is not implemented or superseded on main, so it should continue through normal maintainer review rather than cleanup close.

## Best Possible Solution

Keep the PR open. The best path is to finish review in `extensions/irc`: CAP/BATCH negotiation, IRCv3 tag parsing, inbound/outbound multiline handling, conservative IRC-safe line splitting, focused tests, and docs updates if user-visible behavior or config changes.
