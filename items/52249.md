## Summary

Keep #52249 open. The original direct-resume/root-cause text is stale, but the newer heartbeat-disabled reproduction still matches current main: ACP parent-stream completion enqueues a parent-session system event and requests a heartbeat wake, while the wake layer still depends on a registered/enabled heartbeat runner before anything drains that event. The direct fix PR #60402 is still open in the provided context and is not present in this checkout.

## Best Possible Solution

Keep the issue open until main includes a regression for ACP `streamTo: "parent"` child completion with heartbeat disabled and an implementation that wakes/drains the targeted parent session without depending on optional periodic heartbeat state. The likely fix belongs in the heartbeat wake/runner targeted-wake path or a dedicated ACP completion delivery primitive, with #60402 as the current candidate implementation to review.
