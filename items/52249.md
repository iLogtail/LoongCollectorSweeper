## Summary

Keep #52249 open. Current main still routes ACP stream-to-parent completion through queued system events plus `requestHeartbeatNow`, and the wake runner can still skip delivery when heartbeat infrastructure is disabled or has no configured agents. The linked open PR #60402 appears to target this exact gap, but it is not merged into current main, so the bug is not implemented-on-main or safe to close.

## Evidence

- **ACP relay remains heartbeat-coupled:** `startAcpSpawnParentStreamRelay` enqueues the parent system event and then calls `requestHeartbeatNow(scopedHeartbeatWakeOptions(..., { reason: "acp:spawn:stream" }))`; there is no independent yield-resume or direct parent delivery path here.
  - file: [src/agents/acp-spawn-parent-stream.ts:184](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/agents/acp-spawn-parent-stream.ts#L184)

  - sha: [8e12c24d1721](https://github.com/openclaw/openclaw/commit/8e12c24d17218dc50023aaf5543df1b7c36520aa)
- **Wake request is dropped without a handler:** The heartbeat wake timer returns immediately when no wake handler is registered, leaving pending work undelivered until another handler appears or inbound activity drains it by another path.
  - file: [src/infra/heartbeat-wake.ts:156](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/src/infra/heartbeat-wake.ts#L156)
