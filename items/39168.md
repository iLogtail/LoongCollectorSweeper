## Summary

Keep PR #39168 open. The reported `mcp_exec`/`mcp_read`/`mcp_write` dispatch problem is still plausible on current main, but the PR cannot merge as-is because it patches obsolete `@mariozechner/pi-agent-core@0.55.3` while current main uses 0.70.2 and routes this class of repair through OpenClaw's stream tool-name normalization wrapper.

## Best Possible Solution

Keep the PR open but ask for rework against current main: implement guarded `mcp_` prefix normalization in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, only when the stripped name resolves to an allowed registered tool, preserve existing `mcp__bundle__*` tool names, and add focused regression tests for `mcp_exec`, `mcp_read`, `mcp_write`, a nonexistent `mcp_foo`, and the bare `mcp_` edge case in the existing embedded-runner normalization tests. Do not carry a stale pnpm patch for pi-agent-core 0.55.3.
