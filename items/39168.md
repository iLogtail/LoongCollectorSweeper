## Summary

Keep #39168 open. Current main still does not show an implemented `mcp_` prefix normalization path for embedded runner tool dispatch, and the PR's dependency patch targets an obsolete `@mariozechner/pi-agent-core` version rather than the locked runtime on main.

## Best Possible Solution

Keep the PR or its underlying bug open, but rework it against current main. The best fix is likely to add a guarded `mcp_` prefix candidate to `normalizeToolCallNameForDispatch` or an equivalent current dispatch seam, then add focused regression tests for `mcp_exec`, `mcp_read`, `mcp_write`, bare `mcp_`, and ordinary `exec`/`read`/`write` behavior. Avoid reviving the obsolete 0.55.3 pnpm patch unless maintainers explicitly want to patch upstream dist for the current 0.70.2 package.
