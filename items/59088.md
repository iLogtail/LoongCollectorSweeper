## Summary

Current main has a partial Control UI agent file editor, but it is intentionally scoped to core/bootstrap workspace files and MEMORY.md. The issue asks for a broader workspace file browser/editor with a file tree and support for memory/*.md plus .json/.csv/.txt files, so meaningful requested capability remains missing.

## Evidence

- **Control UI exposes a Core Files editor, not a workspace file browser:** The Agents Files panel is titled "Core Files" and renders tabs plus a textarea/save flow for the files returned by agents.files.list.
