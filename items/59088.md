## Summary

Current main has a partial Control UI agent file editor, but it is still scoped to fixed core/bootstrap files plus MEMORY.md. #59088 asks for a broader workspace browser/editor with a file tree, nested memory/*.md, and .json/.csv/.txt support, so meaningful requested functionality remains missing. The related #63224 context reinforces #59088 as the canonical remaining workspace-browser issue.

## Best Possible Solution

Keep #59088 open as the canonical Control UI workspace browser/editor feature. The best implementation path is a root-bound gateway API for safe workspace tree listing plus explicit get/set/download/preview operations, with authorization scopes, symlink/hardlink escape protection, file-size/type limits, and UI support for memory/*.md plus safe text formats such as .md, .json, .csv, and .txt. It should also carry forward #63224's media preview/download requirements rather than treating the existing Core Files editor as complete.
