## Summary

Keep PR #72055 open. Current main still lacks any Bonjour plugin config path for overriding the mDNS service instance name, while the PR is a focused implementation of the linked #54467 request with manifest schema, plugin-entry wiring, docs, changelog, and regression tests.

## Best Possible Solution

Keep this PR open as the concrete implementation path for #54467. The best product direction is to keep the setting plugin-owned as `plugins.entries.bonjour.config.instanceName`, validate it through the Bonjour manifest schema, read it via `api.pluginConfig` in the Bonjour plugin entry, and ship the accompanying tests/docs/changelog after maintainer review.
