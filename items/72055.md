## Summary

Keep PR #72055 open. Current main still lacks the requested Bonjour per-instance mDNS service-name config: the bundled plugin always derives the advertised name from `ctx.machineDisplayName`, the manifest rejects unknown config properties, and docs do not mention `plugins.entries.bonjour.config.instanceName`. The linked issue #54467 was closed as superseded by this PR, so this PR remains the active implementation path rather than cleanup material.

## Best Possible Solution

Keep #72055 open as the active implementation path for #54467. Rebase it onto current main, preserve the current Bonjour crash-suppression handler wiring, keep the config under `plugins.entries.bonjour.config.instanceName`, and require the Bonjour extension tests plus the changed gate before merge.
