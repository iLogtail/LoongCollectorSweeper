## Summary

Keep open. Current main has generic auth-profile rotation and some provider API-key rotation helpers, but the Z.AI gateway auth path still only exposes single-key env auth (`ZAI_API_KEY` / `Z_AI_API_KEY`) and does not synthesize runtime-only API-key auth profiles from `ZAI_API_KEYS`, numbered env keys, or `OPENCLAW_LIVE_ZAI_KEY`. The PR discussion and bot review comments describe real edge cases in that missing seam, so this is not safe to close as implemented or superseded.

## Best Possible Solution

Keep the PR open, but ask for or prepare a focused rebase that implements the missing Z.AI env-backed runtime auth-profile rotation with narrow shared-contract changes: add API-key-capable runtime external auth overlays, include runtime-only profiles in auth-profile order without breaking cooldown/stale-profile repair, register/scrub Z.AI rotation env vars, preserve SecretRef/non-interactive behavior, update Z.AI/provider docs, and keep the targeted Z.AI/auth-store regression tests.
