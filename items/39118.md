## Summary

Keep open. Current main still does not implement the PR's core safety behavior: dependency-only companion systemd units can still be reported as gateway-like, extra services are reported without active-state filtering, and gateway service repair can still rewrite ExecStart without checking whether the gateway is currently running. The PR also has unresolved review comments that make it not merge-ready rather than obsolete.

## Best Possible Solution

Keep this PR open, but require a revised implementation. The best path is to filter dependency-only companion service references without suppressing real custom gateway units, make any active-service filtering scope-aware for user versus system services, and apply the running-gateway ExecStart guard after all ExecStart-level audit issues, including entrypoint mismatch, have been computed. Add focused tests for companion dependencies, custom-named gateways, system-scope deep scans, inactive extras, and active-running custom launchers before merge.
