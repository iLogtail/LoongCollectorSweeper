## Summary

Keep open. Current main still lacks the PR's runtime protections and only partially narrows extra-service detection; companion services with `After=openclaw-gateway.service` still match the gateway-line marker path, and doctor still repairs entrypoint mismatches without a running-service exception. The PR also has substantive unresolved review comments that need revision rather than cleanup closure.

## Best Possible Solution

Keep #39118 open for maintainer/author revision. The right fix should preserve custom-named and system-scope extra-gateway detection, avoid dependency-only companion false positives, and apply any running-service ExecStart protection after all command/entrypoint rewrite issues are known, with focused tests for companion units, custom gateway units, inactive extras, and mismatch-only running services.
