## Summary

Keep open. Current `main` still uses a hardcoded 5 MB default for the media store, the media server, and sandbox staging, and there is still no top-level `media.maxBytes` config key in the runtime config types/schema/docs. Some callers can pass an explicit `maxBytes` override, but the requested user-configurable global setting is not implemented on `main`.
