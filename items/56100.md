## Summary

Keep open. Current main still treats `sessions_spawn` one-shot subagents as isolated helper sessions, not memory-equal conversational sessions. Active Memory explicitly skips non-user/internal runs, subagent bootstrap filtering excludes `MEMORY.md`, and persistent native subagent sessions still require `thread: true`. The report overstates data loss because subagent transcripts are retained/archived and can be indexed via opt-in session memory search, but the requested automatic memory initialization for ephemeral A2A runs is not implemented.

## Risks / Open Questions

- The issue body mixes three different behaviors: transcript persistence, automatic memory recall (Active Memory), and durable writes to `MEMORY.md`; a maintainer follow-up should narrow which one is actually requested.
- There may be a plugin-based path using subagent lifecycle hooks, but current main does not provide the requested behavior out of the box, so closing as already implemented or cannot reproduce would be premature.
