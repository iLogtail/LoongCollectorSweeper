## Summary

Keep open. Current main still does not implement the requested WhatsApp human-takeover pause. The WhatsApp config schema has self-chat, access-control, debounce, reply, and receipt settings, but no humanTakeover/pause-on-outgoing option. Current inbound access control explicitly drops outbound direct messages marked fromMe instead of recording a per-conversation pause, and tests assert those outbound DMs do not reach the auto-reply handler.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin, not as a close. Add a typed root/account WhatsApp config for human takeover, detect non-gateway outgoing fromMe messages by conversation before they are silently dropped, record a per-account/per-peer pause TTL, and make the auto-reply path skip dispatch while the pause is active. Cover direct chats, selfChatMode, multi-account behavior, gateway-sent echo exclusion, and docs for the new config.
