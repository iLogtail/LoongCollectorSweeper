## Summary

Keep PR #69328 open. Current main does not implement the PR's ACP zero-diff verifier or the active-run `session.message` append path: ACP terminal handling still derives blocked/succeeded only from progress text, and the UI still defers active-run `session.message` reloads instead of appending the event payload. The PR remains useful, but should be revised before merge to address the review concern around untracked files and unborn `HEAD` repositories.

## Best Possible Solution

Keep the PR or a replacement patch open. The best path is to implement a robust ACP terminal verification gate that distinguishes persistent sessions from one-shot runs, handles tracked modifications plus new/unborn worktree cases intentionally, and adds focused regression coverage. The UI half should append active-run `session.message` payloads with identity-based dedupe/optimistic echo replacement while preserving the existing deferred reload fallback.
