## Summary

Keep PR #69328 open. Current main still lacks both central behaviors: the web UI defers `session.message` during an active run instead of appending the payload, and ACP task terminal handling has no worktree-diff verifier or persistent/oneshot zero-diff policy. The PR is not safely closeable as implemented-on-main or superseded, and its own review thread also identifies a real merge-blocking risk in the proposed `git diff --stat HEAD` verifier.

## Best Possible Solution

Keep the PR open, but require an updated patch against current main. The UI side should append active-run `session.message` payloads using the server-provided `messageId`/`messageSeq` metadata with robust dedupe and optimistic echo replacement tests. The ACP side should only add a zero-diff verification gate if it detects tracked, untracked, and unborn-HEAD changes correctly, preserves persistent-session exploratory turns, and has focused regression coverage for persistent, oneshot-with-progress, oneshot-without-progress, untracked-file, and unborn-repo cases.
