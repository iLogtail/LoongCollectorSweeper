## Summary

Current main still leaves the reported TUI/main chat path unsanitized. The shared GLM/DeepSeek special-token sanitizer exists and has regression coverage, but `resolveFinalAssistantText` still returns `finalText` and `streamedText` unchanged, and the TUI stream assembler builds those values through formatter paths that only call `sanitizeRenderableText`. The related PR #46182 was closed unmerged in the provided GitHub context, so it does not resolve #44179.

## Best Possible Solution

Keep #44179 open and wire the existing shared assistant-visible sanitizer into the TUI assistant extraction/finalization path, with focused regression tests for `resolveFinalAssistantText`, `extractContentFromMessage`, and/or `TuiStreamAssembler` showing `<|assistant|>` and full-width `<пЅњ...пЅњ>` tokens are stripped from final and streamed main chat output while examples inside code spans/blocks remain intact.
