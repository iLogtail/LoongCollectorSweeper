## Summary

Current `main` does not implement this PR’s behavior, and the repo now consistently documents and tests the opposite policy: internal hook discovery is skipped until hooks are explicitly configured. That means there is not strong evidence for any allowed close reason, so the conservative cleanup decision is to keep the PR open.

## Evidence
