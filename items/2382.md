## Summary

本 PR 为 metric input 与 aggregator 增加了 panic 自动重启机制，具有采集链路容错价值。但当前实现通过 cc.Run 递归调用 p.Run 且无重试上限与退避机制，存在协程泄漏与资源耗尽风险；同时基线版本过旧已产生合并冲突。属于需重构的有价值陈旧 PR，故保持开启等待作者迭代。

## Best Possible Solution

建议作者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用重构为带最大重试次数与指数退避的调度重试机制（或交由上层控制循环处理），防止插件持续 panic 导致资源耗尽。补充边界测试与重试配置说明后，请求维护者复审。
