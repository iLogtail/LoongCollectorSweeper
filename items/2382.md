## Summary

本 PR 为 timerRunner 引入了 panic 自动重启机制（rerunIfPanic），针对 metric input 与 aggregator 启用该特性以提升管道容错能力。代码具有明确价值，但当前基线过时导致无法合并（mergeable: false），且 panic 回调中采用递归调用 cc.Run 重启任务，缺乏重试上限与退避机制，持续 panic 可能引发协程泄漏。需作者 rebase 并优化重试策略后由维护者审查。

## Best Possible Solution

建议作者先将分支 rebase 至最新 main 解决合并冲突。维护者审查时需重点评估 panicCallback 中递归调用的安全性，建议改为循环重试或增加最大重启次数/指数退避机制，防止资源耗尽。待架构调整完成且 CI 验证通过后合并。
