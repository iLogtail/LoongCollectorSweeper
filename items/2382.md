## Summary

本 PR 为 metric input 与 aggregator 引入了 panic 自动重启机制，具有明确的采集链路容错价值。但当前分支基线过旧存在合并冲突，且 panicCallback 采用无限制的递归重启策略，持续 panic 可能导致协程泄漏或栈溢出。需作者 rebase 并重构重试逻辑后方可进入合并流程。

## Best Possible Solution

建议作者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用改为带最大重试次数与指数退避的循环重试机制，防止资源耗尽。补充边界测试后请求维护者复审。
