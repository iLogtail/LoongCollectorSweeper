## Summary

本 PR 旨在为 metric input 与 aggregator 增加 panic 自动重启机制以提升稳定性，但当前实现采用无上限递归重试且基线陈旧存在合并冲突。属于需重构安全重试逻辑并 rebase 后复审的有价值陈旧 PR，建议保持开启。

## Best Possible Solution

建议作者或维护者先将分支 rebase 至最新 main 解决冲突。将 panic 重启逻辑重构为带最大重试次数限制与指数退避的安全调度机制，或交由上层控制循环统一接管 panic 恢复与生命周期管理。补充边界测试与重试配置说明后，请求维护者复审。

## Evidence

- **architectural_flaw_unbounded_retry:** panicCallback 中直接通过 cc.Run 递归调用 p.Run，无最大重试次数或退避机制。若插件持续 panic，将无限创建新 goroutine 导致资源耗尽。
