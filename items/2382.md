## Summary

本 PR 为 metric input 与 aggregator 增加了 panic 自动重启机制，具有明确的采集链路容错价值。但当前分支基线过旧存在合并冲突，且 panic 回调采用无限制递归重启策略，存在协程泄漏与资源耗尽风险。属于有价值但需重构的陈旧 PR，建议保持开启等待作者更新。

## Best Possible Solution

建议作者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用改为带最大重试次数与指数退避的循环或调度重试机制，防止插件持续 panic 导致资源耗尽。补充边界测试后请求维护者复审。

## Evidence
