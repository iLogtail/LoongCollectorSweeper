## Summary

本 PR 提议为 metric input 与 aggregator 增加 panic 自动重启机制，方向具有价值，但当前实现存在无上限递归调用的架构缺陷，且基线版本过旧导致合并冲突。需重构为带退避与最大重试次数的安全调度机制，并 rebase 至最新 main。

## Best Possible Solution

建议作者或维护者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用重构为带最大重试次数与指数退避的调度重试机制（或交由上层控制循环处理），防止插件持续 panic 导致资源耗尽。补充边界测试与重试配置说明后，请求维护者复审。
