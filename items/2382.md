## Summary

本 PR 提出了为 metric input 与 aggregator 增加 panic 自动重启机制的合理需求，但当前实现存在无界递归派生 goroutine 的架构缺陷，且基线陈旧存在合并冲突，需重构安全重试逻辑并 rebase 后复审。

## Best Possible Solution

建议作者或维护者先将分支 rebase 至最新 main 解决冲突。将 panic 重启逻辑重构为带最大重试次数限制与指数退避的安全调度机制（例如在外部控制循环或独立 goroutine 中接管生命周期与重试，而非在 defer recover 回调中直接递归调用 cc.Run），补充边界测试与重试配置说明后，请求维护者复审。
