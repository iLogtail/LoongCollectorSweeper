## Summary

本 PR 旨在为 metric input 与 aggregator 增加 panic 自动重启机制，具有明确的运维价值。但当前实现存在架构缺陷（panic 回调中直接递归调用 p.Run，无退避与重试上限，持续 panic 将导致 goroutine 泄漏与资源耗尽），且基线版本过旧导致合并冲突。属于可能仍有价值的陈旧 PR，需重构安全重试逻辑并 rebase 后复审。

## Best Possible Solution

建议作者或维护者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用重构为带最大重试次数与指数退避的安全调度机制（或交由上层控制循环统一处理 panic 与重启），防止插件持续 panic 导致资源耗尽。补充边界测试与重试配置说明后，请求维护者复审。
