## Summary

本 PR 为 timerRunner 引入了 rerunIfPanic 容错机制，使 metric input 与 aggregator 在发生 panic 时可通过回调自动重启，提升采集链路稳定性。包含核心逻辑改造、回调式 panicRecover 扩展及对应单元测试，属于有效的核心增强贡献。

## Best Possible Solution

保持开启并进入常规代码审查。建议维护者重点评估：1) panic 回调中通过 cc.Run 递归调用 Run 是否会导致 goroutine 累积或控制流异常；2) 重启时的退避策略（当前直接复用 initialMaxDelay）是否符合生产预期；3) 类型断言放宽对 AggregatorV2 的影响。审查通过且 CI 验证无误后可合并入主线。
