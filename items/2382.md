## Summary

本 PR 旨在为 metric input 与 aggregator 增加 panic 自动重启机制，具有明确的采集链路容错价值。但当前实现存在无限制递归重启风险，且基线过旧导致合并冲突，需作者重构并 rebase 后方可进入合入流程。

## Best Possible Solution

建议作者先将分支 rebase 至最新 main 解决冲突。将 panicCallback 中的递归调用改为带最大重试次数与指数退避的循环或调度重试机制，防止插件持续 panic 导致资源耗尽。补充边界测试后请求维护者复审。
