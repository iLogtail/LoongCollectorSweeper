## Summary

Keep #51903 open. Current main still has the reported mismatch: the `nodes` tool schema keeps `node` optional, but the `describe` action still reads it with `{ required: true }`, so `{ action: "describe" }` is rejected before the default-node resolver can run. The related fix PR #51926 is still open, and PR #51919 was closed unmerged, so there is no shipped mainline fix to close against.

## Best Possible Solution

Keep the issue open until the `describe` branch reads `node` as optional and routes omitted-node calls through an intentional default/discovery path, ideally with a focused regression test in `src/agents/tools/nodes-tool.test.ts` proving `{ action: "describe" }` no longer throws `node required` while targeted actions still require an explicit node.
