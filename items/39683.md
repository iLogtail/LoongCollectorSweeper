## Summary

Keep open. Current main still lacks create/update-time Feishu/Lark missing-target rejection, and this PR is still a plausible focused fix thread, but the submitted blanket `delivery.to` requirement is not the best fix against current target-resolution behavior.

## Best Possible Solution

Keep the PR open, but request a rework rather than merging the current diff. The best fix should validate unresolved announce targets through the current generic delivery-resolution seam, allow explicit `delivery.to` and deterministic session/default/plugin resolution, avoid blocking unrelated update operations on legacy jobs, keep provider-specific target hints in the Feishu/Lark plugin surface, and add focused regression coverage for create-time failure plus valid resolved-target acceptance.
