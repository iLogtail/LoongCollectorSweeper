## Summary

Keep open. Current main still has the reported timeout/failover shape: the embedded run timer marks a generic timeout, only compaction has a special exemption, and assistant failover still rotates/falls back on any timedOut state that is not timedOutDuringCompaction. I found no tool-execution timeout cause/state in the runner. The issue discussion includes concrete logs and a correct root-cause analysis; related closed issues cover adjacent timeout behavior, not this remaining tool-execution misclassification.
