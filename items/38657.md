## Summary

Current main still preserves existing provider apiKey and baseUrl from agent models.json during merge-mode regeneration, so the broader Onboard overwrite/persistence bug remains unresolved. The issue discussion also explicitly kept this issue open after #39103 because that PR only covered a narrower stale baseUrl precedence case.

## Best Possible Solution

Keep this issue open. The best fix is an explicit Onboard/provider-auth update path where a deliberate same-provider edit persists new apiKey/baseUrl across openclaw.json, agent models.json regeneration, and auth-profiles handling, while preserving fallback behavior only when the new config omits those fields. Add regression coverage for same-provider edits, SecretRef-managed credentials, and the documented merge behavior.
