## Summary

Keep #50991 open. Current main has related compaction lifecycle hooks and a pre-compaction memory-flush system, but the bundled `session-memory` hook still only registers and executes for `/new` and `/reset`; the `/compact` command path compacts directly and does not emit `command:compact` or otherwise invoke `session-memory`. The KeepMyClaw comment is a useful external backup workaround, but it does not resolve the requested OpenClaw in-process hook behavior, and the provided related ClawSweeper items cover daily/idle/subagent/output-dir gaps rather than this compact-specific request.

## Best Possible Solution

Keep the issue open and implement a compact-specific preservation path if maintainers want this bundled behavior: either emit `command:compact` before manual and gateway compaction and register `session-memory` for it, or adapt `session-memory` to consume the existing `session:compact:before`/`before_compaction` seam with full previous transcript context. The patch should update the hook manifest/docs and add focused tests proving `/compact` writes a memory entry before compaction without regressing `/new` and `/reset`.
