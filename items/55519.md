## Summary

Keep open. Current main does not implement the PR's central behavior: bare `agentId` is not accepted as a `sessions_send` target, `sessions_spawn` still rejects `streamTo` for `runtime: "subagent"`, and the current system prompt lacks the Feishu-specific long-running group orchestration guidance. The PR is not safe to close as obsolete because it addresses a real gap, but it also has unresolved P1/P2 review findings that should be fixed before merge.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. The best path is a narrower patch that implements bare `agentId` resolution only when explicitly requested without a label, preserves the existing main-session A2A guard semantics or updates them deliberately with tests, ignores ACP-only `streamTo` for native subagents if that compatibility behavior is desired, and gates Feishu-specific prompt text on `runtimeChannel === "feishu"`. Add targeted regression tests for bare-agentId resolution, label-miss no-fallback safety, A2A-disabled main-session behavior, subagent `streamTo`, and non-Feishu prompt output.

## Risks / Open Questions

- Merging the PR as-is could misroute a label-targeted `sessions_send` after a label lookup miss by falling back to the newest session for the supplied agentId.
- Merging the PR as-is could change A2A enforcement for main/orchestrator sessions if the removed `requesterAgentId &&` guard is not restored or intentionally documented.
- The Feishu prompt hint should be gated to Feishu runtime context so other channel deployments do not receive channel-specific guidance.
