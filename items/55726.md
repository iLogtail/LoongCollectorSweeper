## Summary

Keep open. Current main still implements the exposure-matrix checks only for `groupPolicy="open"`, so the PR is not obsolete. The PR also has a valid latest review concern: legacy `dm.policy` should not override an explicit safer `dmPolicy`, matching config precedence.

## Best Possible Solution

Keep the PR or its linked bug active. The best path is to rebase the focused DM-parity exposure-matrix change onto current main, add/keep regression tests for channel and account `dmPolicy="open"`, legacy `dm.policy` only when `dmPolicy` is absent, sandbox suppression, and runtime/filesystem severity, then update audit-check docs if the finding semantics intentionally cover both group and DM policies.
