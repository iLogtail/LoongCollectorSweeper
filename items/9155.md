## Summary

Keep #9155 open. Current main at ee2ab9a6449219e1c425308a4832ddbb7eeb0ca2 still does not implement a primary `cron.defaults.delivery` config path: the cron config contract has no `defaults` block, the strict config schema would reject it, job creation only uses per-job delivery input, and omitted isolated-agent delivery still resolves through `channel: "last"`. The newer `cron.failureDestination` work covers failure notifications only and does not solve the requested default delivery target for normal cron output.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a real config-level primary cron delivery default or a maintainer explicitly chooses a different product direction. A complete fix should add `cron.defaults.delivery` to `CronConfig`, zod/generated schema/help/docs, cron add/create/normalization and relevant UI/CLI surfaces, with tests proving job-level delivery overrides the global default before falling back to `last`. Related #3520/#45806-style fixes and `cron.failureDestination` are useful but do not replace this request.
