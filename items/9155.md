## Summary

Keep #9155 open. Current main still lacks a config-level `cron.defaults.delivery` or equivalent primary cron delivery default; only failure-notification defaults exist, and omitted announce delivery still resolves to `last`.

## Best Possible Solution

Keep the issue open until OpenClaw implements a real primary cron delivery default, likely `cron.defaults.delivery`, or a maintainer explicitly chooses another product direction. A complete fix should add the config type/schema/help/docs surface, apply the default when cron jobs are created or normalized across gateway/tool/CLI/UI paths, and test precedence: explicit job delivery wins, then `cron.defaults.delivery`, then the current `last` fallback.
