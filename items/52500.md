## Summary

Keep #52500 open. The requested generic numeric bound hint is not implemented on current main: config validation still only enriches allowed-value/union errors, numeric `.min()`/`.max()` Zod issues fall through as plain messages, and there are no current tests covering ceiling/floor hint metadata. The directly related PR #62091 was closed unmerged per the provided GitHub context, and its referenced head commit is not present in this checkout.

## Best Possible Solution

Keep the issue open and either revive the focused approach from PR #62091 or implement an equivalent generic mapper for Zod numeric `too_big`/`too_small` config issues. The fix should append normalized maximum/minimum hints for all schema-validated numeric bounds, preserve the existing allowed-values behavior, and add regression tests for both ceiling and floor violations.
