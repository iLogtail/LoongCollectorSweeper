## Summary

Keep #9465 open. Current main still does not implement persisted per-job cron hooks, the narrower shell pre-hook gate, or pre/post/onError lifecycle phases around scheduled execution. Existing plugin hooks can see cron ctx.jobId and may short-circuit some cron-triggered agent runs, but that is a global plugin hook surface, not job-defined hooks in the cron schema, Gateway RPC, CLI/UI, or scheduler lifecycle. Related PR #45129 is still only an open implementation path in the provided context, and PR #9466 was closed unmerged docs-only.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class cron hook lifecycle across persisted cron types, Gateway schemas/RPC, scheduler/manual/startup execution paths, CLI/UI, docs, and tests, or explicitly supersede it with a narrower canonical tracker such as PR #45129 for shell pre-hook gating. Existing plugin hooks can remain a workaround for some deployments but should not be treated as closing the persisted per-job hooks request.
