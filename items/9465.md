## Summary

Keep #9465 open. Current main still does not implement the requested per-job cron hooks lifecycle: the cron job type has no `hooks` contract, Gateway cron schemas cannot accept or return job-level hooks, CLI/UI surfaces expose no hook options, and automatic, startup catch-up, and manual run paths execute jobs without a pre/post/onError hook gate. The provided discussion also includes a recent steipete Codex review keeping this issue open, with PR #45129 identified as a narrower open implementation path rather than shipped behavior.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive per-job cron hook lifecycle across cron types, Gateway schemas/RPC, timer/startup/manual execution paths, agent-tool and CLI/UI surfaces, docs, and tests, or explicitly supersede it with a narrower canonical tracker such as PR #45129 for shell pre-hook gating.
