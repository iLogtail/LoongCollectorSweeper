## Summary

Keep open. Current main still suppresses requested PTY execution when a sandbox runtime is active and does not warn the user, so PR #39349 still addresses the unresolved UX bug tracked by #38601.

## Best Possible Solution

Keep the PR open or land an equivalent fix on main with a small regression test around `createExecTool` when `pty:true` is requested and sandbox routing is active. If maintainers choose docs instead of a runtime warning, update the public exec docs too, but current main still has the silent behavior reported in #38601.
