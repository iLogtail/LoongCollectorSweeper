## Summary

Keep PR #39349 open. Current main still computes `usePty` as `params.pty === true && !sandbox` without adding a warning for the explicit `pty: true` plus sandbox case, so the PR is not implemented or superseded. The linked bug #38601 remains a real behavior gap, and the existing warnings plumbing would surface the proposed message if added.

## Best Possible Solution

Keep the PR open and either merge/rebase its narrow warning change with a targeted regression test, or have a maintainer decide on the alternate product behavior of erroring when explicit PTY is unavailable. Closing as implemented, duplicate, stale, or non-actionable would drop a still-valid fix for #38601.
