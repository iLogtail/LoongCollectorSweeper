## Summary

Keep #39349 open. Current main still silently converts an explicit `pty:true` request to non-PTY whenever the exec call resolves to a sandbox runtime, and I found no current warning branch, regression test, docs update, merged duplicate, or release/changelog evidence that supersedes this PR. The linked #38601 remains the same unresolved core exec UX bug in the provided GitHub context.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow core exec fix. Add a user-facing warning to the existing exec `warnings` path when `params.pty === true && sandbox`, verify it appears in both running and foreground result text, add regression coverage for sandbox and non-sandbox cases, and optionally clarify `https://docs.openclaw.ai/tools/exec` so users know PTY is unavailable in sandbox mode.

A sandboxed call with `pty:true` therefore passes `false` into execution.
