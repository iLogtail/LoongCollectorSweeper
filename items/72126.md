## Summary

Keep open. Current main still passes the OpenAI Realtime env fallback through as a literal string in both Talk browser-session creation and server-side realtime voice bridge creation, so PR #72126 remains a plausible focused fix for #72120 rather than cleanup-close material.

## Best Possible Solution

Keep #72126 open. The best path is to land a reviewed fix that resolves OpenAI Realtime API keys before both `talk.realtime.session` client-secret creation and realtime voice bridge creation, with focused tests for the env fallback/keychain-ref case and clear behavior for failed or transient keychain lookups. If maintainers prefer the existing SecretRef model, adapt the PR toward that contract rather than closing it as obsolete.
