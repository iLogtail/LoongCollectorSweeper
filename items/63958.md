## Summary

PR #63958 is superseded by merged/shipped QA-packaging fixes. Current main solves the linked completion-cache crash without packaging `qa/`, and later release checks explicitly reject private QA suite artifacts in npm packages, so this PR's package change now conflicts with the maintained policy.

## Best Possible Solution

Close #63958 as superseded. Keep the current source-only/private QA packaging policy, the optional scenario-pack fallback, and the completion-cache registration guard. Any future public QA scenario-pack distribution should be opened as a new maintainer-owned product decision rather than reviving this stale PR.
