## Summary

Keep open. Current main has clear evidence for the narrower `/status` fix, but this issue reports all native Discord slash commands returning `âś… Done.`, and the later discussion explicitly says the empty-response bug persisted independently of the duplicate-listener patch. The current implementation still has the same `âś… Done.` zero-output fallback for generic native command dispatch, while the direct-response branch and regression test are status-specific.

## Best Possible Solution

Keep the issue open, but narrow the next maintainer pass to current-main behavior for non-status native Discord commands. Verify `/help`, `/whoami`, `/acp`, `/models`, and `/model` with native commands enabled; if any still reach the zero-output dispatcher fallback, add focused regression coverage and route command-owned informational outputs through a deterministic interaction reply path rather than relying on an agent turn that can return no visible reply.
