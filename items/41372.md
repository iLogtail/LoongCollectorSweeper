## Summary

Keep open. Current main has addressed or documented several parts of the field report, including config schema/doctor repair, Discord gateway/inbound-worker hardening, and much of the cron CLI syntax. But the issue should not be auto-closed because a concrete cron documentation mismatch from the report still exists on main: the docs label `openclaw cron list` as listing all jobs, while the CLI still omits disabled jobs unless `--all` is passed. The report is also a broad bundle of core product/docs/operator issues, not a ClawHub-only idea or a duplicate clearly superseded by one canonical item.

## Best Possible Solution

Keep #41372 open for maintainer triage, but split it into focused follow-ups. The most concrete current fix is to correct the cron docs so `openclaw cron list` is documented as enabled-only by default and `--all` is shown for disabled jobs. Then decide whether remaining operator requests, such as cron upsert/resource warnings, isolated web-search repro coverage, and update-process detachment, need separate core issues or can be closed as already fixed by current config/Discord/onboard hardening.
