## Summary

Keep open. Current main does not implement the PR’s durable post-reset startup window, and the request is a plausible core/Discord session race fix rather than ClawHub or external cleanup. It is not safe to close as implemented or duplicate: #49001 is still open and thread-focused, while #53524 specifically tries to extend the behavior to regular Discord channels. The PR still needs maintainer review and regression coverage before merge.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The best path is to consolidate it with or after #49001 into a tested one-shot post-reset startup-window implementation covering both Discord thread and regular channel sessions. Add focused tests for channel sessions without `MessageThreadId`, non-owner messages arriving before the owner’s first post-reset turn, and successful embedded steer early-return cleanup.
