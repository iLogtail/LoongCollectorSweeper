## Summary

Keep #7524 open. Current main still only collapses direct-message peers through dmScope; group/channel peers remain channel-scoped isolated sessions, and the strict config/schema/docs do not expose groupScope or an equivalent group-to-main route-session override. The broader #19929 is open but has not been made canonical, and the latest maintainer-facing comment explicitly kept #7524 open.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly supersedes it with #19929. The best implementation is an additive core routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through buildAgentPeerSessionKey(), buildAgentSessionKey(), resolveAgentRoute(), route binding overrides, strict schemas/types/help/generated metadata, public docs, and regression coverage for BlueBubbles, Signal, Telegram, and broader channel/group routing.
