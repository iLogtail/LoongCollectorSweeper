## Summary

Keep PR #69835 open. Current main has not implemented the two central changes: explicit gateway `workspaceDir` is not in the public agent protocol and dispatch still only forwards inherited spawned workspace metadata, while workspace bootstrap loading/rendering still preserves `[MISSING]` markers. The PR is a plausible focused fix area, but the provided review comments identify real blockers that need rework before merge.

## Best Possible Solution

Keep the PR open for author or maintainer rework rather than cleanup-closing it. The best path is to gate any explicit `workspaceDir` override behind a trusted internal/admin path or validated session metadata, update the protocol/tests accordingly, and separately make an intentional decision on missing bootstrap files: either update loader, prompt rendering, docs, and stability/cache tests to omit them, or preserve markers and address the symlink case through the #40210/#40230 path.

## Risks / Open Questions

- Forwarding `request.workspaceDir` without an admin/internal guard would let write-scoped gateway callers redirect agent runs to arbitrary filesystem roots.
- Dropping missing bootstrap markers changes documented behavior and breaks current stability/cache/workspace expectations unless all affected tests and docs are updated together.
- Omitting missing bootstrap entries could mask guarded-read failures or the symlink issue tracked by #40210/#40230 unless maintainers explicitly choose that product behavior.
