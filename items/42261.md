## Summary

Do not close #42261. Current main still matches the reported bug: `streamReasoning` is only enabled when `onReasoningStream` is present, and `emitAgentEvent({ stream: "thinking" })` is still behind the same callback guard. The webchat gateway reply options shown in current main do not provide `onReasoningStream`, so WS clients still cannot receive live thinking events through this path.

## Evidence
