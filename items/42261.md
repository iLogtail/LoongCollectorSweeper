## Summary

Keep open. Current main still contains both callback-dependent guards described in #42261, so `reasoningMode: "stream"` without an `onReasoningStream` callback cannot reach the `emitAgentEvent({ stream: "thinking" })` broadcast path. The related fix PRs in the provided context (#15918 and #46121) were closed unmerged, and the current tests I inspected cover reasoning streaming with a callback present rather than the callback-free WS/webchat case.

## Best Possible Solution

Keep #42261 open. The best fix is the narrow decoupling described by the issue: set `streamReasoning` from `reasoningMode === "stream"`, let `emitReasoningStream()` emit `stream: "thinking"` when streaming is enabled even without `onReasoningStream`, and call the callback only when present. Add a regression test for `subscribeEmbeddedPiSession({ reasoningMode: "stream" })` without `onReasoningStream` that observes an agent event, plus a gateway/webchat test proving the emitted thinking event reaches WS clients.
