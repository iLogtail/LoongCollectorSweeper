## Summary

Keep #38597 open. Current main still uses whitespace-only Discord long-line splitting and falls back to a hard 2000-character split for CJK text with no spaces. The focused fix exists as open PR #71384, but it has not merged or shipped.

## Best Possible Solution

Keep #38597 open and use PR #71384 as the active fix path. The best implementation belongs in extensions/discord/src/chunk.ts: preserve whitespace priority for Latin/mixed text, add CJK terminal/closing punctuation fallback, add a safe CJK character-boundary fallback with a minimum-progress guard, and land focused regression tests for Chinese, Japanese, Korean, mixed-script, punctuation, long-token, and surrogate-pair cases.
