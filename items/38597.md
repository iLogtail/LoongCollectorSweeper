## Summary

Keep #38597 open. Current main still uses whitespace-only Discord long-line splitting and falls back to a hard 2000-character split for no-whitespace CJK text. The discussion already converged on a focused fix: @openperf’s #39399 patch was closed unmerged, and @p3nchan’s #71384 is the current open fix path but has not landed on main.

## Best Possible Solution

Keep #38597 open and use PR #71384 as the active fix path. The best implementation belongs in extensions/discord/src/chunk.ts: preserve whitespace priority for Latin and mixed text, add CJK terminal/closing punctuation fallback, add a safe CJK character-boundary fallback with a minimum-progress guard, and land focused Discord chunk tests for Chinese, Japanese, Korean, mixed-script, punctuation, long-token, preserveWhitespace, and surrogate-pair cases.
