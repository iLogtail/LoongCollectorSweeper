## Summary

Keep open. Current `main` still has the exact gap the PR describes: `gatewayWatch` runs `tsdown`, `runtime-postbuild`, and `build-stamp`, but not the standalone `copy-hook-metadata` step; `runRuntimePostBuild()` does not copy bundled hook `HOOK.md` files; hook discovery requires `HOOK.md`; and tsdown only emits bundled hook handlers into `dist/bundled/<hook>/handler`. The provided Greptile review and PR body add useful supporting context rather than superseding the work.

## Best Possible Solution

Keep the PR open for maintainer review. The best code path is a narrow fix that makes the gateway-watch runtime-postbuild path copy bundled hook `HOOK.md` manifests, ideally reusing or sharing the existing `copy-hook-metadata` behavior to avoid drift, with targeted tests for clean dist, missing source dir, and missing manifest cases.
