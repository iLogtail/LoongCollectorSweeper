## Summary

Keep open. Current main still returns `queuedFinal: false` for the exact plugin-bound `inbound_claim` handled path, and Telegram still sends the silent-reply fallback when `queuedFinal`, fallback delivery, dispatch error, and local delivery are all absent. The linked PR #70288 is open and unmerged, so the reported regression remains actionable on main.

## Best Possible Solution

Keep #70286 open until main has a regression-tested fix. The best implementation is to make plugin-bound async claims suppress channel silent-reply fallback without conflating that state with a locally queued final reply: update the dispatch result contract or finalization path, cover the plugin-owned `inbound_claim` handled case through the Telegram buffered dispatcher, and leave the broader typing/timeout behavior as a separate follow-up if maintainers want it.
