## Summary

Keep #14051 open. Current main still does not implement activity-based heartbeat idle pausing: the heartbeat config/schema/docs expose interval, active-hours, routing, prompt, timeout, light-context, isolated-session, and reasoning controls, but no `idleTimeoutMinutes`, `idleOnly`, `triggerWhen`, or equivalent recent-activity gate. The runner remains interval/active-hours/queue/preflight based, and the existing `lastInteractionAt` tracking is used for session reset freshness rather than heartbeat scheduling.

## Best Possible Solution

Keep #14051 open. A proper fix should add a product-approved heartbeat idle-timeout config key to exported types, strict runtime schema, schema help/generated docs, config-honor inventory, user docs, and focused tests. Runtime should track or reuse last allowed inbound user activity at the correct agent/session/channel boundary and gate heartbeat runs or scheduling on that activity window while preserving existing interval, active-hours, retry, targeted wake, task wake, light-context, and isolated-session behavior.
