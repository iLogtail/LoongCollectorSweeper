## Summary

Keep #52105 open. Current main narrows the issue because the `/think` executor catches its own RPC/read failures, but the Control UI dispatch layer still has the swallowed-command seams described in the issue: it clears the draft before dispatch, silently returns when `host.client` is absent, and does not wrap the generic `executeSlashCommand` dispatch/injection path in an error-to-message guard. The linked fix PR #52113 was closed unmerged, so the current tree does not prove the reported Control UI failure is fixed.

## Best Possible Solution

Keep the issue open and fix the Control UI dispatch layer: inject a visible system message when `host.client` is absent, wrap `executeSlashCommand` and result injection so unexpected errors become inline command results instead of swallowed drafts, and add focused `ui/src/ui/app-chat.test.ts` regression coverage for `/think` no-client and thrown-dispatch cases. After that lands, close #52105 against the implementation commit.
