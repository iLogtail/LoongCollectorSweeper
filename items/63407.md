## Summary

Keep PR #63407 open. Current main still has the Dockerfile-side `OPENCLAW_INSTALL_BROWSER` support, but the Podman setup path forwards only `OPENCLAW_DOCKER_APT_PACKAGES` and `OPENCLAW_EXTENSIONS` into `podman build`, and the Podman docs do not list the browser-install env var. The PR is a focused two-file parity fix and a maintainer comment on April 25, 2026 explicitly confirmed it remains valid and not fixed on current main.

## Best Possible Solution

Keep the PR open for maintainer handling. The best path is to rebase it if needed, land the guarded `OPENCLAW_INSTALL_BROWSER` passthrough in `scripts/podman/setup.sh`, and keep the matching Podman docs bullet so Podman reaches parity with the existing Dockerfile behavior.
