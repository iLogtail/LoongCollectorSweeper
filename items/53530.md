## Summary

Keep #53530 open. Current main still does not implement the PR’s central behavior: direct-session/DM session keys do not resolve channel DM `tools` or `toolsBySender`, and Feishu DM config entries still do not accept those policy fields. The related discussion is useful, especially #44886 and the companion #53496, but neither supersedes this narrow PR.

## Best Possible Solution

Keep the PR open and reconcile it with current main. The best path is a smaller rebased implementation that adds direct/DM policy resolution to the existing tool-policy path, wires Feishu `dms.*.tools` and `dms.*.toolsBySender` through schema/config docs, preserves empty-override fallback behavior, and keeps #53496 as the separate Feishu group-policy slice unless maintainers choose to merge the two.
