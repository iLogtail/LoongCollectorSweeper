## Summary

Keep PR #59859 open. Current main still has no native Linux companion app, docs still describe Linux companion apps as planned, and gateway client IDs/native UI allowlisting still exclude an `openclaw-linux` client. The PR is large and needs maintainer review, but it is a plausible implementation track for maintainer-owned issue #75, not implemented, not superseded, and not a ClawHub-only feature.

## Best Possible Solution

Keep #59859 open and route it through maintainer review for issue #75. The best path is to compare the C/GTK implementation in #59859 with the Rust/GTK alternative in #61576, choose or stage a canonical Linux companion direction, then require the selected track to settle gateway client ID/handshake changes, lifecycle/security review, packaging expectations, and unresolved bot review conversations before merge.
