## Summary

Keep #6820 open. Current main still does not expose `xhigh` for base `openai-codex/gpt-5.2`: the OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but the xhigh allowlist only includes the codex-suffixed `gpt-5.2-codex`. The linked fix PRs in the supplied context (#6013 and #43054) are closed and unmerged, and the current tests/harness do not cover the requested provider/model pair.

## Best Possible Solution

Keep #6820 open until the bundled OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `resolveThinkingProfile` or `listThinkingLevels("openai-codex", "gpt-5.2")`, or maintainers explicitly document that base Codex GPT-5.2 remains high-only while `gpt-5.2-codex` is xhigh-capable.
