## Summary

The reported `openai-codex/gpt-5.2` xhigh whitelist gap was real in an older implementation, but current `main` has moved past that code path and the xhigh capability is now handled through provider-based resolution rather than the old hard-coded list.

## Evidence

  - file: `src/auto-reply/thinking.ts`
  - sha: `f7c4d7a5f0c4efece9d93e2c82f35111bb89ee7f`

That matches the original report and confirms it targeted stale code.
