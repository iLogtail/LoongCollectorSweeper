## Summary

Keep #59933 open. Current main still preserves the reported mismatch under the default session-tool visibility contract: group/channel session rows can exist in the gateway/session store, but `sessions_list` filters same-agent sessions outside the current session tree unless `tools.sessions.visibility` is explicitly widened to `agent` or `all`. The provided Discord follow-up narrows the same behavior to the tool layer, not persistence.

## Best Possible Solution

Keep the issue open. Maintainers should decide whether `sessions_list` should include same-agent group/channel sessions by default, whether Telegram/Discord setup diagnostics should temporarily widen to `tools.sessions.visibility="agent"`, or whether the tool result should explicitly report that same-agent sessions were hidden by `tree` visibility. A targeted regression test should cover requester `agent:main:telegram:direct:<id>` with an existing `agent:main:telegram:group:<id>` row and verify the desired `sessions_list({ kinds:["group"] })` behavior.
