## Summary

Keep PR #69903 open for maintainer judgment rather than auto-closing. Current main has largely superseded the product direction with the bundled `diagnostics-otel` plugin, shipped in `v2026.4.24`, which provides opt-in OTLP metrics/traces/log export with redaction. But the PR's crash-path capture/flush and direct Sentry DSN transport are not fully implemented on main, and current plugin APIs do not expose the internal `log.record` stream to general external plugins, so neither `implemented_on_main` nor `clawhub` is a high-confidence close.

## Best Possible Solution

Do not merge the PR as-is. The best path is to keep the shipped OTLP-based diagnostics direction, then either add a focused generic crash/fatal diagnostics flush to `diagnostics-otel` or open a narrow plugin API proposal for a safe, redacted log/error-event stream. Avoid adding a direct `@sentry/node` core dependency unless a maintainer explicitly chooses Sentry-specific core ownership.
