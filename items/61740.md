## Summary

Keep #61740 open. Current main has adjacent ClawHub search, operator RPC, per-agent allowlists, plugin skill loading, and hot-refresh pieces, but it does not implement the requested runtime agent skill discovery: agents still receive a prompt-time `<available_skills>` snapshot, and there is no agent-callable `skill_search` or local semantic resolver. The related items are adjacent, not canonical duplicates.

## Best Possible Solution

Keep this open and re-scope it into a concrete product/API decision. The likely best path is either an optional ClawHub/plugin `skill_search` tool, or a narrow core read-only skill-discovery API that such a plugin can use to rank local plus ClawHub skills while respecting agent allowlists, skill gates, and explicit-install safety. If maintainers want this built into core, it should add tests around prompt snapshots, tool exposure, per-agent skill filters, and no silent skill installation.
