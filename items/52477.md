## Summary

Keep open. Current main still normalizes `chrome-extension://...` through `URL.origin`, which becomes the literal string `"null"`, and then compares that value against `gateway.controlUi.allowedOrigins`. The focused extension-origin regression test is also absent. The linked PR #38290 appears to track the right fix, but it is still open/unmerged, so the bug remains in current main.

## Evidence
