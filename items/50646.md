## Summary

Keep open. Current main still applies restricted session visibility to `sessions_list` and then returns only `count` and `sessions`, so the reported lack of scope/visibility feedback remains. The Gateway `sessions.list` result contract also has no visibility metadata. Ryce's KeepMyClaw/#50621 comment is useful operator context about session visibility and durability, but it is not a duplicate or an implementation of the requested OpenClaw output transparency.

## Best Possible Solution

Keep the issue open and add explicit visibility transparency to the session listing surfaces. A conservative fix would return or render effective visibility metadata for `sessions_list` and, if the protocol contract is extended, Gateway `sessions.list`: configured/effective visibility, whether results are scoped, and a user-facing note such as `tools.sessions.visibility=tree; showing current session tree only`. Add regression coverage for default `tree`, `self`, sandbox-clamped visibility, and `all`, while avoiding hidden-session count leaks unless authorized.
