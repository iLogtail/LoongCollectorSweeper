## Summary

Current `main` still uses the existing numeric config-backup ring (`.bak`, `.bak.1` ... `.bak.4`), and the repo test suite still asserts that behavior. This PR therefore is neither implemented-on-main nor obsolete; it remains a substantive behavior change on a live config-write path and should stay open for maintainer judgment rather than conservative stale cleanup.
