## Summary

Keep #71737 open. Current main has adjacent raw runtime events, internal AgentRunContext, gateway_start getCron access, and generic prompt hooks, but not the requested lifecycle-owned plugin SDK contracts. The linked draft PR #72082 appears to be the right implementation path, but it is still open/draft and not on main.

## Best Possible Solution

Keep #71737 open and use #72082, or a maintainer-accepted successor, as the implementation path. The core SDK should add typed sanitized agent event subscriptions, plugin-namespaced run-context helpers with run-end cleanup, session/plugin-owned scheduler job ownership and cleanup, heartbeat-scoped prompt contributions with deterministic ordering, and fixture tests for the acceptance criteria. After that lands on main, close this issue with the implementing commit SHA.
