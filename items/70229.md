## Summary

Keep #70229 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges`, has no `sandbox.docker.noNewPrivileges` config type/schema/docs surface, and does not report `noNewPrivileges=false` as a dangerous config flag. The PR is a focused fix for the linked open bug #43996, and the later PR commits address the bot's only noted test-coverage gap.

## Best Possible Solution

Keep the PR open and have maintainers review or land a rebased version of this focused fix: add `sandbox.docker.noNewPrivileges` with default `true`, wire it through config schema/types/docs/generated labels, omit the Docker flag only for explicit `false`, and report that weaker setting as dangerous config. This directly addresses #43996 without broadening sandbox defaults.
