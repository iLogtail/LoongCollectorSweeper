## Summary

Keep #70229 open. Current main still unconditionally emits Docker `--security-opt no-new-privileges` and does not expose `sandbox.docker.noNewPrivileges` in config types, schema, docs, resolution, or dangerous-config reporting. The linked bug #43996 has concrete reproduction details, and the PR is a focused fix attempt; Greptile's per-agent test-coverage concern appears addressed in the PR's later commit, so this should remain open for maintainer review/validation rather than cleanup closure.

## Best Possible Solution

Keep #70229 open for maintainer review and normal validation. The best product path is the PR's narrow core config escape hatch: default `sandbox.docker.noNewPrivileges` to true, allow explicit false for incompatible Docker hosts, document the risk, cover agent/shared-scope precedence and Docker args behavior in tests, and report `noNewPrivileges=false` as dangerous configuration.
