## Summary

Keep open. Current main still has the reported stuck-Control-UI failure mode if an embedded timeout/abort completes without a terminal lifecycle event: Control UI stays busy on a non-null chatRunId, chat.send only synthesizes a final event for non-agent-started runs, and the embedded timeout path still relies on agent_end/lifecycle delivery rather than directly emitting a terminal event from abortRun(true).

## Best Possible Solution

Keep the item open or explicitly consolidate it into #42213/#42258. The robust fix should guarantee a terminal chat/lifecycle state for embedded timeout/abort even when agent_end is missed, and add a Control UI recovery path such as an agent.wait-backed stale-run watchdog so chatRunId cannot keep the Stop button and queue blocked indefinitely.
