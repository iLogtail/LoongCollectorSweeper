## Summary

Keep open. Current main still appears to have the failure mode described in #42011: after a Control UI chat run has started, the webchat path depends on a terminal lifecycle/chat event to clear the client run, and the embedded timeout abort path does not itself guarantee that terminal event. Related #58067 is a different subagent progress/freeze issue, and closed #59536 tracks timeout partial-content leakage rather than the stuck Stop state.

## Evidence

- **Issue report has concrete timeout evidence:** The provided issue context includes a non-maintainer Windows 11 report against 2026.3.8 with an `embedded run timeout ... timeoutMs=600000` log followed by healthy gateway RPCs and a Control UI session stuck on Stop. Labels are `bug` and `bug:crash`, with no protected label or maintainer author association forcing keep-open independently.
- **Control UI server path still lacks a started-run fallback finalization:** `agentRunStarted` is set in `onAgentRunStart`; the dispatch `.then` path broadcasts a chat final only when `!agentRunStarted`. If a started embedded run settles without a lifecycle-derived chat event, the `finally` cleanup removes server-side tracking but does not broadcast `final`, `aborted`, or `error` for the client run.
  - file: [src/gateway/server-methods/chat.ts:2503](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/server-methods/chat.ts#L2503)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '2460,2755p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Timeout abort path still aborts the active session directly:** `abortRun(true)` marks the attempt aborted/timedOut, aborts the run signal, aborts compaction, and calls `activeSession.abort()`. This code path does not directly emit a lifecycle `end` or `error` event itself.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1966](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/agents/pi-embedded-runner/run/attempt.ts#L1966)

  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Terminal lifecycle emission remains tied to agent_end handling:** The embedded subscribe lifecycle handler emits lifecycle `error` or `end` from `handleAgentEnd`, after block/media flush and before resolving compaction wait. This supports the report's concern that a timeout/abort path that bypasses or loses `agent_end` can leave consumers without a terminal lifecycle event.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:39](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L39)

  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Client Stop state is cleared only by chat terminal events:** The Control UI chat controller clears `chatRunId` on `final`, `aborted`, or `error` chat payloads. If none is broadcast for the active run, `isChatBusy()` remains true and the Stop control can stay visible.
  - file: [ui/src/ui/controllers/chat.ts:684](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/ui/src/ui/controllers/chat.ts#L684)
