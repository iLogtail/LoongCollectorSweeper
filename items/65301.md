## Summary

Keep open. Current main still uses the broad `hasPollCreationParams` send guard, so metadata-only poll fields can still reject `action="send"`. PR #65301 is a coherent, targeted fix with regression coverage and appears to be the cleaner rebased version of the same work also seen in #65295, not something already implemented on main.

## Best Possible Solution

Keep #65301 open as the actionable fix candidate. Maintainers should review or land a cleaned version that uses a narrower send-side poll intent check, preserves full poll validation for `action="poll"`, and keeps the added regression coverage for metadata-only sends and real poll payload rejection. The older duplicate PR #65295 can be triaged separately if maintainers want one canonical PR.
