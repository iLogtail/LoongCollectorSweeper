## Summary

Keep PR #57366 open. Current main does not implement Teams CDN emoji alt extraction: Teams HTML `<img>` tags are still treated as inline image media for placeholders and downloads. The PR remains relevant, though the final fix likely needs maintainer review to cover both placeholder text and downloaded media behavior.

## Best Possible Solution

Keep the PR open and have maintainers either merge a revised version or implement the equivalent in the Microsoft Teams attachment layer: detect pure Teams emoji CDN HTML images by parsed host plus `alt`, surface the Unicode text as the inbound body, preserve fallback for mixed real media/non-HTML attachments, and add regression tests for both placeholder generation and inbound media download/filtering.
