## Summary

Keep this PR open. Current main does not implement the proposed proactive session archive cleanup at gateway startup or on an hourly timer. The existing cleanup paths are reactive/manual, and the PR discussion contains a real unresolved product/implementation question around custom session.store roots rather than evidence that the work is obsolete.

## Best Possible Solution

Keep the item open or supersede it with a focused maintainer patch that implements proactive core gateway cleanup using the existing session-store discovery surface, including configured/custom session.store roots. The final design should run one startup sweep plus one hourly timer, avoid duplicate concurrent sweeps, count successful deletions accurately, preserve active transcripts, cover .deleted/.reset archives, orphaned writeTextAtomic .tmp files, and numeric/ISO .bak files, and add regression tests for default roots, custom roots, zero retention, tmp age gating, and bak pruning.
