## Summary

Keep this PR open for maintainer judgment rather than auto-closing it. Current main still does not have the PR's proactive gateway startup/hourly archive sweep, and the linked disk-bloat issue #44820 remains the canonical open user problem. The PR is not merge-ready as-is because review identified a real gap for custom `session.store` roots, but that is a changes-needed state, not a close reason.

## Best Possible Solution

Keep #47635 open as a changes-needed PR, or ask for a replacement PR against #44820. The best implementation should add a core-owned proactive sweep at gateway startup and on a timer, but base target discovery on current session-store target APIs so default, templated, custom, and retired agent stores are covered; tests should prove `.deleted`, `.reset`, `.bak`, orphan `.tmp`, active transcript preservation, and custom `session.store` behavior.
