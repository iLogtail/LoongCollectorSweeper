## Summary

Keep #10957 open. Current main has substantial Android node support and local/release-bundle build mechanics, but it still does not provide or document Android beta/public access for an existing Gateway setup. The prior issue comment already kept this open for the same product boundary, and current main still matches that conclusion.

## Best Possible Solution

Keep #10957 open until maintainers either provide and document an Android beta/public distribution path for existing-Gateway phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android node capability and Play/third-party release hardening separately in the Android app and release docs.
