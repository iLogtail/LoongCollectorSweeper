## Summary

Keep #10957 open. Current main has Android node code, existing-Gateway pairing docs, capability surfaces, Play/third-party flavor work, and signed AAB mechanics, but it still does not provide or document Android beta/public access for an existing-Gateway phone node. The existing issue discussion already kept it open on that product boundary, and current main still matches that conclusion.

## Best Possible Solution

Keep #10957 open until maintainers either provide and document an Android beta/public distribution path for existing-Gateway phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android node capability, Play flavor, third-party bundle, and release-hardening work separately in the Android app and release docs.
