## Summary

Keep #72143 open. Current main still implements the reported user-facing prefix on the direct fallback path, has tests asserting that prefix, and does not retry the session resume/follow-up call before falling back. The related PR #72148 in the provided context is open and explicitly scopes out the requested retry/backoff work, so it does not make the issue obsolete yet.

## Best Possible Solution

Keep #72143 open as the canonical issue until main removes the user-facing fallback prefix, adds bounded retry/backoff around the agent session follow-up resume path, and records internal-only resume-failure diagnostics with approval/session/delivery metadata. PR #72148 may be a useful first step for the prefix regression, but it should not close the issue unless the remaining retry/logging scope is either implemented or split into a clearly linked follow-up accepted by maintainers.
