## Summary

Keep open. Current `main` still logs config warnings unconditionally at both `createConfigIO` warning callsites, and the proposed `loggedConfigWarnings` dedupe guard is not present on `main`. This PR addresses a real unresolved cleanup item rather than something already implemented or obsolete.
