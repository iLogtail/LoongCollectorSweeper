## Summary

Keep open. Current `main` does not contain an equivalent fix: device-less operator connects still depend on `sharedAuthOk`, and the websocket auth path still forces the secondary shared-auth probe to `allowTailscale: false`, leaving the reported Tailscale mismatch path unresolved on `main`. Existing tests also still codify the current cleared-scopes behavior rather than the PR's proposed preservation behavior.

## Evidence

- **Shared-auth probe still disables Tailscale:** `resolveConnectAuthState()` authenticates the websocket handshake first, then re-runs a secondary shared-auth probe with `auth: { ...resolvedAuth, allowTailscale: false }`. That means a successful primary Tailscale auth can still produce `sharedAuthOk = false` on the follow-up probe.
