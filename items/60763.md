## Summary

Keep #60763 open. Current main still lacks the requested Discord component `expiresAtMs` and `bindSession` fields in the public component spec, parser, message tool schema, and build wiring. The PR is not implemented on main or superseded by the closed original #60355; it remains useful but needs maintainer review for the security/design concerns raised in the discussion.

## Best Possible Solution

Keep the PR open and have the Discord plugin owner/maintainer decide the product shape. A good implementation would add `expiresAtMs` and `bindSession` to the Discord component spec, parser, message tool schema, docs, and tests, while bounding expiry timestamps and defining safe unbound-session authorization. If `bindSession=false` remains, entries and modals should clear session, agent, and account route overrides consistently and tests should cover both buttons and modals.
