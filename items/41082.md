## Summary

Keep open. Current main has meaningful compaction freshness mitigations, but the specific reported behavior, post-compaction topic switching where a new user ask is still overshadowed by an older summary, is not clearly fixed or covered by a direct regression path. The existing `npx clawaid` comment is a useful gateway-health workaround, but it does not prove a compaction/context-priority fix.

## Best Possible Solution

Keep the issue open and narrow it into a concrete compaction-context regression: build a compacted transcript with an older summary plus a later unrelated user ask, verify the request payload/order seen by the model, and add either prompt/request construction that clearly prioritizes post-compaction messages or a topic-switch heuristic with tests under the agent compaction/session replay surface.
