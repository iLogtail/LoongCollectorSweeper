## Summary

Keep PR #53965 open. Current main still has unguarded `chmodSync` calls in the active secrets write path, while this PR only adds an unused helper file and does not patch the code path identified in its own review comments. The broader JSON/config atomic-write helpers have Windows `EPERM` handling now, but that does not cover the remaining secrets apply/auth/env writes targeted by this PR.

## Best Possible Solution

Do not merge this PR as-is and do not auto-close it as implemented. The best fix is a narrow maintainer patch or refreshed PR that updates the existing secrets write helpers directly, ideally by reusing the shared `src/infra/json-files.ts`/`src/infra/json-file.ts` patterns or by adding equivalent best-effort chmod handling while preserving `ensureDirForFile`, temp-file replacement, rollback behavior, and secure modes. Add focused tests under `src/secrets` that simulate `chmodSync` throwing `EPERM` for auth/env writes and confirm secrets apply still succeeds or restores correctly.
