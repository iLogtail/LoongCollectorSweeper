## Summary

Keep open. Current main still has a real no-payload completion path: subagent announce reads/retries child output, but if no usable child result is found it builds and delivers a `task_completion` event with `result: "(no output)"` instead of fail-closing/defering. The tests also explicitly assert a completion-mode announce succeeds and contains `(no output)` when only user text is present. Related PR #68464 is an open, unmerged candidate with the right fail-closed direction, not shipped main behavior.

## Best Possible Solution

Keep #69767 open and either land a focused fail-closed fix or revive/supersede it with PR #68464. The implementation should make completion-mode announces return `false` and remain retryable when no usable user-facing result, structured delivery payload, or explicitly sanitized fallback exists, while preserving intentional `NO_REPLY`/`ANNOUNCE_SKIP` and terminal error behavior with clear tests.
