## Summary

Keep open. Current main has adjacent subagent-delivery hardening, but it still treats a missing subagent result as deliverable `(no output)` in the announce path, and the regression tests explicitly preserve that behavior. That does not close the reported early-completion/missing-payload race.

## Best Possible Solution

Keep #69767 open. The likely fix belongs in the subagent announce lifecycle: when `expectsCompletionMessage` is true and there is no non-silent frozen result, child findings, or real fallback result, leave cleanup retryable instead of building `(no output)`, refresh/freeze late output before retrying, and add regression tests proving no-payload completion is deferred rather than announced. Related PR #68464 is still useful context but is open/unmerged, so it should not supersede this issue yet.
