## Summary

Keep open. Current main has partial hardening for late subagent output and empty announce-agent payloads, but it still has user-facing completion paths that proceed with `(no output)` when no usable child result was captured. The issue's specific fail-closed request is not fully implemented on main.

## Best Possible Solution

Keep the issue open and fix the completion path so completion-message delivery fails closed when no usable child result, structured payload, sanitized fallback, or explicit error payload exists. In practice, `runSubagentAnnounceFlow`/lifecycle cleanup should return `false` and preserve retryable cleanup state instead of building `(no output)` for user-facing completion announces; tests that currently assert `(no output)` delivery should be updated to assert deferred retry or a real sanitized fallback. PR #68464 is likely the closest existing implementation direction if it is revived or superseded.
