## Summary

Keep open. Current main still hardcodes the 5 MiB media-store and sandbox-staging limit, does not expose `media.maxBytes` in config schema/types/help, and the linked bug #40880 plus PR discussion describe a real remaining user problem. The PR is not ready to merge as-is because later review comments flagged missing imports/startup wiring in the submitted branch, but that is a reason to request fixes/rebase, not to close conservative cleanup.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase plus fixes for the reviewed blockers: add `media.maxBytes` to schema/types/help/generated metadata, initialize the effective limit on every gateway boot with default reset behavior, use that effective limit in store/server/sandbox staging paths, preserve security caps, and add focused tests for default, configured, reset, and oversized staging/server behavior. Treat #67031 as the broader follow-up for other image/tool/provider limits.
