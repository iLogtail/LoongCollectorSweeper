## Summary

Current `main` does not implement this PR's `nvm` preload, and the existing installer flow still makes the underlying report plausible. At the same time, the PR has an unresolved correctness risk already called out in review: forcing `nvm use default` before `check_node()` can pin the shell to an outdated nvm default and cause the later post-install verification to fail. That is not a safe auto-close case.

## Evidence

- **Current main still does direct check/install without nvm preload:** In `main()`, installer step 2 still runs `check_node()` and then `install_node()` directly. There is no `try_load_nvm()` or equivalent preload on current `main`. That means the requested behavior is not already implemented.
  - file: [scripts/install.sh:2326](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/scripts/install.sh#L2326)
  - command: `nl -ba scripts/install.sh | sed -n '2310,2345p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Current fallback is explicit nvm guidance after a failed active-node check:** `ensure_default_node_active_shell()` detects nvm-managed shells and tells the user to run `nvm install 24`, `nvm use 24`, and `nvm alias default 24` before rerunning the installer. That is the current mainline behavior for stale nvm runtimes, which means maintainer intent is not yet aligned with this PR's automatic `nvm` activation.
  - file: [scripts/install.sh:1341](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/scripts/install.sh#L1341)
  - command: `nl -ba scripts/install.sh | sed -n '1330,1415p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Linux install path does not replace an already-prepended nvm runtime:** On Linux, `install_node()` installs Node via the distro package manager/NodeSource and then returns; it does not rewrite `PATH`. If a pre-check helper had already run `nvm use default` and that default was still below the minimum version, the shell would remain pointed at the old nvm binary and the later `ensure_default_node_active_shell()` check would still fail.
  - file: [scripts/install.sh:1408](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/scripts/install.sh#L1408)
