## Summary

Keep open. Current main does not implement an `x-openclaw-workspace` override on `/v1/chat/completions`: the OpenAI-compatible HTTP handler resolves agent/session/message-channel/model headers only, then builds an ingress agent command without `workspaceDir`. There is an existing `workspaceDir` command option, but code and comments scope it to internal spawned subagent workspace inheritance, not public HTTP per-request tenant routing. The KeepMyClaw comments are adjacent snapshot-isolation discussion, not an implementation or replacement for the requested Gateway header.
