## Summary

Keep #51451 open. Current main does not implement `x-openclaw-workspace` for `/v1/chat/completions`: the OpenAI-compatible handler resolves agent/session/channel/model headers, but does not read or forward a workspace override. There is an internal `workspaceDir` seam for spawned subagent workspace inheritance, and tests explicitly reject public workspace injection on the gateway agent method. The Ryce comments point to KeepMyClaw for snapshot isolation, which is related recovery/isolation context, but it does not satisfy this requested core HTTP routing behavior.

## Best Possible Solution

Keep the issue open for an explicit maintainer decision. If accepted, the likely path is a core, opt-in `/v1/chat/completions` workspace override with strict validation, allowed-root policy, tests proving no public injection by default, docs for the OpenAI HTTP API, and security docs clarifying that this is not hostile multi-tenant isolation. If maintainers decide shared-gateway tenant routing is out of scope, close manually with that product/security rationale rather than treating it as already implemented or ClawHub-only.
