## Summary

Keep #72215 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still lacks `cron.enabled` and `agents.defaults.compaction.mode` in the agent-facing gateway config mutation allowlist, while both are real documented config fields. The PR is a focused implementation candidate for open issue #72166, not obsolete or already implemented.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers accept the trust-boundary expansion, merge #72215 or an equivalent scoped change that adds exactly `cron.enabled` and `agents.defaults.compaction.mode` to `ALLOWED_GATEWAY_CONFIG_PATHS` with the proposed positive guard tests. If maintainers reject agent-driven edits for either field, close the linked issue/PR with that product decision and document that direct operator config RPC or CLI config writes remain the supported path.
