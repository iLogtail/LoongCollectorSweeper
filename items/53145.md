## Summary

Keep open. Current main has partial control-token hygiene, but the specific `sessions_send(..., timeoutSeconds: 0)` A2A announce path still runs follow-up/ping-pong plumbing, still uses the raw `Agent-to-agent announce step.` prompt as an injected agent message, and only treats `REPLY_SKIP`/`ANNOUNCE_SKIP` as A2A stop tokens rather than suppressing `NO_REPLY` or `HEARTBEAT_OK` before inter-session delivery.

## Best Possible Solution

Keep #53145 open and fix the A2A sessions_send boundary directly. The likely implementation belongs in `src/agents/tools/sessions-send-tool.a2a.ts` and `src/agents/tools/sessions-send-tokens.ts`: treat exact `NO_REPLY` and `HEARTBEAT_OK` as silent in reply and announce steps, prevent announce/ack follow-up text from being reinjected as user messages for `timeoutSeconds: 0`, and avoid persisting/displaying the raw `Agent-to-agent announce step.` placeholder as normal user-visible history. Add focused regression coverage for async `sessions_send` that asserts no placeholder, no `NO_REPLY`, and no extra echo turn reaches either session/channel.
