## Summary

Close: the Windows ESM path bug is already handled on current `main`, and the same normalization plus regression test are present in the shipped `v2026.4.23` tag.

## Evidence

- **Windows import-path normalization exists in shipped code:** `src/plugins/loader.ts` defines `toSafeImportPath`, explicitly documenting the Windows `ERR_UNSUPPORTED_ESM_URL_SCHEME` failure mode and converting absolute Windows paths like `C:\...` into `file:///C:/...` URLs before import.
  - file: [src/plugins/loader.ts:419](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/loader.ts#L419)
  - command: `git show v2026.4.23:src/plugins/loader.ts | nl -ba | sed -n '415,445p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Loader uses the normalization at actual module-load boundaries:** The plugin loader passes normalized paths into Jiti/native loading for runtime modules, plugin registration loads, setup runtime loads, and CLI metadata loads, so the failing Windows specifier is handled where the crash would occur.
  - file: [src/plugins/loader.ts:2418](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/loader.ts#L2418)
  - command: `git show v2026.4.23:src/plugins/loader.ts | nl -ba | sed -n '1920,1932p;2416,2429p;2507,2515p;3062,3072p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression test covers Windows drive-letter and UNC paths:** `src/plugins/loader.test.ts` asserts that `toSafeImportPath` rewrites both `C:\Users\alice\plugin\index.mjs` to `file:///C:/Users/alice/plugin/index.mjs` and `\\server\share\plugin\index.mjs` to `file://server/share/plugin/index.mjs`.
  - file: [src/plugins/loader.test.ts:6553](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/loader.test.ts#L6553)
  - command: `git show v2026.4.23:src/plugins/loader.test.ts | nl -ba | sed -n '6548,6568p'`
