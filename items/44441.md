## Summary

Keep PR #44441 open. Current main does not contain the proposed macOS watchdog runbook or updating-guide cross-link, and the PR still has substantive review concerns around whether its sample actually detects the advertised alive-but-unhealthy state and whether it targets non-default service profiles correctly.

## Best Possible Solution

Keep the PR open and ask for a focused docs revision: either narrow the runbook to a gateway reachability watchdog, or change the example to use a predicate that actually fails for the degraded state it claims to recover from. The LaunchAgent example should also show how to carry the intended `OPENCLAW_PROFILE`, `OPENCLAW_CONFIG_PATH`, and `OPENCLAW_STATE_DIR` values when the managed Gateway is not the default profile/config.
