## Summary

Keep #44441 open. Current main does not contain the proposed macOS watchdog runbook or updating-guide cross-link, and the PR is focused docs work rather than an obsolete, duplicate, or out-of-repo item. The right maintainer path is revision, not auto-close: the PR discussion already fixed the launchd PATH footgun, but later review comments identify actionable issues around using `openclaw health --json` as the predicate and preserving profile/config environment.

## Best Possible Solution

Keep the PR open and ask for a focused docs revision: either scope the watchdog honestly as a reachability watchdog or use a predicate that inspects the health JSON/deep probe for the intended degraded states, and add profile/config environment propagation for named-profile LaunchAgents. After that, the docs addition can be reviewed as a normal macOS operator runbook rather than closed by cleanup automation.
