## Summary

Keep open. Current main has partial fixes for OAuth refresh serialization and stale session-lock recovery, but the issue’s core multi-agent orchestration request is not fully implemented: concurrent `agents add` is still not serialized/merged, session-store writes still use a store-wide lock with a 10s default, and the A2A path does not implement the requested session-level processing semaphore/queue. The related open items (#44997, #42160, #44925) show remaining work rather than a single canonical replacement that fully supersedes this issue.

## Evidence

- **Concurrent agents add still derives config from one snapshot:** `agentsAddCommand` loads one config snapshot, captures `baseHash`, builds `nextConfig`, then calls `replaceConfigFile`; this can reject on concurrent changes, but it does not queue/re-read/merge all concurrent agent additions as requested.
