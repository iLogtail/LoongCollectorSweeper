## Summary

Keep open. Current main still has the exact 120-second outer Gateway timeout on `openclaw infer model run --gateway`; the server-side agent path can support longer/configured timeouts, but the infer gateway command does not expose or forward one. This is a real remaining product bug for local Ollama turns routed through Gateway, not a ClawHub/plugin-bound request or duplicate of the separate Ollama memory-embedding timeout item.

## Best Possible Solution

Keep the issue open. The best fix is to align `openclaw infer model run --gateway` with `openclaw agent`: add a timeout option or reuse `agents.defaults.timeoutSeconds`, forward `timeout` to the Gateway `agent` RPC, set the outer Gateway `timeoutMs` from that value with a buffer, and add focused CLI tests covering gateway model runs with local/Ollama-style slow responses.
