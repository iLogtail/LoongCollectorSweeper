## Summary

Keep open. Current main does not implement OpenRouter `autoRouter.allowedModels`, and the PR is a real user-facing provider feature request for the already-bundled OpenRouter provider. The branch still appears to need maintainer/contributor follow-up on behavior boundaries, so it is not conservative-cleanup close material.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is a narrowly scoped OpenRouter provider change in `extensions/openrouter/stream.ts` that injects the upstream `auto-router` plugin only for the canonical OpenRouter auto route, preserves existing `params.provider` routing behavior, exposes any test helper through `extensions/openrouter/api.ts` if core tests need it, and adds focused provider/docs/tests for absent, empty, exact-model, wildcard, and non-auto-model cases.
