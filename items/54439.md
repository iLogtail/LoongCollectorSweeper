## Summary

Keep #54439 open. Current main still contains the same unguarded Control UI render path: `app-render.ts` reads `job.payload.kind` while building cron model suggestions, and the cron controller stores `cron.list` rows into UI state without validating or filtering malformed jobs. The linked focused fixes #54550 and #54552 were closed unmerged in the provided GitHub context, and I found no current regression test or changelog entry showing this issue shipped.

## Best Possible Solution

Land a focused Control UI fix: guard or narrow cron jobs before reading `payload.kind` in `ui/src/ui/app-render.ts`, consider shared validation/filtering at the `cron.list` UI boundary, and add the jsdom regression test described by #54552 so malformed cron entries cannot block switching back to Chat.

## Evidence
