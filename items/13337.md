## Summary

Keep #13337 open. Current main does not contain a Vapi phone provider for the Voice Call plugin, and the existing plugin SDK surfaces do not expose a public telephony-provider registration seam that would let an external plugin satisfy the requested `voicecall` config/CLI/RPC behavior. The Clawdia Vapi bridge is useful related work, but it is an external bridge, not a first-class `@openclaw/voice-call` provider implementation.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A bundled fix would add a Vapi provider implementation, config/schema and manifest metadata, env/validation handling, runtime provider selection, focused provider/runtime tests, and Voice Call docs. If maintainers prefer external ownership, first add or document a public phone-provider extension seam, or explicitly endorse the existing Clawdia Vapi bridge as the supported Vapi path and close with that product decision.
