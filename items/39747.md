## Summary

Keep #39747 open. Current main still normalizes `heartbeat.target` through the already-loaded deliverable channel set before any plugin bootstrap path, so plugin-only heartbeat targets can still resolve to `none`. Telegram itself has explicit current coverage, but docs, schema hints, and validation still support non-bundled/plugin channel ids such as `bluebubbles`, so the underlying plugin-channel issue is not obsolete.

## Best Possible Solution

Keep #39747, or a focused successor, open until heartbeat target resolution bootstraps channel plugin metadata before deliverable-channel recognition, retries normalization after registry initialization, and adds a regression test for a plugin-only `agents.defaults.heartbeat.target` with an initially empty active channel registry. Maintainers should also decide whether `heartbeat.target` should count as gateway startup channel intent.
