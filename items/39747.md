## Summary

Keep #39747 open. Current main still normalizes `heartbeat.target` through the already-loaded deliverable channel registry before any plugin bootstrap path, so plugin-only heartbeat targets can still collapse to `none`. Telegram itself is now covered as a bundled channel id, but docs and config validation still allow external/plugin channel ids, which keeps the underlying issue real.

## Best Possible Solution

Keep the PR or a successor open until main bootstraps channel plugin metadata before heartbeat target normalization, then retries channel recognition and includes a regression test for a plugin-only `agents.defaults.heartbeat.target` with an initially empty active channel registry. Maintainers should also decide whether `heartbeat.target` should count as gateway startup channel intent.
