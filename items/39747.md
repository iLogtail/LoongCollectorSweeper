## Summary

Keep open. Current main still resolves `heartbeat.target` through `normalizeDeliverableOutboundChannel(rawTarget)` without passing config or bootstrapping plugin channel metadata first, so a plugin-only heartbeat target can still collapse to `none` when the registry has not already registered that channel. The latest PR diff in the provided context is not present on main and appears to address the earlier no-op review by moving the bootstrap retry into normalization, but it still needs maintainer review and regression coverage.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-owned fix that bootstraps channel plugin metadata before heartbeat target normalization, then retries normalization and adds a focused regression test for a plugin-only `agents.defaults.heartbeat.target` before registry initialization.
