## Summary

Current `main` already separates configured agent listing from disk-backed session discovery. The PR's proposed deletion is stale against that architecture and would regress behavior that `main` now tests explicitly for ACP/disk-only agent sessions.

## Risks / Open Questions

- The PR's change target is outdated: current `main` moved templated session-store discovery into `resolveAllAgentSessionStoreTargetsSync`, so removing disk IDs from `listConfiguredAgentIds` would not be the right fix path anymore.
- Closing this as already fixed would overstate the evidence. `main` intentionally preserves disk-only ACP session visibility, so the PR's requested `sessions.list` narrowing is not implemented on `main`.
