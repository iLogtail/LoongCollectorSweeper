## Summary

Current `main` does not implement the requested `usage.rateLimit` transcript schema. Assistant usage is still normalized to token/cost fields only, replay normalization rewrites persisted usage objects to that narrow shape, and the only direct handling of provider rate-limit headers I found is a live test that logs them for debugging. Main does expose provider quota snapshots via `usage.status`, but that is a separate surface and only for a limited provider set; it is not per-message session `.jsonl` logging of API response headers.
