## Summary

Current main still has the reported TUI footer behavior for the sparse-session case: when the TUI knows the context window but has no positive total token value, it renders `tokens ?/<ctx>` instead of `tokens 0/<ctx> (0%)`. The related fix PR #43044 was closed unmerged and its behavior is not present on main.

## Best Possible Solution

Keep the issue open and land a narrow TUI fix: when `formatTokens` receives a known context window with a null/undefined total, render `0/<ctx> (0%)` or otherwise propagate an explicit zero into the TUI footer. Add focused regression tests for `formatTokens(null, 200000)` and preserve the existing `tokens ?` behavior when both total and context are unknown.
