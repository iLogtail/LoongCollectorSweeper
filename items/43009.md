## Summary

Keep #43009 open. Current main still has the TUI formatter behavior described by the issue: when `totalTokens` is null/undefined but `contextTokens` is known, the footer renders `tokens ?/<ctx>` rather than `0/<ctx> (0%)`. The directly related fix PR #43044 remains open/unmerged, so the original bug report should stay open until a fix lands or a maintainer decides otherwise.

## Evidence

- **TUI footer uses formatter with nullable total/context values:** The footer calls `formatTokens(sessionInfo.totalTokens ?? null, sessionInfo.contextTokens ?? null)`, so a missing total token value reaches the formatter as `null`.
  - file: [src/tui/tui.ts:838](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/tui/tui.ts#L838)
  - command: `nl -ba src/tui/tui.ts | sed -n '810,855p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Current formatter still renders unknown total as question mark:** `formatTokens` sets `totalLabel` to `?` when `total == null`; with a known context it returns `tokens ${totalLabel}/${formatTokenCount(context)}`. For the issue case, that is still `tokens ?/200k`.
  - file: [src/tui/tui-formatters.ts:373](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/tui/tui-formatters.ts#L373)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '340,410p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Session list can still omit totalTokens while preserving contextTokens:** The session row resolves `totalTokens` only from fresh positive session/transcript totals, while `contextTokens` can resolve independently from entry, transcript, or model config. This permits the exact nullable-total/known-context state that the formatter displays as `?/ctx`.
  - file: [src/gateway/session-utils.ts:1354](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/gateway/session-utils.ts#L1354)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1310,1465p'`
  - sha: [e640c0a95fcb](https://github.com/openclaw/openclaw/commit/e640c0a95fcbd06a1311f18fc635518d011d45da)
- **Status output behavior differs from TUI:** The status message path defaults missing entry totals to `(inputTokens ?? 0) + (outputTokens ?? 0)`, so `/status` can show `0/<ctx>` while the TUI footer still shows `?/<ctx>`. This matches the issue's report that `session_status` looked correct but TUI did not.
