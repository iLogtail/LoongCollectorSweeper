## Summary

Keep open. Current `main` still has an unchecked streaming completions path that can coerce non-string `choice.delta.content` payloads into literal `[object Object]` text, which matches the reporter's local-model symptoms and is not shown as fixed in either the latest release or `main`.

## Evidence

- **Reporter provided a stronger reproduction clue:** The follow-up comment says the corruption also appears on `/new`, with session memory showing long `[object Object]` prefixes before the real assistant greeting, and the affected setups are local Mistral/Gemma-family models. That points at streamed assistant text assembly rather than a one-off UI rendering glitch.
- **Current main appends streamed completion content without runtime type checks:** In the OpenAI-compatible completions stream handler, `choice.delta.content` is passed straight into text accumulation with no `typeof === "string"` guard or flattening step.
  - file: [src/agents/openai-transport-stream.ts:1335](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/openai-transport-stream.ts#L1335)
  - command: `nl -ba src/agents/openai-transport-stream.ts | sed -n '1332,1340p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **That text accumulator will stringify objects as `[object Object]`:** `appendTextDeltaInternal` concatenates `text` directly into `currentBlock.text` and re-emits the same delta. If a provider sends an object or array instead of a plain string, JavaScript string coercion produces the exact symptom from the issue.
  - file: [src/agents/openai-transport-stream.ts:1264](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/openai-transport-stream.ts#L1264)
  - command: `nl -ba src/agents/openai-transport-stream.ts | sed -n '1264,1277p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **OpenClaw already has a defensive content flattener elsewhere, but not on this stream path:** `flattenStringOnlyCompletionContent` converts array-of-text-block content into a string for completion messages, showing the codebase already expects some OpenAI-compatible surfaces to need normalization. The buggy stream path above does not use an equivalent normalization step.
  - file: [src/agents/openai-completions-string-content.ts:1](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/openai-completions-string-content.ts#L1)

- **The latest release still carried the same unchecked stream logic:** The released `v2026.4.23` code shows the same `choice.delta.content` passthrough into `appendTextDelta`, so this is not implemented in the current shipped release either.
  - file: [src/agents/openai-transport-stream.ts:1289](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/openai-transport-stream.ts#L1289)
