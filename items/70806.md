## Summary

Keep open. Current main has partial safeguards for content-array prompt/history normalization, but the reported symptom is not fully closed out: the OpenAI-compatible streaming loop still passes `choice.delta.content` into text accumulation without a runtime string/object guard, so malformed local backends can still turn object deltas into `[object Object]`. The reporter also added a concrete `/new` memory-file example on 2026-04-24, so this is not stale or low-signal enough to close.

## Best Possible Solution

Keep the issue open and fix the remaining OpenAI-compatible streaming boundary: normalize `choice.delta.content` through a runtime helper that accepts strings and extracts text from structured content parts, drops unsupported objects instead of concatenating them, and covers both direct and post-tool buffered deltas with a regression test using Mistral/Gemma-style local stream chunks. After that, consider a narrow session/memory cleanup path for already persisted `[object Object]` lines.
