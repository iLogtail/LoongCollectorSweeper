## Summary

Keep open. Current `main` still uses `messagingToolSentMediaUrls` to strip duplicate final-reply media, but the embedded agent paths described in PR #41151 still do not populate that array when tool-result media is emitted directly. I found no equivalent fix on `main` in the inspected code.
