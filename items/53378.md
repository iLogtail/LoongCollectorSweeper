## Summary

Keep open. Current `main` still does not show a channel icon/name in the Control UI Sessions table; it renders the raw session key plus optional label/displayName and a generic direct/group kind badge. Related chat-dropdown work has channel-aware fallback labels, but that helper is not used by the sessions-list view.

## Best Possible Solution

Keep the issue open and implement the missing Sessions-table behavior in `ui/src/ui/views/sessions.ts`: expose/use channel metadata in the Control UI row type and live session-update fields, derive a stable channel label from `row.channel`/delivery metadata/session-key parsing, render it next to the session identity, and add focused UI tests for Telegram/Feishu-style direct keys plus unknown plugin channels. The existing chat session display helper is a good starting point but needs to be made suitable for the Sessions table.
