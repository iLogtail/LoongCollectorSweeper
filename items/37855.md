## Summary

Keep open. Current main still lacks cron support in the internal hook event contract and the gateway cron callback still only broadcasts cron events plus handles finished-job webhook/failure delivery. The related implementation PRs in the supplied GitHub context are closed unmerged, so there is no strong implemented, duplicate, or superseded close path.
