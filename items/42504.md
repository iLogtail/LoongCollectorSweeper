## Summary

Keep #42504 open. Current main can display configured agent avatars and `agents.update` can persist an avatar string into config/IDENTITY.md, but it still lacks the requested Agents-tab upload path: no `agents.avatar.upload` RPC, no upload schema/scope, and no `/agents` file picker that writes image bytes into `workspace/avatars/`. Related avatar-display fixes cover rendering and stale refresh behavior, not this server-side upload workflow.

## Best Possible Solution

Keep the issue open and implement a first-party agent avatar upload flow. The best path is to add a scoped admin RPC or equivalent upload endpoint that validates image type/size, writes bytes safely under the selected agent workspace such as `avatars/<agentId>.<ext>`, updates `IDENTITY.md` and config through the existing identity helpers, then wires the `/agents` avatar click/picker to call it and refresh `agents.list`, `agent.identity.get`, and `/avatar/<agentId>` rendering. Add gateway tests for validation/safe paths/IDENTITY updates and UI tests for the Agents-tab picker.
