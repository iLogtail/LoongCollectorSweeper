## Summary

Keep open. The reporter provided a fresh v2026.4.24 repro and the current main routing code still appears to drop exec-event deliveryContext.threadId when heartbeat is configured with an explicit Slack target/to instead of target="last".

## Best Possible Solution

Fix the exec-event heartbeat delivery path so a pending system event deliveryContext takes precedence over, or is merged into, configured heartbeat targets for exec-event wakes. Add a regression test for Slack with heartbeat target:"slack" plus explicit to and an event deliveryContext.threadId, asserting the outbound Slack payload receives that threadId/thread_ts.
