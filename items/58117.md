## Summary

Keep open. Current `main` still does not implement direct Mistral TTS support for `messages.tts.provider: "mistral"`. The bundled Mistral plugin only registers model, memory-embedding, media-understanding, and realtime transcription capabilities; TTS docs still omit Mistral; and the speech runtime skips unregistered providers and falls back instead. There is a partial workaround through the bundled OpenRouter speech provider, which exposes `mistralai/voxtral-mini-tts-2603`, but that is not the same as native Mistral-provider support requested in #58117.
