## Summary

Keep open. Current main has the underlying Telegram/media delivery capability, but the prompt cleanup requested by #43256 is not clearly implemented: the broad exec/curl prohibition is still present, and the message-tool prompt/test coverage still does not explicitly teach proactive attachment delivery with media/filePath/buffer. The related PR #43280 was closed unmerged, so it is not a canonical implemented fix.
