## Summary

Keep #29372 open. Current main still does not expose `agents.list[].subagents.workspace`, and native same-agent/default subagent spawns still preserve the caller's inherited workspace. The related cross-agent workspace bug is fixed and documented separately, but that does not implement the requested configurable child workspace for omitted-`agentId` spawns.

## Best Possible Solution

Keep #29372 open for maintainer product judgment. A complete implementation would add a documented `agents.list[].subagents.workspace` config field, decide whether `agents.defaults.subagents.workspace` is also needed, update TS types, zod/runtime/generated schemas, config help/docs, path normalization, and native same-agent/default spawn workspace resolution. It should add regression coverage for omitted `agentId`, explicit same-agent `agentId`, schema acceptance/rejection, `requireAgentId`, and cross-agent workspace preservation.
