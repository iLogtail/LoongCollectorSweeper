## Summary

Keep open. Current main still does not implement `agents.list[].subagents.workspace`. The cross-agent workspace inheritance bug related in #40171 is addressed by using the target agent workspace for cross-agent spawns, but this request is specifically for default/same-agent subagents to use a configured child workspace, and that config surface and behavior are still absent.

## Best Possible Solution

Keep #29372 open for maintainer/product judgment. A complete implementation would add a documented `agents.list[].subagents.workspace` config surface, decide whether an `agents.defaults.subagents.workspace` counterpart is needed, update TypeScript types, zod/runtime/generated schema/help/docs/path normalization, and change same-agent/default spawn workspace resolution so omitted `agentId` subagents can use the configured child workspace without regressing the already-fixed cross-agent target workspace behavior. Add regression tests for omitted `agentId`, explicit same-agent `agentId`, strict schema acceptance/rejection, `requireAgentId`, and cross-agent workspaces.
