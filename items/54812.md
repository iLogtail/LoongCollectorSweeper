## Summary

Keep open. Current `main` still implements `configWrites` only at channel/account scope. The core authorization path has no DM/group/topic dimension, and representative strict schemas for Telegram and WhatsApp do not accept nested `configWrites` entries under `groups`, `direct`, or `topics`. The docs and tests also describe only channel-level and multi-account overrides, so this feature request is still unmet and is recent/actionable rather than stale.

## Evidence

- **Core policy only models global, channel, and account scopes:** `ConfigWriteScopeLike`/`ConfigWriteTargetLike` only include global, channel, account, and ambiguous targets, and `resolveChannelConfigWritesShared()` resolves `configWrites` from account-or-channel only. There is no DM/group/topic scope in the authorization model.
