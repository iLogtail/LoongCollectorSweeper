## Summary

Keep open. Current `main` still classifies inbound Feishu `p2p` events solely by `chat_type`, while the rest of the Feishu plugin and docs treat `oc_` IDs as group/chat IDs, so this PR addresses a real inconsistency that is not implemented on `main`. There is also still an unresolved contract question because current tests encode some `oc_` + `p2p` cases as DMs, which argues for maintainer review rather than auto-close.

- Closing this PR would leave `main` with a documented/code-level inconsistency: outbound/config paths treat `oc_` as group chat IDs, but inbound routing still treats `p2p` events as direct solely from `chat_type`.
