## Summary

Keep open. Current main does not implement this PR's low/minimal-to-medium clamp for Claude Opus 4.7. The Anthropic Messages transport still maps `minimal` and `low` to `low`, then emits that as `output_config.effort` for adaptive-thinking models. The linked bug #70322 remains a live, concrete report, and this PR is a focused fix with targeted tests in the provided diff, not obsolete maintainer cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best product/code path is to implement a tested compatibility rule for GitHub Copilot Claude Opus 4.7 so low/minimal thinking cannot emit unsupported `output_config.effort = "low"`; ideally make the rule provider-aware or otherwise confirm the model-level clamp is correct for all Anthropic Messages users.
