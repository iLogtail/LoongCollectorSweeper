## Summary

Keep PR #70584 open. Current main still maps Anthropic adaptive `minimal` and `low` thinking to `output_config.effort: "low"`, and GitHub Copilot Claude models still route through the Anthropic Messages transport where that value is emitted. The PR’s regression tests are not present on main, and the linked #70322 problem remains a real uncovered code path rather than an obsolete or implemented change.

## Best Possible Solution

Keep the PR open, but maintainer review should likely ask for a provider/model-aware adaptive-effort compatibility fix in `src/agents/anthropic-transport-stream.ts`. Add regression tests for `github-copilot/claude-opus-4.7` low and minimal mapping, preserve Opus 4.6 behavior, and consider #69928's evidence before deciding whether high/xhigh should also be clamped or omitted for Copilot only.
