## Summary

Keep #65786 open. Current main still appears to reject the reporter's Feishu config shape: Feishu channel config is validated through the plugin runtime schema, that schema is strict, `streaming` is still boolean-only, and `contextVisibility` is still absent from the Feishu shared config shape even though Feishu runtime/tests use context-visibility behavior. `openclaw doctor --fix` also does not show a Feishu-specific compatibility migration for the reported legacy fields.

## Best Possible Solution

Keep the issue open and fix the Feishu-owned config compatibility path. The likely maintainer path is to add Feishu schema support for common channel fields the runtime already honors, decide how legacy `streaming: "progress"` should map to the current Feishu streaming model, add a doctor compatibility migration if old configs should be repaired automatically, and cover the reporter's sample with `validateConfigObjectWithPlugins` plus doctor-fix regression tests.
