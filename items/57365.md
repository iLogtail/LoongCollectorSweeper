## Summary

Keep open. Current `main` still does not implement source-based channel schema discovery for external plugins, and the repo now documents/tests the opposite manifest-first contract for non-bundled channel plugins. The PR discussion also points to a real affected plugin (`openclaw-zulip`) and only surfaced P2 review notes, so this is not stale or incoherent cleanup material.

## Evidence

## Risks / Open Questions

- Current main, docs, and tests intentionally enforce manifest-first metadata for external channel plugins; merging this PR would change that contract and needs explicit maintainer approval.
- If maintainers do want source-based discovery for external plugins, the docs, diagnostics, and tests around `channelConfigs` will need coordinated updates rather than cleanup closure.
