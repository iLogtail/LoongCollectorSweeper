## Summary

Keep #69251 open. Current main still routes POSIX `exec host=node` through `/bin/sh -lc`, always asks the node to build a `system.run.prepare` plan before the gateway approval policy can matter, and the node-side planner still fail-closes for shell payloads that resolve to mutable/script-like files. The related #69308 / #69077 discussion is orthogonal: it handled non-interactive `SHELL` fallback, not this approval-binding denial.

## Best Possible Solution

Keep the issue open and fix the core node exec approval planner/path. The safest direction is to preserve the POSIX transport wrapper while making `system.run.prepare` compute and revalidate node-local mutable file operands for shell payloads such as `/tmp/test.sh`, `/bin/zsh /tmp/test.sh`, and `python3 /tmp/script.py`, or otherwise pass enough approval-policy context to avoid mandatory stable binding when the node explicitly runs `security: full` with `ask: off`. Add focused tests around `executeNodeHostCommand` and `buildSystemRunApprovalPlan` so #69308's shell fallback remains orthogonal.
