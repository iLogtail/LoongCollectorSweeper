## Summary

Keep open. Current `main` still routes `openclaw wiki status` and `openclaw wiki bridge import` through the local CLI implementation instead of the gateway path, and that implementation still depends on active memory public-artifact registration. The CLI metadata loader only exposes `registerCli`, while `memory-core` registers `registerMemoryCapability` only in its full plugin entry, so the zero-artifact CLI failure described in the issue remains plausible on `main`. The destructive prune path is also still unconditional when `activeKeys` is empty.

That means the stock CLI path still relies on local plugin-state wiring.
