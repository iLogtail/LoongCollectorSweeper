## Summary

Keep open. Current `main` still has the same structural CLI gap described in the issue: `openclaw wiki status` and `openclaw wiki bridge import` run local plugin code, the CLI metadata path for `memory-core` does not register the memory capability/public-artifact provider, and bridge import still prunes existing bridge pages when artifact discovery is empty.

## Evidence

- **Wiki CLI still uses local sync/status path, not gateway RPC:** `runWikiStatus` calls `syncMemoryWikiImportedSources()` and `resolveMemoryWikiStatus()` directly, and `runWikiBridgeImport` calls `syncMemoryWikiImportedSources()` directly. The `wiki status` command wires to that local function path, so the stock CLI still depends on local plugin-state restoration instead of routing through `callGatewayFromCli`.
