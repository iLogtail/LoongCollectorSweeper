## Summary

Keep open. Current main has the Skill Workshop plugin, but it does not yet contain the PR’s central hardening: realpath-aware containment for Skill Workshop writes, a second prepare immediately before atomic write, prompt-budget preview/enforcement exposed through the plugin SDK, or the qa-lab routing regression coverage. The related #71768 is closed and unmerged; #72115 is the active extension-only follow-up rather than a duplicate or shipped change.

## Best Possible Solution

Keep #72115 open for maintainer review. The best path is to land or reshape the PR’s extension-owned hardening on the existing Skill Workshop plugin surface, preserving the plugin-SDK seams only if they are accepted as stable public runtime contracts. If scope remains a concern, split qa-lab validation hardening from the Skill Workshop/security-runtime/skills-runtime changes rather than closing the PR as obsolete.
