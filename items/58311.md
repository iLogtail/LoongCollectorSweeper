## Summary

Keep open. Current `main` still contains the exact `existsSync()` -> `readFileSync()` TOCTOU pattern in all three paths named by the PR, so the request is not implemented on `main`. The review discussion also surfaces an intentional behavior change for non-`ENOENT` errors in `device-auth-store` and `exec-approvals` that merits maintainer judgment rather than auto-closing.
