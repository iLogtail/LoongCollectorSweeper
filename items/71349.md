## Summary

Keep open. Current main has memory-lancedb hook wiring and tests, but the implementation still auto-captures only on `agent_end`, only for user messages, only when a narrow trigger/length/injection filter passes, and only up to the first three capturable texts. That does not prove the reporter's fresh 2026.4.23 failure is fixed, and the adjacent doctor warning path still has a plausible current-code basis.

## Best Possible Solution

Keep #71349 open. Reproduce on current main with `plugins.slots.memory = "memory-lancedb"` and the reporter's Ollama-compatible embedding config, then add targeted coverage for gateway/Telegram/cron `agent_end` auto-capture persistence. Maintainers should also decide whether memory-lancedb autoCapture is intentionally trigger-only or should produce per-session/per-turn entries; if trigger-only is intended, update docs/diagnostics and separately fix the doctor memory-slot warning path.
