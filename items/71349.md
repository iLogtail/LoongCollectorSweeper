## Summary

Keep open. Current `main` does not contain a fix for the reported low auto-capture behavior, and the adjacent doctor warning is still inconsistent with a valid `memory-lancedb` slot configuration. The code also makes silent missed captures plausible because `agent_end` hooks are dispatched fire-and-forget while `memory-lancedb` capture itself is narrow and heuristic-driven.

## Evidence

- **No post-release change in memory-lancedb capture path:** `git diff a9797214338ba31c52c796adbb75afb16e0684a9..734748d4f4fef7436def57f7a7ae3350bba9a091 -- extensions/memory-lancedb/index.ts` returned clean, so current `main` still ships the same `memory-lancedb` auto-capture implementation as the reported `2026.4.23` release.
  - command: `git diff --exit-code a9797214338ba31c52c796adbb75afb16e0684a9..734748d4f4fef7436def57f7a7ae3350bba9a091 -- extensions/memory-lancedb/index.ts`
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Auto-capture is selective, not per-turn:** The plugin only captures on successful `agent_end`, only inspects `role === "user"` messages, and stores at most three filtered texts after duplicate checking. This does not match a per-session or per-turn persistence model.
  - file: [extensions/memory-lancedb/index.ts:613](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-lancedb/index.ts#L613)
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Heuristics are intentionally narrow:** `shouldCapture(...)` rejects short text, over-limit text, recalled/system-like XML blocks, markdown summaries, emoji-heavy content, prompt-injection patterns, and anything that does not match the hard-coded trigger regexes.
  - file: [extensions/memory-lancedb/index.ts:247](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-lancedb/index.ts#L247)
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Default size cap is 500 chars and tests lock it in:** `DEFAULT_CAPTURE_MAX_CHARS` is 500, and the test suite asserts that a longer message is skipped unless `captureMaxChars` is raised. That makes 'rarely fires' plausible for longer real-world user turns.
  - file: [extensions/memory-lancedb/index.test.ts:1545](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/extensions/memory-lancedb/index.test.ts#L1545)

- **agent_end hooks are explicitly fire-and-forget:** The hook runner documents `agent_end` as 'Runs in parallel (fire-and-forget)'. That means asynchronous capture work is not part of the caller's completion path.
  - file: [src/plugins/hooks.ts:592](https://github.com/openclaw/openclaw/blob/734748d4f4fef7436def57f7a7ae3350bba9a091/src/plugins/hooks.ts#L592)
  - sha: [734748d4f4fe](https://github.com/openclaw/openclaw/commit/734748d4f4fef7436def57f7a7ae3350bba9a091)
- **Embedded and CLI runners do not await capture completion:** The embedded runner calls `runAgentEnd(...)` and immediately continues, and the generic harness helper used by CLI runs also dispatches `agent_end` with `void ...catch(...)`. Silent loss is therefore plausible for short-lived runs such as cron/CLI sessions.
