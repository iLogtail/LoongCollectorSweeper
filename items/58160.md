## Summary

Current `main` at `dc7c7034254971fa6e9769e99a18da88f432f8c7` does not contain the proposed late-output snapshot sync, and the existing tests on `main` do not cover the reported race. The PR still appears to address a real gap, so it is not a conservative close candidate.

## Evidence

- **Finished snapshot is frozen at exit on current main:** `appendOutput()` updates the live session's `aggregated` and `tail`, but does not sync any already-finished snapshot afterward. `markExited()` immediately calls `moveToFinished()`, which copies `aggregated`, `tail`, `truncated`, and `totalOutputChars` into `finishedSessions` once.
  - file: [src/agents/bash-process-registry.ts:133](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/bash-process-registry.ts#L133)
  - command: `nl -ba src/agents/bash-process-registry.ts | sed -n '118,230p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **`process log` reads only the finished snapshot for completed backgrounded sessions:** When a session is no longer running, `process log` slices `scopedFinished.aggregated` and can emit `(no output recorded)`. That means any late chunk not copied into the finished snapshot will be invisible to users.
