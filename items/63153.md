## Summary

Keep open. Current `main` still contains the Bonjour watchdog path that restarts the advertiser when it stays non-`announced`, and the test suite explicitly asserts the `stuck in announcing` restart behavior. There are now documented/workable disable paths for LAN Bonjour advertising (`discovery.mdns.mode: "off"`, `openclaw plugins disable bonjour`, or `OPENCLAW_DISABLE_BONJOUR=1`), but that is a workaround, not evidence that the reported self-healing/log-flood behavior was fixed.

## Evidence

- **Current main still restarts the advertiser on stuck announcing:** The bundled Bonjour advertiser watchdog calls `recreateAdvertiser(...)` once a service has remained non-`announced` for at least `STUCK_ANNOUNCING_MS`, logging `bonjour: restarting advertiser (...)` with the `service stuck in ...` reason.
