## Summary

Keep open. Current `main` no longer has the exact file/shape this PR edits, but it still does not provide an equivalent diagnostic log when `web_search` fails to resolve a provider API key, and the submitted patch itself had unresolved review defects.

## Evidence

- **Current web_search entrypoint is only a thin wrapper:** `createWebSearchTool` now delegates straight to `runWebSearch(...)` and returns the JSON result; there is no provider-selection or missing-key diagnostic logging at this layer anymore, so the PR's old-file patch has not been implemented on `main`.
