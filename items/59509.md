## Summary

Keep #59509 open. Current main does not implement the requested per-model `location` override for `google-vertex`; the relevant config and runtime paths still expose only agent-model aliases/params/streaming plus provider-level/model-definition `baseUrl`, and the resolved model path does not consume a configured model-level `location` or `baseUrl` for Vertex endpoint selection.

## Evidence

- **Agent model config has no location field:** `AgentModelEntryConfig`, which backs `agents.defaults.models["provider/model"]`, only defines `alias`, `params`, and `streaming`. The issue's proposed top-level per-model `location` field would not be part of this config surface.
  - file: [src/config/types.agent-defaults.ts:31](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/src/config/types.agent-defaults.ts#L31)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '1,60p'`
  - sha: [b64bfc5d9abf](https://github.com/openclaw/openclaw/commit/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab)
- **Per-model params are request extras, not endpoint selection:** Runtime extra params are merged from `agents.defaults.params`, `agents.defaults.models[provider/model].params`, and agent params. There is no resolver that treats `location` as a Vertex endpoint override.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:67](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/src/agents/pi-embedded-runner/extra-params.ts#L67)

- **Model/provider config has baseUrl but no location:** `ModelDefinitionConfig` supports optional `baseUrl`; `ModelProviderConfig` has provider-level `baseUrl`. Neither type defines `location`, so the requested model config key is not implemented in the typed config contract.
  - file: [src/config/types.models.ts:77](https://github.com/openclaw/openclaw/blob/b64bfc5d9abf078217a0a63c25d4c2bcbf4e6fab/src/config/types.models.ts#L77)
