## Summary

Keep open. Current main does not implement the requested per-model Google Vertex `location` override, and the obvious documented endpoint workaround is not clearly available through runtime model assembly. The Google provider hook aliases include `google-vertex`, but current config/runtime paths resolve Google request URLs from the runtime model `baseUrl`, not from a model-level `location`; `agents.defaults.models` entries only expose `alias`, `params`, and `streaming`, and the provider docs only mention Vertex ADC/project setup rather than per-model region selection.

## Best Possible Solution

Keep the issue open and implement this in the Google provider/runtime path. A conservative fix would add a provider-owned, documented `google-vertex` model-level endpoint override, preferably `agents.defaults.models["google-vertex/<model>"].params.location` or a typed equivalent, with priority `model location -> provider/base config -> GOOGLE_CLOUD_LOCATION`. The implementation should construct the correct `global` or regional `aiplatform.googleapis.com` base URL for the runtime model without duplicating providers, and should include tests for mixed preview/GA Vertex models plus docs under the Google/Vertex model provider section. If maintainers prefer the existing catalog contract, first make `models.providers.*.models[].baseUrl` actually win per model, then document that as the supported Vertex-region override path.
