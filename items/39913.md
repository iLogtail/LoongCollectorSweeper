## Summary

Keep #39913 open. Current main still applies `messages.responsePrefix` on the normal reply path, but message-tool `send` still forwards raw outbound text through plugin dispatch, core `sendMessage`, and gateway/direct delivery without resolving or applying the prefix. The prior Codex keep-open comment remains accurate against current main `4a3030df9efa72c44bab567a4390ac0c4876a73f`.

## Best Possible Solution

Keep #39913 open. Implement prefixing once in the shared message-tool outbound send boundary before plugin/core/gateway dispatch, resolving `responsePrefix` with agent, channel, and account context and using the same idempotent normalization semantics as direct replies. Preserve media-only sends, existing cross-context decoration, mirrors/transcripts, and plugin dispatch behavior. Add regression coverage for `runMessageAction`/`handleSendAction`, `executeSendAction`, and the core/gateway send path.
