## Summary

Keep open. Current main still applies `messages.responsePrefix` on the normal reply path, but the `message` tool outbound `action=send` path passes raw message text through to outbound delivery without resolving or prepending the response prefix. The issue discussion and prior Codex review correctly identify a real inconsistency, and current main has not fixed the relevant outbound code.

## Best Possible Solution

Implement the missing behavior in the shared message-tool outbound path: resolve the effective response prefix for the target channel/account/agent/session, apply the same normalization semantics used by direct replies before dispatch, avoid double-prefixing, and add focused regression tests covering global, channel, and account prefixes for `message(action=send, ...)`.
