## Summary

Keep open. Current `main` still applies `messages.responsePrefix` on the normal reply-routing path, but the message-tool `action=send` path never resolves or prepends that prefix before dispatch. The latest release shares the same outbound message-tool code, so there is no strong evidence this was already fixed or became obsolete.

## Evidence
