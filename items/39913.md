## Summary

Current `main` still has the reported inconsistency. Direct reply delivery resolves and applies `messages.responsePrefix`, including channel/account overrides, but the `message(action="send", ...)` path keeps the outbound text raw through message-tool handling, plugin dispatch, core send, and gateway delivery.

## Best Possible Solution

Keep #39913 open. Implement prefixing at the shared message-tool outbound boundary by resolving `responsePrefix` with agent, channel, and account context, normalizing outgoing text through `normalizeReplyPayload()` or an equivalent shared helper before plugin/core/gateway dispatch, preserving media-only sends, and adding regression tests for `runMessageAction`, core send, and gateway send.
