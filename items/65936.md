## Summary

Current `main` and the latest release still do not include `Asia/Shanghai` in `CRON_TIMEZONE_SUGGESTIONS`, so this PR is not already implemented. At the same time, the Control UI timezone field is freeform and the codebase already validates `Asia/Shanghai` as a valid IANA timezone, so the PR remains a small UX improvement rather than an obsolete or broken change.
