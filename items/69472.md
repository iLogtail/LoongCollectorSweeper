## Summary

Keep PR #69472 open. Current main still has the reported central auto-reply gap: `isSilentReplyPayloadText` only recognizes exact `NO_REPLY` or the strict JSON envelope, and the later mixed-content stripper does not collapse punctuation-separated doubled tokens like `NO_REPLY.NO_REPLY`. The PR remains a relevant fix attempt for #69470, though it should address the existing false-positive review concern before landing.

## Best Possible Solution

Keep #69472 open. The best maintainer path is to port/adapt the central `src/auto-reply/tokens.ts` fix with regression tests in `src/auto-reply/tokens.test.ts`, while addressing the false-positive review concern so #19537 semantics remain intact for substantive replies that merely end with the sentinel.
