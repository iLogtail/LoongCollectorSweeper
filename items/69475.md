## Summary

Keep #69475 open. Current main still has the four flagged bundled skill frontmatter descriptions without explicit trigger/WHEN language, and OpenClaw's system prompt selects skills by scanning the `<description>` entries before reading any SKILL.md body. The `gh-issues` skill also remains oversized at 5006 words / 885 lines with no `references/` split. The related PR #69489 was closed unmerged, and #69529 only mentions a partial two-skill change and is still open/unmerged, so there is no landed canonical fix.

## Best Possible Solution

Keep the issue open and land a focused bundled-skill cleanup: update the four frontmatter descriptions with concise WHAT plus WHEN trigger language, fold in the reporter's follow-up notes about Contacts coverage and Obsidian-specific triggers, then either refactor `skills/gh-issues/SKILL.md` into linked `references/` files or make an explicit maintainer decision to document a different OpenClaw size guideline. A small static audit for bundled skill descriptions/size would prevent drift.
