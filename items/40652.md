## Summary

Current `main` changed the startup flow to auto-restore many invalid configs from a last-known-good copy, but the remaining startup-error path still only suggests `openclaw doctor --fix` and does not mention manual backup recovery. Because that exact behavior is asserted in tests, this issue is not safely closable as already fixed or obsolete.
