## Summary

PR #65376 adds ATOM as a new bundled provider plugin, but current main already has the public provider-plugin and self-hosted OpenAI-compatible helper surfaces needed to ship this outside core. The PR discussion and linked issue #65811 are useful: they show a working extension-scoped implementation and explicitly ask whether a bundled provider is the right path. Given OpenClaw's current scope guidance, this should move to a standalone ClawHub plugin unless maintainers later choose official bundled promotion.

## Best Possible Solution

Move the ATOM implementation from this PR into a standalone provider plugin published through ClawHub, using the existing provider SDK and self-hosted OpenAI-compatible setup helpers. Keep OpenClaw core unchanged unless maintainers explicitly decide ATOM should become an official bundled provider or a concrete missing plugin API is identified.
