## Summary

Keep open. Current main still has no guard that filters native-provider catalog entries inferred from OpenRouter-style model ID prefixes before `/model status` groups by provider. The linked fix PRs are still open/draft in the provided context, so there is no shipped main implementation to close this against.

## Best Possible Solution

Keep #62317 open and review/land one focused fix, preferably in `loadModelCatalog` so downstream `/model`, `/models`, and gateway catalog consumers receive a clean catalog. The fix should suppress only auto-inferred native-provider rows duplicated by a configured aggregator such as OpenRouter, while preserving explicitly configured native providers, and add regression coverage for both cases.
