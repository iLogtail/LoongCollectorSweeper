## Summary

Keep open. Current main still trims context by returning rewritten messages from the context-pruning Pi hook, but it does not emit a `session:prune` or `context:prune` lifecycle event, expose pruning metadata, or document such an event. The issue discussion points to #9761 and #12581; #9761 was closed unmerged, and #12581 remains the direct unmerged implementation path.

## Best Possible Solution

Keep #8606 open until #12581 or a replacement maintainer-approved implementation lands. The likely path is to extend `src/agents/pi-hooks/context-pruning/*` so pruning collects safe metadata, emits an internal `session:prune` or equivalent lifecycle event after successful pruning, adds focused tests, and documents the event in the public hooks and session-pruning docs.
