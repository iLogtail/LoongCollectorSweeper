## Summary

Keep open. Current main improves the original startup-blocking symptom by deferring pricing refresh work, but it does not implement the requested disable switch and still performs remote OpenRouter/LiteLLM pricing fetches with a 60s timeout. Timeout warnings are still covered as expected behavior in tests, and the linked fix PR #68651 remains separate from main rather than shipped.

## Best Possible Solution

Keep #53639 open until a core fix lands. The best path is to either land a maintained version of #68651 with config types, zod schema, help/labels, docs, generated baseline updates, and regression tests, or otherwise make the pricing bootstrap non-disruptive enough that offline/restricted users do not get long timeout noise. Current local `models.providers.*.models[].cost` overrides remain useful but are not a substitute for an explicit bootstrap control.
