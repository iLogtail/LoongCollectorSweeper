## Summary

Keep #62121 open. Current main has shipped phase-aware/commentary safeguards and Telegram can prefer a final assistant-visible text field for text-only cron delivery, but the reported remaining case is untagged DeepSeek preamble in isolated cron delivery. The code still treats unphased assistant text as visible output, cron has no provider-agnostic final-output boundary or supported output-start marker field, and a maintainer comment explicitly left this issue open for a cleaner cron final-output design.

## Best Possible Solution

Keep this issue open for a core cron/reply-delivery design pass. The best fix is provider-agnostic: define the final-output boundary for isolated cron delivery, coordinate with the broader channel leak work in #25592, preserve the shipped commentary/final_answer hardening, and add regression coverage for unphased DeepSeek-style preamble before Telegram delivery. Avoid DeepSeek-specific regexes or emoji marker hacks unless maintainers explicitly choose a configurable sanitizer surface.
