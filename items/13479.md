## Summary

Keep #13479 open. Current main still does not provide click-to-reveal cron job details/summary in the cron dashboard: row clicks select run history, the list renders prompt/system payload inline, and `job.description` is accepted by the data/form contract but not rendered in the cron job card/detail surface.

## Best Possible Solution

Keep the issue open. The best implementation path is to add explicit selected-job detail or expansion state for the cron dashboard in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary alongside the existing prompt/system payload, preserve run-history selection controls, add English i18n strings, and cover the behavior with focused `ui/src/ui/views/cron.test.ts` tests.
