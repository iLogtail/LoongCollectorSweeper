## Summary

Keep open. Current main has useful rate-limit cooldowns, auth-profile rotation, model fallback, and bounded restart-recovery retries, but it still does not implement the requested session-level circuit breaker that suppresses synthetic auto-resume injection after consecutive model errors. The issue is recent, concrete, non-maintainer-authored, and the related open reports/PRs show this remains unresolved rather than safely closeable.

## Best Possible Solution

Keep #52384 open unless maintainers explicitly consolidate it into a canonical session-level model-error breaker tracker. The best implementation should live in the agent/session runtime: persist consecutive model-error state per session, gate main-session and subagent synthetic resume injection before creating another turn, classify 429/RESOURCE_EXHAUSTED through structured model-error metadata, fall back or back off before retrying the same provider, cap synthetic retries, reset the breaker on the first successful model response, and add tests for main restart recovery, subagent orphan recovery, and embedded-runner rate-limit/fallback paths.
