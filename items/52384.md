## Summary

Keep open. Current main has useful rate-limit/fallback handling and bounded restart-recovery retries, but I did not find the requested session-level circuit breaker for consecutive model errors suppressing synthetic auto-resume injection. The issue is recent, concrete, non-maintainer-authored, and has no protected-label override, so conservative cleanup should not close it.

## Evidence

- **Issue scope:** The provided discussion specifically reports a 2026-03-22 Gemini 429 cascade where `429 -> inject 'Continue where you left off' -> 429 -> inject` repeated 8+ times, and asks for suppressing/capping auto-injected resume attempts after consecutive model errors plus 429 fallback/backoff behavior.
- **Main-session recovery injects a synthetic resume message but does not track model-error streaks:** `resumeMainSession` calls the gateway with a synthetic resume message, then clears `abortedLastRun` after the gateway call succeeds.
