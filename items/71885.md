## Summary

Current main does not contain this PR's fix. The SQLite task store still binds `record.task` directly, and an in-memory `node:sqlite` check confirms object values throw during binding, so this focused PR remains potentially useful maintainer work rather than cleanup-close material.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is either to merge a narrow SQLite serialization regression fix with the added test, or to adjust it so task creation/runtime boundaries reject or normalize non-string values before persistence while still covering the SQLite binding failure.
