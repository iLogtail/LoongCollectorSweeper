## Summary

Keep #71885 open. Current main still binds `record.task` directly into the SQLite named parameter map, so the proposed serialization is not already implemented. The PR is a focused, recent bug-fix candidate with regression coverage, but maintainers should verify whether structured `task` values are truly runtime-reachable because the public task registry contracts currently type `task` as `string`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge the narrow SQLite-boundary serialization with the included regression test after confirming a real runtime path can pass structured task values, or ask the contributor to document that path and consider updating the `TaskRecord.task`/detached task contracts if structured values are intentionally supported.
