## Summary

Keep open. Current `main` does not implement `cron.defaults` for model, delivery, or `contextTokens`: the cron config schema has no `defaults` block, cron payload/schema still only supports per-job model overrides, isolated cron jobs still default missing delivery to `announce`, and cron model selection still falls back to the agent/default model when no per-job override is present. The issue is recent and concrete, so there is not enough evidence to close it.

## Evidence

- **Cron config has no `cron.defaults` surface:** `CronConfig` only defines `enabled`, `store`, `maxConcurrentRuns`, `retry`, `webhook`, `webhookToken`, `sessionRetention`, `runLog`, `failureAlert`, and `failureDestination`; there is no `defaults`, model, delivery, or `contextTokens` field under `cron`.
  - file: [src/config/types.cron.ts:30](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/types.cron.ts#L30)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Validated config schema matches the missing surface:** The Zod schema for `cron` accepts only the same fields and is `.strict()`, so `cron.defaults` is not accepted on current `main`.
  - file: [src/config/zod-schema.ts:568](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/config/zod-schema.ts#L568)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Cron payload schema still lacks `contextTokens`:** `agentTurn` cron payload accepts `model`, `fallbacks`, `thinking`, `timeoutSeconds`, `allowUnsafeExternalContent`, `lightContext`, and `toolsAllow`, but no `contextTokens`; delivery is a separate job field, not a global default block.
  - file: [src/gateway/protocol/schema/cron.ts:4](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/gateway/protocol/schema/cron.ts#L4)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Current delivery default is still hard-coded to `announce`:** Cron normalization sets `delivery = { mode: "announce" }` for isolated `agentTurn` jobs when no delivery is provided, and the corresponding test asserts that behavior.
  - file: [src/cron/normalize.ts:631](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/cron/normalize.ts#L631)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Current model precedence still ends at agent/default selection:** Cron model selection checks Gmail override, then per-job payload `model`, then stored session override, then falls back to agent/default model selection; docs describe the same precedence and do not mention any `cron.defaults` layer.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/c65aa1d2a6e573d49619c4dd0698a76010476e7d/src/cron/isolated-agent/model-selection.ts#L104)
  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
