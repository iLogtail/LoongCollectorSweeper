## Summary

Keep open. Current main still does not warn on legacy `CLAWDBOT_*`/`MOLTBOT_*` environment variables: the gateway startup path does not contain a legacy-env preflight, credential/auth resolution only reads `OPENCLAW_*` gateway env names, and existing tests explicitly lock in ignoring at least `CLAWDBOT_GATEWAY_PORT`. The related fix PR #53667 is still open/unmerged in the provided context, so the bug is tracked but not shipped on main.

## Best Possible Solution

Keep #53482 open until a fix lands on main. The best product path is a small gateway startup/doctor diagnostic that scans for `CLAWDBOT_*` and `MOLTBOT_*`, emits a clear migration warning or augments auth/channel missing-token errors, and adds focused tests for warning emission, deduplication, and no-warning behavior for valid `OPENCLAW_*` env vars. PR #53667 appears to be the existing candidate implementation to review or supersede.
