## Summary

Keep open. Current `main` has not implemented this change, and the PR as proposed is not safe to auto-land or auto-close: bootstrap reads still use the passed workspace dir itself as the trust boundary, while the PR discussion already identified that widening to `DEFAULT_AGENT_WORKSPACE_DIR` would incorrectly trust files outside a configured nested workspace.

## Evidence

- **Current main still anchors bootstrap reads to the passed workspace dir:** `loadWorkspaceBootstrapFiles()` resolves `dir` and passes that same `resolvedDir` to `readWorkspaceFileWithGuards()` as `workspaceDir`, so symlinks are only accepted if they stay within that exact directory boundary.
  - file: [src/agents/workspace.ts:602](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/workspace.ts#L602)
  - command: `nl -ba src/agents/workspace.ts | sed -n '600,745p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Extra bootstrap files use the same narrow boundary on main:** `loadExtraBootstrapFilesWithDiagnostics()` also passes `resolvedDir` straight through as the boundary root, so a nested workspace cannot symlink bootstrap files back to a parent workspace root today.
  - file: [src/agents/workspace.ts:695](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/workspace.ts#L695)
  - command: `nl -ba src/agents/workspace.ts | sed -n '600,745p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Call sites pass workspace roots, not agent state dirs:** Bootstrap loading is called with `workspaceDir` from runtime context (`resolveBootstrapFilesForRun`) and from the bundled extra-files hook, confirming the current contract is 'current workspace is the boundary'.
  - file: [src/agents/bootstrap-files.ts:217](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/bootstrap-files.ts#L217)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **OpenClaw supports configurable workspace roots:** `resolveAgentWorkspaceDir()` resolves per-agent `workspace` and `agents.defaults.workspace`, and docs describe `agents.defaults.workspace` as the agent workspace root. That means a fix cannot safely hardcode `DEFAULT_AGENT_WORKSPACE_DIR` as the widened boundary for every case.
  - file: [src/agents/agent-scope-config.ts:152](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/agent-scope-config.ts#L152)

- **PR discussion already surfaced the correctness hole:** The PR review thread flagged that widening to `DEFAULT_AGENT_WORKSPACE_DIR` would let nested configured workspaces trust files from outside their configured root; the author replied that they chose `DEFAULT_AGENT_WORKSPACE_DIR` only to keep the patch minimal. That makes this PR useful discussion, but not strong evidence for closure.
