## Summary

Keep open. The issue discussion links PR #65511 as the intended OpenClaw fix set, but that PR was closed unmerged. Current main does include several adjacent mitigations for auth-profile timeout cooldowns, deferred context maintenance, QA gateway cleanup/readiness, and Telegram polling liveness; however, the central requested phase-aware timeout attribution is not verifiable on main. `agent.wait` and lifecycle snapshots still expose generic timeout status without queue/preflight/provider/post_turn/gateway_draining phase data.

## Best Possible Solution

Keep #65504 open but narrow it to the remaining OpenClaw-owned phase-aware timeout attribution contract for embedded runs and `agent.wait`. The comments correctly point at #65511 and the external lossless-claw PR #416 as prior attempted fixes; current main already covers several mitigations, but maintainers should either implement/verify queue, preflight, provider, post_turn, and gateway_draining attribution on the gateway-visible run state, or open a narrower canonical follow-up before closing this umbrella.
