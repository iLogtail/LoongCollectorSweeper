## Summary

Keep open. Current main still routes Linux gateway service status, runtime reads, restart, and update restart scripts through systemd user units only, while the public gateway docs describe supported system-scoped units under /etc/systemd/system. The PR targets a real gap, but the provided review discussion identifies merge-blocking correctness issues that need a revised patch rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or a revised implementation. The best fix is a shared systemd service-scope resolver that detects a matching /etc/systemd/system/<unit>.service before falling back to the user unit, wires that resolver into status/runtime reads/restart and update restart generation, uses direct systemctl for read-only operations and root callers, uses sudo -n only where a non-root mutating system action needs it, handles system-unit uninstall/removal errors accurately, and adds focused tests for user units, system units, custom OPENCLAW_SYSTEMD_UNIT, restart helper generation, and existing user-scope mock behavior.
