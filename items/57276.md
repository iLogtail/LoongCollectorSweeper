## Summary

Keep open. Current `main` still assumes user-scoped systemd units for Linux gateway status/restart paths, while the docs explicitly describe supported system-scoped installs under `/etc/systemd/system`. This PR addresses a real gap, and the review discussion raises concrete correctness issues that need maintainer follow-up rather than auto-close cleanup.

## Evidence

- **Current main resolves Linux gateway units only in the user systemd directory:** `resolveSystemdUnitPathForName` hard-codes `$HOME/.config/systemd/user/<name>.service`; there is no parallel resolver for `/etc/systemd/system` in current `main`.
