## Summary

Keep open. Current main still resolves the macOS desktop browser-proxy local gateway auth path as plain strings only, so structured SecretRef values for `gateway.auth.token` or `gateway.auth.password` still drop to nil before `MacNodeBrowserProxy` can send auth. The PR is active, targets the implicated code path, and maintainer discussion treats the root cause as real while asking for merge-shape and regression-test tightening.

## Best Possible Solution

Keep #69494 open and review or land it, or an equivalent patch, after final macOS validation. The best implementation path is the current PR direction: add an app-side SecretInput resolver seam for local `gateway.auth.token` and `gateway.auth.password`, keep remote auth behavior unchanged unless product-approved, avoid blocking actor/main endpoint refresh on keychain lookup, and retain regression coverage for the actual `localConfig()` path used by `MacNodeBrowserProxy`.
