## Summary

Keep open. Current main still resolves macOS desktop local gateway token/password only from string values, while the browser proxy depends on those resolved values for its auth headers. The PR is a focused, recent bug fix with maintainer-confirmed root cause and active author follow-up, so it is not a conservative cleanup close candidate.

## Best Possible Solution

Keep PR #69494 open for maintainer review. The best path is to finish reviewing the current branch shape and land a tested macOS-side resolver for local `gateway.auth.token` and `gateway.auth.password` SecretRefs, preserving remote string-only behavior unless explicitly changed.
