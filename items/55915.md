## Summary

Keep open. Current `main` does not implement the requested cloud relay feature. OpenClaw currently supports remote mobile access through direct secure Gateway endpoints (`gateway.remote.url` or Tailscale Serve/Funnel) and has a separate APNs push relay for official iOS wake notifications, but there is no `cloud` config surface, no `openclaw login`/cloud-status CLI, and no gateway registration/heartbeat/WebSocket relay path.

## Risks / Open Questions

- The issue is coherent and plausibly valuable, but it is a large security-sensitive feature: OAuth token storage, gateway presence/heartbeat, and authenticated relay transport would all need new core surfaces.
- The existing iOS APNs relay could be mistaken for prior implementation, but the code/docs show it only handles push delivery, not remote WebSocket access to the Gateway.
