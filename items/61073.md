## Summary

Keep PR #61073 open. Current main partially localizes the Instances screen, but it still leaves hardcoded fallback/metadata strings in the Instances view and still has the empty-reason placeholder path that the PR review caught. The PR is stale against the current generated-locale workflow, but the underlying work is still useful and not fully implemented on main.

## Best Possible Solution

Keep the PR open and rework it against current main: add the missing Instances fallback/scope keys in `ui/src/i18n/locales/en.ts`, regenerate all shipped locale bundles through the Control UI i18n pipeline, update `ui/src/ui/views/instances.ts` so mode/host/scopes/reason all go through locale-backed strings without the empty-placeholder bug, and validate with the UI i18n test plus the normal UI build lane.
