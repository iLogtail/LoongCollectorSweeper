## Summary

Keep #43260 open. Current main has agent-level, cron-level, and pre-turn plugin model routing, but it still lacks a `model`/`preferredModel` field in `SKILL.md` frontmatter and has no skill-invocation-aware temporary model switch contract. Related #11826 and #61391 address adjacent routing paths, not this per-skill frontmatter request.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is either a first-class `SKILL.md`/plugin-skill metadata contract with model/fallback fields and runtime enforcement when a skill is invoked or read, or a narrower plugin API that exposes skill-selection/invocation context to model-routing hooks. Current `before_model_resolve` and cron `payload.model` support should be treated as adjacent infrastructure, not a substitute for the requested per-skill contract.
