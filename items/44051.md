## Summary

Keep open. Current main still applies the realpath containment check uniformly to managed skills, so a symlink/junction-backed skill under `~/.openclaw/skills` can still be rejected before loading. The related fix PR #59219 remains open rather than shipped on main.

## Best Possible Solution

Keep #44051 open until #59219, or an equivalent narrower patch, lands. The best fix is to scope root-realpath containment by skill source so workspace/extra/project and bundled roots stay fail-closed, while managed/personal roots can load symlink-backed skill directories without allowing `SKILL.md` to escape the individual skill root. Add a managed symlink regression test and then close the issue from the merged PR.
