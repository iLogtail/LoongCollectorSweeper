## Summary

Keep #69943 open. Current main still persists raw session-memory transcript text into memory files, and the daily-memory startup path can still load those files on bare /new or /reset. The linked fix PR #69961 is a plausible canonical implementation path, but it is still open and not merged, so the original bug report should remain open until the fix lands or maintainers choose a different solution.

## Best Possible Solution

Keep #69943 open as the canonical bug report until PR #69961 or an equivalent maintainer-approved fix lands. The likely code path is a small session-memory-owned sanitizer applied before transcript text is persisted, with regression tests covering `<|im_end|>`, `<|endoftext|>`, raw `<tool_call>` XML, orphaned role markers, and exact NO_REPLY-only turns.
