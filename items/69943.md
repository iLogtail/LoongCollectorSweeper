## Summary

Keep #69943 open. Current main still has the reported raw `session-memory` persistence path: transcript text is extracted and appended without sanitization, then written verbatim under `## Conversation Summary`. The linked PR #69961 is an open, unmerged implementation path for this issue, while #69943 remains the canonical bug report rather than a duplicate to close now.

## Best Possible Solution

Keep #69943 open as the canonical bug. Land or revise PR #69961, or an equivalent fix, so `session-memory` sanitizes both the transcript read path and final write path, adds regression coverage for leaked chat-template tokens, raw `<tool_call>` XML, orphaned role markers, and `NO_REPLY`, then close the issue through the merged PR.
