## Summary

Keep open. Current main still persists session-memory transcript content without sanitizing chat-template tokens, raw tool-call XML, role-marker leakage, or NO_REPLY-style malformed turns. Related PR #69961 is an active fix attempt, but it is not present on main, and the duplicate #71031 being closed does not supersede this canonical report.

## Best Possible Solution

Keep #69943 open as the canonical bug. Land #69961 or an equivalent core fix that sanitizes the session-memory transcript/write path, adds focused tests for chat-template tokens, raw `<tool_call>` XML, orphaned role markers, and NO_REPLY-style turns, and documents operator cleanup for already-poisoned memory files.
