## Summary

Current `main` already exposes supported solutions for the reported gap: one-Gateway multi-agent orchestration with `sessions_send`/`sessions_spawn`, sandbox persistence options via Docker bind mounts or OpenShell mirror mode, and Discord bot-message handling via `channels.discord.allowBots` / `"mentions"`.

## Close Comment

Closing from a Codex review because current `main` already has supported paths for this use case. OpenClaw now documents one-Gateway multi-agent orchestration as the recommended setup (`docs/gateway/multiple-gateways.md:11`, `docs/concepts/multi-agent.md:8`) and ships built-in cross-session coordination via `sessions_send` / `sessions_spawn` (`docs/concepts/session-tool.md:19`, `docs/tools/multi-agent-sandbox-tools.md:292`). For sandbox persistence, Docker bind mounts are supported via `agents.defaults.sandbox.docker.binds` (`docs/gateway/sandboxing.md:72`, `src/config/zod-schema.agent-runtime.ts:140`), and OpenShell `mirror` mode syncs sandbox writes back to the local workspace (`docs/gateway/sandboxing.md:190`). Discord bot-to-bot handling is also configurable now through `channels.discord.allowBots` / `"mentions"` (`docs/channels/discord.md:1115`, `src/config/types.discord.ts:242`). Given those existing supported mechanisms on current `main`, there is no separate cleanup item left here.
