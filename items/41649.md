## Summary

Current main does not implement this PR's model-change transcript archive/handoff flow. Model-related config reloads only restart the heartbeat, there is no session-reset action in the reload plan/handler, and the reply path has no model-handoff prompt injection. Session transcript archiving still happens only on normal session rollover/reset paths, not on config-driven model changes.
