## Summary

Keep PR #41649 open. Current main has partially improved stale model selection by using explicit session overrides rather than runtime `model`/`modelProvider` for reply routing, and it has provider-specific replay sanitization for model route switches. But it does not implement this PR’s core behavior: detecting default-model config changes, rotating/archiving affected session transcripts, creating one-shot handoff files from recent user messages, or injecting that handoff on the next turn. The related #41907 was closed unmerged in the provided context and only tracked the narrower reset-session-model-fields fix, so it is not a shipped superseder for this PR’s remaining work.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but require a refreshed implementation against current main. The best path is a narrow, tested model-change handoff design that integrates with both hot reload and restart paths, preserves all fresh per-session handoffs instead of a global five-file cap, consumes handoff context only when a model run will actually proceed, and documents how this differs from the existing runtime-field and replay-history sanitization behavior.
