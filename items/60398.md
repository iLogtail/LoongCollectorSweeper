## Summary

Keep open. #60398 reports a clear, recent macOS LaunchAgent install failure for external-APFS home directories, and current main still writes and bootstraps the gateway plist from the resolved HOME path with no boot-volume `/Users/<username>/Library/LaunchAgents` fallback or APFS/volume detection.

## Best Possible Solution

Keep #60398 open. The best fix is a macOS-specific LaunchAgent path resolver that detects when the user's HOME/`~/Library/LaunchAgents` is not bootstrap-safe for launchd and writes/bootstraps the OpenClaw gateway plist from the boot-volume `/Users/<username>/Library/LaunchAgents` location instead. The same resolver should be reused by install, repair/start/restart, update restart helper, restart handoff, status/inspect cleanup paths, and covered with unit tests for external-home fallback plus docs/troubleshooting notes.
