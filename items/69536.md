## Summary

Keep #69536 open. The related #46687 cleanup covered startup-context loading, but current main still preserves the reported dreaming/backfill mismatch: the bundled session-memory hook writes `memory/YYYY-MM-DD-slug.md`, while the dreaming daily ingestion, gateway dream-diary backfill, CLI `rem-backfill`, and short-term promotion path classifier still only accept date-only `YYYY-MM-DD.md` daily files or the separate `.dreams/session-corpus` path.

## Best Possible Solution

Keep the issue open and fix the product boundary directly: either change `session-memory` to append to canonical `memory/YYYY-MM-DD.md`, or introduce a shared memory-file classifier/date extractor used by dreaming daily ingestion, gateway backfill, CLI `rem-backfill`, and short-term promotion so slugged session-memory files are ingested consistently. Add regression tests for `memory/YYYY-MM-DD-slug.md` in the CLI/gateway dreaming paths and update the hook/memory/dreaming docs to describe one coherent convention.
