## Summary

Keep open. Current main still has the reported dreaming/backfill mismatch: `session-memory` writes `memory/YYYY-MM-DD-slug.md`, while live dreaming daily ingestion, grounded `memory rem-backfill`, and short-term promotion path validation still recognize bare `YYYY-MM-DD.md` daily-note paths. Startup context now reads slugged files, which explains related #46687, but that is only a startup mitigation and does not make slugged hook files participate in the dreaming promotion path.

## Best Possible Solution

Keep the issue open and resolve the session-memory/memory-core contract. A conservative fix would either make `session-memory` append structured entries to canonical `memory/YYYY-MM-DD.md`, or broaden memory-core daily/backfill discovery to include date-prefixed slugged files while preserving deterministic ordering, source paths, dedupe state, and exclusions for generated dreaming artifacts. Add regression coverage for live light-phase ingestion and `memory rem-backfill --path ./memory --stage-short-term` with a `YYYY-MM-DD-slug.md` input, then update memory and hook docs to state the contract clearly.
