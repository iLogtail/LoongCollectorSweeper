## Summary

Keep open. Current main still sends Ollama `options.num_ctx` from `model.contextWindow` and the OpenAI-compatible Ollama wrapper also ignores `model.params.num_ctx`; provider model config schema/runtime types do not accept or preserve `models.providers.<id>.models[].params`. The PR is a focused bug fix for #44550/#52206 with useful reproduction detail and addressed review feedback, not an obsolete duplicate of the earlier closed unmerged #44678.

## Best Possible Solution

Keep #69464 open and review it as the active candidate fix for honoring Ollama `params.num_ctx`. The best implementation path is the one the PR is taking: add `params` to provider model config/runtime types, merge provider-model params over agent-default params onto the resolved model, have both native and OpenAI-compatible Ollama payload builders prefer a valid positive `params.num_ctx`, and keep regression coverage for precedence and invalid values.
