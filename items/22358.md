## Summary

Keep #22358 open. Current main has subagent lifecycle hooks and delivery-status plumbing, but not the requested post-subagent-completion, pre-announcement plugin hook that can inspect child output, observe delivery success/failure, and enrich or suppress the announcement.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds this exact scope into a canonical lifecycle/delivery design. The contract should include child/session/run metadata, completion output, outcome/timing/usage where available, delivery path/status/error visibility, and explicit return semantics for announcement enrichment or suppression, with docs and regression tests around ordering.
