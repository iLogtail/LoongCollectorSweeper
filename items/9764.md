## Summary

Keep #9764 open. Current main still only accepts/resolves Google Chat service-account credentials, uses the `chat.bot` auth path for Chat API calls, and lacks the requested user OAuth config, secret contract, token creation, and token selection for reactions, attachment uploads, reaction typing indicators, and proactive DM/email-alias lookup. The existing maintainer-side Codex comment also explicitly kept this open, and the external Google Chat file-upload PR does not implement bundled OpenClaw support.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict user OAuth credential model with redaction/SecretRef support and docs; resolve refresh-token credentials safely; construct a user OAuth client with the required Google Chat scopes; choose that token for reactions, attachment uploads, `typingIndicator="reaction"`, and proactive DM/email-alias lookup; preserve the existing service-account path for webhook verification and bot-level operations; and warn/fallback cleanly when user OAuth is absent.
