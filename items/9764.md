## Summary

Keep open. Current main still lacks Google Chat user OAuth config, schema, credential resolution, auth scopes, and token selection for the requested reaction, media upload, and proactive DM paths. The latest issue discussion also already contains a maintainer/Codex review explicitly keeping this open.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin needs an explicit user OAuth credential model, strict schema and docs, safe secret handling, token selection for reactions/uploads/proactive DMs, preserved service-account behavior for bot/webhook operations, and tests covering both configured OAuth and graceful fallback when user OAuth is absent.
