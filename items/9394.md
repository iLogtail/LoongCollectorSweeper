## Summary

Keep #9394 open. Current main only partially covers the request: BlueBubbles has a limited Private API `sendWithEffect` path for classic effects, but rich text attributes, text sizing, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter animations are not implemented or documented. The legacy `imsg` iMessage path remains plain text/media/reply delivery.

## Best Possible Solution

Keep #9394 open for maintainer product direction. The likely best path is a typed BlueBubbles/iMessage rich-text and text-effect send surface, wired through message actions and outbound delivery with tests and docs; legacy imsg should be documented as unsupported for those attributes if upstream imsg cannot carry them.
