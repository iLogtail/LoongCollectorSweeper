## Summary

Keep #9394 open. Current main still only partially covers iMessage effects through BlueBubbles `sendWithEffect`, while the legacy `imsg` channel remains plain-text/media/reply oriented. Rich text attributes, text sizing, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects are not implemented or documented, and the existing issue discussion already includes a maintainer-side Codex comment keeping this open for those same gaps.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely path is a typed BlueBubbles/iMessage rich-text and text-effect send surface wired through message actions and outbound delivery, with tests and docs; legacy `imsg` should either gain an upstream-supported capability or be explicitly documented as unsupported for those attributes.
