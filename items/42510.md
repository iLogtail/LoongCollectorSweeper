## Summary

Keep #42510 open. Current main still has Google Chat thread propagation paths that can bypass `replyToMode: "off"`: the default typing placeholder is sent into the inbound thread before reply filtering, the first text chunk edits that placeholder into the visible reply, and the attached outbound adapter still turns `threadId ?? replyToId` into a threaded Google Chat send. The related attempted fixes #42705, #43076, and #43112 were closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep the issue open until a Google Chat-specific fix lands. The best path is to suppress only automatic Google Chat thread propagation for `replyToMode: "off"` in the monitor/typing and routed outbound paths, while preserving explicit reply tags, then add focused Google Chat tests for typing placeholders, text/media replies, and outbound `threadId` suppression.
