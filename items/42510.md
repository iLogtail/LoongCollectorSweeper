## Summary

Current main does not have a high-confidence fix for #42510. Even though shared reply payload filtering can strip `replyToId` for `replyToMode: "off"`, Google Chat still sends the default typing placeholder into the inbound thread and then edits that threaded placeholder into the final text reply. The outbound adapter also still turns any `threadId`/`replyToId` that reaches it into a Google Chat thread send without checking `replyToMode`. Related PRs #42705, #43076, and #43112 were closed unmerged in the provided GitHub context, so this should stay open for maintainer review/fix.
