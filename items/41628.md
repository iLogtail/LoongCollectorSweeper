## Summary

Keep open. Current main already implements the safer exec/plugin approval ID prefix lookup, including the exact-resolved-ID guard flagged in PR review, but the PR’s second user-facing feature, replying to an approval message with `/approve <decision>` and extracting the ID from `ReplyToBody`, is still missing on main. The open PR is not obsolete; it should be rebased/narrowed and updated to address the review comments.

## Best Possible Solution

Keep the item open, but ask the contributor to rebase onto current main, drop or adapt the already-shipped prefix lookup changes, and focus the remaining patch on reply-to-message approval: allow `/approve <decision>` only when a replied approval message supplies an ID, extract the full trimmed `ID:` line from `ctx.ReplyToBody`, update the forwarded hint text, and add focused parser/handler tests for success, missing reply body, whitespace-containing IDs, ambiguous prefixes, and authorization behavior.
