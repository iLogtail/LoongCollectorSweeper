## Summary

Keep #43300 open. Current main still rejects loopback-source trusted-proxy connections, and the local credential path still treats trusted-proxy as ineligible for password fallback. The related fixes/RFC (#63379, #64122, #69066) are open and show the problem is still active rather than implemented on main.
