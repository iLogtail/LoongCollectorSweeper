## Summary

Keep #70643 open. Current main has atomic temp-file-and-rename writes plus validation/recovery, but it does not serialize OpenClaw-owned config writes with a shared advisory lock or otherwise linearize read-modify-write across processes. The related fix PR #70655 was closed unmerged, so the central requested prevention is still unshipped.

## Best Possible Solution

Keep the issue open and fix it in core config I/O. The best path is to centralize OpenClaw-owned `openclaw.json` mutation under the existing advisory `withFileLock` primitive, hold that lock across read/merge/validate/backup/write, re-read or conflict-check under the lock, cover gateway `config.set`/`config.patch`, CLI config/model/doctor writes, startup auto-enable writes, and sync migration/recovery paths, then add a regression test that exercises concurrent writers and proves the final file remains valid JSON with linearized changes.
