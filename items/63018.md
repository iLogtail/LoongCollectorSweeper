## Summary

Current `main` already fires `agent_end` centrally for embedded/channel runs and initializes the global hook runner for runtime-loaded plugin registries, so Feishu webhook sessions no longer depend on `extensions/feishu` registering `agent_end` directly.

## Evidence

- **Embedded runner fires `agent_end` for completed runs:** `runEmbeddedPiAgent` calls `hookRunner.runAgentEnd(...)` after the reply attempt completes, with channel/session context attached.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2684](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/agents/pi-embedded-runner/run/attempt.ts#L2684)

  - file: [src/agents/pi-embedded-runner/run.ts:309](https://github.com/openclaw/openclaw/blob/7bd74758c5f54c546b52708359a25c08cae74ad7/src/agents/pi-embedded-runner/run.ts#L309)
