## Summary

Keep #42999 open. Current main still implements the context-window guard with fixed 16,000/32,000 token thresholds, tests assert those constants, and runtime setup passes those fixed values into the guard. The related docs-only PRs were closed unmerged, and the later metabolism PR was closed as ClawHub/plugin scope while explicitly leaving built-in dynamic budget enforcement to focused issues like #42999.

## Best Possible Solution

Keep #42999 open as the focused core tracker for dynamic context-window guard thresholds. If maintainers approve the behavior, implement it in `src/agents/context-window-guard.ts` using the resolved effective context budget, update `src/agents/pi-embedded-runner/run/setup.ts` messaging/tests, and add explicit config schema/docs only if ratio overrides become public API. If broader budget controls in #58826 become the accepted canonical design, link that issue and close #42999 as superseded then.
