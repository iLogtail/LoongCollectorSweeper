## Summary

Keep open. Current main still implements the context-window guard with fixed 16k hard-min and 32k warning thresholds, and tests/docs explicitly assert that behavior. The requested dynamic percentage-based budget is not implemented. The linked PR #43103 was closed unmerged, and the related #43208 context is an open docs/strategy item, not a canonical merged implementation.
