## Summary

Current main still matches the reported bug. `detectAndLoadPromptImages()` returns `existingImages` unsanitized when the prompt has no image refs, and the OpenAI-compatible streaming lifecycle `error` path finalizes the SSE stream without writing an error chunk. The linked PR #62070 is useful prior work, but it remains open/unmerged, so this issue should stay open rather than be closed as implemented or superseded.

## Evidence
