## Summary

Keep #59913 open. Current main still has the core image-sanitization gap: OpenAI-compatible `image_url` parts are converted to `ImageContent` and passed into the embedded runner as `existingImages`, but `detectAndLoadPromptImages()` returns those images before `sanitizeImagesWithLog()` whenever the prompt text has no detected image references. The streaming path has improved since the issue text, but a lifecycle `error` event can still request finalization and close the SSE stream before the async catch writes error content. The linked fix PR #62070 is directly relevant, but it is open and unmerged, so the issue remains a useful canonical bug tracker.

## Best Possible Solution

Keep #59913 open until the fix lands. The best code path is the one proposed by #62070 or an equivalent maintainer-owned patch: sanitize `existingImages` on the no-ref early-return path in `detectAndLoadPromptImages()`, add a regression test for gateway-provided images with no prompt refs, and make the OpenAI compat SSE handler surface lifecycle-error failures before `[DONE]` with a targeted streaming regression test.
