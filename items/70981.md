## Summary

Keep #70981 open. Current main still has the gateway/session-row mismatch this PR targets: `buildGatewaySessionRow` receives `cfg` but still populates row `thinkingDefault` with the model-only resolver, while the Control UI renders the dropdown label from that row field. The provided discussion also says #71033 was closed as the duplicate while #70981 remained the intended landing path. #71779 overlaps and may be a broader alternative, but it is open and not a merged or clearly maintainer-selected superseder.

## Best Possible Solution

Keep #70981 open for maintainer review as a focused fix for #70967. The best path is to land or adapt the config-aware session-row behavior with its regression test, then decide whether to fold in the broader per-agent/defaults parity from #71779 before closing the linked issue and duplicate PRs.
