## Summary

本 Issue 请求为采集任务（如 File2Kafka）增加运行时暂停与恢复能力，以应对流量高峰期的负载调控。经核查当前 main 分支，Pipeline 生命周期仍主要由配置加载/卸载驱动，缺乏独立的 Pause/Resume 控制面 API 或动态配置标记来支持无损暂停与上下文保留。该需求涉及核心调度、Buffer/Checkpoint 保留策略及状态机改造，属于有效的核心架构增强，尚未实现且不适合通过插件系统旁路解决。

## Best Possible Solution

保持开启。建议维护者或贡献者补充详细设计方案（RFC），明确暂停/恢复的触发入口（如动态配置标记、REST API 或系统信号）、状态机转换边界、Buffer/Checkpoint 保留策略及恢复时的 Offset 衔接逻辑。可评估是否通过扩展现有流控/背压机制间接满足场景，或作为 Pipeline 控制面新特性纳入核心开发路线。
