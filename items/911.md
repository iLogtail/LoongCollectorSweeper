## Summary

本 Issue 请求为 File2Kafka 采集任务增加运行时暂停与恢复能力，用于流量高峰期的负载调控。经核查当前 main 分支代码，核心 Pipeline 生命周期主要由配置管理器驱动（Start/Stop/Reset），尚无独立的 Pause/Resume 控制面 API 或插件状态机支持。当前可通过动态配置变更（如修改 enable 字段或移除/下发配置）间接实现任务启停，但无法满足保留完整运行上下文（如内存 Buffer、Checkpoint 状态）并快速恢复的场景。该需求涉及核心调度与控制面改造，具有实际运维价值，且无现有等效实现或插件替代方案，故保持开启以待进一步设计或路线规划。

## Best Possible Solution

保持开启。建议维护者或贡献者补充详细设计方案，明确暂停/恢复的触发入口（如 REST API、信号或配置标记）、上下文保留策略（Buffer/Checkpoint 冻结）及恢复时的 Offset 衔接逻辑。可评估是否通过扩展现有动态配置机制（如新增 `paused: true` 字段触发内部状态机挂起而非销毁）间接满足，或作为 Pipeline 控制面新特性纳入核心路线规划。
