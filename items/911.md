## Summary

本 Issue 请求为 File2Kafka 采集任务增加运行时暂停与恢复能力，用于流量高峰期的负载调控。经核查当前 main 分支代码，Pipeline 生命周期管理仍由配置驱动（Start/Stop/Reset），缺乏独立的 Pause/Resume 控制面 API 与状态机。该特性涉及核心调度改造，具有明确业务价值，且无现有等效实现或插件替代方案，故保持开启。

## Best Possible Solution

保持开启。建议维护者或贡献者补充详细设计方案，明确暂停/恢复的触发入口（如 REST API、信号或配置标记）、上下文保留策略（Buffer/Checkpoint 冻结）及恢复时的 Offset 衔接逻辑。可评估是否通过扩展现有动态配置机制（如新增 paused 字段触发内部状态机挂起而非销毁）间接满足，或作为 Pipeline 控制面新特性纳入核心路线规划。

## Evidence

- **pipeline_lifecycle_gap:** 当前 Pipeline 生命周期仅包含 Start/Stop/Reset 等状态转换，缺乏 Pause/Resume 接口与状态机实现，无法在保留上下文的前提下挂起任务。
  - file: [core/pipeline/Pipeline.go:1](https://github.com/alibaba/loongcollector/blob/7a4d1b1f4619244bf2cd80125ea859c87b402556/core/pipeline/Pipeline.go#L1)
  - command: `rg -n "func.*Pipeline.*(Start|Stop|Reset|Pause|Resume)" core/pipeline/`
  - sha: [7a4d1b1f4619](https://github.com/alibaba/loongcollector/commit/7a4d1b1f4619244bf2cd80125ea859c87b402556)
- **config_driven_control:** 任务启停完全依赖 ConfigManager 动态下发配置，移除或禁用配置会触发 Pipeline 销毁与重建，不符合暂停保留现场的需求。
  - file: [core/control/ConfigManager.go:1](https://github.com/alibaba/loongcollector/blob/7a4d1b1f4619244bf2cd80125ea859c87b402556/core/control/ConfigManager.go#L1)
