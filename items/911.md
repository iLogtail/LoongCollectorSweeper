## Summary

本 Issue 请求为采集任务（如 File2Kafka）增加运行时暂停与恢复能力，以应对流量高峰期的负载调控。经核查，当前主线架构中 Pipeline 生命周期仍由配置加载/卸载驱动，缺乏独立的 Pause/Resume 状态机与控制面 API。该需求涉及核心调度与上下文保留逻辑改造，属于有效的核心增强请求，尚未实现且不适合插件化，应保持开启以待架构设计。

## Best Possible Solution

保持开启。建议维护者或贡献者补充详细设计方案（RFC），明确暂停/恢复的触发入口（如动态配置标记、REST API 或系统信号）、状态机转换边界、Buffer/Checkpoint 保留策略及恢复时的 Offset 衔接逻辑。可评估是否通过扩展现有流控/背压机制间接满足场景，或作为 Pipeline 控制面新特性纳入核心开发路线。
