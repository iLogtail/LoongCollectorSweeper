## Summary

本 Issue 请求为 File2Kafka 采集任务增加运行时暂停与恢复能力，以应对流量高峰期的负载调控。经核查当前 main 分支代码，核心 Pipeline 生命周期仍由配置管理器驱动（Start/Stop/Reload），尚未实现独立的 Pause/Resume 状态机或控制面 API。动态配置变更虽可间接启停任务，但会重置运行上下文（如内存 Buffer 与 Checkpoint），无法满足保留状态并快速恢复的需求。该功能涉及核心调度与控制面改造，具有明确运维价值，且无现有插件或等效实现可替代，故保持开启以待进一步设计。

## Best Possible Solution

保持开启。建议维护者或核心贡献者评估在 Pipeline 状态机中引入 Pause/Resume 状态的可行性，明确触发入口（如 REST API、信号或配置标记 paused: true）、上下文保留策略（冻结 Buffer/Checkpoint 刷新）及恢复时的 Offset 衔接逻辑。可先输出设计文档（RFC）讨论核心改造范围，或评估是否通过扩展动态配置加载器间接支持软暂停。
