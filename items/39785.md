## Summary

Keep open. Current main still does not implement this PR's requested behavior: provider model merge still overwrites an explicit configured `input` with the implicit catalog value, and the existing unit test still asserts that behavior. The related issue #39690 remains open, and the later related PR #70816 was closed unmerged, so there is no canonical shipped fix to supersede this PR.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-owned canonical fix that preserves source-authored `models.providers.*.models[].input` through `mergeProviderModels` while still letting implicit/catalog multimodal metadata win when the user omitted `input`. Add focused regression coverage in `src/agents/models-config.merge.test.ts` for explicit image input, omitted input with implicit image support, and explicit empty input, plus a source-snapshot/defaulted-config case if the final implementation relies on raw source config.
