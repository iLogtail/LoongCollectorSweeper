## Summary

Keep open. Current `main` does not implement this PR's requested merge behavior: `mergeProviderModels` still overwrites explicit model `input` with `implicitModel.input`. `v2026.4.23` added a downstream Gateway/WebChat mitigation that preserves attachments for text-only model paths by offloading them as `media://inbound/*` refs, but that is not the same as honoring explicit `models.providers.*.models[].input` during provider merge. The review-thread concern about the proposed one-line `"input" in explicitModel` check also remains credible on current `main`, because defaulted configs still materialize missing `input`, `resolveModelsConfigInput` still falls back to the passed config when no source snapshot exists, and `src/agents/context.ts` still eagerly calls `ensureOpenClawModelsJson(loadConfig())` on that path.
