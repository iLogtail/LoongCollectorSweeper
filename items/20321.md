## Summary

Current main still does not implement the requested Telegram `groupPolicy: "members"` behavior. The issue also has a fresh maintainer comment explicitly saying to keep it open. The shared config contract, Telegram schema, runtime access checks, docs, and tests still only support `open`, `disabled`, and `allowlist`; the related implementation PRs are closed unmerged, and the linked `openclaw-chat-only-trigger` plugin is useful prior art but not the requested all-members allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit maintainer/product handling. The likely implementation belongs in the bundled Telegram plugin plus shared config contract: add `groupPolicy: "members"`, define exact all-member verification semantics, wire Telegram Bot API membership/count checks with cache/invalidation behavior, update schema/types/docs/security guidance, and add focused Telegram runtime tests. Treat the closed PRs and the `openclaw-chat-only-trigger` plugin as useful prior art, not as a close basis.
