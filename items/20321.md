## Summary

Keep #20321 open. Current main still does not implement Telegram `groupPolicy: "members"`: the shared config contract, schema, Telegram runtime policy evaluation, tests, and docs only cover `open`, `disabled`, and `allowlist`. The closed PR attempts are useful prior art but unmerged, and the linked `openclaw-chat-only-trigger` plugin is a related workaround rather than the requested all-current-members Telegram allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit Telegram plugin and shared config-contract work. The likely path is to add `groupPolicy: "members"` to the shared type/schema and Telegram docs, define fail-closed all-member verification semantics, implement Telegram Bot API membership/count checks with caching and join/leave invalidation in the bundled Telegram plugin, update security/doctor guidance, and add focused runtime tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art, not as a close basis.
