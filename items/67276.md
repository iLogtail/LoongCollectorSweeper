## Summary

Keep open. Current main still has the verbose-off behavior this PR targets: edit failures are treated as mutating tool errors, but the payload builder omits the underlying mismatch reason unless verbose details are enabled. The PR remains a focused, plausible bug fix with regression coverage, and the bot review concern was addressed in the PR context by preserving raw detail for verbose-on users.

## Best Possible Solution

Keep the PR open for normal maintainer review/CI. The best path is to land the narrow payload summarization change if checks pass, then separately decide whether the linked issue's broader request for an edit-disable configuration belongs in core.
