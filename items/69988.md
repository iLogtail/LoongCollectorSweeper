## Summary

Keep #69988 open. Current main still supports compaction config only under `agents.defaults.compaction`; it does not accept `agents.list[].compaction`, does not include compaction in resolved per-agent config, and embedded Pi compaction paths still read defaults-only settings. The PR is recent, focused, and the provided discussion shows active follow-up to review feedback, so this is not a conservative cleanup-close candidate.

## Best Possible Solution

Keep #69988 open as the current focused implementation path for per-agent compaction overrides. Maintainers should review and land an agent-aware config/schema/resolution change that accepts `agents.list[].compaction`, merges it over `agents.defaults.compaction`, threads the effective config into Pi settings, direct/manual compaction, safeguard runtime setup, and adds schema/runtime tests plus docs. After that lands, close the related duplicate feature reports against the merged PR.
