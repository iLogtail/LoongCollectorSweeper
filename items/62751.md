## Summary

Keep open. Current main implements `session:patch` hook dispatch for the `sessions.patch` RPC, but the reported `/model` and `session_status(model=...)` model-change paths still mutate the session store directly and do not call the same hook pipeline. The issue has no maintainer/protected labels and no related canonical item that supersedes this exact gap.

## Best Possible Solution

Keep the issue open. The best fix is to route explicit session model updates through a shared session patch/hook helper, or add the proposed dedicated model-change hook, then add regression tests proving hook delivery for `/model`, `session_status(model=...)`, and `sessions.patch` with the expected changed-field payload.
