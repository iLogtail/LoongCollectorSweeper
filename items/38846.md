## Summary

Keep #38846 open. Current main has not implemented the proposed Windows argument-validation API or PowerShell-pattern hardening, and the PR diff still has unresolved review blockers that make it unsafe to merge as written rather than obsolete or implemented.

## Best Possible Solution

Keep the PR open for a maintainer or author rework. A safe version should validate only actual cmd.exe injection primitives at the wrapper boundary, preserve normal Windows paths, slash switches, scoped package names, and `=` flags, avoid echoing full argv values in errors, and add focused regression tests for `.cmd`/`.bat` wrappers, npm/pnpm fallback, scoped packages, `=` flags, and null/control-byte handling.
