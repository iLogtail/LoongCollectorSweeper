## Summary

Keep PR #61972 open. The central bug is still present on current main: Gemini web search, Grok web_search, and x_search still build requests from hardcoded vendor endpoints, and the current plugin manifests/config schemas do not expose the PR's proposed baseUrl fields. The PR discussion is useful, including the bot formatting note and the follow-up review about legacy Grok baseUrl fallback; that remaining review feedback should be incorporated rather than closing the PR as obsolete.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best implementation is to add plugin-owned `webSearch.baseUrl` support for Google/Gemini and xAI/Grok, add `tools.web.x_search.baseUrl` or a clearly documented xSearch plugin config path, thread the resolved base URL into Gemini web search, xAI web_search, and x_search request helpers, include baseUrl in cache keys, preserve legacy Grok baseUrl fallback for x_search, and update manifests, generated config metadata, docs, and focused tests.
