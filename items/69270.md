## Summary

Keep #69270 open. Current main still lacks the PR's central compaction/reset invariant fixes: reset and compaction hook contexts omit `messageProvider` on several paths, and zero-token compaction results are still ignored by session accounting. The PR is a plausible focused bug-fix for linked reports #69269, #69286, and #69287, but the provided review discussion also shows at least one remaining resolver edge case to settle before landing.

## Best Possible Solution

Keep #69270 open. The best maintainer path is to finish the shared provider resolver so explicit provider values are preserved when intentional, infer providers only from real channel-scoped session keys, keep the zero-token/cache-cost accounting fixes, and retain focused regression tests for command reset, gateway reset, subscribe/engine compaction hooks, and manual/zero-token compaction bookkeeping. If the PR becomes too broad, split it under the canonical #69269 bug cluster rather than closing it as obsolete.
