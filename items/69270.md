## Summary

Keep PR #69270 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks the PR's core behavior: reset/compaction hook contexts do not consistently carry a normalized messageProvider, subscribe-time compaction hooks omit it entirely, gateway resets omit it, and zero-token compaction bookkeeping is still skipped by the `tokensAfter > 0` guard. The PR is not obsolete or implemented on main; it is active bug-fix work for the open canonical issue #69269 and the related token-accounting reports #69286/#69287. However, the provided review discussion also shows at least one meaningful remaining implementation concern around preserving explicit non-gateway providers, so the best maintainer path is to keep the PR open for review/fixup rather than close it.

## Best Possible Solution

Keep #69270 open as the concrete implementation PR for #69269/#69286/#69287. Before landing, refine the shared hook message-provider resolver so explicit providers are preserved and only inferred providers are constrained to real deliverable channel-scoped session keys, then keep the regression tests for reset hooks, subscribe-time compaction hooks, gateway resets, zero-token compaction accounting, and stale cache/cost cleanup.
