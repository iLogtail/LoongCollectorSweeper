## Summary

Keep PR #69701 open. Current main still has the dual loopback bind path for `127.0.0.1` when `::1` is bindable, and gateway startup consumes that returned host list directly. The linked Windows regression #69674 has concrete reproduction details, and the PR’s latest commit addresses Greptile’s test-quality comment by replacing the redundant Windows passthrough test with a non-Windows guard test. This is a focused core gateway bug fix, not ClawHub scope and not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best path is to apply the Windows-only early return in `resolveGatewayListenHosts`, keep the focused regression tests from the PR, and verify the gateway test lane plus a Windows smoke path for #69674 before merging.
