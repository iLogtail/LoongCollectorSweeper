## Summary

Keep open. Current main has partial runtime improvements for invalid/unknown model handling, but it does not yet satisfy this issue’s proactive startup/doctor validation request for retired provider slugs. In particular, OpenRouter dynamic resolution still accepts arbitrary model IDs, so a retired `openrouter/<provider>/<model>` ref can pass gateway warmup and only fail later at request time.

## Best Possible Solution

Keep the issue open for a focused maintainer fix: add a conservative configured-model validation path that can warn on startup and in `openclaw doctor` when a configured primary/fallback model is absent from an authoritative provider catalog after live/catalog refresh, while preserving OpenRouter forward-compatible custom model support. A good implementation would cover `agents.defaults.model.primary`, fallbacks, and related configured model refs, and add regression tests for an invalid `openrouter/anthropic/...` slug.
