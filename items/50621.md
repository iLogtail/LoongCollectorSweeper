## Summary

Current main still has the reported core behavior: `systemEvent` cron jobs do not get the longer/disabled agent-turn timeout, and `sessionTarget: "main"` plus `wakeMode: "now"` still waits on `runHeartbeatOnce()`. The related PR attempts were closed unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep #50621 open and make a maintainer-owned cron behavior decision. The likely fix belongs in `src/cron/service/timeout-policy.ts` and `src/cron/service/timer.ts`: either make main-session `systemEvent` wake-now delivery truly fire-and-forget after enqueueing/requesting heartbeat, or add an explicit/configurable timeout policy for this path, with regression coverage for long-running `sessionTarget: "main"` system events and docs for any remaining timeout/workaround.
