## Summary

Keep open. Current main has partial hardening for plugin register failures, typing cleanup, and repeated no-progress tool loops, but it does not implement the requested per-plugin failure circuit breaker/auto-disable, hard timeout for plugin tool calls, or typing-TTL-driven session recovery.

## Best Possible Solution

Keep #55253 open and split/implement it as core resilience work: add per-plugin health state with a time-window failure counter and temporary disablement for register/hook/tool failures, add a generic timeout/abort contract for plugin tool and hook execution, and define whether typing TTL should mark the run/session failed, restart/recover the session, or surface a user-visible outage notice. Link the narrower Telegram typing keepalive work (#45759) only for the channel-specific typing breaker overlap; it does not supersede this broader plugin/session hardening request.

## Risks / Open Questions

- A plugin tool or hook that never resolves can still block the agent path because the generic execution paths await plugin promises directly.
- Typing TTL can currently hide a stuck run by stopping the indicator without marking the session failed, restarting, or notifying the user.
- Graceful plugin register failure handling is useful, but repeated failures across reloads/restarts are not tracked or auto-disabled.
