## Summary

Keep #41088 open. Current main still launches managed Chrome with only `Translate,MediaRouter` in `--disable-features`, and the sandbox browser entrypoint has no OptGuide/WasmTts disable-features block. The browser config exposes `headless`, `noSandbox`, and `extraArgs`, but no dedicated model-download disable option. The related PR #41213 attempted exactly this change but is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and land a narrow browser-plugin fix: define a shared list of Chrome ML/model-download feature names, apply it in `buildOpenClawChromeLaunchArgs` for local managed Chrome and in `scripts/sandbox-browser-entrypoint.sh`, add regression coverage for launch args and sandbox args, and document either the new default behavior or a first-class config option if maintainers prefer an opt-out/opt-in switch.
