## Summary

Current main has session switching for active session keys, a Sessions page, reset/delete transcript archiving, and compaction-checkpoint restore. It does not provide the requested Web UI path to list or restore reset/deleted session archives after the plus button creates a new session, so the core feature request remains actionable.

## Best Possible Solution

Keep the issue open. The best core path is a small, explicit Web UI/session API design for archived session recovery: list reset/deleted transcript archives for a session key, preview enough context to identify the old chat, and restore or branch into a new/current session entry without weakening existing session maintenance and retention controls. KeepMyClaw can remain documented as an external backup workaround, but it should not replace a native recovery decision if maintainers want accidental /new recovery in the GUI.
