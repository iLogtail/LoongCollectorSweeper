## Summary

Keep #42646 open. Current main has a SQLite-backed memory search index and adjacent active-memory/dreaming/wiki work, but it does not define the requested typed durable-memory record schema or canonical record model. The durable source of truth is still Markdown memory files, while SQLite stores indexed files/chunks/embeddings for retrieval.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a conservative v1 durable-memory record store, likely in the memory-core/memory-host layer: SQLite tables for canonical memory records plus revisions/entities/links as needed, typed record/status/confidence/validity fields, explicit transition semantics, and docs/tests proving Markdown/embedding indexes no longer serve as the canonical durable-memory record model unless that product direction changes explicitly.
