## Summary

Keep #70002 open. Current main still lacks the fork guard in both docs workflows, and both jobs still consume OPENCLAW_DOCS_SYNC_TOKEN for openclaw/docs operations. The PR is a focused two-line workflow fix matching existing repo guard patterns, with positive discussion and no superseding implementation on main.

## Best Possible Solution

Keep the PR open and have maintainers either merge its two job-level repository guards or land an equivalent fork-safe guard in the two docs workflows. The fix belongs in OpenClaw workflow code because the failing behavior is caused by repository-owned GitHub Actions files, not by external docs setup alone.
