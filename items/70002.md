## Summary

Keep #70002 open. Current main still lacks the proposed fork guard on both docs publish workflows, and both jobs still use OPENCLAW_DOCS_SYNC_TOKEN against the separate openclaw/docs publish repo. The PR is a narrow two-line infra fix that matches existing main-only workflow guard patterns elsewhere in the repo; it is not implemented, duplicated, or obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best product/code path is the PR's narrow job-level `if: github.repository == 'openclaw/openclaw'` guard on both docs publish workflows, preserving upstream behavior while skipping fork runs that cannot have the required docs sync token.
