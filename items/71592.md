## Summary

Keep #71592 open. Current main still advertises `/status` and `/compact`, but the TUI local command handler has no local `status` or `compact` cases and its default branch forwards the raw slash text through `sendMessage`. In `openclaw tui --local`, that send path uses `EmbeddedTuiBackend`, whose run path calls `agentCommandFromIngress`, so the reported local embedded model-run fallthrough remains present. The related items in the provided context are distinct and do not supersede this local TUI execution bug.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Recognized shared commands such as `/status` and `/compact` should either execute through a shared local command adapter or return an explicit local unsupported/system response, but they should not fall through to `sendMessage(raw)` in `opts.local` mode. Add focused regression tests for local `/status` and `/compact`, while preserving gateway-connected forwarding for commands intentionally handled by `chat.send`.
