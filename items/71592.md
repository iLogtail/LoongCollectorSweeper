## Summary

Keep #71592 open. Current main still advertises shared slash commands such as `/status` and `/compact` in the TUI, but the TUI command switch has no `status` or `compact` cases. In local embedded mode the fallback path adds the raw slash text as a user turn and sends it through `EmbeddedTuiBackend.sendChat()`, which starts `agentCommandFromIngress()` directly instead of the gateway `chat.send` command-dispatch path. The existing ClawSweeper comment identified this same gap, and the relevant current-main paths still match it at 21c51bc140295e4286f1adc871a96b9b33c6a8b9.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Known shared commands such as `/status` and `/compact` should either execute through a shared local command adapter or return an explicit local unsupported/system response in `opts.local` mode, but they should not fall through to `sendMessage(raw)`. Add focused local-mode regression tests for both commands and keep gateway-connected forwarding behavior intact.

## Risks / Open Questions

- A broad fix that blocks all unknown slash commands in the TUI could regress gateway-mode forwarding of shared, plugin, or gateway-only commands; the fix should distinguish local embedded mode from gateway-connected mode.
- The regression coverage should assert that local `/status` and `/compact` do not reach `sendChat()` / `agentCommandFromIngress()`, while preserving intentional gateway forwarding for shared commands.
