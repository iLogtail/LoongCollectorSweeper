## Summary

Keep #71592 open. Current main still advertises `/status` and `/compact` in the TUI command surface, but the TUI handler has no local `status` or `compact` cases; unmatched slash commands fall through to `sendMessage(raw)`, which adds a user turn and calls `client.sendChat`. In `openclaw tui --local`, that client is `EmbeddedTuiBackend`, whose `sendChat` path calls `agentCommandFromIngress` and then the local agent/model run path rather than the gateway `chat.send` command dispatcher. The related items in the provided context are adjacent but do not supersede this local embedded TUI execution bug.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Recognized shared commands such as `/status` and `/compact` should either execute through a shared local command adapter or return an explicit local unsupported/system response in `opts.local` mode, but they should not fall through to `sendMessage(raw)`. Add focused regression tests for local `/status` and `/compact` asserting they do not reach `sendChat`/`agentCommandFromIngress`, while preserving gateway-connected forwarding for commands intentionally recovered by `chat.send`.
