## Summary

Keep open. Current main still has the IRC behaviors this PR tries to fix: one-argument NickServ IDENTIFY, silent empty PRIVMSG drops, and immediate quit after transient sends. The PR is useful but not merge-ready as-is because the submitted sendRaw closed-guard conflicts with quit(), matching the existing review comments.

## Best Possible Solution

Keep the PR open and ask for a focused update, or land an equivalent maintainer patch: preserve the IRC reliability fixes, add regression tests for fallback-nick IDENTIFY/reclaim and silent-send behavior, fix quit() so QUIT is sent before the closed guard applies, and tie delayed NickServ reclaim timers to client shutdown.
