## Summary

Keep #56813 open. Current main still preserves the reported OpenShell npm name-collision path: the bundled OpenShell plugin declares `openshell@0.1.0`, that package is the Telegram bridge dependency tree, enabled bundled plugin runtime deps are staged before module load, and the OpenShell command resolver still prefers a bundled `openshell` binary over the standalone CLI on PATH. The reporter's follow-up root-cause comment is consistent with current code and upstream package metadata.

## Best Possible Solution

Keep the issue open and fix the OpenShell plugin boundary: remove or replace the unscoped `openshell@0.1.0` dependency, make the default `command: "openshell"` prefer the standalone CLI on PATH unless an explicitly validated bundled runtime exists, and add a regression test that an unrelated npm package named `openshell` is not selected and does not require `TELEGRAM_BOT_TOKEN`.
