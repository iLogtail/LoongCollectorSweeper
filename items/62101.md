## Summary

Keep PR #62101 open. Current main still has the BlueBubbles bare-number allowlist mismatch, so the linked bug is not implemented or obsolete. However, the PR’s shared `AllowFromEntrySchema` transform is too broad because other bundled channel configs also use that shared schema for numeric sender IDs, so this needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open but request a revision that localizes the fix to BlueBubbles: add the all-digit `+` normalization in `normalizeBlueBubblesHandle` and/or a BlueBubbles-owned allowlist schema/parse transform, with regression coverage for numeric and string `allowFrom` entries. Avoid changing shared `AllowFromEntrySchema` unless every channel using `AllowFromListSchema` is audited and updated for the new contract.
