## Close Comment

Closing from a Codex review: this request is already implemented on current `main`. OpenClaw supports model-specific tool policy overrides through `tools.byProvider`, where the key can be either a provider id or a full `provider/model` id (`src/config/types.tools.ts:311`). The resolver in `src/agents/pi-tools.policy.ts:243` builds the full model key and prefers that exact match before falling back to the provider-wide rule, so a config like `tools.byProvider["ollama/gpt-oss:20b"].deny = ["web_search", "web_fetch", "browser"]` works for the safe-fallback case described here. This is also documented in `docs/gateway/configuration-reference.md:2244` and `docs/gateway/sandbox-vs-tool-policy-vs-elevated.md:68`, and the feature is called out in the `2026.1.12` changelog (`CHANGELOG.md:6420`).
Codex evidence: Reviewed against main [a2221d6b4709](https://github.com/openclaw/openclaw/commit/a2221d6b47094cd34c12025ce809aba02c5e1c1c). Fix evidence: release [2026.1.12](https://github.com/openclaw/openclaw/releases/tag/2026.1.12).
