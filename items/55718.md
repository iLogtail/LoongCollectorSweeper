## Summary

Keep open. Current main already supports manual per-session labels in Control UI, which is why related #60006 could close, but this issue asks for channel account configuration to automatically seed friendly labels for new channel sessions. I found no `displayLabel` field, no account `name`/display label propagation into `session.label`, and no docs/tests covering config-driven session labeling.

## Best Possible Solution

Keep the issue open and implement a generic channel-account session-label contract. The likely maintainer path is to reuse the existing `channels.<channel>.accounts.<id>.name` field or intentionally add a documented `displayLabel` alias, then have the inbound session metadata/store path seed `session.label` or `displayName` for newly created channel sessions when the account label is configured and does not overwrite a user-set label. Add gateway/session tests for Telegram or a generic channel fixture, Feishu schema coverage if `displayLabel` is added, Control UI session-picker assertions, and docs under multi-account channel configuration.
