## Summary

Current main already fixes the PR's central runtime-provider ownership bug: explicit provider/model refs that resolve to no activatable owner now remain an explicit empty plugin scope instead of widening into an unrestricted provider load, and regression coverage exists for that behavior. The real 2026.4.8 memory-lancedb failure discussed on the PR is therefore covered by the shipped main path, while any remaining hook-alias optimization should be a fresh narrow change against the current provider-contract/activation system.

## Best Possible Solution

Close PR #63013 as redundant with the shipped explicit-empty plugin scope implementation on main. Keep the current main behavior; if maintainers still want more targeted hook-alias owner activation, that should be a fresh narrow PR against the current provider-contract/activation code instead of reviving this stale branch with unrelated changes.
