## Summary

Keep open. Current main still routes the Dreaming diary “Open Config” button to `config.openFile`, and the gateway’s Windows opener still builds `Start-Process -LiteralPath ...`, which PowerShell rejects with the same `NamedParameterNotFound` failure shown in the issue. This is a current OpenClaw gateway bug, not stale or already implemented.

## Best Possible Solution

Patch `src/gateway/server-methods/config.ts` so the Windows config opener uses a valid PowerShell contract, likely `Start-Process -FilePath '<escaped configPath>'` or another opener that supports literal paths, and update `src/gateway/server-methods/config.test.ts` with a Windows regression case covering quotes/metacharacters and the absence of `-LiteralPath`.
