## Summary

Keep PR #71063 open. Current main still has the unguarded initial `child.stdin.write()` / `end()` path in `createChildAdapter`, and I found no existing non-writable-stdin regression coverage or alternate fix on main. The PR is a small, focused bug fix with a matching test for a real CLI-backend surface: `google-gemini-cli` shells out to a local `gemini` command that docs require to be on PATH.

## Best Possible Solution

Keep the PR open for maintainer review. The best product/code path is to land a narrow process-supervisor fix that guards the initial stdin write/end path on writability and includes the non-writable-stdin regression test, then run the normal changed-lane validation for `src/process/supervisor/adapters/child.ts` and its tests.
