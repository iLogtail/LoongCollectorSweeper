## Summary

Keep PR #71063 open. Current main still has the unguarded initial stdin write/end path in `createChildAdapter`, there is no matching regression test on main, and the PR is a focused bug-fix attempt against a real CLI-spawn surface. The proposed one-line `writable` guard may still need maintainer review because a read-only Node check showed a missing shell-exec target can emit `EPIPE` after a write that began while stdin was still writable.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix likely belongs in `src/process/supervisor/adapters/child.ts` around the initial stdin write/end path: guard unusable stdin, handle synchronous and asynchronous stdin write/end errors consistently with `ManagedRunStdin`, and add a regression test that simulates the missing-binary/non-writable or EPIPE-emitting stdin case. The current PR is a useful narrow start, but should be validated against the actual shell-wrapper missing-command behavior before landing.
