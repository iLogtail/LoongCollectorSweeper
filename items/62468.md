## Summary

Keep open. Current main has Feishu outbound/directory support, and related reports cover other Feishu delivery failures, but this specific bare username case is still actionable: a target like `jaydenli` is still inferred as a group lookup and can fall through to the same `Unknown target` error instead of resolving a user or giving a Feishu-specific username/prefix hint.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin-owned messaging target layer: add Feishu target inference/resolution that can resolve explicit `user:<name|open_id|user_id>` and, if product-approved, bare username fallback after group lookup misses; improve the hint/error text to tell users when `user:` or `chat:` is required; add regression coverage around `jaydenli`, `user:jaydenli`, open_id, chat_id, and ambiguous group/user names.
