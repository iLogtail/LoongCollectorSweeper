## Summary

Keep open. The underlying bug is still present on current main after the approval flow was refactored into the shared handler: `handlePendingApprovalRequest` expires requests with `no-approval-route` before checking `twoPhase`, so a two-phase exec approval with no approval clients, no originating chat route, and no delivery still fails immediately. The PR needs to be rebased/ported to the current shared approval path rather than closed as implemented.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase it onto current main by moving the two-phase guard into `handlePendingApprovalRequest` in `src/gateway/server-methods/approval-shared.ts`, preserving the existing single-phase fast-fail behavior and adding a focused regression test for two-phase/no-client/no-delivery pending behavior.
