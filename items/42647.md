## Summary

Keep open. Current main has adjacent attribution pieces: path/line citations in memory_search results, a source column in the memory index, short-term promotion markers with a single source span, and a provenance-rich memory-wiki companion layer. It still does not implement the requested durable-memory provenance model: required source refs on every durable memory, multiple supporting sources, retained provenance through merges/edits, or a source-less promotion blocker.

## Best Possible Solution

Keep the issue open and implement a typed provenance contract in the memory-core durable write path: SourceRef records for session/message/file origins, sourceRefs arrays on durable records and promotions, merge/edit lineage, review-surface display, and validation that blocks source-less candidate promotion. Align it with #42646's canonical schema work and #42648's write pipeline; memory-wiki claim evidence can inform the shape but should not be treated as a substitute for core durable-memory provenance.
