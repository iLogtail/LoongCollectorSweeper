## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat/model path instead of formatting and delivering it deterministically. There is a duplicate-delivery guard and a test that drains a consumed cron event, but that happens after `getReplyFromConfig` has already been invoked and does not address repeated watcher-generated cron alerts or OpenAI usage. The related PR #15403 in the issue body was a separate shell environment PR and was closed unmerged, so it does not supersede this request.
