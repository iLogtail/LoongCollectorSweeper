## Summary

Keep #16142 open. Current main still sends main-session cron reminder text through the heartbeat model path, and current tests assert that cron reminders produce a `cron-event` model call. Existing docs explain heartbeat/cron cost mitigations, but they do not implement the requested deterministic string-formatting bypass. The prior Codex/maintainer review comment also explicitly kept this issue open, and the linked #15403 PR is unrelated shell-environment work.

## Best Possible Solution

Keep #16142 open until main-session cron reminders have a deterministic relay/formatting path for simple `systemEvent` reminders that does not call `getReplyFromConfig`, while preserving model-mediated behavior for agentic cron events, exec completions, and cases that need reasoning. The likely change belongs around `executeMainSessionCronJob`, `resolveHeartbeatRunPrompt`, or a small delivery helper, with regression tests proving simple reminders do not call the model and docs explaining when heartbeat `lightContext`/`isolatedSession` remain the right cost mitigations.
