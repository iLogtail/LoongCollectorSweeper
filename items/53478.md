## Summary

Keep open. Current `main` still does not export `normalizeAccountId` or `resolvePreferredOpenClawTmpDir` from `openclaw/plugin-sdk`, but the PR's March patch no longer matches the current SDK architecture: legacy root imports now flow through `root-alias.cjs` and `compat.ts`, and those paths also do not expose the requested helpers. The compatibility concern remains plausible, but this PR is stale/incomplete rather than already fixed.

## Evidence
