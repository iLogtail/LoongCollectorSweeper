## Summary

Keep open for maintainer judgment. The exact PR change is not implemented on current main, and the reported external plugin compatibility problem is plausible because `normalizeAccountId` and `resolvePreferredOpenClawTmpDir` exist on supported subpaths but are not exported from the root `openclaw/plugin-sdk` surface. Current main also deliberately keeps the root runtime surface tiny and docs tell plugin authors to use focused subpaths, so this PR should not be auto-closed as implemented; the right outcome needs a maintainer policy decision between adding a compatibility export and requiring the external plugin to migrate imports.

## Best Possible Solution

Keep the PR open for maintainer triage. The likely best path is either a deliberate compatibility change with contract tests and migration notes, preferably through the existing compatibility/root-alias policy, or a maintainer response asking `@tencent-weixin/openclaw-weixin` to import from `openclaw/plugin-sdk/account-id` and `openclaw/plugin-sdk/temp-path` instead of the deprecated root barrel.
