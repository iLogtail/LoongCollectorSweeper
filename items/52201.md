## Summary

Keep open. Current main still has route bindings that select an agent, not a binding-level workspace override; the configured route binding type and Zod schema have no `workspace` field, and docs still say the matched agent determines the workspace/session store. The linked follow-up PR #61460 is closed/unmerged and, from the provided context, does not provide a current canonical open replacement for this specific workspace-override request.
