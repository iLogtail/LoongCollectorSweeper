## Summary

Keep open. Current `main` does not contain the PR’s central fixes: `agentLogoUrl("")` still returns route-relative `favicon.svg`, the sidebar/login/welcome logo paths still depend on that helper, and Control UI static asset responses still receive the global CSP before serving files. The PR needs a current-main rebase because grouped chat avatars have since moved to `ui/src/ui/chat/chat-avatar.ts`, but the linked regression #55600 is still plausibly actionable in core.

## Best Possible Solution

Keep the PR or linked issue open, but ask for a current-main update: change `agentLogoUrl("")` to produce `/favicon.svg`, scope Control UI CSP so HTML/avatar responses remain protected while static favicon/SVG assets avoid the paint-reference breakage, add/refresh tests around empty `basePath` logo URLs and static favicon CSP, and drop or adapt the stale grouped-render hunk now that chat avatars render through `ui/src/ui/chat/chat-avatar.ts`.
