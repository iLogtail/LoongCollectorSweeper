## Summary

Keep #8209 open. Current main still does not provide the requested native cron lifecycle integration: cron emits an internal started event, but the documented/configured external surfaces remain completion-oriented, and the plugin SDK does not expose a generic cron start/finish subscription that would let this move fully to a plugin. The linked PR #8855 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #8209 open for explicit maintainer product/API direction. The best path is a generic, typed cron lifecycle surface, preferably a plugin hook or runtime event subscription that emits start and finish events for both agentTurn and systemEvent jobs. If core webhooks are chosen instead, keep them generic, schema-validated, SSRF-guarded, documented, and covered by focused tests for start, success, failure, timeout, disabled jobs, and invalid destinations.
