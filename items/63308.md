## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still hard-rejects calls containing both `sessionKey` and `label`. The provided context also shows PR #63377 is an open, unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until a fix is reviewed and merged.

## Best Possible Solution

Keep this issue open and review a concrete fix that either adds strong schema guidance or, preferably, normalizes executor behavior so `sessionKey` is authoritative when both `sessionKey` and `label` are present. Because PR #63377 is an open closing PR for this issue, the best maintainer path is to review/land or close that PR, or supersede it with a canonical runtime-normalization PR that includes focused regression coverage.
