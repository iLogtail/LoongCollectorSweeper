## Summary

Keep open. Current `main` does not implement the proposed `openclaw auth rotate` CLI or `gateway.auth.tokenExpiryHours`, and the PR discussion still shows concrete auth-safety issues that need maintainer/author follow-up before this can be considered obsolete or safely closed.

  - file: [src/cli/program/subcli-descriptors.ts:7](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/cli/program/subcli-descriptors.ts#L7)

- **Current main lacks the token-expiry feature:** The only related current-main change in this area is `resolveHttpBrowserOriginPolicy` in `src/gateway/http-utils.ts`; there is no `tokenExpiryHours` config, no `token-expiry-state` module, and no expiry-warning path on `main`.
  - file: [src/gateway/http-utils.ts:67](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/gateway/http-utils.ts#L67)
