## Summary

Keep open. Current `main` still does not implement the core `agent_end` / `before_compaction` provenance-preservation change this PR proposes, so there is no conservative cleanup basis to close it. The report remains plausible from the code: provenance is attached on the persistence path, but current hook payloads still come from in-memory snapshots or omit messages entirely in some compaction paths.
