## Summary

Keep open. Current main has partial ACP mitigations, but the reported zero-output hang is not proven fixed: the discussion identifies `nonInteractivePermissions` and `npx` adapter resolution as root causes, and current main still treats both as operator-facing failure/workaround paths rather than a fully implemented fix.

## Best Possible Solution

Keep the issue open until maintainers either land/pin an ACPX release that handles the npx/global-binary path and permission-default behavior, or add OpenClaw-side handling/tests/docs that prove `sessions_spawn(runtime="acp")` produces output or fails with a bounded, actionable error for the reported Claude/OpenCode paths.
