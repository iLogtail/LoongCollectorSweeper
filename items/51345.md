## Summary

Keep open. Current main mitigates the worst 6-hour relay behavior by applying `runTimeoutSeconds` to ACP child turns and dispatching ACP runs on the background subagent lane, and it documents the permission and npx/probe workarounds. But the underlying no-output ACPX startup class is not fully implemented away: OpenClaw still defaults ACPX non-interactive permission prompts to `fail`, adapters may still be fetched with `npx`, and the related acpx PR to prefer global PATH binaries was closed unmerged after maintainer pushback. The issue discussion added useful concrete workarounds, but there is still a real remaining product/runtime reliability question rather than a safe auto-close.
