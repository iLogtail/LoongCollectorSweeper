## Summary

Keep open. Current main still lacks the core behaviors requested by the PR: legacy target aliases/current-context fallback remain, Discord reads are still raw normalized payloads rather than compact/default-limited payloads, and the message tool has no `detail` read parameter. The PR may still contain useful work, but it is not implemented on main and the supplied review context also shows unresolved PR-head concerns that need maintainer/author handling.

## Evidence

- **Current main still normalizes legacy target aliases and infers current context targets:** `normalizeMessageActionInput` deletes legacy aliases only when an explicit `target` exists, otherwise converts legacy `to`/`channelId` into `target`; it also infers `target` from `toolContext.currentChannelId` for target-required actions. That is the behavior this PR proposed to remove/harden.
