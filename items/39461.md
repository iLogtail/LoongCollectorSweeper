## Summary

Keep #39461 open. Current main still hard-codes QQ Bot state storage under the user's home directory and still recommends `QQBOT_DATA_DIR` in diagnostics without implementing that environment-variable override.

## Best Possible Solution

Implement the storage-root override in the QQ Bot plugin, most likely by honoring `QQBOT_DATA_DIR` inside `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()` so existing session, ref-index, credential-backup, diagnostics, and download call sites inherit it. Add focused tests for the env override and update the QQ Bot docs or remove the diagnostic recommendation if maintainers decide not to support the variable.
