## Summary

Keep open. Current main does not implement the PR's central detached `openclaw update` Windows stale-listener recovery behavior: the detached restart path still calls `waitForGatewayHealthyRestart` without `includeUnknownListenersAsStale`, while the underlying restart-health helper and normal `gateway restart` path already support that option. The PR is not merge-ready as-is because it has drifted into unrelated Tlon/config/Matrix/CI changes and has review-thread risk, but the core update recovery gap is still real enough that this should not be auto-closed as implemented or superseded.

## Best Possible Solution

Keep the item open but require a clean, focused update to the PR: rebase on current main, drop unrelated Tlon/config/Matrix/CI/baseline changes, preserve the detached update restart health fix, and add/update focused `src/cli/update-cli.test.ts` coverage for Windows delayed stale-listener fallback call ordering. If the contributor does not refresh it, a maintainer can lift just the small `maybeRestartService` change into a new core CLI PR.
