## Summary

Keep #72125 open. Current `main` still sends normal Feishu replies through `createFeishuReplyDispatcher`, which calls Feishu-specific send helpers directly and does not emit plugin `message_sent` or internal `message:sent` hooks. The linked #72138 PR is the right active fix path, but it is still a separate PR ref and is not present in `main` at `631552c554095b0c520376db9131dac0f5427f94`.

## Best Possible Solution

Keep the issue open until #72138 or an equivalent maintainer-reviewed patch lands. The fix should emit the same canonical plugin `message_sent` and internal `message:sent` events for Feishu normal replies that shared outbound delivery emits, including success/failure, text/card/media/streaming branches, message IDs where available, and routed session/group metadata from both single-agent and broadcast-active paths.
