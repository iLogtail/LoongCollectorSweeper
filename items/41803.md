## Summary

Keep open. On current main at 350d3221800a18283915cd2f3b612ae04ddd490c, `formatAssistantErrorText` still returns raw unrecognized plain-text errors to the user surface, so the PR’s core bugfix is not implemented on main.

## Evidence

- **Raw fallback still returns unrecognized error text:** After all classifiers, `formatAssistantErrorText` only sanitizes HTTP/JSON-style errors via `formatRawAssistantErrorForUi(raw)`. For everything else it still logs long errors and returns the raw string up to 600 chars: `return raw.length > 600 ? `${raw.slice(0, 600)}…` : raw;`. That matches the problem statement rather than the PR’s generic fallback.
