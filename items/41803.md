## Summary

Keep open. Current main still returns raw unrecognized assistant error text from `formatAssistantErrorText`, and that value is pushed into user-facing embedded-run payloads as an error reply. The PR is not obsolete, though the patch should be revised before merge to address the log redaction/sanitization concerns raised in review.

## Best Possible Solution

Keep the PR open and ask for a small refresh against current main: retain the core behavior change so unknown assistant errors no longer reach chat surfaces, add regression coverage for the orphaned `tool_result`/long-string cases plus preserved actionable errors, and fix the logging preview to redact before truncation and sanitize for logs. The existing compaction repair should remain as a root-cause mitigation, but it does not replace the user-surface suppression requested here.
