## Summary

Keep #15828 open. Current main still exposes bundled `session-memory` as a manual `/new` and `/reset` internal hook only. Idle/daily rollover lifecycle hooks and `gateway_stop` plugin hooks exist, and related PRs partially target auto daily/idle rollovers, but current main does not wire those paths to bundled `session-memory`, does not provide shutdown or periodic/message-count checkpoint triggers, and does not implement an LLM summary-output option beyond raw transcript extraction with LLM slug generation.

## Best Possible Solution

Keep #15828 open unless maintainers explicitly split or redirect it. A complete path should extend the bundled `session-memory` hook with configurable idle/shutdown/message-count triggers, dedupe, summary-generation policy, config schema/docs updates, and focused tests. If maintainers prefer a narrower rollout, active related work such as #61675/#71792 can cover daily/idle rollover first, but shutdown, periodic checkpoints, and summary output should remain tracked here or in clearly linked follow-up issues.
