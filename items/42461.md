## Summary

Keep open. Current main still resolves the TUI Gateway URL from config/process env without applying an active runtime/service port override first, so the reported custom-port behavior remains a meaningful missing capability. The provided GitHub context shows related fix PRs, but the current checkout does not contain their requested helper or regression coverage, and the remaining open PR #42496 is not a reason to close the source bug yet.
