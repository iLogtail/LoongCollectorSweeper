## Summary

Keep open. Current `main` still has the lazy-import failure gap this PR is trying to address, but the proposed fix is not safe to merge as written because it would immediately re-render after clearing the cached promise, creating a likely retry loop on persistent import failures.

## Evidence

- **Current main still lacks lazy-import error handling:** `createLazy()` on `main` only attaches `.then(...)` to the dynamic import and has no `.catch(...)` branch, so this PR's core bug report is not already implemented on `main`. The same helper still backs all nine lazy view modules.
  - file: [ui/src/ui/app-render.ts:158](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/ui/src/ui/app-render.ts#L158)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '152,183p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Render path makes the retry-loop review concern credible:** When a lazy getter returns `null`, `lazyRender()` renders `nothing`. Separately, `renderApp()` stores `_pendingUpdate` as the host `requestUpdate` callback. In this UI, calling `_pendingUpdate?.()` from a catch path would schedule another render through the same lazy getter, so clearing `s.promise` first would immediately retry the failing import.
  - file: [ui/src/ui/app-render.ts:214](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/ui/src/ui/app-render.ts#L214)

  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **PR discussion already identified the blocking flaw in the patch:** The PR's review thread flagged that the proposed `.catch()` resets `s.promise` and immediately calls `_pendingUpdate?.()`, which can spin on permanent import failures. A second review comment from the Codex connector independently raised the same issue. That means the patch is not conservatively closable as 'implemented' or 'cannot reproduce'; it still needs maintainer judgment or a revised fix.
