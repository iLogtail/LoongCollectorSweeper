## Summary

Current `main` still implements the reported restriction: cross-agent `sessions_send` is denied unless `tools.sessions.visibility="all"`, and sandboxed label/session resolution still narrows lookups to spawned sessions. This is not implemented on `main`, not obsolete, and an open linked PR already exists for the proposed fix path.

## Evidence

- **Cross-agent send is still gated by visibility before A2A policy:** `createSessionVisibilityChecker()` returns `forbidden` for any cross-agent target when `visibility !== "all"` before it checks whether `tools.agentToAgent` is enabled or allowed. That matches the issue's root-cause report.

  - sha: [c65aa1d2a6e5](https://github.com/openclaw/openclaw/commit/c65aa1d2a6e573d49619c4dd0698a76010476e7d)
- **Sandboxed send resolution still restricts cross-agent lookup:** `createSessionsSendTool()` rejects cross-agent label lookup in sandboxed `restrictToSpawned` mode and passes `spawnedBy` into `sessions.resolve`, which prevents resolving non-spawned cross-agent targets from sandboxed sessions.
