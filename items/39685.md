## Summary

Keep #39685 open. Current main still has the lower-level SSRF hostname allowlist machinery and a Codex-native search allowDomains option, but it does not expose or enforce the requested global/per-agent allowedDomains/denyDomains egress firewall for web_fetch and managed web_search. The prior Codex bot comment reached the same conclusion, and the source has not changed on this surface at current main 6a688e33f66f3965968343922e874cc0e9440998.

## Best Possible Solution

Keep the issue open and implement a core web tool network policy: add typed/schema/docs support for global and per-agent allowedDomains/denyDomains, resolve it consistently like other tool policies, map allowlists into SSRF hostnameAllowlist for web_fetch, add deny precedence, and define enforceable behavior for managed/native web_search providers with focused config and guard tests. If provider-native search cannot be fully enforced, document that limitation explicitly and fail closed where possible.
