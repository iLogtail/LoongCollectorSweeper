## Summary

Current main still does not implement the requested safe fallback for `read(path="*.docx|*.pdf|*.pptx|*.xlsx")`. The OpenClaw wrapper delegates directly to the pinned upstream read tool, the sandbox MIME hook intentionally reports only images, and the upstream dependency still UTF-8 decodes every non-image buffer. The separate `pdf` tool helps PDF workflows but does not guard the generic `read` tool and does not cover Office documents. Related fix PRs are not landed, so #35406 remains actionable.

## Best Possible Solution

Keep #35406 open and land a small core read-tool guard. The likely implementation belongs in `src/agents/pi-tools.read.ts`: detect known binary document extensions/MIMEs before `executeReadWithAdaptivePaging`, return a concise structured fallback that points PDFs to the `pdf` tool when available and tells users DOCX/XLSX/PPTX need a parser/conversion tool, then add host and sandbox regression tests for at least `.docx`, `.xlsx`, `.pptx`, and `.pdf`. PR #57600 remains the closest active implementation vehicle if maintainers want to revive it.
