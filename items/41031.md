## Summary

Keep open. Current main still binds heartbeat model runs to the resolved heartbeat session, and when no forced session or `heartbeat.session` is configured that resolves to the agent main session. Explicit `heartbeat.target` + `heartbeat.to` is used for delivery target resolution, but current main does not derive the run session through outbound routing. The PR is therefore not obsolete; it is a plausible behavior change/fix that needs maintainer judgment, especially because the current docs say `session` controls run context while `target`/`to` control delivery.

## Best Possible Solution

Do not auto-close. A maintainer should decide whether explicit `heartbeat.target` + `heartbeat.to` should also select the heartbeat run session. If yes, rebase/update the PR, keep the outbound-session-route approach, add/keep regression coverage for Telegram and any generic plugin-route behavior, and update heartbeat docs. If no, close manually with a product rationale and point users to `heartbeat.session` for run-context selection.
