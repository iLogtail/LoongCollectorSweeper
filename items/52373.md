## Summary

Keep #52373 open. Current main has adjacent skill metadata support (`emoji`, `homepage`, requirements, install hints) and the Web Dashboard Skills view can show a safe homepage link in the detail dialog, but there is no `configUrl`/`configButton` metadata contract, no status/protocol fields for it, and no Skills-list configure action. The request is recent, concrete, author is not a maintainer, and it has no protected label in the provided context.

## Best Possible Solution

Keep the issue open for maintainer/API work. The likely implementation is an additive skill metadata/status/UI contract: parse `metadata.openclaw.configUrl` and optional `configButton`, validate the URL with the same safe external-link policy, include the fields in the skill status surface consumed by Control UI, render a clear Configure action in the Skills list or detail view, and update Skills docs/tests. If maintainers prefer not to add new fields, document `homepage` as the supported configuration-link workaround instead.
