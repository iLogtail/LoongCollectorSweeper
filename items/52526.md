## Summary

Keep open. Current main still appears to have the reported result-surface mismatch: `agent --json` builds its returned payload text from parsed CLI stdout/output transforms, while `before_message_write` is applied on the transcript persistence path. The issue comments correctly identify the same seam, and I did not find a current CLI regression test or code path that reuses the post-hook persisted assistant message for the JSON result.
