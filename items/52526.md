## Summary

Keep open. Current main still has the reported result-surface split: `before_message_write` can rewrite the persisted transcript, but the `agent --json` output path is still built from run payloads captured before that persistence hook result is used as the authoritative reply text.

## Best Possible Solution

Keep the issue open. The fix should make the JSON result surface use the same finalized assistant message that is persisted after `before_message_write`, either by threading the hook-mutated assistant message back into the run result or by reading the final transcript record before constructing `payloads`. Add a focused regression where a minimal `before_message_write` hook prepends known text and `agent --json` returns that finalized text.
