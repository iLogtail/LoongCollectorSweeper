## Summary

Keep open. Current main still does not accept or dispatch a top-level `threadId` from `POST /hooks/agent`, even though the cron delivery pipeline now has a valid `delivery.threadId` path. The PR is not merge-ready as described, but the underlying feature remains a real gap.

## Best Possible Solution

Keep the item open or ask for a focused replacement patch. The best fix is to rebase onto current main, add `threadId` parsing to `HookAgentPayload`/`normalizeAgentPayload`, include it in hook replay/dedupe scope, populate `CronJob.delivery.threadId` in `dispatchAgentHook` while preserving existing `deliver` semantics, and add targeted gateway/cron tests plus docs for `POST /hooks/agent`. The backup file and unrelated model-selection change should be removed or split out.
