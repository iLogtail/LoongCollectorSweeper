## Summary

Keep PR #69859 open. The underlying feature is still not implemented on current main: `/hooks/agent` still drops inbound `threadId`, even though downstream cron delivery now supports explicit delivery thread IDs. The PR is relevant but not merge-ready as submitted because it carried unrelated changes and a backup artifact, and its original reviewed head was flagged for incomplete propagation.

## Best Possible Solution

Keep the item open, but ask for a revised, focused implementation: add `threadId` to the direct `/hooks/agent` payload normalization and replay scope, propagate it into the hook-created cron delivery target as `delivery.threadId` or the current equivalent delivery-plan path, update the public hook docs, and add gateway/cron regression tests proving a posted Slack thread timestamp reaches outbound delivery. If the contributor does not update the PR, a maintainer replacement PR should reuse the same scope and close PR #69859 afterward as superseded.
