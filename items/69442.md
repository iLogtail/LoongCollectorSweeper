## Summary

Keep open. Current main has adjacent docs-discovery surfaces, including Docs hubs, docs:list, and public llms.txt, but it does not implement the requested generated docs_map.md/docs-map.md with per-page H2/H3/H4 heading hierarchy. The existing repo script emits a flat file list with front-matter summaries/read_when only, and the public docs_map endpoints currently return 404.

## Best Possible Solution

Keep #69442 open and implement it in the docs tooling: add a docs-owned generator that walks docs/**/*.md, parses H2/H3/H4 headings, emits a stable docs_map.md or docs-map.md artifact that Mintlify publishes beside llms.txt, and wire it into the docs change/check workflow. Link it from Docs hubs and/or llms.txt so agents can discover it without guessing.
