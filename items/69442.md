## Summary

Keep open. Current main has useful docs navigation aids, but it does not implement the requested public, auto-generated `docs_map.md` sibling with H2/H3/H4 heading hierarchy. The issue is recent, concrete, contributor-authored, and has no protected label or maintainer-authored auto-close blocker.

## Best Possible Solution

Keep #69442 open and implement a generated docs map if maintainers want this product surface: add a deterministic script that walks `docs/**/*.md(x)`, extracts H2/H3/H4 headings plus routes, writes a public `docs_map.md` or equivalent publish artifact, and wire it into docs checks or the docs sync workflow so changes under `docs/**` cannot drift.
