## Summary

Keep open. Current main has a partial hardcoded mitigation for malformed built-in file-tool arguments, but it does not implement the requested `tools.validate_before_call` / `tools.retry_on_missing_params` config surface, does not do generic JSON-schema-required validation for all tools, and still returns/logs validation failures as ordinary tool errors rather than doing an automatic hidden retry.

## Best Possible Solution

Keep the issue open as a focused core feature request. A complete implementation would add documented and schema-validated `tools.*` config fields, wire generic JSON-schema-required validation before tool handler dispatch across core/plugin/client tools where safe, preserve existing per-tool semantic validation, and design any missing-parameter retry behavior in the model loop so it cannot hide real tool errors or create retry loops.
