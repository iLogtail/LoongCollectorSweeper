## Summary

Keep #69978 open. Current main still has the TUI streaming watchdog as an injected handler option with a 30s default, but there is no user-facing `/watchdog` command or persisted setting, and the handler still has no per-run warned set to suppress duplicate watchdog messages if the same run emits more deltas after a watchdog reset.

## Best Possible Solution

Keep the issue open. The likely maintainer path is to implement duplicate suppression in `src/tui/tui-event-handlers.ts` with a per-run warned set cleared on finalization/termination/session change, then add a TUI-level `/watchdog` command in `src/tui/commands.ts` and `src/tui/tui-command-handlers.ts` that can show, disable, and set the timeout. If maintainers choose config-file rather than TUI-only persistence, coordinate the timeout-setting portion with open #68596 while still tracking the duplicate-warning suppression here.
