## Summary

Keep #69978 open. Current main still has a 30s TUI streaming watchdog with only an internal test/context override, still emits the watchdog system warning when the timer expires, and has no `/watchdog` command, persisted TUI setting, or documented user-facing control. Related #68596 and PR #69026 remain open/unmerged and do not fully cover both requested pieces.

## Best Possible Solution

Keep this open unless maintainers intentionally close it into a canonical watchdog UX item. The product/code path should either implement a TUI-owned watchdog setting and `/watchdog` command with persistence, or explicitly consolidate that runtime-control request into #68596, while also adding per-run warning suppression or a single softer warning and updating TUI command docs/tests.
