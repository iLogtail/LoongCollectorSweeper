## Summary

Keep open. Current main still emits the embedded-runner `llm_output` hook without checking that an assistant message exists, which leaves the PR's core bug unresolved. The PR is still useful directionally, but it also bundles an unrelated `chat-content` parsing change with unresolved review concerns, so it should not be auto-closed or treated as cleanly superseded.

## Best Possible Solution

Keep the PR open, but ask for a narrow embedded-runner fix: guard `llm_output` emission when no assistant output exists, preserve normal successful `llm_output` behavior, and add focused regression coverage for the timeout/no-output path. The unrelated `chat-content` JSON-string parsing change should be removed, split into a separate PR, or fixed with tests for mixed content and `output_text` before this PR is merge-ready.
