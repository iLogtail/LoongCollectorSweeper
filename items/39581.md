## Summary

Keep PR #39581 open. Current main still emits the embedded-runner `llm_output` hook whenever hooks are registered, without checking that the current attempt produced assistant output, and it still passes `lastAssistant` from the full message history rather than the current attempt boundary. The PR addresses a real remaining issue, but should be narrowed because its branch also includes unrelated `chat-content` JSON parsing changes with unresolved review feedback.

## Best Possible Solution

Keep the PR open, but request a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale full-history `lastAssistant`, preserve normal-output hook emission, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
