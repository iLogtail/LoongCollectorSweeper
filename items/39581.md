## Summary

Keep PR #39581 open. Current main still has the embedded-runner `llm_output` bug: the hook fires whenever it is registered, even when the current attempt produced no assistant output, and it passes a `lastAssistant` value that can be undefined or stale from earlier session history. The PR addresses a real remaining issue, but it should be narrowed because the branch also contains unrelated `chat-content` JSON parsing changes with unresolved review feedback.

## Best Possible Solution

Keep the PR open, but request a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale `lastAssistant`, preserve normal-output hook emission, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.

## Evidence

There is no guard for current-attempt assistant output on current main.
