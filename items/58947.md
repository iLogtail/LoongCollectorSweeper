## Summary

Keep open. Current main still defaults `tools.fs.workspaceOnly` to unrestricted behavior unless the config is explicitly `true`; tests and docs continue to lock/document that default. The related #42207 PR is an open, incomplete implementation attempt rather than a merged or shipped resolution, so this proposal still needs maintainer/product handling.

## Best Possible Solution

Keep this issue open as the product/security tracking item. The best path is for maintainers to decide whether the default should be global `true` or profile-based, then update `src/agents/tool-fs-policy.ts`, host read/write/edit wiring, config types/help/schema, security/config docs, and tests together. Coordinate with #42207 or replace it with a maintainer-owned patch; close only after the behavior is merged or a canonical follow-up explicitly supersedes this proposal.
