## Summary

Keep open. Current `main` has added standalone OTEL spans for runs, model calls, and tool executions, and it carries diagnostic trace context through the runner, but `diagnostics-otel` still exports `openclaw.message.processed` as a separate coarse span and explicitly does not parent diagnostic-event spans into a child-span tree. The issue's core request for child spans under message processing is not implemented yet.

## Evidence
