## Summary

Current `main` already addresses both reported heartbeat isolated-session bugs, and it does so with a safer implementation than this PR's original regex plus dedicated regression tests for the nesting and transcript-reset cases.

## Evidence

- **Safer isolated-key canonicalization is on main:** `main` now routes isolated heartbeat session keys through `resolveIsolatedHeartbeatSessionKey`, which collapses only repeated terminal `:heartbeat` suffixes and preserves legitimate configured base keys that already end with `:heartbeat`. That directly covers the nesting bug while addressing the overbroad-regex concern raised in this PR review.
