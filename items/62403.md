## Summary

Keep PR #62403 open. Current checked-out main still renders the Sessions table rows positionally with `paginated.flatMap(...)`, while the label input patches a row key and the default updated sort can reorder rows after a session update. The PR's keyed-row fix and added browser regression test are not present on main, and the existing sessions tests do not cover the label-edit reorder case.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a scoped keyed-rendering fix for the Sessions table plus the browser regression test, ideally adjusting the PR so each keyed `repeat()` item returns one stable template that includes both the main row and optional expanded checkpoint row.
