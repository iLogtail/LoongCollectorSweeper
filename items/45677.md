## Summary

Keep open. Current `main` still drops `RuntimeLogger` metadata in `createRuntimeLogging`, so this is not implemented or obsolete. The underlying core logging path also treats structured fields as object-first, which means the PR's current `(message, meta)` passthrough likely needs revision before merge rather than auto-close.
