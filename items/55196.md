## Summary

Keep open. Current `main` does not implement a `brave.http` diagnostics flag or equivalent targeted logging for Brave `web_search` requests, responses, or cache hit/miss behavior.

## Evidence

- **No Brave diagnostics flag in runtime:** The Brave web-search runtime builds outbound Brave URLs and calls `withTrustedWebSearchEndpoint(...)`, but there is no `isDiagnosticFlagEnabled(...)`, subsystem logger, or request/response logging in the Brave path.
  - file: [extensions/brave/src/brave-web-search-provider.runtime.ts:77](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/brave/src/brave-web-search-provider.runtime.ts#L77)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Cache path is silent:** Brave cache lookup and cache write happen without any diagnostic logging. A cache hit returns immediately from `readCachedSearchPayload(cacheKey)`, and writes use `writeCachedSearchPayload(...)` with no hit/miss log surface.
  - file: [extensions/brave/src/brave-web-search-provider.runtime.ts:286](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/brave/src/brave-web-search-provider.runtime.ts#L286)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Requested pattern exists for Telegram only:** Telegram has the exact targeted flag pattern the issue references: `isDiagnosticFlagEnabled("telegram.http", cfg)` gates a dedicated HTTP diagnostic logger. No analogous Brave implementation was found.
  - file: [extensions/telegram/src/send.ts:202](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/telegram/src/send.ts#L202)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Docs/examples do not mention brave.http:** Diagnostics flag docs describe subsystem-owned opt-in flags and use `telegram.http` as the concrete example. No `brave.http` example or Brave-specific flag documentation is present.
  - file: [docs/diagnostics/flags.md:9](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/docs/diagnostics/flags.md#L9)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Repo-wide search found no implementation:** A repo search for `brave.http` and Brave-specific diagnostics/logging hooks returned no matches beyond the issue context; the Brave extension files only show request execution and cache plumbing.
  - command: `rg -n "brave\.http|diagnostic|diagnostics|createSubsystemLogger|logger\.|cache hit|cache miss" extensions/brave src/agents/tools src/plugin-sdk`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
