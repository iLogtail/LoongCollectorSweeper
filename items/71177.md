## Summary

Keep open. Current main has useful mitigations, especially opt-in `isolatedSession` and `lightContext`, but non-isolated heartbeat runs still intentionally reuse the existing heartbeat/main session and there is no heartbeat `sessionRotation` policy with turn, age, token, or consecutive-error thresholds. The related #71031 closure may address one poisoning trigger, and #45390 remains the broader session TTL tracker, but this heartbeat-specific lifecycle request is not implemented or fully superseded.

## Best Possible Solution

Keep this open or explicitly consolidate it into #45390 after maintainer review. The concrete core path is a documented heartbeat lifecycle policy under `agents.defaults.heartbeat` or an intentional default change to isolated heartbeat sessions, with schema validation, archive/summary behavior, health/error thresholds, tests for non-isolated heartbeat rollover, and docs explaining interaction with `isolatedSession`, `lightContext`, daily/idle reset, and the related #71031 session-memory poisoning fix.
