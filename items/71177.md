## Summary

Keep open. Current `main` has an opt-in mitigation via `heartbeat.isolatedSession` that runs each heartbeat in a fresh session and archives stale heartbeat transcripts, but I did not find the requested automatic rotation/TTL/error-threshold behavior or matching config knobs on current `main`.

## Evidence

- **Heartbeat config exposes opt-in isolated sessions, not rotation thresholds:** `agents.defaults.heartbeat` includes `isolatedSession?: boolean` with docs describing fresh-session heartbeats and lower token cost, but no `sessionRotation`, turn-count, age, token-percent, or consecutive-error fields.
  - file: [src/config/types.agent-defaults.ts:372](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/config/types.agent-defaults.ts#L372)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '320,430p'`
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Heartbeat runner can start a fresh session per run and archive the previous isolated transcript:** When `heartbeat.isolatedSession === true`, `runHeartbeatOnce` creates a new cron-style session with `forceNew: true`, rewrites the store entry under a `:heartbeat` key, and archives any replaced isolated transcript.
  - file: [src/infra/heartbeat-runner.ts:796](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/infra/heartbeat-runner.ts#L796)
