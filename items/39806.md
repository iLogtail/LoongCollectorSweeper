## Summary

Keep PR #39806 open. Current main still has the Debug Snapshots UX gap from #39777: disconnected refresh returns before any visible state is set, and the Snapshots card body only renders the Status, Health, and Last heartbeat blocks while the button label changes during loading. The PR is a focused attempt to fix a real remaining Control UI bug, not conservative cleanup material.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused fix that passes connection state or a refresh-disabled reason into renderDebug, shows a body-level loading/progress state while debugLoading is true, shows a clear disconnected callout when refresh cannot run, preserves the existing Status/Health/Last heartbeat layout, and closes #39777 once landed.
