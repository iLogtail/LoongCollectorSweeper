## Summary

Keep #39806 open. Current main still has the Debug Snapshots UX gap described by #39777: `loadDebug()` silently returns when disconnected, and the Snapshots card only changes the Refresh button text while loading. The PR remains useful in scope, but the provided review discussion identifies a concrete markup/layout regression in the PR version, so it needs follow-up rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR or its linked issue open. The best path is a small UI follow-up that preserves the existing Status/Health/Last heartbeat stack, passes enough connection state into `renderDebug()` to show an offline callout, renders an in-card loading state while `debugLoading` is true, and adds focused view/controller coverage if the existing UI test setup can support it.
