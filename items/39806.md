## Summary

Keep PR #39806 open. Current main still has the Debug Snapshots UX gap described by #39777: `loadDebug` returns before setting a loading state when disconnected, and the Snapshots card only changes the Refresh button text while rendering the Status/Health/Last heartbeat blocks with no in-card loading or disconnected state.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused fix that adds body-level loading and disconnected states to the Debug Snapshots card, passes or derives connection state cleanly, preserves the existing Status/Health/Last heartbeat layout, and closes #39777 once landed.
