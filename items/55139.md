## Summary

Keep #55139 open. Current main still has only the older fallback guard: `resolveConsoleSettings()` calls `readLoggingConfig()` before `loggingState.resolvingConsoleSettings` is set, and the console `forward()` path still calls `getConsoleSettings()` while formatting timestamped output. The PR's final commit directly addresses the residual path raised in review, so the work is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and either rebase/finish it or apply the same focused fix on main: set `loggingState.resolvingConsoleSettings` before the initial `readLoggingConfig()` call, keep it covering `loadConfigFallback()`, add a pre-`getConsoleSettings()` safe path in console forwarding for warnings emitted during resolution, and add a regression test for warnings from the initial logging-config read path.
