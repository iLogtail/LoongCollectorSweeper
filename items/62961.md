## Summary

The requested task-tier model routing fits OpenClaw's existing plugin/skill extension points rather than core. Current main already exposes prompt-time model override hooks, ACP spawn model overrides, and bundled ACP routing skill ownership, so this is better handled as a ClawHub/plugin feature.

## Evidence

## Risks / Open Questions

- No standalone public ClawHub package implementing this exact task-tier classifier was verified from the issue thread; the strongest ownership proof is the existing extension points plus the bundled ACPX routing skill.
- If maintainers later want a built-in default classifier for UX reasons, that would be a new product decision rather than evidence that current main lacks the necessary extension seam.
