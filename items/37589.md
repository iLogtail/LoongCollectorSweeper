## Summary

Keep #37589 open. Current main still recreates every text part and returns a new message object for array-backed content in `stripInlineDirectiveTagsFromMessageForDisplay`, even when `stripInlineDirectiveTagsForDisplay` reports no change. The related PRs in the provided GitHub context are closed and unmerged, and the existing tests still do not assert reference identity preservation.

## Best Possible Solution

Implement the narrow identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add focused Vitest coverage with `toBe` assertions for unchanged array messages and mixed changed/unchanged text parts. If the helper is obsolete, remove or de-export it deliberately and update tests to cover the current projection API instead.
