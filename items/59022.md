## Summary

Keep open. Current main has a Gateway A2UI route and expected auth/capability protection, so the direct `curl` returning `Unauthorized` is not itself proof that the host is down. However, the reported macOS node failure is still actionable: the Gateway now has a scoped canvas capability refresh method and Android/iOS A2UI paths retry with a fresh capability, but the macOS node A2UI path still navigates once and returns `A2UI_HOST_UNAVAILABLE` without refreshing the scoped capability.

## Best Possible Solution

Keep the issue open and implement macOS parity with iOS/Android: when macOS A2UI readiness fails or the cached scoped canvas URL is stale, request `node.canvas.capability.refresh`, update the cached/derived canvas host URL, navigate to the refreshed A2UI URL, and retry before returning `A2UI_HOST_UNAVAILABLE`. Add macOS-side coverage for stale scoped capability retry, and optionally update user docs/troubleshooting to say direct `/__openclaw__/a2ui` returns 401 without bearer auth or a node-scoped capability URL.
