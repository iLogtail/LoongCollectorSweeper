## Summary

Do not auto-close #59022. Current main clearly has the Gateway-side A2UI host and node-scoped capability support, and the reported unauthenticated curl returning 401 is now documented/tested as expected. However, the reported failing surface is a macOS node `canvas.a2ui.*` invoke, and current macOS node code still lacks the capability-refresh retry path that iOS/Android use for the same `A2UI_HOST_UNAVAILABLE` failure mode. Without live macOS proof or a macOS regression test, this is not strong enough for `implemented_on_main` or `cannot_reproduce`.
