## Summary

Current `main` still has the reported CLI/profile path mismatch when `OPENCLAW_HOME` is set, and this PR remains the concrete fix rather than something already implemented upstream.

## Evidence

- **CLI still appends profile suffix on main:** `resolveProfileStateDir` in current `main` always returns `.openclaw${suffix}` for non-default profiles and has no `OPENCLAW_HOME` special case, so `--profile work` still resolves to `<home>/.openclaw-work` even when `OPENCLAW_HOME` is set.
  - file: [src/cli/profile.ts:63](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/cli/profile.ts#L63)
  - command: `nl -ba src/cli/profile.ts | sed -n '1,220p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Daemon/config side uses unsuffixed .openclaw under OPENCLAW_HOME:** Current config path resolution derives the default state dir from `newStateDir(homedir)`, which is always `<home>/.openclaw`; tests explicitly assert `OPENCLAW_HOME` defaults to `.openclaw/openclaw.json`. That matches the PR description's daemon-side behavior and conflicts with the current CLI profile logic.
  - file: [src/config/paths.ts:39](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/config/paths.ts#L39)
  - command: `nl -ba src/config/paths.ts | sed -n '1,110p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **Current tests lock in the mismatch:** `src/cli/profile.test.ts` on `main` still expects `OPENCLAW_HOME + profile=work` to resolve to `.openclaw-work`, while config tests expect `OPENCLAW_HOME` defaults to `.openclaw`. The mismatch is therefore still present in both implementation and test expectations on `main`.
  - file: [src/cli/profile.test.ts:126](https://github.com/openclaw/openclaw/blob/cf858258c71701ef62dddf88aa777873a4962046/src/cli/profile.test.ts#L126)
  - command: `nl -ba src/cli/profile.test.ts | sed -n '100,220p'`
  - sha: [cf858258c717](https://github.com/openclaw/openclaw/commit/cf858258c71701ef62dddf88aa777873a4962046)
- **PR discussion improved the proposed fix:** The review discussion usefully identified two follow-ups on the PR branch: normalize sentinel `OPENCLAW_HOME` values like `"undefined"`, and cover `dev` profile behavior under `OPENCLAW_HOME`. The provided PR context shows a second commit (`16d0dec468d4eede8d4eb1a635971956a5ae0bb6`) addressing the normalization concern and adding the missing tests, which strengthens the case for keeping the PR open rather than closing it as obsolete.
  - sha: [16d0dec468d4](https://github.com/openclaw/openclaw/commit/16d0dec468d4eede8d4eb1a635971956a5ae0bb6)
