## Summary

Current shipped code already handles this Talk Mode failure mode conservatively: macOS Talk Mode reads the assistant reply from chat history instead of provider-specific streaming, resumes listening if no reply arrives, and falls back to system voice if ElevenLabs playback fails. The shared chat client also now accepts canonical `agent:main:main` completion events and refreshes history instead of getting stuck on a `main`/canonical session-key mismatch.
