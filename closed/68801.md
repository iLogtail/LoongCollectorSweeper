## Summary

Keep open. Current main still has the reported `agentRunStarts` retention behavior, so the PR is not implemented or superseded. The proposed one-line TTL sweep is plausible but needs maintainer-safe revision because `agentRunStarts` is also the fallback source for `startedAt` when terminal lifecycle events omit it, and the documented agent runtime can exceed the 10-minute cache TTL.

## Best Possible Solution

Keep the PR open, but revise it before merge. The safest path is a targeted `agent-job.ts` cleanup that distinguishes orphaned starts from active long-running runs, plus tests for both cases: lost terminal event entries eventually drain, and an active run lasting longer than `AGENT_RUN_CACHE_TTL_MS` still returns `startedAt` when its terminal event omits that field. If that distinction cannot be made with current state, maintainers should choose an explicit lifecycle/active-run tracking contract rather than applying the terminal snapshot cache TTL to active start timestamps.
