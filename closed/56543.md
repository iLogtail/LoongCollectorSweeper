## Summary

Current main already covers the reported stale macOS gateway service drift: package-manager updates refresh the loaded gateway service from the updated install root, doctor detects mismatched service entrypoints and repairs/stages them, and macOS doctor/bootstrap repair handles listed-but-unloaded LaunchAgents.

## Risks / Open Questions

- Manual package-manager swaps done outside the supported `openclaw update`/installer flows can still bypass the auto-refresh path.
- A service that intentionally keeps a version-manager Node/runtime may still surface advisory warnings until it is migrated to a supported system Node.
