## Summary

本 Issue 建议修改 sls_logs.proto 以支持毫秒时间戳。经社区成员确认，LoongCollector 当前版本已统一时间戳精度（含纳秒/毫秒）的配置与处理能力，可通过现有 Processor 插件灵活完成解析与转换，无需变更底层 Proto 定义。因 Issue 提出已超 60 天且未补充详细预期输出与复现路径，结合主线已具备对应能力，予以关闭。

## Best Possible Solution

使用现有 timestamp processor 插件配置时间精度；若需特殊处理可基于插件机制扩展或在新 Issue 中提供完整复现数据。
