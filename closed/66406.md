## Summary

Keep open. Current main still passes the message tool's raw `emoji` string directly into Feishu reaction API `emoji_type`, and the linked fix PR #66478 is open/unmerged rather than shipped.

## Best Possible Solution

Keep #66406 open until a Feishu plugin fix lands. The likely path is to refresh or replace PR #66478 with a tested normalizer that maps common Unicode emoji inputs like 👍 to Feishu reaction type strings such as THUMBSUP before add/remove/list reaction API calls, while preserving explicit Feishu type strings.

## Evidence

A call such as `emoji="👍"` would therefore still reach the API as the raw Unicode character.
