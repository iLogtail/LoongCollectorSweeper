## Summary

Keep #69084 open. Current main has not implemented the PR's requested `showAlerts=false` webchat live-output behavior, but the PR's one-line `showOk` to `showAlerts` swap is not clearly safe to merge as-is because heartbeat OK acknowledgments and alert content are separate visibility contracts.

## Best Possible Solution

Keep the PR open for maintainer review or contributor changes. The best fix is a narrower server-chat/webchat visibility implementation that treats heartbeat OK acknowledgments and alert content separately: preserve `showOk` for OK-only acknowledgments, honor `showAlerts=false` for non-OK alert text, and add focused tests for default, OK-only, alert-disabled, and fully silent configurations. If maintainers prefer the broader event-audience design, link this PR to that canonical work but do not close it as already implemented.
