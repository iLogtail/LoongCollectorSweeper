## Summary

#71657 is a duplicate of the older open canonical issue #70628. The current main branch still contains the same Telegram direct-message no-visible-response path: it synthesizes `NO_REPLY`, applies the direct silent-reply rewrite policy, and has a regression test that currently asserts the visible rewritten message behavior reported here.

## Best Possible Solution

Close #71657 as a duplicate and keep #70628 as the canonical open bug. The fix should land against #70628 by changing the Telegram plugin dispatch path so direct-message no-visible-response turns without an actual dispatch or delivery error stay silent, while preserving real error fallback behavior and updating `extensions/telegram/src/bot-message-dispatch.test.ts` accordingly.
