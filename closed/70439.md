## Summary

Keep open. Current main partially addresses the related Signal metadata problem, so accepted Signal voice attachments can now reach the shared media-understanding path, but it does not implement the requested Signal/channel-agnostic audio preflight hook. Signal still performs group mention gating before downloading/transcribing attachments, has no Signal `transcribeFirstAudio` call, and exposes no `channels.signal.audio` or external preflight command config.

## Best Possible Solution

Keep this issue open and implement the remaining behavior in the Signal channel using the existing shared audio preflight contract: fetch/resolve eligible audio for accepted senders before mention gating when needed, call `transcribeFirstAudio`, use the transcript for mention detection and agent-facing body replacement, and mark the transcribed attachment so the later media-understanding pass does not repeat STT. Prefer a shared channel helper over a Signal-only shell command; if a configurable external hook is added, document Signal template variables and non-WAV conversion requirements.
