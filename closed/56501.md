## Summary

Current `main` already persists provider auth failures into auth-profile unavailable state and the model-fallback loop skips providers whose profiles are unavailable for `auth`/`auth_permanent`, which addresses the reported Moonshot 401 fallback loop.

## Evidence

- **Embedded runner persists auth failures:** Non-timeout failover reasons are converted into auth-profile failure state and written via `markAuthProfileFailure`, so a 401-classified auth failure is not treated as a one-off transient.
  - file: [src/agents/pi-embedded-runner/run.ts:633](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-embedded-runner/run.ts#L633)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '633,663p'`
- **Auth failures become durable unavailable state:** `markAuthProfileFailure` records failure counts and cooldown/disable windows; billing and `auth_permanent` use the disabled lane, while `auth` still records a profile-wide unavailable cooldown.
  - file: [src/agents/auth-profiles/usage.ts:559](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/auth-profiles/usage.ts#L559)
  - command: `nl -ba src/agents/auth-profiles/usage.ts | sed -n '559,626p'`
- **Fallback skips auth-broken providers:** When all profiles for a candidate provider are unavailable and the inferred reason is `auth` or `auth_permanent`, fallback returns `skip` for that provider instead of probing it again.
  - file: [src/agents/model-fallback.ts:663](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/model-fallback.ts#L663)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '663,675p'`
- **Skipped providers are removed from the current fallback pass:** The fallback loop records a `skip_candidate` attempt and continues to the next provider/model candidate, preventing immediate switch-back to the failing provider in the same pass.
  - file: [src/agents/model-fallback.ts:780](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/model-fallback.ts#L780)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '780,803p'`
- **Release tag already contains the auth-skip logic:** The `v2026.4.23` tag contains the same `auth`/`auth_permanent` skip logic in `src/agents/model-fallback.ts`, so this is not only on current `main`.
  - file: [src/agents/model-fallback.ts](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/agents/model-fallback.ts)
  - command: `git show v2026.4.23:src/agents/model-fallback.ts | rg -n "auth_permanent|isPersistentAuthIssue|skipping all models"`
