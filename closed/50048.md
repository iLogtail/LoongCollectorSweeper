## Summary

Current `main` already fixes the reported Discord/Jiti runtime-import problem by loading source `*.runtime.ts` plugin entries through non-native Jiti, so the explicit `.js`→`.ts` import rewrites from PR #50048 are no longer needed.

## Close Comment

Closing from a Codex review: current `main` already fixes this class of Discord runtime failure. The relevant change is in `src/plugins/loader.ts:442`, where source `*.runtime.ts` plugin entries are forced onto the non-native Jiti path specifically because those shims import sibling `.js` specifiers that only exist after build. The Discord files in this PR still use `.js` imports on `main` (`extensions/discord/src/channel.runtime.ts:3` and `extensions/discord/src/voice/manager.runtime.ts:4`), but they now load correctly through that loader behavior, and the expectation is covered by `src/plugins/sdk-alias.test.ts:843` and `src/plugins/sdk-alias.test.ts:1056`. This shipped in `v2026.4.22` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, so there is no remaining maintainer action for this PR.
