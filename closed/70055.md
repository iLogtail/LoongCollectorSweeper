## Summary

Keep #70055 open. Current main partially reduces the reported noise by moving external CLI bootstrap decisions to debug logs and by keeping `models list/status` on no-external-profile store helpers, but it still does not provide the requested config, env, or CLI-level opt-out for external CLI auth overlays. The remaining request is core auth/config behavior, not a stale or ClawHub-only item.

## Best Possible Solution

Add a first-class core auth configuration surface, preferably `auth.externalCliSync` with boolean plus optional provider include/exclude semantics, and thread it through the runtime auth overlay path in `store.ts`/`external-auth.ts`. Keep default-on behavior for compatibility, update the generated config schema and auth docs, and add focused tests proving disabled config prevents Codex/Claude/MiniMax external CLI profile reads and overlays while preserving existing runtime-only behavior when enabled.
