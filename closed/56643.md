## Summary

Current main already implements interval heartbeats as a per-agent scheduler and defaults those runs to the agent main session, which addresses the reported per-session duplicate HEARTBEAT_OK behavior.

## Close Comment

Closing from a Codex review. Current `main` already implements interval heartbeats as a per-agent scheduler rather than a per-session loop: the runner builds one heartbeat state per agent in `src/infra/heartbeat-runner.ts:202`, the interval timer fires without any session key in `src/infra/heartbeat-runner.ts:1390`, and the interval path iterates `state.agents.values()` in `src/infra/heartbeat-runner.ts:1513`. If you still hit duplicate interval heartbeats on a newer build, please open a fresh issue with the exact version, effective heartbeat config, and a minimal session inventory/log excerpt.
