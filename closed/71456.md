## Summary

Keep open. Current `main` still has the reported stop-timeout/no-op restart behavior, but the PR as proposed is not safe to auto-close or land as-is because stale callbacks from the timed-out task can still overwrite runtime state for a replacement task.

## Evidence

That matches the stuck-runtime behavior described in the PR body.
