## Summary

Current `main` still treats persisted `"type": "aws-sdk"` auth-profile entries as invalid during store load, and the only linked fix path (#69753) was closed rather than merged. There is Bedrock fallback coverage for empty auth stores, but there is no shipped fix or regression coverage for persisted aws-sdk markers, so this item is not safe to close.

## Evidence

- **Persisted auth-profile parser still rejects aws-sdk:** `AUTH_PROFILE_TYPES` only allows `api_key`, `oauth`, and `token`, and `parseCredentialEntry(...)` returns `invalid_type` for anything else. On current `main`, that still matches the issue's reported warning path for persisted `"type": "aws-sdk"` entries.
