## Summary

Current main does not implement `before_persistence_write` or an equivalent durable-state write hook. The public hook registry and docs expose `before_tool_call`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; the existing tool hook lacks write-target/session-boundary metadata, and the persistence hooks are scoped to transcript/message persistence. This is a core plugin API gap rather than ClawHub-only work, and related provenance/isolation items remain open rather than superseding it.
