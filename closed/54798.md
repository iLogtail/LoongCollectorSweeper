## Summary

Keep open. Current main has job-level cron overlap guards and configurable cron/subagent concurrency, but it does not provide the requested shared atomic counter/session KV increment, named mutex/exclusive sessions_spawn mode, or agent-facing advisory file-lock primitive. The linked #54325-style skip-running behavior is present only as per-job reservation/skip logic and does not solve shared counters across distinct cron jobs or concurrent sub-agents.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The narrowest implementation path would be a documented core or SDK-backed atomic workspace/session state primitive, such as a small session KV incr/compare-and-set command and agent tool, or a named mutex/exclusive option on sessions_spawn if that is the desired control plane. If maintainers prefer ClawHub, the repo should first bless the plugin-facing contract and document how such a plugin gets enabled for cron sub-agents; the existing plugin-sdk file-lock helper alone does not satisfy the reported built-in cron/sub-agent workflow.
