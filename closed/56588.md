## Summary

Current `main` already addresses this report: the Discord gateway plugin now uses `reconnect.maxAttempts = 50`, reconnect-exhausted errors are classified and routed through the Discord lifecycle supervisor instead of escaping as uncaught process-killing exceptions, and regression tests cover both the exhausted-reconnect path and late teardown errors. The released tag `v2026.4.23` contains the same implementation.
