## Summary

Keep open. Current `main` contains a partial orphaned-user repair, but it does not implement the PR's narrower goal of recovering multiple orphaned trailing turns while preserving legitimate repeats and deduping against the raw prompt/hooks path.

## Evidence

- **Current runner only repairs one leaf:** `runEmbeddedAttempt` reads exactly one trailing leaf via `sessionManager.getLeafEntry()`, merges that single `leafMessage`, and branches/reset once. There is no loop or chain walk to recover multiple orphaned trailing user turns.

  - sha: [bc73141e8284](https://github.com/openclaw/openclaw/commit/bc73141e8284c219933970dbd4f6aeacb05bbb77)
- **Main lacks raw-prompt-aware dedupe and can collapse real repeats:** The active merge strategy only accepts `{ prompt, trigger, leafMessage }`; there is no `rawPrompt` input. Its duplicate check returns true when the orphan text appears anywhere in the normalized prompt (`includes` on the orphan text), which is broader than the PR's 'drop only the newest prompt echo' behavior and can still erase legitimate repeated turns.
