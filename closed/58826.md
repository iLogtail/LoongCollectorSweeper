## Summary

Keep open. Current `main` has adjacent safeguards and observability, but not the requested built-in token/request budget enforcement at per-request, per-session, or per-day scope. The issue discussion is still actionable: loop detection is now available, and rate-limit fallback exists, but operator-defined spend/request caps are not implemented.

## Evidence

- **Usage/quota is observability only:** The usage-tracking docs describe provider usage/quota display surfaces (`/status`, `/usage`, `openclaw status --usage`) and explicitly frame them as provider-reported windows/cost summaries, not enforcement controls.
  - file: [docs/concepts/usage-tracking.md:11](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/concepts/usage-tracking.md#L11)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Loop detection exists, but it only covers repetitive tool-call patterns:** `tools.loopDetection` is a configurable guard for repeated tool-call loops. It is disabled by default and does not provide request/session/day token caps.
  - file: [docs/gateway/config-tools.md:114](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/gateway/config-tools.md#L114)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Core runtime blocks critical tool loops:** Before tool execution, OpenClaw detects stuck tool-call patterns and blocks critical loops. This partially addresses the issue's safety-mechanism ask, but only for loop patterns.
  - file: [src/agents/pi-tools.before-tool-call.ts:192](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/pi-tools.before-tool-call.ts#L192)

- **There is an always-on unknown-tool loop safety net:** The embedded runner keeps an unknown-tool guard on even when general loop detection is disabled, preventing infinite tool-not-found loops. This is still narrower than requested token/request budgets.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:336](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/pi-embedded-runner/run/attempt.ts#L336)

## Risks / Open Questions

- Future reviewers should not mistake `contextTokens` or provider usage dashboards for spend/request budget enforcement; they solve different problems.
- The issue now overlaps existing loop-detection and fallback features, so any eventual implementation should be scoped tightly around operator-defined quota/limit enforcement.
