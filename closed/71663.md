## Summary

Close #71663 as a duplicate of open #71575. Both issues report Telegram exposing Working/tool/progress output during tool execution, and current main still has Telegram preview tool progress enabled by default with tests asserting visible `tool: exec` and command text in partial previews.

## Best Possible Solution

Keep #71575 open as the canonical Telegram bug. The likely product fix is to make Telegram partial preview hide or strictly sanitize tool metadata and raw output by default, possibly by making `streaming.preview.toolProgress` opt-in or generic-only, then update Telegram dispatch tests and docs to match.
