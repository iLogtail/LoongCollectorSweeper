## Summary

Keep #70882 open. Current main still does not implement the PR's schema-aware coercion for stringified object/array properties before bundle MCP tool calls, and the linked #70872 report has a concrete current product path: embedded Pi exposes configured MCP tools and forwards calls through the bundle MCP runtime. The PR remains potentially useful, but should be revised before merge to address the bot-reviewed safety and schema edge cases.

## Best Possible Solution

Keep the PR open and have the author or maintainer revise it into a safe bundle MCP argument-normalization fix in the materialize/runtime path. The best patch should coerce only schema-declared object/array properties, avoid prototype-key pollution, bound JSON parsing, handle JSON Schema string-or-array `type`, and add focused regression tests for #70872.
