## Summary

Current main already contains the OpenRouter streaming fix for reasoning-details-only replies: OpenRouter-compatible completions now treat `reasoning_details.response.output_text` / `response.text` as visible assistant output, and regression tests cover the previously empty-turn path. The fix is present on unreleased main, not in the latest shipped tag.

## Evidence

- **Changelog entry for this exact OpenRouter fix:** The unreleased `2026.4.24` changelog explicitly says OpenRouter-compatible streaming now treats `reasoning_details.response.output_text` and `reasoning_details.response.text` as visible assistant output, matching the issue discussion about replies living in reasoning details while `content` stayed empty.

- **OpenRouter compat marks visible reasoning detail types:** Current main sets `visibleReasoningDetailTypes` to `['response.output_text', 'response.text']` for OpenRouter-like completions, which is the compatibility switch needed to surface these replies as normal assistant text.
